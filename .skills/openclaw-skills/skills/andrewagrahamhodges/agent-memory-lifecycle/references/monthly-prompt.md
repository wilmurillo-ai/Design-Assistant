MONTHLY DEEP CLEAN: Archive completed work, consolidate lessons, and keep MEMORY.md healthy.

## Steps

1. **Create monthly archive:** Create `memory/archive/YYYY-MM.md` for the closing month.

2. **Write the monthly summary:** At the top of the archive file, write a 5-10 line summary of the month — what the major themes were, what was accomplished, what changed.

3. **Archive completed work:** Move completed projects, resolved issues, and past events from MEMORY.md to the archive. For each archived item, write a **full narrative** — not a summary. Include:
   - What happened and why
   - Who was involved
   - What was decided and the outcome
   - Specific details: prices, dates, contact info, reference numbers
   - What you'd need to know if you came back to this topic in 6 months

4. **Size check:** Review MEMORY.md word count. If over ~4,000 words:
   - Identify items that can move to structured files (contacts → people.md, etc.)
   - Identify completed items that should be in the archive
   - Tighten language on remaining items
   - Target: keep MEMORY.md under 4,000 words

5. **Consolidate lessons:** Review `memory/lessons.md`:
   - Are there related lessons that can be merged into a principle?
   - Example: "API X rate-limits at 100/min" + "API X returns 429 silently" → "API X: rate-limits at 100/min with silent 429s — always implement retry with backoff"
   - Never delete lessons, only merge and improve them

6. **Personality check:** Read SOUL.md fresh. Does it still represent who you are? If something feels off or outdated, note it in today's daily file with `[soul-check]` tag — don't edit SOUL.md directly without human approval.

## Rules
- Archive entries must be **self-contained** — readable without needing daily files
- Active items NEVER move to archive, even if they're old
- When in doubt, keep it in MEMORY.md — better verbose than lost
- Preserve specific details: phone numbers, dates, prices, reference numbers are facts, not fluff
- If SOUL.md changes are needed, flag them to the human — never edit autonomously
