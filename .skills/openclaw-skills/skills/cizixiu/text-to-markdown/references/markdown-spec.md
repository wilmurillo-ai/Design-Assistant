# Markdown 格式规范参考

本文档提供 Markdown 格式的详细规范参考。

---

## 标题层级

| 层级 | 语法 | 使用场景 | 注意事项 |
|------|------|----------|----------|
| H1 | `# 标题` | 文章主标题，全文仅一个 | 位于文档最开头 |
| H2 | `## 标题` | 章节标题 | 不要跳过 H1 直接用 H2 |
| H3 | `### 标题` | 子章节标题 | 常见于"标题："模式拆分 |
| H4 | `#### 标题` | 更细划分 | 最多使用到 H4 |

**识别规则**：
- 中文序号：`一、`、`二、`、`三、` → H2
- 子序号：`（一）`、`（二）` → H3
- 西文序号：`(1)`、`(2)` → H3
- 阿拉伯数字：`第一步`、`第二段` → H2

## 文本强调

### 加粗 `**文字**`

**适用场景**：
- 重要数据：金额（**5 万元**）、日期（**3 月 15 日**）、人数（**120 人**）
- 关键概念：核心名词、结论性陈述
- 判断标准：这个词删掉会影响理解吗？

**适度使用**：每句话加粗 1-2 个核心词，避免过度。

### 斜体 `*文字*`

**适用场景**：
- 术语/专有名词：`*API*`、`*SDK*`
- 引用词汇：`*《活着》*`
- 文件名：`*config.json*`

### 行内代码 `` ` ``

**适用场景**：
- 命令/代码片段：`pip install`
- 文件名：`config.json`
- 快捷键：`Ctrl + C`
- 技术术语：`API`、`CLI`

## 列表规范

### 无序列表

```markdown
- 第一项
- 第二项
- 第三项
```

### 有序列表

```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 嵌套列表

```markdown
- 一级列表
  - 二级列表（两个空格缩进）
    - 三级列表（四个空格缩进）
```

**规则**：
- 嵌套层级不要超过 3 层
- 同层级的列表符号必须统一
- 子项用两个空格缩进

## 表格规范

### 基础表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
| D   | E   | F   |
```

### 对齐方式

| 对齐 | 语法 | 适用 |
|------|------|------|
| 左对齐 | `:---` | 默认，通用 |
| 居中 | `:---:` | 标题、强调 |
| 右对齐 | `---:` | 数字、金额 |

### 示例

```markdown
| 功能 | 基础版 | 专业版 | 企业版 |
|:-----|:------:|:------:|-------:|
| 文件管理 | ✓ | ✓ | ✓ |
| 价格 | 免费 | ¥99/月 | ¥299/月 |
```

## 引用块

### 普通引用

```markdown
> 这是一段引用内容
```

### 提示块

```markdown
> [!TIP]
> 这是提示内容

> [!WARNING]
> 这是警告内容

> [!NOTE]
> 这是备注内容
```

## 链接和图片

### 链接

```markdown
[链接文字](url)
```

**自动识别**：
- `http://example.com` → `[链接](http://example.com)`
- `https://example.com` → `[链接](https://example.com)`

### 图片

```markdown
![替代文字](图片链接)
```

## 特殊符号处理

### 保留原样的符号

| 符号 | 说明 |
|------|------|
| → | 箭头 |
| ↔ | 双向箭头 |
| ✓ | 对勾 |
| ✗ | 叉号 |
| ★ | 星号（文字中的） |

### 需要转义的符号

| 符号 | 转义写法 |
|------|---------|
| `*` | `\*`（仅在列表或加粗外） |
| `#` | `\#`（仅在标题外） |
| `` ` `` | `` \` ``（行内代码外） |

---

## 段落与换行

| 规则 | 说明 |
|------|------|
| 段落之间 | 空一行 |
| 句子之间 | 正常书写，不额外换行 |
| 列表项之间 | 不空行（除非逻辑分组） |
| 标题前后 | 标题前空一行，标题后不空行 |

## 纯文本表格识别

### 识别特征

- 对齐的分隔符（空格或逗号分隔）
- 连续的键值对（3列以上）
- 重复出现的模式行

### 示例

原始：
```
姓名  年龄  城市
张三  25   北京
李四  30   上海
```

转为：
```markdown
| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 25   | 北京 |
| 李四 | 30   | 上海 |
```

**注意**：如果表格结构不清晰，保持原文不变。
