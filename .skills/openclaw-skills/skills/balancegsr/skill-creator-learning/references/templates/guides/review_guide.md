# 模块复盘执行指南

> 本文件在复盘模式触发时加载。提供复盘各步骤的详细执行指导。

## 复盘前准备

1. 确认要复盘的模块编号 N
2. 加载该模块的所有学习点笔记：`notes_module_N_*.md`
3. 加载 learning_plan.md 中该模块的规划信息（学习点列表、目标描述）

## 执行步骤

### 第一步：知识整合

目标：从散落的学习点笔记中提炼出模块级别的知识结构。

- 列出该模块所有学习点的核心概念
- 梳理概念之间的关系：哪些是基础→进阶关系？哪些是并列关系？哪些互为补充？
- 用一段话或一个简要结构图向用户描述这个模块的知识架构
- 与用户讨论：这个结构是否符合你的理解？有没有你觉得关系不对的地方？

### 第二步：缺口识别

目标：找出学习过程中遗留的认知薄弱点。

- 汇总所有笔记中"认知盲区"部分的内容
- 逐条判断：
  - 已在后续学习点中覆盖 → 标记为已解决
  - 仍然存在 → 评估严重程度：影响后续学习？还是可以暂时搁置？
- 检查模块内各学习点的"与其他模块的关联"部分，是否有关联关系在学习过程中未被充分建立
- 与用户讨论发现的缺口，共同决定处理方式：
  - 当场补充讲解（如果缺口较小且影响理解完整性）
  - 记录到"仍需深化"中（如果缺口需要更多时间或后续模块会覆盖）

### 第三步：验证测试

目标：检验用户是否真正整合了模块知识，而不只是记住了各个学习点的内容。

设计 1-3 个验证问题（数量根据模块体量调整）：
- 至少 1 个综合性问题：需要组合多个学习点的知识才能回答
- 如模块涉及实践技能：设计一个跨学习点的应用场景
- 如模块涉及判断能力：设计一个需要权衡取舍的决策问题

验证的目的是发现理解深度的问题，不是考试。如果用户回答暴露了理解偏差，针对性补充后继续，不需要"重新验收"。

### 第四步：策略反思

目标：从本模块的学习过程中提取对后续模块有用的经验。

回顾本模块的学习过程，关注：
- 哪种验收方式效果好？哪种流于形式？
- 学习节奏是否合适？（太快导致消化不良？太慢导致注意力分散？）
- 资料质量如何？有没有特别好或特别差的资料来源？
- bottom-up 策略在这个模块中是否合适？

如发现需要调整后续策略，更新 learning_plan.md 中的相关说明。

### 第五步：生成模块总结

按照笔记格式规范中的"模块整合总结"模板，生成 `summary_module_N.md`。

总结应该是：
- 用用户自己的话（而非照搬笔记）重新组织的知识结构
- 聚焦"学完这个模块后我能做什么"，而非"这个模块讲了什么"
- 明确标注仍需深化的点，方便后续回溯

保存后通知用户。
