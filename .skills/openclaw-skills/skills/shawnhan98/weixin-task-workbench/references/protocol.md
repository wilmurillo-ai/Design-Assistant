# 单入口多任务工作台协议（微信）

## 目标

在一个微信/OpenClaw 对话窗口中并行管理多个任务，并保证上下文不串味。

## 用户可见概念

- 任务
- 任务 ID
- 当前任务
- 状态
- 摘要

除非用户明确询问实现细节，不主动暴露：

- `sessionKey`
- 路由内部实现
- subagent / session 机制细节

## 标准命令

- `新建任务：<title>`
- `任务列表`
- `切到 #<id>`
- `继续`
- `总结 #<id>`
- `关闭 #<id>`
- `归档 #<id>`
- `任务状态`

## 边界处理（必需）

### 新建任务缺少标题

如果用户只发送 `新建任务`、`新建任务：` 或等价空标题命令，不要创建空任务。必须提示：

```text
新建任务时还缺一个标题。
可直接发送：
- 新建任务：周报
- 新建任务：招聘方案优化
```

### 指定任务不存在

如果用户请求的 `#<id>` 不存在（如 `切到 #9`、`总结 #9`、`关闭 #9`、`归档 #9`），不要返回原始错误栈。必须提示：

```text
任务 #9 不存在。
先发送「任务列表」查看可用任务。
```

### 无当前任务时继续

如果用户发送 `继续`，但当前没有任务：

```text
当前没有可继续的任务。
先发送「任务列表」查看任务，或发送「新建任务：...」开始。
```

### 已完成 / 已归档任务

- `切到 #<id>`：允许切换到已完成任务，但要在回执中标明状态。
- `继续`：默认只路由到当前任务；如果当前任务为 `completed` 或 `archived`，先提醒用户切换或明确恢复。
- `归档 #<id>`：如果任务已归档，应返回“已归档，无需重复归档”。
- `关闭 #<id>`：如果任务已完成，应返回“已完成，无需重复关闭”。

## 强制门禁

在首次任务命令前，必须确认主机支持：

- `sessions_spawn`
- `sessions_send`
- `sessions_history`

缺任一能力时，立即阻断并返回配置修复指引。

## Registry 隔离范围

按微信账号 + 联系人隔离：

```text
tasks/weixin/<account>/<peer>.json
```

{
  "currentTaskId": 2,
  "tasks": [

## 响应模板（低自由度，优先套用）

以下控制面命令优先使用固定模板回复，避免风格漂移。

### 新建任务成功

```text
已新建任务 #<id>：<title>
当前任务已切换到：#<id> <title>
```

### 切换任务成功

```text
已切换到任务 #<id>：<title> [<状态中文名>]
```

### 继续但无当前任务

```text
当前没有可继续的任务。
先发送「任务列表」查看任务，或发送「新建任务：...」开始。
```

### 指定任务不存在

```text
任务 #<id> 不存在。
先发送「任务列表」查看可用任务。
```

### 重复关闭

```text
任务 #<id> 已完成，无需重复关闭。
```

### 重复归档

```text
任务 #<id> 已归档，无需重复归档。
```

### 新建任务缺标题

```text
新建任务时还缺一个标题。
可直接发送：
- 新建任务：周报
- 新建任务：招聘方案优化
```

### 任务列表

有任务时：

```text
任务列表 / Task List
#1 周报 [已完成]
#2 PPT [进行中] ← 当前
```

无任务时：

```text
任务列表 / Task List
当前还没有任务。
可直接发送：
- 新建任务：周报
- 新建任务：产品需求梳理
```

### 任务回复头（必需）

```text
[任务:#2 PPT]
```

### 任务总结

```text
任务 #2：PPT
- 进展：已完成结构与前两页
- 当前状态：进行中
- 阻塞：缺少最新数据图
- 下一步：补图并统一视觉样式
```

### 任务状态（排障命令）

有当前任务时：

```text
任务状态
- 当前任务：#2 PPT [进行中]
- taskSessionKey: agent:main:openclaw-weixin:direct:<peer>:task:t2
- registry: tasks/weixin/<account>/<peer>.json
```

无任务时：

```text
任务状态
- 当前任务：无
- 任务总数：0
- registry: tasks/weixin/<account>/<peer>.json
- 提示：先发送「新建任务：...」开始
```

注意：`/status` 显示的是 OpenClaw 当前全局会话信息，不是任务工作台状态。

## 推荐状态集

- `todo`
- `in_progress`
- `waiting_input`
- `blocked`
- `completed`
- `archived`

## 路由决策树

1. 若消息是任务控制命令，优先走控制面动作。
2. 若消息显式包含 `#<id>`，路由到指定任务。
3. 否则默认路由到 `currentTaskId`。
4. 若不存在当前任务，提示用户新建或切换任务。
