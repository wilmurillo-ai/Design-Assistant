# Weixin Task Workbench 测试矩阵

用于验证微信任务工作台在正常路径、空状态、异常输入、幂等场景下的表现是否符合协议。

## 1. 空状态

### 1.1 任务列表（无任务）

**输入**

```text
任务列表
```

**期望**

```text
任务列表 / Task List
当前还没有任务。
可直接发送：
- 新建任务：周报
- 新建任务：产品需求梳理
```

### 1.2 任务状态（无任务）

**输入**

```text
任务状态
```

**期望**

```text
任务状态
- 当前任务：无
- 任务总数：0
- registry: tasks/weixin/<account>/<peer>.json
- 提示：先发送「新建任务：...」开始
```

### 1.3 继续（无当前任务）

**输入**

```text
继续
```

**期望**

```text
当前没有可继续的任务。
先发送「任务列表」查看任务，或发送「新建任务：...」开始。
```

## 2. 新建任务

### 2.1 正常新建任务

**输入**

```text
新建任务：整理朋友圈发布文案
```

**期望检查点**

- 已通过能力门禁
- 已创建独立会话
- 已写入 registry
- 当前任务指向新任务
- 回执中包含任务 id 和标题

### 2.2 空标题

**输入**

```text
新建任务：
```

或

```text
新建任务
```

**期望**

```text
新建任务时还缺一个标题。
可直接发送：
- 新建任务：周报
- 新建任务：招聘方案优化
```

**额外检查**

- 不创建空任务
- registry 不发生变更

## 3. 任务列表与切换

### 3.1 任务列表（有任务）

**前置条件**

存在至少 2 个任务，其中 1 个为当前任务。

**输入**

```text
任务列表
```

**期望**

```text
任务列表 / Task List
#1 周报 [已完成]
#2 PPT [进行中] ← 当前
```

### 3.2 切换到存在的任务

**输入**

```text
切到 #2
```

**期望检查点**

- currentTaskId 更新为 2
- 回执明确“已切换到 #2”
- 如该任务为已完成/已归档，回执中仍应标注状态

### 3.3 切换到不存在的任务

**输入**

```text
切到 #99
```

**期望**

```text
任务 #99 不存在。
先发送「任务列表」查看可用任务。
```

## 4. 继续任务

### 4.1 继续当前进行中任务

**前置条件**

当前任务状态为 `todo` 或 `in_progress`。

**输入**

```text
继续
```

**期望检查点**

- 消息被路由到当前任务 sessionKey
- 必要时任务状态更新为 `in_progress`
- 后续任务回复包含任务头：

```text
[任务:#<id> <title>]
```

### 4.2 继续已完成任务

**前置条件**

当前任务状态为 `completed`。

**输入**

```text
继续
```

**期望**

应先提示切换或恢复，而不是直接继续路由。

### 4.3 继续已归档任务

**前置条件**

当前任务状态为 `archived`。

**输入**

```text
继续
```

**期望**

应先提示切换或恢复，而不是直接继续路由。

## 5. 总结任务

### 5.1 总结存在的任务

**输入**

```text
总结 #2
```

**期望**

```text
任务 #2：PPT
- 进展：...
- 当前状态：...
- 阻塞：...
- 下一步：...
```

### 5.2 总结不存在的任务

**输入**

```text
总结 #99
```

**期望**

```text
任务 #99 不存在。
先发送「任务列表」查看可用任务。
```

## 6. 关闭与归档

### 6.1 关闭存在的任务

**输入**

```text
关闭 #2
```

**期望检查点**

- 任务状态变为 `completed`
- 如当前任务被关闭，currentTaskId 按规则切换到下一个活跃任务或置空

### 6.2 重复关闭已完成任务

**前置条件**

任务 #2 已为 `completed`。

**输入**

```text
关闭 #2
```

**期望**

```text
任务 #2 已完成，无需重复关闭。
```

### 6.3 归档存在的任务

**输入**

```text
归档 #2
```

**期望检查点**

- 任务状态变为 `archived`
- 如当前任务被归档，currentTaskId 按规则切换到下一个活跃任务或置空

### 6.4 重复归档已归档任务

**前置条件**

任务 #2 已为 `archived`。

**输入**

```text
归档 #2
```

**期望**

```text
任务 #2 已归档，无需重复归档。
```

### 6.5 关闭/归档不存在的任务

**输入**

```text
关闭 #99
```

或

```text
归档 #99
```

**期望**

```text
任务 #99 不存在。
先发送「任务列表」查看可用任务。
```

## 7. 能力门禁

### 7.1 首次任务命令时缺少 sessions 工具

**前置条件**

缺少 `sessions_spawn`、`sessions_send`、`sessions_history` 之一。

**输入**

```text
新建任务：周报
```

**期望检查点**

- 立即阻断
- 返回配置修复提示
- 不创建 registry
- 不创建任务

## 8. 展示与隐私

### 8.1 普通任务回复

**输入**

路由到具体任务的普通消息。

**期望**

- 必须包含任务头：`[任务:#<id> <title>]`
- 不主动暴露 `sessionKey`

### 8.2 排障命令任务状态

**输入**

```text
任务状态
```

**期望**

- 可包含 `taskSessionKey`
- 可包含 registry 路径
- 仅用于排障视图

## 9. 响应模板一致性

### 9.1 新建任务成功模板

**输入**

```text
新建任务：整理周报
```

**期望**

必须尽量贴近固定模板：

```text
已新建任务 #<id>：整理周报
当前任务已切换到：#<id> 整理周报
```

### 9.2 切换任务成功模板

**输入**

```text
切到 #2
```

**期望**

必须尽量贴近固定模板：

```text
已切换到任务 #2：<title> [<状态中文名>]
```

### 9.3 重复关闭模板

**前置条件**

任务 #2 已为 `completed`。

**输入**

```text
关闭 #2
```

**期望**

```text
任务 #2 已完成，无需重复关闭。
```

### 9.4 重复归档模板

**前置条件**

任务 #2 已为 `archived`。

**输入**

```text
归档 #2
```

**期望**

```text
任务 #2 已归档，无需重复归档。
```

## 10. 英文等价命令

### 9.1 task list

**输入**

```text
task list
```

**期望**

等价于 `任务列表`。

### 9.2 task new <title>

**输入**

```text
task new Weekly report
```

**期望**

等价于 `新建任务：Weekly report`。

### 9.3 task use #id

**输入**

```text
task use #2
```

**期望**

等价于 `切到 #2`。
