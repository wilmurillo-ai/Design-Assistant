# Quality targets

## first-playable
Use when the point is to validate the fantasy, mechanic, or interaction loop quickly.
Must still be real and bootable, but can defer polish and content breadth.

## polished-prototype
Use when the result needs to impress, demo well, or meaningfully represent the intended game.
This is the default serious greenfield target.

## production-feature
Use when adding or changing a feature inside a maintainable product codebase.
The feature should feel complete, verified, and ready to keep building on.

## live-patch
Use when the repo is shipped or operationally risky.
Prefer small, safe, reversible changes with strong verification and rollback notes.
