# Game-dev abstractions

Prefer these abstractions in plans and code organization when they fit:

- `GameManager`
- `InputGrammar`
- `PlayerController`
- `CameraRig`
- `StateMachine`
- `EncounterDirector`
- `ProgressionModel`
- `HUDFlow`
- `FeedbackLayer`
- `SoundLayer`
- `SaveModel`
- `BackendProfile`

Do not overfit the top-level architecture to one specific renderer or framework unless the project already commits to it.
