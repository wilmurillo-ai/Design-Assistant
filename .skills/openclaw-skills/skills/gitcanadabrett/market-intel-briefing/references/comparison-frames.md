# Comparison Frames

Use this reference when the user asks for a side-by-side comparison across jurisdictions, competitors, categories, or strategic options.

## Build one shared frame first

Do not write disconnected mini-profiles.
Compare each option against the same decision criteria so tradeoffs are visible.

Useful shared criteria include:
- power availability and interconnection risk
- permitting complexity and expected speed
- water constraints
- gas access or backup-fuel flexibility when relevant
- tax, policy, or incentive environment
- infrastructure readiness
- speed to execution
- operational risk and evidence quality

## Output pattern for comparison briefs

1. Scope and comparison frame
2. Side-by-side comparison by criteria (include "evidence quality" row or column per option)
3. Key tradeoffs
4. Commercial implications
5. Recommended next 3 actions
6. Client-facing talk track
7. Sources

## Comparison guidance

- Prefer directional comparisons over fake precision
- Flag where evidence quality differs across options with an explicit per-option label (e.g., "strong / moderate / weak" or "primary-sourced / inferred")
- Distinguish province/state-level signals from utility, municipality, or site-specific constraints
- Surface what is still unknown before making hard recommendations
- If the user's ask is broad, narrow to the criteria most relevant to the decision

## Boundary rules

- Do not invent timelines, capacity, or permitting certainty
- Do not collapse unequal evidence into equal confidence
- Do not overread marketing or policy language without operational support
