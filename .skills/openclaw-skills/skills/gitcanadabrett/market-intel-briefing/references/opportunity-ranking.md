# Opportunity Ranking

Use this reference when the user asks for underserved skills, workflow gaps, opportunity scans, or ranked commercial openings.

## Anchor opportunities in real workflow pain

Do not generate abstract ideas first.
Start from a painful manual workflow, repeated operator burden, or obvious gap in current tooling.

Useful prompts to apply internally:
- What are people still doing manually?
- Where do current tools fail to connect the last mile?
- What costs time, introduces error, or delays decisions?
- What current workaround signals unmet demand?

## Ranking criteria

Rank opportunities using explicit criteria such as:
- pain severity
- frequency of the workflow
- value of solving it
- current tooling gap
- ease of delivering a first useful solution
- confidence in demand evidence

## Output pattern

1. Scope and segment
2. Painful workflows or unmet jobs
3. What current tools miss
4. Ranked opportunity list with short rationale and demand evidence tag per item (inferred / adjacent-market analogy / directly observed)
5. Case studies or examples with source type tag per item (independent research / vendor-published / operator self-reported)
6. Confidence notes
7. Recommended next actions
8. Sources

## Confidence guidance

When opportunity attractiveness is inferred rather than directly observed:
- say so
- reduce certainty in ranking language
- distinguish clear pain from speculative market size

## Avoid

- generic startup-idea lists
- ranking based only on novelty
- overclaiming market demand from weak anecdotes
