# Source Triage Reference

## Source hierarchy

### Tier 1 - Primary
Use first when available.
- regulatory filings
- company press releases
- earnings materials and transcripts
- official product or policy documentation
- official government releases

### Tier 2 - Credible secondary
Use to broaden context or when primary is not yet available.
- respected trade publications
- established beat reporters
- reputable industry newsletters with sourcing

### Tier 3 - Community signal
Use sparingly and always label clearly.
- forums
- X posts
- Discord or Slack communities
- LinkedIn posts or comments

## Claim handling rules
- One primary source is better than five rewrites.
- If all sources trace back to the same rumor, treat it as one unverified claim.
- If a secondary source cites unnamed sources, mark as `Likely` at most.
- If a community post contains a specific operational datapoint, keep it only if it changes the user's watch priorities and label it `Unverified`.

## Exclusion rules
Drop items that are:
- repeated coverage of the same announcement with no new facts
- generic opinion or trend commentary
- stale items outside the agreed time window unless they materially affect interpretation

## Confidence shortcuts
- Confirmed: primary source or strong corroboration
- Likely: credible secondary source with consistent reporting
- Unverified: rumor, anecdote, or single weak source
