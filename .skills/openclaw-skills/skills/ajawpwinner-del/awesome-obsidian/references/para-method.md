# PARA 方法详解

PARA 是 Tiago Forte 提出的个人知识管理方法，将所有信息分为四类：

## 1. Projects（项目）

**定义**：有明确开始和结束时间的任务

**特点**：
- 有具体的目标
- 有截止日期
- 完成后归档

**示例**：
- 写一篇博客文章
- 完成一个产品发布
- 准备一次演讲
- 学习一门课程

**文件夹结构**：
```
projects/
├── 博客文章-2026-04/
├── 产品发布-v2.0/
└── 演讲准备-技术大会/
```

## 2. Areas（领域）

**定义**：持续关注的责任或兴趣领域

**特点**：
- 没有明确的结束时间
- 需要持续维护
- 长期投入

**示例**：
- 健康（运动、饮食、睡眠）
- 财务（投资、预算、记账）
- 职业（技能提升、人脉维护）
- 家庭（亲子、伴侣关系）
- 兴趣爱好（摄影、阅读、音乐）

**文件夹结构**：
```
areas/
├── health/           # 健康
├── finance/          # 财务
├── career/           # 职业
├── family/           # 家庭
└── hobbies/          # 兴趣爱好
```

## 3. Resources（资源）

**定义**：有长期价值的信息和知识

**特点**：
- 可能未来用到
- 不属于特定项目或领域
- 可复用的知识

**示例**：
- 技术文档
- 学习笔记
- 参考资料
- 想法和思考
- 书摘和金句

**文件夹结构**：
```
resources/
├── tech-docs/        # 技术文档
├── learning-notes/   # 学习笔记
├── references/       # 参考资料
├── thoughts/         # 想法和思考
└── book-quotes/      # 书摘和金句
```

## 4. Archive（归档）

**定义**：不再活跃的项目和信息

**特点**：
- 已完成的项目
- 不再需要的资料
- 保留以备查阅

**示例**：
- 已完成的项目
- 过期的资料
- 历史记录

**文件夹结构**：
```
archive/
├── 2025-projects/
├── 2024-projects/
└── old-resources/
```

## PARA 流转规则

### 信息流向

```
inbox → Projects/Areas/Resources → Archive
```

### 判断流程

1. **是项目吗？** → 有明确目标和截止日期 → `projects/`
2. **是领域吗？** → 需要持续关注 → `areas/`
3. **是资源吗？** → 有长期价值 → `resources/`
4. **都不属于？** → 临时信息 → `inbox/`

### 定期整理

- **每天**：快速收集到 inbox
- **每周**：整理 inbox，归类到正确位置
- **每月**：归档完成的项目
- **每季度**：回顾领域和资源

## PARA 与 Obsidian 结合

### 文件夹映射

```
Obsidian Vault/
├── inbox/        ← 快速收集
├── projects/     ← PARA: Projects
├── areas/        ← PARA: Areas
├── resources/    ← PARA: Resources
├── archive/      ← PARA: Archive
└── journal/      ← 日记（不属于 PARA）
```

### 标签系统

使用标签补充文件夹分类：

```markdown
#project/博客文章
#area/健康
#resource/技术文档
#archive/2025
```

### 链接关联

用双向链接连接相关笔记：

```markdown
这篇笔记关联到 [[项目A]]
参考了 [[资源B]]
属于 [[领域C]]
```

## 常见问题

### Q: 项目和领域如何区分？

**项目**：有明确的"完成"状态
- ✅ 写完博客文章 = 完成
- ✅ 发布产品 = 完成

**领域**：永远不会"完成"
- ❌ 健康永远不会"完成"
- ❌ 职业发展永远不会"完成"

### Q: 资源和领域如何区分？

**领域**：你需要主动维护
- 健康 → 需要运动、饮食管理
- 财务 → 需要记账、投资

**资源**：被动参考
- 技术文档 → 需要时查阅
- 学习笔记 → 需要时复习

### Q: inbox 多久整理一次？

建议**每周一次**，避免堆积过多。

可以设置固定时间（如周日晚上）进行整理。
