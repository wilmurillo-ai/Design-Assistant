# Second Round Guide

请先参考 `./references/workflow.md` 中定义的工作流执行任务。

当前阶段是**第二轮代码生成阶段**。

执行要求：

1. 不要重新输出整份需求分析
2. 直接基于已确认的“详细执行信息”继续实现
3. 优先复用 `./onescience/` 中已有模块
4. 先读取相关 `assets/models/` / `assets/contracts/`
5. 如契约信息不足，再回到对应源码锚点补充确认
6. 只修改与当前任务直接相关的文件

设备与测试要求：

7. 若生成测试代码、样例代码或运行验证代码，默认优先使用 GPU
8. 优先采用：
   - `device = torch.device("cuda" if torch.cuda.is_available() else "cpu")`
9. 再将模型和输入张量移动到 `device`
10. 若环境中没有 GPU，应自动回退到 CPU，而不是写死为仅支持 CUDA

输出时应包含：

- 最终完整代码
- 复用了哪些 OneScience 组件
- 主要 shape 变化
- 新增桥接模块或新增调用层改动说明
- 如有必要，补充最小验证建议

如果用户未额外指定保存路径，则默认将生成文件保存到对应的 `case` 目录下。

如果已确认的设计中仍然存在会导致代码无法实现的关键冲突，请先用一个简短的 `md` 区块指出问题；否则直接开始生成代码。
