# First Round Guide

请先参考 `./references/workflow.md` 中定义的工作流执行任务。

当前阶段是**第一轮需求整理阶段**，不是最终代码生成阶段。

执行要求：

1. 先理解用户的自然语言需求
2. 若用户明确提到已有模型，优先读取 `./assets/models/` 中对应模型卡
3. 再读取 `./assets/contracts/` 中相关内容
4. 必要时再读取 `./onescience/` 中对应源码
5. 第一轮**不要直接输出最终代码**
6. 先输出一份“详细执行信息”或“结构化规格说明”，供用户确认

这份“详细执行信息”至少要包含：

- 任务目标
- 输入定义
- 输出定义
- 保持不变的部分
- 计划修改的部分
- 推荐复用的 OneScience 组件
- 主干结构建议
- 主要 shape 变化
- 需要用户确认的关键点

在输出结束前，还必须追加一个固定的确认区块，至少包括：

- `## 待确认摘要`
- `## 自检结果`
- `## 用户操作`

额外约束：

- 命名尽量统一使用 `Batch`、`TimeSteps`、`PressureLevels`、`Height`、`Width`
- 如果 `onescience` 里已有合适模块，优先复用
- 如果任务是模型内模块替换，先看模型卡，再看相关组件契约
- 如果需要判断“直接替换是否可行”，第一轮必须明确说明：
  - 能否直接替换
  - 若不能，最小可实现改造路径是什么
  - 哪些部分保持原模型不变
  - 需要修改哪些文件

输出结束时，请明确告诉用户下述信息，同时将本次思考中进行总结，并整理为文档输出：

- 如果用户确认这份详细执行信息，你再继续生成完整代码
