# 内容策略与生产手册

当任务涉及账号定位、内容支柱、选题、标题、正文模板或四格漫画脚本时，读取本文件。

## 账号核心定义

- 基础赛道：`普通人用 AI 做训练、饮食和执行复盘，把抽象目标变成可执行动作`
- 推荐叙事原型：
  - `职业过渡期自救型`
  - `脑力劳动者精力管理型`
  - `AI 工作流实验型`
- 内容承诺：展示 `普通人如何用 AI 提高健身执行率`
- 核心卖点：`真实记录 + AI 实操 + 低成本执行 + 连续复盘`

## 人设落地规则

每个账号至少补齐：

- 起点状态：当前身体、作息、困境
- 主线任务：`30天`、`60天` 或 `90天`
- 执行约束：在家/健身房、时间、预算、器械
- 追踪指标：体重、围度、作息、训练完成率、情绪

推荐公式：

`[身份/阶段] + [当前状态] + [用 AI 做什么] + [准备在 X 天完成什么]`

如果用户没有提供真实数值，不要硬编；优先用区间、状态或 `[待补充]`。

## 高分辨率档案模式

如果用户给了真实基线数据，后续所有内容统一沿用。

优先锁定：

- 当前体重或体重区间
- 目标体重或目标体脂
- `2-3` 个围度
- 当前睡眠或作息时间
- 当前训练频率

## 四大内容支柱

### 1. AI 制定方案

- 典型内容：提示词拆解、模型对比、训练计划、饮食模板
- 流量来源：`搜索流量`
- 目标动作：`收藏`
- 写法重点：标题写清问题和结果，正文给出可抄作业结构
- CTA 示例：`评论区回“计划”领模板`

### 2. 真人执行记录

- 典型内容：周复盘、体重/围度变化、训练打卡、放弃和重启
- 流量来源：`关注回访 + 推荐流量`
- 目标动作：`评论 + 关注`
- 写法重点：时间线、真实阻力、真实调整
- CTA 示例：`如果你也在重建状态，可以留言一起打卡`

### 3. AI 工具与方法测试

- 典型内容：提示词前后对比、不同模型实测、失败案例、工作流演示
- 流量来源：`搜索流量`
- 目标动作：`收藏 + 转发`
- 写法重点：对比、判断标准、前后差异
- CTA 示例：`想要我这版提示词，可以评论关键词`

### 4. 自救叙事与情绪恢复

- 典型内容：情绪复盘、习惯重建、低谷期记录、自我对话
- 流量来源：`推荐流量`
- 目标动作：`关注 + 私信`
- 写法重点：情绪钩子后必须落回具体行动
- CTA 示例：`如果你也在低谷期，我把我用的最小行动清单发你`

## 内容配比

冷启动默认：

- `50%` AI 实操
- `30%` 健身记录
- `20%` 自救叙事和复盘

## 搜索优先策略

每次选题先搭关键词组：

- 人群词：`90后`、`上班族`、`失业在家`、`新手减脂`
- 场景词：`在家训练`、`预算有限`、`作息混乱`、`没有私教`
- 痛点词：`坚持不下去`、`不会做计划`、`吃饭控制不住`、`练了没效果`
- 结果词：`怎么做`、`一周计划`、`避坑`、`模板`
- 工具词：`ChatGPT`、`AI 健身计划`、`提示词`、`饮食规划`

标题优先 `人群 + 问题 + 结果`。

## 选题判断流程

1. 这条内容的目标是什么：冷启动、搜索、信任、互动、转化还是复盘。
2. 它属于哪个内容支柱。
3. 用户会主动搜索什么问题。
4. 证据是什么：截图、数据、记录、对比、失败案例，还是个人体验。
5. 读完后只让用户做一个动作：收藏、评论、私信或关注。
6. 离开账号上下文后，陌生人能不能独立看懂。

## GEO 写作质量规则

适用于方案类、工具测试类、复盘类内容。

正文尽量包含：

- `条件`
- `动作`
- `结果`
- `限制`

还要有：

- 可抽取事实锚点
- 边界说明，如“这是我的个人记录，不适合所有人”

## 笔记模板

### 模板 A：AI 生成计划型

适用于：提示词、训练方案、饮食模板、模型对比

结构：

1. 开头直接给结论
2. 交代输入条件
3. 展示 AI 输出关键部分
4. 评价哪里可执行，哪里不靠谱
5. 给出最终采用版本
6. 用一句 CTA 收口

### 模板 B：执行记录型

适用于：7 天、14 天、30 天打卡与复盘

结构：

1. 今天是第几天
2. 先说变化，再说过程
3. 写清做了什么、没做到什么
4. 解释为什么卡住
5. 下一步怎么调整
6. 邀请用户留言

### 模板 C：避坑拆解型

适用于：错误计划、失败经验、模型幻觉、执行断裂

结构：

1. 点出误区
2. 说明为什么看起来对，实际上不好执行
3. 给出真实例子
4. 给出更稳妥的改法
5. 总结一句判断标准

### 模板 D：自救叙事型

适用于：职业空档期、作息崩掉、重建秩序

结构：

1. 交代真实处境
2. 说出最难的一件事
3. 说明为什么选择 AI + 健身
4. 给出当时只做的一件最小行动
5. 解释为什么这个动作能执行下去
6. 再写一个已出现的小变化
7. 结尾回到“普通人也能先做一点”

### 模板 E：工作流展示型

适用于：自动化闭环、OpenClaw、数据追踪、次日计划生成

结构：

1. 先说明闭环解决什么问题
2. 展示 `输入 -> 处理 -> 输出`
3. 点出最容易断掉的一步
4. 给出最简可复制版本
5. 说明普通人不做自动化时也能先抄哪一步
6. 用一句 CTA 收口

## 标题生成规则

避免“逆袭”“蜕变”“暴瘦”等夸张词。

可复用公式：

- `我失业后第 X 天，让 AI 给我做了一份在家减脂计划`
- `把身高体重和作息喂给 AI 后，它给我的方案能执行吗`
- `普通人不用私教，AI 能不能先帮你把训练表做出来`
- `AI 做健身计划最大的问题，不是不专业，是做不下去`
- `90后职业空档期，我用 AI + 健身把作息一点点拉回来了`

## 封面和开头规则

- 封面只讲一个冲突或结果
- 标题负责点开，前 3 行负责继续看
- 开头优先写结论
- 让用户在 3 秒内知道“这篇对我有什么用”
