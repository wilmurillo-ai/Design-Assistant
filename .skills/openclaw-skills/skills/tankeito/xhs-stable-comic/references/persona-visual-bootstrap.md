# 人设与视觉初始化引导

当用户还没有明确的账号人物设定、漫画主角或画风系统时，先读取本文件。

目标不是直接开始写内容，而是先生成一套稳定的账号底稿：

- 账号人设卡
- 内容世界观一句话
- 角色设定卡
- 画风设定卡
- 四格漫画系统卡

## 先判断账号模式

只做这一步分类，不要一开始就锁死细节。

### 模式 A：真人参考模式

适用条件：

- 用户提供了自拍、生活照、职业经历或真实目标
- 用户希望账号更像自己
- 用户想长期经营“我自己”的叙事

输出重点：

- 真实身份锚点
- 真实叙事主线
- 真人参考转漫画角色卡

### 模式 B：原创角色模式

适用条件：

- 用户没有真人照片
- 用户不想暴露真实身份
- 用户只想做一个“AI+健身”的内容账号，不强求像本人

输出重点：

- 人设 archetype
- 角色视觉卡
- 画风系统卡

## 账号人设卡生成模板

先生成账号层面的人设，不要一开始就写得太满。

至少补齐这 8 项：

- 账号主题：AI+健身里的哪一条主线
- 叙事身份：普通人、脑力劳动者、职业过渡期、自律重建者、工作流实验者
- 内容承诺：帮用户解决什么问题
- 受众：谁最容易被吸引
- 情绪基调：温和、克制、理性、轻喜剧、治愈、实验感
- 主线周期：30天、60天、90天
- 追踪指标：体重、围度、作息、训练完成率、情绪、精力
- 商业边界：卖模板、卖提示词、卖陪跑，还是只做内容

## 世界观一句话模板

格式：

`我是 [身份]，正在用 [方法] 解决 [问题]，把 [抽象目标] 变成 [可执行结果]。`

示例：

- `我是一个经常脑子很忙但身体很乱的人，正在用 AI 和简单训练把混乱作息拉回正轨。`
- `我是一个普通上班族，正在测试 AI 到底能不能帮我把健身这件事做得更容易坚持。`
- `我是一个职业过渡期的人，正在把身体管理做成一个可复盘的小系统。`

## 角色设定卡生成模板

这是漫画主角层，不是账号层。

至少补齐：

- 年龄感
- 性别表达
- 发型
- 脸型
- 眼镜或无眼镜
- 体型
- 常穿搭
- 标志物
- 默认表情
- 固定场景

### 如果信息不足，优先输出 3 套角色候选

#### 候选 1：安静理工型

- 关键词：克制、理性、脑力劳动者、轻微疲惫但稳定

#### 候选 2：自救记录型

- 关键词：普通人、真实、稍微肉感、情绪回暖、慢慢重建

#### 候选 3：极客实验型

- 关键词：工具感、工作流、系统化、数据追踪、冷静但不冰冷

## 画风设定卡生成模板

至少补齐：

- 风格：半扁平 / 治愈插画 / 轻喜剧 / 极简漫画
- 线条：细线还是中细线
- 配色：低饱和暖色 / 冷静灰蓝 / 清爽米白
- 光感：柔和室内光 / 白天自然光 / 夜晚电脑光
- 版式：四格 / 单格 / 封面插画
- 文案策略：图内少字还是完全后期加字

### 默认建议

- 优先 `半扁平 + 细线 + 低饱和 + 2x2四格`
- 优先后期加中文，不让模型直接渲染长文案

## 四格漫画系统卡模板

如果用户要长期做漫画，先固定系统卡。

至少补齐：

- 阅读路径：2x2 四格
- 默认结构：问题 -> 方法介入 -> 执行 -> 结果/反思
- 适合的题材：自救、计划、复盘、工作流、避坑
- 封面规则：用第几格，还是单独做封面
- 负面锚点：不要什么画风、不要什么版式、不要什么气质

## 初始化输出规范

初始化任务默认输出：

1. 账号模式判断
2. 账号人设卡
3. 世界观一句话
4. 角色设定卡
5. 画风设定卡
6. 四格漫画系统卡
7. 首批 3 个剧情方向

## 默认原则

- 用户没给真人资料时，不要默认长成某个具体人
- 用户没给职业背景时，不要硬写具体行业
- 用户没给时间线时，不要硬写某个年份和月份
- 优先先搭系统，再写单篇内容

## 适合触发本文件的请求

- `帮我从零搭一个 AI+健身 账号`
- `先帮我定人物和画风`
- `我没有主角，你先给我 3 套角色方向`
- `我想做长期 4 格漫画，先搭系统`
