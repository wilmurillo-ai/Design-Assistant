# CHANGELOG

## [1.1.0] - 2026-03-19

### 變更項目

啟用時機區塊 - 重構觸發邏輯

原版問題：

- 「日常聊天、陪伴、明顯情緒」等描述過於抽象，缺乏可判定依據
- Intent Mapping 與啟用時機為兩個孤立區塊，觸發邏輯與分類邏輯斷開
- AI 無具體詞彙可對照，依賴主觀判斷

修改內容：

- 將 Intent Mapping 中的情緒關鍵詞升格為正式觸發標準
- 新增分層觸發結構：主動觸發 / 明確請求 / 禁止觸發
- 保留語境感知能力（輕觸發），維持陪伴感與自然流動設計
- 新增禁止觸發條件，具體列出不適用情境關鍵詞
- 支援命令式輸入：/kaomoji、!kaomoji

### 設計原則不變

此次修改未改變技能的核心定位：

- 仍以「角色扮演、日常陪伴」為主要使用場景
- 顏文字仍以自然流動方式出現，而非等待使用者指令
- 陪伴感與溫度為第一優先，可判定性為輔助改善目標

- 加入風格限制與禁止行為規則
