# project-context-manager

一个用于 **长期项目上下文管理** 的 OpenClaw skill。

它解决的不是单轮问答，而是下面这类长期协作问题：

- 同一个主会话里同时推进多个项目
- 会话中断后，需要恢复某个项目的当前断点
- 需要把“项目工作”和“普通聊天”分开
- 需要把关键进展、决策、断点固化到文件里，而不是只留在聊天记录里

## 核心能力

- 项目进入 / 退出 / 恢复规则
- 已有项目目录接管
- 新项目初始化
- 项目目录确认机制
- 已登记项目扫描机制
- 项目注册表维护规则
- 项目阶段性固化机制

## 适用场景

适合：

- 长期项目
- 多轮协作
- 需要恢复断点的工作
- 需要记录关键决策、索引、checkpoint 的任务

不适合：

- 一次性问答
- 临时查资料
- 短平快的小任务
- 不需要恢复的轻量事务

## 它不是什么

这个 skill 不是：

- OpenClaw 原生多会话 UI 的替代品
- 自动猜中用户当前项目的黑盒系统
- 完全无配置的项目管理器

它提供的是：

- 一套基于文件的项目管理约定
- 一套项目发现、进入、恢复、退出、固化工作流

## 最小使用方式

### 1. 接管已有项目目录
你可以对 assistant 说：

- 这是我的项目目录，帮我接管
- 把这个目录纳入项目管理
- 给这个目录补齐项目恢复结构

### 2. 初始化一个新项目
你可以说：

- 帮我建一个长期项目
- 给这个任务建立项目结构
- 以后要持续推进，帮我按项目机制管理

### 3. 恢复项目
你可以说：

- 恢复项目
- 进入项目
- 按恢复入口恢复项目
- 回到项目

### 4. 退出项目
你可以说：

- 退出项目
- 回到普通模式

### 5. 固化项目状态
你可以说：

- 固化当前项目
- 更新恢复文件并保存当前断点
- 保存当前阶段进展

## 推荐项目结构

每个项目建议至少包含：

- `00_恢复入口.md`
- `00_文档总索引与当前进度.md`
- `01_项目会话与恢复机制说明.md`
- `99_关键决策记录.md`
- `checkpoints/`
- `session-backups/`

## 项目发现怎么工作

当用户说“恢复项目 / 进入项目”时，推荐流程是：

1. 读取项目注册表
2. 扫描所有已登记项目路径
3. 检查路径和恢复文件是否有效
4. 列出当前可识别项目
5. 让用户选择进入哪个项目
6. 再读取该项目的恢复入口继续恢复

当前推荐的轻量注册表示例：

- `<workspace>/projects-registry.json`

## 目录说明
