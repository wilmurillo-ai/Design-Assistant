# 会话恢复入口（项目模板）

## 一、用途

当当前项目会话中断、上下文丢失、或需要快速恢复项目工作时，优先从本文件开始。

本文件定义的是**动态恢复协议**，不是静态文件清单。

---

## 二、恢复原则

1. 先读恢复入口，再执行恢复流程
2. 先扫描项目目录，再决定读哪些文件
3. 不依赖固定旧文件名而忽略新增文件
4. 优先恢复当前主线、当前阶段、当前断点、关键决策、下一步

---

## 三、恢复流程

### 第 1 步：扫描当前项目目录
至少识别：
- `*.md`
- `checkpoints/`
- `session-backups/`
- 其他新增的说明、决策、验证、实施、排障类文件

### 第 2 步：读取索引与决策文件
优先读取：
1. `00_文档总索引与当前进度.md`
2. `99_关键决策记录.md`

### 第 3 步：动态定位当前断点
根据索引和目录现状识别：
- 当前进行到哪
- 当前断点在哪个文件
- 最近新增的重要文件有哪些

### 第 4 步：补读最近 checkpoint / memory
如有需要，再补读：
- 最新 checkpoint
- 最近相关 memory 记录
- 其他与当前断点高度相关的新增文件

---

## 四、恢复兜底策略

如果恢复过程中发现以下情况：

- 索引缺失或明显过期
- 恢复入口内容不完整
- 目录结构与索引不一致
- 当前断点无法直接定位

则应执行兜底流程：

1. 先扫描当前目录中的全部相关文件
2. 优先读取最近 checkpoint
3. 再读取最近相关 memory / 摘要记录
4. 给出“低置信恢复”提示
5. 再向用户确认是否按当前判断继续

---

## 五、恢复后必须输出

恢复完成后，至少输出：

- 当前项目主线
- 已完成内容
- 当前断点
- 最近新增文件
- 当前关键决策
- 下一步建议

---

## 六、固化提醒

若恢复后发现：
- 索引已落后
- 当前断点变化但未固化
- 新决策未写入决策文件
- 当前阶段即将结束

应提醒用户考虑执行一次固化。
