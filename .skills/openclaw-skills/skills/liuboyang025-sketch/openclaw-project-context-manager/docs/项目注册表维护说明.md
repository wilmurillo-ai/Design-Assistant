# 项目注册表维护说明

## 一、目的

本文件用于说明 `project-context-manager` 当前版本中 `projects-registry.json` 的职责、维护规则与推荐更新时机。

目标是让“项目发现层”可持续维护，而不是只靠临时补记路径。

---

## 二、注册表的定位

`projects-registry.json` 是当前版本的轻量项目发现入口。

它不替代：
- 每个项目自己的 `00_恢复入口.md`
- 每个项目自己的索引与决策文件
- workspace 级 `PROJECT_SYSTEM.md`

它负责的是：
- 记录有哪些项目已经被纳入 skill 管理
- 记录这些项目当前应去哪里扫描
- 在用户说“恢复项目 / 进入项目”时，为项目发现提供第一批候选对象

一句话：

> 注册表解决“先找到有哪些项目”，项目内文件解决“进入后怎么恢复”。

---

## 三、建议字段

当前版本建议每个项目至少记录以下字段：

- `name`
  - 项目名称
- `path`
  - 项目目录路径
- `status`
  - 当前状态，例如：`active` / `archived` / `invalid`
- `entryFile`
  - 恢复入口文件名，通常为 `00_恢复入口.md`
- `indexFile`
  - 索引文件名，通常为 `00_文档总索引与当前进度.md`
- `decisionFile`
  - 决策文件名，通常为 `99_关键决策记录.md`
- `registeredAt`
  - 首次登记时间
- `notes`
  - 简要说明该项目当前用途或主线

可选字段：
- `lastValidatedAt`
- `lastRecoveredAt`
- `lastCheckpoint`
- `tags`
- `rootMode`（例如 `project-root` / `subdir-entry`）

---

## 四、什么时候应新增项目登记

以下情况应新增项目到注册表：

1. 用户明确要求把某个长期项目纳入 skill 管理
2. 用户提供了项目路径，并确认这是正式项目目录
3. assistant 已完成项目初始化并确认目录存在
4. assistant 已接管一个已有项目目录并确认结构有效

在这些情况下，不应只完成项目结构初始化，还应同步写入注册表。

---

## 五、什么时候应更新已有项目登记

以下情况应更新已有项目的注册表信息：

### 5.1 项目路径变更

当用户说明：
- 项目搬家了
- 我把目录移动了
- 路径改了

应更新：
- `path`
- 必要时补记 `notes`
- 更新验证时间

### 5.2 项目恢复入口结构调整

当项目内部关键文件改名或结构调整时，应更新：
- `entryFile`
- `indexFile`
- `decisionFile`

### 5.3 项目状态变化

当项目已不再活跃、进入归档、或路径已失效时，应更新：
- `status`

推荐状态：
- `active`：当前正常使用
- `archived`：已归档但仍可参考
- `invalid`：路径失效或结构不可用

### 5.4 项目说明变化

当项目主线明显变化时，可更新：
- `notes`

---

## 六、什么时候不应随便修改注册表

以下情况不应轻易改注册表：

1. 只是普通聊天提到某个项目名
2. 用户没有确认项目路径
3. 当前只是逻辑上按项目讨论，但还未正式纳入管理
4. assistant 只是猜测某个目录可能是项目

原则：

> 注册表是“已确认项目”的名单，不是“疑似项目”的缓存池。

---

## 七、恢复项目时的使用规则

当用户说：
- 恢复项目
- 进入项目
- 回到项目

建议顺序：

1. 读取 `projects-registry.json`
2. 过滤 `status = active` 的项目
3. 逐个检查：
   - 路径是否存在
   - 入口文件是否存在
   - 索引/决策文件是否存在
4. 形成当前可识别项目列表
5. 让用户选择进入哪个项目
6. 选择后，再进入对应项目恢复流程

如果某项目登记存在但路径失效：
- 可暂不从候选列表主展示
- 或带着“路径失效”标签提示用户
- 并建议用户更新路径

---

## 八、反打扰规则

注册表维护不应造成高频打扰。

### 应提示用户的情况
- 首次登记项目时
- 项目路径失效时
- 恢复时找不到可用项目时
- 项目路径与恢复结构不一致时

### 不应反复提示的情况
- 项目已登记且可正常恢复
- 只是普通事务对话
- 刚刚确认过路径且状态未变化

### 单次提醒原则
对同一项目的同一目录问题，默认只提醒一次；
只有在用户再次要求恢复、路径再次校验失败、或项目状态变化时，才再次提醒。

---

## 九、当前版本的限制

1. 当前注册表仍是轻量版本
2. 主要依赖手动维护或在接管/初始化时同步更新
3. 尚未建立更完整的自动校验与清理机制
4. 尚未支持更复杂的多入口项目结构
