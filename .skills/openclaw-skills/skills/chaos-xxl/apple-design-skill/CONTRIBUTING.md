# Contributing to Apple Design Skill

Thank you for your interest in contributing! This guide will help you get started.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Include clear steps to reproduce any bugs
- Provide examples of expected vs actual behavior

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-improvement`)
3. Make your changes
4. Ensure all tests pass (`npm test`)
5. Commit with a clear message (`git commit -m "feat: add new layout pattern"`)
6. Push to your fork (`git push origin feature/my-improvement`)
7. Open a Pull Request

### Commit Message Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` — New feature or prompt module
- `fix:` — Bug fix or correction in prompt content
- `docs:` — Documentation changes
- `test:` — Adding or updating tests
- `chore:` — Maintenance tasks

### What You Can Contribute

- **Prompt improvements** — Refine existing prompt modules for better AI output
- **New layout patterns** — Add more Apple-style layout templates
- **Language support** — Extend font stacks or copywriting rules for additional languages
- **Examples** — Add new usage examples showcasing different page types
- **Tests** — Improve test coverage for prompt content validation
- **Documentation** — Fix typos, improve clarity, add translations

### Development Setup

```bash
# Clone the repository
git clone https://github.com/user/apple-design-skill.git
cd apple-design-skill

# Install test dependencies
npm install

# Run tests
npm test
```

### Guidelines

- Keep prompt content concise and actionable
- Follow the existing file structure and naming conventions
- Ensure design token values stay within the documented ranges
- Test your changes before submitting a PR
- Be respectful and constructive in discussions

## Code of Conduct

Be kind, be respectful, and help make this project welcoming for everyone.

## Questions?

Open an issue or start a discussion — we're happy to help!
