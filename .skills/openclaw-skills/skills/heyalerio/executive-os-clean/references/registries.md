# Registries

Use explicit registries before fallback heuristics.

People buckets:
- `Inner Circle` for people whose messages usually matter immediately
- `Active Threads` for live loops and pending next steps
- `Opportunity People` for meaningful operators, founders, creators, investors, or partners
- `Sensitive Institutions` for banking, legal, government, health, billing, or account-security contacts
- `Watchlist` for entities that become important when certain topics appear

Programs:
- HQ
- Interview / Podcast
- Agent OS / Automation
- Network / Opportunities
- Personal Logistics
- Money / Admin
- Health / Personal Care
- General FYI

Rules:
1. map each thread to one primary program
2. add a secondary program only when it changes the decision
3. avoid defaulting everything to General FYI
