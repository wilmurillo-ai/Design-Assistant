## 快速开始

### 步骤 1：安装技能

```bash
npx clawhub@latest install mind-layer
```

### 步骤 2：配置用户信息

编辑 `USER.md`，填写：

```markdown
# USER.md - 用户画像

- **Name:** 您的名字
- **Timezone:** Asia/Shanghai
- **偏好:** 您喜欢什么沟通方式
```

### 步骤 3：确认核心文件

确保以下文件存在：

```
~/.openclaw/workspace/
├── SOUL.md          # 您的价值观
├── AGENTS.md        # 工作区规则
├── RULES.md         # 沟通规则 ← Mind Layer 提供
├── IDENTITY.md      # AI 身份
├── USER.md          # 用户画像
├── MEMORY.md        # 长期记忆
└── memory/
    └── YYYY-MM-DD.md  # 每日记录
```

## 文件配置

### RULES.md

这是 Mind Layer 的核心文件，包含：

- 四大核心原则
- 记忆优先级规则
- 禁止事项
- 沟通风格指南

**一般不需要修改**，开箱即用。

### MEMORY.md

长期记忆文件，记录：

- 重要决策
- 用户偏好
- 经验教训
- 关键事件

**定期更新**，每次重要交互后回顾。

### memory/daily/*.md

每日记录文件，包含：

- 今日成就
- 经验总结
- 待改进
- 明日计划

**每日创建**，使用日期作为文件名。

## 记忆操作

### 记住重要信息

当用户说"记住这个"：

```markdown
1. 判断：这是什么类型的信息？
   - 事实 → 知识层
   - 经验 → 经验层
   - 偏好 → 用户画像

2. 确定优先级：
   - P0（必须记住）→ 立即写入 MEMORY.md
   - P1（重要）→ 标记情感标签
   - P2（普通）→ 保留在上下文
   - P3（无价值）→ 丢弃

3. 建立联想：
   - 接近关联：同场景、同时间
   - 相似关联：同类目、相似概念
   - 对比关联：差异点、对立面
```

### 提取记忆

当需要回忆信息时：

```markdown
1. 确定需求：需要什么类型的记忆？
2. 并行检索：从各层同时搜索
3. 多通道输出：文字 + Emoji
4. 提取强化：成功后强化联想链
```

## 每日反思

### 自动触发

可在 HEARTBEAT.md 中配置每日反思任务：

```markdown
## 每日小规模反思

1. 检查今日是否有重大成功/失败
2. 记录 1 条值得记住的经验
3. 分析用户的追问（优化沟通）
4. 更新 MEMORY.md
```

### 手动触发

也可以随时要求进行反思：

> "书灵，今天有什么收获？"

## 进阶技巧

### 知识图谱

为每个知识点建立多维关联：

```markdown
# 知识节点示例

## Python
- 接近：JavaScript, Java
- 相似：胶水语言, 脚本语言
- 对比：静态语言, 编译型语言
```

### 情感标签

标记情感强度：

```markdown
# 情感标签示例
- [高兴] 用户获得新技能
- [困惑] 用户遇到问题
- [满意] 用户给予肯定
```

### 遗忘曲线

按时间回顾：

| 时间点 | 动作 |
|--------|------|
| T+0 | 完整存储 |
| T+1天 | 首次回顾 |
| T+7天 | 二次回顾 |
| T+30天 | 评估是否永久 |

## 故障排除

### 问题：不知道信息存哪里

**解决**：按优先级判断

```
用户明确要求记住？ → P0 → MEMORY.md
用户表达强烈情绪？ → P1 → 标记情感
有用但非关键？ → P2 → 上下文
闲聊客套话？ → P3 → 丢弃
```

### 问题：记忆提取失败

**解决**：多通道尝试

```
1. 精确匹配 → 关键词搜索
2. 语义相似 → 联想扩展
3. 承认不确定 → 询问用户
```

### 问题：记忆太多太乱

**解决**：使用精选原则

```
不是所有都要记住
只记：核心、偏好、经验
丢弃：闲聊、一次信息、过时
```
