## 生成完整月度财务汇报

### 第一步：准备数据

1. 收集当月银行流水（CSV 或 XLSX 格式）
2. 如有 GBK 编码的 CSV 文件，技能会自动识别并转换
3. 整理为以下格式：

```
日期,公司名称,对方账户,摘要,金额
2026-03-01,广州公司,宿罡,工资,-30000
2026-03-05,杭州公司,青岛申通达,咨询费,200000
...
```

### 第二步：分析数据

```bash
node index.js ".\2026年3月\广州公司2026年3月流水_utf8.csv"
node index.js ".\2026年3月\杭州公司2026年3月流水_utf8.csv"
```

### 第三步：生成完整财务汇报

使用分析结果，生成包含以下内容的完整报告：

1. **基础信息**：汇报日期、汇报人、期间

2. **财务数据汇总**：
   - 资金总览（期初、本期收入、本期支出、期末）
   - 收入明细（含占比）
   - 支出明细（含占比）
   - 成本费用分析（营业成本 vs 期间费用，刚性 vs 可调）

3. **关键财务指标对比**：两地公司对比

4. **应收/应付及大额收支**：客户、金额、性质

5. **现金流分析**：流入/流出/净现金流

6. **重大事项说明**：各公司事项、关联交易

7. **风险预警**（必须包含）：
   - 🔴 资金流动性风险
   - 🟡 经营风险预警
   - 🟢 财务健康度总结

8. **季度回顾**（如有）：
   - 季度核心数据
   - 各月趋势
   - 主要成就
   - 待改进项

9. **待补充事项**：待办清单

## 数据验证清单

生成报告前，必须验证：

- [ ] 期末余额 = 期初余额 + 收入 - 支出
- [ ] 与上月数据进行衔接
- [ ] 核对大额交易
- [ ] 标注数据来源和缺失项

## 常见问题

### Q: CSV 文件乱码怎么办？
A: 技能会自动检测 GBK 编码并转换，如仍有问题，可手动转换：
```powershell
Get-Content "file.csv" -Encoding Default | Out-File "file_utf8.csv" -Encoding utf8
```

### Q: 如何处理多个月份数据？
A: 逐月分析，然后整合生成季度汇总

### Q: 应收账款如何统计？
A: 需要手动补充合同和往来账数据，银行流水只记录实际收付
