# get-note-sifter — Get笔记筛选整理助手

> 版本：v2.0 | 创建：2026-04-07 | 更新：2026-04-07
> 状态：v2.0 已上线（支持 Obsidian CLI 对接）

---

## 技能定位

解决 Get笔记 → Obsidian 这段路的整理效率问题。

每次分析时，先了解你当前的研究方向（基于研究地图 + Obsidian 实时查询），再做有针对性的推荐。

---

## 核心功能

| 功能 | 触发 |
|------|------|
| Get笔记单篇分析 | 你发来 Get笔记 链接/笔记ID |
| 外部链接分析 | 你发来微信文章/公众号/任意网页链接 |
| 批量扫描报告 | 你说「整理一下」，或每周五定时 |

## Obsidian 对接配置

### 方式一：vaultctl CLI（推荐）

**安装：**
```bash
npm install -g vaultctl
vaultctl config set vault "g:\笔记\喜悦天地"
```

**常用命令：**
```bash
vaultctl search "关键词"                    # 全文搜索
vaultctl search --tag "AI教学"              # 按标签搜索
vaultctl tags list                           # 列出所有标签
vaultctl read "课题研究/066 AI应用成熟度模型/README.md"  # 读取笔记
vaultctl info                                # 库概览
```

**特点：**
- stateless，无依赖，开箱即用
- 专为 AI agent 设计
- 输出 JSON 格式（便于解析）

### 方式二：文件系统直接访问（备选）

当 vaultctl 不可用时，直接读取课题目录：

```
g:\笔记\喜悦天地\课题研究\
├── 066 AI应用成熟度模型/
├── 070 AI教学实战课/
├── 071 AI办公助能力提升/
├── 072 本地化智能体研究/
└── ...
```

## 分析流程（v2.0 增强版）

```
收到内容（Get笔记ID 或 URL）
    ↓
读取笔记/文章内容
    ↓
读取研究地图（memory/research-map.json）
    ↓
用 vaultctl / 文件系统 查询相关课题
    ↓
匹配：这篇文章和你的哪个课题相关？
    ↓
对比已有课题内容：有没有重复？填补了什么空白？
    ↓
输出分析报告
    ↓
（如推荐入选）生成 Obsidian Wiki 格式笔记
    ↓
（如需要）写入 Obsidian 课题文件夹
```

```
📄 笔记标题
━━━━━━━━━━━━━━━━━━━━━━
📅 日期 | 来源：Get笔记

🔍 一句话核心观点

📌 内容摘要
   这篇讲了什么...

🔗 与你的课题关联
   → 匹配课题：「课题名」（匹配度：高/中/低）
   → 补充了：具体补充了什么观点
   → 可能重复：[[课题笔记名]]（观点有重叠）

⭐ 建议操作

   💾 推荐收入 Get笔记？
      ✅ 推荐 / 📌 暂时不 / ❌ 不值得

   ✅ 推荐入选 Obsidian？
      ✅ 推荐 → 建议存放：课题研究/[课题文件夹名]/
      📌 暂时不 → 原因：...
      ❌ 不值得 → 原因：...

📝 附注
   如推荐入选，可直接写成 Obsidian Wiki 格式

## Obsidian Wiki 笔记模板

当推荐入选 Obsidian 时，按以下模板生成笔记写入：

```markdown
# {{笔记标题}}

> 整理自：{{来源类型}} | {{日期}}
> 课题：[[课题文件夹名]]
> 标签：#来源标签1 #来源标签2

---

## 🔍 一句话核心观点

{{用1-2句话总结核心}}

---

## 📌 关键要点

{{用结构化方式列出文章的主要观点}}

- **要点1：** ...
- **要点2：** ...
- **要点3：** ...

---

## 💡 我的启发

{{结合你的实际情况写的启发，不只是摘录原文}}

> 这篇解决了我什么问题/补充了什么认知空白

---

## 🔗 关联笔记

- 与 [[课题内已有笔记A]] — 关联说明
- 与 [[课题内已有笔记B]] — 关联说明

---

## 📋 待实践/待补充

- [ ] 待实践：{{从这篇文章学到的，下一步可以做什么}}
- [ ] 待补充：{{这篇文章没讲清楚，还需要了解什么}}

---

*整理：小暖阳 | {{YYYY-MM-DD}}*
```

### 字段说明

| 字段 | 来源 | 说明 |
|------|------|------|
| `{{笔记标题}}` | 文章标题 | 直接引用 |
| `{{来源类型}}` | 微信文章/公众号/Get笔记 | 标注来源 |
| `{{日期}}` | 文章日期 | 标注时效性 |
| `{{课题文件夹名}}` | 研究地图匹配结果 | Obsidian 内的文件夹路径 |
| `{{一句话核心观点}}` | AI 总结 | 不是原文摘录，是重新理解后的一句话 |
| `{{关键要点}}` | AI 提取 | 结构化整理，不要大段复制原文 |
| `{{我的启发}}` | AI 结合你的研究课题写 | 这是最有价值的部分 |
| `{{关联笔记}}` | vaultctl 查询 + 研究地图 | 主动建立双向链接 |
| `{{待实践/待补充}}` | AI 判断 | 这篇文章值得延伸的方向 |

## 外部链接分析（额外输出）

当收到外部链接时，分析报告额外包含：

```
🔗 来源类型：微信文章 / 公众号 / 网页

💾 收入 Get笔记 推荐度
   ★★★☆☆ → 理由

✅ 入选 Obsidian 推荐度
   ★★★☆☆ → 理由

📋 建议行动
   → 如果两条都推荐：直接写成 Wiki 格式存 Obsidian
   → 如果只推荐 Get笔记：告知你手动保存
   → 如果都不推荐：告知原因
```
