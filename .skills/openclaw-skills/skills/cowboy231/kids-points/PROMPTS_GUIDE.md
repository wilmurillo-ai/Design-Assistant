## 🎯 提示设计原则

1. **不强制** - 文字功能可以直接用，语音功能可选
2. **友好引导** - 说明好处，不强迫配置
3. **强调免费** - 消除用户顾虑
4. **简单步骤** - 3-4 步快速配置
5. **多场景提示** - 不同场景不同文案
6. **正面激励** - 配置后体验更好

## 🔧 实现细节

### API Key 检查函数

```javascript
function checkSenseApiKey() {
  try {
    const configPath = path.join(process.env.HOME, '.openclaw/openclaw.json');
    const config = JSON.parse(fs.readFileSync(configPath, 'utf8'));
    const apiKey = config.env?.SENSE_API_KEY;
    return apiKey && apiKey.length > 10;
  } catch (e) {
    return false;
  }
}
```

## 🎨 文案风格

### ✅ 使用正面语言

```
✅ 好："配置后可解锁语音功能，目前基本免费！"
❌ 差："你没有 API Key，语音功能不可用"
```

### ✅ 强调免费

```
✅ 好："注册即送免费额度，个人使用完全足够"
❌ 差："需要申请 API Key"
```

✅ 好：3-4 步清晰指引 + 链接
