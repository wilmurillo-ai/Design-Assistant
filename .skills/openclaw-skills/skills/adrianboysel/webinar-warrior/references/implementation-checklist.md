# Implementation Checklist
## Pre-Launch Through Post-Webinar Close

---

## Critical Path Rule

Test the full funnel yourself before spending a single dollar on traffic.
Self-register → check confirmation email → visit confirmation page → attempt order bump purchase → verify redirect → check thank you page → verify email delivery sequence.
If anything is broken at this stage it will be broken for every registrant.

---

## Pre-Launch (Days 1–2 Before Traffic)

### Pages and Tech
- [ ] Registration page live and converting (test on mobile and desktop)
- [ ] Confirmation page live with order bump visible
- [ ] Thank you page (post-order-bump) live
- [ ] Main offer page live with correct pricing
- [ ] Offer page redirects / closes at stated deadline (test the mechanism)
- [ ] Stripe / payment processor connected and tested with real $1 charge
- [ ] Webinar platform configured (Zoom / StreamYard / WebinarJam)
- [ ] Webinar link generated and tested
- [ ] Calendar .ics file created for "Add to Calendar" button

### Email System
- [ ] Sequence A loaded in CRM (A1, A2, A3, A4)
- [ ] Sequence B loaded and tagged to trigger on replay purchase
- [ ] Sequence C loaded and tagged to trigger on non-buyer post-webinar
- [ ] All sequences tested via self-registration
- [ ] Webinar link inserted into A3 and A4
- [ ] Order bump links correct in B1 and C1
- [ ] Main offer links correct in all post-webinar emails
- [ ] Deadline dates/times correct and match actual close

### Tracking
- [ ] Pixel installed on registration page
- [ ] Pixel installed on confirmation page (purchase event)
- [ ] Pixel installed on main offer thank you page
- [ ] UTM parameters on all ad links
- [ ] Conversion tracking verified (test conversion fires correctly)

### Assets
- [ ] Presenter headshot uploaded to registration page
- [ ] Testimonials / social proof uploaded to registration page
- [ ] Slide deck complete and tested on presentation device
- [ ] Backup slide deck saved to cloud (not just local)
- [ ] Any promised bonus files ready to deliver

## Launch (Days 1–3 of Campaign)

### Traffic
- [ ] Email list announcement sent (Day 1)
- [ ] Paid ads launched — all 5 variations active ($30/day test)
- [ ] Organic amplifiers briefed with content calendar and posting guidelines
- [ ] Daily theme posts distributed to amplifiers
- [ ] Amplifiers posting minimum 1 time per day confirmed

### Monitoring (Daily)
- [ ] Registration count vs. projection
- [ ] Cost per registration (pause if above $3.00)
- [ ] Ad CTR by variation (cut bottom 3 by Day 4)
- [ ] Email open rates and click rates
- [ ] Order bump conversion rate on confirmation page

## Scale (Days 4–7)

- [ ] Bottom 3 ad variations paused
- [ ] Top 2 ad variations scaled ($50/day)
- [ ] Retargeting layer launched (visited page, did not register)
- [ ] Mid-campaign email to list (value email + registration push)
- [ ] Daily amplifier theme posts confirmed

## Final Push (Days 8–10)

- [ ] Final 2 ad variations at push budget ($37/day)
- [ ] Day-before email sent (A2)
- [ ] Final organic push: "Tomorrow we go live"
- [ ] Slide deck loaded on presentation device the night before
- [ ] Water, environment, tech tested night before

## Webinar Day

### Pre-Show (60 Minutes Before)
- [ ] A3 sent (1-hour reminder with live link)
- [ ] Presentation device + backup confirmed
- [ ] Internet connection tested (wired if possible)
- [ ] Camera and lighting checked
- [ ] Audio checked (use a dedicated mic — not laptop built-in)
- [ ] Second monitor or notes device ready (speaker notes visible)
- [ ] Water bottle on desk
- [ ] Phone on silent and face-down
- [ ] Slack, email, notifications all closed
- [ ] Team member or VA on standby to monitor chat

### 10 Minutes Before
- [ ] A4 sent (10-minute reminder)
- [ ] Webinar platform open and presenter admitted
- [ ] Start looping pre-show energy (Brunson method: energy + countdown + teaser)
- [ ] Chat monitored by team or yourself for early questions

### During Webinar
- [ ] Start on time. Not 5 minutes late. On time.
- [ ] Follow delivery structure — do not deviate to "add more value"
- [ ] Trial close every 60–90 seconds
- [ ] Commitment questions after each mechanism
- [ ] Stack before price reveal
- [ ] If-all statements before price
- [ ] Start countdown clock after pitch
- [ ] Run 30-minute Q&A with pre-loaded questions
- [ ] Restate CTA every 5–7 minutes during Q&A window
- [ ] Export all Q&A before closing the webinar

### Post-Webinar (Within 30 Minutes)
- [ ] B1 triggered to replay purchasers
- [ ] C1 triggered to non-buyers
- [ ] Q&A exported and saved
- [ ] Registration count, show-up count, and live sales recorded
- [ ] Initial conversion rate calculated

### Q&A Analysis (Most Important Step)

Export all questions from the webinar chat.
Categorize by type:
1. Content questions (something wasn't clear — fix the slide)
2. Objection questions (reason not to buy — add to close)
3. Proof questions (they need more evidence — add case study)
4. Technical questions (platform / access / process — FAQ on offer page)

**Rebuild slides based on real questions before next run.**
This is how Brunson went from $30,000 → $120,000 → $150,000 in three consecutive webinars.
The questions are the roadmap to a better next run.

### Next Run Decision

If EPR > $5.00 → Scale aggressively. Double ad spend.
If EPR $2.00–$5.00 → Optimize and rerun. Fix top 3 Q&A issues first.
If EPR < $2.00 → Diagnose before scaling. Check: show-up rate / offer / positioning.

Rerun within 2 weeks while audience is warm and interested.
Call it an "Encore" — never a "Replay" — to maintain live energy positioning.
