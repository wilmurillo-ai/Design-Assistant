# 快速定制指南

## 5 分钟快速定制

### 步骤 1: 创建项目 (1 分钟)

告诉技能：

> "帮我基于 quecpython-modbus-iot 技能创建一个新项目，项目名称是 xxx"

技能会自动复制模板到你的工作区。

---

### 步骤 2: 修改核心配置 (2 分钟)

编辑 `config/config.py`，**必须修改以下 5 个参数**：

```python
# 1. 项目名称
PROJECT_NAME = "your-project-name"

# 2. 项目版本
PROJECT_VERSION = "1.0.0"

# 3. 数据上报平台地址 (必须修改！)
URL_REPORT = "https://your-platform.com/api/data"

# 4. OTA 升级地址 (必须修改！)
URL_OTA = "https://your-ota-server.com/ota"

# 5. 上报频率 (秒)
REPORT_INTERVAL_SEC = 60  # 60 秒 = 1 分钟
```

---

### 步骤 3: 修改硬件配置 (1 分钟)

根据你的硬件修改：

```python
# Modbus 波特率 (根据传感器)
MODBUS_BAUDRATE = 9600

# LED 引脚 (根据硬件设计)
LED_NET_PIN = 36
LED_MODBUS_PIN = 44

# 传感器数量
SENSOR_MAX_COUNT = 8
```

---

### 步骤 4: 部署测试 (1 分钟)

将以下 8 个文件上传到设备 `/usr` 目录：

```
config.py
main.py
led_control.py
modbus_rtu.py
sensor_device.py
network_manager.py
system_manager.py
data_reporter.py
```

运行测试：

```python
import main
main.main()
```

### 场景 6: 修改传感器协议

**需求**: 传感器状态码不同

**修改** `src/sensor_device.py` 中的 `read_status()` 方法：

```python
def read_status(self):
    result = self.modbus.read_registers(1, 2000 + self.addr, 1)
    status_value = result['data'][0]
    
    # 根据你的传感器协议修改
    if status_value == 0:
        # 正常
        pass
    elif status_value == 1:
        # 你的自定义状态码
        self.level_1_alarm = 1
    # ... 更多状态
```

## 常见问题

### Q: 不知道平台地址是什么？
**A**: 向客户索要 API 接口文档，里面会有上报地址

### Q: 传感器读取失败？
**A**: 检查波特率、RS485 接线 (A/B 极)

### Q: 网络连不上？
**A**: 检查 SIM 卡是否欠费，天线是否连接

### Q: 数据格式不对？
**A**: 修改 `data_reporter.py` 中的 JSON 构建逻辑

### Q: 如何添加新功能？
**A**: 在 `main.py` 的主循环中添加，或创建新模块
