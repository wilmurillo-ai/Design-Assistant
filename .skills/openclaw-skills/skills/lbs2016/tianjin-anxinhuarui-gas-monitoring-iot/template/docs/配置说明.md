# 配置说明

## 配置文件位置

`config/config.py` - 所有参数集中管理

---

## 参数分类

### 1. Modbus 通信参数

```python
MODBUS_UART = 2              # UART 端口号
```
- **说明**: 使用的 UART 端口
- **可选值**: 1, 2, 3 (根据硬件引脚确定)
- **默认值**: 2

---

```python
MODBUS_BAUDRATE = 9600       # 波特率
```
- **说明**: Modbus 通信波特率
- **可选值**: 2400, 4800, 9600, 19200, 38400, 115200
- **默认值**: 9600
- **注意**: 必须与传感器波特率一致

---

```python
MODBUS_DATABITS = 8          # 数据位
```
- **说明**: Modbus 数据位
- **默认值**: 8 (固定)

---

```python
MODBUS_PARITY = 0            # 校验位
```
- **说明**: Modbus 校验方式
- **可选值**: 0=无校验，1=奇校验，2=偶校验
- **默认值**: 0

---

```python
MODBUS_STOPBIT = 1           # 停止位
```
- **说明**: Modbus 停止位
- **可选值**: 1, 2
- **默认值**: 1

---

```python
MODBUS_FLOWCTL = 0           # 流控制
```
- **说明**: 硬件流控制
- **可选值**: 0=无流控，1=硬件流控
- **默认值**: 0

---

### 2. 传感器参数

```python
SENSOR_BASE_ADDR = 1000      # 浓度数据起始地址
```
- **说明**: 传感器浓度寄存器起始地址
- **默认值**: 1000
- **注意**: 根据传感器协议修改

---

```python
SENSOR_STAT_ADDR = 2000      # 状态数据起始地址
```
- **说明**: 传感器状态寄存器起始地址
- **默认值**: 2000

---

```python
SENSOR_MAX_COUNT = 32        # 最大传感器数量
```
- **说明**: 支持的最大传感器数量
- **默认值**: 32
- **范围**: 1-32

---

```python
SENSOR_SLAVE_ADDR = 1        # Modbus 从站地址
```
- **说明**: Modbus 从站地址
- **默认值**: 1
- **范围**: 1-247

---

### 3. 数据上报平台

```python
URL_OTA = "https://..."      # OTA 升级地址
```
- **说明**: 固件升级服务器地址
- **默认值**: 天津安信华瑞 OTA 服务器
- **注意**: 修改为你的平台地址

---

```python
URL_REPORT = "https://..."   # 数据上报地址
```
- **说明**: 传感器数据上报地址
- **默认值**: 天津安信华瑞平台
- **注意**: 修改为你的平台地址

---

### 4. 上报频率控制

```python
REPORT_INTERVAL_SEC = 60     # 上报间隔 (秒)
```
- **说明**: 数据上报时间间隔
- **默认值**: 60 (1 分钟)
- **建议**: 最小 30 秒，最大 3600 秒
- **注意**: 影响流量消耗

---

```python
NETWORK_CHECK_INTERVAL = 1   # 网络检查间隔 (小时)
```
- **说明**: 多久检查一次网络连接
- **默认值**: 1 小时

---

### 5. LED 指示灯引脚


---

```python
LED_MODBUS_PIN = 44          # Modbus 通信 LED 引脚
```
- **说明**: Modbus 通信指示灯 GPIO 编号
- **默认值**: 44

### 7. 看门狗设置

```python
WDT_TIMEOUT_MS = 320000      # 看门狗超时时间 (毫秒)
```

### 8. 重试次数限制

- **默认值**: 5

---

```python
HTTP_RETRY_MAX = 2           # HTTP 上报最大重试次数
```
- **说明**: HTTP 上报失败重试次数
- **默认值**: 2

---

```python
NETWORK_RETRY_MAX = 5        # 网络失败重试次数
```
- **说明**: 网络连接失败重试次数
- **默认值**: 5


- **说明**: 传感器扫描失败重试次数

### 9. 调试开关

```python
DEBUG_ENABLE = True          # 调试模式总开关
```
- **说明**: 启用/禁用调试输出
- **可选值**: True, False
- **默认值**: True
- **注意**: 生产环境建议设为 False

---

```python

```
- **说明**: 是否输出 Modbus 详细日志
- **可选值**: True, False

---

```python
DEBUG_NETWORK = False        # 网络调试开关
```
- **说明**: 是否输出网络详细日志

---

```python
DEBUG_HTTP = False           # HTTP 调试开关
```

## 常用配置场景

### 场景 1: 修改波特率

传感器波特率为 19200：

```python
MODBUS_BAUDRATE = 19200
```

### 场景 2: 修改上报间隔

每 5 分钟上报一次：

```python
REPORT_INTERVAL_SEC = 300
```

### 场景 3: 修改平台地址

```python
URL_REPORT = "https://my-platform.com/api/upload"
URL_OTA = "https://my-ota.com/firmware"
```

### 场景 4: 修改传感器数量

只使用 8 个传感器：

```python
SENSOR_MAX_COUNT = 8
```

### 场景 5: 生产环境配置

```python
DEBUG_ENABLE = False         # 关闭调试输出
DEBUG_MODBUS = False
DEBUG_NETWORK = False
DEBUG_HTTP = False
```

## 配置检查清单

部署前请确认：

- [ ] URL_REPORT 已修改为实际平台地址
- [ ] URL_OTA 已修改为实际 OTA 地址
- [ ] MODBUS_BAUDRATE 与传感器一致
- [ ] LED_NET_PIN 和 LED_MODBUS_PIN 与硬件一致
- [ ] PROJECT_VERSION 已更新为当前版本
- [ ] DEBUG_ENABLE 在生产环境设为 False
