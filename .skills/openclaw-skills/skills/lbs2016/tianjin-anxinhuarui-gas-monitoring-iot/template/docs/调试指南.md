### 2. Modbus 模块调试

```python
from modbus_rtu import ModbusRTU

# 创建 Modbus 实例
modbus = ModbusRTU(2, 9600, 8, 0, 1, 0)

# 测试读取电池状态 (地址 0)
print("读取电池状态...")
result = modbus.read_registers(1, 0, 1)
print("结果:", result)

# 测试读取传感器状态 (地址 2000)
print("读取传感器状态...")
result = modbus.read_registers(1, 2000, 1)
print("结果:", result)

# 测试读取传感器浓度 (地址 1000)
print("读取传感器浓度...")
result = modbus.read_registers(1, 1000, 1)
print("结果:", result)
```

**预期结果**:
```
{'error': 0, 'data': [xxx]}
```

**常见错误**:
- `error: 1` - 超时，检查接线
- `error: 2` - 异常响应，检查从站地址
- `error: 3` - 长度错误，检查寄存器地址

## 常见错误排查

### 错误 1: ModuleNotFoundError

```
ModuleNotFoundError: no module named 'xxx'
```

**原因**: 模块文件未上传或路径错误

**解决**:
1. 检查文件是否在 `/usr` 目录
2. 使用 `import os; print(os.listdir('/usr'))` 确认

---

### 错误 2: Modbus 超时

```
{'error': 1, 'data': []}
```

**原因**: 
- RS485 接线错误
- 波特率不匹配
- 从站地址错误

**解决**:
1. 检查 A/B 线是否接反
2. 确认波特率与传感器一致
3. 使用 Modbus Poll 等工具测试

---

### 错误 3: 网络注册失败

```
网络连接失败
```

**原因**:
- SIM 卡欠费
- 信号弱
- APN 设置错误

**解决**:
1. 检查 SIM 卡状态
2. 检查天线连接
3. 查看信号强度 `net.csqQueryPoll()`

---

### 错误 4: HTTP 上报失败

```
HTTP 错误，状态码：404
```

**原因**: URL 地址错误

**解决**: 检查 `config.py` 中的 URL_REPORT

---

### 错误 5: 看门狗重启

```
设备自动重启
```

**原因**: 主循环超时，未及时喂狗

**解决**:
1. 检查是否有死循环
2. 减少 Modbus 重试次数
3. 缩短网络超时时间
