## 核心理念

**不要只记录交易，要从交易中学习。**

大多数交易者犯同一个错误：亏钱了不开心，赚钱了开心，但都没有记录为什么。
这个系统的核心是：**每次交易都有完整的上下文，每次复盘都有可操作的结论。**

## 分析维度

### 1. 按策略分析
- 哪种策略胜率最高？
- 哪种策略平均盈亏比最大？
- 哪种策略在特定市场环境下表现最好？

### 2. 按市场环境分析
- 牛市：趋势跟踪策略表现好
- 熊市：超跌反弹策略表现好
- 震荡：网格/区间策略表现好

### 3. 按信号组合分析
- RSI + 成交量 = 胜率多少？
- MACD金叉 + RSI < 40 = 胜率多少？
- 布林下轨 + RSI超卖 = 胜率多少？

### 4. 按时间分析
- 哪个时间段交易表现最好？
- 持仓多久止盈/止损最多？

## 自动规则更新流程

```
每次交易后：
1. 记录完整上下文 → 保存到 data/trades.json
2. 每周分析一次 → 生成规则更新建议
3. 如果某规则连续3次失败 → 标记为"待验证"
4. 如果某规则连续5次成功 → 加入核心规则库

规则分级：
- 🔴 核心规则：连续10次以上验证
- 🟡 备用规则：连续5次以上验证
- ⚪ 待验证规则：少于5次
```

## 在小绿瓶体系中应用

### 立即可用的改动

1. **修改 auto_trade.py 的日志格式**
   - 增加 `context` 字段（技术指标值、市场环境、新闻事件）
   - 增加 `entry_reason`（为什么入场）
   - 增加 `lesson`（从中学到什么）

2. **每周一自动运行分析**
   ```bash
   python3 scripts/analyze_trades.py
   ```
   输出：
   - 胜率最高的策略 Top3
   - 最赚钱的信号组合
   - 需要改进的策略
   - 建议删除的规则

3. **更新 trading_rules.md**
   - 根据分析结果调整止损/止盈参数
   - 根据市场环境调整策略权重

## 关键指标追踪

| 指标 | 公式 | 目标 |
|------|------|------|
| 胜率 | 盈利交易数 / 总交易数 | > 50% |
| 盈亏比 | 平均盈利 / 平均亏损 | > 1.5 |
| 夏普比 | (平均收益 - 无风险利率) / 收益标准差 | > 1 |
| 最大回撤 | 历史最大账户回落 | < 10% |
| 交易频率 | 每月交易数 | 根据策略定 |
