# 电商平台适配指南

本 skill 支持主流电商平台。向用户确认平台后，按对应配置采集数据。
页面改版频繁，若选择器失效先 `browser_snapshot` 获取当前结构再调整。

## 平台速查

| 平台 | 用户中心 | 订单列表 | 商家类型 |
|------|----------|----------|----------|
| 淘宝/天猫 | `https://i.taobao.com/` | `https://buyertrade.taobao.com/trade/itemlist/list_bought_items.htm` | 第三方商家 |
| 拼多多 | `https://mobile.yangkeduo.com/personal.html` | `https://mobile.yangkeduo.com/order.html` | 第三方商家 |
| 抖音商城 | `https://www.douyin.com/user/self` | 抖音 App 内「我的订单」 | 第三方商家 |
| 唯品会 | `https://www.vip.com/` → 个人中心 | 个人中心 → 我的订单 | 自营+第三方 |
| 苏宁 | `https://order.suning.com/` | `https://order.suning.com/order/orderList.do` | 自营+第三方 |
| 小红书 | App 内「我」→ 订单 | 仅 App 内查看 | 第三方商家 |

> 部分平台（拼多多、抖音、小红书）的订单管理主要在 App 内，
> 浏览器版功能有限。此时引导用户手动截图，skill 负责整理和生成文书。

## 通用采集策略

不论平台，需要采集的核心数据一致：

| 数据 | 必采 | 说明 |
|------|------|------|
| 订单详情页 | ✓ | 订单号、下单时间、金额、商品明细 |
| 商品快照/商品页 | ✓ | 下单时的描述、价格、卖家承诺 |
| 物流记录 | ✓ | 发货时间、签收记录、运单号 |
| 售后/退款记录 | 按需 | 已有的售后申请和处理结果 |
| 客服聊天记录 | 按需 | 与商家/平台客服的沟通（承诺和拒绝） |
| 商品评价 | 按需 | 其他买家的差评（证明问题普遍性） |
| 商家资质 | 按需 | 营业执照、经营者信息（起诉时需要） |

## 各平台元素定位

定位优先级：语义文本匹配 > aria 属性 > class 名 > 结构定位

### 淘宝/天猫

| 元素 | 定位策略 | 说明 |
|------|----------|------|
| 订单号 | 文本匹配 `订单号` 相邻元素 | — |
| 金额 | 文本匹配 `实付款` / `订单金额` | — |
| 商品快照 | 订单详情页内商品标题链接 | 淘宝快照有时效，尽早采集 |
| 物流 | 订单详情页「查看物流」 | — |
| 商家信息 | 店铺首页 → 资质详情 | 营业执照页面 |
| 售后入口 | 文本匹配 `退款` / `退货退款` | — |
| 聊天记录 | `https://www.taobao.com/webww/` 或千牛消息 | 网页版消息有限 |

### 拼多多

| 元素 | 定位策略 | 说明 |
|------|----------|------|
| 订单详情 | 移动端页面，元素较简单 | class 命名不稳定 |
| 商家信息 | 订单详情页「联系商家」旁的店铺入口 | — |
| 聊天记录 | 仅 App 内可查看 | 引导用户截图 |

> 拼多多网页版功能受限，优先引导用户在 App 内截图，
> skill 负责整理证据和生成文书。

### 通用元素定位备忘

| 元素 | 通用策略 |
|------|----------|
| 订单号 | 文本匹配 `订单号` / `Order ID` / `订单编号` 相邻文本 |
| 金额 | 文本匹配 `实付` / `应付` / `合计` / `总价` 后的数字 |
| 翻页 | 文本匹配 `下一页` / `>` / `.next` |
| 关闭弹窗 | `[class*="close"]` / `[aria-label="close"]` / Escape |
| 登录检测 | URL 包含 `login` / `passport` / `signin` |

## 平台经营者信息查询

起诉或投诉时需要被投诉方的工商信息：

| 查询方式 | URL | 说明 |
|----------|-----|------|
| 国家企业信用信息公示 | `https://www.gsxt.gov.cn/` | 官方权威 |
| 天眼查 | `https://www.tianyancha.com/` | 信息全面 |
| 企查查 | `https://www.qcc.com/` | 信息全面 |

查询所需信息：平台内的店铺名称、营业执照上的公司全称。

## App-Only 平台的处理策略

部分平台无法通过浏览器自动化采集：

1. 引导用户在 App 内逐页截图
2. 用户将截图保存到约定目录
3. Skill 读取截图，整理为证据清单
4. 基于用户口述的事实经过生成文书
