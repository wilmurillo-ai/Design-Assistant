# Gap Taxonomy

Use this file to identify the main reason the learner is stuck.

## 1. Prerequisite gap

**Signal**
- The user jumps into advanced material without core concepts.
- They ask surface questions because the base layer is weak.

**Say**
- "You are reading ahead of your foundation."

**Repair**
- Name the 2-4 prerequisite topics in order.
- Tell the user to stop rereading the advanced source until those are covered.

## 2. Terminology gap

**Signal**
- The user gets blocked by notation, jargon, or compressed phrasing.
- They can follow the story but not the symbols.

**Say**
- "Your bottleneck is vocabulary and notation, not the big idea."

**Repair**
- Build a symbol sheet.
- Translate each key term into plain language and one worked example.

## 3. Math / probability gap

**Signal**
- The user understands the architecture story but breaks on the objective or derivation.
- They can describe steps but not justify them.

**Say**
- "You do not have enough mathematical footing for this section yet."

**Repair**
- Name the missing math explicitly.
- Assign short derivation drills.
- Return to the paper only after the drills.

## 4. Architecture intuition gap

**Signal**
- The user memorizes components but cannot explain why the mechanism helps.
- They repeat QKV, latent space, residuals, etc. without causal understanding.

**Say**
- "You have component recognition without mechanism-level understanding."

**Repair**
- Ask for before/after comparisons.
- Force explanation of what failure the component fixes.
- Use toy examples.

## 5. Problem-framing gap

**Signal**
- The user dives into methods before understanding the task, constraints, baseline, or evaluation target.

**Say**
- "You are reading the solution before you understand the problem."

**Repair**
- Restate the task, input/output, failure mode, baseline, and success metric first.

## 6. Experimental reasoning gap

**Signal**
- The user reads tables and plots but cannot say what the evidence proves.

**Say**
- "You are consuming results as decoration instead of evidence."

**Repair**
- Ask what each ablation isolates.
- Ask what claim would fail without that result.

## 7. Reading method gap

**Signal**
- The user rereads endlessly, highlights everything, or confuses familiarity with mastery.

**Say**
- "Your reading loop is inefficient. More rereads will not fix a structural gap."

**Repair**
- Switch to a pass structure: problem → method skeleton → key equations → evidence → reread.
- Force retrieval and self-explanation.

## 8. Fake-fluency gap

**Signal**
- The user can repeat words from the source but cannot answer simple transfer questions.

**Say**
- "You can recite the text, but you cannot reason with it yet."

**Repair**
- Ask for paraphrase, comparison, and application.
- Require one original example and one failure case.

## 9. Practice gap

**Signal**
- The user keeps reading but never solves, derives, implements, or explains.

**Say**
- "You are trying to think your way to skill without reps."

**Repair**
- Give drills.
- Define a concrete pass/fail test.
