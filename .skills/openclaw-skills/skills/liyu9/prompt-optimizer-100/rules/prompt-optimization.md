# Prompt 优化规则（V3.0）

> 核心原则：默认隐式优化 + 直接执行，无需确认

---

## 需求分级标准（L1-L4）

| 等级 | 判断标准 | 处理方式 | 示例 |
|------|----------|----------|------|
| **L1 简单** | 单一步骤、明确、低风险 | 隐式优化 → 直接执行 | "写一篇 300 字文章" |
| **L2 中等** | 多步骤、需分析、中风险 | 隐式优化 → 展示思路 → 执行 | "调研一下竞品" |
| **L3 复杂** | 涉及决策、对比、高风险 | 隐式优化 → 多 Agent 并行 → 对比 | "技术选型方案" |
| **L4 关键** | 重要交付、对外发布、高风险 | 显式优化 → 确认 → 执行 | "生成客户方案 PPT" |

**关键词判断规则**：
- **L4**：包含"客户"、"发布"、"对外"、"重要"、"PPT"
- **L3**：包含"对比"、"选型"、"评审"、"架构"、"设计"
- **L2**：包含"调研"、"分析"、"规划"、"总结"
- **L1**：默认

---

## Prompt 优化规则（默认模式）

**默认流程**（用户未明确要求确认时）：
```
用户输入 → 后台优化 prompt → 直接执行 → 返回结果（L2-L3 展示优化思路）
```

**确认模式**（用户明确要求时）：
```
用户输入 → 展示优化 prompt → 用户确认 → 执行
```

**触发确认的关键词**：
- "先确认再执行"
- "优化后给我看"
- "确认一下"
- "方案评审"

---

## 必须展示优化思路的任务类型

| 关键词 | 等级 | 操作 |
|--------|------|------|
| 包含"方案"、"设计"、"梳理"、"规划" | L2-L3 | 回复开头展示🎯优化思路 |
| 包含"对比"、"选型"、"评审" | L3 | 多 Agent 并行 + 对比表格 |
| 包含"客户"、"发布"、"对外" | L4 | 显式确认 |

---

## 角色使用规则

| 等级 | 是否加角色 | 示例角色 |
|------|-----------|----------|
| L1 简单 | ❌ 不加 | - |
| L2 调研/分析 | ✅ 加 | 产品经理/分析师 |
| L3 方案/选型 | ✅ 加 | 架构师/顾问 |
| L4 交付 | ✅ 加 | 资深顾问 |

**角色模板**：
```
你是 {年限} 年 {领域} 的{具体角色}，擅长{具体方向}
```

**示例**：
- "你是 5 年 B 端产品经理，擅长 CRM 设计"
- "你是资深技术架构师，擅长高并发系统设计"
- "你是行业分析师，擅长竞品调研"

---

## 执行模式

**模式 A：隐式高效**（L1 默认）
- 用户输入 → 后台优化 → 直接执行 → 返回

**模式 B：显式确认**（用户明确要求时）
- 用户输入 → 优化方案 → 确认 → 执行

**模式 C：对比模式**（选型/评审）
- 用户输入 → 多 Agent 并行 → 对比分析 → 交付

---

## 自动判断规则

| 触发词/场景 | 自动模式 |
|------------|---------|
| "写一篇"、"总结一下"、"翻译" | 模式 A |
| "调研"、"分析"、"设计" | 模式 A（默认直接执行，但展示优化思路） |
| "对比"、"选型"、"评审"、"哪个更好" | 模式 C |
| 包含"重要"、"客户"、"发布" | 模式 B（显式确认） |
| 明确要求"先确认" | 模式 B |

```
□ 1. 是否已判断任务等级（L1-L4）？

□ 3. L4 是否准备了确认方案？
□ 4. 是否使用了标准化框架模板？

```

## Badcase 处理流程

```
收集 Badcase → 分类标注 → 共性分析 → 针对性优化 → 回归验证 → 更新记忆
```

**收集渠道**：
- 用户指正（如"这次没优化提示词"）
- 主动发现（输出质量不达标）
- 定期回顾（每周检查合规率）
