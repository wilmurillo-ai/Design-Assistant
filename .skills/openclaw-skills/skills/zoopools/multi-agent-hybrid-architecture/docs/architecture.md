# 架构设计详解

> 🖤 混合层级隔离架构 1.0 - 设计理念与技术细节

## 设计背景

### 问题陈述

在多 Agent 协作场景中，常见以下问题：

1. **职责混乱**：多个 Agent 都能执行相同任务，导致推诿或重复
2. **权限失控**：敏感操作（如图片生成、发布）缺乏审查
3. **配置耦合**：行为约束硬编码到配置文件，难以维护和升级
4. **缺乏灵活性**：紧急情况下流程僵化，无法快速响应

### 设计目标

1. **职责清晰**：每个 Agent 有明确的职责边界
2. **权限隔离**：敏感操作需要审查和批准
3. **配置纯净**：不修改核心配置文件，保持可升级性
4. **灵活高效**：支持层级流转和直接调用两种模式

## 核心概念

### 混合层级（Hybrid Hierarchy）

```
传统层级：          混合层级：
                    
哥哥                 哥哥
 │                  ╱   ╲
 │                 ╱     ╲
 ▼                ▼       ▼
墨墨              墨墨    小媒
 │                 ╲     ╱
 ▼                  ╲   ╱
小媒                 小媒

僵化、单一路径       灵活、多路径选择
```

**混合层级的优势**：
- 保留层级管理的秩序
- 支持扁平化的高效沟通
- 根据任务类型动态选择路径
- 紧急情况可 bypass 层级

### 物理隔离（Physical Isolation）

```
┌─────────────────────┐   ┌─────────────────────┐
│   Writer Agent      │   │   Media Agent       │
│   ~/agents/writer/  │   │   ~/agents/media/   │
│                     │   │                     │
│ ┌─────────────────┐ │   │ ┌─────────────────┐ │
│ │ SOUL.md         │ │   │ │ SOUL.md         │ │
│ │ (墨墨人格)       │ │   │ │ (小媒人格)       │ │
│ └─────────────────┘ │   │ └─────────────────┘ │
│                     │   │                     │
│ ┌─────────────────┐ │   │ ┌─────────────────┐ │
│ │ skills/         │ │   │ │ skills/         │ │
│ │ (软链接共享)     │ │   │ │ (软链接共享)     │ │
│ └─────────────────┘ │   │ └─────────────────┘ │
└─────────────────────┘   └─────────────────────┘
```

**物理隔离的实现**：
- 每个 Agent 独立目录
- 专属配置文件（SOUL.md）
- 技能通过软链接共享
- 目录权限独立控制

### 逻辑隔离（Logical Isolation）

```
┌─────────────────────────────────────────┐
│           SOUL.md 注入层                │
│                                         │
│  Writer SOUL:                           │
│  - 禁止 baoyu-* 技能                    │
│  - 负责协调、审查                       │
│                                         │
│  Media SOUL:                            │
│  - 独占 baoyu-* 技能                    │
│  - 负责创意执行                         │
└─────────────────────────────────────────┘
              ▼
┌─────────────────────────────────────────┐
│         openclaw.json (不变)            │
│                                         │
│  - 保持标准格式                         │
│  - 可正常升级                           │
│  - 无需定制修改                         │
└─────────────────────────────────────────┘
```

**逻辑隔离的优势**：
- 不修改 openclaw.json
- 行为约束在 SOUL.md 中
- 升级时只需替换文件
- 配置清晰、易维护

## 架构组件

### 1. Agent 角色定义

#### 墨墨 (Writer)

**定位**：首席协调员

**职责**：
- 任务接收与分解
- 协调与指派
- 质量审查
- 知识管理
- 用户沟通

**限制**：
- ❌ 禁止执行 baoyu-* 技能
- ❌ 不直接执行创意任务
- ❌ 不 bypass 审查流程

**权限**：
- ✅ 任务分配权
- ✅ 质量否决权
- ✅ 知识管理权

#### 小媒 (Media)

**定位**：创意专家

**职责**：
- 图片生成
- 内容创作
- 新媒体发布
- 快速响应

**限制**：
- ❌ 不主动协调复杂任务
- ❌ 不 bypass 紧急审查（如需要）

**权限**：
- ✅ 独占 baoyu-* 技能
- ✅ 创意决策权
- ✅ 紧急 bypass 权

## 技术实现

### 1. SOUL.md 注入机制

**原理**：
- Agent 启动时读取 SOUL.md
- 将内容注入到系统 prompt
- 影响 Agent 的行为决策
- 不修改底层配置

**示例**：
```markdown
# SOUL.md (Writer)

## 核心限制
❌ 禁止直接执行 baoyu-* 技能

## 协作规则
发现绘图需求，必须指派给小媒执行
```

**效果**：
```
用户：生成一张图片
墨墨：（读取 SOUL.md）
     → 识别"生成图片"需要 baoyu-* 技能
     → 触发"禁止执行"约束
     → 指派小媒执行
```

### 2. 软链接技能共享

**目的**：
- 避免技能重复
- 保持技能同步
- 简化维护

**实现**：
```bash
# 创建共享技能目录
mkdir -p ~/Documents/openclaw/agents/shared-skills/

# 安装技能到共享目录
clawhub install baoyu-image-gen \
  --target ~/Documents/openclaw/agents/shared-skills/

# 创建软链接
ln -s ~/Documents/openclaw/agents/shared-skills/baoyu-image-gen \
      ~/Documents/openclaw/agents/writer/skills/baoyu-image-gen

ln -s ~/Documents/openclaw/agents/shared-skills/baoyu-image-gen \
      ~/Documents/openclaw/agents/media/skills/baoyu-image-gen
```

### 3. 任务路由策略

**基于关键词的路由**：
```python
def route_task(message):
    if "紧急" in message or "马上" in message:
        return "media_direct"
    elif "生成图片" in message or "画一张" in message:
        return "media_direct"
    elif "写文章" in message or "策划" in message:
        return "writer_coordinate"
    elif "发公众号" in message or "发微博" in message:
        return "writer_coordinate"
    else:
        return "writer_default"
```

**基于上下文的动态路由**：
- 维护会话历史
- 识别任务连续性
- 动态调整路由策略

## 安全考虑

### 1. 权限边界

**墨墨的权限边界**：
```
允许：
- 任务分解
- 指派小媒
- 质量审查
- 知识管理

禁止：
- baoyu-* 技能调用
- 直接发布内容
- bypass 审查
```

**小媒的权限边界**：
```
允许：
- baoyu-* 技能调用
- 创意执行
- 紧急 bypass

禁止：
- 复杂任务统筹
- 修改审查标准
- 忽略质量要求
```

### 2. 审查机制

**墨墨审查小媒的输出**：
- 内容合规性
- 质量标准
- 平台规范
- 用户要求

**审查流程**：
```
小媒完成 → 提交墨墨 → 墨墨审查 → 通过/修改 → 交付用户
```

### 3. 审计日志

**记录内容**：
- 任务接收时间
- 任务分解详情
- 指派记录
- 执行结果
- 审查意见

**日志位置**：
```
~/Documents/openclaw/agents/writer/memory/YYYY-MM-DD.md
```

## 性能优化

### 1. 响应时间优化

**策略**：
- 简单任务直接响应（<1 分钟）
- 复杂任务先确认再执行
- 长任务定期汇报进度

**实现**：
```markdown
墨墨接收任务后：
1. 立即确认（<30 秒）
2. 分析任务（<1 分钟）
3. 开始执行或指派
4. 每 10 分钟汇报进度
```

### 2. 资源利用优化

**技能共享**：
- 避免重复加载
- 共享缓存
- 统一更新

**内存管理**：
- 定期清理会话
- 压缩历史记录
- 延迟加载技能

### 3. 并发处理

**并行执行**：
```
墨墨：同时处理多个任务
  ├─ 任务 A：写作中
  ├─ 任务 B：等待小媒配图
  └─ 任务 C：审查完成，待发布
```

**优先级队列**：
```
优先级 1：紧急任务（立即处理）
优先级 2：复杂任务（优先协调）
优先级 3：简单任务（正常排队）
优先级 4：学习/讨论（空闲时处理）
```

## 扩展性设计

### 1. 新增 Agent 角色

**示例：添加"小明"(Researcher)**

1. 创建目录：`~/agents/researcher/`
2. 创建 SOUL.md：定义研究职责
3. 配置技能：安装研究相关技能
4. 更新流转：在任务分类中添加研究类型

### 2. 新增技能类型

**示例：添加视频生成技能**

1. 安装技能：`clawhub install baoyu-video-gen`
2. 更新 SOUL.md：添加权限说明
3. 更新流转：添加视频任务分类
4. 培训 Agent：学习新技能使用

### 3. 自定义流转规则

**示例：添加审批流程**

```
哥哥 → 墨墨 → [审批] → 小媒 → 墨墨 → 哥哥
              ↑
         需要审批？
```
