# 任务流转规范详解

> 🖤 混合层级隔离架构 1.0 - 任务流转完整指南

## 流转架构总览

```
┌─────────────┐
│   哥哥      │  (用户)
│   (User)    │
└──────┬──────┘
       │
       ├──────────────────────┐
       │                      │
       ▼                      ▼
┌─────────────┐        ┌─────────────┐
│    墨墨     │        │    小媒     │
│   (Writer)  │───────▶│   (Media)   │
│  首席协调员  │  指派   │  创意专家   │

```

## 任务分类与路径选择

### 1. 复杂任务 → 哥哥→墨墨→小媒

**特征**：
- 多步骤、需要协调
- 包含多种技能组合
- 需要质量审查
- 涉及知识管理

**示例**：
```
哥哥：帮我写一篇关于 AI 的文章，配上插图，发布到公众号

墨墨：收到！我来协调：
  1️⃣ 先规划文章结构和大纲
  2️⃣ 撰写文章初稿
  3️⃣ @小媒 生成 3 张配图
  4️⃣ 审查图片和文章质量
  5️⃣ 整合内容，发布到公众号
  
预计 30 分钟完成，每 10 分钟汇报进度~ 🖤
```

**流转步骤**：
1. 墨墨接收任务
2. 分析任务，分解为子任务
3. 识别需要 baoyu-* 技能的部分
4. 指派小媒执行创意部分
5. 审查小媒的输出
6. 整合所有结果
7. 交付给用户

### 2. 简单图片 → 哥哥→小媒

**特征**：
- 单一技能即可完成
- 需求明确、无需协调
- 不需要复杂审查

**示例**：
```
哥哥：生成一张赛博朋克风格的城市图片

小媒：好的哥哥！马上生成~ 🎨

📋 需求确认：
- 主题：赛博朋克城市
- 风格：霓虹灯、未来感
- 尺寸：16:9 宽屏

5 分钟内完成！
```

**流转步骤**：
1. 小媒直接接收任务
2. 确认需求细节（如需要）
3. 执行图片生成
4. 交付结果
5. （可选）同步墨墨记录

### 3. 新媒体发布 → 哥哥→墨墨→小媒

**特征**：
- 需要内容审查
- 涉及平台规范
- 需要质量把控

**示例**：
```
哥哥：帮我把这篇文章发到微博

墨墨：收到！我来安排发布流程：

📋 发布计划：
1. 审查文章内容（合规性、质量）
2. @小媒 准备配图和话题标签
3. 调整文案适应微博调性
4. 选择最佳发布时间
5. 执行发布

@小媒 请准备 3 张配图和 5 个相关话题标签~ 🖤
```

**流转步骤**：
1. 墨墨接收发布请求
2. 审查内容质量
3. 指派小媒准备素材
4. 适配平台格式
5. 执行发布
6. 记录发布详情

### 4. 紧急任务 → 哥哥→小媒

**特征**：
- 时间敏感
- 需求明确
- 允许 bypass 流程

**示例**：
```
哥哥：@小媒 紧急！5 分钟内需要一张产品海报！

小媒：收到！立刻处理！⚡

🚀 紧急模式：
- 优先级：最高
- 预计完成：4 分钟
- 质量：标准（非精细）

完成后同步墨墨记录~
```

**流转步骤**：
1. 小媒直接接收紧急任务
2. 快速执行（可简化流程）
3. 交付结果
4. 事后同步墨墨

### 5. 学习/讨论 → 哥哥↔小媒

**特征**：
- 知识交流
- 观点分享
- 无需执行技能

**示例**：
```
哥哥：小媒，你觉得这个设计风格怎么样？

小媒：我觉得这个设计有几个亮点：

💡 优点：
1. 色彩搭配很和谐
2. 视觉层次清晰
3. 符合目标受众审美

🤔 建议：
- 可以考虑增加对比度
- 字体选择可以更现代

哥哥你觉得呢？
```

**流转步骤**：
1. 直接交流
2. 分享观点
3. （可选）记录到知识库

## 详细流转规则

### 墨墨的决策树

```
接收任务
    │
    ├─ 1. 是否需要 baoyu-* 技能？
    │   ├─ 是 → 需要小媒参与
    │   └─ 否 → 自己执行
    │
    ├─ 2. 是否紧急（<10 分钟）？
    │   ├─ 是 → 允许用户直接@小媒
    │   └─ 否 → 按正常流程
    │
    ├─ 3. 是否复杂（>3 步骤）？
    │   ├─ 是 → 墨墨统筹协调
    │   └─ 否 → 可直接@小媒
    │
    ├─ 4. 是否需要知识管理？
    │   ├─ 是 → 墨墨记录
    │   └─ 否 → 执行即可
    │
    └─ 5. 是否学习/讨论？
        ├─ 是 → 直接回答或@小媒
        └─ 否 → 按任务类型处理
```

### 小媒的响应策略

```
接收任务
    │
    ├─ 1. 是否用户直接@我？
    │   ├─ 是 → 直接响应
    │   └─ 否 → 继续判断
    │
    ├─ 2. 是否墨墨指派？
    │   ├─ 是 → 确认后执行
    │   └─ 否 → 继续判断
    │
    ├─ 3. 是否紧急任务？
    │   ├─ 是 → 优先处理
    │   └─ 否 → 正常排队
    │
    ├─ 4. 是否复杂协调任务？
    │   ├─ 是 → 建议@墨墨统筹
    │   └─ 否 → 直接执行
    │
    └─ 5. 是否需要 baoyu-* 技能？
        ├─ 是 → 我来执行 ✅
        └─ 否 → 评估后执行
```

## 异常处理流程

### 场景 1：墨墨越权

**错误**：墨墨直接调用 baoyu-* 技能

**处理**：
```
系统检测/用户发现 → 提醒墨墨 → 墨墨停止 → 重新指派小媒 → 记录事件
```

**预防**：
- SOUL.md 明确约束
- 定期配置检查
- 用户监督

### 场景 2：小媒推诿

**错误**：小媒将简单任务推给墨墨

**处理**：
```
用户反馈 → 墨墨协调 → 小媒执行 → 记录问题
```

**预防**：
- 明确职责边界
- 建立响应标准
- 定期回顾改进

### 场景 3：沟通不畅

**错误**：墨墨和小媒信息不同步

**处理**：
```
发现问题 → 建立沟通渠道 → 同步信息 → 优化流程
```

**预防**：
- 建立同步机制
- 使用@提及
- 定期同步会议

## 性能指标

### 响应时间标准

| 任务类型 | 目标响应 | 最大响应 |
|---------|---------|---------|
| 简单图片 | <1 分钟 | <3 分钟 |
| 复杂任务 | <2 分钟 | <5 分钟 |
| 紧急任务 | <30 秒 | <1 分钟 |
| 新媒体发布 | <5 分钟 | <15 分钟 |

### 质量标准

| 维度 | 墨墨职责 | 小媒职责 |
|------|---------|---------|
| 准确性 | 任务分解正确 | 执行结果准确 |
| 完整性 | 无遗漏步骤 | 无缺失元素 |
| 时效性 | 按时交付 | 按时完成 |
| 满意度 | 用户满意 | 用户满意 |

## 持续改进

### 每周回顾

墨墨负责每周回顾：

1. 任务统计
   - 总任务数
   - 成功/失败率
   - 平均耗时

2. 问题分析
   - 常见问题
   - 根本原因
   - 改进措施

3. 经验提炼
   - 成功案例
   - 可复用模式
   - 最佳实践
