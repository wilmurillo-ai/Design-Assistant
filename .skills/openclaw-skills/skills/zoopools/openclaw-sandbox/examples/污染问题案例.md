## 🔴 案例 1: 沙盒复制生产配置

**时间**: 2026-03-09 12:52 PM  
**影响**: ⭐⭐⭐⭐⭐ Gateway 崩溃，WebUI 无法访问

### 问题根因

```bash
# ❌ 错误做法
cp -r ~/.openclaw/* /tmp/openclaw-sandbox/.openclaw/
```

**问题**:
- 沙盒继承了 `plugins.allow: ['lossless-claw']`
- 沙盒共享了 `~/.openclaw/extensions/`
- 两个 Gateway 同时加载有问题的插件

### 连锁反应

```
12:52 PM  创建沙盒（复制生产配置）❌
    ↓
12:55 PM  两个 Gateway 同时加载 lossless-claw ❌
    ↓
12:58 PM  尝试软链接修复 → Gateway 崩溃 ❌
    ↓
13:00 PM  生产环境 WebUI 无法访问 ❌
    ↓
13:07 PM  删除插件配置 + 重启 → 生产恢复 ✅
```

### 正确做法

```bash
# ✅ 正确做法：创建独立配置
mkdir -p /tmp/openclaw-sandbox/.openclaw/{extensions,agents,logs}

cat > /tmp/openclaw-sandbox/.openclaw/openclaw.json << 'EOF'
{
  "gateway": {"port": 18800, "bind": "loopback"},
  "plugins": {"allow": []}
}
EOF
```

### 教训

**三不复制原则**:
1. ❌ 不复制生产配置
2. ❌ 不复制生产插件
3. ❌ 不复制生产数据

## 🔴 案例 2: 共享插件目录

**时间**: 2026-03-09 12:55 PM  
**影响**: ⭐⭐⭐⭐ 插件加载冲突

### 问题根因

### 解决方案

```bash

mkdir -p /tmp/openclaw-sandbox/.openclaw/extensions
```

### 教训

**插件隔离必须彻底**

## 🔴 案例 3: 环境变量污染

**时间**: 2026-03-09 17:40 PM  
**影响**: ⭐⭐⭐⭐ 生产无法访问

### 问题根因

```bash
# 在沙盒目录执行 CLI
cd /tmp/openclaw-sandbox
openclaw status  # OPENCLAW_HOME 未清理
```

### 连锁反应

```
17:40 PM  在沙盒目录执行 openclaw 命令
    ↓
17:41 PM  OPENCLAW_HOME 环境变量未清理
    ↓
17:42 PM  生产 Gateway 读取沙盒配置
    ↓
17:43 PM  生产环境无法访问（配置不兼容）
```

### 解决方案

```bash
# ✅ 执行后清理环境变量
unset OPENCLAW_HOME
```

### 教训

**执行前后必须清理环境变量**

## 🔴 案例 4: memoryFlush 配置污染

**时间**: 2026-03-10 07:20 AM  
**影响**: ⭐⭐⭐ 配置报错

### 问题根因

```bash
# ❌ 添加不支持的配置键
{
  "agents": {
    "defaults": {
      "memoryFlush": 100  # OpenClaw 2026.3.8 不支持
    }
  }
}
```

### 解决方案

```bash
# ✅ 配置前验证
openclaw config validate

# ✅ 配置前备份
cp openclaw.json openclaw.json.bak

# ✅ 报错回滚
cp openclaw.json.bak openclaw.json
```

### 教训

**配置修改前必须验证 + 备份**
