## 📊 成本分析模板

### 请求类型分布

| 类型 | 占比 | 推荐模型 | 实际模型 | 优化空间 |
|------|------|---------|---------|---------|
| JSON 提取 | 30% | GPT-4o-mini | Claude | 高 |
| 简单摘要 | 25% | Haiku | Claude | 高 |
| 翻译 | 15% | GPT-4o-mini | Claude | 高 |
| 事实问答 | 10% | GLM-4 | Claude | 中 |
| 复杂推理 | 20% | Claude | Claude | 无 |

### 优化建议

**高优先级**:
1. JSON 提取任务 → 路由到 GPT-4o-mini (节省 85%)
2. 简单摘要 → 路由到 Haiku (节省 80%)
3. 翻译任务 → 路由到 GPT-4o-mini (节省 85%)

**中优先级**:
1. 事实问答 → 路由到 GLM-4 (节省 70%)

**低优先级**:
1. 复杂推理 → 保持使用 Claude (无需优化)

## 🎓 优化技巧

### 1. 智能路由调优

**技巧**: 根据实际使用情况调整路由策略

```json
{
  "routing": {
    "strategy": "cost_optimized",
    "claude_percentage": 23,  // 根据实际调整
    "fallback_chain": [
      "anthropic/claude-sonnet-4",  // 首选
      "zhipu/glm-4",                // 备选 1
      "deepseek/deepseek-chat",     // 备选 2
      "openai/gpt-4o-mini"          // 备选 3
    ]
  }
}
```

### 2. 缓存优化

**技巧**: 提高缓存命中率

```json
{
  "cache": {
    "enabled": true,
    "ttl_minutes": 10,      // 根据请求重复率调整
    "max_cache_size_mb": 500,
    "deduplication": true
  }
}
```

### 3. Token 压缩

**技巧**: 启用观察压缩 (Agent 任务节省 20-40%)

```json
{
  "compression": {
    "enabled": true,
    "threshold_kb": 180,
    "observation_compression": true,  // 关键！
    "max_compression_ratio": 0.97
  }
}
```

### 4. 模型分配

**技巧**: 根据任务类型手动指定模型

```bash
# 简单任务 - 使用便宜模型
openclaw agent --model gpt-4o-mini --message "提取 JSON"

# 复杂任务 - 使用 Claude
openclaw agent --model claude-sonnet-4 --message "架构设计"
```
