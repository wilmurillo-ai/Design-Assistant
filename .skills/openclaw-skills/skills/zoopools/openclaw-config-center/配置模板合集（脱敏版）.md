## 🔧 1. core.json - 核心配置

```json
{
  "device_id": "{{设备 ID 前缀}}-{{设备名称}}",
  "api_base": "https://api.openclaw.ai",
  "log_level": "info",
  "log_dir": "{{用户主目录}}/.openclaw/logs",
  "gateway_port": 18789,
  "webui_port": 3000,
  "config_version": "v1.0"
}
```

### 字段说明

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| `device_id` | 设备唯一标识 | `{{设备 ID 前缀}}-{{设备名称}}` | ✅ |
| `api_base` | OpenClaw API 地址 | 默认即可 | ✅ |
| `log_level` | 日志级别 | `info`/`debug`/`warn` | ✅ |
| `log_dir` | 日志目录 | `/Users/xxx/.openclaw/logs` | ✅ |
| `gateway_port` | 网关端口 | `18789`（默认） | ✅ |
| `webui_port` | WebUI 端口 | `3000`（默认） | ✅ |

### 获取设备 ID 前缀

```bash
# 方法 1: 从现有配置读取
cat ~/.openclaw/identity/device.json | jq -r .deviceId | cut -c 1-8

# 方法 2: 使用设备名称
hostname  # 如：{{设备名称}}
```

## 🖤 2. agents/writer.json - 协调员 Agent 配置

```json
{
  "name": "{{协调员 Agent 名称}}",
  "role": "coordinator",
  "agent_id": "writer",
  "workspace": "{{用户主目录}}/Documents/openclaw/agents/{{协调员 Agent ID}}",
  "model": {
    "primary": "{{模型提供商}}/{{模型名称}}"
  },
  "restrictions": {
    "forbidden_skills": ["{{禁止的技能模式}}"],
    "allowed_actions": ["review", "coordinate", "manage_memory"]
  },
  "memory": {
    "consolidate_time": "03:00",
    "consolidate_enabled": true
  }
}
```

### 字段说明

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| `name` | Agent 显示名称 | `墨墨`/`Coordinator` | ✅ |
| `role` | Agent 角色 | `coordinator`（协调员） | ✅ |
| `agent_id` | Agent 唯一 ID | `writer` | ✅ |
| `workspace` | 工作目录 | `/Users/xxx/Documents/openclaw/agents/writer` | ✅ |
| `model.primary` | 主用模型 | `bailian/qwen3.5-plus` | ✅ |
| `forbidden_skills` | 禁止的技能 | `["baoyu-*"]` | ⚠️ |
| `consolidate_time` | 记忆巩固时间 | `03:00`（凌晨 3 点） | ⚠️ |

### 常见模型配置

| 提供商 | 模型 | 配置值 | 用途 |
|--------|------|--------|------|
| 通义千问 | Qwen3.5-Plus | `bailian/qwen3.5-plus` | 通用对话 |
| 通义千问 | Qwen-Max | `bailian/qwen-max` | 复杂任务 |

## 📱 4. channels/feishu.json - 飞书渠道配置

**⚠️ 敏感信息警告**: 此文件包含 App Secret，请勿上传到公开仓库！

```json
{
  "enabled": true,
  "connection_mode": "websocket",
  "domain": "feishu",
  "group_policy": "open",
  "dm_policy": "open",
  "plugins_allow": ["feishu-openclaw-plugin"],
  "bots": [
    {
      "app_id": "{{飞书应用 App ID}}",
      "app_secret": "{{飞书应用 App Secret}}",
      "agent_id": "{{协调员 Agent ID}}",
      "webhook_enabled": true
    },
    {
      "app_id": "{{飞书应用 App ID 2}}",
      "app_secret": "{{飞书应用 App Secret 2}}",
      "agent_id": "{{创意专家 Agent ID}}",
      "webhook_enabled": true
    }
  ]
}
```

### 字段说明

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| `enabled` | 是否启用 | `true` | ✅ |
| `connection_mode` | 连接模式 | `websocket`（长连接） | ✅ |
| `app_id` | 飞书应用 ID | `{{飞书应用 App ID}}` | ✅ |
| `app_secret` | 飞书应用密钥 | `{{飞书应用 App Secret}}` | ✅ |
| `agent_id` | 绑定的 Agent | `writer`/`media` | ✅ |

### 获取飞书应用凭证

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 进入"凭证与基础信息"
4. 复制 `App ID` 和 `App Secret`

**⚠️ 安全提醒**:
- 不要将 `app_secret` 上传到 GitHub 等公开仓库
- 建议在 `.gitignore` 中添加 `channels/feishu.json`
- 生产环境可使用环境变量替代

## 🔒 安全建议

### 1. .gitignore 配置

在项目根目录创建 `.gitignore`：

```gitignore
# 敏感配置文件
config/channels/feishu.json
config/core.json

# 保留模板
!config/channels/feishu.json.example
!config/core.json.example
```

### 2. 权限设置

```bash
# 配置目录权限
chmod 755 ~/.openclaw/config
chmod 755 ~/.openclaw/config/agents
chmod 755 ~/.openclaw/config/channels

# 配置文件权限
chmod 644 ~/.openclaw/config/*.json
chmod 644 ~/.openclaw/config/agents/*.json
chmod 600 ~/.openclaw/config/channels/feishu.json  # 密钥文件更严格
```

### 3. 环境变量替代（高级）

对于生产环境，可使用环境变量：

```bash
# ~/.zshrc 或 ~/.bashrc
export OPENCLAW_FEISHU_APP_ID="cli_xxx"
export OPENCLAW_FEISHU_APP_SECRET="xxx"
```

然后在脚本中读取：

```bash
APP_ID=$OPENCLAW_FEISHU_APP_ID
```

## 🚀 快速开始

### 步骤 1: 复制模板

```bash
# 创建配置目录
mkdir -p ~/.openclaw/config/{agents,skills,channels}

# 复制模板文件（脱敏版）
cp ~/Documents/openclaw/agents/writer/config-templates/*.json.example ~/.openclaw/config/
```

### 步骤 2: 修改配置

```bash
# 编辑核心配置
vim ~/.openclaw/config/core.json

# 编辑 Agent 配置
vim ~/.openclaw/config/agents/writer.json
vim ~/.openclaw/config/agents/media.json

# 编辑渠道配置（⚠️ 敏感）
vim ~/.openclaw/config/channels/feishu.json
```

### 步骤 3: 验证配置

```bash
# 验证 JSON 格式
jq '.' ~/.openclaw/config/core.json
jq '.' ~/.openclaw/config/agents/writer.json
jq '.' ~/.openclaw/config/channels/feishu.json

# 所有输出无错误即为格式正确
```

### 步骤 4: 生效配置

```bash
# 生成主配置
~/.openclaw/scripts/generate-main-config.sh

# 同步记忆
~/.openclaw/scripts/update-soul.sh

# 重启网关
openclaw gateway restart --force
```

### 步骤 5: 验证运行

# 检查渠道状态

## 📞 常见问题

### Q1: 设备 ID 怎么获取？

```bash
# 方法 1: 从现有配置
cat ~/.openclaw/identity/device.json | jq -r .deviceId

# 方法 2: 使用 hostname
hostname

# 方法 3: 自定义
echo "my-custom-device-id"
```

### Q2: 飞书 App Secret 泄露了怎么办？

1. **立即重置**: 访问飞书开放平台 → 应用管理 → 重置 App Secret
2. **更新配置**: 修改 `channels/feishu.json`
3. **重启网关**: `openclaw gateway restart --force`

### Q3: 可以只配置一个 Agent 吗？

可以！在 `channels/feishu.json` 的 `bots` 数组中只保留一个对象即可：

```json
{
  "bots": [
    {
      "app_id": "cli_xxx",
      "app_secret": "xxx",
      "agent_id": "writer"
    }
  ]
}
```

### Q4: 配置文件格式错了怎么办？

```bash
# 使用 jq 验证
jq '.' ~/.openclaw/config/core.json

# 如果报错，说明 JSON 格式有误
# 常见错误：
# - 缺少逗号
# - 引号不匹配
# - 注释（JSON 不支持注释！）
```

**⚠️ 重要**: JSON 格式不支持注释！模板中的注释仅用于说明，实际使用时需删除。
