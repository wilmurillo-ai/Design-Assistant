# Frontend Skill

Ship composition first, components second.

## Best for

- Premium landing pages with strong first-screen impact
- Brand-forward websites and product marketing surfaces
- Motion-led interfaces where hierarchy and atmosphere matter
- App/product UI that needs calm, high-signal visual structure

## Not for

- Backend-only tasks with no UI output
- Accessibility/security review as primary task (do separately)
- Dense enterprise forms where visual narrative is not a goal
- “Add more cards” requests without clear information architecture

## 60-second preflight

1) Define visual thesis in one sentence (mood/material/energy).
2) Define section jobs: hero -> support -> detail -> final CTA.
3) Define interaction thesis: 2-3 motion moments.
4) Choose type system (max 2 typefaces) and one accent color.

## Core workflows

### 1) Landing page mode
- Start with one dominant visual anchor in first viewport.
- Keep hierarchy strict: brand -> headline -> body -> CTA.
- Avoid hero cards and card mosaics by default.
- Keep copy short and scannable.

Default section flow:
1. Hero
2. Support proof/feature
3. Detail/story/depth
4. Final CTA

### 2) App/product UI mode
- Prioritize orientation, status, action.
- Build workspace layout first: nav + primary surface + secondary context.
- Use cards only when card itself is interaction.
- Keep color restraint and typography-led hierarchy.

### 3) Imagery mode
- Use real-looking in-situ imagery when narrative matters.
- Preserve calm tonal area for text legibility.
- Avoid cluttered signage/typography inside hero images.
- Avoid synthetic collage look unless explicitly requested.

### 4) Motion mode
Ship 2-3 intentional motions only:
1. Hero entrance,
2. Scroll/sticky depth effect,
3. Hover/reveal/layout transition with UX value.

Motion must be smooth on mobile, visible in quick recording, and removed if ornamental.

## Failure recovery playbook

- Screen feels generic SaaS -> remove card grid, restore one visual anchor.
- Beautiful but weak brand presence -> raise brand line prominence above headline.
- Headline strong but action weak -> simplify to one primary CTA and one backup.
- Layout noisy -> remove competing accents, reduce sections, tighten copy.
- Motion feels gimmicky -> cut to one purposeful transition per section.

## Guardrails

- One dominant idea per section.
- No hero cards by default.
- No filler copy or prompt/meta commentary in UI text.
- Prefer whitespace, scale, contrast, cropping before extra chrome.
- Keep first viewport readable in one glance on mobile.
- Reject split-screen hero unless one side is calm and text-safe.

## Litmus checks

1. Is brand/product unmistakable in first screen?
2. Is there one strong visual anchor?
3. Can user scan headlines and understand page flow?
4. Does each section have one job?
5. Would this still look premium without decorative effects?
