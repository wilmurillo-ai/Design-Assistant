# 论文笔记标准格式模板

严格按照以下格式输出论文笔记：

```markdown
# 一、基本信息
1.paper：《论文标题》

2.github：[链接](URL) 或 未知

3.会议：会议名称 或 未知

# 二、文章理解
## 1. 研究背景与动机 (Motivation)
- 领域背景介绍
- 现有方法/传统方法的演变历程
- 现有方法的局限性
- 引出本文动机

## 2. 核心问题 (Problem Statement)
- 现有方法存在的具体问题（编号列举）
- 每个问题的数学推导/公式分析
- 关键缺陷的理论解释
- 如有重要概念或面试级知识点，用 :::info 块展开详细推导

## 3. 解决方法 (Methodology)
- 方法的数学原理（公式 + 解释）
- 关键设计选择及其理论依据
- 算法实现细节（附 PyTorch 伪代码）
- 工程优化方案（如有）
- 如有复杂算法，用 :::info 块展开说明

## 4. 实验结果 (Experiments)
- 实验设置（模型架构、参数量、扩展率等）
- 稳定性分析（Loss 曲线、梯度范数、信号增益等）
- 性能对比（各 Benchmark 上的表现）
- 扩展性分析（Scaling 规律）
- 具体数值结果
```

## 格式要求

1. **语言**：中文为主，技术术语保留英文
2. **公式**：使用 LaTeX 格式，行内公式用 `$...$`，独立公式用 `$$...$$`
3. **图片**：保留原始图片链接，使用 `![](url)` 格式
4. **重点标注**：使用 `**加粗**` 标记关键概念
5. **扩展知识**：使用 `:::info` 块包裹深度推导、证明过程或面试相关知识点
6. **代码/伪代码**：使用代码块，标注语言类型（如 `python`）
7. **层级结构**：严格遵循 `# → ## → ### → ####` 四级标题体系
8. **数值结果**：使用具体数字说明改进幅度
