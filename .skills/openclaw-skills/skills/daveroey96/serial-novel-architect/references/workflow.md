# 工作流

## 1. 从不可逆的问题开始

先找出整部小说最终要逼向的问题。

例如：

- 主角究竟该打开还是封死那扇门？
- 上古真相值不值得公开？
- 为了阻止灾难，必须失去什么？

如果一个项目无法被压缩成一个不可逆的问题，它通常还没有稳定脊柱。

## 2. 先稳住五个基础

在大量写正文之前，先明确：

- 主角身份
- 主角伤口
- 推着主角往前走的力量
- 主要对立力量
- 拖延的代价

## 3. 把想法翻成压力，不要只停留在标签

较弱的写法：

- “这里有一个古老文明。”

更强的写法：

- “一个被埋起来的古老文明，正在开始影响现实天气、地貌或人命。”

较弱的写法：

- “主角想知道真相。”

更强的写法：

- “如果主角误判这件事，会有更多人死。”

## 4. 谜题要分层释放

更优顺序：

1. 异常事件
2. 物证
3. 解释冲突
4. 历史回声
5. 结构真相
6. 道德选择

较差顺序：

1. 一上来完整解释世界观
2. 很久之后才给后果

## 5. 每个重大真相都要有代价

每次大型揭示，至少触发其中一种：

- 受伤
- 信任破裂
- 关键证据流失
- 选择变难
- 危险升级
- 必须付出代价

## 6. 按分卷功能搭建

每一卷都应该有明确任务。

一个常用结构：

- 第一卷：发现并活下来
- 第二卷：还原并分裂
- 第三卷：选择并收束

## 7. 按章节功能检查

每一章至少完成两件事中的两件：

- 推动情节
- 推进关系
- 提供新证据
- 迫使人物做决定
- 提高危险
- 改变读者理解

如果一章只完成了一件很弱的事，通常说明结构还不够紧。

## 8. 保护人物热度

在设定很重的项目里，要不断追问：

- 谁因此受伤？
- 谁想得到这个东西？
- 谁在隐瞒？
- 谁在害怕？
- 谁明明可以走，却还是留下？

如果答案还是“暂时没有”，说明设定已经跑到人物前面去了。
