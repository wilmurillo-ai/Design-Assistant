## What This Skill Does

1. **要点提取** - 从冗长的会议记录中提取核心要点
2. **结构化纪要** - 生成专业的会议纪要模板
3. **待办提取** - 识别任务、截止日期、负责人
4. **决策记录** - 记录会议形成的决策和结论
5. **行动项汇总** - 汇总所有待办并跟踪进度

## When to Use

- 会议记录太乱需要整理时
- 想快速了解会议结论时
- 担心遗漏任务跟进时
- 需要生成正式会议纪要时

## How It Works

### 1. 要点提取

分析会议内容，提取：
- 讨论的核心问题
- 各方观点和分歧
- 达成的共识
- 未解决的问题

### 2. 结构化纪要

生成标准会议纪要格式：

```
会议纪要

📅 时间: 
📍 地点: 
👥 参会人: 
📝 主持人: 

📋 议程
1. XX议题
2. XX议题

💡 讨论要点
1. ...
2. ...

✅ 决策
1. ...

📎 行动项
1. [任务] - [负责人] - [截止日期]
```

### 3. 待办提取

识别并提取：
- 明确的任务描述
- 被分配的责任人
- 截止日期/时间点
- 优先级标记

### 4. 决策记录

记录会议形成的：
- 明确决策
- 讨论结论
- 方向性共识
- 需要进一步讨论的问题

### 5. 行动项汇总

生成待办表格：

| 序号 | 任务 | 负责人 | 截止日期 | 状态 |
|------|------|--------|----------|------|
| 1 |    |        |          | 待处理 |

## Input Format

### 方式一：粘贴会议记录

```
帮我整理这会议纪要：
[粘贴会议记录/语音转文字内容]
```

### 方式二：描述会议内容

```
今天开了个项目评审会，主要讨论了XX功能的上线时间，张三负责UI，李四负责开发， deadline是下周五
```

### 方式三：生成模板

```
帮我生成一个产品评审会的会议纪要模板
```

## Configuration

### 自定义设置

可以配置：
- 纪要格式：简洁/详细/正式
- 语言：中文/英文
- 模板风格：公司标准/自定义

### 设置命令

```
# 设置默认模板
我的会议纪要用[简洁版]模板

# 设置输出格式
用表格形式展示待办事项
```

## Limitations

- 需要提供会议内容作为输入
- 对模糊表达的理解可能不准确
- 重要会议建议人工复核
