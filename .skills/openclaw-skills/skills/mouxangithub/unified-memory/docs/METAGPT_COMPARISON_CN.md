## 1. 架构对比

### 1.1 MetaGPT 架构

```
┌─────────────────────────┐
│      Environment         │
│  (环境、上下文)          │
└─────────────────────────┘
            ↓
┌─────────────────────────┐
│         Role             │
│  (PM、Architect、Engineer)│
└─────────────────────────┘
            ↓
┌─────────────────────────┐
│        Action            │
│  (分析、设计、实现)       │
└─────────────────────────┘
            ↓
┌─────────────────────────┐
│         Tool             │
│  (LLM、文件、网络)        │
└─────────────────────────┘
            ↓
        输出文档/代码
         (无记忆)
```

**特点**：
- 单次执行，无状态
- 每次都是"第一次"
- 无法积累经验

### 1.2 我们的架构

```
┌─────────────────────────────────────┐
│         记忆系统层                    │
│  ┌──────────┬──────────┬──────────┐ │
│  │ 短期记忆  │ 长期记忆  │ 知识图谱  │ │
│  │ (会话)    │ (LanceDB)│ (关联链)  │ │
│  └──────────┴──────────┴──────────┘ │
│  · 访问追踪 · 置信度衰减 · 自动归档   │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│         Agent 协作层                 │
│  ┌──────────┬──────────┬──────────┐ │
│  │ 工作流引擎│ 角色系统  │ 决策引擎  │ │
│  │SOP + DAG │ 7+ 角色   │ 多因素   │ │
│  └──────────┴──────────┴──────────┘ │
│  · 冲突检测 · 动态分配 · 协作总线    │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│         执行层                       │
│  ┌──────────┬──────────┬──────────┐ │
│  │ LLM 集成  │ 代码沙箱  │ 工具集成  │ │
│  │ 6+ 提供商 │ Docker   │ GitHub等 │ │
│  └──────────┴──────────┴──────────┘ │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│         输出 + 反馈闭环              │
│  文档 · 代码 → Sprint评估 → SOP优化  │
└─────────────────────────────────────┘
                ↓
           存入记忆系统
           (持续积累)
```

**特点**：
- 持续记忆，有状态
- 越用越聪明
- 自我进化
