# QMD 风格搜索

基于 QMD 架构的本地优先搜索系统，支持三层可选增强。

## 三种搜索模式详解

### 模式一：BM25 关键词搜索（默认）

**原理**：基于词频的倒排索引，纯数学计算，无需 LLM。

**特点**：
- ✅ **0 Token 消耗** - 完全不调用 LLM
- ✅ 速度快（~1s 首次索引，后续毫秒级）
- ✅ 精确匹配关键词
- ⚠️ 不理解语义（"苹果"匹配不到"水果"）
- ⚠️ 需要准确的关键词

**适用场景**：
- 知道确切关键词（如 "EvoMap"、"刘总"）
- 快速查找，不关心语义
- Token 预算有限

**启用方式**：
```bash
# 配置文件
memq config --set-config '{"use_vector": false}'

# 或命令行
memq search -q "查询" -m bm25
memq search -q "查询" --no-vector
```

---

### 模式二：向量 + RRF 混合搜索（推荐）

**原理**：BM25 关键词 + 向量语义 + RRF 融合排序。

**特点**：
- ✅ 理解语义（"苹果"能匹配"水果"）
- ✅ 融合关键词和语义，质量更高
- ✅ RRF 本地计算，不额外消耗 Token
- ⚠️ **~100 Token**（1 次 embedding 调用）
- ⚠️ 需要本地 Ollama 运行

**Token 消耗说明**：
- 1 次 `nomic-embed-text` 调用 ≈ 100 Token
- 这是**一次性开销**，无论返回多少结果
- 本地模型，**实际费用为 $0**

**适用场景**：
- 语义搜索（"怎么部署项目"能找到部署文档）
- 不确定准确关键词
- 追求更高搜索质量
- **推荐日常使用**

**启用方式**：
```bash
# 配置文件（默认已开启）
memq config --set-config '{"use_vector": true}'

# 或命令行
memq search -q "查询" -m hybrid
memq search -q "查询"  # auto 模式自动选择
```

---

### 模式三：LLM 智能重排（高级）

**原理**：在混合搜索基础上，用 LLM 对结果重新排序。

**特点**：
- ✅ 最高搜索质量
- ✅ 理解复杂查询意图
- ✅ 上下文感知排序
- ⚠️ **额外 ~300 Token**（小模型）
- ⚠️ 需要配置 LLM 模型
- ⚠️ 增加延迟（~1-2s）

**Token 消耗说明**：
- 基础：~100 Token（向量搜索）
- 重排：~300 Token（小模型推理）
- **总计：~400 Token**
- 使用本地小模型（如 qwen2.5:0.5b）费用仍为 $0

**适用场景**：
- 查询复杂，需要理解意图
- 结果质量要求高
- 不在乎额外 Token
- **值在好用** ✨

**启用方式**：
```bash
# 配置文件
memq config --set-config '{"use_llm_rerank": true, "llm_model": "qwen2.5:0.5b"}'

# 或命令行（临时启用）
memq search -q "查询" --rerank
```

## 配置

```bash
# 查看当前配置
python3 memory_qmd_search.py config

# 设置配置
python3 memory_qmd_search.py config --set-config '{
  "use_vector": true,
  "use_llm_rerank": false,
  "llm_model": "qwen2.5:0.5b"
}'
```

### 配置项

| 配置 | 说明 | 默认值 |
|------|------|--------|
| `use_vector` | 是否启用向量搜索 | `true` |
| `use_llm_rerank` | 是否启用 LLM 重排 | `false` |
| `llm_model` | 重排用的模型 | `qwen2.5:0.5b` |

## 快速使用

```bash
# 自动模式（推荐）
memq search -q "查询内容"

# 纯 BM25（0 Token）
memq search -q "查询" -m bm25

# 启用 LLM 重排（最高质量）
memq search -q "查询" --rerank
```

## 与 QMD 完整对比

### 核心定位差异

| 维度 | unified-memory | QMD |
|------|---------------|-----|
| **定位** | AI Agent 记忆系统 | 文档搜索引擎 |
| **数据类型** | 结构化记忆（分类、时效、关联） | Markdown 文档、会议记录 |
| **使用场景** | Agent 上下文、用户画像、会话记忆 | 知识库检索、笔记搜索 |

### 功能对比

| 功能 | unified-memory | QMD | 说明 |
|------|:-------------:|:---:|------|
| **BM25 搜索** | ✅ | ✅ | 都支持纯关键词搜索 |
| **向量搜索** | ✅ | ✅ | 都支持语义搜索 |
| **RRF 融合** | ✅ | ✅ | 混合搜索质量更高 |
| **本地重排** | ✅ 可选 | ✅ 默认 | QMD 默认启用，我们可选 |
| **片段返回** | ✅ | ✅ | 省 Token |
| **MCP 接口** | ✅ | ✅ | 两者都支持 MCP |
| **CLI 工具** | ✅ | ✅ | 都有命令行工具 |
| **Web UI** | ✅ 38080 | ❌ | 我们有可视化界面 |
| **知识图谱** | ✅ | ❌ | 记忆关系可视化 |
| **用户画像** | ✅ | ❌ | 自动分析用户偏好 |
| **记忆分类** | ✅ | ❌ | preference/fact/decision 等 |
| **时效管理** | ✅ | ❌ | 自动过期、衰减 |
| **健康检测** | ✅ | ❌ | 矛盾/冗余/孤立检测 |
| **云同步** | ✅ | ❌ | S3/WebDAV/Dropbox 等 |
| **多 Agent** | ✅ | ❌ | 协作记忆、冲突解决 |
| **主动注入** | ✅ | ❌ | 预测加载相关记忆 |
| **审计日志** | ✅ | ❌ | 可追溯记忆变更 |

### Token 消耗对比

| 操作 | unified-memory | QMD |
|------|---------------|-----|
| **BM25 搜索** | 0 Token | 0 Token |
| **向量搜索** | ~100 Token (本地免费) | ~100 Token (本地免费) |
| **混合搜索** | ~100 Token | ~100 Token |
| **重排** | +300 Token (可选) | +300 Token (默认) |

**结论**：两者 Token 消耗相当，但 unified-memory 默认不重排，更省 Token。

### 适用场景建议

| 场景 | 推荐 | 原因 |
|------|------|------|
| **AI Agent 记忆** | unified-memory | 结构化、时效、关联、主动注入 |
| **知识库搜索** | QMD | 更轻量、MCP 支持、专注文档 |
| **用户画像** | unified-memory | 自动分析偏好 |
| **笔记检索** | QMD | 简单直接 |
| **多 Agent 协作** | unified-memory | 冲突解决、协作记忆 |
| **MCP 集成** | QMD | 原生支持 |

### 可以结合使用

```bash
# unified-memory 用于 Agent 记忆
mem store "用户偏好使用飞书"
mem search "项目管理"

# QMD 用于文档检索
qmd search "API 文档"
```

**最佳实践**：
- **Agent 记忆** → unified-memory
- **文档/笔记** → QMD
- 两者结合，覆盖所有场景
