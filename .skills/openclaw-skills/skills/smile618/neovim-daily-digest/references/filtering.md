# Filtering Notes

Use these heuristics when you need to explain why a post was kept or dropped, or when you need to manually tune the final digest.

## Strong keep signals

- Mentions a concrete Neovim command, API, or workflow detail
- Announces a new plugin or meaningful plugin update
- Links to a GitHub repo with feature details
- Discusses LSP, Tree-sitter, completion, diffing, file navigation, notes workflows, or editor automation in a reusable way
- Troubleshooting thread reveals a common gotcha or configuration fix

## Strong drop signals

- Sticky/meta thread (`Weekly`, `Monthly`, `Questions Thread`, `Dotfile Review Thread`)
- Pure screenshot, desk setup, or aesthetic showcase with no meaningful feature/workflow detail
- Generic support request with no broader lesson
- Theme/colorscheme post whose main value is appearance rather than functionality

## Borderline cases

- Theme post with a notable feature update: downrank, but keep if the feed is otherwise thin
- AI-related Neovim plugin: keep when it changes workflow materially; drop if it is just another wrapper with no detail
- Help thread: keep only if the summary points to a common debugging lesson other users can reuse

## Ranking intuition

1. Practical tip or workflow improvement
2. Plugin launch/update with concrete feature details
3. Broadly useful debugging/config lesson
4. Interesting but niche tool/plugin
5. Pure showcase or aesthetics
