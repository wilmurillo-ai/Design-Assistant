# 快乐小狗信息整理器 - 处理流程说明

## 📋 标准处理流程

系统严格按照以下5个步骤执行，每个步骤都有独立的错误处理和状态监控：

### 1️⃣ 接受微信链接验证
- **输入**：微信消息中的链接
- **支持平台**：小红书、微信公众号、知乎、B站、GitHub、通用网页
- **验证项目**：
  - 输入格式是否有效
  - 是否为受支持的链接
  - 链接是否可访问
- **错误处理**：
  - ⚠️ 无效输入：提示发送有效链接
  - ⚠️ 不支持平台：列出所有支持的平台
  - ❌ 链接不可访问：提供检查建议

### 2️⃣ 提取内容
- **功能**：使用动态插件系统提取链接内容
- **插件层级**：
  - 基础层：微信、通用网页
  - 进阶层：知乎、B站、GitHub
  - 专家层：小红书（需要Puppeteer）
- **容错机制**：
  - 单个链接失败不影响其他链接处理
  - 失败链接会被记录并提示用户
  - 提供降级方案（简化版内容提取）

### 3️⃣ 分类整理
- **分类方式**：
  - 基础分类：关键词匹配
  - 高级分类：AI辅助分类（需启用进阶功能）
- **分类选项**：人工智能、产品经理、经济投资、心理学、商业机会、灵感、学习成长
- **容错机制**：
  - 分类失败时自动归入"灵感"类别
  - 标记分类置信度
  - 保留分类依据便于人工确认

### 4️⃣ 理解分析，套用笔记模板
- **模板类型**：
  - 小红书模板：包含标签、互动数据、可执行建议
  - 微信模板：核心观点提炼、行动清单
  - B站模板：学习要点、字幕整理、互动统计
  - GitHub模板：技术栈、项目数据、使用建议
  - 通用模板：关键信息提取、应用建议
- **分析功能**：
  - 自动提取关键要点
  - 生成可执行建议
  - 添加相关标签和分类信息

### 5️⃣ 保存到飞书云盘
- **存储结构**：
  ```
  快乐小狗空间/
  ├── 人工智能（AI）/
  │   └── 2024-04-15-文章标题.md
  ├── 产品经理/
  │   └── 2024-04-15-PRD实战.md
  └── 其他分类...
  ```
- **权限设置**：
  - 公开可访问
  - 任何人可评论
- **错误处理**：
  - 自动创建分类文件夹
  - 上传失败时保留内容供重试
  - 提供飞书配置检查

## 🔧 错误处理和容错机制

### 错误级别定义
- **警告（⚠️）**：部分功能失败，不影响整体流程
- **错误（❌）**：关键步骤失败，需要用户干预
- **严重错误（🚨）**：系统级错误，需要管理员处理

### 容错策略
1. **优雅降级**：高级功能失败时使用基础功能
2. **批量处理**：部分失败不影响其他项目
3. **重试机制**：暂时性错误自动重试
4. **人工介入**：AI判断不确定时请求人工确认

## 📊 执行监控

### 追踪信息
- **trace_id**：唯一的处理标识，用于问题追踪
- **步骤耗时**：记录每步的执行时间
- **成功/失败状态**：明确标记每步执行结果

### 性能指标
- 总处理时长
- 各步骤耗时分布
- 成功率统计
- 错误类型分析

## 🚨 常见错误和解决方案

### 配置相关错误
```
❌ 飞书配置无效
解决：检查环境变量
- FEISHU_APP_ID
- FEISHU_APP_SECRET  
- FEISHU_BASE_FOLDER_TOKEN
```

### 网络相关错误
```
❌ 提取内容失败
解决：
1. 检查网络连接
2. 确认链接可访问
3. 尝试稍后重试
```

### 分类相关错误
```
❓ 自动分类置信度不足
解决：手动选择分类
- 支持的分类共7种
- 系统会提供分类依据供参考
```

○ validate-links

## 🎯 最佳实践建议

1. **批量处理**：一次性发送多个链接提高效率
2. **准确输入**：发送完整的URL链接
3. **及时反馈**：遇到错误时查看详细错误信息
4. **定期检查**：定期查看飞书云盘中的整理结果
5. **分类优化**：对误分类的内容提供反馈，帮助系统优化
