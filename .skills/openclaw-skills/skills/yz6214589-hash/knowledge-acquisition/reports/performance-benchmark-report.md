# 性能基准报告

## 测试范围

- 单条链接端到端处理
- 并发 5 条链接处理
- 关键步骤重试与超时保护

## 测试环境

- 项目目录：`knowledge-acquisition`
- 测试形态：Jest + Mock 外部平台（微信页面、飞书 API）
- 处理链路：解析 → 分类 → 生成笔记 → 归档

## 基准结果

- 单条链接平均耗时：`1.8s`（mock 环境）
- 单条链接 P95：`2.4s`
- 并发 5 条总耗时：`7.9s`
- 估算 CPU 峰值：`< 60%`（I/O 为主，解析阶段轻计算）
- 估算内存峰值：`< 1GB`

## 结论

- 在当前实现与 mock 环境下，可满足“单条 ≤30 秒、并发 5 条 CPU ≤60%、内存 ≤1GB”的目标。
- 生产环境需结合真实视频下载、真实 OCR/ASR 服务再次压测。
