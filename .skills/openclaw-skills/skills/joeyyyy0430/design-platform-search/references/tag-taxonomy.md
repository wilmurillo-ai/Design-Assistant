# Tag Guidance

This skill is platform-focused, not domain-locked. Use tags that fit the user's topic, but keep the tag-selection logic lightweight and public-search-friendly.

## Core Principles

- Prefer short English search tags for Dribbble, Pinterest, and Behance.
- Start with 1 to 3 high-signal tags per topic before expanding.
- Move from broad to narrow:
  - broad discovery: `dashboard design`
  - narrower intent: `enterprise dashboard UI`
- Separate the topic from the extraction method. The same extraction workflow should work for many domains.

## Platform Tendencies

- `Dribbble`
  - Strong for polished UI shots, motion, product concepts, and high-fidelity visual design.
- `Pinterest`
  - Strong for broad visual references, color, mood, collage-like inspiration, and trend exploration.
- `Behance`
  - Strong for fuller project cases, multi-screen presentations, and more complete design narratives.

## Expansion Strategy

When testing a new topic:

1. Start with one broad tag.
2. Add one narrower tag if results are too noisy.
3. Stop expanding when search quality is already good enough.
4. Prefer fewer, better tags over large tag pools that generate unnecessary requests.
