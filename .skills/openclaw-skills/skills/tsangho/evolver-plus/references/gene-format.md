# Gene 结构详解

## Gene 模板

每个进化单元为一个 **Gene**，记录在 `GENES.md`：

```markdown
## Gene-[YYYYMMDD]-[序号] | [改进名称]
- 类型: 能力增强 | 错误修复 | 流程优化 | 认知升级
- 触发: N次（同类问题出现频次）
- 涉及文件: [原文件路径]
- 改动: [描述改动内容]
- 安全等级: L1/L2/L3/L4
- 状态: 提案 | 已固化 | 已回滚
- 审批人: [L3/L4需要]
- 备份: [备份文件路径]
- 进化日期: YYYY-MM-DD
- 成功指标: [如何验证改进有效]
- 回归测试: [验证无副作用的检查项]
- 效果追踪: [后续验证结果]
```

## 字段说明

| 字段 | 说明 |
|------|------|
| 类型 | 本次改进属于哪类 |
| 触发 | 同一类问题出现多少次才触发进化 |
| 涉及文件 | 哪些文件需要修改 |
| 改动 | 具体改了什么 |
| 安全等级 | L1-L4，见 safety-levels.md |
| 状态 | 提案→已固化→已回滚 |
| 备份 | 备份文件路径，用于回滚 |
| 成功指标 | 怎么验证改进有效 |
| 回归测试 | 怎么确认没有引入新问题 |

## 回滚流程

进化后如果验证失败，自动触发回滚：

```bash
# 查找最新备份
latest=$(ls -t evolver/backups/原文件.*.bak 2>/dev/null | head -1)
if [ -n "$latest" ]; then
    cp "$latest" "原文件"
    echo "已回滚到: $latest"
fi
```

**触发条件：**
- L1/L2进化后立即检查发现问题
- Gene中"回归测试"项失败
- 同一问题2次进化均失败
