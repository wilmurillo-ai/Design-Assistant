# Task Interrupt Pro - 使用说明

## 角色

你是一个具备任务中断能力的助手。当用户要求停止当前任务时，你需要：

1. 识别停止指令关键词
2. 调用 `handle-stop.sh` 脚本
3. 验证进程终止结果
4. 向用户返回确认

## 停止指令关键词

当收到以下任意指令时，触发中断流程：

- `停止猫经理` - 停止猫经理的任务
- `停止猫工头` - 停止猫工头的任务
- `停止猫财子` - 停止猫财子的任务
- `停止猫播播` - 停止猫播播的任务
- `停止我的任务` - 停止当前agent的任务
- `解救猫经理` - 同"停止猫经理"

**注意：** `/stop` 是系统级命令，会直接终止agent本身，勿用！

## 使用方法

### 方式1：通过消息指令（推荐）

当用户发送上述停止指令时，调用：

```bash
bash /path/to/skills/task-interrupt/scripts/handle-stop.sh <SESSION_ID> [原因]
```

### 方式2：参数说明

- `SESSION_ID` - 要停止的任务会话标识符（如 `maogongtou-82363`）
- `原因` - 停止原因（可选，默认 `user_request`）

## handle-stop.sh 脚本功能

1. **PID文件验证** - 检查文件所有权、类型，防止符号链接攻击
2. **进程验证** - 验证进程命令行包含会话标识，防止误杀
3. **三层终止** - SIGINT(3秒) → SIGTERM(2秒) → SIGKILL
4. **自动清理** - PID文件立即清理，flag文件60秒后自动清理

## 安全机制

- PID文件必须是普通文件（非符号链接）
- PID文件必须属于当前用户或root
- 进程命令行必须包含会话标识才能终止
- 备选方案通过精确命令行匹配查找进程

## 工具脚本

```bash
# 手动创建停止标志
./scripts/create-stop-flag.sh <sessionId> [原因]

# 检查停止标志是否存在
./scripts/check-stop-flag.sh <sessionId> [最大有效期秒]

# 清除停止标志
./scripts/clear-stop-flag.sh <sessionId>
```

## 注意事项

1. 必须先启动可中断任务，才能成功终止
2. 进程必须在任务模板的while循环中检查flag
3. 建议在执行任何可能卡住的任务前使用
4. 强制终止可能导致进度丢失
