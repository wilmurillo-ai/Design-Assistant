# 技术文档翻译风格指南

## 翻译原则

### 1. 准确性优先
- 技术准确性高于语言流畅性
- 保持原文的技术含义和逻辑关系
- 不得添加或删减原文信息

### 2. 一致性
- 同一术语在整个文档中保持统一翻译
- 遵循术语表中的标准译法
- 保持语气和风格的统一

### 3. 可读性
- 使用简洁明了的表达
- 避免过度直译导致的生硬表达
- 符合目标语言的表达习惯

## 中译英规范

### 语气和风格
- 使用主动语态，避免被动语态
- 使用现在时态描述功能和操作
- 保持简洁，避免冗余
- 使用第二人称"you"直接与读者对话

### 句式结构
- 一句话表达一个主要观点
- 避免过长的复合句
- 使用并列结构时保持平行
- 示例：
  - ✓ "Click **Save** to save your changes."
  - ✗ "Your changes will be saved when you click the Save button."

### 标点符号
- 使用英文标点符号（半角）
- 列表使用句号或不用标点（保持一致）
- 代码、参数使用反引号包裹

### 格式要求
- UI 元素名称使用粗体：**Button Name**
- 代码元素使用反引号：`function_name()`
- 文件路径使用反引号：`/path/to/file`
- 重要提示使用 Note、Warning、Important 等标记

## 英译中规范

### 语气和风格
- 使用书面语，避免口语化
- 保持专业、客观的语气
- 避免过度本土化，保留必要的技术感

### 句式结构
- 遵循中文语序和表达习惯
- 适当拆分长句，提高可读性
- 避免欧化句式
- 示例：
  - ✓ "点击**保存**按钮保存更改。"
  - ✗ "当你点击保存按钮时，你的更改将被保存。"

### 标点符号
- 使用中文标点符号（全角）
- 句号、逗号、冒号使用全角：，。：
- 括号使用全角：（）
- 引号使用中文引号：「」或 ""
- 代码、英文专有名词前后使用半角空格

### 专有名词处理
- 产品名称、技术名称保持英文
- 首次出现时可用中文注释：React（一个 JavaScript 库）
- 常用技术术语保持英文：API、SDK、JSON
- 中英混排时注意空格：使用 React 框架开发应用

### 格式要求
- UI 元素名称使用粗体：**按钮名称**
- 代码元素使用反引号：`function_name()`
- 文件路径使用反引号：`/path/to/file`
- 重要提示使用"注意""警告""重要"等标记

## 数字和单位

### 英文文档
- 使用阿拉伯数字：1, 2, 3
- 大数字使用逗号分隔：1,000,000
- 百分比使用 %：50%
- 单位与数字之间加空格：10 GB, 5 minutes

### 中文文档
- 使用阿拉伯数字：1、2、3
- 大数字使用万、亿：100 万、10 亿
- 百分比使用 %：50%
- 单位与数字之间不加空格：10GB、5 分钟

## 代码相关

### 代码注释
- 注释保持简洁明了
- 使用完整的句子（首字母大写，句末标点）
- 或使用短语（全小写，无标点）
- 保持风格统一

### 代码示例
- 代码块不翻译
- 字符串内容根据需要翻译
- 变量名、函数名不翻译
- 注释根据目标语言翻译

## 翻译流程建议

1. **理解原文**：完整阅读原文，理解上下文和技术背景
2. **术语核对**：检查术语表，确保使用标准译法
3. **初步翻译**：按照风格指南进行翻译
4. **格式检查**：确保标点、格式符合规范
5. **一致性检查**：确保术语和风格的一致性
6. **质量审核**：检查准确性、可读性和完整性
