# 日报质量检查清单

## 0. 快速开始

使用内置模板，确保不遗漏板块：
```bash
cp skills/daily-report-catchup/TEMPLATE.md memory/daily-reports/$(date +%Y-%m-%d).md
```

## 1. 内容完整性（最低标准）

- [ ] 字数 ≥ 200 字
- [ ] 包含「今日工作」板块
- [ ] 包含「复盘」板块
- [ ] 包含「明日计划」或「建议」

## 2. 内容质量（推荐标准）

- [ ] 工作内容具体（非空泛描述）
- [ ] 有量化成果（如：完成 X 功能、修复 Y Bug）
- [ ] 复盘有实质性反思
- [ ] 建议有建设性意见

## 3. 格式规范

- [ ] 使用标准模板
- [ ] 无乱码/格式错误
- [ ] 按时发送（工作日 19:30 ± 30分钟）

## 4. 质量评分规则

| 评分 | 条件 | 说明 |
|------|------|------|
| ⭐⭐⭐ | 满足全部推荐标准 | 优秀 |
| ⭐⭐ | 满足最低标准 | 合格 |
| ⭐ | 有板块但内容空洞 | 需改进 |
| ❌ | 缺失核心板块或字数 < 100 | 不合格 |

## 5. 自动检查脚本

```bash
# 检查报告质量
python3 check_report_quality.py memory/daily-reports/YYYY-MM-DD.md
```

## 6. 不合格报告处理

1. 标记为「待改进」状态
2. 发送提醒给 agent
3. 要求补写内容
