# 第二阶段：剧情设计

## 前提条件

首先加载第一阶段生成的 `settings.md` 文件。如果该文件尚未生成，请先执行第一阶段。

## 步骤

### 步骤 1：选择结构

根据用户偏好和作品特点选择叙事结构：

| 结构 | 适用场景 | 特点 |

|------|-------------|------|

| 三幕式结构 | 长篇娱乐科幻片 | 清晰的开头、发展、转折和结尾，强调情感宣泄 |

| 中篇小说 | 强调节奏 | 加速发展，日式韵律 |

| 非线性 | 时间旅行科幻片，实验性作品 | 时间顺序打乱，谜题般的悬念 |

| 群像剧 | 宏大的世界观 | 多重视角交织，对社会的多面性刻画 |

| 框架故事 | 哲学科幻 | 嵌套结构，元问题 |

### 第二步：创作故事梗概

用一句话概括故事的核心：

```

模板：一个故事，其中[主人公]由于[触发因素]而追求[目标]，面临[障碍]，并直面[主题问题]。

示例：一位失去记忆的前飞行员遇到了一个来自过去的自己，在寻找真我的过程中，他卷入了记忆产业的阴谋，最终直面“即使失去了记忆，我还是我自己吗？”这个问题。

```

### 第三步：设计章节结构

三幕式结构的基本模板：

```markdown
# 情节结构

## 第一幕（占总篇幅的 25%）：背景设定和引言

### 第一章：[标题]

- 场景目标：

- 主要事件：

- 需要传达给读者的信息：

- 章节结尾的悬念：

## 第二幕，前半部分（25%）：冲突与转折

### 第 N 章：[标题]

...

## 第二幕，后半部分（25%）：转折点与危机

### 第 N 章：[标题]

...
## 第三幕（25%）：高潮与结局

### 第 N 章：[标题]

...
```

每个章节必须包含以下内容：

- **场景目标**：本章的叙事功能

- **主要事件**：发生了什么

- **揭示**：读者在本章中了解到的信息章节

- **章节结尾悬念**：为下一章留下悬念（问题、危机、转折）

### 第四步：创建情节图

```markdown

## 情节管理表

| ID | 情节（位置） | 位置章节 | 结局章节 | 结局方式 |

|----|-------------|--------|--------|---------|

| F1 | [内容] | 第2章 | 第8章 | [方式] |

| F2 | [内容] | 第3章 | 第12章 | [方式] |

```

### 第五步：检查张力曲线

对每一章节的张力水平进行评分，分值范围为 1-10 分，并确保符合以下模式：

- 避免超过三章出现单调递增或张力平缓的情况

- 在高潮之前安排一个“平静”的章节

- 确保每一幕的转折点处张力都有明显的提升

### 第六步：输出

将完成的图表保存为 `plot.md`。在收集用户反馈后，我们将进入第三阶段。
