# 第三阶段：角色设计

## 前提条件

首先加载 `settings.md` 和 `plot.md` 文件。

## 步骤

### 第一步：角色列表构建

确定故事中所需的角色：

- **主角**：体现主题并经历最显著变化的角色。

- **对手**：阻碍主角实现目标的角色（不一定是反派）。

- **合作者**：帮助主角并提供不同视角的角色。

- **催化剂**：引发故事转折点的角色。

科幻作品特有的考量：

- 是否应包含非人类角色（人工智能、外星人、后人类）？

- 技术如何影响人格（增强智能、情绪控制、记忆操控等）？

### 第二步：角色设定表创建

为每个主要角色设计以下内容，并将其记录在 `characters.md` 文件中：

```markdown

## [角色名称]

### 基本信息

- 年龄/性别/外貌特征：

- 职业/社会地位：

- 科技强化（如有）：

### 内心世界

- 核心欲望（自我意识层面）：

- 真实欲望（无意识层面，与主题相关）：

- 恐惧/弱点：

- 世界观/信仰体系：

### 叙事功能

- 在情节中的作用：

- 与主题的关系：

- 成长弧线：[起始状态] → [转折点] → [最终状态]

### 人物关系

- [其他角色名称]：[关系及变化]

### 人物声音特征

- 用词倾向：

- 口头禅/特色短语：

- 对话示例（3个）模式）：

1. 正常："..."

2. 压力："..."

3. 情绪场景："..."

```

### 第三步：设计科幻外星人

在描绘非人类角色或经过大幅改造的人类时：

1. **认知差异**：定义人类与非人类之间的一个根本区别。

- 例如：非线性时间感知 / 个体与群体无区别 / 以逻辑运算表达情感

2. **表达限制**：决定如何用文字表达这些差异

- 独特的第一人称句式结构 / 比喻的选择 / 信息的优先级

3. **共情点**：找到一个读者能够产生共鸣的点，即使存在差异

### 第四步：关系图

组织文本中人物之间的关系：

```

[主角] ← 信任/依赖 → [合作者]

[主角] ← 冲突/镜像 → [对手]

[主角] ← 师徒/失去 → [催化剂]

```
至少有一种关系必须在故事中发生根本性的变化。

### 第五步：输出

将完成的人物设置保存到 `characters.md`。继续进行第四阶段。
