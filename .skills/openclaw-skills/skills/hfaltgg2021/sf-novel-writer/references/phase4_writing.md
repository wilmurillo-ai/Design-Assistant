# 第四阶段：编写

## 前提条件

首先加载 `settings.md`、`plot.md` 和 `characters.md` 文件。

## 步骤

### 第一步：确定写作风格

与用户确认以下内容：

- **视角**：第一人称 / 仅第三人称 / 第三人称全知视角 / 多视角交替

- **时态**：过去时（标准） / 现在时（强调即时性）

- **写作风格语气**：如未指定，请从以下选项中选择：

```

写作风格：

- 硬朗风格：以短句为主，情感克制，堆砌事实（参考：伊藤计划，格雷格·伊根）

- 诗意风格：运用丰富的比喻，描写五感，强调韵律（参考：雷·布拉德伯里，上田小百合）

- 逻辑严谨风格：逻辑发展，概念逐步揭示（参考：特德·姜，小川和水）

- 对话驱动风格：对话节奏快，以角色口吻叙述（参考：贝基·钱伯斯）

- 实验风格：打破形式，异质叙事（参考：塞缪尔·德莱尼，托·恩乔）

```

### 第二步：逐章写作

**一次只写一章。** 写作每一章时，请包含以下内容：

输入：

1. 相关章节的情节要点（来自 `plot.md` 文件）

2. 上一章的概要（2-3 句话）

3. 相关章节中出现的角色的口吻特征

4. 风格说明（在第一步中确定）

写作注意事项：

- **开头**：用每章的第一句话吸引读者（情境/感受/问题）

- **信息披露**：将背景介绍融入到动作和对话中。连续的说明段落不要超过三个。

- **场景转换**：每章场景转换次数限制在两次或以下。过渡部分使用空白行和对新感受的描述。

- **章节结尾**：以问题、危机或转折结尾，营造引人入胜的叙事，促使读者翻页。

### 第三步：科幻描述指南

**技术描述**：尽量减少对原理的解释，重点关注技术如何影响人类体验。

```

❌ “这种利用量子纠缠的超光速通信设备，使用了贝尔态的非局域关联……”

✅ “我触摸通信面板的那一刻，就听到了女儿的笑声，仿佛是从隔壁房间传来的，而她远在12光年之外。然而，那声音却是六个小时前传来的。”

```

**外星生物的视角**：有意地改变人类的认知框架。

```

示例（以非线性时间感知者的第一人称视角）：

“我已经在为你所谓的‘尚未发生’的事件而哀悼了。”

```

**世界质感**：通过日常细节而非宏大的场景来展现世界。

```

示例：通过对晨间例行活动、用餐、通勤方式和问候的描写来营造未来感。

``

### 第四步：质量检查（章节完成后）

每章完成后，请进行以下自查：

1. 情节是否已展开？

2. 人物的叙述口吻是否一致？

3. 场景设置是否存在不一致之处？

4. 是否与主题保持了联系？

5. 行文节奏是否恰当（是否存在冗余段落）？

### 第五步：输出

将完成的章节保存到 `chapters/chapter_XX.md`。完成所有章节后，请继续执行第五阶段。
