# sf-novel-writer-cn
逐步指导科幻小说（SF）的规划、写作和修改的技能。
触发条件：任何与创作科幻小说相关的请求，例如“我想写一部科幻小说”、“我想创作一部科幻作品”、“我在构思一个科幻世界”、“我要写科幻小说的下一部分”、“我要设计一个科幻情节”、“我要修改我的科幻小说”等等。支持短篇小说、中篇小说和长篇小说。涵盖各种子类型，例如硬科幻、太空歌剧、反乌托邦和赛博朋克。也可用于编辑、审阅和改进现有稿件。

# 科幻小说写作
技能，分六个阶段逐步指导科幻小说的创作。

## 工作流程概述
整个工作流程包含以下几个阶段。用户可以根据自己的请求从任何阶段开始。
1.规划与构思 → 请参阅 `references/phase1_worldbuilding.md`
2.情节设计 → 请参阅 `references/phase2_plot_design.md`
3.角色创建 → 请参阅 `references/phase3_character.md`
4.写作 → 请参阅 `references/phase4_writing.md`
5.修改与编辑 → 请参阅 `references/phase5_revision.md`
6.输出与完成 → 请按照本文件底部的说明进行操作

## 阶段确定
根据用户输入确定要执行的阶段：
- “世界观”、“设定”、“主题”、“构思” → 阶段 1
- “情节”、“概要”、“结构”、“发展” → 阶段 2
- “角色”、“角色”、“角色简介” → 阶段 3
- “写作”、“写作”、“继续”、“正文” → 阶段 4
- “修改”、“审阅”、“ “修正”、“改进” → 阶段 5
- “输出”、“文件”、“完成”、“润色” → 阶段 6
- 如有歧义 → 与用户确认并建议从阶段 1 开始

## 项目初始化
启动新项目时，运行 `scripts/init_project.py` 生成项目目录：
python3 scripts/init_project.py <项目名称>

这将在 `/home/claude/<项目名称>/` 目录下创建以下文件：
- `settings.md` — 世界观和设定素材
- `plot.md` — 情节和章节结构
- `characters.md` — 角色设定
- `chapters/` — 每个章节的主文本目录
- `notes.md` — 笔记和想法

## 阶段 6：输出和润色
最后阶段按以下步骤进行：
1. 最终检查所有章节的一致性（设定不一致、角色行为和对话、情节要点解决）
2. 合并章节文件以生成最终稿件
3. 以 `.md` 或 `.docx` 格式提供给用户
。4. 以 `.docx` 格式输出时，请参考 `/mnt/skills/public/docx/SKILL.md` 进行创建。

## 科幻小说特有的质量标准
作为科幻小说，请始终牢记以下几点：
-科学一致性：已建立的技术体系不应存在内部矛盾。
-奇幻感：故事应包含能够激发读者好奇心和智力思考的元素。
-主题深度：故事应通过技术和未来社会探讨关于人类的根本问题。
-新颖性：故事应具有清晰且原创的科幻概念，并以此推动叙事发展。
