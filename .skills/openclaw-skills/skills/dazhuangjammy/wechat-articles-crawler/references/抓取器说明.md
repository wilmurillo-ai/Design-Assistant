# 公众号抓取龙虾

这是放在 `内容生产龙虾/公众号作者文章抓取/` 目录里的自带抓取器副本，供当前 skill 直接调用。实现脚本放在 `./scripts/`，本文件属于 `./references/` 文档层。

本地版微信公众号文章抓取工具。

特点：

- 登录态只保存在本机 `.playwright-profile/`
- 登录失效后固定生成 `login_artifacts/login_qr.png`、`login_artifacts/login_qr.txt`、`login_artifacts/login_status.json`
- `Mac/Linux` 都能用；没有图形界面的环境也可以直接扫字符二维码，或者把 `login_qr.png` 发到 IM 里扫码
- 首次启动会让你选择输出父目录，并设置单次抓取篇数
- 程序会自动在你选中的目录里创建 `输出文章/`
- 每次抓取都会在 `输出文章/` 下面新建一个按公众号名和时间命名的子文件夹
- 每篇文章同时保存 `Markdown` 和原始 `HTML`

## 首次安装

```bash
cd "/Users/jammy/Desktop/龙虾架构/内容生产龙虾/公众号作者文章抓取/scripts"
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/playwright install chromium
```

## 运行

兼容旧用法，直接传链接：

```bash
cd "/Users/jammy/Desktop/龙虾架构/内容生产龙虾/公众号作者文章抓取"
./scripts/run_fetcher.sh "https://mp.weixin.qq.com/s/你的文章链接"
```

也支持显式命令：

```bash
cd "/Users/jammy/Desktop/龙虾架构/内容生产龙虾/公众号作者文章抓取"
./scripts/run_fetcher.sh fetch "https://mp.weixin.qq.com/s/你的文章链接"
```

或者双击 `run.command` 后按提示输入文章链接。

## Agent 友好 CLI

推荐给 agent / IM / 自动化工具使用这些命令：

```bash
./scripts/run_fetcher.sh ensure-login --json --display silent
./scripts/run_fetcher.sh login-status --json
./scripts/run_fetcher.sh fetch "https://mp.weixin.qq.com/s/你的文章链接" --json --display silent
./scripts/run_fetcher.sh clear-login --json
```

命令说明：

- `ensure-login`
  - 检查并复用现有登录态
  - 如果不能复用，就生成二维码文件并等待扫码成功
- `login-status`
  - 读取当前登录状态文件
  - 适合外层 agent 轮询
- `fetch`
  - 确保登录成功后抓取文章
- `clear-login`
  - 删除本地登录缓存和二维码产物

典型 JSON 状态：

```json
{
  "status": "authenticated",
  "status_file": "/abs/path/login_status.json",
  "profile_dir": "/abs/path/.playwright-profile",
  "login_artifacts_dir": "/abs/path/login_artifacts"
}
```

```json
{
  "status": "waiting_scan",
  "qr_png_path": "/abs/path/login_qr.png",
  "qr_text_path": "/abs/path/login_qr.txt"
}
```

## 配置

首次运行后会生成 `config.json`，后续要改：

- 输出父目录
- 单次抓取篇数
- 并发数

直接编辑 `config.json` 即可。

无界面 Linux / agent 模式下，如果 `config.json` 不存在，程序会自动生成一份默认模板：

- `output_parent`
  - 当前这份 skill 副本默认使用 skill 根目录本身
  - 也就是 `内容生产龙虾/公众号作者文章抓取/`
- `article_limit`
  - 默认 `20`
- `concurrency`
  - 默认 `6`
- `display_mode`
  - 无界面环境默认 `silent`

生成后你直接改 [config.json](/Users/jammy/Desktop/龙虾架构/内容生产龙虾/公众号作者文章抓取/scripts/config.json) 就行，不会要求弹窗配置。

## 二维码登录模式

默认 `display_mode` 是 `auto`，也可以命令行覆盖：

```bash
./scripts/run_fetcher.sh --display auto "https://mp.weixin.qq.com/s/你的文章链接"
./scripts/run_fetcher.sh --display terminal "https://mp.weixin.qq.com/s/你的文章链接"
./scripts/run_fetcher.sh --display image "https://mp.weixin.qq.com/s/你的文章链接"
./scripts/run_fetcher.sh --display silent "https://mp.weixin.qq.com/s/你的文章链接"
```

- `auto`：保存二维码文件；如果当前是终端就打印字符二维码；如果本机有图形界面则尝试打开 PNG
- `terminal`：保存二维码文件，并打印字符二维码
- `image`：保存二维码文件，并优先打开 PNG
- `silent`：只生成二维码文件，不主动展示，适合外层 agent / IM 自己取图发送

## 登录状态存储位置

- 登录缓存目录：`.playwright-profile/`
- 二维码与登录状态目录：`login_artifacts/`

如果要彻底清除当前账号登录态，执行：

```bash
./scripts/run_fetcher.sh clear-login
```
