# 打包与交付

当 skill 已经准备好交付成可分发的成品时，读这份文档。

默认把 `<skill-base>` 理解为当前 skill 的 `SKILL.md` 所在目录。

- 一旦进入这份文档，先固定：`current_path` = `打包`。
- 执行过程中始终显式维护：`current_step` 和 `next_action`。
- 每次进入新步骤，或准备执行打包脚本、复制只读 skill、向用户交付文件前，先复述：
  - 当前路径：`打包`
  - 当前步骤：`Step N`
  - 下一动作：一句话
- 如果任务已经漂移成“继续改 skill”“跑评测”或“优化触发描述”，不要继续沿用这份文档；先回主 `SKILL.md` 的 Step 1 重新判路。

## Step 1：先确认现在真的该打包

- 进入这一步时，更新：
  - `current_step` = `Step 1`
  - `next_action` = 先确认 skill 已经足够稳定，值得进入交付阶段
- 如果 skill 还在频繁改结构、频繁改说明、或评测还没收口，就先不要打包。
- 如果只是想把当前版本暂存成一个包，也要向用户说明它是不是最终交付版。

## Step 2：打包前先做结构检查

- 进入这一步时，更新：
  - `current_step` = `Step 2`
  - `next_action` = 先确认结构干净、可交付，再执行打包
打包前确认：
- 主 body 只剩耐久规则和默认工作流
- 低频细节已经移出主 body
- skill 能通过校验
- 最终目录里没有明显垃圾文件或 build 产物
- 没有把本次运行的绝对路径写进最终交付物
- 没有把不必要的 `config.yaml`、`agents/openai.yaml`、`evals/` 或 workspace 结果一起塞进包里
- 如果这个 skill 带 `# 索引`，它只是恢复入口，不是第二份说明书

## Step 3：必要时先复制已安装或只读 skill

- 进入这一步时，更新：
  - `current_step` = `Step 3`
  - `next_action` = 判断是否需要先复制到可写副本再打包
如果用户是在改一个已安装 skill：
- 除非用户明确要求，否则保留原名
- 如果原位置可能只读，先复制到可写目录再改
- 从可写副本打包，不要直接对只读源目录做操作

## Step 4：执行标准打包流程

- 进入这一步时，更新：
  - `current_step` = `Step 4`
  - `next_action` = 运行打包脚本，产出 `.skill` 文件
- 进入打包命令前，先再做一次状态播报，确认你是在交付，而不是误把中间版本当成成品。
在 `<skill-base>` 下执行：

```bash
cd "<skill-base>" && python3 scripts/package_skill.py <path-to-skill-folder>
```

打包脚本会先做验证，再产出 `.skill` 压缩包。

## Step 5：把结果清楚地交给用户

- 进入这一步时，更新：
  - `current_step` = `Step 5`
  - `next_action` = 告诉用户产物在哪、改了什么、有没有跳过校验
如果环境有文件展示工具，就直接把 `.skill` 文件交给用户。

如果没有：
- 告诉用户精确输出路径
- 简短说明这个包里改了什么
- 如果有跳过的校验或打包步骤，要明确说明

## Step 6：交付后及时停下，或回主流程重判

- 进入这一步时，更新：
  - `current_step` = `Step 6`
  - `next_action` = 判断交付是否完成，还是要回主流程继续处理
- 如果用户只要交付，到这里就停。
- 如果用户接下来要继续改 skill、做评测、做触发优化，回主 `SKILL.md` 的 Step 1，不要凭惯性继续沿用打包路径。

## 索引

- 如果上下文变长、刚看完打包输出、或需要重新找回方向，先复述：
  - `current_path`
  - `current_step`
  - `next_action`
- 然后按当前路径直达对应材料：
  - 打包主线：留在这份文档，继续当前 Step
  - 结构检查：回看这份文档的 `Step 2`
  - 只读 skill / 已安装 skill：回看这份文档的 `Step 3`
  - 标准打包命令：回看这份文档的 `Step 4`
  - 如果任务已漂移成修改 / 评测 / 优化：回主 `SKILL.md` 的 Step 1，再跳对应 reference
- 这个索引只用来快速恢复上下文，不替代上面的 Step 1 到 Step 6。
