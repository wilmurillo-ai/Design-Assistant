## Best Practices | 最佳实践

---

### DO (应该做)

#### 1. 先有概念，后有技术

```
推荐的创作流程：

Step 1: 明确概念
━━━━━━━━━━━━━
在写任何代码之前，先问自己：
- 我想表达什么？
- 我想给观众什么感受？
- 作品的核心是什么？

把你的答案写下来，即使只有一句话。

Step 2: 技术探索
━━━━━━━━━━━━━
思考：什么样的技术最能服务这个概念？
- 静态图像 vs 动态动画 vs 交互体验
- 如果概念改变了，技术需要跟着变吗？

Step 3: 原型验证
━━━━━━━━━━━━━
用最简单的方式验证概念：
- 纸和笔速写
- HTML/CSS快速原型
- 代码片段测试

Step 4: 迭代深化
━━━━━━━━━━━━━
基于反馈调整：
- 概念是否清晰传递？
- 技术是否完美服务概念？
- 哪里可以更深入？

━━━━━━━━━━━━━━━━━━━━━━━━━━
记住：最好的技术选择是那些帮助你更清晰地
表达概念的选择，而不是那些展示技术能力的选择。
```

#### 2. 善用浏览器特性

```
浏览器是独一无二的创作媒介。思考"什么只能用浏览器做到"：

✅ 只有浏览器能做到的：
━━━━━━━━━━━━━━━━━━━━
- 超链接：将作品变成可探索的网络
- 响应式：不同设备不同体验
- 实时交互：即时响应用户输入
- 字体渲染：文字作为视觉元素
- CSS动画：GPU加速的流畅动画
- Canvas/WebGL：像素级控制
- 声音：音频API集成

❌ 传统媒介也能做到的：
━━━━━━━━━━━━━━━━━━━━
- 静态图像（印刷品更清晰）
- 预设动画（视频更流畅）
- 复杂3D建模（专业软件更强大）

💡 练习：
尝试用一句话描述你的作品，其中必须包含
"只有浏览器才能..." 的表述。

示例：
"这是一个探索超链接作为叙事媒介的作品，
只有浏览器才能让观众自由选择故事走向。"
```

#### 3. 追求意外之喜

```
好的艺术作品应该有让创作者自己都惊讶的元素。

如何创造意外之喜：

1. 引入随机性
   - 使用Math.random()但控制范围
   - 让系统在规则内"自由发挥"
   - 考虑用户输入对结果的不可预测影响

2. 拥抱错误
   - 观察"错误"的结果
   - 问自己：这个"错误"是否比原计划更有趣？
   - 有时意外的错误揭示了新的可能性

3. 跨领域灵感
   - 音乐如何影响视觉设计？
   - 舞蹈动作如何转化为动画？
   - 烹饪如何启发粒子系统？

4. 技术探索
   - 尝试超出自己能力范围的技术
   - 观看技术demo，寻找灵感
   - 问：还有什么我没想到的可能性？

练习：
选择一个你已完成的常规项目，
尝试引入一个"失控"的元素，
观察结果是否有意想不到的惊喜。
```

#### 4. 接受不完美

```
故障、意外、迭代都是艺术过程的一部分。

完美主义是创作的大敌。接受不完美：

1. 设定"足够好"的标准
   - 不是所有作品都需要达到博物馆级别
   - 有些作品就是实验、习作、探索
   - 完成比完美更重要

2. 善用迭代
   - 第一版：验证概念
   - 第二版：优化体验
   - 第三版：打磨细节

   不要期待第一版就完美

3. 意外的"不完美"可能是特色
   - 故意保留的手绘痕迹
   - 有机的不规则形状
   - 刻意的不完美节奏

4. 时间限制是朋友
   - 给项目设定截止日期
   - 强迫自己做出决定
   - 接受"现在能做到的最好"

反思：
你最近是否有一个项目因为追求完美而未完成？
如果是，设定一个24小时的"完成"期限，
发布你现在有的版本。
```

#### 5. 建立个人视角

```
不要复制风格，要形成自己的艺术语言。

如何发展个人视角：

1. 探索你的背景
   - 你来自哪里？
   - 你的文化背景如何影响你的审美？
   - 你的人生经历如何塑造你的世界观？

2. 深入你的兴趣
   - 你对什么话题有持久的热情？
   - 你想通过艺术探讨什么问题？
   - 什么样的主题让你废寝忘食？

3. 实验不同的影响
   - 研究多种艺术传统
   - 融合看似不相关的领域
   - 找到你独特的影响组合

4. 保持诚实
   - 创作你真正关心的作品
   - 不要追随潮流
   - 做会让你"不舒服"的实验

练习：
列出三个对你有深刻影响但很少在作品中体现的元素。
考虑如何在下一个作品中引入其中之一。
```

---

### DON'T (不应该做)

#### 1. 不要只追求视觉效果

```
炫技不等于艺术。

⚠️ 警惕这些信号：
━━━━━━━━━━━━━━━━━━
- 你的第一反应是"这看起来好酷"而不是"这让我感受到..."
- 你在技术上花费的时间远超概念探索
- 你的作品描述主要涉及视觉效果而非概念
- 观众问"这是什么"而你只能描述外观

✅ 正确的优先级：
━━━━━━━━━━━━━━━━━━
概念 > 技术 > 视觉效果

正确的问题顺序：
1. 我想表达什么？（概念）
2. 什么技术最能服务这个概念？（技术）
3. 如何让技术发挥最佳效果？（视觉效果）

练习：
给你的作品写一个"电梯演讲"——
用30秒时间向非专业人士解释你的作品。
如果你的解释只涉及视觉效果，说明需要深化概念。
```

#### 2. 不要复制现有作品

```
即使是致敬，也要有自己的转化。

⚠️ 避免的行为：
━━━━━━━━━━━━━━━━━━
- 直接复制某个艺术家的视觉风格
- 照搬现有作品的技术实现
- 使用别人的概念但不注明来源

✅ 健康的致敬方式：
━━━━━━━━━━━━━━━━━━
1. 研究：理解原作的历史和意图
2. 转化：用你自己的视角重新诠释
3. 时代：用当代技术/语境重新想象
4. 个人：加入你自己的经历和感受
5. 透明：如果是明显的致敬，明确说明

例子：
❌ 错误：复制TeamLab的樱花飘落效果
✅ 正确：研究物哀美学，理解"美的短暂性"，
         用你自己的技术重新诠释，体现你对时间的新思考

练习：
选择一件你非常喜欢的艺术作品，
问自己："如果我创作这件作品，
     我会加入什么我自己的东西？"
```

#### 3. 不要忽视用户体验

```
艺术作品也需要考虑观众的感受。

⚠️ 常见问题：
━━━━━━━━━━━━━━━━━━
- 作品无法在某些设备上运行
- 动画过快或过慢，让人不适
- 交互方式不直观
- 作品让人困惑，不知道该做什么

✅ 基本原则：
━━━━━━━━━━━━━━━━━━
1. 测试多种设备和浏览器
2. 考虑不同身体能力的观众
3. 提供清晰的入口和引导
4. 尊重观众的时间和注意力

艺术性与可用性的平衡：

这不是非此即彼的选择。
好的艺术体验往往也是好的人类体验。
思考：你的艺术概念如何通过更好的用户体验得到加强？

练习：
让你的朋友/家人观看你的作品。
不要给任何说明，只是观察他们的反应。
他们是否知道该做什么？
他们是否感到不适？
他们是否想要探索更多？
```

#### 4. 不要过度依赖库

```
尝试自己实现核心效果。

⚠️ 过度依赖的信号：
━━━━━━━━━━━━━━━━━━
- 每个项目都依赖相同的库
- 你不知道库在做什么
- 你无法在没有库的情况下实现同等效果

✅ 什么时候用库：
━━━━━━━━━━━━━━━━━━
- 性能优化（Three.js的WebGL封装）
- 跨平台兼容性（响应式库）
- 非核心功能（工具函数）

❌ 不需要库的场景：
━━━━━━━━━━━━━━━━━━
- CSS可以实现的动画
- 原生Canvas可以实现的粒子系统
- 原生API可以获取的用户输入

练习：
选择你常用的一个库功能，
尝试用原生代码实现它。
这不仅能加深理解，
还可能在过程中发现独特的方法。
```

#### 5. 不要害怕简单

```
有时候最简单的反而最有力。

✅ 简单的价值：
━━━━━━━━━━━━━━━━━━
- 更清晰的传达
- 更快的加载
- 更广泛的可访问性
- 更容易理解和维护

著名的简单作品：
- John Baldessari的单色画
- Agnes Martin的网格画
- Robert Morris的极简雕塑
- 草间弥生的单一圆点重复

练习：
选择一个你觉得"太简单"的草图，
问自己：是否可以用更少的元素表达同样的概念？
是否可以删除某些"装饰"而保留核心？
如果答案是肯定的，删除并观察效果。
```

---

### 总结

```
DO:
✅ 先有概念，后有技术
✅ 善用浏览器特性
✅ 追求意外之喜
✅ 接受不完美
✅ 建立个人视角

DON'T:
❌ 不要只追求视觉效果
❌ 不要复制现有作品
❌ 不要忽视用户体验
❌ 不要过度依赖库
❌ 不要害怕简单
```

记住：最好的艺术作品往往是那些让你觉得"我也能做"但又比你做的好得多的作品。学习技术，但不要被技术所限。保持好奇，保持实验，保持真诚。
