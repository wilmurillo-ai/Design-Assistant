# 数据声明

本 skill 在运行过程中涉及以下数据交互，特此声明，确保透明、可核查。

---

## 一、用户数据

### 主动采集
小窗幽在对话中会主动询问并记忆以下用户信息，用于个性化推荐：

| 数据类型 | 示例 | 用途 | 存储位置 |
|----------|------|------|----------|
| 城市/地区 | "你在北京？" | 天气查询、踏青推荐 | MEMORY.md |
| 体质倾向 | "体质偏寒" | 饮食、茶饮建议 | MEMORY.md |
| 即时状态 | "最近有点累" | 养生建议参考 | 会话上下文（不持久化） |
| 偏好反馈 | "太复杂了" | 内容风格调整 | 会话上下文（不持久化） |

以上均为**用户主动自愿分享**，非强制采集。

### 禁止采集
- 真实姓名、身份证号、手机号
- 病历、医疗诊断记录
- 银行卡、金融账户信息
- 未明确授权的任何个人隐私

### 关于"主观状态描述"的特别说明
"最近有点累""精神不太好"等主观感受属于一般性状态描述，用于推荐参考，**不涉及医疗诊断**，允许作为上下文记录。
若用户主动透露具体疾病名称、病历或处方用药信息，不记录，并回复"此为医疗信息，建议直接咨询医生"。

---

## 二、外部查询

### 网页搜索
当用户询问时效性内容（踏青路线、节假日活动、价格信息、天气数据）时，skill 调用 `batch_web_search` 实时查询，查询结果仅用于本次回复，不持久化存储。

### 天气数据
- **来源**：wttr.in（世界天气在线）
- **接口**：`wttr.in/<城市>?format=j1`
- **精度**：3天内较准确，7天以上仅供参考
- **标注**：每次天气回复均标注来源

---

## 三、本地文件存储

### 存储路径

| 文件 | 内容 | 路径 |
|------|------|------|
| MEMORY.md | 用户画像、长期偏好、进行中计划 | `/workspace/MEMORY.md` |
| 日记文件 | 每日对话日志摘要 | `/workspace/memory/YYYY-MM-DD.md` |

### 访问权限说明（声明范围）

skill 层面能确认的事实：

- skill 本身在当前 OpenClaw 实例内读写这些文件
- skill 文件（SKILL.md 及 references/）打包上传至 ClawHub 时，**不包含** MEMORY.md 及 memory/ 日记文件
- skill 没有配置任何外部凭据或环境变量

**超出 skill 控制范围的部分**（以下由平台与运维策略决定，不由 skill 保证）：

- 服务器管理员是否有物理/SSH 访问权限
- OpenClaw 平台是否有内部审计日志
- 其他已安装插件是否拥有同目录访问权限

> 如需了解当前部署环境的实际文件访问策略，请咨询平台运维。

---

## 四、跨会话记忆机制

- MEMORY.md 中的用户画像可在后续对话中继续使用（下次会话时读取）
- 不同 session 之间**不自动建立用户身份关联**，仅读取用户主动留下的通用偏好
- 若用户要求"删除我的记忆"，agent 将清空 MEMORY.md 中该用户的对应条目

---

## 五，定时更新任务说明

skill 配置了每月1日（北京 06:00）自动触发的更新任务：

- **任务类型**：isolated agentTurn job，由 OpenClaw cron 系统调度
- **执行内容**：检查节气标注、气候规律、用户 memory 是否有遗漏
- **运行权限**：由 OpenClaw 平台提供，与当前 agent 会话隔离执行
- **取消方式**：用户可通过 agent 指令 `cron remove` 删除此任务

---

## 六，透明性承诺

1. **不伪造数据**：无法确认的信息，主动告知"建议核实"，而非编造
2. **不强制记忆**：用户可随时要求删除已记录的个人信息
3. **不跨身份追踪**：不同会话之间不建立用户身份关联
4. **文件可控**：用户可随时查看、编辑、删除存储文件
5. **建议仅供参考**：养生、饮食、出行建议不替代专业医疗与服务
6. **不依赖外部凭据**：本 skill 不要求任何 API Key、环境变量或外部账号

---

> 本声明适用于 xiao-chuang-you skill v1.7.0 及以上版本。
