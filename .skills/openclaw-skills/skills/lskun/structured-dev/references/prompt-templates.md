# Prompt 模板

各阶段可直接复用的提示词模板。根据项目语言和场景替换占位符。

---

## Phase 1: Research

### 通用调研
```
深入阅读 {目标目录}，理解它的工作原理、功能和所有细节。
走通 {核心流程} 的完整调用链。
完成后，把你的发现写成 .dev/research.md，包含：
1. 架构理解（模块关系、数据流）
2. 相关文件清单及职责
3. 现有约定和规范
4. 潜在风险和边界条件
不要遗漏任何细节。浅层阅读不可接受。
```

### 带 Bug 排查的调研
```
深入阅读 {目标目录}，理解 {功能模块} 的完整工作流程。
系统存在一个问题：{问题描述}。
持续排查，直到找出所有可能的原因。
把调研结果和排查发现写成 .dev/research.md。
```

### 多模块调研
```
这个任务涉及 {模块A}、{模块B} 和 {模块C} 的交互。
分别深入理解每个模块，然后重点分析它们之间的接口和数据流。
特别关注：{关注点}。
输出 .dev/research.md。
```

## Phase 2: Plan

### 通用方案
```
基于 .dev/research.md 的调研结果，为以下需求编写详细实现方案：
{需求描述}

输出 .dev/plan.md，必须包含：
1. 实现思路和整体方案
2. 将修改的文件路径及每个文件的具体改动
3. 关键代码片段（实际代码，不是伪代码）
4. 取舍说明（为什么选这个方案）
5. 对现有模块的影响
6. 明确不做什么

先不要实现。
```

### 基于参考实现的方案
```
参考这段实现：
{参考代码或链接}

这是 {项目/库} 中 {功能} 的实现方式。
编写 .dev/plan.md，说明我们如何采用类似方案适配到我们的项目中。
注意我们的 {差异点}。

先不要实现。
```

## Phase 3: Annotate

### 处理批注
```
我在 .dev/plan.md 中添加了批注。
逐条处理所有批注并更新文档。
明确标注每条批注的处理方式。
先不要实现。
```

### 重新审视某一节
```
重新审视 plan.md 中 {章节名} 的设计。
考虑到 {新约束/新信息}，更新这一节。
其他部分保持不变。先不要实现。
```

---

## Phase 4: Todo List

### 生成任务清单
```
在 .dev/plan.md 末尾追加"实施清单"章节。
按阶段拆分为细粒度任务，每个任务一行 checkbox。
任务粒度：一个函数、一个文件修改、一个测试。
先不要实现。
```

---

## Phase 5: Implement

### 标准实现（TypeScript 项目）
```
全部实现。每完成一个任务，在 .dev/plan.md 中标记为已完成。
在所有任务全部完成之前不要停下。
不要添加不必要的注释或 JSDoc，不要使用 any 或 unknown 类型。
持续运行 tsc --noEmit，确保不引入类型错误。
```

### 标准实现（Python 项目）
```
全部实现。每完成一个任务，在 .dev/plan.md 中标记为已完成。
在所有任务全部完成之前不要停下。
不要添加不必要的注释，保持与项目现有代码风格一致。
实现完成后运行 pytest 确保测试通过。
```

### 标准实现（Vue/前端项目）
```
全部实现。每完成一个任务，在 .dev/plan.md 中标记为已完成。
在所有任务全部完成之前不要停下。
保持与项目现有组件风格一致。
实现完成后运行 npm run build 确保无编译错误。
```

### Spawn Coding Agent 实现
```
读取 .dev/plan.md 中的实施清单。
按顺序逐项完成所有任务。
每完成一个任务，更新 plan.md 标记为 [x]。
遇到方案中没覆盖的问题时，写入 .dev/questions.md 并继续下一个任务。
完成后运行项目的 lint 和 typecheck。
```

## Phase 6: Feedback

### 简短修正
```
{具体修正指令}
```

### 参照修正
```
这个 {组件/函数} 应该跟 {参照目标} 完全一样。
相同的 {具体属性}。
```

### 回滚重来
```
回滚所有改动。
现在只做 {缩小后的范围}，别的什么都不要动。
```

## 特殊场景

### 恢复中断的会话
```
读取 .dev/research.md 和 .dev/plan.md，恢复上下文。
继续从实施清单中未完成的任务开始。
```

### 需求变更
```
需求有变化：{变更描述}。
更新 .dev/plan.md 中受影响的部分。
重新评估实施清单，标记需要修改或新增的任务。
先不要实现，等我审查。
```
