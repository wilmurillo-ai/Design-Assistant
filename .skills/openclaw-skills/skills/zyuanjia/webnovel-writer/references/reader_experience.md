# 读者体验模拟指南

> 写完一章不是结束，而是验证的开始。本文档提供系统化方法，模拟5种典型读者的反应，在发布前识别弃书风险。

---

## 一、5种读者类型画像

### 1. 猎奇型读者

**核心驱动力**：新鲜设定、猎奇概念、"没见过这种"

| 维度 | 描述 |
|------|------|
| 弃书触发点 | 设定解释完就跑（设定党，只看创意不看执行）；同类套路重复出现；金手指/系统太常规无新意 |
| 满足点 | 开篇3章内展现独特世界观；每10章左右有新设定信息揭露；"设定层层剥洋葱"的快感 |
| 黄金节奏 | 前3章抛核心设定→5-10章扩展世界观→每卷开新设定层→不要一口气倒完 |
| 弹幕特征 | "这个设定有意思""第一次见这种""等等所以XX是YY？" |

### 2. 情感型读者

**核心驱动力**：共情、角色羁绊、"哭了""心疼"

| 维度 | 描述 |
|------|------|
| 弃书触发点 | 角色行为不可理喻（为剧情强行降智）；情感铺垫不足突然爆发；主要角色长期不互动；过度虐心无回报 |
| 满足点 | 用行为/细节传达情感而非形容词；角色关系有明确的进展里程碑；虐之后有甜、哭之后有暖 |
| 黄金节奏 | 每3-5章一个情感小高潮→每卷一个催泪名场面→虐甜比约3:7→催泪后接1-2章缓冲日常 |
| 弹幕特征 | "呜呜呜""心疼XX""这段看哭了""别说了我眼泪止不住" |

### 3. 推理型读者

**核心驱动力**：猜真相、找伏笔、"我早猜到了"/"反转绝了"

| 维度 | 描述 |
|------|------|
| 弃书触发点 | 伏笔回收敷衍（"就这？"）；推理线长期无进展（超过15章没有新线索）；答案靠天降/主角光环而非推理；设定矛盾被无视 |
| 满足点 | 线索公平递呈（读者可以跟着猜）；伏笔回收有"恍然大悟"感；多层反转（至少2层）；关键信息前面确实埋过 |
| 黄金节奏 | 每5章给1条明确线索→每卷中段设1个误导→卷末反转→跨卷大伏笔不超过50章回收周期 |
| 弹幕特征 | "我就说XX有问题！""这个伏笔早就埋了""等等回到第X章再看一遍""细思极恐" |

### 4. 爽文型读者

**核心驱动力**：爽感、打脸、逆袭、"太爽了"

| 维度 | 描述 |
|------|------|
| 弃书触发点 | 主角连续3章以上被动挨打；装逼被拆台无后续反击；日常拖沓超过2章无冲突；配角戏份过多抢主角风头 |
| 满足点 | 每3章一个小爽点（小胜/小打脸）；每10章一个中爽点（阶段性胜利）；每卷一个大爽点（终极打脸/实力碾压）；爽完立刻接新危机 |
| 黄金节奏 | 短章快节奏（2000-3000字/章）；对话推进剧情不水字；打脸-装逼循环间隔不超过5章；低谷期不超过2章 |
| 弹幕特征 | "爽！""笑死""主角牛逼""前方高能""打脸来得太快" |

### 5. 细节型读者

**核心驱动力**：沉浸感、世界真实感、"像真的一样"

| 维度 | 描述 |
|------|------|
| 弃书触发点 | 明显常识错误（时间/地理/逻辑）；设定前后矛盾；环境描写空洞（"一个房间"）；配角脸谱化 |
| 满足点 | 环境描写有五感细节；角色行为符合其性格逻辑；时间线可追溯；世界观自洽 |
| 黄金节奏 | 关键场景3-5句环境锚定→日常场景1-2句点睛→不堆砌描写但每次描写都有质感→每10章补充1次世界观细节 |
| 弹幕特征 | "这里有个bug""等等时间对不上""作者功课做得好""细节好评" |

## 二、章节级读者体验评估流程

写完每章后，按以下5步模拟：

### 步骤1：标记本章类型

| 章节类型 | 标记 |
|----------|------|
| 日常过渡 | `type:日常` |
| 紧张推进 | `type:紧张` |
| 催泪情感 | `type:催泪` |
| 悬念铺设 | `type:悬念` |
| 高潮反转 | `type:高潮` |
| 信息揭露 | `type:揭露` |

### 步骤2：对每种读者模拟打分（1-5分）

```
读者类型  | 吸引力 | 风险项
猎奇型   | ?/5   | ?
情感型   | ?/5   | ?
推理型   | ?/5   | ?
爽文型   | ?/5   | ?
细节型   | ?/5   | ?
```

### 步骤3：判断标准

- **任意读者 ≤ 2分**：该类型读者高流失风险，需要修改
- **所有读者 ≥ 3分**：及格线，可以发布
- **核心读者群 ≥ 4分**：良好（核心读者群见第四节小说分析）

### 步骤4：连续追踪

每5章画一次雷达图趋势：

```
章节1-5：  猎奇4 情感3 推理3 爽文4 细节3
章节6-10： 猎奇3 情感4 推理4 爽文3 细节4
...
```

**趋势预警**：任何读者类型连续3个5章窗口下降 → 需要针对性补强。

### 步骤5：修改优先级

1. 核心读者群弃书风险 → 必须修
2. 次要读者群弃书风险 → 尽量修
3. 所有读者低于3分 → 必须修
4. 单读者低于3分但非核心 → 可选修

## 四、两部小说的读者画像分析

### 《我的死亡日期消失了》—— 悬疑/都市

**核心设定**：男主能看见人头上的死亡倒计时数字，自己的死亡日期突然消失了。

#### 读者画像

| 维度 | 分析 |
|------|------|
| 核心读者群 | **推理型（40%）+ 猎奇型（30%）** |
| 次要读者群 | 爽文型（15%）+ 情感型（10%）+ 细节型（5%） |
| 获客钩子 | "看见死亡倒计时"——猎奇型秒入；"消失"——推理型被勾住 |
| 留存关键 | 悬疑线的层层推进、伏笔的精妙回收、反转的质量 |
| 风险点 | 悬疑解密后猎奇型流失（需持续提供新谜题）；推理线拖沓导致推理型弃书 |

#### 各读者类型满足策略

- **猎奇型**：死亡倒计时的规则持续扩展（谁能看见？数字能改变吗？消失意味着什么？），每5章左右揭露一个新规则
- **推理型**：核心悬疑线（消失原因）贯穿全书，每卷设子谜题，跨卷大线索串联。公平递呈线索，让读者能参与推理
- **情感型**：陆远看到亲人/朋友头上的数字时的心痛；配角死亡时的催泪。**策略**：用"知道死期却无力改变"的绝望感制造情感钩子
- **爽文型**：利用信息差（知道别人不知道的事）制造"上帝视角爽感"；解谜成功时的智力碾压感
- **细节型**：时间线必须严格（每个死亡日期都对应得上）；世界观规则必须自洽

#### 关键节奏参数

```
悬疑解密周期：每卷一个大谜题，卷末揭晓
伏笔密度：每章至少埋1条微伏笔，每5章1条中伏笔
猎奇型喂料周期：每5-7章一个新设定信息
情感型催泪点：每卷至少1个催泪名场面
```

### 《观众都是我的前世》—— 催泪/都市

**核心设定**：主角的每个前世都是观众，在"轮回影院"里看自己的人生。

#### 读者画像

| 维度 | 分析 |
|------|------|
| 核心读者群 | **情感型（45%）+ 猎奇型（25%）** |
| 次要读者群 | 推理型（15%）+ 细节型（10%）+ 爽文型（5%） |
| 获客钩子 | "前世都是观众"——猎奇型+情感型双重钩子；"看自己的人生"——共情爆点 |
| 留存关键 | 角色关系的深度刻画、催泪场景的密度和质量、前世故事的多样性 |
| 风险点 | 催泪疲劳（连续虐心导致麻木）；前世故事套路化；推理线偏弱导致推理型流失 |

#### 各读者类型满足策略

- **情感型**：每3-5章一个情感高光时刻；前世故事聚焦"遗憾"主题引发共鸣；用日常细节铺情感而非直说。参考风格："温暖日常中藏锋芒、对话活泼、弹幕幽默但关键时刻安静"
- **猎奇型**：前世身份的多样性（皇帝、乞丐、杀手……）；轮回影院的规则持续揭露；每次进入新前世都是新世界
- **推理型**：为什么所有前世都是观众？轮回影院的本质是什么？——这条元悬疑线留住推理型
- **细节型**：每个前世的世界观需要自洽；历史细节（如果涉及古代前世）需要准确
- **爽文型**：利用前世记忆/能力的跨世加成制造爽感（不是核心，但是锦上添花）

#### 关键节奏参数

```
催泪点周期：每3-5章一个小催泪，每卷一个大催泪
虐甜比：3:7（虐是为了甜更甜）
前世故事更新：每卷1-2个前世完整故事
猎奇型喂料：每10章左右揭露轮回影院新规则
推理线推进：每卷给1条关于轮回影院本质的线索
```

## 六、快速参考卡片

**写每章时问自己4个问题：**

1. 猎奇型读者能看到新东西吗？（新设定/新信息）
2. 情感型读者能感到什么吗？（共情/心疼/温暖）
3. 推理型读者能猜什么吗？（新线索/新谜题）
4. 爽文型读者能爽到吗？（小胜/打脸/逆袭）

**每5章回顾时问自己2个问题：**

5. 细节型读者能挑出毛病吗？（时间线/设定矛盾）
6. 哪种读者类型在最近5章的得分在持续下降？

**4个问题中至少覆盖2个 = 及格。覆盖3个 = 良好。覆盖4个 = 优秀。**

## 七、弃书点检测

> 什么样的段落让读者想关掉页面？以下是7类高危弃书点及其检测方法。

### 7.1 弃书点分类与识别

| 弃书点类型 | 典型表现 | 严重度 | 检测方法 |
|-----------|---------|--------|----------|
| **设定倾倒** | 连续超过300字纯世界观解释，无角色动作穿插 | 🔴高 | 统计连续无对话/动作的描写段落长度 |
| **角色降智** | 角色做明显不符合智力/性格的决定，只为推进剧情 | 🔴高 | 对照 character_states.json 的 personality 字段检查行为合理性 |
| **情感跳崖** | 没有铺垫突然大哭/暴怒/表白，前一章还在正常对话 | 🔴高 | 检查情感词密度突变（相邻章节情感基调变化>2级） |
| **重复套路** | 相似情节结构在10章内重复出现≥3次 | 🟡中 | 用相似度检测比对章节结构（打脸→被嘲讽→再打脸的循环） |
| **信息真空** | 连续2章没有新信息（新设定/新线索/新关系进展） | 🟡中 | 标记每章新增信息量，连续低值触发预警 |
| **节奏塌陷** | 连续3章以上紧张度低于3（10分制） | 🟡中 | rhythm_check.py 的连续单调检测 |
| **视角漂移** | 突然切到不相关角色的内心独白超过1段 | 🟡中 | consistency_check.py 视角混乱检测 |

### 7.2 章节开头3秒检测

读者决定是否继续读下一章，通常只看开头3行。检测清单：

- [ ] 开头3行是否有新信息或悬念？（不是上章总结）
- [ ] 开头是否有动作或对话？（不是环境描写）
- [ ] 前100字是否出现人名？（不是代词"他"）
- [ ] 开头是否与上章结尾有明确的衔接或反差？

**不通过任意2项 = 高弃书风险开头，建议重写。**

### 7.3 章末钩子有效性检测

章节末尾是决定读者是否点"下一章"的关键：

- [ ] 最后200字是否有悬念/反转/情感冲击？
- [ ] 章末是否有未解的问题？（读者想知道答案）
- [ ] 章末是否有情感留白？（不是完全收束）
- [ ] 是否避免了"总结式结尾"（把本章事件复述一遍）？

## 八、阅读疲劳检测

> 读者不是机器，连续相同节奏会疲劳。以下是量化检测方法。

### 8.1 疲劳阈值参考表

| 内容类型 | 连续无转折最大安全字数 | 超过后表现 |
|---------|---------------------|----------|
| 打斗场景 | 1500字 | 招式重复感、读者跳读 |
| 日常对话 | 2000字 | 水字感、"什么时候进入正题" |
| 世界观解释 | 800字 | 设定党满足但其他人流失 |
| 情感描写 | 1200字 | "矫情""行了知道你难过" |
| 悬念铺垫 | 2500字 | 期待感耗尽、"到底说不说" |
| 单一情绪状态 | 3000字（约1章） | 情感麻木、催泪失效 |

### 8.2 转折点密度公式

```
转折密度 = 关键转折点数 / (总字数 / 1000)

健康范围：
- 快节奏小说（爽文/悬疑）：2.0-3.0（每千字2-3个转折）
- 慢节奏小说（文艺/催泪）：1.0-2.0（每千字1-2个转折）
- 低于1.0 = 阅读疲劳高风险
```

**关键转折点包括：** 新信息揭露、立场转变、关系变化、意外事件、情绪翻转、伏笔回收。

### 8.3 自动化检测（rhythm_check.py 集成）

rhythm_check.py 的连续单调检测已覆盖节奏塌陷，以下是补充的手动检测项：

1. **同义词堆砌检测**：同一情绪词（如"愤怒"）在500字内出现≥3次 → 疲劳信号
2. **句式重复检测**：连续5句以上相同句式结构（"他...了。他...了。他...了。"）→ 疲劳信号
3. **对话堆砌检测**：连续8行以上纯对话无动作/心理穿插 → 跳读信号
4. **描写堆砌检测**：连续200字以上纯环境/外貌描写无剧情推进 → 跳读信号

## 九、情感共鸣点评估

> 好的小说不是让读者"知道"角色难过，而是让读者"跟着"难过。

### 9.1 共鸣强度评估矩阵

| 共鸣等级 | 读者反应 | 文本特征 | 示例 |
|---------|---------|---------|------|
| 5-爆哭 | 放下手机平复情绪 | 核心情感+细节铺垫+出其不意+角色成长 | 前世终于释怀、生死离别 |
| 4-泪目 | 鼻酸但能继续看 | 情感积累到临界+一个触发细节 | 发现一直被默默保护 |
| 3-触动 | 停下来回味一下 | 共情角色处境+真实细节 | 角色说出读者想说的话 |
| 2-理解 | 知道应该感动但没感觉 | 情感铺垫不足/套路化 | "他悲痛欲绝"（空喊口号） |
| 1-无感 | 跳过不看的冲动 | 强行煽情/煽情时机不对 | 没铺垫突然角色牺牲 |

### 9.2 共鸣三要素检查

每个情感场景检查是否满足以下三要素（缺一个降一级）：

1. **铺垫充分**：情感爆发前至少有2章的细节暗示（小动作、小对话、小变化）
2. **细节真实**：用具体的、感官的、独特的细节传达情感，而非形容词堆砌
3. **情感意外**：不是读者能预判的"这里该哭了"，而是出其不意的触发点

### 9.3 共鸣点分布检查

```
每卷检查：
- Level 5（爆哭）：1-2个
- Level 4（泪目）：2-3个  
- Level 3（触动）：3-5个
- Level 2 以下：0个（有的话需要重写）

警告信号：
- 连续10章无 Level 3+ 共鸣点 → 情感型读者流失风险
- 单章超过2个 Level 4+ 共鸣点 → 情感过载，催泪疲劳
```

### 9.4 反共鸣检测（必须避免）

| 反共鸣模式 | 表现 | 修复方法 |
|-----------|------|----------|
| **强行煽情** | 悲惨事件堆砌但无铺垫 | 删掉事件，先花2章建立角色关系 |
| **情绪通货膨胀** | 每章都在哭/每章都在爆发 | 降低日常章节的情感强度，留空间 |
| **说教代替共情** | 角色长篇大论讲道理 | 用行为和选择传达，不用台词 |
| **全知视角泄露** | 提前揭示角色不知道的信息 | 严格守视角，只写角色能感知的 |
