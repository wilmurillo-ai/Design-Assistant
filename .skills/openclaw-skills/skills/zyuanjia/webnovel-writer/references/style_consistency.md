# 文风一致性指南

> 从多卷衔接实践中提炼，适用于长篇网络小说（14卷+、500章+级别）。

---



### 1. 文风维度定义

#### 四维模型

| 维度 | 定义 | 检测指标 |
|------|------|---------|
| **句式长度** | 平均每句字数 | 长句(>30字)/中句(15-30)/短句(<15) 的比例 |
| **用词风格** | 口语化 vs 文学化程度 | 书面语词汇占比、成语密度、专业术语频率 |
| **描写密度** | 环境描写 vs 叙事推进的比重 | 每1000字中纯描写段落数量 |
| **对话比例** | 对话文本占总文本的比例 | 对话行数 / 总行数 |

#### 基准值（网络小说标准）

| 维度 | 建议范围 | 说明 |
|------|---------|------|
| 句式长度 | 短句40-50%、中句30-40%、长句10-20% | 网文以短句为主，节奏明快 |
| 用词风格 | 口语化70%+ | 网文读者不吃纯文学那套 |
| 描写密度 | 每1000字1-2段环境描写 | 点缀而非堆砌 |
| 对话比例 | 35-50% | 太少不生动，太多像剧本 |

#### 按类型的基准偏移
- **悬疑类**：短句比例可提高到55%+，对话偏少（30-40%），描写密度1段/千字
- **催泪/都市类**：中长句可增多，对话偏多（40-55%），描写密度1.5段/千字
- **战斗/动作场景**：短句可达70%，对话降到20-25%
- **情感/回忆场景**：中长句可达50%，描写密度3段/千字

### 2. 文风检测方法

#### 自动统计脚本指标

每章完成后，应统计以下基础数据（可脚本化）：

```python
# 核心检测指标
metrics = {
    "字数": total_chars,
    "段落数": total_paragraphs,
    "对话段落数": dialogue_paragraphs,
    "对话占比": dialogue_pct,           # 目标: 35-50%
    "平均句长": avg_sentence_length,     # 目标: 18-25字
    "短句占比(<15字)": short_sentence_pct, # 目标: 40-55%
    "环境描写段数": description_paragraphs,
    "描写密度": desc_per_1k,            # 目标: 1-2段/千字
    "问号数量": question_marks,          # 过多=角色总在提问
    "感叹号数量": exclamation_marks,     # 过多=情绪过激
    "省略号数量": ellipsis_count,        # 过多=犹豫不决风格
}
```

#### 偏移阈值

| 指标 | 黄灯（轻微偏移） | 红灯（需要修正） |
|------|-----------------|-----------------|
| 对话占比 | <30% 或 >55% | <20% 或 >65% |
| 平均句长 | <15字 或 >30字 | <12字 或 >40字 |
| 短句占比 | <35% 或 >60% | <25% 或 >75% |
| 描写密度 | <0.5 或 >3段/千字 | <0.2 或 >5段/千字 |
| 感叹号/千字 | >8个 | >15个 |

#### 检测频率
- 每5章做一次统计检查
- 关键章节（卷首、卷末、高潮章）逐章检查
- 发现红灯立即修正，黄灯在下章自然调整

### 3. 文风偏移的修正策略

#### 常见偏移类型及修正

| 偏移类型 | 表现 | 修正方法 |
|---------|------|---------|
| **文学化偏移** | 句子变长、描写变多、用词变雅 | 砍掉形容词、拆长句为短句、加对话打断描写 |
| **口语化过度** | 全是短句、没有描写、像聊天记录 | 每3-5段对话插入1段动作/环境描写 |
| **对话泛滥** | 连续十几行纯对话 | 用动作描写打断（"他端起杯子""她皱了皱眉"） |
| **描写泛滥** | 连续几段纯环境描写 | 用角色反应/内心想法打断，或者直接砍半 |
| **节奏断裂** | 突然从紧张跳到平淡 | 检查段落过渡，加入情绪过渡句 |
| **AI味加重** | 用词过于工整、排比过多、说教感 | 参照标杆段落，刻意制造不工整（口语打断、句式变化） |

#### 修正优先级
1. **不改动剧情**：风格修正不能改变情节走向
2. **不改动人物性格**：对话风格是角色的一部分
3. **最小修改原则**：调句式 > 调段落 > 重写段落
4. **保持节奏**：修正后重读，确保阅读节奏自然

### 4. 风格标杆段落机制

#### 机制说明
每部小说保存3-5段"标杆段落"，代表该小说的理想文风。写新章节时对照标杆，发现偏移及时调整。

#### 标杆选择标准
- **代表性**：能代表该小说的核心风格（不是最好的段落，而是最"对味"的段落）
- **多样性**：覆盖不同场景类型（日常、紧张、情感、对话、描写）
- **长度**：每段200-400字

#### 标杆存储格式

```
# 风格标杆段落

## 小说：《XXX》

### 标杆1：日常场景（来源：第X章）
[段落内容]

### 标杆2：紧张场景（来源：第X章）
[段落内容]

### 标杆3：情感场景（来源：第X章）
[段落内容]

### 标杆4：对话场景（来源：第X章）
[段落内容]

### 标杆5：描写场景（来源：第X章）
[段落内容]
```

#### 使用方法
1. **写作前**：重读标杆段落，进入正确风格频道
2. **写作中**：每写1000字，对照标杆检查句式和用词
3. **审校时**：发现偏移，用标杆段落作为"校准参照"重写

#### 标杆更新
- 每50章回顾一次标杆段落
- 如果小说风格有意转变（新卷基调变化），可以更新标杆
- 保留旧标杆作为历史参考

#### 实际标杆示例（需在实际写作中填充）

**悬疑类标杆（参考《我的死亡日期消失了》风格）：**
> 短句为主、冷静克制、恐怖感通过细节传递、不直说"害怕"但让读者自己害怕

**催泪/都市类标杆（参考《观众都是我的前世》风格）：**
> 温暖日常中藏锋芒、对话活泼、弹幕幽默但关键时刻安静、情感用行为而非形容词表达

## Agent 使用说明

### 初始化
每部新小说开写前：
1. 在小说工作目录创建 `style_benchmark.md`（风格标杆段落）
2. 确定该小说的四维基准值（参考上方"按类型的基准偏移"）
3. 记录在 `style_benchmark.md` 头部

### 写作时
- 跨卷章节：参照"卷首开篇模板"和"卷末收束模板"
- 新角色引入：参照"3章法则"
- 时间跳跃：参照对应跨度的处理方式

### 审校时
- 每5章运行文风统计检测
- 黄灯提醒，红灯必须修正
- 参照标杆段落校准

### Shell 命令参考

```bash
# 统计章节基础数据（需配合 consistency_check.py 或自定义脚本）
python3 scripts/consistency_check.py <正文目录> --dict 设定词典.md

# 查看风格标杆
cat <小说目录>/style_benchmark.md

# 更新标杆
# 手动编辑 style_benchmark.md，选择最具代表性的段落
```
