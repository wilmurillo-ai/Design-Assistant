# 深度审读指南

## 原则

**审读 = 像人类编辑一样逐字读，不是用正则扫。**

脚本检测能发现字面问题（人名写错、数值矛盾），但发现不了：
- 角色行为不符合人设
- 情感断裂（上一章还在悲伤，下一章突然正常）
- 逻辑漏洞（这事在设定下不合理）
- 节奏问题（这段拖了、那段赶了）
- AI味（套话、空泛描写、不自然的比喻）

这些只能靠**理解**。

## 审读前准备

必须先加载（缺一不可）：

1. **大纲.md** — 知道剧情往哪走，才能判断正文是否跑偏
2. **设定词典.md** — 专有名词、称呼规范
3. **character_states.json** — 角色当前状态（关系、情感、能力）
4. **追踪表.md** — 近10章摘要，了解前情
5. **foreshadowing.json** — 未回收的伏笔

## 逐章精读流程

### 第一步：通读全文

- 不要跳读，不要只看开头结尾
- 读的过程中在心里回答：这章在讲什么？
- 如果读完说不出来 → 说明这章没有明确主线

### 第二步：逐段分析

对每一段问6个问题：

1. **推进什么？** — 剧情/情感/伏笔/世界观，至少要占一个
2. **角色反应合理吗？** — 符合人设 + 当前处境 + 上一章状态
3. **读者感受？** — 到这里会无聊/紧张/感动/困惑/想看下去？
4. **衔接上了吗？** — 跟上一段的信息/情绪/时间线是否连贯
5. **多余吗？** — 删掉这段，剧情是否受影响？不受影响就删
6. **AI味？** — 套话/空泛描写/不自然比喻/过度形容词

### 第三步：记录发现

每章写精读笔记，格式：

```markdown
## 第N章：标题

### 剧情概述
（30-50字，用自己话概括。如果概括不出来，说明这章有问题）

### 出场角色
- 角色A：什么状态，做了什么
- 角色B：什么状态，做了什么

### 情感走向
（本章情绪从哪到哪，用一个词概括）

### 发现的问题
- [P0/逻辑] 具体问题 + 为什么不合理
- [P1/角色] 具体问题 + 为什么不符合人设
- [P1/情感] 具体问题 + 情绪断层在哪
- [P2/节奏] 具体问题 + 哪里拖了/赶了

### 写得好的地方
（也要记录，不要只挑毛病）
```

## 跨章分析（全部精读完后）

### 角色弧线检查

1. 主角从第1章到最后一章，心态变化是否连续？
2. 有没有"突然开窍"/"突然黑化"/"突然原谅"的断裂？
3. 配角有没有工具人化（只在需要时出现）？

### 情感曲线检查

1. 整体有没有起伏？连续3章以上平淡就是问题
2. 高潮章的情感是否足够到位？有没有"高潮了但读者没感觉"的情况
3. 低谷章是否在铺垫？还是单纯的拖？

### 伏笔检查

1. 埋的伏笔有没有过期没回收的？
2. 回收伏笔时，读者是否有"啊原来如此"的感觉？
3. 有没有忘了伏笔的情况？

### 时间线检查

1. 事件发生顺序是否合理？
2. 有没有"昨天才发生的事，今天大家都忘了"的情况？
3. 跨天/跨周的时间过渡是否自然？

### 信息一致性

1. 角色A知道的事，后续有没有被"忘记"？
2. 某个设定前几章确立，后面有没有被推翻？
3. 读者的信息量和主角的信息量是否同步？

## 问题分级

| 级别 | 定义 | 示例 |
|------|------|------|
| **P0** | 硬伤，必须修 | 人名写错、时间线矛盾、设定冲突 |
| **P1** | 影响体验，建议修 | 角色行为不合理的、情感断裂、节奏拖沓 |
| **P2** | 锦上添花 | AI味、对话标签单调、段落略长 |

## 输出格式

### 精读报告

```markdown
# 精读报告：小说名

## 总评
- 质量评分：X/10
- 最大优点：...
- 最需要改进：...

## 分章问题清单
（按严重程度排序）

### P0（必须修）
| 章节 | 类型 | 问题 | 建议修改 |
|------|------|------|----------|
| 第N章 | 逻辑 | ... | ... |

### P1（建议修）
...

### P2（可以改）
...

## 跨章问题
### 角色弧线
- ...

### 情感曲线
- ...

### 伏笔
- ...

## 修改依赖
- 修改第N章可能影响第M章（原因）
```

### 修改清单

```markdown
# 修改清单_精读后

## 第N章
- [ ] P0: 修复人名错误（"张三"→"李四"）
- [ ] P1: 角色情绪调整（悲伤不应该这么快消失）

## 第M章
- [ ] P1: 补充情感过渡（修改第N章后需要检查这里是否受影响）
```

## 注意事项

1. **不要一次性读太多章** — 每批3-5章，保证深度理解
2. **不要带着先入为主的判断** — 先读完再下结论
3. **好的地方也要记录** — 审读不是挑刺
4. **问题要有具体建议** — 只说"这里不好"没用，要说"怎么改"
5. **注意修改连锁反应** — 改了一处可能影响其他章节
