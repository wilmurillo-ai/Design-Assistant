# 弹幕角色管理指南

适用于有弹幕/群聊/多角色实时互动机制的小说（如《观众都是我的前世》）。

## 核心原则

1. **每个弹幕角色有独立人格** — 不能只是"第N号"的标签
2. **弹幕推动剧情** — 不是装饰，每个弹幕角色都有功能
3. **发言频率合理** — 不是每章每个角色都发言
4. **互动有化学反应** — 角色之间会互怼、共鸣、争论

## 弹幕角色档案模板

每个常驻弹幕角色需要建立档案：

```markdown
### 001号 赵承乾

**身份：** 唐朝将领，战死沙场
**性格：** 刚烈、忠义、嘴硬心软
**说话风格：** 文言底子+偶尔现代口语混搭，喜欢用军事比喻
**口头禅：** "哼""某当年……""这点小事"
**与主角关系：** 最先认可主角，像长辈
**功能定位：** 力量担当/情感锚点
**出场频率：** 每2-3章至少一次发言
**互动对象：** 经常和777号互怼，与247号有惺惺相惜

### 777号 [名字]

**身份：** 现代程序员，过劳死
**性格：** 毒舌、理性、闷骚
**说话风格：** 网络梗+冷笑话+技术术语
**口头禅：** "笑死""这波啊""寄了"
**功能定位：** 喜剧担当/解构严肃
**出场频率：** 每2章至少一次
**互动对象：** 经常怼001号，和247号组"理性派"
```

## 弹幕管理规则

### 发言顺序
- ❌ 不能每章都是 001→777→247→... 固定顺序
- ✅ 根据场景选择合适的角色发言（战斗场景001，搞笑场景777）
- ✅ 每章发言角色数控制在3-6个

### 弹幕密度
| 场景类型 | 弹幕频率 | 说明 |
|----------|----------|------|
| 日常/轻松 | 每200-300字 | 弹幕多，氛围轻松 |
| 紧张/战斗 | 每500字 | 弹幕少，不抢戏 |
| 催泪/情感 | 每400字 | 弹幕是情感放大器 |
| 揭秘/反转 | 反转前少，反转后爆发 | 制造对比 |

### 弹幕格式
- 编号+名字首次出现，后续可用编号
- 隔5章以上没出场的角色再次出现要带名字
- 弹幕用【】包裹，区分正文

### 去AI味
- ❌ 弹幕角色都一个语气（都是"有趣""有意思"）
- ❌ 每个角色发言都是"分析性"的（像AI在分析剧情）
- ✅ 弹幕要像真人在看直播时的反应（吐槽、惊叹、嘲讽、感动）
- ✅ 不同时代/背景的角色要用不同表达方式

## 弹幕与剧情的关系

### 自称一致性（铁律）
每个弹幕角色必须有固定自称，整部小说不变：
- 001号赵承乾 → "朕"（皇帝身份）
- 389号金万三 → "老夫"（商人身份）
- 666号李归墟 → 极少说话，以"算了"为主
- 520号冷月 → 惜字如金（"嗯""……""危险"）
- 888号孟长卿 → "老夫"（文臣身份）

**写作时必须检查：**
1. 每个角色第一次出场确定自称
2. 后续所有弹幕保持一致
3. 用 `consistency_check.py` 自动检测自称不一致
4. "我"是通用自称，但有个性的角色应该有特色自称

## 弹幕与剧情的关系

1. **弹幕可以给提示** — 但不能直接剧透
2. **弹幕可以误导** — 弹幕角色的猜测不等于真相
3. **弹幕反映读者情绪** — 读者可能在想什么，弹幕角色替他们说出来
4. **弹幕推动主角思考** — 主角从弹幕中获得启发
5. **毕业是弹幕角色的高光** — 毕业前要铺垫足够的情感

## 弹幕角色毕业设计

- 毕业不能太突然，至少提前3章铺垫
- 毕业的触发是角色自己主动申请
- 毕业场景是催泪高潮，要写得慢、写得细
- 毕业后座位变空/星光，留下视觉记忆
- 毕业对其他弹幕角色有影响（悲伤、感慨、珍惜当下）
