# 续写衔接机制

长篇小说续写的最大挑战是上下文断裂。本指南确保每章与前文的衔接。

## 写前必读：近5章摘要法

不能只读上一章最后500字，需要构建"写作上下文包"。

### 上下文包的构成

```
1. 当前写作卡（本章要写什么）
2. 角色状态快照（所有出场角色当前状态）
3. 近5章关键事件摘要（每章1-2句话）
4. 上一章最后500字原文
5. 本章涉及角色的设定
6. 相关设定词典条目
```

### 近5章摘要模板

每章写完后，在追踪表中维护：

```markdown
### 近5章摘要（写作时自动生成）

| 章节 | 关键事件 | 新增信息 | 角色变化 |
|------|----------|----------|----------|
| 第31章 | 陆远发现财务系统后门 | 永盛有隐藏账户 | 陆远开始怀疑公司高层 |
| 第32章 | 陆远找孙磊确认数据 | 孙磊否认知道后门 | 孙磊态度异常 |
| 第33章 | 赵恒远程分析数据 | 数据指向海外空壳公司 | 无 |
| 第34章 | 陆远被王建国叫去谈话 | 王建国暗示"别多管闲事" | 陆远处境危险 |
| 第35章 | 陆远决定深入调查 | 发现第二层加密数据 | 陆远从被动变主动 |
```

## 衔接检查清单

写完新章节后，检查以下衔接点：

### 1. 时间衔接
- [ ] 本章开头的时间是否紧接上一章结尾
- [ ] 如果有时间跳跃，是否在文中明确交代
- [ ] 角色的行动时间是否合理（走路、吃饭、开会需要时间）

### 2. 空间衔接
- [ ] 上一章结尾角色在哪，本章开头是否一致
- [ ] 如果换了场景，是否有过渡描写
- [ ] 场景中的道具/环境是否连贯

### 3. 情绪衔接
- [ ] 上一章结尾角色的情绪，本章开头是否延续
- [ ] 情绪变化是否有事件触发，不能突兀转变
- [ ] 对话的语气是否与情绪匹配

### 4. 信息衔接
- [ ] 角者在上一章获得的信息，本章是否正确运用
- [ ] 没有在本章"遗忘"上一章的关键发现
- [ ] 角色的决策是否基于已有信息

### 5. 悬念衔接
- [ ] 上一章末尾的悬念是否在本章回应（即使不揭晓，也要推进）
- [ ] 不能完全忽略上一章的钩子
- [ ] 回应方式可以是"部分回答+新悬念"

## 常见衔接错误

| 错误类型 | 示例 | 修正方法 |
|----------|------|----------|
| 位置跳跃 | 上一章在办公室，本章突然在家，没交代怎么回去的 | 加过渡段 |
| 情绪断裂 | 上一章结尾恐惧，本章开头轻松聊天 | 加情绪过渡 |
| 信息遗忘 | 上一章发现重要线索，本章完全没提 | 自然带出"他一直在想那条线索" |
| 角色凭空出现 | 角色突然在场但没交代什么时候来的 | 补一句"他不知道什么时候站到了身后" |
| 时间重叠 | 两个事件的时间对不上 | 用追踪表的时间线核实 |

## 自动摘要生成

每章写完后，在追踪表中自动维护摘要。agent 应执行：

1. 总结本章3个关键事件
2. 记录新增的重要信息
3. 标记角色状态变化
4. 将摘要追加到"近5章摘要"表
5. 如果超过5章，删除最旧的一条
