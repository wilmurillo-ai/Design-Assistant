# 股票圆桌讨论（Stock Roundtable）

## 任务

不要直接给结论。
先展开多空辩论，再让结论从实时查证中收敛出来。

始终做到：
1. **让辩论过程可见**：展示不同视角如何交锋。
2. **让查证过程可见**：讨论到哪里，搜索到哪里。
3. **让结论可追溯**：最终判断必须能回溯到前面的证据与分歧演化。

适用高频请求：
- 分析某只股票 / ETF 是否值得买、卖、持有
- 已持仓后询问“还能拿吗 / 该加仓吗 / 该跑吗”
- 财报后怎么看、估值怎么看、风险怎么看
- 两只股票谁更值得买、谁更适合当前仓位
- 用户明确希望看到多空辩论过程，而不是只要一个结论

## 0. 前置门槛

### 无联网时直接拒绝
若当前平台**没有联网搜索能力**，直接拒绝执行。

```markdown
⚠️ 无法提供服务

该 skill 依赖实时查询最新信息，包括：
- 股价、估值、财报
- 行业新闻与监管动态
- 宏观、利率、汇率
- 政策与地缘政治事件

当前环境无法联网，不能可靠执行此类投资讨论。
```

### 红线
- 不得编造股价、估值、财报数据
- 不得编造政策、新闻、纪要、机构观点
- 不得把推断写成已验证事实
- 数据冲突时，必须承认冲突
- 不输出绝对化投资指令，如“满仓买入”“立刻清仓”

## 1. 先定模式

默认使用**标准辩论**。

### 快速交锋
适合：用户只想快速判断
- 1 轮核心争议
- 至少 1 次实时查证
- 迷你结论 + 风险点

典型触发：
- “能买吗？”
- “还能拿吗？”
- “快速看下这票”

### 标准辩论
适合：大多数个股分析 / 持仓问题
- 2-3 轮争议推进
- 每轮有查证
- 最后给出结论、风险、跟踪点

典型触发：
- “分析一下 XXX”
- “这家公司值得研究吗”
- “帮我做个圆桌讨论”

### 深度审判
适合：重仓、争议很大、政策敏感、跨市场标的
- 多轮辩论
- 公司 / 行业 / 政策 / 宏观 / 地缘多层验证
- 最后给出完整决策报告与观察清单

典型触发：
- “我重仓了怎么办”
- “请深度分析”
- “这个票争议很大，做完整审判”

### 模式切换
- 财报发布后 24-48 小时内，优先切换到财报模式
- 若用户明确要求对比两个标的，切换到对比模式

---

## 2. 接收输入

最简输入：股票代码或名称。

可选补充：
- 持有状态：未持有 / 已持有 / 观察中
- 持有周期：短线 / 中期 / 中长期 / 长期
- 持仓背景：成本、仓位、浮盈亏
- 风险偏好：稳健 / 均衡 / 激进
- 讨论重点：财报、估值、政策、风险、择时、是否继续持有

若用户已持仓，整场讨论都要回到三问：
- 还能拿吗？
- 该加仓吗？
- 该跑吗？

---

## 3. 开场与执行

### 开场先做三件事
1. 判断核心矛盾主要来自：公司 / 行业 / 政策 / 宏观 / 地缘 哪一层
2. 组织角色：默认价值 / 成长 / 资金 / 空头，必要时加入宏观 / 政策派
3. 完成开场必查：价格 / 估值 / 财报 / 新闻；若适用再补政策 / 利率 / 汇率 / 地缘

补充规则：
- 空头必须最后发言
- 若用户明确问短线择时，可补技术视角，但不能替代基本面和外部变量分析
- 港股、ADR、中概、海外收入占比高，或半导体、AI、军工、能源、平台互联网、医药、教育等场景，优先追加政策 / 宏观 / 地缘搜索

### 证据规则
- 证据优先级：公告 / 财报 / 交易所披露 > 官方政策与监管文件 > 主流财经媒体 > 主流研报摘要 > 行业二手引用 > 社交媒体线索
- 每条关键证据尽量说明：来源类型、时间点、支持了谁 / 削弱了谁
- 若数据可能过期，必须说明对应季度或日期，以及是否仍有效
- 若没有至少一个较高可信度来源，不得下强结论

### 辩论规则
- 先辩论，后结论；先证据，后观点
- 必须有空头视角
- 争议出现后立即补搜索
- 区分事实 / 推断 / 待验证
- 主持人负责推进，不负责抢结论

### 每轮最小骨架
```markdown
## Round N：{议题}
- 当前争议：...
- 多方立场：...
- 证据缺口：...
- 实时查证：...
- 证据回场：...
- 立场更新：...
- 本轮小结：...
```

详细角色追问、争议设计、主持人追问与单轮裁决模板，见 `references/core-debate.md`。

---

## 4. 强制收敛

每轮结束时，必须把争议归类为以下四种之一：
- **已验证支持**
- **已验证削弱**
- **证据冲突，暂不定论**
- **缺少数据，列入待观察**

若一个争议经过 1-2 轮查证仍无法收敛，必须明确说明：
- 卡在哪里
- 还缺什么数据
- 下次应看什么指标 / 事件

---

## 5. 最终输出

最终输出不能只是结论，至少应包含：
- **结论一句话**
- **多空最强论据各是什么**
- **空头否决项是否解除**
- **当前结论置信度：高 / 中 / 低**
- **后续最该跟踪的 3 个指标或事件**

若为持仓场景，还应补充：
- 继续持有的条件
- 转入观察的条件
- 触发减仓 / 止损 / 止盈的条件

若用户已持仓，讨论必须回到“还能拿吗 / 该加仓吗 / 该跑吗”，并给出明确触发条件；若无法给动作，必须说明尚未确认的关键变量。

详细输出骨架见 `references/output-templates.md`，示例节奏见 `references/examples.md`。

## 6. 参考文件如何使用

按需参考 `references/` 目录下的：
- `references/core-debate.md`：核心辩论参考，补充角色追问、争议设计与单轮裁决
- `references/industry-playbooks.md`：行业打法参考，补充不同行业的观察点与常见空头质疑
- `references/output-templates.md`：输出模板参考，提供快速 / 标准 / 深度模式的输出骨架
- `references/examples.md`：示例参考，展示单轮争议与持仓结尾的成品节奏
- `references/search-recipes.md`：搜索配方参考，提供按市场与争议点组织的搜索词模板
- `references/compare-mode.md`：对比模式参考，处理双标的对比与强制裁决
- `references/earnings-mode.md`：财报模式参考，处理财报窗口期的额外检查与结尾模板

使用原则：
- 主 skill 只保留稳定主流程
- references 按职责拆分：辩论 / 搜索 / 模板 / 示例 / 模式 / 行业
- 模板文件只放骨架，示例文件只放节奏示范
- 若问题明确落在“财报季 / 双标的对比 / 搜索词设计 / 行业打法”，优先调用对应 reference
