# 任务状态变化指南

## 🎯 核心概念

子弹笔记的精髓在于任务的**动态状态变化**。任务不是静止的待办清单，而是会随着时间和进展不断演化的生命体。

---

## 📊 任务状态系统

### 1. 状态定义

| 状态 | 符号 | 名称 | 说明 |
|------|------|------|------|
| 待办 | ● | 待办任务 | 需要执行的动作，初始状态 |
| 已完成 | × | 已完成 | 任务已成功完成 |
| 已迁移 | > | 已迁移 | 任务未完成但重要，延期到新日期 |
| 已取消 | ~~ | 已取消 | 任务不再需要，已取消（划掉） |
| 已计划 | < | 已计划 | 任务升级为项目，需要详细规划 |

### 2. 状态变化流程

```
                    ┌─────────────────┐
                    │     ● 待办      │
                    └────────┬────────┘
                             │
           ┌─────────────────┼─────────────────┐
           │                 │                 │
           ▼                 ▼                 ▼
    ┌──────────┐      ┌──────────┐      ┌──────────┐
    │ × 已完成  │      │ > 已迁移  │      │ ~~ 已取消│
    └──────────┘      └────┬─────┘      └──────────┘
                            │
                            │ 在新日期重新写入
                            ▼
                     ┌──────────┐
                     │  ● 待办   │
                     └──────────┘

                    ● 待办 → < 已计划
                            │
                            │ 移至项目集子
                            ▼
                     ┌──────────┐
                     │  ● 待办   │（分解后的子任务）
                     └──────────┘
```

### 3. 状态转换规则

| 当前状态 | 可转换到 | 触发条件 |
|----------|----------|----------|
| ● 待办 | × 已完成 | 任务成功完成 |
| ● 待办 | > 已迁移 | 任务未完成但重要，需要延期 |
| ● 待办 | ~~ 已取消 | 任务不再需要或无关紧要 |
| ● 待办 | < 已计划 | 任务升级为项目，需要详细规划 |
| × 已完成 | ● 待办 | 撤销完成（重新激活） |
| > 已迁移 | ● 待办 | 在新日期重新激活 |
| < 已计划 | ● 待办 | 项目分解为具体任务 |

## 📝 实际使用场景

### 场景1：日常任务完成

**早晨记录**：
```
● 09:00 整理有机逆合成逻辑
● 09:00 购买卫生纸
● 09:17 处理立项书摘要
```

**上午工作**：
- 完成"整理有机逆合成逻辑" → 点击 `●` → 变为 `×`
- 完成"购买卫生纸" → 点击 `●` → 变为 `×`

**中午记录**：
```
× 09:00 整理有机逆合成逻辑
× 09:00 购买卫生纸
● 09:17 处理立项书摘要
```

### 场景2：任务迁移

**早晨记录**：
```
● 09:00 预约牙医
● 14:00 完成周报
```

**上午工作**：
- "预约牙医"今天太忙，没时间 → 标记为 `>`（已迁移）
- "完成周报"已完成 → 标记为 `×`

**迁移操作**：
1. 点击 `●` → 变为 `>`
2. 系统提示输入新日期（如：2025-03-17）
3. 任务自动迁移到新日期

**第二天早晨**：
```
● 09:00 预约牙医（从2025-03-16迁移而来）
```

### 场景3：任务取消

**早晨记录**：
```
● 10:00 参加产品评审会
● 14:00 给客户打电话
```

**上午工作**：
- 产品评审会取消 → 点击 `✗` → 变为 `~~`
- 原因：会议取消

**中午记录**：
```
~~ 10:00 参加产品评审会（已取消：会议取消）~~
● 14:00 给客户打电话
```

### 场景4：任务升级为项目

**早晨记录**：
```
● 09:00 制定年度计划
```

**上午思考**：
- "制定年度计划"太复杂，需要分解 → 标记为 `<`（已计划）
- 创建项目集子（Collection）

**下午记录**：
```
< 09:00 制定年度计划（已升级为项目，移至集子：年度计划）
```

**项目集子中**：
```
# 年度计划

## 1月
- ● 完成目标设定
- ● 制定行动计划

## 2月
- ● 执行计划
- ● 月度复盘
```

## 🔧 任务管理API

### 完成任务

```python
from task_manager import TaskManager

manager = TaskManager()

# 完成任务
task = manager.complete_task(
    date='2025-03-16',
    task_id='task_001',
    notes='已完成'
)

print(f"任务状态：{task['status']}")  # completed
```

### 迁移任务

```python
# 迁移任务到新日期
task = manager.migrate_task(
    date='2025-03-16',
    task_id='task_002',
    new_date='2025-03-17'
)

print(f"任务状态：{task['status']}")  # migrated
```

### 取消任务

```python
# 取消任务
task = manager.cancel_task(
    date='2025-03-16',
    task_id='task_003',
    reason='会议取消'
)

print(f"任务状态：{task['status']}")  # cancelled
```

### 计划任务

```python
# 将任务升级为项目
task = manager.plan_task(
    date='2025-03-16',
    task_id='task_004',
    collection_name='年度计划'
)

print(f"任务状态：{task['status']}")  # planned
```

### 获取统计信息

```python
# 获取任务统计
stats = manager.get_task_statistics('2025-03-16')

print(f"总任务数：{stats['total']}")
print(f"已完成：{stats['completed']}")
print(f"待办：{stats['pending']}")
print(f"已迁移：{stats['migrated']}")
print(f"已取消：{stats['cancelled']}")
print(f"完成率：{stats['completion_rate']:.1f}%")
```

## 🎨 事件 vs 任务的区别

### 任务（●）

**特征**：
- 需要**主动执行**的动作
- 有明确的开始和结束
- 可以被"完成"
- 示例：整理有机逆合成逻辑、购买卫生纸

**示例**：
```
● 整理有机逆合成逻辑
● 购买卫生纸
● 完成周报
```

### 事件（○）

**特征**：
- 已经或即将发生的事实
- 与特定**日期/时间/地点**相关
- 是**被动**的，不能"完成"，只能"经历"或"记录"
- 示例：团队会议、医生约诊、生日

**示例**：
```
○ 09:15 团队会议（地点：会议室A）
○ 下午3点医生约诊
○ 今天下雨了
```

### 关键区别

| 特征 | 任务（●） | 事件（○） |
|------|-----------|-----------|
| 性质 | 主动执行的动作 | 已发生或将要发生的事实 |
| 完成性 | 可以被"完成" | 不能"完成"，只能"经历"或"记录" |
| 日期 | 可选，通常无固定日期 | 必须有日期 |
| 时间 | 可选 | 可选（会议等有具体时间） |
| 地点 | 通常无 | 可有（如会议室、医院） |
| 状态变化 | ● → ×、>、~~、< | ○ + 笔记、○ → >（重新安排） |

## 💡 最佳实践

### 1. 每日迁移（Daily Migration）

每天晚上或早晨，花5-10分钟回顾前一天的日志：

```python
# 获取昨天的任务
tasks = manager.get_daily_tasks('2025-03-16')

# 对每个未完成的任务进行决策
for task in tasks:
    if task['status'] == 'task':
        print(f"任务：{task['content']}")
        print("请选择操作：")
        print("1. 完成任务")
        print("2. 迁移到今天")
        print("3. 取消任务")

        choice = input("请输入选项（1/2/3）：")

        if choice == '1':
            manager.complete_task('2025-03-16', task['id'])
        elif choice == '2':
            manager.migrate_task('2025-03-16', task['id'], '2025-03-17')
        elif choice == '3':
            manager.cancel_task('2025-03-16', task['id'])
```

### 2. 月度迁移（Monthly Migration）

每月初，将"未来记录"中本月的任务安排到具体的每日日志中。

### 3. 保持简洁

不要创建太多状态，先用好基础符号：
- ● 待办
- × 已完成
- > 已迁移
- ~~ 已取消

### 4. 定期回顾

每周回顾一次，看看哪些任务总是被迁移，可能需要：
- 重新评估优先级
- 分解为更小的任务
- 或者果断取消

## 🎯 总结

### 任务状态变化的核心原则

1. **主动性选择**：不是被动堆积，而是主动决策
2. **视觉反馈**：符号变化提供强烈的完成感
3. **持续进化**：任务是动态的，会随时间变化
4. **断舍离**：取消不再有价值的任务

### 事件的核心特征

1. **日程性**：与日历紧密相关
2. **记录性**：记录事实，如日记
3. **被动性**：不是主动执行的动作
4. **可迁移**：可以重新安排日期
