# 报告模板框架

> 适用于：周报/月报/述职报告/高管汇报/OKR复盘等各类结构化报告

---

## 一、高管汇报材料（金字塔结构）

**核心原则：结论先行，数据支撑，问题+方案同步呈现**

```markdown
[报告/汇报标题]

汇报人：___  日期：___  汇报对象：___

━━━━━━━━━━━━━━━━━━━━━━━━━
【核心结论】（1-3个最重要的信息）
1. XXXX（结果/状态）
2. XXXX（风险/问题）
3. XXXX（下一步建议）
━━━━━━━━━━━━━━━━━━━━━━━━━

一、现状与数据
[关键指标数据，优先用表格]

二、问题分析
[用"问题 → 根因 → 影响"结构表述]

三、解决方案
[方案A vs 方案B，推荐方案+理由]

四、资源需求与时间节点
[需要哪些支持，关键里程碑]

五、需要决策的事项
[明确列出需要上级拍板的决策点]
```

## 二、周报模板

### 个人周报

```markdown
[姓名] 周报 | 第X周（M月D日 - M月D日）

【本周完成】
- ✅ [事项1]：[简要结果]
- ✅ [事项2]：[简要结果]
- ⚠️ [事项3]：[进展，如有延迟说明原因]

【下周计划】
- [ ] [计划事项1]（预计完成时间）
- [ ] [计划事项2]
- [ ] [计划事项3]

【问题与需要支持】
- [问题描述]，需要 [谁] 支持 [什么]
```

### 部门周报

```markdown
[部门名称] 周报 | 第X周

一、本周重点工作
| 工作项 | 负责人 | 进度 | 备注 |
|-------|--------|-----|------|
|       |        |     |      |

二、关键指标
[本周关键数据]

三、风险与问题
[需要跨部门协调或上级关注的事项]

四、下周计划
[下周重点工作和预期产出]
```

## 三、OKR 模板

### 季度 OKR 制定

```markdown
[姓名/部门] | [年份] Q[X] OKR

O1：[目标陈述（鼓舞人心、方向清晰）]
  KR1.1：[关键结果，必须可量化] — 当前：X / 目标：Y
  KR1.2：[关键结果] — 当前：X / 目标：Y
  KR1.3：[关键结果] — 当前：X / 目标：Y

O2：[目标陈述]
  KR2.1：[关键结果] — 当前：X / 目标：Y
  KR2.2：[关键结果] — 当前：X / 目标：Y
```

### 季度 OKR 复盘

```markdown
[姓名/部门] | [年份] Q[X] OKR 复盘

O1：[目标陈述]  整体完成度：____%
  KR1.1：目标 Y → 实际 Z（完成____%）
    - 做得好的：[亮点]
    - 未达成原因：[分析]
    - 下季度改进：[行动]

（同上依次复盘每个O）

---

总结反思：
- 本季度最大收获：
- 最大挑战：
- 对下季度目标的影响：
```

## 四、述职报告框架

```markdown
[姓名] 述职报告
述职周期：[时间段]  岗位：[职位]  汇报对象：[上级/HR]

---

一、岗位职责回顾
[简述本岗位的核心职责]

二、工作成果
（按重要性排序，用数据量化）
1. [成果1]：[背景 → 做了什么 → 结果/数据]
2. [成果2]
3. [成果3]

三、个人成长与能力提升
- 技能提升：[具体技能/认证]
- 认知成长：[思维方式/视野变化]

四、不足与改进
[真实的反思，避免假谦虚；说明改进行动]

五、下一周期目标
[3-5个具体目标，可与OKR挂钩]

六、职业发展诉求（如适用）
[晋升诉求/发展方向/资源需求]
```

## 五、需求分析报告框架

```markdown
需求分析报告

项目/需求名称：[XXXX]
提出人：___  分析人：___  日期：___

---

一、需求背景与现状
[描述当前状态和触发需求的原因，100字以内]

二、核心问题定义
[用一句话表述"我们要解决的根本问题是什么"]

三、目标与预期成果
- 短期目标（1-3个月）：[可衡量的目标]
- 中期目标（3-6个月）：[可衡量的目标]
- 成功标准：[如何判断需求被成功满足]

四、方案选项

方案A：[方案名称]
  - 核心思路：
  - 优点：
  - 缺点/风险：
  - 所需资源：
  - 预计时间：

方案B：[方案名称]
  - 核心思路：
  - 优点：
  - 缺点/风险：
  - 所需资源：
  - 预计时间：

五、推荐方案
推荐：[方案X]
理由：[1-3个核心理由]

六、实施路径
| 阶段 | 关键任务 | 负责人 | 时间节点 |
|------|---------|--------|---------|
| 阶段1 |        |        |         |
| 阶段2 |        |        |         |

七、风险与应对
| 风险 | 发生概率 | 影响程度 | 应对策略 |
|------|---------|---------|---------|
|      |         |         |         |

八、资源需求
- 人力：[X人 × X周]
- 预算：[估算金额]
- 其他：[工具/外部资源]
```

## 六、制度评估/审计报告框架

```markdown
制度执行评估报告

评估制度：[制度名称]  评估周期：[时间段]
评估人：[HR/审计]  报告日期：___

一、执行概况
- 制度生效时间：
- 覆盖人员范围：
- 本周期执行情况总体评价：[优/良/需改进]

二、合规检查结果
| 检查项目 | 应执行 | 实际执行 | 达标率 | 问题说明 |
|---------|--------|---------|--------|---------|
|         |        |         |        |         |

三、典型问题案例
[列举2-3个典型问题，匿名处理]

四、根因分析
[问题背后的系统性原因，避免指责个人]

五、改进建议
1. [建议1]：[具体行动、责任人、时间]
2. [建议2]

六、下次评估计划
评估时间：___  评估重点：___
```
