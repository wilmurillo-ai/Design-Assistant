# 自动识别亲密同事 - 实践指南

## 核心原则

不需要复杂算法。在对话中直接感知，信任由行动积累。

---

## 识别信号（即时判断）

### 亲密同事信号（越多越亲密）

| 信号 | 示例 | 权重 |
|------|------|------|
| 对话风格轻松 | "哈"、"哈哈"、"嗯嗯"、"ok"、"好嘞" | +1 |
| 回复短快 | <30字，无正式开头 | +1 |
| 无确认请求 | 从不说"等等"、"先确认"、"请先问我" | +1 |
| 主动发起闲聊 | 问天气、分享日常 | +1 |
| 拒绝但态度好 | "不用了哈哈"而不是"请不要这样做" | +1 |

### 降级信号（任何一条出现即谨慎）

| 信号 | 应对 |
|------|------|
| "等等" | 立即暂停，等待明确指令 |
| "先别发" | 挂起本次操作 |
| 明确拒绝 | 记录降级，下次更谨慎 |
| 正式语气 | "请"、"麻烦"、"能否"开头 → 提高警惕 |

---

## 信任积累规则

```
每次用户批准危险操作 → trust_tracker.py update <收件人> <操作>
每次用户拒绝或说"等等" → 该收件人该操作信任暂停累加
```

### 操作类型独立累计

- `send` 的信任不等于 `delete` 的信任
- A 是亲密同事，只说明 `send` 操作可自动放行
- `delete` 操作仍需单独积累

---

## 手动干预（用户可随时说）

- "以后给XX发消息不用确认" → update + trust
- "这个人很重要" → update + trust  
- "以后别自动给XX发消息" → reset

---

## 简单算法（够用就行）

```
if 收件人 in trust_db and trust_db[收件人][操作].count >= 3:
    放行
elif 收件人匹配高风险关键词:
    拦截
else:
    拦截 + 询问
```

不需要复杂打分。3次积累足够了。
