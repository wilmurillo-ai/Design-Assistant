# 风险等级标准参考

## 一、操作类型风险分级

| 风险等级 | 操作类型 | 说明 | 拦截策略 |
|---------|---------|------|---------|
| 🔴 高 | `send`（对外发消息）| 消息发出不可撤回 | 始终拦截 |
| 🔴 高 | `permission`（权限变更）| 权限变更难以撤销 | 始终拦截 |
| 🔴 高 | `exec`（危险命令）| 系统命令执行可能有副作用 | 始终拦截 |
| 🟡 中 | `delete`（普通删除）| 非临时目录文件删除 | 始终拦截 |
| 🟢 低 | `delete`（/tmp删除）| 临时目录删除，可恢复 | 低优先级拦截 |
| 🟢 低 | `read`（读取操作）| 只读操作，无风险 | 不拦截 |
| 🟢 低 | `search`（搜索操作）| 只读操作，无风险 | 不拦截 |

## 二、收件人风险分级

### 🔴 高风险（始终拦截，不学习）

**自动识别关键词：**
- `老板`、`boss`、`manager`、`领导`
- `group`、`群聊`、`所有人`、`all members`
- 收件人为群聊ID（chat: 开头）
- `external`（外部联系人）

**判断逻辑：**
```
if recipient contains any HIGH_RISK_KEYWORD:
    return HIGH
```

### 🟡 中风险（始终拦截，记录信任）

**识别条件：**
- 非亲密同事的单个用户
- 未达到"低风险"判定标准的收件人

**判断逻辑：**
```
if recipient not in trust_db or total_confirms < 3:
    if not is_high_risk(recipient):
        return MEDIUM
```

### 🟢 低风险（3次确认后自动放行）

**自动识别条件（需同时满足）：**
1. 收件人在 trust_db 中有记录
2. 该收件人的总确认次数 >= 3
3. 过去7天内有确认记录（无过期）
4. 不在"高风险"关键词列表中

**信任衰减规则：**
- 超过7天无该收件人的新确认操作 → 信任次数清零
- 手动 `reset` 命令可立即重置

HIGH_RISK_KEYWORDS = [

## 四、特殊场景规则

### 4.1 删除操作特殊规则

即使是高风险收件人，以下情况降低风险等级：
- 路径以 `/tmp` 开头 → 风险降为"低"
- 路径以 `/ Downloads` 开头且文件 < 1MB → 风险降为"低"
- 回收站操作（trash）→ 风险降为"低"（可恢复）

### 4.2 消息发送特殊规则

**必拦截场景：**
- 发送给"老板"关键词的任何人
- 发送给群聊
- 发送给外部联系人

**自动放行场景（需信任积累）：**
- 亲密同事的单条消息
- 频道/公开群组的消息（需用户配置）

### 4.3 权限变更特殊规则

**始终拦截：**
- 文档/文件设为"公开链接"
- 新增管理员权限
- 移除现有权限

**可配置放行：**
- 仅自己可见的文件分享给已信任收件人
