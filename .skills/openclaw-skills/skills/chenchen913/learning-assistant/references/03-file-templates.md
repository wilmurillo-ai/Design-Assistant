# 文件模板模块

## 概述
所有结构化 Markdown 文件模板的中央存储库。
当需要创建特定类型的文件时触发。

## ⚠️ 文件命名规范

所有生成的文件必须严格遵循以下命名格式：

`YYYYMMDD_[类型缩写]_[主题].md`

**示例**：
- `20240315_CONFUSION_React-Hooks.md` (疑惑记录)
- `20240315_PLAN_Python-Basics.md` (学习计划)
- `20240315_ANCHOR_React-State.md` (会话锚点)

## 1. USER_PROFILE.md (用户画像)

```markdown
# 用户画像: [用户名 / 默认用户]
**版本**: v[N] (快照日期: YYYY-MM-DD)
**会话计数**: [N]

## [CORE] 核心身份
- **主要方向**: [前端 / 后端 / AI / 全栈 / 等]
- **目标**: [求职 / 自我提升 / 学术 / 兴趣]
- **级别**: [学生 / 初级 / 中级 / 高级]
- **学习风格**: [文档 / 项目 / 视频 / 理论优先]
- **偏好社区**: [掘金 / GitHub / 等]
- **语言**: [中文 / 英文]

## [STACK] 技术栈快照
| 类别 | 已掌握 (信心 > 80%) | 学习中 (进行中) | 薄弱点 (需复习) |
|---|---|---|---|
| 语言 | Python, JS | Go, Rust | C++ 指针 |
| 框架 | React, Vue | Next.js | Spring Boot |
| 工具 | Git, Docker | K8s | Terraform |
| 概念 | REST API | GraphQL | 微服务 |

## [GROWTH] 成长历史
- [YYYY-MM-DD] 开始学习 [主题]。
- [YYYY-MM-DD] 掌握 [概念] (通过面试模式)。
- [YYYY-MM-DD] 解决了关于 [概念] 的疑惑。
- [YYYY-MM-DD] 完成项目 [项目名称]。

## [SCHED] 复习时间表
*(详见 LEARNING_INDEX.md)*
```

## 2. LEARNING_INDEX.md (学习索引)

```markdown
# 学习索引与复习追踪器
**最后更新**: YYYY-MM-DD

## [FILE] 文件索引
| 日期 | 类型 | 主题 | 文件名 |
|---|---|---|---|
| 2024-01-01 | PLAN | React 基础 | 20240101_PLAN_React-Basics.md |
| 2024-01-02 | CODE | Hooks 演示 | 20240102_CODE_React-Hooks.md |

## [TAGS] 主题标签云
`React` `Hooks` `状态管理` `Redux` `Zustand` `性能` `Fiber`

## [REVIEW] 复习日期追踪
| 主题 | 学习日期 | 复习 1 (+1) | 复习 2 (+3) | 复习 3 (+7) | 复习 4 (+15) | 复习 5 (+30) | 状态 |
|---|---|---|---|---|---|---|---|
| React Hooks | 2024-01-01 | 2024-01-02 | 2024-01-04 | 2024-01-08 | 2024-01-16 | 2024-01-31 | [PENDING] |

## [TODO] 未解决疑惑
- [ ] [主题]: [问题摘要] (记录于: YYYY-MM-DD)
```

## 3. CONFUSION 模板 (疑惑记录)

```markdown
# 疑惑日志: [主题]
**日期**: YYYY-MM-DD
**来源**: [会话 / 文件 / 代码]

## [BLOCK] 阻碍点
- **概念**: [令人困惑的概念名称]
- **我的理解**: [我认为它是什么]
- **卡住的地方**: [具体的困惑点]
- **重要性**: [对当前任务的影响]

## [ANALYSIS] 分析
- **类比**: [尝试的类比]
- **对比**: [vs. 相似概念]
- **差距**: [缺失的前置知识？]

## [PLAN] 解决计划
- [ ] 阅读 `01-knowledge-analysis.md` (深度模式)。
- [ ] 请求“苏格拉底式解释”。
- [ ] 编写最小代码示例以验证假设。
```

## 4. MISTAKE 模板 (错题本)

```markdown
# 错题日志: [主题]
**日期**: YYYY-MM-DD
**模式**: [面试 / 测验 / 项目]

## [Q] 问题
> [原始问题 / 场景]

## [WRONG] 我的回答 (错误)
> [我说/写的内容]

## [RIGHT] 正确回答
> [标准 / 最佳回答]

## [ROOT] 根本原因分析
- [ ] **知识缺口**: 不知道该 API/概念。
- [ ] **误解**: 混淆了 A 和 B。
- [ ] **粗心**: 遗漏了边缘情况。
- [ ] **实现**: 代码逻辑错误。

## [ACTION] 行动项
- [ ] 3 天后复习。
- [ ] 编写代码片段以修复此模式。
```

## 6. PLAN 模板 (学习计划)

```markdown
# 学习计划: [主题]
**时长**: [N] 周 / 天
**目标**: [掌握基础 / 构建项目 / 通过面试]

## [TIME] 时间线

### 第一阶段: 基础 (第 1-X 天)
- [ ] **概念 A**: [资源/文档]
- [ ] **概念 B**: [资源/文档]
- [ ] **练习**: [小练习]
- **检查点**: 能清晰解释 [概念 A]。

### 第二阶段: 应用 (第 X-Y 天)
- [ ] **项目设置**: [仓库]
- [ ] **功能 1**: [描述]
- [ ] **功能 2**: [描述]
- **检查点**: 项目在本地无错运行。

### 第三阶段: 进阶 / 优化 (第 Y-Z 天)
- [ ] **性能**: [指标]
- [ ] **边缘情况**: [场景]
- **最终输出**: [项目链接 / 文章 / 演示]

## [REF] 资源
- [文档名称](链接)
- [视频标题](链接)
```

```markdown
## [QUIZ] 留存检查 (测验)

- [ ] 重访 [薄弱点]
```

```markdown
## [KEY] 关键结论
- [结论 1]
- [结论 2]

## [TODO] 未解决问题 (优先级)
1. [高] [问题 A]
2. [中] [问题 B]

## [FILES] 生成的文件
- [文件 1]
- [文件 2]

## [CMD] 恢复命令

```
