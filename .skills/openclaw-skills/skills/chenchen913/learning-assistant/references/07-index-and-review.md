# 索引与复盘系统

## 概述
管理全局 `workspace/LEARNING_INDEX.md` 并生成结构化复盘报告。
在每次会话结束时（步骤 D）或用户要求“复盘”时触发。

## 索引管理 (步骤 D)

在**每次**会话结束时：

1. **追加新文件**：将新生成的文件（SUMMARY, PLAN, CODE 等）添加到 `文件索引表`。
   - **注意**：文件名必须遵循 `YYYYMMDD_[类型]_[主题].md` 规范。
2. **更新标签云**：将新主题/关键词添加到 `主题标签云`。
3. **更新复习计划**：
   - 对于每个新学习的概念，基于艾宾浩斯遗忘曲线计算复习日期：
     - 复习 1: 第 1 天
     - 复习 2: 第 3 天
     - 复习 3: 第 7 天
     - 复习 4: 第 15 天
     - 复习 5: 第 30 天
   - 在 `workspace/LEARNING_INDEX.md` 的 `复习日期追踪表` 中添加条目。
   - ⚠️ **重要**：复习日期必须填写 **绝对日期**（格式：YYYY-MM-DD），**不可**填写相对天数（如"第 7 天"），否则 Step 3 的日期比较逻辑将无法执行。

## 复盘生成 (用户触发)

当用户说“复盘 [主题]”或“复盘最近的学习”时：

1. **定位历史**：在 `workspace/LEARNING_INDEX.md` 中查找相关文件。
2. **提取薄弱点**：
   - 未解决的 `CONFUSION` 条目。
   - 该主题的 `MISTAKE` 记录。
3. **生成 REVIEW 报告**：
   - **3.1 结构生成**：
     - **[OBJ] 目标**：学到了什么？
     - **[QUIZ] 概念留存**：基于过去错误的快速测验。
     - **[LINK] 知识图谱**：更新连接（前置/相关）。
     - **[NEXT] 下一步**：基于当前掌握程度的建议。
   - **3.2 重新出题验证（核心步骤）**：
     - 从 CONFUSION 文件中提取"未解决"条目，逐一重新出题。
     - 从 MISTAKE 文件中提取错题，换一种表达方式重新考察。
     - 用户回答后，判断是否已真正掌握，并更新对应条目状态。

## 疑惑与错误处理

- **CONFUSION**：“我不懂 X”。
  - 记录它。
  - 优先复习。
- **MISTAKE**：“我把 X 答错了”。
  - 记录它。
  - 在未来的复盘中重新测试。

如果在后续会话中解决了 `CONFUSION` 或 `MISTAKE`：
- 在 `workspace/LEARNING_INDEX.md` 中将其标记为 `[Resolved]`。
- 在 `workspace/USER_PROFILE.md` 中添加 `成长记录`。
