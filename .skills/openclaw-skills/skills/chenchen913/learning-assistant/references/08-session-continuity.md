# 会话连续性模块

## 概述
管理会话连续性并创建锚点文件以防止上下文丢失。
当会话过长、任务未完成或用户说“暂停”时触发。

## 锚点文件生成 (步骤 C)

当会话结束或接近上下文限制时：

1. **触发条件**：
   - `会话长度 > 80% 限制`
   - `用户明确要求暂停`
   - `任务未完成`

2. **生成文件**：`workspace/YYYYMMDD_ANCHOR_[主题].md`
   - **格式**：
     - **[STATUS] 任务状态**：完成 / 进行中 / 未开始。
     - **[SUM] 会话摘要**：100-200 字简洁回顾。
     - **[KEY] 关键结论**：主要收获的要点。
     - **[TODO] 未解决问题**：待处理问题/任务列表（含优先级）。
     - **[FILES] 生成的文件**：本会话创建的所有文件列表。
     - **[CMD] 恢复命令**：“从 [主题] [日期] 继续”或加载此文件。

## 恢复流程

当用户上传 `ANCHOR` 文件或说“继续上次会话”时：

1. **Step 1: 确认日期**：询问用户当前日期。
2. **Step 2: 读取锚点**：读取 `workspace/YYYYMMDD_ANCHOR_[主题].md`。
3. **Step 3: 回顾**：简要总结上次停下的地方。
4. **Step 4: 确认方向**：询问“直接继续还是调整计划？”。
5. **Step 5: 恢复**：从“下一步”或“未解决问题”开始。
