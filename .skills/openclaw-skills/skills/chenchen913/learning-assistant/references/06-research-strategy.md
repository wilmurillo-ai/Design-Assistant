# 调研策略模块

## 概述
定义分级调研策略和降级预案。
当用户要求“调研”、“调查”或“检查最新信息”时触发。

## 调研层级

### 1. [STD] 标准调研 (默认)
**目标**：为日常学习提供快速、可靠的答案。
**优先级**：
- **中文技术社区**：掘金, SegmentFault, 博客园, CSDN (已过滤质量)。
- **官方中文文档**：MDN (zh-CN), 框架文档 (Vue/React/Spring/Django 等)。
- **视频教程**：Bilibili (技术区)。
- **用户偏好**：检查 `workspace/USER_PROFILE.md` 中的特定站点。

### 2. [DEEP] 深度调研 (显式触发 / 自动升级)
**目标**：权威、前沿或冷门主题。

**触发逻辑**：

```
IF 用户说"深度挖掘" OR "查原始来源" OR "英文来源"：
    → 切换至 [DEEP] 深度调研

IF 标准调研结果存在以下情况：
    - 信息发布时间 > 1 年
    - 多个来源结论冲突
    - 找不到权威来源
    → 自动升级至 [DEEP] 并告知用户
```

**优先级**：
- **源代码与讨论**：GitHub (Issues, Discussions, README)。
- **问答**：Stack Overflow, Reddit (r/programming, r/MachineLearning)。
- **论文**：arXiv, Papers With Code。
- **模型与数据集**：Hugging Face。
- **标准**：RFCs, W3C Specs, Cloudflare Blog, MDN (en-US)。

## 降级预案 (当调研失败时)

如果网络/搜索失败或无结果：

1. **优先级 1**：使用搜索工具（如果可用）查询官方文档。
2. **优先级 2** (回退)：使用内部知识库。
   - **必须标注**："[WARN] 以下内容基于训练数据，建议核实最新版本"。
3. **优先级 3** (最终回退)：明确说明无法找到可靠信息。
   - 提供 **搜索建议**："[推荐平台] 搜索关键词: [关键词]"
