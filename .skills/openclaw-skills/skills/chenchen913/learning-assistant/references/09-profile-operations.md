# 用户画像操作模块

## 概述
管理用户画像（workspace/USER_PROFILE.md）的生命周期，包括初始化、被动积累和快照重建。
当涉及到用户画像的任何修改操作时触发。

## 1. 首次初始化 (第一层)

**触发条件**：`workspace/USER_PROFILE.md` 文件不存在。

**执行流程**：
1. **询问**：向用户发送以下初始化问卷。
2. **收集**：等待用户回答。
3. **创建**：根据回答内容，使用 `03-file-templates.md` 中的 `workspace/USER_PROFILE.md` 模板创建文件。

### 初始化问卷内容

```
👋 你好！在开始之前，我需要了解你一些基本情况。

1. 你目前主要关注哪个技术方向？（可多选）
   前端 / 后端 / AI·ML / 全栈 / 网络·安全 / 系统·底层 / 其他：___

2. 你的学习目的是？
   求职·面试 / 自我提升 / 项目需要 / 考研·学业 / 兴趣探索

3. 你目前的技术水平大概是？
   在校学生 / 工作 1-2 年 / 工作 3-5 年 / 工作 5 年以上

4. 你偏好的学习方式？
   看文档自学 / 边做项目边学 / 先理论后实践 / 视频课程为主

5. 有偏好的技术社区或参考平台吗？（调研时优先使用）
   例如：掘金、SegmentFault、知乎、Dev.to 等
```

---

## 2. 被动积累 (第二层)

**触发条件**：每次会话结束 Step A。

**IF** 以下任一事件发生：
- 面试/费曼模式答对率 ≥ 80%
- 攻克了 CONFUSION 文件中的某个疑惑
- 开始学习全新主题
- MISTAKE 错题被二次答对
- 用户明确说“我懂了”、“这个搞清楚了”

**THEN**：
1. 在 `workspace/USER_PROFILE.md` 的 `[GROWTH] 成长历史` 部分追加记录。
2. 更新 `[STACK] 技术栈快照` 中对应条目的状态（例如：从“学习中”移至“已掌握”）。
3. **注意**：仅追加和修改对应行，不要重写整个文件。

---

## 3. 快照重建 (第三层)

**触发条件**（由 SKILL.md Step B 定义）：
- 会话计数为 10 的倍数
- OR 用户主动要求（“更新我的学习档案”）
- OR 学习中条目 > 15 个

**执行流程**：
1. **读取数据**：读取 `workspace/USER_PROFILE.md` 的全部成长记录，以及 `workspace/LEARNING_INDEX.md` 的文件索引和主题标签云。
2. **分析状态**：根据成长记录和最近的学习活动，重新评估所有技术栈条目的状态。
3. **合并整理**：合并重复条目，清理过时状态。
4. **归档旧版本**：将当前的 `[STACK]` 快照部分（如果存在）移动到历史记录文件（如果需要）或直接覆盖。
5. **写入新快照**：生成新的 `[STACK]` 表格并写入 `workspace/USER_PROFILE.md`。
6. **更新元数据**：更新文件头部的版本号和快照日期。
7. **用户通知**：向用户展示更新摘要（新增已掌握、仍在学习中、识别到的新薄弱点、建议下一步）。
