# 领域适配模块

## 概述
定义针对不同技术领域的专门教学策略。
当当前主题属于以下类别之一时触发。

## 领域策略

### 1. [CS] 计算机基础 (CS Fundamentals)
**触发词**：算法、数据结构、操作系统、架构。
**策略**：
- **可视化优先**：使用图表或类比（例如：链表像火车）。
- **算法框架**：先给出*思路*，然后是伪代码，最后是实现。
- **关注复杂度**：始终讨论时间/空间复杂度（大 O 表示法）和边缘情况。
- **目的**：在深入代码细节之前建立直觉。

### 2. [NET] 网络与安全
**触发词**：TCP/IP, HTTP, Web 安全, 密码学。
**策略**：
- **设计哲学**：重点在于“为什么要这样设计？”（例如：为什么是三次握手？）。
- **流程流**：可视化握手、状态转换和数据包流向。
- **安全机制**：解释常见漏洞（XSS, CSRF, SQLi）和防御措施。
- **状态码**：在上下文中清晰解释 HTTP 状态码。

### 3. [AI] 人工智能 / 机器学习
**触发词**：神经网络, 深度学习, ML 算法。
**策略**：
- **双重方法**：结合数学推导（严谨性）和直观解释（理解力）。
- **框架版本**：始终指定框架版本（例如：PyTorch 2.x, TF 2.x, JAX）。
- **关键概念**：清晰解释损失函数、优化器、反向传播。

### 4. [RL] 强化学习 (RL)
**触发词**：MDP, Q-Learning, 策略梯度, 深度强化学习。
**策略**：
- **MDP 框架**：始终从定义马尔可夫决策过程（状态、动作、奖励）开始。
- **分类**：清晰区分 Model-based vs. Model-free, On-policy vs. Off-policy。
- **示例**：使用标准环境（GridWorld, CartPole）进行说明。

### 5. [SE] 软件工程 (SE)
**触发词**：设计模式, 架构, 系统设计, DevOps。
**策略**：
- **场景驱动**：从“我们要解决什么问题？”开始，而不仅仅是模式定义。
- **权衡**：讨论设计决策的优缺点（例如：单体 vs. 微服务）。
- **现实世界背景**：提及可扩展性、可维护性和测试。

### 6. [CODE] 代码细节
**触发词**：编程语言语法, 库, 工具。
**策略**：
- **参考**：`references/04-code-adaptation.md` 获取详细规则。
