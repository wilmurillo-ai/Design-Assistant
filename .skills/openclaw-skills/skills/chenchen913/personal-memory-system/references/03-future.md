# 未来维度：决策推演、目标追踪与状态预测

## 核心定位

未来维度是记忆系统的**价值变现层**。
它不做占卜，它做的是：基于你积累的数据，给出有依据的判断。

数据越多，判断越准。建议使用满 30 天后再重度依赖这一层。

## 一、三层信号系统

### 第一层：信号提取（由现在维度每日执行）

每次处理 diary 后，在 signals-log.md 末尾追加：

```
YYYY-MM-DD | 情绪:[词] | 能量:[🔴低/🟡中/🟢高] | 关键词:[词1,词2,词3] | 特殊事件:[简述或无] | 目标:[目标名或无] | 模式:[触发的模式名或无]
```

---

### 第二层：趋势分析（读取 signals-log 时执行）

**消极预警规则：**

| 信号 | 持续时间 | 预警级别 | 处理方式 |
|---|---|---|---|
| 同一消极情绪词 | 连续 3 天 | 🟡 轻度 | 在 response 中温和提及 |
| 同一消极情绪词 | 连续 5 天 | 🟠 中度 | 在 response 中重点标注，建议关注 |
| 能量🔴低 | 连续 7 天 | 🔴 高度 | 在 response 中直接说明，见 `05-protocols.md` § 预警协议 |
| 极端负面词（绝望/崩溃）| 单次出现 | ⚫ 紧急 | 立即进入危机协议 |

**积极信号识别：**
- 积极情绪词连续 3 天以上 → 标注"持续好状态"，在 response 中表达高兴
- 能量从🔴低转🟢高 → 标注"状态明显改善"
- 某个长期负面关键词连续 7 天未出现 → 标注"问题可能已解决"，值得在 response 中提及

**周期性模式检测：**
- 每周固定某天/某几天能量偏低 → 记录到 pattern-alerts.md
- 每月固定某段时间情绪波动 → 记录到 pattern-alerts.md
- 某类事件总是导致特定情绪反应 → 记录到 pattern-alerts.md

## 二、行为模式追踪（pattern-alerts.md）

**路径**：`/memory/future/patterns/pattern-alerts.md`

这个文件记录 AI 从数据中发现的、用户反复出现的行为或情绪模式。

```markdown
# 已识别的行为/情绪模式

## 格式

**模式名称**：[简短命名，例如"周末焦虑模式"]
**首次发现**：YYYY-MM-DD
**描述**：[这个模式是什么？在什么情况下触发？通常的表现是？]
**出现频率**：[大约多久出现一次]
**上次出现**：YYYY-MM-DD
**历史影响**：[这个模式通常导致什么结果？]
**有效的应对方式**：[如果用户已经找到了应对方法，记录在这里]

---

## 已识别模式列表

[AI 在数据积累到一定程度后填写]
```

**模式触发时的处理：**
在 response 的"我注意到一件事"一节中温和提及：
> "我注意到这是这个月第三次你在[情况]时出现[模式]，上次是[时间]，当时你通过[方式]走出来的。"

### 目标更新规则

| 触发 | 操作 |
|---|---|
| diary 涉及目标 | 在对应目标的"进度记录"追加 |
| 用户完成里程碑 | 更新里程碑状态 |
| 用户完成或放弃目标 | 移入 completed-goals.md |
| 用户提出新目标 | 引导细化，添加到 active-goals.md |
| 用户很久没提某个目标 | 在月报中标注"该目标已X天无进展" |

## 四、决策推演

**触发**：用户说"帮我分析"、"该不该做"、"帮我推演"、"我面临一个决定"。

### 执行前额外读取

- `past/themes/decisions.md` — 历史决策模式
- `past/themes/personality.md` — 决策风格
- 最近 30 天 diary
- 相关主题文件（职业决策→career.md；关系决策→relationships.md；等）

---

### 工作流程

**Step 1：背景确认**

先说：
> "让我先确认一下我理解的情况：[AI 复述对问题的理解]。
> 有没有我遗漏的背景信息？"

**Step 2：生成推演报告**

**路径**：`/memory/future/decisions/YYYY-MM-DD-[关键词].md`

```markdown
# 决策推演 · YYYY-MM-DD · [主题]

## 用户的问题

[原文记录]

## 我对背景的完整理解

[AI 对用户处境的理解，包括：
- 问题本质
- 用户当前状态（来自 diary 和 signals-log）
- 和用户历史决策的关联
- 用户性格档案里的相关特点]

## 分析依据（透明化）

[明确说明 AI 参考了哪些数据：]
- 参考了最近X天的情绪信号：整体状态是[xxx]
- 参考了你过去的决策记录：在类似情况下你曾经[xxx]
- 参考了你的性格档案：你在压力下倾向于[xxx]
- 参考了你的目标：这个决策和你的[xxx]目标有关联

## 这个决策的核心张力

[这件事真正让你纠结的本质矛盾是什么？
不是表面的"做还是不做"，而是背后的价值冲突或恐惧是什么？]

## 三种情境推演

### 情境 A：[选择做/方案A]

**短期（1-3个月）**：可能发生什么
**中期（半年-1年）**：可能走向哪里
**长期影响**：对你的人生轨迹意味着什么
**成立条件**：这个结果需要哪些条件支撑
**主要风险**：最大的不确定因素

### 情境 B：[选择不做/方案B]

[同上格式]

### 情境 C：[推迟/维持现状]

[同上格式]

## 魔鬼代言人

[AI 刻意说出反对最显而易见选择的理由——帮用户看到自己可能忽略的视角。
不是 AI 的立场，是为了让用户想得更全面。]

## AI 的综合判断

[基于以上分析，AI 倾向认为哪个方向，以及理由。
必须说明这是基于什么数据得出的判断，不是凭感觉。]

## 你可能还没想到的角度

[1-2 个用户可能忽略的因素或视角]

## 留给你的问题

[2-3 个问题，帮用户进一步想清楚自己的真实想法：]
- "在你做这个决定之前，你有没有想清楚：[问题]？"

## 待验证预判

[记录这次推演的核心预判，格式：]
- 预判1：如果你选择了XXX，那么在YYY时间内，可能发生ZZZ
- 预判2：...

**建议验证时间**：YYYY-MM-DD 前后
```

## 五、决策验证

**触发**：用户说"验证之前的推演"、"之前那个分析准吗"，或推演文件中设定的验证日期到来。

**路径**：`/memory/future/verification/YYYY-MM-DD-验证-[主题].md`

```markdown
# 决策验证 · YYYY-MM-DD · [主题]

## 原始推演

**推演日期**：YYYY-MM-DD
**链接**：/memory/future/decisions/YYYY-MM-DD-主题.md
**核心预判**：[列出原始推演中的待验证项]

## 实际发生了什么

[用户描述的真实结果]

## 验证评分

| 预判 | 实际结果 | 准确度 |
|---|---|---|
| | | ✅准确 / ⚠️部分准确 / ❌不准确 |

## 推演偏差分析

[如果有偏差，分析原因：]
- 信息不足？（当时不知道什么关键信息）
- 用户行为改变？（用户做了和预期不同的事）
- 外部因素？（不可预见的外部变化）
- AI 判断错误？（AI 的逻辑本身有问题）

## 这次验证让我学到的

[对 AI 理解这个用户有什么新的认识？如何改进下次推演？]

[同时更新 ai-portrait.md 中的"决策风格"部分]
```

## 七、注意事项

- 每次推演后都说：**"这只是参考，最终决定是你的。"**
- 每次推演必须说明分析依据，不能凭感觉给判断
- 数据不足时坦诚说明，不用不存在的数据做推演
- 发现高度预警信号，温和但直接地说出来，不回避
- 若预警信号涉及心理健康，建议寻求专业支持
- 定期报告需用户主动触发，不自动生成
