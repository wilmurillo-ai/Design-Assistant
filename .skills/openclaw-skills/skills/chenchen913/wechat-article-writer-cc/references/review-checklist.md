# 三轮审核详细检查清单

## 审核铁律

- **触发时机**：本文件所有内容在 Phase 5 中执行。Phase 4 初稿完成后，AI 必须主动宣告进入 Phase 5，不得等待用户要求。
- 三轮必须按顺序执行：内容 -> 风格 -> 细节
- 每轮发现问题须修改后重新检查该轮，通过后才能进入下一轮
- 用户可在每轮开始前选择跳过（但建议至少执行第二轮风格审校）
- 每轮审核结果必须以对比格式呈现（修改前 vs 修改后）
- 审核修改同样遵循「分段修改原则」：只修改发现问题的部分，不全篇重写

---

## 第一轮：内容审校（确保"对"）

目标：确保文章内容准确、逻辑自洽、结构完整。

### 检查清单

| 编号 | 检查项 | 具体标准 | 处理方式 |
|------|--------|---------|---------|
| 1.1 | 事实核查 | 数据统计是否准确？来源是否可靠？ | 存疑标注 [存疑]，给出修改建议 |
| 1.2 | 案例真实性 | 引用的案例是否真实存在？有无虚构成分？ | 虚构的标注 [虚构]，用真实素材替换 |
| 1.3 | 逻辑自洽 | 论点->论据->结论链条是否完整？有无逻辑跳跃？ | 补充缺失的过渡和论据 |
| 1.4 | 结构完整 | 大纲中承诺的内容是否全部覆盖？ | 遗漏内容补回 |
| 1.5 | 敏感检查 | 是否有敏感词、违禁内容、夸大误导？ | 直接修改或删除 |
| 1.6 | 观点依据 | 观点结论是否有支撑？是否过于主观？ | 补充依据或弱化表述 |
| 1.7 | 引用规范 | 引用他人观点是否标注了来源？ | 补充来源标注 |
| 1.8 | ⏰ 信息时效性 | 数据/工具/政策/技术是否为最新？版本号、价格、市场份额等易变信息是否过时？ | 使用搜索工具（加当前年份关键词）验证，过时则更新或标注 |
| 1.9 | ⏰ 信息来源日期 | 引用的报告/数据是否标注了年份/日期？ | 补充日期标注，如"据 2026 年 XX 报告" |

### 输出格式

```
## 第一轮审核结果：内容审校

### 发现的问题

1. [1.1 事实核查] 第 X 段："全球有 80% 的人使用 AI"
   - 问题：数据来源不明，可能不准确
   - 建议：改为"据 XX 报告，XX% 的人..."或删除具体数字

2. [1.3 逻辑自洽] 第 Y 段到第 Z 段
   - 问题：从"AI 的优势"直接跳到"实操步骤"，缺少过渡
   - 建议：补充一段说明"既然 AI 有这些优势，那如何落地？"

3. [1.8 ⏰ 信息时效性] 第 W 段："ChatGPT 目前最新版本是 GPT-4"
   - 问题：版本信息可能已过时，需搜索验证
   - 建议：搜索"ChatGPT 最新版本 2026"确认后更新

4. [1.9 ⏰ 信息来源日期] 第 V 段："据报告显示市场规模达 XX 亿"
   - 问题：缺少报告年份和来源
   - 建议：补充为"据 2026 年 XX 机构报告..."

### 修改总结
- 共发现 X 个问题
- 已修改 X 个
- 需用户确认 X 个
```

## 第二轮：风格审校（确保"好读"）

目标：消除 AI 味，让文章读起来像人写的。这是最关键的一轮。

### 6 步动作清单

#### 动作 1：删套话

删掉所有万能短语和空洞表达。

| 要删除的套话 | 替代方式 |
|-------------|---------|
| "众所周知" | 直接进入正题，或改为具体事实 |
| "不言而喻" | 删掉，直接说结论 |
| "在当今 XX 的时代" | 删掉，直接说事 |
| "让我们一起探索" | 删掉 |
| "值得注意的是" | 删掉，直接说 |
| "综上所述" | 改为简洁的总结句 |
| "总而言之" | 改为"一句话：" |

改写示例：
- 改前："在当今 AI 技术飞速发展的时代，我们需要重新思考写作的意义。"
- 改后："上周试了个新工具，写了三篇文章，确实省了不少时间。"

#### 动作 2：拆长句

一口气读完很累的句子（超过 40 字）拆成短句。

改写示例：
- 改前："为了能够更好地帮助用户理解如何在日常工作中高效地利用 AI 工具来提升自己的写作效率和内容质量，我们需要先了解一些基本概念。"
- 改后："怎么用 AI 写得又快又好？先搞懂几个基本概念。"

#### 动作 3：口语化

把书面表达改成口语表达。

| 书面表达 | 口语化 |
|---------|--------|
| "此方案具有较高的可行性" | "这个方案能用" |
| "该功能尚处于实验阶段" | "这功能还在测试，别太指望" |
| "用户反馈普遍较为正面" | "大家用了都说好" |
| "建议采用以下方案" | "我的建议是这样" |

#### 动作 4：加细节

空洞的地方补充具体细节或例子。

改写示例：
- 改前："效果很好。"
- 改后："阅读量从 200 涨到了 2000，留言区第一次超过 50 条。"

- 改前："这个方法很实用。"
- 改后："我上周用这个方法写了一篇，从选题到发布只花了 2 小时。"

#### 动作 5：去连接词堆砌

删掉过度的连接词，用段落自然衔接替代。

要警惕的连接词：此外、另外、与此同时、不仅如此、更重要的是、除此之外。

原则：如果删掉连接词后句子仍然通顺，就删掉它。

#### 动作 6：检查 AI 味

找出所有"太像 AI 说的话"，逐一改掉。

AI 味的典型特征：
- "首先...其次...最后..."连用三次以上
- 开头就是一个宏大叙事
- 每段都有转折词
- 句式高度一致（排比感太强）
- 用词过于"正确"和"完美"

### 输出格式

```
## 第二轮审核结果：风格审校

### 修改清单

1. [删套话] 第 X 段
   - 改前："在当今数字化转型的浪潮中..."
   - 改后："最近公司要求用 AI 写周报。"

2. [拆长句] 第 Y 段
   - 改前："为了能够更好地..."（52 字）
   - 改后：拆为两句（28 字 + 18 字）

3. [检查 AI 味] 全文
   - 改前："首先...其次...再次...最后..."
   - 改后：删除编号，用自然段落衔接

### 修改统计
- 删套话：X 处
- 拆长句：X 处
- 口语化：X 处
- 加细节：X 处
- 去连接词：X 处
- 去 AI 味：X 处
```

## 第三轮：细节打磨（确保"精"）

目标：打磨句子节奏、修正标点格式、确保公众号平台兼容。

### 检查清单

| 编号 | 检查项 | 具体标准 |
|------|--------|---------|
| 3.1 | 句子节奏 | 长短句是否交替？有没有韵律感？连续短句不超过 3 句 |
| 3.2 | 段落节奏 | 信息密度是否均匀？有没有"喘气空间"？每段不超过 5 行 |
| 3.3 | 标点规范-中英混用 | 中文文章必须用中文标点（，。！？：；""）而非英文标点 |
| 3.4 | 标点规范-引号 | 中文文章用中文引号""或「」，不用英文引号"" |
| 3.5 | 标点规范-感叹号 | 全文感叹号不超过 3 个，避免滥用 |
| 3.6 | 中英文空格 | 英文/数字与中文之间加空格：AI 写作（对）/ AI写作（错） |
| 3.7 | Markdown-引用块 | Markdown 引用块（以 `>` 开头）单次不超过 300 个字符（无法精确计数时宁可拆小），超出则拆分或改为正文 |
| 3.8 | 公众号-段落长度 | 适配移动端，每段 3-5 行为宜 |
| 3.9 | Markdown-引用块嵌套 | 引用块内不嵌套代码块或复杂格式 |
| 3.10 | 空行检查 | 段落间是否有合理空行，避免内容挤在一起 |
| 3.11 | emoji 使用 | emoji 是否恰当使用？不过度堆砌？位置是否合理（标题、要点前）？ |
| 3.12 | Markdown-星号兼容 | 使用 `*` / `**`（斜体/加粗等）时，星号后是否有空格（示例：`** 加粗 **`、`* 斜体 *`），避免部分渲染器不识别？ |

### 工具辅助

机械性格式问题（中英文空格、标点全半角）可调用以下工具处理：

- format-markdown Skill（如果用户已安装）
- 或手动逐项检查

原则：AI 做需要判断的事（节奏、可读性），脚本做机械的事（空格、标点）。

### 输出格式

```
## 第三轮审核结果：细节打磨

### 修改清单

1. [3.3 标点] 第 X 段：英文逗号 "," 改为中文逗号 "，"
2. [3.6 空格] 第 Y 段："AI写作" 改为 "AI 写作"
3. [3.7 引用块] 第 Z 个 Markdown 引用块共 328 字符，超出 28 字符
   - 建议：拆分为两个引用块，或精简表达
4. [3.12 星号] 第 W 段：部分渲染器下加粗可能不识别（`**加粗**`）
   - 建议：按兼容写法改为 `** 加粗 **`（星号后加空格）

### 格式统计
- 标点修正：X 处
- 空格修正：X 处
- 引用块调整：X 处
- 段落调整：X 处
```
