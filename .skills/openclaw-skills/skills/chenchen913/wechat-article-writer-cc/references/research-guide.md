# 市场调研指南 (Phase 2)

## ⚠️ 调研前置动作（每次调研前必须执行）

**Step 0：确认当前日期**
在开始任何搜索之前，AI 必须先通过系统工具或对话确认今天的日期，并在调研报告开头标注：
"本次调研执行于：[YYYY年MM月DD日]"

**为什么要做这一步**：AI 的训练数据有截止日期，搜索结果也可能混入旧内容。
明确标注调研日期，方便用户判断信息是否过时。

**Step 1：搜索词必须带年份**
所有搜索词默认追加当前年份。
- 错误示例：搜索"ChatGPT 最新功能"
- 正确示例：搜索"ChatGPT 最新功能 2026"

对于时效性强的内容（AI工具、政策、市场数据），还需额外搜索"[关键词] 最新 [当前年份]"作为验证。

**Step 2：对搜索结果标注时间**
引用任何数据或工具信息时，必须标注信息的发布时间或有效期：
- ✅ 正确："据 2026 年 3 月 OpenAI 官网，ChatGPT 当前最新版本为..."
- ❌ 错误："ChatGPT 目前最新版本是..."（无日期）

## 调研策略（分层优先级）

| 优先级 | 平台 | 重点 |
|--------|------|------|
| P0 | 公众号/微信搜索 | 深度长文、爆款规律、创作者视角 |
| P1 | **读者评论区** | 真实痛点、高频疑问、读者原话 |
| P2 | 知乎/Github | 专业视角、高赞回答 |
| P3 | 小红书 | 用户痛点、真实体验 |
| P4 | 微博/Twitter | 热点讨论、舆论风向 |
| P5 | Reddit/YouTube | 国外视角、社区讨论 |

## 关键词策略（三层结构）

AI 每次调研必须覆盖以下三层关键词：

1.  **第一层「核心词」**：文章主题的直接关键词（如"AI 写作"）
2.  **第二层「延伸词」**：核心词 + 修饰词的组合（如"AI 写作 入门"、"AI 写作 避坑"、"AI 写作 2025"）
3.  **第三层「读者语言词」**：读者遇到问题时会搜的词（如"用 AI 写出来的文章很假怎么办"、"AI 代写被发现"）

## 搜索执行要求

*   **标准模式**：至少执行 **5 次独立搜索**（覆盖三层关键词 + 至少 2 个平台）
*   **快速模式**：至少执行 **2 次搜索**（核心词 + 读者语言词各一次）

**精读要求**：
*   搜索结果中，选出最相关的 2-3 篇文章，使用 `fetch` 工具读取完整内容，不只看摘要。
*   精读优先级：阅读量高的 > 评论互动多的 > 结构完整的。

## 新增：读者视角调研

在平台列表中，新增一个调研维度：**读者评论区**。

| 调研维度 | 目标 | 方法 |
| :--- | :--- | :--- |
| **创作者视角** | 了解现有内容生态、结构风格、爆款规律 | 搜索并分析文章正文 |
| **读者视角** | 了解真实痛点、用词习惯、高频疑问 | 重点看文章评论区、知乎问答高赞回答、小红书笔记评论 |

**输出要求**：提炼出 3-5 句"读者原话"（直接引用评论原文），这些原话后续可用于文章开头或金句。

## 调研结果与后续 Phase 的衔接规则

调研完成后，AI 必须输出一个"调研结论 → 写作映射表"：

| 调研发现 | 映射到写作的哪个环节 | 具体建议 |
| :--- | :--- | :--- |
| 读者最关心的痛点是 XX | 文章开头 | 用这个痛点作为开头钩子 |
| 同类文章普遍缺少 YY 角度 | 大纲差异化点 | 在大纲中设置一个专门讲 YY 的部分 |
| 读者常用的表达是"ZZ" | 写作用词 | 在文章中使用这些读者熟悉的表达 |
| 爆款文章都用了 AA 结构 | 大纲结构 | 参考 AA 结构，但加入个人故事做区分 |

## 网络故障三级预案

| 级别 | 条件 | 预案 |
| :--- | :--- | :--- |
| 一级 | 主搜索失败 | 切换备选搜索工具 |
| 二级 | 所有搜索失败 | 基于 AI 知识库"模拟调研"，**必须标注信息截止日期** |
| 三级 | AI 也不熟悉 | 诚实告知，建议用户提供参考文章 |

无论哪级预案，必须告知用户数据来源和信息日期。

调研后向用户提出 3-5 个确认问题，每个附参考答案。
