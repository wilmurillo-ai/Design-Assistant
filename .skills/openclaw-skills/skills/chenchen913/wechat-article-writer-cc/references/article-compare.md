# 文章对比分析指南

## 1. 触发方式

用户说出以下任意表达时，AI 进入对比模式：
- "我有另一篇文章想对比一下"
- "帮我比较这几篇文章"
- "我用 XX 也写了一篇，看看哪个更好"
- "文章对比"
- "对比分析"

**对比模式可在任意时机触发**：
- 在 Phase 4 初稿完成后触发 → 正常流程
- 在 Phase 1 之前触发 → AI 先做对比，再决定是否需要重新写或直接用某篇进入 Phase 5
- 随时触发均可，AI 根据当前 Phase 状态灵活衔接

---

## 2. 对比流程（4步）

### Step 1：收集材料

AI 询问用户提供所有待对比的文章（包括当前这篇），每篇给一个编号（文章A、文章B...）。

AI 询问：本次对比的核心诉求是什么？（选择或组合）：
1. 选出最好的一篇直接用
2. 找出各篇优点合并成一篇
3. 以某篇为主、其他为参考改进

### Step 2：逐维度对比分析

AI 按以下维度输出对比表格：

```markdown
## 文章对比分析报告

| 维度 | 文章A | 文章B | 文章C（如有） |
|------|-------|-------|--------------|
| 开头方式 | [描述] | [描述] | |
| 整体结构 | [描述] | [描述] | |
| 核心观点 | [描述] | [描述] | |
| 论据丰富度 | [★★★☆☆] | [★★★★☆] | |
| 语言风格契合度* | [描述] | [描述] | |
| 金句/亮点 | [摘录] | [摘录] | |
| 明显不足 | [描述] | [描述] | |
| 字数 | [X字] | [X字] | |

*语言风格契合度：与 my-style.md 中定义的个人风格的匹配程度
```

### Step 3：综合推荐

AI 给出明确建议，格式如下：

```markdown
## 对比结论

**推荐方案**：[选A直接用 / 选B为主 / 融合改写]

**推荐理由**：[2-3句话，基于上方对比数据]

**如果选择融合改写，建议如下**：
- 从文章A借鉴：[具体内容，如"第二部分的案例更生动"]
- 从文章B借鉴：[具体内容，如"开头的钩子更抓人"]
- 保留当前文章的：[具体内容]
- 建议舍弃的：[具体内容]
```

### Step 4：执行

根据用户选择的方案，AI 进入对应动作：
- **选A/B直接用** → 对选中文章执行 Phase 5 三轮审核
- **融合改写** → AI 按融合方案重新生成一篇，然后执行 Phase 5
