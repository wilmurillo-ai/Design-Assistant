# 风格学习指南

## 概述

本指南说明如何使用个人素材库（my-articles/）和风格档案（my-style.md）来实现：
1. 事前预防 AI 味（用真实素材替代编造）
2. 让 AI 越来越懂你的写作风格（迭代学习）

---

## 个人素材库（my-articles/）

### 如何使用

用户将自己写过的文章存入 `my-articles/` 文件夹：
- 支持 Markdown（.md）和纯文本（.txt）格式
- 建议存入 5-10 篇最能代表自己风格的文章
- 文件名建议包含日期或主题，方便检索（如 `2026-AI写作心得.md`）

### AI 读取策略（Token 优化）

```
场景判断流程：

1. 检查 my-style.md 是否已有内容（非模板状态）
   |
   ├─ 已有内容 -> 只读 my-style.md（约 100-200 行，节省 Token）
   |   └─ 如需检索素材 -> 按关键词在 my-articles/ 中搜索，只读匹配的文件
   |
   └─ 为空/模板状态 -> 检查 my-articles/ 是否有文件
       |
       ├─ 有文件 -> 采样读取（最多 5 篇），分析风格，生成 my-style.md
       |
       └─ 无文件 -> 跳过，通过对话采集风格信息
```

关键原则：
- 绝对不要每次都读取全部文章
- my-style.md 是"缓存"，读它就够了
- 只有首次使用或用户要求更新时才重新分析文章

### 素材检索方法

当需要从 my-articles/ 中检索相关素材时：

1. 确定当前写作话题的关键词（2-5 个）
2. 在 my-articles/ 中按关键词搜索（使用可用的文件搜索工具，如 Grep、ripgrep 或 IDE 内置搜索）
3. 只读取命中的段落和上下文，不读取整篇文章
4. 将找到的素材（真实经历、观点、案例、金句）整理为清单
5. 呈现给用户确认哪些可以用于本次写作

注意：忽略 `PLACE_YOUR_ARTICLES_HERE.md`，该文件是使用说明，不是用户文章。

## 风格档案（my-style.md）

### 生成流程

首次使用时，AI 读取 my-articles/ 中的文章（采样 3-5 篇），分析以下维度：

1. **用词偏好**
   - 高频词汇（排除通用词）
   - 口头禅或专属表达
   - 偏好的动词、形容词

2. **句式特征**
   - 平均句长（短句为主 / 长短交替 / 长句为主）
   - 段落平均长度
   - 是否常用反问句、设问句

3. **开头方式**
   - 常用的开头模式（痛点切入 / 故事开场 / 数据引入 / 直接观点）

4. **结构习惯**
   - 常用的文章结构（总分总 / 递进 / 并列 / 问答）
   - 是否有固定的段落套路

5. **语气风格**
   - 正式程度（1-10，1=极口语，10=极书面）
   - 幽默程度（1-10）
   - 是否有犀利/温和/中立的倾向

6. **金句风格**
   - 是否常用金句
   - 金句的类型（比喻型 / 反转型 / 排比型 / 短句型）
   - 金句出现的位置（开头 / 段末 / 结尾）

7. **禁忌表达**
   - 用户明确不喜欢的表达方式
   - 从文章中推断出的避免倾向

### my-style.md 模板

```markdown
# 我的写作风格档案

> 本文件由 AI 分析个人素材库后自动生成。
> 最后更新：[日期]
> 分析样本：[文件名列表]

## 用词偏好
- 高频表达：[列表]
- 口头禅：[列表]
- 偏好动词：[列表]

## 句式特征
- 平均句长：约 [X] 字
- 段落长度：约 [X] 行
- 特殊句式：[如常用反问句]

## 开头方式
- 最常用：[模式]
- 次常用：[模式]

## 结构习惯
- 偏好结构：[模式]
- 段落节奏：[描述]

## 语气风格
- 正式程度：[X]/10
- 幽默程度：[X]/10
- 整体倾向：[描述]

## 金句风格
- 频率：[高/中/低]
- 类型：[描述]
- 示例金句：
  - "[从文章中提取的金句 1]"
  - "[从文章中提取的金句 2]"

## 禁忌表达
- [列表]

## 风格总结（一段话）
[用 100 字左右概括用户的整体写作风格]
```

### 更新机制

以下情况触发 my-style.md 更新：

1. 用户在 my-articles/ 中新增了文章，并明确要求"更新我的风格档案"
2. 用户对 AI 写的文章进行了大量手动修改，并反馈了修改原因
3. 用户直接告诉 AI"我的风格有变化"
4. 用户通过素材库管理接口直接修改了风格档案中的字段

更新时：
- 读取新增的文章（不重新读全部）
- 对比现有风格档案，标注变化
- 更新 my-style.md 中有变化的字段
- 记录更新日期和分析样本

## 无素材库时的替代方案

如果用户没有 my-articles/ 中的文章，通过对话采集风格信息：

1. "你平时写文章（或发朋友圈）是什么风格？"
   参考答案："比较口语化，喜欢用短句"
2. "有没有你特别喜欢的博主或写作风格？"
   参考答案："喜欢 XX 的风格，简洁有力"
3. "有没有你特别不喜欢的表达方式？"
   参考答案："不喜欢太正式的书面语"

根据回答手动填充 my-style.md 的关键字段。

## 风格迭代学习

每次交付文章后，提示用户：

```
文章已交付。如果你后续对文章做了修改，可以：
1. 将修改后的版本存入 my-articles/ 文件夹
2. 告诉我你改了哪些地方、为什么这么改

我会从你的修改中学习，更新你的风格档案，下次写得更像你。
```

这实现了"越写越像你"的持续迭代。

## 个人素材库管理接口

用户可在工作流的任意时刻通过自然语言触发素材库管理功能，无需中断当前流程。

### 触发识别

AI 需要识别以下用户意图并执行对应操作：

| 用户可能的表述 | 对应操作 |
|--------------|---------|
| "查看我的写作风格" / "我的风格是什么" | 读取并展示 `my-style.md` |
| "我的风格太口语化了" / "风格太偏激" | 修改 `my-style.md` 中的对应字段 |
| "把正式程度调高一点" | 修改 `my-style.md` 中的具体参数 |
| "重新分析我的风格" | 重新采样 `my-articles/` 分析并重新生成 `my-style.md` |
| "看看我的素材库" / "我有哪些文章" | 列出 `my-articles/` 文件列表和摘要 |
| "在我的文章里搜索 XX" | 按关键词在 `my-articles/` 中检索 |

### 风格修改流程

当用户对风格档案不满意时（如"太口语化""太偏激""太片面"）：

1. 读取当前 `my-style.md`
2. 展示用户反馈涉及的字段当前值
3. 根据用户反馈提出修改方案（展示修改前后对比）
4. 用户确认后，保存修改到 `my-style.md`
5. 确认："✅ 风格档案已更新。下次写作时我会参照最新的风格。"

示例对话：
```
用户：我看了下我的写作风格，太口语化了，正式程度太低了
AI：📄 当前风格档案中：
  - 正式程度：3/10
  - 整体倾向：轻松随意，偏口语化

  我建议调整为：
  - 正式程度：6/10
  - 整体倾向：轻松但不失专业感，保持亲和力的同时增加专业表达

  确认修改吗？
用户：好的
AI：✅ 风格档案已更新。下次写作时我会参照这个更正式的风格。
```

### 操作后的流程恢复

素材库管理操作完成后，AI 需要：
1. 告知用户操作结果
2. 自动判断应回到工作流的哪个步骤（参见全局规则中的"步骤跳跃判断机制"）
3. 如果在写作过程中修改了风格，提示用户当前文章是否需要根据新风格调整
