## 1. 测试策略

### 1.1 测试金字塔

```
                    ┌───────────┐
                    │   E2E     │     端到端测试 (少量)
                    │   Tests   │     验证完整业务流程
                    └─────┬─────┘
                          │
              ┌───────────┴───────────┐
              │      Integration       │     集成测试 (适量)
              │        Tests           │     验证模块间交互
              └───────────┬───────────┘
                          │
        ┌─────────────────┴─────────────────┐
        │           Unit Tests              │   单元测试 (大量)
        │            (核心)                  │   验证单个类/方法
        └─────────────────────────────────┘
```

### 1.2 测试目标

| 测试类型 | 覆盖率目标 | 说明 |
|----------|------------|------|
| 单元测试 | ≥ 80% | 核心业务逻辑、工具类 |
| 集成测试 | ≥ 60% | Controller、Service 层 |
| E2E 测试 | 关键路径 | 核心业务流程 |
