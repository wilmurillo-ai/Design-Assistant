## ⚙️ 功能介绍

### 🎯 缰绳 (Constraints) - 架构约束

定义 AI 必须遵守的规则边界，确保输出符合预期。

**约束类型**：
- 输出格式约束（Markdown、UTF-8）
- 文档结构约束（必须包含的章节）
- 内容约束（禁止占位符、禁止 TODO）
- 质量约束（最小长度、覆盖率）

### 📦 马车 (Context) - 上下文工程

为 AI 提供丰富的上下文知识，确保生成内容准确、完整。

**上下文来源**：
- 项目结构分析
- 技术栈检测
- 数据库结构扫描
- API 接口分析
- 配置文件解读

### 🪞 镜子 (Feedback) - 反馈循环

通过多层校验确保输出质量，形成闭环反馈。

**校验机制**：
- 结构验证（章节完整性）
- 内容验证（禁止占位符）
- 质量验证（长度、格式）
- 一致性验证（术语统一）

### 🧹 车夫 (Entropy Management) - 熵管理

清理输出中的冗余、错误和不一致，保持输出整洁。

**清理内容**：
- 多余空行
- 格式不一致
- 重复内容
- 标点符号

## ⚙️ 配置说明

### 配置文件

在项目根目录创建 `project-analyzer.yaml`：

```yaml
project:
  name: my-project
  type: spring-boot

output:
  base_dir: docs

sdd:
  generate_srs: true
  generate_sad: true
  generate_sdd: true
  generate_dbd: true
  generate_apid: true
  generate_tsd: true

standards:
  auto_create: true
```

### 约束配置

```yaml
documents:
  srs:
    required_sections:
      - 引言
      - 功能需求
      - 非功能需求
    min_length: 2000

  dbd:
    required_sections:
      - 数据库概述
      - 表结构
      - 索引设计
    min_length: 1500

content:
  language: zh-CN
  no_placeholders: true
  no_todos: true
```

## ❓ 常见问题

### Q1: 支持哪些技术栈？

**A**: 支持：
- Java / Spring Boot
- Node.js
- Python
- Go
- 前端项目（React/Vue）

### Q2: 支持哪些数据库？

**A**: 支持：
- MySQL
- PostgreSQL
- Oracle
- SQL Server
- MongoDB

### Q3: 生成的文档准确吗？

**A**: 
- ✅ 基于实际代码扫描生成
- ✅ SQL 文件与 DO 实体类对照验证
- ✅ Controller 与实际接口对照
- ✅ 技术栈基于配置文件判断
- ⚠️ 建议人工审核后使用

### Q4: 可以自定义模板吗？

**A**: 可以。在 `references/templates/` 目录下放置自定义模板。

### Q5: 如何更新生成的文档？

**A**: 
1. 代码变更后重新运行分析
2. 对比文档差异
3. 手动更新或重新生成
