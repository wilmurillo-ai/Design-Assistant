# Report format

## Short report
Use this default structure:
1. input investigated
2. target type
3. strongest matches
4. weaker / uncertain matches
5. breach result if applicable
6. confidence summary
7. caveats

## Extended report
Use when the user wants detail:
- identity summary
- confirmed public profiles
- likely public profiles
- weak / ambiguous findings
- breach exposure section if email was checked
- correlation notes
- sources consulted

## Rule
Lead with the strongest evidence first.
Do not bury the useful matches under weak noise.
