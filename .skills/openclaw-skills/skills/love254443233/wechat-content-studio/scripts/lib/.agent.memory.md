## 文件夹摘要 (Directory Summary)

> 微信公众号内容工作室技能中，与 OpenClaw / Agent 运行时对接的共享环境加载与密钥解析逻辑。

## 文件夹内容结构 (Structure)

> `openclaw_env.js`：从多层 `.env` 与进程环境变量中解析 LLM / DashScope / 微信发布等配置，供 `main.js`、改写、封面、发布脚本复用。

## 遗留问题 (Legacy Issues)

> 无。后续若 OpenClaw 官方约定新的全局变量名，在此集中增加别名即可。

## 特殊决策 (Special Decisions)

> 采用「不覆盖已存在于 `process.env` 的键」策略，与常见 dotenv 行为一致，便于 CI/CD 与本地 `.env` 共存。

## 注意事项 (Common Pitfalls & Conventions)

> 加载顺序：`OPENCLAW_ENV_FILE` → 技能根目录 `.env` → `~/.openclaw/.env` → `~/.workbuddy/.env`。仅补充缺失变量；真实 Shell 导出的变量始终优先。
