## 📋 测试方案（3 个级别）

### 🔹 方案 A: 快速测试（5 分钟）⭐推荐先做

**目标**: 验证基本功能和自动安装

**步骤 1: 安装/更新技能**
```bash
clawhub install human-like-memory-cn
# 或
clawhub update human-like-memory-cn
```

**预期**: OpenClaw 自动检测并安装依赖（后台执行 `npm install`）

**步骤 2: 在聊天对话中测试**

```
/remember 我住在杭州西湖区，喜欢喝拿铁咖啡
```

**预期回复**:
```
✅ 已创建记忆：杭州居住偏好
   - 重要性：75 分
   - 分类：preference
   - 存储：WARM 记忆
```

**步骤 3: 查看统计**

```
/memory stats
```

**预期**: 显示记忆数量、压缩分布、Token 节省等

**步骤 4: 测试搜索**

```
/memory search 咖啡
```

**预期**: 返回拿铁咖啡相关记忆

### 🔹 方案 B: 向量化测试（15 分钟）⭐认真测试

**前提**: 首次使用会自动下载模型（约 200MB，需要网络）

**步骤 1: 创建多条记忆**

```
/remember 我喜欢吃重庆火锅，偏好麻辣口味
/remember 上海菜偏甜，我不太喜欢
/remember 川菜很辣，但是很好吃
/remember 粤菜比较清淡，适合养生
```

**步骤 2: 测试语义搜索**

```
/memory search 我喜欢吃什么口味的菜？
```

**预期**: 返回火锅/麻辣相关记忆（即使问题中没有 exact match「火锅」）

**步骤 3: 测试智能问答**

```
问：我对食物有什么偏好？
```

**预期**: 综合多条记忆回答（麻辣、不喜欢甜、清淡养生）

## 🐛 常见问题

### Q1: 命令没反应

**原因**: 在系统控制台输入，而不是聊天对话

**解决**: 确保在 OpenClaw **聊天窗口**输入命令

### Q2: 提示缺少依赖

**检查**:
```
/memory stats
```

**解决**:
```bash
cd /home/admin/openclaw/workspace/skills/human-like-memory
npm install
```

### Q3: 向量化失败（fetch failed）

**原因**: 网络问题导致模型下载失败

**解决 1**: 使用国内镜像
```bash
export HF_ENDPOINT=https://hf-mirror.com
cd /home/admin/openclaw/workspace/skills/human-like-memory
npm install
```

**解决 2**: 临时关闭向量搜索
```json
// 编辑 config.json
{ "useVectorSearch": false }
```
