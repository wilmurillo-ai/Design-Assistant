## 自动功能（无需手动操作）

### 1. 自动记忆编码

**触发条件：**
- 用户说"记住 XXX"、"记下来"、"别忘了 XXX"
- 检测到重要信息（偏好、决策、联系方式等）
- 每 10 轮对话自动扫描

**示例：**
```
用户：记住我喜欢喝拿铁
→ 自动创建记忆：咖啡偏好 - 喜欢拿铁 (importance: 75)
```

### 2. 自动上下文压缩

**触发条件：** 上下文使用率 >70%

**压缩策略：**
1. 移除<30 分的低价值记忆
2. 将 HOT 记忆摘要化
3. 将 WARM 记忆转为 COLD（只保留摘要）
4. 保留≥80 分核心记忆

### 3. 自动记忆复习

**触发条件：** 记忆到期（默认 7 天）

**复习效果：**
- 复习后重要性 +10 分（上限 100）
- 未复习记忆重要性 -20 分
- <30 分自动遗忘

## 手动命令

### 编码记忆

```
/remember [内容]
例：/remember 我最喜欢的颜色是蓝色
```

```
/remember this
例：（在对话后）/remember this
→ 编码当前对话中的重要信息
```

### 查询记忆

```
/memory search [关键词]
例：/memory search 咖啡
→ 返回所有与咖啡相关的记忆
```

```
/memory stats
→ 显示记忆统计信息
```

```
/memory review
→ 手动触发到期记忆复习
```

### 管理记忆

```

→ 删除指定记忆
```

```
/memory export
→ 导出所有记忆为 JSON
```

```
/memory compact
→ 手动触发记忆压缩
```

## 配置选项

编辑 `skills/human-like-memory/config.json`：

```json
{
  "autoRemember": true,              // 是否自动记忆
  "reviewIntervalDays": 7,           // 复习间隔（天）
  "compactWhenContextPercent": 70,   // 触发压缩的上下文阈值
  "maxHotMemories": 20,              // HOT 记忆上限
  "maxWarmMemories": 100             // WARM 记忆上限
}
```

## 最佳实践

### ✅ 推荐做法

1. **明确指示** — 说"记住 XXX"比隐含表达更可靠
2. **定期复习** — 使用 `/memory review` 保持记忆活跃
3. **添加标签** — 帮助记忆分类和检索
4. **及时清理** — 删除过期或错误的记忆

### ❌ 避免做法

1. **过度记忆** — 不是所有信息都值得保存
2. **从不复习** — 记忆会随时间"衰退"
3. **模糊标签** — 使用具体、可搜索的标签

## 故障排除

### 记忆没有被保存？

1. 检查 `autoRemember` 是否为 `true`
2. 确认信息重要性≥30 分
3. 查看 HOT 记忆文件是否可写

### 记忆检索不到？

1. 尝试不同关键词
2. 检查记忆是否被归档
3. 确认记忆未被遗忘（<30 分）

### 上下文没有压缩？

1. 检查当前上下文使用率是否>70%
2. 手动运行 `/memory compact`
3. 检查是否有≥80 分的核心记忆（不压缩）
