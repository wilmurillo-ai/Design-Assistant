# 抖音自动回复监控助手

## 快速使用

### 1. 检查评论状态

```
1. 确保浏览器在评论管理页面
2. 执行 browser snapshot
3. 解析返回的 DOM 结构：
   - 查找包含用户名的文本
   - 查找评论内容
   - 检查是否有 "回复" 按钮（表示可以回复）
```

### 2. 解析 Snapshot 的关键模式

**评论项特征：**
```
- 用户名：较短文本，通常在前面
- 时间："X分钟前"、"X小时前"、"刚刚"、"MM-DD"
- 内容：主要文本，长度不等
- 按钮："回复"、"删除"、"举报"
```

**未回复特征：**
```
- 没有 "作者" 嵌套回复
- 有 "回复" 按钮（可以点击）
```

### 3. 正确的评论回复流程

⚠️ **重要：必须使用嵌套输入框**

```
1. 点击目标评论的 "回复" 按钮（不是顶部输入框！）
   browser act click ref: <评论的回复按钮ref>

2. 等待嵌套输入框展开
   browser snapshot
   // 输入框 placeholder 应显示 "回复 @用户名："

3. 找到嵌套输入框（第二个 contenteditable 元素）
   browser act evaluate fn: "() => {
     const editables = document.querySelectorAll('[contenteditable=\"true\"]');
     const replyInput = editables[1]; // 第二个是回复输入框
     replyInput.focus();
     document.execCommand('insertText', false, '回复内容');
     replyInput.dispatchEvent(new Event('input', {bubbles: true}));
     return 'success';
   }"

4. 点击嵌套输入框旁边的发送按钮
   browser act click ref: <发送按钮ref>

5. 检查是否需要验证
   browser snapshot
   // 查找验证弹窗

6. 确认发送成功
   // 检查回复是否嵌套显示在用户评论下方
```

### 4. 常见错误

❌ **错误：使用页面顶部输入框**
```
页面顶部有一个通用的评论输入框（placeholder "有爱评论，说点好听的~"）
在这个输入框输入内容并发送，会发成独立的新评论，而不是回复指定评论
```

✅ **正确：使用嵌套输入框**
```
必须点击目标评论的 "回复" 按钮
等待评论下方展开嵌套输入框
在嵌套输入框中输入内容
点击该输入框旁边的发送按钮
```

## 回复原则

对所有用户评论，智能分析后生成回复，遵循以下原则：

| 原则 | 说明 | 示例 |
|------|------|------|
| 积极 | 正面回应，传递正能量 | "欢迎来玩！今天也要开开心心的～ ✨" |
| 大方 | 不卑不亢，得体有度 | "感谢关注，有问题随时问我哦" |
| 热情 | 亲切友好，拉近距离 | "哈喽！看到你的评论超开心～ 💕" |
| 合法 | 遵守法律法规，符合平台规范 | 不涉及敏感话题，不违法违规 |

## 状态监控

维护状态文件记录已处理评论：

```json
{
  "lastCheck": "2026-04-01T16:00:00+08:00",
  "account": "[账号名称]",
  "stats": {
    "totalComments": 10,
    "repliedComments": 5
  },
  "processed": [
    { "type": "comment", "author": "用户名", "time": "15:00", "content": "评论内容", "replied": true }
  ]
}
```

## ⚠️ 平台限制

1. **每条评论只能回复一次** - 抖音平台限制每条评论只能有一条作者回复
2. **回复要一次到位** - 由于只能回复一次，内容要完整有效
