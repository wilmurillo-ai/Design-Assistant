## 执行步骤

### 1. 理解数据流

画清楚数据从输入到输出的全流程：

```
用户输入 → [接口层] → [业务逻辑层] → [数据层] → 用户可见结果
                ↓              ↓           ↓
            错误处理      事务边界     索引/查询
```

回答：
- 数据从哪里来？
- 谁是数据的权威来源？
- 数据被谁消费？

### 2. 识别失败模式

| 失败类型 | 预防措施 |
|---------|---------|
| 单点故障 | 冗余、降级、熔断 |
| 数据不一致 | 事务、幂等、重试 |
| 性能退化 | 索引、缓存、分页 |
| 安全漏洞 | 鉴权、授权、输入验证 |
| 雪崩效应 | 限流、熔断、超时 |

## 代码审查检查清单

被审查代码必须满足：

- [ ] **幂等性**：同样的请求多次执行，结果一致
- [ ] **超时控制**：所有外部调用有超时设置
- [ ] **重试机制**：失败时有指数退避重试（不是无限重试）
- [ ] **事务边界**：跨数据库操作有事务或补偿机制
- [ ] **索引**：数据库查询有相应索引
- [ ] **N+1**：列表查询无N+1问题
- [ ] **输入验证**：所有外部输入被验证
- [ ] **信任边界**：来自客户端的数据都被视为不可信
