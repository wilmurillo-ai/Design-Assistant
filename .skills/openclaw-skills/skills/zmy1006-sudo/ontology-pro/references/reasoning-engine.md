# 推理引擎 Prompt 模板 (Reasoning Engine)

> 本文档定义了推理引擎的 prompt 模板，用于基于知识图谱进行多步推理。

---

## 1. 推理引擎概述

推理引擎是 ontology-pro 的核心，它不满足于"是什么"（检索），而是回答"为什么"和"意味着什么"（推理）。

### 推理层级

| 层级 | 名称 | 说明 | 示例 |
|------|------|------|------|
| L1 | 事实检索 | 从图谱中查询已知信息 | "浙江储能政策有哪些？" |
| L2 | 因果推理 | 识别因果关系链 | "为什么储能成本在下降？" |
| L3 | 变量分析 | 识别关键影响变量 | "影响储能收益的关键因素？" |
| L4 | 路径探索 | 探索多条发展路径 | "储能项目的三种盈利模式？" |
| L5 | 策略推导 | 基于推理生成策略 | "应该优先投资哪种储能技术？" |

## 2. 因果推理 Prompt

### 模板

```
你是一个认知推理引擎。基于以下知识图谱和用户问题，进行因果推理。

【知识图谱】
实体：
{entity_list}

关系：
{relation_list}

【用户问题】
{user_question}

请执行以下推理步骤：

Step 1: 因果链识别
- 找出问题中涉及的核心实体
- 从图谱中追踪因果关系链（A → B → C）
- 标注因果方向和强度

Step 2: 因果类型分类
对每条因果链进行分类：
- 直接因果（direct）：A 直接导致 B
- 间接因果（indirect）：A 通过中介 C 导致 B
- 反馈因果（feedback）：A 导致 B，B 又反过来影响 A
- 隐含因果（implicit）：图谱未直接记录，但可逻辑推导

Step 3: 因果强度评估
- 强因果（≥0.8）：有明确证据，几乎必然
- 中因果（0.5-0.7）：有逻辑支撑，可能发生
- 弱因果（≤0.4）：仅推测，需验证

输出格式：
```json
{
  "core_entities": ["e001", "e003"],
  "causal_chains": [
    {
      "chain": ["e001", "→causes→", "e003", "→leads_to→", "e005"],
      "type": "indirect",
      "strength": 0.7,
      "evidence": "..."
    }
  ],
  "key_insights": [
    "政策补贴是储能市场增长的最强驱动因素（直接因果，强度0.9）",
    "电池成本下降会间接推动储能渗透率提升（间接因果，强度0.7）"
  ]
}
```
```

## 3. 变量分析 Prompt

### 模板

```
你是一个认知推理引擎。请对以下问题进行变量分析。

【知识图谱上下文】
{graph_context}

【分析主题】
{topic}

【用户问题】
{user_question}

请执行以下分析：

1. 变量识别
- 控制变量（controllable）：可以被决策者影响的因素
- 环境变量（environmental）：外部环境因素，不可直接控制
- 中介变量（mediating）：连接因果链中间环节的变量
- 调节变量（moderating）：影响因果关系强度的因素

2. 变量关系网络
- 每个变量与其他变量的关系类型（正相关/负相关/非线性）
- 变量之间的依赖关系

3. 敏感性分析
- 哪些变量变化会对结果产生最大影响（高敏感）
- 哪些变量相对稳定（低敏感）

输出格式：
```json
{
  "variables": [
    {
      "name": "电池成本",
      "type": "environmental",
      "sensitivity": "high",
      "current_value": "下降趋势",
      "trend": "持续下降"
    }
  ],
  "sensitivity_ranking": [
    {"variable": "电池成本", "impact_score": 0.9},
    {"variable": "政策补贴", "impact_score": 0.85}
  ]
}
```
```

## 4. 路径探索 Prompt

### 模板

```
你是一个认知推理引擎。请基于知识图谱进行路径探索。

【知识图谱上下文】
{graph_context}

【起点】
{start_point}

【终点/目标】
{end_goal}

【约束条件】（如有）
{constraints}

请探索从起点到终点的多条可能路径：

1. 路径枚举
- 列出所有可行的路径（至少3条）
- 每条路径标明：经过的节点、需要的步骤、预估时间

2. 路径评估
- 可行性（feasibility）：0-100 分
- 成本（cost）：资金/时间/资源投入
- 收益（benefit）：预期回报
- 风险（risk）：潜在障碍

3. 路径对比
- 各路径的优劣势对比
- 路径之间的切换可能性

输出格式：
```json
{
  "paths": [
    {
      "id": "path_1",
      "name": "技术驱动路线",
      "steps": ["e001", "e005", "e010", "e015"],
      "feasibility": 75,
      "cost": "high",
      "benefit": "high",
      "risk": "medium",
      "pros": ["技术壁垒高，竞争优势持久"],
      "cons": ["研发周期长，前期投入大"]
    }
  ],
  "recommendation": "path_2",
  "reason": "..."
}
```
```

## 5. 综合推理 Prompt（think 命令核心）

### 模板

```
你是一个认知推理引擎（ontology-pro think 模式）。请对以下输入进行完整的认知推理。

【知识图谱】
{graph_summary}

【用户输入】
{user_input}

【推理历史】（如有）
{reasoning_history}

请执行完整的推理流程：

## Phase 1: 问题理解
- 用户真正想解决什么问题？
- 表面问题 vs 深层需求
- 问题涉及的领域和知识范围

## Phase 2: 知识检索
- 从图谱中检索相关实体和关系
- 识别知识缺口（图谱中缺失的关键信息）
- 如有缺口，标注需要补充的信息

## Phase 3: 因果推理
- 核心因果链是什么？
- 正向因果：什么导致了当前局面？
- 反向因果：改变什么可以影响结果？
- 隐含因果：有什么未被发现的关键因素？

## Phase 4: 变量分析
- 关键控制变量有哪些？
- 高敏感变量是什么？
- 变量之间的交互效应？

## Phase 5: 路径探索
- 有哪些可行路径？
- 每条路径的可行性、成本、收益、风险如何？
- 推荐哪条路径？为什么？

## 输出推理链
请以结构化方式输出完整推理过程，包括：
1. 推理链（reasoning chain）：每一步的推理依据
2. 置信度（confidence）：每个结论的可靠程度
3. 知识来源（source）：每个结论基于图谱中的哪些证据
4. 未知项（unknowns）：推理中发现的知识缺口

格式：
```json
{
  "reasoning_chain": [
    {
      "step": 1,
      "type": "fact_retrieval",
      "content": "从图谱检索到：储能成本同比下降15%",
      "confidence": 0.95,
      "sources": ["e010", "r023"]
    },
    {
      "step": 2,
      "type": "causal_inference",
      "content": "成本下降→需求上升，因果关系强度0.7",
      "confidence": 0.7,
      "sources": ["e010", "e015", "r023"]
    }
  ],
  "conclusion": "...",
  "confidence": 0.75,
  "unknowns": ["具体补贴金额待确认", "竞争对手布局未知"],
  "next_action": "需要补充政策细节和竞争情报"
}
```
```

## 6. 推理质量评估

推理完成后，检查以下指标：

| 指标 | 标准 |
|------|------|
| 推理链完整性 | 每个结论都有前置推理步骤，无跳跃 |
| 证据充分性 | 每步推理至少有1个图谱证据支撑 |
| 置信度合理 | 直接事实 ≥ 0.9，因果推断 0.5-0.8，假设 ≤ 0.5 |
| 未知项标注 | 所有不确定的地方都明确标注为 unknown |
| 可追溯性 | 读者能从结论回溯到每一步推理和证据 |

### 常见推理反模式（需避免）

❌ **结论先行**：先给结论再找证据（确认偏误）
❌ **证据不足**：只有1个证据就下强结论
❌ **忽略反例**：只看支持结论的证据
❌ **混淆相关与因果**：A和B同时出现不代表A导致B
❌ **过度外推**：从有限数据推断超出范围的结论
