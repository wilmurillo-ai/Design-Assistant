# 决策输出模板 (Strategy Output)

> 本文档定义了如何将推理结果转化为可执行的决策策略。

---

## 1. 决策输出概述

ontology-pro 的终极目标不是给出分析报告，而是给出**可以直接执行的行动方案**。

### 决策输出三要素

```
推理结果 ──→ 最优路径（做什么） + 风险点（防什么） + 行动建议（怎么做）
```

| 要素 | 说明 | 回答的问题 |
|------|------|-----------|
| 最优路径 | 推荐方案及理由 | 应该做什么？ |
| 风险点 | 潜在障碍和负面因素 | 可能出什么问题？ |
| 行动建议 | 具体的下一步行动 | 怎么开始做？ |

---

## 2. 策略生成 Prompt

### 核心模板

```
你是一个决策策略生成器（ontology-pro strategy 模式）。基于以下推理结果生成可执行策略。

【推理结果】
{reasoning_output}

【用户原始需求】
{original_request}

【约束条件】（如有）
{constraints}

【决策者背景】（如有）
{decision_maker_context}

请生成以下策略内容：

## 1. 策略概览
- 一句话总结推荐策略
- 策略类型：探索型 / 优化型 / 防御型 / 转型型
- 预期时间范围：短期（<3月）/ 中期（3-12月）/ 长期（>1年）

## 2. 最优路径（推荐方案）

- 步骤 1: [具体行动] → 预期成果 → 所需资源
- 步骤 2: [具体行动] → 预期成果 → 所需资源
- ...

### 备选路径
- 方案 B: [描述] → 适用场景 → 与主路径的差异
- 方案 C: [描述] → 适用场景 → 与主路径的差异

## 3. 风险评估

### 高风险（P0）
- [风险描述]
  - 概率：高/中/低
  - 影响：严重/中等/轻微
  - 缓解措施：[具体应对方案]

### 中风险（P1）
- [风险描述]
  - 概率：高/中/低
  - 影响：严重/中等/轻微
  - 缓解措施：[具体应对方案]

### 低风险（P2）
- [风险描述]

## 4. 行动建议（按优先级排序）

### P0 - 立即行动（本周内）
- [ ] [具体行动项]
  - 负责人：[角色]
  - 截止时间：[日期]
  - 成功标准：[可衡量的指标]

### P1 - 近期行动（2周内）
- [ ] [具体行动项]
  - 负责人：[角色]
  - 截止时间：[日期]

### P2 - 计划中（1个月内）
- [ ] [具体行动项]

## 5. 关键指标（KPIs）
- 指标 1: [名称] → 目标值 → 当前值 → 差距
- 指标 2: [名称] → 目标值 → 当前值 → 差距

## 6. 决策依据
- 基于推理链第 X 步的结论：[摘要]
- 基于图谱证据：[实体/关系]
- 置信度：[0-100%]
- 主要假设：[列出关键假设]

输出为结构化 JSON（或根据用户偏好使用 Markdown 格式）。
```

---

## 3. 策略类型定义

| 类型 | 触发条件 | 输出重点 | 示例 |
|------|---------|---------|------|
| **探索型** | 面对新领域、信息不充分 | 信息收集路径、试错策略 | 进入新市场 |
| **优化型** | 现有路径可行但需要改进 | 效率提升、成本优化 | 降低储能项目成本 |
| **防御型** | 面对风险或威胁 | 风险规避、备选方案 | 应对政策变化 |
| **转型型** | 需要根本性改变 | 路径切换、资源重新配置 | 从项目制转产品制 |

## 4. 输出格式模板

### 精简版（日常快速决策）

```markdown
## 🎯 决策建议

**策略**: [一句话]
**类型**: [探索型/优化型/防御型/转型型]

### 最优路径
1. [步骤1]
2. [步骤2]
3. [步骤3]

### ⚠️ 风险提示
- [风险1] - [缓解措施]
- [风险2] - [缓解措施]

### ✅ 行动项
- P0: [立即行动]
- P1: [近期行动]
- P2: [计划中]
```

### 完整版（重大项目决策）

```markdown

**会话ID**: sess_001

> [完整推理链的精简版，每步一行]

1. 事实: 电池成本同比下降 15%（置信度 95%）

### 3. 最优路径

|------|------|---------|---------|------|
| 1 | ... | ... | ... | 1周 |

- 适用场景: [描述]
- 与主路径差异: [描述]

### 4. 风险矩阵

| 风险 | 概率 | 影响 | 优先级 | 缓解措施 |
|------|------|------|--------|---------|

### 5. 行动清单

**P0 立即行动**
- [ ] 行动项1 - 截止: [日期] - 负责人: [角色]

- [ ] 行动项3
```

## 5. 风险评估矩阵

### 概率 × 影响矩阵

```
        影响 →
概率 ↓    轻微     中等     严重
高      [P1]    [P0]    [P0]
中      [P2]    [P1]    [P0]
低      [P2]    [P2]    [P1]
```

### 风险分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 外部风险 | 市场、政策、竞争等不可控因素 | 政策补贴退坡 |
| 内部风险 | 资源、能力、执行等可控因素 | 技术团队不足 |
| 技术风险 | 技术可行性、成熟度 | 新技术不稳定 |
| 时间风险 | 进度延迟、时机错过 | 窗口期有限 |

---

## 6. 决策质量检查

输出策略后，逐项验证：

- [ ] 策略是否直接回答了用户问题
- [ ] 每个行动项是否具体可执行（而非空泛建议）
- [ ] 风险是否有对应的缓解措施
- [ ] 行动优先级是否合理（P0 确实需要立即做）
- [ ] KPI 是否可衡量（有数值目标）
- [ ] 决策依据是否追溯到图谱证据和推理链
- [ ] 是否标注了置信度和关键假设
- [ ] 是否列出了需要补充的信息
