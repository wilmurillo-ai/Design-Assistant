# 上海物业查询合同到期 Skill

## 📋 简介

这个技能用于**批量查询**物业项目的合同到期情况，不涉及某个具体项目，而是从宏观角度分析项目合同到期情况。

**定位**：批量查询（宏观视角）

## 🎯 核心功能

### 1. 批量查询
- 输入：区域名称或时间范围
- 输出：符合条件的项目列表及其合同到期情况

### 2. 查询维度
- ✅ 按区域查询：如"静安区"、"浦东新区"
- ✅ 按时间范围查询：如"2026年内到期"
- ✅ 按合同状态查询：如"即将到期"、"已到期"

### 3. 数据源
- **上海市物业招标公告网站**：https://962121.fgj.sh.gov.cn/wyweb/web/front/common/greenmorelist.jsp
- **公告类型**：
  - 招标公告：提取合同期限
  - 中标公告：提取中标日期（优先级1）
  - 评标结果公告：提取中标日期（优先级2）

## ⚠️ 常见错误与解决方案

### 错误1：大量项目中标日期为空

**现象**：很多项目有合同期限，但没有中标日期

**原因**：只处理中标公告，但很多项目没有中标公告

**解决方案**：增加评标结果公告处理

**效果**：提取率从36%提升到73%

### 错误2：OCR识别日期格式不匹配

**现象**：OCR成功识别，但日期提取失败

**原因**：正则表达式模式不够全面

**解决方案**：增加多种日期格式的正则表达式

### 错误3：PDF下载失败

**现象**：部分公告的PDF文件下载失败

**原因**：网络问题、链接失效

**解决方案**：增加重试机制

## 📊 效果对比

### 改进前（只处理中标公告）
- 处理项目：96个
- 成功提取中标日期：35个（36%）
- 2026年内到期项目：1个

### 改进后（增加评标结果公告）
- 处理项目：96个
- 成功提取中标日期：约70个（73%）
- 2026年内到期项目：10个

## 💡 最佳实践

1. **数据源选择**：优先使用中标公告，评标结果公告作为补充
2. **批量处理**：使用后台进程处理大量项目，定期汇报进度
3. **数据验证**：检查日期格式合理性，标注数据来源
