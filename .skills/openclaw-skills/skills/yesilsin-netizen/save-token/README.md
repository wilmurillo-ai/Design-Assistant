# Save Token Skill

## ⚠️ 重要说明

这是一个**指导性 Skill**，不是执行脚本。

| 是 | 不是 |
|----|------|
| 指导 Agent 如何节省 Token | 自动修改上下文的脚本 |
| 提供策略和最佳实践 | 直接删除重复内容的工具 |
| 帮助 Agent 识别节省机会 | 保证精确节省量的系统 |

   - Emoji: `💰`

## 🎯 Skill 核心功能

### 四大节省策略

1. **摘要代替重发**：摘要早期对话，不重发完整历史
2. **引用代替重复**：引用已读取的信息，不重复读取
3. **去重相同块**：识别并删除重复的文本块
4. **压缩冗长输出**：只保留相关部分，摘要其余内容

### 触发条件

- 用户提到 "token" / "成本" / "省钱"
- 对话超过 10 轮
- 同一文件被多次读取
- 任务前上下文超过 5000 字符

## 📊 预期效果

| 策略 | 适用场景 | 预期节省 |
|------|---------|---------|
| 摘要代替重发 | 长对话 | 30-50% |
| 引用代替重复 | 文件多次读取 | 40-60% |
| 去重相同块 | 重复错误消息 | 20-40% |
| 压缩冗长输出 | 大文件/日志 | 50-70% |

## ⚠️ 注意事项

1. **Agent 必须主动应用策略**：Skill 提供指导，Agent 负责执行
2. **用户应被告知优化结果**：Agent 应报告节省了多少 Token
3. **不影响独特内容**：只优化重复/冗余部分
