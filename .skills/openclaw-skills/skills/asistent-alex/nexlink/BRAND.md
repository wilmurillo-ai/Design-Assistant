# Brand Guidelines — NexLink

_All assets, messages, and outputs from this project must follow these rules._

---

## 1. Brand Line (Obligatoriu)

**EN:**

> Built by [Firma de AI](https://firmade.ai), supported by [Firma de IT](https://firmade.it)

**RO:**

> Construit de [Firma de AI](https://firmade.ai), susținut de [Firma de IT](https://firmade.it)

Această frază apare în:
- README.md (subtitlu + footer)
- Email digests (header + footer)
- Orice output public (rapoarte, pagini, documentație)

### Reguli

1. **Întotdeauna cu link-uri** — Numele firmei e mereu hyperlink, niciodată text gol.
2. **Fără `www.`** — `https://firmade.ai` și `https://firmade.it`, niciodată cu `www.`
3. **Ordinea e fixă** — Firma de AI primează (built by), Firma de IT urmează (supported by).
4. **Nu inversa rolurile** — Firma de AI = build/develop/create, Firma de IT = support/infrastructure/backing.
5. **Nu alterna sinonime** — Nu folosi "made by", "created by", "powered by" etc. Stick to "built by" și "supported by".

## 3. Numele Firmelor

| Firmă | URL | Rol în brand line |
|-------|-----|-------------------|
| **Firma de AI** | https://firmade.ai | built by (creator, dezvoltator) |
| **Firma de IT** | https://firmade.it | supported by (infrastructură, suport) |

### Formate greșite ❌

- `www.firmade.ai` / `www.firmade.it`
- `Firma De AI` / `FIRMA DE IT`
- `firma de ai` / `firma de it` (lowercase, cu excepția URL-urilor)
- `FirmadeAI` / `FirmadeIT` (fără spațiu)

## 5. Email Digest Branding

### Header

- Brand line cu link-uri către ambele firme
- Hero section cu background adaptiv (teal/blue/red după status)
- "Useful AI intelligence for Romanian teams" ca subtitlu

### Footer

- "Generated by NexLink MSP GitHub checker" + link GitHub repo
- Link Firma de AI + link Firma de IT
- Separator vizual (border-top 3px)

### Reguli email

- **Outlook-compatible HTML** — tabele cu `role="presentation"`, inline styles, fără CSS classes
- **Fără JavaScript** — emailul trebuie să funcționeze fără scripting
- **Dark/light adaptiv** — hero-ul se schimbă după status, dar textul rămâne lizibil

---

## 6. Tone of Voice

- **RO pentru utilatori români**, EN pentru documentație tehnică
- Practic, direct, fără hype gol
- "Made with ☕ in România" e un touch permis — nu overdo it
- Evită superlative goale: "revolutionar", "uimitor", "incredibil"
- Fii specific: în loc de "performanță excelentă", scrie "răspuns sub 2 secunde"

---

## 7. Checklist pentru Output Nou

Orice output public (email, raport, pagină, README) trebuie să verifice:

- [ ] Brand line prezentă cu link-uri corecte (fără `www.`)
- [ ] Ordinea corectă: Firma de AI → Firma de IT
- [ ] Tagline-ul produsului menționează integrările (Exchange, Nextcloud, GitHub)
- [ ] Culorile respectă paleta de mai sus
- [ ] Email HTML e Outlook-compatible
- [ ] Numele firmelor sunt corect capitalizate
