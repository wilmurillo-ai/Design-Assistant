# samples/ · 样例文件目录说明

本目录用于存放测试样例, 帮助验证 analyze.py 各功能模块是否正常工作。

---

## 建议的测试结构

```
samples/
├── README.md                   ← 本文件
├── demo_word/
│   ├── 2025年第一批采购公告.docx  ← 含包级表格的 Word 主公告
│   └── 附件_资质要求.xlsx         ← 资质附件 (含"详见附件1"回填场景)
├── demo_pdf/
│   └── 2025年充电桩采购公告.pdf   ← PDF 格式主公告 (测试字号标题提取)
└── demo_nopackage/
    └── 散文件招标公告.docx         ← 无包级表格, 测试正文回退
```

---

## 各测试场景说明

### 场景 1: Word 表格完整抽取
- 文件: `demo_word/` 目录
- 期望: 匹配来源 = `Word表格+Excel`, 资质列由包级字段聚合
- 验证点: 分标编号/包号/资质/概算金额是否正确

### 场景 2: PDF 字号标题提取
- 文件: `demo_pdf/` 目录  
- 期望: 项目名称由 pdfplumber 字号法从封面提取, 而非段落正则
- 验证点: 项目名称是否为封面标题文字

### 场景 3: "详见附件" 资质回填
- 文件: `demo_word/` 目录 (资质列含 "详见附件1")
- 期望: 自动从 `附件_资质要求.xlsx` 按 (分标编号, 包号) 回填
- 验证点: 资质列不再显示 "详见附件1", 改为实际资质内容

### 场景 4: 正文回退 + 手动补盲提示
- 文件: `demo_nopackage/` 目录
- 期望: 匹配来源 = `Word/PDF正文`, 终端提示该项目需手动补盲
- 验证点: 终端末尾是否出现 "⚠️ 以下项目建议手动核查"

### 场景 5: 关键词去冗余
- 文件: 任意含 "咨询服务" 或 "储能系统" 的公告
- 期望: 匹配关键词 = `咨询服务`（不出现 `咨询`）
- 验证点: 输出列 "匹配关键词" 无冗余短词

---

## 如何添加自己的测试样例

1. 将招标公告文件放入对应子目录
2. 运行: `python3 ../scripts/analyze.py`  
   (需先将 `base_dir` 改为 `samples/` 目录路径, 或临时软链接)
3. 检查终端输出与生成的 xlsx 报告

> **注意**: 请勿将含有真实供应商信息的招标文件上传到公开仓库。  
> 如需共享测试样例, 请先脱敏处理。
