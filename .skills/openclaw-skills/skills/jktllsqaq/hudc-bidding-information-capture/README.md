# hbdc · 国网招标文件智能分析工具

> **hbdc** — 半自动招标筛选工具, 专为国家电网（SGCC）招标文件设计。  
> 将你手动下载的招标公告 (Word/PDF) 和附件 (Excel) 自动解析为结构化表格, 筛出符合公司业务方向的可投标项目。

---

## 功能亮点 (v6)

| 功能 | 说明 |
|------|------|
| 🗂️ 多格式解析 | Word (.docx) / PDF / Excel (.xlsx) 三合一 |
| 🔁 PDF 自动重试 | pdfplumber 失败后自动切换参数重试, 明确提示失败原因 |
| 🔠 PDF 字号标题 | 用字符级字号定位封面大字, 项目名称识别更准 |
| 📎 附件资质回填 | 识别 "详见附件X" 占位符, 自动从 xlsx 资质附件补全 |
| 🔑 关键词去冗余 | 咨询服务命中后自动丢弃"咨询"; 储能系统命中后丢弃"储能" |
| 📞 联系人及电话 | 正则 + xlsx 字段双重抓取 |
| 🏛️ 开标时间地点 | 正则抓取开标时间与地点 |
| 💰 投标保证金 | 正则识别金额, 含"不收取"判断 |
| ⚖️ 评标办法 | 综合评估法 / 合理低价法 / 综合评分法 等 |
| 🎨 截止时间染色 | 红(≤3天) / 黄(≤7天) / 灰(已过期) |
| 📝 外部关键词配置 | `config/keywords.json` 随时修改, 无需改脚本 |
| 📊 23列结构化输出 | 含图例说明 Sheet |

## 快速开始

### 1. 安装依赖

```bash
pip3 install python-docx openpyxl pdfplumber --break-system-packages
```

### 2. 准备文件目录

```
~/Desktop/sgcc_files/
├── 项目A/
│   ├── xxx-招标公告.docx      ← 主公告
│   └── 附件_资质要求.xlsx      ← 可选, 资质补充
├── 项目B/
│   └── xxx-采购公告.pdf
└── 散文件_招标公告.docx        ← 也支持根目录散放
```

**文件命名规则:**
- 主公告: 文件名含 `招标公告` / `采购公告` / `招标文件` 优先识别
- 自动忽略: `重要提醒` / `答疑` / `澄清` / `评标办法` / `合同条款` 等

### 3. 运行脚本

```bash
python3 scripts/analyze.py
```

输出报告: `~/Desktop/sgcc_result.xlsx`

## 关键词自定义

编辑 `config/keywords.json`:

```json
{
  "keywords": {
    "新能源汽车充换电类": ["新能源汽车", "充换电", "充电桩"],
    "我的新类别": ["关键词A", "关键词B", "关键词C"]
  },
  "short_keywords": {
    "我的新类别": ["短词"]
  }
}
```

- `keywords`: 精确匹配, 直接命中
- `short_keywords`: 短词匹配, 若对应长词已命中则自动去冗余

## 常见问题

**Q: PDF 文件读取失败怎么办?**  
A: v6 会自动重试一次 (切换 tolerance 参数)。若仍失败会在终端明确提示, 由 AI 手动阅读 PDF 补填字段。

**Q: 资质列显示"详见附件"怎么办?**  
A: v6 会自动识别并尝试从同目录 xlsx 附件中按 (分标编号, 包号) 回填。若 xlsx 附件也没有对应数据, 再降级为项目级通用资质。

**Q: 如何添加公司自己的业务关键词?**  
A: 编辑 `config/keywords.json`, 保存后直接重新运行脚本即可, 不需要修改 Python 代码。

**Q: 匹配来源是"Word/PDF正文"是什么意思?**  
A: 说明该项目的 Word/PDF 表格和 xlsx 附件都没有提取到包级数据, 脚本回退到段落正文扫描。这类项目精度较低, 需要 AI 手动核查。
