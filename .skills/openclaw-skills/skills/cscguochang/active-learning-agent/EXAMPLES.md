## 一、深度追溯示例

### 话题：memory 落地

> 看到群里说"memory 先走 AB 实验"。

只知道一个结论，不知道为什么、怎么来的。

**全源深度追溯（v2.4 做法）**：

```
扫描到群聊提到 memory AB 实验
↓
搜索会议纪要 → 找到 4/2 四方对齐会纪要 → 读取结论和讨论过程
↓
搜索历史 → 3/20 技术评审有 action "定评估标准"，至今未闭环
↓
搜索文档 → Q1 复盘文档中标注"memory 收益确定但上不了线"
↓
回溯群聊 → 3/22 工程侧就提过"先跑 AB 不等完整评估"
↓
检查私聊 → 用户 3/28 和 ZZ 说过"评估标准问题不是技术问题是共识问题"
↓
建立时间线 → 理解完整脉络
```

**基于完整脉络生成的汇总**：

```
memory 落地 timeline（3/15→4/2，跨 4 周）：
▸ 3 月前两周：各方确认收益，卡在评估标准上
▸ 3/20 技术评审定了 action"定评估标准"→ 至今未闭环
▸ 3/22 工程侧提出"先 AB 不等评估"→ 当时未被采纳
▸ 4/1 Q1 复盘标注"上不了线"→ 压力升级
▸ 4/2 四方对齐：最终定了"先 AB"→ 本质是 3/22 方案的延迟采纳
未闭环：AB 实验的成功标准还没对齐
（来源：XX群、产品周会x3、技术评审、四方对齐会纪要、Q1复盘文档）
```

**差异**：这条消息只有在追溯了 4 周跨 6 个信息源之后才能生成。

## 二、防止张冠李戴的实际场景

### 场景 A：区分发言人

**错误**：
```
你在群里提出了"先跑 AB 不等评估"的方案。
```

**实际情况**：这个方案是工程同事 XX 在 3/22 提出的，用户当时没有表态，直到 4/2 四方会才采纳。

**正确**：
```
3/22 XX 提出"先 AB 不等评估"，当时群里没有结论。
4/2 四方对齐会上最终采纳了这个方向（来自四方对齐会纪要）。
```

### 场景 B：区分讨论 vs 定论

**错误**：
```
你决定了油腻治理不继续推。
```

**实际情况**：用户在评审会上说"代价太大不值得"，但这是讨论中的观点，会议没有形成"停止"的正式结论。

**正确**：
```
油腻治理：你在周三评审时提到"代价太大不值得"（来自评审会讨论），
但会议纪要中没有"停止"的结论。群里目前仍在推进方向上讨论。
这两个信号可能需要对齐。
```

### 场景 C：区分不同场景的含义

**错误**：
```
你这周反复强调"可控性"，说明这是当前最核心的关注点。
```

**实际情况**："可控"在需求评审里指"风险可控"，在 OKR 复盘里指"进度可控"。

**正确**：
```
"可控"这周出现了 9 次，但含义不一样：
▸ 需求评审（3次）：指"风险可控"——能不能兜底
▸ OKR 复盘（4次）：指"进度可控"——能不能按时交付
▸ 1:1（2次）：指"预期可控"——结果是否可预测
（分别来自需求初评会、Q1 OKR 复盘、与 XX 的 1:1）
```

## 三、各行动类型 × 全源能力

### 主动分析（跨季度 OKR 对比）

**Agent 执行**：
1. `feishu_search_doc_wiki(query="专家模型 OKR Q3")` → 找到 Q3 OKR 文档
2. `feishu_fetch_doc` → 读取 Q3 目标和结果
3. 重复 Q4、Q1
4. 做跨季度对比

**发送**：
```
专家模型 3 季度 OKR 对比（来源：Q3/Q4/Q1 OKR 文档）：
▸ Q3 目标"上线 v1" → 完成评估未上线 | 阻力：技术可行性
▸ Q4 目标"迭代" → 方向调整 | 阻力：路径选择
▸ Q1 目标"稳定迭代" → 标注不顺利 | 阻力：执行层
阻力在变——不是同一个问题卡了 3 次，是每次解决一个又出新的。
Q2 定目标前可能需要先判断当前阻力属于哪一类。
```

### 预警（跨源信号不一致）

**Agent 发现**：
- 群聊：团队在推进油腻治理
- 评审会纪要：用户说"代价太大"
- 时间：评审会在群聊讨论之后

**发送**：
```
油腻治理信号不一致：
▸ 周三评审你说了"代价太大不值得"（评审会录音 14:23）
▸ 但群里今天还在讨论优化方案（XX群 16:05）
评审结论可能没同步到执行层。
```

### 草案（基于会议纪要起方案）

**Agent 执行**：
1. 读取四方对齐会纪要
2. 读取会议录音中讨论 AB 实验的段落
3. 提取实验相关信息
4. 创建飞书文档

**发送**：
```
基于 4/2 四方对齐会讨论起了 memory AB 实验方案：
[飞书文档链接]
信息来源：四方会纪要 + 会议录音中 14:30-15:10 的讨论段落。
"成功标准"部分空着——纪要和录音里都没有结论。
```

## 四、静默 vs 行动

| 场景 | 决策 | 原因 |
|------|------|------|
| 活跃话题有完整时间线 + 可做分析 | **主动分析** | 有深度信息可交付 |
| 讨论散落 3+ 场景 | **信息汇总** | 帮用户串联 |
| 跨源信号不一致 | **预警** | 可能存在对齐问题 |
| 用户提到"要整理" + 有会议纪要可参考 | **草案** | 能帮用户省时间 |
| 弱信号，信息不够完整 | **静默 + 继续追溯** | 先积累再互动 |
| 信息来源不确定 | **静默** | 宁可不说也不说错 |
