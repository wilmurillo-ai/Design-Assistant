# 场景入口索引

[返回总入口](../../SKILL.md) · [协同地图](../README.md) · [问题分类](../categories/problem-taxonomy.md) · [方法索引](../methods/method-index.md) · [模糊输入识别](../clarification/ambiguity-gate.md)

## 用途说明

- 这不是方法卡，也不是分类体系的替代品。
- 它只解决一件事：当问题已经能大致分清类型后，帮助你快速判断应该先进入哪一类场景入口。
- 如果主领域还拿不准，先回 [problem-taxonomy.md](../categories/problem-taxonomy.md)；如果连问题本身都还没成形，先回 [ambiguity-gate.md](../clarification/ambiguity-gate.md)。

## 标准交接

场景层的标准位置是：

`澄清/重述 -> 分类 -> 场景 -> 方法`

它只决定两件事：

- 从哪个现实入口进入最顺
- 进入后优先看哪 `1` 到 `3` 张方法卡

它不负责：

- 重做澄清
- 替代方法卡正文
- 提前决定输出形式

## 6 类场景入口

1. [work-execution.md](./work-execution.md)
   适合项目推进、任务卡点、资源有限、多人协作下的执行失灵。

2. [learning-growth.md](./learning-growth.md)
   适合学习路线、认知升级、复盘改进、长期积累失焦。

3. [relationship-boundaries.md](./relationship-boundaries.md)
   适合误解、冲突、合作关系、边界设定、谈判沟通。

4. [self-management.md](./self-management.md)
   适合拖延、内耗、状态波动、行动断裂、节奏重建。

5. [life-decisions.md](./life-decisions.md)
   适合重大选择、阶段切换、多个方案比较、时间精力资源取舍。

6. [team-governance.md](./team-governance.md)
   适合带人、统一认识、协同失灵、制度执行、组织节奏与治理问题。

## 最快判断：我先读哪一份

- 你最在意的是“事情为什么推不动、下一步怎么推进”：先读 [work-execution.md](./work-execution.md)。
- 你最在意的是“怎么学、为什么一直学不进去、怎么形成稳定积累”：先读 [learning-growth.md](./learning-growth.md)。
- 你最在意的是“和谁怎么谈、边界怎么画、关系怎么修或怎么止损”：先读 [relationship-boundaries.md](./relationship-boundaries.md)。
- 你最在意的是“我自己为什么总卡住、怎么把状态和行动接起来”：先读 [self-management.md](./self-management.md)。
- 你最在意的是“几个方案怎么比、现在该不该换、该怎么取舍”：先读 [life-decisions.md](./life-decisions.md)。
- 你最在意的是“团队为什么越来越乱、怎么统一方向、怎么把组织拉回正轨”：先读 [team-governance.md](./team-governance.md)。

## 常见跨场景组合

- 工作推进 + 团队治理：事情推不动，不只是执行问题，还涉及分工、角色、组织节奏。
- 关系边界 + 团队治理：表面像沟通冲突，背后其实是角色、权责、联盟结构问题。
- 学习成长 + 自我管理：知道该学什么，但学不进去、练不稳、复盘接不上。
- 生活决策 + 自我管理：不是没有选项，而是迟迟下不了判断、无法承受切换成本。
- 生活决策 + 关系边界：重大选择牵涉伴侣、家人、合伙人或关键合作关系。

如果一个问题同时像两类场景，先选“当前最影响结果”的那类作为主要进入角度，另一类只作为补充视角。

## 每份场景卡通常会告诉你什么

- 这类问题到底适不适合从这里进入。
- 进入该场景后，优先调哪 1 到 3 张方法卡。
- 这类问题通常最该先补什么信息。
- 这类问题常见的输出骨架是什么。
- 这类问题最容易误用的方法和表达边界在哪里。

## 使用提醒

- 场景入口是为了帮助陌生用户更快上手，不是为了替代方法层。
- 一个问题通常只需要 1 个主要进入角度，不要一开始就把 6 类全扫一遍。
- 一旦主入口确定，就尽快交给 `methods/`，不要长期停留在场景层反复描述。
- 如果场景拿不准，回 [confidence-rules.md](../routing/confidence-rules.md) 先做保守判断。
- 如果已经明确命中高风险表达或高风险动作，再进入 [misuse-boundaries.md](../risks/misuse-boundaries.md) 或 [translation-red-lines.md](../risks/translation-red-lines.md)。
