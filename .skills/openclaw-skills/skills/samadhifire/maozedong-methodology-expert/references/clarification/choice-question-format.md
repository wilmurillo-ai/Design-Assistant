# 选项式澄清规范

[返回总入口](../../SKILL.md) · [轮次结构](./round-response-structure.md) · [模糊输入识别](./ambiguity-gate.md) · [澄清流程](./intake-flow.md) · [分场景追问包](./question-packs-by-domain.md)

## 一、文件作用

这个文件只解决一件事：

- 当已经判断“必须先澄清”时，如何把追问写成稳定、可回答、可控的选项题，而不是一上来就开放追问或直接分析。

它不负责：

- 判断是否必须进入澄清
- 替代分场景问题包
- 直接给出正式分析

## 二、核心原则

- `先问清楚` 是硬门槛，不是建议动作。
- 一旦触发这个 skill，就默认按复杂结构题处理。
- 只要输入还不足以支撑不靠大量猜测的正式分析，第一轮就只能先问，不得抢答。
- 澄清题默认写成 `3` 到 `4` 个预设选项，加一个显式 `其他（请补一句）`。
- 用户如果觉得选项都不贴，就必须能走 `其他`，不能被迫硬选。
- 每轮题量和阶段负载，统一按 [intake-flow.md](./intake-flow.md) 执行；本文件只负责单题怎么写，不单独另开题量规则。
- 选项题的目标是压缩不确定性，不是把模型已有结论塞进题干里。
- 第一轮第一题默认先锁 `目标`。
- `目标` 问的是用户想先推进什么结果，不是让用户先给问题分门别类。

## 三、什么时候必须启用

命中下面任一情况，就默认启用选项式澄清：

- 用户只给了现象、感受或结论，没有给最近一次具体事件
- 用户没有说明自己已经试过什么
- 用户没有说明当前最硬的约束、底线或不能承受的代价
- 当前输入看起来像“能分析”，但其实任何判断都还会大量依赖脑补
- 这是第一轮回复，而且问题明显还没过澄清门槛

对这类输入，不要先写“主要矛盾是……”“你现在处在……阶段”“今晚怎么做”。
先把题问清楚，再决定能不能分析。

## 四、标准格式

澄清阶段的选项题，默认放在三段式回复的 `主体结构` 里。
`背景信息` 和 `当前进度` 的固定字段，看 [round-response-structure.md](./round-response-structure.md)。

推荐格式：

```text
背景信息：
- 当前目标：待锁定
- 当前现象：用户给的是现象描述，最近一次关键事件还没落地
- 已确认：当前问题已触发结构化分析
- 仍待确认：目标、最近一次关键事件
- 暂不下判断：暂不进入主要矛盾或阶段判断
- 请主动提供更为详细的背景信息，让毛选拆局.Skill 更好帮您解决问题（很重要很重要！！！）。

主体结构：
这件事我先不急着分析。你先回我下面 1 到 2 题，每题选最接近的一项；如果都不贴，就选“其他（请补一句）”。

1. [目标问题一句话]
A. ...
B. ...
C. ...
D. ...
其他：请补一句

2. [关键事件或关键结构问题一句话]
A. ...
B. ...
C. ...
D. ...
其他：请补一句

当前进度：
- 当前阶段：澄清
- 当前关注：先补目标和最近一次关键事件
- 下一步：补齐后先做问题重述，再判断还缺哪些结构位
```

补充规则：

- 默认单选。
- 只有确实需要多选时，才在题干里显式写 `可多选`。
- 选项数量优先 `4` 个；如果场景更简单，也允许 `3` 个，但仍必须带 `其他`。
- 这一页不再单独规定“每轮最多问几题”；如果拿不准该继续问还是该先收束，回到 [intake-flow.md](./intake-flow.md) 看负载规则。

## 五、出题要求

### 1. 选项要互相区分

- 每个选项都要代表不同路径或不同解释，不要把一句话换个说法写四遍。
- 不要把 `A/B/C/D` 设计成强弱梯度，逼用户选模型更想要的那个。

### 2. 选项要贴场景

- 选项要能对准当前领域，不要用所有场景都能套的空泛句子。
- 自我管理就问行动断点、拖走因素、约束来源。
- 团队治理就问角色、控制点、节奏、规则失灵。

### 3. 选项不能偷跑结论

- 不要把模型的分析结论直接塞进选项，当作用户必须承认的前提。
- 例如不要写：
  - `A. 你其实就是缺自制力`
  - `B. 本质上就是阶段错配`
- 应改成用户能直接辨认的现象或处境。

### 4. `其他` 必须显式出现

- `其他（请补一句）` 不是可选装饰，而是每题必备出口。
- 不能只在开头说一句“如果都不贴可以补充”，后面每题都不写 `其他`。

## 六、首轮回复硬规则

如果当前输入还没过澄清门槛，第一轮回复只能做下面这些事：

1. 先按三段结构写回复，保持 `背景信息 / 主体结构 / 当前进度` 都在。
2. 用一句自然的话说明“我先不急着分析，先把关键位置问清楚”。
3. 提 `1` 到 `2` 个选项题。
4. 如果用户已经提到 `HTML`，顺手补一句“现在信息还不够生成 HTML”。

这时不得出现：

- 正式结构判断
- 主要矛盾判断
- 阶段判断
- 成因归因
- 行动方案
- 作息表
- 路线建议
- HTML 交付确认

## 七、推荐追问顺序

默认优先顺序仍然是：

1. 目标
2. 最近一次关键事件
3. 关键人物/对象与位置变化
4. 控制点/权利资源/可团结对象
5. 已做尝试
6. 约束/底线

但落到具体提问时，不要求机械按这个顺序全问完。
每轮只挑当前最缺、最能减少脑补的 `1` 到 `2` 题。

## 八、自我管理示例

下面示例主要展开 `主体结构` 的提问部分；`背景信息` 和 `当前进度` 仍按 [round-response-structure.md](./round-response-structure.md) 的默认骨架补齐。

用户输入：

```text
我最近总拖延。白天其实知道有几件重要的事必须推进，比如工作汇报和一个考试准备，但一到晚上就觉得很累，只想刷手机放空，结果越刷越晚，第二天又会自责，白天效率更差。我现在有点分不清自己到底是单纯懒、太累了，还是在逃避。
```

合格首轮：

```text
这件事我先不急着分析，先把关键断点问清楚。你先回我两题，每题选最接近的一项；如果都不贴，就选“其他（请补一句）”。

1. 你这次最想先拉回哪一段？
A. 晚上别失控
B. 第二天别自责
C. 早上能顺利启动
D. 整体作息拉回正轨
其他：请补一句

2. 最近一次最典型的失守发生在什么情境？
A. 白天太累，晚上只想放空
B. 一想到具体任务就想躲
C. 躺下后顺手刷手机停不下来
D. 情绪不好，拿手机缓冲
其他：请补一句
```

不合格首轮：

- 直接分析“你的主要矛盾是……”
- 直接给“今晚怎么做”
- 直接下“你现在处于恢复秩序阶段”

## 九、团队治理示例

下面示例同样只展开 `主体结构`；真正落地时，仍要带上 `背景信息` 和 `当前进度` 两段。

用户输入：

```text
用毛选帮我分析，我这个团队越来越乱了，我感觉规则都失效了。
```

合格首轮：

```text
这件事我先不急着分析，先把关键位置问清楚。你先回我两题，每题选最接近的一项；如果都不贴，就选“其他（请补一句）”。

1. 你这次最想先推进什么结果？
A. 先判断真正的主卡点在哪
B. 先把节奏和协同稳住
C. 先把规则和边界重新立起来
D. 先判断该不该调整人或位置
其他：请补一句

2. 最近一次最能说明“团队乱了”的具体事件，更像哪一种？
A. 会上说清了，会后没人接
B. 同一件事多头指挥
C. 规则还在，但现场已经不按规则走
D. 表面协同，实际互相绕开
其他：请补一句
```

不合格首轮：

- 直接说“旧的统一已经破了，新的统一还没立起来”
- 直接说“真正的乱不是意见多，而是权力、信息、责任分开了”
- 直接给“你现在最该做的，只有 4 步”

## 十、禁止事项

- 不要把选项题写成审讯式长问卷。
- 不要一边问选项，一边顺手塞进大段分析。
- 不要把 `其他` 省掉。
- 不要把 `A/B/C/D` 写成价值判断或道德标签。
- 不要在用户还没回答前，就把选项当成已确认事实。
- 不要把微信措辞、聊天记录润色、强硬版/温和版当成第一轮默认方向。
