# “中间力量 / 顽固势力 / 先团结后改造”使用护栏

[返回总入口](../../SKILL.md) · [风险路由](../routing/confidence-rules.md) · [误用边界入口](./misuse-boundaries.md) · [表达红线入口](./translation-red-lines.md)

## 这组表达什么时候可用

- 用来处理对象差异很大、不能一刀切对待的多方局势。
- 用来说明为什么有些对象要留空间争取，有些对象必须设边界，而合作入口又不能等到完全同质才建立。

## 正确含义

- `中间力量` 指可争取、可摇摆、需留空间的对象，不是天然中立者。
- `顽固势力` 指在特定阶段持续破坏共同任务的力量，不是简单的“我不喜欢的人”。
- `先团结后改造` 指先建立合作入口，再在合作中逐步校准和提升，不是无底线妥协，也不是先做纯化审查。

## 禁止翻译

- 不能翻成“中间力量就是谁都不得罪的人”。
- 不能翻成“凡是不完全支持我的，都是顽固势力”。
- 不能翻成“既然先团结，就先别谈条件和边界”。
- 不能翻成“必须先完全改造成自己人，才允许进入合作”。

## 运行期检查

- 使用 `中间力量` 时，必须同时写出它为什么值得争取、它会在什么条件下迁移。
- 使用 `顽固势力` 时，必须同时给出持续破坏共同任务的行为证据，而不是情绪判断。
- 使用 `先团结后改造` 时，必须同时写出合作入口、边界条件和后续校准路径。
- 输出里要保留动态复判，防止把分层做成永久贴标签。
