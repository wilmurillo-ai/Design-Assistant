# “整风 / 开放批评 / 治病救人”使用护栏

[返回总入口](../../SKILL.md) · [风险路由](../routing/confidence-rules.md) · [误用边界入口](./misuse-boundaries.md) · [表达红线入口](./translation-red-lines.md)

## 这组表达什么时候可用

- 用来处理组织内部已经出现偏差、需要纠偏又不能把队伍打散的场景。
- 用来说明为什么批评要进入系统，但目标是修复能力和统一认识，而不是清洗或泄愤。

## 正确含义

- `整风` 或 `整顿` 指同时改认知、改组织、改作风、改表达，不是运动式清洗。
- `开放批评` 指让正确意见进入系统，不是放任攻击和情绪宣泄。
- `治病救人` 指原则性纠偏加组织修复，不是和稀泥，也不是只求气氛团结。

## 禁止翻译

- 不能翻成“批评越狠，纠偏就越彻底”。
- 不能翻成“为了团结，先别碰真实问题”。
- 不能翻成“只要外部批评进来，就必须全盘照收”。
- 不能翻成“整顿就是换人和追责”。

## 运行期检查

- 使用 `开放批评` 时，必须同时说明哪些意见是基于事实、对共同目标有帮助的。
- 使用 `整风` 或 `整顿` 时，必须同时写出要修的路线、机制或作风，而不是只写人。
- 使用 `治病救人` 时，必须同时写出原则边界和组织修复动作，防止滑向和稀泥。
- 输出既要防清洗化，也要防团结化空转。
