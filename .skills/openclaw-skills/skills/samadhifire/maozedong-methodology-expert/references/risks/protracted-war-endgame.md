# “持久战 / 近胜期”使用护栏

[返回总入口](../../SKILL.md) · [风险路由](../routing/confidence-rules.md) · [误用边界入口](./misuse-boundaries.md) · [表达红线入口](./translation-red-lines.md)

## 这组表达什么时候可用

- 用来处理长期不对称竞争、阶段递进和窗口期收紧的局面。
- 用来说明为什么短期不能躁进，但也不能把长期理解成原地等待。

## 正确含义

- `持久战` 是分阶段长期竞争设计，不是拖延主义。
- `最后阶段` 或 `近胜期` 是窗口变窄、节奏收紧、判断门槛提高，不是自动胜利。
- 阶段越接近转段或收束，越需要重估时间表、资源节奏和组织换挡。

## 禁止翻译

- 不能翻成“既然是持久战，就慢慢来就行”。
- 不能翻成“既然进入最后阶段，结果就会自己到来”。
- 不能翻成“只要士气高，就可以宣布提前收官”。

## 运行期检查

- 使用 `持久战` 时，必须同时写出当前所处阶段和下一阶段的转化条件。
- 使用 `近胜期` 时，必须同时写出窗口为什么变窄、哪些标准更严格、哪些动作要收紧。
- 输出里既要防盲目加速，也要防无限拖延。
