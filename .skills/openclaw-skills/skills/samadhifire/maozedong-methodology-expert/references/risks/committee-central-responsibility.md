# “党委制 / 一元化负责”使用护栏

[返回总入口](../../SKILL.md) · [风险路由](../routing/confidence-rules.md) · [误用边界入口](./misuse-boundaries.md) · [表达红线入口](./translation-red-lines.md)

## 这组表达什么时候可用

- 用来处理复杂协同里重大问题需要上桌、责任需要收口的场景。
- 用来说明为什么既要集体决策，又要避免人人负责等于没人负责。

## 正确含义

- `党委制` 指重大问题上桌、会前准备、集体讨论决定，不是凡事会议化。
- `一元化负责` 指主统筹清晰、责任收口明确，不是什么都一个人拍板。
- 这组判断的目标是让班子协同和责任链同时成立，而不是制造开会感或强人依赖。

## 禁止翻译

- 不能翻成“只要开过会，就算集体领导”。
- 不能翻成“既然要一元化负责，别人就不必参与判断”。
- 不能翻成“凡是重要的事，都必须把所有人拉进同一张会”。

## 运行期检查

- 使用 `党委制` 时，必须同时写出哪些问题必须上桌、会前准备怎么做、会后责任怎么落。
- 使用 `一元化负责` 时，必须同时写出主统筹是谁、边界在哪里、哪些事项仍需集体判断。
- 输出既要防集体甩责，也要防个人包办一切。
