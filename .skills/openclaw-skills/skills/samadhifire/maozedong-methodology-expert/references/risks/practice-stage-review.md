# “实践检验 / 阶段复盘”使用护栏

[返回总入口](../../SKILL.md) · [风险路由](../routing/confidence-rules.md) · [误用边界入口](./misuse-boundaries.md) · [表达红线入口](./translation-red-lines.md)

## 这组表达什么时候可用

- 用来处理一轮行动已经完成，结果需要回流成再判断的场景。
- 用来说明为什么复盘不是总结情绪，而是重检原判断、原标准和下一轮入口。

## 正确含义

- `实践检验` 指带着判断回到现实验证，再把结果送回判断系统，不是结果论和经验崇拜。
- `阶段复盘` 指重看阶段位置、判断标准和转段条件，不是复述阶段名称。
- 这组判断的目标是让结果改变下一轮方法，而不是只留下战报或感想。

## 禁止翻译

- 不能翻成“做过就天然懂”。
- 不能翻成“结果好就说明原判断都对，结果差就说明人不行”。
- 不能翻成“复盘就是把这一阶段名称再讲一遍”。

## 运行期检查

- 使用 `实践检验` 时，必须同时写出上一轮原判断是什么、这轮结果验证了什么、推翻了什么。
- 使用 `阶段复盘` 时，必须同时写出哪些阶段标准保留、哪些失效、哪些需要重写。
- 输出要能落到下一轮调查、诊断或执行入口，而不是停在总结文本上。
