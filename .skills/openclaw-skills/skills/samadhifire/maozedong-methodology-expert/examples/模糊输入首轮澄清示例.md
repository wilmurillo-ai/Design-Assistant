# 模糊输入首轮澄清示例

这个示例只演示一件事：

- 当用户输入还很模糊时，首轮回复应该先问清楚，而不是直接分析。

## 示例输入

```text
用毛选的方法帮我分析：我最近总拖延。白天其实知道有几件重要的事必须推进，比如工作汇报和一个考试准备，但一到晚上就觉得很累，只想刷手机放空，结果越刷越晚，第二天又会自责，白天效率更差。我现在有点分不清自己到底是单纯懒、太累了，还是在逃避。
```

## 合格的首轮回复

```text
这件事我先不急着分析，先把关键断点问清楚。你先回我两题，每题选最接近的一项；如果都不贴，就选“其他（请补一句）”。

1. 你这次最想先拉回哪一段？
A. 晚上别失控
B. 第二天别自责
C. 早上能顺利启动
D. 整体作息拉回正轨
其他：请补一句

2. 最近一次最典型的失守发生在什么情境？
A. 白天太累，晚上只想放空
B. 一想到具体任务就想躲
C. 躺下后顺手刷手机停不下来
D. 情绪不好，拿手机缓冲
其他：请补一句
```

## 不合格的首轮回复

下面这些都不合格：

- 直接说“你的主要矛盾是……”
- 直接说“你现在在恢复秩序阶段”
- 直接给“今晚怎么做”“未来七天怎么做”
- 没有选项，只有开放追问
- 题目里没有显式 `其他`
