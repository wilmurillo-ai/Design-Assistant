# A-G 评估矩阵

## 评级标准

| 等级 | 标签 | 分数区间 | 匹配程度 | 建议操作 |
|------|------|----------|----------|----------|
| **A** | 强匹配 | 90-100 | 所有核心维度匹配 | 立即投递，生成定制简历，启动面试准备 |
| **B+** | 良好匹配 | 80-89 | 高度匹配，次要维度有差距 | 优先投递，使用标准简历 |
| **B** | 中等偏上 | 70-79 | 存在匹配但差距可克服 | 建议投递，面试时注意弥补差距 |
| **C** | 中等 | 60-69 | 部分匹配，差距较大 | 观察，无更好选择时投递 |
| **D** | 低匹配 | 40-59 | 最小限度对齐 | 跳过 |
| **E-F-G** | 不匹配 | 0-39 | 几乎/完全无关 | 自动剔除 |

## 五个评估维度（匹配度）

每个维度独立打分 0-20，满分 100。

### 维度 1：角色匹配度 (Role Match) — 20 分

衡量岗位与候选人目标方向的匹配程度。

| 分数 | 标准 |
|------|------|
| 18-20 | 岗位与主要目标方向完全一致 |
| 14-17 | 岗位是目标方向的密切相邻角色 |
| 10-13 | 与目标方向有交叉但有显著差异 |
| 5-9 | 同领域但不同职能 |
| 0-4 | 与目标方向无有意义重叠 |

**示例**：
- 目标方向是「产品经理」：
  - 「AI产品经理」「B端产品经理」→ 18-20
  - 「产品运营」「商业分析师」→ 14-17
  - 「项目管理」「技术支持」→ 5-9

### 维度 2：技能匹配度 (Skill Match) — 20 分

衡量候选人技能对岗位要求的覆盖程度。

| 分数 | 标准 |
|------|------|
| 18-20 | 覆盖所有硬性要求 + 额外加分技能 |
| 14-17 | 满足所有核心要求，最多缺 1-2 个加分项 |
| 10-13 | 满足大部分要求，有 2-3 个显著缺口 |
| 5-9 | 满足不到一半的要求 |
| 0-4 | 几乎不满足任何要求 |

**评估方法**：
- 列出所有岗位要求
- 逐条对照 cv.md 中的技能
- 按要求重要性加权（必须 vs 加分）

**不同岗位方向的技能评估重点**：
- **产品经理**：需求分析、数据能力、原型设计、项目管理、行业理解
- **运营**：数据分析、内容能力、活动策划、用户运营、增长思维
- **市场**：品牌策划、传播策略、渠道运营、竞品分析、文案能力
- **技术/数据**：编程能力、算法、数据处理、系统设计、技术深度
- **设计**：设计工具、作品集质量、用户研究、设计思维、审美能力

### 维度 3：地点适配度 (Location Fit) — 20 分

| 分数 | 标准 |
|------|------|
| 18-20 | 位于候选人首选城市 |
| 14-17 | 位于候选人可接受城市列表（第 2-3 选择） |
| 10-13 | 同区域的一线城市 |
| 5-9 | 需要搬迁到不便的位置 |
| 0-4 | 完全不可接受的地点（跨国家/地区） |

**注**：远程岗位默认 14-17。

### 维度 4：经验契合度 (Experience Alignment) — 20 分

| 分数 | 标准 |
|------|------|
| 18-20 | 有直接相关经验且已证明影响力 |
| 14-17 | 有相关经验，可能在层级或领域上略有差异 |
| 10-13 | 有可迁移经验但缺乏直接领域相关性 |
| 5-9 | 经验水平显著低于要求 |
| 0-4 | 要求的经验候选人完全不具备 |

**补充因素**：
- 实习岗位：评估项目/实习质量，而非年限
- 可量化的成果（数据、用户影响、营收）加分
- 岗位明确要求候选人明显缺乏的经验则扣分
- 相关的课外/竞赛/自学经验可部分弥补正式经验不足

### 维度 5：行业相关性 (Industry Relevance) — 20 分

| 分数 | 标准 |
|------|------|
| 18-20 | 公司在候选人首要目标行业且市场地位强 |
| 14-17 | 在相关/相邻行业，增长潜力好 |
| 10-13 | 不同行业但有价值的跨领域经验 |
| 5-9 | 行业与候选人目标关联度低 |
| 0-4 | 行业完全不对口 |

**评估要点**：
- 同时考虑公司和行业
- 强公司在非目标行业 > 弱公司在目标行业
- 增长期行业（AI、新能源、出海等）可小幅加分

## 竞争度评估（独立维度）

与 A-G 评分并行，单独评估。

| 星级 | 竞争程度 | 判断依据 |
|------|----------|----------|
| ★★★★★ | 极激烈 | 大厂热门岗 + 海量投递 + 大量面经讨论 |
| ★★★★ | 激烈 | 知名公司 + 高讨论度 |
| ★★★ | 中等 | 中型公司 + 正常投递量 |
| ★★ | 较低 | 小公司 / 新业务线 / 急招 |
| ★ | 很低 | 刚发布 / 小众方向 / 冷门公司 |

**判断方法**：
- Web 搜索「{公司} {岗位} 面经」看讨论数量
- 搜索「{公司} {岗位} 投递量/竞争」
- 牛客网上该岗位的讨论帖数量
- BOSS直聘上该岗位的求职者数量

## 综合推荐指数

```
推荐指数 = 匹配度分值 × 竞争度权重

竞争度权重：★★★★★=0.6 | ★★★★=0.75 | ★★★=0.85 | ★★=0.95 | ★=1.0
```

**用途**：推荐指数用于排序投递优先级。A 级 + 极激烈竞争（96×0.6=57.6）的实际优先级可能低于 B+ 级 + 低竞争（85×0.95=80.75）。

**但注意**：推荐指数是辅助决策工具，不是硬规则。高匹配的高竞争岗位仍然值得投递——只是要做好充分准备。

## 自动剔除条件

满足以下任一条件，无需评分直接剔除：

1. **已过期**：申请截止日期已过
2. **层级错误**：高级/资深/总监级别（非实习/初级/校招）
3. **地区错误**：要求在不可接受地区现场办公
4. **重复**：已存在于 `data/scan-history.tsv`
5. **虚假/低质**：描述模糊/泛化，无具体要求，疑似培训机构
6. **毕业年份不符**：明确要求不同届次
7. **海笔海面**：通过 web 搜索确认该公司有大量面试但不发 offer 的口碑
8. **薪资严重偏离**：薪资明显低于市场合理范围（可能是坑）

## 评分示例

```
岗位：阿里巴巴 — AI应用产品经理（2027届实习生）

匹配度评估：
角色匹配：  20/20  （与首选目标岗位完全一致）
技能匹配：  19/20  （所有要求满足，算法笔试略有差距）
地点适配：  18/20  （杭州 = 首选城市）
经验契合：  19/20  （直接相关产品实习 + 独立项目）
行业相关：  20/20  （阿里巴巴 + AI = 首选行业）
───────────────────────────────────
匹配度总分：96/100  → A 级

竞争度评估：★★★★★（大厂热门岗位，大量面经讨论）

推荐指数：96 × 0.6 = 57.6（竞争激烈，但匹配度极高，仍应全力冲击）
```

## 报告格式

每次评估生成 `reports/{YYYY-MM-DD}_A-G_evaluation.md`，包含：

1. 候选人信息和评估日期
2. 评估矩阵说明
3. 按推荐指数排序的岗位列表（高→低）
4. 每个岗位的维度级评分、竞争度、理由
5. 投递优先级汇总（P0 → P1 → P2）
6. 剔除岗位及原因
