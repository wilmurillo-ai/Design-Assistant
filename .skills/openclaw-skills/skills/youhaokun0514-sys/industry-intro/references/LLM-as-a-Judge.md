## 质量评估概述

### 评估目的
确保生成的定义报告符合行业标准、数据可溯源、格式统一。

### 四维度评分体系
| 维度 | 分值 | 评估重点 |
|------|------|---------|
| 有根据性 | 40分 | 所有关键陈述是否有明确来源，来源是否可信 |
| 相关性 | 20分 | 内容是否紧密围绕种子词 |
| 结构合规性 | 20分 | 是否严格遵循四维结构 |
| 可解释性 | 20分 | 表述是否清晰，无歧义 |

### 质量阈值
- **及格线**：总分≥85分
- **优秀线**：总分≥90分
- **修正要求**：总分<85分必须返回阶段4或阶段5进行修正

## 维度1：有根据性（满分40分）

### 评分细则

#### 1.1 溯源标注完整性（20分）

**关键论点的界定**：指以下三类内容必须有来源标注：

1. **技术参数**：如CAS号、纯度、尺寸、熔点、沸点、电压、转化率、视场角、亮度、分辨率、刷新率等理化参数和性能参数
2. **法规标准引用**：如GB/T编号、监管定义、行业标准、药典版本、专利号等
3. **行业共识性结论**：如"主流技术路线是XX"、"行业普遍认为"、"核心技术为XX"等结论性陈述

**评分标准**：
- **20分**：≥5处关键论点有溯源标注，覆盖所有三类关键内容
- **15分**：3-4处关键论点有溯源标注，覆盖至少两类关键内容
- **10分**：1-2处关键论点有溯源标注，覆盖至少一类关键内容
- **5分**：0-1处关键论点有溯源标注
- **0分**：无溯源标注或标注不足

#### 1.2 数据来源可信度（10分）

**可信度层级定义**：

| 层级 | 分值 | 信源类型 | 示例 |
|------|------|---------|------|
| 权威信源 | 10分 | 国家标准/法规、招股书、头部券商研报 | GB/T 18384-2020、专利CN112345678、中金公司研报 |
| 中等信源 | 7分 | 行业协会白皮书、行业媒体 | 汽车工业协会白皮书2023、医药工业信息中心 |
| 一般信源 | 4分 | 普通网页、自媒体 | 企业官网、行业博客 |
| 无标注 | 0分 | 无溯源标注 | - |

**评分标准**：
- **10分**：所有关键论点的来源为权威信源（国家标准/法规、招股书、头部券商研报）
- **8分**：大部分（≥80%）关键论点的来源为权威信源，其余为中等信源
- **7分**：关键论点的来源为中等信源（行业协会白皮书、行业媒体）
- **4分**：关键论点的来源为一般信源（普通网页、自媒体）
- **0分**：无溯源标注

#### 1.3 模糊词汇控制（10分）

**模糊词汇清单**：也许、大概、可能、或许、估计、应该、差不多、约、左右、大约

**评分标准**：
- **10分**：无模糊词汇
- **7分**：≤2处模糊词汇
- **4分**：3-5处模糊词汇
- **0分**：>5处模糊词汇

### 常见问题与修正建议

**问题1**：溯源标注不足或关键论点未标注
- **修正指令**：补充三类关键内容的来源标注（技术参数、法规标准、行业共识）
- **示例**："HUD的视场角为10度×5度" → "HUD的视场角为10度×5度。[来源：专利CN112345678]"
- **示例**："阿司匹林的CAS号为50-78-2" → "阿司匹林的CAS号为50-78-2。[来源：中国药典2020版]"
- **示例**："主流技术路线是激光投影" → "主流技术路线是激光投影。[来源：汽车工业协会白皮书2023]"

**问题2**：数据来源可信度不足
- **修正指令**：提升来源可信度层级，优先使用权威信源（国家标准、专利、头部券商研报）
- **示例**：将来源从"企业官网"（一般信源，4分）替换为"国家标准GB/T 18384-2020"（权威信源，10分）

**问题3**：使用模糊词汇
- **修正指令**：删除所有模糊表述，替换为确定性表述
- **示例**："纯度约为99.5%" → "纯度≥99.5%"
- **示例**："温度可能达到135℃" → "温度为135℃"
- **示例**："亮度大概是8000尼特" → "亮度为8000尼特"

## 维度2：相关性（满分20分）

### 评分细则

#### 2.1 种子词出现频率（10分）
- **10分**：种子词出现≥5次
- **7分**：种子词出现3-4次
- **4分**：种子词出现1-2次
- **0分**：种子词未出现

#### 2.2 主题偏离度（10分）
- **10分**：无偏离主题的段落
- **7分**：1个段落可能偏离主题
- **4分**：2-3个段落可能偏离主题
- **0分**：>3个段落偏离主题

### 常见问题与修正建议

**问题1**：种子词出现频率过低
- **修正指令**：增加核心关键词，确保种子词在报告中出现≥5次
- **示例**：在描述技术原理、应用场景时重复使用种子词

**问题2**：内容偏离主题
- **修正指令**：删除与种子词无关的内容，聚焦核心定义
- **示例**：如果报告包含大量竞争对手分析，但与定义无关，应删除或简化

## 维度3：结构合规性（满分20分）

### 评分细则

#### 3.1 四维结构完整性（12分）
- **12分**：四维结构完整（核心本体界定、技术原理与物理化学属性、病理/功能与下游应用场景、边界划定与统计口径）
- **8分**：缺少1个维度
- **4分**：缺少2个维度
- **0分**：缺少≥3个维度

#### 3.2 标题格式规范（4分）
- **4分**：标题层级完整（##和###）
- **2分**：标题层级不完整
- **0分**：缺少标题结构

#### 3.3 输出格式规范（4分）
- **4分**：包含所有格式元素（分类层级、统计口径、包含、不包含）
- **3分**：≥3个格式元素
- **1分**：1-2个格式元素
- **0分**：无格式元素

### 常见问题与修正建议

**问题1**：四维结构不完整
- **修正指令**：补充缺失的维度内容
- **示例**：如果缺少"边界与口径"部分，必须补充"包含"、"不包含"和"市场规模计算方式"

**问题2**：标题格式不规范
- **修正指令**：使用标准Markdown格式（##和###）
- **示例**：将"1. 核心本体界定"改为"### 1. 核心本体界定"

## 维度4：可解释性（满分20分）

### 评分细则

#### 4.1 语句清晰度（8分）
- **8分**：无过长句子（≤200字符）
- **6分**：1-2个过长句子
- **4分**：3-5个过长句子
- **2分**：>5个过长句子

#### 4.2 术语解释充分性（6分）
- **6分**：专业术语数量≤5个
- **3分**：专业术语数量>5个
- **0分**：专业术语数量>10个且无解释

#### 4.3 逻辑连贯性（6分）
- **6分**：≥3个逻辑连接词（"因此"、"然而"、"此外"等）
- **4分**：1-2个逻辑连接词
- **2分**：0个逻辑连接词
- **0分**：逻辑混乱

### 常见问题与修正建议

**问题1**：句子过长
- **修正指令**：拆分过长句子，提高可读性
- **示例**：将"HUD是一种通过将关键驾驶信息投影到驾驶员视野前方从而提高驾驶安全性的车载光学显示装置"拆分为两句

**问题2**：逻辑连接不足
- **修正指令**：增加过渡词，提高连贯性
- **示例**：在段落之间添加"此外"、"然而"等连接词

## 质量阈值与修正

### 修正流程
1. **评分**：调用`quality_checker.py`进行四维度评分
2. **判断**：如果总分<85分，生成修正指令
3. **修正**：根据修正指令返回阶段4或阶段5进行修改
4. **重评**：修正后重新评分，直到总分≥85分

### 修正指令示例

**指令1（溯源标注）**："第二段缺少技术参数溯源，请补充专利号（权威信源）"
**指令2（模糊词汇）**："第四段存在2处模糊词汇，请删除'可能'和'大概'"
**指令3（结构完整性）**："缺少'边界与口径'部分，必须补充包含、不包含和市场规模计算方式"
**指令4（相关性）**："种子词出现频率过低（仅2次），建议增加至≥5次"
**指令5（来源可信度）**："数据来源为一般信源（企业官网），建议替换为权威信源（国家标准或专利）"

### 迭代限制
- 最多允许3次修正
- 3次修正后仍未达标，报告错误信息并终止

```

- 溯源标注完整（20/20）- 覆盖所有三类关键内容
- 数据来源可信度高（10/10）- 全部为权威信源（国家标准、专利）
- 无模糊词汇（10/10）

相关性：19/20
- 种子词出现6次（10/10）
- 无主题偏离（9/10）

结构合规性：19/20
- 四维结构完整（12/12）
- 标题格式规范（4/4）
- 输出格式规范（3/4）

可解释性：19/20
- 语句清晰度高（8/8）
- 术语解释充分（6/6）
- 逻辑连贯性好（5/6）

总分：96/100，通过
```

### 示例2：需要修正的报告（总分68分）
```
有根据性：23/40
- 源标注不足（10/20）- 仅覆盖两类关键内容
- 数据来源可信度低（7/10）- 主要为中等信源
- 存在模糊词汇（6/10）

相关性：15/20
- 种子词出现3次（7/10）
- 存在1个偏离段落（8/10）

结构合规性：18/20
- 四维结构完整（12/12）
- 标题格式规范（4/4）
- 输出格式规范（2/4）

可解释性：12/20
- 存在3个过长句子（4/8）
- 术语较多（3/6）
- 逻辑连接词少（5/6）

总分：68/100，不通过

修正指令：
1. 补充技术参数和法规标准的来源标注（+8分）
2. 将中等信源替换为权威信源（国家标准、专利）（+3分）
3. 删除"可能"和"大概"等模糊词汇（+4分）
4. 拆分3个过长句子（+4分）
5. 增加"此外"、"因此"等逻辑连接词（+1分）
```
