# 群组定义参考

## group_info 字段

```javascript
const groupInfo = {
  派id: "123456789",       // 数字字符串
  群名: "AI资讯交流群",
  bot_config: null,
};
```

## active_members 字段

每个成员需要：

```javascript
{
  id: "user_001",          // 唯一ID
  role: "user",            // "user" 或 "assistant"
  nickname: "陈默",         // 真实微信风格昵称（不是社交ID）
  persona: "32岁产品经理，...", // 详细角色描述，供 subagent 扮演
  join_time: 46090.333,    // Excel时间序列数
}
```

## AI助手成员示例

```javascript
{
  id: "bot_001",
  role: "assistant",
  nickname: "元宝",
  persona: "群内AI助手，回答群友问题，提供信息检索和总结",
  join_time: toExcelSerial(2026, 1, 1, 0, 0, 0),
}
```

## 昵称风格指南

✅ 好的昵称（真实微信风格）：
- 陈默、张磊、林晓、王斌、孙阳
- 周宇老师、毛毛、程凡、刘青、胡建
- 老陈、小林、张工、Leo张

❌ 避免的风格：
- chenmo_2025、zhang_lei_official
- CoolDesigner、TechGuru99

## 话题设计建议

- 每个话题 40–60 条消息
- 话题1：行业新闻/热点（引发广泛讨论）
- 话题2：工具/技术对比（专业角色主导）
- 话题3：职业/社会议题（情绪性强，引发共鸣）
- 确保每个话题都有：发起者、技术派、小白提问、AI助手被@

## 角色多样性建议

覆盖这些角色类型有助于生成更真实的群聊：

| 角色类型 | 特征 | 示例职业 |
|---------|------|---------|
| 技术专家 | 严谨、纠错、引数据 | 算法工程师、教授 |
| 产品/运营 | 实用主义、爱总结 | PM、运营专员 |
| 创业者 | 讲实战、踩过坑 | CEO、创始人 |
| 媒体人 | 信息面广、写导语 | 科技记者 |
| 学生/新人 | 好奇、提naive问题 | 在校生 |
| 设计师 | 感性、关注视觉 | UI设计、创意 |
| 投资人 | 看战略、看数字 | VC、基金 |
