## 🌐 agent-browser 简介

**agent-browser** 是一个基于 Rust 的快速无头浏览器自动化 CLI 工具，支持：

- ✅ 页面导航、点击、输入、快照
- ✅ AI 代理友好的结构化命令
- ✅ 无障碍树快照 + 引用式元素选择
- ✅ 网页测试、表单填写、数据提取
- ✅ 截图、录屏、PDF 生成

## 🚀 快速开始

### 1. 打开网页

```bash
agent-browser open https://example.com
```

### 2. 获取页面快照（查看可交互元素）

```bash
agent-browser snapshot -i
```

输出示例：
```
[ref=e1] heading "Example Domain"
[ref=e2] paragraph "This domain is for use in documentation..."
[ref=e3] link "Learn more"
```

### 3. 与元素交互

```bash
# 点击链接
agent-browser click @e3

# 获取文本
agent-browser get text @e1

# 获取 URL
agent-browser get url
```

### 4. 关闭浏览器

```bash
agent-browser close
```

## 📋 完整演示场景

### 场景 1：百度搜索

```bash
# 1. 打开百度
agent-browser open https://www.baidu.com

# 2. 获取页面快照
agent-browser snapshot -i

# 3. 找到搜索框并输入
agent-browser fill @e1 "Kimi Agent 生态爪痕"

# 4. 点击搜索按钮
agent-browser click @e2

# 5. 等待页面加载
agent-browser wait --load networkidle

# 6. 获取搜索结果
agent-browser get count ".result"

# 7. 截图保存
agent-browser screenshot baidu_search.png

# 8. 关闭浏览器
agent-browser close
```

---

### 场景 2：表单自动填写

```bash
# 1. 打开表单页面
agent-browser open https://example.com/form

# 2. 获取表单元素
agent-browser snapshot -i

# 3. 填写表单
agent-browser fill @e1 "张三"           # 姓名
agent-browser fill @e2 "zhangsan@email.com"  # 邮箱
agent-browser fill @e3 "13800138000"    # 电话
agent-browser select @e4 "北京"         # 城市下拉选择
agent-browser check @e5                 # 勾选协议

# 4. 提交表单
agent-browser click @e6

# 5. 等待提交完成
agent-browser wait --text "提交成功"

# 6. 关闭浏览器
agent-browser close
```

---

### 场景 3：数据抓取

```bash
# 1. 打开目标网站
agent-browser open https://news.example.com

# 2. 获取新闻列表
agent-browser snapshot -i

# 3. 循环获取每条新闻标题
for i in {1..10}; do
    agent-browser get text @e$i
done

# 4. 或者使用 CSS 选择器
agent-browser get count ".news-item"
agent-browser get text ".news-item:nth-child(1) .title"

# 5. 保存为 JSON
agent-browser eval "
  Array.from(document.querySelectorAll('.news-item')).map(item => ({
    title: item.querySelector('.title').textContent,
    link: item.querySelector('a').href,
    time: item.querySelector('.time').textContent
  }))
" --json > news.json

# 6. 关闭浏览器
agent-browser close
```

### 场景 4：登录并保存会话

```bash
# 第一次：登录并保存状态
agent-browser open https://app.example.com/login
agent-browser snapshot -i
agent-browser fill @e1 "username"
agent-browser fill @e2 "password"
agent-browser click @e3
agent-browser wait --url "/dashboard"
agent-browser state save auth.json

# 后续使用：直接加载状态
agent-browser state load auth.json
agent-browser open https://app.example.com/dashboard
agent-browser snapshot -i

# 关闭浏览器
agent-browser close
```

## 🔧 常用命令速查

### 导航
```bash
agent-browser open <url>      # 打开网页
agent-browser back            # 后退
agent-browser forward         # 前进
agent-browser reload          # 刷新
agent-browser close           # 关闭
```

### 快照
```bash
agent-browser snapshot            # 完整快照
agent-browser snapshot -i         # 仅可交互元素（推荐）
agent-browser snapshot -c         # 紧凑输出
agent-browser snapshot -d 3       # 限制深度
```

### 交互
```bash
agent-browser click @e1           # 点击
agent-browser fill @e2 "text"     # 填写输入框
agent-browser type @e2 "text"     # 输入（不清空）
agent-browser press Enter         # 按键
agent-browser hover @e1           # 悬停
agent-browser check @e1           # 勾选
agent-browser select @e1 "value"  # 下拉选择
```

### 获取信息
```bash
agent-browser get text @e1        # 获取文本
agent-browser get html @e1        # 获取 HTML
agent-browser get value @e1       # 获取输入值
agent-browser get url             # 获取当前 URL
agent-browser get title           # 获取页面标题
```

### 截图/录屏
```bash
agent-browser screenshot          # 截图
agent-browser screenshot path.png # 保存到文件
agent-browser screenshot --full   # 全屏截图
agent-browser record start demo.webm  # 开始录屏
agent-browser record stop             # 停止录屏
agent-browser pdf output.pdf          # 生成 PDF
```

### 等待
```bash
agent-browser wait @e1                     # 等待元素
agent-browser wait 2000                    # 等待 2 秒
agent-browser wait --text "Success"        # 等待文本
agent-browser wait --url "/dashboard"      # 等待 URL
agent-browser wait --load networkidle      # 等待网络空闲
```

## 💡 最佳实践

### 1. 总是先快照再交互
```bash
# ✅ 正确
agent-browser open https://example.com
agent-browser snapshot -i
agent-browser click @e1

# ❌ 错误（不知道元素 ref）
agent-browser open https://example.com
agent-browser click ???
```

### 2. 使用 fill 而不是 type
```bash
# ✅ fill 会先清空再输入
agent-browser fill @e1 "新文本"

# ❌ type 会追加到现有文本后
agent-browser type @e1 "新文本"
```

### 3. 导航后重新快照
```bash
# 导航后 refs 会变化，需要重新快照
agent-browser click @e3
agent-browser wait --load networkidle
agent-browser snapshot -i  # 重新获取 refs
```

### 4. 使用语义化选择器
```bash
# 使用角色查找
agent-browser find role button click --name "提交"

# 使用文本查找
agent-browser find text "登录" click

# 使用标签查找
agent-browser find label "邮箱" fill "user@test.com"
```

## 🐛 故障排除

| 问题 | 解决方案 |
|------|----------|
| 命令未找到 | 检查路径：`which agent-browser` |
| 元素未找到 | 先执行 `snapshot -i` 获取正确的 ref |
| 页面未加载 | 添加 `wait --load networkidle` |
| 需要调试 | 添加 `--headed` 参数显示浏览器窗口 |
| 超时错误 | 添加 `--timeout 30000` 增加超时时间 |
