# 记忆系统架构模板

## 目录结构

```
.memory/
├── SESSION-STATE.md     # 热内存（当前会话状态）
├── WORKING-BUFFER.md    # 工作缓冲区
├── daily/               # 日期记忆（温存储）
│   └── YYYY-MM-DD.md
└── topics/              # 主题记忆（冷存储）
    ├── decisions.md     # 决策记录
    ├── relationships.md # 关系图谱
    ├── concepts.md      # 概念定义
    ├── lessons.md       # 经验教训
    └── preferences.md   # 偏好设置
```

## WAL协议（写前日志）

**重要**：在回复用户之前，先更新SESSION-STATE.md

```
1. 收到用户消息
2. 更新SESSION-STATE.md（记录当前状态）
3. 生成回复
4. 发送回复
5. 如有需要，更新长期记忆
```

这样可以防止上下文丢失导致的状态遗忘。

---

## 记忆污染检测

定期检查以下问题：

1. **重复记忆检测**：是否有相同内容被存储多次？
2. **矛盾记忆检测**：是否有前后矛盾的信息？
3. **过时信息检测**：是否有已过时的信息需要更新？
4. **与SOUL冲突检测**：记忆内容是否与身份设定矛盾？

如发现问题，主动向用户汇报并请求确认。
