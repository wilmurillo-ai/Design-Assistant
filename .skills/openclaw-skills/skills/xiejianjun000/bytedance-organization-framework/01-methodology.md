# 01-方法论：字节跳动组织管理核心原则

## 核心方法论：Context, not Control

### 定义
**Context not Control** 是字节跳动最核心的组织管理理念，由张一鸣提出并实践。

**核心思想**：管理者应该为团队提供充分的信息和背景（Context），让一线员工基于这些信息自主做出决策，而不是通过层层审批和控制（Control）来管理。

### 为什么选择 Context not Control

1. **信息不对称问题**：一线员工最接近用户和市场，掌握最新信息
2. **决策速度**：减少审批层级，加快决策速度
3. **人才激励**：优秀的人才希望有自主权，不喜欢被微观管理
4. **规模化挑战**：公司规模扩大后，创始人/高管无法了解所有细节

### Context not Control 的实施条件

| 条件 | 说明 |
|------|------|
| 人才密度 | 招聘最优秀的人，他们有能力做出好决策 |
| 信息透明 | 所有相关信息对决策者开放 |
| 共同目标 | 所有人理解公司使命和优先级 |
| 反馈机制 | 快速试错，快速学习 |

### 与传统的 Control 模式对比

| 维度 | Control 模式 | Context 模式 |
|------|-------------|-------------|
| 决策权 | 集中在高层 | 分散到一线 |
| 信息流 | 自下而上汇报 | 全面透明共享 |
| 管理者角色 | 审批者、控制者 | 信息提供者、教练 |
| 员工角色 | 执行者 | 决策者 + 执行者 |
| 适用场景 | 标准化、重复性工作 | 创新性、复杂性工作 |

### 字节跳动的具体实践

1. **OKR 全面公开**：所有人的 OKR 对公司内部可见
2. **飞书文档文化**：会议前阅读文档，会上直接讨论决策
3. **默认开放**：信息默认对内部开放，除非有特殊理由保密
4. **减少审批**：尽可能减少审批层级，让一线决策

### 张一鸣的原话（公开采访整理）

> "Control 导致的是局部的最优解，而 Context 才能让每个人基于全局信息做出最优决策。"

> "随着公司规模扩大，创始人不可能了解所有细节。如果你坚持 Control，你会成为瓶颈。"

> "Context not Control 的前提是你有足够好的人，他们能理解 Context 并做出好决策。"

### 实施挑战

1. **初期效率可能下降**：员工需要时间适应自主决策
2. **信息过载风险**：需要提供相关 Context，而非所有信息
3. **人才要求高**：不适合能力不足的员工
4. **文化转变难**：从 Control 到 Context 需要管理层真正放权

### 关键洞察

- Context not Control 不是放任不管，而是换一种管理方式
- 管理者需要花更多时间提供 Context，而不是审批决策
- 这个模式高度依赖人才密度和信息透明度
- 适合知识密集型、创新性强的行业
