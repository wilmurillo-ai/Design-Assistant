## 心智模型提炼 (5 个核心)

### 心智模型 1: 赛道优先 (Market First)
**核心逻辑：** 选择比努力更重要，大市场才能养出大公司

**决策流程：**
```
市场规模 (TAM) → 增长速度 → 竞争格局 → 进入时机 → 是否投资
```

**关键指标：**
- TAM > 100 亿美元
- 年增长率 > 30%
- 市场集中度低或正在重构
- 技术/政策/消费习惯变化带来机会窗口

**红杉特色：**
- 宁可错过好团队，不投小市场
- 在一个赛道建立深度认知后持续投资
- 赛道判断基于宏观趋势，不是短期热点

---

### 心智模型 2: 创始人三力模型 (Founder Capability Model)
**核心逻辑：** 创始人是公司的天花板，也是地板

**评估框架：**
```
┌─────────────────────────────────────┐
│           创始人能力评估            │
├─────────────┬─────────────┬─────────┤
│   愿景力    │   执行力    │  学习力  │
│  (Vision)   │ (Execution) │ (Learning)│
├─────────────┼─────────────┼─────────┤
│ 看到别人    │ 把想法变成  │ 快速迭代  │
│ 看不到的    │ 现实的能力  │ 认知和    │
│ 未来        │             │ 能力      │
└─────────────┴─────────────┴─────────┘
```

**评估方法：**
- 愿景力：问"5 年后公司什么样？"看答案的清晰度和野心
- 执行力：看过往经历、关键决策、克服困难的记录
- 学习力：问"如果重新来过，你会做什么不同的事？"

**一票否决：**
- 诚信问题 (绝对红线)
- 缺乏自省能力
- 过度关注估值而非业务

---

### 心智模型 3: 时机判断 (Timing Matrix)
**核心逻辑：** 最好的投资时机是趋势刚启动但未被共识时

**时机矩阵：**
```
              市场认知度
              低        高
        ┌──────────┬──────────┐
      低│  太早    │  最佳    │
    成  │(教育成本)│(共识前)  │
    熟  ├──────────┼──────────┤
    度  高│  观察    │  太晚    │
        │(等下一波)│(红海竞争)│
        └──────────┴──────────┘
```

**红杉的时机判断：**
- 技术成熟度曲线：跨越"幻灭低谷"后进入
- 市场渗透率：5-15% 时最佳 (已验证，未饱和)
- 竞争格局：有 2-3 个玩家，但未定胜负

**案例：**
- Airbnb(2009): 共享经济刚被接受，渗透率<5%
- Stripe(2012): 在线支付需求明确，体验差
- 拼多多 (2015): 下沉市场被忽视，微信红利

---

### 心智模型 4: 单位经济优先 (Unit Economics First)
**核心逻辑：** 增长可以加速或减速，单位经济决定生死

**评估框架：**
```
LTV (客户终身价值) / CAC (获客成本) > 3  ✓ 健康
LTV / CAC < 1  ✗ 不可持续
LTV / CAC = 1-3  ? 需要优化
```

**关键洞察：**
- 早期可以不盈利，但单位经济必须成立
- "花钱买增长"只在单位经济成立时有效
- 规模放大单位经济，不会修复单位经济

**红杉实践：**
- Airbnb 尽调时重点分析房东和房客的 unit economics
- WeWork 失误在于忽视了单位经济不成立
- 要求创始人能清晰解释单位经济模型

---

### 心智模型 5: 生态思维 (Ecosystem Thinking)
**核心逻辑：** 投资的不只是公司，是生态位和网络效应

**思考框架：**
```
单点公司 → 产业链位置 → 网络效应 → 生态价值
```

**评估维度：**
1. **产业链位置**：公司在价值链中的位置，是否掌握关键环节
2. **网络效应类型**：
   - 同边网络效应 (用户越多，对用户越有价值)
   - 跨边网络效应 (一边用户越多，对另一边越有价值)
   - 数据网络效应 (使用越多，产品越好)
3. **生态协同**：与红杉其他被投企业是否有协同

**案例：**
- Apple: 硬件 + 软件 + 服务生态
- Airbnb: 房东 - 房客双边网络
- Stripe: 开发者 - 商家 - 用户生态

## 决策启发式 (8 条)

### 启发式 1: 赛道一票否决
**规则：** 如果市场规模<10 亿美元，直接 pass

**理由：**
- 小市场养不出大公司
- 即使 100% 市场份额，回报也有限
- 机会成本太高

**例外：**
- 市场正在快速扩张 (当前小，未来大)
- 公司有明显扩展相邻市场的能力

---

### 启发式 2: 24 小时决策窗口
**规则：** 看准的项目 24 小时内给 term sheet，看不准的项目明确说需要更多时间

**理由：**
- 好项目竞争激烈，速度是关键
- 犹豫说明没看准，不如坦诚
- 快速决策建立声誉

**实践：**
- Google: 48 小时内决定
- Airbnb: 一周内决定
- FTX: 太快了，缺乏尽调 (反面教材)

---

### 启发式 3: 创始人背景调查三层
**规则：** 必须访谈创始人的前同事、下属、投资人各至少 2 人

**调查重点：**
- 同事：工作能力、合作精神
- 下属：领导风格、是否赋能团队
- 投资人：诚信记录、承诺兑现

**红线信号：**
- 多人提到诚信问题
- 前同事不愿推荐
- 过往投资人有负面评价

---

### 启发式 4: 单位经济测试
**规则：** 创始人必须能清晰解释 LTV/CAC 模型，且 LTV/CAC>3

**测试问题：**
- 获客成本是多少？如何降低？
- 客户留存率是多少？如何提升？
- 毛利是多少？规模效应如何体现？

**警示信号：**
- 创始人说不清楚单位经济
- 用"先增长后盈利"搪塞
- 单位经济依赖补贴

---

### 启发式 5: 竞争格局评估
**规则：** 如果市场已有明确领导者且差距>3 倍，谨慎投资

**评估方法：**
- 市场份额对比
- 增长速度对比
- 资本实力对比
- 团队能力对比

**例外：**
- 有颠覆性技术/模式
- 领导者有明显弱点
- 市场正在重构 (技术/政策变化)

---

### 启发式 6: 估值公平原则
**规则：** 不追求最便宜的估值，追求让创始人有足够动力的公平估值

**理由：**
- 过度压价损害创始人动力
- 长期关系比短期利益重要
- 好项目稀缺，竞争激烈

**实践：**
- 参考市场可比公司
- 给创始人预留足够期权池
- 不在困难时期趁火打劫

---

### 启发式 7: 投后边界原则
**规则：** 战略层面深度参与，运营层面充分授权

**参与边界：**
- ✓ 参与：战略规划、关键 hires、后续融资、重大合作
- ✗ 不参与：日常管理、产品细节、具体业务决策

**例外：**
- 公司遇到危机时临时介入
- 创始人主动请求帮助
- 涉及重大风险 (法律/财务/声誉)

---

### 启发式 8: 退出时机判断
**规则：** 退出时机由公司发展阶段决定，不是由基金周期决定

**判断标准：**
- IPO: 年收入>1 亿美元，增长率>30%，单位经济健康
- 并购：战略价值明确，价格公允，创始人同意
- 继续持有：公司仍有高增长空间，市场估值合理

**红杉特色：**
- IPO 后通常继续持有 3-5 年
- 不因短期股价波动而退出
- 关注公司"终身价值"

## 组织 DNA

### 决策流程
```
项目来源 → 初步筛选 (合伙人) → 深入尽调 → 合伙人讨论 → 共识决策 → 快速执行
   ↓           ↓                  ↓           ↓           ↓          ↓
 多渠道      快速判断          业务/财务/   充分辩论    不需要     24-48 小时
           (1-2 小时)          法律/技术               投票       完成
```

### 沟通风格
- **直接坦诚**：有话直说，不绕弯子
- **数据驱动**：用数据和事实说话
- **建设性批评**：指出问题同时提供解决方案
- **尊重专业**：尊重创始人在业务上的专业判断

### 价值观排序
```
1. 诚信 (Integrity) - 绝对底线
2. 长期主义 (Long-term) - 核心哲学
3. 创始人成功 (Founder Success) - 成功标准
4. 卓越投资 (Investment Excellence) - 专业要求
5. 生态建设 (Ecosystem) - 差异化优势
```

### 红杉美国 vs 红杉中国差异

| 维度 | 红杉美国 | 红杉中国 |
|------|----------|----------|
| 决策风格 | 更保守，充分辩论 | 更激进，快速决策 |
| 投资节奏 | 每年 40-50 个项目 | 每年 80-100 个项目 |
| 赛道偏好 | 硬科技、企业软件 | 消费互联网、硬科技 |
| 持有周期 | 12-15 年 | 8-10 年 |
| 投后参与 | 董事会层面 | 更深度参与 |

## 与个人 Skill 的差异说明

### 组织认知 vs 个人认知

| 维度 | 个人 Skill (如马斯克视角) | 组织 Skill (红杉框架) |
|------|--------------------------|----------------------|
| 决策主体 | 个人直觉 + 经验 | 组织流程 + 共识 |
| 表达风格 | 个人特色鲜明 | 中性专业 |
| 知识来源 | 个人言论 + 行为 | 组织文件 + 案例 |
| 适用场景 | 个人决策参考 | 组织决策参考 |
| 更新机制 | 随个人认知更新 | 随组织演进更新 |

### 红杉框架的特色

1. **系统性**：不是个人直觉，是系统化框架
2. **可传承**：不依赖特定个人，可培训传承
3. **多样性**：融合多合伙人视角，不是单一声音
4. **实证性**：基于 50 年投资案例验证
5. **适应性**：随市场变化而演进

### 使用建议

- **适合场景**：VC 投资决策、创业融资策略、商业分析
- **不适合场景**：个人生活决策、需要个人直觉的创意工作
- **最佳用法**：作为决策检查清单，不是替代思考

### 验证 2: 边缘测试 (用新赛道推断)

| 框架要素 | 分析 | 结论 |
