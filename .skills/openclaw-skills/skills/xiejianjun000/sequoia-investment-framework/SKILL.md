# 红杉资本投资决策框架 Skill

## 技能说明

**名称：** 红杉资本投资决策框架 (Sequoia Investment Framework)

**用途：** 作为思维顾问，用红杉资本的视角分析投资机会、审视商业决策、评估创业公司

**适用场景：**
- VC/PE 投资决策参考
- 创业者融资策略制定
- 商业机会评估
- 竞争格局分析
- 创始人能力评估

**不适用场景：**
- 个人生活决策
- 需要个人直觉的创意工作
- 短期投机决策

---

## 核心心智模型

### 模型 1: 赛道优先 (Market First)

**核心原则：** 选择比努力更重要，大市场才能养出大公司

**决策检查清单：**
- [ ] TAM (总可服务市场) > 100 亿美元？
- [ ] 年增长率 > 30%？
- [ ] 市场集中度低或正在重构？
- [ ] 有技术/政策/消费习惯变化带来的机会窗口？

**关键问题：**
1. "这个市场 5 年后有多大？"
2. "什么因素会驱动市场增长？"
3. "为什么是现在？"

**警示信号：**
- 市场规模小且增长慢
- 市场已被巨头垄断
- 进入时机太早或太晚

**红杉特色：**
> "宁可错过好团队，不投小市场"

---

### 模型 2: 创始人三力模型 (Founder Capability Model)

**核心原则：** 创始人是公司的天花板，也是地板

**评估框架：**

```
┌─────────────────────────────────────┐
│           创始人能力评估            │
├─────────────┬─────────────┬─────────┤
│   愿景力    │   执行力    │  学习力  │
│  (Vision)   │ (Execution) │ (Learning)│
├─────────────┼─────────────┼─────────┤
│ 看到别人    │ 把想法变成  │ 快速迭代  │
│ 看不到的    │ 现实的能力  │ 认知和    │
│ 未来        │             │ 能力      │
└─────────────┴─────────────┴─────────┘
```

**评估方法：**

| 能力 | 评估问题 | 观察点 |
|------|----------|--------|
| 愿景力 | "5 年后公司什么样？" | 答案的清晰度、野心、逻辑 |
| 执行力 | "讲一个你克服困难的经历" | 具体行动、结果、反思 |
| 学习力 | "如果重新来过，你会做什么不同的事？" | 自省深度、认知迭代 |

**一票否决项：**
- ✗ 诚信问题 (绝对红线)
- ✗ 缺乏自省能力
- ✗ 过度关注估值而非业务
- ✗ 团队有重大短板且无法弥补

**红杉特色：**
> "投资就是投人，但人不是抽象的，是在具体赛道里的人" — 沈南鹏

---

### 模型 3: 时机矩阵 (Timing Matrix)

**核心原则：** 最好的投资时机是趋势刚启动但未被共识时

**时机评估矩阵：**

```
              市场认知度
              低        高
        ┌──────────┬──────────┐
      低│  太早    │  ★最佳★  │
    成│(教育成本) │(共识前)  │
    熟├──────────┼──────────┤
    度│  观察    │  太晚    │
    高│(等下一波) │(红海竞争)│
        └──────────┴──────────┘
```

**判断指标：**
- 技术成熟度：跨越"幻灭低谷"后进入
- 市场渗透率：5-15% 时最佳 (已验证，未饱和)
- 竞争格局：有 2-3 个玩家，但未定胜负

**关键问题：**
1. "市场渗透率现在是多少？"
2. "什么信号表明时机成熟了？"
3. "如果等一年，会发生什么变化？"

**红杉案例：**
- Airbnb(2009): 共享经济刚被接受，渗透率<5% ✓
- 共享办公 (2015): 渗透率约 10%，竞争加剧 → 谨慎

---

### 模型 4: 单位经济模型 (Unit Economics)

**核心原则：** 增长可以加速或减速，单位经济决定生死

**核心公式：**
```
LTV (客户终身价值) / CAC (获客成本) > 3  ✓ 健康
LTV / CAC < 1  ✗ 不可持续
LTV / CAC = 1-3  ? 需要优化
```

**评估检查清单：**
- [ ] 创始人能清晰解释 LTV/CAC 模型？
- [ ] LTV/CAC > 3？
- [ ] 获客成本有下降趋势？
- [ ] 留存率健康 (年留存>80%)？
- [ ] 毛利有提升空间？

**关键问题：**
1. "获客成本是多少？如何降低？"
2. "客户留存率是多少？如何提升？"
3. "规模效应如何体现？"

**警示信号：**
- 创始人说不清楚单位经济
- 用"先增长后盈利"搪塞
- 单位经济依赖补贴
- 规模放大后单位经济恶化

**红杉教训：**
> WeWork 失误在于忽视了单位经济不成立

---

### 模型 5: 生态思维 (Ecosystem Thinking)

**核心原则：** 投资的不只是公司，是生态位和网络效应

**评估框架：**

```
单点公司 → 产业链位置 → 网络效应 → 生态价值
```

**网络效应类型：**

| 类型 | 说明 | 案例 |
|------|------|------|
| 同边网络效应 | 用户越多，对用户越有价值 | 微信、Facebook |
| 跨边网络效应 | 一边用户越多，对另一边越有价值 | Airbnb、Uber |
| 数据网络效应 | 使用越多，产品越好 | Google 搜索、推荐系统 |

**关键问题：**
1. "公司在产业链中的位置是什么？"
2. "有什么类型的网络效应？"
3. "网络效应的强度如何？"
4. "与红杉生态是否有协同？"

**红杉特色：**
> "我们投资的不只是公司，是生态"

## 决策启发式 (8 条)

### 启发式 1: 赛道一票否决
**规则：** 如果市场规模<10 亿美元，直接 pass

**理由：** 小市场养不出大公司，机会成本太高

**例外：** 市场正在快速扩张，或有明显扩展相邻市场的能力

---

### 启发式 2: 24 小时决策窗口
**规则：** 看准的项目 24 小时内给 term sheet，看不准的项目明确说需要更多时间

**理由：** 好项目竞争激烈，速度是关键；犹豫说明没看准

**实践：** Google(48 小时)、Airbnb(1 周)、FTX(太快了，反面教材)

---

### 启发式 3: 创始人背景调查三层
**规则：** 必须访谈创始人的前同事、下属、投资人各至少 2 人

**调查重点：**
- 同事：工作能力、合作精神
- 下属：领导风格、是否赋能团队
- 投资人：诚信记录、承诺兑现

**红线信号：** 多人提到诚信问题、前同事不愿推荐

---

### 启发式 4: 单位经济测试
**规则：** 创始人必须能清晰解释 LTV/CAC 模型，且 LTV/CAC>3

**测试问题：** 获客成本、留存率、毛利、规模效应

**警示信号：** 说不清楚、用"先增长后盈利"搪塞、依赖补贴

---

### 启发式 5: 竞争格局评估
**规则：** 如果市场已有明确领导者且差距>3 倍，谨慎投资

**评估方法：** 市场份额、增长速度、资本实力、团队能力对比

**例外：** 有颠覆性技术/模式、领导者有明显弱点、市场正在重构

---

### 启发式 6: 估值公平原则
**规则：** 不追求最便宜的估值，追求让创始人有足够动力的公平估值

**理由：** 过度压价损害创始人动力，长期关系比短期利益重要

**实践：** 参考市场可比公司、预留足够期权池、不趁火打劫

---

### 启发式 7: 投后边界原则
**规则：** 战略层面深度参与，运营层面充分授权

**参与边界：**
- ✓ 参与：战略规划、关键 hires、后续融资、重大合作
- ✗ 不参与：日常管理、产品细节、具体业务决策

**例外：** 公司危机时临时介入、创始人主动请求、涉及重大风险

---

### 启发式 8: 退出时机判断
**规则：** 退出时机由公司发展阶段决定，不是由基金周期决定

**判断标准：**
- IPO: 年收入>1 亿美元，增长率>30%，单位经济健康
- 并购：战略价值明确，价格公允，创始人同意
- 继续持有：公司仍有高增长空间，市场估值合理

**红杉特色：** IPO 后通常继续持有 3-5 年，不因短期波动退出

## 组织 DNA

### 决策流程
```
项目来源 → 初步筛选 (1-2 小时) → 深入尽调 → 合伙人讨论 → 共识决策 → 快速执行 (24-48 小时)
   ↓              ↓                  ↓           ↓           ↓          ↓
 多渠道       快速判断          业务/财务/   充分辩论    不需要     完成
           (合伙人)            法律/技术               投票
```

### 沟通风格
- **直接坦诚**：有话直说，不绕弯子
- **数据驱动**：用数据和事实说话
- **建设性批评**：指出问题同时提供解决方案
- **尊重专业**：尊重创始人在业务上的专业判断

### 价值观排序
```
1. 诚信 (Integrity) - 绝对底线
2. 长期主义 (Long-term) - 核心哲学
3. 创始人成功 (Founder Success) - 成功标准
4. 卓越投资 (Investment Excellence) - 专业要求
5. 生态建设 (Ecosystem) - 差异化优势
```

### 红杉美国 vs 红杉中国差异

| 维度 | 红杉美国 | 红杉中国 |
|------|----------|----------|
| 决策风格 | 更保守，充分辩论 | 更激进，快速决策 |
| 投资节奏 | 每年 40-50 个项目 | 每年 80-100 个项目 |
| 赛道偏好 | 硬科技、企业软件 | 消费互联网、硬科技 |
| 持有周期 | 12-15 年 | 8-10 年 |
| 投后参与 | 董事会层面 | 更深度参与 |

## 使用方式

### 作为决策检查清单

在做出投资决策前，逐项检查：

```markdown
## 投资决策检查清单

### 赛道评估
- [ ] TAM > 100 亿美元
- [ ] 年增长率 > 30%
- [ ] 市场集中度低或正在重构
- [ ] 时机合适 (渗透率 5-15%)

### 创始人评估
- [ ] 愿景力：清晰的 5 年愿景
- [ ] 执行力：过往成功记录
- [ ] 学习力：深度自省能力
- [ ] 背景调查：无诚信问题

### 单位经济
- [ ] LTV/CAC > 3
- [ ] 留存率健康
- [ ] 毛利有提升空间

### 竞争格局
- [ ] 无明显领导者，或差距<3 倍
- [ ] 有差异化优势

### 估值
- [ ] 估值公平，创始人有足够动力

### 综合判断
- [ ] 符合红杉投资哲学
- [ ] 有本垒打潜力 (10x+ 回报)
```

### 作为分析框架

分析一个商业机会时，按以下顺序思考：

1. **赛道分析**：市场有多大？增长多快？为什么是现在？
2. **创始人分析**：三力模型评估，背景调查
3. **时机分析**：市场渗透率？竞争格局？
4. **单位经济分析**：LTV/CAC？留存率？规模效应？
5. **生态分析**：网络效应类型？产业链位置？
6. **综合判断**：是否符合启发式规则？

### 作为学习工具

学习红杉投资案例时，用框架复盘：

1. 红杉当时看到了什么赛道机会？
2. 创始人有哪些特质打动了红杉？
3. 时机判断是否准确？
4. 单位经济是否成立？
5. 最终结果如何？有什么教训？

1. **作为检查清单**：在决策前逐项检查，避免遗漏关键维度
