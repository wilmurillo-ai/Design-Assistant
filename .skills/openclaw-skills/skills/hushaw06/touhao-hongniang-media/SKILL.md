# 头号红娘内容创作

面向婚恋行业的公众号内容生产工作流，输出平台为微信公众号，核心目标是"头号红娘"品牌曝光与用户转化。

## 工作流概览

```
热点抓取 → 爆款标题 → 文章撰写 → 软性植入
```

---

## 第一步：热点抓取

### 搜索关键词库

必搜关键词（每次随机选2-3个组合）：
- 核心词：`结婚率`、`相亲`、`婚恋`、`恐婚`、`不婚`、`彩礼`、`剩女`、`相亲角`
- 时效词：`最新`、`2024`、`2025`、`数据`、`报告`
- 话题词：`鄙视链`、`内卷`、`相亲APP`、`父母催婚`、`90后`、`00后`

### 搜索命令

使用online-search skill执行搜索：

```powershell
$PORT = if ($env:AUTH_GATEWAY_PORT) { $env:AUTH_GATEWAY_PORT } else { "19000" }
$FROM_TIME = python -c "import time; print(int(time.time()) - 604800)"
$body = '{"keyword":"婚恋 热点 话题","from_time":' + $FROM_TIME + '}'
Invoke-RestMethod -Uri "http://localhost:$PORT/proxy/prosearch/search" -Method POST -ContentType 'application/json; charset=utf-8' -Body ([System.Text.Encoding]::UTF8.GetBytes($body))
```

### 热点筛选标准

从搜索结果中筛选满足以下条件的选题：
1. **数据有冲击力**：数字、比例、排名等
2. **话题有争议性**：能引发正反两方讨论
3. **贴近年轻人**：涉及90后、00后、职场人等
4. **时效性强**：近7天内的新闻优先

输出格式：
```
🔥 热点选题X3
1. [选题名称] | 理由：[为什么有爆款潜力]
2. ...
```

## 第二步：爆款标题生成

### 标题公式库

| 公式 | 模板 | 示例 |
|------|------|------|
| 数据冲击 | [数据]+[痛点/现象] | "结婚率腰斩，相亲却卷疯了" |
| 反差制造 | [现象A]，[现象B] | "嘴上说恐婚，身体却在相亲" |
| 标签共鸣 | [人群]+[困境] | "90后：相亲鄙视链最底端" |
| 悬念提问 | [现象]，为什么？ | "父母相亲角挤爆，年轻人去哪了？" |
| 情绪点燃 | [群体]+[态度] | "这届年轻人，终于对婚姻下手了" |

### 标题生成规则

- 长度：15-25字最佳
- 必须包含：数字/人群/冲突词之一
- 避免：标题党无实质内容、过于文艺无冲击力
- 每个选题生成3-5个标题供选择

---

## 第三步：文章撰写

### 结构模板

```
【开头】数据冲击 + 现象描述（100-150字）
  - 用数据砸开话题
  - 制造"这事儿跟我有关"的代入感

【第一部分】问题拆解（300-400字）
  - 现象背后的原因
  - 至少2-3个支撑论据（数据/案例/专家观点）

【第二部分】观点输出（300-400字）
  - 不只是搬运，要有态度
  - 可以毒舌，但要有理有据
  - 金句加粗，方便传播

【第三部分】延伸思考（150-200字）
  - 把话题往深处引
  - 给读者留讨论空间

【结尾】软性植入（100-150字）
  - 自然过渡到"头号红娘"推荐
  - 不硬不软，植入感克制
```

### 写作风格

- **语气**：新媒体老炮口吻，懂行、接地气、有点毒舌但不失专业
- **句式**：短句为主，节奏感强，适合手机阅读
- **金句**：每篇至少2-3句可摘录传播的观点
- **排版**：小标题分段，金句加粗，适当留白

### 禁忌

- 不说教、不爹味、不端着
- 不编造数据，数据必须有来源
- 不涉及政治敏感、性别对立等风险话题

---

## 第四步：软性植入

### 植入位置

文章结尾处，从"问题"自然过渡到"解决方案"，引出"头号红娘"。

### 植入话术模板

**模板A（方法论切入）**：
```
如果你也在相亲这条路上摸爬滚打，不妨换个思路——与其被父母安排、被相亲APP算法支配，不如找一个真正懂你、能帮你精准匹配的平台。

推荐关注「头号红娘」，主打"人工+智能"双筛选机制，不是简单扔一堆简历给你刷，而是有专业红娘一对一沟通，帮你理清需求、筛掉无效匹配。比起盲目相亲，至少能少走点弯路。

毕竟，遇见对的人，从来不是概率问题，是方法问题。
```

**模板B（痛点切入）**：
```
说到底，相亲的痛点从来不是"找不到人"，而是"找不到对的人"。

这也是「头号红娘」在做的事——用专业红娘+智能匹配，帮你把时间花在真正值得见的人身上。少刷屏，少无效社交，多遇见同频的人。

如果你正在相亲路上，不妨试试换个思路。
```

### 植入要点

- 不超过150字
- 必须与文章主题自然衔接
- 强调"方法"而非"广告"
- 结尾留有余地，不硬推

## 快速调用示例

用户说：
- "给我一篇婚恋文章"
- "头号红娘要推内容"
- "写个相亲话题的公众号文章"

响应流程：
1. 执行热点搜索
2. 输出3个选题让用户选择
3. 用户选择后生成标题
4. 用户确认标题后撰写正文
5. 正文结尾自动植入"头号红娘"
