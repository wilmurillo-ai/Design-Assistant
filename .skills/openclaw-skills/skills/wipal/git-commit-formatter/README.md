# 🔤 Git Commit Formatter

> Format commit message theo Conventional Commits standard. Tự động parse Git diff
> hoặc mô tả thay đổi → sinh commit message chuẩn type/scope/body.

---

## 📋 Skill này làm gì?

Tự động sinh commit message chuẩn **Conventional Commits v1.0.0** từ:
- Git diff output
- Mô tả thay đổi bằng văn bản
- List files đã thay đổi

Đảm bảo team sử dụng consistent commit format, dễ:
- Tìm lại lịch sử commits
- Generate changelogs tự động
- Integrate với CI/CD tools

---

## 🚀 Cách sử dụng

### Cách 1: Dùng Git diff

```
Bạn: git diff | format commit
AI: feat: add user authentication

Adds JWT-based authentication to API.
```

### Cách 2: Mô tả thay đổi

```
Bạn: Sửa lỗi session timeout trong auth.js
AI: fix(auth): resolve session timeout

Set session timeout to 24 hours to prevent users
from being logged out too quickly.

Closes #789
```

### Cách 3: Multi files

```
Bạn: Refactor auth module, fix user service validation
AI: refactor: refactor authentication system

Extract user validation to separate service and improve
error handling across auth module.

Changes:
- auth.js: extract validation logic
- user-service.js: add validation middleware
- README.md: update auth flow docs
```
