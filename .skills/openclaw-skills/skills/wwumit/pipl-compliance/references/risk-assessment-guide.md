# PIPL风险评估指南

## 风险评估概述

### 1. 风险评估目的
- 识别个人信息处理活动中的风险
- 评估风险对个人权益的影响程度
- 确定风险优先级和应对措施
- 满足PIPL合规要求

### 2. 风险评估原则
- **全面性**：覆盖所有个人信息处理活动
- **客观性**：基于事实和证据进行评估
- **持续性**：定期更新和重新评估
- **可操作性**：评估结果可用于指导改进

### 3. 风险评估时机
- 新功能/新服务上线前
- 处理方式发生重大变更时
- 发生安全事件后
- 定期（建议每半年一次）
- 法规变化时

## 风险评估框架

### 风险维度及评分标准

#### 维度1：数据敏感度（权重：30%）

**评分标准**：
- **1分（低敏感）**：仅收集基础身份信息（用户名、邮箱等）
- **3分（中敏感）**：收集一般个人信息（联系方式、地址等）
- **5分（高敏感）**：收集敏感个人信息（生物识别、金融账户等）
- **7分（极高敏感）**：收集未成年人敏感信息或特殊类别信息

#### 维度2：处理规模（权重：20%）

**评分标准**：
- **1分（小规模）**：处理个人数<1000
- **3分（中等规模）**：处理个人数1000-10000
- **5分（大规模）**：处理个人数10000-100000
- **7分（超大规模）**：处理个人数>100000

#### 维度3：处理目的（权重：15%）

**评分标准**：
- **1分（必要性强）**：履行合同所必需
- **3分（正当性高）**：维护公共利益或重大个人利益
- **5分（商业目的）**：用于营销、个性化推荐等
- **7分（敏感性高）**：用于自动化决策、用户画像等

#### 维度4：第三方参与（权重：15%）

**评分标准**：
- **1分（无第三方）**：完全自主处理
- **3分（有限委托）**：仅技术性委托处理
- **5分（多方共享）**：与多个第三方共享数据
- **7分（跨境传输）**：涉及跨境数据传输

#### 维度5：安全保障（权重：20%）

**评分标准**：
- **1分（措施完善）**：有完善的技术和管理措施，定期审计
- **3分（措施基本）**：有基本安全措施，但未定期更新
- **5分（措施不足）**：安全措施不完善，存在明显漏洞
- **7分（无措施）**：无有效安全措施

### 风险等级计算

**计算公式**：
```
总风险分 = Σ(维度分 × 权重)
风险等级 = 总风险分 ÷ 最高可能分(7分) × 100%
```

**风险等级划分**：
- **低风险（<40%）**：基本合规，建议持续优化
- **中风险（40%-70%）**：存在合规问题，需要限期改进
- **高风险（>70%）**：存在严重风险，需要立即整改

## 风险评估实施步骤

### 步骤1：识别处理活动
1. 梳理所有个人信息处理场景
2. 识别每个场景的处理流程
3. 确定涉及的个人信息类型
4. 识别涉及的内部和外部方

**工具支持**：
```bash
# 使用数据映射工具
python3 scripts/data-mapping.py --scope all
```

### 步骤2：收集评估信息
1. **收集制度文件**
   - 隐私政策
   - 用户协议
   - 数据处理协议
   - 安全管理制度

2. **收集技术信息**
   - 系统架构图
   - 数据流图
   - 安全措施说明
   - 审计报告

3. **收集运营信息**
   - 处理个人数统计
   - 用户投诉记录
   - 安全事件记录
   - 合规检查记录

### 步骤3：执行风险评估
1. **逐项评估**
   - 对每个处理活动进行评估
   - 按照五个维度进行打分
   - 记录评分依据和证据

2. **计算风险等级**
   - 计算每个活动的总风险分
   - 确定风险等级
   - 识别主要风险点

**工具支持**：
```bash
# 执行风险评估
python3 scripts/risk-assessment.py --activity "用户注册"
```

### 步骤4：制定应对措施
1. **高风险活动**
   - 立即停止或限制处理
   - 制定详细整改计划
   - 优先分配资源整改
   - 报告管理层和监管机构（如需要）

2. **中风险活动**
   - 制定30-60天整改计划
   - 明确责任人和时间表
   - 定期检查整改进展
   - 验证整改效果

3. **低风险活动**
   - 制定优化计划
   - 纳入持续改进流程
   - 定期监控优化效果

### 步骤5：记录和报告
1. **风险评估报告**
   - 评估范围和方法
   - 评估结果和发现
   - 风险等级和优先级
   - 整改建议和计划

2. **证据材料**
   - 评分表和依据
   - 相关文件副本
   - 访谈记录和会议纪要
   - 整改证据材料

## 特定场景风险评估

### 场景1：自动化决策和用户画像
**特殊风险点**：
- 算法透明度不足
- 算法歧视风险
- 个人权益受限
- 解释困难

**评估要点**：
1. 算法逻辑是否可解释
2. 是否提供拒绝选项
3. 是否定期评估算法公平性
4. 是否提供人工复核机制

### 场景2：跨境数据传输
**特殊风险点**：
- 境外法律环境差异
- 监管不确定性
- 数据主权问题
- 执法合作困难

**评估要点**：
1. 接收方所在国保护水平
2. 传输路径合法性
3. 安全保障措施充分性
4. 应急响应能力

### 场景3：敏感个人信息处理
**特殊风险点**：
- 对个人权益影响大
- 监管要求严格
- 社会关注度高
- 处罚力度大

**评估要点**：
1. 单独同意机制有效性
2. 额外安全保障措施
3. 访问控制严格性
4. 保存期限合理性

### 场景4：未成年人信息处理
**特殊风险点**：
- 特殊保护义务
- 监护人同意要求
- 内容适应性要求
- 社会敏感性高

**评估要点**：
1. 年龄验证机制有效性
2. 监护人同意机制
3. 内容过滤和限制
4. 特殊安全措施

## 持续风险管理

### 1. 风险监控
- **定期监控**：每季度更新风险状态
- **事件触发**：发生重大变化时重新评估
- **趋势分析**：分析风险变化趋势

### 2. 风险报告
- **定期报告**：每半年向管理层报告
- **专项报告**：重大风险事件专项报告
- **监管报告**：按监管要求提交报告

### 3. 风险改进
- **整改跟踪**：跟踪整改措施执行情况
- **效果评估**：评估整改措施有效性
- **持续优化**：基于评估结果持续优化

### 4. 风险文化建设
- **培训教育**：定期进行风险意识培训
- **经验分享**：分享风险评估经验
- **最佳实践**：建立和推广最佳实践
