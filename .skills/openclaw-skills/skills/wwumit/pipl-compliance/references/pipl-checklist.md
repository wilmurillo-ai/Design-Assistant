# PIPL合规检查清单

## 通用检查项（适用于所有个人信息处理活动）

### 1. 告知义务检查
- [ ] **告知内容完整性**
  - 处理目的、方式
  - 处理的个人信息种类、保存期限
  - 个人行使权利的方式和程序
  - 个人信息处理者的名称和联系方式
  - 法律、行政法规规定应当告知的其他事项

- [ ] **告知方式适当性**
  - 以显著方式、清晰易懂的语言
  - 易于个人理解
  - 便于个人查阅和保存

- [ ] **告知时机**
  - 在收集个人信息前告知
  - 处理目的、方式、种类发生变更时重新告知

### 2. 同意机制检查
- [ ] **同意有效性**
  - 取得个人同意
  - 同意基于充分知情
  - 同意是个人自愿、明确作出

- [ ] **同意撤回机制**
  - 提供便捷的同意撤回方式
  - 撤回同意不影响撤回前基于个人同意的处理

- [ ] **单独同意要求**
  - 处理敏感个人信息时取得单独同意
  - 向其他处理者提供个人信息时取得单独同意
  - 公开个人信息时取得单独同意
  - 公共场所安装图像采集设备时取得单独同意

### 3. 数据最小化检查
- [ ] **收集最小化**
  - 仅收集与处理目的直接相关的个人信息
  - 不收集非必要个人信息

- [ ] **处理最小化**
  - 处理限于实现处理目的的最小范围
  - 不进行与处理目的无关的处理

- [ ] **保存期限合理**
  - 保存期限为实现处理目的所必要的最短时间
  - 法律、行政法规另有规定的除外

### 4. 安全保障检查
- [ ] **技术措施**
  - 加密、去标识化等安全技术措施
  - 访问控制机制
  - 安全事件监测和响应机制

- [ ] **管理措施**
  - 内部管理制度和操作规程
  - 个人信息分类分级管理
  - 安全教育和培训
  - 定期安全风险评估

- [ ] **第三方管理**
  - 委托处理时签订数据处理协议
  - 对受托方的处理活动进行监督
  - 确保受托方达到同等安全保护水平

## 特定场景检查项

### 场景A：用户注册和账户管理
- [ ] **注册流程**
  - 提供隐私政策链接
  - 收集必要信息（用户名、密码等）
  - 提供同意选项
  - 允许拒绝非必要信息收集

- [ ] **账户管理**
  - 提供个人信息查询、复制、更正、删除功能
  - 提供账户注销功能
  - 注销后及时删除或匿名化处理个人信息

### 场景B：位置信息收集
- [ ] **告知内容**
  - 明确告知位置信息收集目的
  - 告知位置信息精度和频率
  - 告知位置信息使用场景

- [ ] **同意机制**
  - 取得单独同意
  - 提供位置信息开关
  - 允许随时关闭位置信息收集

- [ ] **最小化要求**
  - 仅在必要时收集位置信息
  - 降低位置信息精度（如仅需城市级）
  - 减少位置信息收集频率

### 场景C：用户画像和个性化推荐
- [ ] **合法性基础**
  - 取得个人同意
  - 提供个性化推荐开关
  - 允许拒绝个性化推荐

- [ ] **透明度要求**
  - 告知用户画像标签和算法逻辑
  - 提供标签管理功能
  - 允许个人查询、更正、删除用户画像

- [ ] **公平性要求**
  - 不基于用户画像进行价格歧视
  - 不进行不合理的差别待遇

### 场景D：跨境数据传输
- [ ] **合法性路径**
  - 通过网信部门组织的安全评估
  - 通过专业机构进行的个人信息保护认证
  - 签订网信部门制定的标准合同
  - 法律、行政法规或网信部门规定的其他条件

- [ ] **告知义务**
  - 告知境外接收方名称和联系方式
  - 告知处理目的、方式、种类
  - 告知个人向境外接收方行使权利的方式

- [ ] **安全保障**
  - 评估境外接收方所在国家或地区的个人信息保护水平
  - 采取必要措施保障境外接收方处理活动安全
  - 发生个人信息泄露等事件时及时通知个人

## 特殊人群检查项

### 1. 未成年人个人信息保护
- [ ] **年龄验证**
  - 建立有效的年龄验证机制
  - 对未成年人进行特殊标记

- [ ] **监护人同意**
  - 处理不满十四周岁未成年人信息时取得监护人同意
  - 制定专门的未成年人个人信息处理规则

- [ ] **内容适配**
  - 提供适合未成年人的服务模式
  - 限制向未成年人推送不适宜内容

### 2. 员工个人信息保护
- [ ] **必要性原则**
  - 仅收集与劳动关系直接相关的个人信息
  - 不收集与劳动关系无关的个人信息

- [ ] **告知同意**
  - 在劳动合同中明确告知个人信息处理事项
  - 取得员工同意（非劳动关系必要条件时）

- [ ] **安全保障**
  - 对员工个人信息进行特殊保护
  - 限制非授权访问员工个人信息

## 检查结果处理

### 1. 风险等级判定
- **高风险（立即整改）**
  - 未取得同意处理敏感个人信息
  - 未履行告知义务
  - 发生个人信息泄露事件
  - 其他严重违反PIPL要求

- **中风险（限期改进）**
  - 告知内容不完整
  - 同意机制不完善
  - 安全保障措施不足
  - 其他一般性违规

- **低风险（持续优化）**
  - 文档不够规范
  - 流程不够完善
  - 其他轻微问题

### 2. 整改建议
- **高风险问题**
  - 立即停止违规处理活动
  - 制定详细整改计划
  - 报告监管机构（如涉及）
  - 通知受影响个人

- **中风险问题**
  - 制定30天整改计划
  - 完善相关制度和流程
  - 加强员工培训
  - 定期检查整改效果

- **低风险问题**
  - 制定优化计划
  - 纳入持续改进流程
  - 定期评估优化效果

### 3. 记录保存
- [ ] **检查记录**
  - 保存检查过程记录
  - 保存检查结果和证据
  - 保存整改计划和执行情况

- [ ] **合规证明**
  - 准备合规证明文件
  - 准备应对监管检查材料
  - 定期更新合规记录

## 检查频率建议

### 1. 定期检查
- **全面检查**：每半年一次
- **专项检查**：每季度一次（针对高风险领域）
- **即时检查**：新功能上线前、重大变更时

### 2. 触发检查
- 法规变化时
- 监管要求时
- 发生安全事件时
- 收到用户投诉时

### 3. 检查报告
- 每次检查后生成检查报告
- 报告包括检查范围、方法、结果、建议
- 报告提交管理层审阅
- 报告作为合规证明文件保存

---

**使用说明**：
1. 根据实际处理活动选择相应检查项
2. 逐项检查并记录结果
3. 根据检查结果确定风险等级
4. 制定和执行整改计划
5. 保存检查记录和证明文件
