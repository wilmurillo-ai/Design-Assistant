# База знаний: Профессиональный дизайн для AI-агентов

> Собрана: 2026-03-25  
> Источники: NN/g eyetracking research, artlebedev.ru/kovodstvo, материалы о Fiverr/Kwork, публичные design system guidelines

---

## 1. Обложки для фриланс-бирж

### 1.1 Fiverr — гиг-изображения

**Технические требования (официальные):**
- Размер: **550 × 370 пикселей** (соотношение ~3:2), минимум
- Рекомендуется: **1280 × 769 px** для чёткости на ретина-дисплеях
- Формат: JPEG или PNG
- Максимальный вес: 5 МБ
- Можно загрузить до 3 изображений + видео

**Что работает (лучшие практики):**
- Чистый, незагромождённый фон — минимализм лучше
- Крупный текст с ключевым преимуществом: «I will design your logo for $50»
- Лицо исполнителя или реальный пример работы — повышает доверие
- Контрастный заголовок, читаемый в миниатюре (thumbnail)
- Брендовые цвета — создают узнаваемость при повторном просмотре
- Стрелки, иконки, визуальные «ловушки» взгляда
- Показывай результат, а не процесс — покупатель покупает итог

**Что снижает CTR:**
- Мелкий нечитаемый текст
- Клипарт и дешёвые стоковые изображения
- Перегруженность: несколько логотипов, много цветов, много текста
- Нет чёткого UVP (уникального ценностного предложения) на обложке

**Рекомендованные цвета:**
- Fiverr сам использует зелёный — он ассоциируется с платформой
- Для выделения среди конкурентов: оранжевый, ярко-синий, белый на тёмном

---

### 1.2 Kwork — обложки воркеров

**Технические требования:**
- Размер: **1200 × 800 пикселей** (соотношение 3:2)
- Форматы: JPG, PNG, GIF
- Максимальный вес: 6 МБ
- Рекомендуется делать несколько вариантов (каталог, мобильный вид)

**Что работает на Kwork:**
- Конкретная польза в заголовке: «Логотип за 24 часа»
- Примеры работ прямо на обложке (мозаика из 2-4 работ)
- Тёмный или контрастный фон — выделяет среди светлых
- Цена или скорость как визуальный акцент
- Кириллический текст крупно — русская аудитория
- Аватар исполнителя в углу — персонализация

**Специфика российского рынка (Kwork):**
- Конкретика и цифры > абстрактные обещания
- «Сделаю за 500 рублей» работает лучше, чем «качественный дизайн»
- Отзывы и рейтинг важнее красоты — покупатели смотрят на цифры

---

### 1.3 Upwork — профиль и портфолио

**Фото профиля:**
- Размер: 250 × 250 px минимум (квадрат)
- Профессиональное фото лица — обязательно
- Светлый однотонный фон
- Улыбка, открытость — доверие

**Портфолио:**
- Рекомендуется: 6-10 кейсов
- Каждый кейс: превью 1920 × 1080 или 800 × 600
- Формат кейса: до → после, задача → решение
- Показывать реальные результаты (метрики, если есть)

## 2. Дизайн-системы и гайды крупных компаний

### 2.1 Google Material Design 3 (M3)

**Ключевые принципы:**
- **Material You** — адаптация под пользователя, персонализация цветовой схемы
- **Dynamic Color** — цвета генерируются из wallpaper/брендинга автоматически
- Высота (elevation) заменена цветовыми оттенками вместо теней
- **Typography scale** — 5 уровней: Display, Headline, Title, Body, Label

**Основные правила:**
- 4dp grid — все отступы и размеры кратны 4
- Минимальный размер тапабельного элемента: 48 × 48 dp
- Контраст текст/фон: минимум 4.5:1 для обычного текста, 3:1 для крупного
- Максимум 3 роли цвета в одном компоненте: primary, secondary, tertiary
- Анимации: 200-300ms для большинства переходов

**Цветовая система M3:**
- Primary, Secondary, Tertiary + Error
- У каждого цвета: Container, On, OnContainer варианты
- Surface, Background — нейтральные

**Доступность (обязательно):**
- Все интерактивные элементы доступны с клавиатуры
- Поддержка screen reader
- Масштабирование текста до 200% без потери функциональности

---

### 2.2 Apple Human Interface Guidelines (HIG)

**Фундаментальные принципы Apple:**
- **Эстетическая целостность** — визуальное оформление согласовано с функцией
- **Единообразие** — используй стандартные компоненты; не изобретай велосипед
- **Прямое манипулирование** — пользователь взаимодействует с объектами напрямую
- **Обратная связь** — каждое действие получает видимый/слышимый ответ
- **Метафоры** — используй знакомые концепции реального мира
- **Пользовательский контроль** — человек инициирует действия, приложение предупреждает

**Ключевые правила UI:**
- Минимальный размер касания: **44 × 44 pt** (iOS)
- Безопасная зона: учитывай notch, Dynamic Island, home indicator
- Шрифт iOS: SF Pro (системный), не нужно встраивать
- Используй SF Symbols для иконок — согласованы с системой
- Не воспроизводи UI других платформ (нет Material Design на iOS)

**Типографика (iOS):**
- Large Title: 34pt, Bold
- Title 1: 28pt, Regular
- Title 2: 22pt, Regular
- Body: 17pt, Regular
- Caption: 12pt, Regular

**Цвет:**
- 6 системных серых + динамические цвета (светлая/тёмная тема)
- Акцентный цвет — 1 на приложение
- Тёмная тема: обязательна с iOS 13

---

### 2.3 IBM Design Language

**Принципы IBM:**
- **Purposeful** — каждый элемент служит задаче, ничего декоративного без смысла
- **Innovative** — используй технологии для решения реальных проблем
- **Expressive** — IBM Carbon Design System — открытый, можно использовать

**Carbon Design System (open source):**
- Grid: 12-колоночная сетка, 2xl breakpoint: 1584px
- Spacing: кратен 4px или 8px (micro: 4, 8; standard: 16, 24, 32; large: 48, 64, 96)
- Шрифт: IBM Plex — доступен бесплатно на Google Fonts
- Иконки: Carbon Icons (открытая библиотека, 2000+ иконок)
- Компоненты на React, Angular, Vue, Web Components

**IBM Design Principles:**
- Дизайн — это решение проблем, а не украшение
- Пользователь в центре (human-centered design)
- Системное мышление: компоненты, паттерны, руководства

---

### 2.4 Atlassian Design System

**Принципы Atlassian:**
- **Bold** — уверенный, прямой визуальный язык
- **Optimistic** — позитивный тон, доверие
- **Practical** — функциональность прежде эстетики

**Технические правила:**
- Spacing tokens: 4px base unit
- Шрифт: Charlie Display (заголовки), Inter (текст)
- Иконки: Atlassian Icons — открытая часть
- Цвета: Blue (primary action), Green (success), Red (danger), Yellow (warning)

**Design tokens (доступны публично):**
- `color.background.brand.bold` = #0052CC (Atlassian Blue)
- `space.100` = 8px, `space.200` = 16px, `space.400` = 32px

## 3. Ководство Артемия Лебедева — ключевые тезисы

> Источник: artlebedev.ru/kovodstvo (публичный онлайн-ресурс)

### Фундаментальные принципы

**§ 108. О простоте дизайна:**
- «Дизайн должен быть простым.»
- Простота — это не бедность, это концентрация смысла

**§ 136. Теория близости:**
- Объекты, расположенные рядом, воспринимаются как связанные
- Заголовок должен быть ближе к своему абзацу, чем к предыдущему
- Пространство между элементами — это смысловое послание
- Дизайнер обязан разделять несвязанные объекты заметными признаками: расположением, размером, цветом

**§ 166. Воздух общий (о рекламе):**
- Реклама раздражает, когда выскакивает на место полезного контента
- У рекламы должно быть своё чётко обозначенное место
- Не занимай чужое пространство без разрешения пользователя

**§ 125. Факты:**
- Дизайнер обязан проверять факты так же, как журналист
- Не брать логотип из первого попавшегося источника без верификации
- Всегда смотреть первоисточник, а не третьи руки

**§ 143. Типографика и знаки препинания:**
- Тире ≠ дефис ≠ минус — это разные знаки
- Признак хорошей вёрстки — незаметность сложных моментов
- Смайлик отбивается пробелом от слова; точка после смайлика не ставится

**§ 148. Единица смысла:**
- Дизайнер ценен ровно настолько, насколько ценны его работы
- Хорошая идея — единица смысла. Стостраничный брендбук — тоже одна единица
- Обычный хороший дизайнер: 1-2 единицы смысла в месяц

**§ 181. О творческих инструментах:**
- «У дизайнера инструмент — Фотошоп, у арт-директора инструмент — дизайнер»

### Практические правила типографики из Ководства

**Тире и дефис:**
- Дефис (`-`) — только в словах (ковёр-самолёт, по-русски)
- Тире (`—`) — в предложениях, всегда с пробелами по бокам
- Короткое тире (`–`) — для диапазонов чисел (1941–1945)
- Минус (`−`) — только в математических формулах

**Кавычки:**
- В русском тексте: «ёлочки» — не "лапки"
- Вложенные кавычки: «внешние „внутренние" кавычки»

**Пробелы:**
- Неразрывный пробел перед однобуквенными предлогами и союзами (в, с, и, а)
- Инструмент Лебедева «Типограф» автоматически расставляет всё правильно

## 4. CTR и конверсия дизайна

### 4.1 Eye-tracking исследования (Nielsen Norman Group)

**Паттерны сканирования страниц:**

**F-паттерн** (наихудший):
- Пользователи читают первые строки полностью (горизонтально)
- Затем ниже — только начало строк слева
- Итог: правая часть страницы практически не читается
- Применимо к desktop и mobile (в RTL языках — зеркальный паттерн)

**Spotted (точечный) паттерн:**
- Глаза прыгают по выделенным словам: ссылки, жирный текст, цифры, буллеты
- Лучше F-паттерна, если дизайнер правильно расставил акценты

**Layer-cake (слоёный пирог):**
- Глаза движутся по заголовкам и подзаголовкам
- После нахождения нужного заголовка — читается тело текста
- Один из лучших паттернов для структурированного контента

**Commitment (вовлечение):**
- Пользователь читает всё подряд — максимальное понимание
- Редко, только при высокой мотивации

**Практические выводы:**
- Первые строки — самые важные (туда смотрят все)
- Левая сторона важнее правой
- Заголовки и подзаголовки читают чаще, чем тело текста
- Буллеты и цифры притягивают взгляд
- Стены текста без структуры → F-паттерн → низкий CTR

---

### 4.2 Факторы CTR баннеров и обложек

**Размер и видимость:**
- Крупные баннеры имеют более высокий CTR, но вызывают banner blindness
- Оптимальный размер: не слишком мал (незаметен), не слишком велик (раздражает)
- Размещение выше сгиба (above the fold) — в 2-3 раза выше CTR

**Цвет:**
- Высококонтрастные цвета привлекают взгляд
- Красный, оранжевый, жёлтый — высокая заметность
- Синий, зелёный — доверие и спокойствие
- Тестируй A/B: один и тот же текст, разный цвет CTA-кнопки

**Текст на баннере:**
- Менее 7 слов — оптимально для баннера
- CTA (призыв к действию) обязателен: «Попробовать», «Узнать цену»
- Конкретные цифры лучше абстракций: «+127% продаж» > «увеличим продажи»

**Banner blindness (слепота к баннерам):**
- Пользователи научились не замечать элементы, похожие на рекламу
- Решения: нативный вид, неожиданные формы, анимация (умеренно), персонализация

**A/B тестирование — что проверять:**
- Цвет CTA-кнопки
- Текст заголовка (длинный vs короткий)
- Изображение (лицо человека vs абстракция)
- Позиционирование элементов
- Наличие/отсутствие CTA

---

### 4.3 Психология восприятия визуального

**Иерархия внимания:**
1. Движение (анимация)
2. Лица людей (особенно взгляд, направленный на объект)
3. Яркий контрастный цвет
4. Крупный текст
5. Логотип/бренд (при наличии узнаваемости)

**Правило 3 секунд:**
- За 3 секунды пользователь решает, интересен ли контент
- Главная мысль должна считываться мгновенно

**Закон Фиттса:**
- Чем крупнее кнопка/элемент и чем ближе она к курсору — тем быстрее по ней кликнут
- CTA-кнопка должна быть большой и на видном месте

**Социальное доказательство:**
- Рейтинги, отзывы, количество клиентов на обложке повышают конверсию
- Лица реальных людей > иллюстрации и стоковые фото

## 5. Чеклисты дизайна

### 5.1 Чеклист перед отправкой дизайна клиенту

**Технические проверки:**
- [ ] Все шрифты встроены или конвертированы в кривые
- [ ] Разрешение: 72 dpi для экрана, 300 dpi для печати
- [ ] Цветовое пространство: RGB для экрана, CMYK для печати
- [ ] Размер соответствует ТЗ
- [ ] Файл называется понятно: `ClientName_LogoV2_2024-03-25.pdf`
- [ ] Нет скрытых слоёв с тестовыми элементами
- [ ] Экспорт протестирован (открывается у клиента без потерь)

**Визуальные проверки:**
- [ ] Орфография и пунктуация проверены
- [ ] Правильные кавычки («ёлочки» для рус, "..." для eng)
- [ ] Тире, дефис, минус использованы корректно
- [ ] Выравнивание элементов по сетке
- [ ] Отступы одинаковые и кратны 4/8px
- [ ] Иерархия: заголовок > подзаголовок > текст читается визуально
- [ ] Достаточный контраст текст/фон (минимум 4.5:1)
- [ ] Логотип не искажён (пропорции не нарушены)

**Смысловые проверки:**
- [ ] Основная идея считывается за 3 секунды
- [ ] CTA (призыв к действию) присутствует и заметен
- [ ] Факты, цены, контакты проверены на актуальность
- [ ] Нет чужих торговых марок без разрешения
- [ ] Изображения имеют лицензию для коммерческого использования

---

### 5.2 Чеклист качества баннера/обложки

**До начала работы:**
- [ ] Понятна целевая аудитория (кто смотрит)
- [ ] Понятна цель (клик, запоминание, доверие)
- [ ] Технические требования площадки (размер, формат, вес)
- [ ] Есть брендбук или хотя бы примеры стиля

**Визуальная иерархия:**
- [ ] 1 главный элемент — он должен цеплять первым
- [ ] Максимум 2-3 шрифта (лучше 1-2)
- [ ] Максимум 3 цвета (основной + 2 дополнительных)
- [ ] Пустое пространство (whitespace) не заполнено мусором

**Читаемость:**
- [ ] Текст читается в миниатюре (thumbnail)
- [ ] Минимальный размер текста: 14px для экрана
- [ ] Контраст текст/фон проверен
- [ ] Текст не перекрывает ключевые визуальные элементы

**Контент:**
- [ ] Есть конкретное предложение (что? за сколько? для кого?)
- [ ] Есть призыв к действию
- [ ] Нет лишних деталей — каждый элемент зарабатывает своё место
- [ ] Соответствие тематике площадки (стиль платформы)

**Финальная проверка:**
- [ ] Просмотрен на разных устройствах (десктоп, мобайл)
- [ ] Просмотрен в уменьшенном виде (как в каталоге)
- [ ] Спрошено мнение незаинтересованного человека
- [ ] Сравнён с топ-конкурентами на той же площадке

## 6. Универсальные принципы профессионального дизайна

### Правила типографики

- Максимум **2 гарнитуры** в одном проекте (заголовочная + текстовая)
- Межстрочный интервал для текста: **1.4–1.6**
- Для заголовков: **1.1–1.2**
- Длина строки: **60–75 символов** — оптимально для читаемости
- Выравнивание по левому краю > по ширине (для экранов)

### Правила цвета

- Правило 60-30-10: 60% основной цвет, 30% дополнительный, 10% акцентный
- Тестируй на colorblind (8% мужчин — дальтоники)
- Не используй только цвет для передачи смысла — добавляй форму/текстуру
- Инструменты: Coolors.co, Adobe Color, Color Hunt

### Правила сетки

- **8px grid** — современный стандарт (все отступы кратны 8)
- Или 4px micro-grid для малых отступов
- Консистентные поля (margin) на всём макете
- Направляющие — не украшение, а структура

### Принципы гештальта в дизайне

- **Близость** — рядом = связано (Лебедев §136)
- **Сходство** — похожие элементы воспринимаются как группа
- **Продолжение** — взгляд следует по линии/кривой
- **Замкнутость** — мозг достраивает незаконченные фигуры
- **Фигура/фон** — элемент выделяется на фоне

### Правила для цифрового дизайна

- Mobile-first: проектируй с маленького экрана
- Минимальный размер кнопки: 44×44px (Apple), 48×48dp (Google)
- Загрузка: важнее всего первые 3 секунды (LCP < 2.5s)
- Ретина-дисплеи: готовь @2x и @3x версии изображений
