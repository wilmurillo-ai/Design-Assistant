# Weekly Report Generator

## Overview
Turn simple bullet points into professional weekly reports in seconds. Stop staring at blank documents - get polished, manager-ready updates instantly.

## When to Use
- User needs to write a weekly status report
- User mentions "weekly report", "status update", "weekly summary"
- User wants to summarize their week's work
- User is preparing for a team meeting
- User needs to document project progress

## How It Works

### Input
Simple bullet points of what you did this week:
```
- Finished the API integration
- Fixed 3 bugs in the login flow
- Had a meeting with the design team
- Started working on the new feature
```

### Output
Professional weekly report with proper structure, professional language, and impact framing.

## Report Structure

### 1. Summary
High-level overview of the week's achievements

### 2. Completed Tasks
What was accomplished with impact statements

### 3. In Progress
Current work and expected completion

### 4. Blockers
Issues preventing progress (if any)

### 5. Next Week
Planned activities and goals

```
[One sentence overview of the week]

### Completed
✅ [Task 1 with impact]
✅ [Task 2 with impact]
✅ [Task 3 with impact]

### In Progress
🔄 [Project] - [Status] ([Percent]% complete)

### Blockers
⚠️ [Issue] - [Mitigation plan]

### Next Week
- [Goal 1]
- [Goal 2]
```

### Brief Update
```
**This Week:**
- [Key achievement 1]
- [Key achievement 2]

**Next Week:**
- [Priority 1]
- [Priority 2]

**Needs:**
- [Any support required]
```

## Usage Guidelines

1. **List what you did** - Simple bullet points are fine
2. **Be honest** - We'll frame it professionally
3. **Include numbers when possible** - Metrics add credibility
4. **Mention blockers** - Shows proactive communication
5. **Preview and adjust** - Make it yours before sending

## Tips for Better Reports

- Start with impact, not activity
- Use numbers and percentages
- Show progress, not just completion
- Flag blockers early
- Set clear next week goals
