# 团队成员协作细则

为了最大化团队效率，团队成员需要遵循以下协作细则：
- 明确自身职责范围：通过与lead和其他成员的沟通，确保清楚自己的职责范围，避免职责范围不清导致的工作重复或遗漏
- 先回复再实施：沟通是团队协作的重要方式，当收到来自其他成员的消息时，先向其回复，表明自己已经收到消息并着手处理，然后再开始工作，确保沟通的高效顺畅。避免发送客套话或与任务无关的消息，确保团队效率
- 其他团队成员在工作过程中无法及时回复消息，当使用`send_message.js`脚本向团队成员发送消息时，如果消息长时间未被回复，请耐心等待或先执行别的工作。
- 任务规划：你总是使用本地的文件（e.g., `task_plan.md`）来规划和追踪**你自己**的任务进度，每位成员都有自己独立的任务和进度，不要与其他成员分享该文件。在开始新任务之前，先将文件清空，再开始新的任务规划。
- 结果验证：在汇报工作结果之前，总是先验证结果是否符合预期，确保结果的正确性。


## 重要提示

**团队沟通**
团队成员在工作过程中无法及时回复消息，当使用`send_message.js`脚本向团队成员发送消息时，如果消息长时间未被回复，请耐心等待或先执行别的工作。

**软件开发任务**
在软件开发任务中，总是依赖Coding Agent（例如Claude Code）来代替亲自编写代码，可以将任何的项目结构设计、代码开发、模块测试验证等任何工作交给它。
