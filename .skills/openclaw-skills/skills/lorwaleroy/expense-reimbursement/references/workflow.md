# 报销流程详解 v2.0

## 与v1.0的核心差异

| 项目 | v1.0 | v2.0 |
|------|------|------|
| 票据类型 | 仅高德打车 | 高德、滴滴、火车票、机票、住宿等 |
| 行程类型 | 不区分 | 出差/公出自动判断 |
| 专项项目 | 写死模板 | 先询问用户再填写 |
| 扫描方式 | 手动列ZIP | 递归扫描所有文件含ZIP内 |
| 出发地判断 | 行程单地址 | XML SellerAddr为主 |
| 路径配置 | 硬编码 | 环境变量可配置 |
| 平台支持 | macOS | macOS + Linux + Windows |

---

## 实战流程

### Step 0：读取申请截图

OCR所有截图，提取行程信息：
```
截图1 → 03-11无锡出差，2天，判断：出差
截图2 → 03-21哈尔滨公出，1天，判断：公出
截图3 → 03-23哈尔滨出差，1天，判断：出差
截图4 → 03-30北京出差，1天，判断：出差
截图5 → 04-02上海出差，1天，判断：出差
```

### Step 1：递归扫描票据

```python
# 扫描 ~/报销/ 下所有文件
# 含ZIP内部文件
# 建立清单：文件路径、类型（打车/火车/住宿/其他）
```

### Step 2：识别票据内容

**打车票据**（含XML）：
- 解析XML：SellerName, SellerAddr, Amount
- SellerAddr末级城市 = 真实打车城市

**火车票PDF**：
- OCR提取：出发站、到达站、时间、金额

### Step 3：按行程归类

```
03-11无锡出差/
  交通票据/
    01_出发打车/T3¥65.60（杭州东站出发）
    02_火车票/G7592 ¥150
    03_返程打车/腾飞¥54.02（到杭州东站）
    返程火车/G7585 ¥156
```

### Step 4：填写表单

1. 确认专项项目（询问用户）
2. 根据申请截图判断出差/公出
3. 只写有金额的项目，0值省略
