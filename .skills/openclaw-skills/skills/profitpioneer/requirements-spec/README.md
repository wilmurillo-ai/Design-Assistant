# Requirements Spec Skill

A comprehensive framework for writing complete, measurable, and actionable requirements documents. This skill guides you through an 8-dimension evaluation system to ensure your PRDs and requirements specs are thorough, clear, and ready for development.

## When to Use

- Drafting a new product requirements document (PRD)
- Writing or structuring a requirements specification
- Reviewing an existing requirements document for completeness
- Ensuring requirements meet professional quality standards

## Features

- **8-Dimension Framework** — Covers Background (15%), Objectives (20%), Scope (20%), Detailed Features & Flows (20%), Non-Functional Requirements (5%), Data Tracking (10%), Acceptance Criteria (5%), and Risk Assessment (5%)
- **Built-in Checklists** — Each dimension includes a verification checklist to catch gaps before they become problems
- **SMART Objectives** — Ensures every goal is Specific, Measurable, Achievable, Relevant, and Time-bound
- **Quality Scoring** — Automatic quality estimation: Excellent (8.0+), Good (6.0-7.9), Acceptable (4.0-5.9), Needs Work (<4.0)
- **Common Pitfalls Guide** — Avoids vague goals, unclear scope, missing acceptance criteria, and ignored risks

## Quick Start

Just ask to write, draft, review, or structure a requirements document, PRD, or product spec. The skill will guide you through each dimension step by step.

## Example Output Structure

```
# [Requirement Name]

## 1. Background
## 2. Objectives
## 3. Scope
## 4. Detailed Features & Flows
## 5. Non-Functional Requirements
## 6. Data Tracking
## 7. Acceptance Criteria
## 8. Risk Assessment
```
