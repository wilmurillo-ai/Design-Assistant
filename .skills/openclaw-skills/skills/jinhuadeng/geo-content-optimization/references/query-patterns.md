# Query Patterns

Use these patterns to shape headings, FAQ blocks, and comparison content.

## Definition queries
- What is X?
- What does X do?
- What is X used for?

## Comparison queries
- X vs Y
- Best X for ...
- X alternatives
- Is X better than Y?

## Commercial research queries
- Is X worth it?
- How much does X cost?
- Who should use X?
- Best tool for [role/use case]

## Implementation queries
- How to use X
- How to implement X
- Common mistakes with X
- X checklist

## GEO mapping tip

A strong page should usually support at least 3 query types:
1. definition
2. comparison
3. implementation
