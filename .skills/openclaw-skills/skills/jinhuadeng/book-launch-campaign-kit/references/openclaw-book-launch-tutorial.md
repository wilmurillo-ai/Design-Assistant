# OpenClaw Book Launch Tutorial

## Goal
Turn one book cover and one core positioning line into a complete English launch system:
- posters
- promo videos
- Amazon conversion copy
- people-holding-book ad concepts
- polished campaign videos
- sales conversion videos
- launch pack documentation

This tutorial summarizes the workflow used to build the OpenClaw book launch assets.

## Source Assets
Primary source files used in the workflow:
- Book cover PNG/JPG
- English manuscript and launch copy
- Existing launch folder for saving outputs

Recommended project structure:
- `en/` for English book files
- `launch-assets/` for image/video outputs
- `openclaw-launch-pack/` for copywriting and metadata packs

## Phase 1 — Lock Positioning
Start with one strong line:

**OpenClaw: Build Your Own AI Agent That Actually Gets Things Done**

Then derive supporting hooks:
- Most AI can talk. Very few can execute.
- Stop building demos. Start building execution.
- From AI chat to AI action.

Do not create assets before the positioning is stable.

## Phase 2 — Build Poster Set
Create 3 static poster sizes first:
- vertical poster for mobile / stories / shorts cover
- square poster for social posts
- horizontal poster for article hero / GitHub / web header

Poster system should include:
- core title
- short supporting line
- clear CTA
- book cover shown prominently

## Phase 3 — Build English Promo Videos
Start with simple English promo videos:
- 30-second explainer version
- 15-second punchy version
- 15-second improved voiceover version

Recommended flow:
1. write short script
2. generate subtitles
3. synthesize voiceover
4. combine poster or ad stills with motion zoom
5. export vertical MP4

## Phase 4 — Improve Voice Quality
If local/offline TTS sounds too robotic, switch to better neural TTS.
A practical, low-cost solution is Edge neural TTS.

Optimization knobs:
- voice selection
- rate
- pitch
- shorter punchier copy

## Phase 5 — Build Launch Copy Pack
Create one launch pack folder containing:
- core launch copy
- platform posts
- asset index
- launch checklist
- Amazon / Kindle conversion pack
- Amazon ready-to-publish pack

This package becomes the control center for release.

## Phase 6 — Build People-Holding-Book Concepts
When static book-cover posters are not enough, generate stronger ad directions.
Use 4 concept families:
1. tech founder style
2. international star style
3. business magazine style
4. movie poster style

Important rule:
Use celebrity-level energy, not exact real celebrity endorsement.

For each style, prepare:
- English image prompt
- negative prompt
- composition guidance
- headline suggestion

## Phase 7 — Turn Concepts into Campaign Video
Convert the strongest people-holding-book visuals into video.

Recommended workflow:
1. render mockups or concept stills
2. choose the strongest directions
3. add light motion (zoom/pan)
4. add subtitles
5. add English voiceover
6. add soft background music for polish

Recommended strongest pair from this workflow:
- International Star Style
- Movie Poster Style

## Phase 8 — Split Brand vs Conversion
Do not use one video for every goal.
Create two lanes:

### Brand lane
Use more premium, emotional, high-aesthetic visuals.
Goal: attention and perceived quality.

### Conversion lane
Use clearer promises and stronger CTA.
Goal: explain why to read/buy now.

## Phase 9 — Amazon Packaging
Prepare final Amazon-ready metadata:
- title
- subtitle
- long description
- short description
- benefit bullets
- backend keywords
- category direction
- A+ copy angles

Core positioning for Amazon:
**This is not another AI theory book. It is a practical guide to execution.**

## Recommended Final Deliverables
Minimum viable launch stack:
- 1 square poster
- 1 vertical poster
- 1 polished brand video
- 1 sales conversion video
- 1 launch copy pack
- 1 Amazon ready-to-publish pack

## Reusable Lessons
1. Lock positioning before design.
2. Make short-form video early.
3. Upgrade voice quality quickly.
4. Separate brand and conversion assets.
5. Turn strong still concepts into moving ads.
6. Package metadata and creative in one place.

## Publishing Workflow
Once assets are complete:
1. export final files into a stable project folder
2. create launch pack documentation
3. package the workflow as a reusable skill
4. publish to GitHub
5. publish the skill to ClawHub

## Output Expectation
After following this workflow, the user should have a repeatable system for launching an English book with:
- visuals
- videos
- copy
- Amazon listing materials
- reusable process documentation
