# Workflow

## End-to-end pipeline

1. Input arrives as one of:
   - audio
   - transcript
   - meeting minutes
   - raw notes
2. Normalize into clean source text
3. Extract a compact article brief
4. Choose article mode and target audience
5. Draft WeChat article
6. Add image placement notes
7. Save markdown with frontmatter
8. If requested, publish to WeChat draft box

## Best practice

- Audio should first become text before article writing.
- If the source is long, compress into a brief before drafting.
- Preserve only useful quotes, not entire transcript flow.
- Keep article centered on one core message.

## Typical user requests this skill should handle

- 这段录音帮我整理成公众号文章
- 根据会议录音写一篇可发文章
- 用这段文字稿加几张图，生成公众号文章
- 录音转文字后直接出公众号成稿
- 带二维码，做成可发布稿
