## 系统概述

news Hot Hub 是一个中文热点数据聚合技能，整合知乎、微博、今日头条、V2EX、36氪、AIBase 六大平台的热搜/热门数据。采用「调度器 + 独立脚本」的插件式架构，每个平台是一个独立的 Python 脚本，由统一的 `hub.py` 调度。

### 设计原则

- **单一职责**：每个平台脚本只负责一个平台的数据获取
- **统一接口**：所有脚本遵循相同的 CLI 参数规范和 JSON 输出格式
- **渐进实现**：V1 先实现知乎，其他平台预留脚手架，V2 逐步补全
- **无状态**：脚本不持久化数据，每次调用独立获取

## 架构设计

```
┌─────────────────────────────────────────────────┐
│                   AI Agent                       │
│  (根据用户意图选择 fetch/all/compare/status)       │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│               hub.py  (调度器)                    │
│                                                   │
│  ┌─────────┐  ┌─────────┐  ┌──────────────────┐ │
│  │ fetch   │  │  all    │  │    compare       │ │
│  │ (单/多) │  │ (并行)  │  │  (词频分析)      │ │
│  └────┬────┘  └────┬────┘  └────────┬─────────┘ │
│       │            │                │            │
│       ▼            ▼                ▼            │
│  ┌──────────────────────────────────────────┐   │
│  │        Platform Registry (注册表)         │   │
│  │  zhihu | weibo | toutiao | v2ex | 36kr | aibase   │   │
│  └──────────────────┬───────────────────────┘   │
└─────────────────────┼───────────────────────────┘
                      │  subprocess.run()
          ┌───────────┼───────────────────┐
          ▼           ▼                   ▼
    ┌──────────┐ ┌──────────┐      ┌──────────┐
    │ zhihu.py │ │ weibo.py │ ...  │ aibase.py │
    └──────────┘ └──────────┘      └──────────┘
          │           │                   │
          ▼           ▼                   ▼
    ┌──────────┐ ┌──────────┐      ┌──────────┐
    │ 知乎 API │ │ 微博 API │      │ AIBase公开│
    └──────────┘ └──────────┘      └──────────┘
```

### 调度器 (hub.py)

- 解析 CLI 参数，路由到对应命令处理函数
- 通过 `subprocess.run()` 调用各平台脚本，实现进程隔离
- `all` 和 `compare` 命令使用 `ThreadPoolExecutor` 并行调度
- 统一处理超时、错误、JSON 解析

### 平台脚本 (zhihu.py, weibo.py, ...)

- 每个脚本是独立可执行的 CLI 工具
- 输入：CLI 参数（command, --limit, --query 等）
- 输出：JSON 到 stdout，错误信息到 stderr
- 退出码：0 成功，1 失败

## 数据流

```
用户输入 → Agent 解析意图 → hub.py 调度
  → 平台脚本获取数据 → JSON stdout
  → hub.py 收集结果 → 输出给 Agent → 展示给用户
```

### fetch 流程

1. 解析平台名（支持别名和逗号分隔）
2. 顺序调用各平台脚本
3. 逐行输出 JSON Lines

### all 流程

1. 并行启动 6 个平台脚本（ThreadPool, max_workers=5）
2. 逐个收集完成结果
3. 逐行输出 JSON Lines

### compare 流程

1. 并行获取全部平台数据
2. 提取各平台标题列表
3. 分词统计关键词词频（中文 ≥2 字 / 英文 ≥2 字符）
4. 计算跨平台出现次数
5. 输出排序后的词频分析 JSON

## 平台注册表

hub.py 中的 `PLATFORMS` 字典定义了所有已注册平台：

| 字段          | 说明                        |
| ------------- | --------------------------- |
| `name`        | 平台中文名（用于输出展示）  |
| `script`      | scripts/ 目录下的脚本文件名 |
| `default_cmd` | 默认子命令                  |
| `commands`    | 支持的子命令列表            |

别名映射在 `ALIASES` 字典中，支持中文名和英文缩写。

## 扩展新平台

添加新平台只需 3 步：

1. 在 `scripts/` 下新建 `<platform>.py`，遵循标准 CLI 接口（见 `platform-guide.md`）
2. 在 `hub.py` 的 `PLATFORMS` 字典中注册
3. 在 `ALIASES` 中添加别名

无需修改调度逻辑，hub.py 会自动发现和调用。
