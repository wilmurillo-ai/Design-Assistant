## 1. 产品概述

### 1.1 产品愿景
打造一个人人可用的个人健康顾问，通过智能穿戴设备数据 + AI知识库，为用户提供个性化的健康洞察和生活方式建议。

### 1.2 目标用户
- **主要用户：** 健康意识较强的个人用户（25-45岁）
- **典型特征：**
  - 已拥有智能穿戴设备（Apple Watch、Garmin、Fitbit等）
  - 希望通过数据驱动改善健康
  - 有具体健康目标（减肥、改善睡眠、缓解压力等）
  - 不需要复杂医疗系统，但要专业建议

### 1.3 核心价值
1. **数据驱动：** 基于真实穿戴设备数据，不是空泛建议
2. **个性化：** 根据用户健康关切定制建议
3. **专业可信：** 基于文献的医学级建议（由NotebookLM提供）
4. **零使用门槛：** 配置简单，日常自动推送

### 1.4 市场定位
- **差异化：**
  - 对比智能设备自带App：更深入的分析，医学级建议
  - 对比健康管理App：不需要手动记录，自动同步
  - 对比AI医生：不替代医疗，专注生活方式干预

## 3. 产品功能规划

### 3.1 MVP功能范围

#### 核心功能（必须）
1. **用户配置系统**
   - 首次对话收集健康关切
   - 穿戴设备连接配置
   - 报告推送时间设置

2. **数据获取模块**
   - 支持Garmin设备（高优先级）
   - 睡眠、步数、HRV、压力、身体电量、运动强度
   - 基础错误处理和重试

3. **AI建议生成**
   - 基于用户健康关切的个性化提示词
   - 调用NotebookLM生成建议
   - 结构化输出（数据洞察 + 睡眠/压力 + 运动 + 饮食）

4. **报告推送**
   - 每日定时推送
   - 飞书消息格式化
   - 支持其他IM平台（微信、Telegram）

5. **历史数据存储**
   - 本地JSON文件存储
   - 保留最近30天数据
   - 支持趋势查询

#### 次要功能（V1.0）
1. 支持Apple Watch（通过Apple Health API）
2. 支持Fitbit
3. 主动查询历史趋势
4. 用户反馈收集（建议是否有用）

#### 3.2.3 AI建议生成模块

**提示词动态生成：**

根据用户健康关切动态组装提示词模板：

```python
def generate_prompt(user_profile, health_data, history):
    base_template = load_base_prompt()

    # 根据用户关切选择子模块
    concern_modules = []
    if "改善睡眠" in user_profile.health_concerns:
        concern_modules.append(load_module("sleep_quality"))
    if "缓解压力" in user_profile.health_concerns:
        concern_modules.append(load_module("stress_management"))
    if "尿酸高" in user_profile.specific_conditions:
        concern_modules.append(load_module("uric_acid_management"))

    # 组装完整提示词
    prompt = base_template.format(
        user_concerns=format_concerns(user_profile),
        health_data=format_data(health_data),
        history=format_history(history),
        analysis_requirements="\n".join(concern_modules),
        privacy_rules=load_privacy_rules()
    )

    return prompt
```

## 5. 隐私与数据安全

### 5.1 数据类型

**本地存储的数据：**
- ✅ 穿戴设备健康数据（睡眠、步数、HRV等）
- ✅ 用户健康关切配置
- ✅ 穿戴设备API凭证（加密存储）
- ❌ **不存储：** 个人身份信息（姓名、电话、邮箱、地址）

**发送到NotebookLM的数据：**
- ✅ 脱敏的健康数据（无具体诊断）
- ✅ 用户健康关切（泛化描述）
- ❌ **不发送：** 任何个人身份信息、具体医疗诊断

### 5.2 隐私保护措施

1. **数据本地化：** 所有健康数据存储在用户本地，不上传到云端
2. **凭证加密：** 穿戴设备API凭证使用加密存储
3. **输出脱敏：** AI建议中不包含具体诊断和指标数值
4. **用户控制：** 用户可随时删除配置和历史数据
5. **透明性：** 清晰说明数据用途和存储位置

### 5.3 合规性考虑

- **符合GDPR：** 数据本地化，用户完全控制
- **符合HIPAA：** 不涉及医疗诊断，仅生活方式建议
- **中国网络安全法：** 数据不出境，本地存储

## 8. 风险与应对

### 8.1 技术风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 穿戴设备API变更 | 高 | 抽象层设计，易于适配 |
| NotebookLM不可用 | 中 | 降级到本地规则引擎 |
| IM平台API限制 | 低 | 多平台支持，降低单点依赖 |

### 8.2 产品风险

| 风险 | 影响 | 应对 |
|------|------|；------|
| 建议过于医学化，用户看不懂 | 中 | A/B测试，优化提示词 |
| 用户担心隐私，不愿使用 | 中 | 强化隐私保护说明，强调本地存储 |
| 配置复杂，用户放弃 | 高 | 简化配置流程，提供详细文档 |

### 8.3 合规风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 涉及医疗诊断，需要医疗资质 | 高 | 明确免责声明，只提供生活方式建议 |
| 数据跨境问题 | 中 | 强调数据本地化，不出境 |
