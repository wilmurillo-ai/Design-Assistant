***"Every morning, a personalized wellness briefing delivered by your private concierge."***

This is an intelligent life assistant that combines **wearable device data** (e.g., your HRV, sleep duration) with the powerful **Google NotebookLM AI**. Instead of creating body anxiety with cold charts, it tells you: *"You didn't get enough deep sleep last night, consider grabbing a latte before your afternoon meeting."*

## ✨ Core Values

- 🧠 **End Data Anxiety**: Translates complex biometric data into "human-readable," specific suggestions.
- 🎯 **Hyper-Focused Goals**: Your report only cares about what matters to you (e.g., fat loss, stress management, joint health).
- 📑 **Backed by Medical Science**: Leverages NotebookLM's powerful literature analysis capabilities rather than generic search engines.
- ☕ **Seamless Automation**: Set it up once, and receive it every morning at 8 AM alongside your morning coffee.

## 📋 Prerequisites

1. **OpenClaw**: Installed and configured with at least one IM platform (Telegram/Discord/WeChat/Feishu).
2. **NotebookLM CLI**: Installed and authorized with Google.
   ```bash
   pipx install notebooklm-py
   notebooklm login
   ```
3. **智能穿戴设备**：目前支持Garmin设备

## 🚀 1-Click Private Health Coach

### 1. Install from App Market
Just run this gently in your terminal:
```bash
clawhub-install <your-github-username>/health-assistant
```
*(The system will automatically install the necessary AI engine environment for you, no need to worry about dependencies!)*

### 2. Authorize Your Wearable Data
We need to read the past few days' data to establish a baseline. For Garmin users:
```bash
# Run the official authentication command in your system terminal (only once)
garth login
```
*Your credentials are safely stored in your local system (`~/.garth`), this assistant only requests read permissions.*

### 3. First Use

Send a message via your IM platform (e.g., Telegram) to OpenClaw:

```text
report
```
If this is your first time, the skill will automatically intercept and guide you through an interactive setup process:

```
🦞 Health Assistant: Hello! I am your personal health advisor. Checking environment... 🔍

...
Now, I'd like to know what health aspects you care about the most (you can select multiple):
A. Improve Sleep
B. Weight Loss / Management
C. Stress Relief
D. Boost Energy
...
```

### 4. Configure Scheduled Delivery

Edit OpenClaw's cron jobs:

```yaml
# Example: Push report every morning at 8:30
name: "daily-health-report"
schedule: "30 8 * * *"
command: "/path/to/health-assistant/venv/bin/python /path/to/health-assistant/src/main.py --daily-report"
```

## 🔒 Privacy & Security Commitment

We understand the paramount importance of health data and account security:

- **No Cloud Data Transfer**: All health signs are flowed and analyzed natively in your local environment. Aside from the official NotebookLM/Garmin API pathways, NO data is uploaded to third-party servers.
- **Credential Sandbox**: Device authentication is handled by the official `garth` tool in a standalone `~/.garth` environment. The config file leverages OS-level strict permissions (`0o600`, reading/writing strictly restricted to the owner) for local isolation.
- **De-identified AI Analysis**: The metrics provided to the LLM are stripped of identity boundaries, maximizing privacy protection.
- **Full Control**: You can always say `reset` to clear all local histories and preferences.

## 🛠️ Troubleshooting

### Issue: Cannot connect to Garmin

**Solution:**
```bash
# Re-authenticate locally
garth login
```

### Issue: NotebookLM queries failing

**Solution:**
```bash
# Check NotebookLM status
notebooklm list

# If not logged in, authorize again
notebooklm login
```
