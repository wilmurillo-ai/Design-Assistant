# Seers 用户指南 (中文)

## Seers是什么?

Seers 是一个多链去中心化预测市场平台，允许用户：

- 创建预测事件（支付 1000 USDT 作为保证金）
- 参与事件交易（买卖 YES/NO 头寸）
- 提交裁定证据
- 基于事件结果获得奖励

Seers 利用资金池式的市场机制对未来事件进行预测和结算，用户投注形成不同选项的资金池，最终按照投注在正确选项资金池中的占比分配奖励。

## Seers 是平台庄家吗?

不是。Seers 不是庄家。

在 Seers 上，用户才是核心驱动者——

- 用户负责创建事件
- 用户承担事件的保证金
- 用户可以从事件的交易手续费中获利

Seers 官方只承担：

- 仲裁
- 安全维护
- 链上机制执行

平台保持严格中立，不会对市场结果进行投注或操控，仅在极少数争议事件需要二次裁决时介入，以维护整体公平和用户利益。

## 事件如何创建?

任何用户都可以创建事件，只需：

1. 连接支持的钱包（如 MetaMask / Phantom）
2. 选择链（Solana / Polygon / Ethereum 等）
3. 设置：
   - 事件标题
   - 描述
   - 规则与裁定标准
   - 到期时间

为满足不同需求，我们提供两种创建方式：
- 传统模式：无需支付保证金，适合大多数普通事件创建。
- 庄家模式：创建者需支付一笔可配置的保证金，以激励提供更高质量的事件并维护平台生态。

## 为什么要交保证金?

保证金用于：

- 防止恶意事件创建
- 提供初始市场信誉
- 用于争议奖励或作废事件时的补偿
- 确保参与者对事件有真实成本参与

保证金会在事件最终结算后自动退还（除非事件违规或出现争议成本）。

## 如何交易?

在任意事件中，你可以选择：

- YES（认为事件会发生）
- NO（认为事件不会发生）

购买对应选项的份额并加入资金池。市场会显示当前各选项的资金占比，最终结算时，赢家按照自己在正确资金池中的份额占比获得奖励。

事件结束时：

- 预测正确 → 获得 payout
- 预测错误 → 头寸归零

## 谁决定事件结果?

Seers 采用混合仲裁机制：

### 1. 用户证据提交（Evidence Submission）

任何人都可以上传：

- 链上数据截图
- 官方公告
- 新闻媒体
- 赛事结果
- 可信第三方信息

### 2. 官方仲裁（Final Arbitration）

Seers 仲裁团队在链上公布最终裁定。

这是为了避免：

- 恶意事件创建者
- 无关第三方操纵
- 数据伪造

## Seers 能操控事件结果吗?

不能。

Seers 的职责是：

- 执行链上规则
- 确保障机制公平透明
- 提供最终仲裁

Seers 不参与市场交易、不对事件下注，仅通过收取合理的手续费（抽水）维持平台运营和可持续发展。

## 支持哪些链?

当前支持：

- Solana
- Polygon
- Ethereum
- BSC
- Base
- BTC（部分功能）

更多链会持续上线。

## 可以创建哪些事件?

只要事件有明确、可验证的结果即可，例如：

- 加密市场（BTC 是否突破 100k）
- 体育赛事
- 科技发布与产品事件
- 政策或社会趋势
- 实时热点话题
- 加密生态数据（Gas、TVL、活跃用户等）

禁止：

- 涉及真人隐私
- 高风险政治煽动性内容
- 无法客观验证的内容

Seers 官方有权在极端情况下拒绝或终止违规事件。

## 什么是 Seers?

Seers 是一个多链去中心化预测市场平台，允许用户创建事件、交易未来结果，并根据事件最终裁定获得收益。

平台采用公平的资金池机制：用户将资金投入YES或NO选项对应的资金池中，所有资金由智能合约托管和管理。事件结束后，由平台进行公正裁定，正确选项的资金池（扣除手续费后）将按用户投注占比分配给所有赢家。
