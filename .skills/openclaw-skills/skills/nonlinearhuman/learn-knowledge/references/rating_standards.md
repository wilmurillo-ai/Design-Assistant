# 能力评级标准详解

本文档详细说明能力评级的评分标准和方法。

---

## 评分维度详解

### 1. 广度 (Breadth)

**定义**：对核心子领域、关键术语的覆盖比例

**评分方法**：

```
广度分数 = (已覆盖的核心子领域数 / 总核心子领域数) × 100
```

**核心子领域识别**：

| 领域类型 | 典型子领域示例 |
|----------|----------------|
| 技术领域 | 核心概念、主要方法、工具框架、应用场景、前沿进展 |
| 理论领域 | 基础理论、主要流派、代表学者、经典著作、当代发展 |
| 应用领域 | 核心功能、主要产品、市场格局、用户场景、发展趋势 |

**评分指南**：

| 分数范围 | 覆盖程度 | 描述 |
|----------|----------|------|
| 90-100 | 全面覆盖 | 所有核心子领域都有涉及 |
| 70-89 | 较好覆盖 | 主要子领域覆盖，少量次要领域缺失 |
| 50-69 | 基本覆盖 | 核心子领域覆盖，但不够全面 |
| 30-49 | 部分覆盖 | 仅覆盖部分核心内容 |
| 0-29 | 覆盖不足 | 仅涉及表层概念 |

---

### 2. 深度 (Depth)

**定义**：对核心原理、技术细节、内在矛盾的理解深度

**评分方法**：

通过知识框架的深度层级来评估：

```
深度分数 = (达到的深度层级 / 最大深度层级) × 100
```

**深度层级定义**：

| 层级 | 深度类型 | 典型问题 |
|------|----------|----------|
| L1 | 定义级 | "是什么？" |
| L2 | 原理级 | "怎么工作？" |
| L3 | 机制级 | "为什么这样设计？" |
| L4 | 细节级 | "数学推导是什么？" |
| L5 | 前沿级 | "有哪些未解决问题？" |

**评分指南**：

| 分数范围 | 深度层级 | 描述 |
|----------|----------|------|
| 90-100 | L4-L5 | 能讨论前沿问题和深层细节 |
| 70-89 | L3-L4 | 理解设计原理和技术细节 |
| 50-69 | L2-L3 | 理解工作机制，但原理不够深入 |
| 30-49 | L1-L2 | 知道是什么，但不理解原理 |
| 0-29 | L1 | 仅停留在定义层面 |

---

### 3. 置信度 (Confidence)

**定义**：所学信息的平均交叉验证可信度

**评分方法**：

```
置信度分数 = Σ(信息源可信度 × 权重) / Σ权重
```

**信息源可信度评分**：

| 来源类型 | 基础分 | 说明 |
|----------|--------|------|
| 顶级学术论文 (Nature, Science, 顶会) | 95 | 同行评审，权威认证 |
| arXiv预印本 | 85 | 学术前沿，未经同行评审 |
| 官方文档/GitHub | 80 | 一手资料，可能不完整 |
| 权威技术博客 | 70 | 专业解读，可能有偏见 |
| 行业报告/白皮书 | 65 | 专业研究，可能有商业倾向 |
| 教程/Medium/知乎 | 50 | 实践经验，质量参差不齐 |
| 用户论坛/问答 | 30 | 非权威，需验证 |

**交叉验证加分**：

- 同一信息有2个独立来源验证：+10分
- 同一信息有3个以上来源验证：+15分
- 信息来自冲突来源（需说明）：-10分

---

### 4. 结构化程度 (Structure)

**定义**：知识被逻辑化、关联化的程度

**评分方法**：

基于知识图谱的关系密度和框架的完整性：

```
结构化分数 = (关系数量 / 实体数量) × 50 + 框架完整度 × 50
```

**框架完整度评估**：

| 检查项 | 分值 |
|--------|------|
| 有明确的层次结构 | 20 |
| 每层有清晰的内容组织 | 20 |
| 层级之间有逻辑关联 | 20 |
| 知识图谱有关系连接 | 20 |
| 问答对覆盖不同难度 | 20 |

---

## 综合评级算法

### 加权平均法

```
综合分数 = 广度 × 0.25 + 深度 × 0.30 + 置信度 × 0.25 + 结构化 × 0.20
```

**权重说明**：
- 深度权重最高(30%)：技术能力更看重理解深度
- 广度和置信度次之(各25%)：需要全面且可靠的知识
- 结构化权重最低(20%)：影响检索效率但不影响正确性

### 等级映射

| 综合分数 | 能力等级 |
|----------|----------|
| 85-100 | 等级4：专家级 |
| 70-84 | 等级3：深度掌握 |
| 50-69 | 等级2：系统理解 |
| 30-49 | 等级1：概念入门 |
| 0-29 | 需要进一步学习 |

## 能力边界声明生成规则

### 正向能力描述

根据综合分数和深度分数，列出具体能做到的事情：

**等级4**：
- 可参与学术辩论
- 可识别研究空白
- 可提出创新思路
- 可批判性分析论文

**等级3**：
- 可进行技术选型
- 可设计初步方案
- 可分析优劣权衡
- 可回答复杂技术问题

**等级2**：
- 可解释核心概念
- 可对比不同方案
- 可提供使用建议
- 可回答常见问题

**等级1**：
- 可解释基本术语
- 可提供入门指引
- 可推荐学习资源
- 可回答"是什么"问题

### 限制描述

根据缺失的子领域和不足的深度，说明能力边界：

**模板**：
```
在{子领域A}方面，我的能力有限，因为{原因}。
建议{后续行动}。
```

**示例**：
```
在"多智能体协同控制"方面，我的能力有限，因为学习时未深入涉及此子领域。
建议提供具体论文让我深入学习，或咨询该领域专家。
```

## 评级报告示例

```markdown
# 📊 能力评估报告：强化学习在机器人控制中的应用

## 综合评分

| 维度 | 分数 | 说明 |
|------|------|------|
| 广度 | 85/100 | 覆盖主流算法、仿真环境、现实迁移等核心子领域；缺失多智能体、基于模型的RL |
| 深度 | 70/100 | 理解算法原理和训练流程；数学推导细节(如信赖域优化)掌握不足 |
| 置信度 | 88/100 | 主要参考arXiv顶会论文、伯克利课程；信息经多源交叉验证 |
| 结构化 | 90/100 | 知识图谱关系密度高；框架5层完整；问答对覆盖基础到进阶 |

**综合分数**：82.5/100

## 能力等级

**等级2：系统理解**，正向等级3过渡

当前可梳理强化学习在机器人控制中的核心逻辑，对比不同算法优劣，解答常见技术问题。正在向深度掌握过渡，能够讨论部分技术细节。

## 能力边界声明

✅ **我能做到**：
- 系统解释RL在机器人控制中的工作流程
- 对比DQN、PPO、SAC等主流算法的优劣
- 设计简单的奖励函数
- 推荐合适的仿真环境和训练框架
- 分析仿真到现实迁移的主要挑战

⚠️ **能力有限**：
- 多智能体协同控制：未深入学习，建议提供相关论文
- 基于模型的RL：仅了解概念，无法深入讨论
- 神经网络架构调参：缺乏实践经验，建议参考专业教程

❌ **建议咨询专家**：
- 前沿研究方向（如元学习、分层RL）：需要领域专家指导
- 特定硬件平台的实际部署问题：需要工程实践经验

## 后续学习建议

1. 阅读"Multi-Agent Reinforcement Learning"综述，补充多智能体知识
2. 学习基于模型的RL方法（如Dreamer、MuZero）
3. 实践：使用Isaac Gym或PyBullet进行仿真训练实验
```
