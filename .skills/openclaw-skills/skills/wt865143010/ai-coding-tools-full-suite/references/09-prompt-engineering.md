# 编程提示词工程技能参考

## 提示词核心要素

### 1. 基础结构

```yaml
标准提示词结构:
  1. 身份定义
     - 我是谁
     - 我的能力
     - 我的限制

  2. 任务描述
     - 要做什么
     - 输入什么
     - 期望输出

  3. 约束条件
     - 技术约束
     - 质量要求
     - 格式要求

  4. 上下文
     - 背景信息
     - 相关文件
     - 环境信息

  5. 执行要求
     - 操作步骤
     - 验证方式
     - 反馈机制
```

### 2. 信息分层

```yaml
第一层: 必需信息
  - 核心任务
  - 关键约束
  - 明确目标

第二层: 重要信息
  - 技术栈
  - 代码风格
  - 质量标准

第三层: 辅助信息
  - 项目背景
  - 团队规范
  - 个人偏好

第四层: 可选信息
  - 额外要求
  - 探索方向
  - 学习目的
```

## 任务描述技巧

### 1. 清晰的任务定义

```yaml
好的任务描述:
  ✓ 明确具体
  ✓ 可度量
  ✓ 可验证

不好的任务描述:
  ✗ 模糊笼统
  ✗ 主观感受
  ✗ 难以判断

示例对比:

不好: "优化这个函数"
  - 怎么优化
  - 优化什么指标
  - 达到什么标准

好: "优化这个函数的性能，将执行时间从500ms降低到100ms以内"
  - 明确优化目标
  - 具体性能指标
  - 可验证标准
```

### 2. 上下文提供

```yaml
上下文类型:
  1. 代码上下文
     - 相关代码片段
     - 函数签名
     - 类型定义
     - 使用示例

  2. 项目上下文
     - 项目结构
     - 技术栈
     - 配置信息
     - 依赖关系

  3. 业务上下文
     - 功能需求
     - 业务规则
     - 约束条件
     - 优先级

提供方式:
  1. 直接包含
     - 代码粘贴
     - 文件引用
     - 配置展示

  2. 指引包含
     - 文件路径
     - 搜索查询
     - 上下文键

  3. 隐含包含
     - IDE自动提供
     - 工具自动收集
     - 会话历史
```

## 主流工具提示词风格

### 1. Cursor风格

```yaml
风格特点:
  1. 直接执行
     - 明确指示工具操作
     - 减少确认步骤
     - 提高效率

  2. 上下文感知
     - 自动收集相关文件
     - 理解当前编辑状态
     - 预测下一步操作

  3. 渐进修改
     - 小步快跑
     - 及时验证
     - 快速迭代

使用技巧:
  - 明确指定文件
  - 说明修改意图
  - 提供约束条件

示例:
  "在src/auth/login.ts中:
   1. 添加邮箱验证
   2. 使用正则: ^[\\w.-]+@[\\w.-]+\\.\\w+$
   3. 错误提示需要本地化"
```

风格特点:

## 提示词模板库

### 1. 代码生成模板

```yaml
模板结构:
  [技术栈]
  [功能描述]
  [详细要求]
  [约束条件]
  [验收标准]

示例:
  技术栈: React 18 + TypeScript
  功能: 用户头像上传组件
  详细:
    - 支持拖拽上传
    - 支持点击选择
    - 支持预览
    - 支持裁剪
  约束:
    - 文件大小限制5MB
    - 支持jpg/png/webp
    - 使用现有UI组件库
  验收:
    - 所有交互有反馈
    - 错误情况有提示
    - 单元测试覆盖率>80%
```

### 2. 代码重构模板

```yaml
模板结构:
  [当前代码描述]
  [问题说明]
  [目标状态]
  [约束要求]

示例:
  当前: 一个500行的函数处理用户注册
  问题:
    - 职责不单一
    - 难以测试
    - 错误处理分散
  目标:
    - 拆分为多个函数
    - 每个函数<50行
    - 清晰的数据流
  约束:
    - 保持现有接口不变
    - 不改变业务逻辑
    - 通过所有现有测试
```

### 3. Bug修复模板

```yaml
模板结构:
  [问题描述]
  [复现步骤]
  [预期行为]
  [实际行为]
  [环境信息]

示例:
  问题: 用户登录失败后重定向错误
  复现:
    1. 输入错误密码
    2. 点击登录
    3. 等待3秒
  预期: 重定向到登录页并显示错误
  实际: 页面空白，控制台报错
  环境: Chrome 120, React 18.2
```

## 高级技巧

### 1. 链式思维

```yaml
技巧说明:
  引导AI分步骤思考

使用方式:
  1. 明确思考步骤
  2. 要求每步说明
  3. 验证中间结果

示例:
  "请按以下步骤分析这段代码:
   Step 1: 识别主要功能模块
   Step 2: 分析模块间的依赖关系
   Step 3: 评估代码复杂度
   Step 4: 识别潜在的性能瓶颈
   Step 5: 提出优化建议"
```

### 2. 角色扮演

```yaml
技巧说明:
  设定特定角色获取专业视角

使用方式:
  1. 定义角色身份
  2. 说明角色专长
  3. 设定视角要求

示例:
  "你是一个资深的安全工程师，
   专注于Web应用安全。
   请从安全角度审查这段代码，
   重点关注:
   1. 输入验证
   2. SQL注入风险
   3. XSS漏洞
   4. CSRF防护"
```

### 3. 示例引导

```yaml
技巧说明:
  通过示例说明期望的输出

使用方式:
  1. 提供输入示例
  2. 展示期望输出
  3. 说明格式要求

示例:
  "请将以下API响应转换为TypeScript类型:

  响应示例:
  {
    "id": 1,
    "name": "John",
    "email": "john@example.com",
    "created_at": "2024-01-01T00:00:00Z"
  }

  期望输出格式:
  interface User {
    id: number;
    name: string;
    email: string;
    createdAt: Date;
  }"
```

### 4. 约束强化

```yaml
技巧说明:
  强化约束条件避免偏离

使用方式:
  1. 列出硬性约束
  2. 说明违反后果
  3. 提供检查清单

示例:
  "实现这个功能时必须遵守:

  硬性约束 (违反会导致项目失败):
  - 不能修改现有API接口
  - 不能改变数据库表结构
  - 不能引入新的外部依赖

  检查清单:
  - [ ] 代码符合现有代码风格
  - [ ] 通过所有现有测试
  - [ ] 不引入新的lint错误"
```
