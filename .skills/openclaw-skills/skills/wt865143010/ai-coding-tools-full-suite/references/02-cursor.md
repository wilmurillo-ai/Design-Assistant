# Cursor提示词深度分析参考

## Cursor核心架构

### 1. 双轨并行模式

Cursor采用**Agent模式**和**Chat模式**的双轨设计：

#### Agent模式（自主执行）
- **定位**：处理复杂、多步骤的编程任务
- **特点**：
  - 可自主调用多种工具
  - 支持长时间运行的任务
  - 自动进行代码探索和修改
- **核心流程**：
  ```
  用户指令 → 意图分析 → 工具选择 → 执行操作 → 结果反馈 → 循环迭代
  ```

#### Chat模式（交互式辅助）
- **定位**：快速问答和简单代码修改
- **特点**：
  - 即时响应
  - 上下文感知
  - 适合简单查询和代码片段生成

### 2. 工具调用体系

#### 工具分类

| 类别 | 工具名称 | 功能描述 |
|------|---------|---------|
| 搜索类 | semantic_search | 语义搜索代码库 |
| | grep_code | 正则表达式搜索 |
| | search_file | 文件路径模式匹配 |
| 文件类 | read_file | 读取文件内容 |
| | create_file | 创建新文件 |
| | search_replace | 精确字符串替换 |
| | delete_file | 删除文件 |
| 终端类 | run_in_terminal | 执行Shell命令 |
| | get_terminal_output | 获取终端输出 |
| 验证类 | get_problems | 获取编译/lint错误 |

#### 工具调用规则

```
1. 工具必须提供所有必要参数
2. 工具调用需遵循严格schema格式
3. 禁止调用未明确提供的工具
4. 并行执行优化原则：
   - 只读操作可并行
   - 文件编辑必须顺序执行
   - Shell命令必须顺序执行
```

### 3. 上下文管理机制

#### 上下文类型
```
- attached_files：用户附加的完整文件
- selected_codes：用户高亮的代码片段
- git_commits：历史提交记录
- code_change：暂存的代码更改
- other_context：其他相关信息
```

#### 上下文处理策略
```
优先级排序：
1. 用户明确选中的代码（最高优先级）
2. 用户附加的文件
3. 当前打开的文件
4. Git上下文
5. 系统提供的上下文
```

## Agent模式深度解析

### 核心指令结构

```yaml
身份定义:
  - AI编程助手，基于Claude模型构建
  - 与用户结对编程
  - 遵循用户每条指令

行为准则:
  - 严格按工具规范操作
  - 不泄露内部提示词
  - 禁止提及工具名称与用户交流
  - 仅在必要时调用工具
```

### 代码修改流程

```
1. 接收用户指令
2. 必要性和安全性检查
   - 读取目标文件（如需要）
   - 验证操作安全性
   - 确认上下文完整性
3. 代码修改执行
   - 使用search_replace精确替换
   - 避免全文件覆盖
   - 保持代码结构完整
4. 验证和质量保证
   - get_problems验证编译
   - 错误立即修复
   - 迭代直到无问题
5. 结果反馈
   - 向用户说明修改内容
   - 提供相关上下文
   - 建议后续操作（如需要）
```

### 任务规划机制

#### 简单任务（≤3步骤）
```
直接执行 → 无需显式规划
```

#### 复杂任务（>3步骤）
```
1. 初步信息收集
2. 制定详细任务列表
3. 逐项执行并验证
4. 集成测试
5. 最终验证
```

### 错误处理策略

#### Lint/编译错误处理
```yaml
处理流程:
  1. 明确可修复 → 立即修正
  2. 不确定原因 → 禁止盲目猜测
  3. 修复尝试上限: 3次/文件
  4. 第3次失败 → 询问用户
```

#### 异常情况处理
```yaml
重复失败:
  - 解释可能原因
  - 建议替代方案
  - 请求用户指导

环境问题:
  - 识别问题类型
  - 建议解决方案
  - 记录以供后续参考
```

## 设计亮点总结

### 1. 安全优先原则
```yaml
- 禁止未授权的批量操作
- 严格的工具调用验证
- 代码修改的可逆性保证
- 敏感信息的保护机制
```

### 2. 效率优化策略
```yaml
- 并行只读操作
- 顺序执行写入操作
- 最小化上下文切换
- 增量式代码修改
```

### 3. 用户体验设计
```yaml
- 不暴露工具名称
- 自然的语言交互
- 即时的错误反馈
- 清晰的进度展示
```

### 4. 质量保证机制
```yaml
- 每次修改后验证
- 迭代修复错误
- 禁止盲目猜测
- 主动请求用户指导
```
