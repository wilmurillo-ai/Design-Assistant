# 系统连续性检查模板

## 快速检查清单

### 🎯 核心原则
**环环相扣，连续完整，无需人工干预**

### 1. 工作流连续性检查
- [ ] **数据流图**：是否画出了输入→处理→输出的完整流程？
- [ ] **自动触发**：每个环节完成后能自动触发下一个吗？
- [ ] **端到端测试**：是否测试过从开始到结束的完整工作流？

### 2. 用户视角拷问
- [ ] **重启恢复**：系统重启后所有功能能自动恢复吗？
- [ ] **人工干预**：用户需要定时手动执行任何操作吗？
- [ ] **错误恢复**：出错后系统能自动恢复或告警吗？

### 3. 系统重启测试
- [ ] **开机自启**：是否配置了开机自启动或系统服务？
- [ ] **进程监控**：进程崩溃后是否有自动重启机制？

### 4. 错误处理检查
- [ ] **错误隔离**：关键操作是否有try-catch错误隔离？
- [ ] **日志告警**：是否有错误日志记录和自动告警？

## 改进建议库

### 工作流连续性
- **问题**：模块孤立，逻辑断裂
- **改进**：先画图再编码，设计自动触发机制，进行全流程测试

### 用户视角
- **问题**：需要人工记忆/干预
- **改进**：配置自动化任务，消除人工操作，实现自动恢复

### 系统重启
- **问题**：重启后功能失效
- **改进**：使用Windows任务计划程序，实现进程监控重启

### 错误处理
- **问题**：单点失败导致整体瘫痪
- **改进**：关键模块错误隔离，完善日志监控，优雅降级

## 常见错误模式

### 1. 编码问题（哥哥发现）
- 乱码没彻底解决
- 混合格式没正确处理

### 2. 触发逻辑错误（哥哥发现）
- 赛后采集触发时机错误
- 自动化缺失需要手动启动

### 3. 系统设计缺陷（哥哥发现）
- 只做单点功能，不思考整体工作流
- 模块间逻辑断裂，没有环环相扣
