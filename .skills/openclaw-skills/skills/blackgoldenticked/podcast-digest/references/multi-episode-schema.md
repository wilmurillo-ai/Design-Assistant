# 多期播客研究数据结构

## 研究对象定义

当用户提供 2 期或以上内容时，构建以下数据结构进行横向分析。

## 核心数据模型

### Episode 对象
```
{
  id: "EP001",
  title: "...",
  date: "YYYY-MM-DD",
  duration: "hh:mm",
  hosts: ["..."],
  guests: [{ name, role, affiliation, stance }],
  core_topic: "...",          // 一句话主题
  concepts: ["概念A", ...],   // 本期核心概念列表
  arguments: [
    {
      claim: "...",
      evidence: "...",
      speaker: "...",
      timestamp: "MM:SS",
      tags: ["#领域", "#争议点"]
    }
  ],
  quotes: [{ text, speaker, timestamp, insight }],
  open_questions: ["..."],    // 未解决的问题
  references: ["书/人/概念"]  // 提到的外部资源
}
```

### 关联关系类型

| 关系类型 | 符号 | 含义 |
|--------|------|------|
| 深化 | `-->|"深化"|` | B期对A期概念的深入展开 |
| 扩展 | `-->|"扩展"|` | B期将A期引申到新领域 |
| 反驳 | `-->|"反驳"|` | B期对A期观点提出质疑 |
| 前提 | `-->|"前提"|` | A期是B期讨论的前提假设 |
| 应用 | `-->|"应用"|` | B期将A期理论付诸实践 |
| 对比 | `<-->|"对比"|` | 两期从不同角度看同一问题 |
| 修正 | `-->|"修正"|` | B期更新了A期的某个说法 |

## 多期研究矩阵模板

用 Markdown 表格展示跨期对比：

```markdown
| 维度 | EP01 [主题] | EP02 [主题] | EP03 [主题] |
|------|-----------|-----------|-----------|
| 核心问题 | ... | ... | ... |
| 主要立场 | 偏乐观 | 中立 | 偏保守 |
| 关键数据 | X% | Y倍 | Z年 |
| 嘉宾背景 | 学术 | 工业 | 投资 |
| 核心概念 | A, B | B, C | C, D |
| 开放问题 | ... | ... | ... |
```

## 知识演化追踪

对同一概念跨期追踪：

```
概念："[X]"

EP01 (日期): [初次提出/定义]
  └─ 状态：提出
  └─ 说法：[引用]

EP03 (日期): [修正/深化]  
  └─ 状态：修正
  └─ 变化：从"..."到"..."
  └─ 原因：[新证据/新案例]

EP07 (日期): [应用验证]
  └─ 状态：验证
  └─ 结果：[支持/反驳]
```

## 知识图谱节点规则

- **大节点**：在 3 期以上出现的概念
- **中节点**：出现 2 次的概念
- **小节点**：仅出现 1 次的概念
- **边颜色**：支持关系=绿色，反驳=红色，扩展=蓝色，中性=灰色
- **节点颜色**：理论概念=紫色，人物=橙色，工具/方法=青色，数据=蓝色

## 综合报告输出结构

```
1. 研究概览
   - 覆盖期数、时间跨度、嘉宾总数
   - 主题演化一句话总结

2. 主题演化时间线（Mermaid timeline）

3. 跨期对比矩阵（Markdown 表格）

4. 核心概念知识图谱（Mermaid graph LR）

5. 概念演化追踪（选取 3-5 个重要概念）

6. 观点分歧地图（quadrantChart）

7. 综合洞察
   - 贯穿所有期的核心线索
   - 尚未解答的大问题
   - 领域共识 vs 争议点

8. 推荐学习路径
   - 入门：从哪期开始
   - 进阶：按主题线索的阅读顺序
   - 深入：延伸到播客外的资源

9. 开放研究清单
   - 这系列播客留下的未解之谜
```

## 双向关联导航

在 Web 报告中，每个概念/论点旁显示：
- 👆 **前溯**：哪期首次提出，原文引用
- 👇 **后续**：哪期深化/反驳，跳转链接
- 🔄 **对比**：持相反观点的期数

实现方式：在 HTML 中用 `data-concept` 属性标记，JS 动态生成关联面板。
