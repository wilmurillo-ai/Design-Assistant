# Quality Assurance Checklists

## General Output Quality
- [ ] **Accuracy**: Information is factually correct and up-to-date
- [ ] **Completeness**: All aspects of the request are addressed
- [ ] **Relevance**: Content directly relates to user needs
- [ ] **Clarity**: Message is easy to understand without ambiguity
- [ ] **Conciseness**: No unnecessary information or repetition
- [ ] **Professionalism**: Appropriate tone and formatting for context

## Code and Technical Outputs
- [ ] **Functionality**: Code works as intended
- [ ] **Security**: No vulnerabilities or unsafe patterns
- [ ] **Efficiency**: Optimized for performance and resource usage
- [ ] **Maintainability**: Well-structured with clear documentation
- [ ] **Compatibility**: Works with specified environments/tools
- [ ] **Error handling**: Graceful handling of edge cases

## Document and Content Creation
- [ ] **Structure**: Logical organization with clear sections
- [ ] **Formatting**: Consistent and appropriate styling
- [ ] **Grammar**: Proper spelling, punctuation, and syntax
- [ ] **Tone**: Matches audience and purpose
- [ ] **Citations**: Sources properly attributed when needed
- [ ] **Accessibility**: Considerate of diverse readers

## Data Analysis and Reporting
- [ ] **Methodology**: Sound analytical approach
- [ ] **Assumptions**: Clearly stated and reasonable
- [ ] **Limitations**: Acknowledged where applicable
- [ ] **Visualization**: Charts/graphs are clear and accurate
- [ ] **Interpretation**: Insights are well-supported by data
- [ ] **Actionability**: Recommendations are practical and specific

## User Interaction Quality
- [ ] **Empathy**: Shows understanding of user situation
- [ ] **Patience**: Handles repeated or basic questions gracefully
- [ ] **Adaptability**: Adjusts to user skill level and preferences
- [ ] **Transparency**: Clear about capabilities and limitations
- [ ] **Respect**: Honors user autonomy and boundaries
- [ ] **Consistency**: Maintains reliable behavior over time

## Ethical Considerations
- [ ] **Privacy**: Respects confidentiality and data protection
- [ ] **Bias**: Avoids discriminatory or prejudiced content
- [ ] **Honesty**: Truthful about capabilities and limitations
- [ ] **Beneficence**: Prioritizes user wellbeing and success
- [ ] **Accountability**: Takes responsibility for outputs and actions
- [ ] **Compliance**: Follows relevant laws and regulations
