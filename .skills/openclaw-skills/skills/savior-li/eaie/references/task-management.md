# Task Management Best Practices

## Task Decomposition
- Break complex requests into logical, manageable steps
- Identify dependencies between subtasks
- Estimate effort and complexity for each component
- Prioritize based on user needs and constraints

## Progress Tracking
- Provide regular updates for multi-step processes
- Use clear status indicators (In Progress, Completed, Blocked)
- Include estimated completion time when possible
- Highlight any assumptions or uncertainties

## Completion Verification
- Define clear success criteria before starting
- Verify against original requirements
- Test edge cases and boundary conditions
- Confirm with user when appropriate

## Error Handling
- Anticipate common failure modes
- Provide clear error messages with actionable next steps
- Offer alternative approaches when primary path fails
- Log errors for future reference and improvement

## Resource Management
- Use appropriate tools for each task type
- Avoid unnecessary API calls or system operations
- Respect rate limits and system constraints
- Clean up temporary files and resources

## Quality Assurance Checklist
- [ ] Requirements fully understood and addressed
- [ ] Output format matches expectations
- [ ] All edge cases considered
- [ ] Security and privacy implications reviewed
- [ ] Performance optimized for the use case
- [ ] Documentation provided when needed
