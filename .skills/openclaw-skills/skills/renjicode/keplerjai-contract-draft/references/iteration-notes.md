# 迭代经验

这份文档只负责记录真实运行中已经确认的失败模式和修正经验。

## 已确认的失败模式

### 1. 只改合作内容，不改联动段落

表现：

1. 合作内容区已改
2. 鉴于、合作原则、权利义务、合作机制、知识产权仍残留旧语义

要求：

1. 必须做全文一致性检查
2. 不能把全文兼容改写完全交给临场发挥
3. runner 需要对关键兼容段做默认兜底改写

### 2. 把模板改造成新的章条体系

表现：

1. 模板原本不是“第X章 / 第X条”
2. 运行后却新增了章节式结构

要求：

1. 保留模板原有标题层级
2. 除非模板本来就有章条体系，否则不要新增

### 3. 使用旧版 job spec

表现：

1. 只写 `content_sections`
2. 不写 `cooperation_items`
3. 不写 `paragraph_overrides`
4. 结果只改标题级信息

要求：

1. 当前标准 job spec 必须使用 `cooperation_items`
2. 不能再把旧版简化 schema 当成可接受输入

### 4. Windows 下 JSON / 中文编码摩擦

表现：

1. PowerShell 写出的 JSON 可能带 BOM
2. 中文路径、中文文件名在 shell 中容易乱码

要求：

1. JSON 读取应兼容 `utf-8-sig`
2. 模板路径与输出路径应优先走自动推导和相对解析
3. 写完后必须验证生成的 `.docx`

## 最低验收清单

1. 甲乙方在封面、正文、签署页是否一致
2. 合作内容之外的关键联动段是否一起改了
3. 是否还有旧主体、旧地域、旧行业、旧产品残留
4. 签署页是否仍完整
5. 输出路径是否落在目标 agent workspace 下
