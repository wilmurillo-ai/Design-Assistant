# 钉钉应用权限配置指南

## 问题诊断

当前应用可以获取 access_token，但调用 API 时返回：
```
errcode: 22
errmsg: Invalid method[submsg=不合法 ApiName]
```

这说明**应用没有配置相应的 API 权限**。

---

## 配置步骤
   
### 步骤 1: 登录钉钉开放平台1111

访问：https://open.dingtalk.com

使用企业管理员账号登录。

### 步骤 2: 进入应用管理

1. 点击顶部菜单「应用开发」
2. 选择「企业内部开发」
3. 找到你创建的应用（按应用名称搜索）
4. 点击进入应用详情

### 步骤 3: 添加权限

在应用详情页面：

1. 点击左侧菜单「权限管理」
2. 点击「申请权限」或「添加权限」
3. 搜索并添加以下权限：

#### 必需权限：
- **通讯录权限**
  - `获取部门用户列表` (用于获取员工 ID)
  - `获取用户详情` (可选，用于获取员工姓名)

- **考勤权限**
  - `获取打卡记录`
  - `获取考勤组信息`
  - `获取考勤报表`

### 步骤 4: 提交审批

1. 添加权限后，点击「提交审批」
2. **需要企业管理员审批**
3. 审批通过后权限才会生效

### 步骤 5: 发布应用

1. 点击「版本管理与发布」
2. 点击「发布应用」
3. 确保应用状态为「已启用」

## 验证配置

权限配置完成后，运行测试脚本验证：

```bash
cd keplerjai-dingtalk-attendance

```

## 常见问题

### Q: 看不到权限管理菜单？
A: 确保你是应用的管理员或开发者。

### Q: 权限申请后还是不能用？
A: 
1. 确认管理员已审批通过
2. 确认应用已发布
3. access_token 有效期为 2 小时，重新获取 token 再试

### Q: 没有企业管理员权限？
A: 需要联系公司的钉钉管理员帮你配置权限。

## 权限说明

| 权限名称 | API 用途 | 是否必需 |
|---------|---------|---------|
| 获取部门用户列表 | 获取员工 ID 列表 | 是 |
| 获取打卡记录 | 获取考勤数据 | 是 |
| 获取用户详情 | 获取员工姓名 | 可选 |
| 获取考勤组 | 获取考勤组配置 | 可选 |

## 配置完成后

权限配置完成并审批通过后，运行：

```bash
node index.js
```

即可获取考勤数据！
