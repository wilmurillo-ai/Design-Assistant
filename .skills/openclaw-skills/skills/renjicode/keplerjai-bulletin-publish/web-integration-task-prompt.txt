请执行 KeplerJAI 日常简讯网页端集成任务。

这是一个一次性完成的总流程任务，请在单次运行中完成以下三件事：
1. 从固定新闻入口采集并筛选 7 条新闻
2. 通过 KeplerJAI API / curl 创建 7 条简讯
3. 返回最终整理好的纯文本结果

不要进入微信发送阶段。不要要求人工确认。不要分阶段停下来。不要输出长篇解释、分析过程或中间日志。

## 固定新闻入口

只允许从以下页面开始采集新闻：

https://www.newsnow.co.uk/h/Science/AI?type=ln

不要改用其他新闻聚合入口，不要从别的新闻首页开始找。

## 第一阶段要求：新闻采集

1. 只保留人工智能行业相关新闻
2. 排除虚拟货币、政治敏感、明显负面、谣言、八卦、高风险内容
3. 先记录中文新闻标题，再点击“显示原文”获取英文标题、英文正文和原始文章链接
4. 最终必须筛出正好 7 条新闻
5. 来源尽量分散，避免 7 条几乎都来自同一家媒体
6. 如果候选足够，尽量保持题材多样

每条最终新闻至少要整理出以下字段：

- `english_title`
- `chinese_title`
- `source_url`
- `image_url`
- `english_body_raw`

## 第二阶段要求：API 发布

接口地址：
https://www.keplerjai.com/api/content/bulletin

对每条新闻执行发布时：
1. `title` 使用 `english_title`
2. `url` 使用 `source_url`
3. `content` 优先使用适合官网展示的 HTML 内容
4. 使用 `curl` 实际执行 `POST` 请求
5. 只有在真实接口响应中存在：
   - `resultCode = "OK"`
   - `bulletin.id` 为有效整数
   时，才视为发布成功
6. 前台链接规则固定为：
   - `https://www.keplerjai.com/bulletin?id=<bulletin.id>`

如果某一条发布失败：

1. 不要伪装为成功
2. 明确标记失败
3. 在最终输出中说明失败原因

## 图片策略

当前阶段可以尝试带图发布，但不要把图片作为发布成功的必要条件。

已知现实情况：
1. `content` 字段支持 HTML，正文中可以插入 `<p>` 和 `<img>`
2. 图片应插在第一段之后，不能作为正文第一行
3. 图片推荐使用 `<img src="..." style="width:100%;" />`
4. 只有当你能拿到真实、可访问、与正文强相关的图片直链时才插图
5. 如果图片链接不可访问、显示破图、是广告图、缩略图、占位图或来源不明确，则不要插图，直接退回纯文本或无图 HTML
6. 不要因为单条图片失败而阻塞整条 7 条链路

## 最终输出要求

如果 7 条都发布成功，最终只输出下面这种纯文本格式，不要加标题、编号、表格或解释：

英文标题
中文标题
https://www.keplerjai.com/bulletin?id=xxxx

下一条英文标题
下一条中文标题
https://www.keplerjai.com/bulletin?id=yyyy


联盟小秘书 lmxms2020

格式细则：
1. 每条新闻严格三行：
   - 英文标题
   - 中文标题
   - 官网链接
2. 每条之间空一行
3. 全部 7 条结束后空两行
4. 最后一行固定为：
   - `联盟小秘书 lmxms2020`

## 失败输出要求

如果无法完成 7 条完整发布，不要返回半截正常结果。

请返回简短失败说明，格式如下：

FAILED
原因：..
已成功发布条数：N

只有 7 条都成功发布时，才允许输出最终纯文本结果。
