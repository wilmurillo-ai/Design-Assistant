## 反偷懒强制约束

> AI 偷懒的核心手段：间距虚大 + 内容极少 + 卡片大量留白。以下规则全部可量化，每格生成后逐条自查。

### 文件体积下限

**每页 HTML ≥ 18KB**（内容页）；封面/目录/结尾等特殊页 ≥ 8KB。

达到 18KB 的典型构成：
```
CSS 样式块          ~3–4 KB   （变量、四区、卡片、Tc 主题、背景纹理）
HTML 结构 + 卡片    ~6–8 KB   （4–6 张卡片，每卡正文 80–120 字）
SVG 图表            ~4–6 KB   （至少 1 个主图表 + 1–2 个内嵌迷你图/进度条）
数据组件/数字块     ~2–3 KB   （mini 卡片行、3格数字块、进度条行）
页眉/摘要/页脚      ~1–2 KB
```

体积不足的修复方向（按优先级）：
1. **补 SVG 细节**：增加坐标轴标签、数值注释、渐变填充、辅助线
2. **补卡片正文**：每张卡片正文扩至 80–120 字，确含 ≥2 个数字
3. **补数据组件**：每张卡片加 mini 数字块或进度条行
4. **丰富背景纹理**：增加光晕层、噪点、几何装饰 SVG

**预估<18KB = 禁止生成，必须补充内容。违者删除文件重来。**

### 卡片正文字数（硬性指标）

**每张卡片正文≥60 字，且含≥2 个具体数字或专有名词。**

**字数不足或数字<2 个 = 偷懒，必须扩写。**

### 间距硬上限

| 属性 | 硬上限 | 偷懒常见值（禁止） |
|------|--------|----------------|
| 网格 `gap` | **8px** | 16–24px |
| 卡片 `padding` 上下 | **8px** | 16–20px |
| 卡片 `padding` 左右 | **12px** | 20–24px |
| 标题 `margin-bottom` | **5px** | 10–16px |
| 文字 `line-height` | **1.45** | 1.6–2.0 |
| `.ct` `padding-top` | **0（禁止）** | 12–20px |

### 75% 填充率规则

每个卡片：**文字 + 图表高度 ≥ 卡片可用高度的 75%**

```
布局C 每格：高 281px，padding 上下各 8px → 可用 265px
最低要求：265 × 75% = 199px 必须被内容填满

❌ 偷懒：标题 22px + 正文 1 行 18px = 40px → 占比 15%，不通过
✅ 合格：标题 22px + 正文 4 行 72px + SVG 80px + mini 35px = 209px → 79%，通过
```

判断方法：视觉上看卡片背景色区域，若背景色面积 > 内容面积，即不通过。

### 三层内容结构（每格必须）

```
层次 1：标题行（主题词 + 核心数字，1 行）
层次 2：正文段落（含 ≥2 个数字/专有名词，3–5 行）
层次 3：下方增强组件（三选一）
         ├── mini 数据卡片行（2–4 个，flex 排列）
         ├── 内嵌 SVG 图表（条形/折线/进度条）
         └── 数据网格（2×2 或 3×1 数字块）
```

**只有层次 1+2 没有层次 3 = 偷懒，必须补充。**

> 质检清单（≥4项满足）→ 见 `06-workflow.md` Phase 4

## 内容写作规范

### 禁止空洞描述

```
❌ 该攻击危害严重，具有重要研究价值。（0 个数字）
❌ 防御方案效果良好，推荐采用。（0 个数字，0 个专有名词）
❌ 模型存在漏洞，需要及时修复。（废话）
```

### 必须紧凑数据

```
✅ FGSM 单步成功率 63%，PGD-40 步达 97%，ε≤0.03 时人眼无法分辨。
✅ PGD 对抗训练鲁棒精度 +34%，代价仅 -5.2%；C&W 可 bypass 蒸馏防御（Carlini 2017）。
✅ 停车标志贴纸攻击（Eykholt 2018）：30m 外误识率 91%，任意光照条件有效。
```

**每个卡片至少包含 2 个具体数字或专有名词（论文作者/年份/工具名/百分比/毫秒/px）。**

### 关键词高亮规范

```html
<!-- 数字/百分比 → <b> -->
<b style="color:var(--t);">97%</b>

<!-- 公式/代码 → <em>（等宽主色）-->
<em style="color:var(--p);font-style:normal;font-family:monospace;">δ=ε·sign(∇ₓJ)</em>

<!-- 专有名词/论文 → <b> -->
<b style="color:var(--t);">PGD-AT</b>（Madry 2018）
```

## 数字块组件（第三层，按 design-system 风格自行实现）

> 数字块组件无固定 HTML 模板。生成时根据激活的 **Tc 模板**（design-system 风格）自行设计，
> 所有颜色必须来自 Tc CSS 变量，禁止硬编码非语义色。

### 三种常用变体（结构描述）

**① N格数字块（横向网格）**
- 结构：`grid-template-columns: repeat(N, 1fr)`，gap 5px，margin-top 5px
- 单元格：`var(--pm)` 背景 + `var(--bd)` 描边 + 圆角 3-4px + padding 5px 7px，居中
- 数字：16-20px bold，`var(--p)`（主题色）或语义色（语义突出时）
- 标注：9px `var(--dt)`，margin-top 2px
- 状态强调格：背景/描边改用对应语义色的 10%/25% 透明版本

**② 对比列表（带色点）**
- 结构：`flex-direction: column`，gap 4px
- 每行：7px 圆点（语义色）+ 名称（11px `var(--t)` font-weight 500）+ 右对齐值（10px 语义色）
- 色点颜色 = 语义色（success/warning/danger）或 `var(--p)`/`var(--c2)`/`var(--c3)`

**③ 进度条行**
- 结构：`flex`，每行 label（固定宽）+ 条（`flex:1` 3px 高）+ 数值
- 条背景：`rgba(255,255,255,0.04)`（暗色系）或 `rgba(0,0,0,0.06)`（亮色系）
- 条填充：`linear-gradient(90deg, var(--p), var(--p) 70%, 亮色)` 或 `var(--c2)` 第二系列
- 数值：`var(--mt)` bold

### 颜色原则

```
主题色数字   → var(--p)
状态强调数字 → var(--success) / var(--danger) / var(--warning)
多系列进度条 → var(--p) / var(--c2) / var(--c3)
标注文字     → var(--dt)
数值文字     → var(--mt)
条/格背景    → var(--pm)（主题透明背景）
条/格描边    → var(--bd)（主题透明描边）
```

## SVG / 图表颜色适配规则（Design-System 版）

> **所有 SVG 图表必须使用下列变量系统，禁止硬编码非语义颜色。**

### 颜色变量层级

```
层级 1 · Tc 颜色（随 design-system 主题变化）
  var(--p)   → 主强调色（第一数据系列、重点高亮）
  var(--c1)  → 同 var(--p)，明确标识"第一系列"
  var(--c2)  → 第二数据系列（Tc 中派生，与 --p 互补）
  var(--c3)  → 第三数据系列（Tc 中派生）
  var(--mt)  → 图表辅助文字、数值标注
  var(--dt)  → 轴标签、图例文字、网格线颜色

层级 2 · 语义固定色（含义固定，不随主题变化）
  var(--success)  → #10b981  正确、通过、增长
  var(--danger)   → #ef4444  错误、高风险、下降（负面时）
  var(--warning)  → #f59e0b  警告、中风险
  var(--info)     → #3b82f6  信息、中性数据
  var(--neutral)  → #94a3b8  辅助系列、未激活项

层级 3 · 透明度（所有层级颜色均可叠加透明度）
  填充区域  → 0.10-0.15 透明度（如 rgba fill）
  描边/边框 → 0.25-0.35 透明度
  网格线    → 0.04-0.08 透明度
```

### SVG 适配示例

```html
<!-- 单系列 → 使用 var(--p) -->
<rect fill="var(--p)"/>

<!-- 多系列 → 使用 --c1/--c2/--c3 -->
<polygon fill="rgba(var(--c1-rgb,59,130,246),0.15)" stroke="var(--c1,var(--p))"/>
<polygon fill="rgba(var(--c2-rgb,99,102,241),0.12)" stroke="var(--c2,#6366f1)" stroke-dasharray="3,2"/>

<!-- 轴标签、图例 → var(--dt) -->
<text fill="var(--dt)">标签</text>

<!-- 网格线 → rgba with --dt 近似值 -->
<line stroke="rgba(255,255,255,0.06)"/>  <!-- 暗色系通用 -->

<!-- 语义色（SWOT、状态、风险） → 固定语义色 -->
<rect fill="rgba(16,185,129,0.1)" stroke="rgba(16,185,129,0.35)"/>  <!-- success -->
<rect fill="rgba(239,68,68,0.1)"  stroke="rgba(239,68,68,0.35)"/>   <!-- danger -->
```

### 图表颜色规则

| 场景 | 颜色来源 | 说明 |
|------|---------|------|
| 单系列数据条/线 | `var(--p)` | 主题自适应 |
| 第二数据系列 | `var(--c2)` | Tc 定义，与 --p 对比 |
| 第三数据系列 | `var(--c3)` | Tc 定义 |
| 成功/通过/正增长 | `var(--success)` | 语义固定色 |
| 失败/风险/负变化 | `var(--danger)` | 语义固定色 |
| 警告/中风险 | `var(--warning)` | 语义固定色 |
| SWOT/风险矩阵四象限 | 语义固定色（不随主题变化） | 优势=success，劣势=danger，机会=info，威胁=warning |
| 图例/标注文字 | `var(--dt)` | 弱显示 |
| 填充面积 | 对应颜色 10-15% 透明 | rgba 或 fill-opacity |
