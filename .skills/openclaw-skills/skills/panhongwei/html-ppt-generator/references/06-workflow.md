# 06 · 工作流程与质量核查

---

## 第零阶段：内容拆解（生成任何 HTML 之前）

> **铁律：S0-3.5 输出物是生成 HTML 的前置条件。跳过 = 拿空弹匣开枪。**
> 若用户已提供结构化 MD 输入 → 跳过 S0-1/S0-2，从 S0-3 开始。

### S0-1 · 联网核实数据

用 web_search 查询主题相关可信数据（Gartner/IDC/McKinsey/权威机构），找：
- 行业报告中的具体数字（带时间、来源）
- 公开可验证的对比数据

**查不到 → 标注 `[待核实]`，禁止凭感觉估算。**

### S0-2 · 主题定性（3 句话）

```
核心信息：[1 句话，含数字，整份报告服务于它]
核心张力：[最反直觉的矛盾点，这是记忆点的原材料]
主要受众：[职位] · [认知距离] · 核心问题：[1句] · 禁忌：[1句]
```

### S0-3 · 逐页内容指令（每页 ≤8 行）

```
### P0X · [页面名]  [情感：警示/积极/中性/展望]
意图：[受众读完这页的内心活动，不是"介绍XXX"]
气质：[1-2词，本页视觉人格，如"压迫·警示""精密·手术台"]
数据：[数字] ← [来源]（已核实/[待核实]）
原料：[论点骨架/对比两端，2-3条毛坯，不润色]
视觉重心：[需要/不需要图] · [内容结构理由]
```

### S0-3.5 · 逐页内容扩写（丰满度保障）

> 每页输出以下四项，HTML 生成前必须完成：

**① 核心论点句**（1 句，含具体数字，可直接用作卡片大标题）

**② 正文段落**（每张卡片 60-120 字）
- 以 S0-3 原料为骨架，展开完整段落
- 每段 ≥2 个具体数字或专有名词
- 关键数字用 `<b>` 标注，专有名词用 `<b>` 或 `<em>` 标注

**③ 数据组件**（图表/数字块/进度条的具体数值）
```
标签 | 数值 | 含义说明
例：AI-SAST 扫描时间 | 4-8小时 | vs 传统 2-3周
```

**④ 摘要栏文字**（≤40 字，直接可复制，不是标题缩写）

扩写密度自查：
```
□ 论点句含具体数字？
□ 每张卡片正文 ≥60 字，含 ≥2 个数字/专有名词？
□ 数据组件有完整标签+数值+含义？
□ 摘要栏 ≤40 字，独立成句？
□ 所有数值来自 S0-1 核实，无估算？
```

### S0-4 · 视觉规格声明

```
风格：[暗/亮色系] + [气质关键词]
记忆点：[具体可执行的视觉元素，不是形容词]（必须出现在多页，非仅封面）
背景方案：body 基底色___ + body::before 纹理层___ + body::after 光晕层___
  ★ 此处定义的背景 CSS 三件套（body/::before/::after）将在每一页原样复制，
    包括只生成单页的场景——单页也必须先在此声明背景，不得省略。
禁用色：___  必用色：___（+语义绑定）
禁止行为：❌ [针对本报告具体化] ❌ [...]
```

### S0-4.5 · 主题专属卡片语言设计（新增强制步骤）

> **铁律：必须在写任何 HTML 之前完成。卡片形状/线条/装饰必须与主题强绑定，禁止套用通用圆角卡壳。**

**① 形状语言**（从下方选择并说明理由）

| 主题类型 | 形状语言 | CSS 实现 |
|---------|---------|---------|
| 战争·军事·史诗 | 切角六边形·锐角·不对称裁切 | `clip-path: polygon(...)` |
| 科技·AI·数字 | 精密直角·内发光框·扫描线 | `border + box-shadow inset` |
| 奇幻·RPG | 卷草纹框·宝石切割·符文角标 | `clip-path` + `::before` 角标 |
| 极简·学术 | 单像素细线·负空间·无填充 | `border-left` + 透明底 |
| 赛博·霓虹 | 故障错位·RGB 分层·像素块 | `box-shadow` 多重错位 |
| 金融·典雅 | 细金线描边·圆弧端·烫金角 | `border-radius` + 渐变描边 |
| 游戏竞技 | 血条感·进度条边框·状态指示灯 | `linear-gradient` border |

**② 线条风格**（具体到像素宽度和颜色语义）
```
主卡边框：___px ___色（来自 --bd 或 --bd-em）
强调左边框：___px ___色（CARD-B / CARD-E 必填）
装饰线条：___（顶部扫光/底部渐变/斜切边）
分隔线宽度：___px，颜色 ___
```

**③ 卡片变体分工表**（明确本报告每类内容用哪种变体）
```
CARD-C（数字核心）→ 用于：___
CARD-B（左条无底）→ 用于：___
CARD-E（语义色框）→ 用于：___（语义色绑定：___色=___含义）
CARD-A（全色填充）→ 用于：___（每页最多1张，放最重要结论）
CARD-STD（通用）  → 用于：___
```

**④ 装饰元素清单**（角标/徽章/符文/勋章/印章等主题专属元素）
```
角标类型：___（例：RPG 切角三角 / 军事菱形印 / 科技扫描点）
角标 CSS：___（::before / ::after 实现，不用图片）
全局装饰：___（例：节标题前的卷草纹符号 / 卡片顶部渐变光束）
```

**⑤ 卡片密度分级**
```
高密度卡（≥80字正文）→ 高度：___px，字号：___px
标准卡（60-80字正文）→ 高度：___px，字号：___px
KPI 卡（数字主导）   → 高度：___px，大数字：___px
```

> 以上 ①-⑤ 输出完成后，将卡片语言写入 Tc 模板的"卡片"字段，进入阶段一规划。

> ⚠️ **输出 S0-4 + S0-4.5 全部内容后，直接进入第一阶段规划，然后开始生成 HTML 文件。**

## 第一阶段：规划

### 步骤 1 · 主题拆解

将 S0-3 的页面清单整理为 8-12 个子维度，参考维度类型：
背景/现状 · 概念/定义 · 分类/类型 · 原理/机制 · 流程/步骤 · 案例/实验 · 风险/威胁 · 检测/评估 · 防御/应对 · 工具/框架 · 趋势/展望

### 步骤 2 · 确定主基调

```
主题 → 主基调 → design-system 文件
技术/AI/架构 → 专业科技 → dark-luxury / developer-tech / dashboard-analytical
风险/威胁/攻击 → 警示严肃 → neon-cyberpunk / glitch-corrupt / military-tactical
防御/安全 → 积极正面 → dark-luxury（绿主色）/ esg-sustainability
数据/统计 → 中性客观 → dashboard-analytical / science-lab
趋势/展望 → 活力创新 → startup-pitch / creative-studio / cyberpunk-anime
金融/投资 → 权威典雅 → finance-banking / real-estate
政府/政策 → 庄重规范 → government-official / legal-profession
文化/艺术 → 人文典雅 → culture-museum / japanese-minimalism
游戏/动漫 → 奇幻热血 → game-rpg / light-novel-isekai / anime-shounen
```

> 配色变量（--p / --bg / --card 等）从所选 design-system 文件的 `:root` 块获取，不再使用 04-color-font。

### 步骤 3 · 逐页三元组规划表

> **"Tc 变体"列填写 `Tc-主基调`/`Tc-对比色`/`Tc-辅助色`，不写父风格名。**
> **"气质"列填写 design-system 风格关键词，替代原来的"配色方案号"。**

```markdown
| 页码 | 主题 | 布局 | 气质/design-system | Tc 变体 | 图表 | 说明 |
|------|------|------|--------------------|---------|------|------|
| P01 | 执行摘要 | D | dark-luxury 蓝主色 | Tc-主基调 | 雷达图 | 主基调 |
| P06 | 风险评估 | C | dark-luxury 红主色 | Tc-警示色 | 热力矩阵 | 对比色（同系换主色）|
| P10 | 趋势展望 | N | startup-pitch 亮色 | Tc-展望色 | 面积图 | 对比风格 |
```

规划约束自查：
```
□ 主基调配色 60-80%？对比色位于逻辑转折点（风险页/总结页）？
□ 同一布局连续 ≤2 次？背景类型统一？全程使用同一 Tc 模板？
□ 10 页使用 ≥6 种布局？
□ 至少 2 页使用业务图谱（diagram-types/）？
```

## 第二阶段：逐页生成

### 前置检查（每页生成前，→ 详细版见 `08-checklist.md`）

```
□ S0-3.5 扩写已完成（每张卡片正文≥60 字，含≥2 个数字）
□ 规划表已输出（三元组完整）
□ Tc 模板已定义（来自 02-design-system.md Step 0→D）
□ 本页 Lc 布局已声明（来自 03-layout.md 自创流程）
```

**任何一项未完成 = 禁止生成 HTML，返回第零阶段。**

### 逐页生成顺序

```
1. 01-canvas.md     → 四区 CSS 骨架
2. Tc 模板 CSS      → 应用专属 Tc（背景/卡片/字体/颜色变量，全来自 design-system，全程统一）
3. 03-layout.md     → 自创 Lc 布局（高度必须精确计算）
4. 05-content.md    → 填内容（对照 S0-3.5 扩写），选图表，遵守 SVG 颜色规则
5. 09-components.md → 选页眉/摘要栏变体
```

> 多样性通过**布局变化**（≥6 种）和**配色变化**实现，不得切换父风格。

### 生成后自查（每页写入 HTML 文件后，立即执行）

> **强制步骤：写入 HTML 文件后，读取技能目录下的 `check.txt`，将其全部内容作为自查提示词逐项核验，输出自查结果。不通过项必须修正并重新写入文件，再次自查直至全部通过。**

### 逐页报告（每页生成后必须输出，→ 完整话术见 `08-checklist.md`）

```
✓ p01.html 生成完成 | 体积：XX.X KB | 布局：X 变体 | 配色：方案 X | 自检：通过/失败

自检清单：
├── 文件体积：XX.X KB ≥ 18KB ✓
├── 卡片数量：X 张
├── 卡片正文字数：__字 / __字 / __字（均≥60 字）✓
├── SVG 图表：X 个（主图表 + 内嵌组件）✓
├── 数据组件：X 组（mini 卡片/数字块/进度条）✓
├── 四区高度：72+580+48+20=720px ✓
└── 布局容器高度：calc(580px - padding) ✓
```

**用户可随时叫停或要求修正。**

## 第三阶段：验证与质检（→ 详细检查表见 `08-checklist.md`）

### 🔴 必须通过（否则重做）

**结构（→ 01-canvas.md）：**
- [ ] 1017×720px，四区 72+580+48+20=720
- [ ] `.ct` 无 padding-top/bottom
- [ ] 布局容器 `height: calc(580px - padding-top)`
- [ ] flex 卡片有 `min-height:0` + 显式 height
- [ ] Grid 行高显式定义（非 1fr 自动撑高）

**内容密度（→ 05-content.md）：**
- [ ] 每页 HTML ≥ 18KB（内容页）；特殊页 ≥ 8KB
- [ ] 每张卡片：标题 + 正文(≥60字,≥2数字) + 第三层组件
- [ ] gap ≤ 8px，padding 上下 ≤ 8px，line-height ≤ 1.45
- [ ] 摘要栏有实质内容（非空）

**设计：**
- [ ] 主基调配色占 60-80%，对比色在逻辑转折处
- [ ] 全程同一 Tc 模板（禁切换父风格）
- [ ] **每页 body/::before/::after 背景 CSS 与 p00（封面）完全一致**（单页场景同样必须先定义背景）
- [ ] 10 页 ≥6 种不同布局
- [ ] 至少 2 页使用业务图谱

### 🟡 应当通过

- [ ] 每页 1-3 个 SVG
- [ ] 大数字 KPI 每页 ≤4 个
- [ ] 同款卡片组合 ≤4 次

### 🟢 加分项

- [ ] SVG 使用 `<linearGradient>`
- [ ] 关键数字用 `<em>` 等宽主色高亮
- [ ] 页脚进度条宽度与页码比例一致
- [ ] 至少 1 页有 SVG 时间轴或甘特图
