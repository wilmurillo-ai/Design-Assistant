# 错误码

## 通用错误码

| err_code | 说明 | 处理建议 |
|----------|------|---------|
| `-1` | 成功 | — |
| `0` | 成功（部分接口用0表示） | — |
| `1001` | 余额不足 | 检查账户余额，充值后重试 |
| `1002` | 积分不足 | 检查积分，积分充值后重试 |
| `2001` | 参数错误 | 检查请求参数格式、必填字段 |
| `2002` | 任务不存在 | 检查 task_id 是否正确 |
| `3001` | 资源不存在 | 检查上传的资源是否成功 |
| `4001` | 权限不足 | 检查 Token 是否有权限操作此任务 |
| `5001` | 服务端内部错误 | 稍后重试，或联系技术支持 |
| `401` | HTTP认证失败 | Token 无效或过期，重新获取 |
| `400` | HTTP请求格式错误 | 检查请求体格式是否符合要求 |

## AES加密相关错误

| 现象 | 原因 | 解决 |
|------|------|------|
| 解密后是乱码 | IV 与服务端不一致 | 确保 `x-ms-at` 时间戳正确传入解密 |
| 解密失败 | pkgname 对应的 Key 错误 | 检查 pkgname → Key 映射是否正确 |
| 响应无法解析 | 服务端返回了未加密的明文 | 尝试直接 JSON.parse 响应文本 |
| 解密文本前有垃圾字符 | 服务端调试信息夹杂在JSON前 | 截取 `text[find('{'):rfind('}')+1]` |

## 异步任务错误

| state | 说明 | 处理建议 |
|-------|------|---------|
| `1` | 处理中 | 继续轮询 |
| `2` | 成功 | 获取并使用返回数据 |
| `3` | 失败 | 查看 `err_message` 或 `data` 中的详细错误信息 |

## 任务创建失败

| err_code | 说明 |
|----------|------|
| `1001` | 余额不足 |
| `1002` | 积分不足 |
| `2001` | 参数错误（file_name/tpl_code/num 等字段问题） |
| `4001` | 未登录或 Token 无效 |

## 需求分析失败

| 原因 | 表现 | 解决 |
|------|------|------|
| plot 为空 | state!=2 | 确保传入有效的需求描述 |
| 视频内容不足 | 分析结果为空 | 确认视频素材是否有效 |
| 轮询超时 | 180s 内 state 未变为 2 | 增加超时时间或重试 |

## 音视频处理错误

| 错误 | 原因 | 解决 |
|------|------|------|
| ffmpeg 抽音频失败 | 视频格式不支持/路径错误 | 检查视频文件是否损坏 |
| ffprobe 解析失败 | 视频元数据缺失 | 换用其他视频素材 |
| 上传到 OSS 失败 | 网络问题/文件过大 | 检查网络，重试上传 |
| 图片 URL 不可访问 | OSS 文件未公开/过期 | 确保图片 `usage=seven_day_temp_img` 有效期足够 |

## TTS相关错误

| 错误 | 原因 | 解决 |
|------|------|------|
| 轮询返回 audio_url 为空 | TTS 处理中 | 继续轮询等待 |
| 下载 zip 失败 | URL 过期 | 下载后立即处理，不要等待 |
| 解压失败 | zip 文件损坏 | 重新下载 |

## 调试建议

1. **保存原始响应**：每次轮询成功后将完整响应保存到 JSON 文件
2. **打印关键字段**：如 `err_code`、`state`、`data` 结构
3. **时间戳记录**：记录每次请求的时间，便于排查超时问题
4. **响应头检查**：注意 `x-ms-at` 响应头，用于解密
5. **明文测试**：某些接口可能直接返回明文 JSON，尝试不解密直接解析
