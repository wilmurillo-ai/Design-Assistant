# 自动化智能日程管理系统 — 使用手册

> 基于 OpenClaw + 钉钉的团队共享日程管理方案。
> 团队所有成员通过钉钉机器人私聊即可操作，无需任何编程知识。

## 1. 系统简介

这是一个运行在阿里云服务器上的智能日程管理系统，具备以下能力：

| 能力 | 说明 |
|------|------|
| 增删改查 | 用自然语言添加、修改、删除、查询日程 |
| 冲突检测 | 新日程与已有日程时间重叠时，自动提示并建议替代时间 |
| 邮件汇总 | 每小时自动发送一封邮件，汇总未来 24 小时的日程 |
| 钉钉提醒 | 日程开始前 15 分钟，通过钉钉推送强提醒 |
| 模糊指令 | 支持"把明天的会往后挪一小时"这样的自然语言操作 |

**数据存储**：所有日程保存在服务器上的 SQLite 数据库中（非临时内存），重启不丢失。

## 2. 首次配置（管理员操作）

> 以下步骤只需管理员操作一次，普通成员不需要关心。

### 2.1 前置条件

确认以下服务已正常运行：

```bash
# 检查 OpenClaw 是否在运行
curl http://localhost:12463/health
# 预期输出: {"ok":true,"status":"live"}
```

确认钉钉机器人已配置并能正常回复消息。

### 2.2 配置邮箱（用于每小时日程汇总邮件）

编辑服务器上的配置文件：

```
文件位置: ~/.openclaw/workspace/schedule/config.json
```

将以下字段替换为**真实信息**：

```json
{
  "smtp": {
    "server": "smtp.qq.com",
    "port": 465,
    "use_ssl": true,
    "sender_email": "your-sender-email@example.com",
    "password": "your-smtp-password",
    "receiver_email": "your-receiver-email@example.com"
  },
  "timezone": "Asia/Shanghai",
  "reminder": {
    "email_interval_hours": 1,
    "qq_advance_minutes": 15
  }
}
```

#### 如何获取 QQ 邮箱 SMTP 授权码

1. 浏览器打开 https://mail.qq.com ，登录你的 QQ 邮箱
2. 点击顶部「设置」→「账户」
3. 向下找到「POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV 服务」
4. 点击「开启」POP3/SMTP 服务
5. 按提示用手机发送短信进行验证
6. 验证成功后会显示一个**授权码**（16 位字母），复制它
7. 粘贴到 `config.json` 的 `password` 字段

> 注意：授权码不是你的 QQ 密码，是专门用于第三方邮件客户端的独立密码。

#### 验证邮件配置是否正确

在服务器终端执行：

```bash
cd ~/.openclaw/workspace
python3 schedule/email_summary.py
```

如果输出 `"status": "success"`，说明邮件配置正确。如果报错，检查邮箱地址和授权码是否填写正确。

### 2.3 启动自动提醒

在钉钉里找到你的机器人，发送以下消息：

```
请帮我设置日程管理系统的自动提醒：
1. 每小时发送一次邮件日程汇总
2. 每 5 分钟检查一次即将开始的日程，提前 15 分钟通过钉钉提醒我
```

机器人会自动注册两个定时任务：
- **邮件汇总**：每小时整点执行，把未来 24 小时的日程发到你配置的收件邮箱
- **钉钉提醒**：每 5 分钟检查一次，如果有日程在 15~20 分钟内开始，自动通过钉钉推送提醒

配置完成后，系统即可投入使用。

## 3. 日常使用（所有成员）

### 3.1 打开钉钉，找到机器人

在钉钉的联系人或消息列表中找到配置好的 OpenClaw 机器人，直接发私聊消息即可。

### 3.2 添加日程

像跟同事说话一样告诉机器人：

```
明天下午2点到3点在会议室A开组会
```

```
后天上午10点讨论毕业论文，大概一个小时
```

```
下周一早上9点部门例会，在3楼大会议室
```

> 如果不说结束时间，默认持续 1 小时。

#### 遇到时间冲突

如果新日程和已有日程时间重叠，机器人会提示：

```
⚠️ 时间冲突！以下日程与新日程时间重叠：
  - #3 组会 (14:00~15:30)

💡 建议可选时间：
  1. 后移至 16:00~17:00
  2. 前移至 13:00~14:00

请问要选择哪个时间？或者直接强制添加？
```

你可以回复"选 1"或"还是强制添加吧"。

### 3.3 查看日程

```
我们明天有什么安排？
```

```
这周的日程有哪些？
```

```
有没有关于论文的安排？
```

### 3.4 修改日程

精确修改：

```
把日程 #3 的时间改到下午4点
```

模糊修改（机器人会自动理解）：

```
把明天下午的那个会往后挪一小时
```

```
组会提前半小时开始
```

```
把讨论会延长到5点结束
```

> 如果描述模糊且匹配多个日程，机器人会列出候选项让你选择。

### 3.5 删除日程

```
取消明天的组会
```

```
删除日程 #5
```

### 3.6 查看提醒状态

```
显示所有的定时提醒
```

```
关闭日程提醒
```

## 4. 进阶功能

### 4.1 模糊指令速查表

| 你说的话 | 系统理解为 |
|----------|-----------|
| "明天下午3点" | 明天 15:00 |
| "下周一上午" | 下周一 09:00 |
| "后天晚上8点" | 后天 20:00 |
| "大后天" | 今天 +3 天 |
| "往后挪一小时" | 开始和结束时间各 +1 小时 |
| "提前半小时" | 开始和结束时间各 -30 分钟 |
| "改到3点" | 调整开始时间，保持原来的时长 |
| "延长到4点" | 只修改结束时间 |
| "取消那个会" | 搜索匹配的会议 → 确认 → 删除 |

### 4.2 多人提醒

默认情况下，钉钉提醒只发送给设置提醒的那个人。如果团队多人都需要收到提醒：

- **方案 A**：每个需要提醒的成员分别给机器人发一次"帮我设置日程提醒"
- **方案 B**：在钉钉群里 @机器人 说"在群里设置日程提醒"，提醒会推送到群

### 4.3 手动触发汇总

不想等每小时自动发送，可以随时说：

```
帮我看看今天还有什么安排
```

```
未来 48 小时的日程有哪些？
```

## 5. 常见问题

### Q: 机器人不回复消息怎么办？

1. 检查 OpenClaw 是否在运行：在服务器终端执行 `curl http://localhost:12463/health`
2. 检查钉钉应用是否已在开放平台发布
3. 查看日志：`openclaw logs --follow`

### Q: 邮件收不到怎么办？

1. 检查 `config.json` 中的邮箱和授权码是否正确
2. 在服务器终端执行 `python3 ~/.openclaw/workspace/schedule/email_summary.py` 查看报错信息
3. 检查收件邮箱的垃圾箱

### Q: 钉钉提醒没收到？

1. 确认已通过钉钉对机器人说过"设置日程提醒"
2. 提醒只在日程开始前 15~20 分钟触发，且每个日程只提醒一次
3. 钉钉可能会限制机器人的主动消息频率

### Q: 日程数据存在哪里？

存储在服务器的 SQLite 数据库中：`~/.openclaw/workspace/schedule/schedules.db`。
重启服务器、重启 OpenClaw 都不会丢失数据。

### Q: 所有人操作的是同一份日程吗？

是的。这是一个**团队共享日程表**，所有通过钉钉与机器人交互的成员看到和操作的都是同一份数据。

## 6. 技术架构

```
团队成员（钉钉私聊）
       │
       ▼
 OpenClaw Gateway（端口 12463）
       │
       ├── 钉钉连接器（dingtalk-connector 插件）
       │
       ▼
 OpenClaw Agent（LLM 理解自然语言意图）
       │
       ├── 读取 SKILL.md（行为规范）
       │
       ▼
 Python 日程管理模块（schedule_manager.py）
       │
       ▼
 SQLite 数据库（schedules.db）
       │
       ├── Cron 定时任务 ─── 邮件汇总脚本 ─── SMTP 发送
       │
       └── Cron 定时任务 ─── 提醒检查脚本 ─── 钉钉推送
```

### 文件说明

| 文件 | 作用 |
|------|------|
| `schedule/schedule_manager.py` | 核心模块，所有日程的增删改查、冲突检测、时间建议 |
| `schedule/email_summary.py` | 每小时邮件汇总脚本 |
| `schedule/check_upcoming.py` | 即将开始的日程检查 + 提醒标记 |
| `schedule/config.json` | 邮箱 SMTP 配置 |
| `schedule/schedules.db` | SQLite 数据库（自动创建） |
| `skills/schedule-manager/SKILL.md` | Agent 行为规范（教 AI 如何使用日程系统） |
