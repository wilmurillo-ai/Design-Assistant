# 一句话需求转 Brief 规则

当用户只给一句话需求时，按下面规则推断：

## 1. 模式判断

- 出现“改写、重写、参考、基于这篇”时，优先判为 `rewrite`
- 否则优先判为 `title_to_article`

## 2. 受众判断

- 提到“产品经理”时：产品经理和业务负责人
- 提到“创业、创始人、老板”时：创业者和中小团队管理者
- 提到“运营、新媒体”时：运营和内容从业者
- 提到“职场、打工人”时：普通职场人
- 提到“程序员、开发、技术”时：技术从业者和技术管理者
- 提到“AI、人工智能”时：关注 AI 趋势与应用的公众号读者
- 没有明显信号时：关注行业趋势的微信公众号读者

## 3. 角度判断

- 提到“为什么”时：围绕原因和后果展开
- 提到“怎么、如何”时：围绕方法和动作建议展开
- 提到“趋势、变化”时：围绕结构性变化展开
- 改写模式默认：从更适合公众号传播的角度重写
- 其他情况默认：找到一个清晰、有传播性的切口

## 4. 重点提取

从用户需求里找这些表达：

- 重点
- 包括
- 要讲
- 讲清
- 分析
- 拆解
- 原因
- 建议
- 案例

把包含这些信号的片段提取成 key points。
