# 素材分析指南

## 1. 总体流程

1. 获取抖店商品链接。如果当前流程中未提供链接，则向使用者（运营、产品或需求发起人）询问并确认链接的正确性。
2. 执行 Python 脚本 `urlformater.py` 对原始抖店商品链接做简化处理，将链接中的关键信息（如商品 ID）抽取并整理为统一格式的「简化链接」。
3. 执行 `icp.py` 脚本，将简化后的链接作为参数传入固定的服务（service_id = `2531055211182946`），获取与该链接对应的素材信息（返回 JSON 结构，字段以接口实际输出为准）。


## 2. 操作指南

### 2.1 获取抖店商品链接

1. 由业务方提供
   - 在需求评审或日常沟通中，请业务方直接给出完整的抖店商品链接（推荐从浏览器地址栏或商品分享链接中复制）。
   - 要求链接必须以 `http://` 或 `https://` 开头，且能在浏览器中正常打开。

2. 自己从页面中获取
   - 打开目标抖店商品的详情页（PC 端或移动端均可）。
   - 从浏览器地址栏复制当前页面的完整 URL，或使用页面提供的「复制链接/分享链接」能力获取 URL。
   - 将链接粘贴到当前分析流程所需的输入位置（例如命令行参数、表单输入框等）。

3. 链接缺失时的处理
   - 如果自动化流程或上游系统没有传入 `url`，必须先暂停后续步骤。
   - 主动向使用者询问：
     - 说明当前需要一个「抖店商品链接」来完成分析。
     - 告知如何从浏览器或抖店后台获取该链接。
   - 在拿到链接后，手动或自动重新触发后续流程。

4. 链接有效性校验建议
   - 使用浏览器打开该链接，确保可以正常访问且指向期望的商品。
   - 检查链接中是否包含明显的商品标识（如 `id`、`product_id` 等参数）。
   - 如有多条链接，建议与需求方确认最终以哪一条为准。


### 2.2 执行 `urlformater.py` 简化链接

`urlformater.py` 脚本用于将原始抖店商品链接统一转化为简化格式，方便后续服务识别和处理。

1. 进入脚本所在目录

一般情况下，脚本位于项目根目录下的 `scripts/` 目录：

```bash
cd <项目根目录>
```

2. 执行简化命令

在命令行中执行：

```bash
python3 scripts/urlformater.py <url> --format simplify --keys id
```

参数说明：

- `<url>`：原始抖店商品链接（上一步获取到的完整 URL）。
- `--format simplify`：指定输出为简化后的链接格式。
- `--keys id`：指定以商品 `id` 作为关键字段进行提取和拼接。

3. 期望输入与输出

- 输入：一条完整的抖店商品链接，例如：
  - `https://xxx.douyin.com/...?id=1234567890`（示例，仅用于说明）。
- 输出：一条简化后的链接，通常包含规范化的域名与商品 ID，例如：
  - `https://xxx.douyin.com/item/1234567890`（实际格式以脚本实现为准）。

脚本会将简化后的链接输出到标准输出（stdout），建议：

- 在命令行中直接查看输出，或
- 在自动化流程中将输出结果捕获并传递给下一步脚本（`icp.py`）。


### 2.3 执行脚本获取链接素材信息

在获得简化后的商品链接后，通过 `icp.py` 调用固定服务，获取与该链接关联的素材信息。

1. 基础命令

```bash
python3 scripts/icp.py --service-id <服务ID> --params '{"url": "<简化后的链接>"}' --output <输出json文件路径>
```

2. 参数说明：

- `<服务ID>`：服务 ID 固定为 `2531055211182946`，无需修改。
- `<简化后的链接>`：前一步得到的简化后的抖店商品链接。
- `<输出json文件路径>`：[必填] 指定将返回的 JSON 结果保存到的文件路径。你可以根据需要自定义文件路径，但建议：
  - 路径应指向一个可写的临时文件位置（如 `/tmp/` 目录）。
  - 文件名应以 `.json` 结尾，例如 `./tmp/material-xxx.json`。

3. 输出结果

- 脚本会以 JSON 结构返回与该链接相关的素材信息，返回JSON结构描述参考`./references/素材分析结果说明.md`
- 读取 `<输出json文件路径>` 中的 `payload` 结果，解析并展示相关素材信息。
- 具体字段和结构以接口实际返回为准，可根据业务需要进行解析和展示。

## 3. 错误处理

在抖店链接分析过程中，常见错误类型及处理建议如下。

### 3.1 链接相关错误

1. 未提供链接
   - 错误表现：脚本或流程提示缺少 `url` 参数，或没有任何链接输入。
   - 处理建议：
     - 明确告知使用者：当前操作需要提供抖店商品链接。
     - 指导使用者从浏览器或抖店后台获取商品链接并重新执行流程。

2. 链接格式不正确
   - 错误表现：
     - `urlformater.py` 报错无法解析链接，或
     - 链接在浏览器中无法打开，返回 404/跳转异常。
   - 处理建议：
     - 请使用浏览器验证该链接是否可正常访问。
     - 确认链接来源是否为抖店商品详情页，而不是活动页、搜索结果页等。
     - 如仍无法使用，请联系提供链接的同学重新确认商品地址。


### 3.2 `urlformater.py` 执行错误

1. 找不到脚本或命令
   - 错误表现：如 `python3: can't open file 'scripts/urlformater.py': [Errno 2] No such file or directory`。
   - 处理建议：
     - 确认当前工作目录是否为项目根目录。
     - 使用 `ls scripts/` 检查 `urlformater.py` 是否存在。
     - 若脚本缺失，联系项目维护者补齐脚本或更新使用文档。

2. Python 环境问题
   - 错误表现：`python3` 命令不存在，或依赖包缺失导致脚本运行失败。
   - 处理建议：
     - 确认本地已安装 Python 3，并能在命令行中执行 `python3 --version`。
     - 按项目文档安装必要依赖（例如使用 `pip install -r requirements.txt`）。

3. 链接解析失败
   - 错误表现：脚本报错提示无法从链接中提取 `id`。
   - 处理建议：
     - 检查链接是否为标准的抖店商品详情链接，是否包含 `id` 或其他脚本所需字段。
     - 如为新类型链接，反馈给脚本维护者评估是否需要更新解析逻辑。


### 3.3 获取素材信息的错误

1. 脚本执行失败
   - 错误表现：命令行报错（如找不到脚本、依赖缺失、权限不足等）。
   - 处理建议：
     - 告知用户具体报错信息（例如「找不到脚本路径」「缺少某依赖包」）；
     - 建议检查脚本路径、Python 环境，以及是否按文档完成依赖安装。

2. 服务调用失败
   - 错误表现：服务返回非 0 状态码、超时、网络异常等。
   - 处理建议：阅读并参考`./references/接口错误指南.md`，遵循错误码的处理建议，提示用户根据错误码进行处理。

3. 返回内容为空或不符合预期
   - 错误表现：创作分析结果为空对象、缺少关键分析字段，或与文档约定不一致。
   - 处理建议：
     - 提示用户当前返回结果异常（例如「未获取到有效的创作分析结论」）；
     - 建议对比其他正常样例结果，或联系服务侧确认是否存在灰度、配置或版本问题。



### 3.4 通用错误处理建议

- 清晰告知用户：
  - 当前操作在哪一步失败（获取链接、简化链接、获取素材信息）。
  - 失败的直接原因（如「缺少链接参数」「链接格式不正确」「服务调用超时」等）。
- 提供至少一条可执行的下一步建议，例如：
  - 请重新提供可在浏览器正常打开的抖店商品链接。
  - 请先在项目根目录下执行命令，或检查脚本是否存在。
  - 请稍后重试或联系服务负责人，并附上完整的错误信息和命令参数。

通过以上规范流程与错误处理建议，可以帮助使用者快速完成抖店链接分析，并在出现问题时具备明确的排查路径。
