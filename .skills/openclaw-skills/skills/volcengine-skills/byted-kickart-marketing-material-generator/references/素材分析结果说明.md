# 接口返回结果说明

本文档基于提供的 JSON 元信息，说明接口返回结果的整体结构、字段含义及必填情况，供调用方和前后端联调参考。

## 一、顶层结构

响应体整体为一个对象，包含两个字段：

- `responseMetadata`：响应结果元信息，表示本次接口调用是否成功。
- `result`：业务结果数据及执行状态。

```jsonc
{
  "responseMetadata": { ... },
  "result": { ... }
}
```

### 1. responseMetadata（响应结果）

表示接口调用成功或失败。

| 字段名 | 类型     | 必填 | 说明                         |
|--------|----------|------|------------------------------|
| code   | integer  | 是   | 响应码，`0` 表示成功，其它为失败 |
| message| string   | 是   | 失败描述，当 `code != 0` 时返回失败原因 |

### 2. result（接口输出）

承载具体的业务数据及执行状态。

| 字段名    | 类型    | 必填 | 说明         |
|-----------|---------|------|--------------|
| data      | object  | 是   | 核心业务数据 |
| queueInfo | object  | 是   | 队列及排队信息 |
| status    | string  | 是   | 执行状态     |


## 二、result.data 结构

`data` 为对象，包含指标信息和业务输出。

| 字段名  | 类型   | 必填 | 说明         |
|---------|--------|------|--------------|
| metrics | object | 是   | 指标信息集合 |
| payload | object | 是   | 业务输出内容 |

### 1. data.metrics

当前定义了一个指标对象 `nodesExecutionTime`。

#### 1.1 metrics.nodesExecutionTime

记录各节点的执行耗时等数值型指标（具体单位与含义以业务定义为准）。

- 类型：object
- 必填：是
- 字段：键为字符串形式的节点 ID，值为 number 类型。

示例：

```jsonc
"metrics": {
  "nodesExecutionTime": {
    "1": 12.3,
    "2": 8.5,
    "3": 0.0,
    // ... 其他节点
    "47": 5.6
  }
}
```


### 2. data.payload（接口输出）

`payload` 为接口的业务输出部分，主要描述商品相关的多媒体素材与基础信息。

| 字段名               | 类型    | 必填 | 说明           |
|----------------------|---------|------|----------------|
| product_detail_images| array   | 是   | 商品详情图片列表 |
| product_main_images  | array   | 是   | 商品主页图片列表 |
| product_title        | string  | 是   | 商品标题       |
| product_videos       | array   | 是   | 商品宣传视频列表 |
| selling_point        | array   | 是   | 商品卖点文案列表 |

#### 2.1 payload.product_detail_images（商品详情图片）

- 类型：array<object>
- 必填：是
- 含义：商品详情页中使用的图片列表。

单个图片对象字段：

| 字段名 | 类型    | 必填 | 说明       |
|--------|---------|------|------------|
| height | integer | 是   | 图片高度   |
| type   | string  | 是   | 图片类型（如格式或业务类型，具体取值以实际为准） |
| url    | string  | 是   | 图片 URL 地址 |
| width  | integer | 是   | 图片宽度   |

#### 2.2 payload.product_main_images（商品主页图片）

- 类型：array<object>
- 必填：是
- 含义：商品主展示区域（如列表/首屏）的图片列表。

单个图片对象字段与 `product_detail_images` 相同：

| 字段名 | 类型    | 必填 | 说明       |
|--------|---------|------|------------|
| height | integer | 是   | 图片高度   |
| type   | string  | 是   | 图片类型   |
| url    | string  | 是   | 图片 URL 地址 |
| width  | integer | 是   | 图片宽度   |

#### 2.3 payload.product_title（商品标题）

- 类型：string
- 必填：是
- 说明：商品展示标题。

#### 2.4 payload.product_videos（商品宣传视频）

- 类型：array<object>
- 必填：是
- 含义：与商品相关的宣传视频列表。

单个视频对象字段：

| 字段名 | 类型    | 必填 | 说明                         |
|--------|---------|------|------------------------------|
| duration | number  | 否   | 视频时长，单位和精度以实际为准 |
| height   | integer | 否   | 视频高度                     |
| type     | string  | 否   | 视频类型（如格式或业务类型）   |
| url      | string  | 否   | 视频 URL 地址                 |
| width    | integer | 否   | 视频宽度                     |

> 注：上述字段在 schema 中未显式标记为必填，实际必填与否以后端实现为准。

#### 2.5 payload.selling_point（商品卖点）

- 类型：array<string>
- 必填：是
- 说明：商品卖点文案列表，每一项为一条卖点描述。


## 三、result.queueInfo 结构

`queueInfo` 描述任务在执行队列中的信息。

| 字段名             | 类型    | 必填 | 说明                         |
|--------------------|---------|------|------------------------------|
| executionStartTime | integer | 是   | 执行开始时间，具体单位以业务定义为准（如毫秒时间戳等） |
| queuePosition      | integer | 是   | 队列位置（排队顺序号），具体含义以业务定义为准 |


## 四、result.status 字段

- 字段名：`status`
- 类型：string
- 必填：是
- 说明：表示当前任务的执行状态，例如「排队中」「执行中」「成功」「失败」等，具体枚举值以业务约定为准。


## 五、使用说明与校验建议

1. 判断接口调用是否成功：
   - 先检查 `responseMetadata.code` 是否为 `0`；
   - 若不为 `0`，可结合 `responseMetadata.message` 提示错误原因。
2. 在调用方解析字段时，应：
   - 严格按照本文档说明的层级取值；
   - 对可能为空或新增的字段做好兼容处理；
   - 对 `nodesExecutionTime`、`queueInfo` 等数值字段的单位和含义与后端达成一致后再做展示或计算。
