# 故事板创作结果说明

本文档基于提供的 JSON 元信息，说明接口返回的故事板创作结果整体结构、字段含义及必填情况，供调用方和前后端联调参考。

## 一、顶层结构

响应体整体为一个对象，包含两个字段：

- `responseMetadata`：接口调用状态信息；
- `result`：业务结果数据及执行状态。

```jsonc
{
  "responseMetadata": { ... },
  "result": { ... }
}
```

### 1. responseMetadata（接口调用状态）

表示本次接口调用是否成功，以及失败原因。

| 字段名  | 类型    | 必填 | 说明                         |
|---------|---------|------|------------------------------|
| code    | integer | 是   | 接口调用状态码，具体成功值以业务约定为准 |
| message | string  | 是   | 接口调用失败说明，当调用失败时用于描述原因 |

> 建议以 `code` 是否为约定的成功值（例如 0）作为调用是否成功的判断依据，具体成功/失败规则与后端统一约定。

### 2. result

承载具体的业务数据及执行状态。

| 字段名    | 类型   | 必填 | 说明         |
|-----------|--------|------|--------------|
| data      | object | 是   | 核心业务数据 |
| queueInfo | object | 是   | 队列及排队信息 |
| status    | string | 是   | 执行状态     |


## 二、result.data 结构

`data` 为对象，包含指标信息和业务输出内容。

| 字段名  | 类型   | 必填 | 说明         |
|---------|--------|------|--------------|
| metrics | object | 是   | 指标信息集合 |
| payload | object | 是   | 业务输出内容 |


### 1. data.metrics

当前定义了一个指标对象 `nodesExecutionTime`，记录各节点的数值指标（通常为执行耗时），具体单位与含义以业务定义为准。

#### 1.1 metrics.nodesExecutionTime

- 类型：object
- 必填：是
- 含义：以节点 ID 为键的指标映射，值为 number 类型。

示例：

```jsonc
"metrics": {
  "nodesExecutionTime": {
    "1": 12.3,
    "3": 8.5,
    "7": 0.0,
    "63": 5.4,
    "64": 2.1,
    "66": 1.7
  }
}
```


### 2. data.payload（业务输出）

`payload` 为接口的业务输出部分，描述故事分镜（storyboard）相关的规划结果。

| 字段名     | 类型  | 必填 | 说明                     |
|------------|-------|------|--------------------------|
| storyboards| array | 是   | 故事列表，可能包含多个备选故事方案 |

#### 2.1 payload.storyboards（故事列表）

- 类型：array<object>
- 必填：是
- 说明：每个元素表示一个完整的故事方案，包含 BGM 描述、分镜镜头列表、故事概述、成片时长与视觉风格等信息。

单个故事对象字段如下：

| 字段名          | 类型    | 必填 | 说明                         |
|-----------------|---------|------|------------------------------|
| bgm_description | string  | 否   | 背景音乐描述，如风格、节奏等 |
| shots           | array   | 否   | 分镜镜头列表                  |
| storyline       | string  | 否   | 故事简述，概括故事内容主线    |
| video_duration  | integer | 否   | 成片总时长，单位以业务约定为准 |
| visual_style    | string  | 否   | 视觉风格说明（如写实、二次元等） |

> 注：上述字段在 schema 中未显式标记为必填，实际必填与否以后端实现为准。

##### 2.1.1 storyboards.shots（分镜镜头列表）

`shots` 为数组，每个元素代表一个镜头/片段。

单个镜头对象字段如下：

| 字段名       | 类型    | 必填 | 说明                                     |
|--------------|---------|------|------------------------------------------|
| duration     | integer | 否   | 镜头时长，单位以业务约定为准            |
| image_indexs | array   | 否   | 使用到的图片索引列表（整数）            |
| image_urls   | array   | 否   | 镜头对应的图片 URL 列表                 |
| prompt       | string  | 否   | 镜头提示词/画面文案，用于描述创作意图   |
| subtitle     | string  | 否   | 镜头字幕文案，用于成片展示              |

字段补充说明：

- `image_indexs`：通常与上游素材列表的顺序对应，用于指明此镜头引用哪些素材；
- `image_urls`：可直接用于预览或生成镜头画面；
- `prompt` 与 `subtitle`：可用于指导生成画面和展示文案，两者内容可能不同，需按业务需求分别使用。


## 三、result.queueInfo 结构

`queueInfo` 描述任务在执行队列中的信息。

| 字段名             | 类型    | 必填 | 说明                         |
|--------------------|---------|------|------------------------------|
| executionStartTime | integer | 是   | 执行开始时间，具体单位以业务定义为准（如毫秒时间戳等） |
| queuePosition      | integer | 是   | 队列位置（排队顺序号），数值越小通常表示越靠前 |


## 四、result.status 字段

- 字段名：`status`
- 类型：string
- 必填：是
- 说明：表示当前任务的执行状态，例如「排队中」「执行中」「成功」「失败」等，具体枚举值以业务约定为准。


## 五、使用说明与校验建议

1. 调用结果判定
   - 首先检查 `responseMetadata.code` 是否为约定的成功状态；
   - 若非成功状态，可结合 `responseMetadata.message` 作为错误提示文案返回给调用方/前端。

2. 字段解析建议
   - 解析数据时需严格按照本文档层级访问字段：如 `result.data.metrics.nodesExecutionTime`、`result.data.payload.storyboards` 等；
   - 对于 `nodesExecutionTime` 的数值字段，与后端确认单位（如毫秒/秒）后再进行展示或统计分析；
   - 对 `storyboards` 及其内部字段（特别是 `shots`、`prompt`、`subtitle` 等），建议在前端或下游服务中做好空值与缺失字段的兼容处理。

3. 业务使用示例
   - 可基于 `storyboards` 中的 `storyline`、`visual_style` 与 `bgm_description`，用于向运营/创作者展示不同故事方案；
   - 利用 `shots` 中的 `duration`、`image_urls`、`subtitle` 等信息，驱动自动化分镜预览或生成视频脚本；
   - 通过 `nodesExecutionTime` 与 `queueInfo` 监控整体处理链路的性能和排队情况，辅助容量评估与优化。
