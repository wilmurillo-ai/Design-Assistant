# 消费成片指南

## 1. 总体流程

1. 从上下文中获取用户指定的故事板数据。
2. 参考 `./references/故事板创作结果说明.md` 对故事板的字段和含义进行理解，确认本次消费成片所需的关键信息是否齐全。
3. 执行 Python 脚本，调用 River 服务的消费成片接口，将用户指定的故事板数据作为入参传入（或从上下文中在脚本内读取）。
4. 获取消费成片接口返回结果，对其中的核心指标、分析结论及建议进行解读，形成对当前故事线的消费成片结果。


## 2. 执行步骤

### 2.1 获取用户指定的故事板数据

1. 从上下文中读取
   - 若当前流程已在前置步骤完成「故事板创作」，通常会将故事板数据以 JSON 形式保存在：
     - 上下文变量（如工作流上下文、任务上下文）；
     - 中间文件（如本地 `*.json` 文件）；
     - 统一的数据存储（如 Redis、数据库等）。
   - 在进入创作分析前，需要明确：
     - 上下文中素材扫描 JSON 的存储位置；
     - 对应 key / 文件路径 / 查询条件。

2. 阅读并理解结果格式
   - 参考文档：`./references/故事板创作结果说明.md`。
   - 重点关注：
     - 故事板标识（如故事板 ID、链接、来源渠道等）；
     - 故事板内容信息（标题、文案、图片/视频结构化信息）；
     - 任何与创作质量、合规性、风险相关的字段。
   - 在正式调用创作分析接口前，建议先人工抽样检查 1~2 条 JSON 结果，确认字段与预期一致。

3. 校验数据完整性
   - 检查必需字段是否存在、类型是否正确（例如字符串、数字、数组等）；
   - 对明显缺失或异常的字段进行记录，必要时在调用创作分析接口前进行补全或过滤。

4. 询问用户确认故事板数据是否符合预期
   - 若故事板数据中包含多个故事板，建议用户人工检查确认每个故事板的字段值是否正确、是否符合预期。
   - 若发现异常或错误字段，建议用户及时反馈并在调用创作分析接口前进行修正。
   - 当用户确认故事板数据无误后，才执行下一步调用创作分析接口。


### 2.2 执行 Python 脚本调用消费成片接口

1. 准备执行环境
   - 确认本地或运行环境已安装 Python 3，并能正常执行 `python3 --version`；
   - 确保消费成片脚本已就绪（例如位于某个固定路径下），并完成依赖安装（如 `pip install -r requirements.txt`）。

2. 脚本调用方式

```bash
python3 icp.py --service-id <消费成片服务ID> --input <故事板JSON文件路径> --output <输出json文件路径> --forward <前置处理流程>
```

其中：
- `<消费成片服务ID>`：实际的消费成片服务 ID，固定为 4910549050609106
- `<故事板JSON文件路径>` 根据消费成片接口的参数约定来设置，传入用户指定的故事板 JSON 文件路径。
- `<输出json文件路径>`：[必填] 指定将返回的 JSON 结果保存到的文件路径。你可以根据需要自定义文件路径，但建议：
  - 路径应指向一个可写的临时文件位置（如 `/tmp/` 目录）。
  - 文件名应以 `.json` 结尾，例如 `./tmp/result-xxx.json`。
- `<前置处理流程>`：固定传入 `storyboard`

3. 输出结果
- 脚本会以 JSON 结构返回基于用户指定的故事板数据的消费成片结果。
- 读取 `<输出json文件路径>` 中的 `video_url` 视频 URL 地址，结合用户任务展示给用户。
- **[强制]从视频URL链接中获取x-expires参数值,并将其转换为可读的时间格式（如2023-12-31 23:59:59）,然后提醒用户在过期前及时下载或使用**
- 具体字段和结构以接口实际返回为准，可根据业务需要进行解析和展示。


## 3. 错误处理

在创作分析流程中，常见错误及处理建议如下。

### 3.1 上下文读取或 JSON 结果错误

1. 未能从上下文中获取素材扫描结果
   - 错误表现：脚本入参为空、读取文件失败、或找不到对应 key。
   - 处理建议：
     - 明确告知用户「当前未获取到素材扫描 JSON 结果」；
     - 提示检查前置素材扫描流程是否成功执行、以及上下文/文件路径配置是否正确；
     - 必要时提供示例路径或参数配置方式，协助用户补充或重新触发素材扫描。

2. JSON 格式不合法
   - 错误表现：解析 JSON 时抛出异常（如 `JSONDecodeError`）、字段类型与约定不符等。
   - 处理建议：
     - 指出错误发生在「素材扫描结果解析」阶段；
     - 建议用户使用 JSON 校验工具或日志打印确认原始内容；


### 3.2 调用创作分析接口错误

1. 脚本执行失败
   - 错误表现：命令行报错（如找不到脚本、依赖缺失、权限不足等）。
   - 处理建议：
     - 告知用户具体报错信息（例如「找不到脚本路径」「缺少某依赖包」）；
     - 建议检查脚本路径、Python 环境，以及是否按文档完成依赖安装。

2. 服务调用失败
   - 错误表现：服务返回非 0 状态码、超时、网络异常等。
   - 处理建议：阅读并参考`./references/接口错误指南.md`，遵循错误码的处理建议，提示用户根据错误码进行处理。

3. 返回内容为空或不符合预期
   - 错误表现：消费成片结果为空对象、缺少关键分析字段，或与文档约定不一致。
   - 处理建议：
     - 提示用户当前返回结果异常（例如「未获取到有效的消费成片结论」）；
     - 建议对比其他正常样例结果，或联系服务侧确认是否存在灰度、配置或版本问题。


### 3.3 向用户反馈的通用建议

在出现错误时，应做到：

- 准确告知用户错误发生的阶段（获取素材扫描结果 / 调用消费成片接口 / 解析返回结果）；
- 明确说明错误原因（例如「上下文缺少素材扫描 JSON」「接口返回参数校验失败」「服务调用超时」等）；
- 同时提供至少一条可执行的下一步建议，如：
  - 请先确认前置故事板任务是否成功，并在上下文中补充 JSON 结果；
  - 请检查并修正入参 JSON 格式后重新调用；
  - 请稍后重试或联系服务负责人，并提供完整的命令和错误日志。

通过以上流程和错误处理建议，可以帮助使用者稳定地完成消费成片任务，并在发生异常时具备清晰的排查路径。
