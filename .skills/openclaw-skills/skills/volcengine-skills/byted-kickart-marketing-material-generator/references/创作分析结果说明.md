# 接口返回结果说明（素材多模态分析）

本文档基于提供的 JSON 元信息，说明接口返回结果的整体结构、字段含义及必填情况，供调用方和前后端联调参考。

## 一、顶层结构

响应体整体为一个对象，包含两个字段：

- `responseMetadata`：接口调用状态信息；
- `result`：业务结果数据。

```jsonc
{
  "responseMetadata": { ... },
  "result": { ... }
}
```

### 1. responseMetadata（接口调用状态）

表示本次接口调用是否成功，以及失败原因。

| 字段名  | 类型    | 必填 | 说明                 |
|---------|---------|------|----------------------|
| code    | integer | 是   | 接口调用状态码      |
| message | string  | 是   | 接口调用失败说明，当调用失败时用于描述原因 |

> 建议以 `code` 是否为约定的成功值（例如 0）作为调用是否成功的判断依据，具体成功/失败规则以业务约定为准。

### 2. result

承载具体的业务数据。

| 字段名 | 类型   | 必填 | 说明         |
|--------|--------|------|--------------|
| data   | object | 是   | 核心业务数据 |


## 二、result.data 结构

`data` 为对象，包含指标信息和业务输出内容。

| 字段名  | 类型   | 必填 | 说明         |
|---------|--------|------|--------------|
| metrics | object | 是   | 指标信息集合 |
| payload | object | 是   | 业务输出内容 |


### 1. data.metrics

当前定义了一个指标对象 `nodesExecutionTime`，记录各节点的数值指标（如执行耗时），具体单位与含义以业务定义为准。

#### 1.1 metrics.nodesExecutionTime

- 类型：object
- 必填：是
- 含义：以节点 ID 为键的指标映射，值为 number 类型。

示例：

```jsonc
"metrics": {
  "nodesExecutionTime": {
    "5": 12.3,
    "7": 8.5,
    "11": 0.0,
    // ... 其他节点
    "198": 6.7
  }
}
```


### 2. data.payload（业务输出）

`payload` 为接口的业务输出部分，描述素材的多模态分析结果。

| 字段名       | 类型   | 必填 | 说明           |
|--------------|--------|------|----------------|
| aspect_ratio | string | 是   | 长宽比，例如 "9:16"、"16:9" 等 |
| duration     | array  | 是   | 时长信息数组，元素为整数，具体含义以业务约定为准 |
| language     | string | 是   | 语言标识，例如内容识别出的语种代码或名称 |
| media        | array  | 是   | 媒体素材列表（图像/视频等） |

#### 2.1 payload.aspect_ratio（长宽比）

- 类型：string
- 必填：是
- 说明：素材画面的宽高比例表示，用于表示出品比适配情况等。

#### 2.2 payload.duration（时长信息）

- 类型：array<integer>
- 必填：是
- 说明：时长相关的整数数组，可用于表示整体时长或分段时长等，具体含义以业务约定为准。

#### 2.3 payload.language（语言）

- 类型：string
- 必填：是
- 说明：素材内容的语言信息，例如识别出的语种。

#### 2.4 payload.media（媒体素材列表）

- 类型：array<object>
- 必填：是
- 说明：素材中各媒体资源的列表，每个元素描述一条媒体（如一帧画面、一个片段等）。

单个媒体对象字段如下：

| 字段名   | 类型    | 必填 | 说明                         |
|----------|---------|------|------------------------------|
| analysis | object  | 否   | 对该媒体的文本分析结果       |
| height   | integer | 否   | 媒体高度（像素）             |
| type     | string  | 否   | 媒体类型（如 image / video 等） |
| url      | string  | 否   | 媒体访问 URL                 |
| width    | integer | 否   | 媒体宽度（像素）             |

> 以上字段在 schema 中未显式标记为必填，实际必填与否以后端实现为准。

##### 2.4.1 media.analysis

`analysis` 为对象，包含对该媒体的语义分析结果。

| 字段名      | 类型   | 必填 | 说明                 |
|-------------|--------|------|----------------------|
| description | string | 是   | 对画面/媒体内容的文字描述 |
| tags        | array  | 是   | 标签列表，每个元素为一个标签字符串 |


## 三、使用说明与校验建议

1. 调用结果判定
   - 首先检查 `responseMetadata.code` 是否为约定的成功状态；
   - 若非成功状态，可结合 `responseMetadata.message` 作为错误提示文案。

2. 字段解析建议
   - 解析数据时需严格按照本文档层级访问字段：`result.data.metrics`、`result.data.payload` 等；
   - 对于未来可能新增的节点 ID 或 payload 字段，应保持向前兼容，不要做强校验抛错；
   - 对 `nodesExecutionTime` 中的数值字段，要与后端确认单位（如毫秒/秒）再用于展示或计算。

3. 业务使用示例
   - 可基于 `aspect_ratio` 和 `duration` 评估素材在不同投放场景下的适配度；
   - 基于 `media.analysis.description` 与 `tags`，实现自动打标、检索或创作辅助；
   - 结合 `nodesExecutionTime` 指标监控各节点性能，进行链路优化。
