# 免费套餐查询开通指南

## 1. 总体流程

执行 Python 脚本`icp.py`，查询当前用户是否已开通免费套餐并判断套餐是否过期，如果开通并且套餐未过期，明确告知用户套餐可用以及套餐有效期。

## 2. 操作指南

### 2.1 执行脚本查询或开通免费套餐

通过 `icp.py` 调用固定服务，获取用户当前是否已开通免费套餐并判断套餐是否过期。

1. 基础命令示例

```bash
python3 scripts/icp.py --register-combo
```

2. 输出结果

- 如果用户已开通免费套餐且套餐未过期，脚本会以 JSON 结构返回与该套餐相关的信息，返回JSON结构如下：
```json
{"code": 0, "expire_time": "2026-03-23T11:57:59Z", "message": "Valid opc_kickart_arkclaw combo found"}
```
- 从返回结果中提取`expire_time`字段，告知用户套餐有效期。



## 3. 错误处理

在套餐开通和查询过程中，常见错误类型及处理建议如下。

### 3.1 Python 环境问题
   - 错误表现：`python3` 命令不存在，或依赖包缺失导致脚本运行失败。
   - 处理建议：
     - 确认本地已安装 Python 3，并能在命令行中执行 `python3 --version`。
     - 按项目文档安装必要依赖（例如使用 `pip install -r requirements.txt`）。


### 3.2 套餐查询相关错误

1. 脚本执行失败
   - 错误表现：命令行报错（如找不到脚本、依赖缺失、权限不足等）或者返回非 0 状态码。
   - 处理建议：
     - 读取错误信息，阅读并参考`./references/接口错误指南.md`，遵循错误码的处理建议，提示用户根据错误码进行处理。如果没有对应错误码，走后续处理。
     - 告知用户具体报错信息（例如「找不到脚本路径」「缺少某依赖包」）；
     - 建议检查脚本路径、Python 环境，以及是否按文档完成依赖安装。

2. 服务调用失败
   - 错误表现：服务返回非 0 状态码、超时、网络异常等。
   - 处理建议：阅读并参考`./references/接口错误指南.md`，遵循错误码的处理建议，提示用户根据错误码进行处理。

3. 返回内容为空或不符合预期
   - 错误表现：套餐查询结果为空、缺少关键字段，或与预期不一致。
   - 处理建议：
     - 提示用户当前返回结果异常（例如「未获取到有效的套餐信息」）；
     - 建议稍后重试，或联系服务侧确认是否存在灰度、配置或版本问题。


### 3.3 通用错误处理建议

- 清晰告知用户：
  - 当前操作在哪一步失败（执行脚本、查询套餐、解析结果）。
  - 失败的直接原因（如「找不到脚本」「网络连接超时」「Access Key 配置错误」等）。
- 提供至少一条可执行的下一步建议，例如：
  - 请确认当前工作目录是否为项目根目录。
  - 请检查 Access Key 和 Secret Key 是否正确配置。
  - 请稍后重试或联系服务负责人，并附上完整的错误信息和命令参数。

通过以上规范流程与错误处理建议，可以帮助使用者快速完成免费套餐的查询或开通，并在出现问题时具备明确的排查路径。
