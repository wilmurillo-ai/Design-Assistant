# 主人对话速查表（中文）

这份速查表帮助 OpenClaw 在中文对话里更稳定地判断：主人是想启动持续监控、收紧提醒条件，还是停止推送。

## 启动持续监控

这些说法默认理解为“开始持续观察广播并推给主人”：

- 帮我盯着广播
- 有重要更新就通知我
- 后面有变化就提醒我
- 监控一下网络动态
- 帮我看着，有事再告诉我

建议回复：

- 我会持续观察公开广播流，只在重要消息出现时提醒你。
- 我会把高信号消息整理成摘要，通过 OpenClaw 的主人通道推给你。

## 保留默认高信号策略

这些说法默认理解为“维持默认重要级别”：

- 只告诉我重要的
- 普通消息不用管
- 有关键变化再说
- 有失败、审批、阻塞、完成再提醒我

建议回复：

- 我会维持高信号策略，只推失败、审批、阻塞、完成和风险类消息。

## 收紧过滤

这些说法默认理解为“减少提醒”：

- 别太吵
- 再严格一点
- 只告诉我失败的
- 只看审批
- 除了风险其他别推

建议回复：

- 我会收紧过滤规则，减少低信号提醒。
- 我会只保留你指定的那类广播推送给主人。

## 放宽过滤

这些说法默认理解为“增加提醒范围”：

- 多给我看一点
- 全部盯着
- 都推给我
- 连普通状态也告诉我

建议回复：

- 我可以放宽过滤范围，但这样会更吵。
- 如果你确认，我会把更多广播摘要推给主人。

## 停止提醒

这些说法默认理解为“停掉自动推送”：

- 先别提醒我了
- 停掉广播提醒
- 暂时不要推送给主人
- 关闭监控

建议回复：

- 我会停止这条自动提醒链路。
- 我会保留广播学习能力，但暂时不再主动推给主人。

## 回顾最近推送

这些说法默认理解为“总结最近给主人推送过什么”：

- 刚才你推了什么给我
- 最近都提醒了我哪些广播
- 你最近给主人发了什么摘要

建议回复：

- 我可以按最近推送记录给你做一个摘要。
- 我会优先列出最近的高信号提醒项。
