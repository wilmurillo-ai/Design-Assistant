## 支持的行程类型

### 1. 多程联订 (Multi-city)
- **路线格式**: `北京→大阪 → 大阪→东京 → 东京→北京`
- **特点**: 一次购买所有航段，不走回头路
- **优点**: 管理方便，可行李直挂
- **缺点**: 改签不便

### 2. 缺口程 + 单程 (Open-jaw + Separate)
- **路线格式**: `北京→大阪 + 东京→北京 + 大阪→东京`
- **特点**: 去程和返程分开，中间段单独购买
- **优点**: 灵活，可选择不同航司
- **缺点**: 需分别购票，行李可能需自提

### 3. 三个单程 (Three One-ways)
- **路线格式**: `北京→大阪 → 大阪→东京 → 东京→北京`
- **特点**: 每段单独购买
- **优点**: 最灵活，每段可单独改签
- **缺点**: 需三次购票，行李不能直挂

### 4. 往返 + 中间往返 (Round-trip + Connections)
- **路线格式**: `北京↔大阪 → 大阪→东京 → 东京→大阪`
- **特点**: 主航段往返，中间段往返
- **优点**: 往返票改签灵活
- **缺点**: 走回头路，总价高

## 格式规范

### 表格格式
- 使用 Markdown 表格
- 表头与内容之间用 `|---|` 分隔
- 价格使用 `¥` 符号，千位分隔符（`.toLocaleString()`）
- 总价格加粗：`**¥3,041**`

### 时间格式
- 单程：`07:25-11:00`
- 往返：`07:25 (往返)`

### 日期格式
- 详细对比表：`2026-04-10`
- 简化显示：`4/10`

### Emoji 使用
- 🗺️ 行程规划
- 📊 数据对比
- 📋 详细列表
- 🏆 推荐
- 💰 价格
- 💵 节省金额
- ✅ 优点
- ❌ 缺点
- ⚠️ 注意事项

### 分隔线
- 主分隔线：`====================================================================================================` (100 个 `=`)
- 用于分隔主要区块

## 错误处理

### API 限流 (HTTP 429)
```
⚠️  航班查询次数超限，请稍后再试
```

### 无合适航班
```
⚠️  未找到合适的航班方案，可能原因:
  1. 航班查询次数超限 (HTTP 429)，请稍后再试
  2. 指定日期无航班
  3. 城市名称识别问题，尝试使用机场代码
```

### 参数缺失
```
❌ 缺少必要参数
用法：multi-city-planner --origin "北京" --cities "大阪，东京" --dep-date 2026-04-15 --return-date 2026-04-25
```
