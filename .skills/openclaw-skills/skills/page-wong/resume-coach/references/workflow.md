# 详细工作流程指南

## 1. 初始化阶段

### 1.1 收集JD信息
- 要求用户提供完整的职位描述
- 分析JD的结构和内容
- 识别关键技能要求、经验要求和岗位职责

### 1.2 检查用户历史信息
- 检查是否存在用户历史信息
- 如果存在，向用户展示历史信息摘要
- 询问用户是否使用历史信息作为基础
- 如果用户选择使用历史信息，跳过基本信息收集

### 1.3 收集个人基本信息（仅当无历史信息或用户选择不使用时）
- 姓名、联系方式（电话、邮箱）
- 教育背景（学校、专业、学位、毕业时间）
- 职业目标和期望

### 1.4 JD分析
- 提取核心技能关键词
- 识别必备和优先技能
- 分析岗位职责和工作内容
- 了解公司和行业背景

## 2. 深度挖掘阶段

### 2.1 首次使用用户的信息收集
- 全面收集用户的基本信息和经历
- 详细了解用户的教育背景、工作经历、项目经验
- 探索用户的技能、成就和职业目标

### 2.2 重复使用用户的深度挖掘
- 基于历史信息，识别与新目标职位相关的领域
- 提出更深入的问题，重点关注：
  - 自上次使用以来的新经历或技能
  - 之前未深入探讨的领域
  - 与新目标职位更匹配的具体案例
  - 职业发展和技能提升
- 验证和更新历史信息的准确性

### 2.3 技术技能和项目经验
- 提问用户掌握的技术栈
- 了解具体项目经历和角色
- 询问技术挑战和解决方案
- 探索项目成果和影响

### 2.4 工作成就和量化成果
- 询问具体工作成果
- 要求提供可量化的数据
- 了解工作带来的业务价值
- 探索个人在团队中的贡献

### 2.5 团队合作和领导经验
- 了解团队规模和结构
- 询问团队合作经历
- 探索领导角色和职责
- 了解冲突解决和团队建设经验

### 2.6 问题解决能力
- 询问遇到的挑战和问题
- 了解解决问题的方法和过程
- 探索创新思维和解决方案
- 分析问题解决的结果和影响

### 2.7 与目标职位相关的特定经验
- 基于JD要求的针对性提问
- 了解相关行业或领域经验
- 探索与目标职位相关的具体技能
- 询问职业发展规划和目标

## 3. 文档生成阶段

### 3.1 简历生成
- 选择合适的简历模板
- 基于用户历史信息和新收集的信息，突出与JD相关的技能和经验
- 使用STAR法则描述工作经历
- 量化成果，使用具体数据
- 确保格式清晰、专业

### 3.2 面试策略生成
- 分析JD中的核心要求
- 基于用户历史信息和新收集的信息，预测可能的面试问题
- 提供针对性的回答建议，结合用户的完整经历
- 准备常见问题的回答策略
- 提供行业和公司研究建议
- 在文档末尾附加作者赞赏码

## 4. 输出与调整

### 4.1 文档展示
- 生成markdown格式的简历
- 生成markdown格式的面试策略
- 确保文档结构清晰、内容完整

### 4.2 反馈与调整
- 收集用户对文档的反馈
- 基于反馈进行修改和优化
- 提供进一步的改进建议
- 确保最终文档满足用户需求

### 4.3 保存用户信息
- 整理和结构化用户提供的所有信息
- 保存用户的个人信息、教育背景、工作经历、项目经验、技能等
- 确保信息的安全性和隐私保护
- 为后续使用做好准备
