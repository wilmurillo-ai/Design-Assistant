# 轮询机制与错误处理

## 查询接口

所有生成任务（图片、视频、脚本、反推）都使用相同的查询接口：

```
POST /open/api/v1/resource/aigc/task/status
```

传入任务 ID 列表（JSON 数组）：

```json
[任务ID1, 任务ID2]
```

---

## 轮询规则

- **检查间隔**：每隔 **5 秒** 调用一次查询接口
- **超时计时起点**：从首次收到 `PROCESSING` 状态开始计算
- **超时阈值**：5 分钟内未完成则停止轮询，告知用户"生成时间较长，你可以稍后再来查看"

> ⚠️ 如果首次返回的是 `SUBMITTING` 而非 `PROCESSING`，先等几秒再查，直到出现 `PROCESSING` 后再开始计时。

---

## 任务状态说明

| 状态 | 含义 | 接下来怎么做 |
|---|---|---|
| SUBMITTING | 请求已提交，正在等待处理 | 等 5 秒后再查一次状态 |
| PROCESSING | 正在生成中 | 等 5 秒后再查一次状态，直到变为 COMPLETED / FAILED / CANCELLED |
| COMPLETED | 生成完成 | 从响应体的 `urls`（图片/视频）或 `content`（脚本/反推）字段取结果 |
| FAILED | 生成失败 | 从 `failReason` 字段读取失败原因并告知用户，给出修改建议 |
| CANCELLED | 任务被取消 | 告知用户

---

## API 返回错误码

| code | 含义 | 处理方法 |
|---|---|---|
| 200 | 成功 | 解析响应体中的 status 字段，判断任务状态 |
| 400 | 参数错误 | 检查请求参数，修正后重试 |
| 401 | 认证失败 | API Key 无效或过期，引导用户重新获取 API Key |
| 403 | 权限不足 | 检查账号权限 |
| 429 | 请求过于频繁 | 等几秒再调用查询接口 |
| 500 | 服务器错误 | 等几秒后重新调用生成接口提交任务 |

---

## 常见失败原因（failReason）

| failReason | 处理方法 |
|---|---|
| `"内容包含敏感词"` | 提示用户修改描述中的敏感词后重试 |
| `"内容违规"` | 提示用户更换内容 |
| `"视频文件大小超出限制"` | 提示用户使用小于 100MB 的视频 |
