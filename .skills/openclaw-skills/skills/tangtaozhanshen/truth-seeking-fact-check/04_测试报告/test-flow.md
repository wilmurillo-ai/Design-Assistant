# Truth (求真) v1.3.0 测试流程

## 测试阶段划分

1. **测试小组功能测试**（产品部）
   - 功能完整性测试：所有声明功能是否都实现
   - 安装可用性测试：能否正常加载运行
   - 基础用例测试：典型场景能否输出正确结果
   - 资源占用测试：CPU/内存是否符合2核2G要求

2. **独立审计部门安全审计**（AuditAgent）
   - 合规性检查：是否符合OpenClaw合规要求
   - 链接有效性检查：所有URL是否正确可用
   - 代码安全检查：是否存在安全问题
   - 描述一致性检查：描述和代码是否一致
   - 资源占用检查：CPU/内存是否不超过70%

3. **发布市场**
   - 只有测试+审计全部通过才能发布
   - 任一环节不通过打回修改

---

## 测试准入条件

- [ ] 所有代码开发完成
- [ ] 所有官方问题已修复
- [ ] 所有名称修改完成
- [ ] 删除了开源地址
- [ ] 目录结构符合规范

---

## 测试完成标准

- [ ] 所有测试用例执行完成
- [ ] 阻塞性问题为0
- [ ] 所有问题已修复并验证
- [ ] 测试报告签字确认

---

## 审计完成标准

- [ ] 合规检查100%通过
- [ ] 所有链接可用
- [ ] 无安全问题
- [ ] 描述和代码一致
- [ ] 资源占用符合要求
- [ ] 审计报告签字确认
