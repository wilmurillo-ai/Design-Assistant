## 二、核心功能设计

### 1. 6层校验体系设计

#### 层级权重分配
| 层级 | 校验维度 | 权重 | 计算方式 |
|------|----------|------|----------|
| 1 | 来源匹配度 | 25% | 关键词搜索匹配度 0-10分 |
| 2 | 逻辑一致性 | 15% | 内部矛盾检测 0-10分 |
| 3 | 常识符合度 | 15% | 基本常识匹配 0-10分 |
| 4 | 信息明确性 | 10% | 模糊词统计 0-10分 |
| 5 | 交叉验证度 | 20% | 多数据源一致性比对 0-10分 |
| 6 | 存证区块链验证 | 15% | 区块链存证验证 0-10分（未验证得5分） |

#### 总分计算公式
```
总分 = Σ(得分 × 权重) / Σ权重
最终得分 = 总分 - (问题句子数 × 0.5)
结果保留1位小数，范围 0-10
```

### 2. 区块链验证设计

#### 设计思路
- 不运行全节点，资源友好（适配2核2G）
- 通过公开区块链浏览器API验证存证哈希
- 仅当文本中包含 `#blockchain:[URL]` 格式时触发验证
- 默认不启用，不占用额外资源

#### 输入格式示例
```
"这则消息已上链存证 #blockchain:https://etherscan.io/tx/0x..."
```

#### 验证流程
```
用户输入 → 提取区块链URL → 调用浏览器API获取存证哈希 → 比对内容哈希 → 返回验证结果 → 计入评分
```

#### 支持的区块链浏览器
- Ethereum: Etherscan
- BSC: BscScan
- 其他: 预留接口，可扩展

#### 输出结构
```python
class BlockchainVerification:
    has_verification: bool        # 是否进行了验证
    blockchain_url: str           # 验证URL
    content_hash: str             # 内容哈希
    on_chain_hash: str            # 链上哈希
    verification_result: bool    # 是否匹配
    explanation: str             # 验证结果说明
    score: float                 # 得分（匹配10分，不匹配0分，未验证5分）
```

## 四、性能设计（适配2核2G）

1. **内存控制**：
   - 单篇最大文本长度：5000字
   - 批量最大总长度：20000字
   - 不缓存全文本，处理完立即释放
   - 区块链验证仅按需调用，验证完成立即释放

2. **并发控制**：
   - 批量核查最大并发 = CPU核心数（2），不超资源
   - 网络请求串行化访问同一数据源，避免被封禁

3. **超时控制**：
   - 单文本核查超时：3秒
   - 网页核查超时：10秒
   - 区块链验证超时：5秒
   - 批量核查超时：每段增加1秒，最大30秒

4. **懒加载**：
   - 数据源客户端按需初始化，不用不创建
   - 区块链模块仅在需要时导入

## 五、合规设计

1. **敏感内容拦截**：
   - 使用开源敏感词库预处理，违规内容直接拒绝核查
   - 不返回违规内容任何结果，直接返回"包含违规内容，已拒绝处理"

2. **隐私保护**：
   - 所有处理在内存完成，不写入磁盘存储用户核查内容
   - 请求处理完成后立即清空内存数据

3. **免责声明**：
   - 输出结果开头必须包含免责声明："本结果仅供参考，不构成任何专业意见，不保证完全准确"
   - 明确告知：结果是AI辅助判断，最终真实性由用户自行判断

4. **区块链合规**：
   - 仅验证公开链上数据哈希，不上链任何用户内容
   - 不触碰违规区块链应用，仅做事实存证验证

## 六、测试计划

### 单元测试用例设计
1. 正常文本核查：完全真实文本 → 高分输出
2. 幻觉文本核查：包含明显错误事实 → 低分输出 + 问题标注
3. 敏感文本测试：包含违规内容 → 直接拒绝
4. 批量核查测试：5段混合文本 → 输出正确批量结果
5. 网页URL测试：正常公开网页 → 成功抓取并核查
6. **新增**：区块链验证测试：有存证且匹配 → 10分；不匹配 → 0分

### 性能测试用例
1. 最大文本测试：5000字文本 → 3秒内完成
2. 最大批量测试：20段文本 → 30秒内完成
3. 内存占用测试：峰值内存 ≤ 100MB
4. 区块链验证测试：单次验证 ≤ 5秒完成
