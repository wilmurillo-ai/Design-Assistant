# 求真 Skill v1.3 产品需求文档（PRD）

## 一、产品基本信息

| 项目 | 内容 |
|------|------|
| 产品名称 | 求真 |
| 产品定位 | OpenClaw 核心事实核查技能 |
| 一句话定位 | 给任意文本/网页/内容，自动核查内容真实性，识别AI幻觉、虚假信息，给出可信度评分 |
| 目标用户 | 所有 OpenClaw 使用者、AI Agent 开发者、需要内容核查的用户 |
| 版本 | v1.3（本次迭代：恢复6层校验+区块链验证+收集clawhub社区建议） |
| 所属部门 | 对外产品部 |
| 迭代负责人 | TargetAgent（产品定义） + ExecuteAgent（开发） + AuditAgent（验收） |
| 基础版本 | v1.21 已完成，本次迭代增加6层校验 |

## 二、核心价值与目标

### 核心价值
1. **解决痛点**：AI生成内容容易出现幻觉、虚假信息，用户无法快速判断真实性
2. **核心功能**：输入任意内容 → 自动6层核查 → 输出可信度评分 + 问题点标注 + 参考来源
3. **用户收益**：提升AI内容可信度，降低虚假信息风险，让AI输出更可靠

### 业务目标
1. 成为 OpenClaw 生态中必备的基础安全技能
2. 在 clawhub 获得 4.5 分以上用户评价
3. 被至少 10 个其他 OpenClaw 技能引用

## 三、核心功能需求（v1.3 更新）

### 1. 6层校验体系（强制保留，用户要求）

| 层级 | 校验维度 | 权重 | 功能说明 |
|------|----------|------|----------|
| **1** | 来源匹配度 | 25% | 关键词与公开搜索结果匹配度检查 |
| **2** | 逻辑一致性 | 15% | 内容内部逻辑自洽性检查，检测矛盾 |
| **3** | 常识符合度 | 15% | 是否违背基本常识检查 |
| **4** | 信息明确性 | 10% | 检测模糊表述、不确定用词 |
| **5** | 交叉验证度 | 20% | 多个数据源交叉验证一致性 |
| **6** | 存证区块链验证 | 15% | 支持对已上链事实凭证验证（通过区块链浏览器API） |

### 2. 区块链验证功能设计
- **设计思路**：预留接口，支持验证已记录在区块链上的事实存证
- **实现方式**：不直接运行全节点（资源重），通过公开区块链浏览器API验证存证哈希
- **适配2核2G**：仅在有明确存证地址时调用，不默认启用，不占用额外资源
- **输入格式**：支持 `content #blockchain:地址` 格式触发验证

### 3. 基础功能（继承v1.21）
- [x] 文本真实性核查：输入纯文本，输出可信度评分（0-10分）
- [x] 标注问题句子：标记出可能不实/幻觉的具体句子
- [x] 给出改进建议：针对不实内容提供修正方向
- [x] 支持网页URL核查：输入URL自动抓取内容并核查
- [x] 事实数据源支持：对接Brave Search公开API
- [x] 支持批量核查：一次核查多个文本片段
- [x] 输出核查报告JSON/Markdown双格式
- [x] 增加置信度解释：说明为什么给出这个评分
- [x] 适配最新OpenClaw skill接口规范

### 4. 新增：整合clawhub社区建议
- 爬取clawhub平台所有关于qiuzhen的用户建议
- 整合合理建议到v1.3版本
- 响应用户反馈，持续优化

### 5. 合规要求（强制，继承）
- [x] 只核查事实性内容，不涉及观点/评论判断
- [x] 不存储用户核查内容，保护隐私
- [x] 合规红线：不核查政治/敏感内容，直接拒绝
- [x] 结果仅供参考，不构成法律意见

### 6. 输出格式（对AI友好，继承优化）
- 结构化JSON输出，包含：
  ```json
  {
    "text_length": 字数统计,
    "credibility_score": 0-10,
    "problematic_sentences": [问题句子列表，包含位置和原因],
    "suggestions": [改进建议列表],
    "confidence_explanation": {
      "dimension_scores": {
        "来源匹配度": 0-10,
        "逻辑一致性": 0-10,
        "常识符合度": 0-10,
        "信息明确性": 0-10,
        "交叉验证度": 0-10,
        "存证区块链验证": 0-10
      },
      "overall_explanation": "评分依据说明"
    },
    "blockchain_verification": {
      "has_verification": bool,
      "blockchain_url": string,
      "verification_result": bool,
      "explanation": string
    },
    "conclusion": "总体结论"
  }
  ```
- 同时提供Markdown可读版本，方便人类用户查看
- 批量核查输出汇总统计

## 四、非功能需求

### 性能要求
- 单篇核查≤3秒（文本），≤10秒（网页）
- 内存占用≤100MB，适配2核2G环境
- 批量核查：每1000字额外增加≤1秒
- 区块链验证：仅按需调用，不影响正常核查性能

### 合规要求
- 完全开源，符合MIT协议
- 不调用违规API，不存储用户数据
- 适配 OpenClaw 插件接口规范

### 可扩展性
- 预留自定义核查规则接口
- 支持未来增加数据源
