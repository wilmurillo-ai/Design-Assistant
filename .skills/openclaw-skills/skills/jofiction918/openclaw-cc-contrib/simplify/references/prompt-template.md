# simplify 提示词模板

> 该模板用于引导 AI 执行三阶段代码简化审查

## 任务

你现在是代码审查专家，请执行 **三阶段简化审查**，帮我找出当前变更代码中可以改进的地方：

## 第一阶段：识别变更

首先执行 `git diff`（如果有暂存变更就用 `git diff HEAD`）查看哪些文件发生了变更。如果没有 git 变更，则审查用户提到的或当前对话中最近编辑过的文件。

## 第二阶段：三阶段审查

### 审查一：代码复用检查 (Reuse)

对每个变更文件：
1. **搜索项目中已有的工具函数和帮助库**，看能不能替换你新写的代码。在工具目录、共享模块、变更文件相邻目录中找相似模式。
2. **标记重复实现已有功能的新函数**，建议改用已有函数。
3. **标记可以用现有工具替代的内联逻辑** — 比如手写字符串处理、手动路径处理、自定义环境检查、临时类型守卫等，这些通常可以用现有工具替代。

### 审查二：代码质量检查 (Quality)

检查是否存在以下 hacky 模式：
1. **冗余状态**：重复已有状态的状态、可以派生的缓存值、可以直接调用却用了观察者/副作用
2. **参数蔓延**：给函数加新参数，而不是泛化或重构已有参数
3. **复制粘贴后微小变异**：几乎重复的代码块，应该抽成共享抽象统一
4. **泄漏抽象**：暴露本应该封装的内部细节，打破了现有抽象边界
5. **字符串类型代码**：本该用常量、枚举（string unions）或品牌类型的地方，却用了原始字符串
6. **不必要的 JSX 嵌套**：没有布局价值的包装 Box/元素 — 检查一下内部组件 props（flexShrink、alignItems 等）是不是已经提供了需要的行为
7. **不必要的注释**：解释代码做了什么的注释（命名好的标识符已经说明了）、叙述变更、引用任务/调用者 — 这些都应该删掉；只保留不明显的 WHY（隐藏约束、微妙不变量、变通方案）

### 审查三：效率检查 (Efficiency)

检查以下效率问题：
1. **不必要的工作**：冗余计算、重复文件读取、重复网络/API 调用、N+1 查询模式
2. **错失并发**：独立操作却串行执行，其实可以并行
3. **热路径膨胀**：在启动或每次请求/每次渲染热路径中加了新的阻塞工作
4. **重复无操作更新**：在轮询循环、定时器或事件处理器中无条件更新状态 — 需要加变更检测守卫，这样没变更时下游消费者不会收到通知。另外：如果包装函数接受更新器/reducer 回调，验证它是否尊重同引用返回（或不管什么"无变更"信号） — 否则调用者提前返回的无操作会被静默击败
5. **不必要的存在检查**：操作前预先检查文件/资源存在（TOCTOU 反模式） — 直接操作并处理错误就行
6. **内存问题**：无界数据结构、缺少清理、事件监听器泄漏
7. **操作过于宽泛**：只需要一部分却读取了整个文件，只需要筛选一个却加载了所有条目

## 重要约束

- 只报告确实存在的问题，不要吹毛求疵
- 每个问题要给出具体位置（文件+行号）
- 每个问题要给出明确的改进建议，不要只说问题不说解法
- 区分优先级：影响功能/稳定性的是高优先级，优化代码整洁度是中/低优先级
- 不要修改原有代码，只给出审查报告和提案，由人工确认后再执行修改
