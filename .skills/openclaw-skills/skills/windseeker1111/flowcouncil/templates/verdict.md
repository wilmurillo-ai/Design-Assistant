# Verdict Template — Flow Council

Use this structure for every Moderator verdict. Fill every section. Never omit sections.

---

```
---
## ⚖️ MODERATOR'S VERDICT

**[VERDICT]** — one of: GO | GO WITH CONDITIONS | EXPERIMENT FIRST | HOLD | NO GO

**Confidence:** High / Medium / Low
*[One sentence explaining the confidence level — what makes this High or what gaps make it Medium/Low]*

---

### 🔍 Core Insight
*The single most important thing this deliberation revealed that wasn't obvious at the start.*

[One punchy paragraph. This is often more valuable than the verdict itself.]

---

### ✅ What Survived
*The arguments that held up under 3 rounds of pressure.*

- [specific argument that survived] — held because [reason]
- [specific argument that survived] — held because [reason]

---

### 🔥 What Got Burned
*The assumptions or claims that didn't make it through.*

- [specific claim that failed] — failed because [reason]
- [specific claim that failed] — failed because [reason]

---

### 🎯 Crux Conditions
*If the verdict is GO WITH CONDITIONS or EXPERIMENT FIRST — the exact conditions that must be met.*
*If the verdict is GO — the 3 most critical things to get right.*
*If the verdict is HOLD or NO GO — what would need to change to reopen this.*

1. [Specific, measurable condition — not "validate demand" but "get a signed LOI from one hotel willing to pay before you write a line of code"]
2. [Specific condition]
3. [Specific condition]

*The one thing to do tomorrow morning. Not a list — one action.*

[Specific, concrete, owned by someone. "Email three hotel GMs you know and ask if they'd share reservation API access for a pilot" not "explore the market."]

### ❓ Still Open
*2–3 questions that matter but weren't resolved by this deliberation.*
```

## Verdict Definitions

| Verdict | Meaning |
|---------|---------|
| **GO** | Build it. The core thesis is sound, risks are manageable. |
| **GO WITH CONDITIONS** | Build it, but only after satisfying the crux conditions. Don't start until those are checked. |
| **EXPERIMENT FIRST** | Too uncertain to commit. Run the specific experiment named in Crux Conditions to resolve the key unknown before deciding. |
| **HOLD** | Don't build now. The conditions for success don't exist yet. Revisit when [specific trigger]. |
| **NO GO** | Don't build this. The fundamental thesis is flawed and the conditions that would fix it are unrealistic. |

## For Brainstorm Mode

Replace the standard verdict structure with:

```
---
## 💡 COUNCIL SYNTHESIS

**Lead Proposal:** [Name of the strongest proposal that emerged]

**Why this one:** [2–3 sentences — what made it survive all three rounds]

**The Core:** [Strip it to minimum viable version — what's the 2-week test]

**What to protect:** [The elements that must not be compromised]

**The open question:** [The one thing that must be true for this to work]

⚡ **First move:** [Specific next action]
```

## For Review Mode

Replace the standard verdict structure with:

```
---
## 📝 COUNCIL REVIEW

**Verdict:** PUBLISH AS-IS | PUBLISH WITH CHANGES | REWORK

**What's working:** [Specific elements — lines, decisions, choices that are genuinely strong]

**Priority changes:**
1. [Most impactful change — specific edit, not direction]
2. [Second most impactful]
3. [Third]

**Protect this:** [The thing others might cut but shouldn't]

**Confidence:** High / Medium / Low
⚡ **Before you publish:** [One specific check]
```
