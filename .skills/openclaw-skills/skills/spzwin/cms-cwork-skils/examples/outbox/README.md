# outbox 示例

## 模块说明
获取当前用户的发件箱汇报列表，支持分页。

## 依赖脚本
`../../scripts/outbox/get-list.py`

## 对应接口
- `POST /work-report/report/record/outbox`

---

## 标准流程（含 3S1R 管理闭环）

### Step 1 — Suggest（建议）
**在拉取发件箱列表之前，先给出建议方案。**

- 建议合适的分页参数（当前脚本默认10条/页）
- 如需高级筛选，可使用完整 JSON 请求体
- 提示发件箱数据量与用户发送频率相关

```
建议：默认拉取最近 10 条发件箱记录。
如需复杂筛选，可直接传完整 JSON 请求体。
```

### Step 2 — Decide（确认/决策）
**涉及数据查询前，必须向用户确认参数。**

- 确认分页大小（默认10）
- 确认是否需要高级筛选 JSON

```
请确认查询参数：
□ 分页大小：10（默认）/ 其他：____
□ 高级筛选 JSON（可选）：____
```

### Step 3 — Execute（执行）
执行发件箱查询脚本。

### Step 4 — Log（留痕）
**查询结果必须完整记录。**

- 记录查询时间、分页参数、过滤条件、返回条数
- 格式：`[LOG] outbox | page:1 size:50 | result:N | ts:ISO8601`

```
[LOG] outbox | page:1 size:10 | result:10 | ts:2026-03-25T13:52:00+08:00
```

```json
{

    "list": [
      {

        "main": "测试开头 20260328-1912 脚本联调",
        "replyCount": 1,
        "fileCount": 1
      }
    ]
  }

```

---

## 注意事项
- 发件箱为只读查询
- 分页参数需严格校验，避免异常偏移
- 日志保留完整查询上下文，供审计使用
