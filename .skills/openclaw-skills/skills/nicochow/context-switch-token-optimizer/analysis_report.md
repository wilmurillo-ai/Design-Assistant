## 2. 相关度计算分析

### 计算逻辑

相似度计算公式：
```
相似度 = (关键词相似度 × 时间衰减因子)
```

**关键词相似度计算：**
1. 精确重叠：`exact_overlap = len(set1 & set2)`
2. 弱关联检测：2字子串 + 首字匹配
3. 相似度 = `(exact_overlap + 0.5*has_partial) / max_length`
4. 弱关联加成：如果存在弱关联且相似度 < 0.7，则加成 0.3，上限 0.65

**时间衰减：**
```python
time_decay = 0.95 ** (time_diff_seconds / 3600)
```

### 测试结果分析

#### 问题发现：时间衰减导致误判

**案例：**
- 主题1：`查看相关度计算列表相关`
- 主题2：`上下文管理相关`
- 关键词相似度：0.65（应该触发 `drift_compress`）
- 但实际动作：`switch`

**原因分析：**
1. 关键词相似度计算正确：0.65
2. 但时间衰减后，最终相似度可能低于 0.35
3. 导致被误判为 `switch` 而非 `drift_compress`

**时间衰减影响示例：**
- 时间差 1 小时：衰减 = 0.95^1 = 0.95，最终相似度 = 0.65 × 0.95 = 0.6175 ✅
- 时间差 3 小时：衰减 = 0.95^3 = 0.857，最终相似度 = 0.65 × 0.857 = 0.557 ✅
- 时间差 10 小时：衰减 = 0.95^10 = 0.599，最终相似度 = 0.65 × 0.599 = 0.389 ✅
- 时间差 20 小时：衰减 = 0.95^20 = 0.358，最终相似度 = 0.65 × 0.358 = 0.233 ❌ (误判为 switch)

## 4. 发现的问题

### 🔴 问题 1：时间衰减导致误判

**现象：**
- 关键词相似度 0.65 应该触发 `drift_compress`
- 但时间衰减后可能低于 0.35，被误判为 `switch`

**影响：**
- 可能导致不必要的上下文切换
- 错过渐变漂移场景的压缩机会

**建议：**
1. 在 `get_context_action()` 中使用**衰减前的相似度**进行判断
2. 或者调整时间衰减因子（当前 0.95 可能过于激进）
3. 或者为 `drift_compress` 设置更宽松的时间衰减阈值

### 🟡 问题 2：相似度计算中空关键词处理

**现象：**
- 当关键词为空时，相似度直接返回 0.0
- 导致 "一般对话" 类型主题无法正确计算相似度

**建议：**
- 考虑使用主题描述（topic）进行相似度计算作为备选

### 🟢 问题 3：日志输出不足

**现象：**
- 代码中有 `logger` 但实际运行时缺少详细日志
- 难以追踪相似度计算的中间过程

**建议：**
- 启用 `CONTEXT_SWITCH_LOG_LEVEL=DEBUG` 查看详细计算过程
- 在关键计算点添加日志输出
