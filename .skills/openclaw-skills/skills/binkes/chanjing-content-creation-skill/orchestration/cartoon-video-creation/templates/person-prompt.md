# 动漫短视频分镜·人物首帧图提示词框架

> **适用场景**：调用文生图模型（如豆包 Doubao-Seedance 系列等），为动漫风格短视频的每个分镜生成首帧关键图。  
> **核心目标**：跨分镜人物外貌高度一致 + 单镜画面表现力。  
> **设计原则**：本框架只定义字段结构与填写指南，不绑定特定画风或具体内容。通过调整各维度的描述精度和侧重点，可适配从写实美型到 Q 版简笔画的任意动漫风格。具体填写方式参见文末案例。

---

## 一、提示词拼装公式

每张图的完整提示词 = **风格锚 + 画质锚 + 角色身份卡 + 一致性强化 + 分镜变量（表情·姿态·手部·衣物动态·场景·镜头·光照·色调·特效·氛围）+ 负向约束**

拼装时保持以上顺序；**风格锚、画质锚、角色身份卡、一致性强化**在所有分镜中逐字保持不变，仅替换**分镜变量**。

---

### 风格适配矩阵

> 各维度的描述精度和侧重点随风格类别变化。下表为快速参照，帮助确定"每个维度应该写到什么程度"。

| 维度 | 写实 / 美型 | 简笔画 / Q 版 | 3D 渲染 | 水墨 / 插画 |
|------|-----------|-------------|---------|-----------|
| **头身比** | 7-9 头身，精确比例 | 2-4 头身，夸张比例 | 6-8 头身，接近写实 | 按画风，可夸张 |
| **面容** | 解剖级：脸型、骨骼感、肤色精确 | 特征化：标志性形状概括 | 解剖级 + 材质渲染 | 笔触 / 墨色概括 |
| **眼睛** | 虹膜 / 瞳孔 / 高光 / 睫毛 / 眼角 | 特征符号（黑点 / 线条 / 色块） | 虹膜 / 高光 / SSS 反射 | 墨点 / 留白 / 意象 |
| **发型** | 发丝层次 / 碎发 / 光泽质感 | 整体轮廓 + 色块形状 | 物理模拟发丝 + 光泽 | 飞白笔触 / 墨迹 |
| **服装** | 逐层描述，面料材质纹理 | 纯色块，极简剪裁 | 布料物理 + 纹理贴图 | 墨色浓淡 / 笔触褶皱 |
| **表情** | 眉 / 眼 / 嘴 / 瞳孔 / 面部肌肉 | 夸张变形 + 漫画符号（汗滴、怒筋） | 微表情 + 面部细节 | 意境暗示 / 少写具体 |
| **手部** | 精确手指描述 / 规避策略 | 简笔画手掌 / 规避策略 | 精确手指 + 接触物理 | 概括 / 隐于衣袂 |
| **衣物动态** | 风向 / 角度 / 每层飘动 | 静态贴合 / 极简飘动 | 布料物理模拟 | 墨迹飞白表现动感 |
| **场景** | 三层纵深 + 比例参照 | 简笔画色块 + 几何概括 | 精细 3D 环境 + 光照 | 大面积留白 + 意境 |
| **光照** | 主光 / 补光 / 轮廓光三光体系 | 平面光 + 色块阴影 | PBR 光照 + 体积光 | 墨色浓淡即光影 |
| **色调** | 冷暖对比 / 饱和度精细控制 | 高饱和纯色 / 简单对比 | 调色 LUT 感 / 色调映射 | 墨色为主 / 少量点朱 |
| **特效** | 灵气 / 粒子 / 光弧等写实特效 | 漫画符号（☆ ♪ 💢 汗滴） | 粒子系统 / 体积光雾 | 飞溅墨点 / 书法笔势 |
| **负向约束** | 防照片写实 / 穿模 / 手部畸变 | 防精细化 / 防写实 / 防成人体型 | 防塑料感 / 防穿模 | 防写实 / 防 3D |

> 根据项目风格在上表中定位所属列，按该列的精度要求填写各维度。案例一和案例二分别演示了"写实/美型"列和"简笔画/Q 版"列的填写方式。

**填写指南**（根据风格适配矩阵选择合适的画质描述）：

| 参数 | 选什么 | 常见选项 |
|------|-------|---------|
| 画幅 | 须与数字人画幅一致（见 `render-rules.md`） | `竖屏9:16` / `横屏16:9` / `方屏1:1` |
| 清晰度 | 根据模型能力和输出需求 | `8K超高清` / `4K高清` / `高清` |
| 渲染质感 | **必须匹配风格锚的美术类型** | 写实美型→`电影级光影，柔和阴影`；简笔画→`平面化极简光影，色块平涂`；3D→`PBR 渲染，次表面散射`；水墨→`墨色渲染，宣纸质感` |
| 品质增强 | 通用品质修饰词 | `极致画质` / `大师级作品` / `超高细节` / `线条清晰锐利` |

> **注意**：画质锚的渲染质感不可与风格锚矛盾（如简笔画风格不应写"电影级光影"，水墨风不应写"赛璐璐着色"）。

> 下表"好的示例"给出写实美型和 Q 版两种风格的示范，说明同一维度在不同风格下的精度差异。

| 维度 | 必须明确的要素 | 好的示例（写实/美型） | 好的示例（Q 版/简笔画） | 差的示例 |
|------|---------------|-------------------|---------------------|---------|
| 头身比 | 动漫比例类型 + 肢体比例感 | `美型8头身，四肢修长，躯干紧凑` | `Q版2.5头身，头大身小，四肢短粗` | 不写 |
| 面容 | 脸型、五官特征、肤色、面部质感 | `瓜子脸微带棱角，下颌线利落，肤色象牙白，鼻梁高挺` | `大圆脸，脸颊鼓鼓，无下颌线，肤色粉米色` | `好看的脸` |
| 眼睛 | 眼型、颜色、高光、标志性特征 | `狭长丹凤眼，虹膜深琥珀色，瞳孔中上方星芒高光，睫毛浓密` | `绿豆小眼，两颗黑色小椭圆点，无虹膜无高光` | `大眼睛` |
| 发型 | 发色、长度、造型结构、标志性轮廓 | `墨黑长发及腰，束高马尾，白玉发冠，鬓角碎发垂至下颌` | `纯黑短发，锯齿形刘海五齿朝下，后脑勺服帖如帽` | `黑色长头发` |
| 体型 | 身高感、体态特征、身体比例 | `身高感185cm，倒三角体态，肩宽腰窄，站姿挺拔` | `5岁幼童体型，圆肚子微凸，四肢短粗如藕节` | `高个子` |
| 服装 | 款式结构、配色、材质、关键剪裁 | `外层白色宽袖长袍，银线云纹暗纹，深青色交领中衣，银白宽腰封` | `大红色圆领短袖T恤纯色块，明黄短裤纯色块，白色运动鞋` | `穿古装` |
| 配饰/武器 | 名称、材质、颜色、位置、携带方式 | `腰左斜挂青鞘仙剑，剑长约90cm，银白剑穗20cm垂落` | `无任何配饰` | `戴些装饰` |
| 特征标记 | 独一无二的识别特征；无则写 `无` | `右眉尾约1cm淡银色旧疤` | `标志性粗眉毛（宽度为眼睛2倍），锯齿刘海` | 不写 |
| 角色色板 | 主色 / 辅色 / 点缀色的精确色名 | `主色：象牙白+浅青；辅色：深青+银白；点缀：琥珀金（瞳色）` | `主色：大红+明黄；辅色：粉米色+纯黑；点缀：白色+蓝色` | 不写 |

**各子维度编写指南**：

#### 5.1 表情·眉眼

眉毛和眼睛是动漫角色情绪传达的第一要素。

| 要素 | 要写什么 | 示例 |
|------|---------|------|
| 眉毛状态 | 上扬/平直/下压/皱拢/单挑、弧度、间距 | `双眉下压紧锁，眉间挤出竖纹` |
| 眼睛开合 | 全睁/半阖/微眯/圆睁/单眼闭 | `双眼圆睁，上眼睑抬至最高` |
| 瞳孔状态 | 收缩/扩张/聚焦/涣散/震颤 | `瞳孔微缩聚焦，虹膜纹理清晰` |
| 眼神方向 | 直视镜头/望向左上/俯视/斜视/目光追踪 | `目光向画面左侧45°远眺，视线略低于水平` |
| 高光与泪光 | 瞳孔高光亮度变化、泪膜反光、含泪未落 | `瞳孔高光明亮呈星芒形，眼底有薄泪光但未溢出` |
| 动漫眼部特效 | 眼神光变形、瞳孔异色、灵力纹路 | `虹膜边缘泛起淡青色灵力光纹` |

#### 5.2 表情·嘴部与脸色

嘴部形态和面部血色变化是情绪的第二锚点。

| 要素 | 要写什么 | 示例 |
|------|---------|------|
| 嘴型 | 闭合/微张/大张/抿紧/咬唇/露齿、嘴角上扬/下拉/角度 | `薄唇紧闭，嘴角微微下沉，下唇前端微翘` |
| 牙齿可见度 | 闭唇不见/微露上齿/咬牙切齿/犬齿外露 | `嘴唇微启露出一线上齿，不露犬齿` |
| 面部血色 | 苍白/红润/潮红/泛青/日晒红 | `颧骨处微泛薄红，鼻尖带一点血色` |
| 面部肌肉 | 放松/紧绷/抽搐/咬肌凸起/法令纹加深 | `咬肌微微凸起，面部肌肉紧绷` |
| 动漫表情符号 | 额角青筋、汗滴、脸红线条、阴影竖线 | `额角浮现一道青筋纹路（漫画式表达）` |

#### 5.3 姿态·身体重心

身体的力学关系决定人物是静态还是动态。

| 要素 | 要写什么 | 示例 |
|------|---------|------|
| 站/坐/蹲/跪/卧 | 基础体位 | `单膝跪地，右膝着地左腿弓步` |
| 重心分布 | 重心偏向哪只脚/哪个方向、是否失衡 | `重心压在右脚前掌，身体前倾15°呈冲刺蓄力` |
| 脊柱线条 | 挺直/微弓/S型/扭转角度 | `脊柱挺直微向后仰，胸腔打开` |
| 肩部姿态 | 耸肩/沉肩/一高一低/后收/前含 | `左肩下沉右肩微抬，身体自然侧倾` |
| 头颈角度 | 正视/侧转角度/仰头/低头/歪头 | `头向右侧转25°，下巴微收，颈部线条舒展` |
| 运动趋势 | 静止/即将起步/运动中/急停/腾空 | `身体处于跳跃最高点的滞空瞬间，双脚离地` |

#### 5.4 姿态·手部细节

手部是文生图的高难度区域，必须明确描述以降低畸变概率。

| 要素 | 要写什么 | 示例 |
|------|---------|------|
| 左手状态 | 握拳/张开/半握/持物/结印/自然下垂/插袖 | `左手五指自然张开，掌心朝上，托着一枚发光的青色灵珠` |
| 右手状态 | 同上 | `右手握剑柄，虎口贴合剑格，食指到小指依次收紧` |
| 手指数量 | 显式强调（降低模型出错率） | `五指分明，无多余手指` |
| 手臂姿态 | 手臂角度、与身体的相对位置 | `右臂前伸与肩同高，肘关节微屈约150°` |
| 手与道具关系 | 握/托/捏/按/推/拉的接触方式 | `右手四指握住剑柄中段，拇指压在食指侧面固定` |
| 规避策略 | 手部不好描述时的替代方案 | `双手负于身后不可见` / `双手插入宽袖中仅露指尖` / `双手被飘动的衣袂遮挡` |

#### 5.5 衣物与发丝动态

布料和头发的运动方向决定画面的动感与物理可信度。

| 要素 | 要写什么 | 示例 |
|------|---------|------|
| 风向 | 风从哪个方向来（须全画面统一） | `画面右侧吹来的山风` |
| 发丝响应 | 头发在风/运动中的飘动方向、弧度、散开程度 | `马尾整体向画面左后方飘起约45°，前额碎发贴向左侧颧骨，发梢散开呈扇形` |
| 上衣动态 | 衣袖/衣襟/领口随风/动作的变化 | `宽袖被风鼓起向左后方展开如翼，右侧衣襟随转身动作向外翻卷` |
| 下摆/裙摆 | 下半身衣物的飘动方向与形态 | `长袍下摆向身后飞扬，下摆边缘卷起可见深青色内衬` |
| 腰带/飘带 | 悬挂件的物理响应 | `银白腰带垂带向左侧飘起，剑穗随剑鞘微微摇晃` |
| 静态场景 | 无风时衣物的自然下垂状态 | `无风，衣袍自然垂落贴合身形，仅下摆因站在台阶边缘微微悬空` |

**一致性要点**：同一画面中所有飘动元素（头发、衣袖、飘带、落叶、烟雾）必须方向一致。

#### 5.6 场景环境

场景分为三层纵深描写，加上天气与环境细节。

| 层次 | 要写什么 | 示例 |
|------|---------|------|

#### 5.7 镜头与构图

#### 5.8 光照与阴影

| 要素 | 要写什么 | 示例 |
|------|---------|------|
| **主光（Key Light）** | 最强光源的方向、强度、色温 | `主光从画面右上方45°照射，强度中等，暖黄色调（黄昏夕阳）` |
| **补光（Fill Light）** | 阴影面的辅助光源 | `左下方环境反射光补充阴影面，强度约主光的30%，偏冷青色` |

| **环境光** | 场景整体的环境光照 | `云海反射的漫射光从下方照亮人物下巴和衣物底部` |

#### 5.9 画面色调

控制单帧的色彩倾向，让不同分镜之间有"调色"变化但角色色板不变。

| 要素 | 要写什么 | 示例 |
|------|---------|------|
| 整体色温 | 暖调/冷调/中性 | `整体偏冷青色调` |
| 主色调 | 画面中面积最大的环境色 | `画面以白云的冷白色和山石的青灰色为主` |
| 强调色 | 与主色调形成对比的小面积亮色（通常是人物的服装/武器色） | `人物白青色衣袍作为画面中的亮色焦点` |

## 六、多角色场景补充规则

当同一部短视频有 **2 个及以上** 固定角色时：

1. **每个角色独立建卡**：各写一份完整的"角色身份卡"（全部 9 个维度），用 `【角色A】`、`【角色B】` 前缀区分
2. **色彩区分原则**：不同角色的角色色板主色调必须有明显差异（如男主白青 vs 女主红金），避免模型混淆
3. **同框分镜变量拆分**：在分镜变量中每个子维度都要为每个角色单独描述，格式如下：
   ```
   【表情·眉眼·角色A】...
   【表情·眉眼·角色B】...
   【姿态·身体重心·角色A】...位于画面左侧...
   【姿态·身体重心·角色B】...位于画面右侧...
   ```
4. **身高与位置对比**：需注明角色间的相对位置（前后/左右）、身高差（如"角色A比角色B高半头"）、面朝方向
5. **交互关系描述**：若两角色有互动（对视/递物/对战），需明确双方的视线交汇点和身体朝向关系

| | `scene_title` | string | 分镜标题，格式：`视角·情境` |
| | `character_ids` | string[] | 本镜涉及的角色 ID 列表 |
| | `variables` | object | 分镜 11 维变量（expression_eyes/expression_mouth_skin/pose_body/pose_hands/cloth_hair_dynamics/scene_env/camera_composition/lighting_shadow/color_tone/effects_particles/mood） |
| | `assembled_prompt` | string | 拼装后的完整提示词 |

### `assembled_prompt` 拼装规则

下游脚本按以下顺序拼接生成 `assembled_prompt`：

```
assembled_prompt =
    global.art_style + "，"
  + global.quality_spec + "，"
  + character.summary + "，"
  + "【头身比】" + identity_card.proportion + "，"
  + "【面容】" + identity_card.face + "，"
  + "【眼睛】" + identity_card.eyes + "，"
  + "【发型】" + identity_card.hair + "，"
  + "【体型】" + identity_card.body + "，"
  + "【服装】" + identity_card.outfit + "，"
  + "【配饰/武器】" + identity_card.accessory + "，"
  + "【特征标记】" + identity_card.distinguishing_marks + "，"
  + "【角色色板】" + identity_card.color_palette + "，"
  + global.consistency_tags + "，"
  + variables.expression_eyes + "，"
  + variables.expression_mouth_skin + "，"
  + variables.pose_body + "，"
  + variables.pose_hands + "，"
  + variables.cloth_hair_dynamics + "，"
  + variables.scene_env + "，"
  + variables.camera_composition + "，"
  + variables.lighting_shadow + "，"
  + variables.color_tone + "，"
  + variables.effects_particles + "，"
  + variables.mood + "，"
  + global.negative_prompt
```

多角色同框时，依次拼接 `character_ids` 中每个角色的身份卡，并在分镜变量中用 `【角色A】`/`【角色B】` 前缀区分各自的表情和动作。

## 八、使用 Checklist

生成每张分镜首帧图前，逐项自检：

**角色一致性**：
- [ ] 风格锚、画质锚是否与前几张逐字相同
- [ ] 角色身份卡 9 个维度是否与前几张逐字相同
- [ ] 一致性强化标签是否已添加
- [ ] 角色色板中的颜色描述是否在所有帧中统一使用

**表情与动作**：
- [ ] 眉毛状态是否具体到弧度/间距/皱纹
- [ ] 眼睛是否描述了开合度+瞳孔状态+眼神方向+高光
- [ ] 嘴部是否描述了唇形+嘴角角度+牙齿可见度
- [ ] 脸色是否有血色变化描述（红润/苍白/潮红）
- [ ] 身体重心和运动趋势是否明确
- [ ] 手部是否明确了每只手的姿态，或使用了规避策略
- [ ] 手指数量是否强调为"五指"

**衣物与物理**：
- [ ] 是否明确了风向/运动方向
- [ ] 头发、衣袖、下摆、飘带的飘动方向是否全部一致
- [ ] 飘动幅度是否与风力/运动强度匹配

**场景与镜头**：
- [ ] 场景是否分前景/中景/远景三层描写
- [ ] 是否有 ≥2 个具体环境元素（非空泛词）
- [ ] 景别是否用了专业术语（特写/近景/中景/全景/远景）
- [ ] 是否明确了机位角度（仰/俯/平）和构图法则
- [ ] 景深是否明确了焦点位置和虚化区域
- [ ] 人物在画面中的位置是否精确到三分法位置

**光照与色彩**：
- [ ] 是否至少描述了主光方向+色温
- [ ] 面部明暗分布是否明确
- [ ] 画面整体色调是否明确（冷/暖/高饱和/低饱和）
- [ ] 画面色调变化是否仅改变环境色而非角色本身色板

**特效与氛围**：
- [ ] 特效类型和密度是否与场景情绪匹配
- [ ] 粒子/飘浮物方向是否与风向一致
- [ ] 安静场景是否克制使用特效（写"无特效"）
- [ ] 情绪氛围是否用了具体可感知的描述而非抽象形容词

**负向约束**：
- [ ] 负向约束是否已添加
- [ ] 多角色同框时是否添加了"无分身/无撞脸"约束
