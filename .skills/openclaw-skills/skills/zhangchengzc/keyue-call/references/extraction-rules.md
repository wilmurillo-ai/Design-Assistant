# 自然语言参数提取规则

把用户的自然语言请求拆成五类信息：
- `mobile`：被叫手机号；若用户未明确提供，则回退到 `config.json` 默认手机号
- `name`：被称呼的人名
- `owner_name`：外呼任务发起人的姓名或身份称呼
- `user_intent`：真正要传达的信息
- `schedule`：立即执行，还是延时/定时执行

## 提取原则

### 1. `name`
- 提取用户想称呼的人名
- 去掉“告诉他 / 提醒他 / 联系他 / 打电话给”这类壳子
- 若用户说“打电话给我”，优先使用默认称呼或当前配置，不要硬编姓名
- 若用户没给姓名，也不需要强行补一个

示例：
- “打电话给张三” → `name = "张三"`
- “给李四打个电话” → `name = "李四"`
- “打电话给我” → `name` 可留空或使用配置默认值

### 2. `owner_name`
- 用于表示是谁发起了这通电话，或机器人在话术里应代表谁来传话
- 只有当上下文明确存在这个“发起人身份”时才提取；不要臆造
- 可取真实姓名，也可取关系/身份称呼，例如“王五”“妈妈”“公司前台”
- 它不同于 `name`：`name` 是接电话时称呼谁，`owner_name` 是谁让这通电话打出去

示例：
- “以王五的名义打给李四，提醒他开会” → `owner_name = "王五"`
- “帮我打电话给妈妈，让她提醒小王取快递” → 若电话话术需要体现发起人，可提取 `owner_name = "妈妈"`
- 普通句子里没有明确发起人身份时 → `owner_name` 可留空

### 3. `user_intent`
- 只保留真正要传达的消息内容
- 去掉“打电话给…告诉他/提醒他/通知他”这些动作包装
- 尽量保留原意，不要擅自改写成完全不同的话
- 尽量简洁，避免把整句命令原样塞给平台

示例：
- “打电话给张三，告诉他下午三点开会” → `user_intent = "下午三点开会"`
- “给李四打电话，提醒他今天下班前提交周报” → `user_intent = "今天下班前提交周报"`
- “打电话给王强，叫他尽快回我微信” → `user_intent = "尽快回我微信"`

### 4. `mobile`
- 若用户明确给出手机号，则使用用户提供的号码
- 只有当用户明确表示“打电话给我 / 给我自己打电话 / 提醒我自己”时，才允许回退到 `config.json` 默认手机号
- 若场景是联系其他人（例如张三、李四、同事、客户、家人），即使给了姓名和内容，也必须要求用户提供手机号
- 若目标明显是第三人，但既没有手机号也没有现成映射，不要臆造号码，也不要偷偷使用默认号码

示例：
- “打电话给 15201515771” → `mobile = "15201515771"`
- “打电话给我” + 无号码 → 使用 `config.json` 默认手机号
- “打电话给张三” + 无号码 → 先向用户索要号码，不直接执行

### 5. `schedule`
- “现在 / 马上 / 立刻” → 立即外呼
- “2分钟后 / 五分钟后 / 下午三点” → 使用 `cron` 调度
- 调度文案里写明这是提醒电话，并带上提取后的关键信息

## 常见句式映射

### 立即外呼
用户：`打电话给张三，告诉他下午三点开会`
- `name = "张三"`
- `owner_name` 视上下文决定，若用户要求体现发起人身份则补齐
- `user_intent = "下午三点开会"`
- `schedule = immediate`
- 因为缺少手机号，先向用户追问号码，不直接执行

拿到号码后执行：
```bash
python3 scripts/create_realtime_call.py \
  --config config.json \
  --mobile "13333333333" \
  --name "张三" \
  --owner-name "王五" \
  --user-intent "下午三点开会"
```

### 指定号码立即外呼
用户：`打个电话给15201515771，告诉他下午三点开会`
- `mobile = "15201515771"`
- `user_intent = "下午三点开会"`
- `schedule = immediate`

执行：
```bash
python3 scripts/create_realtime_call.py \
  --config config.json \
  --mobile "15201515771" \
  --user-intent "下午三点开会"
```

### 延时提醒
用户：`2分钟后打电话给我，提醒我出门拿快递`
- `user_intent = "出门拿快递"`
- `schedule = delay(2 minutes)`

执行模式：
1. 创建 one-shot `cron`
2. 到点后运行本 skill
3. 如需体现是谁发起的提醒，同时传入 `--owner-name`
4. 使用 `--user-intent` 传入提醒内容

## 决策顺序

1. 先判断是不是延时/定时任务
2. 再抽取手机号
3. 再抽取 `name`
4. 若存在明确发起人身份，再抽取 `owner_name`
5. 最后抽取 `user_intent`
6. 构造命令时优先使用：`--mobile`、`--name`、`--owner-name`、`--user-intent`
7. 只有额外变量确实需要时，才补 `--dialog-var`

## 不要这样做

- 不要把整句“打电话给张三，告诉他下午三点开会”原样塞进 `user_intent`
- 不要在用户未提供号码时捏造手机号
- 不要在用户可见回复里暴露密钥、token、完整敏感配置
