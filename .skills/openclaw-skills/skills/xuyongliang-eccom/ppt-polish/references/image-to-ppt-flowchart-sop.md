# 图片转 PPT 流程图 SOP

目标：把一张包含流程图、架构图、步骤图的图片，转成 PowerPoint 中可编辑的流程图或结构图。

## 一、先判断图片类型

拿到图片后，先分 4 类处理：

1. 标准流程图
   - 大量方框、菱形、箭头、连接线
   - 适合直接重建

2. 组织/架构图
   - 树状、分层、分组明显
   - 适合先抽层级，再画图

3. 信息海报式流程图
   - 装饰元素多，流程只是其中一部分
   - 先保核心结构，再决定是否复刻视觉风格

4. 截图/低清图片/拍照图片
   - 先增强清晰度、裁切、矫正
   - 再做 OCR 和结构拆解

## 二、推荐工作流

### 路线 A：高保真人工复刻

适合：要交付客户、要高可编辑性、图比较复杂。

1. 观察整体布局
2. 拆节点
3. 拆连接关系
4. 拆视觉样式
5. 在 PPT 中重建
6. 收尾优化

### 路线 B：半自动重建

适合：批量图、内部使用、先快速出版本。

1. OCR 提取文字
2. 人工整理节点和连接关系
3. 输出中间结构（推荐 Markdown 列表 / Mermaid）
4. 按结构在 PPT 里快速重画

### 路线 C：自动化脚本辅助

适合：经常处理同类流程图，愿意投入一点搭建成本。

自动化通常不直接生成“完美 PPT”，而是先生成：
- 节点清单
- 流程关系表
- Mermaid / draw.io 草图
- PPT 绘制说明

## 三、实际操作 SOP

### 第 1 步：图片预处理

如果图片质量一般，先做：
- 裁掉无关边缘
- 拉正拍摄角度
- 提高清晰度和对比度
- 必要时放大 2x

### 第 2 步：提取文字

需要提取：
- 标题
- 每个节点里的主文案
- 分支条件
- 备注、图例、编号

### 第 3 步：识别结构

按“谁连到谁”整理成关系表：

```text
开始 -> 提交申请
提交申请 -> 审批
审批 -> 通过
审批 -> 驳回
```

如果有层级、泳道、分组，也要标出来。

### 第 4 步：在 PPT 中建骨架

推荐顺序：
1. 先定页面比例
2. 先放主节点
3. 再加次级节点
4. 再接连线
5. 最后补装饰元素

### 第 5 步：统一样式

统一：
- 形状大小
- 线条粗细
- 箭头样式
- 字体字号
- 圆角半径
- 填充色和边框色

### 第 6 步：检查可编辑性

检查是否满足：
- 每个框都能单独编辑
- 每段连线都能调整
- 文字不是一张图片贴上去
- 分支结构清楚

## 四、反推流程结构的办法

1. 先找主路径
2. 再找判断点
3. 再找分组
4. 最后找修饰元素

修饰元素应最后补，不要一开始被它们干扰。

## 五、常见难点

### 箭头交叉很多
- 先保主干
- 次级连接后补
- 必要时改为折线连接

### 图片很糊，OCR 不准
- 先放大、锐化、增强对比
- OCR 后人工校对
- 关键节点宁可人工录入

### 原图太花，像海报
- 先做结构复刻
- 第二版再补视觉风格

### 想一键转成 PPT
- 现实里很难一步到位
- 更合理的目标是：自动提取 60%-80% 结构 + 人工补齐
