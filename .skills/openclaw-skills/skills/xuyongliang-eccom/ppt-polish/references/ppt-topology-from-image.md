# 图片生成可编辑 PPT 拓扑图拆解稿

目标：将源图片中的拓扑/结构信息，转换为 PowerPoint 中可编辑的拓扑图页面，而不是将原图直接贴入 PPT。

## 一、交付目标

交付结果应满足：

1. 可编辑
   - 节点、区域、文字、连线都能在 PPT 中单独修改
2. 结构清晰
   - 能明确表达层次、分组和连接关系
3. 适合演示
   - 页面适合汇报、讲解、二次修改

## 二、推荐页面标题

如无特殊要求，推荐使用：
- 系统拓扑架构图

## 三、页面结构建议

建议采用三层结构：

### 1. 顶层：接入 / 外部层
用于承载：
- 用户终端
- 外部系统
- 第三方平台
- Internet / 公网入口

### 2. 中层：核心业务 / 平台层
用于承载：
- 网关
- 业务平台
- 核心服务
- 管理后台
- 中间件服务

### 3. 底层：数据 / 基础设施层
用于承载：
- 数据库
- 缓存
- 文件存储
- 消息队列
- 日志 / 监控

必要时可额外拆出运维管理区。

## 四、可编辑元素拆解规范

### A. 区域容器
建议使用：
- 圆角矩形
- 浅色或深色统一填充
- 细边框
- 左上角放区域标题

常见区域：
- 外部接入区
- 核心服务区
- 数据存储区
- 运维管理区

### B. 节点元素
建议映射：

- 平台/系统：圆角矩形
- 模块/服务：矩形或圆角矩形
- 数据库：数据库图标或圆柱形
- 云资源：云朵形状
- 网络设备：图标 + 文本框
- 终端：图标 + 文本框

### C. 连线规范
- 主流程 / 主链路：深色实线箭头
- 辅链路：虚线箭头
- 双向交互：双向箭头
- 弱依赖 / 说明性关系：细灰线

## 五、建议绘制顺序

1. 先搭区域框
2. 再放核心节点
3. 再补辅助节点
4. 先连主链路
5. 再补支线和说明
6. 最后统一样式

## 六、汇报型样式建议

- 页面比例：16:9
- 背景：白色 / 极浅灰 / 深色商务风三选一，整页保持统一
- 标题居上
- 中间为主拓扑图
- 下方可加图例

### 配色建议
- 区域框：浅蓝 / 浅灰 / 深色半透明
- 核心节点：蓝色系
- 数据节点：绿色、青色或深灰
- 外部系统：橙色或中性灰
- 运维节点：紫色或灰蓝色

### 文本建议
- 区域名：16-20 pt
- 节点名：12-16 pt
- 备注：10-12 pt

### 对齐建议
- 同层节点尽量等宽等高
- 使用“对齐”和“平均分布”
- 尽量减少线条交叉

## 七、关系表模板

```text
外部用户 -> 统一入口
管理端 -> 管理后台
第三方平台 -> API 网关
统一入口 -> 业务服务A
统一入口 -> 业务服务B
业务服务A -> 数据库
业务服务A -> Redis
业务服务B -> 文件存储
运维平台 -> 业务服务A
运维平台 -> 日志系统
```

## 八、交付原则

优先保证：
1. 信息结构正确
2. 演示可读性强
3. 元素可编辑

视觉高保真是加分项，但不应牺牲可编辑性与清晰度。
