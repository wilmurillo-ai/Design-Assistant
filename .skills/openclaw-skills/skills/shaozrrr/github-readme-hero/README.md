## Overview

Many README pages do not fail because they lack information.  
They fail because the first screen has no presence.

`GitHub-README Hero.skill` is built to solve exactly that problem.  
It does not overwrite everything with generic copy, and it does not replace the best original opening with templated slogans. Its job is more deliberate:

- lift the strongest existing line into a sharper hero opening
- arrange title, quote, badges, horizontal navigation, and subtitle into a cleaner first screen
- preserve the original introduction underneath instead of flattening it
- stay compatible with GitHub / ClawHub style Markdown rendering
- work especially well for Chinese-first and bilingual repositories that need stronger presentation without losing their voice

If the goal is not "rewrite my README" but "make the first screen feel more expensive, more deliberate, and more like a finished piece," this skill is built for that job.

<a id="best-for"></a>

## Best For

- GitHub README hero redesign
- skill landing page polish
- quote-led openings with badges and horizontal navigation
- Chinese-first or bilingual README front-page restructuring
- preserving strong original copy while improving the first screen

<a id="install"></a>

## Install

Place this directory in your Codex skills directory:

```bash
${CODEX_HOME:-$HOME/.codex}/skills/github-readme-hero
```

## Invocation

```text
Use $github-readme-hero to turn this GitHub README into a polished hero-style front page while preserving the strongest original introduction.
```

<a id="example"></a>

## Example

```text
Use this screenshot as the reference and turn my README top section into a centered hero layout. Keep my strongest original line at the top, add badges, horizontal navigation, and a short subtitle, but do not delete the introduction underneath.
```
