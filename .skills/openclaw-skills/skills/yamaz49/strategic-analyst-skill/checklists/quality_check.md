# 报告质量检查清单

## 结构完整性

### 必含章节
- [ ] 执行摘要（核心结论+关键数据+行动建议）
- [ ] 数据溯源与可信度说明 ⭐ 新增必须章节
- [ ] 行业结构分析（五力模型）
- [ ] 市场规模分析
- [ ] 竞争格局分析
- [ ] 趋势分析
- [ ] 关键成功因素
- [ ] 战略建议
- [ ] 附录（数据来源）

### 完整性检查
- [ ] 每个章节有明确标题
- [ ] 章节间逻辑连贯
- [ ] 结论与数据支撑一致
- [ ] 建议与问题对应

### 结构一致性检查（新增）
- [ ] 章节顺序符合标准（执行摘要→数据溯源→行业概览→五力→市场规模→竞争格局→价值链→趋势→CSF→战略启示→附录）
- [ ] 章节命名使用标准标题
- [ ] 表格格式统一（关键数据表包含提取方式列）
- [ ] 评级标准一致（可信度用★星级，优先级用P1/P2/P3）

---

## 数据质量

### 准确性
- [ ] 所有数据有明确来源
- [ ] 关键数据有交叉验证
- [ ] 数据口径一致
- [ ] 数据时效性说明

### 可信度标注
- [ ] 高可信度数据（政府/财报）已标注
- [ ] 中等可信度数据已标注来源
- [ ] 推算/预测数据已说明假设

### 数据透明度（新增）
- [ ] 直接引用数据 vs 推算数据已明确区分
- [ ] 推算数据的计算公式/逻辑已说明
- [ ] 数据时效性分布已说明（2024/2023/更早）
- [ ] 数据缺口和局限已坦诚说明
- [ ] 使用了标准可信度星级（★★★★★）
- [ ] 执行摘要中有数据时效性提示
- [ ] 多模态数据已标注提取方式（PDF提取/网页抓取/OCR识别/直接引用）

---

## 分析质量

### 逻辑性
- [ ] 分析框架应用正确
- [ ] 因果关系清晰
- [ ] 避免了幸存者偏差
- [ ] 考虑了反面证据

### 深度
- [ ] 不仅描述现象，解释原因
- [ ] 不仅罗列数据，形成洞察
- [ ] 区分事实判断和假设推断
- [ ] 指出分析局限

---

## 输出适配

### 用户类型适配
| 检查项 | CEO版 | 高管版 | 学生版 |
|-------|-------|-------|-------|
| 执行摘要在前 | 必须 | 推荐 | 可选 |
| 方法论解释 | 简要 | 适度 | 详细 |
| 案例数量 | 1-2 | 2-3 | 3-5 |
| 行动建议详细 | 必须 | 必须 | 可选 |
| 学习检查清单 | 否 | 否 | 必须 |

---

## 语言与格式

### 专业性
- [ ] 术语使用准确
- [ ] 避免口语化表达
- [ ] 单位使用规范
- [ ] 图表标题清晰

### 可读性
- [ ] 段落长度适中
- [ ] 关键结论前置
- [ ] 表格格式正确
- [ ] 层次分明（标题层级）

---

## 合规与边界

### 免责声明
- [ ] 包含数据局限说明
- [ ] 包含时效性说明
- [ ] 建议专业咨询提示

### 敏感信息
- [ ] 未假设用户敏感信息
- [ ] 未泄露可能涉密内容

---

## 最终检查

### 发布前必做
- [ ] 通读全文检查流畅度
- [ ] 核对所有数据准确性
- [ ] 确认格式正确（表格/列表）
- [ ] 检查错别字
- [ ] 确认用户问题已回答

### 交付确认
- [ ] 报告交付用户
- [ ] 询问是否需要调整
- [ ] 询问是否需要深度专题
- [ ] 询问是否需要持续监控
