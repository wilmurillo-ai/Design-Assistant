# Strategic Analyst HTML 报告风格规范

> 版本: 1.2
> 更新日期: 2026-03-29
> 适用范围: 所有 strategic-analyst 生成的 HTML 格式报告

---

## 一、设计原则

### 1.1 整体风格
- **极简专业**: 去除多余装饰，聚焦内容本身
- **金融研报**: 参考券商/咨询机构研究报告的排版风格
- **紧凑高效**: 减少留白，提升单屏信息密度
- **黑白灰为主**: 仅用黑、灰、红色（强调），避免花哨配色

### 1.2 核心约束
- 禁止直接用模型记忆填充数据，必须基于实时搜索
- 表格数量做减法，能用文字叙述的不用表格
- 段落文字优先于列表，列表优先于表格

## 二、配色方案

### 2.1 主色调
| 用途 | 色值 | 说明 |
|-----|------|-----|
| 主色 (黑色) | `#000000` | 标题、表头下划线、强调文字 |
| 次要色 (深灰) | `#333333` | 二级标题、副标题 |
| 正文色 | `#222222` | 正文文字 |
| 浅灰 | `#666666` | 辅助文字、页脚 |
| 边框灰 | `#cccccc` | 表格边框、分隔线 |

### 2.2 强调色
| 用途 | 色值 | 说明 |
|-----|------|-----|
| 红色 (强调) | `#cc0000` | 重点数据、趋势上升、警示 |
| 深红 | `#990000` | 趋势下降、风险 |

### 2.3 可信度星级颜色
| 星级 | 色值 |
|-----|------|
| ★★★★★ | `#000000` (黑) |
| ★★★★☆ | `#555555` (中灰) |
| ★★★☆☆ | `#888888` (浅灰) |
| ★★☆☆☆ | `#cc3333` (红) |
| ★☆☆☆☆ | `#990000` (深红) |

## 三、排版规范

### 3.1 页面基础
```css
body {
    font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
    font-size: 13px;
    line-height: 1.65;
    color: #222222;
    max-width: 900px;
    margin: 0 auto;
}
```

### 3.2 标题层级
| 级别 | 字号 | 字重 | 下边距 | 其他 |
|-----|------|-----|-------|------|
| H1 (章节) | 1.25em | 600 | 12px | 底部 1px 边框 (#cccccc) |
| H2 (小节) | 1.05em | 600 | 8px | 无 |
| H3 (内容项) | 0.95em | 600 | 6px | 无 |

### 3.3 间距规范
- 章节标题上边距: 28px
- 段落间距: 8px
- 表格外边距: 8px 上下
- 内容区 padding: 0 (无边距)

## 四、封面设计

### 4.1 布局
- **白底黑字**，左对齐
- 底部 2px 黑色实线分隔
- 简洁，无大 padding

### 4.2 元素
```

```

## 五、表格样式

### 5.1 核心特征
**金融研报风格**：
- 白底黑字，**无线框包围**
- 表头底部 **2px 黑色粗线**
- 单元格仅保留**横线分隔**（border-bottom: 1px solid #cccccc）
- 最后一行底部加 **1px 深灰线**收尾

### 5.2 具体规范
```css
table.data-table {
    border-collapse: collapse;
    font-size: 0.85em;
    border: none;
}

table.data-table th {
    background: #fff;
    color: #000;
    border: none;
    border-bottom: 2px solid #000;
    padding: 6px 8px;
    font-weight: 600;
}

table.data-table td {
    border: none;
    border-bottom: 1px solid #cccccc;
    padding: 5px 8px;
}

table.data-table tbody tr:last-child td {
    border-bottom: 1px solid #999;
}
```

### 5.3 使用原则
- **保留表格的场景**：数据来源溯源、市场份额排名、增长预测、PESTEL综合评估、机会/风险评估
- **避免表格的场景**：五力综合评估（改文字）、竞争者画像（改段落）、CSF分析（改文字+仅排序用表）

### 5.4 表格下载交互
HTML 报告中每个数据表格的右上角会在鼠标悬停时显示 **PNG / SVG** 下载按钮，采用纯前端零依赖实现（基于 SVG `<foreignObject>` + Canvas），不影响正常阅读与打印。

- 按钮默认隐藏，鼠标悬停在表格区域时显示
- 点击 **PNG** 可将表格渲染为图片并下载
- 点击 **SVG** 可下载矢量格式文件
- 该功能无需外部网络请求，HTML 报告自包含

## 六、内容结构规范

### 6.1 执行摘要
```markdown
[背景概述段落]

### 核心结论
1. [结论1]
2. [结论2]

### 关键数据
| 指标 | 数值 | 来源 | 可信度 | 来源链接 |
```

### 6.2 波特五力模型
**禁止**使用「五力综合评估表」，改用**叙述性文字总结**：
```markdown
### 1. 现有竞争者竞争强度
[分析段落]

**评分：** ★★★★☆（高强度）

### 综合评估
[对各力量的相互关系与行业吸引力进行总结性文字评述]

**总评：** [高/中/低]
```

### 6.3 竞争者画像
**禁止**使用「企业A/B/C」小表格，改用**段落式描述**：
```markdown
### 主要竞争者画像

**阿里云**以 26.1% 的份额稳居首位。其核心优势在于...近期动态显示...相对短板在于...

**华为云**在 2024 年实现收入...凭借...其主要挑战在于...
```

### 6.4 CSF分析
**重点用文字阐述**，仅在排序时使用简洁表格：
```markdown
### 行业CSF识别

**1. AI智算能力**
[重要性、表现、门槛的文字描述]

**2. 政企/行业解决方案**
...

### CSF优先级排序
| 排序 | CSF | 重要性 | 壁垒 | 紧迫性 |
```

### 6.5 能力建设建议
```markdown
### 能力建设建议

**短期（0-6个月）** [具体行动描述]

**中期（6-18个月）** [具体行动描述]

**长期（18个月+）** [具体行动描述]
```

## 七、Markdown转HTML处理规则

### 7.1 段落包裹逻辑
- 普通文本段落 → `<p>` 包裹
- 以块级标签开头的段落（`div`, `table`, `ul`, `ol`, `blockquote`, `hr`）→ 不包裹
- 以内联标签开头的段落（`strong`, `em`, `code`, `span`）→ 仍需 `<p>` 包裹

### 7.2 内联标签白名单
```python
inline_tags = {'strong', 'em', 'code', 'span', 'a', 'b', 'i', 'u', 's', 'del', 'ins', 'sub', 'sup', 'mark', 'small', 'label', 'img', 'br', 'wbr'}
```

### 7.3 特殊处理
- `---` → `<hr/>`
- `**文字**` → `<strong>文字</strong>`
- `*文字*` → `<em>文字</em>`
- `1. 内容` → `<ol class="numbered-list">` + `<li value="1">`
- `- 内容` → `<ul class="bullet-list">` + `<li>`

## 八、禁止事项

### 8.1 样式层面
- ❌ 禁止黑底白字表头
- ❌ 禁止斑马纹（交替行背景色）
- ❌ 禁止全包围表格边框
- ❌ 禁止大 padding/margin 留白
- ❌ 禁止彩色渐变背景

### 8.2 内容层面
- ❌ 禁止「建议行动清单」/「下一步行动」
- ❌ 禁止简单罗列的小表格（如五力评估表、能力差距分析表）
- ❌ 禁止用模型记忆填充数据（必须用实时搜索）

## 九、一致性检查清单

生成报告前检查：

- [ ] 封面为白底黑字左对齐，底部有 2px 黑线
- [ ] 表格为白底黑字，表头底部 2px 黑线，无全包围边框
- [ ] 所有 `.table-wrapper` 包含 PNG/SVG 下载按钮
- [ ] 表格内的 `[来源](URL)` 链接在 HTML 中被正确渲染为 `<a>` 标签
- [ ] 波特五力综合评估为文字叙述，非表格
- [ ] 竞争者画像为段落文字，非「维度-内容」小表格
- [ ] CSF分析以文字为主，表格仅用于排序
- [ ] 内联标签（strong/em/code）开头的段落被正确包裹在 `<p>` 中
- [ ] 无「建议行动清单」章节
