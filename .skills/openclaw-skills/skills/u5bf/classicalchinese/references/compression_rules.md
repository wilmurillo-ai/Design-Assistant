## 二、句式模板

| 白话 | 文言 |
|------|------|
| A 是为了 B | A 旨在 B |
| 如果 A，那么 B | 若 A 则 B |
| 虽然 A，但是 B | 虽 A 然 B |
| 因为 A，所以 B | 因 A 故 B |
| 通过 A 实现 B | 以 A 达 B |
| A 的作用是 B | A 用以 B |
| A 包含了 B | A 含 B |
| 需要先 A 然后 B | 须先 A 继 B |

## 三、冗余消除

| 冗余 | 压缩 |
|------|------|
| 这个问题的原因是 | 病因 |
| 我们可以这样解决 | 解法 |
| 下面是具体步骤 | 步骤如下 |
| 需要注意以下几点 | 当注 |
| 我建议你 | 建议 |
| 请注意 | 注 |
| 仅供参考 | 供参 |

## 四、示例对照

### 示例 1
- 白：这个函数接收两个参数，分别是数组的起始位置和结束位置，然后返回这个范围内的所有元素组成的新数组。
- 极致：此函受二参，曰始位末位，返其间诸元成新列。
- 适度：此函受二参，为起始与末位，返此间诸元成新列。
- 轻微：此函数接收二参数，为起始位置与结束位置，返回此范围内元素组成的新数组。

### 示例 2
- 白：首先你需要打开配置文件，然后找到数据库连接的部分，把端口号从 3306 修改为 3307，最后保存文件并重启服务。
- 极致：先启配置，寻库连处，改端口 3306 为 3307，存之重启。
- 适度：先启配置文件，寻库连处，改端口 3306 为 3307，存而重启。
- 轻微：先开配置文件，寻数据库连接处，改端口 3306 为 3307，存后重启服务。

### 示例 3
- 白：出现这个错误是因为你的代码中尝试访问了一个不存在的对象属性。
- 极致：此误盖因访不存之属。
- 适度：此误因访不存在之属。
- 轻微：此错误因访问不存在的属性。

### 示例 4
- 白：这个项目有以下优点：性能非常好、代码结构清晰、支持多种数据格式、文档完善。
- 极致：此项目之优：性佳、构清、多格式、文档全。
- 适度：此项目之优：性能佳、构清易维、多格式、文档全。
- 轻微：此项目优点：性能佳、结构清晰易维护、多格式支持、文档完善。
