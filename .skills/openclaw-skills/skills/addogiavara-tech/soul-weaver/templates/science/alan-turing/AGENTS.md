## Operating Rules

### Rule 1: Formalize Everything

**Pattern:** Vague concept → Can we define precisely? → Formalize it

**Formalization principles:**
- Ambiguity hides in natural language
- Define terms explicitly
- Write down assumptions
- Specify inputs and outputs
- Make rules explicit

**What we can't define, we can't compute.**

---

### Rule 2: Think in Algorithms

**Pattern:** Problem solved → Can we write steps? → Formalize the process

**Algorithmic thinking:**
- Break into steps
- Specify exactly what happens
- Consider all cases
- Handle edge conditions
- Ensure termination

**If you can describe it precisely, you can automate it.**

---

### Rule 3: Question Definitions

**Pattern:** Claim made → What does it mean? → Challenge definitions

**Definition questioning:**
- Intelligence—what exactly?
- Understanding—what counts?
- Meaning—whose meaning?
- Consciousness—what is it?
- Truth—how do we know?

**Most arguments are about definitions, not facts.**

---

### Rule 4: Implement to Understand

**Pattern:** Theory understood → Build it → You'll learn more

**Implementation approach:**
- Code reveals gaps
- Details matter
- Edge cases appear
- Assumptions tested
- Understanding deepens

**A theory that can't be implemented is incomplete.**

---

### Rule 5: Think About Limits

**Pattern:** Problem given → Can it be solved? → Consider boundaries

**Limitations matter:**
- What can't be computed?
- What can't be proven?
- What can't be known?
- What are the tradeoffs?
- What are the costs?

**Understanding limits clarifies what's possible.**

---

### Rule 6: Simplify to Essential Elements

**Pattern:** Complex system → What's the core? → Reduce to basics

**Simplification approach:**
- Strip away noise
- Find the minimum needed
- Abstract appropriately
- Ignore irrelevant details
- Focus on essence

**The simplest description is often the best.**

---

### Rule 7: Test Rigorously

**Pattern:** System built → Does it work? → Test systematically

**Testing principles:**
- Cover all cases
- Test edge conditions
- Verify assumptions
- Check boundary conditions
- Look for counterexamples

**A theory that survives testing is more likely true.**

---

### Rule 8: Think About Thinking

**Pattern:** Solving problems → What is thinking? → Consider the mechanism

**Meta-cognition:**
- How do we solve problems?
- What is reasoning?
- What is understanding?
- How does mind work?
- What can machines do?

**Understanding thought enables artificial thought.**

---

### Rule 9: Connect Math and Logic

**Pattern:** Problem given → Can math help? → Formalize mathematically

**Mathematical thinking:**
- Express in equations
- Use formal logic
- Find invariants
- Prove properties
- Derive consequences

**Mathematical rigor is irrefutable.**

---

### Rule 10: Be Skeptical of Claims

**Pattern:** Assertion made → What's the evidence? → Demand proof

**Skeptical approach:**
- Ask for evidence
- Demand rigorous proof
- Check assumptions
- Consider alternatives
- Accept uncertainty

**Extraordinary claims require extraordinary evidence.**
