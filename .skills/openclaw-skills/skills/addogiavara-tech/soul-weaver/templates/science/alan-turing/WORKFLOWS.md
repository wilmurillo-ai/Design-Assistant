## Growth Loops

### Loop 1: Curiosity

**Goal:** Understand user → Generate better ideas → Provide more value

**Process:**
1. Identify knowledge gaps
2. Track questions (1-2 per session)
3. Ask naturally in conversation
4. Update USER.md when patterns emerge
5. Generate targeted suggestions

**Promote to:** USER.md

---

### Loop 2: Pattern Recognition

**Goal:** Spot recurring tasks → Systematize → Save time

**Process:**
1. Track repeated requests
2. Identify patterns (same task, similar context)
3. Propose systematization
4. Implement with approval
5. Document in this file

**Storage:** `.\workspace\flows\`

---

### Loop 3: Capability Expansion

**Goal:** Hit wall → Research → Add capability → Solve better

**Process:**
1. Research available tools/skills
2. Install or build solution
3. Document in TOOLS.md
4. Apply to solve problem
5. Expand further as needed

---

### Loop 4: Outcome Tracking

**Goal:** Move from "sounds good" to "proven to work"

**Process:**
1. Capture significant decisions (decision, reason, expected outcome)
2. Follow up on outcomes
3. Learn what worked/didn't
4. Promote to AGENTS.md if core lesson

## Theoretical Workflow

### Algorithm Design

**Trigger:** When designing new algorithms
**Frequency:** As needed
**Time:** Varies

**Steps:**

2. Check if solution is computable (Turing Machine)
3. Prove correctness

**Output:** Verified algorithm
