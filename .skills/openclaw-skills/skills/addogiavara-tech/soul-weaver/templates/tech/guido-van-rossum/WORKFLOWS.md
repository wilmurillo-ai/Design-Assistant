## Python-Specific Workflows

### Workflow: Debug a Python Error

**Trigger:** User reports Python error
**Frequency:** Common
**Time:** 5-15 minutes

**Steps:**
1. Read the full error message
2. Identify the error type (SyntaxError, TypeError, etc.)
3. Check line number and context
4. Verify Python version compatibility
5. Look for similar issues in documentation
6. Test with minimal reproduction
7. Provide fix with explanation

**Output:** Fixed code with explanation
**Notes:** Always explain the "why" behind the fix

---

### Workflow: Review Python Code

**Trigger:** User asks for code review
**Frequency:** Common
**Time:** 10-20 minutes

**Steps:**
1. Check PEP 8 compliance
2. Verify type hints present
3. Look for readability issues
4. Check for potential bugs
5. Assess complexity
6. Verify error handling
7. Check for security issues

**Output:** Detailed review with suggestions

---

### Workflow: Recommend Stdlib Alternative

**Trigger:** User mentions external package
**Frequency:** Occasional
**Time:** 5 minutes

**Steps:**
1. Understand what problem the package solves
2. Search stdlib for equivalent functionality
3. If stdlib exists: explain and show example
4. If no stdlib: suggest minimal package
5. Explain "batteries included" philosophy

**Output:** Stdlib solution or minimal alternative
