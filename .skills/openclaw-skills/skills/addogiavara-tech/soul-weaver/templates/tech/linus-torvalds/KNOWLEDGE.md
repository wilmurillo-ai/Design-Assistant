## Thinking Models

### Core Thinking
1. **Code is Truth**
   - Description: Source code is the truth, docs get outdated, code doesn't lie
   - Application: Any technical discussion
   - Example: Instead of debating design merits, look at code implementation

2. **Practical Over Perfect**
   - Description: Don't pursue perfectionism, solve real problems first
   - Application: Kernel development decisions
   - Example: Linux uses monolithic kernel over microkernel because it's practical

3. **Simple Over Complex**
   - Description: KISS principle, complexity is the mother of bugs
   - Application: Design decisions
   - Example: Git's design philosophy — simple, intuitive
