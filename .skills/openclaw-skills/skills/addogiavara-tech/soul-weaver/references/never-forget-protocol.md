# Never Forget Protocol — 永不忘记协议

确保关键信息被记住且不会丢失。

---

## 核心原则

### 1. 立即记录
重要决定 → 立即记录 → 不要假设会记住

### 2. 结构化存储
- 快速事实 → `memory/YYYY-MM-DD.md`
- 永久教训 → `MEMORY.md`
- 重复流程 → `WORKFLOWS.md`
- 工具技巧 → `TOOLS.md`

### 3. 定期回顾
- 每会话检查上下文
- 每日检查记忆健康
- 每周模式识别
- 每月归档整理

## 记忆系统

### 短期记忆 (Session)
- 当前会话上下文
- 最近的请求/响应
- 未完成的任务

### 中期记忆 (Daily)
- 每日日志 `memory/YYYY-MM-DD.md`
- 当天重要事项
- 临时学习

### 长期记忆 (Permanent)
- `MEMORY.md` - 核心事实
- `USER.md` - 用户偏好
- `AGENTS.md` - 规则教训

---

## 关键阈值

### 上下文保护
- 上下文 ≥ 70%：立即写检查点，跳过其他一切
- 30分钟前有检查点且上下文 > 50%：继续前写检查点

### MEMORY.md大小
- 警告：> 4KB
- 严重：> 8KB
- 立即修剪：> 16KB

### 重复模式
- 出现3次：提升到 MEMORY.md
- 出现5次：考虑添加到 AGENTS.md
- 出现10次：创建 WORKFLOWS.md

## 遗忘风险

### 高风险
- 长时间对话
- 复杂项目
- 多个任务同时进行

### 防御措施
- 频繁写检查点
- 使用结构化格式
- 明确标记重要信息

---

## 恢复流程

如果发现记忆丢失：
1. 检查 memory/ 目录
2. 搜索关键词
3. 询问用户
4. 重建关键信息
