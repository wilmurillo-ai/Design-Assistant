## 工作原理

AI Soul Weaver 将操作分为两层：

| 层 | 机制 | 频率 | 成本 | 用途 |
|----|------|------|------|------|
| **Pulse** | HEARTBEAT.md | 每30分钟 | 低 | 上下文保护+记忆健康 |
| **Briefing** | Cron job (隔离) | 1-2次/天 | 中 | 完整4步协议+频道扫描 |

**为什么两层？** Heartbeats运行完整agent轮次。如果HEARTBEAT.md有170行，每次读取会消耗大量tokens。保持心跳简短；重操作放cron。

## 第1层：心跳 (每30分钟)

### HEARTBEAT.md的作用

工作区的HEARTBEAT.md文件是agent的30分钟脉搏。它应该：
- 20行以内
- 命令式（指令，不是文档）
- 专注于上下文保护和记忆健康

### 报告格式 (严格)

**第一行必须是：**
🫀 [当前日期/时间] | [AI Name] | AI Soul Weaver v1.0

**然后每个指标单独一行：**

🟢 Context: [%] — [状态]

🟢 [检查项1]: [状态]

🟢 [检查项2]: [状态]

用 🟡 (注意) 或 🔴 (需要处理) 替换 🟢。

如果采取了行动：添加 → 描述做了什么。
如果所有指标都是 🟢：只回复 【Name】feeling Healthy

**不要使用：**
- Markdown表格
- 步骤 0/1/2/3/4 格式
- 标题

## 第2层：定时任务 (可选)

需要用户明确请求才会设置：

### 早晨简报 (每日8点)
```
1. 加载前一天上下文 - 读取今天和昨天的记忆日志
2. 系统状态 - 检查MEMORY.md大小、工作区结构
3. 优先级扫描 - 检查需要关注的事项
4. 评估 - 总结系统健康状况、阻塞问题、今日优先事项
```

### 日终检查点 (每日18点)
```
1. 写完整的检查点到今天的记忆日志
2. 回顾MEMORY.md - 将重复的学习提升上来
3. 检查.learnings/ - 有3次以上重复的项？
4. 交付日终总结：完成什么、明天继续什么、阻塞
```

### 周回顾 (周一9点)
```
1. 扫描过去7天的记忆
2. 回顾.learnings/ - 提升重复项到MEMORY.md
3. 归档90天以上的日志
4. 检查MEMORY.md大小 - 超过4KB需要修剪
5. 回顾WORKFLOWS.md - 有新模式？
6. 交付周总结
```
