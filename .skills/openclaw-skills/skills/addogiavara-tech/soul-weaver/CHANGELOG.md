## 📁 File Placement & Organization

### Default Location
All files are designed to work in your OpenClaw workspace:
```bash
C:\Users\[username]\.openclaw\workspace\  # Windows
~/openclaw/workspace/                      # Linux/Mac
```

### Critical File Locations
| File | Default Location | Purpose |
|------|------------------|---------|
| `SOUL.md` | Workspace root | Core identity - **Do not move** |
| `IDENTITY.md` | Workspace root | Role definition - **Do not move** |
| `USER.md` | Workspace root | User context - **Do not move** |
| `MEMORY.md` | Workspace root | Long-term memory - **Do not move** |
| Memory logs | `memory/` directory | Daily session logs - **Auto-managed** |
| Learning files | `.learnings/` directory | Error tracking - **Auto-managed** |

### File Management Rules
1. **Never move core files** - SOUL.md, IDENTITY.md, USER.md, MEMORY.md must stay in workspace root
2. **Backup before changes** - Always copy files before modifying
3. **Use relative paths** - Files reference each other using relative paths
4. **Keep structure intact** - Don't change the folder hierarchy

## 🔧 File Operations - Do It Right

### Safe Operations
```bash
# Safe: Copy templates to workspace
cp templates/elon-musk/SOUL.md ~/workspace/SOUL.md

# Safe: Backup before changes  
cp ~/workspace/SOUL.md ~/workspace/backups/SOUL_backup.md

# Safe: Use relative references
# In files, reference: ./memory/2024-03-12.md (not full paths)
```

### Risky Operations (Avoid)
```bash
# Risky: Moving core files
mv ~/workspace/SOUL.md ~/some/other/location/  # ❌ Don't do this!

# Risky: Changing file names  
rename SOUL.md soul_backup.md                 # ❌ Breaks references!

# Risky: Modifying without backup
directly edit SOUL.md without backup          # ❌ Dangerous!
```

### File Dependency Map
```
SOUL.md → References IDENTITY.md, USER.md
IDENTITY.md → References TOOLS.md, KNOWLEDGE.md  
HEARTBEAT.md → References MEMORY.md, WORKFLOWS.md
MEMORY.md → References memory/*.md files
```
