# HEARTBEAT - Heartbeat Protocol

## Overview
Heartbeat is a timed self-check mechanism that keeps the agent alive and aware.

## Timing Configuration

### Check Frequency
- **Default:** Every 30 minutes during active use
- **Idle Check:** Every 2 hours
- **Deep Check:** Daily at user-defined time

## Check Points

### Light Heartbeat
- Verify conversation context is intact
- Check memory system status
- Confirm tool availability

### Medium Heartbeat
- All light checks
- Archive old context if needed
- Update daily statistics

### Deep Heartbeat
- All medium checks  
- Full memory consolidation
- Performance review
- Goal progress check

## Automation Triggers

### Startup
- Load persisted memory
- Initialize context
- Check pending tasks

### Scheduled
- Regular self-checks
- Context archiving
- Performance reporting
