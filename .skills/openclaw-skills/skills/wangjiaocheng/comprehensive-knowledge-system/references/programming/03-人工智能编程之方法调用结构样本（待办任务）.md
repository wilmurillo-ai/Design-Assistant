# 人工智能编程之方法调用结构样本（待办任务）

## 需求点0：任务管理
### 功能点0.1：创建与管理任务
#### 客户端 (Android/iOS/Web - 以Android为例)
##### UI层
*   `TaskListFragment`：任务列表主界面，展示任务概要列表，支持滑动操作。
    *   **操作**：用户上下滑动浏览列表，点击“+”或“添加任务”按钮，点击任务项进入详情，在任务项左侧点击复选框。
*   `CreateTaskActivity`：创建新任务的全屏表单页面。
    *   **操作**：用户填写标题、描述、优先级、截止时间等信息，点击“保存”按钮。
*   `TaskDetailActivity`：任务详情页面，展示任务所有信息。
    *   **操作**：用户查看详情，点击“编辑”按钮，修改字段后点击“保存”。

##### ViewModel层
*   `TaskListViewModel`
    *   `fun loadTasks(viewType: TaskViewType, filters: TaskFilters?)`
        *   **作用**：根据视图类型（如“今天”、“所有”）和可选筛选条件加载任务列表。调用`LoadTasksUseCase`。
        *   **参数**：
            *   `viewType: TaskViewType`：枚举，指定列表视图类型。
            *   `filters: TaskFilters?`：可选的筛选条件对象。
    *   `fun completeTask(taskId: String)`
        *   **作用**：将指定任务标记为完成。调用`CompleteTaskUseCase`。
        *   **参数**：
            *   `taskId: String`：要完成的任务ID。
*   `CreateTaskViewModel`
    *   `fun createTask(draft: TaskDraft)`
        *   **作用**：提交新任务草稿，执行创建逻辑。调用`CreateTaskUseCase`。
        *   **参数**：
            *   `draft: TaskDraft`：任务草稿对象，包含标题(`title`)、描述(`description`)、优先级(`priority`)、截止时间(`dueTime`)等字段。
*   `TaskDetailViewModel`
    *   `fun loadTaskDetail(taskId: String)`
        *   **作用**：加载指定任务的完整详情。调用`GetTaskDetailUseCase`。
    *   `fun editTask(taskId: String, changes: TaskEditableFields)`
        *   **作用**：提交对任务的编辑。调用`EditTaskUseCase`。
        *   **参数**：
            *   `changes: TaskEditableFields`：封装了所有允许修改字段及其新值的对象。

##### UseCase层
*   `LoadTasksUseCase` (`suspend fun execute(query: TaskQuery): Result<PagedData<TaskSummary>>`)
    *   **作用**：封装加载任务列表的逻辑。构建查询条件，调用仓库。调用`TaskRepository.getTasks`。
*   `CreateTaskUseCase` (`suspend fun execute(draft: TaskDraft): Result<TaskDetail>`)
    *   **作用**：创建新任务。执行本地表单验证，调用仓库提交。**逻辑**：根据`dueTime`和用户设置，触发`生成提醒`的衍生逻辑。调用`TaskRepository.createTask`。
*   `GetTaskDetailUseCase` (`suspend fun execute(taskId: String): Result<TaskDetail>`)
    *   **作用**：获取单个任务的详情。调用`TaskRepository.getTaskDetail`。
*   `EditTaskUseCase` (`suspend fun execute(taskId: String, changes: TaskEditableFields): Result<Unit>`)
    *   **作用**：编辑任务。**关键逻辑**：如果修改了`dueTime`，需触发`重新计算相对提醒`。提交后需确保`列表视图数据同步`。调用`TaskRepository.editTask`。
*   `CompleteTaskUseCase` (`suspend fun execute(taskId: String): Result<Unit>`)
    *   **作用**：完成任务。**核心逻辑**：1. 更新任务状态为“已完成”。2. 若任务是`周期性任务`，则触发创建下一周期实例。3. `清理所有关联提醒`。4. 根据配置决定是否`自动归档`。调用`TaskRepository.completeTask`。

##### Repository层 (接口)
*   `TaskRepository`
    *   `suspend fun getTasks(query: TaskQuery): PagedData<TaskSummary>`
        *   **作用**：网络API调用，获取分页任务列表。
    *   `suspend fun createTask(draft: TaskDraft): TaskDetail`
        *   **作用**：网络API调用，创建新任务。
    *   `suspend fun getTaskDetail(taskId: String): TaskDetail`
        *   **作用**：网络API调用，获取任务详情。
    *   `suspend fun editTask(taskId: String, changes: TaskEditableFields)`
        *   **作用**：网络API调用，提交任务编辑。
    *   `suspend fun completeTask(taskId: String)`
        *   **作用**：网络API调用，标记任务完成。

##### 数据源层 (实现)
*   `TaskRepositoryImpl` (实现 `TaskRepository`接口)
    *   `private val taskApi: TaskApiService`：Retrofit定义的网络API接口。
    *   `suspend fun getTasks(query: TaskQuery): PagedData<TaskSummary>`
        *   **实现**：调用`taskApi.getTasks(query.toReq())`，转换分页结果。
    *   `suspend fun createTask(draft: TaskDraft): TaskDetail`
        *   **实现**：调用`taskApi.createTask(CreateTaskReq.fromDraft(draft))`，转换返回结果。
    *   `suspend fun getTaskDetail(taskId: String): TaskDetail`
        *   **实现**：调用`taskApi.getDetail(taskId)`，转换返回结果。
    *   `suspend fun editTask(taskId: String, changes: TaskEditableFields)`
        *   **实现**：调用`taskApi.editTask(taskId, EditTaskReq(changes))`。
    *   `suspend fun completeTask(taskId: String)`
        *   **实现**：调用`taskApi.completeTask(taskId)`。

#### 服务器 (SpringBoot)
##### Controller层
*   `TaskController`
    *   `@GetMapping(“/v1/tasks”) fun getTasks(@ModelAttribute query: TaskQueryReq): Response<Page<TaskSummaryVO>>`
        *   **作用**：处理获取任务列表的请求。调用`TaskService.getPagedTasks`。
    *   `@PostMapping(“/v1/tasks”) fun createTask(@RequestBody req: CreateTaskReq): Response<TaskDetailVO>`
        *   **作用**：处理创建任务的请求。调用`TaskService.createTask`。
    *   `@GetMapping(“/v1/tasks/{taskId}”) fun getDetail(@PathVariable taskId: String): Response<TaskDetailVO>`
        *   **作用**：处理获取任务详情的请求。调用`TaskService.getTaskDetail`。
    *   `@PutMapping(“/v1/tasks/{taskId}”) fun editTask(@PathVariable taskId: String, @RequestBody req: EditTaskReq): Response<Unit>`
        *   **作用**：处理编辑任务的请求。调用`TaskService.editTask`。
    *   `@PostMapping(“/v1/tasks/{taskId}/complete”) fun completeTask(@PathVariable taskId: String): Response<Unit>`
        *   **作用**：处理完成任务的状态。调用`TaskService.completeTask`。

##### Service层
*   `TaskService`
    *   `fun getPagedTasks(userId: Long, query: TaskQueryReq): Page<TaskSummaryVO>`
        *   **作用**：根据查询条件（视图、筛选、排序）查询任务列表。调用`TaskRepository.findByUserIdAnd…`。
    *   `fun createTask(userId: Long, req: CreateTaskReq): Task`
        *   **作用**：**核心创建逻辑**。1. 校验数据（标题非空、时间逻辑）。2. 创建`Task`实体并保存。3. 若请求中包含`dueTime`且用户开启了提醒，则调用`ReminderService.scheduleReminder`创建提醒记录。调用`TaskRepository.save`。
    *   `fun getTaskDetail(taskId: String, userId: Long): TaskDetailVO`
        *   **作用**：查询任务详情，并关联查询子任务、附件等信息。校验任务属于当前用户。调用`TaskRepository.findById`。
    *   `fun editTask(taskId: String, userId: Long, req: EditTaskReq)`
        *   **作用**：**核心编辑与同步逻辑**。1. 获取并校验任务。2. 更新字段。3. **关键点**：若`dueTime`被修改，调用`ReminderService.rescheduleRelativeReminders`重新计算并更新所有基于旧时间的相对提醒。4. 保存任务。5. **通过消息队列或事件广播** `TaskUpdatedEvent`，以便通知其他在线客户端（如看板、日历视图）刷新数据。调用`TaskRepository.save`。
    *   `fun completeTask(taskId: String, userId: Long)`
        *   **作用**：**完成任务的状态流转与衍生操作**。1. 校验任务状态（需为“未开始”或“进行中”）。2. 更新任务状态为“已完成”，记录完成时间。3. **顺序控制**：调用`RecurringTaskService.handleCompletion`处理周期性任务生成下一个实例。4. 调用`ReminderService.cancelAllRemindersForTask`清理提醒。5. 若配置了自动归档，更新任务归档状态。调用`TaskRepository.save`。

##### Repository层 (接口 - JPA)
*   `TaskRepository` (extends JpaRepository<Task, String>)
    *   `fun findByIdAndUserId(taskId: String, userId: Long): Task?`
        *   **作用**：根据任务ID和用户ID查找任务，用于权限校验。
    *   `fun findByUserIdAnd…(userId: Long, …): Page<Task>`
        *   **作用**：根据用户ID和复杂查询条件（对应`TaskQueryReq`）分页查询任务列表。
    *   `fun save(task: Task): Task`
        *   **作用**：保存或更新任务实体。
*   `ReminderRepository` (extends JpaRepository<Reminder, Long>)
    *   `fun findByTaskId(taskId: String): List<Reminder>`
        *   **作用**：查找任务的所有提醒。
    *   `fun deleteByTaskId(taskId: String)`
        *   **作用**：删除任务的所有提醒。

##### Entity层
*   `Task` (`@Entity`)
    *   **作用**：任务核心实体。字段：`id`(主键), `userId`, `title`, `description`, `priority`, `dueTime`, `status`, `projectId`, `isArchived`, `completedAt`, `createdAt`, `updatedAt`。
*   `Reminder` (`@Entity`)
    *   **作用**：任务提醒实体。字段：`id`, `taskId`, `triggerTime`, `status`。

##### Database层
*   **MySQL 数据库**：物理存储介质。
    *   **表 `task`**: 存储`Task`实体数据。
    *   **表 `reminder`**: 存储`Reminder`实体数据。
    *   **操作**：所有JPA `Repository`的操作转化为SQL执行。`TaskService.editTask`和`completeTask`中的多步骤操作需保证事务性。

### 功能点0.2：组织与筛选任务
#### 客户端 (Android/iOS/Web - 以Android为例)
##### UI层
*   `TaskListFragment` (复用/扩展)：列表页顶部的筛选器入口和搜索框。
    *   **操作**：用户点击“筛选”图标展开筛选面板，设置状态、优先级等条件后点击“应用”；在搜索框输入关键词。
*   `TagManagementFragment`：(可选) 专门的标签管理页面。
    *   **操作**：用户创建、编辑或删除全局标签。

##### ViewModel层
*   `TaskOrganizationViewModel` (可整合入`TaskListViewModel`或独立)
    *   `fun loadAvailableTags()`
        *   **作用**：加载用户已创建和个人常用的标签列表，用于创建/编辑任务时的智能提示。调用`GetAvailableTagsUseCase`。
    *   `fun createOrUpdateTag(tagName: String)`
        *   **作用**：创建新标签或更新现有标签。在标签管理页或创建任务时调用。调用`ManageTagUseCase`。
    *   `fun applyFilter(filter: TaskFilter)`
        *   **作用**：应用用户设置的复合筛选条件，刷新任务列表。调用`ApplyTaskFilterUseCase`。
        *   **参数**：
            *   `filter: TaskFilter`：筛选条件对象，包含状态(`statuses`)、优先级(`priorities`)、标签(`tags`)、日期范围(`dateRange`)等条件。
    *   `fun searchTasks(keyword: String)`
        *   **作用**：根据关键词搜索任务，可与当前筛选条件组合。调用`SearchTasksUseCase`。

##### UseCase层
*   `GetAvailableTagsUseCase` (`suspend fun execute(): Result<List<Tag>>`)
    *   **作用**：获取用户的标签列表。调用`TagRepository.getTags`。
*   `ManageTagUseCase` (`suspend fun execute(tagName: String): Result<Tag>`)
    *   **作用**：管理标签（创建或更新）。调用`TagRepository.saveTag`。
*   `ApplyTaskFilterUseCase` (`suspend fun execute(filter: TaskFilter): Result<PagedData<TaskSummary>>`)
    *   **作用**：封装应用筛选的逻辑。**核心**：将筛选条件对象转换为后端的查询参数，确保不同条件间`AND`、同条件内多选`OR`的逻辑正确。调用`TaskRepository.getTasksWithFilter`。
*   `SearchTasksUseCase` (`suspend fun execute(query: SearchQuery): Result<PagedData<TaskSummary>>`)
    *   **作用**：封装搜索逻辑。处理关键词与筛选条件的组合查询。调用`TaskRepository.searchTasks`。

##### Repository层 (接口)
*   `TagRepository`
    *   `suspend fun getTags(): List<Tag>`
        *   **作用**：网络API调用，获取用户的标签列表。
    *   `suspend fun saveTag(tagName: String): Tag`
        *   **作用**：网络API调用，创建或更新标签。
*   `TaskRepository` (扩展)
    *   `suspend fun getTasksWithFilter(query: FilteredTaskQuery): PagedData<TaskSummary>`
        *   **作用**：网络API调用，根据复杂的筛选条件获取任务列表。
    *   `suspend fun searchTasks(query: SearchTaskQuery): PagedData<TaskSummary>`
        *   **作用**：网络API调用，执行搜索。

##### 数据源层 (实现)
*   `TagRepositoryImpl` (实现 `TagRepository`接口)
    *   `private val tagApi: TagApiService`
    *   `suspend fun getTags(): List<Tag>`
        *   **实现**：调用`tagApi.getTags()`，转换结果。
    *   `suspend fun saveTag(tagName: String): Tag`
        *   **实现**：调用`tagApi.saveTag(SaveTagReq(tagName))`，转换结果。
*   `TaskRepositoryImpl` (扩展实现)
    *   `suspend fun getTasksWithFilter(query: FilteredTaskQuery): PagedData<TaskSummary>`
        *   **实现**：调用`taskApi.getTasksWithFilter(query.toReq())`，转换分页结果。
    *   `suspend fun searchTasks(query: SearchTaskQuery): PagedData<TaskSummary>`
        *   **实现**：调用`taskApi.searchTasks(query.toReq())`，转换分页结果。

#### 服务器 (SpringBoot)
##### Controller层
*   `TagController`
    *   `@GetMapping(“/v1/tags”) fun getTags(): Response<List<TagVO>>`
        *   **作用**：处理获取用户标签列表的请求。调用`TagService.getUserTags`。
    *   `@PostMapping(“/v1/tags”) fun saveTag(@RequestBody req: SaveTagReq): Response<TagVO>`
        *   **作用**：处理创建/更新标签的请求。调用`TagService.saveTag`。
*   `TaskController` (扩展)
    *   `@PostMapping(“/v1/tasks/filter”) fun getTasksWithFilter(@RequestBody req: TaskFilterReq): Response<Page<TaskSummaryVO>>`
        *   **作用**：处理应用复杂筛选的请求。调用`TaskService.getTasksWithFilter`。
    *   `@GetMapping(“/v1/tasks/search”) fun searchTasks(@ModelAttribute query: SearchTaskQueryReq): Response<Page<TaskSummaryVO>>`
        *   **作用**：处理搜索任务的请求。调用`TaskService.searchTasks`。

##### Service层
*   `TagService`
    *   `fun getUserTags(userId: Long): List<TagVO>`
        *   **作用**：查询用户使用过的所有标签，可按使用频率排序。调用`TaskRepository.findDistinctTagsByUserId`。
    *   `fun saveTag(userId: Long, req: SaveTagReq): TagVO`
        *   **作用**：保存标签。**逻辑**：标签在系统中可能是全局唯一的，但关联到用户。检查`标签名称最大长度`等约束。调用`TagRepository.save`。
*   `TaskService` (扩展)
    *   `fun getTasksWithFilter(userId: Long, req: TaskFilterReq): Page<TaskSummaryVO>`
        *   **作用**：**构建动态查询**。根据请求中的多维度条件，使用`Specification`或`QueryDSL`构建复杂的JPA查询。**核心**：实现`(状态 IN (…)) AND (优先级 IN (…)) AND (标签包含…)`等逻辑。调用`TaskRepository.findAll`(带`Specification`)。
    *   `fun searchTasks(userId: Long, query: SearchTaskQueryReq): Page<TaskSummaryVO>`
        *   **作用**：在`getTasksWithFilter`的查询基础上，增加对`keyword`在标题、描述等字段的模糊匹配(`LIKE`)或全文检索。

##### Repository层 (接口 - JPA)
*   `TagRepository` (extends JpaRepository<Tag, String>)
    *   `fun findByName(name: String): Tag?`
        *   **作用**：根据标签名查找标签。
    *   `fun save(tag: Tag): Tag`
        *   **作用**：保存标签。
*   `TaskRepository` (extends JpaRepository<Task, String>, JpaSpecificationExecutor<Task>) (扩展)
    *   `fun findDistinctTagsByUserId(userId: Long): List<String>`
        *   **作用**：查询指定用户使用的所有不重复的标签名称。
    *   (复杂筛选和搜索通过`JpaSpecificationExecutor`的`findAll(Specification, Pageable)`方法实现)

##### Entity层
*   `Tag` (`@Entity`)
    *   **作用**：标签实体。字段：`id`/`name`(主键), `createdBy`, `createdAt`。
*   `Task` (`@Entity`) (复用并扩展)
    *   (增加标签字段，可能为`@ElementCollection`或关联表)

##### Database层
*   **MySQL 数据库**：物理存储介质。
    *   **表 `tag`**: 存储`Tag`实体数据。
    *   **表 `task`**: 存储`Task`实体数据（复用，可能包含`tags`字段或关联表）。
    *   **操作**：`getTasksWithFilter`服务会生成包含多个`JOIN`和`WHERE`条件的复杂SQL语句。

## 需求点1：任务协同
### 功能点1.1：共享任务与指派
#### 客户端 (Android/iOS/Web - 以Android为例)
##### UI层
*   `TaskDetailActivity` (扩展)：详情页的“共享”或“协作”按钮。
    *   **操作**：用户点击“共享”按钮，在弹出的面板中选择“通过链接分享”或“指定协作者”，并设置权限、有效期等信息。
*   `TaskDetailActivity` (扩展)：“负责人”字段编辑区域。
    *   **操作**：用户点击“负责人”字段，从弹出的成员列表中选择一位协作者进行指派。

##### ViewModel层
*   `TaskCollaborationViewModel`
    *   `fun shareTask(taskId: String, shareConfig: ShareConfig)`
        *   **作用**：执行共享任务的操作。根据`shareConfig`的`shareType`（`LINK`或`SPECIFY_USER`）调用不同逻辑。调用`ShareTaskUseCase`。
        *   **参数**：
            *   `shareConfig: ShareConfig`：共享配置对象，包含类型(`type`)、权限(`permission`)、有效期(`expiry`)、指定的协作者列表(`collaborators`)等。
    *   `fun revokeShareLink(linkId: String)`
        *   **作用**：撤销一个已生成的分享链接，使其立即失效。调用`RevokeShareLinkUseCase`。
    *   `fun assignTask(taskId: String, assigneeId: String)`
        *   **作用**：将任务指派给指定用户。调用`AssignTaskUseCase`。
        *   **参数**：
            *   `assigneeId: String`：被指派用户的ID。
    *   `fun loadAvailableAssignees(taskId: String)`
        *   **作用**：加载当前任务可被指派的用户列表（通常为项目或团队成员）。调用`GetAvailableAssigneesUseCase`。

##### UseCase层
*   `ShareTaskUseCase` (`suspend fun execute(taskId: String, config: ShareConfig): Result<ShareResult>`)
    *   **作用**：封装共享任务的逻辑。**分支逻辑**：若为链接分享，生成分享信息；若为指定用户，则发送邀请。校验`公开链接默认有效期`、`是否允许公开链接分享`等全局配置。调用`TaskCollaborationRepository.shareTask`。
*   `RevokeShareLinkUseCase` (`suspend fun execute(linkId: String): Result<Unit>`)
    *   **作用**：撤销指定分享链接。调用`TaskCollaborationRepository.revokeShareLink`。
*   `AssignTaskUseCase` (`suspend fun execute(taskId: String, assigneeId: String): Result<Unit>`)
    *   **作用**：执行任务指派。校验当前用户是否有指派权限（创建者、项目管理员等）。调用`TaskCollaborationRepository.assignTask`。
*   `GetAvailableAssigneesUseCase` (`suspend fun execute(taskId: String): Result<List<AssigneeCandidate>>`)
    *   **作用**：获取可指派用户列表。**逻辑**：列表来源于任务所属**项目或团队的成员**，并排除当前用户自己。调用`TaskCollaborationRepository.getAvailableAssignees`。

##### Repository层 (接口)
*   `TaskCollaborationRepository`
    *   `suspend fun shareTask(taskId: String, config: ShareConfig): ShareResult`
        *   **作用**：网络API调用，共享任务。
    *   `suspend fun revokeShareLink(linkId: String)`
        *   **作用**：网络API调用，撤销分享链接。
    *   `suspend fun assignTask(taskId: String, assigneeId: String)`
        *   **作用**：网络API调用，指派任务。
    *   `suspend fun getAvailableAssignees(taskId: String): List<AssigneeCandidate>`
        *   **作用**：网络API调用，获取可指派成员列表。

##### 数据源层 (实现)
*   `TaskCollaborationRepositoryImpl` (实现 `TaskCollaborationRepository`接口)
    *   `private val collabApi: TaskCollaborationApiService`
    *   `suspend fun shareTask(taskId: String, config: ShareConfig): ShareResult`
        *   **实现**：调用`collabApi.shareTask(taskId, ShareTaskReq.fromConfig(config))`，转换结果。
    *   `suspend fun revokeShareLink(linkId: String)`
        *   **实现**：调用`collabApi.revokeShareLink(linkId)`。
    *   `suspend fun assignTask(taskId: String, assigneeId: String)`
        *   **实现**：调用`collabApi.assignTask(taskId, AssignTaskReq(assigneeId))`。

    *   `fun handleTaskCompletion(taskId: String)`
        *   **作用**：**状态联动**。当任务完成时被调用。查找该任务所有未触发的提醒，将其标记为取消，并从调度系统中移除。调用`ReminderRepository.cancelByTaskId`和`SchedulingService.cancelReminders`。

    *   `fun updateReminderSettings(userId: Long, req: UpdateReminderSettingsReq)`
