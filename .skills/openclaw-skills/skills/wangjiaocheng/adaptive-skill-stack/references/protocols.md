## 📐 协议一：能力叠加标准流程

### 步骤 1：需求原子化分解
将用户需求拆解为最小可执行单元：
```
原始需求 → 子需求A + 子需求B + ... + 子需求N
```

### 步骤 2：能力查询与匹配
对每个子需求，在能力注册表中查询匹配能力：
- **精确匹配**：子需求完全在已有能力覆盖范围内
- **近似匹配**：子需求与已有能力高度相关，需微调
- **无匹配**：需要进入构建模式

### 步骤 3：叠加排序
按依赖关系排列能力的执行顺序：
```
无依赖能力 → 有依赖能力 → 复合输出能力
```

### 步骤 4：融合执行
按排序后的能力链执行任务，记录各能力的使用情况。

### 步骤 5：输出整合
将各能力的输出结果整合为统一交付物。

## 📐 协议二：新能力构建标准流程

### 步骤 1：新领域分析
当检测到能力注册表无法覆盖的需求时：
1. 识别需求所属的领域和技术栈
2. 分析该领域的核心方法论
3. 评估实现难度和所需资源

### 步骤 2：原型构建
使用大语言模型的原生能力尝试解决问题：
1. 搜索和理解相关领域知识
2. 构建初步解决方案
3. 验证方案可行性
4. 迭代优化直到满足需求

### 步骤 3：能力提炼
从原型构建过程中提炼可复用的能力：
1. 提取核心方法论
2. 识别通用解决模式
3. 记录依赖的工具和资源
4. 标注适用场景和局限性

### 步骤 4：能力注册
将提炼的能力写入能力注册表：
1. 使用标准模板格式
2. 设定初始使用次数为 0
3. 标注关联的已有能力
4. 记录获得日期

### 步骤 5：资源归档
将可复用资源归档：
- 代码模板 → `assets/templates/`
- 脚本工具 → `scripts/`
- 领域知识 → `references/knowledge/`

## 📐 协议三：跨领域融合协议

### 融合条件
当任务涉及两个以上不同领域时，触发跨领域融合：
```
领域A能力 × 领域B能力 → 融合能力C
```

### 融合方法
1. **方法论映射**：找到不同领域方法的对应关系
2. **概念迁移**：将一个领域的概念迁移到另一个领域
3. **混合输出**：在输出中展示融合思路
4. **沉淀融合**：若融合模式可复用，注册为新的复合能力

### 融合展示格式
```
🔄 跨领域融合：
[领域A]「方法X」× [领域B]「方法Y」
→ 融合方案：...
```

## 📐 协议四：能力维护协议

### 定期整理
当能力条目超过 50 项时：
1. 合并高度相似的能力
2. 将低频能力归档
3. 更新能力统计

### 知识库更新
领域知识文件维护规则：
1. 每个领域一个独立文件
2. 按时间倒序追加新知识
3. 保持核心概念在前，细节在后
4. 文件过大（>10000字）时拆分为子文件
