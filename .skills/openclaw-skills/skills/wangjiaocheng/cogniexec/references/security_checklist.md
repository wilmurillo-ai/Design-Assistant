# 安全审查清单

审查对象：`scripts/` 下全部 `.py` 脚本。

---

## 一、命令注入

- [ ] **subprocess shell=True**：列出所有使用处，确认用户输入经过验证/过滤
- [ ] **os.system / os.popen**：确认无残留调用
- [ ] **f-string / .format 拼接命令**：确认拼接的变量已做安全处理
- [ ] **过滤完整性**：shell=True 的过滤是否覆盖 `` ` ``、`$(`、`${`、`;`、`&&`、`||`

## 二、SQL 注入

- [ ] **表名/列名参数化**：确认 SQL 中的标识符使用白名单校验，不直接拼接用户输入
- [ ] **LIMIT / OFFSET 参数**：确认数值参数做正整数校验
- [ ] **子查询安全性**：确认子查询中嵌入的表名经过校验
- [ ] **用户 SQL 语句**：`cmd_query` / `cmd_exec` 直接执行用户 SQL 的场景需标注风险

## 三、路径遍历

- [ ] **tar 解压过滤**：`extractall` 是否使用 filter 或逐条校验，拦截 `../../`
- [ ] **ZIP 解压过滤**：解压前是否校验所有成员路径在目标目录内
- [ ] **GZIP / 单文件解压**：输出路径是否由 `output_dir` + `basename` 组合，不含用户拼接
- [ ] **用户指定输出路径**：`--o` 参数直接用于写入的场景需标注风险

## 四、不安全反序列化 / 动态代码执行

- [ ] **eval / exec**：全量搜索确认无调用
- [ ] **pickle**：确认无 `pickle.load()` 或 `pickle.loads()`
- [ ] **动态 import**：`__import__` / `importlib.import_module` 是否限制模块名白名单
- [ ] **YAML 加载**：是否使用 `yaml.safe_load()` 而非 `yaml.load()`

## 五、输入验证

- [ ] **CLI 参数类型**：数值参数（`--limit`、`--n`、`--timeout`）是否做 `int()` 转换及范围校验
- [ ] **枚举参数白名单**：method / format 等枚举参数是否用白名单而非直接传递
- [ ] **parse_args 逻辑**：自实现的参数解析器索引递增是否正确（`i += 1` vs `i += 2`）

## 六、依赖安全

- [ ] **第三方依赖**：是否基于纯标准库，或第三方依赖是否有版本锁定
- [ ] **pip install 版本锁定**：所有 `pip install` 调用是否指定版本范围

## 七、信息泄露

- [ ] **无硬编码凭据**：密码、token、API key 是否从环境变量或配置文件读取
- [ ] **无敏感日志输出**：密码、token 等是否在日志/标准输出中过滤
- [ ] **命令行参数可见性**：`--password` 等敏感参数通过命令行传入的风险

## 八、密码学

- [ ] **安全随机数**：密码/密钥生成是否使用 `secrets`（CSPRNG）而非 `random`
- [ ] **哈希用途**：MD5/SHA1 仅用于完整性校验而非安全场景
- [ ] **弱加密标注**：XOR 等教学演示功能是否明确标注不适用于生产

## 九、网络安全

- [ ] **SSL 验证默认开启**：HTTP 客户端默认 `verify_ssl=True`
- [ ] **SSL 禁用警告**：`verify_ssl=False` 时是否发出安全警告
- [ ] **HTTP 请求超时**：所有请求是否设置超时，防止无限挂起

## 十、文件操作安全

- [ ] **删除操作确认**：删除/清理操作是否有 `confirm()` 和 `dry-run` 模式
- [ ] **目录创建**：`os.makedirs` 是否使用 `exist_ok=True` 避免竞态
- [ ] **文件写入路径**：输出路径是否限制在预期目录内
