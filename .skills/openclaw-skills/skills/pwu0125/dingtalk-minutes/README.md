# Meeting Minutes

自动整理会议纪要，智能归档管理。

## 功能

- **自动扫描**：识别当日新增会议记录文件
- **智能合并**：根据时间、主题、参会人自动合并相关会议
- **信息提取**：自动提取参会人、核心议题、关键决策、待办事项
- **分类归档**：支持自定义分类，自动归档到对应目录

## 触发方式

输入 `/纪要` 即可触发。

## 首次使用

首次启动时会询问是否对会议档案进行分类：
- **不分类**：所有纪要放在 `meetings/` 根目录
- **使用默认分类**：项目会议 / 部门会议 / 外部会议 / 其他
- **自定义分类**：输入你自己的分类名称

## 支持的会议记录格式

默认扫描 `./会议记录/` 或 `~/Documents/会议记录/` 目录下的 `.txt` 文件：

```
MM-DD 会议类型_ 会议主题.txt
```

示例：
```
03-16 内部会议_ 产品规划讨论.txt
03-16 客户会议_ 合作方案沟通.txt
```

## 输出格式

生成标准 Markdown 格式的会议纪要：

```markdown
# 会议纪要：产品规划讨论

## 基本信息
- **日期**：2026-03-16
- **时间**：14:00 - 15:30
- **类型**：内部会议
- **参与人**：张三、李四、王五
...
```

## 归档规则

文件名格式：`YYYYMMDD_{分类缩写}_{主题}.md`

分类缩写：
- 项目会议 → `proj`
- 部门会议 → `dept`
- 外部会议 → `ext`
- 其他 → `misc`

归档路径：
- 有分类：`./memory/meetings/{分类}/`
- 无分类：`./memory/meetings/`

## 合并规则

当同时满足以下条件时，多个会议记录会被合并为同一个会议纪要：
1. 文件创建时间差 < 15分钟
2. 文件名相似度 > 70%
3. 具体人名（排除"发言人1/2/3"）重叠度 > 2/3

## 安装

```bash
clawhub install meeting-minutes
```

## 依赖

- OpenClaw 环境
- 钉钉或飞书导出的会议记录文本文件
