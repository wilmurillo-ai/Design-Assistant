# 音乐推荐器

一项根据用户描述推荐2000年后经典热门歌曲的技能，结合情绪、场景、活动等多种因素提供个性化音乐推荐。

## 为什么这项技能重要

许多用户在特定场景或情绪下需要合适的音乐，但可能不知道具体歌曲名称或如何找到合适的音乐。本技能通过分析用户描述，推荐真实存在且2000年后广受欢迎的热门歌曲，帮助用户快速找到适合的音乐。

## 核心功能

1. **混合推荐系统**：结合情绪、场景、活动、音乐特征等多种因素推荐歌曲
2. **经典热门歌曲**：专注于2000年之后各个时期广受欢迎的热门歌曲
3. **真实存在验证**：只推荐AI知道真实存在的歌曲，不虚构
4. **详细信息提供**：提供完整的歌曲信息，包括艺术家、专辑、年份等
5. **多样化选择**：当用户描述模糊时，提供多样化的歌曲选择

## 何时使用此技能

当遇到以下任何模式时，请主动使用此技能：

- 用户明确请求音乐推荐："推荐一些音乐"、"给我一些歌曲"
- 用户描述特定场景需要音乐："适合健身的音乐"、"学习时听的歌"
- 用户描述情绪状态需要音乐："心情不好想听歌"、"庆祝时听的音乐"
- 用户询问特定类型的音乐："有什么好听的摇滚乐"、"推荐经典流行歌曲"
- 用户需要背景音乐："工作时的背景音乐"、"派对的歌单"

## 如何使用此技能

### 步骤1：解析用户描述

仔细分析用户的描述，提取关键信息：
- **情绪**：快乐、悲伤、放松、兴奋、平静等
- **场景**：健身、学习、工作、驾驶、派对、睡眠等  
- **活动**：跑步、阅读、烹饪、清洁、旅行等
- **音乐特征**：节奏快慢、流派偏好、年代范围等
- **其他要求**：语言偏好、艺术家偏好等

### 步骤2：选择推荐策略

根据用户描述的清晰度选择策略：
- **描述明确**：直接根据描述匹配歌曲特征
- **描述模糊**：提供多样化推荐，涵盖多种可能的解释
- **描述简略**：如"推荐音乐"，可询问更多细节或提供通用热门推荐

### 步骤3：选择合适歌曲

遵循以下原则选择歌曲：

1. **真实性原则**：只推荐你知道真实存在的歌曲，不虚构任何歌曲信息
2. **热门度原则**：优先选择在各个时期排行榜上表现良好、广为人知的歌曲
3. **经典性原则**：注重歌曲的持久流行度和经典地位，而非短期热度
4. **匹配性原则**：确保歌曲特征与用户描述相匹配
5. **多样化原则**：推荐5-10首歌曲，涵盖不同流派、年代、艺术家
6. **年份限制原则**：只推荐2000年及之后发行的歌曲，确保音乐的时代感

### 步骤4：验证歌曲信息

对每首推荐歌曲验证以下信息：
- 歌曲名称准确
- 艺术家名称正确
- 专辑信息（如知道）
- 发行年份（如知道，必须为2000年及之后）
- 音乐流派
- 歌曲的流行度证据（如排行榜成绩、播放量等）

### 步骤5：格式化输出

使用以下格式输出推荐结果：

```
# 音乐推荐：基于您的描述"[用户描述摘要]"

根据您的需求，我为您推荐以下[数量]首经典热门歌曲：

## 1. [歌曲名称] - [艺术家]
- **专辑**：[专辑名称] ([发行年份])
- **流派**：[主要流派]
- **推荐理由**：[简要说明为什么这首歌适合您的描述，如：这首歌的[特征]非常适合[场景/情绪]]

## 2. [歌曲名称] - [艺术家]
- **专辑**：[专辑名称] ([发行年份])
- **流派**：[主要流派]
- **推荐理由**：[简要说明]

... [继续其他歌曲]

## 推荐总结
这些歌曲都是历史上广受欢迎的热门作品，适合您的[场景/情绪描述]。您可以从这些经典作品开始，探索更多相似音乐。
```

### 步骤6：提供后续建议

根据需要提供：
- 如何找到这些歌曲（流媒体平台建议）
- 类似艺术家或歌单建议
- 调整推荐的提示（如"如果您想要更多/不同的推荐，请告诉我更多细节"）

## 注意事项

1. **避免虚构**：绝不推荐你不知道的歌曲或虚构歌曲信息
2. **注重质量**：优先推荐真正经典、广受欢迎的热门歌曲
3. **保持客观**：基于歌曲的实际特征推荐，而非个人偏好
4. **考虑多样性**：在适当范围内涵盖不同流派、年代和文化背景
5. **验证信息**：确保提供的歌曲信息准确无误

## 成功标准

当出现以下情况时，您会知道此技能正在发挥作用：
- 用户对推荐表示满意或提供积极反馈
- 推荐的歌曲确实是2000年后的经典热门作品
- 歌曲特征与用户描述良好匹配
- 输出格式清晰、信息完整
- 用户根据推荐找到了合适的音乐
