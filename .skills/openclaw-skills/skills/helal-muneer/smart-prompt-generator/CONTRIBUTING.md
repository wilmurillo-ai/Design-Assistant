# Contributing to AI Prompt Generator

First off, thank you for considering contributing to the AI Prompt Generator skill! It's people like you that make this tool better for everyone.

## 🎯 Ways to Contribute

### 1. Report Bugs
If you find a bug, please create an issue with:
- Clear description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

### 2. Suggest Enhancements
Have an idea for improvement? Create an issue with:
- Clear description of the enhancement
- Use case and benefits
- Possible implementation approach

### 3. Submit Templates
Share your successful prompts:
1. Fork the repository
2. Add your template to appropriate file
3. Follow the existing format
4. Submit a pull request

### 4. Improve Documentation
Help make docs clearer:
- Fix typos
- Add examples
- Improve explanations
- Translate to other languages

### 5. Share Examples
Share how you use the skill:
- Real-world examples
- Success stories
- Tips and tricks
- Blog posts

## 🚀 Getting Started

### Prerequisites
- OpenClaw 2.0.0 or higher
- Basic understanding of prompt engineering
- Familiarity with Markdown

### Development Setup

1. **Fork the repository**
   ```bash
   git clone https://github.com/YOUR_USERNAME/skill-prompt-generator.git
   cd skill-prompt-generator
   ```

2. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Edit files as needed
   - Test your changes
   - Update documentation

4. **Commit your changes**
   ```bash
   git add .
   git commit -m "Add: your feature description"
   ```

5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create Pull Request**
   - Go to GitHub
   - Create PR from your fork
   - Describe your changes
   - Link any related issues

## 📝 Style Guidelines

### Markdown
- Use proper Markdown formatting
- Keep lines under 100 characters
- Use semantic headers
- Include code blocks with language tags

### Templates
Follow this structure:
```
### Template Name

Brief description

**Structure:**
```
[Template content]
```

**Example:**
```
[Example usage]
```

**Customization:**
- Option 1
- Option 2

**Notes:**
- Additional notes
```

### Code Examples
- Include complete, working examples
- Add comments for clarity
- Follow Dart/Flutter best practices
- Test before submitting

## 🎨 Template Guidelines

### Quality Standards
- ✅ Clear and specific
- ✅ Includes all necessary sections
- ✅ Provides concrete examples
- ✅ Follows existing format
- ✅ Well-documented
- ✅ Tested with AI tools

### Template Structure
1. **Title**: Clear, descriptive name
2. **Description**: What it does
3. **Structure**: Template format
4. **Example**: Working example
5. **Customization**: Options available
6. **Notes**: Additional information

### Categories
- Flutter App Development
- Flutter Game Development
- Creative Writing
- Problem-Solving
- Educational
- Business & Marketing
- Programming

## ✅ Testing

Before submitting:
1. Test prompts with AI tools (ChatGPT, Claude, etc.)
2. Verify all examples work
3. Check formatting
4. Validate links
5. Review for clarity
