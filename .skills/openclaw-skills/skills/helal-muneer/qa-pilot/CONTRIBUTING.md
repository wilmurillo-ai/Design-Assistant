# Contributing to QA Pilot

First off, thanks for taking the time to contribute! 🎉

## How to Contribute

### Reporting Issues

Found a gap in the methodology? Have an idea for improvement?

1. Check [existing issues](../../issues) to avoid duplicates
2. Open a new issue with:
   - **Clear title** describing the problem or suggestion
   - **Context** — what type of project/agent were you using
   - **Expected vs actual behavior**

### Suggesting Improvements

QA Pilot is a living methodology. We welcome improvements to:

- **Testing patterns** — New ways agents can test their work
- **Edge case coverage** — Scenarios that should be tested
- **Framework-specific guidance** — React, Vue, Django, etc.
- **Agent-specific adaptations** — Claude, GPT, Gemini, Cursor, etc.
- **Reporting templates** — Better ways to communicate results

### Making Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-improvement`)
3. Make your changes
4. Test your changes against a real project (if applicable)
5. Commit with a clear message
6. Open a Pull Request

### Guidelines

- **Keep it accessible** — QA Pilot should be understandable by non-developers
- **Show, don't tell** — Use examples and concrete scenarios
- **Stay framework-agnostic** — Core methodology should work with any tech stack
- **Be practical** — Every suggestion should be actionable by an AI agent
- **No jargon** — Write for vibe coders, not just senior engineers

### Code of Conduct

Be respectful. Be constructive. We're all here to make AI agents less frustrating to work with.

## Questions?

Feel free to open an issue with the `question` label.
