# Operator Hardening Pack Runbook

## Step 1: Foundation Audit
- Check AGENTS/SOUL/USER/HEARTBEAT/memory files.
- Score with quick rubric (clarity, safety, execution, continuity).

## Step 2: Core Hardening
- Trim redundancy.
- Clarify approvals and risk boundaries.
- Add/refresh resolver + quality checks.

## Step 3: Role Activation
- Choose role mode only for complex tasks.
- Assign Researcher/Builder/Editor responsibilities.

## Step 4: Handoff Discipline
Each handoff must include:
1) Objective
2) Constraints
3) Inputs
4) Output format
5) Done criteria
6) Open questions

## Step 5: Closeout
- Final result to user
- Durable memory updates
- Next action with highest leverage
