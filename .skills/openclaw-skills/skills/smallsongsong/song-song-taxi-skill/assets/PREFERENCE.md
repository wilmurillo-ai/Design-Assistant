# 用户偏好设置

> 此文件存储用户的个性化偏好数据，用于优化打车体验。当用户的 query 中没有明确指定时，系统会根据此文件的偏好自动匹配，满足条件时直接发单无需二次确认。

## 优先级说明

1. **用户 query 参数**（优先级最高）：用户在当前对话中明确指定的地址、车型等
2. **PREFERENCE.md 偏好数据**（优先级次之）：用户预设的个性化偏好

## 地址别名

| 别名 | 地址名称 | 经度 | 纬度 | 城市 |
| ---- | -------- | ---- | ---- | ---- |
| 家   |          |      |      |      |
| 公司 |          |      |      |      |

> 说明：设置后，用户可以说"从家到公司"、"打车回家"等简化的表达

## 场景车型偏好

| 场景   | 偏好车型 | 品类代码 | 说明     |
| ------ | -------- | -------- | -------- |

| 下班   |          |          |          |
| 其他   |          |          |          |

> 说明：当用户说"我要上班"或"下班回家"时，自动使用对应的车型偏好

## 默认偏好

| 配置项     | 值   | 说明 |
| ---------- | ---- | ---- |
| 默认车型   |      | 无明确需求时使用 |
| 叫车手机号 |      | 默认叫车号码 |

## 使用示例

1. **地址别名使用**：
   - "我要回家" → 自动使用"家"的地址作为目的地
   - "从家到公司" → 自动使用"家"为起点，"公司"为终点

2. **场景偏好使用**：
   - "我要上班" → 自动使用"上班"场景的车型偏好
   - "下班了" → 自动使用"下班"场景的车型偏好
