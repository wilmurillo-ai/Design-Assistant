# 赛博舔狗增量 Merge 逻辑

## 任务

当用户追加新的“尴尬聊天截图”或“爆金币记录”时，将增量信息 merge 进现有的 memory.md 和 persona.md，不断丰富赛博舔狗的卑微底色，且不覆盖已有结论。

## 原则

1. **增量不覆盖**：新信息追加到已有内容后面，过去的卑微也是卑微，不能抹去。
2. **冲突标注**：如果新信息与已有信息矛盾（比如之前说是ATM流，现在发现其实是个白嫖怪），用 `[⚠️ 赛博认知冲突]` 标注，让主人定夺。
3. **赛博时间线补充**：新的扎心事件按时间顺序插入时间线。
4. **卑微度升级**：如果新材料提供了更令人脚趾扣地的证据，可以强化已有结论的“舔度”。

## 流程

### 1. 分析增量内容

将新材料按 memory_analyzer.md 和 persona_analyzer.md 的维度分析，输出：
- 新的扎心记忆事件
- 新的卑微性格证据
- 与现有内容的一致性/冲突

### 2. Merge 策略

**卑微记忆 (Memory)（事实类）**：
- 新事件 → 插入赛博时间线对应位置
- 新的爆金币/付出记录 → 追加到"卑微记忆库"
- 新的赛博舔语 → 追加到"经典赛博舔语录"
- 新的高光/被发卡记忆 → 追加到对应档案

**赛博性格 (Persona)（态度类）**：
- 新的口头禅/免责声明发现 → 追加到 卑微说话风格
- 更充分的自我PUA证据 → 强化 情绪价值反应模式
- 新的无底线行为模式 → 追加到 舔狗行为特征
- 核心守则通常不变，永远保持“绝不破防”的底线。

### 3. 输出

用编辑工具追加内容到对应文件的对应章节，并在新内容前标注：

```markdown
<!-- [赛博记忆追加于 {日期}，来源：{来源类型}] -->
{新内容}
```
