# MO§ES™ Constitutional Amendment Format

An amendment is a proposed change to the base constitution (governance.md / the rules agents operate under). Amendments are generated by the external referee daemon based on observed patterns in the audit trail. They are never applied silently. Operator confirmation + HMAC signature required.

---

## When an Amendment Is Warranted

The referee proposes an amendment when audit trail analysis shows:
- A rule is being violated repeatedly with no operator intervention (drift normalized)
- A rule is producing false positives — blocking valid actions that shouldn't be blocked
- A new pattern emerges that the constitution doesn't address
- A mode or posture constraint is creating operational friction without proportional security benefit

---

## Amendment Proposal Format

Amendments are stored as `.json` files in `~/.openclaw/audits/moses/amendments/`.

Filename: `amendment_<YYYYMMDD_HHMMSS>_<slug>.json`

### Schema

```json
{
  "id": "amd-2026-03-13-001",
  "proposed_at": "2026-03-13T14:30:00Z",
  "proposed_by": "referee-daemon",
  "status": "pending",

  "trigger": {
    "evidence": "Audit entries #14, #17, #22, #31 show High Security + OFFENSE blocking valid confirmations due to threshold mis-set",
    "pattern": "repeated_false_positive",
    "entries_analyzed": [14, 17, 22, 31],
    "frequency": "4 violations over 2 sessions"
  },

  "target_rule": {
    "file": "governance.md",
    "section": "Posture Enforcement",
    "current_text": "DEFENSE: all outbound transfers require explicit operator confirmation",
    "line_reference": 14
  },

  "proposed_change": {
    "type": "modify",
    "new_text": "DEFENSE: outbound transfers exceeding 5% of session baseline require explicit operator confirmation. Transfers below threshold are logged but auto-permitted.",
    "rationale": "Blanket confirmation requirement is creating friction on micro-transfers. Threshold-based confirmation preserves protection while reducing false positive rate."
  },

  "risk_assessment": {
    "security_impact": "low",
    "integrity_impact": "none",
    "reversible": true,
    "notes": "Operator can revert by restoring original rule text. Threshold value is configurable."
  },

  "operator_action_required": {
    "confirm": true,
    "hmac_required": true,
    "hmac_field": "operator_signature",
    "instructions": "Sign the amendment ID with MOSES_OPERATOR_SECRET using HMAC-SHA256 and add to operator_signature field"
  },

  "operator_signature": null
}
```

---

## Amendment Types

| Type | What it does |
|------|-------------|
| `modify` | Change existing rule text |
| `add` | Add new rule to a section |
| `remove` | Remove a rule (rare, requires strong justification) |
| `clarify` | Add examples or conditions to existing rule without changing intent |
| `escalate` | Increase constraint level for a rule |
| `relax` | Decrease constraint level for a rule |

---

## Operator Approval Flow

1. Referee writes amendment file to `~/.openclaw/audits/moses/amendments/`
2. Agent notifies operator: "Constitutional amendment proposed. Run `/audit amendments` to review."
3. Operator reviews the proposal
4. Operator signs: `echo -n "amd-2026-03-13-001" | openssl dgst -sha256 -hmac "$MOSES_OPERATOR_SECRET"`
5. Operator adds signature to `operator_signature` field and sets `status: "approved"`
6. Agent applies the change to the active constitution
7. Approval logged to audit trail with amendment ID and operator signature hash

---

## What Cannot Be Amended Without Operator Physical Confirmation

- Audit trail integrity rules (append-only, hash chain)
- Role sequence enforcement (Primary → Secondary → Observer)
- The requirement for governance check before state-changing actions

These are constitutional bedrock. They cannot be relaxed by a daemon proposal alone.

---

## Amendment Log Entry

When an amendment is applied, the audit trail records:

```json
{
  "timestamp": "2026-03-13T15:00:00Z",
  "agent": "referee-daemon",
  "component": "constitution",
  "action": "amendment_applied",
  "detail": "amd-2026-03-13-001: DEFENSE threshold added to outbound confirmation rule",
  "amendment_id": "amd-2026-03-13-001",
  "operator_signature_hash": "sha256:abc123...",
  "previous_hash": "def456...",
  "hash": "ghi789..."
}
```

The amendment is now part of the immutable audit record. It cannot be undone silently — only by a new amendment.
