# 进度管理标准

## 迭代管理

### 迭代周期

- 标准周期：2 周
- 紧急修复：随时插入
- 变更窗口：每个迭代第一周

### 迭代流程

1. **迭代计划会**：每个迭代第一天
2. **每日站会**：每天同一时间
3. **迭代评审**：迭代最后一天
4. **迭代回顾**：迭代评审后

## 进度跟踪

### 燃尽图

- 正常：曲线平滑下降
- 异常：曲线平坦/上升
- 风险阈值：剩余工作 > 计划 20%

### 里程碑

| 里程碑 | 交付物 | 容差 |
|--------|--------|------|
| 需求冻结 | 完整 PRD | 0 天 |
| 设计冻结 | 设计稿 + 评审 | -3 天 |
| 开发完成 | 代码提测 | -2 天 |
| 发布上线 | 功能上线 | 0 天 |

## 延期处理

1. 分析延期原因
2. 识别可压缩范围
3. 评估影响（范围/质量/成本）
4. 与干系人沟通
5. 调整计划并记录
