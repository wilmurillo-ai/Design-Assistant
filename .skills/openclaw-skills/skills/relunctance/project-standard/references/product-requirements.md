# 需求标准

## 需求撰写标准

### 需求文档结构

```
## 背景
- 业务问题是什么
- 受影响用户

## 目标
- 量化目标（尽量用数字）

## 用户故事
As a [用户角色]
I want [功能]
So that [价值]

## 验收标准
- [ ] 标准1
- [ ] 标准2

## 边界条件
- 异常情况1
- 异常情况2

## 依赖项
- 依赖方 | 依赖内容 | 预计完成时间
```

## 需求分级

| 级别 | 定义 | 交付要求 |
|------|------|----------|
| P0 | 核心功能，影响主流程 | 必须按时交付 |
| P1 | 重要功能，影响效率 | 尽量交付 |
| P2 | 优化功能，体验提升 | 尽量交付 |

## 需求澄清检查

- [ ] 角色：谁会用这个功能
- [ ] 场景：在什么情况下使用
- [ ] 目标：解决什么问题
- [ ] 验收：如何判断功能完成
- [ ] 异常：边界条件和异常如何处理

## 需求变更标准

1. 变更必须有书面记录
2. P0 变更需项目负责人审批
3. 变更后更新燃尽图
4. 变更影响及时同步相关方
