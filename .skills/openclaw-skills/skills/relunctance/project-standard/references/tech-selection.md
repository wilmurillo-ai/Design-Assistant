# 技术选型标准

## 选型评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 功能匹配 | 25% | 是否满足需求 |
| 学习成本 | 20% | 团队掌握难度 |
| 社区活跃度 | 15% | 生态是否成熟 |
| 稳定性 | 20% | 生产验证 |
| 扩展性 | 10% | 未来扩展能力 |
| 成本 | 10% | 采购/运维成本 |

## 选型决策流程

1. 调研备选方案（2-3个）
2. 技术验证（POC）
3. 团队评审
4. 负责人审批
5. 试点应用
6. 全量推广

## 禁止使用

- 无 LTS 版本的框架
- 社区已停止维护的库
- 有已知高危漏洞的组件
