# 质量标准

## 质量目标

| 指标 | 目标 | 警告线 |
|------|------|--------|
| 测试覆盖率 | ≥ 80% | < 75% |
| 缺陷修复及时率 | ≥ 95% | < 90% |
| 回归测试通过率 | ≥ 98% | < 95% |
| 上线后缺陷密度 | < 0.5%/月 | > 1%/月 |

## 缺陷等级

| 等级 | 定义 | 修复时限 |
|------|------|----------|
| P0 | 核心功能不可用，数据错误 | 4h |
| P1 | 重要功能异常，影响主要流程 | 24h |
| P2 | 次要功能异常，有替代方案 | 72h |
| P3 | 体验问题，不影响使用 | 2周 |

## 测试标准

### 测试通过标准

- 单元测试：≥ 95%
- 集成测试：≥ 90%
- 系统测试：100%
- 回归测试：≥ 98%

### 上线质量门禁

- [ ] 所有 P0/P1 缺陷已修复
- [ ] 核心功能回归通过
- [ ] 性能测试达标
- [ ] 安全扫描通过
