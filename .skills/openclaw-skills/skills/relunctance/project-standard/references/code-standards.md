# 代码规范

## Git 提交规范

### 提交信息格式

```
<类型>: <简短描述>

[可选的正文]

[可选的页脚]
```

### 类型

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 缺陷修复 |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| test | 测试相关 |
| chore | 构建/工具相关 |

### 示例

```
feat: 添加用户登录功能

- 支持手机号+验证码登录
- 支持第三方微信登录
- 登录态 7 天有效

Closes #123
```

## 代码审查标准

### 必须检查

- [ ] 代码逻辑正确
- [ ] 有单元测试
- [ ] 无安全漏洞
- [ ] 符合命名规范
- [ ] 注释清晰

### 审查 Checklist

| 维度 | 检查项 |
|------|--------|
| 正确性 | 逻辑正确、边界处理 |
| 性能 | 无 N+1、无大循环 |
| 安全 | 注入防护、鉴权 |
| 可读性 | 命名清晰、函数短 |
| 可维护性 | 解耦、模块化 |

## 代码风格

- 缩进：4 空格
- 行长度：≤ 120
- 命名：语义化，驼峰/下划线统一
- 函数长度：≤ 50 行

## 单元测试标准

- 覆盖率：≥ 80%
- 核心模块：≥ 95%
- 关键路径：100%
- 测试用例命名：test_功能_场景
