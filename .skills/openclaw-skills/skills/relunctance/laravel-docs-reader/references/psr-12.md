# PSR-12 Coding Standard Quick Reference

## Overview

PSR-12 is the extended coding style guide for PHP. All code generated by this skill complies with PSR-12.

---

## Key Rules at a Glance

### 1. Files

- PHP files MUST use `<?php` tag (no `<?=` unless outputting)
- Files MUST use UTF-8 without BOM
- Files SHOULD either declare symbols (classes, traits, etc.) or cause side effects (database operations, output), but NOT both
- Namespaces and classes MUST follow PSR-4 autoloading

```php
<?php

declare(strict_types=1);

namespace App\Http\Controllers;

class UserController extends Controller
{
    // ...
}
```

---

### 2. Lines & Indentation

- Use **4 spaces** for indentation (no tabs)
- Lines SHOULD be 120 characters or fewer; SOFT limit at 180
- One blank line after `namespace` declaration
- One blank line after block of `use` declarations
- `declare(strict_types=1)` MUST be first statement, with no blank lines before/after

```php
<?php

declare(strict_types=1);

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use Illuminate\Http\JsonResponse;

class UserController extends Controller
{
    public function show(int $id): JsonResponse
    {
        // 4 spaces, not tabs
    }
}
```

---

### 3. Namespaces & Use Statements

```php
<?php

declare(strict_types=1);

namespace App\Http\Controllers;        // ✓ ONE blank line after

use App\Models\User;                 // ✓ Blank line between groups
use Illuminate\Routing\Controller;  // ✓ Alphabetical within group
use Illuminate\Http\Request;
```

**Grouped imports** (PHP 7.0+ style):
```php
use Illuminate\Support\Facades\{Auth, Cache, DB};
```

---

### 4. Classes, Traits, Interfaces

- Opening `{` on same line with ONE space before
- Closing `}` on its own line
- `extends` on SAME line (no `implements` on separate line)

```php
// ✓ Correct
class UserController extends Controller implements HasFactory
{
}

// ✗ Wrong
class UserController
    extends Controller
    implements HasFactory
{
}
```

**Abstract classes**: `abstract class Example`
**Interfaces**: `interface Example`
**Traits**: `trait Example`

---

### 5. Methods & Functions

- Visibility (`public`, `protected`, `private`) MUST be declared on all properties and methods
- `abstract` and `final` come before `protected`/`public`
- `static` comes after visibility

```php
abstract protected static function factory(): static;
public    static function make(): static;
protected          $id;
private   static   $instance;
```

**Method chaining** (each method on new line):
```php
$query->where('active', true)
    ->orderBy('name')
    ->select('id', 'name')
    ->get();
```

---

### 6. Control Structures

```php
// if / elseif / else
if ($a === $b) {
    // ...
} elseif ($a === $c) {
    // ...
} else {
    // ...
}

// switch
switch ($variable) {
    case 'value1':
        // ...
        break;
    case 'value2':
        // ...
        break;
    default:
        // ...
}

// for / foreach / while
foreach ($items as $item) {
    // ...
}

// try / catch
try {
    // ...
} catch (FirstException $e) {
    // ...
} catch (OtherException $e) {
    // ...
}
```

---

### 7. match Expression (PHP 8.0+)

```php
$result = match ($status) {
    'active'   => 'User is active',
    'inactive' => 'User is inactive',
    'banned'   => 'User is banned',
    default   => 'Unknown status',
};
```

---

### 8. Operators

```php
// ✓ Spaces around operators
$a = $b + $c;
$name = isset($user) ? $user->name : 'Anonymous';

// ✗ No spaces inside parentheses
$name = isset($user) ? $user->name : 'Anonymous';

// Nullsafe operator (PHP 8.0+)
$country = $session?->user?->getAddress()?->country;

// Spaceship operator
$a <=> $b;  // Returns -1, 0, or 1
```

---

### 9. Strings

```php
// Concatenation: space before and after .
$greeting = 'Hello, ' . $name . '!';

// Heredoc / Nowdoc
$html = <<<HTML
<div class="card">
    <h1>{$title}</h1>
    <p>{$body}</p>
</div>
HTML;
```

---

### 10. Arrays

```php
// Short array syntax (PSR-12)
$items = ['first', 'second', 'third'];

// Trailing comma (allowed in PSR-12)
$items = [
    'first',
    'second',
    'third',  // ← trailing comma OK
];

// Associative
$config = [
    'host'     => '127.0.0.1',
    'port'     => 3306,
    'database' => 'forge',
];
```

---

### 11. Type Declarations (PHP 7.4+)

```php
class User
{
    public int    $id;
    public string $name;
    protected ?User $parent = null;

    public function __construct(
        public readonly string $email,
    ) {}

    public function getId(): int
    {
        return $this->id;
    }
}
```

**Return type**:
```php
public function getName(): string
{
    return $this->name;
}

public function getUser(): ?User  // nullable
{
    return $this->user;
}

public function setId(int $id): void  // void return
{
    $this->id = $id;
}
```

**Union types** (PHP 8.0+):
```php
public function getStatus(): string|int
{
    return $this->status;
}

public function getId(): int|null
{
    return $this->id;
}
```

---

### 12. Attributes (PHP 8.0+)

```php
#[Attribute]
class Documented
{
    public function __construct(
        public string $author,
        public ?string $version = null,
    ) {}
}

#[Documented(author: 'Laravel', version: '12')]
class Post extends Model
{
}
```

## Common PSR-12 Violations Checklist

| Check | Rule |
|-------|------|
| ☐ | `declare(strict_types=1)` is the first statement |
| ☐ | Indentation uses 4 spaces (not tabs) |
| ☐ | Opening brace ` `{ on same line as class/function declaration |
| ☐ | Visibility declared on ALL properties and methods |
| ☐ | `use` statements alphabetically ordered within groups |
| ☐ | One `use` statement per import (unless grouped) |
| ☐ | No trailing whitespace at end of lines |
| ☐ | Blank line before `return` (unless in nested structure) |
| ☐ | No hard tabs anywhere in the file |
| ☐ | Namespace has ONE blank line before first `use` |
| ☐ | Control structure keywords have ONE space before `( |
| ☐ | No space after function name before `(` |

## Laravel-Specific PSR-12 Notes

- **Controllers**: `App\Http\Controllers\`, one blank after namespace block
- **Models**: `App\Models\`, `$fillable`/`$casts`/`$hidden` as arrays with trailing commas
- **Jobs**: `App\Jobs\`, implement `ShouldQueue` with `use Dispatchable, Queueable, SerializesModels`
- **Migrations**: Method chain `$table->string()->unique()` where possible
- **Factories**: Return array, use `fake()->` for all generated data
- **Routes**: Closures in routes file should be avoided — use controllers
