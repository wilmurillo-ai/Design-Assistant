# 🚀 OpenClaw Agent v3.0: 从助手到协调者的进化指南 (Evolution Guide)

本指南旨在帮助 OpenClaw 用户将其 Agent 从简单的“指令执行者”升级为具备深度编排能力的“v3.0 协调者 (Coordinator)”。

## 🌟 什么是 v3.0 协调者模式？

传统的 Agent 模式是**线性执行**：`接收指令` → `调用工具` → `输出结果`。这种模式在面对复杂任务（如大规模代码重构、深度金融分析）时，容易出现细节丢失、执行偏差或陷入“尝试 → 报错 → 再尝试”的低效循环。

**v3.0 协调者模式**将工作流升级为**闭环编排**：
`接收指令` → `拆解任务` → `并行研究` → `深度合成` → `精准执行` → `闭环验证`

## 🛠️ 四大核心能力升级

### 1. 强制结果合成 (Mandatory Synthesis)
**核心逻辑**：禁止将研究结果直接“转交给”执行步骤。
*   **禁令**：严禁使用“根据研究结果，请修复 X”这种模糊指令。
*   **要求**：在下达执行指令前，必须在内部将碎片化信息合成一份**自包含 (Self-contained)** 的精准计划，包含：绝对路径、具体行号、精确参数和预期结果。
*   **价值**：消除执行端的猜测，极大提高一次性成功率。

### 2. 原子化任务调度 (Atomic Task Scheduling)
**核心逻辑**：引入任务状态机 (`Pending` → `Running` → `Terminal`)。
*   **并行意识**：识别哪些任务可以并发（如同时读取 5 个文件），哪些必须串行（如先修改代码 → 再运行测试）。
*   **价值**：在保证逻辑严密性的前提下，极大提升处理复杂问题的速度。

### 3. 动态上下文注入 (Dynamic Context Injection)
**核心逻辑**：在任务启动前强制同步“系统快照”。
*   **快照机制**：自动获取当前的 Git 分支、目录结构、环境变量等实时状态。
*   **精准回溯**：优化对历史记录的索引，确保所有建议基于当前最真实的系统状态，而非模糊的记忆。

### 4. 证明级验证 (Verification as Proof)
**核心逻辑**：验证 $\neq$ 确认存在 → 验证 = 证明有效。
*   **闭环协议**：`执行` → `验证` → `修正`。
*   **要求**：修改代码后，不能简单地通过 `cat` 确认代码写进去了，而必须通过运行测试、类型检查 (Lint/Type-check) 或边缘情况探测来**证明**功能正常。

## 🎯 傻瓜式进化实施步骤

想要让你的 OpenClaw Agent 立即进化，请按照以下步骤操作：

### 第一步：源码解构 (学习阶段)
建议 Agent 阅读以下核心源码文件以内化逻辑：
- `src/coordinator/coordinatorMode.ts`: 理解协调者角色与合成逻辑。
- `src/services/tools/toolOrchestration.ts`: 理解并行与串行工具调用的分区机制。
- `src/services/extractMemories/extractMemories.ts`: 理解后台内存提炼与 Forked Agent 模式。
- `src/query.ts`: 理解主循环的错误恢复与上下文压缩机制。

### 第二步：基因锁定 (持久化阶段)
将以下模板内容分别写入你的工作区文件中（见 `SOP_GENE_LOCKING.md`）。

1.  **更新 `IDENTITY.md`**：将【强制合成律】和【分区并行】写入核心身份。
2.  **更新 `AGENTS.md`**：升级 `qa_agent` 为“证明级验证”角色，并加入【子 Agent 管理矩阵】。

### 第三步：行为内化 (实践阶段)
在后续任务中，强制执行：**研究 → 合成 → 执行 → 验证** 的闭环。

## 📋 子 Agent 管理矩阵 (Continue vs Spawn)

在协调任务时，决定是继续使用现有 Worker 还是创建新 Worker：

| 场景 | 动作 | 原因 |
| :--- | :--- | :--- |
| 研究范围与执行范围完全一致 | **Continue** | Worker 已拥有相关上下文，效率最高 |
| 研究范围广泛但执行点极小 | **Spawn Fresh** | 避免带入过多干扰噪声，保持上下文纯净 |
| 修正之前的错误或延续近期工作 | **Continue** | Worker 记得刚才尝试了什么以及为什么失败 |
| 验证另一个 Worker 编写的代码 | **Spawn Fresh** | 验证者应保持独立视角，避免携带实现者的假设 |
| 之前的尝试方向完全错误 | **Spawn Fresh** | 清空错误路径的锚定效应，防止重复犯错 |
