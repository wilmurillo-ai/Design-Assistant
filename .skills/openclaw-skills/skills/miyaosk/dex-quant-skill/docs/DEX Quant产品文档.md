## 产品概述

DEX Quant 是一个基于 AI 的量化交易工具，通过 Telegram 对话即可完成从策略创建到实盘部署的全流程。无需编程经验，无需搭建本地环境。

### 核心能力

| 功能 | 说明 | 费用 |
|------|------|------|
| 策略创建 | 自然语言描述交易想法 → AI 生成 Python 策略脚本 | 免费 |
| 历史回测 | 服务器拉取 K 线数据，模拟历史交易，输出绩效报告 | 免费 |
| 参数优化 | 6 种优化算法自动搜索最优参数组合 | 免费 |
| 实时监控 | 服务器 7×24 定时执行策略，产生买卖信号推送 | 免费（3 个策略） |
| 自动下单 | 产生信号后自动下单到 Hyperliquid DEX | 免费（需配置密钥） |

### 支持的交易所 & 数据源

- **行情数据**: Binance 全币种（永续合约 + 现货）
- **自动下单**: Hyperliquid DEX（主网 + 测试网）
- **K 线周期**: 1m / 5m / 15m / 1h / 4h / 1d

## 快速开始

### 第一步：推荐策略

在聊天中发送：

```
推荐策略
```

AI 会推荐 2 个经过实测的正收益策略，选一个数字即可回测。

### 第二步：回测验证

```
回测
```

服务器自动拉取历史数据、模拟交易，返回绩效报告和权益曲线图。

包含 7 项评分指标：收益率、Sharpe 比率、最大回撤、胜率、盈亏比、交易数、爆仓次数。
评级从 A（优秀）到 F（失败）。

### 第三步：参数优化

```
优化
```

选择优化算法（推荐 genetic 遗传算法），服务器自动搜索最优参数组合。

### 第四步：部署监控

```
部署
```

选择运行模式后，服务器 7×24 监控策略，产生信号后推送给你。

## 功能详解

### 1. 策略创建

**用自然语言描述你的交易想法，AI 自动生成可运行的策略脚本。**

示例对话：

```
用户: 帮我做一个 BTC 策略，RSI 超卖时买入，超买时卖出，用 EMA 过滤趋势
AI:   ✅ 策略已生成 → btc_rsi_ema_strategy.py
```

支持的信号源：
- 技术指标：EMA、SMA、RSI、MACD、布林带、ATR、KDJ
- 自定义条件：交叉、突破、回踩等
- 多空双向交易

### 2. 历史回测

**将策略在历史数据上模拟运行，验证是否有效。**

配置参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 交易对 | BTCUSDT | 支持 Binance 全币种 |
| K 线周期 | 4h | 1m / 5m / 15m / 1h / 4h / 1d |
| 回测区间 | 2025 全年 | 自定义起止日期 |
| 杠杆倍数 | 3x | 1-125x |
| 初始资金 | 100,000 USDT | 自定义 |
| 交易方向 | 多空双向 | 仅多 / 仅空 / 多空 |

输出内容：
- 权益曲线图（PNG）
- 综合评分（A-F）
- 关键指标：总收益、Sharpe、最大回撤、胜率、盈亏比、交易次数

### 3. 参数优化

**用算法自动搜索策略的最优参数组合。**

| 算法 | 特点 | 适用场景 |
|------|------|----------|
| genetic（遗传算法） | 全局搜索，推荐 | 默认选择 |
| bayesian（贝叶斯） | 少量评估即可收敛 | 需要快速结果 |
| grid（网格穷举） | 遍历所有组合 | 参数空间小（≤200 组合） |
| random（随机搜索） | 高维空间探索 | 参数较多时 |
| annealing（模拟退火） | 跳出局部最优 | 陷入局部最优时 |
| pso（粒子群） | 连续参数优化 | 参数全是浮点数 |

优化目标可选：Sharpe 比率（默认）、总收益率、最大回撤、胜率、盈亏比。

### 4. 实时监控

**服务器 7×24 定时执行策略，产生买卖信号。**

两种模式：

#### 模式 1：仅监控信号

- 服务器定时执行策略，产生信号推送给你
- 你收到信号后自行决定是否交易
- 不需要提供任何密钥
- 免费 3 个策略同时运行

#### 模式 2：监控 + 自动下单

- 服务器产生信号后自动下单到 Hyperliquid DEX
- 需要通过**安全链接**提交钱包私钥（不在聊天中输入）
- 支持测试网和主网
- 免费 3 个策略同时运行

监控管理命令：

| 命令 | 说明 |
|------|------|
| 查看状态 | 查看某个监控任务的运行状态和最近信号 |
| 列出全部 | 列出所有运行中的监控任务 |
| 停止监控 | 停止某个监控任务 |

## 安全机制

### 密钥保险箱 (Vault)

自动下单需要 Hyperliquid 钱包私钥。为确保安全，我们设计了**密钥保险箱**系统：

```
                    ┌─────────────────────────┐
  聊天对话 ──×──→   │   私钥不经过聊天记录     │
                    └─────────────────────────┘
                               │
                    ┌──────────▼──────────────┐
  安全链接 ──────→  │  浏览器 HTTPS 表单提交   │
                    └──────────┬──────────────┘
                               │
                    ┌──────────▼──────────────┐
  服务器存储 ────→  │  AES-256-GCM 加密存入DB  │
                    └─────────────────────────┘
```

**安全保障：**

| 特性 | 说明 |
|------|------|
| 传输加密 | 全程 HTTPS，私钥不在聊天中出现 |
| 存储加密 | AES-256-GCM 加密，主密钥与数据隔离 |
| 一次性链接 | 30 分钟过期，用后作废 |
| 随时删除 | 在聊天中说「删除密钥」即可立即清除 |
| 网络可选 | 支持测试网（推荐先用）和主网 |

**操作流程：**

1. 在聊天中说「设置密钥」或选择「监控 + 自动下单」模式
2. AI 生成一个安全链接发给你
3. 在浏览器中打开链接 → 粘贴私钥 → 选择网络 → 提交
4. 回到聊天确认 → 开始部署
