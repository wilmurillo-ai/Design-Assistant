# Component & API Design（组件与 API 设计）

设计易用、可扩展、易维护的组件 API 与文件结构。

## What It Does

- 职责单一与组件拆分建议
- Props 设计（必要才加、命名一致、类型明确、可控/不可控）
- 扩展方式：children、slots、className、透传、variant/size
- 事件与回调命名与参数约定
- 文件与目录结构建议（index、types、styles、子组件）

## How to Use

当用户说「设计这个组件」「组件怎么封装」「API 怎么定」「props 怎么设计」时启用，输出《组件设计》含 API 表、事件、插槽、示例与文件结构。

## Requirements

- 无额外依赖
- 适用于 React/Vue 及现有组件库约定
