# 杯柄形态识别算法说明

本文档详细说明杯柄形态识别算法的技术细节和评分标准。

## 算法架构

```
检测流程：
1. 数据获取 → 2. 前置趋势检测 → 3. 杯身检测 → 4. 把手检测 → 5. 突破信号检测 → 6. 综合评分
```

## 1. 数据获取

使用 Tushare Pro API 获取股票日线数据：
- 时间范围：默认365天（可配置）
- 数据字段：开盘价、收盘价、最高价、最低价、成交量

## 2. 前置趋势检测

**目的**：确认杯柄形态形成前已有上升趋势。

**标准**：
- 前期涨幅 ≥ 30%
- 时间跨度：杯身开始前30-60天

**计算方法**：
```
pre_gain = (pre_high - pre_start) / pre_start
```

**评分权重**：前置涨幅越高，评分越高（上限50%）。

## 3. 杯身检测

### 3.1 U型底部检测

**U型 vs V型判别**：

U型特征：
- 底部区域平坦（标准差小）
- 左右两侧斜率对称（下降→上升）
- 底部持续时间较长

V型特征：
- 底部急剧反转
- 底部区域狭窄

**检测方法**：

1. 找到最低点 `bottom_idx`
2. 取底部前后各15%区域计算标准差
3. 计算平坦度得分：
   ```
   flatness_score = 1 - (bottom_std / price_range)
   ```
4. 计算斜率对称性：
   ```
   left_slope = 下降斜率（负值）
   right_slope = 上升斜率（正值）
   slope_score = min(|left_slope|, |right_slope|) / max(|left_slope|, |right_slope|)
   ```
5. 综合U型得分：
   ```
   u_score = flatness_score * 0.4 + slope_score * 0.6
   ```

**阈值**：U型得分 ≥ 0.3 才视为有效杯身。

### 3.2 杯身深度检测

**理想范围**：前期涨幅的1/3到1/2

**计算方法**：
```
cup_high = max(左杯口, 右杯口)
cup_low = 杯底价格
cup_depth = (cup_high - cup_low) / cup_high
```

**评分标准**：
- 深度 15-33%：满分（0.5分）
- 深度 10-15% 或 33-50%：减半（0.2分）
- 深度 <10% 或 >50%：不合格

### 3.3 杯身时间跨度

**理想范围**：30-180天（1-6个月）

检测时遍历不同起点，寻找最优杯身区间。

## 4. 把手检测

### 4.1 把手位置

- 必须位于右杯口之后
- 时间跨度：5-30天（1-4周）

### 4.2 把手形态

**特征**：
- 向下倾斜或横向整理
- 不能明显上涨（斜率 > 0.1 不合格）

**计算方法**：
```
handle_slope = normalize(线性回归斜率)
```

### 4.3 把手深度

**限制**：不超过杯深的1/3

```
handle_depth = (handle_high - handle_low) / handle_high
constraint: handle_depth < cup_depth / 3
```

**保护底线**：把手最低点不能低于杯底。

### 4.4 成交量萎缩

把手形成期间成交量应低于杯身平均水平：

```
avg_cup_vol = mean(cup_volume)
avg_handle_vol = mean(handle_volume)
vol_ratio = avg_handle_vol / avg_cup_vol

vol_score = 1 - vol_ratio  (if vol_ratio < 1)
```

理想状态：成交量萎缩 20-50%。

## 5. 突破信号检测

### 5.1 价格突破

```
breakout_level = 右杯口价格
latest_price ≥ breakout_level * 0.95  # 接近或突破
```

### 5.2 成交量突破

突破时成交量应激增：

```
recent_vol = mean(最近5天成交量)
avg_vol = mean(杯身+把手期间成交量)
vol_breakout = recent_vol > avg_vol * 1.5
```

理想突破：成交量放大50%以上。

## 6. 综合评分

```
total_score = cup_score * 0.4 
            + handle_score * 0.3 
            + breakout_signal * 0.2 
            + vol_breakout * 0.1
```

**评分范围**：0-1

**阈值建议**：
- ≥ 0.7：高质量杯柄形态
- 0.5-0.7：中等质量
- < 0.5：形态不完整或不标准

## 检测失败原因

当检测返回"不匹配杯柄形态"时，可能原因：

1. **未检测到符合条件的杯身**
   - U型得分过低（<0.3）
   - 杯身深度不合理
   - 缺乏前置上升趋势（涨幅<30%）

2. **未检测到符合条件的把手**
   - 把手上涨而非下跌/整理
   - 把手深度超过杯深1/3
   - 把手跌破杯底
   - 时间跨度不符合

3. **数据不足**
   - 日线数据少于180天
   - 数据缺失或异常

## 图表生成

当检测成功时，生成的图表包含：

### 主图（蜡烛图）
- 红色蜡烛：收盘>开盘（上涨）
- 绿色蜡烛：收盘<开盘（下跌）
- 蓝色背景：杯身区域
- 橙色背景：把手区域
- 红色虚线：颈线（突破位）

### 标注点
- 左杯口：杯身起点高点
- 杯底：最低点
- 右杯口：杯身终点高点
- 把手：把手起点

### 信息框
- 形态评分
- 各项参数（杯深、U型得分、把手深度等）
- 突破信号状态
- 目标价计算

### 成交量图
- 柱状图显示成交量
- 颜色对应涨跌
- 区域标注杯身和把手成交量区间

## 算法局限性

1. **形态主观性**：杯柄形态边界有一定主观判断空间，不同分析师可能判断不同
2. **参数敏感性**：阈值参数（如U型得分0.3）可能需要根据市场调整
3. **历史数据依赖**：仅使用历史数据，不考虑未来走势
4. **单一形态**：仅检测杯柄形态，不检测其他技术形态

## 参数配置

可通过脚本参数调整：
- `--days`：数据获取天数（默认365）
- 算法阈值需修改源码中的常量
