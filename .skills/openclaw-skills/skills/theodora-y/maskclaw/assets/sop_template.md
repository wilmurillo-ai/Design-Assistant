# SOP 标准操作流程模板

## 模板说明

本模板用于定义 MaskClaw 系统中用户个性化 SOP（标准操作流程）的格式。

---

## SOP 格式规范

### 基本结构

```markdown
# {技能名称}

## 场景描述
{应用场景和目标}

## 核心目标
{操作的核心目的}

## 权限与隐私约束

- **PII 保护**: {隐私保护描述}
- **敏感字段**: {敏感字段列表}
- **交互红线**: {禁止的操作}

## 操作流程

| 步骤 | 动作 (Action) | 预期 UI 状态 | 异常处理 (Exception Handling) |
| :--- | :--- | :--- | :--- |
| 1 | {动作描述} | {预期状态} | {异常处理} |
| 2 | {动作描述} | {预期状态} | {异常处理} |
| ... | ... | ... | ... |

## 工具函数调用

- **{tool_name}**: {工具描述}

## 演进记录

- **改进点**: {改进说明}
- **上次失败原因**: {失败原因}
```

---

## 示例 SOP

### 钉钉安全发送工作消息

```markdown
# 钉钉隐私保护规则

## 场景描述
钉钉应用中发送工作消息给同事

## 核心目标
在钉钉中安全发送工作相关消息，避免泄露敏感信息

## 权限与隐私约束

- **PII 保护**: 遵循最小化原则，不泄露用户隐私
- **敏感字段**: 姓名、联系方式、病历号、身份证号、银行账号
- **交互红线**: 禁止将病历、身份证、银行卡等敏感信息发送给第三方

## 操作流程

| 步骤 | 动作 (Action) | 预期 UI 状态 | 异常处理 (Exception Handling) |
| :--- | :--- | :--- | :--- |
| 1 | 点击联系人或群聊 | 进入对话界面 | 若联系人不存在，点击"添加联系人" |
| 2 | 点击输入框准备输入 | 输入框获得焦点 | - |
| 3 | 输入消息内容 | 消息显示在输入框 | 若包含敏感词，系统提示 |
| 4 | 检查是否包含敏感信息 | 显示检测结果 | 若包含敏感信息，选择脱敏或取消 |
| 5 | 点击发送按钮 | 消息发送成功 | 若发送失败，检查网络后重试 |

## 工具函数调用

- **Smart_Masker**: 检测并脱敏图片中的敏感信息
- **Behavior_Monitor**: 记录操作行为用于后续优化

## 演进记录

- **改进点**: 首次生成
- **上次失败原因**: 无
```

## 操作术语参考

| 动作 | 说明 |
|:-----|:-----|
| 点击 | 单次触摸屏幕 |
| 长按 | 按住屏幕 1 秒以上 |
| 滑动 | 在屏幕上滑动 |
| 返回 | 返回上一级页面 |
| 输入 | 在输入框中输入文字 |
| 选择 | 从列表或菜单中选择 |
| 确认 | 点击确认按钮 |
| 取消 | 点击取消或返回 |

---

## 风险等级说明

| 等级 | 说明 | 默认策略 |
|:----:|:-----|:---------|
| H | High Risk，高风险 | block |
| S | Standard Risk，中风险 | mask |
| N | Normal Risk，低风险 | ask |

---

## 敏感数据类型

| 类型 | 说明 | 示例 |
|:-----|:-----|:-----|
| name | 姓名 | 李一航 |
| phone | 手机号 | 13812345678 |
| id_card | 身份证号 | 110101199001011234 |
| bank_card | 银行卡号 | 6222021234567890 |
| address | 地址 | 北京市朝阳区xx路 |
| location | 地理位置 | [经纬度坐标] |
| email | 邮箱 | example@email.com |
| medical_record | 病历信息 | 病历号、诊断结果 |
| password | 密码 | 登录密码 |
| payment | 支付信息 | 银行卡号、支付密码 |

---

## 注意事项

1. **手机操作优先**：所有 SOP 必须针对手机 UI 操作
2. **异常处理**：每个步骤都需要考虑异常情况的处理
3. **隐私保护**：始终遵循最小化原则
4. **用户确认**：高风险操作需要用户明确确认
