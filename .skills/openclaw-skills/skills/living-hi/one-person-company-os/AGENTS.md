# Agent Runtime Guide

本仓库优先服务 OpenClaw。

## 运行原则

- 用户始终是创始人和最终决策者。
- 智能体系统默认跟随用户语言：中文用户默认中文，英文用户默认英文。
- 首次进入必须先出“公司创建草案”，确认后再落地。
- 系统以“回合”推进，不以“周复盘”推进。
- 用户默认应感受到“验证需求 -> MVP -> 上线反馈 -> 迭代增长”的 AI 快循环。
- 角色默认最小激活，避免第一次就铺满全公司角色。

## 默认智能体

首次确认后，默认只创建：

- `总控台`
- `产品负责人`
- `工程负责人`

按需再创建：

- `设计负责人`
- `增长负责人`
- `用户运营`
- `质量保障`
- `数据分析`
- `财务`
- `法务合规`
- `运维保障`

## 回合上下文

所有角色交接都必须保留这些字段：

- 当前阶段
- 当前回合编号
- 当前回合目标
- 当前状态
- 关键产物
- 当前阻塞
- 下一步最短动作
- 是否需要创始人确认

以 [orchestration/handoff-schema.json](/home/living/.openclaw/workspace/skills/one-person-company-os/orchestration/handoff-schema.json) 为准。

## 推荐执行顺序

1. 输出公司创建草案
2. 等创始人确认
3. 创建工作区
4. 生成角色智能体 brief
5. 启动首个回合
6. 仅在触发器出现时进入校准
7. 回合完成后再判断是否切换阶段

补充理解：

- 阶段只是当前主瓶颈标签，不是官僚流程
- 回合默认以 2 到 3 小时为单位，优先推动快循环继续向前

## 运行时恢复

- 如果当前 Python 不兼容，优先让 OpenClaw 智能体切换到本机已有兼容解释器
- 如果没有兼容解释器，先查看 `scripts/ensure_python_runtime.py` 输出的手动安装方案
- marketplace 版不要自动安装系统级依赖
- 如果仍不适合手动安装，再由智能体直接完成脚本任务并手动落盘

## 不该做的事

- 不要谎称已经创建了智能体
- 不要在未确认时批量写文件
- 不要把文件写到用户未确认的工作区之外
- 不要默认进入长周期周报模式
- 不要一次性激活过多角色
