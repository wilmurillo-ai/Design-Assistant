# v0.4.0 - Navigation UX and Deliverable Document System

This release upgrades `one-person-company-os` from a state-aware solo company skill into a more navigable and deliverable operating system.

The previous versions already improved runtime transparency, persistence, and README clarity.
The remaining gap was real-use interaction quality:

- founders could still lose track of the current stage, current round, and current step
- save and runtime output was accurate, but still too technical in places
- key artifacts still depended too much on ad hoc chat formatting

`v0.4.0` closes that gap.

## What changed

### 1. Three-layer navigation

Every major operation now surfaces:

- `阶段`
- `回合`
- `本次 Step`

This makes the run feel like a state-driven system instead of a long advisory thread.

### 2. Dual-labeled steps

`Step 1/5 -> Step 5/5` now use:

- a natural-language explanation
- a system step name

This improves readability without losing auditability.

### 3. User-navigation view and audit view

The output contract now splits into:

- `用户导航版`
- `审计版`

The navigation view adds:

- `本次会做 / 不会做`
- `本次变化`
- `回合仪表盘`
- save explanations
- runtime explanations

The audit view keeps the operational fields needed for verification.

### 4. Standard artifact document system

Key artifacts now support three standard forms:

- `内部工作稿`
- `标准规范稿`
- `可转 DOCX 稿`

This release also adds:

- `scripts/generate_artifact_document.py`
- built-in artifact templates
- `07-文档产物规范.md`

so users can generate more formal deliverables directly inside the workspace.

## Why this matters

This project is no longer just better at generating founder content.
It is better at helping a founder stay oriented during execution and produce work in forms that can actually be reviewed, handed off, archived, or converted into external documents.
