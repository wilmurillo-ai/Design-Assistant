# One Person Company OS v0.6.4

`v0.6.4` is a trust-and-bootstrap patch on top of `v0.6.3`.

## What Changed

- removed the hard-coded leaked vertical case from the generated landing copy, interview board, trial intake, feedback capture, and demo surfaces
- enforced a direction-first bootstrap gate so workspace creation now requires confirmed founder inputs instead of placeholder defaults
- aligned README, release docs, validation, and agent prompt metadata with the real runtime contract

## Why This Matters

The previous surface created a serious mismatch:

- the docs said the system would ask for founder direction first
- the actual generated files still behaved like one specific hidden case

This patch fixes that mismatch. The package now behaves like a reusable one-person-company operating system instead of a disguised vertical template.
