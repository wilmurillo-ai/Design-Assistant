# v0.5.8 - Final-Named Deliverables And Real Document Surface

`v0.5.8` fixes a serious product mismatch: generated artifacts still looked like document specification and placeholder management instead of real deliverables.

This release changes that by making final-output naming the default:

- generated DOCX files now use their final deliverable names from the start
- status markers such as `[待生成]` and `[已生成]` are removed from file names
- `07-交付物地图.md` replaces the old document-spec entry point
- `11-交付目录总览.md` replaces the old pending/completed board
- document maturity is expressed inside file content rather than in the file path
- release validation now checks the new final-document model end to end

The result is a cleaner workspace surface, clearer file paths, and a stronger feeling that the system is producing real formal documents instead of document placeholders.
