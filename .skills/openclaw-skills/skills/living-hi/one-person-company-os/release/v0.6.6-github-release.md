# One Person Company OS v0.6.6

`v0.6.6` is a marketplace safety-boundary hardening patch on top of `v0.6.5`.

## What Changed

- disabled automatic system-package installation in `scripts/ensure_python_runtime.py`
- kept Python compatibility guidance, interpreter switching, and target-script reruns when a trusted compatible interpreter already exists
- constrained persisted artifact output and saved role briefs to stay inside the approved company workspace
- disabled implicit invocation in `agents/openai.yaml`
- synchronized README, SKILL, security docs, listing copy, publishing notes, and release material with the new runtime safety boundary

## Why This Matters

The previous release fixed the founder-visible workspace surface, but the marketplace-facing contract still looked riskier than the actual product intent:

- public docs implied the package might change the host environment automatically
- some write-path options were too open-ended for a higher-authority skill
- metadata and public prompts did not explain the write boundary clearly enough

This patch closes that gap.
The package now makes three things explicit:

- script mode expects an existing trusted Python runtime
- the marketplace build does not auto-install system packages
- persisted writes stay inside the founder-approved workspace
