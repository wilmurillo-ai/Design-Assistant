# v0.5.1 - Bilingual Runtime And Document Upgrade

This release makes `one-person-company-os` work cleanly for both Chinese and English founders during real execution.

The workflow was already structured, state-aware, and persistence-aware.
The remaining gap was language continuity across the real product surface:

- runtime reports could still fall back to Chinese-only output
- generated role briefs and formal documents did not fully follow English execution
- public release copy still framed the project too narrowly

`v0.5.1` closes that gap.

## What changed

### 1. Language-aware runtime reports

The core workflow scripts now emit Chinese or English runtime output based on the active language.

### 2. English generated materials

The system now generates:

- English role briefs
- English workspace document content
- English DOCX artifact content

while keeping the canonical workspace layout stable for automation compatibility.

### 3. Bilingual public positioning

README, guides, ClawHub listing copy, media kit text, and publishing notes now describe the project as a bilingual operating system for solo founders.

### 4. Stronger release validation

`scripts/validate_release.py` now verifies the English path in addition to the existing Chinese path.

## Validation

Validated locally with:

```bash
python3 -m py_compile scripts/*.py
python3 scripts/validate_release.py
git diff --check
```
