# v0.5.6 - Founder Intake And Placeholder Deliverable Control

`v0.5.6` upgrades the product in two places that matter during real use:

- the founder no longer has to answer a heavy first-run form before the system can help
- the workspace now makes pending vs completed formal deliverables obvious at a glance

This release adds:

- proactive one-line founder intake with 3 to 4 optional startup directions
- `10-创始人启动卡.md` for low-burden startup guidance inside the workspace
- `11-交付状态总览.md` for a visible pending/completed deliverable board
- starter DOCX files marked `[待生成]` and automatic promotion to `[已生成]`
- review-path and improvement reminders in every runtime report
- stricter validation for placeholder naming, lifecycle promotion, and removal of legacy unnumbered artifact folders

The result is a more useful company OS:

- easier to start
- easier to inspect
- easier to keep formally deliverable
