# v0.3.3 - README Professionalization and Architecture Clarity

This patch release upgrades the public product surface of `one-person-company-os`.

The core system was already stronger after the runtime and state-transparency work.  
The remaining gap was presentation quality:

- the README still did not explain the product fast enough
- the architecture was not explicit enough
- the first impression was not professional enough
- install and start were not minimal enough

This release closes that gap.

## What Changed

### 1. The README is now structured like a product document

The main README now answers, in order:

- what this is
- who it is for
- what problem it solves
- what the user gets
- how the system is architected
- how the operating model works
- how to install it in one line
- how to start it in one line

### 2. The system architecture is now explicit

The README now describes the layered model directly:

- Founder
- Control Tower / Skill Protocol
- Round Engine
- Workspace Persistence Layer
- Runtime Recovery Layer

That makes the product feel much more like an operating system and much less like a prompt pack.

### 3. Onboarding is faster

This release adds:

- one-line install
- one-line start

That lowers the friction between “I understand this” and “I can try this now”.

### 4. Public-facing copy is now aligned

The sharper positioning is now reflected in:

- `README.md`
- `README.zh-CN.md`
- release READMEs
- ClawHub listing copy
- agent metadata
