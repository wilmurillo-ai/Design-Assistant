# OpenClaw 运行约定

## 默认流程

1. 先执行 preflight，判断 `installed / runnable / workspace_created / persisted`
2. 输出公司创建草案
3. 等用户确认
4. 创建中文工作区
5. 生成角色 brief
6. 启动首个回合

## 固定 5 步状态机

1. `Step 1/5` 模式判定
2. `Step 2/5` preflight 与保存策略检查
3. `Step 3/5` 草案 / 变更提议 / 当前状态装载
4. `Step 4/5` 执行与落盘
5. `Step 5/5` 验证与回报

## 执行模式

- `模式 A：脚本执行`
- `模式 B：手动落盘`
- `模式 C：纯对话推进`

脚本失败时优先从 A 降级到 B。

## Python 兼容处理

- 兼容目标默认是 `Python 3.7+`
- 如果当前 Python 不兼容，优先让 OpenClaw 智能体切换到本机已有的兼容解释器
- 如果本机没有兼容解释器，先查看 `scripts/ensure_python_runtime.py` 输出的手动安装方案
- marketplace 版不要自动安装系统级依赖
- 如果不适合手动安装，则由智能体直接完成脚本任务并手动落盘
- 不要只抛给用户一个版本错误后停止

## 默认创建的智能体

- 总控台
- 产品负责人
- 工程负责人

## 默认透明输出

每次关键产出后都要固定给出：

- `状态栏`
- `保存状态`
- `运行状态`

其中 `保存状态` 必须明确：

- 是否已保存
- 保存路径
- 文件名
- 未保存原因

`运行状态` 还应明确：

- `python_supported`
- `兼容目标`
- `可切换解释器`
- `智能体建议动作`

## 注意事项

- 宿主不支持真实智能体时，只能模拟，不得虚构
- 不要在未确认时批量写文件
- 不要把文件写到用户未确认的工作区之外
- 触发器出现前，不要主动进入重型校准
- 当前内容若只存在聊天中，必须明确写出“未保存”
