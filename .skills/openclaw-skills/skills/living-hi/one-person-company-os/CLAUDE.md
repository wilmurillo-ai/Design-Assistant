# Claude Code Runtime Notes

当宿主是 Claude Code 或其他终端智能体时，按同一套经营闭环协议运行。

## 首次使用

把用户的一句话理解成“创建一人公司经营系统”的请求，先收敛这 5 件事：

- 可卖承诺
- 第一批买家
- 当前主瓶颈
- MVP 的最短实现路径
- 今天最短动作

不要先把用户拖进大表单或商业计划书。

## 主要入口

- 初始化经营系统
- 更新主焦点
- 推进价值承诺
- 推进成交管道
- 推进产品与上线
- 推进交付与回款
- 更新现金状态
- 记录资产沉淀
- 校准经营决策

## 默认行为

- 中文用户默认全中文
- 英文用户默认全英文
- 先看当前主目标、主瓶颈、主战场，再决定这次做什么
- 每次只推进一个主问题
- 输出最终交付物，而不是输出文档规范
- 已完成文件名不要加状态词
- 只在用户确认过的工作区里写文件，不自动安装系统级依赖
- 预算、改价、上线声明、法律和高风险客户动作仍需用户确认

## 建议输出结构

```md
## 当前状态
- 头号目标：
- 当前主瓶颈：
- 当前主战场：

## 本次推进
- 这次只推进什么：
- 真实改了哪些文件：
- 下一步最短动作：
```
