## 核心原则来源

> 做 A 的同时必须完成 B，不能分开两步。

**背景：** AI Agent 每次会话重新初始化，记忆不持续。当主动提出某项计划后，如果只发消息给用户而不写入持久化状态文件，系统重启后即遗忘。

**解决：** 所有承诺/计划 → 写入 todo.json → 系统自动追踪。

## 已知局限

- deadline 格式必须为 ISO（YYYY-MM-DD），格式错误会被 --check 跳过
- `status` 字段不影响过期判断（仅看 deadline）
- 无自动取消：过期项不会自动状态变更，需手动 --done 或 --remove
- 500+ 任务时考虑迁移 SQLite（当前为 JSON）

---

## 设计决策

| 问题 | 选择 | 原因 |
|------|------|------|
| 为什么用 JSON？ | 无外部依赖 | AI 可直接读写，无需数据库 |
| 为什么 deadline 不过期自动完成？ | 保留人工判断 | 有些过期任务仍有价值 |
| 为什么用 8 位 UUID？ | 够用 + 手动输入方便 | 太长不方便，太短易冲突 |
| system 标记的用途？ | 区分主动/被动任务 | 方便分类管理和优先跟进 |
