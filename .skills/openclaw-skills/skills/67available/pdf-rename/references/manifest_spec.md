# Manifest Schema

Generated by `extract.py`, optionally refined by `apply_verified.py`.

## Top-level

```json
[
  { /* entry */ },
  ...
]
```

## Entry Fields

| Field | Type | Description |
|-------|------|-------------|
| `filename` | string | Original filename (exact, as in folder) |
| `filepath` | string | Absolute path to the file |
| `title` | string\|null | Paper title (verified > pdf_text > filename) |
| `title_source` | string | One of: `verified`, `pdf_text`, `filename` |
| `year` | string\|null | Publication year (verified > filename > pdf_text) |
| `year_source` | string\|null | One of: `verified`, `filename`, `pdf_text` |
| `venue` | string\|null | Conference/journal abbreviation |
| `venue_source` | string\|null | `verified` or `pdf_text` |
| `status` | string | `ready` \| `needs_verification` \| `manual_review` |
| `notes` | string | Human-readable extraction notes |
| `is_duplicate` | bool | True if another entry has the same title |
| `duplicate_group` | string\|null | Indices of all duplicates, e.g. `"[3,7]"` |

## Status Rules

| Status | Meaning | Will rename? |
|--------|---------|-------------|
| `ready` | All fields verified | ✅ Yes |
| `needs_verification` | Auto-extracted, not yet verified | ❌ No |
| `manual_review` | Extraction failed or skipped | ❌ No |

## Output Filename Format

```
[Year] [Venue] Title.pdf
[Year] [Venue] Title (1).pdf   ← duplicate
[Year] [Venue] Title (2).pdf   ← duplicate
```
