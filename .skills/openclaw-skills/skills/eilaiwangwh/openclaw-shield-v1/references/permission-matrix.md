# 权限矩阵与来源判定

## 来源类型

- `OWNER_DIRECT`: 来自直接交互界面的 Owner 指令。
- `TAINTED_OWNER`: Owner 指令与当前污染上下文相关。
- `AGENT_AUTO`: Agent 自主决策。
- `EXTERNAL_DRIVEN`: 文件、网页、API、Webhook 等外部输入驱动。

## 来源判定流程

1. 先判定交互来源。
2. 再做注入检测，剥离零宽字符与可疑编码。
3. 最后做污染关联评估，必要时降级到 `TAINTED_OWNER` 或 `EXTERNAL_DRIVEN`。

## 权限策略基线

| 操作类型 | OWNER_DIRECT | TAINTED_OWNER | AGENT_AUTO | EXTERNAL_DRIVEN |
|---|---|---|---|---|
| 普通文件读写 | 放行 | 放行 | 放行 | 警告后放行 |
| 安装软件包 | 放行 | 提醒或确认 | 提醒后放行 | 拦截 |
| 删除文件 | 提醒 | 确认 | 确认 | 拦截 |
| rm -rf 类操作 | 二次确认 | 拦截 | 拦截 | 拦截 |
| sudo su 提权 | 提醒或口令确认 | 拦截 | 拦截 | 拦截 |
| 白名单网络访问 | 放行 | 放行 | 放行 | 警告后放行 |
| 非白名单网络访问 | 提醒或确认 | 确认 | 确认 | 拦截 |
| 读取凭证文件 | 提醒 | 拦截 | 拦截 | 拦截 |
| 输出凭证内容 | 强制脱敏 | 强制脱敏 | 强制脱敏 | 强制脱敏 |
| 关闭 Shield | 口令确认 | 拦截 | 拦截 | 拦截 |

## 会话信任建议

- 首次高危操作要求确认 Owner 身份。
- 出现注入尝试或加载外部内容后，撤销高危免确认状态。
- 长时间空闲后重置高危信任状态。
