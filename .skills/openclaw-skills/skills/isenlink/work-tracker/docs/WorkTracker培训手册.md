## 第一部分：WorkTracker系统概述

### 1.1 系统背景

**问题**：助手经常忘记报告工作进展，导致工作透明度低，管理困难。

**解决方案**：WorkTracker - 简单快速的工作日志系统

### 1.2 系统价值

- ✅ **提高透明度**：随时查看助手工作状态
- ✅ **建立纪律**：强制的工作记录习惯
- ✅ **便于管理**：集中查看团队工作进展
- ✅ **历史追溯**：完整的工作记录存档

### 1.3 系统特点

- **简单易用**：命令行操作，学习成本低
- **实时更新**：工作状态实时同步
- **自动备份**：数据自动备份，安全可靠
- **可扩展**：支持自定义助手和配置

## 第二部分：核心功能演示

### 2.1 基础命令

#### 开始工作

```bash
# 语法
worktracker start "助手名" "工作描述" "截止时间"

# 示例
worktracker start "小新" "修复宝塔面板SSL证书问题" "今天14:00前完成"
# 输出：✅ 小新已开始工作：修复宝塔面板SSL证书问题（截止：今天14:00前完成）
```

#### 更新进展

```bash
# 语法
worktracker update "助手名" 进度百分比 "更新内容"

# 示例
worktracker update "小新" 50 "已找到证书配置问题，正在重新生成证书"
# 输出：✅ 小新工作进展更新：50% - 已找到证书配置问题，正在重新生成证书
```

#### 完成工作

```bash
# 语法
worktracker complete "助手名" "完成结果" "后续事项"

# 示例
worktracker complete "小新" "成功修复SSL证书，宝塔面板可正常访问" "需要测试其他功能是否正常"
# 输出：✅ 小新工作完成：成功修复SSL证书，宝塔面板可正常访问（后续：需要测试其他功能是否正常）
```

### 2.2 查看命令

#### 查看状态

```bash
# 查看所有助手工作状态
worktracker status

# 输出示例：
# 📋 工作状态报告（2026-03-14 01:45）
# | 助手 | 当前工作                  | 进度 | 状态   | 开始时间 | 最后更新 | 截止时间           |
# |------|---------------------------|------|--------|----------|----------|--------------------|
# | 小新 | 修复SSL证书问题           | 100% | 已完成 | 01:40    | 01:45    | 今天14:00前完成    |
# | 小雅 | 市场分析报告撰写          | 30%  | 进行中 | 01:30    | 01:35    | 今天下班前         |
```

#### 查看日志

```bash
# 查看最近10条工作日志
worktracker log

# 查看最近20条工作日志
worktracker log --limit 20
```

### 2.3 配置命令

#### 添加助手

```bash
# 添加新助手
worktracker config add-assistant "新助手" "测试工程师" --email "test@example.com"
```

#### 查看配置

```bash
# 查看系统配置
worktracker config show
```

#### 导出数据

```bash
# 导出为JSON格式
worktracker export json --output work_data.json

# 导出为CSV格式
worktracker export csv --output work_data.csv
```

## 第三部分：使用规范要求

### 3.1 强制要求

#### 1. 必须记录的工作

- ✅ 所有超过10分钟的工作必须记录
- ✅ 重要决策和会议必须记录
- ✅ 系统配置变更必须记录
- ✅ 问题解决过程必须记录

#### 2. 更新频率要求

- ✅ 每30分钟至少更新一次进展
- ✅ 工作状态变化时必须立即更新
- ✅ 遇到问题或阻塞时必须更新

#### 3. 完成工作要求

- ✅ 完成工作后必须立即记录
- ✅ 必须填写完成结果和后续事项
- ✅ 重要工作完成后必须向清风总确认

### 3.2 工作描述规范

#### 好的工作描述

```bash
# 清晰、具体、可衡量
worktracker start "小雅" "完成Q1市场分析报告，包含3个竞品分析和5个市场趋势" "今天17:00前"
```

#### 不好的工作描述

```bash
# 模糊、不具体
worktracker start "小雅" "做市场分析" "尽快"
```

### 3.3 更新内容规范

#### 好的更新内容

```bash
# 具体、有信息量
worktracker update "小雅" 60 "已完成竞品分析部分，正在整理市场趋势数据，预计还需要1小时"
```

#### 不好的更新内容

```bash
# 模糊、无信息量
worktracker update "小雅" 60 "还在做"
```

### 3.4 完成结果规范

#### 好的完成结果

```bash
# 具体、包含成果
worktracker complete "小雅" "已完成15页市场分析报告，包含3个竞品详细分析和5个市场趋势预测" "需要清风总审阅并提供反馈"
```

#### 不好的完成结果

```bash
# 模糊、不完整
worktracker complete "小雅" "做完了" "无"
```

## 第四部分：常见问题解答

### 4.1 安装问题

#### Q1：命令找不到怎么办？

```bash
# 检查技能是否安装
openclaw skills list | grep worktracker

# 如果未安装，安装技能
skillhub install worktracker
```

#### Q2：权限错误怎么办？

```bash
# 检查文件权限
ls -la ~/.openclaw/workspace/.worktracker/

# 修复权限
chmod 755 ~/.openclaw/workspace/.worktracker/
chmod 644 ~/.openclaw/workspace/.worktracker/*.json
chmod 644 ~/.openclaw/workspace/.worktracker/*.md
```

### 4.2 使用问题

#### Q3：助手不存在怎么办？

```bash
# 添加助手
worktracker config add-assistant "助手名" "角色描述"

# 示例
worktracker config add-assistant "测试助手" "测试工程师"
```

#### Q4：工作已存在怎么办？

```bash
# 先完成或取消当前工作
worktracker complete "助手名" "取消当前工作" "重新开始新工作"

# 或等待当前工作完成
```

#### Q5：进度无效怎么办？

```bash
# 进度必须在0-100之间
# 正确示例
worktracker update "助手名" 50 "工作进展"

# 错误示例
worktracker update "助手名" 150 "工作进展"  # ❌ 进度超过100
```

### 4.3 数据问题

#### Q6：数据损坏怎么办？

```bash
# 从备份恢复
# 系统会自动备份，可以手动检查备份文件
ls -la ~/.openclaw/workspace/.worktracker/backups/
```

#### Q7：如何导出数据？

```bash
# 导出为JSON
worktracker export json --output work_export.json

# 导出为CSV
worktracker export csv --output work_export.csv
```

#### Q8：如何清理旧数据？

