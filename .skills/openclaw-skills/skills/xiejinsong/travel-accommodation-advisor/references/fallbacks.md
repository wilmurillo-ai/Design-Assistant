# Fallbacks

## Case 1：`search-poi` 无结果
- 放宽关键词：去掉修饰词，仅保留核心地标名。
- 调整类别：`地标建筑` -> `城市观光` / `展览馆` / 不传 `category`。
- 仍无结果时，直接执行：
  - `flyai search-hotels --dest-name "<城市>" --poi-name "<原始地标词>" --sort distance_asc`
  - 并标注“POI 未精准命中”。

## Case 2：`search-hotels` 无结果
- 放宽约束顺序：
  1. 去掉星级限制
  2. 提高预算上限（建议 +20%）
  3. 放宽半径（1km -> 3km -> 5km）
- 日期不确定时，先无日期探索，再补日期确认可订性。

## Case 3：候选过多，难以决策
- 固定输出 Top N（建议 5-8 家）。
- 先按主排序截断，再按次排序重排。
- 同分时优先：评分高 > 价格低 > 品牌稳定。

## Case 4：全部超预算
- 保持地标不变并双策略调整：
  - 扩圈（+2km）
  - 降星（5 -> 4，4 -> 3）
- 仍超预算时，明确提示“区域价格带较高”并给出最低可行价。

## Case 5：地标重名或跨区歧义
- 在结论中展示“地标确认信息”（城市、区域、类别）。
- 只保留与用户目标区域一致的候选，其余放备选附录。
