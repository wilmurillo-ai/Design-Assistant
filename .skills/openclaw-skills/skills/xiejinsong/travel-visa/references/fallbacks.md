# Fallbacks

## Case 1：签证政策结论不一致
- 使用同义查询重试，补充时态和身份条件：
  - `flyai fliggy-fast-search --query "<国籍> 去 <目的地> 旅游签证 最新 要求"`
  - `flyai fliggy-fast-search --query "<国籍> 去 <目的地> 电子签 落地签 条件"`
- 在输出中明确标注“需以官方渠道最终确认为准”。

## Case 2：`search-poi` 找不到签证中心
- 关键词降噪：`<国家名> 签证中心` -> `签证中心` -> `<国家名> 领事馆`。
- 更换办理城市重试，优先省会或一线城市。
- 仍无结果时，保留文字建议并标注“POI 未精准命中”。

## Case 3：多段行程转机规则不清晰
- 先用 `search-flight` 识别转机地，再对每个转机地执行单独核验：
  - `flyai fliggy-fast-search --query "<国籍> 经停 <转机地> 过境签 条件"`
- 输出按“主目的地签证”与“转机地过境要求”分栏展示。

## Case 4：时间过紧，存在来不及出签风险
- 先给出最小可行方案：电子签/免签目的地备选。
- 同步提供原目的地“最晚送签时间”与风险等级（高/中/低）。

## Case 5：用户仅给出模糊目的地
- 先做开放式澄清搜索：
  - `flyai fliggy-fast-search --query "<主题> 近期热门免签目的地"`
- 再收敛到 1-2 个候选目的地，进入标准签证核验流程。
