# Usage Patterns

## Pattern 1 — Daily Assessment Mode
Use when provider quality is uncertain or the system is still being validated.

This is also the recommended first mode for a conversational setup wizard.

Characteristics:
- daily reports
- provider freshness/status visibility
- no trusted signal assumption
- paper mode only
- approval-gated buy/sell workflow can remain enabled

## Pattern 2 — Trusted Paper Mode
Use after data quality is proven stable.

Characteristics:
- paper proposals become more meaningful
- still no real-money execution
- strategy review over time

## Pattern 3 — Live Review Candidate
Use only after strong paper evidence and explicit human approval.

Characteristics:
- strategy under structured review
- still should not auto-promote without explicit approval

## Core rule
Do not treat provider availability as equivalent to provider quality.
Freshness and reliability must be validated before trusting signals.
