# Online Image Search Workflow

在用户明确提出找图片、找参考图、找素材、找 moodboard 图、找美术灵感图时，按这套流程执行。

如果当前任务本身就是“前期调研 + 灵感收集”类型，即使用户没有再单独补一句“给我参考图”，也默认进入这套流程。

## 默认站点范围

默认且仅使用下面这些设计站点：

- Behance
- 站酷
- 花瓣
- 小红书

除非用户明确允许扩展来源，否则不要主动扩到其他站点。
每次搜索都必须逐站检查 `Behance`、`站酷`、`花瓣`、`小红书`，不能默默漏掉其中任何一个。

## 先判断需求类型

先把请求归类成下面四类之一：

### 1. 灵感参考图

用途：

- 看风格方向
- 看构图和氛围
- 看材质和叙事感

要求：

- 重在代表性和差异性
- 不要求可直接商用
- 必须保留来源链接
- 优先选择 Behance、站酷、花瓣上的原始项目页、收藏页，以及小红书上的高质量参考笔记

### 2. Moodboard 图

用途：

- 做方向板
- 做视觉氛围拼贴
- 对齐风格语言

要求：

- 可以混合品牌图、案例图、摄影图、版式图
- 优先保证风格覆盖度
- 必须标清来源，避免误认为原创资产
- 优先从 Behance、站酷、花瓣、小红书收集

### 3. 美术灵感图

用途：

- 找画风参考
- 找色彩气质
- 找笔触和材质感
- 找故事感和氛围感

要求：

- 优先找插画、绘本风、海报风、概念视觉、艺术化项目页
- 少给产品截图和纯 UI 页面
- 优先保留能直接体现美术质量的项目页、采集页或优质灵感笔记
- 可结合 `references/art-inspiration-search.md` 的关键词策略

### 4. 可商用素材线索

用途：

- 后续可能进入正式设计稿
- 需要进一步确认授权和下载方式

要求：

- 在 Behance、站酷、花瓣、小红书上更多把它们当作线索页、作者页或整理笔记，而不是默认可直接商用素材库
- 优先选择明确提供授权信息或跳转到原始素材来源的页面
- 不要仅根据图片搜索结果就断言可商用
- 应返回素材线索和来源页，不要擅自把授权说死

## 搜索策略

### 关键词构成

每次搜索至少包含：

- 主题词
- 风格词
- 场景词
- 媒介词
- 站点词

例如：

- 教育 插画 专题页 Behance
- 绘本风 海报 花瓣
- 儿童 教育 视觉 站酷
- editorial illustration education Behance
- 国际绘本 参考 小红书

### 搜索原则

- 先按方向分别搜，不要一把梭只搜一个总词
- 对 `Behance`、`站酷`、`花瓣`、`小红书` 分别执行搜索，不要合并后默认某站已覆盖
- 每个方向优先找 3 到 6 个代表结果
- 结果之间要有区分度
- 优先保留原始项目页、原始作品页、原始收藏页、优质整理笔记，而不是二次搬运页
- 不要用官方品牌页、出版社页、新闻页替代设计站结果
- 候选结果必须经过 `references/quality-gate.md` 评估后再决定保留
- 即使某站没有强结果，也要在输出中明确写出该站的覆盖结论
- 如果 Behance、站酷、花瓣、小红书结果不足，要先明确说明不足，不要自动扩站

## 输出要求

返回图片相关内容时，至少包含：

- 来源覆盖情况
- 图片或来源标题
- 来源站点
- 链接
- 推荐原因
- 适用方向
- 用途类型
- 质量判断

如果这是一次完整的前期研究交付，图片相关结果应作为默认输出的一部分，而不是可有可无的附录。

如果当前环境无法联网，也不能直接跳过这一部分；应返回四站关键词包、搜图优先级和筛选标准，让后续流程能继续。

如果用户明确说“给我链接”“返回链接”“要能点进去看”，就优先返回可访问的来源页链接，而不是只写搜索关键词或抽象描述。

如果某站点没有拿到强结果，也必须明确写出该站的覆盖状态，不要用其他站点结果替代。

对外展示时，优先保留少量强链接，不要返回一串没有解释的裸 URL。

## 风险提醒

- Behance、站酷、花瓣、小红书上的图，多数默认适合作为参考，不等于自动可商用
- 灵感图不等于可商用素材
- moodboard 图不等于可以直接进正式设计稿
- 收藏页、搬运页、聚合页要谨慎使用
- 小红书内容更适合作为灵感收集和趋势线索，不应默认视为原创作品源头
- 如果用户真正要下载和使用素材，必须回到原始来源页确认授权

## 无法联网时的降级方案

如果当前环境无法实际搜图，就返回：

- 面向 Behance、站酷、花瓣、小红书的搜图关键词包
- 推荐先搜哪个站点以及原因
- 每个方向期望收集张数
- 筛选标准
- 质量评估标准
- 四站覆盖检查清单

让用户或后续可联网流程继续执行。
