# 中文输出模板

返回结果时优先使用下面的结构，目标是让设计团队能直接接着工作。

## 1. 需求翻译

用 2 到 4 句话说明：

- 这个需求本质上想解决什么问题
- 当前是产品升级、活动需求、内容包装，还是视觉探索
- 本轮前期工作最需要先回答什么

## 2. 需求校准

先写出你对当前 brief 的判断：

- 当前输入是否足够支撑方向判断
- 哪些点已经明确
- 哪些点还需要校准

如果 brief 不够清楚，再补 1 到 3 个高影响问题。不要机械罗列问题。

## 3. 已知 / 未知 / 假设 / 风险

分成四组：

### 已知

- 已明确的信息

### 未知

- 还没确认、但会影响方向判断的信息

### 假设

- 为了先推进本轮研究而采用的工作假设

### 风险

- 当前最可能导致返工或方向跑偏的风险

## 4. 关键调研问题

给出 5 到 8 条问题，覆盖：

- 用户和使用场景
- 业务目标和转化目标
- 内容组织
- 视觉调性
- 同类参考

## 5. 方向探索

给 3 个方向，每个方向都包含：

- 方向名称
- 核心关键词
- 适合借鉴的参考类型
- 适用场景
- 风险提醒

方向之间要有明显差异，不要只是同一风格换词。

## 6. 建议收集的参考

按类型列出：

- 竞品参考
- 跨行业参考
- 版式参考
- 动效参考
- 材质或氛围参考

每类说明为什么要收。

## 7. 来源覆盖情况

只要本轮包含灵感参考，就优先加这一段。

必须分别写出：

- Behance：找到强结果 / 只有弱结果 / 没有可用结果 / 当前环境无法联网
- 站酷：找到强结果 / 只有弱结果 / 没有可用结果 / 当前环境无法联网
- 花瓣：找到强结果 / 只有弱结果 / 没有可用结果 / 当前环境无法联网
- 小红书：找到强结果 / 只有弱结果 / 没有可用结果 / 当前环境无法联网

## 8. 灵感图 / 参考链接清单

默认应该加这一段，除非用户明确说“只做分析不要参考”。

每项至少包含：

- 名称或简述
- 来源站点
- 链接或搜图关键词
- 推荐原因
- 用途说明
- 质量判断

## 9. 本轮质量判断

简要说明：

- 结果整体是否可继续用
- 有没有主动删掉弱结果
- 当前是否还存在样本不足或风格重复风险
- 是否完成了需求校准
- 如果做了参考收集，四站覆盖是否完整

## 10. 建议下一步

只选一个最合适的动作：

- 先补需求澄清
- 先做 moodboard
- 先做低保真结构
- 先继续细分美术方向

并说明理由。
