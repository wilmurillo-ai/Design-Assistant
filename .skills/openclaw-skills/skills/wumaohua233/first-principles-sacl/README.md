# First Principles Thinking (第一性原理思维)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

基于 SACL 方法论的系统化思维框架：剥离层 → 基元化 → 约束映射 → 杠杆重构

## 什么是 SACL？

```
问题输入 (Problem)
    ↓
[S] 剥离层 (Strip) → 去除类比/惯例/历史路径
    ↓
[A] 基元化 (Atomize) → 拆解到不可再分的物理/逻辑单元
    ↓
[C] 约束映射 (Constraint) → 区分硬约束 vs 软约束
    ↓
[L] 杠杆重构 (Leverage) → 用现有资源重新组合基元
    ↓
行动输出 (Action)
```

## 适用场景

- 资源有限但需要破局（没钱、没时间、没技能）
- 复杂问题不知道如何拆解
- 被行业惯例限制思维
- 寻找不对称优势
- 第一性原理分析

## 四大工具卡

### 1. Strip (剥离层) — 零点思维

**核心问题**:
- 如果我是刚着陆地球的外星人，没有现有方案的知识，如何描述这个问题？
- 行业术语背后物理上发生了什么？
- 哪些是遗迹（历史遗留）vs 问题的本质？

**工具**: 禁忌清单 —— 列出行业中"每个人都必须做"的事，然后逐一划掉问："不做会怎样？"

### 2. Atomize (基元化) — 找到乐高积木

**基元标准**:
- 不能被其他单元组合替代
- 有清晰的数学/物理/逻辑边界

**快速分解公式**:
```
输入 [X] → 通过 [Y] → 转化为 [Z]，消耗 [资源 W]
```

### 3. Constraint (约束映射) — 承认现实

**约束分类**:

| 硬约束 (物理/逻辑) | 软约束 (资源/惯例) | 伪约束 (认知误区) |
|-------------------|-------------------|------------------|
| 时间不可逆 | 资金有限 | "必须自建流量" |
| 注意力有限 | 单一技能 | "必须用专业 PR" |
| 光速/带宽限制 | 地理位置 | "必须有权权威才能做 SEO" |

### 4. Leverage (杠杆重构) — 重新组装

**四种杠杆模式**:

1. **平台寄生** — 不建平台，只连接，用高权威平台作为你的"基元"
2. **时间套利** — 用你富余的时间换取他人缺乏的即时性
3. **基元替代** — 目标需要基元 A，但你有 B，B 能转化为 A 的输入吗？
4. **约束翻转** — 你的约束也是他人的约束，在此约束下建立专长形成壁垒

## 使用示例

### 零预算 SEO 问题

**S (剥离)**: 划掉"SEO = 在自己网站发内容"，划掉"推广 = 买广告"
**A (基元)**: 需求者 = 搜索关键词，相遇单元 = 信任传递，你的资产 = 代码生成的工具
**C (约束)**: 硬约束 = Google 沙盒期，软约束 = 零预算 → 必须非货币交换获得关注
**L (杠杆)**: 
- 平台寄生：在 Reddit/Zhihu/ Dev.to 发内容
- 原子替代：用代码换反向链接替代买链接
- 开源工具：GitHub 获得自然传播

## 核心哲学

> 第一性原理不会让你全知全能 —— 它让你成为特定约束下的最优设计者。

你的局限（"只会写文章和做小工具"）不是缺陷 —— 它们是精确定义的设计参数。用这些参数，你可以设计出大公司无法复制的解决方案。
