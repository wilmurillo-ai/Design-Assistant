# Framework Reference

## Purpose

This skill gives agents a reusable source-research framework that works across workspaces.

It should help agents:
- discover and record source pools;
- avoid repeating rejected analysis;
- allow manual collection when automation does not exist yet;
- gradually turn repeated valuable work into reusable methods and automation.

## Shared mental model

### Three result layers
1. Source pools
2. Acquisition methods
3. Filtered high-quality source results

### Four execution stages
1. Record source pools
2. Research acquisition/filtering methods
3. Produce high-quality source results
4. Automate monitoring and output

These four stages are **parallelizable** and **non-linear**.

## Default daily logic

- New pool discovered during unrelated work -> record first, research later if needed.
- Historical pool -> judge whether it is worth continued investment.
- Rejected pool -> leave explicit rejection conclusion.
- Valuable but manual pool -> manual usage is allowed now.
- Repeatedly useful pool -> raise priority for methodology, engineering, and automation.

## Workspace mapping rule

If the workspace already has a canonical source-research structure, use it.
If not, create a simple canonical structure under something like:
- `docs/source-research/`

and keep these categories separated:
- source pools;
- acquisition/filtering methods;
- filtered results;
- rejection conclusions;
- automation/engineering assets.
