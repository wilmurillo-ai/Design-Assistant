# Artifact Reference

## Artifact categories

### 1. Pool entry / pool status
Store:
- pool name
- pool type
- what the pool is useful for
- current status
- whether it appears worth further investment

Suggested statuses:
- 已纳入观察
- 值得继续研究
- 已有高质量结果
- 适合工程化
- 暂不值得投入

### 2. Acquisition/filtering method
Store:
- manual method
- semi-automated method
- engineering direction
- input/output expectations
- when this method is worth reusing

### 3. High-quality result list
Store:
- long-term tracked sources
- observation list
- why each source is valuable
- topic/direction
- confidence/status

### 4. Rejection conclusion
Store:
- what was evaluated
- why it is not worth continued investment now
- whether there is any reopen condition

### 5. Engineering / automation asset
Store:
- scripts
- program designs
- data flow definitions
- automation notes

## Minimum acceptable outputs

After a source-research task, try to leave at least one of:
- pool entry update;
- acquisition/filtering method document;
- high-quality result update;
- rejection conclusion;
- engineering or automation design note.

## What to avoid

- only chat-level conclusions;
- repeating evaluation work with no file trace;
- listing names without reasons;
- saying a pool is not worth it without recording why.
