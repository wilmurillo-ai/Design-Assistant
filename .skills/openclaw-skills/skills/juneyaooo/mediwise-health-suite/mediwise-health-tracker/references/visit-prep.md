## 对外介绍话术

当用户只是泛泛地问“你能做什么”时，可以顺手补一句：

```text
如果你最近准备去看医生，我还可以根据你这次的不舒服描述，再结合历史记录，先帮你整理一版简短的就医摘要；如果你需要，我再继续帮你整理成图片或 PDF，方便给医生直接看。
```

这样更容易让用户意识到这个功能的价值。

## 适用场景

当用户最近准备去门诊、复诊、看专科时，可以先让用户用自己的话描述本次不适，再生成一段“就医前摘要”。

适合的说法包括：

- “我最近想去看医生，帮我整理一下情况”
- “我先描述一下，你帮我总结重点”
- “把我最近的病情、病史、过敏和用药整理一下”
- “帮我做一张给医生看的摘要图”

## 推荐流程

默认优先：**先生成一段短文摘要**。

短文发给用户后，顺手补一句：

```text
如果你愿意，我也可以继续把这份内容整理成图片或 PDF，方便就诊时直接出示给医生。
```

只有在用户明确说“要图片”“做成图”“导出 PDF”“给医生直接看”时，再继续生成图片版或 PDF 版。

### 推荐追问话术

生成完短文摘要后，可以按语气自然选择一句：

```text
如果你愿意，我也可以继续把这份摘要整理成图片，方便你到医院时直接给医生看。
```

```text
这版短文你先看看；如果要更方便出示给医生，我可以再帮你排成图片或 PDF。
```

```text
要不要我顺手再帮你整理成一张图，或者导出成 PDF？这样就诊时会更直观。
```

如果用户已经明确说“我要给医生看”“我要打印”“我要发给家人或医生”，优先直接问：

```text
我可以直接帮你整理成图片或 PDF，你更想要哪一种？
```

## 生成方式

### 先生成短文摘要

```bash
python3 {baseDir}/scripts/doctor_visit_report.py text --member-id <id> --description "最近两周反复头晕，起床和翻身时更明显，偶尔恶心"
```

### 生成图片版

```bash
python3 {baseDir}/scripts/doctor_visit_report.py screenshot --member-id <id> --description "最近两周反复头晕，起床和翻身时更明显，偶尔恶心"
```

### 生成 PDF 版

```bash
python3 {baseDir}/scripts/doctor_visit_report.py pdf --member-id <id> --description "最近两周反复头晕，起床和翻身时更明显，偶尔恶心"
```

### 只生成 HTML

```bash
python3 {baseDir}/scripts/doctor_visit_report.py generate --member-id <id> --description "最近两周反复头晕，起床和翻身时更明显，偶尔恶心"
```

### 调试结构化数据

```bash
python3 {baseDir}/scripts/doctor_visit_report.py data --member-id <id> --description "最近两周反复头晕，起床和翻身时更明显，偶尔恶心"
```

## 图文里包含的信息

- 本次主诉与自动提取的重点
- 近期关键指标、异常提醒、最近症状记录
- 最近门诊 / 住院变化
- 相关既往史与近期相关检查
- 当前在用药
- 已知过敏史
- 当前在用药之间可识别的中高风险药物相互作用
- 就诊时建议主动告诉医生的重点

## 使用建议

- 让用户先用自然语言把这次不舒服讲清楚，越具体越好。
- 默认先给短文摘要，更符合聊天习惯。
- 用户需要时再导出图片版或 PDF 版，便于直接展示给医生。
- 如果摘要里缺少重要信息，再补录入后重新生成一次。
- 这份内容是辅助沟通材料，不替代正式病历和医生判断。
