# Client Rules — 跟进规则（客户版）

## 状态机

```
idle → [send-checkin] → sent → [7天未回复] → reminded → [仍未回复] → skipped → idle（下月）
                          ↓                      ↓
                   [collect-reply]         [collect-reply]
                          ↓                      ↓
                        idle（下月）           idle（下月）

特殊状态：
  任何状态 → [红线检测命中] → flagged → 等待规划师处理 → idle
```

## 幂等规则

| 模式 | 前置 state | 行为 |
|------|-----------|------|
| send-checkin | idle | 正常发送 |
| send-checkin | sent/reminded/completed/skipped | 拒绝，提示"本月已发送" |
| collect-reply | sent/reminded | 正常执行 |
| collect-reply | idle/skipped | 拒绝，提示状态不匹配 |

## 高风险红线关键词

检测家长反馈中是否包含以下信号（不需要精确匹配，语义相关即触发）：

### 立即通知规划师：
- 退行信号：之前会的现在不会了、倒退、退步明显
- 情绪极端：持续暴怒、打人咬人加剧、自伤、不吃不睡
- 安全事件：校园霸凌、受伤、走失
- 家庭变故：离婚、丧亲、搬家、重大经济变化
- 医疗信号：确诊、就医、发育迟缓、医生建议

### 处理方式：
1. 不做任何分析或建议
2. state → flagged
3. 立即发消息给规划师，附上家长原文
4. 回复家长："收到你的反馈，我已经第一时间转给规划师，会尽快给你回复。"

{整理后的反馈，保持原意，分条列出}
──────────────────

```

🚨 {孩子姓名} {YYYY年M月} 月度反馈（⚠️ 红线触发）
──────────────────
触发原因：{红线类型}

plan_id：{plan_id}
──────────────────
【家长反馈原文】
{完整原文，不做任何删减}
──────────────────
⚠️ 请规划师优先处理
```

## 闭环终止条件

| 条件 | 动作 |
|------|------|
| followup_count ≥ 12 | 提醒家长联系规划师评估续期 |
| 连续 3 月 skipped | 通知规划师 |
| 孩子超龄 | 提醒家长联系规划师评估新阶段 |
