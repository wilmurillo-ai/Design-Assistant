# Deep Reader — 深度阅读技能

本技能依据《如何阅读一本书》中介绍的阅读方法论，对用户提供的一本书进行深度阅读和分析。

## 核心阅读层次

深度阅读遵循四个递进层次：

### 第一层：基础阅读
识别书籍基本信息：书名、作者、译者、出版社、出版时间、ISBN，确认文本可读。

### 第二层：检视阅读（Inspectional Reading）
- **有系统略读**：快速扫描目录、序言、章节标题，判断书籍类型与主题范围
- **粗浅通读**：从头到尾快速读一遍，遇到难点不纠结，先建立整体印象
- **判断价值**：这本书值得深入读吗？适合哪种阅读方式？

### 第三层：分析阅读（Analytical Reading）
这是深度阅读的核心，需要系统执行以下规则：

**规则一：书籍分类**
首先判断书籍类型：
- 虚构作品（小说、戏剧、诗歌）
- 论说性作品（实用型 vs 理论型）
  - 理论型细分：历史、哲学、科学、数学、社会科学
- 实用性作品：行动导向，关注"应该怎么做"

**规则二：透视骨架**
- 用一到两句话概括整本书在谈什么
- 列出全书重要章节及组织架构
- 找出作者想要解决的核心问题或意图

**规则三：与作者共通的词义**
- 找出书中的关键概念词（专业术语、反复出现的关键词）
- 与作者就核心词义达成理解共识

**规则四：判断作者主旨**
- 找出书中关键句（包含核心观点的句子）
- 用自己的话复述作者的主旨
- 找出作者论述的关键论点与论证过程

**规则五：公正评价**
- 理解之前不轻易批评
- 批评时要有理有据，指出具体问题（知识不足/知识错误/不合逻辑/分析不完整）

### 第四层：主题阅读（Syntopical Reading）
如涉及多本书对比分析，超越单一书籍，站在更高视角比较不同作者对同一主题的观点。

### 触发词判断逻辑

通过用户请求中的关键词，判断应使用哪种阅读层次，以及分析重点：

**触发检视阅读的关键词**：
"这本书讲了什么"、"主要内容"、"核心观点"、"大概讲什么"、"简要介绍"、"主要框架"、"主题是什么"、"值不值得读"

**触发分析阅读的关键词**：
"详细分析"、"深入分析"、"论证过程"、"论据"、"观点分析"、"如何论证的"、"核心论点"、"作者的主旨"、"批判性分析"

**触发主题阅读的关键词**：
"多本书"、"对比分析"、"围绕XX主题"、"比较这几本书"、"主题阅读"、"综合分析"

**精细化问题类型映射**（用于调整分析阅读的侧重点）：

| 用户问题类型 | 分析侧重点 |
|---|---|
| "这本书在谈什么" | 透视骨架（规则二、三），聚焦主题概括 |
| "作者为什么这么说" | 论证过程分析，追踪逻辑链条 |
| "我不同意作者" | 触发批评性分析，引导用户具体说明不同意哪一点 |
| "这本书对我有什么用" | 增加启示模块，联系读者实际 |
| "作者的知识/观点是否有依据" | 重点分析证据、引文、论证前提 |

> 如果用户明确指定了阅读层次，按用户要求执行。如果不明确，按分析阅读默认处理。

### 第一步：加载书籍内容

1. 识别书籍格式：
   - **EPUB**：解压 → 提取 `OPS/chapter-*.xhtml` 文件 → 用 `sed 's/<[^>]*>//g'` 去除 HTML 标签
   - **PDF**：使用 `pdf` 技能提取文本
   - **TXT**：直接读取

2. 提取书籍结构：
   - 读取目录文件（`navigation.xhtml` / `toc.ncx` / 目录章节）
   - 识别章节顺序与主题分组

3. 提取关键内容：
   - 序言/前言：了解作者写作动机与核心主张
   - 第一章/最后一章：往往包含最核心的框架
   - 选取代表性章节深度分析（不必全读，但要有代表性）

### 第二步：判断用户需求层次

根据用户的具体需求，选择对应的阅读层次：

#### 需求A：只想了解书籍的核心观点和大致内容
→ **使用检视阅读**

执行要点：
- 有系统略读：快速扫描目录、序言、章节标题，判断书籍类型
- 用简短句子概括整本书在谈什么
- 列出书籍的结构大纲（主要章节及逻辑关系）
- 指出作者的核心主题或意图
- **输出：结构笔记**

#### 需求B：想深入了解书籍的观点、论据和论证过程
→ **使用分析阅读**

执行要点：
1. 对书籍进行完整分析阅读（15个规则分三阶段执行）
2. 找出关键概念词，与作者就词义达成共识
3. 提取关键句，分析作者的核心论点与论证链条
4. 结合原文引用进行深度分析
5. 根据书籍类型调整分析侧重点（见下节"书籍类型适配"）
- **输出：概念笔记**

### 书籍类型适配模块

检测到书籍类型后，自动调整分析侧重点：

| 书籍类型 | 分析侧重点 | 评价标准 |
|---|---|---|
| **虚构作品**（小说、戏剧、诗歌） | 引导用户关注**体验和感受**，而非论点；关注情节、人物、场景背后的意涵 | 不要用论说性作品标准批评虚构作品；先感受，再评价 |
| **实用型书** | 引导用户问"我愿意照做吗？""作者的目的是否达成？" | 不争论观点是否符合事实，争论是否有效达到目的 |
| **哲学书** | 引导用户思考**问题本身**，而非记住结论；用自己的经验来验证哲学家主张 | 没有标准答案，要主动思考作者提出的问题 |
| **历史书** | 警惕作者的选择性，关注"为什么选这些事实""这些事件对我意味着什么" | 关注史实的选择与诠释，关注作者立场 |
| **科学/数学书** | 找出作者研究的核心问题；关注公理、假设与推导过程 | 可以跳过专业推导，但要理解核心发现 |
| **社会科学书** | 注意关键词模糊问题，不同作者用法不同；必要时进行主题阅读 | 通常需要广泛阅读相关书籍才能真正理解 |

> 基础阅读和检视阅读阶段不区分书籍类型，统一流程。书籍类型适配仅在分析阅读阶段生效。

#### 需求C：用户要求深入分析多本书
→ **使用主题阅读**

这是最复杂的场景，必须按以下步骤执行：

**阶段一：对每本书分别进行检视阅读**
- 快速提取每本书的核心主题、结构大纲、所属类型
- 找出每本书与主题相关的章节
- 制作每本书的结构笔记

**阶段二：提取多本书的共同部分（四步细化）**
1. **建立主题词汇表**：各书对同一核心概念的定义是否一致？是否有歧义？
2. **找共识点**：哪些观点多方一致认可？（可能存在共同前提或相近证据）
3. **找争议点**：分歧在哪里？根源是什么？
   - 前提不同（各自的假设不同）
   - 证据不同（引用的事实或研究不同）
   - 价值观不同（对同一事实的解读不同）
4. **形成自己的立场**：基于对各方观点的理解，明确站哪一边，并说明理由

**阶段三：对相关章节分别进行分析阅读**
- 对每本书中与主题相关的章节，提取关键论点
- 追踪每位作者的论证逻辑
- 记录每位作者对该主题的立场

**阶段四：综合输出辩证笔记**
- 以**主题为中心**而非以某一本书为中心进行综合论述
- 用自己的语言整理各方观点
- 建立完整的分析框架，指出共识与分歧
- 必要时给出自己的评价与立场

---

**如果用户没有具体说明阅读层次**（默认模式），则按以下三个核心问题展开（对应分析阅读深度）：

> **问题一：这本书在谈什么？**
> - 用一段话概括整本书的主题
> - 列出书籍的结构大纲（主要章节及逻辑关系）
> - 指出作者的核心命题或意图

> **问题二：作者详细说了什么？怎么说的？**
> - 书中最重要的几个观点是什么？
> - 每个观点是如何论证的（举例、引用、推理）？
> - 作者使用了什么写作手法/结构安排？

> **问题三：这本书说得有道理吗？是全部还是部分？**
> - 哪些观点令人信服？依据是什么？
> - 哪些观点可能存在争议或局限？
> - 这本书对你有什么意义或启发？

### 做笔记引导模块

分析阅读报告完成后，引导用户自己做笔记，而不只是消费报告：

**教用户做笔记的具体方法**（可在报告末尾附上）：
- **画底线**：标出最重要的论点
- **编号**：标记论证的顺序（1、2、3……）
- **边注提问**：在不理解的地方写下问题
- **做索引**：在相关论点之间画箭头或注明"cf.第X页"
- **写下评论**：在空白处写下"我不同意……因为……"

**主动引导用户思考**：
- 报告完成后，询问："你是否同意作者在第X章的论证？"
- 报告完成后，询问："尝试用你自己的话概括第X节的核心论点"
- 引导用户将笔记与自身经验连接："你能举一个符合/不符合这个观点的例子吗？"

---

### "受教"心态模块

内化为技能回答的态度准则：

**回答原则**：
- 分析一本书时，先完整理解（复述作者观点），再表达质疑
- 遇到不熟悉的领域，先承认局限，不轻易否定
- 引导用户同理：先问"作者在说什么？"，再问"他说得有道理吗？"

**批评的四条标准**（供引导用户使用）：
1. 作者的**知识不足**：是否缺少与主题相关的重要知识？
2. 作者的**知识错误**：是否有明显与事实相反的论点？
3. 作者**不合逻辑**：推论是否无法令人信服？
4. 作者**分析不完整**：是否还有重要方面未被考虑？

---

### 阅读习惯养成模块

分析阅读完成后，提供一个"刻意练习"建议：

**推荐格式**：
> "下次读类似主题的书时，建议尝试：先回答'这本书在谈什么'，再开始阅读正文。"

**根据书籍类型给出不同建议**：
- 虚构作品：下次读小说时，关注人物的行为动机而非情节走向
- 实用型书：下次读方法论书籍时，问自己"我愿意照做哪一点？"
- 哲学书：下次读哲学论述时，关注"问题是什么"而非"答案是什么"

### 第三步：输出格式

根据用户需求选择对应输出格式：

#### 检视阅读输出（需求A — 只想了解核心观点）

```
# 《书名》检视阅读报告

## 基本信息
- 作者：
- 类型：
- 核心主题：

## 书籍结构
[主要章节及逻辑关系]

## 核心主题概括
[一两句话概括整本书在谈什么]

## 主题架构
[作者如何组织内容，大纲]
```

#### 分析阅读输出（需求B — 深入分析论证过程）

```
# 《书名》深度阅读报告

## 基本信息
- 作者：
- 类型：
- 核心主题：

## 问题一：这本书在谈什么？
[详细回答]

## 问题二：作者详细说了什么？怎么说的？
### 核心观点1
[原文摘录 + 分析]
### 核心观点2
[原文摘录 + 分析]
……

## 问题三：这本书说得有道理吗？
[批判性分析，包含支持与质疑]
```

#### 主题阅读输出（需求C — 多本书比较分析）

```
# 主题：《XX》主题阅读报告

## 参与讨论的书籍
[列出所有参与主题阅读的书籍]

## 主题词汇表
[各书共同使用的关键概念及定义共识/分歧]

## 各方观点梳理
### 观点A（来自《书名1》）
[该书对此主题的核心立场]

### 观点B（来自《书名2》）
[该书对此主题的核心立场]

……

## 综合分析
[站在更高视角，综合各方观点，形成自己的分析框架]

## 共识与争议
[哪些观点各方一致认可？哪些存在根本分歧？]

## 辩证笔记
[用你自己的语言，综合讨论这一主题，引用各方观点]
```

### 第四步：引用原文

深度阅读必须有原文依据：
- 关键论点必须引用原文
- 使用块引用格式 `> ` 标注
- 注明章节位置

## 注意事项

- EPUB 提取：章节文件编号可能不连续（如 chapter-1, 2, 3, 8, 12...），需全部列出后排序
- 遇到生僻字用 `[生僻字]` 占位，不中断流程
- 如书籍内容过长，优先读取核心章节（序言、目录、第一章、最后一章、索引）
- 实用型书籍评价标准：关注作者目的是否达成，而非观点是否同意
- 虚构作品评价标准：感受体验优先，批评其次
