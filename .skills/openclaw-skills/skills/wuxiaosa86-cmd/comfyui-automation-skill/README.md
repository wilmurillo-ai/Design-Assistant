# ComfyUI Automation Skill

## 技能介绍

ComfyUI Automation Skill 是一个 OpenClaw 技能，用于自动化执行 ComfyUI 工作流。它能够根据工作流需求收集必要的素材，执行工作流，并监控执行状态获取结果。

## 功能特性

- **工作流识别**：支持通过工作流ID或工作流昵称识别工作流
- **素材收集**：根据工作流需求自动收集必要的素材文件
- **素材完整性检查**：确保所有必要的素材都已收集完整
- **执行确认**：在执行工作流前请求用户确认
- **工作流执行**：使用 ComfyUI 任务1-简易接口执行工作流
- **状态监控**：监控工作流执行状态并获取结果
- **新工作流创建**：当遇到不存在的工作流ID时，可以创建新的工作流
- **工作流分析**：自动分析工作流JSON，识别可修改节点
- **节点选择**：用户可以选择需要配置的节点
- **图片加载**：支持本地文件上传和网络URL加载

## 安装方法

1. 确保已安装 OpenClaw
2. 将技能目录 `comfyui automation skill` 复制到 OpenClaw 的技能目录中
3. 技能目录通常位于：
   - Windows: `%USERPROFILE%\.openclaw\workspace\skills\`
   - macOS/Linux: `~/.openclaw/workspace/skills/`
4. 安装依赖：
   ```bash
   pip install requests
   ```

## 使用方法

1. 在 OpenClaw 中调用技能
2. 提供工作流标识信息（workflowId 或工作流昵称）
3. 根据提示提供必要的素材文件
4. 确认执行工作流
5. 等待工作流执行完成并查看结果

## 执行步骤

1. **技能启动阶段**：接收用户提供的工作流标识信息
2. **工作流分析阶段**：自动获取工作流JSON结构，分析需要用户输入的节点
3. **节点选择阶段**：
   - 显示所有识别到的节点及其当前值
   - 用户可以选择需要配置哪些节点
   - 支持输入节点编号（如 "1 3 5"）或输入 "all" 选择所有节点
   - 也可以不选择任何节点，直接使用工作流默认值
4. **智能参数收集阶段**：
   - 对选中的节点进行配置
   - 自动识别 LoadImage 节点，支持本地文件上传或URL输入
   - 自动识别 CLIPTextEncode 节点，收集文本提示词
   - 自动识别 Seed 节点，收集随机种子
   - 支持保留工作流中的默认值或使用新值
5. **素材完整性检查**：验证所有必要的素材是否已收集完整。如果素材不完整，必须向用户索取完整素材，直到所有必要素材都收集完毕
6. **执行确认阶段**：请求用户确认是否执行运行操作
7. **工作流执行阶段**：
   - 如有自定义参数，使用"ComfyUI任务2-高级"接口执行工作流
   - 如无自定义参数，使用"ComfyUI任务1-简易"接口执行工作流
8. **结果反馈阶段**：监控执行状态，获取返回结果并反馈给用户

## 图片加载方式

技能支持三种图片加载方式：

1. **本地文件上传**：
   - 用户提供本地图片文件路径
   - 技能自动将文件上传到 RunningHub 平台
   - 使用返回的文件名在工作流中加载

2. **网络图片URL**：
   - 用户直接提供网络图片的URL
   - 技能在工作流中直接使用该URL

3. **使用工作流默认值**：
   - 保留工作流 JSON 中已配置的图片文件名
   - 适用于图片已预先上传到平台的情况

## 支持的节点类型

技能会自动识别并处理以下类型的节点：

| 节点类型 | 说明 | 输入方式 |
|---------|------|---------|
| LoadImage | 加载图片节点 | 本地文件路径或URL |
| CLIPTextEncode | CLIP文本编码器 | 文本提示词 |
| CLIPTextEncode (NSP) | CLIP文本编码器(NSP) | 文本提示词 |
| Seed (rgthree) | 随机种子节点 | 数字 |
| RH_Translator | RunningHub翻译器 | 文本提示词 |
| KSampler | K采样器 | seed, steps, cfg, denoise |
| easy sam3ImageSegmentation | 图像分割 | threshold |
| ControlNetInpaintingAliMamaApply | ControlNet应用 | strength |
| ModelSamplingAuraFlow | 模型采样 | shift |

## 工作流操作指南

### 如何在 RunningHub 平台添加新工作流

1. **访问 RunningHub 平台**
   - 打开浏览器访问 [RunningHub](https://www.runninghub.cn)
   - 登录你的账户

2. **创建或上传 ComfyUI 工作流**
   - 进入"ComfyUI 工作流"页面
   - 点击"创建工作流"或"导入工作流"按钮
   - 如果是导入工作流：上传你的 ComfyUI 工作流 JSON 文件
   - 如果是创建工作流：在在线编辑器中构建你的工作流

3. **保存工作流并获取 Workflow ID**
   - 配置好工作流后，点击保存
   - 在工作流详情页或列表中找到该工作流
   - 复制工作流的 `workflowId`（通常是一串数字）

4. **在本技能中使用新工作流**
   - 运行技能时，输入获取到的 `workflowId`
   - 技能会自动分析工作流结构并识别需要的节点

### 工作流命名规范

- 工作流名称必须唯一，不能与已有工作流重复
- 建议使用有意义的名称，如"局部重绘"、"图像生成"等
- 避免使用特殊字符

### 工作流节点配置建议

为了让技能更好地识别和处理工作流，建议：

1. **LoadImage 节点**：
   - 保留默认图片文件名或留空
   - 技能会自动识别并提示用户输入新图片

2. **CLIPTextEncode 节点**：
   - 可以设置默认提示词
   - 也可以留空让用户输入
   - 技能会识别所有的文本编码器节点

3. **Seed 节点**：
   - 建议设置一个默认种子值
   - 用户可以选择使用默认值或输入新种子

### 获取 Workflow ID 的方法

方法1：从工作流详情页获取
- 打开工作流详情页
- 在 URL 或页面信息中找到 workflowId

方法2：从 API 文档获取
- 在 RunningHub API 文档中找到你的工作流
- 查看示例代码中的 workflowId

方法3：使用 API 获取工作流列表
- 调用工作流搜索 API
- 从返回结果中获取 workflowId

## 工作流节点映射

### 工作流节点配置规范

**重要：** 工作流节点配置必须严格按照以下规范，否则技能创建将失败：

- 一个 Workflow ID 只能有一行配置
- 每个 Workflow ID 必须唯一，不得重复
- 每个工作流名称必须唯一，不得重复
- 表格格式必须保持整齐，不得有多余的空行或格式错误
- 节点类型必须用逗号分隔，不得使用其他分隔符

### 工作流节点配置

以下是常见工作流及其需要处理的节点：

| Workflow ID         | 工作流名称 | 已选择需要处理的节点  |
| ------------------- | ----- | ----- |
| 2033490071340453890 | 局部重绘  | 节点9(LoadImage),节点2(CLIPTextEncode),节点27(CLIPTextEncode),节点26(RH_Translator),节点18(Seed) |

### 自定义工作流节点

对于自定义工作流，请根据工作流的实际需求配置需要处理的节点，并确保遵循上述规范。

### 处理未找到的 Workflow ID

如果用户提供的 Workflow ID 不存在，技能将引导用户新建一个工作流。新建工作流的步骤如下：

1. 提示用户输入新工作流的名称
2. 询问用户新工作流需要处理的节点类型
3. 根据用户输入创建新工作流配置
4. 保存新工作流配置并使用它执行任务

## 依赖

- Python 3.6+
- requests 库

## 权限

- 网络访问权限（用于调用 RunningHub API）

## 异常处理

- **异常场景**：API调用失败、工作流执行超时、素材格式错误
- **处理策略**：
  - API调用失败：重试3次，每次间隔2秒
  - 工作流执行超时：设置300秒超时，超时后返回错误信息
  - 素材格式错误：提示用户重新上传正确格式的素材

## 最佳实践

- 确保提供的工作流标识信息正确无误
- 准备好所有必要的素材文件，确保格式符合工作流要求
- 在执行前仔细确认工作流参数，避免不必要的资源消耗
- 定期检查API密钥的有效性
- 对于有默认值的节点，可以选择保留默认值或输入新值
- 建议在 RunningHub 平台先测试工作流，确保能正常运行后再使用本技能

## 示例

### 示例 1：局部重绘工作流（Workflow ID: 2033490071340453890）

- **用户指令**：使用workflowId "2033490071340453890"执行局部重绘工作流
- **技能会自动识别的节点**：
  1. 节点9: LoadImage - 输入图片
  2. 节点2: CLIPTextEncode - 负面提示词
  3. 节点27: CLIPTextEncode - 正面提示词
  4. 节点26: RH_Translator - 翻译器提示词
  5. 节点18: Seed (rgthree) - 随机种子
- **预期输出**：收集用户输入，使用高级接口执行工作流，返回重绘结果

### 示例 2：使用本地图片执行工作流

- **用户指令**：执行工作流，使用本地图片 "D:\test.jpg"
- **技能执行步骤**：
  1. 自动上传本地图片到 RunningHub 平台
  2. 获取上传后的文件名
  3. 将文件名填入 LoadImage 节点
  4. 执行工作流
- **预期输出**：图片自动上传，工作流执行成功

### 示例 3：使用网络图片URL

- **用户指令**：执行工作流，使用网络图片 "https://example.com/image.jpg"
- **技能执行步骤**：
  1. 直接使用提供的URL填入 LoadImage 节点
  2. 无需上传文件
  3. 执行工作流
- **预期输出**：直接使用URL加载图片，工作流执行成功

### 示例 4：自定义选择需要的节点

- **用户指令**：执行工作流，只配置输入图片和正面提示词
- **技能执行步骤**：
  1. 技能显示所有识别到的节点列表
  2. 用户输入 "1 3" 选择节点1（LoadImage）和节点3（CLIPTextEncode）
  3. 只对选中的节点进行配置
  4. 其他节点使用工作流默认值
  5. 执行工作流
- **预期输出**：只配置选中的节点，其他使用默认值，工作流执行成功

### 示例 5：使用所有节点默认值

- **用户指令**：执行工作流，不修改任何参数
- **技能执行步骤**：
  1. 技能显示所有识别到的节点列表
  2. 用户直接按回车或输入无效内容
  3. 技能使用所有节点的默认值
  4. 使用"ComfyUI任务1-简易"接口执行工作流
- **预期输出**：直接使用工作流默认值执行，工作流执行成功
