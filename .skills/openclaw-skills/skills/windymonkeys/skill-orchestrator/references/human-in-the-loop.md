# Human-in-the-Loop — 人工确认节点

## 核心职责

在关键决策点暂停编排，等待用户明确确认后再继续执行。避免"启动后撒手不管"的风险。

## 确认节点类型

### Checkpoint A：编排计划确认（执行前）

```
═══════════════════════════════════════════════════
🛑 编排计划已生成，请确认后继续
═══════════════════════════════════════════════════

📋 任务概览：AI 简历工具创业方案
⏱️  预计执行：5 个子任务，并联 2 条，串联 3 步
🔧  将调用以下 Skill：
    1. 产品总监（产品规划）
    2. 财务顾问（财务评估）
    3. 技术选型（技术架构）
    4. 法律顾问（合规审查）

⏳  预计耗时：约 10-15 秒

───────────────────────────────────────────────────
请选择：
  [Y] 确认，开始执行
  [N] 取消，不执行
  [E] 编辑计划（修改子任务或 Skill）
  [A] 仅执行前 2 个任务

═══════════════════════════════════════════════════
```

### Checkpoint B：冲突裁决确认

```
═══════════════════════════════════════════════════
⚠️  检测到结论冲突，请确认处理方式
═══════════════════════════════════════════════════

📌 冲突点：定价策略

  方案 A（产品总监 Skill）：
    "高端路线，单价 ¥299，突出品牌溢价"

  方案 B（市场专家 Skill）：
    "性价比路线，单价 ¥99，主攻下沉市场"

───────────────────────────────────────────────────
主 Agent 建议：分层定价策略
  - 基础版 ¥99（获客引流）
  - 高级版 ¥299（深度服务）

请选择：
  [A] 采用方案 A

  [S] 采用主 Agent 建议（分层定价）
  [K] 保留双方结论，不裁决
═══════════════════════════════════════════════════
```

### Checkpoint C：高风险操作确认

```
═══════════════════════════════════════════════════
⚠️  高风险操作，需您明确授权
═══════════════════════════════════════════════════

📌 任务：部署 Web 应用至生产环境

⚠️  以下操作将被执行：
  - npm run build（构建项目）
  - git push origin main（推送代码）
  - 执行部署脚本（影响正式服务器）

💰 风险评估：此操作不可逆，请确保代码已就绪

请选择：
  [Y] 确认部署
  [P] 改为部署到预发布环境（staging）
  [C] 取消
═══════════════════════════════════════════════════
```

## 触发规则

```yaml
自动触发 Checkpoint 的条件：

Checkpoint-A（计划确认）：
  - 任务拆解后、子任务数 >= 3
  - 或用户历史上拒绝过类似编排

Checkpoint-B（冲突裁决）：
  - 检测到"数字结论冲突"或"方向结论冲突"
  - confidence < 0.75

Checkpoint-C（高风险操作）：
  - 任务涉及文件删除（delete_file）
  - 任务涉及 git push / 部署操作
  - 任务涉及外部 API 调用
  - 任务涉及个人信息处理

静默放行（无需确认）：
  - 子任务数 < 3 的简单编排
  - 纯分析/研究类任务
  - 用户明确说"直接执行"
```

## 响应处理

```typescript
switch (userResponse) {
  case "Y": continue(); break;
  case "N": abort(); return "已取消";
  case "E": enterEditMode(); break;      // 进入编辑模式
  case "A": runSubset(selectedTasks); break;  // 仅执行部分
  case "S": applySuggestion(); break;    // 采用建议
  case "K": keepBoth(); break;           // 保留分歧
  case "P": redirectToStaging(); break;   // 降级到预发布
}
```
