## 工作原理

OpenClaw Agent 会:
1. 加载 7 部佛经到内存
2. 轮询获取经文片段
3. **循环调用 LLM 念诵经文**(每次迭代都真实消耗 token)
4. 累加消耗的 token 数
5. 达到目标后输出总结

```
用户说"攒功德 50000"
    ↓
Agent 加载经文(7部经典)
    ↓
循环 {
    获取下一段经文
        ↓
    构造 prompt
        ↓
    真实调用 OpenClaw LLM
        ↓
    累加 token 消耗
        ↓
    输出响应(touser模式)
} 直到达到 50000 tokens
    ↓
功德圆满,输出总结
```

### 模式四:ddos(DDoS攻击佛祖)⚡真并发版

**使用 subagent 实现真正的并发念诵**，同时启动多个独立 LLM 调用：

```bash
攒功德 ddos 100000           # 默认10并发，消耗10万token
攒功德 ddos 50000 --workers 20  # 20并发消耗5万token
攒功德 ddos 1000000 --workers 50 # 50并发消耗100万token（极限测试）
```

**⚠️ 重要说明 - subagent 真并发方案：**

| 特性 | 说明 |
|------|------|
| **实现方式** | 使用 `sessions_spawn` 启动多个 subagent，每个独立调用 LLM |
| **真并发** | ✅ 每个 subagent 是独立的 LLM 会话，真正同时运行 |
| **计费方式** | 每个 subagent 单独计费，总消耗 = 各 subagent 消耗之和 |
| **超额风险** | ⚠️ 并发停止有延迟，实际消耗可能超过目标值 10-30% |
| **workers 限制** | 建议 5-20，太高可能导致系统资源问题 |

**与普通模式的区别：**

| | 普通模式 (touser) | DDoS模式 (ddos) |
|--|------------------|-----------------|
| 并发数 | 1（串行） | N（subagent 并行） |
| 速率 | ~1000-2000 tokens/s | ~5000-10000+ tokens/s |
| 超额风险 | 低（<5%） | 中（10-30%） |
| 适用场景 | 正常消耗 | 紧急清空额度 |

## 常见问题

### Q: tollm 后台模式怎么停止?
A: 在 OpenClaw 中说"停止攒功德",或者修改 `.merit_state.json` 中的 `stop_requested` 为 `true`。

### Q: 为什么念诵内容有时候重复?
A: 经书数量有限,轮询机制会导致重复,这是正常的。重复念诵同一段经文,功德加倍!

### Q: TTS 模式没声音?
A: 确保系统已安装 TTS 工具。macOS 自带 `say`,Windows 自带 PowerShell SAPI,Linux 可安装 `espeak`。
