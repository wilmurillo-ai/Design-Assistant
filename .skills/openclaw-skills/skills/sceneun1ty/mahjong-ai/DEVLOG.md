## 第三阶段：核心算法开发

### 3.1 牌面编码系统
设计了简洁的编码：
- **万子：** 1m~9m（m=万）
- **筒子：** 1p~9p（p=筒）
- **条子：** 1s~9s（s=条）

### 3.2 向听数计算
最核心也最难的部分。向听数 = 距离听牌还差几步。

算法思路：
1. 拆出雀头（对子）
2. 数面子（顺子+刻子）和搭子
3. 公式：向听数 = 4 - 面子数 - min(搭子数, 4-面子数)
4. 特殊情况：七对子单独计算

### 3.3 听牌搜索
遍历所有 27 种牌（9万+9筒+9条），逐一模拟摸入，检测是否能胡牌。

### 3.4 出牌建议（14张时）
摸牌后手上 14 张，需要打一张。对每张牌模拟打出：
- 打完后向听数是多少
- 能听几种牌、共多少张
- 该牌的安全度评分

三级排序：向听数最低 → 听牌张数最多 → 安全度最高

### 3.5 安全度评分
评估打出某张牌放炮的风险（0~100分）：
- 🟢 70+ = 安全（场上已出很多）
- 🟡 40~69 = 一般
- 🔴 <40 = 危险（场上少见的中张）

参考因素：
- 场上已出几张同样的牌
- 边张（1、9）比中张（4、5、6）安全
- 剩余张数

## 第六阶段：实战测试

### 6.1 Kevin 的复盘照片
Kevin 打完一局后发了手牌照片让我复盘：
- AI 视觉识别：1条×2, 2条, 3条, 4条, 6条, 7条, 8条, 9条×3
- 加上杠的 5条×4
- 分析结果：**清一色 + 杠，已胡！** ✅
- Kevin 确认：完全正确

### 6.2 Bug 修复
测试中发现向听数计算有 bug：
- **问题：** 启发式算法返回 0（听牌），但实际找不到任何等待的牌
- **原因：** 启发式和精确搜索不一致
- **修复：** 如果启发式说听牌(0)但搜不到等牌，校正为1向听

## 第七阶段：三大优化（v2.0）

Kevin 提出三个优化方向，当场完成：

### 7.1 定缺建议
开局时分析三门花色，推荐定哪门缺：
- 统计每门张数
- 计算搭子价值（对子+3分，刻子+6分，连张+2分）
- 价值最低的定缺
- 如果某门≥7张，提醒有清一色潜力

### 7.2 换三张策略
定缺后换三张的选牌建议：
- 优先换定缺花色的牌
- 选保留价值最低的 3 张
- 提示换回来最想要什么牌
- 处理定缺花色不够3张的情况

### 7.3 对手出牌模式推测
根据对手出的牌分析：
- 推测对手定了什么缺（哪门出最多）
- 检测是否在做清一色（某花色0出牌 = 🔥危险）
- 后期出中张 = 可能已听牌 🚨
- 标出安全牌和危险方向

## 第八阶段：自我训练

### 8.1 生成训练数据
用 Python + Pillow 生成了 10 张随机牌局的标准训练图：

- 场上已出的牌
- 底部附答案（牌面编码）
- 存储在 training/ 目录 + answers.json

### 8.2 算法自测
- 10 局随机牌局：向听数全部正确 ✅
- 5 个特殊场景：全部通过 ✅
  - 普通听牌 → 听4万+3筒=4张
  - 七对听牌 → 听8条3张
  - 清一色听牌 → 听9种23张（九莲宝灯级）
  - 对对胡 → 识别双重牌型
  - 2向听 → 正确计算

### 8.3 麻将桌面图生成
Kevin 要求生成俯视麻将桌图，用于训练视觉识别：
- 第一版：用 Pillow 绘制，1200×1200，绿色桌布+木框
