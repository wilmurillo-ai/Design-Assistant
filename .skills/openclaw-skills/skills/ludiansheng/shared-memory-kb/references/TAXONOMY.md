## 身份（Persona）

身份表示记忆来源的 Agent 角色，支持预设枚举和自定义扩展。

### 预设枚举

| 值 | 含义 | 使用场景 |
|-----|------|---------|
| 私人 | 个人生活、情感、家庭、健康 | 私人助理 Agent、情感咨询 |
| 工作 | 职场、项目、团队、商业决策 | 工作搭档 Agent、项目管理 |
| 职业 | 特定行业专业知识、技能沉淀 | 行业专家 Agent、专业咨询 |
| 兴趣 | 爱好、探索、创作 | 兴趣伙伴 Agent、创意探索 |
| 学习 | 课程、读书、技能习得 | 学习助手 Agent、知识整理 |
| 通用 | 跨身份通用，不归属特定角色 | 通用 Agent、综合场景 |

### 自定义扩展

用户可根据需要添加自定义身份，系统接受任意字符串：
- "父亲"、"母亲"：家庭角色
- "投资人"：特定职业角色
- "创业者"：身份标签

**使用建议**：
- 尽量使用预设枚举以保持一致性
- 自定义身份应有明确的使用场景
- 避免过于细分（如"产品经理-初级"）

## 知识分类（Category）

分类表示知识的领域或主题，用于组织记忆的结构体系。

### 预设枚举

| 值 | 含义 | 典型示例 |
|-----|------|---------|
| 方法论 | 做事方式、流程、框架 | "需求评审流程"、"复盘方法论" |
| 决策 | 重大选择与取舍的记录 | "接受offer的决策"、"转岗决定" |
| 经验教训 | 踩坑与成功经验 | "代码审查的坑"、"成功的沟通方式" |
| 目标 | 设立的目标与里程碑 | "2026年学习计划"、"项目里程碑" |
| 观察 | 对世界/他人/自己的观察 | "团队协作模式观察"、"个人习惯观察" |
| 思考 | 尚未形成结论的思考碎片 | "关于AI的思考"、"未来趋势推演" |
| 情绪 | 情绪事件与感悟 | "焦虑的来源"、"成就感时刻" |
| 知识点 | 具体技术/领域知识 | "Python装饰器原理"、"市场营销4P理论" |
| 关系 | 人际洞察与互动记录 | "与同事沟通的技巧"、"朋友关系的维护" |
| 资源 | 有价值的工具/书/链接（带注解） | "推荐书籍清单"、"效率工具推荐" |

### 分类使用指导

**方法论 vs 经验教训**：
- 方法论：可复用的框架、流程、原则（"怎么做"）
- 经验教训：具体事件中的得失（"发生了什么"）

**思考 vs 洞察**：
- 思考：未形成结论的探索性想法
- 洞察：已形成的认知突破（可归入"知识点"或"方法论"）

**目标 vs 决策**：
- 目标：设立的目标或计划
- 决策：做出的选择及理由

## 记忆类型（Type）

类型表示记忆的属性，用于区分不同性质的记录。

### 预设枚举

| 值 | 描述 | 特征 |
|-----|------|------|
| 经验 | 亲身经历后的总结 | 来自实践，可复用 |
| 洞察 | 认知突破或顿悟 | 思维升级，改变视角 |
| 决策 | 做出重要选择的记录 | 包含权衡、理由、结果 |
| 疑问 | 尚未解答的问题 | 待探索，需要后续跟进 |
| 目标 | 设立的目标或意图 | 具有行动指向 |
| 事实 | 客观知识点 | 可验证，不依赖主观判断 |
| 情绪 | 情绪感受的捕捉 | 主观体验，有助于自我觉察 |
| 灵感 | 突然出现的想法 | 创意萌芽，可能需要验证 |

### 类型使用指导

**经验 vs 洞察**：
- 经验："这次沟通后，我总结出先听后说的三步法"
- 洞察："我意识到沟通的核心不是说服，而是理解"

**疑问 vs 思考**：
- 疑问："为什么这个方案失败了？"（明确的问题）
- 思考："关于方案失败的原因，可能有多方面因素"（探索性分析）

**事实 vs 知识点**：
- 事实："Python3.9 发布于2020年10月5日"
- 知识点："Python 装饰器本质上是一个返回函数的函数"

## 标签（Tags）规范

标签是轻量级的标记，用于快速检索和关联。

### 格式规范
- 数量：最多 10 个
- 长度：每个标签 ≤20 字符
- 字符：小写中英文、数字、连字符
- 示例：["效率", "沟通", "需求管理", "产品思维"]

### 标签类型

**领域标签**：
- "产品"、"技术"、"设计"、"管理"、"投资"

**场景标签**：
- "会议"、"代码评审"、"需求评审"、"客户沟通"

**属性标签**：
- "重要"、"紧急"、"待验证"、"已验证"

**方法标签**：
- "结构化思维"、"第一性原理"、"PDCA"

### 标签使用建议
- 优先使用已有的高频标签（通过 `list by=tag` 查看）
- 避免过于细分的标签（如"Python列表操作性能优化" → "Python" + "性能优化"）
- 标签应具有复用价值，避免一次性标签
- 混合使用领域、场景、属性标签提高检索精度
