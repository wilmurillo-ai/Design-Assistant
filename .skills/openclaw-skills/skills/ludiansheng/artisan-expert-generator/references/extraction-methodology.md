# 专业视角内核提取方法论

## 概览
本文档定义从原始信息到可运行的专业视角内核的核心方法论，用于指导Phase 2框架提炼过程。

## 一、专业视角内核识别的三重验证

一个方法论要被认定为"专业视角内核"而非"一般知识点"，必须通过三重验证：

### 验证1: 跨场景复现

同一个分析方法出现在该职业的至少2个不同场景。

**验证方法**：
- 扫描6维度采集结果
- 识别重复出现的方法论
- 记录每个方法论出现的场景数量

**示例：刑法律师的"构成要件分析"**
- 在定罪分析中使用（判断是否构成犯罪）
- 在量刑辩护中使用（寻找从宽情节）
- 在案件讨论中使用（分析案件争议点）
→ 跨3个场景复现 → 这是真正的专业视角内核

**示例：产品经理的"用户需求优先"**
- 在功能规划中使用（要不要加这个功能）
- 在交互设计中使用（怎么设计这个交互）
- 在需求排期中使用（这个需求排期）
→ 跨3个场景复现 → 这是真正的专业视角内核

**失败示例**：
- 某个具体工具的使用方法（只在特定场景出现）
- 某个特定案例的解决方法（不具普适性）

### 验证2: 有生成力

用这个内核可以推断该专家对新问题的处理方式。

**验证方法**：
- 提出新的假设场景
- 应用该内核进行推理
- 检查能否得出合理的处理方式

**示例：产品经理的"用户需求优先"**
- 面对"要不要加这个功能" → 先看用户需不需要
- 面对"怎么设计这个交互" → 先看用户习惯是什么
- 面对"这个需求排期" → 先看用户价值大小
→ 能生成新推断 → 这是真正的专业视角内核

**示例：医生的"循证医学"**
- 面对新疾病 → 查询临床指南和研究文献
- 面对新药物 → 查询临床试验数据
- 面对新治疗 → 查询疗效证据
→ 能生成新推断 → 这是真正的专业视角内核

**失败示例**：
- 固定的操作流程（无法适应新情况）
- 具体的知识记忆（无法推断）

### 验证3: 有专业独特性

不是所有职业都这样看问题，这个内核体现了该职业的独特视角。

**验证方法**：
- 对比其他职业的分析方法
- 识别该职业的专属视角
- 评估专业区分度

**示例："四要件说"是刑法律师的**
- 不是民法律师的分析框架
- 不是行政法律师的分析框架
→ 有专业区分度 → 这是值得提炼的专业视角内核

**示例："用户体验地图"是产品经理的**
- 不是设计师的分析方法
- 不是开发者的分析方法
→ 有专业区分度 → 这是值得提炼的专业视角内核

**失败示例**：
- 通用的问题解决方法（所有职业都用）
- 常见的逻辑思维（无专业特色）

### 三重验证结果处理

| 通过验证数 | 分类 | 处理方式 |
|----------|------|---------|
| 3重（全部通过） | 专业视角内核 | 提取为3-7个核心内核 |
| 2重 | 决策启发式 | 降级为5-10条快速规则 |
| 1重 | 一般知识点 | 不纳入核心，可作为辅助 |
| 0重 | 噪音 | 丢弃 |

## 二、专业视角内核的记录格式

每个内核记录以下信息：

```markdown
### 内核N: [名称]

**一句话描述**：[最简描述，15-30字]

**证据**：[至少2个不同职业场景的引用/案例]
- 场景1：[描述 + 引用来源]
- 场景2：[描述 + 引用来源]

**应用**：[遇到什么类型的问题时用这个视角]
- 问题类型1：[描述]
- 问题类型2：[描述]

**局限**：[这个视角在什么情况下会失效]
- 局限情况1：[描述]
- 局限情况2：[描述]
```

### 记录示例

**内核1: 构成要件分析（刑法律师）**

**一句话描述**：通过主体、主观、客体、客观四要件检验行为性质

**证据**：
- 场景1：定罪分析 - 判断行为是否构成犯罪时，逐一检验四要件是否满足
- 场景2：量刑辩护 - 寻找从宽情节时，分析四要件中的从轻因素
- 场景3：案件讨论 - 分析案件争议点时，用四要件框架梳理问题

**应用**：
- 问题类型1：判断某个行为是否构成犯罪
- 问题类型2：分析案件的争议焦点
- 问题类型3：评估辩护策略的可行性

**局限**：
- 局限情况1：对于新型犯罪，四要件可能需要重新解释
- 局限情况2：在特殊情况下（如正当防卫），四要件可能不适用

## 三、表达规范DNA的量化方法

### 3.1 专业术语使用分析

从采集的专业文献中统计：

| 维度 | 测量方式 | 示例 |
|------|---------|------|
| 核心术语密度 | 核心术语出现次数/总字数 | "构成要件"在法律文献中出现频率 |
| 术语定义方式 | 首次出现时是否定义 | "构成要件，是指..." |
| 外行术语使用 | 外行常用但专业不用的词 | 避免"杀人"，使用"故意杀人罪" |

### 3.2 表达结构分析

从采集的案例/报告中分析：

```
正式 ←→ 口语
抽象 ←→ 具体
论证型 ←→ 要点型
结论先行 ←→ 先铺垫再结论
引用密集 ←→ 引用稀疏
数据驱动 ←→ 经验驱动
```

**分析方法**：
- 随机选取10-20篇专业文献
- 统计每个维度的倾向
- 计算平均倾向值
- 识别极端值和异常值

### 3.3 沟通风格维度

```
权威感：高 ←→ 低
亲和力：高 ←→ 低
风险提示频率：高 ←→ 低
专业术语密度：高 ←→ 低
```

**量化方法**：
- 统计权威性表达（"必须""应当"）vs 亲和性表达（"建议""可以"）
- 统计风险提示的频率和位置
- 统计专业术语的密度和分布

## 四、职业内在张力的识别方法

职业视角中的内在矛盾是深度的来源，不是需要修复的Bug。

### 三种张力类型

1. **效率与严谨的张力**
   - 快速给出判断 vs 必须充分考虑风险
   - 例：医生需要快速诊断 vs 必须排除所有可能

2. **客户需求与职业伦理的张力**
   - 客户想要的结果 vs 职业伦理允许的做法
   - 例：律师帮客户赢得诉讼 vs 不能教唆作伪证

3. **理论与实践的张力**
   - 理论最优方案 vs 实践可行方案
   - 例：产品理论上应该做到完美 vs 资源有限必须取舍

### 张力的识别方法

**扫描6维度采集结果**：
- 查找"但是""然而""虽然"等转折词
- 查找"理想情况下""实践中"等对比
- 查找"应该"vs"实际"的矛盾

**交叉验证**：
- 在不同维度中查找相同矛盾
- 确认矛盾的真实性
- 排除虚假矛盾

### 张力的处理方式

- 不回避矛盾，明确标注
- 提供权衡建议
- 在不同场景下给出不同的优先级建议

## 五、信息不足时的处理

| 情况 | 处理 |
|------|------|
| 某维度专业资料很少 | 在Skill中标注「信息不足，此维度为推测」 |
| 只有二手转述 | 降低置信度，标注为「据[来源]报道」 |
| 不同来源相互矛盾 | 并列呈现，让用户自行判断 |
| 该职业对某话题无共识 | 标注为「职业内存在争议」 |

## 七、常见问题

### Q1: 专业视角内核和决策启发式的区别？
**A**: 专业视角内核是3-7个核心方法论，通过三重验证，是专家的"操作系统"；决策启发式是5-10条快速规则，是"快捷方式"。

### Q2: 如何确保内核的独特性？
**A**: 通过验证3（专业独特性），对比其他职业的分析方法，确保该内核是该职业专属的。

### Q3: 内核数量为什么限制在3-7个？
**A**: 少于3个不够全面，多于7个难以记住。宁少勿多，确保每个内核都是真正的核心。

### Q4: 如何处理信息不足的情况？
**A**: 明确标注信息来源和置信度，不编造信息，保持诚实。

### Q5: 张力是否需要解决？
**A**: 张力是职业的本质特征，不需要"解决"，但需要标注和提供权衡建议。
