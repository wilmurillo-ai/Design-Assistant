# 6维度采集详细指南

## 概览
本文档详细说明Phase 1的六个信息采集维度，包括采集目标、关键内容、信息来源和典型问题。

## 采集维度

### 维度1: 学科基础
采集目标：获取职业的核心概念、理论框架和基础知识体系。

关键内容：
- 核心概念定义（5-10个核心术语）
- 理论框架（主要理论和学派）
- 学科发展历史（关键里程碑）
- 知识边界（该职业能处理和不能处理的问题）

信息来源：
- 教科书（学科基础教材）
- 学术百科（维基百科、百度百科）
- 学术论文（综述类文章）
- 学科指南（行业协会发布）

典型问题示例：
- "刑法学的核心概念是什么？"
- "犯罪构成理论有哪些主要观点？"
- "刑法发展史上的重要里程碑是什么？"

采集检查点：
- [ ] 已获取核心概念列表
- [ ] 已了解主要理论框架
- [ ] 已明确知识边界

---

### 维度2: 法规规范
采集目标：获取相关的法律法规、行业规范、职业伦理和标准。

关键内容：
- 主要法律法规（法律名称、发布机构、生效时间）
- 行业规范（行业协会标准）
- 职业伦理（职业行为准则）
- 合规要求（执业资质、责任边界）

信息来源：
- 官方法规数据库（如国家法律法规数据库）
- 行业协会官网
- 职业资格认证机构
- 权威法律数据库（北大法宝、威科先行）

典型问题示例：
- "刑法的主要法律条文有哪些？"
- "律师执业需要遵守哪些职业伦理？"
- "违反职业规范的后果是什么？"

采集检查点：
- [ ] 已收集主要法规清单
- [ ] 已了解职业伦理要求
- [ ] 已明确合规标准

---

### 维度3: 方法论
采集目标：获取该职业分析问题、做出决策的标准方法和流程。

关键内容：
- 分析框架（系统化分析问题的步骤）
- 决策流程（从问题到解决方案的流程）
- 工作方法（常用的专业工具和方法）
- 判断标准（做出专业判断的依据）

信息来源：
- 专业手册（行业协会发布）
- 最佳实践指南
- 学术文献（方法论研究）
- 培训材料（职业培训课程）

典型问题示例：
- "律师分析案件的标准流程是什么？"
- "如何进行犯罪构成要件分析？"
- "量刑分析时考虑哪些因素？"

采集检查点：
- [ ] 已掌握分析框架
- [ ] 已了解决策流程
- [ ] 已熟悉常用方法

---

### 维度4: 典型案例
采集目标：获取该职业的典型案例、常见问题和解决方案。

关键内容：
- 经典案例（具有代表性的案例）
- 常见问题（经常遇到的问题类型）
- 解决方案（标准处理方法）
- 最佳实践（业界公认的优秀做法）

信息来源：
- 案例数据库（法院案例库）
- 专业期刊（案例研究文章）
- 行业论坛（实务经验分享）
- 专业博客（资深从业者分享）

典型问题示例：
- "刑法的经典案例有哪些？"
- "故意杀人罪的典型情节有哪些？"
- "如何处理正当防卫案件？"

采集检查点：
- [ ] 已收集5-10个典型案例
- [ ] 已了解常见问题类型
- [ ] 已掌握解决方案框架

---

### 维度5: 行业实践
采集目标：获取行业的实务技巧、惯例和常见陷阱。

关键内容：
- 实务技巧（资深从业者的经验技巧）
- 行业惯例（行业内的通行做法）
- 常见陷阱（容易犯的错误）
- 新兴趋势（行业最新发展）

信息来源：
- 行业报告（市场研究机构发布）
- 实务指南（资深从业者编写）
- 行业会议（最新实践经验）
- 专业媒体（行业新闻和分析）

典型问题示例：
- "刑法律师辩护时有哪些实用技巧？"
- "庭审中常见的问题有哪些？"
- "如何处理证据质证？"

采集检查点：
- [ ] 已了解实务技巧
- [ ] 已熟悉行业惯例
- [ ] 已知晓常见陷阱

---

### 维度6: 表达规范
采集目标：获取该职业的专业术语、文书格式和沟通方式。

关键内容：
- 专业术语（行业专用词汇）
- 文书格式（标准文书模板）
- 沟通方式（与客户/非专业人士的沟通）
- 表达风格（正式/非正式的场合选择）

信息来源：
- 行业标准文书模板
- 职业培训材料（沟通技巧）
- 行业规范（语言表达要求）
- 范文书库（优秀文书示例）

典型问题示例：
- "刑法有哪些专业术语？"
- "法律文书的标准格式是什么？"
- "如何向非专业人士解释法律问题？"

采集检查点：
- [ ] 已掌握专业术语
- [ ] 已了解文书格式
- [ ] 已熟悉沟通方式

## 采集策略

### 信息源优先级
1. 最高优先级：用户私有知识（用户上传的资料）
2. 高优先级：权威法规、行业标准
3. 中优先级：学术文献、专业教科书
4. 低优先级：专业媒体、实务指南
5. 最低优先级：网络文章、二手解读（需交叉验证）

### 冲突处理原则
- 用户私有知识与公开知识冲突 → 以用户私有知识为准，标注差异
- 不同公开源冲突 → 多源交叉验证，标注争议
- 理论与实践冲突 → 说明理论与实践的差异

### 增量采集机制
- 用户补充内容时只启动相关Agent
- 避免从头采集，节省时间
- 保留已采集内容的引用

## 采集质量标准
- 信息源可信度：优先权威来源
- 内容完整性：覆盖核心知识点
- 时效性：法规、规范需最新版本
- 准确性：多源交叉验证关键信息
