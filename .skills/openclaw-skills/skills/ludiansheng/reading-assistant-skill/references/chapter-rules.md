本文档定义了从文本中识别章节信息的规则和优先级。

## 识别规则概述

章节识别使用正则表达式匹配，按优先级从高到低依次尝试。一旦匹配成功，返回识别结果。

### 支持的章节格式

1. 中文章节格式（优先级最高）
2. 带括号的章节（优先级高）
3. 英文章节格式（优先级中）
4. 数字编号格式（优先级中）
5. 页码格式（优先级低）
6. Markdown 标题格式（优先级低）

## 规则详解

### 规则 1：中文章节格式

**正则表达式**：`第[一二三四五六七八九十百千万零\d]+[章节回部篇卷集]`

**匹配示例**：
- ✅ 第一章
- ✅ 第2节
- ✅ 第五回
- ✅ 第十卷
- ✅ 第100章

**不匹配**：
- ❌ 第一章标题（匹配到"第一章"，但不包括后续标题文字）

### 规则 2：带括号的章节

**正则表达式**：`[（\(]第[一二三四五六七八九十百千万零\d]+[章节回部篇卷集][）\)]`

**匹配示例**：
- ✅ （第一章）
- ✅ (第2节)
- ✅ （第十回）

### 规则 3：英文章节格式

**正则表达式**：`Chapter\s+\d+`（不区分大小写）

**匹配示例**：
- ✅ Chapter 1
- ✅ chapter 2
- ✅ CHAPTER 10

### 规则 4：数字编号格式

**正则表达式**：`^\d+[\.\、\s]`

**匹配示例**：
- ✅ 1. （数字加点）
- ✅ 2、（数字加顿号）
- ✅ 3 （数字加空格）

**注意**：必须位于行首

### 规则 5：页码格式

**正则表达式**：`[Pp](\d+)|第(\d+)页`

**匹配示例**：
- ✅ P123
- ✅ p456
- ✅ 第123页

### 规则 6：Markdown 标题格式

**正则表达式**：`^#+\s+(.+)$`

**匹配示例**：
- ✅ # 标题
- ✅ ## 副标题
- ✅ ### 小标题

## 优先级说明

章节识别按以下优先级顺序执行：

1. **优先级 1**：中文章节格式
   - 置信度：0.9
   - 原因：格式标准，误识别率低

2. **优先级 2**：带括号的章节
   - 置信度：0.9
   - 原因：格式明确，不易混淆

3. **优先级 3**：英文章节格式
   - 置信度：0.7
   - 原因：英文书籍标准格式

4. **优先级 4**：数字编号格式
   - 置信度：0.7
   - 原因：常见于笔记和提纲

5. **优先级 5**：页码格式
   - 置信度：0.6
   - 原因：信息有限，仅表示位置

6. **优先级 6**：Markdown 标题格式
   - 置信度：0.5
   - 原因：可能是标题，不一定是章节

## 上下文推断

当无法识别章节时，使用上下文推断机制：

### 继承上一条摘录的章节

如果当前摘录无法识别章节，且存在上一条摘录：
- 置信度：0.6
- 标记为 `inherited`
- 说明："继承上一条摘录的章节信息"

### 关键词推断

如果文本包含章节关键词，但不符合标准格式：
- 检查文本开头是否包含关键词：章、节、回、篇、PART、CHAPTER、SECTION
- 如果包含，提取第一行作为章节信息
- 置信度：0.5
- 标记为 `inferred`

## 特殊情况处理

### 多个章节标记

如果文本中包含多个章节标记，返回：
- 置信度最高的章节
- 同时返回其他匹配结果（最多 3 个）

**示例**：

输入：
```
第1章开头
第2章内容
```

输出：
```json
{
  "chapter": "第1章",
  "chapter_type": "chinese_chapter",
  "confidence": 0.9,
  "all_matches": [
    {"chapter": "第1章", "type": "chinese_chapter"},
    {"chapter": "第2章", "type": "chinese_chapter"}
  ]
}
```

### 无法识别

如果所有规则都无法匹配：
- 返回 `success: false`
- 提示用户手动指定章节

## 使用示例

### 示例 1：标准中文章节

**输入**：
```
第5章 认知放松
当你感觉顺畅、放松时，系统1处于放松状态...
```

**输出**：
```json
{
  "success": true,
  "chapter": "第5章",
  "chapter_type": "chinese_chapter",
  "confidence": 0.9
}
```

### 示例 2：英文章节

**输入**：
```
Chapter 3: The Lazy Controller
System2 is lazy and tries to avoid effort...
```

**输出**：
```json
{
  "success": true,
  "chapter": "Chapter 3",
  "chapter_type": "english_chapter",
  "confidence": 0.7
}
```

### 示例 3：数字编号

**输入**：
```
1. 什么是认知偏差
认知偏差是指...
```

**输出**：
```json
{
  "success": true,
  "chapter": "1.",
  "chapter_type": "numeric",
  "confidence": 0.7
}
```

### 示例 4：继承章节

**输入**：
```
这是一个有趣的实验结果...
```

**上下文**：上一条摘录的章节是"第3章"

**输出**：
```json
{
  "success": true,
  "chapter": "第3章",
  "chapter_type": "inherited",
  "confidence": 0.6,
  "note": "继承上一条摘录的章节信息"
}
```

### 示例 5：无法识别

**输入**：
```
这是一个关于决策的故事...
```

**输出**：
```json
{
  "success": false,
  "error": "未能识别章节信息",
  "suggestion": "请手动指定章节信息"
}
```

## 脚本调用

章节识别通过 `scripts/chapter_parser.py` 实现：

```bash
# 基本用法
python scripts/chapter_parser.py "第5章 认知放松"

# 带上下文
python scripts/chapter_parser.py "这是一个实验结果" "第3章"
```

## 注意事项

1. **优先级判断**：当文本包含多个章节标记时，优先返回置信度最高的
2. **上下文依赖**：继承机制仅在连续摘录时有效
3. **手动指定**：如果识别失败或错误，应允许用户手动指定章节
4. **扩展性**：可以添加新的识别规则，但需注意优先级设置
