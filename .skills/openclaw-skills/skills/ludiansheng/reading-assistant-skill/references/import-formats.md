## 支持的导入来源

| 来源 | 格式 | 文件类型 | 说明 |
|------|------|----------|------|
| 微信读书 | 文本格式 | .txt, .csv | 支持划线、笔记导出 |
| Kindle | My Clippings.txt | .txt | Kindle 设备导出 |
| Kindle App | CSV | .csv | Kindle 手机应用导出 |
| 通用 | CSV | .csv | 自定义格式 |

## 微信读书格式

### 文本格式

微信读书导出的文本格式通常包含：

```
第一章 标题

这是第一段划线内容...

◆ 这是第二段划线内容

第二章 标题

◆ 这是第三章的内容...
```

**识别规则**：
- 章节标题：以"第X章/节"开头
- 划线内容：以"◆"标记开头
- 普通内容：连续的文本行

**示例文件**：

```
第1章 两个系统

系统1的运行是无意识且快速的...

◆ 系统2将注意力转移到需要费脑力的大脑活动上来。

第2章 注意力与努力

◆ 在心流状态中，人们会忘记时间，忘记自我...

这是另一段内容，可能是手动添加的笔记。
```

### CSV 格式

如果微信读书支持 CSV 导出，应包含以下列：

| 列名 | 必需 | 说明 |
|------|------|------|
| book_title | 是 | 书名 |
| chapter | 否 | 章节 |
| content | 是 | 摘录内容 |
| note | 否 | 个人笔记 |
| created_at | 否 | 创建时间 |

## Kindle 格式

### My Clippings.txt

Kindle 设备导出的标准格式：

```
书名 (作者)
- 您在位置 #123-125 的标注 | 添加于 2024年1月1日星期一

这是摘录的内容...
==========
另一本书 (另一作者)
- 您在位置 #456-458 的标注 | 添加于 2024年1月2日星期一

另一段摘录内容...
==========
```

**识别规则**：
- 书名和作者：第一行，格式"书名 (作者)"
- 位置信息：第二行，提取位置编号
- 摘录内容：第三行开始，直到"=========="

### Kindle App CSV

Kindle 手机应用导出的 CSV 格式：

| 列名 | 必需 | 说明 |
|------|------|------|
| Title | 是 | 书名 |
| Author | 否 | 作者 |
| Chapter | 否 | 章节 |
| Highlight | 是 | 摘录内容 |
| Location | 否 | 位置 |
| Date | 否 | 添加日期 |

## 通用 CSV 格式

如果用户从其他来源导入，可以使用通用 CSV 格式：

### 必需列

| 列名 | 必需 | 说明 |
|------|------|------|
| content | 是 | 摘录内容 |

### 可选列

| 列名 | 说明 |
|------|------|
| book_title | 书名（如不提供，使用文件名） |
| chapter | 章节 |
| tags | 标签（用逗号分隔） |
| note | 个人笔记 |
| created_at | 创建时间（格式：YYYY-MM-DD HH:MM:SS） |

### 示例 CSV

```csv
book_title,chapter,content,tags,note,created_at
思考快与慢,第1章,系统1的运行是无意识且快速的,系统1;认知,,2024-01-01 10:00:00
思考快与慢,第1章,系统2将注意力转移,系统2;决策,,2024-01-01 10:05:00
```

## 格式验证

导入脚本会对文件格式进行验证：

### 验证规则

1. **文件存在性**：检查文件是否存在
2. **文件编码**：必须是 UTF-8 编码
3. **文件格式**：检查是否符合预期格式
4. **内容完整性**：至少包含一条有效摘录

### 错误提示

如果验证失败，脚本会返回详细的错误信息：

```json
{
  "success": false,
  "error": "文件格式错误：第5行缺少必需字段 'content'"
}
```

## 使用示例

```bash
# 仅预览，不保存
python scripts/import_notes.py ./wechat_export.txt --source wechat
```

```json

  "success": true,
  "preview": {

      "第2章": 8,

    "sample": [
      {

        "source": "wechat"
      }
    ]
  },
  "imported_count": 0

```

```json
{

  "preview": {
    "total_count": 45,
    "chapters": {...}

  "imported_count": 45

```

```bash

```

## 导入流程

### 标准流程

1. **准备文件**：从阅读平台导出笔记文件
2. **预览导入**：使用脚本预览，检查识别是否正确
3. **确认导入**：确认无误后，执行保存操作
4. **验证结果**：检查导入的笔记数量和内容

### 智能体交互流程

在对话中使用导入功能时：

```
用户：我想导入微信读书的笔记
智能体：请提供导出文件的路径
用户：./wechat_export.txt
智能体：（调用导入脚本，预览模式）
已识别到 45 条摘录，分布如下：
- 第1章：5条
- 第2章：8条
- 第3章：6条
...

示例摘录：
1. "系统1的运行是无意识且快速的..." (第1章)
2. "系统2将注意力转移..." (第1章)

是否确认导入？
用户：确认

已成功导入 45 条摘录到《思考，快与慢》
```

## 注意事项

### 文件编码

- 所有文件必须使用 UTF-8 编码
- 如果导入失败，请检查文件编码

### 重复导入

- 导入脚本不会自动去重
- 如果重复导入，会创建重复的摘录
- 建议：导入前先检查是否已经导入过

### 章节识别

- 导入时会尝试自动识别章节
- 如果识别失败，章节信息为 null
- 可以在导入后手动编辑章节信息

### 标签处理

- 导入时不会自动生成标签
- 可以在导入后使用标签管理功能添加标签

## 故障排查

### 问题 1：导入后中文乱码

**原因**：文件编码不是 UTF-8

**解决**：使用文本编辑器将文件转换为 UTF-8 编码

### 问题 2：章节识别不正确

**原因**：源文件格式不符合预期

**解决**：
1. 检查文件格式是否符合规范
2. 手动编辑文件，调整格式
3. 导入后手动修改章节信息

### 问题 3：部分摘录丢失

**原因**：格式不匹配或内容为空

**解决**：
1. 检查预览结果，确认总数是否正确
2. 查看脚本输出，检查是否有错误提示
3. 调整文件格式后重新导入
