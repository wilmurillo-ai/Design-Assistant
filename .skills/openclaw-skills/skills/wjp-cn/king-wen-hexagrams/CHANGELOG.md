# Changelog

## 1.1.0

- 为首次启用补入命主资料采集约定，默认优先询问农历生日与性别，并允许可选补充出生时辰
- 新增 `onboarding.md` 与 `scripts/init_profile.py`，把首次安装后的命主资料登记做成可直接执行的初始化流程
- 新增 `scripts/postinstall_hint.py` 与 `templates/first-run-message.md`，让安装完成后自动给出初始化提示，并统一首次运行开场话术
- 初始化脚本现在会在完成后追问用户是否需要继续配置每日运势 cron，并新增 `templates/post-init-message.md` 统一初始化后的承接话术
- 增加每日运势模式说明，支持结合 OpenClaw `cron` 定时任务在指定时间自动起卦查看当日气机
- 新增 `daily-fortune.md`，整理每日运势输出口径、OpenClaw CLI 示例与推荐消息模板
- 扩展 `scripts/session_state.py`，可持久化命主资料、每日执行时间与时区
- 为手动抛硬币起卦接入完整流程，支持按六次单次正反记录换算本卦
- 为 `SKILL.md`、`guide.md`、`workflow.md` 与模板补充庄重但克制的起卦引导语和记录规则
- 接入仓库中的 `文王六十四卦详解.md` 作为原典来源，为单卦文件增加卦辞、彖曰、象曰和六爻摘录
- 六爻解释现在优先使用详解文件中的原典表格，遇到个别缺失行时再回退到技能内置解释
- 将六爻解释从“位次框架 + 单条 line_hint”升级为“64 卦各自六条爻位专属解读”，更接近原典爻辞差异
- 为 64 卦增加更贴近经传语义的专属表达数据，让卦义展开、场景判断和动爻提示不再只是通用模板换主题词
- 将 64 个单卦文件从摘要版升级为扩展版，增加卦义展开、结构理解、场景化解读、实务建议和六个爻位的解释框架
- 调整 `scripts/generate_assets.py`，让单卦内容由统一模板批量生成
- 保持 PowerShell 下脚本输出的 UTF-8 中文可读性

## 1.0.0

- 建立 `King Wen Hexagrams` 发布型 skill 目录结构
- 增加核心文档、模板、数据骨架与脚本占位
- 增加 64 卦目录索引与单卦样板文件
- 增加查询、标准化和会话状态脚本的初始实现
