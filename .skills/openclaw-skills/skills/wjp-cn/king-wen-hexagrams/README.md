# King Wen Hexagrams

一个以文王六十四卦为根本、重在问事与开解的 OpenClaw skill。

当前版本：`1.1.0`

它所做的，主要有两件：

- 先替来问之人定清所问与卦象
- 再依本卦、动爻与变卦给出清楚、克制、可继续追问的判断

现在又补上两类更贴近使用感的入口：

- 首次启用时，可先登记命主的农历生日与性别，以增强连续感与仪式感
- 可结合 OpenClaw 定时任务，在每日指定时间起一卦看当天运势
- 已提供安装后初始化脚本，可交互式登记命主资料与每日运势时间
- 安装完成后会自动打印首次启用提示，提醒用户先完成命主资料初始化
- 初始化完成后，会继续提示用户是否需要接入每日运势 cron

## 适用场景

- 用户已经有卦名、卦序或动爻，想要解读
- 用户只有数字、铜钱结果或六爻状态，想要据此定卦
- 用户想亲手抛硬币起卦，并按六次正反记录生成本卦
- 用户需要查询六十四卦基础信息、上下卦、象意和主题
- 用户希望围绕同一问题多轮追问，不想每次都从头整理
- 用户第一次启用技能，希望先登记命主资料
- 用户希望每天固定时间自动看当日运势

## 发布亮点

- 不是随口断语，而是有次第、有凭依的问事与解卦过程
- 文档层、模板层、数据层和脚本层分离，便于后续扩展
- 运行时优先按需读取资料，避免 64 卦全文挤占上下文
- 解读分层清楚，既可继续追问，也便于后续整理
- 每个单卦文件都包含卦义展开、上下卦结构、场景化问事和六个爻位的解释框架
- 64 卦已加入专属语义层，卦义展开与场景判断开始贴近各卦经传语义，而不只是通用模板换主题词
- 六爻位已经细化到爻辞级差异，每卦六条爻位解释不再共用同一条提示
- 单卦文件现已结合仓库中的 `文王六十四卦详解.md`，自动带入卦辞、彖曰、象曰与六爻原典摘录
- 已集成手动抛硬币起卦流程，可将六次正反记录直接标准化为本卦
- 起卦引导保留庄重、敬慎的气息，但不过度夸饰神秘
- 首次启用已纳入命主资料采集流程，默认优先补农历生日与性别
- 已提供 OpenClaw 定时任务示例，可将每日运势接入固定时点自动运行

## 设计重点

- `SKILL.md` 主定启用时机、问卦次第与解卦口径
- `onboarding.md` 负责首次安装后的引导文案与初始化命令
- `templates/first-run-message.md` 负责首次运行时的标准开场话术
- `templates/post-init-message.md` 负责初始化完成后的欢迎与承接话术
- `daily-fortune.md` 负责每日运势模式与 OpenClaw 定时任务接入说明
- 64 卦义理分藏于 `hexagrams/`，按需取用，不一股脑铺陈
- 单卦内容由统一模版生发，但每卦都保留自己的卦义神情、场景判断与爻位差异
- `文王六十四卦详解.md` 中的卦辞、彖曰、象曰与爻辞，已经接入单卦内容
- `templates/` 负责统一口吻，使起卦、开解与追问前后不失其神
- 内部推演与整理之术只作底层依凭，不喧宾夺主，不直接替代解卦口径

## 说明

- 这是一个“依卦问事”的 skill，不是机械吐结论的占断器
- 卦象用于照见局势，不替代现实中的专业判断
- 每日运势更适合作为当日提醒，不替代针对具体事件的正式问卦
- 若后续要补更深的爻辞注解与专题解读，可继续沿现有资料脉络往下增修

## 安装后初始化

如果你希望安装后先把命主资料记下来，可直接运行：

```bash
python scripts/init_profile.py
```

它会按提示依次询问农历生日、性别、可选时辰，以及是否顺手配置每日运势时间。
如果已补齐每日运势时间，初始化完成后还会继续询问你是否需要接入每日运势 cron。

安装完成后，`postinstall` 也会自动打印初始化提示；如果你想手动再次查看这段提醒，可运行：

```bash
npm run show-onboarding
```
