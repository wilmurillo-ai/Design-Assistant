# QA Fix Patterns

## 修复协议

1. 逐页审，标记 Pass / Fix
2. 分级：Critical > Important > Minor
3. **只重跑有问题的页**——删旧 JPG → 修 prompt → 重跑 → 重组装
4. **永远不碰已通过的页**
5. 修复后记录 QA Round 总结

## 10 种常见问题

### 1. 标题被加了 "Slide N:" 前缀


**CRITICAL**: 标题不要加 "Slide N:" 或任何编号前缀。
标题是"[精确文字]"。


### 2. 字号标注泄漏到标题

标题渲染为 "老板的仪表盘(36pt)"。原因：prompt 里样式和文字混在一起。

修复：分离样式描述和标题文字。

标题（大号粗体）：老板的仪表盘

不写 `标题（36pt）：...`

### 3. 英文副标题泄漏


**CRITICAL**: 不要加任何英文副标题或括号。纯中文。


### 4. 中文名乱码 / 英文替代


**CRITICAL**: 名字必须是"[中文名]"。
不能用英文拼音或乱码。
如果无法正确渲染，用头像代替（不显示名字）。


重跑 2 次仍失败 → HTML + Playwright fallback。

### 5. Prompt 指令被渲染为正文

原因：prompt 第一句看起来像标题，被 AI 渲染到页面上。

修复：重写第一句为描述性语句。
- ❌ `策略自进化。背景要求...`
- ✅ `这是一张关于策略淘汰赛机制的页面。`

### 6. 时间轴/列表节点数量错


**CRITICAL**: 精确包含以下 N 个节点（不多不少）：
[节点1] → [节点2] → ... → [节点N]


### 7. 表格/数据缺失

逐行写出每一行内容：

表格是本页最重要的元素。必须完整展示：
| 列A | 列B | 列C |
| 行1 | ... | ... |
| 行2 | ... | ... |
[全部写出]


### 8. 风格不一致（页间跳跃）

启用参考链：生成时传入上一页的 JPG 作为 reference。

### 9. 数字被 AI 篡改


**CRITICAL**: 以下数字必须精确，不要四舍五入或"优化"：
- [数字A]（不是 [错误数字]）
- [数字B]


### 10. 内容被 AI 改写


**CRITICAL**: 以下文案一字不差，不要改写：
"[完整原文]"


## QA Round 总结模板


QA Round N：
- Critical (X 处)：[Slide M 什么改了什么]
- Important (X 处)：[...]
- Minor (X 处)：[...]
其他 N 页未动，已重组装。
