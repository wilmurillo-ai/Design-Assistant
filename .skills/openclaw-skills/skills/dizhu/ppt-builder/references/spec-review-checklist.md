# Spec 自审清单

Spec 写完后至少两轮自审。

## 第一轮：受众视角

站在目标受众的角度读一遍大纲。

### 结构

- [ ] **开场有钩子吗？** 前 30 秒有没有一个让人继续读的理由？
- [ ] **"你/我"比例对吗？** 大部分页在讲受众的世界还是在推销产品？
- [ ] **有没有让受众代入的页面？** 不是抽象概念，是他能映射到自己业务的场景
- [ ] **叙事有节奏吗？** 不是平铺直叙，有起伏（问题 → 故事 → 解决 → 落地）

### 主线（如有场景故事）

- [ ] **有具体的人物吗？** 有名字、身份、动机、顾虑？
- [ ] **时间线合理吗？** 和受众的真实业务周期匹配？
- [ ] **有"失败"场景吗？** 不是一路成功——展示失败怎么恢复更有说服力
- [ ] **有未成功的出路吗？** 70% 没成交的人怎么办？

### 说服力

- [ ] **核心 claim 放太早了吗？** 应该先用故事/数据演示，再在收束页下定义
- [ ] **标题是受众语言还是产品经理语言？** "你每天需要看的 5 个数字" vs "数据分析模块"
- [ ] **数据可信吗？** 没有真实数据就用占位符
- [ ] **有没有受众会抗拒的表述？** 比如暗示"替代你的岗位"

### 落地

- [ ] **落地时间线现实吗？** 不要写理想化的"第一天就上线"
- [ ] **商业信息够吗？** 受众不知道"我是不是目标客户"会焦虑
- [ ] **承诺可以量化和写进合同吗？** 模糊承诺不如不写
- [ ] **CTA 能此刻兑现吗？** 每一个行动都是立刻能做的

## 第二轮：文案质量

- [ ] 禁用词 grep 零正文命中（见 `writing-principles.md`）
- [ ] 核心 claim 只在指定页出现
- [ ] meta 注释没有串进页面内容
- [ ] 占位符统一用 `[待补]`
- [ ] 数字全 deck 一致
- [ ] 人物/品牌名全程一致
