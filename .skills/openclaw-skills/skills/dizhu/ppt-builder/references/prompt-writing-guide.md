# Prompt Writing Guide for Nano Banana Pro

## Prompt 结构

每页 prompt = `STYLE`（全局前缀）+ 页面 prompt（具体内容）

### 全局 STYLE 模板


STYLE = """你是一名顶级PPT设计师。请生成一张16:9比例的专业演示幻灯片图片。

设计风格要求（[风格名称]）：
- 背景：[具体描述——纹理/渐变/纯色]
- 内容卡片：[颜色 hex + 圆角 + 阴影]
- 主标题：[颜色 hex]，粗体大字
- 强调色：[颜色 hex]，用于高亮和装饰
- 正文：[颜色 hex]
- 次要文字：[颜色 hex]
- 字体风格：[现代无衬线体/...]
- 所有中文文字必须清晰可读，绝不能模糊或乱码
- 请只生成一张图片"""


### 页面 Prompt 模板


这是一张[页面类型描述]。[背景和卡片简述]。

标题（[颜色描述 + hex]，[大小]粗体）：[精确文字]

[区域 1 - 如"左半 50%"]：
- [详细内容描述]

[区域 2 - 如"右半 50%"]：
- [详细内容描述]

[底部区域]：
- [锚点文字 / 数据 / 注脚]


## 关键规则

### 1. 文案原文复制

从 content.md 复制，**一字不改**。不让 AI 重新组织语言。

### 2. 颜色用 hex

- ✅ `深墨绿色(#1f4a3c)`
- ❌ `深绿色` / `绿色`

### 3. UI Mockup 详细描述

不要：`一个聊天截图`

要：

一个 iPhone 风格的微信聊天界面 mockup：
- 白色圆角框 + 灰色导航栏，标题"[人名]"
- 一条绿色消息气泡（AI 发的）："[完整消息文字]"
- 气泡右侧一个绿色圆形 AI 头像
- 底部时间 "14:02"


### 4. 中文渲染强制指令

AI 生图经常把中文渲染成乱码。对关键中文内容加：


**CRITICAL**: [字段名]必须是"[精确中文]"。
不能用英文或拼音替代。


### 5. 表格逐行列出

不要：`一个 7 行 4 列的表格`

要：

一个表格（4 列）：
| 列A | 列B | 列C | 列D |
| 行1值 | ... | ... | ... |
| 行2值 | ... | ... | ... |
[写完每一行]


### 6. 列表/节点精确列出


**CRITICAL**: 横向时间轴必须精确包含以下 N 个节点（不多不少）：
[节点1] → [节点2] → ... → [节点N]
其中 [当前节点] 用 [颜色] 高亮。


### 7. 避免 Prompt 泄漏

Prompt 第一句不要写样式指令——AI 可能把它渲染成标题。

- ❌ `[标题关键词]。[背景样式]。`（看起来像标题，会被渲染）
- ✅ `这是一张关于[主题]的页面。`（明确是描述，不是内容）

把样式放在"设计要求"段落里，和"页面内容"段落分开。

## 中文渲染 Fallback

如果某页重跑 2 次中文仍然乱码：

1. **降级为 HTML + Playwright**：写一个 HTML 页面用 Tailwind CSS + 系统中文字体，Playwright 截图 1920×1080，替换该页 JPG
2. **用英文缩写 + 头像代替人名**：不显示全名，用头像或初始字母

## 参考链模式

生成 Slide N 时传入 Slide N-1 的 JPG 作为 visual reference，提升风格一致性。

首次全量生成可不用（速度优先），QA 修复单页时建议启用。
