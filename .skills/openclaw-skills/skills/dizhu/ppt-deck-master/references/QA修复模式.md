# QA 修复模式

## 修复协议

1. 逐页审，标记 Pass / Fix
2. 分级：Critical > Important > Minor
3. **只重跑有问题的页** — 删旧 JPG → 修 prompt → 重跑 → 重组装
4. **永远不碰已通过的页**

## 10 种常见问题

### 1. 标题被加 "Slide N:" 前缀
**CRITICAL**: 标题不要加任何编号前缀。标题是"[精确文字]"。

### 2. 字号标注泄漏（如 "(36pt)"）
分离样式和文字：`标题（大号粗体）：[文字]`，不写 `标题（36pt）：...`

### 3. 英文副标题泄漏
**CRITICAL**: 不要加任何英文副标题或括号。纯中文。

### 4. 中文名乱码
**CRITICAL**: 名字必须是"[中文名]"。不能用英文或乱码。
重跑 2 次仍失败 → HTML + Playwright fallback。

### 5. Prompt 指令被渲染为正文
第一句改为描述句：`这是一张关于[主题]的页面。`

### 6. 时间轴/列表节点数错
**CRITICAL**: 精确包含以下 N 个节点（不多不少）：
[节点1] → [节点2] → ... → [节点N]

### 7. 表格/数据缺失
逐行写出每一行内容。

### 8. 风格不一致
启用参考链：生成时传入上一页 JPG。

### 9. 数字被 AI 篡改
**CRITICAL**: 以下数字必须精确，不要四舍五入：[精确数字]

### 10. 内容被 AI 改写
**CRITICAL**: 以下文案一字不差，不要改写："[完整原文]"

## QA 总结模板

QA Round N：
- Critical (X 处)：[什么改了什么]
- Important (X 处)：[...]
- Minor (X 处)：[...]
其他 N 页未动，已重组装。
