# Nano Banana Prompt 指南

## Prompt 结构

每页 prompt = 全局风格前缀 + 页面具体内容

### 全局风格前缀

STYLE = """你是一名顶级PPT设计师。请生成一张16:9比例的专业演示幻灯片图片。

设计风格要求：
- 背景：[具体描述]
- 内容卡片：[颜色 hex + 圆角 + 阴影]
- 主标题：[颜色 hex]，粗体大字
- 强调色：[颜色 hex]
- 正文：[颜色 hex]
- 字体：现代无衬线体
- 所有中文文字必须清晰可读，不能模糊或乱码
- 请只生成一张图片"""

### 页面 Prompt 模板

这是一张[页面类型]。[背景和卡片简述]。

标题（[颜色 hex]，大号粗体）：[精确文字]

[区域描述]：
- [具体内容]

[底部]：
- [锚点/数据/注脚]

## 6 条铁律

### 1. 文案原文复制
从 content.md 复制，一字不改。

### 2. 颜色用 hex
`深墨绿色(#1f4a3c)` 不是 `深绿色`

### 3. UI mockup 详细描述
不要"一个聊天截图"。要描述每个气泡、头像、时间戳。

### 4. 中文名强制指令
**CRITICAL**: 名字必须是"[中文名]"。不能用英文。

### 5. 表格逐行列出
不说"7 行表格"。写出每一行的每一列。

### 6. 避免 prompt 泄漏
第一句不写样式指令（会被渲染为标题）。

- ❌ `[标题关键词]。[背景样式]。`
- ✅ `这是一张关于[主题]的页面。`

## 中文渲染 Fallback

重跑 2 次仍乱码 → HTML + Playwright 单独生成该页：
1. 写 HTML（Tailwind CSS + 系统中文字体）
2. Playwright 截图 1920×1080
3. 替换该页 JPG

## 参考链模式

生成 Slide N 时传入 Slide N-1 的 JPG 作为参考，提升风格一致性。
首次全量生成可不用，QA 修复单页时建议启用。
