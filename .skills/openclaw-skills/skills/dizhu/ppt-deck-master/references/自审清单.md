# Spec 自审清单

至少两轮自审，不审不进下一步。

## 第一轮：受众视角

### 结构
- [ ] 开场有钩子吗？前 30 秒有让人继续读的理由？
- [ ] 大部分页在讲受众的世界还是在推销产品？
- [ ] 有让受众代入的页面吗？（不是概念，是他能映射到自己业务的场景）
- [ ] 叙事有节奏吗？（问题 → 故事 → 解决 → 落地，不是平铺直叙）

### 主线（如有）
- [ ] 有具体人物吗？（名字、身份、动机、顾虑）
- [ ] 时间线和受众的真实业务周期匹配吗？
- [ ] 有"失败"场景吗？（展示失败恢复比一路成功更有说服力）
- [ ] 有未成功的出路吗？（没成交的人怎么办？）

### 说服力
- [ ] 核心 claim 放太早了吗？（应该先演示后下定义）
- [ ] 标题是受众语言还是产品经理术语？
- [ ] 数据可信吗？（没有就 `[待补]`）
- [ ] 有没有受众会抗拒的表述？

### 落地
- [ ] 落地时间线现实吗？
- [ ] 商业信息够吗？（受众不知道"我是不是目标客户"会焦虑）
- [ ] 承诺可以量化和写进合同吗？
- [ ] CTA 能此刻兑现吗？

## 第二轮：文案质量

- [ ] 禁用词 grep 零正文命中
- [ ] 核心 claim 只在指定页出现
- [ ] meta 注释没有串进页面内容
- [ ] 占位符统一用 `[待补]`
- [ ] 数字全 deck 一致
- [ ] 人物/品牌名全程一致
