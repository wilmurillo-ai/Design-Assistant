# Glossary: HR terms ↔ engineering

| HR term | Skill-HR meaning |
|---------|------------------|
| Job Description (JD) | Structured spec of the user task: competencies, deliverables, success criteria |
| Role / position | A class of work (e.g. "PDF processing", "frontend polish") |
| Incumbent | The Agent Skill selected to execute the task |
| Internal candidate | Installed skill already on disk / in registry |
| External candidate | Skill discovered via web/marketplace search |
| Recruitment | Search, vet, install, register a new skill |
| Probation | Skill is allowed but closely monitored; confirm borderline matches |
| Performance review | Post-task debrief (P05) |
| Termination / offboarding | Remove skill from **eligible pool** in registry; optional filesystem uninstall |
| Headcount plan | Registry `skills[]` and status filters |
| Requisition | User request that triggers P01 |
| Handoff | P03 package given to the incumbent |
| Incident | Immutable audit record of one assignment attempt |

## Status labels

- **active**: standard employee in the pool.
- **on_probation**: hire under watch; lower auto-delegate threshold tolerance.
- **terminated**: former employee; must not receive new assignments via Skill-HR.
- **frozen**: leave of absence; temporarily excluded.
