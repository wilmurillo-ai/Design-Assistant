# START HERE

If you are opening `pm-workbench` for the first time, start here.

`pm-workbench` is **not** trying to be a giant skill library.
It is an **OpenClaw-native PM workbench** for a narrower job:

**take a messy product ask and turn it into a clearer decision, reusable PM artifact, or next move.**

## What kind of repo this is

Use `pm-workbench` when the work sounds like:

- “Help me unpack what this request is really asking for.”
- “Should we do this now or not?”
- “We have too many requests and not enough room.”
- “Turn this into something leadership can react to.”
- “Help me draft the working version, not a bloated document.”

Do **not** expect this repo to be:

- a pile of unrelated PM prompts
- a framework encyclopedia
- a general business operating system
- a giant library you browse before doing real work

The operating principle is simple:

**route fast, ask only what matters, produce something reusable.**

And along the way, the repo should quietly reinforce better PM judgment — without turning into a PM course.

## What the repo is optimized for

`pm-workbench` is strongest when the job is one of these:

- clarify a vague ask before solutioning
- make a go / hold / no-go call
- compare two viable options with a real recommendation
- prioritize under constrained capacity
- draft a lightweight PRD or roadmap artifact
- prepare leadership communication with a concrete ask
- write a useful postmortem instead of a ceremonial recap
