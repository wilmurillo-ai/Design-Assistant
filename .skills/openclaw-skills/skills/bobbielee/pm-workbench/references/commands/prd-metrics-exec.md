# prd-metrics-exec

## Command intent

Use this command when the direction is already chosen and you need to move from working definition to measurable launch communication.

## Best fit

- a feature or initiative that already has directional alignment
- launch preparation that needs both product definition and measurement logic
- teams that want leadership looking at the same launch story as product and design

## Workflow chain

1. `draft-prd`
2. `design-metrics`
3. `prepare-exec-summary`

## What this command protects

It keeps the launch story coherent.
A lightweight PRD without success logic is incomplete, and a launch summary without that logic is weak.

## What good output looks like

By the end, you should have:

- a lightweight working PRD
- a success metric set with guardrails
- a short leadership-ready launch summary

## Example prompt

> Draft the working PRD first, then define how we should measure success, then turn the whole thing into a short leadership-ready launch summary.
