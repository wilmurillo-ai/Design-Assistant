# exec-then-postmortem

## Command intent

Use this command when a launch or initiative is under pressure and you need both the current leadership call and the learning loop after the result lands.

## Best fit

- launch timing decisions under uncertainty
- initiatives where the current decision logic should still be visible later
- teams that want a cleaner bridge between status communication and learning

## Workflow chain

1. `prepare-exec-summary`
2. `write-postmortem`

## What this command protects

It keeps decision communication and later learning connected.
That usually leads to a better postmortem because the team can compare outcome against the real call that was made.

## What good output looks like

By the end, you should have:

- a clear current leadership call
- a reusable frame for expected vs actual
- a cleaner learning loop after the initiative completes

## Example prompt

> Help me write the leadership call we need to make now, and structure it so we can turn the result into a useful postmortem afterward.
