# prioritize-roadmap-exec

## Command intent

Use this command when you need to turn too many requests into a quarter story leadership can approve.

## Best fit

- overloaded quarter planning
- portfolio calls under tight capacity
- product leader situations where a ranked list is not enough

## Workflow chain

1. `prioritize-requests`
2. `build-roadmap`
3. `prepare-exec-summary`

## What this command protects

It keeps three different jobs from collapsing into one muddy answer:

- deciding what is above and below the line
- turning that call into a staged plan
- turning the plan into a leadership-ready ask

## What good output looks like

By the end, you should have:

- an above-the-line / below-the-line call
- a roadmap with a clear stage goal
- a one-page leadership-ready summary

## Example prompt

> Help me turn this overloaded quarter into something leadership can approve. First prioritize the work, then shape it into a roadmap, then compress it into a one-page exec summary with the main ask.
