# clarify-then-compare

## Command intent

Use this command when several directions are already on the table, but the decision frame is still unstable.

## Best fit

- two-path strategy debates
- founder or product-leader trade-off calls
- situations where people are comparing options before agreeing on what the choice is optimizing for

## Workflow chain

1. `clarify-request`
2. `compare-solutions`

## What this command protects

It prevents polished pros-and-cons theater.
The command makes the decision objective explicit first, then compares the options against that objective.

## What good output looks like

By the end, you should have:

- a clarified decision objective
- one current-period recommendation
- an explicit why-not-the-other-path-now explanation

## Example prompt

> First help me clarify what decision we are actually making, then compare these two options and recommend one current-period path.
