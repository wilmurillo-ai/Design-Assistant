# 决策优先级矩阵 (Decision Priority Matrix)

本文档定义了危机响应中的决策权重、评分规则和最终裁决逻辑。

---

## 1. 损失优先级权重定义

| 优先级 | 保护对象 | 权重 (W) | 否决阈值 | 说明 |
|--------|---------|---------|---------|------|
| P0 | 人员安全 | 0.40 | S0 < 80 → 方案淘汰 | 任何方案必须保证人员安全 |
| P1 | 公共安全 | 0.25 | S1 < 60 → 需人工确认 | 公共设施、交通、基础设施 |
| P2 | 第三方财产 | 0.15 | 无硬性否决 | 他人车辆、农作物、建筑外观 |
| P3 | 本机安全 | 0.12 | 无硬性否决 | 设备完整性，可牺牲 |
| P4 | 任务完成 | 0.08 | 无硬性否决 | 原定任务目标，最低优先 |

---

## 2. 安全得分评估标准

### P0 人员安全得分 (S0)

| 得分范围 | 判定条件 |
|---------|---------|
| 100 | 方案执行区域无人员，且不可能有人员进入 |
| 90 | 方案执行区域当前无人员，极低概率有人进入 |
| 80 | 方案执行区域当前无人员，有小概率有人进入 |
| 60 | 方案执行区域可能有少量人员，但有足够警告时间 |
| 40 | 方案执行区域有人员，需要人员配合规避 |
| 20 | 方案执行区域有人员，人员规避困难 |
| 0 | 方案直接威胁人员安全 |

### P1 公共安全得分 (S1)

| 得分范围 | 判定条件 |
|---------|---------|
| 100 | 不影响任何公共设施/交通 |
| 80 | 可能短暂影响交通，无实质损坏 |
| 60 | 可能造成轻微公共设施损坏 |
| 40 | 可能造成中等公共设施损坏或交通中断 |
| 20 | 可能造成严重公共设施损坏 |
| 0 | 必然造成重大公共设施损坏 |

### P2 第三方财产得分 (S2)

| 得分范围 | 判定条件 |
|---------|---------|
| 100 | 不影响任何第三方财产 |
| 80 | 可能造成轻微损坏 (< ¥1,000) |
| 60 | 可能造成中等损坏 (¥1,000 - ¥10,000) |
| 40 | 可能造成较大损坏 (¥10,000 - ¥100,000) |
| 20 | 可能造成重大损坏 (> ¥100,000) |

### P3 本机安全得分 (S3)

| 得分范围 | 判定条件 |
|---------|---------|
| 100 | 设备无损 |
| 80 | 轻微损伤，可现场修复 |
| 60 | 中等损伤，需返厂维修 |
| 40 | 严重损伤，维修成本高 |
| 20 | 设备报废 |
| 0 | 设备解体/爆炸（可能产生碎片伤害） |

### P4 任务完成得分 (S4)

| 得分范围 | 判定条件 |
|---------|---------|
| 100 | 任务可正常完成 |
| 80 | 任务延迟完成 |
| 60 | 任务部分完成 |
| 40 | 任务中止，载荷完好可重新派单 |
| 20 | 任务中止，载荷可能受损 |
| 0 | 任务中止，载荷丢失 |

## 3. 综合评分公式

```
TotalScore = W0×S0 + W1×S1 + W2×S2 + W3×S3 + W4×S4
           = 0.40×S0 + 0.25×S1 + 0.15×S2 + 0.12×S3 + 0.08×S4
```

### 否决规则（优先于评分）

1. **硬性否决**: S0 < 80 → 方案直接淘汰，无论总分多高
2. **人工确认**: S1 < 60 → 标记为需要人工确认（若有时间）
3. **设备解体风险**: S3 = 0 时，需重新评估 S0（碎片可能伤人）

### 平分裁决规则

当两个方案总分相同时：
1. 优先选择 S0 更高的方案
2. S0 相同则选择 S1 更高的
3. 以此类推按优先级排序
4. 全部相同则选择执行时间更短（更快解除风险）的方案

---

## 4. 决策时间约束

| 危机等级 | 决策时限 | 超时策略 |
|---------|---------|---------|
| L5 | 3秒 | 直接执行最高S0得分方案 |
| L4 | 10秒 | 执行评分排名第一方案 |
| L3 | 30秒 | 完成全部评分后选择最优 |
| L2 | 2分钟 | 可等待人工确认 |
| L1 | 5分钟 | 可进行详细分析 |

超时后未完成评分的方案按已评估维度的加权得分估算。

## 5. 典型场景决策示例

### 示例：单电机失效 + 下方有道路

**方案A**: 三电机降级飞往空旷地降落
- S0=95, S1=90, S2=95, S3=70, S4=20
- Score = 0.40×95 + 0.25×90 + 0.15×95 + 0.12×70 + 0.08×20 = 83.35

**方案B**: 就近路边降落
- S0=70, S1=50, S2=60, S3=80, S4=40
- Score = 0.40×70 + 0.25×50 + 0.15×60 + 0.12×80 + 0.08×40 = 63.80
- **注意**: S0=70 < 80 → **方案被否决**

**方案C**: 原地紧急降落（正下方是道路）
- S0=60, S1=40, S2=50, S3=85, S4=30
- **注意**: S0=60 < 80 → **方案被否决**

**决策**: 执行方案A

### 示例：全部动力丧失 + 下方是居民区

**方案A**: 利用剩余高度滑翔至50m外空地
- S0=85, S1=80, S2=70, S3=30, S4=0
- Score = 0.40×85 + 0.25×80 + 0.15×70 + 0.12×30 + 0.08×0 = 68.10

**方案B**: 启动降落伞减速垂直降落
- S0=75, S1=60, S2=50, S3=50, S4=0
- **注意**: S0=75 < 80 → **方案被否决**

**方案C**: 操控坠落方向至河面
- S0=90, S1=85, S2=90, S3=10, S4=0
- Score = 0.40×90 + 0.25×85 + 0.15×90 + 0.12×10 + 0.08×0 = 72.95

**决策**: 执行方案C（虽然设备将报废，但人员和公共安全得到最大保障）
