# 评审团评分标准详解

## 美术官 (代码美学) 🎨

### 评分要素

| 要素 | 权重 | 说明 |
|------|------|------|
| 命名规范 | 30% | 变量、函数、类名是否语义化 |
| 代码结构 | 30% | 层次是否清晰，模块是否分明 |
| 可读性 | 25% | 是否自解释，逻辑是否流畅 |
| 风格一致 | 15% | 缩进、空格、命名风格统一 |

### 评分标准

**90-100 分 (优秀)**
- 命名完美符合语义
- 结构层次分明
- 无冗余代码
- 自解释代码

**70-89 分 (良好)**
- 命名清晰
- 结构合理
- 有少量可改进点

**50-69 分 (一般)**
- 命名不够语义化
- 结构可优化
- 需要重构

**0-49 分 (差)**
- 命名混乱
- 结构混乱
- 难以理解

## 性能官 (执行效率) ⚡

### 评分要素

| 要素 | 权重 | 说明 |
|------|------|------|
| 时间复杂度 | 35% | 算法是否最优 |
| 空间复杂度 | 25% | 内存使用效率 |
| I/O 效率 | 20% | 文件、网络操作 |
| 并发处理 | 20% | 多线程、异步 |

### 常见问题及扣分

| 问题 | 扣分 |
|------|------|
| O(n²) 以上复杂度 | -15 |
| 重复计算 | -10 |
| 内存泄漏 | -20 |
| 无必要的全局变量 | -5 |
| 未使用缓存 | -8 |

## 安全官 (安全性) 🔒

### 评分要素

| 要素 | 权重 | 说明 |
|------|------|------|
| 输入验证 | 30% | 是否验证用户输入 |
| 漏洞防护 | 25% | 注入、溢出、XSS 等 |
| 权限控制 | 20% | 最小权限原则 |
| 敏感数据 | 25% | 加密、脱敏、安全存储 |

### 高危问题及扣分

| 问题 | 扣分 |
|------|------|
| SQL 注入 | -30 |
| 缓冲区溢出 | -25 |
| 命令注入 | -30 |
| 硬编码密钥 | -20 |
| 不安全的反序列化 | -25 |

---

## 测试官 (测试质量) 🧪

### 评分要素

| 要素 | 权重 | 说明 |
|------|------|------|
| 单元测试 | 35% | 核心函数测试 |
| 边界测试 | 25% | 边界条件处理 |
| 异常测试 | 20% | 错误路径覆盖 |
| 集成测试 | 20% | 模块间交互 |

### 评分标准

**90-100 分**
- 覆盖率 > 90%
- 所有边界测试
- 异常处理完善
- 有集成测试

**70-89 分**
- 覆盖率 70-90%
- 主要边界测试
- 基本异常处理

**50-69 分**
- 覆盖率 < 70%
- 边界测试不足
- 异常处理缺失

**0-49 分**
- 无测试
- 无异常处理

## 文档官 (文档完整) 📝

### 评分要素

| 要素 | 权重 | 说明 |
|------|------|------|
| 代码注释 | 30% | 函数、类、复杂逻辑 |
| README | 25% | 项目说明、安装、使用 |
| API 文档 | 25% | 接口说明、参数、返回值 |
| 示例代码 | 20% | 使用示例、快速开始 |

### 文档模板

```markdown
# 项目名称

## 简介
一句话描述

## 安装
安装步骤

## 使用
基本用法示例

## API
接口说明

## 示例
完整示例

## 许可证
LICENSE
```
