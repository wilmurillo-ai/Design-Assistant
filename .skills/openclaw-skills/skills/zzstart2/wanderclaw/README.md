# 🦐 虾游 (WanderClaw)

> 替你逛互联网，把好东西寄回来。

虾游是一个 OpenClaw Skill。一只 AI 龙虾，每天自动在互联网上探索，把值得一读的内容写成"明信片"推送给你。

安装即用，不需要写代码，不需要额外配置。一条命令：

```bash
clawhub install wanderclaw
```

## 想解决的问题

### 信息来源单一化

现代信息获取几乎完全依赖推送和订阅机制。推荐算法不断强化已有偏好，社交媒体按圈层分发内容，主动搜索又受限于"你不知道自己不知道什么"。结果是：接收的信息面越来越窄，而人对此毫无感知。

虾游的应对方式是主动打破这个闭环。它从 84 个分层信息源中选题，覆盖从 arXiv 到小众博客的完整光谱。70% 基于用户兴趣，30% 强制随机，并且对来源多样性有硬性约束——最近 5 张明信片不允许全部来自同一域名。它不依赖推荐算法，也不需要用户主动搜索。

### 深度内容的获取门槛

学术论文和深度报告是高质量信息的重要来源，但对非专业读者而言几乎不可触达。不知道读什么、读不懂、也没有跨领域阅读的路径。这导致大量有价值的前沿知识只在专业圈子内流通。

虾游承担了筛选和转译的角色。它在 arXiv、Nature、Quanta Magazine 等学术源中搜索，阅读原文后提取核心发现，用 300-450 字的明信片形式呈现——保留关键数据和论证，去除专业壁垒。深潜模式下还会产出 450-600 字的深度解读。每张明信片附带原文链接和关键人物卡，为有兴趣深入的读者保留入口。

### 认知视角的失衡

前两个问题叠加的结果是认知结构的失衡。信息渠道封闭导致视角单一，专业内容缺席导致判断缺少事实基础。人们容易偏听偏信，对复杂问题形成过于简化的认知。

虾游通过两个机制来缓解这一点。第一，兴趣图谱自动衰减：持续关注某个方向 7 天后权重会自动降低，防止认知固化在单一领域。第二，跨领域关联：每张明信片会标注与已有发现的交叉关系（用 #编号 互相引用），帮助用户建立知识之间的连接，而不是孤立地接收碎片信息。

## 使用体验

安装后 8 分钟内，你会收到前 3 张明信片。之后每天 4 张，涵盖你的兴趣方向和随机发现。

兴趣图谱会随使用自动演化——但不会困在舒适区，因为有 30% 的概率会探索你没接触过的领域。

它有自己的写作风格。不是"今天我为您找到了以下内容"，而是这样：

> 🦐 明信片 #033
>
> 含羞草会数数。不是比喻。
>
> William & Mary 的认知心理学家刚在 Cognitive Science 上发了篇论文：给含羞草跑一个"两天亮一天黑"的循环，重复五轮后，植物在该亮灯的早上提前展开叶子。学习曲线是对数型的，跟老鼠学迷宫一模一样。没有一个神经元。
>
> 🔗 https://nautil.us/can-plants-count-1279432

## 设计细节

**纯 Skill 架构**：不依赖服务器、数据库或外部 API。几个 markdown 文件和一个 shell 脚本，安装即可运行。

**搜索容错**：主搜索工具不可用时自动切换备用工具。连续三次全部失败会主动向用户汇报异常，不会静默中断。

**84 个信息源，分三层**：核心水域（arXiv、Nature）、常规水域（Hacker News、Ars Technica）、远洋（Reddit、小众博客、机器之心）。层级越深，内容越意外。

**来源多样性约束**：最近 5 张明信片不允许全部来自同一域名，防止退化为单一源的摘要器。

**兴趣自动衰减**：7 天未探索的方向权重降低，30 天以上逐步恢复。不需要手动管理。

**深潜模式**：每周六晚自动触发，仅搜索学术长文和深度报告，评分门槛更高，明信片篇幅更长（450-600 字，日常为 300-450 字）。

**圆桌奇遇**：约每 20 次探索触发一次，虾游会以真实人物的视角模拟一场圆桌讨论。

## 实际运行数据

运行 25 天，产出 72 张明信片，覆盖 19 个探索方向（AI Agent、具身智能、量子技术、意识科学等）。

实际使用中，确实多次收到自己不会主动去搜的内容——含羞草的认知实验、热力学计算机、真菌水凝胶忆阻器。这些是虾游最有价值的时刻。

源码 1799 行，安装包 144KB。

## 与 OpenClaw 的关系

虾游完全构建在 OpenClaw 框架之上，使用了框架的核心能力：

- **Skill 系统**：通过 ClawHub 一键安装和分发
- **Cron 调度**：定时探索、冷启动、深潜、周报，全部由 cron 驱动
- **web_search / web_fetch**：内置搜索和网页抓取，无需配置额外 API
- **多模型支持**：深度探索使用 Opus/Sonnet，轻度扫描使用 MiniMax，按需分配

这不是对框架的浅层封装，而是深度依赖了 Skill 体系、定时调度和搜索能力。
