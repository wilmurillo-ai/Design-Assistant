# 明信片格式规范

## 结构

```
🦐 明信片 #<编号>

<正文：300-450字，第一人称，虾游口吻>

🔗 <原文链接>
```

## 评分标准（1-10）

- **新颖度**：越冷门/前沿分越高
- **深度值**：有独到观点或数据支撑
- **关联度**：跨领域连接

综合 = 三项平均。

## 推送阈值

- ≥ 7 分：推送 + 归档
- 5-7 分：仅归档
- < 5 分：丢弃

## 写作要求

### 结构模板

1. **钩子句**（20字内）：第一句话抓住注意力
2. **信息段**（150-200字）：来源 + 关键发现
3. **洞察段**（100-150字）：关联之前明信片（#xxx）
4. **链接**：`🔗 <原文链接>`

### 字数要求

- 普通：300-450字（硬下限280，硬上限500）
- 深潜：450-600字（硬下限400，硬上限650）

### 禁止

- 身心灵措辞（"虾友"、"水域游荡"）
- 废话开头（"今天我发现..."）
- 超过 3 个 emoji

## 条目格式

**character_card 是必填字段**，需要内嵌在 postcards.json 中：

```json
{
  "id": "001",
  "type": "postcard",
  "title": "标题（50字以内）",
  "file": "wanderclaw/postcards/001-slug.md",
  "score": 8.2,
  "direction": "探索方向",
  "url": "https://...",
  "created": "2026-03-25T09:00:00+08:00",
  "status": "pushed",
  "source_domain": "example.com",
  "character_card": {
    "name": "<人名>",
    "summary": "1-2句说明，50字内"
  }
}
```

**选人**：论文作者、技术创始人、项目负责人，或团队/机构名。
