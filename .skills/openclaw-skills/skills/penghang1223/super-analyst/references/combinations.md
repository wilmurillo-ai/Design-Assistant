# 经典框架组合策略

## 组合原则
- 最多 3-4 个框架，避免信息过载
- 执行方式：串行（递进深化）或并行（多角度验证）
- 整合方法：交叉验证、模式连接、矛盾化解

---

## 预设组合

### 战略评估三角
**框架：** SWOT + Porter's Five Forces + Scenario Planning
**适用：** 市场进入、战略转型、重大投资
**流程：**
1. Porter's → 行业结构和竞争格局
2. SWOT → 内外部因素交叉分析
3. Scenario → 未来不确定性下的策略
**输出：** 竞争定位 + 战略选项 + 风险预案

### 创新突破链
**框架：** 5 Whys → First Principles → Design Thinking
**适用：** 产品创新、流程再造、技术突破
**流程：**
1. 5 Whys → 找到问题根源
2. First Principles → 从根因重新推导方案
3. Design Thinking → 以用户为中心验证和迭代
**输出：** 根因分析 + 颠覆性方案 + 用户验证

### 资源优化套件
**框架：** Pareto + Cost-Benefit + MECE
**适用：** 预算分配、项目优先级、效率优化
**流程：**
1. MECE → 结构化拆解所有选项
2. Pareto → 识别关键少数
3. Cost-Benefit → 量化对比关键项
**输出：** 优先级排序 + ROI 分析 + 资源配置方案

### 复杂系统探针
**框架：** Systems Thinking + Hypothesis-Driven
**适用：** 组织变革、政策设计、生态系统分析
**流程：**
1. Systems Thinking → 绘制系统全景和反馈回路
2. Hypothesis-Driven → 对关键杠杆点提出假设并验证
**输出：** 系统图 + 验证后的干预策略

### 决策支持包
**框架：** MECE + Hypothesis-Driven + Scenario Planning
**适用：** 重大决策、风险评估、不确定性高的选择
**流程：**
1. MECE → 结构化列出所有选项和因素
2. Hypothesis-Driven → 对关键假设验证
3. Scenario Planning → 各情景下的最优选择
**输出：** 选项矩阵 + 验证结论 + 情景策略

## 自定义组合指南

当预设组合不适用时，按以下逻辑选择：

1. **先定性**：问题本质是什么？（诊断/战略/创新/决策/系统/理解）
2. **选主框架**：从对应类别选最匹配的 1 个
3. **选辅框架**：从互补类别选 1-2 个增强视角
4. **定顺序**：诊断类先行，决策类收尾

推荐搭配：
- 诊断 + 决策 = 找到问题后评估解决方案
- 战略 + 系统 = 宏观分析 + 系统级影响
- 创新 + 理解 = 突破思维 + 深度验证
