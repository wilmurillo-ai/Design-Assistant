# 🏆 质量审查检查清单

## 命名与风格
- [ ] 函数/变量命名清晰且一致
- [ ] 遵循语言标准编码规范（PEP8/Airbnb/Google Style）
- [ ] 无魔法数字，使用常量替代
- [ ] 文件组织合理（职责单一）
- [ ] 无未使用的导入和变量

## 错误处理
- [ ] 异常被恰当捕获（不吞异常）
- [ ] 错误信息有上下文（便于排查）
- [ ] 资源释放有保障（finally/context manager）
- [ ] 有降级/兜底方案
- [ ] 用户友好的错误提示

## 设计原则
- [ ] 单一职责（函数/类只做一件事）
- [ ] 开闭原则（扩展优于修改）
- [ ] 依赖注入而非硬编码
- [ ] 接口隔离（不强制实现不需要的方法）
- [ ] DRY — 无不必要的重复代码

## 类型安全
- [ ] 关键数据有类型定义
- [ ] 可空值正确处理（Optional/Null check）
- [ ] 边界值处理正确
- [ ] 枚举/常量优于裸字符串
- [ ] TypeScript 严格模式（如适用）

## 代码可读性
- [ ] 函数长度 < 50 行
- [ ] 每个函数只有一个抽象层级
- [ ] 注释解释 WHY 而非 WHAT
- [ ] 复杂逻辑有说明注释
- [ ] 代码即文档（self-documenting）
