# 🔧 可维护性审查检查清单

## 代码复杂度
- [ ] 圈复杂度 < 10（理想 < 5）
- [ ] 函数嵌套层级 ≤ 3
- [ ] 无超长函数（> 50 行）
- [ ] 无超长文件（> 500 行）
- [ ] 类的方法数量合理（< 20 个公开方法）

## 模块化
- [ ] 职责边界清晰
- [ ] 模块间耦合度低
- [ ] 抽象层合理（不过度/不足）
- [ ] 公共逻辑提取为工具函数
- [ ] 业务逻辑与基础设施分离

## 文档
- [ ] 公共 API 有文档注释
- [ ] 复杂算法有设计说明
- [ ] README/CHANGELOG 更新
- [ ] 关键配置有注释说明
- [ ] 接口变更有文档记录

## 测试
- [ ] 有单元测试覆盖核心逻辑
- [ ] 边界值和异常路径有测试
- [ ] 测试命名清晰（表达意图）
- [ ] 测试可独立运行（无隐式依赖）
- [ ] 覆盖率 > 80%（关键路径）

## 可扩展性
- [ ] 使用设计模式（策略/工厂/观察者等）
- [ ] 配置化而非硬编码
- [ ] 插件/钩子机制（如需扩展点）
- [ ] 版本兼容考虑（API 变更策略）
- [ ] 数据库迁移方案完整

## 技术债务
- [ ] 无 TODO/FIXME 遗留（或已跟踪）
- [ ] 无临时 hack 代码
- [ ] 已知问题有处理计划
- [ ] 依赖无已知过期风险
- [ ] 无死代码（unused code）
