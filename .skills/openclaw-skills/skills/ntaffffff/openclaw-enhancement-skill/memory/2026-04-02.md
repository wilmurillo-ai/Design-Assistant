### 📚 学习笔记

#### 1. 为什么要多Agent？

**单个 Agent 局限：**
- 能力有限，什么都做但做不好
- 模型成本高
- 容错性差

**多 Agent 优势：**
- 专业化：每个Agent只做一件事件
- 可扩展：加Agent就能扩展能力
- 容错：一个挂了其他还能工作
- 成本优化：简单任务用小模型

#### 2. Agent 角色定义

| 角色 | 功能 |
|------|------|
| Planner | 任务规划、分解 |
| Executor | 执行具体操作 |
| Reviewer | 检查结果质量 |
| Router | 路由、分发任务 |

#### 3. 任务分配机制

- **串行模式**：A → B → C（按顺序）
- **并行模式**：同时执行多个任务
- **树形模式**：分叉再汇总

#### 4. 通信机制

- 直接消息（Point-to-Point）
- 广播（Broadcast）
- 消息队列（Message Queue）

---

### 💻 代码实践

✅ 已运行 `multi_agent.py`

```
🚀 开始执行任务: 分析销售数据,生成图表,发送报告

🔍 Planner 分析任务 → 分解为 3 个步骤
⚡ Executor 执行 → 全部完成
🔎 Reviewer 审核 → 通过 (3/3)
🎉 任务完成!
```

---

### ✅ 今日总结

**核心概念：**
1. 多Agent系统 = 团队协作，分工明确
2. 角色定义：Planner（规划）、Executor（执行）、Reviewer（审核）
3. 通信：通过消息（AgentMessage）传递信息
4. 模式：串行（流水线）、并行（同时）、树形（分层）

**类比理解：**
- 单Agent = 一个人做所有事
- 多Agent = 一个团队，有策划、执行、审核

## 🎬 视频剪辑 CLI 工具开发

### 背景
主人提出想要一个自动剪辑瓦罗兰特精彩片段的工具，需求是：
- 主人看视频 → 标记关键时间点 → CLI 自动剪切拼接

### 实现成果

#### 1. 工具代码
- 位置：`~/video-cli/clipper.py`
- 版本：v2.0

#### 2. 功能列表
```bash
# 预览片段
clipper preview -v 视频.mp4 -c "1:20-1:35,3:45-4:00"

# 剪切片段
clipper cut -v 视频.mp4 -c "1:20-1:35,3:45-4:00" -o 成品.mp4

# 加背景音乐
clipper cut -v 视频.mp4 -c "1:20-1:35" -m BGM.mp3 -o 成品.mp4

# 慢动作（精彩击杀）
clipper cut -v 视频.mp4 -c "1:20-1:35" --speed 0.5 -o 成品.mp4

# 加速（开场）
clipper cut -v 视频.mp4 -c "1:20-1:35" --speed 2.0 -o 成品.mp4
```

#### 3. 技术实现
- 核心：FFmpeg 命令封装
- 支持：多片段剪切、背景音乐混音、速度调整、转场效果
- 时间格式：支持 10、1:20、1:20:30 等格式

## 🔥 Claude Code 源码泄露事件分析

### 事件背景（2026-04-01）

**发生了什么：**
- Anthropic 在 2026年3月31日发布 Claude Code v2.1.88
- 意外把 59.8MB 的 Source Map 文件打包进 npm 包
- 导致 51.2万行 TypeScript 源码泄露

### 关键信息
- 这是 Claude Code **第二次**源码泄露（2025年2月也发生过）
- 泄露的是客户端代码，不含核心模型权重
- GitHub 镜像被 Anthropic 发 DMCA 删除
- 社区出现 Python 重写版 `claw-code`，2小时破 5万 star

### 架构分析（值得学习）

#### Claude Code 核心架构
```
UI 层 (React + Ink)
    ↓
REPL 循环 (核心)
    ↓
工具系统 (Tool System) - 40+ 工具
    ↓
LLM 层 (Anthropic SDK)
```

#### 值得学习的点
1. **工具插件化** - 每个工具独立，包含 schema 定义 + 权限
2. **REPL 架构** - 简单但强大的交互循环
3. **Ink UI** - 用 React 组件构建终端界面
4. **权限模型** - 每个工具独立权限控制
