# SEO And Brand Patterns

Use these patterns to create names that balance keyword relevance and brandability.

## What usually works

- exact root + strong suffix
- action root + product suffix
- semantic category word + AI/product suffix
- shortened root + memorable ending

Examples:

- `calc` + `pilot`
- `note` + `forge`
- `draft` + `flow`
- `brief` + `lab`
- `memo` + `base`

## Useful suffixes

Prefer these before inventing stranger endings:

- `ai`
- `hq`
- `lab`

- `flow`

## Useful prefixes

- `get`
- `try`
- `use`
- `my`
- `go`
- `smart`
- `auto`
- `hyper`

Use prefixes sparingly. They often weaken the name if the core root is already clear.

## Root expansion ideas

Expand the user's phrase into:

- head term
- synonym
- stem
- spoken variant
- product term
- outcome term

Examples:

- `calculator` -> `calc`, `calcu`, `calculator`, `calcurator`, `compute`
- `writing` -> `write`, `writer`, `draft`, `scribe`, `copy`
- `meeting notes` -> `note`, `memo`, `brief`, `recap`, `minutes`
- `resume` -> `resume`, `cv`, `career`, `hire`, `job`
- `research` -> `research`, `intel`, `insight`, `signal`, `scan`

## Scoring heuristic

Prefer candidates with:

- 6 to 14 characters before the TLD
- one or two semantic chunks
- easy pronunciation
- obvious product fit
- limited ambiguity

Penalize candidates with:

- 3 or more chunks
- forced spelling
- repeated letters that reduce clarity
- weak search relevance
- resemblance to famous product brands

## Recommended process

1. Generate 30 to 100 candidates quickly.
2. Cut down to 8 to 12 high-quality names.
3. Check live availability.
4. Return the best 3 to 5 confirmed options.
