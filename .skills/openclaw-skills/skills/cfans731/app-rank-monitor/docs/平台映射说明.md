## 📝 文件命名规则

### 格式
```
{platform}_new_apps_{YYYYMMDD_HHMMSS}.xlsx
```

### 示例
- `appstore_cn_new_apps_20260324_161930.xlsx` - App Store 中国区
- `huawei_new_apps_20260324_162000.xlsx` - 华为
- `hongmeng_new_apps_20260324_162100.xlsx` - 鸿蒙
- `xiaomi_new_apps_20260324_162200.xlsx` - 小米

## 🔧 代码实现

### 平台映射字典

```python
platform_map = {
    # App Store 国家/地区
    'line-1-0-0-75': 'appstore_cn',      # App Store 中国区
    'line-1-0-0-72': 'appstore_us',       # App Store 美国
    'line-1-0-0-81': 'appstore_jp',       # App Store 日本
    'line-1-0-0-82': 'appstore_kr',       # App Store 韩国
    'line-1-0-0-111': 'appstore_uk',      # App Store 英国
    'line-1-0-0-113': 'appstore_de',      # App Store 德国
    'line-1-0-0-115': 'appstore_fr',      # App Store 法国
    # Google Play 国家/地区
    'line-2-0-0': 'googleplay',
    # 国内安卓渠道
    'line-3-0-0-1': 'huawei',             # 华为
    'line-3-0-0-2': 'honor',              # 荣耀
    'line-3-0-0-3': 'xiaomi',             # 小米
    'line-3-0-0-4': 'vivo',               # vivo
    'line-3-0-0-5': 'oppo',               # OPPO
    'line-3-0-0-6': 'meizu',              # 魅族
    'line-3-0-0-7': 'tencent',            # 应用宝
    'line-3-0-0-8': 'baidu',              # 百度
    'line-3-0-0-9': 'qihoo360',           # 360
    'line-3-0-0-10': 'wandoujia',         # 豌豆荚
    'line-3-0-0-11': 'hongmeng',          # 鸿蒙
    # TapTap
    'line-4-0-0': 'taptap'
}

# 简化的平台映射（用于 URL 匹配）
simple_platform_map = {
    'line-1': 'appstore',
    'line-2': 'googleplay',
    'line-3': 'android',
    'line-4': 'taptap'
}
```

### 平台识别逻辑

```python
# 从 URL 获取平台信息
current_url = page.url
platform = 'appstore'  # 默认

# 优先匹配详细的平台映射
for key, value in platform_map.items():
    if key in current_url:
        platform = value
        break
else:
    # 如果没有匹配到详细的，使用简化的映射
    for key, value in simple_platform_map.items():
        if key in current_url:
            platform = value
            break

# 生成文件名
new_filename = f"{platform}_new_apps_{datetime.now().strftime('%Y%m%d_%H%M%S')}.xlsx"
```
