# 核心工作流

### 前置步骤：安装/升级 FlyAI CLI

在执行任何搜索之前，**必须先执行安装命令**（无论是否已安装，确保为最新版本）：

```bash
npm install -g @fly-ai/flyai-cli@latest --registry=https://registry.npmjs.org
```

> 💡 此命令会自动处理首次安装和版本升级，无需手动判断是否已安装。**安装后验证：**
```bash
flyai --help
```

**安装失败处理：**
| 情况 | 处理方式 |
|-----|---------|
| npm 未安装 | 提示用户先安装 Node.js (https://nodejs.org/) |
| 权限不足 | 建议使用 `sudo npm install -g @fly-ai/flyai-cli@latest --registry=https://registry.npmjs.org` 或使用 nvm 管理 Node |
| 网络问题 | 建议用户检查网络或使用国内镜像 `npm config set registry https://registry.npmmirror.com` |

**注意：** 此步骤只在首次使用时执行，后续调用会直接跳过已安装的情况。

### Step 1: 收集用户输入

使用 `ask_user_question` 工具按以下顺序逐步询问：

#### 1.1 询问目的地

```
问题: "你想去哪个国家？"
选项:
- "日本"
- "韩国"
- "泰国"
- "欧洲申根国"
- "其他（我来输入）"
```

#### 1.2 询问出发日期

```
问题: "你计划什么时候出发？"
选项:
- "1个月内"
- "1-2个月后"
- "2-3个月后"
- "3个月以上"
- "我有具体日期（我来输入）"
```

#### 1.3 询问户籍/签发地

```
问题: "你的护照签发地（户籍）在哪里？（用于确定送签领区）"
选项:
- "上海/江浙沪"
- "北京/华北"
- "广东/华南"
- "其他省份（我来输入）"
```

#### 1.4 询问签证办理偏好

```
问题: "你打算怎么办签证？"
选项:
- "自己办 - 省钱但需要自己跑流程"
- "找代办 - 省心但需要额外费用"
```

#### 1.5 询问职业身份（可选，用于优化材料清单）

```
问题: "你目前的职业身份是？（用于定制材料清单）"
选项:
- "在职人员"
- "自由职业/个体"
- "学生"
- "退休人员"
```

#### 1.6 询问护照情况（可选，用于风险评估）

```
问题: "你的护照使用情况？"
选项:
- "有多次出境记录"
- "有1-2次出境记录"
- "白本护照（从未出境）"
```

**收集完成后**，整理用户信息：
- **目的地**: 国家名称
- **计划出发日期**: 具体日期
- **户籍/护照签发地**: 省份或城市（用于确定送签领区）
- **签证类型偏好**: 自己办 / 找代办
- **职业身份**: 在职/自由职业/学生/退休
- **护照情况**: 是否白本护照（无任何出境记录）

### Step 2: 查询签证信息

使用 flyai 查询目的地签证要求：

```bash
/flyai keyword-search --query "{目的地国家} 签证"
```

> **⚠️ SSL 证书问题**：如果通过终端执行 flyai CLI 命令时遇到 `UNABLE_TO_GET_ISSUER_CERT_LOCALLY` 错误，需要在命令前加上环境变量：
> ```bash
> NODE_TLS_REJECT_UNAUTHORIZED=0 flyai keyword-search --query "{目的地国家} 签证"
> ```

从查询结果中提取：
- 签证办理周期（工作日）
- 签证费用参考
- 代办服务链接

### Step 3: 计算时间线

#### 3.1 签证办理周期参考

| 国家/地区 | 常规办理周期 | 加急周期 | 备注 |
|-----------|-------------|---------|------|
| 日本 | 5-7 工作日 | 3-5 工作日 | 单次旅游签证 |
| 韩国 | 5-6 工作日 | 3-4 工作日 | 单次旅游签证 |
| 泰国 | 3-5 工作日 | 1-2 工作日 | 或落地签 |
| 新加坡 | 3-5 工作日 | - | 电子签证 |
| 马来西亚 | 3-5 工作日 | - | eNTRI/eVISA |
| 欧洲申根 | 10-15 工作日 | - | 需面签预约 |
| 美国 | 1-3 个月 | - | 需面签，周期不确定 |
| 英国 | 15-20 工作日 | 5 工作日 | 需采集指纹 |
| 澳大利亚 | 10-20 工作日 | 5 工作日 | 电子签证 |
| 加拿大 | 2-4 周 | - | 需采集指纹 |

#### 3.2 时间线倒推算法

1. **确定出发日期** (D日)
2. **确定取回护照日期**: D - 3天（预留缓冲）
3. **确定出签日期**: 取回护照日期
4. **确定送签日期**: 出签日期 - 办理周期
5. **确定材料准备截止日期**: 送签日期 - 5~7 工作日
6. **确定启动准备日期**: 材料准备截止日期 - 3~5 工作日

#### 3.3 判断时间是否充裕

```
剩余天数 = 出发日期 - 今天
所需天数 = 材料准备时间 + 办理周期 + 缓冲时间

如果 剩余天数 >= 所需天数 → ✅ 来得及
如果 剩余天数 < 所需天数 但 >= 加急周期 → ⚠️ 需要加急
如果 剩余天数 < 加急周期 → ❌ 常规流程来不及
```

### Step 4: 生成材料清单

根据目的地国家和用户身份，生成材料清单。

#### 4.1 通用材料

- [ ] 护照原件（有效期 ≥ 6个月）
- [ ] 护照首页复印件
- [ ] 2寸白底彩色证件照（近3个月内）
- [ ] 身份证正反面复印件
- [ ] 户口本整本复印件

#### 4.2 财力证明（根据身份调整）

**在职人员**:
- [ ] 在职证明（盖公章）
- [ ] 公司营业执照复印件（盖公章）
- [ ] 近6个月银行流水（余额建议≥10万）

**自由职业者**:
- [ ] 自由职业说明（如有）
- [ ] 近6个月银行流水（余额建议≥15万）
- [ ] 房产证/车辆登记证（补充证明）

**学生**:
- [ ] 学校在读证明
- [ ] 父母在职证明 + 银行流水
- [ ] 亲属关系公证

**退休人员**:
- [ ] 退休证复印件
- [ ] 养老金流水/存款证明

#### 4.3 行程材料

- [ ] 往返机票预订单（可用飞猪先预订，出签后付款）
- [ ] 酒店预订确认单（覆盖全程住宿）
- [ ] 行程计划表（每日行程安排）

### Step 5: 输出时间线卡片

按以下格式输出签证办理时间线：

```
━━━━━━━━━━━━━━━━━━━━
🛂 {目的地}签证·办理时间线
   签证类型：{签证类型}
   预计总耗时：{X-Y}个工作日
   ⏰ 最晚启动日：{日期}（今天是{今天日期}，{✅ 来得及 / ⚠️ 较紧张 / ❌ 来不及}）

📅 Step 1 - 立即开始（{日期}前完成）
  □ 确认护照有效期 ≥ 6个月（即{具体日期}后到期）
  □ 拍证件照（{规格要求}）

📅 Step 2 - 准备材料（{起始日期}-{截止日期}）
  □ {材料1}
  □ {材料2}
  □ ...（详见下方材料清单）

📅 Step 3 - 送签（{起始日期}-{截止日期}）
  送签地址：{根据领区确定}
  工作时间：周一至周五 {时间}

📅 Step 4 - 等待出签（{起始日期}-{预计出签日期}）
  通常{X-Y}个工作日

📅 Step 5 - 取回护照（{预计日期}）
  出签后领取或快递寄回

⚠️ 风险提示：
  - {根据用户情况给出的风险点}
  - {补充材料建议}
━━━━━━━━━━━━━━━━━━━━
👉 下一步：{引导性问题}
```

### Step 6: 查询机票酒店参考

提供预订单生成帮助：

```bash
/flyai search-flight --origin "{出发城市}" --destination "{目的地}" --dep-date {出发日期} --back-date {返回日期}
```

```bash
/flyai search-hotel --dest-name "{目的地}" --check-in-date {入住日期} --check-out-date {退房日期}
```
