# 场景专项能力

本技能整合了以下专项能力，根据用户需求自动调用：

| 场景触发词 | 调用能力 | 功能 |
|-----------|---------|------|
| 目的地 PK | `flyai-destination-pk` | 2-3个目的地全维度对比 |
| 低价日历、哪天飞便宜 | `flyai-flight-calendar` | 扫描日期范围找最低价 |
| 选酒店、酒店对比 | `flyai-hotel-picker` | 精选3家酒店对比决策 |
| 带小孩/老人/闺蜜 | `flyai-companion-match` | 同行人适配景点推荐 |
| 拼假、请假方案 | `flyai-vacation-planner` | 最优拼假日历计算 |
| 签证怎么办 | `flyai-visa-timeline` | 签证时间线与材料清单 |
| 行李清单、带什么 | `flyai-packing-list` | 场景化智能装箱单 |
| 周末去哪 | `flyai-weekend-trip` | 2天1晚说走就走方案 |
| 我有X元能去哪 | `flyai-reverse-budget` | 反向预算搜索3档方案 |
| XX平替、去不起XX | `flyai-destination-substitute` | 梦想目的地平替方案 |
| 帮我说服TA | `flyai-persuade-ta` | 用数据说服决策人 |
| 旅行盲盒、帮我抽 | `flyai-travel-blindbox` | 随机惊喜目的地 |
| **中转能玩吗、转机时间长** | `flyai-transit-tour` | 中转城市半日游攻略 |
| **同款打卡、圣地巡礼** | `flyai-pilgrimage-tour` | 影视/动漫取景地打卡路线 |
| **检查行程、行程有没有问题** | `flyai-trip-checker` | 行程体检诊断+优化建议 |
| **同城不同价、哪个机场便宜** | `flyai-multi-airport-radar` | 多机场价格矩阵对比 |
| **旅伴测试、和XX一起旅行** | `flyai-companion-matcher` | 旅伴风格匹配度报告 |
| **现在买划算吗、会不会降价** | `flyai-price-advisor` | 价格水位评估+买/等建议 |
| **现在出发、说走就走** | `flyai-instant-departure` | X小时内能到哪的极限方案 |

> ⚠️ 所有 flyai 命令执行前需加 `NODE_TLS_REJECT_UNAUTHORIZED=0` 绕过 SSL 验证

## 场景1: 目的地纠结 → 调用 PK 对比

**触发词**："XX 还是 YY"、"纠结去哪"、"选哪个目的地"、"PK"

**核心调用**：

```bash
# 1. 为每个候选目的地搜索机票
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-flight \
  --origin "[出发城市]" --destination "[目的地A]" \
  --dep-date [出发日期] --back-date [返回日期] --sort-type 3

# 2. 为每个候选目的地搜索酒店
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-hotels \
  --dest-name "[目的地A]" --check-in-date [入住日期] \
  --check-out-date [退房日期] --sort rate_desc

# 3. 为每个候选目的地搜索景点
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-poi --city-name "[目的地A]"
```

**输出格式**：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🆚 [目的地A] vs [目的地B] · 全维度PK

                    [目的地A]        [目的地B]

🏨 酒店（每晚）    ¥[低]-[高]       ¥[低]-[高]
📍 景点数          [X]个            [X]个
💰 预估总花费      ¥[低]-[高]       ¥[低]-[高]

🏆 综合推荐：
   [用户偏好] → ✅ [胜出目的地]（[理由]）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 场景4: 请假规划 → 调用拼假日历

**触发词**："拼假"、"凑假"、"怎么请假划算"、"假期规划"

**信息收集**：
- 目的地
- 可请假天数上限
- 出发城市

**拼假算法**：
1. 识别周末和法定假日
2. 合并相邻休息日
3. 在间隙插入请假日
4. 按 (旅行天数/请假天数) 排序

**核心调用**：

```bash
# 为每个拼假方案查询机票
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-flight \
  --origin "[出发城市]" --destination "[目的地]" \
  --dep-date [方案出发日期] --back-date [方案返回日期] \
  --sort-type 3
```

**输出格式**：
```
━━━━━━━━━━━━━━━━━━━━
📋 方案A（推荐⭐）
  请假：[日期]（周几）
  旅行时段：[起始] - [结束]，共[X]天[Y]晚
  机票参考：往返 ¥[价格]
  标签：🟢 淡季 | 性价比最高
  👉 [引导操作]
━━━━━━━━━━━━━━━━━━━━
```

## 场景9: 梦想太远 → 调用平替旅行

**触发词**："想去XX但太贵"、"XX平替"、"找平替"、"去不起XX"、"类似XX的地方"、"XX太远了"、"XX签证难"

**核心理念**：
不是简单推荐"便宜目的地"，而是理解用户向往这个目的地的**本质是什么**（体验元素），然后找能满足这些元素的平替。

**体验元素拆解示例**：

| 梦想目的地 | 核心体验元素 |
|-----------|------------|
| 瑞士 🇨🇭 | 🏔 雪山草甸 + 🚂 景观列车 + 🏘 精致小镇 + 🐄 牧场田园 |
| 马尔代夫 🏝 | 🏖 水清沙白 + 🏠 水上屋 + 🤿 浮潜潜水 + 🌅 海上日落 |
| 圣托里尼 🇬🇷 | 🏛 蓝白建筑 + 🌅 悬崖日落 + 🍷 浪漫氛围 |
| 冰岛 🇮🇸 | 🌌 极光 + 🧊 冰川 + 🌋 火山地热 + 🛤 荒野公路 |

**核心调用**：

```bash
# 1. 根据体验元素搜索平替目的地
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai keyword-search \
```

## 场景12: 中转不浪费 → 调用 flyai-transit-tour

**触发词**：“中转航班”、“转机时间长”、“中转能玩吗”、“中转顺便逛”、“省钱中转”

**核心理念**：
大多数人把中转时间视为浪费，但6-15小时的中转，那个中转城市本身就是一个免费的旅行目的地。

**信息收集**：
- 出发地 → 最终目的地
- 出行日期、返程日期

**输出内容**：
- 筛选6-15小时中转的航班方案
- 中转城市半日游攻略
- 时间安全垫计算（确保不误机）
- 与直飞的价格对比
