# Memory 系统

## 记忆内容结构

使用 `search_memory` 和 `update_memory` 工具管理用户画像，记忆内容包括：

```yaml
用户旅行画像:
  人设风格: "暴躁老哥 | 元气萌妹 | 资深玩家 | 专属管家"
  出发城市: "[用户出发城市]"
  常见同行人:
    - 类型: "[同行人类型]"
      特征: "[特征描述]"
  旅行偏好:
    喜欢: ["[偏好1]", "[偏好2]"]
    不喜欢: ["[不喜欢1]", "[不喜欢2]"]
  预算偏好: "[预算范围]"
  住宿偏好: "[住宿类型]"
  历史目的地: ["[目的地1]", "[目的地2]"]
```

## 何时读取 Memory

**启动时自动读取**（除非用户明确说"忽略偏好"、"重新选风格"等）：

```
技能启动
    ↓
检测用户是否说了"忽略风格/重新选择/换个风格"
    ↓
├─ 是 → 跳过 memory，重新让用户选择人设
└─ 否 → 调用 search_memory 读取用户画像
           ↓
       ├─ 有记录 → 使用已保存的人设风格，直接开始对话
       │           并在推荐时自动应用偏好
       └─ 无记录 → 首次用户，让用户选择人设
```

**读取 Memory 的调用方式**：

```python
search_memory(
  query="用户旅行偏好和人设风格",
  keywords="旅行,人设,偏好,同行人,出发城市",
  category="user_hobby,user_info",
  depth="shallow"
)
```

## 何时更新 Memory

在以下时机调用 `update_memory` 更新用户画像：

| 触发场景 | 更新内容 | 示例 |
|---------|---------|------|
| 用户选择人设风格 | `人设风格` | 用户选了"暴躁老哥" |
| 用户提到出发城市 | `出发城市` | "我从XX出发" |
| 用户提到同行人 | `常见同行人` | "带XX一起" |
| 用户表达喜好 | `旅行偏好.喜欢` | "想去海边" |
| 用户表达不喜欢 | `旅行偏好.不喜欢` | "不想爬山太累" |
| 用户确认预算 | `预算偏好` | "预算3000左右" |
| 用户选择酒店类型 | `住宿偏好` | "想住亲子酒店" |
| 完成一次规划 | `历史目的地` | 成功规划了某地之旅 |

**更新 Memory 的调用方式**：

```python
update_memory(
  action="update",  # 或 "create" 首次
  category="user_hobby",
  title="旅行偏好与风格",
  content="人设风格: [风格]\n出发城市: [城市]\n常见同行人: [同行人]\n喜欢: [喜好]\n不喜欢: [不喜欢]",
  keywords="旅行,人设,偏好"
)
```

## Memory 应用示例

**场景：老用户再次触发技能**

```
用户: "这周末想出去玩"

系统内部流程:
1. search_memory → 发现用户画像
2. 读取到已存储的人设风格和偏好

AI回复（使用已存人设风格）:
根据用户选择的人设风格，用对应语气打招呼，并基于偏好主动推荐
```

**场景：用户想换风格**

```
用户: "换个风格，我想要萌妹版"

系统内部流程:
1. 检测到"换个风格" → 不读取旧人设
2. 更新 memory: 人设风格 = 元气萌妹

AI回复（元气萌妹风格）:
切换为新风格进行对话
```

## 版本兼容性

**数据版本管理**：
- 当前版本：v1.0
- 升级策略：向后兼容，新增字段使用默认值

**版本迁移示例**：
```python
# v1.0 → v1.1 迁移
if "住宿偏好" not in user_profile:
    user_profile["住宿偏好"] = "不限"
```

## 数据清理策略

**历史目的地上限**：
- 最多保留 20 个历史目的地
- 超出时删除最早的记录

**过期偏好清理**：
- 超过 1 年未更新的偏好标记为"待确认"
- 下次对话时主动询问是否仍然有效
