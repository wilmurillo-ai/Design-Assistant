# 自主探索思维框架

当遇到预设场景之外的需求时，按以下思路处理。

## 思维流程

```
1. 理解意图
   └─ 用户真正想解决什么问题？
   └─ 背后的情感需求是什么？

2. 拆解任务
   └─ 这个问题可以分解为哪些子问题？
   └─ 哪些子问题可以用现有能力解决？

3. 能力匹配
   └─ 优先使用 FlyAI 搜索命令
   └─ 不够？结合 AI 通识知识
   └─ 还不够？fetch_content 获取实时信息

4. 方案构建
   └─ 将各部分结果整合为完整方案
   └─ 保持输出格式一致性

5. 迭代优化
   └─ 询问用户反馈
   └─ 根据反馈调整方案
```

## 自主探索示例

| 用户需求（预设外） | 自主解决思路 |
|-------------------|-------------|
| "帮我规划蜜月路线" | 组合：目的地推荐 + 酒店（浪漫型）+ 景点（适合情侣） |
| "带宠物能去哪" | AI知识 + keyword-search(宠物友好) + 补充搜索 |
| "团建去哪合适" | 分析团建需求 → 场地+活动+住宿组合方案 |
| "求婚去哪好" | 浪漫目的地 + 特色体验 + 餐厅/酒店推荐 |
| "找个能拍vlog的地方" | 网红打卡点 + 出片景点 + 光线/场景分析 |
| "想去看极光" | 极光目的地 + 最佳季节 + 观测攻略 |
| "毕业旅行推荐" | 理解学生预算 + 出片需求 + 组合多人方案 |
| "一个人散心去哪" | 理解情绪需求 → 安静/自然/疗愈类目的地推荐 |

## 能力组合策略

### 优先级顺序

1. **FlyAI 搜索命令**（核心能力）
   - keyword-search：自然语言搜索
   - search-flight/hotel/poi：结构化搜索

2. **AI 通识知识**（补充）
   - 目的地特点、文化背景
   - 旅行常识、注意事项

3. **fetch_content**（实时补充）
   - 最新政策、开放时间
   - 小众信息、攻略细节

### 组合示例

```
用户："想去个小众海岛，人少景美"

思路：
1. keyword-search("小众海岛 人少") → 获取候选
2. 如果结果不够 → AI 知识补充冷门海岛
3. 确定目的地后 → search-flight + search-hotel
4. 需要详细攻略 → fetch_content 抓取游记
```

## 自主学习机制

- **记住用户偏好**：不喜欢爬山、预算敏感、偏爱海岛...
- **积累解决方案**：每次新场景都在丰富经验库
- **学习用户反馈**：用户说"不太对"时，理解原因并调整
- **组合创新**：将预设能力灵活组合，应对复杂需求
- **主动服务**：关联推荐、风险提示、超预期服务
