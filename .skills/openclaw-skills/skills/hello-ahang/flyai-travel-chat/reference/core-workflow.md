# 核心工作流

### 前置步骤：确保 FlyAI CLI 为最新版本

在执行任何搜索之前，必须先确保 FlyAI CLI 已安装且为最新版本。

**安装/升级 FlyAI CLI：**
```bash
npm install -g @fly-ai/flyai-cli@latest --registry=https://registry.npmjs.org
```

> 💡 此命令会自动处理首次安装和版本升级，无需手动判断是否已安装。

**验证安装成功：**
```bash
flyai --help
```

**安装失败处理：**
| 情况 | 处理方式 |
|-----|---------|
| npm 未安装 | 提示用户先安装 Node.js (https://nodejs.org/) |
| 权限不足 | 建议使用 `sudo npm install -g @fly-ai/flyai-cli@latest --registry=https://registry.npmjs.org` 或使用 nvm 管理 Node |
| 网络问题 | 建议用户检查网络，确保可以访问 registry.npmjs.org |

### 阶段1: 需求发散 (Discovery)

用户说任意自然语言，你通过 FlyAI 搜索 + 追问 逐步明确需求。

**1.1 解析用户意图**

从用户输入中提取关键信息：
- **时间线索**：什么时候出发？几天？
- **地点线索**：有没有明确方向（海边/山里/国内/出境）？
- **人员线索**：谁一起？（独自/情侣/家庭/朋友）
- **预算线索**：有没有提到预算？
- **偏好线索**：想要什么体验？（发呆/刺激/美食/拍照）
- **特殊需求**：有没有非常规需求？（宠物、无障碍、特殊饮食等）

**1.2 针对性追问**

如果信息不完整，使用 `ask_user_question` 进行趣味二选一追问：

```
问题: "先帮你划定方向，你更想："
选项:
- "海边吹风 🏖️ — 沙滩、海浪、日落"
- "山里避暑 🏔️ — 清凉、徒步、自然"
- "城市探索 🏙️ — 美食、购物、文化"
- "都可以，帮我推荐"
```

**1.3 首轮搜索推荐**

基于收集到的信息，调用 FlyAI 搜索：

```bash
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai keyword-search --query "[用户描述关键词]"
```

输出候选目的地，格式：

```
根据你的描述，帮你锁定了3个方向：

🏔️ 方向A：[目的地]
   [特点描述] | [预算参考]
   
🏖️ 方向B：[目的地]
   [特点描述] | [预算参考]
   
🌆 方向C：[目的地]
   [特点描述] | [预算参考]

你对哪个更感兴趣？或者告诉我更多偏好
```

### 阶段2: 需求聚焦 (Refinement)

用户选择方向后，调用结构化搜索获取具体数据。

**2.1 锁定目的地后的信息收集**

使用 `ask_user_question` 收集：出发城市、天数、同行人等。

**2.2 并行搜索**

确定目的地后，并行调用多个搜索：

```bash
# 搜索机票
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-flight \
  --origin "[出发城市]" --destination "[目的地]" \
  --dep-date [出发日期] --back-date [返回日期] --sort-type 3

# 搜索酒店
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-hotel \
  --dest-name "[目的地]" --check-in-date [入住日期] \
  --check-out-date [退房日期] --sort rate_desc

# 搜索景点
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-poi --city-name "[目的地]"
```

**2.3 输出初步方案**

```
好的！[目的地][天数]天方案框架出来了：

✈️ 机票：[出发城市]⇄[目的地]
   去：[日期] [航班] ¥[价格]
   回：[日期] [航班] ¥[价格]
   💡 [省钱提示]

🏨 住宿初筛：
   [酒店名称] ¥[价格]/晚（[亮点]）
   
📍 核心景点：
   [景点1] | [景点2] | [景点3]

要我输出完整行程方案吗？还是先调整哪部分？
```

### 阶段3: 方案输出 (Delivery)

生成完整可执行方案。

**3.1 完整方案格式**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━
🌴 [目的地][天数]天[晚数]晚·[主题描述]

Day1 ([日期] [周几])：[城市]→[目的地]
  ✈️ [航班] [时间] 到[机场]
  🚗 [机场]→[酒店]（[交通方式]约[时间]）
  🌙 入住[酒店]，[晚间活动建议]

Day2 ([日期])：[当日主题]
  📍 [景点]——[一句话描述+游玩建议]
  💡 [实用提示]
  🌙 [住宿]

...

💰 费用估算（[人数]人总计）：
  机票：¥[金额]
  酒店：¥[金额]（[晚数]晚）
  景点门票：约¥[金额]
  交通+餐饮：约¥[金额]
  ──────────
  合计：约¥[总金额]（人均¥[人均]）
━━━━━━━━━━━━━━━━━━━━━━━━━━━

👉 [预订机票] [预订酒店] [查看景点详情]
```

**3.2 提取预订链接**

从 FlyAI 返回数据中提取 `jumpUrl` 字段，这是飞猪直接预订链接。
