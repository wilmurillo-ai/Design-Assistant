# 核心工作流

### 前置步骤：安装/升级 FlyAI CLI

在执行任何搜索之前，**必须先执行安装命令**（无论是否已安装，确保为最新版本）：

```bash
npm install -g @fly-ai/flyai-cli@latest --registry=https://registry.npmjs.org
```

> 💡 此命令会自动处理首次安装和版本升级，无需手动判断是否已安装。**安装后验证：**
```bash
flyai --help
```

**安装失败处理：**
| 情况 | 处理方式 |
|-----|---------|
| npm 未安装 | 提示用户先安装 Node.js (https://nodejs.org/) |
| 权限不足 | 建议使用 `sudo npm install -g @fly-ai/flyai-cli@latest --registry=https://registry.npmjs.org` 或使用 nvm 管理 Node |
| 网络问题 | 建议用户检查网络或使用国内镜像 `npm config set registry https://registry.npmmirror.com` |

**注意：** 此步骤只在首次使用时执行，后续调用会直接跳过已安装的情况。

### Step 1: 参数确认

确认以下必需参数：
| 参数 | 示例 | 必填 |
|------|------|------|
| 出发城市 | 上海 | ✅ |
| 目的地 | 大阪 | ✅ |
| 日期范围 | 7月15日-7月31日 | ✅ |
| 行程天数 | 5天 | ✅ |
| 只看直飞 | 是/否 | ❌ |
| 时间偏好 | 早/晚/无要求 | ❌ |

### Step 2: 批量查询机票价格

使用 flyai 的 `search-flight` 命令扫描每天的机票价格。

**单日查询命令**：

```bash
NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-flight \
  --origin "{出发城市}" \
  --destination "{目的地}" \
  --dep-date {出发日期} \
  --back-date {返回日期} \
  --journey-type {1=直飞/不填=含中转} \
  --sort-type 3
```

**参数说明**：
| 参数 | 说明 |
|------|------|
| `--origin` | 出发城市（必填） |
| `--destination` | 目的地城市 |
| `--dep-date` | 出发日期 (YYYY-MM-DD) |
| `--back-date` | 返回日期 (YYYY-MM-DD) |
| `--journey-type` | 1=只看直飞，2=只看中转，不填=全部 |
| `--sort-type` | 3=价格从低到高 |
| `--dep-hour-start/end` | 出发时间范围 (0-23) |
| `--max-price` | 价格上限 |

> **⚠️ SSL 证书问题**：执行 flyai CLI 命令时，需要在命令前加上 `NODE_TLS_REJECT_UNAUTHORIZED=0` 来绕过 SSL 证书验证。

**批量扫描策略**：

1. 根据日期范围计算需要扫描的出发日期列表
2. 对每个出发日期，计算对应的返程日期（出发日期 + 行程天数）
3. 逐日调用 `search-flight` 获取最低价
4. 记录每天的最低价和对应航班信息

**优化提示**：
- 日期范围超过14天时，可以先每隔2-3天采样，找到低价区间后再细查
- 使用 `--sort-type 3` 确保返回结果按价格排序，取第一条即为最低价

### Step 3: 生成低价日历

将扫描结果整理为可视化日历格式：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📅 {出发城市}⇄{目的地} · {月份}低价日历（往返{筛选条件}）

{月份}
┌───┬───┬───┬───┬───┬───┬───┐
│ 一 │ 二 │ 三 │ 四 │ 五 │ 六 │ 日 │
├───┼───┼───┼───┼───┼───┼───┤
│ {日}│ {日}│ {日}│ {日}│ {日}│ {日}│ {日}│
│{价格}│{价格}│{价格}│{价格}│{价格}│{价格}│{价格}│
│ {标}│ {标}│ {标}│ {标}│ {标}│ {标}│ {标}│
└───┴───┴───┴───┴───┴───┴───┘

🟢 最低价  🟡 相对低价  ⬜ 正常价  （无标=偏高）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**价格分级规则**：
- 🟢 最低价：价格最低的1-2天
- 🟡 相对低价：价格低于平均值10%以上
- ⬜ 正常价：接近平均值
- 无标记：高于平均值10%以上

### Step 4: 输出推荐方案

在日历下方输出具体推荐，**必须包含飞猪预订链接**：

**从 API 返回数据中提取 `jumpUrl` 字段**，这是飞猪的直接预订链接，格式如 `https://a.feizhu.com/xxxxx`。

```
🏆 推荐方案：

| 方案 | 出发日期 | 往返价格 | 航班类型 | 预订链接 |
|------|----------|----------|----------|----------|
| 最省钱 | {日期}({周几}) | ¥{价格} | {直飞/中转} | [立即预订]({jumpUrl}) |
| 次优选 | {日期}({周几}) | ¥{价格} | {直飞/中转} | [立即预订]({jumpUrl}) |

✅ 最省钱方案详情：
  {日期}({周几})出发 → {返程日期}返回
  往返含税：¥{价格}/人
  比最贵日期({最贵日期})省 ¥{差价}！
  去程：{航班号} {时间}（{航空公司}）
  回程：{航班号} {时间}（{航空公司}）
  👉 [点击预订]({jumpUrl})
     
💡 规律提示：
  {发现的价格规律，如"周二/三出发最便宜，周五/六/日贵30%-50%"}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**重要**：每个航班方案的 `jumpUrl` 是唯一的预订入口，务必在输出中展示，方便用户一键跳转飞猪下单。

### Step 5: 提供下一步行动

```
👉 下一步：
  [锁定{推荐日期}出发·查看详细航班] [同步搜这几天的酒店价格]
```

引导用户进行后续操作：

1. **查看详细航班信息**：
   ```bash
   NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-flight --origin "{出发城市}" --destination "{目的地}" --dep-date {推荐日期} --back-date {返程日期}
   ```

2. **搜索酒店价格**：
   ```bash
   NODE_TLS_REJECT_UNAUTHORIZED=0 flyai search-hotel --dest-name "{目的地}" --check-in-date {入住日期} --check-out-date {退房日期}
   ```

3. **搜索机+酒套餐**：
   ```bash
   NODE_TLS_REJECT_UNAUTHORIZED=0 flyai keyword-search --query "{目的地} 机票酒店套餐 {日期}"
   ```
