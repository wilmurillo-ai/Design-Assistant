## 任务设计原则

### 核心理念

任务设计应遵循 **"学以致用、AI 协作"** 的原则：

1. **真实场景导向**：任务应来源于真实工程需求
2. **渐进式难度**：从基础操作到复杂集成，逐步提升
3. **AI 辅助明确**：清晰标注哪些环节可借助 AI 工具
4. **可验证结果**：提供明确的验收标准

### 难度分级

| 级别 | 难度描述 | 预计耗时 | 适用对象 |
|-----|---------|---------|---------|
| ⭐ | 基础任务，修改参数或简单逻辑 | 30分钟-1小时 | ROS2 初学者 |
| ⭐⭐ | 中级任务，添加新节点或功能 | 2-4小时 | 有 ROS2 基础 |
| ⭐⭐⭐ | 高级任务，多模块集成 | 1-2天 | 有项目经验 |
| ⭐⭐⭐⭐ | 挑战任务，AI 模型集成 | 数天-数周 | 进阶开发者 |

## 任务示例

### 任务一：机械臂关节控制器参数调优

**难度**：⭐

**所属项目**：SO-ARM100

#### 任务目标
调整机械臂关节控制器的 PID 参数，使关节运动更平滑、响应更快。

#### 涉及 ROS2 知识点
- **参数声明与获取**：学习如何在节点中使用参数
- **ros2_control 框架**：理解控制器参数文件的结构
- **YAML 配置文件**：学习 ROS2 的配置管理方式

#### 前置条件
- [ ] ROS2 Humble 环境已配置
- [ ] 已安装 ros2_control 相关包
- [ ] 已克隆 SO-ARM100 项目代码
- [ ] Gazebo 仿真环境可正常运行

#### 任务步骤

##### 步骤 1：查看当前控制器参数
1. 启动仿真环境
   ```bash
   ros2 launch so_arm100 simulation.launch.py
   ```
2. 查看当前参数
   ```bash
   ros2 param list /joint_state_broadcaster
   ros2 param get /joint_trajectory_controller gains.joint1.p
   ```

##### 步骤 2：修改参数配置文件
1. 打开控制器配置文件：`config/controllers.yaml`
2. 找到 PID 参数部分
3. 尝试调整参数值（建议范围：P: 10-100, I: 0.01-0.1, D: 0.01-0.5）

##### 步骤 3：验证效果
1. 重启仿真
2. 观察关节运动是否更平滑
3. 检查是否有超调或震荡

#### AI 辅助建议
- **代码生成**：可让 AI 帮助生成参数调试脚本
- **调试排错**：若出现震荡或不稳定，可将错误日志发给 AI 分析
- **参数建议**：可询问 AI 关于 PID 参数调优的经验法则

#### 验收标准
- [ ] 机械臂可平稳运动到目标位置
- [ ] 无明显超调或震荡
- [ ] 响应时间在合理范围内（<2秒）

#### 扩展挑战
- 为每个关节设置不同的 PID 参数
- 添加动态参数调整功能（通过服务实时修改参数）
- 编写自动调参脚本

#### 参考资料
- ros2_control 文档：https://control.ros.org/master/index.html
- PID 控制原理：https://en.wikipedia.org/wiki/PID_controller

### 任务二：添加语音控制机械臂功能

**难度**：⭐⭐⭐

**所属项目**：reBot-DevArm

#### 任务目标
为机械臂添加语音控制功能，用户可通过语音指令控制机械臂移动到预设位置。

#### 涉及 ROS2 知识点
- **服务通信**：创建自定义服务类型
- **动作通信**：使用 MoveIt2 的动作接口
- **消息定义**：创建自定义消息类型
- **多节点协作**：语音识别节点与控制节点的协作

- [ ] ROS2 Humble 环境已配置

##### 步骤 4：集成与测试
1. 启动机械臂控制
2. 启动语音识别节点
3. 测试语音指令控制

#### AI 辅助建议
- **代码生成**：可让 AI 帮助生成语音识别和命令解析的代码框架

### 任务三：自然语言导航控制

**难度**：⭐⭐⭐⭐

**所属项目**：lerobot-ros

#### 任务目标
结合 LLM（如 GPT-4、Claude），让机器人能够理解自然语言导航指令并自主导航。

#### 涉及 ROS2 知识点
- **Nav2 导航栈**：理解导航架构和接口
- **动作客户端**：调用 NavigateToPose 动作
- **服务通信**：与 LLM 服务交互
- **坐标变换**：处理地图坐标与语义位置的映射

#### 前置条件
- [ ] ROS2 Humble 环境已配置
- [ ] Nav2 已安装并配置
- [ ] 已获取 LLM API 密钥
- [ ] lerobot-ros 项目可正常运行
- [ ] 地图已建立

#### 任务步骤

##### 步骤 1：创建位置语义数据库
定义语义位置与坐标的映射：
```yaml
locations:
  kitchen:
    x: 2.5
    y: 1.0
    theta: 0.0
  bedroom:
    x: -1.5
    y: 3.0
    theta: 1.57
  entrance:
    x: 0.0
    y: 0.0
    theta: 0.0
```

##### 步骤 2：创建 LLM 接口节点
```python
class LLMNavigationNode(Node):
    def __init__(self):
        super().__init__('llm_navigation_node')
        # 订阅自然语言指令
        self.nl_sub = self.create_subscription(
            String, '/natural_language_command', 
            self.command_callback, 10)
        # 创建 Nav2 动作客户端
        self.nav_client = ActionClient(self, NavigateToPose, 'navigate_to_pose')
```

##### 步骤 3：实现指令解析逻辑
- 将自然语言指令发送给 LLM
- LLM 返回结构化的导航目标
- 调用 Nav2 执行导航

##### 步骤 4：集成与测试
测试指令示例：
- "去厨房"
- "我想去卧室"
- "带我去门口"

#### AI 辅助建议
- **Prompt 设计**：让 AI 帮助设计 LLM 的 prompt，使其返回结构化 JSON
- **Nav2 接口**：询问 AI Nav2 动作客户端的使用方法
- **错误处理**：让 AI 帮助设计导航失败后的重试逻辑

#### 验收标准
- [ ] 可理解至少 10 种自然语言导航指令
- [ ] 指令解析成功率 >90%
- [ ] 导航成功率 >80%
- [ ] 有语音或文字反馈当前状态

#### 扩展挑战
- 支持相对位置指令（"向前走 3 米"）
- 支持多点路径规划（"先去厨房再去卧室"）
- 添加动态避障和路径重规划

#### 参考资料
- Nav2 文档：https://navigation.ros.org/
- OpenAI API：https://platform.openai.com/docs/
- Claude API：https://docs.anthropic.com/

## AI 辅助指南

### 推荐的 AI 工具

| 工具 | 适用场景 | 使用建议 |
|------|---------|---------|
| **Claude** | 代码生成、架构设计、文档解读 | 适合复杂逻辑和长代码生成 |
| **ChatGPT** | 快速问答、代码片段生成 | 适合简单问题和即时反馈 |
| **GitHub Copilot** | 实时代码补全 | 适合编码过程中的辅助 |
| **Cursor** | IDE 集成 AI 编程 | 适合完整项目开发 |

### 使用 AI 的最佳实践

1. **提供清晰上下文**：
   - 说明项目背景和目标
   - 提供相关代码片段
   - 描述预期行为

2. **迭代式开发**：
   - 先让 AI 生成基础框架
   - 逐步完善细节
   - 让 AI 帮助审查和优化

3. **验证 AI 输出**：
   - 检查生成的代码是否正确
   - 在仿真环境中测试
   - 对比官方文档确认语法

4. **学习而非依赖**：
   - 理解 AI 生成的代码原理
   - 查阅相关文档深入学习
   - 尝试自己修改和扩展

### 示例：如何向 AI 提问

❌ **不好的提问**：
```
帮我写一个机械臂控制节点
```

✅ **好的提问**：
```
我正在使用 ROS2 Humble 和 MoveIt2 开发一个 6 自由度机械臂控制节点。
机械臂型号是 SO-ARM100，我已经有了 MoveIt2 配置包。

我需要：
1. 一个 Python 节点，提供服务接口接收目标位置（x, y, z）
2. 调用 MoveIt2 的运动规划服务
3. 执行规划结果并返回成功/失败状态

请帮我生成代码框架，并说明如何集成到现有项目。
```
