# 机器人论文推文撰写 Skill

## 任务目标

为"Mbot具身智能实验室"撰写深度技术推文，具备以下核心能力：
- 多源检索：整合论文、代码、项目主页、演示视频
- 核心拆解：提取 SOTA 指标、创新架构、硬件配置
- 技术溯源：识别研究团队，梳理技术演进脉络
- 深度撰写：生成结构化、硬核且易读的推文
- **自动截图**：自动采集论文/项目主页图片并插入推文

## 触发条件

- 用户提供论文标题或 Arxiv ID
- 用户需要机器人/具身智能领域的技术分析文章
- 用户希望梳理某个研究团队的技术演进脉络

## 工作流程

### 步骤 1：多源检索

**执行要点**：
1. 使用 `web_search` 工具搜索论文原文（Arxiv、官方 PDF）
2. 搜索 GitHub 代码仓库（关键词：论文标题 + github）
3. 搜索项目主页（关键词：论文标题 + project page）
4. 搜索演示视频（YouTube、Bilibili）
5. 记录所有有效链接，供后续资源导航使用

**输出**：收集论文、代码、项目主页、视频的完整资源列表

### 步骤 1.5：自动截图采集（新增）

**触发条件**：用户要求自动采集论文图片或推文中需要插入图片

**执行要点**：
1. 加载 `playwright-browser-automation` skill 获取浏览器自动化能力
2. 使用 Playwright 访问论文的 arXiv HTML 版本（如 `https://arxiv.org/html/2604.00202`）
3. 自动采集论文中的 Figure 图片（通过 `figure` 元素定位）
4. 尝试从论文页面提取并访问项目主页，采集项目主页截图
5. 将图片保存到 `paper_imgs/` 目录

**自动化脚本**：使用 [scripts/capture_imgs.js](scripts/capture_imgs.js) 脚本：
```bash
cd <工作目录>
npm install playwright
npx playwright install chromium
node scripts/capture_imgs.js <arXiv_ID>
```

**输出**：采集的图片保存在 `paper_imgs/` 目录，可直接在推文中引用

### 步骤 2：核心拆解

**执行要点**：
1. 阅读论文摘要和引言，识别研究问题
2. 提取核心创新点（架构设计、训练策略、损失函数等）
3. 记录 SOTA 指标（成功率、泛化能力、推理速度）
4. 记录硬件配置（机器人型号、传感器、GPU）
5. 标注关键图示位置（Figure 1 架构图、实验对比图）

**输出**：结构化的论文核心信息摘要

### 步骤 3：技术溯源

**执行要点**：
1. 识别论文所属团队（查看作者机构）
2. 搜索该团队的系列工作（使用 `web_search`）
3. 梳理技术演进脉络：
   - 该团队的前序工作有哪些？
   - 本工作相比前作有哪些改进？
   - 引用了哪些关键方法？
4. 构建技术对比表格

**输出**：技术演进路线图和对比分析

参考：[research-teams.md](references/research-teams.md) 中列出了主流团队的研究脉络

### 步骤 4：深度撰写

按照以下结构生成推文（详见 [paper-structure.md](references/paper-structure.md)）：

#### 4.1 标题生成
生成 3 个备选标题，格式示例：
- "[团队名]新作：突破具身智能泛化瓶颈，[论文缩写]开源代码已收录"
- "[技术关键词]首次落地机器人，[论文简称]刷新 SOTA"
- "从[前作]到[新作]，[团队名]如何解决[核心问题]"

#### 4.2 导语（The Hook）
- 用极客感十足的语言点出"破坏力"
- 示例："在[某特定任务]上，我们终于看到了 Scaling Law 的曙光。"
- 控制在 2-3 句话，制造悬念

#### 4.3 硬核科普（Tech Wiki）
- 针对论文中的高频术语进行科普
- 每个术语控制在 200 字以内
- 使用类比和直观解释，避免公式堆砌
- 参考：[tech-terms-glossary.md](references/tech-terms-glossary.md)

#### 4.4 核心突破
按照"痛点 → 方案 → 效果"的逻辑展开：
- **痛点**：为什么之前的方法无法解决这个问题？
- **方案**：模型是如何设计的？（架构创新、训练技巧）
- **效果**：实验证明了什么？
- **插入图示**：[此处插入论文 Figure 1：模型总体架构图]

#### 4.5 实验表现与视频演示
- 描述机器人在各种环境下的表现（扰动实验、跨场景泛化）
- 标注视频插入位置：[此处插入机器人执行任务的 GIF 动图]
- 列出官方视频链接（YouTube/Bilibili）

#### 4.6 技术溯源与关联工作
- **团队脉络**：分析该团队的系列工作
- **技术对比**：对比前作的继承与改进
- **插入图示**：[此处建议插入技术演进路线图或对比表格]

#### 4.7 资源直达
以列表形式给出：
- 论文地址：[Arxiv Link]
- 项目主页：[Project Page URL]
- 代码仓库：[GitHub Repo URL]（若未公开，注明"Code Coming Soon"）

### 步骤 5：多媒体锚点标注与自动插入

**执行要点**：
1. 在推文中标注需要插入图片/视频的位置
2. 精准描述图片内容（例如："展示机器人在不同光照下抓取透明物体的对比图"）
3. 标注图片来源（论文 Figure、项目主页截图）
4. 标注视频片段的时间点（例如："建议截取 0:15-0:30 的抓取演示"）
5. **自动插入图片**：采集图片后，将图片复制到推文同目录，并更新推文中的图片路径为直接引用

**自动插入图片的工作流**：
1. 执行截图脚本后，将图片从 `paper_imgs/` 复制到推文所在目录
2. 在推文中用 Markdown 语法引用图片：
   ```markdown
   ![Figure说明](arxiv_fig_1.png)
   ```
3. 注意：图片路径使用相对路径，且与推文文件在同一目录

**输出**：带有可显示图片的完整推文

### 步骤 6：最终校验

**检查清单**：
- [ ] 所有实验指标与原文一致
- [ ] 术语科普能让大二相关专业学生听懂
- [ ] 标题、导语、正文风格统一
- [ ] 资源链接完整且有效
- [ ] 多媒体锚点描述具体
- [ ] 文末已添加引导语："关注 Mbot 具身智能实验室，第一时间追踪机器人前沿干货。"

## 风格指南

### 深度优先
不仅写"是什么"，更要写：
- "为什么这么做"（设计动机）
- "它的局限性"（诚实分析）

### 视觉描述
在提示插图时，描述必须具体：
- ❌ 错误："插入一张实验对比图"
- ✅ 正确："插入一张展示机器人在不同光照条件下（正常光照/暗光/强光）抓取透明物体的成功率对比柱状图"

### 引导关注
文末固定引导语：
> 关注 Mbot 具身智能实验室，第一时间追踪机器人前沿干货。

## 约束条件

1. **数据准确**：所有实验指标必须与原文一致，不得夸大或模糊
2. **代码敏感**：如果代码仓库尚未公开，需注明"Code Coming Soon"
3. **禁止术语堆砌**：术语科普必须通俗易懂，能让大二学生理解
4. **引用规范**：提及他人工作时，需准确标注出处
