## 标题生成规范

### 核心要求
- 生成 3 个备选标题
- 包含：团队名、核心创新点、论文缩写/简称
- 长度：20-35 字符
- 风格：硬核、有信息量、带节奏感

### 三种标题模板

#### 模板 1：团队导向型
```
[团队名]新作：突破[核心问题]，[论文缩写]开源代码已收录
```
示例：
- "DeepMind 新作：突破机器人泛化瓶颈，RT-X 开源代码已收录"
- "Stanford 新作：让机器人学会复杂操作，ACT-2 代码开源"

#### 模板 2：技术导向型
```
[技术关键词]首次落地机器人，[论文简称]刷新 SOTA
```
示例：
- "扩散策略首次落地机器人，Diffusion Policy 刷新 SOTA"
- "世界模型首次驱动机器人，π0 开启具身智能新范式"

#### 模板 3：演进对比型
```
从[前作]到[新作]，[团队名]如何解决[核心问题]
```
示例：
- "从 RT-1 到 RT-2，DeepMind 如何解决机器人跨任务泛化"
- "从 ACT 到 Diffusion Policy，伯克利如何提升操作稳定性"

## 导语写法

### 核心要求
- 极客感十足的语言
- 点出该工作的"破坏力"或"突破性"
- 制造悬念，引发好奇
- 控制在 2-3 句话（50-80 字符）

### 三种导语模板

#### 模板 1：数据冲击型
```
在 [某任务] 上，[本方法] 将成功率从 [前作]% 提升到 [本作]%——我们终于看到了 [某能力] 的曙光。
```

#### 模板 2：范式突破型
```
当业界还在用 [传统方法] 时，[团队名] 证明了 [新方法] 可以让机器人 [某能力]。这不仅是一次架构升级，更是 [某领域] 的范式转移。
```

#### 模板 3：场景代入型
```
想象一下，机器人可以在 [复杂场景] 中 [完成某任务]。[论文简称] 让这个设想成为现实，而且泛化到 [新场景] 也毫无压力。
```

### 示例

> 在跨场景泛化任务上，Diffusion Policy 将成功率从 63% 提升到 97%——我们终于看到了机器人"举一反三"能力的曙光。

> 当业界还在纠结如何设计更好的奖励函数时，DeepMind 证明了 RT-X 可以让机器人直接从大规模数据中学会操作。这不仅是一次架构升级，更是机器人学习范式的转移。

## 术语科普格式

### 核心要求
- 每个术语控制在 200 字以内
- 使用类比和直观解释
- 避免公式和专业术语堆砌
- 目标读者：大二相关专业学生

### 术语科普模板

```markdown
**[术语名称]**
[一句话定义]：[通俗解释]
[类比说明]：就像 [日常生活场景]，[具体类比]
[机器人场景]：在机器人领域，这意味着 [应用价值]
```

### 示例

**Action Chunking（动作分块）**
一句话定义：让机器人一次性预测并执行一段连续动作序列，而不是单步动作。
类比说明：就像你开门时不会分解成"伸手、握把手、旋转、推门"，而是自然地完成整个开门动作。
机器人场景：Action Chunking 让机器人操作更流畅，避免单步决策带来的抖动和犹豫。

**Diffusion Policy（扩散策略）**
一句话定义：用扩散模型生成机器人的动作序列，通过逐步去噪得到最优动作。
类比说明：就像从一团模糊的云雾中，逐渐"雕刻"出清晰的形状。
机器人场景：Diffusion Policy 可以生成多样化的动作，让机器人在复杂任务中表现更稳定。

## 实验表现描述要点

### 核心要求
- 描述机器人在各种环境下的表现
- 突出泛化能力、鲁棒性、稳定性
- 标注视频/GIF 插入位置

### 描述框架

```markdown
## 实验表现与视频演示

### 泛化能力测试
[描述跨场景、跨物体、跨任务的泛化表现]

**跨场景**：机器人在 [场景 A] 训练后，可以直接泛化到 [场景 B]，成功率达到 [数值]。

**跨物体**：即使面对 [未见过的物体类型]，机器人也能 [具体表现]。

### 鲁棒性测试
[描述在扰动条件下的表现]

**动态干扰**：在机器人执行任务时 [加入什么干扰]，仍能保持 [数值] 的成功率。

**光照变化**：在 [不同光照条件] 下，性能仅下降 [百分比]。

### 视频演示
**[此处插入机器人执行任务的 GIF 动图，建议来自项目主页的视频片段]**

**视频链接**：
- YouTube: [链接]
- Bilibili: [链接]
```

## 资源列表格式

### 标准格式

```markdown
## 资源直达

- **论文地址**：[Arxiv Link](URL)
- **项目主页**：[Project Page URL](URL)
- **代码仓库**：[GitHub Repo URL](URL)
- **演示视频**：[YouTube/Bilibili Link](URL)

> 如代码未开源，标注："Code Coming Soon"
```

### 注意事项
- 所有链接必须可点击
- 检查链接有效性
- 如有多个代码仓库（如主仓库 + 数据集仓库），分别列出
