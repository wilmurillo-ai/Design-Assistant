## Install

**Step 1:** Make sure jq is installed (one-time setup):

```bash
brew install jq
```

**Step 2:** Download and unzip the skill, then move it to your OpenClaw skills folder:

```bash
unzip supstack-skill-api.zip -d ~/.openclaw/skills/
mv ~/.openclaw/skills/supstack-skill ~/.openclaw/skills/supstack
```

**Step 3:** Verify it works:

```bash
bash ~/.openclaw/skills/supstack/scripts/supstack.sh search magnesium
```

You should see JSON results. If so, the skill is ready — just message your agent.

No API key needed. The SupStack database is free and public.

## Research Monitor

The Research Monitor automatically checks for new studies on your supplements and messages you when something interesting is published.

**Start it:** Tell your agent "keep me updated on research for my stack" or "track new studies on magnesium, creatine, and vitamin D."

**What it does:**
- Checks for new studies on a schedule you choose (weekly, biweekly, or monthly)
- Messages you only when new research is found — no "nothing new" spam
- Remembers what it's already told you so you never see duplicates
- You can add or remove supplements from tracking at any time

**Change it anytime:**
- "Add ashwagandha to my research tracking"
- "Remove creatine from research tracking"
- "Stop all research updates"

**Enable automatic scheduling:** Copy the cron template to your OpenClaw jobs:

```bash
cat ~/.openclaw/skills/supstack/cron/research-monitor.json
# Then merge into ~/.openclaw/workspace/cron/jobs.json
```

## It Remembers You

The agent builds a profile as you talk. Mention your stack, medications, goals, or health conditions — it saves them and uses them automatically going forward.

> **You:** I'm currently taking magnesium glycinate 300mg at night, creatine 5g, vitamin D 4000 IU, and omega-3 2g.

From that point on, when you say "analyze my stack" or "what should I add?", the agent already knows what you take. No need to list everything again.

**What it remembers:**
- 💊 Your current supplements (names, doses, timing)
- 💉 Your medications (for automatic interaction checking)
- 🎯 Your health goals (so recommendations are personalized)
- 🚫 Your conditions and allergies (factored into safety)
- 📝 Supplements you've tried and stopped (and why)
- 🔍 Supplements you've asked about but aren't taking yet
- ⚙️ Your preferences (budget, pill count, capsule vs powder)

It also notices patterns. If you keep asking about sleep-related supplements, it'll pick up on that:

> **Agent:** I've noticed most of your questions lately are sleep-related. Want me to do a dedicated sleep stack analysis?

And if you've been researching something for a while without starting it:

> **Agent:** You've asked about lion's mane three times now. Want me to do a full breakdown — evidence, dosing, and how it fits with your current stack?

**What you can ask:**
- "What's my current stack?"
- "What medications do I have on file?"
- "I stopped taking ashwagandha — it gave me stomach issues" (moves it to past trials)
- "I just started metformin" (saves it and immediately checks against your stack)

## Troubleshooting

**Agent doesn't seem to use the skill?**
- Make sure the skill folder is at `~/.openclaw/skills/supstack/` (with `SKILL.md` inside)
- Restart your OpenClaw instance
- If you use skill allowlisting, add `supstack` to your `openclaw.json`:
  ```json
  { "skills": { "entries": { "supstack": { "enabled": true } } } }
  ```

**Getting "jq not found" errors?**
- Run `brew install jq`

**Research Monitor not sending updates?**
- Check that the cron job is in your `jobs.json`
- Run `bash ~/.openclaw/skills/supstack/scripts/supstack.sh monitor-status` to verify setup
- Run a manual check: `bash ~/.openclaw/skills/supstack/scripts/supstack.sh monitor-check`

## For Developers

Full API documentation is in `API.md`. The SupStack API is public at `https://supstack.me/api/v1` — 15 endpoints, all GET, no authentication required.

The skill script (`scripts/supstack.sh`) wraps every API endpoint. Run `bash scripts/supstack.sh help` for the full command reference.

State files are stored in `~/.openclaw/workspace/supstack/`:
- `profile.md` — user profile (stack, medications, goals, conditions, preferences)
- `monitor.json` — Research Monitor tracking state
- `experiments/*.json` — N-of-1 experiment data (one file per experiment)
- `onboarded` — first-run marker
