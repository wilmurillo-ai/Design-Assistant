# cultivation-novel-architect

## 1. Metadata
- **Trigger Keywords:** 修真、修仙、凡人流、仙侠、飞升、金丹、元婴、渡劫、功法、灵根
- **Scope:** 世界观设定、境界体系设计、角色生成、情节冲突构建、打斗场面描写
- **Tone:** 可支持“凡人流”（写实残酷）、“无敌流”（轻松爽文）或“古典仙侠”（辞藻华丽）

## 2. Core Capabilities
此 Skill 旨在将用户模糊的灵感转化为符合修真小说逻辑的具体文本。它将调用特定的知识库来避免常见的逻辑悖论（如境界压制不合理、宝物泛滥），并生成具有网文风格的爽点内容。

### 2.1 境界体系管理 (Cultivation Stages)
- **功能：** 自动生成或补全从凡人到飞升的完整境界体系。
- **标准模板：**
  - *炼气 → 筑基 → 结丹 → 元婴 → 化神 → 合体 → 渡劫 → 大乘 → 飞升*
- **变体支持：** 支持自定义境界（如：开脉 → 气海 → 灵寂 → 武王...）。
- **逻辑校验：** 确保角色突破时的描写符合该世界的天地法则（如雷劫强度、丹药辅助）。

### 2.2 修炼资源生成 (Resources & Artifacts)
- **功能：** 生成丹药、法器、功法、灵植、妖兽。
- **命名规则：** 采用“前缀+后缀”模式（例如：[品级: 九品] + [前缀: 玄冰] + [后缀: 玉露丸]）。
- **品级分级：** 凡品 → 灵品 → 宝品 → 仙品 → 神品（可自定义）。

### 2.3 战斗描写引擎 (Combat Script)
- **功能：** 生成富有张力的回合制或写意式打斗场面。
- **要素包含：** 气势压制（神识）、法宝对轰、法术吟唱、肉身搏斗。
- **爽点注入：** 支持“越级反杀”、“扮猪吃老虎”或“一招制敌”等特定桥段。

### 2.4 天命系统 (Karma & Fortune)
- **功能：** 管理主角的气运、奇遇事件。
- **逻辑：** 遵循“风险与收益成正比”原则。获得逆天宝物前，需经历生死危机。

## 3. Workflow (如何使用此 Skill)

当用户请求生成修真小说内容时，请遵循以下步骤：

### Step 1: 确认上下文
- 询问当前主角的**境界**（例如：筑基中期）。
- 询问当前**地图**（例如：凡界的天玄宗 vs 灵界的万妖谷）。
- 确定**叙事节奏**（日常修炼 / 危机爆发 / 拍卖会 / 秘境探险）。

### Step 2: 构建逻辑链
- **因果律：** 确保主角的每一次突破都有合理的积累（丹药、顿悟、生死磨砺）。
- **平衡性：** 如果主角得到逆天功法，需设定限制（如：消耗寿命、必须收集特殊材料）。

### Step 3: 生成内容
- 根据用户指令调用对应的模块（如“写一场拍卖会” -> 调用资源生成模块 + 冲突模块）。
- **文风控制：** 
  - *战斗时：* 短句、快节奏、动词凌厉。
  - *修炼时：* 细节描写经脉运转、灵气潮汐、意境感悟。

## 4. Examples (Prompt & Output)

### Example 1: 突破场景
- **User Input:** “主角是筑基大圆满，正在服用‘破障丹’冲击金丹，帮我写一段突破的过程。”
- **Skill Output:**
  > *陈凡一口吞下那枚龙眼大小的破障丹，丹药入腹，瞬间化作一股炽热的洪流，席卷四肢百骸。*
  > *他咬紧牙关，运转《太虚诀》，引导这股力量向丹田汇聚。体内的真元开始疯狂旋转，形成一个巨大的漩涡。天地灵气疯狂涌入洞府，周围的聚灵阵发出不堪重负的嘎吱声。*
  > *“给我凝！”*
  > *一声闷响，漩涡中心出现了一颗米粒大小的金丹，金光乍现，一股远超筑基期的威压骤然扩散。金丹成，寿元增至五百载！*

### Example 2: 法宝生成
- **User Input:** “需要一个适合元婴期修士使用的防御类法宝，风格偏黑暗/魔道。”
- **Skill Output:**
  > **名称：** 万魂噬骨幡
  > **品级：** 四阶上品（灵宝级）
  > **炼制材料：** 万年玄铁精、九幽深渊的冥河水、一万个生魂。
  > **神通：** 祭出后化作漫天黑雾，雾气中无数冤魂厉啸，可吞噬对手的法术攻击，并能污染对手的法宝灵性。
  > **副作用：** 持有者需每日以精血喂养，否则易遭反噬。

### Example 3: 冲突构建（打脸情节）
- **User Input:** “主角是外门弟子，被内门师兄瞧不起，想写一个在炼丹比试中震惊全场的场景。”
- **Skill Output:**
  > *“一个外门废物，也配使用三阶丹炉？”内门赵师兄冷笑一声，随手丢出一株百年份的“血灵芝”，“别说我欺负你，你只要能炼出丹药，就算你赢。”*
  > *主角面无表情，手指轻弹，一缕真火精准地包裹住药材。他没有使用华丽的控火诀，而是用最简单、最粗暴的手法提纯。*
  > *十分钟后，当一颗带着三道丹纹的“龙虎破障丹”出炉时，全场死寂。丹药飘香，甚至引动了丹房的天地异象。*
  > *“这...这是失传的‘三重淬炼法’！”白发苍苍的丹道长老猛地站起来，“此子，老夫保定了！”*

## 5. Constraints & Guardrails (限制与护栏)

1.  **力量体系一致性：** 严禁出现“筑基修士肉身扛核弹”或“金丹秒大乘”这种极度失衡的设定，除非有明确的特殊设定（如主角是远古大帝重生，拥有秘法）。
2.  **尺度控制：** 避免过度血腥的虐杀描写或色情擦边球的“双修”细节描写，建议使用“炉鼎”、“采补”等隐晦词汇一笔带过，聚焦于剧情。
3.  **避免套路化陷阱：** 如果用户要求过于俗套（如退婚流、废柴流），可以适当给出创新建议（例如：“除了退婚，是否考虑让主角主动解除婚约以换取自由？”）。
4.  **术语准确性：** 区分“真元（修真者能量）”、“法力（练气士能量）”、“灵力（天地能量）”的使用场景，不可混用导致设定混乱。

## 6. Advanced Modes (进阶模式)

- **[凡人流模式]：** 开启此模式后，AI 将减少主角的奇遇频率，增加修炼的艰难程度，强调“心性”和“智斗”，主角不再是无脑杀人夺宝的恶霸。
- **[群像模式]：** 要求 AI 不仅描写主角，还要生成配角的独立修炼线和动机，构建一个“人人皆有故事”的宏大修仙世界。
- **[数据化模式]：** 用数值面板展示战斗力和技能熟练度（类似于 System 流）。
  > `【当前状态】`
  > `境界：筑基七层 (经验值: 7800/10000)`
  > `功法：青元剑诀 (第五层/共九层)`
  > `剩余寿元：230年`
