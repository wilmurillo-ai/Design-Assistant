# 通用语言词汇表（Glossary）

建立团队共享的业务术语体系，确保所有人对同一概念的理解一致。

## 词汇表模板

| 中文术语 | 英文术语 | 类别 | 定义 | 示例 | 备注 |
|---------|---------|------|------|------|------|
|         |         | 实体/值对象/事件/服务 |        |      |      |

**类别说明**：
- **实体（Entity）**：有唯一标识，其生命周期内属性可能变化
- **值对象（Value Object）**：由属性定义，无唯一标识，不可变
- **领域事件（Domain Event）**：领域中已发生的重要业务事件
- **领域服务（Domain Service）**：跨实体的业务行为

## 词汇表建立步骤

1. **收集术语**：与业务专家访谈，列出所有业务术语
2. **消除歧义**：对同一术语的不同理解进行统一
3. **分类归档**：按领域/上下文分组
4. **持续更新**：随着领域理解的深入，不断迭代

## 示例（电商场景）

| 中文术语 | 英文术语 | 类别 | 定义 | 示例 | 备注 |
|---------|---------|------|------|------|------|
| 订单 | Order | 实体 | 客户购买行为的记录，有唯一订单号 | SO-20240101-001 | 生命周期：创建→支付→发货→完成/取消 |
| 订单项 | OrderItem | 实体 | 订单中的单个商品记录 | iPhone 15 x 2 | 包含下单时的商品快照 |
| 订单金额 | Money | 值对象 | 由金额和币种定义 | CNY 5999.00 | 不可变，支持加减运算 |
| 订单创建 | OrderCreated | 事件 | 订单已成功创建 | 事件发生时间：2024-01-01 10:00:00 | 触发后续库存检查 |
| 客户 | Customer | 实体 | 平台注册用户，有唯一客户ID | C-10001 | 包含客户等级信息 |
| 客户等级 | CustomerLevel | 值对象 | 客户等级权益标识 | GOLD | 影响折扣策略 |
| 聚合根 | AggregateRoot | 概念 | 聚合内唯一对外入口 | Order | 外部只能通过ID引用 |
| 限界上下文 | Bounded Context | 概念 | 领域边界和通用语言范围 | 订单上下文、支付上下文 | 每个上下文有独立的领域模型 |

## 常见陷阱

- **同词不同义**：同一个词在不同上下文中含义不同，需要明确上下文
- **同义不同词**：同一个概念用了不同的词表达，需要统一
- **技术术语混入**：避免使用技术实现细节来描述业务概念
