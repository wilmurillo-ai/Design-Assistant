# 上下文映射图（Context Mapping）

## 概念定义

限界上下文之间存在多种集成关系，上下文映射图用于描述这些关系。

## 映射关系类型

| 关系类型 | 说明 | 使用场景 |
|---------|------|---------|
| 共享内核（Shared Kernel） | 两个上下文共享部分模型 | 紧密协作的团队 |
| 客户-供应商（Customer-Supplier） | 上游制定计划，下游消费 | 供应商提供 API |
| conformist（追随者） | 下游完全适应上游模型 | 无法影响上游时 |
| 防腐层（ACL） | 在下游转换模型 | 集成外部系统 |
| 开放主机服务（OHS） | 通过服务协议暴露模型 | 提供公开 API |
| 发布语言（PL） | 稳定的集成协议 | API 版本管理 |
| 独立系统（Separate Ways） | 无需集成 | 完全独立 |
| 大泥球（Big Ball of Mud） | 混乱的集成 | 需要重构 |

## 防腐层（Anti-Corruption Layer, ACL）

**作用**：保护本地上理上下文不受外部系统模型污染，在边界处进行模型转换。

```
[外部系统] --> [防腐层/ACL] --> [本地上下文]
              (模型转换)
```

**实现原则**：
- 在**下游上下文**中建立
- 外部模型转换为本地模型
- 禁止在本地理领域模型中直接引用外部实体

## 共享内核（Shared Kernel）

两个上下文共享一个小的、内聚的模型子集。

**使用约束**：
- 共享模型必须有明确的边界
- 两个团队必须协调变更
- 变更需要双方同意

## 上下游关系识别

**上游特征**：
- 决定模型和接口
- 不依赖下游
- 通常是核心域

**下游特征**：
- 依赖上游模型
- 可以申请变更，但不强制
- 可以通过 ACL 隔离影响

## 上下文映射图示例

```
+------------------+       防腐层 (ACL)       +------------------+
|   订单上下文      | <--------------------> |   支付上下文       |
|                  |      客户-供应商        |                  |
| Order, OrderItem |      OrderId/Payment   | Payment, Invoice  |
+------------------+                         +------------------+
        |                                              ^
        |          共享内核                             |
        +-------------------------------------------> |
                      Money, CustomerId               |
                                                  +------------------+
                                                  |   库存上下文       |
                                                  | InventoryContext |
                                                  +------------------+
```

## 绘制步骤

1. 识别所有限界上下文
2. 确定每个上下文的团队边界
3. 分析上下文间的数据依赖
4. 确定集成模式（共享内核/ACL/Conformist 等）
5. 标注信任边界和通信方式
