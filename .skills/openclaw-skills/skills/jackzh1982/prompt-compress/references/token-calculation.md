# Token 估算与计算

## Token 计算原理

不同语言和模型对 Token 的计算方式不同，以下是通用估算方法。

### 中文 Token 估算

```
中文 Token ≈ 中文字符数 / 1.5

示例：
"你好世界" (4个中文字符) ≈ 2.67 tokens
"这是一个测试提示词" (8个中文字符) ≈ 5.33 tokens
```

### 英文 Token 估算

```
英文 Token ≈ 英文字符数 / 4

示例：
"Hello world" (11个字符) ≈ 2.75 tokens
"Analyze this prompt carefully" (27个字符) ≈ 6.75 tokens
```

### 混合文本

```
总 Token = 中文 Token + 英文 Token + 其他字符 Token

示例：
"请分析 this prompt" 
= (2 中文 / 1.5) + (10 英文 / 4) + (1 空格 / 3)
= 1.33 + 2.5 + 0.33
= 4.16 tokens
```

## Token 消耗场景分析

### 场景 1: 简单问答

```
提示词: "你好，请帮我解释一下什么是机器学习"
Token 消耗: ~30 tokens

压缩后: "[解释]机器学习"
Token 消耗: ~8 tokens
节省: 73%
```

### 场景 2: 长文档分析

```
原始: 完整文档 + 详细要求
Token: 文档(50K) + 要求(500) = 50,500 tokens

压缩策略:
1. 文档摘要: 50K → 5K
2. 要求精简: 500 → 100

压缩后: 5,100 tokens
节省: 90%
```

### 场景 3: 多轮对话

```
原始: 保存完整对话历史
Token: 每轮累积，最终超出限制

压缩策略:
1. 滑动窗口: 只保留最近 N 轮
2. 摘要压缩: 旧对话 → 摘要
3. 关键信息提取: 只保留决策点

节省: 60-80%
```

### 场景 4: 代码审查

```
原始: 完整代码文件
Token: 取决于文件大小

压缩策略:
1. 只传 diff 部分
2. 精简注释
3. 移除空行

节省: 50-70%
```

## Token 消耗公式

```
总消耗 = 输入 Token + 输出 Token + 系统提示 Token

费用 = 输入单价 × 输入 Token + 输出单价 × 输出 Token
```

### 示例费用计算

```
GPT-4 Turbo 定价:
- 输入: $10 / 1M tokens
- 输出: $30 / 1M tokens

请求示例:
- 输入: 10,000 tokens
- 输出: 2,000 tokens

费用 = (10/1000000 × 10000) + (30/1000000 × 2000)
     = $0.10 + $0.06
     = $0.16
```

## Token 节省策略对比

| 策略 | 节省率 | 实现难度 | 适用场景 |
|------|--------|---------|---------|
| 删除礼貌用语 | 10-20% | 低 | 所有 |
| 结构化表达 | 20-40% | 低 | 所有 |
| 分块处理 | 50-80% | 中 | 长文档 |
| RAG 检索 | 70-90% | 高 | 知识库 |
| 摘要压缩 | 60-80% | 中 | 历史/文档 |
| 混合模型 | 30-50% | 低 | 多任务 |
