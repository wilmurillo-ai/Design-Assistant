## 概览
本文档定义了电商比价分析中的风险评估规则，采用显性特征（rule-based）的方式计算风险评分，避免直接"拍值"。

## 风险等级定义

### 低风险（Low Risk）
- **风险评分**: 0-29 分
- **特征**: 渠道可靠、条件简单、售后完善、价格合理
- **典型场景**: 官方自营、官方旗舰店、授权店铺
- **用户建议**: 可放心购买

### 中风险（Medium Risk）
- **风险评分**: 30-59 分
- **特征**: 部分特征存在风险，但整体可控
- **典型场景**: 百亿补贴、限时优惠、需领券、新用户专享
- **用户建议**: 谨慎购买，仔细阅读条款

### 高风险（High Risk）
- **风险评分**: 60-100 分
- **特征**: 多个特征存在显著风险
- **典型场景**: 疑似翻新、非官方渠道、无售后保障、价格异常
- **用户建议**: 高度警惕，不建议购买

## 显性特征评分规则

风险评分由四个显性特征组成，每个特征权重 25 分，总分 0-100 分：

### 1. 渠道风险（0-25 分）

**官方渠道（0 分）**:
- 京东官方自营
- 天猫官方旗舰店
- 官方授权店
- 亚马逊自营

**补贴渠道（10 分）**:
- 拼多多百亿补贴
- 平台补贴类渠道

**第三方渠道（15 分）**:
- 普通第三方店铺
- 个人店铺

**高风险渠道（25 分）**:
- 无明确商家信息
- 商家信誉差（评分 < 4.5）

### 2. 条件复杂度（0-25 分）

**无特殊条件（0 分）**:
- 官方价格，无任何附加条件

**简单条件（2-5 分）**:
- 需领券（2 分）
- 满减（2 分）
- 返现（2 分）
- 单项条件最多 5 分

**复杂条件（5-15 分）**:
- 限时（5 分）
- 限量（5 分）
- 限地区（5 分）
- 需会员（5 分）
- 多项条件累加

**极复杂条件（15-25 分）**:
- 限新用户（10 分）
- 多项复杂条件叠加
- 每项扣 5 分，最高 25 分

### 3. 售后不确定性（0-25 分）

**完善售后（0 分）**:
- 官方渠道
- 全国联保
- 明确退换货政策

**基础售后（5 分）**:
- 授权店铺
- 有保修说明

**不确定售后（15 分）**:
- 第三方渠道
- 售后说明不明确
- 保修期较短

**无售后保障（25 分）**:
- 明确标注"无售后"
- 无保修说明
- 无退换货政策

### 4. 价格偏离度（0-25 分）

**价格合理（0-5 分）**:
- 价格在 P25-P75 主流区间（0 分）
- 价格略低于 P25（< 5%）（5 分）
- 价格略高于 P75（< 10%）（5 分）

**价格偏低（10-20 分）**:
- 价格低于 P25（5-10%）（10 分）
- 价格低于 P10（5-10%）（15 分）
- 价格低于 P10（10-15%）（20 分）

**价格异常（25 分）**:
- 价格低于 P10（> 15%）（25 分）
- 价格高于 P90（> 15%）（25 分）

## 价格异常判断规则

价格异常判断必须使用定量规则：

### 强异常
- **规则**: 价格 < P10 × 0.9
- **含义**: 价格显著低于市场主流区间，超过 10% 偏离
- **处理**: 禁止作为"推荐购买方案"输出，仅作为风险警告
- **示例**: P10 = 5199，价格 < 4679（5199 × 0.9）→ 强异常

### 中异常
- **规则**: 价格 < P25 × 0.95
- **含义**: 价格略低于市场主流区间，接近异常边缘
- **处理**: 可作为推荐，但需标注风险
- **示例**: P25 = 5299，价格 < 5034（5299 × 0.95）→ 中异常

### 正常价格
- **规则**: P25 × 0.95 ≤ 价格 ≤ P90
- **含义**: 价格处于合理区间
- **处理**: 可正常推荐

### 溢价
- **规则**: 价格 > P90 × 1.1
- **含义**: 价格高于市场主流区间
- **处理**: 可作为推荐，但需标注溢价

## 推荐标签判定规则

### 省心
**判定条件**:
- 风险等级 = 低
- 渠道 = 官方自营/官方旗舰店
- 条件 = 无复杂条件
- 价格 = 在主流区间或略高于主流

**理由模板**:
- "官方自营，售后无忧"
- "官方旗舰店，价格合理"

### 性价比
**判定条件**:
- 风险等级 = 低 或 中
- 价格 = 低于主流区间（P25）
- 且不是强异常价格

**理由模板**:
- "价格低于主流区间"
- "价格合理，风险低"

### 高风险
**判定条件**:
- 风险等级 = 高

**理由模板**:
- "价格异常或条件复杂"
- "渠道不可靠，风险较高"

### 中规中矩
**判定条件**:
- 价格 = 在主流区间（P25-P75）
- 风险等级 = 中

**理由模板**:
- "价格在主流区间"
- "条件适中，需注意"

## 人群导向推荐规则

### 性价比最高（价格与风险平衡）
**目标用户**: 追求性价比，愿意承担一定风险
**推荐标准**:
- 价格 ≤ P25（低于主流区间）
- 风险等级 = 中（低风险更佳）
- 条件复杂度 ≤ 10 分

**输出示例**:
```json
{
  "platform": "拼多多",
  "price": 4899,
  "risk_level": "中",
  "tag": "性价比",
  "reason": "百亿补贴，限新用户"
}
```

### 最省心（售后/条件最简单）
**目标用户**: 追求购买便捷，售后无忧
**推荐标准**:
- 风险等级 = 低
- 渠道 = 官方自营/官方旗舰店
- 条件复杂度 = 0 分
- 售后不确定性 = 0 分

**输出示例**:
```json
{
  "platform": "京东",
  "price": 5299,
  "risk_level": "低",
  "tag": "省心",
  "reason": "官方自营，售后无忧"
}
```

### 最低价（如风险可控）
**目标用户**: 追求最低价，能接受一定风险
**推荐标准**:
- 价格 = 所有选项中最低
- 风险等级 ≤ 中（高风险不推荐）
- 不是强异常价格

**输出示例**:
```json
{
  "platform": "拼多多",
  "price": 4899,
  "risk_level": "中",
  "tag": "性价比",
  "reason": "百亿补贴，需注意条件"
}
```

## 典型场景评分示例

### 场景 1：京东官方自营 + 主流价格
**数据**:
- 价格: 5299
- P25-P75: 5199-5499
- 条件: 官方自营，无额外券
- 卖家类型: official

**评分计算**:
1. 渠道风险: 0 分（官方自营）
2. 条件复杂度: 0 分（无特殊条件）
3. 售后不确定性: 0 分（官方渠道）
4. 价格偏离度: 0 分（价格在 P25-P75 区间）
5. **总分**: 0 分
6. **风险等级**: 低
7. **推荐标签**: 省心
8. **推荐理由**: "官方自营，售后无忧"

### 场景 2：拼多多百亿补贴 + 新用户限制
**数据**:
- 价格: 4899
- P10: 5199
- 条件: 百亿补贴，限新用户
- 卖家类型: subsidized

**评分计算**:
1. 渠道风险: 10 分（补贴渠道）
2. 条件复杂度: 10 分（限新用户）
3. 售后不确定性: 15 分（第三方渠道）
4. 价格偏离度: 10 分（低于 P10，5-10%）
5. **总分**: 45 分
6. **风险等级**: 中
7. **推荐标签**: 性价比
8. **推荐理由**: "百亿补贴，限新用户"

### 场景 3：异常低价 + 疑似翻新
**数据**:
- 价格: 3500
- P10: 5199
- 条件: 疑似翻新，非官方渠道
- 卖家类型: third-party

**评分计算**:
1. 渠道风险: 15 分（第三方渠道）
2. 条件复杂度: 5 分（疑似翻新）
3. 售后不确定性: 25 分（无售后保障）
4. 价格偏离度: 25 分（低于 P10，> 15%）
5. **总分**: 70 分
6. **风险等级**: 高
7. **推荐标签**: 高风险
8. **推荐理由**: "价格异常或条件复杂"
9. **异常判断**: 强异常（3500 < 5199 × 0.9 = 4679）

### 场景 4：天猫旗舰店 + 需领券
**数据**:
- 价格: 5199
- P25: 5299
- 条件: 官方旗舰店，需领券200
- 卖家类型: official

**评分计算**:
1. 渠道风险: 0 分（官方旗舰店）
2. 条件复杂度: 2 分（需领券）
3. 售后不确定性: 0 分（官方渠道）
4. 价格偏离度: 5 分（略低于 P25，< 5%）
5. **总分**: 7 分
6. **风险等级**: 低
7. **推荐标签**: 性价比
8. **推荐理由**: "官方旗舰店，需领券"

## 注意事项

1. **显性特征必须定量**: 所有评分必须基于明确的规则，不能主观判断
2. **风险等级统一**: 对用户只展示低/中/高，不展示具体分数
3. **异常价格隔离**: 强异常价格不得作为"推荐购买方案"输出
4. **人群导向**: 推荐结果必须至少包含三类之一（性价比/省心/最低价）
5. **理由简洁**: 简短理由不超过 20 字
6. **结构化输出**: L1 层输出必须可结构化，便于产品与 Agent 调用
