## 概览
本文档为智能体提供电商价格信息的采集指导，包括各平台的特性、采集流程和数据规范。

## 采集原则

### 1. SKU 一致性原则
- 确保采集的价格来自完全相同的商品规格
- 不同颜色、容量、版本、配置的商品不能混在一起
- 示例：iPhone 15 128GB 黑色 vs iPhone 15 256GB 白色（不可比）

### 2. 价格可比性原则
- 优先采集"到手价"（最终支付价格）
- 区分展示价、券后价、满减价
- 不含运费的价格应标注"不含运费"

### 3. 信息完整性原则
- 不仅采集价格，还需采集条件、商家类型、售后政策
- 记录采集时间，便于时效性判断

### 4. 可信度优先原则
- 优先采集官方渠道价格
- 对非官方渠道的价格需额外验证

## 平台特性

### 京东
**渠道类型**:
- 官方自营（京东自营，最高可信度）
- 官方旗舰店
- 授权店
- 第三方商家

**价格特点**:
- 价格通常较稳定
- 自营商品有明确售后保障
- 促销活动：京东618、双11、京东超市日等

**采集要点**:
- 优先采集自营价格
- 标注是否需要领券
- 区分京东PLUS会员价

**常见条件**:
- 自营现货
- 需领券
- 限时秒杀
- PLUS会员专享

### 天猫
**渠道类型**:
- 官方旗舰店（最高可信度）
- 授权店
- 普通店铺

**价格特点**:
- 价格波动较大，常有优惠券活动
- 店铺活动较多：聚划算、百亿补贴等
- 跨店满减是常见促销方式

**采集要点**:
- 确认是否为官方旗舰店
- 区分店铺券和跨店券
- 记录优惠券领取条件和有效期

**常见条件**:
- 店铺满减
- 跨店满减
- 需领券（店铺券/跨店券）
- 88VIP专享价

### 拼多多
**渠道类型**:
- 百亿补贴（平台补贴，价格低但可能有条件）
- 官方品牌店
- 普通店铺

**价格特点**:
- 百亿补贴价格通常最低，但限制较多
- 价格波动频繁，需实时确认
- 同款商品可能有多个店铺

**采集要点**:
- 重点采集百亿补贴价格
- 确认补贴条件（限新用户、限地区等）
- 查看店铺评分和商品评价
- 注意区分"补贴价"和"活动价"

**常见条件**:
- 百亿补贴（通常限新用户）
- 限地区
- 限时秒杀
- 多件优惠
- 关注店铺享优惠

### 抖音
**渠道类型**:
- 品牌自播（官方直播）
- 授权店直播
- 达人带货

**价格特点**:
- 直播间价格可能低于店铺价
- 限时、限量是常见条件
- 价格受主播影响大

**采集要点**:
- 确认是否为品牌官方直播
- 区分直播间价和店铺价
- 记录优惠码和领取方式
- 注意是否为"赠品"而非低价

**常见条件**:
- 直播间专属价
- 限时限量
- 需领券
- 拍一发N（赠品）

### 小红书
**渠道类型**:
- 品牌官方号
- 达人推荐
- 普通用户分享

**价格特点**:
- 多为推荐和分享，非直接购买
- 价格信息可能来自不同平台
- 需甄别广告和真实分享

**采集要点**:
- 小红书主要用于价格参考，而非直接采集
- 关注官方号发布的价格信息
- 查看评论区是否有真实购买反馈

### 淘宝
**渠道类型**:
- 天猫官方旗舰店（同天猫）
- 授权店
- 淘宝个人店铺

**价格特点**:
- 价格区间大，质量参差不齐
- 个人店铺风险较高
- 促销活动多

**采集要点**:
- 区分天猫和淘宝
- 查看店铺评分（皇冠、钻石、心级）
- 查看商品评价和销量
- 注意"包邮"、"假一赔十"等承诺

### 亚马逊（海外）
**渠道类型**:
- 亚马逊自营
- 第三方商家

**价格特点**:
- 价格含税费（如增值税）
- 运费可能较高
- 受汇率影响

**采集要点**:
- 确认是否含税
- 确认运费
- 注意关税政策
- 确认保修政策（是否全球联保）

## 采集流程

### 步骤 1：商品语义对齐
1. 识别商品品牌、型号、关键规格
2. 去除营销词、赠品词、套装描述
3. 确认目标 SKU
4. 排除不可比商品

**示例**:
- 用户输入："iPhone 15 Pro Max 256GB"
- 语义对齐：Apple iPhone 15 Pro Max A3108 256GB
- 排除：iPhone 15 Pro Max 512GB（容量不同）、iPhone 15 Pro（型号不同）

### 步骤 2：平台选择
1. 根据用户指定平台采集
2. 若未指定，默认采集：京东、天猫、拼多多
3. 考虑地区限制（如用户在香港，优先考虑香港平台）

### 步骤 3：价格信息采集
1. **采集展示价**: 商品页面标注的价格
2. **采集到手价**: 计算最终支付价格（考虑优惠券、满减等）
3. **采集条件**: 记录获得该价格的所有条件
4. **采集商家信息**: 商家类型、店铺评分、售后政策
5. **采集时间**: 记录采集时间戳

### 步骤 4：数据标准化
1. 统一价格单位（如均为人民币元）
2. 统一条件描述格式
3. 统一商家类型分类
4. 标注数据来源

## 数据规范

### 价格数据格式
```json
{
  "price": 5299,           // 数字类型，单位：元
  "display_price": 5999,  // 展示价
  "coupon_price": 700,    // 优惠金额
  "shipping": 0,          // 运费（0表示包邮）
  "effective_price": 5299  // 到手价（price + shipping）
}
```

### 条件描述格式
```json
{
  "conditions": "官方自营，无额外券",  // 简洁描述
  "details": {                        // 详细条件
    "need_coupon": false,
    "is_limited_time": false,
    "is_limited_quantity": false,
    "new_user_only": false,
    "region_limited": false,
    "membership_required": false
  }
}
```

### 商家信息格式
```json
{
  "seller_type": "official",          // official/authorized/third-party/subsidized
  "seller_name": "京东自营",
  "seller_rating": 4.9,              // 店铺评分（如果有）
  "after_sales": "全国联保"          // 售后政策
}
```

### 完整数据示例
```json
{
  "platform": "京东",
  "product_name": "Apple iPhone 15 128GB 黑色",
  "price": 5299,
  "display_price": 5999,
  "conditions": "官方自营，无额外券",
  "details": {
    "need_coupon": false,
    "is_limited_time": false,
    "is_limited_quantity": false
  },
  "seller_info": {
    "seller_type": "official",
    "seller_name": "京东自营",
    "seller_rating": 4.9,
    "after_sales": "全国联保"
  },
  "collect_time": "2024-01-15T10:30:00Z"
}
```

## 注意事项

### 1. 时效性
- 电商价格变化频繁，采集时应记录时间
- 建议在回复中标注价格采集时间
- 提醒用户价格可能随时变化

### 2. 条件验证
- 采集后应验证条件的真实性
- 对可疑条件（如"零元购"、"免费领"）应特别警惕
- 避免采集明显不合理的价格

### 3. SKU 验证
- 仔细核对商品规格，确保 SKU 一致
- 对疑似混充不同 SKU 的商品应排除
- 示例：同一型号不同容量的商品价格不能直接对比

### 4. 赠品与捆绑销售
- 区分"低价"和"赠品"
- 拍一发N的赠品不计入价格对比
- 捆绑销售（如+配件）应标注

### 5. 运费与税费
- 查询是否包邮
- 如果有运费，应计入到手价
- 海外商品需考虑关税和增值税

### 6. 促销活动识别
- 识别平台大促（如618、双11）期间的异常低价
- 识别限时秒杀、限量优惠等不稳定价格
- 标注促销活动，提醒用户注意时效

### 7. 用户评价参考
- 查看商品评价，了解真实体验
- 特别关注"中评"和"差评"
- 注意区分"刷单"评价

### 8. 风险商品识别
- 价格显著低于市场价（如低于 P10 20%以上）
- 商家非官方且评分低（< 4.5）
- 无明确售后说明
- 需要预付定金、分期等特殊支付方式
- 要求线下交易、微信转账等

## 采集工具建议

### 优先级 1：网络搜索
- 搜索关键词："商品名 + 平台 + 价格"
- 示例："iPhone 15 128GB 京东价格"
- 查看搜索结果中的商品信息

### 优先级 2：官方应用/网站
- 访问平台官方网站或APP
- 搜索商品
- 查看价格和活动信息

### 优先级 3：比价网站
- 如有比价网站，可作为辅助参考
- 注意比价网站的数据时效性

## 常见问题

### Q1: 如何确认商品是官方旗舰店？
**A**: 
- 查看店铺名称是否包含"官方旗舰店"
- 查看店铺是否有品牌授权证书
- 查看店铺评分和商品评价

### Q2: 如何判断价格是否为"到手价"？
**A**:
- 查看商品详情页是否标注"到手价"
- 计算优惠券、满减等优惠
- 加上运费（如果不包邮）

### Q3: 如何识别"伪低价"？
**A**:
- 价格显著低于市场价（如低于 P10 20%以上）
- 商家非官方且信息不透明
- 需要满足复杂条件（如预付定金、分期等）
- 有大量负面评价

### Q4: 如何处理不同地区的价格差异？
**A**:
- 标注地区信息
- 考虑税费差异
- 考虑运费差异
- 如果用户未指定地区，默认以大陆地区为准

### Q5: 如何确保采集的数据时效性？
**A**:
- 记录采集时间
- 在回复中标注"价格采集时间"
- 提醒用户价格可能随时变化
- 对于大促期间的价格，特别标注时效性

## 总结

智能体在采集价格信息时，应遵循以下核心原则：

1. **准确性**: 确保 SKU 一致，价格准确
2. **完整性**: 不仅采集价格，还需采集条件、商家信息
3. **时效性**: 记录采集时间，标注价格变化
4. **可比性**: 确保采集的价格来自可比的来源
5. **可信度**: 优先采集官方渠道，验证数据真实性

通过遵循本指导，智能体可以采集到高质量的价格数据，为后续的价格分析和风险评估提供可靠基础。
