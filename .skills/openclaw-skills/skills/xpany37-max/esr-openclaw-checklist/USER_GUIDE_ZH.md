## 📋 安全检查项目

### 1. 默认端口使用情况
- **检查内容**: 是否使用默认端口18789
- **安全要求**: 使用17999-18999之间的自定义端口
- **风险等级**: 低风险

### 2. 服务监听地址检查
- **检查内容**: 服务是否绑定到0.0.0.0（公网暴露）
- **安全要求**: 仅绑定到127.0.0.1（本地访问）
- **风险等级**: 高风险

### 3. 内网穿透工具检测
- **检查内容**: 是否运行未授权穿透工具
- **检测工具**: ngrok, frpc, cloudflared等20+种
- **风险等级**: 高风险

### 4. Node.js版本检查
- **检查内容**: Node.js版本是否>=18 LTS
- **安全要求**: 使用稳定版本，及时更新
- **风险等级**: 中风险

### 5. 密码登录模式检查
- **检查内容**: 是否启用密码登录模式
- **安全要求**: 配置 `mode: "password"`
- **风险等级**: 高风险

### 6. Skill数量及官方来源检查
- **检查内容**: 安装的Skill是否来自官方源
- **安全要求**: 仅使用ClawHub官方Skill
- **风险等级**: 中风险

### 7. 配置文件权限检查
- **检查内容**: openclaw.json文件权限是否为600
- **安全要求**: 仅所有者可读写
- **风险等级**: 高风险

### 8. OpenClawd深度安全审计
- **检查内容**: Sandbox等安全配置
- **安全要求**: 启用沙箱隔离
- **风险等级**: 中风险

## ⏰ 定时任务配置

### 当前配置（已预设）
```bash
# 定时任务已自动配置：
# 名称: ESR每日安全检查
# 时间: 每天 17:00 (北京时间)
# 时区: Asia/Shanghai
# 执行: python3 scripts/openclaw_checklist_scheduled.py
# 通知: 发送到钉钉群 cid8NuHF/3BALK8ub6oKUf0Dw==
```

### 定时任务输出
1. **Python脚本执行**：执行8项安全检查
2. **钉钉消息发送**：发送格式化报告到钉钉群
3. **AI代理回复**："ESR每日自动安全检查已成功执行完成。"

### 管理命令
```bash
# 查看定时任务
openclaw cron list

# 立即执行定时任务
openclaw cron run [任务ID]

# 删除定时任务
openclaw cron remove [任务ID]

# 查看运行历史
openclaw cron runs --id [任务ID]
```

## 🔧 自定义配置

### 修改钉钉群ID
编辑配置文件：
```bash
nano ~/.openclaw/skills/ESR_openclaw-checklist-v3.2/config.json
```
修改 `dingtalk_group_id` 字段。

### 修改定时任务时间
需要删除原有定时任务，重新创建：
```bash
# 1. 查找任务ID
openclaw cron list --json | grep -B2 -A5 "ESR每日安全检查"

# 2. 删除任务
openclaw cron remove [任务ID]

# 3. 创建新任务
openclaw cron add --name "ESR每日安全检查" \
  --cron "0 9 * * *" \  # 改为每天9:00
  --tz "Asia/Shanghai" \
  --message "执行Python脚本 ~/.openclaw/skills/ESR_openclaw-checklist-v3.2/scripts/openclaw_checklist_scheduled.py 并在完成后输出：ESR每日自动安全检查已成功执行完成。" \
  --announce \
  --channel dingtalk \
  --to "新的钉钉群ID"
```

## 🧠 运行思路

### 1. 手动执行流程
```
用户: "运行ESR安全检查"
    ↓
OpenClaw: 调用 python3 scripts/openclaw_checklist.py
    ↓
Python脚本: 执行8项安全检查
    ↓
输出: 标准化格式的结果 + 信息安全提示
    ↓
保存: 生成时间戳命名的JSON文件
```

### 2. 定时任务流程
```
系统时间: 每天17:00
    ↓
OpenClaw Cron: 触发定时任务
    ↓
AI代理: 执行Python脚本
    ↓
Python脚本: 
    1. 执行8项安全检查
    2. 格式化钉钉消息
    3. 发送到钉钉群
    ↓
AI代理: 回复完成消息
    ↓
钉钉群: 收到安全检查报告
```

### 3. 数据处理流程
```
安全检查 → 结果分析 → 格式化输出 → 钉钉发送 → 结果保存
    ↓           ↓           ↓           ↓           ↓
8项检查     风险评级     标准格式     群消息     JSON文件
```
