# ESR安全检查输出格式规范 v3.2

## 概述
本文档定义了ESR OpenClaw安全检查技能v3.2版本的**标准化输出格式**，确保在所有主机上执行"运行ESR安全检查"时，输出内容稳定一致。

## 核心原则
1. **格式稳定**：无论主机环境如何，输出格式保持一致
2. **内容完整**：包含所有8项安全检查结果
3. **信息安全提示**：固定包含信息安全部门提示
4. **结果可追溯**：自动保存JSON格式检查结果

## JSON结果文件格式

### 文件名格式
`scan_result_YYYYMMDD_HHMMSS.json`
- 例如：`scan_result_20260316_112658.json`

### 文件内容结构
```json
{
  "scan_time": "2026-03-16T11:26:58.123456",
  "status": "存在风险",
  "issues": [
    {
      "category": "Network",
      "description": "使用默认端口 18789",
      "severity": 1,
      "recommendation": "修改默认端口为 17999-18999 之间，不使用默认端口"
    },
    {
      "category": "Application",
      "description": "未启用密码登录模式",
      "severity": 3,
      "recommendation": "配置 mode: password，启用密码登录"
    },
    {
      "category": "SupplyChain",
      "description": "共安装 262 个 Skill，非官方: [Skill列表]",
      "severity": 2,
      "recommendation": "仅使用官方来源 Skill，核实非官方 Skill 来源"
    }
  ]
}
```
