# GitHub Launch Checklist

Use this checklist before publishing or announcing a new open-source repo.

## 1. Repo Metadata

Set the GitHub `About` panel deliberately:

- sharp one-line description
- relevant high-intent topics
- homepage only if it adds real value

The goal is to help the right people understand the repo in seconds.

## 2. Social Preview

Prepare a `social preview` image that makes the repo legible in:

- GitHub shares
- X / Twitter cards
- LinkedIn shares
- Slack and Discord unfurls

The image should quickly show:

- project name
- core value proposition
- who it is for

## 3. README First Screen

The first screen should answer:

- what this repo does
- why it matters
- who it is for
- why it is better than the default approach

## 4. Release Notes

Prepare a short release note that covers:

- what shipped
- why it matters
- who should care
- what to do next

## 5. Launch Copy

Prepare a founder/operator-style launch post that explains:

- the problem with the old workflow
- what this repo changes
- who should use it
- what is open source today

## 6. Final Rule

Do not treat “repo published” as the finish line.

A strong open-source release needs:

- good packaging
- good presentation
- good positioning

That is what gets the right clicks, stars, forks, and reuse.
