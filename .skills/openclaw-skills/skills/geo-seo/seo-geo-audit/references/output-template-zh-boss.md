# 中文老板版模板

用于管理层、创始人、负责人汇报。目标是让对方在 60 秒内明白：

- 网站整体健康度如何
- 当前最大风险是什么
- 对增长有什么影响
- 接下来先做什么

避免堆术语。不要把报告写成工程日志。

## 输出结构

### 一句话结论

用一句话回答当前状态：

- `基础健康`
- `有明显短板`
- `存在关键风险`

### 总体判断

控制在 3 行内：

- 技术健康度：`强 / 中 / 弱`
- 内容与品牌可见度：`强 / 中 / 弱`
- 站外实体信号：`强 / 中 / 弱`
- AI / GEO 准备度：`强 / 中 / 弱`

### 关键问题

只列 3 到 5 条，且每条必须带业务影响。

格式：

- `[问题]`：`会导致什么后果`

### 优先级建议

只分三层：

- `P0 立即处理`
- `P1 本月完成`
- `P2 后续优化`

每层最多 3 条。

### 管理层建议

给出 2 到 4 条管理动作。

### 可见性补充说明

如果适用，可额外用 2 行说明：

- 站外提及现状：`例如 LinkedIn / Reddit / YouTube / Wikipedia 等是否有有效存在`
- 平台差异：`例如更适合先打 ChatGPT / Perplexity，还是先补 Google AIO / Gemini`

## 语言规则

- 优先中文
- 少用英文缩写；必须出现时，在第一次出现后补中文解释
- 不写过细的实现步骤
- 不写长 checklist
- 对缺失数据只保留一句简短说明

推荐说明：

`本次结论基于页面公开可见信息和站点审计结果，不包含 Search Console、外链数据库和服务器日志。`

如果做过站外平台核查，可补：

`站外实体和社媒可见性仅基于公开可核查平台，不代表完整舆情或品牌声量数据库。`
