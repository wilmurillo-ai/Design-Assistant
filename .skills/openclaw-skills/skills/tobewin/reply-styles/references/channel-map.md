# Channel Map

## Goal

同一句话在不同渠道里，长度、格式、压迫感都不同。

## Supported Channels

### `wechat`

适合：

- 私聊
- 熟悉关系
- 轻商务沟通

特点：

- 句子短
- 口语感更自然
- 不要太像公文

### `email`

适合：

- 正式商务
- 跨团队协作
- 客户与管理层沟通

特点：

- 结构完整
- 开头称呼明确
- 结尾收口更正式

### `dm`

适合：

- 平台私信
- 陌生关系沟通

特点：

- 更快进入主题
- 少铺垫
- 减少冗长寒暄

### `group`

适合：

- 群消息
- 社群运营
- 公告 / 提醒 / 规则说明

特点：

- 更清楚
- 少个人情绪
- 信息组织感强

### `comment`

适合：

- 评论区
- 公开互动

特点：

- 更短
- 不宜解释过长
- 注意公开场合体面

### `support-ticket`

适合：

- 客服处理
- 工单回复

特点：

- 清楚回应问题
- 动作和承诺更明确
- 降温优先

## Practical Rule

- 默认私聊用 `wechat`
- 正式商务优先 `email`
- 投诉与客服优先 `support-ticket`
- 群体沟通优先 `group`
