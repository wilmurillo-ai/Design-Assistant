# Mode Packs

回复模式决定一句话的行进方式，不只是语气。

## 推荐模式

| Mode | 中文定位 | 适合场景 | 结构 |
|---|---|---|---|
| `soothe-first` | 先安抚后回应 | 投诉、情绪高、敏感反馈 | 先接情绪，再说事实，最后给动作 |
| `clear-reject` | 清楚拒绝 | 礼貌拒绝、划边界、止损 | 轻接一下，明确结论，礼貌收口 |
| `push-forward` | 推进落地 | 催进度、压节点、推进执行 | 对齐节点，说明目标，压实下一步 |
| `repair-trust` | 修复信任 | 道歉、售后补救、关系修复 | 先认影响，再讲责任，最后给修复动作 |
| `public-safe` | 公开稳妥 | 评论区、社媒、品牌公开说明 | 确认已收到，谨慎表态，留正式出口 |
| `concise-update` | 结论先行 | 给老板、管理同步、简短更新 | 先给结论，补充状态，收在节点 |

## 什么时候一定要用模式

- 用户说“不要太软，要把话说清楚”：优先 `clear-reject`
- 用户说“对方现在很炸”：优先 `soothe-first`
- 用户说“帮我催一下但别太冲”：优先 `push-forward`
- 用户说“要公开发出去”：优先 `public-safe`
- 用户说“给老板回一句就行”：优先 `concise-update`
