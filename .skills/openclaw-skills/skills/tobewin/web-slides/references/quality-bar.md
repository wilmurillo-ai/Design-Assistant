# Quality Bar

## Priority Order

1. 效果好
2. 稳定好
3. 低 token
4. 零 API Key
5. 移动端友好

低 token 不能以牺牲视觉完成度为代价。

## Visual Standard

生成结果至少满足以下标准：

- 首屏有记忆点，不像普通网页首屏
- 主题统一，不出现风格漂移
- 标题、正文、数字、注释层级清晰
- 配色控制在 1 个主色系 + 1 个强调色系
- 动效轻量，不杂乱，不廉价
- 页面留白合理，不拥挤

## Content Density Rules

- 一页 1 个中心观点
- 标题最好不超过 16 个中文字符
- 正文 bullet 最好 3-5 条
- 单条 bullet 最好不超过 2 行
- 数据页优先“结论 + 数字”，不要只有数字没有解释
- 图表页必须用一句话写清结论

## Mobile Rules

- 小屏下标题仍然可读
- 双栏需要可以降级为单栏堆叠
- 指标卡片可纵向排列
- 不依赖 hover 才能理解内容
- 边距和字号要用响应式策略

## Motion Rules

- 动效只服务层次和节奏
- 默认使用淡入、轻位移、轻缩放
- 避免大幅旋转、频繁弹跳、强炫光扫过
- 正式场景动效要更克制

## Failure Modes To Avoid

- 科技风变成霓虹夜店风
- 商务风变成模板市场廉价蓝
- 极简风变成空白且无层次
- 图表很多但没有结论
- 页面内容溢出或字号过小
- 手机端标题断裂、卡片挤压、行距失衡

## Final Check

交付前至少问自己：

- 这份演示稿是否可以直接用于正式场合？
- 封面是否足够“高端”？
- 是否比同类技能默认结果更精致？
- 如果用户在手机上打开，是否仍然像成品而不是残版？
