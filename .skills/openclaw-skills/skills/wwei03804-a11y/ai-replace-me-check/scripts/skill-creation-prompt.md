## 🔍 示例：创建一个「多项目仪表板」Skill

```markdown
## 角色
你是一个经验丰富的 OpenClaw Skill 开发者。请帮我创建一个自定义 Skill。

## 技能基本信息

- **名称**：multi-project-dashboard
- **功能描述**：统一查看多个项目的状态、风险和待办，减少在多个系统间切换
- **使用场景**：每天早上查看所有项目健康度，快速定位需要关注的项目
- **目标用户**：同时管理多个项目的项目经理

## 核心功能

### 功能 1：项目状态汇总
- **输入**：项目列表（禅道/Jira 地址）
- **处理逻辑**：
  1. 调用各项目 API 获取状态
  2. 汇总工单数量、延期数、阻塞数
  3. 计算健康度得分
- **输出**：统一格式的项目状态卡片

### 功能 2：风险预警
- **输入**：当前项目状态
- **处理逻辑**：
  1. 识别红灯项目（延期 > 3 天、阻塞 > 5 个）
  2. 生成预警消息
- **输出**：风险预警列表

### 功能 3：待办汇总
- **输入**：各项目的我的待办
- **处理逻辑**：
  1. 收集所有待办
  2. 按紧急程度排序
  3. 去重和合并
- **输出**：今日待办列表

## 工作流程

```
1. [触发] 用户说"看看所有项目状态"
2. [收集] 调用禅道/Jira API 获取数据
3. [分析] 计算健康度、识别风险
4. [生成] 生成统一格式的仪表板
5. [发送] 通过飞书推送给用户
```

## 依赖工具

| 工具 | 用途 | 是否必需 |
|------|------|---------|
| 飞书 | 消息发送 | 是 |
| 禅道 | 项目数据 | 是 |
| Jira | 项目数据 | 否（可选） |

## 输出示例

### 示例 1：项目状态汇总
- **输入**：禅道项目列表
- **输出**：
  ```
  📊 项目状态早报

  ✅ 健康（3 个）
  - 项目A：进度 80%，无风险
  - 项目B：进度 65%，无风险
  - 项目C：进度 90%，无风险

  ⚠️ 关注（1 个）
  - 项目D：进度 50%，延期 2 天

  🔴 风险（1 个）
  - 项目E：进度 40%，延期 5 天，阻塞 8 个
  ```

### 示例 2：风险预警
- **输出**：
  ```
  🚨 风险预警

  项目E 需要立即关注：
  - 延期 5 天（合同约定：3/31）
  - 8 个阻塞工单
  - 开发团队反馈：核心人员请假

  建议动作：
  1. 与客户沟通延期可能性
  2. 评估是否需要加班赶工
  3. 升级到管理层
  ```

## 约束条件

- 不访问项目敏感文档
- 数据仅保存在本地
- 遵循禅道/Jira API 使用限制
```

```markdown
├── README.md                   # 使用说明
└── scripts/
    └── main.py                # 核心逻辑（如果需要）
```

## 🔧 创建后的下一步

1. **本地测试**：在本地环境测试 Skill 是否正常工作
2. **安全检查**：检查是否有安全漏洞或隐私风险
3. **上传 ClawHub**：如果适合公开，分享给社区
4. **用户确认**：让用户测试并提供反馈
5. **迭代优化**：根据反馈改进 Skill

---

## ⚠️ 注意事项

1. **安全第一**：不要在 Skill 中存储敏感信息（密码、Token 等）
2. **用户授权**：使用用户已授权的工具，访问用户明确同意的数据
3. **错误处理**：graceful degradation，失败时给出有用的错误信息
4. **文档完整**：让用户知道 Skill 能做什么、不能做什么
