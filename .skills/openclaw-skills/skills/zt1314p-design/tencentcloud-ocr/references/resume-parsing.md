# 多语种简历结构化识别指引

## 概述

本 reference 提供基于通用高精度 OCR 的多语种简历结构化识别能力。支持将中文、英文、日文、韩文、法文、德文、西班牙文等多语种简历图片/PDF 进行 OCR 识别与格式化解析，自动提取关键信息并输出排版精美的可读 Markdown 文本。

## 处理流程

```
用户上传简历图片/PDF
        │
        ▼
  调用 scripts/main.py 进行文字识别（通用高精度 OCR）
        │
        ▼
  获取 raw_text 原始文本
        │
        ▼
  大模型根据 Prompt 模板直接提取关键信息并格式化输出可读文本
```

**Step 1**：使用 `scripts/main.py` 对简历图片/PDF 进行 OCR 识别，获取 `raw_text`。

调用示例：

```bash
# 识别简历图片
python scripts/main.py --image-url "https://example.com/resume.jpg"

# 识别简历 PDF
python scripts/main.py --image-url "https://example.com/resume.pdf" \
  --is-pdf true --pdf-page-number 1

# 使用 Base64 文件调用
python scripts/main.py --image-base64 /path/to/resume.jpg
```

**Step 2**：根据下方的 **Prompt 模板** 从 `raw_text` 中一步完成字段提取与格式化输出。

## Prompt 模板

获取到 `raw_text` 后，请按照以下 Prompt 直接提取并格式化输出：

```
你是一个多语种简历解析专家。请根据以下 OCR 识别出的简历原始文本，提取关键信息并按照指定格式排版输出。

## 提取要求（极其重要）

1. **完整保留原文，禁止删减**：必须完整保留简历中每一条描述的全部内容，逐字逐句输出，绝对不能概括、缩写、省略或精简任何内容。每个工作职责、项目描述、技能说明都必须完整呈现原文的所有细节。
2. **语言检测**：先识别简历语言（中文/英文/日文/韩文/法文/德文/西班牙文等）
3. **保留原文语言**：字段值保留简历原文语言，不进行翻译
4. **日期格式**：统一为 "YYYY年MM月" 或 "YYYY.MM" 格式
5. **跳过空模块**：简历中未提及的信息模块直接跳过，不要展示空模块，也不要展示 "⚠️ 该部分信息未在简历中找到" 提示
6. **容错处理**：OCR 可能存在误差，不确定的内容标注在末尾备注中

## 需要提取的信息模块

按以下顺序依次提取（仅展示简历中实际包含的模块）：

1. **基本信息**：姓名、性别、出生日期、年龄、电话、邮箱、现居地、国籍、LinkedIn、个人网站/作品集、个人简介/自我评价
2. **教育经历**（按时间倒序）：学校名称、学位、专业、起止时间、GPA
3. **工作/实习经历**（按时间倒序）：公司名称、职位、起止时间（在职标注"至今"）、工作职责与业绩（逐条完整列出，必须保留原文每一条的全部描述文字，不得缩减）
4. **项目经历**：项目名称、角色、起止时间、项目描述（完整保留原文）、技术栈、项目成果（逐条完整列出）
5. **技能专长**：按类别分组（编程语言、框架、数据库、工具等），每个技能项的描述必须完整保留
6. **语言能力**：语言名称、熟练程度、相关证书
7. **证书资质**：证书名称、颁发机构、获得时间
8. **荣誉奖项**：奖项名称、获奖时间、说明
9. **论文发表**：论文标题、发表刊物/会议、发表时间
10. **兴趣爱好**

## 输出格式

请严格按照下方的 Markdown 排版模板输出可读性强的格式化文本，**不要输出 JSON**，**不要使用 ASCII 边框字符**。

OCR 原始文本：
---
{raw_text}
---

请完整提取所有信息并按照排版模板输出，不要删减任何内容。
```

## 格式化规则

1. **完整性第一（最重要）**：必须完整保留简历中每一条描述的全部文字，逐字逐句输出，绝对禁止概括、缩写、省略、精简或改写任何内容。如果原文一条职责有 3 句话，输出也必须是完整的 3 句话
2. **仅展示非空模块**：如果某个信息模块在简历中完全不存在，则跳过该模块，不要展示空模块，不要显示 "⚠️ 该部分信息未在简历中找到" 之类的提示
3. **使用 Markdown 排版**：使用标准 Markdown 格式（标题、表格、列表、分隔线、引用等），**不要使用 ASCII 边框字符**（如 ┌─┐│└┘├┤═ 等），因为中英文混排时这些字符无法对齐
4. **日期格式化**：
   - `start_date` 有值但 `end_date` 为 null 时，显示为 "至今"
   - 日期格式统一为 "YYYY年MM月" 或 "YYYY.MM"
5. **列表项处理**：职责、成果、技能等列表项使用 Markdown 无序列表（`-`）逐条展示
6. **分隔线统一**：每个模块之间用 Markdown 分隔线 `---` 分隔
7. **特殊标注**：如果 OCR 识别存在误差或不确定内容，在末尾备注中说明
8. **一句话摘要**：在最末尾用引用格式输出一句话总结简历亮点

## 注意事项

1. **完整性第一（最重要）**：必须完整保留简历原文的每一条内容，绝对禁止概括、删减、精简。每条职责、每条成果都要完整输出原文
2. **使用 Markdown 排版**：使用标准 Markdown 格式输出，禁止使用 ASCII 边框字符（┌─┐│└┘├┤═ 等），避免中英文混排导致的对齐问题
3. **跳过空模块**：简历中不存在的信息模块直接跳过，不要显示空模块或 "未找到" 提示
4. **语言检测优先**：在解析前先识别简历语言，不同语种的简历在版式、惯用写法上有差异
5. **保留原文语言**：字段值保留简历原文语言，不进行翻译，确保信息不失真
6. **容错处理**：OCR 可能存在识别误差，对于模糊或不确定的内容标注在末尾备注中
7. **隐私合规**：输出结果可能包含个人隐私信息，使用时需注意数据安全与合规
8. **最终输出格式**：**必须**输出 Markdown 格式的可读文本，**不要输出 JSON 数据**，**不要使用 ASCII 边框字符**
