# 数据获取能力清单与降级策略

## 架构设计

面向SaaS场景：运营方在飞书AI账号层面预接通权威数据源API，终端用户零配置直接使用。

三层优先级：`权威API（待接入） > 免费公开数据（当前可用） > web搜索（兜底）`

---

### 缓存机制

所有API调用结果自动缓存到本地SQLite数据库（`memory/vc-cache.db`），相同查询不重复调用，节省API费用。

| 数据类型 | 缓存时长 | 原因 |
|---------|---------|------|
| 企业工商详情 | 30天 | 变化极慢 |
| 股权结构/团队 | 14天 | 偶有变更 |
| 知识产权 | 7天 | 新专利周期 |
| 司法风险 | 3天 | 诉讼变化快 |
| 舆情监控 | 6小时 | 需较实时 |
| 政策文件 | 1天 | 每天可能有新政策 |
| 行业数据 | 3天 | 变化频率中等 |

## 降级策略

当权威API未接入时，自动降级到免费公开数据源：

```
天眼查(企业工商) → 东方财富+雪球+web搜索
人民数据(政策舆情) → 央行+发改委+web搜索
IT桔子(融资事件) → 36氪API+清科+web搜索
Wind(金融数据) → 东方财富+同花顺+雪球+公开财报
NMPA(审评进度) → NMPA官网+医药魔方+FDA+web搜索
```

**当前（API未接入）的输出质量：**
- 宏观政策分析：✅ 可用（央行数据已验证）
- 行业趋势：⚠️ 基础可用（36氪快讯+公开信息）
- 赛道企业清单：⚠️ 基础可用（公开搜索+36氪，企业数据不够精确）
- 企业评估：⚠️ 基础可用（缺少工商、股权、知识产权等精确数据）

**API接入后的输出质量：**
- 宏观政策分析：✅✅ 精确（人民数据+央行+发改委）
- 行业趋势：✅✅ 精确（IT桔子融资数据+Wind行业数据）
- 赛道企业清单：✅✅ 精确（天眼查搜索+筛选）
- 企业评估：✅✅ 精确（天眼查完整画像+舆情风险检查）

## 安全规则

- API Key仅存储在环境变量或本地memory文件中
- 不在对话输出中暴露完整Key
- 不转发Key到任何第三方
- 日志不记录API响应中的敏感数据
