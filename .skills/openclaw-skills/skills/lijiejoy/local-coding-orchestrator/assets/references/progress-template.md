# Intelligent Progress Feedback Template

Use this structure for progress updates:

## Status
- Codex: running/completed/failed
- Claude: running/completed/failed
- OpenCode: running/completed/failed

## Key output
- What changed since last check
- Which files or artifacts were created
- Whether results landed in the intended directory

## Assessment
- Is the task now actionable?
- Are results aligned or conflicting?
- Is there an environmental issue blocking progress?

## Next action
- Continue waiting
- Relaunch failed worker
- Merge results
- Ask user to choose between prototype vs maintainable path
