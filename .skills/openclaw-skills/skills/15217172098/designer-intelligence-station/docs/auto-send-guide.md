# 设计师情报站 - 自动发送日报指南

## 📬 输出优化说明

### v1.5.0 起，日报生成后会自动发送：

1. **📊 日报摘要** - 情报数量、分级统计、抓取成功率
2. **📄 MD 文件** - 完整日报（v1.3.3 格式）

---

## 🤖 Agent 执行流程

### 步骤 1: 执行抓取脚本

```bash
./execute_daily.sh
```

### 步骤 2: Agent 调用 web_fetch 抓取网页

根据 `/tmp/dis_daily/web_fetch_instructions.json` 中的列表，并行抓取网页源。

### 步骤 3: 合并结果

```bash
python3 tools/fetch_all.py --merge \
  /tmp/dis_daily/rss_items.json \
  /tmp/dis_daily/api_items.json \
  /tmp/dis_daily/web_items.json \
  --output /tmp/dis_daily/all_items.json
```

### 步骤 4: 生成 v1.3.3 格式日报

Agent 读取 `/tmp/dis_daily/all_items.json`，按 5 维筛选标准判断，生成：
- `temp/intelligence-daily-YYYY-MM-DD.md`

### 步骤 5: 发送给用户

**发送内容**：
1. 📊 **日报摘要**（文字消息）
2. 📄 **MD 文件**（附件）

**发送方式**：
```python
# 使用 message 工具发送
message(
  action="send",
  channel="openim",
  target="group_713131094",  # 或用户 ID
  filePath="/path/to/intelligence-daily-YYYY-MM-DD.md",
  caption="📊 设计师情报站 · YYYY-MM-DD 日报（v1.3.3 格式 · XX 条情报）"
)
```

## 🔗 详情链接规范（v1.6.0 新增）

### 核心要求

**每一条情报都必须提供「详情」超链接入口**，让用户可以点击跳转查看原文。

### 表格格式规范

**所有情报表格必须包含「详情」列**，格式统一为：

```markdown
| 公司/机构 | 事件 | 影响 | 来源 | 详情 |
| --- | --- | --- | --- | --- |
| 谷歌 | Stitch 更新 | 形成新规范 | 优设网 | [→ 详情](https://example.com) |
```

**详情列格式**: `[→ 详情](完整 URL)`

### 链接覆盖范围

| 内容类型 | 链接要求 |
|----------|---------|
| **头条情报** | 每条必须有原文链接 |
| **手机/AI/硬件/设计领域** | 每条必须有原文链接 |
| **周刊内容** | 所有产品/工具/素材必须有链接 |
| **趋势洞察** | 底部添加「📎 相关阅读」链接（2-3 条） |
| **今日设计思考** | 可选添加相关案例链接 |

### 链接来源优先级

1. **原始来源链接** — 情报对应的原文 URL
2. **深度阅读链接** — 相关分析文章
3. **官方链接** — 产品官网/GitHub 仓库
4. **中文翻译** — 如有官方中文版优先提供

### 禁止行为

- ❌ 表格缺少「详情」列
- ❌ 详情链接为空或占位符
- ❌ 使用短链接/重定向链接（除非原文如此）
- ❌ 趋势洞察无「相关阅读」链接

### 示例对比

**❌ 错误（无详情列）**:
```markdown
| 公司/机构 | 事件 | 影响 | 来源 |
| --- | --- | --- | --- |
| 谷歌 | Stitch 更新 | 形成新规范 | 优设网 |
```

**✅ 正确（有详情列）**:
```markdown
| 公司/机构 | 事件 | 影响 | 来源 | 详情 |
| --- | --- | --- | --- | --- |
| 谷歌 | Stitch 更新 | 形成新规范 | 优设网 | [→ 详情](https://www.ftium4.com/Google-Stitch-new-features-5-trends-in-AI-design-UI-in-advance.html) |
```

### 周刊深度抓取链接规范

**Moonvy 设计素材周刊 / 优设网体验碎周报**:

| 栏目 | 链接要求 |
|------|---------|
| 设计资讯 | 每条资讯必须有详情链接 |
| 产品推荐 | 产品名称 + 详情链接 |
| 设计素材 | 素材名称 + 详情链接 |
| 大产品小细节 | 每个细节必须有详情链接 |
| 产品发现 | 每个产品必须有详情链接 |
| 推荐阅读 | 每篇文章必须有详情链接 |

**示例**:
```markdown
| 产品 | 描述 | 详情 |
|------|------|------|
| Motion Core | GSAP/Three.js 动画组件库 | [→ 详情](https://motion-core.dev) |
```

## 📰 周刊深度抓取执行规范（v1.6.0）

### 适用周刊

| 周刊名称 | 更新频率 | 抓取优先级 |
|----------|---------|-----------|
| Moonvy 设计素材周刊 | 每周 | P0（必须深度抓取） |
| 优设网体验碎周报 | 每周 | P0（必须深度抓取） |

### 抓取流程

```
步骤 1: 检测 RSS 源中的周刊条目
   ↓
步骤 2: 提取期数，构建详情页 URL
   ↓
步骤 3: 并行调用 web_fetch 抓取详情页
   ↓
步骤 4: 解析 Markdown 内容，按栏目分类
   ↓
步骤 5: 结构化输出到日报「🎨 设计领域」板块
```

### URL 构建规则

**Moonvy 设计素材周刊**:
```
基础 URL: https://moonvy.com/blog/post/设计素材周刊/
期数格式：{期数}/
示例：206 期 → https://moonvy.com/blog/post/设计素材周刊/206/
```

**优设网体验碎周报**:
```
基础 URL: https://www.ftium4.com/ux-weekly-
期数格式：{期数}.html
示例：273 期 → https://www.ftium4.com/ux-weekly-273.html
```

### web_fetch 调用示例

```python
# Moonvy 设计素材周刊 206 期
web_fetch(
  url="https://moonvy.com/blog/post/设计素材周刊/206/",
  extractMode="markdown"
)

# 优设网体验碎周报 273 期
web_fetch(
  url="https://www.ftium4.com/ux-weekly-273.html",
  extractMode="markdown"
)
```

### 输出格式要求

**1. 周刊标题格式**:
```markdown

```

**2. 表格格式**（所有栏目）:
```markdown
| 产品/主题 | 描述 | 详情 |
|----------|------|------|
| 产品名称 | 1-2 句核心描述 | [→ 详情](链接) |
```

**3. 链接保留规则**:
- ✅ 必须保留所有原始链接
- ✅ 链接格式：`[→ 详情](完整 URL)`
- ✅ GitHub/App Store/Chrome 商店等特殊链接保留原格式

### 禁止行为

- ❌ 仅输出 RSS 摘要（如"设计素材周刊 206 期"无具体内容）
- ❌ 省略产品/工具链接
- ❌ 不分类别混合呈现
- ❌ 缺少「详情」列
- ❌ 手动输入期数（必须从 RSS 自动提取）
