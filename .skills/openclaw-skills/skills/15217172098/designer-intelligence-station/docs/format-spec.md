## 详情链接规范

### 核心要求

**每一条情报都必须提供「详情」超链接入口**，让用户可以点击跳转查看原文。

### 表格格式

**所有情报表格必须包含「详情」列**：

```markdown
| 公司/机构 | 事件 | 影响 | 来源 | 详情 |
| --- | --- | --- | --- | --- |
| 谷歌 | Stitch 更新 | 形成新规范 | 优设网 | [→ 详情](https://example.com) |
```

**详情列格式**: `[→ 详情](完整 URL)`

### 链接覆盖范围

| 内容类型 | 链接要求 |
|----------|---------|
| 头条情报 | 每条必须有原文链接 |
| 手机/AI/硬件/设计领域 | 每条必须有原文链接 |
| 周刊内容 | 所有产品/工具/素材必须有链接 |
| 趋势洞察 | 底部添加「📎 相关阅读」链接（2-3 条） |

### 禁止行为

- ❌ 表格缺少「详情」列
- ❌ 详情链接为空或占位符
- ❌ 使用短链接/重定向链接
- ❌ 趋势洞察无「相关阅读」链接

## 周刊深度抓取规范

### 适用周刊

| 周刊名称 | 更新频率 | 抓取优先级 |
|----------|---------|-----------|
| Moonvy 设计素材周刊 | 每周 | P0（必须深度抓取） |
| 优设网体验碎周报 | 每周 | P0（必须深度抓取） |

### 抓取流程

```
步骤 1: 检测 RSS 源中的周刊条目
   ↓
步骤 2: 提取期数，构建详情页 URL
   ↓
步骤 3: 并行调用 web_fetch 抓取详情页
   ↓
步骤 4: 解析 Markdown 内容，按栏目分类
   ↓
步骤 5: 结构化输出到日报「🎨 设计领域」板块
```

### URL 构建规则

**Moonvy 设计素材周刊**:
```
基础 URL: https://moonvy.com/blog/post/设计素材周刊/
期数格式：{期数}/
示例：206 期 → https://moonvy.com/blog/post/设计素材周刊/206/
```

**优设网体验碎周报**:
```
基础 URL: https://www.ftium4.com/ux-weekly-
期数格式：{期数}.html
示例：273 期 → https://www.ftium4.com/ux-weekly-273.html
```

### 期数自动提取

**从 RSS 标题提取期数**:
```python
# Moonvy 示例
标题："设计素材周刊 206 期"
提取：期数 = 206
URL: https://moonvy.com/blog/post/设计素材周刊/206/

# 优设网示例
标题："体验碎周报第 273 期（2026.3.23）"
提取：期数 = 273
URL: https://www.ftium4.com/ux-weekly-273.html
```

### 输出格式要求

**周刊标题格式**:
```markdown
### 📖 Moonvy 设计素材周刊 {期数} 期（深度抓取）
### 📖 优设网体验碎周报 {期数} 期（深度抓取）
```

**表格格式**（所有栏目）:
```markdown
| 产品/主题 | 描述 | 详情 |
|----------|------|------|
| 产品名称 | 1-2 句核心描述 | [→ 详情](链接) |
```

### 禁止行为

- ❌ 仅输出 RSS 摘要（如"设计素材周刊 206 期"无具体内容）
- ❌ 省略产品/工具链接
- ❌ 不分类别混合呈现
- ❌ 缺少「详情」列
- ❌ 手动输入期数（必须从 RSS 自动提取）

## 趋势洞察格式

### 三层结构

```markdown
## 💡 趋势洞察：[趋势名称]

### 👁️ 观察（事实层）
- 客观描述现象
- 关键信号 1、2、3

### 🧠 分析（解读层）
- 为什么现在发生？
- 设计师视角的解读

### 🔮 预测（预判层）
- 短期影响（1-3 个月）
- 中期影响（3-12 个月）

📎 **相关阅读**: [链接 1](url) | [链接 2](url)
```

### 示例

```markdown
## 💡 趋势洞察：AI 视频生成遇冷：从 hype 到现实

### 👁️ 观察（事实层）
- OpenAI 宣布关闭 Sora，推出仅 15 个月
- 迪士尼十亿美元合作随之放弃
- 缺乏可持续的用户价值

### 🧠 分析（解读层）
- 纯 AI 社交 feed 模式尚未找到产品 - 市场契合点
- 技术能力强≠产品成功
- 视频生成技术仍在，但应用场景需要重新思考

### 🔮 预测（预判层）
- 短期：AI 视频公司将转向 B 端工作流
- 中期：C 端应用将整合到现有平台中

📎 **相关阅读**: [OpenAI 关闭 Sora 详情](https://www.theverge.com/ai-artificial-intelligence/899850/openai-sora-ai-chatgpt)
```
