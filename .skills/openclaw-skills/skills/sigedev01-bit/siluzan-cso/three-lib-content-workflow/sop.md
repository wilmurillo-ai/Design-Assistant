# 文案生成 SOP

## 1. 先定账号人设
阅读 `references/persona.md`获取人设列表让用户选择人设
这一步是总开关。

**人设不定，后面全会跑偏。**
## 2. 再定内容目标
先想清楚这篇文案是为了什么：

- 拉流量
- 做认知占位
- 做品牌势能
- 做转化
- 做私信咨询
- 做评论互动
- 做系列内容的一篇

不同目标，写法完全不同。

## 3. 选题与热点匹配
如果是热点型内容，就先做筛选：
- 这个热点够不够大
- 够不够反差
- 能不能讲出“生意判断”
- 是否适合对应账号
- 能不能翻译成老板听得懂的话
- 全球局势
- 供应链重构
- 成本传导
- AI、电力、出海、产业带

## 4. 拆素材
把原始信息拆成可用素材：
- 事实
- 数据
- 案例
- 风险点
- 术语
- 场景
- 争议点
- 可讲的人话版本

这一步是在回答：
**“我到底要写什么内容。”**

## 5. 找流量因子
看这篇内容的钩子在哪里：

- 反差
- 冲突
- 焦虑
- 稀缺
- 趋势
- 认知差
- 价格变化
- 全球秩序变化

这一步是在回答：
**“它为什么会有人看。”**

## 6. 匹配产品资产
如果内容里涉及产品、服务、行业能力，就要接到产品资产库：

- 产品卖点是什么
- 能解决什么问题
- 适合什么场景
- 能沉淀成什么长期资产
- 哪些内容可以系列化

这一步是在回答：
**“它能不能变成长期可复用的东西。”**

## 7. 选烹调方法
接着从烹调方法库里挑结构：

常见有：
- 反常识开头
- 观点先行
- 现象 → 本质
- 案例 → 判断
- 问题 → 解法
- 风险预警 → 行动建议
- 冲突 → 结论 → 升维

这一步是在回答：
**“这篇怎么讲最顺、最好看。”**

## 8. 组织成提纲
先出提纲，不直接写正文。

常见提纲是：

1. 开头钩子  
2. 核心判断  
3. 事实/案例支撑  
4. 生意启示  
5. 行动建议/收口

提纲的作用是先确认逻辑，不让正文写散。

## 9. 生成正文
按照人设 + 目标 + 结构 + 素材，写成完整文案。

这里会同时兼顾：

- 好懂
- 有节奏
- 有判断
- 有画面
- 有结尾

## 10. 质检
生成后要检查：
- 有没有老板视角
- 有没有趋势判断
- 有没有“这事和我有关”的感觉
- 有没有生意感
- 是否足够强势、有画面
- 有没有实操步骤
- 是否新手能照着做
- 是否够具体
- 有没有避坑点
- 能不能引导咨询 / 私信

## 11. 根据用途再改
最后再按用途微调：

- **爆款版**：更强钩子、更冲突
- **转化版**：更实操、更落地
- **口播版**：更短句、更口语
- **视频号版**：更稳、更顺、更自然
- **标题版**：更短、更有钩子

## 总结
**先定人设，再定目标；先找素材，再找流量因子；先选结构，再写正文；最后按账号用途做质检和改写。**

## 总链路
**人设 → 目标 → 选题 → 拆素材 → 找因子 → 配资产 → 选结构 → 出提纲 → 写正文 → 质检 → 改写**
