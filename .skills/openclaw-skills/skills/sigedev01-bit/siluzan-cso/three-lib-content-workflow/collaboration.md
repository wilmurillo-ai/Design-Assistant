# 三库协作说明

## 总原则
- **流量因子库负责选题**
- **产品资产库负责价值**
- **烹调方法库负责表达**

## 协作逻辑
### 1. 流量因子库：决定“写什么能被看见”
负责判断：
- 题材有没有热点
- 有没有反差
- 有没有情绪点
- 有没有认知差
- 有没有传播性

这是**选题和流量入口**。

### 2. 产品资产库：决定“写什么能长期有价值”
负责判断：
- 这篇内容对应什么产品/服务/能力
- 能不能沉淀成长期资产
- 能不能复用
- 能不能变成系列内容

这是**内容背后的商业沉淀层**。

### 3. 烹调方法库：决定“怎么写最好看”
负责判断：
- 开头怎么抓人
- 中间怎么推进
- 转折怎么做
- 结尾怎么收
- 用什么结构最顺

这是**表达和结构层**。

## 工作顺序
1. 先从流量因子库挑题。
2. 再从产品资产库找落点。
3. 再从烹调方法库选结构。
4. 最后生成文案。
5. 再做质检和改写。

## 一句话总结
**流量因子库负责选题，产品资产库负责价值，烹调方法库负责表达。**

## 类比
- **流量因子库 = 选菜**
- **产品资产库 = 看这道菜有没有营养**
- **烹调方法库 = 怎么把菜做得好吃**
