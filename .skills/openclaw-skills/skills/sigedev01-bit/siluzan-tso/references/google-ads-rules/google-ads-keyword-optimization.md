## 一、优化闭环总览

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  拉取数据    │ ──→ │  分析诊断    │ ──→ │  执行动作    │ ──→ │  验证效果    │
│ google-     │     │ 指标对比     │     │ 加词/减词    │     │ 同比/环比    │
│ analysis    │     │ 搜索词分桶   │     │ 调匹配/加否定│     │ 2-4周等待    │
└─────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
       ↑                                                              │
       └──────────────────────────────────────────────────────────────┘
```

**核心原则**：每次优化只改一个变量（关键词、匹配类型、出价、落地页），等待 2-4 周验证，再进入下一轮。

## 二、核心指标解读与判断层级

### 2.1 三层指标体系

| 层级 | 指标 | 用途 | CLI 数据来源 |
|---|---|---|---|
| **决策层**（先看） | CPA / ROAS / 转化率 | 决定关键词去留 | `google-analysis keywords` → `costPerConversion`、`conversionRate` |
| **诊断层**（解释原因） | CTR / CPC / Quality Score | 解释为什么决策指标好或差 | 同上 → `ctr`、`averageCpc`、`qualityScore` |
| **机会层**（发现增长） | 搜索展示份额 / 因排名丢失 / 因预算丢失 | 发现被抑制的潜力 | 同上 → `searchImpressionShare`、`searchRankLostImpressionShare`、`searchBudgetLostImpressionShare` |

### 2.2 Quality Score 对 CPC 的实际影响

| QS | CPC 相对基准 | 行动 |
|---|---|---|
| 10 | ~50% 折扣 | 维持，不动 |
| 8-9 | ~25% 折扣 | 优秀，微调即可 |
| 6-7 | 基准线 | 正常，按优先级逐步提升 |
| 4-5 | ~30% 溢价 | 需要优化广告相关性或落地页 |
| 1-3 | ~100% 溢价（2 倍成本） | 紧急：换文案/换落地页/暂停 |

### 2.3 2025 行业基准参考

| 指标 | 搜索网络平均值 | 说明 |
|---|---|---|
| CTR | ~6.4% | 低于 2% 通常说明广告相关性差 |
| CPC | $1-$9（行业差异大） | 单独看无意义，须结合转化率 |
| 转化率 | 4.4-7.0% | 关键词转化率低于账户均值一半 → 需关注 |
| CPA | $30-$100+（行业差异大） | 与业务利润目标挂钩 |

> **CPC 可承受上限公式**：`Max CPC = 目标 CPA × 转化率`
> 例：目标 CPA = $50，转化率 = 5% → Max CPC = $2.50

## 三、关键词决策框架

### 3.1 暂停关键词

| 条件 | 阈值 | 说明 |
|---|---|---|
| 零转化 + 高花费 | 花费 > 2× 目标 CPA 且 0 转化 | 如目标 CPA=$50，花费>$100 仍无转化 |
| 持续高 CPA | CPA > 1.5× 目标，持续 14 天，≥3 次转化 | 排除偶发因素 |
| QS 崩塌 | QS ≤ 3 持续 2 周以上 | 换文案无效后暂停 |
| 最低数据量 | 至少 20-30 次点击才做判断 | 低量词需更长观察期 |

> ⚠️ 暂停前**必须检查转化延迟**（见第六章）。近期数据可能有未归因转化。

### 3.2 提升出价

| 条件 | 操作 | 上限 |
|---|---|---|
| CPA < 目标 且 因排名丢失展示份额 > 20% | 提升出价 | 单次 +20-30% |
| 转化率 > 账户均值 且 展示份额 < 70% | 提升出价 | `新出价 = 当前出价 × (目标CPA / 实际CPA)` |

### 3.3 降低出价

| 条件 | 操作 | 下限 |
|---|---|---|
| CPA 为目标的 1.2-1.5× | 降低出价 | 单次 -15-20% |
| 展示份额 > 80% 但转化率低于均值 | 降低出价 | 过度竞价，降位后 CPA 可能改善 |

### 3.4 调整匹配类型

| 场景 | 方向 | 前提 |
|---|---|---|
| 精确匹配表现好但量太低（<50 展示/周） | Exact → Phrase/Broad | 配合密切监控搜索词 |
| 广泛匹配 >30% 花费在无关查询 | Broad → Exact + 加否定 | 先补否定词再调类型 |
| 测试广泛匹配 | 用 Experiments 50/50 分流 | 需 ≥30 转化/月 + Smart Bidding |

### 3.5 移到其他广告组

| 信号 | 说明 |
|---|---|
| 关键词意图与组内其他词明显不同 | 信息类词混在交易类组里 |
| CTR 低于广告组均值 30%+ | 文案无法同时覆盖 |
| QS 被拖低 | 广告相关性评分「低于平均」 |

### 3.6 加为否定关键词

| 条件 | 否定层级 |
|---|---|
| 搜索词明显无关（1 次点击即可判断） | 广告组/系列级 |
| 10+ 点击 0 转化 | 广告组级（精确否定） |
| 花费 > 3× 目标 CPA 且 0 转化 | 系列级 |
| 发现无效主题模式（如 "free"、"jobs"） | 共享否定词列表（短语否定） |

## 四、搜索词报告分析工作流

### 4.1 四桶分类法

拉取搜索词数据后，将每条搜索词分入四个桶：

| 桶 | 条件 | 动作 |
|---|---|---|
| **A — 提升** | 高转化率 + CPA ≤ 目标，≥3 次转化 | 加为 Exact Match → 专属广告组 + 专属文案 |
| **B — 观察** | 有点击无转化但意图相关 | 保留 1-2 周再评估 |
| **C — 否定（词级）** | 明显无关或 10+ 点击 0 转化 | 加为精确匹配否定词 |
| **D — 否定（主题级）** | 暴露一类无效流量的模式词 | 加为短语/广泛否定词 |

### 4.2 N-Gram 模式挖掘

不逐条处理，而是找高频出现在无效搜索词中的**单词/双词组合**：

```
步骤：
1. 导出 30 天搜索词，筛选 0 转化项
2. 拆词统计频率：哪些单词在无效查询中反复出现？
3. 高频无效词 → 一条广泛/短语否定词 = 屏蔽数千变体
```

> 例：发现 "free" 出现在 47 条无效查询中 → 加 "free" 为广泛否定词（但先确认 "hassle free"、"free shipping" 等是否也会被误伤）。

### 4.3 查询雕刻（Query Sculpting）

通过否定词**主动引导**查询流向最佳广告组/系列：

| 目标 | 操作 |
|---|---|
| 紧急查询由紧急系列承接 | 通用系列加否定 "emergency"、"urgent" |
| 品牌系列保持纯净 | 品牌系列加否定通用词 |
| 高客单查询由高客单系列承接 | 低客单系列加否定高端修饰词 |

### 4.4 搜索词可见性限制（2025-2026）

Google 因隐私原因减少可见搜索词数量，很多长尾归入「其他搜索词」。应对方式：

- 使用 **Search Term Insights**（主题聚类视图）替代逐条分析
- 关注趋势和主题，不执着于每条查询
- AI Max / Broad Match 生成的独特查询比报告可见数量多得多

## 五、优化节奏与任务清单

### 5.1 每日（5-10 分钟）

```
□ 检查预算消耗进度：系列是否因预算受限？
  → google-analysis campaigns -a <CID>  查看 searchBudgetLostImpressionShare
□ 检查异常：CPC 突增、CTR 骤降、转化追踪是否正常
  → google-analysis overview -a <CID>
□ 检查广告是否被拒审
```

### 5.2 每周（30-60 分钟）— 核心优化周期

```
□ 搜索词报告分析（四桶分类）
  → google-analysis search-terms -a <CID> --limit 100
  → 加否定：ad keyword-negative-create
  → 加新词：ad keyword-create

□ 关键词效果审查
  → google-analysis keywords -a <CID> --limit 100
  → 花费 > 1× 目标 CPA 且 0 转化的词 → 标记，下周仍无转化则暂停

□ 出价微调
  → CPA 偏离目标 >20% 且数据充足（≥20 点击）的词调整出价

□ QS 检查
  → QS 周环比下降 ≥2 分的词 → 检查广告相关性和落地页
```

### 5.3 每月（2-4 小时）

```
□ 全量关键词审计
  → 按 CPA 排序，暂停底部 10%（数据充分的前提下）
  → 按匹配类型分析效果差异

□ 广告组重组
  → 高量词但表现分化 → 拆为独立广告组

□ 落地页交叉审查
  → 高 CTR 低转化词 → 落地页问题
  → 高跳出率关键词 → 意图不匹配

□ 预算再分配
  → 低效系列 10-20% 预算转移到高效系列

□ 否定词列表清理
  → 检查是否有否定词误屏有价值查询
```

### 5.4 每季度（半天）

```
□ 账户架构审查：系列结构是否匹配当前业务目标
□ 竞争格局分析：Auction Insights 90 天趋势
□ 关键词扩展调研：Keyword Planner + 搜索词挖掘 + 竞品工具
□ 归因模型审查：转化延迟是否变化，归因窗口是否合适
□ 季节性规划：下季度关键词列表和预算预案
```

## 六、转化归因与延迟处理

### 6.1 为什么这很重要

多数广告主使用 30 天点击归因窗口，但如果实际平均转化延迟只有 2 天，30 天窗口会高估关键词价值。反过来，在评估近期数据时，短窗口内的转化可能尚未归因。

### 6.2 检查转化延迟

```
Google Ads → 工具 → 转化 → 转化路径 → 「转化天数」报告
也可用 → 归因 → 「时间滞后」报告
```

统计：80%+ 转化在几天内完成？

### 6.3 决策协议

| 原则 | 说明 |
|---|---|
| **评估窗口** | 评估关键词效果时，排除最近 N 天数据（N = 平均转化延迟） |
| **暂停等待期** | 暂停前等待 ≥ 2× 平均转化延迟的数据量 |
| **归因窗口调整** | 如 80% 转化在 7 天内完成，考虑从 30 天缩短到 7 天 |
| **对照指标** | 同时看「所有转化」与「转化」列，差异 = 延迟转化量 |

> 实测案例（SearchEngineLand 2026.03）：将归因窗口从 30 天缩短到 7 天后，ROAS 指标提升 62%（减少了与其他渠道的重复归因）。

### 6.4 Smart Bidding 与延迟

Smart Bidding 内部会考虑转化延迟，自动对近期数据打折。但更准确的归因窗口仍能提升其表现。最低数据要求：
- Target CPA：30-50 转化/30 天
- Target ROAS：50+ 转化/30 天

## 七、预算再分配策略

### 7.1 70/20/10 框架

| 分配 | 用途 | 允许 CPA |
|---|---|---|
| **70%** | 已验证的高效关键词/系列 | ≤ 目标 CPA |
| **20%** | 测试区：新词、新匹配类型、新受众 | ≤ 1.5× 目标 CPA |
| **10%** | 实验区：全新系列类型、竞品词、广泛发现 | 可接受更高 CPA |

### 7.2 月度再分配流程

```
1. 按效率排序所有系列（线索用 CPA，电商用 ROAS）
2. 找「饥饿赢家」：因预算丢失展示份额 >10% 且 CPA < 目标
   → google-analysis campaigns → searchBudgetLostImpressionShare
3. 找「预算捐赠者」：CPA > 1.5× 目标 持续 30 天 且 无改善趋势
4. 从捐赠者转移 10-20% 预算到饥饿赢家
5. 单次削减不超过 30%（避免算法震荡）
6. 2 周后验证效果再决定下一步
```

### 7.3 边际收益递减检测

- 预算 +20% 但 CPA 上升 >10% → 触达边际递减，回退到原预算
- 使用 Google Ads 的**预算模拟器**预估不同预算水平的增量转化

### 7.4 扩量阶梯

| 阶段 | 月花费 | 策略 |
|---|---|---|
| 验证期 | $0-10K | Exact Match 为主，验证单元经济 |
| 扩展期 | $10-30K | 加入 Phrase Match，扩展关键词主题 |
| 规模化 | $30K+ | 叠加 Broad + Smart Bidding + PMax |

> $20-30K 通常是「扩量墙」——Exact Match 流量用尽，必须拓宽匹配类型。

## 八、衰减检测与关键词生命周期

### 8.1 关键词见顶信号

| 信号 | 含义 |
|---|---|
| 展示份额 >90% 且 CPA 上升 | 已捕获几乎所有需求，继续推高只增成本 |
| 出价 +20% 但转化增量 <10% | 边际递减 |
| 搜索量持平或下降（Google Trends） | 市场需求在缩小 |
| CPA 连续 3 周上升，花费持平 | 关键词饱和 |
| CTR 持续下降，展示量稳定 | 广告疲劳或竞争加剧 |

### 8.2 见顶后的操作

| 操作 | 说明 |
|---|---|
| **锁定出价** | 不再提高，维持当前水平 |
| **横向扩展** | 搜索词报告 + Keyword Planner 找同义词、长尾变体、邻近意图词 |
| **刷新文案** | 新 headline/description 对抗创意疲劳 |
| **多渠道分流** | Search 饱和 → 测试 Demand Gen、YouTube、PMax |
| **季度复测** | 搜索行为变化快，3-6 个月后可能有新增量 |

## 九、竞争响应策略

### 9.1 竞争监控指标

通过 Auction Insights（如有 CLI 支持）或网页端导出，每月追踪：

| 指标 | 含义 |
|---|---|
| **展示份额** | 你的份额 vs 前 3-5 名竞争对手 |
| **重叠率** | 竞争对手与你同时展示的频率 |
| **排名胜出率** | 竞争对手排在你上方的频率 |
| **超越份额** | 你排在竞争对手上方（或他不展示时你展示）的综合比例 |

### 9.2 竞争响应决策树

| 场景 | 响应 |
|---|---|
| **新竞争者进入** | 观察 2-4 周，不急加价。多数新进者会测试后退出 |
| **竞争者加价（你的展示份额下降）** | CPA 仍达标 → 适度提价 10-15%（仅限高转化词）；CPA 已超标 → 不跟价，转向 QS 优化和文案差异化 |
| **竞争者出价你的品牌词** | 确保品牌系列在投 + 加满附加信息 + 考虑商标投诉 |
| **全行业 CPC 上涨** | 转向低竞争长尾词 + 加强 QS + 评估单元经济是否仍可行 |

## 十、落地页与关键词联动优化

### 10.1 关键词数据驱动落地页优化

| 关键词数据表现 | 诊断 | 落地页优化方向 |
|---|---|---|
| 高 CTR + 低转化率 | 广告吸引人但落地页不达标 | 检查：加载速度、文案匹配、CTA 清晰度、价格异议 |
| 低 CTR + 高转化率 | 广告不够吸引但落地页强 | 优化广告文案以匹配落地页优势 |
| 高跳出率关键词 | 搜索意图与落地页不匹配 | 创建更精准的落地页或调整关键词定向 |
| 移动端转化率远低于桌面端 | 移动落地页体验差 | 优先修复移动端 UX |

### 10.2 落地页检查清单（与 QS 直接相关）

```
□ 目标关键词出现在：页面标题、H1、首段、正文
□ 加载速度 < 2 秒（PageSpeed Insights 目标 90+）
□ 移动端响应式适配
□ HTTPS + 有效 SSL
□ 首屏有清晰的价值主张 + 与广告承诺一致
□ 单一明确的 CTA（不分散注意力）
□ 社会证明（评价/评分/案例）
□ 无阻挡性弹窗
```

### 10.3 高消费词专属落地页

对花费前 20 的关键词，创建专属落地页变体：
- 标题直接呼应搜索意图
- CTA 与广告文案的 CTA 一致
- 实测结果：专属落地页 + 文案优化 → CTR +67%、CPC -34%

## 十一、CRM / 离线数据反馈闭环

### 11.1 问题

Google Ads 优化的是你追踪的「转化」（表单、电话）。但很多线索永远不会成交。**没有离线数据，算法优化的是线索数量而非质量**。

### 11.2 实施路径

```
1. 表单提交时捕获 GCLID（Google Click ID）→ 存入 CRM
2. 线索推进（MQL → SQL → 机会 → 成交）时，上传转化事件回 Google Ads
3. 使用 Enhanced Conversions for Leads：附加 hashed email 提升匹配率
4. 设主转化目标为下游事件（如 SQL 或成交），而非表单提交
5. Smart Bidding 据此优化真正产生收入的行为
```

### 11.3 价值分层

| 阶段 | 赋值示例 | 说明 |
|---|---|---|
| 表单提交 | $10 | 量大但质量参差 |
| 合格线索（MQL） | $50 | 初筛通过 |
| 销售合格线索（SQL） | $200 | 进入销售流程 |
| 成交 | $2,000+ | 实际收入 |

所有阶段都上传 → Smart Bidding 学习完整漏斗。

### 11.4 关键词级洞察

```
运行报告：关键词 → GCLID → CRM 阶段
```

常见发现：
- 部分高量低 CPA 词产生的线索**从不成交** → 降低出价或暂停
- 部分高 CPA 词产生的线索**高成交率 + 高客单** → 值得加大投入

## 十二、自动化规则与脚本

### 12.1 Google Ads 自动规则模板

| 规则 | 条件 | 动作 | 频率 |
|---|---|---|---|
| 暂停高 CPA 零转化词 | CPA > $X 且转化 = 0 且时间窗 = 14 天 | 暂停 | 每天 |
| 提价受限赢家 | CPA < 目标 且 因排名丢失 > 20% | 出价 +15%，设上限 | 每周 |
| 降价超支词 | CPA > 1.3× 目标 且点击 > 30 | 出价 -10% | 每周 |
| 预算预警 | 系列花费 > 日预算 90%（下午 2 点前） | 发送邮件 | 每天 |

### 12.2 高价值脚本类别

| 脚本 | 功能 |
|---|---|
| 效果暂停脚本 | 扫描关键词，花费超阈值 0 转化 → 自动暂停 + 记录到 Google Sheet |

### 12.3 脚本安全协议

- 初始阈值设保守，逐步放宽
- 所有变更记录到 Google Sheet
- 每周 review 脚本操作日志

## 十三、CLI 命令实操流程

### 13.1 每周优化完整流程

```bash
# 1. 拉取账户总览（确认整体趋势）
siluzan-tso google-analysis overview -a <CID> --start <7天前> --end <昨天>

# 2. 拉取关键词效果（按花费降序）
siluzan-tso google-analysis keywords -a <CID> --limit 100

# 3. 拉取搜索词报告
siluzan-tso google-analysis search-terms -a <CID> --limit 100

# 4. 拉取系列数据（看预算受限情况）
siluzan-tso google-analysis campaigns -a <CID>

# 5. 根据分析结果执行动作

# 加否定词（搜索词分析结果中的 C/D 桶）
siluzan-tso ad keyword-negative-create --keywords "free,jobs,DIY" ...

# 提升新发现的高转化词为精确匹配
siluzan-tso ad keyword-create --keywords "[高转化词1],[高转化词2]" ...

# 暂停低效词
# （通过网页端或 ad keyword-update 操作）
```

### 13.2 月度审计流程

```bash
# 1. 拉取 30 天全量数据
siluzan-tso google-analysis keywords -a <CID> --start <30天前> --end <昨天> --limit 200

# 2. 拉取按日趋势（看 CPA 趋势）
siluzan-tso google-analysis daily-metrics -a <CID> --start <30天前> --end <昨天>

# 3. 设备分析（移动端 vs 桌面端转化差异）
siluzan-tso google-analysis devices -a <CID>

# 4. 地域分析（哪些地区效率高/低）
siluzan-tso google-analysis geographic -a <CID>

# 5. QS 诊断
siluzan-tso google-analysis ads-index -a <CID>

# 6. 根据分析调整关键词、预算、落地页
```

### 13.3 诊断速查

| 我看到... | 可能原因 | 检查命令 | 动作 |
|---|---|---|---|
| CPA 突然升高 | 竞争加剧 / 转化追踪故障 / 落地页问题 | `overview` + `keywords` + `campaigns` | 检查转化追踪 → 检查竞争 → 检查落地页 |
| CTR 持续下降 | 广告疲劳 / 竞争对手新文案 / QS 下降 | `keywords`（看 QS 列）+ `ads` | 刷新广告文案 → 优化 QS |
| 展示量骤降 | 预算耗尽 / 关键词暂停 / 被拒审 | `campaigns`（看预算丢失）+ `keywords` | 检查预算 → 检查广告状态 |
| 转化率下降、点击不变 | 落地页变动 / 季节性 / 受众漂移 | `daily-metrics` + `devices` + `geographic` | 检查落地页 → 分设备/地域诊断 |
| 花费集中在少数词 | 广泛匹配过度扩展 / 出价不均 | `keywords`（top 20 占比）+ `search-terms` | 加否定词 → 平衡出价 |

## 十四、信号污染纠正协议

当优化方向偏离（CPA 恶化、线索质量下降）时：

```
1. 检查转化目标：是否有弱转化事件（页面浏览、滚动）作为主目标？→ 移除
2. 检查客户列表：是否 >18 个月未更新？→ 上传新鲜数据
3. 检查系列隔离：品牌/非品牌是否混合？高低客单是否共用 ROAS 目标？→ 拆分
4. 刷新广告创意：疲劳的创意 = 衰退的信号
5. 等待 5-10 天稳定期，不做额外调整
```
