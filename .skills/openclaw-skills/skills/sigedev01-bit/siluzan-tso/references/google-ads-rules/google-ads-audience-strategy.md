# Google 广告受众策略与分析指南

> **所属 skill:** siluzan-tso
> **适用场景:** 受众数据分析（`google-analysis audience`）、受众策略建议、再营销策略规划、RLSA 策略
> **更新日期:** 2026-04-01

---

## 1. 受众类型全览

### 1.1 Google 预定义受众（SystemDefined）

通过 `siluzan-tso google-analysis audience -a <CID> --type SystemDefined` 查看表现。

| 类型 | 说明 | 典型用途 |
|------|------|----------|
| **In-market（有购买意向）** | 近期正在研究或比较特定品类产品的用户 | 漏斗中下层，获取高意向新客 |
| **Affinity（兴趣相似）** | 基于长期浏览行为反映生活方式和兴趣的用户 | 漏斗上层，品牌认知扩展 |
| **Life Events（生活事件）** | 即将经历重大生活变化的用户（搬家、毕业、结婚等） | 时间敏感型产品推广 |
| **Detailed Demographics（详细受众特征）** | 基于教育程度、婚姻状况、房屋所有权、家庭收入等 | 精细化人群定位 |

### 1.2 自定义受众（UserDefined）

通过 `siluzan-tso google-analysis audience -a <CID> --type UserDefined` 查看表现。

| 类型 | 说明 | 创建方式 |
|------|------|----------|
| **Custom Segments（自定义细分）** | 按关键词、URL、应用定义的用户群 | Google Ads UI → 受众管理器 |
| **Customer Match（客户匹配）** | 上传 CRM 数据（邮箱/电话/地址）匹配 Google 用户 | Google Ads UI → 受众管理器 → 上传列表 |
| **Website Visitors（网站访客再营销）** | 通过 Google Tag 追踪的网站访问者 | Google Ads UI → 受众管理器 → 创建规则 |
| **App Users（应用用户）** | 应用安装者、活跃用户、应用内事件用户 | Firebase/Google Analytics 连接 |

### 1.3 相似受众与组合细分

- **Similar/Lookalike 受众:** 已于 2023 年弃用，被**优化定位（Optimized Targeting）**和**受众扩展（Audience Expansion）**替代。不再需要手动创建。
- **Combined Segments（组合细分）:** 支持 AND/OR 逻辑组合多个受众。例如："In-market: 运动鞋" AND "25-34 岁男性"。在 Google Ads UI 中创建。

> **重要提示:** CLI 工具仅能分析受众数据，所有受众列表的创建和管理必须在 Google Ads UI 中完成。

## 2. 受众策略矩阵

按业务类型和漏斗阶段推荐受众组合：

### 2.1 电商（Ecommerce）

| 漏斗阶段 | 推荐受众 | 模式 | 预期效果 |
|----------|---------|------|----------|
| 拉新（Prospecting） | In-market 相关品类 + Affinity | 观察 → 验证后定向 | 触达高意向新客 |
| 考虑（Consideration） | 网站访客（浏览未加购，7-30d） | 定向，单独广告系列 | 推动加购行为 |
| 转化（Conversion） | 加购未购买（1-7d） | 定向，最高出价 | 挽回弃购用户 |
| 交叉销售（Cross-sell） | 已购买客户（30-90d） | 定向 | 提升 LTV |

### 2.2 线索获取（Lead Gen）

| 漏斗阶段 | 推荐受众 | 模式 | 预期效果 |
|----------|---------|------|----------|
| 拉新 | In-market 相关服务 + Detailed Demographics | 观察 | 发现高质量人群 |
| 转化 | 表单页访客（未提交，1-14d） | 定向，单独广告系列 | 挽回高意向线索 |
| 排除 | Customer Match（已签约客户） | 排除 | 避免浪费预算 |
| 追加销售 | Customer Match（现有客户，低层级） | 定向 | 升级服务套餐 |

### 2.3 SaaS

| 漏斗阶段 | 推荐受众 | 模式 | 预期效果 |
|----------|---------|------|----------|
| 拉新 | In-market "商业软件"相关 + Custom Segments（竞品 URL） | 观察 | 获取精准新客 |
| 试用转付费 | Trial 用户（注册未付费，7-14d） | 定向 | 促进付费转化 |
| 召回 | 流失用户（90-180d） | 定向，专属优惠素材 | Win-back |
| 排除 | 当前付费用户 | 排除 | 避免重复花费 |

## 3. 观察模式 vs 定向模式

这是受众设置中**最关键的区别**，选错模式会导致流量暴跌或预算浪费。

### 3.1 核心差异

| 维度 | 观察模式（Observation） | 定向模式（Targeting） |
|------|------------------------|----------------------|
| 覆盖范围 | 不缩小范围，面向所有用户展示 | 仅面向指定受众展示 |
| 出价调整 | 可对受众单独设置出价调整 | 自动仅投放给受众 |
| 数据收集 | 适合收集受众表现数据 | 适合已验证的高价值受众 |
| 风险 | 低（不影响现有投放） | 高（覆盖面可能大幅缩小） |

### 3.2 决策框架

```
第 1 步：新受众 → 始终以观察模式添加
第 2 步：收集 2-4 周数据（至少 1,000 次展示）
第 3 步：评估指标
  ├─ 受众 CPA < 账户平均 CPA × 0.8（低 20% 以上）→ 考虑切换为定向模式
  ├─ 受众 CPA ≈ 账户平均 CPA → 保持观察，调整出价 +10-20%
  └─ 受众 CPA > 账户平均 CPA × 1.2（高 20% 以上）→ 降低出价或移除
```

### 3.3 Smart Bidding 下的建议

使用 Smart Bidding（tCPA/tROAS/Maximize Conversions）时：
- **推荐默认使用观察模式** — 受众信号会自动被 Smart Bidding 纳入优化，无需手动限制
- 观察模式为 Smart Bidding 提供更多信号维度，不会限制其优化空间
- 仅在需要"完全隔离"特定受众（如再营销专属广告系列）时使用定向模式

## 4. 再营销策略深度

### 4.1 再营销列表分层（按时效）

| 层级 | 时间窗口 | 用户状态 | 出价调整建议 | 策略 |
|------|---------|---------|-------------|------|
| 热（Hot） | 1-7 天 | 近期活跃，意向最强 | +50% ~ +100% | 积极追投，紧迫性素材 |
| 温（Warm） | 8-30 天 | 有一定意向，尚未遗忘 | +20% ~ +50% | 持续曝光，强调差异化价值 |
| 凉（Cool） | 31-90 天 | 意向减弱，需唤醒 | +0% ~ +20% | 促销/限时优惠素材 |
| 冷（Cold） | 91-540 天 | 可能已遗忘品牌 | -20% 或仅观察 | 品牌提醒，低预算试探 |

> **注意:** 再营销列表最长有效期为 540 天。超过此期限的用户无法再追踪。

### 4.2 弃购/弃表单专项策略

针对加购未购买或表单页未提交的用户，建议创建**独立广告系列**：
- 独立预算分配（通常占再营销预算 40-60%）
- 出价高于常规广告系列 30-50%
- 广告素材针对性：强调产品/服务卖点、限时优惠、免运费等
- 频次上限：每天 3-5 次（避免过度骚扰）

### 4.3 排除已转化用户

**必须始终排除已转化用户**，排除窗口根据复购周期设定：
- 快消品：排除近 7-14 天转化用户
- 一般电商：排除近 30 天转化用户
- 高客单/低频：排除近 60-90 天转化用户
- B2B/SaaS：排除已签约客户（Customer Match 排除）

### 4.4 交叉销售再营销

已购买产品 A 的用户 → 向其展示产品 B/C 广告：
- 在 Google Ads UI 中基于转化事件或产品页创建受众
- 使用定向模式，独立广告系列
- 素材中可引用"感谢购买 A，您可能也需要 B"等信息

## 5. RLSA（搜索广告再营销列表）

RLSA 将再营销受众叠加到搜索广告系列上，实现"搜索意图 + 历史行为"双重精准。

### 5.1 前提条件

- 再营销列表最低用户数：**1,000**（搜索网络要求）
- 展示网络最低：**100**
- 需要 Google Tag 正确部署并持续回收数据

### 5.2 三大核心策略

**策略一：出价提升（Bid Boost）**
- 在现有搜索广告系列中以**观察模式**添加再营销受众
- 对回访用户提高出价 +20% ~ +80%
- 适用场景：通用关键词广告系列中，优先展示给已知用户
- 优势：不影响新客获取，同时提升回访用户转化

**策略二：宽泛匹配扩展（Broad Match + Audience）**
- 创建独立广告系列，使用广泛匹配关键词 + **定向模式**锁定再营销受众
- 适用场景：对已知用户放宽关键词限制，捕获更多搜索需求
- 关键词范围可以比常规广告系列宽泛 2-3 倍
- 风险可控：因为受众已是已知用户，广泛匹配不会浪费在完全无关人群

**策略三：独立再营销搜索广告系列**
- 完全独立的广告系列 + 定向模式 + 专属预算
- 关键词覆盖更广（含竞品词、行业通用词）
- 广告文案针对回访用户定制（如"欢迎回来""专属优惠"）
- 适用场景：再营销列表规模较大（>10,000 用户）且转化率显著高于新客

## 6. Customer Match 策略

### 6.1 核心用途

| 用途 | 说明 | 操作位置 |
|------|------|----------|
| 排除现有客户 | 在拉新广告系列中排除已签约/已购买客户 | Google Ads UI |
| 追加销售 | 向现有低层级客户推广升级方案 | Google Ads UI |
| 高 LTV 客户扩展 | 上传高价值客户列表，Google 自动寻找相似人群 | Google Ads UI（优化定位） |
| 流失召回 | 上传流失客户列表，投放 Win-back 广告 | Google Ads UI |

### 6.2 匹配率预期

| 数据类型 | 预期匹配率 | 提升建议 |
|----------|-----------|----------|
| 邮箱 | 30-60% | 优先使用 Gmail 地址，清洗格式 |
| 电话号码 | 20-40% | 包含国家代码，E.164 格式 |
| 邮寄地址 | 15-30% | 需姓名+地址配合 |
| 组合上传（邮箱+电话+地址） | 50-70% | **强烈推荐**，多信号交叉匹配 |

### 6.3 操作规范

- **数据更新频率:** 每月至少更新一次列表，保持数据新鲜度
- **隐私合规:** 数据上传前自动哈希处理（SHA-256），必须获得用户同意
- **最低列表规模:** 建议 1,000+ 条记录以确保有效匹配
- **所有操作在 Google Ads UI 中完成**，CLI 仅用于分析 Customer Match 受众表现

## 7. 受众数据分析框架

使用 CLI 工具分析受众数据的标准流程。

### 7.1 关键评估指标

| 指标 | 含义 | 判断标准 |
|------|------|----------|
| CTR（点击率） | 受众与广告的相关性 | 高于账户均值 → 受众匹配度好 |
| 转化率 | 受众的转化能力 | 核心评估指标 |
| CPA（单次转化费用） | 获客效率 | **低于账户均值 20% 以上为优质受众** |
| ROAS（广告支出回报率） | 投资回报 | 电商核心指标 |
| 展示份额 | 受众规模与竞争度 | 过低说明受众太窄或出价不足 |

### 7.2 分析工作流（CLI 命令序列）

```bash
# 第 1 步：拉取 Google 预定义受众数据
siluzan-tso google-analysis audience -a <CID> --type SystemDefined

# 第 2 步：拉取自定义受众数据
siluzan-tso google-analysis audience -a <CID> --type UserDefined

# 第 3 步：拉取广告系列数据作为基准对比
siluzan-tso google-analysis campaigns -a <CID>

# 第 4 步：拉取关键词数据辅助分析
siluzan-tso google-analysis keywords -a <CID>
```

### 7.3 受众表现 → 行动决策矩阵

| 受众表现 | CPA 对比账户均值 | 转化量 | 建议行动 |
|----------|-----------------|--------|----------|
| 优秀 | CPA < 均值 × 0.7 | ≥ 10 次转化 | 提高出价 +20-30%，考虑独立广告系列 |
| 良好 | CPA < 均值 × 0.8 | ≥ 5 次转化 | 提高出价 +10-20%，保持观察 |
| 一般 | CPA ≈ 均值（±20%） | ≥ 3 次转化 | 维持现状，继续观察 |
| 较差 | CPA > 均值 × 1.3 | < 3 次转化 | 降低出价 -20-30% |
| 极差 | CPA > 均值 × 1.5 | 高展示零转化 | 排除该受众 |

> **数据充分性提醒:** 做出决策前确保受众有足够展示量（建议 ≥ 1,000 次展示）和统计显著性。数据不足时保持观察，不急于调整。

## 8. 受众排除策略

排除策略常被忽视，但对预算效率影响巨大。

### 8.1 必须排除的受众

| 排除对象 | 适用广告系列 | 操作方式 |
|----------|-------------|----------|
| 已转化用户 | 拉新/获客广告系列 | Google Ads UI：创建转化用户列表并排除 |
| 现有客户 | 拉新广告系列 | Google Ads UI：Customer Match 上传排除 |
| 内部员工 | 所有广告系列 | Google Ads UI：IP 排除 + 内部邮箱列表排除 |
| 已知低质量流量 | 所有广告系列 | Google Ads UI：排除高跳出/零互动受众 |

### 8.2 条件性排除

| 排除条件 | 判断阈值 | 操作 |
|----------|---------|------|
| 已转化用户（防重复） | 窗口 = 复购周期（7-90 天不等） | 按业务类型设定排除窗口 |
| 高展示零转化受众 | 展示 > 2,000 且转化 = 0 | 排除或大幅降低出价 |
| 高跳出率受众 | 跳出率 > 80% 且页面停留 < 10s | 排除 |
| 频次过高用户 | 30 天内展示 > 15 次未转化 | 排除或加入冷却列表 |

### 8.3 排除清单审查频率

- **周度:** 检查是否有新的已转化用户需要排除
- **月度:** 审查受众表现，排除持续低效受众
- **季度:** 更新 Customer Match 排除列表，审查整体排除策略

## 9. 2025-2026 受众趋势

### 9.1 隐私优先时代

- **第三方 Cookie 淘汰影响:** Chrome 已逐步限制第三方 Cookie，传统基于 Cookie 的受众追踪能力下降
- **应对策略:** 加速第一方数据收集（注册、订阅、CRM），减少对第三方信号的依赖
- **Enhanced Conversions（增强型转化）:** 部署增强型转化以提高归因准确度

### 9.2 Google 优化定位（Optimized Targeting）

- 逐渐替代手动受众选择，Google AI 根据广告素材和转化信号自动寻找最佳受众
- **对策略的影响:** 观察模式成为主要使用方式，手动定向模式的使用场景收窄
- **建议:** 保持受众以观察模式添加，为 AI 提供信号参考而非硬限制

### 9.3 Customer Match 重要性上升

- 随着第三方信号减弱，第一方数据（Customer Match）成为最可靠的受众来源
- Google 持续提升 Customer Match 的匹配率和功能
- **建议:** 优先建设 CRM 数据管道，每月更新 Customer Match 列表

### 9.4 PMax 受众信号

Performance Max 广告系列中的受众信号使用方式：
- 受众信号是**引导**而非**限制** — Google AI 可能会超出信号范围投放
- 建议添加的信号：Customer Match 列表、网站访客列表、In-market 细分、Custom Segments
- 受众信号的质量直接影响 PMax 的冷启动速度和初期表现
- 通过 CLI 分析 PMax 广告系列的受众洞察来评估信号效果

## 10. CLI 操作流程

### 10.1 每周受众审查

```bash
# 1. 全量受众概览
siluzan-tso google-analysis audience -a <CID>

# 2. 分类型查看
siluzan-tso google-analysis audience -a <CID> --type SystemDefined
siluzan-tso google-analysis audience -a <CID> --type UserDefined

# 3. 对比广告系列基准
siluzan-tso google-analysis campaigns -a <CID>
```

**审查要点:**
- 各受众 CPA 与账户均值对比
- 新增/流失受众的表现变化
- 再营销列表的用户规模变化趋势

### 10.2 受众表现审计（月度）

执行完整的受众效率审计，按以下顺序：

```bash
# 第 1 步：获取所有受众数据
siluzan-tso google-analysis audience -a <CID>

# 第 2 步：获取广告系列数据作为基准
siluzan-tso google-analysis campaigns -a <CID>

# 第 3 步：获取关键词数据辅助分析
siluzan-tso google-analysis keywords -a <CID>
```

**阈值决策规则:**

| 条件 | 行动 | 执行方 |
|------|------|--------|
| CPA < 账户均值 × 0.7 且转化 ≥ 10 | 建议提高出价或创建独立广告系列 | 用户在 UI 中操作 |
| CPA 介于均值 × 0.7 ~ 0.9 且转化 ≥ 5 | 建议小幅提高出价 +10-20% | 用户在 UI 中操作 |
| CPA 介于均值 × 0.9 ~ 1.2 | 保持观察，下周期再评估 | 无需操作 |
| CPA > 均值 × 1.3 且转化 < 3 | 建议降低出价或考虑排除 | 用户在 UI 中操作 |
| 展示 > 2,000 且转化 = 0 | 建议立即排除该受众 | 用户在 UI 中操作 |

### 高效受众 TOP 5（按 CPA 排序）
| 受众名称 | 类型 | 展示 | 点击 | 转化 | CPA | vs 均值 |
