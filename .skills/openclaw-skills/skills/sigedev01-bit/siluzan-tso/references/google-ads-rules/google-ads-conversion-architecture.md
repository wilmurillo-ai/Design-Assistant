# Google 广告转化信号架构设计指南

> 所属 skill: siluzan-tso
> 适用场景: 转化追踪策略设计、信号质量诊断、离线转化导入规划、利润导向出价

---

## 1. 核心理念：信号质量决定投放上限

Smart Bidding 的效果上限由转化信号质量决定，而非广告素材或关键词。关键词平庸但信号精准的广告系列，长期 ROI 会优于关键词完美但信号粗糙的广告系列。原因：Smart Bidding 根据「什么行为带来了转化价值」做竞价决策——你喂什么信号，它就优化什么。

**信号质量阶梯**（从低到高）：

| 层级 | 信号类型 | Smart Bidding 决策质量 | 典型 tCPA 误差 |
|------|---------|----------------------|---------------|
| L1 | 页面浏览 Page View | 极差 — 无法区分意向 | ±80%+ |
| L2 | 表单提交 Form Submit | 基础 — 知道谁有兴趣 | ±40-60% |
| L3 | 合格线索 MQL | 良好 — 知道谁是真实潜客 | ±20-35% |
| L4 | 成交 + 收入 | 优秀 — 知道谁带来收入 | ±10-20% |
| L5 | 成交 + 利润 | 最佳 — 直接优化利润 | ±10-15% |

**关键原则**：
- 宁可信号少但准，不要信号多但杂：100 个精准成交信号优于 2000 个表单提交
- 数据量门槛：Smart Bidding 需要每月 30-50 个转化。信号不足 30 个/月时用上一层级
- 信号一致性比完美更重要：频繁更改转化定义会触发 2 周重学习期

## 2. 转化动作设计框架

### 按业务类型

| 业务类型 | 主要转化 | 次要转化 | 常见错误 |
|---------|---------|---------|---------|
| 电商 | Purchase（动态收入值，Every） | Add to Cart, Begin Checkout | — |
| 线索获取 | Qualified Lead（非表单提交） | Form Submit, Phone Call | 把表单提交设为主要 → 优化垃圾线索 |
| SaaS | Paid > SQL > MQL > Trial（按数据量选） | 上游漏斗事件 | 转化窗口太短 |
| 本地服务 | 电话 ≥60s 或 在线预约 | 方向导航, 短时电话 | 不设通话时长阈值 |

### 主要 vs 次要转化

- **主要（Primary）**：参与出价优化。每个转化目标类型下只应有 1 个主要转化
- **次要（Secondary）**：仅报告，不影响出价
- 错误示范：Purchase 和 Add to Cart 都设为 Primary → 重复计数、信号混乱

### 转化计数与窗口

**计数方式**：Every = 每次都有价值（购买）；One = 只关心是否发生（线索）

**点击归因窗口**：

| 业务类型 | 推荐窗口 | 理由 |
|---------|---------|------|
| 冲动消费 <$50 | 7-14 天 | 决策快，长窗口引入噪音 |
| 一般电商 $50-500 | 30 天 | 平衡精度与覆盖 |
| 高客单 >$500 | 60-90 天 | 多次研究对比 |
| B2B / SaaS | 90 天 | 决策链长 |

**浏览型窗口**：默认 1 天即可；展示/视频广告可设 3-7 天。

## 3. 多阶段漏斗价值设计

**核心公式**：阶段价值 = 该阶段→成交转化率 × 平均成交金额

**线索获取示例（法律行业）**：

| 漏斗阶段 | →成交率 | 案件均值 | 赋值 |
|---------|--------|---------|------|
| 表单提交 | 5% | $10,000 | $500 |
| 电话咨询 | 15% | $10,000 | $1,500 |
| 面谈预约 | 30% | $10,000 | $3,000 |
| 签约 | 100% | $10,000 | $10,000 |

**SaaS 示例**：Trial(8%→$6k=$480) → MQL(20%→$1,200) → SQL(40%→$2,400) → Paid(100%→$6,000)

**计算方法**：从 CRM 拉 6-12 个月数据 → 统计各阶段→成交转化率 → 乘以均值 → 每季度复核

### 优化层级选择（数据量权衡）

| 月度成交量 | 推荐优化层级 |
|-----------|------------|
| <15 | MQL 或表单提交（数据不足） |
| 15-30 | 可尝试成交，波动大 |
| 30-50 | 优化成交 |
| 50+ | 成交 + 利润值（理想状态） |

**迁移路径**：表单 → MQL → SQL → 成交。每级切换需 MQL ≥50/月后再升级，每次切换触发 ~2 周学习期。

## 4. Enhanced Conversions 实施要点

### EC for Web（网页增强型转化）

将哈希第一方数据（邮箱/电话/地址）发给 Google 匹配已登录账户，弥补 Cookie 丢失。预期恢复 5-15% 丢失归因。

实施方式：gtag（需开发）、GTM（推荐中小团队）、Google Ads API（最灵活）。匹配优先级：Email > Phone > Name+Address。

### EC for Leads（线索增强型转化）

无需存储 GCLID，通过邮箱匹配离线转化。流程：表单提交时 tag 发送哈希邮箱 → CRM 成交时上传同一邮箱 → Google 自动匹配。回溯窗口 63 天。

### 选择决策

| 场景 | 推荐 | 理由 |
|------|------|------|
| 纯电商 | EC for Web | 不涉及离线 |
| 线索获取，无 GCLID | EC for Leads | 无需改造 CRM |
| 线索获取，有 GCLID | 传统离线导入 + EC for Web | GCLID 匹配率接近 100% |
| SaaS 长周期 | EC for Leads + 传统导入并行 | 63 天窗口可能不够 |

## 5. 离线转化导入策略

### GCLID 导入

流程：点击广告(gclid) → 表单存 gclid → CRM 状态更新 → 上传 gclid+转化+时间+价值
- 存储：GCLID 最长 100 字符，VARCHAR(100)+，精确时间含时区，有效期 90 天
- 频率：理想每 24h，可接受 48h，最低每周（但延迟大）

### EC for Leads 导入

流程：表单→tag 发哈希邮箱 → CRM 更新→上传哈希邮箱+转化。无需 GCLID，63 天回溯。

### 匹配率

| 匹配率 | 状态 | 操作 |
|-------|------|------|
| <30% | 严重不足 | 检查 GCLID 捕获/邮箱格式 |
| 30-50% | 需改善 | 增加字段、修复数据 |
| 50-70% | 合格 | 持续优化 |
| >70% | 优秀 | 保持 |

提升技巧：邮箱统一小写去空格、GCLID 覆盖所有入口页、检查字段截断。

### CRM 延迟

| 行业 | 典型延迟 | 影响 |
|------|---------|------|
| 电商 | 即时-3天 | 无 |
| 本地服务 | 1-7天 | 轻微 |
| SaaS SMB | 7-30天 | 中等 |
| SaaS Enterprise | 30-180天 | 严重，必须上传中间阶段 |
| B2B 高客单 | 30-90天 | 严重 |

延迟 >14 天时，必须上传中间阶段（如 MQL，延迟仅 3-7 天）作为及时信号。

## 6. Consent Mode v2

自 2024 年 3 月 EEA 强制执行。允许 Google 在用户拒绝 Cookie 时通过建模估算缺失转化。

**两种模式**：
- **Basic**：未同意前不加载标签。合规但无法建模，损失全部未同意数据
- **Advanced（推荐）**：未同意时发送无 Cookie 匿名 ping，Google 建模补全。合规且恢复大部分数据

**数据恢复效果**：同意率 70% 时，无 CM 可见 70%，有 Advanced CM 可见 ~93%；同意率 50% → ~87%；30% → ~77%

**必需同意类型**：`ad_storage`、`analytics_storage`、`ad_user_data`（v2 新增）、`ad_personalization`（v2 新增）

**实施**：集成 CMP（Cookiebot/OneTrust/Usercentrics）→ 与 GTM 对接 → EEA 默认 denied，其他 granted → Tag Assistant 验证

**影响**：无 CM 则 Smart Bidding 在欧洲数据不足；未实施 CM 无法用 Enhanced Conversions；无法再营销。

## 7. 利润导向出价（Profit-Based Bidding）

### 核心：喂利润而非收入

收入 ≠ 利润。$1000 收入 5% 利润 = $50；$200 收入 50% 利润 = $100。用收入出价，Smart Bidding 选错方向。

**电商**：转化值 = 售价 × 利润率。进阶含退货率：售价 × 利润率 × (1-退货率)

| 产品 | 售价 | 毛利率 | 利润值 | 收入值 |
|------|------|-------|-------|-------|
| 高端配件 | $500 | 60% | $300 | $500 |
| 电子产品 | $800 | 15% | $120 | $800 |

用收入值 → 算法追电子产品；用利润值 → 算法追高端配件。

**线索获取**：转化值 = 成交概率 × 成交额 × 利润率。例：8% × $20,000 × 40% = $640

### Conversion Value Rules

UI 内按条件调整值，无需改代码：地理（加州 +20%）、受众（回头客 +50%）、设备（Desktop +15%）

### 出价策略选择

| 场景 | 策略 |
|------|------|
| 有明确利润目标 | tROAS + 利润值 |
| 利润率波动大 | Max Conversion Value + 利润值 |
| 起步数据不足 | Max Conversions → 过渡 tROAS |

### PMax High Value Mode（2025）

标记高价值客户特征，PMax 优先寻找类似高价值用户。结合利润值：上传利润值 → 启用 High Value Mode → 算法识别高利润模式。

## 8. 信号质量诊断

### 常见红旗

| 问题 | 发现方式 | 严重度 |
|------|---------|--------|
| 同一目标多个 Primary | `conversion-actions` 多个 Primary | 高 |
| 转化价值 = 0 | `conversion-actions` value 字段 | 高 |
| 页面浏览做主要转化 | `conversion-actions` type 字段 | 高 |
| 转化窗口过短（B2B 用 7 天） | `conversion-actions` window 字段 | 中 |
| 未启用 EC | `conversion-actions` EC 状态 | 中 |
| 线索用 Every 计数 | `conversion-actions` 计数方式 | 中 |

### 诊断 CLI 流程

```bash
# 1. 转化动作配置
siluzan-tso google-analysis conversion-actions -a <CID>
# → 检查：Primary 数量、value、window、EC 状态

# 2. 广告系列转化表现
siluzan-tso google-analysis campaigns -a <CID>
# → 检查：转化量合理性、异常转化率、系列间差异

# 3. 关键词验证
siluzan-tso google-analysis keywords -a <CID>
# → 检查：高消费零转化词、品牌/非品牌差异
```

### 信号质量评分卡

| 检查项 | 是/否 | 分值 |
|-------|------|------|
| 主要转化对齐核心业务目标 | +20 / 0 | 核心 |
| 每目标类型仅 1 个 Primary | +15 / -10 | 核心 |
| 转化价值设置且合理 | +15 / -5 | 高 |
| 窗口匹配业务周期 | +10 / -5 | 中 |
| EC 已启用 | +10 / 0 | 中 |
| 计数方式正确 | +10 / -5 | 中 |
| 离线转化已导入 | +10 / 0 | 高 |
| 利润值而非收入 | +10 / 0 | 进阶 |

评分：80+ 优秀 / 60-79 良好 / 40-59 需改善 / <40 严重问题

## 9. 转化信号迁移路径

每次重大变更触发 Smart Bidding ~2 周重学习。分阶段执行，每次只改一个变量。

### Phase 1：修复基础（第 1-2 周）
- 修正 Primary/Secondary（每目标类型 1 个 Primary）
- 修正计数方式（线索 One，购买 Every）
- 调整窗口匹配业务周期
- 启用 EC for Web
- CLI 验证：`siluzan-tso google-analysis conversion-actions -a <CID>`
- 预期：CPA 波动 20-30%，1-2 周恢复

### Phase 2：添加离线信号（第 3-6 周）
- 配置离线导入（GCLID 或 EC for Leads）
- 设为次要转化（不参与出价）
- 定期上传（24-48h），监控匹配率 >50%
- 观察 2-4 周验证数据质量

### Phase 3：切换主要转化（第 7-10 周）
- 前提：新转化 ≥30/月（推荐 ≥50）、匹配率 >50%、已稳定上传 4 周+
- 深层转化升为 Primary，原 Primary 降为 Secondary
- 重设 tCPA/tROAS 目标值
- 预留学习期预算缓冲（+10-20%）

### Phase 4：利润导向（第 11-14 周）
- 前提：Phase 3 稳定 4 周+，有可靠利润数据
- 切换值为利润或启用 Value Rules
- 重评 tROAS 目标

**风险管理**：不同时执行多阶段、保留回滚方案、避开旺季、提前沟通波动预期。

## 10. 按行业推荐配置

### 电商
主要转化: Purchase（动态利润值） | 次要: Add to Cart, Checkout | 计数: Every | 窗口: 30天(高客单60天) | EC: Web | 出价: tROAS

### SaaS
主要转化: Paid>SQL>MQL>Trial（按数据量） | 次要: 上游事件 | 计数: One | 窗口: 90天 | EC: Leads | 值: 阶段加权 | 出价: tCPA(<50/月) / tROAS(>50/月)

### 法律
主要转化: 电话≥120s 或 CRM合格咨询 | 次要: 表单, 短电话 | 计数: One | 窗口: 30天 | EC: Leads | 值: 按案件类型(人伤$5k, 家庭法$1.5k) | 出价: tCPA

### 金融/保险
主要转化: Qualified Application | 次要: 表单, 报价请求 | 计数: One | 窗口: 60天 | EC: Leads | 值: 按产品利润 | 出价: tROAS

### 房地产
主要转化: 看房预约/CRM合格线索 | 次要: 咨询, 电话 | 计数: One | 窗口: 90天 | EC: Leads | 值: 按房产类型 | 出价: tCPA

### 本地服务
主要转化: 电话≥60s + 在线预约 | 次要: 导航, 短电话 | 计数: One | 窗口: 14-30天 | EC: Web | 值: 固定(如$150) | 出价: Max Conv / tCPA

### 教育
主要转化: 报名/试听完成 | 次要: 咨询, 资料下载 | 计数: One | 窗口: 30-60天 | EC: Leads | 值: 按课程价格 | 出价: tCPA / tROAS

## 11. CLI 操作流程

### 11.1 诊断转化设置

```bash
siluzan-tso google-analysis conversion-actions -a <CID>  # 转化动作配置
siluzan-tso google-analysis overview -a <CID>             # 账户转化概览
siluzan-tso google-analysis campaigns -a <CID>            # 系列转化数据
```

分析清单：
1. Primary 转化数量 → 每目标类型仅 1 个？
2. Value → >0 且合理？
3. Window → 匹配业务周期？
4. 各系列转化率 → 存在异常差异？
5. 高消费零转化系列 → 追踪缺失？

### 11.2 评估转化延迟

```bash
siluzan-tso google-analysis conversion-actions -a <CID>
siluzan-tso google-analysis campaigns -a <CID>
```

分析：近 7 天 vs 30 天转化比例是否正常（<7/30 说明延迟）；离线上传时间是否 <48h；延迟 >14 天需添加中间阶段。

### 11.3 信号质量排查

```bash
siluzan-tso google-analysis conversion-actions -a <CID>
siluzan-tso google-analysis campaigns -a <CID>
siluzan-tso google-analysis keywords -a <CID>
```

| 症状 | 可能原因 | 诊断 |
|------|---------|------|
| CPA 远超目标 | Primary 过多/重复 | conversion-actions 查 Primary 数 |
| 转化量骤降 | 追踪故障 | campaigns 近7天 vs 前7天 |
| 转化率 >20% | 信号过宽（页面浏览） | conversion-actions 查 type |
| ROAS 高但利润低 | 用收入非利润 | conversion-actions 查 value |
| 品牌/非品牌转化率无差异 | 浅层转化 | keywords 对比 |
