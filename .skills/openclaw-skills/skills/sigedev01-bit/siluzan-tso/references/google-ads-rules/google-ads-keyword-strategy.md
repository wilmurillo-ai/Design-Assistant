## 一、2025-2026 关键词范式转变

**核心认知**：关键词从「精确匹配目标」变成「意图信号」。

| 旧范式（2020 前） | 新范式（2025-2026） |
|---|---|
| 关键词 = 广告触发器 | 关键词 = 意图信号，Google Gemini 推断用户真实意图 |
| 越多精确匹配越好 | 广泛匹配 + Smart Bidding + 受众信号 = 推荐组合 |
| 管理 30,000 个关键词 | 管理信号质量（转化数据、受众、创意、落地页） |
| 按匹配类型拆分广告系列 | 合并广告系列，让算法获得足够数据 |
| 长尾精确匹配逐一添加 | 长尾由广泛匹配自动捕获，高转化词再提升为精确匹配 |

> **Ginny Marvin（Google Ads Liaison, 2026.02）**：「用关键词告诉 Google 你想匹配哪类查询，而不是试图精确命中每个搜索词。」

**对 Agent 的影响**：生成关键词时不必追求覆盖所有长尾变体，重点选好 10-20 个代表性意图词，配合正确匹配类型和否定词。

## 二、匹配类型高级策略

### 2.1 当前匹配行为

| 类型 | 实际行为（2025-2026） | 适用场景 |
|---|---|---|
| **Broad Match** | Gemini 驱动，综合转化信号、受众、落地页内容匹配用户意图 | 品类探索、有充足转化数据的账户 |
| **Phrase Match** | 匹配相同含义/意图，词序不再重要（已吸收 BMM） | 商业意图词、平衡覆盖与精准 |
| **Exact Match** | 匹配相同含义，含近似变体和改述 | 最高价值已验证转化词 |

### 2.2 推荐组合

| 账户阶段 | 推荐策略 | 前提条件 |
|---|---|---|
| **新账户 / <30 转化/月** | Phrase + Exact | 数据不足，广泛匹配浪费预算 |
| **成熟账户 / ≥30 转化/月** | Broad + Smart Bidding（tCPA/tROAS） | 干净的转化追踪 + 充足否定词 + 受众信号 |
| **品牌词** | Exact + Phrase | 隔离品牌流量，保护品牌 SERP |
| **竞品词** | Phrase + Exact | 避免广泛匹配触发商标问题 |

**关键数据**：Broad + Smart Bidding 对比 Exact-only → CPC 低 34%，展示量高 67%，转化率高 23%（机构实测 $23M+ 花费）。

### 2.3 已过时做法

- ❌ 按匹配类型拆分广告系列（分散数据，降低 Smart Bidding 效率）
- ❌ 同一关键词三种匹配类型各建一个广告组（保留最宽松的即可）
- ❌ 依赖 BMM（已不存在，等同 Phrase Match）

## 三、关键词分组结构

### 3.1 三种模型对比

| 因素 | SKAG（单词广告组） | STAG（单主题广告组） | SIAG（单意图广告组） |
|---|---|---|---|
| 每组关键词数 | 1 | 3-20 | 5-15 |
| Smart Bidding 兼容性 | 差 | 优 | 优 |
| 数据积累速度 | 慢 | 快 | 快 |
| 管理复杂度 | 极高 | 低 | 中低 |
| 2026 推荐度 | 基本淘汰 | **默认推荐** | 进阶推荐 |

**SIAG 示例**：「buy protein powder」和「order whey protein」共享购买意图 → 同组；「protein powder reviews」是调研意图 → 分组。

### 3.2 推荐广告系列架构

| 系列类型 | 匹配类型 | 预算占比 | 说明 |
|---|---|---|---|
| 品牌系列 | Exact | 10-15% | 隔离品牌词，防止信号污染 |
| 品类系列 | Broad + 受众信号 | 40-50% | STAG/SIAG 结构，核心转化来源 |
| 竞品系列 | Phrase + Exact | 10-15% | 独立预算，预期 CPC 较高 |
| 探索系列 | Broad / DSA / AI Max | 15-20% | 发现新机会 |
| 再营销/RLSA | 关键词 + 受众叠加 | 10-15% | 受众信号为主 |

### 3.3 合并原则

- 每个广告系列 5-20 个广告组，每组有明确主题
- 落地页相同 + 创意相似 + 转化目标一致的广告组 → 可合并
- 每个广告组目标：每周 ≥1,000 次展示（Smart Bidding 有效学习的最低数据量）

## 四、否定关键词高级技巧

### 4.1 限制（2025 更新）

| 层级 | 上限 | 说明 |
|---|---|---|
| 账户级否定词 | 1,000 | 全局排除 |
| 共享否定词列表 | 每列表 5,000（最多 20 个列表） | 按主题分组 |
| 广告系列级 | 10,000 | PMax 从 100 扩展到 10,000（2025.03） |
| 广告组级 | 不限（实际建议 <500） | 防止组间重叠 |

**重要更新**：2024.06 起否定关键词自动屏蔽拼写错误变体。

### 4.2 N-Gram 分析法

不逐条添加坏查询，而是分析搜索词中的**高频无效单词/双词组合**，用一条广泛匹配否定词屏蔽数千变体。

| 传统做法 | N-Gram 做法 |
|---|---|
| 否定 "blue running shoes size 10 free shipping" | 否定 "free"（广泛匹配否定词） |
| 需 100+ 条否定词 | 1 条否定词 = 同等效果 |

> ⚠️ 广泛匹配否定词 "free" 会同时屏蔽 "hassle free"、"free shipping"。使用前确认不会误伤。

### 4.3 分层否定词框架

| 层级 | 内容 | 示例 |
|---|---|---|
| 账户级（全局） | 永远无关的类别 | 成人内容、完全无关行业 |
| 共享列表（按主题） | 竞品品牌、免费/廉价词、求职词、B2C 排除词 | 每主题一个列表 |
| 广告系列级 | 系列语境相关 | 高意图系列排除信息类词 |
| 广告组级 | 防组间重叠 | 确保查询路由到最相关组 |

### 4.4 维护节奏

| 频率 | 操作 |
|---|---|
| **每周** | 审查搜索词报告，10+ 点击 0 转化 → 加否定 |
| **每月** | 检查否定词是否误屏有价值查询 |
| **每季度** | 全面清理过时否定词 |

## 五、长尾关键词现代用法

### 5.1 策略转变

| 旧做法 | 新做法 |
|---|---|
| 为每个长尾词建精确匹配 | 用长尾词作为意图信号，广泛匹配自动捕获变体 |
| 添加数百个低搜索量关键词 | 聚焦 3-5 个主题词，让算法扩展覆盖 |

很多长尾词现在会被标记为「低搜索量」而无法直接投放。

### 5.2 实操流程

1. 将长尾关键词添加到广告组 → 即使显示低搜索量，也能引导匹配方向
2. Broad Match 自动匹配长尾查询 → 从搜索词报告发现高转化长尾
3. 高转化长尾 → 提升为 Exact Match，配专属广告文案和落地页
4. 语音搜索查询平均 9 词，本质是长尾 → 广告文案适配口语化表达

### 5.3 行业长尾模式

| 行业 | 长尾关键词模式 |
|---|---|
| 电商 | 产品属性组合：「organic cotton baby blanket hypoallergenic」 |
| 本地服务 | 地点 + 服务 + 修饰：「emergency plumber north sydney 24 hour」 |
| B2B | 问题/方案：「how to reduce employee turnover SaaS company」 |
| SaaS | 功能 + 对比：「project management tool with gantt chart free trial」 |

## 六、Quality Score 优化

### 6.1 三要素与关键词联动

| QS 要素 | 关键词层面优化 | 目标 |
|---|---|---|
| **预期 CTR** | RSA 中 ≥2 条 headline 含主关键词；用 DKI `{KeyWord:默认}`；低 CTR 词移除 | CTR 高于同行平均 |
| **广告相关性** | STAG 分组确保组内词共享意图；广告文案直接回应关键词需求；Pin 含关键词的 headline 到位置 1 | 相关性评分「高于平均」 |
| **落地页体验** | 落地页兑现广告承诺；加载 <3s；移动适配；内容匹配查询意图 | 体验评分「高于平均」 |

**目标**：所有活跃关键词 QS ≥ 7。QS < 6 且花费大的关键词优先优化。

### 6.2 结构性提升

- 品牌词和非品牌词分系列（QS 差异大，混合会拖累非品牌）
- 高客单和低客单产品不共用 ROAS 目标（混合导致算法困惑）
- 将低效 SKAG 合并为 STAG，整体 QS 反而上升（数据密度提升）

## 七、出价策略与关键词意图

### 7.1 意图分层出价

| 意图层级 | 关键词模式 | 出价策略 | 优先级 |
|---|---|---|---|
| **交易意图（BOFU）** | buy, order, hire, near me, pricing, quote | tROAS / tCPA | 最高 |
| **商业调研（MOFU）** | best, reviews, compare, vs, top | tCPA / Max Conversions + CPA cap | 中高 |
| **信息意图（TOFU）** | how to, what is, guide, tips | Max Clicks + CPC cap / 不投 Search | 低 |

### 7.2 Smart Bidding 选择

| 策略 | 最低数据要求 | 适用场景 |
|---|---|---|
| **Target ROAS** | ≥30 转化/月 | 成熟电商 |
| **Target CPA** | ≥15 转化/月 | 线索生成、增长期 |
| **Max Conversions** | 无硬性门槛 | 数据积累期，2-4 周后切 tCPA |
| **Manual / ECPC** | — | 新账户无转化数据时的过渡方案 |

### 7.3 价值出价（Value-Based Bidding）

不只计数转化，分配不同价值：
- 电商：每笔交易收入
- 线索生成：表单 = $10，合格线索 = $100，成交 = $5,000
- 从 CRM 导入离线转化数据 → 真正的利润优化

## 八、搜索词挖掘与精炼

### 8.1 每周挖掘流程

```
1. 导出搜索词报告（所有活跃系列）
2. 按花费降序 → 找高花费零转化词 → 加否定
3. 找高转化词不在关键词列表中 → 提升为 Exact Match + 专属广告组
4. N-Gram 分析 → 发现无效词根模式 → 批量否定
5. 跨系列查重 → 同一查询触发多个系列 → 用否定词路由到最佳系列
```

### 8.2 查询雕刻（Query Sculpting）

通过否定关键词 **主动引导** 查询流向最佳广告组：

| 场景 | 操作 |
|---|---|
| 通用系列匹配到「emergency plumber」 | 在通用系列加否定 "emergency" → 确保由紧急系列承接 |
| 品牌系列匹配到通用查询 | 在品牌系列加否定通用词 → 保持品牌系列纯净 |

### 8.3 搜索词可见性变化

- Google 因隐私持续减少单条搜索词展示，很多长尾归入「其他搜索词」
- 使用 **Search Term Insights**（主题聚类视图）替代逐条分析
- AI Max / Broad Match 产生更多独特查询，但报告中可见比例更低

## 九、竞品关键词策略

### 9.1 竞品征服（Conquesting）

| 要素 | 建议 |
|---|---|
| 合法性 | 多数地区合法，但**禁止在广告文案中使用竞品商标** |
| 匹配类型 | Phrase / Exact（Broad 触发面太广） |
| 预期 | CPC 高、QS 低（你的广告对品牌搜索相关性天然较低） |
| 广告文案 | 强调差异化：「Switch and Save 30%」、「The #1 Rated Alternative」 |
| 落地页 | 专用对比页，展示你的优势 |
| 预算 | 独立系列，占搜索预算 10-15% |

### 9.2 品牌防御

- 始终运行品牌广告系列保护品牌 SERP
- Auction Insights 监控是否有竞品出价你的品牌词
- 发现竞品 → 提高品牌词出价 + 完善附加信息 + 考虑商标投诉

### 9.3 竞品情报工具

| 工具 | 用途 |
|---|---|
| **Auction Insights** | 看谁和你竞争相同拍卖，按关键词细分 |
| **SEMrush / SpyFu / Ahrefs** | 反查竞品付费关键词、广告文案、落地页 |
| **Google Alerts** | 监控竞品产品发布和促销 |

## 十、季节性与趋势关键词

### 10.1 季节性调整工具

Google Ads 内置 **Seasonality Adjustments**（工具 > 出价策略 > 高级控制）：
- 为短期可预测事件（黑五、双十一、新品发布）设置预期转化率变化
- Smart Bidding 自动处理渐进式季节性趋势，**无需手动调整**
- 仅用于突变型事件，不用于长期趋势

### 10.2 预算管理

- **Campaign Total Budgets**（2025 扩展至 Search/Shopping）：设固定推广窗口总预算，AI 按实时需求分配日花费
- 在峰值前 2-3 周提升预算，给 Smart Bidding 学习时间
- 创建自动规则在特定日期调整预算

### 10.3 行业季节性速查

| 行业 | 关键时间节点 |
|---|---|
| 电商 | 黑五/网一、双十一、情人节、母亲节、返校季 |
| B2B | 预算规划季（Q4/Q1）、行业会议、财年末 |
| 本地服务 | 天气依赖（HVAC/管道）、春季家装、报税季 |
| 旅游 | 暑期（春季预订）、节假日出行（提前 2-3 月预订） |

## 十一、关键词自蚕食防治

### 11.1 四种蚕食类型

| 类型 | 表现 | 后果 |
|---|---|---|
| 关键词重叠 | 相同词在多个广告组/系列竞价 | 数据碎片化、CPC 虚高 |
| 地域重叠 | 「纽约市」系列 + 「美国」系列覆盖同一地区 | 自竞争 |
| PPC vs SEO | 付费广告抢自己的自然排名流量 | 浪费预算 |
| 跨系列蚕食 | PMax + Search + DSA 争同一查询 | 最优系列拿不到流量 |

### 11.2 防治策略

| 策略 | 操作 |
|---|---|
| 否定词交叉映射 | 低相关系列加否定 → 路由到最佳系列 |
| 匹配类型合并 | 同一关键词仅保留最宽松匹配类型 |
| 品牌/非品牌隔离 | 品牌词独立系列 + 非品牌系列加品牌否定 |
| 地域排除 | 本地系列覆盖区域从全国系列排除 |
| PPC/SEO 协调 | 自然排名前 3 的词，测试暂停付费广告的增量贡献 |

### 11.3 审计频率

- **每月**：Google Ads Editor 运行「查找重复关键词」
- **每季度**：全账户结构审计

## 十二、PMax / AI Max 与关键词协同

### 12.1 PMax 与 Search 优先级

| 场景 | 胜出者 |
|---|---|
| Search 有 Exact/Phrase 匹配词 | **Search 优先** |
| 查询无精确匹配 | Ad Rank 最高者胜出 |
| PMax 有更好的 Ad Rank | PMax 胜出 |

**策略**：用 Search + 关键词控制最重要的查询；用 PMax 捕获跨网络增量。

### 12.2 AI Max for Search（2025 推出，2026 扩展）

| 功能 | 说明 |
|---|---|
| Search Term Matching | AI 基于落地页内容和广告素材扩展查询覆盖 |
| Text Customization | AI 动态生成 headline/description |
| Final URL Expansion | AI 可将用户导向不同落地页（如预测效果更好） |
| 精细控制 | 广告组级品牌设置、位置设置、URL 包含/排除 |

> **关键影响**：落地页内容变成定向标准。页面标题要像广告文案一样清晰传达主题（Marvin）。

### 12.3 信号层级

| 信号类别 | 权重 | 示例 |
|---|---|---|
| 主信号（真相） | 最高 | 离线转化导入、价值出价目标 |
| 次信号（上下文） | 高 | 第一方客户列表、广告创意环境 |
| 三级信号（意图） | 中 | 关键词（含低量长尾）、落地页信号 |
| 噪音信号（污染） | 负面 | 滚动/点击等弱转化、过时客户列表 |

## 十三、第一方数据 × 关键词

### 13.1 Customer Match 要求

| 要素 | 建议 |
|---|---|
| 最小列表 | 1,000 条 hashed emails |
| 增加匹配率 | 附加电话号码（+15% 匹配率） |
| 按行为分段 | 新客 vs 复购、高 LTV vs 低 LTV |
| 更新频率 | 每 14-28 天（过期列表训练算法找错误人群） |

### 13.2 受众 × 关键词乘数效应

Broad Match + 受众信号 = 伪装成广泛定向的精准投放。

**实测数据**：Broad + 客户信号 → ROAS 2.8x vs Broad 无信号 → ROAS 1.4x（提升 100%）。

### 13.3 受众叠加优先级

1. **Customer Match** → 实际客户数据，按 LTV 分层
2. **网站行为受众** → 3+ 页浏览、90s+ 停留、购物车放弃者
3. **Custom Segments** → 用竞品 URL + 高意图关键词构建
4. **In-market Audiences** → Google 预建的活跃调研人群

**操作**：先以「观察」模式叠加 → 查看各受众表现 → 高效受众切换为「定向」模式。

## 十五、信号污染与算法漂移

### 15.1 识别污染源

| 污染源 | 症状 |
|---|---|
| 垃圾线索做主转化目标 | CPA 低但线索质量差，销售团队反馈无效 |
| 弱转化事件（滚动、页面浏览） | 转化量虚高但收入不涨 |
| 过时客户列表（>18 个月未更新） | 算法寻找已过时的用户画像 |
| 高低客单混用同一 ROAS 目标 | 算法偏向低价高量产品 |
| 品牌和非品牌不隔离 | 品牌词高转化率虚抬非品牌预期 |

### 15.2 纠正协议

1. 移除弱/误导转化目标
2. 上传新鲜客户列表
3. 拆分系列隔离意图层级
4. 刷新广告创意
5. 等待 5-10 天稳定期，不做额外调整

## 十六、Keyword Planner 输出解读与使用技巧

### 16.1 命令用法与参数

CLI 命令格式：

```
siluzan-tso keyword -k "搜索词" [--url <URL>] [--include <包含词>] [--exclude <排除词>] [--json]
```

| 参数 | 说明 |
|---|---|
| `-k` | 核心搜索词（**必填**），Keyword Planner 以此为起点生成建议 |
| `--url` | 配合网站 URL 获取更相关的建议；Google 会分析页面内容，返回与该网站更匹配的关键词 |
| `--include` | 结果必须包含的词（逗号分隔），用于缩小建议范围 |
| `--exclude` | 排除包含这些词的建议（逗号分隔），过滤无关结果 |
| `--json` | 输出 JSON 格式，便于程序化处理或进一步分析 |

**组合用法建议**：先用 `-k` 拉大探索范围，拿到初步结果后，再用 `--include` / `--exclude` 精筛，逐步收窄到高相关关键词集合。

### 16.2 输出字段解读

| 字段 | 含义 | 注意事项 |
|---|---|---|
| **搜索量（Monthly Searches）** | Google 给出的月均搜索量，通常是**范围值**（如 1K-10K）而非精确数字 | 低搜索量词（<100）谨慎投放，可能数据不足以支撑 Smart Bidding；注意季节性波动，单月数据不代表全年趋势 |
| **竞争度（Competition）** | Low / Medium / High，反映**广告主之间**的竞争激烈程度 | 这是广告竞争指标，**不是 SEO 竞争**；Low 竞争不代表容易排名，只代表出价的广告主少 |
| **建议出价（Suggested Bid / Top-of-page Bid）** | 分为 Low range 和 High range；Low range 是页面顶部出价的较低端估算，High range 是较高端估算 | 用于预估预算：日预算 ≈ High range × 预期日点击量；实际 CPC 可能偏离建议值 |
| **关键词相关性** | 建议词与输入搜索词之间的语义距离 | 语义距离越近越相关；距离远的词可能代表新的市场机会，也可能是噪音 |

### 16.3 数据解读技巧

| 数据信号 | 解读 |
|---|---|
| 搜索量 = 0 | 不一定没人搜索。长尾词的搜索量可能低于 Google 报告阈值，仍然有投放价值 |
| 建议出价差异大（Low range 与 High range 差距悬殊） | 竞争激烈且 CPC 波动大，出价需要更大弹性空间 |
| Competition Low + 搜索量中等 | **可能的机会词**——有需求但广告主覆盖不足，值得优先测试 |
| Competition High + 建议出价高 | 验证该词有商业价值（别人愿意花钱买），但需要充足预算支撑才能持续投放 |

### 16.4 实战工作流

```
Step 1: 宽泛探索
  keyword -k "CRM software"
  → 拿到大范围建议列表，了解市场关键词全貌

Step 2: 细分筛选
  keyword -k "CRM" --include "small business" --exclude "free,open source"
  → 聚焦目标客群，排除低商业价值词

Step 3: 竞品参考
  keyword -k "CRM" --url "https://competitor.com"
  → 获取与竞品网站相关的关键词建议，发现对手覆盖的词

Step 4: 整理为关键词列表
  → 按意图分层：BOFU（交易意图）/ MOFU（商业调研）/ TOFU（信息意图）
  → 分配匹配类型：高价值已验证词 → Exact；商业意图词 → Phrase；探索词 → Broad
  → 导出 JSON（--json）供批量导入或进一步加工
```

### 16.5 与其他章节的协同

| 拿到 Planner 数据后 | 参考章节 | 操作 |
|---|---|---|
| 关键词分组 | 第三章（STAG/SIAG 分组） | 将建议词按主题/意图聚类为广告组 |
| 意图分层与出价 | 第七章（出价策略与关键词意图） | BOFU 词高出价、MOFU 中等、TOFU 低出价或不投 Search |
| 否定词预判 | 第四章（否定关键词框架） | 从 Planner 结果中识别明显无关词，提前加入否定词列表 |
