# 成本优化建议

## 模型选择策略
- 简单问答、分类、摘要 → 优先用 claude-3-haiku 或 gpt-3.5-turbo，成本降低 90%+
- 复杂推理、代码生成 → 用 claude-3-sonnet 或 gpt-4o，平衡质量与成本
- 仅在必要时使用 claude-3-opus 或 gpt-4（最贵，适合高精度任务）

## 减少 API 调用
- 批量处理：将多个小任务合并为一次调用
- 缓存常用结果：相同输入 24 小时内复用结果
- 避免重复处理：任务开始前检查是否已有结果

## 控制 Token 消耗
- 精简 system prompt：去掉冗余说明，保留核心指令
- 限制输出长度：设置 max_tokens 避免超长回复
- 使用结构化输出（JSON mode）减少废话

## 存储优化
- 定期清理 /tmp 下的临时文件（超过 7 天）
- 压缩历史日志（超过 30 天的归档）
- 删除重复的中间文件

## 调度优化
- 非紧急批量任务安排在低峰时段（凌晨 2-6 点）
- 避免多个 agent 同时运行大规模任务
- 设置任务超时，防止死循环

## 异常预防
- 为每个 agent 设置单次任务成本上限
- 循环任务必须有终止条件
- 测试环境使用更便宜的模型
