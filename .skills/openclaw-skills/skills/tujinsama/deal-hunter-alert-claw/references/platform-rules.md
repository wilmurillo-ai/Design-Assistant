## 采集策略

### 请求频率控制
- 同一平台请求间隔：≥ 3秒（避免触发反爬）
- 高频平台（闲鱼/转转）：每5分钟一轮，每轮间隔随机 3-8 秒
- 低频平台（政府采购/法拍）：每15-30分钟一轮

### 关键字段提取规则

**房产类**
```
标题、价格（总价/单价）、面积、地址/区域、楼层、建成年份、
联系方式、发布时间、房源ID、图片链接
```

**二手商品类**
```
标题、价格、成色/品相、卖家信誉分、发布时间、
商品ID、图片链接、地区、是否支持验货
```

**招标类**
```
项目名称、预算金额、采购单位、发布时间、截止时间、
项目类型、地区、联系方式、公告链接
```

### 去重策略
- 使用 `平台名称 + 商品/房源ID` 作为唯一键
- 已推送过的标的，状态变化（降价、即将截止）时再次推送

## 反爬应对

- User-Agent 轮换：维护10+个常见浏览器UA
- 请求头模拟：携带 Referer、Accept-Language 等正常浏览器头
- 遇到 429/503：指数退避，等待 30s/60s/120s 后重试
- 部分平台需要 Selenium 模拟浏览器（闲鱼、链家）

## 数据存储

监控结果存储在 `~/.openclaw/workspace/deal-hunter-data/` 目录：
- `seen_ids.json`：已处理的标的ID集合（去重用）
- `alerts_log.jsonl`：所有推送记录（JSONL格式，每行一条）
- `rules.json`：当前生效的监控规则
