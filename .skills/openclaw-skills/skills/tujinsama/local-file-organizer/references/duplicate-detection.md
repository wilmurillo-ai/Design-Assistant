# 重复文件检测算法

## 检测策略

### 1. 精确匹配（哈希比对）
- MD5：快速，适合大批量扫描
- SHA256：更安全，适合重要文件
- 流程：先按文件大小分组，再对同大小文件计算哈希

### 2. 相似度检测（图片）
- 感知哈希（pHash）：检测视觉相似图片
- 阈值：汉明距离 ≤ 10 视为相似
- 依赖：pillow 库

### 3. 快速模式（--quick）
- 仅比对文件大小 + 文件名
- 速度快，误报率较高
- 适合初步筛查

## 保留策略

当发现重复文件时，按以下优先级保留：
1. 保留最新修改时间的版本
2. 保留文件名最规范的版本（符合命名规范）
3. 保留分辨率最高的版本（图片）
4. 默认保留路径最短的版本

## 处理方式

- `--trash`：移动到系统回收站（推荐，可恢复）
- `--delete`：直接删除（不可恢复，需二次确认）
- `--report`：仅生成报告，不执行操作
