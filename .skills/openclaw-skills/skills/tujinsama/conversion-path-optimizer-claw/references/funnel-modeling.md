# 销售漏斗建模指南

## 标准漏斗阶段

| 阶段 | 英文标识 | 说明 |
|------|----------|------|
| 初次接触 | lead | 首次触达客户 |
| 需求确认 | qualification | 确认客户有真实需求 |
| 方案演示 | demo | 产品演示/方案讲解 |
| 报价谈判 | negotiation | 价格与条款谈判 |
| 合同签订 | closed_won | 成交 |
| 流失 | closed_lost | 未成交 |

不同行业可自定义阶段，但需保持字段一致性。

## 转化率计算

```
阶段转化率 = 进入下一阶段数 / 本阶段总数 × 100%
整体转化率 = closed_won 数 / lead 总数 × 100%
```

## 行业健康转化率基准

| 行业 | lead→qualification | qualification→demo | demo→negotiation | negotiation→closed_won |
|------|-------------------|-------------------|-----------------|----------------------|
| SaaS | 40-60% | 50-70% | 60-80% | 20-40% |
| 电商 | 30-50% | 60-80% | 70-90% | 30-60% |
| B2B | 20-40% | 40-60% | 50-70% | 15-30% |

## 流失预警信号

- 某阶段转化率低于行业基准 20% 以上 → 重点排查
- 单阶段停留时间超过平均值 2 倍 → 客户可能失温
- 同一销售的流失率持续高于团队均值 → 需要话术辅导

## 异常检测规则

1. **骤降型**：某阶段转化率环比下降 >15%，需立即分析原因
2. **长尾型**：大量客户卡在同一阶段超过 30 天，需优化该阶段话术
3. **季节型**：结合历史数据判断是否为正常季节性波动
