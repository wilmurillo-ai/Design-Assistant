# 对账规则库

## 匹配模式

### 精确匹配（默认）

条件：金额完全一致 **且** 日期在容差范围内（默认 ±3 天）

```
match_score = 100
if abs(amount_diff) == 0 and abs(date_diff_days) <= date_tolerance:
    → 精确匹配 ✅
```

### 模糊匹配

条件：金额在误差范围内（如含手续费场景）**且** 摘要关键词相似度 ≥ 0.6

```
if abs(amount_diff) <= amount_tolerance and keyword_similarity >= 0.6:
    → 模糊匹配 ⚠️（需人工确认）
```

默认 `amount_tolerance = 1.0`（元），可通过 `--amount-tolerance` 参数调整。

### 多对一匹配

识别拆分付款、合并付款场景：

- **拆分付款**：一笔订单对应多笔流水，金额之和等于订单金额
- **合并付款**：多笔订单对应一笔流水，流水金额等于订单金额之和

## 容差配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--date-tolerance` | 3 | 日期容差（天），适用于 T+N 到账场景 |
| `--amount-tolerance` | 1.0 | 金额容差（元），适用于含手续费场景 |
| `--keyword-threshold` | 0.6 | 摘要相似度阈值（0~1） |

## 异常分级

| 级别 | 标记 | 定义 | 建议处理 |
|------|------|------|---------|
| 红 🔴 | UNMATCHED | 流水有记录但订单/发票无对应 | 立即核查，可能漏单或错账 |
| 黄 🟡 | AMOUNT_DIFF | 匹配到对应条目但金额不符 | 核查手续费、折扣、退款冲销 |
| 橙 🟠 | DUPLICATE | 同一笔交易出现多次 | 核查是否重复入账 |
| 蓝 🔵 | DATE_OFFSET | 超出容差范围的日期差异 | 核查到账延迟、跨月结算 |

## 特殊场景规则

### 手续费扣除

银行流水金额 = 实际到账金额（已扣手续费），订单金额 = 应收金额。
→ 设置 `--amount-tolerance` 为预期手续费上限。

### 退款冲销

退款流水金额为负数，与原订单正向匹配后标记为"已退款"。

### 分期付款

多笔流水金额之和 = 订单金额，日期跨度在合理范围内（如 30 天内）。

### 跨月结算

月末订单可能在次月到账，建议将 `--date-tolerance` 设为 5~7 天。
