# 脱敏规则

## 原则

- 所有数据处理在本地完成，不调用外部 AI 接口处理原始财务数据
- 脱敏副本用于匹配计算，原始数据不离开本地环境
- 金额字段**不脱敏**（对账核心字段）

## 字段脱敏规则

| 字段类型 | 脱敏方式 | 示例 |
|---------|---------|------|
| 银行账号 | 保留前4后4，中间替换为 `****` | `6222 **** **** 1234` |
| 姓名 | 保留姓氏，名字替换为 `**` | `张**` |
| 手机号 | 保留前3后4，中间替换为 `****` | `138****5678` |
| 身份证号 | 保留前6后4，中间替换为 `********` | `310101********1234` |
| 邮箱 | 用户名保留首字符，其余替换为 `***` | `z***@example.com` |
| 金额 | **不脱敏** | 原始值 |
| 日期 | **不脱敏** | 原始值 |
| 流水号/订单号 | **不脱敏** | 原始值 |

## 实现方式

脱敏在 `scripts/reconcile.py` 的 `desensitize_dataframe()` 函数中实现，在数据加载后、匹配计算前自动执行。

脱敏后的数据仅用于：
- 生成对账报告的展示列
- 写入飞书多维表格的展示字段

匹配计算使用原始金额和日期，不受脱敏影响。
