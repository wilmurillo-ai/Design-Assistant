# 版本管理规则

## 版本号格式

`v主版本号.次版本号.修订号`，例如 `v3.2.1`

| 变更类型 | 递增位置 | 示例 |
|---------|---------|------|
| 重大修订（结构性变化） | 主版本号 | v2.0 → v3.0 |
| 内容新增/删除 | 次版本号 | v3.0 → v3.1 |
| 文字修正/格式调整 | 修订号 | v3.1.0 → v3.1.1 |

## 版本状态

| 状态 | 说明 |
|-----|------|
| 草稿 | 尚未审批，不可引用 |
| 待审核 | 已提交审批，等待批准 |
| 生效中 | 当前有效版本，唯一 |
| 已废止 | 被新版本替代，仅供查阅 |
| 历史版本 | 主动归档的旧版本 |

## 核心规则

1. 同一文件同一时间只能有一个"生效中"版本
2. 新版本归档时，旧版本自动变更为"已废止"
3. 每次版本变更必须记录：变更时间、操作人、主要变更内容
4. 废止版本永久保留，不可删除，仅可查阅

## 变更记录格式

```
版本号: v3.0
生效日期: 2026-04-01
操作人: 张三
变更内容:
  - 新增远程办公政策（第5章第3节）
  - 调整年假计算规则（第8章第2节）
废止版本: v2.0（废止日期: 2026-04-01）
```
