# Learning Rules

## What to Learn

### High-value signals (always learn)
- User explicitly corrects your behavior → immediate rule
- User confirms a non-obvious approach → stable preference
- Repeated pattern across 3+ interactions → stable convention
- User states a project rule ("we always use X for Y") → project convention

### Medium-value signals (learn if repeated)
- prompt-refiner version choice (refined vs original) — only after consistent pattern
- Output format preference (bullet vs prose, short vs detailed)
- Tool usage preference (which tools to use/avoid)

### Low-value signals (usually skip)
- One-off debugging choices
- Temporary workarounds
- Context-specific decisions unlikely to recur

## What NOT to Learn

### Never store
- Full prompt text generated by prompt-refiner
- API keys, passwords, tokens, or any secrets
- Personally identifiable information
- File paths that will change (use patterns instead)
- Function signatures (derivable from code)

### Skip unless explicitly asked
- Obvious programming best practices
- Framework defaults (e.g., "use async/await in Node")
- Things already documented in project README or config

## Judging prompt-refiner Preference Signals

| Pattern | Action |
|---------|--------|
| User picks refined version 5+ times in a row | Store: "user prefers refined prompts" |
| User picks original once after picking refined many times | Don't overfit — likely scenario-specific |
| User says "stop showing me the original" | Immediately store as rule |
| User says "I want to always see both options" | Immediately store as rule |
| User switches between refined and original roughly equally | Don't store — no clear preference yet |

## Merge Strategy Examples

### Adding a new rule
```
Before: (nothing in CLAUDE.md about test strategy)
Signal: User says "always write tests before implementation"
After:  - 测试优先：先写测试再写实现代码
```

### Updating a conflicting rule
```
Before: - 使用 AskUserQuestion 弹窗确认
Signal: User says "don't use popups, just show me in chat"
After:  - 使用纯文本确认，不使用 AskUserQuestion 弹窗
```

### Compressing redundant rules
```
Before:
- 不要记录完整 prompt 文本
- prompt-refiner 产生的具体完整提示词文本不要记录
- 只学习用户偏好规则，不记录完整 prompt 内容

After:
- 只学习偏好规则，不记录完整 prompt 文本
```
