# Security Policy

## Supported Versions

Security updates are applied to:

- The latest commit on `master`
- The most recent published release of this skill

Older snapshots may not receive fixes.

## Reporting a Vulnerability

Do not open public GitHub issues for suspected vulnerabilities.

Preferred reporting path:

1. Use GitHub's private vulnerability reporting for this repository if it is available.
2. If private reporting is not enabled, contact the maintainer privately through the repository owner profile before disclosing details publicly.

Include:

- Affected version or commit
- Clear reproduction steps
- Expected impact
- Any proof-of-concept or logs needed to validate the issue

## Disclosure Expectations

Please allow time for triage and remediation before public disclosure. Reports will be reviewed, validated, and prioritized based on severity and exploitability.
