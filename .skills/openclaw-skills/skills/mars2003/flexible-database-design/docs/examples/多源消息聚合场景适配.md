# 多源消息聚合场景适配

> **应用案例**：聚合美国国债收益率、美元外汇等多源金融时序数据。  
> 验证场景：多 source、联合视图、按日期对齐。

---

## 一、场景特征

| 维度 | 说明 |
|------|------|
| **多源** | 至少 2 个 source（如 treasury_yield、forex） |
| **消息流** | 每条记录 = 日期 + 指标 + 数值 |
| **软字段** | data_type、items[]、trend（可选） |

---

## 二、实现方式

本案例采用**场景适配**：结构化时间序列、字段固定，使用单表 + 联合视图，而非 records + dynamic_data。

| 层级 | 实现 |
|------|------|
| **原始层** | records（每源 1 条批次元信息） |
| **业务层** | treasury_yield_daily、forex_daily 固定表 |
| **视图层** | v_daily_combined（按日期 JOIN 两源） |

---

## 三、验证结果

- 国债：约 1.1 万行
- 外汇：约 1.6 万行
- v_daily_combined 正确按日期内连接

---

## 四、与外汇数据库的对比

| 维度 | 外汇数据库 | 多源消息聚合 |
|------|------------|--------------|
| 来源数 | 单源 | 多源 |
| 联合分析 | 无 | 有（国债+外汇按日期对齐） |
