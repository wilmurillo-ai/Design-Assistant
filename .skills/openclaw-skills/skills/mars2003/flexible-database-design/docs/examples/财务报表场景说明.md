# 财务报表收集场景说明

> **应用案例**：基于 Flexible Database Skill 构建的财务报表数据库。  
> 本 Skill 保持通用定位，本文档为财报场景的扩展说明。

---

## 一、货币与单位约定（P1）

### 1.1 统一存储单位

**约定**：金额类字段（revenue、net_income、total_assets 等）**统一存「元」**。

| 原始单位 | 归档时换算 | 示例 |
|----------|------------|------|
| 万元 | × 10000 | 100 万元 → 1000000 |
| 亿元 | × 100000000 | 1.5 亿元 → 150000000 |
| 千元 | × 1000 | 500 千元 → 500000 |

**实现**：在抽取器或归档前预处理中做单位换算；查询/展示时再按需格式化（如除以 1e8 显示为「亿元」）。

## 二、字段映射（已内置）

core 已支持财报场景的字段映射，**无需改 core 代码**：

| extracted 字段 | 映射到 | 说明 |
|----------------|--------|------|
| `report_type` | category | 报表类型（年报/季报/中报）作为分类 |
| `report_date` | data_date | 报告期作为数据日期 |

示例：

```json
{"company":"某公司","report_type":"年报","report_date":"2024-12-31","revenue":1000000000,"net_income":80000000}
```

## 三、派生指标与视图（P2）

ROA、ROE、净利率等可建视图，减少重复计算。

### 3.1 v_financial_ratios 示例

```sql
-- 派生指标视图（需 dynamic_data 中有 revenue、net_income、total_assets 等）
CREATE VIEW IF NOT EXISTS v_financial_ratios AS
SELECT
    r.record_id,
    r.source,
    r.created_at,
    MAX(CASE WHEN d.field_name = 'company' THEN d.field_value END) AS company,
    MAX(CASE WHEN d.field_name = 'report_type' THEN d.field_value END) AS report_type,
    MAX(CASE WHEN d.field_name = 'report_date' THEN d.field_value END) AS report_date,
    MAX(CASE WHEN d.field_name = 'revenue' THEN d.numeric_value END) AS revenue,
    MAX(CASE WHEN d.field_name = 'net_income' THEN d.numeric_value END) AS net_income,
    MAX(CASE WHEN d.field_name = 'total_assets' THEN d.numeric_value END) AS total_assets,
    CASE WHEN MAX(CASE WHEN d.field_name = 'total_assets' THEN d.numeric_value END) > 0
         THEN 100.0 * MAX(CASE WHEN d.field_name = 'net_income' THEN d.numeric_value END)
              / MAX(CASE WHEN d.field_name = 'total_assets' THEN d.numeric_value END)
         ELSE NULL END AS roa_pct,
    CASE WHEN MAX(CASE WHEN d.field_name = 'revenue' THEN d.numeric_value END) > 0
         THEN 100.0 * MAX(CASE WHEN d.field_name = 'net_income' THEN d.numeric_value END)
              / MAX(CASE WHEN d.field_name = 'revenue' THEN d.numeric_value END)
         ELSE NULL END AS net_margin_pct
FROM records r
LEFT JOIN dynamic_data d ON r.record_id = d.record_id
WHERE r.is_deleted = 0
GROUP BY r.record_id, r.source, r.created_at;
```

## 四、文件归档模式（P2）

### 4.1 从文件读取（已支持）

长文本、PDF 提取后的全文，可用 `-F` 避免命令行长度限制：

```bash
python3 scripts/archive_item.py -F extracted_report.txt -s "pdf" -e '{"company":"A","report_type":"年报","report_date":"2024-12-31"}'
```

### 4.2 PDF 归档扩展模式

PDF 需先提取文本再归档。可扩展 `archive_pdf.py` 模式：

1. 用 PyMuPDF / pdfplumber 等提取 PDF 文本；
2. 可选：用 LLM 抽取结构化字段；
3. 调用 `archive_item(content=..., extracted=...)` 或 `archive_item.py -F temp.txt -e '...'`。

示例骨架：

```python
# archive_pdf.py 示例（项目内扩展，非 Skill 内置）
import fitz  # PyMuPDF
from flexible_db import FlexibleDatabase

def archive_pdf(pdf_path: str, source: str = "pdf", extracted: dict = None):
    doc = fitz.open(pdf_path)
    text = "\n".join(page.get_text() for page in doc)
    doc.close()
    db = FlexibleDatabase()
    success, rid = db.archive_item(content=text, source=source, extracted_data=extracted)
    db.close()
    return success, rid
```

## 五、单文件多记录策略（P2）

### 5.1 取舍

| 策略 | 适用 | 优点 | 缺点 |
|------|------|------|------|
| **单条多科目** | 一张 PDF 含多张表（资产负债表+利润表+现金流量表） | 一条 record 对应一个文件，关联简单 | 查询需按 field_name 过滤 |
| **多条按报表类型** | 每张表单独一条 record | 按 report_type 查询清晰，便于建视图 | 需在抽取阶段拆分 |

### 5.2 推荐做法

- **单 PDF 多报表**：推荐**多条按报表类型**。抽取时按表拆分，每条 record 的 `report_type` 为「资产负债表」「利润表」「现金流量表」等，`raw_content` 存该表原文。
- **单 PDF 单报表**：单条即可，`report_type` 标识类型。

## 六、query_items 按视图查询（P2 可选）

当前 `query_items` 按 `category`、`field_name` 查询。若需按扁平视图（如 v_financial_ratios）查询，可：

1. **项目内扩展**：写 `query_view.py`，对指定视图执行 `SELECT ... WHERE company=? AND report_date=?`；
2. **或** 使用 `query_items --field company --value "某公司"` 配合 `--field report_date` 多条件（当前 CLI 暂不支持多字段 AND，可调用 `flexible_db.query_dynamic()` 或直接 SQL）。
