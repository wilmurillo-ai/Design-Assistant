# 外汇数据库场景适配

> **应用案例**：基于 Flexible Database Skill 框架构建的外汇日度数据库。  
> 本案例展示**结构化时间序列**场景下如何按需裁剪、而非照搬三层模型。

---

## 一、Skill 与场景的适配关系

| 维度 | flexible-database-design Skill | 外汇数据库实际实现 |
|------|-------------------------------|-------------------|
| **目标场景** | 非结构化、多源异构（知识库、政策、表单等） | 结构化时间序列（外汇日度收盘价） |
| **数据特征** | 字段易变、来源多样、需全文检索 | 字段固定、单一来源、数值查询为主 |
| **三层模型** | 原始层 → 软字段层(JSON/键值对) → 业务视图层 | 未采用，改为单表 + 元数据表 |

外汇数据是结构化金融时间序列，与 Skill 主打的「非结构化、多源异构」不同，因此做了**场景适配**，而不是照搬三层模型。

---

## 二、从 Skill 继承的部分

| 继承项 | 说明 |
|--------|------|
| **目录结构** | `references/` + `scripts/` + `data/` 与 Skill 推荐一致 |
| **技术选型** | SQLite，便于本地部署和查询 |
| **PRAGMA 设置** | `foreign_keys = ON`、`encoding = 'UTF-8'` |
| **元数据表** | `forex_meta` 类似 Skill 中的来源、版本等元信息 |
| **文档规范** | README 含数据源、目录结构、使用方法、表结构 |
| **作者/日期** | 按用户规则标注作者和日期 |

---

## 三、针对外汇场景的定制

| 定制项 | 实现方式 |
|--------|----------|
| **Schema** | 单表 `forex_daily`：date 主键 + 15 个数值列（美元指数 + 14 个货币对） |
| **导入脚本** | `import_forex_csv.py`：解析 CSV、处理千分位、`INSERT OR REPLACE` |
| **索引** | 按日期、美元指数、USD/JPY 建索引，支持时间序列和常用分析 |
| **无 FTS** | 无全文检索需求，未启用 FTS |
| **无软字段** | 字段固定，无需 JSON 或键值对表 |

---

## 四、交付物清单

```
外汇数据库/
├── README.md                    # 项目说明、用法、表结构
├── data/
│   └── forex.db                 # SQLite 数据库（约 1.6 万条日度记录）
├── references/
│   └── schema_forex.sql         # 建表脚本
└── scripts/
    └── import_forex_csv.py      # CSV 导入脚本
```

---

## 五、结论与启示

| 启示 | 说明 |
|------|------|
| **Skill 作为框架** | 提供目录结构、技术选型和文档规范，外汇项目沿用了这些约定 |
| **按场景裁剪** | 外汇数据结构化、字段稳定，采用单表 + 元数据表即可，无需三层灵活模型 |
| **可复用模式** | 未来若有「多源外汇数据」「不同货币对组合」等需求，可再考虑引入软字段或视图层 |
| **验证结果** | 数据已成功导入，可支持美元走势分析等查询，满足当前使用需求 |

整体上，Skill 提供了**方法论和项目骨架**，外汇数据库在此基础上做了合理的**场景适配**，实现了简洁、可用的 SQLite 外汇数据存储与查询。
