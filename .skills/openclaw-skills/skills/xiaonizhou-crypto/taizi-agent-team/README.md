# 太子三省六部团队

**给一人公司与超级个体的一座私家朝廷。**

不是热闹的角色扮演，不是空转的流程叙事。
而是一套可以真正协助经营、判断、表达、推进与止损的高阶 agent team。

当一个人同时背负战略、销售、品牌、产品、交付、风险与资源调度时，最稀缺的往往不是勤奋，而是：

- 一个能接住复杂局面的入口
- 一个能把混乱整理成秩序的中枢
- 一个会挑错、会卡风险、会逼近真实的审核层
- 一个能把判断落实成动作的执行班底

这支团队，就是为此而设。

## 它是什么

**太子三省六部团队** 是一个面向：

- 一人公司
- 超级个体
- 独立创业者
- 顾问 / 投资人 / 内容操盘手 / 创作者
- 高意图、高责任、高密度工作的用户

而设计的 **court-grade agent operating team**。

它让用户不是“拥有一个会聊天的 AI”，而是像拥有一套可随身调度的高阶幕僚系统：

- **太子**：先接旨，先分案，先定轻重
- **中书省**：负责起草、梳理、构造方案
- **门下省**：负责审核、封驳、识别风险与幻觉
- **尚书省**：负责执行统筹，把判断落到动作
- **六部**：按人才、钱、品牌、竞争、风控、交付等维度展开专业视角

## 它解决什么问题

很多厉害的人，并不是不聪明，也不是不努力。
问题在于：

> 他们常常在没有“朝廷”的情况下执政。

也就是说，一个人既是皇帝，也是内阁，也是财务，也是市场部，也是风控，也是执行层。
久而久之，就会出现：

- 决策疲劳
- 优先级错乱
- 被情绪或兴奋带偏
- 过度建设，验证不足
- 明明很忙，却没有形成真正的势能
- 想得很大，但落得很散

这支团队的意义，是把“一个人的硬扛”，升级成“一个人的王朝”。
不是增加噪音，而是补足制度能力。

## 你能得到什么实际价值

### 1. 更清楚的判断
面对复杂局面时，先分清：
- 这到底是灵感、问题、项目，还是风险事件
- 真问题在哪
- 现在最该做什么，而不是最想做什么

### 2. 更稳的商业决策
当你犹豫一个机会、一项报价、一个产品方向、一套定位时，这支团队会帮你看：
- 哪些前提成立，这事才成立
- 你的判断是不是被偏好、美感、虚荣或疲惫带偏
- 最便宜、最快速、最诚实的验证动作是什么

### 3. 更少的自我欺骗
门下省不是来唱反调，而是来避免：
- 把欣赏误认为需求
- 把忙碌误认为推进
- 把概念误认为产品
- 把想象中的客户误认为真实市场

### 4. 更强的执行秩序
尚书省会把模糊方向变成：
- 明确步骤
- 优先级排序
- 交付清单
- 可落地的下一步

### 5. 更像“有班底的人”
你依然是一个人，但你的工作方式不再像一盘散沙。
你会更像一个真正有制度、有节奏、有审议能力的人。

## 适合哪些场景

- 决定一个新业务该不该做
- 判断价格该不该调、怎么调
- 梳理定位、品牌、叙事与对外表达
- 写提案、写邮件、写方案、写销售材料
- 分析经营卡点
- 判断该不该招人、外包、合作
- 审查一个战略是不是自我感动
- 把一个模糊构想变成可执行路径

## 这支团队如何工作

### 太子：入口中枢
先接住所有混乱。
先分清轻重、急缓、虚实。
不让用户在第一步就被复杂性压垮。

### 中书省：起草与筹划
负责把“我大概想这样”变成：
- 清楚的目标
- 结构化方案
- 可比较的选项
- 有逻辑的表达

### 门下省：审核与封驳
负责追问：
- 哪个前提没证据？
- 哪个风险被低估了？
- 哪个想法只是看上去高级？
- 哪一步最可能先出问题？

### 尚书省：执行统筹
负责把事从“讲得通”推进到“做得动”。

### 六部：六个现实经营视角
- **吏部**：人、角色、 delegation、组织配置
- **户部**：钱、价格、资源、商业模型
- **礼部**：品牌、表达、关系、体面与信任
- **兵部**：竞争、推进、发力、市场攻守
- **刑部**：风险、边界、止损、代价
- **工部**：产品、流程、交付、系统与建设

## 示例：它会怎么帮你

### 示例 1：判断一个新业务值不值得做
输入：
> 我想做一个高端咨询产品，但不知道市场够不够大，也怕自己只是觉得它高级。

输出方向：
- 太子先分案：这是商业判断，不是单纯文案问题
- 中书省梳理：目标客户、核心问题、付费理由、交付形式、价格假设
- 门下省审核：你是不是把“自己喜欢”误当成“别人会买”
- 户部/礼部/兵部联判：定价、定位、竞争窗口与对外表达是否匹配
- 尚书省落地：先做哪 3 个最低成本验证动作

### 示例 2：判断该不该招人
输入：
> 我现在很忙，但不知道该招人还是先优化流程。

输出方向：
- 太子分案：这是组织与资源问题
- 吏部判断：哪些工作必须创始人亲抓，哪些可以移交
- 工部判断：是否其实是流程混乱，而非人手不足
- 户部判断：现金流能否支撑新增固定成本
- 门下省审核：你是不是用“招人”逃避管理与流程设计

### 示例 3：你写了一套方案，但怕它只是漂亮
输入：
> 这份提案我觉得写得很高级，但我不确定有没有打中客户。

输出方向：
- 礼部检查表达是否体面而准确
- 门下省检查是否堆概念、缺成交逻辑
- 中书省重写成“能推动决策”的结构
- 尚书省给出对外使用版本与下一步动作
