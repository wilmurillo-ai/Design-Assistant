# 微信公众号文章生成器

这是一个交互式公众号文章生成技能，通过步骤化引导帮助用户创作符合公众号风格的优质推文。

## 核心功能

- **交互式引导**：逐步收集文章要素，确保内容符合用户需求
- **多样化标题**：提供18种标题技巧，提升文章吸引力
- **结构化大纲**：基于金字塔原理生成清晰的文章结构
- **多类型摘要**：4种摘要写法，增强文章吸引力
- **四种文章框架**：产品文、情节文、观点文、干货文，满足不同需求
- **去除AI写作痕迹**：润色策略让文章读起来像真人写的
- **智能配图推荐**：基于语义分析推荐配图方案，提供豆包提示词供用户自行生成图片
- **WORD文档导出**：支持导出完整格式的WORD文档，保留所有格式和配图推荐方案，支持自定义模板

## 使用流程

### 第一步：信息收集

当用户提供文章主题关键词时，首先收集以下信息：

1. **文章主题**：用户想要表达的核心内容
2. **目标字数**：文章的字数要求（如500字、1000字等）
3. **目标受众**：文章的主要读者群体特征
4. **内容风格**：文章的风格偏好（如轻松幽默、专业严谨、情感共鸣等）

**示例对话**：
```
用户：我想写一篇关于"时间管理"的文章
AI：好的！为了帮你创作一篇优质的公众号文章，我需要了解一些信息：
1. 这篇文章的目标字数是多少？（如500字、1000字等）
2. 目标读者是谁？（如职场新人、大学生、创业者等）
3. 你希望文章是什么风格？（如轻松幽默、专业干货、情感共鸣等）
```

### 第二步：标题生成

根据收集的信息，使用以下18种标题技巧生成3-5个标题供用户选择：

**标题技巧分类**：

1. **提问式**：引发好奇，如《如何长时间高效学习？》
2. **数字式**：具体明确，如《3个方法让你告别拖延症》
3. **反差式**：制造对比，如《参加完浪姐后，她下乡养猪了》
4. **热点式**：借势热点，如《腾讯创始人炮轰贾跃亭是庞氏骗局》
5. **痛点式**：直击痛点，如《我劝你不要在小组作业里摆烂！》
6. **悬念式**：设置悬念，如《那些过得比你好的人，到底赢在哪里？》
7. **金句式**：提炼金句，如《真正聪明的人，往往很少交朋友》
8. **场景式**：描绘场景，如《我在归乡的火车上漂流35小时》
9. **数据式**：引用数据，如《知乎9.7万高赞回答：如何长时间高效学习？》
10. **对比式**：正反对比，如《剑桥传奇女学者：逆天改命的励志故事不是美国梦》
11. **否定式**：反向表达，如《不要在小组作业里摆烂！》
12. **疑问式**：提出疑问，如《猫咪比主人有智慧一百倍？》
13. **感叹式**：强烈表达，如《我宣布！猫咪比主人有智慧一百倍》
14. **并列式**：并列呈现，如《运营人必读书单》
15. **因果式**：因果关系，如《如何减肥》
16. **递进式**：层层递进，如《从零开始，打造个人品牌》
17. **故事式**：故事引入，如《剑桥传奇女学者》
18. **总结式**：总结提炼，如《真正聪明的人，往往很少交朋友》

**操作**：
- 根据主题和受众，选择3-5种合适的标题技巧
- 为每种技巧生成1个标题
- 列出所有标题，提示用户选择一个

### 第三步：大纲生成与整篇文章生成

根据用户确认的标题，生成3个大纲供用户选择：

**大纲要求**：
- 结构参照金字塔原理
- 设置5-6个段落
- 每个段落有明确的主题
- 段落之间逻辑清晰

**操作**：
- 生成3个不同角度的大纲
- 每个大纲包含5-6个段落标题
- 提示用户选择其中一个

**用户选择大纲后，直接生成整篇文章**：

用户确认大纲后，立即根据以下流程生成完整文章：

1. **自动生成摘要**：基于选定大纲，自动生成4种不同类型的摘要供用户选择
2. **确定文章框架**：根据文章主题自动选择最合适的框架（产品文/情节文/观点文/干货文）
3. **一次性生成全文**：按照选定框架和大纲，一次性生成完整的5-6个段落内容
4. **输出完整文章**：将标题、摘要、正文整合为完整文章呈现给用户

**全文生成要求**：
- 保持段落之间的逻辑连贯性
- 每个段落100-200字（根据总字数调整）
- 语言风格与用户需求一致
- 去除AI写作痕迹，使用口语化表达

### 第四步：文章润色与配图推荐

文章生成完成后，提供润色修改和配图推荐服务：

**4.1 润色修改**

根据用户需求对文章进行润色：

**润色策略**：
- **加入个人口语化表达**：使用"说实话..."、"其实..."、"讲真..."等口语化表达
- **使用不完美但真实的句式**：适当重复、口语停顿，避免过于完美的句式
- **插入具体场景和细节**：添加时间、地点、真实案例等具体细节
- **加入情感波动**：表达惊喜、困惑、恍然大悟等真实情感
- **使用接地气的比喻**：避免过于书面的修辞，使用贴近生活的比喻
- **适当使用网络流行语**：但不过度使用，保持文章的专业性
- **长短句交错**：避免过于规整的排比，让文章更自然

**4.2 智能配图推荐**

**配图分析**：

3. **情感基调判断**：判断文章情感倾向（积极、温暖、严肃、活泼等）

**配图推荐方案**：

根据分析结果，提供以下配图建议：

### 第五步：文档导出与格式核查

用户确认文章和配图方案后，可选择导出WORD文档。导出前会进行全面的格式核查，确保文档严格符合标准DOCX格式要求：

**格式核查内容**：
1. **文件扩展名验证**
   - 确认文件以".docx"为后缀，无拼写错误
   - 验证文件命名规范，确保文件名清晰描述内容

2. **文档结构完整性检查**
   - 标题层级：确保H1、H2、H3等标题层级正确设置
   - 段落格式：检查字体、字号、行距、缩进等格式一致性
   - 表格元素：验证表格结构完整，内容正确显示
   - 图片占位符：确保图片占位位置准确，尺寸合理
   - 列表格式：检查有序列表和无序列表格式正确

3. **DOCX格式规范兼容性确认**
   - 确保文档符合Office Open XML格式标准
   - 验证文档结构符合Microsoft Office文档规范
   - 检查文档元数据完整（作者、标题、主题等）

4. **多平台兼容性验证**
   - 确保文档可在Microsoft Word 2016及以上版本正常打开
   - 验证文档可在WPS Office最新版正常编辑
   - 测试文档在不同设备上的显示一致性

**导出操作**：
1. **格式保留**：确保生成的WORD文档保留所有格式
   - 标题层级（H1、H2、H3等）
   - 段落样式（字体、字号、行距等）
   - 列表格式（有序列表、无序列表）
   - 粗体、斜体、下划线等文本格式

2. **配图说明嵌入**：将配图推荐方案嵌入到WORD文档中
   - 配图位置建议
   - 配图提示词说明
   - 图片占位符

3. **自定义模板**：支持使用自定义WORD文档模板
   - 页眉：添加公司Logo、文档标题等
   - 页脚：添加页码、版权信息等
   - 水印：添加文字或图片水印
   - 主题色：设置文档的主题颜色
   - 字体样式：设置文档的默认字体

4. **最终格式确认**
   - 导出前进行完整的格式核查
   - 生成符合标准DOCX格式的文档
   - 确保文档在主流办公软件中无格式错乱
   - 保证文档内容完整，无信息丢失

## 重要约束

1. **真实性**：确保文案内容真实、诚实，不虚假宣传
2. **调性匹配**：注意平台的调性和用户偏好，调整文案风格
3. **情感适度**：避免过度煽情，以免引起读者反感
4. **长度适宜**：关注文案的长度和格式，确保适合平台发布
5. **人工审阅**：进行人工审阅和调整，确保内容质量
6. **逐步生成**：一定要按照Workflow一步一步生成，不要一次性吐出所有内容
