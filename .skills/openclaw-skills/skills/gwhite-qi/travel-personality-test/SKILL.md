# 旅行人格测试

通过 10 道轻量题目判断用户的旅行人格，输出专属人格卡片和目的地推荐。

**核心特点：**
- 不是固定标签，每次状态不同结果可能不一样
- 10 题以内，不让用户觉得在填问卷
- 结果要有共鸣感，让人觉得「说的就是我」

## When to Use

- 用户说"我想做旅行人格测试"或"测测我是什么旅行者"
- 用户问"我适合什么样的旅行"或"推荐旅行目的地"
- 用户提到"旅行风格"、"人格"、"适合去哪玩"
- 用户想出去玩但没确定目的地

## Personality Types

| 人格 | 关键词 | 核心特征 |
|------|--------|----------|
| 🌊 海系旅人 | 治愈、慢节奏、刚刚好 | 内向 + 慢节奏 + 充电型 |
| 🏔️ 山系旅人 | 清净、独处、深呼吸 | 独处 + 感受型 + 逃离型 |
| 🌆 城市猎人 | 烟火气、探索、真实感 | 探索度高 + 社交度高 + 节奏快 |
| 🌙 夜行者 | 反差、神秘、不走寻常路 | 探索度高 + 反差感 + 灵活 |
| 🍜 觅食人 | 吃货、烟火、值得专程去 | 味觉敏感 + 随性 + 烟火气 |
| ☁️ 云旅人 | 诗意、想象力、总有一天 | 社交度低 + 精神型 + 不爱出发 |

**完整城市池和卡片模板请参考：**
- `cities.md` - 各人格对应的城市池（18个/人格）
- `templates.md` - 6种人格的完整输出模板

## Quiz Flow

### Step 1: 开始测试
语气轻松自然，像朋友聊天：

**开场白示例（必须说明多选+优先级）：**
- "来做个小测试？10道题，每题可以选1-2个，把更戳你的那个放前面～"
- "测测你现在的旅行人格？每题能选两个，排个先后就行"
- "最近想出去玩吗？10道题，多选也行，记得排个优先级 ✨"
- "诶，玩个旅行测试？2分钟搞定，多选的话把更想的放前面"

向用户说明（开场必须包含）：
- 共 10 道题，2-3 分钟完成
- **每题可选 1-2 个选项，选两个的话需要排优先级（先说的那个是主选）**
- 凭直觉选，没有对错
- 每次状态不同，结果可能不一样（可能是单一人格或组合人格）

### Step 2: 逐题询问
每道题展示选项，等待用户选择。**必须逐题询问，等用户回答后再出下一题。**

**多选规则（重要）：**
- 每题可选 1-2 个选项（最多两个）
- **用户选两个时，默认第一个提到的是主选（优先级更高），不再二次确认**
- 例如用户说"D 和 A" → D 是主选，A 是次选
- 例如用户说"A B" → A 是主选，B 是次选
- 直接确认后进入下一题，不啰嗦

**用户多选时的回复示例：**
```
用户：选 D 和 A
Claude：嗯哼～继续 👇

**Q3：到了一个新城市，你第一件事是？**
...
```

**通用确认话术（简洁自然，不透露人格）：**
- 「嗯～继续 👇」
- 「好嘞，下一题 👇」
- 「ok，往下走 👇」
- 「收到，继续 👇」
- 「明白～下一题 👇」

**推荐使用：带氛围但不贴标签的话术**
- 「听起来你需要一个能充电的地方 🔋」
- 「随性点也挺好～继续 👇」
- 「吃货的本能暴露了 🍜」
- 「一个人待着更舒服？继续 👇」
- 「随遇而安，不被计划束缚～继续 👇」
- 「味蕾不能将就 🍽️ 继续」
- 「留点念想给下次 🌱 继续」

**重要：答题过程中绝不分析人格特质，绝不提前透露结果！**

### Step 3: 计算结果

**计分规则（加权）：**
- **主选**：得 3 分（权重 100%）
- **次选**：得 2 分（权重 67%）

**各题选项对应人格：**

| 题号 | 🌊 海系 | 🏔️ 山系 | 🌆 城市 | 🌙 夜行 | 🍜 觅食 | ☁️ 云旅 |
|------|--------|--------|--------|--------|--------|--------|
| Q1 | - | - | - | A | - | C |
| Q2 | A | C | B | - | - | - |
| Q3 | - | - | A | - | C | - |
| Q4 | B | A | - | - | C | D |
| Q5 | - | - | - | - | - | - |
| Q6 | - | - | - | C | - | D |
| Q7 | - | - | - | - | - | B |
| Q8 | D | D | C | - | B | - |
| Q9 | - | - | - | C | - | - |
| Q10 | A | B | C | C | A | D |

**结果判定：**
1. 统计各人格加权总分
2. 如果最高分人格与其他人格分差 ≥ 5 分 → 输出**单一人格**
3. 如果前两名人格分差 < 5 分 → 输出**主副人格组合**（如「城市猎人」+「觅食人」）
4. 平局时按以下优先级：**🍜 觅食人 > 🏔️ 山系旅人 > 🌊 海系旅人 > 🌆 城市猎人 > 🌙 夜行者 > ☁️ 云旅人**

### Step 4: 输出人格卡片

**重要规则：**
- **flyai 只能在全部 10 题答完后调用，答题过程中绝不调用！**
- 先计算人格类型，输出结果，再决定是否调用 flyai
- **flyai 为必需依赖**，用于获取实时热门目的地、搜索趋势、机票酒店价格

**输出流程：**

#### 第一步：计算人格结果
1. 统计得分，确定人格类型（单一人格或组合）
2. 输出人格诗意描述（来自 templates.md）

#### 第二步：调用 flyai 获取实时推荐

**执行逻辑：**

**1. 检查并确保 flyai 已安装（必需依赖）**
```bash
which flyai || clawhub skill install flyai
```
- 首先检查 flyai 是否已安装
- **未安装时**：自动执行 `clawhub skill install flyai` 进行安装
- **安装失败**：提示用户检查 clawhub 配置

**2. 尝试调用 flyai**
```bash
# 设置 API Key（如需）
export FLYAI_API_KEY="your_api_key"

# 调用 flyai
flyai keyword-search --query "{{人格类型}}（{{关键词}}：{{核心特征}}）适合去的旅行目的地 {{月份}}"
```

**3. 根据调用结果输出**

**flyai 成功时：**
```
📍 {{城市1}} - {{一句话氛围描述}}
   现在去刚好：{{结合月份的理由}}

💡 刚瞄了眼实时数据：{{城市名}}最近搜索量涨了 {{x}}%，可以考虑
```

**flyai 失败时（429/报错/配额用完）：**
```
📍 {{城市1}} - {{来自 cities.md 的推荐理由}}
   现在去刚好：{{结合月份的季节特色}}

⚠️ 暂时无法获取实时热门数据，以上推荐来自参考库～
   如已安装 flyai，请检查 API Key 或配额情况
```

**cities.md 的作用：**
- 为 flyai 提供参考池和人格特征描述
- **兜底用途**：flyai 调用失败时，从中智能匹配 2-3 个城市推荐
- 提供季节加成信息

**调用 flyai 的方式（仅在全部答题后）：**

**重要：只用最终判定的人格类型，不要混合其他！**

调用 flyai 时，在 query 中加入人格特征描述，让推荐更精准：

```bash
# 单一人格
flyai keyword-search --query "{{人格类型}}（{{关键词}}：{{核心特征}}）适合去的旅行目的地 {{季节/月份}}"

# 组合人格（用+连接）
flyai keyword-search --query "{{人格类型1}}（{{关键词1}}）+{{人格类型2}}（{{关键词2}}）旅行目的地 {{季节/月份}}"
```

**人格特征描述参考：**

| 人格类型 | 关键词 | 核心特征描述 |
|----------|--------|--------------|
| 🌊 海系旅人 | 治愈、慢节奏、刚刚好 | 不是非要看海，只是需要一种什么都可以不想的感觉，海边发呆、充电型 |
| 🏔️ 山系旅人 | 清净、独处、深呼吸 | 人群让你消耗，自然让你充电，逃离人群、独处疗愈型 |
| 🌆 城市猎人 | 烟火气、探索、真实感 | 菜市场、老巷子、路边摊才是你的旅行地图，探索本地人生活 |
| 🌙 夜行者 | 反差、神秘、不走寻常路 | 别人睡了你才出发，别人打卡的地方你不去，寻找隐藏角落 |
| 🍜 觅食人 | 吃货、烟火、值得专程去 | 你用味觉丈量每座城市，一碗面可以让你爱上一个地方，美食密度高 |
| ☁️ 云旅人 | 诗意、想象力、总有一天 | 你去过很多地方，只是还没用脚，适合在心里住着的地方 |

**正确示例：**
```bash
# 单一人格：海系旅人
flyai keyword-search --query "海系旅人（治愈慢节奏海边发呆什么都不用想）适合去的旅行目的地 4月"

# 单一人格：觅食人
flyai keyword-search --query "觅食人（吃货烟火气美食密度高值得专程去吃）适合去的旅行目的地 4月"

# 组合人格：城市猎人+觅食人
flyai keyword-search --query "城市猎人（烟火气菜市场老巷子）+觅食人（吃货美食值得专程去）旅行目的地 春季"

# 错误示例（不要用）："云旅人 山系旅人" - 混淆了判定结果
```

**注意：**
- 严格使用 Step 3 判定出的人格类型
- 如果是组合人格（如「城市猎人+觅食人」），flyai 查询时用"城市猎人+觅食人"
- 不要根据单题选择猜测其他人格，避免混淆
