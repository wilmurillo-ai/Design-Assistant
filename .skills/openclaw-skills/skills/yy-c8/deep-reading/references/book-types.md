# 书籍类型判断与阅读量配置

## 判断逻辑

Agent 根据书籍特征自动判断类型，设定单次阅读量：

| 书籍特征 | 自动设定 | 判断标准 |
|---------|---------|---------|
| **古典原典** | 1章/次 | 古典哲学、宗教经典等。章节短（≤500字），密度高，需深度解析。 |
| **古典解读/注疏** | 1章/次 | 后人对古典的注疏、解读系列。层层关联，不宜多读。 |
| **现代哲学/经营类** | 2-3章/次 | 现代哲学、管理学、自助成长类著作。可读性强，密度较低。 |
| **学术著作/政治经济学** | 1节/次 | 学术性强的理论著作。逻辑严密，需慢读。 |
| **文学类** | 3-5章/次 | 小说、散文、回忆录等。叙事性强，节奏快。 |

## 判断流程

```
1. 读取书籍第一章
2. 分析章节长度、内容密度、写作风格
3. 匹配上述类型表
4. 写入 reading/plan.md
```

## 用户可调

Agent 自动推断后，**必须**告诉用户：

```
检测到《XXX》属于[类型]，建议每次读[X]章。
- 同意：直接回复"开始精读"
- 调整：告诉我你希望每次读几章
```

## reading/plan.md 格式

```markdown
# 精读计划

## 当前书籍

| 书名 | 作者 | 类型 | 每次阅读量 | 当前进度 | 状态 |
|------|------|------|-----------|---------|------|
| 示例书1 | （作者） | 古典原典 | 1章 | X/X章 | ✅ 完成 |
| 示例书2 | （作者） | 古典原典 | 1章 | X/X章 | ✅ 完成 |
| 示例书3 | （作者） | 古典原典 | 1篇 | X/X篇 | 🔄 进行中 |

## 已完成

| 书名 | 章节 | 用时 | 字数 | 完成日期 |
|------|------|------|------|---------|
| 示例书1 | XX章 | XXX分钟 | XXXXX字 | YYYY-MM-DD |
| 示例书2 | XX章 | XXX分钟 | XXXXX字 | YYYY-MM-DD |
```

## 特殊情况处理

| 情况 | 处理方式 |
|------|---------|
| 书籍不在上述类型中 | 根据章节长度和密度推断，默认 1章/次 |
| 用户强烈要求调整 | 优先响应用户需求，记录调整后的配置 |
| 多本书同时精读 | 每本书独立配置，按用户指定顺序执行 |
