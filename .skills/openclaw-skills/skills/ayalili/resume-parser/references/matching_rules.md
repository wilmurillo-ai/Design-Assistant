# 简历匹配严格规则
## 核心原则
客观公正，宁缺毋滥，严格按照JD要求匹配，禁止主观加分和过度美化。

## 核心要求判定
### 必须判定为核心要求的内容
1. JD中明确标注「必须」「要求」「需具备」「必备」的技能/经验
2. 岗位名称对应的核心能力：
   - AI算法岗：深度学习、机器学习、Python/ C++、模型训练/调优
   - 后端开发岗：编程语言（Java/Go/Python等）、数据库、框架、微服务
   - 前端开发岗：JavaScript/TypeScript、框架（React/Vue等）、CSS/HTML
   - AI应用开发岗：大模型、RAG、Agent、提示词工程、AI工具链
3. 明确的工作年限要求（如3年以上相关经验）
4. 明确的学历要求（如本科及以上计算机相关专业）

### 判定为加分要求的内容
1. JD中标注「优先」「加分项」「最好有」的内容
2. 非核心技能（如熟悉某款工具、了解某领域知识）
3. 软技能要求（如沟通能力、学习能力）

## 扣分规则
1. 核心要求每缺失1项，整体评分扣20分
2. 核心要求部分满足的，按满足程度给分，最多给50%的分
3. 加分项不做强制要求，满足的额外加分，但总分不超过100
4. 如果核心要求缺失超过2项，直接判定为「不匹配」，评分不超过40分

## 评分校准规则
### 避免过度美化的校准项
1. 「了解」「熟悉」某技能但没有实际项目经验的，最多给30%的分
2. 项目经验和JD要求相关但不直接匹配的，最多给50%的分
3. 工作年限刚好达到要求的，不额外加分；超过要求的可适当加5-10分
4. 跨行业经验的，除非明确说明可迁移，否则不额外加分

### 必须明确说明的内容
1. 匹配结果必须明确：完全匹配/基本匹配/勉强匹配/不匹配
2. 必须列出所有未满足的核心要求
3. 必须说明核心差距，不能用模糊表述
4. 建议必须具体可操作，不能空泛
