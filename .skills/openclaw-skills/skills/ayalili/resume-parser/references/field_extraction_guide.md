# 简历字段提取指南

## 个人信息提取规则
1. **姓名**：通常在简历最顶部，2-3个汉字，优先提取第一个出现的人名
2. **电话**：11位数字，格式可能有空格/横杠，提取纯数字
3. **邮箱**：标准邮箱格式，包含@符号
4. **年龄**：从出生年份计算，或直接提取文本中的年龄数字
5. **性别**：直接提取"男"/"女"，或从其他信息推断（可选）
6. **所在地**：城市名，通常在个人信息区域
7. **工作年限**：从工作经历的起止时间计算，或直接提取

## 教育经历提取规则
1. 学校名称优先提取985/211/双一流等标签
2. 学历优先级：博士 > 硕士 > 本科 > 专科 > 高中
3. 专业要完整提取，包括细分方向
4. 起止时间统一格式为"YYYY-MM"
5. GPA如果有就提取，没有留空
6. 相关课程只提取和岗位相关的核心课程

## 工作经历提取规则
1. 公司名称优先提取全称，知名公司可保留简称
2. 职位要准确，不要泛化为"工程师"等
3. 工作内容要提取核心职责，不要冗余描述
4. 业绩成果优先提取有量化数据的内容（如"提升30%"、"服务10万用户"）
5. 技术栈提取明确提到的编程语言、框架、工具
6. 时间如果是"至今"，end_date填"至今"

## 项目经历提取规则
1. 项目名称要准确，不要简化
2. 角色要明确（如"项目负责人"、"核心开发"、"测试工程师"）
3. 项目描述提取核心功能和价值
4. 技术栈提取项目中使用的所有技术
5. 成果优先提取量化指标，没有量化的提取定性成果

## 技能提取规则
1. 技术技能：编程语言、框架、中间件、数据库、工具等
2. 软技能：沟通能力、团队协作、项目管理、学习能力等
3. 语言能力：英语四六级、雅思托福、其他外语能力

## 注意事项
1. 所有提取的信息必须完全来自简历原文，不要编造
2. 信息不确定时留空，不要猜测
3. 重复的信息只保留一次
4. 时间顺序按最新到最旧排序
5. 尽量保留原始表述，不要改写
