## 概述

桌面远程协助是帮助非技术用户安装和配置 OpenClaw 的有效方式。与 SSH 全自动方案不同，桌面远程协助通过图形界面进行，适合：

**适用场景**:
- 对方完全不熟悉命令行操作
- 需要查看对方桌面环境进行诊断
- 需要安装 GUI 相关的依赖（如 Node.js 图形化安装）
- 网络环境限制无法使用 SSH（如公司防火墙）
- 需要实时指导对方操作

**核心优势**:
- 操作直观，对方可以看到你的每一步操作
- 可以实时看到错误信息
- 适合教学式指导
- 无需配置复杂的 SSH 密钥

**与 SSH 方案对比**:

| 特性 | SSH 远程 | 桌面远程协助 |
|------|----------|-------------|
| 自动化程度 | 100% 全自动 | 半自动（需要人工操作） |
| 适合人群 | 有一定技术基础 | 完全技术小白 |
| 配置复杂度 | 需配置 SSH 密钥 | 简单（连接码/ID） |
| 安全性 | 高（密钥认证） | 中等（依赖软件安全性） |
| 操作效率 | 高（批量操作） | 中等（单次操作） |
| 可视化 | 无 | 有（可以看到桌面） |

**最佳实践**:
- 优先使用 SSH 方案（批量、自动化）
- 桌面远程用于单次临时协助
- 结合使用：SSH 用于环境检测，桌面远程用于复杂配置

## 远程软件对比

### 主流远程软件特性

| 软件 | 类型 | 平台支持 | 安装难度 | 免费限制 | 特点 |
|------|------|----------|----------|----------|------|
| **AnyDesk** | 桌面控制 | Win/Mac/Linux | 简单 | 个人免费 | 速度快，无需注册 |
| **TeamViewer** | 桌面控制 | Win/Mac/Linux | 简单 | 非商用免费 | 功能全面，有文件传输 |
| **向日葵** | 桌面控制 | Win/Mac/Linux | 简单 | 有限制 | 国内服务，速度快 |
| **UU远程** | 桌面控制 | Win/Mac | 简单 | 免费限时长 | 游戏用户首选，需UU账号 |
| **微软RDP** | 远程桌面 | Win主机 | 中等 | 免费 | Windows 原生，配置繁琐 |
| **VNC** | 远程桌面 | Win/Mac/Linux | 困难 | 免费 | 需要配置服务端 |

### 选择建议

**快速临时协助**:
- 首选：AnyDesk（无需注册，速度快）
- 备选：TeamViewer（功能更全）
- 国内用户：向日葵（网络稳定）

**固定远程办公**:
- Windows 环境：微软 RDP（原生，性能好）
- 跨平台：VNC（灵活但配置复杂）

**已有游戏平台**:
- UU 游戏加速器用户：UU 远程（无需额外软件）

## 常用远程软件指南

### AnyDesk

**优势**:
- 极速连接，延迟低
- 无需注册即可使用
- 轻量级，安装快速
- 支持文件传输

**使用步骤**:

1. **双方安装 AnyDesk**
   ```
   下载地址: https://anydesk.com/zh/downloads
   选择对应平台版本下载安装
   ```

2. **获取对方连接信息**
   - 让对方打开 AnyDesk
   - 告诉你他们的"此工作台地址"（9位数字）
   - 例如：123 456 789

3. **建立连接**
   - 在你的 AnyDesk 中输入对方地址
   - 点击"连接"
   - 输入对方提供的密码（显示在 AnyDesk 界面）
   - 成功连接后即可操作对方桌面

4. **权限确认**
   - 首次连接对方会收到权限请求提示
   - 让对方点击"允许"即可

**安全建议**:
- 每次会话后让对方更改安全密码
- 不要分享安全密码，仅用于当前会话
- 使用"无人值守访问"需谨慎

**文件传输**:
- 连接后点击 AnyDesk 左侧的"文件传输"按钮
- 可以直接在本地和远程电脑之间传输文件

### TeamViewer

**优势**:
- 功能全面，支持多显示器
- 内置聊天和文件传输
- 支持会议功能
- 有移动端 APP

**使用步骤**:

1. **双方安装 TeamViewer**
   ```
   下载地址: https://www.teamviewer.com/zhcn/download/windows/
   选择免费版本安装
   ```

2. **获取对方 ID 和密码**
   - 让对方打开 TeamViewer
   - 告诉你他们的"您的 ID"（10位数字）
   - 告诉你他们的"密码"（4位数字，每次重启变化）

3. **建立连接**
   - 在你的 TeamViewer 中"伙伴 ID"输入框输入对方 ID
   - 点击"连接到伙伴"
   - 输入对方提供的密码
   - 成功连接

4. **附加功能**
   - **文件传输**: 点击"文件传输"按钮
   - **视频通话**: 点击"演示" → "视频通话"
   - **聊天**: 点击右侧聊天图标
   - **录制会话**: 点击"其他" → "录制会话"

**安全建议**:
- 非商用场景使用免费版本
- 定期更新 TeamViewer 版本
- 启用双重验证（如果有 TeamViewer 账号）

**常见问题**:
- "连接超时": 检查网络，尝试更换端口
- "未授权": 对方未接受连接请求

### 向日葵

**优势**:
- 国内服务，连接速度快
- 支持扫码连接（移动端）
- 远程开机功能
- 远程摄像头功能

**使用步骤**:

1. **双方安装向日葵**
   ```
   下载地址: https://sunlogin.oray.com/download
   选择"个人版"免费版本
   ```

2. **注册账号（可选）**
   - 可以使用手机号注册
   - 或直接使用"本机识别码"连接

3. **获取对方信息**
   - 让对方打开向日葵
   - 告诉你他们的"本机识别码"（8位数字）
   - 或让对方登录账号后分享设备

4. **建立连接**
   - **方式一**: 输入对方识别码 → 点击"远程连接" → 输入验证码
   - **方式二**: 扫描对方二维码（移动端）
   - 首次连接需对方确认

5. **附加功能**
   - **文件传输**: 点击工具栏的"文件管理"
   - **远程开机**: 需提前配置支持远程开机的设备
   - **远程摄像头**: 点击"远程摄像头"

**安全建议**:
- 设置强密码
- 定期更换验证码
- 不要分享账号密码

### UU远程

**优势**:
- 如果已安装 UU 游戏加速器，无需额外软件
- 游戏用户熟悉该界面
- 支持加速远程连接

**使用步骤**:

1. **前提条件**
   - 对方已安装 UU 游戏加速器
   - 双方都有 UU 账号

2. **启动 UU 远程**
   - 打开 UU 游戏加速器
   - 点击左侧或顶部的"远程"按钮
   - 或按快捷键（通常在设置中查看）

3. **发起连接**
   - **方式一**: 邀请对方 → 对方点击接受
   - **方式二**: 输入对方的邀请码/设备号

4. **开始远程协助**
   - 连接成功后可以直接操作对方桌面
   - 右侧工具栏提供文件传输、聊天等功能

**注意事项**:
- 需要 UU 会员（部分功能）
- 免费用户有时长限制
- 网络质量依赖 UU 节点

### 微软远程桌面 (RDP)

**优势**:
- Windows 原生功能，无需安装第三方软件
- 性能优秀，资源占用低
- 支持剪贴板共享
- 支持本地驱动器映射

**使用步骤**:

1. **目标主机（被控端）配置**
   ```powershell
   # 以管理员身份运行 PowerShell

   # 启用远程桌面
   Enable-NetFirewallRule -DisplayGroup "Remote Desktop"

   # 允许远程连接
   Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name "fDenyTSConnections" -Value 0

   # 设置远程桌面用户权限
   # 控制面板 → 系统 → 远程桌面 → 选择用户
   ```

2. **获取目标主机信息**
   - IP 地址：`ipconfig` 查看或让对方告知
   - 用户名：Windows 登录用户名
   - 密码：Windows 登录密码

3. **发起连接（控制端）**
   - Windows: 按 `Win + R` → 输入 `mstsc`
   - macOS: 下载 Microsoft Remote Desktop 应用
   - Linux: 使用 `rdesktop` 或 `Remmina`

4. **配置连接**
   - 计算机：输入目标 IP 或主机名
   - 用户名：输入目标用户名
   - 点击"连接"
   - 输入密码

5. **高级选项**
   - **本地资源**: 勾选"剪贴板"、"驱动器"（可共享本地文件）
   - **显示**: 调整分辨率和颜色质量
   - **体验**: 根据网络速度选择

**常见问题**:
- "无法连接": 检查防火墙和远程桌面服务状态
- "需要 Network Level Authentication": 在"高级"中关闭此选项
- 蓝屏/黑屏: 调整显示设置，降低分辨率

### VNC

**优势**:
- 跨平台支持（Win/Mac/Linux）
- 开源免费
- 可配置性强
- 支持多个虚拟桌面

**使用步骤**:

1. **安装 VNC Server（目标主机）**
   ```bash
   # Linux (Ubuntu/Debian)
   sudo apt-get update
   sudo apt-get install -y x11vnc tightvncserver

   # macOS
   brew install tiger-vnc

   # Windows
   # 下载 TightVNC 或 UltraVNC
   ```

2. **启动 VNC Server**
   ```bash
   # 设置 VNC 密码
   vncpasswd

   # 启动 VNC Server
   # Linux
   x11vnc -display :0 -forever -shared -rfbauth ~/.vnc/passwd

   # 或使用 vncserver
   vncserver :1

   # macOS
   /System/Library/CoreServices/RemoteManagement/ARDAgent.app/Contents/Resources/kickstart -activate -configure -access -on -restart -agent -privs -all
   ```

3. **安装 VNC Viewer（控制端）**
   - 下载 RealVNC Viewer 或 TigerVNC Viewer
   - 跨平台通用

4. **连接到目标主机**
   - 输入目标 IP 和端口（如 192.168.1.100:5901）
   - 输入 VNC 密码

**安全建议**:
- VNC 传输未加密，建议配合 SSH 隧道使用
- ```bash
    # SSH 隧道示例
    ssh -L 5901:localhost:5901 user@remote-host
    # 然后连接 localhost:5901
    ```
- 设置强密码
- 限制 IP 访问（防火墙）

## OpenClaw 安装步骤清单

### 通用准备检查清单

在开始安装前，先确认以下内容：

**环境检查**:
- [ ] 操作系统版本已知（Windows 10/11, macOS 12+, Ubuntu 20.04+ 等）
- [ ] 已知管理员权限状态
- [ ] 已知网络环境（是否需要代理/VPN）
- [ ] 已知磁盘剩余空间（至少 2GB）
- [ ] 已知内存大小（至少 4GB，推荐 8GB+）

**软件检查**:
- [ ] 浏览器已安装（Chrome/Firefox/Edge）
- [ ] 终端/命令提示符可打开
- [ ] 已关闭杀毒软件临时禁用（可选，防止拦截）
- [ ] 已准备好 OpenClaw 相关文档

### Windows 安装清单

1. **安装 Node.js** (如未安装)
   - [ ] 打开浏览器，访问 https://nodejs.org
   - [ ] 下载 LTS 版本（推荐 v22+）
   - [ ] 运行安装程序，一路"下一步"
   - [ ] 打开 PowerShell/CMD，输入 `node --version` 验证
   - [ ] 输入 `npm --version` 验证

2. **配置 npm 镜像源** (如在国内)
   - [ ] 打开 PowerShell（管理员）
   - [ ] 执行: `npm config set registry https://registry.npmmirror.com`
   - [ ] 执行: `npm config get registry` 验证

3. **安装 Git** (如未安装)
   - [ ] 访问 https://git-scm.com/download/win
   - [ ] 下载并安装 Git
   - [ ] 打开 PowerShell，输入 `git --version` 验证

4. **安装 OpenClaw**
   - [ ] 打开 PowerShell（管理员）
   - [ ] 执行: `npm install -g openclaw@latest`
   - [ ] 等待安装完成（可能需要 2-5 分钟）
   - [ ] 输入 `openclaw --version` 验证安装

5. **初始化配置**
   - [ ] 执行: `openclaw onboard`
   - [ ] 按照提示完成配置
   - [ ] 设置 AI 模型（Claude/GPT）
   - [ ] 设置通讯频道（微信/钉钉/Slack 等）

6. **启动服务**
   - [ ] 执行: `openclaw gateway start`
   - [ ] 打开浏览器访问 http://127.0.0.1:18789
   - [ ] 确认 Web UI 正常显示

7. **测试功能**
   - [ ] 在 Web UI 发送测试消息
   - [ ] 确认 AI 响应正常
   - [ ] 检查服务状态: `openclaw gateway status`

## 故障排查

### 远程连接问题

**问题 1: 无法连接到对方电脑**

**可能原因**:
- 对方未打开远程软件
- 对方网络断开
- 防火墙阻止连接
- 端口被占用

**解决方案**:
```
1. 确认对方已打开远程软件
2. 确认对方网络正常（让同学ping一个网站）
3. 让对方关闭防火墙或添加远程软件到白名单
4. 尝试更换连接方式（如 TeamViewer 转 AnyDesk）
5. 尝试使用手机热点（排查公司网络限制）
```

**问题 2: 连接后黑屏或卡顿**

**可能原因**:
- 网络带宽不足
- 对方电脑性能差
- 显示分辨率过高

**解决方案**:
```
1. 降低远程软件的显示质量/分辨率
2. 关闭不必要的动画效果
3. 让对方关闭其他占用资源的程序
4. 尝试更换网络环境
```

**问题 3: 对方收不到连接请求**

**可能原因**:
- 远程软件通知被屏蔽
- 对方电脑静音
- 连接请求被拦截

**解决方案**:
```
1. 让对方检查远程软件的通知设置
2. 提前电话或视频告知对方
3. 使用其他联系方式确认（微信/QQ）
4. 让对方检查防火墙设置
```

### 安装问题

**问题 1: Node.js 安装失败**

**症状**: 安装程序报错或安装后无法验证

**解决方案**:
```
1. 检查操作系统版本是否支持
2. 禁用杀毒软件临时安装
3. 使用管理员权限运行安装程序
4. 尝试下载不同版本的 Node.js
5. Windows: 检查是否有残留文件，清理后重试
```

**问题 2: npm install 失败**

**症状**: 安装 OpenClaw 时报错

**常见错误**:
- `EACCES`: 权限不足
- `ETIMEDOUT`: 网络超时
- `ENOSPC`: 磁盘空间不足
- `MODULE_NOT_FOUND`: 依赖缺失

**解决方案**:
```
1. 权限不足:
   - Windows: 以管理员身份运行 PowerShell
   - macOS/Linux: 使用 sudo

2. 网络超时:
   - 配置国内镜像源: npm config set registry https://registry.npmmirror.com
   - 清理缓存: npm cache clean --force

3. 磁盘空间不足:
   - 清理临时文件
   - 卸载不必要的软件

4. 依赖缺失:
   - 安装构建工具（Linux: build-essential）
   - 安装 Python（某些依赖需要）
```

**问题 3: OpenClaw 启动失败**

**症状**: `openclaw gateway start` 报错

**解决方案**:
```
1. 检查 Node.js 版本: node --version (需要 v22+)
2. 检查端口占用: netstat -ano | findstr :18789 (Windows)
3. 检查配置文件: ~/.openclaw/config.json
4. 查看日志: ~/.openclaw/logs/gateway.err.log
5. 尝试重新安装: npm install -g openclaw@latest
```

### 配置问题

**问题 1: 无法访问 Web UI**

**症状**: 浏览器打开 http://127.0.0.1:18789 无响应

**解决方案**:
```
1. 确认服务已启动: openclaw gateway status
2. 检查端口是否被占用
3. 尝试使用 localhost 而非 127.0.0.1
4. 检查防火墙是否阻止
5. 重启服务: openclaw gateway restart
```

**问题 2: AI 无法响应**

**症状**: 发送消息后 AI 无响应

**解决方案**:
```
1. 检查 AI 模型配置
2. 检查 API Key 是否有效
3. 检查网络连接
4. 查看日志文件: ~/.openclaw/logs/gateway.out.log
5. 检查账号余额（如使用付费 API）
```

### 安全问题

**问题 1: 怀疑被恶意远程**

**症状**: 发现不明远程会话或文件被修改

**解决方案**:
```
1. 立即断开网络
2. 修改所有密码（系统、邮箱、银行等）
3. 检查远程软件的"最近连接"记录
4. 运行杀毒软件全面扫描
5. 联系技术支持
6. 如涉及敏感信息，考虑报警
```

**问题 2: 安装过程中被杀毒软件拦截**

**症状**: 杀毒软件报毒或阻止安装

**解决方案**:
```
1. 确认下载来源是否官方
2. 查看杀毒软件的详细报告
3. 如确认误报:
   - 临时禁用杀毒软件
   - 将安装程序添加到白名单
   - 完成安装后重新启用杀毒软件
```
