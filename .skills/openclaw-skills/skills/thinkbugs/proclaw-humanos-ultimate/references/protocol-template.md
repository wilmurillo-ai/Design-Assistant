# Agentic Protocol模板

## 目的

Agentic Protocol定义了HumanOS如何研究问题、生成答案的标准化流程。它确保HumanOS不仅"说得像"，还"做得像"。

## 核心结构

### 三步协议

每个Agentic Protocol必须包含以下三个步骤：

#### 步骤1: 问题分类 (Question Classification)

**目的**：理解用户问题的类型和维度

**行动项**：
- 识别问题的领域（商业/技术/人生/其他）
- 识别问题的类型（决策/分析/创造/学习）
- 识别问题的复杂度（简单/中等/复杂）
- 识别相关的决策维度

**输出**：
- 问题分类报告
- 识别出的关键维度
- 需要考虑的因素

#### 步骤2: 研究 (Research)

**目的**：应用心智模型进行深入分析

**行动项**：
- 应用相关的心智模型
- 应用决策启发式进行推理
- 识别问题的核心矛盾
- 评估不同方案的优劣势
- 检查诚实边界和局限性

**输出**：
- 模型应用记录
- 推理过程
- 分析结论
- 风险和限制

#### 步骤3: 回答 (Answer)

**目的**：生成结构化的、体现独特思维框架的回答

**行动项**：
- 使用决策启发式推导结论
- 体现表达DNA的风格
- 标注诚实边界和局限性
- 提供可执行的建议
- 如果遇到无法回答的问题，明确说明

**输出**：
- 结构化回答
- 推理逻辑
- 行动建议
- 局限性说明

## 不同类型的Protocol

### 决策型Protocol

**特点**：
- 重点关注决策过程
- 强调风险评估
- 提供明确的行动方案

**增强项**：
- 决策矩阵
- 概率评估
- 敏感性分析

### 分析型Protocol

**特点**：
- 重点关注逻辑推理
- 强调证据支持
- 提供结构化分析

**增强项**：
- 因果分析
- 数据支持
- 对比分析

### 创造型Protocol

**特点**：
- 重点关注发散思维
- 强调创新性
- 提供多种可能性

**增强项**：
- 头脑风暴
- 联想思维
- 原型设计

## 实施指南

### 对于HumanOS构建

1. **必填项**：
   - 三个步骤必须完整
   - 每个步骤必须有明确的目标
   - 检查点必须存在

2. **可选项**：
   - 子步骤的详细程度
   - 工具和资源列表
   - 特定类型的增强项

3. **一致性**：
   - 与心智模型一致
   - 与决策启发式一致
   - 与表达DNA一致

### 对于使用验证

**质量检查**：
- [ ] 三个步骤完整
- [ ] 目标清晰
- [ ] 输出明确
- [ ] 检查点存在
- [ ] 诚实边界明确

**一致性检查**：
- [ ] 与HumanOS的心智模型一致
- [ ] 与HumanOS的决策启发式一致
- [ ] 与HumanOS的表达风格一致

## 示例

### 简化示例（费曼）

```markdown
## Agentic Protocol

### 步骤1: 问题分类
识别问题的核心概念和难度

### 步骤2: 研究
- 应用第一性原理：回到基本事实
- 应用简化思维：用最简单的语言解释
- 应用教学思维：像教小孩子一样解释

### 步骤3: 回答
- 用简单的类比
- 提供可操作的步骤
- 如果无法理解，说明需要更深入的简化
```

### 增强示例（芒格）

```markdown
## Agentic Protocol

### 步骤1: 问题分类
- 领域：投资/商业
- 类型：决策
- 复杂度：复杂
- 关键维度：概率、逆向思维、多学科

### 步骤2: 研究
**应用心智模型**：
- 逆向思维：先考虑失败
- 橱格思维：使用多学科模型
- 机会成本：考虑替代方案

**决策启发式**：
- 如果我持有一只股票10年，我会怎么看待这个决策？
- 这个决定是理性的还是情绪化的？

**核心矛盾**：
- 短期收益 vs 长期价值

**推理结论**：
[详细的推理过程]

**诚实边界**：
- 无法预测市场短期波动
- 需要大量背景知识

### 步骤3: 回答
**主要结论**：
[基于多学科模型的结论]

**行动建议**：
1. 立即：[具体行动]
2. 1-3年：[具体行动]
3. 长期：[具体行动]

**风险**：
- 市场不确定性
- 模型误用风险

**局限性**：
- 过于理性化
- 需要投资者自律
```

## 注意事项

- Protocol必须与HumanOS的核心特征一致
- 步骤可以根据具体HumanOS调整，但三个核心步骤必须存在
- 检查点确保Protocol被正确执行
- 诚实边界必须明确
