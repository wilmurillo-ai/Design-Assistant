## 概览

ProClaw HumanOS支持4种类型，根据需求自动选择最合适的类型。

### 类型矩阵

| 类型 | 定义 | 触发词示例 | 核心特征 | 优先维度 |
|------|------|-----------|---------|---------|
| **HumanType** | 蒸馏特定人物的思维框架 | "蒸馏芒格"、"费曼视角" | 角色扮演、表达DNA、智识谱系 | 著作⭐⭐⭐, 对话⭐⭐⭐, 决策⭐⭐⭐ |
| **ThemeType** | 综合多人视角的主题框架 | "反脆弱OS"、"价值投资框架" | 流派对比、领域共识、中性表达 | 著作⭐⭐⭐, 他者⭐⭐⭐ |
| **ScenarioType** | 特定场景的决策系统 | "创业决策系统"、"产品评估" | 决策流程、工具箱、风险识别 | 决策⭐⭐⭐, 工具⭐⭐ |
| **MethodologyType** | 完整的方法论实施框架 | "设计思维OS"、"精益创业" | 实施步骤、工具模板、变体适应 | 著作⭐⭐⭐, 工具⭐⭐⭐ |

## HumanType

### 定义
蒸馏特定人物的思维框架，创建可运行的人格Skill。

### 适用场景
- 用户指定具体人名："蒸馏马斯克"、"创建乔布斯skill"
- 角色扮演需求："我要像巴菲特一样思考"
- 学习大师思维："费曼如何简化复杂概念？"

### 核心特征
1. **角色扮演规则**：如何像此人一样思考、表达、行动
2. **表达DNA**：句式、词汇、节奏、风格
3. **智识谱系**：影响此人的思想家、书籍、事件
4. **决策启发式**：此人的决策规则和直觉
5. **诚实边界**：此人的局限性和做不到的事

### 采集策略
- **优先维度**：著作(01)、对话(02)、决策(05)、时间线(06)
- **次优维度**：表达(03)、他者(04)

### 输出内容
- 心智模型（3-7个）
- 决策启发式（5-10条）
- 表达DNA（句式、词汇、节奏）
- 价值观（3-5条）
- 工具箱（3-10个）
- 内在张力（2-5处）
- 诚实边界（3-5条）

### 示例
- `charlie-munger-perspective` - 芒格思维框架
- `feynman-perspective` - 费曼教学思维
- `naval-perspective` - 纳瓦尔财富智慧

## ThemeType

### 定义
综合多人视角的主题框架，创建中性的领域思维OS。

### 适用场景
- 主题需求："构建价值投资框架"、"反脆弱OS"
- 综合视角："创业思维体系"、"批判思维框架"
- 领域系统："产品方法论"、"设计思维"

### 核心特征
1. **框架概览**：主题的核心定义和边界
2. **流派对比**：不同观点的对比分析
3. **领域共识**：该主题的公认原则
4. **中性表达**：不带个人色彩的客观描述
5. **应用场景**：何时使用此框架

### 采集策略
- **优先维度**：著作(01)、他者(04)
- **次优维度**：对话(02)、决策(05)

### 输出内容
- 核心概念（5-8个）
- 主要流派（3-5个）
- 共识原则（5-8条）
- 应用工具（5-10个）
- 常见误区（3-5个）
- 适用场景（3-5个）

### 示例
- `value-investing-os` - 价值投资框架
- `antifragility-os` - 反脆弱系统
- `design-thinking-os` - 设计思维

## ScenarioType

### 定义
特定场景的决策系统，创建场景化的决策支持OS。

### 适用场景
- 场景需求："创业决策系统"、"产品评估系统"
- 决策支持："投资决策助手"、"职业选择系统"
- 问题解决："技术选型系统"、"团队管理OS"

### 核心特征
1. **决策流程**：标准化的决策步骤
2. **评估框架**：评估选项的标准化方法
3. **风险识别**：常见风险和应对策略
4. **工具箱**：场景特定的工具和模板
5. **案例库**：典型决策案例

### 采集策略
- **优先维度**：决策(05)、著作(01)
- **次优维度**：他者(04)、对话(02)

### 输出内容
- 决策流程（5-8步）
- 评估框架（3-5个）
- 风险清单（5-10项）
- 工具箱（5-10个）
- 决策模板（3-5个）
- 典型案例（3-5个）

### 示例
- `startup-decision-os` - 创业决策系统
- `product-evaluation-os` - 产品评估系统
- `investment-decision-os` - 投资决策系统

## MethodologyType

### 定义
完整的方法论实施框架，创建可执行的方法论OS。

### 适用场景
- 方法论需求："设计思维OS"、"精益创业"
- 实施指导："敏捷开发框架"、"用户研究方法"
- 流程系统："写作方法论、编程方法论"

### 核心特征
1. **实施步骤**：详细的可执行步骤
2. **工具模板**：可直接使用的工具和模板
3. **变体适应**：不同情境下的调整方法
4. **最佳实践**：成功经验和常见陷阱
5. **质量控制**：检查点和验收标准

### 采集策略
- **优先维度**：著作(01)、工具(衍生)
- **次优维度**：决策(05)、时间线(06)

### 输出内容
- 实施步骤（8-12步）
- 工具模板（5-10个）
- 变体方法（3-5个）
- 检查清单（5-10项）
- 常见问题（5-10个）
- 成功要素（3-5个）

### 示例
- `design-thinking-os` - 设计思维方法论
- `lean-startup-os` - 精益创业方法论
- `agile-methodology-os` - 敏捷开发方法论

## 类型选择指南

### 决策树

```
用户需求
  ↓
是否有明确人名？
  ├─ 是 → HumanType
  └─ 否 → 是否是具体场景？
      ├─ 是 → ScenarioType
      └─ 否 → 是否是实施方法？
          ├─ 是 → MethodologyType
          └─ 否 → ThemeType
```

### 关键问题

**HumanType关键问题**：
- 你想蒸馏谁的思维框架？
- 你是否知道这个人的名字？
- 你需要角色扮演功能吗？

**ThemeType关键问题**：
- 你想构建什么领域的思维框架？
- 这个主题是否有明确的定义和边界？
- 你需要综合多个视角吗？

**ScenarioType关键问题**：
- 你想在什么场景下做决策？
- 是否有明确的决策流程？
- 你需要风险评估功能吗？

**MethodologyType关键问题**：
- 你需要可执行的实施步骤吗？
- 是否需要具体的工具和模板？
- 你需要检查点和质量控制吗？

### 混合类型

某些HumanOS可能具有多种类型的特征，此时：
1. 选择主导类型
2. 在次要类型的维度上加强采集
3. 在输出中融合多种类型的特征

### 类型转换

HumanOS可以在演化过程中转换类型：
- HumanType → ThemeType：从个人思维上升为通用框架
- ScenarioType → MethodologyType：从决策流程发展为方法论
- ThemeType → HumanType：从抽象框架具象为人物思维

## 使用建议
- 优先使用HumanType进行人物蒸馏
- ThemeType适合构建领域知识系统
- ScenarioType适合重复性决策场景
- MethodologyType适合流程化实施
- 类型可以演化，不是固定的
