# 质量检查清单使用指南

---

## 📋 检查清单概览

### 五大核心问题

| 序号 | 问题 | 检查要点 |
|------|------|----------|
| 1 | 开场三句话，读者会不会继续往下读？ | 具体场景、用"你"称呼、不提定义 |
| 2 | 错误答案，读者会不会觉得"我之前也这么想"？ | 2-3 种误解、致死反例、有杀伤力 |
| 3 | 正确答案，读者能不能给别人讲明白？ | 一句话本质、例子支撑、逻辑完整 |
| 4 | 触类旁通，读者会不会觉得"原来这么多领域都相关"？ | 4-5 个领域、各有案例、有洞察 |
| 5 | 结尾，读者会不会有"值了"的感觉？ | 回到开场、更深洞察、不说教 |

### 方法二：手动检查

写作完成后，逐个回答以下问题：

#### ✅ 开场检查
- [ ] 第一句是否有具体场景（时间、地点、人物）？
- [ ] 是否用"你"称呼读者？
- [ ] 是否避免"本文将探讨"等学术腔？
- [ ] 是否提出核心问题而非直接给答案？

#### ✅ 错误答案检查
- [ ] 是否列出至少 2 种常见误解？
- [ ] 每种误解是否有具体反例？
- [ ] 反例是否有杀伤力（不是温和反驳）？
- [ ] 读者会不会觉得"我之前也这么想"？

#### ✅ 核心洞察检查
- [ ] 能否用一句话说清本质？
- [ ] 是否有至少 2 个具体例子支撑？
- [ ] 逻辑链条是否完整？
- [ ] 读者能否转述给别人？

#### ✅ 跨领域检查
- [ ] 是否覆盖 4-5 个不同领域？
- [ ] 每个领域是否有具体案例？
- [ ] 领域之间是否有明显差异？
- [ ] 每个案例是否有洞察总结？

**推荐领域组合：**
- 商业/职场
- 历史/文化
- 艺术/创作
- 科技/AI
- 日常生活/教育/医疗

#### ✅ 结尾检查
- [ ] 是否回到开场场景？
- [ ] 是否给出更深层洞察？
- [ ] 是否有可执行的建议？
- [ ] 是否避免"所以我们应该"等说教？

## 📊 结构检查清单

| 检查项 | 标准 | 检查方法 |
|--------|------|----------|
| 段落数量 | 6-8 个主要段落 | 目测或数空行 |
| 总字数 | 4000-6000 字 | `wc -m article.md` |
| 框架标题 | 不出现"开场故事"等 | 搜索禁用词 |
| 数据对比 | 至少 2-3 组数据 | 搜索数字和百分比 |
| 书籍推荐 | 10 本，分 3 级 | 数《》数量和分级词 |

## 🚫 禁用词清单

### 绝对不要出现的框架标题
```
开场故事
错误答案
正确答案
触类旁通
对比制造
结尾升华
七步框架
```

### 避免的学术腔
```
本文将探讨
笔者认为
综上所述
由此可见
在 xx 现象学原理中
从 xx 理论来看
根据 xx 模型
```

### 避免的说教语气
```
所以我们应该
因此必须
希望大家
让我们
读者应该
```


## 📝 检查流程建议

### 写作前
1. 确定核心问题/概念
2. 想好开场场景
3. 列出 2-3 个常见误解
4. 准备 4-5 个领域的案例

### 写作中
1. 按照七步框架写（不显示标题）
2. 每写完一段，对照检查清单
3. 确保用"你"称呼，避免学术腔

### 写作后
1. 运行自动检查脚本
2. 手动回答五大核心问题
3. 根据建议修改
4. 再检查一遍

## 🎯 快速评分表

| 维度 | 1 分 | 3 分 | 5 分 |
|------|------|------|------|
| 开场吸引力 | 抽象定义 | 有场景但平淡 | 具体场景+代入感 |
| 错误答案 | 1 种或无反例 | 2 种有反例 | 3 种反例狠 |
| 核心洞察 | 模糊不清 | 能理解但难转述 | 一句话能说清 |
| 跨领域 | 1-2 个领域 | 3 个领域 | 4-5 个领域 |
| 结尾 | 简单总结 | 有洞察但说教 | 闭环+洞察+行动 |

**总分 20-25 分**：可以发布  
**总分 15-19 分**：需要修改  
**总分<15 分**：建议重写
