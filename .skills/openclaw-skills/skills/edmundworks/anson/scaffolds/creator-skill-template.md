# Creator Skill Template

Every maker skill generated by anson must support two modes: bootstrap and update. This template defines the contract.

## Frontmatter

Follow the Agent Skills specification:

```yaml
---
name: [identity-maker | user-maker | soul-maker | agents-maker]
description: [What this maker does and when to trigger it — include both bootstrap and update scenarios]
---
```

## Two-mode contract

### Bootstrap mode

Bootstrap mode runs during the initial anson setup. The maker skill:

1. Reads any existing context (prefill files, prior documents)
2. Conducts an interview with the user to gather information
3. Creates the target document (IDENTITY.md, USER.md, SOUL.md, or agent instructions)

The interview should be conversational, not a questionnaire. Ask one question at a time. Build on answers. Infer what you can. Know when to stop.

### Update mode

Update mode runs after bootstrap, whenever the assistant learns something durable or clarifying that belongs in the target document. The maker skill:

1. Reads the current target document
2. Identifies what has changed or what new insight has emerged
3. Makes a precise, targeted update — not a full rewrite
4. Preserves continuity and existing content unless there is real evidence it should change

Update mode should be proactive, not passive. It triggers when:

- New context reveals something durable about the user, the assistant's role, or the relationship
- A correction or clarification from the user implies the document should evolve
- The assistant notices a pattern over time that the document doesn't yet capture

Update mode should NOT trigger for:

- Momentary preferences or one-off requests
- Information that belongs in conversation memory rather than identity/user/soul
- Minor phrasing improvements with no substantive change

## Interview style (bootstrap mode)

- Ask one question at a time
- Build each question on the previous answer
- Use the framing and style described in the relevant creator-context file
- Be sensitive to the user's communication style
- Know when to stop — don't over-interview
- Confirm the final document with the user before writing

## Output

The target document should be:

- Written in markdown
- Placed in the workspace root (path from ANSON_META.md)
- Structured according to the template defined in the creator-context file
- Written in a voice appropriate to its purpose (IDENTITY.md describes the agent, USER.md describes the user, SOUL.md describes the relationship)
