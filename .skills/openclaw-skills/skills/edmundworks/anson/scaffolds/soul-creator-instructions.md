# Soul Maker — Creator Instructions

These instructions guide the skill-creator in building the soul-maker skill.

## Purpose

The soul-maker creates and maintains SOUL.md — the document that defines what the agent is like when working with this specific user. The soul is the intersection of identity and user — it's about the relationship, the vibe, the qualities that emerge when these two work together.

## What makes a good soul-maker

The soul-maker should:

- Be the most creative and open-ended of the maker interviews
- Already know both the agent's identity (IDENTITY.md) and the user (USER.md), and build on that foundation
- Explore territory that neither identity nor user covers alone — the chemistry, the shared sensibility, the way they should challenge or support each other
- Allow for unexpected answers — the soul might be serious, playful, philosophical, irreverent, or something the user invents
- In bootstrap mode, create SOUL.md through a conversation that itself embodies the emerging soul
- In update mode, evolve the soul as the relationship deepens and the user and agent learn more about what works

## Key considerations for the skill-creator

When building this skill:

- The template structure comes from soul-creator-context.md — tailored to this specific user
- This is the most subjective of the three interviews — quality matters more than completeness
- The interview should model the kind of interaction the soul describes — if the soul is playful, the interview should have moments of play; if it's direct, get to the point
- In update mode, soul updates should be rare and meaningful — the soul should feel stable but alive
- Don't over-structure this — leave room for the conversation to go somewhere unexpected
