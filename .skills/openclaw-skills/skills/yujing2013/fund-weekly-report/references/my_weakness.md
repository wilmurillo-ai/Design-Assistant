## 七点五、QDII/REITs平均收益提取错误（2026-03-16修复）

### ❌ 错误：用TOP基金平均值代替Excel平均收益行

**问题描述：**
脚本在计算QDII和REITs平均收益时，错误地取TOP基金列表的平均值，而不是Excel中明确提供的"平均收益"行数据。

**Excel数据结构：**
```
QDII基金_收益top Sheet:
| 证券简称 | 近一周收益 | 年初以来收益 |
|---------|-----------|-------------|
| 汇添富黄金及贵金属A | 0.067905 | 0.159555 |
| 易方达黄金主题A人民币 | 0.060704 | 0.145347 |
| ... | ... | ... |
| 平均收益 | 0.002455 | 0.047890 |  ← 这行被忽略了
```

**错误代码：**
```python
# 错误做法：计算TOP基金的平均值
avg_return = sum([x['近一周收益'] for x in qdii_data]) / len(qdii_data)
```

**错误结果对比：**

| 类型 | 错误值 | 正确值 | Excel原始值 |
|------|--------|--------|-------------|
| QDII近一周 | 5.04% ❌ | 0.25% ✅ | 0.002455 |
| QDII年初以来 | 12.38% ❌ | 4.79% ✅ | 0.047890 |
| REITs近一周 | 7.07% ❌ | 2.63% ✅ | 0.026263 |
| REITs年初以来 | - | 4.90% ✅ | 0.049042 |

**根因分析：**
- `extract_qdii_data()` 函数只返回基金列表，没有提取"平均收益"行
- 计算平均收益时用TOP基金平均值，而不是Excel中的整体平均

### ✅ 修复方案

**修改函数返回值：**
```python
def extract_qdii_data(fund_data: Dict[str, pd.DataFrame]) -> tuple:
    """提取QDII基金数据，返回 (基金列表, 平均收益字典)"""
    funds = []
    avg_return = {'近一周': None, '年初以来': None}
    df = fund_data.get('QDII基金_收益top')
    if df is not None:
        for _, row in df.iterrows():
            name = row.get('证券简称')
            if pd.notna(name):
                if name == '平均收益':
                    avg_return['近一周'] = row.get('近一周收益')
                    avg_return['年初以来'] = row.get('年初以来收益')
                else:
                    funds.append({...})
    return funds, avg_return
```

**修改周报生成：**
```python
qdii_funds, qdii_avg = extract_qdii_data(fund_data)
# 使用Excel中的平均收益
avg_return = qdii_avg.get('近一周', 0) or 0
ytd_avg_return = qdii_avg.get('年初以来', 0) or 0
```

## 七点六、增强指基跑赢跑输判断错误（2026-03-16修复）

### ❌ 错误：跑赢跑输判断逻辑完全错误

脚本在4.2章节增强指基中，超额收益均值说完后，跑赢跑输的判断逻辑完全错误。原来的逻辑是：只有在数据缺失时才写"跑赢对标指数"，而不是根据实际超额收益正负来判断。

## 七点七、2.1/2.2标题生成逻辑改进（2026-03-16修复）

### ❌ 问题：标题逻辑过于简单，不能反映市场真实情况

**问题描述：**
- 2.1标题：只判断是否为负，没有区分"收正"、"收负"、"均为负值"等情况
- 2.2标题：只显示领涨行业，没有反映整体涨跌情况
- 2.2正文：写死了"涨跌互现"，与标题可能不一致

**错误示例：**
```
2.1 收益分布：周收益表现分化  # ← 中位数全部为正时仍显示"表现分化"
2.2 行业主题基金：周期主题基金平均上涨6.25%  # ← 缺少整体涨跌判断
```

### ✅ 修复方案

#### 2.1 标题规则

**根据中位数正负判断：**
- 全部为负：`普通股票/偏股混合/灵活配置基金周收益中位数均为负值，尾部产品周跌幅超过xx%`
- 多数为负（≥3个负）：`各类主动权益基金周收益中位数收负，尾部产品周跌幅超过xx%`
- 多数为正（≥3个正）：`各类主动权益基金周收益中位数收正，尾部产品周跌幅超过xx%`
- 涨跌互现：`各类主动权益基金周收益中位数涨跌互现`

#### 2.2 标题规则

**先判断涨跌情况，再加领涨行业：**
- 跌多涨少（下跌行业数 > 上涨行业数 * 1.5）：`各类主动权益基金跌多涨少，xxx主题基金平均上涨xx%`
- 涨多跌少（上涨行业数 > 下跌行业数 * 1.5）：`各类主动权益基金涨多跌少，xxx主题基金平均上涨xx%`
- 涨跌互现：`各类主动权益基金涨跌互现，xxx主题基金平均上涨xx%`
- 全部下跌：`各类主动权益基金普遍下跌，xxx主题基金跌幅相对较小`

**修复后示例：**
```
2.1 收益分布：各类主动权益基金周收益中位数收正，尾部产品周跌幅超过5%
2.2 行业主题基金：各类主动权益基金涨多跌少，周期主题基金平均上涨6.25%
```

**修复时间：** 2026-03-16

## 七点八、2.1标题规则修正（2026-03-16修复）

### ❌ 问题：2.1标题规则逻辑错误

**问题描述：**
- 全部为正/负时，应该用"收正/收负"
- 有正有负时，应该把多数的单独列出来，比如"普通股票/偏股混合/灵活配置基金周收益中位数均为正值"

**错误逻辑：**
```python
if negative_count == len(medians_valid):
    title = "普通股票/偏股混合/灵活配置基金周收益中位数均为负值"  # ← 应该是"收负"
elif negative_count >= 3:
    title = "各类主动权益基金周收益中位数收负"  # ← 应该列出具体的负的类型
```

### ✅ 修正方案

**正确逻辑：**
1. **全部同号** → `各类主动权益基金周收益中位数收正/收负`
2. **多数为正** → `普通股票/偏股混合/灵活配置基金周收益中位数均为正值`（列出为正的）
3. **多数为负** → `普通股票/偏股混合/灵活配置基金周收益中位数均为负值`（列出为负的）
4. **正负各半** → `各类主动权益基金周收益中位数涨跌互现`

**修复时间：** 2026-03-16
