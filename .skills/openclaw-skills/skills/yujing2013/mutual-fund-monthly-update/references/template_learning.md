## 学习目标

精确学习用户Excel模板，确保生成的Excel与模板结构完全一致。

## 学习流程

### 第一步：结构扫描

```python
# 扫描模板所有单元格
for sheet in workbook:

            记录：
            - 单元格坐标 (A1, B3, ...)
            - 单元格值
            - 单元格样式
            - 是否合并单元格
```

### 第二步：字段识别

**识别哪些单元格需要填充数据：**

| 单元格类型 | 特征 | 处理方式 |
|-----------|------|---------|
| 行标题 | 文本，在第一列 | 作为字段名，建立映射 |
| 列标题 | 文本，在第一行 | 作为日期标识 |
| 数据单元格 | 空值或数值 | 需要填充数据 |
| 固定文本 | 非空文本（非标题） | 保持不变 |
| 公式单元格 | 包含公式 | 保持公式不变 |

**示例：**

```
模板：
| A        | B      | C      | D      |
| 指标     | 10月   | 11月   | 12月   |  ← 列标题
| 久期     |        |        |        |  ← 行标题，B/C/D空，需填充
| YTM      | 5.54%  |        |        |  ← B有示例值，C/D需填充
| 基金名称 | 汇丰   | 汇丰   | 汇丰   |  ← 固定文本，保持不变

AI学习结果：
- 行标题：久期、YTM（需填充）
- 列标题：10月、11月、12月（日期）
- 数据区域：B2:D3（需填充）
- 固定文本：基金名称行（保持不变）
```

## 模板类型识别

### 类型1：横向时间对比

```
| 指标 | 10月 | 11月 | 12月 |
| 久期 |      |      |      |
```

**特点**：日期在列标题，数据按月横向展开

### 类型2：纵向时间对比

```
| 日期   | 久期 | YTM  |
| 10月   |      |      |
| 11月   |      |      |
```

**特点**：日期在行标题，数据按月纵向展开

### 类型3：多Sheet对比

```
Sheet1: 汇丰亚洲债券
Sheet2: 摩根亚洲债券
```

**特点**：每个Sheet一个产品，Sheet内按月对比

## 错误处理

### 情况1：字段无法映射

```
模板字段："组合久期"
PDF中找不到匹配字段

AI处理：
1. 尝试模糊匹配（久期、Duration）
2. 如果仍无法匹配，标记为"未找到"
3. 在输出Excel中用红色高亮显示
```

### 情况2：PDF数据缺失

```
模板要求：信用评级分布（AAA/AA/A/BBB...）
PDF中只有：评级分布（无详细分类）

AI处理：
1. 提取可用的评级数据
2. 无法填充的单元格标记为"N/A"
3. 在输出说明中注明
```

### 情况3：格式冲突

```
模板要求：百分比格式（5.54%）
PDF数据：小数格式（0.0554）

AI处理：
- 自动转换为模板要求的格式
- 0.0554 → 5.54%
```

## 字段模糊匹配规则（v2.1新增）

### 匹配策略

**三层匹配机制：**

1. **精确匹配**（优先级最高）
   - 模板字段名完全等于PDF字段名
   - 示例：模板"久期" = PDF"久期"

2. **别名匹配**（优先级中等）
   - 使用预定义的字段别名表
   - 示例：模板"组合久期" → PDF["久期", "Duration", "平均年期"]

3. **模糊匹配**（优先级最低）
   - 关键词匹配（包含关系）
   - 语义相似度匹配（基于词向量）
   - 示例：模板"到期收益率" → PDF"平均最低孳息率"（含"收益率"）

## 数据类型保持规则（v2.1新增）

### 核心原则

**不要改变模板中每个格子的数据类型！**

### 数据类型识别

```python
def get_cell_data_type(cell):
    """识别单元格的数据类型"""
    
    # 1. 检查数字格式
    if cell.number_format:
        if '%' in cell.number_format:
            return 'percentage'  # 百分比
        elif cell.number_format.startswith('0.00'):
            return 'decimal'  # 小数
        elif cell.number_format.startswith('0'):
            return 'integer'  # 整数
    
    # 2. 检查值类型
    if isinstance(cell.value, (int, float)):
        return 'number'  # 数字
    elif isinstance(cell.value, str):
        if cell.value.endswith('%'):
            return 'percentage_text'  # 百分比文本
        else:
            return 'text'  # 文本
    
    # 3. 默认
    return 'unknown'
```

### 数据类型转换规则

| 模板类型 | PDF数据 | 转换规则 | 示例 |
|---------|---------|---------|------|
| 百分比格 | 6.7 | `value / 100` → `0.067` → 单元格显示 `6.7%` | 6.7 → 0.067 |
| 百分比格 | 6.7% | 去掉%，转为小数 | "6.7%" → 0.067 |
| 小数格 | 2.3年 | 提取数字 | "2.3年" → 2.3 |
| 文本格 | 开放申购 | 保持文本 | "开放申购" → "开放申购" |
| 整数格 | 253.25百万 | 提取数字（按情况处理） | "253.25百万" → 253.25 |

### 错误示例（避免）

```
❌ 错误做法：
模板单元格格式：百分比
PDF数据：6.7
AI写入：ws['F12'] = '6.7%'  # 错误！写入文本

✅ 正确做法：
模板单元格格式：百分比
PDF数据：6.7
AI写入：ws['F12'] = 0.067  # 正确！写入小数，Excel自动显示为6.7%
```

## 学习结果存储

**每次学习都是临时的，不永久保存：**

```python
# 学习结果存储在内存中
template_learning_result = {
    "structure": {...},
    "mapping": {...},
    "styles": {...},
    "merged_cells": [...]
}

# 生成Excel后清除
del template_learning_result
```

**原因**：
- 每次用户的模板可能不同
- 不需要"记住"之前的模板
- 每次都是全新的学习
