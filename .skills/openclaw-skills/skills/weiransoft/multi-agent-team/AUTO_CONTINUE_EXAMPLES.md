## 示例 3: 紧急 Bug 修复

### 场景描述
广告拦截功能失效，大量用户投诉，需要紧急修复。

### 执行流程

#### 第一阶段：问题定位（0-30%）
```

🔍 任务分解:
- [in_progress] 任务 1: 复现问题，收集日志
- [pending] 任务 2: 分析根因
- [pending] 任务 3: 制定修复方案
- [pending] 任务 4: 立即修复
- [pending] 任务 5: 快速验证

进度：0%
```

#### 第二阶段：根因分析（30-50%）
```
🔍 任务 1 完成：问题复现成功

复现步骤:
1. 访问 YouTube
2. 播放视频
3. 观察：广告未拦截

日志分析:
- 错误信息：CSS selector ".ad-banner" not found
- 调用栈：AdBlocker.block() line 127
- 原因：YouTube 更新了广告 DOM 结构

🔍 任务 2：根因分析

5-Why 分析:
Why 1: 为什么未拦截？→ CSS 选择器未匹配
Why 2: 为什么未匹配？→ YouTube 更新了 DOM 结构
Why 3: 为什么未检测？→ 缺少动态 DOM 监控
Why 4: 为什么缺少？→ 架构设计未考虑动态场景
Why 5: 为什么未考虑？→ 需求分析不全面

根因：架构设计未考虑动态 DOM 场景

进度：50%
```

```

- 添加 3 个备用选择器

## 关键特性总结

### 1. 用户无感知体验

**用户看到的**:
```
"正在重构用户模块... (进度 65%)"
↓ （思考次数超限）
"继续执行中... (进度 75%)"
↓ （自动继续）
"重构完成！所有测试通过 ✅"
```

**用户不需要的**:
- ❌ 手动重复需求
- ❌ 手动检查进度
- ❌ 手动触发继续
- ❌ 担心任务中断

**系统自动做的**:
- ✅ 自动保存进度
- ✅ 自动恢复上下文
- ✅ 自动继续执行
- ✅ 自动验证结果

### 2. 进度持久化

**存储位置**: `.trae-multi-agent/progress.md`

**记录内容**:
- 任务列表和状态
- 代码变更历史
- 验证结果（编译/测试）
- 下一步计划
- 阻塞与风险
- 自动续期记录

**支持操作**:
- 自动更新（每次操作后）
- 手动回滚（到任意进度点）
- 进度导出（分享/备份）

### 3. 智能续期策略

**续期判断**:
- 剩余思考次数 < 2 次
- 评估当前任务完成度
- 决策：暂停保存 or 优化收尾

**续期执行**:
1. 保存所有上下文
2. 输出进度提示
3. 自动加载继续
4. 保持任务连贯性

### 4. 分块处理策略

**大任务分解**:
```
原任务：重构整个用户模块（20 个文件）
↓
分块 1: 用户模型层（2 个文件）
分块 2: 用户控制层（1 个文件）
分块 3: 用户安全层（3 个文件）
分块 4: 用户测试层（2 个文件）
```

**每块独立验证**:
- 完成分块 → 立即验证
- 验证通过 → 提交 → 继续
- 验证失败 → 修复 → 重验
