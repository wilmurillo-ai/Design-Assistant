# 使用指南

## 1. 技能介绍

### 1.1 技能概述

**trae-multi-agent** 是一个多智能体协作技能，能够根据任务类型动态调度到合适的智能体角色（架构师、产品经理、测试专家、独立开发者）。支持多智能体协作、共识机制和完整项目生命周期管理。

### 1.2 核心功能

- **智能体调度**: 根据任务类型自动分配给合适的智能体角色
- **规范驱动开发**: 基于项目规范和文档进行开发
- **代码地图生成**: 生成项目代码结构和依赖关系图
- **项目理解**: 快速读取项目文档和代码，生成角色特定的理解文档
- **多语言支持**: 支持中英文双语

## 2. 安装与配置

### 2.1 安装方法

1. **克隆技能到本地**
   ```bash
   git clone <repository-url> .trae/skills/trae-multi-agent
   ```

2. **安装依赖**
   ```bash
   cd .trae/skills/trae-multi-agent
   pip install -r requirements.txt
   ```

### 2.2 配置文件

- **skills-index.json**: 技能配置文件，定义技能的基本信息、关键词和依赖
- **requirements.txt**: Python依赖包列表

## 3. 使用方法

### 3.1 基本使用

1. **启动技能**
   在Trae IDE中，通过技能面板启动 `trae-multi-agent` 技能。

2. **输入任务描述**
   输入您的任务描述，例如：
   - "设计一个微服务架构的电商系统"
   - "为用户注册功能编写测试用例"
   - "分析项目代码结构并生成架构文档"

3. **智能体自动调度**
   技能会根据任务类型自动选择合适的智能体角色：
   - **架构师**: 负责系统架构设计、代码审查、技术选型
   - **产品经理**: 负责需求分析、PRD编写、产品规划
   - **测试专家**: 负责测试用例设计、测试执行、缺陷管理
   - **独立开发者**: 负责全栈开发、项目管理、部署运维

### 3.2 高级使用

#### 3.2.1 规范驱动开发

1. **创建项目规范**
   - 在 `docs/spec/` 目录中使用模板创建项目规范文档
   - 填写项目信息、功能需求、非功能需求等

2. **分析规范**
   技能会自动分析项目规范，生成任务分解和执行计划。

3. **执行开发**
   根据规范执行开发任务，确保最终交付符合规范要求。

#### 3.2.2 代码地图生成

1. **生成代码地图**
   ```bash
   python scripts/code_map_generator.py --project-path /path/to/project
   ```

2. **查看代码地图**
   - 生成的代码地图会保存为JSON和Markdown格式
   - 包含目录结构、依赖关系、关键组件等信息

#### 3.2.3 项目理解

1. **生成项目理解文档**
   ```bash
   python scripts/project_understanding.py --project-path /path/to/project
   ```

2. **查看角色特定文档**
   - 架构师文档: 关注系统架构、技术选型、代码质量
   - 产品经理文档: 关注需求分析、功能规划、用户体验
   - 测试专家文档: 关注测试策略、测试用例、缺陷管理
   - 独立开发者文档: 关注全栈实现、项目管理、部署运维

## 5. 智能体工作流程

### 5.1 任务接收与分析

1. **接收任务**
   技能接收用户输入的任务描述。

2. **任务分析**
   分析任务类型、复杂度、所需专业知识。

3. **智能体选择**
   根据任务分析结果选择合适的智能体角色。

### 5.2 规范处理

1. **规范读取**
   读取项目规范文档，了解项目要求。

2. **规范分析**
   分析规范的完整性、一致性、可行性。

3. **任务分解**
   根据规范分解任务，制定执行计划。

### 5.3 任务执行

1. **角色协作**
   多个智能体角色协作完成任务。

2. **进度跟踪**
   跟踪任务执行进度，确保按时完成。

3. **质量保证**
   进行代码审查、测试验证，确保交付质量。

### 5.4 结果交付

1. **成果整理**
   整理任务执行结果，生成交付文档。

2. **结果展示**
   向用户展示任务执行结果。

3. **反馈收集**
   收集用户反馈，持续改进。

## 6. 最佳实践

### 6.1 任务描述最佳实践

- **清晰明确**: 任务描述应清晰明确，避免模糊不清
- **具体详细**: 提供足够的细节，帮助智能体理解任务需求
- **目标明确**: 明确任务的目标和预期结果
- **范围合理**: 任务范围应合理，避免过于庞大或复杂

### 6.2 规范编写最佳实践

- **完整全面**: 规范应包含所有必要的信息
- **结构清晰**: 规范结构应清晰，便于阅读和理解
- **术语一致**: 使用一致的术语，避免歧义
- **可执行性**: 规范应具有可执行性，能够指导实际开发

### 6.3 多智能体协作最佳实践

- **明确分工**: 明确各智能体的职责和分工
- **有效沟通**: 确保智能体之间的有效沟通
- **协同工作**: 各智能体应协同工作，共同完成任务
- **质量控制**: 建立质量控制机制，确保交付质量

## 7. 常见问题与解决方案

### 7.1 问题: 智能体选择不准确

**解决方案**:
- 提供更详细的任务描述
- 明确指定所需的智能体角色
- 提供更多的上下文信息

### 7.2 问题: 代码地图生成失败

**解决方案**:
- 确保项目路径正确
- 确保项目结构完整
- 检查Python依赖是否安装正确

### 7.3 问题: 项目理解文档生成不完整

**解决方案**:
- 确保项目文档齐全
- 确保代码结构清晰
- 增加项目理解的时间和资源

### 7.4 问题: 多智能体协作效率低

**解决方案**:
- 明确任务分工和责任
- 建立有效的沟通机制
- 优化智能体之间的协作流程

## 8. 示例

### 8.1 架构设计示例

**任务描述**:
"设计一个基于Spring Boot的微服务架构电商系统，包含用户服务、商品服务、订单服务和支付服务。"

**执行流程**:
1. 技能选择架构师智能体
2. 架构师分析需求，设计系统架构
3. 生成架构设计文档，包含模块划分、API设计、数据库设计等
4. 提供技术选型建议和部署方案

### 8.2 测试用例设计示例

**任务描述**:
"为用户注册功能编写测试用例，包括正常场景和异常场景。"

**执行流程**:
1. 技能选择测试专家智能体
2. 测试专家分析需求，设计测试用例
3. 生成测试计划和测试用例文档
4. 提供测试执行建议和缺陷管理方案

### 8.3 项目理解示例

**任务描述**:
"分析一个现有的React前端项目，生成项目理解文档。"

**执行流程**:
1. 技能执行项目理解工具
2. 分析项目结构、代码质量、依赖关系
3. 生成角色特定的理解文档
4. 提供项目改进建议

**trae-multi-agent** 技能为项目开发提供了全方位的支持，通过智能体协作和规范驱动开发，提高开发效率和代码质量。使用本技能，可以：

- **提高开发效率**: 自动化任务分配和执行，减少人工干预
- **保证代码质量**: 多智能体协作，从不同角度确保代码质量
- **规范开发流程**: 基于规范进行开发，确保项目一致性
- **降低开发成本**: 减少重复工作，提高资源利用效率
