# 最终交付确认 — 动态拼装指南

> **重构说明**：原版枚举了 8 种固定场景，新版改为动态拼装逻辑，适配任意文档组合，维护成本大幅降低。

---

## 如何使用本指南

1. 根据用户实际选择的文档，从「文档块」中选取对应条目
2. 按顺序拼装：**开头 + 文档块（每个选择的文档） + 分析特点（标准模式用）+ 通用收尾**
3. 填入实际文件名（含日期）、质量检查结果，然后展示并等待用户回复

---

## 文档块（按用户选择拼装）

### 业务流程分析报告
```
**[产品名称]-业务流程分析-[快速/标准]版-YYYYMMDD.md**
内容：需求概述 · 高层级流程图 · 详细流程图（标准版含）· 异常场景分析 · 处理策略
适用：业务对齐、产品评审
```

### 功能架构设计
```
**[产品名称]-功能架构设计-[快速/标准]版-YYYYMMDD.md**
内容：功能架构图 · 分层设计 · 详细功能规格 · 依赖关系图 · 迭代规划（MVP→完整版）
适用：技术团队实施
```

### 埋点方案（快速版）
```
**[产品名称]-埋点清单-快速版-YYYYMMDD.md**
内容：页面埋点清单 · 核心事件定义 · 漏斗定义（3页内精简版）
适用：数据团队快速接入
```

### 埋点方案（标准版）
```
**[产品名称]-埋点方案设计-标准版-YYYYMMDD.md**
内容：埋点概述 · 分类体系 · 页面/事件/用户属性埋点 · 转化漏斗 · 数据看板 · 隐私合规 · 测试计划（13章完整）
适用：完整数据体系建设
```

### 完整 PRD 文档
```
**[产品名称]-PRD-[快速/标准]版-YYYYMMDD.md**
内容：产品概述 · 用户分析 · 功能需求 · 业务流程 · 非功能需求 · 数据需求 · 界面原型 · 技术建议 · 实施计划 · 风险假设 · 运营策略 · 监控优化（含埋点）· 附录（13章完整）
适用：正式立项、对外展示
```

## 标准模式特点块（标准模式必须加）

```
**本次分析特点**：
- ✓ 标准模式深度分析
- ✓ 应用了 [X] 个分析框架
- ✓ 识别了 [X] 个异常场景（含四大类）
- ✓ [包含外部调研与竞品分析 / 基于已知知识的行业分析]
- ✓ 提供了完整的迭代规划（MVP→完整版）
```

## 通用收尾块

```

**请确认**：
1. 文档内容是否符合您的需求？
2. 是否需要修改或补充任何内容？
3. 是否需要生成其他格式的文档？

- **满意** → 回复「确认」，任务完成
- **需修改** → 告知具体修改部分，我将调整后重新交付
- **需补充文档** → 告知需要哪些额外文档，我将继续生成

💡 提示：文档使用 Mermaid 图表，推荐用 Typora、VS Code 或 Notion 查看。
💡 如已在格式选择中选择了 Draw.io（选项 5），对应 .drawio 文件与文档同步生成，可用 Draw.io 或 diagrams.net 打开编辑。
```
