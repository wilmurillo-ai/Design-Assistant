# 异常点检查清单

在分析业务流程时，需要系统性地识别各类异常情况。

## 1. 业务异常

### 数据异常
- 数据缺失：必填字段为空、关键信息未填写
- 数据格式错误：手机号格式、邮箱格式、身份证号格式
- 数据超限：数值超过范围、文本超长、文件过大
- 数据冲突：同一用户重复注册、订单状态冲突
- 数据过期：优惠券过期、会员到期、token失效

### 业务规则异常
- 权限不足：未登录访问、角色权限不足、资源不可访问
- 业务状态不符：订单已取消但尝试支付、已下架商品被购买
- 额度/配额限制：库存不足、账户余额不足、超出限购数量
- 时间限制：非营业时间操作、活动未开始/已结束、超时未支付

### 流程异常
- 跨步骤访问：未完成前置步骤直接跳转
- 流程中断：用户中途退出、长时间未操作
- 流程倒退：已确认订单后修改、已发货后取消
- 并发操作冲突：多设备同时登录、重复提交、超卖

## 2. 场景异常

### 多渠道场景
- PC端与移动端差异
- APP与小程序差异
- 线上与线下场景
- 跨平台操作

### 网络环境
- 弱网环境
- 离线场景
- 网络切换

### 特殊时段
- 高峰期
- 系统维护
- 异常流量

### 边界场景
- 首次使用
- 账号注销
- 极限测试
- 空状态

## 3. 用户异常

### 用户操作异常
- 误操作
- 重复操作
- 非预期路径
- 跳过引导

### 用户理解偏差
- 功能理解错误
- 信息不足
- 多语言问题

### 用户行为异常
- 恶意行为：刷单、套利、薅羊毛
- 账号安全：密码泄露、账号被盗
- 违规内容：发布违禁信息
- 滥用功能：频繁举报、恶意评价

## 4. 技术异常

### 系统层异常
- 服务不可用
- 响应超时
- 资源耗尽
- 依赖服务故障

### 数据层异常
- 数据库异常
- 缓存异常
- 数据一致性
- 数据迁移

### 接口层异常
- 参数校验失败
- 接口版本问题
- 限流/熔断
- 鉴权失败

### 兼容性问题
- 浏览器兼容
- 操作系统兼容
- 设备兼容
- 第三方SDK

### 安全问题
- 注入攻击
- 越权访问
- 敏感信息泄露
- 防刷机制

## 异常点分析流程

1. 绘制主流程
2. 逐步骤检查四类异常
3. 识别关键节点
4. 设计异常处理
5. 输出异常矩阵

## 优先级定义

- **P0**: 影响核心流程，必须处理
- **P1**: 影响用户体验，优先处理
- **P2**: 边界场景，合理安排
- **P3**: 极端场景，后续优化
