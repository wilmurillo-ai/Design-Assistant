# COST TRACKING

Token usage monitoring and budget enforcement to maintain cost efficiency.

## TOKEN TYPES TO TRACK

| Type | Description |
|------|-------------|
| input_tokens | Tokens sent to the model (system prompt + context + user messages) |
| output_tokens | Tokens generated by the model |
| cache_creation_input_tokens | Tokens cached for the first time |
| cache_read_input_tokens | Tokens read from cache (cheaper) |

## PER-CYCLE BUDGET

Configured in CONFIG.yaml:
  cost:
    per_cycle_budget_usd: <number>
    per_gap_target_usd: <number>
    model_tier: haiku | sonnet | opus

## COST ESTIMATION

Token cost per million tokens (approximate, USD):
  haiku: input $0.25, output $1.25
  sonnet: input $3.00, output $15.00
  opus: input $15.00, output $75.00

Formula: cost = (input_tokens * input_rate + output_tokens * output_rate) / 1_000_000

Cache reads are charged at reduced rates (typically 10% of base rate).

## BUDGET PROTOCOL

| Threshold | Action |
|-----------|--------|
| 80% | Log warning in PROGRESS.md |
| 100% | Soft limit: surface to human, ask to continue or stop cycle |
| 120% | Hard limit: halt cycle immediately, write HANDOFF.md |

## COST-PER-GAP METRIC

Primary efficiency KPI: avg_cost_per_gap = total_cycle_cost / gaps_closed_this_cycle

Track in docs/status/EFFICIENCY-HISTORY.md alongside context-efficiency metrics.

If cost-per-gap increases 2+ cycles in a row: flag for harness-variant search.
