# 渠道集成指南

支持的集成方式：
- Webhook（任何能 POST JSON 的平台）
- 微信 / 企业微信：使用企业微信消息 API 或第三方接入（需要 CorpID/Secret）
- 邮件：使用 SMTP/IMAP 拉取未读邮件并调用 /webhook 接口处理
- 工单系统：Zendesk/Freshdesk/Feishu – 使用其 REST API 创建工单（示例见下）

凭证配置建议：
- 使用环境变量或受限凭证文件：不要在代码中写入明文 Key
- 权限分离：机器人仅需消息读取/回复与创建工单权限

速率与错误处理：
- 对于外部渠道的 webhook，确保幂等与重试逻辑（重复消息可能发生）
- 日志与告警：将错误写入监控并在连续失败时通知运维
