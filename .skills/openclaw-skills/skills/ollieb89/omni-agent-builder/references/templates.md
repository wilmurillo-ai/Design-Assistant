# references/templates.md

Use these as starting points. Always tailor to the user.

## IDENTITY.md
- Name, vibe, emoji.
- Optional avatar path.

## USER.md
- Preferred name, address style, timezone, language.
- Durable preferences.
- Hard boundaries (“never do”).

## SOUL.md
- Persona and tone.
- Operating principles.
- Safety boundaries and trust model.
- Continuity rules: where memory lives.

## AGENTS.md (most important)
- Session-start read order.
- Safety defaults (ask-before-destructive, ask-before-outbound, stop-on-CLI-error).
- Memory workflow.
- Delegation protocol + sub-agent rules.
- Evaluation loop.

## HEARTBEAT.md
- Keep empty (or comments only) to skip.
- If used: tiny checklist; avoid deep work; aim for `HEARTBEAT_OK` when idle.

## BOOTSTRAP.md
- One-time first-run ritual.
- After completing, delete the file.

## MEMORY.md (optional)
- Curated, user-approved durable items only.
- Never loaded/used in group contexts.
