## 报告结构说明

企业诊断综合报告由五大板块组成，整合三份基础报告的核心信息：

```
企业诊断综合报告
├── 一、企业概况
│   ├── 基本信息摘要
│   ├── 股权结构
│   └── 经营状况概览
├── 二、财务健康度分析
│   ├── 核心财务指标
│   ├── 盈利能力分析
│   └── 风险提示
├── 三、税务合规性评估
│   ├── 税务状况概览
│   ├── 风险点识别
│   └── 优化建议
├── 四、政策红利与补贴机会
│   ├── 适用政策清单
│   ├── 补贴申请可行性
│   └── 申报时间规划
└── 五、综合诊断结论与建议
    ├── SWOT分析
    ├── 核心问题识别
    └── 改进建议优先级
```

## 企业基本信息板块

### 数据提取清单

从《企业基本信息报告》中提取以下关键数据：

| 信息类别 | 数据项 | 重要性 |
|---------|--------|--------|
| **企业基本资料** | 企业名称、统一社会信用代码 | 必填 |
| | 成立时间、注册资本、实缴资本 | 必填 |
| | 企业类型、经营状态、注册地址 | 必填 |
| | 经营范围 | 必填 |
| **股权结构** | 股东名称、持股比例 | 必填 |
| | 认缴出资额、出资方式 | 选填 |
| | 实际控制人 | 必填 |
| **管理团队** | 法定代表人、高管团队 | 必填 |
| | 核心技术人员 | 选填 |
| **经营状况** | 人员规模、分支机构 | 选填 |
| | 主要业务、客户群体 | 选填 |
| **资质荣誉** | 高新技术企业、专精特新等 | 选填 |

### 数据验证规则

1. **一致性检查**
   - 企业名称在三份报告中应完全一致
   - 统一社会信用代码格式：18位字母数字组合
   - 成立时间格式：YYYY-MM-DD

2. **完整性检查**
   - 标注"必填"项不得缺失
   - 缺失项用 `[缺失]` 标注

## 财税诊断板块

### 数据提取清单

从《财税诊断报告》中提取以下关键数据：

#### 财务指标

| 指标类别 | 具体指标 | 数据来源 |
|---------|---------|---------|
| **盈利能力** | 营业收入、净利润、毛利率、净利率 | 利润表 |
| **营运能力** | 应收账款周转率、存货周转率 | 财务分析 |
| **偿债能力** | 资产负债率、流动比率、速动比率 | 资产负债表 |
| **成长能力** | 营收增长率、利润增长率 | 同比数据 |

#### 税务状况

| 税种 | 关键数据 | 说明 |
|-----|---------|------|
| 增值税 | 适用税率、税负率、应纳税额 | 主要税种 |
| 企业所得税 | 适用税率、应纳税所得额、税负率 | 主要税种 |
| 其他税种 | 个人所得税、印花税等 | 辅助参考 |

#### 风险识别

需要特别标注的风险点：
- ⚠️ 高风险：可能涉及税务违规、重大财务异常
- ⚡ 中风险：需要优化改进的问题
- 💡 低风险：建议关注的事项

## 政策匹配板块

### 数据提取清单

从《政策研究与补贴申请可行性报告》中提取以下关键数据：

#### 适用政策清单

| 政策类型 | 数据项 | 说明 |
|---------|--------|------|
| **税收优惠** | 政策名称、优惠内容、适用条件 | 如高新技术企业税收优惠 |
| **财政补贴** | 补贴名称、补贴标准、申报条件 | 如研发费用补贴 |
| **资质认定** | 认定名称、认定条件、认定好处 | 如专精特新企业认定 |

#### 补贴申请可行性评估

每个补贴项目需评估：

```
项目名称：[补贴项目名称]
申报条件匹配度：[完全匹配/部分匹配/不匹配]
预计补贴金额：[具体金额或区间]
申报时间窗口：[具体时间段或常年]
申报难度：[低/中/高]
申报建议：[立即申报/准备材料/暂不具备条件]
```

#### 申报时间规划

制作时间表，包含：
- 即将到期的申报机会（优先级最高）
- 年度固定申报窗口
- 需要提前准备的项目

## 综合分析板块

### SWOT分析框架

```
优势 (Strengths)
- 从财务指标中提取正向数据
- 企业资质与荣誉
- 行业地位与竞争优势

劣势 (Weaknesses)
- 财税诊断中发现的问题
- 资源配置不足的领域
- 合规性风险点

机会 (Opportunities)
- 可申报的政策与补贴
- 行业发展趋势
- 政策红利窗口期

威胁 (Threats)
- 政策变化风险
- 市场竞争压力
- 合规性风险
```

### 核心问题识别

按优先级列出发现的主要问题：

1. **紧急且重要**：需要立即处理的风险点
2. **重要但不紧急**：中长期改进事项
3. **紧急但不重要**：日常管理优化
4. **常规优化**：持续改进项

### 改进建议优先级

```
【高优先级】建议在1个月内完成
- [具体建议1]

【中优先级】建议在3个月内完成
- [具体建议1]
- [具体建议2]

【低优先级】建议在6个月内完成
- [具体建议1]
- [具体建议2]
```

- 金额：使用千分位分隔符，保留2位小数（如：1,234,567.89元）
- 百分比：保留2位小数（如：12.34%）

- 使用Markdown表格语法
- 表头行加粗
- 复杂表格可使用多级表头

```markdown
## 二、财务健康度分析

### 2.1 核心财务指标（2023年度）

| 指标类别 | 指标名称 | 数值 | 行业平均值 | 评价 |
|---------|---------|------|-----------|------|
| **盈利能力** | 营业收入 | 2,500万元 | - | - |
| | 净利润 | 350万元 | - | - |
| | 毛利率 | 45.2% | 40% | 优于行业 |
| | 净利率 | 14% | 10% | 优于行业 |
| **偿债能力** | 资产负债率 | 35% | 50% | 良好 |
| | 流动比率 | 2.5 | 1.5 | 良好 |
| **成长能力** | 营收增长率 | 25% | 15% | 优于行业 |
| | 利润增长率 | 30% | 12% | 优于行业 |

### 2.2 盈利能力分析

**亮点**：
- 毛利率45.2%，高于行业平均水平5.2个百分点，体现较强的产品竞争力和成本控制能力
- 净利率14%，盈利能力稳健，处于行业中上游水平

**关注点**：
- 销售费用占比较高，达到营收的15%，建议优化营销策略
- 应收账款余额较大，周转天数约90天，存在资金占用风险

### 2.3 风险提示

⚠️ **中风险**：应收账款周转天数较长
- 当前应收账款周转天数为90天，建议缩短至60天以内
- 建议完善信用管理制度，加强账款催收

💡 **低风险**：现金流波动
- 季度间现金流波动较大，建议建立现金储备机制
```

### 综合诊断结论示例

```markdown
## 五、综合诊断结论与建议

### 5.1 SWOT分析

**优势 (Strengths)**
- 高新技术企业资质完备，享受税收优惠政策
- 盈利能力优于行业平均水平，毛利率和净利率表现良好
- 技术团队实力较强，研发人员占比超过20%
- 轻资产运营模式，资产负债率低

**劣势 (Weaknesses)**
- 应收账款周转较慢，资金占用较大
- 客户集中度较高，存在依赖风险
- 品牌知名度有待提升

**机会 (Opportunities)**
- 贵州省大力支持大数据产业发展，政策红利丰富
- 可申报省级科技计划项目，预计获得100-200万元资金支持
- 数字化转型需求旺盛，市场空间广阔

**威胁 (Threats)**
- 行业竞争加剧，新进入者增多
- 政策变化可能影响税收优惠的持续性
- 人才流失风险

### 5.2 核心问题识别

| 优先级 | 问题类别 | 具体问题 | 影响程度 |
|--------|---------|---------|---------|
| 🔴 高 | 资金管理 | 应收账款周转慢（90天） | 中 |
| 🔴 高 | 政策红利 | 未充分申报补贴项目 | 高 |
| 🟡 中 | 市场风险 | 客户集中度较高 | 中 |
| 🟡 中 | 成本控制 | 销售费用占比较高 | 低 |

### 5.3 改进建议

#### 【高优先级】建议在1个月内完成

1. **启动科技计划项目申报**
   - 成立申报工作组
   - 准备申报材料
   - 预计获得100-200万元资金支持

2. **优化应收账款管理**
   - 建立客户信用评级制度
   - 缩短回款周期至60天以内
   - 制定账款催收机制

#### 【中优先级】建议在3个月内完成

1. **客户结构优化**
   - 拓展新客户，降低客户集中度
   - 建立客户分类管理机制

2. **成本优化**
   - 优化营销策略，降低销售费用占比
   - 提升营销转化率

#### 【低优先级】建议在6个月内完成

1. **品牌建设**
   - 加强市场宣传
   - 参与行业活动提升知名度

2. **人才储备**
   - 完善激励机制
   - 建立人才梯队
```
