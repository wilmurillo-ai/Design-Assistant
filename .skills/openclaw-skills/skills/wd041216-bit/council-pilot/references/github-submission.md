# GitHub Submission

The final phase of the autonomous pipeline. Submits the completed, maturity-scored artifact to GitHub.

## Submission Gate

Before submission, ALL of these must be true:

- [ ] Maturity score = 100 (breadth=25, depth=25, thickness=25, effectiveness=25)
- [ ] Verification loop all PASS (build, types, lint, tests, security, diff)
- [ ] No coverage gaps flagged
- [ ] Council consensus on submission readiness
- [ ] `pipeline_state.json` status = `converged`

## Branch Naming

```
council-pilot/<domain_id>
```

Example: `council-pilot/ai-reliability`

If the branch already exists, append the iteration number:
```
council-pilot/ai-reliability-v2
```

## Commit Message Format

```
feat(council-pilot): <domain> maturity 100/100

Breadth: 25/25 | Depth: 25/25 | Thickness: 25/25 | Effectiveness: 25/25
Expert council: <council_name> (<expert_count> experts)
Iterations: <iteration_count>
Expert forum: <forum_root_path>
```

Example:
```
feat(council-pilot): ai-reliability maturity 100/100

Breadth: 25/25 | Depth: 25/25 | Thickness: 25/25 | Effectiveness: 25/25
Expert council: ai-reliability-main (5 experts)
Iterations: 7
Expert forum: knowledge/expert_forums
```

## Commit Contents

The commit includes:

1. **Project source code** — all generated files
2. **Expert forum data** — the full forum root directory:
   - `forum_index.json`
   - `domains/` — domain definitions
   - `experts/` — all expert profiles and distillates
   - `councils/` — council definitions
   - `scoring_reports/` — all scoring history
3. **Pipeline state** — `pipeline_state.json` with final status
4. **Build logs** — key build logs from iterations
5. **Maturity report** — final comprehensive report

Files to EXCLUDE from commit:
- `.env` / `.env.*` — environment secrets
- `__pycache__/` — Python cache
- `node_modules/` — dependencies
- `candidates/` — intermediate candidate files (optional)
- `source_dossiers/` — raw source data (optional, may contain large content)
- `promotion_audits/` — intermediate audit files (optional)

## CI Integration

After PR creation:
1. Wait for CI checks to complete (if configured)
2. If CI fails, treat as a DEBUG failure and re-enter the loop
3. If CI passes, the pipeline is complete

## Post-Submission

After successful PR:
1. Update `pipeline_state.json` status to `submitted`
2. Record PR URL in pipeline state
3. Generate final maturity report
4. Print summary:
   - PR URL
   - Total iterations
   - Final scores per axis
   - Council composition
   - Experts added mid-loop
   - Build failures encountered
   - Total pipeline duration
