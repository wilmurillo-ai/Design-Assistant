# Security Policy

If you find a bypass, dangerous false negative, or a rule that may expose secrets or private data, please report it privately before opening a public issue.

## Please report

- bypasses for blocked-command or protected-path rules
- prompt-injection patterns that evade current detection
- redaction failures for secret-like strings
- audit logging issues that hide important events

Do not post working secrets, tokens, or private keys in public issues.
