# 📊 市场调研示例 - 竞品分析

> 使用晴晴自动激活 + 多源搜索 + 数据摘要

---

## 🎯 场景

你想分析竞品：
- 产品功能对比
- 定价策略分析
- 用户反馈整合
- 市场趋势调研
- 竞争优势识别

---

## 🚀 晴晴自动激活流程

### 步骤 1：一句话启动

```bash
# 直接说需求
"帮我分析一下竞品"

# 晴晴自动识别：市场调研场景
# 自动激活技能组合:
# 1. multi-search-engine (多引擎搜索)
# 2. playwright (网页数据采集)
# 3. summarize (报告摘要)
# 4. trend-researcher (趋势分析)
# 5. feedback-synthesizer (反馈整合)
```

---

### 步骤 2：多源搜索 (自动)

```
multi-search-engine 自动搜索:
├─ Google: 竞品官方网站、产品介绍
├─ Bing: 行业报告、市场分析
├─ 百度: 中文资料、用户评价
├─ 知乎: 用户讨论、专业分析
└─ 小红书: 用户体验、真实反馈

搜索范围：2026 年、最近 3 个月、中国 + 全球
```

---

### 步骤 3：数据采集 (自动)

```
playwright 自动抓取:
├─ 竞品网站：功能列表、定价页面
├─ 电商平台：销量、评价
├─ 社交媒体：用户反馈、口碑
└─ 行业报告：市场规模、趋势

采集深度：3 层链接 (首页 → 产品页 → 详情页)
```

---

### 步骤 4：报告摘要 (自动)

```
summarize 自动生成报告:
├─ 产品功能对比表
├─ 定价策略分析
├─ 用户反馈整合
├─ 市场份额估算
└─ 竞争优势识别

报告长度：3,000-5,000 字
```

---

### 步骤 5：趋势分析 (自动)

```
trend-researcher 分析:
├─ 行业趋势 (2026 年 AI、电商、SaaS)
├─ 技术趋势 (React、Node.js、AI 集成)
├─ 市场趋势 (增长、竞争、机会)
└─ 用户趋势 (需求、痛点、期望)
```

---

### 步骤 6：反馈整合 (自动)

```
feedback-synthesizer 整合:
├─ 正面反馈 (功能好、体验佳)
├─ 负面反馈 (价格高、功能少)
├─ 中性反馈 (建议、期望)
└─ 优先级排序 (高频 → 低频)
```

---

## 📋 完整命令示例

### 自动激活 (推荐)

```bash
# 一句话启动全流程
"帮我分析一下竞品，包括产品功能、定价策略、用户反馈，重点关注 2026 年 AI 电商趋势"

# 晴晴自动执行:
# 1. 识别：市场调研场景
# 2. 激活：multi-search-engine + playwright + summarize
# 3. 搜索：多引擎交叉验证
# 4. 采集：网页数据抓取
# 5. 分析：趋势 + 反馈整合
# 6. 交付：完整调研报告
```

### 手动调用 (可选)

```bash
# 多源搜索
/multi-search-engine "搜索 2026 年 AI 电商竞品，包括 Shopify、WooCommerce、Magento"

# 数据采集
/playwright "抓取竞品网站功能列表和定价页面"

# 报告摘要
/summarize "生成竞品分析报告，包括功能对比、定价分析、用户反馈"

# 趋势分析
/trend-researcher "分析 2026 年 AI 电商趋势"

# 反馈整合
/feedback-synthesizer "整合用户反馈，识别高频痛点和机会"
```
