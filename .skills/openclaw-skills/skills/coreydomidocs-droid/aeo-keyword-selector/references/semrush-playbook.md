# Semrush playbook

## Role of semrush
Semrush is the primary validation and support tool, not the automatic starting point for dense clusters.

For dense topic clusters, use semrush only after reviewing existing site coverage and current page intent.
In those cases, semrush should help confirm whether a proposed gap is real, distinct, and worth separating from existing URLs.

For broader or less-developed topic areas, semrush may still be used earlier to help identify viable search demand.

## What semrush is good for here
- validating whether a candidate topic or question has real search behavior behind it
- checking whether a proposed gap is distinct enough to justify separation
- finding support for question-style phrasing after architecture review
- sanity checks for crowded query spaces
- confirming that a topic family actually exists in search behavior

## What semrush should not do here
- drive adjacent query exploration in dense clusters
- create multiple near-duplicate candidate paths from slight phrasing shifts
- override existing URL ownership of primary intent
- manufacture a case for `new-url` when `expand-existing` or `no-go` is stronger

## What semrush should not decide alone
- whether a zero-volume long-tail question is still worth creating for ai-native search behavior
- whether an existing site url already owns the primary ranking intent
- whether the right action is `new-url` versus `expand-existing`

## Zero-volume ai-native exception
A zero-volume keyword may be selected only when all are true:
1. it is naturally askable in long-form question language
2. it is a logical derivative of a semrush-supported topic cluster
3. browser SERP inspection suggests that people also ask / ai-overview style phrasing supports the question pattern
4. fetched-url review shows no direct cannibalization risk

Treat this as an exception, not the default mode.

## Browser usage
Browser SERP inspection is allowed as a supporting method to:
- inspect people also ask
- inspect visible question phrasing in results
- judge whether a crowded query space has narrower question angles worth targeting

Do not let browser SERP inspection override architecture review or create a false case for a new URL.
