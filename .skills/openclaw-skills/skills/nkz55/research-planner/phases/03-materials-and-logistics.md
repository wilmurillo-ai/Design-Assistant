# Phase 3: Research Materials and Logistics

Goal of this phase: translate the approved research plan into **ready‑to‑use materials and an operational checklist**.

Inputs:

- Approved research plan from Phase 2

Outputs:

- Drafts of all required research materials
- A logistics plan for running the study

---

## 1. Map Plan Components to Materials

From the research plan, list which materials are required. Typical items:

- Recruitment screener
- Participant invitation and consent wording
- Interview / discussion guide
- Usability test script and task list
- Survey questionnaire
- Observer / note‑taking templates

**Output (example):**

```markdown
### Required Materials
- Recruitment screener: Yes
- Invitation & consent: Yes
- Interview guide: Yes
- Usability test script: No
- Survey questionnaire: No
- Observation / note template: Yes
```

Use the templates in the `templates/` directory as the starting point for each material.

---

## 2. Recruitment and Consent Materials

### 2.1 Recruitment Screener

Use `templates/screener-and-survey-template.md` as a base and fill in:

- Study title and short description
- Eligibility questions mapped to inclusion / exclusion criteria
- Disqualification rules
- Minimal demographic / firmographic questions needed for quotas

Ensure:

- Wording is clear and non‑leading
- Logic (who qualifies / disqualifies) is explicit

### 2.2 Invitation and Consent

Draft:

- A concise email / message invitation
- Consent wording that covers:
  - Purpose of the study
  - What participation involves (time, activities, recording)
  - Incentive and payment, if any
  - Data handling and privacy
  - Voluntary participation and right to withdraw

Flag where the user’s legal / compliance team should review.

---

## 3. Session / Study Instruments

### 3.1 Interview / Discussion Guide

Use `templates/interview-guide-template.md` as a scaffold and adapt it to:

- The research objectives and key questions
- The target user profile and scenario

Ensure:

- Flow roughly follows: intro → warm‑up → core themes → wrap‑up
- Questions are open‑ended and non‑leading
- There is space for probes and follow‑ups

### 3.2 Usability Test Script

Use `templates/usability-test-script-template.md` as a base and:

- Define realistic, user‑centric task scenarios
- For each task, define success criteria and failure indicators
- Include moderator instructions and observation notes space

### 3.3 Survey Questionnaire

If a survey is part of the design, expand the survey section in `templates/screener-and-survey-template.md`:

- One section per construct or topic
- Clear question wording, consistent scales
- Any skip logic described in plain language

---

## 4. Logistics: Timeline, Roles, and Checklist

Turn the plan into an **operational schedule**.

### 4.1 Timeline

Break down:

- Recruitment start and end dates
- Fieldwork window (interviews, tests, survey live dates)
- Debrief or interim check‑ins (if any)

**Output (example):**

```markdown
### Timeline
- Week 1: Finalize materials, kick off recruitment
- Week 2–3: Conduct interviews (N=12)
- Week 4: Debrief sessions and hand‑off to analysis skill
```

### 4.2 Roles and Responsibilities

Define:

- Moderator(s)
- Note‑taker(s) / observer(s)
- Logistics / scheduling owner
- Stakeholders invited to observe (if applicable)

### 4.3 Operational Checklist

Create a simple checklist to confirm readiness:

```markdown
### Operational Checklist
- [ ] Recruitment screener finalized and implemented
- [ ] Invitation and consent text approved
- [ ] Interview / test / survey instrument finalized
- [ ] Scheduling tool or process set up
- [ ] Recording and note‑taking tools tested
- [ ] Incentive process confirmed
```

---

## 5. Handoff to Fieldwork and Analysis Skills

Once materials and logistics are confirmed:

- Clearly indicate that the next phases (fieldwork, analysis, reporting) are **outside this planning skill’s scope**
- Provide a concise package (plan + materials + timeline + checklist) that can be used directly by:
  - Users running the study themselves, and/or
  - Other skills focused on data collection and analysis
