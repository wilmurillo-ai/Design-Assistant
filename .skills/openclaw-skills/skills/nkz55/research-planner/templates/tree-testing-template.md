# Tree Testing Template

Template for planning tree testing to evaluate information architecture (IA) structures separate from visual design.

## How to Use

- Use after card sorting when you need to **validate** a proposed IA or navigation scheme.
- Test the **text-only hierarchy** (“tree”) without visual design to isolate structural issues.

## Tree Testing Template

```markdown
# Tree Test – [Site / Product IA]

Version: v[...]
Date: [...]
Owner: [...]

## 1. Objectives

- What navigation or findability questions should this tree test answer?
  - [...]
- Which parts of the IA are in focus (e.g., certain sections, deep levels)?
  - [...]

## 2. Tree Structure

Provide a text-only representation of the IA levels to be tested.

Example format:

- Home
  - Products
    - Product A
    - Product B
  - Support
    - FAQs
    - Contact us

Note any differences from the live site if this is a prototype tree.

## 3. Tasks

Write realistic, user-centered tasks that require finding specific items in the tree.

For each task:

- Task ID:
- Task wording (what the user sees):
  - E.g., “You want to find how to change your billing address. Where would you go?”
- Correct answers (paths or endpoints in the tree):
- Success definition:
  - [e.g., “participant reaches any of these nodes without backtracking more than X levels”]

## 4. Participants

- Target users/segments:
  - [...]
- Sample size:
  - [e.g., 20–50, depending on segments and complexity]
- Any quotas:
  - [...]

## 5. Logistics

- Tool/platform:
  - [e.g., dedicated tree-testing tool, simple survey tool with branching]
- Mode:
  - [remote unmoderated / remote moderated / in-person]
- Estimated time per participant:
  - [e.g., 10–20 minutes]
```
