## 3. Introduction and Ground Rules (Script)

```markdown
Hi [participant name], thanks for joining today.

My name is [...], and I’ll be guiding you through this session.

**Purpose**
Today, we’re going to look at [product / prototype] and see how easy or difficult it is to use for tasks like [high‑level tasks]. We’re testing the product, **not** you. There are no right or wrong answers.

**Recording**
With your permission, we’ll record the session so our team can review it later. The recording will only be used internally for research.

**Think‑aloud**
It’s very helpful if you can think aloud—tell me what you’re trying to do, what you expect to happen, and anything that is confusing or surprising.

If at any point you feel uncomfortable or want to stop, just let me know. You can skip any question or task.

Do you have any questions before we start?
```

## 4. Background Questions (Short)

Keep background brief—focus on what matters for interpreting usability findings.

## 5. Task Scenarios

Present tasks as **realistic scenarios** in user language. For each task, define:

- Scenario text (what you read to the participant)
- Starting point (what they see at the beginning)
- Success criteria
- Observations and notes

### Task 1 – [Short task name]

```markdown
**Scenario (read aloud)**
You are [...]. You want to [...]. Using [product], please show me how you would do this.

**Starting point**
- Screen / state: [...]

**Success criteria**
- Participant completes [goal] without assistance
- Key actions: [...]

**Moderator guidance**
- Allow silence and exploration
- If participant is stuck for more than [X] seconds, offer a neutral prompt: "What would you expect to do next?"

**Observation notes**
- Path taken:
- Errors or confusions:
- Verbal comments:
```

---

## 6. Post‑Task / Post‑Study Questions

Ask a few reflective questions after tasks, while the experience is fresh.

```markdown
### Post‑Study Questions
1. Overall, how easy or difficult was it to complete these tasks?
   - Probe: What made it easy? What made it difficult?

2. What, if anything, did you like about using [product / prototype]?

3. What, if anything, was confusing or frustrating?

4. If you could change one thing to improve this experience, what would it be?
```

Optionally, include a short rating question (e.g., overall satisfaction or likelihood to use), if aligned with the research plan.
