# Card Sorting Template

Template for planning and documenting open, closed, or hybrid card-sorting studies used to understand users’ mental models and inform information architecture.

## How to Use

- Use early in IA design to explore how users group concepts and label categories.
- Choose **open** card sorting to discover users’ own labels; choose **closed** to validate an existing category scheme.

## Card Sorting Study Template

```markdown
# Card Sorting Study – [Project / IA Area]

Version: v[...]
Date: [...]
Owner: [...]

## 1. Objectives

- What do we want to learn from this card sort?
  - E.g., “Understand how users group content for [site/app area]”
  - E.g., “Validate whether the proposed navigation labels make sense”
- How will results influence IA, navigation, or labeling decisions?
  - E.g., “Update top-level navigation”, “Rename key categories”, “Inform future tree testing”

## 2. Scope

- Content / items included:
  - List of pages, features, topics, or content snippets to represent on cards.
- Out-of-scope items (if any):
  - Content we deliberately exclude or defer to future rounds.

## 3. Method

- Type of card sort:
  - [ ] Open (participants create their own groups and labels)
  - [ ] Closed (participants sort items into pre-defined categories)
  - [ ] Hybrid (pre-defined categories + ability to create new ones)
- Format:
  - [ ] Remote – online card sorting tool
  - [ ] In-person – physical cards
- Session style:
  - [ ] Individual sessions
  - [ ] Group sessions

## 4. Participants

- Target users / segments:
- Number of participants:
  - Aim for: [e.g., 15–30 participants, depending on variety of content and segments]
- Any quotas (e.g., by segment, experience level, market):

## 5. Materials

- Card list:
  - Card ID / label / description:
  - Example:
    - C01 – “Billing & invoices” – “All content related to payments and invoices”
- For closed sorts: category labels and descriptions:
  - Category ID / label / description:

## 6. Procedure

### 6.1 Participant Instructions

What participants see at the start:

- Brief explanation of purpose (high-level, not leading).
- Clear instructions for how to sort:
  - For open sorts: “Group cards in a way that makes sense to you, then name each group.”
  - For closed sorts: “Place each card into one of the given categories; if it doesn’t fit, use [‘Other’ / ‘Does not fit’].”

### 6.2 Example / Practice Round (Optional)

- Short practice with a few neutral items, if needed.

### 6.3 Sorting Rules

- Clarify:
  - Whether cards can belong to more than one group.
  - How to handle “doesn’t fit” or “I don’t know” items.

## 7. Data Capture and Analysis (High Level)

- How results will be captured:
  - Online tool exports (including similarity matrices, dendrograms, etc.)
  - Photos of physical card layouts
- How results will be analyzed (summary level):
  - Identify common groupings and labels
  - Note disagreements or ambiguous items
  - Propose candidate IA structures based on clusters
```

## Key References for Card Sorting

- Donna Spencer, *Card Sorting: Designing Usable Categories* – foundational book on card-sorting technique and analysis.
- Nielsen Norman Group, “Card Sorting: How Many Users to Test” and related articles (`https://www.nngroup.com/articles/` – search for “card sorting”).
- User Interviews and similar UX research field guides on card sorting as a discovery / IA method (`https://www.userinterviews.com/ux-research-field-guide-module/user-research-methods`).
