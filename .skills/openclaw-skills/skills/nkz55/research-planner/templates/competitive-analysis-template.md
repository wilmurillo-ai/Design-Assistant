# Competitive Analysis Template

Template for structured competitive and comparative analysis focused on user experience, value propositions, and market positioning.

## How to Use

- Use to capture **evidence-based** comparisons, not just opinions.
- Combine with user research where possible to avoid evaluating only from an internal perspective.

## Competitive / Comparative Analysis Template

```markdown
# Competitive / Comparative Analysis – [Product / Market]

Version: v[...]
Date: [...]
Owner: [...]

## 1. Objectives

- What questions should this analysis answer?
  - E.g., “How do competitors onboard new users compared to us?”
  - E.g., “How do pricing and packaging differ across key competitors?”
- How will findings affect strategy, design, or roadmap?
  - E.g., “Identify table-stakes features vs. differentiators”, “Inform navigation redesign”, “Support positioning decisions”.

## 2. Competitor / Comparator Set

- Direct competitors:
  - [...]
- Indirect or alternative solutions (including “do nothing” and workarounds):
  - [...]
- Internal benchmarks (if any):
  - [...]

## 3. Dimensions of Comparison

For each competitor, assess along key dimensions (columns in a comparison table).

- Target users / segments:
  - Who they seem to target (based on messaging, features, case studies).
- Value proposition and positioning:
  - What problems they claim to solve; how they differentiate.
- Key features and workflows:
  - Core capabilities; completeness of workflows; integrations.
- Pricing and packaging:
  - Pricing model (subscription, freemium, usage-based); tiers; limitations.
- UX quality:
  - Onboarding:
  - Navigation and information architecture:
  - Visual design and content clarity:
  - Responsiveness and performance:

## 4. Evaluation Framework

- Criteria and rating scale:
  - Define explicit criteria (e.g., “ease of starting a trial”, “clarity of dashboard”, “support discoverability”).
  - Use a simple scale (e.g., 1–5; 1=poor, 3=acceptable, 5=excellent) and **define what each level means**.
- Evidence sources:
  - Hands-on product trials:
  - Documentation and help content:
  - Marketing websites and demos:
  - Customer reviews, case studies, or public feedback (if relevant).

Consider capturing results in a table:

- Rows: competitors + our product.
- Columns: criteria + qualitative notes.

## 5. Findings Summary

Summarize key observations:

- For each competitor:
  - Strengths:
  - Weaknesses:
  - Notable UX patterns or innovations:
- For the category overall:
  - Common patterns and “expected” behaviors (table stakes):
  - Areas where most competitors underperform:

## 6. Implications

- Design implications:
  - UX improvements to consider:
  - Patterns to learn from vs. patterns to avoid:
- Product / strategy implications:
  - Differentiation opportunities:
  - Features or areas where we must “catch up”:
```

## Key References for Competitive Analysis

- Michael E. Porter, *Competitive Strategy* – high-level frameworks for industry and competitor analysis.
- Marty Cagan, *Inspired* – product-centric view of understanding markets and competitors.
- Nielsen Norman Group and various UX field guides on UX competitive reviews and teardown methodologies (`https://www.nngroup.com/articles/` – search for “competitive analysis”, “competitive usability evaluations”).
