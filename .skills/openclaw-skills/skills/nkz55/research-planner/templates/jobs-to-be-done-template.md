# Jobs to Be Done (JTBD) Template

Template for capturing user “jobs,” pains, and gains using the Jobs to Be Done framework.

## How to Use

- Use when you need to understand **why** users “hire” a product or service in a given situation.
- Focus on situations and desired outcomes, not demographics or product features.

## Key References for Jobs to Be Done

- Clayton M. Christensen et al., *Competing Against Luck* – foundational JTBD theory and case studies.
- Tony Ulwick, *What Customers Want* – outcome-driven innovation framing related to JTBD.
- Various product and UX strategy resources summarizing JTBD patterns and job statements (e.g., Reforge, Intercom on Product Management).
