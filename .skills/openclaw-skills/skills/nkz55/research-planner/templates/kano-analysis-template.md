# Kano Analysis Template

Template for planning and documenting Kano-model-based analysis of feature satisfaction and expectations.

## How to Use

*- Use Kano when you need to understand how different features affect satisfaction (basic vs. performance vs. delight).*
- Combine with other prioritization methods (e.g., effort/cost estimates).

## Kano Analysis Template

```markdown
# Kano Analysis – [Product / Feature Set]

Version: v[...]
Date: [...]
Owner: [...]

## 1. Objectives

- Why use Kano for this set of features?
  - [...]
- What decisions will it inform (e.g., prioritization, roadmap, MVP scoping)?
  - [...]

## 2. Features / Attributes

- List of features or attributes to evaluate:
  - Feature 1: [...]
  - Feature 2: [...]
  - Feature 3: [...]

## 3. Kano Questions (per feature)

For each feature, define:

- Functional form (feature is present):
  - "If [feature] is available, how do you feel?"
- Dysfunctional form (feature is absent):
  - "If [feature] is not available, how do you feel?"

## 4. Response Scales

Use standard Kano response options:

- I like it that way.
- It must be that way.
- I am neutral.
- I can live with it that way.
- I dislike it that way.
```
