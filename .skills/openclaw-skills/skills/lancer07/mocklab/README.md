# MockLab - 智能接口工具箱

MockLab 是一个智能接口工具箱，包含AI Skill 和 Mock Server，目标： 接口开发调试一站式解决。
AI skill 可以将任意格式的接口文档整理成JSON Schema
Mock Server 支持 Mock 数据生成、请求转发、数据复现

---

## 解决什么问题

| 角色 | 痛点 | MockLab 如何解决 |
|------|------|-----------------|
| 前端开发 | 后端接口还没 ready，只能干等 | 用 Mock 数据独立调试，不依赖后端 |
| 后端开发 | 调本地接口只能写代码或用 Postman | UI 可视化，一键转发到 localhost |
| 测试工程师 | 造测试数据费时，不真实 | AI 自动生成合规数据（身份证、手机号等）|
| 第三方联调 | 对方测试环境迟迟不就绪 | 拿到文档 5 分钟上线，不用等 |
| 团队协作 | Schema 无法共享，各改各的 | 一键导入导出，分享给队友 |
| 问题复现 | 随机数据每次不同，无法复现 | Seed 固定化，相同数据可重现 |

## 核心功能

### Mock 数据生成
AI 自动解析接口文档，生成合规 Mock 数据：

| 字段类型 | 生成结果 |
|---------|---------|
| 手机号 | 真实号段（130～199）|
| 身份证 | 含校验位的 18 位号码 |
| 车牌号 | 合规省份+车牌格式 |
| 金额 | 指定范围随机金额 |
| 利率 | 指定范围年利率 |
| 枚举 | 随机取值 |
| 订单号 | 日期+自增序号 |

### 请求代理转发
后端开发人员的利器：填写目标地址，一键转发到真实接口，原样返回响应。

### 状态串联
自动管理 token、订单号等跨接口状态，不用手工填写。

### 延迟 & 错误注入
- **延迟**：模拟网络延迟，测试弱网场景
- **错误**：模拟 500/429/超时等异常，测试异常处理

### 自定义请求头
发送请求时添加 Authorization 等自定义头部。

### 必填校验
发送前检查必填字段，未填写弹窗提示。

### 导入/导出
- **导出**：下载当前项目 Schema JSON 文件
- **导入**：上传 Schema JSON，一键加载

### Seed 固定化
设置固定 Seed，生成相同数据，方便问题复现和对比测试。

## 快速开始

```
你：  启动 MockLab，这是接口文档 [粘贴 / 上传文件 / 给 URL]
AI：  解析文档 → 生成 Schema → 启动服务
你：  打开 http://localhost:18080，开始使用
```

## 规则引擎

### 基础规则

| 规则 | 生成内容 |
|------|---------|
| `phone` | 11位真实号段手机号 |
| `idcard` | 18位合法身份证 |
| `name` | 中文姓名 |
| `plate` | 合规车牌号 |
| `amount:lo-hi` | 指定范围金额 |
| `rate:lo-hi` | 指定范围利率 |
| `random:6` | 6位随机数字 |
| `timestamp:ms` | 毫秒时间戳 |
| `token:32` | 32位随机字符串 |
| `fixed:VALUE` | 固定常量 |
| `enum:a,b,c` | 随机枚举值 |
| `sequence:key` | 自增序列号 |
| `skip` | 不生成（sign 等）|
| `nested:xxx` | 嵌套 JSON 对象 |
| `array:xxx` | 对象数组 |

### 动态响应规则

| 规则 | 说明 |
|------|------|
| `mirror` | 取请求 body 中同名字段 |
| `body:data.order_id` | 取请求 body 嵌套字段 |
| `path:order_id` | 取 URL 路径参数 |
| `query:page` | 取 URL query 参数 |
| `header:Authorization` | 取请求头 |
| `state:token` | 取跨次调用状态 |
| `when` 条件 | 满足条件才返回 |
