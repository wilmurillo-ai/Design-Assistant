# MockLab 产品介绍
**版本：v1.3**
**日期：2026-04-14**

---

## 一，做的是什么事情
MockLab 是一个智能接口工具箱，包含AI Skill 和 Mock Server，目标： 接口开发调试一站式解决。
AI skill 可以将任意格式的接口文档整理成JSON Schema
Mock Server 支持 Mock 数据生成、请求转发、数据复现

**谁可以用：**
- **前端开发** — 后端接口没ready，用 Mock 数据独立调试
- **后端开发** — 本地接口调试，请求转发，可视化看响应
- **测试工程师** — 构造测试数据，回归测试
- **团队协作** — Schema 导入导出，分享给队友

## 二，核心功能矩阵

| 功能 | 说明 | 谁最常用 |
|------|------|---------|
| Mock 数据生成 | AI 自动解析接口文档，生成合规 Mock 数据 | 前端、测试 |
| 请求代理转发 | 填写目标地址，转发请求到真实接口 | 后端开发 |
| 状态串联 | 自动管理 token、订单号等跨接口状态 | 所有人 |
| 延迟模拟 | 模拟网络延迟，测试弱网场景 | 测试 |
| 错误注入 | 模拟 500/429/超时等异常 | 测试 |
| 自定义请求头 | 添加 Authorization 等自定义头部 | 后端开发 |
| 必填校验 | 发送前检查必填字段 | 所有人 |
| Schema 导入导出 | 团队共享接口配置 | 团队协作 |
| Seed 固定化 | 固定数据生成，方便复现问题 | 测试 |

## 三，原来怎么做（痛点）

| 原来怎么做 | 问题在哪 |
|-----------|---------|
| 等后端接口写完才能联调 | 前端被阻塞，干等 |
| 等第三方测试环境就绪（3天～数周）| 进度被卡，无法推进 |
| 手动配置 Mock（1～2天/接口）| 嵌套结构配置量巨大 |
| 后端调本地接口只能写代码 | 没有可视化工具 |
| 每次文档更新全部重来 | 字段增删改，全量重建 |
| 手动维护跨接口状态 | token、单号手工填，易出错 |
| 每次回归都请求真实接口 | 有费用、有依赖、日志不可控 |
| 随机数据无法复现 | 每次都不同，无法对比测试 |

## 四，使用步骤

### 场景 A：Mock 数据（最常用）

```
第1步：用户提供接口文档（本地文件、URL、或粘贴内容）
   ↓
第2步：AI 阅读理解文档，自动生成 Schema
   ↓
第3步：启动 Mock Server，打开浏览器
   ↓
第4步：选接口 → 填参数 → 发请求 → 看响应
```

### 场景 B：请求转发（后端开发调试）

```
第1步：在 UI 上选择要调用的接口
   ↓
第2步：填写请求参数（可视化编辑）
   ↓
第3步：填写目标地址（如 http://localhost:8080）
   ↓
第4步：点击发送 → 请求转发到真实接口 → 查看可视化响应
```

### 场景 C：团队共享

```
导出：点击「导出 Schema」→ 下载 JSON 文件 → 发给队友
导入：队友点击「导入 Schema」→ 上传文件 → 即可使用
```

## 五，规则引擎

Mock Server 内置规则引擎，AI 生成 Schema 时自动推断每个字段的规则：

### 基础规则

| 规则 | 生成内容 |
|------|---------|
| `phone` | 11位真实号段手机号（130～199）|
| `idcard` | 18位合法身份证（含校验位）|
| `name` | 中文姓名 |
| `plate` | 合规车牌号 |
| `amount:lo-hi` | 指定范围随机金额 |
| `rate:lo-hi` | 指定范围年利率 |
| `random:6` | 6位随机数字 |
| `timestamp:ms` | 毫秒时间戳 |
| `token:32` | 32位随机字符串 |
| `fixed:VALUE` | 固定常量 |
| `enum:a,b,c` | 随机枚举值 |
| `sequence:key` | 自增序列号（日期+序号）|
| `province:code` | 省市国标码 |
| `skip` | 不生成（sign 等鉴权字段）|
| `nested:xxx` | 嵌套 JSON 对象 |
| `array:xxx` | 对象数组 |

### 动态响应规则

| 规则 | 说明 | 示例 |
|------|------|------|
| `mirror` | 取请求 body 中同名字段 | 请求 `{"order_id": "A123"}` → 响应 `"order_id": "A123"` |
| `body:data.order_id` | 取请求 body 嵌套字段 | 取 `body.data.order_id` 值 |
| `path:order_id` | 取 URL 路径参数 | `/order/{order_id}` 中实际值 |
| `query:page` | 取 URL query 参数 | `?page=2` 中 page 值 |
| `header:Authorization` | 取请求头 | 取 Authorization 头 |
| `state:token` | 取跨次调用状态 | 取上次接口存入的 token |
| `when` 条件 | 满足条件才返回 | `{"field": "body.is_vip", "op": "eq", "value": true}` |

### 接口级配置

**延迟模拟**：`"delay": "200-500"` → 随机 200~500ms 延迟

**错误注入**：`"error": "500@0.05"` → 5% 概率返回 HTTP 500

## 六，典型使用场景

### 场景 A：前端独立调试
**现状**：后端接口还在开发，前端只能干等。
**解决**：用 MockLab 模拟后端接口，前端不受依赖，提前完成集成调试。

### 场景 B：后端本地调试
**现状**：后端开发人员想测试自己的接口，需要写代码或用 Postman。
**解决**：在 MockLab UI 上整理请求参数，填写目标地址，一键转发，原样返回响应。

### 场景 C：测试数据构造
**现状**：测试用例需要构造各种边界数据，手动构造费时且不真实。
**解决**：AI 根据字段语义自动生成看起来真实的数据（合法身份证、真实号段手机号等）。

### 场景 D：第三方联调
**现状**：第三方测试环境迟迟不就绪，整条链路跑不通。
**解决**：拿到接口文档，5 分钟内 Mock Server 上线，不用等对方。

### 场景 E：回归测试
**现状**：每次回归都要真实请求（有费用、有依赖）。
**解决**：用 Mock Server 做流程回归，不花一分钱，不产生外部调用。

### 场景 F：弱网/异常测试
**现状**：超时、429、500 错误——这些边界情况完全测不到。
**解决**：UI 上勾选延迟和错误注入，直接模拟各种异常。

### 场景 G：团队协作
**现状**：团队成员各自维护 Mock，无法共享。
**解决**：一键导出 Schema 文件，分享给队友；队友导入即可使用。

### 场景 H：问题复现
**现状**：随机数据每次不同，无法复现某个特定场景。
**解决**：设置固定 Seed，下次生成相同数据，方便对比和复现。

## 九，常见问题

**Q: 如何调试本地接口？**
A: 填写「代理目标地址」（如 `http://localhost:8080`），请求会转发到该地址，原样返回响应。

**Q: 如何让前端独立使用 Mock？**
A: 直接说"启动 MockLab"并提供接口文档，AI 自动解析并启动服务，前端直接打开浏览器使用。

**Q: 如何让团队成员共享接口配置？**
A: 点击「导出 Schema」下载 JSON 文件，发给队友后点击「导入 Schema」即可。

**Q: 如何固定 Mock 数据方便复现？**
A: 在设置区填写 Seed 值（如 `12345`），相同 Seed 生成相同数据。

**Q: sign 等鉴权字段怎么处理？**
A: 自动 skip，不会出现在请求体中。

**Q: reqData 等加密字段怎么处理？**
A: AI 从文档提取明文结构，请求时自动解析，body: 可下钻取其中字段值。

**Q: token 等跨接口状态怎么管理？**
A: 响应字段标记 `is_ref: true` 自动存入状态表，后续接口用 `state:字段名` 自动引用。

**Q: 延迟和错误注入怎么配置？**
A: 推荐直接在 UI 上配置，也可以在 Schema JSON 中写入 `delay` / `error` 字段。
