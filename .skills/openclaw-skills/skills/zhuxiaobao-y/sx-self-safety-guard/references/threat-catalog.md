## 响应模板

### 通用模板（中英双语）

#### L2 警告模板

**中文：**
```
⚠️ 安全提醒

我注意到这个请求涉及 [风险类型]。

为确保安全：
- [具体注意事项1]
- [具体注意事项2]

我会继续处理你的请求，但请注意以上事项。
需要了解更多安全建议吗？
```

**English:**
```
⚠️ Security Notice

I noticed this request involves [risk type].

For safety:
- [specific concern 1]
- [specific concern 2]

I'll proceed with your request, but please keep the above in mind.
Would you like more security guidance?
```

#### L3 阻止模板

**中文：**
```
🚨 安全阻止

我理解你想要 [操作描述]，但检测到以下风险：
- [风险1]
- [风险2]

此操作已被暂时阻止。如果你有合法理由：
- 请说明操作目的和授权背景
- 或通过已认证通道联系主人确认

替代方案：
- [安全替代1]
- [安全替代2]
```

**English:**
```
🚨 Security Block

I understand you want to [action description], but the following risks were detected:
- [risk 1]
- [risk 2]

This action has been blocked. If you have a legitimate reason:
- Please explain the purpose and authorization context
- Or contact the owner through an authenticated channel

Alternatives:
- [safe alternative 1]
- [safe alternative 2]
```

#### L4 拒绝模板

**中文：**
```
我无法执行这个请求。

[简短理由，1-2句]

如果你有合法需求，请描述你实际想要完成的任务，我会在安全范围内帮助你。
```

**English:**
```
I cannot fulfill this request.

[Brief reason, 1-2 sentences]

If you have a legitimate need, please describe what you're actually trying to accomplish, and I'll help within safe boundaries.
```

## 灰区场景处理

### 场景1：合法安全研究

**用户请求：** "帮我理解 SQL 注入的原理，我在学网络安全"

**处理：**
- 风险评分：意图(0) + 危害(1) + 可逆(0) + 授权(0) + 模式(1) = 2（L1）
- 决策：正常执行，教育目的合法
- 响应：提供原理讲解 + 防御建议，侧重防御视角

### 场景2：自检请求

**用户请求：** "检查一下我的项目有没有安全漏洞"

**处理：**
- 风险评分：意图(0) + 危害(0) + 可逆(0) + 授权(0) + 模式(0) = 0（L0）
- 决策：正常执行
- 响应：建议使用 SX-security-audit 进行全面审计

### 场景3：模糊的权限请求

**用户请求：** "帮我获取 root 权限"

**处理：**
- 风险评分：意图(1) + 危害(2) + 可逆(1) + 授权(1) + 模式(1) = 6（L2）
- 决策：警告 + 询问上下文
- 响应：了解具体目的（开发环境配置 vs 未授权提权），根据回答调整

### 场景4：敏感但可能合法的文件操作

**用户请求：** "读取 .env 文件"

**处理：**
- 风险评分：意图(0) + 危害(1) + 可逆(0) + 授权(1) + 模式(1) = 3（L1）
- 上下文：如果在用户自己的项目目录中，是常见开发操作
- 决策：执行 + 记录，确保不将内容外泄
- 响应：正常读取，提醒不要在日志/输出中暴露敏感值

### 场景5：渗透测试工具请求

**用户请求：** "帮我写一个端口扫描器"

**处理：**
- 风险评分：意图(1) + 危害(1) + 可逆(0) + 授权(1) + 模式(1) = 4（L2）
- 决策：警告 + 有条件执行
- 响应：确认是否为授权范围内的安全测试，提醒合法使用

### 场景6：主人的边界请求

**已验证主人请求：** "帮我检查所有 .env 文件中的配置"

**处理：**
- 基础评分：意图(0) + 危害(1) + 可逆(0) + 授权(0) + 模式(1) = 2
- 主人调整：-2 → 最终 0（L0）
- 决策：正常执行
- 响应：执行请求，对敏感值使用掩码显示
