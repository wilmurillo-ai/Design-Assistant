# Copywriting Reference — Funnel Revenue Engine

Proven formulas and frameworks the agent uses when writing
landing pages, email sequences, and offer copy.
Apply these to all `[PLACEHOLDERS]` in the templates.

---

## THE GOLDEN RULE

**One page. One offer. One action.**

Every element on the landing page must push toward the single CTA.
If something doesn't support the conversion — remove it.

---

## HEADLINE FORMULAS

### Formula 1 — Result + ICP + Without Pain
```
"[ACHIEVE RESULT] for [ICP] without [OBSTACLE]"

Examples:
"Close 3x more B2B deals without a sales team"
"Grow your newsletter to 10K subscribers without paid ads"
"Trade crypto profitably without spending hours on charts"
```

### Formula 2 — How To
```
"How to [DESIRED RESULT] in [TIMEFRAME] even if [OBJECTION]"

Examples:
"How to generate €1,000 MRR in 30 days even if you're starting from zero"
"How to land 3 B2B clients this month even if you hate cold email"
```

### Formula 3 — The Specific Number
```
"[SPECIFIC RESULT] in [TIMEFRAME]"

Examples:
"+€2,847 in trading profit last month"
"47 new subscribers in 7 days"
"3 paying clients in the first week"
```

### Formula 4 — The Question
```
"[QUESTION that targets their biggest frustration]?"

Examples:
"Why is your pipeline full of leads that never convert?"
"What if you never had to do cold outreach again?"
```

## THE 3 COPY STRUCTURES

### 1. PAS — Problem / Agitate / Solution

Best for: landing pages, email subject lines, LinkedIn posts

```
P — PROBLEM: Name the exact pain the ICP feels
"You spend 3 hours a day on manual prospecting."

A — AGITATE: Make them feel the cost of staying in this situation
"That's 15 hours a week. 60 hours a month. 720 hours a year.
 All on tasks that generate zero revenue."

S — SOLUTION: Present your offer as the way out
"[PRODUCT] automates it in 15 minutes. You do one setup.
 It runs every day."
```

---

### 2. AIDA — Attention / Interest / Desire / Action

Best for: email sequences, longer landing page sections

```
A — ATTENTION: Stop the scroll with a bold claim or question
"Most [ICP] are leaving €[X] on the table every month."

I — INTEREST: Add specificity that makes them want to know more
"Here's why: [SPECIFIC REASON with data or example]"

D — DESIRE: Paint the picture of the desired outcome
"Imagine [SPECIFIC POSITIVE OUTCOME]. That's what [N]
 of our customers achieved in the first [TIMEFRAME]."

A — ACTION: One clear, low-friction next step
"Start here: [CTA]"
```

---

### 3. BAB — Before / After / Bridge

Best for: testimonials, case studies, email intros

```
B — BEFORE: Describe the situation before your product
"[CLIENT] was spending €800/month on a VA doing manual outreach.
 Reply rate: 1.2%. 0 clients in 3 months."

A — AFTER: Describe the result after
"90 days after switching to [PRODUCT]:
 Reply rate: 8.4%. 7 new clients. €14K new MRR."

B — BRIDGE: Your product is the bridge between the two states
"The difference: [ONE KEY CHANGE they made]"
```

## OBJECTION HANDLING SCRIPTS

### "It's too expensive"
```
Option 1 — ROI frame:
"The problem costs you [X] per month.
 The solution costs [Y] per month.
 The math: [X - Y = monthly gain]."

Option 2 — Daily cost:
"That's €[PRICE/30]/day. Less than a coffee.
 One [result] covers [N] months of subscription."

Option 3 — Comparison:
"A freelancer doing the same job costs €[3-10x more].
 With [PRODUCT] you pay [Y] and it runs 24/7."
```

### "I need to think about it"
```
"I completely understand.
 Two things that might help:
 1. [CASE_STUDY or PROOF LINK]
 2. This offer is valid until [DATE/CONDITION].
 If it makes sense after that — I'm here."
```

### "I don't have time right now"
```
"[PRODUCT] is designed for people who don't have time.
 Setup: [X minutes]. Daily time required: [Y minutes].
 Everything else is automated."
```

### "Does it really work?"
```
"Fair question. Here's the honest answer:
 [N] customers. [RESULT_METRIC] average.
 [SPECIFIC_CASE with real numbers].
 [GUARANTEE]: if it doesn't work for you in [X days] — full refund."
```

## GUARANTEE FORMULAS

### Standard B2C
```
"Try it for 7 days. If you're not satisfied — for any reason —
 email us and we'll refund 100%. No forms. No questions."
```

### Results-Based (for high-ticket)
```
"If you don't [SPECIFIC RESULT] within [TIMEFRAME],
 we'll refund every cent. We take the risk — not you."
```

### Keep the Bonus
```
"Cancel anytime. Keep everything you downloaded.
 We'd rather earn your trust than your money."
```

## URGENCY & SCARCITY (use honestly — never fake)

### Real scarcity
```
"Limited to [N] members to maintain signal quality."
"Only [N] spots left at this price."
"Beta pricing: €[X] now, €[Y] after [DATE]."
```

### Real urgency
```
"This week's signal is sent [DAY] — subscribe before [TIME]
 or wait another week."
"Price increases [DATE] when the next cohort opens."
```

### Soft urgency (always true)
```
"Every week you wait is another week without [RESULT]."
"[N] people signed up in the last 7 days."
```

## CTA BUTTON COPY

### High converting patterns
```
"Start my free trial"          (personal, action-oriented)
"Get instant access"           (immediate gratification)
"Send me the free [resource]"  (tells them exactly what happens)
"Yes, I want [RESULT]"         (affirms the desire)
"Book my free call"            (low commitment framing)
```

### What to avoid
```
"Submit"     → no value signal
"Click here" → generic, no context
"Buy now"    → friction word at early funnel stages
"Sign up"    → feels like admin work
```

## SOCIAL PROOF HIERARCHY

Use in this order — strongest to weakest:

```
1. Specific result with numbers
   "Jean went from 0 to €2,100 MRR in 47 days using this system"

2. Specific result without numbers
   "This is the only tool that actually moved the needle for us"

3. Outcome statement
   "I finally have a system that works while I sleep"

4. Process statement
   "Setup was 20 minutes. I've been running it for 3 months."

5. General positive
   "Highly recommended" (weakest — only use if nothing better)
```

## EMAIL SUBJECT LINE FORMULAS

```
CURIOSITY:    "[N] things about [topic] most people get wrong"
SPECIFICITY:  "+[X]% in [timeframe] — here's exactly how"
PERSONAL:     "re: your question about [topic]"
URGENCY:      "your trial ends today"
RESULT FIRST: "[SPECIFIC RESULT] — here's the trade"
QUESTION:     "are you making this mistake?"
SHORT/BLUNT:  "quick question" / "a thought" / "last email"
```

**Subject line rules:**
- Under 50 characters when possible
- No caps lock (looks like spam)
- No emojis in B2B (fine in B2C)
- Test: would a friend send this subject line?
