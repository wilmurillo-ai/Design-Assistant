# 工作空间结构示例（精简版）

## 核心原则
- **结构反映工作流**：步骤顺序体现实际工作流程
- **文件按逻辑放置**：输入/输出文件放在对应步骤目录
- **避免硬套模板**：根据任务需求创建专属结构

## 示例1：文档分析任务
```
workspace/
├── input/2026-03-13_文档格式分析/
└── output/2026-03-13_文档格式分析/
    ├── 01_文档读取与内容提取/
    ├── 02_文档结构与样式分析/
    ├── 03_模板匹配与模式识别/
    ├── 04_最终文档生成/
    └── 05_结果验证与整理/
```
**工作流**：读取 → 分析 → 匹配 → 生成 → 验证

## 示例2：数据管道任务
```
workspace/
├── input/2026-03-16_网页数据抓取管道/
└── output/2026-03-16_网页数据抓取管道/
    ├── 01_页面解析开发/
    ├── 02_数据提取实现/
    ├── 03_数据清洗转换/
    ├── 04_数据存储设计/
    └── 05_监控与维护/
```
**工作流**：开发 → 实现 → 转换 → 存储 → 监控

## 实践建议
1. **分析任务需求**：明确交付物和工作阶段
2. **创建3-5个步骤**：通常足够覆盖主要工作流
3. **使用动作导向名称**："数据清洗"而非"数据处理阶段"
4. **适时调整**：工作过程中可重命名、拆分或合并步骤
