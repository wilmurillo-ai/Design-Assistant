# 专业型文档修改工作流程

## 适用文档类型

- 项目建议书、可行性研究报告、项目计划书
- 立项报告、商业计划书、投资分析报告
- 技术方案、实施计划书
- 其他涉及数据修改、逻辑调整的专业文档

---

## 第一阶段：自动分析

### 联动自动识别

读取文档后，自动扫描并识别以下联动关系：

1. **数据联动**：被修改数据的计算衍生项（总计、占比、结论）
2. **引用联动**：基于被修改内容的总结、推论、引用
3. **结构联动**：编号、目录、交叉引用

### 确认方式

- 简单修改（无联动）：直接执行，无需询问
- 复杂修改（有联动）：列出自动识别的联动项，确认是否需要同步修改

---

## 第二阶段：执行修改

### 高亮颜色分配规则

1. 阅读所有修改需求，识别包含几个不同类别
2. 每个类别分配一个颜色
3. 关联跟随的内容，跟随其主修改的类别颜色

| 颜色 | 代码 | 适用场景 |
|------|------|----------|
| 黄色 | FFFF00 | 默认主修改色 |
| 绿色 | 00B050 | 数据/预算调整 |
| 蓝色 | 00B0F0 | 逻辑/结论调整 |
| 粉色 | FF99CC | 表达润色 |

### XML级别高亮实现

```xml
<w:shd w:val="clear" w:fill="颜色代码" w:color="auto"/>
```

### 修改后高亮标记

修改后的内容必须添加高亮，同一类的修改用同一颜色。

## 第三阶段：输出校验

### 完成后必做检查清单

```
修改校验清单

[ ] 高亮标记完整性
    - 用户要求的修改是否都标记了？
    - 颜色分类是否正确？

[ ] 数据一致性
    - 数字是否全文统一？
    - 计算口径是否一致？

[ ] 逻辑联动检查
    - "X大问题"->"Y大问题"，全文引用都改了吗？
    - 数据改了，相关结论是否同步调整？
    - 基于被改内容的衍生表述是否需要调整？

[ ] 格式统一性
    - 字体、字号是否与原文一致？
    - 编号样式是否统一？

[ ] 遗漏扫描
    - 扫描全文，确认没有遗漏的关联内容
```

### 校验问题生成

根据修改内容自动生成针对性校验问题：

```
示例（数据修改场景）：
"预算从XX万改到YY万，相关的收益测算、投资回收期、结论是否已同步调整？"

示例（结构修改场景）：
"你提到要改'三大挑战'为'四大挑战'，文中所有引用这个说法的地方都同步了吗？"

示例（定位/品牌修改场景）：
"新增了'[名称]'定位，后续全文中相关引用是否需要统一？"
```

## 第四阶段：输出报告

### 修改清单输出格式

```
修改完成：

位置：[章节名称]
颜色：[颜色名称]
内容：[修改内容简述]

关联调整：
- [需要同步修改的关联项]

需确认：
- [可能遗漏的检查项]
```

---

## 分支逻辑

### 简单修改（单点修改）

直接修改 -> 高亮标记 -> 快速校验

### 复杂修改（涉及联动）

自动识别联动项 -> 确认执行 -> 修改 -> 高亮标记 -> 完整校验

## 技术工具

### 工具选择优先级

1. **pandoc**（推荐）：如果用户环境已安装，先用 `pandoc` 快速读取结构
2. **python-docx**：精确修改Word文档，保留格式
3. **解压XML编辑**：复杂格式控制场景

### 推荐工作流

**读取文档**：
```bash
pandoc "文件.docx" -o "内容.md"
```

**修改文档**：
- 优先使用 python-docx（`pip install python-docx`）
- 或解压docx编辑XML后重新打包

**验证结果**：
```bash
pandoc "修改后.docx" -o "验证.md"
```

### 工具对比

| 工具 | 适用场景 | 优点 |
|------|----------|------|
| pandoc | 快速读取结构 | 一行命令，解析快 |
| python-docx | 精确修改 | API友好，保留格式 |
| 解压XML编辑 | 复杂格式控制 | 最精确，保留所有细节 |
