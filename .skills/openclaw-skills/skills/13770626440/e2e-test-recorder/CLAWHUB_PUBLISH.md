# ClawHub技能发布指南

## 🦀 ClawHub简介

ClawHub是一个AI技能市场，开发者可以发布和分享AI技能。你的E2E测试录制技能非常适合在ClawHub上发布。

## 📋 准备工作

### 1. ClawHub账号
- 访问 https://clawhub.com
- 注册账号（支持GitHub登录）
- 完成邮箱验证

### 2. 技能要求
确保你的技能符合以下要求：
- ✅ 有清晰的README.md
- ✅ 有完整的SKILL.md（符合ClawHub格式）
- ✅ 有package.json（包含版本和依赖）
- ✅ 有使用示例
- ✅ 代码质量良好

### 3. 技能信息准备
准备好以下信息：
- **技能名称**: e2e-test-recorder
- **技能描述**: 自动化端到端测试录制工具，支持浏览器操作录制和测试过程录制
- **技能版本**: 1.0.0
- **技能分类**: 开发工具 / 测试工具
- **技能标签**: e2e-testing, screen-recording, automation, testing
- **技能图标**: 可选，建议尺寸 512x512
- **技能截图**: 展示使用界面的截图

## 🚀 发布流程

### 步骤1：登录ClawHub
1. 访问 https://clawhub.com
2. 点击右上角 "登录"
3. 使用GitHub账号登录（推荐）

### 步骤2：创建新技能
1. 点击右上角用户头像
2. 选择 "我的技能"
3. 点击 "发布新技能"
4. 填写基本信息：
   - **技能名称**: e2e-test-recorder
   - **显示名称**: E2E Test Recorder
   - **技能描述**: 自动化端到端测试录制工具，支持浏览器操作录制和测试过程录制
   - **技能分类**: 选择 "开发工具" → "测试工具"
   - **技能标签**: 输入 `e2e-testing`, `screen-recording`, `automation`, `testing`, `puppeteer`, `nodejs`
   - **开源协议**: 选择 MIT License

### 步骤3：上传技能文件
1. **上传方式**: 选择 "GitHub仓库"
2. **仓库URL**: 输入你的GitHub仓库URL
   - 例如: `https://github.com/YOUR_USERNAME/e2e-test-recorder`
3. **分支**: 选择 `main` 或 `master`
4. **技能根目录**: 留空（如果是仓库根目录）
5. **技能版本**: 输入 `1.0.0`

或者选择 "上传文件":
1. 打包你的技能文件：
   ```bash
   cd D:\knowledge\coding\e2e-test
   tar -czf e2e-test-recorder-1.0.0.tar.gz \
     --exclude=node_modules \
     --exclude=recordings \
     --exclude=logs \
     --exclude=test-data/temp \
     .
   ```
2. 上传打包文件

### 步骤4：配置技能
1. **入口文件**: 选择 `SKILL.md`
2. **技能类型**: 选择 "工具类技能"
3. **运行环境**: 选择 "Node.js"
4. **依赖安装命令**: 输入 `npm install`
5. **启动命令**: 输入 `node scripts/cli.js`
6. **技能图标**: 上传图标文件（可选）
7. **技能截图**: 上传3-5张截图，展示：
   - CLI使用界面
   - 录制过程
   - 输出文件
   - 配置界面

### 步骤5：填写详细说明
1. **功能特性**:
   ```
   - 🎥 浏览器操作录制：基于Puppeteer的浏览器录制
   - 🎯 端到端测试录制：完整的测试流程录制
   - 🔄 格式转换：支持MP4、GIF、WebM格式
   - ⚡ 自动化测试集成：可与测试框架集成
   - 📊 性能监控：录制过程中的性能数据
   - 🎨 视频编辑：基础视频编辑功能
   ```

2. **安装说明**:
   ```
   ## 安装方式
   
   ### 从ClawHub安装
   ```bash
   npx skills add YOUR_USERNAME/e2e-test-recorder@e2e-test-recorder
   ```
   
   ### 从GitHub安装
   ```bash
   git clone https://github.com/YOUR_USERNAME/e2e-test-recorder.git
   cd e2e-test-recorder
   npm install
   ```
   
   ### 全局安装CLI
   ```bash
   npm install -g .
   ```
   ```

3. **使用示例**:
   ```
   ## 快速开始
   
   ### 录制网页操作
   ```bash
   e2e-test record https://example.com --output demo.mp4
   ```
   
   ### 录制端到端测试
   ```bash
   e2e-test test configs/test.json --record
   ```
   
   ### 查看帮助
   ```bash
   e2e-test --help
   ```
   ```

4. **配置说明**:
   ```
   ## 配置文件
   
   技能支持JSON配置文件，位置：`configs/default.json`
   
   ```json
   {
     "recorder": {
       "fps": 24,
       "quality": 80,
       "outputDir": "./recordings"
     },
     "browser": {
       "headless": false,
       "viewport": {
         "width": 1280,
         "height": 720
       }
     }
   }
   ```
   ```

### 步骤6：设置价格和权限
1. **技能价格**: 选择 "免费"
2. **技能权限**: 
   - ✅ 允许用户安装
   - ✅ 允许用户复制
   - ✅ 允许商业使用（根据MIT协议）
3. **技能可见性**: 选择 "公开"

### 步骤7：提交审核
1. 点击 "预览" 检查所有信息
2. 确认无误后点击 "提交审核"
3. 等待ClawHub审核（通常1-3个工作日）
