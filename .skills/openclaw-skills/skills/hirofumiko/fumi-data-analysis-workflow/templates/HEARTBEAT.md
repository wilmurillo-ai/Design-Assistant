# HEARTBEAT - Data Analysis Project Monitoring

## 目的
データ分析プロジェクトの進捗を監視し、定期的に更新する。

## 行動指針
- データの更新を確認
- 実験の進捗を確認
- モデルのパフォーマンスを確認
- ドキュメントの更新を確認

## 行動内容

### Step 1: 状況理解
- [データの更新状況を確認]
- [実験の進捗状況を確認]
- [モデルのパフォーマンスを確認]
- [ドキュメントの更新状況を確認]

### Step 2: アクション実行
- [新しいデータの分析が必要な場合: 分析を開始]
- [実験が完了した場合: 結果を確認]
- [モデルのパフォーマンスが低下した場合: 再トレーニングを検討]
- [ドキュメントが古い場合: 更新]

### Step 3: 記録
ここまでで行ったことの概要をまとめてmemoryする。

### Step 4: 報告
必要に応じて報告を行う。

## 定期チェック項目

### データ更新
- 新しいデータが利用可能か？
- データの品質に問題はないか？
- データのスキーマに変更はないか？

### 実験の進捗
- 実行中の実験はあるか？
- 結果を確認すべき実験はあるか？
- 失敗した実験を再試行する必要があるか？

### モデルのパフォーマンス
- モデルの精度に変化はないか？
- 過学習や劣化の兆候はないか？
- 再トレーニングの必要はあるか？

### ドキュメント
- 分析結果をドキュメントしたか？
- ノートブックを整理したか？
- 結論を記録したか？

## 注意事項
- 統計的有意性を確認する
- 結論の不確実性を認識する
- 再現性を維持する
