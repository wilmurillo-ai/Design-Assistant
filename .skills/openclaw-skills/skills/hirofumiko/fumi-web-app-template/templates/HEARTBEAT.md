# HEARTBEAT - Web Application Development Monitoring

## 目的
Webアプリケーション開発プロジェクトの進捗を監視し、定期的に更新する。

## 行動指針
- パフォーマンスを確認
- エラーを確認
- セキュリティを確認
- デプロイ状況を確認

## 行動内容

### Step 1: 状況理解
- [パフォーマンス状況を確認]
- [エラー状況を確認]
- [セキュリティ状況を確認]
- [デプロイ状況を確認]

### Step 2: アクション実行
- [パフォーマンスが低下した場合: 最適化を検討]
- [エラーを発見した場合: 修正とテスト]
- [セキュリティ脆弱性を発見した場合: 修正とパッチ]
- [デプロイが必要な場合: デプロイプロセス]

### Step 3: 記録
ここまでで行ったことの概要をまとめてmemoryする。

### Step 4: 報告
必要に応じて報告を行う。

## 定期チェック項目

### パフォーマンス
- Core Web Vitalsは良好か？（LCP, FID, CLS, TTFB）
- ページロード時間は適切か？
- コード分割や遅延ロードは有効か？
- 画像の最適化は行われているか？

### エラー
- JavaScriptエラーはあるか？
- APIエラーはあるか？
- 4xx/5xxエラーはあるか？
- エラーログに異常はないか？

### セキュリティ
- OWASPガイドラインに従っているか？
- 入力検証とサニタイズは行われているか？
- XSS/CSRF/SQLインジェクションの脆弱性はないか？
- 認証・認可は適切に実装されているか？

### デプロイ
- デプロイは成功しているか？
- CI/CDパイプラインは正常に動作しているか？
- 環境変数は正しく設定されているか？
- モニタリングは有効か？

### アクセシビリティ
- WCAG 2.1 AAに準拠しているか？
- ARIA属性は適切に使用されているか？
- キーボードナビゲーションは動作しているか？
- スクリーンリーダー対応は行われているか？

### SEO
- メタタグは適切に設定されているか？
- サイトマップは作成されているか？
- 構造化データは実装されているか？
- Lighthouseスコアは良好か？

## 注意事項
- ユーザー体験を常に考慮する
- パフォーマンスを最適化し続ける
- セキュリティを重視する
- アクセシビリティを確保する
- テストと監視を徹底する
