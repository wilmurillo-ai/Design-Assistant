# HEARTBEAT - CLI Tool Development Monitoring

## 目的
CLIツール開発プロジェクトの進捗を監視し、定期的に更新する。

## 行動指針
- コードの品質を確認
- テストの実行状況を確認
- ドキュメントの更新を確認
- バグや問題を追跡

## 行動内容

### Step 1: 状況理解
- [コードの品質状況を確認]
- [テストの実行状況を確認]
- [ドキュメントの更新状況を確認]
- [バグや問題を確認]

### Step 2: アクション実行
- [コードの品質に問題がある場合: リンティングや型チェックを実行]
- [テストが失敗した場合: 修正を検討]
- [ドキュメントが古い場合: 更新]
- [バグを発見した場合: 修正とテスト]

### Step 3: 記録
ここまでで行ったことの概要をまとめてmemoryする。

### Step 4: 報告
必要に応じて報告を行う。

## 定期チェック項目

### コードの品質
- リンティングのエラーはあるか？
- 型チェックのエラーはあるか？
- コードのスタイルは一貫しているか？

### テストの状況
- ユニットテストはパスしているか？
- 統合テストはパスしているか？
- テストカバレッジは十分か？
- 新しいテストが必要か？

### ドキュメント
- READMEは最新か？
- ヘルプメッセージは明確か？
- 使用例は動作するか？
- CHANGELOGは更新されているか？

### バグと問題
- オープンな問題はあるか？
- 新しいバグを発見したか？
- 回帰（regression）を検出したか？
- ユーザーからのフィードバックはあるか？

## 注意事項
- ユーザー体験を常に考慮する
- 明確なエラーメッセージを提供する
- テスト駆動開発を維持する
- ドキュメントを常に更新する
