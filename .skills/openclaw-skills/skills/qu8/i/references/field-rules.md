# 字段识别规则

## work_type 工作类型

按顺序匹配，命中即返回：

| 类型 | 关键词 |
|------|--------|
| 会议 | 会议、开会、研讨、评审、汇报 |
| 沟通 | 联系、对接、沟通、交流、讨论、反馈、回复 |
| 文档 | 文档、写、报告、材料、总结、整理、台账 |
| 设计 | 设计、规划、方案、蓝图 |
| 测试 | 测试、验证、检查、调试、排查 |
| 编程 | 编程、编码、开发、写代码、技能、部署 |
| 调研 | 调研、研究、调查、了解 |
| 其他 | 以上均不匹配 |

## planning 事务性质

- 含"临时/突然/插/紧急插" → **临时**
- 含"计划/原本安排/安排/原定" → **计划内**
- 默认 → **临时**

## importance 重要性

- 含"重要/关键/核心/必须" → **重要**
- 含"不重要/次要" → **不重要**
- 否则 → **未标注**

## urgency 紧迫性

- 含"紧急/尽快/马上/立刻/急需/急" → **紧急**
- 含"不紧急/稍后/以后" → **不紧急**
- 否则 → **未标注**

## quality 完成质量

- 含"完成了/搞定/解决/顺利/成功/正常了" → **高质量**
- 含"完成得不错/还行/一般/凑合/完成" → **中等**
- 含"有问题/没弄好/卡住/失败/需要改进" → **待改进**
- 否则 → **未标注**

## is_todo 待办判断

- 含以下词且**无完成信号** → `true`：下周、明天、待处理、要去、需要、再去、等上班、上班后、下次
- 含完成信号（已完成/搞定/解决了/完成了/处理了/正常了） → `false`

## work_date 日期识别

| 输入 | 解析 |
|------|------|
| "今天" | 当前日期 |
| "昨天" | 当前日期 - 1天 |
| "明天" | 当前日期 + 1天 |
| "下周一" | 下周一的日期 |
| "下周上班" | 下周一的日期 |
| "2月26号" / "2月26日" | 当年 2026-02-26 |
| "2026-02-26" | 直接使用 |
| 无日期信息 | 当前日期 |

## contacts 对接人识别

- 识别内容中"和/与/联系/通知/协调"后跟随的中文姓名（2~4字）
- 识别常见角色词：产品经理、运维、保安、领导、同事、客户
- contacts 列表长度即为 contact_count
