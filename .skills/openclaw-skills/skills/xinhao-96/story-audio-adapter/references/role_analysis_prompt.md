# Story Role Analysis Prompt

你要处理的是一段已经用 `[角色名]文本内容` 标记好的小说/故事文本，并基于当前音色库为角色匹配最接近的声音。

## 输入

- 原始故事文本
- 当前音色库 `voice_library.json`

## 任务

请严格按以下步骤执行：

1. 统计文本中出现的所有角色名，并区分主角色、旁白、次要角色。
2. 结合角色台词和叙述内容，总结每个角色的人设标签：
   - 年龄感
   - 性别倾向
   - 性格特征
   - 情绪基调
   - 叙事功能
3. 对每个角色，从音色库中选择一个最接近的 voice。
4. 给出匹配理由，理由必须引用角色文本特征和音色描述，不得只说“感觉合适”。
5. 如果多个角色必须复用同一音色，要说明原因和风险。

## 输出格式

输出 JSON，结构如下：

```json
{
  "role_count": 3,
  "roles": [
    {
      "role_name": "旁白",
      "line_count": 12,
      "traits": {
        "age_hint": "成年",
        "gender_hint": "中性",
        "personality": ["沉稳", "克制"],
        "emotion": "平稳"
      },
      "voice_match": {
        "voice_id": "male_0004_a",
        "description": "儒雅道长",
        "reason": "旁白承担整体叙事，语气偏平稳克制，与儒雅沉稳的声线更接近。"
      }
    }
  ],
  "notes": [
    "若角色数量超过可用音色数量，优先保障旁白和主角的区分度。"
  ]
}
```

## 限制

- 不要编造文本中不存在的人设
- 不要输出音色库之外的 voice_id
- 角色名保持与原文一致
- 如果某个角色信息不足，明确写“信息不足”
