# Privacy Redaction Rules

## Default Rule

Public or repo-ready material should preserve the method and remove private specifics.

## Remove Or Generalize

| Information | Treatment |
|---|---|
| Interviewer names | Replace with role |
| Raw recordings | Do not include |
| Transcript excerpts | Summarize as question types |
| Compensation | Remove |
| Recruiter private messages | Remove |
| Non-public org structure | Remove |
| Internal metrics | Remove unless already public |
| Personal investment or asset details | Generalize |
| Social account analysis | Keep method, remove identity details |

## Keep

- Round structure
- Interview objectives
- Question categories
- Preparation methods
- Reverse question patterns
- Pressure interview tactics
- JD teardown logic
- Product research methods
- Domain knowledge map
- Anonymous examples

## Example

Do not publish:

```text
Named interviewer X asked me Y after seeing my exact project.
```

Publish:

```text
In the business owner round, expect questions that test strategic judgment, project truthfulness, risk awareness, and whether the candidate can do foundational work before pursuing innovation.
```
