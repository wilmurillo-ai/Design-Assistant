# Case Interview And 30/60/90 Plans

## Use When

- User has a product case.
- User has take-home homework.
- User is preparing for final round.
- User targets lead or director roles.
- User needs a 30/60/90 plan.

## Case Structure

```md
Conclusion:
...

User and scenario:
...

Problem diagnosis:
...

Options:
1. ...
2. ...
3. ...

Recommendation:
...

MVP:
...

Metrics:
...

Risks:
...

Next 30/60/90 days:
...
```

## Product Experience Report

Must include:

- Main user path
- Friction points
- Risk points
- Competitor comparison
- Business impact
- Prioritized recommendations

## 30/60/90 Plan

30 days:

- Learn business context
- Meet stakeholders
- Validate metrics and roadmap
- Find real bottlenecks
- Identify quick wins

60 days:

- Align priority
- Start 1-2 focused initiatives
- Build operating rhythm
- Define success metrics

90 days:

- Deliver visible result
- Establish review mechanism
- Propose 6-12 month roadmap
- Earn trust for larger scope

## Director-Level Standard

Director-level cases must include:

- Business loop
- Resource constraints
- Organizational friction
- Risk control
- Metrics
- Sequencing
- Communication plan
