# Mock Interview Scoring

## Use When

- User provides an answer.
- User provides transcript.
- User asks whether the answer is good enough.
- User wants final-round simulation.

## Scorecard

| Dimension | Weight | What Good Looks Like |
|---|---:|---|
| Role fit | 25% | Directly maps to JD and interviewer objective |
| Business judgment | 20% | Talks users, money, priority, metrics, tradeoffs |
| Domain depth | 20% | Can withstand wallet, DeFi, data, AI follow-ups |
| Structure | 15% | Clear conclusion, logic, and pacing |
| Evidence | 10% | Real projects, metrics, decisions, outcomes |
| Risk awareness | 10% | Sees safety, compliance, org, cost, and execution risk |

## Output Format

```md
Hiring recommendation:
Strong yes / yes / borderline / no

Score:
...

What worked:
1. ...

Biggest risks:
1. ...

Likely follow-ups:
1. ...

Stronger answer:
...

Next drill:
...
```

## Harsh But Useful Standard

For senior candidates, a good answer must include:

- Judgment
- Evidence
- Tradeoff
- Execution path
- Metric or validation
- Risk control
