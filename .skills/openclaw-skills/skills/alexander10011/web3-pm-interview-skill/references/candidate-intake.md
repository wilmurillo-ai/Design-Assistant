# Candidate Intake

## Required Fields

Use this structure when the user provides a resume or background.

```yaml
candidate:
  alias:
  current_role:
  years_experience:
  target_level:
  target_company:
  target_role:
  interview_stage:
  time_left:
  strongest_projects:
  web3_experience:
  technical_depth:
  management_experience:
  founder_experience:
  english_level:
  strongest_edge:
  biggest_risk:
  specific_questions:
```

## Initial Diagnosis

Classify the candidate:

- Web3 native PM
- Web2 PM transitioning to Web3
- Founder returning to organization
- Data/platform PM moving into on-chain data
- Growth/BD/operator moving into product
- Senior IC moving into lead/director

Then judge:

- Is the target level realistic?
- Does the candidate have direct proof or only transferable proof?
- What must be translated?
- What must be learned?
- What must be de-risked?

## Risk Patterns

- Founder risk: hard to manage, unstable, only wants innovation.
- Web2 transition risk: lacks real wallet/DeFi user feel.
- Seniority risk: project scope does not match target level.
- Technical risk: cannot withstand wallet, DeFi, or data follow-ups.
- Strategy risk: has opinions but no metrics or execution path.
- English risk: cannot operate in global org meetings.
