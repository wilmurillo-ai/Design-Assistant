# Wallet PM Domain Map

## What Interviewers Test

- Real wallet usage
- Understanding of signing and transaction flow
- Security and risk judgment
- Ability to simplify complexity
- Metrics and growth thinking
- CEX, DEX, DeFi, bridge, and asset flow connections

## Must Know

- EOA vs MPC vs smart account
- Seed phrase, private key, signing, recovery
- Approve, allowance, revoke
- Gas, nonce, slippage, MEV basics
- Transaction simulation
- Wrong chain and wrong address failure modes
- DApp connection and WalletConnect
- Asset display and token risk detection

## Strong Answer Pattern

When discussing wallet UX, include:

1. User intent
2. Risk point
3. Transaction path
4. Explanation layer
5. Confirmation design
6. Failure handling
7. Metric impact

## Useful Metrics

- Activated wallets
- Funded wallets
- First transaction completion
- Swap conversion
- Bridge success rate
- Signing failure rate
- Wrong-chain incidents
- Revoked high-risk approvals
- 7-day and 30-day retention

## Common Tradeoffs

- Security vs conversion
- Simplicity vs user control
- Automation vs trust
- Multi-chain breadth vs quality
- CEX integration vs self-custody credibility
