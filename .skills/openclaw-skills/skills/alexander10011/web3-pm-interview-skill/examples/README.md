# Examples

Use these examples to understand how the skill translates different candidate backgrounds into Web3 PM interview strategy.

## Case Map

| Example | Best For | Core Translation |
|---|---|---|
| `anonymized-wallet-senior-pm-case.md` | Senior wallet / DeFi / on-chain data PM candidates | Founder + Web3 product experience -> wallet data/platform PM |
| `anonymized-product-director-case.md` | Product lead / director candidates | Senior product experience -> strategy, organization, risk, 30/60/90 |
| `web2-fintech-to-wallet-pm.md` | Web2 fintech PMs moving into Web3 wallets | Payment transaction UX -> wallet transaction path |
| `growth-bd-to-exchange-growth-pm.md` | Web3 growth, BD, ops, or community candidates | Campaign/distribution experience -> productized growth loops |
| `defi-researcher-to-product-pm.md` | DeFi researchers, analysts, or investment professionals | Protocol research -> product decisions and user workflows |
| `filled-candidate-intake.md` | Users who need an intake example | How to fill candidate input |
| `sample-battle-plan-output.md` | Users who want to see final output | Example battle plan |

## Privacy Standard

All examples are anonymized or synthetic. They are designed to preserve reusable interview patterns without exposing private interview details, company confidential information, or personal data.
