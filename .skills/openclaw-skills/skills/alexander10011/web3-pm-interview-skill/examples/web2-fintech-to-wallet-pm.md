# Anonymous Case: Web2 Fintech PM -> Web3 Wallet PM

## Candidate Type

Senior Web2 fintech PM with strong transaction, onboarding, risk, and data product experience. Limited Web3 building experience, but some wallet usage.

## Target Role

Web3 Wallet Senior PM covering asset management, swap, bridge, onboarding, and transaction safety.

## Role Reality

This is not simply a "crypto app feature" role.

The real hiring model is:

> A financial transaction product owner who can improve wallet activation, signing confidence, swap/bridge conversion, transaction success, and user trust in a self-custody environment.

## Core Translation

| Candidate Asset | Web3 Translation |
|---|---|
| Web2 payment transaction flow | Wallet send / swap / bridge transaction path |
| KYC and risk review | Transaction risk explanation and user protection |
| Failed transfer reduction | Failed transaction, wrong-chain, slippage, and bridge failure reduction |
| Data dashboard | Wallet funnel, success rate, and support-ticket metrics |
| Partner API integration | Wallet, DEX, bridge, and protocol integration collaboration |

## Biggest Risks

1. Interviewer may think the candidate is "too Web2."
2. Candidate may not understand self-custody and irreversible transaction risk.
3. Candidate may use payment language instead of wallet language.
4. Candidate may lack DeFi and on-chain data depth.

## Winning Mainline

> I am a financial transaction PM. My strongest value for a wallet team is improving user activation, transaction success, and trust across high-risk flows like funding, signing, swap, bridge, and asset management.

## De-risking Language

For "you are too Web2":

> The ownership model is different in Web3 because users hold keys and transactions are irreversible. But the product discipline transfers: understand intent, reduce failure points, explain risk clearly, measure conversion and error rates, and build recovery paths where possible.

For "you do not know DeFi deeply":

> I would not pretend to be a protocol expert on day one. My first priority is to master the flows users touch every day: wallet creation, funding, approvals, swap, bridge, revoke, transaction simulation, and failed transaction handling. That is the fastest path to becoming useful to a wallet team.

## High-Probability Questions

1. Why move from fintech to Web3 wallet?
2. What is different between Web2 payments and wallet transactions?
3. How would you improve first transaction conversion?
4. How would you reduce failed swap or bridge transactions?
5. How do you balance safety and conversion?
6. Which wallets have you used, and what would you improve?
7. What is approval risk?
8. What is slippage?
9. What metrics would you own in the first 90 days?
10. What Web3 topics are you still learning?

## Must-know Prep

P0:

- Seed phrase, private key, signing
- Gas, nonce, slippage
- Approval, allowance, revoke
- Swap and bridge user paths
- Transaction simulation
- Wallet activation funnel

P1:

- AMM basics
- Lending basics
- Account abstraction basics
- WalletConnect and DApp connection

P2:

- Deep MEV
- Advanced restaking
- Low-level indexer architecture

## Reverse Questions

1. Where is the biggest drop-off in the current wallet transaction path: funding, signing, swap, bridge, or post-transaction trust?
2. How does the team currently balance risk warnings with conversion?
3. If I joined and did well in six months, which metric would most likely reflect that?

## 7-day Prep Plan

Day 1:

- Rewrite the JD into wallet transaction and trust problems.

Day 2:

- Use 3 wallets and complete send, swap, bridge, approve, revoke.

Day 3:

- Learn wallet fundamentals and write a transaction flow map.

Day 4:

- Learn DeFi basics: AMM, lending, APY, TVL.

Day 5:

- Convert 3 fintech projects into wallet-relevant STAR stories.

Day 6:

- Mock hiring manager questions.

Day 7:

- Finalize self-introduction and reverse questions.
