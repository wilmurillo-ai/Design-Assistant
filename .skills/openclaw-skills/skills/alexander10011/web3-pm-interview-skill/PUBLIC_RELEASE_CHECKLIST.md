# Public Release Checklist

Use this checklist before switching the repository from private to public.

## Repository

- [x] `README.md` explains Why / What / How in English and Chinese.
- [x] `SKILL.md` includes usage modes and first-response guidance.
- [x] `LICENSE` exists.
- [x] `CONTRIBUTING.md` exists.
- [x] Examples are anonymized or synthetic.
- [x] Templates are usable without private context.
- [x] GitHub description and topics are set.
- [ ] Repository visibility changed from private to public.

## Privacy

- [x] No raw interview recordings.
- [x] No private recruiter messages.
- [x] No compensation or package details.
- [x] No named interviewer analysis.
- [x] No non-public company information.
- [x] No internal metrics.
- [x] Real cases are role-anonymized.

## Content Quality

- [x] Contains at least one filled candidate intake example.
- [x] Contains at least one sample battle plan output.
- [x] Contains multiple candidate transition examples.
- [x] Contains domain references for wallet, DeFi / on-chain data, and AI Wallet.
- [x] Contains mock scoring rubric.
- [x] Contains privacy redaction rules.

## Launch Assets

- [x] X / Twitter launch copy prepared.
- [x] LinkedIn launch copy prepared.
- [x] Chinese community launch copy prepared.
- [x] FAQ prepared.
- [x] GLM review report generated.
- [x] GLM retest passed after P0 fixes.

## Recommended Launch Sequence

1. Switch the repository to public.
2. Create a GitHub release: `v0.4-public-preview`.
3. Publish the X / LinkedIn / Chinese community posts from `LAUNCH_COPY.md`.
4. Pin the repo link in the first comment or thread.
5. Invite feedback from:
   - Web3 PM candidates
   - Wallet / DeFi PMs
   - Hiring managers
   - Recruiters
6. Collect issues and examples for V0.5.

## V0.5 Ideas

- Lightweight intake form.
- More candidate examples.
- Contribution templates.
- GitHub issue templates.
- Automated file/reference validation.
- Public website or landing page.
