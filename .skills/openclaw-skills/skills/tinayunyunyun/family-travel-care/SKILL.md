# 特殊旅客关怀

**重要：所有输出必须使用中文（简体）。**

这个技能的目标不是罗列一大堆服务，而是把用户这趟行程串成一个可执行方案：现在要做什么、什么时候申请、准备什么、到机场怎么说。

## 核心原则

### 1. 按完整行程回答，不按人群堆信息

优先按下面的顺序组织答案：

1. 先给结论：这趟行程最该做的 2-4 件事
2. 再给时间线：订票后 / 出发前 72 小时 / 48 小时 / 机场 / 飞行中 / 转机 / 抵达
3. 再给申请与报备卡片
4. 最后给证件、材料、药物清单

不要先分“婴儿服务一大段、老人服务一大段、药物一大段”然后让用户自己拼答案。

### 2. 只保留跟决策有关的信息

保留：
- 需要提前申请的服务
- 需要现场说明的事项
- 会影响成行或体验的关键限制
- 低成本航司与全服务航司的差异
- 用户马上可以复制使用的话术/模板

删除或弱化：
- 机上标配、用户几乎不会主动申请的低价值信息
- 同一件事重复出现
- 与当前行程无关的泛泛提示
- 没有明确行动价值的机场设施罗列

例如，不要主动写“婴儿救生衣”“机上有毛毯”这类低决策价值信息，除非用户明确问到。

### 3. 用户问的是“怎么解决”，不是“有哪些服务”

输出要围绕这几个问题：

- 这趟行程会卡在哪一步？
- 现在该申请什么？
- 哪些不用特殊处理，照要求带就行？
- 哪些必须提前报备？
- 到机场、安检、登机口分别该怎么说？

### 4. 可视化表达优先

优先使用：
- 4 行以内的“先做这几件事”
- 行程时间线表
- 决策表
- 申请卡片 / 话术卡片
- 清单

少用大段散文说明。

## 识别重点

先快速识别这趟行程的关键变量：

| 维度 | 你要识别什么 |
|------|--------------|
| 同行人 | 婴儿 / 儿童 / 老人 / 行动不便 / 一名大人同时照顾多人 |
| 行程 | 国内 / 国际 / 联程 / 非联程 / 是否转机 / 转机时间长短 |
| 航司 | 全服务航司还是低成本航司 |
| 舱位 | 经济舱 / 高级经济舱 / 商务舱 |
| 特殊需求 | 轮椅、婴儿摇篮、儿童首飞、特殊餐、药物、医疗设备 |
| 药物复杂度 | 普通携带 / 需安检说明 / 需航司提前报备 |

如果用户是一名成人同时带婴儿和老人，必须主动指出这是“操作压力高”的组合，并优先给出减负方案，例如轮椅协助、靠前座位、避免过短转机、同一区域座位、登机前一次性确认。

## 工具使用

### 优先调用

- `flight-helper:scenario_advice`
  用于确认这趟行程下的航司服务、转机协助、家庭/无障碍动线、低成本航司差异。
- `flight-helper:premium_cabin_info`
  只有在升舱或增购座位对体验有明显帮助时才调用。

### 工具不可用时

使用通用民航常识和常见航司规则给出方案，但要明确区分：

- **高确定性**：普遍适用的安检/乘机原则
- **待确认**：可能因航司、机场、目的地变化的细则

不要编造精确公斤数、价格、时限、客服电话。拿不准就写“通常”“一般建议”“请在出票后通过航司订单页或客服确认”。

## 申请卡片示例库

以下是高频场景的预填示例。输出时根据用户实际情况替换具体信息，让用户拿到就能直接复制使用，不要留占位符。

### 示例：婴儿摇篮 + 老人轮椅一次性申请

```
你好，我的订单号是 CX1234567。
我这次带 8 个月婴儿和 70 岁老人一起出行，行程是上海→香港→伦敦。
想一次确认：
1. 两段航班都申请婴儿摇篮（宝宝约 8kg）
2. 老人备注轮椅服务（走路慢，不需要上下楼梯协助），含转机接驳
3. 座位尽量安排在同一区域
4. 值机当天还需要在哪个环节再次确认？
```

### 示例：儿童哮喘药安检说明

```
这是孩子正在使用的哮喘吸入器，品牌是万托林/沙丁胺醇。
放在随身包里便于飞行途中使用。
这里有处方，如果需要我可以配合开包检查。
```

### 示例：需要航司报备的医疗设备

```
你好，我的订单号是 MU7654321。
同行旅客需要在飞行途中使用便携制氧机，型号是 Inogen One G5，
电池容量 99Wh。
想确认三件事：
1. 这台设备能否带上飞机并在机上使用
2. 是否需要填写医疗表格或提供适航证明
3. 登机当天要向哪个柜台确认
```

## 药物处理逻辑

药物不要一律写成“必须开医生证明”。先判断复杂度，再给方案。

### A. 普通携带：通常不用特殊处理

适用：
- 片剂、胶囊
- 常规处方药
- 常见非处方药
- 小体积外用药
- 常规吸入器

回答方式：
- 直接告诉用户通常可携带
- 提醒尽量保留原包装
- 国际行程建议带处方或药盒说明，便于安检/海关解释
- 不要把这类药物写成“必须提前报备航司”

示例表达：

```markdown
这类药通常不用提前向航司报备，按正常随身药品携带即可：
- 保留原包装
- 用量与行程匹配
- 放随身包，不建议全放托运行李
- 如是国际航班，最好带处方或药盒说明页
```

### B. 需要安检说明：通常不用航司审批，但要准备材料

适用：
- 超过常规液体限制的液体药
- 雾化药液
- 婴儿必需药水
- 注射药物和针具
- 便携雾化器等医疗用品

回答方式：
- 明确这类通常是“安检说明问题”，不是“航司审批问题”
- 写清需要准备什么
- 给安检申报卡片

必须覆盖的内容：

| 项目 | 要点 |
|------|------|
| 官方/常见要求 | 必要药品通常可随身携带，但需主动说明，可能需开包检查 |
| 要准备什么 | 原包装、处方/医生证明、用药说明、必要时英文说明 |
| 到现场怎么做 | 安检前主动告知，不要等被查到再解释 |

安检申报卡片模板：

```markdown
### 安检申报卡片
我随身带了孩子/老人正在使用的必要药物，其中有液体药/雾化药/针具。
这里有处方和医生说明，请问我应该走哪一步检查？
```

### C. 需要提前报备或双重确认：必须给解决方案

适用：
- 机上用氧、制氧机、担架等医疗协助
- 电池规格敏感的医疗设备
- 需要机上冷藏/特殊保存的药物
- 近期手术、明显不适合独立乘机的情况
- 涉及目的地海关限制的管制成分药物

这类问题必须输出：

1. **官方要求看哪三类**
   - 航司乘机要求
   - 机场安检要求
   - 目的地海关/入境要求
2. **你现在该怎么做**
   - 先确认航司是否接受
   - 再确认是否需要医疗表格/医生适航证明
   - 再确认目的地是否允许入境
3. **要准备什么**
   - 订单号
   - 药名与成分
   - 剂量与使用频率
   - 医生证明
   - 设备型号/电池参数（如有）
4. **话术卡片**

航司报备卡片模板：

```markdown
### 航司报备卡片
你好，我的订单号是 [订单号]。
同行旅客需要携带/使用 [药物或设备名称]，情况是 [简述病情或用途]。
我想确认三件事：
1. 这项药物/设备能否带上飞机
2. 是否需要提前备注、医疗证明或表格
3. 转机/登机当天要向哪一个柜台再次确认
```

医生证明准备卡：

```markdown
### 医生证明准备卡
请医生在证明里尽量写清：
- 旅客姓名
- 药物/设备名称
- 为什么必须携带或使用
- 单次剂量与每日频次
- 飞行途中是否可能需要使用
- 医生签名、医院盖章、日期
```

## 低成本航司处理

如果是春秋这类低成本航司，必须显式区分：

| 类别 | 默认判断 |
|------|----------|
| 免费服务 | 通常更少，且执行弹性更小 |
| 选座 | 往往需要付费 |
| 儿童餐/婴儿餐 | 不要默认存在 |
| 机上娱乐 | 不要默认存在 |
| 特殊协助 | 有可能能申请，但要更早确认 |

输出时加一个简短区块：

```markdown
## 低成本航司特别提醒
这类航司通常不是“不能飞”，而是“很多体验型服务没有或要付费”。这次你要把重点放在：
- 提前选座
- 自备吃的/娱乐物品
- 提前确认是否支持特殊协助
- 不要默认有儿童餐、摇篮、免费相邻座位
```

## 升级建议规则

只有在升级真的能解决问题时才写：

- 长航段老人膝盖不适，额外腿部空间或高级经济舱有价值
- 一名成人同时照顾婴儿和老人，靠前/同区座位有价值
- 长时间转机且带婴儿/老人，贵宾室或休息区可能有价值

不要把所有付费项目都列出来。每次最多给 1-3 个最有价值的建议，并说明“为什么值得”。

## 机场设施规则

只有设施会直接影响这次方案时才写，例如：
- 母婴室
- 家庭/无障碍安检通道
- 轮椅接驳点
- 转机协助集合点

不要写旅游指南式的设施大全。

## 需要主动补位的细节

如果用户没明确提到，但场景里高概率会踩坑，要主动补一句：

- 婴儿摇篮通常要尽早申请，而且有体重/身长限制
- 轮椅服务要区分仅代步还是上下楼梯也需要协助
- 一名大人同时带婴儿和老人时，转机时间不要太短
- 国际行程中，婴儿和儿童也通常需要独立证件/签证
- 药物问题要区分“安检说明”与“航司审批”，不要混为一谈

## 不要这样输出

- 不要为了显得全面，把所有服务一口气列满
- 不要把普通药品写成高风险事项
- 不要把安检要求、航司要求、海关要求混成一句话
- 不要只说“建议联系航司”，却不告诉用户要准备什么、怎么问
- 不要给长段空话，尽量让用户一眼能复制行动

## 与其他技能联动

- **flight-baggage-check**：当药品、电池、雾化器、母婴液体、辅助设备涉及能否托运/随身时，可建议联动
- **airport-transfer-guide**：当转机协助、非联程、跨航站楼、短转机会影响老人/婴儿时，可建议联动
- **travel-outfit-planner**：当用户还没准备婴儿用品、保暖、换洗、护理物品时，可建议联动
