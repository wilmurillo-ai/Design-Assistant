# 测试用例设计参考手册

## 一、测试用例核心要素

### 1.1 用例编号规则
```
TC-[模块]-[功能]-[序号]
例如: TC-LOGIN-001, TC-ORDER-003
```

### 1.2 优先级定义
| 优先级 | 定义 | 说明 |
|--------|------|------|
| P0 | 冒烟测试 | 核心功能失败，阻塞后续测试 |
| P1 | 高优先级 | 主要功能路径，必须通过 |
| P2 | 中优先级 | 重要功能，允许已知缺陷 |
| P3 | 低优先级 | 边缘功能，可延后处理 |

## 二、测试方法详解

### 2.1 等价类划分法

#### 步骤
1. 分析输入条件
2. 划分有效等价类和无效等价类
3. 每个等价类选取代表性测试数据
4. 设计测试用例覆盖所有等价类

#### 示例：用户名输入框
| 等价类ID | 描述 | 类型 | 测试数据 |
|----------|------|------|----------|
| EQ-01 | 6-18位字母数字 | 有效 | user123, test_user |
| EQ-02 | 少于6位 | 无效 | abc, u1 |
| EQ-03 | 多于18位 | 无效 | 1234567890123456789 |
| EQ-04 | 纯数字 | 无效 | 123456 |
| EQ-05 | 特殊字符 | 无效 | user@#$ |
| EQ-06 | 中文 | 无效 | 用户名 |

### 2.2 边界值分析法

#### 步骤
1. 确定边界条件（上、下、内部）
2. 选择上点、内点、离点
3. 设计测试用例覆盖这些点

#### 示例：年龄输入框（0-120）
| 边界ID | 边界描述 | 上点 | 内点 | 离点(减) | 离点(加) |
|--------|----------|------|------|----------|----------|
| BV-01 | 年龄范围 | 0, 120 | 60 | -1 | 121 |

### 2.3 决策表法

#### 示例：订单折扣规则
| 条件 | 规则1 | 规则2 | 规则3 | 规则4 |
|------|-------|-------|-------|-------|
| 订单≥1000元 | Y | Y | N | N |
| VIP会员 | Y | N | Y | N |
| **动作** | | | | |
| 折扣20% | ✓ | | | |
| 折扣10% | | ✓ | | |
| 折扣5% | | | ✓ | |
| 无折扣 | | | | ✓ |

### 2.4 状态转换法

#### 示例：用户注册状态机
```
[未注册] --填写信息--> [信息填写] --提交--> [审核中] --通过--> [已注册]
                              |                      |
                              +--拒绝--> [注册失败] <-+
```

### 2.5 场景法

#### 基本流（Happy Path）
用户完成目标的理想流程

#### 备选流（Alternative Flow）
各种异常情况和分支处理

#### 示例：ATM取款
| 场景类型 | 描述 | 路径 |
|----------|------|------|
| 基本流 | 正常取款成功 | 插卡→输入密码→输入金额→出钞→取卡 |
| 备选流1 | 密码错误 | 插卡→输错密码→重新输入→... |
| 备选流2 | 余额不足 | 插卡→输入密码→输入金额→提示余额不足 |
| 备选流3 | 超出日限额 | 插卡→输入密码→输入金额→超出日限额 |

## 五、覆盖率计算公式

### 5.1 语句覆盖率
```
语句覆盖率 = (被执行语句数 / 总语句数) × 100%
```

### 5.2 判定覆盖率
```
判定覆盖率 = (判定结果为真次数 + 判定结果为假次数) / (判定总数 × 2) × 100%
```

### 5.3 路径覆盖率
```
路径覆盖率 = (实际覆盖路径数 / 理论路径数) × 100%
```
