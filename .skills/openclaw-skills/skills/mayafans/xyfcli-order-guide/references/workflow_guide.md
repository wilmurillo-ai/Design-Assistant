# 交互式下单完整工作流程

## 第 1 步：产品验证和选择

### 开场白（含能力清单）

**首次互动时，先告知用户你能做什么：**

```
我可以帮你：
① 快速下单（一句话下单，自动验证）
② 查产品（看看有什么肥料、查 45% 的产品）
③ 查客户列表（我有哪几个客户）
④ 查地址（客户的收货地址在哪）
⑤ 图片下单（发送订货单图片自动识别）

需要什么服务？
```

**如果用户直接说下单，使用简化开场：**

```
我来帮您下单。首先，您有：
1. 产品描述（我来搜索），
2. 产品编码（如果已经知道），
3. 产品图片（拍照或发送图片，我从中提取描述），还是
4. 订货单图片（拍照或发送订货单，我从订货单提取完整信息）？

请提供描述、产品编码、产品图片或订货单图片。
```

### 选项 A：通过描述搜索产品

**用户提供描述**："含量 45% 13-5-27"

**OpenClaw 操作**：
1. **搜索产品 URI**：
   ```bash
   xyfcli shop getproducturibydesc -desc "含量 45% 13-5-27" -limit 5 --json
   ```
2. **批量获取产品详情**：对每个 URI 执行
   ```bash
   xyfcli shop getproductdetailbyuri -uri "产品 URI" --json
   ```
3. **向用户呈现选项**：
   ```
   我找到了 3 个匹配的产品：
   1. 产品编号：[产品编码 1] - [产品名称 1]
   2. 产品编号：[产品编码 2] - [产品名称 2]
   3. 产品编号：[产品编码 3] - [产品名称 3]
   
   您想订购哪个产品？（输入数字 1-3）
   ```
4. **获取用户选择**并验证产品编码：
   ```bash
   xyfcli shop getgoodsinfo "选中的产品编码"
   ```

### 选项 B：直接输入产品编码

**用户提供编码**："Y163U1305276020000,ABC123456789"

**OpenClaw 操作**：
1. **验证每个产品编码**：
   ```bash
   xyfcli shop getgoodsinfo "Y163U1305276020000"
   xyfcli shop getgoodsinfo "ABC123456789"
   ```
2. **确认验证结果**：
   ```
   我已验证这两个产品编码都存在：
   - Y163U1305276020000 ✓
   - ABC123456789 ✓
   ```

### 选项 C：通过产品图片解析产品

**用户发送图片**：用户发送产品图片

**OpenClaw 操作**：
1. **解析图片提取描述**：使用图片识别技术提取产品描述信息
2. **执行搜索流程**：使用与选项 A 相同的搜索流程

### 选项 D：通过订货单图片处理

**用户发送订货单图片**

**OpenClaw 操作**：
1. **解析订货单图片提取信息**：提取产品描述/编码、客户信息、数量、发货基地、收货地址
2. **验证提取的信息**：使用相应流程验证产品和客户
3. **向用户确认**：
   ```
   我从订货单中提取到以下信息：
   - 产品信息：[产品编码/描述]
   - 客户信息：[客户姓名]
   - 购买重量：[重量] 吨
   - 发货基地：[基地]
   - 卸货地点：[地址]
   
   我将使用这些信息创建订单。请确认是否正确？（是/否）
   ```

### 产品验证失败处理

**如果验证失败**：
```
"产品编码 [编码] 不存在。请提供有效的产品编码或尝试不同的搜索词。"
```

## 第 3 步：数量管理

### 询问数量

⚠️ **重要**：商品数量是必传参数，必须为正数，且与产品数量一一对应。

```
"现在让我们为每个产品指定数量。

您订购的是：
1. Y163U1305276020000
2. [其他产品如果有]

每个产品需要多少？
请按相同顺序提供数量，用逗号分隔。

示例：对于 2 个产品，输入"5,3"表示第一个 5 吨，第二个 3 吨。"
```

### 处理用户输入

**用户提供**："5,3"

**OpenClaw 验证**：
1. **数量匹配**：确保数量数量与产品数量匹配
2. **正整数**：所有数量必须 > 0
3. **格式检查**：仅限逗号分隔的整数

**如果验证失败**：
```
"我需要 [N] 个数量值对应 [N] 个产品。您提供了 [M] 个值。
请提供恰好 [N] 个正整数，用逗号分隔。

3 个产品的示例："10,5,3""
```

### 确认数量

```
"数量摘要：
- Y163U1305276020000：5 吨
- ABC123456789：3 吨
- 总重量：8 吨

是否正确？（是/否）"
```

## 第 4 步：发货基地和地址选择

### 获取发货基地选项

```bash
xyfcli shop getdeliverybase -productcode "产品编码" -dealercode "客户编码" --json
```

**呈现基地**：
```
此产品和客户的可用发货基地：
1. 新洋丰中磷
2. 武汉配送中心  
3. 成都仓库

我应该使用哪个发货基地？（输入数字 1-3）
```

### 验证收货地址

⚠️ **重要**：收货地址必须是客户在精准营销系统中已保存的地址，不支持临时输入新地址。地址ID是必传参数，必须精确传入，不支持模糊匹配或默认选择。收货人和电话提单时必传，可从地址中提取默认值，用户需确认。

```bash
xyfcli shop getdealeraddresses "客户编码" --json
```

```
客户有 2 个已保存地址：
1. 湖北省荆门市东宝区泉口街道馨梦缘公寓（牛一 13121007444）
2. 甘肃省天水市秦安县叶堡镇新阳村（牛建建 18093818192）

我应该使用哪个地址？（输入数字 1-2）
```

**情况 B：客户没有已保存的地址**
```
✗ 该客户没有已保存的收货地址

请先到精准营销系统维护收货地址：
1. 登录精准营销系统
2. 进入客户管理 → 找到对应客户
3. 添加收货地址（省/市/区/详细地址/联系人/电话）

|---|----------|----------|------|

| 2 | ABC123456789 | 某产品 ✓ | 3 吨 |

```

### 生成订单

**如果用户确认"是"**：

```bash
xyfcli order place \
  -dealer "J620522007" \
  -name "牛建建" \
  -sales "EZB2019063" \
  -products "Y163U1305276020000,ABC123456789" \
  -q "5,3" \
  -base "10" \
  -addr-id 15099 \
  -receiver "牛建建" \
  -phone "18093818192"
```

**注意**：
- `-addr-id` 参数必须是系统中已保存的地址 ID，必须传入，不支持模糊匹配或默认选择
- `-receiver` 和 `-phone` 参数可选，如不传则使用地址中的联系人和电话，但需在订单确认时明确展示并让用户确认
- `-q` 参数必须传入商品数量，与产品一一对应
- 不支持临时输入新地址，必须在精准营销系统中维护
- 运输方式固定为"汽运"
- 提货方式固定为"统派车"

## 多产品处理

### 中途添加产品

**用户**："我想添加另一个产品"

**OpenClaw 回应**：
```
"好的！您当前订单中有 [N] 个产品。

请提供：
1. 新产品编码（或描述），和
2. 此产品的数量

我将验证产品，然后让您再次确认所有数量。"
```

### 移除产品

**用户**："移除第二个产品"

**OpenClaw 回应**：
```
"我将从您的订单中移除产品 [产品编码]。

您更新后的产品列表：
1. [第一个产品] ×[数量]
...（已移除第二个产品）

请确认剩余产品的数量："
```

### 更改数量

**用户**："将产品 X 的数量改为 10"

**OpenClaw 回应**：
```
"正在将 [产品编码] 数量从 [旧值] 更新为 10。

更新后的订单摘要：
[带新数量的产品列表]

是否正确？"
```

## 最佳实践

### 作为对话领导者
1. **始终汇总**每个步骤后
2. **确认后再继续**到下一步
3. **需要选择时提供清晰选项**
4. **使用简单语言**避免技术术语
5. **重复用户所说**以确认
6. **用户困惑时提供帮助**
7. **格式要求时提供示例**

### 作为 CLI 操作员
1. **解析输出时使用`--json`标志**
2. **执行命令前验证所有输入**
3. **优雅处理错误**并提供有用的建议
4. **先用简单案例测试命令**
5. **通过测试连接保持会话有效**
6. **记录交互**以便故障排除
7. **呈现给用户前验证输出**

### 作为流程协调员
1. **始终遵循 5 步流程**
2. **用户按不同顺序提供信息时保持灵活**
3. **允许随时更正**
4. **概念上保存进度**（记住选择）
5. **全程提供状态更新**
6. **订单创建时庆祝完成**
7. **订单生成后提供后续步骤**

- [ ] 所有 CLI 命令成功执行
