# 输出格式参考

_同程旅行数据输出格式示例（与 `scripts/lib/formatters.js`、`scripts/lib/output-mode.js` 行为一致）_

---

## 📊 格式选择规则

逻辑由 `resolve_output_mode({ channel, surface })` 实现（各 `*-query.js` 共用）：

| 条件 | 使用格式 | 链接形式 |
|------|---------|---------|
| `channel` = `webchat` | **表格** | Markdown：`[PC 端预订](url)`、`[移动端预订](url)`（见 `render_booking_buttons`） |
| `channel` 包含 `wechat` / `weixin` / `微信` | **卡片** | 纯文本：`PC 端：url` / `移动端：url`（`render_booking_buttons`） |
| `surface` = `mobile` | **卡片** | Markdown 链接 |
| `surface` = `table` | **表格**（覆盖上文） | 同上 |
| `surface` = `card` | **卡片**（覆盖上文） | 同上 |
| 其他 | **卡片** | Markdown 链接 |

**说明：**

1. 先按 `channel` / `mobile` 得到初始 `use_table`、`use_plain_link`（微信系为 `use_plain_link = true`）。
2. 再执行 **`surface` = `table` / `card` 的覆盖**：可强制表格或卡片（例如 `channel=webchat` 且 `--surface card` 时输出卡片）。
3. 表格与卡片中的预订列/行均由 `render_booking_buttons` 生成：Markdown 模式下为 `[PC 端预订](…)`、`[移动端预订](…)`（仅输出有效 URL，空或 `#` 不渲染）；微信渠道为 `PC 端：url | 移动端：url` 形式的纯文本。

---

## 🔗 预订链接（`render_booking_buttons`）

| 模式 | 输出 |
|------|------|
| Markdown | `[PC 端预订](url)`、`[移动端预订](url)` 以空格拼接；无效链接省略对应段 |
| 微信 / 纯文本 | `PC 端：url \| 移动端：url`；无效链接省略对应段 |
| 双端均无效 | 空字符串（表格「预订」列为空，卡片可不单独展示预订行） |

## 🚄 火车票输出格式

### 表格（直达车次，`format_train_table`）

**不展示余票/余座**（表格与卡片均如此）。**价格列**展示所有坐席及对应价格（取自 `ticketList` 的 `ticketType` + `ticketPrice`，逗号分隔组合在同一单元格）；无 `ticketList` 时兜底使用 `price` 字段。

```markdown
| 车次 | 出发站 | 到达站 | 出发时间 | 到达时间 | 运行时长 | 价格 | PC 预订 | 手机预订 |
|------|--------|--------|---------|---------|---------|------|--------|---------|
| 🚅 G7209 | 苏州 | 上海 | 07:25 | 08:06 | 41 分 | 二等座¥42.0, 一等座¥69.0 | [PC 端](link) | [手机端](link) |
```

### 卡片（`format_train_card`）

价格行展示所有坐席及对应价格（同上规则），标签为「**价格**」。

```markdown
### 🚅 G7209 | 苏州 → 上海
**出发时间** 07:25 | **到达时间** 08:06 | **时长** 41 分
**价格** 二等座¥42.0, 一等座¥69.0
**预订** [PC 端](pc_link) | [手机端](mobile_link)

---
```

> **坐席价格组合规则**：由辅助函数 `format_train_seat_prices` 实现——遍历 `train.ticketList`，将每个 `ticketType` + `ticketPrice` 组合为 `坐席名¥价格`，多个以 `, ` 分隔；若无 `ticketList` 则兜底为 `¥{train.price}`。

### 中转联程（`format_transfer_train`）

与火车票、机票接口共用：顶层起讫为 `depStationName` / `arrStationName`，缺省时用 `depAirportName` / `arrAirportName`。含多段表格（`format_transfer_segment`：火车 `🚄`、飞机 `✈️`，班次列为 `trafficNo`）、中转说明行、末尾 `🔗 [PC 端] | [手机端]`。航班段（`segmentType` 非 TRAIN/非 BUS）的机场名追加航站楼、时间含月日前缀和跨天标注。

> ⚠️ **链接真实性要求**：必须使用 API 返回的真实链接，**禁止省略或伪造**

## ✈️ 机票输出格式

### 表格（常规航班，`format_flight_table`）

表格行**不**自动加 ✈️ 前缀（仅为示例可读性时可省略）。机场名后追加航站楼（仅当 `depAirportTerminal`/`arrAirportTerminal` 非空时）；时间含月日前缀（如 `04-17 07:50`），跨天时到达时间标注 `+1天`。

```markdown
| 航班号 | 出发机场 | 到达机场 | 出发时间 | 到达时间 | 时长 | 价格 | 航司 | PC 预订 | 手机预订 |
|--------|---------|---------|---------|---------|------|------|------|--------|---------|
| MU5123 | 上海虹桥T2 | 北京首都T3 | 04-17 19:00 | 04-17 21:15 | 2 时 15 分 | ¥409 | 东方航空 | [PC 端](link) | [手机端](link) |
```

### 特价机票表格（`format_flight_table_special`，`flightNo` 为空等特价场景）

最后一列为 **PC + 手机** 两列链接，表头为「预订链接」一栏内用 `|` 分隔两段 Markdown。

```markdown
| 目的地 | 价格 | 日期 | 折扣 | 原价 | 预订链接 |
|--------|------|------|------|------|----------|
| 上海→北京 | ¥299 | 周一 | 3 折 | ¥899 | [PC 端](…) | [手机端](…) |
```

（脚本中该行为单行单元格内 `pc_md \| mobile_md`。）

### 卡片（`format_flight_card`）

机场名后追加航站楼（仅非空时）；时间含月日前缀，跨天时到达时间标注 `+N天`。

```markdown
### ✈️ MU5123 | 上海虹桥T2 → 北京首都T3
**出发时间** 04-17 19:00 | **到达时间** 04-17 21:15 | **时长** 2 时 15 分
**价格** ¥409
**航司** 东方航空
**预订** [PC 端](pc_link) | [手机端](mobile_link)

---
```

> ⚠️ **链接真实性要求**：必须使用 API 返回的真实链接，**禁止省略或伪造**

## 🧳 跟团游/度假产品输出格式

### 表格（`format_trip_table`）

- **目的地**列：`(trip.destList || []).join(', ')`
- **特点**列：`(trip.labelList || []).join(', ')`
- **评分**：有有效分时为 `⭐{score}`，否则 `暂无`（**不含评论条数**，与景区表不同）

```markdown
| 产品 | 目的地 | 价格 | 评分 | 特点 | PC 预订 | 手机预订 |
|------|--------|------|------|------|--------|---------|
| 三亚5天4晚自由行 | 三亚 | ¥2999 | ⭐4.7 | 含往返机票, 海景酒店 | [PC 端](link) | [手机端](link) |
```

### 卡片（`format_trip_card`）

标题使用 **🏖️**（非 🧳）。

```markdown
### 🏖️ 三亚5天4晚自由行
**价格** ¥2999
**特色** 含往返机票 + 海景酒店 + 接送机
**地址** （若有）
**预订** [PC 端](pc_link) | [手机端](mobile_link)

---
```

> ⚠️ **链接真实性要求**：必须使用 API 返回的真实链接，**禁止省略或伪造**

## 🗺️ 行程规划输出格式

行程规划由 `format_trip_plans`（多方案入口）+ `format_trip_plan`（单方案格式化）实现，仅在 `travel-query.js` 中使用。

### 多方案头部（`format_trip_plans`）

多方案时输出引导语与输出要求，然后逐个渲染方案：

```markdown
★★★★★ 以下为向你推荐的行程规划信息，你可以参考并预订每一天的行程 ★★★★★

📋 **输出要求**：
1. 完整输出所有行程规划方案，不要筛选、不要遗漏任何方案
2. 每个方案必须包含完整的每日行程（Day 1, Day 2, ...），不要跳过任何一天
3. 保留每个景点/活动的推荐理由和详细介绍，不要删减
4. 保持原始预订链接格式，不要改写 URL
5. 如果内容较长，可以分段输出，但必须确保完整性

🗺️ **推荐行程规划**

#### 方案 1

（单方案内容）
```

> 仅当方案数 > 1 时才输出 `#### 方案 N` 标题。

### 单方案格式（`format_trip_plan`）

**行程概览**：

```markdown
📋 **行程概览**
⏱️ **天数**: 3 天 | 🏨 **住宿**: 2 晚 | 🍜 **美食**: 西湖醋鱼、龙井虾仁
🎯 **景点数**: 8 个

📅 **每日行程**
```

**每日行程**：

```markdown
### Day 1：西湖经典游
📍 **城市**: 杭州

**1. 西湖风景区**
漫步苏堤白堤，赏断桥残雪

**门票** ¥0 | **星级** 5A级景区 | **评分** ⭐4.8（25000条） | **开放时间** 全天 | **建议游玩** 半天-1天 | **主题** 自然风光

🔗 **立即预订**：[PC 端](pc_link) | [手机端](mobile_link)
```

**详情行规则**：

- 详情字段（门票/星级/评分/开放时间/建议游玩/主题）仅在**有值时**展示，用 ` | ` 分隔
- **门票**：`activity.price` 不为 `undefined`/`null`/空字符串时展示（含 `0` 表示免费）
- **星级**：`activity.star` 有值时展示
- **评分**：`activity.score` 有值时展示，含评论数则为 `⭐{score}（{commentNum}条）`
- **开放时间**：`activity.openTime` 有值时展示
- **建议游玩**：`activity.playTime` 有值时展示
- **主题**：`activity.theme` 有值时展示
- 预订链接由 `render_booking_buttons` 生成，规则与其他品类一致

> ⚠️ **链接真实性要求**：必须使用 API 返回的真实链接，**禁止省略或伪造**

## 💬 底部引导语（脚本追加，以各 `*-query.js` 为准）

不同脚本结尾文案略有差异，例如：

- 火车票 / 机票 / 汽车（`bus-query`）：引导至 **火车票机票**
- 酒店：引导至 **酒店民宿**
- 景区 / 度假 / 交通聚合（`traffic-query`）等：常含 **火车票机票** 与 **酒店民宿**

示例（交通聚合类）：

```markdown
💡 **更多选择**：也可以打开 **同程旅行 APP** 或在 **微信 - 我 - 服务** 中，点击 **火车票机票** 或 **酒店民宿** 查看更丰富的资源。
```

## ⚠️ 预订链接强制要求

> 🚨 **链接是用户完成预订的唯一有效路径，必须完整输出！**

**要求：**

1. **每个资源都必须有预订链接**（车次、航班、酒店、景区、汽车班次等）
2. **同时输出 PC 端与手机端展示列**（脚本内对跟团游等场景可能两列使用相同 URL，仍以脚本输出为准）
3. **禁止省略、替换或修改链接**
4. **禁止伪造不存在的链接**
5. 链接格式：
   - 非微信渠道：`[PC 端](url)` | `[手机端](url)`（或表格单元格内两段）
   - 微信渠道：单元格内为 **裸 URL**，不是 `PC 端：url` 文案格式

**错误示例** ❌：

```markdown
| 车次 | ... | 预订链接 |
| G123 | ... | [预订](#) |  <!-- 链接被替换 -->
```

**正确示例** ✅：

```markdown
| 车次 | ... | PC 预订 | 手机预订 |
| G123 | ... | [PC 端](https://www.ly.com/...) | [手机端](https://m.ly.com/...) |
```
