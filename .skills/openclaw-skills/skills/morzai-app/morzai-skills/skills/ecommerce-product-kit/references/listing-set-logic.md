# Listing Set Narrative Logic

This guide defines how a full ecommerce image set should be structured so the asset package can move from attention to trust to conversion.

---

## 1. Core P1-P7 narrative

| Slot | Role | Business job | Visual focus |
| :--- | :--- | :--- | :--- |
| **P1** | **Hero Image** | Grab attention | Clear product focus, strong first click potential, high compliance where required. |
| **P2-P3** | **Core Benefits** | Build trust | Visualize key selling points, upgrades, materials, or use-case benefits. |
| **P4** | **Detail Hero** | Deepen sensory understanding | Macro texture, craftsmanship, hardware, stitching, or functional close-ups. |
| **P5** | **Multi-view / Coverage** | Provide full context | Front, side, back, angle variation, or fit coverage. |
| **P6** | **Lifestyle** | Create desire | Show the product in a believable real-life or aspirational context. |
| **P7** | **Decision-making image** | Support final conversion | Size chart, package contents, guarantee, support info, or purchase reassurance. |

---

## 2. Consistency rules

- The full set should share one lighting language and one commercial style system.
- Typography, palette, and icon style must feel related across the set.
- Alternate between cleaner layouts and denser informational layouts to avoid repetition fatigue.
- Each slot should do a different job; do not make all 7 images visually or functionally identical.

---

## 3. Execution sequence

1. Extract or summarize the main selling points from the source image and product context.
2. Match the visual direction to the target platform and market.
3. Generate the image set according to P1-P7 role separation.
4. Review the set for consistency, trust signals, and conversion clarity.

---

## 4. Agent implications

Use this guide when the user asks for:
- a full listing set
- a 7-image set
- a full product-page package
- a complete ecommerce asset set

For single-image requests, cross-reference `output-spec.md` and only use the relevant slot logic.
