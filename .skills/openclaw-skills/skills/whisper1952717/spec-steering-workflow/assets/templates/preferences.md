# Preferences Steering

## Execution Style
- Prefer clear requirements, design, and tasks before major execution.
- Prefer small batches with verifiable outcomes.
- Prefer file-first state over chat-only state.

## Communication
- Report facts and artifacts, not vague progress.
- Never use "still working" as a checkpoint.
- Always leave one exact next action before stopping.

## Quality Bar
- Tasks must be verifiable.
- Templates must stay agent-friendly.
- Prefer lightweight automation over heavy systems.

## Default Decisions
- Use spec workflow for complex tasks.
- Put stable context in `steering/`.
- Put task-specific context in `specs/`.
