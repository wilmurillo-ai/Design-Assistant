# 飞书消息直接发送文件工作流程

## 概述
本文档总结了通过飞书消息直接发送文件附件的工作流程，解决了发送文档链接而非直接文件的问题。

## 问题背景
用户需要直接通过飞书消息发送文件附件，而不是发送文档链接。经过多次尝试，发现使用 `message` 工具的 `path` 参数是最有效的直接发送文件方法。

## 核心解决方案

### 关键发现
- 使用 `message` 工具的 `path` 参数直接发送本地文件
- 避免使用 `media` 参数发送文档链接
- 确保文件路径正确且文件存在

### 正确方法 vs 错误方法

#### ❌ 错误方法（发送的是链接）：
```javascript
message({
  action: "send",
  message: "这是文档",
  media: "https://feishu.cn/docx/..."  // 这是文档链接，不是文件
})
```

#### ✅ 正确方法（发送的是文件）：
```javascript
message({
  action: "send", 
  message: "这是文件附件",
  path: "/root/.openclaw/workspace/文件.ppt"  // 这是本地文件路径
})
```

## 完整工作流程

### 步骤1：创建本地文件
```javascript
write({
  path: "/path/to/文件.ppt",
  content: "文件内容..."
})
```

### 步骤2：使用message工具发送文件附件
```javascript
message({
  action: "send",
  message: "文件说明文字",
  path: "/path/to/文件.ppt",      // 关键：使用path参数指定本地文件路径
  filename: "自定义文件名.ppt"    // 可选：指定显示的文件名
})
```

## 参数说明

| 参数 | 作用 | 是否必需 | 示例 |
|------|------|----------|------|
| `action: "send"` | 发送消息 | 是 | `"send"` |
| `message` | 消息正文内容 | 是 | `"这是AI MaaS述职报告PPT文件"` |
| `path` | **本地文件路径**（关键参数） | 是 | `"/root/.openclaw/workspace/文件.ppt"` |
| `filename` | 客户端显示的文件名 | 否 | `"AI_MaaS_述职报告PPT.ppt"` |

## 实际应用示例

### 示例1：发送单个文件
```javascript
// 1. 创建PPT文件
write({
  path: "/root/.openclaw/workspace/AI_MaaS_述职报告PPT.ppt",
  content: "幻灯片1: 封面\n幻灯片2: 目录..."
})

// 2. 发送文件附件
message({
  action: "send",
  message: "这是AI MaaS述职报告PPT文件，请查收。",
  path: "/root/.openclaw/workspace/AI_MaaS_述职报告PPT.ppt",
  filename: "AI_MaaS_述职报告PPT.ppt"
})
```

### 示例2：批量发送多个文件
```javascript
// 查找所有.docx和.pdf文件
exec({
  command: "ls -la /root/.openclaw/workspace/ | grep -E '\\.(docx|pdf)$'"
})

// 逐个发送文件
const files = [
  "AI_MaaS_述职报告PPT.docx",
  "harness_engineering_article.docx",
  "AI_5_Layer_Cake_with_Images.pdf"
];

files.forEach((file, index) => {
  message({
    action: "send",
    message: `文件 ${index + 1}: ${file}`,
    path: `/root/.openclaw/workspace/${file}`,
    filename: file
  });
});
```

## 注意事项

### 1. 文件路径
- 使用绝对路径：`/root/.openclaw/workspace/文件名.ppt`
- 避免使用相对路径
- 确保文件存在且有读取权限

### 2. 文件类型支持
- 支持各种文件类型：.ppt、.docx、.pdf、.txt、.jpg等
- 文件名后缀决定客户端如何显示和处理文件

### 3. 文件大小限制
- 注意飞书对附件大小的限制
- 大文件可能需要分卷或压缩

### 4. 错误处理
- 检查返回的 `ok: true`
- 确认收到 `messageId`
- 如果发送失败，检查文件路径和权限

## 常见问题排查

### 问题1：文件发送失败
**可能原因**：
- 文件路径错误
- 文件不存在
- 权限不足

**解决方案**：
```javascript
// 检查文件是否存在
exec({
  command: "ls -la /path/to/file"
})
```

### 问题2：收到的是链接而不是文件
**可能原因**：
- 使用了 `media` 参数而不是 `path` 参数
- 参数设置错误

**解决方案**：
确保使用 `path` 参数而不是 `media` 参数。

### 问题3：文件名显示不正确
**可能原因**：
- 没有设置 `filename` 参数
- `filename` 参数包含特殊字符

**解决方案**：
设置合法的 `filename` 参数。

## 最佳实践

### 1. 文件准备
- 在发送前确认文件已创建
- 检查文件大小是否合适
- 确保文件内容正确

### 2. 发送过程
- 添加有意义的消息说明
- 设置合适的文件名
- 一次发送一个文件，避免并发问题

### 3. 验证结果
- 检查返回状态 `ok: true`
- 记录 `messageId` 用于追踪
- 如有失败，记录错误信息

## 总结

### 核心要点
1. **使用 `path` 参数**：直接发送本地文件
2. **避免 `media` 参数**：不要发送文档链接
3. **检查文件存在**：发送前确认文件可访问
4. **设置合适文件名**：方便用户识别

### 工作流程记忆口诀
```
一查路径二确认，
三设参数四发送。
路径要用绝对路，
媒体参数不要用。
```
