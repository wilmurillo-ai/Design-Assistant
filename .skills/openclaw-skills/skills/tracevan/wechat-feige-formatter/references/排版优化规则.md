# 公众号排版优化规则 v2.3

这份文档定义了如何将普通 Markdown 文档智能优化为公众号排版格式。使用本 Skill 时，AI 会按照这些规则进行优化。

---

## 核心原则

**用户不需要懂任何排版语法。** 你的工作是读懂内容、判断意图、加上合适的排版标记。

**优化分为两个阶段**：
1. **AI 优化阶段**（本文档的范围）：识别内容语义，添加排版标记，生成标准 Markdown
2. **排版转换阶段**：formatter.py 根据标记生成 HTML

**黄金原则**：
- 不改变原文意思，只做格式标记
- 宁可少标记，也不要过度标记
- 每种格式用在最合适的场景，不要强行凑
- 检查列表里说禁止的，一定不能做

---

## 版式元素全览

| 版式 | formatter 语法 | 触发场景 |
|------|---------------|---------|
| PART 章节卡片 | `## 章节标题` | 文章大章节分隔（也支持旧写法 `## PART 01 标题`） |
| BREAKING 开篇框 | `## BREAKING` 块 | 文章开头的亮点提炼（formatter 会自动生成，无需手动写） |
| STEP 步骤卡片 | `> STEP 01 标题` + `> 说明` | 有顺序的操作步骤、流程 |
| ✨ 要点框 | `> ✨ 标题` + `> 内容` | 核心结论、关键信息、重要观点 |
| 💡 提示框 | `> 💡 标题` + `> 内容` | 小贴士、实用建议、注意/警告 |
| 关键词高亮 | `==词==` | 专业术语、核心概念、产品名、需要强调的词 |
| 🤖 指令框 | `> 🤖 PROMPT:` + `> 内容` | AI 指令、代码示例、命令、配置 |
| 代码块 | ` ```lang ``` ` | 代码片段 |
| ### 小标题 | `### 标题` | 章节内部的小节标题 |
| **加粗** | `**词**` | 句子中需要强调但不至于高亮的词 |

## 任务 1：识别 STEP 步骤

**触发信号**（有任意一项即可）：
- 关键词：`第一步`、`第二步`、`首先`、`然后`、`接着`、`最后`、`Step X`、`步骤 X`
- 有编号的连续列表：`1.`、`2.`、`3.` 或 `1）`、`2）`
- 描述连续动作的句子组合（如"打开→输入→点击"）
- 操作类说明：安装、配置、部署、下载、上传等步骤

**转换方式**：

```markdown
# 原始
第一步：打开应用
第二步：输入参数，确认配置正确
第三步：点击确认按钮

# 转换后
> STEP 01 打开应用

> STEP 02 输入参数
> 确认配置正确

> STEP 03 点击确认按钮
```

**规范**：
- 标题部分简洁，不超过 15 个字
- 说明内容放在标题行下方，每行以 `> ` 开头，**中间不能有空行**
- STEP 编号从 01 开始，自动递增（formatter 会重新编号，不用担心顺序）
- 每个步骤独立成块，步骤之间用空行分隔

## 任务 2：识别 ✨ 核心要点框

**触发信号**：
- 段落中出现"最重要的是"、"核心是"、"关键在于"、"一句话说"、"总结来说"
- 整段是对前面内容的提炼或结论
- 明显值得读者特别注意的观点
- 文章中作者语气最强、最有洞见的那句话

**转换方式**：

```markdown
# 原始
最重要的是：在开始之前一定要先备份数据。

# 转换后
> ✨ 关键提醒
> 在开始之前一定要先备份数据
```

**规范**：
- 标题简短，概括要点（3-8 字）
- 内容是核心信息本身，不是废话
- 同一段内容只用一个 ✨ 框，不要堆叠
- **每个章节（## 标题之间）最多出现 1 个 ✨ 框**，只留最重要的那个
- **框内多行内容，每行都要以 `> ` 开头，中间不能有空行**

---

## 任务 3：识别 💡 提示框

**触发信号**：
- 正文中出现"注意"、"提醒"、"小贴士"、"tips"、"建议"、"推荐"
- 文章中的补充说明、经验分享
- 对普通用户友好的实用建议
- 警告性内容（踩坑、避免错误）

**与 ✨ 的区别**：
- ✨ → 核心观点、重要结论（读者必读）
- 💡 → 实用建议、补充提示（读者参考）

**转换方式**：

```markdown
# 原始
注意：这个操作在 Windows 和 Mac 上的路径不同。

# 转换后
> 💡 注意：不同系统的路径
> Windows 和 Mac 上的路径不同，请根据自己的系统选择对应的命令
```

**规范**：
- 标题简短，说明这是什么类型的提示
- **每个章节（## 标题之间）最多出现 1 个 💡 框**；✨ 和 💡 加起来每章节也最多 1 个
- 只有原文中作者明确有"注意/提示/小贴士"类表达时才加，不要主动补充
- **框内多行内容，每行都要以 `> ` 开头，中间不能有空行**

---

## 任务 4：识别关键词高亮

**触发信号**：
- 专业术语、行业名词（首次出现时标记）
- 产品名称、技术名词、品牌名
- 文章中反复强调的核心概念
- 作者自己造的词或专有说法

**转换方式**：

```markdown
# 原始
WorkBuddy 是腾讯推出的 AI 智能体平台，支持通过 Skill 扩展功能。

# 转换后
==WorkBuddy== 是腾讯推出的 ==AI 智能体== 平台，支持通过 ==Skill== 扩展功能。
```

**规范**：
- 只标记词语，不标记短句或整段
- 同一个词在全文中首次出现时标记一次即可，后续出现不重复标记
- **密度控制（重要）：每 3 行正文最多出现 1 个高亮词**，宁可少标也不要堆
- 一句话里不超过 1 个高亮词（极少数情况可 2 个，绝不超过 2 个）
- 常见词不需要标记（如"方法"、"结果"、"技术"等）
- 支持嵌套加粗：`==**核心概念**==`

---

## 任务 5：识别 🤖 指令/代码框

**触发信号**：
- AI 使用的 Prompt 指令
- 命令行命令（如 `npm install`、`python3 xxx`）
- 代码片段（如果不是独立代码块的话）
- 配置文件内容

**转换方式**：

```markdown
# 原始
运行以下命令安装依赖：
npm install && npm run build

# 转换后
> 🤖 PROMPT: 安装并构建
> npm install && npm run build
```

**规范**：
- **框内多行内容，每行都要以 `> ` 开头，中间不能有空行**
- 如果代码较长，用代码块（\`\`\`）更合适
- 指令框适合短命令、单条 Prompt

---

## 任务 6：识别 ### 小标题

**触发信号**：
- 章节内部有明显的子话题分组
- 原文中有加粗的"标题性文字"但不足以升格为 `##`
- 列举多个方面/维度时，每个维度可以作为小标题

**转换方式**：

```markdown
# 原始
**工具选择的三个维度**
易用性：...
稳定性：...
价格：...

# 转换后
### 工具选择的三个维度

易用性：...
稳定性：...
价格：...
```

## 任务 7：处理 BREAKING 开篇框

**说明**：formatter 会自动从 `#` 标题生成 BREAKING 框（不填副标题，避免与正文重复）。**正常情况下无需手动写**。

只有在以下情况下才需要手动写 BREAKING 块：
- 需要定制副标题（全文一句话概括，与正文不重复）
- 需要"旧方式 → 新方式"的对比句型

**手动写法**（⚠️ 每个字段必须以 `- ` 开头，缺少 `- ` 会被当成正文输出）：
```markdown
## BREAKING
- bold: 新方式的核心标题（大号黑字）
- subtitle: 全文一句话概括（不要复制正文第一句）
- strikethrough: 旧方式（删除线效果，可选）
- highlight: 一句亮点话（主色字体，可选）
- date: 2026.03
```

## 格式规范（必须遵守）

### ⚠️ 最重要的规范：特殊框内不能有空行

```markdown
# ✅ 正确：每行都有 > 前缀，中间无空行
> ✨ 核心要点
> 第一行内容
> 第二行内容

# ❌ 错误：中间有空行
> ✨ 核心要点

> 第一行内容（这行会被当成新块，格式崩坏）
```

### STEP 块格式

```markdown
# ✅ 正确：说明和标题一起在框内
> STEP 01 打开文件
> 找到桌面上的图标，双击打开

# ✅ 也正确：只有标题，无说明
> STEP 01 打开文件
```

### 不要做的事

- ❌ 不要为每个 `##` 标题都加 `> 📖` 解释文字（formatter 不支持，会显示成 > 符号）
- ❌ 不要在章节标题下方生成"📖 本节xxx"、"本节介绍xxx"、"本节说明xxx"等任何概述性文字，原文没有的内容一律不新增
- ❌ 不要改变原文的核心意思
- ❌ 不要过度标记高亮词（每 3 行最多 1 个，一句话最多 1-2 个）
- ❌ 不要把短句强行拆成 STEP
- ❌ 不要为每个章节都凑一个 ✨ 或 💡 框（每章节最多 1 个，原文没有明确结论/提示就不加）
- ❌ ✨ 和 💡 不要在同一章节内同时出现（二选一）

## 完整优化示例

### 原始文档

```markdown
# 用 AI 工具写公众号，我的完整流程

离职后我开始在公众号上写文章。内容不是问题，但排版每次都很费劲。

## 工具选择

我试了好几款 AI 写作工具，最后选了 WorkBuddy。
原因很简单：安装即用，不需要配服务器。

注意：Windows 用户安装路径和 Mac 不同，注意区分。

## 实际使用步骤

第一步：把文章草稿粘贴给 AI
第二步：说"帮我排版"
第三步：预览 HTML 效果
第四步：一键复制，粘贴到公众号编辑器

整个过程不超过 5 分钟。最重要的是：排版风格完全统一，再也不用每篇手动调。
```

# 用 AI 工具写公众号，我的完整流程

## 优化检查清单

完成优化后，逐项检查：

- [ ] **STEP**：所有流程步骤都转为 `> STEP XX` 格式，编号递增，框内无空行
- [ ] **✨/💡**：核心结论和小贴士都用对应框标记，框内无空行
- [ ] **关键词**：专业术语首次出现时用 `==词==` 标记，不过度标记
- [ ] **🤖 框**：代码/命令/Prompt 用指令框标记，框内无空行
- [ ] **## 标题**：每个章节标题用 `##`，formatter 自动渲染为渐变卡片
- [ ] **格式正确**：所有特殊块内部无空行，`> ` 前缀完整
- [ ] **内容完整**：原文没有被删减，核心意思保留
