# 软件著作权登记源代码格式要求

## 一、基本要求

### 1. 源代码提交数量
- **一般要求**：提交源程序的前30页和后30页
- **不足60页**：应当提交全部源代码
- **超过60页**：选取前30页和后30页，中间部分可以省略

### 2. 页面要求
- **每页行数**：不少于50行（不含空行）
- **字体要求**：清晰可辨，建议使用等宽字体（如Consolas、Courier New）
- **字号要求**：建议使用5号或小5号字体
- **颜色要求**：黑色打印，A4纸张

## 二、格式规范

### 1. 页面标识
每页源代码应当包含：
- 页码标识
- 软件名称
- 版本号

示例：
```
=======================================================================
软件名称：{{软件名称}}    版本号：V{{版本号}}    页码：第X页
=======================================================================
```

### 2. 代码格式
```java
// 正确的代码格式示例
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

### 3. 禁止事项
- ❌ 代码模糊不清
- ❌ 使用图片或扫描件
- ❌ 代码行数不足50行/页
- ❌ 包含大量空行或注释行

## 三、文档鉴别材料要求

### 1. 文档鉴别材料是什么
文档鉴别材料是说明软件功能特点、技术特点等的文档材料，通常包括：
- 软件说明书
- 设计文档
- 用户手册
- 操作手册

### 2. 文档格式要求
- **页数要求**：一般不少于10页
- **内容要求**：应当详细描述软件的功能和技术特点
- **格式要求**：A4纸张，清晰的打印件

### 3. 文档内容建议
1. 软件概述（功能、用途）
2. 技术架构说明
3. 主要功能模块介绍
4. 界面截图（如有）
5. 操作说明
6. 技术创新点说明

## 四、提交文件清单

| 文件类型 | 格式要求 | 份数 |
|----------|----------|------|
| 源代码 | 纸质打印件 + 电子件 | 各1份 |
| 说明书 | 纸质打印件 + 电子件 | 各1份 |
| 申请表 | 原件1份 | - |

## 五、常见错误避免

### ❌ 常见错误1：代码行数不足
**问题**：每页代码少于50行
**解决**：确保每页打印不少于50行有效代码

### ❌ 常见错误2：格式混乱
**问题**：代码缩进不一致，格式混乱
**解决**：使用统一的代码格式化工具整理代码

### ❌ 常见错误3：关键代码缺失
**问题**：提交的代码片段无法体现软件的核心功能
**解决**：选择能够体现软件主要功能的代码段提交

### ❌ 常见错误4：说明书过于简单
**问题**：说明书只有1-2页，内容过于简单
**解决**：详细描述软件的功能和技术特点，不少于10页

## 六、电子文件要求

### 1. 源代码电子文件
- 格式：TXT、PDF、ZIP
- 命名：`源程序_{软件名称}_{版本号}.zip`
- 大小：不超过100MB

### 2. 说明书电子文件
- 格式：PDF、DOC/DOCX
- 命名：`说明书_{软件名称}.pdf`
- 分辨率：300dpi以上

## 七、审查注意事项

登记机构审查时重点关注：
1. 源代码是否能够体现软件的原创性
2. 说明书是否完整描述了软件功能
3. 申请信息与提交材料是否一致
4. 材料格式是否符合要求
