# 任务分级与执行管控

## 任务分级表

| 级别 | 定义 | 流程 | 示例 |
|------|------|------|------|
| 🔴 紧急/简单 | 5 分钟内可完成 | 直接派工造院，跳过标准化 | 错别字修复、一行代码修改 |
| 🟡 常规 | 多步任务，有明确标准 | 经略院 → 工造院 → 明鉴院 | 功能模块开发、文档编写 |
| 🟢 复杂 | 大型项目，需多轮迭代 | 经略院多轮评审 → 工造院 → 明鉴院 | 系统架构设计、跨模块重构 |

## 前置检查清单（10 项）

任务启动前，逐项确认：

1. [ ] 任务目标明确，无歧义
2. [ ] 验收标准（DoD）已定义
3. [ ] 任务级别已判定
4. [ ] 所需资源已确认
5. [ ] 依赖项已识别
6. [ ] 风险点已评估
7. [ ] 时间预估合理
8. [ ] 负责人已指定
9. [ ] 知识库已检索（有无现成标准）
10. [ ] 沟通渠道已确认

## 测试强制要求

- 所有工造院交付物必须附带自测报告
- 自测通过率 ≥ 90%
- 自测报告包含：测试范围、测试用例、通过/失败详情、遗留问题

## 执行管控规则

- **里程碑时限**：单个里程碑 ≤ 30 分钟
- **超时预警**：超过时限 2 倍立即上报管理者
- **进度透明**：里程碑完成即汇报，不堆叠
- **变更管控**：任务范围变更需经略院重新评估 DoD

## 失败处理规则

| 失败分类 | 重试上限 | 处理方式 |
|---------|---------|---------|
| 工造院执行失败 | 3 次 | 每次退回需明确修改方向，3 次后升级管理者介入 |
| 明鉴院质检不通过 | 2 次 | 退回工造院返工，2 次后升级管理者重新评估 DoD |
| 整体任务失败 | 3 次 | 经略院重新拆解任务，3 次后上报决策者 |

## 知识复用流程

1. **任务前**：搜索知识库，查找同类任务标准流程
2. **有现成标准**：直接复用，标注来源
3. **无现成标准**：执行任务，完成后产出标准流程文档
4. **任务后**：新标准存入知识库，供后续复用
5. **定期维护**：每季度审视知识库，标记过时标准
