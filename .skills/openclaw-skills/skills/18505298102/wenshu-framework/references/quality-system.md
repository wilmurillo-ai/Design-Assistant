# 质量评分体系

## 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 准确性 | 40% | 内容正确无误，无事实性错误，代码运行通过 |
| 完整性 | 20% | 覆盖所有需求点，无遗漏，边界情况已处理 |
| 规范性 | 20% | 符合编码/文档规范，命名一致，格式统一 |
| 效率 | 10% | 方案简洁，无过度设计，资源使用合理 |
| 创新性 | 10% | 有优化思路，超越基本需求，体现专业水准 |

## 等级定义

| 等级 | 分数 | 含义 | 处置 |
|------|------|------|------|
| S | 95+ | 卓越，超出预期 | 标杆案例，纳入知识库 |
| A | 85-94 | 优秀，完全达标 | 直接交付，表扬团队 |
| B | 70-84 | 良好，基本达标 | 可交付，标注改进建议 |
| C | 60-69 | 及格，有不足 | 限期改进，补充说明 |
| D | <60 | 不合格 | 退回返工，分析原因 |

## 评分操作指南

1. **逐项评分**：按 5 个维度分别打分（0-100）
2. **加权计算**：总分 = 准确性×0.4 + 完整性×0.2 + 规范性×0.2 + 效率×0.1 + 创新性×0.1
3. **等级判定**：根据总分对照等级表
4. **评语编写**：每个维度至少一句评语，说明得分理由
5. **改进建议**：C 级及以下必须提供具体改进方向

### 评分示例

```
质量评分：88/100 (A)
- 准确性 (90/100)：核心逻辑正确，边界情况已处理
- 完整性 (85/100)：覆盖主要需求，次要场景可补充
- 规范性 (90/100)：命名规范，注释清晰
- 效率 (80/100)：方案合理，个别函数可优化
- 创新性 (85/100)：有优化思路，超出基本要求
```

> 评分需客观公正，避免主观偏好影响结果。
