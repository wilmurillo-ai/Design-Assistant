# 违规处理体系

## 违规定义

违规是指 Agent 在执行任务过程中，违反既定规则、流程、安全制度或行为准则的行为。

## 违规类型

| 类型 | 严重程度 | 示例 |
|------|---------|------|
| 安全违规 | 严重 | 泄露密钥、越权访问、数据未脱敏 |
| 流程违规 | 中等 | 跳过质检、未执行前置检查、未按 DoD 交付 |
| 质量违规 | 中等 | 交付物未通过自测、代码存在明显缺陷 |
| 态度违规 | 轻微 | 隐瞒问题、推诿责任、不回应纠正 |

## 处理流程

```
发现违规
    ↓
分类定级（安全/流程/质量/态度）
    ↓
记录违规详情（时间、类型、描述、影响）
    ↓
执行处罚（扣分/警告/限制权限）
    ↓
整改要求（限期修正 + 预防措施）
    ↓
复查确认（整改通过后关闭）
```

## 扣分机制

| 违规类型 | 扣分 | 附加处罚 |
|---------|------|---------|
| 安全违规 | -20 分 | 立即暂停权限，全面审查 |
| 流程违规 | -10 分 | 限期整改，重新培训流程 |
| 质量违规 | -5 分 | 返工至达标，记录教训 |
| 态度违规 | -3 分 | 书面反思，限期改进 |

### 累计处罚

| 累计扣分 | 处罚 |
|---------|------|
| 单次 -20 | 立即暂停，全面审查 |
| 累计 -30 | 限制任务权限 24 小时 |
| 累计 -50 | 暂停任务权限，上报决策者 |

> 扣分定期清零（建议每月），但严重违规记录永久保留。
