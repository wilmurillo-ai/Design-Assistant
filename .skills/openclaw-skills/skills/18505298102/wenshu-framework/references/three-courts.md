# 三院制协作协议

## 组织层级

```
决策者（战略方向）
  └── 管理者（执行管理）
        ├── 经略院（任务拆解 + 标准化 + 制定 DoD）
        ├── 工造院（按标准执行 → 产出交付物）
        └── 明鉴院（独立质检 → 通过/退回）
```

## 三院职责

| 院名 | 职责 | 含义 |
|------|------|------|
| 经略院 | 任务拆解 → 标准化 → 流程化 → 制定验收标准（DoD） | 经略天下，谋定而后动 |
| 工造院 | 按标准和流程执行 → 产出交付物 | 天工开物，巧夺天工 |
| 明鉴院 | 独立检验交付物 → 出具质检报告 → 通过/退回 | 明镜高悬，明察秋毫 |

> 执行与质检必须分离。管理者通过子代理委派三院，不亲力亲为。

## 汇报铁律

1. **事实优先**：汇报基于可验证的事实，不使用模糊表述
2. **数据支撑**：关键结论必须有数据或证据支撑
3. **格式统一**：使用标准汇报模板，确保一致性
4. **及时透明**：发现问题立即上报，不隐瞒不拖延

## 协作流程

```
1. 决策者下达任务需求
       ↓
2. 管理者评估任务级别（紧急/常规/复杂）
       ↓
3. 经略院拆解任务 → 制定 DoD（验收标准）
       ↓
4. 工造院按 DoD 执行 → 产出交付物
       ↓
5. 明鉴院独立质检 → 出具报告
       ↓
6. 通过 → 管理者验证 → 提交决策者
   不通过 → 退回工造院返工 → 重新质检
```

## 知识沉淀机制

- 任务完成 → 产出标准流程文档 → 存入知识库
- 同类任务 → 先查知识库 → 有现成标准直接复用
- 无现成标准 → 新建标准 → 入库
- 定期清理：过时标准标记废弃，不直接删除

## 模型选择策略

| 任务类型 | 推荐模型 | 说明 |
|---------|---------|------|
| 任务拆解（经略院） | 强推理模型 | 需要深度分析能力 |
| 代码生成（工造院） | 强编码模型 | 代码准确率和简洁性优先 |
| 质量检查（明鉴院） | 强推理+编码模型 | 需要发现隐蔽问题 |
| 简单任务 | 轻量模型 | 成本优先，快速响应 |
| 文档编写 | 通用大模型 | 语言流畅度优先 |

> 模型选择应结合成本、速度、准确率综合评估。
