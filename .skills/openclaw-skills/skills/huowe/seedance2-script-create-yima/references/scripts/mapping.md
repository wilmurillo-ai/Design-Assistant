## 脚本模板说明

### 模板命名规则
```
@script_{分类前缀}_{时长}_{版本}
```
- **分类前缀**: 一级分类的缩写（如fmcg、elec、fashion等）
- **时长**: 视频时长（9s、12s、15s、30s）
- **版本**: 脚本版本（v1、v2、v3）

### 模板可用性
| 时长 | 适用场景 | 典型用途 |
|------|----------|----------|
| 9秒 | 短视频平台 | 抖音、快手、小红书 |
| 12秒 | 中等时长 | 品牌广告、产品介绍 |
| 15秒 | 标准广告 | 电视广告、视频前贴片 |
| 30秒 | 详细说明 | 产品演示、功能展示 |

## 映射验证

### 验证规则
1. **完整性**: 每个分类reference必须有至少一个关联脚本
2. **可用性**: 脚本模板必须实际存在
3. **一致性**: 脚本模板的分类必须与reference匹配

### 自动验证脚本
```bash
# 检查所有reference是否有关联脚本
for ref_file in references/categories/**/*.md; do
  if ! grep -q "## 关联脚本" "$ref_file"; then
    echo "错误: $ref_file 缺少关联脚本"
  fi
done
```

## 扩展说明

### 添加新分类
1. 在对应目录创建新的reference文件
2. 在本映射表中添加新的映射关系
3. 创建对应的脚本模板文件
4. 更新验证脚本

### 添加新脚本模板
1. 在templates目录创建新的脚本文件
2. 更新映射表，将模板关联到相应分类
3. 更新reference文件中的关联脚本列表

## 使用示例

### 查询脚本
```markdown
产品分类: FMCG > food
需求时长: 15秒

- @script_fmcg_9s_v1
- @script_fmcg_15s_v1
- @script_fmcg_30s_v1
```

基于分类reference自动推荐脚本：
```yaml

  - @script_fmcg_9s_v1 (推荐用于短视频)

  - @script_fmcg_30s_v1 (推荐用于详细展示)
```
