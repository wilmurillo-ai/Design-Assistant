# 项目宪法模板

> 这是项目工程原则和质量标准的定义文档，所有规范和实现都应遵循此宪法。

## 🎯 核心原则

### 1. 测试驱动开发（TDD）

- **测试先行**：所有功能实现前必须编写测试
- **覆盖率要求**：单元测试 ≥ 70%，总体覆盖率 ≥ 80%
- **测试类型**：包含单元测试、集成测试、契约测试

### 2. 代码质量标准

- **语言规范**：使用严格模式（如 TypeScript strict mode）
- **代码风格**：遵循项目选定的 linter 规则
- **注释要求**：关键逻辑必须有注释，使用中文

### 3. 架构原则

- **关注点分离**：模块清晰分离，单一职责
- **依赖注入**：避免硬编码依赖
- **API 优先**：RESTful API 设计

### 4. 文档规范

- **同步更新**：代码变更时同步更新文档
- **语言**：所有文档使用中文
- **结构**：遵循 speckit 目录结构

### 5. 安全规范

- **身份验证**：所有 API 必须进行身份验证
- **数据加密**：敏感数据必须加密存储
- **最小权限**：信息交互最小化

### 6. 性能要求

- **响应时间**：API 响应时间 < 200ms
- **并发支持**：支持 100+ 并发请求
- **资源限制**：内存使用 < 512MB
