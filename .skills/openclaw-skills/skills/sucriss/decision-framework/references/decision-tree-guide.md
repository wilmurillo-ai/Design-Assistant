# Decision Tree Guide - 决策树指南

## 什么是决策树

决策树是一种图形化的决策分析工具，通过树状结构展示：
- **决策节点** - 你需要做的选择
- **机会节点** - 可能的结果（带概率）
- **终端节点** - 最终收益/损失

## 核心公式

**期望值 (Expected Value) = Σ(概率 × 结果值)**

## 基本结构

```
核心决策？
├── 选项 A
│   ├── 结果 1 (概率 P1, 收益 V1)
│   └── 结果 2 (概率 P2, 收益 V2)
│   期望值：P1×V1 + P2×V2
└── 选项 B
    └── ...
```

## 使用步骤

### Step 1: 定义核心决策

明确你要做什么选择，例如：
- 要不要投资这个创业项目？
- 要不要接受这个工作 offer？
- 要不要现在买房？

### Step 2: 列出所有选项

穷举所有可行选择：
- 投资 / 不投资
- 接受 / 拒绝 / 协商
- 买 / 租 / 再等等

### Step 3: 识别可能的结果

对每个选项，列出可能的结果及其概率：

**示例：投资创业**
```
投资 100 万
├── 成功 (30%) → 退出收益 500 万
├── 持平 (40%) → 收回 100 万
└── 失败 (30%) → 损失 80 万
```

### Step 4: 计算期望值

```
投资期望值 = 0.3×500 + 0.4×100 + 0.3×(-80)
           = 150 + 40 - 24
           = 166 万

不投资期望值 = 0（维持现状）

决策建议：投资（期望值 +166 万）
```

### Step 5: 敏感性分析

测试关键假设变化的影响：

```
如果成功率降到 20%：
期望值 = 0.2×500 + 0.4×100 + 0.4×(-80) = 68 万（仍为正）

如果失败损失增加到 100 万：
期望值 = 0.3×500 + 0.4×100 + 0.3×(-100) = 160 万
```

## 完整示例

### 示例 1：是否辞职创业

```
是否辞职创业？

├── 辞职创业 (投入 50 万积蓄)
│   ├── 大成功 (15%) → 3 年后估值 1000 万
│   ├── 小成功 (35%) → 3 年后年利润 100 万
│   ├── 持平 (30%) → 3 年后收回成本
│   └── 失败 (20%) → 损失 50 万
│   
│   期望值 (3 年)：
│   0.15×1000 + 0.35×300 + 0.30×0 + 0.20×(-50)
│   = 150 + 105 + 0 - 10 = 245 万
│   
│   年化期望值：245/3 ≈ 82 万/年

└── 继续工作
    └── 稳定收入 (100%) → 3 年总收入 90 万 (30 万/年)
        期望值：90 万
        年化：30 万/年

决策建议：创业期望值更高，但风险大
关键考量：风险承受能力、机会成本、个人偏好
```

### 示例 2：是否买房子

```
是否现在买房？(总价 500 万，首付 150 万)

├── 现在买
│   ├── 房价上涨 (40%) → 5 年后涨到 700 万
│   │   收益：200 万 - 利息成本 50 万 = 150 万
│   ├── 房价持平 (40%) → 5 年后仍 500 万
│   │   收益：0 - 利息成本 50 万 = -50 万
│   └── 房价下跌 (20%) → 5 年后跌到 400 万
│       收益：-100 万 - 利息成本 50 万 = -150 万
│   
│   期望值：0.4×150 + 0.4×(-50) + 0.2×(-150)
│          = 60 - 20 - 30 = 10 万

└── 继续租房 (月租 5000)
    └── 5 年租金：30 万
        投资收益 (假设首付 150 万理财年化 4%)：150×(1.04^5-1) ≈ 32 万
        净收益：32 - 30 = 2 万

决策建议：买房期望值略高 (10 万 vs 2 万)，但差距不大
关键考量：居住稳定性、杠杆风险、流动性需求
```

## 多阶段决策树

有些决策是序列性的，需要多阶段分析：

```
第一阶段：要不要做市场调研？(成本 5 万)
├── 做调研
│   ├── 结果积极 (60%) → 进入第二阶段：投资
│   │   ├── 成功 (50%) → 收益 200 万
│   │   └── 失败 (50%) → 损失 100 万
│   └── 结果消极 (40%) → 放弃，损失 5 万
└── 不做调研 → 直接决定投或不投
```

**逆向归纳法 (Backward Induction)：**
从后往前计算每个节点的期望值，选择最优路径。

## 常见陷阱

1. **概率估计偏差** - 过度乐观/悲观
   - 对策：参考历史数据、咨询专家、做敏感性分析

2. **忽略隐性成本** - 时间、机会成本、心理成本
   - 对策：全面列出所有成本项

3. **结果值难以量化** - 幸福感、成就感等
   - 对策：用 1-10 分主观打分，或承认局限性

4. **概率和不为 1** - 遗漏某些结果
   - 对策：确保所有互斥结果概率和=1

5. **混淆相关性和因果** - 错误归因
   - 对策：仔细分析因果链条

## 何时使用决策树

✅ 适合：
- 有明确的风险/不确定性
- 可以估计概率和结果值
- 多阶段序列决策
- 需要量化分析支持

❌ 不适合：
- 纯情感决策（如选择伴侣）
- 无法估计概率的情况
- 结果值完全不可量化
- 需要快速直觉判断

## 与其他框架配合

- **决策树 + SWOT** - 先用 SWOT 识别内外部因素，再用决策树量化风险
- **决策树 + 10-10-10** - 用 10-10-10 评估长期影响，作为决策树的结果值
- **决策树 + 加权打分** - 对多选项先用加权打分筛选，再对 top 选项做决策树分析
