# 项目式学习(PBL)指南

## 项目设计框架

### 项目启动阶段（第1周）

#### 任务1.1：问题选择
**目标**：确定一个真实的、有挑战性的数学建模问题

**活动**：
1. 浏览真实世界案例库，选择感兴趣领域
2. 与导师/同伴讨论问题的可行性和教育价值
3. 明确项目的数学学习目标

**交付物**：项目提案（1页）
- 问题背景与意义
- 涉及的数学知识点
- 预期成果形式

#### 任务1.2：背景调研
**目标**：深入理解问题背景，收集必要信息

**活动**：
1. 阅读相关科普文章、新闻报道
2. 查找学术论文（摘要和引言部分）
3. 访谈领域专家（如可能）

**交付物**：背景调研报告
- 关键概念术语表
- 数据来源清单
- 已有研究综述

#### 任务1.3：问题定义
**目标**：将模糊的现实问题转化为明确的数学问题

**活动**：
1. 界定问题范围（明确包含什么、排除什么）
2. 提出可研究的数学问题
3. 设定成功标准

**交付物**：问题定义文档
- 研究问题陈述
- 假设与简化
- 评价指标

---

### 模型构建阶段（第2-3周）

#### 任务2.1：变量识别
**目标**：识别并定义所有相关变量

**活动**：
1. 头脑风暴列出所有可能影响因素
2. 分类：自变量、因变量、参数、常数
3. 确定变量间的依赖关系

**工具**：变量关系图
```
[输入变量] → [中间变量] → [输出变量]
     ↓              ↓
[参数]         [约束条件]
```

#### 任务2.2：建立关系
**目标**：建立变量间的数学关系

**活动**：
1. 基于物理/经济/生物规律建立方程
2. 使用类比方法借鉴已知模型
3. 数据驱动的关系发现（如适用）

**检查点**：
- [ ] 量纲一致性检查
- [ ] 极端情况验证
- [ ] 与已知特例对比

#### 任务2.3：模型整合
**目标**：形成完整的数学模型

**活动**：
1. 将各部分组成有机整体
2. 明确模型输入输出
3. 编写模型文档

**交付物**：数学模型描述
- 模型假设清单
- 数学方程系统
- 变量定义表

---

### 求解验证阶段（第4-5周）

#### 任务3.1：求解策略
**目标**：选择合适的求解方法

**决策树**：
```
解析解可能？
├── 是 → 尝试解析方法（代数、微积分）
└── 否 → 数值方法
    ├── 常规问题 → 标准算法（牛顿法、Runge-Kutta等）
    └── 复杂问题 → 专业软件（MATLAB、Mathematica）
```

**活动**：
1. 评估解析解可能性
2. 选择数值方法（如需要）
3. 确定计算工具

#### 任务3.2：计算实现
**目标**：获得模型的数值结果

**活动**：
1. 编写计算代码（Python/MATLAB等）
2. 调试与验证代码正确性
3. 运行计算，记录结果

**最佳实践**：
- 模块化编程
- 版本控制（Git）
- 详细注释

#### 任务3.3：结果验证
**目标**：检验模型的合理性和准确性

**验证方法**：
1. **内部验证**：参数敏感性分析
2. **外部验证**：与实验数据/历史数据对比
3. **专家验证**：请领域专家评估

**交付物**：验证报告
- 数值结果
- 与对比基准的差异分析
- 模型可信度评估

---

### 成果展示阶段（第6周）

#### 任务4.1：报告撰写
**目标**：清晰完整地记录项目全过程

**报告结构**：
```markdown
1. 引言
   - 问题背景
   - 研究目标

2. 模型建立
   - 假设与简化
   - 变量定义
   - 数学模型

3. 求解方法
   - 解析/数值方法选择
   - 算法描述

4. 结果与分析
   - 主要发现
   - 敏感性分析
   - 模型局限性

5. 结论与展望
   - 主要结论
   - 实际应用建议
   - 未来改进方向

6. 参考文献
7. 附录（代码、数据）
```

#### 任务4.2：演示准备
**目标**：有效传达项目成果

**演示结构**（15-20分钟）：
1. 问题引入（2分钟）：为什么重要？
2. 方法概述（3分钟）：如何解决？
3. 核心结果（5分钟）：发现了什么？
4. 演示互动（5分钟）：可视化展示
5. 问答讨论（5分钟）：回应质疑

**可视化建议**：
- 使用图表代替表格
- 动画展示动态过程
- 交互式演示（如可能）

#### 任务4.3：同行评议
**目标**：通过互评提升质量

**评议维度**：
1. **问题定义**（20%）：是否清晰？是否有意义？
2. **模型合理性**（30%）：假设是否恰当？数学是否正确？
3. **求解质量**（20%）：方法是否合适？结果是否可靠？
4. **展示效果**（20%）：表达是否清晰？可视化是否有效？
5. **创新性**（10%）：是否有独特见解？

**评议表**：

## 常见问题解答

**Q: 项目太难，无法完成怎么办？**
A: 及时与导师沟通，调整问题范围。记住：完成比完美更重要。

**Q: 找不到合适的数据怎么办？**
A: 可以使用模拟数据，但要明确说明。或选择理论性更强的项目。

**Q: 数学工具不够怎么办？**
A: 这是学习的机会。可以自学、求助同学或调整方法。

**Q: 如何平衡项目与课程学习？**
A: 制定详细时间表，将项目工作分散到每周，避免最后突击。

## 附录：项目选题示例

### 示例1：校园共享单车调度优化
- **背景**：校园内共享单车分布不均，高峰期供需矛盾
- **数学工具**：图论、优化理论、概率统计
- **数据**：骑行记录、停车点位置、时间分布

### 示例2：食堂排队系统分析
- **背景**：午餐高峰期排队时间过长
- **数学工具**：排队论、随机过程、优化
- **数据**：到达率、服务时间、队列长度

### 示例3：图书馆座位预约系统评估
- **背景**：预约系统存在违约、资源浪费现象
- **数学工具**：博弈论、机制设计、统计分析
- **数据**：预约记录、实际使用情况、用户调查
