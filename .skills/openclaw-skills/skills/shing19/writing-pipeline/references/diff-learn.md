# Diff-and-Learn 流程

AI 直接分析 draft vs final 的差异，提取写作偏好规律，提议更新对应的 writing-style skill。

---

## 触发条件

用户完成手动改稿，`final.md` 已就绪。

## 流程

### Step 1: 段落对齐

按段落对齐 `draft-v1.md` 和 `final.md`：
- 识别对应段落（基于位置和内容相似度）
- 标记新增段落、删除段落、修改段落
- 对于大幅重写的段落，标注为"重写"而非逐字对比

### Step 2: 分类修改

将每处修改归入以下 8 种类型：

| 类型 | 代码 | 说明 | 示例 |
|------|------|------|------|
| 用词替换 | `WORD` | 换了一个词 | "深耕" → "做了十几年" |
| 句式调整 | `SYNTAX` | 句子结构变了 | 长句拆短句、被动改主动 |
| 删除冗余 | `DELETE` | 删掉了内容 | 删掉"说实话"开头 |
| 新增内容 | `ADD` | 加了原稿没有的 | 补充数据、个人观点 |
| 语气调整 | `TONE` | 同样的意思换了语气 | 正式 → 口语 |
| 结构调整 | `STRUCT` | 段落顺序或结构变了 | 合并段落、调换顺序 |
| 标点格式 | `FORMAT` | 标点、空格、格式 | 中文标点统一 |
| 事实修正 | `FACT` | 修正了错误信息 | 数字纠正、名称纠正 |

### Step 3: 提取规律

对分类后的修改进行统计分析：

- **频率统计**：同类修改出现 3 次以上 = 一条规律
- **方向判断**：修改是朝哪个方向走的？（更口语/更精确/更简洁）
- **模式识别**：是否有"每次都删 X""每次都把 A 改成 B"的模式

**规律提取模板：**

```
规律：[描述]
证据：
  - 修改1: "A" → "B"
  - 修改2: "C" → "D"
  - 修改3: "E" → "F"
频率：X 次 / 全文 Y 个段落
方向：[更口语/更精确/更简洁/更有观点/...]
建议动作：[加入禁止清单 / 更新风格要点 / 新增特殊规则]
```

### Step 4: 生成 diff-analysis.md

输出结构：

```markdown
# Diff Analysis: [文章标题]

## 概览
- 总段落数：X
- 修改段落数：Y（Z%）
- 主要修改类型：WORD(N), SYNTAX(N), DELETE(N)...

## 修改详情

### 用词替换 (WORD)
| 原文 | 改后 | 分析 |
|------|------|------|
| ... | ... | ... |

### 句式调整 (SYNTAX)
...

（每种类型一个表格）

## 提取的规律

### 规律 1: [名称]
- 描述：...
- 证据：[列出 3+ 个具体修改]
- 建议动作：...

### 规律 2: [名称]
...

## 建议的 Skill 更新

### 禁止清单新增
- ...

### 风格要点调整
- ...

### 新增规则
- ...
```

### Step 5: 提议 Skill 更新

基于 diff-analysis.md 中的"建议的 Skill 更新"部分：

1. 读取当前使用的 writing-style skill（从 README.md 的 `style_skill` 字段获取）
2. 列出建议的修改点，向用户逐条确认
3. 用户确认后，执行对应 skill 文件的更新
4. 更新后检查是否有自相矛盾的规则，如有则提请用户裁决

---

## 规则归纳

当一个 writing-style skill 的规则积累过多时（禁止清单超过 20 条，或风格要点超过 10 条），执行归纳：

1. 检查是否有重复或可合并的规则
2. 将具体规则抽象为更高层次的原则
3. 删除已被更高层原则覆盖的具体规则
4. 保留禁止清单中的具体示例（因为 AI 看例子比看规则学得快）

---

## 注意事项

- diff-learn 的核心是**从修改中学习偏好**，不是评判对错
- 3 次以上才算规律，1-2 次可能是个案
- 事实修正（FACT）不纳入偏好学习，只做记录
- 每次 skill 更新后，建议用同一篇材料重新生成一段，验证更新效果
