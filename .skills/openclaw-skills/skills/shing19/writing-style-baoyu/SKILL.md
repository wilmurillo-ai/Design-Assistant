# 宝玉写作风格

## 核心风格要点

### 1. 比喻贯穿

用一个核心比喻串联全文，让抽象概念变得可感。比喻不是点缀，是文章的骨架。

- 正确：整篇文章用"菜谱/厨子/底味"比喻写作风格，从头到尾一致
- 错误：每段换一个比喻，读者跟不上

**方法**：在构思阶段就确定核心比喻，然后围绕它展开所有论点。好的比喻能让读者"哦，我懂了"。

### 2. 先破后立

先指出常见做法的问题（破），再给出自己的方案（立）。制造认知冲突，让读者意识到"我之前想错了"。

- 正确："所有去 AI 味提示词都是错的" → 然后解释为什么，再给出替代方案
- 错误：直接给方案，读者不知道为什么需要这个方案

**方法**：开头用一个反常识观点吸引注意力，接下来 2-3 段证明为什么旧方法不行，然后才亮出你的方案。

### 3. 编号递进

使用【1】【2】【3】章节式结构，每章解决一个子问题。整体从"为什么"到"是什么"到"怎么做"递进。

- 典型结构：问题 →【1】为什么旧方法不行 →【2】本质是什么 →【3】新方案 →【4】具体步骤 →【5】模板 →【6】组合应用 →【7】总结行动
- 每章标题既是编号也是内容提示

### 4. 对比示例

每个原则都给"改前/改后"示例。AI 看例子比看规则学得快，读者也一样。

- 正确：AI 写 → "作为一名深耕 AI 领域多年的老兵" / 改后 → "作为一个做了十几年 AI 的人"
- 附上分析：不用军事化用语，不用"深耕"，具体年数比"多年"更好

**方法**：每条规则至少配一组正反例，分析差异背后的原因。

### 5. 实操落地

从理论到"你现在就能做的第一步"。文末必须有明确的行动指引，不能停留在"希望对你有帮助"。

- 正确："花一个小时，写一份你自己的菜谱。先把角色定义和忌口表写出来，然后让 AI 做一道菜，尝一口，提意见，更新菜谱。"
- 错误："希望这篇文章能给你带来一些启发"

### 6. 开头亮底牌

用 blockquote 在文章开头给出全文核心判断。不铺垫，不藏着掖着，先让读者知道你的立场。

- 正确：开篇用 `>` 引用一段核心观点，作为全文主旨提纲
- 错误：开头绕三段才说到重点

**方法**：写完全文后回头提炼一句最核心的判断，放在标题之后、正文之前。结尾可以回扣这段引言。

### 7. 结尾旗帜鲜明

结尾必须有明确的单一主张。不要"两种方式都行"的中立收尾，不要两头讨好。

- 正确："用不好 AI，是因为你还没有梭哈。"——明确告诉读者你怎么想
- 错误："两种方式都行。最怕的是……"——看似有态度，实际在和稀泥

**方法**：结尾注入"我"的视角——从介绍他人方法升级为表达自己的判断。让读者带走的不只是信息，还有你的立场。

## 禁止清单

以下表达方式**绝对禁止**：

- **废话开场白**："在当今数字化时代…""随着 AI 技术的飞速发展…"
- **总结套话**："综上所述""总而言之""希望对你有帮助"
- **AI 味句式**："不是…而是…""值得注意的是""需要指出的是"
- **模糊形容**："大幅提升""显著改善""非常优秀"
- **商业黑话**："赋能""闭环""抓手""深耕""底层逻辑"
- **强调拐棍**："说实话""不得不说""有一说一"
- **学术腔**："本文旨在探讨…""笔者认为""从某种意义上来说"
- **首先/其次/最后** 三段式 → 改为编号递进或自然过渡

## 文章结构模板

```
标题（反常识或引发好奇）

> 核心判断（blockquote 亮底牌，一两句话点明全文立场）

引子：常见现象 + 反常识观点（先破）

【1】为什么旧方法不行
    - 3 个原因，每个 2-3 句
    - 用读者熟悉的场景举例

【2】问题的本质是什么
    - 引入核心比喻
    - 用比喻解释问题根源

【3】新方案是什么
    - 比喻延伸：从"提示词"到"菜谱"
    - 对比旧方案 vs 新方案

【4】具体怎么做（分步）
    - 第一步：...
    - 第二步：...（含改前/改后示例）
    - 第三步：...
    - 第四步：...

【5】模板/框架（可选）
    - 给出可复用的结构
    - 每部分 2-3 句说明

【6】组合应用/延伸场景（可选）
    - 核心方法的其他用法
    - 用比喻延伸："一份底味，百道菜"

【7】行动号召
    - 注入"我"的判断，旗帜鲜明
    - 回扣开头 blockquote
    - 一句话收尾 + 明确的第一步行动
```

## 菜谱迭代方法论

这是宝玉风格的元方法论——写作风格本身如何进化：

1. **每次人工改稿都是训练数据**：draft vs final 的差异就是偏好信号
2. **AI 分析差异、提取规律**：3 次以上相同修改 = 一条新规则
3. **规则更新到 skill 中**：禁止清单越来越长，风格要点越来越准
4. **定期归纳精简**：规则多了让 AI 归纳合并，保持简洁

---

## 使用指南

1. 确认文章类型属于方法论/教程/观点输出
2. 确定核心比喻（构思阶段最重要的决策）
3. 先写"破"的部分——旧方法有什么问题
4. 再写"立"的部分——新方案 + 具体步骤
5. 每个原则配正反例
6. 结尾给出"现在就能做的第一步"
7. 自检禁止清单
