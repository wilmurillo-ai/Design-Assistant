> 这是去敏后的示例，展示如何记录错误。

## 2026-03-11

### 错误 #003 - 浏览器工具超时
- **时间**: 12:00-12:03
- **任务**: 访问 GitHub 查看开源项目
- **错误**: `browser.snapshot` 超时，多次重试失败
- **原因**: 服务器环境浏览器可能未正确配置或网络问题
- **修正**: 改用 `curl` + GitHub API 获取项目信息
- **模式标签**: #浏览器不可用 #降级方案
- **相似错误**: 无

---

### 错误 #002 - git clone 超时
- **时间**: 11:51-11:58
- **任务**: 克隆开源项目仓库
- **错误**: `git clone` 命令多次超时被 SIGTERM
- **原因**: 网络慢，默认超时时间不足
- **修正**: 改用 `curl` 下载单个文件，或增加 `yieldMs` 到 60s+
- **模式标签**: #网络超时 #大文件下载
- **相似错误**: 无

---

### 错误 #001 - 内容发布缺少图片
- **时间**: 11:58
- **任务**: 帮用户发布图文内容
- **错误**: 用户未提供图片，无法发布
- **原因**: 平台发图文必须有图片，但未提前检查
- **修正**: 发布前必须验证图片/视频是否存在
- **模式标签**: #文件校验 #发布前检查
- **相似错误**: 无
