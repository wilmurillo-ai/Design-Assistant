一个魂们！你的粉丝牌亲密度还在手动挂机涨吗？

我把之前那个 A-SOUL 自动签到工具升级了一波（v4.0），这次是真·大更新：

【核心更新：亲密度终于能自动涨了】

之前版本只能自动签到发弹幕，亲密度还得自己开直播间挂。

这次直接搞定了 B站最新的移动端心跳协议，成员开播后全自动：
发弹幕点亮牌子 → 心跳挂机涨亲密度 → 下播自动停

实测效果：30/30 亲密度直接拉满，粉丝牌直接升级

每个成员每天 30 亲密度上限，5 分钟涨 6 点，挂 25 分钟就满了。你甚至不需要打开 B站。

【和之前版本比有什么不一样】

v3.0：心跳协议过期了，看起来成功实际上不涨亲密度（B站偷偷改了接口）
v4.0：换成了最新的移动端心跳协议，纯 Python 实现，不需要装 Node.js 也不需要额外服务

简单说就是：以前的版本白挂了，现在这个是真挂真涨

----

【完整功能一览】

自动的：
- 每 30 分钟检测一次成员是否开播
- 开播了自动发 10 条弹幕点亮粉丝牌
- 自动心跳挂机涨亲密度（每天 30 满额）
- 下播了自动停止 + Discord/微信通知
- 每 2 天自动给新视频点赞

手动也行：
- python3 heartbeat.py --check-only（看谁在播）
- python3 heartbeat.py --members 贝拉（指定成员挂机）
- python3 checkin.py --live-only（只发弹幕点亮）

----

【怎么用】

方式一：纯 GitHub Actions（不涨亲密度，只做视频/动态点赞）
→ Fork 仓库 + 填 Cookie 就行，教程在 README 里

方式二：搭配 OpenClaw 使用（全自动涨亲密度 + 点亮牌子）
→ 安装 OpenClaw 后告诉它：
「帮我设置一个定时任务，每 30 分钟检测 A-SOUL 成员是否在直播，如果在播就帮我挂机涨亲密度并点亮粉丝牌」

不需要你的电脑一直开着，OpenClaw 会在后台帮你盯着。

----

【技术细节（给感兴趣的同学）】

B 站 2020 年的 x25Kn 心跳协议虽然还能调通（返回成功），但服务端已经不计亲密度了。

v4.0 使用 mobileHeartBeat 端点，签名算法是 5 层 hash 链：
sha512 → sha3_512 → sha384 → sha3_384 → blake2b

纯 Python 标准库实现，不需要 Node.js 签名服务，不需要 pm2，不需要 Docker。

一个 Python 文件搞定一切。
