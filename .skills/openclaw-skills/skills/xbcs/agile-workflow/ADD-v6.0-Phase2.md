## 2. 系统架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                  Web 仪表盘 (前端)                        │
│  Vue.js 3 + ECharts + WebSocket                         │
│  ├─ 系统概览  ├─ 任务进度  ├─ Agent 状态  ├─ 告警中心    │
└────────────────────┬────────────────────────────────────┘
                     │ WebSocket (实时推送)
┌────────────────────▼────────────────────────────────────┐
│                  API 服务器 (后端)                        │
│  Express + Socket.IO                                    │
│  ├─ REST API  ├─ WebSocket  ├─ 数据聚合  ├─ 权限控制    │
└────────────────────┬────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────┐
│                  数据层                                  │
│  SQLite (状态存储) + 内存缓存 (实时数据)                 │
│  ├─ 任务数据  ├─ Agent 数据  ├─ 资源数据  ├─ 质量数据    │
└─────────────────────────────────────────────────────────┘
```

### 2.2 数据流

```
模块状态 → 数据采集 → 数据处理 → 数据存储 → WebSocket 推送 → 前端更新
   ↓          ↓          ↓          ↓          ↓           ↓
 质量验证   定时采集   聚合过滤   SQLite    实时推送    仪表盘刷新
 创造性评分   1 秒/次   汇总平均   内存缓存   事件驱动    响应式更新
 Token 管理
 Agent 管理
 熔断恢复
```

## 5. API 设计

### 5.1 REST API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/overview` | GET | 获取系统概览 |
| `/api/tasks` | GET | 获取任务列表 |
| `/api/tasks/:id` | GET | 获取任务详情 |
| `/api/agents` | GET | 获取 Agent 列表 |
| `/api/agents/:name` | GET | 获取 Agent 详情 |
| `/api/agents/:name/start` | POST | 启动 Agent |
| `/api/agents/:name/stop` | POST | 停止 Agent |
| `/api/alerts` | GET | 获取告警列表 |
| `/api/metrics` | GET | 获取性能指标 |

### 5.2 WebSocket 事件

| 事件 | 方向 | 说明 |
|------|------|------|
| `connect` | 客户端→服务器 | 连接建立 |
| `disconnect` | 客户端→服务器 | 连接断开 |
| `metrics:update` | 服务器→客户端 | 指标更新 |
| `task:update` | 服务器→客户端 | 任务状态更新 |
| `agent:update` | 服务器→客户端 | Agent 状态更新 |
| `alert:new` | 服务器→客户端 | 新告警通知 |
