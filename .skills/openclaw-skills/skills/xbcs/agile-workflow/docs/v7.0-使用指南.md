## 🎯 核心问题与解决方案

### 问题
> Agent 并发是否会造成数据污染？如何通过新建 Agent 进行并发并避免数据依赖？

### 第一性原理答案
**数据污染不来自大模型本身，而来自共享存储层的并发写入**

- ✅ 大模型是 stateless 的，每次 API 调用完全独立
- ❌ 污染根源 = 多个 Agent 同时修改同一文件/变量
- ✅ 解决方案 = **写入隔离 + 依赖显式化 + 策略化合并**

## 📦 核心模块

### 1. WriteDomainIsolator - 写入域隔离管理器

**职责**: 为每个 Agent 分配独立写入域，防止越界访问

```javascript
const WriteDomainIsolator = require('./write-domain-isolator');

const isolator = new WriteDomainIsolator({
  baseDir: '/tmp/agents'
});

// 为任务创建隔离域
const domainPath = isolator.createDomain('task-001', {
  ttl: 3600000,      // 1 小时后过期
  quota: 104857600   // 100MB 配额
});

// 安全写入（自动验证边界）
isolator.safeWrite('task-001', 
  path.join(domainPath, 'output.txt'), 
  'Hello World'
);

// 越界写入会被阻止
try {
  isolator.safeWrite('task-001', '/tmp/outside.txt', 'Fail');
} catch (error) {
  console.log('写入越界被阻止:', error.message);
}
```

**核心功能**:
- ✅ 自动创建隔离域（带唯一哈希前缀）
- ✅ 写入边界验证
- ✅ 配额管理（文件大小 + 总容量）
- ✅ TTL 过期自动清理
- ✅ 域统计（文件数、总大小）

### 2. DependencyGraphManager - 依赖图管理器

**职责**: 管理任务依赖关系，确保正确执行顺序

```javascript
const DependencyGraphManager = require('./dependency-graph-manager');

const graph = new DependencyGraphManager({
  maxDepth: 50,        // 最大依赖深度
  timeoutMs: 300000,   // 默认超时 5 分钟
  allowCycles: false   // 禁止环路
});

// 添加任务（声明依赖）
graph.addTask('世界观', [], { priority: 1 });
graph.addTask('人物设定', ['世界观'], { priority: 2 });
graph.addTask('大纲设计', ['人物设定'], { priority: 3 });

// 检查任务是否可执行
console.log(graph.canExecute('大纲设计')); // false（依赖未完成）

// 获取可并行执行的任务
console.log(graph.getExecutableTasks()); // ['世界观']

// 获取分层执行计划（用于并行优化）
const layers = graph.getLayeredExecutionPlan();
// [['世界观'], ['人物设定'], ['大纲设计']]

// 标记任务完成
graph.markCompleted('世界观', { result: '完成' });
console.log(graph.canExecute('人物设定')); // true
```

**核心功能**:
- ✅ DAG 依赖图管理
- ✅ 自动环路检测
- ✅ 拓扑排序
- ✅ 分层执行计划（最大化并行）
- ✅ 关键路径分析
- ✅ 执行历史追踪

### 3. MergeStrategyManager - 合并策略管理器

**职责**: 定义和应用合并策略，处理并发结果整合

```javascript
const MergeStrategyManager = require('./merge-strategy-manager');

const merger = new MergeStrategyManager();

// 查看可用策略
console.log(merger.getAvailableStrategies());

// 策略 1: 追加合并（适合列表、日志）
const appendResult = merger.applyStrategy('append', [
  [1, 2, 3],
  [4, 5, 6]
]);
// merged: [1, 2, 3, 4, 5, 6]

// 策略 2: 覆盖合并（适合配置、状态）
const overwriteResult = merger.applyStrategy('overwrite', [
  { a: 1, b: 2 },
  { a: 10, c: 3 }
]);
// merged: { a: 10, b: 2, c: 3 }
// conflicts: [{ key: 'a', values: [1, 10], ... }]

// 策略 3: 智能合并（适合文档、代码）
const smartResult = merger.applyStrategy('smart', [
  { name: 'Alice', age: 25 },
  { name: 'Bob', age: 30 }
]);
// 检测到冲突，标记需要人工审核

// 策略 4: 聚合合并（适合统计、分析）
const aggResult = merger.applyStrategy('aggregate', [
  [10, 20, 30],
  [40, 50]
], { operation: 'avg' });
// merged: 30 (平均值)

// 策略 5: 深度合并（适合嵌套对象）
const deepResult = merger.applyStrategy('deep', [
  { a: { b: 1 } },
  { a: { c: 2 } }
]);
// merged: { a: { b: 1, c: 2 } }
```

**7 种合并策略**:
| 策略 | 适用场景 | 冲突处理 |
|------|----------|----------|
| append | 列表、日志、章节 | 无冲突 |
| overwrite | 配置、状态 | 最后写入胜出 |
| smart | 文档、代码 | 标记人工审核 |
| aggregate | 统计、数值 | 数学运算 |
| deep | 嵌套配置 | 递归合并 |
| custom | 特殊需求 | 自定义函数 |
| file | 文件合并 | 拼接/覆盖 |

### 4. ConcurrentExecutor v2.0 - 并发执行器

**职责**: 整合所有模块，安全地并发执行任务

```javascript
const ConcurrentExecutor = require('./concurrent-executor-v2');

const executor = new ConcurrentExecutor({
  workspace: '/tmp/my-workspace',
  maxConcurrency: 3,    // 最大并发数
  timeoutMs: 300000,    // 默认超时
  retryCount: 2,        // 失败重试次数
  cleanupOnComplete: true
});

// 事件监听
executor.on('task:submitted', (e) => console.log('提交:', e.taskId));
executor.on('task:start', (e) => console.log('开始:', e.taskId));
executor.on('task:complete', (e) => console.log(`完成：${e.taskId} (${e.duration}ms)`));
executor.on('task:error', (e) => console.error(`失败：${e.taskId} - ${e.error}`));
executor.on('execution:complete', (summary) => {
  console.log('执行完成:', summary);
});

// 提交无依赖任务（完全并行）
executor.submitTask('chapter-1', async (domainPath, context) => {
  // 在隔离域内写入
  fs.writeFileSync(path.join(domainPath, 'content.md'), '# 第一章');
  return { chapter: 1, words: 3000 };
});

executor.submitTask('chapter-2', async (domainPath) => {
  fs.writeFileSync(path.join(domainPath, 'content.md'), '# 第二章');
  return { chapter: 2, words: 3200 };
});

// 提交有依赖任务
executor.submitTask('summary', async (domainPath, context) => {
  // 访问依赖结果
  const ch1 = context.dependencies['chapter-1'];
  const ch2 = context.dependencies['chapter-2'];
  
  const totalWords = ch1.words + ch2.words;
  return { totalChapters: 2, totalWords };
}, ['chapter-1', 'chapter-2']); // 依赖声明

// 执行所有任务
const summary = await executor.executeAll();
console.log('完成:', summary.completedTasks.length, '个任务');

// 合并结果
const merged = executor.mergeResults(
  ['chapter-1', 'chapter-2'],
  'append'
);
console.log('合并结果:', merged.merged);
```

## 🚀 完整示例：小说创作并发工作流

### 场景：同时创作 5 个章节，然后汇总

```javascript
const ConcurrentExecutor = require('./concurrent-executor-v2');
const fs = require('fs');
const path = require('path');

async function createNovelChapters() {
  const executor = new ConcurrentExecutor({
    workspace: '/tmp/novel-workspace',
    maxConcurrency: 5  // 5 个章节并行
  });

  // 提交 5 个章节创作任务（完全并行）
  for (let i = 1; i <= 5; i++) {
    executor.submitTask(`chapter-${i}`, async (domainPath) => {
      console.log(`开始创作第${i}章，隔离域：${domainPath}`);
      
      // 模拟创作过程（在隔离域内写入）
      const content = `# 第${i}章\n\n这里是第${i}章的内容...\n`;
      fs.writeFileSync(path.join(domainPath, 'chapter.md'), content);
      
      // 返回章节元数据
      return {
        chapter: i,
        wordCount: content.length,
        filePath: path.join(domainPath, 'chapter.md'),
        createdAt: Date.now()
      };
    });
  }

  // 提交汇总任务（依赖所有章节）
  executor.submitTask('summary', async (domainPath, context) => {
    const chapters = Object.values(context.dependencies);
    const totalWords = chapters.reduce((sum, ch) => sum + ch.wordCount, 0);
    
    const summaryContent = `# 小说汇总\n\n- 总章节：${chapters.length}\n- 总字数：${totalWords}\n`;
    fs.writeFileSync(path.join(domainPath, 'summary.md'), summaryContent);
    
    return {
      totalChapters: chapters.length,
      totalWords,
      avgWordsPerChapter: Math.round(totalWords / chapters.length)
    };
  }, ['chapter-1', 'chapter-2', 'chapter-3', 'chapter-4', 'chapter-5']);

  // 执行
  const result = await executor.executeAll();
  
  console.log('执行完成:');
  console.log(`  总任务：${result.totalTasks}`);
  console.log(`  完成：${result.completedTasks.length}`);
  console.log(`  失败：${result.failedTasks.length}`);
  console.log(`  耗时：${result.duration}ms`);
  
  // 合并所有章节
  const merged = executor.mergeResults(
    ['chapter-1', 'chapter-2', 'chapter-3', 'chapter-4', 'chapter-5'],
    'append'
  );
  
  console.log('合并结果:', merged.merged);
  
  // 导出报告
  executor.exportReport('/tmp/novel-workspace/report.json');
  
  return result;
}

// 运行
createNovelChapters().catch(console.error);
```

## 🎯 最佳实践

### 1. 选择合适的并发数
```javascript
// CPU 密集型任务：并发数 = CPU 核心数
const executor = new ConcurrentExecutor({ maxConcurrency: 4 });

// I/O 密集型任务：可以更高
const executor = new ConcurrentExecutor({ maxConcurrency: 10 });

// 大模型 API 调用：考虑速率限制
const executor = new ConcurrentExecutor({ maxConcurrency: 3 });
```

### 2. 合理声明依赖
```javascript
// ❌ 错误：隐式依赖
executor.submitTask('B', async () => {
  // 假设 A 已经完成
});

// ✅ 正确：显式声明依赖
executor.submitTask('B', async (domainPath, context) => {
  const aResult = context.dependencies['A'];
}, ['A']);
```

### 3. 选择合适的合并策略
```javascript
// 章节创作 → append
executor.mergeResults(chapterIds, 'append');

// 配置生成 → overwrite
executor.mergeResults(configIds, 'overwrite');

// 数据分析 → aggregate
executor.mergeResults(analysisIds, 'aggregate', { operation: 'avg' });

// 复杂文档 → smart（检测冲突）
executor.mergeResults(docIds, 'smart');
```

### 4. 错误处理与重试
```javascript
executor.submitTask('flaky-task', async () => {
  // 可能失败的任务
}, [], {
  retryCount: 3,     // 重试 3 次
  timeout: 60000     // 60 秒超时
});
```

### 5. 资源清理
```javascript
// 自动清理
const executor = new ConcurrentExecutor({ cleanupOnComplete: true });

// 手动清理
await executor.executeAll();
executor.cleanup();
```
