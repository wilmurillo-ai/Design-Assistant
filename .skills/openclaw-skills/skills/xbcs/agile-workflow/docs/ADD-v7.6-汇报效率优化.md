# ADD v7.6: 汇报效率优化 - 去重原则

**版本**: v7.6.0  
**日期**: 2026-03-13  
**问题**: 暂停项目重复汇报、已完成任务重复汇报  
**根因**: 汇报逻辑未区分状态变化 vs 状态保持

---

## 🎯 第一性原理分析

### 报告的本质
```
报告 = 传递有价值的信息

价值判断：
- ✅ 新信息 = 有价值
- ✅ 状态变化 = 有价值
- ❌ 重复信息 = 无价值
- ❌ 状态保持 = 低价值

核心原则：
只汇报变化的、重要的、需要关注的信息
```

### 问题分析

**当前问题**:
1. ❌ 暂停项目每次都汇报（状态未变）
2. ❌ 已完成任务每次都汇报（状态未变）
3. ❌ 报告冗长，重点不突出

**正确做法**:
1. ✅ 暂停项目只在暂停/恢复时汇报
2. ✅ 已完成任务只在本次完成时汇报
3. ✅ 聚焦当前进行中的任务

---

## 📐 MECE 拆解

### 维度 1: 项目状态

| 状态 | 汇报频率 | 说明 |
|------|----------|------|
| **活跃** | 每次汇报 | 有任务进行中 |
| **暂停** | 状态变化时 | 只在暂停/恢复时汇报 |
| **完成** | 完成时一次 | 完成后不再汇报 |

### 维度 2: 任务状态

| 状态 | 汇报频率 | 说明 |
|------|----------|------|
| **进行中** | 每次汇报 | 当前正在执行 |
| **本次完成** | 汇报一次 | 本次会话新完成 |
| **历史完成** | 不汇报 | 之前会话已完成 |
| **待执行** | 不汇报 | 未开始 |

### 维度 3: 汇报内容优先级

| 优先级 | 内容 | 汇报频率 |
|--------|------|----------|
| **P0** | 当前执行中任务 | 每次 |
| **P1** | 本次新完成任务 | 一次 |
| **P2** | 状态变化（暂停/恢复） | 变化时 |
| **P3** | 历史完成/暂停项目 | 不汇报 |

---

## 🏗️ 汇报原则

### 原则 1: 活跃项目优先

```
只汇报活跃项目（有任务进行中）
暂停项目不汇报（除非状态变化）
```

### 原则 2: 增量汇报

```
只汇报本次新完成的任务
历史已完成任务不重复汇报
```

### 原则 3: 聚焦进行中

```
重点汇报当前正在执行的任务
待执行任务简略提及
```

### 原则 4: 状态变化通知

```
项目暂停/恢复时汇报
任务状态变化时汇报
状态保持时不汇报
```

### 方案 3: 更新汇报逻辑

**伪代码**:
```javascript
function generateReport() {
  const projects = loadProjects();
  const state = loadState();
  
  // 只汇报活跃项目
  const activeProjects = projects.filter(p => p.status === 'active');
  
  // 只汇报本次新完成任务
  const newCompleted = getNewCompletedSinceLastReport();
  
  // 检查状态变化
  const statusChanges = checkStatusChanges(state);
  
  report = {
    activeProjects,      // P0
    newCompleted,        // P1
    statusChanges,       // P2
    // 不汇报：历史完成、暂停项目（无变化）
  };
  
  return report;
}
```
