## 🎯 核心问题

### 问题背景

**v5.1 问题**：Agent 熔断后工作中断，需要人工干预恢复

```
常见熔断场景:
1. 连续失败 5 次→熔断，工作中断
2. 执行超时→熔断，任务卡住
3. 资源耗尽→熔断，无法恢复
4. 心跳丢失→熔断，需要手动重启

结果：
- 工作中断率：30%
- 平均恢复时间：30 分钟 +
- 人工干预：每次熔断都需要
```

### v5.2 解决方案

**自动熔断恢复闭环**：
```
正常 → 预警 → 半熔断 → 全熔断 → 冷却 → 探测 → 恢复
              ↓          ↓          ↓         ↓
           降级处理   拒绝请求   自动恢复   渐进放量
```

## 📊 核心功能

### 1. 熔断检测（增强）

**检测维度**：
| 维度 | 阈值 | 动作 |
|------|------|------|
| **连续失败** | 5 次 | 半熔断 |
| **错误率** | > 50% | 预警 |
| **执行超时** | 3 次 | 熔断 |
| **资源耗尽** | 内存>90% | 熔断 |
| **心跳丢失** | 90 秒 | 熔断 |

**错误分类**：
- TIMEOUT（超时）
- RESOURCE（资源）
- HEALTH（健康）
- DEPENDENCY（依赖）
- BUSINESS（业务）

### 2. 熔断状态机（新增）

**六状态流转**：
```
正常 (NORMAL) ←→ 预警 (WARNING) ←→ 半熔断 (HALF_OPEN)
                                  ↓
                              全熔断 (OPEN)
                                  ↓
                              恢复中 (RECOVERING)
```

| 状态 | 请求处理 | 持续时间 | 转换条件 |
|------|----------|----------|----------|
| **正常** | 全部接受 | - | 错误率 < 阈值 |
| **预警** | 接受 + 降级 | 5 分钟 | 错误率 > 50% 阈值 |
| **半熔断** | 50% 接受 | 2 分钟 | 连续失败 5 次 |
| **全熔断** | 全部拒绝 | 5 分钟 | 连续失败 10 次 |
| **恢复中** | 10% 接受 | 1 分钟 | 冷却期结束 |

### 3. 自动恢复（新增）

**恢复流程**：
```
全熔断 → 冷却期（5 分钟）→ 探测请求 → 渐进恢复 → 正常
                      ↓
                  10% → 30% → 60% → 100%
```

**恢复策略**：
| 错误类型 | 恢复策略 | 预计时间 |
|----------|----------|----------|
| **超时** | 增加超时 + 重试 | 8 分钟 |
| **资源** | 清理 + 重启 + 监控 | 11 分钟 |
| **健康** | 重启 + 验证 | 8 分钟 |
| **默认** | 冷却 + 探测 + 渐进 | 11 分钟 |

### 4. 预防机制（新增）

**错误隔离**：
- 识别错误影响范围
- 隔离受影响任务
- 防止错误扩散

**降级策略**：
- 一级降级：使用缓存
- 二级降级：简化模式
- 三级降级：排队重试

**预警通知**：
- 错误率 > 10% → 预警
- 响应时间 > 2x → 预警
- 内存使用 > 80% → 预警

### 集成到工作流

```javascript
// 在 agent-manager.js 中集成
const { CircuitBreakerManager } = require('./circuit-breaker.js');

const cbManager = new CircuitBreakerManager();

// 分配任务前检查熔断状态
async function assignTask(agentName, task) {
  // 检查是否可以接受请求
  if (!cbManager.canAcceptRequest(agentName)) {
    console.warn(`⚠️ Agent ${agentName} 熔断中，拒绝分配任务`);
    return null;
  }

  // 分配任务并执行
  const startTime = Date.now();
  try {
    const result = await executeTask(agentName, task);
    
    // 记录成功
    cbManager.recordResult(agentName, true, null, Date.now() - startTime);
    return result;
  } catch (error) {
    // 记录失败
    cbManager.recordResult(agentName, false, error, Date.now() - startTime);
    throw error;
  }
}
```

### 配置选项

```javascript
{
  // 熔断阈值
  failureThreshold: 5,          // 连续失败 5 次→熔断
  successThreshold: 3,          // 连续成功 3 次→恢复
  warningThreshold: 0.5,        // 错误率 50%→预警
  
  // 超时配置
  timeout: 30000,               // 任务超时 30 秒
  resetTimeout: 300000,         // 熔断重置 5 分钟
  recoveryTimeout: 600000,      // 完全恢复 10 分钟
  
  // 资源阈值
  memoryThreshold: 0.9,         // 内存 90%→预警
  cpuThreshold: 0.95,           // CPU 95%→预警
  
  // 恢复策略
  gradualRecovery: true,        // 启用渐进恢复
  autoRetry: true,              // 启用自动重试
  maxRetries: 3                 // 最大重试 3 次
}
```

## ⚠️ 注意事项

### 1. 熔断阈值

- 默认连续失败 5 次触发熔断
- 可根据 Agent 类型调整
- 关键 Agent 可提高阈值

### 2. 恢复时间

- 冷却期 5 分钟（不可跳过）
- 渐进恢复 5-10 分钟
- 总恢复时间 10-15 分钟

### 3. 预警通知

- 错误率 > 10% 发送预警
- 建议配置告警通知
- 及时响应预警防止熔断

### 4. 日志管理

- 状态变化日志：`state-changes.log`
- 备份文件自动清理（保留 7 天）
- 建议配置日志轮转
