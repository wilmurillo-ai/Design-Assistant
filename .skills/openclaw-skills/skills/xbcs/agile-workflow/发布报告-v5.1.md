## 🎯 核心改进

### 问题背景

**v5.0 问题**：所有 Agent 常驻，资源占用大

```
20 个 Agent 常驻:
- 内存占用：4GB
- CPU 占用：15%
- 电费成本：约 50 元/月
```

```
有任务→启动，1 小时无任务→释放

- CPU 占用：2-5% (节省 70%)
- 电费成本：约 15 元/月 (节省 70%)
```

## 📊 核心功能

### 1. Agent 状态管理（新增）

**四状态机**:
```
离线 (offline) ←→ 空闲 (idle) ←→ 活跃 (active)
    ↑                ↓
    └──── 休眠 (sleeping) ────┘
```

| 状态 | 说明 | 资源占用 | 启动时间 | 转换条件 |
|------|------|----------|----------|----------|
| **活跃** | 正在执行任务 | 100% | - | 分配任务 |
| **空闲** | 已启动无任务 | 30% | - | 任务完成 |
| **休眠** | 暂停可快速恢复 | 10% | 5 秒 | 30 分钟空闲 |
| **离线** | 已释放 | 0% | 30 秒 | 1 小时空闲 |

### 2. 按需启动（新增）

**任务分配流程**:
```
分配任务 → 检查状态 → 离线？→ 启动 → 验证 → 分配
             ↓          ↓
           活跃/空闲 → 直接分配
```

**测试结果**:
```bash
$ node agent-manager.js assign chapter_writer "第 1 章创作"

📋 分配任务给 Agent chapter_writer: 第 1 章创作
🚀 启动 Agent chapter_writer...
✅ Agent chapter_writer 启动成功 (PID: 12345)
✅ 任务已分配给 Agent chapter_writer
```

### 3. 空闲释放（新增）

**自动检测**:
- 30 分钟无任务 → 休眠（保留状态，快速恢复）
- 1 小时无任务 → 释放（停止进程，释放资源）

**测试结果**:
```bash
$ node agent-manager.js idle-check

⏰ Agent chapter_writer 空闲 30 分钟，进入休眠...
⏰ Agent world_builder 空闲 1 小时，释放资源...
✅ 处理了 2 个空闲 Agent
```

### 4. 健康监控（增强）

**监控维度**:
- 心跳检测（30 秒一次）
- 日志监控（实时）
- 进程监控（实时）
- 自动恢复（失败重启）

**测试结果**:
```bash
$ node agent-manager.js status

📊 Agent 状态统计:
总计：5
活跃：1
空闲：2
休眠：1
离线：1
内存：512.0 MB
CPU: 8%
```

## 🚀 使用指南

### 快速开始

```bash
# 1. 查看 Agent 状态
node /skills/agile-workflow/core/agent-manager.js status

# 2. 启动指定 Agent
node /skills/agile-workflow/core/agent-manager.js start chapter_writer

# 3. 分配任务（自动启动）
node /skills/agile-workflow/core/agent-manager.js assign chapter_writer "第 1 章创作"

# 4. 手动检测空闲
node /skills/agile-workflow/core/agent-manager.js idle-check

# 5. 停止 Agent
node /skills/agile-workflow/core/agent-manager.js stop chapter_writer
```

### 集成到工作流

```javascript
// 在 agile-workflow-engine-v5.js 中集成
const { AgentStateManager } = require('./agent-manager.js');

const agentManager = new AgentStateManager();

// 分配任务时自动启动
async function assignTask(subtask, projectId) {
  await agentManager.assignTask(subtask.agent, subtask);
}

// 完成任务时更新状态
function completeTask(taskId) {
  const task = this.state.tasks[taskId];
  agentManager.completeTask(task.agent, true);
}
```

## ⚙️ 配置选项

### 超时配置

```javascript
{
  idleToSleep: 30 * 60 * 1000,    // 30 分钟空闲→休眠
  idleToOffline: 60 * 60 * 1000,  // 1 小时空闲→离线
  heartbeatInterval: 30000,        // 30 秒心跳检测
  startupTimeout: 30000            // 启动超时 30 秒
}
```

### 资源限制

```javascript
{
  maxConcurrentAgents: 5,   // 最大同时运行 Agent 数
  minIdleAgents: 1          // 最小空闲 Agent 数
}
```

### Agent 配置

```javascript
{
  agents: {
    'chapter_writer': {
      command: 'node agents/chapter-writer.js',
      port: 3001,
      autoStart: false  // 不自动启动，按需启动
    },
    // ...其他 Agent
  }
}
```

## ⚠️ 注意事项

### 1. 启动延迟

- 离线 Agent 启动需要 30 秒
- 建议：常用 Agent 保持空闲状态
- 配置：`minIdleAgents: 1`

### 2. 状态备份

- 休眠/释放前自动备份
- 备份位置：`/workspace/logs/agent-manager/backup-*.json`
- 建议：定期清理旧备份（保留 7 天）

### 3. 健康监控

- 心跳丢失 90 秒自动恢复
- 日志实时监控
- 建议：配置告警通知

### 4. 资源限制

- 最大并发 Agent 数：5
- 超出限制的任务排队等待
- 建议：根据服务器配置调整
