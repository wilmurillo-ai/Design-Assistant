## 📈 功能特性

### 负载检测

**检测指标**：
- 当前任务数
- 负载百分比
- CPU/内存占用
- 响应时间
- 错误率
- 心跳状态

**检测频率**：
- 实时检测：1 秒/次
- 评分更新：5 秒/次
- 健康检查：30 秒/次

### 智能分配

**评分算法**：
```
Agent 得分 = 100 - 负载率×50% - 错误率×30% - 响应时间×20% + 健康加分
```

**分配策略**：
1. 得分优先（选择得分最高）
2. 负载均衡（避免单点过载）
3. 亲和性（相同任务优先同一 Agent）
4. 备用选择（最优不可用时选次优）

### 冲突解决

**冲突类型**：
| 类型 | 解决策略 | 说明 |
|------|---------|------|
| **资源冲突** | 排队等待 | 多个任务竞争同一资源 |
| **依赖冲突** | 等待依赖 | 依赖任务未完成 |
| **优先级冲突** | 抢占资源 | 高优先级插队 |

### 过载保护

**保护机制**：
- 过载检测（负载>90%）
- 限流（拒绝新任务）
- 降级（降低质量要求）
- 熔断（暂停服务）

**队列管理**：
- 最大队列大小：100
- 超时时间：30 秒
- 优先级排序

### 评分算法

```javascript
function calculateScore(agent) {
  let score = 100;
  
  // 负载率扣分 (50%)
  score -= agent.loadPercent * 0.5;
  
  // 错误率扣分 (30%)
  score -= agent.errorRate * 100 * 0.3;
  
  // 响应时间扣分 (20%)
  score -= Math.min(agent.responseTime / 1000, 10) * 0.4;
  
  // 健康加分
  if (agent.healthy) score += 10;
  if (agent.recentSuccess) score += 5;
  
  return Math.max(0, Math.min(100, score));
}
```

```javascript

// 注册 Agent

// 分配任务
```
