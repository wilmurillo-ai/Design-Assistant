# 问题严重级别定义

## 级别总览

| 级别 | 标识 | 修复要求 | 合并影响 |
|------|------|----------|----------|
| **P0** | 🔴 Critical | 必须修复 | 阻止合并 |
| **P1** | 🟠 High | 强烈建议 | 建议修复后合并 |
| **P2** | 🟡 Medium | 建议优化 | 可接受但需跟进 |
| **P3** | 🟢 Low | 可选改进 | 不影响合并 |

---

## P0 — Critical（必须修复）

### 定义
- 会导致程序崩溃、数据损坏或安全漏洞
- 任何未定义行为（UB）

### C++ 典型场景
```
- 空指针解引用
- 数组越界访问
- use-after-free
- 数据竞争（data race）
- 未初始化变量使用
- 缓冲区溢出（strcpy/sprintf/scanf 系列）
- 命令注入（用户输入拼接到 system/popen）
```

### 修复要求
```
必须修复后才能继续
评审报告需明确标注 P0 问题及风险
```

---

## P1 — High（强烈建议）

### 定义
- 不直接导致崩溃，但可能引发 P0 级问题
- 严重的可维护性或性能问题
- 违反重要最佳实践

### C++ 典型场景
```
- 裸 new[]/delete[]（应用智能指针或容器）
- double-free 风险
- 整数溢出风险（安全相关）
- 缺少异常安全性
- 严重的内存泄漏
- 虚函数滥用（性能）
- std::auto_ptr 使用（已废弃）
- 缺少边界检查的指针运算
```

### 修复要求
```
强烈建议修复
可在后续迭代中处理，但需有明确跟踪
```

---

## P2 — Medium（建议优化）

### 定义
- 代码可正常工作，但有更好的实现方式
- 可维护性或性能改进空间

### C++ 典型场景
```
- 不必要的拷贝（缺少 const ref 或 move）
- 可以 constexpr 的常量使用了宏
- 循环内重复计算可提取
- 建议使用 emplace_back 替代 push_back
- 缺少 override 说明符
- 缺少 const 成员函数
- 过于复杂的模板元编程
- 建议使用 std::array 替代 C 风格数组
```

### 修复要求
```
建议修复
可通过 IDE 自动修复或手动优化
```

---

## P3 — Low（可选改进）

### 定义
- 代码风格、可读性、优化层面的建议
- 不影响功能、正确性或性能

### C++ 典型场景
```
- 命名不规范（但符合项目风格）
- 缺少 JSDoc/Doxygen 注释
- magic number 未定义为常量
- 建议使用 auto 简化类型声明
- 建议使用范围 for 循环
- 建议使用 std::string_view（C++17）
- 过多的嵌套层级
- 建议添加 assert 增强可调试性
```

### 修复要求
```
可选
不影响合并，可在后续迭代中改进
```

---

## 升级 / 降级规则

### 升级条件（P3 → P2 → P1 → P0）

1. **上下文风险**：当 P2/P3 问题出现在安全关键代码路径时，升级
2. **累积效应**：多个相同类型的 P3 问题可能升级为 P2
3. **反馈循环**：P3 问题的修复成本极低时（如 rename），升级为 P2

### 降级条件（P0 → P1 → P2 → P3）

1. **误报**：评审工具误报（实际代码有保护）
2. **受控环境**：仅在特定平台/配置下触发
3. **防御性代码**：已有其他机制缓解风险

## 评分计算公式

```
总分 = 内存安全得分 × 0.30
     + 逻辑正确性得分 × 0.25
     + 性能得分 × 0.20
     + 安全得分 × 0.15
     + 可读性得分 × 0.10

各维度得分 = (满分问题数 - 扣分问题数) / 满分问题数 × 100

扣分规则：
- P0 问题：-25 分
- P1 问题：-15 分
- P2 问题：-5 分
- P3 问题：-1 分

下限：0 分（最低不会低于 0）
```

## 示例

### 示例 1: 内存泄漏 + UB

```cpp
void process() {
    int* p = new int[100];
    // 忘记 delete[]
    if (condition) return; // P0: 内存泄漏 + P0: UB（使用未初始化）
    delete[] p;
}
```

**结果**：
- 内存安全维度：P0 × 2 → -50 分
- 总分会被显著拉低

### 示例 2: 性能问题

```cpp
void copy_data(std::vector<int> v) { // P2: 不必要的拷贝
    for (auto& x : v) { ... }
}
```

**结果**：
- 性能维度：P2 × 1 → -5 分
- 总分轻微影响
