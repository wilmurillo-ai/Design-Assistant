# C++ 专项检查清单

## 1. 内存安全（Memory Safety）

### 🔴 必须检查
- [ ] 是否有裸 `new[]` / `delete[]`？建议改用 `std::vector` 或 `std::unique_ptr`
- [ ] 是否存在悬垂指针（dangling pointer）？
- [ ] 是否存在 double-free？
- [ ] 是否有 use-after-free 风险？
- [ ] 内存分配 / 释放是否配对？

### 🟡 建议检查
- [ ] 是否有 `std::auto_ptr`？（已废弃，应用 `std::unique_ptr`）
- [ ] 是否有 `malloc`/`free` 与 `new`/`delete` 混用？
- [ ] 异常安全性：资源获取是否使用 RAII？

---

## 2. 未定义行为（Undefined Behavior）

### 🔴 必须检查
- [ ] 变量是否在定义前使用？
- [ ] 是否访问了数组越界？
- [ ] 是否有符号整数溢出？（`int` 溢出会 UB）
- [ ] 是否有除零操作？
- [ ] 是否有空指针解引用？

### 🟡 建议检查
- [ ] 是否有数据竞争（data race）？
- [ ] 是否有类型双关（type punning）？应使用 `std::bit_cast`
- [ ] 移位操作是否超过类型宽度？
- [ ] `union` 使用是否涉及活跃成员之外的写操作？

---

## 3. RAII / 所有权（Ownership）

### 🟡 建议检查
- [ ] 构造函数 / 析构函数顺序是否正确？
- [ ] 是否有成员初始化顺序问题？（按声明顺序初始化）
- [ ] `std::unique_ptr` 是否正确使用？（禁止复制，使用 `std::move`）
- [ ] `std::shared_ptr` 是否有循环引用风险？（使用 `std::weak_ptr` 打破）
- [ ] 锁的获取 / 释放是否配对？是否使用 `std::lock_guard`？

---

## 4. 性能（Performance）

### 🟢 可选检查
- [ ] 是否存在不必要的拷贝？（传参应使用 `const T&` 或值语义 + `std::move`）
- [ ] 是否存在不必要的临时对象？
- [ ] 循环内是否有重复计算可以提取？
- [ ] 是否需要内联建议？（频繁调用的小函数）
- [ ] 是否使用了虚函数但可以静态派发？
- [ ] 容器操作是否高效？（`push_back` vs `emplace_back`）

---

## 5. 安全（Security）

### 🔴 必须检查
- [ ] 是否有 `strcpy`/`sprintf`/`scanf`？（缓冲区溢出风险）
- [ ] 是否有用户输入直接拼接到命令字符串？（命令注入）
- [ ] 是否有未验证的指针运算？
- [ ] 是否有硬编码的敏感数据？（密钥、密码）
- [ ] 是否有路径遍历风险？（用户输入拼接到文件路径）

### 🟡 建议检查
- [ ] 格式化字符串是否安全？（`printf` vs `printf("%s", str)`）
- [ ] 整数操作是否做边界检查？（防止整数溢出绕过安全检查）
- [ ] 错误信息是否泄露敏感信息？

---

## 6. 现代 C++（Modern C++）

### 🟢 可选检查
- [ ] 是否使用了 `constexpr` 可以替代宏定义？
- [ ] 是否可以使用 `std::array` 替代 C 风格数组？
- [ ] 是否可以使用 `std::optional` 替代 `nullptr` 表示无效值？
- [ ] 是否有 `auto` 可以使用但未使用？（提升可读性）
- [ ] 是否有范围 `for` 循环可以替代普通 `for` 循环？
- [ ] 是否使用了 C++20 `std::format` 替代 `sprintf`？
- [ ] 是否有 `std::string_view` 可以替代 `const std::string&`？（避免不必要的拷贝）
- [ ] 是否有 Concepts（C++20）可以替代 `static_assert`？

---

## 7. 代码可读性

### 🟢 可选检查
- [ ] 头文件是否过于庞大？（分解为更小的头文件）
- [ ] 模板元编程是否过于复杂？（添加注释或简化）
- [ ] 是否有过于复杂的嵌套 lambda？
- [ ] 是否有 magic number 需要定义为常量？
- [ ] 是否有 TODO/FIXME/HACK 未处理？
