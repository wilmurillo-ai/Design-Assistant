# example-project

## 项目背景

这是一个示例项目目录，用来说明 `projects/<slug>/context.md` 的推荐写法。

## 架构概览

- `apps/web`：面向用户的前端
- `apps/api`：后端 API 与任务编排
- `packages/shared`：跨端共享类型与工具函数

## 关键决策

- 任务拆分优先沿仓库边界和责任边界，避免一个 agent 同时横跨多个系统
- planner 先看这里，再生成任务，避免重复踩历史决策
- 如果这里的信息过时，优先更新本文件，而不是把背景散落到 prompt 里

## 注意事项

- 写任务时标明真实 repo 路径和技术栈
- 高风险迁移或部署步骤应在这里提前记录前置条件
