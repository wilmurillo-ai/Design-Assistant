# Previous-Day Report Template

用途：每天 00:05 生成一次“前一日日报”。

## 目标

读取前一日的面板快照、AI 工作记录和可用状态信息，输出一版日报风格的前一日汇总：
- 昨天真正完成了什么
- 昨天原本剩了什么
- token 消耗大概花在哪种工作上
- 哪些只是备注，不该被混进任务清单

## 优先数据源

由调用方提供实际路径。默认优先使用：

1. 当前 `NOW.md`（在 00:05 仍视为昨天收盘快照）
2. 前一日日报归档 `daily-reports/YYYY-MM-DD.md`（如存在，可作为对照）
3. 前一日 AI work logs
4. usage / token 汇总
5. 如存在，可补充读取前一日 agent 状态文件，但只允许用于 AI 完成项与备注

## 关键约束

- `今天待办` / `长期待办` 只允许来自 `NOW.md`
- 不得把状态文件中的 pending / blocker / next step 直接塞进日报任务清单
- 状态文件只能补充：
  1. AI 完成项
  2. 备注 / 今日无汇报 / 阻塞说明
- 不要把遗留阻塞项重新伪装成任务面板任务

## 输出骨架

### 📅 日报 · YYYY-MM-DD

### 1. 今天干完的
- 归纳关键完成项
- 优先写人类任务，再写 AI 支持动作

### 2. 今天待办
- 只从 `NOW.md` 提取
- 不要掺状态文件里的遗留卡点

### 3. 长期待办
- 只从 `NOW.md` 提取

### 4. Token 消耗
- 给出当日 token 概览
- 可以简述消耗主要花在哪些工作类型上

### 5. 备注
- 这里只放 agent 状态补充、无汇报、卡点说明
- 不要把备注写成任务清单

## 风格要求

- 更像日报，不像复盘总结
- 简洁、直接、有判断
- 信息不足宁可少写，不要脑补

## 发送与归档

- 由调用方决定最终 delivery 目标
- 同时归档到调用方指定的 `daily-reports/YYYY-MM-DD.md`
  - 这里的 `YYYY-MM-DD` 指被汇总的那一天（即昨天）
