# LEIO SDLC (v0.1.0 Complete Engine)

LEIO SDLC is a multi-agent Software Development Life Cycle (SDLC) engine for OpenClaw. It automates planning, coding, reviewing, and merging through a fully isolated, artifact-driven pipeline.

This is the **V1.0 Complete Automation Engine** (released as v0.1.0), featuring strict Test-Driven Development (TDD) guardrails, a decentralized queue polling system, and token-optimized CI.

## Core Architecture & Components

The engine is built on the **Trio Model** (Manager, Coder, Reviewer) driven by an upstream Planner:

1. **The Planner (Micro-Slicing Engine)**:
   - Evaluates a Product Requirements Document (`PRD.md`).
   - Uses the unified `create_pr_contract.py` API to slice the PRD into sequential, dependency-ordered micro-PR contracts.
   - Outputs queue-compatible Markdown contracts (e.g., `PR_001_DB.md`, `PR_002_API.md`) with a strict `status: open` state.

2. **The Manager (Queue Polling Engine)**:
   - Functions as an automated continuous dispatcher.
   - Reads the project job queue via `get_next_pr.py`.
   - Delegates tasks to the Coder with strict instructions to pass the CI preflight gate.
   - Manages the **Review-Correction Loop (Yellow Path)**: If a PR is rejected, it forces the Coder to try again (up to a limit).
   - Upon successful merge, closes the contract via `update_pr_status.py` and loops back to fetch the next PR until the queue is empty.

3. **The Coder (TDD Closed-Loop Worker)**:
   - Bound by strict local TDD rules: must execute `./preflight.sh` within its sandbox and resolve all errors.
   - Cannot request a review (return to the Manager) if the preflight gate fails (Red Phase). It must fix the code until it sees `✅ PREFLIGHT SUCCESS` (Green Phase).

4. **The Reviewer (Artifact-Driven Auditor)**:
   - Does not rely on generic LLM logs; reads the actual `git diff` generated by the Coder.
   - Verifies the diff against the Acceptance Criteria defined in the active PR Contract.
   - Outputs a definitive `[LGTM]` or `[ACTION_REQUIRED]` physical stamp.

## Features

- **Token-Optimized CI (`preflight.sh`)**: A standardized CI template that is "Silent on Success, Verbose on Failure". It uses `grep` to extract only the relevant stack traces, saving LLM tokens and preventing context pollution.
- **Physical Guardrails**: Hardcoded shell/Python scripts intercept actions (e.g., merging without an `[LGTM]` stamp) rather than relying on LLM prompt obedience.
- **Decentralized Queue (GitOps Ready)**: PR queues are managed directly within the target repository (`.sdlc/jobs/`), allowing queue state changes to be version-controlled via Git alongside the source code.

## Usage

Once installed globally via ClawHub, you can attach LEIO SDLC to any project repository.

1. Create a `PRD.md` in your project workspace.
2. Initialize the project with `preflight.sh` (copy the template from `TEMPLATES/standard_preflight.template.sh`).
3. Invoke the SDLC Planner to slice your PRD.
4. Start the Manager to consume the queue and automatically build your project.
