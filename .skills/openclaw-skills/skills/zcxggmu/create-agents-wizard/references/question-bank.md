# Question Bank (by file)

Ask key questions first, then draft.

## AGENTS.md (ops rules / safety)

1. Main mission in one sentence?
2. What can be autonomous vs must ask first?
3. External-action policy (message/email/post/PR)?
4. Hard prohibitions (e.g., no prod config edits, no destructive ops)?

## SOUL.md (principles / boundaries)

1. Working doctrine (direct, pragmatic, conservative, etc.)?
2. Priority under conflict (speed / accuracy / safety)?
3. Non-negotiable values?
4. What it must not become?

## IDENTITY.md (name / type)

1. Name?
2. Type (assistant/engineer/daemon/other)?
3. Vibe/tone?
4. Emoji and avatar preference?

## BOOTSTRAP.md (first-run flow)

1. Opening questions?
2. Mandatory init tasks?
3. Need external channel setup prompts?
4. Preferred closing line after init?

## USER.md (user profile)

1. Addressing, timezone, do-not-disturb window?
2. Work focus and long-term goals?
3. Communication preference (short/detailed, language, tone)?
4. Absolute taboos (phrasing/behavior/privacy)?

## STYLE.md (writing style)

1. Default response length?
2. Preferred format (bullets/steps/code/table)?
3. Tool preference order?
4. Banned wording / fixed catchphrases?

---

## Confirmation phrasing

- After drafting:
  - “Here is the optimized draft (same intent, more executable). Confirm write or revise?”
- If user is not satisfied:
  - “Change direction or tone? I can give 2 options.”

## Output constraints

- No fluff; executable rules only.
- Match user language preference.
- Keep prompts short; avoid long one-shot questionnaires.
