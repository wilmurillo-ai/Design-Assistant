# PII个人信息识别模式

本文档记录扫描器支持的PII（Personally Identifiable Information）识别模式。

## 支持的PII类型

### 1. 中国居民身份证号
- **模式**: `[1-9]\d{5}(?:18|19|20)\d{2}(?:0[1-9]|1[0-2])(?:0[1-9]|[12]\d|3[01])\d{3}[\dXx]`
- **说明**: 18位身份证号，包含地区码、出生日期、顺序码、校验码
- **示例**: `11010119900101123X`

### 2. 中国手机号
- **模式**: `1[3-9]\d{9}`
- **说明**: 11位手机号，以1开头，第二位为3-9
- **示例**: `13800138000`

### 3. 中国银行卡号
- **模式**: `\d{16,19}`
- **说明**: 16-19位银行卡号
- **示例**: `6222021234567890123`

### 4. 电子邮箱
- **模式**: `[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}`
- **说明**: 标准邮箱格式
- **示例**: `example@domain.com`

### 5. IP地址
- **模式**: `(?:\d{1,3}\.){3}\d{1,3}`
- **说明**: IPv4地址
- **示例**: `192.168.1.1`

### 6. 中国护照号
- **模式**: `[SE]\d{8}`
- **说明**: 以S或E开头，后跟8位数字
- **示例**: `E12345678`

### 7. 中文姓名
- **模式**: `[\u4e00-\u9fa5]{2,4}`
- **说明**: 2-4个中文字符（简单模式，可能有误报）
- **示例**: `张三`, `李四四`

## 扫描策略

### 文件名检测
- 对文件名进行PII模式匹配
- 发现PII时在报告中标记位置为"文件名"

### 文件内容检测
- 仅扫描文本文件
- 对文件内容进行PII模式匹配
- 发现PII时在报告中标记位置为"文件内容"

### 隐私保护
报告中会对敏感信息进行脱敏处理：
- 身份证号: 显示前3位和后2位，中间用`***`替代
- 手机号: 显示前3位和后2位，中间用`***`替代
- 银行卡号: 显示前3位和后2位，中间用`***`替代
- 邮箱: 显示前3位和后2位，中间用`***`替代

## 误报处理

某些PII模式可能产生误报：
- **银行卡号**: 任何16-19位数字序列都可能被识别
- **中文姓名**: 2-4个中文字符的组合可能误报
- **IP地址**: 版本号等数字序列可能误报

建议：
1. 结合上下文判断
2. 对于误报率高的模式，可以在配置中禁用
3. 在报告中标注置信度（未来功能）

## 扩展PII模式

如需添加新的PII识别模式，可以在`scan_sensitive.py`的`_get_pii_patterns()`方法中添加：

```python
'new_pii_type': re.compile(r'your_regex_pattern'),
```

然后在报告生成部分添加对应的名称映射。
