# 花叔 - 著作与系统性思考

## 核心论点（反复出现 ≥3次）

### 1. Harness Engineering（给AI造缰绳）
> "AI模型是马，力气再大，没有缰绳就是一匹乱跑的野马。Harness Engineering，就是给AI造缰绳的工程方法论。"

- **来源**：《Harness Engineering橙皮书》（80页，微信读书上架）
- **核心逻辑**：
  - OpenAI Codex团队：7个人，5个月，零行手写代码，交付100万行产品
  - 模型没换，换的是套在模型外面的那层东西
  - Harness五个组件：指令、约束、反馈、记忆、编排
  - 反直觉原则：**Harness不是越多越好**
- **应用场景**：AI编程、Agent开发、工作流设计

### 2. AI Native Coder宣言
> "我不会写代码。一行都不会。但我用AI做出了App Store付费榜Top1的产品。"

- **来源**：GitHub主页、个人简介
- **核心逻辑**：
  - 经济学专业出身，完全不会传统代码
  - 所有产品都用AI编程工具完成
  - 从"对话驱动"到"规范驱动"的进化
- **应用场景**：零基础开发者、独立开发者、产品经理转型

### 3. 三阶段进化论
> "从好用到可用，差的不是手感，而是工程能力"

- **来源**：《AI Coding从好用到可用》文章、掘金
- **核心逻辑**：
  - **第一阶段：Vibe Coding** - 对话驱动，随机好用
  - **第二阶段：Spec Coding** - 先设计再执行，稳定可控
  - **第三阶段：SDD Coding** - 规范驱动开发，可验证
- **关键判断**：换模型是最容易的决策，也往往是最没用的决策

### 4. 精准投放原则
> "真正值得投入的，是把AI Coding从'随机好用'变成'稳定可控'"

- **核心逻辑**：
  - 用最少的token传递最高密度的信号
  - 四层分开：静态常驻 / 按需取用 / 参照代码 / 动态背景
  - 关键约束不能靠运气
- **案例**：某电商中后台团队采纳率从50%提升到97.9%

### 5. 知识资产护城河
> "AI编码的护城河，不是更好的Prompt，而是更厚的知识资产"

- **核心逻辑**：
  - AI使用方式变成团队资产，而不是个人技巧
  - CLAUDE.md + rules/ + /plan命令 = 最小启动配置
  - 知识资产分层：开发规范、代码模板、领域知识

## 代表性观点

### 关于AI编程
- "AI Coding从好用到可用，差的不是手感，而是工程能力"
- "换模型是最容易的决策，也往往是最没用的决策"
- "让AI抄代码，比让AI写代码更可控"

### 关于独立开发
- "找到真实需求，用AI快速实现，完全开源"
- "做AI内容博主门槛确实不高，但能做出差异化的人很少"

### 关于学习
- "不要问'AI时代怎么最挣钱'，要问你有没有什么强烈的热爱和好奇心，然后用AI把它做到极致"
- "手册化的工作会被替代，判断力的工作AI替代不了"
