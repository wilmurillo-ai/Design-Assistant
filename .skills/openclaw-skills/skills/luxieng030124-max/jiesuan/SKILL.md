## 📊 结算模式详解

### 1️⃣ 达标瓜分（Guaranteed）
**适用场景**：完成指定任务即获得奖励，人人有份

**算法**：
```
人均金额 = 奖池总额 / 达标人数
```

**示例**：
- 奖池10万元，完成量≥100的用户达标
- 达标200人 → 每人500元
- 达标100人 → 每人1000元

---

### 2️⃣ 排名赛（Ranking）
**适用场景**：激励头部优秀者，按排名差异化奖励

**算法**：
```
根据排序字段排名，按名次段发放固定金额
```

**示例**：
| 名次段 | 奖励金额 | 人数 | 小计 |
|--------|----------|------|------|
| 第1-3名 | 5000元 | 3 | 15000元 |
| 第4-10名 | 2000元 | 7 | 14000元 |
| 第11-20名 | 1000元 | 10 | 10000元 |

**合计**：39,000元

---

### 3️⃣ 混合不互斥（Hybrid）
**适用场景**：多重激励并存，达标和排名奖励可同时获得

**算法**：
```
用户总奖金 = Σ(各奖池奖金)
```

**示例**：
- 用户A：播放量1.5万（达标），点赞排名第5
- 达标奖：500元
- 排名奖：5000元
- **总奖金：5500元**

---

### 4️⃣ 权重分配（Weighted）
**适用场景**：按贡献度分配，多劳多得

**算法**：
```
用户奖金 = 奖池总额 × (用户权重 / 总权重)
```

**示例**：
- 奖池10万元，按播放量权重分配
- 用户A播放量10万（占总量20%）→ 2万元
- 用户B播放量5万（占总量10%）→ 1万元

## 🐛 错误处理与恢复

### 常见错误及解决方案

#### 1. AI解析超时
```
⚠️ 请求超时（60秒）

解决建议：
· 将超时时间调整为90秒或120秒
· 简化规则描述，分多次对话完善
· 检查网络连接状态
```

#### 2. 数据格式错误
```
⚠️ 无法识别数据格式

解决建议：
· 确保文件为.xlsx/.xls/.csv格式
· CSV文件建议使用UTF-8编码
· 检查第一行是否为列标题
```

#### 3. 缺少必需字段
```
⚠️ 缺少必需字段：作者ID、播放量

解决建议：
· 检查数据文件列名是否正确
· 支持的列名变体：作者ID/用户ID/账号/ID
· 使用对话功能映射字段名
```

#### 4. 计算异常
```
⚠️ 检测到异常值：播放量包含负数

解决建议：
· 检查数据源是否正确
· 使用过滤条件排除异常数据
· 手动修正后重新上传
```

### 自动重试机制
```python
max_retries = 3      # 最大重试次数
retry_delay = 2s     # 重试间隔（指数退避）
timeout = 90s        # 单次超时时间

for attempt in range(max_retries):
    try:
        result = process_with_timeout(timeout)
        return result
    except TimeoutError:
        if attempt < max_retries - 1:
            wait(retry_delay * (2 ** attempt))  # 2s, 4s, 8s
        else:
            raise SettlementError("多次重试失败，建议调整超时配置")
```
