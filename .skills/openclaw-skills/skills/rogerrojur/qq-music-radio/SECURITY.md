# 安全说明 (Security Policy)

## 概述

QQ 音乐电台播放器是一个安全、透明的本地应用，所有代码开源可审查。

## 🔒 安全特性

### 1. 本地优先架构
- **本地服务器** - 运行在 `localhost:3000`，不监听外部网卡
- **无持久化** - 不保存用户数据或历史记录
- **进程隔离** - 独立的 Node.js 进程，不影响系统

### 2. 网络行为
本应用的网络请求仅限于：

**必需的 API 调用：**
- `c.y.qq.com` - QQ 音乐搜索和歌单 API
- `u.y.qq.com` - QQ 音乐播放链接获取
- `y.gtimg.cn` - 专辑封面图片（CDN）

**可选的隧道服务：**
- `serveo.net` - SSH 反向代理（仅当 `ENABLE_TUNNEL=true` 时）

**无其他外发连接：**
- ❌ 不连接任何分析服务
- ❌ 不上传用户数据
- ❌ 不包含追踪代码

### 3. 公网隧道（可选）

**默认行为：** 启用（`ENABLE_TUNNEL=true`）

**工作原理：**
```bash
ssh -R 80:localhost:3000 serveo.net
```

- 仅转发 HTTP 流量到本地端口
- 不暴露其他服务或端口
- 隧道地址是临时的（关闭即失效）

**如何禁用：**
```bash
# 方法 1: 环境变量
ENABLE_TUNNEL=false ./start.sh

# 方法 2: 配置文件
# 创建 .env 文件，内容：
# ENABLE_TUNNEL=false
./start.sh
```
./start.sh
```

### 4. 代码审查

**核心文件：**
- `start.sh` - 启动脚本（83 行，可读）
- `server-qqmusic.js` - Node.js 服务器（240 行）
- `public/index.html` - 前端页面
- `public/app-auto.js` - 前端逻辑

**审查重点：**
1. **无混淆** - 所有代码都是明文，未压缩
2. **无外部依赖** - 仅使用知名 npm 包（express, axios）
3. **无 eval** - 不使用 eval() 或动态代码执行

### 5. 依赖安全

**生产依赖：**
```json
{
  "express": "^4.18.2",  // Web 框架
  "axios": "^1.6.0",     // HTTP 客户端
  "cors": "^2.8.5",      // CORS 中间件
  "dotenv": "^16.3.1"    // 环境变量
}
```

所有依赖都是：
- ✅ 官方维护的热门包
- ✅ 定期安全更新
- ✅ 无已知高危漏洞

## 🛡️ 使用建议

### 推荐配置（高安全）
```bash
# 创建 .env 文件
ENABLE_TUNNEL=false           # 禁用公网访问
SHOW_SECURITY_WARNING=true    # 保持安全提示
```

**适用场景：**
- 个人使用
- 公司内网
- 对安全敏感的环境

### 默认配置（平衡）
```bash
# 创建 .env 文件
ENABLE_TUNNEL=true            # 启用公网访问
SHOW_SECURITY_WARNING=true    # 显示安全提示
```

**适用场景：**
- 需要分享给朋友
- 手机访问电脑播放器
- 临时演示

### 自托管配置（高级）
```bash
# 使用自己的反向代理
ENABLE_TUNNEL=false
# 然后手动配置 nginx/caddy 等
```

## 🚨 风险披露

### 已知风险

1. **QQ 音乐 API**
   - 使用非官方 API，可能随时失效
   - 部分歌曲有版权限制
   - API 响应速度受 QQ 音乐控制

2. **公网隧道（启用时）**
   - 任何人都可以访问播放器（如果知道 URL）
   - 隧道服务可能不稳定
   - 建议仅在需要时启用

3. **浏览器安全**
   - 依赖浏览器的 HTML5 Audio API
   - 某些旧浏览器可能不支持

### 不存在的风险

- ❌ 不会窃取个人信息
- ❌ 不会感染病毒或恶意软件
- ❌ 不会篡改系统文件
- ❌ 不会开放其他端口
