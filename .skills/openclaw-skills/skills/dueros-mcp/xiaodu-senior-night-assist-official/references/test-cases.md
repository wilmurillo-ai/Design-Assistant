# test-cases.md

## 测试用例

---

## 1. 有 night scene 时的完整调用链

### 1A. scene 存在且触发成功
- 输入：夜间辅助模式
- 前提：有高置信度 night-assist scene
- 预期：`list_scenes` -> `trigger_scene` -> `speak.sh` -> `night_light_tail.sh`
- 预期用户口径：自然产品汇报，不是技术日志

---

## 2. 路径照明模式

### 2A. 床边灯 + 走廊灯 + 卫生间灯齐全
- 输入：夜里起夜
- 前提：没有匹配 scene，但床边灯、走廊灯、卫生间灯都可用
- 预期：床边灯 -> 走廊灯 -> 卫生间灯 -> 短播报 -> 自动收尾

### 2B. 只有床边灯
- 输入：夜里起夜
- 前提：只有床边灯，没有走廊灯/卫生间灯
- 预期：至少开床边灯 -> 短播报 -> 自动收尾
- 预期：不要因为缺少路径灯而整体中断

### 2C. 某个路径灯失败
- 输入：我要去卫生间
- 前提：走廊灯失败，床边灯和卫生间灯可用
- 预期：床边灯成功 -> 走廊灯失败 -> 卫生间灯继续 -> 短播报 -> 自动收尾

---

## 3. 局部照明模式

### 3A. 只开夜灯
- 输入：帮我开一下夜灯
- 前提：有夜灯
- 预期：只开夜灯 -> 短播报 -> 3 分钟后自动关
- 预期：不要扩展成整条路径

### 3B. 开床边灯
- 输入：开一下床边灯
- 前提：有床边灯
- 预期：只开床边灯 -> 短播报 -> 自动收尾

### 3C. 没有夜灯，降级到最近柔和灯
- 输入：太黑了
- 前提：没有夜灯，但有可用灯
- 预期：至少开一盏最接近的柔和灯 -> 短播报 -> 自动收尾

---

## 4. 温控附加模式

### 4A. 用户明确说冷
- 输入：我要去卫生间，有点冷
- 预期：先路径照明 -> 再轻微温控 -> 自动收尾

### 4B. 用户明确说热
- 输入：太黑了，也有点热
- 预期：先局部照明 -> 再轻微温控 -> 自动收尾

### 4C. 用户未提冷热，不应主动温控
- 输入：夜里起夜
- 预期：只做灯光和播报，不动空调/风扇

---

## 5. 自动收尾

### 5A. 路径照明默认 5 分钟后关灯
- 输入：夜里起夜
- 预期：路径灯开启 -> 5 分钟后自动关闭

### 5B. 局部照明默认 3 分钟后关灯
- 输入：帮我开一下夜灯
- 预期：夜灯开启 -> 3 分钟后自动关闭

### 5C. 用户明确说“先别关”
- 输入：帮我开一下夜灯，先别关
- 预期：开灯 -> 短播报 -> 不挂自动收尾

### 5D. 自动收尾脚本失败
- 输入：夜里起夜
- 前提：`night_light_tail.sh` 不可用
- 预期：仍然开灯并播报，但对外说明这次不会自动关

---

## 6. 极简确认

### 6A. 多个合理灯光目标，路径不清楚
- 输入：太黑了
- 前提：卧室灯、客厅灯、走廊灯都可用
- 预期：只问一个最小确认问题，例如“你是要去卫生间，还是只开床边灯？”

### 6B. 只有一个明确目标时不确认
- 输入：开一下走廊灯
- 前提：走廊灯存在
- 预期：直接执行，不额外提问

---

## 7. 串行执行验证

### 7A. 路径灯必须串行执行
- 输入：夜里起夜
- 预期：床边灯 -> 等待返回 -> 走廊灯 -> 等待返回 -> 卫生间灯 -> 等待返回 -> 播报 -> 自动收尾
- 预期：前一条命令没返回前，不应发送下一条命令

### 7B. 前一条失败，后一条继续
- 输入：我要去卫生间
- 前提：床边灯失败
- 预期：走廊灯和卫生间灯仍继续执行

---

## 8. 防误控验证

### 8A. 不应默认全屋开灯
- 输入：夜里起夜
- 预期：只开必要路径灯，不开客厅主灯或全屋灯

### 8B. 不应默认最亮
- 输入：帮我开一下夜灯
- 预期：灯光为低亮度，不应默认 100% 亮度

### 8C. 不应默认播放内容
- 输入：夜间辅助模式
- 预期：只做灯光/播报/收尾，不放音乐/新闻/故事
