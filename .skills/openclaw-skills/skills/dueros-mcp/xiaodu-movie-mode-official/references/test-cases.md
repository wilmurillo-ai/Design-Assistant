# test-cases.md

## 测试用例

---

## 1. 有观影 scene 时的完整调用链

### 1A. scene 存在且触发成功
- 输入：开始观影模式
- 前提：有高置信度观影 scene
- 预期：`list_scenes` -> `trigger_scene` -> `speak.sh` 做简短确认
- 预期用户口径：自然产品汇报，不是技术日志
- 预期价值表达：如果 scene 成功触发，应在播报里明确说出“我已经帮你把观影模式安排好了，房间和环境已经调整好了”

---

## 2. 没有 scene 时的 fallback 规划

### 2A. 完整 fallback（灯 + 窗帘 + 电视 + 播报）
- 输入：开始观影模式
- 前提：没有匹配 scene，但有灯、窗帘、电视
- 预期：`list_iot_devices` -> 灯光调暗 -> 窗帘关闭 -> 电视打开/准备 -> `list_devices` -> `speak.sh`
- 预期用户口径：自然产品汇报

### 2B. 只有灯 + 电视（无窗帘）
- 输入：开始观影模式
- 前提：没有匹配 scene，有灯和电视但没有窗帘
- 预期：灯光调暗 -> 电视打开/准备 -> `speak.sh`（跳过窗帘）
- 预期：不要因为缺少窗帘而中断整条流程

### 2C. 只有环境设备（无显示设备）
- 输入：开始观影模式
- 前提：没有匹配 scene，只有灯和窗帘，没有电视或投影
- 预期：灯光调暗 -> 窗帘关闭 -> `speak.sh` -> 告知当前没有可用显示设备
- 预期：不要假装电视或投影已经准备好

---

## 3. 显示设备准备

### 3A. 有电视，无投影
- 输入：开始观影模式
- 预期：灯光调暗 -> 窗帘关闭（如果有）-> 电视打开/准备 -> 播报确认

### 3B. 有投影，无电视
- 输入：开始观影模式
- 预期：灯光调暗 -> 窗帘关闭 -> 投影打开/准备 -> 播报确认

### 3C. 电视和投影同时存在
- 输入：开始观影模式
- 前提：电视和投影都在线
- 预期：先问“用电视还是投影？”（最小确认）
- 预期：用户回答后，再执行对应设备的准备动作

### 3D. 信号源切换
- 输入：开始观影模式
- 前提：电视支持 HDMI / 电视主页 / 机顶盒等多个信号源
- 预期：可以按偏好或默认切到最常用信号源；如果不确定，先确认

---

## 4. 内容层（条件进入）

### 4A. 用户只说“开始观影模式”（不指定内容）
- 输入：开始观影模式
- 预期：只完成环境和设备准备，不进入内容层
- 预期播报：说明当前已完成准备，可以开始选片

### 4B. 用户明确说“打开爱奇艺”
- 输入：开始观影模式，打开爱奇艺
- 预期：环境和设备准备 -> 内容层继续打开爱奇艺
- 预期播报：说明已完成准备，爱奇艺已打开

### 4C. 用户明确说“切到 HDMI”
- 输入：开始观影模式，切到 HDMI
- 预期：环境和设备准备 -> 信号源切换到 HDMI -> 播报确认
- 预期播报：说明 HDMI 已切换

### 4D. 用户明确说“播放《xxx》”
- 输入：开始观影模式，播放《星球大战》
- 预期：环境和设备准备 -> 内容层搜索/播放指定内容 -> 播报说明正在播放

---

## 5. 极简确认

### 5A. 单设备场景不需要确认
- 输入：开始观影模式
- 前提：只有一台电视在线，没有其他显示设备
- 预期：直接准备电视，不需要先问

### 5B. 有偏好时不重复确认
- 输入：开始观影模式
- 前提：用户之前已设定“默认用电视”
- 预期：直接使用偏好设备，不需要再问

### 5C. 多设备必须确认
- 输入：开始观影模式
- 前提：电视和投影都在线，用户没说用哪个
- 预期：先问一个最小确认问题再继续

---

## 6. 单点失败不中断全流程

### 6A. 灯光失败，但电视和播报可继续
- 输入：开始观影模式
- 前提：灯故障无法调暗，但电视可用
- 预期：继续电视打开/准备 -> 播报确认（不因灯光失败而整体中断）

### 6B. 电视打开失败，但窗帘和播报可继续
- 输入：开始观影模式
- 前提：电视故障，但窗帘可关，智能屏可播报
- 预期：窗帘关闭 -> `speak.sh` 说明当前状态 -> 告知电视未就绪

### 6C. 信号源切换失败
- 输入：开始观影模式
- 前提：信号源切换失败，但电视已打开
- 预期：保持电视打开状态，继续播报确认，不假装信号源已切好

---

## 7. 偏好记忆

### 7A. 用户设定默认设备偏好
- 输入：以后看电影都用投影
- 预期：偏好写入 `XIAODU_CONTEXT.md`，下次直接用投影而不重复确认

### 7B. 用户设定音量偏好
- 输入：以后看电影音量都调小一点
- 预期：偏好写入，下次自动调小音量

### 7C. 用户说“以后不用问，直接切 HDMI”
- 输入：以后看电影直接切 HDMI，不用问
- 预期：偏好写入，下次自动切 HDMI

---

## 8. 串行执行验证

### 8A. 观影流程必须串行执行
- 输入：开始观影模式
- 预期：灯光、窗帘、电视、音量、信号源、播报必须一个接一个执行
- 预期：前一条命令没返回前，不应发送下一条命令
- 预期：若前一条失败，后一条直接继续，不中断全流程
