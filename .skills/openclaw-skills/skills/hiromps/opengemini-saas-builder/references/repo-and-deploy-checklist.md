# Repo and Deploy Checklist

## Repo creation checklist

- Confirm project name
- Confirm repo visibility
- Confirm `gh auth status`
- Initialize git if needed
- Create baseline docs
- Create initial commit
- Create remote repo
- Push default branch
- Record repo URL in README or deployment notes

## Deploy checklist

- Confirm deployment was explicitly requested
- Confirm `vercel` CLI exists
- Confirm `vercel whoami` works
- Confirm framework/build settings
- Confirm env vars are documented
- Run deployment
- Record deployment URL
- Add post-deploy verification notes

## Good naming rules

- Keep repo names short and readable
- Match product or function
- Avoid dates unless useful
- Avoid generic names like `test-app-final`
