# SaaS Build Modes

Use one of these modes when shaping scope.

## 1. Idea mode
Use when the user only has a rough concept.
Output should focus on:
- positioning
- target user
- MVP
- differentiators
- risks

## 2. MVP mode
Use when the user wants a buildable first version.
Output should focus on:
- tight feature set
- architecture
- delivery order
- shortcuts that are acceptable

## 3. Production mode
Use when the user wants a more serious system.
Output should focus on:
- reliability
- auth
- observability
- CI/CD
- env separation
- deploy and rollback notes

## 4. Launch mode
Use when the user also needs landing page, copy, SEO, and deploy.
Output should include:
- product app plan
- marketing site plan
- conversion copy needs
- analytics and deployment checklist
