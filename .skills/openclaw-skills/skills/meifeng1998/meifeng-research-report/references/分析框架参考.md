# 国网人力资源管理研究分析框架参考

本文件提供常见研究主题的分析框架，供撰写报告时参考选用。实际使用中应根据项目背景和输入材料灵活调整，不要机械套用。

---

## 一、公司治理管控分析框架

### 核心分析维度
1. **股权结构层面**：出资人关系、控股比例、多元主体的权利义务
2. **管理架构层面**：委托代理链条、管理层级、管理权限的划分
3. **治理机制层面**：决策机制、监督机制、信息传导机制
4. **权责边界层面**：哪些事项由谁决策、哪些授权可以下放、哪些必须收口

### 国网特色问题
- **股权关系与管理关系分离**：持股方与实际管理方不一致，导致考核权责模糊
- **管理授权边界模糊**：省公司、地市公司、平台公司、原集体企业之间的职责界面不清晰
- **双线并行协调难题**：省级条线（省公司→省平台→省管原集体企业）与属地条线（省公司→地市公司→属地原集体企业）的协调问题

### 典型分析模型
```
【治理管控链条图】
出资人（省公司/国网总部）
    ↓ 股权管理（持股）
平台公司（晋电科技等）
    ↓ 管理授权（委托管理）
原集体企业
    ↑ 业务协同/主办关系
原主办单位（电科院/物资公司等）
```

## 二、企业功能定位分类框架

### 国有企业功能分类标准（参考国资委口径）
| 类型 | 定义 | 考核导向 |
|------|------|---------|
| 商业竞争类 | 主要承担市场竞争业务，以盈利为主要目标 | 重经营绩效、财务回报 |
| 公益保障类 | 主要承担公共服务或政策性功能 | 重服务质量、保障水平 |
| 特殊功能类 | 承担特殊战略使命（安全、能源等） | 重战略目标、安全底线 |

### 国网原集体企业分类参考维度
- **业务形态**：纯市场化业务 vs. 国网主业协同业务 vs. 劳务服务类
- **市场竞争程度**：完全竞争 / 准竞争 / 垄断服务
- **与国网主业关联度**：深度嵌入型 / 协同配套型 / 相对独立型
- **历史沿革**：原电力系统剥离企业 / 职工安置安排 / 新兴业务衍生

### 分类结果对考核设计的影响
- 功能定位→考核目标侧重点→指标体系设计→工资总额联动机制

## 三、绩效考核体系设计框架

### 指标体系设计"五步法"
1. **战略解码**：将公司战略目标分解为可量化的经营目标
2. **维度构建**：确定考核维度（通常为财务、运营、创新、团队等4-6个维度）
3. **指标遴选**：每个维度选取2-4个核心指标，遵循 SMART 原则
4. **权重设置**：根据战略优先级和企业功能定位分配权重
5. **标准制定**：为每个指标设定基准值、目标值、挑战值

### 常见指标维度参考
| 维度 | 常见指标 | 适用类型 |
|------|---------|---------|
| 财务效益 | 营收增长率、净利润率、成本控制率 | 商业竞争类为主 |
| 经营质量 | 合同履约率、客户满意度、服务及时率 | 所有类型 |
| 创新发展 | 新业务收入占比、技术成果转化数 | 科技类企业 |
| 安全合规 | 安全事故次数、合规检查通过率 | 所有类型（底线指标） |
| 人才队伍 | 关键岗位保留率、培训完成率 | 视需要 |
| 改革推进 | 改革任务完成率、"三个全面"达标情况 | 国企改革阶段 |

### 差异化考核的设计要点
- **同一指标框架，不同权重配比**：对不同类型企业，同一套指标下权重差异化
- **指标内容差异**：部分指标为特定类型企业专属
- **标准值差异**：相同指标，基准值根据企业规模/发展阶段有所不同
- **考核周期差异**：季度/年度结合，不同类企业节奏可有差异

### 考核链条设计要素
```
省公司
  ↓ ①制定总指标框架 + ②下达任务目标值
平台公司（晋电科技 / 地市平台）
  ↓ ③指标二次设计（增加特色指标）+ ④分解下达
原集体企业
  → ⑤执行落地 + ⑥过程监控
  ← ⑦结果上报 + ⑧兑现激励约束
```

**链条设计关键问题**：
- 一级考核与二级考核的界面如何划分？
- 同一指标是否存在"重复考核"风险？
- 平台公司在指标分解中的自主空间有多大？

## 四、工资总额管理框架

### 工资总额核定逻辑
```
【联动因素】
经济效益（利润总额/营收）× 增效系数
+ 劳动力市场水平对标
+ 人工成本投入产出比
+ 改革任务完成情况加成/扣减
= 工资总额核定结果（上级审批）
```

### 工资总额内部分配框架
| 项目 | 比例范围（参考） | 说明 |
|------|--------------|------|
| 固定工资（基本工资+岗位工资） | 60%-70% | 相对稳定，保障基本生活 |
| 绩效工资（浮动部分） | 30%-40% | 与考核结果挂钩 |
| 专项激励 | 0%-10% | 超额完成目标、特殊贡献等 |

### 激励约束机制设计要点
- **正向激励上限**：防止绩效工资占比过高导致"保底逻辑"失效
- **负向约束底线**：达不到基准值时工资总额缩减的幅度
- **分配系数**：高低岗位之间的工资倍差（如1:2.5或1:3）
- **兑现时机**：年度考核后一次性兑现 vs. 分季度预兑现

### 国网体系特殊性
- 工资总额受主管单位（地市公司/省公司）核定约束，企业自主分配空间受限
- "双轨制"历史：改制前后人员待遇标准存在历史差异
- 劳务外包人员（如众辉公司）工资管理与在册员工存在差异

## 五、原集体企业改革路径分析框架

### 改革历程阶段划分（参考）
| 阶段 | 时间背景 | 主要任务 |
|------|---------|---------|
| 初步清查阶段 | 2010年代初 | 摸清家底、确定改革范围 |
| 规范整合阶段 | 2015-2019 | 清产核资、公司化改制、集中管理 |
| 深化改革阶段 | 2020年至今 | "三个全面"、股权规范、治理完善 |

### "三个全面"政策分析框架
（注：具体内容以国网总部实际文件为准，以下为参考框架）
- **全面清查**：历史资产、人员、业务的彻查摸底
- **全面规范**：治理结构、劳动关系、财务管理的规范化
- **全面整合**：低效企业的整合、退出和资源优化配置

### 历史遗留问题分析维度
1. **产权历史遗留**：集体所有制的历史来源与现代公司制改造之间的张力
2. **人员遗留问题**：老职工待遇、历史欠账、身份认同等问题
3. **管理关系遗留**：股权关系、挂靠关系、主办关系历史演变留下的管理真空
4. **制度遗留**：原有规章制度与新要求之间的衔接问题

### 改革路径评估框架
```
现状诊断 → 问题归因 → 路径选择 → 方案设计 → 推进策略
         ↑                        ↑
    历史沿革分析              合规性审查（政策约束）
```

## 六、管理模式优化通用框架

当项目目标是"优化某种管理模式"时，报告通常遵循以下分析路径：

```
一、现状梳理
    （一）现行模式的主要特征
    （二）各方主体的角色与职责
    （三）运行机制与流程

二、问题诊断
    （一）主要问题识别（基于调研数据）
    （二）问题成因分析（制度性 / 机制性 / 历史性）
    （三）问题的影响与风险

三、对标借鉴
    （一）国内外（或系统内）先进经验
    （二）可供借鉴的做法与启示
    （三）在[本单位]情境下的适用性分析

四、优化方案
    （一）优化方向与总体思路
    （二）具体优化举措
        1. [举措一]
        2. [举措二]
        3. [举措三]
    （三）实施路径与时序安排

五、保障机制
    （一）组织保障
    （二）制度保障
    （三）资源保障
```

## 七、报告类型与对应重点

| 报告类型 | 核心输出 | 主要章节重点 |
|---------|---------|------------|
| 现状调研报告 | 摸清家底、发现问题 | 现状描述、问题归因 |
| 机制研究报告 | 理论+实践分析 | 理论基础、对标分析、机制设计 |
| 方案设计报告 | 可操作的改进方案 | 问题诊断、方案设计、实施路径 |
| 实施手册 | 操作指导 | 流程说明、工具方法、注意事项 |
| 综合研究报告 | 全面研究成果 | 以上各章综合，逻辑闭合 |
