## 使用说明

### 严重性级别定义

| 级别 | 含义 | 是否阻塞合并 |
|------|------|-------------|
| `[CRITICAL]` | 安全漏洞、数据丢失、崩溃 | 是 |
| `[MAJOR]` | Bug、逻辑错误、显著性能问题 | 是 |
| `[MINOR]` | 改进建议、维护成本优化 | 否 |
| `[NIT]` | 风格偏好、命名建议 | 否 |

### 审核维度

1. **安全性** - SQL 注入、XSS、认证授权、密钥管理、输入验证
2. **并发安全** - 线程安全、竞态条件、锁机制
3. **代码质量** - 命名规范、代码结构、重复代码、注释
4. **可维护性** - DRY 原则、魔法数字、函数长度、依赖方向

### 报告编写原则

1. **只列出问题** - PASS 的检查项不需要列出
2. **具体明确** - 指出具体文件、行号、代码片段
3. **说明风险** - 解释为什么这是问题
4. **给出建议** - 提供具体的修复方案或代码示例
5. **分级清晰** - 使用 [CRITICAL]/[MAJOR]/[MINOR]/[NIT] 标记

### 示例对比

❌ **不好的反馈**:
> 这里有问题，修复一下

✅ **好的反馈**:
> **[MAJOR] Random 实例重复创建**
> - **文件**: `BusinessProperties.java:45`
> - **代码**: `Random random = new Random();`
> - **风险**: Random 不是线程安全的，高并发下可能产生相同结果
> - **建议**: 使用 `ThreadLocalRandom.current()`
