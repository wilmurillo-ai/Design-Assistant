## ⚡ 最快中断方式

### 方式 1：直接说关键词（推荐）
```
用户：中断
用户：停止
用户：别做了
用户：任务不对
用户：方向错了
```
**响应时间：** <1 秒  
**Token 节省：** 立即停止，不浪费额外 token

## 🎯 什么时候应该中断？

### ✅ 立即中断
- [ ] 任务方向明显错误
- [ ] 需求理解有误
- [ ] 缺少关键信息
- [ ] 发现更简单方案
- [ ] 外部依赖失败

### ⏸️ 暂停确认
- [ ] 执行速度慢（可能在等待 API）
- [ ] 即将完成（>80%）
- [ ] 只是小问题

### ❌ 不建议中断
- [ ] 正常执行中
- [ ] 只差最后一步
- [ ] 只是输出格式问题

## 🆘 常见问题

**Q: 中断后能恢复吗？**  
A: 不能，但会保存中断记录，可以重新开始

**Q: 中断会清理所有资源吗？**  
A: 默认清理子代理和后台进程，临时文件可选

**Q: 如何确认中断成功？**  
A: 会收到"🛑 已中断"的确认消息

**Q: 中断后能立即执行新任务吗？**  
A: 可以，清理完成后立即可用
