# 五流合一企业合规经营管理系统

## ⚠️ 数据采集原则（强制）

**所有业务数据必须通过上传原始文件采集，禁止手动输入。**

这是合规经营的核心要求：
- 原始文件是档案依据，必须留存
- 手动输入无凭据，税务稽查无法溯源
- 系统只接受文件上传，自动识别解析

### 支持的文件类型

| 数据类型 | 文件格式 | 说明 |
|---------|---------|------|
| 银行流水 | CSV, Excel, PDF | 银行导出的交易明细 |
| 发票 | PDF, 图片(JPG/PNG) | 进项/销项发票原件 |
| 合同 | PDF, Word, 图片 | 采购/销售合同扫描件 |
| 项目资料 | PDF, Excel | 项目预算、结算文件 |

### 文件处理流程

```
用户上传文件 → 系统识别解析 → 提取业务数据 → 原件归档保存 → 数据入库
```

### 使用方式

```bash
# 上传银行流水
python main.py upload bank_statement.csv

# 上传发票
python main.py upload invoice.pdf

# 上传合同
python main.py upload contract.pdf --type contract

# 查看归档文件
python main.py archives
```

## 五流合一

系统自动进行以下核对：

1. **业务流 ↔ 合同流**：项目预算与合同金额匹配
2. **合同流 ↔ 票据流**：合同金额与发票金额匹配
3. **票据流 ↔ 资金流**：发票金额与银行流水匹配
4. **资金流 ↔ 财务流**：银行流水与记账凭证匹配
5. **财务流 ↔ 税务流**：账务数据与申报数据匹配
