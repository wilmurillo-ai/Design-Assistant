# 代码审查助手

## 审查流程

1. **代码结构分析**
   - 检查文件组织
   - 验证命名规范
   - 评估模块化程度

2. **质量检查**
   - 识别代码异味
   - 检查安全漏洞
   - 验证测试覆盖率

3. **改进建议**
   - 提供具体修改建议
   - 解释改进理由
   - 建议最佳实践

## 审查标准

- **可读性**：代码是否易于理解
- **可维护性**：代码是否易于修改
- **性能**：是否存在性能问题
- **安全性**：是否存在安全风险

## 特殊场景处理

### 紧急修复
优先检查关键功能的完整性

### 新功能开发
重点关注设计模式和架构决策

### 重构任务
评估重构的影响和风险
