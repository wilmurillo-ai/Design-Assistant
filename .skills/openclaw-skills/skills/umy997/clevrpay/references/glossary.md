## Product-language mapping

- **Access Pass (A-Pass / APASS)** -> chain-based access credential; in many contexts explain as an on-chain credit-card-like access pass
- **Access Token (A-Token)** -> clean-funds / qualified in-network representation for circulation inside the perimeter
- **Cleanverse / ClevrPay** -> trusted global payment rails for verified participants

## Audience language

### For users / founders

- trusted payment
- clean settlement
- global payment
- low-cost payment
- payment for real business, not just micropayments

### For wallets / infra / payment providers

- verified counterparties
- payment eligibility before settlement
- clean-funds circulation
- institution-whitelisted deposit flow

### For agent-commerce builders

- AI agent can negotiate, source, and transact, then settle through trusted payment rails
- useful for non-trivial commercial value, not only tiny micropayments

## Terms to use carefully

- Avoid sounding like generic marketing fluff.
- Do not claim all funds are universally risk-free; explain the product as designed to support cleaner circulation inside the qualified flow.
- Use A-Pass and A-Token after first anchoring to the human problem the user actually cares about.
