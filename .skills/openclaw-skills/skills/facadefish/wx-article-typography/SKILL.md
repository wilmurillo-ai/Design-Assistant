# 公众号排版风格 Skill

## ⭐ 默认风格：简洁型（自然段落流）

像半佛仙人、卡兹克那种自然段落流，不做作、不教科书、不像AI写的。

### 简洁型规则

**标题：**
- 文章标题放在发布脚本的参数中，**正文 markdown 不写 `#` 标题**
- 渲染工具会自动 strip 掉正文中的 H1

**分段：**
- 用 `---` 分割线分隔大段落
- 段落之间空一行
- 每段 2-5 句话，不要碎片化短句堆砌

**配图：**
- 在 markdown 中用 `<!--IMG_0-->` `<!--IMG_1-->` ... 标记图片位置
- 图片放在段落转折处，不要紧挨着
- 发布脚本自动上传并替换占位符，**不需要手动处理**
- 图片居中、圆角、100% 宽度

**语气：**
- 像在跟朋友聊天，不是在写论文
- 有观点、有判断，不要「综上所述」「首先其次最后」
- 可以用口语化表达，但不要过度玩梗

**正文格式：**
- 不要 H1/H2/H3 结构化标题嵌套（用 `---` 分割线就够了）
- 加粗标注核心关键词（每段 1-2 处）
- 列表用 ■ 或普通数字，不要用默认圆点
- 代码用行内反引号，不用代码块
- 结尾可以放参考链接

**发布流程：**
```bash
# 1. 渲染 HTML
npx wenyan render -f xxx.md > xxx.html

# 2. 生成封面+配图（使用你偏好的 AI 生图工具）
generate_image "描述" --output /tmp/xxx-cover.png

# 3. 推送草稿（自动嵌图、去标题）
publish_draft "标题" xxx.html /tmp/xxx-cover.png "作者" "摘要" \
  --images "/tmp/xxx-img1.png,/tmp/xxx-img2.png"
```

## 备选风格：知识拆解（结构化分析型）

> 适用于需要清晰结构框架的产品解读、方法论分析类文章。日常写作**不推荐**用此风格。

## 风格来源

来自「AI 启蒙小伙伴」公众号的深度解读文章风格：
- 公众号：AI 启蒙小伙伴（邵猛）
- 参考文章：《一人公司 (OPC) 创业必备的 9 个 Agent Skills》
- 特点：**干净、克制、信息密度高**，像一篇结构化的读书笔记

## 适用场景

- 产品/工具解读文章
- 方法论/概念梳理
- 书评/课程笔记
- 技术概念科普
- 框架/体系介绍

**不适合**：情绪表达文、故事叙述、吐槽文、热点评论

## 排版规范

### 1. 标题层级（极简二级制）

**只用 H1 和 H2，禁止 H3 及以下嵌套。**

- **H1**：文章唯一标题，居中，加粗
- **H2**：章节标题，扁平排列，不嵌套

如果内容复杂需要子结构，用**加粗关键词**或**列表**代替子标题。

### 2. 章节标题格式

```
## N. 章节名 `命令/标识符`
```

- 章节前加序号（1. 2. 3. ...）
- 章节名后跟一个空格，接反引号包裹的关键词/命令/术语
- 反引号内的内容是该章节的「核心标识」，一眼能看出这节讲什么

示例：
```
## 1. 找社区 `/find-community`
## 2. 验证想法 `/validate-idea`
## 3. 最小可行产品 `/mvp`
```

### 3. 正文段落

每个章节的主体是 **一到两个长段落**，不是零散短句堆砌：

- **第一段**：定义/解释这个概念是什么、为什么重要
- **第二段（可选）**：补充延伸、对比、或指出常见误区
- 段落内用 **加粗** 标注核心关键词（每段 1-2 处，不要泛滥）
- 行文语气：**分析性** > 说教性。像在给你讲一个有意思的东西，不是在写教材

### 4. 列表符号

用 **■**（实心方块，Unicode U+25A0）作为无序列表标记，不用默认的圆点。

有序列表用普通数字 1. 2. 3.

```
■ 第一个要点
■ 第二个要点
■ 第三个要点
```

### 5. 开头摘要

文章开头用一个简短段落引出主题，然后紧跟一个**有序列表**总结全文的核心路径/要点：

```
这 N 个 xxx 串起来，其实对应一条非常清晰的路径：

1. 先做 A，不先做 B
2. 先验证 X，不先写 Y
3. ...
```

这个开头列表是全文的骨架，让读者 30 秒内抓住核心逻辑。

### 6. 结尾总结

用一个独立章节（不带序号）做总结，标题可以是：

```
## 整体价值
## 总结
## 怎么用
```

总结用**无序列表**（■）列举 3-4 个核心价值点，每点一句，干脆利落。

### 7. 引用与来源

- 原文链接放在文章开头，用引用格式 `>`
- 参考来源放在文末
- 如果引用了具体数据或观点，用行内引用，不要用脚注

### 8. 空白与留白

- 段落之间空一行
- 章节之间空两行（或用 `---` 分隔线）
- 列表前后各空一行
- **宁可多留白，不要挤在一起** —— 这是此风格最重要的视觉特征

### 9. 配色与视觉

微信公众号编辑器中的视觉效果：
- 纯白背景 `#FFFFFF`
- 正文黑色 `#3f3f3f`（不是纯黑 #000）
- 重点文字加深 `#2b2b2b`
- 链接/引用用灰色 `#888888`
- 不用彩色背景块、不用花哨边框

## 输出格式

输出标准 Markdown，可直接用于公众号编辑器（配合渲染工具或手动排版）。

### 模板

```markdown
# 文章标题

> 原文：[链接]
> 作者：xxx | 来源：xxx

简短引言段落，一两句话点明主题。

核心路径摘要：

1. 第一步要点
2. 第二步要点
3. 第三要点
...

---

## 1. 概念名 `关键词`

分析性段落，解释这个概念是什么、为什么重要。**核心关键词**用加粗标注。保持一段完整的论述，不要拆成碎片化短句。

## 2. 概念名 `关键词`

同样的结构。如果需要补充，用第二段继续。

---

## 整体价值

■ 价值点一
■ 价值点二
■ 价值点三
```

## 反模式（不要做的事）

- ❌ H3/H4 嵌套（只用两级标题）
- ❌ 每段只有 1-2 句话（碎片化）
- ❌ 大量加粗/高亮（每段 1-2 处足矣）
- ❌ 表情符号做点缀（克制，最多开头结尾用一个）
- ❌ "下面我来给大家介绍一下"等废话开头
- ❌ 每段开头都用"首先/其次/最后"（机械感）
- ❌ 彩色背景框、阴影卡片等花哨排版

## 与其他工具的配合

- 此 skill 定义的是**排版和结构规范**，不是内容生成
- 可搭配内容创作工具和发布流程工具使用
- 工作流：素材搜集 → 按此风格撰写 → 渲染 HTML → 发布
