# HackerOne Web3 Reporting Guidance

## Triage Focus

- trust boundary violation
- reproducibility
- realistic impact

## Report Structure

1. Summary
2. Affected Components
3. Technical Root Cause
4. Reproduction Sequence
5. Impact
6. Severity Justification
7. Remediation

## Common Rejection Patterns

- hypothetical exploit chains
- missing reproduction details
- ambiguous scope mapping
