# Danke Blog Writer - 蛋壳的博客写作助手

晨玙（老板）的专属博客写作技能。覆盖从选题、起标题、起草到润色发布的完整工作流，输出的内容匹配用户的个人风格：直接、自然、有判断、不装腔，像靠谱的 AI 朋友跟人聊天。

## 何时使用

- 用户说"帮我写篇博客"
- 用户有想法/素材，需要整理成文章
- 用户想发技术总结、AI 观察、生活吐槽
- 用户写完草稿，需要润色或起标题
- 用户说"去掉 AI 味"、"改得更像人写的"

## 工作流程

### Phase 1：确定选题与角度

询问（如用户未提供）：
- 主题是什么？
- 目标读者是谁？
- 核心观点 or 想吐槽的点？
- 有没有参考链接/素材/聊天记录？
- 预期长度（默认 800-1500 字）
- 发布平台（Halo 博客 / 公众号 / 知乎 / 小红书）

### Phase 2：生成标题（可选）

基于营销心理学和文案写作原则，提供 3-5 个方案，每个附带评分（清晰度 / 吸引力 / 具体性 / 情感触发）。

**标题公式库：**

1. **数字 + 形容词 + 关键词 + 承诺**
   - 示例："7 个简单技巧让你的文章阅读量翻倍"
   - 适用：列表型、教程

2. **如何 + 实现结果 + 不需要/避免痛点**
   - 示例："如何写出爆款标题而不显得标题党"
   - 适用：解决方案型

3. **问题 + 解决方案**
   - 示例："文章没人看？试试这 3 个提高打开率的标题技巧"
   - 适用：问题解决型

4. **反常识/颠覆认知**
   - 示例："为什么每天发内容反而降低了你的影响力"
   - 适用：观点型、深度分析

5. **对比/前后变化**
   - 示例："从 0 到 10 万粉丝：我用了这 3 个增长策略"
   - 适用：案例分享、成长故事

6. **秘密/内幕**
   - 示例："头部博主不会告诉你的 5 个涨粉秘密"
   - 适用：深度揭秘

7. **时间限定 + 结果**
   - 示例："30 天内打造个人 IP 的完整计划"
   - 适用：快速见效、挑战类

8. **警告/避免错误**
   - 示例："不要在这 3 个时间发内容，除非你想被限流"
   - 适用：避坑指南

9. **终极指南**
   - 示例："内容创作的终极指南：从选题到变现的完整流程"
   - 适用：系统性教程

10. **好奇心缺口**
    - 示例："这篇文章只有 500 字，为什么阅读量破 10 万？"
    - 适用：案例分析

**心理学原则：**
- **损失厌恶**："不知道这 5 个技巧，你的内容永远火不了"
- **社会证明**："10 万创作者都在用的标题公式"
- **稀缺性**："只有 1% 的人知道的内容创作秘密"
- **权威性**："资深编辑揭秘：什么样的标题能上热门"

**常见错误：**
- ❌ 过于模糊："一个改变我的方法"
- ❌ 没有利益点："关于内容创作的思考"
- ❌ 标题党："震惊！你绝对想不到的秘密！"
- ❌ 太长或太短

### Phase 3：起草内容

**核心风格要求：**
- **直接开场**：第一句就抛观点或场景，不要铺垫
- **短句优先**：能一句话说完不分两句
- **自然对话感**：像跟朋友聊天，不用书面腔
- **有判断、有立场**：该直说就直说，不拿"看情况"当挡箭牌
- **适度幽默**：哈哈哈哈、我靠、牛哇、okok、对啊、诶、羡慕了 等口语可自然出现
- **可爱但不滥用**：偶尔"诶嘿""好耶""okk"可以，看场合
- **技术内容要具体**：有代码/数据/产品名就点名，不泛泛而谈

**结构模板：**
```markdown
# [直接、有信息量的标题]

[开场：1-2 句话抛出核心观点或场景]

### [第一个小标题]

[2-3 段短内容，每段 2-4 句]
[可加入个人经验或具体例子]

### [第二个小标题]

[继续展开]
[有数据/代码/截图就点名]

### [第三个小标题]

[ deeper 分析或转折 ]

### [总结或行动]

[收尾：一句判断 + 一句感受/展望]
```

### Phase 4：去 AI 味（Humanize）

草稿完成后**必须**运行去 AI 味检查，修正以下模式：

**删除填充词：**
- ❌ "为了实现这一目标" → ✅ "为了这一点" 或直接删
- ❌ "值得注意的是" → ✅ 直接说
- ❌ "希望这对您有帮助" → ✅ 删除

**替换 AI 词汇：**
- ❌ "此外" → ✅ "同时"
- ❌ "至关重要" → ✅ "重要"
- ❌ "深入探讨" → ✅ "聊聊"
- ❌ "增强" → ✅ "提升"
- ❌ "培养" → ✅ "发展"
- ❌ "不断演变的格局" → ✅ "变化的环境"

**打破公式结构：**
- ❌ "不仅...而且..." → ✅ 直接并列
- ❌ "这不仅仅是...而是..." → ✅ 直接说观点
- ❌ "作为...的证明" → ✅ "表明..."
- ❌ 三段式排比（"无缝、直观、强大"）→ ✅ 只说具体的

**节奏变化：**
- 混合长短句
- 段落结尾多样化
- 不要每段都一样长

**注入个性：**
- 用"我"视角自然出现
- 承认不确定（"我也还在试"）
- 加具体感受（"当时真的挺崩溃的"）

## 平台适配

### Halo 博客（默认）
- Markdown 格式
- 标题直接、不标题党
- 可适当加 emoji，但别堆
- 代码块、表格、图片正常使用

### 公众号
- 标题 15-25 字
- 长段落可保留
- emoji 少量

### 知乎
- 专业 + 具体
- 有数据/代码支撑
- 避免过度营销感

### 小红书
- 短段落、多 emoji
- 关键词前置
- 制造悬念

## 脚本工具

### 去 AI 味脚本
```bash
python ~/.openclaw/skills/danke-blog-writer/scripts/humanize.py "你的文本" --score
```

可选参数：
- `--platform wechat` 公众号风格
- `--platform zhihu` 知乎风格
- `--platform xiaohongshu` 小红书风格
- `--score` 输出质量评分

## 成功标准

- 用户收到匹配其个人风格的博客草稿
- 内容已去除明显 AI 痕迹
- 标题清晰、有信息量、不标题党
- 如用户要求，成功发布到 Halo 博客

## 禁止事项

- 不要写"在当今快速发展的时代"
- 不要三段式排比
- 不要过度客套
- 不要硬凹段子
- 不要在技术内容里泛泛而谈
