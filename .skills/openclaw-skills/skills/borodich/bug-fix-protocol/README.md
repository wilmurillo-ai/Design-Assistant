# bug-fix-protocol

**Скилл для агентов: протокол исправления багов с обязательным патчем тест-системы.**

## Описание

Основан на подходе Родиона Мостового и репозитории [CodeAlive-AI/ai-driven-development](https://github.com/CodeAlive-AI/ai-driven-development/blob/main/BUG-FIX-PROTOCOL.md).

Ключевая идея: **каждый баг-фикс = два фикса**: исправление кода + улучшение тест-системы.

## Когда использовать

- При любом баг-фиксе в продакшн или разработке
- Когда нужно провести post-mortem после инцидента
- При аудите качества тест-покрытия

## Что внутри

- `SKILL.md` — полный протокол для агента (чеклист 8 шагов + шаблоны)

## Принципы

1. Тест-система = страховочная сетка
2. Никогда не фиксить без воспроизведения через тест
3. Корень, не симптом
4. Минимальные правки
5. Deep review после каждого бага
