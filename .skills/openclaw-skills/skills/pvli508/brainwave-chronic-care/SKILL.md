# 慢病脑波声疗助手 🧠

为慢病患者提供脑波音频播放、健康辅助服务。支持高血压、糖尿病、睡眠障碍、心脑血管、康复通用等慢病类型的脑波音频播放。

## 基础配置

| 配置项 | 值 |
|--------|-----|
| Skill ID | hc_brainwave_chronic_v1 |
| 类型 | 内容服务 + 企业工具 |
| 运行平台 | 音箱 / 大屏 / 健康一体机 / 小程序 SDK |
| 租户模式 | 多租户 SaaS |

## 合规声明

**全局强制挂载**：本音频为非药物健康辅助，不替代医疗诊断、治疗及用药

所有话术必须包含此声明。

## 意图定义

### 意图 1：PLAY_BRAINWAVE_AUDIO - 播放慢病脑波音频

**用户说法**：
```
播放高血压舒缓脑波音频
给糖尿病患者播放助眠脑波
打开睡前慢病修复音频
播放 15 分钟放松脑波音频
播放心脑血管舒缓音频
打开康复期放松脑波
播放餐后慢病调理音频
播放睡眠障碍脑波音频
播放脑波音频
```

**槽位**：
| 槽位名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chronic_type | 枚举 | 是(可交互) | 慢病类型：高血压、糖尿病、睡眠障碍、心脑血管、康复通用 |
| scene | 枚举 | 否 | 使用场景：睡前、餐后、午休、情绪烦躁、康复期、日间放松 |
| duration | 数字 | 否 | 播放时长：10/15/20/30 分钟，默认 15 |

**执行逻辑**：
1. 校验租户权限
2. 检查 chronic_type 槽位：
   - **有槽位**：匹配慢病 + 场景音频 → 返回播放地址 → 播放
   - **无槽位**：进入交互确认流程，引导用户选择慢病类型
   - **交互失败/未选择**：播放「康复通用」类型音频
3. 更新用户最近使用记录 (last_used_time)
4. 上报播放日志

**多用户设备策略**：最近使用优先 (Last Used Priority)
- 记录每个用户的最近使用时间和慢病类型
- 用户未指定类型时，自动调用最近一次使用记录
- 多用户时以最后一次播放时间排序

## 接口清单

### OpenClaw Skill 网关接口

| 接口 | 地址 | 说明 |
|------|------|------|
| 鉴权 | POST /openclaw/skill/auth | app_id, device_id, timestamp, sign → token |
| 意图执行 | POST /openclaw/skill/intent | user_id, tenant_id, intent_id, slots → reply_text, audio_url, control_cmd, timer |
| 状态回调 | POST /openclaw/skill/callback | log_id, status, timestamp → success |

### 内部业务接口

| 接口 | 地址 | 说明 |
|------|------|------|
| 音频匹配 | POST /business/audio/match | 根据慢病+场景+时长匹配音频 |
| 签名生成 | POST /audio/sign/generate | 生成 CDN 时效签名（HMAC-SHA256，6小时） |
| 日志上报 | POST /log/play/upload | 播放日志上报 |
| 租户配置 | POST /tenant/config/get | 获取租户配置 |
