# 实习报告使用技巧与最佳实践

## 📋 实习报告写作核心原则

### 1. 告别流水账
最大的痛点就是把实习报告写成"今天做了A，明天做了B"的流水账。
解决方案：用 **结构化框架** 替代流水记录。

- 每件事都问自己三个问题：
  1. 为什么做？（背景和目的）
  2. 怎么做的？（方法和过程）
  3. 做出了什么？（结果和收获）

- 用动词开头：负责、参与、主导、完成、优化、协助
- 用数据说话：3个页面、2周内完成、响应时间降低40%

### 2. 写作金字塔
好的实习报告结构：
```
       总结观点
      /    |    \
   论点1  论点2  论点3
  /  \   / \   /  \
事例  事例 事例  事例
```

## 🎯 使用场景速查

### 场景A：每日/每周记录
```bash
# 日报 — 每天花5分钟记录
intern.sh daily "参与需求评审,完成登录页开发,修复2个Bug"

# 周报 — 每周五花15分钟总结
intern.sh weekly "完成用户模块,学习React Hooks,参与代码评审"
```
💡 坚持日报可以为最终总结积累素材！

### 场景B：写实习总结（最常见）
```bash
# 第一步：生成框架
intern.sh summary "公司名" "岗位" "天数"

# 第二步：根据框架填充内容
# 第三步：做深度反思，提炼亮点
intern.sh reflect "你做过的事情列表"
```

### 场景C：准备答辩
```bash
# 生成8页PPT大纲
intern.sh defense "你的实习工作内容"

# 包含：
# - 每页的要点提示
# - 讲述思路建议
# - 常见答辩问题及应对
```

### 场景D：深度反思与职业规划
```bash
# 从经历中提炼可迁移能力
intern.sh reflect "参与产品开发,学习敏捷流程,独立完成模块设计"

# 输出：
# - WHAT-SO WHAT-NOW WHAT反思模型
# - 可迁移能力清单
# - 职业规划思考框架
# - 行动计划
```

## 🏆 答辩PPT技巧

### PPT设计原则
- **每页只讲一个要点**，不要堆砌文字
- 正文字号不小于24pt，标题不小于32pt
- 使用图表代替文字墙
- 配色统一，不要太花哨

### 答辩演讲技巧
- 控制在8-10分钟
- 提前练习3遍以上
- 第4、5页（工作内容）花最多时间讲
- 准备好常见问题的答案

### 常见答辩问题
1. 你在实习中最大的收获是什么？
2. 遇到最大的困难是什么？怎么解决的？
3. 如果重新来过，你会怎么做？
4. 这段实习对你的职业规划有什么影响？
5. 学校教育和实际工作的差距在哪里？

### 回答框架：STAR法则
- **S**ituation：当时是什么情况
- **T**ask：我的任务/目标是什么
- **A**ction：我具体做了什么
- **R**esult：结果如何

## 🔍 深度反思功能详解

`reflect` 命令使用 **WHAT-SO WHAT-NOW WHAT** 反思模型：

1. **WHAT**（发生了什么）
   - 客观回顾经历
   - 识别关键事件

2. **SO WHAT**（这意味着什么）
   - 分析自己的优势和劣势
   - 理解自己的工作价值观
   - 发现适合自己的工作环境

3. **NOW WHAT**（接下来怎么做）
   - 制定具体的行动计划
   - 明确短中长期目标

### 可迁移能力自动提取
系统会根据你的经历自动匹配相关的可迁移能力：
- 开发类 → 编程能力、逻辑思维、问题拆解
- 设计类 → 审美能力、用户思维、创意表达
- 产品类 → 需求分析、用户研究、项目推进
- 运营类 → 数据分析、内容创作、活动策划

## 📊 实习报告评分标准（参考）

| 维度 | 占比 | 要点 |
|------|------|------|
| 内容完整性 | 30% | 工作内容描述清晰、有深度 |
| 收获与反思 | 25% | 有真实的感悟，不是套话 |
| 专业性 | 20% | 展现了专业知识的应用 |
| 结构与表达 | 15% | 逻辑清晰、格式规范 |
| 态度与参与度 | 10% | 积极主动、有独立思考 |

## 💡 高分秘诀

1. **数据化表达**
   - ❌ "完成了很多任务"
   - ✅ "独立完成5个前端页面开发，累计代码量3000+行"

2. **真实的反思比完美的结果更重要**
   - 老师更看重你的思考过程，而不是你做了多牛的事

3. **坚持记录**
   - 每天用 `daily` 记录，周末用 `weekly` 总结
   - 最后写总结时不用回忆，直接整合

4. **适度展现个性**
   - 不要写八股文，加入你自己的真实感受
   - 一个有温度的故事胜过十个干巴巴的数据

5. **配合其他工具**
   - 用 `ppt-outline` 做更精美的答辩PPT
   - 用 `job-hunter` 把实习经历包装进简历
