# Test & Publish Checker - tips.md

## 发布事故TOP5及预防

1. **环境变量漏配** → deploy命令检查，.env.example同步
2. **数据库Migration未执行** → 自动化Migration，发布流水线集成
3. **缓存未清理** → CDN purge + 版本化静态资源
4. **回滚方案未测试** → 每次发布前演练回滚
5. **没人值班** → 发布日必须有值班oncall

## 发布时间选择

| 推荐 | 不推荐 |
|------|--------|
| 周二~周四 上午10点 | 周五下午 |
| 工作日白天 | 节假日前一天 |
| 团队齐全时 | 凌晨(无人响应) |

## SemVer速查

- `1.0.0` → 第一个稳定版
- `1.1.0` → 新功能,向后兼容
- `1.1.1` → Bug修复
- `2.0.0` → 不兼容的改动
- `1.0.0-beta.1` → 预发布

## 灰度发布节奏

1. **1%** — 观察10分钟,看error rate
2. **10%** — 观察30分钟,看核心指标
3. **50%** — 观察1小时,看性能
4. **100%** — 全量,持续监控24h
