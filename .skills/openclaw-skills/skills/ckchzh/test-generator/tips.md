# Test Generator Tips

1. **Follow the AAA pattern** — Every test should have Arrange (setup) → Act (execute) → Assert (verify), keeping structure clear and readable
2. **One assertion per test** — Unit tests should verify a single behavior; when they fail, you immediately know what broke
3. **Always test boundaries** — Use the `edge` command to generate: null values, zero, max values, negatives, special characters
4. **Mock only external dependencies** — Mock databases, APIs, and file systems, never mock the code under test itself
5. **Reuse fixtures** — Use the `fixture` command to generate shared test data instead of duplicating setup in every test
6. **Coverage != Quality** — 80% coverage is a good target, but covering critical paths and edge cases matters more than hitting a number
7. **Set performance baselines** — Tests generated by `benchmark` should include baseline thresholds for automatic CI regression detection
8. **Refactor tests too** — Test code is still code; periodically clean up duplication and remove obsolete tests
