# 公文写作使用技巧与最佳实践

## 📋 公文写作核心原则

### 1. 格式规范
公文有严格的格式要求，不是自由写作。
- 标题：发文机关 + 事由 + 文种
- 正文：缘由 → 事项 → 要求
- 排版：字体、字号、页边距都有标准

### 2. 用语正式
公文是正式的行政文书，杜绝口语化。
- ❌ "我觉得差不多可以搞一下"
- ✅ "经研究决定，拟开展..."

### 3. 一事一文
- 通知只通知一件事
- 请示只请示一件事
- 不要在一篇公文里夹带多个主题

## 🎯 使用场景速查

### 场景A：写公文
```bash
# 通知
official.sh notice "关于开展安全检查的通知" "安全生产"

# 请示
official.sh request "购置设备" "需要购置电脑10台"

# 报告
official.sh report "上半年工作" "各项工作进展"

# 批复
official.sh reply "关于购置设备的请示" "同意购置"
```

### 场景B：检查已有公文
```bash
# 格式检查
official.sh format-check "你的公文文本"

# 语气检查
official.sh tone "你的公文文本"
```

### 场景C：查模板
```bash
# 查看所有模板类型
official.sh template "all"

# 查看特定类型
official.sh template "通知"
official.sh template "请示"
official.sh template "会议纪要"
```

## 📊 格式检查功能详解

`format-check` 命令检查7大维度：
1. **标题规范** — 是否包含"关于...的通知/报告/请示/批复"
2. **发文字号** — 是否有标准编号格式
3. **主送机关** — 是否明确行文对象
4. **日期标注** — 是否有发文日期
5. **结尾用语** — 是否使用规范结尾
6. **签章区域** — 是否有落款
7. **段落编号** — 是否使用规范编号

输出包含：
- 格式评分（0-100分）
- 发现的具体问题
- 改进建议
- 标准格式规范参考

## 🗣️ 语气检查功能详解

`tone` 命令检查30+种口语化表达，自动推荐公文替换用语：

### 常见替换对照
| 口语化 | 公文用语 |
|--------|---------|
| 我觉得 | 经研究认为 |
| 大概 | 约/预计 |
| 搞 | 开展/推进 |
| 要 | 应/须/需 |
| 不行 | 不可/不宜 |
| 马上 | 立即/即刻 |
| 然后 | 随后/继而 |
| 所以 | 因此/故 |
| 这个 | 该/此/本 |
| 老板 | 领导/负责人 |
| 嗯/吧/呢 | [删除] |

### 公文常用句式
**开头：**
- "根据...精神/要求"
- "为了进一步..."
- "为贯彻落实..."

**过渡：**
- "经研究决定..."
- "现将有关事项通知如下..."

**要求：**
- "各单位要..."
- "务必..."
- "切实做好..."

**结尾：**
- "特此通知"
- "特此报告"
- "妥否，请批示"

## 📄 模板库功能详解

`template` 命令提供6种公文模板：

### 通知
向下级/平级传达事项。
结构：缘由 → 事项（分条列出） → 要求

### 报告
向上级汇报工作/反映情况。
结构：基本情况 → 做法和成效 → 问题 → 计划

### 请示
向上级请求指示/批准。
结构：缘由 → 请示事项 → 预算（如涉及）
⚠️ 一文一事，只能主送一个机关

### 批复
上级对下级请示的答复。
结构：引叙原文 → 批复意见 → 具体要求

### 会议纪要
记录会议情况和议定事项。
结构：概况 → 议题讨论 → 决定事项 → 任务分工

### 工作总结
总结一定时期的工作。
结构：总体评价 → 工作成绩 → 经验做法 → 问题不足 → 下步计划

## 💡 公文写作实战技巧

### 标题怎么写
```
格式：发文机关 + 关于 + 事由 + 的 + 文种

✅ 国务院关于加强城市基础设施建设的意见
✅ XX公司关于开展年度安全检查的通知
❌ 安全检查通知（缺少发文机关和事由）
```

### 段落编号规范
```
一、（一级标题，顶格）
  （一）（二级标题，空两格）
    1.（三级标题，空两格）
      （1）（四级标题，空两格）
```

### 数字使用规范
- 年份：阿拉伯数字（2026年）
- 编号：阿拉伯数字（第3期）
- 概数：汉字数字（三四天、七八个）
- 序号：一二三（大层次）、1 2 3（小层次）

## ⚠️ 常见错误

1. **多头请示** — 请示只能主送一个上级机关
2. **请示夹带报告** — 请示就是请示，不要汇报工作
3. **报告夹带请示** — 报告结尾不能要求批复
4. **通知不分层** — 事项要分条列出，不要写大段文字
5. **抄送不规范** — 需要抄送的单位列在版记区域
