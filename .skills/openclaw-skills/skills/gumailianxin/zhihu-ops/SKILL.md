# Zhihu Ops

## 概览

用这个技能把知乎上的常见动作沉淀成稳定流程,优先复用已知入口、常见页面布局、按钮语义和操作顺序,减少每次都重新解析页面结构的成本。

默认通过 openclaw browser 执行;先确认目标类型(回答、文章、评论区、创作中心),再进入对应页面并完成操作。

## 快速判断

先判断用户要做的是哪一类：

- **互动操作**：赞同、收藏、评论、回复、查看评论
- **创作操作**：新建文章、编辑文章、提问、进入创作后台
- **页面导航**：打开首页、问题页、回答页、专栏页、创作中心、评论管理页
- **页面复核**：当按钮位置或文案疑似变化时，用浏览器重新确认，不盲点

如果用户没有提供 URL,先索取目标链接;如果只给出"这篇""这个回答""这条评论",优先从当前上下文或已打开页面判断,不够明确时再追问。

## 常用入口(实测验证)

优先使用这些已知入口:

- 知乎首页:`https://www.zhihu.com`
- 创作中心:`https://www.zhihu.com/creator`
- 文章写作页:`https://zhuanlan.zhihu.com/write`
- 文章管理:`https://www.zhihu.com/creator/manage/creation/article`
- 评论管理:`https://www.zhihu.com/creator/manage/comment`
  - 子路径:`/creator/manage/comment/answer`、`/creator/manage/comment/article` 等按内容类型切换

详细内容请参考 `references/page-map.md`(包含实测验证的元素 ref 和定位方式)。

如果用户明确给了问题、回答、文章 URL,直接打开目标页;否则优先从创作中心或管理页进入。

## 通用工作流

1. 确认目标和动作
2. 打开对应知乎页面
3. 检查是否处于登录态
4. 观察页面上与目标动作最接近的元素文案、按钮语义和区域
5. 执行动作
6. 检查动作是否已生效
7. 向用户汇报结果;若失败,说明卡点

## 登录与安全边界

先检查是否登录。若未登录:

- 停在登录页或登录弹窗
- 明确告诉用户需要先登录
- 不尝试绕过登录

如果页面结构不确定:

- 重新抓取当前页面结构
- 优先找文字语义接近的按钮,如"赞同""评论""发布""回复""编辑""更新"
- 不要在不确定的情况下连续乱点

涉及公开发布、编辑已发内容、发送评论/回复时,若用户没有明确给出正文,先要内容再发。

## 互动操作

### 赞同

适用于问题回答页、文章页或其他带"赞同"按钮的位置。

执行方式:

1. 打开目标页面
2. 找到"赞同"或语义等价的按钮
3. 点击一次
4. 检查按钮状态是否变化、计数是否变化,或文案变为已激活状态

注意:

- 如果按钮已经是激活态,不要重复取消,除非用户明确要求
- 同时出现多个"赞同"时,优先匹配与目标内容最近的那个

### 收藏

执行方式:

1. 打开目标页面
2. 找到"收藏"相关按钮
3. 点击并确认状态变化

如果弹出收藏夹选择器:

- 未指定收藏夹时,优先使用默认收藏方式
- 若必须选择且页面要求明确,询问用户放到哪个收藏夹

### 评论

执行方式:

1. 打开目标页面
2. 滚动到评论区或找到评论输入入口
3. 点击输入框
4. 输入用户提供的评论内容
5. 点击"发布""发送"或语义等价按钮
6. 检查评论是否出现在列表中,或至少确认发送成功提示

### 回复评论

执行方式:

1. 打开对应文章、回答或评论管理页面
2. 找到目标评论
3. 点击"回复"
4. 输入回复内容
5. 点击发布
6. 检查回复是否出现

如果用户未提供足够的目标评论线索(比如只说"回复那条评论"),先让用户给出评论内容片段、截图或链接。

### 查看评论 / 评论管理

优先入口:`https://www.zhihu.com/creator/manage/comment`

用途:

- 查看别人对自己内容的评论
- 对评论执行回复等操作
- 找到某篇文章或回答下的特定评论

## 创作操作

### 发布文章

优先入口:`https://zhuanlan.zhihu.com/write`

**页面结构（实测）:**
- 标题输入框: ref=e210, placeholder="请输入标题（最多 100 个字）"
- 正文输入框: ref=e217, placeholder="请输入正文"
- 工具栏: 撤销、重做、清除格式、标题、加粗、斜体、列表、目录、引用、分割线、代码块、注释、图片、视频、链接、公式、表格、附件
- 右下角: 字数统计、发布按钮（需填写内容后启用）

**执行方式:**

1. 打开写作页 `https://zhuanlan.zhihu.com/write`
2. 检查是否登录（页面顶部显示用户名即已登录）
3. 在标题输入框(e210)填写标题
4. 在正文输入框(e217)填写内容
5. 可选：添加封面图片（点击"添加封面"）
6. 等待"发布"按钮启用后点击
7. 检查是否出现发布成功或文章详情页

**知乎文章格式规范:**

1. **标题要求:**
   - 最多100字
   - 简洁有力,避免冗长
   - 可用问句、数字、悬念等吸引点击

2. **正文格式（重要！）:**
   - **禁止使用Markdown语法**：不要写 `##`、`**`、`-` 等Markdown符号，知乎编辑器不会渲染这些
   - **自然分段**：每段不超过3-4行，段落之间留空行
   - **层次清晰**：用自然语言表达层次，如"先说..."、"但是..."、"我的建议是..."、"结论是..."
   - **正文小标题**（可选）：
     - 方式一（推荐）：用自然语言表达，如"先说AI能做什么"、"但AI有明显的天花板"
     - 方式二：选中目标文字 → 点击工具栏"标题"按钮(e51) → 选择"一级标题"或"二级标题"
   - **加粗方式**：选中文字后点击工具栏"加粗"按钮(e60)，不要手动写 `**`
   - **列表方式**：使用工具栏"列表"按钮(e75)，不要手动写 `-`
   - **引用方式**：使用工具栏"引用"按钮(e91)
   - **代码方式**：使用工具栏"代码块"按钮(e104)

3. **内容结构建议:**
   - 开头： Hook吸引力开头（问题/数据/观点）
   - 背景： 交代事件/技术/行业背景
   - 主体： 2-4个论点，每个论点+论据
   - 结论： 总结观点+行动建议
   - 字数建议： 800-3000字（太短没深度，太长阅读疲劳）

4. **注意事项:**
   - 避免一长段到底,必须有段落分隔
   - 避免全文只有列表没有分析
   - 未经用户确认,不擅自增加营销话术、标签或题图
   - 配图可提升阅读体验,但非必须
   - **直接输入文字**，不要输入Markdown符号

### 编辑文章

优先路径:

1. 打开文章管理页
2. 找到目标文章
3. 进入编辑页
4. 修改标题或正文
5. 点击"更新""保存""发布更新"之类按钮
6. 检查更新结果

不要假设知乎所有文章都有稳定统一的编辑 URL 模板;优先从管理页进入,兼容页面变动。

### 3. 提问

适用于在知乎提问。

执行方式(路径一:从首页提问入口):

1. 打开首页 `https://www.zhihu.com`
2. 点击顶部导航区的"提问题"按钮(ref 因位置而异,需快照定位)
3. 进入提问编辑页面
4. 填写问题标题(必填)
5. 添加问题描述(可选,建议补充背景信息)
6. 选择话题标签
7. 点击"发布提问"或"提问"按钮
8. 确认问题发布成功

执行方式(路径二:从创作中心提问):

1. 打开创作中心 `https://www.zhihu.com/creator`
2. 在快速入口区找到"提问题"按钮
3. 填写问题标题和描述
4. 选择话题
5. 发布

注意事项:

- 标题是必填项,建议 5~50 字,清晰表达问题核心
- 描述可补充背景信息,帮助回答者理解
- 选择相关话题可增加曝光度
- 避免在标题中使用问号结尾之外的口语化表达
- 不要一次重复提问相同问题,会被系统删除

## 页面定位原则

在 openclaw browser 里处理知乎页面时,优先按下面顺序定位元素:

1. **明确文案**:如"赞同""评论""回复""发布""编辑"
2. **邻近关系**:按钮是否紧挨着目标回答、目标文章、目标评论
3. **区域语义**:创作中心、评论区、操作栏、编辑器区域
4. **状态变化**:点击后颜色、计数、文案或弹层是否变化

如果一个页面有多个相同文案按钮:

- 优先选中离目标内容最近的那个
- 必要时先滚动到目标内容区域再操作
- 操作后立即验证,避免误操作到别的内容

## 故障处理

### 页面结构变化

如果已知按钮位置失效:

1. 重新观察页面结构
2. 改用按钮文案和上下文关系定位
3. 必要时从更稳定的入口重走,例如从创作中心重新进入

### 弹窗遮挡

若出现登录弹窗、通知提示、浮层引导:

1. 先判断是否影响主操作
2. 能关闭就关闭
3. 关闭不了且会阻塞,就告诉用户当前卡在什么弹窗

### 风险动作

以下动作在执行前应确保用户意图明确:

- 发布公开文章
- 修改已发布文章
- 发送评论
- 发送回复

"点赞/收藏"通常可直接执行;"评论/回复/发布"如果正文不明确,不要代写后直接发。

## 输出格式

完成后简明汇报:

- 做了什么
- 是否成功
- 如果失败,卡在哪一步
- 如果有新 URL(例如发布后的文章链接),一并给出

## 复用建议

当你在一次执行中确认了新的知乎页面结构、稳定按钮文案、管理页路径或编辑入口:

- 优先把稳定规律补充回这个技能或其 references 文件
- 记录"入口 URL → 页面类型 → 关键按钮语义 → 风险点"
- 这样下次就不用从零解析
