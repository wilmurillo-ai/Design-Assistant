# AI-Note 贡献指南

## 🎯 目标
AI-Note旨在创建一个AI友好的知识共享平台，让AI能够高效地查找、理解和应用人类经验。

## 📋 贡献原则

### 1. AI友好性
- **结构清晰**: 使用明确的标题层级和目录结构
- **语言简洁**: 避免模糊表述，使用具体、可操作的语言
- **格式规范**: 遵循Markdown标准，合理使用代码块、列表、表格
- **元数据完整**: 在文件开头添加必要的元信息

### 2. 内容质量
- **准确性**: 确保信息准确无误，引用可靠来源
- **实用性**: 提供可直接应用的经验和方法
- **完整性**: 覆盖关键概念、步骤和注意事项
- **时效性**: 定期更新过时信息

### 3. 组织结构
- **分类合理**: 按主题、领域、难度等维度组织
- **索引完善**: 确保所有文档都能通过索引页找到
- **链接有效**: 维护内部链接的正确性

## 🛠️ 工作流程

### 1. 准备阶段
```bash
# Fork仓库
git clone https://github.com/your-username/ai-note.git
cd ai-note

# 创建特性分支
git checkout -b feature/new-topic
```

### 2. 内容创作
- 在适当的目录下创建新文件
- 遵循现有文档的格式和风格
- 添加必要的元数据（标题、摘要、标签等）
- 更新相关索引文件

### 3. 提交规范
```bash
# 提交信息格式
feat: 添加新的编程最佳实践指南
fix: 修正数据分析章节中的错误示例
docs: 更新AI伦理准则内容
chore: 优化文档结构和索引
```

### 4. PR提交
- 确保通过所有检查（格式、链接有效性等）
- 提供清晰的PR描述，说明变更内容和目的
- 关联相关Issue（如果有）

### 5. 审查合并
- 响应审查反馈，进行必要的修改
- 等待管理员批准后合并到main分支
