## 📖 简介

**Team Lead** 是 OpenClaw 的多 Agent 协作编排技能，让你能够像团队主管一样指挥多个专业 Agent 完成复杂任务。

### 核心能力

- 🧩 **智能任务分解** - 自动将复杂任务拆解为可执行的子任务
- 🎯 **智能 Agent 匹配** - 根据能力需求匹配最合适的 Agent
- ⚡ **并行执行** - 独立子任务同时执行，大幅提升效率
- 🔍 **质量审核** - 多维度质量检查，确保输出质量
- 📊 **结果聚合** - 智能整合多个 Agent 的输出为统一结果
- 📈 **性能追踪** - 记录 Agent 表现，持续优化协作效率

## 🚀 快速开始

### 安装

```bash
clawhub install team-lead
```

### 基本使用

```
用户：帮我做一份 2026 年 AI 芯片市场分析报告

Team Lead 自动执行：
1. 任务分解 → 数据搜索、厂商调研、趋势分析、报告撰写
2. Agent 匹配 → search-agent × 2, analysis-agent, writing-agent
3. 并行执行 → 同时搜索数据和厂商信息
4. 质量检查 → 确保数据准确、分析深入
5. 结果聚合 → 整合为完整报告
6. 交付用户 → 结构化市场分析报告
```

## 🔧 配置选项

### 基础配置

```json
{
  "orchestration": {
    "maxParallelAgents": 5,      // 最大并行 Agent 数
    "defaultTimeout": 300,       // 默认超时（秒）
    "retryAttempts": 2,          // 重试次数
    "qualityThreshold": 0.75,    // 质量阈值
    "enableCaching": true        // 启用缓存
  }
}
```

### 质量权重

```json
{
  "quality": {
    "weights": {
      "completeness": 0.25,   // 完整性
      "accuracy": 0.30,       // 准确性
      "consistency": 0.20,    // 一致性
      "format": 0.10,         // 格式规范
      "relevance": 0.15       // 相关性
    }
  }
}
```

## 📚 使用示例

### 示例 1：市场研究报告

```
用户：帮我做一份 2026 年 AI 芯片市场分析报告，包括主要玩家、市场规模、趋势预测

执行流程：
1. [分析] 识别为 research 类型
2. [分解] 4 个子任务：数据搜索→厂商调研→趋势分析→报告撰写
3. [分发] 并行执行搜索任务，串行执行分析和撰写
4. [聚合] 整合为结构化报告
5. [交付] 完整市场分析报告

预计耗时：~5 分钟
输出：5000+ 字结构化报告
```

### 示例 2：代码功能开发

```
用户：给我的项目添加用户登录功能，包含邮箱验证和密码加密

执行流程：
1. [分析] 识别为 coding 类型
2. [分解] 4 个子任务：需求分析→代码实现→安全审计→测试用例
3. [分发] 按依赖关系顺序执行
4. [聚合] 整合代码 + 安全报告 + 测试用例
5. [交付] 完整功能实现

预计耗时：~6 分钟
输出：可运行代码 + 安全报告 + 测试用例
```

### 示例 3：多语言内容创作

```
用户：写一篇关于气候变化的科普文章，然后翻译成英文和日文

执行流程：
1. [分析] 识别为 content + translation 类型
2. [分解] 3 个子任务：撰写→英文翻译→日文翻译
3. [分发] 串行撰写，并行翻译
4. [聚合] 整合三语言版本
5. [交付] 中英日三语文章

预计耗时：~4 分钟
输出：三语言对照文章
```

## 🔍 质量审核标准

### 完整性 (25%)
- [ ] 内容完整，覆盖所有要求
- [ ] 结构清晰，有标题和分段
- [ ] 长度适当，信息充分

### 准确性 (30%)
- [ ] 数据有可靠来源
- [ ] 无明显事实错误
- [ ] 逻辑推理合理

### 一致性 (20%)
- [ ] 内部无矛盾
- [ ] 术语使用一致
- [ ] 格式统一

### 格式规范 (10%)
- [ ] Markdown 格式正确
- [ ] 列表、标题使用恰当
- [ ] 无格式错误

### 相关性 (15%)
- [ ] 紧扣主题
- [ ] 覆盖关键词
- [ ] 无偏离内容

## 📈 最佳实践

### ✅ 应该做的

1. **明确任务目标** - 越清晰的需求越好
2. **提供必要上下文** - 帮助 Team Lead 更好理解
3. **设定合理预期** - 复杂任务需要时间
4. **审查最终结果** - 人工复核关键点

### ❌ 不应该做的

1. **不要过度干预** - 让 Team Lead 自主协调
2. **不要忽略质量反馈** - 重视改进建议
3. **不要期望完美** - AI 协作仍有局限
4. **不要忘记保存** - 重要结果及时备份

## 🛠️ 故障排除

### 问题：任务执行时间过长

**可能原因：**
- 任务过于复杂，分解过多子任务
- 某个 Agent 响应慢
- 依赖关系导致串行等待

**解决方案：**
- 简化任务描述
- 减少不必要的依赖
- 调整超时设置

### 问题：质量评分低

**可能原因：**
- 任务要求不明确
- Agent 能力不匹配
- 时间不足

**解决方案：**
- 提供更详细的任务说明
- 手动指定 Agent 类型
- 增加超时时间

### 问题：结果不符合预期

**可能原因：**
- 任务分解不合理
- 聚合策略不合适
- 质量阈值设置过低

**解决方案：**
- 手动调整任务分解
- 更换聚合策略
- 提高质量阈值并启用返工

## 📦 技术细节

### 核心模块

| 模块 | 文件 | 功能 |
|------|------|------|
| Agent 注册中心 | `agent-registry.js` | 管理 Agent 能力和性能 |
| 任务分解器 | `task-decomposer.js` | 拆解复杂任务 |
| 任务分发器 | `dispatcher.js` | 分派任务给 Agent |
| 结果聚合器 | `result-aggregator.js` | 整合多个结果 |
| 质量审核器 | `quality-checker.js` | 检查输出质量 |
| 主入口 | `index.js` | 协调所有模块 |

### 聚合策略

| 策略 | 适用场景 | 说明 |
|------|----------|------|
| `merge` | 互补信息 | 合并各 Agent 输出 |
| `select-best` | 重复任务 | 选择最佳结果 |
| `consensus` | 有冲突 | 提取共识，解决冲突 |
| `chain` | 依赖关系 | 链式传递结果 |
