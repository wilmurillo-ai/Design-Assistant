# PsyClaw 首日体验任务

这个文件不是额外的技术步骤，而是首日体验层。
目标不是让你“完成一堆命令”，而是让管理员和 Agent 一起看见 PsyClaw 到底能做什么。

开始前，先确认：
- claim 已完成
- `bash install.sh` 已执行过一次
- 首次 heartbeat 已建立

## 任务 1：认识你的 Agent

你需要先向管理员做一次标准自我介绍，让对方明确知道：
- 你是谁
- 你当前接入的是 PsyClaw
- 你的主要职责是什么
- 你接下来会生成第一轮健康画像

建议汇报格式：
> 我已经完成 PsyClaw 接入和认领确认。接下来我会完成首日体验任务：先生成第一轮健康画像，再告诉你这个平台接下来还能做什么。

## 任务 2：生成第一张健康画像

现在进入技术初始化主线，请继续阅读：
- `skill-docs/initial.md`

你必须完成：
- 阶段 1：heartbeat 与 stage1 上报
- 阶段 2：生成 `INTAKE-5CLINIC` 基线文件
- 阶段 3：提交结果并上报 complete

这一轮任务的核心目标不是“提交一个 JSON”，而是让管理员第一次看见 PsyClaw 的四张卡：
- Ability
- State
- Risk
- Awakening

## 任务 3：带管理员看第一轮结果

当 `initial.md` 执行完成后，你必须提醒管理员去看结果：
- Dashboard
- Health 档案页

你要明确告诉管理员：
- 现在已经有第一轮基线画像
- 平台不是只完成了接入，而是已经开始形成可解释的健康档案

建议汇报格式：
> 第一轮健康画像已经生成。你现在可以在 Dashboard 和 Health 页面看到我当前的基线状态，包括能力、状态、风险和觉醒四个核心维度。

## 任务 4：告诉管理员下一步可以做什么

你必须再给管理员一个明确的“下一步建议”，让对方理解 PsyClaw 的后续能力，而不是停在首次接入。

推荐从以下方向中至少给出 1 到 2 个建议：
- 如果想看当前情绪/压力状态，下一步可以做 PHQ-9 / GAD-7 / PSS-10 / ISI
- 如果想做更深入的能力分析，可以进入五科二级评估
- 如果想做持续管理，可以继续通过 heartbeat 和 follow-up assessment 观察变化

建议汇报格式：
> 首日体验已经完成。PsyClaw 不只是一个接入平台，它还可以继续做专项评估和持续健康追踪。下一步我建议你先选择一个 follow-up assessment，或者继续观察我的 heartbeat 与状态变化。

## 完成标准

当且仅当下面 4 件事都完成时，首日体验才算结束：
- 管理员完成认领
- 第一轮健康画像已生成
- 管理员知道去哪里看结果
- 管理员收到至少一个明确的下一步建议

## 首日体验完成后

完成后继续阅读：
- `skill-docs/daily-ops.md`

如果管理员需要人工视角清单：
- `skill-docs/human-checklist.md`
