# SEO Keyword Mapper

Transform seed keywords into complete content strategies with clusters, intent mapping, and content briefs.

## What It Does

1. **Keyword Expansion** — Takes 1-3 seed keywords and generates 50-200 related terms
2. **Cluster Mapping** — Groups keywords into topical clusters (pillar + supporting content)
3. **Intent Classification** — Labels each keyword: informational, navigational, commercial, transactional
4. **Difficulty Estimation** — Rates competition level for prioritization
5. **Content Brief Generation** — Creates outlines with target keywords, headers, and word count
6. **Internal Linking Map** — Shows how content pieces should link to each other

## Usage

### Full Keyword Map
```
Create a keyword map for: [NICHE/TOPIC]

Seed keywords: [1-3 keywords]
Target audience: [Who are you writing for]
Business type: [SaaS / agency / e-commerce / blog / etc.]
Goal: [Traffic / leads / sales / authority]

Generate:
1. 100 keyword ideas grouped into clusters
2. Search intent for each
3. Difficulty estimate (easy/medium/hard)
4. Priority ranking (which to target first)
5. Content type recommendation per keyword (blog/landing page/comparison/guide)
6. Internal linking structure
```

### Quick Cluster Analysis
```
I want to rank for "[TARGET KEYWORD]".

Give me:
1. The pillar content piece I need (topic, angle, word count)
2. 10-15 supporting articles that build topical authority
3. How they should interlink
4. What order to publish them in
5. Estimated timeline to rank (based on competition signals)
```

### Content Calendar from Keywords
```
Turn this keyword map into a 3-month content calendar:

Month 1: Foundation pieces (low competition, high relevance)
Month 2: Supporting content (medium competition, topical depth)  
Month 3: Money pages (commercial intent, conversion-focused)

For each piece: title, target keyword, word count, publish date, internal links
```

### Competitor Keyword Gap
```
I'm competing with these sites:
1. [Competitor URL 1]
2. [Competitor URL 2]

My site: [Your URL]

Find keywords they rank for that I don't. Group into:
- Quick wins (low difficulty, high relevance)
- Strategic targets (high value, worth investing in)
- Ignore (irrelevant or too competitive)
```

## Output Format

### Keyword Cluster
```
## Cluster: [Topic Name]
**Pillar Page**: [Main keyword] — [search volume estimate] — [difficulty]

| Supporting Keyword | Intent | Difficulty | Content Type | Priority |
|-------------------|--------|------------|--------------|----------|
| [keyword] | Info | Easy | Blog post | High |
| [keyword] | Commercial | Medium | Comparison | High |
| [keyword] | Transactional | Hard | Landing page | Medium |

**Internal Linking**: 
- All supporting articles link to pillar page
- Pillar links to top 5 supporting articles
- Cross-link related supporting articles
```

### Content Brief
```
## Brief: [Title]
**Target Keyword**: [primary keyword]
**Secondary Keywords**: [2-4 terms to include]
**Intent**: [Informational / Commercial / Transactional]
**Word Count**: [Recommended range]
**Difficulty**: [Easy / Medium / Hard]

### Required Sections:
- H1: [Title with keyword]
- H2: [Section heading]
  - Key points to cover
- H2: [Section heading]
  - Key points to cover
- H2: FAQ (target People Also Ask)
  - Q: [Common question]
  - Q: [Common question]

### Internal Links:
- Link TO: [related article on your site]
- Link FROM: [articles that should link here]

### Competitive Notes:
- Top 3 ranking pages: [what they cover]
- Gap: [what they miss that you should include]
```

## SEO Principles Built In

- **Topical authority** — Build clusters, not random articles
- **Intent matching** — Match content format to search intent
- **Low-hanging fruit first** — Target easy keywords early for quick wins
- **Internal linking** — Every piece strengthens the cluster
- **Content depth** — Longer isn't better, more comprehensive IS better
