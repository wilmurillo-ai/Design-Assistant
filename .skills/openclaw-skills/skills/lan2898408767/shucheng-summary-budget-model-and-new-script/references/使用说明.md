# 技能更新操作指南

## 快速开始

### 触发总结

只需在对话中发送以下任一指令：

- `【总结对话内容】`
- `【更新技能】`
- `【记录这个脚本】`
- `【保存新的数据模型】`

### AI 将自动执行

1. **回顾对话历史** - 查找新的脚本写法和数据模型
2. **分类整理** - 区分脚本操作和数据模型内容
3. **更新技能文件** - 追加到对应的技能参考文件中
4. **记录日志** - 在 update_log.md 中记录更新内容
5. **创建备份** - 备份原文件以防需要回滚

## 更新内容类型

### 脚本操作类

以下类型的脚本内容会被记录：

✅ **会被记录：**
- 新的 API 调用方式（如 DataModelUtils 的新用法）
- 特殊的数据处理逻辑
- 新的函数或工具类使用
- 脚本模板或最佳实践
- 错误处理和边界情况处理
- 性能优化技巧

❌ **不会被记录：**
- 已有脚本的简单重复
- 明显的语法错误示例
- 临时性的测试代码

### 数据模型类

以下类型的数据模型内容会被记录：

✅ **会被记录：**
- 新的数据表（表名、描述、用途）
- 新的字段（字段名、类型、约束）
- 表间关联关系
- 业务规则和验证逻辑
- 默认值和计算规则

❌ **不会被记录：**
- 临时表或测试表
- 已废弃的字段
- 个人自定义字段（非通用）

## 备份与回滚

### 备份位置

所有更新前的原文件会备份到：
```
skills/SummaryBudgetModelAndNewScript/references/backup/
```

### 备份命名

```
文件名_YYYYMMDD_HHMMSS.扩展名
```

### 回滚操作

如需回滚：
1. 找到需要恢复的备份文件
2. 复制回原位置覆盖当前文件
3. 在 update_log.md 中记录回滚原因

## 最佳实践

### 1. 及时总结

建议在以下时机触发总结：
- 完成一个复杂脚本的编写后
- 讨论完一个新的数据模型后
- 解决了一个特殊问题后
- 对话告一段落时

### 2. 清晰描述

提供新内容时，尽量包含：
- 使用场景
- 输入参数
- 输出结果
- 注意事项

### 3. 验证内容

AI 会在更新前验证：
- 语法正确性
- API 存在性
- 命名规范性
- 无重复内容

### 4. 定期审查

建议定期审查更新内容：
- 检查是否有误添加
- 确认内容的准确性
- 清理过时的内容
