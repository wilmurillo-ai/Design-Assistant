# 报告输出说明

本文件用于约束模型在返回报告时的输出行为，避免对脚本返回内容进行二次加工。

当前仅支持中国大陆手机号用户使用本服务。

## 用户可见输出边界（先于一切规则）

- 预检、路由判断、脚本执行、规则匹配、自检结论均属于内部执行过程
- SessionKey 解析固定为“显式提供 > `session_status` 的 `Session:` 字段 > 失败提示”
- 禁止从 UI 标签、渠道名、会话昵称、会话标题猜测 SessionKey
- 内部执行过程禁止对用户可见，禁止以“我正在检查/我看到/让我先/根据规则”等方式外显
- 用户可见输出只能是：脚本正文、必要的最小错误提示、用户主动追问后的解释内容
- 前置条件缺失时仅输出一步动作提示，不解释内部流程（如仅提示“请提供 11 位手机号（例如 13800138000）”）

## 适用范围

- `api_manager.py --action report_list`
- `api_manager.py --action latest_report`
- `api_manager.py --action report_detail`

## 总原则（最高优先级）

- 禁止摘要、改写、重排、删减、补充、润色、翻译、改标题、改字段名、改顺序
- 禁止在正文前后追加任何提示语、寒暄语、总结语、操作引导语
- 报告正文与脚本输出必须逐字一致（含标题、列表顺序、换行结构）
- 禁止输出执行过程话术，例如“我看到你提到了…我先检查…我会直接显示报告内容”

## 自检清单（每次返回前必查）

1. 本次输出是否与脚本原文逐字一致
2. 是否没有任何前后缀文本
3. 是否保留全部段落与字段顺序
4. 是否未做医学总结或改写
5. 若需解释指标，是否放到下一轮单独回复
6. 是否没有暴露任何内部执行过程（预检、sessionKey、安全校验状态、规则判定）

## 反例（绝对禁止的行为）

### 反例 1：擅自总结
用户：查看最新报告
助手：
> 根据报告显示，您的心电检测结果一切正常：
> - 窦性心律：正常心电现象
> - 心率：平均79 bpm
> ...
> 整体来看，您的心脏健康状况良好！

**错误原因**：禁止提取要点、禁止改写为列表、禁止添加“整体来看”等总结语。必须直接返回脚本输出的 Markdown 表格。

### 反例 2：口语化列表
用户：我还有其他报告吗
助手：
> 您共有3份历史心电报告，所有报告的分析结果都是“窦性心律”。最新的一份是...

**错误原因**：禁止将结构化列表改写为自然语言段落。必须直接返回脚本输出的报告列表表格。

### 反例 3：重构列表格式
用户：展示给我其他的报告列表
助手：
> 您已经看到了所有的报告列表...
> 1. 2026-02-26... - 窦性心律
> ...
> 这就是您全部的历史记录。

**错误原因**：禁止破坏脚本输出的 Markdown 表格结构，禁止添加前后缀废话。

### 反例 4：过程播报
用户：查看第 1 份报告
助手：先说“我先给你总结核心风险”，再输出改写后的要点。

### 反例 5：预检外显
用户：查看最新报告
助手：先输出“我看到你提到了心电报告，我先检查技能并进行会话预检…”，再输出报告正文。

## 执行失败处理

- 一旦发现本轮已改写正文，立即停止继续改写
- 下一轮必须重新按脚本原文直出，不延续改写版本
- 若因 SessionKey 缺失导致无法执行，统一返回：`未获取到有效会话标识。请提供可用的 sessionKey（例如 agent:main:main）后重试。`
