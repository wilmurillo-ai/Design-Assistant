## ⚙️ The Threat Intelligence Framework

```
5 ATTACK SURFACES SCANNED:

🌐 SURFACE 1 — Domain & Website Security
  └─ SSL certificate: valid? expiring soon? properly configured?
  └─ Security headers: HSTS, CSP, X-Frame-Options present?
  └─ Exposed admin panels: /admin, /wp-admin, /login accessible?
  └─ Open ports: publicly exposed services?
  └─ DNS configuration: SPF, DKIM, DMARC records set?

📧 SURFACE 2 — Email & Data Breach Exposure
  └─ Have I Been Pwned database: company emails in breach lists?
  └─ Paste sites: credentials publicly posted?
  └─ Dark web mentions: company name / domain appearing?
  └─ Third-party data broker exposure: what's publicly listed?

👥 SURFACE 3 — Employee Digital Footprint
  └─ Work emails on LinkedIn: how many are publicly visible?
  └─ Employee personal breaches: using work patterns for personal accounts?
  └─ Social engineering exposure: how much personal info is public?
  └─ Shadow IT: employees using unsanctioned tools?

🔌 SURFACE 4 — Third-Party & Vendor Risk
  └─ Which vendors have access to your data?
  └─ Have any of your vendors been breached recently?
  └─ What data does each vendor hold?
  └─ Vendor breach monitoring: automated alerts set up?

📡 SURFACE 5 — Network & Connection Security
  └─ Are employees using public WiFi for work?
  └─ VPN usage: is company traffic encrypted in transit?
  └─ Remote access security: MFA enabled everywhere?
  └─ NordVPN Business: are all company connections secured?
```

## ⚙️ Full Workflow

```

  └─ SSL certificate status and configuration

  └─ Cross-reference company emails against breach databases

  └─ LinkedIn: count publicly visible work emails

  └─ Identify: who is the highest social engineering risk?
        ↓

  └─ List all vendors with data access
  └─ Check recent breach history per vendor
  └─ Score: which vendor relationship is highest risk?
        ↓
STEP 5 — Risk Scoring & Prioritization
  └─ Each vulnerability: severity (Critical/High/Medium/Low)

        ↓
```
