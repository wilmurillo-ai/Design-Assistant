# 业务模板库

本文档提供常见业务场景的表结构模板，作为参考而非硬性规范。

**使用原则：**
- 模板是参考基础，根据实际业务调整
- 不要生搬硬套，理解设计思路更重要
- 简单优先，只取需要的部分

---

## 模板1：客户关系管理（CRM）

### 适用场景
- 销售团队管理客户和商机
- 跟进潜在客户直到成交
- 需要记录完整的沟通历史
- 关注销售漏斗和转化率

### 核心对象
- 客户（公司或个人）
- 跟进记录
- 商机/订单

### 推荐表结构

#### 表1：客户主表
管理客户基础信息和当前状态

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 客户名称 | 文本 | 必填，公司名或个人名 |
| 客户类型 | 单选 | 选项：企业客户、个人客户 |
| 当前阶段 | 单选 | 选项：潜在客户、意向沟通、方案报价、谈判中、已成交、已流失 |
| 客户来源 | 单选 | 选项：转介绍、线上咨询、线下活动、主动开发、广告投放 |
| 负责人 | 人员 | 多选，销售人员 |
| 联系人 | 文本 | 主要联系人姓名 |
| 联系电话 | 电话 | 用于快速拨打 |
| 邮箱 | 邮箱 | - |
| 公司地址 | 文本 | - |
| 首次接触时间 | 日期 | 记录来源 |
| 预计成交日期 | 日期 | 用于预测业绩 |
| 成交概率 | 数字 | 百分比，0-100% |
| 预估金额 | 数字 | 货币格式 |
| 备注 | 多行文本 | 记录重要信息 |
| 标签 | 多选 | 选项：重点客户、VIP、需跟进等 |

#### 表2：跟进记录表
记录每次客户沟通的详细内容

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 跟进对象 | 关联 | 必填，关联到"客户主表"，显示"客户名称" |
| 跟进时间 | 日期 | 必填，包含时间 |
| 跟进方式 | 单选 | 必填，选项：电话、微信、面谈、邮件、其他 |
| 沟通摘要 | 多行文本 | 必填，记录沟通要点 |
| 客户反馈 | 多行文本 | 客户的态度和需求 |
| 下一步计划 | 多行文本 | 明确后续动作 |
| 下次跟进时间 | 日期 | 包含时间，用于提醒 |
| 跟进人 | 人员 | 记录谁跟进的 |

#### 表3：订单/商机表
记录成交后的订单信息

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 订单编号 | 文本 | 自动编号 |
| 客户 | 关联 | 必填，关联到"客户主表" |
| 产品/服务 | 文本 | - |
| 订单金额 | 数字 | 货币格式 |
| 成交时间 | 日期 | - |
| 付款状态 | 单选 | 选项：未付款、部分付款、已付款 |
| 交付状态 | 单选 | 选项：未开始、进行中、已完成 |
| 备注 | 多行文本 | - |

### 典型流程
潜在客户 → 意向沟通 → 方案报价 → 谈判 → 成交签约 → 交付服务

### 推荐视图
- **客户主表**：
  - "全部客户"（表格视图）
  - "我的客户"（按负责人筛选）
  - "本周待跟进"（按"下次跟进时间"筛选）
  - "成交漏斗"（按"当前阶段"分组看板）
- **跟进记录表**：
  - "全部记录"（按时间倒序）
  - "今日跟进"（按日期筛选今天）

### 推荐自动化
1. 当"下次跟进时间"到期前1天，发送通知给负责人
2. 当"当前阶段"变更为"已成交"时，自动创建订单记录

## 模板2：内容生产管理

### 适用场景
- 自媒体团队管理内容创作
- 文章/视频从选题到发布的全流程
- 多人协作的内容生产
- 需要统计内容数据和发布计划

### 核心对象
- 内容选题
- 内容正文/素材
- 发布记录
- 数据复盘

### 推荐表结构

#### 表1：内容选题池
管理所有内容创意和选题

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 选题标题 | 文本 | 必填 |
| 内容类型 | 单选 | 选项：图文、视频、音频、直播 |
| 选题来源 | 单选 | 选项：原创、改编、热点、用户投稿 |
| 优先级 | 单选 | 选项：高、中、低 |
| 当前状态 | 单选 | 选项：灵感池、待创作、创作中、待审核、已发布、已废弃 |
| 负责人 | 人员 | 内容创作者 |
| 审核人 | 人员 | - |
| 预计发布时间 | 日期 | - |
| 目标平台 | 多选 | 选项：公众号、小红书、抖音、B站等 |
| 标签 | 多选 | 内容分类标签 |
| 核心观点 | 多行文本 | 内容大纲 |
| 备注 | 多行文本 | - |

#### 表2：内容素材库
管理内容的详细信息和素材

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 关联选题 | 关联 | 必填，关联到"内容选题池" |
| 素材类型 | 单选 | 选项：正文、图片、视频、音频 |
| 素材链接 | URL | 存储云盘链接 |
| 素材文件 | 附件 | 直接上传文件 |
| 完成状态 | 单选 | 选项：未开始、进行中、已完成 |
| 备注 | 多行文本 | - |

#### 表3：发布记录表
记录已发布内容和数据表现

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 内容 | 关联 | 必填，关联到"内容选题池" |
| 发布平台 | 单选 | 选项：公众号、小红书、抖音、B站 |
| 发布时间 | 日期 | 必填，包含时间 |
| 发布链接 | URL | - |
| 阅读量/播放量 | 数字 | - |
| 点赞数 | 数字 | - |
| 转发数 | 数字 | - |
| 评论数 | 数字 | - |
| 收藏数 | 数字 | - |
| 数据更新时间 | 日期 | - |

### 典型流程
灵感池 → 筛选选题 → 分配创作 → 内容审核 → 定时发布 → 数据复盘

### 推荐视图
- **选题池**：
  - "全部选题"（表格视图）
  - "待创作"（按状态筛选）
  - "本周发布计划"（按"预计发布时间"日历视图）
  - "创作看板"（按"当前状态"分组）
- **发布记录**：
  - "数据总览"（表格视图，显示所有数据指标）

### 推荐自动化
1. 当"当前状态"变更为"待审核"时，通知审核人
2. 当"预计发布时间"到达时，提醒负责人确认发布

## 模板3：项目任务协同

### 适用场景
- 团队项目管理
- 任务分配和进度跟踪
- 多人协作的工作流
- 需要看整体进度和个人任务

### 核心对象
- 项目
- 任务
- 子任务

### 推荐表结构

#### 表1：项目主表
管理所有项目信息

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 项目名称 | 文本 | 必填 |
| 项目状态 | 单选 | 选项：未开始、进行中、已完成、已暂停 |
| 项目负责人 | 人员 | 必填 |
| 项目成员 | 人员 | 多选 |
| 开始时间 | 日期 | - |
| 截止时间 | 日期 | - |
| 项目进度 | 数字 | 百分比，0-100% |
| 优先级 | 单选 | 选项：高、中、低 |
| 项目描述 | 多行文本 | - |
| 项目目标 | 多行文本 | - |

#### 表2：任务表

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 任务名称 | 文本 | 必填 |
| 所属项目 | 关联 | 必填，关联到"项目主表" |
| 任务状态 | 单选 | 必填，选项：未开始、进行中、已完成、已阻塞 |
| 任务负责人 | 人员 | 必填 |
| 任务优先级 | 单选 | 选项：高、中、低 |

| 截止时间 | 日期 | - |

#### 表3：子任务表
拆解更细的执行步骤

**关键字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 子任务名称 | 文本 | 必填 |

| 完成状态 | 复选框 | - |
| 负责人 | 人员 | - |
| 截止时间 | 日期 | - |

### 典型流程
项目立项 → 任务拆解 → 分配执行 → 进度跟踪 → 完成验收

### 推荐视图
- **项目表**：
  - "全部项目"（表格视图）
  - "进行中的项目"（按状态筛选）
  - "项目看板"（按"项目状态"分组）
- **任务表**：
  - "全部任务"（表格视图）
  - "我的任务"（按负责人筛选）
  - "本周到期"（按"截止时间"筛选）
  - "任务看板"（按"任务状态"分组）

### 推荐自动化
1. 当"截止时间"到期前1天，提醒任务负责人
2. 当"任务状态"全部完成时，自动更新"项目进度"

| 字段名 | 类型 | 说明 |

| 客户 | 关联 | 关联到客户表 |
| 订单金额 | 数字 | 货币格式，公式计算 |

| 小计金额 | 数字 | 公式：数量×单价 |

  - "全部订单"（表格视图）
  - "待处理订单"（按"订单状态"筛选）

## 如何使用这些模板

### 1. 匹配模板
根据用户业务描述，找到最接近的模板。

### 2. 定制调整
基于模板，根据用户实际需求调整：
- 删除不需要的字段
- 增加业务特定字段
- 调整字段选项
- 合并或拆分表

### 3. 从零设计
如果用户业务完全不符合任何模板：
- 借鉴多个模板的部分思路
- 参考"常见模式"（见 `common-patterns.md`）
- 从业务本质出发设计

---

**记住：模板只是参考，理解设计思路比生搬硬套更重要。**
