# 常见表结构模式

本文档总结飞书多维表格设计中的常见模式，帮助快速理解业务本质并设计合理的表结构。

---

## 一、表关系模式

### 1. 主表 + 子表模式（一对多）

**适用场景：**
- 一个对象有多条记录
- 需要记录历史或详细日志
- 子表需要独立管理

**典型例子：**
- 客户主表 ← 跟进记录表
- 项目主表 ← 任务表
- 订单表 ← 订单明细表

**设计要点：**
```
主表（一）                子表（多）
├─ 客户主表                ├─ 跟进记录表
│  ├─ 客户名称              │  ├─ 跟进对象（关联到客户主表）
│  ├─ 当前阶段              │  ├─ 跟进时间
│  └─ 负责人                │  ├─ 跟进方式
│                           │  └─ 沟通摘要
│
关联方向：子表 → 主表
```

**关键字段：**
- 子表必须有**关联字段**指向主表
- 关联字段建议设置为必填
- 用查找引用显示主表的关键信息

**视图建议：**
- 主表：表格视图 + 看板视图
- 子表：按时间排序的表格视图

---

### 2. 主表 + 结果表模式（流程产出）

**适用场景：**
- 业务流程产生结果
- 需要追踪从线索到结果的转化
- 结果数据需要独立分析

**典型例子：**
- 客户主表 → 订单表
- 内容选题池 → 发布记录表
- 招聘候选人表 → 入职记录表

**设计要点：**
```
主表（流程）              结果表（产出）
├─ 客户主表                ├─ 订单表
│  ├─ 客户名称              │  ├─ 订单编号
│  ├─ 当前阶段              │  ├─ 客户（关联到客户主表）
│  └─ 负责人                │  ├─ 订单金额
│                           │  └─ 成交时间
│
关联方向：结果表 → 主表
```

**关键字段：**
- 结果表关联到主表
- 主表可通过公式统计结果数量

---

### 3. 标签表 + 主表模式（多维分类）

**适用场景：**
- 需要多维分类
- 一个对象属于多个分类
- 分类需要独立管理

**典型例子：**
- 文章表 + 标签表（多对多）
- 产品表 + 分类表
- 任务表 + 项目表

**设计要点：**
```
主表                    标签表
├─ 文章表                ├─ 标签表
│  ├─ 文章标题            │  ├─ 标签名称
│  ├─ 文章内容            │  ├─ 标签类型
│  └─ 标签（多选关联）    │  └─ 标签描述
│
关联方向：主表.标签字段 → 标签表（多对多）
```

**替代方案：**
- 简单场景用多选字段即可
- 复杂场景才需要独立的标签表

---

### 4. 维度表 + 事实表模式（数据分析）

**适用场景：**
- 需要多维度分析
- 数据仓库设计
- 复杂报表

**典型例子：**
- 时间维度表 + 销售事实表
- 商品维度表 + 订单事实表
- 客户维度表 + 交易事实表

**设计要点：**
```
维度表（描述属性）        事实表（业务数据）
├─ 时间维度表              ├─ 销售记录表
│  ├─ 日期                │  ├─ 日期（关联到时间维度）
│  ├─ 年                  │  ├─ 商品（关联到商品维度）
│  ├─ 月                  │  ├─ 客户（关联到客户维度）
│  ├─ 季度                │  ├─ 销售金额
│  └─ 星期                │  └─ 销售数量
│
关联方向：事实表 → 各维度表
```

**注意：**
- 大部分业务场景不需要这么复杂
- 仅适用于需要深度数据分析的场景

## 二、字段组合模式

### 1. 状态管理模式

**基础版：**
```
- 当前状态（单选）：未开始、进行中、已完成
```

**进阶版：**
```
- 当前状态（单选）：未开始、进行中、已完成、已阻塞
- 状态变更时间（日期）：记录最后状态变更时间
- 负责人（人员）：当前负责人
```

**高级版：**
```
- 当前状态（单选）
- 状态变更时间（日期）
- 历史状态（多行文本）：记录状态变更历史
- 或用独立的"状态变更记录表"（子表模式）
```

---

### 2. 进度跟踪模式

**简单版：**
```
- 完成进度（进度）：0-100%
```

**详细版：**
```
- 完成进度（进度）：0-100%
- 已完成子任务数（公式）：COUNTIF(子任务表.完成状态 = 已完成)
- 总子任务数（公式）：COUNT(子任务表)
- 完成百分比（公式）：已完成数 / 总数 × 100%
```

---

### 3. 优先级管理模式

**基础版：**
```
- 优先级（单选）：高、中、低
```

**四象限版：**
```
- 紧急程度（单选）：紧急、不紧急
- 重要程度（单选）：重要、不重要
- 优先级（公式）：根据紧急和重要程度自动计算
```

**加权版：**
```
- 优先级分数（数字）：1-10分
- 优先级等级（公式）：
  - 8-10分：高优先级
  - 5-7分：中优先级
  - 1-4分：低优先级
```

---

### 4. 时间跟踪模式

**基础版：**
```
- 开始时间（日期）
- 截止时间（日期）
```

**进阶版：**
```
- 开始时间（日期）
- 截止时间（日期）
- 预估工时（数字）：单位小时
- 实际工时（数字）：单位小时
- 工时差异（公式）：实际工时 - 预估工时
```

**提醒版：**
```
- 截止时间（日期）
- 提醒时间（公式）：截止时间 - 1天
- 是否逾期（公式）：IF(截止时间 < 今天(), "是", "否")
```

---

### 5. 人员协作模式

**单人负责：**
```
- 负责人（人员）：单选
```

**多人协作：**
```
- 负责人（人员）：单选，主要负责人
- 协作人（人员）：多选，协作成员
```

**审批流程：**
```
- 申请人（人员）
- 审批人（人员）
- 审批状态（单选）：待审批、已通过、已拒绝
- 审批时间（日期）
- 审批意见（多行文本）
```

---

### 6. 联系信息模式

**基础版：**
```
- 联系人（文本）
- 联系电话（电话）
- 邮箱（邮箱）
```

**多联系人版：**
```
主表：客户表
├─ 客户名称
└─ ...

子表：联系人表
├─ 客户（关联到客户表）
├─ 联系人姓名
├─ 职位
├─ 电话（电话）
├─ 邮箱（邮箱）
└─ 是否主要联系人（复选框）
```

---

## 三、业务流程模式

### 1. 线性流程模式

**适用场景：**
- 流程阶段固定
- 按顺序推进
- 每个阶段明确

**典型流程：**
```
阶段1 → 阶段2 → 阶段3 → 阶段4 → 完成
```

**设计要点：**
```
- 当前阶段（单选）：所有阶段作为选项
- 阶段变更记录（子表）：记录每次阶段变更
- 阶段负责人（人员）：每个阶段的负责人
- 阶段截止时间（日期）
```

**例子：**
- 销售流程：潜在客户 → 意向沟通 → 方案报价 → 谈判 → 成交
- 招聘流程：简历筛选 → 初试 → 复试 → 终试 → 发offer
- 内容流程：选题 → 创作 → 审核 → 发布 → 复盘

---

### 2. 状态机模式

**适用场景：**
- 状态之间有明确的转换规则
- 某些状态只能从特定状态转换
- 需要控制流程走向

**设计要点：**
```
- 当前状态（单选）
- 前置状态（单选或公式）：记录从哪个状态转换来的
- 状态转换条件（多行文本）：说明转换条件
```

**例子：**
```
订单状态转换规则：
- 未付款 → 已付款（付款后）
- 已付款 → 已发货（发货后）
- 已发货 → 已完成（客户确认收货）
- 已发货 → 已退款（客户申请退款）
```

---

### 3. 并行流程模式

**适用场景：**
- 多个任务同时进行
- 需要汇总多个任务的进度
- 团队协作场景

**设计要点：**
```
主表：项目表
├─ 项目名称
├─ 项目进度（公式）：AVG(任务表.完成进度)
└─ ...

子表：任务表
├─ 所属项目（关联到项目表）
├─ 任务名称
├─ 负责人
├─ 完成进度
└─ ...
```

---

### 4. 审批流程模式

**适用场景：**
- 需要多人审批
- 审批顺序固定
- 需要记录审批意见

**设计要点：**
```
主表：申请表
├─ 申请人
├─ 申请内容
├─ 当前审批节点（单选）
└─ ...

子表：审批记录表
├─ 申请（关联到申请表）
├─ 审批人
├─ 审批顺序（数字）
├─ 审批状态（单选）：待审批、通过、拒绝
├─ 审批时间
└─ 审批意见
```

---

## 四、数据统计模式

### 1. 汇总统计模式

**从子表统计到主表：**
```
子表：订单明细表
├─ 订单（关联到订单表）
├─ 商品
├─ 数量
└─ 小计（公式）：数量 × 单价

主表：订单表
├─ 订单金额（公式）：SUM(订单明细表.小计)
├─ 商品数量（公式）：COUNT(订单明细表)
└─ 平均单价（公式）：订单金额 / 商品数量
```

---

### 2. 时间统计模式

**按日期统计：**
```
- 创建日期（日期）
- 年份（公式）：YEAR(创建日期)
- 月份（公式）：MONTH(创建日期)
- 星期（公式）：WEEKDAY(创建日期)
```

**按时间段筛选：**
```
- 本月记录：筛选"创建日期"在本月
- 本周记录：筛选"创建日期"在本周
- 今日记录：筛选"创建日期"在今天
```

---

### 3. 转化率统计模式

**销售漏斗转化率：**
```
主表：客户表
├─ 当前阶段（单选）
└─ ...

统计表（可选）：
├─ 阶段名称
├─ 客户数量（公式）：COUNTIF(客户表.当前阶段 = 阶段名称)
├─ 转化率（公式）：当前阶段数量 / 上一阶段数量
└─ 总体转化率（公式）：当前阶段数量 / 初始阶段数量
```

## 五、常见反模式（避免）

### ❌ 反模式1：万能大表

**问题：**
- 一张表包含所有信息
- 字段数量超过30个
- 混合不同性质的数据

**例子：**
```
❌ 错误：客户表
├─ 客户名称
├─ 联系电话
├─ 跟进记录1（多行文本）
├─ 跟进记录2（多行文本）
├─ 跟进记录3（多行文本）
├─ 订单1金额
├─ 订单1时间
├─ 订单2金额
├─ 订单2时间
└─ ...（超过50个字段）
```

**正确做法：**
```
✅ 正确：拆分成多张表
- 客户主表（基础信息）
- 跟进记录表（子表）
- 订单表（关联到客户）
```

---

### ❌ 反模式2：重复存储数据

**问题：**
- 相同数据在多处存储
- 数据不一致
- 更新困难

**例子：**
```
❌ 错误：
订单表：
├─ 订单编号
├─ 客户名称（文本）
├─ 客户电话（文本）
├─ 客户地址（文本）
└─ 订单金额

订单明细表：
├─ 订单编号
├─ 客户名称（文本） ← 重复
├─ 客户电话（文本） ← 重复
└─ ...
```

**正确做法：**
```
✅ 正确：用关联
客户表：
├─ 客户名称
├─ 客户电话
└─ 客户地址

订单表：
├─ 订单编号
├─ 客户（关联到客户表） ← 用关联
└─ 订单金额
```

---

### ❌ 反模式3：用文本存储结构化数据

**问题：**
- 用文本字段存储状态
- 用多行文本存储列表
- 无法筛选和统计

**例子：**
```
❌ 错误：
- 状态（文本）：输入"进行中"、"已完成"
- 标签（多行文本）：输入"产品、运营、增长"
```

**正确做法：**
```
✅ 正确：
- 状态（单选）：选项：未开始、进行中、已完成
- 标签（多选）：选项：产品、运营、增长
```

---

### ❌ 反模式4：缺少状态字段

**问题：**
- 无法跟踪流程进度
- 无法做看板视图
- 业务状态不明确

**正确做法：**
```
✅ 每张流程表都应该有状态字段
- 任务表：任务状态（单选）
- 订单表：订单状态（单选）
- 客户表：客户阶段（单选）
```

---

## 六、设计检查清单

在设计表结构时，检查以下问题：

### 表结构层面
- [ ] 一张表只管理一类对象
- [ ] 字段数量控制在 8-15 个
- [ ] 表数量不超过 5 张（MVP阶段）
- [ ] 表之间的关联关系清晰

### 字段层面
- [ ] 每个字段都有明确的用途
- [ ] 字段类型选择正确
- [ ] 必填字段尽可能少
- [ ] 没有冗余字段
- [ ] 有状态/阶段字段（如果需要）

### 关联层面
- [ ] 避免数据重复，用关联代替
- [ ] 没有循环关联
- [ ] 关联方向正确（通常是子表→主表）
- [ ] 关联字段设置为必填

### 扩展性
- [ ] 预留了扩展空间
- [ ] 没有过度设计
- [ ] 可以后续增加字段
- [ ] 可以后续增加表

---

**记住：好的表结构应该简单、清晰、易维护。**
