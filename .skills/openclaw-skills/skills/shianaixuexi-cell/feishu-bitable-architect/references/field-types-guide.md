# 飞书多维表格字段类型指南

本文档详细说明飞书多维表格的字段类型、适用场景和配置建议。

---

## 字段类型总览

飞书多维表格支持以下字段类型：

| 字段类型 | 适用场景 | 可统计 | 可筛选 |
|---------|---------|--------|--------|
| 文本 | 短文本、名称、编号 | ❌ | ✅ |
| 多行文本 | 长文本、描述、备注 | ❌ | ❌ |
| 单选 | 互斥的状态、分类 | ✅ | ✅ |
| 多选 | 多个标签、分类 | ✅ | ✅ |
| 人员 | 负责人、协作者 | ✅ | ✅ |
| 群组 | 团队、部门 | ✅ | ✅ |
| 日期 | 时间点、日期 | ✅ | ✅ |
| 数字 | 金额、数量、百分比 | ✅ | ✅ |
| 电话 | 电话号码 | ❌ | ✅ |
| 邮箱 | 邮箱地址 | ❌ | ✅ |
| URL | 链接地址 | ❌ | ❌ |
| 复选框 | 是/否、开关 | ✅ | ✅ |
| 进度 | 百分比进度 | ✅ | ✅ |
| 评分 | 星级评分 | ✅ | ✅ |
| 附件 | 文件、图片 | ❌ | ❌ |
| 关联 | 关联其他表 | - | ✅ |
| 查找引用 | 显示关联表的其他字段 | - | ✅ |

| 创建记录 | 记录创建人和时间 | - | ✅ |

## 详细字段说明

### 1. 文本

**适用场景：**
- 名称、标题
- 编号、代码
- 简短描述
- 地址、公司名等

**配置建议：**
- 默认长度：单行文本
- 可设置唯一值约束（如订单编号）
- 可设置默认值

**常见错误：**
- ❌ 用文本字段存储金额（无法计算）
- ❌ 用文本字段存储状态（无法统计）
- ❌ 用文本字段存储长段落（应该用多行文本）

**示例：**
- 客户名称
- 订单编号
- 产品名称
- 联系地址

---

### 2. 多行文本

**适用场景：**
- 详细描述
- 备注信息
- 长篇内容
- 会议纪要

**配置建议：**
- 适合存储超过50字的文本
- 不用于需要分析的内容
- 不用于需要结构化的数据

**常见错误：**
- ❌ 用多行文本记录应该独立成表的内容（如跟进历史）
- ❌ 用多行文本存储结构化数据（如多个标签）

**示例：**
- 项目描述
- 任务详情
- 会议纪要
- 客户备注

---

### 3. 单选

**适用场景：**
- 流程状态（互斥）
- 分类（单一分类）
- 优先级
- 阶段

**配置建议：**
- 选项必须互斥（不能同时属于两个选项）
- 选项要完整（覆盖所有可能性）
- 选项要简洁（2-6个字）
- 建议设置默认值
- 可设置颜色标识

**常见错误：**
- ❌ 选项有重叠（如"高"和"非常高"）
- ❌ 选项不完整（缺少某些情况）
- ❌ 选项过多（超过8个考虑用其他方式）

**示例：**
- 任务状态：未开始、进行中、已完成
- 优先级：高、中、低
- 客户阶段：潜在客户、意向、成交

---

### 4. 多选

**适用场景：**
- 多个标签
- 多维分类
- 技能标签
- 服务项目

**配置建议：**
- 可同时选择多个选项
- 选项之间是"和"的关系
- 建议不超过15个选项

**常见错误：**
- ❌ 用多选存储应该用单选的内容
- ❌ 选项过多导致选择困难

**示例：**
- 文章标签：产品、运营、增长、设计
- 服务项目：设计、开发、测试、运维
- 技能标签：Python、Java、Go、React

---

### 5. 人员

**适用场景：**
- 负责人
- 协作者
- 执行人
- 审批人

**配置建议：**
- 支持单选或多选
- 多选时注意：一个人只能被选中一次
- 可配合权限使用

**常见错误：**
- ❌ 用文本字段记录人员（无法@提醒）
- ❌ 多选时重复选择同一人

**示例：**
- 任务负责人
- 项目成员
- 销售人员
- 审核人

---

### 6. 群组

**适用场景：**
- 部门
- 团队
- 项目组

**配置建议：**
- 用于多人协作场景
- 可配合权限管理

**示例：**
- 所属部门
- 项目组
- 销售团队

---

### 7. 日期

**适用场景：**
- 时间点
- 截止时间
- 开始时间
- 纪念日

**配置建议：**
- 可选择是否包含时间
- 可设置默认值（如今天）
- 可设置时间范围

**常见错误：**
- ❌ 需要精确时间时未勾选"包含时间"
- ❌ 用文本字段存储日期（无法按日期筛选和排序）

**示例：**
- 截止时间
- 开始时间
- 发布时间
- 生日（不包含时间）
- 会议时间（包含时间）

---

### 8. 数字

**适用场景：**
- 金额
- 数量
- 百分比
- 评分
- 排序权重

**配置建议：**
- 可设置格式：常规、货币、百分比
- 可设置小数位数
- 可设置单位
- 可设置最小值/最大值

**常见错误：**
- ❌ 用文本字段存储数字（无法计算和排序）
- ❌ 金额字段未设置货币格式
- ❌ 百分比未设置为百分比格式

**示例：**
- 订单金额（货币格式，2位小数）
- 商品数量（常规格式，整数）
- 完成进度（百分比格式）
- 评分（1-5的整数）

---

### 9. 电话

**适用场景：**
- 手机号
- 座机
- 联系电话

**配置建议：**
- 支持点击拨打
- 自动识别电话格式

**常见错误：**
- ❌ 用文本字段存储电话（无法快速拨打）

**示例：**
- 联系电话
- 手机号
- 客服电话

---

### 10. 邮箱

**适用场景：**
- 邮箱地址

**配置建议：**
- 自动验证邮箱格式
- 支持点击发送邮件

**示例：**
- 联系邮箱
- 工作邮箱

---

### 11. URL

**适用场景：**
- 链接地址
- 文档链接
- 发布链接

**配置建议：**
- 支持点击跳转
- 可设置链接文本

**示例：**
- 文档链接
- 发布链接
- 参考资料链接

---

### 12. 复选框

**适用场景：**
- 是/否
- 开/关
- 完成标记
- 布尔值

**配置建议：**
- 只有两个状态
- 勾选或取消勾选

**常见错误：**
- ❌ 用单选（是/否）代替复选框

**示例：**
- 是否完成
- 需要提醒
- 已付款
- 已发布

---

### 13. 进度

**适用场景：**
- 任务完成度
- 项目进度
- 学习进度

**配置建议：**
- 显示为百分比
- 可视化进度条

**示例：**
- 任务进度
- 项目进度
- 学习进度

---

### 14. 评分

**适用场景：**
- 满意度评分
- 质量评分
- 星级评价

**配置建议：**
- 可设置最大值（如5星）
- 可视化显示

**示例：**
- 客户满意度
- 产品评分
- 面试评分

---

### 15. 附件

**适用场景：**
- 图片
- 文档
- 音视频
- 其他文件

**配置建议：**
- 支持多种文件格式
- 注意存储空间

**示例：**
- 产品图片
- 简历附件
- 设计稿

---

### 16. 关联

**适用场景：**
- 连接两张表
- 建立数据关系
- 避免数据重复

**配置建议：**
- 必须关联到另一张表
- 可设置关联条件
- 可设置是否允许关联多条
- 可选择显示字段

**常见错误：**
- ❌ 重复录入数据（应该用关联）
- ❌ 循环关联（A关联B，B又关联A）
- ❌ 关联了错误的表

**示例：**
- 订单表的"客户"字段 → 关联到客户表
- 跟进记录表的"客户"字段 → 关联到客户表
- 任务明细表的"所属任务" → 关联到任务表

---

### 17. 查找引用

**适用场景：**
- 显示关联表的其他字段
- 自动同步关联表的数据

**配置建议：**
- 必须基于关联字段
- 可显示关联表的任意字段
- 数据自动同步

**示例：**
- 在"订单明细表"中，关联"商品表"后，用查找引用显示"商品价格"
- 在"跟进记录表"中，关联"客户表"后，用查找引用显示"客户阶段"

---

### 18. 公式

**适用场景：**
- 自动计算
- 数据处理
- 条件判断

**配置建议：**
- 支持数学运算
- 支持文本处理
- 支持日期计算
- 支持条件判断

**示例：**
- 小计金额 = 数量 × 单价
- 订单总金额 = SUM(订单明细表.小计金额)
- 是否逾期 = IF(截止时间 < 今天(), "是", "否")
- 年龄 = YEAR(TODAY()) - YEAR(生日)

---

### 19. 按钮

**适用场景：**
- 快速操作
- 流程触发
- 状态变更

**配置建议：**
- 可执行预设操作
- 可配置按钮文字

**示例：**
- "标记完成"按钮
- "发送通知"按钮
- "归档"按钮

---

### 20. 修改记录

**适用场景：**
- 记录最后修改人
- 记录最后修改时间

**配置建议：**
- 自动记录，不可手动修改
- 可选择记录人和/或时间

**示例：**
- 最后修改人
- 最后修改时间

---

### 21. 创建记录

**适用场景：**
- 记录创建人
- 记录创建时间

**配置建议：**
- 自动记录，不可手动修改
- 记录后不变更

**示例：**
- 创建人
- 创建时间

## 字段选择决策树

```
需要存储什么类型的数据？

├─ 文字类
│  ├─ 短文本（<50字）→ 文本
│  ├─ 长文本（>50字）→ 多行文本
│  ├─ 互斥选项 → 单选
│  ├─ 多个标签 → 多选
│  └─ 电话/邮箱/链接 → 电话/邮箱/URL
│
├─ 人员类
│  ├─ 单个人 → 人员（单选）
│  ├─ 多个人 → 人员（多选）
│  └─ 团队/部门 → 群组
│
├─ 时间类
│  ├─ 只需要日期 → 日期（不含时间）
│  └─ 需要精确时间 → 日期（含时间）
│
├─ 数字类
│  ├─ 金额 → 数字（货币格式）
│  ├─ 数量 → 数字（常规格式）
│  ├─ 百分比 → 数字（百分比格式）或进度
│  └─ 评分 → 数字或评分
│
├─ 是/否类 → 复选框
│
├─ 文件类 → 附件
│
└─ 关联类 → 关联 + 查找引用
```

## 常见字段组合模式

### 1. 状态管理组合
```
- 当前状态（单选）+ 状态变更时间（日期）
- 任务状态（单选）+ 完成进度（进度）
```

### 2. 人员协作组合
```
- 负责人（人员）+ 协作人（人员，多选）
- 创建人（创建记录）+ 修改人（修改记录）
```

### 3. 时间跟踪组合
```
- 开始时间（日期）+ 截止时间（日期）
- 创建时间（创建记录）+ 修改时间（修改记录）
```

### 4. 关联关系组合
```
- 关联字段（关联）+ 显示字段（查找引用）
- 主表字段（关联）+ 关联记录数（公式）
```

## 最佳实践

### ✅ 推荐做法

1. **金额必须用数字字段**
   - 方便统计和计算
   - 支持货币格式

2. **状态必须用单选字段**
   - 便于筛选和分组
   - 支持看板视图

3. **人员必须用人员字段**
   - 支持@提醒
   - 支持按人员筛选

4. **避免数据重复，用关联**
   - 客户信息只存一次
   - 其他表通过关联引用

5. **必填字段要少**
   - 只填真正必须的
   - 降低录入门槛

### ❌ 避免做法

1. **用文本字段存储数字**
   - 无法计算和统计

2. **用多行文本记录结构化数据**
   - 无法分析和检索

3. **字段数量过多（>20个）**
   - 考虑拆表
   - 精简字段

4. **用单选存储应该用多选的内容**
   - 限制灵活性

5. **缺少状态字段**
   - 流程无法推进

## 字段命名规范

### 推荐命名方式

| 类型 | 示例 | 说明 |
|------|------|------|
| 状态类 | 任务状态、订单状态、客户阶段 | 名词+状态/阶段 |
| 时间类 | 截止时间、开始时间、创建时间 | 名词+时间 |
| 人员类 | 负责人、创建人、审核人 | 角色+人 |
| 数量类 | 订单金额、库存数量、实际工时 | 名词+金额/数量/工时 |
| 标识类 | 订单编号、客户ID、产品代码 | 名词+编号/ID/代码 |

### 命名原则

1. **简洁明确** - 2-6个字
2. **用业务语言** - 避免技术术语
3. **保持一致** - 同类概念用相同命名模式
4. **避免歧义** - 不要用"类型"、"分类"这种模糊词

---

**记住：选择正确的字段类型是设计好表结构的第一步。**
