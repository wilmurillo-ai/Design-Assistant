# 交互式文档编写：对话示例

以下是一个白皮书从 Discovery 到第一章完成的真实对话节奏示例。

---

## Phase 0: Discovery 片段

**AI**：这篇文档的核心目标是什么？你希望读者看完后产生什么认知或做出什么行动？

**用户**：写一个智慧工厂管控平台的白皮书，给客户看的。

**AI**：明白，面向客户的白皮书。追问一下——你希望客户看完后的主要行动是什么？是"认可你们的技术能力并进入商务洽谈"，还是"理解这个品类的价值并把你们列入候选名单"？这两个定位会影响内容的深度和侧重。

**用户**：前者，我们已经在候选名单里了，这个是加深信任的。

**AI**：那白皮书的定位就是"能力证明 + 信任建立"，而不是"市场教育"。这意味着内容应该偏重具体的技术方案和落地案例，而不是花大篇幅讲行业痛点。

接下来确认一下读者——主要决策者是什么角色？CTO/IT总监级别，还是生产/运营VP级别？

**用户**：两者都有，分管IT的副总和生产总监会一起看。

**AI**：好，这意味着我们需要两层叙事——业务语言讲清楚"能解决什么问题"给生产总监看，技术语言讲清楚"怎么实现的、是否可靠"给IT副总看。建议每个模块都按"业务价值 → 技术实现"的结构来组织。

到这里我对文档定位已经有了比较清晰的理解，我来总结一下：

```
## 文档定义摘要
- **名称**：智慧工厂管控平台（MOM）白皮书
- **类型**：白皮书（能力证明型）
- **核心目标**：加深已进入候选名单的客户对平台能力的信任
- **目标读者**：IT副总（关注技术可靠性）+ 生产总监（关注业务价值）
- **预期篇幅**：约 8000-12000 字
- **风格基调**：专业严谨但不晦涩，业务语言优先、技术细节适度
- **术语偏好**：中文 + 英文缩写注释（如"制造执行系统（MES）"）
```

这个定义准确吗？有需要调整的地方吗？

---

## Phase 2: Chapter Loop 片段（第一章）

**AI**：（进度：0/8 章，进入第1章「引言」）

第1章是引言，通常控制在 200-300 字，用一段话概括白皮书的核心主张和阅读价值。你希望开篇第一句传达什么核心信息？

**用户**：制造企业的竞争力取决于能不能把计划变成可控的执行。

**AI**：这个切入点很好——从"痛"入手。我来写一下引言。

*（写入文档文件）*

### 审计意见（第1章·引言）

> 审计身份：竞品公司资深文档顾问，初次阅读

- ✅ 逻辑严谨性：从挑战切入 → 引出平台定位 → 说明白皮书价值，链条完整
- ✅ 完整性：引言该交代的三件事（背景、方案定位、阅读收益）都覆盖了
- 💡 改进建议："覆盖供应链协同、仓储物流、计划排程……等关键场景"这个罗列在引言中略显冗长，建议精简为"覆盖从供应链到生产执行的全链路关键场景"，把详细模块留给后续章节展开

你看这个引言是否满意？要按建议精简那句话吗？

**用户**：可以，按你的建议改。

**AI**：已修改，进度已保存。（已完成 1/8 章）我们进入第2章「制造管理的关键挑战」——这一章你想覆盖哪些核心痛点？
