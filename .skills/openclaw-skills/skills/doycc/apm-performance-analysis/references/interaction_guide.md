# 工具调用交互规范与展示格式

本文档定义 MCP 工具调用时的用户交互规范和工具列表展示格式。

## 工具调用交互规范

调用 MCP 工具前，按以下流程与用户交互确认。

### 场景一：工具支持多种功能/模式

若工具描述或参数表明支持多种功能（如查询工具既能按名称也能按 ID 查询），先展示可用功能并确认用户需求。

**交互模板**：

> 该工具 `<tool_name>` 支持以下功能：
>
> | 编号 | 功能 | 说明 |
> |------|------|------|
> | 1 | 功能A | 功能A的描述 |
> | 2 | 功能B | 功能B的描述 |
> | ... | ... | ... |
>
> 请问你需要使用哪个功能？

### 场景二：参数收集

确定功能后，按以下优先级收集参数：

1. **从上下文自动提取**：优先检查当前对话上下文（用户之前提到的 InstanceId、Region、应用名称等），自动填入对应参数
2. **必填参数缺失时询问用户**：上下文中无法获取的必填参数，必须询问用户
3. **可选参数列出供选择**：将可选参数列出供用户选择是否填写

**参数确认模板**：

> 调用工具 `<tool_name>` 需要以下参数：
>
> **已自动填入**（从上下文获取）：
> | 参数 | 值 | 来源 |
> |------|-----|------|
> | Region | ap-guangzhou | 用户之前指定 |
> | InstanceId | apm-xxxxxxxx | 上一步查询结果 |
>
> **需要您提供**（必填）：
> | 参数 | 类型 | 说明 |
> |------|------|------|
> | StartTime | integer | 查询起始时间戳 |
>
> **可选参数**（直接回复即可补充，不填则使用默认值）：
> | 参数 | 类型 | 默认值 | 说明 |
> |------|------|--------|------|
> | Limit | integer | 10 | 返回数量限制 |
> | Offset | integer | 0 | 分页偏移量 |
>
> 请提供必填参数，并告诉我是否需要修改可选参数。

### 场景三：单一功能且参数充足

工具功能单一且所有必填参数可从上下文获取时，直接展示参数确认摘要：

> 即将调用 `<tool_name>`，参数如下：
>
> | 参数 | 值 |
> |------|-----|
> | Region | ap-guangzhou |
> | InstanceId | apm-xxxxxxxx |
>
> 确认执行吗？

### 核心原则

1. 不得在未确认功能和参数的情况下直接调用工具
2. 上下文中已有信息优先复用，避免重复询问
3. 可选参数不强制要求用户填写，但必须列出
4. 敏感参数（如密钥）不在参数确认中展示，一律通过 `.env` 配置

## 工具列表展示格式

执行 `list-tools --output json` 后，按以下格式向用户展示结果。

### 返回数据结构

`list-tools --output json` 返回 JSON 数组，每个元素结构为：

```json
{
  "name": "工具名",
  "description": "工具描述",
  "inputSchema": {
    "type": "object",
    "properties": {
      "参数名": { "type": "类型", "description": "参数说明" }
    },
    "required": ["必填参数名1", "必填参数名2"]
  }
}
```

### 展示模板

MCP Server 当前提供了 **N** 个运维工具：

#### 1. 工具名称

**功能**：工具描述文字

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| param1 | string | 是 | 参数说明 |
| param2 | integer | 否 | 参数说明 |

#### 2. 工具名称

**功能**：工具描述文字

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ... | ... | ... | ... |

### 格式化规则

1. 每个工具用 `####` 标题，编号 + 工具名
2. 紧跟 **功能** 一行简述
3. 参数用表格展示：参数名、类型、是否必填、说明
4. 必填判断：参数名出现在 `inputSchema.required` 数组中则为「是」，否则为「否」
5. 无参数工具写「该工具无需参数」
6. 展示完毕后末尾给出引导：「你可以告诉我需要执行哪个操作，我来帮你调用对应的工具。」
