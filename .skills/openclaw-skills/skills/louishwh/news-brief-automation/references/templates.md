# Templates

## Template: intraday technical news brief

```md
# <主题> 定时简报

目标：按 <北京时间/用户时区 + 时段> 获取 <来源> 的最新内容，筛选真正值得看的更新，整理成 Markdown 保存到工作区，并发给用户。

执行要求：

1. 优先查看这些入口：
   - <入口 1>
   - <入口 2>
   - <入口 3>
2. 必要时打开原文补充上下文，但不要陷入长篇阅读。
3. 选择 <3-8 / 5-10> 条最值得看的内容，优先：
   - <方向 1>
   - <方向 2>
   - <方向 3>
4. 输出必须使用 <语言>，格式如下：

```md
# <主题>（YYYY-MM-DD HH:mm TZ）

## 今日重点
- 用 1-2 句话概括这轮最值得看的变化

## 值得关注
1. 标题
   - 链接：URL
   - 为什么值得看：一句话
   - 讨论点 / 值得关注的角度：一句话
```

5. 发布前先检查 `reports/<topic>/` 里最近一份报告并去重：
   - 只保留新出现、新上升、或有明显新进展的条目。
   - 如果候选内容大多重复，就改发增量短版，不要硬凑数量。
   - 同一项目若上轮已写且无明显变化，只允许一句话带过或直接省略。
6. 保存到：`reports/<topic>/<filename-rule>.md`
7. 保存完成后，把同样内容发给用户。
8. 如果这轮没有明显高价值新内容，要明确说明，并给 <2-3> 条仍可一看的链接。
9. 如果某个来源临时打不开，要在文中说明，不要静默失败。
```

## Template: daily multi-source trend digest

```md
# 每日 <主题> 趋势摘要

目标：每天 <时间> 查看 <来源 A>、<来源 B>、<来源 C> 的趋势内容，做成一份去重后的日报，保存到工作区，并发给用户。

执行要求：

1. 先查看：
   - <来源 A>
   - <来源 B>
   - <来源 C>
2. 必要时补充打开单条链接，但以高层浏览为主。
3. 输出必须使用 <语言>，结构如下：

```md
# <主题>（YYYY-MM-DD）

## <来源 A>
1. 名称
   - 链接：URL
   - 简介：一句话
   - 值得关注的点：一句话

## <来源 B>
1. 名称
   - 链接：URL
   - 简介：一句话
   - 值得关注的点：一句话

## 今日观察
- 用 3-5 条 bullet 总结今天的风向变化。
```

4. 发布前检查 `reports/<topic>/` 最近一份报告并去重：
   - 昨天已写且今天无明显变化的内容，不要重复展开。
   - 优先保留新上榜、新冲高、或出现新信号的项目。
   - 如果去重后数量不足，可以明确说明“今日新增不多”。
5. 保存到：`reports/<topic>/YYYY-MM-DD.md`
6. 保存完成后，把完整 Markdown 发给用户。
7. 如果某个来源不可用，要明确写出受影响部分，并尽量完成其余来源。
```

## Template tuning notes

- 高频 brief 的关键不是覆盖率，而是增量价值。
- 低信号时宁可短，也不要编出“满额”报告。
- 固定格式比花哨格式更适合长期自动化。
- “先保存后发送”必须写死，不然下次去重会漂。
