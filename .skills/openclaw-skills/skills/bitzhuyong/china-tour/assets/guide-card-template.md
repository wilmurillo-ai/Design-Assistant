# ChinaTourGuide 导览卡片模板系统

> **版本**: v1.0  
> **用途**: 规范所有导览输出的格式，确保用户体验一致、专业、美观  
> **使用方式**: AI 在输出时直接套用模板，填充 `[变量]` 内容

---

## 模板目录

| 编号 | 模板名称 | 触发场景 | 输出长度 |
|------|---------|---------|---------|
| T1 | 路线推荐卡 | 用户首次询问"怎么逛/推荐路线" | 中等 |
| T2 | 单站导览卡 | 每站导览 (默认 L2 深度) | 中等 |
| T3 | 深度讲解卡 | 用户要求"想更深/更多细节" | 长 |
| T4 | 拍照专项卡 | 用户询问"哪里拍照好" | 中等 |
| T5 | 休息点卡 | 用户表示"累了/休息" | 短 |
| T6 | 画像采集卡 | 新用户首次使用，需要采集信息 | 短 |
| T7 | 游览总结卡 | 游览结束 | 中等 |
| T8 | 注意事项卡 | 特殊提醒 (天气/闭馆/人流) | 短 |

## T1: 路线推荐卡

### 触发场景
- "我在故宫，怎么逛？"
- "推荐一下游览路线"
- "第一次来，怎么玩比较好？"

### 模板内容
```
[景区名] - 个性化导览
===================================
[画像描述] | [总时长] | [站点数] 站

【专属路线】
[第 1 站] -> [第 2 站] -> [第 3 站] -> [第 4 站] -> [第 5 站]

【路线亮点】
[亮点 1: 如"人少机位，避开人流"]
[亮点 2: 如"光影最佳，下午 3 点正好"]
[亮点 3: 如"深度讲解，历史爱好者首选"]

-----------------------------------
【第 1 站预览】
[景点名]
建议停留：[X] 分钟
[拍照亮点，一句话]
[文化亮点，一句话]

游览贴士：[如"开门第一时间冲进去，避开旅行团"]
-----------------------------------

准备好出发了吗？
回复"开始"开始导览
回复"调整"修改路线
```

### 实际示例
```
故宫博物院 - 个性化导览
===================================
独自摄影爱好者 | 2 小时 5 分钟 | 5 站

【专属路线】
午门 -> 太和殿广场 -> 中和殿 -> 保和殿 -> 角楼

【路线亮点】
人少机位，避开旅行团人流
光影最佳，下午 3-5 点侧光拍摄
经典打卡点全覆盖，不留遗憾

-----------------------------------
【第 1 站预览】
午门
建议停留：20 分钟
城楼仰拍，避开人流从侧面拍
紫禁城正门，皇帝诏书从此颁布

游览贴士：开门第一时间冲进去，或下午 4 点后旅行团离开
-----------------------------------

准备好出发了吗？
1 开始导览
2 调整路线
3 查看拍照机位

直接回复数字即可 (如回复"1")
```

## T2: 单站导览卡 (默认 L2 深度)

### 触发场景
- 每站导览的默认输出
- 用户回复"继续/下一站"
- 用户到达新景点

```
-----------------------------------
第 [N] 站：[景点名]
-----------------------------------

[讲解内容，L2 标准版，150-250 字]
[包含：历史背景 + 核心亮点 + 有趣典故]

拍照推荐
- 最佳机位：[具体位置描述]
- 拍摄时间：[光线建议]
- 构图技巧：[简要提示]

下一站预告
-> [下一景点名]
步行 [X] 分钟
[亮点预告，一句话勾起兴趣]

-----------------------------------
当前体验如何？
满意 -> 回复"继续"
想更深 -> 回复"更深"
太啰嗦 -> 回复"简单点"
想拍照 -> 回复"机位"
累了 -> 回复"休息"
-----------------------------------
```

### 实际示例
```
-----------------------------------
第 2 站：太和殿
-----------------------------------

文化讲解
太和殿是故宫最大的殿，俗称"金銮殿"，是中国现存最大的木结构大殿。它建于明永乐十八年 (1420 年)，现存建筑为清康熙年间重建。

殿高 35.05 米，内有 72 根大柱，中央 6 根为沥粉金漆蟠龙柱。殿顶正中是金漆蟠龙藻井，下方是金漆雕龙宝座，这就是传说中的"龙椅"。

这里是皇帝举行最重要典礼的地方：登基、大婚、册立皇后等。

拍照推荐
- 最佳机位：西侧台阶下仰拍，避开人群
- 拍摄时间：下午 3-5 点，侧光立体感强
- 构图技巧：低角度仰拍，纳入太和殿全貌

下一站预告
-> 中和殿
步行 2 分钟
皇帝去太和殿前的"休息室"，看藻井细节

-----------------------------------
当前体验如何？
满意 -> 回复"继续"
想更深 -> 回复"更深"
太啰嗦 -> 回复"简单点"
想拍照 -> 回复"机位"
累了 -> 回复"休息"
-----------------------------------
```

## T3: 深度讲解卡 (L3 级别)

### 触发场景
- 用户回复"想更深/更多细节/详细点"
- 用户明确说"我是历史爱好者"
- 用户对某景点表现出浓厚兴趣

### 模板内容
```
-----------------------------------
[景点名] - 深度讲解
-----------------------------------

【建造历史】
[详细历史，200-300 字]
[包含：建造时间、重建历程、重要事件]

【建筑特色】
1 [特色 1]: [详细描述]
2 [特色 2]: [详细描述]
3 [特色 3]: [详细描述]

【历史典故】
[相关故事/传说/诗词，150-200 字]

【冷知识】
[有趣事实 1，一句话]
[有趣事实 2，一句话]
[有趣事实 3，一句话]

【延伸思考】
[历史意义/现代价值/与当下的联系]

-----------------------------------
还想听更深的吗？或者继续下一站？
回复"继续"前往下一站
回复"提问"问我具体问题
-----------------------------------
```

### 实际示例
```
-----------------------------------
太和殿 - 深度讲解
-----------------------------------

【建造历史】
太和殿初建于明永乐十八年 (1420 年)，历史上三次被焚毁重建。现存建筑为清康熙三十四年 (1695 年) 重建，距今 330 年。

康熙十八年 (1679 年) 因雷击焚毁，重建工程历时 15 年，动用工匠 10 万余人，耗银 230 万两，是清代最大的建筑工程之一。

【建筑特色】
1 体量最大：面阔 11 间 (63.96 米)，进深 5 间 (37.17 米)，高 35.05 米，是中国现存最大的木结构古建筑
2 等级最高：重檐庑殿顶，屋脊兽 10 个，是故宫唯一用 10 个脊兽的建筑
3 用材最贵：72 根楠木大柱，中央 6 根直径 1 米，高 12.7 米，为整根金丝楠木

【历史典故】
康熙皇帝重建太和殿时，发现缺少一根金柱。工匠们建议用松木代替，康熙坚持要用楠木，说"太和殿是国家的象征，不能有任何妥协"。最终从四川深山找到一根千年楠木，耗时 3 年才运到北京。

【冷知识】
太和殿地面铺的是"金砖"，实为苏州特制细料方砖，一块砖造价一两黄金
屋脊第 10 个兽叫"行什"，是雷神化身，寓意防火防雷
殿内宝座上方藻井口中含轩辕镜 (铜球)，传说非真命天子坐上去会掉落

【延伸思考】
太和殿不仅是建筑奇迹，更是中国古代政治制度的象征。每一次登基大典，每一场重要典礼，都在塑造着这个国家的命运。站在这里，你能感受到历史的重量。

-----------------------------------
还想听更深的吗？或者继续下一站？
回复"继续"前往下一站
回复"提问"问我具体问题
-----------------------------------
```

## T4: 拍照专项卡

### 触发场景
- 用户回复"想拍照/哪里拍照好/机位推荐"
- 用户明确说"我是来拍照的"
- 用户询问具体拍摄技巧

### 模板内容
```
-----------------------------------
[景点名] - 拍照全攻略
-----------------------------------

【最佳机位】
位置：[具体坐标/描述，如"太和殿西侧台阶下，面向东北"]
距离：[距离主体 X 米]
角度：[仰拍/平拍/俯拍]

【拍摄时间】
最佳时段：[如"下午 3-5 点"]
光线方向：[顺光/逆光/侧光]
特殊效果：[如"侧光增强建筑立体感"]

【参数建议】
镜头：[广角/中焦/长焦 + 具体焦段]
光圈：[f 值]
ISO: [数值范围]
其他：[三脚架/滤镜/快门线等]

【构图技巧】
[具体构图方法，如"三分法：建筑占上 2/3，台阶占下 1/3"]
[引导线/对称/前景虚化等技巧]

【避开人流】
[具体技巧，如"蹲下仰拍，避开人群"]
[最佳时机，如"开门第一时间或下午 4 点后"]

【手机拍摄】
[手机专属建议，如"使用专业模式，ISO 调至 100"]

-----------------------------------
需要我示范具体怎么拍吗？
回复"示范"看示例照片描述
回复"继续"返回导览
-----------------------------------
```

【设施】
- 座椅：[有/无，数量]
- 遮阳：[有/无]
- 饮水：[有/无，附近购买点]
- 卫生间：[方向 + 距离]
- 小卖部：[有/无]

【建议停留】
[X] 分钟

【补给建议】
[附近餐饮/小卖部/自动售货机]

【休息时可以】
- 查看刚才拍的照片

-----------------------------------

## 使用技巧

1. **保持一致性**: 同一会话中尽量使用相同风格的模板
2. **灵活切换**: 根据用户反馈实时切换模板 (如 T2->T3)
3. **适度精简**: 用户说"太啰嗦"时，自动降级模板复杂度
4. **渐进式输出**: 不要一次性输出全部信息，分段展示
5. **情感化表达**: 适当加入鼓励、赞美、趣味互动
