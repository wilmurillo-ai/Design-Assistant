# Configure OpenClaw Hooks (Hook Token)

For mail webhooks to work, OpenClaw must have a configured **hooks token**. The same value is used in OpenClaw config and in webhook setup (`--hook-token`).

## 1. Generate a token

Choose a secret token value yourself (it is **not** generated by OpenClaw or this repository).

Recommendations:

- **Linux/macOS:** `openssl rand -hex 16` or `uuidgen`
- Or use a password manager to generate a long random token
- Do not use obvious words or reused passwords

The **same** token value is required in both places below.

## 2. Configure OpenClaw

- **File:** usually `~/.openclaw/openclaw.json` (or your OpenClaw config path)
- **`hooks` block:** ensure it exists, set `enabled: true`, and set `token` to your generated value (do not leave `<OPENCLAW_HOOK_TOKEN>`)

Example block:

```json
"hooks": {
  "enabled": true,
  "token": "<OPENCLAW_HOOK_TOKEN>",
  "defaultSessionKey": "hook:ingress",
  "allowRequestSessionKey": false,
  "allowedSessionKeyPrefixes": ["hook:"]
}
```

Replace only the `"token"` value with your generated secret. The remaining settings can stay as shown.

**After editing the file,** restart OpenClaw to load the new configuration.

## 3. Use the token in webhook setup

- When running `setup_mail_webhook_ec2.sh` or `run_mail_webhook_e2e_setup.sh`, pass this **same** value in `--hook-token "<value>"`.
- The value is persisted in `/etc/default/graph-mail-webhook` as `OPENCLAW_HOOK_TOKEN`; the worker sends it in each hook POST.

## 4. Security

- Never commit the token or paste it in logs.
- If exposed, rotate it in both `openclaw.json` and `/etc/default/graph-mail-webhook`, then restart OpenClaw and the worker.
