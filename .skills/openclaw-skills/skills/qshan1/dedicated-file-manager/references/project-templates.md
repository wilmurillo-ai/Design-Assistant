# 项目目录模板

## 通用项目模板

适合大多数非代码类项目（文档、设计、素材管理）。

```
[项目名]/
├── 📄 README.md              # 项目说明
├── 📋 manifest.md            # 文件清单（Agent维护）
├── 📁 00_文档/               # 所有文档
│   ├── 设计文档.md
│   └── 会议记录.md
├── 📁 01_素材/               # 原始素材
├── 📁 02_产出/               # 最终成果
├── 📁 03_过程/               # 中间版本
└── 📁 _archive/              # 归档
```

## Agent 专属文件夹模板

Agent 的工作领地，自己管理文件、定期整理。

```
[专属文件夹]/
├── 📥 inbox/              # 收件箱（所有新文件入口）
├── 📁 projects/          # 项目工作区
│   ├── [项目A]/          #   按项目分子目录
│   └── [项目B]/
├── 📚 references/        # 参考资料、文档
├── 📦 archives/          # 归档区（不删除，只转移）
├── 🔧 scratch/           # 草稿、临时文件
├── 📋 manifest.md        # 文件清单（自动维护）
└── ⚙️ config.json        # 配置（路径、规则）
```

### 各目录用途

| 目录 | 用途 | 整理规则 |
|------|------|----------|
| inbox | 所有新文件的入口 | 每周自动分类分流 |
| projects | 项目相关文件 | 按项目分子目录 |
| references | 长期保留的参考资料 | 不主动移动 |
| archives | 旧文件、版本文件 | 超期临时文件移入 |
| scratch | 草稿、实验性文件 | 超30天自动归档 |

### 入口规则

**文件进入**：所有来源的文件 → inbox → Agent 按规则分类

**分类逻辑**：
```
文档类 (.md, .pdf, .docx) → references/
代码类 (.py, .js, .cs)    → projects/
素材类 (.png, .mp3)       → scratch/（等待进一步分类）
临时文件 (_temp, ~)       → 直接归档
```

### manifest.md 结构

```markdown
# 文件清单

更新时间：YYYY-MM-DD

## 统计
- 总文件数：X
- inbox：X（待整理）
- projects：X
- references：X
- archives：X
- scratch：X

## 待整理（inbox）
[文件列表...]

## 项目列表
[各项目文件统计...]
```

### 自动化

- **每周整理**：扫描 inbox → 按规则分类 → 更新 manifest → 生成报告
- **新项目**：扫描项目文件夹 → 建立清单 → 询问分类方式
- **临时清理**：超过30天的 scratch 文件 → archives

## 选择模板的指导

| 项目类型 | 推荐模板 |
|----------|----------|
| 文档写作、笔记 | 通用项目 |
| 软件开发 | 代码项目 |
| Unity/Unreal 开发 | 游戏项目 |
| 毕业论文、毕设 | 毕设/论文项目 |
| 图片/音频/视频管理 | 素材库 |
| AI/Agent 工作区 | Agent 专属文件夹 |

## 自定义模板

用户可以通过说"我要自定义模板"来：

1. 指定目录名称
2. 添加/删除目录
3. 设置默认归宿规则
4. 保存为命名模板

自定义模板会保存在 `~/.workbuddy/skills/file-manager/` 中。
