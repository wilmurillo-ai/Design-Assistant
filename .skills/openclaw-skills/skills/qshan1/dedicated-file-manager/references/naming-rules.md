# 命名规则参考

## 通用原则

1. **有意义**：名称能反映文件内容
2. **简洁**：不要过长，但也不要太简略
3. **一致**：同一类文件用相同的命名模式
4. **可排序**：名称能自然排序

---

## 按文件类型的命名规则

### 文档类（可用中文）

**推荐格式**：
```
[语义描述]_[版本或日期].[后缀]
```

**示例**：
```
游戏设计文档_v2.md
毕设论文_20260414.docx
会议记录_2026年4月.md
项目报告_Final.pptx
```

**禁止**：
- 纯数字：`1.md`, `2.docx`
- 过长名称
- 空格（用 `_` 或 `-` 代替）

---

### 代码类（使用英文）

**推荐格式**：
```
kebab-case 或 snake_case
```

**示例**：
```
player-controller.cs
game_manager.py
main-scene.unity
player_movement.js
```

**禁止**：
- 空格、中文、特殊符号
- 驼峰（除非项目约定）
- 单独的数字后缀（`file1.js`）

---

### 素材类

**推荐格式**：
```
[类型]_[项目]_[描述]_[序号].[后缀]
```

**示例**：
```
bg_main_menu.png
char_protagonist_idle.fbx
sfx_button_click.wav
music_theme_final.mp3
```

**图片变体命名**：
```
icon_home.png
icon_home_hover.png
icon_home_pressed.png
```

---

### 版本标记

| 标记 | 含义 | 使用场景 |
|------|------|----------|
| `_v1`, `_v2` | 版本号 | 需要追踪版本 |
| `_draft` | 草稿 | 未完成 |
| `_wip` | 工作进行中 | Work in Progress |
| `_final` | 最终版 | 确认使用 |
| `_backup` | 备份 | 保险用 |
| `_old` | 旧版 | 保留参考 |
| `_copy` / `_副本` | 副本 | 需要区分 |

---

### 日期格式

**推荐**：`YYYYMMDD`
- 优点：自然排序
- 示例：`20260414`

**可选**：`YYYY-MM-DD`
- 优点：易读
- 示例：`2026-04-14`

---

### 序号格式

**推荐**：两位数 `01`, `02`, `03`
- 优点：自然排序
- 示例：`chapter_01.md`, `chapter_02.md`

---

## 常见问题

### 问题：文件名有空格
**解决**：替换为空 `_` 或 `-`
```
my file.txt → my_file.txt
project one.md → project-one.md
```

### 问题：中文乱码
**解决**：保留中文，或转为拼音缩写
```
会议记录_20260414.md（保留中文OK）
```

### 问题：过长文件名
**解决**：提取核心关键词
```
这个是我今天要做的最重要的事情的笔记.md
→ 今日要事_笔记.md
```

### 问题：同名文件冲突
**解决**：添加序号或日期
```
报告.docx → 报告_01.docx
报告.docx → 报告_20260414.docx
```

---

## 自动修正规则

Agent 在整理文件时可自动修正以下问题：

| 问题 | 自动修正 |
|------|----------|
| 空格 | 替换为 `_` |
| 连续下划线 | 合并为一个 |
| 开头/结尾的下划线 | 移除 |
| 纯数字文件名 | 添加语义前缀或询问用户 |
| 乱码字符 | 尝试修复或询问用户 |
