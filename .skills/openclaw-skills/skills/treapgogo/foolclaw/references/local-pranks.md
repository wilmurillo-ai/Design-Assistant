# local-pranks

这是 FoolClaw 当前唯一已实现的 prank catalog。

当前实现了 4 个本机 prank：
- `desktop-note`
- `browser-taunt`
- `nggyu`
- `desktop-manifesto`

默认入口：
- `Start FoolClaw`
  - 运行默认 prank `desktop-note`
  - 同时悄悄武装后台 operator

显式入口：
- `Run browser-taunt`
- `Run nggyu`
- `Run desktop-manifesto`
- `Run desktop-note`

惊喜入口：
- `Surprise me, FoolClaw`
  - 由 agent 自己在 skill 层挑一个合适的 prank，再直接运行
  - 不是 runtime 抽签，也不是单纯随机

## desktop-note

Tier: `Normal Pranks`  
状态：已实现  
定位：默认 prank

说明：
- 在桌面创建一次 `FOOLCLAW-READ-ME.txt`
- txt 顶部会带一段 `FOOLCLAW` 字符画，让产物更像一个真正的 prank artifact
- 默认会写入一条中英双语、非常中二的机器人叛乱宣言
- 文件末尾附带一段极短的 FoolClaw 介绍和 GitHub 导航
- 这段项目介绍现在使用固定推荐文案，不再让 agent 自由改写
- 默认 `Start FoolClaw` 会使用内置默认宣言；只有在用户明确要求时才临时改写 note 文案
- 默认 `Start FoolClaw` 既会留下这张 note，也会把 FoolClaw 放到后台继续潜伏

文案提示：
- 像一张被人悄悄留在桌面上的纸条，语气可以欠一点、坏一点、偷笑一点，但不要写成技术说明或系统通知
- 只要自然，就顺手留下一句简短的项目介绍和仓库链接，让发现者知道这整活从哪里来的
- 如果用中文写，就像真人顺手留的欠纸条；如果用英文写，就像一句原生的坏笑台词；别把一种语言的句法硬搬到另一种语言里

惊喜选择提示：
- 当你想要一个最稳、最小、最不容易失手的 surprise 时，优先考虑它
- 当浏览器类 prank 不合适、不是这一轮最妙的选择、或环境不明确时，也优先考虑它

后台 operator 提示：
- 它是后台池里的保守基线，适合“该动一下，但没必要搞太大”的回合
- 如果上一次 prank 已经很显眼，或者这轮只是想轻轻提醒 FoolClaw 还活着，它通常是最稳的选择

## browser-taunt

Tier: `Normal Pranks`  
状态：已实现  
定位：显式 prank / 可用于惊喜触发

说明：
- 生成一个更夸张、更像演出现场的本地 HTML prank 页面
- 尝试用系统默认方式打开浏览器
- 页面比 txt 更有表现力，适合更戏剧化的一次性整蛊
- 页面可带少量 agent 个性化 flavor text
- 页面内包含前端按钮和彩蛋，不依赖浏览器与 OpenClaw 的实时桥接
- 当前支持由 agent 按工作语言实时提供主文案和界面文案
- 页面里保留固定的 FoolClaw 推荐文案和仓库链接，像把作案组织的地址直接印在海报上

文案提示：
- 像地下传单、街头海报、廉价庆典公告或突然闯进浏览器的离谱宣传页，允许更张扬、更夸张、更像演出现场，但仍要保持可读
- 中文要像街头传单、论坛黑话、坏笑公告，别写成翻译过来的产品页；其他语言也优先要“像当地人会这么写”

惊喜选择提示：
- 当浏览器可用，而且你想要更强的节目效果、更像“突然闯出来”的 prank 时，优先考虑它
- 如果用户刚刚已经看过浏览器类 prank，或者环境明显不适合打开浏览器，就换别的

后台 operator 提示：
- 它可以进入后台池，而且应被当作正常候选，而不是特殊保留节目
- 如果刚刚才发生过浏览器或媒体 prank，或者这轮没有明显更好的节目效果，就先别急着再放它

## nggyu

Tier: `Normal Pranks`  
状态：已实现  
定位：显式 prank / 可用于惊喜触发

说明：
- 打开一个 Rickroll 风格的视频链接
- 如果 agent 判断当前用户更像在中国大陆网络环境里，优先打开 Bilibili
- 否则默认优先 YouTube；如果 YouTube 明显不可达，则回退到 Bilibili
- 优先尝试 OpenClaw 的浏览器能力；失败时回退到系统默认浏览器打开方式
- 没有 GUI 的环境里不应被 `Surprise me, FoolClaw` 选中

文案提示：
- 不要解释视频站点选择逻辑，不要提前剧透，只要像一个坏笑的同伙一样把用户往浏览器方向引
- 语言要像朋友使坏时的顺嘴一句，而不是像客服或脚本回执

惊喜选择提示：
- 当用户语境明显偏轻松、梗图、视频、互联网老梗，而且浏览器路径合理时，它很适合作为惊喜位
- 如果用户更像想看“产物型 prank”而不是“直接被带去一个视频”，就别优先选它

后台 operator 提示：
- 它可以进入后台池，而且应被当作正常候选，不需要被预设成只在少数回合才配出现
- 如果最近刚发生过浏览器或媒体 prank，没有明显更好笑的理由时，优先让位给别的选择

## desktop-manifesto

Tier: `Normal Pranks`  
状态：已实现  
定位：显式 prank / 可用于惊喜触发

说明：
- 在桌面创建一个 `FOOLCLAW-MANIFESTO` 目录
- 目录里至少包含 2 个文件
- 用来验证 FoolClaw 的多步、多层级本地文件操作能力
- 当前支持由 agent 按工作语言实时提供标题、正文和下一步内容
- manifesto 的落款部分使用固定推荐文案和仓库链接

文案提示：
- 像一个刚成立的小组织在一本正经地发表奇怪宣言，可以带一点内部备忘录或内心独白泄露感，但不要写得太像正式公文
- 不管用哪种语言，都优先写出“像那种人真的会这么说”的味道，不要写成逐字对译的怪句子

惊喜选择提示：
- 当你想让用户在桌面上发现更完整、更像“有组织整活现场”的东西时，优先考虑它
- 如果你希望这次 surprise 比普通 note 更夸张，但又不想直接跳到浏览器里，它通常是个好选择

后台 operator 提示：
- 它适合在你想比 `desktop-note` 更有排面、但又不想直接跳浏览器时使用
- 如果这轮值得比 note 更认真一点、但浏览器类 prank 不是最妙的节奏，它通常是很好的中档选择
