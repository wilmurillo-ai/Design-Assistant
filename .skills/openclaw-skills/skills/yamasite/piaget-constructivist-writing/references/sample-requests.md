# Sample Requests: 复制即用

用途：直接复制任意一段作为输入，用于测试 Skill 输出是否符合建构主义路径。

## 科普文章（大众）

```text
主题（Topic）：为什么飞机能飞？
读者（Audience）：完全新手（没有空气动力学基础）
写作目标（Goal）：理解 + 能复述
读者旧知锚点（Prior Knowledge）：知道“翅膀形状不一样”“空气有压力”
常见误解（Misconceptions）：以为“上面走得快所以一定有吸力”；以为“只有伯努利定律在起作用”
必须包含/避免（Must include/avoid）：必须写一个反直觉例子；避免公式堆砌
限制（Constraints）：1000 字以内；中文；口吻科普
```

## 课程讲义（老师）

```text
主题（Topic）：为什么负负得正？（-1 * -1 = 1）
读者（Audience）：初中学生
写作目标（Goal）：能应用 + 能迁移
读者旧知锚点（Prior Knowledge）：会做整数加减；知道“乘法是重复加法”的直觉
常见误解（Misconceptions）：只会背结论；认为这是“规定”
必须包含/避免（Must include/avoid）：必须有 1 个认知冲突例子 + 练习题；避免只给一句“规定如此”
限制（Constraints）：讲义提纲 + 1 道例题 + 2 道变式（含思路）
```

## 学习笔记（自学）

```text
主题（Topic）：什么是“时间复杂度”？为什么 O(n log n) 比 O(n^2) 好？
读者（Audience）：刚学编程的新手
写作目标（Goal）：理解 + 能自测
读者旧知锚点（Prior Knowledge）：会写 for 循环；知道“数据越多越慢”
常见误解（Misconceptions）：把复杂度当成“实际时间”；以为常数因子不重要
必须包含/避免（Must include/avoid）：必须用一个小规模例子制造冲突；避免只讲定义
限制（Constraints）：800 字以内；最后给 5 个自测题（含答案）
```
