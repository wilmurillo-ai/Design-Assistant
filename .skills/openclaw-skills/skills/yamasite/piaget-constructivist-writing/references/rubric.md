# Rubric: Success Criteria & Scoring (Constructivist Writing)

用途：把用户目标转成可验证成功标准，并用于最终“目标达成评估”。

## 评分尺度（建议）

- 0 = 不满足（缺失或明显偏离）
- 1 = 部分满足（有但不稳定/有硬伤）
- 2 = 满足（可用且基本稳定）
- 3 = 超出预期（非常清晰/非常贴合目标）

## 核心项（默认必选）

- 旧知锚点：是否明确读者已知，并以此作为起点（而不是从术语开讲）。
- 可控冲突：是否用可验证的小问题/反例引出“旧模型解释不了”的点。
- 支架渐进：是否分步引入概念与术语；每步都有定义与最小例子。
- 迁移练习：是否提供从熟悉到新场景的迁移题，并给出思路。
- 误解纠偏：是否覆盖关键误解并解释“为什么容易错”。

## 可选项（按场景选 0-5 条）

- 类比严谨：是否写清相似点/不同点，避免误导迁移。
- 节奏与负荷：信息密度是否匹配 Audience（不跳步、不灌输）。
- 证据边界：是否区分普遍结论与条件结论。
- 可复盘性：笔记/讲义是否能被快速复用（层级清楚、关键信息前置）。

## 判定规则（从分数到结论）

- `已达成`
  - 核心项中没有 0 分项
  - 核心项平均分 >= 2.3
- `部分达成`
  - 核心项存在 1 分项，或核心项平均分在 1.5-2.2
- `未达成（需澄清）`
  - 任一核心项 = 0，或核心项平均分 < 1.5
  - 或输入上下文不足，无法构造可信的旧知锚点/冲突/支架

输出评估时必须写清：

- 哪些核心项没达标
- 为什么没达标
- 最值得优先修的 1-3 项
