# Example Outputs: 从输入到成品

用途：对齐“建构式讲解”的颗粒度与节奏。需要校准风格时先看最接近的例子。

## 示例 1：学习笔记（时间复杂度）

### 输入

```text
主题（Topic）：什么是“时间复杂度”？为什么 O(n log n) 比 O(n^2) 好？
读者（Audience）：刚学编程的新手
写作目标（Goal）：理解 + 能自测
读者旧知锚点（Prior Knowledge）：会写 for 循环；知道“数据越多越慢”
常见误解（Misconceptions）：把复杂度当成“实际时间”；以为常数因子不重要
必须包含/避免（Must include/avoid）：必须用一个小规模例子制造冲突；避免只讲定义
限制（Constraints）：800 字以内；最后给 5 个自测题（含答案）
```

### 输出（示例）

```md
## 概念地图

- 核心概念：输入规模 n、操作次数、增长趋势、O 表示法、对数 n log n
- 关系：复杂度描述“增长趋势”，不是某台机器的具体秒数

## 学习路径

- 旧知锚点：for 循环里做事次数越多，程序越慢
- 认知冲突：两段代码 n=10 时差不多，但 n=10 万时差距会爆炸
- 新概念：用 O(·) 只描述增长趋势，忽略常数与低阶项
- 迁移：能判断常见代码的复杂度，并知道什么时候该优化

## 分层讲解（短讲解）

1) 你可以先把“时间复杂度”理解成：当数据量 n 变大时，代码需要做的“工作量”怎么变。

2) 认知冲突（小例子）：
- 代码 A：两层循环，各跑 n 次 -> 总次数大约 n^2
- 代码 B：每次把范围对半缩小（像二分查找）-> 需要做大约 log n 次
当 n=10 时，n^2=100，log n 也就几次，差不多。
当 n=100000 时，n^2=10^10 次，很容易把时间拖爆；而 n log n 仍然可控很多。

3) 为什么说 O(n log n) “更好”：
- 不是说一定更快（常数也会影响）
- 而是说 n 变大时，它增长得更慢，更不容易在大规模输入下失控

## 练习题（含思路与答案）

1. 概念题：时间复杂度描述的是“具体秒数”吗？
   - 答案：不是，描述的是增长趋势；具体秒数取决于机器、实现、常数因子等。
2. 辨析题：O(n) 一定比 O(n log n) 快吗？
   - 答案：趋势上更慢增长，但小 n 或常数差异大时不一定；看场景与规模。
3. 应用题：一层循环跑 n 次，里面做 O(1) 的操作，复杂度是？
   - 答案：O(n)。
4. 应用题：排序 + 一次遍历，复杂度大约是？
   - 答案：排序通常 O(n log n)，遍历 O(n)，整体 O(n log n)。
5. 迁移题：为什么很多算法喜欢“分治/对半”？
   - 答案：因为把规模不断减半会带来 log n 的层数，整体更可控。

## 误解纠偏

- 误解：复杂度就是“运行时间”
  - 为什么容易错：我们直觉里只看秒数，但忽略输入规模变化
  - 正确模型：复杂度是增长趋势；秒数 = 常数因子 * 操作次数 + 其他开销
  - 自检：换 n 从 1e3 到 1e6，趋势是否仍成立？

## 目标达成评估

- 结论：已达成
- 理由：旧知锚点明确、有可控冲突、逐步引入 O 表示法，并给了迁移题
```

## 示例 2：人文/社会科学（什么是“社会角色”）

### 输入

```text
主题（Topic）：什么是“社会角色”？为什么同一个人在不同场合会表现得不一样？
读者（Audience）：高中生
写作目标（Goal）：理解 + 能复述 + 能举例应用
读者旧知锚点（Prior Knowledge）：知道“在家和在学校表现不一样”；知道“老师/学生/父母”这些身份
常见误解（Misconceptions）：以为“社会角色”就是“演戏装出来”；以为角色变化等于“虚伪”；把“身份标签”和“角色期待”混为一谈
必须包含/避免（Must include/avoid）：必须从日常经验出发；必须有 1 个反例；避免空泛定义
限制（Constraints）：1000 字以内；中文；像一段可直接上课的讲解
```

### 输出（示例）

```md
## 概念地图

- 核心概念：社会角色、身份、角色期待、场景规范、行为调整
- 关系：
  - 身份告诉别人“你是谁”
  - 社会角色强调“在某个关系和场景中，别人期待你怎么做”
  - 同一个人可以同时拥有多个角色

## 学习路径

- 旧知锚点：你在家里、在班级里、在社团里，说话做事可能都不一样
- 认知冲突：如果“做自己”最重要，为什么同一个人会在不同场景下切换表现？这是不是“虚伪”？
- 新概念：社会角色不是装出来的戏，而是社会关系对行为的期待
- 迁移：能分析一个人为什么在不同场景里会采取不同表达方式

## 分层讲解（短讲解）

1) 先从你熟悉的经验开始：同一个同学，在老师面前和在好朋友面前，讲话方式往往不一样。这个差异不一定说明他“变了一个人”，而是他所在的关系和场景不同。

2) 认知冲突：
- 我们常说“做人要真实”
- 但现实中，一个人既可能是学生，也是女儿/儿子，也可能是社团负责人
- 如果“真实”只等于“永远一种说话方式”，那很多正常的场景切换就解释不通

3) 这时就要引入“社会角色”：
- 社会角色不是演戏，而是一个人在某种社会关系中，被期待采取的一套行为方式
- 比如“学生”这个角色，会被期待按时上课、完成作业、遵守课堂规则
- “班长”这个角色，则会多出组织、协调、传达信息的期待

4) 这能解释为什么同一个人在不同场景下会有不同表现：
- 不是他没有“真实的自己”
- 而是不同角色有不同任务、责任和互动规则

5) 但角色也不是无限变化的。反例是：如果一个人在所有场合都完全只顾自己、完全不理会基本规则，那不是“更真实”，而可能是没有完成对应角色应承担的责任。

## 练习题（含思路与答案）

1. 概念题：社会角色和身份标签是一回事吗？
   - 答案：不完全一样。身份标签更像“你是谁”，社会角色更强调“在某种关系中，别人期待你怎么做”。
2. 辨析题：一个人在不同场景说话方式不同，就是虚伪吗？
   - 答案：不一定。很多时候，这是在回应不同角色的责任与场景规范，而不是故意伪装。
3. 应用题：为什么班长在同学聚会和在班会上的表现可能不同？
   - 答案：因为场景不同，角色期待不同。聚会时更偏朋友互动，班会时更偏组织与协调。
4. 迁移题：为什么医生面对病人和面对家人时表达方式可能不同？
   - 答案：面对病人时承担专业角色，需要清晰、负责、谨慎；面对家人时更多体现亲密关系角色。

## 误解纠偏

- 误解：社会角色就是“装”
  - 为什么容易错：因为我们把“不同表现”直觉地理解为“不真诚”
  - 正确模型：角色差异更多是在不同关系里回应不同期待与责任
  - 自检：这个变化是在逃避责任，还是在适应场景任务？

- 误解：一个人只能有一种稳定角色
  - 为什么容易错：我们习惯用单一标签看人
  - 正确模型：同一个人往往同时处在多个关系网络中，因此也拥有多个角色
  - 自检：把“身份”“角色”“责任”三个词分开看

## 目标达成评估

- 结论：已达成
- 理由：从高中生日常经验出发，引入可控冲突，再用“身份/角色/期待”的关系重建概念，并给出了应用与辨析题
```
