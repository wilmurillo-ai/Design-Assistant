# “AI 味”与“人味”的本质差别（用于诊断与取舍）

## 本质差别（可操作版本）

- **意图与代价**：人写作往往背后有真实目标与代价（要说服谁、要承担后果）；AI 默认“安全正确”，缺少真实押注与取舍。
- **证据与可证伪**：人能提供来源、细节、反例、边界；AI 容易给“对的废话”或“听起来像证据的句子”但不可核验。
- **视角与偏好**：人有稳定偏好与语气脾性（甚至小缺点）；AI 输出更趋于平均化、滴水不漏、同质化。
- **信息结构**：人会围绕关键冲突/关键决定组织材料；AI 更倾向模板结构（总分总、先说结论、分点罗列）。
- **语言节奏**：人有自然的长短句波动与不对称表达；AI 更易出现节奏均匀、连接词堆叠、段段都总结。

## 信息传递视角（把“去味”从文风拉回内容）

如果把写作看成“信息传递”而非“文字拼接”，那么“AI 味”最常见的根因不是某个词，而是：

- **信息密度低**：句子很完整，但新增信息很少（读完记不住可复述的事实）
- **可证伪性弱**：结论很强，但没有可核验锚点（来源/口径/复现步骤）
- **决策与代价缺失**：只讲“重要/建议/提升”，不讲“选什么/不选什么/要付出什么”
- **边界缺失**：没有“不适用条件/反例/失败模式”，导致内容看起来“永远正确”

因此：去味的第一步是补信息与约束，让文本能被验证、能指导行动；去指纹（口癖/格式）只能作为最后一遍清理。

## 三个“根因分流”（决定该不该只润色）

1. **信息不足型**：没有数字、案例、约束、边界、对比、来源。先追问补信息，别硬润色。
2. **结构模板型**：信息其实够，但被“首先其次最后/三段论/列表化”淹没。先重组结构再改句子。
3. **口吻格式型**：内容没问题，但暴露高频指纹（过度帮助式收尾、乱加粗、客服共情）。走 De-Fingerprint Pass 清理即可。

## 额外可加的检查范式（建议）

- **可证伪性检查**：每段至少落地到 1 个可核验点（人/事/数/时间/机制/约束/边界）。
- **结论强度校准**：结论有多强，证据就要匹配多强；不够就降级措辞或补证据。
- **取舍与边界**：给 1 次明确取舍 + 1 个不适用条件（避免“两边都对”的空泛中立）。
- **来源纪律**：不写“研究表明/数据显示”除非有来源；没有就改成“经验判断/常见情况”，并提示补链接。
- **受众/媒介适配**：同一句话在“对外公关/内部协作/学术论文/社媒种草”允许的温度与格式不同，先对齐场景再去味。

## Hallucination Gate：拦截“语法正确但不真实”的句子（默认启用）

目标：不是“抓错别字”，而是拦截那些看起来很专业、语法完美、但缺乏可核验锚点的内容，尤其是乱编参考文献/来源。

### 触发条件（任一满足即触发）

- **强引用但不可核验**：出现具体论文/报告/机构/期刊/年份/DOI/会议/作者组合，但用户无法提供链接、PDF、截图或可复制的原文片段。
- **假证据句式**：`研究表明/数据显示/大量实践证明/权威机构指出/据某某报告` 但没有最小出处（机构名+年份+链接/文档名）。
- **强数字无口径**：出现精确数字/百分比/排名/时间线，但没有口径、样本、范围、计算方法或数据源。
- **步骤可执行但不可复现**：操作指南给了步骤，却缺少可观察输出（预期页面/返回值/日志/错误码/版本）。
- **过度自洽且无代价**：结论很满、语气很稳，但不说明边界、反例、失败条件、成本与限制。

### 处理策略（按优先级）

1. **先要证据**：把结论变成问题，向作者索要最小可核验锚点：
   - 链接/DOI/报告原文截图/引用段落/数据表/实验设置/版本号
2. **拿不到就降级**（禁止编造）：
   - 把“事实/研究结论”降级为“经验判断/可能性/常见情况”，并明确前提条件。
   - 删除虚构的参考文献条目；不要为了“显得专业”保留假引用。
3. **补边界与反例**：
   - 增加“什么时候不成立/不适用”“最常见的失败模式/反例”。
4. **最小复现闭环**（操作类内容）：
   - 每步补齐：输入 → 操作 → 预期输出 → 常见错误与排查。

### 快速识别小抄（写作时自检）

- 每个“强断言”是否能回答：`我如何在 5 分钟内验证它？`
- 每个“引用”是否具备：`机构/作者 + 年份 + 可打开的链接/可贴出的原文片段`
- 每个“数字”是否具备：`口径 + 范围 + 来源`
