# 日本語：AIっぽさのサインと自然化の手筋

## よくある「AIっぽい」サイン

- 型が強い：`まず/次に/最後に`、`結論として`、`まとめると` が機械的に並ぶ
- 語尾が単調：`です。/ます。` が連続し、リズムが一定
- 無難すぎる：当たり障りのない一般論、両論併記で判断がない
- 具体性不足：数字・条件・失敗談・例外・制約が少ない
- いかにも説明口調：読者を“講義”してしまう（場面に不一致）
- 形式の癖：箇条書きだらけ、見出しだらけ、強調（太字/記号）が多すぎる
- 「重要/核心」連発：`重要/本質/ポイント/鍵` が多いのに根拠が薄い
- 過度に丁寧な締め：`お役に立てれば幸いです` などの定型締めが頻発

追加でバレやすいサイン（高頻度）：

- 受け身・責任回避の定型：`〜が求められる/必要がある/推奨される` が多い（誰が、何のために、が消える）
- 断定回避の定型：`〜と言えるでしょう/〜と考えられます/〜かもしれません` が安全運転で連発
- 抽象語の多用：`様々な/多くの/いくつか/一定の/適切な/効果的な` が多いのに中身がない
- 指示語でぼかす：`このような/そのような/こうした` が続き、具体物が出てこない
- 前置きが長い：`現代社会において/近年/周知の通り` で始まり、本題が遅い
- 質問→即答の型：`なぜ〜なのでしょうか？それは〜だからです。` を段落ごとに繰り返す
- 擬似エビデンス：`研究によると/データによれば` と言うが、出典・年・リンクがない

## 直し方（優先順）

- 具体を入れる：
  - 5W1H、数字、条件、例外、比較（何と比べてどう良い/悪い）
- 判断 + 境界を出す：
  - `私はAに寄せる（理由）/ただしBの条件ならAは避ける`
- 語尾と文の長さを混ぜる：
  - 短文で切る、体言止めを少し混ぜる、接続を言い換える
- テンプレ接続を減らす：
  - `さらに/また/一方で` を連打しない。段落の意味で繋ぐ
- 形式を整える：
  - 箇条書きは並列だけ。背景・温度・経験は本文で書く
  - 強調は最小限（要所だけ）
- 定型締めを避ける（デフォルト）：
  - `お役に立てれば幸いです` を消し、次の行動や確認事項で終える

追加の手筋（よく効く）：

- 受け身を能動に戻す：
  - `〜が求められます` → `私たちは〜をやる必要があります` / `運用上は〜を必須にします`
- 主語を復活させる：
  - `重要です` → `この場面ではXが重要です（理由：Yが起きるから）`
- 抽象語を削るか、定義する：
  - `様々な課題` → `具体的にはA（例：…）、B（例：…）`
- 指示語を具体物に置き換える：
  - `このような` → `たとえば「◯◯の承認フロー」` のように名詞を出す
- 断定回避を“条件付き断定”に変える：
  - `と言えるでしょう` → `Xの条件ならYになる/ただしZでは崩れる`
- 擬似エビデンスは Gate を通す：
  - `研究によると` を書くなら、最低限 `機関/著者 + 年 + リンク/論文ID` を出す
  - 出せないなら `経験上/現場では` に降格し、前提条件を明記する

## 定型フレーズ置換（即効）

- `〜することができます` → `〜できます`（冗長さを減らす）
- `〜という点が挙げられます` → `〜がある` / `〜が起きる`
- `現代社会において` → 直接本題へ（必要なら2文目で背景）
- `一般的に/多くの場合` → 条件を書く（`Xのときは`）

## 出典・引用がある文章の注意（幻覚対策）

- 参考文献・論文・統計を出す場合、最低限「検索できる錨」が必要：
  - 論文：タイトル（完全一致で検索できる形）+ 著者 + 年 + DOI/URL
  - 書籍：ISBN + 出版社/版
- 錨が出せない引用は削除し、主張を「仮説/経験」に降格する（詳しくは `references/citation-verification.md` と `references/principles.md` の Hallucination Gate）

## 「サイン→最小のBefore/After例」（`references/signs-of-ai-writing.md` 対応）

1) まとめ過ぎ / 反復する締め
- Before：`まとめると、これは非常に重要です。お役に立てれば幸いです。`
- After：`次にやること：今週の運用データで1回試して、工数とミス率を比べる。`

2) 接続詞・路標の使い過ぎ
- Before：`さらに、改善が必要です。また、コミュニケーションも重要です。`
- After：`問題は引き継ぎです。担当者と期限を固定し、手順を1枚にまとめます。`

3) 「3点セット」癖
- Before：`3つのポイントは、効率・品質・コストです。`
- After：`結局、効率が最優先です。理由：締め切りが固定で、遅れると損失が出る。`

4) 受け身/断定回避でぼかす（機構がない）
- Before：`改善が求められます。適切に対応する必要があると言えるでしょう。`
- After：`私たちは承認フローを1段減らします。例外は金額が一定以上の申請だけにします。`

5) 修辞が多いが情報が増えない
- Before：`これは単なる改善ではなく、変革です。`
- After：`重複していた確認を1回に減らしました。その結果、処理時間が約3割短くなりました。`

6) 形式ツール箱（見出し/箇条書き/強調）痕跡
- Before：`## 結論\n- **重要**：効率化\n---\n結論として…`
- After：`結論：まず重複確認を削る。次に、例外条件だけを明文化する。`

## 情報不足なら先に聞く

- 目的（誰に、何をしてほしいか）
- 前提（状況、制約、期限、読者の知識レベル）
- 具体（数字、実例、失敗、比較対象、リンク/資料）
