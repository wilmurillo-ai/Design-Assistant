# Search patterns for recipe discovery

Use targeted search queries instead of generic searches. Start with the dish or key ingredients, then add the practical constraint that matters most.

## Ingredient mode
- `<ingredient 1> <ingredient 2> recipe`
- `<ingredient 1> <ingredient 2> weeknight dinner`
- `<ingredient 1> <ingredient 2> air fryer recipe`
- `<ingredient 1> <ingredient 2> kid friendly recipe`
- `<ingredient 1> <ingredient 2> healthy recipe`
- `<ingredient 1> <ingredient 2> one pan recipe`

## Surprise mode
- `best easy weeknight dinners 2026`
- `most popular family dinner recipes 2026`
- `healthy high protein dinner recipes 2026`
- `best air fryer dinners 2026`
- `kid friendly dinner ideas 2026`

## Diet or style modifiers
Append one of these when relevant:
- vegan
- vegetarian
- gluten free
- dairy free
- high protein
- low carb
- Mediterranean
- comfort food
- budget

## Comparison workflow
1. Search broadly for 3 to 6 candidate recipes.
2. Fetch the top candidates.
3. Compare for ingredient overlap, complexity, equipment requirements, and credibility.
4. Prefer the recipes that minimize missing ingredients while still sounding genuinely good.

## Practical ranking hints
Higher rank when the recipe:
- uses most of the user's available ingredients
- fits the user's available cookware and appliances
- has clear steps and realistic timing
- has obvious family-friendly or child-adjustable variations
- avoids techniques the user dislikes
