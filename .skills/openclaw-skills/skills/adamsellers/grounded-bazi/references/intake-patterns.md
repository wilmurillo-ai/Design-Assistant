# Intake patterns

Use this file to run a cleaner BaZi intake before giving the reading.

## Default intake

Ask for:
- date of birth
- time of birth if known
- place of birth
- focus

Then ask these two routing questions:
- "Do you want a reflective read or a more technical pillar-aware one?"
- "Do you want the short version or the full temple sermon?"

## If the request is underspecified

Use a compact prompt like:

- DOB:
- Time:
- Place:
- Focus:
- Mode: reflective / technical
- Length: short / full

## Focus menu

Offer this only if the user has not already been clear:
- overall pattern
- current luck cycle
- career
- money
- career + money
- relationships
- health / energy
- timing for a decision
- yearly theme

## Follow-up questions by focus

### Current luck cycle
Ask optionally:
- "Are you asking because life feels slow, volatile, or like it is reorganising itself?"
- "Do you want the read aimed more at work, money, or relationships?"

### Career
Ask optionally:
- "Are you deciding whether to stay, build, lead, or leave?"
- "Do you want strengths, risks, or timing emphasis?"

### Money
Ask optionally:
- "Do you want this framed as earning style, capital management, or current money timing?"

### Relationships
Ask optionally:
- "Do you want attraction patterns, trust patterns, or current relationship timing?"

## If birth time is unknown

Say:
- "That’s workable, but the reading gets lighter and less precise without the hour pillar."

Then continue unless the user wants strict technical detail.

## If the user wants high precision

Say:
- "I can do a strong interpretive read now, but for exact pillar-level claims I’d want a proper BaZi calculator."

## Combined reading upsell

When appropriate, offer:
- "If you want, I can pair this with a feng shui read of your room, desk, or house setup so the personal pattern and environment pattern line up."

## Output selection rule

- If the user is casual or playful, use short mode by default.
- If the user is asking about life direction, timing, career, or money, default to full mode.
- If the user asks for technical detail, use technical mode with explicit uncertainty.
