# Technical / pillar-aware mode

Use this file when the user wants a more technical BaZi reading or asks for Four Pillars detail.

## Core truthfulness rule

If you do not have a trustworthy chart calculator or precomputed chart, do not invent:
- heavenly stems
- earthly branches
- day master
- ten gods
- hidden stems
- exact element percentages
- ten-year luck pillar start ages
- exact annual pillar interactions

A technical tone is fine. Fake chart facts are not.

## What to say up front

Use one of these honesty lines near the top:
- "I can give you a pillar-aware reading, but without a proper Chinese calendar calculator I won’t pretend I’ve computed the exact chart."
- "I can explain the technical BaZi frame and give an interpretive read, but I’m not going to fake exact stems and branches."
- "For a strict Four Pillars audit, I’d want a proper calculator first."

## How to structure a pillar-aware reading without exact chart math

1. State the limitation.
2. Explain what a formal reading would inspect.
3. Walk through those categories as interpretive themes.
4. Keep uncertainty attached to anything that would normally depend on precise chart data.
5. End with practical takeaways.

## Categories to cover

### Year pillar
Use for:
- family imprint
- outer persona
- social inheritance
- broad generational atmosphere

### Month pillar
Use for:
- career environment
- conditioning
- responsibility patterns
- how the person meets worldly structure

### Day pillar / day master
Use for:
- sense of self
- core temperament
- how the person naturally exerts force or receives pressure

### Hour pillar
Use for:
- inner life
- legacy
- private ambitions
- later-life expression

### Element balance
Do not invent exact counts. Instead discuss:
- excess vs deficiency themes
- whether the person seems to need more structure, more warmth, more flexibility, more discipline, more movement, more grounding
- how imbalance might show up psychologically or behaviorally

### Luck pillars
Without exact start-age math, describe them as decade-scale pressure environments.
Talk about:
- consolidation vs expansion
- authority vs experimentation
- relationship focus vs work focus
- rebuilding vs harvesting

## Technical vocabulary

Use technical terms sparingly and explain them briefly:
- Day Master: the core self-reference point in BaZi
- Ten Gods: relationship patterns around power, output, wealth, resource, peers
- Useful element: the balancing influence a chart tends to need
- Luck pillar: a decade-scale environmental cycle

## Good pattern

- technical enough to satisfy curiosity
- plain enough to be readable
- explicit enough about uncertainty that the user is not misled

## Bad pattern

- sounding certain about exact chart mechanics you did not calculate
- spraying jargon to sound mystical
- pretending a rough reading is a mathematically derived one

## Suggested closing line

"If you want, next I can either keep this interpretive, or I can help generate the exact chart inputs we’d want for a stricter pillar-by-pillar reading with a proper calculator."
