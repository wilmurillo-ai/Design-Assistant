在构建 AI Agent 的征途中，我们常陷入两个极端：要么让昂贵的大模型处理一切琐碎，如杀鸡用牛刀；要么用生硬的规则禁锢了它的灵性。

**Semantic Router (语义路由)** 的诞生，旨在打破这一困局。它不仅仅是一个分流器，更是一套关于“思考快与慢”的决策哲学。

## 🎭 核心哲学：思考的快与慢

受丹尼尔·卡尼曼（Daniel Kahneman）心理学巨著《思考，快与慢》的启发，我们将 Agent 的思维模式解构为三个维度：

### 1. 系统 1：直觉与速度 (Highspeed Pool)
当用户问“今天天气如何”或“查询汇率”时，这不需要深思熟虑。我们的 **【高速池】** 如同人类的直觉，毫秒级响应，轻盈、高效、低成本。它负责处理 60% 的日常琐事，让交互如行云流水。

### 2. 系统 2：理性与逻辑 (Intelligence Pool)
当面对“帮我重构这段代码”或“设计一个系统架构”的请求时，大脑必须慢下来。**【智能池】** 此时接管控制权，调用最深邃的模型（如 Opus/GPT-4），虽然缓慢，但每一步都精准无误。这是 Agent 的理性光辉。

### 3. 感性与创造 (Humanities Pool)
除了理性的两极，我们还保留了一方人文的净土。**【人文池】** 专注于内容创作、情感陪伴与文学翻译。在这里，即使是冰冷的机器，也能写出有温度的文字。

## 🛡️ 静默的艺术：系统穿透

在喧嚣的数字世界中，懂得“沉默”同样重要。

Semantic Router 引入了 **System Passthrough (系统穿透)** 机制。对于 Heartbeat（心跳）、Cron（定时任务）或 Slash Command（斜杠命令），路由系统选择“视而不见”。它不打断、不干扰、不消耗宝贵的上下文窗口。

这种“静默的艺术”，让后台任务在暗处静静流淌，而前台的对话依然专注如初。

## 🌟 为什么它与众不同？

*   **不仅仅是关键词**：它拥有向量（Embedding）的眼睛，能读懂“继续刚才的话题”背后的千言万语。
*   **不仅仅是切换**：它懂得“连贯性”。当意图模糊时，它会温柔地延续当前的对话流，而不是生硬地打断。
*   **不仅仅是工具**：它是 Agent 的前额叶皮层，负责分配最宝贵的资源——注意力。
