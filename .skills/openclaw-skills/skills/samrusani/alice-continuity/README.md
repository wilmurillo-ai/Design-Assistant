# Alice Continuity for OpenClaw

**Alice is a local-first memory and continuity engine for AI agents.**

This skill is for OpenClaw users who want stronger long-term memory, cleaner resumption, correction-aware recall, open-loop tracking, and provenance-backed context.

## What Alice adds

With Alice, an OpenClaw workflow can gain:

- better recall across sessions
- resumption briefs for interrupted work
- correction-aware memory
- open-loop tracking
- provenance-backed explanations
- trust-aware memory promotion

Instead of treating memory as a pile of chat history, Alice normalizes imported or connected workflow data into structured continuity objects so your agent can answer higher-value questions like:

- What did we decide?
- What changed?
- What am I waiting on?
- What should happen next?

## Use cases

Alice is useful if you want OpenClaw to have:

- stronger long-term memory
- cleaner work resumption
- explicit correction and supersession
- more transparent provenance
- continuity that survives across days and weeks of work

## Integration paths

Alice currently supports:

- importing existing OpenClaw memory into Alice
- normalizing imported data into Alice continuity objects
- running recall and resumption against imported work
- layering Alice continuity workflows on top of an existing OpenClaw setup
