# 端口扫描脚本使用指南

## port_scan.py

### 功能说明
扫描目标主机的开放端口，识别潜在风险服务。

### 使用方法

```bash
# 扫描本机
python scripts/port_scan.py localhost

# 扫描指定主机
python scripts/port_scan.py 192.168.1.1

# 指定端口范围
python scripts/port_scan.py 192.168.1.1 --start 1 --end 1000

# 快速扫描（仅高危端口）
python scripts/port_scan.py 192.168.1.1 --fast
```

2. 数据库仅允许本地访问
3. 使用强密码或密钥认证

## 常见端口参考

| 端口 | 服务 | 风险 | 建议 |
|------|------|------|------|
| 21 | FTP | 高 | 关闭，使用 SFTP |
| 22 | SSH | 中 | 禁用密码，使用密钥 |
| 23 | Telnet | 极高 | 立即关闭 |
| 25 | SMTP | 中 | 限制中继 |
| 80 | HTTP | 低 | 改用 HTTPS |
| 443 | HTTPS | 低 | 启用 HSTS |
| 3389 | RDP | 高 | 仅内网，启用 NLA |
| 3306 | MySQL | 高 | 绑定本地，禁用远程 |
| 5432 | PostgreSQL | 高 | 绑定本地，禁用远程 |
| 6379 | Redis | 极高 | 绑定本地，设置密码 |
| 27017 | MongoDB | 高 | 绑定本地，设置密码 |

## 风险评估规则

- 开放数据库端口（3306, 5432, 6379, 27017）→ 🔴 高危
- 开放远程管理端口（22, 3389）→ 🟡 中危
- 开放 Telnet(23) → 🔴 极高危
- 开放未加密服务（21, 23）→ 🔴 高危
