# 横纵分析法 Deep Research Skill

## 方法论溯源

横纵分析法由数字生命卡兹克（Khazix）提出，融合了：
- **索绪尔的历时-共时分析**（语言学）
- **社会科学纵向-横截面研究**（社会科学）
- **商学院案例研究法**
- **竞争战略分析**

核心原则：**纵向追时间深度，横向追同期广度，最终交汇出判断。**

---

## 双轴框架

### 纵向分析（Diachronic / Longitudinal）

沿时间轴，完整还原研究对象从诞生到现在的发展全貌。

**五个子维度：**
1. **起源追溯** — 诞生的背景、技术/理念/需求来源、创始团队、当时的行业环境
2. **诞生节点** — 明确首次发布/成立时间，最初形态和定位
3. **演进历程** — 按时间顺序梳理所有关键节点（版本更新、融资、团队变动、战略转向、技术架构变化、里程碑、危机）
4. **决策逻辑** — 每个关键节点上还原"为什么选了A而不是B"，当时的约束条件
5. **阶段划分** — 分为萌芽期、快速增长期、转型期等，每个阶段有核心特征和核心矛盾

**叙事要求：** 不要写成干巴巴的年表。要用故事的方式串起来，让读者感受到因果关系和时代脉络。

**字数：** 6000-15000字（历史越长越靠近上限）

---

### 横向分析（Synchronic / Cross-sectional）

以当前时间点为切面，与同赛道竞品/同类进行全面对比。

**竞品场景判断：**
- **场景A（无直接竞品）** — 全新品类，分析为什么没有竞品、未来最可能从哪个方向冒出竞争者、间接替代方案
- **场景B（1-2个竞品）** — 逐一深入详细分析
- **场景C（3个及以上竞品）** — 选取3-5个最具代表性的进行对比，其余简要提及

**四个对比维度：**
1. **核心差异** — 技术路线/方法论、产品形态/商业模式、目标用户、核心优势与短板、定价策略
2. **用户视角** — 真实口碑、社区评价、用户选择它的真实理由
3. **生态位** — 在整个赛道版图中占据什么位置、填补了什么空白
4. **趋势判断** — 在竞争格局中的走向、机会和风险

**字数：** 3000-10000字（视竞品数量调整）

---

### 横纵交汇洞察

这是整篇报告的精华段。

**回答的核心问题：**
- 历史如何塑造了当下的竞争位置？
- 竞品的起源和演变路径有什么不同？如何导致了今天各自的特点？
- 今天的每个核心优势，能追溯到历史上的哪个节点或决策？
- 今天的每个核心劣势，能追溯到哪个历史决策？当初的"好决策"有没有变成今天的包袱？
- **未来推演三个剧本：** 最可能的、最危险的、最乐观的，每个要有逻辑支撑

**字数：** 1500-3000字

## 写作风格

**核心：** 不是冷冰冰的咨询报告，是让人能从头读到尾的深度特稿。

**具体要求：**
- 叙事驱动，不是罗列驱动，有故事弧线、起承转合
- 句子时长时短，段落之间跳跃自然，制造节奏感
- 观点要有，但必须建立在事实之上，先摆事实再给判断
- 避免：套话（"首先...其次..."）、空洞形容词（"赋能""抓手""打造闭环"）
- 对比要有温度，讲清楚每个竞品"活成了什么样"
- 结尾前后回环呼应，把开头埋的钩子收回来

**字数分配参考：**
研究产品 → 纵轴重点版本迭代/技术路线/用户增长；横轴重点功能对比/用户体验
研究公司 → 纵轴重点创始团队/融资历程/战略转向；横轴重点商业模式/市场份额
研究概念 → 纵轴重点起源/如何流行/争论演变；横轴重点与相近概念区别
研究人物 → 纵轴重点职业轨迹/关键决策/成长曲线；横轴重点同领域人物对比

## 执行流程

### 1. 确认研究信息
- 研究对象：具体的产品名/公司名/概念名/人名
- 类型：产品、公司、概念、人物、还是其他？
- 研究动机（可选）：为什么要研究它？最近发生了什么？
- 特别关注点（可选）：有没有特别想深入的方向？

### 2. 并行子 Agent 信息收集

**子Agent 1 — 纵向信息：**
- 研究对象的起源、创始人背景
- 发展历程、关键事件
- 版本迭代、融资、战略转向、危机

**子Agent 2 — 横向信息：**
- 竞品识别
- 各竞品的特点和用户口碑
- 行业对比评测、市场份额

**子Agent 3（复杂对象才需要）：**
- 创始人深度背景
- 行业环境变化
- 用户社区讨论（GitHub issues、Reddit、Twitter/X、知乎等）

**信息来源优先级：**
一手来源优于二手来源
- 产品更新/技术决策 → 官方博客、GitHub Release Notes、创始人推文
- 融资/商业数据 → 公司官方公告、SEC/工商文件
- 用户口碑 → GitHub Issues、Reddit讨论、Twitter/X、知乎帖子
- 学术/技术原理 → arXiv论文、Google Scholar

### 3. 自检清单

- [ ] 纵轴是叙事故事体？读起来有因果逻辑和时代脉络？不是年表流水账？
- [ ] 创始人/发起者的背景和动机有足够深度？
- [ ] 每个关键节点都展开了，没有为压缩跳过重要细节？
- [ ] 决策逻辑有还原？不只是"发生了什么"，还有"为什么这么选"？
- [ ] 横轴的竞品场景判断正确？竞品分析够深？
- [ ] 用户口碑引用了真实用户声音？不只是官方宣传？
- [ ] 横纵交汇产出新判断，不是前面内容缩写版？
- [ ] 未来推演三剧本都有逻辑支撑？
- [ ] 写作风格有节奏感、有可读性？
- [ ] 搜不到的信息诚实标注"暂缺"，没有编造？
- [ ] 总字数在10000-30000字范围内？

### 4. PDF 报告生成（如需）

使用 `scripts/md_to_pdf.py` 脚本：
```bash
pip install weasyprint markdown --break-system-packages
python scripts/md_to_pdf.py input.md output.pdf --title "研究对象名称" --author "横纵分析法"
```

## Prompt 版（轻量级，直接复制使用）

见同目录下的 `横纵分析法-Prompt.md`，把第一行的"研究对象 =" 改成你想要研究的目标即可。适用于：ChatGPT DeepResearch、Claude深度研究、豆包专家模式、DeepSeek专家模式等。

## Skill 版 vs Prompt 版

| | Prompt版 | Skill版 |
|--|---------|---------|
| 复杂度 | 轻量，直接复制使用 | 重量级，自动联网搜索+arxiv论文+PDF生成 |
| 使用方式 | 复制到AI对话里 | 装入Agent，直接说"帮我研究一下XXX" |
| 输出 | AI直接输出报告 | 自动调用工具搜集信息，最终生成排版精美的PDF |
| 适用场景 | 临时研究、单一报告 | 深度持续研究、体系化沉淀 |
