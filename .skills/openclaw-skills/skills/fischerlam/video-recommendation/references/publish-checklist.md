# Publish Checklist

Use this before publishing the skill to ClawHub.

## Positioning

- Is the one-line promise clear and memorable?
- Does the description distinguish the skill from generic recommendation tools?
- Is the anti-addiction / context-aware angle obvious enough?

## Trigger quality

- Will the description trigger on realistic user requests?
- Does it cover both casual and project-linked asks?
- Does it avoid being so broad that it triggers unnecessarily?

## Skill body quality

- Is SKILL.md concise enough?
- Is the workflow clear?
- Does each reference file have a distinct purpose?
- Is there duplicated guidance that should be merged or cut?

## Reference quality

- Do examples feel realistic?
- Does the scoring rubric actually improve quality?
- Does source curation make the skill more opinionated?
- Are taste profiles useful rather than gimmicky?

## Practicality

- Can the skill produce direct links, not just search suggestions?
- Does it still work when web search is blocked?
- Is the helper script simple and justified?

## Product quality

- Does this skill feel like a real capability someone would install?
- Is the value obvious from the name and description?
- Would a user know when to use it?

## Before publishing

- Remove anything that feels repetitive
- Tighten wording where possible
- Confirm file structure is clean
- Test with at least 3 realistic prompts
