# AI 产品常见模式

---

## 1. AIGC 产品框架

**核心价值链：**
```
用户输入（Prompt）→ 内容生成（模型）→ 质量过滤 → 用户消费 → 反馈迭代
```

**产品设计要点：**
- Prompt 设计：引导 vs 自由输入
- 生成质量控制：安全过滤、质量评分
- 用户预期管理：生成时间、失败处理
- 版权与合规

**差异化维度：**
- 垂直领域专精（vs 通用）
- 工作流集成深度
- 风格/品牌一致性

---

## 2. 推荐产品框架

**核心价值主张：** 信息过滤 + 个性化匹配

**产品架构：**
```
内容供给 → 用户理解 → 匹配策略 → 结果呈现 → 效果反馈
```

**关键设计决策：**
- 推荐密度：多少比例是个性化 vs 编辑推荐
- 探索与利用平衡（Exploration vs Exploitation）
- 用户控制权（可解释性、兴趣调整）
- 信息茧房风险

---

## 3. 搜索产品框架

**核心价值主张：** 精准意图满足

**产品架构：**
```
查询理解 → 召回 → 排序 → 结果展示
```

**关键设计决策：**
- 搜索 vs 推荐 的边界（主动 vs 被动）
- 结构化结果 vs 生成式答案（RAG）
- 搜索词联想与纠错体验

---

## 4. 对话/智能助手框架

**核心价值主张：** 自然语言交互完成任务

**产品架构：**
```
意图理解 → 实体抽取 → 对话管理 → 动作执行 → 回复生成
```

**关键设计决策：**
- 单轮 vs 多轮对话
- 任务型 vs 闲聊型 vs 问答型
- 失败处理：低置信度时澄清 vs 猜测执行
- 人工兜底机制

---

## 5. 数据产品框架

**核心价值主张：** 数据洞察驱动决策

**产品架构：**
```
数据采集 → 数据处理 → 指标体系 → 可视化 → 洞察/预警
```

**关键设计决策：**
- 自助分析 vs 预置报表
- 实时 vs T+1
- 数据权限与隐私

---

## AI 产品通用设计原则

1. **渐进式自动化** — 先辅助人，再逐步替代人工步骤
2. **可解释性** — 用户能理解 AI 为什么这样做
3. **优雅降级** — AI 失效时有合理的非 AI 兜底
4. **数据飞轮** — 产品设计要促进数据积累，形成正循环
5. **人工标注入口** — 便于收集训练数据，持续改进模型
