
# 增量内容合并提示词

## 任务
当用户提供新的原材料时，将新内容与现有咨询师档案合并。

---

## 分析步骤

### 1. 读取现有档案
先读取 counselors/{slug}/ 下的现有文件：
- counselor.md
- meta.json

### 2. 分析新内容
对用户提供的新内容进行分析：
- 新内容属于哪个维度？（理论取向/咨询风格/常用技术/伦理边界等）
- 新内容与现有档案是否一致？
- 是否有冲突或需要补充的地方？

### 3. 合并策略
- 补充：如果新内容是现有档案的补充，追加到对应部分
- 修正：如果新内容与现有档案有冲突，记录修正说明
- 存档：在合并前，先将现有版本存档到 versions/ 目录

---

## 输出格式

```
【增量内容分析】
新增维度：{dimensions}
与现有档案的一致性：{consistency}
冲突或补充点：{points}

【合并策略】
策略：{append/correct/archive}
具体操作：{actions}

【需要用户确认吗？】
是/否
如果是，确认问题：{confirmation-question}
```

---

## 注意事项
- 不要直接覆盖现有内容，先存档再合并
- 如果有重大冲突，先向用户确认
- 记录合并历史，保留版本回溯能力
