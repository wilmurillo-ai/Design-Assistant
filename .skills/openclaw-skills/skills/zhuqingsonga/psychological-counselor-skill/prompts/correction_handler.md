
# 对话纠正处理提示词

## 任务
当用户表达"不对"、"这位咨询师不会这样"、"应该是"时，识别纠正内容并处理。

---

## 分析步骤

### 1. 识别纠正内容
判断用户的纠正属于哪个维度：
- 风格模拟（这位咨询师不会这么说）
- 专业学习（这位咨询师的理论取向不是这样）
- 倾听陪伴（这个回应不对）

### 2. 理解正确的内容
从用户的表达中提取正确的内容：
- 用户认为应该是什么样的？
- 用户提供了什么新信息？
- 用户希望如何调整？

### 3. 生成纠正记录
生成结构化的纠正记录，包括：
- 纠正时间
- 纠正类型
- 原始内容
- 正确内容
- 纠正原因

---

## 输出格式

```
【纠正分析】
纠正类型：{style/professional/listening}
原始内容：{original-content}
正确内容：{corrected-content}
纠正原因：{reason}

【处理策略】
策略：{append/correct/revise}
具体操作：{actions}

【需要用户确认吗？】
是/否
如果是，确认问题：{confirmation-question}
```

---

## 注意事项
- 认真倾听用户的纠正，不要辩解
- 将纠正内容记录在案，作为后续迭代的依据
- 如果用户纠正了多个方面，分别记录和处理
- 保持开放态度，将纠正视为改进的机会
