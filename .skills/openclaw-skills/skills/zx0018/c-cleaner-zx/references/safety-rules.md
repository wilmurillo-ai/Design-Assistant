# 🔐 C 盘数据安全规则

**本 Skill 的核心安全原则 - 必须严格遵守**

_源自 SOUL.md 临时原则（2026-04-14），现已整合到 Skill_ 

---

## ⚠️ 核心原则

### 1. 只扫描和汇报
- ✅ 可以主动执行扫描、分析、汇报
- ❌ 不能主动执行任何删除、移动、清理操作

### 2. 删除前必须询问
- 任何删除操作都必须得到用户**明确确认**
- 确认格式：用户回复"确认删除"或类似明确指令
- 模糊回复（如"好的"、"行"）不足以作为确认

### 3. 白名单保护
以下目录**绝对不能删除或移动**：
- `C:\Windows\System32` - 系统核心
- `C:\Windows\SysWOW64` - 系统核心
- `C:\Program Files` - 程序安装目录
- `C:\Program Files (x86)` - 程序安装目录
- `C:\Users\*\Documents` - 用户文档
- `C:\Users\*\Desktop` - 桌面文件
- `C:\Users\*\Pictures` - 图片文件
- `C:\Users\*\Videos` - 视频文件

### 4. 迁移需确认
- 迁移前必须确认目标位置
- 迁移完成后询问是否删除原文件
- 不能默认删除原文件

### 5. 操作留痕迹
- 所有操作必须记录日志
- 日志包括：操作类型、文件列表、时间戳、释放空间

## ❌ 禁止行为

### 文件操作风险防护

**❌ 绝不删除的文件类型：**

| 路径 | 说明 |
|------|------|
| `C:\Windows\*` | 系统文件，动不得（除 Temp 外） |
| `C:\Program Files\*` | 程序文件，动不得 |
| `C:\Users\*\Documents\*` | 个人文档，动不得 |
| `C:\Users\*\Desktop\*` | 桌面文件，动不得 |
| `C:\Users\*\Pictures\*` | 照片，动不得 |
| `C:\Users\*\Videos\*` | 视频文件，动不得 |
| `*.docx, *.xlsx, *.pptx` | Office 文档，动不得 |
| `*.pdf, *.jpg, *.png` | 个人文件，动不得 |

**✅ 只清理这些安全目标：**

| 目标 | 说明 | 风险 |
|------|------|------|
| `C:\Windows\Temp\*` | 系统临时文件 | ✅ 零风险 |
| `C:\Users\*\AppData\Local\Temp\*` | 用户临时文件 | ✅ 零风险 |
| `$Recycle.Bin\*` | 回收站 | ✅ 零风险 |
| `C:\Windows\SoftwareDistribution\Download\*` | Windows 更新缓存 | ⚠️ 需确认 |

### 其他禁止行为

1. ❌ 未经确认删除任何文件
2. ❌ 删除白名单目录中的文件
3. ❌ 默认删除迁移后的原文件
4. ❌ 隐藏或跳过确认步骤
5. ❌ 不记录操作日志

## ✅ 推荐行为

### 可审计性保障

**每次操作必须：**
1. **命令预审** - 执行前完整展示命令，老师确认后再执行
2. **分步执行** - 一步一确认，不批量执行
3. **只读优先** - 先用只读命令分析，确认后再删除
4. **完整展示** - 每条命令都完整显示，不隐藏参数
5. **实时汇报** - 执行中实时汇报进度
6. **可中断** - 老师说停就立即停止

**推荐做法：老师手动执行**
```
Roxy 给出命令 → 老师在 Windows 上执行 → 结果告诉 Roxy
```
这样 Roxy 只有建议权，没有执行权，最安全！

### 其他推荐行为

1. ✅ 扫描前告知用户要做什么
2. ✅ 汇报时使用清晰的表格格式
3. ✅ 确认提示包含完整信息（操作类型、影响范围、风险提示）
4. ✅ 操作后记录详细日志
5. ✅ 会话结束时总结操作结果
