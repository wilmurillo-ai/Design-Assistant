## 📖 技能说明

Skill Composer 是一款智能技能编排工具，通过任务分解、技能匹配、流程编排、执行优化，帮助用户自动编排多技能协作流程，最大化执行效率。

**核心价值**:
- 自动化多技能协作
- 智能执行策略优化
- 技能依赖关系分析
- 结果自动合成

## 📋 核心功能

### 1. 任务分解（第一性原理）

**分解流程**:
```
复杂任务
    ↓
识别子任务（5Why 分析）
    ↓
定义子任务依赖关系
    ↓
生成任务 DAG（有向无环图）
```

**分解示例**:
```
输入：「分析技术架构问题并生成报告」

输出：
1. 问题分析（first-principle-analyzer）
   └─ 依赖：无
2. 技能编排（meta-skill-weaver）
   └─ 依赖：1
3. 报告撰写（copywriting）
   └─ 依赖：1, 2
4. 演示文稿（pptx）
   └─ 依赖：3
```

### 2. 技能匹配（能力 - 任务匹配）

**匹配算法**:
```javascript
匹配分数 = 能力匹配度×0.4 + 技能评分×0.3 + 执行效率×0.3

能力匹配度：技能能力描述 vs 任务需求
技能评分：六维评估综合得分
执行效率：历史执行时间/资源消耗
```

**匹配示例**:
```
任务：深度分析问题

匹配结果:
1. first-principle-analyzer (0.92)
   - 能力匹配：0.95
   - 技能评分：0.71
   - 执行效率：0.85

2. meta-cognition-assistant (0.78)
   - 能力匹配：0.80
   - 技能评分：0.65
   - 执行效率：0.75
```

### 3. 流程编排（并行/串行决策）

**决策规则**:
```
if (子任务间无依赖) {
  执行策略 = 并行执行
} else if (子任务间有依赖) {
  执行策略 = 串行执行
} else if (部分依赖) {
  执行策略 = 混合执行（关键路径串行，其他并行）
}
```

**编排示例**:
```
任务 DAG:
A → B → D
A → C → D

执行策略:
1. 执行 A（串行）
2. 并行执行 B 和 C
3. 执行 D（等待 B、C 完成）

预计时间：A + max(B,C) + D
相比全串行：节省 (B+C) - max(B,C) 时间
```

### 4. 执行优化（资源调度）

**优化目标**:
- 最小化执行时间
- 最小化资源消耗
- 最大化执行质量

**优化策略**:
```
1. 关键路径分析
   - 识别影响总工期的关键任务
   - 优先保障关键任务资源

2. 资源平衡
   - 避免资源争抢
   - 均衡负载分布

3. 动态调整
   - 监控执行进度
   - 动态调整资源分配
```

### 5. 结果合成（多技能结果聚合）

**合成策略**:
```
1. 结果收集
   - 收集所有技能输出
   - 验证结果完整性

2. 结果整合
   - 合并相似内容
   - 解决冲突（如有）

3. 结果优化
   - 格式统一
   - 逻辑连贯性检查

4. 结果交付
   - 生成最终报告
   - 交付给用户
```

## 💡 使用技巧

### 技巧 1：明确任务目标

描述任务时明确目标和约束：
```
✅ 好：「分析技术架构问题并生成报告，2 小时内完成」
❌ 差：「帮我做分析」
```

### 技巧 2：提供技能偏好

如有偏好技能，提前说明：
```
✅ 好：「优先使用 first-principle-analyzer 进行分析」
❌ 差：「随便用技能」
```

### 技巧 3：定义成功标准

明确定义任务完成标准：
```
✅ 好：「报告需要包含问题分析、解决方案、实施计划」
❌ 差：「生成报告就行」
```

## ❓ FAQ

### Q1: 如何编排多技能协作？

**A**: 描述复杂任务，技能编排师会自动分解任务、匹配技能、编排流程。

### Q2: 并行执行有什么好处？

**A**: 并行执行可以显著减少总执行时间，特别是对于独立子任务。

### Q3: 如何处理技能执行失败？

**A**: 技能编排师会检测执行失败，自动重试或切换到备用技能。

### Q4: 结果合成会丢失信息吗？

**A**: 不会，结果合成会保留所有关键信息，并解决可能的冲突。
