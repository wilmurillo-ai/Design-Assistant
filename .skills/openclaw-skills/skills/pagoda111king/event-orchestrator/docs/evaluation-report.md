## 应用的设计模式

### 1. 事件驱动架构 (EDA)
**来源**: 2026-04-09 23:00 小时学习

**应用内容**:
- EventBus 作为核心通信机制
- 事件生产者与消费者完全解耦
- 事件用过去分词命名（`skill.completed`, `task.failed`）
- 支持事件订阅、发布、过滤
- 事件历史记录支持审计

**代码体现**:
- `EventBus` 类：核心事件总线（event-bus.js）
- `EventValidator` 类：事件 Schema 验证
- 订阅者管理：优先级排序、过滤器支持

### 2. Middleware 链模式
**来源**: 2026-04-09 13:00 小时学习（DeerFlow 架构）

**应用内容**:
- 事件处理支持中间件链
- 每个中间件可修改事件、跳过后续处理、记录日志
- 支持中间件注册、执行顺序控制

**代码体现**:
- `LoggingMiddleware`: 日志记录
- `ValidationMiddleware`: 事件验证
- `RetryMiddleware`: 自动重试
- `RateLimitMiddleware`: 速率限制
- `MiddlewareChainExecutor`: 中间件链执行器

### 3. 状态机模式
**来源**: DeerFlow 架构学习

**应用内容**:
- 编排任务有明确状态（pending, running, completed, failed）
- 状态转换有严格规则（STATE_TRANSITIONS）
- 支持状态持久化和恢复（serialize/deserialize）
- 状态历史追溯

**代码体现**:
- `OrchestrationState` 枚举：状态定义
- `STATE_TRANSITIONS`：转换规则
- `StateMachine` 类：状态机核心
- `OrchestrationTask` 类：任务封装
