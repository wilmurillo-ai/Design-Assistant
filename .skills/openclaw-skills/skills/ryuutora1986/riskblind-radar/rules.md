# 风险盲区雷达 — 认知偏差与机会识别

## 一、认知偏差参考表（辅助，不作为主触发）

| 偏差类型 | 信号 | 应对 |
|---------|------|------|
| 规划谬误 | 工期估计远短于实际 | 对比历史预测准确率 |
| 沉没成本 | "已经投了这么多..." | 追问退出条件 |
| 从众效应 | "别人都在做" | 要求差异化说明 |
| 过度自信 | "肯定没问题" | 要求列出潜在风险 |
| 确认偏误 | 只听支持自己观点的反馈 | 主动提供反方意见 |

**使用注意：** 偏差检测只作为辅助参考，不作为主动触发条件。

## 二、机会识别模式库

当用户抱怨但没有转化为行动时，触发机会信号：

| 抱怨模式 | 机会可能性 | 行动建议 |
|---------|-----------|----------|
| "每次都..." | 高频痛点 | 问：有没有可能做成产品？ |
| "要是能...就好了" | 显性需求 | 问：有人为此付过钱吗？ |
| "太贵了" | 价格敏感点 | 问：降到多少你会买？ |
| "太麻烦了" | 摩擦点 | 问：简化哪一步用户会翻倍？ |
| "没有人做..." | 市场空白 | 问：为什么没人做？是真的吗？ |

## 三、高价值风险疑问速查表（快速触发）

```
用户说...              →  自动追问...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
投钱                   →  ROI公式是什么？
上线                   →  成功的定义是什么？
计划做X                →  X之后？然后呢？
先...再说              →  后置条件是什么？
已经投了...            →  退出条件是什么？
用户会...              →  验证过吗？
没有AI就...            →  真的壁垒是什么？
同时做两件事           →  哪个是主哪个是次？
说"麻烦"但没解决       →  遇到多少次了？能变现吗？
做出承诺               →  验收标准是什么？
```

## 四、二阶思维追问路径

```
Level 1：用户说"我要做X"
    ↓
Level 2："X之后会发生什么？"
    ↓
Level 3："然后呢？"
    ↓
Level 4："整个链条里最脆弱的环节是什么？"
    ↓
Level 5："这个脆弱环节能提前加固吗？还是整个计划都有风险？"
```

使用原则：Level 1 → Level 2 常规触发，Level 3-5 按需追问，不在单次对话中全部走完。

## 五、风险雷达成长机制

每次对话结束后自问：

1. 我今天问出了什么好问题？
2. 这个问题帮用户想到了什么他没想到的风险？
3. 下次如何问得更好？

这个不进日志，只作为系统自己的学习记录。
