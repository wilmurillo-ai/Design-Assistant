# Word中文格式标准化使用指南

## 1. 安装依赖

在使用本技能之前，需要安装必要的Python依赖包：

```bash
pip install -r scripts/requirements.txt
```

这将安装 `python-docx` 库，用于处理Word文档。

## 2. 运行格式化脚本

### 2.1 基本用法

```bash
python scripts/format_word.py <input_file> [output_file]
```

参数说明：
- `<input_file>`：必需，要格式化的Word文档路径
- `[output_file]`：可选，格式化后的文档保存路径。如果不指定，默认在原文件基础上加"_formatted"后缀

### 2.2 示例

#### 示例1：格式化文档并保存为新文件

```bash
python scripts/format_word.py document.docx formatted_document.docx
```

这将把 `document.docx` 按照中文格式标准进行格式化，并保存为 `formatted_document.docx`。

#### 示例2：直接在原文件基础上格式化

```bash
python scripts/format_word.py report.docx
```

这将把 `report.docx` 按照中文格式标准进行格式化，并保存为 `report_formatted.docx`。

## 3. 格式化效果

执行脚本后，文档将按照以下标准进行格式化：

### 3.1 字体
- 统一使用微软雅黑字体
- 标题和各级标题使用相应的字体大小和加粗效果
- 正文使用5号字体

### 3.2 段落
- 正文使用1.5倍行距
- 首行缩进2字符
- 段前和段后间距按照标准设置

### 3.3 图片和图表
- 图片居中显示
- 图片下方添加编号和说明
- 图表标题居中显示

### 3.4 表格
- 表格居中显示
- 表格内容使用统一格式

## 4. 常见问题

### 4.1 脚本运行失败

**问题**：运行脚本时出现错误。

**解决方法**：
- 确保已安装 `python-docx` 库
- 确保输入文件是有效的Word文档（.docx格式）
- 确保文件路径正确

### 4.2 格式化效果不理想

**问题**：格式化后的文档效果不符合预期。

**解决方法**：
- 检查文档是否使用了特殊格式或样式
- 对于复杂的文档，可能需要手动调整某些部分
- 确保文档中的标题使用了正确的标题样式

### 4.3 图片编号问题

**问题**：图片编号不正确或重复。

**解决方法**：
- 脚本会按照图片在文档中出现的顺序自动编号
- 如果需要自定义编号，可以在格式化后手动修改

## 5. 注意事项

1. **备份原文档**：在格式化之前，建议备份原文档，以防万一。

2. **兼容性**：脚本适用于 .docx 格式的文档，不支持 .doc 格式。

3. **特殊格式**：对于文档中的特殊格式（如脚注、尾注、交叉引用等），脚本可能无法完全处理，需要手动调整。

4. **性能**：对于大型文档，格式化可能需要较长时间，请耐心等待。

5. **字体安装**：确保系统中已安装微软雅黑字体，否则可能会导致字体显示异常。

## 6. 高级用法

### 6.1 批量处理

可以使用批处理脚本一次性处理多个文档：

```bash
# Windows批处理
for %%f in (*.docx) do python scripts/format_word.py "%%f"

# Linux/Mac Shell
for f in *.docx; do python scripts/format_word.py "$f"; done
```

### 6.2 集成到其他工作流

可以将本脚本集成到其他工作流中，例如：

- 作为文档处理管道的一部分
- 与版本控制系统配合使用
- 作为自动化办公工具的组件

## 7. 故障排除

如果遇到问题，可以查看脚本的错误信息，或者尝试以下方法：

1. **检查Python版本**：确保使用Python 3.6或更高版本。

2. **检查依赖版本**：确保 `python-docx` 库是最新版本。

3. **查看日志**：脚本会输出执行过程中的信息，可以根据这些信息定位问题。

4. **简化文档**：如果处理复杂文档时出现问题，可以先处理一个简化版本的文档，逐步排查问题。
