# Roadmap Planning Under Uncertainty

This framework helps AI PMs shift from feature-based roadmaps to outcome-oriented planning in an environment where AI capabilities change rapidly.

## When to use this framework
- Defining the product strategy for the next quarter or year.
- Communicating priorities to stakeholders when specific delivery dates are impossible to guarantee.
- Deciding whether to build, buy, or partner for AI capabilities.

## 1. Shift to Outcome-Oriented Planning
Instead of asking "What features will we build?", ask "What user outcomes do we want to achieve, and what user behaviors do we want to enable?"
- **Action:** Define the core problem and the desired change in user behavior.
- **Example:** Instead of "Build an AI chatbot," the outcome is "Reduce customer support resolution time by 30%."

## 2. The Now, Next, Later Framework
Use this model to communicate priorities without committing to rigid timelines.
- **Now:** High-confidence initiatives currently in development. Specific features and near-term delivery dates.
- **Next:** Medium-confidence initiatives planned for the near future. Focus on outcomes and broad solutions, not specific features.
- **Later:** Low-confidence, long-term strategic bets. Focus on high-level problems and exploratory research.
- **Template:** Use `templates/outcome_roadmap.md` to structure this.

## 3. The U.S.I.D.O. Lifecycle
Apply this structured lifecycle to AI product development:
- **Understand:** Define the problem, user needs, and business value.
- **Specify:** Determine the required data, model capabilities, and success metrics.
- **Implement:** Develop the model, build the prototype, and integrate with the product.
- **Deploy:** Launch the feature, monitor performance, and gather user feedback.
- **Optimize:** Continuously refine the model and the user experience based on data.

## 4. The Build / Buy / Bake Strategy
Evaluate how to acquire the necessary AI capabilities.
- **Build:** Develop the AI model in-house. Best for core competitive advantages and unique data assets. High cost and time.
- **Buy:** Purchase an off-the-shelf AI solution (e.g., an API). Best for common problems and fast time-to-market. Lower differentiation.
- **Bake (Partner):** Co-develop a solution with a specialized vendor. Balances speed and customization.
