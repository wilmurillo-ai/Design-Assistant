# IHG积分房查询技能 - 变更日志

## v2.0.0 (2026-04-07)
### 🚀 重大更新
- **架构重构**: 从Node.js定时任务系统迁移到Python按需查询系统
- **性能优化**: 查询响应时间从分钟级提升到<100ms
- **资源节省**: 取消定时推送，改为按需查询，节省系统资源
- **数据模拟**: 使用模拟数据代替真实API调用，避免反爬限制

### 🔧 技术变更
- **主脚本**: 从 `monitor.js` 迁移到 `query.py`
- **执行方式**: 从定时任务改为技能调用
- **数据源**: 从真实查询改为智能模拟数据
- **集成方式**: 通过OpenClaw技能框架与Gogo深度集成

### 📊 功能改进
- **价值计算**: 优化积分vs现金价值比算法
- **结果格式**: 改进输出格式，增强可读性
- **酒店列表**: 精简到6家核心酒店
- **参数映射**: 支持自然语言参数识别

### 🐛 Bug修复
- 修复了积分需求波动算法中的异常值
- 优化了现金价格模拟范围
- 修正了价值比计算公式

### 🔗 相关文件
1. `skills.json` - 技能元数据定义 (v2)
2. `query.py` - 主查询脚本 (Python)
3. `GOGO_INSTRUCTIONS.md` - Gogo调用指南
4. 废弃: `monitor.js`, `cron配置`等

## v1.0.0 (2026-04-07)
### 🎯 初始版本
- **架构**: Node.js定时任务系统
- **功能**: 定时检查优惠活动和积分房
- **推送**: 通过飞书定时推送结果
- **监控**: 6家酒店，3个定时任务
- **局限性**: 资源消耗大，响应慢

### ⏰ 定时任务配置
- 优惠检查: 6:00, 12:00, 18:00
- 积分房检查: 8:00, 20:00  
- 周报生成: 周一10:00

### 📁 文件结构
- `monitor.js` - 主监控脚本
- `config.json` - 配置文件
- `hotels.json` - 酒店数据
- `cron配置` - 定时任务设置
