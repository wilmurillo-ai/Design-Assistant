## 文件说明

### 地址模式参考

技能使用以下正则表达式模式识别地址组件：

- **省份模式**: `(北京|天津|河北|山西|内蒙古|辽宁|吉林|黑龙江|上海|江苏|浙江|安徽|福建|江西|山东|河南|湖北|湖南|广东|广西|海南|重庆|四川|贵州|云南|西藏|陕西|甘肃|青海|宁夏|新疆|台湾|香港|澳门)`
- **城市模式**: `([\u4e00-\u9fa5]{2,8}?(?:市|州|地区|盟|自治州))`
- **区县模式**: `([\u4e00-\u9fa5]{2,10}?(?:区|县|市|自治县|旗|自治旗))`
- **乡镇模式**: `([\u4e00-\u9fa5]{2,15}?(?:镇|乡|街道|苏木|民族乡))`
- **村庄模式**: `([\u4e00-\u9fa5]{2,20}?(?:村|社区|居委会|大队|组|巷|弄))`
- **道路模式**: `([\u4e00-\u9fa5\d]+?(?:路|街|大道|胡同|弄|巷|条|里))`
- **门牌模式**: `[\u4e00-\u9fa5]?[\d\-]+[\u4e00-\u9fa5]?号?`
- **楼栋模式**: `[\u4e00-\u9fa5]?[\d]+[\u4e00-\u9fa5]*(?:栋|号楼|幢|座|单元)`

### 高德地图API文档

**地理编码API**: `https://restapi.amap.com/v3/geocode/geo`

**请求参数**:
- `address`: 结构化地址信息
- `key`: 高德Key
- `city`: 指定查询的城市
- `output`: 返回数据格式
- `batch`: 批量查询控制

**响应格式**:
```json
{
  "status": "1",
  "info": "OK", 
  "infocode": "10000",
  "count": "1",
  "geocodes": [{
    "formatted_address": "完整地址",
    "province": [],
    "city": [], 
    "district": [],
    "township": [],
    "neighborhood": [],
    "building": [],
    "adcode": [],
    "street": [],
    "number": [],
    "location": "经度,纬度",
    "level": "匹配级别"
  }]
}
```

### 地址标准化层级

标准地址路径格式：
```
省-市-区/县-乡镇/街道-村庄/社区-道路号POI(楼栋号)
```

示例：
- `北京市-朝阳区-建国门外大街-1号-国贸大厦(A座1208室)`
- `上海市-浦东新区-陆家嘴环路-1000号-恒生银行大厦(25楼)`
- `广东省-深圳市-南山区-深南大道-9988号-深圳湾1号(T7栋2801)`

### 常见POI类型

- **商务建筑**: 大厦、大楼、广场、中心、写字楼
- **居住区域**: 小区、花园、苑、山庄、公寓
- **商业设施**: 商场、超市、酒店、餐厅、银行
- **公共设施**: 医院、学校、邮局、车站、机场、公园
- **企业机构**: 公司、企业、工厂、机构、部门、局、院、所
