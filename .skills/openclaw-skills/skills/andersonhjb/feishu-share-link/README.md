# feishu-share-link

**飞书专属分享链接“双保险”生成规范 (Feishu Share Link)**

## 💡 为什么需要这个 Skill？(产生的意义)

在使用 AI 助手处理飞书云文档、多维表格或知识库时，系统 API 默认返回或 AI 习惯生成的通常是飞书的通用根域名链接（如 `https://feishu.cn/...`）。这在实际办公场景中会带来严重的体验割裂：

1. ❌ **强行跳转外部浏览器，打断心流**
   点击基础的 `feishu.cn` 链接，飞书客户端往往会将其当作普通的外部链接，强制唤起系统自带的浏览器打开。这不仅需要重新加载网页，甚至可能要求用户重新扫码登录，完全打破了在飞书软件内沉浸式工作的连贯性。

2. ✅ **企业专属域名 = 软件内原生打开**
   经过深度测试发现：**只有带有企业专属前缀域名**（如 `https://czpn2fds56.feishu.cn/...`）的链接，才能被飞书客户端原生识别，并直接在软件内部的标签页中丝滑打开！这是本 Skill 解决的核心痛点。

3. 🛡️ **“双保险”稳妥防失联机制**
   在某些复杂的企业网络环境（如特殊的 VPN 策略、内外网隔离、或是把链接发给外部临时协作人员）下，专属租户域名偶尔会遇到解析或权限拦截问题。因此，本规范强制要求 AI **必须同时发送专属链接与通用根域名链接**。前者提供极致的原生体验，后者作为 100% 稳妥的兜底方案。

## 🚀 核心机制与表现

当 OpenClaw 安装此 Skill 后，AI 在处理完毕文档或表格时，会：
1. 主动读取本地环境配置（`TOOLS.md`）获取当前主人的企业专属域名。
2. 以如下“双保险”格式向用户发送结果：

> 🏢 **专属链接**: `https://[你的企业前缀].feishu.cn/base/...` *(点击在飞书内直接打开)*
> 🌐 **通用链接**: `https://feishu.cn/base/...` *(备用兜底，外部浏览器稳妥访问)*

## ⚙️ 如何配置
在你的 `~/.openclaw/workspace/TOOLS.md` 中添加如下配置，AI 即可自动读取：
```markdown
### Feishu
- Feishu Custom Domain: [你的专属前缀].feishu.cn
```
