## ⚠️ API 限制说明

### 当前 Remnawave API 版本限制

| 功能 | 支持状态 | 说明 |
|------|----------|------|
| 创建账号 | ✅ 支持 | 完整支持 |
| 分配内部分组 | ✅ 支持 | 需要完整 UUID |
| 设置流量限制 | ✅ 支持 | bytes 单位 |
| 设置设备限制 | ✅ 支持 | 整数 |
| 设置过期时间 | ✅ 支持 | ISO 8601 格式 |
| **流量重置策略** | ❌ **不支持** | API 参数会被忽略 |
| 更新用户信息 | ❌ 不支持 | PATCH/PUT API 未实现 |

### 流量重置策略解决方案

由于 API 不支持，需要手动配置：

#### 方案 A: 管理后台手动配置（推荐）
1. 登录 Remnawave 管理后台
2. 进入用户列表
3. 找到对应用户
4. 点击编辑
5. 设置流量重置策略
6. 保存

#### 方案 B: 浏览器抓包分析
1. 打开浏览器开发者工具
2. 在管理后台修改用户流量重置
3. 抓取实际发送的请求
4. 分析正确的 API 格式
5. 更新脚本

#### 方案 C: 等待 API 更新
关注 Remnawave 更新日志，后续版本可能支持

## 🔍 故障排查

### 问题 1: 内部分组未分配

**症状：** `activeInternalSquads: []`

**原因：** UUID 格式不正确

**解决：**
```bash
# 更新组配置
node update-squads-config.js
```

### 问题 2: 流量重置未生效

**症状：** `trafficResetInterval: null`

**原因：** API 不支持此参数

**解决：** 手动在管理后台配置（见步骤 2）

### 问题 3: 邮件发送失败

**症状：** SMTP 连接错误

**解决：**
1. 检查 `smtp.json` 配置
2. 验证邮箱密码
3. 确认端口和加密方式
