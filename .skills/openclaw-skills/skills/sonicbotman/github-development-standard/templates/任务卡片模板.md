## 验收清单（15 项）

### A. 需求一致性

- [ ] A1. 我能用一句话说清这次修复的目标
- [ ] A2. 我知道这次"不打算修"的内容有哪些
- [ ] A3. 代码改动与 issue 描述一致

### B. 技术正确性

- [ ] B1. 我基于正确版本开始修改
- [ ] B2. 我没有重写整个文件
- [ ] B3. 数据结构变化已同步所有引用点
- [ ] B4. 新逻辑不会破坏旧逻辑

### C. 测试验证

- [ ] C1. 语法检查通过
- [ ] C2. 导入检查通过
- [ ] C3. 最小样例验证通过
- [ ] C4. 回归测试通过

### D. 发布质量

- [ ] D1. diff 大小与任务规模匹配
- [ ] D2. release note 与实际代码一致
- [ ] D3. 版本号、文档、注释已同步
- [ ] D4. 我可以指出这次改动的风险点

---

**⚠️ 这 15 项只要有 1 项答不上来，就不要发布。**
