## B. 技术正确性（4 项）

### B1. 我基于正确版本开始修改

**如何验证：**
```bash
# 检查起始版本
git log --oneline | head -5
git describe --tags
```

**错误示例：**
- ❌ 从上次没提交的改动继续改
- ❌ 从别人的分支继续改
- ❌ 从旧版本继续改

**❓ 你的回答：** 基线版本：_______________

---

### B2. 我没有重写整个文件

**如何判断：**
```bash
# 检查改动比例
git diff --stat scripts/xxx.py

# 如果一个文件改动超过 50%，要警惕
```

**正确做法：**
- ✅ 复制旧代码 → 修改 → 替换
- ❌ 凭记忆重写

**❓ 你的回答：** 改动比例：_______%

---

### B3. 数据结构变化已同步所有引用点

**示例：**
```python
# 修改前
stats = {
    'kept': len(kept),
    'removed': len(removed)
}

# 修改后（新增字段）
stats = {
    'kept': len(kept),
    'removed': len(removed),
    'content_preserved': len(kept_older) + len(kept_recent)  # 新增
}

# 必须检查
- 所有读取 stats 的地方
- 所有写入 stats 的地方
- 所有序列化/反序列化的地方
```

**如何验证：**
```bash
# 搜索所有使用点
grep -r "stats\[" .
grep -r "stats\." .
grep -r "stats\.get" .
```

**❓ 你的回答：** ☐ 已检查所有引用点

---

### B4. 新逻辑不会破坏旧逻辑

**如何验证：**
1. 运行已有的测试用例
2. 手动测试旧功能
3. 检查新代码的输入输出范围

**示例：**
```python
# 旧逻辑
def compress(text):
    return text[:100]

# 新逻辑（错误）
def compress(text):
    return text[:100] + "..."  # 增加了输出长度！

# 新逻辑（正确）
def compress(text):
    return text[:100]  # 行为不变
```

**❓ 你的回答：** ☐ 旧功能仍正常

    assert 'removed' in result
