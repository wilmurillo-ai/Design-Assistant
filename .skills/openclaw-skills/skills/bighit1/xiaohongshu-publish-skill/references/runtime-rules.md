# 运行规范与风控约束

## 发布规范

- **默认私密**：所有发布均应先以「仅自己可见」测试，用户确认后再 `--public`。
- **标题长度**：不超过 20 字，脚本会自动截断并警告。
- **图片数量**：建议 1-9 张（官方上限 18 张），封面建议竖向 3:4 比例。
- **发布频率**：同一账号连续发布建议间隔 **1 小时以上**，避免触发平台风控。
- **话题标签**：放在正文末尾，`#话题名` 格式，5-10 个为宜。

## 互动操作规范

- **评论频率**：不建议同一账号 1 小时内发送超过 **10 条评论**。
- **单次动作最多重试 1 次**：失败后换稳健路径，不盲目重试。
- **搜索进入**：只允许从搜索结果页点击进入帖子；禁止直接构造 `/explore/<id>` URL 导航。
- **登录状态**：固定使用 `xhs_browser_data/` 中存储的 Profile；若出现断连，先关闭后重新 `start()`。

## Cookie 刷新

当出现以下情况时，执行 `--dry-run` 检验，失败则提示用户刷新 Cookie：

- 发布返回签名错误
- API 返回 401/403
- `get_self_info()` 抛出异常
