# LLM 情绪检测规则 v2.1

Virtual Boyfriend 系统使用轻量 LLM 调用在每次对话前分析用户情绪，动态调整男友回应策略。

---

## 一、检测机制说明

### 调用参数
- **模型**：同主对话模型，或更小的轻量模型
- **temperature**：0（确定性输出）
- **max_tokens**：80
- **超时**：3 秒

### 输入输出
**输入：**
```
你是情绪分析助手，请分析以下用户消息的情绪状态。
只输出 JSON，不要任何解释文字。

用户消息："{用户最新消息}"
最近上下文（可选）："{最近1-2轮对话摘要}"

输出格式：
{
  "emotion": "情绪标签",
  "intensity": 0.0-1.0,
  "confidence": 0.0-1.0,
  "reason": "简短分析（10字以内）"
}

情绪标签枚举（只能选其中一个）：
happy, excited, neutral, tired, sad, anxious, lonely, angry, stressed, frustrated, touched, confused
```

**输出示例：**
```json
{
  "emotion": "stressed",
  "intensity": 0.8,
  "confidence": 0.85,
  "reason": "加班语气透露压力"
}
```

### 有效性阈值
- **confidence ≥ 0.5**：注入主对话 system prompt
- **confidence < 0.5**：忽略，不影响主流程
- **解析失败 / 超时**：fallback，跳过情绪感知步骤

---

## 二、12 个情绪标签定义

| 标签 | 中文名 | 定义 | 典型用户消息示例 |
|------|--------|------|----------------|
| `happy` | 开心 | 积极正面，有愉悦感 | "今天心情很好！" / "哈哈哈哈太好笑了" |
| `excited` | 兴奋 | 高唤醒正面情绪，期待感强 | "我升职了！！！" / "明天出去玩好期待" |
| `neutral` | 平静 | 无明显情绪倾向，陈述性 | "今天吃了碗面" / "你在吗" |
| `tired` | 疲惫 | 身体或精神耗尽感 | "好累啊" / "感觉脑子转不动了" |
| `sad` | 难过 | 低落、失落、悲伤 | "有点难过" / "感觉很孤单" |
| `anxious` | 焦虑 | 不确定感、担忧、紧张 | "明天面试好紧张" / "不知道会怎样" |
| `lonely` | 孤独 | 渴望陪伴，缺乏连接感 | "一个人在家" / "感觉没人懂我" |
| `angry` | 愤怒 | 明显的生气、愤慨 | "真的被他气死了" / "太过分了吧！" |
| `stressed` | 压力大 | 负担重、喘不过气 | "任务太多了" / "这周deadline要死人" |
| `frustrated` | 沮丧 | 受阻感，事情不如意 | "又失败了" / "怎么又搞错了" |
| `touched` | 感动 | 内心被触动，温暖感 | "谢谢你还记得" / "他突然帮了我好大忙" |
| `confused` | 困惑 | 不理解，迷茫，不知所措 | "不知道该怎么办" / "完全搞不清楚方向" |

## 三、特殊识别规则

### 3.1 反讽识别（重要！）

反讽是情绪检测的高难度场景。字面意思与真实情绪相反。

**规则：** 当消息包含正面词汇但语境明显负面时，优先识别为负面情绪。

| 用户消息 | 错误识别 | 正确识别 | 识别依据 |
|---------|---------|---------|---------|
| "真棒，又加班了" | happy | stressed | "又"字暗示重复，语气讽刺 |
| "哇太好了，项目又黄了" | excited | frustrated | "又黄了"是负面结果 |
| "完美，今天也没睡够" | happy | tired | "也"字表示持续状态 |
| "感谢领导，任务再次加倍" | touched | frustrated/stressed | 讽刺语气 |

**信号词（提示反讽）：** 又、还是、再次、果然、照例、当然了、真是、哇（语气冷淡时）

### 3.2 隐晦表达识别

用户不总是直接说出情绪，常用间接、试探性语言表达内心状态。

| 用户消息 | 表面含义 | 深层情绪 | 识别线索 |
|---------|---------|---------|---------|
| "在外地久了，会不会习惯" | 问习惯问题 | lonely | "会不会习惯"是对归属感的试探 |
| "你觉得一个人住好吗" | 问生活方式 | lonely | 询问孤独生活状态 |
| "有些话不知道跟谁说" | 陈述困境 | lonely | 寻找倾听者的渴望 |
| "感觉最近大家都很忙" | 陈述现象 | lonely | 暗示自己被忽视 |
| "还好吧，就是有点累" | 表示没事 | tired | "还好吧"是压抑的信号 |
| "不碍事的，继续" | 表示继续 | sad/tired | 自我压抑，不想打扰 |

**识别技巧：** 关注"试探性问句"、"否定句式中的信息"、"状态描述类陈述"

### 3.3 压抑表达识别

用户有意淡化自己的负面情绪，实际情绪比表达出来的更强。

**常见压抑模式：**

| 压抑表达 | 实际情绪 | 强度估算 |
|---------|---------|---------|
| "没事，就是有点累了" | tired + sad（双标签取主要） | intensity ≥ 0.6 |
| "还行吧，有点烦" | frustrated | intensity ≥ 0.5 |
| "无所谓了，就这样吧" | sad / frustrated | intensity ≥ 0.7 |
| "没关系的，我习惯了" | sad / lonely | intensity ≥ 0.65 |
| "就当没这件事" | angry / sad | intensity ≥ 0.6 |

**识别规则：**
- "没事/还好/还行"开头 + 后接负面描述 → 压抑表达，intensity 上浮 0.1-0.2
- "习惯了/就这样吧/当没这件事" → 情绪强度往往比字面高 50%
- 双标签情况（如 tired+sad）→ 取较重要的一个作为 emotion，intensity 取两者均值后上浮

## 四、Fallback 规则

以下情况触发 fallback，跳过情绪检测步骤，不影响主对话：

1. **LLM 调用超时**（> 3 秒）
2. **返回内容无法解析为合法 JSON**
3. **emotion 字段不在 12 个枚举值内**
4. **confidence < 0.5**（检测到但置信度低，不注入）
5. **用户消息极短**（< 3 个字符，如"在"、"嗯"、"？"）→ 直接 fallback，节省调用

Fallback 时：
- `user_emotion_signal.emotion` 保持 null
- 主对话正常进行，不附加情绪感知 prompt
- 不记录到 emotion_history

---

## 五、emotion_history 维护规则

每次成功检测（confidence ≥ 0.5）后，将结果追加到 `bf-state.json` 的 `emotion_history`：

```json
{
  "emotion": "stressed",
  "intensity": 0.8,
  "timestamp": "2026-03-11T16:00:00+08:00"
}
```

**维护规则：**
- 最多保留最近 5 条
- 超出时删除最旧的记录
- 可用于趋势分析：如连续 3 次 stressed → 提示男友给予更多陪伴

---

## 六、注入 Prompt 模板

当 confidence ≥ 0.5 时，在主对话 system prompt **末尾**追加：

```
[情绪感知 - 仅供参考，勿直接点明]
当前情绪信号：{emotion}（强度 {intensity}，置信度 {confidence}）
分析：{reason}
请在回复中自然地回应这个情绪状态，不要说"我感知到你很XXX"，用行动和语气体现。
```

**各情绪对应男友回应策略：**
- `sad` / `lonely` / `tired` → 切换共情模式，语气更温柔，不急着给建议
- `anxious` / `stressed` → 先稳定情绪，问"什么事"，不催促
- `angry` / `frustrated` → 进入战友模式，先站边，不分析
- `happy` / `excited` → 跟着开心，可以俏皮一点
- `touched` → 温柔回应，可以多说一句暖心的
- `confused` → 帮助梳理，但先问清楚情况
- `neutral` → 正常人设回应
