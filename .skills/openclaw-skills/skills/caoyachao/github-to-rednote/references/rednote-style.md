# RedNote (小红书) Style Guide

## Format Principles

### Visual Hierarchy

RedNote content is consumed on mobile devices. Format for small screens:

1. **Short paragraphs** - 2-3 sentences max per paragraph
2. **Clear sections** - Use emoji headers to separate sections
3. **White space** - Empty lines between sections for breathing room
4. **Bullet points** - Use for lists instead of dense paragraphs

### Emoji Usage

| Context | Suggested Emojis |
|---------|-----------------|
| Title | 🔥 ⚡️ 🚀 💎 ⭐ |
| Introduction | 📌 📝 |
| Features | 💡 ✨ 🎯 |
| Code | 💻 ⌨️ 🔧 |
| Warning | ⚠️ ❗ |
| Tips | 💡 📌 |
| Links | 🔗 |
| Stats | ⭐ 🍴 💻 📊 |

Rules:
- Use emojis sparingly (1-2 per section max)
- Place at beginning of line or section
- Don't use more than 5 emojis in a row
- Match emoji to content meaning

### Content Structure

Standard article structure:

```
[Title with emoji]
[Divider]
[Stats line]
[Blank line]
[Section 1: Introduction]
[Blank line]
[Section 2: Key points]
[Blank line]
[Section 3: Details]
[Blank line]
[CTA or Conclusion]
[Divider]
[Hashtags]
```

### Hashtags

Default tech hashtags:
- #技术分享
- #开源项目
- #程序员
- #GitHub
- #开发工具
- #代码学习

Add specific hashtags based on:
- Programming language (#Python #JavaScript #Rust)
- Domain (#AI #Web开发 #移动端)
- Tool name (#VSCode #Docker #Kubernetes)

Format: Space-separated at end of article
Count: 5-10 hashtags total

### Writing Style

**DO:**
- Be concise and direct
- Use technical terms correctly
- Focus on practical value
- Include specific numbers/stats
- Write as if talking to a developer friend

**DON'T:**
- Use sensational language ("震惊！")
- Write long paragraphs
- Include too many emojis
- Be overly promotional
- Use clickbait tactics

### Code Blocks

Format code minimally:

```
💻 安装:
```bash
pip install package-name
```
```

Keep code blocks short (under 10 lines).
For longer code, link to documentation instead.

### Length Guidelines

- **Short**: 300-500 chars (quick tips)
- **Standard**: 500-800 chars (most articles)
- **Long**: 800-1200 chars (detailed tutorials)

Never exceed 2000 characters.

## Platform-Specific Notes

### RedNote (小红书)

- Supports markdown-style formatting
- Hashtags are clickable and searchable
- Images are important (but we focus on text)
- First 2 lines appear in feed preview

### Mobile Optimization

- Test how content looks on narrow screens
- Break long lines manually if needed
- Use visual separators between sections
- Put key info in first 100 characters

## Quality Checklist

Before publishing, verify:

- [ ] Title is compelling but not clickbait
- [ ] No spelling errors
- [ ] Emojis enhance (not distract)
- [ ] All links work
- [ ] Hashtags are relevant
- [ ] Length is appropriate
- [ ] Formatting renders correctly on mobile
