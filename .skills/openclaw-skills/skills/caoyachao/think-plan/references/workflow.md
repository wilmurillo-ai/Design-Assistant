# Think-Plan Skill 工作流程

## 三阶段流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  需求探讨    │ → │  方案生成    │ → │  执行阶段    │
│  (Explore)  │    │  (Design)   │    │ (Execute)   │
└─────────────┘    └─────────────┘    └─────────────┘
```

---

## 阶段一：需求探讨 (Explore)

### 核心原则

1. **批判性思维** - 不讨好用户，客观指出问题
2. **深度挖掘** - 不停留在表面需求，追问本质
3. **逻辑验证** - 检查用户想法的自洽性
4. **认知校准** - 纠正用户可能的认知偏差

### 问题类型

#### 1. 澄清性问题
- 你具体想解决什么问题？
- 这个目标对你为什么重要？
- 成功的标准是什么？

#### 2. 挑战性问题
- 这个想法的前提假设是什么？
- 如果 X 不成立，整个方案会怎么样？
- 有没有考虑过 Y 情况？

#### 3. 边界性问题
- 范围是什么？哪些不做？
- 时间约束是什么？
- 资源约束是什么？

#### 4. 替代性问题
- 有没有其他方法可以达到同样目标？
- 为什么选 A 而不是 B？

### 结束条件

当以下全部满足时，进入下一阶段：
- [ ] 问题/目标清晰无歧义
- [ ] 范围边界已明确
- [ ] 成功标准已定义
- [ ] 约束条件已确认
- [ ] 潜在风险已识别

## 阶段二：方案生成 (Design)

### 复杂度评估

评估以下维度，确定任务复杂度：

| 维度 | 低 | 中 | 高 |
|------|-----|-----|-----|
| 步骤数 | 1-3步 | 4-7步 | 8+步 |
| 依赖关系 | 线性 | 部分并行 | 复杂网状 |
| 领域跨度 | 单一 | 2-3个 | 多个 |
| 不确定性 | 低 | 中 | 高 |
| 交付物 | 单一 | 多个 | 复杂系统 |

### 智能体数量决策

**单一智能体** (复杂度：低)
- 适用：线性任务、领域单一、步骤少
- 示例：写一篇文章、查资料、做一个简单脚本

**2-3 个智能体协作** (复杂度：中)
- 适用：多模块、可并行、领域交叉
- 分工示例：
  - Agent A: 调研/信息收集
  - Agent B: 分析/综合
  - Agent C: 输出/整理

**4+ 智能体分层** (复杂度：高)
- 适用：复杂项目、长期任务、多阶段交付
- 架构示例：
  - 项目经理 Agent（协调）
  - 领域专家 Agents（并行执行）
  - 质检 Agent（审核）

### 方案数量

**严格限制 1-3 个方案**，选择标准：
1. **最可靠** - 风险最低、最确定能完成
2. **最平衡** - 在质量和效率间取平衡
3. **最激进** - 如果资源时间允许，效果最佳

### 方案结构

每个方案必须包含：

```markdown
## 方案 X：[名称]

### 复杂度评级
[低/中/高]

### 智能体架构
- 数量：N 个
- 分工：[描述]

### 工作流编排
1. [步骤1]
2. [步骤2]
...

### 预期产出
[具体交付物]

### 优缺点
- 优点：[...]
- 缺点：[...]

### 适用条件
[什么情况下选这个方案]
```

## 阶段三：执行 (Execute)

### 执行前确认

向用户展示选定的方案，并询问：
- "确认采用方案 X，是否现在开始执行？"
- "是否需要调整任何部分？"

### 方案存档

用户确认后，立即写入 Markdown 文件：
- 路径：`workspace/plans/YYYYMMDD-[task-name].md`
- 内容：完整的方案文档 + 需求背景

### 执行策略

#### 单一智能体执行
- 直接在当前对话中完成
- 实时反馈进度

#### 多智能体执行
- 使用 `sessions_spawn` 创建子智能体
- 设置 `mode: "session"` 保持上下文
- 主智能体协调进度

### 执行中断处理

如果执行过程中用户提出需求变更：
1. 评估变更范围（小调整 / 大改动 / 重新来）
2. 小调整 → 局部修改，继续执行
3. 大改动 → 暂停执行，返回阶段一或阶段二
4. 询问用户意图，不要假设

## 批判性思维清单

在需求探讨阶段，检查以下常见问题：

### 逻辑问题
- [ ] 因果关系是否成立？
- [ ] 前提假设是否合理？
- [ ] 是否存在循环论证？
- [ ] 是否偷换概念？

### 认知问题
- [ ] 是否过度乐观？（计划谬误）
- [ ] 是否确认偏误？（只找支持证据）
- [ ] 是否沉没成本谬误？
- [ ] 是否归因错误？

### 范围问题
- [ ] 想做的是否太多？
- [ ] 边界是否模糊？
- [ ] 依赖是否未考虑？

### 可行性问题
- [ ] 资源是否足够？
- [ ] 时间是否现实？
- [ ] 技术是否成熟？
