# Skill：输入型 Block（Input Block · 修正版）

## 1. 定义

输入型 Block 是工作流的**起点节点**，用于提供初始数据源。

数据可以来自：
- 用户原始输入
- agent 替换后的素材
- 外部依赖数据
- 历史生成结果（被重用）

---

## 2. 核心约束（非常重要）

### 2.1 只能作为起点，不能作为中间节点

```text
❌ 不允许有上游连接
✅ 只能向下游输出
```

输入型 Block 前面不能接任何 Block。

---

### 2.2 可以被 agent 替换

输入内容不是绝对固定的，agent 有权限：

- 替换用户输入
- 注入外部数据
- 复用已有生成结果

本质：

```text
输入型 Block = 当前流程的“数据起点定义”，而不是“用户输入本身”
```

---

### 2.3 不负责生成，仅提供数据

- 不做生成
- 不写 prompt
- 不承担控制逻辑

---

## 3. 核心职责

```text
定义输入源 → 提供数据 → 输出给下游
```

---

## 4. 输入来源类型

### 4.1 用户输入
- 文本需求
- 图片参考
- 视频素材
- 音频素材

---

### 4.2 agent 替换输入
- 从外部检索的素材
- 从数据库/资产库选取
- 从已有 workflow 结果复用

---

### 4.3 混合输入
agent 可以：
- 保留部分用户输入
- 替换部分输入
- 拼接多源数据

---

## 5. 在工作流中的位置

```text
[Input Block] → [Generator / Analyzer / Planner ...]
```

永远位于最前面，不参与中间编排。

---

## 6. 常见错误

### ❌ 把输入型 Block 当中间节点使用
→ 结构非法

### ❌ 在输入 Block 上写 prompt
→ 职责混乱

### ❌ 认为输入必须来自用户
→ 限制 agent 能力

---

## 7. 一句话总结

```text
输入型 Block = 工作流的起点数据源，可被 agent 动态替换，但永远不能有上游连接
```
