# 详情页工作流程

## 比例规格（2025-2026 实测）

| 比例 | 尺寸 | 适用场景 | 文字质量 | 推荐度 |
|------|------|---------|---------|--------|
| **1:4** | 2048×8252 | 简单项目一键直出详情页 | ✅ 好 | ⭐⭐⭐⭐⭐ |
| **1:8** | 1408×11728 | Shopee/Lazada 超长滚动 | ⚠️ 小字略弱 | ⭐⭐⭐⭐ |
| **9:16** | — | TikTok Shop 视频封面 | ✅ 好 | ⭐⭐⭐⭐ |
| **3:4** | — | 标准竖版详情页 | ✅ 好 | ⭐⭐⭐⭐ |
| **1:2** | — | 短版详情页 | ✅ 好 | ⭐⭐⭐ |

> **结论**：优先用 **1:4**（细节好、提示词好写），超长滚动用 **1:8**

## 超长详情页拼接模式（1:4 × N）

当详情页内容较多，一张 1:4 不够时，**分段生成再拼接**：

### 拼接策略

| 总长度 | 切割方式 | 生成张数 |
|--------|---------|---------|
| 适中（10-14章节） | 上半部分 + 下半部分 | 2张 1:4 竖拼 |
| 超长（14+章节） | 上段 + 中段 + 下段 | 3张 1:4 竖拼 |
| 横向故事流 | 左图 + 右图 | 2张 3:4 横拼 |

### 切割原则

```
规则1：每段必须包含完整的"区块/章节"，不可在章节中间切断
规则2：相邻段落之间预留20-30%的内容重叠，确保拼接自然
规则3：每段使用相同的背景色/主色调，保持视觉统一
规则4：每段都要有独立的"区块标题"，拼接后标题顺序连续
```

### 分段 Prompt 模板

**第一段（上半部分）**：
```
【{品类}详情页 Part 1 of 2（上半部分）｜{产品名}｜{主色调}】

▶️ 整体构图：竖版长图（1:4），包含前4-5个区块：
- 区块①/章节①-②：产品主视觉 + 核心卖点
- 区块②/章节③-④：功能/技术展示（前半部分）
- 区块③/章节⑤：细节特写
- 确保本段结尾是一个完整的模块（不可截断）

▶️ 风格统一：与 Part 2 保持完全一致
▶️ 结尾处理：底部留白约15%，为Part 2的顶部衔接做准备
```

**第二段（下半部分）**：
```
【{品类}详情页 Part 2 of 2（下半部分）｜{产品名}｜{主色调}】

▶️ 整体构图：竖版长图（1:4），包含后3-4个区块：
- 区块③/章节⑤：与Part 1衔接的内容
- 区块④/章节⑥-⑦：后半部分展示
- Social Proof / 品牌保障
- CTA + Footer + 品牌Logo

▶️ 开头处理：顶部约15%与Part 1底部重叠，保持视觉延续
▶️ 风格统一：与Part 1完全一致的背景色调、文字风格、字体层级
```

### 拼接示例（童装详情页，分2张 1:4）

| 段 | 内容 | 章节 |
|----|------|------|
| Part 1 | 上半部分 | ①品牌封面 → ②品牌故事 → ③产品功能 → ④面料工艺 → ⑤尺码指南（前半） |
| Part 2 | 下半部分 | ⑤尺码指南（后半） → ⑥穿搭场景 → ⑦用户证言 → ⑧品牌保障 → ⑨CTA |

## 模式 A — 4 区块模板

适合：消费电子/智能家电/工具/数码/有明确技术卖点的产品
基于 Banana 2 + 行业最佳实践

```
┌──────────────────────────┐
│  区块① 产品主视觉区       │
├──────────────────────────┤
│  区块② 核心功能/技术区    │
├──────────────────────────┤
│  区块③ 产品细节/爆炸图区  │
├──────────────────────────┤
│  区块④ 场景/对比/CTA区   │
└──────────────────────────┘
```

**Prompt 公式（高质量版）**：

```
【{品类}电商详情页主视觉 + 功能模块组合图｜{产品名}｜{主色调}主色调｜产品外观绝对统一｜所有分镜中机身颜色、{关键部件A}、{关键部件B}、{关键部件C}设计必须完全一致】

▶️ 整体构图：竖版长图（1:4或1:8比例），分为四大核心区块：

① 区块①「{产品核心卖点Slogan}」区 → {背景色}渐变背景 + 白色粗体大字"{主标题}"，下方小字说明"{副标题内容}"。中央为{产品形态}模型，底部环绕{品牌色}光环，周围分布{数量}个功能图标，每个图标带发光边框+简体中文标签，呈放射状对称布局；

② 区块②「{技术名/功能名}」区 → {场景描述} + {技术可视化描述}；左上角标题"{技术标题}"，副标题"{技术副标题}"；

③ 区块③「{核心部件名}区」 → {爆炸图/透视描述}，展示内部结构：{部件A}、{部件B}、{部件C}、{部件D}，每项用{品牌色}圆角标签标注；背景为{纹理}，突出科技感与精密感；

④ 区块④「{场景/CTA}区」 → {使用场景描述} 或 {对比效果描述} + {购买引导}；

▶️ 文字细节：
- 主标题：思源黑体 Heavy，白色带轻微外发光；
- 副标题：微软雅黑 Regular，字号适中，行距1.5倍；
- 所有文字居中对齐或左对齐，严禁斜排或艺术变形；

▶️ 色彩系统：
- 主色：{主色值}（{主色名}）、{科技色值}（{科技色名}）
- 文字：#FFFFFF（纯白）、#E2E8F0（浅灰）

▶️ 风格要求：
- 商业广告级渲染，照片级写实；
- 所有分图中产品角度、比例、配色必须严格统一，不可随意更改。
```

## 模式 B — 8 章节模板

### 场景 1：标准种草型（配件/童装/快消）

```
P1: Hero 封面     → 产品主图 + 一句话核心卖点
P2: 痛点共鸣      → 使用场景图 + 痛点描述
P3: 解决方案      → 产品功能点（3-5个）各配图
P4: 规格参数      → 尺寸/材质/重量表
P5: 使用场景      → 2-3张场景图，生活方式展示
P6: 细节特写      → 2-4张细节图，工艺/材质
P7: Social Proof  → 评价截图 + 用户晒图
P8: CTA           → 行动按钮 + 促销信息
```

### 场景 2：节日促销型（圣诞/618/开学季等）

```
P1: 节日封面     → 限定标语 + 产品主图 + 节日元素
P2: 节日氛围     → 节日刺绣特写 + 节庆背景
P3: 产品功能展示  → 3-4个核心卖点各配功能图
P4: 尺码指南     → 尺码表 + 测量方法说明
P5: 面料与工艺   → 质感特写 + 材质展示
P6: 穿搭场景     → 节日穿搭 + 日常穿搭
P7: 品牌保障     → 品牌介绍 + 品质承诺
P8: CTA + Footer → 立即购买 + 品牌Logo
```

### 场景 3：对比测评型（工具/数码/消费电子）

```
P1: 测评封面     → "为什么要买XX" 悬念型标题
P2: 痛点场景     → 旧产品/竞品痛点可视化
P3: 技术对比     → 新旧/竞品参数表格对比
P4: 核心升级     → 3大技术突破各配图
P5: 细节展示     → 关键部件放大特写
P6: 场景演示     → 真实使用场景图
P7: 权威认证     → 检测报告/专利/奖项
P8: CTA         → 限时优惠 + 购买链接
```

### 场景 4：痛点解决型（个护/家居/健康）

```
P1: 共情封面     → 目标用户痛点场景图
P2: 痛点放大     → 痛点的3个具体表现
P3: 解决方案     → 产品如何解决每个痛点
P4: 成分/原理    → 核心技术/成分说明
P5: 真实体验     → 使用前后对比
P6: 安全认证     → 质检/认证/背书
P7: 用户证言     → 真实评价精选
P8: CTA         → 限时优惠引导
```

### 场景 5：品牌故事型（高客单/礼品/设计师品牌）

```
P1: 品牌封面     → 品牌调性大图
P2: 品牌理念     → 创始人故事/品牌愿景
P3: 设计来源     → 设计灵感/工艺传承
P4: 产品展示     → 全系列产品图
P5: 细节品质     → 高端工艺特写
P6: 选材溯源     → 原材料/产地说明
P7: 品牌服务     → 售后/定制/保养
P8: CTA          → 专属购买引导
```
