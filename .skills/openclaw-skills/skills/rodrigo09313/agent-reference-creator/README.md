# Reference Creator Skill

Скилл для создания Reference-файлов системы агентов OpenClaw.

## Что делает этот скилл

Создаёт нормативные файлы специализаций (references) которые описывают профессии агентов, а не конкретных агентов. По референсу агент-архитектор может создать любого агента данной специализации.

## Установка

### Автоматическая установка (рекомендуется)

```bash
# Из корня проекта OpenClaw
cp -r reference-creator-skill OpenClaw\ KB/skills/reference-creator
```

### Ручная установка

1. Создайте директорию для скилла:
```bash
mkdir -p "OpenClaw KB/skills/reference-creator"
```

2. Скопируйте файлы:
```bash
cp reference-creator-skill/SKILL.md "OpenClaw KB/skills/reference-creator/"
cp reference-creator-skill/LICENSE.txt "OpenClaw KB/skills/reference-creator/"
```

## Назначение скилла агенту

### В SOUL.md агента

Добавьте в раздел Skills:

```markdown
## Skills

- `reference-creator` — создание Reference-файлов специализаций для системы OpenClaw
```

### В agent spec (YAML)

```yaml
skills:
  - reference-creator
```

## Кому назначить этот скилл

Рекомендуется назначить:
- **Агенту-архитектору** — тот кто создаёт других агентов
- **Методологу** — тот кто разрабатывает структуру системы
- **Документалисту** — тот кто ведёт базу знаний

## Использование

После установки скилла агент может:

1. **Создать референс через интервью:**
   Просто запустите диалог и отвечайте на вопросы скилла

2. **Создать референс из готового описания:**
   Заполните блок "Описание новой специализации" в конце SKILL.md

3. **Валидировать существующий референс:**
   Проверить файл на соответствие всем требованиям

## Структура выходного файла

Файлы сохраняются по пути:
```
OpenClaw KB/references/[specialization].md
```

Каждый референс содержит 13 обязательных разделов:
1. Специализация
2. Ключевые компетенции  
3. Персонализация
4. Домен
5. Делает (фиксировано)
6. Не делает (фиксировано)
7. Skills
8. Routing
9. Стандарты работы
10. Пайплайн
11. Инструкции для агента-архитектора
12. Модель по умолчанию
13. Changelog

## Примеры существующих референсов

В системе уже есть референсы:
- `database-administrator` — хранитель базы знаний
- `kaizen-coach` — коуч непрерывных улучшений
- `strategist` — системный стратег
- `secretary-operator` — операционный секретарь
