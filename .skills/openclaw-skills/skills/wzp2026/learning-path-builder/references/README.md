## 一、核心原则（详细）

### 1.1 权威资源优先原则

| 推荐等级 | 来源类型 | 示例 |
|---------|---------|------|
| ✅ 首选 | 国家标准 | GB/T xxx-xxxx（对应领域标准） |
| ✅ 首选 | 官方文档 | 教育部/工信部/人社部官方文档 |
| ✅ 首选 | 权威平台 | 中国大学MOOC（清华/北邮/上交大等） |
| ⚠️ 参考 | 技术社区 | GitHub开源项目、OWASP、权威技术组织 |
| ⚠️ 参考 | 学术期刊 | 中国知网/计算机学报相关论文 |
| ❌ 避免 | 非权威 | 个人博客、社交媒体、商业推广 |

**权威资源来源：**
- 技能人才评价工作网（国家职业标准查询系统）：https://www.osta.org.cn/
- 国家标准全文公开系统：https://openstd.samr.gov.cn
- 中国大学MOOC：https://www.icourse163.org
- 教育部官网：https://www.moe.gov.cn
- 工信部官网：https://www.miit.gov.cn

### 1.2 搜索优先原则

**搜索来源优先级：**

| 优先级 | 来源类型 | 示例 | 标注 |
|--------|---------|------|------|
| P1 最高 | 国家职业标准 | osta.org.cn | ✅ 权威来源 |
| P1 最高 | 国家标准 | openstd.samr.gov.cn | ✅ 权威来源 |
| P1 最高 | 政府机关 | miit.gov.cn, moe.gov.cn, gov.cn | ✅ 权威来源 |
| P1 最高 | 高校/学术 | edu.cn, ac.cn 域名下的官方平台 | ✅ 权威来源 |
| P2 中等 | 学术数据库 | CNKI知网、万方、国家哲学社会科学文献中心 | ⚠️ 参考来源 |
| P2 中等 | 权威技术组织 | 官方技术文档、权威行业组织 | ⚠️ 参考来源 |
| P3 最低 | 行业社区 | 技术博客、开发者社区 | ⚠️ 参考来源（需核实） |
| ❌ 禁止 | 非权威网络 | 个人博客、贴吧、论坛、微博、公众号等 | 禁止使用 |

**搜索格式示例：**

```
site:osta.org.cn {关键词} 职业标准
site:openstd.samr.gov.cn {关键词} 国家标准
site:edu.cn {关键词} 课程/教材
site:gov.cn {关键词} 政策/法规
site:moe.gov.cn {关键词} 教育文件
site:miit.gov.cn {关键词} 标准/文件
site:cnki.net {关键词} 学术论文
```

**搜索-推论流程：**

```
① 使用 site:osta.org.cn / site:gov.cn / site:edu.cn / site:openstd.samr.gov.cn 搜索
   → 找到权威来源 → ✅ 权威来源
② 扩大范围搜索（不限 site）
   → 找到支撑来源 → ⚠️ 参考来源
③ 搜索不到或不确定 → 大模型推论
   → 标注为 🤖 推断内容
④ 推论内容必须注明"建议结合权威资料核实"
```

## 二、三层层次结构（详细）

### 2.1 层次说明

```
L1 大节点（领域层）
├── 如「算法选型及调优」（可变学时：12~20h，按L1数量动态分配）
│   └── L2 中节点（模块层）
│       ├── 如"机器学习基础"（可变学时：4~12h）
│       │   └── L3 叶节点（固定0.5学时/节点）
│       │       ├── 概念A（0.5h）✅
│       │       ├── 概念B（0.5h）⚠️
│       │       └── 概念C（0.5h）🤖
│       └── …多个中模块
└── L2/L3 继续展开
```

### 2.2 学时分配说明

| 领域 | L1总学时 | L2数量 | 说明 |
|------|---------|--------|------|
| （动态，由PDF原文的职业功能数量决定） | 12~20h/领域 | 动态 | 基于PDF原文动态提取 |

> 示例：NLP方向6个职业功能 → 每领域约 13h（共约80h）

### 2.3 来源分类详解

| 来源类型 | 标识 | 说明 |
|---------|------|------|
| **权威来源** | ✅ | 有明确权威来源：国家标准编号、官方URL、真实MOOC课程 |
| **参考来源** | ⚠️ | 有参考但非原文：学术期刊参考、技术社区文档 |
| **推断内容** | 🤖 | 无原始依据：大模型根据领域知识逻辑推导 |

**来源判定规则：**
```
if 包含国家标准编号(如GB/T) and 有官方文档URL:
    → 权威来源
elif 有中国大学MOOC中真实高校课程:
    → 权威来源
elif 有官方技术文档:
    → 权威来源
elif 有学术期刊/知网论文参考:
    → 参考来源
elif 有GitHub/技术博客参考:
    → 参考来源
else:
    → 推断内容（需人工审核）
```

## 四、多方向标准检测与交互式处理（详细）

### 4.1 检测流程

当输入的标准文档包含多个专业方向时（如人工智能工程技术人员有5个方向）：

1. **方向检测**：分析标准文档，列出所有专业方向
2. **推荐分析**：用大模型推理各方向的学习热度、材料丰富度、应用前景
3. **默认推荐**：基于分析给出综合推荐
4. **用户确认**：展示所有方向选项，让学习者选择

### 4.2 多方向标准示例

**示例：人工智能工程技术人员**

```
检测结果：本标准包含5个专业方向

方向列表：
1. AI基础研究 — 从事AI理论、算法研究
2. AI平台研发 — 从事AI框架、平台、工具开发
3. 智能系统开发 — 从事AI应用系统设计与实现
4. AI行业应用 — 推动AI在具体行业的落地实施
5. AI伦理治理 — 从事AI伦理、隐私、安全、合规工作

推荐分析：
- 智能系统开发（⭐⭐⭐⭐⭐）：学习热度最高，材料最丰富，就业面最广
- AI行业应用（⭐⭐⭐⭐）：次热门，政策扶持力度大
- AI平台研发（⭐⭐⭐⭐）：技术门槛高，薪资最高
- AI基础研究（⭐⭐⭐）：学术导向，适合深造
- AI伦理治理（⭐⭐）：新兴方向，政策驱动，未来潜力大

默认推荐：智能系统开发（方向3）
```

**动态L1领域提取示例：**

当选择方向后，从PDF原文动态提取职业功能领域：

```
动态提取的L1领域（示例，NLP方向为6个）：
L1_1: 人工智能算法选型及调优（1.1节）
L1_2: 人工智能算法实现及应用（1.2节）
L1_3: 自然语言及语音处理需求分析（2.1节）
L1_4: 自然语言及语音处理设计开发（3.1节）
L1_5: 自然语言及语音处理测试（4.1节）
L1_6: 自然语言及语音处理产品交付与运维（5.1/6.1节）
```

> **注意**：示例中的方向名称和数量仅供参考，实际运行时根据输入的标准文档自动检测。不同职业标准的方向数量和命名由原文决定。

## 五、工作流程（详细）

### 5.1 执行流程

```
【第零步】多方向标准检测（快速）
  → 通过当前会话发送通知：「已检测到N个方向，已推荐[默认方向]，请确认」
  → 等待用户选择（或直接使用默认推荐）

【第一步】PDF文本提取
  → 执行：pdftotext 提取全文
  → 写入：/tmp/kg_pdf_text_{timestamp}.txt
  → 通过当前会话发送通知：「文本提取完成，共X行，准备构建知识图谱...」

【第二步】结构分析与节点规划
  → 执行：解析PDF标准章节结构，动态提取职业功能领域（L1）
  → 解析PDF内容，识别初级/中级/高级各等级的"工作内容"与"相关知识要求"
  → 写入：/tmp/kg_structure_{timestamp}.json
  → 通过当前会话发送通知：「结构分析完成：从标准中动态提取了L1×N个职业功能领域」

【第三步】逐域构建（逐L1领域执行，每个完成后通知）
  → 写入：/tmp/kg_build_progress_{timestamp}.json（增量写入）
  → 通过当前会话发送通知：「[领域名] 构建完成，N个节点，来源标注：✅N个/⚠️N个」

【第四步】生成多格式文件
  → 执行：生成 JSON + CSV_nodes + CSV_edges + JSON-LD
  → 通过当前会话发送通知：「多格式文件生成完成，共X个文件」

【第五步】生成来源说明文档 + 质量检查
  → 写入：{name}_kg_sources.md
  → 通过当前会话发送通知：「✅ 知识图谱构建完成！总学时Xh，节点数：L1×N（动态提取）/ L2×N / L3×M」
```

### 5.2 进度通知模板

每步完成后发送会话消息，格式如下：

```markdown
📊 知识图谱构建进度

✅ [步骤名称] 已完成
- 用时：X分X秒
- 产出：[产出描述]
- 状态：正常/有小问题

→ 接下来：[下一步名称]，请稍候...
```

### 5.3 超时与续接策略

| 情况 | 处理方式 |
|------|---------|
| 某步骤执行中 | 等待完成，不设硬性超时 |
| 某步骤完成通知后 | 等待用户确认或自动继续 |
| 某步骤失败 | 通过当前会话发送通知失败原因，从该步骤重试 |
| 用户中断 | 保留已生成的文件，下次可续接 |

### 5.4 文件命名规范

```
/tmp/kg_pdf_text_{job_id}.txt      # PDF原始文本
/tmp/kg_structure_{job_id}.json    # 结构分析结果
/tmp/kg_build_progress_{job_id}.json  # 构建进度（增量）

{output_dir}/{name}_kg.json        # 主JSON
{output_dir}/{name}_kg_nodes.csv   # 节点表
{output_dir}/{name}_kg_edges.csv    # 关系表
{output_dir}/{name}_kg.jsonld       # JSON-LD
{output_dir}/{name}_kg_sources.md  # 来源说明
```
