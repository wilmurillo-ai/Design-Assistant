# 能力地图（自包含版）

这份文档把 feishu-writing-bundle 整合的 7 类核心能力的实际规则直接写进来，不依赖外部 skill 文件。

---

## 能力 1：读取飞书文档（feishu_fetch_doc）

**工具**：`feishu_fetch_doc`

**支持格式**：
- wiki 链接：`https://xxx.feishu.cn/wiki/Xxxxx`
- docx 链接：`https://xxx.feishu.cn/docx/Xxxxx`
- 裸 doc_id：`HdWfdXXXXXXXX`

**核心规则**：
- 改稿任务必须先 fetch，不能凭记忆或猜测内容
- 读完才能判断用哪种 update 模式
- 读取多维表格用 `feishu_bitable_app_table_record`（需从 URL 提取 `app_token`）

---

## 能力 2：创建飞书文档（feishu_create_doc）

**工具**：`feishu_create_doc`

**核心规则**：
- `title` 传标题，正文不再重复一级标题（从 `##` 开始）
- `markdown` 字段用 Lark-flavored Markdown
- 位置参数三选一：`wiki_space`（推荐 `my_library`）/ `wiki_node` / `folder_token`
- 长文档（>2000字）拆成：create（骨架）+ 多次 update(append)
- 调用后立即回 `doc_url`，这是完成标准

**文档要自包含**：开头补足背景、来源、时间、适用对象，不能默认读者看过聊天上下文。

---

## 能力 3：更新飞书文档（feishu_update_doc）

**工具**：`feishu_update_doc`

**7 种模式**（按使用频率排序）：

| 优先级 | 模式 | 用途 |
|-------|------|------|
| ★★★ | `append` | 追加新章节，分段写长文档 |
| ★★★ | `replace_range` | 替换某段/某章节 |
| ★★ | `insert_before/after` | 在特定位置前后插入 |
| ★★ | `delete_range` | 删除过时内容 |
| ★ | `replace_all` | 全文替换关键词 |
| ⚠️ | `overwrite` | 全文清空重写（慎用，会清评论/图片关联） |

**定位参数**：
- `selection_by_title: "## 章节标题"` → 自动匹配整章，首选
- `selection_with_ellipsis: "开头...结尾"` → 范围匹配

**核心规则**：默认不 overwrite；除非用户明确要求"整篇重写"，否则用局部模式。

## 能力 4：局部精准改稿

**没有专属工具，是一套操作策略**。

适合：演讲稿、说明文、方案文档的保守精修。

**操作流程**：
1. `feishu_fetch_doc` 读原文
2. 判断改什么：表达？结构？语气？信息补充？
3. 只改必要部分，保留原文作者意图
4. 用 `replace_range`（按标题或内容定位）精确落到文档
5. 回链

**核心规则**：
- 不要把作者原本的观点洗成别的东西
- 改稿 ≠ 重写；除非原文逻辑已经无法救

---

## 能力 5：去 AI 腔 / 人味化

**没有专属工具，是写作风格规则**。

**触发时机**：文档信息对但读感像模板拼出来的。

**操作清单**：
- 删掉空泛拔高句（"这充分体现了…""对此我们有深刻认识"）
- 去掉机械排比（"第一…第二…第三…"填鸭式）
- 消掉过度总结句（段尾再重复一遍段首观点）
- 保留信息密度，但让句子更像真人会写的样子

**注意**：
- 人味化 ≠ 加情绪词汇
- 人味化 ≠ 写得随意
- 目标是：**可信、自然、不像完成任务的机器**

---

## 能力 6：Proposal 正式化

**没有专属工具，是写作风格规则**。

**触发时机**：用户说"改成能发的 proposal""要正式一点""给老师/评审看"。

**操作清单**：
- 重建论证结构（问题→分析→方案→预期效果）
- 收紧语气，减少口语感和聊天体
- 每个论点都要有依据，不要只列观点
- 减少碎点拼贴，改用完整段落
- 去掉"其实""大概""感觉上"等不确定词

**区分**：
- 普通文档：能读懂就行
- Proposal：能拿去正式提交/评审

---

## 能力 7：飞书交付回链

**操作**：任务结束后，必须显式返回 `doc_url`。

**标准格式**：
```
✅ 已整理成飞书文档：[一句话说明]
🔗 https://www.feishu.cn/docx/XxxXxxxXxx
```

**核心规则**：
- 不说"我写完了"，而是给链接
- 不让用户追问"链接呢"
- 如果文档落在用户无法访问的空间，这件事还没完成 → 继续处理权限
