# Workflows

## Workflow A：资料检索 → 飞书文档

适合：用户让你“检索资料并写个飞书文档”。

步骤：
1. 搜集资料
2. 读核心材料
3. 组织结构
4. 需要时做人味化
5. 创建飞书文档
6. 返回飞书文档链接

输出标准：
- 不是只回一段聊天文本
- 必须有 doc_url

---

## Workflow B：已有飞书文档 → 局部改稿

适合：用户说“改这篇飞书文档”“改这篇演讲稿”。

步骤：
1. 读取原文
2. 明确修改目标
3. 只改需要改的部分
4. 增量更新到原文档
5. 回链，并说明改动点

输出标准：
- 默认不整篇覆盖
- 保留原结构

---

## Workflow C：草稿 → 正式 proposal

适合：用户说“改成能发的 proposal”“要正式一点”。

步骤：
1. 读取原稿
2. 重建论证结构
3. 去 AI 腔
4. 创建新文档或更新既有文档
5. 返回链接

输出标准：
- 更像正式提交材料
- 不是聊天体
- 不是碎列表拼贴
