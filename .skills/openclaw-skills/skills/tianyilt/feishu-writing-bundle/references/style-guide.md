# Style Guide

## 核心原则

1. 文档是交付物，不是聊天记录
2. 优先可读、可协作、可继续编辑
3. 优先结构稳定，不追求花哨排版
4. 默认写完就回飞书文档链接

## 结构建议

- 标题明确
- 层级不要过深
- 多用短段落
- 在关键信息处用 callout
- 数据或对比信息可用表格
- 流程可用 mermaid

## 风格建议

- 少套话
- 少空泛总结
- 少“显得很厉害”的装饰句
- 保持信息密度
- 对正式材料收紧口语感

## 交付建议

当任务目标是“写个飞书文档”时，最终交付应包括：
- 文档已创建 / 已更新
- 明确的飞书链接
- 一句简短说明：这份文档是什么，做了什么处理
