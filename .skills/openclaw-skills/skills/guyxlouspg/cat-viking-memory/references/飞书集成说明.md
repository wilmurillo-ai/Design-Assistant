# 飞书集成说明

## 概述

Viking 记忆系统可以与 OpenClaw 的飞书插件集成，实现群聊会话的自动保存功能。

## 功能特性

- **自动检测会话结束**：通过监测会话活跃时间，自动识别会话结束时刻
- **自动保存记忆**：在会话结束后自动提取关键信息并保存到 Viking 记忆系统
- **智能摘要**：自动生成会话摘要，包含主题、决策、待办事项等
- **无缝集成**：作为飞书插件的扩展功能，无需额外配置（除启用外）

## 工作原理

飞书插件中的 `FeishuSessionManager` 类负责：

1. **会话跟踪**：记录每个飞书群的最后活跃时间
2. **超时检测**：配置的检查间隔（默认5分钟）检测是否超时（默认30分钟）
3. **触发保存**：当会话超时时，调用 `memory-session-hook.sh` 脚本执行保存

```typescript
// 关键配置
const DEFAULT_CHECK_INTERVAL_MS = 5 * 60 * 1000; // 5分钟检查
const DEFAULT_TIMEOUT_MS = 30 * 60 * 1000;      // 30分钟超时
const SESSION_HOOK_PATH = "~/.openclaw/skills/memory-pipeline/scripts/memory-session-hook.sh";
```

## 启用方式

### 1. 确保飞书插件已启用

在 OpenClaw 配置中确保飞书插件已加载：

```bash
# 检查插件状态
openclaw plugins list | grep feishu
```

### 2. 配置 Session Manager

在飞书插件的配置文件中添加（或确认）以下配置：

```json
{
  "sessionManager": {
    "enabled": true,
    "checkIntervalMs": 300000,
    "timeoutMs": 1800000
  }
}
```

### 3. 验证脚本路径

确保 `memory-session-hook.sh` 脚本存在于正确路径：

```bash
ls -la ~/.openclaw/skills/memory-pipeline/scripts/memory-session-hook.sh
```

## 相关文件

- `session-manager.ts`：飞书会话管理器核心实现
- `memory-session-hook.sh`：会话结束时的保存钩子脚本
- `memory-pipeline`：Viking 记忆管道主程序
- `memory-auto-save.sh`：自动保存脚本（定时任务）
- `memory-tier-cron.sh`：记忆层级降级脚本

## 注意事项

1. **权限要求**：飞书插件需要执行 `memory-session-hook.sh` 的权限
2. **Viking 配置**：确保 `memory-pipeline` 的配置正确（见 references/README.md）
3. **存储空间**：自动保存会增加记忆存储，注意监控磁盘使用
4. **测试建议**：初次启用时建议在测试群验证功能正常

## 故障排查

### 自动保存未触发

1. 检查 `FeishuSessionManager` 是否正在运行：
```bash
ps aux | grep session-manager
```

2. 查看日志：
```bash
tail -f ~/.openclaw/logs/feishu-session.log
```

3. 确认超时配置是否合理，过短的超时可能导致频繁保存

### 保存失败

检查 `memory-session-hook.sh` 的退出码和输出：

```bash
# 手动测试
~/.openclaw/skills/memory-pipeline/scripts/memory-session-hook.sh "test_chat_id" "test_account"
```

## 进阶配置

如需自定义 Hook 脚本行为，可修改 `memory-session-hook.sh`，它接受两个参数：

```bash
# 参数说明
$1 = 聊天会话ID (chatId)
$2 = 账户ID (accountId)
```

脚本内部会提取会话内容并调用 `memory-pipeline mp_global` 保存。
