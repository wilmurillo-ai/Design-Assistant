# Viking 记忆系统技能 - 使用教程

## 简介

**cat-viking-memory** 是一个完整的 Viking 记忆系统技能包，为 OpenClaw AI Agents 提供五级记忆层级管理、向量语义检索、动态重要性调整以及飞书群聊会话自动保存功能。

> **特点**：自动降级、重要保护、跨 Agent 共享、飞书集成

---

## 功能概览

| 功能 | 描述 |
|------|------|
| **五级记忆层级** | L0→L4 自动降级，0-1天完整 → 90天+仅标题 |
| **重要记忆保护** | 标记 `important` 的记忆永不降级 |
| **向量相似度检测** | 被反复提及的记忆自动提升重要性 |
| **飞书自动保存** | 会话结束30分钟无消息自动保存 |
| **跨 Agent 共享** | 全局 workspace 团队共享知识 |
| **LLM 智能压缩** | 使用 Gemini CLI 自动摘要降级 |

## 配置说明

### 1. 基础配置

编辑技能目录下的 `config.json`：

```json
{
  "memory_workspace": "~/.openclaw/viking-{agent}",
  "auto_save_enabled": true,
  "feishu_integration": {
    "enabled": false,
    "check_interval_ms": 300000,
    "timeout_ms": 1800000
  }
}
```

**关键配置项：**

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `memory_workspace` | Agent 私有的记忆存储目录 | `~/.openclaw/viking-{agent}` |
| `auto_save_enabled` | 自动保存开关 | `true` |
| `feishu_integration.enabled` | 飞书自动保存开关 | `false` |
| `feishu_integration.check_interval_ms` | 检查间隔（毫秒） | `300000` (5分钟) |
| `feishu_integration.timeout_ms` | 会话超时（毫秒） | `1800000` (30分钟) |

### 2. 设置定时任务（推荐）

每天凌晨 3:00 自动执行记忆降级和索引重建：

```bash
# 编辑 crontab
crontab -e

# 添加以下行
0 3 * * * ~/.openclaw/workspace/agents/maozhuli/cat-viking-memory/memory-pipeline/memory-tier-cron.sh
```

> **注意**：请根据实际路径调整脚本路径

### 3. 飞书自动保存配置（可选）

如需启用飞书群聊会话自动保存，请按以下步骤：

#### 3.1 确保 feishu 插件已启用

```bash
# 检查插件状态
openclaw plugins list | grep feishu
```

如果未安装，先安装：
```bash
openclaw plugins install feishu
```

#### 3.2 修改 feishu 插件配置

编辑 OpenClaw 配置文件（通常位于 `~/.openclaw/config.yaml` 或 `~/.config/openclaw/config.yaml`）：

```yaml
plugins:
  feishu:
    enabled: true
    sessionManager:
      enabled: true
      checkIntervalMs: 300000
      timeoutMs: 1800000
```

#### 3.3 修改技能配置

在 `cat-viking-memory/config.json` 中启用飞书集成：

```json
"feishu_integration": {
  "enabled": true,
  "check_interval_ms": 300000,
  "timeout_ms": 1800000
}
```

#### 3.4 重启 OpenClaw Gateway

```bash
openclaw gateway restart
```

#### 3.5 验证功能

查看日志确认会话管理器已启动：
```bash
tail -f ~/.openclaw/logs/gateway.log | grep session-manager
```

应该看到类似：
```
[INFO] FeishuSessionManager started, checkInterval=300000ms, timeout=1800000ms
```

## 使用方式

### 1. 加载记忆上下文

启动时自动加载最近7天的记忆（hot + warm）：

```bash
# 执行一次加载，测试是否正常
~/.openclaw/workspace/agents/maozhuli/cat-viking-memory/memory-pipeline/memory-pipeline mp_autoload
```

或在 agent 配置的 hooks 中自动调用。

### 2. 保存记忆到 Viking

#### 自动保存（推荐）

任务完成后，只需简单调用：
```bash
memory-pipeline mp_global "完成了用户偏好分析任务，记录：用户喜欢简洁汇报、重视效率、偏好直接沟通"
```

系统会自动：
1. 提取关键词和标签
2. 判断重要性
3. 存储到当前热区（L0）
4. 触发向量索引更新

#### 手动详细保存

如需精细控制，使用 `mp_store`：

```bash
# L0 层：完整记录（0-1天）
memory-pipeline mp_store \
  --title "用户需求分析" \
  --content "背景：xxx\n过程：xxx\n结果：xxx\n待办：xxx" \
  --tags "需求,分析" \
  --tier L0 \
  --importance high

# L1 层：核心轮廓（2-7天）
memory-pipeline mp_store \
  --title "周会总结" \
  --content "核心：本周完成3个功能开发，待解决2个bug" \
  --tags "周会" \
  --tier L1
```

### 3. 搜索记忆

```bash
# 语义搜索（向量相似度）
sv_find "董事长偏好"

# 关键词搜索
sv_find "紧急 bug"

# 查看具体记忆文件
cat ~/.openclaw/viking-maojingli/agent/memories/hot/2026-03-18_*.md
```

### 4. 查看记忆统计

```bash
# 查看各层级文件数
ls -la ~/.openclaw/viking-maojingli/agent/memories/hot/ | wc -l
ls -la ~/.openclaw/viking-maojingli/agent/memories/warm/ | wc -l
```

## 记忆文件格式

Viking 使用 **Frontmatter + Markdown** 格式：

```yaml
---
id: mem_20260318_001
title: "用户需求分析"
importance: high
important: true          # 此记忆永不降级
tags: [需求, 分析, 用户]
created: 2026-03-18T10:30:00Z
last_access: 2026-03-18T14:22:00Z
access_count: 5
retention: 90
current_layer: L0
level: 0
weight: 14.2
---
```

```markdown
# 用户需求分析 [L0]

## 背景
用户希望简化汇报流程，强调效率优先。

## 过程
与用户进行了2轮深度沟通，整理了核心诉求清单。

## 结果
确定了3个核心需求：1) 简洁汇报 2) 快速响应 3) 数据透明

## 待办
- [ ] 细化需求文档
- [ ] 原型设计
```

## 飞书自动保存详解

### 工作流程

```
┌─────────────────┐
│ 飞书群收到消息   │
└────────┬────────┘
         ↓
   更新会话活跃时间
         ↓
   每5分钟检查一次
         ↓
   是否超时？(30分钟无消息)
         ↓
     是 → 调用 memory-session-hook.sh
              ↓
        提取会话摘要 (LLM)
              ↓
        调用 mp_global 保存
              ↓
        更新会话状态 (saved=true)
```

### 关键文件

| 文件 | 说明 |
|------|------|
| `session-manager.ts` | 飞书会话管理器（运行在插件中） |
| `memory-session-hook.sh` | 会话结束时的保存钩子 |
| `memory-pipeline` | 主程序（实际执行保存） |

### 日志定位

```bash
# 查看会话管理器日志
tail -f ~/.openclaw/logs/feishu-session.log

# 或查看 gateway 日志
tail -f ~/.openclaw/logs/gateway.log | grep -i session
```

### 测试方法

1. 在飞书群聊中发消息
2. 等待30分钟不发言（或修改配置缩短超时）
3. 检查记忆是否自动保存：
```bash
ls -lt ~/.openclaw/viking-maojingli/agent/memories/hot/ | head -5
```

## 常见问题

### Q1: 自动保存不触发？

**排查步骤：**

1. 检查 FeishuSessionManager 是否运行：
```bash
ps aux | grep -i feishu | grep session
```

2. 查看日志是否有错误：
```bash
tail -n 50 ~/.openclaw/logs/gateway.log | grep -A5 -B5 session
```

3. 确认配置 `feishu_integration.enabled = true`

4. 检查 `memory-session-hook.sh` 是否可执行：
```bash
ls -l ~/.openclaw/workspace/agents/maozhuli/cat-viking-memory/memory-pipeline/memory-session-hook.sh
```

### Q2: 记忆降级没执行？

确认 crontab 有定时任务：
```bash
crontab -l | grep memory-tier
```

手动测试：
```bash
~/.openclaw/workspace/agents/maozhuli/cat-viking-memory/memory-pipeline/memory-tier-cron.sh
```

### Q3: 如何标记重要记忆？

在存储时添加 `--important` 参数或 frontmatter 中设置 `important: true`：

```bash
memory-pipeline mp_store \
  --title "关键决策" \
  --content "..." \
  --important
```

带 `important: true` 的文件**永不降级**，始终保留完整细节。

### Q4: 跨 Agent 共享怎么看？

全局记忆存储在 `~/.openclaw/viking-global/`：

```bash
# 查看全局任务
cat ~/.openclaw/viking-global/shared/tasks/2026-03-18.md

# 读取全局配置
cat ~/.openclaw/viking-global/boss/preferences.md
```

使用 `mp_global` 写入的内容都会进入全局空间：
```bash
memory-pipeline mp_global "董事长今日重点：关注用户体验优化"
```

### Q5: 向量搜索不准？

1. 确保 embedding 服务可用（默认使用 Ollama）：
```bash
curl http://localhost:11434/api/tags
```

2. 重建向量索引：
```bash
~/.openclaw/workspace/agents/maozhuli/cat-viking-memory/memory-pipeline/memory-embed.sh rebuild ~/.openclaw/viking-maojingli
```

3. 调整相似度阈值：编辑 `memory-session-hook.sh`，修改 `SIMILARITY_THRESHOLD=0.63`

## 进阶话题

### 动态重要性机制

Viking 会跟踪记忆的 `access_count`（访问次数）：

| 条件 | 行为 |
|------|------|
| `access_count > 10` | low → medium |
| `access_count > 20` | 自动标记 `important: true` |
| 30天未访问 | 重要性降级 |
| 90天未访问 | 封存到 `archived/` |

这意味着：**经常被搜索的记忆会自动变得重要**！

### 与 simple-viking 配合

`simple-viking/sv` 提供底层向量搜索能力，可用于自定义脚本：

```bash
# 混合搜索（语义+关键词）
sv hybrid "用户偏好"

# 列出所有层级
sv layers

# 自动加载上下文（为后续对话注入背景）
sv autoload maojingli
```
