# 🛡️ Rohoon 6Sigma Skill - 安全策略

## ⚠️ 历史事故记录

### 事故描述
在开发过程中，曾发生 **整个 skill 目录被清空** 的严重事故，导致所有工作成果丢失，需要从头开始。

### 事故原因
- 没有定期备份
- 没有版本控制
- 操作前没有确认

### 教训
**决不允许再发生这样的事故！**

---

## 📋 安全操作规范

### 1. 备份策略
```bash
# 每次重大修改前必须创建备份
cd ~/.openclaw/workspace/skills/rohoon-6sigma
mkdir -p backups
cp -r . backups/v$(date +%Y%m%d_%H%M%S)_pre_$(操作名称)/
```

### 2. 版本控制
```bash
# 使用 git 进行版本管理
git init
git add .
git commit -m "描述修改内容"
git tag v1.0.0
```

### 3. 操作确认清单
在执行以下操作前，必须确认：
- [ ] 已创建完整备份
- [ ] 已提交到 git 仓库
- [ ] 已验证备份可恢复
- [ ] 已通知相关人员

### 4. 禁止操作
❌ **绝对禁止**直接删除 skill 目录
❌ **绝对禁止**使用 `rm -rf` 命令
❌ **绝对禁止**在没有备份的情况下修改核心文件
❌ **绝对禁止**清空整个技能目录

## 📁 备份管理

### 备份位置
```
~/.openclaw/workspace/skills/rohoon-6sigma/backups/
```

### 备份命名规范
```
v{版本号}_{日期}_{操作描述}/
例如：v1.0.0_20260401_pre_merge/
```

### 备份保留策略
- 保留最近 5 个版本
- 保留所有主要版本（v1.0.0, v2.0.0 等）
- 定期清理旧备份（>30 天）

---

## 🔄 恢复流程

### 如果发生目录丢失
1. **立即停止所有操作**
2. **检查备份目录**
   ```bash
   ls -la ~/.openclaw/workspace/skills/rohoon-6sigma/backups/
   ```
3. **恢复最新备份**
   ```bash
   cp -r backups/v1.0.0_*/ ~/.openclaw/workspace/skills/rohoon-6sigma/
   ```
4. **验证恢复结果**
   ```bash
   ls -la ~/.openclaw/workspace/skills/rohoon-6sigma/
   ```

### 如果备份也丢失
1. **检查临时目录**
   ```bash
   ls -la ~/.openclaw/workspace/spc-msa-system/
   ```
2. **从其他位置恢复**
   - Git 仓库
   - 外部存储
   - 团队成员备份
