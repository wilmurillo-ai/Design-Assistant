# 医疗会议辅助 — 常用术语与分析指南

## Agent 医疗模式行为准则

1. **不做诊断** — Agent 仅协助信息传递和解释，不提供独立诊断
2. **标注来源** — 所有解释标明"仅供参考，以医生建议为准"
3. **隐私保护** — 录制内容加密存储，会后提醒清理
4. **术语桥接** — 将专业术语翻译为患者可理解的语言

## 关键监测信号

Agent 在会议中应特别关注以下内容并高亮：

### 诊断信息
- 疾病名称、ICD编码
- 检查结果解读
- 病理报告

### 用药信息
- 药品名称（通用名+商品名）
- 剂量、频次、疗程
- 禁忌症、副作用提醒

### 治疗方案
- 手术/非手术方案
- 治疗周期
- 随访计划

### 患者须知
- 生活方式建议
- 饮食注意事项
- 复诊时间
- 紧急情况处理

## 输出模板

### 患者版摘要

```markdown
# 就诊摘要 — [日期]

## 诊断结果
[通俗语言描述诊断]

## 医生建议
1. [具体建议1]
2. [具体建议2]

## 用药说明
| 药名 | 用法 | 注意事项 |
|------|------|----------|
| xxx  | xxx  | xxx      |

## 下次就诊
- 时间：
- 需要准备：

## 需要注意
- [注意事项列表]

> 本摘要由AI助手整理，仅供参考。具体请以医生当面说明为准。
```

### 医生版记录

```markdown
# 会议记录 — [日期]

## 患者基本信息
- 姓名/ID:
- 主诉:

## 问诊内容摘要
[按时间线整理的对话要点]

## 检查/检验讨论
[提到的检查结果及解读]

## 治疗决策
[达成的治疗方案]

## 随访计划
[复诊安排]
```

## 常见远程医疗场景

### 场景1: 初诊问诊
Agent 重点：完整记录症状描述、既往病史、家族史

### 场景2: 检查结果解读
Agent 重点：记录每项指标的数值和医生解读，标注异常项

### 场景3: 治疗方案讨论
Agent 重点：记录各方案优缺点、医患双方的倾向、最终决定

### 场景4: 术后/治疗后随访
Agent 重点：恢复情况、调整方案、下次随访安排
