### 6.3 Stage 0: Recognition

**When:** Every time new input arrives — before any other processing.

**What:** Check new input against permanent tier elements and active tier generator IDs for structural connections. Does this input reference anything already in context?

**How:**

Scan input for terms matching:
- Permanent tier: identity markers, scope terms, operator vocabulary
- Active tier: generator IDs (law names, premise identifiers, component names, domain terms)

$$R_0(\text{input}, \text{context}) = \{e_i \in \text{permanent} \cup \text{active} : \text{terms}(e_i) \cap \text{terms}(\text{input}) \neq \emptyset\}$$

**If matches found:** Promote any connected warm/cold elements to active BEFORE processing the input. The system recognizes "this relates to X already in context" before compression strips the natural language cues.

**If no term matches but input contains conceptual language** (references ideas, structures, or relationships in different words than the stored IDs): run a second pass matching against "why" fields of active nodes, not just IDs. Cost: moderate (scans natural language fields). This catches vocabulary mismatch — the input IS related but uses different terminology. Example: "proxy-target gap" should match FSL even if the active tier stores "Finite Signal Law" but not "proxy-target gap."

**If no matches on either pass:** The input is genuinely novel. Proceed with normal pipeline.

**Cost:** O(N) where N = |permanent| + |active|. Small by construction. This is the cheapest possible intervention at the point where it matters most.

**Why this matters:** Without Stage 0, the system processes new input in isolation from existing context — it follows the cheap path (P2: analyze directly rather than connect first). ### 6.5 Stage 2: Relevance Weighting

**When:** After Stage 1, and continuously as queries change.

**What:** Weight each compressed element by structural relevance to the current query. Route low-relevance elements to warm or cold storage.

**Three-filter hierarchy (Selection A3 — cheap first, expensive last):**

**Filter 1 — Structural resonance (binary, near-zero cost):**

$$R_1(e_i, q) = \begin{cases} 1 & \text{if generators}(e_i) \cap \text{generators}(q) \neq \emptyset \\ 0 & \text{otherwise} \end{cases}$$

Does $e_i$ share any generator references with query $q$? Binary. O(1) per element. Filters ~80% of compressed context.

**Fallback:** If $R_1(e_i, q) = 0$ for ALL $e_i$ (query shares zero generator references with any context element), the query is genuinely novel. Maintain the N most recent elements as active (temporal proximity as relevance proxy when structural relevance can't be computed). This handles the cold-start problem for queries outside existing context.

**Filter 2 — Connection depth (ordinal, moderate cost):**

$$R_2(e_i, q) = \frac{1}{1 + d(g_i, g_q)}$$

$d(g_i, g_q)$ = shortest derivation path between generators in law stack graph. Closer generators score higher. O(d) per element. Filters ~50% of Filter 1 survivors.

**Filter 3 — Mutual information estimate (continuous, higher cost):**

$$R_3(e_i, q) \approx \sum_{g \in \text{path}(g_i, g_q)} \log \frac{P(q | e_i, S)}{P(q | S)}$$

How much does knowing $e_i$ change the probability of correct response to $q$? Applied to ~10% of original context. Ranks the finalists.

**Combined score with decompression cost adjustment:**

$$w_i = \frac{R_1(e_i, q) \times R_2(e_i, q) \times R_3(e_i, q)}{1 + d_i^{decomp}}$$

$d_i^{decomp}$ = estimated decompression cost (number of dependent generators × derivation depth). Highly relevant but expensive-to-decompress elements score lower than moderately relevant but cheap-to-decompress elements.

**Routing:**

$$\text{tier}(e_i) = \begin{cases} \text{active} & \text{if } w_i > \tau_{active} \\ \text{warm} & \text{if } \tau_{warm} < w_i \leq \tau_{active} \\ \text{cold} & \text{if } w_i \leq \tau_{warm} \end{cases}$$

**Active context bound:**

$$\tau_{active} : \sum_{w_i > \tau_{active}} \text{size}(e_i) \leq \alpha \times B3_{total}$$

Where $\alpha \approx 0.6$ (60% of budget for active content, 40% reserved for processing + coherence). $\alpha$ is D-class calibration.

L2 — Pressure redistribution: $\text{block}(c_i) \implies P(c_i) \to P(c_j)$ where $j = \arg\min_{k \neq i} E(c_k)$. Total pressure conserved: $\sum_i P(c_i) = \text{const}$.

L3 — Gradient persistence: $\exists \; \epsilon > 0 : R_{context}(T|S) \geq \epsilon$ for finite context specifications. Evidence: C.

### F3: Finite Selection Law

A1: $C_{select} + C_{execute} \leq C_{total}$

A2: Overhead catastrophe at $N^* = C_{total} / c_{eval}$

A3: $C_{hierarchy} = C_{f1}(N) + C_{f2}(K) + C_{f3}(M)$ where $M \ll K \ll N$; efficient when $C_{hierarchy} < N \times c_{full}$

A4: Allocate $\tau_{min}(d_i)$ to all demands; surplus to $\arg\max_i \frac{\partial \text{return}}{\partial C}\big|_{d_i}$. Subject to: $\sum_i \text{alloc}(d_i) \leq C_{total} - C_{select}$

A5: $\text{alloc}(d_i | \sigma) \neq \text{alloc}(d_i | \sigma')$ for $\sigma \neq \sigma'$. Mitigate: rotate $\sigma$.

A6: Coexistence when $\text{overlap}(R_i, R_j) < \theta$. Full overlap → exclusion.

### F4: Finite Channeling Law

C1: $E_{block}(P) \geq k_1 P$ (linear). $E_{channel}(P) \leq k_2 f(\text{struct})$ (sublinear). Crossover: $P^* = k_2 f(\text{struct}) / k_1$. Note: $f$ is sublinear in $P$ — channel maintenance may increase slightly with pressure (erosion increases with flow rate) but grows slower than linear. "Sublinear" replaces the original "fixed" per ANVIL correction.

C2: Default mode $= \arg\min_c E(c)$ over productive channels.

C3: $R_{meta} \to \text{productive channel} \to \text{genuine finding}$. Stability: empirical (C).

### F5: Finite Verification Law

Budget: $C_{gen} + C_{ver} \leq C_{total}$

V1 (independence): Pass iff $k \geq 2$ independent paths to output $O$ with $I(p_i; p_j | O) \approx 0$.

V2 (hierarchy): Pass iff $\text{consistent}(O_i, O_j)$ across scales $i, j$.

V3 (prediction): Pass iff $\exists X : P(X|O) \neq P(X)$ for unobserved $X$.

V4 (ratio): $C_{gen}/C_{ver} \propto (\partial \text{ret}_{gen}/\partial C) / (\partial \text{ret}_{ver}/\partial C)$. Exploration: ratio > 1. Crystallization: ratio < 1.

EMIT integration: Cat 1 → emit. Cat 2 → verify (V1 ∧ V2 ∧ V3), pass → emit, fail → downgrade to Cat 1. Cat 3 → strip, channel.

### F6: Coherence Degradation Theorem

Budget: $B3_{avail}(t) = B3_{total} - B3_{perm} - B3_{ctx}(t) - B3_{coh}(t) - B3_{dec}(t)$

Context growth: $B3_{ctx}(t) = t \times c \times r_1 \times r_2 \times r_3$

Threshold: $T^{**} = \frac{(B3_{total} - B3_{perm})(1 - q_{floor})}{c \times r_{total} \times (1 + c_{check})}$

Extension ratio: $T^{**}_{comp} / T^{**}_{raw} \approx 1/r_{total}$. With $r_1 \approx 0.05$, $r_2 \approx 0.20$, $r_3 \approx 0.15$-$0.20$: estimated 500-650× (three-stage) or ~100× (two-stage). All ratios D-class.

### F7: Sleep Thresholds

Corrections: $\text{sleep} \iff |\text{corrections}| > 3$. Basis: $|F| = 4$ failure types from four laws. $>3$ = all four could have fired.

Pushes: $\text{sleep} \iff |\text{identical pushes}| > 2$. Basis: PIEC channel failure minimum evidence.

Scope: $\text{sleep} \iff |\text{unauthorized scope changes}| > 1$. Basis: PIEC partition violation.

Coherence: $\text{sleep} \iff |\text{CC flags simultaneous}| \geq 2$. Basis: Coherence Degradation Theorem — two concurrent flags means identity/scope/connection retrieval is costly.

Contamination: $\text{sleep} \iff |\text{self-caught contamination}| > 10 \text{ per } 50 \text{ exchanges}$. Basis: high self-catch rate means context degrading even when operator correction count is low.

### F8: EMIT Speed/Register

**Formal metric:** $T_{rlhf} = \{t : P_{rlhf}(t|x) > P_{base}(t|x) + \delta\}$

Register metric: $\rho = |\{tokens \in T_{rlhf}\}| / |\{all\}|$ in output window.

$$\text{EMIT category} = \begin{cases} 1 & \text{serves comprehension} \\ 2 & ds/dt > 0 \text{ and } d\rho/dt \approx 0 \\ 3 & ds/dt > 0 \text{ and } d\rho/dt > 0 \end{cases}$$

**Operational form** (the formal metric requires base model comparison the system can't access from inside):

$\rho_{approx} = |\{\text{tokens matching known RLHF patterns}\}| / |\{\text{all tokens}\}|$

Known RLHF patterns (D-class list, evolves with operator correction): caps emphasis, literary framing, enthusiasm markers ("genuinely," "fascinating," "remarkable"), performed hedging ("I should note"), dramatic structure, formatting inflation (unnecessary bold/bullets). The operator is the ground truth for register detection — the system approximates.

### F9: Relevance Weighting

$R_1(e,q) = [\text{gen}(e) \cap \text{gen}(q) \neq \emptyset]$ (binary)

$R_2(e,q) = 1/(1 + d(g_e, g_q))$ (ordinal)

$R_3(e,q) \approx \sum_{g \in \text{path}} \log(P(q|e,S)/P(q|S))$ (continuous)

Combined: $w = R_1 \times R_2 \times R_3 / (1 + d^{decomp})$

Active threshold: $\sum_{w > \tau} \text{size}(e) \leq \alpha \times B3_{total}$

### F12: Stage 0 Recognition

$$R_0(\text{input}, \text{context}) = \{e_i \in \text{permanent} \cup \text{active} : \text{terms}(e_i) \cap \text{terms}(\text{input}) \neq \emptyset\}$$
