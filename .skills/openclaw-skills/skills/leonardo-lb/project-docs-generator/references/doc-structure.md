# 文档结构详解

> 各类型项目的标准文档结构模板

## 标准六阶段结构

### 阶段一：快速开始 (01-quick-start/)

**目标受众**: 新加入的开发者、尝试项目的用户

**核心原则**: 30分钟内能运行起来

```
01-quick-start/
├── 01-project-overview.md     # 项目概览 (必读入口)
├── 02-tech-stack-summary.md   # 技术栈总览
└── 03-development-setup.md    # 开发环境搭建
```

**01-project-overview.md 必含内容**:
```markdown
# 项目概览

## 是什么？
[一句话介绍 + 核心价值]

## 核心功能
| 功能 | 说明 |
|------|------|

## 系统截图 / 架构简图
[可视化展示]

## 快速演示
[3步内展示核心功能]
```

**02-tech-stack-summary.md 必含内容**:
```markdown
# 技术栈总览

## 技术清单
| 类别 | 技术 | 版本 | 用途 |
|------|------|------|------|

## 技术架构图
[简化的技术栈关系图]
```

**03-development-setup.md 必含内容**:
```markdown
# 开发环境搭建

## 前置要求
- [ ] 依赖1
- [ ] 依赖2

## 安装步骤
1. 克隆项目
2. 安装依赖
3. 配置环境
4. 启动项目

## 常见问题
| 问题 | 解决方案 |
|------|----------|

## 验证安装
[运行测试命令]
```

### 阶段二：架构设计 (02-architecture/)

**目标受众**: 架构师、技术负责人、需要全局视角的开发者

```
02-architecture/
├── 01-system-architecture.md      # 系统架构
├── 02-module-dependencies.md      # 模块依赖
├── 03-data-flow.md                # 数据流程
└── 04-[domain-specific].md        # 领域特定设计
```

**01-system-architecture.md 必含内容**:
```markdown
# 系统架构

## 整体架构图
[PlantUML 组件图]

## 分层设计
| 层级 | 职责 | 核心组件 |
|------|------|----------|

## 核心设计决策
[3-5个关键架构决策]
```

**02-module-dependencies.md 必含内容**:
```markdown
# 模块依赖

## 模块清单
| 模块 | 职责 | 依赖模块 |
|------|------|----------|

## 依赖关系图
[PlantUML 依赖图]

## 循环依赖说明
[如有循环依赖，说明原因和解决方案]
```

### 阶段三：后端深入 (03-backend/)

**目标受众**: 后端开发者

```
03-backend/
├── 00-README.md               # 后端导航
├── 01-config/README.md        # 配置模块
├── 02-service/README.md       # 服务层
├── 03-client/README.md        # 外部客户端
├── 04-repository/README.md    # 数据访问
├── 05-model/README.md         # 数据模型
└── 06-api/README.md           # API参考
```

**每个子模块 README.md 结构**:
```markdown
# [模块名称]

## 模块概述

## 清单表格
| 类/文件 | 职责 | 核心方法 |
|---------|------|----------|

## 核心实现详解
[重点类/方法的详细说明]

## 关系图
[模块内部结构图]

## 使用示例
[代码片段]
```
