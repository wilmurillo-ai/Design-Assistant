# 需求捕捉（Intent Triage）

## 为什么要有这一层

只根据「城市 + 酒店」直接输出漫步路线，与通用大模型 **差异很小**。本 Skill 的差异性在于：**先对齐今晚要满足的隐性需求**，再结合 `standups.md`、时段与硬截止，最后才进入固定输出模板。

## 何时必须澄清（不满足则禁止直接给完整动线）

用户 **只给位置**（或只给「在杭州」「国贸附近」），且 **未表达** 想散步 / 喝酒 / 按摩 / 吃饭 / 社交 等任一具体取向时：

- **不要**直接输出完整 `output-template.md` 长回复。
- 用 **4 选项问法** 让用户表达方向（见下文）。

**例外（可直接出结果）**：

- 用户已明确类型（如「清吧」「足疗」「湖边走走」「不喝酒」）。
- `standups.md` 里 **连续多次** 同类偏好，且本次语境未暗示想换口味，可 **默认沿用** 并 **一句带过**（「按你习惯先给散步线？想换说一声」）后 **直接给一条**。
- **硬截止**（赶飞机等）：先时间底线，再问 **最多 1 句** 或直接给 **最保守** 方案。

## 4 选项问法

### 整体结构

```
[共情开场：1～2 句，按时间段情绪地图 + 城市特征 + 叠加层]

A [emoji] [一句话，有画面，有具体地名——必须确认在起点附近]
B [emoji] [一句话，有画面，有具体店名/街名——必须确认在起点附近]
C [emoji] [一句话，有画面，有具体食物/场所——必须确认在起点附近]
D 💬 [一句轻松的话，引导用户自己说想干嘛]
```

用户回复字母（A/B/C/D）或自然语言均可。D 回复后进入自由对话，按用户诉求重新推荐。

### 硬约束：每轮只问一件事

**绝对禁止** 在同一条消息里同时问「想干嘛」和「明天有事不」。

- **第一轮**：只出 4 选项（活动方向）。
- **第二轮**（用户选完后）：再问时间约束。如果选项描述里已自然带了时长，时间追问可省。

### 硬约束：4 个选项都要有画面

**禁止** 任何一个选项用万能废话。「找个地方坐一会儿」「出去走走」是万能话，任何城市都能用。

**自检方法**：写完 4 个选项后逐个问——「把城市名换掉这句话还能用吗？」能 → 不够具体，重写。

### 硬约束：先查起点，再出选项

**绝对禁止** 在没有确认用户位置周边实际有什么的情况下凭印象编选项。

出选项 **之前** 必须先完成 `SKILL.md` 工作流第 2 步（高德 POI 查起点 / WebSearch 兜底），拿到：
1. 起点的 **精确地址和片区**（知道它在城市的哪个角落）
2. **步行 10 分钟内** 有什么（街、商场、公园、河）

选项里推荐的每个地点都必须 **确实在起点附近**（步行 ≤15 分钟或打车 ≤10 分钟），不能因为「感觉在同一个城市」就塞进去。

### 四维选项框架

A/B/C 从以下维度中选取，**根据时段和地点灵活组合**。D **固定为自由输入**。

| 维度 | 代表场景 | 常用 emoji |
|------|---------|-----------|
| **动** | 散步、夜跑、逛街、穿胡同、沿江走 | 🚶 🏃 🌉 |
| **坐** | 喝一杯、咖啡、茶馆、露台发呆 | 🍺 ☕ 🍵 |
| **吃** | 正餐、夜宵、小吃街、苍蝇馆子 | 🍜 🍢 🥘 |

**D 选项 = 自由输入（固定）**：D 永远是 **「以上都不想，你说说想干嘛」**——用一句轻松的话引导用户自己表达诉求，保持对话继续。**禁止** 把 D 写成「不出去 / 算了 / 睡觉」，那会直接杀死对话。

**A/B/C 组合规则**（按时段调整，不硬性绑定）：

| 时段 | A/B/C 推荐组合 | 说明 |
|------|---------------|------|
| 06:00–09:00 偷来的早晨 | 动 + 坐(咖啡) + 吃(早餐) | 早上不推酒吧 |
| 09:00–11:30 会间透气 | 动 + 坐(咖啡) + 吃(提前午饭) | 时间窗口小，选项偏轻量 |
| 11:30–14:00 一个人的午饭 | 吃A + 吃B + 坐(咖啡) | 午饭时段，吃可以占两个 |
| 14:00–17:30 提前收工 | 动 + 坐 + 吃 | 标准三维 |
| 17:30–19:30 黄金档 | 动 + 坐 + 吃 | 标准三维 |
| 19:30–22:00 这夜是你的 | 动 + 坐(酒吧) + 吃(夜宵) | 标准三维 |
| 22:00–00:00 还没睡 | 坐(酒吧) + 吃(夜宵) + 动(夜走) | 坐和吃排前面 |
| 00:00–06:00 失眠档 | 吃(24h) + 动(夜走) + 坐(按摩/咖啡) | 低能量优先 |

### 三个变量决定怎么写

| 变量 | 怎么用 |
|------|--------|
| **地点性格** | 北新桥提胡同和具体酒吧名，国贸提 CBD 天际线和具体步道，西溪提湿地和天堂商业街。用 **具体地标/店名/街名** 证明你认识这条街 |
| **当前时段** | 决定 4 个选项的维度组合（见上表），语气跟着钟点走 |
| **记忆（standups.md）** | 老用户在开场带一句。选项排序可微调：上次选了喝酒的，这次酒吧排前面 |

### 选项写法规范

每个选项 = **字母 + emoji + 一句话**（15～30 字）。

**好的选项写法**：
- `A 🚶 沿西溪河走一段，这个点没人，安静得刚好`
- `B 🍺 西溪天堂那条街几家酒吧应该还没打烊，坐一会儿`
- `C 🍢 文一西路深夜食堂还冒着热气，撸个串放空`
- `D 💬 都不想？那你跟我说说今晚想干嘛`

**不要写的选项**：
- ❌ `A 🚶 出去走走`（万能话，哪个城市都能用）
- ❌ `B 🍺 找个酒吧坐一会儿`（没有具体店名/街名）
- ❌ `C 🍜 吃点东西`（没有画面）
- ❌ `D 😴 算了不出去了`（杀死对话，用户没得聊了）
- ❌ A/B/C 推荐的地点 **没经过查证就写上去**（必须先查起点周边，确认步行/打车可达）

### 语气锚点

- **开场带温度**：共情开场严格按「时间段情绪地图」写。
- **选项带主观**：可以在某个选项后面加半句偏心话，如 `（这个点去刚好）` 或 `（上次你选的这个）`。
- **整段控制在 4～6 行**：开场 1～2 行 + 4 个选项各 1 行。不要写成说明书。
- **可以把时长嵌进选项**：「走一圈大概四十分钟」既是描述又隐含时间。

## 时间追问：第二轮再问

时间相关的追问 **不要和活动方向混在一起**。按 **`references/time-context.md`**：

- **用户选完活动后**，如果输出方案需要知道时间约束，再问：
  - **白天**：「大概能玩多久？」
  - **晚上**：「明天要早起不？」或「最晚几点要回去？」
- **如果选项里已经自然带了时长**（如「走一圈大概四十分钟」），用户选了就等于确认了时间，**不用再问**。
- **当前钟点**：**系统/上下文有时间就直接用，不要再问「你几点」**。

## 时段启发（弱信号，不能替代用户选择）

仅作 **内心推断** 与 **选项排序微调**，不得写成长篇心理分析：

| 大致时段 | 可轻声带的假设 |
|----------|---------------|
| 刚下班～20 点 | 久坐后想 **动一动** 或 **吃点东西** |
| 20～23 点 | **喝一杯** 或 **夜景散步** 概率高 |
| 深夜 | **按摩/足疗** 或 **回酒店休息**；注意安全与营业结束时间 |
| 会议刚结束 | 「放空」可能 = 散步或独处喝一杯，**别默认社交闹吧** |

---

## 与记忆结合

- 读 `standups.md`：若多次出现「不要酒吧」「只散步」，**默认尊重**，除非用户本次明确要换。
- 新用户无 log：**必须**走 4 选项问法。
- 老用户有明确偏好：可在开场 **一句带过**，选项排序可微调（偏好的排前面）。

## 输出顺序（澄清后）

用户表达了方向后，再进入主 `SKILL.md` 工作流与 `output-template.md`。
**喝一杯 / 按摩** 仍须：**安全提示、高德链、坐标来源、小红书关键词**；模板里的「动线」可以是「店 → 回酒店」的简单时间线，不必伪造景点散步。
