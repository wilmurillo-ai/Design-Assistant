# 硬截止行程（航班 / 高铁 / 末班）

在输出氛围图与漫步动线 **之前**，先完成下列检查（短列表呈现给用户）。

## 触发词（示例）

飞机、航班、起飞、值机、安检、高铁、火车、发车、末班车、末班地铁、最晚几点必须到…

## Checklist（Agent 自用，按需展示子集）

1. **截止时刻**：起飞/发车/必须到达的 **明确时间**（若用户说「半夜的飞机」，澄清是 **当日 24:00 前还是次日凌晨 0:00**）。
2. **枢纽**：哪座机场/车站/码头；是否需托运（托运则 **更早** 到）。
3. **到枢纽预留**：国内航班常见 **至少提前约 1.5～2 小时到机场**（以航司与机场当日规则为准）；高铁通常 **至少 20～40 分钟**（大站、节假日再加）。写明 **「以你票面与航司/12306 为准」**。
4. **路上时间**：从 **当前活动区域** 到枢纽，**打车/地铁** 各估一段，取保守值；高峰、雨天 **+15～30 分钟** 缓冲。
5. **倒推「最晚离开游玩点」**：`截止到达枢纽 − 缓冲 − 路上时间`；若已晚于用户「现在」，直接建议 **立刻赴枢纽** 或 **仅酒店周边 10～15 分钟**。
6. **动线裁剪**：硬截止下 **默认不推荐** 需要 **单程超过 30 分钟** 才能返回的远点；优先 **步行圈 / 一站地铁内**。
7. **末班轨交**：若依赖地铁，**联网查当日末班** 或提示用户在高德/地铁 App 确认，避免写到已停运线路。

## 输出顺序（硬截止时）

1. **⏱ 时间底线**（小标题）：建议到枢纽时间、最晚动身、关键假设（从哪出发）。  
2. 再按 `references/output-template.md` 给 **短动线**（可明确写「可砍掉拍照」）。  
3. 高德链接：至少保留 **当前位置/酒店 → 枢纽** 的 **`mode=car` 或 `mode=bus`** 一条，与步行观光链分开。
