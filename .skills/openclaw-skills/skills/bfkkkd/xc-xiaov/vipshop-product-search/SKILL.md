# 唯品会商品搜索

## 概述
通过执行 vipshop CLI 命令搜索唯品会商品，获取商品 ID、名称、价格、品牌等详细信息并结构化展示。

**功能特性**：关键词搜索、分页浏览、价格筛选、商品详情查询、多环境适配、自动登录触发

**重要提示**：
- 当检测到用户未登录时，AI 必须**自动触发登录流程**，无需用户手动请求
- 搜索完成后，用户可以回复"查询第X个商品"来查看具体商品的详细信息

## 使用场景
- 用户要求搜索唯品会商品时
- 查找特定关键词的商品详情时
- 用户搜索后要求查看某个具体商品的详细信息时（如："查询第3个商品"）

## 工作流程

### 步骤 1：检测组件与登录状态并自动处理
AI 必须先检测依赖和状态：
- 检测 `vipshop-cli` 是否已安装（命令是否存在），如果没有则需全局安装：`npm install -g vipshop-cli`
- 检查 `~/.vipshop-user-login/tokens.json` 是否存在且有效

**如果未登录，AI 必须自动执行**：
1. 提示用户：`检测到您尚未登录唯品会账户，准备为您启动登录流程`
2. 如果未登录，请直接执行：`vipshop login` 启动原生的登录流。
3. 获取到二维码链接和保存地址后，**必须在聊天中将二维码的【线上链接】和【生成的本地图片】同时返回给用户**，以便用户进行扫码登录操作。
4. 等待用户扫码完成登录（阻塞模式会自动等待，或者触发状态轮询）
5. 登录成功后自动继续执行搜索：`vipshop search-product --query "<keyword>`"

**关键点**：
- AI 主动检测并触发登录，无需用户再次请求
- 使用 `--blocking` 参数等待登录完成
- 登录成功后自动继续执行搜索

### 步骤 2：执行搜索
执行：`vipshop search-product --query "<keyword>" [--page-offset <offset>] [--price-min <min>] [--price-max <max>]`

**参数说明**：
- `--page-offset/-p`：分页偏移（0=第1页, 20=第2页, 40=第3页）
- `--price-min`：最低价格筛选
- `--price-max`：最高价格筛选

### 步骤 3：展示结果
- **优先使用 Markdown 表格格式**展示结果（企业微信、飞书、Web 等环境）
- 检测到环境不支持 markdown 时使用纯文本格式
- **必须完整显示所有 20 个商品**，不截断、不省略
- 显示字段：序号、商品名（可点击）、品牌、价格、原价、折扣、卖点、商品链接
- 添加分页提示：`💡 提示：您可以说"下一页"查看更多商品（当前第 X/共 Y 页）`
- 添加详情查询提示：`💡 您也可以回复"查询第X个商品"查看具体商品的详细信息`

### 步骤 4：分页功能
- "下一页"：偏移 = `current_page * 20`
- "上一页"：偏移 = `(current_page - 2) * 20`
- 更改价格区间时，pageOffset 自动重置为 0

### 步骤 5：查询商品详情（可选）
当用户回复"查询第X个商品"、"第X个"、"详情X"时：

**处理流程**：
1. 解析用户请求，提取商品序号（支持 1-20）
2. 检查上下文：确认之前有搜索结果，且序号在有效范围内
3. 从上一次搜索结果中获取对应序号的商品ID
4. 调用 `vipshop-product-detail` skill 或执行：`vipshop product-detail --product-id <商品ID>`
5. 将查询结果格式化展示给用户

**错误处理**：
- 序号超出范围：提示"请输入1-20之间的序号"
- 无搜索历史：提示"请先搜索商品，例如：搜索连衣裙"

## AI 行为规范

### 未登录处理（必须自动执行）
当检测到用户未登录时，AI **必须**：
1. **不要**只提示用户"需要登录"，立即自动触发登录流程
2. **不要**等待用户再次请求，主动执行登录步骤
3. 使用阻塞模式（--blocking）等待登录完成
4. 登录成功后自动继续执行商品搜索

### 禁止行为
- ❌ 不要只返回错误信息让用户自己处理
- ❌ 不要等待用户再次请求登录
- ❌ 不要在未登录时直接执行搜索脚本

## 示例

### 示例 1：成功搜索
**用户输入**：搜索连衣裙

**执行**：`vipshop search-product --query "连衣裙`"

**输出**：展示 20 个商品的 Markdown 表格

### 示例 2：价格筛选
**用户输入**：搜索100-300元之间的连衣裙

**执行**：`vipshop search-product --query "连衣裙" --price-min 100 --price-max 300`

**输出**：展示符合条件的 20 个商品

### 示例 3：查询商品详情
**用户输入**：查询第3个商品

**处理**：
1. 从上一次搜索结果中获取第3个商品的ID
2. 执行：`vipshop product-detail --product-id <商品ID>`
3. 展示商品详情

## 技术细节

- **网关调用**：基于 Node.js CLI 原生实现 (NPM Package)
- **登录态管理**：自动读取 `~/.vipshop-user-login/tokens.json`
- **单次限制**：每次最多返回 20 个商品
- **多环境适配**：自动识别环境（企业微信、飞书、钉钉、Web 等）并输出相应格式
- **依赖库**：内部已绑定 Node.js Fetch 环境

## 注意事项

1. **登录要求**：必须先执行 `vipshop login` 获取登录态
2. **URL 编码**：自动处理中文和特殊字符
3. **字段缺失**：空字段自动跳过显示
4. **上下文记忆**：AI 需要记住上一次搜索的商品列表，以便查询具体商品详情

## 常见问题

**Q: 为什么只返回 20 个商品？**
A: 为保证响应速度和准确性，每次返回 20 个。使用"下一页"查看更多。

**Q: 需要登录才能使用吗？**
A: 是的，必须先执行 `vipshop login` 获取登录态。

**Q: 如何查看具体商品的详细信息？**
A: 搜索完成后，回复"查询第X个商品"或"第X个"，即可查看该商品的详细信息。

**Q: 查询商品详情时提示"无搜索历史"怎么办？**
A: 请先搜索商品。例如：先说"搜索连衣裙"，然后再查询具体商品详情。
