# 唯品会商品详情查询

## 概述
唯品会商品详情查询助手，根据商品ID查询商品的详细信息，包括基本信息、价格、品牌、服务保障等，并智能分析总结关键信息。

## 功能特性

- 📦 **商品ID查询**：支持输入任意商品ID查询详情
- 🖼️ **商品图片**：自动提取前3张商品图片
- 💰 **价格信息**：展示到手价、市场价、折扣信息
- 🏷️ **优惠信息**：展示活动优惠、优惠券信息
- ✅ **正品保障**：展示正品保障、品牌授权等认证信息
- 📝 **用户评论**：展示前2条精华评论内容
- 🎯 **智能分析**：自动提取关键信息并生成分析总结
- 📋 **结构化输出**：脚本输出JSON格式数据，AI格式化展示给用户
- ⚠️ **异常处理**：完善的错误处理和用户提示机制
- 🔑 **登录态支持**：自动读取 vipshop login 的登录态
- 🔄 **自动登录触发**：检测到未登录时自动触发登录流程

**注意**：商品辅助信息接口当前不可用，仅返回商品主信息。

## 工作流程

### 1. 接收输入
- 接受用户输入：商品ID
- 确保商品ID不为空

### 2. 检查登录状态
通过检查文件 `~/.vipshop-user-login/tokens.json` 来验证用户是否有有效的登录令牌：
- 如果已登录，进入步骤3
- 如果未登录（AI **必须自动执行**，不要等待用户请求）：
  1. 提示用户准备启动登录流程
  
  
  2. 自动调用 `vipshop login`生成二维码
  5. 等待用户扫码确认登录
  6. 登录成功后，自动继续执行步骤3

### 3. 执行查询脚本
- 执行：`vipshop product-detail --product-id <product_id>`

### 4. 展示结果
- 解析 JSON 数据并格式化输出
- 展示商品分析总结，包括：
  - 商品图片（前3张）
  - 商品信息（品牌名｜商品名称）
  - 价格信息（到手价、市场价、折扣）
  - 价格计算公式
  - 优惠信息
  - 优惠券信息
  - 服务标签
  - 正品信息
  - 精华评论（前2条）
  - 商品链接

## 实现要求

### 登录状态验证
在执行脚本之前，务必验证用户登录状态：
1. 检查 `~/.vipshop-user-login/tokens.json` 是否存在且包含带有 `cookies` 字段的有效数据
2. 如果用户未登录，自动处理登录流程（安装 skill、生成二维码、等待确认）
3. 仅在确认用户拥有有效登录令牌后才继续执行脚本

### 脚本执行

要查询商品详情，执行 CLI 命令：

```bash
vipshop product-detail --product-id <product_id>
```

**示例：**
```bash
vipshop product-detail --product-id 6921714935983149512
```

### 输出格式化
执行脚本后：
1. 解析 JSON 输出
2. 提取分析总结部分，包括：
   - 商品图片：从 previewImages.imageUrl 字段提取前3张
   - 商品信息：品牌名和商品名称拼接
   - 价格信息：市场价、到手价、折扣
   - 价格计算公式：formula.detail 下的 t 字段拼接
   - 优惠信息：foldTips、svipFoldTips 字段
   - 优惠券信息：foldCoupons 字段
   - 服务标签：serviceTags 字段（去重）
   - 正品信息：commitment4 字段
   - 精华评论：前两条用户评价内容（从 productReputationList.reputation.content 提取）
   - 链接：PC端商品详情链接
3. 向用户显示格式化的结果

## 接口信息

**登录态要求**：
- 使用查询功能前，用户必须先通过 `vipshop login` 登录
- `mars_cid` 作为请求参数从登录态文件读取
- `PASSPORT_ACCESS_TOKEN` 通过 Cookie 方式传入

### 商品主信息接口
- **URL**：`https://mapi-pc.vip.com/vips-mobile/rest/shopping/skill/detail/main/v6`
- **方式**：POST
- **主要参数**：`productId`（商品ID，必需）、`mars_cid`（设备ID）、`scene`（固定为detail）
- **Cookie**：`PASSPORT_ACCESS_TOKEN`（访问令牌）
- **返回**：商品基本信息、价格信息、品牌信息、服务保障、优惠信息等

### 商品辅助信息接口
- **注意**：当前接口不可用，暂不提供尺码表、尺码推荐等辅助信息。

## 注意事项

1. **登录要求**：使用查询功能前必须先通过 `vipshop login` 登录，未登录时系统会自动处理
2. **输出格式**：AI工具优先使用 Markdown 格式输出，提供更好的可读性；如果AI工具不支持 Markdown，则使用普通文本格式
3. **参数固化**：大部分接口参数已固化，无需手动调整
4. **网络要求**：需要正常的网络连接才能调用接口
5. **接口更新**：如果接口参数失效，会提示接口可能已更新
6. **URL编码**：所有中文和特殊字符会自动进行URL编码
7. **字段缺失**：如果某些字段为空，会自动跳过该字段的显示
8. **辅助信息失败**：辅助信息接口失败时，自动降级为仅展示主信息
9. **登录态管理**：脚本会自动读取 `~/.vipshop-user-login/tokens.json` 文件
10. **智能分析**：自动提取关键信息并生成分析总结
10. **原始数据**：同时保留原始数据供调试使用

## 常见问题

**Q: 如何获取商品ID？**
A: 可以通过 vipshop-product-search skill 搜索商品获取商品ID。

**Q: 支持哪些语言的搜索？**
A: 支持中文、英文等多种语言，会自动进行URL编码处理。

**Q: 接口调用失败怎么办？**
A: 检查网络连接是否正常，如果持续失败可能是接口参数已更新，请联系管理员。

**Q: 商品ID有什么用？**
A: 商品ID是唯品会商品的唯一标识，可以用于查询商品详情。

**Q: 为什么有些商品没有显示所有字段？**
A: 某些商品可能缺少某些字段，系统会自动跳过空字段，只显示有值的信息。

**Q: 是否需要登录才能使用？**
A: 是的，必须先通过 `vipshop login` 登录才能使用查询功能。未登录时系统会自动为您处理登录。

**Q: 如何登录？**
A: 使用 `vipshop login` 进行登录。您可以说"登录唯品会"或"唯品会登录"。如果您的系统中没有安装此 skill，系统会自动为您安装。

**Q: 原始数据有什么用？**
A: 原始数据包含完整的接口返回数据，可用于调试和扩展分析。

**Q: 为什么没有尺码表信息？**
A: 商品辅助信息接口当前不可用，因此无法提供尺码表、尺码推荐等信息。
