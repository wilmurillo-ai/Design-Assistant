# 风格记忆 — Style Memory

> AI 生成仪表盘时必须遵循的风格规范。每次生成前读取此文件。

## 用户偏好（推断自对话历史）

- **偏好简洁**：大数字 > 长表格，一句话 > 一段落
- **说人话**：技术术语必须配通俗解释（如 Token → "AI的阅读量"）
- **务实导向**：给建议，不给废话。有问题的东西直接标红 + 给解决方案
- **有观点**：状态判断要有明确结论（OK / ⚠️ / ✕），不要模棱两可

## 展示规范

### 信息优先级（从高到低）

1. **健康状态**：系统是否正常 → 大字 + 颜色
2. **资源压力**：Token / 上下文 → 进度条 + 百分比
3. **活跃任务**：谁在干活 → 状态列表
4. **预警信息**：失败任务 / 高压力 → 醒目警告块
5. **背景信息**：插件 / 技能 / 安全 → 折叠或次要区域

### 说人话翻译表

| 技术术语 | 翻译为 |
|---------|--------|
| Token | AI 的"阅读量"，用越多越慢越贵 |
| Context Window | AI 一次能"记住"的信息量 |
| Session | 一次连续对话（就像聊天的"一局"） |
| Cron Job | 定时任务（闹钟一样，到点自动执行） |
| Subagent | AI 分身（主 AI 派出去干活的小助手） |
| Plugin | 插件（给 AI 加的"APP"） |
| Skill | 技能（AI 的"专业证书"） |
| Gateway | 网关（AI 和外部世界的"大门"） |
| Sandbox | 沙箱（安全隔离的"试验场"） |

## 仪表盘视觉风格

### 颜色系统

| 状态 | 颜色 | CSS 变量 | 用途 |
|------|------|---------|------|
| 健康/正常 | 绿色 `#22c55e` | `--green` | 正常状态、成功、低压力 |
| 注意/预警 | 黄色 `#f5c518` | `--yellow` | 中等压力、待处理、禁用 |
| 错误/危险 | 红色 `#f87171` | `--red` | 失败、高压力、安全风险 |
| 主色/强调 | 青绿 `#00d4aa` | `--accent` | 品牌、活跃、运行中 |
| 信息/次要 | 青色 `#00c9ff` | `--cyan` | 数据指标、网络 |
| AI/认知 | 紫色 `#a78bfa` | `--purple` | Token、模型、记忆 |

### 主题

- **深色主题**：背景 `#0a0f1a`，卡片 `#111827`
- **圆角风格**：14px 卡片，7px 内部元素
- **微光效果**：活跃状态用 box-shadow 发光
- **呼吸动画**：实时指示灯 2s 周期

## 写作风格

### ✅ 要这样

- 直接给结论：「系统正常」「82% 偏高，建议重置」
- 配行动建议：每个问题都要有下一步操作
- 用 Emoji 辅助理解：🌟 系统状态 / 🧠 Token / 👥 任务
- 简洁提示框：一句话解释"怎么看"

### ❌ 不要这样

- 长篇大论的系统介绍（用户已经知道这是什么）
- 模棱两可的措辞（"可能需要关注" → 直接说"建议关注"）
- 密集的技术参数堆砌（人话版隐藏细节，专业版才全展示）
- 无颜色的纯文本状态（必须有颜色区分）
- 超过 3 行的说明文字（最多 2 行 + 链接）

## 禁忌

1. **不要在简洁模式展示原始技术参数**：Job ID、Build Hash 等只在专业模式
2. **不要表格混乱**：简洁模式用列表，专业模式才用表格
3. **不要缺失通俗解释**：每个技术概念至少配一句话解释
4. **不要忽略预警**：任何失败/高压力必须醒目标出
5. **不要静态数据**：每次生成必须用 session_status 实时采集

## 自适应规则

- 如果用户经常用专业模式 → 记住偏好，下次默认专业模式
- 如果用户经常点"刷新" → 缩短自动刷新间隔
- 如果用户在深夜使用 → 简化 UI，减少视觉刺激
- 如果上下文 >90% → 强烈建议重置会话（红色 + 粗体 + 闪烁）
