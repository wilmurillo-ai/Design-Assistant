# Путь внедрения — AI-Лиды PRO ## День 1: Фундамент ### Шаг 1. Заполнить config.yaml
Открыть `config.yaml` и заполнить **честно и точно**:
- `business:` — ниша, продукт, средний чек, гео
- `target:` — кто ваш клиент, боли, триггеры
- `channels:` — текущие каналы, бюджет, цель по лидам
- `economics:` — текущие CPL, CAC, LTV (если знаете) **Критично:** вымышленные данные → вымышленные результаты. Если не знаете CPL — напишите 0, система рассчитает по факту. ### Шаг 2. Построить аватар
```
аватар: [ваша ниша], B2B/B2C, чек [N] руб, гео [регион]
```
Получите: 6 блоков — демография, психография, поведение, путь покупки, ценовой профиль, ключевое сообщение. ### Шаг 3. Найти каналы
```
где брать лидов для [ниша], бюджет [N] руб/мес, цель [N] лидов/мес
```
Получите: 10+ каналов с CPL, ROI, конкретными действиями и планом на 3 месяца. ### Шаг 4. Первый лид-магнит
```
лид-магнит для [аудитория из аватара], канал [telegram/email]
```
Получите: концепция + механика раздачи + цепочка касаний после скачивания. **Итог дня 1:** аватар + каналы + лид-магнит. Фундамент системы. --- ## Неделя 1: Первые касания ### Шаг 5. Первое касание
```
первое касание: [контекст — где нашли лида, что знаем о нём]
```
5 шаблонов: холодный TG, email, после сигнала, по рекомендации, follow-up. ### Шаг 6. Сегментация базы
```
сегментируй базу [N] контактов: [описание — сколько горячих, тёплых, холодных]
```
BANT-баллы + температура + действие для каждого сегмента + фокус недели. ### Шаг 7. Квалификация горячих
```
квалифицируй: [описание лида — кто, откуда, что сказал, бюджет, сроки]
```
Для каждого горячего лида: BANT-скоринг 0-100 + следующий шаг. ### Шаг 8. Запуск прогрева
```
прогрей: [сегмент — сколько контактов, температура, продукт, канал]
```
Цепочка 14 дней: 4 фазы AIDA с конкретными текстами и триггерами. ### Шаг 9. Первая автоворонка
```
автоворонка для [продукт], цена [N], канал [telegram], источник [каналы]
```
6 этапов: захват → знакомство → прогрев → предложение → оплата → cross-sell. **Итог недели 1:** работающая система — касания идут, лиды квалифицированы, прогрев запущен, воронка построена. --- ## Месяц 1: Аналитика и оптимизация ### Шаг 10. Unit-экономика
```
экономика лидов: [данные по каналам — бюджет, лиды, клиенты, выручка]
```
CPL, CAC, LTV/CAC, ROI по каждому каналу + вердикт: масштабировать/держать/отключить. ### Шаг 11. CEO-отчёт
```
отчёт лиды за [период]: [данные воронки]
```
7 блоков: воронка, unit-экономика, каналы, динамика, узкое место, прогноз, топ-3 действия. ### Шаг 12. Реактивация базы
```
реактивируй: [описание спящей базы]
```
5 стратегий + план по дням + прогноз конверсии. Дополнительные сделки без бюджета на рекламу. ### Шаг 13. A/B тест
```
ab тест: [что тестируем — тема, креатив, CTA, время]
```
Гипотеза + дизайн + выборка + план запуска + критерий победы. ### Шаг 14. Корректировка каналов
По результатам unit-экономики:
- Убыточные каналы → отключить или оптимизировать
- Прибыльные каналы → увеличить бюджет
- Новые каналы → запустить на тест (10% бюджета) **Итог месяца 1:** знаете стоимость лида и клиента по каждому каналу, первый прогноз выручки, слабые каналы отключены, сильные масштабируются. --- ## Квартал 1: Масштабирование ### Что должно работать к концу квартала 1. **Все каналы протестированы A/B.** - CPL оптимизирован (снижение 30-50% от старта) - Минимум 4 A/B теста завершены 2. **Автоворонка генерирует клиентов автономно.** - Лид-магнит → прогрев → оффер → оплата — без ручного участия - Собственник подключается только к горячим лидам (BANT 70+) 3. **Unit-экономика стабильна.** - LTV/CAC ≥ 3x по всем активным каналам - Pipeline покрытие ≥ 3x от плана 4. **Масштабирование предсказуемо.** - Знаете: вложить X руб → получить Y лидов → Z клиентов - Прогноз выручки ±15% 5. **CEO-отчёт — еженедельный ритуал.** - Воронка + unit-экономика + прогноз — каждый понедельник - Решения по каналам — на цифрах, не на ощущениях --- ## Чек-лист готовности (самопроверка) | # | Критерий | Неделя |

| 12 | Прогноз выручки ±15% | Квартал 1 | --- ## Частые ошибки при внедрении 1. **Пропустить config.yaml.** Без настройки система работает «в среднем по больнице». С настройкой — под ваш бизнес. 2. **Пропустить прогрев.** Отправить лид-магнит и сразу оффер = потерять 70% лидов. Прогрев 14 дней — обязателен. 3. **Не считать unit-экономику.** Без CPL/CAC/ROI по каналам вы не знаете что масштабировать. Это как ехать без спидометра. 4. **Прогревать всех одинаково.** Горячий лид (BANT 70+) не нужно прогревать 14 дней — ему нужен звонок сейчас. Сегментация обязательна. 5. **Не реактивировать базу.** Мёртвые контакты = деньги на полу. 3-5% базы конвертируется при правильной реактивации — и это бесплатно.
