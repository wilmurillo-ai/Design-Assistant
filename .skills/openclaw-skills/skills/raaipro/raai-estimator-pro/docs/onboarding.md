# AI-Сметчик PRO — Путь внедрения Пошаговый план от установки до системного сметного контура.
Каждый этап — фундамент для следующего. --- ## День 1: Установка и первая эталонная смета **Время: 2–3 часа** ### Шаг 1: Заполнить config.yaml (30–45 минут) Обязательные поля на старте:
- `region`: регион, ТЕР-код, коэффициенты
- `normatives`: нормативная база, НДС, непредвиденные
- `pricing.control_rates`: контрольные расценки под ваш регион и тип работ
- `typical_objects`: 2–3 типовых объекта с площадью и составом работ
- `company`: тип компании, основной тип объектов **Правило:** контрольные расценки ставьте реальные для вашего города — они используются для аудита и проверки. ### Шаг 2: Первая эталонная смета (60–90 минут) Возьмите самый типовой объект (например, квартира 60 м²).
Команда: `смета` Что произойдёт:
- система попросит состав работ и объёмы
- посчитает ПЗ → НР → СП → НДС
- покажет структуру и контрольные расценки Сохраните эту смету как эталон для будущих объектов. ### Шаг 3: Первый аудит подрядной сметы (30 минут) Возьмите любую смету от подрядчика.
Команда: `проверь смету` Система найдёт: завышенные расценки, дубли, некорректные НР/СП. **Результат дня 1:** config настроен, эталонная смета готова, первый аудит проведён. --- ## Неделя 1: Формирование документального контура **День по дням:** | День | Задача | Результат |
|---|---|---|
| Пн | config + эталонная смета | Базовая настройка |
| Вт | `дефектовка` по типовому объекту | Шаблон дефектной ведомости |
| Ср | `объёмы` по квартире | Шаблон объёмной ведомости |
| Чт | `кс-2` + `кс-3` по тестовому объекту | Шаблоны актов |
| Пт | `тендер` / `кп` — первый тендерный пакет | Шаблон КП | **Что должно произойти к концу недели 1:**
- библиотека шаблонов под типовые объекты
- первый аудит подрядной сметы с цифрами завышений
- время на подготовку типовой сметы сократилось минимум вдвое --- ## Месяц 1: Система начинает работать ### Неделя 2: Углубление - Аудит 2–3 смет подрядчиков — уже с отчётом по замечаниям
- `сравни` — сравнение двух вариантов материалов или подрядчиков (TCO)
- `материалы` — ведомость материалов с запасом и стоимостью ### Неделя 3: Тендерный контур - `тендер` — подготовить КП на реальный тендер
- Добавить 2 варианта цены: эконом и оптимал
- Ссылки на нормативы в КП — усиление позиции ### Неделя 4: Обзор и стандартизация - Проверить: где по-прежнему делаем вручную?
- Зафиксировать типовые шаблоны под каждый класс объектов
- Первый отчёт: сколько часов сэкономили, сколько завышений нашли ### Чек-лист конца месяца 1: - [ ] Эталонные сметы по 2–3 типам объектов готовы
- [ ] Шаблоны КС-2, КС-3, дефектовки, КП — сохранены
- [ ] Аудит подрядчиков стал стандартной процедурой
- [ ] Время на смету сократилось ≥50%
- [ ] Первый тендерный пакет с обоснованием цены --- ## Квартал 1: Системный сметный контур ### Месяц 2: Зрелость - Все 10 режимов освоены и используются по типу задачи
- Объёмная ведомость и дефектовка — стандарт ДО подписания акта
- Библиотека: 5+ типовых объектов с шаблонными позициями
- Аудит подрядчиков: проверяем каждую смету >500 000 руб ### Месяц 3: Результат - Процент выигранных тендеров: замерить и сравнить с периодом до
- Найденные завышения: посчитать суммарную экономию
- Время на смету: сравнить с первым месяцем ### Чек-лист конца квартала 1: - [ ] Среднее время сметы сократилось ≥60% против старта
- [ ] Найдено завышений от подрядчиков минимум на сумму продукта × 3
- [ ] КС-2 и КС-3 готовятся за 30 минут, не за день
- [ ] Процент выигранных тендеров вырос или количество участий выросло
- [ ] Нет ни одного акта «задним числом» — документы ведутся по ходу --- ## Типичные ошибки первого месяца | Ошибка | Почему плохо | Что делать |
|---|---|---|
| Не заполнить контрольные расценки | Аудит без базы = только структурные ошибки | Потратить 20 минут на control_rates |
| Считать без объёмов | Смета — приблизительная | Всегда начинать с объёмной ведомости |
| Не делать аудит каждой сметы | Завышения копятся в каждом объекте | Аудит >500К — обязателен |
| Использовать только режим `смета` | 1 из 10 режимов | Добавлять по 2 режима в неделю |
| КП без обоснования | Заказчик видит цифру, не видит логику | Всегда добавлять ссылки на нормы |
| Дефектовка после приёмки | Уже поздно — конфликт неизбежен | Дефектовка до акта, не после | --- ## Рекомендуемый ритм (после первого месяца) ```
НА КАЖДЫЙ ОБЪЕКТ: Старт → объёмная ведомость → смета → КП По ходу → дефектовка → акты скрытых работ Завершение → КС-2 / КС-3 → сдача документации ДЛЯ КАЖДОГО ПОДРЯДЧИКА: Смета пришла → аудит → переговоры по замечаниям → подписание ДЛЯ КАЖДОГО ТЕНДЕРА: Запрос пришёл → быстрая смета (1 день) → КП с 2 вариантами → подача ЕЖЕМЕСЯЧНО: — Обзор выигранных тендеров и найденных завышений — Обновление контрольных расценок (рынок меняется) — Обновление библиотеки типовых смет
``` --- *AI-Сметчик PRO | RAAI | Путь внедрения*
