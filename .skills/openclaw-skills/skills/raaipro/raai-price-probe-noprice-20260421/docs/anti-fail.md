# AI-Поддержка PRO — Ограничения и честные границы Что продукт НЕ делает. Где он НЕ поможет. Что нужно на вход.
Честность = доверие. Доверие = высокий чек. --- ## Где AI-Поддержка PRO НЕ поможет ### 1. Не заменяет руководителя поддержки AI-Поддержка PRO — это операционная система. Но решения по политике сервиса принимает человек. Система НЕ заменяет:
- принятие решений по нестандартным кейсам;
- формирование политики компенсаций и возвратов;
- управление конфликтами с VIP-клиентами в сложных ситуациях;
- мотивацию и развитие команды операторов. Система ЗАМЕНЯЕТ:
- рутинную категоризацию;
- ручной SLA-контроль;
- ручное отслеживание angry clients;
- повторяющийся онбординг новых операторов. ### 2. Не работает без реального SLA Самый частый паттерн провала: ```
"У нас SLA — отвечаем в течение рабочего дня."
"А как замеряете?" — "Ну, примерно отслеживаем."
``` Если SLA существует только на словах:
- нет baseline для улучшения;
- система не знает, что считать breach;
- daily report не имеет смысла. **Что нужно:** хотя бы приблизительные SLA-цели по типам обращений (critical, high, medium, low). ### 3. Не работает без категоризации тикетов Автоматическая категоризация работает на обученных паттернах. Но:
- если исторические тикеты не классифицированы — стартуем с нуля;
- если бизнес-специфика очень нестандартная — нужна настройка;
- если операторы сами не используют категории — аналитика неточная. ### 4. Не решает кадровые проблемы поддержки Система НЕ поможет, если:
- операторы не соблюдают стандарты (это HR-вопрос);
- L2 саботирует эскалации;
- нет ответственного за L3. Эскалация работает, когда на каждом уровне есть реальный owner. ### 5. Не обрабатывает данные о клиентах без CRM Режим `карточка клиента` и полный контекст обращения работают при наличии CRM-данных.
Без интеграции система работает только с текстом обращения — без истории LTV и предыдущих тикетов. ### 6. Не гарантирует NPS-рост сам по себе Поддержка — один из факторов NPS. Если проблема в продукте, цене или доставке — поддержка не вытянет NPS в одиночку. AI-Поддержка PRO улучшает компоненту поддержки в NPS. Остальное — за продуктом и бизнесом. --- ## Что нужно на вход ### Минимальные требования: | Требование | Почему обязательно |
|---|---|
| 100+ тикетов в месяц | Нужен поток для категоризации и статистики |
| Назначенный L2 и L3 owner | Без них эскалации уходят в никуда |
| Хотя бы примерные SLA-цели | Без них нет baseline и нет контроля |
| Готовность вводить данные по тикетам | Без данных нет категоризации |
| Ежедневный ревью отчётов руководителем | Без этого метрики не превращаются в действия | ### Что улучшает результат (но не обязательно): - CRM-интеграция для доступа к истории клиента и LTV
- Исторические данные по тикетам (тип, время решения, оценка)
- Зрелая команда с опытом работы по SLA
- NPS-инфраструктура для замера качества --- ## Честные границы по каждому режиму | Режим | Что делает хорошо | Где ограничен |
|---|---|---|
| Категоризация | Быстро классифицирует типовые обращения | Нестандартные кейсы требуют ручного тюнинга |
| SLA-контроль | Показывает риски в реальном времени | Нужны заполненные SLA-параметры |
| Эскалация | Применяет правила автоматически | Работает только при назначенных L2/L3 |
| Sentiment | Определяет тон и злость | Граничные случаи могут ошибаться |
| Ответ клиенту | Генерирует в правильном тоне | Специфика бизнеса требует настройки шаблонов |
| Возвраты | Применяет правила политики | Нужно заполнить политику возвратов в config |
| NPS отчёт | Показывает тренды и причины | Требует накопленных данных (2-4 недели) |
| Онбординг оператора | Даёт чек-лист и стандарт | Не заменяет наставника в первые дни | --- ## Резюме: формула успеха ```
Результат = Дисциплина данных × Вовлечённость руководителя × Системность использования Где: Дисциплина данных = реальные SLA + чистые категории + назначенные owner'ы Вовлечённость руководителя = ежедневный report + еженедельный ревью Системность = все 10 режимов в рабочем ритме Если хотя бы один множитель ≈ 0, результат ≈ 0.
``` AI-Поддержка PRO — честный продукт. Он не обещает, что клиенты перестанут злиться. Он обещает, что злость будет захватываться вовремя и не станет репутационным риском. --- *AI-Поддержка PRO | RAAI | Честные границы продукта*
