# AI-Прораб PRO — Путь внедрения Пошаговый план от установки до системного управления объектом.
Не спешить. Каждый этап — фундамент для следующего. --- ## День 1: Установка и первый план дня **Время: 1.5–2 часа** ### Шаг 1: Заполнить config.yaml (30–45 минут) Обязательные поля на старте:
- `object`: название, тип, площадь, стадия, дедлайн, критический путь
- `team`: прораб, бригада, специальности, субподрядчики
- `quality`: стандарт, основные нормы, допуски
- `safety`: ТБ-требования, список работ повышенной опасности
- `supplies`: поставщики, сроки поставки, требования к сертификатам **Правило:** заполняйте честно. Объект с «площадь = 0» и «этапы = нет» не даёт нужного контура. ### Шаг 2: Первый план дня (15 минут) Команда: `план дня` Что получите:
- приоритеты по зонам и этапам на сегодня
- кто что делает (owner по задаче)
- риски на сегодня ### Шаг 3: Базовый журнал (10 минут) Команда: `журнал` Заполните первую запись по КС-6а: что выполнялось сегодня, кто, объём, замечания. **Результат дня 1:** config настроен, первый план есть, журнал начат. --- ## Неделя 1: Выстраивание документального ритма **Ежедневно:** | День | Задача | Результат |
|---|---|---|
| Пн | `план дня` + `журнал` | Первый день по ритму |
| Вт | `входной контроль` (если поставка) | Принимаем материал по норме |
| Ср | `акт скрытых работ` (если есть закрываемый этап) | Первый акт зафиксирован |
| Чт | `безопасность` — разбор ситуации по ТБ | Риски на площадке |
| Пт | `отчёт прораба` — первый недельный отчёт | Статус, риски, следующий шаг | **Что должно произойти к концу недели 1:**
- все задачи на объекте имеют owner'а
- журнал КС-6а заполняется ежедневно
- первый акт скрытых работ подписан до закрытия этапа
- входной контроль проведён хотя бы для 1 партии
- первый недельный отчёт по статусу объекта --- ## Месяц 1: Система начинает работать ### Неделя 2: Расширение контура - `гант` — построить Гант по объекту с зависимостями
- `субподрядчик` — синхронизировать расписание внешних исполнителей
- `приёмка` — первая приёмка этапа по чек-листу с допусками ### Неделя 3: Углубление - `бригада` — наряд-задание с измеримым результатом и дедлайном
- Проверить: все ли скрытые работы фиксируются? (Нет пропусков?)
- `исполнительная документация` — чек-лист текущей укомплектованности ### Неделя 4: Итоги месяца - `отчёт прораба` — полный отчёт за месяц
- Проверить: процент переделок vs плановый
- Обновить Гант с учётом фактического прогресса ### Чек-лист конца месяца 1: - [ ] Журнал КС-6а: 0 пропущенных рабочих дней
- [ ] Акты скрытых работ: все критичные этапы зафиксированы
- [ ] Входной контроль: каждая партия материалов
- [ ] Гант обновлён — реальный прогресс vs план
- [ ] Ни одного простоя бригады из-за поставок
- [ ] Прораб тратит <30 мин/день на систему --- ## Квартал 1: Полный операционный контур ### Месяц 2: Зрелость - Все 10 режимов используются по типу ситуации
- Приёмка по нормам — стандарт, а не исключение
- Исполнительная документация актуальна всегда
- Споры с заказчиком решаются за 1 день по документам, а не за неделю на эмоциях ### Месяц 3: Результат - Сдача объекта: все документы готовы ЗА ДЕНЬ до приёмки
- Итоговый рапорт: переделки, штрафы, простои vs прогноз
- Анализ: что было сделано хорошо, что улучшить на следующем объекте ### Чек-лист конца квартала 1: - [ ] rework_percent ≤ 3% (целевой)
- [ ] 0 штрафов за задержку сдачи
- [ ] Исполнительная документация полностью готова к сдаче
- [ ] Заказчик подписал акт в день приёмки (без задержек по документам)
- [ ] Бригады работают по плану, а не по устным указаниям прораба --- ## Типичные ошибки первого месяца | Ошибка | Почему плохо | Что делать |
|---|---|---|
| Акты скрытых работ «потом» | При сдаче нет документов → вскрытие → конфликт | До закрытия этапа, не после |
| Входной контроль «потом» | Брак смонтирован → дорогая переделка | При приёмке партии, сразу |
| Журнал КС-6а 1 раз в неделю | Пропуски — юридическая дыра | Каждый рабочий день |
| Задачи без owner'а | «Думали, что другой сделает» | У каждой задачи — один owner + дедлайн |
| Приёмка «нормально выглядит» | Спор при сдаче без критерия | Всегда с чек-листом по СП |
| Отчёт прораба «когда попросят» | Руководство узнаёт о проблемах постфактум | Еженедельно, независимо от запроса | --- ## Рекомендуемый ритм (после первого месяца) ```
ЕЖЕДНЕВНО (15–20 мин): Утро: `план дня` — приоритеты, люди, риски Вечер: `журнал` — запись по КС-6а ПО СОБЫТИЮ (5–15 мин): При поступлении материала → `входной контроль` При закрытии скрытого этапа → `акт скрытых работ` При несоответствии на приёмке → `приёмка` + фиксация При риске ТБ → `безопасность` ЕЖЕНЕДЕЛЬНО (30–40 мин): Пятница → `отчёт прораба` + обновление Ганта Пятница → `исполнительная документация` чек-лист ПРИ СДАЧЕ ЭТАПА / ОБЪЕКТА: `приёмка` с чек-листом → замечания → акт Комплект документов — готов заранее, не в последний день
``` --- *AI-Прораб PRO | RAAI | Путь внедрения*
