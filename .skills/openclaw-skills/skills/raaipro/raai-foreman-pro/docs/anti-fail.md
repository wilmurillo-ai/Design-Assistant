# AI-Прораб PRO — Ограничения и честные границы Что продукт НЕ делает. Где он НЕ поможет. Что нужно на вход.
Строительство — физический процесс. AI усиливает управление, но не заменяет присутствие. --- ## Где AI-Прораб PRO НЕ поможет ### 1. Не заменяет прораба на площадке Самое важное ограничение. AI-Прораб PRO — это система управления информацией об объекте.
Физический контроль — это прораб, который вышел, посмотрел, замерил, подписал. Система не может:
- выйти на объект и проверить качество стяжки рейкой
- остановить опасные работы
- принять материал и проверить партию
- подписать акт скрытых работ Без физического присутствия прораба все режимы системы работают на 30–40% эффективности. ### 2. Не исправляет хаос без owner'ов Если задачи ставятся без ответственного — система не поможет.
Правило простое: у каждой задачи на объекте должен быть один owner с именем и дедлайном. «Сделайте» → не работает.
«Иванов, штукатурка зала, завершить до пятницы 18:00» → работает. ### 3. Не лечит игнорирование ТБ Система даёт:
- чек-листы по ТБ для конкретного вида работ
- напоминания об обязательных инструктажах
- алгоритм действий при инциденте Система НЕ делает:
- не заставляет надевать каску
- не останавливает работника, игнорирующего инструкцию
- не несёт ответственности за несчастный случай ТБ — это дисциплина, а не функция AI. ### 4. Не работает без фиксации фактов Garbage in → garbage out в строительстве: ```
Скрытые работы выполнены → акт не подписан
Материал принят → в журнал не внесён
Этап завершён → в журнале КС-6а пропуск
``` При сдаче объекта: нет записи = работа юридически не существует.
Заказчик вправе потребовать подтверждения. ### 5. Не предотвращает конфликт, который уже начался Система помогает предотвратить конфликты через документацию.
Но если конфликт уже начался (заказчик отказался подписывать, идёт претензионная работа) — нужен юрист, не AI. Система покажет: что было зафиксировано, что нет. Это аргументы. Остальное — переговоры. ### 6. Не заменяет проектную документацию AI-Прораб PRO работает с управлением объектом, а не с проектированием. Систему не нужно использовать для:
- разработки проекта или рабочих чертежей
- расчёта конструкций и нагрузок
- согласований с надзорными органами
- сложных инженерных решений --- ## Garbage in → Garbage out Самый опасный паттерн: ```
Прораб: «некогда заполнять — потом внесём»
Неделя 1: 3 пропуска в журнале
Неделя 3: акты скрытых работ не велись
Месяц 2: «потом» не наступает
Сдача: нет документов → конфликт → штраф
``` «Потом» в строительстве не существует. Только «сейчас» или «проблема при сдаче». **Правило:** занять 10 минут сегодня → сэкономить 3 дня на сдаче. --- ## Что нужно на вход ### Минимальные требования: | Требование | Почему обязательно |
|---|---|
| Ответственный прораб — физически на объекте | Без него нечего фиксировать |
| Объект с этапами и сроками | Нет этапов — нет критического пути |
| Бригада / субподрядчики | Нужны owner'ы задач |
| Стандарт качества определён | Приёмка без критерия = спор |
| Готовность вести журнал ежедневно | Без этого документация пустая | ### Что улучшает результат: - Фотоотчёт по ходу работ (особенно для скрытых)
- Данные поставок в config (планирование без простоев)
- Реальные сроки в Ганте (не «хотелось бы», а «реально»)
- Заказчик, который участвует в подписании актов --- ## Антипаттерны управления объектом ### «Документы ведём по завершении» При завершении большого объекта не помнят 60% деталей.
Акты скрытых работ задним числом — юридически слабые.
Журнал КС-6а задним числом — видно, что заполнен одним днём. Документация ведётся по ходу. Это 10 минут ежедневно, а не 3 недели в конце. ### «Каска — не обязательно, мы аккуратные» Один несчастный случай = остановка объекта + уголовная ответственность прораба.
Стоимость комплекта СИЗ: 2 000–5 000 руб на человека.
Стоимость одного НС: миллионы + репутация. ### «Принимаем материал на глаз» Не та партия плитки (другой тон) — узнали когда уложили 40% площади.
Переделка: снять, заказать новую партию, положить.
Стоимость: 80 000–150 000 руб + 10 дней задержки. Входной контроль — 20 минут при поступлении партии. ### «Задача поставлена в чат — значит выполнят» В чате теряются 30–40% задач. Нет дедлайна — нет срочности.
Каждая задача: owner + дедлайн + измеримый результат. Иначе — будет переспрошено. --- ## Честные границы по каждому режиму | Режим | Что делает хорошо | Где ограничен |
|---|---|---|
| Управление бригадой | Наряд-задание с owner'ом и критерием | Исполнение всё равно требует физического контроля |
| Приёмка | Чек-лист по СП/ГОСТ с допусками | Нужны реальные замеры — не «на глаз» |
| Журнал КС-6а | Структура записи, шаблон | Прораб должен ввести факты ежедневно |
| Акты скрытых работ | Шаблон, перечень обязательных | Нужны подписи обеих сторон в момент выполнения |
| Входной контроль | Чек-лист, что проверять | Физическая проверка партии — за прорабом |
| Безопасность | Алгоритм по ситуации | Исполнение правил ТБ — за людьми |
| Гант | Критический путь, зависимости | Нужны реальные сроки, не оптимистичные |
| Отчёт прораба | Структура и визуализация | Нужны реальные данные от прораба | --- ## Резюме: формула успеха ```
Результат = Дисциплина прораба × Качество фиксации × Регулярность системы Где: Дисциплина = ежедневный журнал + акты в момент выполнения Качество фиксации = реальные данные, не "потом уточним" Регулярность = план дня каждое утро + отчёт каждую пятницу Если хотя бы один множитель ≈ 0, результат ≈ 0.
``` AI-Прораб PRO — честный инструмент.
Он не обещает стройку без проблем. Он обещает, что проблемы будут видны заранее — и можно успеть среагировать. --- *AI-Прораб PRO | RAAI | Честные границы продукта*
