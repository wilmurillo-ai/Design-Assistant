# AI-офис PRO — Путь внедрения Пошаговый план от установки до полной системы управления.
Не спешить. Каждый этап — фундамент для следующего. --- ## День 1: Установка и первая стратегическая сессия **Время: 2-3 часа** ### Шаг 1: Заполнить config.yaml (30-45 минут) Это самый важный шаг. Чем точнее заполнен config — тем точнее работает система. Обязательные поля:
- Профиль CEO (имя, роль, стиль управления, зона перегруза)
- Компания (стадия, модель, выручка)
- Финансы (текущая и плановая выручка, маржа)
- Команда (кто подчиняется CEO напрямую)
- Стратегия (годовая цель, узкие места, конкурентное преимущество)
- Расписание (когда брифинг, когда weekly review) **Правило:** если не знаете точную цифру — поставьте приблизительную. Лучше примерно, чем пусто. ### Шаг 2: Стратегическая сессия — постановка OKR (60-90 минут) Отправить: `стратегическая сессия` Что произойдёт:
- Система проведёт вас через анализ текущей ситуации
- Поможет сформулировать 2-3 Objectives на квартал
- Декомпозирует каждый до 3-4 Key Results
- Разобьёт KR на недельные задачи **Результат дня 1:** config заполнен, OKR квартала поставлены, декомпозированы. ### Шаг 3: Первый брифинг (5 минут) Отправить: `брифинг` Даже если данных мало — увидите формат. С каждым днём брифинг будет наполняться. --- ## Неделя 1: Выстраивание утреннего ритма **Ежедневно (5-10 минут утром):** | День | Что делать | Что получаете |
|---|---|---|
| Пн | `брифинг` + `приоритеты` (список задач на неделю) | Фокус дня + матрица задач |
| Вт | `брифинг` | Метрики + решения + фокус |
| Ср | `брифинг` + `делегировать` (1-2 задачи) | Первый опыт с системой делегирования |
| Чт | `брифинг` + `зафиксировать решение` (любое решение дня) | Первая запись в Decision Log |
| Пт | `weekly review` | Первый полноценный EOS-ревью | **Что должно произойти к концу недели 1:**
- Утренний брифинг стал привычкой (3 минуты, каждый день)
- OKR квартала поставлены и декомпозированы
- 1-2 задачи делегированы с checkpoint'ами
- 2-3 решения зафиксированы в Decision Log
- Первый weekly review проведён **Чего НЕ нужно делать на неделе 1:**
- Не пытаться использовать все 9 режимов сразу
- Не перегружать систему данными — начните с главного
- Не ожидать идеальных метрик — они наберутся --- ## Месяц 1: Система начинает работать ### Недели 2-4: углубление **Неделя 2:**
- Брифинг ежедневно (уже привычка)
- Матрица Эйзенхауэра 2-3 раза в неделю
- Первый checkpoint по делегированным задачам
- Второй weekly review — уже с данными прошлой недели для сравнения **Неделя 3:**
- Трекинг OKR: отправить `трекинг OKR` — увидеть прогресс за 3 недели
- Если KR отстаёт — первая коррекция курса (это нормально и правильно)
- Делегировать ещё 3-5 задач с checkpoint'ами
- Decision Log: 8-10 записей к этому моменту **Неделя 4:**
- Weekly Review №4 — уже видны тренды (4 недели данных)
- Первая приёмка делегированной задачи (режим 7.3)
- Оценить: что из системы работает, что пока не нужно ### Чек-лист конца месяца 1: - [ ] Утренний брифинг — каждый рабочий день
- [ ] Weekly Review — каждую пятницу
- [ ] OKR трекаются еженедельно
- [ ] 3+ задачи делегированы с checkpoint'ами
- [ ] 10+ решений в Decision Log
- [ ] Матрица Эйзенхауэра используется 2-3 раза в неделю
- [ ] CEO тратит на систему 30-40 минут/день (не больше) --- ## Квартал 1: Полная система ### Месяц 2: зрелость - Все 9 режимов доступны и понятны
- Делегирование с checkpoint'ами — стандартный процесс (команда привыкла)
- Decision Log: 20-30 записей. Первые ревью решений (30 дней)
- OKR: ясно видно, какие KR достигаются, какие нет
- Если есть инвестор — первый Investor Update через систему ### Месяц 3: результат - Последний weekly review квартала — полная картина: что сработало, что нет
- Decision Log ревью: какие решения были сильными, какие ошибочными
- Стратегическая сессия на Q2 (или следующий квартал): НА ДАННЫХ **Это ключевой момент.** Первая стратегическая сессия на основе данных (OKR прогресс + Decision Log + KPI тренды) — качественно другой уровень планирования. ### Чек-лист конца квартала 1: - [ ] OKR квартала ≥70% выполнены (или понятно почему нет)
- [ ] 30+ решений в Decision Log, 5+ ревью проведены
- [ ] Делегирование с checkpoint'ами — норма, не исключение
- [ ] CEO тратит 30-40 мин/день на систему
- [ ] CEO освободил 3-5 часов/день от операционки
- [ ] Стратегическая сессия Q2 проведена на данных
- [ ] Команда знает ритм: брифинг, checkpoint, weekly review --- ## Типичные ошибки первого месяца | Ошибка | Почему плохо | Что делать |
|---|---|---|
| Пропускать weekly review | Без него OKR деградируют за 2-3 недели | Поставить в календарь как неприкосновенный слот |
| Заполнить config.yaml формально | Система даёт шаблоны, а не решения | Потратить 45 минут на честное заполнение |
| Использовать только брифинг | 1 из 9 режимов = 11% системы | Добавлять по 1 режиму в неделю |
| Не вводить реальные данные | Брифинг без метрик = пустой шаблон | Даже приблизительные данные лучше нуля |
| Делегировать без критериев | Checkpoint бессмысленен без критерия успеха | Каждая задача = измеримый результат |
| Ожидать магию без дисциплины | Система не работает сама — работает CEO | Первые 2 недели = формирование привычки | --- ## Рекомендуемый ритм (после первого месяца) ```
ЕЖЕДНЕВНО (5-10 мин): 09:00 — Брифинг ПО НЕОБХОДИМОСТИ (5-15 мин каждый): — Делегирование (по мере появления задач) — Decision Log (при значимых решениях) — Матрица Эйзенхауэра (при завале) — Стратегические приоритеты (при потере фокуса) ЕЖЕНЕДЕЛЬНО (30-45 мин): Пятница 16:00 — Трекинг OKR Пятница 17:00 — Weekly Review ЕЖЕКВАРТАЛЬНО (3-4 часа): Последняя неделя квартала — Стратегическая сессия (опционально) — Investor Update
``` --- *AI-офис PRO | RAAI | Путь внедрения*
