# Quick Start — 3 готовых промпта для первого запуска После установки (`install.sh` + заполнение 5 OBLIGATORY-полей в `config.yaml`) — проверьте коробку этими 3 триггерами в вашем боте/агенте. **Важно:** если в конфиге не заполнены интеграции (Google Sheets / Calendar), бот честно скажет «данных нет, нужен ввод». Это **не баг** — так работает 7 из 9 режимов без внешних подключений. --- ## 1. `брифинг` — утренний брифинг CEO **Что отправляете боту:**
```
брифинг
``` **Что ожидаете в ответе (5 блоков):**
```
1. Метрики вчера • Выручка день/MTD • Воронка (лиды/сделки) • Ключевые сделки 2. Критические решения на сегодня • Контекст • Варианты A/B • Рекомендация 3. Топ-3 приоритета дня 1. ... 2. ... 3. ... 4. Риски и возможности 5. Календарь
``` **Если данных нет** — бот выдаст структуру с пометкой «данных нет, нужен вход в Google Sheets». Это ожидаемое поведение. --- ## 2. `цели на квартал` — OKR-система **Что отправляете:**
```
цели на квартал
``` **Что ожидаете (по Doerr — 3 Objective × 5 KR):**
```
Objective 1: [качественная цель]
Обоснование: [почему]
Владелец: [имя] KR 1.1: [метрика X → Y к дате], текущее Z, %
KR 1.2: ... Статус: ON TRACK / AT RISK / OFF TRACK
``` Бот НЕ спрашивает «утверждаем?» — сразу выдаёт каркас + 3 задачи недели с владельцами и дедлайнами. --- ## 3. `зафиксировать решение` — Decision Log по Bezos **Что отправляете:**
```
зафиксировать решение
``` **Что ожидаете — шаблон полей (не лекцию о концепции):**
```
Decision Log
ID: DL-001
Дата: [сегодня]
Тип: 1 (необратимое) / 2 (обратимое)
Контекст: —
Варианты с компромиссами:
- A) ...
- B) ...
Выбор: —
Ожидаемый результат: —
Критерии успеха: —
Дата ревью: +30 дней (тип 2) / +90 дней (тип 1) Какое решение фиксируем?
``` **Мета-запрос:** если напишете `зафиксировать решения по всем дырам` — бот сначала перечислит список дыр, потом выдаст отдельный Decision Log-блок на КАЖДУЮ. --- ## Что делать если не работает 1. Проверьте smoke-test: `bash test/smoke-test.sh` — должно быть PASS.
2. Проверьте что в config.yaml заполнены 5 OBLIGATORY полей.
3. Проверьте что триггерные слова написаны **точно как в SKILL.md** (см. раздел triggers).
4. См. `docs/anti-fail.md` — типовые ошибки. Полная библиотека примеров — `examples/example_prompts.md` (46 КБ, 9 режимов со всеми сценариями).
