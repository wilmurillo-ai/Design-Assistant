# AI-Аналитик бизнеса PRO — Ограничения и честные границы Что продукт НЕ делает. Где он НЕ поможет. Что нужно на вход.
Честность = доверие. Доверие = высокий чек. --- ## Где AI-Аналитик бизнеса PRO НЕ поможет ### 1. Не починит плохой учёт AI-Аналитик бизнеса PRO — это аналитическая надстройка, а не система учёта. Если расходы не ведутся или ведутся частично — маржа будет неверной.
Если выручка учитывается «примерно» — дашборд покажет «примерно».
Если данные о клиентах разрознены — unit-экономика будет нечёткой. **Аналогия:** навигатор не поможет, если карта неправильная. Система не может быть точнее данных, которые в неё вводят. ### 2. Не работает без ответственного за цифры Для работы системы нужен человек, который:
- вводит данные регулярно (или настроил интеграцию)
- отвечает за качество вводных
- использует аналитику при принятии решений Если нет owner'а по цифрам — система деградирует за 2–3 недели. ### 3. Не заменяет CFO и финансовую функцию Система делает:
- расчёт unit-экономики, BEP, LTV/CAC
- построение прогнозов и финмоделей
- анализ воронки и когорт
- investor-ready отчётность Система НЕ делает:
- бухгалтерский учёт
- управление дебиторкой и кредиторкой
- налоговое планирование
- юридически значимые документы AI-Аналитик бизнеса PRO — управленческая аналитика, а не бухгалтерия. ### 4. Не гарантирует рост без действий Аналитика показывает картину. Решения принимает CEO. Если воронка показала узкое место, но никто не меняет скрипт продаж — выручка не вырастет.
Если CAC по каналу убыточный, но бюджет не перераспределяется — деньги продолжат гореть. Система = зеркало. Что с этим делать — за CEO. ### 5. Не работает для бизнеса без регулярной выручки Когортный анализ, retention, churn — требуют минимум 3 месяцев регулярной истории.
BEP — требует понятной структуры постоянных и переменных расходов.
Прогноз — требует хотя бы 2 месяцев данных для линейной регрессии. Стартапы без выручки и повторяющейся бизнес-модели — не аудитория продукта. --- ## Garbage in → Garbage out Это самый частый паттерн провала: ```
День 1: Заполнили config: monthly_revenue = 0, gross_margin = 0
День 2: Запустили дашборд — получили пустой шаблон
День 3: "Система не работает"
``` **Система работает.** Данных нет — нечего анализировать. Минимум для работы системы:
- текущая выручка (хотя бы за последние 2 месяца)
- основные статьи расходов (ФОТ, маркетинг, постоянные)
- список продуктов с ценами и хотя бы примерной себестоимостью Без этого система даёт шаблоны, а не аналитику. --- ## Что нужно на вход ### Минимальные требования: | Требование | Почему обязательно |
|---|---|
| Регулярная выручка (хотя бы 2 мес) | Нет данных — нет аналитики |
| Хотя бы примерный учёт расходов | Без расходов маржа неизвестна |
| Список продуктов с ценой | Нужно для unit-экономики и BEP |
| Owner по цифрам (CEO или CFO) | Данные кто-то должен вводить |
| Желание видеть честную картину | Не «красивые цифры», а правда | ### Что улучшает результат (не обязательно): - CRM с историей сделок → точный CAC и воронка
- Выгрузка из 1С / Finolog → автоматическая P&L
- Данные по когортам → точный LTV и retention
- Разбивка расходов по направлениям → маржа по продуктам --- ## Антипаттерны использования ### «Смотрим только выручку» Самый опасный паттерн. Бизнес с выручкой 10 млн/мес и gross margin 15% — убыточный.
Бизнес с выручкой 5 млн/мес и gross margin 65% — здоровый.
Выручка без маржи — это полуправда. ### «CAC считается от общего бюджета» Агрегированный CAC скрывает, что один канал прибыльный, второй — убыточный.
При масштабировании деньги уходят в убыточный канал — это незаметно до потери месяца. ### «Churn мониторим раз в квартал» К моменту квартального отчёта потеряна целая когорта.
Churn нужно смотреть ежемесячно — отклонение от нормы видно на 2-й месяц. ### «Прогноз не нужен — и так понятно» Кассовый разрыв «сюрприз» — прямое следствие отсутствия прогноза.
Cash runway должен мониториться постоянно: менее 3 месяцев = опасная зона. ### «Investor Report сделаем когда попросят» На сборку уходит 2–3 дня + данные несогласованы.
Investor Report нужно обновлять ежеквартально — тогда он готов за 15 минут. --- ## Честные границы по каждому режиму | Режим | Что делает хорошо | Где ограничен |
|---|---|---|
| Дашборд | Дерево метрик и единая картина | Нужны данные в config |

| Unit-экономика | CAC, LTV, payback по логике | Нужны данные по каналам и стоимостям |
| Воронка | Узкие места в деньгах | Нужны конверсии по этапам |
| Прогноз | 3 сценария | Нужна история ≥2 месяцев |
| Финмодель | P&L + CF структура | CEO вводит расходы, не AI |
| BEP | Точка безубыточности | Нужны постоянные и переменные расходы |
| Когорты | Retention-таблица | Нужна история сделок ≥3 мес |
| Investor Report | Структура и язык | Данные вводит CEO | --- ## Резюме: формула успеха ```
