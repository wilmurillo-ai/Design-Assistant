# AI-Аналитик бизнеса PRO — Путь внедрения Пошаговый план от установки до полного аналитического контура.
Каждый этап строится на предыдущем — не пропускать. --- ## День 1: Установка и первый срез аналитики **Время: 1.5–2 часа** ### Шаг 1: Заполнить config.yaml (30–45 минут) Это самый важный шаг. Чем честнее заполнен config — тем точнее работает система. Обязательные поля на старте:
- `leader`: имя, роль, стоимость часа
- `company`: отрасль, модель, стадия
- `finance`: текущая выручка, gross margin, расходы
- `business_metrics`: north_star, CAC, LTV (хотя бы приблизительно)
- `products`: список продуктов с ценой и переменными расходами
- `funnel`: этапы воронки и текущие конверсии **Правило:** если не знаете точную цифру — ставьте приблизительную. Лучше 60% точности сразу, чем 100% через месяц. ### Шаг 2: Первый дашборд (15 минут) Команда: `дашборд` Что получите:
- дерево метрик Revenue → GM → EBITDA → Net
- разбивка по продуктам
- первые аномалии и отклонения ### Шаг 3: KPI-панель (10 минут) Команда: `покажи KPI` Что получите:
- светофор по ключевым метрикам
- красные зоны — где проблема прямо сейчас
- зелёные — где бизнес в норме **Результат дня 1:** config заполнен, первый дашборд собран, красные зоны видны. --- ## Неделя 1: Формирование аналитического контура **Ежедневно (10–15 минут):** | День | Что делать | Что получаете |
|---|---|---|
| Пн | `дашборд` + `покажи KPI` | Базовая картина бизнеса |
| Вт | `unit экономика` | CAC, LTV, payback по каждому продукту |
| Ср | `воронка` | Где теряются клиенты и сколько это стоит |
| Чт | `план факт за [текущий мес]` | Отклонения от плана + причины |
| Пт | `финмодель` | P&L + Cash Flow + точка безубыточности | **Что должно произойти к концу недели 1:**
- понятна реальная gross margin (часто сюрприз)
- CAC посчитан хотя бы по 2 каналам
- воронка разложена по этапам
- есть один красный KPI с планом действий **Чего НЕ нужно делать на неделе 1:**
- не пытаться идеально заполнить все поля config — это займёт недели
- не ждать «правильных» данных — начните с приблизительных
- не использовать все 12 режимов сразу — по 2–3 в день достаточно --- ## Месяц 1: Система начинает работать ### Неделя 2: Углубление - Когортный анализ: `когорты` — посмотреть retention по месяцам
- Сравнение периодов: `сравни [пред. мес] и [текущий]` — динамика
- Прогноз: `прогноз на 3 месяца` — базовый / опт / пессимистичный ### Неделя 3: Маржинальность продуктов - `маржа продуктов` — какой продукт зарабатывает, какой съедает
- Пересчитать unit-экономику с уточнёнными данными
- Первый анализ: откуда реально берётся прибыль? ### Неделя 4: Сводка за месяц - `план факт за [месяц]` — полная сводка отклонений
- `финмодель` — актуальный P&L за месяц
- Обновить прогноз на следующий месяц ### Чек-лист конца месяца 1: - [ ] Дашборд запускается ежедневно (5 минут)
- [ ] Unit-экономика посчитана по всем продуктам
- [ ] Воронка разложена с узкими местами
- [ ] Есть план/факт за месяц
- [ ] Прогноз на следующий квартал построен
- [ ] Определён главный красный KPI + план действий --- ## Квартал 1: Полный аналитический контур ### Месяц 2: Зрелость - Все 12 режимов используются по необходимости
- Investor Report готов в любой момент: `investor report`
- Когортный анализ показывает retention-тренды
- Появилась привычка: «не принимать решение без расчёта BEP» ### Месяц 3: Стратегический уровень - Стратегическая сессия НА ДАННЫХ: P&L тренды + unit-экономика + прогноз
- Сравнение Q1 vs ожидания
- Decision check: какие решения прошлого квартала были основаны на аналитике? **Ключевой момент:** первое квартальное планирование на основе финмодели и реальной unit-экономики — качественно другой уровень принятия решений. ### Чек-лист конца квартала 1: - [ ] Gross margin стабильно считается и видна в реальном времени
- [ ] CAC по каналам — знаем, где прибыльно, где нет
- [ ] LTV/CAC ≥ 3 по ключевым каналам (или есть план как прийти)
- [ ] Churn под мониторингом — отклонение видно на 2-й месяц
- [ ] Cash runway рассчитывается автоматически — нет кассовых сюрпризов
- [ ] Investor Report готовится за 15 минут, а не 3 дня
- [ ] CEO принимает <30% решений без аналитической базы --- ## Типичные ошибки первого месяца | Ошибка | Почему плохо | Что делать |
|---|---|---|
| Заполнить config формально («0», «»,«Название») | Система выдаёт шаблоны, не аналитику | Потратить 45 минут на честное заполнение |
| Смотреть только выручку | Без маржи картина неполная | Добавить gross_margin_percent в config |
| Не считать CAC по каналам | Неизвестно, где деньги работают | Каналы + бюджеты в channels[] |
| Пропускать воронку | Теряем клиентов — не видим где | `воронка` раз в неделю |
| Ждать «правильного» момента | Аналитика «потом» = аналитика никогда | Приблизительные данные сейчас лучше точных позже |
| Использовать только 1–2 режима | 11% системы = слабый результат | Добавлять по 2 режима в неделю | --- ## Рекомендуемый ритм (после первого месяца) ```
ЕЖЕДНЕВНО (5–10 мин): 09:00 — Дашборд + KPI-панель ПО НЕОБХОДИМОСТИ (10–20 мин каждый): — Unit-экономика (при изменении каналов или продуктов) — Воронка (при просадке конверсии) — Финмодель (при планировании расходов) — Прогноз (при стратегических решениях) ЕЖЕНЕДЕЛЬНО (30 мин): Пятница — План/факт за неделю + корректировка прогноза ЕЖЕМЕСЯЧНО (1–1.5 ч): — Полный P&L за месяц — Когортный анализ — Сравнение периодов — Обновление финмодели ЕЖЕКВАРТАЛЬНО (2–3 ч): — Investor Report — Стратегическая сессия на данных — Пересмотр пороговых значений KPI
