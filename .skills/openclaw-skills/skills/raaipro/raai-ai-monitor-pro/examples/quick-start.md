# AI-Монитор PRO — Quick Start (3 сценария) > Три готовых команды для первого запуска. Скопируйте, подставьте свои данные. --- ## Сценарий 1 — Фото-отчёт с объекта (строй-ниша) **Контекст:** Прораб прислал фото, директору нужен структурированный отчёт. **Команда:**
```
фото-отчёт
Объект: жилой дом ул. Ленина 15, 3-й этаж
Дата: сегодня
Бригада: бригада Иванова (монтаж перекрытий)
Плановые работы: укладка плит перекрытия секция А
Описание по фото: [опишите что видите на фото: уложено 8 из 12 плит, крепёж выполнен частично, трещина в одной плите, строительный мусор не убран]
``` **Ожидаемый результат:**
- Структурированный акт: выполнено / не выполнено / нарушения
- Алёрт если есть несоответствие нормам (трещина = дефект)
- Рекомендации по устранению --- ## Сценарий 2 — Дашборд инфраструктуры (ИТ-ниша) **Контекст:** CTO хочет утренний снимок состояния. **Команда:**
```
дашборд
Серверы: prod-01 (CPU 45%, RAM 67%, Disk 52%), db-primary (CPU 71%, RAM 88%, Disk 61%)
Сервисы: API Gateway OK 120ms, Payment Service WARNING 1840ms, Auth OK 95ms, PostgreSQL WARNING 340ms
Активных пользователей: 1247
Выручка в час: 24300 руб
``` **Ожидаемый результат:**
- Полная таблица с цветовой кодировкой (✅ OK / ⚠️ WARNING / ❌ DOWN)
- Активные алёрты с приоритетом
- Бизнес-влияние текущих проблем в рублях --- ## Сценарий 3 — Инцидент по playbook **Контекст:** Упал сервис, нужно быстро разобраться. **Команда:**
```
инцидент: payment-service отдаёт 502, с 14:35, клиенты не могут платить
``` **Ожидаемый результат:**
- Классификация severity (CRITICAL — прямые потери выручки)
- Пошаговый playbook диагностики (2 минуты → классификация → действие)
- Checklist уведомлений (команда / клиенты / статусная страница)
- Критерий восстановления --- ## Как запустить 1. Заполните config.yaml (OBLIGATORY поля — 10 минут)
2. Откройте Claude или OpenClaw
3. Скопируйте один из сценариев выше
4. Подставьте свои данные
5. Отправьте **Полная библиотека сценариев:** [full-library.md](full-library.md)
