# AI-Монитор PRO — Ограничения и честные границы Что продукт НЕ делает. Где он НЕ поможет. Что нужно на вход.
Честность = доверие. Доверие = высокий чек. --- ## Где AI-Монитор PRO НЕ поможет ### 1. Не чинит слабую архитектуру AI-Монитор PRO — это операционный контур реагирования. Он не устраняет фундаментальные проблемы:
- SPOF (single point of failure) в архитектуре
- Отсутствие резервирования критичных компонентов
- Технический долг, который генерирует инциденты Система замечает инциденты быстрее и помогает реагировать. Но если сервис падает каждую неделю из-за плохого кода — это задача рефакторинга, не мониторинга. ### 2. Не работает без определённой стоимости простоя CEO-отчёт и ROI-аргументы работают только если:
- команда знает, сколько стоит 1 час critical downtime;
- есть хотя бы базовое понимание бизнес-влияния по сервисам. Если `cost_of_1h_major_downtime_rub: 0` — CEO-отчёт становится шаблоном без смысла. ### 3. Не работает без post-mortem дисциплины Самый частый паттерн провала: ```
Инцидент случился → решили за 2 часа → "молодцы" → следующий такой же инцидент
``` AI-Монитор PRO даёт шаблон post-mortem и напоминает о дедлайне. Но если команда не доводит разбор до action items с owner'ами — паттерны инцидентов не ломаются. ### 4. Не заменяет senior DevOps / SRE Система поможет:
- Стандартизировать реакцию junior-инженеров по playbook
- Снизить время на первичную диагностику
- Удержать контекст при смене дежурных Система НЕ поможет:
- Принять сложное решение об архитектурных изменениях
- Отладить специфическую проблему кастомного ПО
- Заменить экспертизу при нестандартном инциденте ### 5. Не работает без определённой критичности сервисов Если все сервисы имеют `business_criticality: "critical"` — приоритизация бессмысленна.
Честная карта критичности: 20% сервисов критичны, 30% важны, 50% менее срочны. ### 6. Не автоматически собирает метрики AI-Монитор PRO — это интеллектуальная надстройка над данными. Но данные нужно подавать:
- вручную через команды (в базовом сценарии)
- через интеграции с Prometheus/Grafana (опционально) Система не читает серверы сама без конфигурации. --- ## Что нужно на вход ### Минимальные требования: | Требование | Почему обязательно |
|---|---|
| 3+ серверов и 5+ сервисов | Нет нагрузки — нечего мониторить |
| Определённая стоимость простоя | Без этого нет языка денег для CEO |
| Назначенный incident commander | Без ответственного — хаос при инциденте |
| Готовность вести post-mortem | Без разбора паттерны повторяются |
| SLA-цель (хотя бы приблизительная) | Нет цели — нет контроля буфера | ### Что улучшает результат (но не обязательно): - Интеграция с Prometheus/Grafana для автоматической подачи метрик
- Исторические данные по инцидентам (MTTR, частота, стоимость)
- Зрелая DevOps-культура с опытом post-mortem
- Документация по архитектуре для быстрой диагностики --- ## Честные границы по каждому режиму | Режим | Что делает хорошо | Где ограничен |
|---|---|---|
| Дашборд | Структурирует снимок состояния | Данные нужно подавать — автосбора нет в базовом режиме |
| Алертинг | Приоритизирует и рекомендует действие | Качество зависит от правильно выставленных порогов |
| Playbook | Даёт сценарий первых шагов | Не заменяет экспертизу для нестандартных инцидентов |
| SLA | Считает буфер и предупреждает | Нужно правильно заполнить sla.default_target |
| CEO-отчёт | Переводит в язык денег | Работает только при заполненной cost_of_1h_downtime |
| Capacity | Прогнозирует по трендам | Нужна история данных, без неё — грубый прогноз |
| Post-mortem | Структурирует разбор | Action items нужно делать — AI только помогает зафиксировать | --- ## Резюме: формула успеха ```
Результат = Операционная дисциплина × Качество данных × Системность использования Где: Операционная дисциплина = post-mortem + playbook + ревью алертов Качество данных = честная карта критичности + стоимость простоя Системность = все 10 режимов, а не только дашборд Если хотя бы один множитель ≈ 0, результат ≈ 0.
``` AI-Монитор PRO — честный продукт. Он не обещает, что инциденты прекратятся. Он обещает, что реакция станет быстрее, предсказуемее и не будет стоить так дорого. --- *AI-Монитор PRO | RAAI | Честные границы продукта*
