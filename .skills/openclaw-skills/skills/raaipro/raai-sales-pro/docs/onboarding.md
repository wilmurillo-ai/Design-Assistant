# Путь внедрения — AI-Продажник PRO Пошаговый план от установки до полноценной системы продаж.
Не торопиться. Каждый этап — фундамент для следующего. --- ## День 1 — Старт **Время: 1.5–2 часа** ### Шаг 1: Заполнить config.yaml (30–40 минут) Это фундамент всей системы. Система точна ровно настолько, насколько точны данные на входе. Обязательные поля:
- Продукт/услуга: описание, ключевые выгоды, дифференциаторы
- Ниша: кто ваш клиент, его боли, типичные возражения
- Цены: прайс, средний чек, цикл сделки (в днях)
- Команда: количество менеджеров, уровень подготовки
- Воронка: текущие этапы, конверсии по этапам (если знаете) **Правило:** неточные данные лучше пустых. Поставьте приблизительно — потом скорректируете. ### Шаг 2: Первый скрипт звонка (15–20 минут) Команда: `скрипт звонка: [ваша ниша]` Что получаете:
- Скрипт с ветвлениями по возможным ответам клиента
- Варианты открытия разговора
- Переходы к выявлению потребности
- Блок с отработкой топ-3 возражений вашей ниши **Проверка:** скрипт должен звучать как "наш язык", а не шаблонный. Если не попал — уточните нишу и повторите. ### Шаг 3: SPIN-вопросы под нишу (15 минут) Команда: `SPIN для [ниша]` Получаете 20+ вопросов по четырём уровням:
- Ситуационные (что сейчас у клиента)
- Проблемные (что его беспокоит)
- Извлекающие (к чему это ведёт)
- Направляющие (что изменится, если проблему решить) ### Шаг 4: Первая квалификация лида (10 минут) Команда: `BANT: [описание лида]` Система оценит лида по четырём критериям:
- Budget (есть ли бюджет)
- Authority (это ЛПР?)
- Need (есть ли реальная потребность)
- Timeline (когда планирует решение) **Результат дня 1:** config заполнен, первый скрипт получен, SPIN под нишу готов, BANT понятен. --- ## Неделя 1 — Базовая система **Задача: перевести всех менеджеров на единый стандарт** | День | Действие | Результат |
|---|---|---|
| Пн | Скрипт → все менеджеры | Единый стандарт первого контакта |
| Вт | Топ-5 возражений + ветвления | Скрипт обработки возражений |
| Ср | BANT для текущей воронки | Все активные лиды квалифицированы |
| Чт | Pipeline: все сделки по этапам | Воронка видна, прогресс ясен |
| Пт | Follow-up цепочка | Менеджеры знают что делать после звонка | **Чего НЕ делать на неделе 1:**
- Не пытаться использовать все 15 режимов сразу
- Не пропускать квалификацию лидов — это фильтр пустой траты времени
- Не ждать идеальных данных для pipeline — вносить то, что есть сейчас **Что должно произойти к концу недели 1:**
- Все менеджеры работают по единому скрипту
- Все текущие лиды прошли BANT-квалификацию
- Pipeline заведён: стадии, суммы, прогнозируемые даты
- Follow-up цепочка настроена — менеджер знает следующий шаг по каждому лиду --- ## Месяц 1 — Аналитика и корректировка ### Неделя 2: углубление методологий - Challenger-питч для ключевых сегментов: `Challenger для [сегмент]`
- MEDDIC для Enterprise-лидов: `MEDDIC: [описание сделки]`
- КП в enterprise-формате с ROI: `КП для [клиент, продукт]`
- Pipeline Review: `pipeline report` — первый прогноз выручки ### Неделя 3: первые данные для анализа - Первый win/loss анализ (достаточно 5–10 закрытых сделок): `win/loss анализ`
- Unit-экономика: `unit экономика` — CAC, LTV, конверсия по этапам
- Корректировка скриптов по реальным паттернам поражений ### Неделя 4: система работает - Менеджеры открывают утро с pipeline — знают приоритеты
- РОП проводит еженедельный Pipeline Review по данным системы
- Первые паттерны побед и поражений видны ### Чек-лист конца месяца 1: - [ ] Все менеджеры работают по единому скрипту
- [ ] Топ-5 возражений закрыты скриптом с ветвлениями
- [ ] Pipeline обновляется каждую неделю
- [ ] Все входящие лиды проходят BANT-квалификацию
- [ ] Первый win/loss анализ проведён
- [ ] КП переведено в enterprise-формат с ROI
- [ ] Pipeline Report формируется еженедельно
- [ ] Follow-up цепочка работает — нет "потерянных" лидов --- ## Квартал 1 — Полная система продаж ### Месяц 2: зрелость - Все 15 режимов понятны и используются по назначению
- Win/loss паттерны накопились — видно управляемые причины поражений
- Конверсия по этапам измерима — РОП управляет по цифрам, не по ощущениям
- Challenger-питч откалиброван под 2–3 реальных сегмента ### Месяц 3: результат - Win/loss паттерны квартала: управляемые причины устранены из процесса
- Конверсия выросла на 20–40% от стартовой (реалистичный ожидаемый диапазон)
- Pipeline предсказуем: прогноз выручки с точностью ±15%
- Менеджеры работают автономно по системе — РОП контролирует по данным
- Система продаж работает как машина — её можно масштабировать ### Чек-лист конца квартала 1: - [ ] Конверсия выросла на 20–40% от стартовой
- [ ] Pipeline Report → прогноз выручки ±15%
- [ ] Win/loss анализ: управляемые причины поражений устранены
- [ ] РОП контролирует по pipeline, не микроменеджит
- [ ] SPIN и Challenger встроены в стандарт работы менеджера
- [ ] Unit-экономика считается ежемесячно — видны CAC и LTV
- [ ] Скрипты скорректированы по реальным данным 3+ раз
- [ ] Менеджеры работают автономно по системе — без постоянных подсказок РОПа
- [ ] Следующий квартал планируется на данных, а не на интуиции
- [ ] Система масштабируема — можно добавить нового менеджера без потери стандарта --- ## Типичные ошибки первого месяца | Ошибка | Почему плохо | Что делать |
|---|---|---|
| Скрипт получили, менеджеры не используют | Система без применения = 0 | Обязательный стандарт, контроль РОПа |
| Config.yaml заполнен примерно | Скрипты не попадают в нишу | 30 минут на точное заполнение |
| Pipeline не обновляется | Прогноз врёт — управление слепое | Еженедельный ритм: обновление обязательно |
| Win/loss не делается | Одни и те же ошибки повторяются | Раз в месяц по 5+ закрытым сделкам |
| Используют 2–3 режима из 15 | Частичная система даёт частичный результат | Неделя 2 — добавить 3 новых режима |
| РОП не смотрит Pipeline Report | Данные есть, управление не меняется | Pipeline Review = обязательная встреча раз в неделю | --- ## Рекомендуемый рабочий ритм (после первого месяца) ```
ЕЖЕДНЕВНО (5–10 мин, каждый менеджер): Утро — открыть pipeline, приоритеты на день После каждого звонка — обновить статус лида ПО НЕОБХОДИМОСТИ: Новый лид → BANT-квалификация Питч сложному клиенту → Challenger-подготовка Лид не квалифицирован → SPIN-вопросы Enterprise-сделка → MEDDIC КП нужно → режим КП с ROI ЕЖЕНЕДЕЛЬНО (30–45 мин, РОП): Пятница — Pipeline Review + прогноз выручки Корректировка приоритетов по воронке ЕЖЕМЕСЯЧНО (60–90 мин, РОП + руководитель): Win/Loss анализ — паттерны побед и поражений Unit-экономика — CAC, LTV, конверсии Корректировка скриптов по данным
``` --- *AI-Продажник PRO | RAAI | Путь внедрения*
