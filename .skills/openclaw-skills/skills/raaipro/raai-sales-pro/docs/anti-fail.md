# Ограничения и честные границы — AI-Продажник PRO Что продукт НЕ делает. Где он НЕ поможет. Что нужно на вход.
Честность = доверие. Доверие = высокий чек. --- ## Где AI-Продажник PRO НЕ поможет ### 1. Нет продукта или услуги Если нечего продавать — скрипты бессмысленны. Система не создаёт продукт, не формирует ценностное предложение с нуля. Нужны реальные преимущества и реальные клиенты. ### 2. Нет лидов AI-Продажник PRO работает с воронкой — структурирует, квалифицирует, помогает закрывать. Но лидов он не генерирует. Pipeline будет пустым, если нет входящего потока. Система — для конвертации лидов, а не для их привлечения. ### 3. Менеджеры саботируют Самый частый паттерн провала: ```
Неделя 1: "Попробовали скрипт — интересно!"
Неделя 2: "Ну, я по-своему продаю лучше..."
Неделя 3: "Система не работает."
``` **Система работает. Менеджеры — нет.** Без обязательного использования системы РОПом и менеджерами — это файл за 30 000 руб. ### 4. РОП не контролирует Pipeline без еженедельного ревью деградирует за 2 недели. Win/loss без анализа — просто список. Система требует управленческого участия — РОП должен смотреть данные и принимать решения по ним. ### 5. Чек ниже 5 000 руб при разовых продажах SPIN, Challenger Sale и MEDDIC — методологии для сложных и дорогих продаж с циклом сделки от 1 недели. Для продаж с чеком 500–5 000 руб и мгновенным решением клиента эти методологии избыточны. Система даст скрипты, но ROI будет минимальным. --- ## Garbage in → garbage out | Что заходит на вход | Что выходит на выходе |
|---|---|
| config.yaml с выдуманными данными о нише | Скрипты не попадают в боли реальных клиентов |
| Скрипт без адаптации под конкретный продукт | Шаблонный текст, который клиент слышит каждый день |
| Win/loss без честных причин поражения | Отчёт, который не меняет результат |
| Pipeline с неактуальными данными | Прогноз выручки врёт — решения принимаются вслепую |
| КП без данных о клиенте | Общее предложение, которое не продаёт |
| BANT без реального разговора с клиентом | Формальная квалификация, не отражающая реальность | **Правило:** потратьте 30 минут на точный config.yaml — сэкономите 30 часов на переделке скриптов. --- ## Честные границы каждого режима | Режим | Что может | Чего НЕ может |
|---|---|---|
| Скрипт звонка | Адаптирует структуру под нишу и возражения | Не заменяет живой тон и интонацию менеджера |
| SPIN-вопросы | Генерирует 20+ вопросов под нишу | Не слушает ответы клиента в реальном времени |
| Challenger Sale | Готовит питч, который бросает вызов статус-кво клиента | Не знает специфику бизнеса клиента если она не описана |
| BANT-квалификация | Структурирует оценку лида по четырём критериям | Не может квалифицировать лида вместо менеджера |
| MEDDIC | Даёт чек-лист для Enterprise-сделок | Не интегрируется с CRM автоматически |
| Pipeline Management | Структурирует воронку, строит прогноз | Данные вводит менеджер, не система |
| Pipeline Report | Еженедельный отчёт с прогнозом выручки | Точен только при актуальных данных в воронке |
| Win/Loss анализ | Находит паттерны побед и поражений | Не знает причины если менеджер их не записал |
| КП (enterprise) | Строит структурированный документ с ROI | Не знает бизнес клиента если он не описан |
| Follow-up цепочка | Генерирует серию касаний под стадию воронки | Не отправляет письма автоматически — нужен менеджер |
| Обработка возражений | Ветвления под топ-10 возражений ниши | Не предугадывает уникальное возражение нового клиента |
| Unit-экономика | Считает CAC, LTV, конверсии по этапам | Работает с данными которые ввёл РОП |
| Подготовка к встрече | Чек-лист и сценарий переговоров | Не знает историю отношений с клиентом если не описана |
| Дожим сделки | Тактики финального закрытия под тип клиента | Не принимает решение за клиента |
| Скрипт холодного звонка | Структура первого контакта с незнакомым лидом | Не генерирует базу контактов для обзвона | --- ## Как получить максимум **Четыре условия, без которых система работает вполсилы:** 1. **Точный config.yaml** — потратить 30–45 минут на первое честное заполнение. Не "примерно", не "потом". Сразу. 2. **Первая неделя — ежедневная работа** — калибровка скриптов, квалификация всех текущих лидов, заполнение pipeline. Если первую неделю пропустить — система не запустится правильно. 3. **Pipeline обновляется каждую неделю** — без актуальных данных Pipeline Report врёт, РОП управляет вслепую. 4. **Win/loss анализ — минимум раз в месяц** — это единственный механизм улучшения системы на основе реального опыта продаж. Без него ошибки повторяются. --- ## Если не сработало — чек-лист диагностики Перед выводом "система не работает" — проверить по порядку: | # | Вопрос | Если нет — причина в этом |
|---|---|---|
| 1 | config.yaml заполнен точно и честно? | Скрипты не попадают в нишу |
| 2 | Менеджеры используют скрипты в работе? | Система не применяется |
| 3 | Pipeline обновляется каждую неделю? | Данные устарели — управление слепое |
| 4 | РОП проводит еженедельный Pipeline Review? | Данные есть, решений нет |
| 5 | Win/loss анализ проводился хотя бы раз? | Нет понимания причин результата |
| 6 | BANT используется для входящих лидов? | Время тратится на неквалифицированных лидов |
| 7 | Challenger-питч откалиброван под нишу? | Питч звучит как и у конкурентов |
| 8 | Средний чек выше 15 000 руб? | ROI системы снижается при маленьком чеке | **Правило:** если все 8 пунктов — да, но результата нет — проблема в качестве лидов или в продукте, а не в системе продаж. --- ## Резюме: формула успеха ```
Результат = Дисциплина команды × Качество данных × Системность использования Где: Дисциплина = ежедневная работа менеджеров + еженедельный Pipeline Review РОПа Качество данных = честный config.yaml + актуальный pipeline + честный win/loss Системность = все 15 режимов по назначению, а не 2–3 любимых Если хотя бы один множитель ≈ 0, результат ≈ 0.
``` AI-Продажник PRO — честный продукт. Он не обещает магию. Он обещает систему.
Система × дисциплина = рост конверсии.
Система × 0 = дорогой файл. --- *AI-Продажник PRO | RAAI | Ограничения и честные границы*
