1. Заполнить `config.yaml` — ниша, продукт, средний чек, цикл сделки, топ-3 возражения.
2. Зафиксировать 3 главные боли в продажах прямо сейчас.
3. Запустить `скрипт звонка` — получить первый скрипт под нишу.
4. Запустить `SPIN` — первая квалификационная карта под продукт. ### Неделя 1
1. Все менеджеры переходят на единый скрипт звонка.
2. `follow-up` — настроить цепочку под первую стадию воронки.
3. `pipeline` — завести текущие сделки с вероятностями.
4. `скоринг` — прогнать топ-10 лидов через BANT+MEDDIC.
5. Первый разбор диалога — найти и исправить системную ошибку. ### Месяц 1
1. Собрать данные по первым сделкам — где застревают, где сливаются.
2. Первый `win/loss` разбор — паттерны слива на реальных кейсах.
3. Скорректировать скрипты по данным, а не по ощущениям.
4. Запустить `unit-экономику` — считать стоимость сделки, а не только выручку.
5. Pipeline показывает реальный прогноз на следующий месяц. ### Квартал 1
1. Полный цикл win/loss — паттерны видны, конкуренты разобраны.
2. Конверсия по стадиям выросла на 2+ этапах воронки.
3. Онбординг нового менеджера — 3 недели вместо 3 месяцев.
4. Продажи идут по системе. РОП управляет по цифрам. Подробный путь внедрения: `docs/onboarding.md` --- ## Частые возражения перед покупкой ### «У нас уже есть CRM»
CRM хранит данные, но не продаёт. Нет скриптов с ветвлениями, нет SPIN-квалификации, нет win/loss, нет follow-up логики. AI-Продажник PRO — надстройка над любой CRM, а не замена. ### «Менеджеры не будут пользоваться новым инструментом»
Это не новый инструмент для менеджеров — это система скриптов, которую РОП контролирует через pipeline. Менеджер получает готовый скрипт, а не учится новой платформе. ### «Мы и так продаём»
Вопрос не «продаёте ли», а «сколько теряете». Win/loss покажет: сколько сделок слито, по какой причине, и что это стоило в деньгах. ### «Дорого для набора скриптов»
Это не скрипты — это система продаж. Скрипты — 1 из 15 режимов. Внутри: квалификация, pipeline, follow-up, win/loss, unit-экономика, дожим, КП с ROI. 30К — цена системы, а не промпта. ### «Нам нужен живой тренер по продажам»
Тренер по продажам: 100–300К/мес, 2–4 сессии. Между сессиями — ничего. AI-Продажник PRO: 30К единовременно, 24/7, навсегда. И скрипты не уходят с тренером. --- ## Proof: что показывает продукт 1. **Менеджеры говорят по-разному** → единый скрипт звонка с ветвлениями сократил разброс конверсии между менеджерами с 3× до 1.4×.

- скрипты заведены, но менеджеры игнорируют процесс;
- данные по сделкам не вносятся в pipeline — прогноза не будет;
- win/loss проводится раз в год, а не после каждой потери;
- config.yaml заполнен формально — нет реальных данных о нише и возражениях;
- собственник хочет «волшебный скрипт», но не готов строить процесс. **Garbage in → garbage out.** Подробнее: `docs/anti-fail.md` --- ## Структура файлов ```text
