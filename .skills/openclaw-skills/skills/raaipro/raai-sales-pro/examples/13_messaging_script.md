# Пример 13: Скрипт переписки — реактивация холодной базы **Запрос:**
```
скрипт переписки: реактивация базы — 50 лидов которые интересовались 3-6 месяцев назад,
не купили. Ниша: малый бизнес (магазины, услуги). Продукт: AI-бот для поддержки, 8 000 руб/мес.
Канал: WhatsApp
``` --- **Вывод агента:** ```
💬 СКРИПТ РЕАКТИВАЦИИ: Холодная база, WhatsApp, малый бизнес ━━━━━━━━━━━━━━━━━━━━━━━━━━━━ СООБЩЕНИЕ 1 — РЕАКТИВАЦИЯ (триггер — новинка/повод):
«[Имя], несколько месяцев назад общались про автоматизацию поддержки. Запускаем новую функцию — [конкретная новинка]. Для малого бизнеса это [конкретная выгода]. Актуально снова?» [Не надо: "Как ваши дела?", "Давно не виделись", длинного текста] → ЕСЛИ "да" / "расскажите": СООБЩЕНИЕ 2
→ ЕСЛИ "нет / уже не актуально": СООБЩЕНИЕ 1б
→ ЕСЛИ нет ответа 48ч: СООБЩЕНИЕ 1в СООБЩЕНИЕ 1б (ответил "нет"):
«Понял. Можно один вопрос — что тогда остановило? [Цена / Не подошло / Решили другим способом]?»
→ ЕСЛИ называет причину: обработать конкретно → может открыть новый диалог
→ ЕСЛИ нет: "Хорошо, удачи!" — дальше не давить СООБЩЕНИЕ 1в (не ответил 48ч):
«[Имя], понимаю — много сообщений. Один вопрос: поддержка клиентов для вас сейчас актуальна или нет?»
→ Если нет ответа ещё 48ч → снять с реактивации, в базу через 3 мес ━━━━━━━━━━━━━━━━━━━━━━━━━━━━ СООБЩЕНИЕ 2 — ВЫЯВЛЕНИЕ ИЗМЕНЕНИЙ:
«Отлично! Расскажите — что изменилось за это время? Сколько обращений в день сейчас обрабатываете?» [Задача: понять изменилась ли ситуация → адаптировать под новую боль] → Дождаться ответа
→ Если ситуация изменилась (больше клиентов, расширились): усилить актуальность
→ Если то же самое: вернуться к старому возражению, обработать по-новому СООБЩЕНИЕ 3 — ПРЕЗЕНТАЦИЯ:
«Понял. Именно для магазинов/услуг у нас обновление: [конкретная фича]. [Магазин из их ниши] за первый месяц: администратор отвечает 24/7, +15 заказов, −2 часа работы в день. Покажу как это выглядит — 15 минут демо, удобно сегодня или завтра?» ━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ИНСТРУКЦИЯ ПО РАБОТЕ С БАЗОЙ 50 ЧЕЛОВЕК: Сегментация перед рассылкой:
• Группа А — сказали "дорого": добавить к сообщению 1 — «+ теперь есть пакет от 4 500 руб»
• Группа Б — сказали "подумаю": упомянуть что появились новые кейсы из их ниши
• Группа В — просто пропали: стандартное сообщение 1 Порядок:
1. Сначала отправить группе А и Б (теплее)
2. Через 3 дня — группа В
3. Не рассылать всем сразу — WhatsApp может пометить как спам при массовой рассылке
4. Личные сообщения, не рассылка Ожидаемый результат:
• Ответят: 30-50% (WhatsApp хорошо открывается)
• Заинтересуются: 20-30% от ответивших
• Купят из реактивации: 5-15% от всей базы = 2-7 сделок
• Выручка: 16 000-56 000 руб от одного прохода по базе
``` --- **Признак сильного выхода:** Скрипт содержит ветки для всех сценариев ответа (да / нет / молчание), сегментацию базы по причине предыдущего отказа с адаптированным первым сообщением для каждой группы, порядок отправки исключающий спам-блокировку, и прогноз воронки с конкретными цифрами конверсии и выручки от одного прохода.
