# 📐 MECE 原则

> 相互独立，完全穷尽 (Mutually Exclusive, Collectively Exhaustive)

---

## 📖 什么是 MECE 原则

MECE 是一种**结构化思考**原则，确保分析框架：

- **Mutually Exclusive (相互独立)** - 各部分不重叠
- **Collectively Exhaustive (完全穷尽)** - 所有部分加起来完整覆盖

---

## 🎯 适用场景

- ✅ 问题拆解与分析
- ✅ 报告结构与撰写
- ✅ 分类与归纳
- ✅ 确保思考完整性

---

## 🔧 应用方法

### 方法 1：二分法
```
问题 → A / 非 A

示例：
- 客户 → 新客户 / 老客户
- 收入 → 线上收入 / 线下收入
- 成本 → 固定成本 / 变动成本
```

### 方法 2：流程法
```
按时间/流程顺序拆解

示例：
- 用户旅程 → 认知 → 考虑 → 购买 → 使用 → 复购
- 项目阶段 → 启动 → 规划 → 执行 → 监控 → 收尾
```

### 方法 3：公式法
```
使用数学公式拆解

示例：
- 利润 = 收入 - 成本
- 收入 = 客单价 × 客户数
- 客户数 = 新客户 + 留存客户
```

### 方法 4：矩阵法
```
使用 2×2 矩阵分类

示例：
- 重要/紧急 矩阵
- 内部/外部 × 有利/不利 (SWOT)
- 高/低 × 高/低 矩阵
```

## 📊 经典案例

### 案例 1：利润下降分析

```
利润下降
├── 收入下降
│   ├── 销量下降
│   │   ├── 客户数减少
│   │   └── 购买频次降低
│   └── 单价下降
│       ├── 折扣增加
│       └── 产品结构变化
└── 成本上升
    ├── 固定成本
    │   ├── 租金
    │   └── 人工
    └── 变动成本
        ├── 原材料
        └── 物流
```

### 案例 2：市场分析

```
目标市场
├── 地理维度
│   ├── 国内市场
│   └── 海外市场
├── 人群维度
│   ├── 年龄
│   │   ├── 18-25 岁
│   │   ├── 26-35 岁
│   │   └── 36 岁+
│   └── 收入
│       ├── 高收入
│       ├── 中收入
│       └── 低收入
└── 场景维度
    ├── 工作场景
    ├── 生活场景
    └── 娱乐场景
```

## ⚠️ 检查清单

### ME 检查 (相互独立)
- [ ] 各类别之间有明确边界
- [ ] 没有模棱两可的分类
- [ ] 每个元素只能属于一个类别

### CE 检查 (完全穷尽)
- [ ] 所有可能的情况都已覆盖
- [ ] 没有"其他"类别（或有明确定义）
- [ ] 类别加起来等于整体

### 常见陷阱
| 陷阱 | 表现 | 修正方法 |
|------|------|----------|
| 重叠 | 客户分类：年轻人/上班族 | 改为：年龄/职业 两个维度 |
| 遗漏 | 市场：国内/国外 | 加上：港澳台/特殊地区 |
| 层级混乱 | 混用不同拆解维度 | 统一拆解标准 |
