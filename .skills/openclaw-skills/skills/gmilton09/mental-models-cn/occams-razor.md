# ✂️ 奥卡姆剃刀

> "如无必要，勿增实体" (Entities should not be multiplied beyond necessity)

---

## 📖 什么是奥卡姆剃刀

奥卡姆剃刀（Occam's Razor）是一种**简化原则**：在多个解释或方案中，**最简单的那个往往是正确的**。

### 核心思想
- 简单 ≠ 简陋
- 简单 = 没有不必要的复杂
- 优先选择假设最少的解释

---

## 🎯 适用场景

- ✅ 假设验证与选择
- ✅ 模型/方案设计
- ✅ 问题诊断
- ✅ 决策简化

---

## 🔧 应用方法

### 方法 1：假设计数
```
解释 A: 需要 3 个假设
解释 B: 需要 5 个假设

→ 优先选择解释 A
```

### 方法 2：必要性质疑
```
对每个元素问：
- 这个元素是必要的吗？
- 去掉它会怎样？
- 有没有更简单的方式实现同样功能？
```

### 方法 3：复杂度评估
```
方案评分 = 效果 / 复杂度

优先选择性价比最高的方案
```

## 📊 经典案例

### 案例 1：医学诊断
```
症状：发烧 + 咳嗽 + 乏力

解释 A (简单): 普通感冒
- 假设 1: 病毒感染

解释 B (复杂): 罕见病 + 并发症
- 假设 1: 罕见病原体
- 假设 2: 免疫系统异常
- 假设 3: 环境触发因素

→ 优先按感冒治疗，无效再考虑复杂解释
```

### 案例 2：产品设计
```
目标：用户注册流程

方案 A (复杂):
- 10 步表单
- 邮箱验证
- 手机验证
- 人脸识别
- 风险评估

方案 B (简单):
- 3 步表单
- 邮箱验证

→ 方案 B 转化率通常更高
```

## ⚠️ 使用要点

### 适用条件
- ✅ 多个方案效果相当
- ✅ 假设可验证
- ✅ 简单不牺牲核心功能

### 不适用场景
- ❌ 问题本身复杂（不能过度简化）
- ❌ 关键细节被忽略
- ❌ 简单=错误

### 常见陷阱
| 陷阱 | 表现 | 避免方法 |
|------|------|----------|
| 过度简化 | 忽略关键因素 | 验证核心假设 |
| 错误应用 | 在复杂问题上强行简化 | 承认复杂性 |
| 混淆概念 | 简单≠容易 | 区分复杂度与难度 |
