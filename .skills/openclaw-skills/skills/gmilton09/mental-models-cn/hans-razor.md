# 🤷 汉隆剃刀

> "能解释为愚蠢的，就不要解释为恶意" (Never attribute to malice that which is adequately explained by stupidity)

---

## 📖 什么是汉隆剃刀

汉隆剃刀（Hanlon's Razor）是一种**归因原则**：在解释他人行为时，**优先假设是无知/疏忽，而非恶意**。

### 核心思想
- 恶意需要动机 + 计划
- 疏忽/无知更常见
- 优先选择概率更高的解释

---

## 🎯 适用场景

- ✅ 人际冲突分析
- ✅ 组织行为理解
- ✅ 舆情分析
- ✅ 减少不必要的对抗

## 🔧 应用框架

### 归因分析
```
事件：_______________________

可能的解释：

1. 恶意解释
   - 动机：________
   - 计划：________
   - 证据：________
   - 概率：___%

2. 疏忽解释
   - 原因：________
   - 情境：________
   - 证据：________
   - 概率：___%

3. 无知解释
   - 缺乏信息：________
   - 误解：________
   - 证据：________
   - 概率：___%

→ 选择概率最高的解释
```

## 📊 经典案例

### 案例 1：同事没回复邮件

```
事件：重要邮件发出后 3 天没回复

恶意解释:
- 他故意忽视我
- 他想让我难堪
- 概率：10%

疏忽解释:
- 邮件被淹没了
- 他太忙忘记了
- 概率：70%

无知解释:
- 他没看到邮件
- 他以为别人会处理
- 概率：20%

→ 友好提醒，而非直接对抗
```

### 案例 2：产品上线出 Bug

```
事件：重要功能上线后出现严重 Bug

恶意解释:
- 开发人员故意留 Bug
- 概率：1%

疏忽解释:
- 测试覆盖不足
- 时间压力导致疏漏
- 概率：80%

无知解释:
- 边界情况未考虑到
- 依赖变更未通知
- 概率：19%

→ 改进流程，而非追责个人
```

## ⚠️ 使用要点

### 适用条件
- ✅ 解释他人行为
- ✅ 信息不完整时
- ✅ 减少冲突场景

### 不适用场景
- ❌ 有明确恶意证据
- ❌ 重复性伤害行为
- ❌ 需要立即制止的危险行为

### 常见陷阱
| 陷阱 | 表现 | 避免方法 |
|------|------|----------|
| 过度宽容 | 忽视真正的恶意 | 收集证据，设定边界 |
| 自我欺骗 | 为伤害找借口 | 区分单次与模式 |
| 忽视模式 | 重复发生仍归因疏忽 | 识别行为模式 |
