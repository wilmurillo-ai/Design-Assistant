# /mvp-to-stories

Split one MVP into executable stories.

## Task

Produce:

1. gap cards
2. nearly-done cards
3. lightweight sprint order

## Rules

- do not flatten everything into one backlog
- keep one story id namespace
- sprint means ordering, not staffing
- if something is already mostly done, mark it as such

## Output

- gap cards
- nearly-done cards
- sprint 1
- sprint 2
