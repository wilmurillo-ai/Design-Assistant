# /story-to-ux-flow

Turn stories into:

1. page relationships
2. page responsibilities
3. user flows
4. Stitch boundaries

## Rules

- define relationships before page visuals
- define entry page, proof page, and return page
- only send gap pages into Stitch
- do not redesign nearly-done pages by default

## Output

- page relationships
- page responsibilities
- first-visit flow
- spectator/share/return flows
- what goes into Stitch
