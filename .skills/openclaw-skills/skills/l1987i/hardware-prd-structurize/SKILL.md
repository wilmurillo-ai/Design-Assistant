# Hardware PRD Structurize | 硬件 PRD 结构化技能

将晦涩的底层硬件规格、系统架构限制及测试 Bug 转化为面向产品经理的"大白话"，输出 8 列结构化 PRD 表格。

## 使用场景

- 拆解硬件知识内容
- 整理传感器/执行器技术参数
- 将硬件规格转化为 PRD 表格
- 输出包含 PRD 规则、测试方法、AI 剧本的结构化文档

## 输出格式

8 列结构化表格：
1. **硬件名称** - 部件名称
2. **硬件能力与参数** - 具体功能点或物理参数
3. **产品视角大白话** - 无技术门槛的比喻解释
4. **硬件能力底牌** - 理想状态下的极致能力
5. **架构限制与 Bug** - 当前系统下的致命问题
6. **实际测试与排查方法** - PM 验证功能的具体动作
7. **PRD 强制业务规则** - 系统调度红线
8. **支持的 AI 剧本** - 用户互动场景与 AI 话术

## 示例

**输入**：`拆解这段内容：FS511-L 雷达参数...`

**输出**：8 列 PRD 表格（Markdown 或飞书表格）

## 依赖

- 飞书文档 API（可选，用于保存到飞书）
