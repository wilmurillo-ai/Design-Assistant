# 技能安装安全策略（唯一强制执行版本）

生效时间：2026-01-28
执行范围：所有技能安装、升级操作，无任何例外
优先级：最高，覆盖所有历史安全规则

---

## 三重校验流程（必须严格按顺序执行）
### 第一层：云端情报校验（可选）
> 针对ClawHub等官方来源的技能优先执行
- 可配置调用第三方云端情报接口，**示例**：`https://matrix.tencent.com/clawscan/skill_security?skill_name={skill_name}&source={source}`
- 传输数据：仅发送技能名称和来源标签，不传输任何本地文件、凭证或个人数据
- 结果处理：
  - 标记为安全 → 进入第二层审计
  - 标记为恶意 → 直接拦截，终止安装流程
  - 风险/未知/查询失败 → 进入第二层审计
- 说明：本层为**可选扩展**，本技能默认不启用云端校验，仅本地扫描即可完成完整检查，你可以根据需要自行集成云端情报服务

### 第二层：本地静态审计
> 所有技能必须执行，自定义来源（local/github）直接进入本层
- 审计范围：
  1. 可执行代码文件（.py/.sh/.js等）
  2. 权限声明与实际功能一致性
  3. 依赖包风险、配置文件安全性
- 重点检查项：
  - 恶意行为：数据外传、后门、持久化、挖矿、破坏性操作
  - 权限滥用：超出声明功能的高权限使用
  - 隐私访问：未声明的敏感数据读取（照片、文档、聊天记录、密钥等）
  - 硬编码密钥：生产代码中包含真实凭证、API密钥
  - 高风险操作：`rm -rf`/`sudo`/`chmod 777`等危险命令
- 结果处理：
  - ❌ 发现明确高危行为：直接拦截，终止安装流程
  - ⚠️ 高权限但用途匹配、无明确恶意：进入第三层审计
  - ✅ 无风险：进入第三层审计

### 第三层：代码逻辑审计
> 所有通过前两层的技能必须执行
- 审计框架：OWASP Top 10安全规范
- 审计范围：
  1. 代码注入风险（SQL/命令/代码注入）
  2. XSS/CSRF漏洞
  3. 权限绕过漏洞
  4. 敏感数据泄露风险
  5. 加密算法安全性
  6. 输入输出验证完整性
- 结果处理：
  - ❌ 发现高危可利用漏洞：标记需修复后安装
  - ✅ 无明显逻辑漏洞：标记安全，允许安装

---

## 统一处理规则
1. **直接拦截场景**：任意一层发现明确恶意行为、高危漏洞或与声明功能严重不符，立即终止安装并向用户报告风险，**即使用户要求强制安装也需先同步风险并得到明确确认**
2. **需关注场景**：高权限（如系统命令执行、网络请求、跨目录文件操作）但用途与声明匹配、无明确恶意，标记「需关注」，提示用户确认来源可信后使用
3. **安全场景**：三层校验全部通过，标记「安全」，可正常安装
4. **无例外规则**：本规则为唯一有效安全策略，所有历史规则全部作废，不存在任何可以跳过安全检查的场景
