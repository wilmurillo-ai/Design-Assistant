# 日期选择器处理指南

## 概述

本文档提供日期选择器的完整操作指导。

**适用场景**：
- 执行浏览器自动化任务时遇到日期选择器（LOG阶段、DATASAVER录制工作流验证阶段）
- 从DATASAVER录制工作流.json或执行日志中识别日期选择器场景（WORKFLOW阶段、DATASAVER录制工作流转换阶段）
- 生成SKILL.md时需要描述日期选择操作（CREATOR阶段）
- 用户使用生成的Skill操作日期控件（SKILL使用阶段）

**核心原则**：
- 直接使用 `chrome_accessibility_snapshot` 识别元素
- 观察 → 判断 → 操作 → 验证

---

## 识别条件

**满足以下任一条件即判定为日期选择器场景**：

1. **selector包含关键词**：`date`, `calendar`, `picker`, `month`, `year`, `time`, `date-row`, `date-cell`
2. **description是纯数字**：1-31范围内的数字（如"23"、"9"）
3. **连续多个event-click步骤**：连续点击不同的数字
4. **文本特征明显**：元素或附近文本包含"日期"、"时间"、"开始"、"结束"、"~"、"-"等

---

## 操作指导

### 第一步：定位并打开日期选择器

**使用工具**：`chrome_accessibility_snapshot`

**识别日期选择器的通用特征**：

```
1. 文本特征：包含"日期"、"时间"、"开始"、"结束"、"~"、"-"等关键词
2. 格式特征：显示日期格式如 "YYYY-MM-DD" 或 "开始时间 ~ 结束时间"
3. 元素类型：通常是 input、label 或带有日历图标的容器
4. 位置特征：通常在筛选区域，靠近时间相关标签

找到后点击打开日期面板
```

**执行方式**：
- 使用 `chrome_accessibility_snapshot` 获取页面元素树
- 根据上述特征查找日期选择器
- 使用 `chrome_click_element` 点击（通过uid或selector）

---

### 第二步：识别日历结构

打开日历后，再次通过 `chrome_accessibility_snapshot` 观察日历面板：

```
关键元素识别：

1. 月份标题：
   - 通常显示 "YYYY年MM月" 格式
   - 双月日历会显示两个月份标题
   - 通常是 h2、h3 或 div 元素

2. 导航按钮：
   - 通常是 span 或 button 元素
   - 文本为空但尺寸较小（如 16x16）
   - 位于月份标题两侧
   - 左侧按钮向前（上个月），右侧按钮向后（下个月）
   - 可能有单箭头< >控制月份和双箭头<< >>控制年份

3. 日期网格：
   - div 元素，文本为数字 1-31
   - 同一个数字可能出现多次（当前月 + 相邻月的灰色日期）
   - 通常按周排列（周一到周日）
```

---

### 第三步：判断月份位置

**双月日历布局判断**：

```
1. 从 snapshot 读取两个月份标题的文本（如 "2025年12月" 和 "2026年1月"）
2. 解析目标日期的月份（如 {start_date} = "2025-11-15" → 目标月份 = 11）
3. 比较目标日期的月份与当前显示的月份
4. 决定导航方向：
   - 目标月份 < 左侧月份 → 点击向前按钮 <
   - 目标月份 > 右侧月份 → 点击向后按钮 >
   - 目标月份在可见范围内 → 直接选择日期
```

**导航操作**：
- 点击导航按钮（使用 `chrome_click_element`）
- 重新获取 `chrome_accessibility_snapshot`
- 确认当前显示的月份
- 重复直到目标月份可见

---

### 第四步：区分同名日期

当日历显示多个相同数字时（如多个"8"），需要判断哪个是目标月份的日期：

```
方法1：根据月份标题位置推断
- 左侧月份的日期通常在左半区域
- 右侧月份的日期通常在右半区域
- 灰色日期（相邻月份）通常在边缘

方法2：观察日期的排列顺序
- 每个月的日期按周排列（周一到周日）
- 月初1号前面的数字是上月末尾
- 月末后面的数字是下月开头

方法3：通过坐标相对位置判断
- 比较日期元素与对应月份标题的 x 坐标
- 日期应该在其所属月份标题的下方区域
- 可以通过bbox信息判断
```

---

### 第五步：执行选择

**操作顺序（重要！）**：

```
1. 先点击开始日期
   - 如需要，导航到开始日期所在月份
   - 点击对应的日期数字（使用 chrome_click_element）
   - 该日期会高亮显示

2. 再点击结束日期
   - 如需要，导航到结束日期所在月份
   - 点击对应的日期数字
   - 选择完成，日历通常会自动关闭

3. 验证结果
   - 使用 chrome_accessibility_snapshot 检查日期输入框
   - 确认显示的日期范围是否正确
   - 如日历未自动关闭，查找并点击"确定"按钮
```

## 通用技巧

### 技巧1：导航按钮识别

```
导航按钮的通用特征：

1. 位置：在月份标题的左右两侧
2. 尺寸：通常较小（12x12 到 20x20）
3. 文本：通常为空，或包含 < > << >> 符号
4. 元素类型：span 或 button

识别方法：
- 找到月份标题元素
- 查找其左右两侧的小尺寸可点击元素
- 通过位置关系判断：左侧向前，右侧向后
```

### 技巧2：处理跨月选择

```
当开始日期和结束日期跨越多个月时：

1. 计算导航次数
   例：当前显示 2025-12，目标 2025-10
   需要向前导航 2 次

2. 循环导航
   - 点击导航按钮
   - 获取新的 accessibility_snapshot
   - 确认当前显示的月份
   - 重复直到目标月份可见

3. 导航完成后再选择日期
```

### 技巧3：避免点击错误日期

```
日历边缘的灰色日期属于相邻月份，不要点击！

识别灰色日期：
- 通常在月初（1号之前）或月末（最后一天之后）
- 从 accessibility_snapshot 可能无法直接判断颜色
- 但可以通过逻辑判断：
  - 如果当前月是12月，看到的28、29、30在1号前面 → 是11月的日期
  - 如果当前月是12月，看到的1、2、3在31号后面 → 是1月的日期
```

## 常见问题

### 问题1：导航按钮找不到

**症状**：无法定位月份导航按钮

**解决方案**：
1. 使用 `chrome_accessibility_snapshot` 查找月份标题周围的小尺寸元素
2. 导航按钮通常是空文本的 span 或 button，尺寸16x16左右
3. 尝试通过bbox坐标判断位置关系
4. 如果仍找不到，尝试滚动日历面板使导航按钮可见

### 问题2：点击了错误的日期

**症状**：点击了相邻月份的灰色日期

**解决方案**：
1. 严格判断日期是否在目标月份范围内
2. 对于每个月，1号之前和最后一天之后的日期都是其他月份
3. 使用坐标位置或月份标题位置辅助判断
4. 优先点击bbox中心坐标在月份标题正下方的日期元素

### 问题3：日历没有自动关闭

**症状**：选择完日期后日历面板仍然打开

**解决方案**：
1. 某些日历需要点击"确定"按钮
2. 使用 `chrome_accessibility_snapshot` 查找"确定"、"确认"、"应用"等按钮
3. 点击后验证日期输入框是否已更新

### 问题4：日期范围顺序错误

**症状**：结束日期早于开始日期，日历报错

**解决方案**：
1. 必须先点击开始日期，再点击结束日期
2. 不要颠倒顺序
3. 如果需要修改，可能需要重新打开日历面板

---

## 注意事项

1. **操作顺序**：必须先选开始日期，再选结束日期
2. **每次导航后刷新状态**：点击导航按钮后重新获取 accessibility_snapshot
3. **验证选择结果**：确认日期输入框显示正确的范围
4. **UID必须在10秒内使用**：获取 accessibility_snapshot 后，必须立即使用返回的 UID 进行点击操作，超过10秒页面可能变化导致UID失效

## 禁止事项

<mandatory-rule>

**❌ 禁止直接用 textQuery 搜索日号**

错误做法：
```
chrome_get_interactive_elements(textQuery="19")  // ❌ 会匹配到多个"19"，包括灰色日期
```

正确做法：
```
1. 使用 chrome_accessibility_snapshot 获取完整日历结构
2. 分析月份标题位置，判断目标日期属于哪个月份
3. 通过 bbox 坐标或位置关系找到正确的日期元素
4. 使用该元素的 UID 进行点击
```

**原因**：日历中同一个数字可能出现3次以上（左侧月份、右侧月份、灰色日期），直接搜索会匹配到错误的日期。

</mandatory-rule>

## 失败重试策略

当日期选择操作失败时：

### 情况1：点击了错误的日期
1. 重新获取 `chrome_accessibility_snapshot`
2. 检查当前日期输入框显示的值
3. 如果值错误，点击日期输入框重新打开日历
4. 按正确流程重新选择

### 情况2：日历面板意外关闭
1. 重新点击日期输入框打开日历
2. 从头开始执行选择流程
3. 注意检查之前是否已选中开始日期

### 情况3：导航按钮点击无响应
1. 重新获取 `chrome_accessibility_snapshot`
2. 确认导航按钮的 UID 是否已变化
3. 使用新的 UID 重试点击

### 情况4：UID失效（超过10秒）
1. 重新调用 `chrome_accessibility_snapshot`
2. 获取新的 UID
3. 立即使用新 UID 进行操作

## SKILL.md 生成模板

当CREATOR.md生成包含日期选择器步骤的SKILL.md时，**必须**按以下模板生成：

```markdown
### 步骤N：设置日期范围

**工具组合**：`chrome_accessibility_snapshot` + `chrome_click_element`

**参数**：
- start_date: `{start_date}` - 开始日期，格式YYYY-MM-DD
- end_date: `{end_date}` - 结束日期，格式YYYY-MM-DD

---

##### 子步骤 N.1：打开日期选择器

1. 使用 `chrome_accessibility_snapshot` 识别日期选择器
   - **特征**：文本包含"日期"、"时间"、"开始"、"结束"或日期格式 "YYYY-MM-DD"
   - **元素类型**：input、label 或带日历图标的容器

2. 使用 `chrome_click_element` 点击打开日历面板

##### 子步骤 N.2：识别日历结构

3. 使用 `chrome_accessibility_snapshot` 观察日历面板，识别以下元素：
   - **月份标题**：左右两个月份（如 "2025年9月" 和 "2025年10月"）
   - **导航按钮**：< >（月份）、<< >>（年份）
   - **日期网格**：文本为 1-31 的 div 元素

##### 子步骤 N.3：通用日期选择流程

**对开始日期和结束日期分别执行以下流程**：

**A. 解析目标日期**
- 示例：`"2025-09-19"` → 目标年份=2025，目标月份=9，目标日号=19

**B. 导航到目标月份**（如需要）
- 读取当前显示的左侧月份和右侧月份
- 判断导航方向：
  - 目标月份 < 左侧月份 → 点击 **<** 向前
  - 目标月份 > 右侧月份 → 点击 **>** 向后
  - 目标月份在可见范围 → 跳过导航
- 每次点击后重新 `chrome_accessibility_snapshot`，重复直到目标月份可见

**C. 查找并点击目标日期**
- 从 snapshot 查找所有文本为目标日号的元素
- **区分同名日期**（关键！）：
  - 比较日期元素的 **x坐标** 与目标月份标题的 x坐标
  - 日期应在其所属月份标题的正下方区域（通过 bbox 判断）
  - **避免**：月初1号前、月末最后一天后的灰色日期
- 使用 `chrome_click_element` 点击（通过 **UID**，⚠️ 10秒内使用）

##### 子步骤 N.4：执行日期选择

4. **选择开始日期**：对 `{start_date}` 执行子步骤N.3流程

5. **选择结束日期**：对 `{end_date}` 执行子步骤N.3流程
   - 如果与开始日期同月，跳过导航直接选择

##### 子步骤 N.5：验证结果

6. 使用 `chrome_accessibility_snapshot` 检查日期输入框
7. 确认显示的日期范围格式正确（如 "2025-09-19 ~ 2025-09-30"）
8. 如果日历未自动关闭，查找并点击"确定"按钮

```
