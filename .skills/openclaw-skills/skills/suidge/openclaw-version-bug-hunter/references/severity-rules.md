# 严重程度判定规则

**用途**：帮助判断 OpenClaw 版本 bug 的严重程度，辅助升级决策。

---

## 🔴 Critical / 严重级别

**定义**：导致系统无法正常使用、数据丢失、安全漏洞的问题。

### 典型症状

| 症状 | 示例 | 建议 |
|------|------|------|
| **崩溃/退出** | Gateway 启动后 6-7 分钟崩溃 | 暂缓升级 |
| **数据丢失** | 配置不持久化、memory 文件被覆盖 | 暂缓升级 |
| **无限循环** | 应用自我复制、CPU 100% | 立即回退 |
| **安全漏洞** | 凭证泄露、未授权访问 | 暂缓升级 |
| **系统冻结** | macOS 系统级卡顿、强制关机 | 暂缓升级 |

### 判断标准

- Issue 标题包含：`Critical`、`CRASH`、`data loss`、`security`
- 用户反馈：无法启动、无法使用核心功能
- 影响范围：所有用户或特定平台全体用户

---

## 🟠 Regression / 回归问题

**定义**：之前版本正常工作，当前版本失效的功能。

### 典型症状

| 症状 | 示例 | 建议 |
|------|------|------|
| **性能下降** | API 调用从 1 秒变 30-60 秒 | 谨慎升级 |
| **功能失效** | OAuth 认证失败、MCP 工具不响应 | 谨慎升级 |
| **配置忽略** | 代理变量、环境变量不生效 | 谨慎升级 |
| **兼容性破坏** | 旧配置格式不兼容、插件加载失败 | 谨慎升级 |

### 判断标准

- Issue 标签包含：`regression`、`bug:behavior`
- Issue 标题包含：`Regression`、`broken`、`failed`、`ignored`
- 用户反馈：「之前版本正常，升级后不行」

---

## 🟡 General Bugs / 一般问题

**定义**：不影响核心功能，但有缺陷或体验问题。

### 典型症状

| 症状 | 示例 | 建议 |
|------|------|------|
| **UI 问题** | 按钮错位、样式异常 | 可升级 |
| **非核心功能失效** | 某个冷门渠道发送失败 | 可升级 |
| **文档错误** | 文档与实际行为不符 | 可升级 |
| **偶发问题** | 特定条件下才触发 | 可升级 |

### 判断标准

- Issue 标签仅有：`bug`（无 Critical/regression）
- 影响范围：少数用户或特定场景
- 有 workaround 或替代方案

---

## 📊 升级决策矩阵

| Critical | Regression | General | 建议 |
|----------|------------|---------|------|
| 0 | 0-2 | 任意 | ✅ 推荐升级 |
| 0 | 3-5 | 任意 | ⚠️ 谨慎升级（评估影响） |
| 1-2 | 任意 | 任意 | ⚠️ 谨慎升级（看是否有 workaround） |
| 3+ | 任意 | 任意 | ❌ 暂缓升级 |
| 任意 | 5+（核心功能） | 任意 | ❌ 暂缓升级 |

---

## 🧪 验证方法

升级后建议执行以下验证：

```bash
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 检查配置持久化
openclaw config show

# 3. 测试核心功能
openclaw doctor

# 4. 测试常用渠道
# （根据主人使用的渠道）

# 5. 观察 24 小时
# 留意日志是否有异常
```

---

## 📝 Issue 可信度评估

不是所有 Issue 报告都同等可信。评估维度：

| 维度 | 高可信度 ✅ | 低可信度 ❌ |
|------|------------|------------|
| **复现步骤** | 详细、可复现 | 无或模糊 |
| **日志/截图** | 提供日志、截图 | 无技术细节 |
| **影响范围** | 多用户确认 | 单用户报告 |
| **情绪化程度** | 客观描述 | 情绪化投诉 |

**示例**：
- #64552（性能回归）：高可信度 ✅（详细复现步骤、多用户确认）
- #64745（macOS 崩溃）：低可信度 ❌（无日志、无复现步骤、情绪化）

*银月注：此规则供主人在升级前快速判断风险～🌙*
