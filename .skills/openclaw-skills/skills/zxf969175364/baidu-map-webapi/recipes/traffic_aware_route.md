# 未来路况路线：预测未来某时刻出发的耗时

> **适用摘要**: 用户想知道"下午3点出发要多久"、"明天早高峰的路况如何"。⚠️ 需要高级权限。

## 触发意图

遇到以下用户需求时使用本 recipe：
- "下午3点出发去机场大概要多久？"
- "明天早上8点出发，路况怎样？"
- "预测一下[未来时间]出发的耗时"
- "比较现在和晚高峰出发的路线耗时"

## 前置条件

| 条件 | 要求 |
|---|---|
| 必要输入 | 起点坐标（BD09LL）、终点坐标、未来出发时间 |
| 坐标系 | BD09LL（百度坐标系） |
| 权限 | ⚠️ **高级权限**：需在百度地图控制台开通「未来驾车路线规划」高级服务 |
| 时间范围 | 出发时间必须在未来 **7天** 内（超出范围将退化为估算） |

## 调用链

```
用户输入: 起终点 + 未来出发时间
  → [可选] Step 1: administrative_region_search（地名 → 坐标 + uid）
  → Step 2: driving_route_planning（含未来出发时间参数）
  → 返回按未来路况预测的耗时和路线
```

> 设置出发时间后，路线规划返回的耗时是**按该时刻预测路况**计算的结果，
> 而不是当前时刻的路况耗时。这是「未来路线规划」和普通路线规划的核心区别。

## 分步说明

### [可选] Step 1: administrative_region_search（当终点为地名时）

**目的**: 将目的地名称转为坐标和 uid

**参考**: `../references/administrative_region_search.md`

**需要提供**: 目的地名称、城市名（提高准确率）

**从响应获取**: 目的地坐标和 POI 唯一标识符（uid）

### Step 2: driving_route_planning（含未来出发时间）

**目的**: 按未来指定时刻预测路况，规划路线并返回耗时

**参考**: `../references/driving_route_planning.md` → 高级权限: `../references/capabilities/future_driving_route.md`

**需要提供**: 起点坐标、终点坐标、终点 uid（若有）、未来出发时间（UNIX 时间戳，秒级，限未来7天内）

**从响应获取**:
- 按指定出发时刻预测路况的耗时
- 路线距离
- 各路段路况详情

## 常见错误

| 错误 | 原因 | 解决方法 |
|---|---|---|
| 时间戳单位错误 | 使用了毫秒级时间戳 | 确保使用秒级 UNIX 时间戳 |
| 接口返回正常但预测不准 | 超出7天范围 | 出发时间限制在未来7天内，超出后结果仅为估算 |
| 参数无效错误 | 设置了过去的时间 | 只能预测未来时刻 |

## 变体

- **查询历史路况下的耗时**（不同高级权限）: 改用历史出发时间参数，参考 `../references/capabilities/driving_route_duration.md`
- **建议出发时间**（反向推算）: 改用期望到达时间参数，参考 `smart_departure_time.md`
- **终点是地名**: 先用搜索建议接口获取坐标，参考 `route_to_named_place.md`
