# POI 搜索到详情：关键词搜索 → 获取完整 POI 信息

> **适用摘要**: 用户输入关键词（地名/商家名），获取该 POI 的完整详情（地址、电话、评分、营业时间、坐标等）。

## 触发意图

遇到以下用户需求时使用本 recipe：
- "查一下北京故宫的详细信息"
- "百度大厦的地址和电话是什么？"
- "搜索[POI名称]的营业时间"
- "获取某某餐厅的详细信息"

## 前置条件

| 条件 | 要求 |
|---|---|
| 必要输入 | POI 名称关键词 |
| 城市信息 | 建议提供城市名，避免搜到同名地点 |
| 权限 | 标准 AK 即可 |

## 调用链

```
用户输入关键词
  → Step 1: administrative_region_search（关键词 → uid + 基本信息）
  → Step 2: place_detail_search（uid → 完整 POI 详情）
  → 返回 POI 完整信息（地址、电话、评分、营业时间等）
```

> **为什么不直接用关键词查详情？**
> 详情接口只接受 POI 唯一标识符（uid），不支持关键词搜索。
> 必须先通过搜索建议接口实时获取 uid，不能使用历史缓存的 uid。

## 分步说明

### Step 1: administrative_region_search（关键词搜索获取 uid）

**目的**: 将用户输入的关键词转换为 POI uid

**参考**: `../references/administrative_region_search.md`

**需要提供**: 关键词、城市名（建议提供，避免同名歧义）

**从响应获取**: POI 唯一标识符（uid），传入 Step 2

> 结果按相关度排序，第一条最匹配。若有多个候选，可向用户展示选择列表。

### Step 2: place_detail_search（获取 POI 完整详情）

**目的**: 用 uid 获取 POI 的完整详细信息

**参考**: `../references/place_detail_search.md`

**需要提供**: Step 1 获取的 POI 唯一标识符、开启完整详情返回

**从响应获取**:
- POI 名称、详细地址
- 联系电话
- 坐标（BD09LL）
- 综合评分
- 营业时间
- 简介描述
- 导航引导点坐标（大型 POI 的入口坐标）
- 营业状态

## 常见错误

| 错误 | 原因 | 解决方法 |
|---|---|---|
| 直接硬编码 uid 字符串 | uid 会随数据更新而失效 | 每次必须通过搜索建议接口实时获取 uid |
| 详情字段为空（无评分等） | 未开启完整详情返回模式 | 开启详情返回级别获取完整信息 |
| 搜到错误城市的同名地点 | 未限制城市 | 提供城市名并开启城市限制 |
| 批量查询超过上限 | API 单次请求限制 | 批量查询需分批请求 |

## 变体

- **批量获取多个 POI 详情**: 使用批量 uid 查询，单次有数量上限
- **展示多个候选让用户选择**: Step 1 返回所有候选，展示给用户选择后，再用选中的 uid 查详情
- **规划导航路线**: 获取详情中的导航引导点坐标（比主坐标更精确，直达 POI 入口），传入路线规划接口
