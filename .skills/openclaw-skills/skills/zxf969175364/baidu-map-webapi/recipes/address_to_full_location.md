# 地址解析：输入地址文本 → 获取坐标 + 行政区划信息

> **适用摘要**: 将非结构化地址文本（如"北京市海淀区上地十街10号"）解析为坐标及结构化行政区划信息（省/市/区/街道及编码）。

## 触发意图

遇到以下用户需求时使用本 recipe：
- "把这个地址解析成经纬度：北京市海淀区上地十街10号"
- "这批地址需要地理编码，顺便获取行政区划编码"
- "需要知道某地址对应的省市区街道及其编码"
- "地址解析并需要结构化行政区划信息"

## 前置条件

| 条件 | 要求 |
|---|---|
| 必要输入 | 文本地址（越详细越精确，建议包含省市区） |
| 坐标系 | 输出坐标默认为 BD09LL（经纬度），可指定返回 GCJ02 或 BD09MC |
| 权限 | 标准 AK 即可 |

## 调用链

```
输入地址文本
  → geocoding（地理编码）
  → 返回坐标 + 省/市/区/街道名称及编码
```

> 开启 `extension_poi_infos=true` 可同时获取行政区划详情（省、市、区、街道及 `town_code`）。

## 分步说明

### Step 1: geocoding（地理编码，一步完成）

**目的**: 解析地址文本，返回坐标和行政区划信息

**参考**: `../references/geocoding.md`

**关键参数**:
- `address`：待解析的地址文本
- `city`：（可选）城市名，用于同名地址去歧义
- `extension_poi_infos=true`：开启后额外返回行政区划详情
- `ret_coordtype`：指定返回坐标系（默认 bd09ll）

**从响应获取**:
- 坐标（经纬度）
- `result.precise`：是否精确打点（1=精确，0=模糊）
- `result.comprehension`：地址理解度评分（0-100，越高越准确）
- `poi_infos[0]`（开启 extension_poi_infos 时）：
  - 省（`province`）、市（`city`）、区县（`district`）
  - 街道/乡镇（`town`）及编码（`town_code`）

## 常见错误

| 错误 | 原因 | 解决方法 |
|---|---|---|
| 模糊打点（precise=0） | 地址不够详细，无法精确匹配 | 提供更完整的地址（如带门牌号），或接受模糊结果 |
| 行政区划字段为空 | 未开启 extension_poi_infos | 请求时加上 `extension_poi_infos=true` |
| 同名地址返回错误城市 | 未指定城市 | 传入 `city` 参数限定城市范围 |
| 地址理解度低（comprehension < 50） | 地址格式混乱或过于简短 | 提供标准格式的完整地址 |

## 变体

- **批量地址解析**（数据清洗场景）: 循环调用本接口，注意限速（避免超出配额）
- **仅获取坐标（轻量地理编码）**: 不传 `extension_poi_infos`，响应更轻量，只取 `result.location`
- **逆向查询**（坐标 → 地址）: 改用 `../references/global_reverse_geocoding.md` 或 `../references/reverse_geocoding_agent.md`
