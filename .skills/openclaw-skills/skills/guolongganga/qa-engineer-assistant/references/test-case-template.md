# 测试用例标准模板

## 表格格式

每次输出测试用例时，使用以下 Markdown 表格格式：

| 用例编号 | 模块 | 用例标题 | 前置条件 | 测试步骤 | 输入数据 | 预期结果 | 优先级 | 用例类型 |
|---------|------|---------|---------|---------|---------|---------|--------|---------|
| TC-001  | 登录模块 | 正确账号密码登录成功 | 用户已注册；系统正常运行 | 1. 打开登录页<br>2. 输入用户名<br>3. 输入密码<br>4. 点击登录 | 用户名: test@example.com<br>密码: Test@1234 | 登录成功，跳转至首页，顶部显示用户名 | P0 | 正向用例 |

---

## 字段说明

| 字段 | 说明 |
|------|------|
| **用例编号** | 格式：`模块缩写-序号`，如 `LOGIN-001`、`ORDER-003` |
| **模块** | 所属功能模块名称 |
| **用例标题** | 一句话描述测试目的，格式：`[条件] 执行[操作] 结果[预期]` |
| **前置条件** | 执行本用例前必须满足的环境/数据条件 |
| **测试步骤** | 按顺序编号，步骤要精确可复现 |
| **输入数据** | 具体的测试数据，不能写"正确的密码"，要写真实值或说明取值规则 |
| **预期结果** | 可验证、可观测的结果描述（界面变化/返回值/数据库状态） |
| **优先级** | P0=冒烟必测 / P1=核心功能 / P2=重要 / P3=边缘场景 |
| **用例类型** | 正向用例 / 负向用例 / 边界用例 / 性能用例 / 安全用例 |

## 设计覆盖检查清单

生成测试用例后，附上以下覆盖情况确认：

```
## 覆盖检查清单
- [ ] 正常流程（Happy Path）
- [ ] 必填字段为空
- [ ] 字段超出最大长度限制
- [ ] 字段低于最小长度限制
- [ ] 特殊字符输入（如 `<script>`, `'`, `--`）
- [ ] 重复提交 / 幂等性
- [ ] 无权限用户访问
- [ ] 并发操作（如适用）
- [ ] 网络异常 / 超时（如适用）
- [ ] 边界值（最大值、最小值、临界值±1）
```

## 完整示例：用户登录功能测试用例

> 场景说明：用户名为邮箱格式，密码 8-20 位含大小写字母和数字，连续失败 5 次锁定账号 30 分钟

| 用例编号 | 模块 | 用例标题 | 前置条件 | 测试步骤 | 输入数据 | 预期结果 | 优先级 | 类型 |
|---------|------|---------|---------|---------|---------|---------|--------|------|
| LOGIN-001 | 登录 | 有效账号密码 → 登录成功 | 账号已注册且状态正常 | 1.打开登录页 2.输入邮箱 3.输入密码 4.点击登录 | 邮箱: user@test.com 密码: Test@1234 | 跳转首页，显示用户昵称，token 写入 cookie | P0 | 正向 |
| LOGIN-002 | 登录 | 密码错误 → 提示错误且不登录 | 账号已注册 | 同上，密码输入错误值 | 邮箱: user@test.com 密码: Wrong@123 | 停留登录页，提示"账号或密码错误"，失败次数+1 | P0 | 负向 |
| LOGIN-003 | 登录 | 邮箱格式非法 → 前端校验拦截 | 无 | 输入非邮箱格式用户名 | 邮箱: notanemail 密码: Test@1234 | 邮箱输入框下方显示"请输入有效邮箱"，不触发接口请求 | P1 | 负向 |
| LOGIN-004 | 登录 | 邮箱为空 → 不可提交 | 无 | 邮箱留空，密码填写，点击登录 | 邮箱: (空) 密码: Test@1234 | 登录按钮置灰 或 提示"邮箱不能为空" | P1 | 负向 |
| LOGIN-005 | 登录 | 密码最短边界值 8 位 → 登录成功 | 账号密码为 8 位 | 正常登录操作 | 邮箱: user@test.com 密码: Aa123456 | 登录成功 | P2 | 边界 |
| LOGIN-006 | 登录 | 密码 7 位（低于最小长度）→ 校验拦截 | 无 | 输入 7 位密码 | 密码: Aa12345 | 提示"密码长度不符合要求" | P2 | 边界 |
| LOGIN-007 | 登录 | 密码最长边界值 20 位 → 登录成功 | 账号密码为 20 位 | 正常登录操作 | 密码: Aa123456789012345678 | 登录成功 | P2 | 边界 |
| LOGIN-008 | 登录 | 密码 21 位（超出最大长度）→ 校验拦截 | 无 | 输入 21 位密码 | 密码: Aa1234567890123456789 | 提示"密码长度不符合要求" | P2 | 边界 |
| LOGIN-009 | 登录 | 连续失败 5 次 → 账号锁定 | 账号正常，已失败 4 次 | 再次输入错误密码 | 密码: WrongPwd1 | 提示"账号已锁定，请 30 分钟后重试"，后续正确密码也无法登录 | P1 | 负向 |
| LOGIN-010 | 登录 | SQL 注入尝试 → 安全拦截 | 无 | 邮箱字段输入注入语句 | 邮箱: `' OR 1=1--` | 提示格式错误或返回 400，不返回任何用户数据 | P1 | 安全 |
