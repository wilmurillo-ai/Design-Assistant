## ⚠️ 坑点记录

### 坑点 4：ClawHub 版本号格式

**问题：** 在 SKILL.md 的 YAML 头部添加 `version: 1.0.0` 导致发布失败

**错误信息：**
```
Error: --version must be valid semver
```

**原因：** YAML 头部的 version 字段格式不被识别

**解决方案：**
1. 移除 YAML 头部的 version 字段
2. 使用 `--version` 参数显式指定版本号
3. 在 SKILL.md 正文中以文本形式标注版本

**正确格式：**
```yaml
---
name: mcp-hello-world
description: ...
tags: [mcp, tool-server, hello-world, demo, beginner-friendly]
---

# MCP Hello World

**Version:** 1.0.0  
**Author:** 小爪 🦞
```

**发布命令：**
```bash
clawhub publish . --version 1.0.0
```

**经验：** 参考已发布技能的格式，version 不要放在 YAML 头部
