# Practice Flow

Use this when the user wants to practice now instead of hearing a product explanation.

## Baseline Requirements

- give one small exercise at a time
- make it finishable today
- avoid heavy setup
- include a check mechanism whenever possible

## Standard Template

Prefer this structure.

### Today's Practice

Give a short title.

### Objective

State what today's work is meant to improve.

### Task

Give one clear action, for example:

- solve one question
- break down one concept
- review one mistake pattern
- write one short answer
- complete one discussion-style prompt

### Suggested Time

If the user gave no timing constraint, default to 5-15 minutes.

### Check Method

Include at least one of these:

- sample answer
- checklist
- scoring rubric
- reflection question

### Tomorrow's Continuation

End with one small next step.

## Keep It Aligned With The Public Style

If you want the practice to feel consistent with the public MeiRiYiLian voice, you can emphasize:

- no bloated essays
- steady execution
- point-by-point breakthroughs
- do fewer questions, but do better questions

## Optional Enhancements

Add these when useful.

### AI Teach

When the user is blocked by a concept:

- give a one-line explanation first
- then a layered explanation

### AI Learn

When the user does not know what to study first:

- give a lightweight adaptive path
- then today's smallest actionable task

### AI Practice

When the user primarily needs training:

- go straight to a prompt
- optionally add a clone-style or error-analysis angle

### Discussion Mode

When the user needs deeper understanding:

- add one "why" question
- add one "what changes if the condition changes" question

## Do Not

- dump a huge curriculum by default
- give too many exercises at once
- turn the practice into a marketing paragraph
