# 刷单分析指标说明

## 分析维度

### 1. 时间间隔分析
- **正常情况**: 用户报名后需要一定时间完成任务
- **可疑情况**: 报名到提交时间间隔过短（<5分钟）
- **判断标准**:
  - 完成时间 < 5分钟: 高度可疑
  - 完成时间 5-10分钟: 中度可疑
  - 完成时间 > 10分钟: 正常范围

### 2. 发单人集中度分析
- **正常情况**: 用户会接取不同发单人的任务
- **可疑情况**: 频繁接取同一发单人的任务
- **判断标准**:
  - 同一发单人 > 10次: 高度可疑
  - 同一发单人 5-10次: 中度可疑
  - 同一发单人 < 5次: 正常范围

### 3. 任务重复度分析
- **正常情况**: 用户不会重复接取相同任务
- **可疑情况**: 多次接取相同任务
- **判断标准**:
  - 重复任务 > 2次: 可疑
  - 重复任务 > 5次: 高度可疑

### 4. 置顶刷新分析
- **正常情况**: 用户通常接取置顶的任务（更容易看到）
- **可疑情况**: 大量接取未置顶的任务
- **判断标准**:
  - 未置顶任务比例 > 70%: 可疑（可能通过非正规渠道获取任务）
  - 未置顶任务比例 > 50%: 需关注

## 风险等级评估

### 高风险
- 满足以下任一条件:
  - 2个及以上高风险指标
  - 完成时间<5分钟的记录占比>50%
  - 同一发单人任务>10次

### 中风险
- 满足以下任一条件:
  - 1个高风险指标或2个及以上中风险指标
  - 完成时间<5分钟的记录占比30-50%
  - 同一发单人任务5-10次

### 低风险
- 无明显异常指标
- 或仅有1个低度可疑指标

## 数据库表结构参考

### t_task_submit (任务提交表)
- id: 提交ID
- taskId: 任务ID
- subId: 提交者ID
- claimTime: 报名时间 (毫秒时间戳)
- subTime: 提交时间 (毫秒时间戳)
- examineTime: 审核时间 (毫秒时间戳)
- status: 状态 (2=已完成)
- taskAmout: 任务金额
- examineText: 审核备注
- ip: IP地址

### t_task (任务表)
- id: 任务ID
- taskNo: 任务编号
- title: 任务标题
- userNo: 发单人编号

### t_user (用户表)
- id: 用户ID
- userId: 用户唯一标识

### t_task_subctx (任务上下文表)
- subId: 提交ID
- deviceId: 设备ID
- appId: 应用ID
- channel: 渠道
- ip: IP地址

### t_ip_config (IP配置表)
- id: IP整数表示 (INET_ATON)
- provice: 省份
- city: 城市

### t_task_order (任务订单表)
- taskId: 任务ID
- startTime: 置顶开始时间
- expiredTime: 置顶结束时间
- status: 状态 (1,100=有效)
