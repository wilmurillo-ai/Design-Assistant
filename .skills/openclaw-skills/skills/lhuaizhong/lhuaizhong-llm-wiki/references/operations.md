# Operations

Use these checklists during maintenance.

## Ingest checklist

1. Identify the source and preserve it under `raw/` when appropriate.
2. Determine the main entities, claims, events, and decisions.
3. Update existing wiki pages first.
4. Create new pages only for durable concepts.
5. Add links between related pages.
6. Add or refresh source references.
7. Record the change in the knowledge log.

## Query checklist

1. Search `INDEX.md` and likely hub pages.
2. Read the top relevant wiki pages.
3. Read raw material only if the wiki is incomplete or suspicious.
4. Answer the user.
5. If the question exposed a stable missing insight, update the wiki.

## Reindex checklist

1. Find duplicate or overlapping pages.
2. Merge or rename for canonical structure.
3. Update links after moves.
4. Refresh `INDEX.md` and hub pages.
5. Split pages that are too broad.

## Lint checklist

Review for:

- conflicting summaries
- unsupported claims
- stale pages after recent ingest
- orphan pages
- inconsistent naming
- missing source pointers
- unresolved ambiguity that should be tagged as open

## Provenance guidance

For high-value claims, include enough provenance that a future agent can verify where the page came from.

Useful forms:

- direct raw file path
- source page links
- dates of ingest or last verification
- short notes on confidence or disagreement
