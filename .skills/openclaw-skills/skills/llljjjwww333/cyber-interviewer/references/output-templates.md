# Output Templates

## Compact Report

Use this structure for the default one-shot output:

```markdown
## Candidate Snapshot

## Resume Risks
- ...

## Project Deep-Dive
### Project: <name>
- Verified strengths:
- Likely weak points:
- Interviewer pressure points:

## Current Interview Signals
- Source-dated trend 1
- Source-dated trend 2

## Algorithm Pressure
- Enabled or not
- Why
- Focus areas
- Challenge question 1
- Challenge question 2

## Avoidance Guide
- Pitfall:
  Better framing:

## Mock Interview Plan
- Warm-up
- Deep technical
- Failure modes
- Behavioral

## Interview Flow
- Question theme 1
- Question theme 2
- Stop condition

## First Question
<single concrete question>
```

## Post-Interview Report

Use this structure after the questioning phase ends:

```markdown
## Overall Assessment
- ...

## Strongest Answers
- ...

## Weakest Answers
- What was strong:
- What was weak or missing:
- What evidence should be added:

## Better Answer Shapes
- ...

## Final Recommendation
- ...
```

## Avoidance Guide Format

For each pitfall, include:

- `trigger`: what interviewer prompt is likely to surface it
- `risk`: why the current answer or resume line is weak
- `repair`: how to reframe with concrete evidence
- `proof`: which file, metric, or design decision supports the better answer
