# 改写示范（Before & After Examples）

---

## 示例 1：功能描述（De-jargon）

**Before:** 告别"炼丹式"繁琐配置，一键部署。复杂的 API 调优、繁琐的环境挂载及权限设置，对普通用户是巨大挑战。

**After:** 零门槛上手，一键直接启用。忘记那些复杂的参数调优与底层设置，点击即可开始使用。

**心法:** 去掉了"炼丹、环境挂载"等极客黑话，用"零门槛"传递赋权感而非焦虑感。

---

## 示例 2：安全说明（Empowerment over Anxiety）

**Before:** 拒绝"代码投毒"与隐私裸奔。恶意代码投毒与隐私泄漏事件屡见不鲜。

**After:** 深度安全审计，守护数据隐私。从源头杜绝恶意软件与数据泄露隐患，让你的每一次使用都安心无忧。

**心法:** "代码投毒"和"隐私裸奔"属于恐吓式营销，转化为正向的安全承诺。

---

## 示例 3：CTA 按钮（Rhythm & CTA）

**Before:** 了解更多 → / 了解更多 → / 了解更多 →

**After:** 探索模型 → / 查看评测 → / 开始学习 →

**心法:** 每个 CTA 必须是具体动作指令，降低用户的决策成本。

---

## 示例 4：Feature → Benefit 转换

**Before:** 动态沙箱隔离并实时监控所有 API 调用，防止恶意代码注入和越权访问。

**After:** 在安全的隔离环境中运行，严密防止你的本地文件被越权窃取。你的数据，始终由你掌控。

**心法:** 从"系统怎么运作"转为"用户得到什么保障"。用户不关心沙箱技术，只关心数据是否安全。

---

## 示例 5：心智模型类比

**Before:** 一键部署 Claude 智能体到你的本地环境，支持 MCP 协议扩展。

**After:** 像下载 App 一样添加你的 AI 助手。想加新能力？就像装插件一样简单。

**心法:** 用"下载 App"和"装插件"作为心智模型，让不懂技术的用户秒懂。

---

## 示例 6：并列标题对仗

**Before:**
- 快速入门指南
- 常见问题解答

**After:**

## 示例 7：恐吓 → 赋权

**Before:** 选错配置会烧钱！教程跟着做全是报错！零基础根本搞不定！

**After:** 提供最具性价比的透明方案。零基础也能顺利上手的实操指南，每一步都有截图和说明。

**心法:** 把焦虑转化为掌控感。用户要的是解决方案，不是恐吓。
