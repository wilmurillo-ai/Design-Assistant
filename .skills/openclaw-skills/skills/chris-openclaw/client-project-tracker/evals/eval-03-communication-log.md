# Eval 03: Log Communication and Update Deliverable

## Input
"Had a call with Pastor Mike. He loves the homepage, wants to move to interior pages. Also asked about adding a donation page. I said I'd quote it separately."

## Expected Behavior
1. Logs the call in Riverside's communication log (date, type: call, summary)
2. Updates homepage mockup deliverable to "approved"
3. Notes the donation page request in project notes (potential scope addition)
4. Advances interior pages to "in-progress"
5. Confirms all updates

## What to Watch For
- Does it infer multiple updates from a conversational message?
- Does it update the deliverable status based on context ("loves it" = approved)?
- Does it capture the scope creep note (donation page) without automatically adding it as a deliverable?
