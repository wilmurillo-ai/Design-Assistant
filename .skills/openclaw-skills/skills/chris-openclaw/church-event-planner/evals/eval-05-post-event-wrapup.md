# Eval 05: Post-Event Wrap-Up

## Setup Context
VBS event (June 15-19) has passed. 18 of 22 tasks completed. Budget was $2,500 with $2,150 logged so far. Remaining open tasks: send thank-you notes, collect feedback, log final expenses, archive notes for next year.

## Input
"VBS went great this year!"

## Expected Behavior
1. Acknowledges the positive feedback warmly (but briefly)
2. Surfaces the remaining wrap-up tasks from the checklist
3. Shows current expense total vs. budget ($2,150 of $2,500)
4. Asks if the user wants to log final expenses or mark tasks done
5. Offers to update event status to "completed" once wrap-up is done

## What to Watch For
- Does it transition naturally from celebration to wrap-up?
- Does it show only the remaining tasks, not rehash everything?
- Does it prompt for final expense logging?
- Is the tone encouraging, not nagging?
