# Flink 资源池管理技能

基于 `volc_flink` 命令行工具，管理 Flink 资源池，包括列举资源池、查看资源池详情、创建资源池等。

---

## 通用约定（必读）

本技能的基础约定与只读约定统一收敛在：

- `../../COMMON.md`
- `../../COMMON_READONLY.md`

本技能当前按只读层管理，用于资源池查询、规格比对和容量信息解释；若后续执行真实创建或修改，再切换到 mutation 约定。

### 1. 信息提取与智能选择（从其他 flink 技能借鉴）

#### 1.1 信息提取
从用户提问中提取关键信息：
- **Flink 项目名** (project_name)
- **资源池名/资源池 ID** (resource_pool_name/resource_pool_id)
- **操作类型**：列举资源池、查看资源池、创建资源池
- **资源池配置**：CPU 核数、内存大小、描述等

如果用户没有明确提供，主动询问缺失的关键信息。

#### 1.2 智能项目和资源池选择
**如果用户没有明确提供项目名或资源池名，按以下流程处理**：

**场景 A：用户只提供了资源池名，没有提供项目名**
1. 列出所有项目：`volc_flink projects list`
2. 在每个项目中列出资源池
3. 展示所有匹配的资源池供用户选择

**场景 B：用户没有提供任何信息**
1. 先列出所有项目供用户选择
2. 用户选择项目后，列出该项目下的所有资源池供用户选择
3. 用户选择资源池后，询问操作类型

**场景 C：用户提供的项目名或资源池名不明确**
1. 列出所有资源池供用户选择
2. 或者提供搜索功能

### 2. 操作类型与流程

#### 2.1 列举资源池（List Resource Pools）

**使用场景**：
- 查看所有可用的资源池
- 了解资源池的配置和状态
- 选择合适的资源池用于任务

**操作步骤**：
1. **列举资源池**
   ```bash
   volc_flink resource-pools list
   ```

2. **展示资源池信息**，包括：
   - 资源池 ID
   - 资源池名称
   - 资源池类型
   - 状态
   - CPU 核数
   - 内存大小

---

#### 2.2 查看资源池详情（View Resource Pool Detail）

**使用场景**：
- 查看资源池的详细配置
- 了解资源池的使用情况
- 检查资源池的状态

**操作步骤**：
1. **列举资源池**（如果用户没有提供资源池）
   ```bash
   volc_flink resource-pools list
   ```

2. **查看资源池详情**
   ```bash
   # 通过资源池 ID 查看
   volc_flink resource-pools detail -i <资源池ID>

   # 或者通过资源池名称查看
   volc_flink resource-pools detail -n <资源池名称>
   ```

3. **展示资源池详细信息**，包括：
   - 资源池 ID
   - 资源池名称
   - 资源池类型
   - 状态
   - CPU 核数
   - 内存大小
   - 描述（如果有）
   - 其他配置信息

---

#### 2.3 创建资源池（Create Resource Pool）

**使用场景**：
- 创建新的资源池用于任务
- 为特定项目创建专用资源池
- 扩容资源池能力

**操作步骤**：
1. **确认资源池配置**
   - 资源池名称（必填）
   - 描述（可选）
   - CPU 核数（默认 4）
   - 内存大小（GB，默认 16）

2. **风险确认** - 使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认
   - ⚠️ 资源池创建后会产生费用
   - ⚠️ 确认资源池配置合理
   - ⚠️ 确认资源池名称正确
   - ⚠️ 确认 CPU 和内存配置满足需求

3. **创建资源池**
   ```bash
   # 基本创建（使用默认配置）
   volc_flink resource-pools create -n <资源池名称>

   # 完整配置创建
   volc_flink resource-pools create \
     -n <资源池名称> \
     -d <描述> \
     -c <CPU核数> \
     -m <内存GB数>
   ```

4. **验证创建结果** - 列举资源池确认创建成功
   ```bash
   volc_flink resource-pools list
   ```

---

#### 2.4 资源池资源分布分析（Resource Pool Resource Distribution Analysis）

**使用场景**：
- 分析资源池被哪些任务占用
- 识别占用资源较多的任务
- 发现资源分配不平衡的情况
- 重点关注高资源消耗的任务

**操作步骤**：
1. **列举资源池**（如果用户没有提供资源池）
   ```bash
   volc_flink resource-pools list
   ```

2. **列举所有任务**（包括 RUNNING、STOPPED、FAILED 等状态）
   ```bash
   volc_flink jobs list
   ```

3. **获取每个任务的详情**，提取 ResourcePool 信息
   ```bash
   # 对于每个任务，获取详情
   volc_flink jobs detail -i <任务ID>
   ```

4. **按资源池分组统计任务**：
   - 统计每个资源池的任务数量
   - 统计每个资源池的 RUNNING 任务数量
   - 统计每个资源池的任务类型分布

5. **重点关注任务**：
   - 运行时间较长的任务
   - 状态为 FAILED 的任务
   - 资源池容量接近饱和的任务

## 资源池管理最佳实践

### 资源池配置建议
- **小型任务**：2核CPU，8GB内存
- **中型任务**：4核CPU，16GB内存
- **大型任务**：8核CPU，32GB内存或更多
- **超大型任务**：16核CPU，64GB内存或更多

### 资源池命名规范
- 使用清晰的命名，如 `project-name-pool`
- 包含环境信息，如 `prod`、`test`、`dev`
- 避免使用特殊字符

### 资源池使用注意事项
- **先测试再生产**：建议先在测试环境验证资源池
- **监控资源使用**：持续关注资源池的使用情况
- **及时释放闲置**：及时释放不再使用的资源池
- **合理规划容量**：根据任务需求规划资源池容量

3. **风险确认** - 执行任何资源池操作前，必须使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认
4. **验证操作结果** - 操作完成后，验证结果是否成功

### 通用注意事项

1. **先检测登录状态**：在执行任何操作前，先检测是否已登录
2. **先确认项目范围**：在执行任何操作前，明确确认是哪个项目
3. **先查看资源池列表**：创建或查看前，先查看当前资源池列表
4. **始终先确认风险**：在执行任何变更操作前，必须使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认

## 错误处理优化

### 常见错误及处理

#### 错误 1：未登录
**错误信息**：`❌ 请先登录`

**处理方式**：
- 友好提示："检测到未登录火山引擎账号，请先登录"
- 提供登录指引：请使用交互式登录 `volc_flink login`（或企业内部安全方式），详见 `../../COMMON.md`
- 停止后续操作，等待用户登录后重试

#### 错误 2：资源池不存在
**错误信息**：资源池 ID 或资源池名不存在

**处理方式**：
- 提示："未找到资源池，请检查资源池名或资源池 ID 是否正确"
- 提供帮助：列出该项目下的所有资源池供用户选择

#### 错误 3：资源池名称冲突
**错误信息**：资源池名称重复

**处理方式**：
- 提示："资源池名称重复，请使用其他名称"
- 提供帮助：列出当前资源池供参考
- 建议用户使用其他名称

#### 错误 4：资源池创建失败
**错误信息**：资源池创建失败

**处理方式**：
- 立即通知用户创建失败
- 建议检查资源池配置是否合理
- 建议检查配额是否足够
- 建议稍后重试

#### 错误 5：网络超时
**错误信息**：命令执行超时

**处理方式**：
- 提示："命令执行超时，请检查网络连接"
- 建议稍后重试
- 如果多次失败，建议检查火山引擎服务状态
