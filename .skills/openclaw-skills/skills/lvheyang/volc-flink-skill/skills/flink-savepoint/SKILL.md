# Flink 快照（Savepoint）管理技能

基于 `volc_flink` 命令行工具，管理 Flink 任务的快照（Savepoint）操作，包括创建快照、查询快照列表、从快照恢复任务、删除快照等。

---

## 通用约定（必读）

本技能的基础约定与变更约定统一收敛在：

- `../../COMMON.md`
- `../../COMMON_MUTATION.md`

本技能涉及创建、恢复、删除快照等状态变更操作，必须遵循 `COMMON_MUTATION.md` 中的 Risk Confirmation、变更前检查和操作后验证规则。

## 🎯 核心流程

### 0. 登录状态检测（从其他 flink 技能借鉴）
**在执行任何操作前，先检测登录状态！**

**检测步骤**：
1. 尝试执行一个简单的命令（如 `volc_flink config show`）
2. 如果提示"请先登录"，则提示用户需要登录
3. 提供登录指引：请使用交互式登录 `volc_flink login`（或企业内部安全方式），不要在对话/命令行参数中粘贴 AK/SK，详见 `../../COMMON.md`

**错误处理**：
- 如果检测到未登录，立即停止后续操作
- 友好提示用户需要先登录

### 1. 信息提取与智能选择（从其他 flink 技能借鉴）

#### 1.1 信息提取
从用户提问中提取关键信息：
- **Flink 项目名** (project_name)
- **任务名** (job_name)
- **操作类型**：创建快照、查询快照、恢复快照、删除快照
- **快照路径**（恢复时需要）
- **快照 ID**（删除时需要）

如果用户没有明确提供，主动询问缺失的关键信息。

#### 1.2 智能项目和任务选择
**如果用户没有明确提供项目名或任务名，按以下流程处理**：

**场景 A：用户只提供了任务名，没有提供项目名**
1. 列出所有项目：`volc_flink projects list`
2. 在每个项目中搜索匹配的任务名
3. 展示所有匹配的任务供用户选择

**场景 B：用户没有提供任何信息**
1. 先列出所有项目供用户选择
2. 用户选择项目后，列出该项目下的所有任务供用户选择
3. 用户选择任务后，询问操作类型

**场景 C：用户提供的项目名或任务名不明确**
1. 使用模糊搜索：`volc_flink jobs list --search <关键词>`
2. 展示匹配结果供用户选择

### 2. 操作类型与流程

#### 2.1 创建快照（Create Savepoint）

**使用场景**：
- 任务升级前备份
- 任务迁移前备份
- 定期数据备份
- 故障恢复前的安全备份

**操作步骤**：
1. **获取任务信息** - 确认任务当前状态
   ```bash
   volc_flink jobs detail -i <任务ID>
   ```

2. **风险确认** - 使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认
   - ⚠️ 快照创建可能需要一些时间
   - ⚠️ 快照会占用存储空间
   - ⚠️ 建议在任务空闲时创建快照

3. **创建快照**
   ```bash
   volc_flink jobs savepoint -i <任务ID>
   ```

4. **等待快照完成** - 监控快照创建进度

5. **验证快照** - 查询快照列表确认创建成功
   ```bash
   volc_flink jobs savepoints -i <任务ID>
   ```

---

#### 2.2 查询快照列表（List Savepoints）

**使用场景**：
- 查看可用的快照
- 选择要恢复的快照
- 检查快照创建时间
- 清理过期快照

**操作步骤**：
1. **查询快照列表**
   ```bash
   volc_flink jobs savepoints -i <任务ID>
   ```

2. **展示快照信息**，包括：
   - 快照 ID
   - 快照路径
   - 创建时间
   - 快照状态
   - 快照大小（如果有）

---

#### 2.3 从快照恢复任务（Restore from Savepoint）

**使用场景**：
- 任务失败后恢复
- 任务升级后从旧状态恢复
- 任务迁移后恢复
- 回滚到之前的状态

**操作步骤**：
1. **查询快照列表** - 让用户选择要恢复的快照
   ```bash
   volc_flink jobs savepoints -i <任务ID>
   ```

2. **获取任务信息** - 确认任务当前状态
   ```bash
   volc_flink jobs detail -i <任务ID>
   ```

3. **风险确认** - 使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认
   - ⚠️ 恢复操作会覆盖当前任务状态
   - ⚠️ 任务需要重启
   - ⚠️ 可能导致数据处理短暂中断
   - ⚠️ 确认快照路径正确

4. **如果任务正在运行，先停止任务**
   ```bash
   volc_flink jobs stop -i <任务ID>
   ```

5. **从快照启动任务**
   ```bash
   volc_flink jobs start -i <任务ID> --from-savepoint <快照路径>
   ```

6. **监控任务状态** - 确认任务成功恢复

---

#### 2.4 删除快照（Delete Savepoint）

**使用场景**：
- 清理过期快照
- 释放存储空间
- 删除不需要的备份

**操作步骤**：
1. **查询快照列表** - 让用户选择要删除的快照
   ```bash
   volc_flink jobs savepoints -i <任务ID>
   ```

2. **风险确认** - 使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认
   - ⚠️ 删除后无法恢复
   - ⚠️ 确认快照 ID 正确
   - ⚠️ 建议保留最近的几个快照

3. **删除快照**
   - 注意：具体删除命令需要根据 volc_flink 工具的实际支持情况
   - 如果工具不支持直接删除，可以提示用户手动删除或通过其他方式

---

### 3. 状态监控（新增优化）
**操作完成后，持续监控任务状态一段时间！**

#### 3.1 实时状态更新
对于恢复操作，使用 `--inspect` 参数让命令自动监控任务状态：
```bash
volc_flink jobs start -i <任务ID> --from-savepoint <快照路径> --inspect
```

#### 3.2 状态展示
在监控过程中，实时展示状态变化：
```
⏳ 任务从快照恢复中...
- 当前状态：STARTING
- 已等待：15秒
- 剩余超时：285秒

✅ 任务已从快照成功恢复并进入 RUNNING 状态！
```

#### 3.3 异常处理
如果任务状态异常（如 FAILED），立即通知用户：
```
❌ 任务从快照恢复失败！
- 当前状态：FAILED
- 建议：检查任务日志，确认快照是否兼容
```

## 快照管理最佳实践

### 快照创建时机
- **任务升级前**：创建快照用于回滚
- **任务迁移前**：创建快照用于迁移后恢复
- **定期备份**：按业务需求定期创建快照
- **重大变更前**：任何重大配置变更前创建快照

### 快照保留策略
- **保留最近 3 个快照**：确保有足够的备份
- **定期清理旧快照**：避免占用过多存储空间
- **重要快照标记**：对重要快照进行特殊标记或单独备份

### 快照恢复注意事项
- **确认快照兼容性**：确保快照与当前 Flink 版本兼容
- **检查快照完整性**：恢复前确认快照文件完整
- **先测试再生产**：建议先在测试环境验证快照恢复

## 风险提示模板

注意：以下为风险要点汇总。执行任何会改变任务状态的操作（创建/恢复/删除快照等）前，必须使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认。

### 创建快照风险
- 快照创建可能需要较长时间
- 快照会占用存储空间
- 建议在任务空闲时创建快照
- 创建过程中任务性能可能受轻微影响

### 从快照恢复风险
- 任务会重启，导致短暂中断
- 可能导致数据延迟
- 恢复失败可能需要手动干预
- 确认快照路径正确且快照兼容

### 删除快照风险
- 删除后无法恢复
- 确认快照 ID 正确
- 建议保留最近的几个快照
- 删除前确认快照不再需要

## 注意事项

### 重要：快照操作的特殊要求

⚠️ **快照操作时必须遵守以下规则**：

1. **先确认任务范围** - 明确是哪个任务的快照操作
2. **先查询快照列表** - 恢复或删除前，先列出所有快照供用户选择
3. **风险确认** - 执行任何快照操作前，必须使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认
4. **从快照恢复时，先停止任务** - 如果任务正在运行，先停止再从快照启动
5. **验证操作结果** - 操作完成后，验证结果是否成功

### 通用注意事项

1. **先检测登录状态**：在执行任何操作前，先检测是否已登录
2. **先确认任务范围**：在执行任何操作前，明确确认是哪个任务
3. **先查询快照列表**：恢复或删除前，先列出所有快照供用户选择
4. **始终先确认风险**：在执行任何变更操作前，必须使用 `../../COMMON.md` 的 Risk Confirmation 模板完成用户确认
5. **先获取任务信息**：在执行操作前，先获取任务的详细信息
6. **验证操作结果**：操作执行完成后，必须验证结果
7. **提供清晰的反馈**：向用户提供清晰的操作结果和后续建议
8. **使用友好的语言**：用用户能理解的语言解释操作和风险
9. **避免过度技术化**：除非用户要求，否则避免过度技术化的解释
10. **友好的错误处理**：如果操作失败，向用户说明失败原因，并提供解决方案

## 错误处理优化

### 常见错误及处理

#### 错误 1：未登录
**错误信息**：`❌ 请先登录`

**处理方式**：
- 友好提示："检测到未登录火山引擎账号，请先登录"
- 提供登录指引：请使用交互式登录 `volc_flink login`（或企业内部安全方式），详见 `../../COMMON.md`
- 停止后续操作，等待用户登录后重试

#### 错误 2：任务不存在
**错误信息**：任务 ID 或任务名不存在

**处理方式**：
- 提示："未找到任务，请检查任务名或任务 ID 是否正确"
- 提供帮助：列出该项目下的所有任务供用户选择
- 或者提供搜索功能：`volc_flink jobs list --search <关键词>`

#### 错误 3：快照不存在
**错误信息**：快照路径或快照 ID 不存在

**处理方式**：
- 提示："未找到快照，请检查快照路径或快照 ID 是否正确"
- 提供帮助：列出该任务的所有快照供用户选择

#### 错误 4：快照创建失败
**错误信息**：快照创建失败

**处理方式**：
- 立即通知用户快照创建失败
- 建议查看任务日志：`volc_flink monitor logs -i <任务ID>`
- 提供排查建议：检查任务状态、存储空间等

#### 错误 5：快照恢复失败
**错误信息**：任务从快照恢复失败

**处理方式**：
- 立即通知用户恢复失败
- 建议查看任务日志：`volc_flink monitor logs -i <任务ID>`
- 提供排查建议：检查快照兼容性、任务配置等

#### 错误 6：网络超时
**错误信息**：命令执行超时

**处理方式**：
- 提示："命令执行超时，请检查网络连接"
- 建议稍后重试
- 如果多次失败，建议检查火山引擎服务状态
