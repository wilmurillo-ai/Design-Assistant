# Flink SQL 语法校验技能

用于验证 SQL 草稿的语法正确性，在发布前进行预检查，避免语法错误导致的发布失败。

---

## 通用约定（必读）

本技能的基础约定与只读约定统一收敛在：

- `../../COMMON.md`
- `../../COMMON_READONLY.md`

本技能只负责 SQL 校验与预检查，不负责发布、启动或修改任务状态。

---

## 🎯 核心功能

### 0. 登录状态检测（从 flink-volc 借鉴）
**在执行任何操作前，先检测登录状态！**

**检测步骤**：
1. 尝试执行一个简单的命令（如 `volc_flink drafts apps`）
2. 如果提示"请先登录"，则提示用户需要登录
3. 引导用户使用 `flink-volc` 技能进行登录

**错误处理**：
- 如果检测到未登录，立即停止后续操作
- 友好提示用户需要先登录
- 引导使用 flink-volc 技能

---

### 1. 信息提取与智能选择

#### 1.1 信息提取
从用户提问中提取关键信息：
- **项目名** (project_name)
- **草稿名** (draft_name)
- **草稿 ID** (draft_id)
- **SQL 内容**（可选，直接提供 SQL 代码）

#### 1.2 智能草稿选择
**如果用户没有明确提供草稿信息，按以下流程处理**：

**场景 A：用户只提供了草稿名，没有提供项目名**
1. 列出所有项目：`volc_flink projects list`
2. 在每个项目中搜索匹配的草稿名
3. 展示所有匹配的草稿供用户选择

**场景 B：用户没有提供任何信息**
1. 先列出所有项目供用户选择
2. 用户选择项目后，列出该项目下的所有草稿供用户选择
3. 用户选择草稿后，执行校验

**场景 C：用户提供的草稿名不明确**
1. 使用模糊搜索：列出所有草稿后让用户选择
2. 展示匹配结果供用户选择

---

### 2. SQL 语法校验

使用 `volc_flink drafts validate` 进行 SQL 语法校验。

#### 2.1 校验草稿
**命令格式**：
```bash
# 使用草稿 ID 校验
volc_flink drafts validate -i <草稿ID>

# 使用草稿路径校验
volc_flink drafts validate -i <草稿路径>

# 输出原始 JSON
volc_flink drafts validate -i <草稿ID> --raw
```

#### 2.2 校验结果分析

**校验通过的情况**：
- 没有语法错误
- SQL 语法正确
- 可以安全发布

**校验失败的情况**：
- 语法错误
- 类型不匹配
- 缺少必要参数
- 连接器配置错误
- 等等...

#### 2.3 错误定位和提示
当校验失败时，提供：
- 错误类型
- 错误位置（行号、列号）
- 错误描述
- 修复建议
- 相关文档链接（如适用）

---

### 3. 直接校验 SQL 内容（高级功能）

如果用户直接提供了 SQL 代码，可以：
1. 先创建一个临时草稿
2. 对临时草稿进行校验
3. 获取校验结果
4. 删除临时草稿
5. 向用户反馈校验结果

**注意**：这个功能需要谨慎使用，避免创建过多临时草稿。

---

## 工具调用顺序

### 校验已有草稿
1. **检测登录状态** - 确认已登录
2. **智能选择草稿** - 如果用户没有提供，列出项目和草稿供选择
3. **执行校验** - `volc_flink drafts validate -i <草稿ID>`
4. **分析结果** - 解析校验结果
5. **反馈给用户** - 展示校验结果和修复建议

### 预发布校验流程（推荐）
在发布草稿前，总是先进行校验：
1. **检测登录状态** - 确认已登录
2. **获取草稿内容** - `volc_flink drafts content -i <草稿ID>`
3. **执行校验** - `volc_flink drafts validate -i <草稿ID>`
4. **如果校验通过** - 继续发布流程
5. **如果校验失败** - 展示错误，让用户修复后重试

## 错误处理

### 常见错误及处理

#### 错误 1：未登录
**错误信息**：`❌ 请先登录`

**处理方式**：
- 友好提示："检测到未登录火山引擎账号，请先登录"
- 引导使用 flink-volc 技能："请使用 flink-volc 技能进行登录"
- 停止后续操作

#### 错误 2：草稿不存在
**错误信息**：`Draft not found`

**处理方式**：
- 提示："未找到草稿，请检查草稿名或草稿 ID 是否正确"
- 列出可用草稿：`volc_flink drafts apps`
- 让用户重新选择

#### 错误 3：校验命令执行失败
**错误信息**：校验命令执行超时或失败

**处理方式**：
- 提示："校验执行失败，请稍后重试"
- 建议检查草稿内容是否正确
- 建议手动检查 SQL 语法

## 常见 SQL 错误类型及修复建议

### 语法错误
**错误示例**：
- 缺少分号
- 关键字拼写错误
- 括号不匹配

**修复建议**：
- 检查 SQL 语法
- 使用 SQL 格式化工具
- 参考 Flink SQL 文档

### 连接器配置错误
**错误示例**：
- 缺少必要的连接器参数
- 参数值不正确
- 连接器类型不支持

**修复建议**：
- 检查连接器文档
- 确认所有必需参数都已提供
- 验证参数值的正确性

### 类型不匹配
**错误示例**：
- 字段类型不匹配
- 函数参数类型不正确
- 隐式类型转换失败

**修复建议**：
- 显式转换类型
- 修改字段定义
- 使用正确类型的函数

## 注意事项

### 重要：预发布校验

⚠️ **在发布草稿前，总是先进行校验！**

- 可以避免语法错误导致的发布失败
- 节省时间和资源
- 提高开发效率

### 重要：校验的局限性

⚠️ **语法校验通过 ≠ 运行时一定成功**

- 语法校验只检查语法正确性
- 不检查运行时错误（如连接问题、权限问题等）
- 不检查数据格式问题
- 发布后仍需监控任务运行状态

### 通用注意事项

1. **先检测登录状态**：在执行任何操作前，先检测是否已登录
2. **预发布校验**：在发布前总是先进行校验
3. **提供详细的错误信息**：校验失败时，提供详细的错误位置和修复建议
4. **说明校验的局限性**：告诉用户语法校验通过不代表运行时一定成功
5. **友好的错误处理**：如果操作失败，向用户说明失败原因，并提供解决方案
