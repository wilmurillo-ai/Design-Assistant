# 简历评分维度基准说明

> 本文档定义了 AI-Resume-Judge 评分系统中各维度的评分标准与参考基准。

---

## 评分体系总览（100分）

| 维度 | 满分 | 占总分比 |
|------|------|----------|
| 基本信息完整性 | 15 | 15% |
| 成就量化程度 | 25 | 25% |
| 技术深度与广度 | 20 | 20% |
| 工作经历质量 | 20 | 20% |
| 教育背景 | 8 | 8% |
| 个人品牌/影响力 | 7 | 7% |
| 表达能力 | 5 | 5% |

---

## 维度1：基本信息完整性（满分15）

### 评分要素

| 要素 | 满分 | 检测标准 |
|------|------|----------|
| 邮箱 | 2分 | 含 `@` 的标准邮箱格式 |
| 手机号 | 2分 | 1开头11位手机号或 `XXX-XXXX-XXXX` 格式 |
| GitHub | 3分 | 包含 `github.com/username` |
| LinkedIn | 3分 | 包含 `linkedin.com/in/username` |
| 所在地 | 1分 | 一二三线城市名 |
| 简历结构 | 4分 | 含个人简介(1.5) + 工作经验(1) + 教育背景(1) + 技能(0.5) |

### 100分简历表现
```
📧 xxx@example.com　｜　📱 138-XXXX-XXXX
🌐 github.com/yourhandle　｜　🔗 linkedin.com/in/yourhandle
📍 北京 / 远程
```
- 同时具备邮箱、手机、GitHub、LinkedIn、所在地
- 简历结构完整（个人简介、工作经历、教育背景、技能、项目经验）

### 60分简历表现
```
📧 example@email.com | 📱 138-0000-0000
```
- 仅提供基础联系方式
- 无 GitHub / LinkedIn
- 结构不完整（无个人简介、无项目经历章节）

## 维度2：成就量化程度（满分25）

### 评分要素

| 要素 | 满分 | 说明 |
|------|------|------|
| 量化指标密度 | 8分 | `%`、`倍`、`万`、`亿`、`QPS`、`P99` 等指标数量 × 1.5 |
| 强动词主导 | 6分 | 「主导」「负责」「从零」「构建」等主动动词数量 × 1.0 |
| 百分比行数 | 6分 | 含数字/百分比的描述行数 × 1.2 |
| 业务影响词 | 5分 | 「日均」「节省」「提升」「覆盖」等业务价值词 × 0.5 |

### 扣分项
- 出现「参与」「配合」「协助」「做一些」等模糊动词，每次 -1.5分

### 100分简历表现
```
主导将单体推理服务迁移至 vLLM + Kubernetes 平台，
QPS 提升 4.2 倍，P99 延迟从 3.2s 降至 0.8s，
每月推理成本节省 ¥180 万
```

### 60分简历表现
```
参与了AI项目开发，负责模型训练，配合团队完成NLP任务
```
- 无任何数字
- 全程模糊动词

## 维度3：技术深度与广度（满分20）

### 评分要素

| 要素 | 满分 | 说明 |
|------|------|------|
| AI技术栈覆盖 | 10分 | LLM/NLP/ML/DL 等关键词命中数量 × 0.8 |
| 前沿技术深度 | 5分 | vLLM/Agent/RAG/RLHF 等前沿词命中数量 × 1.0 |
| 技能层次 | 3分 | 有「精通/熟练/了解」分层描述 |
| 工程工具 | 2分 | Docker/K8s/AWS 等工程能力 |

### 100分简历表现
```
### 大语言模型与 Generative AI
- 模型训练与微调：SFT、RLHF、DPO、LoRA / QLoRA（精通）
- 推理优化：vLLM、TensorRT-LLM（熟练）
- RAG 系统：Milvus、Qdrant（了解）
```

### 60分简历表现
```
## 技能
- Python
- TensorFlow / PyTorch
- 机器学习
- 深度学习
```
- 平铺列表，无层次区分
- 无前沿技术关键词

## 维度4：工作经历质量（满分20）

### 评分要素

| 要素 | 满分 | 说明 |
|------|------|------|
| 公司背书 | 5分 | 知名公司关键词命中数 × 1.5 |
| 工作段数 | 4分 | 每段经历有年份 + 职位 × 1.5 |
| 时间跨度 | 4分 | 5年以上=4分，3年=2分，1年=1分 |
| 主导动词 | 4分 | 「主导」「负责」「独立」等 × 0.8 |
| 职位级别 | 3分 | Senior/Lead/专家/总监 等关键词 |

### 100分简历表现
```
### 某头部 AI 科技公司｜资深 AI 工程师 Lead｜2022.03 — 至今
（3段递进：创业公司 → 互联网大厂 → 头部AI公司）
```

### 60分简历表现
```
### AI工程师 · 某互联网公司
*2021 – 至今*
- 参与了AI项目开发
```
- 单一公司描述，无递进感
- 无公司名称辨识度

| 要素 | 满分 | 说明 |
|------|------|------|
| GitHub Star | 3分 | 1k+=3分，100+=2分，10+=1分 |
| 开源贡献 | 1分 | GitHub/开源 关键词 |
| 演讲分享 | 1.5分 | 演讲/大会/顶会 关键词 |

---

## 维度7：表达能力（满分5）

- 结构化标题 `#` 数量 × 0.3（最多2分）
- 自我评价有套话：-2分（出现2条以上），-1分（出现1条）
- 简历内容不足10行：-1分
- 有表格：+1分

## 跨行业等价换算表

| 行业 | 系数 | 说明 |
|------|------|------|
| AI / 大模型 | 1.00 | 基准行业 |
| 金融 / 量化 | 0.92 | 高门槛，但无开源生态 |
| 自动驾驶 | 0.88 | 硬件+算法双重门槛 |
| 游戏 / 图形学 | 0.82 | 技术栈深度要求高 |
| 前端 / 移动 | 0.70 | 入门相对友好 |
| 后端 / DevOps | 0.72 | 技能可迁移 |
| 数据分析 | 0.65 | 门槛较低 |
| 产品经理 | 0.60 | 非技术岗 |
| 财务 / 会计 | 0.55 | 高度规范化 |
| 行政 / HR | 0.45 | 非技术岗 |

### 换算公式
```
行业等价分 = max(0, min(100, round(原始能力分 × 行业系数)))
```

**解读**：如果一个建筑行业从业者简历原始得分70分，在建筑行业中等价于 70 × 0.65 = 45.5 分，相当于 AI 行业的 45.5 分水平。
