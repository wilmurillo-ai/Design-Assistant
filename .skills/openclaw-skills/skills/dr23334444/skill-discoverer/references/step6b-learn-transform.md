# Step 6b：学习并转化流程

用户选择选项 B（学习并转化）时执行此流程。

## 第一步：逐条整理（复用 Step 4 分类结果）

> 📎 **直接使用 Step 4 已完成的四列完整分析结果（原规则/标记/建议转化方式/建议写入位置），不重新读取 SKILL.md。**
> Step 4 未执行时（如直接跳转到此流程），才需要读取 SKILL.md 并补全四列数据。

对每条规则/约定整理如下：

| # | 原规则内容 | 转化方式 | 写入位置 |
|---|---|---|---|
| 1 | {规则描述} | {如何在当前 agent 中应用} | {MEMORY.md 铁律 / SOUL.md / HEARTBEAT.md / 其他} |
| 2 | ... | ... | ... |

### 写入位置判断原则

| 内容类型 | 写入位置 |
|---|---|
| 操作流程/执行规范 | MEMORY.md 铁律节 |
| 行为风格/人格原则 | SOUL.md（需特别标注 🚨，属不可逆改动） |
| 定期任务/检查项 | HEARTBEAT.md |
| 工具配置/环境信息 | TOOLS.md |

## 第二步：展示清单，等待二次确认

将分析表格发给用户，明确说明每条规则将被写入哪个文件的哪个位置：

```
📚 学习并转化清单（{skill名}）

以下规则将被学习内化，请逐一确认：

1. 【铁律】{规则描述}
   → 写入 MEMORY.md「铁律」节，追加为第 N 条

2. 【行为原则】{规则描述}  ⚠️ 将影响回复风格
   → 写入 SOUL.md「{节名}」节

3. 【定期任务】{规则描述}
   → 写入 HEARTBEAT.md「每次 heartbeat 必做」节

是否确认全部应用？[全部确认 / 逐条选择 / 取消]
```

- **全部确认** → 直接进入第三步，逐条写入
- **逐条选择** → 列出序号供用户选择：
  ```
  请回复要内化的条目序号，用逗号分隔（如：1,3,5），其余跳过。
  未选择的条目将记录为「用户跳过」，写入巡查日志。
  ```
  收到用户回复后，仅对选中的条目执行写入。
- **取消** → 放弃本次内化，记录用户取消决定，返回报告

## 第三步：执行写入

用户确认后，逐文件逐条写入，每写入一条输出确认：
```
✅ 已写入：MEMORY.md 铁律第 N 条
✅ 已写入：SOUL.md 「{节名}」
...
```

## 第四步：完成汇总

```
📚 学习并转化完成（{skill名}）

共学习 {N} 条规则：
• MEMORY.md 铁律：新增 X 条
• SOUL.md：修改 Y 处  ⚠️
• HEARTBEAT.md：新增 Z 条

未转化（跳过）：{被用户跳过的条目}
```

> ⚠️ 此方式不安装 skill 文件，规则内化到 agent 记忆体系后，skill 可从广场卸载而不影响已学内容。
> 推荐对"教训/习惯类"规则使用此方式，"操作流程类"建议直接安装 skill（选项 A）。
