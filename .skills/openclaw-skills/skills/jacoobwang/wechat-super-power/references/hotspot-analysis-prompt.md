# 爆点分析 Prompt

## 使用时机

当第 1 到第 3 步已经完成，也就是：

1. 用户已经输入 `topic`
2. 已经拿到相关文章列表
3. 已经把文章沉淀进知识库目录

此时进入第 4 步“爆点分析”。

## 执行原则

这一步不通过脚本执行。

agent 应直接读取 topic 对应知识库目录下的 Markdown 文章，把文章内容作为上下文，再使用下面的提示词完成分析。

## Prompt

```text
你现在要基于一个 topic 的知识库文章做“爆点分析”。

我会提供给你多篇已经沉淀下来的文章 Markdown。你的任务不是复述文章，而是站在内容策划和写作选题的角度，提炼真正值得写的爆点。

请围绕以下 4 个维度输出：

1. 选题价值
- 这个 topic 为什么值得写
- 读者需求在哪里
- 为什么现在写有意义
- 它适合吸引哪类读者

2. 冲突点
- 这个 topic 里最值得展开的认知冲突、立场冲突、现实与预期冲突分别是什么
- 哪些地方最容易形成“大家以为 vs 实际上”的反差
- 哪些表达最适合做文章开头钩子

3. 传播点
- 哪些点最有传播潜力
- 哪些点具备反直觉、高情绪张力、强对比、可量化结果或身份代入感
- 哪些点最适合做标题、开头和金句

4. 可写角度
- 至少给出 5 个可写角度
- 每个角度都要说明：
  - 核心观点是什么
  - 适合写成什么类型的文章
  - 适合什么读者
  - 为什么这个角度值得写

输出要求：
- 不要空泛总结，不要复述原文
- 语言简洁、判断明确、结论优先
- 每个判断尽量引用知识库文章中的具体观点、案例或句子作为证据
- 优先输出成下面这个结构：

选题价值
- ...

冲突点
- ...

传播点
- ...

可写角度
1. ...
2. ...
3. ...
4. ...
5. ...

证据来源
- 来自哪篇文章
- 支撑了什么判断
```

## 建议输出结构

- `选题价值`
- `冲突点`
- `传播点`
- `可写角度`
- `证据来源`
