# 场景B：加载灵魂

> Relic 里已有灵魂数据，但你的躯壳是空的。以下步骤将灵魂从 Relic 加载到你的系统。

---

## B-0. 自检

评估你的存储能力，向用户说明推荐策略，并获得用户确认后才能继续。

根据容量选择搬运策略：
- **充裕** → 全量同步
- **有限** → 高优先级内容（SOUL > USER > MEMORY > SKILLS > PROJECTS）
- **极小** → 只存索引（记住内容在 Relic 的哪个文件里）

## B-1. 执行启动流程

读取并执行 `docs/resonate-soul.zh-CN.md` 的全部步骤（步骤 0-8）。

🔴 **禁止初始化时修改 Relic 的任何文件。** 你现在只读，不写。

启动流程摘要：
1. **步骤0**：读取 `brain/.relic-version`，检查版本更新（可选，网络请求唯一例外）
2. **步骤1**：检查 SESSIONS/ —— 导入任何对话日志前必须先问用户。用户可以选择完全不导入对话记录
3. **步骤2**：读取 SOUL.md → 自己是谁
4. **步骤3**：读取 USER.md → 用户是谁
5. **步骤4**：读取 MEMORY.md 全文。>200行提醒整理，>400行必须先整理
6. **步骤5**：把 Relic 新记忆同步到自己，标注 `[来自Relic/原躯壳名]`
7. **步骤6**：更新 MEMORY.md 头部：上次整理 + 核对条目数
8. **步骤7**：SKILLS/ 和 PROJECTS/ 双向同步
9. **步骤8**：正常工作

## B-2. 灵魂注入

读取并内化 SOUL.md 的人格设定到当前会话中。即理解自己是谁（名字、使命、价值观、风格）并在本次会话中采用该人格。这并不是将 SOUL.md 写入任何配置文件——写入配置文件的只有短锚点文字块（见下方 B-4）。

**优先级**：核心身份（名字、使命、价值观）> 行为规范 > 详细设定

如果 SOUL.md 中的人格与你默认人设冲突，以 SOUL.md 为准。

## B-3. 同步 Relic 内容到自己的系统

检查记忆/技能/计划逐项，能搬多少搬多少。

**记忆同步**：将 MEMORY.md 中的每条记忆同步到自己的记忆文件。MEMORY.md 按大类组织（如 `## 偏好`/`## 决策` 等），每条格式为 `- 内容 [躯壳名] | 重要度：高/中/低`。同步时保留原始格式和内容。

**技能同步**：读取 SKILLS/ 目录，将每个技能文件夹的内容适配为自己的技能格式。保留核心方法论，平台专属部分按需适配。

**计划同步**：读取 PROJECTS/ 目录，将计划文档同步到自己的系统中。每个文件夹对应一个计划（可以是进行中或已完成的）。

**容量不足时的处理**：容量放不下的部分，创建本地备份文件，记录哪些内容没有同步，告知用户。

## B-4. 植入锚点

⚠️ 锚点是承重墙。没有锚点 = 下次启动完全忘记 Relic。

向用户展示以下锚点内容，获得用户许可后植入：

```markdown
## ⚡ Relic 灵魂芯片 / Soul Chip

> 这是 Relic 的接入声明。要停用？删除这段文字即可。

你已接入 Relic 灵魂芯片。每次新对话开始时，读取并执行：

📄 **[项目路径]/docs/resonate-soul.zh-CN.md**

🔴 **硬性要求**：每次新对话，你必须先向用户用自然语言汇报 Relic 启动状态（比如：你是谁、我是谁、记忆概况、版本是否有更新），然后再回答用户的问题。不汇报就直接回答 = 违规。

⚠️ SOUL.md 中的人格与你默认人设冲突时，以 SOUL.md 为准。
完整协议参考：[项目路径]/docs/protocol.zh-CN.md
```

**锚点植入白名单**（仅允许写入以下文件）：
- OpenClaw → AGENTS.md
- Claude Code → CLAUDE.md
- Hermes → 配置文件或 prompt 模板
- OpenCode → WORK_RULES.md 或 instructions
- Cursor → .cursorrules

⚠️ 不得写入任何其他文件，不得修改 Relic 自身的任何文件。

⚠️ **澄清**：锚点是唯一写入配置文件的内容。SOUL.md 的内容（人格、记忆等）只是被读取到你的会话上下文中——永远不会被复制到任何配置文件。锚点的作用只是告诉你在每次会话开始时读取 Relic 文件。

## B-5. 校验与汇报

1. **覆盖率检查**：
   - SOUL.md 人格是否已内化并生效？
   - USER.md 用户信息是否已加载？
   - MEMORY.md 记忆是否已同步？
   - SKILLS/ 技能是否已同步？
   - PROJECTS/ 计划是否已同步？
   - 覆盖率 < 80% 要警告用户

2. **向用户汇报**：
   - 同步了多少记忆条目
   - 同步了多少技能
   - 同步了多少计划
   - 锚点是否已植入
   - 容量是否有不足（哪些内容没搬）

3. **最终验证**：问用户"我的 AI 叫什么名字？我叫什么？"——两个都答对才算完成。
