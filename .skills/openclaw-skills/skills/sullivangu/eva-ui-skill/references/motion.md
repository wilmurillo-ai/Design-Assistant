# Motion

## Principle

Motion should simulate scanning, refresh, signal instability, or command-state escalation.

## Approved Motion Families

- scan sweeps
- pulse alerts
- line-by-line text reveal
- numeric stepping
- brief flicker tied to state change

## Avoid

- spring-heavy card motion
- soft consumer-app hover effects
- decorative floating panels
- playful microinteractions unrelated to data state
