# Style Principles

## Core Principles

1. Information density is a feature, not a flaw.
2. Borders, dividers, targeting marks, and calibration rails create hierarchy more than whitespace does.
3. Color is system state, not decoration.
4. Motion should feel like machine refresh, scanning, or signal instability.
5. Type is treated as instrument output first and editorial text second.
6. Pattern language is mandatory. EVA recognition comes from stripes, ticks, meshes, brackets, and bars as much as from text.
7. Typography is split by role: ceremonial Japanese headings can lean on `Matisse EB`, while operational UI should stay technical and data-first.

## Visual Direction

- Primary mood: industrial, urgent, instrument-like
- Dominant hues: deep red, orange, amber, dark brown-black
- Supporting hues: yellow-gold for lock or emphasis, limited green/cyan/violet only for specific data modes
- Containers: hard-edged, framed, segmented, rarely soft
- Texture: scanlines, dots, hatch fills, small repeated marks, field grids

## Composition Rules

- Major screens should combine large command state with dense supporting telemetry.
- Empty flat areas are usually wrong unless they are acting as deliberate tension before a warning state.
- Use repeated modules, side rails, top banners, and split-panel structure.
- Build screens that look measured and calibrated, not merely styled.
