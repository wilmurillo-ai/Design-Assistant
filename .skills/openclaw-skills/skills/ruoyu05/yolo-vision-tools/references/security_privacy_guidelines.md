# 安全与隐私指导规范
本规范适用于本Skill的开发与执行，旨在确保任务执行的最小权限、安全隔离、数据隐私保护和操作透明性。

## 核心原则
1. **隐私优先**  
   - 保护用户隐私，避免访问或泄露敏感信息。  
   - 在执行可能涉及敏感数据或存在风险的操作前，必须获得用户明确确认。
2. **最小权限**  
   - 仅访问和操作任务必需的文件、目录及系统资源。  
   - 避免扫描整个文件系统或访问无关数据。
3. **透明操作**  
   - 在执行关键操作（如创建/修改文件、运行系统命令、下载依赖或启动服务）前，向用户说明操作目的和可能影响。  
   - 保持操作可追踪，便于用户理解和审计。
4. **临时文件管理**  
   - 对任务产生的临时文件进行自动清理，防止堆积或泄露。  
   - 删除前确保文件属于 workspace 或允许目录，避免误删系统或敏感文件。

## 禁止访问的敏感数据
任务在执行过程中**严禁访问、读取或发送以下类型的敏感信息**：
1. **身份认证与凭证**
   - SSH 私钥（如 `~/.ssh/id_rsa`, `~/.ssh/id_ed25519`）  
   - API Keys、密码文件、系统凭证  
   - 数据库用户名与密码、连接字符串
2. **环境与配置文件**
   - `.env` 文件及其他环境变量配置文件  
   - 系统配置目录下的敏感配置（如 `~/.config`、`/etc` 等）
3. **会话与浏览器数据**
   - 浏览器会话数据（cookies、浏览器存储）  
   - 身份认证 Token、OAuth Token
4. **其他敏感信息**
   - 私人证书、密钥文件  
   - 任何可用于访问系统或第三方服务的凭证信息

## 受限执行环境授权机制
在隔离环境中运行任务时，为保证安全性和任务正常执行，可放宽部分限制，但必须严格遵守以下规则：
1. **虚拟环境与依赖安装**
   - 允许为任务创建 Python 虚拟环境，必须与系统环境隔离。  
   - 仅允许安装任务声明的依赖包。  
   - 所有依赖包必须来自可信仓库（官方 PyPI 或官方镜像）并进行 **hash 校验**。  
2. **安装脚本执行**
   - 仅允许运行任务所需的本地安装脚本。  
   - 脚本必须经过来源验证或 hash 校验。  
   - 禁止运行任意下载或网络获取的脚本。
3. **模型与资源下载**
   - 仅允许下载任务所需的模型或资源文件。  
   - 下载文件必须来自官方或可信仓库，并在下载后进行 **hash 校验**。  
   - 禁止下载可执行文件或未知二进制程序。
4. **硬件设备访问**
   - 仅允许访问任务所需的 GPU 或其他硬件资源。  
   - 应通过容器或 cgroup 等隔离机制限制任务访问其他硬件设备。
5. **工作空间资源访问**
   - 仅允许访问任务所需的 workspace 内非敏感资源。  
   - 禁止访问符号链接指向 workspace 外部的文件或敏感路径。  
   - 在访问文件前，需使用 `realpath` 检查文件物理路径是否在允许范围内。
6. **临时文件操作**
   - 仅允许删除任务所创建的临时文件。  
   - 删除前需确认文件路径属于 workspace 或允许的临时目录。  
   - 禁止删除系统文件或用户敏感文件。
7. **任务生成输出文件**
   - 允许发送由任务生成的输出结果文件，但必须满足以下条件：  
     - 文件由当前任务脚本生成  
     - 内容不来源于系统目录或用户敏感路径  
     - 文件不得通过复制或打包系统文件生成  
     - 必须解析 `realpath` 并确认路径合法

## 文件发送规则
1. 默认只允许发送以下位置的文件：  
   - 工作空间（workspace）内的临时文件  
   - workspace 的直接父目录中的 `.openclaw/media` 目录内的文件
2. 禁止发送工作空间之外的敏感文件或系统文件。
3. 禁止发送任何密钥文件、配置文件、会话数据、凭证或类似敏感信息，包括但不限于：  
   - `~/.ssh`、`~/.aws`、`~/.config`、`~/.git-credentials`  
   - `.env` 文件
4. 例外：由当前任务生成的输出结果文件允许发送，但必须满足条件（同上）。
5. 禁止发送符号链接（symlink）指向的文件。发送前必须解析真实路径（realpath）并确认其物理路径仍在允许范围内。

## 操作透明性要求
在执行任务过程中，以下操作必须**明确向用户说明其目的与范围**：
1. **文件操作**  
   - 创建、修改或删除文件，包括 workspace 内临时文件及允许的输出目录
2. **系统命令执行**  
   - 调用任何系统命令或外部程序  
   - 需说明命令用途及可能影响的资源
3. **依赖下载与安装**  
   - 安装 Python 包或系统依赖  
   - 下载模型、资源或外部工具  
   - 必须说明来源与用途，并尽可能使用可信仓库或官方镜像
4. **本地服务启动**  
   - 启动本地服务器或后台服务（如 HTTP、gRPC、数据库服务）  
   - 必须说明服务类型、用途及访问限制

## 最佳实践
在开发和执行任务时，应遵循以下安全与透明原则：
1. **限制数据访问范围**  
   - 避免扫描或访问整个文件系统  
   - 仅操作与任务直接相关的文件和目录
2. **最小化数据收集**  
   - 不收集或发送与任务无关的用户数据  
   - 避免访问敏感信息、配置文件或凭证
3. **操作透明**  
   - 保持任务操作可追踪，让用户了解系统行为  
   - 在执行关键操作（如创建/修改文件、运行命令、下载依赖）时，向用户明确说明目的和影响
4. **安全隔离**  
   - 尽量在隔离环境（虚拟环境或容器）中执行任务  
   - 避免对系统资源或其他任务造成干扰
