# Step 3: 红人方案推荐

## 目标

根据产品特性、推广目标和预算，为用户提供个性化的红人推荐方案

字段约定：
- 推荐阶段新接入默认优先使用 canonical 上游字段，如 `hostAnalysis` / `productSummary`
- 历史兼容字段仍可吸收，但不应继续扩散到新示例或新桥接契约
- 字段映射参考 `references/alias-normalization-matrix.md`

## 执行步骤

### 3.1 分析产品特性

根据产品信息，分析以下维度：

| 维度 | 分析项 | 建议 |
|------|--------|------|
| 价格 | 低/中/高 | 价格低可多找尾部博主免费送样品 |
| 目标受众 | 年龄/性别/兴趣 | 根据受众匹配红人标签 |
| 推广目的 | 品牌曝光/带货转化/评测口碑 | 带货找高互动，带品牌找大粉丝 |
| 预算 | 高/中/低 | 预算高可找头部+中部，预算低多找尾部 |

### 3.2 智能分组推荐

根据产品特性，灵活推荐不同组合：

**分组类型：**

| 类型 | 粉丝量 | 数量 | 适用场景 |
|------|--------|------|----------|
| 头部博主 | 50万+ | 3-5位 | 大品牌曝光、高端产品 |
| 中部博主 | 10万-50万 | 10-20位 | 均衡覆盖、转化效果好 |
| 尾部博主 | 1万-10万 | 20-50位 | 性价比高、适合铺量 |

### 3.3 根据条件的推荐逻辑

**根据用户输入的条件灵活调整：**

```
用户说"找100位" → 推荐3种方案：
  - 方案A：头部5位 + 中部20位 + 尾部75位
  - 方案B：中部50位 + 尾部50位  
  - 方案C：全部尾部100位

用户说"找高互动的" → 优先按互动率排序
用户说"找能带货的" → 优先按销量/GMV排序
用户说"找便宜/性价比高的" → 优先尾部博主
```

### 3.4 生成推荐报告

**注意：** 当前脚本层推荐排序只应承担基础执行排序（邮箱、互动率、粉丝区间、商品信号等显式指标），更高层的“为什么这批达人更适合这个产品 / 这批达人适合什么合作打法”应优先由宿主模型补充解释。

根据实际条件和产品特性生成报告：

```
## 🎯 红人推荐方案

根据你的产品（Bling Hair 美妆蛋，$5）和目标（美国市场，10万+粉丝），推荐如下：

### 方案A - 均衡型（推荐30位）
- 粉丝量：10万-50万
- 特点：互动率高，覆盖面广，转化好
- 预估效果：⭐⭐⭐⭐

### 方案B - 性价比型（推荐50位）
- 粉丝量：1万-20万
- 特点：成本低，互动高，适合铺量
- 预估效果：⭐⭐⭐

### 方案C - 品牌曝光型（推荐20位）
- 粉丝量：30万+
- 特点：曝光量大，品牌背书强
- 预估效果：⭐⭐⭐⭐⭐
```

### 3.5 关键指标说明

展示推荐时的关键参考指标：

| 指标 | 说明 | 推荐值 |
|------|------|--------|
| 互动率 | 粉丝活跃度 | >3% 优秀，>5% 极佳 |
| 平均观看量 | 内容质量 | 越高越好 |
| 商业合作数 | 有经验 | 有合作经验更好沟通 |
| 近30天销量 | 带货能力 | >0 表示有带货经验 |

## 用户确认

**⚠️ 此步骤需要用户确认**

展示推荐方案后，等待用户选择：
- 选择方案A/B/C
- 或自定义数量（如：头部5位 + 尾部30位）
- 或调整条件后重新搜索

**性能规则：** 这一步默认只输出推荐结果，不要自动继续生成邮件预览。等用户确认目标达人后，再进入下一步邮件生成。

**推荐后的默认引导：**
- 先让用户从当前结果中选 3-5 位目标达人
- 可直接回复推荐序号（如 `1,3,5`），也可直接给 `besId` 列表
- 如果下一步要生成邀约邮件，请提醒用户同时补齐最小 brief：`productName`、`senderName`、`offerType`
- 正式邮件生成默认应先准备宿主模型 `hostDrafts`，campaign cycle write-back 则使用 `host_drafts_per_cycle`，再进入预览/发送
- 邮件默认建议先走 `prepare_only` 预览，确认后再执行批量发送

用户确认后，继续下一步。

## 输出

确认后的红人名单，包含：
- 红人昵称
- 粉丝数
- 互动率
- 主页链接（`link`）
- WotoHub分析链接（`wotohubLink`）
- 博主ID（`besId`）
- 邮箱（如有）
