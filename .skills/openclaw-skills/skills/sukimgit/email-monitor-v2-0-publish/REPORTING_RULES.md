## 🎯 核心认知

**客户的需求：**
- ✅ 收到任务完成的结果
- ✅ 知道任务完成度（100% or 50%）
- ✅ 知道交付物是什么
- ❌ 不在乎是谁汇报（小技 or Monet）

**Monet 的责任：**
- ✅ 确保客户收到结果
- ✅ 验证结果质量
- ✅ 转达完整信息

## 📋 汇报流程

### 标准流程（推荐）⭐

```
小技完成任务
  ↓
小技状态变为 done（自动）
  ↓
Monet 定期检查 subagents list（每 5-10 分钟）
  ↓
Monet 发现小技完成
  ↓
Monet 检查小技的工作成果（文件/代码/文档）
  ↓
Monet 验证完成度（100% or 50%）
  ↓
Monet 通过飞书 Webhook 发送给客户
  ↓
客户收到结果 ✅
```

## 🛠️ 技术实现

### 方案 1：Monet 主动检查（当前方式）⭐

**优点：**
- ✅ 可靠（Monet 控制）
- ✅ 可以验证质量
- ✅ 可以过滤不完整的结果

**缺点：**
- ⚠️ 需要 Monet 记得检查

**实现：**
```python
# Monet 每 5-10 分钟检查一次
import subprocess
import json

def check_subagents():
    result = subprocess.run(['openclaw', 'subagents', 'list', '--recent', '10'], 
                          capture_output=True, text=True)
    data = json.loads(result.stdout)
    
    for agent in data['recent']:
        if agent['status'] == 'done' and not agent.get('reported', False):
            # 发现完成的小技，检查成果
            verify_and_report(agent)
            agent['reported'] = True
```

### 方案 3：混合模式（最可靠）⭐⭐⭐

**流程：**
```
小技完成任务
  ↓
小技尝试发送飞书消息（如果会的话）
  ↓
Monet 同时检查 subagents list
  ↓

```

**优点：**
- ✅ 双重保障（小技 + Monet）
- ✅ 即使小技不会发，Monet 也会发
- ✅ Monet 可以验证质量

## 📝 Monet 的工作规则

### 检查频率

**小技进行中：** 每 5-10 分钟检查一次
```bash
openclaw subagents list --recent 10
```

**检查内容：**
- [ ] 是否有 status=done 的小技
- [ ] 是否有未汇报的小技
- [ ] 小技的 runtime（判断是否卡住）

### 验证成果

**发现小技完成后：**
1. 检查文件是否创建
2. 检查完成度（100% or 50%）
3. 检查质量（代码/文档是否合格）

### 汇报给客户

**汇报格式：**
```
✅ [任务名称] 完成！

## 完成情况
[列出所有交付物及状态]

## 完成度
X/Y 完成（Z%）

## 文件位置
[路径]

## 验证结果
[Monet 的验证意见]

## 下一步建议
[客户需要做什么]
```

**汇报方式：**
- ✅ 飞书 Webhook（直接发送到群里）
- ✅ 或者当前会话（如果客户在线）

---

## 🚨 异常情况处理

### 情况 1：小技卡住了（>30 分钟未完成）

**处理：**
1. 检查小技状态
2. 如果还在 active，继续等待
3. 如果超时，重新启动或放弃

### 情况 2：小技完成了但成果不完整（50%）

**处理：**
1. 如实汇报给客户
2. 说明完成度（50%）
3. 说明缺失什么
4. 询问客户是否继续

**汇报示例：**
```
⚠️ [任务名称] 部分完成！

## 完成情况
✅ README.md - 已创建
❌ test_report.md - 未创建

## 完成度
1/2 完成（50%）

## 原因分析
小技只创建了 1 个文件就返回了

## 下一步建议
[ ] 重新安排小技创建剩余文件
[ ] Monet 手动创建
[ ] 暂时够用，以后再说

客户，你选哪个？
```

### 情况 3：小技完成了但没汇报

**处理：**
1. Monet 主动检查发现
2. Monet 验证成果
3. Monet 汇报给客户
4. 记录问题（小技为什么没汇报）
