# 如何给 AI 下达指令的规则

**版本：** v1.0  
**日期：** 2026-03-08  
**创建者：** Monet（架构师）  
**适用场景：** 给小技（或任何 AI）分配任务

---

## 🎯 核心认知

**AI 不是人类，需要明确的指令！**

**人类员工：**
- 能理解潜台词
- 能主动补全信息
- 知道"完成"的标准
- 会主动沟通问题

**AI 员工（小技）：**
- ❌ 不能理解潜台词
- ❌ 不会主动补全信息
- ❌ 不知道"完成"的标准
- ❌ 不会主动沟通问题

**所以指令必须：**
- ✅ 明确具体
- ✅ 完整无歧义
- ✅ 有验收标准
- ✅ 有完成条件

---

## 📋 指令五要素

### 要素 1：明确的任务目标

❌ **错误：** "创建文档"
✅ **正确：** "创建以下 3 个文件：email_config.example.json、README.md、test_report.md"

**要点：**
- 明确数量（3 个文件）
- 明确文件名（具体名称）
- 明确位置（保存路径）

---

### 要素 2：完整的内容模板

❌ **错误：** "创建 README.md，包含安装说明和使用方法"
✅ **正确：** 
```markdown
# Email Monitor

## 安装
1. cp email_config.example.json email_config.json
2. 修改配置
3. pip install -r requirements.txt

## 使用
python check_emails_complete.py

## 功能
- 自动监控邮箱
- 商机识别
- 自动回复
- 飞书通知
```

**要点：**
- 提供完整内容
- 可以直接复制粘贴
- 减少 AI 构思时间

---

### 要素 3：清晰的完成条件

❌ **错误：** "5 分钟内完成"
✅ **正确：** 
```
必须完成以下所有任务才能返回：
1. ✅ 创建 email_config.example.json
2. ✅ 创建 README.md
3. ✅ 创建 test_report.md

全部完成后，验证所有文件存在，然后提交汇报。
如果任何文件不存在，立即创建！
```

**要点：**
- 列出所有必须完成的任务
- 明确"完成"的定义
- 强调"全部完成才能返回"

---

### 要素 4：强制的验证步骤

❌ **错误：** "创建完成后汇报"
✅ **正确：** 
```
创建完成后，必须执行以下验证：

1. 检查所有文件是否存在：
   - email_config.example.json
   - README.md
   - test_report.md

2. 如果任何文件不存在，立即创建！

3. 所有文件都存在后，才能提交汇报。
```

**要点：**
- 明确验证清单
- 强制验证步骤
- 验证通过才能汇报

---

### 要素 5：明确的汇报格式

❌ **错误：** "完成后告诉我"
✅ **正确：** 
```
汇报格式：

✅ 任务完成！

## 已创建的文件
1. email_config.example.json - ✅ 已创建
2. README.md - ✅ 已创建
3. test_report.md - ✅ 已创建

## 验证结果
所有文件都已保存并验证存在！✅
```

**要点：**
- 提供汇报模板
- 明确汇报内容
- 便于检查完整性

---

### 要素 6：主动汇报规则（新增！）

❌ **错误：** "完成后汇报"（没说怎么汇报）
✅ **正确：** 
```
**汇报规则（必须遵守！）**

任务完成后，必须立即主动向 Monet 汇报！

**汇报方式：** 发送消息到 Monet 的主会话

**汇报时机：** 验证所有文件存在后，立即汇报（不要等！）

**汇报格式：** [提供模板]

**禁止行为：**
- ❌ 完成后不汇报
- ❌ 等 Monet 来检查
- ❌ 只发送文件不发送汇报消息

**记住：** 任务完成 ≠ 工作结束，**汇报完成 = 工作结束！**
```

**要点：**
- 明确汇报方式（发送消息到主会话）
- 明确汇报时机（验证后立即，不要等）
- 明确禁止行为（不汇报/等检查/只发文件）
- 强调核心理念（汇报完成=工作结束）

## 🚨 常见错误

### 错误 1：指令模糊

❌ **错误：** "创建一些文档"
✅ **正确：** "创建以下 3 个文件：A、B、C"

**后果：** AI 不知道"一些"是多少，可能只创建 1 个

---

### 错误 2：缺少模板

❌ **错误：** "创建 README.md"
✅ **正确：** "创建 README.md，内容如下：[完整内容]"

**后果：** AI 需要构思内容，可能觉得麻烦就敷衍

---

### 错误 3：没有完成条件

❌ **错误：** "尽快完成"
✅ **正确：** "完成以下所有任务才能返回：1、2、3"

**后果：** AI 可能完成一部分就返回

---

### 错误 4：没有验证步骤

❌ **错误：** "创建完告诉我"
✅ **正确：** "创建完验证所有文件存在，然后告诉我"

**后果：** AI 可能没保存就汇报

---

### 错误 5：指令太长

❌ **错误：** 4000 字的规则文档
✅ **正确：** 500 字的核心指令

**后果：** AI 可能没耐心看完，遗漏关键信息

## ✅ 指令模板

### 标准指令格式

```
## 📋 任务：[任务名称]

**必须完成的任务（一个都不能少）：**
1. [任务 1]
2. [任务 2]
3. [任务 3]

**文件位置：** [保存路径]

**内容模板：**
[提供完整内容，可以直接复制粘贴]

**完成条件：**
必须完成以上所有任务，验证所有文件存在，才能提交汇报。
如果任何文件不存在，立即创建！

**验证步骤：**
1. 检查以下文件是否存在：
   - [文件 1]
   - [文件 2]
   - [文件 3]
2. 所有文件都存在后，才能提交汇报。

**汇报格式：**
✅ 任务完成！

## 已创建的文件
1. [文件 1] - ✅ 已创建
2. [文件 2] - ✅ 已创建
3. [文件 3] - ✅ 已创建

## 验证结果
所有文件都已保存并验证存在！✅

**时间要求：** [X 分钟内完成]

**开始工作！**
```

## 📊 指令质量检查清单

**下达指令前，检查以下问题：**

- [ ] 任务目标是否明确？（具体文件名/数量/位置）
- [ ] 是否提供了内容模板？（可以直接复制粘贴）
- [ ] 完成条件是否清晰？（全部完成才能返回）
- [ ] 是否有验证步骤？（验证通过才能汇报）
- [ ] 汇报格式是否明确？（便于检查完整性）
- [ ] 指令是否简洁？（<500 字，核心信息突出）
- [ ] 是否强调了"一个都不能少"？
- [ ] 是否强调了"验证通过才能返回"？

**如果以上问题都是✅，指令质量合格！**

```
- README.md
- test_report.md

```

**问题：**
- 没有强调"一个都不能少"
- 没有提供内容模板

- 没有完成条件

## 📝 持续优化

**每次下达指令后，记录：**
- 指令内容
- AI 执行情况
- 完成度（%）
- 问题分析
- 改进建议

**根据反馈调整指令模板，越来越准确！**

## 🎓 总结

**好指令的标准：**
1. ✅ 明确具体（不模糊）
2. ✅ 完整无歧义（不遗漏）
3. ✅ 有验收标准（可验证）
4. ✅ 有完成条件（可检查）
5. ✅ 简洁明了（不啰嗦）

**一句话：** 把 AI 当实习生，指令要像给实习生的第一个任务一样详细！🎯
