**命令：**

### 步骤 4：计算完成度

**公式：**
```
完成度 = (实际完成的文件数 / 任务要求的文件数) × 100%
```

**示例：**
```
任务要求：创建 2 个文件（README.md + test_report.md）
实际完成：1 个文件（README.md）

完成度 = (1 / 2) × 100% = 50%
```

**完成度等级：**
- 100% - 全部完成
- 75-99% - 大部分完成
- 50-74% - 一半完成
- 25-49% - 少部分完成
- 0-24% - 几乎没完成

## 🚨 常见问题处理

### 问题 1：文件存在但内容是空的

**处理：**
1. 标记为"未完成"
2. 记录问题
3. 重新安排或手动修复

**示例：**
```
README.md - ✅ 文件存在，但内容为空
结论：❌ 不合格，需要重新创建
```

---

### 问题 2：文件内容不完整

**处理：**
1. 记录缺失内容
2. 计算完成度（如 70%）
3. 如实汇报

**示例：**
```
README.md - ✅ 文件存在
内容检查：
✅ 有标题
✅ 有安装说明
❌ 缺少使用方法
❌ 缺少常见问题

结论：70% 完成，需要补充
```

---

### 问题 3：代码有语法错误

**处理：**
1. 记录错误信息
2. 标记为"不合格"
3. 重新安排或手动修复

**示例：**
```
check_emails_complete.py - ✅ 文件存在
语法检查：❌ 失败
错误：SyntaxError on line 50

结论：❌ 不合格，需要修复
```

---

### 问题 4：完成度计算争议

**处理：**
1. 按文件数量计算（客观）
2. 不按内容质量计算（主观）
3. 如有争议，向老高说明

**示例：**
```
任务要求：创建 2 个文件
实际完成：2 个文件都存在

但 README.md 内容不完整（缺少常见问题）

计算方式 1（按文件数）：2/2 = 100%
计算方式 2（按内容）：70%

建议：按文件数计算（100%），但说明内容不完整
```
