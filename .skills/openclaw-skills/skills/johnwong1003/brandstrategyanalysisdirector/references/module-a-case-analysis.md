# 模块A：拆案例

## 任务目标

帮助设计师快速看懂案例，不只是"看过"，而是"拆开"。
从多个品牌案例中提取共性、差异、策略和方法。

## 输入识别

用户提供：品牌案例名称、参考图、竞品列表、行业描述等。

## 处理步骤

1. 识别每个案例的品牌类型、行业属性、目标人群
2. 提取视觉共性与表达差异
3. 分析背后的品牌策略与设计意图
4. 提炼可复用的方法
5. 输出结构化总结

## 输出格式

### 1. 案例共同特点

**视觉层共性**
- （色彩、字体、版式、图形语言等维度的共同特征）

**内容层共性**
- （信息架构、叙事方式、内容策略的共同特征）

**品牌表达共性**
- （调性、情绪、价值主张的共同特征）

---

### 2. 背后设计策略

**为什么这样做**
- （策略动机：市场定位、竞争差异化、用户心理等）

**服务了哪类用户**
- （目标用户画像及其决策场景）

**对品牌认知产生了什么作用**
- （认知建立、情感连接、记忆点强化等）

---

### 3. 可复用的方法

**可学的结构**
- （信息组织方式、版式逻辑）

**可学的表达**
- （语气、叙事角度、内容策略）

**可学的视觉策略**
- （色彩运用、字体搭配、图形系统）

**使用边界与注意点**
- （这些方法在什么情况下有效，什么情况下不适用）

---

### 4. 给当前项目的启发

**哪些适合借鉴**
- （结合当前项目背景，可直接参考的部分）

**哪些不适合照搬**
- （需要警惕的陷阱或不匹配的地方）

**建议如何本地化转换**
- （如何把这些方法转化为适合当前项目的版本）
