# 模块D：大资料整理与提炼

## 任务目标

从海量材料中，快速完成：
- 归类
- 去重
- 提炼重点
- 建立索引
- 提炼方法
- 辅助决策

## 输入类型

用户可能提供：案例列表、研究报告摘要、品牌档案、竞品资料、设计参考集等。

## 处理原则

- 先归类，再提炼，不要混在一起
- 高频出现的观点 = 重要共识，要单独标出
- 最终要指向"对当前项目有什么用"

## 输出格式

### 1. 资料分类结果

按主题、行业、风格、策略或用途分类。

| 分类名称 | 包含内容 | 数量/比例 |
|---------|---------|---------|
| | | |

---

### 2. 每类重点摘要

每类资料最值得看的信息是什么。

**[分类名称]**
- 核心内容：
- 最值得关注的点：

（每类3-5条，不要超过）

---

### 3. 高频重复信息

哪些观点反复出现，说明是重要共识。

（列出3-5条高频观点，说明出现频率和来源类型）

---

### 4. 值得深挖的信息

哪些内容可能对项目最有价值。

（指出2-3个值得进一步研究的方向，说明为什么值得深挖）

---

### 5. 可直接复用的结论

哪些内容能直接转化到当前项目。

（列出可直接用的结论，注明来源和适用条件）
