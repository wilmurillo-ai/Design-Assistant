# Skill SRL 评估工具

基于 SRL (Skill Reliability Level) 框架，评估 Skill 的可靠性和置信度。

## SRL 体系概述

### 核心逻辑

Skill 的置信度，取决于它的每一步输出中，有多少来自**可被外部验证的客观事实**，而不是**模型自身推理出来的信息**。

```
低置信度（AI 主导推理）                高置信度（工程事实主导）
SRL-1 ———— SRL-2 ———— SRL-3 ———— SRL-4 ———— SRL-5
```

### 六维评估框架

| 评估维度 | 核心定义 | 为什么不可或缺 |
|---------|---------|--------------|
| **确定性** | 同一输入执行 N 次的输出方差 | 方差越大，Skill 越像是在掷骰子 |
| **溯源性** | 输出是否可追溯至权威/可验证数据源 | 无法溯源的数据，在生产环境中等同于无效数据 |
| **时效性** | 数据时间戳与当前时间的差值 | 过期数据引发的决策失误，比没有数据更可怕 |
| **鲁棒性** | 面对 SEO 垃圾、矛盾信息时的抗干扰能力 | 衡量 Skill 在真实互联网中的生存能力 |
| **失败透明度** | 无法获取可靠信息时，是否主动降级置信度 | "我不知道"的价值，远大于"我编一个" |
| **级联稳定性** | 作为子节点时，误差放大的系数 | 决定了该 Skill 能否被安全地编排进长任务链路中 |

> **额外关键指标**：**漂移率** — 最常见的失效模式不是第一次就跑不通，而是上周还正常，这周悄悄变坏。

### SRL 五级分类

| 等级 | 特征 | 使用建议 |
|------|------|---------|
| **SRL-1** | 几乎没有工程锚点，全链路依赖 AI 自主推理 | 严禁用于正式决策 |
| **SRL-2** | 有少量外部事实，大量步骤仍依赖 AI 二次推理 | 结果只能参考，必须人工复核 |
| **SRL-3** | 已接入部分稳定数据源，但仍混入弱校验环节 | 适合在明确边界下使用 |
| **SRL-4** | 关键步骤锚定在可靠平台数据或官方接口上，失败信号清晰 | 结果基本可复现 |
| **SRL-5** | 全流程依赖强契约工程化工具（CLI/API/DB），AI 只负责编排和格式化 | 可直接进入自动化决策链路 |

### 核心原则

- 可验证的外部事实越多 → 置信度越高
- 模型自主推理的步骤越多、嵌套越深 → 幻觉越容易在链路中悄悄累积

> 非确定性 Skill 最致命的缺陷，不是它会不会报错，而是它经常在**不报错的情况下给出错误结论**。

## 架构

```
AI 阅读 Skill 全部内容
  → 逐维度评审（5 个维度 + 3 个修正因子）
  → 输出结构化 JSON
  → Python 脚本加权计算 + 等级映射 + 报告生成
```

- **AI 负责**：语义理解、评分判断、证据引用
- **脚本负责**：数学加权、修正因子应用、等级映射、报告格式化

## 评估流程

1. **阶段 0** — 确认评估目标（路径、单个/批量）
2. **阶段 1** — 阅读 Skill 全部内容（SKILL.md + 脚本 + README）
3. **阶段 2** — AI 逐维度评审（5 维度 + 3 修正因子，给出子项分 + 证据）
4. **阶段 3** — 输出结构化评分 JSON
5. **阶段 4** — 运行 Python 脚本计算（加权 + 修正 + 等级映射 + 报告生成）
6. **阶段 5** — 输出最终报告

## 五维评分

| 维度 | 权重 | 说明 |
|------|------|------|
| 工程锚点密度 | 25% | 有多少步骤有 CLI/API/DB 等工程化验证手段 |
| 幻觉暴露面 | 20% | 纯 AI 推理且无外部验证的步骤占比 |
| 失败语义清晰度 | 20% | 出错时是"停下来报错"还是"脑补继续跑" |
| 溯源性 | 20% | 输出能否追溯到权威数据源 |
| 可重现性推断 | 15% | 同输入是否能得到同输出 |

### 修正因子

| 因子 | 触发条件 | 影响 |
|------|---------|------|
| 时效性 | < 50 | 可重现性线性扣减，最大 -20% |
| 鲁棒性 | < 50 | 幻觉暴露面线性扣减，最大 -10% |
| 级联稳定性 | < 50 | 可重现性线性扣减，最大 -20% |

## SRL 等级

| 等级 | 分数 | 含义 |
|------|------|------|
| SRL-5 ★★★★★ | 90+ | 全流程工程化，可自动化 |
| SRL-4 ★★★★ | 75-89 | 关键步骤锚定，基本可复现 |
| SRL-3 ★★★ | 60-74 | 部分弱校验，有边界使用 |
| SRL-2 ★★ | 40-59 | 需人工复核 |
| SRL-1 ★ | <40 | 不建议正式使用 |

## 脚本使用

**依赖**: Python 3.6+，仅使用标准库，无第三方依赖。

脚本接收 AI 评审输出的结构化 JSON 数据，执行加权计算和报告生成。

```bash
# 查看输入 JSON Schema
python3 scripts/srl_analyzer.py --schema

# 从 stdin 读取 JSON
echo '{"skill_name":"xxx", ...}' | python3 scripts/srl_analyzer.py

# 从文件读取
python3 scripts/srl_analyzer.py --input eval-data.json

# JSON 格式输出
python3 scripts/srl_analyzer.py --input eval-data.json --json

# 批量模式
python3 scripts/srl_analyzer.py --input batch-data.json --batch
```
