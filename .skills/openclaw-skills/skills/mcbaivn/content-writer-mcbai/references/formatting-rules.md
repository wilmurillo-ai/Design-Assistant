# Formatting Rules — CRITICAL Reference

These rules are ABSOLUTE. Violating ANY of them means the output is unusable on LinkedIn.

## Why These Rules Exist

LinkedIn renders posts as plain text. It does NOT support:
- Markdown bold (**text**)
- Markdown italic (*text*)
- Markdown headers (# text)
- Markdown links ([text](url))
- Em dashes (—)

If you include any of these, the raw characters appear in the post and it looks broken.

## The Rules

### 1. ZERO Asterisks (*)
- No bold: ~~**text**~~ → Use CAPS for 1-2 key words
- No italic: ~~*text*~~ → Just write normally
- No bullets: ~~* item~~ → Use 1. 2. 3. or → arrows

### 2. ZERO Markdown
- No headers: ~~# Title~~ → Just write the title as a line
- No links: ~~[text](url)~~ → Don't include URLs
- No code blocks, no backticks, no formatting characters

### 3. ZERO Em Dashes (—)
- ~~"This is important — really important"~~
- "This is important - really important" ✓
- "This is important, really important" ✓

### 4. ZERO URLs in Post Body
- Source URLs are NEVER included in the text
- If the user wants to reference a source, mention the company/publication name only
- Example: "According to TechCrunch..." not "According to https://techcrunch.com/..."

### 5. Plain Text Only
The final output should be copyable directly into LinkedIn's post editor with zero modifications.

## What IS Allowed

| Element | How to Use |
|---------|-----------|
| Emphasis | CAPS for 1-2 key words: "This is the REAL opportunity" |
| Lists | Numbered: 1. 2. 3. |
| Sub-details | → arrows: "→ Revenue grew 40%" |
| Emoji | Sparingly, max 2-3 per post, for section breaks |
| Line breaks | Empty lines between paragraphs |
| Questions | End with engagement questions |

## Pre-Publish Checklist

Before presenting the post to the user, verify:

- [ ] No * characters anywhere
- [ ] No ** bold markers
- [ ] No # header markers
- [ ] No [] or () link syntax
- [ ] No — em dashes (only - hyphens allowed)
- [ ] No URLs in the text
- [ ] Short paragraphs (1-2 sentences)
- [ ] Specific data in every major claim
- [ ] Hook in first 1-2 lines
- [ ] CTA or question at the end
- [ ] Word count within target range
- [ ] Correct language (EN or VN as specified)
