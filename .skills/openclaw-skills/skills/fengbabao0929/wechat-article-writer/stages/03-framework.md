# 阶段03：框架搭建

## 目标
设计文章结构，确保逻辑清晰、层层递进

---

## 执行步骤

### 步骤1：选择文章类型框架

根据确认的文章类型，选择对应框架（目标2500-3000字）：

---

**干货教程类：**
```
开篇：痛点场景引入（150字）
第一部分：为什么需要这个方法（300字）
第二部分：方法详解 - 分4-5步（1200字）
第三部分：实际案例演示（600字）
第四部分：常见问题/避坑指南（500字）
结尾：行动号召 + 鼓励（250字）
---
总计：3000字
```

---

**观点文章类：**
```
开篇：现象/问题引入（200字）
第一部分：现象背后的原因（400字）
第二部分：我的核心观点（600字）
第三部分：支撑观点的论据1（600字）
第四部分：支撑观点的论据2（600字）
结尾：总结升华（400字）
---
总计：2800字
```

---

**故事案例类：**
```
开篇：结果前置/悬念引入（150字）
第一部分：背景设定（400字）
第二部分：冲突/挑战（600字）
第三部分：转折/突破（800字）
第四部分：收获/启示（600字）
结尾：金句升华（250字）
---
总计：2800字
```

---

**列表盘点类：**
```
开篇：价值承诺（150字）
第一部分：盘点清单（逐个介绍，共2200字）
  - 项目1-10：每个150-250字
  - 名称+一句话介绍+详细说明+适用场景
结尾：使用建议+行动号召（350字）
---
总计：2700字
```

---

### 步骤2：细化小标题

为每个部分设计吸引人的小标题：

```
❌ 抽象： "第一部分"、"方法介绍"
✅ 具体： "01｜别人都在做，你为什么不做？"
✅ 数字： "02｜4个步骤，从0到1"
✅ 提问： "03｜是不是你的问题出在这里？"
✅ 情绪： "04｜太爽了！终于搞定了这个问题"
```

---

### 步骤3：风格检查

在设计框架时，确保：

```
□ 段落是否简短？（2-3句一段）
□ 能否加入情绪表达？（爽/不爽/兴奋）
□ 能否加入真实体验？（我的踩坑经历）
□ 是否有实用价值？（删掉废话）
□ 口语化程度够吗？（像聊天一样）
```

---

### 步骤4：确认框架

向用户展示框架：

```
【文章框架】

标题：XXX
预计字数：2500-3000字

结构：
01｜XXX（200字）
02｜XXX（400字）
03｜XXX（800字）
04｜XXX（800字）
结尾｜XXX（300字）

这个框架可以吗？需要调整吗？
```

---

## 输出模板

确认后记录到 `framework.yaml`：

```yaml
title: "文章标题"
estimated_length: 2700
style_guide: "口语化、情绪外放、段落简短、实用主义、真实体验"

structure:
  - section: "开篇"
    subtitle: "XXX"
    word_count: 200
    key_points: []
    style_notes: "痛点共鸣+情绪表达"
  - section: "第一部分"
    subtitle: "XXX"
    word_count: 400
    key_points: ["要点1", "要点2"]
    style_notes: "真实体验+具体例子"
  # ... 更多部分
```
