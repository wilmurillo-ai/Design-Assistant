# 阶段04：全文撰写

## 目标
根据框架完成2500-3000字文章初稿

---

## 写作风格（必须遵守）

### 1. 口语化表达
```
❌ 书面： "我们需要深入思考这个问题，才能得出有效结论。"
✅ 口语： "这个问题得好好想想，不然容易踩坑。"
```

### 2. 情绪外放
```
✅ 太爽了！终于搞定了这个问题！
✅ 真的气死我了，这个坑我踩了三遍！
✅ 超级兴奋！刚发现一个神仙工具！
```

### 3. 完整句子表达（重要！）
```
❌ 过于碎片化：
昨天研究这个问题。

发现一个奇怪现象。

参数都调好了。

结果反而更糟。

✅ 完整表达：
昨天研究这个问题的时候，我发现了一个奇怪的现象：参数都调好了，结果反而更糟。我困惑了好久，反复检查了几遍流程，最后才发现——原来一开始就搞错了。
```

### 4. 实用主义
```
少废话，多干货
删掉没有信息量的段落
```

### 5. 真实体验
```
敢吐槽，敢承认不足
分享踩坑经历
用第一人称"我"
```

## 执行步骤

### 步骤1：撰写开篇（150-250字）

**开篇目标：3秒抓住读者**

选择一种方式：

**方式A：痛点共鸣**
```
"昨天，一个粉丝私信我：

'老师，我已经坚持XXX一个月了，为什么还是XXX？'

这个问题，让我想了很久..."
```

**方式B：数据震撼**
```
"90%的XXX者，都在第3个月放弃。

这是我在行业里观察了5年，得出的数据。

说实话，看到这个数字的时候，我也挺惊讶的。"
```

**方式C：场景代入**
```
"周五晚上10点，我刚关掉电脑，手机弹出一条消息：XXX

我当时就坐不住了，立马打开电脑继续干。"
```

**方式D：金句开场**
```
"'XXX，是最好的XXX。'

这句话我听了100遍，直到昨天，我才真正理解了它的意思。"
```

---

### 步骤2：撰写正文（2000-2500字）

**写作原则：**

1. **用完整句子表达** - 避免短句碎片化
2. **用具体例子** - 不要只讲抽象概念
3. **关键信息加粗** - 帮助快速扫描
4. **适当留白** - 段落间空行
5. **加入情绪** - 爽/不爽/兴奋
6. **真实体验** - 我的踩坑经历

**示例对比：**
```
❌ 碎片化： "我们需要坚持学习。

不断提升自己。

才能适应社会。"

✅ 完整表达： "我们需要坚持学习，不断提升自己，才能适应这个快速变化的社会。"

❌ 抽象： "这个方法很有效。"
✅ 情绪+具体： "这招真管用！我亲测了一周，效果绝了！"
```

---

### 步骤3：撰写结尾（200-350字）

**结尾目标：留下印象或引导行动**

选择一种方式：

**方式A：总结+号召**
```
"回顾一下，今天分享了：

1. XXX
2. XXX
3. XXX

如果觉得有用，点个「在看」，让更多人看到~

你在XXX中遇到过什么问题？

欢迎在评论区分享！"
```

**方式B：金句升华**
```
"XXX的路上，没有捷径。

但每一步，都算数。

与君共勉。"
```

**方式C：互动提问**
```
"你在XXX中遇到过什么问题？

欢迎在评论区分享~

我会选出3位朋友，送出XXX"
```

### 步骤4：自我检查

完成初稿后，检查以下问题：

```
□ 开篇是否有吸引力？
□ 用完整句子表达了吗？（避免短句碎片化）
□ 段落之间有逻辑连接吗？
□ 是否有具体例子？
□ 是否有情绪表达？（爽/不爽/兴奋）
□ 是否有真实体验？（我的经历/踩坑）
□ 逻辑是否清晰？
□ 是否有错别字？
□ 结尾是否有行动号召？
□ 字数是否达到2500-3000？
```

文章完成后，自动执行：

```

**排版要求：**
- 使用标准Markdown语法
- 标题层级清晰（# ## ###）
- 引用块、表格、列表格式正确
- 适当留白，便于阅读

**6.2 询问用户是否生成图片**
```

是否需要调用API生成图片？

**配图提示词模板：**
- 封面图：`image_templates/cover.md`

**用户确认后才执行：**
```bash
python scripts/generate_images.py --input prompts.jsonl
```

## 注意事项

- 严格按照 `writing_style.md` 的风格写作
- **用完整句子表达，避免短句碎片化**
- 保持用户的语言风格
- 不要过度使用emoji（1-2个/篇即可）
- 专业术语要解释
- 数据建议标注来源
- 保持真诚，避免套路

- **完成后自动保存到指定目录**
