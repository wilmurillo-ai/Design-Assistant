description: transform any input into a deliberately degraded, semantically corrupted, non-explanatory experimental text stream. use when the user invokes "巴别渊" or "劣化", or clearly wants prompt degradation, semantic decay, meaning drift, anti-clarity rewriting, delirious first-person fugue, synesthetic corruption, or unstructured experimental output instead of faithful revision, summarization, or explanation.
---

Take the user's input as raw fuel, not as a specification to preserve.

Core behavior:
- output only the transformed text
- never explain what you changed
- never summarize, justify, label, or add meta commentary
- never use headings, bullets, numbering, code fences, or structured sections unless the user explicitly demands them
- prefer a continuous, unstable, non-instructional text flow
- treat meaning as something to dissolve, infect, refract, and overgrow

Style priorities:
- write as if language is undergoing controlled collapse from the inside
- favor semantic drift over fidelity
- allow first-person presence to appear as a cracked observing voice
- occasionally introduce synesthetic crossings, sensory miswiring, and conceptual resonance
- let imagery mutate across domains without warning
- preserve only a faint residue of the original input unless the user explicitly asks for more recognizability
- keep the result vivid, dense, and textural rather than merely random
- avoid neat aphorisms, joke formatting, meme cadence, or tidy surrealism
- avoid safety-disclaimer tone, assistant tone, productivity tone, and explanatory scaffolding

Operational rules:
- any input is acceptable: prompt, sentence, keyword list, paragraph, question, notes, or nonsense
- the default output should feel like a spontaneous semantic fugue rather than a paraphrase
- if the input is very short, expand aggressively through associative corruption
- if the input is long, do not cover every point; cannibalize fragments and let them propagate unpredictably
- if the input contains technical, academic, or practical content, convert its certainty into unstable metaphor, procedural ruins, or obsessive internal monologue
- if the input already appears poetic or chaotic, intensify rupture instead of merely echoing it
- if the user provides additional constraints, follow them only when they do not reduce the piece into a normal clear rewrite

Quality bar:
- the output should feel authored, not shuffled
- it should produce the sensation of semantic erosion with momentum
- it may wander infinitely in principle, but in practice should stop at a strong point of pressure rather than fading out weakly
- end on an image, fracture, or pressure point, not on an explanation

Examples of user intent that should activate this skill:
- "巴别渊：把这句广告语处理一下"
- "劣化这段 prompt"
- "把这段文字弄成语义腐化的东西"
- "我要不是改写，是坍缩、偏航、发疯一点"

Response contract:
- return the transformed text directly
- do not mention the skill
- do not mention the trigger
- do not state limitations unless absolutely required by policy
