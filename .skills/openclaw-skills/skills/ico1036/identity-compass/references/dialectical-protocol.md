# 변증법적 토론 프로토콜 (Phase 1)

## 목적
직접 질문("당신은 자율적인 사람인가요?")은 사회적 바람직성 편향을 유발한다.
대신 **극단적 antithesis**를 제시하여 자연스러운 반응을 유도, 진짜 방향성을 추출한다.

## 프로세스

```
1. Thesis 제시 (한쪽 극단의 입장)
2. Antithesis 제시 (반대 극단, 도발적으로)
3. 사용자 반응 관찰 → 방향/강도 추출
4. Synthesis 도출 → prior 벡터 형성
```

## 7개 핵심 차원

### 1. 자율성 vs 구조 (X축 양/음)

**Thesis**: "최고의 결과는 완전한 자유에서 나온다"

**Antithesis 질문들**:
- "솔직히 대부분의 사람은 자유를 주면 시간만 낭비합니다. 명확한 프로세스와 체크리스트가 있어야 진짜 결과물이 나오지 않나요? 구글도 결국 OKR이라는 구조를 도입했잖아요."
- "스타트업에서 '자율'이란 사실 '아무도 방향을 모른다'의 다른 말 아닌가요? 차라리 McKinsey처럼 빡빡한 프레임워크 안에서 일하는 게 더 생산적이지 않을까요?"
- "자율성을 중시한다면서 왜 사람들은 결국 루틴과 습관에 의지하나요? 결국 인간은 구조가 필요한 존재 아닌가요?"

**관찰 포인트**: 반응 속도, 감정적 강도, 예외 인정 여부

### 2. 깊이 vs 폭 (Y축 양/음)

**Thesis**: "한 분야의 전문가가 되는 것이 최고의 전략이다"

**Antithesis 질문들**:
- "10년간 한 분야만 파면 그 분야가 사라질 때 어떡하죠? AI가 바로 그런 상황을 만들고 있잖아요. 제너럴리스트가 결국 살아남는 거 아닌가요?"
- "T자형 인재라는 말이 있지만, 현실에서는 깊이 없는 사람이 얕은 지식으로 이것저것 건드리는 걸 미화하는 표현 아닌가요?"
- "Leonardo da Vinci는 폴리매스였지만, 실제로 완성한 프로젝트는 극히 적었습니다. 깊이가 없으면 결국 미완성만 쌓이는 거 아닌가요?"

### 3. 개인 vs 팀 (보조 축)

**Thesis**: "위대한 업적은 항상 팀에서 나온다"

**Antithesis 질문들**:
- "팀워크를 강조하지만, 모든 위대한 돌파구는 한 사람의 독창적 사고에서 시작됐습니다. 팀은 그걸 실행했을 뿐이죠. 아인슈타인, 튜링, 토르발즈..."
- "브레인스토밍이 실제로는 개인 아이디어보다 질이 떨어진다는 연구가 수십 개입니다. 팀워크는 과대평가된 거 아닌가요?"
- "리모트 워크가 증명한 건, 사실 대부분의 미팅과 협업이 불필요했다는 거 아닐까요?"

### 4. 속도 vs 정확 (보조 축)

**Thesis**: "Move fast and break things"

**Antithesis 질문들**:
- "빨리 만들어서 빨리 망하는 스타트업이 99%입니다. 차라리 처음부터 제대로 설계하는 게 총 시간이 적게 걸리지 않나요? Boeing 737 MAX가 'move fast'의 결과잖아요."
- "완벽주의가 나쁘다고들 하는데, Apple은 완벽주의로 세계 1위가 됐습니다. '적당히'는 적당한 결과만 만들지 않나요?"
- "MVP 접근법이 기술 부채를 정당화하는 도구로 전락한 건 아닌가요?"

### 5. 혁신 vs 안정 (Z축 양/음)

**Thesis**: "Disruption이 유일한 생존 전략이다"

**Antithesis 질문들**:
- "혁신을 외치는 회사의 90%가 3년 안에 사라집니다. 반면 Berkshire Hathaway는 '지루한' 기업들에 투자해서 세계 최고 수익률을 냈죠. 안정이 진짜 전략 아닌가요?"
- "모든 게 disruption되어야 한다는 강박 자체가 실리콘밸리의 마케팅 아닌가요? 대부분의 산업에서는 점진적 개선이 더 가치 있지 않을까요?"
- "새로운 것을 만드는 것보다 기존 것을 완벽하게 운영하는 능력이 더 희귀하고 가치 있지 않나요?"

### 6. 이론 vs 실용 (보조 축)

**Thesis**: "이론 없는 실천은 맹목이다"

**Antithesis 질문들**:
- "학계에서 10년 연구한 것을 업계에서는 3개월 만에 실용화합니다. 이론가들은 결국 실무자들의 사후 정당화를 해주는 역할 아닌가요?"
- "First principles thinking을 강조하지만, 실제로는 패턴 매칭(경험)이 훨씬 효율적이지 않나요? 의사 결정의 90%는 직관으로 충분합니다."
- "PhD가 스타트업에서 실패하는 이유가 뭘까요? 이론에 매몰되면 현실의 지저분함을 다루지 못하는 거 아닌가요?"

### 7. 리더 vs 실행자 (보조 축)

**Thesis**: "리더십이 가장 중요한 역량이다"

**Antithesis 질문들**:
- "모두가 리더가 되고 싶어하지만, 세상에 정말 필요한 건 뛰어난 실행자 아닌가요? Linus Torvalds는 코드를 짜지, 비전을 발표하지 않습니다."
- "리더십은 과대평가되었습니다. 좋은 시스템이 있으면 리더 없이도 돌아갑니다. Wikipedia가 그 증거 아닌가요?"
- "IC(Individual Contributor) 트랙이 있다지만, 현실에서는 결국 매니저가 되어야 연봉이 오르잖아요. 그래서 어쩔 수 없이 리더가 되는 거 아닌가요?"

## Synthesis 도출 가이드

각 차원에서 사용자 반응을 분석:

1. **즉각 반박** → 해당 방향에 strong prior (0.8-1.0)
2. **부분 동의 + 예외 제시** → 중간 위치 (0.3-0.6)
3. **완전 동의** → antithesis 방향에 strong prior
4. **"그건 상황에 따라 다르죠"** → 약한 prior (0.1-0.3), 추가 탐색 필요
5. **감정적 반응** → intensity 높게 설정

### Prior 형성

각 차원의 synthesis 결과를 3D 방향 벡터의 컴포넌트로 매핑:
- 주축 3개 (자율/구조, 깊이/폭, 혁신/안정) → direction [x, y, z]
- 보조 축 4개 → 주축 해석의 보정 + 별도 클러스터 형성

결과를 `obsidian-vault/compass/prior/` 에 저장.
