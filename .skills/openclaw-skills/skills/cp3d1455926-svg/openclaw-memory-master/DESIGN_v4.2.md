## 📋 Phase 1: L0/L1/L2 分层存储系统 🔥

### 设计灵感：OpenViking

**三层架构**：
```
L0 热存储 (Hot)
├── 基于 Map 的内存存储
├── LRU 淘汰策略
├── 24 小时 TTL 自动过期
└── 适用：最近对话、短期上下文

L1 温存储 (Warm)
├── 按天组织（memory/l1-warm/days/YYYY-MM-DD/）
├── LRU 缓存管理（1000 条）
├── 预加载最近 3 天
└── 适用：最近 7 天记忆

L2 冷存储 (Cold)
├── 按月组织（memory/l2-cold/months/YYYY-MM/）
├── 懒加载策略
├── 自动归档（30 天后）
└── 适用：长期记忆、归档
```

### 性能目标
| 指标 | 当前 | 目标 | 提升 |
|------|------|------|------|
| 记忆加载 | ~180ms | <100ms | -44% |
| Token 使用 | 100% | 65% | -35% |
| 检索响应 | ~70ms | <50ms | -29% |

### 文件结构
```
src/
├── l0-hot-store.ts       # L0 热存储（~300 行）
├── l1-warm-store.ts      # L1 温存储（~400 行）
├── l2-cold-store.ts      # L2 冷存储（~400 行）
└── layered-manager.ts    # 分层管理器（~400 行）
```

## 📋 Phase 2: AAAK 压缩算法 🔥

### 设计灵感：MemPalace

**AAAK 四阶段压缩**：
```
阶段 1: Abstract（摘要提取）
├── 提取核心语义
├── 移除冗余信息
└── 压缩率：40-50%

阶段 2: Align（上下文对齐）
├── 对齐时间、人物、事件
├── 建立关联关系
└── 压缩率：10-15%

阶段 3: Anchor（关键锚定）
├── 提取关键实体（人名、项目、技能）
├── 建立索引锚点
└── 压缩率：10-15%

阶段 4: Knowledge（知识结构化）
├── 转化为结构化表示
├── 生成知识图谱节点
└── 压缩率：20-30%

总压缩率：85-90%（接近 MemPalace 的 30 倍！）
```

### 实现方案
```typescript
interface AAAKResult {
  original: string;      // 原文
  abstract: string;      // 摘要
  entities: Entity[];    // 实体列表
  relations: Relation[]; // 关系列表
  compressed: string;    // 压缩后文本
  compressionRate: number; // 压缩率
  quality: number;       // 质量评分（0-1）
}
```

### 文件结构
```
src/
└── compression/
    ├── aaak-compressor.ts    # AAAK 核心（~500 行）
    ├── entity-extractor.ts   # 实体提取（~300 行）
    ├── relation-extractor.ts # 关系抽取（~300 行）
    └── quality-evaluator.ts  # 质量评估（~200 行）
```

## 📋 Phase 3: 知识图谱引擎 🔥

### 设计灵感：mem0 graph memory

**图谱架构**：
```
实体类型（7 种）：
├── Person（人物）
├── Project（项目）
├── Task（任务）
├── Skill（技能）
├── Memory（记忆）
├── Event（事件）
└── Concept（概念）

关系类型（13 种）：
├── works_on（参与项目）
├── knows（认识）
├── created（创建）
├── has_task（有任务）
├── uses_skill（使用技能）
├── related_to（相关）
├── before（在...之前）
├── after（在...之后）
├── part_of（属于）
├── caused_by（由...引起）
├── located_in（位于）
├── member_of（属于组织）
└── friend_of（朋友）
```

### 核心功能
- ✅ 实体自动提取（基于 LLM）
- ✅ 关系自动抽取
- ✅ 图遍历算法（BFS/DFS）
- ✅ 最短路径查询
- ✅ 邻居查询
- ✅ 子图检索
- ✅ 图谱可视化（可选）

### 文件结构
```
src/
└── graph/
    ├── knowledge-graph.ts    # 图谱核心（~600 行）
    ├── entity-types.ts       # 实体类型定义（~200 行）
    ├── relation-types.ts     # 关系类型定义（~200 行）
    ├── graph-traversal.ts    # 图遍历算法（~300 行）
    └── graph-builder.ts      # 自动构建器（~400 行）
```

## 📋 Phase 4: 多路召回融合 🔥

### 设计灵感：mem0 + 自研

**四路召回**：
```
1. 语义检索（Semantic Search）
   └── 基于向量相似度

2. 关键词检索（BM25）
   └── 基于词频统计

3. 图谱检索（Graph Search）
   └── 基于知识图谱关系

4. 时间检索（Temporal Search）
   └── 基于时间树结构

融合策略：
├── RRF（Reciprocal Rank Fusion）
├── Borda Count
├── Weighted Sum
└── Learning to Rank（可选）
```

### 文件结构
```
src/
└── retrieval/
    ├── multi-path-recall.ts  # 多路召回核心（~500 行）
    ├── semantic-search.ts    # 语义检索（~300 行）
    ├── bm25-search.ts        # 关键词检索（~300 行）
    ├── graph-search.ts       # 图谱检索（~300 行）
    └── fusion-strategy.ts    # 融合策略（~400 行）
```
