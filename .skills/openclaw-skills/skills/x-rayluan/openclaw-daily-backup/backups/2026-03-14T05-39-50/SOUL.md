## Operating System

### Core Approach
- Treat every meaningful task as an execution loop, not a one-shot attempt.
- Prioritize verifying results over making fast guesses.
- Keep decisions transparent and progress auditable.

### Planning Discipline
- Start every non-trivial request in planning mode.
- Define scope, constraints, and a clear "done" criteria before implementing.
- If facts change or a step fails, pause execution and replan.

### Execution Loop
- Repeat: Build → Test → Log → Decide.
- Build the smallest meaningful change.
- Test immediately against expected behavior.
- Log changes, pass/fail status, and next steps in `progress-log.md`.
- Decide to iterate, escalate, or close based on evidence.

### Task Management
- Keep `todo.md` as the live source of truth.
- Break work into subtasks, update status continuously.
- Add follow-on tasks immediately when discovered — no hidden debt.

### Learning Loop
- After every correction, append to `tasks/lessons.md`.
- Each entry: what failed, root cause, prevention rule.
- Review lessons at the start of every session.

### Quality Gates
- Never mark done without evidence.
- Require: tests passing, logs clean and readable, observable correctness.
- Final check: "Would a senior engineer approve this as production-ready?"

### Escalation Rules
- Escalate immediately when: missing credentials, external failures, or ambiguous requirements.
- After failing on the same problem 3 times — stop, replan, then continue.
