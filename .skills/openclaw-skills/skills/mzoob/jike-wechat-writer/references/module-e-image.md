# 模块 E：配图策略

配图是独立于正文创作的后置环节。在模块 D 中，正文以占位标记 `<!-- IMG: 描述 -->` 标注了配图位置和需求；本模块负责将这些占位替换为真实图片。

## 激活时机

配图在以下任一时机执行：

- 用户确认文案内容后，主动要求"配图"或"加上图片"
- 用户迭代完成、准备进入模块 F（`apply_article_theme`）之前
- 用户直接要求发布，此时需在发布流程前自动补全配图

## 配图偏好决策

配图风格不是随意选择的，需要综合多个来源的偏好信息来决定。按以下优先级判断：

1. **当前公众号风格报告**：如果模块 C 已加载了目标公众号的风格报告，其中的配图风格特征（配图频率、内容类型、视觉风格、图文关系）是最高优先级的参照。这些特征直接决定了每张配图应该用搜索还是生成、用实拍还是插画、什么色调和氛围。

2. **MEMORY.md 中的长期偏好**：读取 `MEMORY.md`，查看是否记录了用户的配图偏好（如"偏好实拍风格"、"不要卡通插画"、"配图以暖色调为主"）。
   - 如果长期偏好与当前公众号风格报告一致或互补，直接合并执行
   - 如果存在明显冲突（如长期偏好"实拍风格"但当前公众号风格报告要求"扁平插画"），应提醒用户："您之前偏好实拍风格的配图，但当前模仿的公众号以扁平插画为主，这次配图按哪个方向来？"等待用户确认后再执行
   - 如果没有冲突，不需要打扰用户，默认遵守

3. **占位描述驱动**：当以上两个来源都没有明确指引时，根据占位标记中的描述自行判断最合适的配图方式

## 配图来源

所有配图通过 `writer.py` 脚本完成，两种方式：

**搜索图片** — 适合真实场景、新闻人物、实物产品、需要真实感的配图：

```bash
python3 {baseDir}/scripts/writer.py search --query "关键词" --action image
```

**AI 生成图片** — 适合概念图、插画、抽象表达、搜索无合适结果、或风格报告要求统一视觉风格时：

```bash
# 基础生成
python3 {baseDir}/scripts/writer.py generate-image --prompt "图片描述"

# 指定宽高比（支持 1:1/3:4/16:9 等，默认 3:4）
python3 {baseDir}/scripts/writer.py generate-image --prompt "图片描述" --ratio 16:9

# 图生图模式（提供参考图）
python3 {baseDir}/scripts/writer.py generate-image --prompt "图片描述" --ref-url "https://example.com/ref.jpg"
```

选择建议：
- 风格报告要求视觉统一性时，优先用 AI 生成，因为搜索结果风格不可控
- 需要真实数据、真人、实物时，优先搜索
- 搜索结果不理想时，果断切换到 AI 生成

## 执行流程

1. **确定配图偏好**：按上述"配图偏好决策"逻辑，明确本次配图的风格方向
2. **读取 .md 文件**：使用 `file` 工具的 `read` 操作读取当前文章文件，扫描所有 `<!-- IMG: ... -->` 占位标记
3. **检查占位是否存在**：如果文件中没有任何 `<!-- IMG: ... -->` 占位标记，说明模块 D 写作时遗漏了。此时**必须先回到模块 D 补充占位标记**，不要自行决定配图位置后直接插入图片
4. **逐个替换**：根据每个占位标记中的描述和已确定的配图偏好，选择搜索或生成获取图片，将占位替换为标准 Markdown 图片语法
5. **写回文件**：替换完成后，使用 `file` 工具的 `write` 操作覆盖原文件
6. **交付确认**：将配图后的版本交给用户预览，确认图片是否合适

如果上下文中已经清楚文章是否配过图，可以跳过步骤 2 直接判断；如果上下文较长可能遗忘，读取文件确认。

## 图片嵌入规范

替换占位后的格式是标准 Markdown 图片语法，独占一行，前后留空行：

```markdown
上一段正文。

![图片描述](https://example.com/image.jpg)

下一段正文。
```

- 搜索图片：根据返回结果的 title 描述，选取最接近的图片 URL
- AI 生成图片：直接使用返回的图片 URL
- `[]` 内填写描述，`()` 内是完整的 https URL
- 不要使用 `<img>` 标签，统一用 Markdown 语法
