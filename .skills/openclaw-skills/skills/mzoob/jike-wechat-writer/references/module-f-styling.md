# 模块 F：公众号样式应用

纯 Markdown 格式无法满足微信公众号的排版需求。此模块负责将文章转换为带有公众号排版样式的 HTML 内容。

## 激活条件（满足任一即激活）

- 用户明确要求"加上公众号样式"、"排版一下"、"生成公众号格式"
- 用户希望获得可直接粘贴到微信编辑器的排版内容
- 上下文中存在用户指定的公众号排版样式偏好

## 样式选择

通过 `render.py` 脚本将 `.md` 文件渲染为带内联样式的 HTML。所有样式以 fragment 模板形式内联，确保微信编辑器兼容。

主题的确定规则（按优先级）：

```
样式确定流程：
│
├── 1. 检查 MEMORY.md 是否记录了用户的排版主题偏好
│   ├── 有主题 ID → 直接使用
│   └── 无 → 继续
│
├── 2. 询问用户是否有喜欢的公众号文章样式
│   ├── 用户提供了公众号文章链接
│   │   → 执行 extract-style <URL> --name "主题名" --id theme-id
│   │   → 自动抓取并拆解样式 → 保存主题到 styles/
│   │   → 将主题 ID 记录到 MEMORY.md
│   │   → 使用该主题渲染
│   ├── 用户想看现有主题
│   │   → 执行 list-themes 展示可选主题
│   │   → 用户选择后记录到 MEMORY.md
│   └── 用户无偏好
│       → 使用默认 blue-minimal
│
└── 3. 使用确定的主题执行渲染
```

关键说明：
- 在"直接用默认"之前，增加一个主动询问环节，让用户有机会自定义样式
- 支持用户通过 `extract-style` 从任意公众号文章链接拆解排版样式
- 选择结果记录到 MEMORY.md，后续创作不再重复询问

可通过 `python3 {baseDir}/scripts/render.py list-themes` 查看所有可用主题。

## 执行流程

1. **确认文章已定稿**：检查模块 D 中的文章是否已获得用户确认。如果未确认，先引导用户完成内容确认
2. **确定主题**：按照上述"样式选择"规则确定主题 ID；均无则使用默认值 `blue-minimal`
3. **调用渲染脚本**：执行 `python3 {baseDir}/scripts/render.py render article.md --theme blue-minimal -o article.html`，生成带有公众号排版样式的 HTML 文件
4. **交付预览**：将生成的 HTML 文件路径交给用户，用户可在浏览器中预览最终排版效果
5. **微信粘贴**：用户在浏览器中打开 HTML 文件，复制 body 内的内容粘贴到微信公众号编辑器即可保留样式

## 图片占位处理

- `.md` 文件中的 `<!-- IMG: 描述 -->` 占位标记**不阻塞**样式应用——即使未执行配图（模块 E），也可以直接渲染
- 渲染时，占位标记会被替换为 `assets/placeholder.png` 占位图片，同时保留图片描述文字，方便后续识别和替换
- 如果后续需要配图，正确流程是：先执行模块 E 将真实图片 URL 写入 `.md` 文件，再重新执行 `render.py` 渲染
- **不要擅自执行配图**（模块 E 会消耗算力），除非用户明确要求

## 行为规则

- 使用 `render.py` 脚本进行排版渲染，不要手写 HTML/CSS
- 如果用户没有要求公众号样式，**不要主动执行此模块**——`.md` 文章文件本身就是有效的交付物
