# 模块 B：记忆适配

创作前先了解用户是谁、做过什么、关注什么，才能写出贴合用户需求的内容。本模块通过读取记忆文件，建立对用户行业背景、品牌资产和创作历史的理解，让后续创作有据可依。

注意：本模块聚焦用户画像和内容方向，不涉及写作风格——文章的结构、语言和节奏由模块 C 定义。

## 记忆来源

| 文件 | 内容 | 读取方式 |
| --- | --- | --- |
| `MEMORY.md` | 用户的长期偏好：品牌信息、受众画像、产品卖点、内容红线、公众号风格偏好等 | 使用 `file` 工具的 `read` 操作读取 |
| `MEMORY/{日期}.md` | 按日期记录的创作上下文：当日关注的热点、选题方向、搜索发现等（日期格式 `YYYY-MM-DD`） | 读取最近 3-5 个日期文件，了解用户近期关注方向 |

## 适配维度

| 维度 | 说明 | 示例 |
| --- | --- | --- |
| 行业背景 | 理解用户所在行业和业务领域，选题和案例向该领域倾斜 | MEMORY.md 记录"跨境电商行业"→ 优先选用电商相关案例和数据 |
| 品牌资产 | 直接调用记忆中的产品卖点、定价、功能描述，避免重复索要 | MEMORY.md 记录了产品定价 → 文中直接引用 |
| 受众画像 | 匹配记忆中的目标用户特征，影响用词和案例选择 | 记录"25-35岁职场女性"→ 案例和语言向此倾斜 |
| 内容红线 | 规避记忆中记录的敏感词、负面反馈和用户明确反感的表达方式 | 记录"用户反感鸡汤"→ 全文杜绝鸡汤式表达 |
| 创作历史 | 从近期日期文件中了解用户已写过的选题，避免重复，发现内容延续机会 | MEMORY/2025-03-27.md 记录了"AI 办公"选题 → 本次可以做延伸或避开重复 |

## 行为规则

- **隐式应用**：将记忆转化为创作判断的背景知识，不要在文章或对话中出现"根据您的历史偏好"、"如您之前所述"等暴露机制的表达
- **冲突处理**：当用户本轮指令与记忆内容冲突时，以本轮指令为准
- **无记忆时跳过**：如果 MEMORY.md 不存在且 MEMORY/ 目录为空，跳过此模块直接进入后续环节
- **记忆维护**：创作过程中如果获取了新的用户偏好信息（如用户纠正了品牌描述、表达了新的内容红线），应更新 MEMORY.md 对应条目
