# 模块 A：选题与方向确认

创作的第一步是明确"写什么"。在动笔之前，必须从上下文中提取并确认以下要素：

## 必须明确的要素

- **核心选题**：文章围绕什么主题展开
- **目标受众**：这篇文章写给谁看
- **内容目的**：科普教育、产品推广、观点输出、情感共鸣、还是热点借势
- **关键信息点**：文章必须覆盖的核心事实、数据或卖点

## 信息来源优先级

1. **用户当前指令**：用户在本轮对话中明确提出的主题和要求，最高优先级
2. **上游选题输出**：如果用户在之前的对话中已经完成选题挖掘并选定了某个选题，直接继承该选题的标题、机会点和方向作为创作输入
3. **上下文推断**：从对话历史中推断用户的创作意图（如用户分享了一篇竞品文章并说"写一篇类似的"）

## 选题挖掘

当用户需要从零开始寻找选题时，根据场景选择合适的方式。

### 热点趋势发现

适合借势热点创作，发现当日全网热门话题。

1. **获取热点列表**：

   ```bash
   python3 {baseDir}/scripts/writer.py trend
   ```

   返回当日多平台聚合的热点新闻列表，包含排名、标题、来源平台和链接。

2. **筛选高潜力选题**：从热点列表中筛选与用户行业/领域相关的 5-10 条高潜力新闻。

3. **深度解读**：对于最有潜力的选题，搜索更多相关报道，或使用 `web_fetch` 工具读取热点原文链接获取完整背景：

   ```bash
   python3 {baseDir}/scripts/writer.py search --query "热点关键词" --action news
   ```

4. **整合交付**：基于完整信息，提炼并整合出 3-5 个具体、可执行的选题角度。

### 关键词热点总结

适合用户已有明确的关键词方向，想看看当日热点中有没有相关内容可以借势。

1. **关键词热点总结**：

   ```bash
   python3 {baseDir}/scripts/writer.py trend-summary --keyword "关键词"
   ```

   脚本会自动从当日热点新闻中筛选与关键词相关的内容，通过大模型归纳合并为若干"内容点"，每个内容点包含标题、摘要、与关键词的关联说明。

2. **记录兴趣领域**：如果总结结果中存在相关内容点（即 summary 不是"今日未发现..."），说明用户对该领域持续关注，应使用 `file` 工具将总结结果追加写入 `memory/{日期}.md`（日期格式 `YYYY-MM-DD`，如 `memory/2025-03-28.md`），便于后续创作时回溯用户的关注方向。文件格式：

   ```markdown
   ## {关键词} — 热点总结
   
   {summary 内容}
   
   ### 内容点
   - **{point.title}**：{point.description}（关联：{point.relevance}）
   ```

3. **补充搜索**：如果总结结果中内容点较少（不足 3 个），围绕关键词搜索更多实时资讯作为补充：

   ```bash
   python3 {baseDir}/scripts/writer.py search --query "关键词" --action news
   ```

4. **整合交付**：将热点总结的内容点与补充搜索结果结合，提炼出 3-5 个选题角度。

### 通用搜索研究

适合选题方向不依赖热点趋势，或需要研究特定领域的场景。

1. **多角度搜索**：围绕用户主题拆解 2-3 个搜索角度分别搜索：

   ```bash
   # 网络新闻搜索 — 获取最新资讯和行业动态
   python3 {baseDir}/scripts/writer.py search --query "关键词" --action news
   
   # 微信公众号文章搜索 — 了解同领域公众号的内容方向和竞品动态
   python3 {baseDir}/scripts/writer.py search --query "关键词" --action wechat_article
   ```

2. **深度解读**：对搜索结果中信息密度高的链接，使用 `web_fetch` 工具获取原文，提取关键事实、数据和案例。

3. **交叉验证**：对比 news 和 wechat_article 两种来源的结果，识别反复出现的主题和信息差。

4. **整合交付**：基于从多个来源获得的全面理解，构建选题建议。

## 脚本命令速查

| 命令 | 用途 | 关键参数 |
| --- | --- | --- |
| `search` | 统一搜索 | `--query`（必填）、`--action`（news/wechat_article/image，默认 news） |
| `trend` | 获取最新热点新闻 | 无 |
| `trend-summary` | 关键词热点总结 | `--keyword`（必填） |

所有命令支持 `--json` 获取原始 JSON 输出。

**使用建议**：

- `trend`：热点发现的起点，返回多平台聚合的热点榜单。获取列表后需要进一步筛选和深度解读，**严禁**仅凭标题直接创作
- `trend-summary`：当用户有明确的关键词方向时优先使用，它会自动完成"热点获取→相关性筛选→合并归纳"的全流程。如果返回"今日未发现与该关键词明确相关的热点内容"，应改用通用搜索研究的方式
- `search`：将复杂问题分解为集中的、单一主题的搜索，优先进行有针对性的搜索而不是宽泛的搜索

## 选题输出与确认

完成选题挖掘后，以表格形式呈现 3-5 个选题供用户选择确认：

| 序号 | 选题标题 | 机会点 | 推荐方向（可作为文章标题） | 时效性 |
| --- | --- | --- | --- | --- |
| 1 | 核心话题/关键词 | 需求证据 + 现有内容缺陷 + 差异化优势（80-150字） | 2-4 个可直接使用的标题方向 | urgent/short/evergreen/none |

**选题质量要求**：

- **机会点**禁止写成"热度高、关注度大"这类空话，必须说明白：需求证据（数据）+ 现有缺陷（具体问题）+ 你的优势（为什么你能做）
- **推荐方向**必须是可直接作为封面的完整标题，包含具体数字、痛点或反常识点，禁止出现"可以从价格角度写"这类空泛描述
- 各选题之间应有足够差异性，覆盖不同角度
- 如果某个关键词经验证无机会，标注"建议放弃，现有内容已过度饱和"，用其他优质选题补足总数

呈现表格后等待用户选择，用户确认选题后直接进入模块 D 创作环节。

## 行为规则

- 如果上下文中选题方向已经足够清晰，**直接进入创作**，不要反复确认浪费用户时间
- 如果存在关键歧义（如主题模糊、受众不明、多个方向无法取舍），使用 `ask` 向用户提出**最多 2 个**聚焦问题来消除歧义
- 禁止在选题已明确的情况下，仍然输出"我将为您创作一篇关于XX的文章"之类的废话——直接开始写
- 当用户需要选题挖掘时，完成挖掘后以表格形式交付，等待用户确认后再进入创作
- 禁止在回复中使用"工作流A"、"工作流B"等内部术语，直接执行对应操作即可
