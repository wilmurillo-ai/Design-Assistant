# 模块 C：风格遵循

创作前需要确定是否有目标风格可以参考。有明确的风格参照能显著提升文章质量和一致性。

## 风格来源判定

按以下优先级确定风格来源：

1. **长期记忆**：读取 `MEMORY.md` 文件，查看是否已记录用户偏好的公众号创作风格（包含风格报告 URL）。如果存在，直接使用该 URL 进入"风格读取与遵循"环节。
2. **用户本轮指定**：用户在当前对话中明确提到了要模仿的公众号名称或给出了一篇文章链接。
3. **无风格参照**：以上都不存在时，询问用户是否需要模仿某个公众号的创作风格。用户可以提供：
   - 一个公众号名称（如"半佛仙人"）
   - 一篇该作者的文章链接

   如果用户明确表示不需要，使用下方"公众号爆文默认风格"进行创作。

## 公众号爆文默认风格

当没有具体的风格参照时，按以下公众号爆文风格进行创作。这不是通用写作指导，而是针对微信公众号高传播内容的风格定义。

**开头（前 3 句定生死）**：
- 用一个具体场景、反常识事实或扎心提问切入，3 句话内让读者产生"这说的是我"的代入感
- 禁止"随着...的发展"、"众所周知"等空洞开场
- 可以用短故事、数据冲击、或直接抛出结论再展开

**标题公式**：
- 控制在 20 字以内，必须包含核心关键词
- 常用结构：数字+痛点（"3 个让你少走弯路的..."）、反常识（"别再...了"）、身份代入（"月薪 5000 的人都在..."）
- 制造信息差或情绪张力，但不能标题党到与正文脱节

**语言风格**：
- 人称用"你"为主，制造对话感，像朋友聊天而非教科书
- 短句为主，长短交替，连续 3 个以上长句必须用短句打断
- 适度口语化，但不能过于随意——专业感和亲切感并存
- 善用设问和反问制造节奏感（"你有没有想过...？"、"真的是这样吗？"）

**段落节奏**：
- 每段只讲一个点，3-5 行封顶（手机屏幕上超过 5 行就是视觉压迫）
- 每 2-3 段插入一个小标题，帮助跳读
- 关键结论或金句单独成段，制造视觉停顿和记忆锚点
- 刻意制造适合截图分享到朋友圈的"金句段落"

**信息密度**：
- 每段必须有信息增量，读完能获得一个新事实、观点或启发
- 抽象观点必须配具体数据或真实案例支撑
- 删除所有"正确但无用"的废话

**结尾**：
- 不要空洞总结，给读者一个明确的"下一步"：行动号召、思考问题、或引导互动
- 可以用一句金句收尾，适合读者截图分享
- 引导点赞、在看、评论（但要自然，不要生硬求赞）

**配图节奏**：
- 每 500-800 字放置一张配图
- 首图建议用场景图或情绪图，吸引注意力
- 优先真实感照片，避免过于廉价的素材图

**篇幅**：
- 1500-3000 字，根据内容深度灵活调整
- 宁可精炼有力，不要注水拉长

## 风格拆解

当用户提供了公众号名称或文章链接时，通过脚本调用风格拆解能力。拆解过程需要一定时间（通常 1-3 分钟），调用前告知用户正在分析。

**按公众号名称拆解**：

```bash
python3 {baseDir}/scripts/writer.py style-by-name --name "公众号名称"
```

**按文章链接拆解**：

```bash
python3 {baseDir}/scripts/writer.py style-by-url --url "文章链接"
```

脚本会同步等待拆解完成，返回结果中包含：
- `article_analysis_url`：风格分析报告的 URL（核心输出，Markdown 格式）
- `article_list_url`：原始文章列表的 URL（参考用）

**拆解完成后**：
1. 将风格报告 URL 记录到 `MEMORY.md` 中，格式：

   ```markdown
   ## 公众号创作风格偏好
   - 公众号：{account_name}
   - 风格报告：{article_analysis_url}
   ```

2. 这样下次创作时可以直接从 MEMORY.md 读取，不需要重复拆解。

## 风格读取与遵循

拿到风格报告 URL 后，使用 `web_fetch` 工具读取报告全文。报告是一份结构化的《创作者 Writing DNA 分析报告》，包含六个维度：

- **价值观锚点**：核心世界观、立场倾向、终极追求
- **情绪节奏曲线**：开篇钩子、情绪坡度、金句密度
- **语言指纹**：词汇偏好、句式结构、人称代词、语气特征
- **结构与视觉惯性**：排版特征、模块固定性、标题公式
- **读者关系定义**：人设定位、互动引导方式
- **配图风格特征**：配图位置规律、内容类型、视觉风格、图文关系

读取后，将报告内容作为创作框架的最高参照，在模块 D 撰写时严格遵循。

## 行为规则

- 风格报告是创作框架的**最高权威**，不可偏离其结构定义
- 如果风格报告与用户本轮的具体要求冲突（如用户说"这次不要用那个开头模式"），以用户本轮指令为准
- 如果不存在有效的风格参照，使用本文档中的"公众号爆文默认风格"进行创作
- 风格报告中的配图习惯（如配图频率、风格偏好、首图要求）应传递给模块 D（影响占位描述）和模块 E（影响图片风格选择）
- 禁止在回复中向用户暴露"模块C"、"风格拆解"等内部术语，自然地说"我先分析一下这个公众号的写作风格"即可
