# Narrative Master 知识图谱

> 编剧的思维基础设施

---

## 一、实体定义

### 1.1 Character（角色）

```yaml
基础属性:
  name: 角色名
  age: 年龄
  profession: 职业

三层结构:
  mask: 外在表现（给外界看到的）
  self: 内在真实（真实的自己）
  ghost: 核心创伤（塑造现在的根源）

核心动力:
  want: 外在目标（角色追求的）
  need: 内在需求（真正需要的）
  
弧光:
  type: 弧光类型
  from: 起始状态
  to: 目标状态

关系:
  conflicts: 矛盾特征
  arc: 变化轨迹
```

### 1.2 Story（故事）

```yaml
基本属性:
  format: 格式（概念/短片/长片/剧集）
  genre: 类型
  duration: 时长

方法论选择:
  mckee: 价值转变
  snyder: 15节拍
  harmon: 8步圆环

核心要素:
  logline: 一句话概念
  premise: 故事前提
  theme: 主题
```

### 1.3 Scene（场景）

```yaml
标识:
  id: 场景编号
  location: 地点
  time: 时间

价值转变:
  start_value: 开始价值
  end_value: 结束价值

戏剧动作:
  goal: 目标
  obstacle: 阻碍
  gap: 鸿沟
```

## 二、方法论映射

### 2.1 McKee体系

| 概念 | 定义 |
|------|------|
| 价值转变 | 每个场景必须有价值从正到负或从负到正 |
| 鸿沟 | 期望与现实的落差 |
| Ghost | 核心创伤 |

### 2.2 Snyder体系

| 节拍 | 分钟 | 内容 |
|------|------|------|
| Opening | 1 | 开场画面 |
| Theme | 5 | 主题陈述 |
| Catalyst | 12 | 催化剂 |
| Midpoint | 55 | 中点 |
| All Is Lost | 75 | 尽失 |
| Finale | 85-105 | 高潮 |

### 2.3 Harmon体系

```
YOU → NEED → GO → SEARCH → FIND → TAKE → RETURN → CHANGE
```

## 四、决策规则

### 4.1 格式选择

```
输入 → 格式
"1-3分钟"或"概念" → 概念超短片
"5-10分钟"或"短片" → 短片
"90分钟"或"电影" → 长片
"剧集"或"连续剧" → 剧集
```

### 4.2 方法论选择

```
故事类型 → 方法论
商业/类型 → Snyder
成长/冒险 → Harmon
角色/文艺 → McKee
非线性 → 诺兰
意外/反转 → 昆汀
```
