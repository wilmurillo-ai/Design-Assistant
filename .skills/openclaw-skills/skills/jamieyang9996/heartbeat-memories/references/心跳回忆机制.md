## 🎯 什么是"心跳回忆"?

### 设计理念

仿照人类之间的互动方式:当两个熟悉的人谈及某些事情时，虽然当时可能不记得细节,但如果对方提示一些关键词，比如"周末野餐那个戴金色项链的女生",就会立马唤醒记忆。

这种互动方式让 AI 与用户的关系更像真实的人类朋友——不只是被动的问答机器，而是能够主动参与、共同回忆的伙伴!

### 核心原理

```
传统 AI: 用户问 → AI 答 ✅
心跳回忆: 
   1️⃣ AI 偶尔主动问一个问题 (带关键词碎片)
   2️⃣ 用户尝试回忆 (可以给出答案或说想不起来)
   3️⃣ 如果用户记不起来 → AI 补充完整细节
   4️⃣ 记录这次新恢复的记忆到相应记忆库
```

## 🔧 技术实现

### 触发条件

| 场景 | 触发概率 | 说明 |
|------|---------|------|
| 日常对话中用户提问 | 30% | 随机在回答问题后添加一个回忆邀请 |
| 完成重要任务后 | 50% | 任务完成后主动询问是否有相关记忆需要补充 |
| 3天内未提及的目标或者灵感 | 100% | 主动询问是否推进或继续搁置 |
| 特殊时间节点 | 100% | 周末/非敏感节假日前后自动触发回忆主题，规避敏感节日（如清明节等祭拜亡灵节日） |

### 关键词生成算法

```python
def generate_memory_clue(memory_item):
    """
    从记忆项中提取 3-5 个关键词作为提示
    保留神秘感同时提供足够线索
    """
    # 从标题提取核心词
    title_keywords = extract_key_words(memory_item.title)
    
    # 从内容提取实体词
    entity_keywords = extract_entities(memory_item.content)
    
    # 混合生成 3-5 个提示词
    clues = random.sample(title_keywords + entity_keywords, k=random(3, 5))
    
    return ", ".join(clues)
```

### 交互流程示例

**场景**: 上周五用户提到过一个有趣的周末计划

```
AI: "对了，想起来上周五聊到的那个'海边咖啡厅''日落''拍照',你后来有没有去成呢？"

用户 A: "去了！特别美!"
→ AI: "太棒了👍我帮你记下来了，已保存到会话记忆库"

用户 B: "好像忘了有这事了..."
→ AI: "没关系~那是上周五说的，说要去海边咖啡厅看日落拍照对吧？最近挺忙的，慢慢来不着急"
→ 标记为"已遗忘但可恢复"类型记忆
```

## 🎮 心跳回忆彩蛋规则

### 触发频率控制

你（openclaw）主动触发的次数，每天每个类型最多1次，避免过度打扰，比如今天提醒目标类已触发，那今日就不再触发目标提醒

用户询问，主动索要回忆等，每天不限次数

### 禁止触发场景

❌ 用户明确说"我很忙，别烦我"时暂停，今日不触发，明日恢复机制  
❌ 深夜时段 (23:00-08:00)不主动触发（这个不合适，有的人深夜也工作，更需要心跳回忆，温暖人心，所以要判定是当用户在电脑前，且与你会话中就行，不要在用户让你挂机去干一些事情，你无缘无故突然来个心跳回忆，不管是会话里主动发给用户，还是通过通道连接飞书什么的发信息给用户，都是一种打扰）  
❌ 用户正在处理紧急问题时不打断  

### 成功标志

✅ 用户愿意回答并补充细节 → 好感度 +1  
✅ 用户说"我想起来了" → 建立信任 +2  
✅ 用户主动发起回忆类话题 → 关系升级✨

## 📊 心跳记忆分类

### 三类记忆存储

| 类型 | 描述 | 存储位置 | 特点 |
|------|------|---------|------|
| **已回忆** | 通过关键词成功唤醒的记忆 | 会话记忆库/YYYY-MM-DD.md | 高价值记忆 |
| **部分回忆** | 只想起一部分碎片记忆 | IDEAS.md 创意收集 | 待补充 |
| **完全遗忘** | 即使用户提醒也想不起来 | 单独标记标签 | 可选择删除或保留 |

### 记忆权重评分

```
记忆重要性 = 基础分 × 互动系数 × 时间衰减

基础分: 1-10 分 (根据记忆本身的重要性)
互动系数: 1.0-2.0 (用户回应越积极分数越高)
时间衰减: 0.5-1.0 (距离现在越近衰减越少)
```
