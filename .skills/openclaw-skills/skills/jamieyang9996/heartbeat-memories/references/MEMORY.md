## 🧠 记忆系统架构说明

OpenClaw 采用**模块化记忆系统设计**，将不同类型的记忆存储在不同的文件和目录中，便于管理和检索。

### 📁 五大核心记忆库 (2026-03-17 创建 | 2026-03-18 定名 | 2026-03-20 重构)

| # | 记忆库名称 | 存储路径 | 内容类型 | 作用 |
|---|-----------|---------|----------|------|
| 1️⃣ | **目标记忆库** | `memory/目标记忆库/` | GOALS.md | 追踪用户目标（包含已完成/进行中/搁置目标） |
| 2️⃣ | **会话记忆库** | `memory/会话记忆库/` | YYYY-MM-DD.md | 每日会话摘要和上下文 |

## 🔧 日常维护指南

### 每日任务 (Daily)

```bash
# 1. 查看当日会话记录
cat memory/会话记忆库/$(date +%Y-%m-%d).md

# 2. 回顾情感记忆（可选）
cat memory/情感记忆库/DAILY_EMOTIONS.md | tail -20

# 3. 查看今日待办目标进度
grep -A5 "P0\|P1" memory/目标记忆库/GOALS.md

# ⚠️ 注意：因阿里云自动镜像，无需执行每日自动备份命令
# 📅 建议改为每周/每月手动备份一次即可
```

### 每周任务 (Weekly)

```bash
# 1. 周日晚上回顾一周经验
cat memory/经验记忆库/TIPS.md | grep -E "^###|^-"

# 2. 清理临时文件（空文件、日志等）
find . -type f -name "*.log" -mtime +7 -delete
find . -type f -size 0 -delete

# 3. 检查存储空间
df -h /home
```

### 每月任务 (Monthly)

```bash
# 1. 归档旧会话记录（保留最近 3 个月）
mkdir -p reports/archive/$(date +%Y%m)
mv memory/会话记忆库/2026-0[12]-*.md reports/archive/

# 2. 全面备份记忆数据
cp -r memory/ ~/semantic_backup_$(date +%Y%m)/

# 3. 月度记忆回顾报告
python3 scripts/local_memory_system_v2.py report --period month
```

## ⚠️ 重要提醒

### 1. 数据安全
- **定期备份**: 建议每天凌晨 2 点执行 cron 任务自动备份
- **分区监控**: 保持 /home 使用率 <80%，避免磁盘溢出
- **内存注意**: ChromaDB 启动需 100-300MB 内存，低内存环境慎用

### 2. 操作规范
- **不覆盖原则**: 在修改任何记忆文件前，先创建备份副本
- **时间戳格式**: 所有记录必须带精确时间戳 (YYYY-MM-DD HH:MM)
- **来源可溯**: 每条经验/目标都要注明创建时间和触发事件

### 3. 隐私保护
- **敏感信息**: 不要在记忆文件中记录密码、API Key 等敏感数据
- **授权检查**: 在共享上下文中不要泄露 MEMORY.md 中的个人偏好
- **加密存储**: 如有需要，可将私密记忆单独加密保存

## 🔄 RAG优化项目启动 (2026-03-25)

### 🎯 优化目标
通过检索增强生成技术，结合五大记忆库内容，提升OpenClaw回答的准确性、相关性和个性化程度。

### 📋 用户要求整合
| 要求 | 实现方案 | 默认状态 |
|------|----------|----------|
| **Token超支风险** | 限制搜索结果数量 + 去重算法，做成开关 | 默认关闭 |
| **加载速度慢** | 内存缓存机制，做成开关 | 默认关闭 |
| **日志占用磁盘** | 全部留存备份，按自然月自动压缩（精简版） | 自动执行 |
| **不需要** | 自愈能力、多用户支持、自然语言界面、重要性记忆权重、跨设备同步 | 不实现 |
