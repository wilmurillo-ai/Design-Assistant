# 常用 Prompt 模板集合

## 模板 1：基础功能测试

```
请根据需求生成测试用例：
- 格式：Markdown 表格
- 字段：用例 ID、用例名称、前置条件、测试步骤、预期结果、优先级
- 测试类型：功能测试
- 覆盖范围：正常场景、异常场景
- 每个功能点至少 3 个用例
```

## 模板 2：完整测试套件

```
请生成完整的测试用例：
- 格式：Markdown 表格，包含用例 ID、模块、用例名称、前置条件、测试步骤、预期结果、优先级、备注
- 测试类型：功能测试 + 边界测试 + 安全测试 + 性能测试
- 覆盖范围：
  * 正常场景：主要业务流程
  * 异常场景：错误输入、网络异常、数据异常
  * 边界值：最小值、最大值、空值、特殊字符
  * 安全测试：SQL 注入、XSS、权限绕过
- 优先级规则：
  * P0: 核心功能，阻塞发布
  * P1: 重要功能，影响体验
  * P2: 次要功能，可延后修复
- 语言：中文
```

## 模板 3：敏捷测试（用户故事）

```
基于以下用户故事生成验收测试：
- 格式：Given-When-Then 格式
- 结构：
  * Given [前置条件]
  * When [操作]
  * Then [预期结果]
- 覆盖：happy path + edge cases
- 输出：每个场景一个测试用例
```

## 模板 4：API 测试

```
请为 API 接口生成测试用例：
- 格式：表格，包含用例 ID、接口、方法、请求参数、预期状态码、预期响应、优先级
- 测试类型：
  * 正常请求：有效参数
  * 参数验证：必填项、类型、范围
  * 错误处理：无效参数、超时、服务不可用
  * 安全测试：未授权访问、Token 过期
- 覆盖 HTTP 状态码：200、400、401、403、404、500
```

## 模板 5：UI/UX 测试

```
请为界面生成测试用例：
- 格式：表格，包含用例 ID、页面/组件、测试点、操作步骤、预期表现、优先级
- 测试类型：
  * 功能测试：交互、导航、数据展示
  * 兼容性测试：浏览器、分辨率、设备
  * UI 测试：布局、样式、响应式
  * 可用性测试：可访问性、键盘导航
- 覆盖：主流浏览器（Chrome、Firefox、Safari、Edge）
```

## 模板 6：性能测试

```
请生成性能测试用例：
- 格式：表格，包含用例 ID、场景、并发用户数、响应时间要求、吞吐量要求、资源限制
- 测试类型：
  * 负载测试：预期并发量
  * 压力测试：极限并发量
  * 稳定性测试：长时间运行
- 指标：响应时间、TPS、CPU/内存使用率、错误率
```

## 模板 7：BDD 格式

```
请生成 BDD 风格的测试场景：
- 格式：Gherkin 语法（Feature/Scenario/Given-When-Then）
- 要求：
  * 业务语言描述，非技术术语
  * 每个场景独立可执行
  * 覆盖主要业务流程和异常分支
- 输出结构：
  Feature: [功能名称]
    Scenario: [场景名称]
      Given ...
      When ...
      Then ...
```

## 模板 8：JSON 格式（机器可读）

```
请生成 JSON 格式的测试用例：
- 格式：JSON 数组
- 字段：id, name, module, precondition, steps, expected, priority, type
- 示例：
[
  {
    "id": "TC-001",
    "name": "登录成功",
    "module": "auth",
    "precondition": "用户已注册",
    "steps": ["输入邮箱", "输入密码", "点击登录"],
    "expected": "登录成功，跳转首页",
    "priority": "P0",
    "type": "functional"
  }
]
```
