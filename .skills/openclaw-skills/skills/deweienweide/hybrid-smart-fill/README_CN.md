# Hybrid Smart Fill Skill - 使用说明

## 技能概述

`hybrid-smart-fill` 是一个基于混合检索模式（BM25语义检索 + TF-IDF向量检索）的智能模板自动填充技能。它可以高精度地从知识库中匹配字段并自动填充Word文档和Excel表格。

## 安装方法

### 方法1：通过ZIP包安装

1. 下载 `hybrid-smart-fill.zip` 文件
2. 解压到 `~/.workbuddy/skills/` 目录
3. 完成！技能已自动加载

### 方法2：直接复制

1. 复制整个 `hybrid-smart-fill/` 文件夹
2. 粘贴到 `~/.workbuddy/skills/` 目录
3. 重启WorkBuddy使技能生效

## 使用指南

### 基本使用流程

1. **准备知识库**
   - 创建JSON格式的知识库文件
   - 包含公司信息、财务数据、人力数据等
   - 参考文档：`references/kb_format.md`

2. **准备模板**
   - Word文档（.docx）或Excel表格（.xlsx）
   - 待填写字段留空或使用占位符（如"XX基金"）

3. **运行填充脚本**
   ```bash
   cd ~/.workbuddy/skills/hybrid-smart-fill
   python scripts/smart_filler.py
   ```

4. **查看结果**
   - 填充后的文件保存在输出目录
   - 文件名添加"已填写"后缀

### 常见使用场景

#### 场景1：批量银行申请表

**需求**：向5家不同银行提交申请，每份表格需要相似信息（公司名称、法人、联系信息、财务数据等）

**解决方案**：
1. 将所有公司数据编译到一个知识库JSON
2. 准备5家银行的申请模板
3. 运行一次 `smart_filler.py`
4. 自动获得5份填写完整的申请表

**节省时间**：2-3小时 → 5分钟

#### 场景2：年度报告更新

**需求**：年度报告模板需要多年财务数据（2022-2025）

**解决方案**：
1. 将历年Excel文件解析到知识库
2. 系统自动按年份组织数据
3. 模板字段如"2024年总资产"自动匹配
4. 一次运行完成所有年份填充

**准确率**：年份数据100%匹配

#### 场景3：占位符替换

**需求**：模板包含"XX基金"作为公司名称占位符

**解决方案**：
1. 系统自动检测"XX基金"模式
2. 用知识库中的实际公司名称替换
3. 处理文本和表格中的多个出现

**覆盖率**：100%替换率

## 技术特性

### 混合检索系统

本技能使用混合检索方法，结合两种算法：

1. **BM25（最佳匹配25）**：基于词频和文档频率的统计排序
   - 考虑文档长度归一化
   - 惩罚过度常见的术语
   - 评分公式：`IDF × (TF × (k1 + 1)) / (TF + k1 × (1 - b + b × doc_length / avgdl))`

2. **TF-IDF（词频-逆文档频率）**：向量相似度搜索
   - 将文本转换为向量空间
   - 计算查询与文档之间的余弦相似度
   - 超越精确关键词的语义匹配

3. **混合得分**：两种结果的加权融合
   - 公式：`final_score = 0.5 × BM25_score + 0.5 × TF-IDF_score`
   - 平衡精确度（BM25）和语义理解（TF-IDF）

### 匹配策略

系统使用多级匹配策略：

1. **精确匹配**：字段名与知识库键完全匹配
2. **包含匹配**：字段名包含或被知识库键包含
3. **关键词匹配**：多关键词组合匹配
4. **特殊处理**：自动替换占位符（如"XX基金" → "国寿安保基金"）

## 知识库格式

### Excel内容格式（文本化）

```
=== Sheet: SheetName ===
A1[Header1] | A2[Value1] | B1[Header2] | B2[Value2]
```

## 故障排查

### 问题1：脚本未找到

**错误**：`ModuleNotFoundError: No module named 'docx'`

**解决方案**：安装所需依赖
```bash
pip install python-docx openpyxl
```

### 问题2：没有字段被填充

**原因**：知识库格式不兼容

**检查项**：
- JSON文件为UTF-8编码
- Excel内容使用`A1[Value]`格式
- `smart_filler.py`中的文件路径正确

### 问题3：填充了错误的值

**原因**：字段名歧义

**解决方案**：
- 在模板中使用更具体的字段名
- 调整代码中的检索权重
- 检查知识库是否有正确的值

## 性能优化建议

1. **组织知识库**：按逻辑分组相关字段
2. **清理模板**：删除可能导致错误匹配的不必要文本
3. **使用具体名称**："公司法人代表"比"代表"更好
4. **先测试**：批量处理前先在一个模板上测试
