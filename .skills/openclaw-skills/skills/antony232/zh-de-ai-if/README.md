# zh_de_ai_ify

中文去 AI 味写作技能。

它用于把中文文本中常见的 AI 腔、官话、模板味、企业黑话和空泛表达去掉，让文字更自然、更具体、更像真人写出来。

## 文件结构

```text
zh_de_ai_ify/
  SKILL.md
  README.md
  examples/
    before_after.md
  resources/
    patterns.md
```

## 适合处理

- AI 生成的中文初稿
- 公众号文章
- 小红书 / 知乎 / 博客文案
- 邮件草稿
- 产品介绍
- 演讲稿
- 项目说明
- 自我介绍
- 总结材料中需要“降官腔”的部分

## 不适合强行处理

- 法律合同
- 学术论文
- 政府公文
- 医疗、金融、合规文本
- 必须保持严格正式语体的材料

这些文本可以轻度润色，但不要强行改得太口语。

## 核心能力

这个 skill 会重点处理：

1. 宏大背景开头  
   例如：“在当今快速发展的时代……”

2. 正确废话  
   例如：“具有重要意义”“提供有力支撑”“推动高质量发展”

3. 企业黑话  
   例如：“赋能”“闭环”“抓手”“沉淀”“颗粒度”“打通链路”

4. 机械结构  
   例如：“首先、其次、最后”“一方面、另一方面”

5. 过度拔高  
   例如：“开启新时代”“产生深远影响”“标志着进入新阶段”

6. AI 格式痕迹  
   例如：滥用粗体、列表、emoji、破折号和不必要表格

7. 伪权威表达  
   例如：“多家媒体报道”“受到业界广泛关注”但不给来源

## 改写强度

### mild

适合正式文本。

- 轻度删套话
- 替换明显黑话
- 基本保留原结构

### standard

默认模式。

- 去掉明显 AI 腔
- 打散机械结构
- 改成自然中文

### aggressive

适合博客、社媒、公众号和个人表达。

- 大幅删官话
- 语气更直接
- 允许重写句子结构
- 更明显地改成人话

## 设计原则

这个 skill 的重点不是把文字改得更“高级”，而是让文字更可信、更具体、更自然。

不要虚构：

- 数据
- 案例
- 引用
- 媒体报道
- 行业评价

如果原文缺少依据，应标记出来，而不是替用户补一个假的。

## 示例

原文：

```text
在当前人工智能技术快速发展的背景下，企业应积极拥抱智能化转型，通过多维度能力建设和体系化运营，持续赋能业务增长。
```

改写：

```text
企业用 AI，先别谈全面转型。先找最耗人的流程，比如客服回复、报表整理、合同初审，一个个试。
```

更多示例见：`examples/before_after.md`
