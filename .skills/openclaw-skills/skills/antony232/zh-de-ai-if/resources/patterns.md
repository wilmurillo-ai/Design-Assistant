# 中文去 AI 味规则词表

供 `zh_de_ai_ify` skill 按需读取。主 `SKILL.md` 保持精简，这里放更完整的中文 AI 味识别清单、替换表和处理建议。

---

## 1. 套话 / 空话

重点警惕：

- 在当今快速发展的时代
- 随着科技不断进步
- 在数字化浪潮席卷全球的背景下
- 近年来，随着人工智能技术的飞速发展
- 值得注意的是
- 不难发现
- 可以看出
- 从某种意义上讲
- 在一定程度上
- 不可否认的是
- 事实上
- 显而易见
- 本文将从以下几个方面展开
- 具有重要意义
- 发挥重要作用
- 提供有力支撑
- 注入新动能
- 推动高质量发展
- 具有广阔前景
- 带来新的机遇和挑战
- 促进可持续发展
- 提升核心竞争力
- 实现价值最大化
- 构建完整生态
- 打造闭环体系

处理原则：

- 没信息量就删
- 能直接说就直接说
- 能具体就别抽象
- 有数据写数据，有例子写例子，没有就标记缺口

## 2. 企业黑话替换表

| AI / 企业黑话 | 建议替换 |
|---|---|
| 赋能 | 帮助、支持、让……更容易 |
| 抓手 | 重点、办法、切入口 |
| 闭环 | 走完整个流程、把事情做完 |
| 沉淀 | 积累、记录、整理出来 |
| 拉齐 / 对齐 | 统一、说清楚 |
| 落地 | 执行、真正用起来 |
| 提效 | 提高效率、省时间 |
| 优化 | 改进、调好 |
| 体系化 | 系统整理 |
| 方法论 | 方法、经验 |
| 颗粒度 | 细节程度 |
| 打通 | 连接、整合 |
| 链路 | 流程、路径 |
| 场景化 | 放到具体场景里 |
| 价值闭环 | 具体收益、实际结果 |
| 数智化 | 数字化、自动化，或直接说明技术内容 |

示例：

```text
该方案通过多维能力赋能业务增长，形成完整价值闭环。
```

改成：

```text
这个方案主要解决两件事：减少人工录入，自动生成业务报表。
```

## 3. 模糊数量词

减少：

- 多个
- 多种
- 各类
- 诸多
- 一系列
- 全方位
- 多维度
- 系统性
- 深度
- 持续
- 显著
- 有效
- 充分

处理方式：

- 能写数字就写数字
- 能举例就举例
- 不知道就别装知道

示例：

```text
该系统支持多种业务场景。
```

改成：

```text
该系统目前支持客服问答、订单查询和报表生成。
```

如果不知道具体场景：

```text
该系统支持哪些业务场景，需要补充具体例子。
```

## 4. 模板化结构

警惕：

- 首先、其次、再次、最后
- 一方面、另一方面
- 不仅……而且……
- 更重要的是
- 总而言之
- 综上所述
- 那么，应该如何……呢？答案是……

处理方式：

- 有真实逻辑顺序才保留
- 没必要就打散
- 不强行凑三点
- 直接说结论

## 5. 过度拔高

警惕：

- 具有里程碑意义
- 产生深远影响
- 奠定坚实基础
- 成为重要标志
- 推动行业变革
- 开启新时代
- 标志着进入新阶段
- 对某领域产生重要影响
- 国内首创
- 世界一流
- 行业领先
- 广受认可
- 深受用户喜爱
- 市场反响热烈

处理方式：

- 有证据就保留并补依据
- 没证据就降调或删除
- 写具体变化，不写宏大判断

## 6. AI 格式痕迹

参考中文维基关于 AI 生成文特征的观察，重点警惕：

### 滥用粗体

AI 常在不必要的地方加粗，像说明书、营销稿或 PPT。

处理：

- 删除大部分不必要粗体
- 一段最多保留一个真正重点
- 不要把列表标题全部加粗

### 滥用列表

AI 常把简单内容拆成工整列表，尤其是“标题 + 解释”的格式。

处理：

- 内容适合散文就改成自然段
- 列表确实更清楚才保留

### 滥用 emoji

常见：✅ 🚀 💡 🔥 📌 🎯 🌟

处理：

- 正式文本中删除
- 社媒文案中少量保留
- 不让 emoji 代替逻辑

### 滥用破折号

示例：

```text
AI 不只是工具——它是推动未来变革的重要力量。
```

改成：

```text
AI 是工具。它能不能带来变化，取决于用在哪些具体任务上。
```

### 不恰当表格

保留表格的条件：

- 有多组数据对比
- 有参数、价格、时间、版本
- 表格确实比自然段清楚

改成自然段的情况：

- 只是概念解释
- 只是优缺点泛泛而谈
- 每格里都是完整句子
- 表格比正文更难读

## 7. 内容真实性检查

必须检查：

- 链接是否能打开
- 来源是否真的支持原文说法
- 新闻、论文、报告是否真实存在
- DOI / ISBN 是否有效
- 引用时间是否合理
- 是否把 A 来源的观点错归给 B 来源

遇到这些表达要追问来源：

- 多家媒体报道
- 引发广泛关注
- 受到业界关注
- 获得市场认可
- 在社交平台引发热议
- 被认为是行业代表
- 具有一定知名度
- 具有较高关注度

如果无法确认：

```text
[需要核实来源]
```

不要编。
