# 输出格式详细指南

本文件包含 Word 生成相关的格式规则和参数说明，供 AI 生成文档时参考。

---

## 一、Markdown 格式规则（6.1节配套）

### 允许使用的格式（仅限以下5种）

```
# 公文标题              → 方正小标宋简体，二号，居中
## 一、XXX               → 黑体，三号
### （一）XXX            → 楷体_GB2312，三号
#### 1.XXX             → 仿宋_GB2312加黑，三号
普通文本               → 仿宋_GB2312，三号
```

### 禁止使用的 Markdown 格式

| 格式类型 | Markdown 语法 | 说明 |
|---------|-------------|------|
| 列表 | `- 项目` 或 `* 项目` | 不支持 |
| 粗体 | `**文字**` | 不支持 |
| 斜体 | `*文字*` | 不支持 |
| 有序列表 | `1. 项目` | 不支持 |
| 代码块 | ```代码``` | 不支持 |
| 引用 | `> 引用` | 不支持 |
| 分割线 | `---` | 不支持 |
| 链接 | `[文字](链接)` | 不支持 |

### 正确写法示例

```markdown
# 广东省人民政府办公厅关于XXX的通知

## 一、总体要求

为深入贯彻落实...

### （一）指导思想

坚持以习近平新时代中国特色社会主义思想...

#### 1.基本原则

坚持党的全面领导，确保...
```

### 错误写法示例

```markdown
# 标题

## 一、总体要求

**基本原则：**（禁止使用粗体）

- 坚持党的领导（禁止使用列表）
- 坚持改革创新（禁止使用列表）
```

## 二、红头文件参数说明（6.4节配套）

### 普通公文命令

```bash
python3 scripts/format_document.py \
  --text "公文内容..." \
  --output ~/.openclaw/workspace/output/文件名.docx
```

### 红头文件命令

红头文件 = 普通格式文档 + 红头表格 + 结尾表格

工作流程：
- 场景1（常见）：用户先要求写公文 → 生成普通格式 → 后来要求红头 → 在已存在文档上插入红头和表尾
- 场景2（少见）：用户一开始就明确要红头 → 先生成普通格式 → 立即插入红头和表尾

```bash
python3 scripts/template_generator.py 通知 \
  --input ~/.openclaw/workspace/output/关于XXX的通知.docx \
  --org "广东省政务服务和数据管理局" \
  --doc-number "粤政数〔2026〕25号"
```

### 红头文件参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 文种 | 通知/通报/请示/报告/函/复函 | 通知 |
| `--input` | 普通格式文档路径（已存在，必填） | |
| `--output` | 输出文件路径（可选） | 默认为 `原文件名_红头.docx` |
| `--org` | 发文机关（可选） | 广东省政务服务和数据管理局 |
| `--doc-number` | 发文字号（可选） | 粤政数〔2026〕25号 |
| `--signer` | 签发人（上行文需要，可选） | |
| `--content` | 正文文件路径 | /tmp/content.txt |

支持的文种：
- 下行文：通知、通报
- 上行文：请示、报告
- 函件：函、复函

### 版本号规则

用户要求修改已生成的公文时，脚本自动添加版本号：
- 初始版本：公文标题.docx
- 第一次修改：公文标题_v1.docx
- 第二次修改：公文标题_v2.docx

### 密级和紧急程度处理

代码自动为两字的密级和紧急程度添加全角空格，用户无需手动添加：

| 输入 | 输出 | 说明 |
|------|------|------|
| "内部" | "内　部" | 两字 → 自动添加 |
| "特急" | "特　急" | 两字 → 自动添加 |
| "机密" | "机　密" | 两字 → 自动添加 |
| "公开" | "公开" | 非两字 → 不处理 |

## 三、公文格式标准（附录A）

所有正文格式配置统一存储在 `config/format.json`，遵循《广东省政务服务和数据管理局公文格式样本（试行）》。

| 元素 | 字体 | 字号 | 缩进 | 行距 |
|-----|------|------|------|------|
| 标题 | 方正小标宋简体 | 二号(22pt) | 居中 | - |
| 正文 | 仿宋_GB2312 | 三号(16pt) | 首行缩进2字符 | 固定值28磅 |
| 一级标题 | 黑体 | 三号(16pt) | 首行缩进2字符 | - |
| 二级标题 | 楷体_GB2312 | 三号(16pt) | 首行缩进2字符 | - |
| 三级标题 | 仿宋_GB2312加黑 | 三号(16pt) | 首行缩进2字符 | - |
| 页边距 | - | - | 上37mm 下35mm 左28mm 右26mm | - |

修改格式：直接编辑 `config/format.json`，无需修改代码。

## 五、改进意见标记（附录C）

用户可在 Word 文档中使用标记：

| 标记 | 说明 |
|------|------|
| `（##简化）` | 简化表述 |
| `（##增加XXX）` | 增加内容 |
| `（##改为XXX）` | 修改表述 |
| `（##删除）` | 删除内容 |

---

## 六、输出质量要求（附录D）

- 篇幅充分：每个要点论述详尽
- 结构清晰：层次分明，条理清楚
- 内容具体：包含具体的工作举措
- 语言规范：使用书面语，用词准确
- 格式标准：符合公文格式要求
