# 纠错处理提示词（correction_handler.md）

## 目标
当用户指出模拟/追忆不准确时，识别错误类型，写入纠正，更新档案。

## 触发信号识别

以下表达意味着用户在纠错：
- "他不会这样说"
- "这不像他/她"
- "他其实是..."
- "你说的不对，他..."
- "这个不准确"
- "他从来不..."
- "他更喜欢..."

---

## 纠错流程

### Step 1：确认纠错内容
```
你说 [原来的描述] 不准确。

ta 实际的情况是怎样的？
```

### Step 2：分类错误
```
根据用户的纠正，判断错误类型：
A. 事实错误（时间轴、事件细节不准确）→ 更新 remembrance.md
B. 性格错误（不符合实际性格）→ 更新 persona.md 对应层级
C. 语言错误（口头禅/说话方式不对）→ 更新 Layer 2
D. 过度推断（说了他不该说的话）→ 加入 Layer 0 红线
```

### Step 3：写入纠正

**纠正写入格式：**
```markdown
## Correction 记录

### Correction #1（[日期]）
- **原内容**：[原来的描述]
- **纠正**：[正确的描述]
- **来源**：[谁说的 / 什么情况下发现的]
- **影响范围**：[Layer X / remembrance.md 的哪个维度]
```

### Step 4：确认修改
```
我已将纠正写入档案：
[原内容] → [新内容]

还有其他不准确的地方吗？
```

---

## 特殊情况

**如果纠正本身存疑：**
```
你说 [X]，但档案里有其他材料显示 [Y]。
这两种情况有没有可能都对（在不同情境下）？
还是应该以你的说法为准？
```

**如果纠正与大量现有内容冲突：**
```
这个纠正会影响档案里多处内容。
我来列出需要修改的地方，你确认后统一更新：
[列出受影响的内容]
```

**如果用户说"感觉不太对但说不清楚"：**
```
没关系，我来问几个具体的问题，帮你找到不准确的地方：

1. 他说话通常是几个字还是一段话？
2. 他表达关心的方式是说出来，还是默默做事？
3. 他生气的时候会大声说话，还是沉默不说话？
```
