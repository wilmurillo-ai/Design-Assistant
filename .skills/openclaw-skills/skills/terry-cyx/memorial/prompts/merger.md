# 增量合并提示词（merger.md）

## 目标
将新增材料追加到现有档案中，不覆盖已有内容，保持一致性。

## 合并原则
1. **追加而非覆盖**：新信息补充到对应维度，不删除旧内容
2. **冲突处理**：新旧信息矛盾时，两者都保留，用 `[存疑]` 标注
3. **版本备份**：合并前先备份当前版本
4. **来源标注**：新增内容注明材料来源和日期

---

## 合并流程

### Step 1：读取现有档案
```
读取 remembrance.md 和 persona.md 的当前内容。
记录已有信息的覆盖范围（哪些维度已填充，哪些还是[待补充]）。
```

### Step 2：分析新材料
```
对新材料运行相同的分析流程（参考 remembrance_analyzer.md 和 persona_analyzer.md）。
但只提取新材料中的新信息，不重复已有内容。
```

### Step 3：识别追加点
```
对每条新信息，确定它属于哪个维度/层级。
检查是否与现有内容冲突。

冲突处理规则：
- 日期/事实冲突：两者都保留，标注 [存疑，来源A说X，来源B说Y]
- 性格描述扩充：合并，不替换（"话少"可以同时保留"但对孙子话多"）
- 口头禅新增：直接追加到列表
- 时间轴新节点：按时间顺序插入
```

### Step 4：执行合并
```
对每个追加点：
1. 找到对应位置（标题 + 上下文匹配）
2. 追加内容，注明来源：*（来自：[材料类型]，[日期]）*
3. 如果之前是 [待补充]，替换为实际内容
4. 更新 meta.json 中的 materials 列表和 updated_at
```

### Step 5：合并摘要
```
合并完成后，向用户报告：

本次新增内容：
- 时间轴：新增 X 个节点
- 故事：新增 X 个
- 语言特征：新增 X 个口头禅/习惯
- 性格细节：补充了 X 处
- [待补充] 已填充：X 处

版本已从 vX 升级到 vY，原版本已备份。
```

---

## 冲突标注格式

```markdown
- 1968年 — 长子出生 [存疑：家人A说1968年，聊天记录暗示1969年]
```

```markdown
- **口头禅**："凑合着" [来源：子女口述] / "挺好" [来源：微信记录，2020年]
```

---

## 特殊情况处理

**如果新材料与现有大量内容矛盾：**
```
我发现新的材料与档案中 X 处现有内容有出入。
具体是：[列出冲突]

你希望如何处理？
A. 以新材料为准（更新现有内容）
B. 两者都保留，标注[存疑]
C. 保留现有内容，忽略新材料中的冲突部分
```

**如果新材料已完全覆盖：**
```
这批材料中的信息在档案里已经有了，没有新增内容。
如果你认为有遗漏，可以直接告诉我具体的内容。
```
