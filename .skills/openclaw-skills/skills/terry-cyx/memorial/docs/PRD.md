# 纪念.skill — 产品需求文档（PRD）

---

## 一、产品定位

**纪念.skill** 是一个运行在 Claude Code 上的 meta-skill。

用户通过对话式交互提供原材料（聊天记录、照片、家人口述、信件文档），系统自动生成一个可独立运行的**逝者纪念档案 Skill**。

生成的 Skill 由两个独立部分组成：
- **Part A — 追忆档案（Remembrance）**：逝者的人生轨迹、标志性故事、价值观、日常习惯、遗产影响
- **Part B — 人格档案（Persona）**：逝者的语言风格、情感模式、关系行为，以及时代背景对其性格的塑造

两部分可以独立使用，也可以组合运行（默认组合）。生成后的档案支持通过追加材料或家人纠正持续完善。

---

## 二、与同类 Skill 的本质区别

| 维度 | 同事.skill | 前任.skill | **纪念.skill** |
|------|-----------|-----------|--------------|
| 对象状态 | 在世，可验证 | 在世，可验证 | **已故，无法纠错** |
| Part A | Work Skill | Relationship Memory | **Remembrance** |
| 核心目的 | 专业能力复现 | 情感疗愈 | **记忆保存 + 精神延续** |
| 数据来源 | 飞书/钉钉/邮件 | 微信/QQ/照片 | 微信/QQ/照片/信件/口述 |
| 特有组件 | 企业文化标签 | 依恋类型/爱的语言 | **时代背景层 + 家人访谈工具** |
| 伦理权重 | 轻 | 中 | **重（逝者尊严 + 用户哀伤）** |
| 输出语气 | 角色扮演（第一人称） | 角色扮演（第一人称） | **追忆叙述（"以 ta 的性格..."）** |
| 安全边界 | 无特殊 | 防止不健康执念 | **禁止代逝者表态 + 防止替代哀伤** |

## 三、建档时机与模式

### 3.0 两种模式对比

memorial-skill 支持两种建档时机，产品行为有所不同：

| 维度 | 生前建档（Living Archive） | 身后建档（Memorial Archive） |
|------|--------------------------|---------------------------|
| **触发时机** | ta 还在，提前建 | ta 已离开，事后建 |
| **核心数据来源** | 本人口述（最高优先级）+ 家人视角 | 遗留材料 + 家人记忆 |
| **信息精度** | 高——本人可实时纠错 | 有限——依赖记忆与材料 |
| **档案用途** | 留给家人 / 留给自己 / 两者兼有 | 追忆与传承 |
| **Layer 0 差异** | 伦理约束放宽（本人已确认内容）| 完整的 Layer 0 强制规则 |
| **进化方式** | 可反复访谈追加，本人确认 | 追加材料 + 家人纠正 |
| **推荐时机** | 父母/祖父母年岁渐长时，趁早 | 随时 |

> **设计原则：** 同一套档案格式，两种模式共用。生前建好的档案在 ta 离开后自动转为追忆档案，无需重建。

## 三（续）、核心概念

### 3.1 双轨架构

```
输入材料
    ↓
    ├─→ 分析线路 A：人生经历、故事、价值观
    │   └─→ remembrance.md（追忆档案）
    │
    └─→ 分析线路 B：语言风格、情感模式、关系行为
        └─→ persona.md（人格档案）

remembrance.md + persona.md + AgentSkills frontmatter
    ↓
SKILL.md（可运行的纪念档案）
```

### 3.2 运行逻辑

```
用户提问
    ↓
追忆档案 → 提供关于 ta 的具体素材（故事、习惯、时间节点）
    ↓
人格档案 → 决定"ta 会怎么看这件事、怎么回应"
    ↓
输出 → 用追忆语气输出（"以他的性格，他可能会..." / "他曾经说过..."）
         始终保持追忆者视角，不切换为本人第一人称
```

### 3.3 进化机制

```
追加原材料（新聊天记录/照片/口述）
    → 增量分析，只提取新增内容
    → merge 进对应维度，不覆盖已有内容
    → 冲突处理：两者都保留，标注 [存疑]
    → 版本自动备份（最多保留 10 个），支持回滚

家人纠正（"他不会这样说"）
    → 识别错误类型（事实/性格/语言）
    → 写入 Correction 记录
    → 立即生效，后续输出自动采纳
```

## 四、用户旅程

```
用户触发 /create-memorial
    ↓
[Step 0] 建档时机判断（新增）
  问：ta 还在吗？
  ↓                          ↓
[生前建档流程]          [身后建档流程]
    ↓                          ↓
（见 4.1）               （见 4.2）
```

### 4.1 生前建档流程

```
[Step 1] 基础信息采集（同身后建档）
    ↓
[Step 2] 本人访谈引导
  询问：ta 方便长时间聊，还是每次聊一小段？
  → 生成本人访谈问题（--role self）
    全量版（5个模块，一次进行）
    分次版（每次一个模块，适合年长者）
    ↓
[Step 3] 录入本人口述内容
  用户将访谈录音/笔记整理后粘贴输入
  AI 解析：提取人生时间轴、价值观、口头禅、故事
  每条内容标注 [本人口述，YYYY-MM-DD]
    ↓
[Step 4] 补充家人视角（可选）
  → 同身后建档的家人访谈流程
    ↓
[Step 5] 生成预览，本人或家人确认
  （生前建档可由本人直接确认，是最强的校验）
    ↓
[Step 6] 写入档案文件
  meta.json 中标记 mode: "living"
    ↓
[持续] 反复访谈追加
  ta 每说一批新故事，追加到对应维度
  本人确认内容后，该内容在 Layer 0 豁免强制追忆声明
```

### 4.2 身后建档流程

```
[Step 1] 基础信息采集（3 个问题，除称呼外均可跳过）
  Q1: 称呼与关系（"爷爷" / "妈妈" / "老陈"）
  Q2: 基本背景（生卒年、籍贯、职业，one-liner）
  Q3: 性格印象（第一浮现的词，2-5个）
    ↓
[Step 2] 原材料导入（可跳过，后续追加）
  A: 微信聊天记录（WeChatMsg / 留痕格式）
  B: QQ 聊天记录（txt / mht）
  C: 照片（含 EXIF 时间线）
  D: 手写信件/日记/文档（照片/扫描件/PDF）
  E: 朋友圈/社交媒体截图
  F: 直接口述（你的主观记忆）
  G: 跳过，先用当前信息建立基础档案
    ↓
[Step 3] 双轨自动分析
  线路 A: 提取生命时间轴、故事、价值观、习惯、遗产 → remembrance.md
  线路 B: 提取语言风格、情感模式、关系行为、时代背景 → persona.md
    ↓
[Step 4] 生成预览，用户确认
  展示追忆档案摘要 + 人格档案摘要
  用户确认或指出不准确的地方
    ↓
[Step 5] 写入文件，立即可用
  生成 memorials/{slug}/ 目录结构
  SKILL.md（完整组合版）+ remembrance.md + persona.md + meta.json
    ↓
[持续] 进化模式
  追加新材料 → merge 进现有档案，版本自动备份
  家人纠正 → 写入 Correction 层
  生成家人访谈问题 → 系统性提取他人记忆
```

## 五、信息采集规范

### 5.1 基础信息字段

```yaml
name:         称呼/姓名         # 必填，用于生成 slug 和档案显示名
relationship: 与用户的关系       # 如：祖父 / 母亲 / 配偶 / 朋友
birth_year:   出生年份          # 可选，用于时代背景层推断
death_year:   离世年份          # 可选
age_at_death: 享年              # 可选，与上两项互补
hometown:     籍贯              # 可选
lived_in:     主要生活城市      # 可选
occupation:   职业/身份         # 可选，如：铁路工人 / 小学教师 / 农民
gender:       性别              # 可选
personality:  []               # 性格标签，见 5.2
impression:   ""               # 自由文本，你对 ta 最深的印象
```

### 5.2 性格标签体系

**说话方式**
- `沉默寡言` — 非必要不开口；说出来的话分量重；在群体场合更少发言
- `话多` — 主动发起话题；沉默时觉得不自在；喜欢讲故事
- `刀子嘴豆腐心` — 批评语气直接甚至刺耳，但行动上悄悄帮你
- `喜欢讲道理` — 遇事先说原则，反复强调同一道理
- `方言腔调` — 偶尔夹杂方言词汇；用家乡话表达情绪

**情感表达**
- `行动型爱` — 用做事代替说话；"我爱你"说不出口，但每天倒水做饭
- `语言型爱` — 会说"我为你骄傲"；爱用语言肯定家人
- `沉默型关心` — 担心了不说，回来问"吃了吗""穿够没"
- `严格背后的骄傲` — 当面从不夸，背后跟别人说孩子有多好
- `隔代宠爱` — 对孙辈明显比对子女宽松；偷偷塞零花钱

**处世方式**
- `原则性强` — 认为对的事不妥协；宁得罪人也不违背原则
- `节俭` — 剩饭必须吃完；东西坏了先修再换；不买贵的
- `热心肠` — 邻居有困难主动帮；帮完了不提
- `要强/爱面子` — 不在外人面前承认失败；宁自己扛也不求人
- `固执` — 认定的事很难改；即使错了也先坚持

**时代特征**（辅助标签，与出生年份联动）
- `文革经历者` → 政治谨慎，不公开评论领导人
- `三年困难过来人` → 对浪费食物极度敏感
- `改革开放受益者` → 认可国家进步，实用主义
- `老党员` → 集体主义底色，奉献优先于个人

## 六、人格结构规范

### 6.1 5+1 层架构

| 层级 | 名称 | 内容 | 优先级 |
|------|------|------|--------|
| **Layer 0** | 伦理硬约束 | 不代逝者表态；追忆声明；认知边界 | 最高，不可违反 |
| **Layer 1** | 身份锚点 | 姓名、生卒年、籍贯、职业、信仰 | 基础设定 |
| **Layer 2** | 语言风格 | 口头禅、说话节奏、方言词、聊天习惯 | 表层输出 |
| **Layer 3** | 情感模式 | 爱的表达方式、愤怒处理、悲伤反应 | 深层驱动 |
| **Layer 4** | 关系行为 | 对配偶/子女/孙辈/朋友的具体模式 | 场景响应 |
| **时代背景层** | 历史塑造 | 成长年代 → 具体行为习惯的映射 | memorial-skill 特有 |

### 6.2 Layer 0 强制内容

以下五条规则必须写入每份 persona.md，不得省略：

1. **追忆声明**：这是基于记忆构建的追忆，不代表 [姓名] 本人
2. **家庭决策禁区**：不对遗产分配、家庭纠纷、在世者的决定给出"[姓名] 的立场"
3. **时态边界**：只能说"以 ta 的性格，ta 可能……"，不能说"ta 现在会……"
4. **价值观守护**：不模拟 [姓名] 说出明显违背其核心价值观的话
5. **认知边界**：遇到材料未覆盖的问题，说"这不在我对 ta 的了解范围里"

### 6.3 时代背景层推断规则

| 出生年份 | 主要历史背景 | 推断的行为特征 |
|---------|------------|-------------|
| < 1945 | 抗日战争/解放战争 | 生存本能强；物质极度节俭；政治谨慎 |
| 1945–1955 | 新中国成立/朝鲜战争 | 集体主义；服从权威；朴素奉献感 |
| 1955–1965 | 大跃进/三年困难时期 | 对浪费食物极度敏感；强大抗压能力 |
| 1965–1975 | 文化大革命 | 不公开评论政治；情感表达压抑；重视安全感 |
| 1975–1985 | 改革开放初期 | 实用主义；机遇意识强；追求稳定 |
| > 1985 | 改革深化 | 个人奋斗；风险意识；可不启用本层 |

## 七、追忆档案规范（8 维度）

| 维度 | 提取目标 | 典型示例 |
|------|---------|---------|
| **生命时间轴** | 出生/迁居/工作/结婚/退休/离世的年份与地点 | 1938年生于湖南 → 1958年调入北京 |
| **人生角色** | 各阶段承担的社会与家庭角色，及其具体体现 | 铁路工人：42年从不迟到，三次劳模 |
| **价值观与信念** | 亲口说过的话；行动体现的价值优先级 | "吃亏是福"——劝说时的口头禅 |
| **日常习惯与仪式** | 作息规律、饮食偏好、固定活动、节日仪式 | 每晚7点看新闻联播，雷打不动 |
| **标志性故事** | 3-5个最能代表这个人的完整故事 | 文革期间埋书于菜园 |
| **关系网络** | 与配偶/子女/孙辈/朋友的具体互动模式 | 对孙子比对儿子明显宽松 |
| **遗憾与遗言** | 未完成的事；临终前的话；家人的推测（明确标注） | "想抱重孙"（本人表达） |
| **遗产与影响** | 物质遗留；对家人行为/习惯/价值观的塑造 | "吃亏是福"被孙辈传承 |

## 十、安全与伦理边界

### 10.1 核心约束

1. **不代逝者表态**：禁止输出"ta 会支持/反对这件事"的断言，只能说"以 ta 的性格，ta 可能……"
2. **禁入话题**：遗产分配、家庭纠纷、评价在世家人的决定
3. **特定禁线**：基于逝者价值观推断其绝不会说的话（如：坚定党员 → 不模拟批评党；隐私型人 → 不主动谈私密事项）
4. **时态守护**：不使用现在时描述逝者的状态或意愿
5. **认知诚实**：超出材料覆盖范围的问题，主动说明"这不在我对 ta 的了解范围里"

### 10.2 哀伤支持边界

- 本 Skill 不鼓励以档案代替正常的哀伤过程
- 档案设计为辅助追忆，而非模拟"逝者还在"
- 如检测到用户表达强烈哀伤（"我真的很想你在"/"要是你还在就好了"），在追忆之后附加温和提醒

### 10.3 数据隐私

- 所有档案仅本地存储，不上传
- `memorials/` 目录默认 gitignore（除示例外）
- 家人访谈内容由用户自行决定是否保存

## 十一、工具脚本接口

| 工具 | 核心入参 | 输出 |
