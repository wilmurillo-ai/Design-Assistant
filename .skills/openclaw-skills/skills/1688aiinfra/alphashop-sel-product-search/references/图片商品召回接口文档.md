## 接口概述

| 项目 | 内容 |
|------|------|
| 接口名称 | 图片商品召回API |
| 接口功能 | 根据用户上传的图片URL，通过AI图像识别和向量检索技术，返回全平台同款/相似商品列表及其详细信息 |
| 请求方式 | POST (HSF泛化调用) |
| Content-Type | application/json |
| 响应耗时 | 10秒内 |
| Endpoint | https://api.alphashop.cn/opp.selection.image.search/1.0 |

## 入参说明

### 请求参数结构

```json
{
  "imageUrl": "https://img.alicdn.com/imgextra/xxx.jpg",
  "cropRegion": "0,0,200,200",
  "platform": "amazon",
  "region": "US",
  "recallNum": 10,
  "similarityValue": "0.5",
  "userId": "12345678"
}
```

### 参数列表

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| imageUrl | String | 是 | 商品图片URL，必须是HTTP/HTTPS协议的完整URL | "https://img.alicdn.com/imgextra/xxx.jpg" |
| cropRegion | String | 否 | 图片裁剪区域，格式为"x,y,width,height"，用于指定图片中的特定区域进行识别 | "0,0,200,200" |
| platform | String | 是 | 目标平台，枚举值：amazon/tiktok | "amazon" |
| region | String | 是 | 目标国家/地区代码，取值范围受platform参数限制（详见下方约束说明） | "US" |
| recallNum | Integer | 否 | 召回商品数量，默认10，取值范围：1-50 | 10 |
| similarityValue | String | 否 | 相似度阈值，默认0.5，取值范围：0.0-1.0。值越大返回结果越精准但数量可能越少 | "0.5" |
| userId | String | 是 | 用户ID，用于身份验证和访问控制 | "12345678" |

### 参数约束

#### 1. platform 与 region 的合法性校验

后端会根据platform的值校验region的合法性：

**当 platform = "amazon" 时**，region 合法值列表：
- US (美国)
- UK (英国)
- ES (西班牙)
- FR (法国)
- DE (德国)
- IT (意大利)
- CA (加拿大)
- JP (日本)

**当 platform = "tiktok" 时**，region 合法值列表：
- ID (印度尼西亚)
- VN (越南)
- MY (马来西亚)
- TH (泰国)
- PH (菲律宾)
- US (美国)
- SG (新加坡)
- BR (巴西)
- MX (墨西哥)
- GB (英国)
- ES (西班牙)
- FR (法国)
- DE (德国)
- IT (意大利)
- JP (日本)

#### 2. imageUrl 约束
- 必须是合法的HTTP/HTTPS URL
- 支持的图片格式：JPG, JPEG, PNG, WEBP
- 图片大小限制：不超过10MB
- 图片必须可公开访问

#### 3. recallNum 约束
- 默认值：10
- 最小值：1
- 最大值：50

#### 4. similarityValue 约束
- 默认值：0.5
- 最小值：0.0（最宽松，召回更多商品）
- 最大值：1.0（最严格，只召回最相似商品）
- 必须是数字格式的字符串

## 技术细节

### 核心流程

1. **参数校验**: 验证所有输入参数的合法性和一致性
2. **图片向量化**: 调用AI embedding模型将图片转换为高维向量
3. **向量检索**: 在OpenSearch中基于向量相似度召回商品
4. **结果过滤**: 根据平台、地区、相似度阈值过滤结果
5. **数据转换**: 将商品数据转换为API响应格式
6. **返回结果**: 返回格式化的商品列表

## 使用建议

### 1. 图片质量要求
- **清晰度**: 图片应清晰，避免模糊或分辨率过低
- **背景**: 建议使用纯色背景或简单背景的商品图
- **角度**: 建议使用正面或主视图
- **光线**: 确保光线充足，避免过暗或过曝

### 2. 参数调优建议

#### similarityValue 相似度阈值
- **0.3-0.5**: 适合发现同品类商品，召回量大
- **0.5-0.7**: 适合寻找相似款式，平衡召回量和精准度
- **0.7-0.9**: 适合寻找同款或极相似商品，召回量小但精准度高

#### recallNum 召回数量
- **5-10**: 适合快速浏览Top商品
- **10-20**: 适合常规选品场景
- **20-50**: 适合详细对比和分析场景

### 3. cropRegion 使用场景
- 图片中包含多个商品时，可裁剪指定商品区域
- 图片背景复杂时，可裁剪商品主体部分
- 格式: "x,y,width,height"，其中x,y为左上角坐标，width和height为裁剪区域的宽高

### 4. 平台和地区选择
- 根据目标市场选择对应的平台和地区
- 注意不同地区的商品库存可能不同
- 同一商品在不同地区的价格和供应商可能不同

## 常见问题 (FAQ)

### Q1: 为什么有时候返回的商品数量少于recallNum？
**A**: 可能的原因：
1. 相似度阈值设置过高，符合条件的商品较少
2. 该地区该平台的商品库存量有限
3. 输入图片比较特殊，难以匹配到相似商品

**建议**: 适当降低similarityValue参数值，或更换其他地区尝试。

### Q2: similarityScore 的值是如何计算的？
**A**: similarityScore是基于图片向量的余弦相似度计算得出，取值范围0-1：
- 0.9-1.0: 极高相似度，可能是同款商品
- 0.7-0.9: 高相似度，款式非常接近
- 0.5-0.7: 中等相似度，属于同品类相似商品
- 0.3-0.5: 较低相似度，品类相关但差异较大

### Q3: 支持哪些图片格式？
**A**: 目前支持 JPG、JPEG、PNG、WEBP 格式的图片，图片大小不超过10MB。

### Q4: cropRegion 的坐标如何确定？
**A**: cropRegion格式为"x,y,width,height"：
- x, y: 裁剪区域左上角的坐标（相对于原图）
- width, height: 裁剪区域的宽度和高度
- 示例: "100,50,300,400" 表示从坐标(100,50)开始，裁剪300x400像素的区域

### Q5: 接口调用失败怎么办？
**A**: 请检查以下几点：
1. 确认图片URL可公开访问
2. 确认platform和region的组合是合法的
3. 确认所有必填参数都已提供
4. 查看返回的错误码和错误信息
5. 如果问题持续，请联系技术支持并提供traceId

### Q6: 如何提高召回精准度？
**A**: 建议：
1. 使用高质量的商品图片
2. 适当提高similarityValue阈值
3. 使用cropRegion裁剪掉背景和无关元素
4. 选择商品主图而非使用场景图
