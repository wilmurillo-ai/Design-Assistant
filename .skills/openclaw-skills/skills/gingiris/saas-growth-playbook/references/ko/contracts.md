# 4부: 계약 템플릿 체계

## 계약 시스템 개요

| 문서 | 설명 |
|:---|:---|
| **SOR** (요구사항 정의서) | 클라이언트 작성, 비즈니스 목표와 요구사항 기술 |
| **SOW** (작업 범위 기술서) | 벤더 작성, 범위, 산출물, 타임라인 정의 |
| **MSA** (기본 서비스 계약) | 법적 마스터 계약, 일반 조건 정의 |
| **PCR** (프로젝트 변경 요청) | 범위 변경 관리 |
| **견적서** | 비용 구성, 지불 조건 |

## 3. MSA 핵심 조항

| 조항 | 요점 |
|:---|:---|
| **서비스** | 범위 정의, SOW 참조 |
| **지불** | 방법, 주기, 통화, 연체 페널티 |
| **지적재산** | 커스텀 → 클라이언트; 프레임워크 → 벤더 |
| **기밀유지** | 상호 NDA |
| **데이터와 보안** | 소유권, 사용, GDPR/CCPA 준수 |
| **SLA** | 업타임 (99.9%), 응답 시간, 구제 조치 |
| **책임 제한** | 상한 (보통 ≤ 계약 금액) |
| **기간과 종료** | 기간, 조기 종료 조건 |
| **준거법** | 적용 법률, 분쟁 해결 |
