# 第五部分：数据驱动的增长闭环

## 1. 北极星指标 (North Star Metric)

**定义：** 能够最好地衡量产品为客户创造核心价值的单一指标

**要求：**
- 整个公司（产品、研发、市场、销售）共同为之奋斗
- 直接反映客户获得的价值

**示例：**
| 产品类型 | 北极星指标 |
|:---|:---|
| AI视频工具 | 每周成功导出的视频数 |
| AI会议助手 | 每周生成的会议摘要数 |
| 协作平台 | 每周活跃协作的团队数 |

---

## 2. AARRR 漏斗指标体系

| 阶段 | 核心问题 | 关键指标示例 |
|:---|:---|:---|
| **拉新 (Acquisition)** | 用户从哪里来？ | 各渠道UV、Sign-up数、CAC（用户获取成本） |
| **激活 (Activation)** | 用户是否体验到核心价值？ | "Aha Moment"转化率、新手引导完成率 |
| **留存 (Retention)** | 用户是否会反复使用？ | 次日/7日/30日留存率、功能留存率 |
| **收入 (Revenue)** | 用户是否愿意付费？ | 付费转化率、ARPU、LTV |
| **推荐 (Referral)** | 用户是否愿意推荐他人？ | NPS、病毒系数K-factor |

---

## 3. 关键指标定义

### 获客指标

| 指标 | 定义 | 计算方式 |
|:---|:---|:---|
| **CAC** | 用户获取成本 | 总市场销售费用 / 新增用户数 |
| **LTV** | 客户生命周期价值 | ARPU × 平均客户生命周期 |
| **LTV/CAC** | 获客效率 | 健康值 > 3:1 |

### 激活指标

| 指标 | 定义 |
|:---|:---|
| **Aha Moment转化率** | 完成核心价值体验的用户占比 |
| **Time to Value** | 从注册到首次获得价值的时间 |

### 留存指标

| 指标 | 计算方式 |
|:---|:---|
| **次日留存率** | 第2天回访用户 / 第1天新增用户 |
| **7日留存率** | 第8天回访用户 / 第1天新增用户 |
| **30日留存率** | 第31天回访用户 / 第1天新增用户 |

### 收入指标

| 指标 | 定义 |
|:---|:---|
| **MRR** | 月度经常性收入 |
| **ARR** | 年度经常性收入 = MRR × 12 |
| **ARPU** | 每用户平均收入 |
| **Net Revenue Retention** | 净收入留存率（含扩展、流失） |

## 5. 增长会议最佳实践

### 频率
每周或每双周一次

### 参与者
产品、市场、销售、数据分析师

### 议程
1. 复盘上周期实验结果
2. 分析数据变化原因
3. 确定下一周期增长实验方向

### 原则
- 所有改动基于明确假设
- 通过A/B测试验证
- 决策必须有数据支撑

---

## 6. 增长实验框架

### ICE评分法

| 维度 | 说明 | 评分(1-10) |
|:---|:---|:---|
| **Impact** | 对北极星指标的潜在影响 | |
| **Confidence** | 对假设成立的信心程度 | |
| **Ease** | 实施的难易程度 | |

**优先级 = Impact × Confidence × Ease**

### 实验文档模板

```
实验名称: 
假设: 如果我们[做某事]，那么[某指标]将会[某变化]，因为[原因]
成功指标: 
实验设计: 
   - 对照组: 
   - 实验组: 
   - 样本量: 
   - 运行时间: 
结果: 
结论: 
下一步: 
```
