# Product Hunt 런칭

## 타이밍 고려사항

| 요소 | 권장 |
|:---|:---|
| 제품 성숙도 | 트래픽 급증에 대비한 안정성 확보 |
| 커뮤니티 기반 | Discord/Telegram 커뮤니티 준비 |
| GitHub Stars | PH 런칭 전 500+ 권장 |
| 경쟁 | 유명 제품과 같은 날 런칭 피하기 |
| 요일 | 화-목이 최적; 월요일은 경쟁 치열, 주말은 낮은 트래픽 |

---

## 헌터 선택

| 유형 | 장점 | 단점 |
|:---|:---|:---|
| **탑 헌터** | 팔로워 많음, 강한 추천 | 비쌈, 일정 빡빡 |
| **중간급 헌터** | 가성비 좋음, 원활한 소통 | 영향력 적음 |

## PH 페이지 자산

| 자산 | 요구사항 |
|:---|:---|
| 태그라인 | 한 줄 설명, 웹사이트와 일치 |
| 설명 | 상세한 가치 제안, 차별화 강조 |
| 5개 스크린샷/GIF | 각 핵심 판매 포인트에 대응 |
| 브랜드 비디오 | 첫 번째 위치, 사용자의 첫인상 |

## 업보트 전략

### 투표 클리어링 메커니즘
- PH는 약 2시간마다 투표 클리어
- 시간당 약 100표 제한
- 다양한 IP 주소 확보
- 단기간 집중 투표 피하기

### 비상 대응 계획
| 상황 | 대응 |
|:---|:---|
| Featured 안됨 | support@producthunt.com 연락 |
| 타인이 먼저 런칭 | PH에 연락, 소유권 증명 제공 |
