# Product Hunt 发布 (PH Launch)

Product Hunt发布是一场为期24小时的集中冲刺战役。

## PH发布时机选择

| 因素 | 建议 |
|:---|:---|
| 产品成熟度 | 确保产品足够稳定，能承受大量新用户涌入 |
| 社区基础 | 至少拥有一定规模的Discord/Telegram社区 |
| GitHub Stars | 建议达到500+后再发布PH |
| 竞争环境 | 避免与知名产品同天发布 |
| 发布日期 | 周二至周四发布，周一竞争激烈，周末流量低 |

**实用工具：** [PH Deck](https://www.phdeck.com/) — 每日查看Featured产品

## Hunter选择策略

### 三类Hunter对比

| 类型 | 代表人物 | 优点 | 缺点 |
|:---|:---|:---|:---|
| **海外顶级Hunter** | Chris Messina | 粉丝多，背书强 | 价格高，档期紧张 |
| **海外高性价比Hunter** | Kevin William David, Ben Lang | 性价比高 | 影响力不及顶级 |
| **国人Hunter** | 国内有PH经验的创业者 | 沟通成本低，价格实惠 | 海外影响力有限 |

### 与Hunter合作流程

1. **预约（L-6周）**：Twitter DM或邮件联系
2. **会议（L-4周）**：30-60分钟视频会议，听取官网和PH页面建议
3. **Draft Page（L-2周）**：创建Draft Page，邀请Hunter Review
4. **Maker Profile（L-1周）**：确保所有团队成员完善个人Profile
5. **Notify Page（PH Day前一天）**：点击"Notify Followers"

## PH页面物料准备

| 物料 | 要求 |
|:---|:---|
| Tagline | 一句话介绍，与官网保持一致 |
| Description | 详细描述产品价值，突出差异化 |
| 5张核心截图/GIF | 每张图对应一个核心卖点 |
| 品牌视频 | 放在首位，用户了解产品的第一印象 |
| 创始人视频 | 30-60秒，真诚介绍产品和团队故事 |

### Maker评论准备

```
Hi PH! I'm [Name], [role] at [Product Name].

We built this because [genuine story about the problem].

[Product Name] is different from [competitors] because [key differentiator].

We'd love to hear your feedback — especially on [specific aspect].

Ask us anything!
```

## Upvote策略与应急预案

### 清票机制说明

- PH大约每2小时进行一次清票
- 每小时上限约100票
- 确保Upvote来源IP地址多样化
- 不要短时间内集中大量Upvote
- 优先引导真实用户主动Upvote

### 各渠道拉票策略

| 渠道 | 预期效果 |
|:---|:---|
| LinkedIn策略 | 3人力可带来200-300票 |
| LinkedIn（5会员+实习生） | 1000票/天 |
| Telegram/Facebook群组 | 发布PH链接引导Upvote |
| 邮件激励 | 提前筛选有PH账号用户，Token激励 |
| 竞品票仓挖掘 | 爬取同赛道产品Review区，DM响应率约60% |

### 应急预案

| 情况 | 处理方式 |
|:---|:---|
| 未被Feature | 联系support@producthunt.com请求人工Review |
| 被他人提前发布 | 联系PH官方，提供所有权证明，请求删除或合并 |
| 被大量清票 | 暂停集中推广，分散式引导真实用户Upvote |

### 二次转化Tips

- 官网嵌入PH Code Block，将官网流量导回PH页面
- 拿到Badge后嵌入官网，方便后续冲周榜/月榜
