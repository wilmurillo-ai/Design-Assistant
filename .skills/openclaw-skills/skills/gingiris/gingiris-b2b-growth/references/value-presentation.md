# 第三部分：价值呈现的艺术

## 1. 价值主张案例

### Supabase: "Build in a weekend, scale to millions."

- **"Build in a weekend"** — 击中开发者快速启动、快速验证的痛点
- **"Scale to millions"** — 打消"能否用于生产环境"的疑虑
- **一句话覆盖个人开发者和企业决策者的核心关切**

### Vercel: "The Frontend Cloud"

- 没有描述具体功能，而是定义了一个全新品类
- 将自身与"云"概念绑定（可扩展、高可用）
- 从"Next.js托管平台"提升到"下一代Web开发基础设施领导者"

> **好的2B价值主张三要素：**
> 1. **一句话定位** — 你是谁，解决什么问题
> 2. **量化成果** — 能带来什么具体收益
> 3. **社会认同** — 谁在用你，谁为你背书

## 2. 2B 销售 Deck 9页结构

| 页码 | 内容 | 目的 |
|:---|:---|:---|
| 1 | **封面页** | 公司Logo + 客户Logo + 一句话价值主张 |
| 2 | **问题页** | 描绘客户当前的痛点和挑战，引发共鸣 |
| 3 | **解决方案页** | 阐述如何从根本上解决问题 |
| 4 | **产品演示页** | 核心功能截图或短视频 |
| 5 | **价值收益页** | 量化收益（如"提升30%效率"） |
| 6 | **客户案例页** | 同行业知名企业的成功案例 |
| 7 | **企业方案页** | 安全、合规、服务与支持特性 |
| 8 | **定价与方案页** | 不同版本的定价和功能差异 |
| 9 | **总结与下一步** | 核心价值总结 + 明确行动计划 |

## 3. 客户成功案例叙事 — 英雄之旅框架

### Vercel × Frame.io 案例解析

**第一幕：旧世界的挑战 (The Challenge)**
- Frame.io 网站迭代速度慢、性能差
- 旧技术栈成为业务发展瓶颈

**第二幕：英雄的顿悟 (The Epiphany)**
- 意识到必须将前端体验提升到极致
- 需要市场团队灵活发布 + 工程师掌控性能

**第三幕：寻找神兵利器 (The Solution)**
- 选择 Vercel + Next.js
- 利用 ISR、Edge Functions、Preview Deployments

**第四幕：新世界的胜利 (The Results)**
- 页面加载性能提升 **3倍**
- 发布新页面时间从**几天→几分钟**
- 全球站点可用性达到 **99.99%**
- 最终被 Adobe 高价收购

> **核心启示：** 将客户塑造为故事的"英雄"，将你的产品定位为英雄战胜挑战所获得的"神兵利器"

## 4. 客户案例标准化生产流程

1. **候选客户筛选**
   - 深度使用产品
   - 取得显著业务成果
   - 愿意分享

2. **访谈与信息收集**
   - 合作前的挑战
   - 选择你的原因
   - 实施过程
   - 量化成果

3. **故事线撰写与审核**
   - 按"英雄之旅"框架撰写
   - 交由客户审核

4. **多格式内容制作**
   - 网页版详细案例
   - PDF版本（供销售邮件发送）
   - 一页纸版本（线下展会）
   - 社交媒体卡片
   - 视频访谈（最具说服力）
