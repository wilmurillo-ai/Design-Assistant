# 发布前准备 SOP (T-4周 至 T-1周)

## 1. 战略规划与定位 (T-4周)

### 回答核心问题

| 关键问题 | 目标与说明 |
|:---|:---|
| **谁是你的用户?** | 定义目标开发者画像，分析其技术栈、工作场景和核心痛点 |
| **你解决了什么问题?** | 用一句话清晰描述项目解决的关键问题 |
| **给用户提供了什么价值?** | 提炼独特优势：效率提升、成本降低、技术创新 |
| **用户为什么要用你?** | 建立与闭源竞品的比较性优势，突出不可替代性 |
| **产品是否达到自来水程度?** | 评估核心功能是否稳定，有足够吸引力让早期用户自发推荐 |

### 明确商业模式

- **开源目的**: 提升影响力？探索商业化？招聘核心人才？为融资做准备？
- **商业模式**: To Enterprise 和 To Developer 的混合模式，API/SDK/Self-host 版本定价
- **资源评估**: 有多少人力可以投入到项目维护和社区运营？

### 确立价值主张 (Positioning)

> **实战方法**: 把市场上 Top 3 的竞品官网并排放在一起，学习其 Tagline 的表述方式，然后定义出自己的、更具吸引力的价值主张。

> **案例**: AppFlowy.IO 的定位非常清晰："The Open Source Alternative To Notion"

## 2. 核心资产准备 (T-3周)

### GitHub Readme

**专业、信息全面、易于理解** — 这是体现项目专业度的门面

- **顶部**: Tagline + Landing Page 链接
- **核心内容**: 清晰的项目描述、Quick Start、部署教程、开发者文档入口
- **社区与贡献**: Contributor 贡献指南，社区链接 (Discord/Telegram)
- **生态**: 生态合作伙伴链接，与其他开源项目抱团取暖

### 官网 (Landing Page)

- **首页 3 秒内**必须展示核心 Tagline
- **强制植入**: 官网与 GitHub Readme 互相链接，让用户在 3 分钟内反复看到关键概念
- **导航清晰**: Header 设置吸顶，放置 GitHub、文档、社区的明确入口

### PR稿 (1篇核心文章)

**故事性强，适合二次传播** — 这是其他自媒体二次传播的主要参考内容

- **内容**: 围绕价值主张讲述动人故事，包含核心功能、使用场景、技术亮点
- **链接**: 必须包含官网和 GitHub 链接
- **发布**: 发布在 Medium 等平台，作为后续所有渠道内容的基础蓝本

### 开发者文档

**用户友好，降低上手门槛** — 开发者验证项目是否合格的基操

- 清晰的 API/SDK 文档
- 详尽的安装、配置和部署指南
- 常见问题 QA 列表

### 素材包 (Content Package)

- Logo (不同尺寸和颜色版本)
- 产品截图、GIF 或短视频 Demo
- 统一的品牌色板和字体规范
- 为 KOL、媒体和社区准备的简介模板

## 3. 渠道与资源准备 (T-2周)

| 渠道 | 准备工作 |
|:---|:---|
| **KOL/Influencer** | 根据 KOL 清单建联询价，确定合作名单、形式和排期 |
| **Reddit** | 确定目标 Subreddit，准备高权重账号，分析社区规则 |
| **内容平台** | 准备 Medium、Dev.to、Hacker News、V2EX 等账号 |
| **海外群组** | 整理 Telegram、Discord、LinkedIn 群组列表 |
| **开源收录** | 整理 DevHunt、GitHub Topic 等平台列表 |
| **付费渠道** | 调研付费 Newsletter，了解价格和排期 |

---

## 4. 团队分工与预算 (T-1周)

- **明确分工**: 制定 `by小时 by渠道` 的 To-do List，将每个任务分配到个人
- **预算审批**: 根据 KOL 报价和付费渠道费用，制定详细预算
- **沟通机制**: 建立发布期间的即时沟通渠道（微信群、Slack）

---

## 5. 发布前冲刺 (T-3天)

### 文案最终版确认

- **Twitter**: 所有 KOL 合作文案（Thread, Quote, Single Post）和官方发布文案
- **Reddit**: 针对不同 Subreddit 的帖子标题和正文
- **Medium**: 核心 PR 稿最终版本
- **DM/Comment**: Twitter、LinkedIn、Medium 的私信和评论模板

### KOL 最终沟通

- 与所有已确认合作的 KOL 进行最后沟通
- 确认发布时间（精确到小时）和发布内容
- 发送最终的 Content Package

### AI Automation 设置

- **Twitter Auto Comment**: 使用 Phantombuster 设置自动评论工作流
- **LinkedIn DM**: 设置 LinkedIn 自动化联系流程
- **目标用户列表**: 准备竞品 follower 列表

### 内部总动员

- 召集所有参与发布的团队成员
- 同步最终的发布日小时级执行计划
- 确保人人明确自己的职责
