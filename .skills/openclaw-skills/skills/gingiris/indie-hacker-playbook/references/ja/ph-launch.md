# Product Huntローンチ

## タイミング考慮事項

| 要因 | 推奨 |
|:---|:---|
| 製品成熟度 | トラフィック急増に対応できる安定性を確保 |
| コミュニティ基盤 | Discord/Telegramコミュニティを準備 |
| GitHub Stars | PHローンチ前に500+を推奨 |
| 競争 | 有名製品と同日ローンチを避ける |
| 曜日 | 火-木がベスト；月曜は競争激しい、週末は低トラフィック |

---

## ハンター選択

| タイプ | メリット | デメリット |
|:---|:---|:---|
| **トップハンター** | フォロワー多、強い推薦 | 高価、スケジュールタイト |
| **中堅ハンター** | コスパ良、スムーズな連携 | 影響力は劣る |

---

## PHページアセット

| アセット | 要件 |
|:---|:---|
| タグライン | 一言説明、ウェブサイトと一致 |
| 説明文 | 詳細な価値提案、差別化を強調 |
| 5枚のスクリーンショット/GIF | 各コア販売ポイントに対応 |
| ブランドビデオ | 最初の位置、ユーザーの第一印象 |

## アップボート戦略

### 投票クリアリングメカニズム
- PHは約2時間ごとに投票をクリア
- 1時間あたり約100票の制限
- 多様なIPアドレスを確保
- 短時間での集中投票を避ける

### 緊急対応プラン
| 状況 | 対応 |
|:---|:---|
| Featuredされない | support@producthunt.comに連絡 |
| 他者に先にローンチされた | PHに連絡、所有権証明を提供 |
