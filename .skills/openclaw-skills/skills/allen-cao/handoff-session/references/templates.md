# Templates V2

## 八槽交接包模板

```md
# Handoff：{主题}

- 时间：{YYYY-MM-DD HH:MM}
- Handoff ID：{handoff_id}
- Session ID：{session_id}

## 当前任务
- {一句话描述当前任务}

## 当前状态
- {当前阶段 / 进度 / 卡点}

## 已完成
- {已完成事项 1}
- {已完成事项 2}

## 关键决策
- {只写结论}

## 关键约束
- {不能违反的限制}

## 关键文件
- `path:line`：{作用 / 为什么下一会话先看它}

## 下一步
- {下一会话直接执行的动作}

## 待确认
- 无 / {唯一阻塞项}
```

填写规则：

- 8 个槽位必须都写
- 没内容时写 `未记录` 或 `无`
- `关键文件` 只保留最关键的 1-5 个
- `下一步` 必须是动作句
- 目标长度 100 行内
- 建议不超过 120 行
- 复杂任务可以放宽到 150 行内

## `handoff out` 固定输出

```text
Handoff saved.

  ID:   20260308-1723-019cccb6
  File: ~/.agents/handoff_context/20260308-1723-019cccb6/handoff.md

  Next session → handoff in 20260308-1723-019cccb6
```

## `handoff in` 恢复顺序

1. 先用 3-5 条 bullets 复述恢复结果
2. 明确说明准备执行的下一步
3. 先征得用户同意
4. 用户同意后再打开 `关键文件`
5. 用户同意后再执行 `下一步`
6. 只有在 `待确认` 阻塞时才提问

## 应删掉的内容

- 长聊天记录
- 多轮来回讨论
- 已否决方案
- 不能帮助下一会话开工的背景
- 情绪化、解释性、礼貌性废话
