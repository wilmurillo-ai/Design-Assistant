# 翻译与 Word 输出规范

## 一、总体原则

对外文新闻进行中文处理时，应优先保证**事实准确、措辞克制、结构清晰、来源可追溯**。翻译的目标不是改写成评论文章，而是生成适合中文读者阅读的新闻型译文与简报型汇编。所有译文必须保留原始标题、原文链接、来源名称、发布时间、作者信息（若页面可得）以及原文中的关键事实关系。

对于中文网站，不做“翻译”，只做必要的**规范化清洗**与**提炼摘要**。对于英文网站，则同时生成以下内容：原文标题、中文标题、中文摘要、中文全文译文。若原文极长，可在全文译文前插入“要点摘要”，但不能只给摘要而丢失正文。

## 二、输出字段规范

每篇文章至少输出以下字段。

| 字段 | 必填 | 说明 |
|---|---|---|
| source_name | 是 | 媒体名称 |
| source_region | 是 | International / Singapore / China Mainland / Greater China |
| section | 否 | World / Singapore / China / Business 等频道名 |
| article_url | 是 | 正文链接 |
| title_original | 是 | 原始标题 |
| title_zh | 是 | 中文标题 |
| published_at | 尽量 | 原站发布时间，保留原时区或注明原时区 |
| author | 尽量 | 作者或机构署名 |
| language_original | 是 | 原文语言 |
| summary_zh | 是 | 120 至 220 字中文摘要 |
| body_original | 尽量 | 原文正文清洗结果 |
| body_zh | 是 | 中文正文，分段呈现 |
| keywords_zh | 是 | 3 至 8 个中文关键词 |
| topic_label | 是 | 政治 / 国际 / 财经 / 科技 / 社会 / 冲突 / 灾害 等 |
| extraction_note | 否 | 对缺失内容、付费墙、动态加载失败等进行说明 |

## 三、中文翻译规则

### 1. 标题翻译

标题翻译应优先保留事件主体、动作和结果，不做夸张词替换，不自行加入立场。若原标题较短或较口语化，可适度补足主语，但不得添加原文没有的信息。

### 2. 正文翻译

正文翻译应采用标准书面中文，按新闻段落组织，不使用机器腔式直译。数字、时间、机构、人名和地点必须优先保证准确。缩略语第一次出现时建议采取“中文全称（英文缩写）”写法，例如“世界卫生组织（WHO）”。

### 3. 专有名词处理

已存在通行译名的机构、人名、条约、地区或公司，应优先使用通行中文译名；如不确定，可保留英文原文并在首次出现时附于括号中。对存在歧义的地名与机构名，不得擅自猜测。

### 4. 引述处理

引语应保持原意，必要时用间接引语表达。若原文为核心人物直接表态，可保留引号。不得把作者分析、编辑解释与受访者原话混为一体。

### 5. 不确定性标注

当原文存在“reportedly”“according to officials”“preliminary”“could/may”等不确定性表述时，中文中必须保留这种不确定性，而不是翻译成确定事实。

## 四、摘要生成规则

中文摘要应在 120 至 220 字之间，回答以下四个问题：发生了什么、谁是主体、为什么重要、下一步可能看点是什么。摘要不得只是标题改写，也不得夹带评论性判断。若文章属于持续更新的突发新闻，摘要中应明确注明“截至发稿时”或“当前已知信息”。

## 五、Word 文档结构

默认每次运行生成一份“每日新闻中文汇编”Word 文档。推荐结构如下。

| 层级 | 内容 | 说明 |
|---|---|---|
| 封面 | 报告标题、日期、批次、覆盖源 | 例如《每日新闻中文汇编（2026-04-16 早报）》 |
| 目录 | 按栏目与媒体列出 | 建议自动生成 |
| 执行摘要 | 5 至 10 条总体要点 | 面向快速阅读 |
| 正文栏目一 | 国际新闻 | 按重要性排序 |
| 正文栏目二 | 新加坡新闻 | 按重要性排序 |
| 正文栏目三 | 中国新闻 | 按重要性排序 |
| 正文栏目四 | 财经与市场 | 可选栏目 |
| 附录 | 来源列表、抓取说明、失败清单 | 保持可追溯性 |

## 六、单篇文章在 Word 中的呈现模板

每篇文章建议按如下顺序呈现：

1. 中文标题
2. 原始标题
3. 来源、频道、发布时间、作者、链接
4. 中文摘要
5. 中文全文译文
6. 原文关键引述或原文首段（可选）
7. 提取备注

## 七、排版建议

中文正文建议使用常规宋体或等价中文字体，英文与 URL 可保留无衬线字体。一级标题建议 16 至 18 磅，二级标题 14 磅，正文 11 至 12 磅。段前段后保持适中留白，避免一页出现过多 URL 裸链。链接应以内嵌文本形式出现，例如“查看原文”。

## 八、失败与异常处理

若正文未能完整提取，应在 `extraction_note` 中明确写出原因，例如“文章存在订阅限制，仅提取到导语”“正文为动态脚本加载，首轮抓取失败”“页面为视频页，无可读正文”。不得用推测内容补全缺失段落。

## 九、最低质量门槛

只有在以下条件同时满足时，文章才应进入最终 Word 文档：

| 条件 | 要求 |
|---|---|
| 可追溯性 | 必须保留原文链接与来源 |
| 可读性 | 中文标题和摘要通顺可读 |
| 完整性 | 至少提取到核心正文段落，而不只是标题 |
| 忠实性 | 不得添加未经原文支持的事实 |
| 去重性 | 与同批次中其他文章不构成明显重复 |

## 十、与模板文件的关系

具体占位格式与文档章节标题，可直接复用 `../templates/daily_news_digest_template.md`。若需要生成 JSON、Markdown 或 DOCX，中间层字段命名应保持与本文件一致。
