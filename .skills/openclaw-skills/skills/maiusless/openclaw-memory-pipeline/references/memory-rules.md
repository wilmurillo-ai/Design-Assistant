# Memory rules

## Default mode

Default to **concise / curated memory**.

Record what will matter later, not the whole conversation.

## Good candidates for memory

Store these by default:

- user preferences
- explicit decisions
- todos and reminders
- project status changes
- recurring workflow constraints
- mistakes, fixes, and lessons learned
- context needed to resume a multi-step task later

## Usually skip these

Do not store by default:

- casual chatter with no future value
- repeated confirmations
- low-value emotional filler
- full transcript copies
- temporary noise that will not help future work

## Density controls

Increase capture density when the user says:

- remember this
- 记一下
- keep this
- save this for later

Decrease or suppress capture when the user says:

- don’t remember this
- 别记
- this is temporary

## Writing style

When writing memory:

- prefer facts over commentary
- summarize instead of quoting
- avoid vague phrases like “recently” when exact dates or times are available
- deduplicate near-identical items
- keep bullets compact and scannable

## File placement guidance

- place new short-term items into `memory/inbox.md`
- place date-based archive output into `memory/YYYY-MM-DD-raw.md`
- place daily highlights into `memory/YYYY-MM-DD.md`
- place durable global rules into `MEMORY.md`
- place project-specific durable context into `memory/projects/`
- place system- or workflow-level durable context into `memory/system/`
