# T 型版式 (顶部横贯 + 底部偏心)

> 重力场：T 形悬臂 -- 顶部横跨的总览大梁 + 底部非均分的深潜区

适用：先总览再展开，且底部信息有明确的主次关系。与英雄式的区别在于底部是非均分的。

## 重力结构

```
Grid: 2fr 1fr 列 x auto 1fr 1fr 行
顶部横条: grid-row: 1; grid-column: 1 / -1（必须跨两列）
底左大卡: grid-row: 2 / 4; grid-column: 1（跨后两行）
底右上: grid-row: 2; grid-column: 2
底右下: grid-row: 3; grid-column: 2
```

## 灵动化指引

### 顶部大梁的多重人格
- **宣言式**：一行 28px 核心论断 + 横向排列的 3-4 个 KPI 数据药丸，像新闻速报的标题栏
- **氛围式**：极淡的配图底纹 + 半透明文字叠加，用 `glass` 让顶部成为画面的"天空"
- **数据带**：横向的进度条/对比柱排列，用 `transparent` 让数据裸露在画面顶部

### 底部深潜区的层次
- 底左大卡是"详细展开"的主战场 -- 图表 + 解读 + 论述，用 `filled` 或 `elevated` 给予厚重感
- 底右两个小卡是"侧面低语" -- 紧凑的补充数据，用 `outline` 或 `transparent` 保持轻盈
- 三者之间的 card_style 必须有至少 2 种差异

### 关键 Grid 约束

| 属性 | 必须性 | 说明 |
|------|-------|------|
| 顶部横条 `grid-column: 1 / -1` | **必须** | 不写则只占一列 |
| 底左大卡 `grid-row: 2 / 4` | **必须** | 不写则只占一行 |
| 每个卡片都需要明确 grid 定位 | **必须** | T 型不能依赖自动排列 |
