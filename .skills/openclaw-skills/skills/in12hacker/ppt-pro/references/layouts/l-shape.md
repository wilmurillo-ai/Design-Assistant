# L 型版式 (主体 + 侧翼 + 底部)

> 重力场：L 形的视觉轨迹 -- 从主体深潜，滑向侧面佐证，沉入底部锚定

适用：核心展示 + 辅助数据 + 底部总结/标签条。信息有三个层级时的黄金版式。

## 重力结构

```
Grid: 2fr 1fr 列 x 1fr 1fr auto 行
主卡片: grid-row: 1 / 3; grid-column: 1（跨前两行+左列）
右上辅助: grid-row: 1; grid-column: 2
右下辅助: grid-row: 2; grid-column: 2
底部横条: grid-row: 3; grid-column: 1 / -1（必须跨两列）
```

## 灵动化指引

### 三层级的情绪剧变
- **第一层（主区域）** 是画面的重力核心 -- 可以是浓墨重彩的数据风暴，也可以是极简的单一论断悬浮在大片留白中
- **第二层（右侧双卫星）** 是侧翼呐喊 -- 紧凑但精准，两个卡片用不同 card_style（如 `elevated` vs `outline`）制造高低错落
- **第三层（底部横条）** 是底部锚定 -- 可以是跨越全宽的标签云、指标行、或一句有力的结论。用 `transparent` 让它轻盈，或 `accent` 让它重重收官

### 变奏可能
- 主区域用 `transparent` + 120px 超大数据，右侧用 `filled` 承载细节 -- "左侧震撼、右侧解读"
- 右侧两卡片一个 `accent` 一个 `transparent` -- "一个燃烧一个呼吸"
- 底部横条用 `glass` 覆盖在一个极淡的渐变背景装饰上 -- 制造雾中锚定感

### 关键 Grid 约束

| 属性 | 必须性 | 说明 |
|------|-------|------|
| 主卡片 `grid-row: 1 / 3` | **必须** | 不写则主区域只占一行 |
| 底部横条 `grid-column: 1 / -1` | **必须** | 不写则底部只占一列 |
| 每个卡片都需要明确 grid 定位 | **必须** | L 型不能依赖自动排列 |
