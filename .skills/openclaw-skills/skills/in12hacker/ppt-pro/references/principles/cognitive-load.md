# 认知负荷与信息密度

> 观众的工作记忆有限（7 +/- 2 个信息块）。超出容量的信息不是"丰富"，而是"噪音"。

## 核心理论

### Miller's Law（7 +/- 2 法则）
- 人一次能处理的信息块约 5-9 个
- **PPT 应用**：每页核心要点不超过 5 个；列表项控制在 4-6 条；图表的数据系列不超过 5 组
- 超出时：用分组（chunking）把信息打包成更大的块

### 信噪比（Signal-to-Noise Ratio）
- Garr Reynolds《演示之禅》：删除一切不帮助传达信息的元素
- **PPT 应用**：每个装饰元素都要问"删掉它信息是否丢失？"；如果答案是"不会"，考虑删掉
- 装饰是为了引导注意力，不是为了"好看"

### Mayer 多媒体学习理论
- **冗余原则**：同时用文字和语音说同一件事，反而降低理解（现场演讲时文字要精简）
- **临近原则**：相关的文字和图像要放在一起（不要图在左边、说明在右边一页之隔）
- **分割原则**：复杂信息分步展示比一次呈现效果好（一步一页 > 一页全写）

## 信息密度光谱

```
← 太空          合适          太满 →
---|-------------|-------------|---
章节封面        内容页         数据仪表盘
(25-40%)       (60-75%)      (70-80%)
```

- **太空**：观众觉得"什么都没讲"（浪费他们的时间）
- **太满**：观众觉得"信息轰炸"（记不住任何东西）
- **合适**：观众能说出"这页讲的是 XXX"（一页一个核心论点）

## 一页一观点法则

每页应该能用一句话总结核心信息。如果需要两句话，考虑拆成两页。

**测试方法**：写完策划稿后，为每页写一句 `goal`（"这页最想让观众记住什么"）。如果 goal 包含"和"字（"A 和 B"），大概率应该拆页。

## 自检

- 这页删掉 30% 的内容，核心信息是否仍然清晰？
- 观众 5 秒能否抓住这页的核心论点？
- 每页的 card_type 种类是否超过 3 种？（超过可能信息过载）
