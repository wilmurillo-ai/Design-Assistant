# 数据可视化原则

> 数据可视化的目标不是"展示数据"，而是"让数据讲故事"。一个好图表让观众 3 秒内理解结论。

## Tufte 的数据墨水比

> 数据墨水比 = 用于展示数据的墨水 / 总墨水量

- **最大化**数据墨水：每个视觉元素都在传达数据
- **最小化**非数据墨水：删除装饰性网格线、3D 效果、渐变填充
- PPT 中的暗线：图表背景透明、轴线极细（1px, 20%透明度）、标签精简

## 图表选择的黄金法则

| 你想表达什么 | 用什么 | 不用什么 |
|------------|--------|---------|
| 占比/份额 | 环形图 / 堆叠条 / 树图 | 折线图 |
| 趋势/变化 | 折线图 / 面积图 | 饼图 |
| 对比/排名 | 柱形图 / 对比条 | 雷达图（超过5项时） |
| 单个KPI | KPI 卡 / 大数字 | 复杂图表 |
| 多维能力 | 雷达图（3-5维） | 表格 |
| 流程转化 | 漏斗图 | 饼图 |
| 时间节点 | 时间线 | 表格 |

**最常犯的错误**：用饼图展示超过 5 个分类（人眼无法比较细小的扇区差异）。

## Stephen Few 的仪表盘原则

- **一屏完整**：所有关键指标在一个视图内，不需要滚动
- **标注异常**：正常数据用中性色，异常数据用 accent 色高亮
- **提供上下文**：单独的数字没有意义，"37%" 不如 "37%（行业均值 22%）"
- **趋势 > 快照**：一个带趋势的数字比一个静态数字信息量大 10 倍

## 数据诚信

- **不截断 Y 轴**（除非明确标注）：从非零开始的柱形图歪曲比例
- **标注数据来源**：没有来源的数据 = 编造
- **区分事实和推测**：预测数据用虚线/浅色，历史数据用实线/深色

## 自检

- 每个数据可视化是否有明确的"结论"（不是让观众自己找结论）？
- 去掉图表只看数字，信息是否仍然清晰？（如果是，图表可能多余）
- 看数字不看图表，信息是否清晰？（如果是，标注可能不足）
