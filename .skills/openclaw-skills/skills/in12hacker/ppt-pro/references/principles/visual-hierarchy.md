# 视觉层级与 CRAP 原则

> 人眼不是逐字阅读，而是按视觉权重跳跃扫描。设计的核心任务是用视觉手段控制阅读顺序。

## CRAP 四原则（Robin Williams）

### Contrast（对比）
- **原理**：差异越大，层级越清晰。相似的元素会被视为同级
- **PPT 应用**：标题 vs 正文至少差 2 级字号；强调数据用 accent 色而非加粗；不同层级的卡片用不同 card_style（不同的"空间存在感"）
- **常见错误**：标题和正文字号差异微弱（看不出层级）；所有文字都加粗（等于没加粗）；所有卡片都用同一种 card_style

### Repetition（重复）
- **原理**：重复使用相同视觉元素创造统一感和可预测性
- **PPT 应用**：所有页面使用同一套 CSS 变量（色彩基因一致）；标题装饰手法跨页保持一致的系统；风格的"DNA"贯穿全篇
- **常见错误**：每页用不同的装饰风格（看起来像拼凑）；风格缺乏统一的基因
- **灵动平衡**：重复是"基因统一"，不是"所有页长一样"。基因一致的前提下，每页的版式、card_style 组合、装饰细节都应该灵动变化

### Alignment（对齐）
- **原理**：页面上没有任何元素是随意放置的，每个元素都应与其他元素有视觉连接
- **PPT 应用**：使用 CSS Grid 严格对齐（而非绝对定位猜坐标）；卡片间距统一；对齐方式统一（左对齐是最安全的选择）
- **灵动平衡**：对齐是规则，但刻意打破对齐也可以是灵动的设计手段。关键是"有意识的对齐"或"有意识的破格" -- 两者都比"无意识的错位"强

### Proximity（亲密性）
- **原理**：相关的信息靠近，不相关的信息远离。间距传达关系
- **PPT 应用**：同一卡片内元素紧密（标题-正文间距小于卡片间距）；不同层级的间距有明确梯度
- **灵动平衡**：间距不必永远恒定。可以用极大的留白把某个核心元素"孤立"出来，制造戏剧性的呼吸感 -- 这就是用亲密性原则的"反向应用"来制造视觉锚点

## 视觉重量阶梯

视觉权重从高到低（大致参考，非固定值）：

```
超大数据(64px+) > 页面标题(28px+) > 配图 > 卡片标题(16-20px) > 数据数字(36px+) > 正文 > 标注/注释
```

**一页只有一个视觉锚点**：最先吸引视线的元素。如果两个元素争夺锚点（如两个超大数字并排），观众不知道先看哪个。

## 自检

- 闭上眼再睁开，第一个看到的元素是否是这页最重要的信息？
- 能否在 3 秒内识别出页面的层级结构？
- 跨页翻阅时，视觉系统是否一致（不是每页一种风格）？
- 有没有刻意的灵动变化让每页视觉新鲜感不同？
