# 风格系统

## 核心理念

> **8 个预置风格是参考调色板，不是固定制服。** 它们提供经过验证的色彩搭配和装饰思路，但你完全可以：
> - 基于预置风格微调色值
> - 混搭两种风格的特质
> - 根据主题完全自创配色方案（只需遵守下方的色彩原则）
>
> **唯一不可突破的底线**：所有颜色必须通过 CSS 变量引用，确保全局一致性。
>
> **灵动法则**：同一套风格变量下，每一页的装饰组合应当不同。装饰工具箱提供的是可自由组合的技法，不是固定套餐。每页选 2-3 种技法组合，且相邻页的组合不应雷同。

## style.json 数据模型

style.json 不仅是 CSS 变量的容器，更是整个 PPT **视觉灵魂的基因型定义**。它必须承载三层信息：色值基因、灵魂描述、变奏策略。

### 完整结构（自创或选用风格时都必须输出）

```json
{
  "style_id": "自定义 ID 或预置 ID",
  "style_name": "风格名称",
  "mood_keywords": ["3-5 个情绪关键词，如 '精密仪器' '深空冷寂' '微光脉搏'"],
  "design_soul": "一句话灵魂宣言 -- 用感性语言描述这套风格想传达的核心情绪体验。这句话将作为每页 HTML 生成时的情绪锚点。例如：'故宫红墙上一缕金线划过黄昏，庄重中透出温暖的光'",
  "variation_strategy": "跨页变奏策略 -- 描述在这套风格基因下，页与页之间如何制造灵动变化。例如：'深底页用网格点阵+角标装饰线制造精密仪器感，浅转场页用大面积光晕呼吸+极简留白，两种极端交替形成视觉心跳'",
  "decoration_dna": {
    "signature_move": "这套风格最标志性的装饰手法（1 种），每 3-4 页至少出现一次",
    "forbidden": ["明确禁止的装饰手法（会破坏风格一致性的元素），如 '网格点阵' '光晕效果'"],
    "recommended_combos": [
      "推荐的装饰技法组合 A（如 '标题下划线 + 编号气泡 + 渐隐分隔线'）",
      "推荐的装饰技法组合 B（如 '角标装饰 + 大号水印 + 品牌色块'）"
    ]
  },
  "css_variables": {
    "bg_primary": "#主背景",
    "bg_secondary": "#渐变过渡色",
    "card_bg_from": "#卡片渐变起",
    "card_bg_to": "#卡片渐变止",
    "card_border": "rgba(边框色)",
    "card_radius": "12px",
    "text_primary": "#标题/正文",
    "text_secondary": "rgba(辅助文字)",
    "accent_1": "#主强调",
    "accent_2": "#次强调",
    "accent_3": "#第三强调",
    "accent_4": "#第四强调"
  },
  "font_family": "PingFang SC, Microsoft YaHei, system-ui, sans-serif"
}
```

### 字段说明

| 字段 | 作用 | 影响什么 |
|------|------|---------|
| `mood_keywords` | 3-5 个情绪关键词，指导每页生成时的装饰氛围选择 | 策划师的 `decoration_hints` 选择 + 设计师的装饰实现 |
| `design_soul` | 一句话灵魂宣言 -- 用感性语言描述风格的核心情绪体验 | 注入到每页 prompt 中，作为设计师的情绪锚点 |
| `variation_strategy` | 跨页变奏策略 -- 描述页间变化的节奏和手法 | 引导设计师在统一基因下制造灵动跳变 |
| `decoration_dna` | 装饰基因（标志手法/禁止手法/推荐组合） | 确保装饰选择既灵动又不跳戏 |
| `css_variables` | 纯色值变量 -- 全局一致性的硬保证 | 直接映射为 CSS :root 变量 |

> **关键区分**：`design_soul` 和 `variation_strategy` 是给 LLM 的"设计意图上下文"，而非具体的 CSS 代码。它们解决的是"为什么选这些颜色、页间怎么变化"的灵魂问题，而不是"这个元素用什么 CSS"的实现问题。

## 色彩原则（自创配色时的硬底线）

| 原则 | 要求 | 目的 |
|------|------|------|
| 对比度安全 | 文字与背景的对比度 >= 4.5:1 | 确保可读性 |
| 60-30-10 法则 | 背景色 60%、卡片/辅助色 30%、accent 色 10% | 避免"花哨" |
| accent 克制 | 同一页 accent 色不超过 2 种（不同页可以不同） | 视觉焦点清晰 |
| 深浅一致 | 深色背景配浅色文字，浅色背景配深色文字 | 基本可读性 |
| 卡片与背景区分 | 卡片背景与页面背景有可感知的色差（至少 5% 明度差） | 卡片边界清晰 |

## 风格决策流程

### 决策优先级

| 优先级 | 信号 | 行动 |
|--------|------|------|
| 1 | 用户**明确指定**了风格或品牌色 | 直接用 / 基于品牌色自创 |
| 2 | 主题有强烈的情绪倾向 | 选择匹配情绪的调色板，可微调 |
| 3 | 受众有明确偏好 | 参考下方受众映射 |
| 4 | 均无明显倾向 | blue_white 作为安全默认 |

### 受众 -> 情绪 -> 调色板

| 受众 | 期望的情绪 | 参考调色板 |
|------|----------|----------|
| 企业管理层/投资人 | 专业可信 | blue_white / minimal_gray / 自创冷色系 |
| 政府/事业单位 | 稳重正式 | blue_white / royal_red / 自创庄重色系 |
| 学术/研究人员 | 冷静克制 | minimal_gray / 自创低饱和度色系 |
| 开发者/技术团队 | 极客未来感 | dark_tech / 自创深色科技系 |
| 消费品/生活方式品牌 | 温暖高级感 | warm_earth / 自创暖色系 |
| 年轻消费者 | 活泼创意 | vibrant_rainbow / 自创亮色系 |
| 高端客户 | 奢华质感 | luxury_purple / 自创深色高级系 |
| 健康/自然主题 | 有机清新 | fresh_green / warm_earth / 自创绿色系 |

### 灵动创作方法论 -- 像设计总监一样思考

> 不要只是"选一个调色板然后输出 CSS 变量"。你是在为这个特定项目**调配独一无二的视觉灵魂**。

**第一步：提炼情绪关键词**

从主题、受众、叙事结构中提炼 3-5 个**感性情绪词**，不是"蓝色""深色"这种色彩描述词，而是"精密仪器感""深空冷寂""温暖炉火"这种**通感体验词**。

| 主题类型 | 情绪关键词示例 |
|---------|-------------|
| AI/技术产品 | 精密仪器、深空冷寂、微光脉搏、数据洪流 |
| 企业商务 | 稳如磐石、蓝图清晰、信赖锚点、专业呼吸 |
| 文化/历史 | 宫墙黄昏、金缕暗纹、厚重沉淀、仪式庄严 |
| 健康/自然 | 晨光透叶、清泉细流、有机呼吸、温柔生长 |
| 创意/年轻 | 弹跳色块、棱镜折射、派对蜜液、不设限的笑 |

**第二步：从情绪推导色彩**

不是"选蓝色因为是科技"，而是"选深海蓝+青光脉搏的组合因为要传达'精密仪器在深空中有节奏地呼吸'的感觉"。

| 推导路径 | 示例 |
|---------|------|
| 情绪 -> 自然类比 -> 色彩 | "精密仪器" -> 天文台穹顶内部 -> 深蓝黑底 + 冷青扫描光 |
| 情绪 -> 场景联想 -> 色彩 | "宫墙黄昏" -> 夕阳打在朱墙上 -> 深朱红 + 暗金侧光 |
| 情绪 -> 触觉通感 -> 色彩 | "温柔生长" -> 手指触碰新叶 -> 极浅薄荷 + 暖阳金绿 |

**第三步：写灵魂宣言**

用一句话把这套色彩要传达的**画面感**写出来。这句话将注入到每页的 HTML 生成 prompt 中，成为设计师的情绪锚点。

好的灵魂宣言：
- "天文台穹顶内部，深蓝黑幕中冷青的仪器扫描光有节奏地划过 -- 精密、冷寂、但每一次扫描都暗含脉搏"
- "故宫红墙上一缕金线划过黄昏，庄重中透出温暖的光 -- 每一处纹理都有千年的重量"
- "晨光透过嫩叶的筛孔，在白纸上投射出薄荷色的呼吸 -- 轻柔、有机、不急不徐"

坏的灵魂宣言：
- "蓝色科技风格，专业商务" -- 没有画面感，LLM 只会输出千篇一律的蓝色方块
- "深色底+亮色accent" -- 这是色值描述不是灵魂

**第四步：设计变奏策略**

在统一基因下，页与页之间如何制造灵动变化。变奏策略不是"每页用不同装饰"（这谁都知道），而是描述**变化的节奏型**。

好的变奏策略：
- "深底页用网格点阵+角标装饰线制造精密仪器感（紧张），浅转场页用大面积光晕呼吸+极简留白（释放），两种极端交替形成视觉心跳"
- "奇数页偏暖偏重（金色装饰厚涂），偶数页偏冷偏轻（仅细线+水印），制造宫廷画卷翻页的明暗交替"

坏的变奏策略：
- "每页用不同的装饰" -- 废话
- "交替使用不同布局" -- 这是布局问题不是风格变奏

### 自创配色指引

当预置调色板都不完美匹配时，自创：

1. 完成上述"灵动创作四步法"
2. 选择一个最接近的预置调色板作为**起点**
3. 按以下策略微调：
   - 换 accent 色 -> 保持背景和文字不变，只调强调色
   - 换色温 -> 冷色系换暖色系（蓝->橙）或反之
   - 换明度 -> 浅底换深底（同时翻转文字色和卡片色）
4. 输出完整的 style.json（**必须包含 mood_keywords / design_soul / variation_strategy / decoration_dna**，不能只有 css_variables）

## 装饰技法工具箱

> **装饰不是固定清单，是可自由组合的技法。** 每种技法有其视觉灵魂和适用场景。每页至少 2-3 种技法的组合。

### 背景层技法

| 技法 | 视觉灵魂 | 适合的情绪 |
|------|---------|----------|
| 网格点阵 | 精密仪器感，像坐标纸的底纹 | 科技、工程、数据驱动 |
| 渐变色块 | 柔和的光照感，像营火旁的光晕 | 温暖、自然、亲和 |
| 光晕效果 | 未来感的光球，像晏暗中的聚光灯 | 发布会、科幻、极客 |
| 几何线条 | 建筑的骨架感，像蓝图上的辅助线 | 工程、结构、严谨 |
| 纹理底图 | 物质感，像纸张/绸缎/水墨的触感 | 文化、质感、传统 |

### 卡片层技法

| 技法 | 视觉灵魂 | 效果 |
|------|---------|------|
| 左侧强调线 | 栅栏的竖条，引导读者视线的入口 | 视觉锚点 |
| 编号气泡 | 圆形的序号牌，像导航标记 | 序号感 |
| 顶部色带 | 卡片的"帽子"，一条彩色横线标识类别 | 分类标记 |
| 卡片悬浮感 | 多层阴影制造的 Z 轴凸起 | 层次感（仅浅色风格） |

### 页面层技法

| 技法 | 视觉灵魂 | 效果 |
|------|---------|------|
| 角标装饰 | 精致的边框点缀，像相框的角尖 | 精致感 |
| 渐隐分隔线 | 双端渐隐的一条细线，像雾中的地平线 | 区域轻柔划分 |
| 大号水印 | 超大尺寸极低透明度的数字/文字，像汪洋中隐约的冰山 | 页面识别度 |
| 标题下划线 | accent 色的短横带，像标题的"基座" | 标题强调 |
| 品牌色块 | 大面积低透明度的圆角矩形，像跳板上的水泡 | 风格标记 |

## 多语言排版优化

### 英文场景

| 层级 | 中文默认 | 英文调整 | 原因 |
|------|---------|---------|------|
| H0 封面标题 | 48-56px | 44-52px | 英文字母视觉比中文汉字小 |
| H1 页面标题 | 28px | 26px | 同上 |
| Body 正文 | 13-14px | 13px | 英文 13px 已够可读 |
| 行高 | 1.8 | 1.6 | 英文不需那么大的行高 |
| 字间距 | 默认 | `letter-spacing: 0.01em` | 微调提升可读性 |

英文字体推荐栈：
```css
font-family: 'Inter', 'SF Pro Display', 'Segoe UI', system-ui, -apple-system, sans-serif;
```

### 日韩语言

| 语言 | 字体推荐 |
|------|---------|
| 日文 | `'Hiragino Kaku Gothic ProN', 'Yu Gothic', 'Meiryo', system-ui, sans-serif` |
| 韩文 | `'Apple SD Gothic Neo', 'Malgun Gothic', 'Nanum Gothic', system-ui, sans-serif` |

排版规则与中文相同（CJK 字符共享类似的排版特征）。
