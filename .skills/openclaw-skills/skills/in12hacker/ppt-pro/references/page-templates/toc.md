# 目录页 -- 演讲的地图俯瞰

> 目录页不是"白纸上列了几行字"。它是整场演讲的**空间全景** -- 让观众在正式深入之前，先从高空俯瞰整个旅程的轮廓。

## 目录页的灵魂

目录页的核心使命是**让观众在 3 秒内理解整场演讲的结构**。它传达的不是内容而是"路线图" -- 观众看完目录应该知道"接下来会经历几个阶段，大概讲什么"。

**一个好目录页只需要做到一件事**：
让演讲的结构一目了然

## 灵动设计指引

### Part 结构展示是唯一的"必须"
- 必须清晰展示所有 Part 的编号和标题
- 展示方式完全自由 -- 不一定是列表

### 所有构图手法都是开放的
以下只是灵感种子：
- 可以用不同 accent 色 + 不同大小的色块暗示各 Part 的"重量"（重点章节视觉更大）
- 编号可以超大极淡，作为每个 Part 区域的背景水印
- 可以用渐进式编排 -- 第一个 Part 的展示面积大，后面递减，暗示"渐入佳境"
- 可以给每个 Part 配一个微小的图标/色块/缩略图暗示内容气质
- 也可以极度简洁 -- 一列精致的带编号的标题行就足够了
- 如果 Part 数量少（2-3 个），可以用横向大面积卡片让每个 Part "铺展开"
- 如果 Part 数量多（5+），紧凑的编号列表反而更清爽

### 绝对禁止
- **禁止** 做成纯文字的编号列表 + 居中标题（那是 Word 目录不是 PPT 目录）
- **禁止** 所有 Part 完全等大等权（应该有视觉重量差异暗示主次）

## 唯一硬约束

- 所有颜色通过 CSS 变量引用
- 装饰用真实 DOM 节点
