# Memory System - 产品说明文档

> 让 AI 记住一切，建立持续有效的长期记忆

## 产品概述

Memory System 是 OpenClaw 的核心记忆系统，采用纯文件系统架构，无需数据库，即可实现：

- 📝 **结构化记忆存储** - 四类记忆分类，规范存储
- 🔍 **语义搜索** - 理解你的意图，而非机械匹配
- ⚡ **自动加载** - 会话开始时记忆自动就位
- 💾 **智能持久化** - 上下文接近满时自动保存
- 🔄 **双向反馈** - 纠正和确认同样重要

---

## 核心创新：四类记忆分类

### 为什么需要分类？

传统记忆系统的痛点：
- ❌ 记忆混杂，难以检索
- ❌ 只记"错误"，忘记"正确"
- ❌ 隐私和共享边界模糊
- ❌ 时间久了记忆失效

### 解决方案

| 类型 | 作用域 | 解决的问题 |
|------|--------|-----------|
| **User** | 私密 | 了解"我在跟谁说话" |
| **Feedback** | 私密/团队 | 记住"什么该做、什么不该做" |
| **Project** | 团队 | 跟踪"项目动态和决策" |
| **Reference** | 团队 | 定位"外部资源在哪里" |

---

## 核心创新：Feedback 双向记录

### 传统做法（只记录纠正）

```
用户说：不要mock数据库
AI 记忆：不要mock数据库 ❌

用户后来接受了一个不常规的做法，但AI已经忘记为什么
```

### 我们的做法（双向记录）

```
negative feedback:
不要mock数据库
Why: 上季度mock测试通过了但生产迁移失败
How to apply: 集成测试必须用真实数据库

positive feedback:
接受单PR而非多个小PR
Why: 拆分反而造成不必要的开销
How to apply: 重构类需求优先合并为大PR
```

**效果**：AI 既能避免重复错误，也能复用成功经验。

## 适用场景

### 个人使用
- 记录你的专业背景和技术栈
- 记住你的沟通偏好（简洁/详细）
- 保存临时但重要的决策理由

### 团队协作
- 共享项目进展和决策上下文
- 记录代码规范和最佳实践
- 维护外部工具入口文档

### 长期项目
- 跟踪项目演变历史
- 保留已废弃方案的教训
- 记录 stakeholder 偏好

## 技术特点

### 文件系统优先
- 纯 Markdown 文件，可直接编辑
- Git 版本控制友好
- 无数据库依赖，部署简单

### 智能降级
- 有 Ollama → 语义搜索
- 无 Ollama → 关键词匹配
- 不影响基本功能使用

### 隐私优先
- User 类型强制私密
- Feedback 可选私密/团队
- 明确的边界控制
