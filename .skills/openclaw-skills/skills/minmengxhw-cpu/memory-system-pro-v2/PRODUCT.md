# Memory System Pro

> 让 AI 拥有持续进化的记忆能力

**版本**: 2.0.0 | **发布**: 2026-03-31

---

## 一句话介绍

Memory System Pro 是一个具有**自动记忆整合**能力的 AI 记忆系统。区别于传统记忆系统的"存储-检索"模式，它引入了类似 Claude Code AutoDream 的自动整理机制，让记忆始终保持新鲜、结构化。

---

## 核心创新

### 1. AutoDream：自动记忆整合 🌙

传统记忆系统的痛点：
- ❌ 记忆越积越多，难以维护
- ❌ 过时记忆造成误导
- ❌ 索引混乱，难以检索

AutoDream 的解决方案：
- ✅ **定时自动执行** - 每天 22:00 自动整合
- ✅ **LLM 驱动** - MiniMax M2.7 模型智能分析
- ✅ **自动清理** - 删除过时、重复记忆
- ✅ **索引自动更新** - 保持 MEMORY.md 整洁

### 2. 四类记忆分类

| 类型 | 作用域 | 解决的问题 |
|------|---------|-----------|
| **User** | 私密 | 了解"我在跟谁说话" |
| **Feedback** | 私密/团队 | 记住"什么该做、什么不该做" |
| **Project** | 团队 | 跟踪"项目动态和决策" |
| **Reference** | 团队 | 定位"外部资源在哪里" |

### 3. Feedback 双向记录

传统做法的盲区：
- 只记录"错误"，忘记"正确"
- AI 变得过于保守

我们的做法：
```markdown
# Negative - 避免重复错误
不要mock数据库
Why: mock测试通过了但生产迁移失败

# Positive - 复用成功经验
接受单PR而非多个小PR
Why: 拆分反而造成不必要的开销
```

## 使用场景

### 场景 1：日常对话
```
用户: 我是民盟上海市委宣传部的
AI: (保存到 user 类型)
    记住了，您是民盟上海市委宣传部副部长

用户: 以后播客报告发到播客群就行
AI: (保存到 feedback 类型)
    好的，已记录您的偏好设置
```

### 场景 2：项目协作
```
用户: 我们现在在做民盟史料库项目
AI: (保存到 project 类型)
    已记录。民盟史料库项目，当前阶段...

用户: 报告格式参考这个文档
AI: (保存到 reference 类型)
    已记录，报告格式参考文档位置...
```

### 场景 3：AutoDream 整合
```
(每天 22:00 HEARTBEAT 自动触发)

AI: [AutoDream 开始执行]
    - 扫描了 32 个记忆文件
    - 识别出 3 个可能过时的记忆
    - 更新了 MEMORY.md 索引
    整合完成。
```

## 安装与配置

### 安装
```bash
clawhub install memory-system
```

### 配置
```json
{
  "skills": {
    "memory-system": {
      "memoryDir": "~/.openclaw/workspace/memory",
      "autoDream": {
        "enabled": true,
        "minHours": 24,
        "minSessions": 3
      }
    }
  }
}
```

### HEARTBEAT 集成
已在 HEARTBEAT.md 中自动配置：
- 每天 22:00 检查并执行
- 静默执行，无变化不打扰
- 手动触发：说"执行记忆整合"
