## 任务记录格式

每个任务使用以下格式记录：

```markdown
## [任务 ID] 任务标题

- **创建时间:** YYYY-MM-DD HH:MM
- **任务类型:** 一次性任务 | 持续性任务
- **当前状态:** 未开始 | 执行中 | 已完成 | 已暂停
- **任务描述:** 详细描述任务内容和目标
- **调用系统功能:** 列出使用的 OpenClaw 功能/skill
- **所需权限:** 列出需要的权限（如文件读写、网络访问、执行命令等）
- **依赖软件:** 任务执行所需的外部软件
- **最后更新:** YYYY-MM-DD HH:MM
- **备注:** 其他信息
```

## 自动化规则

**必须遵守**：
1. 任何新任务分配后，必须在 5 分钟内记录到 TASKS.md
2. 每次修改任务后必须更新统计信息
3. 保持任务 ID 自增不重复
4. 更新任务时必须更新"最后更新"时间
5. 任务记录要完整，不得遗漏关键字段

## 任务类型定义

### Cron 任务
- **特征**: 有明确执行时间（一次性或周期性）
- **示例**: "每天早上 9 点推送资讯"
- **记录**: 记录 Cron 表达式、下次执行时间

### Heartbeat 任务
- **特征**: 定期执行（心跳间隔）
- **示例**: "每次心跳检查邮件"
- **记录**: 记录检查频率、状态文件路径

### Subagent 任务
- **特征**: 多步骤、复杂、需要拆解
- **示例**: "调研竞品并写报告"
- **记录**: 记录 subagent 列表、任务拆解

### Task 记录
- **特征**: 持续性、长期追踪
- **示例**: "持续监控股票走势"
- **记录**: 记录追踪目标、更新频率

### Immediate 任务
- **特征**: 单步、立即执行
- **示例**: "查一下天气"
- **记录**: 可选（简单任务可不记录）

## 状态流转

```
未开始 → 执行中 → 已完成
              ↓
           已暂停 → 执行中 → 已完成
```

状态变更时：
1. 更新"当前状态"字段
2. 更新"最后更新"时间
3. 在"备注"中记录变更原因
