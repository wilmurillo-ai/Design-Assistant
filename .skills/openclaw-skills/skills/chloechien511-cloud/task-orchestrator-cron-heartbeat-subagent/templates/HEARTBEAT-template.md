## ⚠️ Important: Configuration Instructions

**When creating heartbeat task, MUST specify**:
- **Executing Agent**: Which agent is responsible for executing heartbeat
- **Sending Account**: Which Feishu account sends the message

**Standard Configuration Command**:
```bash
openclaw cron add \
  --agentId "{AGENT_ID}" \
  --session "{AGENT_ID}" \
  --name "{Task Name}" \
  --cron "*/30 * * * *" \
  --channel feishu \
  --to "user:{YOUR_USER_ID}" \
  --accountId "{ACCOUNT_ID}" \
  --message "Read HEARTBEAT.md and execute check"
```

**Find Agent's Account ID**:
```bash
cat ~/.openclaw/openclaw.json | python3 -c "
import sys, json
d = json.load(sys.stdin)
for b in d.get('bindings', []):
    print(f\"Agent {b['agentId']}: {b['match']['accountId']}\")
"
```

```bash
mkdir -p workspace/memory
```

```bash
openclaw cron add \
  --agentId "{AGENT_ID}" \

  --name "{Task Name}" \
  --cron "*/30 * * * *" \
  --channel feishu \
```

### 6. Enable Heartbeat

Ensure OpenClaw Gateway has configured heartbeat interval (recommended 30 minutes).

## Best Practices

✅ **Recommended**:
- Keep heartbeat tasks lightweight (<2 minutes to complete)
- Use state files to avoid repeated execution
- Set quiet hours to avoid disturbance
- Add pre-check before expensive operations
- Specify agentId and accountId explicitly

❌ **Avoid**:
- Execute time-consuming operations in heartbeat (>5 minutes)
- Repeated checks without state tracking
- Frequent reports without time period distinction
- Unlimited API calls
- Not specifying agentId (defaults to main execution)
- Using isolated session (doesn't inherit channel binding)
