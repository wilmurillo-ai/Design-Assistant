# Changelog

All notable changes to Task Orchestrator will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [2.0.0] - 2026-04-04

### ✨ Added
- Dynamic configuration loading via YAML files
- Deadlock prevention mechanism with three-dimensional monitoring (cost/time/progress)
- Intelligent aggregation rules (majority/unanimous/veto)
- 4-level risk confirmation boundaries (LOW/MEDIUM/HIGH/CRITICAL)
- 6-point checklist for task setup
- Clarification confirmation环节 for ambiguous tasks
- Pipeline selection based on complexity score
- Conflict resolution strategy with role priority

### 🎯 Enhanced
- Task type identification accuracy improved
- Complexity scoring algorithm enhanced with dependency detection
- Risk assessment now includes 4 distinct levels
- Configuration system now supports runtime loading
- Template library expanded with best practices from 17 skills

### 📝 Documentation
- Complete English documentation added
- Quick start guide with step-by-step examples
- Release guide for ClawHub publication
- Utility command reference
- Troubleshooting section

### 🔧 Technical
- Python utility scripts enhanced with new commands
- YAML configuration schema defined
- Task registry JSON structure standardized
- Deadlock prevention thresholds configurable

### 🙏 Acknowledgments
