# Invest Analyzer 详细使用指南

## 概述

Invest Analyzer（投资分析器）是一个基于四流派投资体系的个股深度分析工具。它整合了 [sunheyi6/investTemplate](https://github.com/sunheyi6/investTemplate) V5.5.6 的核心方法论，提供标准化的投资分析流程。

## 四大投资流派

### 🛡️ 纯硬收息型

**特征**：
- 股息率 > 5%
- 连续5年以上派息记录
- 无有息负债或极低
- 经营现金流/净利润 > 100%
- 央国企背景优先

**适用标的**：电信、公用事业、REITs

**分析重点**：派息可持续性

### 🔍 价值发现型

**特征**：
- PE < 10 或 FCF/市值 > 10%
- 轻资产、商业模式清晰
- 周期拐点或低估
- 经营现金流充裕

**适用标的**：消费、制造、服务

**分析重点**：真实盈利能力

### 🚬 烟蒂股

**特征**：
- 净现金 > 市值
- PB < 0.3
- 靠近历史低点
- 无有息负债

**适用标的**：港股小盘、国企

**分析重点**：清算价值、管理人诚信

### 🔗 关联方资源型

**特征**：
- 国企背景
- 母公司资源可量化
- 控股平台折价

**适用标的**：物业、基建、公用事业

**分析重点**：利益输送风险、母公司支持力度

## 实战参考流程（推荐）：从年报 PDF 还原三表 → 计算 FCF

> 这段是我们在中国卫星（600118）实战中走通的“可复用流水线”，目的是把数据链路固定下来，避免因网页反爬/环境变量/解析器差异导致分析不稳。

### 0) 前提：搜索 Key 的放置（稳定，不依赖 shell / launchd）

推荐把百度 AI Search 的 key 放到固定文件路径：

- `~/.openclaw/env/BAIDU_API_KEY`
  - 文件内容可以是纯 key（单行），也可以是 `BAIDU_API_KEY=...` 的 .env 风格。

这样无论是后台 gateway / cron / skill 运行，都不会因为“终端能 echo、服务拿不到”而翻车。

### 1) 用 baidu-search 找到年报 PDF（优先官方披露渠道）

建议优先顺序：
1) 公司 IR（投资者关系）官网（通常提供“标准文本版 PDF”）
2) 交易所/披露平台（深交所/上交所/巨潮资讯等）直链（确定性高）
3) 第三方聚合平台（东方财富/中财网等）仅作为兜底/交叉验证

示例：
```bash
python3 skills/baidu-search/scripts/search.py '{
  "query":"中国卫星 600118 2024 年年度报告 PDF site:cninfo.com.cn",
  "search_recency_filter":"year"
}'
```

拿到类似 `static.cninfo.com.cn/finalpage/...PDF` 的直链后，进入下载。

### 2) 下载 PDF 到本地（建议统一放 /tmp/openclaw）

```bash
mkdir -p /tmp/openclaw
curl -L --fail -o /tmp/openclaw/<公司>_<代码>_<年份>年报.pdf "<PDF直链>"
```

### 3) 用本地解析器“定位页码 → 抓表格”（避免 OCR/摘要缺表）

#### 3.0 【重要经验】先判定是否为“扫描版/图片版 PDF”

如果 PDF 主要由图片构成（几乎抽不到连续文本层），**不要硬解析**。直接按以下策略处理：

- ✅ **放弃该 PDF 的文本抽取流程**（PyMuPDF / 常规解析器会得到空文本）
- ✅ **改从官方渠道下载“标准年报 PDF（含文本层）”** 再进入后续步骤
  - 优先渠道：公司 IR（投资者关系）页面、交易所/披露平台直链（如：巨潮资讯 cninfo、上交所/深交所披露站点）

> 判断方法（快速）：抽取前几页/随机几页，看是否能拿到连续长文本；或检查页面是否“每页 1 张大图”。

年报 PDF 往往是“正文+三表+附注”混在一起。推荐用关键词搜索先定位：

- 合并资产负债表
- 合并利润表
- 合并现金流量表
- 短期借款 / 长期借款 / 一年内到期的非流动负债 / 租赁负债
- 购建固定资产、无形资产和其他长期资产支付的现金（CAPEX）

#### 方案 A：PyMuPDF 解析（快）
如果你安装了 `pymupdf-pdf-parser-clawdbot-skill`：

```bash
cd skills/pymupdf-pdf-parser-clawdbot-skill
python3 ./scripts/pymupdf_parse.py /tmp/openclaw/<年报>.pdf --format both --outroot /tmp/openclaw/pymupdf-output
```

会输出：
- `output.md`
- `output.json`（包含每页 `text`，可程序化检索关键词 → 得到页码）

> 经验：先用 `output.json` 扫关键词拿到页码，再从 PDF 中精确抽取那几页，比“整份 PDF 直接 LLM 总结”稳定很多。

#### 方案 B：直接用 OpenClaw 的 pdf 工具按页抽取（稳）
当你已经知道三表页码后，按页喂给模型提取数值即可。

### 4) invest-analyzer 的关键计算口径（落地）

- **FCF（粗口径）**：`FCF ≈ 经营活动现金流净额 - 购建固定资产/无形资产等支付的现金`
- **净现金（口径需一致）**：`现金及等价物 - 有息负债`
  - 有息负债至少包括：短期借款、长期借款、应付债券、租赁负债（以及“一年内到期的非流动负债”中属于有息的部分）
  - 这部分最好去附注拆清，避免重复计入（例如租赁负债的流动/非流动拆分）

### 5) 输出
按本 README 的《决策输出格式》出报告即可；并在报告末尾注明：
- 年报来源（PDF直链）
- 三表页码（便于复核）

### 5分钟快速排除法

```
□ 地缘政治核查（30秒）
  - 非洲项目占比 > 10%？→ 排除
  - 东南亚冲突区业务？→ 排除

□ 价格位置检查
  - 距52周高点 < 30%？→ 排除（已反弹过高）

□ 负债快速扫描
  - 财务费用 > 5亿（非周期股）？→ 排除

□ 现金覆盖率
  - 现金 < 市值20%？→ 排除（安全边际不足）

□ 盈利连续性
  - 连续2年经营现金流为负？→ 排除
```

### 快速通过信号

- 🏆 财务费用/市值 < 50% → 无有息负债企业
- 🐢🍊 股价距52周低点 < 80% → 标准烟蒂股
- 🟠 价格位置不佳但估值合理 → 试探建仓 + 网格加仓

## FCF估值计算

### 核心公式

```
FCF = 经营现金流 - 维持性CAPEX
```

⚠️ **重要**：股息不扣除！股息是筹资活动现金流，不影响FCF计算。

### 剔除净现金FCF倍数

```
剔除净现金FCF倍数 = (市值 - 净现金) / FCF

其中：
- 净现金 = 现金及等价物 - 有息负债
- 企业价值(EV) = 市值 - 净现金
```

### 估值区间参考

| FCF倍数 | 评级 | 操作建议 |
|---------|------|---------|
| < 3倍 | 🐢🍊极品 | 极限仓位，重仓买入 |
| 3-5倍 | 🐢🍊金龟 | 推荐建仓区间 |
| 5-8倍 | 合理偏低 | 分批建仓 |
| 8-10倍 | 估值合理 | 持有/观望 |
| > 10倍 | 高估 | 卖出/回避 |

### FCF计算防坑指南

1. **股息不扣除**：FCF是经营+投资活动现金流，股息是筹资活动
2. **CAPEX要合理**：账面CAPEX可能含扩张性支出，需调整
3. **防双标**：不能"抵扣负债但不加现金"
4. **交叉验证**：PE和FCF两种方法都算，确保一致

## 试探建仓与网格加仓

### 建仓梯度

| 层级 | 仓位 | 条件 |
|------|------|------|
| 试探 | 3% | 初步通过初筛 |
| 首次 | 5% | 估值合理且价格可接受 |
| 二次 | 8% | 下跌10%后加仓 |
| 重仓 | 12% | 极限点位 |

### 持仓状态标签

- 🟢 正常：正常持有
- 🔵 已回本：股价回到成本价
- 🟡 高位：距低点涨幅>50%
- 🟠 关注：需要重点监控
- 🔴 遗留：历史遗留仓位
