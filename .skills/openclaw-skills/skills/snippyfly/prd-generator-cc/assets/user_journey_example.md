# 用户旅程图说明（User Journey Map Guide）

> **说明**：本文件是用户旅程图的构建指南与结构参考。  
> 在生成 PRD 时，当涉及复杂用户流程时，Claude 应提示用户参考此结构绘制旅程图，或在 PRD 的"关键场景"部分用文字形式描述旅程结构。

---

## 什么是用户旅程图？

用户旅程图（User Journey Map）是一种可视化工具，用于描述用户从产生需求到完成目标的完整路径，包括每个阶段的行为、想法、情绪和痛点。

**核心价值**：
- 帮助团队建立用户共情
- 识别体验断点和优化机会
- 对齐跨职能团队（产品、设计、工程、运营）对用户路径的理解

---

## 标准旅程图结构

一张完整的用户旅程图包含以下维度（横向为阶段，纵向为维度）：

```
┌─────────────────────────────────────────────────────────────┐
│                    [用户旅程图标题]                          │
│           用户角色（Persona）：[角色名称及描述]              │
│           场景（Scenario）：[用户目标/情境描述]              │
└─────────────────────────────────────────────────────────────┘

阶段划分：
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│  阶段 1  │  阶段 2  │  阶段 3  │  阶段 4  │  阶段 5  │
│  [发现]  │  [考虑]  │  [操作]  │  [等待]  │  [结果]  │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│ 用户行为 │          │          │          │          │
│ 用户想法 │          │          │          │          │
│ 情绪曲线 │  😐      │  😊      │  😟      │  😊      │
│ 痛点     │          │          │ ← 关键痛点│          │
│ 机会点   │          │          │          │ ← 优化点  │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

## 具体示例：电商订单查询旅程

**用户角色**：张女士，30岁，职场白领，习惯手机购物，偶尔需要查询历史订单做报销  
**场景**：需要找到3个月前的一笔订单，截图提交报销申请

| 维度 | 产生需求 | 进入平台 | 查找订单 | 获取凭证 | 提交报销 |
|------|----------|----------|----------|----------|----------|
| **用户行为** | 发现需要报销 | 打开 App，找到"我的" | 在订单列表里上滑翻找 | 截图或导出 PDF | 上传至报销系统 |
| **用户想法** | "我上次买的那个东西多少钱来着？" | "订单在哪里找？" | "怎么翻了这么久还没找到…" | "有没有办法直接导出？" | "终于搞定了" |
| **情绪** | 😐 中性 | 😐 中性 | 😟 轻度沮丧 | 😤 较沮丧 | 😊 如释重负 |
| **痛点** | — | 入口不够醒目 | 无时间筛选，需手动翻 | 无导出功能，只能截图 | — |
| **机会点** | — | 在首页增加"我的订单"快捷入口 | 增加日期范围筛选器 | 增加"导出 PDF/CSV"功能 | — |

## 如何在 PRD 中引用旅程图

**方式一：文字描述（Claude 默认方式）**  
在 PRD 第 4 节"用户故事与使用场景"中，按阶段描述用户路径，并标注关键痛点和情绪低谷。

**方式二：推荐用户绘制旅程图**  
当用户场景复杂（跨多渠道、多角色、情绪波动明显）时，Claude 应在 PRD 中添加：

> 💡 **建议**：本功能涉及 [X] 个用户阶段，建议产品团队绘制用户旅程图以直观呈现各阶段体验。  
> 可使用以下工具：  
> - **Miro**（在线协作白板）：https://miro.com  
> - **Figma FigJam**：支持团队实时协作  
> - **Whimsical**：简洁的旅程图模板  
> 
> 绘制时可参考 `assets/user_journey_example.md` 中的结构框架。

## 旅程图的适用场景

**需要绘制旅程图的情况：**
- 功能涉及 3 个以上用户操作阶段
- 跨越线上/线下多个触点
- 涉及多个用户角色的协作流程
- 产品体验问题的根因分析

**不需要绘制的情况：**
- 简单的单步功能（如：修改密码）
- 纯后台管理功能
- 技术性功能（如：配置项调整）
