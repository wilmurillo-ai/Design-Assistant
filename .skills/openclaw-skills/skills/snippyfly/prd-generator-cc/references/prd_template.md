### 5.2.4 [功能四：批量导入]

#### 5.2.4.1 功能描述

通过模板批量导入主数据，并自动触发同步/异步处理和校验。

#### 5.2.4.2 页面说明

原型图：

![附件3 [导入页原型图]]([prototype-image-path]){width=6.5in}

**流程**
1. 下载模板
2. 上传文件
3. 同步/异步处理
4. 查看结果

**规则**
- 文件格式：[xlsx/xls/csv 等]
- 单次导入数据量 ≤ [上限]
- 文件大小 < [上限]
- ≤ [阈值] 条走同步导入
- [阈值+1]-[上限] 条走异步导入，并自动创建任务中心任务
- 超过条数或大小上限时阻断上传，不转异步
- 失败提供“下载失败文件”

**更多配置**
- 文件内重复数据处理：默认“取第一条”，重复行跳过；可配置为“拒绝导入”。
- 与系统已有数据重复处理：默认“拒绝导入”；可配置为“覆盖更新”。
- 失败数据处理：默认“继续导入”，可配置为“导入终止”。
- 失败文件范围：默认“仅导入失败数据”，可配置为“全部数据”。

#### 5.2.4.3 状态流转逻辑

- 同步导入：上传成功 → 文件校验 → 逐行校验 → 写入数据 → 返回导入结果。
- 异步导入：上传成功 → 文件校验 → 创建任务中心任务 → 后台解析导入 → 任务中心展示导入结果。
- 异步任务状态：待处理 / 处理中 / 成功 / 部分成功 / 失败。

#### 5.2.4.4 权限控制

- 仅 [管理员/具备导入权限的角色] 可导入。

#### 5.2.4.5 异常处理规则

- 文件格式、表头、大小、条数任一不合法：阻断上传并提示明确原因。
- 同步导入逐行失败：按失败数据处理策略决定继续或终止。
- 异步导入失败：任务中心展示失败状态，并提供失败文件下载入口。
- 覆盖更新失败：该行记录失败原因，不影响其他行继续导入，除非用户选择导入终止。

#### 5.2.4.6 数据模型结构（字段级说明）

与新增/编辑一致，需说明模板字段与主从明细关系。

#### 5.2.4.7 业务规则（可测试）

- **FR-030**：导入超过同步阈值且不超过上限时自动转异步。
- **FR-031**：上传文件需校验格式、表头一致性、文件大小、条数上限。
- **FR-032**：任一前置校验失败需给出明确错误原因并终止导入。
- **FR-033**：同步导入需逐行校验必填、格式、重复。
- **FR-034**：导入选项支持“继续导入/导入终止”。
- **FR-035**：失败文件支持“仅失败数据/全部数据”两种范围。
- **FR-036**：失败文件在模板末尾追加“导入失败原因”列。
- **FR-037**：失败文件可直接修正后再次导入。
- **FR-038**：异步导入需创建任务中心任务，并沿用同步导入的解析与校验逻辑。

**配置矩阵**

| 维度 | 主标识类型 | 真实性校验 | 明细必填字段 | 格式校验 | 备注 |
|---|---|---|---|---|---|
| [国家/场景 A] | [主类型] | 是/否 | [字段列表] | [规则] | [说明] |
| [国家/场景 B] | [主类型] | 是/否 | [字段列表] | [规则] | [说明] |

**配置项要求**
- 配置必须支持字段展示名、字段顺序、必填条件、默认值、格式规则、长度规则、真实性校验开关。
- 主类型用于列表摘要展示；多明细场景默认展示主类型值。
- 明细字段结构不允许前端硬编码，应由配置驱动。
- 未配置规则时应定义兜底校验策略。

#### 5.3.2 校验策略 / 保存策略

**保存时必须校验（同步阻断）**
- 必填字段缺失
- 格式/长度校验失败
- 配置一致性不满足

**保存时不阻断（异步）**
- 真实性校验
- 第三方校验超时
- 异步失败进入待确认或风险提示状态

#### 5.3.3 删除与审计策略

- 删除采用软删除，保留关键标识、操作人、操作时间与删除原因。
- 已被下游业务引用的数据不允许物理删除。
- 删除后的数据不出现在默认列表、查询结果和业务选择结果中。

#### 5.3.4 非功能约束

- 列表查询 P95 响应时间目标 ≤ [阈值]。
- 同步导入目标处理时间 ≤ [阈值]，超时需提示用户转异步处理。
- 异步导入任务需可在任务中心追踪，任务结果至少保留 [天数]。
- 涉及个人信息、财税信息、支付信息或敏感数据时，传输过程必须使用 HTTPS，导出/下载文件需按租户权限隔离。
- 删除、导入、覆盖更新、校验结果变更必须记录审计日志。
