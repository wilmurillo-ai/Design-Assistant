# 周报输出模板

以下为标准输出格式。根据实际数据填充，结构不变。

```
团队周报 {周数} ({日期范围})
========================================

【部门管理】

  任务：{任务名}
  ─────────────
  - {姓名}：{本周完成} | {用时}h

  - {姓名}：{本周完成} | {用时}h

  任务：{任务名}
  ─────────────
  - {姓名}：{本周完成} | {用时}h

【项目管理】

  {项目名}
  ─────────────
  任务：{任务名}
  - {姓名}：{本周完成} | {用时}h

  - {姓名}：{本周完成} | {用时}h

  任务：{任务名}
  - {姓名}：{本周完成} | {用时}h

========================================
本周总结：

📊 用时统计：
  - {姓名}：{总用时}h（{各任务用时明细}）
  - {姓名}：{总用时}h（{各任务用时明细}）

📋 与上周计划对比：

  ❌ 未达成：
  - {姓名}：{未完成的上周计划项}

  ➕ 新增（上周未计划）：
  - {姓名}：{新增的工作项}

⚠️ 需关注：
  - {风险或未达成项说明}

💡 下周重点：
  - {从各成员下周计划中提取的关键事项}
```

## 正文结构说明

1. **部门管理**：直接按「任务」分组，同一任务下按成员排列，无项目层级
2. **项目管理**：按「项目 > 任务」两层结构，同一项目下按任务分，同一任务下按成员排列
3. 正文每人每条只写本周完成内容和用时，**不写上周计划、不写达成标记**
4. 纵向对比统一放在最后的「📋 与上周计划对比」板块

## 对比板块规则

- **只展示 ❌ 未达成 和 ➕ 新增**，不展示已达成项
- 只显示有 ❌ 或 ➕ 的成员，无差异的成员不出现
- 某个分类下为空则省略该分类
- 首次收集（无上周数据）时标注"首次收集，无上周数据，无法对比"

## 输出要点

1. 两大维度严格分开：部门管理在前，项目管理在后
2. 部门管理按任务分，项目管理按项目 > 任务分
3. 正文简洁：姓名 + 本周完成 + 用时，不夹杂对比
4. 对比聚焦缺失和新增，达成的不显示
5. 总结部分必须有：用时统计、对比、风险项、下周重点
6. 纯文本，不使用 markdown 表格
