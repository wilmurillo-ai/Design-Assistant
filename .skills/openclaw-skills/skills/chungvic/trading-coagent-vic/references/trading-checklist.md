# Trading Checklist - 交易檢查清單

## 訊號評估 (Signal Evaluation)

### 必檢項目

- [ ] 訊號來源可信度
- [ ] 歷史準確率
- [ ] 當前市場狀況
- [ ] 流動性檢查
- [ ] Gas 價格/交易費用
- [ ] 滑點風險

### 風險評分

```
高風險：新 token / 低流動性 / 高波動
中風險：已驗證項目 / 中等流動性
低風險：主流 token / 高流動性 / 穩定
```

## 交易前確認

### 參數核對

- [ ] Token 合約地址正確
- [ ] 交易數量正確
- [ ] 滑點容忍度設置
- [ ] Gas 價格合理
- [ ] 錢包餘額充足
- [ ] 交易方向正確（Buy/Sell）

### 安全檢查

- [ ] 合約已驗證
- [ ] 無 Honeypot 風險
- [ ] 無異常持倉集中
- [ ] 交易對正常運作

## 交易後驗證

### 立即確認

- [ ] 交易哈希有效
- [ ] 交易已確認
- [ ] 代幣已入賬
- [ ] 數量與預期一致

### 後續跟進

- [ ] 價格走勢符合預期
- [ ] 設置止盈/止損
- [ ] 記錄交易結果

## 失敗分析檢查清單

### 常見失敗原因

1. **Gas 相關**
   - [ ] Gas 價格過低
   - [ ] Gas 限制不足
   - [ ] 網絡擁堵

2. **滑點相關**
   - [ ] 滑點設置過低
   - [ ] 價格波動過大
   - [ ] 流動性不足

3. **RPC 相關**
   - [ ] RPC 節點故障
   - [ ] 請求速率限制
   - [ ] 連接超時

4. **合約相關**
   - [ ] 合約暫停交易
   - [ ] 黑名單限制
   - [ ] 交易冷卻時間

5. **錢包相關**
   - [ ] 餘額不足
   - [ ] Nonce 衝突
   - [ ] 簽名錯誤

## 補救行動指南

### 買錯 Token

```
1. 立即評估當前價格
2. 計算虧損幅度
3. IF 虧損 < 10%：立即賣出
4. IF 虧損 10-30%：評估反彈可能
5. IF 虧損 > 30%：考慮持有或止損
```

### 交易失敗

```
1. 檢查失敗原因
2. 調整參數（Gas/滑點）
3. 確認機會仍存在
4. 重新執行交易
```

### 錯失機會

```
1. 分析錯失原因
2. 評估是否追入
3. 尋找類似機會
4. 優化監控邏輯
```
