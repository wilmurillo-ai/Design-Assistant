# Error Code Explanation

This section describes the most common error codes in the CoinW API, including their meanings, possible causes, and recommended resolution steps.

## Common Error Codes

### 500 Internal Server Error (Futures)

```json
{"code": 500, "msg": "Internal server error"}
```

This error usually occurs when the requested API endpoint address is incorrect or when requesting a non-existent endpoint.

**Troubleshooting Suggestions:**

- Please carefully check if the requested API endpoint address is correct.
- Retest the request.
- If the error still exists, please contact customer support.

### 6003 Insufficient Permissions (Futures)

```json
{"code": 6003, "message": "require more permission", "retry": false, "success": false}
```

This error indicates that the current API Key does not have the required permissions enabled.

**Troubleshooting Suggestions:**

1. Confirm that the API Key is correct, activated, and has the necessary permissions enabled.
2. Log in to the CoinW user backend and enter the "API Management" page.
3. Edit the current API Key and enable the corresponding permissions (such as futures trading, read-only, withdrawal, etc.).
4. Save the changes and retest the request.
5. If the problem persists, please contact customer support.

### 402 Missing Parameters (Futures)

```json
{"code": 402, "msg": "param required"}
```

This error indicates that required parameters are missing in the interface request.

**Troubleshooting Suggestions:**

- Please refer to the API documentation for the requested interface to ensure all required fields are correctly filled.
- Check the interface address and retry the request.
- If the problem persists, please contact customer support.

### 29001 Request Frequency Too High (Futures and Spot)

```json
{"code": 29001, "msg": "API access frequently"}
```

Request frequency exceeds the limit threshold of the interface or global limit.

**Troubleshooting Suggestions:**

- Please refer to CoinW's frequency limit instructions and appropriately reduce the request frequency.
- Frequency limit documentation address: [Link]

### -103 User API Error (Spot)

```json
{"code": "-103", "data": null, "msg": "用户api出错", "success": false, "failed": true}
```

This error may be caused by illegal parameters or signature errors.

**Troubleshooting Suggestions:**

- Verify all parameter names and their values.
- Confirm that the API Key and Secret Key are correct and valid.
- Check the request headers and signature logic.
- If there are still issues, please refer to the examples in the API documentation and contact the CoinW support team.

### 6001 Signature Error (Futures)

```json
{"code": 6001, "message": "sign error", "retry": false, "success": false}
```

Signature verification failed, usually due to incorrect signature algorithm or parameter concatenation format.

**Troubleshooting Suggestions:**

- Please refer to the signature example code in the official API documentation.
- Ensure the signature format meets the requirements of the CoinW backend.
- If the problem persists, please contact the CoinW support team.

### 1004 Invalid Session (Futures)

```json
{"result": false, "errorCode": 1004, "errorMsg": "Invalid session"}
```

WebSocket session expired or invalid.

**Troubleshooting Suggestions:**

- Please re-establish the connection or re-authenticate.
- If there are still issues, please contact the CoinW support team.

### -3 API Access Error, Please Contact Official Support (Spot)

```json
{"code": "-3", "data": null, "msg": "API access error, please contact us", "success": false, "failed": true}
```

A general API interface access error.

**Troubleshooting Suggestions:**

- Check if the request structure and parameters comply with the documentation specifications and try again.
- If the problem persists, please contact CoinW support.

### 1 System Error (Futures)

```json
{"code": 1, "msg": "System error"}
```

Platform-level general error.

**Troubleshooting Suggestions:**

- This may be caused by inconsistent incoming parameters with the interface documentation.
- It may also be caused by system maintenance, please try again later.
- If multiple attempts are ineffective, please contact CoinW support.

### 9012 Position Does Not Exist or Has Been Closed (Futures)

```json
{"code": 9012, "msg": "仓位不存在或已平仓"}
```

The position requested for operation does not exist or has been closed.

**Troubleshooting Suggestions:**

- Please confirm whether the incoming position ID is correct. If the position has been closed or never existed, this error will be returned.
- If you are operating through the OpenAPI interface, please confirm whether the position was generated by the user's independent order. Positions generated through the copy trading system cannot be operated through the OpenAPI interface. Attempting to operate will return this error.

### 9111 Trading Suspended (Futures)

```json
{"code": 9111, "msg": "暂停交易"}
```

The contract or trading variety is temporarily closed and cannot be traded.

**Troubleshooting Suggestions:**

- Please follow CoinW's official announcements to confirm whether there are maintenance or delisting notifications.
- Avoid continuing to place orders for this trading variety.
- If necessary, you can cancel orders and close positions.

### 429 Too Many Requests (Futures and Spot)

```json
{"code": 429, "message": "Too Many Requests", "retry": false, "success": false}
```

This error is usually triggered by the gateway and is not necessarily caused by the user's request frequency being too high.

**Troubleshooting Suggestions:**

- If you encounter this error, please pause the request and try again later.
- It is recommended to follow the interface and global frequency limit rules.
- If the problem occurs frequently, please contact CoinW support.

### 6000 API Error (Futures and Spot)

```json
{"code": 6000, "message": "api error", "retry": false, "success": false}
```

This error indicates improper use of the API Key or authentication failure.

**Troubleshooting Suggestions:**

- The common reason is that private interfaces are not signed and authenticated.
- Please refer to the authentication examples in the CoinW API documentation to ensure the correct implementation of the signature process.
- Ensure that the API Key has not been disabled or deleted. If it is invalid, please re-create the Key.
- If the problem still exists, please contact the CoinW support team.

### -200 Business Exception (Spot)

```json
{"code": "-200", "data": null, "msg": "业务异常", "success": false, "failed": true}
```

Business logic exception, often occurs during system updates.

**Troubleshooting Suggestions:**

- Please try again later.
- If the error persists for a long time, please contact customer support.

### 9031 Your current order amount plus your position has exceeded the maximum limit for this contract (Futures Only)

```json
{"code": 9031, "msg": "您当前下单的金额加上您的持仓已经超过您能买这种合约的最大限制"}
```

Please confirm that the order quantity does not exceed the maximum position limit for the contract.

**Troubleshooting Suggestions:**

- You can query the position limit for the specified contract through the "Get all trading variety margin requirements" interface.
- Check the maximum position limit in the "Trading Rules" section on the web page.
