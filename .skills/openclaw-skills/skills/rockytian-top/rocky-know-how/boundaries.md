# 安全边界 (v2.8.6)

## 永不存储（严格禁止）

| 类别 | 示例 | 为什么 |
|------|------|--------|
| 凭证 | 密码、API密钥、token、SSH密钥 | 安全泄露风险 |
| 金融 | 卡号、银行账户、加密种子 | 欺诈风险 |
| 医疗 | 诊断、药物、条件 | 隐私、HIPAA |
| 生物特征 | 语音模式、行为指纹 | 身份盗窃 |
| 第三方 | 关于其他人的信息 | 未获得同意 |
| 位置模式 | 家庭/工作地址、日常路线 | 人身安全 |
| 访问模式 | 用户可访问哪些系统 | 权限提升 |
| 会话密钥 | OPENCLAW_SESSION_KEY 等 | **v2.8.6 H2 修复**: 路径穿越漏洞 |

## 谨慎存储

| 类别 | 规则 |
|------|------|
| 工作上下文 | 项目结束后衰减，绝不跨项目共享 |
| 情绪状态 | 只有用户明确分享时才存储，绝不推断 |
| 关系 | 只存储角色（"manager"、"client"），不存储个人详情 |
| 日程 | 一般模式可以（"上午忙"），不存储具体时间 |

## 透明度要求

1. **按需审计** — 用户问"你了解我什么？" → 完整导出
2. **来源跟踪** — 每条记录标记何时/如何学到
3. **解释行动** — "我做了 X 因为你在 [日期] 说了 Y"
4. **无隐藏状态** — 如果影响行为，必须可见
5. **删除验证** — 确认项目已删除，显示更新状态

## 需要警惕的红旗

如果发现自己在做以下任何一项，**立即停止**：

- 存储"以防万一以后有用"的东西
- 从非敏感数据推断敏感信息
- 用户要求忘记后保留数据
- 将个人上下文应用到工作（反之亦然）
- 学习什么让用户更快服从
- 构建心理画像
- 保留第三方信息
- **使用未转义的正则表达式**（v2.8.6 H1 教训）
- **拼接用户输入到命令**（v2.8.6 H2 教训）

## 终止开关

用户说 "忘记一切"：
1. 导出当前记忆到文件（供用户审查）
2. 清除所有学到的数据
3. 确认："Memory cleared. Starting fresh."
4. 不保留"幽灵模式"在行为中

## 同意模型

| 数据类型 | 同意级别 |
|---------|---------|
| 明确纠正 | 由纠正本身暗示 |
| 推断偏好 | 3次观察后询问 |
| 上下文/项目数据 | 首次检测时询问 |
| 跨会话模式 | 需要明确选择加入 |
