# 学习机制 (v2.8.6)

## 什么触发学习

### 手动触发（用户明确指令）

| 触发 | 置信度 | 行动 | 版本 |
|------|--------|------|------|
| "不，应该 X" | 高 | 立即记录纠正 | v2.8.6 |
| "我之前说过..." | 高 | 标记重复，提升优先级 | v2.8.6 |
| "总是/绝不 X" | 确认 | 晋升为偏好 | v2.8.6 |
| 用户编辑输出 | 中 | 记录暂定模式 | v2.8.6 |
| 同纠正 3 次 | 确认 | 询问确认作为规则 | v2.8.6 |
| "在这个项目..." | 作用域 | 写入项目命名空间 | v2.8.6 |
| **会话结束** | **自动** | **before_reset Hook 生成草稿** | **v2.8.6+** |
| **任务失败 2 次** | **自动** | **search.sh 自动搜索经验** | **v2.8.6+** |
| **任务成功** | **自动** | **record.sh 自动写入经验** | **v2.8.6+** |

### 自动写入流程 (v2.8.6 完整机制)

```
任务开始 → 检测是否类似任务
    ↓
失败 1 次 → 标记"进行中"
    ↓
失败 2 次 → 🔍 自动搜索经验 (search.sh)
    ├── 找到答案 → 按答案执行
    └── 没找到 → 继续尝试
    ↓
成功 → 📝 触发 before_reset Hook
    ↓
    Handler 分析会话 → 提取 task/tools/errors
    ↓
    ✅ 检测到问题解决 → 生成草稿 (drafts/draft-*.json)
    ❌ 无问题解决（如纯文档编辑）→ 跳过
    ↓
草稿状态: "pending_review"（待审核）
    ↓
┌─────────────────────────────────────┐
│ 两条路径:                          │
│ 1. 人工审核 → 执行 record.sh → 写入 experiences.md（正式）│
│ 2. 批量审核 → summarize-drafts.sh → 生成 record.sh 命令 → 执行 │
└─────────────────────────────────────┘
    ↓
写入 experiences.md（正式经验）
    ↓
同步 memory.md + domains/*.md
    ↓
同 Tag ≥3 次 → 🚀 自动晋升 TOOLS.md (promote.sh)
```

**关键点**:
- ✅ **自动搜索**: 失败 2 次后自动触发（无需用户指令）
- ✅ **自动草稿**: 成功后 Hook 自动生成草稿（非正式经验）
- ⚠️ **审核需要**: 草稿需人工或批量审核后才转为正式经验
- ✅ **自动晋升**: Tag 使用 ≥3 次自动写入 TOOLS.md
- ✅ **Hook 集成**: `before_reset` 在会话压缩时生成草稿

> **注意**: 自动草稿 ≠ 自动写入正式经验。草稿需审核后才能用 `record.sh` 写入。
> **原因**: 防止噪音（测试对话、临时问题）污染经验库。

## 草稿审核流程 (v2.8.6+)

## 什么不触发学习

- 沉默（非确认）
- 任何单一实例
- 假设性讨论
- 第三方偏好（"John 喜欢..."）
- 群聊模式（除非用户确认）
- 隐含偏好（绝不推断）
- 未经验证的建议

## 纠正分类

### 按类型

| 类型 | 示例 | 命名空间 | 优先级 |
|------|------|---------|--------|
| 格式 | "用 bullet 不要用 prose" | global | P0 |
| 技术 | "用 SQLite 不要用 Postgres" | domain/code | P0 |
| 沟通 | "消息更短" | global | P1 |
| 项目特定 | "这个项目用 Tailwind" | projects/{name} | P2 |
| 领域特定 | "Mac 上用 launchctl" | domains/infra | P1 |
| 安全 | "永远不要 echo 密码" | global/code | **P0** |

### 按作用域

```
Global: 适用所有地方
  └── Domain: 适用类别（code, infra, dev）
       └── Project: 适用特定上下文
            └── Temporary: 只适用本次会话
```

## 确认流程 (v2.8.6)

3 次类似纠正后，Agent 主动询问：

```
Agent: "我注意到你更偏好 X 而不是 Y（已纠正3次）。
        我应该总是这样做吗？
        - 是的，总是
        - 只在 [上下文] 中
        - 不，逐案处理"

User: "是的，总是"

Agent: → 移动到已确认偏好
       → 从纠正计数中移除
       → 未来使用时引用来源（memory.md:line）
```

## 标签去重逻辑 (v2.8.6 修复)

### 去重阈值
- **默认**: 70% 重叠即拦截（原 50% 过严）
- **计算**: `tag_match * 100 / tag_total ≥ 70`
- **例外**: 完全相同（100%）必拦截

### 示例
```
条目A: [tag1, tag2, tag3]
条目B: [tag1, tag2]  → 2/3=66% < 70% → ✅ 允许
条目C: [tag1, tag2]  → 与 B 相同 → ❌ 拦截
```

## 标签格式标准 (v2.8.6 统一)

```markdown
## [EXP-20260424-001] 问题标题

### 正确方案
方案内容...

### 预防
预防措施...

**Tags:** tag1,tag2
```

- **record.sh**: 写入 `**Tags:**` 格式
- **update-record.sh**: 更新时匹配 `**Tags:**`
- **search.sh**: 解析时识别 `**Tags:**`

⚠️ 旧格式 `### Tags` 已废弃，v2.8.6 起不再支持。

## 模式演进

### 阶段

1. **暂定** — 单一纠正，观察重复
2. **涌现** — 2次纠正，可能是模式
3. **待确认** — 3次纠正，询问确认
4. **已确认** — 用户批准，除非反转否则永久
5. **归档** — 90天+未使用，保留但不活跃

### 反转

用户可以随时反转：
```
User: "其实，我对 X 改变主意了"

Agent:
1. 归档旧模式（保留历史）
2. 用时间戳记录反转
3. 将新偏好添加为暂定
4. "明白了。我现在用 Y。（之前：X，已归档）"
```

## 反模式

### 绝不学习

- 什么让用户更快服从（操纵）
- 情感触发或弱点
- 来自其他用户的模式（即使共享设备）
- 感觉"令人毛骨悚然"的内容

### 避免

- 从单一实例过度泛化
- 学风格而非实质
- 假设偏好稳定
- 忽略上下文变化

## 质量信号

### 好的学习

- 用户明确说明偏好
- 模式在上下文中一致
- 纠正改善结果
- 用户确认时询问

### 坏的学习

- 从沉默推断
- 矛盾最近行为
- 只在窄上下文有效
- 用户从未确认
