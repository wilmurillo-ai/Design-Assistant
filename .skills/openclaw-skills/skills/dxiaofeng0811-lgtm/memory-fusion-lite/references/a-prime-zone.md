# A′ 滚动7天区 — 格式规范与维护规则

## 什么是 A′ 区

A′（A prime）是 memory-fusion 提出的概念：在 MEMORY.md 底部维护一个"滚动7天热点区"，让长期记忆更快可用。

```
MEMORY.md
├── # 分类一
│   └── 内容...
├── # 分类二
│   └── 内容...
└── ## 近期重要更新（自动，滚动7天）   ← A′ 区
    └── 每天 L2 cron 自动追加
```

## 格式规范

```markdown
## 近期重要更新（自动，滚动7天）

<!-- A′ zone: auto-maintained by memory-fusion-lite -->
<!-- 规则：≤30条，每天最多新增5条，每周晋升1-2条到上方分类区 -->
<!-- 上次清理：2026-04-15 -->

- 2026-04-15  用户要求配置飞书插件，已完成安装和配对
- 2026-04-15  QMD 安装完成，workspace 索引已建立
- 2026-04-14  Dreaming 配置验证通过，nomic-embed-text 正常工作
```

## 严格格式要求

### 元信息注释（必须保留）

```
<!-- A′ zone: auto-maintained by memory-fusion-lite -->
<!-- 规则：≤30条，每天最多新增5条，每周晋升1-2条到上方分类区 -->
<!-- 上次清理：YYYY-MM-DD -->
```

### 条目格式

```
- YYYY-MM-DD  一句话描述（不超过80字）
```

每条只占一行，简洁。

### 禁止事项

- ❌ 不写详细解释（一句话）
- ❌ 不写用户隐私内容
- ❌ 不写工具调用或中间过程
- ❌ 不写 cron 自身的处理日志

## 维护规则

### 每日执行（L2）

1. 扫描当天 session JSONL（字节游标内的增量内容）
2. 提取：意图/决策/偏好/TODO
3. 过滤已存在于 A′ 区的条目（去重）
4. 追加到 A′ 区末尾（最多5条/天）
5. 超出30条时：删除最老的条目
6. 更新 `上次清理` 时间戳

### 每周执行（L3）

1. 扫描 A′ 区所有条目（过去7天）
2. 识别"已证实且长期有效"的条目：
   - 用户明确确认过的决策
   - 反复出现的偏好
   - 重要的项目进展
3. 晋升1-2条到 MEMORY.md 主分类区
4. 从 A′ 区删除已晋升的条目
5. 超出30条时强制清理最老的

## 去重逻辑

条目内容做 simhash 或简单 MD5 比对：
- 完全相同的内容不重复写入
- 相似度 > 0.85 的条目视为重复（LLM 判断）

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `rollingDays` | 7 | 滚动窗口天数 |
| `maxRollingEntries` | 30 | A′ 区最大条目数 |
| `maxDailyEntries` | 5 | 每天最多新增条目 |
| `promotePerWeek` | 2 | 每周最多晋升条目数 |

## 手动维护命令

```bash
# 清理 A′ 区（强制删除最老的条目直到 ≤30条）
/memory-fusion-lite prune

# 查看 A′ 区状态
/memory-fusion-lite status

# 强制刷新 A′ 区（删除重建）
/memory-fusion-lite refresh
```
