# 项目管理 行业知识包

## 业务背景
项目团队使用飞书群协调任务、跟进里程碑、处理风险和阻塞问题，群里包含任务进展、问题暴露、决策讨论等信息。

## 重要信息类型

1. **任务进展** — 具体任务的完成状态、进度百分比
2. **里程碑** — 关键节点达成或延期情况
3. **阻塞问题** — 影响进度的依赖、资源缺失、外部阻塞
4. **风险预警** — 可能导致延期或超预算的风险
5. **决策记录** — 团队做出的重要决定、方案选择
6. **资源需求** — 人力、预算、工具的额外需求
7. **变更请求** — 需求变更、范围调整

## AI 判断标准

**应该记录的消息：**
- 包含任务名 + 状态变化（完成/延期/阻塞）
- 提到具体日期 + 里程碑
- 出现"blocked"、"阻塞"、"等待"、"依赖"等关键词
- 包含风险描述 + 影响评估
- 团队做出明确决策
- 提到资源不足或需要协调

**应该忽略的消息：**
- 纯日常打招呼
- 重复的状态同步（无变化内容）
- 无关项目的闲聊

## 信息结构

- task_name: 任务或里程碑名称
- owner: 负责人
- status: 状态（未开始/进行中/完成/延期/阻塞）
- due_date: 计划完成日期
- blocker: 阻塞原因（如有）
- impact: 对整体项目的影响

## 建议模板

1. **进度分析** — 整体项目健康度评估和风险点
2. **阻塞解决** — 针对当前阻塞的解决路径建议
3. **资源调配** — 根据进度情况的人力调配建议
4. **沟通建议** — 向上汇报或跨团队协调的沟通要点
5. **延期应对** — 里程碑延期时的补救和调整建议
