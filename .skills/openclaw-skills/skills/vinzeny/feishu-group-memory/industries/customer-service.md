# 客户服务 行业知识包

## 业务背景
客服团队使用飞书群处理客户问题、投诉、工单跟进，群里包含问题描述、处理进展、解决方案、客户反馈等。

## 重要信息类型

1. **问题工单** — 客户反馈的具体问题、故障、投诉
2. **处理进展** — 问题当前处理状态、负责人、预计解决时间
3. **解决方案** — 已采用的解决方案、临时方案
4. **升级事件** — 需要上级介入、重大投诉、紧急问题
5. **客户满意度** — 客户对处理结果的反馈、评价
6. **重复问题** — 多个客户反映同一问题（产品 Bug 信号）
7. **SLA 风险** — 即将超出响应/解决时限的工单

## AI 判断标准

**应该记录的消息：**
- 描述客户遇到的具体问题
- 包含工单号、问题编号
- 提到处理状态变更（已解决、待排查、升级等）
- 包含"超时"、"紧急"、"投诉"、"不满"等关键词
- 涉及退款、赔偿讨论
- 多人提到同一问题（重复性 Bug）

**应该忽略的消息：**
- 内部打招呼、闲聊
- 已关闭工单的常规确认
- 纯表情/截图无文字说明

## 信息结构

- customer_id: 客户标识
- issue_type: 问题类型（功能异常/使用咨询/投诉/退款等）
- priority: 优先级（P0紧急/P1高/P2中/P3低）
- status: 状态（待处理/处理中/待客户确认/已解决）
- owner: 负责处理的同事
- sla_deadline: SLA 截止时间

## 建议模板

1. **处理优先级** — 根据问题严重程度建议处理顺序
2. **回复话术** — 针对不同类型投诉的安抚和处理话术
3. **升级判断** — 判断是否需要升级，以及升级路径
4. **根因分析** — 重复问题的可能根本原因提示
5. **SLA 预警** — 即将超时的工单提醒和加速建议
