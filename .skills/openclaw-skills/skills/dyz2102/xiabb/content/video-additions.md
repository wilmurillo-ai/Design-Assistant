## 新增段落1：为什么用 LLM 而不是 Whisper（插入到架构讲解后）

[画面：Whisper vs Gemini 对比表]

很多人会问：为什么不用 Whisper？Whisper 是免费的啊，还不需要网络。

我跟你们说说我的亲身体验。我用 MacWhisper 跑 Whisper large-v3-turbo，说了一段中英文混的话。

[屏幕录制：MacWhisper 输出结果，没有标点，有错别字]

你看到了吗？没有标点符号，"帮我"变成了"棒我"，完全没有语义理解。

为什么？因为 Whisper 是一个纯 ASR 模型——它只做声音到文字的机械转换。它不理解你在说什么。

[画面：字幕强调]

**Whisper = 听写机器人，只听不懂**
**Gemini = 一个懂语言的人在帮你记录**

Gemini 是一个大语言模型，它先听懂你说的话，再输出文字。所以它有完美的标点符号，能纠正口音导致的错误，中英文切换也很自然。

而且 Whisper large-v3 要占 3GB 硬盘 + 4GB 内存来跑。ClawBB 整个 app 只有 280KB。

[画面：280KB vs 3GB 的视觉对比]

当然，Whisper 的优势是离线、隐私。以后我们可以加一个离线模式，用 whisper.cpp 的 tiny 模型（75MB）做 fallback。但主力一定是 LLM。

## 新增段落2：国内用户怎么用（插入到结尾前）

[画面：Z 面向镜头]

我知道很多观众是在国内的，用不了 Claude Code，也用不了 Google Gemini。

别慌，有完整的替代方案。

[画面：对比表]

**编程工具替代：**
- Claude Code → **Kimi Code**（月之暗面出的，命令行工具，免费，体验类似）
- 或者用 **Trae**（字节跳动出的免费 IDE）

**语音 API 替代：**
- Google Gemini → **阿里云 Paraformer**（每天 2 小时免费，中文识别极强）
- 或者 **讯飞**（新用户 10 万次免费）

代码改动很小，就是把 API 地址和认证方式换一下。

[画面：GitHub 链接]

我在 GitHub 上会放一个国内版的配置指南，你们可以直接参考。
