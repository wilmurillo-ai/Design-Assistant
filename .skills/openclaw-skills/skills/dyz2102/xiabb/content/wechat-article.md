# 虾BB：我用AI一晚上做了个免费语音输入神器，然后开源了

---

## 你有没有遇到过这种情况

你对着Mac说一句话——

"帮我写一封email，subject是 project update，内容说明一下这周的进度。"

然后Siri给你返回的是：

"好的，我帮你在Safari里搜索 'email project update 这周的进度'。"

……

我当时的表情，大概就是那种盯着屏幕三秒钟、深呼吸、然后默默关掉Siri的那种。

## 语音输入这件事，一直是个谎言

语音输入这个功能，在科技公司的PPT里出现了至少二十年了。

每次新品发布会，总有人拿出手机说："你看，我只需要说……"然后说出一句完美的普通话，完美地被识别，完美地变成文字。

但我们真实的使用场景是什么？

是中英文混着说，是带着口音，是在嘈杂的咖啡厅里，是说到一半改了主意。

**现有的语音输入工具，基本上是为一种假想用户设计的——那个人说话标准、只讲一种语言、思路永远清晰。**

我不是那种人。

我是一个创业者，脑子里永远有七件事同时在转。我说话中英夹杂，想到哪说到哪。我的工作语言既不是纯中文也不是纯英文，而是一种只有我自己和身边几个人才能听懂的混合体。

我试过Siri——中英混的识别率，呵呵。

我试过Whisper本地跑——延迟高，而且每次都要打开终端，谁受得了。

我试过某几个付费工具——订阅费每月$15，结果识别质量还不如Gemini随手一调。

我的结论是：**要么将就，要么自己做。**

---

## 凌晨一点，灵感来了

那天晚上我在查Google Gemini API的文档，顺手试了一下语音识别的接口。

我发给它一段混着中英文的录音，它干净利落地还给我一段文字，准确率高得让我有点不敢相信。

> 我突然意识到：这玩意是免费的，它能识别中英混杂，它有API。那为什么没有人用它做一个像样的Mac语音输入工具？

答案可能是：因为懒。

或者更准确地说：因为这件事听起来很麻烦。

录音、调API、把文字输出到光标位置——每一步单独拎出来都不算难，但把它们串在一起，还要做成一个好用的小工具，要处理系统权限、要打包成app、要让它在任何输入框里都能工作……

这就是"听起来一晚上能搞定，实际上搞了一周"的那种项目。

我决定试试。

---

## 第一版：Python原型，三小时出来了

先用Python写个原型，验证思路。

逻辑很简单：

1. 监听一个全局快捷键（按住Globe键）
2. 按住时开始录音
3. 松开后把音频发给Gemini API
4. 把返回的文字粘贴到当前光标位置

Python端写起来很快。`sounddevice`录音、`pyperclip`复制粘贴、`pynput`监听按键——三小时原型跑通了。

我说了句"帮我写个todo list，包括buy groceries和回邮件"，它准确地输出了这句话。

我当时高兴得想拍桌子。

然后我尝试把它打包成一个独立的app，分享给朋友用。

然后我遇到了macOS的权限地狱。

---

## macOS权限：一个比老板还难搞的东西

macOS的权限体系，是一套由苹果精心设计的、让开发者痛苦的系统。

麦克风权限、无障碍权限、输入监控权限——每一个单独申请都要在系统偏好设置里手动开启。Python打包出来的app，还要处理代码签名的问题，不签名就会被Gatekeeper拦住，签名要花钱买开发者账号。

更致命的是：Python版本的全局Event Tap（用来监听Globe键）不稳定。macOS在某些情况下会偷偷把Event Tap关掉，完全不通知你，然后快捷键就失灵了。

我调试了大概两天，结论是：**用Python做这件事，是在和macOS掰手腕。**

打包体积也是个问题。一个Python app带上所有依赖，轻轻松松100MB起步。

我想做的是一个轻量小工具，不是一个工程项目。

## 转折点：用Swift重写，Claude Code一个文件搞定

我不是Swift程序员。准确地说，我会写Swift，但我写得很慢，而且对macOS系统API不熟。

但我有Claude Code。

我打开Claude Code，把我的需求描述了一遍：全局Globe键监听、录音、Gemini API、文字注入、系统权限请求、一个悬浮HUD显示状态。

然后我就……基本上开始旁观了。

Claude Code开始写Swift。

它处理了`AVFoundation`录音、`CGEventTap`全局按键监听、`AXUIElement`辅助功能API文字注入——这些我自己查文档要查半天的东西，它一气呵成，还顺手加了错误处理和权限检查。

最终的结果是：**一个Swift文件，1700行，覆盖了所有核心功能。**

> 我第一次有一种感觉：AI不是在帮我写代码，而是在替我把一个"脑子里有但手太慢"的东西，直接具现出来。

当然，中间也遇到了坑。

## 那些让我一度怀疑人生的坑

**坑一：Event Tap权限丢失**

macOS有一个不成文的规矩：如果你的app没有正确持有辅助功能权限，`CGEventTap`会在某个时机被系统静默禁用。没有报错，没有通知，就是突然不工作了。

解法是加一个保活机制：每隔几秒检查Event Tap是否还活着，死了就重新注册。

**坑二：文字注入的兼容性**

把文字"打"到光标位置，有好几种方式。`NSPasteboard`+模拟粘贴最通用，但有些app会拦截剪贴板操作。`AXUIElement`直接写入最干净，但并不是所有app都支持。

最后做了一个优先级：先试AXUIElement，失败了fallback到粘贴板模拟。

**坑三：API Key放哪里**

第一版我直接把Gemini API Key存在UserDefaults里，明文。

后来想了想，不对，用户的Key不能这么存。改成了macOS Keychain，虽然代码多了几十行，但至少睡得着觉。

**坑四：HUD设计**

第一版的HUD是一个红点。

朋友看了之后说："这是在录音还是在报警？"

然后我换成了一个麦克风图标，加了声波动画，加了识别中的旋转指示器。

再后来，灵光一现，换成了一只小龙虾举着麦克风的图标。

因为这个工具叫虾BB。虾当然要出镜。

## 为什么叫"虾BB"

这个名字是凌晨三点想到的。

"虾BB"谐音"瞎BB"。

BB这个字在粤语和北方话里，都有"乱说话"的意思。"瞎BB"就是"乱说一气"。

我觉得这个词完美地描述了语音输入的本质：**你不用想好再说，你直接说，说出来就是输出。**

也完美地描述了我这个创业者的状态：每天满脑子想法，很多都像在瞎说，但偶尔有几句是对的。

> **虾BB的核心哲学：最好的想法，说出来都像在瞎BB。但不说出来，一个都没有。**

所以就叫虾BB了。

## 产品本身，说几个我自己最喜欢的设计

**按住Globe键说话，松开文字出现。**

就这一个交互，没有别的。不用打开任何窗口，不用切换任何界面。在任何输入框、任何app里，按住Globe键，说话，松开，文字就出现在光标位置。

这是我认为语音输入应该有的形态：**它应该是透明的，不应该打断你的工作流。**

---

**280KB。**

整个app，280KB。

我手机里随便一张截图都比它大。

这不是我故意追求小体积，而是Swift写的原生app本来就小。没有Electron，没有Python运行时，没有各种框架——就是一个macOS原生app。

---

**每天250次，免费。**

Gemini API有免费额度，每天250次请求。

对于正常使用来说，250次够了。你一天要说超过250段话才能用完，相当于每小时16次，平均每四分钟一次——这已经是非常重度的使用了。

如果真的不够用，可以在Google AI Studio里申请更高的免费额度，或者付费升级。但大多数人用不到。

**我不想做一个收钱的工具。我想做一个让人用得爽的工具。**

---

**HUD的设计哲学。**

悬浮窗口显示三种状态：

- 🦞🎤 按住录音中，声波动画跳动
- ⏳ 识别中，旋转等待
- ✅ 完成，短暂显示再消失

HUD会自动吸附到屏幕边缘，不挡你的工作区。你可以拖动它到任何位置，它记住位置，下次还在那里。

小细节，但用起来顺手的感觉，就来自这些小细节。

## 为什么要开源

老实说，一开始我没想开源。

这个工具我自己用着挺顺手的，打包一下、写个下载页、交差了事，多简单。

但后来想了想，这个工具能做成什么样，很大程度上取决于有多少人用、多少人提反馈、多少人贡献。

我想要的功能可能和你想要的不一样。有人可能想要热词替换，有人可能想要多语言切换，有人可能想支持其他API提供商。

如果是闭源的，这些都得我一个人做，我做不过来。

开源之后，**它就不是我一个人的瞎BB了，而是一群人的瞎BB**。

> **一个人的瞎BB是噪音，一群人的瞎BB是产品。**

另外，坦白说：我用Claude Code写的代码，让更多人看到、审计、改进，这本身也是一种对AI辅助开发的验证。

如果有人看完代码之后说"这代码写得真好"——谢谢Claude。

如果有人说"这地方有bug"——欢迎提PR。

## 怎么安装

**方法一：下载Release（推荐）**

去GitHub的Releases页面，下载最新的`.dmg`文件，拖进Applications文件夹，打开。

首次打开会要求几个权限：

- 麦克风权限（废话，它要录音）
- 辅助功能权限（用来注入文字到其他app）
- 输入监控权限（用来监听Globe键）

全部允许，然后在设置里填入你的Gemini API Key（免费申请地址：aistudio.google.com），就可以用了。

**方法二：自己编译**

```bash
git clone https://github.com/xiabb/xiabb-app
open xiabb-app/xiaBB.xcodeproj
```

在Xcode里点Build，完事。

需要Xcode 15+，macOS 13+。

## 接下来想做什么

产品功能上，我有几个方向在想：

**多API支持。** Gemini免费够用，但有人可能更信任OpenAI的Whisper，或者用国内的讯飞、腾讯。做成可切换的。

**热词词典。** 有些专业词汇或者人名，语音识别老是识错。允许用户自定义替换规则，说"张三"自动替换成"张三丰"这种。

**识别历史。** 有时候你说完一句话，文字出来了，然后你不小心删掉了，不知道说的是什么。加一个本地历史记录，随时可查。

**Windows版。** 有人问了。我不会Windows开发，但如果有人愿意Port过去，完全欢迎。
