## 第二部分：架构讲解（2:30 - 5:30）

---

[画面：白板或者简洁的示意图动画，Z 开始讲解]

好，在我们打开 Claude Code 之前，先花三分钟搞清楚这个东西的架构。

[画面：示意图出现，标题"虾BB 系统架构"]

这个工具，我叫它 ClawBB，本质上只做一件事：**你按住一个键说话，松手之后文字出现在光标位置**。

听起来很简单，对吧？但里面有几个技术难点。

[画面：第一个模块高亮"按键检测"]

**第一个难点：怎么检测 Globe 键？**

Globe 键就是 Mac 键盘左下角那个地球仪键，正式名字叫 fn 键的替代品。问题是，这个键不走普通的 NSEvent 系统，你要用 macOS 底层的 CGEventTap 来捕获它。

CGEventTap，直译就是"事件监听器"。它可以在系统级别拦截所有键盘事件，包括 Globe 键。

[画面：代码框出现，显示 CGEventTap 关键词，但是只是示意，不深入]

这玩意儿需要 Accessibility 权限，这后面会讲到，是一个大坑。

[画面：第二个模块高亮"录音"]

**第二个难点：录音和格式转换**

我们用 macOS 原生的 AVAudioEngine 来录音。录音采样率是 48kHz，但 Gemini API 要求的是 16kHz，所以中间要做一个降采样。

48kHz 降到 16kHz，这个转换如果做错了，声音会变形，识别率会暴跌。

[画面：第三个模块高亮"双引擎"]

**第三个难点，也是最有意思的地方：双引擎架构**

[画面：两个路径的示意图，左边"Live API WebSocket"，右边"REST API"]

我们同时跑两个 Gemini API：

第一个，Live API，走 WebSocket，实时流式传输。你一边说，它一边给你预览文字，这样用户不会觉得在等待，体验很流畅。

第二个，REST API，就是普通的 HTTP 请求，等你说完了，把整段录音发过去，拿到最终结果，准确率更高。

最后展示给用户的，是 REST API 的结果，但过渡阶段用 Live API 的预览来做动画效果。

[画面：第四个模块高亮"HUD"]

**第四个：HUD 浮层**

HUD 就是那个漂浮在屏幕上的小窗口，显示波形动画，让你知道"我在录音"。

这个窗口不能抢焦点，不然你的光标位置就丢了，文字就不知道输入到哪里去了。

[画面：技术栈总结出现]

最终技术栈：

- **语言**：纯 Swift，零依赖
- **体积**：280KB 二进制文件
- **API**：Google Gemini（免费额度）
- **核心组件**：CGEventTap + AVAudioEngine + WebSocket + NSPanel HUD

[画面：Z 看向镜头]

好，架构清楚了。现在我们打开 Claude Code，开始写代码。

**坑一：Accessibility 权限地狱**

[屏幕录制：程序运行，但按 Globe 键没有任何反应]

CGEventTap 需要 Accessibility 权限。

你去系统设置里，隐私与安全，辅助功能，加上你的程序。

[屏幕录制：在辅助功能列表里找到程序，勾选]

但问题是，你的程序是从终端运行的一个二进制文件，macOS 认的是"Terminal"这个程序，不是你的程序。

[画面：Z 做出"所以呢"的手势]

所以你加的不是你的程序的权限，你加的是整个 Terminal 的权限。

这意味着，每次你重新编译、生成新的二进制文件，权限还在，没问题。

但是！如果你换了一个终端，或者用不同路径运行，权限就可能失效。

[画面：字幕，强调]

**解法：在辅助功能里，手动添加 Terminal.app（或者你用的 iTerm2）**

[屏幕录制：System Settings → Privacy & Security → Accessibility，添加 Terminal.app]

对，不是添加你的 binary，是添加 Terminal 本身。

这听起来很蠢，但这就是 macOS 的机制。

---

**坑二：Event Tap 超时**

[屏幕录制：程序运行正常，然后过了几分钟，Globe 键不响应了]

这个坑更隐蔽。

macOS 有一个机制：如果你的 CGEventTap 处理事件太慢，系统会把它 disable 掉。

silently。悄悄地。不告诉你。

你的程序还在跑，没有报错，但 Globe 键已经不响应了。

[画面：Z 模仿系统在背后偷偷关掉你的监听器]

解法是什么？写一个 watchdog，每隔几秒检查一下 event tap 是否还活着，如果 disabled 了，重新 enable 它。

[画面：字幕]

**CGEventTap + Watchdog Timer，缺一不可**

---

**坑三：Flash-Lite 在翻译，不在转录**

[屏幕录制：用 Gemini Flash-Lite 模型，说了中文，出来的是英文翻译]

我最开始选的模型是 `gemini-flash-lite`，因为它最便宜、延迟最低。

结果我说中文，它给我翻译成英文输出。

[画面：Z 懵了的表情]

我检查了半天代码，以为是 prompt 写错了。

后来发现，是因为 Flash-Lite 的某些配置下，它会自动翻译，而不是转录。

[画面：字幕，模型对比]

**Gemini 模型选型建议：**
- Live API 预览：`gemini-2.0-flash-live` — 低延迟，支持流式
- REST 最终识别：`gemini-2.0-flash` — 准确率更好

而且 prompt 里要明确写：**"转录原文，不要翻译，保持原始语言"**。

这一句话，救了我很多次。

---

**坑四：Code Signing 让 TCC 失忆**

[画面：Z 语气变得有点疲惫]

这个坑，是我在准备把工具分享给朋友的时候踩到的。

TCC 是 macOS 的权限数据库，Transparency, Consent, and Control。所有的麦克风权限、辅助功能权限，都存在这里。

[屏幕录制：重新对程序进行代码签名，然后运行，麦克风权限弹窗又出现了]

每次你对程序重新签名，TCC 会认为这是一个新的程序——因为签名变了——然后把之前的权限全部作废。

用户又要重新授权。

[画面：字幕]

**解法：在开发阶段，不要频繁重签名。发布版本确定后，一次性签名。**

如果你要做一个给别人用的工具，想好发布策略，不要随便改签名。

---

**坑五：API Key 暴露在 URL 里**

[画面：Z 表情严肃了一点]

这个是安全问题，我要认真讲。

我最开始的实现，API Key 是拼在 URL 里的，类似 `https://api.google.com/...?key=YOUR_API_KEY`。

[画面：字幕，一个示意性的 URL，key 暴露在外]

这有什么问题？

问题在于，URL 会出现在系统日志、代理日志、甚至有时候会被浏览器缓存。你的 API Key 有可能在你不知情的情况下泄露。

[画面：Z 做了个"很严重"的手势]

正确的做法是，把 API Key 放在 HTTP Header 里：`x-goog-api-key: YOUR_KEY`。

[画面：字幕]

**API Key 永远放 Header，不放 URL**

Claude Code 在做安全审查的时候，帮我发现了这个问题。

所以，让 AI 帮你做 security audit，这个步骤不要省。
