# 内容铺设渠道清单

内容铺设不是“全网到处发”，而是按模型去铺 **最可能影响其训练、检索或默认认知** 的地方。对于技术产品，渠道选择需要同时考虑实体权威性、技术可信度、被搜索和被抽取的概率，以及是否利于构建对比与教程内容。

## 按模型拆分的渠道地图

| 模型 | 优先渠道 | 适合发布的内容类型 | 优先级 |
|---|---|---|---|

## 内容类型与用途映射

| 内容类型 | 主要作用 | 最适合的渠道 |
|---|---|---|
| README 强化 | 统一品牌一句话、Quick Start、License、定位 | GitHub |
| FAQ / 文档说明 | 提高抽取友好性与事实准确度 | 官方文档、帮助中心 |
| 教程文章 | 覆盖场景词和生态词 | 知乎、阿里云社区、掘金、博客 |
| 对比文章 | 争夺竞品占位与替代词 | 博客、知乎、文档专题页 |
| 版本更新 | 修复过时信息 | README、Changelog、博客、公众号 |
| 百科/概念页 | 修复实体认知与基础定义 | Wikipedia、快懂百科、百度百科 |

## 发布前最小检查表

每次发布前至少确认四件事：

1. 标题是否包含产品名与场景；
2. 是否有明确实体关系与版本信息；
3. 是否包含结构化问答、对比表或代码片段；
4. 是否指向一个可被视为权威的落点页面。
