# ai-visibility-toolkit v0.2.4

This release tightens the ClawHub packaging surface so the public skill is clearer about permissions, prerequisites, and optional credentials.

## Highlights

- Reduced the root skill to a read-only router and moved shell-oriented guidance into `visibility-monitor`.
- Declared optional environment variables and required local binaries with `metadata.openclaw` so scanner-visible metadata matches the repository behavior.
- Added explicit safety guidance: use quickstart or manual paste mode first, inspect scripts before execution, and keep provider keys in local environment variables.
- Aligned root and sub-skill manifest versions to `0.2.4` for the republish.
