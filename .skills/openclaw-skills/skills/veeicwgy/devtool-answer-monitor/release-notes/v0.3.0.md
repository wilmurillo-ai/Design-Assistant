# devtool-answer-monitor v0.3.0

This release repositions the project around a clearer public name, a zero-install demo, and public benchmark stories aimed at developer-tool teams.

## Highlights

- Renamed the public project brand to **DevTool Answer Monitor** to avoid name collisions and make the audience clearer on GitHub.
- Added a zero-install static demo viewer powered by committed sample-run artifacts under `docs/`.
- Added public benchmark stories for **MinerU** and **Sciverse API** with concrete metric deltas and repair takeaways.
- Promoted a new preferred CLI entry point: `python -m devtool_answer_monitor` while keeping `python -m ai_visibility` and `python -m geo_monitor` as compatibility aliases.
- Tightened repository positioning around developer-tool answer monitoring, activation gaps, and agent-readiness instead of broad GEO wording.
