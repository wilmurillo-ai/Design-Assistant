# Publication Readiness

A publishable version must:
- preserve the mainline clearly
- avoid private/local leakage
- make honest claims
- show that the bridge mechanism is general rather than language-by-language hard-coding

## Release checklist

### The skill is publishable only if:
- the mainline is clear in `SKILL.md`
- support notes stay lightweight and subordinate to the mainline
- no local/private environment detail is embedded in the skill package
- claims match actual evidence
- the skill reads like an execution aid, not a research diary

### The skill is not yet publishable if:
- it still depends on environment-specific examples to make sense
- support references compete with the mainline instead of supporting it
- it over-claims retrieval improvements that have not yet been evidenced
- it drifts back toward per-language rule packs or example sprawl

If a file does not help that, trim it.
