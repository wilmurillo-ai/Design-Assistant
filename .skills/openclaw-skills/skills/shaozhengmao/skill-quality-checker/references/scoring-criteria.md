# 📏 Skill 质量评分标准详细说明

本文档是 `check_quality.py` 的评分依据，也供人类审查者参考。

## 总览

| 维度 | 分值 | 权重 |
|------|------|------|
| 问题-方案匹配度 | 0-20 | 20% |
| 完成度 | 0-20 | 20% |
| 容错性 | 0-20 | 20% |
| Description 精度 | 0-20 | 20% |
| Token 效率 | 0-20 | 20% |

满分 100，评级标准：
- ⭐⭐⭐⭐⭐ 优秀：90-100
- ⭐⭐⭐⭐ 良好：75-89
- ⭐⭐⭐ 合格：60-74
- ⭐⭐ 较差：40-59
- ⭐ 很差：0-39

---

## 1. 问题-方案匹配度（20分）

**核心思想**：skill 解决的问题类型，应该决定它的实现方式。

### 任务类型判断
通过 description 中的关键词分类：
- **执行/操作类**（关键词：自动化、脚本、执行、运行、部署、扫描、生成、创建文件、API调用、浏览器操作）
  → 应有 scripts/ 目录
- **知识/指导类**（关键词：指南、规范、最佳实践、格式、模板、流程说明、规则）
  → 纯 Prompt 即可

### 评分规则
| 情况 | 分数 | 说明 |
|------|------|------|
| 有 scripts/ 且任务需要脚本 | 18-20 | 完美匹配 |
| 纯 Prompt 且任务只需指导 | 18-20 | 完美匹配 |
| 有 scripts/ 但任务不需要 | 12-15 | 过度工程化 |
| 无 scripts/ 但任务需要执行 | 5-10 | 方案不匹配 |

### 示例
- `healthcheck` skill 有脚本做安全扫描 → 匹配 ✅
- `code-git` skill 纯指导 Git 配置 → 匹配 ✅
- 一个"翻译指南" skill 带了空 scripts/ → 过度工程化 ⚠️

---

## 2. 完成度（20分）

**核心思想**：交付物是否完整、可用。

### 检查清单
| 检查项 | 分值 | 判定方法 |
|--------|------|----------|
| SKILL.md 存在且有 YAML 头 | 5 | 检查 `---` 分隔的 frontmatter |
| 脚本文件非空 | 5 | scripts/ 下文件 > 10 字节 |
| 无 TODO/FIXME/placeholder | -3/个 | 正则扫描（最多扣5分） |
| 辅助文件完整 | 5 | references/ 等目录下文件非空 |
| 脚本有 shebang + 可执行权限 | 5 | 检查首行和 file mode |

### 说明
- 如果没有 scripts/ 目录（纯 Prompt 型），脚本相关项不扣分，但总分按比例调整
- TODO 在注释中也算，但 "TODO" 作为任务描述不算

---

## 3. 容错性（20分）

**核心思想**：出错时能否优雅处理，而非直接崩溃。

### Python 脚本检查
- `try/except` 块数量 vs 函数数量
- `ImportError` 处理（可选依赖）
- 文件操作是否有异常处理
- 网络请求是否有超时和重试

### Shell 脚本检查
- `set -e` 或等效错误处理
- 关键命令后的 `if`/`||` 检查
- 变量是否有默认值 `${VAR:-default}`

### 纯 Prompt 型检查
- 是否包含容错指导词：「如果失败」「fallback」「降级」「备选方案」「出错时」
- 有则得 12-16 分，无则 5-10 分

### 覆盖率计算
```
coverage = (错误处理块数) / max(函数数, 1) * 100%
```
| 覆盖率 | 分数 |
|--------|------|
| > 70% | 16-20 |
| 30-70% | 10-15 |
| < 30% | 5-10 |
| 0% | 0-5 |

---

## 4. Description 精度（20分）

**核心思想**：description 是 skill 路由的唯一依据，必须精准。

### 检查清单
| 检查项 | 分值 | 判定方法 |
|--------|------|----------|
| description 存在 | 5 | YAML 中有 description 字段 |
| 长度 50-300 字符 | 5 | len(description) |
| 包含中文触发词 | 5 | 正则匹配中文词 |
| 包含英文触发词 | 5 | 正则匹配英文词 |

### 扣分项
- 包含泛化词（"所有"、"任何"、"任意"、"everything"、"anything"、"all kinds"）→ 每个 -2 分
- 太短（< 30字符）→ -3 分
- 太长（> 500字符）→ -2 分

---

## 5. Token 效率（20分）

**核心思想**：SKILL.md 每次都会被完整加载，越小越省 token。

### 文件大小评分
| SKILL.md 大小 | 基础分 |
|---------------|--------|
| < 2KB | 20 |
| 2-5KB | 15-19 |
| 5-8KB | 10-14 |
| 8-15KB | 5-9 |
| > 15KB | 0-4 |

### 加分项
- 有 references/ 等外部文件（渐进式披露）→ +3 分（上限20）

### 扣分项
- SKILL.md 中有大段重复内容 → -3 分
- SKILL.md 中内联了本可外置的大段数据 → -2 分
