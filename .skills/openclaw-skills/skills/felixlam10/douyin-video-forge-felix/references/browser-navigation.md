# 抖音页面结构参考

本文档供 SKILL.md Phase 2 浏览器导航步骤参考，描述抖音 Web 端的 URL 模式、页面元素和常见异常。

---

## URL 模式

| 场景 | URL | 说明 |
|------|-----|------|
| 热榜 | `https://www.douyin.com/hot` | 抖音热点榜，含热搜、热榜、品牌榜等标签页 |
| 关键词搜索 | `https://www.douyin.com/search/<keyword>?type=video` | `type=video` 限定视频结果；不加则返回综合结果 |
| 用户主页 | `https://www.douyin.com/user/<sec_uid>` | sec_uid 以 `MS4wLj` 开头；页面含 profile 区 + 作品列表 |
| 视频详情页 | `https://www.douyin.com/video/<video_id>` | 含视频播放器、作者信息、点赞/评论/收藏/转发数据、评论区 |

---

## 常见 UI 元素

### 热榜页 (`/hot`)
- 热点列表：排名序号、标题文本、热度值（数字）
- 标签页切换：热榜总榜 / 热搜榜 / 品牌热 DOU 榜等

### 搜索结果页 (`/search/...`)
- 搜索框（顶部，可输入关键词）
- 筛选栏：综合 / 视频 / 用户 / 直播等
- 视频卡片列表，每张卡片包含：
  - 封面缩略图
  - 视频标题
  - 作者名称
  - 点赞数（❤️ 图标旁）
  - 播放量（部分卡片显示）
- 滚动加载更多结果

### 视频详情页 (`/video/...`)
- 视频播放器
- 作者信息区：头像、昵称、粉丝数、关注按钮
- 互动数据：点赞数、评论数、收藏数、转发数
- 评论区：默认展示热门评论，需滚动加载更多
- 相关推荐视频列表

### 用户主页 (`/user/...`)
- Profile 区：头像、昵称、抖音号、简介、粉丝数、关注数、获赞数
- 作品列表：按时间倒序排列的视频卡片
- 标签页：作品 / 喜欢

## 反爬行为与异常处理

| 现象 | 表现 | 处理方式 |
|------|------|----------|
| 验证码弹窗 | 页面弹出滑块验证或点选验证 | 暂停操作，提示运营手动完成验证后继续 |
| 登录弹窗 | 遮罩层要求登录/扫码 | 尝试关闭弹窗；若无法关闭，提示运营登录后继续 |
| 空白页 | 页面加载但无内容 | 等待 5 秒重试；仍空白则换用 web_search 降级 |
| 内容加载失败 | "加载失败，请重试"提示 | 刷新页面重试一次；仍失败则降级 |
| 频率限制 | 连续访问后页面响应变慢或报错 | 步骤间等待 3-5 秒；严重时暂停 30 秒 |

## 数据精度说明

浏览器中看到的数值是**近似值**，不是精确数字：

| 页面显示 | 含义 |
|----------|------|
| `1.2万` | 约 12,000（范围 11,500 - 12,499） |
| `368.5万` | 约 3,685,000 |
| `1亿` | 约 100,000,000 |
| `9999` | 精确值（未达万级缩写阈值） |

**对筛选的影响**：低粉高赞筛选使用近似值即可（例如"粉丝 ≤ 5万"判断不受影响）。无需追求精确数字，浏览器可见的量级足以支撑内容决策。
