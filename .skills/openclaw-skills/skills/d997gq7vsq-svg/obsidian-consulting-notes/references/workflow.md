# 详细操作工作流

---

## 工作流一：全流程整理（`Obsidian MD笔记整理`）

```
输入：用户指定的文件/文件夹路径
  ↓
[Step 1] 扫描并列出所有文件（MD、Word、PDF）
  ↓
[Step 2] 自动筛选
  ├── 保留（含诊断/方案/工具/模板/案例/话术且有实操）→ 进入拆分
  ├── 临时归档（有价值但无法拆分）→ 转MD后存入 临时参考/
  └── 剔除（纯理论/宣传/无实操）→ 跳过，输出报告说明
  ↓
[Step 3] Word/PDF 转 MD（如有）
  ↓
[Step 4] 原子化拆分
  ↓
[Step 5] 命名（前缀-关键词.md）
  ↓
[Step 6] 自动归类（确定目标路径）
  ↓
[Step 7] 打标（写入 YAML frontmatter）
  ↓
[Step 8] 关联提示（写入 ## 关联笔记 模块）
  ↓
[Step 9] 写入文件（write_to_file）
  ↓
输出：操作汇总报告（文件名 → 目标路径，含标签列表）
```

---

## 工作流二：核心拆分（`拆分咨询文档为MD原子笔记`）

适用场景：用户提供一个或多个长文档，需要快速拆分。

```
输入：指定文档路径（支持 .md / .docx / .pdf）
  ↓
[Step 1] 读取文档全文
  ↓
[Step 2] 识别独立知识点（按标题/工具名/案例/话术模板）
  ↓
[Step 3] 列出拟拆分清单，展示给用户确认（每条显示：编号、拟命名、核心内容摘要）
  ↓
  用户确认 → 继续
  ↓
[Step 4] 逐条生成 MD 文件内容（含 YAML frontmatter + 正文 + 关联笔记）
  ↓
[Step 5] 按归类规则确定目标路径（参照 taxonomy.md）
  ↓
[Step 6] 使用 write_to_file 写入各 MD 文件
  ↓
输出：拆分结果报告（共拆分 N 条，每条：文件名 → 路径）
```

### 拆分粒度判断标准

| 情况 | 处理方式 |
|---|---|
| 文档 < 500 字且只有 1 个核心主题 | 不拆分，直接作为一条原子笔记 |
| 文档有 2-3 个独立主题 | 拆分为 2-3 个 MD 文件 |
| 文档 > 2000 字且有多个独立工具/案例 | 按每个工具/案例/话术各拆一条 |
| 文档是纯目录/索引文件 | 跳过拆分，临时归档 |

## 工作流三：清理（`清理Obsidian无效MD笔记`）

```
输入：目标文件夹路径
  ↓
[Step 1] 扫描所有 MD 文件，读取文件名和前200字内容
  ↓
[Step 2] 标记无效文件（空文件 / 纯标题无内容 / 字数 < 50）
  ↓
[Step 3] 识别重复笔记
  规则：文件名去掉前缀后相似度 ≥ 80%，或正文前200字相似度 ≥ 80%
  操作：保留最新修改时间的文件，其余文件名末尾追加 "_重复" 标记
  （使用 replace_in_file 修改文件名中的 YAML title 字段，或直接生成重命名建议）
  ↓
[Step 4] 识别应归临时参考的文件（无法匹配任何一级库关键词）
  ↓
[Step 5] 输出清理报告（分三类：无效/重复/应归档），等待用户确认
  ↓
  用户确认 → 执行重命名/移动操作
  ↓
输出：操作完成报告
```

## 工作流四：补标（`给Obsidian MD笔记补全标签`）

```
输入：目标文件夹路径（可指定单个文件或整个文件夹）
  ↓
[Step 1] 扫描所有 MD 文件，读取 YAML frontmatter
  ↓
[Step 2] 识别缺标文件
  - 无 tags 字段
  - tags 字段为空
  - 缺少一级标签（#问题诊断/#解决方案/#交付输出）
  - 缺少领域标签
  - 缺少辅助标签
  ↓
[Step 3] 对每个缺标文件，读取文件名 + 正文前300字
  ↓
[Step 4] 按 taxonomy.md 中的关键词映射推断三层标签
  ↓
[Step 5] 输出补标预览表（文件名 → 拟添加标签），等待用户确认
  ↓
  用户确认 → 执行写入
  ↓
[Step 6] 使用 replace_in_file 更新每个文件的 YAML frontmatter
  写入格式：
  ---
  tags:
    - "#一级标签"
    - "#领域标签"
    - "#辅助标签"
  ---
  ↓
输出：补标完成报告（共处理 N 个文件）
```

---

## 输出规范：YAML frontmatter 标准格式

```yaml
---
tags:
  - "#问题诊断"
  - "#战略管理"
  - "#可复用工具"
---
```

注意事项：
- 标签值用英文双引号包裹
- 每个标签单独一行，前缀 `- `
- `#` 号写在引号内
- 三层标签严格按顺序：一级库 → 领域 → 辅助

---

## 批量操作安全规则

1. 每批写入不超过 20 个文件；
2. 写入前必须先展示操作清单，等用户确认；
3. 使用 `write_to_file` 新建文件，使用 `replace_in_file` 更新已有文件；
4. 不直接删除任何文件（重复文件仅追加 `_重复` 标记）；
5. 操作完成后输出操作汇总（成功 N 条，跳过 N 条，失败 N 条及原因）。
