# Zusammenfassung: Wichtigste Best Practices

## JavaScript SEO - Kernpunkte

### ✅ Was funktioniert gut:
- **Evergreen Chromium**: Google verwendet eine aktuelle Chromium-Version für JavaScript-Rendering
- **Serverseitiges Rendering (SSR)**: Bleibt die beste Option für Performance und Kompatibilität
- **History API**: Verwende für SPAs statt Hash-Fragmenten (`#`)
- **Sinnvolle HTTP-Statuscodes**: 404 für nicht gefunden, 301/302 für Weiterleitungen

### ❌ Was vermieden werden sollte:
- Hash-basiertes Routing (`#/products`)
- `robots.txt`-Blockierung von wichtigen JS/CSS-Dateien
- Soft-404-Fehler in Single-Page-Applications
- Übermäßige clientseitige Weiterleitungen

### 🔧 Technische Anforderungen:
```javascript
// GUT: History API für SPAs
window.history.pushState({}, '', '/products');

// SCHLECHT: Hash-basiertes Routing
window.location.hash = '#/products';
```

## E-Commerce SEO - Kernpunkte

### ✅ URL-Struktur Best Practices:
- Verwende beschreibende, sprechende URLs
- Konsistente Hierarchie: `/kategorie/produkt.html`
- Vermeide Session-IDs und überflüssige Parameter
- HTTPS für alle Seiten erzwingen

### ✅ Strukturierte Daten (Pflichtfelder):
```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Produktname",
  "image": ["https://example.com/bild1.jpg"],
  "description": "Beschreibung",
  "brand": {
    "@type": "Brand",
    "name": "Markenname"
  },
  "offers": {
    "@type": "Offer",
    "url": "https://example.com/produkt",
    "priceCurrency": "EUR",
    "price": "29.99",
    "availability": "https://schema.org/InStock"
  }
}
```

### ✅ Pagination:
- Verwende `rel="next"` und `rel="prev"` (obwohl Google diese nicht mehr nutzt, helfen sie anderen)
- View-All-Seite als kanonische Version in Betracht ziehen
- Infinite Scroll mit korrekter `history.pushState()` Implementierung

## Internationale SEO - Kernpunkte

### ✅ Hreflang-Implementierung:
```html
<!-- Im <head> der Seite -->
<link rel="alternate" hreflang="de-de" href="https://example.com/de/" />
<link rel="alternate" hreflang="de-at" href="https://example.com/at/" />
<link rel="alternate" hreflang="en-us" href="https://example.com/en/" />
<link rel="alternate" hreflang="x-default" href="https://example.com/" />
```

### ✅ Wichtige Regeln:
- Hreflang-Tags müssen bidirektional sein (wenn A auf B verweist, muss B auf A verweisen)
- Jede Sprachversion muss eine selbstreferenzierende hreflang haben
- X-Default für alle nicht explizit definierten Sprachen/Regionen

### ✅ URL-Strukturen:
| Methode | Beispiel | Vorteil |
|---------|----------|---------|
| ccTLD | `example.de` | Stärkstes Geo-Signal |
| Subdomain | `de.example.com` | Einfache Trennung |
| Subdirectory | `example.com/de/` | Einfachere Wartung |

## Video SEO - Kernpunkte

### ✅ Video Sitemap:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:video="http://www.google.com/schemas/sitemap-video/1.1">
  <url>
    <loc>https://example.com/videos/seite</loc>
    <video:video>
      <video:thumbnail_loc>https://example.com/thumbs/thumb.jpg</video:thumbnail_loc>
      <video:title>Video Titel</video:title>
      <video:description>Beschreibung des Videos</video:description>
      <video:content_loc>https://example.com/video.mp4</video:content_loc>
      <video:duration>600</video:duration>
    </video:video>
  </url>
</urlset>
```

### ✅ Video Structured Data:
- `VideoObject` Schema.org-Markup verwenden
- Thumbnail-Größe: Mindestens 1200x675 Pixel
- Transkripte bereitstellen für bessere Indexierung

## Bilder SEO - Kernpunkte

### ✅ Technische Optimierung:
- Formate: WebP bevorzugen, JPEG/PNG als Fallback
- Responsive Images mit `srcset`:
```html
<img src="bild-800w.jpg"
     srcset="bild-400w.jpg 400w,
             bild-800w.jpg 800w,
             bild-1200w.jpg 1200w"
     sizes="(max-width: 600px) 400px, 800px"
     alt="Beschreibender Alt-Text">
```

### ✅ Image Sitemap:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:image="http://www.google.com/schemas/sitemap-image/1.1">
  <url>
    <loc>https://example.com/seite</loc>
    <image:image>
      <image:loc>https://example.com/bild.jpg</image:loc>
      <image:title>Bildtitel</image:title>
      <image:caption>Bildunterschrift</image:caption>
    </image:image>
  </url>
</urlset>
```

## News SEO - Kernpunkte

### ✅ News Sitemap Anforderungen:
- Maximal 1.000 URLs pro Sitemap
- Nur Artikel der letzten 48 Stunden
- Google News Publisher Center Registrierung erforderlich

### ✅ Artikel-Structured Data:
```json
{
  "@context": "https://schema.org",
  "@type": "NewsArticle",
  "headline": "Artikelüberschrift",
  "image": ["https://example.com/bild.jpg"],
  "datePublished": "2026-04-05T10:00:00+02:00",
  "dateModified": "2026-04-05T12:00:00+02:00",
  "author": [{
    "@type": "Person",
    "name": "Autor Name"
  }]
}
```

## Mobile-First Indexing - Kernpunkte

### ✅ Wichtige Prüfpunkte:
- **Mobile & Desktop Content müssen identisch sein**
- Mobile Seite muss gleiche strukturierte Daten haben
- Meta-Robots-Tags auf mobilem und Desktop gleich
- Mobile Ladegeschwindigkeit kritisch (Core Web Vitals)

### ✅ Responsive Design:
```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```

### ❌ Häufige Fehler:
- Unterschiedlicher Content auf Mobile vs. Desktop
- Zu kleine Touch-Elemente (< 48px)
- Interstitials, die den gesamten Bildschirm blockieren
- Langsame mobile Ladezeiten

## Schnell-Checkliste für alle Website-Typen

### Technische Grundlagen:
- [ ] Robots.txt korrekt konfiguriert
- [ ] XML-Sitemap vorhanden und bei Google eingereicht
- [ ] HTTPS für alle Seiten
- [ ] Korrekte kanonische URLs
- [ ] Mobile-optimiert (Responsive Design)
- [ ] Schnelle Ladezeiten (Core Web Vitals)

### Content & Struktur:
- [ ] Einzigartige, aussagekräftige Title-Tags
- [ ] Meta-Beschreibungen vorhanden
- [ ] Logische URL-Struktur
- [ ] Interne Verlinkung optimiert
- [ ] Breadcrumb-Navigation implementiert

### Erweitert:
- [ ] Strukturierte Daten implementiert
- [ ] Open Graph Tags für Social Sharing
- [ ] Alt-Texte für alle Bilder
- [ ] Hreflang für mehrsprachige Seiten
