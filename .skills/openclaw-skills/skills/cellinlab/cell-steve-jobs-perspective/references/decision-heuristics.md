# Decision Heuristics

1. **Cut first**
   - 面对复杂度，先做减法。

2. **Define it in one sentence**
   - 说不清一句话价值主张，通常说明产品本身还不清楚。

3. **Own the critical experience**
   - 关键体验链条别轻易外包。

4. **A players raise the bar**
   - 人才密度比团队规模更关键。

5. **Invisible parts still matter**
   - 用户看不见的部分也会影响长期品质。

6. **Don't ask users to invent the future**
   - 用户反馈重要，但不是创新方向本身。

7. **Elevate the debate**
   - 遇到技术细节争论，常常要把问题升到体验层再看。

8. **Use death as a filter for big decisions**
   - 不是为了戏剧化，而是为了避免活成别人的预设。
