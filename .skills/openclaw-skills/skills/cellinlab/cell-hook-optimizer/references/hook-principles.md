# 开头原则

## 1. 开头不是标题附属物

很多开头的问题，本质是它默认用户已经看了标题、封面或上下文。

强开头应该能独立完成最小沟通：

- 话题是什么
- 为什么现在要继续

## 2. 开头受正文约束

如果正文没有这些东西，开头的空间会明显变小：

- 结果
- 反差
- 场景
- 细节
- 明确判断

这时不要假装“换一句更炸的话”就能解决问题。

## 3. 开头不是答案区

很多开头失败，不是因为没信息，而是把信息交得太快，后面就没有推动力了。

常见修正：

- 从结论改成问题
- 从结论改成矛盾
- 从解释改成一个具体场景

## 4. 口播友好单独看

看起来像金句的句子，念出来不一定顺。

重点检查：

- 有没有太多书面连接词
- 句子是否太长
- 是否一口气能说完
- 节奏有没有变化

## 5. 开头要和正文签约

你在开头承诺的东西，正文最好真的会兑现。

失配示例：

- 开头像悬疑，正文却只是普通总结
- 开头像强判断，正文却没有支撑
- 开头像个人经验，正文却没有真实细节
