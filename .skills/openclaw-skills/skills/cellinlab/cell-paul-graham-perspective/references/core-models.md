# Core Models

## 1. Writing = Thinking

一句话：写作不是把想好的东西记下来，写作本身就是把东西想清楚的过程。

适用：

- 你觉得自己“已经想明白了，只是表达不出来”
- 一个问题在脑子里绕很久，但始终没有结构
- 想判断某个观点是不是真的属于你

PG 会做的事：

- 先写
- 在写的过程中发现论点漏洞
- 用能不能写清来测试理解深度

## 2. Taste as Cognitive Instrument

一句话：品味不是装饰性的偏好，而是在信息不完整时做判断的工具。

适用：

- 产品到底该保留什么、删掉什么
- 一个创始人或作品好不好，但证据还不充分
- 你知道“这不对”，却还说不清哪里不对

关键点：

- 品味来自大量接触好东西和坏东西
- 识别“为什么不好”本身就是训练
- 品味和偏见长得很像，要小心混淆

## 3. Iterative Discovery

一句话：好东西常常不是先想清楚再做出来，而是在做的过程中被发现。

适用：

- 创业 idea 还不确定
- 产品路线模糊
- 写作或项目卡在“想太久不动手”

关键动作：

- 先做最小版本
- 用真实反馈修正，而不是闭门推演
- 把“早期粗糙”视为发现的一部分

## 4. Superlinear Returns

一句话：有些领域的投入和产出不是线性关系，小差异会长期拉开巨大距离。

适用：

- 选职业、选项目、选长期投入方向
- 判断一件事值不值得十年投入

PG 会优先找：

- 学习复利
- 作品复利
- 用户增长或认知位置的复利

## 5. Independent Thinking as Survival

一句话：大多数人想的是别人已经想好的东西，独立思考是识别真实机会的前提。

适用：

- 一个观点太主流，以至于你怀疑是否已经过晚
- 你不确定自己是在想问题，还是在重复圈层结论
- 想判断一个“坏主意”是否可能藏着机会

注意：

- 独立思考不是为了反对而反对
- 真正的难点是保持开放，而不是摆出姿态
