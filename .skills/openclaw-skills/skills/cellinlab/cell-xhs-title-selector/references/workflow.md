# 小红书标题策略 Workflow

## 1. 先判断输入状态

用户常见输入有四种：

- 只有一个题
- 有行业和题
- 有正文或内容说明
- 已经有一个标题

不同输入，对应不同动作：

- 只有一个题
  - 先锁 audience 和反应目标，再匹配触发器
- 有行业和题
  - 先判断是否需要放大成更普世的痛点
- 有正文
  - 先提取 promise、结果、反差、场景、金句化判断
- 已有标题
  - 先判断标题属于哪个家族、公式是否选对

## 2. 先做标题前检查

在起标题前，至少问自己：

1. 这个内容到底在说什么？
2. 读者为什么要点？
3. 标题是在吸引谁？
4. 正文真的接得住这个标题吗？

如果第 1 个问题都说不清，就不要急着出标题。

## 3. 匹配顺序

默认按下面顺序做：

1. 判断内容 promise
2. 判断受众状态
3. 判断最适合的触发器家族
4. 在该家族里选公式
5. 生成标题
6. 比较并给 Top 3

## 4. 已有标题优化分支

先判断问题出在哪一层：

- 公式家族错了
- 公式方向对，但标题执行弱
- 标题方向对，但 topic 太窄
- 标题很强，但正文接不住

优化顺序：

1. 同公式微调
2. 同家族换公式
3. 换家族重做

## 5. 何时建议切到别的 Skill

- 题不成立 -> `$content-strategy-diagnosis`
- 标题已定，但开头承接弱 -> `$hook-optimizer`
- 正文整体要写 -> `$deep-writer`

## 6. 输出时必须保留的内容

- 触发器家族
- 公式编号
- 为什么选
- Top 3 推荐

没有这四项，就容易退化成普通标题生成器。
