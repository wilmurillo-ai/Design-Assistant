# Writing Workshop

> 在场景 B 和场景 C 使用：起草短帖 / Thread，或审阅现有内容。

## 短帖的三种高频任务

### 1. 观点帖

适合：

- 你有明确判断
- 想筛选认同者
- 想引发讨论

结构：

- 先给判断
- 再给一个具体锚点
- 最后留一个张力或问题

### 2. 结果帖

适合：

- 上线、实验、里程碑
- 节省了多少时间
- 获得了什么反馈

结构：

- 结果
- 过程里最意外的点
- 为什么它值得被别人关注

### 3. 方法帖

适合：

- 你真的总结出一套做法
- 可以用 3-5 个点讲清

结构：

- 问题
- 方法
- 一个最关键提醒

## Hook Families

常用的五类起手：

1. **结果锚点**
   - 适合有数字、有里程碑的内容
2. **反常识判断**
   - 适合有明确 stance 的内容
3. **具体场景**
   - 适合把抽象观点压回现实
4. **错误纠正**
   - 适合指出误区与反模式
5. **张力问题**
   - 适合引发回复和对话

好 hook 至少满足两项：

- 有具体性
- 有立场
- 有悬念
- 有可信度锚点

## Thread 基础结构

默认四段：

1. **Open**
   - 说明这是关于什么的
   - 读者为什么要继续
2. **Build**
   - 3-7 个要点
   - 每段都要推进，而不是换说法重复
3. **Compress**
   - 用 1-2 条把前文收束
4. **Next Step**
   - CTA、问题、后续动作都可以

## 格式规则

- 每条尽量单段或短段
- 一屏内能扫懂核心意思
- 如果是英文 X，避免中文长句直译
- 不要为了“像 X”故意把句子切得支离破碎

## 英文 X 写法提醒

- 优先自然、清楚、可读
- 少用空泛形容词
- 结果、动作、对象比抽象名词更重要
- 一条英文帖里只保留一个中心判断

## 改写时的两条路径

### 同路径优化

原方向对，但执行差：

- hook 更具体
- 节奏更紧
- 句子更短
- 可信度更清晰

### 换路径重写

原方向本身就不对：

- 结果帖改成观点帖
- 方法帖改成案例帖
- 自说自话改成用户问题切入

## 常见失败模式

- 只有结论，没有支撑
- 只有过程，没有 promise
- 写给所有人，结果谁都不在意
- 明明是 Thread 题，硬塞进一条短帖
- 英文内容像翻译腔
