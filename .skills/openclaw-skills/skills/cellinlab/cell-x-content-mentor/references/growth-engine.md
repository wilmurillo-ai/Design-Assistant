# Growth Engine

> 在场景 D 使用：用户问增长、定位、分发、导流和阶段策略。

## 先判断账号阶段

### 0-1K：冷启动

核心目标：

- 找到最初一批真关注者
- 明确你让人记住的主题
- 建立持续发和持续互动的节奏

优先动作：

- 高频短帖测试题材
- 在相关对话里留下高质量回复
- 从自己正在做的事里产出内容
- 不急着追求“大而完整”的内容体系

### 1K-10K：验证期

核心目标：

- 找到高表现内容簇
- 知道什么在拉新，什么在留人
- 把零散高表现内容沉淀成稳定栏目

优先动作：

- 开始做更稳定的 Thread 或系列
- 把高表现短帖扩成更深内容
- 建立内容复盘与模板提取
- 开始尝试导流自有资产

### 10K+

核心目标：

- 放大已经验证过的认知位置
- 建立跨平台与产品/服务联动
- 从“写得出”升级到“系统化复利”

优先动作：

- 做系列化、资产化内容
- 明确导流路径
- 让高表现主题跨格式复用

## 历史上常见的高价值信号

这些通常比点赞更值得重视，但精确权重会变：

- 回复质量与回复链条
- 主页点击
- 收藏 / 书签
- 停留时长
- 被目标圈层引用或二次讨论

更安全的表达方式：

- 说“历史上通常更重要”
- 不说“算法一定给这个 150x 权重”

## 增长排查顺序

当用户说“涨不动了”，按这个顺序看：

1. **定位**
   - 别人知道你主要讲什么吗
2. **题材**
   - 你发的是别人会在 X 上讨论和转发的东西吗
3. **写法**
   - 前两行能不能让人停下来
4. **互动**
   - 你有没有主动进入对话，而不是只发不回
5. **资产承接**
   - 高表现内容有没有被扩成后续资产

## 对话优先原则

对很多账号来说，增长不是“多发一点”，而是“更会进入对话”。

优先做：

- 在相关大号和同量级账号的讨论里留下有内容的回复
- 用问题、判断、反例引发回复
- 发出去之后继续参与评论区

少做：

- 只求被点赞
- 发完就走

## 内容复利飞轮

一个合理的复利链条通常是：

```text
短帖验证
  -> Thread 展开
  -> 长文 / 邮件 / 视频深化
  -> 反哺下一轮短帖
```

如果每一层都重新从零想，说明系统没有建立起来。

## 中文创作者做英文 X 的增长提醒

- 不需要装成本地母语表达者
- 但需要对国际受众的 context 负责
- 信息差与真实过程，比语言花哨更重要
- 中国市场一手经验可以成为差异化资产

## 常见误区

- 题太散，什么都发一点
- 把增长问题误判成语言问题
- 只看单条爆不爆，不看长期内容结构
- 把“多平台同步发”误当成策略
- 没有自有资产承接，只把 X 当终点
