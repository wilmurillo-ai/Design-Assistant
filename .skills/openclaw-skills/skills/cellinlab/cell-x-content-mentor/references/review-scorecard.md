# Review Scorecard

> 在场景 C 和场景 E 使用：审阅单条内容，或从账号数据回看内容结构。

## 单条内容审阅

### Hook

检查：

- 前两行能否让目标读者停下来
- 是否说清了讲什么
- 是否有一个具体锚点
- 是否有值得继续看的张力

### Body

检查：

- 是否在推进，而不是重复
- 是否把一个中心意思讲清了
- 是否有多余背景和铺垫
- 是否有读者能带走的东西

### Fit

检查：

- 这是 X 原生内容吗
- 这是短帖该做的事吗，还是 Thread 题
- 语言和受众是否匹配

### Action

检查：

- 想要回复、关注、点击，还是只是完成表达
- 文本有没有为这个动作服务

## Hook 评分建议

可以粗分为：

- `8-10`
  - 方向对、具体、有抓力
- `5-7`
  - 可读，但缺少真正让人停下来的东西
- `1-4`
  - 过泛、过虚、无锚点、无张力

## 账号复盘要看的不是“哪条火了”

而是：

- 哪些内容簇反复有效
- 哪些题材虽然你常发，但很少形成结果
- 哪些内容带来对话，哪些只带来空点赞
- 哪些时间段、语言、形式更适合当前账号

## 反模式

- 只发“我做了什么”，不解释为什么别人要关心
- 全是观点，没有经历或证据
- 全是过程，没有结论
- 一味模仿流行表达，丢掉自己的识别度
- 把英文 X 写成中文长文的切碎版

## 数据复盘的基础指标

优先级通常是：

1. 回复与对话质量
2. 收藏 / 书签
3. 主页点击或个人资料访问
4. 转发或引用
5. 点赞

如果用户能提供更完整数据，再补：

- 曝光
- 关注转化
- 外链点击

## 诊断句式建议

先给结论，再给依据：

- 这条问题不在文笔，在题材 promise 太弱。
- 这条不是不能发，是更像 Thread 的第一条。
- 这条有态度，但没有可信度锚点，所以停留不够。
- 你账号的问题不是“没有努力发”，而是题材结构太散。
