# Topic Engine

> 在场景 A 使用：用户不知道发什么，或不知道怎样把项目、经验、观察转成 X 内容。

## 先判断“这件事值不值得发”

至少满足下面之一再进入写作：

- 有清晰观点
- 有具体经历或结果
- 有反常识观察
- 有可迁移的方法
- 有能引发对话的问题

如果只有“我做了个东西”但没有：

- 为什么重要
- 对谁重要
- 你的独特视角在哪里

先别写。

## 把素材转成 X 原生角度

常见的六种切法：

1. **Build in Public**
   - 适合：产品进展、实验、上线、失败复盘
2. **Learn in Public**
   - 适合：新知识、工具测试、方法拆解、论文理解
3. **Sharp Observation**
   - 适合：行业变化、平台现象、用户行为
4. **Operator Breakdown**
   - 适合：拆流程、拆系统、拆决策
5. **Case and Contrast**
   - 适合：A vs B、做过和没做过、前后变化
6. **Tension or Dilemma**
   - 适合：你真正纠结的选择、矛盾、取舍

## 4A 角度矩阵

一个主题至少尝试从四个方向看：

- **Actionable**：教人怎么做
- **Analytical**：解释为什么
- **Aspirational**：展示变化、可能性、拉力
- **Anthropological**：揭示人性、误区、真实处境

如果同一个主题四个角度都出不来，通常说明主题过薄。

## X 原生题材判断

### 更适合短帖

- 一个锋利判断
- 一个反常识观察
- 一个可引发回复的问题
- 一个有数字或前后对比的小结果

### 更适合 Thread

- 需要 3 层以上展开
- 需要步骤、案例、对比或逻辑链
- 结论本身不够，需要过程支撑

### 更适合系列内容

- 同一主题会持续出现
- 你正在长期做某个项目
- 需要建立一个稳定认知位置

### 不适合直接发 X

- 观点还没成型
- 需要太多背景信息才能讲清
- 本质上更适合长文、播客、课程、研究报告

## 选题优先级

优先级从高到低：

1. 你亲自做过、测过、踩过
2. 你能说出别人没有说清的那一层
3. 你有独特地理、行业或语言视角
4. 你只是转述别人说过的事

如果排到第 4 层，尽量加上：

- 你的验证
- 你的取舍
- 你的反对意见

## 中文创作者做英文 X 的转法

不要直接翻译中文内容。

先重构：

- 这个话题对国际受众真正重要的是什么
- 你的中国语境经验能提供什么信息差
- 哪些表达在中文里成立，在英文里会显得啰嗦或自说自话

优先发：

- 真实实验
- 产品过程
- 中国市场的一手观察
- 对全球 AI / creator / indie maker 语境有价值的细节

少发：

- 纯鸡汤
- 纯态度输出
- 只对本土语境有效但没有背景解释的判断
