# Account Diagnosis

> 在场景 E 使用：用户要求系统分析账号。

## 目标

账号诊断不是“看几条高赞帖然后夸两句”，而是回答：

- 这个账号到底在被什么内容定义
- 哪些内容真正有复利，哪些只是看起来忙
- 增长瓶颈出在定位、题材、写法、互动，还是承接结构

## 采集优先级

按可获得性从高到低：

1. 用户直接提供导出的 analytics / CSV / JSON
2. 用户提供最近 30-100 条帖子文本和基础互动
3. 浏览器或可视化方式抓取公开主页样本

如果只能拿到部分样本，也可以做，但必须写明：

- 样本条数
- 时间跨度
- 缺失了哪些指标

## 建议存档结构

```text
user-data/x-content-mentor/<username>/
├── profile.md
├── posts_<date>.json
├── posts_<date>.md
├── report_<date>.md
└── strategy.md
```

## profile.md 建议记录

- 用户名
- 账号 bio 概述
- 语言分布
- 主题印象
- 当前阶段判断
- 最近一次诊断日期

## report 结构

至少包含六部分：

1. **样本说明**
2. **核心发现**
3. **内容结构**
4. **互动结构**
5. **时间与语言观察**
6. **下一阶段动作**

## strategy.md 应沉淀什么

- 当前最值得加码的内容簇
- 不值得继续高频投入的内容簇
- 推荐的语言与形式配比
- 当前账号的一句话定位
- 接下来 2-4 周最重要的动作

## 诊断优先级

先看这四件事：

1. **内容簇**
   - 账号被哪几类内容定义
2. **结果差异**
   - 哪类内容明显更高 ROI
3. **互动结构**
   - 回复 / 书签 / 引用是否健康
4. **动作缺口**
   - 发了之后有没有承接、复盘和扩展

## 输出原则

- 先给 3-5 条最关键发现
- 发现要能落到动作
- 不要把报告写成泛泛复盘作文
- 如果缺数据，清楚写出来
