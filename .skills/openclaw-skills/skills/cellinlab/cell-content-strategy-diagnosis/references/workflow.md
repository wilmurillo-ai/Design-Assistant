# 内容策略诊断 Workflow

## 1. 输入状态判断

先判断用户给你的是什么：

- 只有一个题
- 已经有内容计划
- 已经有草稿
- 内容已经发过，但想复盘问题

不同输入状态，对应不同诊断重心：

- 只有一个题
  - 重点看题是否成立、适合什么载体、需要补什么材料
- 有计划但没成稿
  - 重点看形式、平台、目标动作路径
- 有草稿
  - 重点看 promise、表达效率、材料分布、是否选错载体
- 已发内容复盘
  - 重点看内容本体问题，不要把一切先归因为平台算法

## 2. 目标先行

至少判断用户更偏向哪一种目标：

- 建认知
- 建信任
- 获取咨询或私信
- 直接成交
- 做观点表达或公开思考

如果目标不同，内容标准也不同。

例如：

- 建认知
  - 更重第一眼理解和记忆点
- 建信任
  - 更重具体案例、判断过程和可信细节
- 成交
  - 更重是否能自然连接到产品或服务

## 3. 推荐载体的判断顺序

优先问三个问题：

1. 这个内容靠什么成立：逻辑、画面、情绪、结果、步骤，还是人本身？
2. 用户需要停留多久才能真正感受到价值？
3. 用户看完后希望发生什么动作？

常见匹配：

- 强观点 + 高压缩表达
  - 更适合短视频或短帖
- 复杂逻辑 + 需要背景
  - 更适合文章或长视频
- 步骤、清单、对比
  - 更适合图文或 carousel
- 需要现场互动和反驳处理
  - 更适合直播或对谈

## 4. 症状与根因分离

优先排查以下错位：

### 4.1 标题问题，其实是 promise 问题

如果用户自己都说不清“这一条到底要说什么”，就不要先讨论标题。

### 4.2 开头问题，其实是材料问题

如果正文没有数据、反差、场景、案例、锐利判断，开头再用力也只是在透支。

### 4.3 平台问题，其实是形式问题

有些内容不是平台错，而是把应该写成长文的东西硬压成短视频。

### 4.4 AI 味问题，其实是作者感问题

真正的问题常常不是 AI，而是没有个人判断、没有体验痕迹、没有具体对象。

## 5. 结论写法

默认按这个顺序交付：

1. 一句话判断
2. 推荐形式和平台
3. 五维诊断
4. 主问题
5. 第一动作
6. 是否建议交给其他 Skill

## 6. 何时停止

以下情况可以止步于诊断，不必继续给大量建议：

- 用户只问“问题在哪”
- 输入信息明显不足
- 核心问题已经足够明确
- 后续更适合交给别的 Skill
