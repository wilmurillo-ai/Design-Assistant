# Block Taxonomy

## 总原则

素材块不是“多写一点说明”，而是未来写作时可以直接调用的模块。

优先使用下面六类块，前四类最常用：

- 观点块
- 故事块
- 金句块
- 概念块
- 对比块
- 动作块

## 1. 观点块

适合承载中心判断、反常识结论、结构判断。

建议字段：

- 标题
- 观点母句
- 它在反驳什么
- 为什么重要
- 适合写在哪类文章里
- 可继续展开的 2-3 个方向
- 依据来源

## 2. 故事块

适合承载人物、案例、实验、历史片段、微场景。

建议字段：

- 标题
- 人物 / 场景
- 冲突
- 转折
- 这个故事证明什么
- 适合出现在文章哪个位置
- 依据来源

## 3. 金句块

适合做开头、转折、点题、结尾。

建议字段：

- 标题
- 类型：原句 / 金句化改写
- 句子
- 适用位置：开头 / 小节标题 / 转折 / 收束 / 结尾
- 后面最适合接什么内容
- 风险提示：是否过满、是否需要上下文支撑

## 4. 概念块

适合解释核心术语、认知框架、关键区分。

建议字段：

- 标题
- 概念定义
- 它和什么容易混淆
- 解释这个概念最好的类比
- 适合嵌入哪类主题
- 依据来源

## 5. 对比块

适合写冲突、误解、认知升级。

建议字段：

- 标题
- 对比双方
- 真正差异
- 为什么读者容易混淆
- 适合做哪种段落推进
- 依据来源

## 6. 动作块

适合把书中的判断落到“下一步该怎么做”。

建议字段：

- 标题
- 行动建议
- 适用对象
- 前置条件
- 可能收益
- 可能代价或边界
- 依据来源

## 组合建议

最常见的高可用组合：

- 观点块 + 故事块
- 概念块 + 对比块
- 观点块 + 动作块
- 金句块 + 观点块

如果未来目标是交给 `$deep-writer`，优先把素材块组合成：

- 开头块
- 主论点块
- 机制解释块
- 反例或边界块
- 结尾块
