# 模仿颗粒度检查

## 1. 不要只说“我在学他”

必须能说清：

- 你学的是哪个层
- 你没学的是哪个层
- 你为什么先学这一层

## 2. 可检查的颗粒

至少可以检查以下项目：

- 内容主题与切口
- 内容形式
- 发布频率
- 标题与包装风格
- 私域或咨询承接
- 产品层级
- 转化路径
- 交付方式

## 3. 常见伪模仿

### 3.1 学了语气，没学结构

结果：

- 看起来像
- 但做不出同样的结果

### 3.2 学了内容，没学产品

结果：

- 内容越来越多
- 商业结果没有同步出现

### 3.3 学了表层动作，没学节奏

结果：

- 知道别人做什么
- 但不知道别人怎么持续做

## 4. 更适合升级成 case research 的情况

如果你已经发现某个 benchmark：

- 经营信号很强
- 模型能大致看懂
- 值得长期研究

就不要停留在筛选，转交给 `$opc-case-research` 做更完整的公开信息拆解。
