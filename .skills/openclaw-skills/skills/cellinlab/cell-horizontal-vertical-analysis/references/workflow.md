## 1. Scope and Mode

先把问题缩成一个明确研究任务：

- 研究对象是谁 / 是什么
- 为什么现在研究它
- 目标是理解、比较、写作准备，还是策略判断
- 输出用在什么场景

默认模式：

- `quick`: 快速摸底，适合先判断值不值得深挖
- `standard`: 默认模式，适合做完整研究报告
- `deep`: 需要更密集的证据、矩阵、追加验证和正式交付件

## 2. Research Skeleton

任何判断之前，先搭四根骨架：

1. `一句话定义`
2. `关键时间线`
3. `横向候选名单`
4. `主要证据池`

如果四根骨架有明显缺口，先补资料，不要急着写洞察。

## 3. Vertical Axis

纵轴关注“它怎么一步一步变成今天这样”。

最少回答：

- 它解决什么旧问题或踩中了什么新机会
- 核心推动者是谁，他们为什么有资格做这件事
- 初始定位是什么，后来发生了哪些变化
- 哪些节点改变了速度、方向、资源结构或外界认知
- 哪些决策形成了路径依赖

### 3.1 推荐写法

先写起源，再按阶段展开：

- 萌芽期
- 放大期
- 转向期
- 稳定期 / 冲突期 / 再定义期

每个阶段至少要写：

- 阶段特征
- 核心矛盾
- 关键节点
- 决策逻辑
- 阶段结算

### 3.2 纵轴质量标准

好的纵轴应该像一条故事线，而不是事件年表：

- 有因果
- 有约束
- 有代价
- 有反复出现的主题

## 4. Horizontal Axis

横轴关注“它今天和谁一起站在场上”。

### 4.1 竞品场景判断

- 场景 A：几乎没有直接竞品
  - 重点分析为什么没有，未来可能从哪里冒出竞争
- 场景 B：只有 1-2 个主要对手
  - 适合做深度对照
- 场景 C：竞品充分
  - 选最有代表性的 3-5 个，其他点到为止

### 4.2 横轴核心问题

- 研究对象当前怎么定位自己
- 用户为什么选它，而不是选别家
- 和主要对手相比，最本质的差异在哪里
- 竞争来自直接对手、替代方案，还是新的路线
- 当前格局在朝什么方向变化

### 4.3 横轴不要犯的错

- 不要只做功能堆表
- 不要只写官方卖点
- 不要忽略真实用户口碑
- 不要把所有对手都写成“各有优劣”

## 5. Crossing Insights

这部分至少要回答五个问题：

1. 哪个历史节点决定了它今天的位置
2. 哪个旧优势，今天仍在提供护城河
3. 哪个旧决策，今天反而变成了包袱
4. 主要竞品的起点和路径有什么不同
5. 下一阶段最可能、最危险、最乐观的剧本分别是什么

交汇段要有判断，但判断必须能回扣到前面的证据。

## 6. Confidence Handling

默认把信息分成三层：

- `事实`
  - 有明确来源支持，可复核
- `推断`
  - 基于多条事实推出来，但不是对象明说
- `未知`
  - 目前找不到足够信息，或者来源冲突

宁可收窄，也不要硬补。
