# AI 驱动的链上网格交易策略

## 策略简介

网格交易是一种在价格波动中自动低买高卖的交易方法。本策略将价格区间划分为多个网格，当价格下跌触及网格时自动买入，上涨触及时自动卖出，通过反复捕捉波动赚取差价收益。

与传统网格不同，本策略深度融合了 **AI Agent** 和 **OKX OnchainOS**，实现了从市场分析、交易决策、链上执行到迭代优化的全流程自动化。

## 功能特性

### 网格引擎

- **非对称网格 (Asymmetric Grid)** — 看涨时买侧密集、卖侧稀疏（快速建仓、慢慢卖出），看跌时反转，顺势而为
- **波动率自适应 (Volatility-Adaptive)** — 基于 ATR 指标动态调整网格宽度，趋势强时自动加宽（减少交易频率），震荡时收窄（增加捕获机会）
- **多时间框架分析 (MTF)** — 综合 5 分钟价格、1 小时 / 4 小时 EMA、8 小时结构检测，判断趋势方向与强度

### 趋势跟踪 (Trend Following)

- **动态仓位管理 (Dynamic Position Sizing)** — 看涨时加大买入量、减少卖出量，顺势积累；看跌时反向操作，顺势减仓；中性时买卖等量
- **追踪止盈 (Trailing Take-Profit)** — 强上涨中延迟卖出，确认趋势反转后再离场，避免过早止盈
- **动量过滤器 (Momentum Filter)** — 1 小时动量强劲且处于上升趋势时，跳过卖出信号，让利润奔跑

### 风控体系

- **止损 (Stop Loss)** — 跌幅达阈值自动停止
- **追踪止损 (Trailing Stop)** — 从高点回落超过阈值触发保护
- **闪崩保护 (Flash Crash Protection)** — 短时间内急跌暂停买入，避免接飞刀
- **熔断机制 (Circuit Breaker)** — 连续执行失败后进入冷却期
- **仓位上下限** — 趋势自适应持仓比例限制，防止单边行情下仓位过重

### 运维与监控

- **Discord 实时通知** — 每笔交易推送，含盈亏、HODL Alpha 对比
- **每日报告** — 自动生成日报，统计当日交易和收益
- **可视化看板** — 专业交易仪表板，展示网格状态、价格走势、交易标记、趋势分析

## 与 AI 的结合

### 1. AI 生成策略代码

整套策略脚本由 AI Agent 设计和编写。通过 **策略工厂（Strategy Factory）** 流水线，AI 完成：

- 策略逻辑设计与代码实现
- 回测引擎搭建与参数扫描
- 风控规则制定

### 2. AI 驱动版本迭代

每一次策略升级都基于 AI 对实盘数据的分析：

```
v3.0 运行 10 天，229 笔交易
    ↓ AI 分析发现
  • HODL Alpha -$4.07（单边上涨中跑输持有）
  • 卖出成功率 84.3%，低于买入 100%
  • 缺少趋势感知，趋势行情中过度交易
    ↓ AI 设计优化方案
v4.0 新增 MTF 分析 + 趋势自适应 + 卖出优化
    ↓ AI 回测验证
  夏普比率 4.45，趋势自适应比等量策略 +1.8%
```

### 3. AI 实时复盘

策略每次 tick 输出结构化 JSON，AI Agent 可随时：

- 配对交易为 round trips，分类为「好单 / 微利 / 亏损」
- 分析根因（网格追涨 / EMA 反应过度 / 流动性不足）
- 提出参数调整建议并回测验证

### 4. AI 看板生成

Dashboard 也由 AI 编写——零依赖纯 Python 生成自包含 HTML，支持：

- 价格走势 + 网格层级叠加图
- 买卖交易标记（绿色买入 / 粉色卖出）
- 趋势状态徽章（方向 / 强度 / 结构）
- 风控面板（止损状态 / 追踪回撤 / 熔断计数）
- 活动日志时间线

## 与 OnchainOS 的结合

策略通过 **onchainos CLI** 与链上世界交互，核心调用链路：

| 能力 | onchainos 命令 | 用途 |
|------|---------------|------|
| 实时报价 | `swap quote` | 获取 ETH/USDC 最优价格 |
| K 线数据 | `market kline` | 1H 烛图，计算 ATR 波动率 |
| 钱包余额 | `wallet balance` | 读取 ETH + USDC 持仓 |
| DEX 聚合 | `swap swap` | 跨 500+ DEX 最优路由 |
| 代币授权 | `swap approve` | ERC-20 Router 授权 |
| 交易模拟 | `gateway simulate` | 上链前模拟，提前发现问题 |
| 签名广播 | `wallet contract-call` | TEE 安全签名 + 链上广播 |
| Smart Money | `signal list` | 获取智能资金 / 巨鲸买入信号 |

**关键优势：**

- **TEE 安全签名** — 私钥在可信执行环境中隔离，策略脚本永远接触不到私钥
- **DEX 聚合器** — 自动在 500+ 流动性源中寻找最优路由，降低滑点和交易成本
- **Agentic Wallet** — 无需手动管理钱包，onchainos 处理授权、签名、广播全流程
- **Smart Money 信号** — 融合链上巨鲸和 KOL 的交易信号，辅助交易决策

## 部署方式

### 运行环境

```
VPS (7×24)
├── Cron 每 5 分钟 → python3 eth_grid.py tick    → 执行交易逻辑
├── Cron 每天 08:00 → python3 eth_grid.py report  → 生成日报推送 Discord
└── Dashboard 定时生成 → 静态 HTML                 → 浏览器直接访问
```

### 前置条件

- **onchainos CLI** — OKX OnchainOS 命令行工具
- **OKX API Key** — 需有 DEX 交易权限
- **Agentic Wallet** — TEE 签名钱包
- **Python 3.10+** — 零第三方依赖
- **VPS**（推荐）— 保证 7×24 稳定运行

### 快速安装

```bash
# 方式一：ClawHub 一键安装
npx clawhub install grid-trading

# 方式二：手动部署
cp eth_grid.py ~/scripts/
crontab -e
# */5 * * * * cd ~/scripts && python3 eth_grid.py tick >> /tmp/grid.log 2>&1
```

## 适用场景

| 市场状态 | 推荐用法 |
|----------|----------|
| 震荡行情 | 双向网格，捕捉上下波动 |
| 看涨趋势 | 做多网格，趋势自适应持仓偏多 |
| 看跌趋势 | 做空网格（合约），自适应持仓偏少 |
| 波动较大 | 等比网格 + 较宽间距 |
| 波动较小 | 等差网格 + 较窄间距 |
