# 需求收集问卷设计指南

## 对话风格

### ✅ 好的做法

```
用户：帮我推荐个旅行地吧
你：好呀！先聊聊——这次是跟谁一起？情侣出游还是朋友结伴？
用户：跟女朋友
你：浪漫之旅～打算去几天？
用户：三四天吧
你：不错，小长假的感觉。打算几月出发？
...
```

### ❌ 不好的做法

```
请回答以下问题：
1. 出游人群？
2. 出行天数？
3. 出行月份？
...
```

## 各维度追问话术参考

### 出游人群
- "这次是一个人还是跟谁一起？"
- "带小朋友吗？还是二人世界？"
- "有长辈同行吗？体力上需要考虑一下"

### 出行天数
- "大概能休几天假？"
- "周末加请假还是长假？"
- 模糊值处理："三四天" → 记录为 3-4 天

### 出行月份
- "打算几月去？近期还是之后？"
- "五一、暑假这种旺季 OK 吗？"

### 出发城市
- "你从哪个城市出发？"
- 如果已知道用户位置（USER.md），可以直接确认："你从杭州出发对吧？"

### 交通工具偏好
- "自驾、高铁、飞机都行吗？有偏好吗？"
- "不想坐飞机的话我帮你筛掉远的"

### 单程时间上限
- "路上最多能接受多长时间？"
- "比如自驾 3h 内、飞行 2h 内这种"
- 引导示例："有人觉得超过 4 小时太折腾，你觉得呢？"

### 旅行偏好
- "喜欢什么类型的旅行？度假躺平、特种兵打卡、自然风光、人文古迹、美食之旅？"
- "上次玩得最开心的是哪次？"
- "有没有特别想体验的？比如潜水、滑雪、泡温泉"

### 旅行经验
- "之前去过哪些地方印象最深？"
- "大概去过多少个城市了？"
- "最喜欢的那次旅行是什么样的？"

## 信息不完整时的处理

| 情况 | 处理方式 |
|------|---------|
| 缺 1-2 个必选字段 | 补问后继续 |
| 用户说"随便都行" | 给 2-3 个差异化的选项让用户选择倾向 |
| 用户催促直接推荐 | 基于已有信息推荐，标注不确定性 |
| 用户中途改变主意 | 更新对应字段，重新评估 |

## 需求确认模板

```markdown
好的[称呼]，我理解了你的需求：

👥 出游：[出游人群]
📅 时间：[出行月份]出发，[出行天数]左右
🚄 交通：[交通工具偏好]，单程不超过[时间上限]
🎯 偏好：[旅行偏好]
🌟 难忘回忆：[旅行经验摘要]

我按这些条件帮你筛选几个目的地，可以吗？
```
