# GTM Launch Plan Template

## Purpose
Produce a full go-to-market launch plan from a project brief. This is the strategic foundation — everything else (calendar, scripts, retention) derives from it.

---

## Structure

Output the GTM plan with these sections in order:

---

### 1. Project Summary
One paragraph. State what the project is, what chain it's on, who it's for, and what problem it solves. No hype. Plain language.

**Format:**
> [Project Name] is a [product type] on [chain] that [core value prop in one sentence]. It is designed for [target audience] who [pain point or current frustration].

---

### 2. GTM Positioning

Define the market position clearly.

| Field                | Content                                              |
|----------------------|------------------------------------------------------|
| Category             | What category does this compete in?                  |
| Primary differentiator | What makes this genuinely different?               |
| Audience segment     | Who is the first 1,000 users?                        |
| Entry wedge          | What's the single use case that gets people in?      |
| Narrative hook       | One sentence that explains why this matters NOW      |

**Narrative hook formula:**
> "At a time when [market condition], [Project Name] lets [audience] [outcome] without [pain/tradeoff]."

---

### 3. Launch Phases

Define 3 phases. Adapt timelines based on the brief's launch date or default to 4-week runway.

#### Phase 1: Pre-Launch (Weeks 1–2)
Goal: Build awareness and waitlist/early community before launch.

Key activities:
- Announce on X with a single clear "what is this" post (no hype)
- Open Discord and establish 3 core channels: #announcements, #general, #early-access
- Publish a short explainer thread on X (5–7 tweets) covering: problem → solution → how it works → why now → CTA
- Identify 5–10 ecosystem partners or protocols on the same chain for cross-promotion
- Set up a simple landing page or Linktree with waitlist form

Metrics targets:
- Discord members: 200–500
- X followers: 500–1,000
- Waitlist signups: 100–300

#### Phase 2: Launch Week (Week 3)
Goal: Drive first users to product, generate on-chain activity.

Key activities:
- Launch announcement post on X (pinned)
- Live AMA or Twitter Space with founding team
- Discord: post step-by-step onboarding guide in #getting-started
- Telegram: launch channel for real-time updates
- Publish first use-case tutorial (written or video)
- Activate ecosystem partner cross-posts

Metrics targets:
- Unique wallets connected: 100–500 (adjust by product type)
- On-chain transactions: 50–200
- Discord messages per day: 30+

#### Phase 3: Post-Launch Growth (Week 4+)
Goal: Convert early users to retained, active community members.

Key activities:
- Weekly product update posts (X + Discord)
- User spotlight or case study content
- Begin retention loop (see onchain-retention.md)
- Collect and publish early user feedback publicly
- Expand to secondary communities (Reddit, Farcaster, Lens if applicable)

Metrics targets:
- 30-day retention rate: >30% of launch-week users return
- Weekly active wallets: growing week-over-week
- Community NPS or sentiment: net positive in Discord

---

### 4. Channel Strategy

| Channel     | Role                        | Frequency          | Content Type                        |
|-------------|-----------------------------|--------------------|-------------------------------------|
| X (Twitter) | Top-of-funnel awareness     | 3–5x per week      | Threads, updates, ecosystem news    |
| Discord     | Community hub & retention   | Daily moderation   | Announcements, support, events      |
| Telegram    | Real-time updates           | 2–3x per week      | Launch news, milestones, alerts     |
| On-chain    | Retention signal            | Event-driven       | Reward triggers, streak milestones  |

---

### 5. Success Metrics (30-Day)

Define 3 primary KPIs based on product type:

| Product Type  | KPI 1                    | KPI 2                        | KPI 3                      |
|---------------|--------------------------|------------------------------|----------------------------|
| DeFi          | TVL (target: $X)         | Unique wallets (target: N)   | Avg sessions/wallet        |
| GameFi        | DAU (target: N)          | Matches/sessions played      | Wallet retention (D7/D30)  |
| NFT           | Mint rate                | Secondary volume             | Holder retention           |
| Payments/Infra| API calls / requests     | Integrating partners         | Recurring usage rate       |
| Oracle/Data   | Subscriptions or queries | Developer integrations       | Uptime / accuracy trust    |
| Social dApp   | MAU                      | Content pieces created       | Return visit rate          |

Adapt targets to the specific project. Do not leave generic placeholders — estimate based on product type and chain maturity.
