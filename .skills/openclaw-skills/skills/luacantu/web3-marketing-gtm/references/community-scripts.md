# Community Engagement Scripts

## Purpose
Produce platform-native, ready-to-use engagement scripts for Discord, Telegram, and X. Scripts must feel human-written, not bot-generated. Adapt all [BRACKETS] to the specific project.

---

## Platform Rules

| Platform  | Max Length         | Tone              | Format                        |
|-----------|--------------------|-------------------|-------------------------------|
| Discord   | No hard limit      | Conversational    | Use line breaks, minimal bold |
| Telegram  | 200–400 chars      | Direct, clear     | Plain text, no markdown       |
| X         | 280 chars/tweet    | Punchy, specific  | Threads max 7 tweets          |

---

## Discord Scripts

### Script D1 — Server Welcome Message (pin in #general)

```
Welcome to [Project Name] 👋

We're building [one-sentence description] on [chain].

A few things to know:
→ What we're about: #about
→ How to get started: #getting-started
→ Ask anything: #general
→ Stay updated: #announcements

We're early. Your feedback matters here more than anywhere else.

— [Founder handle or team name]
```

---

### Script D2 — Launch Day Announcement

```
[Project Name] is live on [chain].

Here's what you can do right now:
→ [Action 1: e.g. Connect your wallet and explore the dashboard]
→ [Action 2: e.g. Make your first trade / query / transaction]
→ [Action 3: e.g. Check your on-chain activity and claim your early user role]

Link: [URL]

Drop your wallet address (or ENS) below if you're one of the first 100 in. We're tracking early adopters.
```

---

### Script D3 — Weekly Community Update (every Sunday)

```
Week [N] Recap

Numbers:
• [Metric 1]: [value] (↑/↓ from last week)
• [Metric 2]: [value]
• [Metric 3]: [value]

What happened:
• [Key event or milestone]
• [Bug fix or improvement shipped]
• [Community highlight]

What's next:
• [Feature or event coming next week]
• [Specific date if known]

Questions? Drop them below or hit us in #general.
```

---

### Script D4 — Community Engagement Prompt (use 2–3x/week)

Choose one per post. Never stack multiple questions.

```
Option A:
What's one thing about [product category] that still doesn't work the way you want it to?

Option B:
If [Project Name] could do one more thing, what would it be?

Option C:
How did you first hear about us? (Genuinely curious — it helps us.)

Option D:
Tag someone who'd actually use this. Tell us why.
```

---

### Script D5 — AMA Opening Message

```
AMA starts now. 

[Founder/Team member name] is here for the next 60 minutes.

Rules:
→ One question per person to start
→ Follow-ups welcome after the first round
→ No off-topic, no price talk

Ask away.
```

### Script X2 — Explainer Thread (7 tweets)

```
Tweet 1 (hook):
[Problem statement — no fluff. E.g.: "Most DeFi dashboards show you data. None of them tell you what to do with it."]

Tweet 2 (introduce product):
[Project Name] fixes this.

Here's how it works: 🧵

Tweet 3 (mechanism — what it does):
[Clear explanation of core feature. No jargon unless your audience is technical.]

Tweet 4 (why this chain):
We built on [chain] because [specific reason: speed, cost, ecosystem, tooling].

Not because it's trendy. Because it makes [product] work better.

Tweet 5 (who it's for):
If you [specific user behavior], [Project Name] is for you.

If you're looking for [hype thing], look elsewhere.

Tweet 6 (proof or milestone — use real number):
[X] people are already [using/on the waitlist/testing].

That happened in [timeframe].

Tweet 7 (CTA):
[Project Name] is live / launching [date].

→ [Link]
→ [Discord link]

Questions? Reply here.
```

## Script Customization Guide

| Variable            | How to fill it                                                   |
|---------------------|------------------------------------------------------------------|
| [Project Name]      | Exact project name from brief                                    |
| [chain]             | Chain name (capitalize: Avalanche, Base, Ethereum, Solana)       |
| [metric]            | Real number. Never estimate or round up beyond 10%              |
| [link]              | Actual URL. Never use placeholder if URL is known               |
| [timeframe]         | Specific: "72 hours", "first week", not "recently"              |
| [audience]          | One specific group: "DeFi traders", "indie game devs", etc.     |
