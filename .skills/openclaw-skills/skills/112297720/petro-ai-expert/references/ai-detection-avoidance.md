# AI检测规避技巧清单

## 一、核心原则

AI检测系统的检测逻辑主要基于：
1. **文本特征**：句式熵、词汇多样性、段落结构
2. **语义分布**：语义重复度、话题一致性
3. **语言模型困惑度**：Perplexity / Burstiness 指标

## 二、句式层面规避

### 1. 增加句式熵（Sentence Entropy）
AI生成的文本句式过于规整、统一，人类写作则相反。

**规避方法：**
- 长句（40-60字）与短句（10-15字）交错
- 避免每句话都以"首先、其次、最后"开头
- 适当使用倒装句、被动句、复合句混合
- 偶尔出现口语化表达

**反面示例（AI特征明显）：**
> 数字化转型是石油石化行业发展的重要方向。数字化转型可以提高生产效率。数字化转型可以降低运营成本。

**正面示例（人类写作）：**
> 谈到数字化转型，业内普遍认为这是降本增效的必由之路——尤其是那些经历过2015年前后油价断崖式下跌的企业，感受更为深刻。事实上，不少老油田这些年正是靠着这套系统，把原来需要跑现场才能确认的数据，直接搬到了管理平台上。

### 2. 词汇多样性
避免同一词汇反复出现。

**常用替换表：**

| 常见词 | 替换表达 |
|-------|---------|
| 重要作用 | 关键驱动因素 / 核心抓手 |
| 提高效率 | 释放人力成本 / 压缩作业周期 |
| 数字化转型 | 数智化升级 / 信息化建设 |
| 安全问题 | 隐患治理 / 风险管控 |
| 大数据 | 海量数据资产 / 多元异构数据集 |

### 3. 专业口语化穿插
人类写作常带有行业经验性语言。

**常用口语化表达：**
- "说起来你可能不信，但在我们油田现场…"
- "实际跑过项目的都知道，最大的坑其实在接口对接上"
- "厂家宣称的指标是一回事，真到戈壁滩上跑一圈完全是另一回事"
- "干了十几年的老工程师都知道，这个环节最怕的不是技术，是配合"

## 三、段落结构规避

### 1. 避免"总-分-总"机械结构
AI偏好完美的三段论结构，人类写作则更松散。

**规避方法：**
- 在段落开头加入背景交代或问题引出
- 结尾留有思考余地，不要过于圆满
- 段落长度不一（50-300字均有）

### 2. 增加过渡段落
在章节之间加入承上启下的段落，降低机械感。

**过渡段落示例：**
> 以上讨论了技术层面的可行性，但真正落地远不止技术选型这么简单——人员培训、流程再造、供应商配合，每一项都是实打实的硬仗。接下来本文结合某油田的实际改造案例，来看一看这些坑是怎么踩过来的。

## 四、逻辑结构规避

### 1. 增加因果跳跃
人类写作常有逻辑跳跃，AI则过度连贯。

**规避方法：**
- 在论证过程中加入"虽然…但…"的转折
- 适当使用"这里有一个有趣的现象…"引出
- 承认研究局限性，而非一味正面表述

### 2. 增加反例讨论
人类写作会主动讨论反对意见。

**示例：**
> 当然，也有人质疑这套系统的实际价值——投入大、周期长、效果难以量化。这些担忧并非没有道理。事实上，在本文调研的几个案例中，确实有一个项目因为数据质量不达标而导致模型准确率偏低。这是后续研究需要重点关注的方向。

## 五、内容层面规避

### 1. 增加行业特异性细节
AI生成内容缺乏具体的企业/项目名称、时间、数据。

**规避方法：**
- 加入真实年份（2019年、2021年等）
- 加入具体企业/项目名称（如"中石化西北油田分公司顺北处理站"）
- 加入具体数字范围，而非精确数字

### 2. 引用真实文献
在正文中加入对具体文献的引用讨论。

**示例：**
> 王建平等（2021）在《油气田地面工程》发表的论文中指出，小断块油藏的数字化改造需要充分考虑老旧设备的接口兼容性——这与笔者在胜利油田调研时听到的现场反馈高度吻合。

## 六、快速检测自检清单

完成写作后，用以下标准自检：

- [ ] 句式是否足够多样（长句/短句交错）？
- [ ] 是否存在连续3句以上相同句式开头？
- [ ] 是否缺少承上启下的过渡段落？
- [ ] 是否有过多"总-分-总"机械结构？
- [ ] 段落长度是否过于均匀（人类写作应该有差异）？
- [ ] 是否缺少行业口语化表达？
- [ ] 是否缺少具体企业/项目/时间细节？
- [ ] 是否缺少对反对意见的讨论？
- [ ] 词汇是否过于重复？

9项中超过3项"否"，建议做一轮修改再提交。
