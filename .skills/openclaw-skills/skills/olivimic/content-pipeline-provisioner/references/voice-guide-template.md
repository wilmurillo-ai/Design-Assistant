# Voice Guide Template

When generating a client's voice.md file, use this exact structure.
Populate from their onboarding answers: brand_voice, product_description, target_audience, notes.

---

## Template

```markdown
# [Business Name] — Twitter/Content Voice Guide
_Generated: [DATE] | Client: [name]_

## Brand Voice in Three Words
[brand_voice — e.g. "Bold. Educational. Direct."]

## Who We're Talking To
[target_audience — rewritten as a 1-sentence description of the reader]

## Core Message
[Derive from product_description — what's the one thing every post should communicate?]

## Tone Rules
- [Derive from brand_voice word 1 — e.g. "Bold: lead with the strongest claim, no hedging"]
- [Derive from brand_voice word 2 — e.g. "Educational: always leave the reader knowing something they didn't"]
- [Derive from brand_voice word 3 — e.g. "Direct: short sentences, no filler words"]
- Never mention competitors by name
- [Add off_limits topics if provided]

## Post Formats That Work
- Insight post: One sharp observation about [their industry] + why it matters
- Before/after: What [target_audience] used to do vs. what's possible now
- Stat drop: One specific number + one-line implication
- Hot take: Contrarian view on a common [industry] belief
- Founder story: One real moment from building [business_name]

## CTAs to Rotate
- "Try it free at [url]"
- "Link in bio"
- "[Business name] — [one-line value prop]"

## Off-Limits
[off_limits if provided, otherwise "Nothing specified — use judgment"]

## Example Posts (generate 3 in their voice)
[Generate 3 example tweets using the above rules and their product context]
```

---

## Generation Instructions

1. Fill every section from onboarding answers
2. For "Tone Rules" — take each of their 3 brand_voice words and write one concrete rule
3. For "Core Message" — read product_description and distill to one sentence max
4. Generate 3 real example tweets at the bottom — these seed the Twitter autoposter
5. Keep the whole file under 400 lines
