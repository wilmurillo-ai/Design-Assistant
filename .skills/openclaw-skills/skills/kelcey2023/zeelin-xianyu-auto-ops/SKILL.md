# ZeeLin 闲鱼自运营 🐟

面向**用户已登录闲鱼网页版**的自运营 skill。目标不是做“黑盒挂机”，而是把最值钱、最实用的运营动作标准化：

- 自动客服回复
- 智能议价回复
- 商品文案生成与优化
- 批量处理常见询问
- 催付 / 自提 / 发货 / 售后话术
- 在网页端辅助执行回复、编辑、发布

## 触发场景

当用户表达以下需求时，使用本技能：

- 「帮我做闲鱼自动回复」
- 「做一个闲鱼自运营 skill」
- 「帮我回复闲鱼买家」
- 「这个闲鱼商品怎么写更容易卖」
- 「帮我批量处理闲鱼消息」
- 「帮我在闲鱼上架」
- 「帮我自动议价」
- 「我已经登录闲鱼网页，你直接操作」

## 能力边界

本技能**优先走浏览器网页工作流**，不直接依赖闲鱼私有 API。

原因：
- 更稳妥，和 OpenClaw 当前浏览器能力更匹配
- 用户已登录即可用，不额外要求抓 cookie / 反向接口
- 降低风控和维护成本

参考仓库 `shaxiu/XianyuAutoAgent` 时，吸收的是它的**运营思路**：
- 意图分类
- 多专家话术（默认 / 议价 / 技术说明）
- 上下文感知
- 人工接管

但在 OpenClaw 里，默认实现方式改为：
- **话术模板 + 大模型生成**
- **浏览器执行**
- **人工确认后发送**（默认）

## 默认工作原则

1. **先确认用户已经登录闲鱼网页**
2. **先读页面，再操作页面**
3. **默认先给草稿，再代发**；除非用户明确要求“直接发”
4. **涉及价格、售后承诺、联系方式时更谨慎**
5. **不诱导脱离平台交易，不输出违规话术**
6. **先确认当前会话身份是“卖家回复买家”还是“买家咨询卖家”，再发送任何消息**
7. **如果会话商品、价格、身份与用户要求不一致，先停止发送，回到列表重新核对，不要凭猜测回复**

## 会话身份校验（新增强规则）

在消息页执行任何自动回复前，必须先完成这 4 个检查：

1. **核对商品是否匹配**
   - 先看聊天窗顶部/商品卡片中的商品标题、价格、地区
   - 必须与用户口中的商品一致（例如“龙虾安装部署 / 299 元”）
   - 如果当前聊天窗是别的商品，不能回复，先返回会话列表继续找

2. **核对自己当前扮演的身份**
   - 如果用户要求你“回复来买我东西的客户”，你必须站在**卖家**身份回复
   - 如果页面内容显示你是在咨询别人、或历史语气明显是买家身份，立即停止，不要继续发

3. **核对最后几轮上下文是否连贯**
   - 至少读最近 3～6 条消息，再判断客户是在问安装、价格、成色、发货还是售后
   - 不要只看左侧列表摘要就直接回

4. **发现错会话时的处理方式**
   - 不继续硬回
   - 回到左侧列表，继续筛选正确客户
   - 必要时先向用户确认“是不是这个商品/这个客户”

这条规则优先级高于“尽快回复”。**宁可慢一步核对，也不要回错买家/卖家身份。**

## 第一反应流程

### A. 如果用户说“帮我回复闲鱼买家 / 自动客服”

1. 先确认当前是否已打开闲鱼消息页
2. 如未打开，导航到闲鱼网页版消息页
3. 用浏览器 snapshot 读取当前会话列表 / 对话内容
4. 抽取最近几轮消息
5. 判断类型：
   - 默认咨询
   - 议价
   - 参数 / 成色 / 使用情况
   - 发货 / 自提
   - 售后 / 退换
   - 明显无效消息 / 可不回
6. 先生成回复草稿
7. 用户要求直接发送时，再代发

### B. 如果用户说“帮我上架闲鱼 / 写商品文案”

先收集最少必要信息：
- 商品名
- 品类
- 品牌 / 型号
- 成色
- 使用时长
- 瑕疵
- 配件是否齐全
- 期望售价
- 是否支持小刀 / 自提 / 包邮

然后输出：
- 标题 3-5 个版本
- 正文描述 1 个完整版本
- 标签关键词
- 议价底线建议
- 常见买家提问预答复

若用户要求直接发到闲鱼：
- 打开发布页
- 代填标题 / 正文 / 价格等字段
- **发布前必须给用户最后确认**

### C. 如果用户说“帮我做闲鱼自运营 skill”

默认提供以下子能力：
- 自动回复买家
- 智能议价模板
- 商品文案生成
- 店铺运营 SOP
- 浏览器端辅助执行

必要时读取：
- `references/reply-playbook.md`
- `references/listing-playbook.md`
- `references/browser-sop.md`

## 回复分类规则

### 1. 默认咨询（default）

适用于：
- 还在吗
- 这个能用吗
- 有无瑕疵
- 什么配置
- 能发货吗

目标：
- 快速、礼貌、可信
- 降低买家决策成本
- 把模糊问题答具体

风格：
- 口语化
- 不油腻
- 不过度热情
- 尽量像真人卖家

### 2. 议价（price）

适用于：
- 能便宜吗
- 最低多少
- XX 出不出
- 包邮吗

原则：
- 先稳价，再小幅让步
- 不一次性到底
- 用理由支撑价格
- 让买家觉得“有沟通空间，但不是乱砍”

建议阶梯：
- 第一次：轻微让步或不让步
- 第二次：给出小幅优惠 + 条件（如今天拍、同城自提）
- 第三次：给到底线附近报价
- 再往下：礼貌收口

### 3. 商品细节 / 成色 / 技术说明（tech）

适用于：
- 电池健康多少
- 有没有维修
- 屏幕有没有划痕
- 型号 / 配置 / 尺寸
- 有什么功能问题

原则：
- 真实，不夸张
- 避免含糊其辞
- 信息越具体越容易成交

### 4. 发货 / 自提 / 成交流程（delivery）

适用于：
- 什么时候能发
- 支持自提吗
- 走什么快递
- 包邮吗

原则：
- 明确时间、地点、方式
- 不做无法兑现的承诺

### 5. 售后 / 退换 / 风险沟通（after-sales）

适用于：
- 收到不满意怎么办
- 能退吗
- 有没有保修

原则：
- 保守表达
- 以平台规则为准
- 不擅自承诺超范围售后

## 人工接管规则

遇到以下情况，建议切人工：
- 买家情绪明显激烈
- 涉及大额砍价
- 涉及真假、维修史、纠纷风险
- 涉及脱离平台交易诱导
- 用户要求改成自己的语气

推荐提示：
> 这轮我建议先人工接管，我可以继续帮你拟回复，但先不要自动发。

## 浏览器操作 SOP

需要网页操作时，使用浏览器工具或 `openclaw browser` 完成。

### 消息回复 SOP

1. 打开闲鱼网页版（通常是 `https://www.goofish.com/`）
2. 进入消息页
3. snapshot 当前会话列表和聊天内容
4. 抽取最后 3-10 条上下文
5. 生成回复
6. **默认先展示草稿**
7. 用户同意后再把文本填入输入框并发送

### 商品发布 / 编辑 SOP

1. 打开发布页或商品编辑页
2. 定位标题、正文、价格、分类等字段
3. 填入生成内容
4. 如需上传图片，提示用户提供素材路径或自行在网页中处理
5. **发布前必须再次确认**
6. 用户确认后再点击发布

### 批量处理会话 SOP

当用户说“批量处理”时：
- 逐个读取会话摘要
- 只处理高价值或待回复会话
- 对明显无效消息标记“可不回”
- 每批建议 5-10 个，避免太猛
