# 浏览器执行 SOP

## 前提

- 用户已登录闲鱼网页版
- 优先使用浏览器快照 + 定位元素 + 填写 / 点击
- 页面结构可能变化，因此每次操作前都应重新 snapshot

## 1. 消息回复流程

1. 打开闲鱼主页或消息页
2. snapshot 获取会话列表
3. 点击目标会话
4. 读取聊天区最近消息
5. 生成回复草稿
6. 默认先给用户确认
7. 如用户同意，定位输入框并填入文本
8. 点击发送

## 2. 商品发布流程

1. 打开发布页
2. snapshot 找标题 / 描述 / 价格 / 分类字段
3. 依次填写
4. 如需上传图片，等待用户提供路径或手动处理
5. 发布前再次确认
6. 点击发布

## 3. 批量处理消息

建议每次处理 5-10 个会话：
- 先识别是否值得回复
- 对低价值会话跳过
- 对议价会话优先给 1 条稳妥回复
- 对售后风险会话优先提示人工介入

## 4. 失败恢复

如果页面断开、ref 失效、弹层遮挡：
- 重新 snapshot
- 必要时刷新页面
- 不要基于旧 ref 反复点击

## 5. 风险控制

- 未经确认不要直接发布商品
- 默认不要连续批量自动发送过多消息
- 遇到异常争议对话停止自动发送
