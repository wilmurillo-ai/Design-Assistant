# 案例表达摘要

以下为可公开分享的通用案例表达方式，用于统一方案写作质量和结构。

---

## 案例1：平台合作增长提案

**结构**：合作空间数据 → 共建目标 → 三大打法 → 逐打法详述

**关键数据写法**：
- 使用真实基线数据（流量、转化、留存、成本）
- 无数据统一标注 `[待补充：XX数据]`

**目标写法**：量化目标 + 时间边界 + 责任主体

**三大打法写法**：
1. 权益加磅/膨胀
2. 活动玩法（抽奖、裂变、任务）
3. 专属价格/专属货盘

**文档风格**：每个打法用表格列出"类型 / 示例 / 链路示意"，链路配截图。

---

## 案例2：场景化心智打法

**结构**：by场景分类心智

| 场景 | 触点 | 用户心智 | 示例 |
|------|------|----------|------|
| 购物场景 | 商品流、活动频道 | 更省钱、更划算 | 返现、专属价、加价券 |
| 互动场景 | 游戏、任务、大促 | 多拿奖励、参与感 | 抽奖、裂变、签到 |
| 权益场景 | 签到页、会员页 | 领更多权益 | 权益加码、双倍领取 |

**文档风格**：极简，一页一个场景，心智+示例。

---

## 案例3：AI合作对外材料

**结构**：用户需求洞察 → 行业案例对标 → 合作方案

**用户需求三阶段（通用）**：
1. 普适知识获取（完整性、专业度）
2. 场景化决策辅助（个性化）
3. 主动化智能服务（持续陪伴）

**合作方案写法**：
- 接入形式：原生接入 / API 调用 / 智能体接入
- 能力输出：推荐、诊断、分析、问答、订阅

**文档风格**：对外材料，强调需求洞察、方案价值和落地路径。

---

## 链路图风格总结

7张链路图的共同特征：

- **排布**：手机屏幕截图从左到右横排
- **标注**：每屏上方标注步骤名（前端入口 → 权益透传 → 下载/唤起 → 站内页 → 目标行为）
- **过桥页**：建议保留唤起/下载承接页
- **分场景**：不同场景分行展示（如互动、权益、频道）
- **数据标注**：关键位置标注UV或转化数据
