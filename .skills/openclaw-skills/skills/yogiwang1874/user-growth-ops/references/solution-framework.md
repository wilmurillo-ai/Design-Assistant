## 方案设计方法论

**拆解 → 发散 → 收敛** 三步法，避免直接跳到打法层拍脑袋。

1. **拆解**：用结构化框架把场景拆透（人货场 / 成本体验效率 / 用户链路）
2. **发散**：每个拆解维度下穷举可能的打法方向，不设限
3. **收敛**：从可落地性、ROI、用户体验三个标尺筛选，归拢为3-5个核心打法

---

## 结构化拆解框架

遇到新场景时，从以下框架中选择合适的进行拆解：

### 框架1：人-货-场

| 维度 | 拆什么 | 关键问题 |
|------|--------|---------|
| **人** | 用户是谁、从哪来、要什么 | 目标用户画像？新客潜力占比？当前心智和行为习惯？ |
| **货** | 能给用户什么 | 权益（券/积分/补贴）、商品（低价SKU）、服务（工具/内容）、玩法（抽奖/裂变）？ |
| **场** | 在什么场景下触达 | 页面哪个位置？什么时机触发？用户此刻情绪和意图是什么？ |

### 框架2：成本-体验-效率

| 维度 | 拆什么 | 关键问题 |
|------|--------|---------|
| **成本** | 多方各出什么 | 补贴谁出？产研资源谁投？边际成本能否递减？ |
| **体验** | 用户体验是否丝滑 | 链路是否过长？每步流失率？过桥页体验？回端后承接是否到位？ |
| **效率** | 投入产出比 | 单客获取成本？留存质量？能否规模化复制到其他频道？ |

### 框架3：用户链路逐环节拆解

把用户从"没听说过"到"成为有效新客"的全链路拆开，每个环节找优化机会：

```
曝光（看到） → 认知（知道是什么） → 兴趣（想了解） → 行动（点击/下载）
→ 过桥（唤起/下载） → 落地（首屏体验） → 转化（完成目标行为） → 留存（回来）
```

每个环节问：转化率多少？流失去哪了？能做什么提升？

## 打法发散维度库

拆解完之后，从以下维度发散打法思路。不限于此，鼓励结合场景自由发散：

### 一、商品结合

通过商品本身驱动用户跨端。

- **专属价格**：定向 SKU 低价，用价格差驱动转化
- **货盘共建**：多业务方共建商品池，嵌入导流引导
- **低价钩子商品**：1分购、限时专享等低门槛激活
- **跨品类导流**：商品消费行为 -> 高价值业务转化

### 二、权益结合

通过权益加码驱动用户行为。

- **权益膨胀**：外部权益在目标端可翻倍或加码
- **权益加磅**：签到/任务/返现等场景叠加金融补贴
- **限端专属权益**：仅目标端可领取或使用
- **权益互通**：积分、券、会员权益跨场景兑换

### 三、玩法结合

通过互动玩法吸引用户参与。

- **免单/抽奖**：业务方提供资源，活动方包装玩法引流
- **裂变传播**：邀好友助力/碰一碰/拼团，社交裂变获客
- **双端互动**：游戏化场景双端联动，完成任务得额外道具
- **签到联动**：双端签到互通，连签奖励拉长用户生命周期
- **任务体系**：完成指定行为得奖励，逐步引导深度转化

### 四、营销噱头结合

借势热点、节点和场景创造传播力。

- **节日大促**：618/双11/年货节等大促节点联合活动
- **热点借势**：结合社会热点包装限时活动（如金价暴涨→黄金专区）
- **IP联名**：品牌联名/明星合作制造传播话题
- **限时稀缺**：限量名额、倒计时、先到先得制造紧迫感

### 五、内容/信息结合

通过内容价值吸引和留住用户。

- **教育内容**：知识解释、策略解读等内容引导转化
- **KOL/达人种草**：达人内容引导目标场景转化
- **UGC激励**：用户分享赚钱案例/使用心得，口碑传播
- **资讯推送**：个性化行情/热点推送，制造打开APP的理由

### 六、工具/服务结合

通过实用工具和服务创造粘性。

- **辅助工具**：预测、推荐、筛选等付费/免费工具
- **AI助手**：智能客服、持仓分析、个性化推荐
- **生活服务嫁接**：话费充值、洗车、保险等生活服务引流
- **会员等级体系**：存量资产分级，高等级享专属服务

### 七、渠道/流量结合

拓宽获客的渠道和方式。

- **异业合作**：与目标用户重合的产品互相导流
- **三方换量**：与外部APP互相嵌入引流入口
- **第三方平台上架**：在支付宝/微信等流量平台分发
- **线下场景联动**：地推、线下活动扫码引导下载

## 收敛为可落地方案

发散完之后，用三个标尺筛选：

| 标尺 | 判断标准 |
|------|---------|
| **可落地性** | 现有产研资源能做吗？合规能过吗？上线周期可接受吗？ |
| **ROI** | 预估获客成本和用户质量，值得投入吗？ |
| **用户体验** | 链路是否自然？用户感受是"有用的福利"还是"被骚扰"？ |

通过筛选后保留3-5个核心打法，排优先级后输出方案。

## 合作提案撰写要点

- 先给数据建立合作空间（流量规模、新客潜力）
- 提出清晰的共建目标（量化）
- 用结构化拆解展示思考深度，再收敛为核心打法
- 每个打法配示例 + 链路示意
- 强调双赢（合作方获得什么、自身获得什么）

---

## 方案质量检查清单

- [ ] 经过结构化拆解，不是拍脑袋直接出打法
- [ ] 打法不局限于已有模式，有发散和创新
- [ ] 最终收敛为3-5个可落地打法
- [ ] 每个打法有具体示例和链路说明
- [ ] 数据全部来自用户提供，无杜撰
- [ ] 考虑了用户体验（不是纯薅流量）
- [ ] 考虑了多方收益和成本分摊
- [ ] 执行节奏合理，有优先级
