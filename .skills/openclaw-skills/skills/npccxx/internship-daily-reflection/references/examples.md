# Examples

## 示例 1：零散笔记转日报

### 输入

- 上午看了 OpenClaw 的 skill 机制文档，整理了触发逻辑
- 下午把 internship-daily-reflection 的 SKILL.md 写完并放到项目目录
- 花了 1.5 小时排查路径问题（全局目录和项目目录冲突）
- 目前问题：还不太清楚发布页面里 description 怎么写更容易被检索
- 明天计划：补 2 个样例，尝试第一次发布

### 期望输出特征

- 输出为中文，结构完整（完成事项、进度、问题、解决、时间、明日计划）
- 明确写出路径问题的影响和当前处理结果
- 明日计划可执行，且包含具体产出

## 示例 2：给导师的进展同步

### 输入

- 今天主要在学习 OpenClaw skill 发布流程
- 已完成：基础版 skill 目录搭建、SKILL.md 编写、项目内路径迁移
- 仍在推进：准备发布材料（示例和模板）
- 阻塞：对标签和描述词不确定
- 预计明天完成首次发布

### 期望输出特征

- 语气专业，可直接转发给导师或 leader
- 尽量量化投入时间和完成度
- 阻塞项应带下一步方案

## 示例 3：信息缺失时的处理

### 输入

- 今天改了技能
- 遇到一些问题
- 明天继续

### 期望输出特征

- 不编造细节
- 主动追问最多 3 个关键问题（任务、耗时、阻塞）
- 对未补充内容标注“待补充”
