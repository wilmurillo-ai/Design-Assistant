# Color-Filer 命名规则详解

## 命名格式

```
[表情][序号]_标题.扩展名
```

### 示例

```
📝_01_Redis数据库详解.md
🖥️_01_系统监控脚本.sh
⚙️_01_nginx配置.conf
📦_01_grafana安装包.tar.gz
```

## 序号补零规则

### 规则说明

- **≤ 99 个文件**：使用 2 位序号（01, 02, ... 10, 11）
- **≤ 999 个文件**：使用 3 位序号（001, 002, ... 010, 011）
- **超过 999 个文件**：使用 4 位序号（0001, 0002, ... 1000）

### 为什么要补零？

补零是为了确保文件按**字典序**正确排序：

```
未补零：1.py, 10.py, 2.py, 3.py  ❌ 错误顺序
补零：01.py, 02.py, 03.py, 10.py  ✅ 正确顺序
```

## 最佳实践

1. **保持语义**：文件名应保留原核心关键词
2. **简洁清晰**：避免过长文件名（建议不超过 50 字符）
3. **统一格式**：同一类别使用相同表情符号
4. **序号连续**：尽量保持序号连续，避免跳号
5. **更新索引**：重命名后同步更新索引文件
