# Level 3 测试 Skill

## 核心职责
- 完成核心任务
- 有完整的错误处理
- 有基准测试和红队测试
- 有用户反馈循环
- 有版本管理和迭代记录

## 何时使用
- 生产环境
- 需要高可靠性的场景
- 需要完整评估的场景

## 何时不使用
- 简单测试场景（用 Level 1/2 即可）

## 工作流程
1. 接收任务并验证输入
2. 执行任务（带错误处理）
3. 验证输出（带质量检查）
4. 生成报告（带详细日志）
5. 收集用户反馈
6. 持续迭代优化

## 工具依赖
- bash: 执行脚本
- python3: 数据处理
- pytest: 运行测试
- promptfoo: 基准测试

## 评估标准
- 任务完成率 > 95%
- 错误率 < 3%
- 测试覆盖率 > 95%
- 红队测试通过率 > 90%
- 用户满意度 > 4.5/5
- 平均执行时间 < 20 秒
- 平均成本 < $0.30/次
