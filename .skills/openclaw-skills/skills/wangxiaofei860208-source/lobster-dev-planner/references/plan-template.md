# 规划方案输出模板

完整规划方案格式，将占位符替换为实际内容后输出给用户。

## 输出格式示例

输出规划时使用以下结构：

一、项目基本信息
- 项目名称：{项目名称}
- 一句话描述：{描述}
- 开发类型：{前端/后端/全栈}
- 部署方式：{部署方式}

二、技术栈配置
【前端】React/Vue + TypeScript + Tailwind + Vite
【后端】Node.js/Python + Express/FastAPI + Prisma + PostgreSQL
【基础设施】Docker + GitHub

三、功能模块规划
【模块1：{名称}】{功能点列表}
【模块2：{名称}】{功能点列表}

四、Agent 团队分工
主控Agent - 规划调度、质量把关
后端Agent - 数据库、API、业务逻辑
前端Agent - 页面、组件、样式
测试Agent - 单元/接口/E2E 测试
文档Agent - 实时同步三份文档

五、MCP 工具规划
GitHub MCP   → 自动 commit/push/Issues
数据库 MCP   → 直接执行 SQL
Docker MCP   → 构建镜像验证
web_search   → 查文档、解决报错

六、开发节点明细（共 N 个节点）
【第一阶段：基础架构】能并行的标注[并行]
  节点1: {名称} [后端+前端并行]
  节点2: {名称} [后端Agent]
【第二阶段：核心功能】
  节点3: {名称}
【第三阶段：联调测试】串行执行
  节点N: 全栈联调+集成测试

七、文档产出规划
1. DEV_DOCUMENT.md   含架构图/ER图/流程图/环境配置
2. {项目名}_API.md   含所有接口详情/示例
3. {项目名}_STYLE.md 含规范/设计系统

结尾提示用户选项：
- 回复"确认"或"开始生成文档" → 生成三份文档
- 回复"修改XXX" → 定向修改
- 回复"重新规划" → 从头来过
