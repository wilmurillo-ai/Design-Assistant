# MQuant 注意事项

> 来源：华泰官方 MQuant 常见问答：https://inst.htsc.com/helpOnline?productId=4&moduleCode=1


---

## 1. MQuant 日常使用须知

### 支持的业务
- A股普通交易及两融交易
- ETF申赎与买卖（内置ETF策略模板）
- 可转债（买卖、可转债转股）交易
- CDR
- 股指期货交易
- 期权交易
- 新股申购

### 策略实例运行须知
1. MQuant 内嵌在 Matic 客户端内部，**不支持 Linux 操作系统**
2. MQuant 可同时运行多个策略实例，**上限为 128 个**
3. Matic 客户端仅限用户**单点登录**
4. MQuant 接受单用户**每秒消息上限为 500 条**（委托数量较多时使用批量报单接口）

### 第三方包
已内置 numpy、pandas、ta-lib，其他第三方库使用【MQuant】-【策略工具】功能安装。

### 多线程
- 策略初始化和结束在另外的线程中，其余回调在一个线程
- **API 接口不是线程安全的**，建议不要多线程调用 MQuant 的 API
- Python 多线程受 GIL 限制，不是真正并行执行

### 多账户
如有关联多账户需求，请联系客户经理办理业务。

### 策略固化
可用策略固化和批量启动，把多个策略固化到一个分组里同时启动。

## 2. 策略执行监控

- **暂停**策略实例：不会重新初始化，进程存在但所有信号不推送
- **停止**策略：进程停止
- 查看日志：需先选中实例，否则不显示

## 3. 数据交互规则

### 读文件限制
- **仅**可在 `initialize()` 和 `on_strategy_end()` 读文件
- 其他回调函数中默认不允许读文件
- 初始化函数中**不可使用** mongodb/redis 或本地 sql 数据库
- 只允许访问内存数据库 **sqlite**
- sqlite 是实例内存数据库，**不支持实例间共享**，不支持数据落地

### 网络访问
- 默认**不允许访问外部网络**（华泰 Insight 行情服务除外）
- 默认**不允许跨进程通信**
- 如果有策略程序需要用到网络通信、进程通信、文件读写这种能力的话，可咨询营业部的服务经理

### 全局变量 g
- 没有任何成员
- 用于存储数据，内容由用户自行定义

### 策略实例间交互
- 可通过 `start_strategy()` 启动新策略实例
- 频率**最少 10 分钟一次**
- 策略运行中修改参数：`on_strategy_params_change(params, path)`

## 4. 回调函数

### 执行顺序
- **顺序执行，不是并发**
- callback1 修改 context 变量后，进入 callback2 的一定是修改后的值

### 订单状态推送
- `open` 状态（已报）：毫秒级推送
- 撤单**没有回报**，能收到撤单导致的原单回报
- 废单在界面日志和系统日志查看原因

### 订单终结判断
- 终结状态：已成、已撤、废单、部撤
- **部撤判断**：订单状态是部撤，且 委托数量 = 撤单数量 + 成交数量

### 注意事项
- 订单回报可能延迟，回报中不要进行大量查询
- 订单状态**不能保证按序**，需策略自行处理

## 5. 报单相关

### 基本规则
- **异步报单**：无需等第一笔成交后再下第二笔
- 初始化**未成功**时**不可以报单**
- 买卖方向：amount **正为买，负为卖**

### order() 和 orders()
- `order()`：单笔报单
- `orders()`：批量报单（更快，最多 20笔/组）

### 撤单
- `cancel_order()`：单笔撤单
- `cancel_orders()`：批量撤单
- 报单和撤单都是**异步执行**，不能保证立即成功

### 报单失败原因
- 资金不足、持仓不足
- 参数填写错误
- 查看策略运行界面日志

### 废单
- 篮子废单：一笔废单整个篮子都废单
- 不同市场分开控（上海/深圳、主板/创业板）
- 市价单按涨停价冻结资金
- 废单可通过 `get_orders()` 查询

### 价格精度
- 期权：精确到后 4 位
- 股票：后 2 位
- 基金：后 3 位
- 上海债券：2 位
- 深圳债券：3 位

## 6. 查询持仓和资金

### 持仓查询
```python
# 方式1：动态获取单只股票
context.portfolio.positions[symbol]
# 注意：如果没有持仓返回 None，需判断

# 方式2：获取所有持仓
get_positions()

# 方式3：推荐
get_positions_ex()  # 支持所有品种
```

### 资金查询
```python
context.portfolio.available_cash  # 可用资金
```

### 更新延迟
- 实盘资金和持仓更新**有延迟**，一般在 **1.5s 以内**
- 强烈建议：**不要强依赖资金和持仓**，建议在策略中维护虚拟资金和持仓

## 7. 行情类

### 支持的行情
- **普通股票**：level-1 实时行情、分钟K线
- **指数**：中证指数、国证指数、申万指数
- **期权**：一档行情
- **期货**：中金所股指期货、国债期货

### K线
- 支持 1/5/15/30/60 分钟、日K线
- **5分钟及以上 K 线仅在收盘后提供**

### 订阅注意事项
- 不要订阅全市场行情（网络带宽和机器处理要求高）
- 先订阅 `subscribe()` 才有推送
- `get_current_tick()` 必须先订阅，建议 sleep 2-3秒后再查询

### Tick
- `tick.datetime` 是**交易所时间戳**，非本地时间

### 停牌股票
```python
tick.trading_phase_code == TradingPhaseCode.TemporarilySuspended
# "8" 表示停牌
```

## 8. 两融交易

### 查询信用资金和持仓
```python
s = MarginTradeHandler.get_margin_assert()
# 信用资产 30s 更新一次
```

### 融券卖出
- 买券还款、卖券还款接口返回 **None**
- 可通过 `get_margin_contract` 查询融券仓位
- 使用 `get_margin_security_info` 查询可融券额度

### 融资买入
- 数量填负数会变成卖券还款
- 建议使用专用接口 `margincash_close()`

---

## 9. 定时类

### 重要
- 定时函数执行时间是**本地时间**，非交易所时间
- **无法从外部停止**定时函数
- 短时间定时信号积压不会丢失，执行后会继续推送
- **已不支持** `run_daily()` 和 `run_monthly()`
- 登录 Matic 后**每 8 小时强制重启**

---

## 10. 期货交易

### 夜盘
- **不支持**期货夜盘

### 投保类型
- 测试环境只支持**投机模式**，不支持套利和保值

### 平仓方式 (close_direction)
- `DEFAULT`('0')：默认，优先平老仓
- `CLOSE_TODAY`('1')：平今仓
- `CLOSE_OLD`('2')：平老仓

### 市价单
- 目前**仅支持中金所市价单**

---

## 11. 期权交易

### 获取合约
```python
# ETF期权
get_symbol_list(ExchangeType.SZ, SecurityType.OptionType, '')
get_symbol_list(ExchangeType.SZ, SecurityType.OptionType, SecuritySubType.ETFoption)
```

### 行权
- 方向填卖出，action 填行权，covered_flag 填非备兑
- 限价价格随便填
- 只有权利仓可以行权
- 义务仓到期自动结算

## 12. 日志

### 自定义日志
- 可用自定义日志打印 debug 信息，写入文件不显示在界面
- 调用 `enable_console_output(enable, label='')` 控制输出

### 注意事项
- **不支持**打印对象（exception 对象获取字符串后可打印）
- **不支持**打印 list 对象
- 日志时间是本地时间
- 只保留**自然日 3 天**日志

### 日志文件
- `XTader`：matic 主进程
- `HTPEM`：Matic 进程 MQuant 专用
- `HTSRE`：每个策略进程
