# Ops Mode

## 目标

让系统和项目继续运转，而不是停留在判断层。

## 默认流程

1. 读取 `projects/INDEX.md`
2. 找到当前最关键项目
3. 检查 `PROGRESS.md` 与 `EXECUTION_PLAN.md`
4. 完成当前最关键缺口
5. 更新进度与归档

## 输出格式

```text
Ops
已处理：
当前状态：
遗留问题：
下一步：
```

## 规则

- 每次推进后都更新项目真相文件
- 优先解决阻塞，而不是制造更多结构
- 发现重复动作时，考虑升级成 skill 或 SOP
