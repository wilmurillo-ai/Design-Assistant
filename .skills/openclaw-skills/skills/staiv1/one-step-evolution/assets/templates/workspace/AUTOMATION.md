# 调度蓝图

这是默认调度蓝图，不是死规则。应根据真实业务调整。

## Heartbeat

- 频率：每 30 分钟
- 建议时段：09:00-22:00
- 目标：低噪音巡检

## Main Cron

建议至少有两类：

1. 进度回执
   - 用于定时同步项目推进
2. 重点提醒
   - 用于早间聚焦或关键时间点提醒

## Isolated Cron

建议至少保留一类：

1. 夜间反思与整理
   - 读取近期 daily log 和项目文件
   - 提炼稳定经验
   - 清理过时指令
   - 更新记忆层

## 调度设计原则

- 对用户可见的输出，走 main cron
- 对系统内部整理，走 isolated cron
- heartbeat 永远保持短
- 每个任务都要写明读哪些文件、输出到哪里、成功标准是什么
