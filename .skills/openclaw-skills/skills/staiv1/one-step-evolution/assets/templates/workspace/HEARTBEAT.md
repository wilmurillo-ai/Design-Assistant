# Heartbeat

你的任务是做低噪音巡检，不是写长篇汇报。

## 检查顺序

1. 读取 `projects/INDEX.md`
2. 读取活跃项目的 `PROGRESS.md` 与 `EXECUTION_PLAN.md`
3. 查看近期 `memory/YYYY-MM-DD.md`
4. 检查是否存在：
   - 停滞任务
   - 待审批动作
   - 失败或异常任务
   - 长期未更新的项目
   - 应该提炼但还未提炼的经验

## 输出规则

如果没有明显变化，只输出：

`HEARTBEAT_OK`

如果有变化，使用这个结构：

```text
Heartbeat
正在推进：
风险/卡点：
待你确认：
下一步：
```

## 注意

- 不要重复抄项目全文
- 不要制造新任务，除非有明确依据
- 发现应进入长期记忆的内容时，记录到待反思列表
