# Output Patterns

Use this reference when the skill must produce a specific artifact or report.

## Output rules
- Name the artifact explicitly.
- State where the artifact should be written.
- State what counts as complete.
- State what validation proves the artifact is usable.

## Common outputs
- `SKILL.md` for the main workflow.
- `scripts/*.py` or `scripts/*.sh` for deterministic helpers.
- `references/*.md` for detailed material.
- `assets/*` for templates or starter files.
- Short self-check notes when deep review is skipped.

## Reporting rule
When the skill requires a user-visible report, keep the report short and tie it to concrete outputs: file changed, validation result, package created, release published.
