# Objection Handling Templates by Product Category

Common buyer objections and copy responses, organized by category. Use as a starting point — always customize for the specific product.

---

## Universal Objections (Apply to Most Products)

| Objection | Copy Response Pattern | Example |
|---|---|---|
| "Too expensive" | Cost-per-use framing | "At $34.99, that's $0.10/day over a year — less than a sip of your morning latte" |
| "Too expensive" | Value comparison | "Compare to [competitor] at $X — same quality, but we include [extra feature]" |
| "What if I don't like it?" | Risk reversal | "Try it for 30 days. If it's not right, return it free — no questions, no hassle" |
| "I'm not sure it'll work for me" | Use case specificity | "Works for [specific scenario A], [scenario B], and [scenario C]" |
| "I've been burned before" | Proof stacking | "[X] verified reviews. [Y] stars. Used by [Z notable customers]" |
| "I'll think about it" | Gentle urgency (only if true) | "Current batch ships tomorrow. Next batch: 2–3 weeks" |

## Apparel & Fashion

| Objection | Copy Response Pattern |
|---|---|

|---|---|

## How to Use These Templates

1. **Identify the top 3 objections** for the specific product (not just the category)
2. **Customize the response** with the product's actual proof points
3. **Place strategically** — price objections near CTA, fit/suitability in benefit blocks, trust in proof section
4. **Never fabricate proof** — if you don't have the data, recommend the merchant gather it
5. **Match the voice** — a luxury skincare brand handles objections differently than a budget kitchen gadget
