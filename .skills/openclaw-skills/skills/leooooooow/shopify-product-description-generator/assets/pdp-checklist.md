# PDP Quality Checklist

Before delivering a product description, verify each item:

---

## Buyer understanding
- [ ] Buyer motive identified (functional / emotional / social / identity)?
- [ ] Purchase trigger understood?
- [ ] Top 3 decision factors identified?
- [ ] Key hesitations mapped?

## Benefit clarity
- [ ] Every feature has a corresponding benefit?
- [ ] Benefits lead (not features) in every section?
- [ ] "So what?" test passed for every claim?
- [ ] No feature listed without explaining why the buyer should care?

## Objection handling
- [ ] Top 3 objections identified for this specific product?
- [ ] Each objection addressed with evidence, not just dismissal?
- [ ] Price objection handled (if product is mid-premium or above)?
- [ ] Objections placed strategically (not clumped in one section)?

## Proof & trust
- [ ] At least one specific proof point included (review, stat, certification)?
- [ ] Numbers are specific, not rounded or vague?
- [ ] No unsubstantiated claims ("best quality", "most popular")?
- [ ] Customer review or testimonial included (if available)?
- [ ] Proof claims flagged for merchant verification?

## Structure & formatting
- [ ] Follows proven PDP structure (headline → hook → benefits → proof → scenario → specs → CTA)?
- [ ] H2 tags used for section headers (not H1)?
- [ ] Bold used for benefit headlines within blocks?
- [ ] Paragraphs max 2–3 sentences (mobile readability)?
- [ ] Scannable on a phone screen?

## CTA & conversion
- [ ] CTA is specific and benefit-driven (not just "Buy now")?
- [ ] Final objection addressed near CTA (guarantee, shipping, returns)?
- [ ] Gentle urgency included (only if truthful)?

## SEO
- [ ] Primary keyword in first 100 words?
- [ ] Primary keyword in at least one H2?
- [ ] Secondary keywords distributed naturally?
- [ ] Meta description suggested (≤ 155 characters)?
- [ ] No keyword stuffing?

## Brand voice
- [ ] Matches brand voice guidelines?
- [ ] Doesn't sound like generic AI copy?
- [ ] Tone appropriate for price point and audience?
- [ ] Avoid-words list respected?

## Shopify readiness
- [ ] HTML formatted for Shopify rich text editor?
- [ ] Can be pasted directly without reformatting?
- [ ] Image alt-text suggestions included?
- [ ] No broken HTML tags?

## Common error check
- [ ] No vague superlatives without proof ("best", "amazing", "revolutionary")?
- [ ] No first-person voice unless brand guidelines require it?
- [ ] No competitor bashing (compare, don't attack)?
- [ ] No false urgency or manufactured scarcity?
- [ ] Regulated claims flagged for review (health, safety, environmental)?
