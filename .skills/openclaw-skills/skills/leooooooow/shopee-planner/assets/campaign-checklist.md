# Shopee Campaign Quality Checklist

Use this checklist before, during, and after every Shopee campaign cycle.

---

## 1. Product Selection (Pre-Campaign)

- [ ] Flash Sale SKUs have ≥4.8★ rating
- [ ] Flash Sale SKUs have ≥50 reviews
- [ ] Flash Sale SKUs have 90-day sales rank in top 20% of store
- [ ] Stock confirmed ≥3× estimated daily sales velocity for each Flash Sale SKU
- [ ] Voucher hero SKUs have AOV in top 30% of store
- [ ] Coins Cashback SKUs have ≥40% gross margin confirmed
- [ ] No SKUs selected that had fulfilment issues in past 30 days
- [ ] New SKUs (<20 reviews) excluded from Flash Sale applications

---

## 2. Pricing & Margin Verification

- [ ] Floor price calculated for all Flash Sale SKUs (COGS + fees + shipping × 1.05)
- [ ] Flash Sale price confirmed above floor price
- [ ] Flash Sale discount ≥10% (Shopee minimum) and ≤30% (margin protection)
- [ ] Gross margin at Flash Sale price ≥20%
- [ ] Voucher minimum spend set at ≥1.5× current AOV
- [ ] Coins rate set only on SKUs with ≥40% gross margin
- [ ] Voucher total issuance cap calculated (daily orders × days × 0.4)
- [ ] No voucher + Flash Sale stack exceeds 35% total discount depth

---

## 3. Campaign Application & Setup

- [ ] Flash Sale application submitted ≥7 days before event (14 days for mega events)
- [ ] Flash Sale slot time selected (12pm / 3pm / 9pm preferred)
- [ ] Flash Sale quantity set at 20–30% of available stock
- [ ] Voucher type selected (store-wide vs. product-specific)
- [ ] Voucher audience selected (all buyers vs. followers only)
- [ ] Voucher validity dates confirmed (no premature expiry)
- [ ] Coins Cashback start/end dates set correctly
- [ ] All campaign settings previewed and confirmed in Seller Centre

---

## 4. Stock & Logistics

- [ ] Inventory count completed for all campaign SKUs
- [ ] Replenishment order placed if stock is below 3× daily velocity
- [ ] Processing time set to 1 business day or less
- [ ] Logistics partner notified of volume spike (if 3PL/warehouse)
- [ ] Shopee preferred seller status confirmed as active
- [ ] Listings updated: hero image, title, description reviewed and optimised
- [ ] All product images show clearly on mobile (test on phone)

---

## 5. Pre-Launch Final Review

- [ ] Campaign calendar confirmed — no scheduling conflicts
- [ ] No overlapping vouchers create unintended stacking discount
- [ ] All Flash Sale applications approved by Shopee
- [ ] Shop decoration / homepage banner updated with campaign creative
- [ ] Shopee Ads (if running) targeting set and budget confirmed
- [ ] Campaign announcement posted on Shop Feed or via Chat Broadcast (optional)

---

## 6. Live Monitoring (During Campaign)

- [ ] Live dashboard checked at campaign start
- [ ] Flash Sale sell-through rate tracked every 30 min
- [ ] Add-to-cart vs. purchase conversion rate monitored (target ≥2.5%)
- [ ] Stock depletion alert set — pause slot if inventory drops below buffer
- [ ] Voucher redemption rate checked hourly (flag if <5% after 2 hrs)
- [ ] No unexpected errors in product listings (price, images, stock showing correctly)
- [ ] Customer messages addressed within 1 hr during Flash Sale windows

---

## 7. Post-Campaign Review (Within 48 Hours)

- [ ] Campaign Report downloaded from Marketing Centre
- [ ] Flash Sale sell-through rate documented (target ≥80%)
- [ ] Flash Sale ROAS calculated and logged
- [ ] Voucher redemption rate documented
- [ ] Basket uplift for voucher orders vs. non-voucher orders measured
- [ ] New followers gained during campaign period noted
- [ ] BSR rank change before/after campaign documented
- [ ] Coins repeat-purchase rate tracked (14-day window)
- [ ] Campaign log updated with key learnings
- [ ] Next cycle adjustments noted (product selection, discount, timing)

---

## 8. Platform Health (Monthly)

- [ ] Seller account health score checked (target: Good or above)
- [ ] Penalty points reviewed — no Flash Sale cancellation penalties
- [ ] Shop rating maintained ≥4.8★
- [ ] Chat response rate ≥90%
- [ ] Shopee Mall eligibility criteria reviewed (if applicable)
- [ ] Preferred Seller status active and renewal date noted
