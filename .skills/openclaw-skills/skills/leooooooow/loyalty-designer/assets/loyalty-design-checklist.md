# Loyalty Program Design Quality Checklist

Use before launching a new program and at quarterly program health reviews.

---

## Pre-Launch Design Checklist

### Financial Foundation
- [ ] Gross margin confirmed for program math baseline
- [ ] Max loyalty program cost calculated (% of revenue that still achieves target contribution margin)
- [ ] Earn rate cost modeled at 20%, 30%, 40% breakage scenarios
- [ ] Tier perk costs estimated for projected enrollment mix
- [ ] Referral incentive cost checked against blended CAC
- [ ] Total program cost is below max allowable % of revenue

### Program Structure
- [ ] Program type matches purchase frequency pattern (points for repeat, tiers for high-AOV, referral for high-CAC categories)
- [ ] All three components integrated: points, tiers, referral reinforce each other
- [ ] Program name and positioning aligned with brand voice
- [ ] Enrollment mechanism defined (opt-in vs. automatic at first purchase)

### Points Mechanics (if applicable)
- [ ] Earn rate set from margin math, not from competitor copycat
- [ ] Minimum redemption threshold requires 1.5–2× AOV in spending
- [ ] Breakage rate assumption documented (minimum 20%)
- [ ] Point expiry policy set (12-18 months recommended)
- [ ] Bonus earn events planned (birthday, double points days, new product launches)

### Tier Mechanics (if applicable)
- [ ] Thresholds based on actual purchase frequency data or reasonable estimates
- [ ] Tier 2 will be reached by no more than 40% of customers
- [ ] Tier 3 will be reached by at least 5% of customers (otherwise not motivating)
- [ ] Each tier includes at least one non-discount perk
- [ ] Downgrade policy defined with customer notification timeline

### Referral Mechanics (if applicable)
- [ ] Dual-sided incentive (both referrer and referee rewarded)
- [ ] Referral reward is less than current blended CAC
- [ ] Unique referral codes or links configured
- [ ] Attribution window defined (30 days recommended)
- [ ] Fraud protection measures in place

### Reward Catalog
- [ ] Mix of discount and non-discount rewards
- [ ] Free shipping option included (high perceived value, variable cost)
- [ ] At least one high-perceived-value, low-cost reward (experience, early access)
- [ ] Minimum redemption thresholds reasonable (achievable within 2-3 normal purchase cycles)

---

## Technical and Operational Checklist

- [ ] Points tracking system selected and configured
- [ ] Referral link generation and attribution tracking tested
- [ ] Member-facing portal or account page shows points balance and tier status
- [ ] Email automation sequences set up for all enrollment, milestone, and expiry triggers
- [ ] Customer support team trained on program mechanics and FAQs
- [ ] Terms and conditions documented (expiry, redemption rules, fraud policy)

---

## Communication Plan Checklist

- [ ] Welcome email template written and tested
- [ ] Points milestone email templates (50%, 100%) built
- [ ] Tier upgrade notification template ready
- [ ] Tier downgrade warning email (30 days out) ready
- [ ] Point expiry warning (30 days out) ready
- [ ] Referral introduction email (triggers after 2nd purchase) ready
- [ ] SMS variants created if program supports SMS communications

---

## Quarterly Health Review Checklist

### Engagement Metrics
- [ ] Enrollment rate (new customers joining program) trending up
- [ ] Points redemption rate reviewed (check breakage against projections)
- [ ] Tier distribution checked (are thresholds still appropriate to customer mix?)
- [ ] Referral conversion rate (link clicks → first purchase) reviewed
- [ ] Average time-to-first-redemption measured

### Financial Health
- [ ] Actual program cost as % of revenue compared to budget
- [ ] Point liability on balance sheet reviewed
- [ ] Referral channel CAC vs. paid acquisition CAC compared
- [ ] LTV of loyalty program members vs. non-members calculated

### Red Flags — Act Immediately
- [ ] **Redemption rate < 30%** → Rewards may be too hard to reach; lower threshold or add easier reward options
- [ ] **Program cost > max allowable %** → Reduce earn rate or increase breakage buffer
- [ ] **Tier 2 enrollment > 50%** → Thresholds too low; plan threshold adjustment with grandfather clause for existing members
- [ ] **Referral fraud spike** → Activate per-account limits and same-address restrictions immediately
- [ ] **Points expiry complaints > 0.5% of enrolled members** → Extend expiry or add activity-based reset rule
