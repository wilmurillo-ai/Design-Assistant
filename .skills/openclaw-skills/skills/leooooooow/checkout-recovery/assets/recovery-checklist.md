# Checkout Recovery — Audit Quality Checklist

Use this checklist to verify that a completed checkout recovery audit covers all critical areas and meets quality standards before it is shared with the seller.

---

## Category 1 — Input Quality (5 items)

- [ ] Checkout completion rate provided as a specific percentage (not just "low")
- [ ] Platform(s) identified clearly (Shopify, TikTok Shop, Lazada, etc.)
- [ ] Target markets specified (not just "international")
- [ ] Current payment methods listed explicitly
- [ ] Error signals or customer complaints described (not just "some issues")

**Category score: __ / 5**

---

## Category 2 — Payment Coverage Assessment (7 items)

- [ ] Missing payment methods identified per market (not just globally)
- [ ] Local/regional payment rails checked for each target market (not only cards)
- [ ] Estimated buyer population affected by coverage gap noted
- [ ] Integration options for missing methods recommended (PSP names)
- [ ] Marketplace facilitator coverage confirmed per platform (TikTok Shop, Amazon, etc.)
- [ ] BNPL options assessed where relevant (Klarna, Atome, Tabby, etc.)
- [ ] COD (Cash on Delivery) assessed where relevant (SEA, MENA)

**Category score: __ / 7**

---

## Category 3 — Friction Audit Coverage (8 items)

- [ ] Error message copy quality assessed (not just "present or absent")
- [ ] Specific error message rewrites provided for critical error types
- [ ] Guest checkout availability confirmed (forced account creation flagged if present)
- [ ] Shipping cost visibility in checkout flow assessed
- [ ] Mobile checkout quality addressed (even if only based on platform description)
- [ ] Trust signals near payment button assessed (SSL, logos, guarantee)
- [ ] Checkout step count noted; progress indicator presence noted
- [ ] Severity rating assigned to each friction item (Critical/High/Medium/Low)

**Category score: __ / 8**

---

## Category 4 — Recovery Sequence Assessment (6 items)

- [ ] Existence of current recovery sequence confirmed (present vs. absent)
- [ ] First touch timing assessed vs. 1-hour best practice
- [ ] Recovery channels documented (email only vs. email + SMS + push)
- [ ] Sequence length assessed vs. 3-touch best practice
- [ ] Recovery email copy quality reviewed (product reference present? direct checkout link?)
- [ ] Discount timing assessed (discount in email 1 flagged as problematic if present)

**Category score: __ / 6**

---

## Category 5 — Remediation Plan Quality (6 items)

- [ ] Recovery Roadmap contains 4–6 prioritized fixes
- [ ] Fixes ordered by expected impact (highest impact first)
- [ ] Each fix includes a specific implementation path (not just "improve error messages")
- [ ] Copy templates provided where relevant (error messages, email subject lines)
- [ ] Timeline estimate included for each fix
- [ ] Retry sequence template included if current recovery is absent or weak

**Category score: __ / 6**

---

## Category 6 — Benchmarking Accuracy (4 items)

- [ ] Benchmark used matches the correct platform (not a different platform's norm)
- [ ] Benchmark accounts for product vertical where relevant (fashion vs. electronics vs. beauty)
- [ ] Gap calculation is stated as a specific percentage difference from benchmark
- [ ] Mobile vs. desktop split mentioned if relevant to the seller's traffic mix

**Category score: __ / 4**

---

## Category 7 — Output Structure (5 items)

- [ ] Three required sections present: Payment Coverage Gap Report, Friction Audit, Recovery Roadmap
- [ ] Payment Coverage Gap Report includes all four required columns
- [ ] Friction Audit uses a table format with severity ratings
- [ ] Recovery Roadmap is numbered and ordered by priority
- [ ] Disclaimer present noting that improvements depend on platform/traffic/market conditions
