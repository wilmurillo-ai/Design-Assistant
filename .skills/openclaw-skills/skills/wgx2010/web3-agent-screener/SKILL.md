# Web3 AI Agent 项目专业筛选器

## 使用方法

用户输入项目名称或描述，直接调用本 skill 进行评估。

## 评估框架（11维度）

对每个项目，从以下维度进行评估：

| # | 维度 | 说明 |
|---|---|---|
| 1 | 流动性 | 是否已上线主流交易所（Binance/Coinbase/OKX等），日均成交量是否支撑大资金进出 |
| 2 | TVL/收入趋势 | TVL 或协议收入是否呈增长趋势，剔除昙花一现的项目 |
| 3 | 核心投资逻辑 | 项目解决的真实问题是什么，叙事是否与实际技术/商业能力匹配 |
| 4 | 真实用户/场景 | 是否有真实活跃用户和可验证的使用场景，剔除纯叙事无落地项目 |
| 5 | 代币价值捕获 | Token 与网络价值/收入是否有直接捕获关系，剔除无效用 Token |
| 6 | 护城河 | 项目是否具有不可复制的竞争壁垒（技术/网络效应/品牌/生态） |
| 7 | 发展阶段 | 评估为：早期（高风险高回报）/ 成长期（最优买入窗口）/ 成熟期（稳定但弹性弱） |
| 8 | 行业痛点解决 | 项目是否真正解决了 Agent 经济赛道的某个结构性痛点 |
| 9 | 竞争优势 | 与行业主要竞争对手相比，差异化优势和领先程度是否明显 |
| 10 | 投资方质量 | 代币持有人/投资者的背景，是否为顶级机构或行业精英 |
| 11 | 核心风险 | 最大单一风险点（技术/监管/代币解锁/竞争/流动性） |

## 工作流程

1. **接收项目名称** → 用户提供待评估项目
2. **信息搜索** → 使用 batch_web_search 搜索项目最新数据（价格、市值、TVL、收入、竞争动态）
3. **11维度评估** → 按框架逐一评估，每个维度需有数据支撑
4. **综合研判** → 结合宏观背景，给出三年+ 投资评级
5. **格式输出** → 按标准格式输出报告

## 数据搜索策略

搜索项目时使用复合关键词：
- `[项目名] [代币符号] 市值 TVL 收入 2026`
- `[项目名] [代币符号] 交易所上线 竞争 护城河`
- `[赛道名] [项目名] 投资方 机构 2026`

## 适用标准（筛选通过门槛）

| 维度 | 通过门槛 |
|---|---|
| 流动性 | 必须上线≥3家主流交易所 |
| TVL/收入 | 必须在增长或稳定 |
| 真实场景 | 必须有可验证用户数据 |
| 代币捕获 | 必须有直接价值销毁/质押机制 |
| 护城河 | 至少一项明显竞争壁垒 |
| 综合评分 | ≥6.5/10 才值得输出完整报告 |

低于以上标准的项目，直接给出"不符合筛选标准，建议放弃"的简短结论。
