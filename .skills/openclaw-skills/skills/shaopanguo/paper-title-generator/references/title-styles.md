# Title Styles Reference Guide

## Venue Analysis

### IEEE Transactions on Automatic Control (TAC)
- **Style:** Highly formal, method-first, theory-focused
- **Structure:** [Method] + [Problem] + [System Type] (occasionally)
- **Forbidden:** Questions, catchy phrases, colons (often), "novel"
- **Word count:** 10-15 words
- **Example:** "Event-Triggered Consensus Control of Multi-Agent Systems With Semi-Markov Switching Topologies"
- **Example:** "Output Feedback Stabilization of Networked Control Systems With Event-Triggered Sampling"
- **Avoid:** "Novel", "New", "Study on", "Research on"

### IEEE Transactions on Industrial Electronics (TIE)
- **Style:** Applied, industry-relevant, practical impact
- **Structure:** [Method/Technology] + [Application Domain] + [Problem]
- **Example:** "Event-Triggered Communication for Formation Control in Multi-UAV Systems"
- **Note:** Can emphasize experimental validation

### IEEE Transactions on Robotics (TRO)
- **Style:** Technical, robotics-specific, precise
- **Structure:** [Method] + [for/in] + [Robot Type/Application]
- **Example:** "Collision-Free Formation Reconfiguration for UAV Swarms Using Distributed Event-Triggered Control"

### Automatica
- **Style:** Rigorous math, theoretical depth, clean prose
- **Structure:** [Contribution] + [Method] + [Problem Class]
- **Example:** "Adaptive Event-Triggered Consensus of Linear Multi-Agent Systems Under Switching Topologies"
- **Note:** Often includes system type: "multi-agent systems", "singular systems", "nonlinear systems"

### CDC / ACC / ECC (IFAC)
- **Style:** Balanced theory/application, clear contribution
- **Structure:** Varies more freely — descriptive or compound
- **Word count:** 12-16 words
- **Example (CDC):** "Event-Triggered Control for Multi-Agent Systems With Guaranteed Convergence"
- **Example (ACC):** "Distributed Formation Control With Event-Triggered Communication: A Hybrid Systems Approach"

### ICRA / IROS (Robotics Conferences)
- **Style:** Engaging, application-aware, can be creative
- **Structure:** Often shorter, robot-specific
- **Example:** "Fly Fast, Talk Less: Event-Triggered Formation Control for Aerial Swarms"
- **Example:** "From Consensus to Collision-Free: Real-Time Formation Control for Multi-Rotor UAVs"

### Nature / Science / Cell
- **Style:** Broad impact, accessible, no jargon
- **Structure:** [Discovery/Insight] + [Context/Implication]
- **Example:** "Adaptive Communication Strategies Enable Efficient Multi-Robot Coordination Without Sacrificing Performance"
- **Avoid:** Acronyms, technical terms (or explain them)

## Title Templates

### Template 1: Method-First (IEEE-style)
```
[Event-Triggered/Adaptive/Distributed] + [Method] + [for/of] + [Problem] + [System]
Example: "Event-Triggered Consensus Control of Linear Multi-Agent Systems"
```

### Template 2: Contribution-First (Automatica)
```
[Contribution] + for/of + [Problem] + in/of + [System]
Example: "Guaranteed-Performance Formation Control for Multi-UAV Systems With Event-Triggered Communication"
```

### Template 3: Question/Curiosity (broad venues)
```
[Question or Hook] + [Method] + [Application]
Example: "Can Multi-Agent Systems Communicate Less Without Losing Control? An Event-Triggered Approach"
```

### Template 4: Compound (ICRA/broader)
```
[Method] + for/of + [Application] + [Problem]
Example: "Consensus-Based Event Triggering Enables Collision-Free UAV Formation Reconfiguration"
```

### Template 5: Innovation Highlight (general)
```
[Novel Aspect] + [Method] + for/of + [Problem]
Example: "Breaking the Communication-Accuracy Tradeoff in Multi-UAV Formation Through Adaptive Event Triggering"
```

## Rejection Patterns (Avoid These)

1. ❌ "A Novel [Method] for [Problem]" — "novel" is red flag at IEEE
2. ❌ "Study on / Research on / Investigation of" — too vague
3. ❌ ">20 words" — titles should be scannable
4. ❌ "Using / Based on" without specificity — what exactly is novel?
5. ❌ All acronym, no plain language — especially for broad venues
6. ❌ Generic: "A New Approach to Formation Control" — what approach?
