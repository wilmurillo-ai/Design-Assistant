# File Organizer

基于 AI 语义分析的智能文件整理技能，采用 Johnny Decimal 分类标准，动态创建分类。

## 特性

| 特性 | 说明 |
|------|------|
| **AI 分类** | 分析文件名语义，动态生成合适的分类 |
| **Johnny Decimal** | 遵循 Johnny Decimal 编号标准和命名规范 |
| **交互模式** | 询问源/目标目录，确认后执行 |
| **目标优先** | 优先使用目标目录现有分类 |
| **零脚本** | 纯 Bash + AI 分析，无外部脚本依赖 |

## 安装

将此技能放置在以下位置之一：

```
~/.claude/skills/file-organizer/SKILL.md
```

或项目本地：

```
.claude/skills/file-organizer/SKILL.md
```

## 使用方法

```bash
# 完整调用
/organize --source ~/Downloads --target ~/Documents/Archives

# 简写
/organize ~/Downloads ~/Documents/Archives

# 仅指定源目录，询问目标目录
/organize ~/Downloads

# 交互式询问所有参数
/organize
```

## Johnny Decimal 分类标准

### 编号规则

采用 `XX.YY` 格式的分类号：
- **XX**：区域编号（00-99）
- **YY**：具体项目编号（00-99）

### 分类示例

```
00-09_系统配置/
10-19_文档资料/
    10.01_报告文档/
    10.02_会议记录/
20-29_图片媒体/
    20.01_截图/
    20.02_设计图/
30-39_代码项目/
    30.01_Python项目/
    30.02_前端项目/
...
```

### 目录命名格式

执行完成后自动更新为：`编号[数量]类别名称`

- **编号**：Johnny Decimal 格式（如 `01-01`, `01-10`）
- **[数量]**：两位数补零（如 `[07]`, `[28]`）
- **类别名称**：语义化的分类名称

## AI 语义分析

- **不依赖扩展名**：通过理解文件名内容进行分类
- **多语言支持**：支持中文和英文文件名
- **动态分类**：根据实际文件内容智能生成分类
- **上下文理解**：识别文件名中的关键词和语义

## 执行流程

1. 解析参数或询问用户
2. 扫描目标目录，检查现有分类
3. 扫描源文件列表
4. AI 分析文件名，生成或匹配分类
5. 显示分类计划
6. 用户确认
7. 创建目录并移动文件
8. 更新目录名称（添加文件数量）
9. 输出统计报告

## 注意事项

1. 建议先备份重要文件
2. 大量文件处理时可能需要较长时间
3. 源目录和目标目录不应为同一目录
4. 需要对目标目录有写入权限
5. 隐藏文件（`.` 开头）会被自动跳过
