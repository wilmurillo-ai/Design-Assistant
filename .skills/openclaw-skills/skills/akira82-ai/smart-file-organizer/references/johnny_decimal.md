# Johnny Decimal 分类标准

## 概述

Johnny Decimal 是一种简单但强大的文件分类系统，使用编号和层级结构来组织文件和文件夹。

## 核心标准

### 标准1：区域划分原则

将所有内容按主题划分为 10 个连续的区域（00-09, 10-19, ..., 90-99），每个区域包含一个主要领域或类别，确保内容有明确归属。

**示例：**
- 00-09: 系统配置与设置
- 10-19: 文档与资料
- 20-29: 图片与媒体
- 30-39: 代码与项目
- 40-49: 工作与职业
- 50-59: 学习与教育
- 60-69: 金融与财务
- 70-79: 健康与生活
- 80-89: 娱乐与爱好
- 90-99: 归档与备份

### 标准2：编号规则规范

采用 `XX.YY` 格式的分类号，其中：

- **XX**（两位整数）：表示区域内的主分类（00-99）
- **YY**（两位小数）：表示该分类下的具体项目（00-99）

**示例：**
- `10.01` - 文档区域第 1 个分类
- `20.05` - 媒体区域第 5 个分类
- `30.12` - 代码区域第 12 个分类

### 标准3：可扩展性设计

通过添加额外小数位实现无限细分（如 `10.01.01`, `20.03.02.01`），保持层级清晰的同时支持内容的不断扩展。

**示例：**
```
30-39_代码项目/
├── 30.01_Python项目/
│   ├── 30.01.01_Web爬虫/
│   └── 30.01.02_数据分析/
└── 30.02_JavaScript项目/
    └── 30.02.01_前端组件/
```

### 标准4：命名规范统一

文件/文件夹命名采用 `[分类ID]_[描述性名称]_[可选信息]` 格式，利用自然排序特性，无需手动调整顺序。

**示例：**
- `10.01_技术文档`
- `20.03_设计草图_v2`
- `30.01_Python爬虫项目_2024`

### 标准5：固定位置映射

每个分类号对应唯一的物理位置（文件夹/目录），形成稳定的 ID-位置映射关系，便于快速定位和检索。

**优势：**
- 分类号即位置，无需记忆复杂路径
- 编号排序自动反映分类层级
- 易于扩展和重组

## 实际应用示例

### 基础结构

```
00-09_系统配置/
    00.01_系统设置/
    00.02_软件配置/
10-19_文档资料/
    10.01_报告文档/
    10.02_会议记录/
    10.03_技术文档/
20-29_图片媒体/
    20.01_截图/
    20.02_设计图/
    20.03_照片/
30-39_代码项目/
    30.01_Python项目/
    30.02_前端项目/
    30.03_工具脚本/
```

### 完整分类编号

**编号递增规则（关键）：**
- **主分类编号（第一部分）**：根据**主题领域**递增
  - 同一大类主题下的所有子分类共享相同的主分类编号
  - 例如：所有技术相关用 `01-XX`，所有商业相关用 `02-XX`，所有行业相关用 `03-XX`
  - 每个主分类编号最多包含 99 个子分类（`01-01` 到 `01-99`）
  - 当发现不同的主题领域时，必须递增主分类编号

- **子分类编号（第二部分）**：在同一主题领域内按顺序递增（`01`, `02`, `03`...）

**示例：**
```
01-01_人工智能技术    # 技术领域
01-02_网络安全        # 技术领域
01-03_编程开发        # 技术领域
02-01_金融经济        # 商业领域（新主题，主编号递增）
02-02_消费零售        # 商业领域
02-03_商业管理        # 商业领域
03-01_教育培训        # 行业领域（新主题，主编号再递增）
03-02_医疗健康        # 行业领域
```

### 分类原则

1. **先识别主题领域**（技术/商业/行业/娱乐等），再细分具体类别
2. 根据文件名的语义相似性分组
3. 分类名称简洁明了（中文或英文）
4. 每个分类包含相关联的文件
5. 优先识别常见模式（代码、文档、图片、压缩包等）

## 最佳实践

1. **保持一致性**：整个系统使用相同的编号格式
2. **预留空间**：每个区域留有足够的编号空间，方便后续扩展
3. **语义化命名**：分类名称应该清晰描述其内容
4. **定期审查**：随着内容增长，可能需要重新分类或调整编号
5. **文档化结构**：维护一份分类索引，记录每个编号的含义

## 常见问题

**Q: 如果某个分类的文件超过 99 个怎么办？**
A: 使用三位小数位（如 `10.01.01` 到 `10.01.99`）进行细分。

**Q: 可以重新分配编号吗？**
A: 可以，但需要谨慎。重新编号可能影响现有文件的查找。建议在系统初期规划好编号结构。

**Q: 如何处理跨分类的文件？**
A: 选择最相关的分类，或者使用符号链接/快捷方式在不改变文件位置的情况下从多个分类访问。

**Q: 编号必须连续吗？**
A: 不必。可以跳过编号（如 `10.01`, `10.05`, `10.10`）以便在中间插入新的分类。
