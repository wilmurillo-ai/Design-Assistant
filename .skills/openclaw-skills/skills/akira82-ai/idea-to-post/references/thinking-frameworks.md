# Detailed Framework Explanations

This document provides detailed explanations of the various thinking frameworks integrated into the `idea-to-post` skill.

---

## 1. 5W1H Analysis Method

### Overview

5W1H is the most comprehensive information collection framework, ensuring coverage of all dimensions of a problem.

### Six Dimensions

| Dimension | Question | Purpose |
|-----------|----------|---------|
| What | What is it? | Clarify core concept/problem/product |
| Why | Why? | Explore reasons, motives, significance |
| Who | Who? | Identify subjects, audiences, stakeholders |
| When | When? | Time dimension, timing selection |
| Where | Where? | Spatial dimension, scenario positioning |
| How | How? | Methods, paths, implementation |

### Applicable Scenarios

- Need to comprehensively introduce a topic
- Project initiation and planning
- Complex problem decomposition and analysis
- Default choice when unsure which framework to use

### Post Example Structure

```
[What] What is AI? A simple definition
[Why] Why is AI important? The changes it brings
[Who] Who is using AI? From individuals to enterprises
[When] Is this the AI era? Timing analysis
[Where] Where is AI applied?
[How] How to start using AI?
```

## 2. SCQA Storytelling Model

### Overview

SCQA is a narrative structure commonly used by McKinsey consultants, very suitable for story-type content.

### Four Stages

| Stage | Meaning | Content |
|-------|---------|---------|
| S (Situation) | Situation | Describe current stable state/background |
| C (Complication) | Conflict | Problems/changes/challenges that appear |
| Q (Question) | Question | How to solve this problem? |
| A (Answer) | Answer | Provide solution/insight |

### Applicable Scenarios

- Telling product/project stories from 0 to 1
- Sharing problem-solving experiences
- Workplace growth stories
- Entrepreneurship journey sharing

### Post Example Structure

```
[S-Situation] Everything looked normal, we worked routinely...

[C-Conflict] Until one day, user complaints surged 30%, the system crashed...

[Q-Question] Why did this problem occur? How should we respond?

[A-Answer] After in-depth analysis, we found the root cause... and found a solution...
```

### Usage Tips

- Make the situation resonate with readers
- The conflict should be dramatic, create tension
- The question should naturally lead to the answer
- The answer should be surprising or inspiring

## 3. Golden Circle Rule (Golden Circle)

### Overview

A classic model proposed by Simon Sinek, thinking from inside out: Why → How → What

### Three Levels

| Level | Question | Description |
|-------|----------|-------------|
| Why (Core) | Why? | Beliefs, original intentions, mission, meaning of existence |
| How (Middle) | How? | Methodology, process, implementation |
| What (Outer) | What? | Specific products, services, results |

### Applicable Scenarios

- Brand stories and philosophy expression
- Product value propositions
- Personal/team mission introduction
- Entrepreneurship original intention sharing

### Post Example Structure

```
[Why-Why] We believe everyone should have the ability to control their own data...

[How-How] Therefore we built an end-to-end encrypted decentralized storage system...

[What-What] The final product presented to users is a simple, secure personal cloud drive...
```

### Usage Tips

- Start with Why - this is the most powerful part
- How should demonstrate unique methodology
- What is the result, but not the focus
- Most people start with What, starting with Why makes you stand out

## 4. AIDA Marketing Model

### Overview

Classic marketing funnel model, guiding readers from attention to action.

### Four Stages

| Stage | Goal | Method |
|-------|------|--------|
| Attention | Attract attention | Shocking data, counter-intuitive viewpoints, suspense |
| Interest | Spark interest | Relevance, pain point resonance |
| Desire | Create desire | Show benefits, social proof |
| Action | Prompt action | Clear action guidance, urgency |

### Applicable Scenarios

- Product promotion
- Course/event promotion
- Conversion-focused copy
- Marketing campaigns

### Post Example Structure

```
[Attention-Attention] 90% of people are learning programming the wrong way...

[Interest-Interest] You spend a lot of time but make slow progress, it's not that you're not smart...

[Desire-Desire] Imagine if there was a method that could help you build independent projects in 3 months...

[Action-Action] Click the link to get my programming learning roadmap for free...
```

### Usage Tips

- Attention should be shocking enough
- Interest should touch pain points
- Desire should paint a beautiful vision
- Action should be specific and clear

## 5. PREP Viewpoint Expression Method

### Overview

A concise and powerful viewpoint expression structure, suitable for expressing opinions and comments.

### Four Parts

| Part | Content | Description |
|------|---------|-------------|
| Point | Viewpoint | Directly state your core viewpoint |
| Reason | Reason | Explain why you hold this view |
| Example | Example | Support with specific cases |
| Point | Summary | Reiterate viewpoint, reinforce impression |

### Applicable Scenarios

- Expressing professional opinions
- Commenting on hot topics
- Sharing experiences and insights
- Workplace reflections

### Post Example Structure

```
[Point-Viewpoint] Remote work is the future trend, enterprises should actively embrace it.

[Reason-Reason] Remote work can reduce costs, improve efficiency, expand talent selection range...

[Example-Example] After our company went fully remote, per capita output increased 40%, turnover rate dropped 50%...

[Point-Summary] So, don't resist remote work, think about how to do it better.
```

### Usage Tips

- Point should be direct, no beating around the bush
- Reason should be logically clear
- Example should be specific and tangible
- Ending Point can echo the beginning, slightly elevating it

## 6. 5-Why Deep Analysis Method

### Overview

The core method of Toyota Production System, finding root causes through continuous questioning.

### Method

Ask "why" five times in a row for each problem, going deeper layer by layer until finding the root cause.

```
Problem: Machine stopped
Why? - Because the fuse blew
Why? - Because the load was too high
Why? - Because the bearing was dry
Why? - Because the oil pump failed
Why? - Because the oil pump had no filter (root cause)
```

### Applicable Scenarios

- Problem root cause analysis
- Failure retrospectives
- Deep technical analysis
- Product problem analysis

### Post Example Structure

```
[Problem] Our user retention rate keeps failing to improve

[Why 1] Why? Users don't come back after using once
[Why 2] Why? Because they can't find the core function
[Why 3] Why? Because product navigation is too complex
[Why 4] Why? Because we're doing addition not subtraction
[Why 5] Why? Because we lack the "less is more" design philosophy (root cause)

[Conclusion] The problem isn't with users, but with our product philosophy
```

### Usage Tips

- Answer honestly at each layer
- Don't stop at surface reasons
- Find actionable root causes
- Can be fewer than 5 times if root cause is found

## 7. First Principles

### Overview

A thinking method advocated by Elon Musk, starting from the most basic facts, decomposing to essence and then reconstructing.

### Three Steps

| Step | Description |
|------|-------------|
| 1. Decompose | Break down the problem to the most basic facts/truths |
| 2. Question | Question all assumptions and "common sense" |
| 3. Reconstruct | Rebuild the solution from basic principles |

### Applicable Scenarios

- Innovation breakthrough
- Disruptive thinking
- Technical architecture design
- Product innovation

### Post Example Structure

```
[Decompose-Basic Facts] What's the essence of a battery? It's a polymer of metals cobalt, nickel, aluminum, carbon...

[Question-Break Assumptions] The market says batteries are expensive, is this true? If buying directly from raw materials...

[Reconstruct-New Solution] Actually raw material cost is only $80/kWh, far below market price of $600...

[Conclusion] So we can make our own batteries, reducing cost by 80%
```

### Usage Tips

- Distinguish "facts" from "opinions"
- Don't be constrained by "everyone does it this way"
- Ask "is this physically possible?"
- Start from physics/economics first principles

## 8. FBA Product Introduction Method

### Overview

Classic product marketing structure, from features to benefits to advantages.

### Three Levels

| Level | Content | Example |
|-------|---------|---------|
| Feature | Feature | What function does the product have |
| Benefit | Benefit | What does this function bring to users |
| Advantage | Advantage | How is it different from other products |

### Applicable Scenarios

- Product introduction
- Feature launch
- Service description
- Comparison reviews

### Post Example Structure

```
[Feature-Feature] Our note-taking software supports bidirectional linking

[Benefit-Benefit] This means you can build knowledge networks, let ideas connect, discover hidden associations...

[Advantage-Advantage] Unlike traditional note software's folder structure, this works like the brain...
```

### Usage Tips

- Feature should be concise and specific
- Benefit should start from user perspective ("what you get")
- Advantage should highlight differentiation
- Can have multiple Feature-Benefit-Advantage combinations

## Framework Selection Decision Tree

```
Your inspiration
    │
    ├─ Contains "why/essence/mission" → Golden Circle
    ├─ Contains "problem/challenge/story" → SCQA
    ├─ Contains "promotion/conversion/marketing" → AIDA
    ├─ Contains "viewpoint/opinion/suggest" → PREP
    ├─ Contains "deep dive/root cause/underlying" → 5-Why
    ├─ Contains "innovation/breakthrough/disrupt" → First Principles
    ├─ Contains "product/feature/advantage" → FBA
    └─ Other/unclear → 5W1H
```

---

## Framework Combination Usage

For complex content, you can combine multiple frameworks:

- **5W1H + SCQA**: First understand background comprehensively, then tell the story
- **Golden Circle + FBA**: Explain philosophy clearly, then introduce product
- **AIDA + PREP**: First attract attention, then express viewpoint
- **5-Why + First Principles**: Find root cause, then reconstruct solution

Don't be limited by frameworks - frameworks are tools, not goals.
