# Publish Evaluation

Use this reference when deciding whether a skill is ready for ClawHub or skills.sh.

## Goal

Decide whether the skill is publish-ready, needs work, or should be held back.

## Checklist

- `name` and folder slug are clear and stable
- `description` includes job, object, outcome, and trigger phrasing
- Example prompts are realistic and visible early
- Commands are portable and use `{baseDir}` where appropriate
- References support the workflow without clutter
- Release blockers are explicit, not implied

## Publish Verdicts

- `ready`: no critical issues, no misleading claims
- `needs-work`: missing trust signals, weak examples, or unclear workflow
- `hold`: broken structure, invalid metadata, or dangerous ambiguity

## Output Guidance

End with a simple verdict:

- ready to publish
- publish after fixes
- do not publish yet
