# Comparison Workflow

Use this reference when comparing two versions of a skill.

## Goal

Identify regressions, not just differences.

## Workflow

1. Evaluate the current version.
2. Evaluate the baseline version.
3. Compare overall score and per-dimension scores.
4. Focus on drops in trigger quality, structure, content, or testability.
5. Explain whether a score change is intentional, harmless, or a real regression.

## Questions to Answer

- Did the trigger surface become clearer or more confusing?
- Did required files, examples, or references disappear?
- Did the skill become broader in a way that hurts precision?
- Did new constraints or prerequisites appear without documentation updates?

## Output Guidance

Report:

- score deltas
- changed strengths
- changed weaknesses
- recommended fixes for each real regression
