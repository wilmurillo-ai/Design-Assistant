## WeGene

**Extension**: `.txt` (tab-separated)
**Header**: First line is column header: `rsid	chromosome	position	genotype`
**Encoding**: UTF-8
**Reference build**: GRCh37 (hg19)

### Example
```
rsid	chromosome	position	genotype
rs548049170	1	69869	TT
rs13328684	1	74792	--
rs9283150	1	565508	AA
```

### Parsing Notes
- Genotype `--` means no call (missing data) — skip these entries
- Genotype is a 2-character string (e.g., `AG`, `TT`) representing both alleles
- Some entries may have `D` or `I` for deletion/insertion markers
- Typical file contains 500,000–600,000 SNPs

## 23andMe

**Extension**: `.txt` (tab-separated)
**Header**: Comment lines starting with `#`, then column header
**Encoding**: UTF-8
**Reference build**: GRCh37 (hg19) for v4/v5 chips

### Example
```
# This data file generated by 23andMe at: ...
# rsid	chromosome	position	genotype
rs548049170	1	69869	TT
rs13328684	1	74792	--
rs9283150	1	565508	AG
```

### Parsing Notes
- Skip all lines starting with `#`
- Format is identical to WeGene after stripping comments
- Genotype `--` means no call
- Indels may appear as `DI`, `DD`, `II`
- v5 chip has ~640,000 SNPs; v4 has ~570,000

### Detection
A file is likely 23andMe if it contains a comment line with "23andMe" in the first 10 lines.

## AncestryDNA

**Extension**: `.txt` (tab-separated)
**Header**: Comment lines starting with `#`, then `rsid	chromosome	position	allele1	allele2`
**Reference build**: GRCh37 (hg19)

### Example
```
#AncestryDNA raw data download
rsid	chromosome	position	allele1	allele2
rs548049170	1	69869	T	T
rs13328684	1	74792	0	0
```

### Parsing Notes
- **Two separate allele columns** instead of one combined genotype column
- Combine allele1 + allele2 to get genotype string (e.g., `T` + `T` → `TT`)
- `0` means no call
- Typical file contains ~700,000 SNPs

### Detection
A file is AncestryDNA if it has "AncestryDNA" in comment header OR has 5 tab-separated
columns (rsid, chromosome, position, allele1, allele2).

## VCF

**Extension**: `.vcf` or `.vcf.gz`
**Standard**: VCF v4.x (typically v4.1 or v4.2)
**Reference build**: check `##reference` or `##contig` header lines

### Key Fields
```
#CHROM	POS	ID	REF	ALT	QUAL	FILTER	INFO	FORMAT	SAMPLE
1	69869	rs548049170	T	C	50	PASS	.	GT:DP	0/0:30
```

### Parsing Notes
- `GT` field encodes genotype: `0/0` = homozygous reference, `0/1` = heterozygous, `1/1` = homozygous alt
- Convert numeric GT to actual alleles using REF and ALT columns
- Multi-allelic sites: ALT may have comma-separated alternatives (e.g., `A,G`)
- `DP` field gives read depth — useful for quality assessment
- WGS VCF typically has 4-6 million variants; chip VCF has 500K-700K
- Check `FILTER` column: `PASS` indicates high-quality calls

### Quality Metrics to Extract
- **Ti/Tv ratio**: transitions / transversions (expect ~2.0-2.1 for WGS, ~1.8 for exome)
- **Average depth**: from DP field
- **Variant count by chromosome**: sanity check for completeness
- **Het/Hom ratio**: typically ~1.5-2.0 for WGS

### Building rsid Index
Not all VCF variants have rsids in the ID column (may show `.`). Build a dual index:
1. `rsid → (chrom, pos, ref, alt, genotype)` for entries with valid rsids
2. `chr:pos → (ref, alt, genotype)` for position-based lookups as fallback

## CRAM/BAM

**Extension**: `.cram` or `.bam`
**Size**: Typically 15-30 GB for 30x WGS

### Purpose
Alignment files are NOT parsed for genotyping directly. They serve as:
1. **Variant verification**: confirm key high-risk SNPs by examining raw reads
2. **Depth analysis**: assess sequencing quality at specific positions
3. **Structural variant evidence**: check for large deletions/insertions

### Tools Required
- `samtools` (v1.10+) for viewing alignments and extracting regions
- `bcftools` (v1.10+) for calling variants at specific positions

### Usage Pattern
```bash
# View reads at a specific position
samtools view -b aligned.cram chr1:12345-12345 | samtools mpileup -

# Call a specific variant
bcftools mpileup -r chr1:12345 -f reference.fa aligned.cram | bcftools call -m
```

### Important: Reference Genome
CRAM files require the reference genome they were aligned against. If the reference is not
available locally, samtools can fetch it from the EBI reference server if the CRAM header
contains the correct `UR` and `M5` tags. Ensure internet access is available.

## Auto-Detection Logic

```python
def detect_format(filepath):
    """Detect genetic data file format from first few lines."""
    with open(filepath, 'r') as f:
        lines = [f.readline() for _ in range(20)]

    text = ''.join(lines)

    if any('23andMe' in line for line in lines):
        return '23andme'
    if any('AncestryDNA' in line for line in lines):
        return 'ancestrydna'

    # Check column structure
    for line in lines:
        if line.startswith('#'):
            continue
        cols = line.strip().split('\t')
        if len(cols) == 4 and cols[0].startswith('rs'):
            return 'wegene'  # or generic chip format
        if len(cols) == 5 and cols[0].startswith('rs'):
            return 'ancestrydna'
        if len(cols) >= 8 and cols[0] in [str(i) for i in range(1,23)] + ['X','Y','MT']:
            return 'vcf'

    # Check file extension
    if filepath.endswith('.vcf') or filepath.endswith('.vcf.gz'):
        return 'vcf'
    if filepath.endswith('.cram'):
        return 'cram'
    if filepath.endswith('.bam'):
        return 'bam'

    return 'unknown'
```
