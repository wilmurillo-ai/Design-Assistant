# 安全边界说明

本文件定义 `c-drive-safe-cleanup` 技能在扫描、交互和删除阶段必须遵守的安全边界。

## 1) 目录边界

- 仅允许扫描并处理白名单目录。
- 白名单外路径一律不扫描、不删除。
- 不触碰系统核心目录，包括但不限于：
  - `C:\Windows\System32`
  - `C:\Windows\WinSxS`
  - `C:\Program Files`
  - `C:\Program Files (x86)`

## 2) 删除边界

- 删除动作仅可发生在用户明确选择“是”的目录组。
- 只删除目录内容，不删除目录本身。
- 文件占用、权限不足时必须跳过并记录，不得强制处理。

## 3) 交互边界

- 每个目录组必须先展示中文说明：
  - 目录作用
  - 垃圾来源
  - 删除影响
- 每项必须独立询问用户，不允许默认全选执行。

## 4) 数据与系统边界

- 不扫描全盘。
- 不自动删除下载/桌面大文件（仅提醒）。
- 不修改注册表。
- 不执行与清理无关的系统配置更改。

## 5) WPS 特殊规则

- 仅检测到 WPS 安装痕迹时，才纳入 WPS 缓存/暂存目录。
- 未检测到时自动跳过 WPS 目录，不报错。
- WPS 清理仅针对缓存/暂存，不删除用户正式文档。

## 6) 演练模式（DryRun）

- 启用 `-DryRun` 时，允许完整执行扫描、说明、确认与日志流程。
- 演练模式禁止执行任何删除动作。
- 日志必须明确标注“DryRun（演练模式）”。
