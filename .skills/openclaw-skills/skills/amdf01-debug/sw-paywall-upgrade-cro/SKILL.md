# Paywall & Upgrade CRO Skill

## Trigger
Design upgrade triggers and paywall experiences that convert free to paid.

**Trigger phrases:** "paywall design", "upgrade flow", "free to paid", "pricing page optimisation", "trial to paid", "feature gating"

## Process

1. **Identify**: Which features drive upgrade intent?
2. **Design gates**: Hard gate vs soft gate vs usage limit
3. **Craft messaging**: What the user gains, not what they lose
4. **Timing**: When to show upgrade prompts
5. **Pricing page**: Optimise for conversion

## Gate Types

### Usage Limits (most effective for tools)
- "You've used 8 of 10 free reports this month"
- Shows value before asking for payment
- Natural upgrade trigger

### Feature Gates (best for clear tier differentiation)
- Core features free, advanced features paid
- Show locked features with preview
- "Unlock this with Pro"

### Time-Limited Trial (best for full-featured products)
- 14 days free, then paywall
- Show countdown: "6 days left in your trial"
- Don't remove access abruptly — grace period

## Pricing Page Principles

- 3 tiers maximum (good-better-best)
- Highlight recommended tier visually
- Annual pricing shown first (higher LTV)
- Feature comparison table with check marks
- FAQ addressing common objections
- Social proof near CTA
- Money-back guarantee reduces risk

## Output

Upgrade flow specification: gate design, trigger points, messaging copy, pricing page structure, and A/B test recommendations.
