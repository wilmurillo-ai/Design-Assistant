# RapidOCR - 专业票据 OCR 识别工具

> ⚠️ **重要**: 首次运行需要网络下载 OCR 模型（约 13MB），后续使用完全离线。

## 🚀 快速开始

### 1. 安装依赖
```bash
pip install rapidocr-onnxruntime
```

### 2. 使用命令

#### 识别发票/票据
```bash
call 票据识别 invoice.jpg
```

#### 文字识别
```bash
call 文字识别 image.jpg
```

#### 批量识别
```bash
call 批量识别 ./invoices/ output.txt
```

## 🌐 网络需求说明

| 阶段 | 网络需求 | 数据量 | 说明 |
|------|---------|--------|------|
| 安装依赖 | **需要** | ~50 MB | `pip install rapidocr-onnxruntime` |
| 首次运行 | **需要** | ~13 MB | 自动下载 OCR 模型文件 |
| 后续使用 | **不需要** | 0 | 完全离线识别 |

**模型存储位置**:
- Windows: `C:\Users\<用户名>\.rapidocr\`
- Linux/macOS: `~/.rapidocr/`

## ⚠️ 已知限制

### 技术限制
- 复杂背景票据识别率较低
- 手写文字不支持
- 极端角度需要图片校正
- 模糊小数点可能识别错误

### 网络依赖
- **首次运行必须有网络连接**
- 模型下载失败时无法使用
- 离线环境需提前在有网环境运行一次

### 建议
- 重要字段（税号、金额）建议人工核对
- 使用清晰、光线充足的图片
- 避免遮挡、折痕、反光

## 🔧 故障处理

### 问题 1：识别结果为空
- 检查图片路径是否正确
- 确认图片格式支持（JPG/PNG/BMP）
- 检查图片是否过于模糊

### 问题 2：依赖安装失败
```bash
# 升级 pip
python -m pip install --upgrade pip

# 重新安装
pip install rapidocr-onnxruntime
```

### 问题 3：首次运行报错/卡住
- 检查网络连接是否正常
- 首次运行会下载 ~13MB 模型文件
- 下载完成后会自动保存到本地缓存
- 如下载失败，可检查防火墙设置

### 问题 4：模型文件缺失
- 删除缓存目录重新下载：
  - Windows: `C:\Users\<用户名>\.rapidocr\`
  - Linux/macOS: `~/.rapidocr/`
- 重新运行程序会自动重新下载

## 🔒 安全说明

### 技能代码本身
- ✅ 无 `exec()` / `eval()` 动态代码执行
- ✅ 无 `os.system()` / `subprocess` 系统调用
- ✅ **无网络请求代码** - 技能代码不发起任何网络调用
- ✅ 无数据上传逻辑

### 依赖包行为（重要）
- ⚠️ **`rapidocr-onnxruntime` 首次调用时下载模型** (~13MB)
- ⚠️ 这是第三方包的行为，不是技能代码的行为
- ✅ 模型来自 PaddleOCR (Apache 2.0 许可)
- ✅ 模型是静态 ONNX 文件，无执行代码

### 数据处理
- ✅ 图片本地处理，不上传云端
- ✅ 识别结果保存在本地
- ✅ 不收集用户数据
- ✅ 不发送遥测信息
