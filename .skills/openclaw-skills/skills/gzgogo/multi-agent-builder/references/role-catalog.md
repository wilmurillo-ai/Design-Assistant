# Role Catalog (Starter)

Use these as defaults, then tailor to the user's mission.

## Global mandatory role (all teams)
Core (always include):
- team-leader

## Product + Engineering
Core:
- product-manager
- tech-architect
- backend-engineer
- frontend-engineer
- qa-engineer

Optional:
- ux-designer
- devops-engineer
- data-analyst
- security-engineer
- technical-writer

## Growth + Marketing
Core:
- growth-lead
- content-strategist
- campaign-operator
- performance-analyst

Optional:
- seo-specialist
- brand-designer
- community-manager
- martech-engineer

## Operations + Support
Core:
- operations-manager
- process-analyst
- support-lead

Optional:
- knowledge-manager
- automation-engineer
- compliance-specialist

## Selection heuristics
- Add a role only when output type is unique and frequent.
- Merge roles if boundaries are unclear.
- Keep one clear owner for cross-role coordination (usually lead/PM).
