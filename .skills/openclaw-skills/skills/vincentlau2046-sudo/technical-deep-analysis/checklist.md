# 技术分析质量检查清单

> 确保每份分析报告符合方法论标准

---

## 📋 分析前检查清单

### 目标定义

- [ ] **分析对象已明确**: 清楚定义了分析什么
- [ ] **分析范围已界定**: 明确了范围边界，排除内容已列出
- [ ] **目标受众已确定**: 知道谁是报告读者
- [ ] **输出形式已定义**: 报告/PPT/备忘录，页数/字数要求

### 信息收集

- [ ] **L1 来源充足**: 至少有 3 个官方文档/论文等 L1 来源
- [ ] **L2 来源补充**: 有行业报告/专家文章作为补充验证
- [ ] **来源清单已建立**: 所有来源已记录，含 URL 和可信度分级
- [ ] **关键信息可验证**: 核心数据有出处可查

---

## 📋 分析中检查清单

### Part 1: 概念定义

- [ ] **术语已溯源**: 谁首次提出、原始定义
- [ ] **演进有时间线**: 概念如何随时间演变
- [ ] **有工作定义**: 明确了本次分析的操作性定义
- [ ] **来源 ≥2 个**: 定义有至少 2 个独立来源支持

### Part 2: 问题诊断

- [ ] **问题有案例**: 每个问题有具体案例或数据支撑
- [ ] **已做根因分析**: 用 5 Why 分析了根本原因
- [ ] **区分症状和根因**: 不是只描述表面现象
- [ ] **问题有优先级**: 按严重程度/影响范围排序

### Part 3: 组件/方案分析

- [ ] **组件有定义**: 每个组件有输入/输出/职责说明
- [ ] **架构图清晰**: 有可视化架构图
- [ ] **组件边界清晰**: 无职责重叠或遗漏
- [ ] **选型有场景**: 选型建议明确了适用场景

### Part 4: 案例研究

- [ ] **案例有代表性**: 能代表一类实践
- [ ] **案例数据完整**: 背景/做法/结果/经验四要素齐全
- [ ] **有跨案例比较**: 对比了共性和差异
- [ ] **区分官方/独立数据**: 标注了数据来源类型

### Part 5: 市场竞争分析

- [ ] **市场规模有依据**: 有数据来源和计算逻辑
- [ ] **竞争者已识别**: 直接/间接/潜在竞争者都已分析
- [ ] **区分事实和推测**: 明确标注了推测内容
- [ ] **四象限图正确**: 竞争格局可视化呈现

### Part 6: 投资风险分析

- [ ] **投资逻辑有数据支撑**: 不是纯主观判断
- [ ] **风险覆盖主要维度**: 市场/技术/竞争/监管/执行
- [ ] **风险有优先级**: 按影响×概率排序
- [ ] **建议有适用条件**: 明确了在什么情况下适用

### Part 7: 框架关系

- [ ] **概念层级清晰**: 有层级图展示关系
- [ ] **特性有定义**: 每个对比维度有明确含义
- [ ] **对比基于文档**: 不是主观印象，有官方文档支撑
- [ ] **提供选择决策树**: 帮助读者做选择

### Part 8: 趋势建议

- [ ] **趋势有时间范围**: 不是无限期预测
- [ ] **标注置信度**: 高/中/低置信度明确
- [ ] **区分必然和可能**: 哪些是确定趋势，哪些是可能趋势
- [ ] **建议有条件限定**: "如果...那么..."格式

---

## 📋 分析后检查清单

### 内容质量

- [ ] **核心结论突出**: 每章有一句话核心结论
- [ ] **论证逻辑完整**: 结论 → 论据 → 分析 → 结论
- [ ] **不确定性已说明**: 已知/未知/假设都列出了
- [ ] **无逻辑矛盾**: 结论与论据无矛盾

### 来源引用

- [ ] **所有来源已引用**: 没有遗漏的参考内容
- [ ] **来源有链接**: URL 或具体位置可查
- [ ] **可信度已标注**: L1/L2/L3/L4 明确标注
- [ ] **格式统一**: 来源引用格式一致

### 可读性

- [ ] **结构清晰**: 章节层次分明，易于导航
- [ ] **表格/图表使用得当**: 复杂信息用可视化呈现
- [ ] **专业术语有解释**: 生僻术语有定义
- [ ] **语言简洁**: 无冗余废话

### 完整性

- [ ] **所有部分已完成**: 8 个部分都有内容
- [ ] **附录完整**: 信息源清单、术语表齐全
- [ ] **检查清单已填写**: 本清单所有项已检查

---

## 📋 特别注意事项

### 禁止项

- ❌ **禁止无来源断言**: 没有来源支撑的事实性陈述
- ❌ **禁止混淆事实和观点**: 观点必须标注为观点
- ❌ **禁止隐藏不确定性**: 必须说明已知和未知
- ❌ **禁止过度推断**: 推测必须说明依据

### 必须项

- ✅ **必须有多源验证**: 核心结论 ≥2 个独立来源
- ✅ **必须标注可信度**: 所有来源必须 L1-L4 分级
- ✅ **必须说明不确定性**: 每部分都有不确定性说明
- ✅ **必须引用来源**: 每部分末尾有来源引用

---

## 📊 质量评分标准

| 维度 | 优秀 (90-100) | 良好 (70-89) | 及格 (60-69) | 不及格 (<60) |
|------|--------------|-------------|-------------|-------------|
| **来源质量** | L1 来源 ≥5 个 | L1 来源 3-4 个 | L1 来源 1-2 个 | 无 L1 来源 |
| **论证严谨** | 所有结论多源验证 | 核心结论多源验证 | 部分结论有验证 | 结论无验证 |
| **结构完整** | 8 部分完整 | 7 部分完整 | 5-6 部分完整 | <5 部分完整 |
| **不确定性** | 全部说明 | 核心部分说明 | 部分说明 | 未说明 |
