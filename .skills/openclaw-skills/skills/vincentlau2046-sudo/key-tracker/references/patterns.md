# 检测模式库

Key Tracker 的智能检测规则。

---

## 一、时间类检测

### deadline（截止日期）

```regex
# 日期 + 截止词
(\d{1,2}月\d{1,2}[日号]).*(截止|之前|前完成|deadline)
(截止|deadline|之前完成).*(\d{1,2}月\d{1,2}[日号])

# 相对时间
(下周|下月|月底|年底).*(完成|交付|上线)
(本周|这周).*前
```

**示例**：
- "3月20日前完成" → deadline: 2026-03-20
- "项目截止日期是下周五" → deadline: 下周五

### milestone（里程碑）

```regex
里程碑|milestone|节点|阶段性|第一阶段|第二阶段
```

## 二、问题类检测

### issue（遗留问题）

```regex
# 问题词
问题|bug|issue|缺陷|错误|故障

# 未完成状态
尚未|还未|待.*完成|未完成|还在做

# 需要解决
需要.*解决|待解决|要处理
```

### blocker（阻塞项）

```regex
卡住|阻塞|等.*才能|需要等|被.*卡住
依赖.*完成|前置条件
```

### risk（风险）

```regex
风险|可能.*问题|担心|隐患|潜在.*问题
可能导致|存在风险
```

## 八、复合检测规则

### 决策 + 原因
```
"决定用A，因为B"
→ decision: 选择A
→ rationale: B
```

### 问题 + 阻塞
```
"问题还没解决，卡在缺少资源"
→ issue: 问题未解决
→ blocker: 缺少资源
```

### 承诺 + 时间
```
"我明天发给你"
→ commitment: 发送
→ deadline: 明天
```

## 九、优先级判断

| 优先级 | 条件 |
|--------|------|
| **高** | 提到截止日期 + 重要事项 |
| **高** | 风险 + 影响大 |
| **中** | 决策 + 多方参与 |
| **中** | 承诺 + 有时间约束 |
| **低** | 纯粹想法/灵感 |
