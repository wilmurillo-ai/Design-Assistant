# 严重等级判断指南

## 判断标准

### 高等级（High）

**定义：** 对患者安全、数据完整性或系统稳定性有直接威胁的问题。

**响应时间：** 24 小时内

**典型场景：**
| 场景 | 示例 |
|------|------|
| 患者安全风险 | 试剂量计算错误导致检测结果偏差 |
| 数据丢失 | 测试结果未保存或丢失 |
| 系统崩溃 | 软件死机、无法启动 |
| 错误结果 | 检测结果与预期严重不符 |
| 核心功能失效 | 无法执行检测流程 |
| 安全隐患 | 电气安全、机械安全相关 |

**判断问题：**
- 是否可能导致患者误诊或延误治疗？
- 是否会导致检测数据丢失或错误？
- 是否会使设备完全无法使用？

**答案任一为"是"→ 高等级**

---

### 中等级（Medium）

**定义：** 影响功能使用但不危及患者安全的问题。

**响应时间：** 3 个工作日内

**典型场景：**
| 场景 | 示例 |
|------|------|
| 功能异常 | 某功能无法正常使用 |
| 性能问题 | 操作响应慢、卡顿 |
| 偶发故障 | 特定条件下出现问题 |
| 流程阻塞 | 需要变通方法才能继续 |
| 数据展示错误 | 界面显示不正确但数据正确 |

**判断问题：**
- 是否影响正常工作流程？
- 是否需要额外的操作步骤？
- 是否影响工作效率？

**答案任一为"是"→ 中等级**

---

### 低等级（Low）

**定义：** 不影响功能使用的问题，主要是体验优化。

**响应时间：** 下个版本

**典型场景：**
| 场景 | 示例 |
|------|------|
| 界面问题 | 按钮位置、颜色、字体 |
| 文字错误 | 拼写错误、翻译不准确 |
| 提示优化 | 错误提示不够友好 |
| 体验建议 | 操作流程可以优化 |
| 性能微调 | 可以更快但不是瓶颈 |

**判断问题：**
- 是否只是"看起来不舒服"？
- 是否不影响实际功能？
- 是否属于"有更好，没有也行"？

**答案任一为"是"→ 低等级**

---

## 边界情况处理

### 情况 1：问题等级不确定

**处理原则：** 就高不就低

如果难以判断，先标记为较高等级，由开发负责人评估后调整。

### 情况 2：问题可能升级

**处理原则：** 动态调整

某些问题初期表现为低等级，但可能发展为高等级：
- 偶发问题 → 频率增加
- 界面问题 → 导致误操作

发现升级趋势时，立即更新严重等级。

### 情况 3：多个问题关联

**处理原则：** 按最高等级处理

如果一个问题引发多个症状，按最严重的症状定级。

---

## 特殊场景

### 医疗器械特有考量

**患者安全永远是第一位。**

即使问题看起来很小，如果涉及：
- 试剂用量计算
- 检测结果判定
- 样本标识追踪
- 质控数据处理

都应提高一个等级评估。

### 监管合规相关

涉及以下法规要求的问题，至少中等级：
- NMPA 注册要求的功能
- 医疗器械软件注册审查指导原则要求
- ISO 13485 质量管理体系要求

---

## 等级调整流程

```
报告人初判 → 开发负责人复核 → 质量部门确认（高等级）
                                    ↓
                            必要时报告管理层
```

### 升级条件
- 问题影响范围扩大
- 发现潜在安全风险
- 客户投诉升级

### 降级条件
- 确认不影响核心功能
- 有临时解决方案
- 问题定位后风险可控

---

## 示例分析

### 示例 1：检测结果偏差 5%

**分析：**
- 影响检测结果准确性 → 可能误导诊断
- 涉及患者安全

**等级：** 高

### 示例 2：界面按钮点击后响应慢 2 秒

**分析：**
- 影响工作效率
- 不影响功能和安全

**等级：** 中

### 示例 3：某个提示文字有错别字

**分析：**
- 不影响功能
- 只是显示问题

**等级：** 低

### 示例 4：偶发性软件死机（100 次出现 1 次）

**分析：**
- 系统稳定性问题
- 虽然偶发但影响严重

**等级：** 高

## 快速判断表

| 问题类型 | 患者安全 | 数据完整 | 功能可用 | 用户体验 | 建议等级 |
|----------|----------|----------|----------|----------|----------|
| 结果错误 | ✓ | ✓ | - | - | 高 |
| 系统崩溃 | - | ✓ | ✓ | - | 高 |
| 功能失效 | - | - | ✓ | - | 中 |
| 性能问题 | - | - | △ | ✓ | 中 |
| 界面问题 | - | - | - | ✓ | 低 |

✓ = 直接影响  △ = 间接影响  - = 无影响
