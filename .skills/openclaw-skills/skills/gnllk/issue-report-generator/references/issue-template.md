## 填写规范

### 问题标题
- 简洁明了，20 字以内
- 包含模块名和问题类型
- 示例：`试剂仓模块 - 更换操作与判读流程时序冲突`

### 严重等级
- **高**：24 小时内响应
  - 患者安全风险
  - 数据丢失/损坏
  - 系统崩溃/无法启动
  - 检测结果错误
- **中**：3 个工作日内响应
  - 功能异常但不影响核心流程
  - 性能问题
  - 偶发性故障
- **低**：下个版本修复
  - 界面显示问题
  - 文字/拼写错误
  - 体验优化建议

### 复现概率
- **必现**：每次都能复现
- **高**：>50% 概率
- **中**：10-50% 概率
- **低**：<10% 概率
- **偶发**：无法稳定复现

### 复现步骤
- 按时间顺序编写
- 每一步清晰可执行
- 包含前置条件
- 标注关键操作点

### 影响分析
从三个维度评估：
1. **患者安全**：是否可能导致误诊、延误治疗等
2. **测试结果**：是否影响检测数据的准确性
3. **用户体验**：是否严重影响操作流程
