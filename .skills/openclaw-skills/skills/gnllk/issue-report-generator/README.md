# 🐛 医疗设备问题报告生成器

根据用户输入的缺陷描述，自动生成符合医疗设备软件规范的问题报告。

## 📌 功能简介

本 Skill 帮助测试工程师、质量工程师和软件开发人员快速生成标准化的问题报告，确保问题描述完整、严重等级准确、便于开发团队定位和修复。

## 🎯 解决什么问题

- ❌ 问题描述不完整，开发无法复现
- ❌ 严重等级判断不准确，优先级混乱
- ❌ 报告格式不统一，增加沟通成本
- ❌ 缺少影响分析，难以评估风险

## ✨ 核心功能

1. **智能分级** - 根据关键词自动判断严重等级（高/中/低）
2. **结构化模板** - 包含现象、复现步骤、环境信息、影响分析
3. **填写引导** - 自动提示需要补充的字段
4. **多平台兼容** - 输出可直接复制到 Jira/禅道/质量管理系统

## 🚀 使用方式

### 触发词

- `生成问题报告`
- `问题报告`
- `缺陷报告`
- `bug 报告`
- `issue report`

### 使用示例

**输入：**
```
生成问题报告：试剂仓更换与进核心舱时序冲突，满载测试时试剂仓 A 耗材用完立即更换会卡住，要等试剂仓 B 判读完成才能操作
```

**输出：**
```markdown
# 🐛 问题报告

**报告编号：** ISSUE-20260329-001

## 基本信息
| 字段 | 内容 |
|------|------|
| 问题标题 | [待完善] 试剂仓更换与进核心舱时序冲突 |
| 严重等级 | 中 |
| 软件版本 | [待填写] |
| 复现概率 | [待填写] |

## 问题描述
**现象：** 满载测试时，试剂仓 A 耗材用完立即更换会卡住

**复现步骤：**
1. 启动满载测试
2. 等待试剂仓 A 耗材用完
3. 立即尝试更换试剂仓 A

**预期结果：** 可以随时更换耗材
**实际结果：** 更换操作卡住
```

## 📊 严重等级标准

| 等级 | 判断标准 | 响应时间 |
|------|----------|----------|
| **高** | 患者安全风险、数据丢失、系统崩溃 | 24 小时内 |
| **中** | 功能异常、性能问题、偶发故障 | 3 个工作日内 |
| **低** | 界面问题、文字错误、体验优化 | 下个版本 |
