# 测试用例分类参考

## 按测试类型分类

### 功能测试（Functional Test）

**目的：** 验证软件功能是否符合需求规格。

**典型用例：**
| 用例编号 | 用例名称 | 优先级 |
|----------|----------|--------|
| TC-001 | 样本检测流程验证 | 高 |
| TC-002 | 试剂余量计算准确性 | 高 |
| TC-003 | 检测结果判定逻辑 | 高 |
| TC-004 | 历史数据查询功能 | 中 |
| TC-005 | 数据导出功能验证 | 中 |

**适用场景：**
- 新功能开发完成
- 功能逻辑修改
- Bug 修复验证

## 测试用例编写规范

### 用例结构

```markdown
## TC-XXX: 用例名称

**类型：** 功能测试/异常测试/UI 测试/...  
**优先级：** 高/中/低  
**模块：** 模块名称

### 前置条件
1. 条件 1
2. 条件 2

### 测试步骤
1. 步骤 1
2. 步骤 2
3. 步骤 3

### 预期结果
- 结果 1
- 结果 2

### 实际结果
[执行时填写]

### 备注
[其他说明]
```

### 编写要点

1. **标题清晰** - 一眼看懂测试目的
2. **步骤详细** - 他人可按步骤执行
3. **预期明确** - 可判断通过/失败
4. **可重复** - 多次执行结果一致
5. **可追溯** - 关联需求和代码
