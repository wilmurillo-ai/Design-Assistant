# 🧪 回归测试用例生成器

当软件发生变更时，根据变更影响范围自动推荐需要执行的回归测试用例，并生成优先级排序。

## 📌 功能简介

本 Skill 帮助测试团队快速确定回归测试范围，根据变更内容智能推荐相关测试用例，按风险等级排序，确保有限时间内聚焦最关键的测试。

## 🎯 解决什么问题

- ❌ 变更影响范围不清晰，测试范围难以确定
- ❌ 回归测试用例太多，不知道优先测什么
- ❌ 时间紧张，需要快速聚焦核心用例
- ❌ 担心遗漏重要测试，导致问题流出

## ✨ 核心功能

1. **智能匹配** - 根据变更内容自动匹配相关测试用例
2. **优先级排序** - 按风险等级分为高/中/低三档
3. **工时估算** - 自动计算预计执行时间
4. **灵活裁剪** - 根据时间情况灵活调整测试范围

## 🚀 使用方式

### 触发词

- `生成回归测试`
- `回归测试`
- `测试用例推荐`
- `regression test`
- `测试建议`

### 使用示例

**输入：**
```
生成回归测试：修改缓冲液消耗计算逻辑，支持单人份独立配置
```

**输出：**
```
变更：修改缓冲液消耗计算逻辑

【高优先级 - 必须执行】
1. TC-032: 缓冲液不足时提示并停止实验
   类型：功能测试 | 预计：15 分钟
2. TC-045: 连续多个实验的缓冲液累计消耗
   类型：功能测试 | 预计：20 分钟

【中优先级 - 建议执行】
3. TC-078: 不同人份数的缓冲液计算
   类型：功能测试 | 预计：15 分钟

【低优先级 - 可选执行】
4. TC-089: 界面显示消耗量格式
   类型：UI 测试 | 预计：5 分钟

预计总耗时：65 分钟
```

## 📊 优先级说明

| 优先级 | 说明 | 执行要求 |
|--------|------|----------|
| **高** | 核心功能、患者安全、数据完整性 | 必须执行 |
| **中** | 重要功能、用户体验 | 建议执行 |
| **低** | 边缘功能、界面展示 | 可选执行 |
