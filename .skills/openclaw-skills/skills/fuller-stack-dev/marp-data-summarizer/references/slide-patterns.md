# Marp Slide Patterns

## Principles

- Summarize, do not transcribe.
- Lead with the answer.
- Use visual hierarchy before decoration.
- Prefer fewer slides with clearer claims.

## Suggested Deck Flow

1. Title and framing
2. What the data says
3. Why it matters
4. Evidence and visuals
5. Actions or conclusion

## Visual Choices

- Use Mermaid for process, sequence, and relationship diagrams.
- Use sparing color accents to highlight deltas, risks, and wins.
- Use short callouts for the strongest insight on each slide.
- Use comparison slides for before/after or option tradeoffs.
- If a slide starts to crowd vertically, split the content across multiple slides rather than shrinking typography too far.
- Render-check the deck before delivery; Marp source that looks fine in Markdown can still overflow in the final slide frame.

## Input Mapping

- CSV or tables: extract trends, outliers, and comparisons.
- JSON or logs: group events, count patterns, and summarize anomalies.
- Research notes: cluster themes, then present the top findings.
- Mixed data: normalize into a few clear claims before drafting slides.

## Slide Writing

- Title: outcome, not topic.
- Bullets: short, concrete, and parallel.
- Captions: explain the visual in one sentence.
- Closing: say what to do next.
- One dense visual plus one short callout is usually the upper limit for a single slide.
