# Question Patterns

## Default shape

When the mode is `deep` or `report`, prefer questions in this shape:

1. `Question`
2. `Why this matters`
3. `Working hypothesis`
4. `Suggested answers`
5. `Free-form`

## Quality rules

- Every question should change the implementation path, validation plan, or sequencing.
- Prefer structural, directional, and decision-shaping questions over generic filler.
- Ask at most one clearly thought-provoking question per round.
- Suggested answers should be plausible, not exhaustive, and should make it easier for the user to
  react quickly.

## Suggested answer options

- Offer `2-4` plausible options when it helps.
- Always leave a free-form answer path.
- Use options to surface likely directions, not to lock the user into a false choice.

## Anti-patterns

- asking for information that can be inspected directly from the repo
- asking duplicate questions across sections
- asking generic filler questions that do not alter implementation choices
- pretending a proposal is final when it is only a working hypothesis
