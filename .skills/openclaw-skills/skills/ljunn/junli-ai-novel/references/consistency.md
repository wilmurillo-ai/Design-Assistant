# 连贯性保证机制

为确保长时间创作的故事连贯性：

## 写前必读

每次开始写新章节前：
1. 阅读 `docs/大纲.md` 中所有已完成章节的摘要
2. 读取上一章文件，了解当前悬念
3. 检查人物状态（位置、情绪、关系）

## 穿针引线

在新章节中：
- 呼应前文埋下的伏笔和线索
- 提及之前发生的事件（自然融入）
- 让人物行为与之前保持一致

### 伏笔回收检查

| 伏笔类型 | 回收标志 |
|----------|----------|
| 物品伏笔 | 物品被使用/破坏/转赠/再次出现 |
| 人物伏笔 | 真实身份被揭示/背景故事展开 |
| 设定伏笔 | 谜团被解开/规则被解释 |
| 承诺伏笔 | 承诺被履行/打破/被提及 |
| 预言伏笔 | 预言被证实/推翻/重新解读 |

### 伏笔识别模式

```
物品类：    "那玉佩..." → 后文必有用途
人物类：    "他究竟是..." → 后文必有揭示  
设定类：    "从未有人..." → 后文必有例外
对话类：    "我会回来..." → 后文必有践行
环境类：    "那座山..." → 后文必有故事
情绪类：    "他从不..." → 后文必有破例
```

### 伏笔记录格式

伏笔记录在 `plot/伏笔记录.md`，格式如下：

```markdown
## 活跃伏笔
| 伏笔名称 | 埋设章节 | 伏笔类型 | 关联章节 |
|----------|----------|----------|----------|
| 玉佩的来历 | 第1章 | 物品伏笔 |          |
| 神秘组织 | 第2章 | 设定伏笔 |          |

## 已回收伏笔
| 伏笔名称 | 埋设章节 | 回收章节 | 备注 |
|----------|----------|----------|------|
| 师父的背叛 | 第1章 | 第5章 | 揭露真相 |
```

## 人物状态跟踪

注意人物在各章节中的变化和成长：
- 位置变化（人在哪里）
- 情绪状态（当前心情）
- 关系变化（与其他角色关系）
- 能力变化（获得新技能/信息）

## 悬念线延续

确保主线悬念逐步推进：
- 每章至少回应一个旧悬念
- 提出新悬念或升级现有悬念
- 不要遗忘任何未解的悬念

## 视角边界

写作和返修时额外检查：

- 本章主 POV 是否稳定
- 是否出现同段乱切脑内
- 非当前 POV 的内心、真实动机和即时判断是否被越权暴露
- 重要未知是否仍通过动作、语气、细节和信息差维持悬念

## 一致性检查清单

- [ ] 人物行为符合其性格设定
- [ ] 前后伏笔有呼应，逻辑闭环
- [ ] 高潮低谷分布合理，节奏恰当
- [ ] 时间线连贯（没有时间跳跃错误）
- [ ] 场景转换自然（没有凭空出现）
- [ ] 主 POV 稳定，没有视角穿帮
- [ ] 信息暴露符合 POV 边界，没有作者越权透题
