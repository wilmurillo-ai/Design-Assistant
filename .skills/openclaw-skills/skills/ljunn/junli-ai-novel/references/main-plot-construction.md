# 主线情节构建专项规则

适用场景：

- 从零立项，但主线发虚
- 有题材和人设，却总想先套结构模板
- 写到中段容易卡文、推翻大纲或重复冲突
- 有事件，但事件之间缺少连锁反应

## 1. 使用顺序

主线规划默认使用这条顺序：

主角目标 -> 起爆事件 -> 连锁反应 -> 结构模板

不要先选三幕式、英雄之旅或章节模板，再倒塞内容。

结构模板只是承载器，不是剧情发动机。

## 2. 第一步：先定终点站

终点站不是抽象主题，而是主角在这本书里最终要拿到、守住、证明或改变的东西。

至少写清：

- 主角最终想要什么
- 他为什么非要得到
- 如果得不到，会失去什么
- 这个目标会逼他变成什么样的人

如果目标还糊，用“纠错提问法”逼清：

1. 先让 AI 或自己用一句话概括主角想要什么
2. 找出不对劲的地方
3. 一轮轮修正目标、欲望、执念和误判
4. 直到角色的行动方向、说话方式和选择逻辑开始稳定

判断标准：

- 看到这个目标，你能自然推演出主角会主动做什么
- 目标本身能持续制造选择、代价和对抗

## 3. 第二步：建立起爆事件

起爆事件的作用，是打破主角原有生活，让故事进入不可逆状态。

合格的起爆事件通常同时满足：

- 带有突发性或强转折
- 不是小噪音，而是会改变主角后续路线
- 一旦发生，主角不能轻易回到原来的生活
- 它会立刻触发下一轮行动，而不是停在设定说明

反向检查：

- 如果删掉这个事件，故事还能原样成立，它就不够硬
- 如果事件只负责热闹，不负责改变局面，它就不是起爆事件

## 4. 第三步：设计连锁反应

主线不是“发生一个大事”就结束，而是让这个大事持续长出后果。

每个核心事件后，至少追问这几层：

### 4.1 直接触发

- 这个意外第一时间造成了什么
- 主角立刻被迫做什么

### 4.2 新境地

优先同时考虑三类后果：

- 好：主角意外得到机会、资源、情报或盟友
- 坏：主角暴露、受损、被追杀、被误解、失去退路
- 意外：出现更高层的人物、力量、规则或变量，把局势带偏

### 4.3 阻碍或转机人物

- 谁会因为这个事件进入故事
- 谁来制造阻碍
- 谁来提供错误希望、短期帮助或危险转机

不要只写“出了事”，要写“谁因此出手”。

## 5. 危机解除与悬念设计

阶段危机的设计，优先用结果倒推法。

先决定：

- 这段危机最后是怎么过掉的
- 主角必须付出什么代价才能过掉
- 危机解除后，什么问题反而更清楚、更麻烦了

结尾不要彻底封口，至少留一个前台未决问题。

常见留钩方式：

- 问题只解决一半，另一半马上可见但被打断
- 新变数突然闯入，迫使主角改选方案
- 危机刚解除，就暴露更大的代价或误判
- 主角拿到了答案的一部分，却因此必须立刻进入下一局

## 6. 最简主线骨架

可反复套用这条链：

目标 -> 起爆事件 -> 直接触发 -> 新境地 -> 阻碍/转机 -> 阶段解法 -> 新悬念

如果任何一段接不上，优先回头检查：

- 主角目标是不是太空
- 起爆事件是不是不够不可逆
- 连锁反应是不是只有坏消息，没有局势变化

## 7. 使用警告

- 不要先写完整大纲，再回头找主角欲望
- 不要把模板当拐杖，模板只在三步法明确后才有意义
- 不要让起爆事件只负责炫设定，不负责推剧情
- 不要让危机解除后一切归零，旧问题最好换形回来

## 8. 逆推式大纲法

主线三步法拎直之后，默认进入逆推式大纲。

不要从“第一章主角出生后做什么”一路顺推到结尾。那样最容易写成雾里开车，写着写着就偏航。

更稳的顺序是：

终极锚点 -> 逆推问题链 -> 分卷锚点 -> 正排成主线

### 8.1 先定终极锚点

先写清全书最后一定要抵达的结果。

可用问题：

- 这本书最后主角到底做成了什么
- 他最终打倒、推翻、揭穿、守住或夺回了什么
- 这个结果为什么值得整本书去写

终极锚点越清楚，中间路线越容易收束。

### 8.2 逆推关键问题

围绕终极锚点往回问，不要怕问题多。

高频问题至少包括：

- 大反派或终极阻碍是谁
- 他为什么必须被打倒，而不是绕开
- 主角最后靠什么赢
- 如果单挑，需要什么境界、筹码、认知或代价
- 如果借势，需要什么伙伴、势力、资源和资格
- 哪些人会阻止主角走到这一步
- 哪件事必须先发生，后面的胜利才成立

每个答案再继续拆子问题，直到你能看见明确剧情事件，而不是只有概念。

### 8.3 分卷锚点

整本书有终极锚点，每一卷也最好有自己的阶段锚点。

常见形式：

- 当众打脸
- 首次越阶胜利
- 身份翻盘
- 揭开阶段黑幕
- 拿到关键资源
- 换地图或换阵营

分卷锚点要满足：

- 够爽，够硬，够能作为一卷结算
- 能反推这一卷前半程必须发生哪些铺路事件

可直接问：

- 这一卷最强爽点是什么
- 为了让这个爽点成立，主角前面必须失去什么、获得什么、练成什么、看清什么
- 哪个反派、哪个场合、哪群观众能把这一击放到最大

### 8.4 正排成线

当终极锚点、问题链和分卷锚点都清楚后，再把答案按时间顺序排列。

这时候大纲不是“凭感觉想章节”，而是把已经被问清的问题，重新排成因果链。

### 8.5 动态调整原则

大纲是导航，不是监工。

如果新灵感更强，而且仍然能通向终极锚点，可以改。

但出现这些信号时，要立刻回到大纲：

- 开始灌水
- 写着写着不知道下一步干嘛
- 事件很多，但都不像主线
- 读者反馈节奏散、冲突弱、期待点不清

## 9. 逆推式卷设计示例问题

给某一卷定锚点后，可直接沿着这条链往回问：

- 为什么这场胜利非赢不可
- 为什么非得在这个场合赢
- 为什么这个对手值得被打
- 主角现在凭什么能赢，凭什么以前赢不了
- 为了赢，他前面必须先失去什么、积攒什么、忍到什么程度
- 这一卷结束后，他拿到了什么新位置，能进入哪张新地图
