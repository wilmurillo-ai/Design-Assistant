# 复杂文档工作流

**适用**：≥ 800 字，或有多个小节，或有"步骤/对比/时间线"等明显结构信号。

## 流程

1. **理解内容**：通读全文，识别结构（教程型 / 评测型 / 长文 / 混合）
2. **转 md**（若非 md 输入），写入 `source.md`
3. **输出结构提案**（必须等用户确认）
4. **根据确认后的提案生成 `article.mdx`**
5. **图片本地化** → **渲染** → **写 meta.json** → **汇报**

## 结构提案格式（Step 3 的输出）

    ## 结构提案

    整体：[教程型/评测型/...]，分 X 小节

    - 开头：[如何处理]
    - 第 1 小节「XXX」：[原文有什么特征] → [建议用什么组件或纯 md]
    - 第 2 小节「YYY」：... → ...
    - ...

    元数据建议：
    - title: ...
    - category: ...
    - tags: [...]

    确认 / 调整？

## 等待用户反馈

用户回复后：

- **"OK" / "继续"** → 按提案生成
- **具体调整**（如"第 3 段不要轮播，用普通图 grid"）→ 按调整后的版本生成
- **整体推翻** → 重新出提案

## 渐进式输出

同简单流程，每步骤即时告知。

## 组件选择纪律

- 严格按 `component-selection.md` 的硬规则（Timeline ≥3 步、ImageCarousel ≥3 张图）
- 组件数量由内容决定，不强行控制比例
- 能用 md 表达清楚的段落**坚决不包组件**
