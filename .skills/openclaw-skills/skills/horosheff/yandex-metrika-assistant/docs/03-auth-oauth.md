# Авторизация (OAuth)

Источник: https://yandex.ru/dev/metrika/ru/intro/authorization  
Быстрый старт (дублирует шаги получения токена): https://yandex.ru/dev/metrika/ru/intro/quick-start

## Требование

Для API Яндекс.Метрики нужен **OAuth-токен** Яндекса. Токен передаётся **в каждом запросе** в заголовке:

```http
Authorization: OAuth <token>
```

Пример из документации (управление):

```http
GET /management/v1/counters HTTP/1.1
Host: api-metrika.yandex.net
Authorization: OAuth 05dd3dd84ff948fdae2bc4fb91f13e22bb1f289ceef0037
Content-Type: application/x-yametrika+json
```

В примерах быстрого старта для JSON также используется `Content-Type: application/json`.

## Ошибки

- **401 Unauthorized** — нет токена, неверный формат заголовка или недействительный токен.
- **403 Access denied** — см. раздел «Возможные проблемы» ниже.

## Создание приложения и получение токена

1. Создать приложение: https://oauth.yandex.ru/?dialog=create-client-entry  
   Тип: **«Для доступа к API или отладки»**.
2. Заполнить название, почту, **доступ к данным** (scopes).
3. Скопировать **Client ID**.
4. Открыть в браузере (implicit flow):

```
https://oauth.yandex.ru/authorize?response_type=token&client_id=<application_id>
```

5. Скопировать выданный **токен** со страницы.

**Важно:** владелец токена — аккаунт, под которым открыли ссылку авторизации, а не обязательно владелец приложения.

## Scopes (доступы приложения)

| Scope | Назначение |
|--------|------------|
| `metrika:read` | Статистика, чтение параметров своих и доверенных счётчиков, список счётчиков |
| `metrika:write` | Создание/изменение счётчиков, загрузка данных |
| `metrika:expenses` | Загрузка расходов в счётчики |
| `metrika:user_params` | Загрузка параметров посетителей |
| `metrika:offline_data` | Офлайн-данные: CRM, офлайн-конверсии, звонки |

Если указан **`metrika:write`**, отдельно **`metrika:expenses`**, **`metrika:user_params`**, **`metrika:offline_data`** не обязательны.

## Организации (porg / логины организации)

Добавить в права приложения **`passport:business`**, если нужен токен от организации.

## 403 — типичные причины

**Приложение:** нет доступа к Метрике — для чтения отчётов и параметров счётчика нужен `metrika:read`, для управления и загрузок — `metrika:write`.

**Токен:** истёк срок; сменили пароль на аккаунте; токен выдан другому пользователю или другому `client_id`.

**Метрика:** у владельца токена нет прав на счётчик (нужен владелец или гостевой просмотр/редактирование — см. справку по доступу).

**Запрос:** заголовок `Authorization` задан неверно или не передаётся.

## Безопасность

Не передавать токен третьим лицам; хранить в секретах, не коммитить в репозиторий.
