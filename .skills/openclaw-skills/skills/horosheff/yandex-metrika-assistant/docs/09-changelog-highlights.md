# История изменений API — выжимка для разработки

Полная лента: https://yandex.ru/dev/metrika/ru/intro/changelog  

Ниже — пункты, полезные при проектировании интеграций (не дублирует весь changelog).

## 2025

### 17 декабря 2025

- Обновлены **квоты**: **200** запросов за **5 минут** к API отчётов с URL **`/stat/v1/data/`**; **400** за **5 минут** к отчётам в интерфейсе.  
  См. также `04-quotas.md`.

### 15 октября 2025

- **Logs API / события**: новые поля (массивы e-commerce, звонки); офлайн-конверсии через **`ym:pv:goalsID`**.
- **Logs API / визиты**: переименования — `ym:s:eventsProduct` → **`ym:s:product`**, `ym:s:products` → **`ym:s:purchasedProduct`**; старые имена временно поддерживаются, затем будут отключены.

### 24 июня 2025

- **Метрика Про**: в визитах логов — **`ym:s:isRobotPro`**; также в Data Streaming (справка Pro).

### 24 февраля 2025

- Новые **метрики** и **группировки** для отчёта «Загрузки файлов» (ссылки в полном changelog на stat/metrics и stat/attributes).

## 2024

### 11 октября 2024

- Управление счётчиками: параметры **`measurement_tokens`**, **`measurement_enabled`** (Measurement Protocol).  
  См. https://yandex.ru/dev/metrika/ru/data-import/measurement-about

### 2 октября 2024

- API **фильтров доступа** (создание, удаление, получение, список, изменение) — пути `.../access_filter/...` в доке management.

### 3 сентября 2024

- **Мониторинг доступности сайтов** в Метрике закрыт.

### 31 июля 2024

- Новые группировки по источнику трафика в **рекламных расходах**: `ym:ev:TrafficSource`, `ym:ev:SourceEngine`.

### 24 июля 2024

- Новые группировки по **событиям** и **дате/времени** (URL события, параметры событий, точность даты и т.д.) — см. полный changelog со ссылками на stat/attributes.

## Рекомендация

Перед релизом интеграции сверять типы полей логов, названия metrics/dimensions и квоты с **актуальной** страницей changelog и openapi-справочниками.
