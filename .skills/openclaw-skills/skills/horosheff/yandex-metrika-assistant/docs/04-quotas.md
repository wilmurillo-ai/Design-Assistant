# Квотирование

Источник: https://yandex.ru/dev/metrika/ru/intro/quotas  
Уточнение лимитов отчётов: https://yandex.ru/dev/metrika/ru/intro/changelog (17.12.2025)

## Ответ при превышении

- HTTP **429 Too Many Requests**
- В теле ответа — информация о **типе** превышенной квоты

## Таблица ограничений

| Ограничение | Квота | Разблокировка |
|-------------|--------|----------------|
| Запросы с одного **IP** (API в целом) | **30** в секунду | Когда за последнюю секунду запросов станет **< 30** |
| Запросы с одного **IP** (**Logs API**) | **10** в секунду | Когда за последнюю секунду **< 10** |
| Параллельные запросы на пользователя (`user_login`) | **3** | Когда одновременных запросов **< 3** |
| Запросы на пользователя в сутки | **5000** | В **00:00 по GMT** |
| Запросы к **API отчётов** URL вида `/stat/v1/data/` | **200 за 5 минут** на пользователя | Через **5 минут** после превышения |
| Запросы к отчётам в **веб-интерфейсе** | **400 за 5 минут** | (аналогично по смыслу ожидания) |
| Добавление **представителей** на один счётчик | **3 в час** | В начале каждого часа (**00 минут**) |

## Лимиты на доступ к одному счётчику

Интенсивность считается **отдельно по группам методов**:

- **`client_read`** — чтение параметров счётчика/аккаунта; относится к методам **API управления** (чтение).
- **`client_write`** — сохранение параметров счётчика/аккаунта.
- **`report_read`** — данные отчётов Метрики: **API отчётов**, конструктор, **GA-совместимый API**.

(Детали по каждой группе — в полном тексте статьи про квоты на сайте Яндекса.)

## Практика для интеграций

- Разделяйте нагрузку: не дергать `/stat/v1/data/` в цикле без задержек — учитывать **200/5 мин**.
- Logs API — отдельный лимит **10 req/s** с IP.
- При 429 — exponential backoff и повтор после интервала из таблицы.
