# Установка Yandex Metrika Assistant — для человека

Читайте **по порядку**. Сначала ставим навык и приложение в Яндексе, потом получаете токен и ключ в OpenClaw. Вопросы по сценарию — [**канал поддержки в Telegram**](https://t.me/maya_pro).

---

## Шаг 1. Установите навык / плагин в OpenClaw

1. Склонируйте репозиторий или скачайте архив:
   ```bash
   git clone https://github.com/Horosheff/yandex-metrika-assistant.git
   ```
2. Установите как **плагин** (рекомендуется, в корне есть `openclaw.plugin.json`):
   ```bash
   openclaw plugins install ./yandex-metrika-assistant
   ```
   Либо скопируйте папку в каталог навыков и используйте `openclaw skills install …` — как принято в вашей среде.
3. В конфиге OpenClaw **включите** плагин `yandex-metrika-assistant` в `plugins.entries` (плагины из workspace часто выключены по умолчанию — включите явно).
4. Перезапустите gateway при необходимости: `openclaw gateway restart`.
5. Назначьте навык нужному агенту (как у вас принято в `skills` / профиле агента).

> Файл **`HOOK.md` не нужен** — это навык + плагин, не hook. Подробности: [`OPENCLAW-AGENT.md`](./OPENCLAW-AGENT.md).

---

## Шаг 2. Создайте приложение в Яндекс OAuth и отметьте доступы к Метрике

1. Откройте [oauth.yandex.ru](https://oauth.yandex.ru/) → создайте приложение.
2. Тип: **«Для доступа к API или отладки»** (или аналог в интерфейсе).
3. Платформа: **Веб-сервисы**. **Redirect URI** (строго так):
   ```text
   https://oauth.yandex.ru/verification_code
   ```
4. В блоке доступа к **Яндекс.Метрике** отметьте галочки — в зависимости от задач:

| Задача | Что включить в приложении (как в кабинете OAuth) |
|--------|---------------------------------------------------|
| Смотреть отчёты, статистику | **Получение статистики, чтение параметров своих и доверенных счётчиков** |
| Менять счётчики, цели, настройки | **Создание счётчиков, изменение параметров своих и доверенных счётчиков** |
| Грузить расходы на рекламу | **Загрузка расходов** |
| Параметры посетителей | **Загрузка параметров пользователей** |
| CRM, офлайн-конверсии, звонки | **Загрузка офлайн данных (данные из CRM, офлайн конверсии, звонки)** |

Для большинства сценариев «спросить Метрику у AI» достаточно **чтения + записи** (первые две строки). Остальные — если реально грузите данные в Метрику.

5. Сохраните приложение и скопируйте **Client ID**.

Подробнее про `scope` и ссылки авторизации: [`INSTRUCTION-GET-TOKEN-RU.md`](./INSTRUCTION-GET-TOKEN-RU.md).

## Шаг 4. Получите OAuth-токен и добавьте ключ в OpenClaw

1. Пройдите выдачу токена по инструкции: [**INSTRUCTION-GET-TOKEN-RU.md**](./INSTRUCTION-GET-TOKEN-RU.md) (ссылка с `response_type=token` и вашим `client_id`).
2. Скопируйте **`access_token`** из адреса после `#` (не путайте с «кодом подтверждения» из другого потока).
3. В OpenClaw вставьте токен в конфиг плагина **`yandex-metrika-assistant`** → поле **`oauthToken`**, либо задайте переменную **`YANDEX_METRIKA_OAUTH_TOKEN`** (если ваш хост так настроен).
4. Опционально задайте **`defaultCounterId`**, чтобы агент не спрашивал номер счётчика каждый раз.

## После добавления ключа — с чего начать

Напишите агенту одну из фраз (или свою):

1. **«Покажи список моих счётчиков Метрики»** — проверка, что токен и права работают.  
2. **«Визиты по дням за последние 7 дней для счётчика …»** — базовый отчёт по датам.  
3. **«Откуда трафик за месяц?»** / **«Сводка по источникам»** — источники и каналы.  
4. **«Топ страниц по просмотрам»** — какие URL набирают больше всего просмотров.  
5. **«Сколько было достижений цели …»** — конверсии (нужен id или название цели).  

Больше идей: [`10-user-intents-matrix.md`](./10-user-intents-matrix.md), примеры команд: [`EXAMPLES.md`](./EXAMPLES.md).
