# Как получить ключ доступа к API Яндекс.Метрики

**Поддержка и вопросы по установке:** канал [**@maya_pro в Telegram**](https://t.me/maya_pro). Токены и пароли в открытый чат не отправляйте.

Перед первым получением токена удобно пройти полную установку: **[INSTALL-FOR-HUMANS-RU.md](./INSTALL-FOR-HUMANS-RU.md)**.

---

Для запросов к API Метрики нужен не отдельный «API-ключ из кабинета Метрики», а **OAuth-токен** Яндекса. Его выдаёт сервис [Яндекс OAuth](https://oauth.yandex.ru/) после входа в аккаунт и согласия прав.

Ниже — способ, который использует **неявный поток** (`response_type=token`): после входа токен появляется **в адресной строке браузера** (во фрагменте URL после символа `#`).

---

## Что понадобится заранее

1. Аккаунт на Яндексе с доступом к нужным счётчикам Метрики.
2. **Приложение** в [Яндекс OAuth](https://oauth.yandex.ru/). Ниже — что выбрать при создании и в настройках.

### Тип приложения

- Выберите вариант вроде **«Для доступа к API или отладки»** (название в интерфейсе может слегка отличаться).

### Доступ к данным: отметьте нужные пункты по Метрике

В блоке прав доступа к **Яндекс.Метрике** включите те возможности, которые вам нужны. Для полного набора (отчёты + управление счётчиками + загрузки) отметьте **все** перечисленные ниже — формулировки приведены как в кабинете OAuth (если у вас текст чуть другой, ориентируйтесь по смыслу):

- [ ] **Получение статистики, чтение параметров своих и доверенных счётчиков** — соответствует `metrika:read`.
- [ ] **Создание счётчиков, изменение параметров своих и доверенных счётчиков** — соответствует `metrika:write`.
- [ ] **Загрузка расходов** — `metrika:expenses` (по документации Метрики не обязателен в ссылке, если уже есть `metrika:write`, но галочка в приложении нужна, если вы грузите расходы).
- [ ] **Загрузка параметров пользователей** — `metrika:user_params` (аналогично: для загрузки параметров галочка должна быть в приложении).
- [ ] **Загрузка офлайн данных (данные из CRM, офлайн конверсии, звонки)** — `metrika:offline_data` (нужна для CRM/офлайн-конверсий/звонков).

Если какой-то из типов доступа вам **не нужен**, снимите соответствующую галочку и **уберите** лишнее из параметра `scope` в ссылке авторизации (см. таблицу ниже), иначе возможна ошибка `invalid_scope`.

### Платформа и Redirect URI

- Укажите платформу **«Веб-сервисы»** (или аналог), добавьте **Redirect URI**:

  `https://oauth.yandex.ru/verification_code`

  Он должен быть **точно таким же** в настройках приложения и в ссылке из шага 1 ниже.

### После сохранения приложения

- Сохраните приложение и скопируйте **Client ID** (идентификатор приложения) из карточки приложения — он понадобится в ссылке авторизации.

## Права Метрики в параметре `scope`

В ссылке авторизации права перечисляются в `scope` через **пробел** (в URL пробел кодируют как `%20`). В кабинете [oauth.yandex.ru](https://oauth.yandex.ru/) должны быть включены те же возможности, что вы перечисляете в `scope`, иначе возможна ошибка `invalid_scope`.

| Возможность | Значение в `scope` |
|-------------|-------------------|
| Получение статистики, чтение параметров своих и доверенных счётчиков | `metrika:read` |
| Создание счётчиков, изменение параметров своих и доверенных счётчиков | `metrika:write` |
| Загрузка расходов | `metrika:expenses` (не обязателен, если есть `metrika:write`) |
| Загрузка параметров пользователей | `metrika:user_params` (не обязателен, если есть `metrika:write`) |
| Загрузка офлайн данных (данные из CRM, офлайн конверсии, звонки) | `metrika:offline_data` (не обязателен, если есть `metrika:write`) |

Если в приложении отмечены все нужные галочки Метрики, часто достаточно запросить **`metrika:read`** и **`metrika:write`** — остальные три scope из таблицы дублируют часть возможностей `metrika:write` и могут не пониматься в ссылке, если не добавлены отдельно в настройках приложения.

## Шаг 1. Соберите ссылку для входа

Подставьте **свой** `client_id` вместо `ВАШ_CLIENT_ID`.

Права в примере: чтение и запись Метрики (`metrika:read` и `metrika:write`). Если в приложении не включены какие-то из запрошенных прав, авторизация может завершиться ошибкой `invalid_scope` — тогда уберите лишнее из параметра `scope` или добавьте галочки в настройках приложения.

```
https://oauth.yandex.ru/authorize?response_type=token&client_id=ВАШ_CLIENT_ID&redirect_uri=https%3A%2F%2Foauth.yandex.ru%2Fverification_code&scope=metrika%3Aread%20metrika%3Awrite
```

**Рабочий пример** (замените на свой Client ID, если отличается):

```
https://oauth.yandex.ru/authorize?response_type=token&client_id=c244cafa27d047dd87dfa6189aa48fc4&redirect_uri=https%3A%2F%2Foauth.yandex.ru%2Fverification_code&scope=metrika%3Aread%20metrika%3Awrite
```

Откройте ссылку **в браузере** (лучше в режиме обычного окна, чтобы было проще скопировать адрес).

---

## Шаг 2. Войдите и разрешите доступ

1. Войдите под тем аккаунтом Яндекса, для которого нужен доступ к Метрике.
2. На экране согласия нажмите разрешение доступа приложению.

---

## Шаг 3. Скопируйте токен из адреса

После успешного входа браузер перенаправит на страницу вида:

`https://oauth.yandex.ru/verification_code#access_token=ДЛИННАЯ_СТРОКА&token_type=bearer&expires_in=...`

1. Посмотрите на **полный адрес** в строке браузера (иногда фрагмент `#...` не виден сразу — нажмите на адресную строку или «Изменить» / полный URL).
2. Найдите в части после `#` параметр **`access_token=`**.
3. Скопируйте значение **от первого символа токена до следующего символа `&`** (без слова `access_token=`).

Это значение и есть **ключ**, который нужно передавать в API Метрики в заголовке:

```http
Authorization: OAuth <скопированный_access_token>
```

Пример для проверки (подставьте свой токен):

```bash
curl -s -H "Authorization: OAuth ВАШ_ТОКЕН" "https://api-metrika.yandex.net/management/v1/counters?per_page=5"
```

При успехе вернётся JSON со списком счётчиков и код ответа **200**.

## Срок жизни токена

У токена есть срок действия (в URL обычно есть `expires_in` в секундах). Когда срок истечёт, нужно **снова открыть ссылку из шага 1** и получить новый токен.

Для серверных сценариев без постоянного открытия браузера чаще используют поток с **кодом** (`response_type=code`) и **обмен на токен** с `client_secret`, плюс **refresh_token** — это описано в [документации Яндекс OAuth](https://yandex.ru/dev/id/doc/ru/codes/code-url).

## Безопасность

- Не отправляйте токен в чаты, не публикуйте в скриншотах и **не коммитьте** в репозиторий.
- Храните только в защищённом месте: менеджер секретов, переменные окружения на сервере, зашифрованные настройки.
- Если токен утёк — отзовите доступ приложения к аккаунту или перевыпустите токен через повторную авторизацию; при необходимости смените **Client secret** в кабинете OAuth.

## Частые ошибки

| Проблема | Что сделать |
|----------|-------------|
| `invalid_scope` | В ссылке запрошены права, которых нет у приложения в [oauth.yandex.ru](https://oauth.yandex.ru/). Упростите `scope` (например, только `metrika%3Aread`) или добавьте галочки в настройках приложения. |
| Нет `#access_token` в адресе | Убедитесь, что в ссылке указано `response_type=token` и разрешён такой тип для вашего приложения. Проверьте, что `redirect_uri` совпадает с настройками. |
| 401 у API Метрики | Токен истёк или скопирован не полностью; заголовок должен быть ровно `Authorization: OAuth ` + токен без кавычек. |
