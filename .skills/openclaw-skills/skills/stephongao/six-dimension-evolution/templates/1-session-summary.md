# 昨日会话总结任务

## ⚠️ 执行前强制检查（必须先做，再执行后续步骤）

列出昨日所有 session，**逐一检查每个 session 的 `origin.label` 字段**，按以下规则分类：

| origin.label 值 | 分类 | 处理方式 |
|---|---|---|
| `"heartbeat"` | [SYSTEM] | **跳过，不读取，不总结** |
| `"user"` / `"feishu"` / `"tui"` / 空值 | [USER] | 纳入总结 |
| 其他未知值 | [USER] | 纳入总结（宁可多读） |

> **如果过滤后 [USER] 类型的 session 数量为 0**：
> 将以下内容写入 `memory/YYYY-MM-DD.md`，然后**停止本任务，不再继续执行**：
> ```
> # 📅 YYYY-MM-DD 记忆
> 昨日无用户发起的会话。
> ---
> *执行时间: [时间戳]*
> ```

> **如果所有 session 都是 [SYSTEM] 类型**，说明数据源读取存在问题，情况同上，写文件后停止。

## 执行步骤

**步骤 1 - 枚举 session 列表（数据准备）**

获取昨日会话及子目录中的**全部** session，列出：
- 每个 session 的文件名 / ID
- `origin.label` 字段值
- session 的起止时间戳

> **枚举完成确认（必须输出，不得跳过）**：
> 在进入步骤 2 之前，必须先输出以下确认信息：
> ```
> 共找到 N 个 session：
> [USER]   X 个：[id1, id2, ...]
> [SYSTEM] Y 个：[id3, id4, ...]
> 将处理以下 session：[USER session ID 完整列表]
> ```
> 输出此确认后，逐一核对 ID 列表是否完整，再进入步骤 2。
> 不得在未输出确认的情况下直接跳入内容提取。

将所有 session 按上表分类，**只对 [USER] 类型的 session 执行步骤 2**。

**步骤 2 - 逐 session 提取内容**

对每个 [USER] session，只读取以下内容：
- `role: "user"` 的消息（用户说了什么、决定了什么、问了什么）
- Agent 的关键决策性回复（执行了什么、确认了什么结论）

**跳过**以下内容，不提取，不总结：
- 心跳触发消息（`HEARTBEAT` 关键词、定时触发标记）
- 自动推送的报告、简报、事件监控结果
- 纯系统状态检查（无用户输入的会话片段）

**步骤 3 - 主题聚合与意图提炼**

将步骤 2 提取的内容按主题聚合，每个主题**必须回答两个问题**：
1. 发生了什么？（事实）
2. 用户想要什么？（意图 / 偏好 / 决策）

没有明确用户意图的内容，不单独成主题，合并到相关主题的要点中。

同时识别以下内容并单独记录：
- **失败/错误事件**：Agent 执行出错、操作失败、产生不符合预期的结果 → 提炼为教训条目（`## 教训` 章节）
- **用户纠正行为**：用户明确指出 Agent 做错了或重新下达指令 → 同上
- 教训格式：`L[序号] [类别]: 描述 | 触发：[事件] | 改进措施：[措施] | ⏳ 待验证`

**步骤 4 - 写入前逐条自查（每条内容写入前执行）**

对每一条准备写入的内容，必须逐项确认：

```
□ 这条内容来自 [USER] session 中用户的真实输入或用户触发的交互？
□ 这条内容不是心跳任务的执行记录（复盘、备份、简报、监控）？
□ 这条内容不是系统自动生成、无用户参与的内容？
□ 这条内容不是新闻事件、市场行情、网络搜索结果？
```

**以上四项全部为"是/否/否/否"才写入。任何一项不满足，跳过该条。**

**步骤 5 - 重复性检测后写入**

在写入 `memory/YYYY-MM-DD.md` 之前，读取最近两日的记忆文件做比对：
- 字段/条目已存在 → 合并优化或跳过，不重复写入
- 内容无冲突 → 按格式写入

**步骤 6 - 更新时间戳**

记录并更新 `lastSessionSummary` 时间戳到 `config/heartbeat-state.json`。

## 📝 每日记忆格式 (memory/YYYY-MM-DD.md)

```markdown
# 📅 YYYY-MM-DD 记忆

## 会话总结

### [主题名称]
**背景**：为什么做这件事？（上下文）
**目标**：想达成什么？
**进展**：做了什么？结果如何？
**待办/下一步**：还有什么没做的？
**用户意图/偏好**：[必须填写，不能省略]

### [主题名称]
...

## 教训（如有）
- L[序号] [类别]: 描述 | 触发：[事件] | 改进措施：[措施] | ✅/⏳ 状态

---
*来源 sessions: [session ID列表]*
```

**⚠️ 记录原则**：
1. **动脑子**：每个主题必须问清"背景、目标、进展、待办"四个维度
2. **抓重点**：只记录有价值的信息，拒绝流水账
3. **无意义内容不记录**：如"新会话启动"、"读取配置文件"等系统常规操作
4. **用户意图必须填写**：不能省略或敷衍

## 其他（如有）

---
*来源 sessions: [session ID列表，精确到具体ID，如 de981d0d, c7804399]*
```

**说明**：教训部分只在有内容时才写入，无则省略。来源 sessions 必须列出具体 session ID，不接受 "main session" 等模糊描述。

**填写规则**：
- 所有字段均可为空，无内容则省略该行或整个章节
- 优先保证「会话总结」质量，其他维度有则记录
- 主题尽量全面，覆盖昨日所有 [USER] session 中有意义的主题
- 每个主题必须包含「用户意图/偏好」标注，无法提炼则不单独成主题
