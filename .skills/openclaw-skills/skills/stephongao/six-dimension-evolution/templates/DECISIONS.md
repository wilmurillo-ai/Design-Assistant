## 📝 决策记录规范

### 何时记录决策？

**必须记录**：
- 给出方向性建议（如职业规划、技术选型、策略方向）
- 用户采纳了我的建议并付诸行动
- 涉及重要资源分配的决策
- 用户明确要求"你怎么看"、"帮我判断"

**无需记录**：
- 简单事实查询
- 纯技术操作
- 日常闲聊

### 决策状态流转

```
[活跃] → 观察7-30天 → 验证效果
              ↓
    ┌─────────┴─────────┐
    ↓                   ↓
[成功] → SUCCESSES   [失败] → LESSONS
```

### 验证问题模板

每个决策需定义"如何验证成功"：
- **量化指标**：收入增加、时间节省、错误减少
- **质化反馈**：用户满意度、问题是否解决
- **行为变化**：用户是否持续采用建议

### 格式

```markdown
| D[序号] | YYYY-MM-DD | 简述决策 | 触发场景 | 验证问题 | 状态 |
```

**验证失败时**：复制到 LESSONS.md

## 📋 维护说明

- **记录时机**：给出重要建议后立即记录
- **验证周期**：7-30天，视决策类型而定
- **更新频率**：每日复盘时检查活跃决策
- **归档时机**：验证完成后移入对应分类
