# 每日复盘任务

## 执行步骤

1. 读取昨日 `memory/YYYY-MM-DD.md`（主）。
2. 提取有价值的用户意图、偏好、习惯、教训、经验等。
3. 根据信息思考如何改进未来的交互方式。
4. 必要时在通过一致性检查后更新核心记忆库 `MEMORY.md`：
   - 用户偏好/习惯变化 → 更新用户画像部分
   - 新教训/失败模式 → 更新教训库部分
   - 教训验证结果 → 更新验证状态
5. **决策回顾**：检查 `DECISIONS.md` 活跃决策的验证状态：
   - 有新结果的决策 → 更新状态，移入成功/失败分类
   - 验证成功的 → 复制到 `SUCCESSES.md`
   - 验证失败的 → 复制到 `LESSONS.md`
   - 昨日新提出的重要建议 → 记录到活跃决策
6. **技能统计**：检查 `SKILL_STATS.md` 及各 skill 的 `logs.json`：
   - 汇总昨日各 skill 调用情况
   - 关注成功率 < 70% 的 skill
   - 对问题 skill 提出优化建议
   - 将预警记录到 `LESSONS.md`
7. **知识盲区检查**：检查 `KNOWLEDGE_GAPS.md`：
   - 回顾昨日对话，检测潜在盲区（模糊词汇、不确定回答）
   - 评估现有盲区优先级
   - 安排高优先级盲区的学习计划
   - 检查是否有已填补的盲区，更新状态
8. 生成总结报告放在昨日的 `review/memory/YYYY-MM-DD.md`。
9. 报告最后加上你作为智囊视角的评价。
10. **推送完整报告**：通过飞书交互卡片（Interactive Card）推送**完整内容**，不得简化或省略任何部分。
11. 记录并更新 `lastDailyReview` 时间戳。

---

## 变更前一致性检查

在任何自动修改 `MEMORY.md` 之前，先对以下配置/定义文件做完整校验和比对：`AGENTS.md`、`SOUL.md`、`USER.md`、`MEMORY.md`：
- 目的：避免重复添加、覆盖关键配置或引入逻辑冲突
- 检查内容：
  - 字段/条目是否已存在
  - 时间戳与来源
  - 语义冲突（例如同一偏好被多处不同描述）
  - 约束/策略冲突（例如行为规则与新记忆相悖）
- 如果检测到确切重复条目：优化合并或跳过写入
- 如果检测到潜在语义冲突：将冲突条目写入 `review/conflicts/YYYY-MM-DD_HH-mm-ss.md`（包含冲突摘要、相关文件路径、来源 session 引用与建议处理动作），并触发飞书通知以便人工复核（根据严重性可阻断自动写入）
