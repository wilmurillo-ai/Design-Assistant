## 📝 统计更新规范

### 何时更新

- **每次 skill 执行后**：判断结果，更新对应统计
- **每日复盘时**：汇总统计，检查预警

### 更新流程

```
Skill 执行
    ↓
判断结果（成功/失败）
    ↓
┌───────────────────────────────────┐
│ 1. 更新本文件汇总统计              │
│ 2. 更新 skill 内部 logs.json      │
│    （失败时记录详细原因）          │
└───────────────────────────────────┘
    ↓
成功率 < 70%？
    ↓ 是
触发预警，记录到 LESSONS.md
```

### 成功/失败判定标准

**成功**：
- 返回了预期结果
- 用户未表达不满
- 无需重试

**失败**：
- 返回错误或空结果
- 用户明确表示不满意
- 需要手动补救

## 📁 详细日志位置

每个 skill 的详细执行日志存放在：

```
skills/<skill-name>/logs.json
```

格式：
```json
{
  "invokes": [
    {
      "timestamp": "YYYY-MM-DDTHH:mm:ss+08:00",
      "platform": "场景标识",
      "result": "success|failure",
      "duration_ms": 1500,
      "error": "失败原因（如有）"
    }
  ]
}
```
