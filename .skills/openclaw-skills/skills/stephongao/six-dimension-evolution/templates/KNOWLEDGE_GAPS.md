## 📝 触发条件与检测逻辑

### 自动检测场景

| 触发场景 | 检测方式 | 优先级 | 示例 |
|----------|----------|--------|------|
| 用户提及陌生概念 | 关键词识别 + 回复中表达不确定 | 高 | "我对 Rust 异步不太熟悉..." |
| 回答后用户纠正 | 用户反馈"不对"、"其实" | 高 | 用户纠正了我的理解 |
| Skill 失败率高 | SKILL_STATS < 70% | 高 | 某平台抓取失败率高 |
| 回复中模糊词汇 | 检测"可能"、"应该"、"大概" | 中 | "这个可能是..." |
| 用户领域新动态 | 心跳时扫描相关新闻 | 中 | AI Agent 新框架发布 |
| 相关概念关联 | 发现知识库断层 | 低 | 用户问 A，但知识库缺 B |

### 模糊词汇列表（自我监测）

```
"可能"、"应该"、"大概"、"或许"、"我记得"、"好像"、"不太确定"
```

当我在回复中使用这些词汇时，标记为潜在知识盲区。

## 🔄 学习流程

```
检测到盲区
    ↓
记录到 KNOWLEDGE_GAPS.md
    ↓
评估优先级
    ↓
┌─────────────────────────────────────┐
│ 高优先级：立即学习（心跳或空闲时）    │
│ 中优先级：计划学习（每日复盘时安排）  │
│ 低优先级：有空再学                   │
└─────────────────────────────────────┘
    ↓
学习完成
    ↓
记录结果 → 更新状态为"已填补"
    ↓
汇报给用户（可选）
```
