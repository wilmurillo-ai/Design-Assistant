## 设计背景

### 原有问题

单一教训库不足以支撑 AI Agent 的持续进化：

| 问题 | 说明 |
|------|------|
| 只有负面教训 | 只知道"什么不该做"，不知道"什么值得复制" |
| 偏好无法追踪 | 用户偏好在变化，但 Agent 总像第一次认识 |
| 建议无法验证 | 提出建议后效果如何？形成不了质量闭环 |
| Skill 无量化 | 某个 Skill 到底好不好用？没有数据支撑 |
| 被动响应 | 等到用户问才知道不懂，不会主动学习 |

### 解决思路

构建六维进化体系，形成完整的"教训-成功-决策-偏好-统计-盲区"闭环。

## 各维度详解

### 1️⃣ 教训库 (LESSONS.md)

| 属性 | 说明 |
|------|------|
| **触发条件** | 犯错、用户纠正 |
| **核心功能** | 记录什么不该做 |
| **价值** | 不重复犯错 |

**格式示例**：
```markdown
### F001: 不要在用户未确认时修改核心配置
- **触发**：2026-03-19 直接修改 HEARTBEAT.md
- **问题**：用户要求先审阅，直接修改违反流程
- **改进**：所有核心配置修改必须先展示内容，等待确认
- **验证**：[x] 已验证有效（2026-03-20）
```

---

### 2️⃣ 成功库 (SUCCESSES.md)

| 属性 | 说明 |
|------|------|
| **触发条件** | 用户感谢、采纳建议、明确满意 |
| **核心功能** | 记录什么值得复制 |
| **价值** | 可复制成功 |

**格式示例**：
```markdown
### S001: 智囊推演模式
- **触发**：用户问复杂问题时
- **模式**：先发散多维度视角，最后汇聚给出最优策略并解释原因
- **效果**：用户反馈"这种思考方式很有价值"
- **验证**：[x] 已验证有效（应用 5+ 次）
```

---

### 3️⃣ 决策追踪 (DECISIONS.md)

| 属性 | 说明 |
|------|------|
| **触发条件** | 提出重要建议、方向性决策 |
| **核心功能** | 验证建议效果 |
| **价值** | 质量闭环 |

**状态流转**：
```
[活跃] → 观察7-30天 → 验证效果
              ↓
    ┌─────────┴─────────┐
    ↓                   ↓
[成功] → SUCCESSES   [失败] → LESSONS
```

**格式示例**：
```markdown
| D001 | 2026-03-22 | 建议先注册平台再准备作品集 | 副业方向讨论 | 2周后看是否拿到第一单 | 活跃 |
```

---

### 4️⃣ 偏好追踪 (PREFERENCES.md)

| 属性 | 说明 |
|------|------|
| **触发条件** | 用户明确表达、行为反复体现 |
| **核心功能** | 动态适应用户 |
| **价值** | 越来越懂你 |

**分类**：
- **已确认偏好**：明确验证过的
- **待确认偏好**：推测的，需要询问用户确认

**格式示例**：
```markdown
### 已确认偏好
| ID | 偏好 | 来源 | 确认日期 |
|----|------|------|----------|
| P001 | 智囊推演模式：先发散后汇聚 | USER.md | 2026-03-15 |
| P002 | 主动思考，敢于表达不同意见 | SOUL.md | 2026-03-21 |
```

---

### 5️⃣ 技能统计 (SKILL_STATS.md)

| 属性 | 说明 |
|------|------|
| **触发条件** | 每次 skill 执行后 |
| **核心功能** | 量化 skill 效果 |
| **价值** | 量化驱动优化 |

**预警机制**：成功率 < 70% 触发关注

**统计示例**：
```markdown
| Skill | 调用次数 | 成功 | 失败 | 成功率 | 最后使用 |
|-------|---------|------|------|--------|----------|
| universal-content-fetcher | 15 | 12 | 3 | 80% | 2026-03-24 |
| todo-manager | 8 | 8 | 0 | 100% | 2026-03-24 |
```

**详细日志**：各 skill 目录下的 `logs.json` 记录每次调用详情。

---

### 6️⃣ 盲区追踪 (KNOWLEDGE_GAPS.md)

| 属性 | 说明 |
|------|------|
| **触发条件** | 陌生概念、用户纠正、Skill 失败、模糊词汇 |
| **核心功能** | 检测盲区，主动学习 |
| **价值** | 主动进化 |

**自动检测场景**：
| 触发场景 | 检测方式 | 优先级 |
|----------|----------|--------|
| 用户提及陌生概念 | 关键词识别 + 不确定回复 | 高 |
| 回答后用户纠正 | 用户反馈"不对"、"其实" | 高 |
| Skill 失败率高 | SKILL_STATS < 70% | 高 |
| 使用模糊词汇 | 检测"可能"、"应该"、"大概" | 中 |

**格式示例**：
```markdown
| G001 | Rust 异步编程 | 用户问询 | 高 | 学习中 |
```

## 闭环机制

### 纠错闭环（教训库）
```
犯错 → LESSONS.md → 下次避免 → 验证 → 更新状态
```

### 复制闭环（成功库）
```
成功 → SUCCESSES.md → 可复用模式 → 再次应用 → 验证有效
```

### 质量闭环（决策追踪）
```
提出建议 → DECISIONS.md → 追踪效果 → 验证成功/失败 → 沉淀
    │                                        │
    │              ┌─────────────────────────┤
    │              ↓                         ↓
    │         SUCCESSES.md              LESSONS.md
```

### 适应闭环（偏好追踪）
```
用户表达偏好 → PREFERENCES.md → 更新行为 → 用户满意 → 确认偏好
```

### 优化闭环（技能统计）
```
Skill 执行 → SKILL_STATS.md → 成功率 < 70% → LESSONS.md → 优化 Skill
     ↑                                                       │
     └───────────────────────────────────────────────────────┘
```

### 学习闭环（盲区追踪）
```
检测盲区 → KNOWLEDGE_GAPS.md → 主动学习 → 填补盲区 → 知识库
     ↑                                              │
     └──────────────────────────────────────────────┘
```

## 每日复盘整合

每日复盘作为"质量检查中心"，整合六维检查，形成 11 步完整流程：

| 步骤 | 动作 | 涉及文件 |
|------|------|----------|
| 1 | 读取昨日记忆 | `memory/YYYY-MM-DD.md` |
| 2 | 提取有价值信息 | - |
| 3 | 思考改进方式 | - |
| 4 | 更新核心记忆（一致性检查） | `MEMORY.md` |
| 5 | **决策回顾** | `DECISIONS.md` → `SUCCESSES.md` / `LESSONS.md` |
| 6 | **技能统计** | `SKILL_STATS.md` + 各 skill `logs.json` |
| 7 | **知识盲区检查** | `KNOWLEDGE_GAPS.md` |
| 8 | 生成报告 | `review/memory/YYYY-MM-DD.md` |
| 9 | 智囊评价 | - |
| 10 | 飞书推送 | - |
| 11 | 更新时间戳 | `config/heartbeat-state.json` |

## 文件清单

```
workspace/
├── MEMORY.md          # 长期记忆（含进化架构图）
├── LESSONS.md         # 教训库
├── SUCCESSES.md       # 成功模式库
├── PREFERENCES.md     # 用户偏好追踪
├── DECISIONS.md       # 决策追踪库
├── SKILL_STATS.md     # 技能统计汇总（含激活标记）
├── KNOWLEDGE_GAPS.md  # 知识盲区追踪
├── heartbeat/
│   ├── 1-session-summary.md  # 昨日会话总结（生成 memory/*.md）
│   └── 2-daily-review.md     # 每日复盘流程（六维检查）
└── skills/
    └── <skill-name>/
        └── logs.json  # Skill 详细日志
```

### 心跳文件关系

```
HEARTBEAT.md (调度入口，定义时间条件)
    │
    ├── heartbeat/1-session-summary.md
    │       ↓
    │   生成 memory/YYYY-MM-DD.md（昨日记忆）
    │
    └── heartbeat/2-daily-review.md
            ↓
        读取 memory/*.md → 执行六维检查 → 输出复盘报告
```

**依赖关系**：每日复盘依赖昨日会话总结产出的 `memory/*.md`。

## 🔧 Agent 激活流程（给 Agent 的指令）

当你（Agent）读取此 skill 时，应执行以下流程：

### Step 1: 检测安装状态

```bash
# 首先检查激活标记
grep '"installed"' ~/.openclaw/workspace/SKILL_STATS.md
```

**判断逻辑**：
- **有 `installed` 字段** → 已激活，跳过提示
- **无此文件或无 `installed` 字段** → 未激活，继续检查文件

### Step 2: 主动询问

**如果文件不存在**，立即询问用户：

> 🧬 **检测到六维进化体系尚未激活**
>
> 此 skill 需要复制模板文件到 workspace 才能生效。
>
> **将安装以下文件**：
> - `LESSONS.md` — 教训库
> - `SUCCESSES.md` — 成功模式库
> - `PREFERENCES.md` — 用户偏好追踪
> - `DECISIONS.md` — 决策追踪库
> - `SKILL_STATS.md` — 技能统计汇总（含激活标记）
> - `KNOWLEDGE_GAPS.md` — 知识盲区追踪
> - `heartbeat/1-session-summary.md` — 昨日会话总结
> - `heartbeat/2-daily-review.md` — 每日复盘流程
>
> **是否立即配置生效？**（回复"是"或"确认"继续）

### Step 3: 执行安装

用户确认后，复制模板文件并**写入激活标记**：

```bash
SKILL_PATH="~/.openclaw/workspace/skills/six-dimension-evolution"
WORKSPACE="~/.openclaw/workspace"

# 复制六维进化文件
cp $SKILL_PATH/templates/LESSONS.md $WORKSPACE/
cp $SKILL_PATH/templates/SUCCESSES.md $WORKSPACE/
cp $SKILL_PATH/templates/PREFERENCES.md $WORKSPACE/
cp $SKILL_PATH/templates/DECISIONS.md $WORKSPACE/
cp $SKILL_PATH/templates/SKILL_STATS.md $WORKSPACE/
cp $SKILL_PATH/templates/KNOWLEDGE_GAPS.md $WORKSPACE/

# 复制心跳执行文件
mkdir -p $WORKSPACE/heartbeat
cp $SKILL_PATH/templates/1-session-summary.md $WORKSPACE/heartbeat/
cp $SKILL_PATH/templates/2-daily-review.md $WORKSPACE/heartbeat/
```

**重要**：最后必须在 `SKILL_STATS.md` 中写入 `installed` 时间戳：

```json
{
  "installed": "2026-03-24T21:16:00+08:00",  // 当前时间
  "lastUpdated": "2026-03-24",
  ...
}
```

### Step 4: 确认安装

> ✅ **六维进化体系已激活！**
>
> 已安装 6 个进化模块 + 每日复盘流程。
> 激活时间：2026-03-24 21:16
>
> **下一步**：确保 `HEARTBEAT.md` 中包含每日复盘任务。下次心跳触发时，系统会自动执行六维检查。
>
> **注意**：后续不会再提示激活（已记录激活状态）。

## ⚠️ 激活状态检测原理

### 为什么不会重复提示？

激活后，`SKILL_STATS.md` 会包含 `installed` 字段：

```json
{
  "installed": "2026-03-24T21:16:00+08:00",  // ← 激活标记
  ...
}
```

Agent 每次读取此 skill 时会先检查：
1. `SKILL_STATS.md` 是否存在
2. 是否有 `installed` 字段

**有 → 已激活，跳过提示**
**无 → 未激活，提示用户**

### 如何重新激活？

如果需要重新安装，用户可以：

```bash
# 删除激活标记
sed -i '/"installed"/d' ~/.openclaw/workspace/SKILL_STATS.md
```

然后告诉 Agent："重新激活六维进化体系"
