# 可信来源定义

## 来源优先级（高 → 低）

| 优先级 | 来源类型 | 说明 |
|--------|----------|------|
| 1 | 官方文档 | 官方 API 文档、官方网站、官方博客 |
| 2 | 权威媒体 | Wikipedia、MDN、RFC、W3C |
| 3 | 技术博客 | 知名技术博客、开发者文章 |
| 4 | GitHub | 开源项目源码、issue、discussion |
| 5 | 社区 | Reddit、Hacker News、Stack Overflow |
| 6 | 社交媒体 | X/Twitter、微博、小红书 |

## 来源标记格式

在 memo.md 中标记：

```markdown
## 子问题 1: [标题]

### 来源 A [P1-官方] ([URL])
- 关键点 1

### 来源 B [P4-GitHub] ([URL])
- 关键点 1
- ⚠️ 与来源 A 冲突：XXX ???
```

### 标记含义
- `[P1-官方]` → 优先级 1，官方文档
- `[P4-GitHub]` → 优先级 4，GitHub
- `[P5-Reddit]` → 优先级 5，Reddit
- `???` → 冲突待验证

## 冲突处理

**规则**：
- P1 vs P1：看发布日期，取新
- P1 vs 其他：信 P1
- P1+P2 vs P3：信多数

**不阻塞退出**：冲突标为"不确定点"，不阻止输出报告
