# Output Contract

Create one run directory per improvement cycle.

- Directory format: `<repo>/.openclaw-self-improve/<YYYYMMDD-HHMMSS>/`

## Required Files

### run-info.md
- `# Run Info`
- timestamp
- mode
- repo
- objective
- scope
- validation gate

### baseline.md
- `# Baseline`
- `## Objective`
- `## Scope`
- `## Repo State`
- `## Reproduction`
- `## Metrics`
- `## Risks`
- `## Status`

### hypotheses.md
- `# Hypotheses`
- `## Hypothesis 1`
- `## Hypothesis 2` (optional)
- `## Hypothesis 3` (optional)
- `## Ranking`

### proposal.md
- `# Proposal`
- `## Selected Hypothesis`
- `## Planned Changes`
- `## Files To Edit`
- `## Validation Gate`
- `## Rollback Plan`
- `## Approval Status`

### validation.md
- `# Validation`
- `## Commands Run`
- `## Results`
- `## Baseline vs New`
- `## Pass/Fail`
- `## Status`

### outcome.md
- `# Outcome`
- `## Summary`
- `## Evidence`
- `## Residual Risk`
- `## Next Iteration`
- `## Status`

## Status Rules
- Use one of: `pass`, `fail`, `blocked`, `inconclusive`.
- Use one of these approval values in `proposal.md`: `pending`, `approved`, `approved and implemented`, `rejected`, `blocked`.
- If `blocked`, include blocker and dependency owner.
- If `inconclusive`, include smallest next check.
- If `fail`, include rollback action and verification note.

## Quality Rules
- Keep metrics measurable (numbers or clear binary checks).
- Keep claims tied to validation evidence.
- Keep planned changes minimal and scoped.
- Validate the finished run with `scripts/validate-improvement-run.sh`.

## Machine-Readable Companion Files
Generate these when a completed run will feed automation, CI, or another tool.

### run-info.json
- Generated by `scripts/export-improvement-run-json.py`
- Contains:
  - `timestamp_utc`
  - `mode`
  - `repo`
  - `objective`
  - `scope`
  - `validation_gate`
  - `git_commit`
  - `git_branch`
  - `generated_at_utc`

### summary.json
- Generated by `scripts/export-improvement-run-json.py`
- Contains:
  - `run_dir`
  - `timestamp_utc`
  - `mode`
  - `objective`
  - `scope`
  - `approval_status`
  - `baseline_status`
  - `validation_status`
  - `outcome_status`
  - `selected_hypothesis`
  - `next_iteration`
  - `generated_at_utc`

When these JSON files are required by downstream automation or CI, validate the run with `scripts/validate-improvement-run.sh --run-dir <run-dir> --require-json`.
