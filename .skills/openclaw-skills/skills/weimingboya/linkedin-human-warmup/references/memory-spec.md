# Memory Spec

> 记忆文件位于 `memory/linkedin/` 目录下。用自然语言 markdown 格式书写，不要求固定字段结构。

## 首次运行

若 `memory/linkedin/` 目录不存在或为空：
- 创建目录和 `memory/linkedin/MEMORY.md`
- 写入初始内容：新号养号阶段，默认保守策略
- 执行结束后照常写入当天日志

## daily 日志：`memory/linkedin/YYYY-MM-DD.md`

每次执行结束必须写。若当天文件已存在，**追加到末尾，不要覆盖**。用自然语言记录即可，建议包含：

- 今天选了什么剧本、做了什么
- 看到了什么内容（随意，增加"人味"）
- 是否有 connect（如果有，从哪里触发的、是否看了 profile）
- 是否遇到风险或异常
- 给未来自己的提醒/约束

**示例（同一文件内两次追加）：**

```markdown
## 2026-02-06 14:25

选了 ExploreTrail。在 feed 刷了几条帖子，打开了一个评论区，点进一个评论者的 profile 看了看，做 marketing 的，没有 connect。整体正常，继续保守。

## 2026-02-06 21:10

选了 PureFeed。随便刷了两屏就退了，没做任何互动。
```

## 长期记忆：`memory/linkedin/MEMORY.md`

自主判断何时更新。用自然语言记录，建议包含：

- 账号当前状态（养号第几天、整体健康度）
- 累计关键数据（大致的总 connect 数、活跃天数）
- 风控事件记录（何时触发、什么类型、是否仍在冷却期）
- 当前生效的约束（如"X 天内禁 connect"、"暂停 StrongSocial"）
- 内容偏好/风格（账号一直在看什么类型的内容，保持一致性更像真人）
