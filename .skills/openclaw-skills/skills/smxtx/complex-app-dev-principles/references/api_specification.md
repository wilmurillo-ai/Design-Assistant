# API接口设计规范

## 1. 概述

本文档定义了I漫剧APP模块间API接口的设计标准，确保各模块间的数据能够顺畅共享和理解识别。

## 2. 通用规范

### 2.1 数据格式
- 所有接口统一使用 **JSON** 格式
- 响应编码为 **UTF-8**
- 时间格式使用 **ISO 8601** 标准

### 2.2 标准响应格式

```json
{
    "code": 0,
    "message": "success",
    "data": {},
    "request_id": "uuid",
    "timestamp": "2026-03-24T03:52:44Z"
}
```

| 字段 | 类型 | 说明 |
|-----|------|------|
| code | int | 状态码，0表示成功 |
| message | string | 状态信息 |
| data | object | 业务数据 |
| request_id | string | 请求追踪ID |
| timestamp | string | 响应时间戳 |

### 2.3 错误码定义

| 错误码 | 说明 | 处理建议 |
|-------|------|---------|
| 0 | 成功 | - |
| 1001 | 参数错误 | 检查输入参数 |
| 1002 | 认证失败 | 重新登录 |
| 2001 | 资源不存在 | 检查资源ID |
| 3001 | 权限不足 | 申请权限 |
| 5000 | 系统错误 | 联系技术支持 |

## 3. 模块间数据共享标准

### 3.1 数据包标准格式

```json
{
    "id": "数据包唯一标识",
    "type": "数据类型",
    "source_module": "来源模块",
    "target_module": "目标模块（可选）",
    "version": "数据版本",
    "timestamp": "生成时间",
    "signature": "数据签名（可选）",
    "payload": {
        "key": "value"
    }
}
```

### 3.2 必填字段

| 字段 | 说明 | 验证规则 |
|-----|------|---------|
| id | 全局唯一标识 | UUID v4 |
| type | 数据类型 | 符合命名规范 |
| version | 数据格式版本 | 语义化版本 |
| timestamp | 时间戳 | ISO 8601 |

### 3.3 命名规范

| 类型 | 规范 | 示例 |
|-----|------|------|
| 模块名 | 小写+下划线 | user_auth |
| 数据类型 | 小写+下划线 | user_profile |
| 接口名 | 动词+名词 | get_user_info |
| 字段名 | 小写+下划线 | user_id |

## 4. 接口设计模板

### 4.1 请求接口

```
POST /api/v1/{module}/{resource}
```

### 4.2 响应接口

```
GET /api/v1/{module}/{resource}/{id}
```

## 5. 版本管理

- 接口版本号使用URL路径管理
- 主版本号变更需评估兼容性
- 建议保持向后兼容

## 6. 安全性要求

- 敏感数据必须加密传输
- 接口调用需携带认证Token
- 记录完整的调用日志
