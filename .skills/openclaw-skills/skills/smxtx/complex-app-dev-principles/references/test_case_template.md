# 测试用例模板

## 测试用例编写规范

### 基本信息

| 字段 | 说明 |
|-----|------|
| 用例编号 | 模块-功能-序号 |
| 用例标题 | 简洁描述测试目的 |
| 测试类型 | 单元测试/集成测试/系统测试 |
| 优先级 | P0/P1/P2 |
| 前置条件 | 执行前需要满足的条件 |
| 测试步骤 | 详细操作步骤 |
| 预期结果 | 期望的输出或行为 |
| 实际结果 | （测试后填写） |
| 测试状态 | 未执行/通过/失败/阻塞 |
| 测试人员 | |
| 测试日期 | |

## 示例格式

```markdown
### TC-001: 用户登录成功

| 字段 | 内容 |
|-----|------|
| 用例编号 | AUTH-001 |
| 用例标题 | 用户使用正确账号密码登录成功 |
| 测试类型 | 功能测试 |
| 优先级 | P0 |
| 前置条件 | 用户已注册，账号为test@test.com |
| 测试步骤 | 1. 打开登录页面<br>2. 输入账号test@test.com<br>3. 输入密码Test123<br>4. 点击登录按钮 |
| 预期结果 | 登录成功，跳转至首页，显示用户信息 |
| 实际结果 | |
| 测试状态 | 未执行 |
| 测试人员 | |
| 测试日期 | |
```
