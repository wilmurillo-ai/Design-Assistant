# 问题修复说明

## 问题描述

用户反馈：使用 `ai-news-xiaohongshu` 技能生成 AI 资讯时，返回的结果都是"模拟搜索结果"的数据，而不是真实搜索得到的数据。

## 根本原因

**脚本设计缺陷**：`create-xiaohongshu-content.js` 脚本中的 `mockSearchResults()` 函数返回的是**硬编码的演示数据**，并非真实搜索。

**技术限制**：Node.js 独立脚本无法直接调用 OpenClaw 的 `web_search` 工具。`web_search` 是 OpenClaw 的内部工具，只能在 OpenClaw 主流程中使用。

## 解决方案

### 方案架构

```
┌─────────────────────────────────────────────────────────┐
│  OpenClaw 主流程                                         │
│                                                          │
│  1. 使用 web_search 工具搜索真实 AI 资讯                    │
│  2. 处理搜索结果（去重、格式化）                           │
│  3. 调用脚本生成内容                                      │
└────────────────────┬────────────────────────────────────┘
                     │
                     │ --news-json '[{...}]'
                     ▼
┌─────────────────────────────────────────────────────────┐
│  create-xiaohongshu-content.js                          │
│                                                          │
│  1. 接收 JSON 格式的新闻数据                               │
│  2. 生成小红书文案                                        │
│  3. 生成 HTML 封面                                        │
│  4. 输出到 output/日期 目录                                │
└─────────────────────────────────────────────────────────┘
```

### 修改内容

#### 1. 重写 `create-xiaohongshu-content.js`

**修改前**：
- 内置 `mockSearchResults()` 函数返回硬编码数据
- 无法接收外部数据

**修改后**：
- 删除 `mockSearchResults()` 函数
- 添加 `--news-json` 参数支持，接收外部传入的新闻数据
- 无传入数据时使用演示数据（向后兼容）

**使用方式**：
```bash
# 使用真实数据
node scripts/create-xiaohongshu-content.js --news-json '[{"title":"...", "url":"..."}]'

# 使用演示数据（测试）
node scripts/create-xiaohongshu-content.js
```

#### 2. 创建 `run-full-flow.js`

用于快速测试的演示脚本，使用内置演示数据运行完整流程。

```bash
node scripts/run-full-flow.js
```

#### 3. 创建 `openclaw-integration.md`

详细说明 OpenClaw 主流程如何正确调用此技能，包括：
- 为什么脚本使用演示数据
- 如何用真实数据调用
- 代码示例
- 数据格式说明

#### 4. 更新 `SKILL.md`

- 添加"快速开始"章节
- 更新工作流程说明
- 添加"重要说明：真实数据 vs 演示数据"章节

## 使用指南

### 方式一：OpenClaw 主流程调用（推荐，真实数据）

```javascript
// 在 OpenClaw 主流程中

// 1. 搜索真实资讯
const results = await web_search({
  query: "AI 大模型 24 小时 新闻 2026 年 3 月",
  count: 10
});

// 2. 格式化数据
const newsData = results.results.map(r => ({
  title: r.title,
  content: r.snippet || r.description,
  url: r.url,
  time: "X 小时前",
  source: new URL(r.url).hostname.replace('www.', '')
}));

// 3. 调用脚本
const { execSync } = require('child_process');
execSync(`node scripts/create-xiaohongshu-content.js --news-json '${JSON.stringify(newsData)}'`);
```

### 方式二：演示模式（测试用）

```bash
cd ~/.openclaw/workspace/skills/ai-news-xiaohongshu
node scripts/run-full-flow.js
```

## 总结

**问题根源**：脚本设计时预留了接口但未实现真实搜索功能，导致一直使用演示数据。

**解决方案**：明确分工 —— OpenClaw 主流程负责搜索，脚本负责生成。通过 `--news-json` 参数传递真实数据。

**向后兼容**：无传入数据时使用演示数据，确保现有用法不受影响。
