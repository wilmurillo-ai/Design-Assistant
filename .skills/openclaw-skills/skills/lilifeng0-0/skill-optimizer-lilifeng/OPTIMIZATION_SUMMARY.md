# skill-optimizer 技能优化总结

## 🎯 优化目标

让 skill-optimizer 技能更容易被触发，从被动等待转变为主动识别和响应。

---

## ✅ 优化内容

### 1. 增强触发描述（Description）

**优化前**：
```
Analyzes, audits, and optimizes other SKILL.md files. It checks for design pattern adherence, instruction clarity, and context efficiency without altering the original skill's core functionality. Use when you want to improve an existing agent skill.
```

**优化后**：
```yaml
description: |
  技能优化专家。当用户想要优化、改进、重构、审核任何 SKILL.md 文件时自动触发。
  典型触发场景：
  - "优化这个技能"、"改进 skill"、"重构技能定义"
  - "审核这个 agent"、"检查技能质量"、"技能诊断"
  - "让技能更好用"、"提升技能效果"、"技能调优"
  - "应用设计模式"、"技能架构升级"、"标准化技能"
  核心功能：分析、审核、优化 SKILL.md 文件，应用 5 大设计模式，提升技能质量和可靠性。
```

**改进点**：
- ✅ 使用中文描述，更符合本地用户习惯
- ✅ 列出 12 个典型触发场景，覆盖常见表达方式
- ✅ 明确核心功能，便于意图识别

### 3. 新增自动触发规则章节

**新增内容**：
```markdown
## 🎯 自动触发规则

**当检测到以下任一情况时，立即主动调用此技能（无需用户确认）：**

1. **用户明确请求优化**：包含"优化"、"改进"、"重构"、"审核"、"检查"、"诊断"、"调优"等关键词
2. **用户提及技能文件**：提到"skill"、"技能"、"agent"、"SKILL.md"等术语
3. **文件变更检测**：当 SKILL.md 文件被创建或修改后的前 3 分钟内
4. **质量评分低于阈值**：如果系统有技能质量评估，评分<80 分时

**触发优先级**：高（覆盖一般性任务，除非用户明确说"不需要优化"）

**执行模式**：
- 用户明确请求 → 立即执行完整流程
- 文件变更触发 → 先询问"检测到技能文件变更，是否需要优化？"
- 质量触发 → 建议式触发"发现技能可优化点，是否查看？"
```

**改进点**：
- ✅ 定义 4 种触发条件，覆盖主动请求和被动检测
- ✅ 设置高优先级，确保及时响应
- ✅ 区分 3 种执行模式，平衡主动性和用户体验

**触发检测**：首先判断触发类型：

## 🎯 核心优化原则

1. **主动不冒进**：主动识别触发条件，但非明确请求时会二次确认
2. **全面不遗漏**：13 个触发词 + 4 种触发条件，覆盖所有可能场景
3. **高效不打扰**：高优先级响应，但提供关闭选项（"不需要优化"）
4. **智能不机械**：区分触发类型，采用不同执行模式
