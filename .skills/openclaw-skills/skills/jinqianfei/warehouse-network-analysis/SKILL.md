## 概述

本Skill用于分析企业客户门店分布数据，生成仓配网络规划建议。帮助企业决策：
- **设仓方案** - 在哪里建仓库最合适
- **配送方案** - 怎么分配配送路线
- **调拨方案** - 如何调配库存
- **报货周期** - 各区域报货频率和时效
- **干线发运** - 仓库之间的调拨计划

## 输入

- Excel文件（.xlsx格式）
- 包含字段：城市、省市区、地址等信息
- 或包含：机构编码、机构名称、省、市、区、详细地址

## 第二部分：设仓决策逻辑（核心方法论）

### 2.1 核心原则

**原则一：门店密度驱动的"区域重心法"**
- 将库存部署在离需求（门店集群）最近的地方
- 最大化区域覆盖，最小化出库干线运输距离
- 在每个门店密集的区域内寻找"重心"设立RDC（区域中心仓库）

**原则二：成本最优的"轴辐式"多级网络结构**
- RDC（设仓）：全国设少数几个大仓，集中存储，形成库存规模效应
- FSL（配送基地）：在更靠近终端市场的区域设立前置仓或转运中心，负责区域内的"共配"
- 货物从RDC批量调拨至FSL，再由FSL执行最后一公里配送

**原则三：核心与外围分治**
- 对于门店密度较低的"长尾"区域，采用外包的代发模式
- 通过委托第三方物流（3PL）进行"代发"，将固定成本转化为可变成本

### 2.2 计算规则流程

```
输入分析 
    ↓
FSL指派（预定义服务地图 → 指派FSL）
    ↓
RDC选择（在企业现有基地中，时效最短+成本最优）
    ↓
检查RDC→FSL是否有直达干线
    ↓
    ├─ 有 → 直接调拨
    ↓
    ├─ 无 → 寻找TC中转仓
           ↓
           选择链路最短/成本最低的TC
    ↓
定义例外策略（代发模式标记）
```

#### 2.3 TC中转仓核心原则

- **必须有干线**：FDC→TC、TC→FSL之间必须有企业内部干线调拨线路
- **TC数量最少**：在满足条件的情况下，中转仓数量尽量少
- **链路最短**：整个配送链路综合成本最优、距离最短

### Step 4: 选择RDC（设仓优化）

**核心逻辑**：基于企业现有仓配网络基地，在客户约束条件下，以"时效最短+成本最优"为目标，选择RDC。

#### 4.1 约束参数（需向用户询问）

| 参数 | 说明 | 示例 |
|------|------|------|
| **设仓数量上限** | 最多可设仓数量（RDC数） | 3个、5个、6个等 |
| **最大中转次数** | TC中转最大跳数 | 1次、2次等 |
| **覆盖率目标** | RDC覆盖门店比例（越高越好） | 80%、90%、100%等 |
| 时效要求 | 门店到仓库的最大允许时效 | 次日达、1-2天、2-3天 |
| **强制仓库** | 某些区域必须使用的仓库 | 必须用成都仓 |
| **禁止仓库** | 某些区域禁止使用的仓库 | 不用某仓 |

**注意**：以上约束均基于企业当前仓配网络和干线网络进行优化选择。

**开始分析前需确认**：
```
请提供以下约束参数：
1. 设仓数量上限是多少？（如6个）
2. 最大中转次数？（如2次）
3. 时效要求是什么？（如1-2天）
4. 是否有必须使用的仓库？（如：成都仓）
5. 是否有禁止使用的仓库？（如：不用某仓）
```

#### 4.2 覆盖率定义

**RDC覆盖门店**：门店的配送链路能连接到RDC，包括：

| 情况 | 配送链路 | 是否RDC覆盖 |
|------|----------|-------------|
| **RDC = FSL** | RDC → 门店（FSL有库存直配） | ✅ 是 |
| **RDC → FSL（有直达）** | RDC → FSL → 门店 | ✅ 是 |
| **RDC → TC → FSL（中转）** | RDC → TC → FSL → 门店 | ✅ 是 |
| **无法连接** | 无有效干线链路 | ❌ 否（FSL直配）

#### 4.3 优化目标

```
优化目标：min(总成本)
约束条件：
  - 时效满足要求（基于当前干线网络时效）
  - 设仓数量 ≤ 上限
  - 覆盖率 ≥ 目标（如80%）
  - 强制仓库必须使用
  - 禁止仓库不使用
  - 中转次数 ≤ 最大中转次数
```

**约束条件说明**：
- 所有约束均基于企业当前仓配网络（17个配送基地）
- 所有约束均基于企业当前干线网络（76条干线线路）
- 选择RDC时需要检查与FSL之间的干线连通性
- 选择TC时需要检查RDC→TC和TC→FSL的干线连接

**成本构成**：
- 设仓成本：仓储出入库操作成本（装卸、拣货、复核等）
- 干线调拨成本：RDC→FSL 或 RDC→TC→FSL 运输成本
- 配送成本：FSL→门店 最后一公里配送成本

```
min(总成本) = 设仓成本 + 干线调拨成本 + 配送成本
```
- 配送成本：FSL→门店配送成本

#### 4.4 选择算法

```
1. 获取所有配送基地（FSL）
   - 从企业现有配送网络中获取
   
2. 每个FSL都是RDC（都需要设仓）
   - 因为FSL需要库存才能直接配送
   
3. 计算配送链路
   - 检查RDC→FSL是否有直达
   - 如无直达，寻找TC中转
   
4. 输出：
   - RDC列表（=FSL列表）
   - 覆盖率 = 100%（每个门店都被覆盖）
   - 链路类型分布（同仓/直达/中转）
```

#### 4.5 输出示例

```
输入：
  - 配送基地列表：21个

输出：
  RDC数量: 21个（每个FSL都设仓）
  
  配送链路分布：
  - 同仓直配（RDC=FSL）: 1789家 (86%)
  - 直达调拨: 0家 (0%)  
  - TC中转调拨: 147家 (7%)
  - 无链路（需新增）: 141家 (7%)
  
  覆盖率: 100%（所有门店都能被配送）
```

### Step 5: 构建网络层级（RDC→FSL，含TC中转）

**核心逻辑**：将步骤4选定的RDC与步骤3指派的FSL关联，形成完整配送链路。

#### 5.1 链路匹配规则

| 类型 | 链路 | 说明 | 是否RDC覆盖 |
|------|------|------|-------------|
| **直接调拨** | RDC → FSL → 门店 | RDC到FSL有直达干线 | ✅ 是 |
| **TC中转** | RDC → TC → FSL → 门店 | 无直达，需要中转仓 | ✅ 是 |
| **代发模式** | 无有效链路 | 建议3PL代发 | ❌ 否 |

#### 5.2 TC中转仓选择逻辑

**选择条件**：
- FDC→TC 之间必须有干线调拨线路
- TC→FSL 之间必须有干线调拨线路
- TC仓数量尽量少
- 链路尽量短（综合成本最优）

**选择步骤**：
```
1. 检查RDC→FSL是否有直达干线
   → 有：直接调拨
   → 无：进入步骤2

2. 寻找可用的TC中转仓
   遍历所有仓库，找出同时满足：
   - RDC→TC有干线
   - TC→FSL有干线
   
3. 选择最优TC
   - 优先选择链路最短的组合
   - 次选成本最低的组合
```

**示例**：
```
场景：成都仓 → 泉州仓
分析：无直达干线
方案A：成都 → 长沙 → 泉州 (链路短) ✓
方案B：成都 → 广州 → 泉州
选择：长沙仓
```

#### 5.3 链路成本权衡

| 考量因素 | 优先级 |
|----------|--------|
| 干线跳转次数最少 | 高 |
| 综合运输距离最短 | 高 |
| 时效满足 | 中 |
| 成本最优 | 中 |

#### 5.4 覆盖率计算

```
覆盖率 = (RDC覆盖门店数) / (总门店数)
       = (直接调拨 + TC中转) / 总门店数
       
输出：
- 直接调拨门店数
- TC中转门店数
- 代发模式门店数
- 总覆盖率
```

### Step 6: 报货周期规划

根据配送距离和门店数量，确定报货频次：

| 配送距离 | 门店数量 | 建议报货计划 |
|----------|----------|--------------|
| ≤100km | 任意 | 一日一配 |
| 100-300km | ≥10家 | 一周三次 |
| 100-300km | <10家 | 一周二次 |
| >300km | 任意 | 一周一次/二次 |

### Step 6: 干线发运规划

对于需要跨仓库调拨的情况，规划干线发运：

| 起始仓库 | 目的仓库 | 建议发车日 | 预计时效 |
|----------|----------|------------|----------|
| 杭州仓 | 上海仓 | 周二/周六 | 次日达 |
| 南京仓 | 廊坊仓 | 周四 | 三日达 |
| 长沙仓 | 南昌仓 | 周一/周五 | 当日达 |
| 廊坊仓 | 石家庄仓 | 周五 | 次日达 |
