# v1.0.0

## Status

Initial public release.

This note captures the first public-safe version of the `repo-growth-operator` skill.

## Goals

- provide a lightweight growth-planning skill for public GitHub repositories
- keep recommendations tied to releases, docs, and visible repo assets
- prefer small follow-up actions over big campaigns
- make growth work reusable and public-safe

## Included

- `SKILL.md` with the main workflow
- `README.md` for GitHub-facing context
- `CHANGELOG.md` for version tracking
- `agents/openai.yaml` for UI metadata

## Success criteria

- the skill stays practical and grounded
- recommendations are small, clear, and actionable
- outputs help a repo show visible maintenance and follow-up signals

## Notes

- This release is intended for public GitHub and ClawHub distribution.
- Keep future updates focused on practical growth operations, not generic marketing fluff.
