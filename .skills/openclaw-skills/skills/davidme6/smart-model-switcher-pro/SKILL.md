## ⚠️ 核心规则：分层切换策略

### 🔒 规则 1：多模态优先（最高优先级）

**所有会话都生效（主窗口 + 子代理）！**

| 输入类型 | 自动切换模型 | 说明 |
|----------|-------------|------|
| **图片** | qwen3.5-plus | 视觉理解首选 |
| **视频** | qwen3.5-plus | 视频理解 |
| **音频** | qwen3.5-plus | 音频理解 |
| **OCR 文字** | qwen3-vl-plus | 纯图片 OCR |

**为什么子代理也要切换？**
- ✅ 保证子代理能识别图片/视频/音频
- ✅ 防止子代理收到多模态输入时失败
- ✅ 保持专业能力（代码/推理不切换）

### 🔑 规则 2：主窗口智能切换（类 Trae 模式）

**主窗口执行完整智能切换：**

| 任务类型 | 自动切换模型 |
|----------|-------------|
| **代码任务** | glm-5 / qwen3-coder-plus |
| **复杂推理** | qwen3-max / qwq-plus |
| **创意写作** | qwen3.5-plus |
| **日常对话** | qwen3.5-plus |
| **长文档** | qwen3.5-plus (256K) |
| **Office 文档** | MiniMax-M2.5 |

### 🔐 规则 3：子代理专业能力保持

**子代理在**非多模态**情况下保持预设模型：**

| 子代理角色 | 预设模型 | 说明 |
|-----------|---------|------|
| **技术专家** | glm-5 | 代码能力最强 |
| **推理专家** | qwq-plus | 复杂推理 |
| **市场猎手** | qwen3-max | 商业分析 |
| **产品经理** | qwen3-max | 需求分析 |
| **设计师** | qwen3.5-plus | 视觉设计 |

## 🔄 自动切换逻辑（类 Trae 模式）

`javascript
// 第一优先级：多模态检测（所有会话）
IF 消息包含图片/视频/音频 → 切换到 qwen3.5-plus

// 第二优先级：主窗口智能切换
IF 主窗口 {
    IF 代码任务 → glm-5
    ELSE IF 推理任务 → qwen3-max
    ELSE IF 创意写作 → qwen3.5-plus
    ELSE IF Office 文档 → MiniMax-M2.5
    ELSE → qwen3.5-plus (默认最优)
}

// 第三优先级：子代理保持专业模型
IF 子代理 {
    使用预设模型（不切换，除非多模态）
}
`
