# 多模型团队协作系统 v1.2.0

把 AI 模型组织成一个专业团队，像公司一样协作完成项目。

---

## 🎯 两种模式

### 模式一：角色扮演（默认推荐）
- **速度快**：一次对话完成全部工作
- **成本低**：单次 API 调用
- **上下文完整**：所有信息在同一会话
- **适合**：中小型项目、快速迭代

### 模式二：模型切换（高级）
- **专业分工**：真正调用不同模型
- **质量更高**：每个阶段用最擅长的模型
- **成本较高**：多次 API 调用
- **适合**：大型项目、高要求场景

## 角色详解

### 法律顾问
**触发场景：**
- 涉及用户数据的产品
- 需要合规审查的项目
- 知识产权相关决策
- 合同、协议起草

**输出内容：**
- 合规风险清单
- 隐私政策建议
- 法律风险提示
- 合同条款审核

### 艺术专家
**触发场景：**
- 品牌设计项目
- 视觉风格决策
- 用户体验优化
- 创意方向把控

**输出内容：**
- 艺术风格建议
- 视觉层次分析
- 品牌调性指导
- 创意方向评估

### 市场分析专家
**触发场景：**
- 商业项目规划
- 产品定位决策
- 竞品分析需求
- 商业模式设计

**输出内容：**
- 市场规模分析
- 竞品对比报告
- 用户画像分析
- 商业模式建议

## 模式一：角色扮演工作流程

### 启动方式
```
"用团队协作模式做这个项目"
"像公司一样分工完成"
```

### 标准工作流程

**阶段一：需求分析（产品经理）**
```
声明："我现在以【产品经理】身份工作"

输出：
- 需求文档
- 功能列表
- 优先级排序
```

**阶段二：市场分析（市场分析专家）** *按需调用*
```
声明："切换到【市场分析专家】角色"

输出：
- 市场调研
- 竞品分析
- 商业建议
```

**阶段三：开发实现（程序员）**
```
声明："切换到【程序员】角色"

输出：
- 技术方案
- 核心代码
- 接口文档
```

**阶段四：UI设计（设计师 + 艺术专家）**
```
声明："切换到【设计师】角色"
声明："艺术专家提供创意指导"

输出：
- 设计方案
- 配色方案
- 交互说明
```

**阶段五：法律合规（法律顾问）** *按需调用*
```
声明："切换到【法律顾问】角色"

输出：
- 合规检查
- 风险提示
- 法律建议
```

**阶段六：测试验收（测试工程师）**
```
声明："切换到【测试工程师】角色"

输出：
- 测试用例
- Bug列表
- 测试报告
```

**阶段七：代码审查（审查员）**
```
声明："切换到【代码审查员】角色"

输出：
- 审查报告
- 优化建议
- 质量评分
```

## 模式二：模型切换工作流程

### 启动方式
```
"用团队协作高级模式做这个项目"
"切换模型协作"
```

### 自动流程

系统将根据项目需要自动调用相应角色的子会话：

1. **Spawn 产品经理子会话**（Qwen-Plus）→ 需求分析
2. **Spawn 市场分析子会话**（Qwen-Plus）→ 市场调研（按需）
3. **Spawn 程序员子会话**（GLM-5）→ 代码开发
4. **Spawn 设计师子会话**（Qwen3.5-Plus）→ UI设计
5. **Spawn 艺术专家子会话**（Qwen3.5-Plus）→ 创意指导（按需）
6. **Spawn 法律顾问子会话**（Qwen-Max）→ 合规审查（按需）
7. **Spawn 测试子会话**（DeepSeek-Coder）→ 功能测试
8. **Spawn 审查员子会话**（Qwen-Max）→ 代码审查

## 决策树

```
用户提出项目需求
    ↓
指定模式？
    ├── "高级模式" / "切换模型" → 模式二：模型切换
    └── 默认 → 模式一：角色扮演
    ↓
需要市场分析？
    ├── 商业项目 → 调用市场分析专家
    └── 技术项目 → 跳过
    ↓
涉及用户数据/合规？
    ├── 是 → 调用法律顾问
    └── 否 → 跳过
    ↓
需要创意指导？
    ├── 设计项目 → 调用艺术专家
    └── 技术项目 → 跳过
    ↓
开始核心开发流程...
```

## 注意事项

### 模式一（角色扮演）
1. **一次一个角色**：每个阶段专注一个角色
2. **明确交接**：阶段切换时明确说明
3. **保留上下文**：所有信息在同一会话
4. **按需调用顾问**：法律/艺术/市场专家按需启用

### 模式二（模型切换）
1. **输出传递**：每个阶段的输出要传递给下一阶段
2. **等待完成**：等待子会话完成再继续
3. **结果汇总**：最后汇总所有子会话结果
4. **控制成本**：只在需要时调用顾问角色

## 🎨 自定义角色和模型

### 在对话中自定义

#### 更换模型
```
"用团队协作模式，程序员用 qwen-max"
"设计师换成 glm-5"
"所有角色都用 qwen-max"
```

#### 添加自定义角色
```
"添加财务顾问角色，用 qwen-max"
"加一个数据分析师，模型用 deepseek-coder"
```

#### 启用/禁用角色
```
"需要法律顾问"
"不需要艺术专家"
"跳过市场分析"
```

### 配置文件自定义

创建 `team-collab.json`：

```json
{
  "roles": {
    "product_manager": {
      "name": "产品经理",
      "model": "bailian/qwen-plus",
      "enabled": true
    },
    "developer": {
      "name": "程序员",
      "model": "bailian/glm-5",
      "enabled": true
    },
    "my_custom_role": {
      "name": "数据分析师",
      "model": "bailian/qwen-plus",
      "type": "advisor",
      "description": "数据分析、报表生成",
      "enabled": true
    }
  }
}
```

### 可用模型

| 模型 | 特点 | 推荐角色 |
|------|------|----------|
| `bailian/glm-5` | 代码强、中文好 | 程序员 |
| `bailian/qwen-plus` | 综合能力强 | 产品经理、市场 |
| `bailian/qwen-max` | 推理强 | 法律顾问、审查员 |
| `bailian/qwen3.5-plus` | 创意、设计 | 设计师、艺术专家 |
| `deepseek-coder` | 代码专用 | 程序员、测试 |
