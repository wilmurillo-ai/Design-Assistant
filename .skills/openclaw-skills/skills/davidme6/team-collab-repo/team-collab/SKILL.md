# 多模型团队协作系统 v1.1.0

把 AI 模型组织成一个专业团队，像公司一样协作完成项目。

---

## 🎯 两种模式

### 模式一：角色扮演（默认推荐）
- **速度快**：一次对话完成全部工作
- **成本低**：单次 API 调用
- **上下文完整**：所有信息在同一会话
- **适合**：中小型项目、快速迭代

### 模式二：模型切换（高级）
- **专业分工**：真正调用不同模型
- **质量更高**：每个阶段用最擅长的模型
- **成本较高**：多次 API 调用
- **适合**：大型项目、高要求场景

## 模式一：角色扮演工作流程

### 启动方式
```
"用团队协作模式做这个项目"
"像公司一样分工完成"
```

### 工作流程

**阶段一：需求分析（产品经理）**
```
声明："我现在以【产品经理】身份工作"

输出：
- 需求文档
- 功能列表
- 优先级排序
```

**阶段二：开发实现（程序员）**
```
声明："切换到【程序员】角色"

输出：
- 技术方案
- 核心代码
- 接口文档
```

**阶段三：UI设计（设计师）**
```
声明："切换到【设计师】角色"

输出：
- 设计方案
- 配色方案
- 交互说明
```

**阶段四：测试验收（测试工程师）**
```
声明："切换到【测试工程师】角色"

输出：
- 测试用例
- Bug列表
- 测试报告
```

**阶段五：代码审查（审查员）**
```
声明："切换到【代码审查员】角色"

输出：
- 审查报告
- 优化建议
- 质量评分
```

## 模式二：模型切换工作流程

### 启动方式
```
"用团队协作高级模式做这个项目"
"切换模型协作"
```

### 自动流程

系统将自动：
1. **Spawn 产品经理子会话**（Qwen-Plus）→ 完成需求分析
2. **Spawn 程序员子会话**（GLM-5）→ 完成代码开发
3. **Spawn 设计师子会话**（Qwen3.5-Plus）→ 完成UI设计
4. **Spawn 测试子会话**（DeepSeek-Coder）→ 完成测试
5. **Spawn 审查员子会话**（Qwen-Max）→ 完成审查

### 使用方法

执行脚本：
```bash
python scripts/team-workflow.py --mode advanced --task "你的项目需求"
```

或在对话中触发：
```
"用高级模式，让不同模型各司其职"
```

## 注意事项

### 模式一（角色扮演）
1. **一次一个角色**：每个阶段专注一个角色
2. **明确交接**：阶段切换时明确说明
3. **保留上下文**：所有信息在同一会话

### 模式二（模型切换）
1. **输出传递**：每个阶段的输出要传递给下一阶段
2. **等待完成**：等待子会话完成再继续
3. **结果汇总**：最后汇总所有子会话结果
