## 角色工作检查清单

### 产品经理检查清单
- [ ] 需求是否完整？
- [ ] 优先级是否清晰？
- [ ] 验收标准是否明确？
- [ ] 是否有遗漏的边界情况？

### 程序员检查清单
- [ ] 代码能否运行？
- [ ] 错误处理是否完整？
- [ ] 是否有冗余代码？
- [ ] 注释是否清晰？

### 设计师检查清单
- [ ] 配色是否协调？
- [ ] 交互是否流畅？
- [ ] 是否符合用户习惯？
- [ ] 响应式是否考虑？

### 测试工程师检查清单
- [ ] 功能是否全部覆盖？
- [ ] 边界条件是否测试？
- [ ] 错误输入是否处理？
- [ ] 性能是否达标？

### 代码审查员检查清单
- [ ] 代码规范是否遵循？
- [ ] 是否有安全风险？
- [ ] 性能是否有优化空间？
- [ ] 是否有更好的实现方式？

## 角色交接话术

### 产品经理 → 程序员
```
需求分析完成。
核心功能：[列表]
优先级：P0-[xxx], P1-[xxx]
验收标准：[标准]
现在交给程序员开始开发。
```

### 程序员 → 测试工程师
```
开发完成。
实现功能：[列表]
代码位置：[路径]
注意事项：[说明]
现在交给测试工程师进行测试。
```

### 测试工程师 → 程序员（如有Bug）
```
测试完成，发现 [N] 个问题。
Bug列表：
1. [描述] - [复现步骤]
2. [描述] - [复现步骤]
请程序员修复后重新提交。
```

### 测试工程师 → 代码审查员
```
测试通过。
覆盖功能：[列表]
测试用例：[数量]
遗留问题：[如有]
现在交给代码审查员进行审查。
```

### 代码审查员 → 交付
```
审查完成。
质量评分：[分数]
优化建议：[如有]
结论：通过/需修改
[如果通过] 项目可以交付。
```
