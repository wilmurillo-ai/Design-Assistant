# 🧠 BeastXA Memory Pro — 产品方案

## 一句话定位
OpenClaw 记忆增强 Skill：让你的 AI 助手永远不断片，越用越聪明。

---

## 产品是什么

一个 OpenClaw Skill，安装后自动帮你的 AI 助手：
1. **结构化记忆** — 把杂乱的笔记自动整理成主题文件
2. **自动归档** — 每天自动整理、去重、归档，不用你管
3. **抗断片** — 对话压缩前自动保存关键信息，压缩后无缝恢复
4. **零依赖** — 纯本地运行，不需要任何外部 API 或服务

## 安装流程（小白版）

### 方法一：一键安装（推荐）

```bash
# 第1步：安装 Skill
clawhub install beastxa-memory-pro

# 第2步：运行安装脚本
cd ~/.openclaw/skills/beastxa-memory-pro
bash scripts/install.sh
```

安装脚本会自动：
- ✅ 检测你的 OpenClaw 版本和配置
- ✅ 创建 memory/ 目录结构
- ✅ 生成 session-notes.md 模板
- ✅ 设置每日/每周自动整理 cron
- ✅ 验证一切正常

### 方法二：手动安装

如果你想自己控制每一步：

**第1步：创建记忆目录**
```bash
cd your-workspace/
mkdir -p memory/topics
```

**第2步：创建会话笔记模板**
在 `memory/session-notes.md` 里写入：
```markdown
# Session Notes
# Current State
_在这里写当前正在做什么_

# Task Specification
_在这里写核心任务_

# Errors and Corrections
_在这里记录踩过的坑_

# Worklog
_在这里记录工作流水_
```

**第3步：设置自动整理**
安装脚本会自动创建两个 cron 任务：
- 每天 23:30 — 日常整理（提取当天关键信息到主题文件）
- 每周日 23:00 — 深度整理（去重、合并、精简）

**第4步：验证**
```bash
ls memory/topics/
# 应该看到按主题分类的 .md 文件
ls memory/session-notes.md
# 应该看到会话笔记模板
```

## 使用方法

### 安装完就不用管了

Skill 安装后是全自动的：
- 每次对话快压缩时，自动保存关键信息到 `memory/YYYY-MM-DD.md`
- 每天 23:30 自动整理当天笔记到主题文件
- 每周日 23:00 深度整理（去重、合并、精简）

### 手动操作（可选）

**拆分已有的 MEMORY.md：**
```bash
python3 scripts/split_memory.py
# 自动把 MEMORY.md 按主题拆分到 memory/topics/
# 生成 memory/MEMORY-INDEX.md 索引
# 原文件不动，零风险
```

**搜索记忆：**
```bash
openclaw memory search "你想搜的关键词"
```

## 原理说明（给想了解的人）

### 为什么 AI 助手会"断片"？

AI 助手的对话窗口有大小限制（比如 200K tokens）。聊多了，早期的对话会被压缩或丢弃。这就是"断片"——你昨天跟它说的事，今天它不记得了。

### 我们怎么解决？

**三层记忆架构：**

```
┌─────────────────────────────────┐
│  Layer 1: 会话笔记              │  ← 当前对话的结构化摘要
│  (session-notes.md)             │     压缩前自动更新
├─────────────────────────────────┤
│  Layer 2: 每日日志              │  ← 每天的原始记录
│  (memory/YYYY-MM-DD.md)        │     自动追加，不覆盖
├─────────────────────────────────┤
│  Layer 3: 主题文件              │  ← 按主题分类的长期记忆
│  (memory/topics/*.md)           │     自动整理，去重合并
└─────────────────────────────────┘
```

全部存在本地文件里，纯 Markdown，不需要任何外部服务。

## 常见问题

**Q: 我的数据会发到外面吗？**
A: 不会。所有数据都存在本地 Markdown 文件里，不需要任何外部 API。

**Q: 安装后需要改代码吗？**
A: 不需要。运行安装脚本就行，全自动配置。

**Q: 支持哪些 OpenClaw 版本？**
A: 2026.3.x 及以上。

**Q: 我已经有 MEMORY.md 了，会被覆盖吗？**
A: 不会。拆分脚本只读取 MEMORY.md 生成主题文件，原文件完全不动。

**Q: 能和其他记忆 Skill 一起用吗？**
A: 可以。这个 Skill 不修改 OpenClaw 核心代码，只是文件层面的增强。
