# Step 1: 基础信息收集

## 目标
用3个快速问题收集迭代所需的最小必要信息。

## 三个问题

### 1.1 涉及角色

**提问方式**：
> "这个流程涉及哪些角色？每个角色一句话职责。"

**示例格式提供给用户**：
```
- 客服专员：执行客户回访任务，记录回访结果
- 业务部门：配置回访模版，分配任务，审核质量
```

**注意事项**：
- 角色一般3-6个
- 每个角色职责控制在20字以内
- 如果用户列出超过10个角色，提醒"是否可以合并相似角色"

### 1.2 现有主流程

**提问方式**：
> "请用3-7步描述现有的核心业务流程。"

**示例格式提供给用户**：
```
1. 业务部门配置回访模板
2. 业务部门创建回访任务并分配给客服专员
3. 客服专员执行回访并记录结果
4. 业务部门审核回访质量
5. 系统生成回访数据报表
```

**注意事项**：
- 控制在3-7步，不要太细
- 每步应该是一个完整的业务动作
- 如果用户描述超过10步，提醒"请概括为核心主流程"

**补充确认**：
如果流程中有关键分支，追问：
- "步骤X是手动还是自动触发？"
- "步骤Y是每次都执行还是按条件执行？"

### 1.3 迭代需求

**提问方式**：
> "目前流程存在什么问题？为什么要迭代？需要增加/修改/删除什么功能？"

**引导用户结构化描述**：
```
问题：（描述现状痛点）
需求：（想要达成的目标）
涉及模块：（如果知道的话）
```

**示例**：
```
问题：业务部门手动创建任务太费时，每次要导入Excel客户名单
需求：希望系统能自动根据规则生成回访任务
涉及模块：任务管理模块
```

**注意事项**：
- 允许用户描述多个问题和需求
- 如果需求描述模糊，追问具体场景
- 记录用户提到的关键词（会用于Step 2推断模块）

## 输出

创建 `project/00-input/baseline.md`，格式如下：

```markdown
# 基础信息

## 涉及角色

| 角色 | 职责 |
|------|------|
| [角色1] | [职责描述] |
| [角色2] | [职责描述] |

## 现有主流程

1. [步骤1]
2. [步骤2]
3. [步骤3]
...

## 迭代需求

### 需求1
- **问题**：[痛点描述]
- **需求**：[目标描述]
- **涉及模块**：[用户提到的模块，如未提到则留空]

### 需求2
...
```

**用 `present_files` 呈现，询问用户**：
> "我整理的基础信息如上，确认准确吗？如有遗漏或错误请告诉我。"

**等用户确认后进入Step 2。**
