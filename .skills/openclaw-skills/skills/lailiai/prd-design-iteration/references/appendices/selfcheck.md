# 迭代设计自检清单（v3.0）

在完成所有步骤后，使用此清单进行最终检查。

## ✅ Step 1: 基础信息收集

- [ ] 收集了涉及角色（3-6个）
- [ ] 收集了现有主流程（3-7步）
- [ ] 收集了迭代需求（问题+需求描述）
- [ ] 用户已确认基础信息准确

## ✅ Step 2: 调整清单

- [ ] AI推断出了功能模块列表
- [ ] 输出了本次调整清单（模块+类型+说明）
- [ ] 调整类型标注准确（新增/调整/删除）
- [ ] 用户已确认调整范围

## ✅ Step 3: 材料收集

### 新增界面（如有）
- [ ] 每个新增界面都有需求描述
- [ ] 每个新增界面都有字段清单
- [ ] 每个新增界面都有前后跳转说明/截图
- [ ] 字段表包含：字段名、类型、说明、必填

### 调整界面（如有）
- [ ] 每个调整界面都有调整说明
- [ ] 每个调整界面都有当前界面截图或详细描述
- [ ] 每个调整界面都标注了变更位置
- [ ] 如果有截图，截图清晰可见

### 删除界面（如有）
- [ ] 每个删除界面都有流程影响分析
- [ ] 每个删除界面都有数据影响分析
- [ ] 每个删除界面都有依赖检查
- [ ] 用户已确认删除风险

### 材料完整性
- [ ] 所有材料已用 present_files 呈现
- [ ] 用户已最终确认所有材料

## ✅ Step 4: 合理性检查

### 信息完整性
- [ ] 所有新增界面材料完整
- [ ] 所有调整界面材料完整
- [ ] 所有删除界面材料完整
- [ ] 无缺失项或缺失项已补充

### 流程闭环性
- [ ] 重建了调整后的主流程
- [ ] 主流程有明确的开始和结束
- [ ] 每个步骤都有执行角色
- [ ] 步骤之间衔接合理（无断裂）
- [ ] 所有新增/调整界面的跳转路径闭环
- [ ] 无"孤岛"界面（无法到达）
- [ ] 无"死胡同"界面（只能进不能出）

### 角色权限覆盖
- [ ] 所有新增界面都明确了操作角色
- [ ] 所有调整界面都明确了权限变化
- [ ] 删除模块未导致角色失去必要功能
- [ ] 角色权限变更已记录

## ✅ Step 4: 最终输出

### 差量PRD
- [ ] 包含迭代概述（背景、目标、范围）
- [ ] 包含现有流程回顾
- [ ] 包含调整后流程
- [ ] 包含新增功能详述（如有）
- [ ] 包含调整功能before/after对比（如有）
- [ ] 包含删除功能影响说明（如有）
- [ ] 包含数据字段变更表
- [ ] 包含角色权限变更表
- [ ] 包含回滚预案（触发条件+回滚步骤+数据恢复）
- [ ] 包含灰度发布建议

### 修改后原型
- [ ] 新增界面：有完整的交互原型
- [ ] 新增界面：包含所有字段和操作
- [ ] 新增界面：有前后跳转模拟
- [ ] 调整界面：有before/after对比视图
- [ ] 调整界面：变更部分有高亮标注
- [ ] 删除界面：有"已删除"通知和影响说明
- [ ] 原型有清晰的导航菜单
- [ ] 原型可以点击跳转模拟流程

### 输出质量
- [ ] PRD和原型已用 present_files 呈现
- [ ] PRD语言清晰、无歧义
- [ ] 原型界面美观、交互流畅
- [ ] PRD和原型内容一致

## ✅ 一致性检查

### PRD vs 原型一致性
- [ ] PRD中描述的新增功能在原型中都能找到
- [ ] PRD中描述的调整在原型的before/after中清晰展示
- [ ] PRD中的字段表和原型中的表单字段一致
- [ ] PRD中的跳转说明和原型中的跳转路径一致

### Step 1-3 vs 输出一致性
- [ ] 输出的主流程和Step 1收集的基础流程匹配
- [ ] 输出的调整清单和Step 2确认的清单匹配
- [ ] 输出的详细设计和Step 3收集的材料匹配
- [ ] 无遗漏的调整项
- [ ] 无多余的调整项

## 🎯 最终检查

- [ ] 用户对PRD满意
- [ ] 用户对原型满意
- [ ] 用户理解回滚预案
- [ ] 用户知道下一步如何推进（灰度发布/工程实施）
