# Step 5 — 泳道图 + 状态机图

## 5.1 泳道图（Mermaid `flowchart TB`）

每个角色一个 subgraph。**允许特殊泳道**（v2 升级）：
- `系统(定时任务)` — 定时触发
- `系统(外部集成)` — 对接外部系统取数/推送
- `系统(自动流转)` — 无人工自动状态流转

特殊泳道在图下方"角色说明"标注触发条件。

规则：
- 每个用户故事的主要动作必须有节点
- 必须有终态节点（归档/完成/结束）

→ `02-flows/swimlane.md`

## 5.2 状态机图（吸收自 create-prd ch10）🆕

为每个**核心业务对象**画状态机（如"工资单"、"质检单"、"申请单"）。Mermaid `stateDiagram-v2`：

```
stateDiagram-v2
    [*] --> 已计算: 系统取数计算
    已计算 --> 已发放: HR触发发放
    已计算 --> 异常: 取数失败/无模板
    异常 --> 已计算: HR处理异常
    已发放 --> [*]
```

规则：
- 必须有起始 `[*]` 和终态 `[*]`
- 每条迁移必须标注触发条件（who/what 触发）
- 异常状态必须有恢复路径

→ `02-flows/state-machine.md`

两图都**STOP** 等用户确认。
