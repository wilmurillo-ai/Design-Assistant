# Step 3 — 7 条机械校验

输出 ✅/❌ 清单。任一 ❌ 打回对应输入项。

| # | 检查项 | 不通过的标志 |
|---|---|---|
| 1 | 角色完整性 | 痛点/期望提到的角色不在 roles.md |
| 2 | 角色必要性 | roles.md 中角色未在痛点或期望出现 |
| 3 | 痛点可解性 | 痛点描述无法靠流程系统解决（如"老板不重视"） |
| 4 | 期望具体性 | 出现"智能/自动/好用/方便"等模糊词且无具体动作 |
| 5 | 角色-期望对应 | 期望中操作没有明确角色主语 |
| 6 | 闭环性 | 期望共同推导的流程没有终态 |
| 7 | **业务流三段完整性** | 缺少以下任一段：① 输入来源（外部取数/用户录入/定时触发/上游流程）② 流转中间节点（处理/审核/计算/分配类动作）③ 输出去向（落库归档/通知下游/推送用户/对外发布） |

每个 ❌ 必须给出**具体修复建议**（指出缺什么 + 给示例）。

全部 ✅ 才能进 Step 4。
