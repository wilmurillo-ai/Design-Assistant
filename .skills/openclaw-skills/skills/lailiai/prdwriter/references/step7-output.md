# Step 7 — 高保真原型 + 12 节 PRD

## 7.1 高保真可交互原型 `07-output/prototype.html`

强制要求：
- 顶部 tab 切换角色视角
- **每个 P0 操作节点入口按钮必须绑定真实交互**（弹窗/抽屉/页面切换/alert），禁止死按钮
- "新建/编辑"按钮打开的表单必须含 `04-modules.md` 全部必填字段及约束提示
- 涉及状态机的列表用不同颜色 badge
- 列表必须处理空状态
- 单 HTML + Tailwind CDN + 原生 JS
- ToB 风格（中性色、表格密度高）

**自检 5 项**（生成后逐条验证）：
- [ ] 每个角色 tab 切换正常
- [ ] 每个"+ 新建/+ 添加"按钮都有弹窗
- [ ] 每个表单都有提交反馈
- [ ] 至少演示一次状态流转
- [ ] 没有 onclick 缺失或空函数

## 7.2 12 节 PRD `07-output/prd.md`

```
1. 产品概述
2. 角色定义
3. 业务现状与痛点
4. 产品目标
5. 用户故事清单（含 AC）
6. 业务流程图（嵌入泳道图 + 状态机图）
7. ER 数据模型 🆕
   - 实体清单 + 关系图（mermaid erDiagram）
8. 功能模块详述
   每个操作节点必须包含:
   - 输入: 数据来源/触发条件/前置依赖
   - 处理逻辑: 关键计算公式、判断分支（伪代码或步骤列表）
   - 输出: 落库字段/通知对象/状态变更/下游触发
   - 异常处理
9. 字段总览（含计算逻辑）
   - 衍生字段必须给计算公式: 实发金额 = 应发 - 扣减
   - 状态字段必须给状态转移表
   - 引用字段必须给引用源
10. RBAC 权限矩阵 🆕（功能权限表 + 数据权限规则）
11. 数据埋点规划 🆕（事件表 + 核心指标）
12. 风险与决策记录
    - 用户在设计阶段做出的高风险决策
    - 未覆盖的边界场景（明确写本期不做哪些）
    - 来自 03.5-field-decisions.md 的关键决策
    - 每条注明: 描述/影响/用户已确认/后续 review 时点
```

**`[TODO:]` 标注规则**（吸收自 create-prd）🆕：
某节信息确实不足时，用 `[TODO: 具体需要补什么]` 占位。例如：
- `[TODO: 请补充考勤系统的 API 文档地址，确认取数字段]`
- `[TODO: 请确认贷款证明 PDF 是否需要财务章/电子签]`

绝不编造内容填空。

PRD **不含接口/API 设计**，技术方案归 `tech-design` skill。

End: "0→1 设计完成（含 PRD / 原型 / RBAC / 埋点）。下一步可调用 tech-design skill。"
