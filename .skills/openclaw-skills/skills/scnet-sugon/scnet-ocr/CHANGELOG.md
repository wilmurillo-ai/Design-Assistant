## [1.0.1] - 2025-03-26

### 优化
重试机制：

    在 recognize_with_retry 函数中，遇到 429 状态码时，采用指数退避策略等待（默认初始1秒，每次重试翻倍，如1,2,4秒），最多重试3次。

    若 API 响应中包含 retry_after 字段，则优先使用该值作为等待时间。

    重试过程中输出友好提示到 stderr，不影响正常 JSON 输出。

避免并发：

    该机制确保了单个脚本实例在遇到限流时不会立刻退出，而是内部等待后重试。这减少了 AI 代理在外部发起新请求的冲动（因为脚本尚未结束，AI 会等待结果）。

    如果 AI 仍可能并发启动多个脚本实例，你还可以在技能描述（SKILL.md）中明确建议用户不要同时发起多个 OCR 请求。

兼容性：

    所有修改均向后兼容，不改变原有功能，仅增加重试逻辑。
