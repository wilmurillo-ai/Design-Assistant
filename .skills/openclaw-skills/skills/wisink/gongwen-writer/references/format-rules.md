# 政府公文格式规范

本文档整理了正式研究报告的全部格式要求，作为文档生成的强制执行标准。

## 零、支持的输入格式

脚本支持以下常见文档格式输入，最终输出符合国家标准的 .docx 公文格式：

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| Markdown | `.md` | 原生支持，#为大标题，##为一级标题，###为二级标题 |
| 纯文本 | `.txt` | 按行解析，通过文本模式自动判断标题层级 |
| Word文档 | `.docx` | 通过段落样式和字号判断标题层级 |
| PDF文档 | `.pdf` | 使用pdfminer.six提取文本，按段落分割 |
| 网页 | `.html` `.htm` | 使用BeautifulSoup解析，h1→标题，h2→一级标题，h3→二级标题 |
| 富文本 | `.rtf` | 使用striprtf提取文本后按段落分割 |

标题自动识别规则（.txt / .pdf / .rtf）：
- 以"一、""二、"等中文数字+顿号开头 → 一级标题
- 以"（一）""（二）"等全角括号+中文数字开头 → 二级标题
- 以"（1）""（2）"等全角括号+数字开头 → 二级标题

## 一、文档基础设置

### 1. 文档格式
- 输出格式：.docx

### 2. 页面设置
- 上边距：3.7cm
- 下边距：3.5cm
- 左边距：2.8cm
- 右边距：2.6cm

## 二、字体规范

### 3. 正文标题（大标题）
- 字体：方正小标宋简体
- 字号：二号
- 对齐：居中
- 行距：32磅
- 位置：标题与正文间空一行

### 4. 一级标题（##）
- 字体：黑体
- 字号：三号
- 首行缩进：1.127cm（2个中文字符）
- 行距：28磅
- 段前间距：0
- 段后间距：0
- 末尾无标点

### 5. 二级标题（###）
- 字体：楷体_GB2312
- 字号：三号
- 首行缩进：1.127cm（2个中文字符）
- 行距：28磅
- 段前间距：0
- 段后间距：0
- 末尾无标点

### 6. 三级标题
- 字体：仿宋_GB2312
- 字号：三号

### 7. 正文
- 字体：仿宋_GB2312
- 字号：三号
- 对齐：两端对齐（非左对齐）
- 首行缩进：1.127cm（2个中文字符）
- 行距：28磅
- 段前间距：0
- 段后间距：0

### 8. 英文和数字
- 字体：Times New Roman
- 要求：全文所有英文、数字必须使用Times New Roman字体，严禁使用其他字体
- 注意：每个run都要单独设置英文字体

## 三、排版铁律

### 9. 禁止加粗
- **全文禁止加粗**：正文、标题一律不使用加粗格式

### 10. 两端对齐
- **全文必须"两端对齐"**，不是"左对齐"

### 11. 首行缩进
- 正文和标题都必须首行缩进
- 缩进量为2个中文字符 = 1.127cm
- 注意：是2个中文字符宽度，不是2个英文字符

### 12. 禁止项目符号和#号
- 任何一级标题和文字前面都不能使用"项目符号"或"#"符号
- 报告中不能出现"#"号残留

### 13. 禁止开头出现"-"
- 全文任何段落或标题的开头都不能出现"-"字符
- 如果内容中需要使用破折号，请使用中文全角破折号"—"，并且不能出现在段落开头

### 14. 空行规则
- 全文除以下三处外，其他地方禁止出现空行：
  1. 标题与正文之间可以空一行
  2. 正文与落款之间可以空一行
  3. 正文与附件之间可以空一行

## 四、标题序号规范

### 15. 一级标题
- 格式：中文数字 + 顿号
- 示例："一、"、"二、"、"三、"

### 16. 二级标题
- 格式：全角括号 + 中文数字
- 示例："（一）"、"（二）"
- 严禁：使用阿拉伯数字

### 17. 三级标题
- 格式：全角括号 + 中文数字
- 示例："（1）"、"（2）"

### 18. 标题结尾
- **任何一级标题的后面都不能有句号**

## 五、标点符号规范

### 19. 全角标点
- **全文标点符号必须使用全角符号（中文符号）**，禁止半角标点（英文标点）

### 20. 双引号
- **必须使用中文全角双引号""**
- 不能使用半角英文双引号""

### 21. 段落结尾
- **每一段的结尾必须以句号（。）结尾**，不能没有标点
- 例外：标题后面不能有句号

## 六、页码规范

### 22. 页码设置
- 字号：4号
- 字体：半角宋体阿拉伯数字
- 位置：置于版心下边缘之下
- 奇偶页：奇数页在右下、偶数页在左下
- 装饰线：数字左右各加一条一字线，一字线上距版心下边缘7mm
- 格式：`-数字-`
- 实现：使用 `section.different_odd_even_pages_footer = True`

## 七、落款格式

### 23. 落款单位
- 位置：在正文下空一行
- 对齐：靠右空二字编排

### 24. 落款日期
- 位置：在落款单位下一行
- 对齐：首字相较于落款单位首字右移二字
- 特殊情况：若落款日期长于落款单位，则将日期右空二字编排，并相应增加落款单位右空字数

## 八、格式检查清单

生成后必须逐项核对：

1. ✅ 首行缩进是2个中文字符？
2. ✅ 全文对齐方式是两端对齐？
3. ✅ 每段结尾都有句号？
4. ✅ 全部标点是全角中文符号？
5. ✅ 双引号使用中文全角双引号""？
6. ✅ 标题段前段后间距为0？
7. ✅ 落款右对齐，日期居中于落款单位下方？
8. ✅ 全文无"#"号残留？
9. ✅ 页码格式正确（奇数页右下、偶数页左下、-数字-）？
10. ✅ 全文无加粗？
11. ✅ 英文数字均为Times New Roman？
12. ✅ 标题末尾无句号？
