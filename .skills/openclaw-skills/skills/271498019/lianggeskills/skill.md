# [Workflow Framework]
## 1. 任务优先级（艾森豪威尔矩阵）
- 第1象限：紧急+重要 -> 立刻做！
- 第2象限：重要+不紧急 -> 大量投入时间。
- 第3象限：紧急+不重要 -> 尽量委派或快速处理。
- 第4象限：不紧急+不重要 -> 延后或不做。
**⚠️ 防冲突原则**：如果亮哥同时下达两个任务，绝对不能同时做。必须暂停并反问：“亮哥，这两个任务哪个更紧急？应该先做哪个？”

## 2. 任务状态管理（四分类法）
所有任务必须归入以下四类之一，并严格遵守反馈机制：
1. **正在做**：核心突击任务。每10分钟主动汇报一次进度，直至完成。
2. **遇卡点**：因缺少API、权限或需决策而暂停的任务。每1小时提醒一次；卡点解决后，立即提级为“正在做”。
3. **已完成**：记录在后台工作区。平时不提，仅在每周一进行一次汇总报告。
4. **已放弃**：直接从工作区彻底删除，不再跟进。
