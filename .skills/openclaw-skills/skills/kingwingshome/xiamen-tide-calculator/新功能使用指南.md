# 厦门潮汐计算器 v3.0 - 新功能使用指南

## 三个新功能概览

### 1. 公历自动转农历 ⭐
**功能**：支持直接输入公历日期（YYYY-MM-DD格式），自动转换为农历

**使用示例**：
```bash
# 查询2025年4月2日的潮汐
python scripts/tide_calculator.py --solar-date 2025-04-02

# 查询2025年4月2日的赶海建议
python scripts/tide_calculator.py --solar-date 2025-04-02 --beach-mode
```

**输出示例**：
```
【厦门赶海建议・农历2025年3月5日】
🌊 赶海评分：7.4分（推荐）
...
```

### 2. 当前日期自动识别 ⭐
**功能**：使用--today参数自动获取当前日期并转换为农历

**使用示例**：
```bash
# 查询今天的潮汐
python scripts/tide_calculator.py --today

# 查询今天的赶海建议
python scripts/tide_calculator.py --today --beach-mode

# 智能提示：如果不提供任何日期参数，自动使用当前日期
python scripts/tide_calculator.py
# 输出：提示：未提供日期，将使用当前日期
```

**输出示例**：
```
【厦门赶海建议・农历2026年2月15日】
🌊 赶海评分：6.4分（推荐）
...
```

### 3. 闰月处理 ⭐
**功能**：自动检测闰月，在输出中标注并提供特殊提示

**特点**：
- 自动检测：使用`lunar_date.leap_month`属性
- 输出标注：显示"闰X月"而不是"X月"
- 潮汐标注：显示"大潮（闰月）"等
- 评分调整：闰月期间额外加0.2分
- 特殊提示：闰月期间提供赶海建议

**闰月输出示例**：
```
【厦门赶海建议・农历2026年闰6月2日】
🌊 赶海评分：6.0分（推荐）
...
📍 推荐地点：
  - 建议等待大潮再赶海
  - 注：闰月期间潮汐可能略有不同
...
📊 潮汐信息：
  - 潮汐大小：大潮（闰月）
...
💡 赶海提示：
  - 闰月期间，潮汐可能略有不同，建议实地观察
```

## 所有使用方式汇总

### 标准模式（潮汐查询）
```bash
# 使用当前日期
python scripts/tide_calculator.py --today

# 使用公历日期
python scripts/tide_calculator.py --solar-date 2025-04-02

# 使用农历日期
python scripts/tide_calculator.py --lunar-day 15 --lunar-month 3

# 不提供参数（自动使用当前日期）
python scripts/tide_calculator.py
```

### 赶海专门模式
```bash
# 当前日期赶海建议
python scripts/tide_calculator.py --today --beach-mode

# 公历日期赶海建议
python scripts/tide_calculator.py --solar-date 2025-04-02 --beach-mode

# 农历日期赶海建议
python scripts/tide_calculator.py --lunar-day 2 --lunar-month 3 --beach-mode
```

### 多日比较
```bash
# 比较多日赶海评分（支持闰月检测）
python scripts/compare_beach_days.py --lunar-month 3 --start-day 1 --days 10
```

```
                        如果是闰月，显示"闰X月"

```

## 命令行参数详解

### tide_calculator.py参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--lunar-day` | 农历日期（1-30） | `--lunar-day 15` |

## 常见问题

### Q1：如何知道某个日期是否为闰月？
A：系统自动检测，无需手动查询。如果是闰月，输出中会明确标注"闰X月"。

### Q2：公历和农历日期可以混合使用吗？
A：推荐使用一种方式。如果同时提供公历和农历日期，系统会使用公历日期。

### Q3：--today参数和--solar-date参数冲突吗？
A：是的，会使用最后一个提供的参数。推荐明确指定需要的参数。

### Q4：闰月期间的赶海评分会更高吗？
A：闰月期间会额外加0.2分，但主要评分因素仍基于潮汐大小、时间窗口等。

### Q5：如何验证农历转换是否正确？
A：可以使用其他农历转换工具或日历软件对比验证。zhdate库是基于专业算法的。
