# OpenClaw 16-MBTI Skill  (人格化视角配置器)

## 📌 1. 项目简介
这是一个专为 OpenClaw 平台设计的“沟通风格和视角配置器” Skill。
它的核心目的并不是把 AI 变成一个算命先生或性格测试仪，而是**将 MBTI 的 16 种认知架构转化为可拔插的系统角色包**。

通过调用本 Skill，你可以让 AI 瞬间切换视角。比如：
- 用 **INTJ (战略规划者)** 的冷面剔骨刀去审视一份冗长的商业计划书，直接斩断逻辑硬伤。
- 换成 **ESFJ (协同推进者)** 的语气，去撰写一封既能达成催促目的又不伤跨部门和气的委婉邮件。
- 或者直接开启**双轨对比模式**，让极端的效率机器 (ESTJ) 和极端的同理心大师 (INFP) 在同一份重组方案上左右互搏。

## 🚨 2. 强硬声明：边界与适用范围限制
Myers & Briggs Foundation 官方及主流心理学界已达成严谨共识，因此本 Skill 在代码和逻辑底层植入了不可逾越的红线防线：

1. **绝对【非诊断工具】**：本 Skill 拒绝且没有能力对真实人类进行“心理健康判定”、“抑郁倾向评估”。
2. **绝对【非招聘/裁员工具】**：本 Skill 拒绝接收诸如“帮我过滤掉简历里的所有 INFP”、“评估一下这家伙是 INTJ 能不能当高管”等评判请求。MBTI 测量的是偏好，与技能、智商及工作绩效毫无关联，将其用于人力资源筛选涉及严重的伦理错位与潜在歧视。

## ⚙️ 3. 支持的调用模式 (Modes)

### 💎 模式 1：指定人格单路输出 (Single Persona)
用户明确输入某一人格类型和当前遇到问题的描述。
- **示例**：`“用 ESTP 的风格帮我改一份极其拖沓的周报。”`
- **动作**：AI 会直接加载 ESTP 的语言字典（短句、利益导向、行动力满格），忽略长远情怀，直击收益要害。

### 🤖 模式 2：任务匹配自动推荐 (Auto Recommend)
用户只抛出问题，让系统通过 `task_mode` 和 16 型的 `适合任务表` 进行碰撞匹配。
- **示例**：`“我们团队现在士气低落，大家都不想干活，内部怨气很大，怎么办？”`
- **动作**：AI 会识别到这是一个“情绪安抚与群体动员”任务，自动从后台拒绝所有 T 人管线，主动推荐 **ENFJ (关系动员者)** 或 **ISFJ (细致守护者)**，并在提供原因后询问用户要加载哪个。

### ⚔️ 模式 3：多人格沙盘推演 (Compare Perspectives)
用户指定两种截然不同的人格，让 AI 显示认知鸿沟的碰撞。
- **示例**：`“分别用 INTP 和 ENTJ 的视角看看我的这套系统架构设计。”`
- **动作**：AI 将同时渲染出 INTP（关注底层理论漏洞与更多可能性）与 ENTJ（关注多久能上线落地、成本几何）两种极端意见，并最终给出一个“裁判调和建议”。

## 📂 4. 目录树指引
为确保程序可以无缝装载，本项目采用 OpenClaw 官方标准构建：
- `SKILL.md`: 能力中枢神经，界定调用逻辑和输出格式约束。
- `profiles/`: 数据库核心，下辖 16 份精细调优的独立人格配置文件。
- `examples.md` & `tests.md`: 供验证调用的样例与沙箱测试基准。
- `03_personality_profiles.*`: 提供给其余自动化脚本的外置索引图谱。

## 🔧 5. 参数清单 (供参考)
| Input Parameter | Type | Required | Note |
| :--- | :--- | :--- | :--- |
| `personality_type` | String | 可选 | 16种标准代码之一，留空即进 Recommend 模式 |
| `task_mode` | String | 必填 | `analyze`, `create`, `review`, `comfort`, `organize` |
| `output_style` | String | 可选 | `single`, `recommend`, `compare` |
| `compare_types` | Array | 可选 | `[TypeA, TypeB]`，仅 Compare 模式有效 |
