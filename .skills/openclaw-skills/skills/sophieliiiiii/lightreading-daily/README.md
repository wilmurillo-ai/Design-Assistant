# 科技新闻每日推送技能 v5.9

自动抓取 LightReading 和 TechCrunch 科技新闻网站文章，AI 阅读英文全文后总结成中文要点并自动推送到企业微信。

## 快速开始

### 1. 安装依赖

```bash
pip install cloudscraper
```

### 2. 配置企业微信 Webhook

编辑推送脚本文件，修改 `WEBHOOK_URL`：

```python
WEBHOOK_URL = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY_HERE"
```

### 3. 手动测试推送

```bash
cd skills/lightreading-daily

# 合并推送（推荐）：LightReading + TechCrunch
python push_combined.py

# 单独推送（备用）
python push_wechat.py  # 仅 LightReading
```

### 4. 设置定时任务（可选）

使用 OpenClaw cron 每 3 小时自动推送：

```bash
openclaw cron add --name "科技新闻每日推送" \
  --schedule "every 3h" \
  --message "运行自动化推送流程"
```

**推送时间表：**
```
00:00, 03:00, 06:00, 09:00, 12:00, 15:00, 18:00, 21:00
```

## 推送格式

### 合并推送（默认）
- **重点推荐**（1 篇）：优先 Iain Morris 文章，2-4 个要点
- **其他精选**（3 篇）：从两个网站选择，每篇 2-4 个要点
- **总计**：4 篇文章 ✅

### 单独推送（可选）
- **LightReading**：1 篇重点推荐 + 3 篇其他精选
- **TechCrunch**：3 篇精选文章

## 工作流程

```
每 3 小时自动执行（10 分钟超时）：

【第 1 步】运行 push_combined.py
- 抓取 LightReading + TechCrunch RSS
- 抓取网页全文
- ✅ 验证内容 ≥ 200 字符
- ✅ 不足则自动更换文章
- ✅ 确保所有文章都有充足内容
- 保存英文文件（4 篇）
   ↓
【第 2 步】AI 总结
- 读取 combined_en.txt
- ✅ 总结全部 4 篇文章
- ✅ 1 篇重点推荐 + 3 篇其他精选
- 阅读英文全文，理解内容
- 总结成 2-4 个中文要点
- 不是翻译，是总结
- 更新 combined_cn.txt
   ↓
【第 3 步】推送
- 检查占位符
- 有占位符 → 跳过
- 无占位符 → 推送 ✅
```

## 常见问题

**Q: 推送显示"今日推送内容尚未生成"？**  
A: 先运行 `push_combined.py` 生成英文文件，然后 AI 总结并保存 `combined_cn.txt`。

**Q: 企业微信收不到推送？**  
A: 检查 Webhook URL 是否正确，机器人是否启用。

**Q: 想修改推送时间？**  
A: 调整 cron 任务 schedule 参数。

**Q: 为什么有时不推送？**  
A: 如果 3 小时内没有新文章，或者内容不足 200 字符，脚本会跳过推送。

**Q: 每次推送都包含两个网站吗？**  
A: 是的！v5.1+ 确保每次推送至少包含 LightReading 和 TechCrunch 各 1 篇。

**Q: 中文要点是翻译还是总结？**  
A: v5.2+ 使用 AI 总结（非翻译）。AI 阅读英文全文后理解内容，总结成 2-4 个中文要点。

**Q: 有占位符会推送吗？**  
A: 不会！v5.1+ 检测到 `[AI 翻译]` `[待翻译]` 等占位符时会自动跳过推送。

**Q: 内容不足 200 字符怎么办？**  
A: v5.3+ 会自动更换新文章，不再说明"信息有限"，确保推送的文章都有充足内容。

**Q: 为什么只推送 1 篇文章？**  
A: v5.4+ 明确要求 AI 总结全部 4 篇文章（1 篇重点 + 3 篇其他），不能遗漏。

**Q: 为什么 LightReading 文章内容很少？**  
A: LightReading 网站使用 JavaScript 动态加载内容，普通抓取只能获取有限内容（约 800-1000 字符）。v5.5+ 会自动使用 RSS 描述补充，确保有足够内容总结。

**Q: 如何确保推送内容准确性？**  
A: v5.6+ 实施标准化验证流程：
- 作者信息自动从网页抓取，不编造
- 每个要点都要有原文对应句子
- 推送前显示预览供人工验证
- 严格执行检查清单，不遗漏任何项

**Q: 中英文内容如何保持同步？**  
A: v5.8+ 实施中英文同步机制：
- 保存英文文件后自动删除旧中文文件
- Cron 任务强制 AI 重新翻译最新内容
- 推送前检查中文文件时间戳
- 确保中英文内容实时匹配

**Q: 作者信息如何确保准确？**  
A: v5.9+ 实施多种作者提取方式：
- JSON-LD 结构化数据（TechCrunch）
- Meta 标签 author 属性
- Author class CSS 类名
- By 开头的文本模式（LightReading）
- 自动过滤邮箱和无效信息
- 确保作者信息准确可靠
