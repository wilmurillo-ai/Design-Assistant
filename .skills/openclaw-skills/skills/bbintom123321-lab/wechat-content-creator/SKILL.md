# WeChat Content Creator

Create high-quality, high-eCPM WeChat public account articles that drive engagement and revenue.

## Core Principles

### High-Value Niches Only (高单价领域)
Focus exclusively on these high-CPM niches:
- **Legal rights (法律维权)**: Labor law, consumer rights, contract disputes
- **Finance/Investment (理财投资)**: Savings, investing, financial planning
- **Career development (职场提升)**: Workplace skills, salary negotiation, career growth
- **Car ownership (汽车使用)**: Maintenance, buying guides, usage tips

**Avoid**: Politics, entertainment, social unrest, sensitive topics

### Topic Selection Formula
```
Article Topic = Long-tail Keyword + Current Hot Topic
```

**Example**: 
- Keyword: "周末加班几倍工资"
- Hot topic: Labor rights awareness
- Article: "劳动仲裁员透露：这类加班费纠纷公司必输，但90%的人没去要"

## Writing Standards

### Article Specifications
| Element | Standard |
|---------|----------|
| Word count | 1100-1300 characters |
| Structure | SCQA + Emotional progression |
| Paragraphs | 1-3 lines (mobile-friendly) |
| Subheadings | Scenario-based, no "一/二/三" |
| Ad slots | 30% and 70% positions |
| Ending | Interactive question + Compliance statement |

### 8 Golden Opening Templates (8大黄金开头)

1. **Pain Point Hook (痛点描述)**
   - "你是不是也以为..."
   - Example: "你是不是也以为，年假只有干满一年才能休？"

2. **Story Hook (故事描写)**
   - Specific person + scenario
   - Example: "工作5年才懂：年假根本不是按入职算的"

3. **Cognitive Dissonance (颠覆认知)**
   - Challenge common beliefs
   - Example: "月薪5000存不下钱，真的不是因为你赚得少"

4. **Scene Setting (场景演绎)**
   - Specific situation entry
   - Example: "工资到账第一件事做对了，一年多存2万"

5. **Data Impact (数据冲击)**
   - Shocking statistics
   - Example: "90%的人加班费都算错了"

6. **Personal Experience (个人经历)**
   - "I discovered after X years..."
   - Example: "我算了3遍才发现公司少给了一半"

7. **Suspense Question (悬念提问)**
   - Curiosity-inducing question
   - Example: "劳动仲裁员透露：这类加班费纠纷公司必输，但90%的人没去要"

8. **Contrast (对比反差)**
   - Before/after comparison
   - Example: "月薪5000和月薪30000，储蓄率差距不到1%"

### Title Formula
```
Title = Pain Point/Hook + Specific Number + Benefit/Curiosity
```

**High-performing title patterns:**
- "X%的人[action]错了，[consequence]"
- "[Authority]透露：[insider knowledge]"
- "[Action]做对了，[benefit]"
- "[Common misconception]？[reality]"

**Elements to include:**
- Specific numbers (90%, 5 days, 20,000)
- Pain points (白白少休、没去要)
- Authority (央行数据显示、劳动仲裁员透露)
- Clear benefits (一年多存2万)

## Content Requirements

### Real Data Only
**Prohibited:**
- Fictional cases ("我朋友小张")
- Fabricated policies ("2026年新规定")
- Made-up statistics

**Required sources:**
- Government documents (《职工带薪年休假条例》)
- Industry reports (央行城镇储户调查报告)
- Official statistics (劳动仲裁委数据)
- Authoritative research (《中国居民储蓄行为研究报告》)

### Long-tail Keyword Integration

**Characteristics:**
- ≥7 Chinese characters
- Question-based (怎么/如何/怎么办)
- Natural placement (not forced)

**Keyword categories:**

**Career/Legal:**
- 年假怎么算、试用期有年假吗
- 周末加班几倍工资、加班费怎么计算
- 劳动仲裁流程、加班费证据、离职补偿

**Finance:**
- 365天存钱法、52周存钱法
- 拿铁因子是什么、货币基金怎么买
- 指数基金定投、副业技能学习
- 工资理财方法、月光族存钱

**Car:**
- 新车磨合期注意事项
- 汽车保养周期表、油耗怎么算
- 车险怎么买划算、二手车检测

## Article Structure (SCQA)

**S - Situation (情境)**
- Set the scene with relatable context
- Use data or common experience

**C - Conflict (冲突)**
- Reveal the problem or misconception
- Create tension

**Q - Question (问题)**
- Pose the central question
- Make reader want to know the answer

**A - Answer (答案)**
- Provide 3-5 practical methods/insights
- Each with specific implementation

## Formatting Guidelines

### Mobile-First Layout
- Font: System default sans-serif
- Body text: 16px
- Subheadings: 18px bold
- Line height: 1.8
- Paragraph spacing: 8-15px
- Text color: #333333

### Subheading Style
Use descriptive scenarios, not numbered sections:

❌ Bad: "一、年假计算方法"
✅ Good: "先算清楚你有几天年假"

❌ Bad: "二、存钱技巧"
✅ Good: "工资到账先转走，这叫'先存后花'"

### Compliance Statement
Always end with:
```
你今年[topic相关]? 欢迎在评论区聊聊。

本文为个人经验与观点分享，仅供参考。
```

## Workflow

1. **Topic Selection**: Long-tail keyword + Hot topic
2. **Title Creation**: Apply 8 templates, include numbers + hook
3. **Outline**: SCQA structure with 3-5 main points
4. **Writing**: Real data only, natural keyword placement
5. **Formatting**: Mobile-optimized paragraphs
6. **Review**: Fact-check, compliance check
7. **Publishing**: Upload cover, post to draft
