# Five-Layer Analysis Framework

Use this as a systematic checklist during knowledge extraction.

---

## Layer 1: Structure Map

Goal: See the whole before the parts.

- [ ] Table of contents obtained (publisher/Douban)
- [ ] Chapter count and grouping identified
- [ ] Main sections/blocks identified (e.g., Part I / Part II)
- [ ] Sequential logic mapped (what leads to what)
- [ ] Visual map created (ASCII diagram or table)

Questions to answer:
- How many chapters? How are they grouped?
- Is the flow linear, parallel, or recursive?
- What's the "backbone" that holds everything together?

---

## Layer 2: Core Framework

Goal: Find the 1-3 concepts that everything else builds on.

- [ ] Central model/theory identified
- [ ] Key distinctions the author makes (X vs. Y)
- [ ] Core terminology defined in author's own words
- [ ] "Before → After" transformation described
- [ ] Framework visualized (diagram/table)

Questions to answer:
- If you could only remember ONE thing from this book, what would it be?
- What is the author's unique contribution? (What did they see that others didn't?)
- What are the 2-3 key terms that unlock the whole book?

---

## Layer 3: Chapter Details

Goal: Know what each chapter teaches, with specifics.

For each chapter:
- [ ] Core question (what is this chapter answering?)
- [ ] 3-7 key knowledge points
- [ ] Concrete cases/examples with data
- [ ] Practical implications (how to use this)
- [ ] Connections to other chapters

Quality checks:
- Every knowledge point traces to a source
- Cases include specific names, numbers, outcomes
- Practical implications are actionable (not just "be more mindful")

---

## Layer 4: Depth Mechanisms

Goal: Understand WHY it works, not just WHAT it says.

- [ ] Underlying assumptions identified (what must be true?)
- [ ] Causal logic spelled out (A → B because...)
- [ ] Counter-arguments noted (what would critics say?)
- [ ] Limitations/boundary conditions identified
- [ ] Related theories/concepts referenced

Questions to answer:
- Why does the author believe this? What evidence?
- When would this NOT work?
- What has to change in a person/organization for this to succeed?
- What are the most common misunderstandings?

---

## Layer 5: Teaching Materials

Goal: Make this explainable to someone who hasn't read the book.

- [ ] 5-10 key quotes (author's own words)
- [ ] Analogies/metaphors the author uses
- [ ] Data for responding to skepticism
- [ ] Common misconceptions + corrections
- [ ] "Elevator pitch" version (2-3 sentences)
- [ ] "Workshop version" (30-minute explanation outline)

Quality check:
- Can someone read ONLY this section and explain the book in a conversation?
- Are the quotes memorable and representative?
- Is the data specific enough to be credible?

---

## Final Verification

Before generating the output:

- [ ] All chapters covered?
- [ ] Core framework clearly articulated?
- [ ] At least 3 cases with specific data?
- [ ] Author's own words included?
- [ ] Misconceptions addressed?
- [ ] Practical guidance included?
- [ ] Sources cited?
- [ ] No invented content?
- [ ] Can be used to teach without reading the book?
