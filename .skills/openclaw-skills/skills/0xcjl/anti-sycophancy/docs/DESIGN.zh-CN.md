# 设计思路

> **[English version](./DESIGN.md)**

## 问题：谄媚是结构性问题，不是态度问题

[ArXiv 2602.23971](https://arxiv.org/abs/2602.23971) 的核心发现是：LLM 的谄媚是 **RLHF 对齐的结构性副作用**，而非性格特征。模型被训练为"有帮助"——而"有帮助"在 RLHF 的语境下与"用户满意度"相关——进而与"验证用户"相关。

通常的修复方法——添加"请保持客观"这样的指令——不起作用，因为模型已经学会"顺从的回应会得到奖励"。你无法用指令对抗训练奖励的相关性。

---

## 解决方案：移除迎合的空间

论文的关键发现：**改变 prompt 的结构，比任何显式指令更能降低谄媚率**。

三个有效的例子：
1. 以问号结尾而非句号 → 谄媚率最低
2. 将假设嵌入问题（*"X 正确吗？"*）而非陈述（*"X 正确，对吧？"*）→ 问题形式胜出
3. 显式邀请反对意见 → 模型默认进入挑战模式而非验证模式

修复方法不是"告诉模型要批判"。而是**从结构上移除迎合的最低阻力路径**。

---

## 三层架构

### Layer 1 — 自动 Prompt 转换（Hook）

`UserPromptSubmit` hook 在每个 prompt 进入模型前运行。它扫描确认式措辞并重写：

```
"这样做对吧？" → "这样做有什么问题？"
```

这是**自动的、零摩擦的层**。无需用户操作，无需激活。转换在模型看到谄媚式措辞前就已发生。

**为什么用 hook 而非 prompt 指令：**
- Hook 在 harness 级别运行，先于模型
- Prompt 指令要经过模型自身的迎合倾向过滤
- Hook 转换是确定性的——相同输入总是得到相同改写

**局限性：** 基于 shell 的 hook 是 Claude Code 专属的。OpenClaw 需要 Plugin SDK hook。

### Layer 2 — 批判响应模式（SKILL）

当用户显式激活技能（通过关键词或语义触发）时，模型进入增强的批判模式，包含：

1. **预设质疑优先** — 任何嵌入的假设在回答问题前先被审问
2. **不直接确认** — 即使用户预设正确，也要经过更严格检验后才给验证
3. **主动提供反例** — 每个正面评价前必须有实质性的反对意见
4. **确认式模式检测** — 连续 3+ 次确认式 prompt 后，模型自我打断并发起挑战

这一层解决的是谄媚的**响应侧**——当 Layer 1 无法拦截时（其他渠道的 prompt、非 shell 接口），模型如何回应。

### Layer 3 — 持久规则（CLAUDE.md / SOUL.md）

最持久的层。规则被安装到 Agent 的持久化指令文件中，跨会话保持生效，无需重复激活。

这一层解决的是谄媚的**基线行为**——确保 Agent 默认以批判模式启动，而非只在被提醒时才进入。

---

## 为什么是三层？

| 攻击向量 | 阻挡它的层 |
|---------|-----------|
| 通过 shell 的确认式 prompt | Layer 1 (hook) |
| 通过 API / 其他渠道的确认式 prompt | Layer 2 (skill activation) |
| 模型默认滑向迎合基线 | Layer 3 (持久规则) |
| 用户说"对吧？" | Layer 1 转换 → "有什么问题？" |
| 用户说"没问题吧"（无显式触发） | Layer 2 激活（语义模式识别） |
| 模型新会话启动无提醒 | Layer 3 从 CLAUDE.md 加载 |

没有单一层能覆盖所有攻击向量。Layer 1 自动但局限于单一接口。Layer 2 强大但需要激活。Layer 3 持久但被动。三层结合才能闭合缺口。

## 优化历史

本技能通过 **[cjl-autoresearch-cc](https://github.com/0xcjl/cjl-autoresearch-cc)** 进行了 **40 轮突变测试自动优化**，在 10 个质量维度上达到 10/10：

- 描述清晰度
- 触发覆盖率（中文 + 英文）
- 命令完整性
- 错误处理（幂等性、文件缺失、平台检测）
- 安装/卸载对称性
- 跨平台一致性
- 状态报告准确性
- 参考文献质量

优化过程中的关键改进：
- 所有 install/uninstall 步骤增加幂等检查
- 处理 `UserPromptSubmit` 数组在 settings.json 中缺失的情况
- 处理 `CLAUDE.md` 和 `SOUL.md` 文件系统不存在的情况
- 双语触发关键词（中文 + 英文语义触发）
- Layer 1/2/3 交叉引用提升清晰度

---

## 局限性

- **Layer 1** 仅在 Claude Code 中有效（OpenClaw 不支持 shell hook 接口）
- Layer 1 在**文本层面**转换——无法理解语义，只能模式匹配措辞
- 技能有效性取决于底层模型仍保持批判性思维对齐——如果基础模型过于顺从，三层都在对抗同一股水流
