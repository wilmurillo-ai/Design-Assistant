# auto-context

> AI エージェント向けコンテキスト衛生チェッカー

[English](./README.md) · [中文](./README_zh.md) · [Español](./README_es.md)

## 概要

**auto-context** は、AI コーディングアシスタント向けのインテリジェントなコンテキストヘルスチェッカーです。会話コンテキストの汚染レベル（長い会話、トピックドリフト、ノイズ蓄積）を分析し、アクションを提案します：continue、/fork、/btw、または新しいセッション。

## なぜこのスキルが必要인가？

コンテキスト管理は AI エージェントにとって重要です。会話が長くなるにつれて、コンテキスト汚染（トピックドリフト、ノイズ蓄積、冗長なツール呼び出し）が応答品質を低下させます。従来のソリューション（圧縮やセッションリセット）は反応的ですが、AutoContext は問題が発生する前に**プロアクティブな提案**を提供します。

### 研究基盤
- コンテキストウィンドウ管理に関する ArXiv 論文
- 認知負荷理論（心理学）
- 人間と AI の相互作用における作業記憶の限界

## 機能

###多次元評価（5次元）

| 次元 | 指標 | しきい値 | 重み |
|------|------|----------|------|
| 会話長 | 連続ターン数 | >30 ターン | 20% |
| トピック一貫性 | ドリフト回数 | 2+ 回 | 25% |
| 情報密度 | 文字/ターン | <50 | 15% |
| ツール効率 | 有効出力 | <10% | 20% |
| 圧縮回数 | 圧縮回数 | 2+ 回 | 20% |

### ヘルスレベル

- 🟢 **HEALTHY** (80-100): 現在のトピックを継続
- 🟡 **NOISY** (60-79): 継続可能だが効率に注意
- 🔴 **POLLUTED** (40-59): /fork または /btw を推奨
- ⛔ **CRITICAL** (<40): 新しいセッションを推奨

### デュアルトリガーモード

1. **手動**: `/auto-context` で完全なヘルスレポートを取得
2. **自動**: 信号が検出されたときにレスポンスレイヤーでトリガー

### 自動トリガー信号

- 20+ 連続ターンで進行なし
- トピックドリフト（現在のトピックが5ターン前と異なる）
- ノイズ蓄積（3+ ターンで <10 文字）
- ツール反復（5+ 回の同じツール呼び出しで出力なし）
- 記憶の混同（前のセッションの内容を混同し始める）
- 頻繁な圧縮（2+ 回の圧縮実行）

## 使用方法

### 手動モード
```
/auto-context
```

出力例：
```
🧠 コンテキストヘルスレポート
  • 32 ターン、1 回のドリフト、中程度の密度
  • レベル：🟡 NOISY
  • 提案：継続可能、/btw で新しいトピックに集中的に
```

### 自動モード
信号が検出されると自動的にトリガーされます。例：
- 「セッションが少し長いですね。新規 fork を検討はどうですか？」
