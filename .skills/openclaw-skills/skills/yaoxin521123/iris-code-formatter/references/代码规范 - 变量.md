# 代码规范 - 变量

# 变量

1. 代码中的命名均不能`$`、`#`等特殊符号开始或结束，因为`$`、`#`等是系统方法、变量，禁止混淆。

反例：

```java
$name
_name
```

正例：

```java
name
```

2. 严禁代码中使用拼音与英文混合方式命名，更不允许直接使用中文。

反例：

```java
Dazhe 
getDZname
[打折]
```

正例：

```java
discount
getDiscountName
```

3. 参数名、成员变量、局部变量都统一使用`lowerCamelCase`风格。

反例：

```java
Params
```

正例：

```java
params
```

4. 常量命名全部大写。力求完整语义表达清楚，不要嫌长。

反例：

```java
MaxCount = 10
```

正例：

```java
MAXCOUNT = 10
```

5. 杜绝不规范缩写，避免词不达意。

反例：

```java
com
condi
```

正例：

```java
complete
condition
```

6. 长度为7个以内单词不需要缩写，当变量长度过长时需考虑缩写，且缩写应为常用正规缩写，避免无意义缩写。

反例：

```java
st
nb
```

正例：

```java
start
number
num
```

7. 避免无意义变量。

反例：

```java
s a = "1"
s b = "2"
s c = a * b
```

正例：

```java
s fitstNum = "1"
s secondNum = "2"
s sum = firstNum * secondNum
```

8. 临时`global`、进程`global`命名严格使用`^TMP`、`^||TMP`，且携带`pid`。格式为`领域名 + 类名 + 方法 + pid + 标识`。

- 提前把`TMP`开头的`global`名称配置到临时`TEMP`命名空间，配置不写日志。

反例：

```java
s ^||YX("TMP", pid, "Login")
```

正例：

```java
s ^||TMP("IRIS", "IRIS.MOB.Base", "Login", pid)
```

9. 布尔变量不要用`is`开头，因为其他框架引用的话会解析错误。例如`Java`的`Getter`的方法也叫`isDisp()`。

反例：

```java
isDisp
```

正例：

```java
dispFlag
```

10. 所有表的`ID`，都用表的前缀。

反例：

- `BS_IRIS_Student`：`ID`
- `CF_IRIS_Config`：`configID`

正例：

- `BS_IRIS_Student`：`bis`
- `CF_IRIS_Config`：`cic`

11. 所有引用`global`数据的变量，用表`ID + data`方式命名，多节点用`ID + 节点 + Data`方式。

反例：

```java
s data=^BS.IRIS.StudentD(inci)
```

正例：

```java
s bisData = ^BS.IRIS.StudentD(inci)
```

12. 方法中仅出现一次`dataStr`分割字符串分割索引均为`i`长度均为`len`。

反例：

```java
s prescNolen=$l(prescNoStr, ",")
f index = 1 : 1 : len q:ret'=0  d
.s prescNo = $p(prescNoStr, ",", index)
```

正例：

```java
s len = $l(prescNoStr, ",")
for i = 1 : 1 : len {
	s prescNo = $p(prescNoStr, ",", i)
}
```

13. 创建的私有对象可以加`m`，例如有`Execute`类的初始化对象。

反例：

```java
s execute = ##(IRIS.Execute).%New()
```

正例：

```java
s mExecute = ##(IRIS.Execute).%New()
```

14. 不要将系统保留字或`SQL`保留字做为变量。

反例：

```java
s SQLCODE="0"
```

正例：

```java
s code = "0"
```

15. 对于调用其他方法获取返回值的变量，禁止使用`err`的名称，推荐`ret`。

反例：

```java
s errRet=##class(XXX).Save()
s retErr=##class(XXX).Save()
```

正例：

```java
s ret = ##class(XXX).Save()
```

16. 禁止变量超过`31`个字符。（局部变量名称限制为`31`个字符。可以指定长度超过`31`个字符的名称，但仅使用前`31`个字符。）

示例：

```java
ClassMethod Var()
{
	s abcdefghiklmnopqrstuvwxzy123456789 = "1"
	w abcdefghiklmnopqrstuvwxzy123456789,!
	
	s fitst = 1
	s fitstSum = fitst +abcdefghiklmnopqrstuvwxzy123456789
	w fitstSum,!
        
	s second = 2
	s abcdefghiklmnopqrstuvwxzy1234567891 = "2"
	w abcdefghiklmnopqrstuvwxzy1234567891,!
	
	s secondSum = second + abcdefghiklmnopqrstuvwxzy123456789
	w secondSum,!
	b
}
```

结果：

```java
DHC-APP>d ##class(M82).Var()
1
2
2
4
 
 b }
 ^
<BREAK>zVar+11^M82.1
DHC-APP 2d1>w
 
<Private variables>
abcdefghiklmnopqrstuvwxzy123456=2
fitst=1
fitstSum=2
second=2
secondSum=4
```

17. 百分比变量命名要以`%z`开头，避免与系统变量冲突。

反例：

```java
s %msg = ""
```

正例：

```java
s %zXX("XX") = ""
```

18. `.inc`文件里的通用变量，为了防止与系统变量重复。加上前缀标识。

反例：

```java
#define Error "XX"
```

正例：

```java
#define XXError "XX"
```
