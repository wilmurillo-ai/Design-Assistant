# Forge Examples

## Good Descriptions
"Structured OSINT research on people, companies, and organizations. Use when the user wants a provenance-backed brief, entity resolution across public sources, or a free-first research workflow."
→ Clear what it does, when to use it, what distinguishes it.

"Behavior-driven taste model built from real consumption signals. Use for personalized recommendations grounded in actual behavior."
→ Specific, grounded, distinguishes from generic recommendations.

## Bad Descriptions
"Helps with many tasks related to information retrieval."
→ Too vague. Could match anything.

"A powerful AI-driven research assistant."
→ Branding, not routing.

## Anti-Patterns
- Build spec that produces a planning memo instead of a package
- SKILL.md that reads like a design diary
- Support directories with zero files
- Descriptions that match every prompt
- Storage inside skill package directories
- Missing journal command or journal reference
