# Conversion Rate Optimization Fundamentals

## When to Use
Use this skill when starting a CRO program or when unsure where to focus optimization efforts.

## Core Rules

### 1. Diagnose Before You Optimize
The most common CRO mistake is changing things based on intuition. Before making any change, run a diagnostic: check analytics for drop-off points, review session recordings for confusion, and run a heatmap for attention patterns. Data first, solutions second.

### 2. Focus on the Biggest Drop-Off Points First
Improving a step where 80% of visitors drop off has 10× the impact of improving a step where 20% drop off. Map your entire conversion funnel with step-by-step completion rates. The step with the lowest completion rate is your highest-priority optimization target.

### 3. Test One Change at a Time
A/B testing multiple changes simultaneously (multivariate testing) requires significantly more traffic for statistical significance. For most products, simple A/B tests with one variable are more practical and produce faster, clearer results.

### 4. Consider the Full Funnel, Not Just the Landing Page
Landing page conversion is one step in a chain. A landing page with a 5% conversion rate that leads to 80% onboarding completion beats a 10% conversion rate that leads to 20% activation. Optimize for end-of-funnel outcomes, not intermediate metrics.

### 5. Document Every Test and Its Results
A CRO testing log — hypothesis, variant, result, what was learned — compounds value over time. Teams that document tests stop repeating failed experiments and build on what works. Teams that don't restart from zero every quarter.

## Quick Reference

| CRO Priority | Funnel Stage |
|------------|-------------|
| Landing page clarity | Top of funnel |
| Sign-up form friction | Mid funnel |
| Onboarding activation | Post sign-up |
| Upgrade triggers | Monetization |
| Retention features | Post-purchase |

## Common Mistakes to Avoid
- Optimizing for click-through rate without tracking downstream conversions
- Running tests for less than 2 weeks — day-of-week patterns skew results
- Making changes during major external events (holidays, launches) that confound test results
