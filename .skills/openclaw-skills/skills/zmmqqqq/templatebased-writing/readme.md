# Template Report Generator

## 1. 功能简介

本 Skill 用于完成“基于模板的正式文档生成”任务。  
它能够读取一个 Word 模板文档，自动分析模板在结构、样式、内容组织和多模态呈现方面的特征，并结合用户提供的数据源生成一份新的、符合模板风格的正式文档。

该 Skill 适用于以下场景：
- 月报、季报、年报生成
- 政府/监管/单位正式报告生成
- 基于历史模板的批量写作
- 结合结构化数据与文本资料生成综合文档

---

## 2. 输入内容

本 Skill 支持两类核心输入：

### 2.1 模板输入
一个 Word 模板文档，用于分析以下内容：
- 封面
- 目录
- 章节结构
- 标题层级
- 字体样式
- 正文段落风格
- 图表和表格布局
- 固定表达与写作框架

### 2.2 数据输入
用于生成文档的数据来源可以包括：
- 知识库
- 多篇参考文档
- CSV文件
- 关系表
- SQL查询结果
- 其他结构化/半结构化数据

---

## 3. 核心能力

本 Skill 主要包含四个模块：

### 3.1 模板结构分析模块
自动识别模板的外部结构和内部层级，包括：
- 封面形式
- 目录结构
- 正文章节层级
- 标题编号方式
- 图表位置与编号方式

### 3.2 模板风格分析模块
分析模板的写作与排版风格，包括：
- 标题与正文的字体样式
- 字号、粗细、对齐方式
- 段落长度与组织方式
- 报告语言风格
- 常用表达方式

### 3.3 数据理解与归纳模块
对输入数据进行抽取、清洗和归纳，包括：
- 多文档内容抽取与去重
- CSV/关系表字段理解
- SQL结果汇总
- 核心指标提炼
- 趋势与分类信息整理

### 3.4 文档生成模块
根据模板分析结果和数据归纳结果，生成完整文档内容，包括：
- 封面文字
- 目录草案
- 正文内容
- 图表插入建议
- 表格插入建议

---

## 4. 支持的数据形式

### 4.1 文本型数据
- 知识库材料
- 多篇参考文档
- 历史报告
- 摘要材料

### 4.2 结构化数据
- CSV
- 关系表
- SQL查询结果
- 统计结果表

### 4.3 混合型数据
同时包含文本说明和结构化指标的数据包。

---

## 5. 输出结果

本 Skill 的输出通常包括三部分：

### 5.1 模板分析结果
总结模板的：
- 结构
- 样式
- 内容写法
- 图表使用方式
- 生成约束规则

### 5.2 数据归纳结果
提炼用户输入数据中的：
- 核心事实
- 统计指标
- 可用于图表的字段
- 可用于表格的字段
- 可写成段落的分析结论

### 5.3 最终文档内容
生成一份可直接复制到 Word 中的正式文档文本，尽量与模板保持一致。

---

## 6. 使用的工具

本 Skill 使用 OpenClaw 本地环境中的以下工具能力：

### 6.1 read
用于读取本地模板文件、数据文件和参考文档。

### 6.2 write
用于输出生成后的结果文件或中间结果。

### 6.3 exec
用于必要的数据处理，例如：
- 读取 CSV
- 统计数量
- 整理结构化字段
- 处理 SQL 查询结果的格式化内容

---

## 7. 模型配置

本 Skill 设计为调用云端大模型：

- 模型名称：glm-5

模型负责：
- 模板结构与风格理解
- 多源数据归纳
- 正式文档生成
- 段落写法对齐

---

## 8. 部署方式说明

本 Skill 在 OpenClaw 环境中运行，整体设计满足以下要求：

- 工具本地化运行
- 数据本地存储
- 模型云端调用
- 支持模板驱动写作
- 支持多源数据输入

这与“本地工具 + 云端大模型”的要求一致。

---

## 9. 典型工作流程

### 步骤1：读取模板
读取 Word 模板文档，分析封面、目录、章节、标题、字体和图表结构。

### 步骤2：读取数据
读取知识库、多篇文档、CSV、关系表或 SQL 查询结果。

### 步骤3：数据归纳
提炼核心信息、统计指标、趋势特征和重点内容。

### 步骤4：生成文档
依据模板结构与语言风格生成完整文档。

### 步骤5：输出结果
输出模板分析、数据归纳和最终文档内容。

---

## 10. 适用场景

本 Skill 特别适合以下任务：
- 基于已有月报模板生成新月份报告
- 基于监管模板自动生成行业监测报告
- 基于历史文档模板自动更新报告内容
- 将多文档、CSV、SQL结果整合为正式材料

---

## 11. 注意事项

1. 若输入数据不足，不应虚构具体数值。
2. 若模板中的某些字体细节无法完全自动识别，应优先保持结构和语言风格一致。
3. 对于图表内容，可输出“建议插入图/表”的说明。
4. 多来源数据应先统一口径，再生成正文。
5. 输出应尽量便于直接复制到 Word 中继续排版。

---

## 12. 总结

本 Skill 的核心目标是：

**将“模板分析”与“数据驱动写作”结合起来，实现一个可复用、可扩展、可落地的模板化正式文档生成能力。**
