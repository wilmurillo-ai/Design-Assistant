# 🔐 安全模式使用指南

## 什么是安全模式？

**安全模式**不会在配置文件中保存你的密码，而是使用浏览器的登录状态（类似于浏览器记住密码的功能）。

### 安全特性

- ✅ **不保存密码**：配置文件中不包含任何敏感信息
- ✅ **浏览器会话**：使用浏览器保存的登录状态
- ✅ **首次手动登录**：只需要登录一次
- ✅ **后续自动同步**：之后每次都自动使用已保存的登录状态

---

## 🚀 使用方法

### 首次使用

第一次运行时，脚本会：

1. **打开浏览器窗口**
2. **显示登录提示**
3. **等待你手动登录**（最多 5 分钟）
4. **保存登录状态**
5. **自动完成同步**

```bash
cd <skill-directory>

# 运行安全模式脚本
./sync_safe.sh
```

### 后续使用

之后每次运行，脚本会：

1. **自动使用已保存的登录状态**
2. **完成同步**（无需再次登录）

```bash
# 直接运行即可
./sync_safe.sh
```

## 🔧 高级选项

### 自定义配置

```bash
# 自定义输出目录
FLOMO_OUTPUT_DIR="/path/to/obsidian/vault" ./sync_safe.sh

# 自定义标签前缀
FLOMO_TAG_PREFIX="flomo/" ./sync_safe.sh

# 自定义浏览器数据目录
FLOMO_USER_DATA_DIR="my_browser_data" ./sync_safe.sh
```

### 直接使用 Python 脚本

```bash
python3 scripts/auto_sync_safe.py \
  --output "~/Documents/Obsidian/flomo" \
  --tag-prefix "flomo/" \
  --no-headless \
  --verbose
```

**参数说明：**
- `--output`: Obsidian vault 输出目录（必需）
- `--tag-prefix`: 标签前缀（可选）
- `--download-dir`: 下载目录（默认：`flomo_downloads`）
- `--user-data-dir`: 浏览器数据目录（默认：`flomo_browser_data`）
- `--no-headless`: 显示浏览器窗口
- `--verbose`: 显示详细日志

## 🔒 安全说明

### 登录状态保存在哪里？

登录状态保存在 `flomo_browser_data` 目录中，类似于浏览器的用户数据目录。

**目录结构：**
```
flomo-to-obsidian/
├── flomo_browser_data/        # 浏览器数据（包含登录状态）
│   ├── Default/
│   │   └── Cookies            # 登录 Cookie
│   └── ...
├── .flomo_sync_state.json     # 同步状态（不含密码）
└── scripts/
    └── auto_sync_safe.py      # 安全模式脚本
```

### 如何保护登录状态？

1. **不要分享** `flomo_browser_data` 目录
2. **定期清理**：如果不再使用，删除该目录
3. **权限控制**：该目录只有你自己可以访问

### 如何重新登录？

如果登录状态过期或想重新登录：

```bash
# 删除浏览器数据和状态文件
rm -rf flomo_browser_data .flomo_sync_state.json

# 重新运行脚本
./sync_safe.sh
```

## 🆚 对比：安全模式 vs 密码模式

| 特性 | 安全模式 | 密码模式 |
|------|---------|---------|
| 密码保存 | ❌ 不保存 | ✅ 保存在 .env |
| 首次使用 | 🔐 手动登录 | 🤖 自动登录 |
| 后续同步 | 🤖 自动 | 🤖 自动 |
| 安全性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 便捷性 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 推荐场景 | 个人使用 | 服务器/自动化 |

### 推荐方案

- **个人电脑**：使用安全模式（更安全）
- **服务器/定时任务**：使用密码模式（更便捷）

## ❓ 常见问题

### Q1: 首次登录超时怎么办？

**A**: 脚本会等待 5 分钟。如果超时：
1. 关闭浏览器
2. 重新运行 `./sync_safe.sh`
3. 更快地完成登录

### Q2: 登录状态会过期吗？

**A**: 会的。flomo 的登录会话会在一段时间后过期（通常几周）。过期后：
1. 脚本会自动提示你重新登录
2. 在浏览器中手动登录即可
3. 登录成功后会继续同步

### Q3: 可以在多台电脑上使用吗？

**A**: 可以，但每台电脑需要单独登录一次。登录状态保存在本地，不会同步到其他电脑。

### Q4: 如何查看同步日志？

**A**: 
```bash
# 查看日志文件
cat auto_sync.log

# 实时查看日志
tail -f auto_sync.log
```

### Q5: 如何切换到密码模式？

**A**: 使用原来的 `sync.sh` 脚本：
```bash
./sync.sh
```
