## 详细说明

### 🔐 安全模式

#### 优点
- ✅ **更安全**：不在配置文件中保存密码
- ✅ **简单直观**：首次登录后，后续自动
- ✅ **浏览器保护**：利用浏览器的安全机制
- ✅ **易于清理**：删除目录即可

#### 缺点
- ⚠️ **首次需要手动登录**：需要在浏览器中输入密码
- ⚠️ **登录会过期**：几周后需要重新登录
- ⚠️ **不适合服务器**：无人值守时无法重新登录

#### 工作原理
1. 首次运行时打开浏览器
2. 等待用户手动登录（最多 5 分钟）
3. 浏览器保存登录状态到 `flomo_browser_data` 目录
4. 后续运行自动使用已保存的登录状态

#### 使用方法
```bash
# 首次运行（会打开浏览器让你登录）
./sync_safe.sh

# 后续运行（自动使用已保存的登录状态）
./sync_safe.sh
```

#### 适合你吗？
✅ 如果你：
- 在个人电脑上使用
- 注重安全性
- 能接受首次手动登录
- 每次同步时都在电脑旁

---

### 🤖 密码模式

#### 优点
- ✅ **完全自动**：一次配置，永久有效
- ✅ **无人值守**：适合定时任务和服务器
- ✅ **不会过期**：登录凭证长期有效
- ✅ **设置简单**：只需配置 .env 文件

#### 缺点
- ⚠️ **密码保存**：需要在 .env 文件中保存密码
- ⚠️ **安全风险**：如果电脑被入侵，密码可能泄露
- ⚠️ **需要保护**：必须设置文件权限

#### 工作原理
1. 从 `.env` 文件读取账号密码
2. 使用 Playwright 自动登录 flomo
3. 自动导出、下载、转换

#### 使用方法
```bash
# 1. 创建配置文件
cat > .env << EOF
FLOMO_EMAIL=your_phone_or_email
FLOMO_PASSWORD=your_password
EOF

# 2. 设置文件权限
chmod 600 .env

# 3. 运行同步
./sync.sh
```

#### 适合你吗？
✅ 如果你：
- 需要在服务器上运行
- 需要完全无人值守
- 需要定时自动同步
- 能接受密码保存在本地

## 🎯 推荐选择

### 场景 1：个人电脑日常使用
**推荐：🔐 安全模式**

理由：
- 更安全，不保存密码
- 首次登录后，后续完全自动
- 登录会话可以保持几周
- 即使登录过期，重新登录也很简单

```bash
./sync_safe.sh
```

---

### 场景 2：服务器 / VPS / 云主机
**推荐：🤖 密码模式**

理由：
- 完全无人值守
- 适合 cron 定时任务
- 不需要人工干预
- 登录不会过期

```bash
./sync.sh
```

---

### 场景 3：定时任务（每天自动同步）
**推荐：🤖 密码模式**

理由：
- 安全模式的登录会话可能过期
- 密码模式更可靠
- 不需要人工重新登录

```bash
# crontab 示例
0 22 * * * cd /path/to/flomo-to-obsidian && ./sync.sh >> /tmp/flomo_sync.log 2>&1
```

---

### 场景 4：偶尔手动同步
**推荐：🔐 安全模式**

理由：
- 更安全
- 使用频率不高，登录不太可能过期
- 即使过期，重新登录也很快

```bash
./sync_safe.sh
```

## 🔄 如何切换？

### 从密码模式切换到安全模式
```bash
# 1. 删除密码配置（可选）
rm .env

# 2. 使用安全模式
./sync_safe.sh
```

### 从安全模式切换到密码模式
```bash
# 1. 创建密码配置
cat > .env << EOF
FLOMO_EMAIL=your_phone_or_email
FLOMO_PASSWORD=your_password
EOF

chmod 600 .env

# 2. 使用密码模式
./sync.sh
```

## ❓ 常见问题

### Q1: 可以同时使用两种模式吗？
**A**: 可以！两种模式使用不同的脚本和状态文件，互不干扰。

### Q2: 哪种模式更快？
**A**: 速度相同。安全模式首次需要手动登录（5分钟内），之后和密码模式一样快。

### Q3: 安全模式的登录多久过期？
**A**: 通常几周。flomo 的登录会话类似于浏览器记住密码的时间。

### Q4: 密码模式安全吗？
**A**: 如果：
- ✅ 文件权限设置正确（`chmod 600 .env`）
- ✅ 不分享 `.env` 文件
- ✅ 电脑有密码保护
- ✅ 已加入 `.gitignore`

那么是安全的！但安全模式仍然更安全。

### Q5: 推荐哪种？
**A**: 
- **个人使用**：🔐 安全模式（更安全）
- **服务器/定时任务**：🤖 密码模式（更可靠）
