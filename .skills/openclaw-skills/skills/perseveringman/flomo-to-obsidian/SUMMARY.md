## 🎯 两种同步模式

### 🔐 安全模式（推荐）

**特点**：
- ✅ 不保存密码
- ✅ 使用浏览器登录状态
- ✅ 首次手动登录（5 分钟内）
- ✅ 后续自动同步

**适合**：
- 个人电脑日常使用
- 注重安全的用户
- 偶尔手动同步

**使用**：
```bash
./sync_safe.sh
```

---

### 🤖 密码模式

**特点**：
- ✅ 完全自动化
- ✅ 适合定时任务
- ⚠️ 保存密码到配置文件

**适合**：
- 服务器部署
- 定时任务
- 完全自动化场景

**使用**：
```bash
./sync.sh
```

## 🚀 快速开始

### 方式1：安全模式（推荐）

```bash
cd <skill-directory>

# 运行安全模式脚本
./sync_safe.sh

# 首次会打开浏览器，手动登录即可
# 后续自动使用已保存的登录状态
```

---

### 方式2：密码模式

```bash
cd <skill-directory>

# 创建配置文件
cat > .env << EOF
FLOMO_EMAIL=your_phone_or_email
FLOMO_PASSWORD=your_password
EOF

chmod 600 .env

# 运行同步
./sync.sh
```

## 🔒 安全性

### 🔐 安全模式
- ✅ **不保存密码**：配置文件中不包含任何敏感信息
- ✅ **浏览器保护**：使用浏览器的安全机制保存登录状态
- ✅ **本地存储**：登录状态保存在 `flomo_browser_data` 目录
- ✅ **权限控制**：目录权限仅当前用户可访问

### 🤖 密码模式
- ⚠️ **密码保存**：账号密码存储在 `.env` 文件
- ✅ **权限保护**：文件权限设置为 `600`（仅所有者可读写）
- ✅ **Git 忽略**：已添加到 `.gitignore`
- ✅ **本地处理**：不会上传任何数据到外部服务器
