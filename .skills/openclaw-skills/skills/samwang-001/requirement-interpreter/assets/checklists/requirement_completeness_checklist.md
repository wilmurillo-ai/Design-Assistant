# 需求完整性检查清单

在向AI描述需求前，请使用此检查清单确保需求描述完整、清晰。

## 基础信息检查

### 项目概览
- [ ] **项目名称**: 是否明确了项目或功能的名称？
- [ ] **需求提出人**: 是否说明了谁提出这个需求？
- [ ] **优先级**: 是否明确了需求的紧急程度？（高/中/低）
- [ ] **时间要求**: 是否说明了期望的完成时间？

### 背景与目标
- [ ] **业务背景**: 是否说明了为什么需要这个功能？
- [ ] **核心问题**: 是否明确了要解决的具体问题？
- [ ] **项目目标**: 是否说明了期望达到的效果？
- [ ] **成功标准**: 是否有明确的成功衡量标准？

## 功能需求检查

### 功能描述
- [ ] **主要功能**: 是否列出了所有主要功能点？
- [ ] **功能优先级**: 是否区分了核心功能和附加功能？
- [ ] **功能边界**: 是否明确说明了包含什么、不包含什么？

### 用户与场景
- [ ] **用户角色**: 是否明确了谁使用这个功能？
- [ ] **使用场景**: 是否描述了在什么情况下使用？
- [ ] **用户流程**: 是否说明了用户的操作步骤？

### 输入输出
- [ ] **输入数据**: 是否说明了需要什么输入？
  - [ ] 数据字段是否明确？
  - [ ] 数据类型是否指定？
  - [ ] 数据格式是否说明？
  - [ ] 数据验证规则是否定义？

- [ ] **输出结果**: 是否说明了期望得到什么输出？
  - [ ] 输出格式是否明确？
  - [ ] 成功/失败的不同输出是否定义？
  - [ ] 错误信息格式是否说明？

### 处理逻辑
- [ ] **核心逻辑**: 是否说明了数据处理的主要逻辑？
- [ ] **业务规则**: 是否说明了需要遵循的业务规则？
- [ ] **异常处理**: 是否考虑了异常情况的处理？
  - [ ] 输入数据错误的处理方式？
  - [ ] 系统错误的处理方式？
  - [ ] 网络中断的处理方式？

## 非功能需求检查

### 性能需求
- [ ] **响应时间**: 是否有具体的响应时间要求？
  - [ ] 页面加载时间要求？
  - [ ] API响应时间要求？
  - [ ] 复杂操作的处理时间？

- [ ] **并发能力**: 是否有并发用户数的要求？
  - [ ] 最大并发用户数？
  - [ ] 每秒事务数(TPS)？
  - [ ] 同时在线用户数？

- [ ] **数据处理能力**: 是否有数据处理量的要求？
  - [ ] 单次处理数据量？
  - [ ] 批量处理能力？
  - [ ] 数据导入导出速度？

### 安全需求
- [ ] **认证授权**: 是否有明确的认证授权要求？
  - [ ] 认证方式？（用户名密码/OAuth/SSO）
  - [ ] 权限模型？（RBAC/ABAC）
  - [ ] 会话管理要求？

- [ ] **数据安全**: 是否有数据安全保护要求？
  - [ ] 数据加密要求？
  - [ ] 数据脱敏规则？
  - [ ] 访问控制要求？

- [ ] **应用安全**: 是否有应用安全防护要求？
  - [ ] 输入验证要求？
  - [ ] 防攻击措施？（XSS/CSRF/SQL注入）
  - [ ] 安全审计要求？

### 可用性需求
- [ ] **系统可用性**: 是否有可用性指标要求？
  - [ ] 系统可用性百分比？（如99.9%）
  - [ ] 故障恢复时间要求？
  - [ ] 维护窗口安排？

- [ ] **用户体验**: 是否有用户体验相关要求？
  - [ ] 界面响应要求？
  - [ ] 操作便捷性要求？
  - [ ] 帮助文档要求？

### 兼容性需求
- [ ] **浏览器兼容**: 是否有浏览器兼容性要求？
  - [ ] 支持的浏览器类型？
  - [ ] 最低版本要求？

- [ ] **设备兼容**: 是否有设备兼容性要求？
  - [ ] 支持的设备类型？（桌面/移动/平板）
  - [ ] 屏幕分辨率要求？

- [ ] **操作系统**: 是否有操作系统兼容性要求？
  - [ ] 支持的操作系统？
  - [ ] 最低版本要求？

### 完整性等级

- **良好 (70-89%)**: 需求描述基本完整，可能需要少量澄清

2. 对于每个项目，确保需求描述中包含了相关信息
3. 对于无法确定或不需要的项目，可以标记为"不适用"
4. 根据完成率评估需求描述的完整性
5. 根据改进建议补充缺失的信息
