# AI沟通检查清单

在与AI沟通需求时，使用此检查清单确保沟通效果最佳化。

## 沟通前准备

### 需求梳理
- [ ] **需求归类**: 是否明确了需求属于哪种类型？
  - [ ] 技术开发
  - [ ] 数据分析
  - [ ] 业务流程
  - [ ] 内容创作
  - [ ] 问题解决

- [ ] **复杂度评估**: 是否评估了需求的复杂程度？
  - [ ] 简单（1-2个功能点）
  - [ ] 中等（3-5个功能点）
  - [ ] 复杂（6+个功能点，多模块）

- [ ] **优先级设定**: 是否明确了需求的优先级？
  - [ ] P0: 紧急且重要（立即处理）
  - [ ] P1: 重要但不紧急（本周内）
  - [ ] P2: 一般需求（可排期）
  - [ ] P3: 优化需求（有时间再做）

### 信息收集
- [ ] **背景资料**: 是否收集了相关背景信息？
  - [ ] 业务背景说明
  - [ ] 用户使用场景
  - [ ] 现有系统情况
  - [ ] 相关文档链接

- [ ] **技术信息**: 是否明确了技术相关信息？
  - [ ] 技术栈偏好
  - [ ] 数据库类型
  - [ ] 第三方服务需求
  - [ ] 部署环境要求

- [ ] **约束条件**: 是否明确了各种约束？
  - [ ] 时间限制
  - [ ] 预算限制
  - [ ] 技术限制
  - [ ] 合规要求

## 沟通内容检查

### 清晰度检查
- [ ] **避免模糊词汇**: 是否避免了以下模糊词汇？
  - [ ] 快/慢 → 使用具体时间（如：<500ms）
  - [ ] 好/差 → 使用具体标准（如：符合XX规范）
  - [ ] 简单/复杂 → 使用具体描述（如：3步完成）
  - [ ] 强大/弱 → 使用具体能力（如：支持1000并发）

- [ ] **具体化描述**: 是否提供了具体信息？
  - [ ] 具体数字（数量、时间、大小）
  - [ ] 具体示例（输入输出示例）
  - [ ] 具体标准（行业标准、设计规范）
  - [ ] 具体指标（性能指标、成功标准）

- [ ] **结构化组织**: 是否使用了结构化描述？
  - [ ] 分点列表
  - [ ] 层级标题
  - [ ] 表格对比
  - [ ] 流程图/示意图

### 完整性检查
- [ ] **5W2H覆盖**: 是否涵盖了5W2H要素？
  - [ ] **What**: 需要做什么？
  - [ ] **Why**: 为什么需要？
  - [ ] **Who**: 谁使用？
  - [ ] **When**: 什么时候需要？
  - [ ] **Where**: 在哪里使用？
  - [ ] **How**: 如何实现？
  - [ ] **How much**: 需要多少资源？

- [ ] **功能完整性**: 是否描述了完整功能？
  - [ ] 核心功能列表
  - [ ] 输入输出定义
  - [ ] 处理逻辑说明
  - [ ] 异常处理方案

- [ ] **非功能完整性**: 是否考虑了非功能需求？
  - [ ] 性能要求
  - [ ] 安全要求
  - [ ] 可用性要求
  - [ ] 兼容性要求

### 示例提供检查
- [ ] **输入示例**: 是否提供了输入示例？
  - [ ] 示例数据格式
  - [ ] 示例数据内容
  - [ ] 示例数据边界值

- [ ] **输出示例**: 是否提供了输出示例？
  - [ ] 成功输出示例
  - [ ] 失败输出示例
  - [ ] 不同场景输出示例

- [ ] **使用示例**: 是否提供了使用场景示例？
  - [ ] 典型用户操作流程
  - [ ] 异常情况处理示例
  - [ ] 界面交互示例

## 沟通技巧检查

### 分步骤描述
- [ ] **步骤分解**: 是否将复杂需求分解为步骤？
  - [ ] 识别主要处理步骤（3-7步）
  - [ ] 描述每个步骤的输入、处理、输出
  - [ ] 定义步骤之间的依赖关系
  - [ ] 考虑每个步骤的异常情况

- [ ] **渐进式沟通**: 是否采用渐进式沟通？
  - [ ] 先描述整体目标
  - [ ] 再分解具体功能
  - [ ] 逐步补充细节
  - [ ] 及时确认理解

### 上下文提供
- [ ] **背景说明**: 是否提供了充分背景？
  - [ ] 业务需求背景
  - [ ] 技术环境背景
  - [ ] 用户使用背景
  - [ ] 项目约束背景

- [ ] **关联信息**: 是否提供了关联信息？
  - [ ] 相关系统信息
  - [ ] 相关文档链接
  - [ ] 相关标准参考
  - [ ] 相关案例参考

### 约束明确
- [ ] **技术约束**: 是否明确了技术约束？
  - [ ] 必须使用的技术
  - [ ] 不能使用的技术
  - [ ] 性能约束
  - [ ] 安全约束

- [ ] **资源约束**: 是否明确了资源约束？
  - [ ] 时间约束
  - [ ] 预算约束
  - [ ] 人力约束
  - [ ] 设备约束

- [ ] **合规约束**: 是否明确了合规约束？
  - [ ] 数据保护法规
  - [ ] 行业标准
  - [ ] 公司政策
  - [ ] 国际标准

## 沟通效果优化

### 验证理解
- [ ] **复述确认**: 是否要求AI复述理解？
  - [ ] "请用你自己的话复述一下这个需求"
  - [ ] "根据你的理解，主要功能有哪些？"
  - [ ] "你认为这个需求的核心难点是什么？"

- [ ] **示例验证**: 是否通过示例验证理解？
  - [ ] "如果输入是X，你认为输出应该是什么？"
  - [ ] "这个场景下，系统应该如何处理？"
  - [ ] "异常情况下，应该返回什么错误？"

### 及时反馈
- [ ] **纠正误解**: 是否及时纠正误解？
  - [ ] "这里理解有误，应该是..."
  - [ ] "还需要补充一点..."
  - [ ] "这个细节很重要..."

- [ ] **补充信息**: 是否及时补充信息？
  - [ ] "忘了说明一个约束条件..."
  - [ ] "这里有个重要背景需要补充..."
  - [ ] "还有一个相关要求..."

### 迭代优化
- [ ] **分阶段沟通**: 是否采用分阶段沟通？
  - [ ] 第一阶段：明确整体框架
  - [ ] 第二阶段：细化功能细节
  - [ ] 第三阶段：确认技术实现
  - [ ] 第四阶段：验证完整方案

- [ ] **持续改进**: 是否记录沟通问题？
  - [ ] 记录AI误解的地方
  - [ ] 分析沟通不畅的原因
  - [ ] 优化下一次沟通方式

## 特殊场景处理

### 复杂需求沟通
- [ ] **分层描述**: 是否采用分层描述？
  - [ ] 高层：业务目标、用户价值
  - [ ] 中层：功能模块、系统架构
  - [ ] 底层：技术细节、实现方案

- [ ] **分模块沟通**: 是否分模块沟通？
  - [ ] 先沟通核心模块
  - [ ] 再沟通辅助模块
  - [ ] 最后沟通集成部分

### 模糊需求沟通
- [ ] **探索式沟通**: 是否采用探索式沟通？
  - [ ] "我有个想法，但不太确定..."
  - [ ] "能不能帮我分析一下可行性？"
  - [ ] "有哪些可能的实现方案？"

- [ ] **原型反馈**: 是否要求原型反馈？
  - [ ] "先做个简单的原型看看效果"
  - [ ] "根据原型提供改进建议"
  - [ ] "迭代优化设计方案"

### 技术难题沟通
- [ ] **问题分解**: 是否分解技术难题？
  - [ ] 明确具体的技术问题
  - [ ] 分析问题的根本原因
  - [ ] 提出可能的解决方案

- [ ] **方案对比**: 是否对比不同方案？
  - [ ] 列出各种技术方案
  - [ ] 分析各方案优缺点
  - [ ] 推荐最优方案

## 沟通效果评估

### 理解准确性评估
- [ ] **需求复述**: AI能否准确复述需求？
- [ ] **功能识别**: AI能否识别主要功能点？
- [ ] **难点分析**: AI能否分析实现难点？
- [ ] **方案提出**: AI能否提出合理方案？

### 沟通效率评估
- [ ] **往返次数**: 沟通需要几次往返？
- [ ] **澄清问题**: AI提出了多少澄清问题？
- [ ] **理解速度**: AI理解需求的速度如何？
- [ ] **方案质量**: AI提出的方案质量如何？

### 改进空间识别
- [ ] **沟通障碍**: 存在哪些沟通障碍？
- [ ] **信息缺失**: 哪些信息提供不足？
- [ ] **表述问题**: 哪些表述可以改进？
- [ ] **工具使用**: 是否有效使用沟通工具？

## 最佳实践总结

### 沟通前
1. **明确需求类型和复杂度**
2. **收集相关背景和约束信息**
3. **准备具体示例和参考文档**

### 沟通中
1. **使用具体化、结构化描述**
2. **提供充分上下文和示例**
3. **明确各种约束条件**
4. **分步骤、分层级描述复杂需求**

### 沟通后
1. **验证AI的理解准确性**
2. **及时纠正误解和补充信息**
3. **记录沟通问题和改进点**
4. **优化下一次沟通方式**

### 持续改进
1. **建立个人沟通模板库**
2. **总结不同类型需求的沟通要点**
3. **分享有效的沟通案例**
4. **学习新的沟通技巧和工具**

---

**沟通效果评分卡**:
- **优秀 (9-10分)**: AI完全理解，无需澄清，提出高质量方案
- **良好 (7-8分)**: AI基本理解，需要少量澄清，方案合理
- **一般 (5-6分)**: AI部分理解，需要较多澄清，方案一般
- **不足 (0-4分)**: AI理解困难，需要大量澄清，方案不成熟

**改进建议**:
根据评分结果，针对性地改进沟通方式和内容。
