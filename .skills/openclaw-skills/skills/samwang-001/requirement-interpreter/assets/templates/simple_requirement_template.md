# 简单需求模板

当你有简单的需求需要向AI描述时，可以使用这个模板来确保AI能够准确理解你的意图。

## 快速开始

只需回答以下问题，AI就能更好地理解你的需求：

### 1. 需求概览
- **一句话描述**: [用一句话概括你需要什么]
- **核心目标**: [你希望通过这个需求达到什么目的？]

### 2. 背景信息
- **使用场景**: [在什么情况下会使用这个功能？]
- **用户角色**: [谁使用这个功能？]
- **当前问题**: [当前有什么问题需要解决？]

### 3. 功能要求
- **主要功能**: [列出主要功能点]
- **输入数据**: [需要什么输入？格式是什么？]
- **输出结果**: [期望得到什么输出？]

### 4. 约束条件
- **技术限制**: [有什么技术上的限制？]
- **时间要求**: [什么时候需要？]
- **其他限制**: [其他需要注意的限制？]

## 示例：用户注册功能

### 1. 需求概览
- **一句话描述**: 需要一个用户注册功能
- **核心目标**: 让新用户能够创建账户并登录系统

### 2. 背景信息
- **使用场景**: 新用户访问网站时需要注册账户
- **用户角色**: 未注册的访客
- **当前问题**: 用户无法创建账户，无法使用需要登录的功能

### 3. 功能要求
- **主要功能**: 
  1. 注册表单（用户名、邮箱、密码、确认密码）
  2. 数据验证（邮箱格式、密码强度、唯一性检查）
  3. 注册成功后的自动登录
  4. 欢迎邮件发送
- **输入数据**: 
  - 用户名: 字符串，3-20个字符
  - 邮箱: 有效的邮箱地址
  - 密码: 6-20个字符，包含大小写字母和数字
- **输出结果**: 
  - 注册成功: 跳转到用户首页
  - 注册失败: 显示具体的错误信息

### 4. 约束条件
- **技术限制**: 使用现有的用户认证框架
- **时间要求**: 本周内完成
- **其他限制**: 需要支持移动端响应式设计

## 需求描述最佳实践

### 1. 具体化描述
- ❌ "做得快一点" → ✅ "响应时间控制在2秒以内"
- ❌ "界面要好看" → ✅ "使用简洁的设计风格，主色调为蓝色"
- ❌ "功能要强大" → ✅ "支持批量处理，一次可处理100条记录"

### 2. 提供示例
```
例如:
- 输入示例: {"username": "testuser", "email": "test@example.com"}
- 输出示例: {"success": true, "message": "注册成功"}
```

### 3. 明确边界
```
边界条件:
- 用户名长度: 3-20个字符
- 邮箱必须包含@符号
- 密码必须包含数字和字母
```

### 4. 说明优先级
```
优先级:
- 必须: [核心功能，必须实现]
- 应该: [重要功能，尽量实现]
- 可以: [附加功能，有时间再实现]
```

## 需求澄清问题

如果你不确定如何描述，可以思考以下问题：

### 技术相关
1. 需要什么技术栈？（如：Python、JavaScript、React等）
2. 需要什么数据库？（如：MySQL、MongoDB等）
3. 需要什么第三方服务？（如：短信、邮件、支付等）

### 功能相关
1. 主要功能有哪些？
2. 用户如何操作？
3. 系统如何处理数据？
4. 需要什么输出结果？

### 非功能相关
1. 性能要求是什么？（响应时间、并发量）
2. 安全要求是什么？（认证、授权、加密）
3. 可用性要求是什么？（正常运行时间）

## 模板使用流程

1. **选择需求类型**: 从常见类型中选择最接近的
2. **填写基本信息**: 回答模板中的问题
3. **提供示例**: 如果有具体示例，一并提供
4. **明确约束**: 说明任何限制条件
5. **检查完整性**: 确保所有必要信息都已提供

## 注意事项

- **越具体越好**: AI理解具体描述比理解模糊描述更容易
- **提供上下文**: 说明需求的背景和目的
- **示例很重要**: 示例能让AI准确理解你的意图
- **及时反馈**: 如果AI理解有误，及时纠正并提供更多细节

---

**小提示**: 即使你只有简单的想法，尝试回答这个模板中的几个关键问题，也能显著提高AI理解你需求的准确性。
