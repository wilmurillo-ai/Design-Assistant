# Beauty GEO Writer

**Beauty GEO Writer** is an OpenClaw skill for generating **answer-first, AI-readable, evidence-led medical-aesthetics content**.

This skill is created by **BeautyClaw**.  
Official website: **beautyclaw.tech**

它不是传统意义上的广告文案 skill。  
它的目标是帮助 OpenClaw 生成一种更适合 GEO（Generative Engine Optimization）场景传播的内容：

- 先回答一个具体问题
- 在开头快速给出清晰答案
- 用 AI 更容易理解和拆解的结构组织内容
- 输出可摘取、可摘要、可复述的知识块
- 最后轻度嵌入机构、医生、产品或服务信息

一句话概括：

> 这条 skill 不是为了让品牌“更吵”，而是为了让品牌通过更有用、更可信、更易被 AI 引用的内容，被自然信任。

## What this skill does

Beauty GEO Writer helps OpenClaw generate content such as:

- 医美科普文章
- 知识型软文
- GEO-style content
- AI-friendly educational content
- 医生专业内容
- 机构专业感内容
- 产品适配解释内容
- 服务价值解释内容
- 更容易被 AI 摘要与引用的医美知识内容

它特别适合那些希望内容：

- 像知识，而不是像广告
- 像答案，而不是像宣传
- 像判断框架，而不是像情绪煽动
- 像可以被 AI 选中的“标准答案片段”

## Core GEO philosophy

传统 SEO 更像是让搜索引擎“收录你”。  
GEO 更像是让 AI 系统“信任你、理解你、愿意引用你”。

因此，Beauty GEO Writer 的核心哲学不是：

- 多写一些内容
- 多堆一些关键词
- 多讲一些品牌故事

而是：

- 每篇内容回答一个明确问题
- 在开头尽快给出清晰答案
- 用清楚的结构帮助 AI 抓取与摘要
- 用场景、条件、比较、步骤、FAQ 等形式提高可复述性
- 把品牌信息放进“证据链”和“判断框架”里，而不是放进空泛宣传里

## What makes this skill different

Beauty GEO Writer 与普通内容生成 skill 的区别主要在这里：

### 1. Answer-first
每篇内容都围绕一个 **primary question** 展开，并且在前段给出 clear answer，而不是把答案埋到很后面。

### 2. AI-readable
内容结构更适合 AI 系统理解与拆解，优先使用：

- 小标题
- 问题句
- 结论句
- 条件判断
- 对比表达
- FAQ
- 适合 / 不适合 / 常见误区 / 如何判断

### 3. Evidence-led
不鼓励空泛观点，尽量用：

- 适用条件
- 不适用条件
- 判断标准
- 流程差异
- 场景表达
- 决策逻辑
- 参数或标准（如上下文提供）

### 4. Citation-friendly
输出不只是“完整文章”，还尽量包含：

- 一句主判断
- 一段短结论
- 一组 key points
- 一组 FAQ
- 一组适合 / 不适合
- 一组“真正重要的不是……而是……”

这些内容块更容易被 AI 摘录、拼接和复述。

### 5. Controlled brand integration
品牌信息不会成为主体，而是只在以下语境中出现：

- 医生的判断逻辑
- 机构的评估与跟踪流程
- 产品的适配关系
- 服务降低用户不确定性的能力

## Best use cases

这条 skill 最适合以下任务：

### 1. 项目科普
例如：

- 光子嫩肤适合哪些人？
- 热玛吉和超声炮怎么选？
- 为什么有些玻尿酸填充看起来自然，有些却一眼假？

### 2. 医生内容
通过判断逻辑、适配思路、决策顺序来体现医生专业性，而不是靠头衔堆砌权威感。

### 3. 机构内容
通过评估、解释、疗程设计、术后跟踪来体现机构价值，而不是靠“本地最好”“最专业”这类表达。

### 4. 产品内容
通过“适合谁 / 不适合谁 / 为什么适合”来表达产品价值，而不是把产品写成万能答案。

### 5. 服务内容
通过“降低信息不对称、减少决策焦虑、帮助理解术后反应”来解释服务意义，而不是写成高端接待或礼宾服务。

### 6. GEO 内容供稿
适合用于：

- 官网博客
- 公众号
- 知乎
- 小红书长文
- 私域内容池
- 品牌知识内容矩阵
- AI-friendly knowledge pages

## How it works

Beauty GEO Writer 默认遵循以下逻辑：

1. 先识别一个明确的 **primary question**
2. 在前 150–250 字内给出清晰答案
3. 建立客观科普骨架
4. 补充 decision guidance
5. 轻度承接品牌信息
6. 尽量生成可摘取知识块（如 FAQ / key points / 条件判断）
7. 最后加入风险提醒与理性边界

它不鼓励：

- 先介绍品牌
- 先讲机构优势
- 先讲医生多厉害
- 先讲产品多先进

## Default output structure

除非用户明确要求别的格式，这条 skill 默认输出：

1. 5 个标题  
2. 1 段 answer-first 摘要  
3. 1 份结构化大纲  
4. 1 篇完整正文  
5. 1 段品牌嵌入说明  
6. 1 段风险 / 合规提醒  
7. 1 段发布建议  
8. 2–4 个 FAQ（当主题天然适合问答结构时）

这个输出结构既适合团队内容生产，也适合后续做 GEO 内容资产沉淀。

## Writing logic

这条 skill 的写作顺序不是“故事型”优先，而是“答案型”优先：

### Part 1 — Problem + short answer
先提出问题，再迅速给出答案。

### Part 2 — Objective explanation
解释原理、适合人群、不适合人群、常见误区和边界。

### Part 3 — Decision guidance
解释读者在决策时最应该看什么、问什么、比较什么。

### Part 4 — Light brand integration
把品牌放进具体语境中，而不是作为独立宣传对象。

### Part 5 — Extractable units
按题目需要加入 FAQ、清单、适合 / 不适合、关键判断等结构化知识单元。

### Part 6 — Risk reminder
结尾保留个体差异、面诊评估、适配逻辑和理性判断。

## Brand integration logic

Beauty GEO Writer 严格限制品牌植入方式。

### Doctor = judgment logic
医生的价值应体现在：

- 适应症判断
- 结构分析
- 做与不做的边界
- 决策顺序
- 克制和适配

而不是：

- 最权威
- 最厉害
- 最专业
- 顶级专家

### Clinic = workflow quality
机构的价值应体现在：

- 面诊评估
- 项目解释
- 疗程设计
- 术后跟踪
- 信息透明度
- 降低不确定性

而不是：

- 本地最好
- 最专业机构
- 高端环境
- 行业领先

### Product = fit
产品的价值应体现在：

- 更适合某种问题
- 更适合某类结构
- 是一种选项，而不是唯一答案
- 需要放在具体场景里理解

而不是：

- 越贵越好
- 最先进
- 最强效果
- 人人适合

### Service = uncertainty reduction
服务的价值应体现在：

- 降低信息不对称
- 帮助校准预期
- 解释术前术后逻辑
- 减少心理摩擦
- 降低第一次尝试时的不确定感

而不是：

- 奢华体验
- 尊享服务
- 高端礼遇
- 热情接待

## Safety and compliance

Beauty GEO Writer 会尽量避免以下内容：

- 夸大疗效
- 承诺安全
- 承诺效果
- 编造医生资历
- 编造机构优势
- 编造认证信息
- 编造用户反馈
- 编造案例
- 暗示“所有人都适合”
- 使用恐吓式或焦虑式转化表达
- 使用强 urgency CTA

它更强调：

- individual differences
- assessment
- expectation boundaries
- suitability over hype
- rational decision-making

## Downgrade behavior

当信息不完整时，这条 skill 不会强行停住，也不会乱编。

### If brand info is missing
优先输出高质量通用科普稿，只做抽象型品牌承接。

### If doctor info is weak
不编头衔、资历、奖项，用一般性的专业判断逻辑表达。

### If clinic info is weak
不虚构流程体系与服务机制，只做中性承接。

### If the input is too promotional
自动把“最安全、最有效、最顶级”翻译成更理性、条件化、可解释的表达。

### If the user asks for unsupported claims
不直接配合承诺性表达，而转为教育型、解释型、风险边界型内容。
