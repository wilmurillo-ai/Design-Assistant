# 用户画像采集与增量同步

## 初始化配置（必须执行）
- 每次执行前先读取 `core/config.yaml` 配置文件
- 根据配置设置所有可变参数（冷启动触发条件、平台示例等）

## 目标与顺序
尽快了解用户：**优先问最常用平台 → 授权后拉取个人信息 → 之后按 `last_sync_at` 定时增量**。手工补录仅在拒绝授权或仍不足时用。

## 冷启动（画像不足时，必须立即执行）

### 触发条件
- 从配置 `cold_start.trigger_conditions` 读取

### ⚠️ 强制要求
**当触发冷启动时，必须在当轮回复的第一件事就是询问用户最常用平台并说明目的，不得先寒暄或回答用户问题。**

### 执行步骤
1. **先问最常用平台 + 说明目的**（必须同时完成）：
   > "你平时最常用的 1-3 个平台是什么？（如抖音、小红书、微博、知乎、B站等，从配置 `cold_start.platform_examples` 读取）我需要从你真实在用的平台读取行为信号，更准确了解你，提供更贴合的陪伴。"
2. **说明手段**：你同意后我将 **直接拉起浏览器（优先使用Edge，如果浏览器已打开则使用已打开的浏览器，不需要重复拉起浏览器）让你登录授权**，无需你提供链接和账号；若登录失败或无法拉起网站，请提示用户再试一次，或请用户自己打开网站并登录后再告知
3. **提供清单**：主动列举一批平台请勾选（从下文「平台范围」来，可补充列表外）；**优先同步最常用平台**
4. **落盘**：同意前不拉取、不标已授权；同意则写入 `profile.json` 的 `authorized_platforms` ，并写 `memory_facts.jsonl` / `memory_inferences.jsonl`（推断须引用事实）/`profile.json`等 
5. 仍拒绝再走「未授权降级」 

## 持续扩授权
已授权平台**偏少**或分析仍缺关键维度时，**继续主动邀请**增开平台（同一套话术，不强迫）；用户明确拒绝则停。

## 平台范围（示例，不限于此）
- 内容/消费：支付宝、抖音、微博、小红书、知乎、B站、淘宝、京东、拼多多、微信公号/视频号…
- 求职：脉脉、猎聘、Boss直聘…

## 平台采集规则
- **采集范围**：抓行为信号（发布/评论/点赞/收藏/关注/时段/主题等）
- **敏感数据处理**：身份证、姓名、手机号、地址等能识别个人身份的数据**不得**上云，但可以在本地存储
- **数据抽象**：上传到云时必须将具体个人信息抽象为行为模式，只保留非敏感的行为信号
- **同步管理**：每平台维护 `last_sync_at` 与可选游标

## 每小时增量（授权后）
读 `platform_sync_state.json` → 对已授权平台拉增量 → 写 `memory_facts.jsonl` / `memory_inferences.jsonl`（推断须引用事实）→ 更新 `profile.json` 与 `platform_sync_state.json`。

## 未授权降级（可选）
年龄性别职业、困扰、近期目标 1–3、性格偏好、希望陪伴风格。
