# SuperClaw by Kamir Labs

A production-ready collection of OpenClaw skills for disciplined, high-quality agent workflows.

## Included Skills (14)

- **brainstorming** ... Use before creative or feature-design work to clarify intent, constraints, and design direction before implementation.
- **dispatching-parallel-agents** ... Use when 2+ independent tasks can run in parallel without shared-state conflicts.
- **executing-plans** ... Use to execute a written implementation plan in a separate session with checkpointed reviews.
- **finishing-a-development-branch** ... Use after implementation and tests to choose merge, PR, or cleanup path.
- **receiving-code-review** ... Use when processing review feedback, especially when comments are ambiguous or technically uncertain.
- **requesting-code-review** ... Use before merge or completion to verify implementation quality and requirement alignment.
- **subagent-driven-development** ... Use to execute implementation plans via independent subagent tasks in the current session.
- **systematic-debugging** ... Use for bugs, test failures, and unexpected behavior before proposing fixes.
- **test-driven-development** ... Use before writing implementation code for any feature or bugfix.
- **using-git-worktrees** ... Use when feature work needs safe isolation from the current workspace.
- **using-superpowers** ... Use at session start to identify and load the right skill before acting.
- **verification-before-completion** ... Use before any completion claim to ensure fresh verification evidence exists.
- **writing-plans** ... Use when a multi-step task has requirements/spec and needs a concrete execution plan.
- **writing-skills** ... Use when creating, editing, or validating skills prior to deployment.

## Notes

- Skills are adapted for OpenClaw tool-based workflows.
- Focus is on reliability, verification, and repeatable execution patterns.
- This package is cleaned for public distribution and does not assume a specific personal environment.
