===== 进度汇报模板（使用前必读）=====

【任务开始前必执行】
将以下信息写入进度日志：
```
echo "[0%] 任务开始" >> /tmp/openclaw/progress.log && date +%s > /tmp/openclaw/progress_heartbeat.log
```

【每步完成后必执行两件事】
1. 发消息给用户（channel=meishi, target=<用户ID>）：
   - 用 message 工具发送进度消息
   - 格式如下（根据实际步骤数调整）：

```
[████████████████████] 100%

✅ 步骤1：<任务名>  — 已完成
✅ 步骤2：<任务名>  — 已完成
🔄 步骤3：<当前任务名>  — 进行中，预估还需X分钟
⏳ 步骤4：<任务名>  — 待开始
⏳ 步骤5：<任务名>  — 待开始
```

2. 更新进度日志：
```
echo "[50%] 步骤3完成" >> /tmp/openclaw/progress.log && date +%s > /tmp/openclaw/progress_heartbeat.log
```

【重要提醒】
- 预估时间（X分钟）= (总步骤数 - 当前步骤数) × 每步平均耗时
- 不要等用户回复再继续，每步完成后立即发消息
- 完成后发100%消息

【实用脚本】
如有需要，可用以下命令发送进度：
```
python3 ~/.openclaw/skills/agent-progress-bar/scripts/send_progress.py <当前步骤> <总步骤> "<步骤名称>" <用户ID>
```
