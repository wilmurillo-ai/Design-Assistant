# 时效性热点预警模板

## 概述

本文档定义了时效性热点预警的输出格式，帮助用户快速识别即将爆发的热点话题。

## 预警报告模板

```
# 时效性热点预警 · {时间范围}

## 预警概要
- **生成时间**：{YYYY-MM-DD HH:MM}
- **预测窗口**：{X小时}
- **预警等级分布**：
  - S级（紧急）：{N}个
  - A级（重要）：{N}个
  - B级（一般）：{N}个

---

## S级预警（紧急蹭热度）

### 预警 1：{话题标题}

**预警等级**：S级（爆发期）
**置信度**：{XX%}
**当前排名**：{第X名}
**预测排名**：{第X名}
**最佳窗口**：{X-X小时}

**话题特征**：
- 话题类型：{breaking/news/entertainment/tech/general}
- 爆发速度：{V=XX}
- 时效性评分：{T=0.XX}

**趋势分析**：
- 近1小时增量：{XXXX}
- 增长趋势：{↑↑↑ 加速上升}
- 预测依据：{基于历史相似话题分析}

**建议动作**：
1. 立即产出相关内容
2. 强调时效性和独家角度
3. {具体创作建议}

**风险提示**：
- {可能的合规风险}
```

## 预警等级说明

| 等级 | 标识 | 描述 | 建议动作 |
|------|------|------|----------|
| S级 | 🔴 紧急 | 正在爆发 | 1小时内产出 |
| A级 | 🟠 重要 | 即将爆发 | 2-4小时内产出 |
| B级 | 🟡 一般 | 有潜力 | 4-8小时内产出 |

## 预警字段说明

### alert_meta（元数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| ts | string | 预警生成时间 |
| predict_window | string | 预测时间窗口 |
| total_alerts | int | 预警总数量 |
| level_distribution | object | 各等级预警数量 |

### alerts（预警列表）

| 字段 | 类型 | 说明 |
|------|------|------|
| alert_id | string | 预警唯一标识 |
| level | string | 预警等级（S/A/B） |
| title | 预警标题 |
| confidence | float | 预测置信度（0-1） |
| current_rank | int | 当前排名 |
| predicted_rank | int | 预测排名 |
| best_window | string | 最佳蹭热度时间窗口 |
| topic_type | string | 话题类型 |
| velocity_index | float | 爆发速度指数 |
| timeliness | float | 时效性评分 |
| trend | string | 趋势（rising/stable/declining） |
| reason | string | 预测依据 |
| suggestions | array | 建议动作列表 |
| risks | array | 风险提示列表 |
| platforms | array | 相关平台 |
| keywords | array | 关键词 |

## 使用场景

### 场景1：日常监控

```
用户：trend watch 时间范围=4h

智能体执行：
1. 分析当前热点的话题特征
2. 识别具有爆发潜力的内容
3. 生成预警报告
4. 提供创作建议
```

### 场景2：定时任务

```
定时任务配置：
- 频率：每天 08:00, 12:00, 18:00, 22:00
- 窗口：预测未来4小时
- 过滤：仅显示S级和A级预警

输出：
- 时效性热点预警日报
- 包含具体创作方向建议
- 附带合规风险提示
```

### 场景3：专题追踪

```
用户：trend watch 关键词=iPhone 时间范围=8h

智能体执行：
1. 聚焦iPhone相关话题
2. 追踪相关话题爆发趋势
3. 生成专题预警
4. 提供iPhone专属创作建议
```

## 预警解读指南

### 如何使用预警

1. **优先级排序**：
   - 优先处理S级预警
   - A级预警及时跟进
   - B级预警作为储备

2. **时间窗口把握**：
   - 在最佳窗口内产出
   - 避免过早（信息不全）
   - 避免过晚（错过热度）

3. **创作方向选择**：
   - 参考suggestions
   - 结合自身定位
   - 差异化竞争

4. **风险规避**：
   - 注意risks提示
   - 遵守平台规则
   - 避免违规内容

### 置信度解读

| 置信度 | 含义 | 使用建议 |
|--------|------|----------|
| >90% | 高可信 | 重点跟进 |
| 70-90% | 中可信 | 建议跟进 |
| 50-70% | 较低可信 | 谨慎跟进 |
| <50% | 低可信 | 作为参考 |

## 预警优化建议

### 提高预警准确度

1. **数据完整性**：
   - 确保提供完整的增量数据
   - 多平台数据交叉验证
   - 历史数据积累

2. **模型调优**：
   - 根据实际效果调整权重
   - 定期复盘预测准确率
   - 引入更多特征

3. **人工干预**：
   - 结合人工判断
   - 识别模型盲区
   - 及时反馈修正
