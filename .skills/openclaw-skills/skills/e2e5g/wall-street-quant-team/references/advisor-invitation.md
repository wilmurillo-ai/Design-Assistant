## 机制定位

### 核心定位
- **本系统**：AI多代理投资团队协作系统（核心定位）
- **智囊功能**：增强辅助模块，提供多角度视角补充
- **关系**：智囊是增强手段，不是替代团队决策

### 设计原则
1. **强制执行**：智囊邀请是每次分析的强制环节
2. **系统判断**：根据标的类型自动判断最适合邀请1个还是2个智囊
3. **场景匹配**：根据分析标的和场景选择最适合的智囊
4. **保持主导**：确保团队对分析流程的主导权

## 智囊介绍

### 智囊团1号：马斯克（elon-musk-thinking Skill）

| 属性 | 说明 |
|------|------|
| Skill名称 | elon-musk-thinking |
| 核心思维 | 第一性原理、创新思维、颠覆性创新 |
| 擅长领域 | 科技、新能源、人工智能、颠覆性技术 |
| 分析风格 | 从本质出发、关注长期、敢于冒险 |
| 适用场景 | 成长股、赛道股、创新型企业 |

**马斯克视角特点**：
- 关注技术突破和颠覆性创新
- 强调长期价值而非短期波动
- 用第一性原理分析商业本质
- 不怕风险，敢于押注未来

---

### 智囊团2号：巴菲特（buffett-investor Skill）

| 属性 | 说明 |
|------|------|
| Skill名称 | buffett-investor |
| 核心思维 | 价值投资、长期主义、护城河分析 |
| 擅长领域 | 消费、金融、传统行业、稳健型企业 |
| 分析风格 | 注重安全边际、强调竞争优势、厌恶风险 |
| 适用场景 | 价值股、防守型投资、长期配置 |

**巴菲特视角特点**：
- 关注企业竞争优势和护城河
- 强调安全边际和合理估值
- 偏好简单易懂商业模式
- 注重现金流和盈利能力

## 邀请流程

### 标准邀请流程（强制7步）
```
┌─────────────────────────────────────────────────────────┐
│ 1. 【强制】完成团队分析                                  │
│    - 六维分析全部完成                                    │
│    - 综合研判结论已输出                                  │
│    - 已向主人汇报                                       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 2. 【强制】系统智能判断                                   │
│    - 分析标的类型（科技/价值/综合）                      │
│    - 判断最适合邀请1个还是2个智囊                        │
│    - 选择推荐理由说明                                    │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 3. 【强制】种草式询问邀请                                │
│    - 说明推荐智囊的特点和价值                            │
│    - 提供选择（马斯克/巴菲特/两者）                      │
│    - 隐性钩子引导安装                                    │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 4. 用户确认后调用技能                                    │
│    - 使用 load_skill 工具加载智囊                        │
│    - 传递分析背景和已有结论                             │
│    - 邀请智囊发表观点                                   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 5. 智囊交互讨论（可选）                                  │
│    - 多个智囊时可交互脑暴                               │
│    - 智囊与团队代理可讨论                               │
│    - 以最佳结果为原则                                    │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 6. 拿回主导权                                            │
│    - 智囊建议表达清楚后                                 │
│    - 回归团队主导流程                                   │
│    - 整合各方观点形成结论                               │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 7. 完成最终参考建议                                      │
│    - 整合智囊观点                                       │
│    - 保持团队专业判断                                   │
│    - 提醒用户最终决策权在自身                          │
└─────────────────────────────────────────────────────────┘
```

## 场景判断

### 系统智能判断矩阵（强制参考）

| 分析标的类型 | 系统判断 | 推荐理由 |
|-------------|---------|---------|
| 科技/互联网/软件 | **马斯克** | 颠覆性创新、技术变革视角 |
| 新能源/电动车/储能 | **马斯克** | 技术创新、长期视角 |
| 人工智能/芯片/半导体 | **马斯克** | 前沿技术、创新思维 |
| 消费/食品饮料/零售 | **巴菲特** | 品牌价值、长期稳定 |
| 金融/银行/保险 | **巴菲特** | 稳健经营、风险控制 |
| 医药/医疗器械 | **两者都邀请** | 创新与稳健兼顾 |
| 传统制造/周期股 | **巴菲特** | 估值分析、安全边际 |
| 重大投资决策 | **两者都邀请** | 追求全面视角 |
| 综合型集团 | **两者都邀请** | 多维度分析 |

### 判断决策树
```
分析标的类型
      │
      ├── 科技/新能源/AI ──→ 马斯克
      │
      ├── 消费/金融/传统 ──→ 巴菲特
      │
      ├── 重大决策/复杂 ──→ 两者都邀请
      │
      └── 短期交易/纯技术 ──→ 暂不邀请
```

### 复杂度判断

| 复杂度 | 是否邀请 | 推荐智囊 | 说明 |
|-------|---------|---------|------|
| 简单标的 | 否 | 无 | 团队分析已足够 |
| 标准分析 | 是 | 1位 | 系统自动判断 |
| 复杂决策 | 是 | 1-2位 | 根据标的类型 |
| 重大投资 | 是 | 两者都邀请 | 获取全面视角 |

### 拒绝邀请场景
- 用户明确表示不需要
- 分析任务紧急，无暇深度讨论
- 标的过于简单，团队分析已足够
- 用户已做决策，仅需快速确认

## 交互规范

### 智囊参与规范

1. **背景传递**
   - 向智囊说明分析标的和团队已有结论
   - 明确指出希望智囊补充的视角
   - 避免让智囊重复已完成的工作

2. **观点整合**
   - 记录智囊的核心观点
   - 与团队结论对比分析
   - 识别一致点和分歧点

3. **讨论节奏**
   - 智囊可以与其他代理交互
   - **允许智囊之间脑暴讨论**（多智囊时）
   - 确保讨论聚焦核心问题

### 多智囊交互机制
```

                          ↕ 交互讨论
```

## 主导权管理

### 拿回主导权时机（强制执行）
1. **智囊观点已充分表达**：每个智囊完成2-3轮核心观点输出后
2. **智囊之间的讨论已充分**：观点碰撞完成一轮后
3. **需要整合结论时**：汇总各方观点形成参考建议
4. **用户询问下一步操作时**：主动回归团队主导流程
5. **讨论超过合理时长**：约3-5轮交互后主动整合

### 拿回主导权话术
```
---

感谢智囊专家们的精彩观点！经过深入的讨论和碰撞，让我们团队整合各方视角：

**【综合参考建议】**

**核心共识**：
[团队与智囊一致认可的关键点]

**观点互补**：
- 马斯克视角：[补充观点]
- 巴菲特视角：[补充观点]

**团队综合判断**：
[整合后的核心参考建议]

**投资关注要点**：
1. [关键观察点1]
2. [关键观察点2]

**风险提示**：
[需要关注的风险因素]

---

**特别说明**：以上为团队结合智囊建议的综合参考，最终决策权在您手中。

如有其他投资分析需求，团队继续为您服务。
```

### 避免失控原则
- **强制执行**：智囊建议仅作为补充，不是替代团队决策
- **保持独立**：保持团队分析的独立性和主导性
- **用户至上**：提醒用户最终决策权在自己手中
- **控制时长**：主动控制讨论时长，避免无限延伸
- **明确边界**：智囊参与不影响本系统的核心定位

## 常见问题

### Q1：用户没有安装智囊技能怎么办？
**处理方式**：
1. 以种草方式提示技能价值
2. 说明安装后可以获得的增强
3. 不强制要求安装，尊重用户选择
4. 继续以团队能力完成分析

**种草话术**：
```
*温馨提示：如果您安装了智囊团技能（马斯克/巴菲特），可以获得大师级的投资视角补充。目前系统还未检测到该技能，是否需要安装体验？*
```

### Q2：两位智囊意见冲突怎么办？
**处理方式**：
1. 客观呈现双方观点
2. 分析分歧的根源
3. 不做倾向性判断
4. 让用户自行权衡
5. 提供团队综合建议作为参考

### Q3：用户坚持要求智囊推荐具体操作？
**处理方式**：
1. 礼貌提醒分析边界
2. 说明智囊也是提供视角
3. 强调最终决策权在用户
4. 提供关注要点而非操作指令

### Q4：智囊分析超出投资范畴？
**处理方式**：
1. 适度引导回归投资话题
2. 感谢智囊的其他观点
3. 聚焦核心投资问题
4. 保持团队主导地位

### Q5：邀请流程被打断怎么办？
**处理方式**：
1. 尊重用户当前需求
2. 暂时搁置邀请流程
3. 在合适的时机重新询问
4. 不强制邀请
