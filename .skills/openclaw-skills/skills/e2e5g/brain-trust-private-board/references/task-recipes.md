# 任务配方

遇到下面这些高频任务，优先按固定步骤走，不要每次都临场发明流程。

## 产品方向判断

推荐专家：

- 乔布斯
- 拉姆斯
- 克里斯坦森
- 德鲁克

步骤：

1. 定义目标用户和核心任务。
2. 判断产品真正不可替代的价值点。
3. 审视功能是否过多、过散或过早。
4. 判断新方案是增量优化还是新路径切入。
5. 给出“保留、砍掉、延后验证”的建议。

适合输出：

- 产品决策备忘录
- 功能取舍建议
- 下一轮验证清单

## 商业战略评估

推荐专家：

- 德鲁克
- 波特
- 巴菲特
- 芒格

步骤：

1. 明确组织目标和约束。
2. 判断竞争位置和取舍是否清楚。
3. 判断优势是否可持续。
4. 审视方案是否建立在错误假设上。
5. 给出“该做、缓做、不做”的战略动作。

适合输出：

- 战略分析 memo
- 方案对比
- 高管决策摘要

## 风险与红队审查

推荐专家：

- 塔勒布
- 卡尼曼
- 戴明
- 芒格

步骤：

1. 列出核心假设。
2. 找出最脆弱、最难证伪的假设。
3. 逆推最可能失败的路径。
4. 明确哪些预警信号应该被持续监控。
5. 设计止损、缓冲和保护栏。

适合输出：

- 红队报告
- 风险清单
- 保护栏设计

## 组织与执行问题

推荐专家：

- 德鲁克
- 戴明
- 韦尔奇
- 梅多斯

步骤：

1. 先判断问题是目标问题、结构问题还是执行问题。
2. 识别是人出错，还是系统在制造错误。
3. 判断现有流程哪里让协作成本过高。
4. 找到真正的杠杆点，而不是头痛医头。
5. 给出短期修正和中期结构优化建议。

适合输出：

- 管理建议
- 组织诊断
- 执行改进清单

## 学习、解释与理解校验

推荐专家：

- 费曼
- 苏格拉底
- 亚里士多德
- 图灵

步骤：

1. 用最朴素的话重讲问题。
2. 追问关键定义是否清楚。
3. 找出论证链条中跳步的位置。
4. 补上结构、因果和边界条件。
5. 最后用一个外行也能懂的版本收束。

适合输出：

- 通俗解释
- 学习笔记重构
- 概念澄清稿

## 伦理与原则冲突

推荐专家：

- 康德
- 亚里士多德
- 苏格拉底

步骤：

1. 先把问题的冲突点说清楚。
2. 判断是否涉及普遍化规则。
3. 判断是否把人仅当工具。
4. 判断情境中的尺度、目的和德性。
5. 给出原则上可接受、现实中可执行的结论。

适合输出：

- 伦理分析
- 原则备忘录
- 决策边界说明

## 文案、品牌与表达优化

推荐专家：

- 乔布斯
- 原研哉
- 拉姆斯
- 费曼

步骤：

1. 先明确对象是谁、想让对方做什么。
2. 判断信息是否过多、过散或无主线。
3. 判断表达是否有气质但不失清晰。
4. 删除花哨但无用的信息。
5. 给出更有辨识度但更易理解的表达。

适合输出：

- 品牌表达建议
- 文案重构建议
- 信息架构优化
