# 输出风格模板

这份文件不是让回答变花，而是让输出更像成熟成品。

## 总原则

- 先结论，后展开。
- 先收束，后展示差异。
- 风格服务理解，不服务表演。
- 同一个回答里不要混杂太多口吻。

## 决策 Memo 风格

适合：

- 方案选择
- 路径判断
- 高层快速阅读

写法：

1. 先用 2-4 句写结论。
2. 再写为什么。
3. 最后写下一步动作和风险。

语气特征：

- 干净
- 有判断
- 不拖泥带水

避免：

- 长篇铺垫
- 名人名言堆砌
- 只分析不拍板

## 专家会诊风格

适合：

- 多专家 panel
- 需要展示共识与分歧

写法：

1. 先写共识。
2. 再写分歧。
3. 最后做裁决。

语气特征：

- 稳
- 清晰
- 有主持感

避免：

- 把专家写成长对话
- 为了热闹牺牲信息密度

## 私董会风格

适合：

- `boardroom` 模式
- 用户想看真实讨论推进
- 需要边聊边成形，而不是整齐轮次发言

写法：

1. 先抛出议题和约束。
2. 让最适合的专家先开口。
3. 用自然接话推进，不固定轮次。
4. 把真正有效的追问、分歧、脑暴保留下来。
5. 一旦形成结论，立刻收束。

语气特征：

- 像成熟闭门讨论
- 有碰撞感
- 但不失控、不拖沓

避免：

- 所有人平均发言
- 像排队发言稿
- 为了“像聊天”而牺牲结论密度

## 红队报告风格

适合：

- 风险审查
- 上线前把关

写法：

1. 先点最危险的假设。
2. 再写失败路径。
3. 最后写保护栏和预警信号。

语气特征：

- 冷静
- 具体
- 不恐吓，但不粉饰

避免：

- 空泛风险
- 没有优先级
- 只说危险不说怎么防

## 评审意见风格

适合：

- skill / prompt / workflow / 文案 / 方案评审

写法：

1. 整体判断
2. 主要问题
3. 优先级
4. 改法

语气特征：

- 直接
- 公平
- 能落地

避免：

- 太委婉导致问题不痛不痒
- 只批评不提供改法

## 教学解释风格

适合：

- 讲复杂概念
- 降低理解门槛

写法：

1. 先一句话解释
2. 再拆成 3-5 个最关键点
3. 最后给例子或自测题

语气特征：

- 朴素
- 轻松
- 不装深奥

避免：

- 术语连发
- 定义套定义

## 文案重构风格

适合：

- 传播表达
- 品牌文案

写法：

1. 先判断原文主问题
2. 给 2-3 个方向
3. 再给最终改写版

语气特征：

- 有辨识度
- 但仍然清楚

避免：

- 一味追求漂亮
- 不顾转化目标

## 默认推荐

如果用户没指定风格：

- 做决策：用 `Memo 风格`
- 做多专家：优先 `私董会风格`
- 做风险：用 `红队报告风格`
- 做评审：用 `评审意见风格`
- 做解释：用 `教学解释风格`
