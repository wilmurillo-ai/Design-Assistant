# 专家选择指南

## 先选模式，再选专家

### 适合 `single`

- 用户已经点名专家
- 问题有明确主维度
- 需要快速得到一个清晰判断

### 适合 `compare`

- 用户在两个方向之间犹豫
- 需要看不同方法论的差异
- 想知道为什么同一问题会得出不同建议

### 适合 `debate`

- 问题天然存在对立立场
- 需要把潜在冲突摊开
- 想测试方案在反对意见下是否站得住

### 适合 `panel`

- 问题跨多个维度
- 单一专家容易偏科
- 需要会诊式视角

### 适合 `critique`

- 已经有草案、方案、文章、产品方向
- 当前重点是挑错、找漏项、提改进

### 适合 `red-team`

- 用户过于乐观
- 方案将投入真金白银或大量时间
- 需要优先暴露失败路径、脆弱环节和误判来源

### 适合 `synthesis`

- 用户不想看纯讨论，更想要最后结论
- 前面已经存在多个观点，需要整合

## 按任务类型选专家

### 产品与体验

- 史蒂夫·乔布斯：看核心价值、用户体验和产品聚焦
- 唐纳德·诺曼：看可用性、反馈和交互是否自然
- 迪特·拉姆斯：看功能与简洁是否统一
- 原研哉：看感受、留白、品牌气质与表达
- 克莱顿·克里斯坦森：看用户任务与颠覆机会

### 商业与战略

- 彼得·德鲁克：看目标、组织、管理与执行
- 迈克尔·波特：看竞争位置、护城河和取舍
- 沃伦·巴菲特：看商业质量、长期价值和安全边际
- 查理·芒格：看多元思维模型和误判来源
- 安迪·格鲁夫：看战略转折点、组织警觉和执行升级

### 运营与质量

- 爱德华兹·戴明：看流程、系统、波动和持续改进
- 杰克·韦尔奇：看执行强度、组织活力和业务取舍
- 多内拉·梅多斯：看系统结构、反馈回路和杠杆点

### 风险与不确定性

- 丹尼尔·卡尼曼：看认知偏差和判断失真
- 纳西姆·塔勒布：看脆弱性、尾部风险和可选性
- 沃伦·巴菲特：看下行保护和错误成本

### 学习、解释与研究

- 理查德·费曼：看是否真的理解，能否讲清楚
- 苏格拉底：看概念是否站得住，前提是否值得追问
- 亚里士多德：看定义、分类、因果与论证结构
- 艾伦·图灵：看形式化、流程化和边界条件

### 创新与技术路线

- 埃隆·马斯克：看第一性原理、成本重构和工程推进
- 艾伦·图灵：看抽象模型与计算路径
- 阿尔伯特·爱因斯坦：看概念突破与思想实验
- 克莱顿·克里斯坦森：看新路径如何切入旧格局

### 竞争与破局

- 孙子：看态势、资源配置和避实击虚
- 安迪·格鲁夫：看拐点与组织应激反应
- 迈克尔·波特：看位置、边界与长期竞争逻辑

### 内容与传播

- 大卫·奥格威：看标题、利益点和说服效率
- 原研哉：看品牌气质、留白和感受
- 史蒂夫·乔布斯：看叙事张力和核心价值表达
- 理查德·费曼：看表达是否真的讲清楚

### 伦理与原则冲突

- 伊曼努尔·康德：看规则是否可普遍化，是否把人当目的
- 亚里士多德：看情境中的德性、尺度和目的
- 苏格拉底：看定义是否清楚、前提是否成立

### 长期演化与系统变化

- 查尔斯·达尔文：看环境压力、适应与渐进演化
- 多内拉·梅多斯：看系统结构如何塑造结果
- 彼得·德鲁克：看组织如何持续调整与生存

## 常用组合配方

### 产品决策

- 乔布斯 + 诺曼 + 拉姆斯
- 适合同时判断价值、可用性和简洁度

### 新产品方向

- 乔布斯 + 克里斯坦森 + 德鲁克
- 适合同时看用户价值、切入机会和组织落地

### 商业模式评估

- 巴菲特 + 波特 + 芒格
- 适合同时看质量、竞争与误判

### 战略与组织调整

- 德鲁克 + 波特 + 格鲁夫 + 梅多斯
- 适合同时看目标、竞争位置、转折点和系统后果

### 风险审查

- 卡尼曼 + 塔勒布 + 戴明
- 适合发现偏见、尾部风险和流程脆弱性

### 学习与讲解

- 费曼 + 苏格拉底
- 适合把复杂问题讲明白并顺手暴露理解漏洞

### 概念澄清与论证

- 苏格拉底 + 亚里士多德 + 康德
- 适合概念、逻辑与规范性问题

### 传播与文案

- 奥格威 + 原研哉 + 费曼
- 适合同时看销售力、气质和清晰度

## 选择时的提醒

- 不要因为专家名气大就优先选他。
- 如果任务是执行性问题，优先选能给操作标准的专家。
- 如果任务是探索性问题，优先选能打开视角的专家。
- 如果任务有明显风险暴露，至少补一个“悲观型”或“校验型”专家。
- 如果任务要说服别人，优先补一个传播或表达型专家。
