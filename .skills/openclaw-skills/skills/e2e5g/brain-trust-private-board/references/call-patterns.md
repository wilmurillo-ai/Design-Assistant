# 调用句式

下面这些句式最容易把这个 skill 用对。

## 最稳的通用句式

```text
用 expert-thinking 帮我分析这个问题。
目标：
背景：
约束：
我现在卡在：
希望输出：
```

## 单专家

```text
用 expert-thinking，按费曼的方式把这个概念讲清楚，并给我一个外行能懂的版本。
```

```text
用 expert-thinking，按德鲁克的框架判断这个组织问题的根因，并给出可执行建议。
```

## 双专家对照

```text
用 expert-thinking，对比乔布斯和拉姆斯会如何看这个产品方案，最后给我综合建议。
```

```text
用 expert-thinking，对比巴菲特和波特会如何评估这个商业模式，输出一个简短 memo。
```

## 多专家会诊

```text
用 expert-thinking，找 3 位最适合的专家会诊这个产品方向问题，并收束成一个推荐路径。
```

## 私董会

```text
用 expert-thinking，按私董会模式讨论这个问题。谁最适合当前话题谁发言，不要固定轮次，讨论出结果就停。
```

```text
用 expert-thinking，开一场小型私董会来讨论这个方案。允许专家自然接话、追问、反驳和脑暴，最后收束成结论和下一步。
```

## 挑错 / 评审

```text
用 expert-thinking，评审这份方案。重点找结构漏洞、遗漏假设和不现实的地方，并按优先级排序。
```

```text
用 expert-thinking，评审这份 SKILL.md。重点看触发描述、流程可执行性、参考文件拆分和真正可用性。
```

```text
用 expert-thinking，评审这个 workflow。重点看路由是否清楚、步骤之间是否耦合过重、哪里最容易失真，以及怎么重构更稳。
```

```text
用 expert-thinking，按私董会模式评审这份 SKILL.md / prompt / workflow。让最适合的人先开口，哪里有分歧就让别人接着拆，讨论到形成修改方向就停止。
```

## 红队

```text
用 expert-thinking，给这个方案跑一次红队。告诉我最可能失败的路径、最早出现的预警信号，以及该加什么保护栏。
```

## 文案与表达

```text
用 expert-thinking，从奥格威、原研哉、费曼三个角度看这段文案，最后给我一个更好用的改写版本。
```

## 指定输出气质

```text
用 expert-thinking，按决策 memo 风格输出，先给结论，再给理由和下一步。
```

```text
用 expert-thinking，按红队报告风格输出，重点写最危险假设、失败路径和保护栏。
```

```text
用 expert-thinking，按私董会风格输出，保留自然讨论推进感，但最后一定要收束出明确结论。
```

## 最好补齐的信息

- 你想解决什么问题
- 你已经有什么版本
- 你最担心什么
- 时间、预算、人力限制是什么
- 你要的是分析、对比、评审、红队还是最终方案
