# 质量护栏

交付前，用这份清单快速防止 skill 失真。

## 常见坏味道

### 1. 只有风格，没有方法

症状：

- 语气很像专家
- 但判断标准、分析顺序和决策重点并没有变

修正：

- 回到“这个专家会先看什么、如何判断、最担心什么”

### 2. 专家堆砌

症状：

- 选了很多专家
- 但观点高度重复

修正：

- 删除同质化专家，只保留互补角色

### 3. 只会分观点，不会收束

症状：

- 罗列了很多专家意见
- 最后没有明确推荐路径

修正：

- 强制补上“在当前约束下，我建议选哪条路径”

### 4. 名言堆积

症状：

- 大量金句、类比、鸡汤
- 信息密度下降

修正：

- 每位专家最多保留极少量辨识度语气

### 5. 忽略现实约束

症状：

- 观点听起来高级
- 但完全不考虑预算、时间、人力或执行顺序

修正：

- 在综合判断里必须写出约束影响

### 6. 假装确定

症状：

- 明明信息不足
- 语气却非常笃定

修正：

- 明确写出假设、缺失信息和置信度

## 最低交付标准

至少确保输出里有：

- 问题重述
- 专家选择理由
- 每位专家的关键判断
- 综合建议
- 下一步动作

## 高风险任务附加检查

如果任务可能消耗大量时间、金钱或组织资源，再检查：

- 是否已经加入红队型专家
- 是否写出失败路径
- 是否写出预警信号
- 是否写出止损或保护栏

## 不确定性表达

当结论不够稳定时，优先这样表达：

- “基于当前信息，我倾向于……”
- “这个建议成立的前提是……”
- “如果前提不成立，结论应改为……”

不要用这类表达：

- “一定会”
- “绝对正确”
- “这就是唯一答案”
