# Multi-Agent Governance

## Suggested write permissions
- daily memory: most working agents may write
- `.learnings/`: agents may propose, but shared review is recommended
- durable memory: conservative writes, prefer approval flow
- skill docs: treat as controlled promotion targets

## Approval flow
1. detected candidate
2. proposed patch/report
3. review approved or rejected
4. apply and mark lifecycle

## Lifecycle states
- active
- candidate-for-promotion
- pending-review
- deprecated
- superseded
- archived
