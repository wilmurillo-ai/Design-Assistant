# Shared Memory OS 中文说明

## 这是什么
`Shared Memory OS` 不是单纯的记忆文件，而是一套共享记忆治理规则。

目标是把 workspace 里的知识分层管理，让多个会话、多个 agent、跨天工作时都能：
- 记得住
- 找得到
- 能复用
- 能持续进化

## 三层记忆
### 1. 长期记忆
用于保存：
- 用户长期偏好
- 稳定约束
- 长期决策
- 持久工作方式

当前 workspace 通常对应：
- `MEMORY.md`

### 2. 当天/近期工作记忆
用于保存：
- 当前任务状态
- 临时上下文
- 阻塞
- 当天结论

当前 workspace 通常对应：
- `memory/YYYY-MM-DD.md`

### 3. 经验库 / 可复用教训
用于保存：
- 用户纠正
- 排障结果
- 成功路径
- 更优策略
- 工具或外部 API 的行为规律

当前 workspace 通常对应：
- `.learnings/`

## 核心规则
### 1. 状态和经验要分开
- 任务状态写到 daily memory
- 长期规则写到 `MEMORY.md`
- 可复用经验写到 `.learnings/`

### 2. 默认主动维护经验库
以下情况默认主动写 `.learnings/`：
- 用户纠正了 agent
- 失败被排查并修复
- 找到了更优的通用策略
- 多次尝试后终于拿到稳定结果
- 外部工具/API 出现异常并找到可靠 workaround

重点：
- 不只是纠正才记
- 成功路径也要记
- 不用等用户提醒

### 3. 只有“会长期成立”的东西才进长期记忆
短期状态不要直接塞进 `MEMORY.md`。

### 4. 共享记忆要可复用、可治理
尽量记录通用原则，不要堆原始日志、敏感信息、一次性细节。

## 标准流程
### 做事前
- 读长期记忆
- 读当天/近期记忆
- 看经验库有没有可复用结论

### 做事中
- 临时状态写到 daily memory
- 长期变化再更新 `MEMORY.md`
- 留意这次有没有值得沉淀的经验

### 做事后
做一次收尾检查：
- 用户有没有纠正我？
- 有没有失败并修好？
- 有没有发现更通用的规则？
- 有没有经过多次尝试才找到稳定路径？
- 有没有内容该从 daily memory 提升到长期记忆？
- 有没有旧内容该归档、合并、清理？

如果前四项任一为“是”，默认更新 `.learnings/`。

## 自学习进化
Shared Memory OS 现在不仅记录，还会推动“自我进化”：
1. 从真实任务里 harvest lessons
2. 写入 `.learnings/`
3. 重建 `.learnings/INDEX.md`
4. 检查重复经验和 stale 规则
5. 找出 promotion candidates
6. 把稳定重复的经验提升成长期规则

## 自动维护
当前已经支持：
- 健康检查
- `.learnings/` 索引重建
- 重复经验检测
- promotion candidate 检测
- 定时自动维护

## 适合谁
适合：
- 长期运行的 OpenClaw workspace
- 多 agent / 多 session 协作
- 需要跨天持续维护上下文
- 希望把排障经验真正沉淀下来的人

## 一句话
它的目标不是“多记点东西”，而是：

> 让共享记忆变成一个能持续自我维护、自我进化的工作系统。
