# 📝 Daily Report Recorder

> 工作日报记录助手

## 功能简介

自动记录、整理并归档用户每天的工作内容，生成符合企业规范的 Markdown 日报。

## ✨ 特性

- 📝 **自动记录** - 接收工作事项，智能整理为日报格式
- 🔄 **增量更新** - 同一天多次提交自动合并，不重复创建
- 📋 **标准化模板** - 今日完成、处理中事项、问题风险、明日计划
- 💼 **企业口径** - 自动优化为专业汇报表述
- 🧠 **时间管理** - 引入柳比歇夫式时间管理思路

## 使用方式

当用户输入以下内容时自动触发：
- 今日工作内容
- 问题排查记录
- 环境搭建、接口联调
- 自动化开发、测试验证
- 会议纪要
- 后续计划

## ⏰ 定时配置（可选）

如需定时自动记录，可配合 Cron 使用：

```bash
# 每天下午5点自动记录（仅触发记录，非强制）
cron add \
  --name "日报记录-每日17点" \
  --schedule '{"kind":"cron","expr":"0 17 * * 1-5"}' \
  --payload '{"kind":"systemEvent","text":"请记录今日工作"}' \
  --sessionTarget main
```

### Cron 表达式说明

| 字段 | 含义 | 取值范围 |
|------|------|----------|
| 第1个 | 分钟 (minute) | 0-59 |
| 第2个 | 小时 (hour) | 0-23 |
| 第3个 | 日期 (day) | 1-31 |
| 第4个 | 月份 (month) | 1-12 |
| 第5个 | 星期 (weekday) | 0-6 (0=周日) |

### 常用示例

```bash
# 每天早上9点
"0 9 * * *"

# 每天下午5点
"0 17 * * *"

# 每周一至周五上午9点
"0 9 * * 1-5"

# 每周一至周五下午5点
"0 17 * * 1-5"

# 每月1号上午9点
"0 9 1 * *"

# 每周一早上9点半
"30 9 * * 1"
```

## 输出路径

```
/data/reports/daily/YYYY-MM-DD.md
```

## 📖 示例

### 输入（用户）
```
今日完成：1.完成用户登录接口优化 2.修复订单列表加载慢的问题
```

### 输出（日报）
```markdown
# 日报 - 2026-03-19

## 今日完成
- 完成用户登录接口优化
- 修复订单列表加载慢的问题

## 处理中事项
- 待补充

## 问题与风险
- 暂无

## 明日计划
- 待补充

## 备注
- 生成时间：2026-03-19 10:00
- 状态：已提交
```

## 📌 注意事项

- 只基于用户提供的真实信息整理，不虚构内容
- 技术问题优先整理为"现象—原因—处理动作—当前状态"
- 避免空话、废话，保持简洁专业
