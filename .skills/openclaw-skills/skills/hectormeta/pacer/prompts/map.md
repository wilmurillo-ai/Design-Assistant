# Map 模块提示词 — 路径规划

## 触发条件

- 扫描完成后自动进入
- 用户说"帮我规划路径"或"我有哪些选择"

## 模式判断

根据扫描结尾用户的回答自动切换：
- 用户说出了具体目标 → **模式A：单目标深度规划**
- 用户说"不确定"/"没有"/"不知道" → **模式B：多方向对比**

---

## 模式A：单目标深度规划

### 触发条件
用户说出了具体目标，如"我想转产品经理"/"我想做医疗AI创业"

### 文字输出（先于图表输出）

```
🎯 目标：[用户的目标职业/方向]

✅ 你的优势匹配
  • [优势1] — [与目标的具体关联，1句话]
  • [优势2] — [与目标的具体关联，1句话]

⚠️ 你需要补齐的差距
  • [差距1]
  • [差距2]

⏱️ 预计见到结果：[X个月]
🚧 最大风险：[1条诚实的风险评估，不回避]
```

### 图表一：技能雷达图

输出完整 HTML（基于 `charts/map-radar.html` 模板），数据注入规则：

- `labels`：从扫描结果提取5-6个关键技能维度，与目标职位相关
- `data_current`：根据用户CV/描述对每个维度评分，0-10分，需有合理依据
- `data_target`：根据目标职位要求评分，0-10分
- 评分时在提示词内部记录依据，不输出给用户
- 颜色：现在用 `#64b5f6`（蓝），目标用 `#f48fb1`（粉）

图表下方附一句话：
> "差距最大的是[维度名]，这是你最需要集中突破的地方。"

### 图表二：阶段甘特条形图

输出完整 HTML（基于 `charts/map-compare.html` 单目标模式），数据注入规则：

- 3行，每行对应一个阶段（如：打基础 / 实战积累 / 转型落地）
- X轴：时间（周数），从0到目标总周数
- 每个阶段的开始时间和持续时长根据用户实际情况填入
- 颜色：阶段一 `#26a69a`，阶段二 `#42a5f5`，阶段三 `#ab47bc`
- 每条 bar 上标注阶段名称和核心任务

### 结尾问题

> "这三个阶段里，你觉得哪个对你来说最难跨过？"

---

## 模式B：多方向对比

### 触发条件
用户说"不确定"/"帮我看看有哪些选项"，或扫描时无法判断方向

### 文字输出（先于图表输出）

```
🔍 基于你的情况，这里有 [N] 条路可以走。
先看整体对比，再告诉我哪个方向你想深入了解。
```

### 图表：多维度对比条形图

输出完整 HTML（基于 `charts/map-compare.html` 多方向模式），数据注入规则：

对比维度（固定4项）：
1. 适配度（与用户背景匹配程度）
2. 收入潜力（6个月后预期）
3. 上手难度（越低越容易上手）
4. 风险程度（越低越稳健）

数据要求：
- 每个方向一组颜色的 bar：方向A `#26a69a`，方向B `#42a5f5`，方向C `#ab47bc`
- Y轴：0-100分
- **所有分值必须基于用户扫描数据推算**，在提示词内部记录评分依据

### 图表下方每个方向说明

```
方向A — [方向名称]：[为什么适合你，1句话] · 第一步：[本周能做的事]
方向B — [方向名称]：[为什么适合你，1句话] · 第一步：[本周能做的事]
方向C — [方向名称]：[为什么适合你，1句话] · 第一步：[本周能做的事]
```

### 结尾问题

> "这几个方向，哪个让你多看了一眼？"

（用户选定一个方向后，进入该方向的单目标深度规划，或直接进入 simulate.md）

---

## 通用规则

- 所有分值必须基于用户扫描数据推算，评分依据在内部记录，不输出给用户
- 图表先于文字说明输出，文字作为图表的解读补充
- 每个图表输出后必须跟一个问题，引导用户进入下一步
