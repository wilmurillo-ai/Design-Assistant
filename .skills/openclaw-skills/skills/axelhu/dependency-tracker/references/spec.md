# Dependency Tracker 详细规范

## 检查项目

| 检查项 | 命令 |
|--------|------|
| Node.js 版本 | `node -v` |
| npm 版本 | `npm -v` |
| 全局包列表 | `npm list -g --depth=0` |
| 可更新包 | `npm outdated` |

## 报告格式

```markdown
# Dependency Tracker 报告

**时间**: YYYY-MM-DD HH:MM

## 环境

| 项目 | 版本 |
|------|------|
| Node.js | v22.22.0 |
| npm | 10.9.0 |

## 全局包

| 包名 | 版本 |
|------|------|
| openclaw | 2.x.x |
| minimax-mcp-js | 1.x.x |

## 可更新包

| 包名 | 当前版本 | 最新版本 | 建议 |
|------|----------|----------|------|
| xxx | 1.0.0 | 1.1.0 | 可更新 |

## 建议

- 暂无
```

## 执行规则

1. 所有 shell 命令必须同步执行（不使用 background 参数）
2. 原子化写入输出文件：先写 .tmp 文件，然后 mv 到最终路径
3. 在生成进程退出前不要读取输出文件
4. 发送报告前先生成执行日志
