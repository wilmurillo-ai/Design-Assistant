# 每日日志详细规范

## 必须包含

| 字段 | 说明 |
|------|------|
| **what** | 做了什么 |
| **why** | 为什么这样做（关键决策） |
| **files** | 变更的文件路径 |
| **commands** | 执行的命令及输出 |
| **errors** | 错误信息（如果有） |

## 禁止

- 事后概括性总结
- 只有结果没有过程
- 过于简略

## 示例格式

```markdown
# 2026-03-16 工作日志

## 任务：xxx

### 执行步骤
1. 做了 xxx → 结果 xxx

### 决策
- 选择方案 A 而非 B，因为：A 更适合

### 输出
- 新增：xxx
- 修改：xxx

### 错误
- xxx 失败 → 原因 → 解决
```

## 写入流程

1. 读取当前日记文件（如已存在）
2. 智能整合新内容（保留旧内容，只追加/优化）
3. 写回文件

**注意：** 不要删除旧内容，只做增量优化。

## 自检清单

- [ ] 有 what
- [ ] 有 why
- [ ] 有文件路径
- [ ] 有命令输出
- [ ] 非事后总结
