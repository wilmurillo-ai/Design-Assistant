# TODO List 详细规范

## 文件位置

所有 TODO list 文件存放在：`memory/todo/`

## 初始化

首次使用需要创建目录和空文件：

```bash
mkdir -p memory/todo
touch memory/todo/TODO.md memory/todo/DONE.md
```

初始内容：

```markdown
# TODO

## 进行中

（暂无）

## 待做

（暂无）
```

```markdown
# DONE

## 已完成

（暂无）
```

## 任务生命周期

```
新增 → 进行中 → 已完成
```

### 1. 新增

当有明确任务时，添加到 TODO.md 的"待做"区域：

```markdown
- [ ] 任务描述 · 原因 · 日期 @负责人
```

### 2. 进行中

开始处理时，移动到"进行中"区域：

```markdown
- [ ] 任务描述 · 原因 · 开始日期 @负责人
```

### 3. 已完成

任务完成后：
1. 从 TODO.md 移除
2. 添加到 DONE.md

```markdown
- [x] 任务描述 · 如何完成 · 完成日期 @负责人
```

## 记录规范

### TODO 简洁记录

| 字段 | 说明 | 必须 |
|------|------|------|
| 任务描述 | 要做什么 | ✅ |
| 原因 | 为什么做 | ✅ |
| 日期 | 添加日期 | ✅ |
| 负责人 | 谁来做 | ✅（可写@自己） |

### DONE 简洁记录

| 字段 | 说明 | 必须 |
|------|------|------|
| 任务描述 | 做了什么 | ✅ |
| 如何完成 | 怎么做的 | ✅ |
| 日期 | 完成日期 | ✅ |
| 负责人 | 谁做的 | ✅ |

## 示例

### 添加新 TODO

```
- [ ] 调研 Gstack 项目 · 借鉴优秀实践 · 2026-03-20 @小爪子
```

### 标记为进行中

保留原内容，只是在 TODO 文件内移动位置到"进行中"

### 标记为 DONE

```
- [x] 调研 Gstack 项目 · 读原repo，整理15角色分析，写各agent学习指南 · 2026-03-20 @小爪子
```

## 不记录什么

- 日常琐事（这是日记的事）
- 模糊的想法（只有明确任务才记）
- 详细过程（只记结论）

## 更新时机

| 时机 | 操作 |
|------|------|
| 接到明确任务 | 添加到 TODO |
| 开始处理 | 移到"进行中" |
| 任务完成 | 移到 DONE |
| 有进展 | 更新当前任务状态 |
| 发现新任务 | 添加到 TODO |

## 示例场景

### 场景1：用户分配任务

**输入**："调研一下 gstack 项目，然后给各 agent 写学习指南"

**操作**：
1. 添加 TODO：调研 Gstack 项目
2. 调研完成后，添加 TODO：为各 agent 写学习指南
3. 完成后移到 DONE

### 场景2：主动发现需要做的事

**输入**："日志里发现 Token Plan API 数据不一致，需要查一下"

**操作**：
1. 添加 TODO：调查 Token Plan API 数据不一致原因
2. 完成后移到 DONE，说明调查结果

### 场景3：每日检查

**输入**：（无输入，每隔一段时间主动检查）

**操作**：
1. 读取 TODO.md
2. 看有哪些进行中的任务
3. 有完成的移到 DONE
