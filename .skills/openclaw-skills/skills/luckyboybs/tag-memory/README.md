# TagMemory - 标签化长期记忆系统

让 AI 记忆具备人类审核机制的长期记忆系统。

**核心理念：记忆不是 AI 自己总结的，而是经过用户确认的。**

## 功能特点

- 🏷️ **标签化存储** — 为记忆打上语义标签
- 🔍 **BM25 搜索** — 快速准确的关键词搜索
- ⏰ **时间标签** — 按时间段查询历史记忆
- 🤝 **闲时核对** — 主动检查记忆正确性
- 📊 **定期归纳** — 生成总结，主动询问确认
- 🛡️ **隐私优先** — 本地存储，不上传云端

## 快速开始

### 安装

```bash
cd ~/.openclaw/workspace/skills/tag-memory
python src/database.py --init  # 初始化数据库
```

### 基础用法

```bash
# 存储记忆
python src/skill.py store "用户偏好 tabs 缩进" "#偏好"

# 查询记忆
python src/skill.py query "tabs 缩进"

# 列出所有记忆
python src/skill.py list

# 查看统计
python src/skill.py stats
```

## 设计理念

TagMemory 的核心理念是**人类参与记忆维护**：

```
┌─────────────────────────────────────────────┐
│              人类审核机制                      │
│                                              │
│  存储 ──→ 闲时核对 ──→ 定期归纳 ──→ 确认存档  │
│    │          │            │           │     │
│    │          │            │           │     │
│    ↓          ↓            ↓           ↓     │
│  打标签    纠正错误      生成总结      用户确认 │
└─────────────────────────────────────────────┘
```

**为什么这样做？**

1. AI 自动存储的记忆可能有错误
2. 没有人类审核，错误会累积
3. 定期归纳 + 确认 = 确保记忆准确
4. 闲时核对 = 发现错误及时纠正

## 标签说明

| 标签 | 用途 | 示例 |
|------|------|------|
| `#偏好` | 用户偏好习惯 | "喜欢 tabs 缩进" |
| `#决定` | 重要决策 | "选择了 PostgreSQL" |
| `#项目` | 项目背景 | "在做电商项目" |
| `#人` | 人物信息 | "项目经理是张三" |
| `#事件` | 发生的事情 | "上周系统宕机了" |
| `#知识` | 学到的知识 | "学会了用 Docker" |
| `#错误` | 犯过的错误 | "之前选错了方案" |
| `#总结` | 归档的总结 | 自动添加 |
