# Competitive Analysis Framework

## Why Competitive Analysis Drives ASO

Your app doesn't rank in a vacuum. The App Store and Google Play rank apps relative to each other. Understanding what competitors do well (and where they're weak) is how you find the gaps that translate directly into ranking opportunities and conversion improvements.

## Step 1: Define Your Competitive Set

### Tiers of Competition

**Tier 1 — Direct competitors** (same function, same audience)
- Apps users would consider instead of yours
- Usually 3-7 apps; these are your primary benchmarks

**Tier 2 — Indirect competitors** (different approach, same problem)
- Solve the same user problem differently
- Important for keyword gap analysis

**Tier 3 — Category neighbors** (same category, different function)
- Share keywords and browse surface with you
- Monitor for keyword cannibalization

### How to Find Competitors
1. Search your primary keywords — the top 10 results are your main competitors
2. Check "You Might Also Like" / "Similar Apps" on your own listing
3. Ask: what app would your target user download instead of yours?
4. Check your reviews — users often mention switching from or comparing to specific apps

## Step 2: Metadata Audit

For each Tier 1 competitor, document:

| Field | Their Current Value | Character Count | Keywords Used |
|-------|-------------------|-----------------|---------------|
| App Name | | | |
| Subtitle (iOS) | | | |
| Short Description (Android) | | | |
| Keyword Field (iOS, if visible) | | | |
| First line of description | | | |

Look for:
- Which keywords appear in EVERY competitor's title? (table stakes — you need them too)
- Which keywords appear in SOME competitors? (opportunity to differentiate or follow)
- What's completely absent? (potential gap)

## Step 3: Keyword Gap Matrix

Build this matrix for your top 5-10 priority keywords:

```
Keyword: [keyword]
Your rank: [#]

Competitor A: [rank or "not ranking"]
Competitor B: [rank or "not ranking"]
Competitor C: [rank or "not ranking"]

Assessment: [gap / table stakes / your advantage]
```

**Gap types:**
- **You're missing, they have it**: Add this keyword to your strategy
- **You both rank, they're higher**: Improve content quality / authority signals
- **You rank, they don't**: Defend this position; it's your advantage
- **Nobody ranks well**: Potentially underexploited niche

## Step 4: Visual Analysis

### Icon Comparison
Screenshot all top 10 icons in your category. Analyze:
- Dominant colors (what's saturated vs. missing?)
- Style (illustrative, photographic, geometric, typographic?)
- Complexity level
- What would stand out vs. blend in?

### Screenshot Strategy Decode
For each top competitor:
- What's the first screenshot promise? (the headline benefit)
- What sequence do they follow?
- What's their visual tone (dark/light, lifestyle/product, minimal/detailed)?
- What do they NOT show? (potential opportunity)

### Conversion Clues
If a competitor has:
- Higher rating with more reviews → better user satisfaction or better review prompting
- Older app, lower visual quality → chance to win on modern design
- Heavy text in screenshots → opportunity to win with cleaner visuals

## Step 5: Rating & Review Analysis

### What to Extract from Competitor Reviews

**1-2 star reviews reveal:**
- What users hate most (pain points you can solve / avoid)
- Features competitors are missing
- Promises competitors make but don't deliver

**4-5 star reviews reveal:**
- What users love (validate these features in your own messaging)
- Language users actually use (steal this for your description copy)
- Use cases you might not have considered

### How to Use This
- Mine competitor 1-star reviews for your "we solved this" marketing angle
- Copy the exact words happy users use about competitors — these are the keywords and phrases your audience resonates with
- If a competitor has hundreds of reviews saying "wish it had X", build X and make it prominent in your screenshots

## Step 6: Update Velocity Analysis

Track competitor updates:
- How often do they update screenshots and metadata?
- Do they run seasonal campaigns?
- Do their rankings improve after updates?

This tells you:
- Whether they're actively A/B testing (frequent small changes = testing)
- Whether you're in a static or dynamic competitive environment
- What triggers ranking changes in your category

## Competitive Analysis Output Template

```markdown
## Competitive Analysis: [Your App] vs. Market

### Competitive Set
- Direct: [App A], [App B], [App C]
- Indirect: [App D], [App E]

### Keyword Opportunity Map
| Keyword | Your Rank | Best Competitor | Gap | Action |
|---------|-----------|-----------------|-----|--------|
| [kw1]   | 15        | App A (#3)      | -12 | Add to title |
| [kw2]   | —         | App B (#8)      | —   | Add to keywords |
| [kw3]   | 4         | App A (#7)      | +3  | Defend |

### Visual Positioning Gap
[App A] dominates dark/minimal aesthetic. [App B] uses lifestyle photography.
Opportunity: Clean, product-focused, light theme is underrepresented.

### Review Mining Insights
Top complaint in competitor reviews: "[quote]" — opportunity to solve this and call it out
Top praise phrase: "[quote]" — use this language in our description

### Recommended Positioning Shift
[1-2 sentence strategic recommendation based on the analysis]
```

## Tools for Competitive Research

If you have access to ASO tools, here's what to use for what:
- **AppTweak / AppFollow**: Keyword ranking history, competitor keyword sets
- **Sensor Tower / data.ai**: Download estimates, revenue estimates, category rankings
- **MobileAction**: Ad intelligence, which competitors are running search ads on which keywords
- **Manual research**: App Store search + screenshots (free, often sufficient)

If no tools are available, manual research covers 80% of what you need for most apps.
