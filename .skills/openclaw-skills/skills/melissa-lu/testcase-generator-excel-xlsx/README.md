# testcase-generator

测试用例生成器 - 根据PRD自动生成和更新测试用例

## 目录结构

```
testcase-generator/
├── SKILL.md                          # 主文档（完整流程说明）
├── README.md                         # 本文件
├── references/
│   └── uhost-testcase-style.md       # UHost用例风格规范参考
├── scripts/
│   ├── version_manager.py            # 用例库版本管理
│   └── case_generator.py             # 用例生成工具
└── templates/
    └── (暂无，未来可扩展)
```

## 快速开始

### 1. 检查用例库版本


```
[阶段1] 解析PRD...

```

## 核心功能

### 📚 用例库版本管理
- 检查当前用例库版本（用例数、最后更新时间）
- 支持增量更新（按UUID覆盖）
- 自动保存元数据

### 📖 PRD解析
- 支持 PDF、Word、Markdown
- 提取核心功能点和变更详情
- 生成 PRD 摘要

### 🎯 测试点分析
- 按模块分类测试场景
- 识别测试类型（CRUD、边界、异常等）
- 生成测试点清单

### 📊 用例库对比
- 搜索相关现有用例
- 判断：可复用 vs 需新增
- 生成对比分析报告

### ✍️ 用例生成
- **新增用例**: 12列标准格式，用例ID留空，模块路径加 /ai 前缀
- **更新用例**: 17列（原12列 + 新5列）
- 严格遵循 UHost 用例库格式规范

## 格式规范（关键点）

### 新增用例特殊规则
1. **用例ID**: 留空
2. **所属模块**: 加 /ai 前缀
   - 例: `/ai/镜像常规/镜像常规/功能/自制镜像tab下/导入镜像`

### 更新用例特殊规则
1. **保留原12列**: 完整信息
2. **新增5列**: 更新类型、更新位置、原内容、更新后内容、更新原因

### 通用格式规则
- 步骤描述：【1】【2】【3】...
- 预期结果：【1】【2】【3】...（与步骤对应）
- 前置条件：1. 2. 3. ...

## 用例库配置

```python
# scripts/version_manager.py 中的配置

    '主机用例库': '/Users/user/testdata/脚本任务/测试用例_UHost.xlsx',
    '镜像用例库': '/Users/user/Downloads/测试用例_UHost (1).xlsx'
}
```

## 使用建议

1. **每次执行前先检查用例库版本**
2. **关键节点需要人确认**（测试点分析、用例对比）
3. **优先更新现有用例**，避免重复
4. **严格遵循格式规范**，确保质量
