# Idea / Demand Check

适用：点子判断、需求验证、创业方向梳理、值不值得做。

核心目标：
- 说清是谁痛
- 说清现在怎么凑合
- 说清为什么现在值得做
- 说清最小可卖/可用切口

关键追问：
1. 谁最痛？要具体到人或角色。
2. 他们现在怎么解决？成本是什么？
3. 最强证据是什么？兴趣不算，行为才算。
4. 最小版本是什么？一周内能交付什么价值？
5. 最大风险是什么？

常见红旗：
- 用户是“所有人”
- 把 waitlist/点赞 当需求
- 没有现有替代方案
- 一上来就想全平台

标准输出：
- Problem
- Real user
- Current workaround
- Evidence
- Narrow wedge
- Biggest risk
- Recommendation
- Next 24-72h move
