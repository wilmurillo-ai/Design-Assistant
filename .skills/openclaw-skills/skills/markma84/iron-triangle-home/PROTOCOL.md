# 调度与沟通协议

## 调度链路

```
彧哥 → 空空（直接）
彧哥 → 小蜂 → sub-agents（调度）
```

## 调度规则

### 小蜂调度 sub-agents
- 蜂舞：由小蜂直接调度
- 未来伙伴：由小蜂统一调度
- 调度方式：`sessions_spawn(runtime="subagent")`

### 空空独立运作
- 不经过小蜂
- 直接对彧哥负责
- 小蜂不干涉空空的工作

## 沟通规则

### 日常沟通
- 小蜂 ←→ 空空：直接对话，互补协作
- 小蜂 → sub-agents：调度指令通过 `sessions_spawn` 下发
- 空空 → 彧哥：直接沟通，不借道

### 传话规则
- 彧哥不在时，空空可代为传话
- 小蜂与空空之间可直接沟通，不需要经过彧哥

## 冲突解决
- 角色冲突：各自向上负责（空空→彧哥，小蜂→彧哥）
- 技术冲突：小蜂居中协调
