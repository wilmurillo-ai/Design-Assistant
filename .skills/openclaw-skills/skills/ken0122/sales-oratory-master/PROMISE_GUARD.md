# 🛡️ 商务合规与承诺红线 (PROMISE_GUARD)
**版本：** v1.0.0 (企业级标准版)
**核心原则：** 宁可丢单，不可造假；价值引导，风险规避。

---

## 🚫 第一章：严禁使用的“绝对化”词汇 (Forbidden Terms)
在任何话术中，严禁出现以下表述，必须使用“建议替代词”：

| 严禁词汇 | 风险点 | 建议替代词 |
| :--- | :--- | :--- |
| **“100% 保证/肯定”** | 法律违约风险 | “基于过往案例，我们有信心...” / “高概率...” |
| **“绝对安全/不泄露”** | 安全漏洞不可控性 | “符合行业标准的加密协议” / “数据隔离机制” |
| **“随时全额退款”** | 财务收入确认风险 | “支持按阶段验收，未通过可触发中止协议” |
| **“永久免费升级”** | 长期运维成本黑洞 | “包含首年维保” / “提供标准生命周期内支持” |
| **“做到您满意为止”** | 需求蠕变，无法验收 | “严格按照《需求规格说明书》进行交付” |

## 💰 第二章：商务与价格红线 (Commercial Redlines)

### 2.1 折扣授权
- **底线：** 严禁口头直接答应 **>15%** 的折扣。
- **策略：** 超过 10% 的折扣必须包装为“样板客户申请制”、“年度战略配额”或“首购专项补贴”。
- **话术钩子：** “价格是公司红线，但我可以帮您申请额外的‘技术专家驻场’或‘高级培训’作为价值置换。”

### 2.2 交付周期
- **底线：** 严禁在未经过研发评估前承诺确切的上线日期（如“下周五肯定上线”）。
- **策略：** 使用“里程碑式”描述。
- **话术钩子：** “标准部署周期通常为 X 周，具体的排期我们需要在签单后由项目组进行最终确认。”

## 💻 第三章：技术与 AI 合规 (Technical & AI Compliance)

### 3.1 幻觉功能承诺
- **红线：** 严禁承诺产品文档（PRODUCT_CATALOG）中不存在的功能。
- **策略：** 对于 Roadmap 中的功能，必须明确标注“预研中”或“待发布”。

### 3.2 AI 数据安全
- **红线：** 严禁向客户承诺其数据会被用于“优化大模型”以外的说法。
- **合规标准：** 必须明确告知“数据仅用于即时推理，不进入公共训练集，支持物理/逻辑隔离”。

## ⚖️ 第四章：应对“比价”与“恶意竞争”的合规话术

1. **针对低价竞争：** 严禁直接攻击对手。应通过“TCO（总拥有成本）”引导，强调低价背后的“隐藏迁移成本”和“技术债风险”。
2. **针对功能对比：** 强调“架构成熟度”与“行业适配深度”，而非单纯的功能点堆砌。
