## 📋 技能描述

**面向场景**: 从问题定义到 PRD 输出的端到端流程  
**目标用户**: 产品经理、产品负责人  
**核心价值**: 整合全部 11 个需求分析技能，输出完整 PRD

---

## 🎯 工作流阶段

### 阶段 1: 问题定义 (20 分钟)
1. 运行 `problem-statement`
2. 运行 `proto-persona`
3. **输出**: 问题定义 + 用户画像

### 阶段 2: 需求探索 (60 分钟)
1. 运行 `customer-journey-map` 或 `jobs-to-be-done`
2. 运行 `problem-framing-canvas`
3. 运行 `company-research`
4. **输出**: 需求分析报告

### 阶段 3: 方案设计 (60 分钟)
1. 运行 `opportunity-solution-tree`
2. 运行 `prioritization-advisor`
3. 运行 `feature-investment-advisor`
4. **输出**: 方案选项 + 优先级 + ROI

### 阶段 4: 需求输出 (60 分钟)
1. 运行 `user-story`（P0 需求）
2. 运行 `epic-hypothesis`（大型功能）
3. 运行 `finance-based-pricing-advisor`（定价相关）
4. **输出**: 用户故事 + 史诗假设 + 定价策略

### 阶段 5: PRD 整合 (40 分钟)
1. 运行 `prd-development`
2. 整合所有输出
3. **输出**: 完整 PRD 文档
