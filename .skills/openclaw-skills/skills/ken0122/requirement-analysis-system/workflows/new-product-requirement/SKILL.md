## 📋 技能描述

**面向场景**: 从 0 到 1 的新产品需求分析  
**目标用户**: 产品负责人、创业团队、创新项目负责人  
**核心价值**: 系统化完成新产品需求分析，避免遗漏关键环节

---

## 🎯 工作流阶段

### 阶段 1: 问题定义 (20 分钟)
1. 运行 `problem-statement` — 明确核心问题
2. 运行 `proto-persona` — 定义目标用户
3. **输出**: 问题定义 + 用户画像

### 阶段 2: 需求探索 (40 分钟)
1. 运行 `jobs-to-be-done` — 深度挖掘需求
2. 运行 `customer-journey-map` — 绘制用户旅程
3. **输出**: 需求分析报告

### 阶段 3: 方案设计 (40 分钟)
1. 运行 `opportunity-solution-tree` — 生成方案
2. 运行 `prioritization-advisor` — 优先级排序
3. **输出**: 方案选项 + 优先级

### 阶段 4: 商业验证 (30 分钟)
1. 运行 `company-research` — 市场调研
2. 运行 `feature-investment-advisor` — ROI 评估
3. **输出**: 商业可行性分析

### 阶段 5: PRD 整合 (40 分钟)
1. 运行 `epic-hypothesis` — 定义关键假设
2. 运行 `prd-development` — 整合 PRD
3. **输出**: 完整 PRD 文档

## 🔧 使用方式

```markdown
/new-product-requirement
```

**预计耗时**: 2.5-3 小时  
**输出**: 完整 PRD + 附件（用户画像/旅程图/优先级清单）
