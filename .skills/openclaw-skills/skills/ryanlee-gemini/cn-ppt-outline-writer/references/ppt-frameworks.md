### 模板 1：年终述职报告（15-20分钟，20页）

**逻辑框架：** 时间线 + 金字塔

```
第1页  封面
       标题：[姓名] 2024年度述职报告
       副标题：[岗位] | [部门] | [日期]

第2页  目录
       1. 年度工作回顾
       2. 核心成果亮点
       3. 问题与反思
       4. 2025年规划

第3页  [章节标题页] 一、年度工作回顾

第4页  工作全景图
       要点：完成X个项目 | 覆盖X个业务线 | 服务X个团队
       视觉：数字卡片 3格

第5页  重点项目 A（最重要的）
       要点：背景/目标 → 行动 → 结果（数据）
       视觉：流程图 + 数字

第6页  重点项目 B
       （同上结构）

第7页  重点项目 C
       （同上结构）

第8页  [章节标题页] 二、核心成果亮点

第9页  关键 KPI 完成情况
       要点：3-5个核心指标达成率（红绿灯）
       视觉：数据表格/进度条

第10页 超预期的成果（亮点）
       要点：列出2-3件超出预期的事
       视觉：√ 检查清单

第11页 团队协作贡献
       要点：跨部门协作案例/帮助他人的贡献
       视觉：关系连线图

第12页 [章节标题页] 三、问题与反思

第13页 遇到的挑战（3个）
       要点：挑战描述 → 应对 → 学到了什么
       视觉：3列卡片

第14页 能力成长盘点
       要点：新学了什么技能/方法
       视觉：能力雷达图（对比今年vs去年）

第15页 如果重来，会做什么不同
       要点：2-3个改进点（展示成熟度）
       视觉：文字为主，简洁

第16页 [章节标题页] 四、2025年规划

第17页 2025年目标
       要点：3个核心目标（SMART格式）
       视觉：OKR 表格

第18页 实施路径
       要点：每个目标的关键里程碑
       视觉：甘特图/时间线

第19页 需要的支持
       要点：资源/授权/跨部门协作需求
       视觉：清单

第20页 结尾
       一句话：感谢 + 核心承诺
       视觉：全屏图片 + 一句话
```

### 模板 2：项目汇报（10分钟，12页）

**逻辑框架：** 结论先行（金字塔）

```
第1页  封面
       [项目名称] 进展汇报 | [日期] | 汇报人

第2页  核心结论（1分钟）
       结论：[项目当前状态：正常/有风险/需决策]
       3个支撑：进度/质量/风险各一句话

第3页  项目背景（快速对齐）
       目标/范围/关键交付物 → 30秒带过

第4页  进度总览
       里程碑时间线：已完成 ✅ / 进行中 🔄 / 待开始 ⬜
       视觉：时间线

第5页  本期重点进展（最重要的成果）
       3个主要完成事项 + 数据

第6页  本期重点进展（续）
       （如有更多）

第7页  问题与风险
       风险矩阵：高/中/低优先级
       每项：风险描述 → 影响 → 应对措施

第8页  需要决策/支持的事项
       清单：每项说明"需要谁在何时给出什么决定"

第9页  下阶段计划（2周/1个月）
       关键任务 + 负责人 + 截止日期
       视觉：表格

第10页 资源/预算状态
       预算使用进度 / 人力投入情况

第11页 总结
       3句话：当前状态 + 最大风险 + 下一步行动

第12页 附录（可选）
       详细数据/支撑材料
```

## 二、金字塔原理应用示例

### 核心原则

> 「先说结论，再给理由，理由由数据支撑。」

### 标准结构（适用于向上汇报）

```
【结论层】 我们建议：Q3 重点投入短视频渠道
    │
    ├── 【论点1】 短视频 ROI 最高
    │       ├── 数据：短视频获客成本 ¥12，图文获客成本 ¥45
    │       └── 数据：短视频用户留存率高 23%
    │
    ├── 【论点2】 竞品已验证可行
    │       ├── 案例：XX品牌 Q1 通过短视频增长 300%
    │       └── 案例：YY品牌短视频占新增用户 60%
    │
    └── 【论点3】 我们有执行基础
            ├── 现有：视频团队 5 人，月均产出 20 条
            └── 差距：需补充达人合作资源
```

### 常见错误 & 修正

| 错误写法 | 金字塔写法 |
|---------|---------|
| "我们做了很多工作……最终结果是……" | "结果是X，因为我们做了A/B/C" |
| 按时间顺序汇报 | 按重要性排列，最重要的先说 |
| 标题用动词"分析、研究" | 标题用结论句"销售额提升 20%" |
| 用"此外"、"另外"罗列 | 用"第一/第二/第三"体现逻辑顺序 |

## 三、SCQA 框架示例

### 应用示例：向管理层汇报数据安全风险

**S（情境）：** 公司过去两年用户数据存储在自建服务器，业务运营正常。

**C（冲突）：** 《数据安全法》实施后，现有存储方案在3个维度不合规，且行业已出现2起罚款案例（最高 500 万元）。

**Q（问题）：** 我们是否需要在 Q4 前完成合规改造？代价是多少？

**A（答案）：** 建议分两阶段完成合规：
- 阶段一（2个月）：完成数据分类分级和权限审计，成本约 50 万
- 阶段二（4个月）：迁移至符合等保三级的云存储，成本约 200 万
- 不改造的罚款风险：最高 500 万 + 业务暂停风险

---

### SCQA 快速模板

```
情境（S）：[大家都认同的现状，1-2句]
冲突（C）：但是，[发生了什么变化/什么问题出现了]
问题（Q）：那么，[我们面临的核心决策/问题是什么]？
答案（A）：我们的建议是 [核心结论]，因为 [1-3个理由]。
```

### 适用场景

- ✅ 向决策层汇报问题并寻求资源支持
- ✅ 商业提案中建立紧迫感
- ✅ 改变现状/推动变革的方案汇报
- ❌ 日常进展汇报（用时间线更清晰）
- ❌ 培训分享（用能力递进框架）

## 四、PPT 制作 Tips

### 每页结构清单

- [ ] 标题是否是结论句而非标题词？（"销售额提升20%" > "销售额分析"）
- [ ] 正文是否 ≤3 个要点？
- [ ] 有没有数字/案例支撑每个要点？
- [ ] 视觉元素是否辅助内容而非装饰？
- [ ] 单页文字是否 ≤80 字？

### 颜色使用

- 主色：公司品牌色（1种）
- 辅色：强调/对比（1种）
- 背景：白色或浅灰（不要深色背景+浅色字）
- 警示：红色仅用于问题/风险

### 字体规范

- 标题：24-32pt，加粗
- 正文：16-20pt，常规
- 注释：12-14pt，灰色
- 全程只用 1-2 种字体（中文：思源黑体/微软雅黑；英文：Arial/Calibri）
