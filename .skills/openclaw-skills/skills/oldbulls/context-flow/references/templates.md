# Context Governance Templates

## Group collaboration template

Replace role labels with your local agent names if needed.

```md
分工安排：
- @controller：负责任务编排与收口
- @worker：负责当前执行任务

当前状态：
- @controller：已派发
- @worker：处理中

协作摘要：
目标：一句话说明这轮要做什么
边界：一句话说明不要做什么
回报：完成后先回 controller，不直接发散
```

## Child agent task card

```md
任务：
- 这轮只做什么

目标：
- 最终要产出什么

范围：
- 只允许处理哪些文件 / 哪些对象 / 哪一部分

必读材料：
- 当前任务直接相关的文档
- 当前阶段需要的稳定背景材料

已知约束：
- 约束1
- 约束2
- 约束3

不要做：
- 不要扩展范围
- 不要改无关文件
- 不要处理历史待办

输出格式：
- 文件清单
- 修改建议
- 风险点
```

## Subagent local task card

```md
你现在只处理一个局部问题。

局部任务：
- 只检查 / 只实现 / 只验证这一个点

背景最小化：
- 当前主目标是 X
- 你这里只负责 Y
- 不需要处理 Z

输入：
- 文件 / 文档 / 目录

判断标准：
- 什么算完成
- 什么算异常

输出：
- 只返回结论 + 证据 + 下一步建议
```

## Controller sync and acceptance templates

### Receipt timeout / pending verification

```md
分工安排：
- @智能管家：负责核验目标侧证据并统一收口
- @执行 agent：负责目标任务执行

当前状态：
- @智能管家：处理中
- @执行 agent：已派发

协作摘要：
当前出现回执超时，不能直接判定执行失败。
已进入待核验状态，正在检查目标侧执行证据。
未核验前，相关外部写操作不重复补发。
```

### Partially complete

```md
分工安排：
- @智能管家：负责继续补治理闭环并统一收口
- @执行 agent：负责已完成主线任务

当前状态：
- @智能管家：处理中
- @执行 agent：已完成

协作摘要：
主线任务已完成，但治理闭环尚未结束。
当前缺口可能是记忆未落盘、实体依赖未同步、索引未补或二次验收未完成。
本轮状态应按“部分完成”处理，不直接宣称全部完成。
```

### Rule-update secondary acceptance task card

```md
你是 [agent 名称]。请执行一次“规则更新后二次验收”自检，只回答结果，不要长解释。

请检查并回报：
1. 你是否已理解本次新增 / 更新规则
2. 你是否仅在当前会话中知晓，还是已写入长期可复用记忆
3. 你是否已同步实体、依赖关系或等价结构化知识
4. 若还未落盘，请直接补写，再返回写入位置
5. 你是否还需要主控继续验收

输出格式：
- A. 配置状态（如适用）
- B. 记忆状态
- C. 实体与依赖状态
- D. 本次补写位置
- E. 是否需要主控再验收
```

## Minimum evidence rules

- `执行已确认` 至少要有对象 ID / URL / messageId / 第三方对象主键，或目标侧成功结果且对象可唯一定位。
- `禁止补派` 必须在 `执行已确认` 成立且已记录去重锚点后才能使用。
- `部分完成` 只适用于主线完成但治理缺口未闭环的情况，不能拿来包装未完成主线。
- `已完成` 必须同时满足：执行结果已确认、必要证据已取得、当前轮要求的收口项已完成。
- `失败` 不能因为“没回我”直接判定，必须经过核验与允许范围内的补派 / 降级尝试。

- ...

- ...

- ...
- ...

## Feishu group examples

### Example: real collaboration handoff in a Feishu group

```md
分工安排：
- @方案agent：整理可执行方案和边界
- @执行agent：收口、执行、统一对外同步

当前状态：
- @方案agent：已派发
- @执行agent：处理中

协作摘要：
目标：先把方案范围收窄，再由主控 agent 收口执行。
边界：群里不贴长日志，不把完整执行上下文贴进协作群。
回报：方案 agent 完成后先回主控 agent，再决定是否同步到群里。
```

### Example: only one agent is working, so do not fake collaboration

```md
不要输出：
- 分工安排：
- 当前状态：
- 协作摘要：

直接输出普通执行进展，例如：
- 已确认需要更新当前任务相关文档
- 我先处理这部分，改完再给你收口结论
```

### Example: the Feishu thread is getting noisy and needs a reset

```md
协作摘要：
当前以后只以这一版状态为准。

已完成：
- 已确认当前主线和边界

处理中：
- 正在处理当前阶段的核心文档或对象

未开始：
- 下一阶段的复核或扩展项

下一步：
- 先完成当前阶段，再决定是否进入下一阶段

说明：
旧讨论不再作为当前执行依据。
```

### Example: sending a compact child-agent task after a Feishu group handoff

```md
任务：
- 只检查当前指定文档或对象是否需要更新

目标：
- 给出应更新点，不扩写无关内容

范围：
- 只看当前任务明确指定的文件、文档或对象

已知约束：
- 不改无关规则文件
- 不处理历史待办
- 输出要短

不要做：
- 不要扫整个工作区
- 不要把群聊天全文重发进任务

输出格式：
- 对象
- 应补内容
- 风险点
```
