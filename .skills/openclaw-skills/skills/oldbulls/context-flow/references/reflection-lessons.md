# Reflection Lessons

## Purpose

This file captures durable reflection patterns that help turn messy collaboration into reusable operating rules.

## Core lessons

- Repeated coordination mistakes should be promoted into stable rules
- Long process notes should not remain the main entry point once a shorter rule exists
- A useful reflection result is compact, actionable, and easy to reuse
- If a collaboration pattern keeps recurring, move it into templates or anti-patterns
- Governance docs should stay thinner than the process history that produced them

## Preferred distillation pattern

1. Keep the raw process record for audit
2. Extract the stable lesson
3. Promote the lesson into rules, templates, or anti-patterns
4. Stop using the long process note as the main reference

## Practical rule

When a lesson is stable enough to guide future behavior, store it in the skill references or local operating rules rather than relying on historical memory alone.
