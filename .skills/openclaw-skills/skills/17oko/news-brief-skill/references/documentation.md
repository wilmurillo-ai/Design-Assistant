## 功能概述

News Brief Skill 是一个智能新闻简报助手，能够：

- **个性化定制**: 根据用户兴趣关键词自动筛选相关新闻
- **多源聚合**: 集成API、爬虫、RSS三种获取方式
- **智能摘要**: 自动生成50-80字的新闻摘要
- **多渠道推送**: 支持微信、邮件、钉钉、Telegram等渠道
- **持续优化**: 用户反馈驱动的智能优化机制
- **自动维护**: 渠道自动检测和工具动态调整

## 安装配置

### 系统要求
- Python 3.8+
- OpenClaw v2.0+
- Windows/Linux/MacOS

### 依赖安装
```bash
pip install requests beautifulsoup4 cryptography feedparser python-dateutil lxml
```

### 初始设置
1. 将skill目录复制到OpenClaw skills目录
2. 首次使用时会引导用户进行初始配置
3. 可选：配置API密钥（如NewsAPI、邮件SMTP等）

## 使用指南

### 首次使用
1. 发送"新闻简报"或"每日新闻"激活技能
2. 按照引导设置：
   - 兴趣领域（科技/财经/时政等）
   - 兴趣关键词（可选）
   - 推送渠道和时间
   - 简报格式偏好

### 日常使用
- **自动推送**: 每日按设定时间自动推送简报
- **查看详情**: 回复新闻序号查看详细信息
- **优化设置**: 回复"反馈"或"优化"进入优化流程
- **手动触发**: 随时发送"新闻简报"获取最新内容

### 优化流程
每次推送后会自动询问优化需求：
- **回复1**: 优化内容（关键词、摘要详略）
- **回复2**: 优化格式（排版、条数）
- **回复3**: 调整推送渠道/时间
- **回复0**: 无需调整

## 模块详解

### 1. 交互模块 (Interaction Module)
- **功能**: 处理用户输入，提供反馈选项
- **特点**: 支持关键词触发，处理模糊输入
- **话术**: 标准化的交互提示

### 2. 优化模块 (Optimization Module)
- **功能**: 执行用户优化请求
- **类型**: 内容优化、格式优化、渠道优化
- **机制**: 预览确认 + 版本回溯

### 3. 推送模块 (Push Module)
- **功能**: 多渠道智能推送
- **特性**: 主备渠道切换，状态反馈
- **安全**: 敏感信息加密存储

### 4. 工具模块 (Tools Module)
- **功能**: 管理新闻获取工具
- **类型**: API接口、网页爬虫、RSS订阅
- **智能**: 动态优先级调整

### 5. 新闻获取模块 (News Sources Module)
- **功能**: 渠道管理和新闻抓取
- **特性**: 自动更新，智能去重
- **应急**: 备用获取方案

### 6. 摘要生成模块 (Summary Module)
- **功能**: 生成标准化新闻摘要
- **规范**: 标题→摘要→来源→可信度
- **灵活**: 支持多种排版样式

## 配置说明

### 用户配置文件结构
```json
{
  "user_id": "wx_123456",
  "domains": ["tech", "finance"],
  "interest_keywords": ["AI", "芯片", "大模型"],
  "primary_channel": "wechat",
  "backup_channels": ["email"],
  "push_time": "08:00",
  "frequency": "daily",
  "max_items": 5,
  "summary_style": "standard",
  "show_credibility": true,
  "email_user": "encrypted_data",
  "email_password": "encrypted_data"
}
```

### 工具配置 (config/tools.json)
- **api_interfaces**: API接口配置
- **web_crawlers**: 网页爬虫配置  
- **rss_feeds**: RSS订阅配置
- **优先级**: 数字越小优先级越高

### 新闻源配置 (config/sources.json)
- **按领域分类**: tech, finance, politics等
- **主备域名**: 自动切换失效域名
- **状态管理**: active/inactive状态

### 模板配置 (config/templates/)
- **brief_template.txt**: 简报整体模板
- **news_item_template.txt**: 单条新闻模板

## 故障排除

### 常见问题

#### 1. 无法获取新闻
- **原因**: 渠道失效或网络问题
- **解决**: 系统会自动切换备用渠道，也可手动更新渠道配置

#### 2. 推送失败
- **原因**: 渠道配置错误或服务不可用
- **解决**: 检查渠道配置，系统会自动切换备用渠道

#### 3. 摘要质量差
- **原因**: 内容源质量或关键词不匹配
- **解决**: 调整兴趣关键词，优化摘要模式

### 日志位置
- **新闻获取日志**: `data/logs/news_fetch_YYYYMMDD.log`
- **推送日志**: `data/logs/push_log_YYYYMMDD.log`  
- **用户配置备份**: `data/backups/{user_id}/`

### 恢复配置
- **查看备份**: `data/backups/{user_id}/`
- **恢复命令**: 通过管理界面选择历史版本恢复

## 安全隐私

### 数据加密
- **敏感信息**: 使用AES-256加密存储
- **加密密钥**: 存储在`config/encryption.key`
- **传输安全**: HTTPS加密通信

### 隐私保护
- **不收集**: 用户身份信息、聊天记录
- **仅存储**: 必要的推送配置和偏好设置
- **定期清理**: 自动清理过期日志

### 权限控制
- **用户隔离**: 每个用户独立配置目录
- **访问控制**: 基于用户ID的权限验证
- **数据安全**: 配置文件权限限制

## 扩展开发

### 新增推送渠道
1. 在`pusher.py`中添加新的推送函数
2. 在`channels`字典中注册新渠道
3. 更新用户配置模板

### 新增新闻源
1. 在`sources.json`中添加新源配置
2. 在`tools.json`中配置对应的爬虫
3. 测试源的可用性

### 自定义摘要算法
1. 修改`summarizer.py`中的摘要生成逻辑
2. 可集成第三方AI摘要API
3. 支持多语言摘要
