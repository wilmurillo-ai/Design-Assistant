# 元素定位策略（源码优先，图像兜底）

## 推荐优先级
1. `getByRole(...)` + 稳定可访问名称（accessible name）
2. `getByLabel(...)`、`getByPlaceholder(...)`、`getByText(...)`（仅稳定文本）
3. 稳定 `data-testid` / `data-qa`
4. 简短且稳定的 CSS 选择器
5. XPath（最后手段）

## 源码优先策略
- 先看组件/模板源码，找稳定定位锚点。
- 优先语义化与可访问性属性，避免纯视觉 class。
- 避免使用构建产物或 CSS-in-JS 生成的动态类名。

## 截图/图像识别兜底策略
当源码或 DOM 钩子不明确时：

1. 截取当前页面状态截图。
2. 识别唯一视觉锚点（按钮文案、图标、相邻标签等）。
3. 将视觉锚点反向映射为可维护 DOM 定位（role/name/testid）。
4. 若只能坐标点击，必须标注为脆弱方案，并加 TODO 尽快替换。

## 防抖/防脆弱规则（Anti-flake）
- 能用状态断言等待就不用固定 `sleep`。
- 定位器尽量限定在稳定父容器范围内。
- 跳转/重渲染后重新查询元素。
- 仅对已知瞬时网络问题增加有限重试。
