# Workout Timer App 集成说明

> Created: 2026-04-12
> 项目地址: gitee.com/kaiji1126/workout-timer
> 技术栈: Flutter (Dart), SQLite

## 概述

Workout Timer 是一款健身计时+训练记录App，支持Android/iOS。部分用户使用此App记录训练数据，并通过App的「AI分析」功能生成数据报告发送给AI。

**⚠️ 兼容性原则**：本技能同时服务使用和不使用此App的用户。仅在用户发送的数据包含App特征（如下方格式）时，才按此文档处理。

## 如何识别App生成的数据

用户从App「AI分析」功能复制的数据报告通常包含以下特征：
- 标题包含「训练数据报告」
- 包含「训练密度」「肌群容量分布」「恢复状态」「常用动作TOP10」等App专属指标
- 包含「用户画像」和「训练计划规则」段落
- 结尾要求输出JSON格式的训练计划

## App数据体系

### 训练容量（Volume）
- 计算公式: **组 × 次 × 重量**（sets × reps × weight）
- 这是所有分析的基石指标
- 等长收缩动作（如Plank）无法计算容量，不建议安排在计划中

### 肌肉分类
- **6大主肌群**: chest, back, shoulders, arms, legs, core
- **18个子肌群**: 上胸/中胸/下胸、背阔肌/上背/菱形肌/下背、前束/中束/后束、二头/三头/前臂、股四头/腘绳/臀肌/小腿、腹直肌/腹外斜肌
- 容量分配权重: 主肌群子分类占70%，次要肌群占30%

### 统计指标说明

| 指标 | 含义 | 用途 |
|------|------|------|
| 总训练量 | 所有动作组×次×重量之和 | 衡量整体训练负荷 |
| 训练密度 | 组数/训练时长(分钟) | 衡量训练效率 |
| 肌群容量分布 | 各肌群训练量及占比 | 发现不平衡 |
| PR记录 | 每个动作历史最大重量 | 安排重量区间参考 |
| 1RM估算 | Epley公式: weight × (1 + reps/30) | 力量评估 |
| 恢复状态 | 各肌群距上次训练天数 | 判断是否可训练 |
| 趋势变化 | vs上一周期的各指标变化 | 判断进步/退步 |
| 力量突破 | 本周期新的1RM记录 | 鼓励+调整参考 |
| 肌群不平衡 | 上肢:下肢比、胸:背比等 | 计划调整依据 |

### 数据报告各段落含义

1. **基本信息**: 周期、训练次数、总训练量、密度、平均每次
2. **趋势变化**: vs上周/上月的总量和各肌群变化（百分比+箭头）
3. **肌肉容量分布**: 各肌群训练量绝对值+占比%，按量排序
4. **PR记录**: 各动作历史最大重量（英文名），取top10
5. **恢复状态**: 各肌群已休息天数 + ✅可训练/⚠️建议休息/🔴今日刚训练
6. **常用动作TOP10**: 按训练次数排序的动作列表
7. **训练洞察**: 自动检测的力量突破、薄弱部位、肌群不平衡、过度训练风险
8. **用户画像**: 目标、经验、频率、设备、重点肌群
9. **训练计划规则**: 根据目标动态生成（增肌/减脂/力量/耐力各有不同规则）
10. **输出格式要求**: 纯JSON

## 基于App数据的计划生成

当收到App数据报告时，生成计划应：

1. **分析趋势**：重点关注容量下降的肌群（需要调整）和持续增长的肌群（保持）
2. **参考PR**：用PR数据安排重量区间，确保渐进超负荷
3. **尊重恢复状态**：⚠️的肌群不要安排在当天
4. **利用洞察**：肌群不平衡、薄弱部位直接影响动作选择
5. **遵循用户画像中的规则**：目标、经验、频率、设备限制

### 输出格式

- **默认**: Markdown人类可读格式（聊天直接看）
- **有App的用户**: 同时提供JSON格式，方便导入App的「计划」→「导入分析」功能
- JSON中的exerciseName必须使用数据库标准英文名（与App内嵌的free-exercise-db一致）

## JSON导入格式（App专用）

```json
{
  "name": "计划名称",
  "days": [
    {
      "dayOfWeek": 1,
      "targetMuscles": ["chest", "shoulders"],
      "exercises": [
        {"exerciseName": "Barbell Bench Press", "targetSets": 4}
      ]
    }
  ]
}
```

字段说明：
- `dayOfWeek`: 1=周一 ... 7=周日
- `targetMuscles`: chest, back, shoulders, arms, legs, core
- `exerciseName`: 必须与free-exercise-db的name字段匹配
- `targetSets`: 3-5

## 没有App的用户

对于不使用App的用户：
- 通过对话收集训练信息（频率、动作、感受）
- 用自然语言描述计划，Markdown表格格式
- 不需要生成JSON
- 手动追踪进度，通过对话了解训练情况
