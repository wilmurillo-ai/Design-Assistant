# Reply Patterns

Use this reference when drafting platform-native replies.

## General Pattern

Compose replies in this order:

1. acknowledge the specific problem
2. offer a useful idea, answer, or framing
3. mention the product only if it is clearly relevant
4. keep the CTA soft unless the thread explicitly asks for tools

## Help-Only Pattern

Use when risk is moderate or the fit is not strong enough for a product mention.

Structure:

- identify the pain precisely
- give one practical suggestion
- optionally share a non-promotional heuristic

## Soft-Mention Pattern

Use when the thread is relevant and recommendation context exists, but a hard pitch would feel forced.

Structure:

- answer the question directly
- mention the product as one possible fit
- explain why it matches the described use case
- avoid hype adjectives

## Direct-Mention Pattern

Use when the author is explicitly asking for a tool, alternative, or recommendation now.

Structure:

- state the fit clearly
- mention one or two reasons it matches
- keep the CTA brief

## Reddit Style Notes

Prefer:

- concrete language
- low-drama tone
- specifics over slogans
- acknowledgment of tradeoffs

Avoid:

- “game changer”
- “supercharge”
- “revolutionary”
- generic product marketing copy

## X Style Notes

Prefer:

- short sentences
- strong first line
- one idea per reply
- conversational rhythm

Avoid:

- long setup paragraphs
- emoji-heavy promo tone
- replies that read like ads

## Variation Rules

When drafting multiple replies:

- vary the opening
- vary sentence length
- vary whether the product appears early or late
- vary the CTA strength

Do not produce a batch of replies that can be recognized as the same template with swapped nouns.
