# Intent Signals

Use this reference when converting a product brief into search queries or when deciding whether a thread represents real demand.

## High-Intent Signals

Treat these as strong buyer or switching signals:

- asking for a recommendation
- asking for alternatives or replacements
- saying current tooling is too expensive, slow, or unreliable
- asking how others solve the same workflow
- describing active evaluation between named options
- asking for something “simple”, “cheap”, “faster”, or “works with X”
- saying “what do you use”, “any tool for this”, or “need a better way”

## Medium-Intent Signals

Treat these as possibly useful but not urgent:

- venting about pain without asking for help
- discussing process quality problems
- mentioning a workaround that is clearly unsatisfying
- asking broad market questions without immediate action

## Low-Intent Signals

Usually skip these:

- generic industry news
- memes, jokes, or hot takes
- academic curiosity
- broad discussions with no practical need
- competitor praise with no unmet need

## Pain-Language Mapping

Translate product features into user language.

Do not search for:

- your internal feature names
- category jargon unless users actually use it
- polished positioning copy from your landing page

Instead search for:

- workflow complaints
- “I hate doing X”
- “this takes forever”
- “any easier way”
- “tool for X”
- “alternative to Y”

## Query Families

Build queries from several families, not just one:

- recommendation: “best tool for ...”, “what do you use for ...”
- pain: “struggling with ...”, “annoying to ...”
- alternatives: “alternative to ...”, “replace ...”
- migration: “moving off ...”, “switch from ...”
- urgent need: “need help with ...”, “looking for ...”
- comparison: “X vs Y”

## Prioritization Heuristic

Prioritize threads where the author has:

- clear ownership of the problem
- current pain
- budget, urgency, or switching context
- willingness to try something new

Deprioritize threads where the author is:

- only debating theory
- obviously not the buyer or user
- asking for free support unrelated to the product
