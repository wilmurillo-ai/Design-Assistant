# Market Research

Use this reference when the user needs category research before outreach or positioning.

## Goal

Understand the market well enough to answer:

- who the product is really for
- what alternatives users already know
- how users describe the pain
- what differentiation angle is actually available
- which channels are worth testing first

## Research Scope

Cover four layers:

### 1. User Layer

Identify:

- primary user
- buyer if different from user
- urgent use cases
- recurring objections
- trust requirements

Focus on observed behavior and language, not just the builder's assumption.

### 2. Competitor Layer

Map:

- direct competitors
- adjacent tools
- manual alternatives
- incumbent products

For each, capture:

- target user
- apparent positioning
- obvious strengths
- obvious weaknesses
- likely switching trigger

### 3. Category Layer

Estimate whether the category is:

- crowded but weakly differentiated
- crowded with clear leaders
- fragmented by niche
- early and undefined

This determines whether to launch with:

- comparison angle
- niche angle
- simplicity angle
- workflow angle
- trust angle

### 4. Channel Layer

Infer where discovery is likely to happen:

- search
- recommendations
- community discussions
- directories
- creator or founder sharing

## Output Pattern

Return the research in a compact decision format:

- market summary
- ideal user summary
- competitor or substitute table
- recurring pain language
- available positioning angles
- recommended first channels
- key risks

## Heuristics

Prefer signals over speculation:

- repeated complaints
- repeated recommendation requests
- repeated comparison patterns
- repeated trust objections

Do not overfit to brand-name competitors. Manual workflows and partial substitutes often matter as much as direct competitors.
