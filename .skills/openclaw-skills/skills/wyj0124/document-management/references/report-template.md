# 报告模板

最终只输出一份完整报告。

严格按照以下结构生成：

# PDF 文件夹分析总报告

## 1. 文件夹概览
说明：
- 文件夹路径
- 共发现多少篇 PDF
- 成功处理多少篇
- 失败多少篇
- 这些文档整体围绕哪些主题或问题

## 2. 主题分类结果
列出主题分类概览，例如：
- 信息检索：3 篇
- 大模型应用：4 篇
- 系统架构：2 篇

并对每个主题补充一句说明。

## 3. 各主题下的文档归并
对每个主题分别写：

### 主题：{topic_name}
- 文档数量：{count}
- 涉及文档：
  1. {title_1}（{file_name_1}）
  2. {title_2}（{file_name_2}）

- 本主题概述：
  {概括这一主题下文档主要关注的问题、内容方向、常见方法或方案}

## 4. 总体结论
总结：
- 这批文档总体在讨论什么
- 主要集中在哪些方向
- 哪些主题最值得关注
- 整体上能提供什么认识

## 5. 单篇文档卡片
对每篇文档，按以下模板输出：

### {标题}
- 文件名：{file_name}
- 主题：{topic}
- 核心摘要：{summary}

- 关键观点：
  1. {point_1}
  2. {point_2}
  3. {point_3}

- 方法或方案：{method_or_solution}
- 结论：{conclusion}
- 风险/不足：{risks_or_limits}

## 6. 处理失败的文件（如有）
- {file_name}：{error}
