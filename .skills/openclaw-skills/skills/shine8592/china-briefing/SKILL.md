# China Briefing — 多尺度中文资讯简报

## 核心能力

**任意尺度，一处生成：**
- 全球 → 区域（亚太/欧洲/中东） → 国家 → 省 → 市 → 区 → 街道→ 社区
- 每层自动选择匹配的搜索源

## 简报结构（固定，不省略）

```markdown
### 🌍 核心结论
[1-2句总结本期最重要的信号/趋势/结论]

### 📌 {日期} {地点} 速报
**头条1 + 头条2 + 头条3**（最多3个，每个含嘟嘟点评）

### 📊 分类热点
[2-4个主题分类，每类2-4条]

### 📈 数据看板
[可选：关键数据表格]

### 🎯 行动建议
[优先级表格：高/中/低]
```

## 使用流程

### 用户说 "国际简报"
1. 用全球尺度源：Reuters + BBC + CNN + 半岛 + NYT + 新浪国际
2. 搜索词："international news latest" + "国际新闻 最新"
3. 输出 5-8 来源，标注出处

### 用户说 "江川路街道简报"
1. 用区县街道尺度源
2. 搜索词："江川路街道 2026 民生" + "江川路街道 实事" + "闵行区 江川路 新闻"
3. 输出 3-5 来源，标注出处

### 用户说 "上海简报"
1. 省市尺度源
2. 搜索词："上海 2026 最新动态" + "上海 经济 民生 政策"
3. 输出 4-6 来源

### 用户说 "AI行业简报"
1. 行业垂直源 + 社区
2. 搜索词："AI news latest 2026" + "AI 大模型 最新"
3. 来源：行业媒体 + 社区 + 研报

## 评论公式

每条头条必须写"嘟嘟点评"，遵守：

```
❌ 只是描述："XX报道了XXX"
✅ 加背景："XX做XXX，结合之前的YYY，说明ZZZ"
✅ 给观点："这意味着...我看好/担心..."
✅ 连点："这和上个月的XX配对，说明趋势在..."
✅ 对读者："对我们来说，意味着..."
```

## 信息筛选

| 问题 | 不通过 → |
|------|---------|
| 读者会分享给朋友？ | 去掉 |
| 有实质内容还是标题党？ | 去掉 |
| 来源可靠？ | 换来源 |
| 和读者相关？ | 标注为什么 |
| 能加独特观点？ | 加 |

## 来源多样性规则

| 尺度 | 最少来源数 | 必须包含 |
|------|----------|---------|
| 全球 | 5+ | Reuters + 1家中国官媒 + 1家社区 |
| 全国 | 4+ | 至少2家不同立场 |
| 省市 | 3+ | 官方 + 本地 |
| 区县/街道 | 2+ | 政府官网 + 至少1个来源 |
| 行业 | 3+ | 行业媒体 + 社区 |

**每条信息必须标注来源。**

## 字数控制

- 核心结论：50-100字
- 每条头条：150-300字（含点评）
- 分类热点：每条30-80字
- 总篇幅：800-2500字
