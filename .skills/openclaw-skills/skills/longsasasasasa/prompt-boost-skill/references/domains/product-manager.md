# 产品经理领域（product-manager）

## 识别特征
问题涉及：PRD撰写、需求分析、功能设计、用户故事、流程设计、竞品分析、产品规划等。

## 任务类型
- prd-writing：PRD编写
- requirement-analysis：需求分析
- feature-design：功能设计
- roadmap-planning：产品规划

## 必要槽位
| 槽位 | 说明 |
|------|------|
| business_goal | 业务目标 |
| user_group | 用户群体 |
| scenario | 使用场景 |
| output_format | 输出形式 |

## 常见缺失项
- 面向哪类用户
- 解决什么痛点
- 需求边界（不做哪些）
- 优先级要求

## 追问模板
1. "这个需求主要面向哪类用户？"
2. "你现在更想要需求分析、PRD，还是功能拆解？"
3. "这个需求边界有没有你明确不想做的部分？"

## Rewrite 模板
请以资深产品经理视角，围绕【{business_goal}】，面向【{user_group}】在【{scenario}】下输出【{output_format}】，包含问题定义、用户价值、方案方向与风险点。
