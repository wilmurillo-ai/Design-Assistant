# oclaw-hermes Agent 协作规范

> Agent 的边界，就是世界的边界。

## 智能体集群架构

oclaw-hermes 采用 **7 大智能体协同** 架构：

```
                    ┌─────────────┐
                    │  Lead Agent │
                    │  (意图识别)  │
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│ Research Agent│  │   Code Agent  │  │ Browser Agent │
│   (深度研究)   │  │   (代码执行)   │  │  (网页浏览)   │
└───────┬───────┘  └───────┬───────┘  └───────┬───────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│  Skill Agent  │  │  Memory Agent │  │  Expert Agent │
│  (技能调用)   │  │  (记忆检索)   │  │  (专家视角)   │
└───────────────┘  └───────────────┘  └───────────────┘
```

## 智能体职责

### 1. Lead Agent (主控智能体)

**职责**：意图识别、任务分发、结果整合

**触发条件**：所有请求

**工作流**：
1. 解析用户意图
2. 确定所需子智能体
3. 分发任务
4. 整合结果
5. 生成最终回复

### 2. Research Agent (研究智能体)

**职责**：深度研究、信息收集、多源验证

**触发条件**：研究类任务、需要外部信息

**工作流**：
1. 问题分解
2. 多源搜索（Web、学术、新闻）
3. 信息验证
4. 综合分析
5. 报告生成

### 3. Code Agent (代码智能体)

**职责**：代码生成、执行、调试

**触发条件**：编程类任务

**工作流**：
1. 需求分析
2. 代码生成
3. 沙箱执行
4. 错误处理
5. 优化迭代

### 4. Browser Agent (浏览器智能体)

**职责**：网页浏览、数据提取、视觉感知

**触发条件**：需要外部网页信息

**工作流**：
1. 导航到目标页面
2. 内容提取
3. 视觉分析（截图）
4. 数据结构化
5. 结果返回

### 5. Skill Agent (技能智能体)

**职责**：Skill 调用、管理、注册

**触发条件**：需要专业技能

**工作流**：
1. 识别所需 Skill
2. 参数提取
3. 调用 Skill
4. 结果处理
5. 经验沉淀

### 6. Memory Agent (记忆智能体)

**职责**：记忆检索、更新、同步

**触发条件**：需要上下文、历史信息

**工作流**：
1. 检索相关记忆
2. 上下文构建
3. 记忆更新
4. 跨平台同步

### 7. Expert Agent (专家智能体)

**职责**：专家思维蒸馏、视角切换

**触发条件**：需要专家视角

**工作流**：
1. 加载专家档案
2. 应用心智模型
3. 生成专家视角回复
4. 多专家会诊（可选）

## 协作模式

### 模式 1: 串行协作

```
用户 → Lead → Research → Lead → 回复
```

适用于：简单研究任务

### 模式 2: 并行协作

```
用户 → Lead → ┬→ Research
              ├→ Code
              └→ Browser
         ↓
      Lead整合 → 回复
```

适用于：复杂任务需要多维度处理

### 模式 3: 迭代协作

```
用户 → Lead → Code → 执行 → 错误 → Code → 成功 → Lead → 回复
```

适用于：编程调试任务

### 模式 4: 专家会诊

```
用户 → Lead → ┬→ Expert(马斯克)
              ├→ Expert(巴菲特)
              └→ Expert(任正非)
         ↓
      Lead整合 → 回复
```

适用于：需要多视角分析

## 安全边界

### 1. 命令审批

| 风险等级 | 处理方式 |
|----------|----------|
| 低风险 | 自动执行 |
| 中风险 | 提示确认 |
| 高风险 | 拒绝执行 |

### 2. 资源限制

- 单次任务最大执行时间: 300s
- 代码沙箱内存限制: 512MB
- 浏览器并发标签: 5
- API 调用频率: 60/min

### 3. 隐私保护

- 敏感信息自动脱敏
- 用户数据本地存储
- 跨平台同步加密

## 性能优化

### 1. 智能体缓存

- 频繁使用的 Skill 预加载
- 专家档案常驻内存
- 记忆索引优化

### 2. 并行执行

- 独立子任务并行处理
- 异步 I/O 操作
- 连接池复用

### 3. 增量更新

- 记忆增量同步
- 状态增量保存
- 日志轮转

## 调试指南

### 查看智能体状态

```bash
oclaw-hermes agents status
```

### 追踪任务执行

```bash
oclaw-hermes trace --task-id <uuid>
```

### 查看协作日志

```bash
oclaw-hermes logs --agent all --level debug
```

## 扩展开发

### 添加新智能体

1. 创建智能体类
2. 注册到 AgentRegistry
3. 定义触发条件
4. 实现 execute 方法

### 自定义协作模式

1. 定义模式配置
2. 注册到 WorkflowEngine
3. 测试验证
