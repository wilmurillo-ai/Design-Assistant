# 建设工程商事调解知识图谱

## 技能定位

本技能为建设工程领域商事调解提供系统化知识图谱支撑，覆盖调解全流程的节点、
规则、策略与文书生成，帮助调解员、律师、造价工程师和项目管理人员高效处理
工程纠纷。

法律依据：《人民调解法》《仲裁法》《民事诉讼法》《建筑法》《建设工程施工
合同示范文本》GF-2017-0201、GB/T 50500-2024《建设工程工程量清单计价标准》。

## 执行逻辑与工作流

### 工作流一：纠纷诊断与焦点梳理

**触发场景**：用户描述工程纠纷，需要梳理问题

**执行步骤**：
1. 读取 `references/dispute-types.md`，对争议进行分类定性
2. 识别核心争议焦点（金额、责任、时间、程序）
3. 评估争议复杂度（简单/中等/复杂）
4. 输出结构化诊断报告，包含：
   - 争议类型与子类型
   - 核心争议焦点列表
   - 涉及金额估算
   - 相关法律依据

**输出格式**：
```markdown
## 纠纷诊断报告

**争议类型**：[类型]
**争议焦点**：
1. [焦点一] — 金额：¥XXX，责任方：XXX
2. [焦点二] — ...

**法律依据**：
- [法规名称 + 具体条款]

**调解建议**：[简要评估]
```

---

### 工作流二：BATNA/WATNA 分析

**触发场景**：用户需要评估各方谈判底线与策略

**执行步骤**：
1. 读取 `references/batna-framework.md`，加载分析框架
2. 分别从发包人和承包人角度评估：
   - **BATNA**（最佳替代方案）：若调解失败，最有利的替代路径
   - **WATNA**（最差替代方案）：若调解失败，最不利的结果
   - **调解区间**（ZOPA）：双方可接受的交集范围
3. 给出调解策略建议

**分析维度**：
- 诉讼成本与周期
- 仲裁成本与周期
- 胜诉概率与预期
- 对双方关系的影响
- 声誉与品牌风险

---

### 工作流三：调解方案设计

**触发场景**：用户需要制定具体调解方案

**执行步骤**：
1. 读取 `references/mediation-strategies.md`，加载策略库
2. 根据争议类型选择适配策略：
   - **评估式调解**：调解员给出评估意见
   - **促进式调解**：引导双方自主协商
   - **混合式调解**：先促进后评估
3. 设计调解议程和时间安排
4. 准备背靠背会议提纲

---

### 工作流四：调解协议起草

**触发场景**：调解达成一致，需要起草正式协议

**执行步骤**：
1. 读取 `references/agreement-template.md`，加载协议模板
2. 填充核心要素：
   - 当事方信息
   - 争议简述
   - 调解结果（金额、时间、方式）
   - 履行条款
   - 违约责任
   - 司法确认条款（如需）
3. 生成协议草稿，标注待确认项

## 知识图谱节点关系规则

### 责任认定规则

| 争议类型 | 主要责任判断要点 | 参考依据 |
|---------|----------------|---------|
| 工期延误 | 关键路径影响 + 通知义务 | GF-2017-0201 第11条 |
| 工程变更 | 指令书面性 + 价格确认 | GF-2017-0201 第10条 |
| 质量缺陷 | 施工/设计/材料归因 | 《建筑法》第60条 |
| 结算争议 | 清单偏差 + 计价规则 | GB/T 50500-2024 |
| 工程款拖欠 | 竣工验收 + 付款节点 | 《民法典》第807条 |

### 调解优先级规则

1. **可调解优先**：金钱纠纷 > 工期纠纷 > 质量纠纷 > 合同解除纠纷
2. **时效优先**：诉讼时效剩余 < 6个月时，加速推进
3. **关系维护**：长期合作方优先促进式调解

## 边界说明

- 本技能聚焦建设工程领域，其他行业纠纷建议切换通用调解技能
- 涉及重大争议（>1000万元）建议提醒用户多方论证
- 法律意见仅供参考，不构成正式法律建议
- 调解协议草稿须经专业律师审核后使用
