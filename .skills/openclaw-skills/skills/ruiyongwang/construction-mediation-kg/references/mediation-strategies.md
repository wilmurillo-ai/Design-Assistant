# 调解策略与技巧库

## 一、调解模式选择

### 1.1 促进式调解（Facilitative Mediation）

**适用场景**：
- 当事方关系尚可，有继续合作意愿
- 信息相对对称，争议事实较清晰
- 当事方均有灵活协商空间

**调解员角色**：中立引导者，不发表实质性意见

**核心技巧**：
- 积极倾听，复述确认（"我理解您的意思是..."）
- 重构问题（将立场转化为利益）
- 分离情绪与事实
- 激发创造性解决方案

**典型流程**：
1. 开场白：建立信任，说明规则
2. 各方陈述：不打断，完整呈现立场
3. 问题界定：共同确认核心争议清单
4. 利益探索：挖掘立场背后的真实需求
5. 方案生成：头脑风暴，不评价
6. 方案评估：对照各方利益筛选
7. 协议起草：固化达成共识

---

### 1.2 评估式调解（Evaluative Mediation）

**适用场景**：
- 当事方立场固化，自主协商失败
- 争议存在明显法律或技术判断标准
- 一方对自身立场有不切实际的预期

**调解员角色**：评估者，给出有方向性的意见

**核心技巧**：
- 提出案情评估报告（优势/劣势分析）
- 给出调解建议方案（非裁决，仅建议）
- 指出诉讼/仲裁的不确定性和成本

**适用于建设工程**：
- 工程款金额评估（参照计价规范）
- 工期延误责任划分（参照进度计划）
- 质量缺陷归因（借助技术专家）

---

### 1.3 混合式调解（Med-Arb / Hybrid）

**适用场景**：
- 部分问题可协商，部分问题需裁判
- 时效压力下需快速解决

**模式**：调解不成部分直接转入仲裁，同一调解员可继续担任仲裁员（需当事方同意）。

## 二、会议设计

### 2.1 联席会议设计

**开场白要素**：
1. 自我介绍，建立调解员中立性
2. 说明调解规则（保密、自愿、无约束力）
3. 确认各方代表的授权范围
4. 设定议程和时间安排

**陈述阶段管理**：
- 每方陈述时间：10-15分钟
- 不打断，但可记录要点
- 陈述完毕后，复述确认关键信息

**联席会议常见挑战**：

| 挑战 | 处理技巧 |
|-----|---------|
| 情绪激动、人身攻击 | 立即叫停，重申规则，转移焦点到问题本身 |
| 一方滔滔不绝 | 礼貌打断，使用"我注意到...让我们..."句式 |
| 双方观点差异极大 | 先搁置分歧，聚焦共识点 |
| 一方沉默不语 | 背靠背会议单独沟通 |

---

### 2.2 背靠背会议（Caucus）设计

**使用时机**：
- 当事方在对方面前无法坦诚表达
- 需要单独核实信息或探测底线
- 联席会议陷入僵局

**背靠背会议提纲**：

**第一轮（了解真实利益）**：
1. "在联席会议上，您提到了XXX，能否告诉我这背后最重要的考虑是什么？"
2. "如果您理想的结果是什么？最低可接受的结果是什么？"
3. "您最担心的风险是什么？"

**第二轮（评估替代方案）**：
1. "如果今天调解不成，您接下来会怎么做？"
2. "您对诉讼/仲裁的结果有多大把握？预计需要多长时间？"
3. "那个结果与您今天的期望相比，哪个更好？"

**第三轮（推动让步）**：
1. "如果对方做出XX调整，您是否有可能接受？"
2. "有没有非金钱的方式可以弥补差距？"
3. "您觉得对方最在意的是什么？"

**保密原则**：未经当事人明确授权，不向对方披露背靠背会议内容。

## 三、常用调解技巧

### 3.1 重构技巧（Reframing）

将立场语言转化为利益语言：

| 立场语言（当事方说） | 重构为利益语言（调解员问） |
|------------------|------------------------|
| "他们必须赔偿全部损失" | "您最关心的是什么？是钱的问题，还是有其他的考虑？" |
| "绝对不可能减少一分钱" | "您的底线考虑是什么？有哪些约束条件？" |
| "他们根本不讲诚信" | "您希望通过这次调解，修复什么？解决什么？" |

### 3.2 单一文本技巧（Single Negotiating Text）

当双方方案差距较大时，由调解员起草一份综合方案，各方在此基础上修改，逐步缩小差距。

**步骤**：
1. 调解员起草第一稿（不代表任何一方立场）
2. 分别征求各方意见
3. 根据反馈修改，形成第二稿
4. 重复上述步骤直至各方接受

### 3.3 切割议题技巧（Issue Splitting）

将复杂争议切割为小议题，逐个推进：
- 先易后难：先解决争议较小的问题，建立合作氛围
- 捆绑交换：将若干小议题打包，形成整体交换方案
- 搁置争议：暂时搁置无法协商的议题，先处理其余部分

### 3.4 时间压力技巧

适当提示：
- 诉讼时效即将届满的风险
- 工程继续推进的成本（如项目停工损失）
- 季节性施工窗口的紧迫性

## 四、建设工程特有调解策略

### 4.1 工程款争议：分阶段支付方案

当全额支付存在争议时，推动以下方案：
1. **无争议部分先行支付**：双方确认无争议的工程量及价款，立即支付
2. **争议部分设立专户**：双方同意将争议金额存入第三方托管账户
3. **时间表约定**：约定争议部分的解决时间和方式

### 4.2 质量争议：修复方案优先

优先推动修复方案而非金钱赔偿：
- 由原施工方修复（成本最低）
- 由第三方修复，费用由责任方承担
- 金钱折减（降价接受有瑕疵工程）

### 4.3 工期延误：责任共担方案

当工期延误责任难以清晰界定时：
- 按比例分担违约金（根据各方贡献度）
- 互相豁免违约金，双方共同向前
- 以工程款结算优惠换取工期违约豁免

## 五、调解僵局处理

### 僵局类型与应对

| 僵局类型 | 识别特征 | 处理策略 |
|---------|---------|---------|
| 信息不对称僵局 | 各方认知事实不同 | 委托专家意见；开示证据 |
| 立场固化僵局 | 重复同一立场，不肯让步 | 背靠背会议探测真实底线 |
| 情绪僵局 | 敌对情绪主导，无法理性讨论 | 暂停调解；情绪疏导；分开陈述 |
| 权限僵局 | 代表无决策权 | 要求更高层级代表出席 |
| 价值观僵局 | 双方原则性立场冲突 | 聚焦具体利益，淡化原则争议 |

### 调解员最后手段（Last Resort）

1. **现实测试**：请当事方描述如果诉讼败诉会发生什么
2. **最终提案**：调解员提出不可更改的一次性调解方案
3. **休会重整**：宣布暂时休会，留时间让当事方重新评估
4. **终止调解**：如确实无法推进，正式终止，出具终止证明
