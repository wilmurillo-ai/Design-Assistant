## 一、系统定位与设计理念

### 1.1 核心价值
本系统模拟造价工程师的思维过程，通过**结构化询问**收集必要参数，**动态调用**设计规范数据库，**智能组合**配比规则，最终生成符合行业规范的结构化估算报告。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **规范前置** | 设计规范是计算规则的前提，不是备选 |
| **渐进式询问** | 每次只问1-2个关键问题，避免信息过载 |
| **参数必填/可选项分离** | 明确区分必须参数和可选参数 |
| **默认值容错** | 可选参数缺失时自动使用推荐值 |
| **实时校验** | 每步输入立即校验合理性 |
| **透明可解释** | 每步计算都标注规范依据 |

## 三、动态规则调用机制

### 3.1 规则编排引擎（Rule Orchestration Engine）

```python
class RuleOrchestrator:
    """
    规则编排引擎
    根据输入参数动态组合和调用配比规则
    """
    
    def __init__(self, knowledge_base):
        self.kb = knowledge_base
        self.rule_chain = []
        
    def build_rule_chain(self, params: dict) -> list:
        """
        根据参数构建规则调用链
        每个规则节点包含：触发条件、规则ID、优先级、依赖项
        """
        chain = []
        
        # 规则1：确定钢筋含量配比（优先级最高）
        chain.append(RuleNode(
            id="REINFORCEMENT_RATIO",
            priority=1,
            condition=lambda p: p.get('structure_type') and p.get('seismic_grade'),
            handler=self._get_reinforcement_rule,
            depends_on=[]
        ))
        
        # 规则2：确定混凝土含量配比
        chain.append(RuleNode(
            id="CONCRETE_RATIO",
            priority=2,
            condition=lambda p: p.get('building_type'),
            handler=self._get_concrete_rule,
            depends_on=[]
        ))
        
        # 规则3：模板综合配比
        chain.append(RuleNode(
            id="FORMWORK_RATIO",
            priority=3,
            condition=lambda p: True,  # 始终适用
            handler=self._get_formwork_rule,
            depends_on=[]
        ))
        
        # 规则4：砌体含量配比（剪力墙结构除外）
        chain.append(RuleNode(
            id="MASONRY_RATIO",
            priority=4,
            condition=lambda p: '剪力墙' not in p.get('structure_type', ''),
            handler=self._get_masonry_rule,
            depends_on=[]
        ))
        
        # 规则5：电气安装配比
        chain.append(RuleNode(
            id="ELECTRICAL_RATIO",
            priority=5,
            condition=lambda p: p.get('building_type') in ELECTRICAL_APPLICABLE,
            handler=self._get_electrical_rule,
            depends_on=[]
        ))
        
        # 规则6：给排水配比
        chain.append(RuleNode(
            id="PLUMBING_RATIO",
            priority=6,
            condition=lambda p: p.get('building_type') in PLUMBING_APPLICABLE,
            handler=self._get_plumbing_rule,
            depends_on=[]
        ))
        
        # 规则7：通风空调配比（仅商业/医院/酒店）
        chain.append(RuleNode(
            id="HVAC_RATIO",
            priority=7,
            condition=lambda p: p.get('building_type') in HVAC_APPLICABLE,
            handler=self._get_hvac_rule,
            depends_on=[]
        ))
        
        # 规则8：地下室调整（当有地下室时）
        chain.append(RuleNode(
            id="BASEMENT_ADJUST",
            priority=8,
            condition=lambda p: p.get('basement_area', 0) > 0,
            handler=self._apply_basement_adjust,
            depends_on=['CONCRETE_RATIO']
        ))
        
        # 规则9：桩基工程（当有桩基时）
        chain.append(RuleNode(
            id="PILE_FOUNDATION",
            priority=9,
            condition=lambda p: p.get('has_pile', False),
            handler=self._apply_pile_adjust,
            depends_on=[]
        ))
        
        return chain
```
