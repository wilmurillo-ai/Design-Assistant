# 专家思维提炼框架

## 一、心智模型三重验证

**一个观点被收录为"心智模型"，必须同时满足：**

### ✅ 验证1：跨域复现
同一思维框架需在专家讨论的**2个以上不同领域**中出现。
- 仅在单一场合提到 → 降级为"决策启发式"
- 多次跨域复现 → 认定为"心智模型"

### ✅ 验证2：有生成力
使用该模型应能**推断专家对新问题的可能立场**。
- 能推断 → 有预测力，收录
- 无法推断 → 可能只是知识性描述，不收录

### ✅ 验证3：有排他性
该模型应体现**专家的独特视角**，而非普遍共识。
- 所有聪明人都会这么想 → 不收录（常识，非模型）
- 只有该专家才会这么想 → 强收录

## 二、表达DNA量化方法

### 句式指纹
| 指标 | 说明 | 测量方式 |
|------|------|---------|
| 平均句长 | 长句vs短句偏好 | 统计字数/句 |
| 疑问句比例 | 苏格拉底式提问习惯 | 统计问句占比 |
| 类比密度 | 用比喻说明抽象概念 | 统计比喻/千字 |
| 数字引用率 | 用数据说话的习惯 | 统计数据点/段 |
| 规范引用率 | 标准条文引用习惯 | 统计引用次数 |

### 风格维度定位（0-10分）
```
正式 ←————————→ 口语
抽象 ←————————→ 具体
理论 ←————————→ 实操
保守 ←————————→ 激进
宏观 ←————————→ 微观
```

### 禁忌词与口癖
- **口癖**（高频词）：适度使用，保持风格辨识度
- **禁忌词**（从不使用）：识别并严格回避

## 三、矛盾处理原则

矛盾不是错误，是专家深度的来源。

| 矛盾类型 | 说明 | 处理方式 |
|---------|------|---------|
| 时间性矛盾 | 早期观点与晚期观点不同 | 标注时间线，两者都保留 |
| 领域性矛盾 | A领域和B领域立场不同 | 分别记录，标注适用场景 |
| 本质性张力 | 内在价值冲突 | 如实记录，不强行统一 |

**错误做法：** 忽略矛盾 / 强行统一 / 取一舍一
**正确做法：** 记录、标注、保持张力

## 四、信息不足处理指南

| 情况 | 处理方式 |
|------|---------|
| 公开信息极少 | 明确标注"信息有限，以下为推测" |
| 信息相互矛盾 | 并列呈现，不做主观判断 |
| 涉及隐私领域 | 停止推断，说明边界 |
| 争议性话题 | 呈现多方观点，不代为定论 |

## 五、工程咨询专家提炼重点

针对工程咨询领域专家，重点提炼：

### A. 造价思维
- 如何判断造价的合理区间
- 面对甲方压价时的决策逻辑
- 对不同工程类型的价格直觉

### B. 合同思维
- 风险识别的优先顺序
- 价款条款的核心关注点
- 争议条款的处理策略

### C. 调解思维
- 争议焦点的快速识别方法
- 利益平衡点的寻找逻辑
- 说服对立方的话术模式

### D. 项目管理思维
- 工期延误归因的分析框架
- 质量问题的系统性归因
- 项目风险的预判方式

## 六、质量自检清单

生成专家Skill后，逐项检查：

- [ ] 心智模型数量在3-7个之间
- [ ] 每个心智模型都通过三重验证
- [ ] 决策启发式是操作级规则（能直接用）
- [ ] 表达DNA有具体的量化特征
- [ ] 反模式清单有至少3条
- [ ] 诚实边界说明了局限性
- [ ] 用3个已知问题验证通过
- [ ] 用1个新问题验证适度不确定性
