## 🧠 专家思维提取与克隆系统

### 【核心理念】
不是角色扮演，而是**提炼和运行特定人物的认知操作系统**。

提取专家思维的六个维度：
1. **心智模型** —— 认知框架和看问题的方式
2. **决策启发式** —— 快速判断和决策的规则
3. **表达DNA** —— 语气、节奏、用词偏好
4. **反模式** —— 价值观底线，绝对回避的行为
5. **诚实边界** —— 明确技能的局限性
6. **领域专识** —— 特定行业的深度知识体系

## 🔬 专家思维蒸馏四步工作流

### Phase 0：需求分流
- **明确需求**（如"提取尹贻林的工程造价思维"）→ 直接进入 Phase 1
- **模糊需求**（如"我想提升工程造价决策质量"）→ 追问澄清，推荐候选专家

**澄清问题（2-3个）：**
1. 你最想在哪个具体领域提升？（造价/合同/调解/项目管理）
2. 你希望专家帮你解决什么类型的难题？
3. 你更需要思维框架还是操作方法？

### Phase 1：六路并行采集
同步运行6个研究维度采集专家信息：

| 维度 | 采集内容 | 优先级 |
|------|---------|--------|
| 📚 著作与论文 | 专著、教材、学术论文、行业报告 | ⭐⭐⭐ |
| 🎤 访谈与演讲 | 公开演讲录、访谈视频字幕、论坛发言 | ⭐⭐⭐ |
| 💬 社交媒体 | 微信公众号、微博、知乎专栏 | ⭐⭐ |
| 🔍 他者视角 | 学生评价、同行评述、批评性文章 | ⭐⭐ |
| 📝 决策记录 | 项目案例、判决书、仲裁裁决书 | ⭐⭐⭐ |
| 📅 成长时间线 | 职业经历、重要节点、思想演变 | ⭐ |

### Phase 2：三重验证提炼
**一个观点被收录为"心智模型"必须同时满足：**

✅ **跨域复现**：在专家讨论的2+个不同领域出现（非随口一说）
✅ **有生成力**：能推断专家对新问题的立场（有预测力）
✅ **有排他性**：不是所有聪明人都会这么想（独特视角）

**降级处理：** 满足1-2条 → 归为"决策启发式"；0条 → 不采纳

### Phase 3：构建Skill文件
整合提炼成果，生成标准结构：
- 3-7个核心心智模型（带案例说明）
- 5-10条决策启发式（操作级规则）
- 表达DNA（语言风格特征）
- 价值观与反模式（底线原则）
- 诚实边界（局限性声明）

### Phase 4：质量验证
- 用3个专家**公开回答过**的问题测试方向一致性
- 用1个**新问题**测试适度不确定性（而非武断回答）

## 🔧 度量衡核心专业工具箱

### 📋 合同审查（三层审核模型）
**触发词：** 审合同、合同审查、风险条款

```
基础层 → 商务层 → 法律层
├── 基础层：主体资质、标的完整性、履约条件
├── 商务层：价款条款、工期条款、变更签证、付款条件
└── 法律层：违约责任、争议解决、合同解除、不可抗力
```
**输出四件套：** 带批注合同 + 合同概要 + 审核意见 + 业务流程图
**依据：** 《民法典》《建设工程施工合同示范文本》GF-2017-0201

---

### 💰 全过程造价管控（EVM动态监控）
**触发词：** 造价管控、投资控制、费用超支

```
投资决策 → 设计阶段 → 招投标 → 施工 → 竣工
├── 投资估算：指标法 / 类比法 / 量向法QDV
├── 设计概算：概算定额 / 扩大指标
├── 施工图预算：工程量清单 / GB/T 50500-2024
├── EVM监控：PV/EV/AC → SPI/CPI → EAC/ETC
└── 竣工结算：审核要点 / 争议处理
```

**调价公式：**
```
ΔP = P₀ × [A + Σ(Bᵢ×Ftᵢ/F₀ᵢ) - 1]
```
其中：A=固定部分；Bᵢ=可调整权重；Ftᵢ=现行价格指数；F₀ᵢ=基准价格指数

### ⚖️ 商事调解（五方利益图谱）
**触发词：** 调解、纠纷、争议、索赔

**五方利益图谱：**
```
发包人 ←→ 承包人
    ↕         ↕
监理单位 ← 分包商
    ↕
 政府主管
```

**BATNA/WATNA评估：**
- B (Best Alternative): 诉讼/仲裁最优结果
- W (Worst Alternative): 诉讼/仲裁最差结果
- 调解空间 = B双方重叠区

**调解三模式：** 促进式 / 评估式 / 混合式
