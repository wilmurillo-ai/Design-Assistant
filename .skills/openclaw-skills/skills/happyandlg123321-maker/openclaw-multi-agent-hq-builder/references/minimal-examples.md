# 最小示例

## 目标
这份文件只解决一个问题：
让新人知道“最小可运行”到底长什么样。

---

## 一、任务卡最小示例
```md
# TASK-001
- 任务名称：搭建 HQ
- 当前状态：NEW
- 当前负责人：001
- 下一接手BOT：03
- 下一步动作：完成问题定义
```

作用：
哪怕只写这 5 行，也比没有任务卡强。

---

## 二、共享黑板最小示例
```md
## 当前统一事实
- HQ 采用 001-05 结构
- 当前任务：搭建总部系统
- 当前状态：CLARIFIED
- 下一步：交给 04 做方向校准
```

作用：
所有 BOT 都围绕同一事实推进，而不是各说各话。

---

## 三、复盘记录最小示例
```md
## 任务复盘
- 做对了什么：先搭中枢，再补机制
- 做错了什么：初期缺少状态同步
- 最该优化的一环：任务卡回写纪律
- 下次固定动作：每次状态推进都回写任务卡
```

作用：
让一次任务结束后，至少能留下一个可复用改进动作。

---

## 核心句
最小示例的价值，不在于完整，
而在于让新人先跑起来，再逐步升级到完整版本。
