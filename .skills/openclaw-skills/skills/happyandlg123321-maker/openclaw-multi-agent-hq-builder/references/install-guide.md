让一个新人拿到这份指南后，能够从零完成一套 **001 母bot + 4 子bot** 的 OpenClaw 多智能体总部搭建，并达到可运行标准。

## 二、推荐安装顺序

### 第一步：先定总部结构
固定核心中枢：
- 001：总控与拍板
- 02：价值与资源
- 03：问题与逻辑
- 04：方向与原则
- 05：执行与交付

### 第二步：先写总部文档
不要一开始就写很多 bot 人设。
先把规则写出来。

### 第三步：安装 001-05 四件套
每个 bot 都必须有完整四件套。
不完整，不算安装完成。

### 第四步：安装 dispatcher
让 001 具备真实调度能力。

### 第五步：安装 L5 运行骨架
安装：
- 状态机
- 任务卡模板
- 共享黑板协议
- 复盘机制
- 升级日志

### 第六步：创建首张任务卡
至少创建 1 张真实任务卡，并跑完整链路。

## 三、零散文档 → skill/package 的 5 步迁移法

如果用户手里已经有很多零散文档，不要重写一切，按这 5 步收敛：

### 第 1 步：先分层
把现有材料分成 5 类：
- 组织层
- 角色层
- 调度层
- L5运行层
- 教学验收层

### 第 2 步：先定最小中枢
先只保留 001-05 作为 HQ 核心。
其他 agent 先放外围池，不要一起塞进正式版。

### 第 3 步：把零散文档映射成固定文件
优先映射到这些目标文件：
- HQ docs
- 每个 bot 四件套
- dispatcher / runbook
- 状态机 / 任务卡 / 黑板 / 复盘

### 第 4 步：补缺口，不重写全部
先找最关键缺口：
- 缺 dispatcher
- 缺状态机
- 缺任务卡
- 缺验收标准

只补关键缺口，不做无效重构。

### 第 5 步：用一张真实任务卡做验收
至少创建 1 张真实任务卡，跑通一次。
如果跑不通，说明 skill/package 还没真正成立。

## 四、新人最容易失败的地方

### 1. 一开始就扩很多 bot
错。
先把 001-05 跑通。

### 2. 只有人设，没有调度
错。
没有 dispatcher，多智能体就只是多人聊天。

### 3. 只有流程，没有状态
错。
没有状态机，任务一多就乱。

### 4. 只有文档，没有任务卡
错。
没有任务卡，就没有共享事实源。

### 5. 只有交付，没有复盘
错。
没有复盘，系统不会进化。

## 五、验收标准
安装完成后，检查以下 10 项：

如果用户给了明确截止时间，优先输出“最短可完成路径”，确保新人先完成 100 分安装，再考虑额外美化或扩展。 

1. 是否已有 001-05 总部结构
2. 是否已有总部文档
3. 是否已有 dispatcher
4. 是否已有任务状态机
5. 是否已有任务卡模板
6. 是否已有共享黑板协议
7. 是否已有复盘机制
8. 是否已有升级日志
9. 是否已有至少 1 张真实任务卡
10. 是否能解释完整任务链路：001 → 子bot → 001

全部满足，才算 100 分安装。

---

## 六、核心句
新人安装最重要的不是“装了多少bot”，
而是是否搭出了一套：
**可运行、可调度、可复盘、可升级的多智能体总部系统。**
