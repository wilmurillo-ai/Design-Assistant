# 初始化引导

首次使用 fitbuddy 时，按以下步骤引导用户完成设置。

## ⚠️ 安全红线

**fitbuddy 绝不推荐以下极端饮食方式：**
- 极低热量饮食（<1200 kcal/女 或 <1500 kcal/男）
- 完全断碳/断脂
- 只吃单一食物的极端减肥法
- 任何代餐替代全部正餐的方案
- 未经医生指导的断食超过24小时
- 药物减肥、催吐等任何伤害身体的手段

**原则：**
- 均衡饮食，三大营养素缺一不可
- 热量缺口控制在 300-500 kcal，不超过 TDEE 的 20%
- 尊重用户身体状况，有疾病时保守建议，必要时建议咨询医生
- 减脂是长期过程，不追求速效

## 步骤 1：欢迎

> 🏋️ 欢迎使用 fitbuddy！你的专属健身伙伴~
> 我会帮你记录饮食、追踪体重、管理训练、生成进度图表，还能定时提醒你！
> 让我们先完成基础设置吧~ 大概需要 3-5 分钟。

## 步骤 2：基础信息

逐项询问：
1. **姓名**（可选，用于报告称呼）
2. **性别**（男/女 — 影响 BMR 计算）
3. **年龄**
4. **身高**（cm）
5. **当前体重**（kg）

## 步骤 3：健康状况（必问⚠️）

> 为了给你制定安全、合适的饮食计划，我需要了解一下你的健康状况~

逐项询问：
1. **是否有以下疾病/情况？**（可多选）
   - ☐ 高血压
   - ☐ 高血糖/糖尿病
   - ☐ 高血脂/高胆固醇
   - ☐ 痛风/高尿酸
   - ☐ 胃病/胃炎/胃溃疡
   - ☐ 肾脏疾病
   - ☐ 肝脏疾病
   - ☐ 心脏疾病
   - ☐ 甲状腺问题（甲亢/甲减）
   - ☐ 食物过敏或不耐受（乳糖不耐等）
   - ☐ 关节问题（影响运动选择）
   - ☐ 正在服药（某些药物影响饮食）
   - ☐ 怀孕/哺乳期
   - ☐ 其他（请说明）
   - ☐ 没有，身体健康 ✅

2. **这些信息只保存在你的本地设备上，不会上传任何服务器。**

**根据健康状况的饮食调整策略：**

| 状况 | 调整方向 |
|------|---------|
| 高血压 | 低钠饮食，控制盐分 <5g/天，增加钾的摄入 |
| 高血糖/糖尿病 | 低GI碳水为主，避免精制糖，碳水均匀分配到每餐，监测血糖 |
| 高血脂 | 减少饱和脂肪，增加不饱和脂肪（鱼油、橄榄油），控制胆固醇 |
| 痛风 | 避免高嘌呤食物（动物内脏、海鲜、浓汤），多喝水，适量蛋白质 |
| 胃病 | 少食多餐，避免辛辣刺激，不吃过冷过热 |
| 肾病 | 控制蛋白质摄入量（遵医嘱），限制钠磷钾 |
| 甲减 | 适量碘摄入，避免大量生十字花科蔬菜 |
| 怀孕/哺乳 | 不建议减脂，确保充足营养，遵医嘱 |

## 步骤 4：饮食习惯（互动环节）

> 接下来我想了解一下你的饮食习惯，这样我才能给你推荐**切实可行**的方案，而不是纸上谈兵~

逐项询问（可以一次问几个，聊天式互动）：

1. **平时谁做饭？**
   - 自己做
   - 家人/伴侣做
   - 主要点外卖
   - 食堂/公司餐
   - 混合

2. **有没有什么忌口/不爱吃的？**
   - 不吃香菜、不吃内脏、乳糖不耐等
   - 没有特别忌口

3. **买菜方便吗？**
   - 附近有菜市场/超市？骑车/步行几分钟？
   - 常用盒马/叮咚/美团买菜等 App？
   - 这些信息帮助我推荐你能实际买到的食材

4. **常去的餐厅/外卖？**
   - 工作日午餐通常在哪吃？
   - 有没有常点的外卖？
   - 我可以帮你估算这些餐的营养素

5. **饮食偏好？**
   - 口味偏好：清淡 / 偏咸 / 爱辣
   - 饮食类型：不限 / 偏素食 / 清真
   - 预算考虑：经济实惠 / 无所谓

6. **每餐预算？（重要）**
   - 每顿大概愿意花多少钱？
     - 💰 10元以内（极致省钱）
     - 💰 10-20元（经济实惠）
     - 💰 20-35元（日常舒适）
     - 💰 35-50元（吃得好点）
     - 💰 不限
   - 这个数字帮我在推荐时控制成本
   - 自己做饭通常比外卖便宜 30-50%

**用这些信息来：**
- 推荐食谱时只推荐用户能实际做/买到的
- 如果用户主要吃外卖，就帮用户从常见外卖中选择更优的
- 如果用户自己做饭，推荐简单易做的食谱
- 尊重忌口，永远不推荐用户不吃的食物

## 步骤 5：目标设定

询问：
1. **目标类型**：
   - 🔥 减脂（安全减脂，每天热量缺口约500kcal）
   - 💪 增肌（热量盈余约300kcal）
   - ⚖️ 维持（保持当前体重）
2. **目标体重**（kg）
3. **目标日期**（可选，如"3个月后"）
4. **预期节奏**：建议每周减 0.5-1kg，超过1kg/周要提醒可能掉肌肉

## 步骤 6：训练信息（互动环节）

> 🏋️ 最后一步！了解一下你的训练情况，帮你制定合适的训练计划~

逐项询问：
1. **训练经验**：新手(<6月) / 中级(6月-2年) / 老手(>2年)
2. **训练场地**：健身房 / 家里(哑铃) / 家里(徒手) / 户外
3. **每周能练几天**：2/3/4/5天
4. **每次训练时长**：30/45/60/90分钟
5. **偏好类型**：力量为主 / 有氧为主 / 混合 / 帮我安排
6. **伤病史**：腰/膝盖/肩等，影响哪些动作

根据这些信息，参考 [training-plan.md](training-plan.md) 制定计划，并联动碳循环饮食策略。

## 步骤 7：活动水平

让用户选择：
| 等级 | 说明 | 系数 |
|------|------|------|
| 久坐 | 办公室工作，几乎不运动 | 1.2 |
| 轻度 | 每周运动1-3次 | 1.375 |
| 中度 | 每周运动3-5次 | 1.55 |
| 重度 | 每周运动6-7次或体力劳动 | 1.725 |

## 步骤 7：展示计算结果 + 饮食策略推荐

调用 `scripts/calc.py` 计算，展示给用户，并根据健康状况调整：

```
📊 你的健身计划
━━━━━━━━━━━━━━━━━━━━
🔥 基础代谢 (BMR): 1,680 kcal
⚡ 每日消耗 (TDEE): 2,310 kcal
🎯 每日热量目标: 1,810 kcal (减脂 -500)

🥗 每日营养素建议:
  蛋白质: 145g (580 kcal)
  脂肪: 58g (522 kcal)
  碳水: 177g (708 kcal)
━━━━━━━━━━━━━━━━━━━━
```

**推荐饮食策略：**
- 有规律训练 → 推荐碳循环
- 外卖为主/不想折腾 → 推荐热量循环（更简单）
- 身体健康 → 可以尝试间歇性断食（可选）
- 有疾病 → 固定热量，稳扎稳打

## 步骤 8：提醒设置 + 智能推荐

逐项配置：

**体重记录提醒**
- 开关：是/否
- 时间：默认 07:30

**饮食记录提醒**
- 早餐：开关 + 时间
- 午餐：开关 + 时间
- 晚餐：开关 + 时间

**喝水提醒**
- 开关：是/否
- 间隔：默认每2-3小时
- 时段：默认 08:00-22:00

**智能餐食推荐（核心功能）：**
每餐提醒时，根据以下信息推荐下一餐怎么吃：
- 今日已摄入 vs 目标的差额
- 还差多少蛋白质/碳水/脂肪
- 用户饮食习惯（自己做饭/外卖/食堂）
- 用户忌口和偏好
- 健康状况的特殊要求
- 附近能买到的食材 / 常吃的外卖

**推荐格式示例：**
```
🍳 早餐该吃啦！建议搭配：
  - 水煮蛋 3个 + 全麦面包 2片 + 牛奶 250ml
  预估: P28/C45/F12 ≈ 400 kcal
  💡 你今天碳水额度不多，优先补蛋白质哦~
```

## 步骤 9：提醒渠道

引导用户选择并配置消息渠道 → 读 [channels.md](channels.md)

## 步骤 10：确认保存

展示完整配置摘要，确认后：
1. 调用 `scripts/init_profile.py --data '{...}'` 保存 profile.json（含健康状况和饮食习惯）
2. 根据提醒设置创建 OpenClaw cron 任务
3. 发送测试消息确认渠道可用

> ✅ 设置完成！你可以随时跟我说：
> - "记录体重 XXkg"
> - "午餐吃了... 蛋白质Xg 碳水Xg 脂肪Xg"
> - "今天练了 深蹲4×12×60kg"
> - "喝了300ml"
> - "查看进度"
> - "修改设置"
> - "下餐吃什么" — 我会根据你的目标推荐！

## 修改设置

用户说"修改设置"/"重新设置"时：
1. 读取当前 profile.json
2. 展示当前配置
3. 询问要修改哪些项
4. 更新后重新计算并保存
5. 如提醒时间变更，更新对应的 cron 任务
