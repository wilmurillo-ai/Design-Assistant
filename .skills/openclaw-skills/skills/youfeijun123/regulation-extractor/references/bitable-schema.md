# 飞书多维表格字段规范

## 表格结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 条文编号 | 文本 | 如 `3.0.1`、`6.4.4` |
| 所属章节 | 文本 | 如 `基本规定`、`操作平台` |
| 页码 | 文本 | PDF原始页码 |
| 质量 | 文本 | `✅ 干净` 或 `⚠️ OCR错误` |
| 条文内容 | 文本 | 条文全文 |

## 章节映射

默认映射（可根据规范调整）：

```json
{
    "1": "总则", "2": "术语和符号", "3": "基本规定",
    "4": "临边与洞口作业", "5": "攀登与悬空作业",
    "6": "操作平台", "7": "交叉作业", "8": "建筑施工安全网"
}
```

## 去重规则

- **同一条文编号只保留一条**（保留最早写入的）
- 子条目独立编号（如6.4.④转为6.4.4后独立计数）
- 不同编号但内容相似的条文**不去重**（避免误合并）

## 质量标记

| 标记 | 含义 | 处理 |
|------|------|------|
| ✅ 干净 | 无OCR错误特征字符 | 可直接使用 |
| ⚠️ OCR错误 | 含OCR特征字符 | 需人工核对 |

## API使用

### 创建新表格

通过飞书多维表格API创建，字段类型：
- 条文编号：文本
- 所属章节：文本
- 页码：文本
- 质量：文本
- 条文内容：文本

### 批量操作

- 写入：逐条调用 `create_record`（每条约1秒）
- 删除：使用 `batch_delete` 接口（一次最多500条）
- 查询：使用 `list_records`（一次最多500条，支持分页）

### 限速

飞书API限频约100次/分钟，大批量写入时需适当sleep。
