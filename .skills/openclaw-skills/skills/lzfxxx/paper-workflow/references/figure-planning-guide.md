# Figures

## Purpose

Use this guide when deciding what figures to create and how to integrate them into the paper.

## Figure Planning Rules

Before drawing a figure, answer three questions:

1. Which section needs this figure?
2. What claim does this figure support?
3. Why is the figure better than prose alone?

If those questions are unclear, do not draw the figure yet.

## Common Figure Types

- architecture diagram
- end-to-end workflow diagram
- sequence diagram
- state transition diagram
- artifact or contract flow diagram
- UI evidence grid
- result table or summary chart

## Tool Routing

Use `prompt-to-drawio` for:

- architecture diagrams
- workflow diagrams
- sequence diagrams
- state machines
- contract-flow diagrams

Use screenshots for:

- teacher or user interface evidence
- output examples
- pipeline stages that are easier to show than redraw

## Screenshot Rules

- Treat screenshots as evidence, not product marketing.
- Choose screenshots that reveal workflow stages, not decorative views.
- Prefer English UI when available.
- If source screenshots contain distracting local-language text, anonymize or replace with an English version.
- When multiple screenshots are needed, prefer one composite grid with a clear caption.

## Caption Rules

- State what the figure shows.
- State the role of each panel if it is a multi-panel figure.
- Use natural prose; do not explain that the figure was translated or relabeled unless that fact matters scientifically.

## Iteration Pattern

1. First pass: get the structure right.
2. Second pass: improve publication quality.
3. Third pass: verify paper layout and caption fit.
4. Final pass: check body text, caption, and figure terms for exact consistency.
