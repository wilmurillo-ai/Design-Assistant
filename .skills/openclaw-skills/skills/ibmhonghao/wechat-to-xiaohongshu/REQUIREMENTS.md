# 系统要求 - WeChat to Xiaohongshu

## 硬件需求

- **操作系统：** Windows 10+ / macOS 10.15+ / Linux (Ubuntu 18.04+)
- **浏览器：** Chrome 90+ 或 Chromium-based 浏览器（Edge, Brave 等）
- **内存：** 最少 4GB（建议 8GB+）
- **网络：** 稳定互联网连接（大陆内网推荐）

## 软件依赖

### EasyClaw 本体

- **版本：** 0.8.0 或更高
- **组件要求：**
  - ✅ Browser Relay 已激活（OpenClaw 浏览器扩展）
  - ✅ Session 支持（会话管理）
  - ✅ 工具集：`browser`、`exec`（用于控制浏览器）

### 浏览器扩展

- **OpenClaw Browser Relay**
  - 版本：最新版本（自动检查）
  - 状态：**必须在要使用的标签页上激活**（绿色 ON 图标）
  - 获取：从 EasyClaw UI 的扩展管理页面安装

## 账号和访问权限

### WeChat Official Account（微信公众号）

| 项目 | 要求 | 备注 |
|------|------|------|
| 账号类型 | 订阅号 / 服务号 | 任意正常公众号 |
| 后台地址 | https://mp.weixin.qq.com | 需提前登录 |
| 权限 | 发表文章权限 | 能在"已发表"中看到自己的文章 |
| 登录状态 | **必须保持登录** | Session 保活 ≥ 30 天 |

**验证方法：**
```
1. 打开 mp.weixin.qq.com
2. 确认已登录（右上角显示昵称）
3. 能看到"素材管理" > "已发表"列表
```

### Xiaohongshu Creator Platform（小红书创作平台）

| 项目 | 要求 | 备注 |
|------|------|------|
| 账号类型 | 小红书账号 | 需创作者认证（可有可无，但推荐） |
| 平台地址 | https://creator.xiaohongshu.com/ | 需提前登录 |
| 权限 | 发布笔记权限 | 通常新账号需要 1-7 天审核期 |
| 登录状态 | **必须保持登录** | 建议设置"30 天记住我" |

**验证方法：**
```
1. 打开 creator.xiaohongshu.com
2. 确认已登录（右上角显示头像）
3. 能进入 "笔记管理" 或 "发布笔记" 页面
```

### 网络访问

- ✅ 能访问 `mp.weixin.qq.com`（微信后台）
- ✅ 能访问 `creator.xiaohongshu.com`（小红书平台）
- ✅ 能访问外部链接（微信文章的分享链接需可公开访问）
- ⚠️ 如在国外，可能需要 VPN 连接中国内网

## 浏览器配置

### 必需配置

| 设置项 | 值 | 说明 |
|------|-----|------|
| Cookie 和 Session | **启用** | 保持登录状态 |
| JavaScript | **启用** | 页面交互必需 |
| 弹窗 | **允许（有限制）** | 小红书导入会打开弹窗 |
| 跨域请求 | **允许** | 微信→小红书 链接导入 |

### 推荐配置

- ⚠️ **隐私模式：** 不推荐（会丢失登录状态）
- ⚠️ **代理/VPN：** 如使用，确保目标网站可访问
- ✅ **多标签页：** 支持，甚至推荐（分别打开微信和小红书标签页）
- ✅ **暗色主题：** 不影响功能

## 网络要求

| 场景 | 带宽 | 延迟 | 说明 |
|------|------|------|------|
| 标准使用 | ≥ 1 Mbps | ≤ 100ms | 足够完成导入 |
| 大量图片文章 | ≥ 5 Mbps | ≤ 50ms | 图片上传/处理 |
| 批量发布 | ≥ 2 Mbps | ≤ 100ms | 连续多篇操作 |

**网络诊断：**
```
如果导入超时或图片上传失败，可能是：
- 网络不稳定（丢包率 > 1%）
- 访问微信后台慢（IP 被限流）
- 小红书服务器繁忙

解决方案：
1. 切换到 4G/5G（如有）
2. 清空浏览器缓存（F12 → 缓存 → 清空）
3. 等待 1-2 小时再试
```

## 故障排查清单

- [ ] mp.weixin.qq.com 标签页已打开并显示文章列表
- [ ] creator.xiaohongshu.com 标签页已打开
- [ ] 网络连接正常（能打开这两个网站）
- [ ] 浏览器 Cookie 未被禁用
- [ ] 近 30 天未被迫注销（登录 Session 有效）

✅ 全部满足 → 可以开始使用

## 已知限制

### 功能限制

| 限制 | 原因 | 解决方案 |
|-----|------|--------|
| 仅支持桌面浏览器 | 小红书手机版 UI 不同 | 使用桌面版或 iPad 版 |
| 不支持视频文章 | 视频处理需额外权限 | 手动上传视频，文案自动导入 |
| 不支持原创保护 | 小红书 API 限制 | 手动在小红书打开原创保护 |
| 单次最多 10 张图片 | 微信→小红书 图片转换 | 超过 10 张建议分多篇 |

### 性能限制

| 限制 | 数值 | 说明 |
|-----|------|------|
| 单篇文章导入时间 | 30-60 秒 | 含网络延迟 |
| 单次批量发布 | ≤ 10 篇 | 超过会超时 |
| 文章字数上限 | 10,000 字 | 小红书笔记限制 |
| 每日发布建议 | ≤ 5 篇 | 避免被限流 |

### 兼容性说明

**完全支持：**
- ✅ Windows 10/11 + Chrome/Edge
- ✅ macOS 11+ + Chrome/Safari（Webkit relay）
- ✅ Linux (Ubuntu 20.04+) + Chrome

**有限支持：**
- ⚠️ macOS Safari：需额外配置（CORS 设置）
- ⚠️ Firefox：不支持（使用 WebDriver 协议不同）

**不支持：**
- ❌ 移动浏览器（iOS Safari, Android Chrome）
- ❌ 国内 QQ 浏览器、UC 浏览器（UI 不兼容）

## 更新和维护

### 定期检查

- **每月：** 检查 EasyClaw 和 Browser Relay 是否有新版本
- **每周：** 清空浏览器缓存（Settings → Privacy）
- **如需要：** 更新浏览器（Chrome 自动更新，手动检查其他）

### 故障排查

如果技能突然不工作：

```
1. 检查 EasyClaw 是否崩溃（看 Help → Status）
2. 重启浏览器标签页（刷新 F5）
3. 重新激活 Relay（点扩展图标）
4. 如仍不行，重启 EasyClaw 和浏览器
5. 清空缓存 + 重新登录微信和小红书
```
