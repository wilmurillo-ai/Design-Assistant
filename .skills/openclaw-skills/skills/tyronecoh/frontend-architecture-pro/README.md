# 🏗️ Frontend Architecture Pro

> Give your developers solid foundations — CSS systems, layout frameworks, and implementation paths that scale.

**Frontend Architecture Pro** is an OpenClaw Skill that turns architectural decision-making into a repeatable process. Instead of debating grid systems or token naming for the hundredth time, let this skill hand you a battle-tested foundation to build on.

---

## ✨ Features

### CSS Architecture
- Complete design token system (colors, typography, spacing, shadows, transitions)
- Light / dark / system theme with zero-friction toggle
- Mobile-first responsive strategy
- No magic numbers — everything references a token

### Layout Framework
- Container system with 4 breakpoints (sm/md/lg/xl)
- Grid patterns: 2-col, sidebar, auto-fit cards
- Semantic component naming conventions
- BEM-lite hierarchy

### Component Architecture
- 4-layer component hierarchy (Layout → Content → Interactive → Utility)
- Naming conventions that prevent CSS conflicts
- Composition patterns for real-world screens

### Developer Experience
- ThemeManager class — copy/paste into any project
- Handoff specs ready for downstream developer agents
- Performance budgets and accessibility baselines included

## 🔬 Architecture At a Glance

```
Token System
├── Colors (brand + semantic + theme variants)
├── Typography (scale + weight + line-height)
├── Spacing (4px base, 0.25rem increments)
├── Shadows (xs / sm / md / lg / xl)
└── Transitions (fast / normal / slow)

Layout System
├── Container (sm → xl breakpoints)
├── Grid (2-col, sidebar, auto-fit)
└── Section (vertical rhythm)

Component Layers
├── Layout (.container, .grid-*)
├── Content (.card, .avatar, .badge)
├── Interactive (.btn, .input, .modal)
└── Utility (.text-*, .sr-only, .flex)
```
