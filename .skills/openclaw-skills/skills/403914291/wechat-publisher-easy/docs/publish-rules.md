## 📋 核心工作流程

### 步骤 1：读取上次发布记录

```
1. 查找 memory/wechat-scheduled-*.md 最新文件
2. 提取已发布的 15 条新闻标题/内容
3. 存储为去重比对基准
```

### 步骤 2：实时收集新闻

**优先收集方向：**
1. 🌍 国外 AI 新闻（TechCrunch AI, MIT Technology Review, The Verge）
2. 🇨🇳 国内 AI 头条（36 氪，机器之心，量子位）
3. 🏢 国内 AI 公司新闻（百度、阿里、腾讯、字节、小米、华为、京东）

### 步骤 3：去重比对

```
for 每条新新闻 in 收集的新闻:
    if 新闻标题/核心内容 not in 上次发布:
        添加到待发布列表
    else:
        跳过（重复新闻）
```

### 步骤 4：数量验证

```
if 待发布新闻数量 >= 15:
    进入步骤 5
else:
    换方向收集更多新闻
    返回步骤 3
```

### 步骤 5：空数据处理

```
if 待发布新闻数量 == 0:
    换方向收集：
    - 方向 1：国内 AI 政策/行业动态
    - 方向 2：国内 AI 公司融资/产品发布
    - 方向 3：AI 应用落地案例
    返回步骤 3
```

### 步骤 6：最终验证（强制检查）

```
检查清单：
□ 新闻总数 >= 15 条
□ 每条新闻字数 >= 80 字（逐条检查）
□ 四板块结构完整
□ 无重复内容
□ 来源标注清晰

if 全部满足:
    ✅ 进入步骤 7（发布）
else:
    ❌ 返回步骤 2（重新收集/补充）
```

**字数检查工具：**
```powershell
# PowerShell 检查每条新闻字数
$newsList | ForEach-Object {
    if ($_.Length -lt 80) {
        Write-Warning "新闻 '$($_.Title)' 字数不足 80，当前 $($_.Length) 字"
    }
}
```

### 步骤 7：发布公众号

```
1. 生成 HTML 内容（V5 模板）
2. 调用微信 API 创建草稿
3. 保存 DraftID 到结果文件
4. 汇报发布结果
```

## ⚠️ 发布条件（必须全部满足）

| 条件 | 要求 | 说明 |
|------|------|------|
| **新闻总数** | ≥ 15 条 | 四板块合计 |
| **每条字数** | ≥ 80 字 | **核心要求：每条新闻必须包含完整信息** |
| **重复率** | 0% | 与上次发布相比无重复 |
| **内容分类** | 四板块 | 今日摘要 + 国外 AI + 国际科技 + 国内大厂 |
| **信息来源** | 可追溯 | 必须标注来源（TechCrunch、36 氪等） |

### 📝 每条新闻≥80 字 写作规范

**❌ 错误示例（太简单）：**
```
13. 【小米】小爱同学大模型升级
小米 AI 助手接入大模型。
```

**✅ 正确示例（完整信息）：**
```
13. 【小米】小爱同学接入大模型，语音交互更自然
小米宣布小爱同学全面接入自研大模型，语音对话更自然流畅，支持复杂任务规划执行，已推送至小米 14 系列和 MIX Fold 4 等旗舰机型，用户满意度提升 40%。
```

**写作要点：**
1. **标题**：包含核心信息（谁 + 做了什么 + 结果）
2. **正文**：至少 2 句话，包含具体数据/影响/时间/范围
3. **信息完整**：读者看完能理解事件全貌，无需额外搜索

## 🔄 去重规则

### 标题去重
- 完全相同 → 跳过
- 核心关键词相同（80% 匹配）→ 跳过

### 内容去重
- 同一事件不同表述 → 选择信息量更大的
- 同一公司连续发布 → 只保留最新/最重要的

### 时间去重
- 超过 24 小时的旧闻 → 跳过
- 重复报道的同一事件 → 只保留首次发布
