# 修复检查清单

本文档提供网页修复的完整检查清单，确保修复质量和完整性。

## 修复前检查

### 信息收集

- [ ] 获得详细的问题描述
- [ ] 记录复现步骤
- [ ] 明确预期行为
- [ ] 确认实际行为
- [ ] 收集环境信息（浏览器、设备、操作系统）
- [ ] 获取截图或录屏（如果有）

### 问题分类

- [ ] 确定问题类型（样式/功能/性能/兼容性）
- [ ] 评估问题严重程度（严重/中等/轻微）
- [ ] 识别影响范围（所有用户/特定条件/边缘情况）
- [ ] 确认修复优先级

## 根因分析检查

### 控制台检查（2026 增强）

- [ ] 检查所有错误消息（红色）
- [ ] 检查所有警告（黄色）
- [ ] 查看完整的堆栈跟踪
- [ ] 记录错误代码和行号
- [ ] 识别错误模式
- [ ] **新增**：使用 AI 辅助分析错误
- [ ] **新增**：检查 Performance Observer 监控指标
- [ ] **新增**：验证 Core Web Vitals (LCP, FID, CLS)
- [ ] **新增**：检查是否有未处理的 Promise 拒绝

### 元素检查

- [ ] 验证 HTML 结构正确性
- [ ] 检查 DOM 层级关系
- [ ] 确认元素是否正确渲染
- [ ] 查看计算后的样式
- [ ] 检查盒模型（margin、border、padding、width、height）
- [ ] 验证定位（position、top、left、right、bottom）

### 样式分析

- [ ] 检查 CSS 选择器特异性
- [ ] 识别样式覆盖问题
- [ ] 验证样式文件加载顺序
- [ ] 检查 !important 使用
- [ ] 确认 CSS 变量值
- [ ] 验证媒体查询断点

### JavaScript 分析

- [ ] 检查变量作用域
- [ ] 验证函数调用顺序
- [ ] 确认事件监听器绑定
- [ ] 检查异步操作（Promise、async/await）
- [ ] 验证 DOM 加载时机
- [ ] 检查错误处理

### 网络请求检查

- [ ] 验证所有资源加载状态
- [ ] 检查失败的请求（非 200 状态码）
- [ ] 查看请求头和响应头
- [ ] 确认 API 响应格式
- [ ] 检查 CORS 配置
- [ ] 验证资源加载时间

### 性能验证（2026 增强）

#### Lighthouse 评分

- [ ] Performance ≥ 90
- [ ] Accessibility ≥ 90
- [ ] Best Practices ≥ 90
- [ ] SEO ≥ 90

#### 关键指标（Core Web Vitals）

- [ ] First Contentful Paint (FCP) < 1.8s
- [ ] Largest Contentful Paint (LCP) < 2.5s
- [ ] Cumulative Layout Shift (CLS) < 0.1
- [ ] First Input Delay (FID) < 100ms
- [ ] Time to Interactive (TTI) < 3.8s

#### 2026 新增性能检查

- [ ] **Performance Observer API**：实时监控性能指标
- [ ] **Time to First Byte (TTFB)** < 600ms
- [ ] **First Input Delay (FID)** < 100ms（替代 Interaction to Next Paint）
- [ ] **Interaction to Next Paint (INP)** < 200ms（新指标）
- [ ] **Total Blocking Time (TBT)** < 300ms
- [ ] **渲染阻塞资源**：已识别并优化
- [ ] **CLS 调试**：使用 Layout Shift Attribution API
- [ ] **LCP 调试**：使用 Element Timing API

#### 资源优化

- [ ] 图片压缩和格式优化
- [ ] CSS/JS 压缩
- [ ] 启用 gzip/brotli 压缩
- [ ] 使用 CDN
- [ ] 实施缓存策略
