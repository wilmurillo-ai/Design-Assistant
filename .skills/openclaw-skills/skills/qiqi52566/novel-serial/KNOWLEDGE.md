## 主角年龄-能力对照规则

```
示例（系统能力）：
  Lv1（0-20成长度）：仅显示角色状态面板
  Lv2（20-50成长度）：解锁角色魅力判定
  Lv3（50-80成长度）：解锁主角情绪共鸣（影响周围人情绪）
  Lv4（80+成长度）：解锁角色技能合成

⚠️ 故事需要主角有超常能力，必须在 book-bible.md 阶段标注清楚，不能写到一半突然添加。
```

## Book Bible 模板

**必须包含（金手指边界是重点）：**

```
- 核心金手指（一句话）+ 边界/限制
- 主角各阶段年龄和能力（从故事开始到结束）
- 金手指各阶段等级和升级条件
- 主角人设（姓名/背景/性格）
- 前10章主线（每章一句话）
- 核心悬念清单（至少3个，必须在20章内交代）
```

## 新书创建流程（完整版）

当用户说「开始第N本书」时执行：

```
Step 1：检索趋势
  搜索番茄小说Top榜 + 起点月票榜，确认当前热门题材+标签组合

Step 2：生成新书信息卡（发给用户确认后再写）

📋 《{书名}》新书信息卡

书本名称：
目标读者：男频 / 女频
作品标签：
主角名：
作品简介：
```

Step 3：创建书籍目录结构

```
workspace/novels/novel-{N}/
├── book-bible.md              # Book Bible（300-400字）
├── memory/
│   ├── 人物性格.md             # 主角各阶段能力/性格状态
│   ├── 世界观设定.md           # 金手指详细规则+边界
│   └── 伏笔钩子.md             # 埋线记录+回收追踪
├── chapters/
│   ├── ch01.md
│   └── ...
└── meta.json
```

Step 4：写 Book Bible + 前两章（写完自检后发给用户）

## 书名公式（番茄爆款模板）

```
结构1：重生/穿越 + 结果 + 系统/能力
结构2：反差/身份 + 场景/氛围
结构3：身份反转/马甲 + 悬念
结构4：直播/任务 + 震撼结果
结构5：XX后 + YY + 结局
```

检查清单：≤16字 / 含1-2个热搜标签 / 有冲突感或悬念感 / 题材一眼可辨
