# Ingest 规则

> 指导 Agent 如何处理不同类型的资料，生成专业、严谨、深度的知识内容。

## 核心写作原则

### 专家视角与第一性原理思维

**必须遵循的原则**：

1. **第一性原理思维**：
   - 从最基本的原理（物理/数学/经济/心理学）出发
   - 推导概念的本质，而非表面现象
   - 建立因果关系，而非相关性描述

2. **专家角色定位**：
   - 以该领域资深学者/高级工程师的视角撰写
   - 展现深度理解，而非浅层知识罗列
   - 使用专业术语，同时提供精准定义

3. **辩证思维**：
   - 分析优势与局限
   - 比较不同理论的异同
   - 批判性地评价，而非简单接受

4. **严谨性要求**：
   - 提供数学公式和推导（如适用）
   - 引用实验数据和统计显著性
   - 明确假设前提和适用边界

### 写作风格指南

#### ✅ 应该做的

- **深度分析**：解释"为什么"，而不仅仅是"是什么"
- **建立联系**：连接到已有理论、实践、其他概念
- **提供证据**：实验数据、数学证明、实证案例
- **批判思考**：分析局限、挑战、争议点
- **前瞻视角**：讨论未来方向、开放问题

#### ❌ 避免的做法

- **浅层描述**：仅列出要点，缺乏深度分析
- **简单总结**：机械概括，缺少专家洞见
- **缺乏批判**：全盘接受，不分析局限
- **孤立知识**：不建立与其他概念的联系
- **模糊表述**：使用"可能"、"大概"等不确定表达

## 通用原则

1. **提取而非复制**：不要大段复制原文，要提取结构化洞察，并深度分析
2. **连接而非孤立**：每个新概念都要链接到现有概念，建立知识网络
3. **溯源**：每个声明都要能追溯到具体来源
4. **专家视角**：以该领域专家的身份，提供深度见解和批判性分析

## 按资料类型的处理策略

### 学术论文

**提取内容**：
- 研究问题
- 方法创新
- 关键结果
- 局限性和未来方向

**创建页面**：
- 论文同名页面（包含完整信息）
- 方法页面（如果方法是通用贡献）
- 更新相关领域综述页面

### 技术博客/文章

**提取内容**：
- 核心观点
- 代码模式（如果有）
- 实践经验/教训

**创建页面**：
- 技术主题页面
- 模式库（如果是实现技巧）

### 代码仓库

**提取内容**：
- 架构设计
- 关键抽象
- 依赖关系

**创建页面**：
- 项目页面
- 设计模式页面（提取通用模式）

### 笔记/想法

**提取内容**：
- 原始洞察
- 待验证的假设
- 问题/待办

**创建页面**：
- 草稿页面（标记为 draft 状态）
- 或追加到相关现有页面

## 页面创建决策树

```
新概念？
  ├─ 是 → 是否已存在相关页面？
  │        ├─ 是 → 更新现有页面，添加链接
  │        └─ 否 → 创建新页面
  └─ 否 → 更新现有页面
```

## 合并策略

当新资料与现有页面相关时：

1. **补充**：添加新信息到合适段落
2. **对比**：如果新资料有不同观点，添加「不同视角」段落
3. **更新**：如果新资料是更新的版本，替换旧信息并注明
4. **冲突标记**：如果存在矛盾，保留两者并标记 `⚠️ 矛盾：来源X说...，来源Y说...`
