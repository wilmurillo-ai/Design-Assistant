# GTD 个人知识库框架

这个框架把 GTD 的 `Capture → Clarify → Organize → Reflect → Engage`，映射到一个适合 OpenClaw 长期维护的知识系统里。

## 1. 顶层目录

推荐骨架：

```text
00-收集箱
10-工作
20-项目
30-记录
40-学习
50-知识输出
98-归档
99-资源库
```

### 目录职责

| 目录 | GTD 角色 | 用途 |
| --- | --- | --- |
| `00-收集箱` | Capture | 一切未澄清输入的落点 |
| `10-工作` | Organize | 与工作职责直接相关的进行中事项 |
| `20-项目` | Organize | 需要持续推进的独立项目 |
| `30-记录` | Reflect | 日志、灵感、会议、阶段记录 |
| `40-学习` | Clarify / Organize | 对输入材料形成自己的理解 |
| `50-知识输出` | Engage | 文章、分享、成品 |
| `98-归档` | Reflect | 完结或暂停的内容归档 |
| `99-资源库` | Capture / Reference | 原始材料、模板、规范、参考文档 |

## 2. 知识流向

推荐默认流向：

```text
00-收集箱 / 99-资源库
        ↓
    澄清和判断
        ↓
10-工作 / 20-项目 / 30-记录 / 40-学习
        ↓
    提炼与输出
        ↓
      50-知识输出
        ↓
      98-归档
```

### 判断规则

| 问题 | 去向 |
| --- | --- |
| 只是临时想法、待处理碎片？ | `00-收集箱` |
| 是外部文章、视频、资料原文？ | `99-资源库` |
| 是跟岗位职责绑定的推进事项？ | `10-工作` |
| 是需要多步推进的独立主题？ | `20-项目` |
| 是今天发生了什么、想到什么？ | `30-记录` |
| 是我已经消化过的学习笔记？ | `40-学习` |
| 是要对外表达或沉淀成品？ | `50-知识输出` |
| 已完结、不再活跃？ | `98-归档` |

## 4. 标签策略

不要让标签承担目录职责。目录先回答“它处于哪个阶段”，标签再回答“它是什么”。

推荐只保留四类标签：

### 内容类型标签

如：
- `日志`
- `会议`
- `灵感`
- `项目`
- `读书笔记`
- `课程笔记`
- `文章笔记`
- `知识输出`

### 领域标签

如：
- `人工智能`
- `编程`
- `方法论`
- `写作`
- `游戏`
- `职场`
- `历史`
- `心理学`

### 来源标签

如：
- `微信文章`
- `知乎`
- `B站`
- `开源项目`
- `原创`

### 状态标签

只在确实需要时用：

- `待处理`
- `进行中`
- `等待中`
- `已归档`

### 推荐组合

- 学习笔记：`文章笔记 + 领域标签 + 来源标签`
- 项目文档：`项目 + 领域标签`
- 日志：`日志 + 主题标签`
- 输出稿：`知识输出 + 领域标签`

## 5. 命名规则

推荐统一成一眼能看懂的规则：

| 类型 | 命名建议 |
| --- | --- |
| 日志 | `YYYY-MM-DD：周X记录.md` |
| 会议 | `YYYY-MM-DD：会议主题.md` |
| 文章笔记 | `YYYY-MM-DD-文章标题.md` |
| 原始文章 | `来源-标题.md` |
| 规范 | `XX规范.md` 或 `XX体系.md` |
| 模板 | `XX模板.md` |

跨平台同步时，尽量避免文件名里的双引号、尖括号、问号等字符。

## 6. GTD 例行节奏

### 每日

- 清空 `00-收集箱`
- 看当天 TODO
- 处理待阅读队列
- 写当日日志

### 每周

- 回顾 `10-工作` 和 `20-项目`
- 检查 `等待中` 与 `未来/也许`
- 归档已结束的事项
- 更新下周重点

### 每月

- 收敛标签
- 合并重复笔记
- 清理长期闲置项目
- 从 `40-学习` 里挑选可输出内容进入 `50-知识输出`

## 7. 迁移旧知识库时的原则

- 先审计，再迁移
- 先补规范，再大整理
- 先把“收集箱”和“归档”加上，再动中间层
- 如果旧目录已经被高频使用，优先兼容，不要为了漂亮重构到不可用
