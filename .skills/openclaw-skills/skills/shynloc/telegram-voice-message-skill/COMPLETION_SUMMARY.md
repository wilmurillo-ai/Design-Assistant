## 🎯 核心功能

### 1. 音频处理流程
```
文本输入 → TTS生成 → 格式转换 → 发送消息 → 清理
```

### 2. 支持的功能
- ✅ **多TTS服务支持**: 阿里云、OpenAI、Google Cloud
- ✅ **智能格式转换**: WAV/MP3 → OGG (libopus)
- ✅ **错误处理**: 重试机制、超时处理、错误恢复
- ✅ **批量发送**: 支持并行发送多个消息
- ✅ **安全保护**: 环境变量配置、临时文件清理
- ✅ **测试工具**: 集成测试、快速测试

### 3. 解决的核心问题
- ❌ **格式错误**: WAV格式 → ✅ OGG格式 (libopus编码)
- ❌ **参数错误**: 缺少asVoice参数 → ✅ 正确使用sendVoice端点
- ❌ **URL过期**: TTS音频URL过期 → ✅ 立即下载+重试机制
- ❌ **文件太大**: 超过Telegram限制 → ✅ 自动压缩和验证
- ❌ **依赖缺失**: 缺少必需工具 → ✅ 依赖检查和安装指导

## 🚀 使用指南

### 快速开始
```bash
# 1. 设置环境变量
export TELEGRAM_BOT_TOKEN="your_token"
export TELEGRAM_CHAT_ID="your_chat_id"
export ALIYUN_TTS_API_KEY="your_aliyun_key"

# 2. 运行快速测试
./scripts/quick_test.sh

# 3. 发送语音消息
./scripts/tts_generator.sh "你好，世界！" | \
    xargs ./scripts/telegram_sender.sh
```

## 🎖️ 核心规则

### 必须遵守的规则
1. **格式规则**: 必须是OGG格式（libopus编码）
2. **参数规则**: 必须使用`asVoice: true`参数
3. **时间规则**: TTS音频URL立即下载（几秒内过期）
4. **安全规则**: 不硬编码敏感信息

### 必须避免的错误
1. ❌ 发送WAV格式文件
2. ❌ 缺少asVoice参数
3. ❌ 使用caption参数
4. ❌ 延迟下载TTS音频
