# 输入数据格式说明

本文档详细说明 r-ggplot-quickplot skill 的输入数据要求。

## 文件要求

- **格式**: CSV (逗号分隔值)
- **编码**: UTF-8 或 GBK
- **表头**: 必须包含表头行
- **缺失值**: 使用 `NA` 或留空

## 列定义

### 必需列

| 列名 | 数据类型 | 描述 | 示例 |
|------|----------|------|------|
| `x` | 数值 | X轴变量 | 2.3, 3.1, 1.8 |
| `y` | 数值 | Y轴变量 | 5.1, 4.8, 6.2 |

### 可选列

| 列名 | 数据类型 | 描述 | 示例 |
|------|----------|------|------|
| `category` | 文本 | 分类变量 | A, B, C, Control, Treatment |
| `value` | 数值 | 数值指标 | 23, 45, 12 |
| `group` | 文本 | 分组变量 | Control, Treatment |
| `time` | 数值 | 时间点 | 1, 2, 3 |

## 图表自动生成逻辑

脚本会根据数据中存在的列自动选择生成相应的图表：

### 规则 1: 散点图

**条件**: 包含 `x` 和 `y` 列

**生成**:
- `01_scatter_basic.png` - 基础散点图
- `02_scatter_advanced.png` - 带分组着色和大小的散点图
- `09_publication_style.png` - 出版级散点图

### 规则 2: 柱状图

**条件**: 包含 `category` 和 `value` 列

**生成**:
- `03_barplot_vertical.png` - 垂直柱状图
- `04_barplot_horizontal.png` - 水平排序柱状图

**数据处理**: 自动按 `category` 分组计算 `value` 的平均值

### 规则 3: 箱线图

**条件**: 包含 `category` 和 `value` 列

**生成**:
- `05_boxplot.png` - 带数据点的箱线图

### 规则 4: 折线图

**条件**: 包含 `time` 和 `value` 列

**生成**:
- `06_lineplot.png` - 分组折线图

**数据处理**: 自动按 `time` 和 `group` 分组计算 `value` 的平均值

### 规则 5: 直方图

**条件**: 包含 `value` 列

**生成**:
- `07_histogram.png` - 直方图

### 规则 6: 分面图

**条件**: 包含 `category` 列 + (`x`, `y` 至少一个)

**生成**:
- `08_facet.png` - 按分类分面的散点图

## 数据示例

### 示例 1: 最小数据 (仅散点图)

```csv
x,y
2.3,5.1
3.1,4.8
1.8,6.2
4.2,3.9
3.5,5.5
```

### 示例 2: 分类数据 (散点图 + 柱状图 + 箱线图)

```csv
x,y,category,value
2.3,5.1,A,23
3.1,4.8,A,45
1.8,6.2,B,12
4.2,3.9,B,67
3.5,5.5,C,34
```

### 示例 3: 时间序列数据

```csv
time,value,group
1,23,Control
1,45,Treatment
2,12,Control
2,67,Treatment
3,34,Control
3,56,Treatment
```

### 示例 4: 完整数据 (生成所有图表)

```csv
x,y,category,value,group,time
2.3,5.1,A,23,Control,1
3.1,4.8,A,45,Treatment,1
1.8,6.2,B,12,Control,2
4.2,3.9,B,67,Treatment,2
3.5,5.5,C,34,Control,3
2.9,4.2,C,56,Treatment,3
```

## 注意事项

1. **列名大小写**: 列名严格匹配，如 `value` 与 `Value` 不同
2. **数值格式**: 使用小数点 `.`，不要使用逗号 `,`
3. **中文支持**: 支持中文字段名和数据值
4. **日期处理**: 如需日期轴，建议转换为数值 (如天数)

## 数据验证

脚本会自动验证数据：

1. 检查文件是否存在
2. 检查是否为有效的 CSV 格式
3. 检查必需的列是否存在
4. 报告检测到的可用图表类型

## 常见问题

### Q: 我的数据只有一列数值，能生成图表吗？

A: 可以。如果只有 `value` 列，将生成直方图。

### Q: 我的列名不是这些标准名称怎么办？

A: 目前脚本使用固定列名。后续版本将支持自定义列名映射。

### Q: 数据有缺失值怎么办？

A: 脚本会自动跳过缺失值，使用 `na.rm = TRUE` 参数。
