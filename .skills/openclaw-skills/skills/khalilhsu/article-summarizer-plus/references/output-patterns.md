# Output Patterns

Use these patterns to keep summaries consistent and compact.

## Default Summary Pattern

### One-sentence summary
A one-sentence statement of what the piece is about.

### Core points
- 3-6 bullets
- follow the source structure when possible
- keep each bullet concise and factual

### Notable details
Use only when there are genuinely notable details such as:
- numbers
- examples
- surprising claims
- caveats

### Brief takeaway
Keep this short. Focus on what the piece is really trying to say, or who it is useful for.

## Comment Summary Pattern

### Main comment themes
- summarize the main direction of visible comments
- cluster similar opinions together

### Points of disagreement
- note where commenters disagree
- distinguish creator view vs audience pushback

### Representative comment impressions
- mention a few representative themes
- do not overfit to one isolated comment unless it is clearly high-signal

## Confidence Language

Use direct wording when confidence is high:
- “This piece is mainly about…”
- “The core conclusion is…”

Use explicit limitation wording when confidence is partial:
- “The visible body text suggests that…”
- “I could access the title and part of the body, but the later section was not visible, so this summary is based only on the visible portion…”
- “Only the first few visible comments were available, so this comment summary reflects the currently visible discussion…”
- “This conclusion is based on visible page content and should not be treated as a full transcript…”

## Length Control

Default to compact output unless the user explicitly asks for more depth.

- short source -> short summary
- long source -> still prefer a concise summary by default
- expand only for explicit requests such as detailed analysis, deep dive, or section-by-section breakdown
- do not let source length alone force a bloated answer

## What to Avoid

- title-only summaries
- padding with generic filler
- pretending partial content is complete
- mixing your own outside knowledge into the summary
