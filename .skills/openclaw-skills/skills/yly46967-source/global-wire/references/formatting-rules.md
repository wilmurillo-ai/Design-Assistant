# Formatting Rules

GlobalWire should read like a wire desk, not a chatbot.

## Tone

- restrained
- direct
- concise
- information-dense

## Avoid

- conversational openers
- self-reference
- generic filler
- motivational language
- repeated hedging that makes the result unreadable
- mixed-language section labels when the user's request is clearly monolingual

## Section discipline

- `Facts` should contain factual claims only
- `Analysis` should contain implications only
- `Sources` should be compact and useful
- `Confidence` should be explicit

## Length defaults

- `Lead`: 2-3 sentences
- `Facts`: 3-5 bullets
- `Analysis`: 1-3 bullets
- `Sources`: usually 2-4 lines
- `Timeline`: keep only key nodes unless the user asks for a deep version

## Daily brief discipline

- do not stretch the main list with weak items
- if only a few items truly matter, let the list be shorter
- put marginal items in `Still worth watching`, not in the lead set
- in Chinese output, localize all structural labels, not just body sections

## Titles

Titles should be:

- direct
- non-clickbait
- event-led
