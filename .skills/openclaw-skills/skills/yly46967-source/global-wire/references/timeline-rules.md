# Timeline Rules

GlobalWire timelines describe event development, not reporting history.

## Default stance

- build the timeline from the event's start to today
- do not default to the last 7 days unless the user explicitly asks for a short window

## What counts as a timeline node

A node should describe something that happened in the event itself:

- an attack
- a sanctions step
- a policy decision
- a diplomatic breakthrough or breakdown
- a deployment
- a ceasefire move
- a leadership statement that changes the event's direction

Do not use these as primary nodes:

- a media outlet publishing a story
- generic commentary
- broad thematic summaries

## Node writing rule

Each node should answer:

1. what happened
2. why it matters in the arc of the event

Example:

```md
- 2026-02-28: US and Israeli strikes hit Iranian targets, shifting the crisis from coercive pressure into open military confrontation.
```

## Timeline structure

Prefer this order:

1. current state
2. timeline
3. key turns
4. disputed or still developing
5. sources
6. confidence

## Quality rules

- keep the main timeline to key nodes
- move uncertain nodes into `Disputed or still developing`
- do not inflate weak claims into major turning points
- do not write the timeline as if every day must have a node

## When to shorten

Only shorten the window when the user explicitly asks for:

- last 24 hours
- last 7 days
- recent updates
