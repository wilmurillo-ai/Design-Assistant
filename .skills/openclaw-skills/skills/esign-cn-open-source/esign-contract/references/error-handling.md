# e签宝异常处理规范

## 签署人信息不匹配（1437328）

**错误**：`该签署人账号：{手机号}对应用户已存在个人信息，传入的指定签署人信息与已有个人信息不一致`
**原因**：姓名与该手机号在 e签宝平台已注册的实名信息不一致。
**处理**：提示用户 `⚠️ 手机号 {手机号} 已在 e签宝实名注册，传入的姓名「{传入姓名}」与注册信息不一致。请提供该手机号对应的真实姓名，或更换其他手机号。`

## 签署人参数缺失（1435002）

**错误**：`参数错误: 需传入{手机号}账号对应姓名`
**处理**：提示用户补充签署人姓名，需与 e签宝实名信息一致。

## 文件相关错误

- **文件不存在**：检查路径，提示用户确认
- **上传失败（HTTP 非 200/201）**：提示重试，可能是网络问题或文件过大
- **关键字检索无结果**：尝试 `scripts/extract_text.py layout` 布局分析；仍无法定位则 `freeMode: true`

## 认证错误（401）

Token 可能过期，脚本会自动重试（先签名鉴权，再 Token 鉴权）。仍失败则提示用户检查 `.env` 中 `ESIGN_APP_ID` 和 `ESIGN_APP_SECRET`。

## 限流（429）

脚本内置指数退避重试（最多 3 次）。仍失败提示用户稍后再试。

## 撤销相关错误

- **流程状态不允许撤销**：先查询状态告知用户，仅签署中可撤销
- **已签署方未同意**：提示需获得已签署方同意，存在法律风险

## 下载相关错误

- **流程未完成**：脚本本地校验 `signFlowStatus=2`，非已完成时提示用户待全部签署后再下载
- **下载地址获取失败**：提示稍后重试或检查流程是否正常完成

## WeasyPrint 不可用

脚本输出 `WeasyPrint 不可用` 时，说明缺少系统级 C 库（Pango/Cairo）。
- macOS：`brew install pango cairo libffi glib`
- Linux：`apt install libpango-1.0-0 libcairo2 libgdk-pixbuf2.0-0`
安装后重新运行排版命令。**不要尝试替代方案**（reportlab、浏览器渲染等）。

## 通用原则

- API 中文错误信息**直接展示给用户**，不用技术术语替代
- 可恢复错误：引导用户修正后重试
- 不可恢复错误：给出明确排查建议
- **不自行猜测或替换签署人信息**，必须由用户确认
