# Analysis rubric for self taste

Analyze the user's own Douban history carefully and by category.

## Category priority

Respect the user's requested category first.

- For book questions, use books as primary evidence.
- For movie questions, use movies as primary evidence.
- For music questions, use music as primary evidence.
- For game questions, use games as primary evidence.

You may cite small amounts of other categories only as supporting context, not as the main basis.

## Priority slices inside a category

Pay extra attention to these subsets:

1. items with comments
2. high-rated items
3. low-rated items
4. recent items

Treat these as the most informative slices when summarizing taste.

## Evidence strength

Rank signals roughly like this:

1. repeated high ratings across time
2. repeated commented high ratings
3. repeated creators / genres / themes / tags
4. repeated low ratings or repeated complaints
5. recent shifts that appear in several items
6. isolated one-off comments

## What to analyze

### Stable preferences

Look for:

- recurring high-rated genres, creators, themes, or regions
- recurring tags
- recurring praise patterns in comments

### Aversions

Look for:

- recurring low ratings
- repeated criticism themes
- categories or styles that consistently fail

### Recent shifts

Look for:

- recent clusters in tags, creators, moods, or styles
- changes in scoring behavior
- newer interests that do not yet have long history

### Behavior patterns

Look for:

- rating generosity vs harshness
- whether the user comments frequently or sparsely
- whether comments focus on plot, craft, ideas, or emotion

## Recommendation reasoning

When judging whether the user may like a candidate work:

1. summarize the candidate's core traits
2. compare against the requested category first
3. cite matching high-rated or commented history items
4. cite any relevant low-rated counterexamples
5. mention recent trend support if present
6. conclude with confidence

## Good output pattern

1. data scope
2. freshness and cache source
3. strongest positive signals
4. strongest negative signals
5. recent signals
6. conclusion with confidence
