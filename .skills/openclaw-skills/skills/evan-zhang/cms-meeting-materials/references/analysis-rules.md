# AI 分析规范 & 实时模式

## 防幻觉约束（所有分析场景必须遵守）

禁止虚构以下信息：
- 时间信息（会议时间、截止时间、时间节点）
- 人物信息（参会人员、负责人）
- 具体数据（数字、百分比）
- 决策结论

专业术语（药品名、疾病名、公司名）结合上下文判断语音识别错误，无法确定时保持原文。
AI 归纳的框架/分类需标注"AI 根据会议内容归纳"。

## 输出格式

**会议总结**：会议主题 / 讨论要点 / 决策结论 / 遗留问题

**待办提取**：`[待办内容] → [责任人] → [截止时间/优先级]`
- 责任人、截止时间必须严格来自原文，不可推测
- 无法确认的标注 ⚠️ 待确认

**专题分析**：相关摘要 + 涉及发言人（如有）+ 大致时间位置 + 关联决策/待办

进行中会议总结必须告知"这是截至目前的内容，会议仍在进行中"。

## 会中实时模式（v1.10.1+）

触发条件（任一）：
- 用户说"实时跟踪"、"会中监控"
- 查询进行中会议后，可主动询问是否实时跟踪

| 命令 | 说明 |
|---|---|
| `start-stream <meetingChatId>` | 启动实时监控 |
| `stop-stream <meetingChatId>` | 停止监控 |
| `stream-status` | 查看所有监控流状态 |

**实时速记**：每 2-8 秒增量拉取，3 秒内展示。二次转写到达时静默替换旧文本。
**滚动摘要**：每 45 秒更新，包含议题关键词 / 决策记录 / 行动项，基于纯正则+词频（不依赖 LLM）。
**同时监控上限**：≤ 3 场。
**会后自动切换**：检测到会议结束（combineState=2）自动切换，生成完整纪要。

向用户隐藏：轮询间隔、数据库、事件类型、缓存机制、API 字段名、接口调用细节等所有技术过程。

## 约束速查

- 获取原文必须走 `get-transcript.py`，禁止直接调用 4.4/4.10/4.8
- 路由严格：无会议号→4.1（禁止从上下文自动取会议号走 4.11）
- 无结果必须主动引导用户提供会议号
- 时间戳向用户展示时必须转换，禁止展示原始数字
- API 返回业务错误（resultCode≠1）时停止执行，不重试
- 所有 API 字段名全部翻译为人类语言后呈现，不展示 JSON key
