# 记忆与状态管理规则

## 存储路径约定

| 用途 | 路径 | 说明 |
|------|------|------|
| 持久记忆 | `06_memory/YYYY-MM-DD.md` | 按日期归档，跨会话有效 |
| 今日工作记录 | `06_memory/YYYY-MM-DD.md`（追加） | 当天所有决策、问题、规则变更 |
| 全局规则 | `06_memory/RULES.md` | 不过期的工厂硬性规则 |

## 记忆写入规则

**写入 HOT 记忆的条件（任一）**：
- 用户明确纠正了某个行为（"用X不用Y"）
- 用户陈述了持久偏好（"以后都这样做"）
- 同一 Lesson 在 7 天内出现 3 次

**不写入的情况**：
- 单次上下文指令
- 用户沉默（不代表认可）
- 未经验证的假设

## Learning Signals

### 纠正（Corrections）
用户说"不对"/"用这个不用那个"/"停止这样做"：
→ 立即记录到 `06_memory/YYYY-MM-DD.md`
→ 3 次重复后提升为 RULES.md 永久规则

### 偏好（Preferences）
用户说"以后总是..."/"永远不要..."：
→ 直接写入 RULES.md，不需要等待重复

### 反思（Reflections）
有意义的工作完成后，记录：
```
CONTEXT: [任务]
REFLECTION: [发生了什么]
LESSON: [下次怎么改]
```

## 会话状态恢复（Context Recovery）

context 丢失时，按顺序读取：
1. `06_memory/RULES.md`（全局规则）
2. `06_memory/YYYY-MM-DD.md`（今日记忆）
3. SOUL.md（角色定位）
4. USER.md（用户信息）
5. 以上均读完后才询问用户补充缺失信息

**原则**：先自救，不要一上来就问用户"我们上次做到哪里了"。

## 记忆促进/衰退

- 7 天内重复 3 次 → 提升到 RULES.md
- 30 天未引用 → 标记 WARM（保留但不主动加载）
- 90 天未引用 → 归档（移入 archive/）
- 确认的偏好：永不删除，除非用户明确要求
