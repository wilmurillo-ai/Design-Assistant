# DESIGN - create-xgjk-skill

## 1) 目标
- 提供玄关开发者平台 Skill 生命周期能力：发现、创建、发布、更新、下架。
- 新增并固化 CWork 工作汇报协同发送能力：发送前确认、姓名解析、附件稳态上传、确认后发送。
- 不解决：跨系统审批流编排、非开放 API 的内部页面行为模拟。

## 2) 用户体验目标
- 先确认再执行：外发前必须展示发送前确认单并等待“确认发送”。
- 稳定可回执：发送后可给出回执信息（结果码/ID）并支持后续核对。
- 失败兜底：
  - fail-soft 用于“草稿准备/确认阶段”（可继续修正）；
  - strict 用于“发送阶段”（未确认或字段不齐直接阻断）。

## 3) 核心流程
- 发现/创建/发布主流程（既有）。
- CWork 汇报扩展流程：
  1) `prepare` 生成发送前确认单（标题/正文/To/CC/建议人/审批人/附件）；
  2) 姓名解析（精确/单候选/别名纠偏/候选回显）；
  3) 附件存在性校验；
  4) 人工确认后 `send`；
  5) 先上传附件 `uploadWholeFile`，再提交 `report/record/submit`（`fileVOList=fileId+name+type`）。

## 4) 风险与防护
- 误发风险：通过“确认口令 + readyToSend + 未解析阻断”控制。
- 姓名歧义风险：候选回显，不做盲发。
- 附件失败风险：采用已验证稳定链路，避免提交时携带不必要字段。
- 平台波动风险：对 `resultCode=200000` 做有限重试。

## 5) 版本策略
- 当前版本：v1.0.7 基础上扩展 work-report 流程（本轮改造）。
- 升级原则：先兼容现网接口，再加能力；破坏性变更需先确认。
