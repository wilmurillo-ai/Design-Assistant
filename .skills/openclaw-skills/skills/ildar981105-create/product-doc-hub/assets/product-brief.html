Tideo 产品逻辑文档

最后更新：2026-04-15 · 版本：visual1（视觉稿还原版）

为什么做这个产品

腾讯云旗下有云点播、云直播、媒体处理（MPS）等多个音视频产品，它们之间存在大量公用功能（字幕擦除、翻译、配音、剪辑等）。如果分别嵌入各个产品内部，开发成本高、功能冗杂。

因此决定做一个统一平台，把这些跨产品的能力集合在一起使用。

立项有三个核心原因：

1. 能力整合

云点播、云直播、MPS 各自有视频处理能力，但分散在不同产品线。用户要用"视频译制"需要同时调 MPS 的擦除、翻译、配音三个接口，操作分散。Tideo 把它们整合成一键流程。

2. 开箱即用

中小企业和内容团队有直接使用这些能力的需求，但未必具备 API 对接的开发能力。Tideo 提供零代码、可视化的操作界面，上传视频就能用，不需要写一行代码。

3. 简化账号管理

腾讯云的账号体系需要多处授权（CAM 策略、服务角色、跨产品权限），流程繁琐。Tideo 用一套轻量账号体系统一管理，降低使用门槛。

产品是什么

Tideo 是腾讯云音视频能力的轻量化统一平台 — 整合云点播、云直播、MPS 的公用功能，提供开箱即用的可视化操作界面。
首期聚焦三大核心功能：视频译制 · 视频生成 · 直播剪辑，后续持续扩充。

功能能力来源用户价值状态
视频译制MPS 字幕擦除 + 翻译 + AI 配音上传视频，自动输出多语言版本✅ 已跑通

用户是谁

  • 中小企业 — 没有技术团队，无法自己对接 API，需要开箱即用的音视频处理工具
  • 短剧出海团队 — 中文短剧翻译成英语/日语/韩语发行海外，需要高效批量处理
  • 内容创作者 — 把视频翻译成多语言版本，扩大海外受众
  • 企业市场部 — 产品宣传片、培训视频的多语言本地化
  • 直播运营团队 — 需要快速从直播流中剪辑精彩片段分发

视频译制 — 核心流程

首期已跑通的完整流程：

打开产品 → 上传视频 → 选择模式 → 等待处理 → 查看成片 → 精调（可选） → 下载

处理模式

模式包含能力适用场景
完整模式（full）字幕擦除 + 字幕翻译 + AI 配音完整的视频本地化
字幕擦除（erase）仅擦除画面中的字幕和水印清理画面
字幕翻译（subtitle）识别语音 + 翻译 + 字幕压制只要字幕，不改配音
配音（voice）语音识别 + 翻译 + AI 克隆配音只要换配音

步骤页面用户做什么系统做什么
1. 进入create.html打开产品首页展示功能卡片和智能输入框

体验设计 — 产品核心差异

传统媒体处理产品的等待体验：提交任务 → 进度条/空白页 → 完成通知。用户在等待期间无事可做，只能离开页面。

Tideo 的做法：把原本无聊、不可视的媒体处理过程，变成一个可交互的游戏沙盘。用户在等待期间能看到"工作室里正在发生什么"，增加趣味性和参与感，降低感知等待时间。

这是 Tideo 区别于所有同类产品的核心体验差异。

四人虚拟工作坊

处理过程被具象化为一间像素风工作室，四个 AI 角色各司其职：

角色真名职责对应处理阶段
🎬 导演林雨晨统筹分工、输出结果任务调度 + 最终合成
🎨 后期陈默画面处理字幕/水印擦除
📝 翻译李明远字幕识别和翻译OCR/ASR + 翻译
🎙️ 配音苏雅AI 克隆配音配音合成

等待期间用户能看到什么

  • 角色实时对话 — 四个角色在聊天面板轮流汇报进度，有性格、有台词，不是机械的状态文字
  • 进度卡片 — 每个角色下方有子任务卡片，逐条从「排队中」→「处理中」→「完成」
  • 像素工作室场景 — 等距视角的工作室画面，角色在各自工位上忙碌
  • 节奏同步 — 所有视觉反馈与真实 MPS 处理进度完全同步，不是假动画

设计目标

  • 降低用户在处理等待期间的流失率
  • 让"等待"本身变成产品体验的一部分，而不是体验的断裂
  • 建立用户对 AI 处理过程的信任感（看得见每一步在做什么）

技术架构

前端

  • 纯 HTML/CSS/JS，无框架，轻量快速，托管在 GitHub Pages
  • COS JS SDK 前端直传（不经过服务器）
  • api-client.js 封装所有 API 调用
  • tracker.js 全自动埋点（用户测试用）

后端

  • 腾讯云 SCF 云函数（tideo-api）— 唯一的后端，约 150 行
  • 职责：API 签名代理（前端不暴露 SecretKey）
  • 路由：/translate · /erase · /clip · /task/:id · /upload-credential · /track · /track-data

云服务依赖

服务用途
腾讯云 MPS视频处理引擎（擦除+翻译+配音）
腾讯云 COS视频/成片/埋点数据存储
腾讯云 STS生成临时上传凭证（前端安全直传）
GitHub Pages前端静态托管

数据流

用户选视频 → COS 直传（前端 → COS）
→ 带 videoUrl 跳转到 translate-v8
→ SCF 云函数调用 MPS ProcessMedia
→ 前端每 5 秒轮询 DescribeTaskDetail
→ MPS 完成后成片写入 COS
→ 前端拿到成片 URL，播放器 + 下载
