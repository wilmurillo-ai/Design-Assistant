# Product Discovery Template — 模式 C：产品发现报告

追踪特定平台/生态的破圈产品时使用此模板。

```markdown
# {平台名} 产品发现 — {YYYY-MM-DD}

> 扫描来源：GitHub / HackerNews / 官方市场 / X / YouTube | 发现产品数：{N}

---

## 最出挑的产品

### 🥇 {产品名}

- **一句话**：{做什么的}
- **数据**：⭐ {stars} | 📥 {downloads} | 👥 {用户数}
- **最近更新**：{YYYY-MM-DD}
- **链接**：[GitHub]({url}) | [官网]({url}) | [Demo]({url})
- **为什么破圈**：{2-3 句——解决了什么痛点，增长速度如何异常}
- **社区评价**：
  - [{HN 讨论标题}]({url}) — {N} points
  - [{X 帖子摘要}]({url})

### 🥈 {产品名}

{同上格式}

### 🥉 {产品名}

{同上格式}

---

## 值得关注但还没破圈的

| # | 产品 | 描述 | Stars/下载 | 链接 | 潜力判断 |
|---|------|------|-----------|------|---------|
| 1 | {name} | {一句话} | {N} | [{url}]({url}) | {为什么值得关注} |
| 2 | {name} | {一句话} | {N} | [{url}]({url}) | {为什么值得关注} |
| 3 | {name} | {一句话} | {N} | [{url}]({url}) | {为什么值得关注} |

---

## 生态趋势

{基于以上发现，总结这个平台/生态的整体方向：}

- **主流方向**：{大多数产品在做什么}
- **新兴方向**：{少数但增长快的方向}
- **空白机会**：{还没人做但有需求的方向}

---

## PM 启示

1. **值得集成/接入**：{具体产品} — {理由}
2. **值得借鉴的设计**：{某产品的某个设计} — {为什么好}
3. **竞品预警**：{某产品可能与你的方向重叠} — {建议动作}
```

## "破圈"判断标准

| 信号 | 权重 | 怎么验证 |
|------|------|---------|
| GitHub stars 周增速 > 100% | 高 | 看 star-history 或创建日期 vs 当前 stars |
| HackerNews 首页 ≥ 200 points | 高 | HN API 搜索 |
| 非技术媒体报道（TechCrunch、Reuters 等） | 高 | Web 搜索 |
| X/Twitter 上有非开发者讨论 | 中 | 搜索 site:x.com |
| ProductHunt 上榜 | 中 | Web 搜索 "{name} site:producthunt.com" |
| YouTube demo 视频播放量 > 10k | 中 | YouTube 搜索 |
| 在其他热门项目的 README 中被引用 | 低 | GitHub 搜索 |
