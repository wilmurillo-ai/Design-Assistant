# Aristotle Reference

Goal Clarifier is built on four Aristotle-inspired ideas.

## 1. Telos before mechanism

An action is easier to judge when its end is explicit. In AI product work, people often name the mechanism first: dashboard, agent, workflow, automation, memory system. This skill reverses that order and asks what purpose the mechanism serves.

## 2. Classification stabilizes reasoning

Aristotle treats disciplined classification as a precondition for sound thought. Goal Clarifier uses that idea by separating:

- outcome from implementation
- constraints from preferences
- blocking ambiguity from working assumption
- human-owned judgment from machine-owned execution

## 3. Function matters more than material

A thing is not understood only by what it is made of. It is understood by what it is for. In AI terms, listing tools and APIs is not enough. A useful brief must explain the function the system is supposed to perform.

## 4. Practical wisdom is contextual

Phronesis means good judgment inside a real situation. That is why Goal Clarifier does not treat every request as a generic form-filling exercise. It asks pointed questions shaped by domain, risk, and reversibility.

## Design implication

The skill exists to prevent a common failure mode: building a polished solution for the wrong problem.
