# Question Patterns

Use the smallest set of questions that can complete the brief.

## Product or agent request

- What decision or action should become easier if this exists?
- Who is the primary user?
- What would count as success in the first 30 days?

## Dashboard or reporting request

- Who will look at it?
- What decision should they make differently because of it?
- What existing pain does it replace?

## Automation request

- What human task is currently being done?
- Which part is repetitive, and which part still requires judgment?
- What mistakes are unacceptable?

## Optimization request

- What is underperforming right now?
- Compared to what baseline?
- What is the strongest constraint on improvement?

## High-risk workflow

- Which decision must stay human-owned?
- What evidence is required before action affects a real person?
- What would a harmful false positive or false negative look like?

## Closing rule

Once the essentials are present, stop asking and produce the six-part brief.
