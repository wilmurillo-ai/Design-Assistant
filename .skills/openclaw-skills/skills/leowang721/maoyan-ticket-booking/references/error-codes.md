# 错误码详细说明

## 错误码对照表

| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 1000 | 通用错误 | 向用户解释错误信息，提供重试或兜底选项 |
| 1001 | 缺少必要字段 | 提示用户补充缺失的信息 |
| 1002 | Token 无效 | 引导用户重新获取 Token，或提供猫眼兜底 |
| 1003 | Token 过期 | 告知用户 Token 已过期，重新获取或跳转猫眼 |
| 1004 | 座位已被占用 | 说明情况，提供重新选座或切换场次 |
| 1005 | 场次已满 | 推荐同影院其他场次或附近影院 |
| 1006 | 订单已关闭 | 说明支付超时，座位已释放，引导重新开始 |
| 1007 | 出票中 | 告知用户正在处理，请稍后再查 |
| 1008 | 出票失败 | 解释支付成功但出票异常，提供重试或支持 |
| 1009 | 网络超时 | 建议检查网络，稍后重试 |
| 1010 | 系统维护 | 告知系统正在维护，提供猫眼兜底或稍后重试 |

## 兜底场景处理

### 座位不满意
- 提供同场次的另一组座位
- 提供同影院的其他场次
- 提供附近其他影院
- 提供猫眼相关页面兜底跳转

### 未提供 Token 或拒绝提供 Token
- 允许继续浏览影院、电影和场次摘要
- 不要进入座位图、选座、订单或状态查询步骤
- 如果用户仍希望继续购买，提供猫眼兜底

### 支付超时
- 说明订单可能已关闭，座位可能已释放
- 提供重新开始购票流程的引导
- 可以推荐同场次或其他场次作为替代

### 创建订单失败（1004/其他）
- 检查错误信息，判断是否为参数错误：
  - `seatNo` 是否使用了接口返回的原始值（不同影院格式不同）
  - `sectionId` 是否正确
  - `rowId`、`columnId` 是否与 `seatNo` 匹配
- 如确认参数错误，修复后重试创建订单
- 如参数正确但仍失败，建议用户换场次或换影院

### 出票失败
- 明确解释支付和出票是两个不同阶段
- 如适合，建议用户稍后再次查询（5-10分钟后）
- 如果有可用支持流程，转入支持或兜底流程
- 提供猫眼相关页面作为替代方案

### 用户中途放弃
- 确认用户是否真的要放弃
- 如果确认，提供猫眼相关页面链接，方便用户后续自行购买
- 记录当前进度，方便用户后续恢复

### 网络异常
- 告知用户网络问题
- 建议检查网络连接
- 提供稍后重试的选项
- 如持续失败，提供猫眼兜底
