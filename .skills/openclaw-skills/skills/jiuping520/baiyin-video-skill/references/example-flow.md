# 执行流程示例

以下是两个完整交互示例，展示正确的步骤顺序和交互方式。

---

## 示例 1：文生视频（可灵）

**用户输入：** "用可灵生成海边日出"

**第 1 步** — 静默执行 `fetch-config.sh`，无输出。

**第 2 步** — 用户已指定"可灵"，预选 model_code:"kling-video"，跳过模型选择。无输出。

**第 3 步** — 执行 `model-detail kling-video`，得到 types: ["文生视频","首尾帧","多图参考","视频参考"]。弹出选择器：

```
[选择器 1] 生成类型: 文生视频 / 首尾帧 / 多图参考 / 视频参考
[选择器 2] 时长: 5秒 / 10秒
[选择器 3] 分辨率: 720P / 1080P
[选择器 4] 画幅: 16:9 / 9:16 / 1:1
```

用户选择：文生视频、5秒、1080P、16:9。执行 `match kling-video 文生视频 5 1080 16:9`。

**第 4 步** — 文生视频无需媒体，跳过。

**第 5 步** — "海边日出"较薄弱，直接弹出润色选择器（无前置文字）：

```
[选择器] header: 提示词 / question: 选择提示词版本
  1. 使用原始版 — "海边日出"
  2. 使用润色版 — "金色朝阳从海平线缓缓升起，天空染成橙红色，波光粼粼的海面倒映着霞光，镜头缓缓推进"
```

用户选择润色版。

**第 6 步** — 弹出配置摘要：

```
模型：可灵视频3.0 Omni
生成模式：文生视频
提示词："金色朝阳从海平线缓缓升起..."
画面比例：16:9
分辨率：1080P
时长：5 秒

选项：确认提交 / 修改参数 / 取消
```

用户确认。

**第 7 步** — 执行 `submit-task.sh`，展示 taskId 和 requestId。

**第 8 步** — 每 30 秒轮询，直到 succeeded 返回视频 URL。

## 示例 2：首尾帧（PixVerse，last_frame: 1 可选）

**用户输入：** "帮我生成一个视频"

**第 1 步** — 静默执行 `fetch-config.sh`。

**第 2 步** — 未指定模型，6 个模型。输出编号列表：

```
1. 可灵视频3.0 Omni — 支持文生视频/首尾帧/多图参考/视频参考
2. PixVerse视频生成 — 文生视频/首尾帧，简单直接
3. Vidu视频Q3 — 速度快，长提示词
...
```

用户回复"2"。

**第 3 步** — 执行 `model-detail pixverse-video`，得到 types: ["文生视频","首尾帧"]。弹出选择器：

```
[选择器 1] 生成类型: 文生视频 / 首尾帧
[选择器 2] 时长: 5秒 / 8秒 / 10秒
[选择器 3] 分辨率: 360P / 540P / 720P / 1080P
[选择器 4] 画幅: 16:9 / 1:1 / 9:16
```

用户选择：首尾帧、5秒、720P、16:9。执行 `match pixverse-video 首尾帧 5 720 16:9`，输出含 `last_frame: 1`。

**第 4 步** — `last_frame: 1`（可选），输出：

```
请提供起始帧图片 URL（如需结束帧，可用换行/空格/逗号/分号分隔添加）：
```

用户回复：`https://example.com/start.jpg`（仅提供起始帧）。

**第 5 步** — 用户未描述场景（只说了"帮我生成一个视频"），主动询问提示词。用户回复"海边日出"，薄弱，弹出润色选择器。

**第 6 步** — 摘要确认。

**第 7 步** — 提交。

**第 8 步** — 轮询。

---

## 关键要点

- 第 1-2 步之间、第 2-3 步之间**不输出任何文字**
- 生成类型**必须展示所有可用类型**，不能只列一种
- 提示词在第 5 步收集，**绝不能**提前到第 3 步之前
- 媒体收集的 `last_frame` 值从 `match` 输出读取，不猜测
