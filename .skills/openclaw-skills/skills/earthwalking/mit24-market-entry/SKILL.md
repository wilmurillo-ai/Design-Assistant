# MIT有序创业24步法 - 阶段四：市场进入策略

第四阶段聚焦于"如何进入市场"。在定义好产品和定价之后，需要建立可重复、可扩展的获客和销售体系，并通过科学实验验证商业假设。本阶段包含步骤15-18。

## 前置条件

建议先完成前三个阶段，确保产品定义清晰后再设计市场进入策略。

## 工作流程

### 步骤 15: 计算获客成本（CAC）

量化获取一个新客户的成本，评估市场进入的经济可行性。

**CAC计算公式：**
```
CAC = (营销总成本 + 销售总成本) / 新增客户数量
```

**成本构成：**
- 营销成本：广告投放、内容营销、品牌建设、市场活动
- 销售成本：销售人员薪酬、销售工具、差旅费用

**关键健康指标：**
- CLV/CAC比率 > 3（客户终身价值至少是获客成本的3倍）
- CAC回收期 < 12个月（获客投入在一年内收回）
- 毛利率 > CAC/CLV（确保利润空间覆盖获客成本）

**CAC优化方向：**
- 缩短销售周期
- 提高转化率
- 优化渠道组合
- 利用口碑和推荐

**输出要求：** CAC计算结果、CLV/CAC健康度评估、CAC优化建议。

### 步骤 16: 设计销售流程

建立可重复、可扩展的销售流程，使获客系统化而非依赖个人能力。

**七阶段销售流程：**
1. 潜在客户开发（Lead Generation）：如何发现和吸引潜在客户
2. 资格审查（Qualification）：如何判断客户是否值得跟进
3. 需求分析（Needs Analysis）：如何深入了解客户需求
4. 方案展示（Presentation）：如何展示产品价值和方案
5. 异议处理（Objection Handling）：如何应对客户疑虑
6. 成交（Closing）：如何促成交易
7. 售后跟进（Follow-up）：如何维护客户关系、促进复购和推荐

**每个阶段需明确：**
- 关键活动和工具
- 转化率基准
- 里程碑和判断标准
- 需要准备的物料

**B2B vs B2C销售流程差异：**
- B2B：决策链条长，需要多角色覆盖，重视试用和POC
- B2C：决策快，强调即时转化，重视用户体验和评价

**输出要求：** 销售流程图 + 各阶段的转化率目标 + 关键销售话术/物料清单。

### 步骤 17: 识别关键假设

识别商业模式中最危险的假设，这些假设如果被证伪，整个商业模式将不成立。

**假设类型清单：**
- 需求假设：客户真的有这个痛点吗？紧迫程度如何？
- 市场容量假设：市场真的有那么大吗？
- 产品功能假设：这些功能真的能解决问题吗？
- 定价假设：客户真的愿意付这个价格吗？
- 渠道假设：真的能通过这个渠道触达目标客户吗？
- 获客成本假设：真的能以这个成本获取客户吗？
- 单位经济假设：毛利率真的能达到这个水平吗？

**假设优先级排序：**
1. 最致命的假设（如果错了，一切都错）优先
2. 最不确定的假设（团队内部分歧最大的）优先
3. 最容易验证的假设（低成本快速测试）优先

**输出要求：** 关键假设清单，按优先级排序，标注每个假设的致命程度和验证成本。

### 步骤 18: 测试关键假设

通过科学实验验证步骤17中识别的关键假设。

**常用验证方法：**

| 方法 | 适用场景 | 成本 | 速度 |
|------|----------|------|------|
| 客户访谈 | 验证需求假设 | 低 | 快 |
| 问卷调查 | 验证需求规模 | 低 | 中 |
| Landing Page测试 | 验证价值主张和定价 | 低 | 快 |
| A/B测试 | 对比不同方案效果 | 中 | 快 |
| 预售/候补名单 | 验证购买意愿 | 低 | 中 |
| 假门测试 | 验证功能需求 | 低 | 快 |
| 小规模试点 | 验证完整商业模式 | 高 | 慢 |

**实验设计框架：**
1. 明确假设（Hypothesis）
2. 定义成功标准（Success Metric）
3. 设计最小实验（Minimum Viable Experiment）
4. 执行并收集数据
5. 分析结果并做出决策

**输出要求：** 假设验证实验计划表，包含假设、验证方法、成功标准、预期时间。

## 输出格式

完成4个步骤后，生成《市场进入策略报告》，包含：
1. CAC计算与健康度评估
2. 销售流程图与转化率目标
3. 关键假设清单（按优先级排序）
4. 假设验证实验计划表

## 实用建议

- 步骤17-18是MIT 24步法中最关键的一步——很多创业失败源于未验证的假设
- 先验证最致命的假设，即使它不是最容易验证的
- 不要因为1-2个客户的正面反馈就停止验证——需要统计显著性
- 如果用户已有运营数据，可以用实际数据来验证或修正假设
