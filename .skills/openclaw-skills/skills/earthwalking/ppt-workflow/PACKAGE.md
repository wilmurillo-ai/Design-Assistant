## 📋 技能简介

基于标准化工作流的 PPT 制作技能，从内容搜索到美化优化的全流程自动化。

**核心特点**:
- ✅ 7 阶段工作流自动化
- ✅ 智能模型路由 (Qwen3.5 + Kimi 2.5 + Minimax 2.5)
- ✅ 学术规范支持
- ✅ 多种模板选择 (5/15/30/60 分钟)
- ✅ 质量检查清单
- ✅ 完整交付包

## 🚀 快速开始

### 安装

```bash
# 通过 ClawHub 安装
openclaw skills install ppt-workflow

# 或手动安装
openclaw skills install ./ppt-workflow-v1.0.zip
```

### 使用

```
帮我做一个 PPT，主题是：[你的主题]
```

**示例**:
```
帮我做一个 PPT，主题是：AI 在心理学研究中的应用
- 时长：15 分钟
- 受众：学术会议
```

## 📊 工作流

| 阶段 | 模型 | 时间 | 任务 |
|------|------|------|------|
| 需求确认 | Qwen3.5 | 1 分钟 | 确认时长、受众、核心信息 |
| 内容搜索 | Kimi 2.5 | 30 分钟 | 文献搜索、资料整理 |
| 框架设计 | Qwen3.5 | 20 分钟 | 结构设计、故事线 |
| 内容填充 | Qwen3.5 | 50 分钟 | 学术写作、图表 |
| 幻灯片制作 | Qwen3.5 | 40 分钟 | PPT 创建、格式 |
| 美化优化 | Minimax 2.5 | 20 分钟 | 视觉优化、文案 |
| 输出交付 | Qwen3.5 | 10 分钟 | 质量检查、导出 |

**总计**: 2.5-4 小时 (取决于报告时长)

## 💡 最佳实践

### ✅ 推荐做法

1. **提前规划**: 至少提前 1 天开始
2. **明确需求**: 时长、受众、核心信息
3. **提供资料**: 相关文献、数据、图片
4. **及时反馈**: 24 小时内给出修改意见
5. **多次排练**: 至少排练 3 次

### ❌ 避免错误

1. **临时抱佛脚**: 提前 1 小时才说要做 PPT
2. **需求模糊**: "随便做一个"
3. **反复修改**: 修改超过 5 次
4. **不提供资料**: 所有材料都要 AI 找
5. **不排练**: 直接上台演讲

**Q: 支持哪些输出格式？**

A: PPTX (可编辑) + PDF (分享) + PDF with notes (演讲备注)
