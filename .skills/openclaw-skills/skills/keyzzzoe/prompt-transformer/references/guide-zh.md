# @prompt v1.0 正式版（中文说明）

这是一个把自然语言需求转换成**更适合 AI 理解和执行的 Prompt** 的 OpenClaw Skill。

## 这个 Skill 是做什么的？

你只需要在消息前加上：
- `@prt`
- `@prompt`

然后输入你的白话需求，它会帮你把需求整理成一段更结构化、更可直接复制使用的 Prompt。

适合场景：
- 写代码
- 做原型
- 写产品方案
- 查资料 / 做研究型整理
- 复杂任务拆解

## 它怎么用？

### 基本写法

```text
@prt 帮我写一个 Python 脚本，批量重命名文件
@prompt 帮我设计一个面向独立开发者的登录页原型
@prt帮我查找近五年管理学论文，并给我硕士选题思路
```

支持：
- `@prt帮我...`
- `@prt 帮我...`
- `@prt    帮我...`

## 它会怎么工作？

这个 Skill 会根据输入自动分成三类处理：

1. **Transform**
   - 把你的需求转成结构化 Prompt

2. **Clarify**
   - 如果关键信息不够，就先追问最少的问题

3. **Bypass**
   - 如果只是打招呼、天气、闲聊，就直接正常回复

## 主要特性

- 支持 `@prt` / `@prompt`
- 支持无空格、单空格、多空格触发
- 会避免误判类似 `@promptify...` 这种相似前缀
- 简单任务不会过度包装
- 模糊任务会克制地澄清
- 普通问答会自然旁路
- 生成后默认收住，不乱开 1 / 2 / 3 分支
- `Prompt 关键处理` 默认只写 2 条，复杂任务最多 3 条

## 输出风格

默认输出：
- 一段可直接复制使用的 Prompt
- 一个简短的 `Prompt 关键处理` 摘要

不会再默认输出：
- 很长的提示词教学解释
- 内部路由术语
- 多余的系统腔说明

## 调试与迭代记录（摘要）

这个版本主要修过这些问题：
- 生成后话题粘连
- 旁路时暴露内部术语
- 结尾过长、主动给 1/2/3 选项
- `@prt帮我...` 附着触发边界
- 澄清问题问得太多
- 错把 `@prt` 请求当成真实执行请求

更详细记录见：
- `debug-history.md`

