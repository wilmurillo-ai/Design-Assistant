# TransPrompt｜一句指令，把需求变成可直接发给 AI 的专业 Prompt

TransPrompt 是一个给 **OpenClaw** 用的 Prompt 转换 Skill。

它解决的不是“AI 不够聪明”，而是很多人**知道自己想做什么，却不会把需求写成 AI 最容易理解的 Prompt**。

所以它做的事情很直接：

- 你输入一句白话需求
- 前面加上 `@prt` 或 `@prompt`
- 它帮你整理成一段更专业、更清晰、可直接复制使用的 Prompt

## 为什么值得装？

### 1. 一句指令，随手调用
不用打开额外工具，不用切换页面，不用学习复杂格式。

直接这样写就行：

```text
@prt 帮我写一个 Python 脚本，批量重命名文件
@prompt 帮我设计一个面向独立开发者的登录页原型
@prt帮我查找近五年管理学论文，并给我硕士选题思路
```

支持：
- 无空格触发
- 单空格触发
- 多空格触发

### 2. 不只是 Prompt 生成，更重要的是“该转才转”
很多类似工具的问题是：
- 什么都想转
- 什么都想套模板
- 最后用户只会觉得啰嗦

TransPrompt 的核心差异在于：

- **需要 Prompt 转换时**，它会帮你整理出可直接使用的 Prompt
- **不需要转换时**，它会直接正常回复

比如：
- `@prt 哈喽`
- `@prt 明天天气怎么样？`
- `@prt 你在吗`

这些都不会被硬塞成 Prompt 模板。

### 3. 输出克制，真的能直接拿去用
默认输出只有两部分：

1. **可直接复制使用的 Prompt**
2. **简短的 `Prompt 关键处理` 摘要**

不会默认给你：
- 冗长的提示词理论课
- 一大堆内部机制说明
- 多余的 1 / 2 / 3 选项菜单

### 4. 经过多轮真实测试，不是一次性写完就发
这个版本不是拍脑袋发的。

它经历过：
- 多轮真实聊天测试
- 脏输入回归测试
- 发布前 smoke test
- 多次边界问题修正

重点修过的问题包括：
- 生成后话题粘连
- 旁路时冒出内部术语
- 结尾过长、乱开分支
- `@prt帮我...` 的附着触发边界
- 澄清问题问太多
- 把 `@prt` 请求误当成真实执行请求

## 它适合谁？

适合这些人：
- 不擅长写 Prompt 的普通用户
- 想把需求更清晰地交给 GPT / Claude / Gemini / Cursor / Claude Code 的人
- 想减少来回试错成本的人
- 想在聊天里“随手调用” Prompt 转换，而不是专门开一个 Prompt 工具的人
