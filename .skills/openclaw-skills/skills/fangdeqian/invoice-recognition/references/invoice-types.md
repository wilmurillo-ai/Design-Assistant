## 分类体系概述

### 按发票性质分类

| 类别 | noteType | 说明 |
|-----|---------|------|
| 增值税发票 | 1 | 可抵扣税款的发票，含税额信息 |
| 非增值税发票 | 2 | 不可抵扣税款的发票 |

### 按票据介质分类

| 介质类型 | 说明 |
|---------|------|
| 纸质发票 | 传统印刷发票，需要扫描识别 |
| 电子发票 | PDF/OFD/XML格式，可直接解析 |
| 区块链发票 | 基于区块链技术，不可篡改 |

## 增值税发票

### 增值税专用发票

**编码**：1（noteType=1，发票代码第1位=0）

**特征**：

- 购买方和销售方信息完整
- 包含税号、地址、电话、银行账号
- 有税额信息，可用于进项抵扣
- 通常为防伪税控系统开具

**识别要点**：

- 发票标题包含"增值税专用发票"
- 发票代码12位，第1位为0
- 有"税额"行
- 购买方和销售方信息完整

### 增值税普通发票

**编码**：2（noteType=1，发票代码第1位=1）

**特征**：

- 与专用发票格式相似
- 税额不可抵扣（购买方为个人或小规模纳税人）
- 可作为入账凭证

**识别要点**：

- 发票标题包含"增值税普通发票"
- 发票代码12位，第1位为1
- 有"税额"行
- 价税合计金额明确

## 普通发票

### 通用机打发票

**编码**：3（noteType=2）

**特征**：

- 非增值税发票
- 由各省税务机关监制
- 格式多样，不同地区有差异

**识别要点**：

- 无税额信息
- 通常没有税号字段
- 金额直接显示价税合计

### 增值税电子普通发票

**编码**：4

**特征**：

- 电子格式，可通过邮箱、短信接收
- 与纸质普通发票同等效力
- 可打印使用

**识别要点**：

- 格式与纸质普通发票一致
- 文件格式为PDF或OFD
- 有电子发票专用章

## 区块链电子发票

### 区块链电子发票（通用）

**编码**：1009

**特征**：

- 基于区块链技术开具
- 防伪性能强，不可篡改
- 开票速度快，即开即用
- 通过二维码获取

**识别要点**：

- 发票代码20位，以"1"开头
- 支持扫码枪扫描
- 发票样式与传统发票不同
- 有区块链特征标识

### 区块链定额发票

**编码**：1010

**特征**：

- 面额固定（如100元、500元等）
- 无需填写信息
- 适合小额消费

**识别要点**：

- 面额印在发票上
- 无需手写或打印信息
- 通常为卷式发票

## 医疗电子票据

### 医疗电子票据（住院）

**编码**：1011

**特征**：

- 医院开具的住院费用票据
- 包含多项费用明细
- 可用于医保报销

**识别要点**：

- 标题包含"医疗住院收费票据"
- 有住院号、出院日期
- 费用明细分项清晰

### 医疗电子票据（门诊）

**编码**：1012

**特征**：

- 医院开具的门诊费用票据
- 费用相对简单
- 即时可用

**识别要点**：

- 标题包含"医疗门诊收费票据"
- 有挂号费、诊疗费、药费等明细
- 单次就诊记录

## 分类决策树

```
┌─────────────────────────────────────────────────────────┐
│                    开始分类                             │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  是否包含税额信息？                                     │
│  ├─ 是 → 进入增值税发票判断                              │
│  └─ 否 → 进入普通发票判断                                │
└─────────────────────┬───────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────┐
│  [增值税发票分支]                                       │
│  发票代码第1位？                                        │
│  ├─ 0 → 增值税专用发票                                  │
│  └─ 1 → 增值税普通发票                                  │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  [普通发票分支]                                         │
│  是否为电子格式？                                        │
│  ├─ 是 → 区块链电子发票/医疗票据判断                     │
│  │   ├─ 发票代码以"1"开头(20位) → 区块链电子发票          │
│  │   ├─ 标题含"住院" → 医疗住院票据                       │
│  │   └─ 标题含"门诊" → 医疗门诊票据                      │
│  └─ 否 → 通用机打发票                                   │
└─────────────────────────────────────────────────────────┘
```

## 特殊类型说明

### 二手车销售统一发票

- **编码**：8
- **用途**：二手车交易专用
- **特点**：票面有"二手车"字样

### 货物运输业增值税专用发票

- **编码**：10
- **用途**：运输服务专用
- **特点**：已逐步停用

### 出租车发票

- **编码**：11
- **用途**：出租车费用
- **特点**：面额小，格式固定

### 航空行程单

- **编码**：12
- **用途**：机票行程凭证
- **特点**：由航空公司开具
