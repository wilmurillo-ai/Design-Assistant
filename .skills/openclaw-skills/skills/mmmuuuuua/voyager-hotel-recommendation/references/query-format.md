# 查询格式参考文档

## 查询对象结构说明

每个查询对象必须包含以下字段：

### 必填字段
1. **subQuery** - 查询描述
   - 类型: `string`
   - 描述: 本次查询的原意描述，用于系统识别和日志记录
   - **重要**: 此字段必填，不能为空
   - 示例: `"东京银座附近的酒店"`

### 可选字段
2. **checkInDate** - 入住日期
   - 类型: `string`
   - 格式: `YYYY-MM-DD`
   - 默认: 当前日期 + 7天
   - 示例: `"2026-03-25"`

3. **checkOutDate** - 退房日期
   - 类型: `string`
   - 格式: `YYYY-MM-DD`
   - 默认: 当前日期 + 8天
   - 示例: `"2026-03-26"`

4. **country** - 国家
   - 类型: `string`
   - 描述: 酒店所在国家
   - 示例: `"日本"`, `"中国"`

5. **city** - 城市
   - 类型: `string`
   - 描述: 酒店所在城市
   - 默认: 用户当前所在城市
   - 示例: `"东京"`, `"上海"`

6. **nearby** - 附近地标(POI)
   - 类型: `string`
   - 描述: 单一地名或地标作为中心点
   - **重要**: 只能是一个地点，多个地点需分解为多个查询对象
   - 示例: `"迪士尼乐园"`, `"梅田车站"`

7. **hotelNames** - 特定酒店名称列表
   - 类型: `array`
   - 描述: 指定的酒店名称列表
   - 示例: `["半岛酒店", "四季酒店"]`

8. **brandNames** - 品牌名称列表
   - 类型: `array`
   - 描述: 酒店品牌列表
   - **重要**: 使用纯净品牌名，不加后缀
   - 正确示例: `["万豪", "希尔顿"]`
   - 错误示例: `["万豪酒店", "希尔顿酒店集团"]`

## 查询对象示例

### 示例1: 基本酒店搜索
```json
{
  "subQuery": "东京酒店推荐",
  "checkInDate": "2026-04-01",
  "checkOutDate": "2026-04-05",
  "country": "日本",
  "city": "东京"
}
```

### 示例2: 地标附近搜索
```json
{
  "subQuery": "大阪梅田附近的酒店",
  "checkInDate": "2026-05-10",
  "checkOutDate": "2026-05-12",
  "country": "日本",
  "city": "大阪",
  "nearby": "梅田"
}
```

### 示例3: 品牌特定搜索
```json
{
  "subQuery": "东京的万豪和希尔顿酒店",
  "city": "东京",
  "country": "日本",
  "brandNames": ["万豪", "希尔顿"]
}
```

### 示例4: 酒店名称指定搜索
```json
{
  "subQuery": "半岛酒店和四季酒店价格查询",
  "city": "上海",
  "country": "中国",
  "hotelNames": ["半岛酒店", "四季酒店"]
}
```

### 示例5: 多地点组合查询（必须在不同查询对象中）
```json
[
  {
    "subQuery": "香港迪士尼附近的酒店",
    "city": "香港",
    "country": "中国",
    "nearby": "迪士尼乐园"
  },
  {
    "subQuery": "上海迪士尼附近的酒店",
    "city": "上海",
    "country": "中国",
    "nearby": "迪士尼乐园"
  }
]
```

## 常见错误

### ❌ 错误示例1: subQuery为空
```json
{
  "subQuery": "",  // 错误: 不能为空
  "city": "东京"
}
```

### ❌ 错误示例2: nearby包含多个地点
```json
{
  "subQuery": "酒店查询",
  "city": "香港",
  "nearby": "尖沙咀和旺角"  // 错误: 只能是一个地点
}
```

### ❌ 错误示例3: 品牌名有后缀
```json
{
  "subQuery": "品牌查询",
  "city": "北京",
  "brandNames": ["万豪国际酒店集团", "希尔顿酒店"]  // 错误: 应使用纯净品牌名
}
```

## 响应处理规则

1. **数据源约束**: 所有酒店信息必须来自API实际响应
2. **无数据处理**: 如果没有返回数据，明确说明"查询未返回匹配结果"
3. **参数提取**: 从用户查询中智能提取相关参数
4. **语言一致性**: 保持参数语言与用户查询一致
5. **默认值处理**: 默默使用默认值，不向用户询问
