# 彼岸图网下载机制备注

## 真实 4K 下载链路

彼岸图网手机壁纸详情页中，页面展示的 `src` / `data-pic` 往往不是最终 4K 原图。

真正的 4K 下载流程是：

1. 打开详情页：
   - 例如 `https://pic.netbian.com/tupian/42413.html`
2. 登录后点击“高清原图(2400x3840)”
3. 页面会请求：
   - `/e/extend/netbiandownload.php?id=42413&t=随机数`
4. 返回 JSON：
   - `{"msg":6,"info":null,"pic":"/e/extend/netbiandownload.php?token=..."}`
5. 再访问 `pic` 字段给出的 token URL，才会真正返回 4K JPEG

## 判断是否真 4K

不要只看文件大小，也不要只看第一个 JPEG SOF 标记。

原因：
- 一些 JPEG 内部带有缩略图 SOF，比如 `100x160`
- 实际主图分辨率在后面的 SOF 标记里

正确做法：
- 遍历 JPEG 内所有 SOF 标记
- 取最后一个 SOF 标记作为实际分辨率
- 默认宽度 `>= 2000` 才视为真 4K

## 常见情况

### 情况 1：未登录
- token 接口无法返回有效图片地址
- 或者只能拿到缩略图、中图

### 情况 2：已登录但页面仍显示未登录
- 登录发生在别的浏览器 / 手机微信内
- 当前自动化浏览器没有同步 cookie
- 需要在当前浏览器上下文重新完成扫码登录

### 情况 3：遇到滑块验证
- 默认策略：暂停自动化，让用户手动完成滑块/验证码
- 不要承诺稳定自动通过滑块
- 不要持续高频重试、反复刷新、或做激进自动化尝试
- 如用户明确同意，可尝试一次视觉 AI / 浏览器自动化辅助
- 一旦失败，应立即回退到人工处理，验证通过后重新读取当前浏览器 cookie

### 情况 4：拿到 token 仍不是 4K
- 说明这张图本身没有对应的真 4K源
- 应删除该文件并继续补下一张

## 推荐策略

- 下载前先确认登录 cookie 完整
- 如遇滑块验证，优先人工完成，再继续自动流程
- 如尝试视觉 AI 辅助过滑块，应限制尝试次数，失败立即回退人工
- 每张间隔 180 秒
- 下载后立即校验分辨率
- 非 4K 立刻删除并继续补齐目标数量
