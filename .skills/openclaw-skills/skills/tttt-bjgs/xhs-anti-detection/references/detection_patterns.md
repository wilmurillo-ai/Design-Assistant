# 小红书 AI 检测特征库

## 已知检测模式

### 1. 元数据指纹

#### 高风险字段
- `Software`: "Stable Diffusion", "Midjourney", "DALL-E", "Adobe Firefly"
- `CreatorTool`: "AI", "Artificial Intelligence", "Generated"
- `Creator`: 包含 "AI"、"GPT"、"Bot" 等关键词
- `CreationDate`: 与账号历史发布时间模式不符（如凌晨 3 点批量创建）

#### 中风险字段
- `Make`/`Model`: 不存在的相机型号
- `ExifVersion`: 非标准版本号
- `UserComment`: 包含生成器标识

### 2. 像素级特征

#### 完美几何形状
- 绝对直线（像素级笔直）
- 完美圆形/椭圆
- 精确对称图案
- AI 倾向于生成"过于完美"的几何图形

#### 纹理缺失
- 纯色块区域缺少噪点
- 渐变过于平滑（无自然抖动）
- 缺少镜头光学特征（如轻微色差、镜头畸变）

#### 文字异常
- 中文字符笔画结构错误（常见：厌、储、磨等）
- 英文字母间距完全均匀
- 文字与背景融合度低（无阴影/描边）
- 多语言混排时字体不一致

### 3. 统计特征

#### 频谱特征
- DCT 系数分布异常（JPEG 压缩指纹）
- 高频分量缺失或过于规律
- 通道间相关性异常

#### 色彩分布
- 色相分布过于集中
- 饱和度分布不自然
- 缺少真实相机的色彩科学特征

### 4. 行为模式（发布侧）

#### 发布频率
- 短时间内大量发布（如 1 小时内 10+ 张）
- 发布时间集中在非活跃时段（凌晨 2-5 点）
- 新账号前 10 篇均为高质量图片

#### 内容特征
- 多张图片风格高度一致
- 主题跳跃但视觉风格不变
- 缺少"不完美"内容（如模糊、过曝、抖动）

## 对抗策略映射

| 检测模式 | 对抗方法 | 强度建议 |
|---------|---------|---------|
| 元数据指纹 | 清除 + 伪造相机信息 | 必做 |
| 完美几何 | 添加 ±1px 抖动 | 必做 |
| 纹理缺失 | 添加 5% 颗粒感 | 必做 |
| 文字异常 | 提示词强化 + 文字区域保护 | 必做 |
| 频谱指纹 | 重新编码 + 打乱 DCT | 必做 |
| 色彩分布 | 轻微色相偏移（±1°） | 推荐 |
| 发布频率 | 人工控制发布节奏 | 必做（行为层） |

## 测试方法

### 自检清单
- [ ] EXIF 中无 AI 生成器标识
- [ ] 线条有轻微抖动（非绝对笔直）
- [ ] 背景有细微纹理（非纯色）
- [ ] 文字清晰可读但有自然融合
- [ ] 色彩不过于饱和或鲜艳
- [ ] 文件大小与同类相机照片相当（±20%）

### 发布测试
1. 使用测试账号发布
2. 观察 24 小时内是否被标记"可能包含 AI 生成内容"
3. 检查互动数据是否正常（异常降权需调整参数）
