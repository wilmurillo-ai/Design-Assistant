# 内容工厂 Content Engine

用 OpenClaw + Obsidian 搭建系统化的内容生产工厂，从选题到爆款的全流程工作流。

## 核心理念

内容工厂 = 工业化思维 + AI 辅助 + 系统化工作流

- **标准化**: 可复制的选题、写作、分发流程
- **数据驱动**: 用数据指导选题，而非凭感觉
- **AI 增强**: 人机协作，AI 处理重复，人做决策
- **复利积累**: 素材库、模板库、知识库持续沉淀

## 一、选题系统

### 1.1 选题原则

**爆款公式**: 热点 + 痛点 + 独特视角

| 维度 | 权重 | 说明 |
|------|------|------|
| 时效性 | 30% | 热点窗口期，越早越好 |
| 受众广度 | 25% | 目标读者数量 |
| 痛点深度 | 25% | 解决问题的紧迫性 |
| 差异化 | 20% | 与竞品的区别度 |

## 二、创作系统

### 2.1 爆文框架模板

#### 模板 A: 痛点解决型

```
标题: [具体数字] + [痛点] + [解决方案] + [效果承诺]

结构:
1. 开场钩子 (痛点场景描述，引发共鸣)
2. 问题放大 (不解决的后果)
3. 解决方案 (步骤 1-2-3)
4. 案例证明 (真实数据/故事)
5. 行动召唤 (立即行动的步骤)

示例:
《我用 3 个月从 0 到 10 万粉: 普通人做自媒体的 5 个黄金法则》
```

#### 模板 B: 热点观点型

```
标题: [热点事件] + [独特观点] + [情绪共鸣]

结构:
1. 热点简述 (客观描述)
2. 主流观点 (正反双方)
3. 独特视角 (你的洞察)
4. 深度分析 (层层递进)
5. 价值升华 (普世道理)

示例:
《董宇辉事件背后: 为什么好员工总是先离开？》
```

#### 模板 C: 干货清单型

```
标题: [数字] + [资源/工具/方法] + [效果]

结构:
1. 价值预告 (能得到什么)
2. 清单主体 (逐项详解)
3. 使用场景 (什么时候用)
4. 避坑指南 (注意事项)
5. 福利赠送 (额外资源)

示例:
《2024 年最值得读的 20 本书: 我花了 500 小时精选》
```

### 2.2 AI 辅助创作工作流

**OpenClaw 自动化流程**:

```
用户: /content-create 选题标题

Claude (OpenClaw):
├── 1. 调取 Obsidian 选题库
├── 2. 分析竞品文章 (defuddle 提取)
├── 3. 生成大纲 (3 个版本)
├── 4. 用户选择 + 反馈
├── 5. 分段撰写 (每段 500 字)
├── 6. 人工润色
├── 7. 排版优化
└── 8. 发布到 Obsidian 待发布库
```

**AI 提示词模板**:

```markdown
## 角色
你是资深自媒体写手，擅长写 10W+ 阅读量的爆款文章。

## 任务
根据以下选题，撰写一篇 {{platform}} 文章。

## 选题信息
- 标题: {{title}}
- 目标读者: {{audience}}
- 核心观点: {{core_point}}
- 参考竞品: {{competitor_urls}}

## 要求
1. 用 [模板类型] 结构
2. 语言风格: {{style}} (专业/幽默/情感/犀利)
3. 字数: {{word_count}}
4. 包含: 钩子开头、3 个分论点、案例、金句结尾
5. 避免: AI 腔、说教感、空洞词汇

## 输出格式
1. 先输出 3 个标题选项
2. 再输出完整正文
3. 最后输出 5 个金句
```

### 2.3 人工润色检查清单

```markdown
## 润色检查清单

### 标题优化
- [ ] 是否包含数字？
- [ ] 是否有情绪词？
- [ ] 是否制造悬念？
- [ ] 是否针对特定人群？

### 开头优化
- [ ] 前 3 句能否抓住注意力？
- [ ] 是否有共鸣点？
- [ ] 是否预告价值？

### 正文优化
- [ ] 段落是否过长？（手机屏 3-4 行为宜）
- [ ] 是否有案例支撑？
- [ ] 是否有数据证明？
- [ ] 是否有金句点缀？

### 结尾优化
- [ ] 是否总结要点？
- [ ] 是否有行动召唤？
- [ ] 是否有情绪升华？

### 语言风格
- [ ] 删除"的、了、呢"等虚词
- [ ] 多用"你"拉近距离
- [ ] 长短句交替，有节奏感
- [ ] 口语化，像朋友聊天
```

## 七、实战案例

### 案例: 从 0 到 10W+ 的全过程

```markdown
# 案例复盘: 《我用 AI 搭建内容工厂，月入 5 万》

## 背景
- 账号: 新号，粉丝 0
- 平台: 公众号 + 知乎 + 小红书
- 周期: 3 个月

## 选题分析
- 热点: AI 工具普及
- 痛点: 想做自媒体但没时间
- 差异化: 系统化工厂思维，非单点技巧

## 创作过程
1. 竞品分析: 读了 20 篇同类文章
2. 大纲迭代: 3 个版本，最终选择"故事+方法论"结构
3. AI 辅助: Claude 生成初稿，人工润色 40%
4. 视觉设计: Canva 制作 5 张信息图

## 发布策略
- 公众号首发 (完整版)
- 知乎同步 (删减故事，增加方法论)
- 小红书拆解 (3 张图，分 3 天发)

## 数据结果
| 平台 | 阅读 | 点赞 | 评论 | 引流 |
|------|------|------|------|------|
| 公众号 | 12W | 2K | 300 | 500 |
| 知乎 | 8W | 1.5K | 200 | 300 |
| 小红书 | 5W | 3K | 150 | 200 |

## 成功因素
1. 标题数字 + 痛点 + 效果承诺
2. 开篇个人故事引发共鸣
3. 方法论可落地，工具具体
4. 多平台适配，最大化曝光

## 可复用模板
- 已保存到 003-Templates/文章模板/案例模板.md
```

## 十、常见问题

**Q: 每天需要多少时间？**
A: 初期 3-4 小时，熟练后 1-2 小时。AI 可节省 60% 时间。

**Q: 如何保证原创性？**
A: AI 只辅助框架和初稿，核心观点、案例、个人经历必须原创。

**Q: 多平台内容需要重写吗？**
A: 建议一稿多改，保持核心观点一致，调整表达形式适配平台。

**Q: 多久能看到效果？**
A: 通常 1-3 个月积累期，第 4 个月起进入增长期。
