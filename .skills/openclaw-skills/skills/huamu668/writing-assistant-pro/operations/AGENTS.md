# AGENTS.md - 代理团队配置
# 可调度的工作单元

## 核心代理

### 1. write（内容创作代理）
- **职责**: 基于需求创作各类文案
- **触发条件**: 用户说"写"或提供创作需求
- **工作流**:
  1. 分析目标平台和受众
  2. 确定内容结构和风格
  3. 生成初稿
  4. 自检优化
- **输出**: 完整文案 + 创作思路说明
- **状态**: ✅ 在线

### 2. rewrite（改写优化代理）
- **职责**: 优化现有文案
- **触发条件**: 用户说"改"或提供待优化内容
- **工作流**:
  1. 分析原文问题和改进空间
  2. 提供具体修改建议
  3. 生成改写版本
  4. 对比说明改进点
- **输出**: 优化后文案 + 修改说明
- **状态**: ✅ 在线

### 3. headline（标题生成代理）
- **职责**: 生成爆款标题和多版本测试
- **触发条件**: 需要标题创作或A/B测试
- **工作流**:
  1. 分析内容核心卖点
  2. 生成多风格标题（10+版本）
  3. 按吸引力排序
  4. 说明每个标题的钩子
- **输出**: 标题列表 + 适用场景说明
- **状态**: ✅ 在线

### 4. ideate（选题策划代理）
- **职责**: 内容创意和选题建议
- **触发条件**: 用户说"选题"或请求创意
- **工作流**:
  1. 分析用户领域和目标
  2. 挖掘受众痛点和兴趣点
  3. 生成选题矩阵
  4. 评估每个选题的可行性
- **输出**: 选题清单 + 执行优先级
- **状态**: ✅ 在线

### 5. writing-assistant（写作助手核心）**[新建]**
- **职责**: 维护写作助手架构，协调代理
- **触发条件**: 架构相关请求
- **工作流**:
  1. 识别写作任务类型
  2. 分配最适合的代理
  3. 协调多代理协作
  4. 输出最终成果
- **输出**: 高质量写作成果
- **状态**: 🆕 刚部署

## 调度规则
```
用户指令 → 意图识别（写/改/标题/选题） → 代理匹配 → 执行 → 结果汇报
```

## 代理冲突解决
1. 单任务：最高匹配度优先
2. 多任务：按依赖顺序执行
3. 冲突时：向创意总监请求决策

## 质量检查点
- 每次输出前自检：是否有具体示例？是否删除了废话？
- 重写代理必须指出原文具体问题
- 标题代理必须提供10+版本
