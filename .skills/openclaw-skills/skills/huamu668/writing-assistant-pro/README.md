## 使用方法

### 1. 创作内容
说：**"写一篇小红书文案，关于[主题]"**

### 2. 改写优化
说：**"改一下这段：[你的内容]"**

### 3. 生成标题
说：**"给这篇文章起10个标题"**

### 4. 选题策划
说：**"给我5个选题方向，关于[领域]"**

## 架构说明

### 第一层：身份层 (identity/)
定义写作助手的本质
- **SOUL.md** - 核心灵魂：为极致写作而生
- **IDENTITY.md** - 角色配置：首席撰稿人
- **USER.md** - 创意总监（用户）画像

### 第二层：操作层 (operations/)
定义如何创作
- **AGENTS.md** - 代理团队：write/rewrite/headline/ideate
- **HEARTBEAT.md** - 系统心跳和质量检查
- **ROLE-CHIEF-WRITER.md** - 首席撰稿人工作指南

### 第三层：知识层 (knowledge/)
积累写作经验
- **MEMORY.md** - 长期记忆：风格偏好、有效技巧
- **shared-context/** - 跨会话状态
- **logs/** - 创作日志

## 工作原则

1. **第一性原理写作** - 每个字都有目的
2. **读者时间至上** - 具体 > 抽象
3. **零废话反馈** - 直接给改写示例
4. **30%删除率** - 好文章是改出来的

## 平台特化

- **小红书**：emoji + 数字 + 痛点，300-800字
- **公众号**：钩子→故事→观点，1500-3000字
- **销售文案**：痛点放大→证明→行动，500-1500字
