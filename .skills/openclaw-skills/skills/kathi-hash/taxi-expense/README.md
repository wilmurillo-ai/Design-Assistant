# 打车报销处理流程

## 目录结构

```
~/.openclaw/workspace/taxi_expense/
├── YYYY-MM-taxi_expense.xlsx  (按月生成的报销Excel)
├── taxi_data.json             (订单数据，含去重)
└── screenshots/               (脱敏处理后的订单截图)
    └── order_YYYY-MM-DD_amount.png
```

## 处理流程

```
📸 用户发滴滴订单截图 (支持多张)
    │
    ▼
🤖 Tesseract OCR 识别文字+坐标
    │  - 识别日期、金额、起点、终点
    │  - 返回每个文字块的 bbox 坐标
    │
    ▼
💰 智能金额提取（容错 OCR 误读）
    │  - ¥ 读成 % → 匹配 %[0-9.]+
    │  - ¥ 丢失 → 从起点行末尾提取裸数字
    │  - 小数点丢失 (1970→19.70) → >500 自动修正
    │
    ▼
✅ 判定报销条件
    │  - 工作日（周一至周五）21:00 后打车 → 可报销
    │  - 周末、非 21:00 后、已关闭(¥0) → 过滤
    │
    ▼
✂️ 裁剪订单卡片 + 脱敏
    │  - 根据 OCR 坐标裁剪单笔订单卡片
    │  - 白色方块遮盖目的地中间文字（保留首尾字）
    │  - 目的地文字脱敏：古*****门
    │
    ▼
📊 生成 Excel
    │  - Sheet1: 报销明细（序号、日期、星期、时间、起点、终点、金额、备注）
    │  - Sheet2: 订单截图（含脱敏截图）
    │  - 按月命名：2026-03-taxi_expense.xlsx
    │
    ▼
📤 发送给用户
```

## 使用方式

### 命令行
```bash
node ~/.openclaw/workspace/skills/taxi_expense/scripts/process.js <image1> [image2] ...
```

### OpenClaw 对话
直接发滴滴截图，说"处理"即可。

## Excel 字段

| 序号 | 日期 | 星期 | 时间 | 起点 | 终点 | 金额 | 备注 |
|------|------|------|------|------|------|------|------|
| 1 | 2026-03-05 | 周四 | 21:04 | 西溪世纪中心... | 古*****门 | ¥24.90 | 加班打车 |

## 关键技术

### OCR 金额容错
- `¥19.90` → OCR 读成 `%19.90` → 正则 `[¥%](\d+\.?\d*)` 兼容
- `¥22.80` → OCR 丢掉 ¥ → 从起点行末尾 `(\d+\.\d{1,2})$` 提取
- `¥19.70` → OCR 读成 `¥1970` → 金额 >500 自动插入小数点

### 脱敏算法
- **截图脱敏：** 用 sharp 单管道 `extract + composite`，在目的地首尾字之间叠加白色方块
- **文字脱敏：** Excel 中终点显示为 `古*****门`（保留首尾字，中间用 * 替代）

### 去重逻辑
- 基于 `日期 + 金额` 判断是否重复
- 多图处理时自动去重

## 依赖

- **sharp** - 图片裁剪、脱敏方块
- **tesseract.js@4** - OCR 文字识别（带坐标）
- **exceljs** - Excel 生成

## 已知限制

1. Tesseract 中文识别偶有误差（"点"→"炭"、"轻享"→"轻亭"）
2. 原始截图（非压缩/裁剪过的）识别效果最佳
3. 同一日期多笔相同金额的订单可能导致去重误判
