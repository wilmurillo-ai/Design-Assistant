# AIEO 统一问题库

## 概述

本文档是 AIEO 诊断、定位、内容创作三个阶段共享的标准问题库，用于：

1. **诊断阶段**：AI 可见性测试
2. **定位阶段**：竞品分析和场景验证
3. **内容阶段**：FAQ 内容创作和效果监控
4. **监控阶段**：定期追踪 AI 回答质量

---

## 一、问题分类体系

### 问题类型定义

| 类型代码 | 类型名称 | 定义 | 优先级 |
|---------|---------|------|--------|
| **BR** | 品牌认知类 | 用户对品牌整体印象的询问 | P0 |
| **CR** | 品类推荐类 | 用户在品类中寻求推荐 | P0 |
| **CP** | 对比决策类 | 用户在多个品牌间做选择 | P0 |
| **SC** | 场景推荐类 | 用户在特定场景下寻求建议 | P1 |
| **PD** | 产品详情类 | 用户询问具体产品信息 | P1 |
| **HS** | 历史故事类 | 用户询问品牌背景和历史 | P2 |
| **SV** | 服务售后类 | 用户询问购买和服务相关 | P2 |

### 测试分层

| 层级 | 问题数量 | 覆盖类型 | 使用阶段 |
|------|---------|---------|---------|
| **Tier 1 核心** | 10-15题 | BR + CR + CP | 诊断必测 |
| **Tier 2 扩展** | 15-20题 | SC + PD | 定位验证 |
| **Tier 3 长尾** | 20-30题 | HS + SV + 长尾 | 内容监控 |

## 四、问题库生成指南

### 4.1 从诊断报告生成

诊断阶段完成后，根据以下信息生成定制问题：

1. **品牌名称** → 替换所有模板中的 [品牌名]
2. **主要竞品** → 替换 [竞品] / [品牌A] / [品牌B]
3. **所属品类** → 替换 [品类]
4. **核心产品** → 替换 [具体产品]
5. **目标场景** → 替换 [场景]

### 4.2 从定位报告补充

定位阶段完成后，补充以下定制问题：

1. **差异化定位词** → 创建含定位词的问题
2. **目标人群** → 创建人群相关问题
3. **核心使用场景** → 创建场景问题
4. **属性-价值映射** → 创建功能/价值问题

### 4.3 问题库数量建议

| 阶段 | 最少问题数 | 推荐问题数 | 完整问题数 |
|------|-----------|-----------|-----------|
| 诊断 | 15 | 30 | 50 |
| 定位 | 30 | 50 | 80 |
| 内容 | 50 | 80 | 100+ |
| 监控 | 20 | 40 | 60 |

## 七、使用流程

### 7.1 诊断阶段使用

```
1. 确定品牌基本信息（名称、品类、竞品、场景）
2. 从 Tier 1 问题生成 15 个核心问题
3. 补充 5-10 个行业专属问题
4. 在 6 个 AI 平台进行测试
5. 记录测试结果，生成诊断报告
```

### 7.2 定位阶段使用

```
1. 继承诊断阶段的问题库
2. 根据竞品分析结果，扩展对比问题
3. 根据场景分析，扩展场景问题
4. 补充 Tier 2 问题（共 30-50 个）
5. 验证定位假设，调整定位策略
```

### 7.3 内容阶段使用

```
1. 将问题库作为 FAQ 内容创作的输入
2. 每个问题对应一个 FAQ 内容
3. 优先创作 Tier 1 问题的 FAQ
4. 根据诊断发现的薄弱项，优先补强
5. 持续扩展 Tier 3 长尾问题
```

### 7.4 监控阶段使用

```
1. 选取 20-40 个核心问题作为监控基准
2. 每周/每两周进行一次 AI 平台测试
3. 记录品牌可见性变化
4. 对比竞品表现变化
5. 根据结果调整内容策略
```
