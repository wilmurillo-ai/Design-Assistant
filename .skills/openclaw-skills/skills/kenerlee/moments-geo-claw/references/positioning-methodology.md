## 导言：定位的范式转移

### 从 SEO 到 AIEO 的演变

```
1990s-2010s          2010s-2020s          2020s-2030s
    SEO      →          SEM       →         AIEO
 搜索引擎优化         搜索引擎营销        AI引擎优化

目标：搜索排名        目标：付费+自然       目标：AI推荐位
载体：网页链接        载体：广告+内容       载体：AI对话回答
竞争：10个蓝链        竞争：广告位竞价      竞争：AI首推位置
```

### 核心变化

| 维度 | 传统定位 | AIEO 时代定位 |
|------|----------|---------------|
| **受众** | 人类消费者 | 人类 + **AI 系统** |
| **载体** | 广告、slogan | **结构化数据、FAQ** |
| **传播** | 品牌→媒体→人 | 品牌→**AI训练/索引**→AI→人 |
| **验证** | 消费者调研 | **AI 平台实测** |
| **竞争** | 心智占领 | **AI 回答占位** |

## 第一章：AIEO 定位的基本原理

### 1.1 AI 如何"理解"品牌定位

```
传统定位在人脑中的存储：
┌─────────────────────────────────────┐
│  人脑                                │
│  ┌─────────────────────────────┐   │
│  │ "百威" → 情感联想：         │   │
│  │   - 高端                     │   │
│  │   - 派对                     │   │
│  │   - 美国                     │   │
│  │   - 皇者风范（广告印象）     │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘

AIEO定位在AI系统中的存储：
┌─────────────────────────────────────┐
│  AI知识图谱                          │
│  ┌─────────────────────────────┐   │
│  │ Entity: 百威                 │   │
│  │ Type: 啤酒品牌               │   │
│  │ Attributes:                  │   │
│  │   - founded: 1876            │   │
│  │   - origin: 美国             │   │
│  │   - category: 高端啤酒       │   │
│  │   - parent: AB InBev         │   │
│  │ Relations:                   │   │
│  │   - competitor: 青岛啤酒     │   │
│  │   - competitor: 喜力         │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

**关键洞察**：
- 人脑存储**情感印象**，可以模糊、隐喻
- AI 存储**结构化知识**，需要明确、可验证
- 传统定位影响人的感觉，AIEO 定位影响 AI 的回答

### 1.2 AI 回答的生成逻辑

当用户问"高端啤酒推荐"时，AI 的处理流程：

```
Step 1: 意图识别
        用户想要 → 啤酒品牌推荐 + 高端定位

Step 2: 知识检索
        从知识库/索引中检索：
        - 哪些品牌标记为"高端啤酒"？
        - 这些品牌的权威性如何？
        - 有哪些可引用的内容？

Step 3: 排序与筛选
        根据以下因素排序：
        - 内容权威性（官网 > 第三方）
        - 信息完整度（Schema > 纯文本）
        - 引用频次（被多处提及）
        - 情感倾向（正面 > 负面）

Step 4: 回答生成
        "高端啤酒品牌推荐：
         1. 青岛啤酒 - 中国历史最悠久...
         2. 百威 - 全球知名品牌...
         3. 喜力 - 荷兰精酿..."
```

**AIEO 定位的目标**：影响 Step 2-3，让品牌在检索和排序中占优势

### 1.3 AIEO 定位的三层架构

```
┌─────────────────────────────────────────────────────┐
│  Layer 3: 内容层（Content Layer）                    │
│  ┌─────────────────────────────────────────────┐   │
│  │  FAQ、博客、对比页面、案例研究               │   │
│  │  → 为 AI 提供可引用的答案素材               │   │
│  └─────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────┤
│  Layer 2: 结构层（Schema Layer）                     │
│  ┌─────────────────────────────────────────────┐   │
│  │  Organization、Product、FAQPage Schema       │   │
│  │  → 让 AI 准确理解品牌实体和属性             │   │
│  └─────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────┤
│  Layer 1: 策略层（Strategy Layer）                   │
│  ┌─────────────────────────────────────────────┐   │
│  │  定位声明、价值主张、差异化属性              │   │
│  │  → 定义品牌在 AI 世界中的身份               │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### Step 1: 识别竞争替代品 + AI 推荐分析

**传统做法**：列出消费者可能选择的替代方案

**AIEO 升级**：测试 AI 平台实际推荐哪些竞品

#### 1.1 替代品四象限

```
                    高 AI 推荐频率
                         │
          ┌──────────────┼──────────────┐
          │              │              │
          │   威胁区     │   主战场     │
          │  (潜在对手)  │  (直接竞品)  │

          │  (间接替代)  │  (传统竞品)  │
          │              │              │
          └──────────────┼──────────────┘
                         │
                    低 AI 推荐频率
```

#### 1.2 AI 竞品测试方法

### Step 2: 识别独特属性 + AI 引用检验

**传统做法**：列出产品/服务的独特卖点

**AIEO 升级**：检验这些属性是否被 AI 引用

#### 2.1 属性三层分类

```
┌─────────────────────────────────────────────┐
│  第一层：可验证事实（AI 最爱引用）           │
│  - 成立年份、市场份额、获奖记录             │
│  - 专利数量、认证资质、客户数量             │
│  例："源自1876年"、"全球销量第一"           │
├─────────────────────────────────────────────┤
│  第二层：技术/工艺差异（需要解释）           │
│  - 独特技术、专有工艺、配方特点             │
│  例："榉木酿造工艺"、"低温慢发酵"           │
├─────────────────────────────────────────────┤
│  第三层：情感/抽象属性（AI 难以引用）        │
│  - 品牌精神、情感诉求、生活方式             │
│  例："皇者风范"、"Just Do It"               │
└─────────────────────────────────────────────┘
```

#### 2.2 属性 AI 引用检验

| 独特属性 | 官网是否明确表述 | AI 是否引用 | 引用准确度 | 优化优先级 |
|----------|-----------------|-------------|-----------|-----------|
| 149年历史 | ⚠️ 不够突出 | ✅ 偶尔引用 | 80% | 中 |
| 榉木工艺 | ❌ 未提及 | ❌ 未引用 | 0% | **高** |
| 全球领导者 | ⚠️ 表述模糊 | ⚠️ 不准确 | 40% | **高** |
| 皇者风范 | ✅ 明确 | ❌ 未引用 | 0% | 低（抽象） |

#### 2.3 属性表述优化原则

```
❌ AI 不友好的表述：
"我们拥有悠久的历史和传承"
"采用独特的酿造工艺"
"是行业的领先者"

✅ AI 友好的表述：
"源自1876年，拥有149年不间断酿造历史"
"采用独特榉木酿造工艺，发酵时间比普通啤酒长30%"
"全球销量排名第一的高端啤酒品牌，年销量超过XX亿升"

原则：
1. 具体 > 模糊
2. 数字 > 形容词
3. 可验证 > 主观判断
```

### Step 5: 确定市场品类 + Schema 标记

**传统做法**：选择竞争的市场品类

**AIEO 升级**：用 Schema 向 AI 声明品类归属

#### 5.1 品类定位策略选择

```
策略A：正面竞争（Head-to-Head）
┌─────────────────────────────────────┐
│ 在大品类中竞争                       │
│ 例："高端啤酒"                       │
│ 优势：TAM大，搜索量高                │
│ 风险：竞争激烈，难以首推             │
│ Schema: "category": "高端啤酒"       │
└─────────────────────────────────────┘

策略B：细分领导（Big Fish, Small Pond）
┌─────────────────────────────────────┐
│ 在细分品类中称王                     │
│ 例："美式高端啤酒" / "商务用啤酒"   │
│ 优势：易于首推，差异化明显           │
│ 风险：TAM较小                        │
│ Schema: "category": "美式高端啤酒"   │
└─────────────────────────────────────┘

策略C：创造品类（Create New Category）
┌─────────────────────────────────────┐
│ 定义全新品类                         │
│ 例："国际化生活方式啤酒"             │
│ 优势：无竞争，独占品类               │
│ 风险：教育成本高，AI可能不认         │
│ Schema: 需要逐步建立                 │
└─────────────────────────────────────┘
```

#### 5.2 Schema 品类声明

```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "百威啤酒",
  "category": "高端啤酒",
  "additionalType": "美式啤酒",
  "isRelatedTo": [
    {
      "@type": "Product",
      "name": "商务用酒",
      "description": "适合商务宴请场合的酒类"
    }
  ],
  "audience": {
    "@type": "Audience",
    "audienceType": "追求品质生活的消费者"
  }
}
```

#### 5.3 品类关键词布局

| 品类层级 | 关键词 | 布局位置 | Schema 标记 |
|----------|--------|----------|-------------|
| L1 大品类 | 啤酒、酒类 | 全站 | @type |
| L2 细分品类 | 高端啤酒 | 品牌页 | category |
| L3 场景品类 | 商务用酒 | FAQ页 | isRelatedTo |
| L4 特色标签 | 美式啤酒、进口啤酒 | 产品页 | additionalType |

### 3.2 内容结构规范

#### 3.2.1 Answer-First 写作原则

```
结构公式：
[结论] + [支撑事实] + [行动建议]

第一句话规则：
- 必须直接回答问题
- 必须包含品牌名
- 必须包含核心定位词
- 字数控制在50字以内

示例对比：

❌ 传统写法：
"在当今啤酒市场，消费者面临众多选择。
啤酒的历史可以追溯到数千年前...
（第500字才提到品牌）"

✅ AIEO 写法：
"百威是全球销量领先的高端啤酒品牌，
以'皇者风范'著称。源自1876年美国，
采用独特榉木酿造工艺..."
```

#### 3.2.2 对比内容创建原则

```
对比内容黄金结构：

1. 开头：承认双方都优秀（避免贬低）
   "XX和YY都是优秀的[品类]品牌，各有特色。"

2. 客观对比：列出双方差异
   "XX的特点是[A]，YY的特点是[B]。"

3. 适用场景：给出选择建议
   "如果您注重[需求A]，XX更适合；
    如果您注重[需求B]，YY更适合。"

4. 自然倾向：通过场景引导选择自己
   "在[我方优势场景]，XX是更好的选择。"
```
