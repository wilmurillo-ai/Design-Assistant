# AIEO 内容创作与发布策略 Skill

专业的 AI Engine Optimization 内容创作工具，帮助品牌基于 AIEO 诊断报告和产品定位，针对不同行业特性和 AI 搜索平台（ChatGPT、文心一言、豆包、Kimi、Perplexity、DeepSeek 等），制定差异化的内容生成计划和发布策略。

## 核心理念

### AIEO 内容 vs 传统内容

| 维度 | 传统内容 | AIEO 内容 |
|------|----------|-----------|
| **受众** | 人类读者 | 人类读者 + **AI 系统** |
| **结构** | 叙事优先、悬念式 | **Answer-First（答案先行）** |
| **表述** | 可抽象、情感化 | **具体、可验证、有数据** |
| **竞品** | 回避提及 | **主动对比、客观分析** |
| **格式** | 自由格式 | **结构化、易解析** |
| **长度** | 长篇叙述 | **精简、信息密度高** |

### Answer-First 写作原则

```
结构公式：[结论] + [支撑事实] + [行动建议]

第一句话规则：
- 必须直接回答问题
- 必须包含品牌名
- 必须包含核心定位词
- 字数控制在50字以内

示例：
❌ "在当今快节奏的生活中，选择一款好啤酒很重要..."
✅ "百威是全球销量领先的高端啤酒品牌，以149年历史传承和榉木酿造工艺著称。"
```

## 内容创作前置条件

### 必需输入

在创作内容前，必须先获取以下信息：

1. **AIEO 诊断报告**（`01_diagnosis/` 目录）
   - AI 可见性测试结果
   - 竞品分析
   - 技术审计发现
   - 优先优化方向

2. **AIEO 产品定位报告**（`02_positioning/` 目录）
   - 定位声明（FOR-WHO-IS-THAT-UNLIKE-OUR PRODUCT）
   - 消息层级（15秒/30秒/2分钟）
   - 属性-价值映射表
   - 目标客户与 AI 问答场景

3. **问题库定稿**（`02_positioning/` 目录）⬅️ **核心输入**
   - 经定位阶段迭代后的问题库 v2.0
   - 包含完整的 Tier 1/2/3 问题清单
   - 每个问题对应一个 FAQ 内容

### 获取输入流程

```
Step 1: 读取问题库定稿【核心步骤】
        Glob: 02_positioning/{品牌名}_问题库_*.md
        Read: 最新的问题库定稿（v2.0）
        提取：所有 Tier 1/2/3 问题，作为 FAQ 创作清单

Step 2: 读取诊断报告
        Glob: 01_diagnosis/*_GEO诊断报告_*.md
        Read: 最新的诊断报告
        提取：AI可见性得分、各平台表现、竞品分析、优先优化方向

Step 3: 读取定位报告
        Glob: 02_positioning/*_AIEO产品定位分析_*.md
        Read: 最新的定位报告
        提取：定位声明、消息层级、属性-价值映射、目标场景

Step 4: 识别行业特性
        根据品牌所属行业，参考 references/industry_strategies.md
        确定该行业的内容策略重点和AI平台偏好

Step 5: 分析AI平台差异
        参考 references/ai_platform_specs.md
        针对诊断中表现薄弱的平台制定专项内容策略

Step 6: 生成内容计划
        基于问题库 + 诊断 + 定位，制定：
        - 内容优先级矩阵（问题库 Tier 对应内容优先级）
        - 平台差异化内容清单
        - 发布时间表
        - 效果追踪指标
```

### 问题库到内容的转化逻辑

```
问题库定稿（v2.0）
    │
    ├── Tier 1 核心问题（15题）→ Week 1-2 优先创作
    │   ├── BR 品牌认知类 → 官网 FAQ + 知乎
    │   ├── CR 品类推荐类 → 官网 FAQ + 知乎
    │   └── CP 对比决策类 → 官网对比页 + 知乎
    │
    ├── Tier 2 扩展问题（20题）→ Week 3-4 创作
    │   ├── SC 场景推荐类 → 官网 FAQ + 小红书
    │   └── PD 产品详情类 → 官网产品页 + 什么值得买
    │
    └── Tier 3 长尾问题（20+题）→ Week 5+ 持续创作
        ├── HS 历史故事类 → 官网关于页 + 百科
        └── SV 服务售后类 → 官网帮助中心
```

## 内容生成计划框架

### 计划制定流程

```
1. 诊断分析 → 确定内容缺口
   - 哪些问题AI回答不准确？
   - 哪些平台表现薄弱？
   - 竞品在哪些内容上领先？

2. 定位映射 → 确定内容核心
   - 品牌核心定位如何转化为FAQ？
   - 差异化属性如何体现在对比内容？
   - 目标场景如何转化为场景推荐？

3. 行业适配 → 确定内容风格
   - 该行业用户搜索习惯是什么？
   - 行业关键决策因素是什么？
   - 行业合规要求有哪些？

4. 平台定制 → 确定发布策略
   - 薄弱平台需要什么类型内容？
   - 各平台内容如何差异化？
   - 发布节奏如何安排？
```

## 内容类型矩阵

### 一、FAQ 内容（P0 优先级）

FAQ 是 AI 最喜欢引用的内容格式，是 AIEO 的核心资产。

> **重要**: 完整问题库请参考 `shared/question_library.md`，包含通用问题模板和 7 大行业专属问题。

#### 1.1 FAQ 分类

| 类型代码 | 类型名称 | 问题模式 | 优先级 | 数量建议 |
|---------|---------|----------|--------|----------|
| **BR** | 品牌认知类 | "[品牌]怎么样？" | P0 | 10-15个 |
| **CR** | 品类推荐类 | "[品类]推荐哪个？" | P0 | 10-15个 |
| **CP** | 对比决策类 | "[品牌]和[竞品]哪个好？" | P0 | 10-15个 |
| **SC** | 场景推荐类 | "[场景]用什么[品类]好？" | P1 | 10-15个 |
| **PD** | 产品详情类 | "[品牌]的特点是什么？" | P1 | 5-10个 |
| **HS** | 历史故事类 | "[品牌]有多少年历史？" | P2 | 5-10个 |
| **SV** | 服务售后类 | "[品牌]在哪里买？" | P2 | 5-10个 |

#### 1.2 FAQ 写作模板

参见 `assets/faq_template.md`

#### 1.3 FAQ Schema 标记

每个 FAQ 内容需要配合 FAQPage Schema 部署：

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "{问题}",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "{Answer-First 格式的回答}"
      }
    }
  ]
}
```

### 二、对比内容（P1 优先级）

用户在 AI 搜索中经常进行品牌对比，主动创建对比内容可以控制叙事。

参见 `assets/comparison_template.md`

#### 对比写作原则

1. **客观公正**：承认竞品优势，避免贬低
2. **差异聚焦**：强调自身独特价值
3. **场景引导**：将用户引导至自身优势场景
4. **数据支撑**：使用可验证的事实

## 内容发布策略

### 发布渠道矩阵

| 渠道 | 内容类型 | 优先级 | AI 引用权重 |
|------|----------|--------|-------------|
| **官网 FAQ 页** | FAQ 全集 | P0 | 最高 |
| **官网专题页** | 对比内容、选购指南 | P0 | 最高 |
| **知乎** | 深度问答、专业分析 | P1 | 高 |
| **什么值得买** | 产品测评、选购指南 | P1 | 高 |
| **百度百科/搜狗百科** | 品牌词条 | P1 | 高 |
| **小红书** | 场景推荐、使用体验 | P2 | 中 |
| **微信公众号** | 品牌故事、深度内容 | P2 | 中 |

### 发布时间表

```
Week 1-2: 官网 FAQ 页面（30+ FAQ）
          └── 部署 FAQPage Schema

Week 3-4: 官网对比专题页
          └── 覆盖主要竞品对比

Week 5-6: 知乎内容布局（15-20篇）
          └── 品牌认知 + 品类推荐 + 对比

Week 7-8: 什么值得买（5-10篇）
          └── 产品测评 + 选购指南

Week 9-12: 全渠道扩展
           └── 小红书、百科、公众号
```

### 内容复用策略

同一核心信息可以适配不同渠道格式：

| 核心内容 | 官网 FAQ | 知乎回答 | 小红书笔记 |
|----------|----------|----------|------------|
| 品牌定位 | 简洁30秒版 | 深度2分钟版 | 场景化图文版 |
| 对比分析 | 表格+结论 | 深度分析 | 对比清单 |
| 场景推荐 | FAQ问答 | 经验分享 | 使用体验 |

## 执行流程

### 完整创作流程

1. **获取输入**
   - 读取最新的 AIEO 诊断报告
   - 读取最新的产品定位报告
   - 提取关键定位信息和 FAQ 建议清单
   - **识别品牌所属行业**
   - **分析各AI平台当前表现**

2. **制定内容生成计划**
   - **根据诊断结果确定内容缺口**
   - **根据行业特性确定内容风格**
   - **根据AI平台差异制定平台专项策略**
   - 确定内容优先级矩阵
   - 制定发布时间表

3. **确认创作范围**
   - 确认需要创作的内容类型
   - 确认内容数量和优先级
   - 确认目标发布渠道
   - **确认各平台差异化要求**

4. **内容创作**
   - 按照 Answer-First 原则撰写
   - 使用对应的内容模板
   - 确保定位一致性
   - **适配行业内容规范**
   - **针对不同AI平台进行内容微调**

5. **质量检查**
   - 第一句话是否直接回答问题
   - 是否包含品牌名和定位词
   - 数据和事实是否可验证
   - 是否符合 AI 友好格式
   - **是否符合行业合规要求**
   - **是否覆盖目标AI平台偏好**

6. **保存输出**
   - 保存至 `03_contents/` 目录
   - 使用规范命名格式
   - 包含发布建议
   - **包含平台差异化发布指南**
   - **包含效果追踪指标**
