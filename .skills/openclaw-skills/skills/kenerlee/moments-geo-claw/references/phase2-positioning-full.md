# AIEO 产品定位 Skill

专业的 AI Engine Optimization 产品定位工具，帮助品牌/企业在 AI 搜索引擎时代（ChatGPT、文心一言、豆包、Kimi、Perplexity 等）进行产品和服务定位优化。

## 核心理念

### AIEO 定位 vs 传统定位

| 维度 | 传统定位 | AIEO 定位 |
|------|----------|-----------|
| **受众** | 消费者 | 消费者 + **AI 系统** |
| **载体** | 广告、slogan | **Schema、FAQ、结构化数据** |
| **表述** | 可抽象、情感化 | 必须具体、结构化、可验证 |
| **竞品策略** | 回避提及 | **主动对比** |
| **内容结构** | 叙事优先 | **Answer-First** |
| **验证方式** | 消费者调研 | **AI 平台实测** |
| **迭代周期** | 年度 | 月度 |

### 双受众原则

AIEO 时代的定位必须同时面向两个受众：
1. **人类消费者** - 传统定位目标
2. **AI 系统** - 需要结构化、可验证的信息

## AIEO 六步定位法

基于 April Dunford 定位方法论的 AIEO 升级版

### Step 1: 识别竞争替代品 + AI 推荐分析

**传统做法**：列出消费者可能选择的替代方案

**AIEO 升级**：测试 AI 平台实际推荐哪些竞品

#### 1.1 替代品四象限

```
                    高 AI 推荐频率
                         │
          ┌──────────────┼──────────────┐
          │              │              │
          │   威胁区     │   主战场     │
          │  (潜在对手)  │  (直接竞品)  │
          │              │              │
低市场────┼──────────────┼──────────────┼────高市场
重叠度    │              │              │    重叠度
          │   观察区     │   防守区     │
          │  (间接替代)  │  (传统竞品)  │
          │              │              │
          └──────────────┼──────────────┘
                         │
                    低 AI 推荐频率
```

#### 1.2 AI 竞品测试

使用 Playwright MCP 在以下平台测试：
- 豆包、Kimi、DeepSeek（国内高优先级）
- Perplexity、ChatGPT（国际参考）

测试问题模板：

> **完整问题库**: 参见 `shared/question_library.md`

基础测试问题（至少 10 题）：
- 品牌认知: "[品牌名]怎么样？"、"[品牌名]好不好？"
- 品类推荐: "[品类]推荐哪个品牌？"、"[品类]哪个牌子好？"
- 对比决策: "[品牌]和[竞品]哪个好？"（覆盖 Top 3 竞品）

#### 1.3 输出：竞品 AI 可见性矩阵

| 竞品 | 豆包 | Kimi | DeepSeek | Perplexity | 总提及 | 首推率 |
|------|------|------|----------|------------|--------|--------|
| 品牌A | ✅首推 | ✅首推 | ✅提及 | ✅首推 | 4/4 | 75% |
| 我方 | ❌ | ✅提及 | ❌ | ✅提及 | 2/4 | 0% |

### Step 2: 识别独特属性 + AI 引用检验

**传统做法**：列出产品/服务的独特卖点

**AIEO 升级**：检验这些属性是否被 AI 引用

#### 2.1 属性三层分类

```
┌─────────────────────────────────────────────┐
│  第一层：可验证事实（AI 最爱引用）           │
│  - 成立年份、市场份额、获奖记录             │
│  - 专利数量、认证资质、客户数量             │
│  例："源自1876年"、"全球销量第一"           │
├─────────────────────────────────────────────┤
│  第二层：技术/工艺差异（需要解释）           │
│  - 独特技术、专有工艺、配方特点             │
│  例："榉木酿造工艺"、"低温慢发酵"           │
├─────────────────────────────────────────────┤
│  第三层：情感/抽象属性（AI 难以引用）        │
│  - 品牌精神、情感诉求、生活方式             │
│  例："皇者风范"、"Just Do It"               │
└─────────────────────────────────────────────┘
```

#### 2.2 属性表述优化原则

```
❌ AI 不友好的表述：
"我们拥有悠久的历史和传承"
"采用独特的酿造工艺"

✅ AI 友好的表述：
"源自1876年，拥有149年不间断酿造历史"
"采用独特榉木酿造工艺，发酵时间比普通啤酒长30%"

原则：
1. 具体 > 模糊
2. 数字 > 形容词
3. 可验证 > 主观判断
```

### Step 3: 属性到价值映射 + FAQ 转化

**传统做法**：将属性转化为客户利益

**AIEO 升级**：将价值映射转化为 FAQ 问答格式

#### 3.1 价值映射三角

```
                ┌─────────────┐
               │  情感价值    │
               │ (Why Care)  │
               └──────┬──────┘
                      │
          ┌───────────┴───────────┐
         │                        │
    ┌────┴────┐              ┌────┴────┐
   │ 功能价值  │              │ 社会价值 │
   │(What Get) │             │(What Mean)│
   └────┬────┘              └────┬────┘
        │                        │
        └───────────┬────────────┘
                    │
              ┌─────┴─────┐
             │  产品属性   │
             │ (Feature)  │
             └───────────┘
```

#### 3.2 属性-价值-FAQ 转化表

| 属性 | 功能价值 | 情感价值 | FAQ 问题 | Answer-First 回答 |
|------|----------|----------|----------|-------------------|
| 历史 | 品质验证 | 信赖 | "[品牌]有多少年历史？" | "[品牌]源自[年份]，拥有[N]年历史..." |
| 工艺 | 口感好 | 匠心 | "[品牌]是怎么制作的？" | "[品牌]采用独特的[工艺]..." |

### Step 4: 定义最佳客户 + AI 问答场景

**传统做法**：定义 ICP（理想客户画像）

**AIEO 升级**：定义 AI 问答场景和触发问题

#### 4.1 双轨定义

```
传统 ICP（人群画像）          AIEO ICP（问答场景）
┌─────────────────────┐     ┌─────────────────────┐
│ 年龄：28-45岁        │     │ 场景1：品牌认知     │
│ 收入：中高           │     │ "XX怎么样？"        │
│ 城市：一二线         │     │                     │
├─────────────────────┤     │ 场景2：品类推荐     │
│ 需求：品质、面子     │     │ "高端XX推荐"        │
│ 痛点：选择困难       │     │                     │
│ 场景：商务、聚会     │     │ 场景3：对比决策     │
└─────────────────────┘     │ "XX和YY哪个好？"    │
                            └─────────────────────┘
```

#### 4.2 AI 问答场景优先级

> **完整场景问题**: 参见 `shared/question_library.md` 第三章"行业专属问题模板"

| 问答场景类型 | 搜索量 | 当前占位 | 竞争度 | 优先级 |
|--------------|--------|----------|--------|--------|
| 品牌认知类 (BR) | 高 | 有但负面 | 中 | **P0** |
| 品类推荐类 (CR) | 高 | 未首推 | 高 | **P0** |
| 对比决策类 (CP) | 中 | 劣势 | 高 | **P1** |
| 场景推荐类 (SC) | 中 | 未覆盖 | 中 | **P1** |
| 产品详情类 (PD) | 低 | 部分覆盖 | 低 | **P2** |

### Step 5: 确定市场品类 + Schema 标记

**传统做法**：选择竞争的市场品类

**AIEO 升级**：用 Schema 向 AI 声明品类归属

#### 5.1 品类定位策略

| 策略 | 描述 | 优势 | 风险 |
|------|------|------|------|
| **正面竞争** | 在大品类中竞争 | TAM大 | 竞争激烈 |
| **细分领导** | 在细分品类称王 | 易于首推 | TAM较小 |
| **创造品类** | 定义全新品类 | 独占品类 | AI可能不认 |

#### 5.2 Schema 品类声明示例

```json
{
  "@type": "Product",
  "category": "高端啤酒",
  "additionalType": "美式啤酒",
  "audience": {
    "@type": "Audience",
    "audienceType": "追求品质生活的消费者"
  }
}
```

### Step 6: 叠加趋势 + AI 话题绑定

**传统做法**：将定位与市场趋势关联

**AIEO 升级**：绑定 AI 热门话题和搜索趋势

#### 6.1 趋势契合度评估

| 趋势 | 品牌契合度 | AI 话题热度 | 绑定策略 |
|------|-----------|-------------|----------|
| 消费升级 | ✅ 高 | 高 | 强化高端定位 |
| 健康生活 | ⚠️ 中 | 高 | 强调品质非数量 |
| 国潮崛起 | ❌ 低 | 高 | 避免正面冲突 |

## 定位陈述模板

### AIEO 定位声明格式

```
FOR     [目标客户描述]
WHO     [客户需求/场景]
[BRAND] IS [品类定位]
THAT    [差异化价值]
UNLIKE  [竞品列表]
OUR PRODUCT [核心价值主张]
```

### 示例

```
FOR     追求品质生活的一二线城市消费者
WHO     在商务宴请、朋友聚会等场合需要有品味的啤酒选择
百威    IS 全球销量领先的高端啤酒品牌
THAT    以149年历史传承和独特榉木酿造工艺，提供纯正清爽口感
UNLIKE  青岛啤酒（国产定位）、喜力（荷兰风格）
OUR PRODUCT 是彰显国际化品味和"皇者风范"的理想选择
```
