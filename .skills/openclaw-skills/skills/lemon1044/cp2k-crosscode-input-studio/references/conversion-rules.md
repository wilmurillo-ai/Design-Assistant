# Conversion rules

This skill may convert a CP2K draft generated by this skill into draft inputs for other quantum-chemistry or materials codes.

## Scope

Supported conversion targets:
- Gaussian (`.gjf` / `.com`)
- VASP (`INCAR`, `POSCAR`, `KPOINTS`)
- ORCA (`.inp`)
- Quantum ESPRESSO (`pw.in`)

Best supported source path today:
- CP2K drafts generated by this skill itself

Less reliable but still possible:
- manually edited CP2K drafts that remain close to this skill's emitted structure

## Important boundary

Do not claim universal CP2K-to-anything translation.

The supported path is:
1. parse the normalized job spec and/or
2. parse the limited CP2K draft structure emitted by this skill's renderer
3. map the result into another code's draft input

This conversion logic is self-contained and does not rely on a local CP2K executable or installed CP2K library files.

If a CP2K input contains advanced manual blocks not represented in the normalized spec, treat conversion as partial and emit warnings.

## Gaussian mapping

Use for molecules or finite clusters.

Core mapping:
- `RUN_TYPE ENERGY` -> single-point route
- `RUN_TYPE GEO_OPT` -> `Opt`
- `RUN_TYPE VIBRATIONAL_ANALYSIS` -> `Freq`
- combine when needed, e.g. `Opt Freq`
- map `xc_functional` to a Gaussian route keyword when obvious (`PBE`, `B3LYP`, `PBE0`)
- preserve charge and multiplicity
- use Cartesian coordinates in Angstrom

Do not pretend periodic crystal jobs are naturally valid Gaussian jobs. Emit review warnings.

## ORCA mapping

Use mainly for molecules and finite clusters.

Core mapping:
- `RUN_TYPE ENERGY` -> single-point ORCA draft
- `RUN_TYPE GEO_OPT` -> `Opt`
- `RUN_TYPE VIBRATIONAL_ANALYSIS` -> `Freq`
- preserve charge and multiplicity
- use Cartesian coordinates in Angstrom

Warn if the source job is periodic.

## VASP mapping

Use for periodic systems, slabs, and periodic-like drafts.

Generate:
- `INCAR`
- `POSCAR`
- `KPOINTS`

Rules:
- `RUN_TYPE ENERGY` -> static SCF-like INCAR draft
- `RUN_TYPE GEO_OPT` / `CELL_OPT` -> ionic relaxation INCAR draft
- `PERIODIC NONE` molecular jobs may still be boxed into a large cell for a rough VASP-style draft, but mark this as a review item
- `KPOINTS` should mirror the heuristic grid if available
- never fabricate `POTCAR`; tell the user it must be assembled manually because it is pseudopotential-library dependent

## Quantum ESPRESSO mapping

Generate a conservative `pw.x` style `pw.in` draft.

Rules:
- map run type into `calculation='scf'` or `calculation='relax'` / `vc-relax`
- preserve charge and spin intent when practical
- map periodicity into k-point behavior heuristically
- use explicit `ATOMIC_POSITIONS angstrom`
- use `CELL_PARAMETERS angstrom` when cell data exists or is boxed heuristically
- do not fabricate exact pseudopotential filenames; emit placeholders and warnings

## Mandatory warnings

Always warn when:
- converting periodic jobs to Gaussian or ORCA
- converting nonperiodic molecular jobs to VASP or Quantum ESPRESSO periodic drafts
- exact basis/pseudopotential/library equivalence is impossible
- the source CP2K file contains content outside the parser's supported subset
