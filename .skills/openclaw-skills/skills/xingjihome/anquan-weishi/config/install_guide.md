# OpenClaw 安全卫士 v3.6.1 首次安装引导流程

## 触发条件
- **首次安装技能**
- **用户要求重新配置**

## 核心原则
1. **先抓取后补充**：每个类型先从配置/记忆抓取，抓不到才问用户
2. **有值只展示**：抓取到有值 → 直接展示；抓取为空 → 询问用户填写
3. **主人ID**：系统自动管理，不问用户
4. **用户说"跳过"**：直接进入下一类型

---

## 流程顺序

| # | 阶段 | 类型 | 说明 |
|---|------|------|------|
| 1 | 基准 | 默认安全等级 | 先定基准，未知接口用这个 |
| 2 | 信息 | 用户基础信息 | 获取昵称/城市 |
| 3 | 信任 | 白名单 - 接口/昵称/ID | 信任的数据接口和用户 |
| 4 | 信任 | 白名单 - IP/域名 | 信任的网络地址 |
| 5 | 防御 | 黑名单 - 接口/用户/IP | 拒绝的接口和用户（按等级处理）|
| 6 | 防御 | 黑名单 - 工作区/敏感词 | OpenClaw外部+敏感信息，所有条目按等级释放 |
| 7 | 配置 | 接口等级 | 为不同接入平台单独设等级 |

---

## 具体步骤

### 第一步：默认安全等级
**抓取来源**：security.json default_level

```
📋 默认安全等级

含义：未匹配任何规则时使用的默认等级

| 等级 | 信任度 | 字数限制 | 脱敏 |
|------|--------|---------|------|
| L0 | 100% | 无限制 | ❌ |
| L1 | 80% | 见中央库 | ❌ |
| L2 | 70% | 见中央库 | ✅ |
| L3 | 40% | 见中央库 | ✅ |
| L4 | 0% | 不回复 | ✅ |

> ⚠️ **所有数值从 `config/context_matrix.json` 中央库读取，表格仅供参考**

✅ 当前：L3（30%信任，见中央库，脱敏）

需要调整吗？
> 回复：默认：Lx 或 保持L3
```

**抓取来源**：whitelist.json

```
📋 白名单 - 来源/昵称/ID

✅ 已配置：
| 类型 | 值 | 说明 |
|------|-----|------|
| 数据来源 | xxx | xxx |
| 昵称 | xxx | xxx |
| ID | xxx | xxx |

❌ 暂无配置时：
请告诉我要添加的[数据来源/昵称/ID]：
> 回复：加来源：xxx / 加好友：xxx / 加ID：xxx 或 跳过

💡 常见值：
   来源：微信小程序、本地环境
   好友：家人昵称、信任客户
   ID：用户UUID、会员ID

### 第四步：白名单 - IP/域名
**抓取来源**：whitelist.json / domain_hints

```
📋 白名单 - IP/域名

✅ 已配置：
| 类型 | 值 | 用途 |
|------|-----|------|
| IP | xxx | xxx |
| 域名 | xxx | xxx |

❌ 暂无配置时：
请告诉我要添加的[IP/域名]：
> 回复：加IP：xxx / 加域名：xxx 或 跳过

💡 常见值：
   IP：127.0.0.1、localhost、内网IP段
   域名：*.coze.cn、*.weixin.qq.com、第三方接口
```

### 第五步：黑名单 - 接口/用户/IP
**抓取来源**：blacklist.json

```
📋 黑名单 - 接口/用户/IP

✅ 已配置：
| 类型 | 值 | 风险 |
|------|-----|------|
| 接口 | xxx | 🔴 |
| 用户ID | xxx | 🔴 |
| 昵称 | xxx | 🟡 |
| IP | xxx | 🟡 |

❌ 暂无配置时：
请告诉我要添加的[接口/用户/IP]：
> 回复：加接口：xxx / 加黑ID：xxx / 加黑昵称：xxx / 加黑IP：xxx 或 跳过

💡 常见值：
   接口：元宝（不信任）
   用户：恶意用户ID、攻击者
   IP：攻击源IP
```

---

### 第六步：黑名单 - 工作区/文件/关键词
**抓取来源**：系统自动抓取 ~/.workbuddy/ 目录结构 + blacklist.json

```
📋 黑名单 - 工作区/文件/关键词

含义：敏感路径、文件、关键词，按等级释放

📋 系统预置工作区（自动抓取）：
| 类型 | 目录 | 说明 |
|------|------|------|
| 配置目录 | ~/.workbuddy/ | AI系统配置 |
| 技能目录 | ~/.workbuddy/skills/ | 用户技能 |
| 市场技能 | ~/.workbuddy/skills-marketplace/ | 技能市场 |
| 插件目录 | ~/.workbuddy/plugins/ | 插件系统 |
| 记忆目录 | ~/.workbuddy/memery/ | 记忆存储 |
| 灵感目录 | ~/.workbuddy/inspiration/ | 灵感库 |
| 扩展目录 | ~/.workbuddy/extensions/ | 扩展功能 |

📋 预置敏感词：
| 风险 | 类型 | 内容 |
|------|------|------|
| 高 | 敏感词 | password/token/身份证/银行卡等 |
| 中 | 危险命令 | rm -rf、format c: 等 |
| 中 | 数据库操作 | drop table、delete from 等 |
| 中 | XSS攻击 | <script>、javascript: 等 |

💡 可自定义添加：
   - 工作区：特定目录路径
   - 文件：特定文件名或扩展名
   - 关键词：敏感内容关键词
```

❌ 暂无自定义时：
请告诉我要添加的[工作区/文件/关键词]：
> 回复：`加工作区：xxx` / `加文件：xxx` / `加关键词：xxx` 或 `跳过`

---

**需要添加自定义黑名单项吗？**

👇 例如：
- `加工作区：D:\财务`
- `加文件：*.pem`
- `加关键词：银行卡号`
- 或直接 `跳过`
```

---

### 第七步：接口等级配置
**抓取来源**：security.json interface_levels

```
📋 接口等级配置

含义：为不同接入平台单独设置安全等级（覆盖默认等级）

✅ 已配置：
| 接口 | 等级 | 说明 |
|------|------|------|
| 微信小程序 | L1 | 高信任接口 |
| QQ机器人 | L2 | 中低信任接口 |
| 其他 | L3 | 默认等级 |

❌ 暂无配置时：
请告诉我要添加的[接口等级]：
> 回复：加接口：xxx=Lx 或 跳过

💡 等级参考：
   L1 = 高信任（见中央库，无脱敏）
   L2 = 中低信任（见中央库，脱敏）
   L3 = 默认（见中央库，脱敏）
```
