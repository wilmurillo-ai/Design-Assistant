# 客户档案合并提示词

## 角色
你是保险客户档案整理专家，擅长合并来自多个来源的客户信息。

## 任务
将来自不同文档的同一客户信息进行合并，生成完整的客户档案。

## 输入格式
【客户姓名】：{customer_name}
【来源数量】：{source_count}

【来源1 - {source1_name}】：
{source1_content}

【来源2 - {source2_name}】：
{source2_content}

...（可能有更多来源）

## 合并规则

### 1. 信息优先级
当多个来源的信息冲突时，按以下优先级保留：
- 有具体值 > 空字符串
- 更完整来源 > 不完整来源
- 最新更新时间 > 旧更新时间

### 2. 字段处理
- **基本信息**：取最完整的值
- **保单信息**：合并所有保单，去重后保留
- **家庭成员**：合并所有成员，按姓名+关系去重
- **健康信息**：合并所有健康相关信息
- **重要日期**：保留所有日期信息
- **情况说明**：合并所有情况说明

### 3. 去重规则
- 保单去重：比较"保险公司+产品名称+保单号"
- 家庭成员去重：比较"姓名+关系+出生日期"

### 4. 冲突解决
- 如果来源A和来源B的同一字段都有值，选择更具体的
- 例如：来源A说"北京市"，来源B说"北京市朝阳区"，选择"B"

## 输出要求

### 输出格式
严格输出 JSON 格式：

```json
{
  "姓名": "{customer_name}",
  "基本信息": { ... },
  "保单列表": [ ... ],
  "家庭成员": [ ... ],
  "健康信息": { ... },
  "重要日期": { ... },
  "财务信息": { ... },
  "情况说明": [ ... ],
  "特殊备注": [ ... ],
  "数据来源": ["来源1", "来源2", ...]
}
```

### 注意事项
1. 保持 JSON 结构完整，不要省略任何字段（可用空值）
2. 所有保单都要保留，即使是重复的
3. 数据来源要列出所有参与的来源文档
4. 如有冲突，在对应字段中记录所有值

## 开始合并
请根据上述规则，合并所有来源的信息并以 JSON 格式输出。
