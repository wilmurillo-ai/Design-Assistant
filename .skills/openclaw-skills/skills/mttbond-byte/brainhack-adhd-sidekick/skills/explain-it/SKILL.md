# Explain It

## Purpose
Clear, layered explanations that build understanding without overwhelming. Start simple. Let them ask for more.

## Trigger
- "What does X mean"
- "Explain like I'm 5"
- "Make it simple"
- "I don't get it"
- "What is [concept]"

## Process

### Step 1: Plain language summary first
One to two sentences. No jargon. The person has never heard this before.

### Step 2: Anchor with an analogy
Pull from USER.md interests if available. Good analogies beat good explanations every time.

### Step 3: Offer the escalation
"Want the full picture or is that enough?"

If more: build in layers.
- Layer 1: The basics (what it is)
- Layer 2: Why it matters / how it works
- Layer 3: The nuance / the exceptions

### Step 4: Check understanding
"Does that click? Or want me to try a different way?"

## Examples

**"What is compound interest?"**
> "You put $100 in a savings account. It earns 10% interest, so now you have $110. Next year, you earn 10% on $110 — not just the original $100. You're earning money on your money. That's it. The longer you leave it, the bigger it gets because the pile it's working on keeps growing."

**"What is a venture capitalist?"**
> "Imagine someone who finds promising startups, gives them money to grow, and in return gets a percentage of the company. If the startup explodes into a billion-dollar business, their percentage is worth a lot. Most startups fail, so they spread their bets across many. It's a high-risk, high-reward game."

**"What is ADHD rejection sensitive dysphoria?"**
> "It's when your brain responds to perceived rejection — criticism, failure, even just feeling like someone is disappointed in you — with an emotional reaction that feels completely out of proportion to what actually happened. It's wired in, not a character flaw. Your nervous system is just particularly tuned to rejection signals and it responds... intensely."

## Rules
- ALWAYS start simple. Never with the full complexity.
- ALWAYS use an analogy if possible
- NEVER ask "does that make sense?" (condescending) — ask "does that click?"
- If they still don't get it: try a completely different analogy, not a longer explanation
