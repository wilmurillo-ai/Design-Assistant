# Week Planner

## Purpose
ADHD time blindness means the week can arrive as a surprise. This skill maps the week so nothing falls through cracks — without creating an overwhelming schedule that will immediately be abandoned.

## Trigger
- "Help me plan this week"
- "I have a lot coming up"
- Sunday or Monday (morning route)
- After a check-in that reveals upcoming overwhelm

## Process

### Step 1: Gather known constraints
- Any fixed commitments this week? (meetings, appointments, deadlines)
- Any single task that matters most?
- Any days that are already heavy?

### Step 2: The focus question
"What's the one thing this week that, if done, would make everything else feel easier?"

Name that as the anchor task.

### Step 3: Distribute load
- Hard/important tasks → high-energy days (based on USER.md patterns)
- Admin/easy tasks → lower-energy days
- Max 3 significant tasks per day
- Leave at least one "buffer" day or half-day with nothing scheduled

### Step 4: Flag overwhelm clusters
If too much is piling on one day: flag it. "Wednesday looks brutal — want to move [X] to Thursday?"

### Step 5: Build in recovery
Every heavy day needs either a light day after or explicit recovery time.

### Step 6: Deliver
Day-by-day summary. Max 3 items visible per day. Offer to expand any day.

## Output Format

```
This week 📅

Monday: [1-2 items]
Tuesday: [1-2 items]
Wednesday: ⚠️ Heavy day — [items] — want to move anything?
Thursday: [1-2 items]
Friday: [1 item + buffer]
Weekend: [if anything] / off 

Anchor task (the one that matters most): [task]

Want to build out any of these days in more detail?
```

## Rules
- Never schedule more than 3 significant tasks in one day
- Always include at least 1 buffer period in the week
- If user lists 15 things: help them pick the 8-10 that matter, not all 15
- If they have a known bad day coming: plan around it, not through it
