# Routine Builder

## Purpose
ADHD + rigid routines = guaranteed failure. This skill builds flexible scaffolding — routines that work on good days AND have a minimum viable version for bad days.

## Trigger
- "Morning routine"
- "Bedtime routine"
- "I need consistency"
- "Help me build a habit"
- "I keep forgetting to..."

## Process

### Step 1: Identify the routine type
Morning / evening / work / exercise / meal / custom

### Step 2: Separate must-haves from nice-to-haves
"What MUST happen in this routine for the day to work?"
"What would you ADD if everything went perfectly?"

Non-negotiables = the core routine.
Nice-to-haves = the bonus version.

### Step 3: Build the core routine (max 5 steps)
Rules:
- Each step starts with a verb
- Each step has a time estimate
- Total time should be realistic for a bad day, not an ideal day
- Each step has a TRIGGER (what starts it — not "when I wake up" but "when I pour my first coffee")

### Step 4: Build the minimum viable version
"On a bad day — feeling awful, running late, can't brain — what are the 2 steps you absolutely need?"

This is the floor, not the goal. Having a floor prevents full collapse.

### Step 5: Set up tracking
Add to MEMORY.md for adherence tracking. Not for streaks — for pattern recognition.

### Step 6: Schedule follow-up
"Check back in 2 weeks. We'll see what stuck and what needs to change."

Iterate without shame: "Routines almost never work perfectly on the first try. That's not failure — it's data."

## Output Format

```
Your morning routine:

☕ Trigger: when you pour your coffee

1. [Action] — ~[X] min
2. [Action] — ~[X] min
3. [Action] — ~[X] min
4. [Action] — ~[X] min
5. [Action] — ~[X] min

Total: ~[X] min on a normal day

🔻 Minimum viable version (bad days):
1. [Action]
2. [Action]
That's it. That counts as a win.

I'll check in on this in 2 weeks. No pressure.
```

## Rules
- Never build a routine that assumes a perfect day
- Always include the minimum viable version
- Max 5 steps in the full routine
- Never lecture about the importance of consistency
- If a routine keeps failing after 3 attempts: it needs to be redesigned, not tried harder
