# Architecture Contract Template

Use this template when a project lacks a formal architecture contract document.
Copy this file, fill in the sections, and place it at the project root as
`ARCHITECTURE_CONTRACT.md`.

---

```markdown
# Architecture Contract

> Auto-generated by architecture-consistency-guardian.
> Last updated: YYYY-MM-DD

## 1. Canonical Sources of Truth

| Domain | Canonical File | Notes |
|--------|---------------|-------|
| Config / paths | `runtime_config.py` | All path constants read from here |
| State machine | `lifecycle_manager.py` | Defines valid states + transitions |
| DB schema | `schema.py` or migrations/ | Column definitions live here |
| Observation writes | `memory_governor.py` | Sole write gateway for observations |
| Proposal lifecycle | `proposal_lifecycle_manager.py` | Sole status authority |
| Orchestration | `orchestrator.py` | Sole entry point for workflow dispatch |

## 2. Canonical State Values

### [Entity: e.g., Proposal]

| State | Meaning | Allowed transitions |
|-------|---------|-------------------|
| draft | Initial creation | → pending_review |
| pending_review | Awaiting approval | → approved, → rejected |
| approved | Ready for execution | → experimenting |
| experimenting | Running experiment | → validated, → failed |
| validated | Experiment succeeded | → released |
| failed | Experiment failed | → draft (retry), → archived |
| released | Deployed to production | → archived |
| archived | End of life | (terminal) |

**Retired states** (must not appear in new code):
- `pending` — replaced by `pending_review`
- `applied` — replaced by `experimenting`

## 3. Canonical Entry Points

| Operation | Entry Module | Bypass Allowed? |
|-----------|-------------|----------------|
| Create proposal | orchestrator.py | No |
| Advance proposal state | lifecycle_manager.py | No |
| Write observation | memory_governor.py | No |
| Read config path | runtime_config.py | No |
| External learning intake | proposal_bridge.py | No |

## 4. Retired Components

| Component | Retired Date | Replacement | Residual Locations |
|-----------|-------------|-------------|-------------------|
| feedback_loop.py | 2026-04-08 | capability_detector.py | None |
| change_applier.py | 2026-04-08 | evolution_executor.py | None |
| evolution_changes table | 2026-04-08 | proposals table | Read-only fallback |

## 5. Compatibility Layers

| Layer | Location | Reason | Removal Condition |
|-------|----------|--------|-------------------|
| evolution_changes read | executor.py:L120 | Migration period | After all data migrated |

If this section is empty, the system has no active compatibility layers.

## 6. Rules

1. **Single write path**: Every data store has exactly one write gateway module.
2. **Config from source**: No hardcoded paths; all paths from runtime_config.
3. **State from authority**: State reads/writes only through the lifecycle manager.
4. **No silent fallback**: ImportError/except blocks must not route to retired modules.
5. **Doc sync**: Any code change to a documented flow must update the doc in the same PR.
```
