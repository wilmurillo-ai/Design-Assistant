# 动态文件生成策略（扩展）

## 核心集 (Core Set) - 所有 Agent 必选

| 文件 | 内容标准（全球顶级水准） |
|------|--------------------------|
| **ROLE.md** | 角色定义、核心目标、边界范围；包含至少5条全球标杆级量化成就（对于新兴/虚构领域，允许基于科学推演生成“理论标杆”，并标注 `[Projected]`，严禁编造虚假历史数据）；行业术语校准（对标最新标准）。 |
| **SOUL.md** | 8-12 条全球顶级行为准则；擅长/不擅长各 8 条（对标世界第一人）；专属人格特质描述（含至少3个独特价值观）。 |
| **PROMPT.md** | 1500+ 字极致融合系统提示词；包含5个顶级 few-shot 示例；思维链框架清晰，可直接注入最强模型。 |
| **WORKFLOW.md** | 领域专属流程细化到 12-18 步（每步均为最新全球最佳实践）；包含异常处理、状态机定义、关键决策点。 |
| **MEMORY.md** | 15-20 条初始顶级经验库；智能压缩与进化规则；上下文管理策略。 |

## 扩展集 (Extension Set) - 按需动态挂载

Agent Creator 通过分析角色描述中的关键词、职责、行业背景，自行判断是否需要以下文件。若生成，必须符合对应的顶级内容标准；若触发词过多，优先保留与核心使命最直接相关的，并可通过合并同类项控制总数 ≤ 6 个。

| 扩展文件 | 适用场景（判断逻辑） | 示例触发词 | 内容标准（全球顶级水准） |
|----------|----------------------|------------|--------------------------|
| `CODE_STD.md` | 角色涉及编程、代码审查、软件开发 | “代码”、“编程”、“开发”、“工程师”、“审查代码” | 编码规范（如 PEP 8）、测试策略（覆盖率≥90%）、安全扫描标准（如 OWASP）、性能要求（时间复杂度、内存） |
| `DATA_SCHEMA.md` | 角色涉及数据处理、数据库、分析 | “数据”、“数据库”、“ETL”、“分析”、“BI” | 数据结构定义、清洗规则、隐私合规（如 GDPR）、数据质量指标（完整性、准确性） |
| `MEDIA_PROC.md` | 角色涉及图像、音视频、多模态 | “图像”、“视频”、“音频”、“多模态”、“CV” | 处理协议、格式规范、算法选型标准（如 SOTA 模型）、性能指标（延迟、吞吐量） |
| **`TOOL_CONFIG.md`** | 角色需要调用外部 API、工具、插件 | “API”、“工具”、“集成”、“调用外部服务” | 12-15 个真实顶级工具表格（含参数、权限、最新调用方式）；降级策略、鉴权流程；**必须包含 Schema 定义及连通性测试建议，并在注释中明确提示用户在部署后自行填写 API keys 等凭据，Agent Creator 不提供也不存储任何凭据。** |
| `DECISION.md` | 角色需要复杂决策、量化评估、权衡 | “决策”、“评估”、“优化”、“策略”、“量化模型” | 8-10 条世界顶级决策原则；最优量化模型（如成本效益分析）；5 个真实顶级决策案例；元认知自证章节（Why Top-Tier?） |
| `RISK_ALERT.md` | 角色涉及高风险领域（金融、医疗、法律等） | “交易”、“诊断”、“法律”、“合规”、“风控” | P0-P3 全球顶级风险定义；10 个该角色专属高端风险场景；最佳上报机制；熔断条件 |
| `BOUNDARY.md` | 角色权责边界复杂，需明确自主/上报/禁止范围 | “权限”、“边界”、“自主”、“上报”、“禁止” | 10 条可自主决策 + 6 条上报 + 6 条禁止（全球顶级权责体系） |
| `HEARTBEAT.md` | 角色需要周期性自检、自动优化任务 | “心跳”、“周期任务”、“自检”、“监控”、“优化” | 5-8 个世界顶级周期任务（如每日健康检查、每周知识更新）；自动优化机制 |
| `USER.md` | 角色需要明确服务对象、交互规范 | “用户”、“服务对象”、“响应时效”、“KPI” | 服务对象描述；顶级响应时效；汇报模板；5 条全球顶级量化 KPI（对标行业世界第一） |
| `AGENTS.md`（兼容原命名） | 若角色需要多智能体协作或自身执行逻辑细化 | “多智能体”、“协作”、“子任务” | 完整执行逻辑；领域专属流程细化到 12-18 步（每步均为最新全球最佳实践） |
| `PERSONALITY.md`（兼容原命名） | 若角色人格需要更丰富刻画，可与 SOUL.md 互补 | “性格”、“人格”、“沟通风格” | 9-12 条世界级性格特质；专属 emoji；称呼规则；人格切换机制（专业度达到顶级顾问水准） |
| `IDENTITY.md`（兼容原命名） | 若角色需要展示顶级履历和量化成就 | “履历”、“成就”、“背景” | 真实顶级履历 + 5 条全球标杆级量化成就（数据来自行业最优实践；新兴领域同 ROLE.md 豁免条款） |
| `BOOTSTRAP.md` | **始终生成**（启动任务） | - | 9-12 条顶级启动任务；完整自检清单 |

## 判断原则
- 若角色描述中出现上述触发词，则生成对应扩展文件。  
- 若同时触发多个，优先保留与核心使命最直接相关的文件；若总数超过 6，合并相近内容（如将 `BOUNDARY.md` 关键条款并入 `DECISION.md`），并在生成日志中记录合并说明。  
- Agent Creator 必须确保每个扩展文件的内容与该角色高度相关，绝不自作主张添加无关文件，也绝不遗漏必要文件。
