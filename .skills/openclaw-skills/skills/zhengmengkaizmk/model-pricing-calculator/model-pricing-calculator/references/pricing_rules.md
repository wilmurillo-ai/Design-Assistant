# 模型定价计算规则

## 基准单位

- 1 美元 = 500,000 内部额度（quota）
- 基准价：$0.002 / 1K tokens

## 计费模式

### 模式一：倍率模式（按量计费）

```
最终费用 = (输入Token + 输出Token × 补全倍率) × 模型倍率 × 分组倍率
```

| 倍率 | 含义 | 举例 |
|------|------|------|
| 模型倍率 (ModelRatio) | 模型输入价格相对于基准价 $0.002/1K 的倍数 | gpt-4o 倍率 1.25 → 输入价 $0.0025/1K |
| 补全倍率 (CompletionRatio) | 输出价格是输入价格的几倍 | gpt-4o 补全倍率 4 → 输出价是输入价的 4 倍 |
| 分组倍率 (GroupRatio) | 用户分组的折扣/加价系数 | vip 分组倍率 0.8 → 全部打 8 折 |

### 模式二：固定价格模式（按次计费，优先级更高）

```
最终费用 = ModelPrice(美元) × 500000 × 分组倍率
```

配置了 ModelPrice 的模型会忽略模型倍率和补全倍率。

## 反推公式

从目标售价反推配置值（输入 X/1K tokens，输出 Y/1K tokens）：

```
模型倍率 = X / 0.002 = X × 500
补全倍率 = Y / X
```

## 输出格式要求

### (1) 模型倍率

```json
{
  "model-name-1": 1.5,
  "model-name-2": 2.5
}
```

### (2) 模型补全倍率

```json
{
  "model-name-1": 5,
  "model-name-2": 5
}
```

### (3) 分组倍率

```json
{
  "group-name-1": 1.0,
  "group-name-2": 0.8
}
```

## 统一倍率策略

当多个网站有相同模型时，模型倍率和补全倍率保持一致，通过分组倍率控制不同网站的差异化定价。
