3.2 场景管理
设备需要和场景绑定后，刷掌才会上传核验记录，核验记录可以根据场景来查看。
● 场景组：表明区域，如“腾讯大厦-腾讯大厦 31 楼”，“北京大学-北京大学图书馆”，
均可建立为场景组；
● 场景：表明真实的铺设点位，如“31 楼进门闸机”；
3.2.1 场景组管理
成员登录管理后台后，根据自身角色，仅可查看所管理的场景组数据，仅可针对所管理的场
景组进行新建，编辑，删除，查看其下场景的操作。
3.2.1.1 新建场景组
● 页面新建：点击场景组后方的设置按钮，点击新建场景组，输入场景组名称完成添加，
新增场景组为当前场景组的下级场景组。可根据实际管理需求，建立多级场景组；
注意：同一个场景组下不可添加同名的场景组；如果当前场景组存在同名的场景，创建的时
候提醒“当前场景组存在同名场景，请修改名称后新建”
3.2.1.2 编辑场景组
● 页面编辑：点击场景组后方的设置按钮，点击编辑场景组，可修改名称；

● 模组升级：指的是对生物识别模组（下位机）上的固件进行更新和优化的过程。模组固
件是嵌入式控制程序，负责硬件底层驱动、信号处理及协议通信。模组升级主要目的是提升
硬件性能、增加新的功能支持、修复已知的硬件故障或提高系统稳定性。
● 系统升级：指的是对上位机的系统固件进行更新和优化的过程。系统固件是运行在主控
单元的操作系统底层程序，负责设备管理框架和资源调度能力。通过系统升级，可以优化设
备管理、提高资源分配效率、增强设备与模组间的通信能力，并修复系统级的错误或漏洞。
● 应用升级：指的是对上位机的应用软件进行更新和优化的过程。应用软件是面向具体场
景的业务逻辑实现层，提供用户交互界面和系统集成能力。应用升级通常涉及改进用户体验、
增强应用功能、修复应用层面的缺陷，或是根据业务需求加入新的功能模块。
在进行多种升级时，请遵循以下的顺序：系统升级 →模组升级→应用升级，自动重启后完成
升级。

常见问题解决
未注册
若出现该页面，则表明该设备未绑定刷掌管理平台，需要在管理后台进行设备录入：
● 在机身背面标签处找到设备 SN 码
● 根据设备 SN 码，在平台进行绑定，详见管理后台用户手册
授权异常如何处理
授权异常可能是由于设备时间不对（拉不到证书）、设备序列号拿不到/重置发生变化、设备
部件更换导致异常、设备被攻击等。
解决方案：重新走授权流程 （可能会超出合同数量）参考上文：刷掌鉴权。
授权过期如何处理
请联系对应销售进行续期。
核验机/门禁机通电不显示
● 注意使用原装电源适配器，如发现电源电压不足或过压保护，用万用表量适配器电压是
否达到标准 12V。
设备刷掌速度慢，怎么办？
● 确认已撕掉保护膜，检查摄像头区域是否清洁。
● 调整手掌高度（5-12 厘米）并保持掌心对准摄像头，避免逆光环境。
● 确保掌心干净没有脏污、汗渍。
● 若多次失败，尝试删除原有信息并重新注册。
● 可能因网络延迟（云端验证设备），建议检查 Wi-Fi/4G 信号。
● 手掌位置偏移或光线过强也会影响响应速度。

推荐使用哪种网络连接方式？
● 优先推荐有线连接，10 台以上不建议使用 Wi-Fi。
有线网络布线的具体要求是什么？
● 网线长度不超过 100 米（建议 80 米内），中间无接口、无死弯或挤压。
● 使用 CAT5e 线缆及标准水晶头，布线前用测线仪测试通过。
● 线缆需用 PVC 管或线槽保护，确保安全美观。
● 每个窗口刷掌机需独立插座，避免与其他设备共用空气开关。
有线网络连接时不同设备数量对应的带宽和组网要求？
● 1-4 台 ：共用网络带宽≥20Mbps，直连路由器。
● 5-30 台 ：带宽≥100Mbps，用企业级路由器+管理交换机（≤4 个）。
● 31 台以上
● ：专线带宽≥100Mbps，需光猫+企业级路由+千兆交换机，每设备划分独立 VLAN。
有线网络连接时交换机部署的限制条件是什么？
● 同一局域网内，交换机串联不得超过 3 级，避免网络延迟和性能下降。
无法连接有线网络
● 网线水晶头没压好：检查水晶头是否有松动，线是否压到位。
● 网线两端水晶头线序没统一：检查网线两端水晶头线序是否统一标准打法（常规使用国
际标准）
● TIA/EIA-568-B 做法。
无法连接 Wi-Fi
● WiFi 网络信号不好：确认设备与无线路由器距离是否太远，或是否有金属类屏蔽物遮挡
设备，如果无上述问题，可尝试更换信号强的 WiFi 网络；如设备安装在地下室 WiFi 信号
无法满足时建议使用有线网络。
● 使用了频段不匹配：确认路由器 2.4GHz 频段已开启
● 密码使用了特殊字符：检查密码是否含特殊字符（建议使用字母+数字组合）
Wi-Fi 已连接但无网络
● 确认是否同时使用了 WIFI 和以太网，WIFI 和以太网只单独使用一种
● 检查无线路由器和其他网络设备是否能正常上网
注册模式，识别模式有什么区别？
● 注册模式下仅支持注册，可以在批量注册时使用。
● 识别模式下仅支持识别，可以在所有人员注册后使用。
