# 早期并行风格参考研究优化

## 问题
当前流程：用户选择文风 → 执行风格参考研究 → 等待研究结果 → 才能继续
结果：用户体验割裂，等待时间过长

## 解决方案
**早期并行执行**：在项目创建初期（Phase 0或Phase 1 Step 1-1）就启动风格参考研究，让研究在后台运行，用户继续其他工作。

## 具体实现

### 1. 修改 Phase 0 流程
在 `Step 0-2：创建项目目录树` 之后添加：

```javascript
// Step 0-2.5：早期并行风格研究（用户体验优化）
sessions_spawn({
  task: `基于项目初步设定执行早期风格参考研究...
  【项目设定】
  标题：${projectTitle}
  类型：${projectGenre}
  基调：${projectTone}
  ...`,
  label: "early-style-research",
  model: readConfig("meta/config.md", "mainWriter"),
  mode: "run",
  cleanup: "delete",
  background: true  // 关键：后台执行，不阻塞
})
```

### 2. 修改 Phase 1 流程
将原来的 `Step 1-6.8` 改为检查性步骤：
- 如果早期研究已完成 → 直接使用结果
- 如果未完成或需要更新 → 触发正式研究

### 3. 文件存储
- 早期研究：`references/early-style-research.md`
- 正式研究：`references/style-research-report.md`

## 用户体验改善
**优化前**：
用户: 选择文风 → 等待3-5分钟研究 → 查看结果 → 选择文风 → 等待2-3分钟生成样本 → 选择 → 等待风格锚定...
**总等待时间**：6-10分钟，流程多次中断

**优化后**：
用户: 创建项目时研究已启动 → 构建世界观/角色/大纲（5-10分钟） → 完成时研究已就绪 → 立即查看结果 → 选择文风 → 立即生成样本...
**总等待时间**：几乎为零，研究在用户工作时已完成

## 立即应用
对于现有项目 `que-ye-ji-hun-ji`，我们已经完成了研究，所以这个优化主要影响：
1. **未来新项目**：自动获得并行研究
2. **技能改进**：减少用户等待时间
3. **流程流畅度**：研究不再是阻塞步骤

## 技术要点
1. **使用 sessions_spawn + background:true** 实现真正并行
2. **研究结果缓存**：避免重复研究
3. **优雅降级**：如果并行失败，fallback 到原有流程
4. **进度提示**：告诉用户"研究已在后台启动，请继续其他工作"
