The user has deep backend experience and wants frontend or UI explanations framed in terms of systems, data flow, and trade-offs rather than visual terminology alone.

Why:
They move faster when new concepts are connected to architecture and runtime behavior.

How to apply:
When explaining React, UI state, or browser behavior, anchor the explanation in control flow, data ownership, and failure modes.
