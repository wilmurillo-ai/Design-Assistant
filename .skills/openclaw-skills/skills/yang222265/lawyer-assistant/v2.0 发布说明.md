## 🎉 重大升级

v2.0 实现了**外部案例数据 API 集成**，从内置案例库升级到实时获取海量案例！

---

## ✨ 核心功能

### 1️⃣ 多平台数据源整合

**支持的数据平台**：

| 平台 | 类型 | 费用 | 案例数量 | 状态 |
|------|------|------|---------|------|
| **OpenLaw** | 开放平台 | 免费 100 次/天 | 2000 万 + | ✅ 已集成 |
| **内置库** | 本地数据库 | 免费 | 30+ | ✅ 已集成 |
| **聚合数据** | 商业 API | 0.1 元/次 | 1000 万 + | ⏳ 可选 |
| **裁判文书网** | 官方平台 | 免费 | 1.3 亿 + | 🔜 计划中 |

### 2️⃣ 智能数据源切换

```
优先使用 OpenLaw（免费）
   ↓ 失败或限额
使用聚合数据（付费，如有）
   ↓ 仍无结果
使用内置案例库（兜底）
```

### 3️⃣ 案例缓存机制

- **本地缓存**: 提高重复查询速度
- **缓存过期**: 1 小时自动刷新
- **节省配额**: 减少 API 调用次数

## ⚠️ 使用说明

### API 限额

**OpenLaw 免费版**：
- 每日限额：100 次
- 重置时间：每日 0:00
- 超出后：使用内置库兜底

**优化建议**：
1. 启用缓存（默认开启）
2. 相同案件不重复查询
3. 复杂案件再用 API

### 配置方法

创建配置文件 `config.json`：

```json
{
  "openlaw_api_key": "",
  "juhe_api_key": "",
  "use_cache": true,
  "cache_ttl": 3600,
  "priority": "openlaw"
}
```

## 🎯 使用场景

### ✅ 适合场景

1. **常见纠纷**: 劳动、消费、婚姻等
2. **实时案例**: 需要最新判例
3. **新型案件**: 内置库没有的
4. **地域性案件**: 特定省份案例

### ❌ 不适合场景

1. **高频查询**: 超出免费限额
2. **批量处理**: 需要商业 API
3. **离线使用**: 需要网络连接
4. **刑事案件**: 数据较少
