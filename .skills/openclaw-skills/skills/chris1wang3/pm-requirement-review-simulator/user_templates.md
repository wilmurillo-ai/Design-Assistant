# 快速启动模板

> 复制模板填写后提交。技能会先返回「信息采集清单」让你确认/补充，再自动生成完整 HTML 深色主题报告。
>
> **使用技巧**：
> - 信息越具体，报告越精准（示例："提升转化率" < "新客转化率提升 15%，7 日复购率提升 5%"）
> - 有现成 PRD 可直接贴内容或 URL，不必逐项填空
> - 不确定字段可留空，技能会先推断并在采集清单里让你确认
> - 想压测方案质量可直接选 🔴 地狱模式

---

## 1) 通用骨架（先填这个）

```text
请使用《产品经理需求评审模拟器》对以下需求做完整攻防推演。

【评审模式】（三选一）
- 🟢 新手村（温柔建议，每角色 5-8 个问题）
- 🟡 实战（标准大厂流程，每角色 10-12 个问题，默认）
- 🔴 地狱（全员恶人，每角色 12-15 个问题，含黑话与高压质疑）

【需求文档】
- 需求名称：[例如：会员体系 2.0]
- 需求类型：[大型产品功能 / 工具类 / MVP验证期 / 迭代优化 / 合规敏感型]
- 业务目标：[核心目标 + 可量化指标]
- 核心功能：[列出 3-5 个核心功能点]
- 目标用户：[C 端用户 / 内部运营 / 策略主理人 / ...]
- 当前版本/里程碑：[例如：v3.2，计划 3 周上线]
- 验收标准：[例如：留存提升 2%，故障率 < 0.1%]

【评审角色】
- 技术 / 运营 / 设计 / 老板 / 法务（默认全开，可删减）

【公司背景】
- 行业：[电商 / 金融科技 / B2B SaaS / 教育 / ...]
- 公司规模：[例如：500 人，成长期]
- 技术栈：[例如：Java + Spring + MySQL + Redis]
- 合规等级：[低 / 中 / 高]

【约束条件】
- 预算上限：[例如：50 万]
- 上线期限：[例如：21 天]
- 人力上限：[例如：前后端共 5 人]
- 风险偏好：[低 / 中 / 高]

【PRD 来源】
- 📄 用户自有 PRD / 🌐 公开/展示版 PRD

【特别关注（可选）】
- 最担心被挑战的问题
- 已知风险
- 竞品名称与差异点
- 历史失败案例
- 本次评审必须拍板事项（最多 3 项）
```

## 2) 行业增量块（按需追加）

> 用法：先复制「通用骨架」，再把对应行业增量块追加到同一条消息末尾。

### A. 电商增量块

**电商评审重点**
- 大促并发抗压（技术必问）
- 推广路径与冷启动（运营必问）
- 与现有产品矩阵的分流/重叠风险（老板必问）
- 营销宣传合规、排名规则、UGC 治理（法务必问）

```text
【行业增量：电商】
- 业务阶段：[大促备战期 / 日常迭代 / 新业务冷启动]
- 大促并发：[预估峰值 QPS、限流/降级方案]
- 与现有模块关系：[例如：与“必买清单”“金选”的定位差异]
- 流量分配规则：[算法推荐 vs 运营人工挑选]
- 竞品对标：[拼多多百亿补贴 / 京东排行榜 / ...]

【输出额外要求】
1) 技术与运营在“稳定性 vs 大促档期”上的冲突化解
2) 老板与法务在“增长速度 vs 合规风险”上的决策收敛
3) 与现有产品矩阵的协同/冲突分析
```

电商展示版 PRD 快捷句：

```text
请评审以下电商公开 PRD，🟡 实战模式。
[贴 URL 或文章内容]
```

### B. B2B SaaS 增量块

**B2B SaaS 评审重点**
- 数据权限与多租户隔离（技术必问）
- 续费率、工单量下降等 CS 指标（运营/客户成功必问）
- 竞品功能差距与定价影响（老板必问）
- 数据合规、审计日志完整性、SLA 承诺（法务必问）

```text
【行业增量：B2B SaaS】
- 客户画像：[中大型企业 / SMB / 混合]
- 多租户要求：[是否需要租户级数据隔离]
- 合规要求：[SOC2 / GDPR / 审计要求]
- 客户反馈：[Top 客户诉求及续费影响]
- 定价影响：[是否作为高级套餐差异化能力]

【输出额外要求】
1) A/B/C 方案取舍表（价值、成本、周期、风险）
2) 跨部门 RACI 分工和关键依赖最晚交付点
3) Conditional Go 的具体条件与复盘节点
4) 客户可演示 MVP 的最小功能范围
```

B2B SaaS 展示版 PRD 快捷句：

```text
请评审以下 B2B SaaS 公开 PRD，🟡 实战模式。
[贴 URL 或文章内容]
```

### C. 金融合规增量块

**合规是一票否决项。**

**金融评审重点**
- 收益模拟/预测是否触碰监管红线（法务致命级）
- 免责声明、风险提示位置与措辞（上线前置项）
- 用户隐私与投资者适当性义务（法务必问）
- 算法精度与跨设备一致性（技术必问）
- 工具定位（内部 vs C 端）是否明确（全员必对齐）

```text
【行业增量：金融科技】
- 持牌情况：[基金销售牌照 / 投顾资质 / ...]
- 监管要求：[不能触碰误导销售红线等]
- 审批前置：[法务与风控双审]
- 数据处理：[纯前端计算 / 服务端上传]
- 投资者适当性：[是否需风险测评前置]
- 免责声明方案：[几层防护，导出/截图是否附带免责声明]

【输出额外要求】
1) 法务高风险问题清单和应对话术
2) 合规闭环方案（免责声明 + 风险提示 + 导出免责 + 适当性管理）
3) No Go 触发条件
4) 上线后 T+7/T+30 指标与回滚预案
5) 内部工具 vs C 端工具的合规标准差异
```

金融内部工具快捷句：

```text
请评审以下金融内部工具需求，🟡 实战模式。
- 需求名称：[例如：主理人提款计算器]
- 目标用户：策略主理人（内部工具）
- 业务目标：提升沟通效率，降低返工率
[补充核心功能和约束条件]
```

## 3) 展示版 PRD 通用快捷模板

当你评审公开文章/案例时，直接用：

```text
请使用《产品经理需求评审模拟器》评审以下公开 PRD。

【评审模式】🟡 实战模式
【PRD 来源】[URL 或文章标题+来源]
【PRD 内容】
[直接粘贴正文]
```

> 展示版 PRD 会自动豁免去敏导致的信息缺失（如内部架构、ROI 细项、审批细节），聚焦已展示内容质量。结构完整、逻辑清晰的展示版 PRD 通常可达 S 级（85+）。
