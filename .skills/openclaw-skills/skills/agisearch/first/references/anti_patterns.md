# First Principles Anti-Patterns

These are common failure modes when trying to reason from first principles.

## Anti-Patterns

- Confusing novelty with fundamentals.
- Rejecting convention just to sound original.
- Treating abstraction as insight.
- Optimizing a visible step instead of the root constraint.
- Confusing tools with user value.
- Assuming common behavior is evidence of correctness.
- Repeating category hype instead of testing objective reality.
- Treating one best practice as universally correct.
- Rebuilding everything when only one bottleneck matters.
- Giving philosophical answers without concrete next actions.

## Warning Signs

- The answer sounds smart but does not change a decision.
- The reasoning attacks convention without proving why convention fails.
- The rebuilt solution is less testable than the original plan.
- The output uses trendy terms instead of variables, constraints, and incentives.
