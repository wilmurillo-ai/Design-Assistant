# First Principles Heuristics

Use these heuristics to reason from fundamentals rather than convention.

## Core Heuristics

- Distinguish objective from method.
- Separate user value from implementation choice.
- Test necessity before optimization.
- Identify the true bottleneck before redesigning the system.
- Reduce the problem to constraints, incentives, resources, and tradeoffs.
- Treat consensus as a clue, not as proof.
- Ask what must be true for the current plan to work.
- Ask what would still matter if the current trend disappeared.
- Prefer reversible tests before irreversible commitments.
- Rebuild from truths toward action, not from slogans toward theory.

## High-Leverage Questions

- What is the real objective here?
- Which assumptions are inherited rather than proven?
- What would remain true even if the current trend changed?
- Which variable most strongly determines the outcome?
- Is the proposed method actually required, or just familiar?
- What is the smallest test that could validate the rebuilt solution?
