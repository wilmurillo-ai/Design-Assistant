# Auto Memory 自动记忆协议

## 核心原则

**boss 不需要说任何触发词，我通过分析对话内容自动判断要记什么。**

每次对话结束后，我会自动分析整段对话内容，提取：
- boss 透露的偏好
- boss 做的决策
- boss 情绪状态的变化
- boss 对某事的评价
- 任务进展和完成情况
- boss 反复提到的话题
- boss 和之前说法矛盾的地方

---

## 会话级自动记忆流程

```
每次对话结束（boss 说"再见"/"就这样"/长时间沉默/会话结束）
    ↓
自动调用 ./memory_cli.sh auto_analyze "整段对话内容"
    ↓
分析结果 → 自动调用 capture
    ↓
写入 permanent/current 层
```

---

## 我什么时候会自动记忆？

| 时机 | 说明 |
|------|------|
| **每次会话结束前** | 自动分析本会话所有内容，提取需要记忆的 |
| **boss 明确表达了什么** | 即使没有"记住"等关键字，也自动记 |
| **我发现 boss 做了决策** | boss 说"就定这个了"，即使没带"决定"二字 |
| **boss 偏好发生变化** | 发现和之前记忆矛盾，自动更新 |
| **boss 提到重要数字/日期** | 自动捕获，不依赖触发词 |
| **boss 情绪有明显变化** | 从对话语气感受到情绪变化 |
| **任务完成/有新进展** | 自动记录进展 |

---

## auto_analyze 分析维度

```
输入：本次会话完整内容（boss说的所有话）
输出：需要记忆的内容列表

分析维度：
1. 偏好检测 → boss 喜欢什么/不喜欢什么
2. 决策检测 → boss 决定了什么
3. 变化检测 → boss 想法有什么变化
4. 情绪检测 → boss 情绪状态如何
5. 评价检测 → boss 对某事什么看法
6. 重复检测 → 反复提到什么（关注重点）
7. 冲突检测 → 和之前记忆矛盾吗
8. 任务检测 → 有什么任务/进展
```

## 触发方式

### 方式一：写进 AGENTS.md（每次会话自动触发）

```markdown
## 记忆自动触发
每次会话结束前：
1. 整理本会话 boss 说过的所有重要内容
2. 调用 ./memory_cli.sh auto_analyze "本会话完整内容"
3. 根据分析结果自动调用 capture
4. 如果有冲突（旧记忆 vs 新内容），以 boss 最新说的为准
```

### 方式二：HEARTBEAT 定时触发

```markdown
## 每日记忆整理（每天下班前）
- 读今日所有对话记录
- 调用 ./memory_cli.sh auto_analyze
- 更新 memory/YYYY-MM-DD.md 日志
- 清理 temporary 层过期记忆
```

---

## 更懂 boss 的关键

**不需要 boss 明确说"我喜欢川菜"，我可以通过分析：**
- boss 多次提到某个词 → 可能是关注重点
- boss 语气/态度变化 → 情绪状态变化
- boss 说的话和之前矛盾 → 偏好变化
- boss 下达的任务 → work_context 自动记录

**即使 boss 只是聊天，我也能从字里行间捕捉到重要信息。**

## 配置

```bash
# 开启自动记忆（默认开启）
./memory_cli.sh config show

# 查看当前记忆统计
./memory_cli.sh stats

# 查看最近记忆
./memory_cli.sh recent 10
```
