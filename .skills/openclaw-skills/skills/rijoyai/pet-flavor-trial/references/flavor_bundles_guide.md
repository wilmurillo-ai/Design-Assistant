## Assortment shapes

| Shape       | Flavors | Use when                          |
|-------------|---------|-----------------------------------|
| Mini trial  | 2–3     | Low commitment; small catalog     |
| Standard    | 4       | "Taste the range" hero offer      |
| Full range  | 5–6     | Large line; "try everything"      |

Prefer one unit per flavor for a true trial; two per flavor if the user wants a "stock-up" assortment.

## Naming patterns

- **Explorer**: "Flavor Explorer," "Taste the Range."
- **Trial**: "4-Flavor Trial Pack," "Flavor Trial Box."
- **Family bucket**: "Flavor Family Bucket," "All Flavors Assortment."
- Keep one name consistent across PDP, cart, and analytics.

## Copy patterns (EN)

- PDP: "Let your pet taste the range. One pouch of each: [list flavors]."
- Cart: "Flavor Explorer (4 pouches) — Chicken, Beef, Salmon, Sweet Potato."
- Banner: "Try all 4 flavors in one pack — discover their favorite."

## Scope

- **Single product line**: e.g. freeze-dried only, or chew sticks only.
- **Cross-line**: Only if the user explicitly wants a mixed treat type bundle; usually keep one line per bundle for clarity.

Use this file when you need to double-check assortment size or copy without re-reading the full SKILL.
