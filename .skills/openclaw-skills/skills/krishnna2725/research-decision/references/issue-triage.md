# Issue Triage

当问题更像 bug、冲突、异常、兼容性故障时，优先把检索和判断重心放在"已知问题"和"止损路径"上，而不是强行深挖根因。

## 优先回答的问题

- 这是不是已知问题？
- 是否与特定版本、平台、依赖组合有关？
- maintainer 或官方是否确认过？
- 有没有 workaround、补丁、替代配置或绕路方案？
- 社区是否普遍建议升级、降级、锁版本、关闭某特性或放弃该方案？
- 继续排查的收益是否高于止损成本？

## 常见检索方向

中文：
- `"<主题> bug"`
- `"<主题> 冲突"`
- `"<主题> workaround"`
- `"<主题> 已知问题"`
- `"<主题> 降级"`
- `"<主题> 回滚"`

英文：
- `"<topic> bug"`
- `"<topic> conflict"`
- `"<topic> workaround"`
- `"<topic> known issue"`
- `"<topic> downgrade"`
- `"<topic> rollback"`

## 判断优先级

优先检查：

1. 是否已有多人复现
2. 是否有 maintainer 明确回复
3. 是否影响生产或核心路径
4. 是否存在现成 workaround
5. 是否已在新版本修复
6. 是否长期未修、且影响面持续扩大

## 输出倾向

面对难解 bug 或冲突时，优先给：

- 更可能的问题方向
- 最值得先试的 workaround
- 是否建议升级 / 降级 / 锁版本 / 回滚
- 是否建议绕开某特性或替换方案
- 何时停止继续排查，转为止损

不要默认建议用户继续无上限排查。
