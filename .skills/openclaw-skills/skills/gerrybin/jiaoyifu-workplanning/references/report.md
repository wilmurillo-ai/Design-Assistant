# 汇报复盘类方案模板（工作汇报/月度复盘/数据汇报）

> 适用：月度工作汇报、季度复盘、项目复盘、数据汇报、向上汇报材料

---

## 汇报复盘方案结构

### 第一节：核心结论

```
## 一、核心结论

### 1.1 本期总结
- 核心成果：[最重要的1-2件事，有数字]
- 目标达成：[核心指标当前值 vs 目标值，达成率]
- 一句话判断：[本期整体表现的定性结论]

### 1.2 亮点 vs 问题
**做成了**（3条以内，有数据）：
- [成果1]：[数字证明]
- [成果2]：[数字证明]

**没做成**（2条以内，有根因）：
- [问题1]：[数据差距 + 一句话根因]
- [问题2]：[数据差距 + 一句话根因]

### 1.3 下期重点（3条以内）
1. [优先级最高的事 + 目标数字]
2. [次优先级]
3. [次次优先级]
```

### 第二节：数据复盘

```
## 二、数据复盘

### 2.1 核心指标达成
[表格：指标 | 目标值 | 实际值 | 达成率 | 趋势↑↓]

### 2.2 关键路径执行情况
[表格：路径/项目 | 预期产出 | 实际产出 | 完成率 | 说明]

### 2.3 异常项说明
[达成率 <80% 或 >120% 的指标，每条单独分析]
- [指标]：目标[X]，实际[Y]，差距[Z]
  - 根因：[2-3句话，数据支撑，不是主观猜测]
  - 影响：[对下期/整体目标的影响]
```

### 第三节：根因分析

```
## 三、根因分析（仅针对P0问题）

### 3.1 问题定义
[用数据把问题锁死：不是"续费率低"，是"4月续费率4%，行业均值8%，差距4个点"]

### 3.2 根因拆解（5Why法）
Why 1：[直接原因]
Why 2：[为什么会有直接原因]
Why 3：[更深层原因]
Why 4：[系统性原因]
Why 5：[根本原因]

→ 根本原因：[一句话结论]

### 3.3 人物场归因
**人（执行层）**：[是人的问题吗？技能/意愿/资源]
**物（工具/产品层）**：[是工具的问题吗？]
**场（场景/机制层）**：[是机制的问题吗？]

→ 核心归因层：[人/物/场，选最关键的一层]
```

### 第四节：改进方案

```
## 四、改进方案

### 4.1 短期行动（下期内可完成）
[表格：问题 | 改进动作 | 负责人 | 截止时间 | 预期效果]

### 4.2 中期优化（1-3个月）
[表格：优化方向 | 具体措施 | 资源需求 | 预期指标改善]

### 4.3 不做什么（明确放弃的事）
[本期证明无效/性价比低的投入，下期不再做]
- [放弃项1]：[放弃原因/数据证明]
- [放弃项2]：[放弃原因]
```

### 第五节：下期计划

```
## 五、下期计划

### 5.1 目标设定
[表格：指标 | 当前值 | 下期目标 | 目标合理性说明]

### 5.2 关键路径
[3条以内，每条有明确的执行步骤和时间节点]

**路径一**：[名称]
- 目标：[数字]
- 关键动作：Step 1（第X天）/Step 2（第X天）
- 里程碑：[中期检查节点]

### 5.3 资源申请（如有）
[预算/人力/技术需求 + 申请理由 + 预期ROI]
```

## 汇报场景快速对应表

| 汇报类型 | 重点节 | 省略节 | 特殊要求 |
|---------|-------|-------|---------|
| 月度例会汇报 | 第一节+第二节 | 第三节（口头解释） | 控制在10分钟内 |
| 季度复盘 | 全五节 | 无 | 第三节必须有根因拆解 |
| 项目收尾复盘 | 第二节+第三节+第四节 | 第五节（变经验总结） | 重点在可复用的经验 |
| 向上汇报（老板） | 第一节 → 按需展开 | 按需 | 结论优先，细节按需 |
| 数据周报 | 第一节（简版）+第二节 | 第三-五节 | 数据驱动，图表优先 |

## 汇报写作通用原则

1. **结论优先**：老板先看结论，再看证据。不要从背景开始写
2. **数字锁死**：每一个「好」「差」「改善」背后必须有数字
3. **问题不隐瞒**：没做到的事明确说，说清楚根因，比粉饰太平更有价值
4. **下期可执行**：改进方案必须有人有时间有验收标准，不是愿望清单
5. **篇幅控制**：向上汇报 ≤ 2页 A4；内部复盘 ≤ 5页；周报 ≤ 1页
