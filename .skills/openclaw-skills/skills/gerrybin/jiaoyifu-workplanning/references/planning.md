# 规划类方案模板（月度/季度/年度）

> 适用：月度业务规划、季度OKR、年度战略规划、目标拆解方案

---

## 月度/季度规划结构

### 第一节：核心结论

```
## 一、核心结论

### 1.1 目标 vs 现状
- 目标GMV/指标：XX
- 现有路径预计：XX
- 缺口：XX（达成率XX%）

### 1.2 核心判断
[一句话：为什么当前路径不够，需要什么新路径]

### 1.3 解决方案概览
- X个增长方向，预计贡献XX
- 资源需求：预算XX / 技术XX人天 / 人力XX人

### 1.4 预计增长路径产出
[表格：路径 | 预计贡献 | ROI | 确定性]

### 1.5 关键风险
[1-3条，每条附应对方案]
```

### 第二节：现状诊断

```
## 二、现状诊断

### 2.1 上期数据复盘
[表格：指标 | 目标 | 实际 | 达成率 | 原因]

### 2.2 增长路径执行对比
[表格：路径 | 预期产出 | 实际产出 | 差距原因]

### 2.3 人物场分析
**人（团队）**：[岗位 | 人数 | 人效 | 主要职责]
**物（资源）**：[预算余额 / 技术能力 / 内容资产]
**场（场景）**：[主要用户场景 / 转化路径 / 激活方式]

### 2.4 核心问题识别
1. [问题1：数据证据 + 影响]
2. [问题2：数据证据 + 影响]
3. [问题3：数据证据 + 影响]
```

### 第三节：目标确定

```
## 三、目标确定

### 3.1 目标合理性分析
- 历史基线：[上期实际]
- 增长来源：[可量化的路径贡献]
- 不确定项：[新路径/新渠道的测试期折扣]
- 最终目标：[数字] = [确定项XX] + [不确定项XX]

### 3.2 目标拆解
[表格：产品线/路径 | 目标 | 占比 | 关键假设]

### 3.3 关键指标体系
[表格：指标 | 当前值 | 目标值 | 达标标准]
```

### 第四节：方法路径（增长路径设计）

```
## 四、增长路径设计

### 路径一：[路径名称]

**流程图**：
[用户池] → [分层标准] → [A组行动] → [B组行动] → [转化目标]

**数据测算**：
[表格：环节 | 数量/转化率 | 计算方式 | 产出]

**选择理由**：
1. 确定性：[为什么这条路径确定性高]
2. 边际成本：[成本结构]
3. 提升空间：[当前vs目标的绝对差值]

**执行方案**：
Step 1（第X天）：[具体动作]
Step 2（第X天）：[具体动作]
Step 3（持续）：[日常动作]

### 路径二～N：[同上结构]

### 路径汇总
[表格：路径 | 投入预算 | 预计产出 | ROI | 负责人]
```

### 第五节：执行方案

```
## 五、资源需求与里程碑

### 5.1 预算申请
[表格：项目 | 金额 | 用途 | 预期ROI]

### 5.2 技术需求
[表格：需求 | 工作量 | 优先级 | 业务价值]

### 5.3 里程碑
[表格：时间节点 | 里程碑 | 负责人 | 验收标准]
```
