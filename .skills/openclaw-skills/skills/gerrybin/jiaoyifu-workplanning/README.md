# jiaoyifu-workplanning

> 运营方案与工作规划 Claude Code Skill — 专为教育行业运营人设计

## 是什么

一个让 Claude Code 帮你写出**真正能落地的方案**的 Skill。

不是好看的方案，是能执行、能追踪、能交差的方案。

核心机制：**五节主结构 × 脚姨夫六维审核**

每写完一个模块，内置的「脚姨夫视角」会自动审核 6 个维度：
- R1 承诺可兑现（别承诺做不到的事）
- R2 执行不自欺（人力/时间/成本算过了吗）
- R3 受益方清晰（这是给用户的，还是给自己的）
- R4 逻辑一致（没有悄悄偏离整体目标）
- R5 数字可追（目标、方法、测算都有数字）
- R6 病因命中（方法对准病因，不是症状）

## 覆盖场景

| 场景 | 典型任务 |
|------|---------|
| 规划类 | 月度/季度/年度工作规划 |
| 活动类 | 活动方案、营销策划、ROI测算 |
| 服务体系类 | 学员服务体系、运营体系方案 |
| 汇报复盘类 | 工作汇报、月度复盘、数据报告 |

## 触发词

在 Claude Code 中说出以下关键词，Skill 自动激活：

`工作规划` / `运营方案` / `服务体系方案` / `活动方案` / `季度规划` / `年度规划` / `工作汇报` / `复盘报告`

## 设计背景

这个 Skill 来自一次真实的踩坑：

写一份5+模块的方案时，每个模块单独生产都没问题，但最后合并时发现——前面模块的内容已经从 Claude 的上下文里消失了，合并出来的是一份残缺版本。

解决方案：每个模块确认后立即写入 `.md` 文件，合并时通过读取文件而非依赖记忆——彻底绕开上下文溢出问题。

这个解法经过 Darwin 评分体系评估，最终得分 89/100。
