# Health Checkup Report v2.0

这不是“把体检单逐项复读一遍”的 skill。

它更像一个体检结果分层解读助手：把一堆箭头、指标和影像结论，整理成用户真正关心的几个判断：

- 先看哪几项
- 哪些只是轻度偏离，不用先吓自己
- 现在是观察、复查、挂门诊，还是尽快线下处理
- 去医院前还要补什么信息、问什么问题

## 核心定位

它解决的不是“这个名词是什么意思”，而是“这份体检报告我现在到底该怎么理解、先做什么”。

默认会帮用户完成这些事：
- 把异常项翻译成普通人能看懂的话
- 按优先级排序，而不是按表格顺序解释
- 把应该放在一起看的指标合并解读
- 区分轻度波动和真正值得处理的问题
- 给出下一步建议，而不是只停在术语解释

## 适合什么问题

- “这个体检有问题吗？”
- “报告好多箭头，到底先看哪几个？”
- “这几个指标偏高是什么意思？”
- “我要不要赶紧去医院？”
- “这个是先观察，还是该复查？”
- “超声写了结节/脂肪肝/囊肿，我该怎么理解？”

## 它会怎么输出

默认输出会收敛到这几个部分：

- `一句话结论`
- `最值得先关注的异常`
- `可以先观察的轻度偏离`
- `这些指标建议放在一起看`
- `建议下一步`
- `去医院前可以准备什么`

也就是说，它不只说“异常”，还会直接告诉用户：
- 重要程度怎么排
- 哪些不用和最严重的放在同一级别
- 下一步更像复查、门诊咨询，还是尽快线下处理

## 它和普通体检解读 skill 的区别

很多体检解读类 skill 的问题，是容易变成“按行解释器”：
- 逐项翻译
- 说了很多术语
- 但用户还是不知道该先处理什么

这个版本重点改成了“分层”和“行动”：
- 不再把每个小箭头都讲成同样严重
- 不再只逐项解释，而是会做指标组合判断
- 不再停在概念说明，而是给出下一步建议

## 它特别重要的安全边界

这不是诊断工具，也不应该装成医生结论。

| 能做 | 不能做 |
|------|--------|
| 解读体检和化验结果 | 直接下疾病诊断 |
| 标出优先级和风险层级 | 承诺“肯定没事” |
| 提示复查 / 门诊 / 尽快线下就医 | 给药物、剂量、停药建议 |
| 根据模式建议通常先挂哪个科 | 用单个指标断言严重疾病 |

如果用户同时有明显不适，或者报告里出现 `危急值`、`建议急诊`、`立即复查` 之类措辞，这个 skill 应该先提示及时线下就医，而不是继续长篇分析。

## 隐私提醒

这个 skill 很适合处理敏感健康信息，所以默认建议用户：
- 打码姓名、身份证号、电话、住址、就诊卡号
- 尽量提供异常项目、数值、单位、参考范围
- 图片不清楚时，优先手打异常项，避免误读

## 典型价值

用户真正想要的，通常不是“知道 ALT 是什么”。

他们想要的是：
- 这几个异常里先处理哪个
- 这次要不要紧张
- 哪些先观察就行
- 去医院前我该准备什么

这个 skill 的价值，就是把一张难读的报告，变成一个能行动的结论。

## 安装

```bash
clawhub install health-checkup-report
```

## 适合放在列表页的短介绍

把体检报告、化验单和影像结论翻译成清晰的中文判断，告诉你先看哪几项、急不急、哪些可以先观察，以及下一步该复查、挂门诊还是尽快线下就医。
