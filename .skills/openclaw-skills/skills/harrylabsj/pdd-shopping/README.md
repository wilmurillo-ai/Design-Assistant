# PDD Shopping Skill v2.1

拼多多不是“谁最便宜就买谁”。

这个 skill 的价值，是把拼多多里最容易让人犹豫的几件事一起判断清楚:
- 这个低价是不是要靠拼团、补贴、券门槛才能成立
- 这个商家能不能买
- 这个价格值不值得承担发货和售后风险
- 现在该买、该等、该换店，还是该换平台

它不是商品说明书，而是一个行动型购物决策助手。

## 适合什么场景

适合这些用户表达:
- “这个拼多多链接值不值得买？”
- “帮我看看百亿补贴是真的假的划算”
- “这个商品要不要拼团”
- “这张券是不是套路”
- “这个店售后行不行，退货麻不麻烦”
- “是现在买，还是换 JD / 天猫 更稳”

## 它会给出什么样的答案

默认不是泛泛分析，而是直接收敛到一个动作：
- 现在买
- 参加现有拼团
- 等更好的补贴或券
- 换店铺或换 SKU
- 改去 JD / 天猫
- 直接跳过这个 deal

典型输出会包含：
- `Recommended Move`
- `Price Reality`
- `Risk Check`
- `Why This Wins`
- `Next Step`

## 为什么用户愿意装

因为它解决的是“差一点就下单”的那一下犹豫。

它特别适合拼多多这种场景：
- 便宜是真的便宜，但条件很多
- 券、补贴、拼团一叠加就容易看花
- 商家差异大，售后体验差异也大
- 不是每个低价都值得冒险

这个 skill 会把“信息说明”升级成“买还是不买、怎么买更对”的建议。

## 核心判断框架

按这个顺序判断：
1. 真实到手价
2. 拼团值不值
3. 补贴是不是官方且稳定
4. 店铺风险大不大
5. 售后和退款扛不扛得住
6. 有没有更稳的替代平台

## 安全边界

| 操作 | Agent | 用户 |
|------|-------|------|
| 搜索/浏览 | ✅ | - |
| 看补贴/拼团/店铺信息 | ✅ | - |
| 加购/生成预览 | ✅ | - |
| 支付/最终提交订单 | ❌ | ✅ |

它会在支付前停下，把最终决定交还给用户。

## 使用示例

### 示例 1

用户：
“这台吹风机拼多多 299，百亿补贴，能买吗？”

Agent：
- `Recommended Move`: 可以买，但只建议买官方补贴页里的旗舰店版本
- `Price Reality`: 299 是有效价，券没有额外价值
- `Risk Check`: 普通店铺版本便宜 20 元，但售后弱，不值得
- `Next Step`: 只选带假一赔十和退货包运费的 SKU

### 示例 2

用户：
“这个零食拼团比单买便宜 6 块，要不要拼？”

Agent：
- `Recommended Move`: 不值得专门等拼团，直接单买
- `Why This Wins`: 价差小，成团和时效不确定，收益不够覆盖麻烦

## 一句话卖点

低价不是重点，重点是这单到底值不值得买。
