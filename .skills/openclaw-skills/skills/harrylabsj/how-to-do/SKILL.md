# How-To-Do: OpenClaw实用指南

你的OpenClaw使用教练 - 从零开始到高效精通。提供具体步骤、实用技巧、个性化推荐，解决"安装了但不会用"的普遍问题。

## 快速参考

| 情景 | 操作 |
|------|------|
| 刚安装OpenClaw | 运行初始化向导，学习基本命令 |
| 不知道装什么skill | 完成需求评估，获取个性化推荐 |
| 想做具体任务 | 获取该任务的逐步指导 |
| 遇到问题卡住 | 诊断问题，提供针对性解决方案 |
| 想提升使用水平 | 获取进阶工作流建议 |
| 定期检查进展 | 回顾进步，调整方向，设定新目标 |

## 核心理念

### 为什么需要这个skill
大多数用户安装OpenClaw后：
- 不知道从哪里开始
- 安装了错误的或太多的skill
- 只用了基础功能
- 遇到问题就放弃
- 错过了能帮助他们的高级功能

这个指南通过提供：
1. **具体步骤**：明确的、可操作的操作指南
2. **个性化推荐**：根据你的实际需求推荐skill
3. **问题解决**：遇到困难时的具体解决方案
4. **渐进式学习**：从基础到进阶的成长路径
5. **中文友好**：针对中文用户的使用习惯和场景

## 实用任务指南

### 常见任务的具体步骤

#### 任务：总结一篇文章
```
步骤1：获取内容
  - URL：使用web_fetch或browser工具
  - 文件：使用read工具读取
  - 文本：直接输入

步骤2：选择总结方式
  - 快速总结：使用summarize skill（自动）
  - 详细总结：手动指定要点
  - 结构化总结：使用模板（要点、引用、行动项）

步骤3：执行和优化
  - 运行总结
  - 检查质量
  - 根据需要调整参数
  - 保存到合适位置

步骤4：后续处理
  - 添加到笔记/知识库
  - 如果需要，分享给团队
  - 安排后续跟进
```

#### 任务：自动化每日天气查看
```
步骤1：设置
  - 安装weather skill
  - 配置地点
  - 设置喜欢的输出格式

步骤2：自动化
  - 创建定时检查（cron/提醒）
  - 设计输出格式（简洁/详细）
  - 选择发送方式（消息/通知/文件）

步骤3：集成
  - 连接到日历安排
  - 如果需要，连接服装建议
  - 如果有旅行，添加相关影响

步骤4：优化
  - 根据使用模式调整时间
  - 添加严重天气提醒（暴雨、极端温度）
  - 如需，扩展到多个地点
```

#### 任务：管理GitHub issues
```
步骤1：设置
  - 安装github skill
  - 配置认证
  - 设置默认仓库

步骤2：设计workflow
  - 每日issue查看流程
  - 分类和标签系统
  - 分配和跟踪
  - 进度监控

步骤3：自动化
  - 自动标签规则
  - 模板回复
  - 状态更新提醒
  - PR链接自动化

步骤4：报告
  - 每周issue总结
  - 团队表现指标
  - 瓶颈识别
  - 改进建议
```

## 问题诊断与解决

### 常见问题及解决方案

#### "OpenClaw启动不了"
```
1. 检查gateway状态：openclaw gateway status
2. 验证安装：openclaw --version
3. 查看日志：~/.openclaw/logs/
4. 常见问题：
   - 端口冲突：更改默认端口
   - 权限问题：检查文件权限
   - 依赖缺失：重新安装依赖
5. 寻求帮助：Discord社区、GitHub issues
```

#### "Skill不工作"
```
1. 验证安装：clawhub list
2. 查看文档：阅读SKILL.md
3. 测试基础功能：简单命令测试
4. 检查依赖：可能需要外部工具
5. 更新skill：clawhub update [skill名称]
6. 搜索已知问题：GitHub、社区论坛
```

#### "不知道用哪个skill"
```
1. 描述任务：你想做什么？
2. 搜索skill：clawhub search [关键词]
3. 查看推荐：基于你的评估结果
4. 尝试替代：相似的skill可能更好用
5. 从简单开始：先用基础skill，再加复杂度
```

#### "Skill太多，感觉混乱"
```
1. Skill审计：列出所有已安装，评分有用性
2. 优先级排序：保留最常用的5-7个
3. 卸载移除：删除不用或低价值的
4. 专注掌握：先精通核心skill，再添加新的
5. 计划添加：逐步添加新skill（每周1-2个）
```
