# Alibaba Shopping Skill v2.0

阿里巴巴生态总入口购物助手 Skill。

它不是单个平台助手，而是先帮用户判断：
- 淘宝、天猫、1688 该去哪一个
- 这次是应该追求正品和售后，还是追求品类和价格，还是追求批发价
- 平台选对之后，再把搜索、比价、看评论、加购推进到支付前

## 功能特性

- 🧭 **平台分流**：先判断淘宝 / 天猫 / 1688 哪个最适合
- 🔍 **商品搜索**：支持淘宝、天猫、1688 多平台搜索
- 📖 **用户评价**：读取评论、评分、晒单图片
- 💰 **价格对比**：多店铺/多平台比价
- 🛒 **购物车操作**：登录态下添加商品
- 🎫 **优惠券**：自动检查并应用可用优惠券
- 📋 **订单预览**：生成完整订单信息（不含支付）
- 🏪 **平台选择**：智能推荐最适合的购物平台

## 支持平台

| 平台 | 特点 | 适用场景 |
|------|------|----------|
| 天猫 | 品牌正品，高信任 | 买品牌、电子产品、重售后 |
| 淘宝 | 品类丰富，价格灵活 | 买特色商品、找同款、零售比价 |
| 1688 | 批发价，工厂直供 | 批量采购、开店货源、工厂拿货 |

## 安全边界

| 操作 | Agent | 用户 |
|------|-------|------|
| 搜索/浏览 | ✅ | - |
| 比价/看评论 | ✅ | - |
| 加购/用券 | ✅ | - |
| **支付/提交订单** | ❌ | ✅ |

## 工作流程

```
平台选择 → 搜索 → 对比 → 选择 → 加购 → 用券 → 预览 → [用户支付]
```

## 使用示例

```
用户：我想买一套咖啡杯，淘宝、天猫、1688 去哪更合适？
Agent：先判断平台 → 再搜索对比 → 最后推进加购和订单预览
用户：👤 完成支付
```
