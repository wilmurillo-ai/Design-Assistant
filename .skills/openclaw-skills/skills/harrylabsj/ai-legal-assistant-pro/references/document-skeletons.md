# 文书骨架

本文件用于生成第一轮结构化草稿。不要假装缺失事实已经明确；缺什么，就明确列出来。

## 1. 民事起诉状骨架

### 建议输出结构
1. 受理法院
2. 原告信息
3. 被告信息
4. 诉讼请求
5. 事实与理由
6. 证据目录
7. 落款

### 草稿模板
**致：** [人民法院]

**原告：** [姓名 / 性别 / 出生年月 / 地址 / 联系方式]

**被告：** [姓名 / 主体信息 / 地址 / 联系方式]

## 诉讼请求
1. [请求 1]
2. [请求 2]
3. [如需，可加诉讼费用承担请求]

## 事实与理由
- 双方关系
- 关键事实时间线
- 违约 / 争议焦点
- 损失或影响
- 法律依据方向（高层级）

## 证据目录
1. [合同 / 协议]
2. [付款记录]
3. [聊天记录 / 通知]
4. [其他支持材料]

**具状人：** [原告]

**日期：** [日期]

## 2. 答辩状骨架

### 建议输出结构
1. 案件识别信息
2. 核心答辩观点
3. 事实争议点
4. 程序 / 法律抗辩点
5. 证据目录

### 草稿模板
## 答辩状

**答辩人：** [姓名]

就 [原告] 提起的案件，答辩如下：

### 主要答辩观点
1. [答辩点 1]
2. [答辩点 2]
3. [答辩点 3]

### 事实与理由
- 已承认的事实
- 有争议的事实
- 为什么原告诉求应被驳回 / 调整 / 进一步澄清

### 证据
1. [证据项]
2. [证据项]

## 3. 证据提纲骨架

### 建议输出结构
| 序号 | 证据名称 | 证明目的 | 来源 | 弱点 / 备注 |
|---|---|---|---|---|
| 1 | 合同 | 证明双方关系 | 原件 / 签章件 | 需核对签字页 |

## 缺失事实清单

起草时，显式询问并标出以下缺失项：
- 双方完整身份信息
- 关键日期与时间线
- 争议金额
- 法院 / 管辖线索
- 已掌握的书面证据
- 是否发过催告 / 通知

## 提醒

这些模板仅用于第一轮准备，不应直接视为可提交法院的最终版本，正式使用前应进一步审核。
