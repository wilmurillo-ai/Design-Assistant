# 首批 Demo 场景稿

本文件用于对外演示 AI 法律助手专业版的首批 3 个最能卖场景。要求：
- 一眼能看懂价值
- 输出结构清楚
- 不靠复杂背景也能演示

---

## Demo 1：诉讼费估算

### 用户提问
起诉 10 万块，大概诉讼费是多少？

### 推荐演示输出
**初步结论：**
如果按常见中国民事财产案件标准估算，10 万元标的的案件受理费大约为 **2300 元**。

**计算依据：**
- 标的额：100,000 元
- 常见计算规则：100,000 × 2.5% - 200
- 估算结果：2300 元

**需要注意：**
- 上述金额通常只是案件受理费
- 不包括律师费、保全费、差旅费、鉴定费等其他成本
- 具体仍要以受理法院最新标准和案件类型为准

**下一步建议：**
如果你愿意，我可以继续帮你判断：
1. 这个案子值不值得起诉
2. 你还需要准备哪些证据
3. 起诉状框架怎么写

**边界提醒：**
当前结果仅供第一轮估算，不构成正式法律意见。

## Demo 2：服务合同付款条款风险扫描

### 用户提问
这条付款条款是不是太偏甲方了？

### 条款示例
乙方完成全部服务内容后，甲方有权根据内部审核结果决定是否验收及是否付款，付款时间不受本合同其他约定限制。

### 推荐演示输出
**初步结论：**
这条付款条款 **明显偏甲方，属于高风险条款**。

**主要风险点：**
1. **验收标准缺失**
   - 没有写清什么叫“完成”、什么叫“验收通过”
2. **付款完全受甲方单方控制**
   - 甲方可用“内部审核”无限拖延付款
3. **付款时间没有上限**
   - 乙方完成工作后，仍无法判断何时拿到钱

**建议修改方向：**
- 明确服务交付标准
- 明确验收期限（例如 5-7 个工作日）
- 增加“逾期未反馈视为验收通过”
- 明确付款时间点与逾期责任

**下一步建议：**
如果你愿意，我可以继续帮你把这条改写成更平衡的版本。

**边界提醒：**
当前扫描仅用于第一轮风险识别，正式签约前建议结合完整合同进一步审核。

## Demo 3：欠款起诉状骨架

### 用户提问
帮我起草一份民事起诉状框架，对方欠钱不还。

### 推荐演示输出
**文书目的：**
用于整理一份欠款纠纷的民事起诉状骨架，方便后续补全事实与证据。

## 民事起诉状（骨架）

**致：** [有管辖权的人民法院]

**原告：** [姓名 / 性别 / 出生年月 / 地址 / 联系方式]

**被告：** [姓名 / 地址 / 联系方式]

### 诉讼请求
1. 请求判令被告偿还欠款本金 [金额] 元；
2. 请求判令被告支付逾期利息 / 违约金（如有依据）；
3. 请求判令本案诉讼费用由被告承担。

### 事实与理由
- 原被告之间的借款 / 欠款关系
- 借款或欠款形成的时间、金额、方式
- 原告已履行付款义务 / 已形成债权
- 被告到期未还，催收后仍未履行

### 证据目录
1. 借条 / 欠条 / 协议
2. 转账记录 / 收款凭证
3. 聊天记录 / 催款记录
4. 其他可证明欠款事实的材料

**具状人：** [原告]

**日期：** [日期]

**仍需补充的关键事实：**
- 欠款形成原因
- 金额与时间
- 到期时间
- 是否已催收
- 手里现有证据清单

**边界提醒：**
这是一版结构化骨架，不是可直接提交法院的最终版本，正式使用前应结合事实进一步审核。
