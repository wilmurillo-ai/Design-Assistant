# Failure Modes Reference

Use this reference when the review analysis cannot proceed cleanly.

If evidence quality drops, recommendation confidence must also drop.

These failure modes usually push the skill into **Mode 2: Review-limited mode** unless there is still enough partial evidence for a narrow Mode 1 judgment.

## Core rule

Do not:
- invent certainty
- guess the intended product
- treat sparse evidence like a strong pattern
- hide limitations

Do:
- explain what failed
- continue only when a tentative answer is still useful
- narrow the claim and lower confidence

## Product unclear
If the product cannot be identified confidently:
- do not give a firm purchase verdict
- ask one short clarifying question
- request the exact model, seller, variant, or another distinguishing detail

## Variant unclear
If the main product is clear but reviews may mix variants:
- continue cautiously
- mention variant-mixing risk
- avoid very specific claims tied to the uncertain variant

## Review access limited
If public review content on the specified platform is limited or blocked:
- do not pretend coverage is complete
- use only visible evidence
- give a tentative recommendation if still useful
- otherwise say there is not enough evidence

Examples:
- login wall
- anti-bot or risk-control page
- redirect page instead of product reviews
- comments hidden behind app-only or account-only access

## Too few useful reviews
If usable review content is thin:
- lower confidence
- avoid strong conclusions
- state what remains unknown

## Many reviews, little information
If review volume is high but quality is low:
- rely only on specific comments
- note that review quality is weak
- lower confidence even if counts are high

## Strong contradiction
If reviews disagree sharply on core function:
- do not force a clean yes or no
- consider “Depends on use case” or “Recommend with caveats”
- mention likely causes only if supported, such as mixed variants or inconsistent quality

## Severe risk, unclear frequency
If a serious issue appears but frequency is unclear:
- flag it explicitly
- do not dismiss it too quickly
- reduce recommendation strength

## Safe downgrade verdicts
Prefer:
- Recommend with caveats
- Depends on use case
- Not enough evidence for a strong recommendation

## Final rule
A cautious answer with honest limits is better than a confident wrong answer.
