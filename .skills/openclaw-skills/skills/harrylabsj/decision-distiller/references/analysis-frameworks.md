# Decision Analysis Frameworks

## Framework 1: Weighted Decision Matrix

Use when comparing multiple options against multiple criteria.

### Steps

1. **List Options**
   - Identify all viable alternatives

2. **Define Criteria**
   - What matters for this decision?
   - Assign weights (1-5 or %)

3. **Score Each Option**
   - Rate each option per criterion (1-5)

4. **Calculate Weighted Scores**
   - Score × Weight for each cell
   - Sum for each option

5. **Compare Results**
   - Highest score indicates best fit

### Template

```markdown
## Decision Matrix

| Criteria | Weight | Option A | Option B | Option C |
|----------|--------|----------|----------|----------|
| Cost | 30% | 4 (1.2) | 3 (0.9) | 5 (1.5) |
| Quality | 25% | 5 (1.25) | 4 (1.0) | 3 (0.75) |
| Speed | 25% | 3 (0.75) | 5 (1.25) | 4 (1.0) |
| Risk | 20% | 4 (0.8) | 3 (0.6) | 2 (0.4) |
| **Total** | 100% | **4.0** | **3.75** | **3.65** |
```

## Framework 3: 10-10-10 Rule

Consider impact at three time horizons.

### Questions

- **10 minutes**: How will I feel about this decision in 10 minutes?
- **10 months**: How will I feel about this decision in 10 months?
- **10 years**: How will I feel about this decision in 10 years?

### Use When
- Personal or subjective decisions
- Emotional factors are significant
- Long-term consequences matter

## Framework 4: Pre-Mortem

Imagine the decision failed and work backwards.

### Steps

1. **Assume Failure**
   - "It's 6 months later and this decision was a disaster"

2. **Identify Causes**
   - What went wrong?
   - What did we miss?

3. **Preventive Actions**
   - How to avoid those causes?

### Use When
- High-stakes decisions
- Risk mitigation is critical
- Team over-optimism is a concern

## Framework 5: Cynefin Framework

Classify the decision context first.

### Domains

| Domain | Characteristics | Approach |
|--------|-----------------|----------|
| **Clear** | Known knowns, cause-effect clear | Best practice, SOP |
| **Complicated** | Knowable, requires analysis | Good practice, expert analysis |
| **Complex** | Unknown unknowns, emergent | Probe-sense-respond |
| **Chaotic** | No clear cause-effect, urgent | Act-sense-respond |
| **Confused** | Unclear which domain | Break down, clarify |

### Use When
- Uncertain how to approach a decision
- Need to match approach to context
- Team disagrees on methodology

## Framework 6: Opportunity Cost Analysis

Explicitly consider what you give up.

### Questions

1. What resources (time, money, attention) does this require?
2. What else could those resources be used for?
3. Is this the best use of those resources?
4. What is the value of the alternatives?

### Use When
- Resource constraints exist
- Multiple competing priorities
- Need to justify opportunity cost
