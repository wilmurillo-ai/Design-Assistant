# Deals and Group-buy

Use this file when the user asks whether a Dianping group-buy, set meal, coupon, voucher, or package is worth buying.

## Deal judgment checklist

Check these before saying a deal is good:
1. How many people is it really for?
2. Does it include the items the user would actually order?
3. Is it available on weekends / holidays / prime dinner hours?
4. Does it require advance booking?
5. Is it dine-in only?
6. Are there extra fees, tea charges, service charges, or mandatory upgrades?
7. How does it compare to normal ordering?

## Three outcome labels

### 划算
Use when:
- clear real discount exists
- restrictions are acceptable
- package contents match the user’s needs

### 一般
Use when:
- discount exists but is offset by awkward restrictions
- package is okay but not obviously better than ordering normally

### 不建议冲
Use when:
- restrictions are too strong
- store has repeated complaints about group-buy discrimination
- package forces irrelevant items or weak portions

## Common traps

- “cheap” but only valid at dead hours
- one package says 2–3 people but portions only fit 1–2
- weekends excluded
- need to call and reserve but booking is hard
- group-buy users seated worse or served slower
- key dishes excluded, forcing add-on spend

## What to compare

When possible, compare:
- package price vs estimated single-item total
- convenience cost: reservation, timing, transport, waiting
- experience loss: reduced choice, weaker seating, lower service priority

## Good output pattern

> 结论：这个套餐算一般，不是不能买，但不值得专门冲。  
> 原因：账面上便宜，但周末晚餐不可用，而且核心菜不在套餐里，实际还要加点。  
> 风险：如果你是两个人周末去，体验大概率不如直接单点。
