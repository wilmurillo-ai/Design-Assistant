# Feed Workflow

Use this file while sourcing posts from the feed.

## Discovery rules

1. Open the openclaw managed chrome.
2. Navigate to `x.com` and open the home feed.
3. CRITICAL NON-NEGOTIABLE: select the **For You** tab. Never use the Following tab or any other tab during this workflow.
4. CRITICAL NON-NEGOTIABLE: once you are on the feed, do not refresh or reload the page for any reason for the entire run, even between comments. Navigate back to this same feed state after each post using the back button only.
5. Scroll before acting; do an initial pass across multiple screenfuls so you are not choosing only from the first visible posts.
4. Never use search for this workflow unless the user explicitly overrides that rule.
5. Open only posts that are relevant to the requested topic set.
7. Prefer posts with a clear claim, launch, opinion, demo, or lesson.
8. Open posts only by clicking from the feed; do not visit post URLs directly.
9. Back out after reviewing the post using the back button, land back on the same feed state, and continue sourcing the next candidate.

## Selection bar

Pick posts that are:

- Fresh
- Specific
- Worth adding to
- Aligned with the user's requested topic focus

Skip posts that are:

- Too vague to comment on intelligently
- Pure engagement bait
- Off-topic
- Duplicate takes on the same thing unless the angle is genuinely different
- Consecutive near-identical takes in a row when a different relevant topic is available

## Cadence

CRITICAL NON-NEGOTIABLE: do not pick all `N` posts first and then comment in a batch.
CRITICAL NON-NEGOTIABLE: use strict sequence for every iteration: click post in feed -> read -> like -> comment -> back button -> same feed state -> scroll -> click next post.
CRITICAL NON-NEGOTIABLE: do not keep multiple candidate posts open and hop between them.
CRITICAL NON-NEGOTIABLE: never refresh or reload the feed at any point during the run, including between comment iterations. The only valid way to return to the feed is the back button.
CRITICAL NON-NEGOTIABLE: after all `N` comments are done, close the `x.com` tab immediately. Do not leave it open.
