# Chinese Discovery Prompts

Use these only during requirement discovery. Keep the exchange short.

## Core Questions

1. "这个 Skill 最主要要帮你解决什么问题？"
2. "用户通常会怎么开口，才应该触发这个 Skill？"
3. "它要接收什么输入，产出什么结果？"
4. "这是新建 Skill，还是改已有 Skill？"

## Follow-Ups

- "如果这个 Skill 做错，最容易错在哪里？"
- "你更在意稳定流程，还是灵活发挥？"
- "有没有现成脚本、模板或文档要一起放进去？"

## Confirmation Prompt

"我理解的是：这个 Skill 用来 [目标]，典型触发语是 [触发]，输入是 [输入]，输出是 [输出]。我会按 [模式] 来设计，对吗？"

## Rules

- Prefer one question at a time.
- Use plain Chinese, not framework jargon.
- Stop asking once the build path is clear.
- If the user is already specific, summarize and move on.
