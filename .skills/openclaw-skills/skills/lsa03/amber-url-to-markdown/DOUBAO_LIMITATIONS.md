# 豆包链接抓取限制说明

## 问题描述

豆包（https://www.doubao.com）的内容完全依赖 JavaScript 动态渲染，具有以下特点：

1. **动态内容加载** - 内容通过 API 异步加载，不在初始 HTML 中
2. **反自动化检测** - 检测 Playwright、Scrapling 等自动化工具
3. **会话依赖** - 内容可能需要登录或特定会话状态

## 测试结果

| 方案 | 状态 | 说明 |
|------|------|------|
| Playwright 无头浏览器 | ❌ 失败 | 检测到自动化，只返回框架 HTML |
| Scrapling 静态抓取 | ❌ 失败 | 无 JavaScript 执行，无法获取动态内容 |
| 独立测试脚本 | ⚠️ 部分成功 | 有时能获取 4000+ 字符，但不稳定 |
| 手动复制 | ✅ 推荐 | 最可靠的方式 |

## 建议方案

### 方案 1：手动复制（推荐）
直接复制豆包对话内容，粘贴到文档中。

### 方案 2：使用分享功能
检查豆包是否有公开分享/导出功能。

### 方案 3：截图 + OCR
对于短内容，可以截图后使用 OCR 工具提取文字。

## 技术细节

### 为什么独立测试能成功但脚本失败？

独立测试中有时能获取内容（4224 字符），但脚本中始终失败，原因可能是：

1. **执行环境差异** - 脚本执行速度更快，触发了更严格的反爬
2. **会话状态** - 独立测试可能有缓存的会话
3. **时间窗口** - 豆包的反爬策略可能动态变化

### 检测点

豆包可能通过以下方式检测自动化：
- `navigator.webdriver` 属性
- 浏览器指纹
- 行为模式（滚动速度、鼠标移动轨迹）
- 请求频率和时序
