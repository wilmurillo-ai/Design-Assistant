# Amber Url to Markdown - 自动触发优化方案总结

## 问题回顾

**原始问题：**
- 技能发布到 ClawHub 后，用户安装无法自动触发
- SKILL.md 中描述的"自动触发"只是文档约定，不是真正的自动触发机制
- 需要 AI 主动检测 URL 并调用脚本，但 AI 可能不会每次都正确识别

**根本原因：**
- OpenClaw 的技能系统没有内置的"消息监听 + 自动触发"机制
- 技能的触发依赖于 AI 的判断，而不是系统级别的事件监听

## 解决方案

使用 OpenClaw 的 **Hooks 系统** 实现真正的自动触发。

### Hook 系统简介

OpenClaw Hooks 是一个事件驱动的自动化系统，可以监听各种事件（包括 `message:received`），并在事件触发时自动执行自定义脚本。

**核心特点：**
- ✅ 系统级别的事件监听
- ✅ 自动触发，不依赖 AI 判断
- ✅ 异步执行，不阻塞消息处理
- ✅ 可配置，可启用/禁用

### 实现方案

创建了 `url-auto-fetch` Hook，实现以下功能：

1. **监听 `message:received` 事件**
   - 当用户发送消息时自动触发

2. **检测 URL 链接**
   - 使用正则表达式匹配消息中的 URL

3. **判断触发条件**
   - 纯 URL 消息（消息中只有 URL）
   - URL + 意图关键词（解析、转换、markdown 等）

4. **执行脚本**
   - 异步调用 `amber_url_to_markdown.py`
   - 不阻塞消息处理

5. **保存结果**
   - Markdown 文件保存到 `/root/openclaw/urltomarkdown/`
   - 图片保存到 `images/knowledge_YYYYMMDD_HHMMSS/`

## 技术实现

### handler.ts 核心逻辑

```typescript
const handler = async (event: any) => {
  // 只处理 message:received 事件
  if (event.type !== "message" || event.action !== "received") {
    return;
  }

  const content = event.context?.content || "";
  
  // URL 匹配正则
  const urlPattern = /https?:\/\/[^\s<>"{}|\\^`\[\]]+/gi;
  
  // 意图关键词
  const intentKeywords = ["解析", "转换", "转成", "markdown", "md", ...];
  
  // 检测 URL
  const urls = content.match(urlPattern);
  if (!urls || urls.length === 0) {
    return;
  }
  
  // 判断触发条件
  const isPureUrl = content.trim().replace(urlPattern, "").trim().length === 0;
  const hasIntentKeyword = intentKeywords.some(k => 
    content.toLowerCase().includes(k.toLowerCase())
  );
  
  if (!isPureUrl && !hasIntentKeyword) {
    return;
  }
  
  // 执行脚本（异步）
  const { exec } = require("child_process");
  exec(`python3 "${scriptPath}" "${targetUrl}"`, ...);
};
```

## 注意事项

### 1. Hook 需要手动启用

出于安全考虑，Hook 默认不会自动启用，需要用户手动执行：

```bash
openclaw hooks enable url-auto-fetch
```

### 2. 异步执行

Hook 使用异步方式执行脚本，这意味着：

- ✅ 不会延迟 AI 回复
- ⚠️ 无法立即返回抓取结果
- ✅ 适合后台处理

### 3. 结果通知

目前 Hook 不会主动发送完成通知（避免骚扰），用户可以在以下位置查看结果：

- `/root/openclaw/urltomarkdown/` 目录

未来可以添加配置选项，让用户选择是否接收完成通知。

### 4. 安全考虑

- 只处理用户发送的消息
- 跳过机器人自己的消息
- URL 经过转义后传递给脚本
- 脚本执行超时设置为 2 分钟
