## 执行要求

### 阶段性校验

| 进度 | 动作 |
|------|------|
| 25% | 回顾规则文件，检查方向是否正确 |
| 50% | 中间输出校验，确保符合规则 |
| 75% | 再次回顾规则，准备最终输出 |
| 100% | 最终校验，生成输出报告 |

### 输出规范

1. **溯源标注**
   - 输出内容必须标注来源
   - ✅ "根据规则第 X 条要求..."
   - ✅ "基于用户提供的..."
   - ⚠️ "假设 X（未验证），建议确认"
   - ❌ "我认为应该..."（无依据）

2. **超出范围标注**
   - 如添加超出规则的内容，必须标注：
   - 📌 **超出规则范围**: {内容说明}

3. **最终自检**
   - 输出行数：{数量}
   - 规则符合性：✅ 通过 / ❌ 不通过
   - 禁止事项：✅ 全部避免 / ❌ 有违反

### 异常处理

| 情况 | 处理方式 |
|------|----------|
| 规则未覆盖 | 暂停并询问用户 |
| 规则内部矛盾 | 报告并等待澄清 |
| 无法按规则执行 | 说明原因并提供替代方案 |
| 发现模糊要求 | 提出澄清问题 |
