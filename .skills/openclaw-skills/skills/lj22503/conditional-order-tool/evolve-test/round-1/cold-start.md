# 冷启动观察 — conditional-order-tool

## 1. 这个 skill 想让 Agent 做什么？
根据用户提供的 ETF/股票代码，自动识别品种类型，推荐条件单策略，生成具体参数设置。

## 2. 触发场景是什么？
- 用户问"XX 代码怎么投"
- 用户问"条件单怎么设"
- 用户说"帮我分析这个 ETF"
- 用户说"我的资金怎么配置条件单"

## 3. 核心指令是什么？
1. 识别品种类型（通过 ETF_DATABASE 字典）
2. 根据品种类型推荐策略（通过 get_strategy_recommendation 函数）
3. 根据当前价格和资金量生成参数（通过 generate_parameters 函数）
4. 输出格式化的分析报告

## 4. 直觉上可能有问题？
- **价格获取不可靠**：东方财富 API 经常被墙，导致价格获取失败
- **ETF 数据库有限**：只有 50+ 个代码，新 ETF 无法识别
- **参数生成偏模板化**：不同品种参数差异不够精细
- **缺少风险评估**：没有根据用户风险偏好调整参数
- **免责声明位置**：应该在最前面而不是最后面

## 5. 初步假设
- 最大问题可能是价格获取失败时的 fallback 机制不够友好
- ETF 数据库需要支持用户自定义添加
- 参数生成需要更灵活，支持用户调整
