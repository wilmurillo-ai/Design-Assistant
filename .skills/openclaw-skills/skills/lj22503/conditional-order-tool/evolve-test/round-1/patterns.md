# 错误模式表 — Round 1

## P01: 价格获取失败导致参数无法生成
- 出现次数：5 次（所有 prompt）
- 表现：用户看到"无法获取实时价格"，但没有具体参数（网格区间、格距、止损止盈价）
- 根因：`fetch_east_money_price()` 函数依赖东方财富 API，该 API 从服务器访问被限制
- 影响面：高（核心功能失效）

## P02: 资金量不影响参数输出
- 出现次数：2 次（prompt 1 和 prompt 4，5 万 vs 50 万）
- 表现：5 万和 50 万资金的输出完全一样，没有根据资金量调整仓位和参数
- 根因：`generate_parameters()` 函数中，参数计算没有根据资金量做差异化处理
- 影响面：中（用户体验差，但策略逻辑正确）

## P03: 错误输入映射为股票不够准确
- 出现次数：1 次（prompt 5）
- 表现：999999 被识别为"未知品种"并归类为"股票"，但实际可能是用户输错了代码
- 根因：`identify_code()` 函数中，未知代码默认归类为股票
- 影响面：低（边缘场景，但不影响核心功能）

## 本轮改进优先级
1. P01 — 价格获取失败是最大问题，需要解决或提供友好 fallback
2. P02 — 资金量差异化是核心体验问题
3. P03 — 错误处理优化，次要
