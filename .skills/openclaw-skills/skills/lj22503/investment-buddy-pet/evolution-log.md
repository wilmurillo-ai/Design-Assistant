## 改进原则（压缩自 Round 1-2）

1. **SKILL.md 是执行指南，不是产品文档**
   - 告诉 Agent"当用户说 X 时，你应该说 Y"
   - 提供示例对话，不只是功能描述

2. **三层架构 + 能力检测**
   - 对话层：任何 Agent 都能执行
   - 执行层：有执行能力的 Agent
   - 数据层：有文件系统的 Agent
   - 执行前先看"Agent 能力检测"

3. **渐进式披露**
   - 核心流程在 SKILL.md（<400 行）
   - 详细配置在 references/

4. **脚本调用标准化**
   - 明确输入参数
   - 明确输出格式（JSON/Markdown）
   - 提供示例输出
