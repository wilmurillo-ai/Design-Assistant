# DAF Intent Guard 🎯

**自动检测多步任务中的意图漂移，决定是继续、回退还是放弃。**

---

## 它解决什么问题

你让Agent帮你订机票，已经锁好座位了，然后你说"改去纽约"。

大多数Agent会直接重来，或者继续用错误的参数执行。

DAF Intent Guard会先测量这次改动有多大，然后告诉Agent：
- **patch**：小改动，直接更新参数继续
- **replan**：改动较大，回退到受影响的步骤重新规划
- **abort**：方向完全变了，停下来确认

---

## 安装

```bash
clawhub install daf-intent-guard
```

## 工作原理

DAF用两个维度衡量改动大小：

**锚点轴**：任务执行到哪一步了？改动会影响哪些已完成的步骤？

**语义轴**：约束值本身变化多大？
- 今天→明天：距离 0.10（小）
- 新加坡→纽约：距离 0.80（大）
- buy→cancel：直接abort

两个维度合并后给出最终决策。

## 使用场景

- 订票/订酒店中途改目的地或日期
- 写邮件/文档中途改要求
- 任何多步骤任务中用户反悔或修改

## 技术细节

- 纯Python，零外部依赖
- 支持时间、地点、分类、数量四种槽位类型的语义距离计算
- 支持action类型检测（buy→cancel自动abort）
- 累积漂移追踪，防止多次小改动积累成大问题
