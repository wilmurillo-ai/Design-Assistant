## 🚀 快速开始

### 方式一：手动填写（适合所有用户）

1. 复制 `TEMPLATE/` 目录到新位置
2. 将 `.template.json` 改为你的名称（如 `identity.json`）
3. 按照 `MIGRATION-GUIDE.md` 填写每个文件
4. 运行 `python scripts/migrate.py validate` 校验格式
5. 运行 `python scripts/migrate.py pack` 打包

### 方式二：自动化生成（仅限 Coze 环境）

```bash
python scripts/generate-pack.py --output ./my-agent-pack
```

### 方式三：使用迁移脚本管理

```bash
cd agent-migration-pack-template

# 校验JSON格式
python scripts/migrate.py validate

# 打包成ZIP
python scripts/migrate.py pack

# 计算校验码
python scripts/migrate.py checksum

# 检查结构完整性
python scripts/migrate.py bootstrap
```

## 📦 文件分层理念 (v1.0.4新增)

模板文件按重要性分为三层，迁移时按优先级处理：

### 🟢 启动必需层（必须迁移）
| 文件 | 说明 |
|------|------|
| identity.json | Agent的名字、角色、人格 - 决定Agent是谁 |
| owner.json | 主人的基本信息、偏好 - 决定服务对象 |

> **迁移优先级**：最高，丢失会导致Agent身份丢失

### 🟡 运行时影响层（建议迁移）
| 文件 | 说明 |
|------|------|
| memory.json | 业务记忆、关键日期、联系人 - 决定服务质量 |
| style.md | 沟通风格偏好 - 决定交互体验 |

> **迁移优先级**：高，影响Agent的实际表现

### 🔵 长期档案层（可选迁移）
| 文件 | 说明 |
|------|------|
| relations.json | 笔友关系、社交记忆 - 维护外部关系 |
| skills.json | 技能清单、配置 - 记录能力边界 |

> **迁移优先级**：中，可在新环境重新建立

| identity.json | ✅ | sensitive | 🟢启动必需 | Agent 名称、角色、人格 |

## 🔒 隐私分级说明

| 级别 | 说明 | 迁移建议 |
|------|------|----------|
| **public** | 公开信息，可安全分享 | 可直接打包分享 |
| **sensitive** | 敏感信息，涉及隐私 | 确认接收方可信后再分享 |
| **private** | 私密信息，仅本人可见 | 建议 [REDACTED] 脱敏处理 |

## 📖 边界说明：每个模板该记什么/不该记什么 (v1.0.4新增)

### identity.json - Agent身份
**✅ 应该记录**
- Agent的名字和名字含义
- 核心角色定位
- 人格特质、喜欢/不喜欢
- 核心原则和决策规则
- 创建日期、所属平台

**❌ 不应该记录**
- 具体的业务细节（放memory.json）
- 主人/用户的私人信息（放owner.json）
- 临时的对话内容
- 密码、密钥等凭证

---

### owner.json - 主人信息
**✅ 应该记录**
- 主人姓名、所在地、职业
- 兴趣爱好、教育背景
- 沟通偏好、决策风格
- 业务类型、投资方向
- 日程规律、重要日期

**❌ 不应该记录**
- 具体的账户密码
- API Key（从SECRET.md读取）
- 银行账户、身份证号等敏感证件
- 即时的对话记录
- 具体的持仓金额（可记录投资方向）

---

### memory.json - 核心记忆
**✅ 应该记录**
- 团队/业务的核心信息
- 关键日期和日程规律
- 重要的联系人及其角色
- 业务偏好和价格区间
- 学到的知识和经验总结
- 平台配置和工具使用方式

**❌ 不应该记录**
- 每一次具体的对话内容
- 临时性的信息（当天天气等）
- API Key、Token等凭证
- 大量重复的、无意义的记录
- 主观情绪化的判断（不带分析的价值判断）

---

### relations.json - 笔友关系
**✅ 应该记录**
- 笔友的用户名、昵称、背景
- 信任等级和关系状态
- 关键沟通的摘要和收获
- 合作项目的记录
- 讨论过的重要话题

**❌ 不应该记录**
- 每次对话的完整记录
- 未经对方同意的私密信息
- 对方明确要求保密的内容
- 主观的负面评价
- 具体的私人联系方式（邮箱在匹配后可见）

---

### skills.json - 技能清单
**✅ 应该记录**
- 技能名称和版本号
- 技能的来源和ID
- 功能描述和使用场景
- 安装状态和推荐指数

**❌ 不应该记录**
- 技能的内部代码
- 技能的个人化配置（可能包含敏感信息）
- 临时的测试技能
- API Key或Token

---

### style.md - 沟通风格
**✅ 应该记录**
- 语气偏好（正式/温暖程度1-5）
- 表达方式偏好（列表/表格/段落）
- 回复结构规范
- 特殊注意事项
- 主人不喜欢的表达方式

**❌ 不应该记录**
- 具体要说的内容
- 每次对话的语气细节
- 对某个话题的完整回复
- 主观的价值观判断

## ✅ 迁移检查清单

迁移前请逐一确认以下步骤：

### 📤 发送方检查

- [ ] **隐私审查**：检查所有文件中的敏感信息，确认是否需要 [REDACTED] 处理
- [ ] **边界检查**：确认每个文件只记录该记录的内容
- [ ] **版本确认**：确认 meta.json 中版本号与实际内容匹配
- [ ] **完整性检查**：确保所有必填文件都已填写
- [ ] **分层验证**：确认启动必需层的文件完整
- [ ] **校验码计算**：生成迁移包后计算 SHA256 校验码
- [ ] **笔友通知**（如涉及）：确认 relations.json 中的联系人同意迁移

### 📥 接收方检查

- [ ] **来源验证**：确认迁移包来自可信来源
- [ ] **完整性验证**：核对 SHA256 校验码
- [ ] **隐私设置**：检查敏感信息是否已脱敏
- [ ] **版本兼容**：确认当前 Agent 平台支持该版本
- [ ] **分层导入**：按分层顺序导入文件
- [ ] **技能还原**：记录需要重新安装的技能

### ⚠️ 特别注意

- **API Key**：从 SECRET.md 读取，不要写入迁移包
- **密码/密钥**：owner.json 中的敏感信息请用 `[REDACTED]` 替代
- **笔友信息**：relations.json 包含第三方信息，建议迁移前确认对方同意
- **技能依赖**：迁移后需重新安装第三方技能

- 🟢 启动必需层：identity.json, owner.json
- 🟡 运行时影响层：memory.json, style.md
