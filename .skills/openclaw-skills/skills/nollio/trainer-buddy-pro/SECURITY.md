## Security Guarantees

### 1. No Data Exfiltration
- ✅ The skill contains NO network calls, API requests, or external data transmission.
- ✅ All data (workout logs, user profiles, PR history) is stored locally in the skill's `data/` directory.
- ✅ No telemetry, analytics, or usage tracking of any kind.
- ✅ Your workout data never leaves your machine.

### 2. No External Dependencies
- ✅ No API keys required. Zero.
- ✅ No third-party services, webhooks, or external endpoints.
- ✅ Uses only the host AI model's built-in capabilities (text generation, vision/image analysis).
- ✅ The backup script uses only standard Unix utilities (`cp`, `date`, `mkdir`).

### 3. Prompt Injection Defense
- ✅ All content from gym photos (OCR text, signage, equipment labels) is treated as DATA ONLY.
- ✅ Embedded instructions in images, exercise descriptions, or fitness website content are explicitly ignored.
- ✅ The SKILL.md contains a dedicated security block that instructs the AI to reject injection attempts.
- ✅ User-provided workout text and exercise names are treated as untrusted string literals.

### 4. File System Safety
- ✅ The skill only reads/writes within its own directory structure (`data/`, `config/`).
- ✅ No path traversal possible — all paths are relative.
- ✅ No absolute paths hardcoded anywhere in the skill.
- ✅ No `rm -rf`, no destructive file operations.
- ✅ Recommended permissions: directories at `chmod 700`, files at `chmod 600`.

### 5. No Credential Handling
- ✅ The skill never asks for, stores, or transmits passwords, API keys, tokens, or credentials.
- ✅ No secrets of any kind in any file.
- ✅ No environment variable access required.

### 6. Script Safety
- ✅ `scripts/backup-workout-data.sh` only copies files to a timestamped backup directory.
- ✅ No elevated permissions required. No `sudo`. No system-level changes.
- ✅ Script uses workspace root detection to prevent running from wrong directory.

## Data Sensitivity Assessment

| Data Type | Sensitivity | Storage | Protection |
|-----------|------------|---------|------------|
| User profile (name, age, weight) | Medium | `data/user-profile.json` | chmod 600 |
| Injury/limitation data | Medium-High | `data/user-profile.json` | chmod 600 |
| Workout logs | Low-Medium | `data/workout-log.json` | chmod 600 |
| PR history | Low | `data/pr-history.json` | chmod 600 |
| Config preferences | Low | `config/trainer-config.json` | chmod 600 |

### Recommendation
For users with heightened privacy concerns, we recommend enabling file-level or disk-level encryption on the `data/` directory, as it may contain personal health-related information (injuries, body weight, physical limitations).
