# Writing Coach Pro — Dashboard Integration Spec

## Overview

Writing Coach Pro integrates with the NormieClaw Dashboard to display writing analysis trends, style profile evolution, and improvement tracking over time.

## Requirements

- NormieClaw Dashboard v1.0+
- Dashboard must support the standard manifest format (see `manifest.json`)
- Tables auto-created on first data sync if using Supabase backend

## Route

**Path:** `/writing-coach`
**Nav label:** Writing Coach
**Icon:** `pencil-line` (Lucide icon set)

## Data Flow

1. After each Writing Coach Pro review session, the agent writes a summary row to `wc_sessions`
2. Individual findings are written to `wc_issues` with a foreign key to the session
3. Any style profile changes are logged to `wc_profile_history`
4. Dashboard reads these tables to render widgets

## Page Layout

### Top Row: Key Stats
Four stat cards in a horizontal row:
- **Total Sessions**: Count of all review sessions
- **Words Reviewed**: Sum of all word counts
- **Current Grade Level**: Readability grade from the most recent session
- **Acceptance Rate**: Overall percentage of accepted suggestions (measures agent learning quality)

### Chart Row 1: Readability Trend
Line chart showing Flesch-Kincaid grade level over time. One data point per day (averaged if multiple sessions). Helps users see if their writing complexity is converging on their target.

### Chart Row 2: Side by Side
- **Left — Issues by Category**: Bar chart breaking down grammar vs. style vs. tone vs. consistency issues. Shows where the user's writing needs the most work.
- **Right — Acceptance Rate Over Time**: Line chart tracking what percentage of suggestions the user accepts. A rising line means the agent is learning the user's preferences. A flat or falling line means the profile needs tuning.

### Table: Recent Sessions
Sortable table showing the last 20 sessions with columns:
- Date
- Mode (Quick/Full/Rewrite/Coach)
- Word Count
- Issues Found
- Issues Accepted
- Readability Grade
- Context Type

Click a row to drill into the session detail (if session logs are stored).

## Table Schemas

See `manifest.json` for complete column definitions. Key relationships:

- `wc_issues.session_id` → `wc_sessions.id`
- All tables use text primary keys (UUIDs generated by the agent)
- Timestamps are `timestamptz` for timezone-safe queries

## Data Retention

No automatic purging. Users can clear their data by:
- Deleting `~/.openclaw/skills/writing-coach-pro/data/` (local logs)
- Running `DELETE FROM wc_sessions` / `wc_issues` / `wc_profile_history` (dashboard tables)
