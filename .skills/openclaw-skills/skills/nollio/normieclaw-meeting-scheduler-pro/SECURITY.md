# Meeting Scheduler Pro — Security Considerations

## Overview

Meeting Scheduler Pro accesses your Google Calendar, email (for prep context), and stores meeting notes locally. This document outlines what data flows where and what you should be aware of.

## Data Flow

### Google Calendar Access
- The `gog` CLI authenticates via Google OAuth 2.0
- Calendar reads and writes go directly to Google's API
- OAuth tokens are stored by the `gog` CLI in its default credential location
- Meeting Scheduler Pro does not store a copy of your calendar — it queries live data on each request

### Meeting Prep — Web Search
- When `prep.include_web_search` is enabled, the agent searches the web for attendee and company context
- Search queries include attendee names and company names derived from your calendar events
- Search results are processed by your configured AI model to generate the prep brief
- If you're uncomfortable with attendee names being used in web searches, disable this in settings.json

### Meeting Prep — Email Context
- When `prep.include_email_context` is enabled, the agent queries Gmail via `gog` for recent threads with meeting attendees
- Email content is processed by your configured AI model to extract relevant context
- No email content is stored separately — it's read on demand and included in the prep brief

### Meeting Notes & Follow-Ups
- Follow-up notes, action items, and agendas are stored as local markdown files in the `meeting-notes/` directory (configurable)
- These files contain meeting summaries, attendee names, discussion points, and action items
- Files persist until you delete them — they feed future meeting prep

### Dashboard Metrics
- Aggregated metrics (meeting counts, prep rates, follow-up stats) are tracked for dashboard display
- Metrics contain no PII — only counts, rates, and time-based aggregations

## AI Model Processing

Meeting prep briefs, agendas, and follow-up summaries are generated by your configured AI model (via OpenClaw). The model processes:
- Calendar event details (title, description, attendees)
- Email snippets (if enabled)
- Web search results (if enabled)
- Past meeting notes (from local files)

Understand your AI provider's data handling policies. Content processed by the model is subject to that provider's terms.

## Authentication & Credentials

| Credential | Managed By | Storage |
|------------|-----------|---------|
| Google OAuth token | `gog` CLI | gog's default credential store |
| OpenClaw API keys | OpenClaw | OpenClaw's secure config |
| No additional credentials | — | Meeting Scheduler Pro adds no new auth |

Meeting Scheduler Pro does not introduce any new API keys or credentials. It relies entirely on `gog` (for Google services) and OpenClaw (for AI processing).

## Recommendations

1. **Review gog permissions:** Ensure you've granted only the scopes you're comfortable with (Calendar read/write, Gmail read)
2. **Audit meeting notes regularly:** The `meeting-notes/` directory accumulates context over time. Review and prune as needed.
3. **Disable email context if sensitive:** If your email contains highly sensitive information, set `prep.include_email_context: false` in settings.json
4. **Disable web search if private:** If you don't want attendee names used in web searches, set `prep.include_web_search: false`
5. **Understand your AI provider:** Meeting content is processed by your configured model. Review their data retention and training policies.

## What This Tool Does NOT Do

- Does not create new cloud accounts or services
- Does not store credentials beyond what `gog` and OpenClaw already manage
- Does not share your calendar with third parties
- Does not send emails without your explicit confirmation
- Does not record or transcribe meetings (that's a separate tool's job)
