# GitHub 入门引导

## 适用场景
- 你要把本地项目放到 GitHub。
- 你后续要用 Vercel 自动部署网站。
- 你几乎没有代码经验。

## 前置准备
- 一个邮箱（可收验证码）。
- 已安装并打开浏览器。
- 本地已有项目文件夹。
- 已安装 Git（Git 是“上传版本记录工具”，可理解为项目传输工具）。

## 术语白话
- 仓库：放项目文件的“网盘文件夹”。
- 分支：同一项目的不同修改线路，`main` 通常是主线路。
- 远程地址：你在 GitHub 仓库对应的网络地址，通常以 `.git` 结尾。

## 页面英文看不懂时怎么做
- GitHub 页面可用浏览器翻译成中文。
- 记住常见按钮位置：右上角头像区是账号设置，绿色按钮多是“新建/提交”。
- “New”一般是新建，“Create”一般是创建，“Upload”一般是上传。
- 看到红色报错先截图，再按报错关键词处理。

## 本地安装
当检测到用户未安装 Git 时，请询问用户的操作系统，并提供对应的终端安装命令：

* **Windows 系统 (使用系统自带 Winget)**：
    ```bash
    winget install --id Git.Git -e --source winget
    ```
    *(注：如果用户不熟悉终端，也可引导直接前往 `https://git-scm.com/` 下载安装包)*
* **macOS 系统 (使用 Homebrew)**：
    ```bash
    brew install git
    ```
* **Linux - Ubuntu / Debian 体系**：
    ```bash
    sudo apt update && sudo apt install git -y
    ```
* **Linux - CentOS / RedHat / Fedora 体系**：
    ```bash
    sudo dnf install git -y
    ```

## 步骤清单示例
**请一步一步带着用户做，如果有任何不确定的请用通俗易懂的语言询问用户。**
1. 打开浏览器，进入 GitHub 官网。  
   你会看到什么：右上角有“Sign up（注册）”或“Sign in（登录）”。
2. 如果你没有账号，先点“Sign up（注册）”完成注册。  
   你会看到什么：注册完成后页面提示你可以登录。
3. 点击“Sign in（登录）”并输入账号信息。  
   你会看到什么：登录后右上角会出现你的头像。
4. 点击右上角 `+`，选择“New repository（新建仓库）”。  
   你会看到什么：出现创建仓库表单页面。
5. 输入仓库名，并保持仓库为“空仓库初始化”。  
   操作要点：不要勾选 `Add a README file`、不要勾选 `Add .gitignore`、不要勾选 `Choose a license`。  
   你会看到什么：这三个初始化选项都保持未勾选状态。
6. 勾选“Public（公开）”或“Private（私有）”。  
   你会看到什么：对应选项变成已选中状态。
7. 点击“Create repository（创建仓库）”。  
   你会看到什么：进入新仓库主页，页面会显示后续要执行的 Git 命令示例。
8. 打开本地终端，进入项目目录。  
   你会看到什么：命令行前缀显示当前目录已是你的项目目录。
9. 运行 `git init` 初始化仓库。  
   你会看到什么：终端出现初始化成功提示。
10. 在执行 `git add .` 前，先创建或检查 `.gitignore`（防误传）。  
   操作要点：确保 `.gitignore` 至少包含以下内容：`node_modules/`、`.env`、密钥/证书私钥文件（如 `*.key`、`*.pem`、`*.p12`）、系统临时文件（如 `.DS_Store`、`Thumbs.db`）。  
   你会看到什么：项目目录里存在 `.gitignore` 文件，打开后能看到上述规则。
11. 上传前安全检查：运行 `git status`，确认待上传列表里没有 `.env`、私钥文件、证书私钥和系统临时文件。  
   你会看到什么：终端显示待上传文件列表，敏感文件不在列表中。
12. 运行 `git add .` 添加文件。  
   你会看到什么：终端通常无报错，表示已加入待提交列表。
13. 运行 `git commit -m "first commit"` 提交一次。  
   你会看到什么：终端显示提交成功和文件数量。
14. 按 GitHub 仓库页面提示，复制远程地址。  
   你会看到什么：地址通常以 `.git` 结尾。
15. 运行 `git remote add origin <仓库地址>`。  
   你会看到什么：终端通常无报错。
16. 运行 `git branch -M main`。  
    你会看到什么：终端通常无报错。
17. 运行 `git push -u origin main`。  
    你会看到什么：终端显示上传进度。
18. 刷新 GitHub 仓库页面。  
    你会看到什么：仓库页面能看到你的项目文件。

## 成功判定
- GitHub 仓库页面能看到完整项目文件。注意引导用户上传到 main 分支。
- 最新提交时间是刚刚。
- 你刷新页面后文件仍存在。

## 常见问题与处理
- 问题：`git` 命令提示找不到。  
  处理：说明 Git 未安装，先安装 Git 后重开终端。
- 问题：推送时提示认证失败。  
  处理：重新登录 GitHub；按提示使用浏览器授权；不要重复输入错误密码。
- 问题：提示仓库已存在内容，无法直接推送。  
  处理：先中止并回报，不建议新手自行执行 `git pull origin main --allow-unrelated-histories`。
- 问题：上传后仓库没有文件。  
  处理：先执行 `git status` 看是否有未提交文件，再补 `git add` 和 `git commit`。
