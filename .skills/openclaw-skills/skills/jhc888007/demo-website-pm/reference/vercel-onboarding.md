# Vercel 部署引导

## 适用场景
- 你已经把项目上传到 GitHub。
- 你要生成一个可分享的网站链接。
- 你希望后续代码更新后自动上线。

## 前置准备
- 你有 GitHub 账号并能正常登录。
- 你的项目仓库在 GitHub 可见。
- 你能打开浏览器并访问 Vercel（强调需要打开VPN）。
- 你知道项目默认分支（通常是 `main`）。

## 术语白话
- Production：正式线上版本链接，给别人访问时优先用这个链接。

## 页面英文看不懂时怎么做
- 先启用浏览器翻译成中文。
- 看按钮位置：中间大按钮多是“导入项目”，右上角多是账号菜单。
- “Import”是导入，“Deploy”是部署，“Settings”是设置。
- 看不懂配置项时，先保持默认值，不要乱改高级选项。

## 步骤清单
1. 打开 Vercel 官网。  
   你会看到什么：页面右上角有登录或注册入口。
2. 完成注册或登录。  
   你会看到什么：登录后进入控制台主页。
3. 选择“Continue with GitHub（用 GitHub 继续）”。  
   你会看到什么：页面跳转到 GitHub 授权页。
4. 在 GitHub 授权页点击确认授权。  
   你会看到什么：页面会跳回 Vercel 控制台。如果GitHub连接失败，还有其他一些注册方式，让用户基于网站上的指引来完成。
5. 在 Vercel 点击“Add New... -> Project”。  
   你会看到什么：出现可导入的 GitHub 仓库列表。
6. 找到你的仓库并点击“Import”。  
   你会看到什么：进入项目配置页。
7. 检查构建设置（Build 设置）是否自动识别。  
   你会看到什么：页面显示框架类型或构建命令；多数项目可直接用默认值。
8. 若项目包含数据库，部署前先在 Vercel 的 Storage 里添加 Neon（或由你指定的其他数据库/存储服务），并且连接项目。  
   你会看到什么：服务商控制台里出现可用的数据库/存储实例与连接信息。
9. 如果项目需要环境变量（环境变量是“运行时要用的账号键值”），在设置区填写；上一步的数据库连接信息通常也填在这里。  
   你会看到什么：变量名称和变量值输入框。
10. 点击“Deploy（部署）”。  
    你会看到什么：页面显示构建进度条和日志。
11. 等待部署完成。  
    你会看到什么：状态显示为“Ready”。
12. 点击生成的网址。  
    你会看到什么：网站页面可在公网打开。
13. 访问说明：Vercel 默认分配的地址（如 `xxx.vercel.app`）在国内网络环境下访问常需 VPN。  
    若不希望访客依赖 VPN：请按下方「独立域名」一节购买并绑定自己的域名，再按 Vercel 提示完成 DNS 解析。

## 独立域名
*与步骤 13 呼应：默认 `.vercel.app` 在国内常需 VPN；购买并绑定自有域名可减轻访客对 VPN 的依赖，按下列步骤操作。*

1. **进入域名设置**：在 Vercel 项目面板，点击顶部的 **`Settings`**，然后在左侧菜单点击 **`Domains`**。
2. **添加新域名**：点击 `Buy`，然后输入自己设想中的域名，如果可用，然后用外币信用卡付款。
3. **等待生效**：回到 Vercel 页面等待几分钟，看到域名状态变成蓝色的 ✔️ 并且带有小锁头，说明绑定和 HTTPS 证书全部配置成功！

## 成功判定
- Vercel 显示状态为“Ready”或“Production”。
- 打开部署链接后页面正常显示。
- 在你实际使用的网络环境下（默认 `.vercel.app` 在大陆常需 VPN；绑定自有域名后可在目标访客网络下再测）手机与桌面均可访问。

## 常见问题与处理
- 问题：仓库列表里看不到你的项目。  
  处理：回 GitHub 检查仓库是否存在；再回 Vercel 重新授权 GitHub 仓库访问权限。
- 问题：部署失败，日志一片红字。  
  处理：先看第一条红色错误；优先检查构建命令和环境变量是否缺失。
- 问题：页面打开空白或 404。  
  处理：确认访问的是最新 Production 链接，不是旧预览链接。
- 问题：更新 GitHub 后网站没更新。  
  处理：确认代码推送到了默认分支；然后在 Vercel 手动点“Redeploy（重新部署）”。
