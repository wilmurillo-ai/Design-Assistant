# 本地环境排错

## 适用场景
- 你在本地启动项目失败。
- 你看到红色报错但看不懂。
- 你要先在本地跑通，再上传 GitHub 和部署 Vercel。

## 前置准备
- 已安装 Cursor 或其他编辑器。
- 已安装终端工具（mac 自带 Terminal）。
- 已安装 Node.js（Node.js 是“让前端项目能运行的基础程序”）。
- 已安装 Python3（Python3 是“让后端脚本能运行的基础程序”）。
- 已安装 pip3（pip3 是“给 Python 安装依赖包的工具”）。
- 你知道项目根目录位置。

## 页面英文看不懂时怎么做
- 把报错整段复制到翻译工具，不要只翻一行。
- 重点看关键词：`error`（错误）、`failed`（失败）、`not found`（找不到）。
- 看不懂英文命令提示时，先按“最短路径”操作，不要一次执行很多命令。
- 每次只改一个点，避免问题叠加。

## 步骤清单
1. 打开终端，进入项目目录。  
   你会看到什么：终端路径显示为你的项目文件夹。
2. 先确认你要走哪条路线：前端路线（Node）或后端路线（Python）。  
   你会看到什么：你能明确这次只先跑一条路线，避免命令冲突。
3. 前端路线（Node）：运行 `node -v` 检查 Node.js。  
   你会看到什么：出现版本号，如 `v20.x.x`。
4. 前端路线（Node）：运行 `npm -v` 检查 npm。  
   你会看到什么：出现版本号，如 `10.x.x`。
5. 前端路线（Node）：运行 `npm install` 安装依赖（依赖是“项目运行要用到的外部包”）。  
   你会看到什么：终端出现下载进度，最后显示安装完成。
6. 前端路线（Node）：运行 `npm run dev` 启动本地服务。  
   你会看到什么：终端显示本地地址，如 `http://localhost:3000`。
7. 前端路线（Node）：打开浏览器访问本地地址。  
   你会看到什么：页面成功加载，不是空白报错页。
8. 前端路线（Node）：如果端口占用，先按提示改为新端口。  
   你会看到什么：终端显示新的端口号。
9. 前端路线（Node）：重新运行 `npm run dev`。  
   你会看到什么：终端给出新的本地地址。
10. 后端路线（Python）：运行 `python3 --version` 检查 Python3。  
    你会看到什么：出现版本号，如 `Python 3.11.x`。
11. 后端路线（Python）：运行 `pip3 --version` 检查 pip3。  
    你会看到什么：出现版本号和安装路径信息。
12. 后端路线（Python）：运行 `python3 -m venv .venv` 创建虚拟环境（虚拟环境是“给当前项目单独隔离依赖的环境”）。  
    你会看到什么：项目目录出现 `.venv` 文件夹。
13. 后端路线（Python）：运行 `source .venv/bin/activate` 激活虚拟环境。  
    你会看到什么：命令行前面出现 `(.venv)` 标记。
14. 后端路线（Python）：运行 `pip3 install -r requirements.txt` 安装依赖。  
    你会看到什么：终端出现安装进度，最后显示安装成功或已满足依赖。
15. 后端路线（Python）：运行 `python3 app.py` 启动服务。  
    你会看到什么：终端显示服务已启动和本地访问地址（如有地址提示）。
16. 后端路线（Python）：打开浏览器访问本地地址。  
    你会看到什么：页面成功加载，不是空白报错页。
17. 后端路线（Python）：如果端口占用，先按提示改为新端口后重启。  
    你会看到什么：终端显示新的端口号或新的访问地址。
18. 如果仍失败，只记录第一条红色报错。  
   你会看到什么：你能明确保存一条完整错误信息。
19. 停止继续尝试。  
    你会看到什么：终端不再新增报错信息。

## 成功判定
- `npm install` 无红色致命错误。
- `npm run dev` 能输出本地访问地址。
- `pip3 install -r requirements.txt` 无红色致命错误。
- `python3 app.py` 能输出启动成功信息或本地访问地址。
- 浏览器可打开本地页面。
- 你能重复关闭再启动一次，结果一致。

## 常见问题与处理
- 问题：`node` 或 `npm` 找不到。  
  处理：重装 Node.js；安装后重开终端再试。
- 问题：`npm install` 失败。  
  处理：先检查网络；再删掉 `node_modules` 和锁文件后重装；仍失败就记录第一条错误。
- 问题：`python3` 找不到。  
  处理：先确认 Python3 已安装；重开终端再试 `python3 --version`；仍失败就检查安装路径是否在系统 PATH（PATH 是“系统找命令的目录列表”）里。
- 问题：`pip3` 找不到。  
  处理：先运行 `python3 --version` 确认 Python3 正常；再尝试 `python3 -m pip --version`；仍失败就重装 Python3。
- 问题：`pip3 install -r requirements.txt` 失败。  
  处理：先检查网络和文件是否存在；再升级安装工具后重试：`python3 -m pip install --upgrade pip`；仍失败就记录第一条错误。
- 问题：提示端口已被占用。  
  处理：关闭占用该端口的旧进程，或用新端口重新启动（Node 和 Python 路线都适用）。
- 问题：虚拟环境异常（如已激活但依赖仍找不到）。  
  处理：先运行 `deactivate` 退出，再重新 `source .venv/bin/activate`；若 `.venv` 损坏，就删除后重新创建。
- 问题：本地地址打开是白屏。  
  处理：先看终端是否有运行时红错；再按第一条错误定位，不要盲改多处。
- 问题：权限不足（permission denied）。  
  处理：把项目放到当前用户可写目录（如桌面/文稿）后重试。

## 何时中止并回报
- 同一错误连续出现 3 次。
- 你需要执行删除系统文件或提权命令，但不确定风险。
- 你准备使用 `sudo pip3` 或修改系统级 Python，但不确定影响范围。
- 你发现虚拟环境与系统 Python 混用，且无法判断当前生效的是哪一个解释器。
- 报错涉及账号密钥、环境变量泄露风险。
- 你无法判断哪条报错是“第一条关键错误”。
- 中止后请回报：执行命令顺序、第一条红错原文、当前系统版本。
