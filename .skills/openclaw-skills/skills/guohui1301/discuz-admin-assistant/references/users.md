认证设置
所谓用户认证，顾名思义就是确认用户某些特定的身份。比如：实名认证、视频认证、商家认证等。那么什么情况下需要用户认证呢?下面就举两个简单的例子说明一下。比如网站中有一个团购版块，每天会有很多的团购交易生成，这时我们就需要有一个团购商家的认证。只有通过团购商家认证的用户才可以在团购版块发起交易。一方面可以将高质量的商家推荐给用户，另一方面也可以做为网站的增值服务。再比如，网站中有一个交友征婚版块。交友征婚是一个严肃性的话题，终生大事必须实名制啊，用户一般都需要征婚信息的可靠度。这时我们就需要实名认证功能，来验证用户的真实信息并留底，提高征婚信息的可靠度与严肃性。接下来我们就一一讲解一下用户认证功能的使用方法。

用户认证概述
用户可以通过认证需求来完善认证所需要的信息，管理员在后台审核这些信息，通过审核的用户即可成为认证用户。认证默认有7种模式，我们可以依据需求来设置商家认证、实名认证等。通过认证的用户在前台会有一个认证标识，管理员还可以设置不同的用户认证在不同版块的权限。

用户认证的设置方法
现在，我们就来介绍一下用户认证的设置方法：操作路径：【后台】=>【用户】=>【认证设置】进入用户认证页面，见下图这里的【认证设置】中默认提供 7 种模式，我们可以依据具体需求来设置不同的认证。其中实名认证和视频认证的认证名称是固定的，管理员不可以进行修改，视频认证需要在开启漫游应用的前提下启用才能生效。下面我们以【商家认证】为例来讲解一下设置认证类型的具体操作流程。首先，点击【编辑】按钮进入用户认证的设置页面，编辑认证时的各项说明：1、认证名称，可以设置认证的名称，如：商家认证。2、是否启用此认证。3、显示认证图标：选择“是”可以设置未认证的图标和认证图标。4、可选资料项：可选资料项的信息是用户栏目中的项目。如果想设置更多项目可以点击右上角的“点此新增资料项”，进入用户栏目添加即可。5、允许参与认证的用户组：设置允许参与认证的用户组，可以按住 CTRL 多选， 所有用户组都不选代表都可以参于认证。商家认证设置完成，见下图：提交后，在认证设置页面效果如下，这里也可以直接开启/关闭认证或者直接修改认证名称。备注：当启用了 设置好的【商家认证】后，点击提交后【商家认证】就会自动加载到左侧菜单的管理项目中。后台认证设置完成后，前台用户就可以进行认证了。

用户组
用户组分为系统组、会员组和自定义组，其中自定义用户组可以设置为多个晋级用户组，会员组及晋级组以积分确定组别和权限，其他组是人为设定，不会由系统自行改变。
系统组和非晋级组的设定不需要指定积分，Discuz! 预留了从站点管理员到游客等的 8 个系统头衔，非公开用户组的用户无法自行加入，需管理员人工添加。
会员用户组至少分为两组，其积分下限分别为负值(任一负数)和 0，否则将导致部分用户无法与用户组匹配的问题。

晋级用户组至少分为两组，其积分下限为 0，否则将导致部分用户无法与用户组匹配的问题。

 3种用户组介绍
1、会员用户组会员用户组是普通会员的用户组，该用户组以积分确定组别和权限。在此处可以设置各个用户组需要的最低积分，提交修改的时候系统会自动修改上一用户组的最高积分。点击用户组后面的详情可以设置该用户组的详细权限。
2、自定义用户组自定义用户组是管理员手工创建的用户组。点击“添加新自定义用户组”，可以添加一个新的用户组，可以自定义组头衔，星星数，头衔颜色。自定义用户组的用户需要在编辑会员时将其加入，用户被加入自定义用户组后，将不受会员用户组级别的限制。可以点击“查看”来查看当前有多少用户在这个自定义用户组里面，点击“编辑”来设置自定义用户组的权限。

现在以我们在自定义用户组中添加的“VIP用户”用户组为例，讲解一下用户组的详细设置。操作路径：【后台】=>【用户】=>【用户组】=>【自定义用户组】，添加了一个名为“VIP用户”的自定义用户组之后，点击“编辑”进行详细权限设置1、基本设置1)用户组头衔：设置用户组头衔名称。2)组图标：设置该用户组的图标，可以选择是输入 URL 或者是上传图片，使用方法同管理组中的组图标一样。3)访问站点权限禁止访问：禁止用户浏览本站任何页面。正常访问：当站点开放的时候，可以正常浏览页面。超级访问：即便站点关闭也可以访问，一般用于管理员或站内测试组。4)阅读权限：设置用户浏览帖子或附件的权限级别，范围 0～255，0 为禁止用户浏览任何帖子或附件。当用户的阅读权限小于帖子或附件的阅读权限许可(默认时为 1)时，用户将不能阅读该帖子或下载该附件。5)最多好有数：该用户组的用户能够拥有的最多好友数量，0为无限。除了用户组的好友数限制外，如果想单独给某个用户组的成员单独增加好友数，可以到【用户】=>【用户管理】=>【详情】中添加额外好友数。6)二级域名最短长度：站点开启二级域名功能后有效，0为禁止使用二级域名。7)允许隐身：是否允许用户在站点上隐身。8)允许积分转账：设置是否允许用户在个人中心将自己的交易积分转让给其他用户。注意：本功能需在 全局设置中启用交易积分后才可使用。9)允许发送短消息：设置是否允许用户发送短消息。选择“是”后可以设置24小时内发布短消息的最大数及是否可以给任何人发短消息，选择不允许给任何人发短消息，那么选择“否”当对方设置为只接受好友短消息，将无法对其发送短消息。10)允许使用 HTML 代码：是否允许用户发布信息的时候使用 HTML 代码。注意：开放 HTML 功能将产生安全隐患，请慎用。建议只在十分必要的情况下使用，并限制只开放给最核心的管理人员11）允许发站外URL：该项设置可以有效减少广告帖子。12)允许查看站点统计：是否允许用户查看站点统计。选择“是”用户在访问站点的时候页面底部会出现“站点统计”链接，点击即可查看。13)允许全文搜索：是否允许用户进行全文搜索，此设置只对搜索论坛有效。注意：当数据量大时，全文搜索将非常耗费服务器资源，请慎用。14)允许使用搜索：选择为该用户组用户开启哪些模块的搜索。包括搜索文章、论坛、日志、相册和圈子。15)操作理由短消息通知作者：设置用户在对他人评分或管理操作时是否强制输入理由和通知作者。16)允许自定义头衔：设置是否允许用户设置自己的头衔名字并在帖子页面中显示。选择“是”用户在【个人资料】中设置自己的头衔后效果如下：17)不受时间段限制：设置用户是否可以不受全局设置中设置的特殊时间段禁令限制。18)会员每小时发主题数限制：设置允许会员每小时最多的发表的主题数量，可以配合灌水预防功能进一步限制会员的发帖，可设置为 1～255 范围内的数值，0 为不限制。此功能会轻微加重服务器负担, 且对游客无效。19）会员每小时发回帖数限制：设置允许会员每小时最多的发回帖数量，可以配合灌水预防功能进一步限制会员的发帖，可设置为 1～255 范围内的数值，0 为不限制。此功能会轻微加重服务器负担，且对游客无效。20)启用防灌水验证码或验证问答机制：开启防灌水验证机制可以防止灌水机等，但会增加用户操作易用度。修改密码不受此限制，开启防灌水验证码或验证问答后，该设置有效。21)发表不受限制：设置发表是否不受灌水预防和最大字数限制等。22)忽略需要审核的关键字：选择“是”则此用户组的用户在发表时不会触发需要审核的关键字。23）允许用户创建淘专辑的数量：“0”表示不允许创建淘专辑。 24）最多允许关注淘专辑的数量：允许用户最多关注的淘专辑数量。
2、用户组体制特殊用户组体制分为“非公众用户组”、“公众用户组”、“收费公众用户组”、“免费公众用户组”。根据不同的用户组体制设置来决定的。1)公众用户组：设置是否为公众用户组。注意：使用此功能前请确定是否确实需要，否则用户一旦自行加入本组，即便之后再设置本组为非公众用户组，仍然无法将用户从该组中批量去除，除非删除本用户组。公众用户组不能关联管理组，以免产生安全隐患。2)非公众用户组：选择“否”为非公众用户组，当用户组设置为“非公众用户组”时，无论是以主用户组的形式，还是以扩展用户组的形式，均只能由管理员手工将用户加入本组。选择“是”会出现“收费公众用户组日价格”和“收费公众用户组一次最短购买时间”的设置3)收费公众用户组日价格：如果您需要用户支付相应的交易积分后才能加入本用户组，请在此输入本组的日购买价格，单位为交易积分所使用的单位。例如设置为 2，则用户购买为期 30 天的本组权限，需要支付 2*30=60 个交易积分，30 天有效期后该用户将自动从本组中退出。如设置为 0，则本组为免费公众用户组。注意: 本功能需要首先设定用户组为公众用户组且设置了有效的交易积分后才有效。4)收费公众用户组一次最短购买时间(天)：设置用户在购买本组权限时，一次支付价值至少多少日的交易积分，请设置为大于等于 1 的数值。例如日价格为 2，最短购买时间为 10，则用户至少需要花费 2*10=20 个交易积分，获得至少 10 天本组权限，才能加入本组。注意: 本功能需要首先设定用户组为收费公众用户组才有效。

后台管理团队
Discuz! 将以往的单一论坛模式下的管理组管理模式升级成了通过管理团队的分工和协作来管理站点内容的模式。细分每个管理团队成员的管理范围和内容，甚至可以细化到某一项独立的管理动作上；管理团队内不同职务的管理人员有着各自不同的管辖范围，可以避免站内的管理工作互相冲突；负责不同区域的管理人员之间权力不能互通，可以达到很好的分权效果。
