# Cron 表达式示例参考

## 📅 基本格式

```
*    *    *    *    *
│    │    │    │    │
│    │    │    │    └── 星期 (0-6) (0=周日)
│    │    │    └─────── 月份 (1-12)
│    │    └──────────── 日期 (1-31)
│    └───────────────── 小时 (0-23)
└────────────────────── 分钟 (0-59)
```

## 🕐 常用示例

### 每天固定时间
- `0 9 * * *` - 每天上午9点
- `0 12 * * *` - 每天中午12点
- `0 18 * * *` - 每天下午6点
- `0 22 * * *` - 每天晚上10点

### 工作日
- `0 9 * * 1-5` - 周一至周五上午9点
- `0 17 * * 1-5` - 周一至周五下午5点

### 每周特定日期
- `0 10 * * 1` - 每周一上午10点
- `0 15 * * 5` - 每周五下午3点
- `0 9 * * 0` - 每周日上午9点

### 每月
- `0 9 1 * *` - 每月1日上午9点
- `0 10 15 * *` - 每月15日上午10点

### 每小时/每半小时
- `0 * * * *` - 每小时整点
- `30 * * * *` - 每小时30分
- `*/15 * * * *` - 每15分钟
- `*/30 * * * *` - 每30分钟

## 📅 ISO 时间格式示例

### 一次性任务时间格式
- `2026-03-26T09:00:00+08:00` - 2026年3月26日上午9点（北京时间）
- `2026-12-25T18:30:00+08:00` - 2026年圣诞节下午6:30
- `2026-06-01T08:00:00+08:00` - 2026年儿童节上午8点

### 相对时间格式
  （注：需要检查openclaw cron是否支持相对时间）
- `tomorrow 09:00` - 明天上午9点
- `+1 hour` - 1小时后
- `+30 minutes` - 30分钟后
- `next monday 10:00` - 下周一上午10点

## 🔧 OpenClaw Cron 特定格式

### 创建一次性任务
```bash
openclaw cron add \
  --name "会议提醒" \
  --at "2026-03-26T14:30:00+08:00" \
  --session isolated \
  --message "下午2:30有团队会议" \
  --announce \
  --channel feishu \
  --to "ou_xxxxxx" \
  --delete-after-run
```

### 创建周期性任务
```bash
openclaw cron add \
  --name "每日早安" \
  --cron "0 9 * * *" \
  --session isolated \
  --message "早上好！新的一天开始了" \
  --announce \
  --channel feishu \
  --to "ou_xxxxxx"
```

## ⚠️ 注意事项

1. **时区问题**：OpenClaw使用UTC时间，中国时区需要+08:00
2. **时间格式**：必须使用ISO 8601格式：`YYYY-MM-DDTHH:MM:SS+08:00`
3. **测试验证**：创建任务后，使用 `openclaw cron list --json` 验证配置
4. **执行检查**：重要任务建议在执行前30分钟检查系统状态

## 🚀 最佳实践

1. **命名规范**：任务名称清晰明确，如"每日AI新闻收集"、"每周报告生成"
2. **消息内容**：提醒消息简洁明了，包含必要信息
3. **测试任务**：先创建测试任务验证流程，如5分钟后的测试提醒
4. **日志记录**：重要任务的创建和执行记录到MEMORY.md中
5. **错误处理**：设置失败通知机制，确保任务异常时能及时知道
