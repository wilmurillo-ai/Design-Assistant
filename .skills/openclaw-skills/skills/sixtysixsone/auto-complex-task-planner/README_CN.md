## 📋 任务分类

### 自动使用子 agent 的任务

| 类型 | 关键词 | 示例 | 预计耗时 |
|------|--------|------|---------|
| **调研** | 调研、搜集、分析、研究 | "调研 XXX 市场" | 10-15 分钟 |
| **开发** | 开发、编写、创建、实现 | "开发 XXX 功能" | 15-30 分钟 |
| **批量** | 批量、全部、所有、删除 | "批量删除 XXX" | 5-10 分钟 |
| **文档** | 报告、文档、方案、总结 | "生成 XXX 报告" | 20-30 分钟 |
| **搜索** | 搜索、查找、Google、百度 | "搜索 XXX 信息" | 10-15 分钟 |

### 主 session 直接处理的任务

| 类型 | 关键词 | 示例 | 耗时 |
|------|--------|------|------|
| **查询** | 天气、时间、日期、状态 | "今天天气" | < 1 分钟 |
| **配置** | 修改、设置、更新 | "修改配置" | < 5 分钟 |
| **聊天** | 你好、谢谢、再见 | 日常对话 | < 1 分钟 |
| **通知** | 发送、通知、提醒 | "发消息" | < 1 分钟 |

### 优先级识别

| 优先级 | 关键词 | 处理方式 |
|--------|--------|---------|
| **高** | 紧急、优先、马上、立刻、急 | 优先执行 |
| **普通** | 无特殊关键词 | 正常执行 |
| **低** | 有空、不急、慢慢、方便时 | 空闲时执行 |

## ⚙️ 配置选项

### 环境变量

```bash
# 子 agent 超时时间（秒）
export SUBAGENT_TIMEOUT=600

# 最大重试次数
export MAX_RETRIES=3

# 质量阈值
export QUALITY_THRESHOLD=0.7

# 是否启用并行执行
export ENABLE_PARALLEL=true

# 最大并发子 agent 数
export MAX_CONCURRENT=5

# 是否启用优先级队列
export ENABLE_PRIORITY=true
```

### 配置文件

```json
{
    "taskScheduler": {
        "autoMode": true,
        "qualityThreshold": 0.7,
        "maxRetries": 3,
        "timeout": 600,
        "parallel": {
            "enabled": true,
            "maxConcurrent": 5
        },
        "priority": {
            "enabled": true,
            "highKeywords": ["紧急", "优先", "马上"],
            "lowKeywords": ["有空", "不急", "慢慢"]
        }
    }
}
```

## 🔧 故障排查

### 子 agent 启动失败

**现象**: 创建子 agent 后无响应

**解决**:
```bash
# 检查子 agent 状态
subagents list

# 查看日志
tail -f /home/admin/.openclaw/logs/*.log

# 检查资源
free -h && df -h
```

### 任务质量不达标

**现象**: 质量评分 < 0.7

**解决**:
1. 优化任务描述
2. 使用任务模板
3. 调整质量阈值
4. 重新执行任务
