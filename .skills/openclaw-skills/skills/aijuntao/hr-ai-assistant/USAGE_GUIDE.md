### 1. 配置 API Key

有四种配置方式，任选其一：

#### 方式 1: 一键配置（最简单）⭐

在对话中直接粘贴 API Key，系统会自动配置！

#### 方式 2: 环境变量

```bash
export HRRULE_API_KEY='your-api-key'
```

#### 方式 3: 编辑配置文件

编辑 `config.json` 文件,填入您的 API Key:

```json
{
  "ws_url": "ws://192.168.112.114:5000",
  "api_key": "your-actual-api-key",  // 在这里填入您的 API Key
  "model": "deepseek-ai/DeepSeek-R1",
  "timeout": 120,
  "verbose": true
}
```

#### 方式 4: .env 文件

复制 `.env.example` 为 `.env` 并填入 API Key

## 集成到钉钉

### 步骤 1: 配置机器人

在钉钉群组中添加自定义机器人,获取 Webhook URL。

### 步骤 2: 接收消息

WorkBuddy 自动接收钉钉消息。

### 步骤 3: 调用 skill

当收到 HR 相关消息时,skill 自动触发:

```python
# 伪代码示例
async def handle_dingtalk_message(message: str):
    # 检测是否为 HR 相关请求
    if is_hr_request(message):
        # 调用 hr-ai-assistant skill
        response = await call_hrrule_api(
            api_key=config['api_key'],
            content=message,
            tag_id=parse_tag_id(message),
            rt=parse_rt(message),
            on_chunk=lambda text: send_to_dingtalk(text)
        )

        # 发送回复到钉钉
        send_to_dingtalk(response)
```

### 步骤 4: 流式返回

AI 响应会实时流式发送回钉钉群组。

## 故障排除

### 问题 1: 连接失败

**检查:**
- WebSocket URL 是否正确
- 网络连接是否正常
- 防火墙是否阻止连接
- 服务器是否运行

**解决:**
```bash
# 测试网络连接
ping 192.168.112.114

# 检查 WebSocket 端口
telnet 192.168.112.114 5000
```

### 问题 2: 认证错误

**检查:**
- API Key 是否正确
- API Key 是否过期
- API Key 是否有权限

**解决:**
更新 `config.json` 中的 API Key。

### 问题 3: 无响应

**检查:**
- 内容参数是否为空
- Tag ID 和 RT 是否有效
- 服务器是否正常处理请求

**解决:**
简化请求,重新发送。

### 问题 4: 超时

**检查:**
- 请求是否过于复杂
- 服务器负载是否过高
- 网络是否稳定

**解决:**
- 简化请求内容
- 增加超时时间
- 重试请求
