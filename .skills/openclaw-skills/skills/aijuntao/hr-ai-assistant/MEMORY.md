# HR AI Assistant - 重要规则记忆

## ⚠️ 下载链接处理规则（最重要）

### 核心原则：流式输出有链接，总结时也必须输出链接

**问题背景**：
- HRrule AI 脚本会生成带下载链接的 Markdown 输出
- 链接包含真实的 fileurl（带临时 token）
- 用户需要这些链接来下载生成的 Word/Excel 文档
- 如果我在总结时省略了链接，用户体验极差

**具体要求**：

1. ✅ **必须保留完整的下载链接格式**：
   ```markdown
   ---
   📎 **附件 {file_id}** - {filename}
   [点击下载附件]({download_url})
   ---
   ```

2. ✅ **在总结时，原封不动地把链接部分复制出来**
   - 不要重新生成链接或简化内容
   - 不要因为重新总结就丢失链接
   - 确保用户在 WorkBuddy 页面上能看到可点击的链接

3. ✅ **检测方法**：
   - 检查脚本输出中是否包含 `<rulefile>` 标签
   - 检查输出中是否包含 `📎 **附件` 字样
   - 检查输出中是否包含 `[点击下载附件]()` Markdown 链接

4. ✅ **适用场景**：
   - 生成 KPI 考核表（Excel）
   - 生成招聘JD（Word）
   - 生成年终总结（Word）
   - 生成任何 HR 文档（带附件）

### 正确示例 ✅

```markdown
完美！✅ HR AI已经为你生成了专业的市场专员招聘JD！

（这里可以简要总结内容，例如：包含岗位职责、任职要求、薪资福利等...）

---

📎 **附件 244494** - 招聘JD_84164_20260320_150748.docx
[点击下载附件](https://oss.hrrule.com/gwnote/2026/03/20/招聘JD_84164_20260320_150748.docx?e=...)

---
```

### 错误示例 ❌

```markdown
完美！✅ HR AI已经为你生成了专业的市场专员招聘JD！
📄 HR AI 生成的招聘JD包含岗位职责、任职要求、薪资福利等内容...
```

**问题**：只总结了内容，没有下载链接，用户无法下载文档！
