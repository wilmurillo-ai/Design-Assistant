## 1. 安装设置 / Installation

### 前提条件 / Prerequisites

- OpenClaw 已安装并运行
- Node.js >= 18.0.0

### 安装步骤 / Installation Steps

```bash
# 进入技能目录
cd ~/.openclaw/workspace/skills/session-password

# 安装依赖
npm install

# 运行设置向导
node scripts/setup.js
```

## 2. 首次配置 / First-time Setup

### 步骤 1：选择语言 / Step 1: Select Language

```
请选择语言 / Select language:
  1. 中文 (zh-CN)
  2. English (en)
[1/2, default 1]: 
```

### 步骤 2：创建管理员用户 / Step 2: Create Admin User

输入用户名（默认：admin）和口令。

**口令要求 / Password Requirements:**
- 至少 12 个字符
- 包含大写字母 (A-Z)
- 包含小写字母 (a-z)
- 包含数字 (0-9)
- 包含特殊符号 (@$!%*?&)

### 步骤 3：安全邮箱 / Step 3: Security Email

**必填项** - 用于账户恢复。忘记口令时，恢复码将发送到此邮箱。

### 步骤 4：安全问题（可选）/ Step 4: Security Question (Optional)

设置一个安全问题作为备用验证方式。

### 步骤 5：设置 / Step 5: Settings

- **超时时间：** 闲置多久后需要重新认证（默认 60 分钟）
- **自动重置：** 认证后是否自动执行 /new 或 /reset

## 3. 日常使用 / Daily Usage

### 认证流程 / Authentication Flow

1. 启动会话时，系统提示输入口令
2. 输入正确口令后，获得访问权限
3. 闲置超过设定时间后，需要重新认证

### 认证状态 / Authentication Status

认证后，系统会记录：
- 认证时间
- 最后活跃时间
- 失败尝试次数

### 自动超时 / Auto Timeout

- 默认 60 分钟闲置后超时
- 超时后需要重新输入口令
- 可在配置中调整超时时间

## 4. 口令恢复 / Password Recovery

### 方法一：邮箱恢复 / Method 1: Email Recovery

当忘记口令时：

1. 输入："忘记口令" 或 "forgot password"
2. 系统发送 6 位恢复码到安全邮箱
3. 输入收到的恢复码
4. 设置新口令

### 方法二：安全问题 / Method 2: Security Question

如果设置了安全问题：

1. 输入："忘记口令"
2. 选择安全问题验证
3. 正确回答问题后设置新口令

---

## 5. 修改口令 / Change Password

在已认证状态下：

1. 输入："修改口令" 或 "change password"
2. 输入当前口令确认身份
3. 输入新口令（需满足复杂度要求）
4. 确认新口令

## 7. 常见问题 / FAQ

### Q: 口令忘记了怎么办？

**A:** 使用"忘记口令"指令，通过安全邮箱或安全问题恢复。

### Q: 账户被锁定了怎么办？

**A:** 等待 15 分钟后自动解锁，或通过邮箱恢复重置。

### Q: 如何修改超时时间？

**A:** 编辑 `memory/auth-config.json` 中的 `timeoutMs` 值（单位：毫秒）。

### Q: 安全邮箱可以更换吗？

**A:** 可以，编辑 `memory/auth-users.json` 中对应用户的 `email` 字段。

### Q: 支持多用户吗？

**A:** 当前版本支持单用户，多用户功能计划在未来版本中实现。

## 安全建议 / Security Recommendations

1. ✅ 使用强口令（超过 12 位，混合字符类型）
2. ✅ 定期更换口令
3. ✅ 使用真实有效的安全邮箱
4. ❌ 不要使用常见单词或日期作为口令
5. ❌ 不要与他人分享口令
