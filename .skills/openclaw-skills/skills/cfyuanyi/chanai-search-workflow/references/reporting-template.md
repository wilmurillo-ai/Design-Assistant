# 搜索结果汇报模板

每次使用本 skill 完成搜索后，优先按下面结构汇报。保持简洁，但要把路线讲清楚。

## 标准模板

```markdown
## 搜索结论
[一句话结论]

## 搜索路线
- 路由：domestic-first / global-first / mixed
- 主搜索入口：[engine/site]
- 交叉验证：[engine/site 或 无]
- 是否发生 fallback：[有/无]

## 关键信息
- [信息点 1]
- [信息点 2]
- [信息点 3]

## 可靠性判断
- [高 / 中 / 低]
- 原因：[例如：双源一致 / 仅聚合页 / 时效性强未完全确认]
```

## 极简模板

适合用户只要结果时：

```markdown
- **结论**：[一句话]
- **路线**：[domestic-first / global-first / mixed]；主用 [X]，补充 [Y]
- **可靠性**：[高/中/低]
```

## 什么时候必须写清楚路线
以下情况必须明确说明：
- 航班、价格、库存、政策、新闻这类高时效信息
- 搜索中发生 fallback
- 搜索结果来自聚合页而非原始来源
- 用户问“走的什么通路”

## 可靠性分级建议

### 高
- 两个以上独立来源一致
- 有原始来源或官方来源
- 关键信息明确

### 中
- 有多个来源，但主要是聚合页
- 时间较新，但没有直接原始来源
- 结论可信但仍建议以官方页为准

### 低
- 只有单一来源
- 页面抓取不完整
- 搜索结果弱或明显噪音大
- 需要用户自行进一步确认

## 示例：航班查询

```markdown
## 搜索结论
2026-03-13 上海到赤峰查到了直飞信息，公开结果显示有 PVG -> CIF 班次。

## 搜索路线
- 路由：domestic-first
- 主搜索入口：Baidu / 公开搜索结果页
- 交叉验证：Bing CN / 聚合航班页
- 是否发生 fallback：有，改用公开搜索页抓取

## 关键信息
- 检索到 FM2135 / MU2135 相关结果
- 公开结果显示 PVG -> CIF 直飞班次
- 最终是否可售仍建议以实际订票页为准

## 可靠性判断
- 中
- 原因：结果来自公开搜索与聚合页，能交叉印证航线存在，但可售状态未完全确认
```

## 示例：GitHub 文档查询

```markdown
- **结论**：OpenClaw 的 memory 索引现在支持本地 Ollama embeddings。
- **路线**：global-first；主用 Google `site:github.com`，补充官方 docs
- **可靠性**：高
```
