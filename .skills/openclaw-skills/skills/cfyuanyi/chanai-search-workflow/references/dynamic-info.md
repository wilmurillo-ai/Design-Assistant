# 动态信息白名单与边界规则

## 目的
把“可用公开搜索初筛”和“必须以实时页/官方页为准”区分开，避免把聚合页、摘要页、旧时刻表误当成最终结论。

## 动态信息白名单
以下问题默认视为 **高动态信息**：

- 航班 / 机票 / 中转 / 余票 / 舱位
- 酒店房价 / 房态 / 可订状态
- 商品价格 / 库存 / 优惠活动
- 股票 / 币价 / 基金净值
- 新闻快讯 / 突发事件 / 热点状态
- 政策生效状态 / 报名状态 / 开售状态
- 演出 / 车票 / 门票 / 名额

## 核心规则

### 规则 1：公开搜索只做初筛
公开搜索结果适合：
- 找有没有这条航线/商品/政策
- 找大概价格区间
- 找候选平台和候选页面
- 找历史趋势和背景信息

公开搜索结果**不适合直接当最终结论**：
- 最终可售价格
- 余票/库存
- 是否还有名额
- 当日是否实际可订
- 退改/限时优惠等即时信息

### 规则 2：最终决策必须回到实时页
对高动态信息，最终结论优先级：
1. 官方实时页
2. OTA / 平台实时可售页
3. 官方公告 / 原始来源
4. 公开搜索摘要页

### 规则 3：摘要页不能替代实时页
如果搜索结果里只有：
- 搜索摘要
- 聚合页标题
- “低至 xx 元起”
- “未来 3 个月均价”

则只能写成：
- **参考价 / 区间价 / 趋势价**
- 不能写成“当前就是这个价格”

## 航班 / 票价类边界规则

### 可以从公开搜索得到的
- 是否存在直飞
- 大致航线（PVG -> CIF）
- 常见航司 / 航班号候选
- 大概飞行时长
- 历史或聚合价格区间

### 不应直接从公开搜索下结论的
- 当天/明天准确可售班次
- 实时票价
- 余票状态
- 是否仅剩少量
- 舱位差异
- 最终退改规则

### 航班类标准做法
1. 先用公开搜索定位航线和候选航班
2. 再去 OTA/官方页确认实时班次
3. 最终下单建议只基于实时页信息给出

## 航班案例反例
如果公开搜索抓到：
- 某班次的旧时刻表
- 某站点的计划班次
- 某聚合页的历史航班号

而实时 OTA 页面显示：
- 仅剩一个可售直飞班次
- 起降时间不同
- 价格和余票状态不同

则必须以 **实时 OTA 页** 为准。

## 输出要求
遇到动态信息时，必须说明：
- 哪些是**实时结果**
- 哪些只是**公开搜索参考**
- 是否已进入**最终可售页/官方页**
- 如果没有进入，只能给出初筛建议，不能替用户拍板
