# 测试用例文档格式说明

## 标准Markdown格式

测试用例文档采用标准Markdown格式，遵循以下规则：

### 标题层级

```markdown
# 一级标题 - 产品名称
## 二级标题 - 功能模块
### 三级标题 - 子功能
#### 四级标题 - 功能点
```

### 测试点格式

```markdown
- **测试点：XXX验证**
  - [ ] 验证项1
  - [ ] 验证项2
  - [ ] 验证项3
```

### 关键元素

1. **测试点标识**：使用粗体标记测试点名称，如 `**测试点：XXX验证**`
2. **验证项列表**：使用 `- [ ]` 格式的待办事项列表
3. **层级缩进**：使用2个空格缩进表示层级关系
4. **文本处理**：
   - 移除checkbox标记 `[ ]` 和 `[x]`
   - 保留粗体标记 `**文本**`
   - 支持emoji和特殊字符

## 示例结构

```markdown
# 产品名称 - 测试用例思维导图

## 一、功能模块A

### 1.1 子功能A1

#### 1.1.1 功能点A1-1
- **测试点：触发条件验证**
  - [ ] 验证条件1
  - [ ] 验证条件2
  - [ ] 验证条件3

- **测试点：展示验证**
  - [ ] 验证展示项1
  - [ ] 验证展示项2

#### 1.1.2 功能点A1-2
- **测试点：交互验证**
  - [ ] 验证交互1
  - [ ] 验证交互2

### 1.2 子功能A2
...
```

## 转换规则

### Markdown到XMind的映射

| Markdown元素 | XMind元素 |
|-------------|----------|
| # 一级标题 | 根主题 |
| ## 二级标题 | 一级子主题 |
| ### 三级标题 | 二级子主题 |
| #### 四级标题 | 三级子主题 |
| - 测试点 | 四级子主题 |
| - [ ] 验证项 | 五级子主题（带☐标记） |

### 特殊处理

1. **Checkbox转换**：
   - `[ ]` → `☐`（未完成）
   - `[x]` → `☑`（已完成）

2. **粗体处理**：
   - 保留粗体文本内容，移除`**`标记

3. **层级判断**：
   - 标题层级：根据`#`数量判断
   - 列表层级：根据缩进空格数判断（每2个空格一级）

## 最佳实践

1. **清晰的层级结构**：确保标题层级清晰，避免跨级
2. **完整的测试点**：每个功能点都应有对应的测试点
3. **详细的验证项**：验证项应具体、可执行
4. **统一的格式**：保持整个文档格式一致
