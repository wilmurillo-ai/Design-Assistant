# 需求深度分析指南

## 1. 深度需求分析方法

**必须对需求进行深度分析，不要只停留在表面！**

### 1.1 逐句分析法

对需求文档中的每一句话进行以下分析：

1. **识别关键词**：
   - 条件词：当、如果、若、满足...时、在...情况下
   - 动作词：点击、输入、选择、提交、展示、隐藏、跳转、调用
   - 数据词：创建、读取、更新、删除、保存、加载、同步
   - 状态词：启用、禁用、显示、隐藏、选中、未选中
   - 关系词：与、和、或、且、包含、属于、关联

2. **提取测试要素**：
   ```
   需求句子："当用户已触发OSA，则在页面底部展示OSA常驻入口"
   
   分析过程：
   - 条件：用户已触发OSA
   - 动作：展示
   - 对象：OSA常驻入口
   - 位置：页面底部
   
   提取验证项：
   - 验证用户已触发OSA时入口展示
   - 验证用户未触发OSA时入口不展示
   - 验证入口展示位置在页面底部
   - 验证入口为常驻特性
   ```

### 1.2 业务流程分析法

对于涉及业务流程的需求：

1. **绘制流程图**：
   - 识别流程的起点和终点
   - 识别流程中的每个步骤
   - 识别流程中的判断分支
   - 识别流程中的循环结构

2. **测试每个节点**：
   - 每个流程节点都应有对应的测试点
   - 每个判断分支都应有对应的验证项
   - 每个异常退出都应有对应的异常测试

### 1.3 数据流分析法

对于涉及数据交互的需求：

1. **追踪数据流**：
   - 数据从哪里来（数据源）
   - 数据如何处理（数据处理）
   - 数据到哪里去（数据存储/展示）

2. **测试每个环节**：
   - 数据获取：验证数据获取的正确性
   - 数据处理：验证处理逻辑的正确性
   - 数据存储：验证存储的完整性
   - 数据展示：验证展示的准确性

### 1.4 状态转换分析法

对于涉及状态变化的需求：

1. **识别所有状态**：
   - 初始状态
   - 中间状态
   - 终止状态

2. **测试状态转换**：
   - 每个状态转换都应有对应的验证项
   - 验证状态转换的条件
   - 验证状态转换的时机
   - 验证状态转换的结果

### 1.5 用户场景分析法

从用户角度分析：

1. **识别用户角色**：
   - 不同角色可能有不同的权限和操作
   - 每个角色都应有对应的测试场景

2. **分析使用场景**：
   - 正常使用场景
   - 异常使用场景
   - 极端使用场景
   - 组合使用场景

## 2. 验证项生成详细指导

在生成验证项时，请遵循以下步骤：

1. **提取需求中的所有条件**：
   ```
   需求："当用户满足物料条件时，展示OSA入口"
   提取验证项：
   - [ ] 验证满足物料条件时入口正确展示
   - [ ] 验证不满足物料条件时入口不展示
   - [ ] 验证物料条件变化时入口动态更新
   - [ ] 验证页面刷新后入口状态保持
   - [ ] 验证物料条件为空时的处理逻辑
   ```

2. **分析需求中的所有动作**：
   ```
   需求："点击电话图标触发拨号"
   提取验证项：
   - [ ] 验证点击电话图标触发拨号功能
   - [ ] 验证拨号界面正确展示经纪人号码
   - [ ] 验证取消拨号后返回页面正常
   - [ ] 验证拨号成功后的记录
   - [ ] 验证电话权限拒绝时的提示
   - [ ] 验证无网络时的拨号提示
   ```

3. **识别需求中的所有状态**：
   ```
   需求："入口在页面底部展示"
   提取验证项：
   - [ ] 验证入口在页面底部正确展示
   - [ ] 验证入口不遮挡页面内容
   - [ ] 验证滚动页面时入口位置固定
   - [ ] 验证入口与页面底部安全距离正确
   - [ ] 验证横竖屏切换时入口位置正确
   - [ ] 验证不同屏幕尺寸下入口展示正确
   ```

4. **挖掘隐含的业务规则**：
   ```
   需求："入口物料与小区详情页共用"
   隐含验证项：
   - [ ] 验证入口物料与小区详情页一致
   - [ ] 验证物料推送策略正确执行
   - [ ] 验证物料内容符合用户画像
   - [ ] 验证物料更新逻辑正确
   - [ ] 验证物料类型完整性（工具/动态/情报/求助）
   - [ ] 验证不同业务类型展示（专属管家/交易专家/金融专家）
   ```

## 3. 需求分析示例

```
需求原文："小区在售房源列表页，进入页面满足物料条件就出，则在该页面底部展示OSA常驻入口"

深度分析：
1. 页面类型：小区在售房源列表页
2. 触发条件：满足物料条件
3. 触发时机：进入页面
4. 展示位置：页面底部
5. 入口类型：OSA常驻入口

提取测试点：
- 触发条件验证（物料条件）
- 触发时机验证（进入页面）
- 展示位置验证（页面底部）
- 入口类型验证（常驻特性）
- 入口物料验证（物料类型和内容）

每个测试点进一步细化验证项...
```
