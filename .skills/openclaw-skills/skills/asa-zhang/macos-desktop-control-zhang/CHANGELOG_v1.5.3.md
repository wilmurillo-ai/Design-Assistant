## 📊 质量评分系统

### 评分规则

```
质量评分 = 借鉴次数 - 2 × 失败次数
```

### 示例计算

```
操作 A: 👁️ 100 次 ❌ 5 次  → 评分 = 100 - 2×5 = 90  ⭐⭐⭐⭐⭐
操作 B: 👁️ 80 次  ❌ 2 次  → 评分 = 80 - 2×2 = 76  ⭐⭐⭐⭐
操作 C: 👁️ 120 次 ❌ 30 次 → 评分 = 120 - 2×30 = 60 ⭐⭐⭐
```

### 排序结果

```
同一应用多个操作：
1. 操作 A - 评分 90 (优先推荐)
2. 操作 B - 评分 76
3. 操作 C - 评分 60
```

---

## 🎯 为什么是 2 倍？

### 权重设计

**失败的影响应该是成功的 2 倍**：

| 场景 | 计算 | 评分 | 说明 |
|------|------|------|------|
| 完美操作 | 100 - 2×0 | 100 | 从不失败 |
| 优秀操作 | 100 - 2×5 | 90 | 偶尔失败 |
| 一般操作 | 100 - 2×25 | 50 | 失败较多 |
| 差劲操作 | 100 - 2×50 | 0 | 一半失败 |
| 危险操作 | 100 - 2×100 | -100 | 失败更多 |

### 心理学依据

- **损失厌恶**: 失败比成功更让人印象深刻
- **信任建立**: 10 次成功建立信任，1 次失败摧毁信任
- **风险规避**: 用户倾向于选择失败少的操作

## 🔧 技术实现

### 1. 失败记录

```python
# 操作失败时
memory.increment_failure(
    app_name="Safari",
    command="打开 Safari"
)

# 效果：失败次数 +1
# 打开 Safari ❌ 0 → ❌ 1
```

### 2. 质量评分计算

```python
# 解析操作时自动计算
usage = op.get('usage_count', 0)
fails = op.get('fail_count', 0)
op['quality_score'] = usage - 2 * fails

# 排序
similar_ops.sort(key=lambda x: x['quality_score'], reverse=True)
```

### 3. 智能分析

```python
quality = usage - 2 * fails

if quality > 100:
    return True, f"🔥 热门优质操作（评分{quality}）"
elif quality > 50:
    return True, f"⭐ 高质量操作（评分{quality}）"
elif quality > 0:
    return True, f"✅ 可靠操作（评分{quality}）"
elif quality > -20:
    return True, f"⚠️ 一般操作（评分{quality}）"
else:
    return True, f"❗ 低质量操作（评分{quality}，失败较多）"
```

## 🎯 总结

### 核心理念

**用成功 -2×失败 反映真实质量**

- 不只看成功 → 也看失败
- 不只看数量 → 更看质量
- 不只看表面 → 更看本质

- 高质量操作 → 评分高 → 优先推荐 → 更多人用
- 低质量操作 → 评分低 → 自然淘汰 → 避免踩坑

### 一句话总结

**让质量评分说话，让可靠操作优先！**
