# Domotics Feature Requests

Automation, reliability, observability, safety, and energy optimization capabilities requested during smart-home operations.

**Areas**: lighting | climate | security | access_control | energy | sensors | routines | voice_assistants | integrations | network
**Statuses**: pending | in_progress | resolved | wont_fix

## Common Request Categories

| Category | Examples |
|----------|----------|
| Automation Governance | Rule conflict detector, precedence simulator, anti-loop guardrails |
| Sensor Reliability | Drift detection, stale-value alarms, confidence scoring |
| Connectivity Resilience | Zigbee/Z-Wave health probes, MQTT disconnect auto-recovery, retry strategies |
| Integration Stability | Schema diff monitor, webhook contract tests, token expiry alerts |
| Safety Controls | Human-confirmation workflows, notify-only fallback, rollback templates |
| Energy Optimization | Tariff-aware schedules, occupancy + climate co-optimization |
| Observability | Routine timeline tracing, latency/jitter dashboards, failure clustering |

## Safety Reminder

Feature ideation must preserve safe defaults.
High-impact routines must support human confirmation and conservative fallback behavior.
