# Article Checklist

## Before writing
- [ ] Search the target keyword on Google
- [ ] Analyze top 5 results: what they cover, format, length
- [ ] Identify gap: what none of them cover (your differentiator)
- [ ] **Topic research**: read primary sources (official docs, data reports, research). If about your own product, read your own docs. If external topic, 5-10 web searches + fetch top authoritative sources.
- [ ] **Research brief compiled**: key facts with sources, user pain points, gaps to fill, debunked assumptions
- [ ] Determine format: listicle, guide, comparison, how-to
- [ ] Build keyword map: primary → title/H1/intro/conclusion, secondary → H2s/body, questions → FAQ

## Title & Meta
- [ ] Generate 3 title options with character count + keyword position analysis
- [ ] Final title: 60 chars max, keyword near start
- [ ] Meta description: 150-160 chars, includes primary keyword + CTA
- [ ] URL: /blog/keyword-slug (short, clean)

## Structure
- [ ] H1: exact or close to target keyword
- [ ] H2/H3: secondary keywords, related questions, no level skipping (H1→H2→H3)
- [ ] Intro: 2-3 sentences, hook + value promise, core answer in first 150 words
- [ ] TL;DR or Key Takeaways section near the top
- [ ] Length: 1500-2500 words (pillar), 800-1200 (tier 2/3)
- [ ] Comparisons in tables, not paragraphs
- [ ] One topic per section, paragraphs 3-5 sentences max
- [ ] Conclusion answers opening question + gives next steps

## CORE-EEAT Quality
- [ ] ≥3 query variants/synonyms covered
- [ ] Audience stated ("this article is for...")
- [ ] ≥5 precise numbers with units
- [ ] ≥1 external citation per 500 words
- [ ] Every claim backed by evidence or source
- [ ] Full names for all people/orgs/products (never "a company")
- [ ] Covers questions competitors don't (gap filling)
- [ ] Includes at least one checklist, template, or decision framework
- [ ] No filler, consistent terminology

## CTAs
- [ ] CTA at TOP of article (after intro)
- [ ] CTA at BOTTOM of article (before/after FAQ)
- [ ] Text above CTA adapted to article topic (not generic)

## Links
- [ ] Internal links: 2-5 to other blog articles, descriptive anchor text
- [ ] External links: 2-3 to authoritative sources backing specific claims

## Images (minimum 2-3)
- [ ] Hero image (generated or relevant visual)
- [ ] Screenshots of your product where relevant
- [ ] Competitor screenshots if doing a comparison
- [ ] Infographics: cost comparison, feature comparison, workflow
- [ ] Alt text optimized for SEO on every image
- [ ] Format: WebP, lazy loading

## FAQ & Featured Snippets
- [ ] ≥3 FAQ questions
- [ ] FAQ answers 40-60 words (snippet-optimized)
- [ ] Use definition/list/table/how-to formats where applicable

## Technical SEO
- [ ] Schema markup: Article + FAQ JSON-LD (for rich snippets)
- [ ] Core Web Vitals awareness (image optimization, lazy loading)
- [ ] Canonical URL set
- [ ] Mobile-first structure (short paragraphs, no wide tables)
- [ ] Open Graph tags
- [ ] Target keyword in: title, H1, first paragraph, meta description, at least one H2
- [ ] Secondary keywords naturally distributed across H2/H3

## SEO Self-Score (target ≥8/10)
- [ ] Title ✓ | Meta ✓ | H1 ✓ | Keyword placement ✓ | H2s ✓
- [ ] Internal links ✓ | External links ✓ | FAQ ✓ | Readability ✓ | Word count ✓

## Quality
- [ ] Fact-checked: all stats, pricing, and claims verified
- [ ] AI-tone check: no filler phrases, buzzwords, or template patterns
- [ ] Reads like a practitioner wrote it, not a content mill
- [ ] Product mentions feel natural and earned
