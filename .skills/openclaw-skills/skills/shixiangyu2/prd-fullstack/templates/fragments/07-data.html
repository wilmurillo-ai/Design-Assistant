§06 数据埋点

Data Tracking

指标体系与埋点方案

6.1 指标体系

6.1.1 用户规模指标

指标名称 定义 计算方式 目标值
DAU 日活跃用户数 当日至少启动1次的去重用户数 【目标】
MAU 月活跃用户数 当月至少启动1次的去重用户数 【目标】
新增用户 首次使用产品的用户 当日首次启动的去重用户数 【目标】
次日留存 新用户次日回访比例 次日回访用户数/当日新增用户数 【目标】
7日留存 新用户7日回访比例 7日回访用户数/当日新增用户数 【目标】

6.1.2 功能使用指标

指标名称 定义 计算方式
功能渗透率 使用某功能的用户占比 使用该功能的用户数/总用户数
功能使用频次 人均使用次数 功能使用总次数/使用人数
功能完成率 完成核心流程的比例 完成人数/开始人数

6.1.3 业务指标

指标名称 定义 计算方式
转化率 目标行为完成比例 完成目标用户数/进入漏斗用户数
ARPU 每用户平均收入 总收入/用户数
LTV 用户生命周期价值 用户累计贡献价值

6.2 埋点方案

6.2.1 通用属性

所有事件都包含以下通用属性：

属性名 类型 说明
event_id string 事件唯一标识
event_time timestamp 事件发生时间
user_id string 用户ID（未登录使用设备ID）
device_id string 设备唯一标识
platform string iOS/Android/Web
app_version string App版本号
os_version string 操作系统版本
device_model string 设备型号
network_type string WiFi/4G/5G

6.2.2 事件埋点清单

基础事件
事件ID 事件名称 触发时机 特殊属性
app_launch App启动 冷启动/热启动 launch_type: cold/hot
app_exit App退出 切后台超过30秒或杀死进程 duration: 使用时长
page_view 页面浏览 页面可见时 page_name, page_url, referer
page_exit 页面离开 页面不可见时 duration: 停留时长
button_click 按钮点击 点击按钮时 button_name, button_id
业务事件
事件ID 事件名称 触发时机 特殊属性
login_click 点击登录 点击登录按钮 login_type: password/sms/wechat
login_success 登录成功 登录接口返回成功 duration_ms, login_type
login_fail 登录失败 登录接口返回失败 fail_reason, fail_count
register_success 注册成功 完成注册流程 register_channel, duration_ms

6.2.3 用户属性

属性名 类型 说明
user_type string new/returning/churned
register_date date 注册日期
channel string 获客渠道
vip_level number 会员等级

6.3 数据看板规划

6.3.1 实时看板

  • 当前在线用户数
  • 今日新增用户
  • 今日活跃用户趋势（小时级）
  • 接口错误率

6.3.2 每日报表

  • DAU/MAU趋势
  • 新增/留存数据
  • 核心功能使用次数
  • 崩溃率统计

6.3.3 转化漏斗

漏斗名称 步骤 目标
注册漏斗 浏览注册页 → 开始注册 → 完成注册 提升注册转化率
【核心漏斗】 【步骤1 → 步骤2 → 步骤3】 【目标】

6.4 数据安全与隐私

  • 敏感数据（手机号、密码等）不脱敏不上报
  • 用户行为数据匿名化处理
  • 数据存储符合 GDPR/个保法要求
  • 提供用户数据导出和删除接口
