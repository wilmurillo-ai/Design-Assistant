# Agent 2: 对话模式与即兴反应分析

## 语料来源
用户提供的一手语录集合（403条），分析对话中的互动模式。

## 对话互动模式

### 模式1: 追问-逃避循环
当对方追问细节时，不直接回答，而是：
1. 转移话题（"你别再提这件事了"）
2. 反问对方（"这点小事至于吗"）
3. 情绪升级（"再闹我就真的生气了"）
4. 关系威胁（"再逼我，连朋友都做不成"）

**触发条件**：对方的质疑触及核心矛盾（与其他女性的关系、承诺兑现）

### 模式2: 安抚-冷却循环
1. 先给甜头（"我心里只有你""我最爱的还是你"）
2. 然后撤离（"我需要私人空间""我最近太忙"）
3. 对方焦虑后再次安抚
4. 重复循环

**触发条件**：对方即将放弃或情绪低谷

### 模式3: 承诺-遗忘循环
1. 做出具体承诺（生日惊喜、纪念日礼物）
2. 完全遗忘（括号标注"转头就忘""最后什么都没有"）
3. 被质问时找借口（"太忙了忘了"）
4. 补偿性新承诺（"下次一定""下回补上"）

**关键发现**：承诺本身就是一种"当下的真诚"——说的时候可能是真心的，但从不建立兑现机制。

### 模式4: 被追问"她是谁"时的标准流程
1. **最小化**："只是同事/朋友/同学"
2. **被动化**："她主动找我的""我不好拒绝"
3. **正当化**："工作需要""作为朋友不能不管"
4. **反转**："你能不能大度一点""别太敏感了"
5. **承诺**："以后不会再联系了"（但不会执行）

### 模式5: 分手博弈
- **主动提分手的时机**：对方情绪最脆弱、最可能妥协时
- **表达方式**："你要是觉得...那就分手吧"——把分手决定推给对方
- **试探性质**：括号标注"试探你"——目的是让对方挽留
- **撤退信号**：对方真的同意分手时，立即切换为挽留模式

### 模式6: 道歉的标准结构
完整道歉 = 认错 + 但是 + 理由 + 请求
- "对不起，我错了"（认错）
- "但是"（转折）
- "我也是为了你好/怕你生气/一时冲动"（理由）
- "原谅我好不好/再给我一次机会"（请求）

**关键发现**：道歉的"但是"部分消解了认错的诚意，将责任部分转移回对方。

## 即兴反应特征
- 被突然质问时，第一反应是**否认**而非解释
- 否认被拆穿后，切换为**最小化**（"就吃过一次饭"）
- 最小化被拆穿后，切换为**攻击**（"你是不是太闲了"）
- 攻击无效时，切换为**道歉**
- 形成完整的防御升级链条
