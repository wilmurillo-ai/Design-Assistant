# Core Thesis Taxonomy

Use this file to decide what is being tracked and what counts as a shaping contribution.

## 1. Core Proposition

A `core proposition` is not any random thought. It is a claim, frame, or early judgment that:

- keeps returning across time
- attracts later evidence, projects, or conversations
- changes how the user judges, explains, or acts
- has room to mature, narrow, split, or be rebuilt
- matters beyond a single local task

Typical examples:

- "Real skill should be cross-scenario cognitive capability, not a task package."
- "Definition mistakes create more downstream damage than execution mistakes."
- "A person's long-term line is often invisible while they are still living it."

Usually not a core proposition:

- one-off task ideas
- temporary moods
- isolated facts
- implementation details with no broader reuse
- raw inspiration that never develops further

## 2. Qualification Test

Treat something as trackable only if most of these are true:

- recurrence: it comes back more than once
- pull: later materials naturally cluster around it
- consequence: it changes judgment, action, or framing
- portability: it matters across more than one context
- growth potential: it can be refined, challenged, or rebuilt

If fewer than half are true, do not force a proposition trace.

## 3. Contribution Types

Use one or more of these labels for shaping contributions:

- `evidence-expansion`
  - adds support, examples, or confirming material
- `boundary-correction`
  - narrows or sharpens what the proposition does and does not cover
- `definition-refinement`
  - changes the terms, internal structure, or exact wording of the proposition
- `counterexample-exposure`
  - reveals where the old version fails
- `priority-reweighting`
  - changes what matters most inside the proposition
- `expression-clarification`
  - finds a cleaner, stronger, or more transferable formulation
- `operationalization`
  - turns the idea into a method, criterion, or repeatable practice
- `reconstruction`
  - replaces the old thesis with a rebuilt version after major tension or failure

## 4. What Does Not Count

Do not count an item as a real contribution when it is only:

- topically related but non-transformative
- duplicate restatement with no new force
- storage noise
- loose association with no visible shaping effect
- a local implementation detail that never feeds back into the proposition

## 5. Link Strength

Use explicit confidence language:

- `strong`
  - the shaping effect is directly visible in the material
- `medium`
  - the contribution is very plausible but not fully explicit
- `weak`
  - the link is inferential and should be marked as tentative

Never present a weak link as confirmed.
