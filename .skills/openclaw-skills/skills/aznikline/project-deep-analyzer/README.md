# Project Deep Analyzer Skill

A structured methodology for deep auditing and understanding software projects. Based on a set of core questions to analyze project boundaries, concepts, architecture, algorithms, and technical decisions.

## Features

- **System Boundary Analysis**: Goals, platforms, interfaces, and tech stack.
- **Concept System**: Key terms, definitions, and code mapping.
- **Module Architecture**: Physical and logical structure, dependencies, and code metrics.
- **Core Algorithms**: Identification, performance, and code location.
- **Technical Selection**: Evaluation of design choices and self-developed parts.
- **Error Handling**: Structured debugging and trace analysis.

## How to use in Trae

This skill is designed to be invoked when you need a deep understanding of a project or when troubleshooting complex issues.
