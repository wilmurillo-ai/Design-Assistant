# EML to SQLite Indexer Skill (V7.0.0 - 管理與匯出強化版)

此技能可將指定目錄下的 EML 郵件檔案索引至 SQLite 資料庫，並提供功能豐富的網頁介面進行查詢與管理。具備自動去重、IP 存取控制、Excel 匯出、以及可透過網頁設定的 **JSON 格式定時備份系統**。

## 功能特點

- **高效索引**：使用 MD5 指紋技術自動去重，確保同一封郵件不會重複匯入。支援百萬級郵件數據處理。
- **關鍵資訊提取**：自動解析並儲存郵件的寄件者、收件者、主旨、內文及寄發時間。
- **網頁查詢介面**：提供基於 Flask 的網頁介面，支援：
    - **進階搜尋**：關鍵字 (主旨/內文，不分大小寫)、寄件者 (模糊比對)、收件者 (模糊比對)、日期區間篩選。
    - **Excel 匯出**：將搜尋結果匯出為 Excel 相容的 CSV 檔案，包含**原始檔案路徑**。
    - **檔案刪除**：可從網頁直接刪除特定郵件，同時移除**資料庫記錄**與**磁碟實體檔案** (僅限管理員)。
    - **分頁顯示**：優化大量數據載入，避免網頁卡頓。
- **IP 存取控制**：可設定允許存取網頁的 IP 白名單，增強安全性。預設僅允許 `localhost` 及 `127.0.0.1` 存取。
- **網頁設定定時備份**：
    - **定時模式**：可透過網頁設定「每隔幾天」以及「在幾點」執行自動備份。
    - **JSON 格式**：備份檔案將郵件數據匯出為結構化 JSON 並壓縮為 ZIP，檔名格式為 `eml-indexer_YYYYMMDD.zip`。
    - **自動循環覆蓋**：系統會自動保留設定的份數，並刪除最舊的備份。
    - **手動管理**：透過網頁介面一鍵下載 JSON 備份 ZIP，或上傳 ZIP 進行資料還原。
- **管理員專屬介面**：當從 `localhost` 或 `127.0.0.1` 存取時，網頁會顯示「系統設定」分頁，提供 IP 管理、備份設定與刪除功能。

## 安裝與部署

### 1. 環境要求
- Python 3.8+ (建議使用最新穩定版)
- 建議作業系統：Windows, Linux

### 2. 依賴項安裝
請確保您的 Python 環境已安裝以下套件：
```bash
pip install -r requirements.txt
# 或手動安裝：
pip install Flask tqdm
```

### 3. 檔案結構
```
eml_indexer/
├── app.py              # 網頁應用程式 (Flask) - 包含定時備份執行緒
├── indexer.py          # EML 索引核心腳本
├── requirements.txt    # Python 依賴項清單
├── SKILL.md            # 技能說明文件 (英文)
├── config.json         # 運行時配置 (包含允許的 IP、備份週期、備份時間、保留份數)
├── emails.db           # SQLite 資料庫檔案 (執行 indexer.py 後產生)
├── backups/            # JSON 備份存放目錄 (自動生成)
└── templates/
    ├── detail.html     # 郵件詳情頁面模板
    └── index.html      # 郵件查詢與管理主頁面模板
└── references/
    └── SKILL-TW.md     # 繁體中文版技能說明文件
```

## 使用方法

### 1. 索引 EML 郵件
執行 `indexer.py` 將指定目錄下的 EML 檔案匯入資料庫。每次執行時，它會自動跳過已索引的郵件。
```bash
python indexer.py <EML目錄路徑> <資料庫路徑(預設為emails.db)>
```

### 2. 啟動網頁查詢介面 (含定時備份)
執行 `app.py` 啟動 Flask 網頁伺服器。啟動後，背景定時備份執行緒也會同步啟動。
```bash
python app.py
```
啟動後，請在瀏覽器訪問 `http://localhost:5000`。

### 3. 管理系統設定
從 `localhost` 或 `127.0.0.1` 訪問網頁時，點擊「⚙️ 系統設定」分頁，您可以：
- **IP 管理**：新增或移除允許存取的 IP 位址。
- **備份設定**：設定「備份週期 (天)」、「備份時間 (時)」與「保留份數」。
- **手動備份**：點擊「建立並下載備份 (ZIP)」立即產生 JSON 備份。
- **手動還原**：上傳 JSON 備份 ZIP 檔案進行還原。
