# Contributing

Thanks for helping improve `comfyui-skill-public`.

## Contribution principles

- Keep the skill install-agnostic and portable.
- Prefer discovery-first guidance over hardcoded assumptions.
- Do not add personal model inventories, local filesystem paths, or private environment details.
- Keep changes focused and minimal.

## How to contribute

1. Fork the repository and create a branch.
2. Make your changes with clear, focused commits.
3. Update docs if behavior or usage expectations change.
4. Open a pull request describing:
   - What changed
   - Why it changed
   - Any compatibility impact

## Content scope

Good contributions include:

- Better onboarding and troubleshooting guidance
- Safer API and validation patterns
- More portable examples for model/workflow families

Avoid contributions that:

- Assume a fixed local ComfyUI path or model inventory
- Include secrets, personal data, or private infrastructure details
