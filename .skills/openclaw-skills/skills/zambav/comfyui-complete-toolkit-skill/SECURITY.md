# Security Policy

## Supported versions

This repository is currently maintained on the default branch. Security fixes are applied there first.

## Reporting a vulnerability

If you discover a security issue, please do **not** open a public issue with exploit details.

Instead:

1. Open a GitHub issue with minimal detail and clearly mark it as a security concern, **or**
2. Contact the maintainer directly through private channels if available.

Please include:

- A clear description of the vulnerability
- Reproduction steps
- Impact assessment
- Suggested remediation (if known)

We will acknowledge reports as quickly as possible and work to validate and remediate confirmed issues.

## Security expectations for contributors

- Never commit secrets, API keys, or private credentials.
- Avoid machine-specific absolute paths in reusable guidance.
- Keep install-specific configuration user-owned and out of shared defaults.
