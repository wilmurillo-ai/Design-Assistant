# 自然语言解析示例

## 查询类型

### 1. 简单查询
- "帮我查一下北京到上海的机票"
- "北京到上海机票"

### 2. 日期查询
- "3月16日北京到成都的机票"
- "明天上海到广州的航班"
- "下周一深圳到北京的机票"

### 3. 价格监控查询

## 解析规则

### 1. 城市识别规则
1. 优先识别明确的城市名
2. 支持"到"、"至"、"飞往"等连接词
3. 支持"从...到..."格式

### 2. 日期识别规则
1. 绝对日期：YYYY-MM-DD、MM月DD日
2. 相对日期：今天、明天、后天
3. 周几：周一、周二...
4. 特殊时间：周末、节假日

### 3. 动作识别规则
1. 查询类关键词：查、查询、找
2. 监控类关键词：监控、订阅、关注
3. 管理类关键词：取消、删除、查看

### 4. 默认值规则
1. 日期未指定：默认为当天
2. 类型未指定：默认为flight
3. 舱位未指定：默认为economy

## 常见问题

### Q: 解析失败怎么办？
A: 提示用户重新输入或使用标准格式：
- "出发城市 到 到达城市 日期 机票"
- 如："北京到上海明天机票"

### Q: 如何支持更多城市？
A: 在 `natural_language_parser.py` 中扩展城市字典

### Q: 如何处理模糊查询？
A: 提供交互式确认，如：
```
用户: "查一下机票"
系统: "请问从哪里飞到哪里？"
```
