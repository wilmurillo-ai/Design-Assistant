# Interview Question Bank

This reference file contains categorized interview questions for various roles and industries.

---

## Technical Questions

### Software Engineering

**Core Concepts:**
- "Explain the difference between a stack and a queue. When would you use each?"
- "What is the difference between process and thread?"
- "Explain the concept of Big O notation and why it matters"
- "What's the difference between SQL and NoSQL databases?"

**System Design:**
- "How would you design a URL shortener?"
- "Design a real-time chat application"
- "How would you scale a web application to millions of users?"

**Coding/Algorithms:**
- "Write a function to reverse a linked list"
- "How would you detect a cycle in a graph?"
- "Implement a caching mechanism for an API"

**Debugging:**
- "Tell me about a challenging bug you fixed"
- "How do you approach debugging a complex issue?"
- "Describe your process for troubleshooting production issues"

### Data Science / Analytics

**Technical Skills:**
- "Explain overfitting and how you prevent it"
- "What's the difference between supervised and unsupervised learning?"
- "How do you handle missing data in your analysis?"

**Problem-Solving:**
- "How would you analyze user churn for a subscription service?"
- "Design an A/B test to improve conversion rates"
- "How would you detect fraudulent transactions?"

### DevOps / Engineering

**Infrastructure:**
- "Explain CI/CD and why it's important"
- "How do you manage configuration across environments?"
- "Describe your experience with containerization (Docker/Kubernetes)"

**Monitoring & Reliability:**
- "How do you set up monitoring for a distributed system?"
- "What's your approach to incident response?"
- "How do you ensure high availability?"

## Tips for Using This Question Bank

1. **Tailor to the Role**: Select questions relevant to the specific position
2. **Adjust Difficulty**: Match question complexity to the candidate's experience level
3. **Follow Up**: Use follow-up questions to dig deeper
4. **Take Notes**: Record key points for later evaluation
5. **Be Flexible**: Adapt questions based on the conversation flow
6. **Stay Legal**: Avoid questions about protected characteristics (age, religion, etc.)
