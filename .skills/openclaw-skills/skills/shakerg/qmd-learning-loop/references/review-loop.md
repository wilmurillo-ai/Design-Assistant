# Review Loop

## Natural review points

Review captured learnings at these moments:
- after finishing a substantial task
- when the same failure recurs
- before editing durable guidance, principles, or decision docs
- during periodic memory maintenance

## Review sequence

1. Ask whether the captured item still matters.
2. Check whether a durable rule already exists.
3. Decide whether to:
   - leave in chronology only
   - log in an incident or feature-request file
   - promote into a runbook, decision, or guidance file
4. If promoted, keep the promoted rule concise and prevention-oriented.

## Prevention-oriented writing

Prefer:
- "Always specify the repo explicitly outside a checkout"
- "Keep generated artifacts out of indexed collections"

Avoid:
- long incident narratives copied into durable guidance
- verbose postmortems pasted into workflow or tooling files
