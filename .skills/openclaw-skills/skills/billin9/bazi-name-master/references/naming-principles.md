# Naming Principles

## Core Rule

BaZi and favorable-element analysis guide the naming direction, but they do not mechanically produce the final name. Once the `病药` hierarchy is clear, final recommendations must first pass that directional gate before they are compared on clarity, sound, structure, and real-world usability.

## Name Length

- Ask for the user's preference: single-character, double-character, or no preference.
- If there is no preference, recommend a double-character given name by default.
- Treat single-character names as a constrained stylistic option rather than the default.

## Character Selection

Prefer characters that are:

- easy to read
- stable in pronunciation
- semantically clear
- visually balanced
- culturally resonant without being stale

Avoid characters that are:

- multi-pronunciation unless the user clearly accepts the tradeoff
- overly obscure
- too difficult to write
- semantically vague
- commonly mocked in everyday speech

## Gender Temperament

Keep the overall temperament clearly aligned with the user's preference:

- masculine: upright, clear, steady, expansive, refined
- feminine: luminous, graceful, clear, spirited, elegant
- neutral only if the user explicitly asks for it

Do not let "modern" drift into muddled or characterless naming.

## Trend Avoidance

- Repeated high-frequency characters are warning signs, not automatic bans.
- Build an avoid-list from current online sampling.
- Prefer names that feel fresh without becoming strange for the sake of novelty.

## Explanation Standard

Every candidate name should survive four checks:

1. `命理`
Can you explain how the name supports the favorable-element strategy?

2. `出处`
Can you point to a real source line and explain the semantic bridge?

3. `音形义`
Does it sound good, look balanced, and mean something coherent?

4. `使用场景`
Is it workable in school, documents, daily speech, and long-term identity use?

## Candidate Name Expression

When presenting each name, do not default to a dry `音形义解析` label.

Prefer:

- `听感/情绪`
- `社交示范`

### 听感/情绪

- Describe the tonal rhythm, calling feel, emotional color, and overall aura of the name.
- The sentence should feel vivid and human, not like textbook phonology.
- It is good to mention tonal shape when useful, but always turn it into a felt impression.

Good style examples:

- `音律轻灵，叫起来像清泉碰玉，有一种干净而灵动的明澈感。`
- `仄平相映，读起来端庄含蓄，像一位有书卷气的少女缓缓走来。`
- `发音清脆，自带一点清冷与智性，不甜不腻，反而更显高级。`

### 社交示范

- Add a natural sentence that could plausibly be used in朋友圈、满月文案、家庭分享文案.
- The copy should match the name's temperament.
- It should feel personal and warm, not like a generic ad line.

Good style examples:

- `山间清溪，掌上明珠。我们的小溪玥，欢迎光临地球。`
- `把生活过成诗，听风起雨落的泠泠之音。`
- `愿你一生清澈明朗，像初雪落在春光里。`

## Analysis Expression

The BaZi section should not read like a neutral lab report.

- State whether the day master is strong or weak in a clear, high-conviction sentence.
- Explain the core structural tension as `病`, then explain the naming remedy as `药`.
- When the chart supports it, say directly that a certain element is the first priority, the second priority, or should be avoided.
- Do not settle for `喜金水，忌木火` as a thin conclusion. Explain the causal chain.

Prefer this shape:

1. `日元旺衰`
Describe why the day master is strong or weak based on month令,泄耗,生扶, and major branch interactions.

2. `核心病药`
Point out what is damaging the structure most and what kind of naming补益 should repair it.

3. `性格潜质`
Translate the structure into temperament, expression style, talent profile, and likely social aura in a readable way.

## Five-Element Strategy Expression

Prefer a decisive hierarchy:

- `首选补X`
- `次选补X`
- `绝对避讳`
- optional `取名策略说明`

This section should sound like an actionable prescription, not a loose summary.

## Supplementary Systems

Three talents and five grids may be mentioned briefly as supplementary commentary if helpful, but they must not override the BaZi-centered naming logic.
