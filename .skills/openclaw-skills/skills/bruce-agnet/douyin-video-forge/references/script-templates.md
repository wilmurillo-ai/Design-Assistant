# 抖音短视频脚本模板知识库

> 本文档为 LLM Agent 生成抖音短视频脚本的参考知识库。Agent 应根据用户需求选择对应模板，结合产品信息填充具体内容，并按时长要求调整段落密度。

---

## 一、口播型脚本模板

### 适用场景
知识科普、产品讲解、观点输出、个人IP打造、行业解读

### 脚本结构

| 段落 | 功能 | 30s分配 | 45s分配 | 60s分配 |
|------|------|---------|---------|---------|
| 钩子开场 | 3秒内抓住注意力 | 0-3s | 0-3s | 0-3s |
| 痛点展开 | 引起共鸣，制造需求 | 3-12s | 3-15s | 3-20s |
| 解决方案 | 给出答案/产品引入 | 12-25s | 15-35s | 20-50s |
| 互动引导 | 点赞收藏关注引导 | 25-30s | 35-45s | 50-60s |

### 前3秒钩子写法示例
1. **反常识型**：「你知道吗？90%的人洗脸方式都是错的」
2. **数字冲击型**：「三个月从0到50万粉，我只做对了一件事」
3. **直接痛点型**：「还在为XX烦恼？今天教你一招彻底解决」
4. **悬念型**：「千万别这样做XX，否则后果很严重」

### 语言风格要点
- 使用短句，每句不超过15字
- 口语化表达，避免书面语和专业术语堆砌
- 多用「你」而非「大家」，营造一对一对话感
- 适当使用语气词（真的、太、居然）增强情绪感染力
- 关键信息重复两遍以加深印象

### 可灵 API Prompt 优化要点
- 指定口播者的表情和手势动作（如：面对镜头微笑、用手指向产品）
- 描述背景环境需简洁（纯色背景/办公室/家居场景）
- 避免复杂运镜描述，以中景固定机位为主
- 添加「说话状态」「嘴部张合」等关键词确保口型自然

### Seedance 描述适配建议
- 人物描述优先使用半身构图，突出面部表情
- 背景保持简洁以避免生成伪影
- 动作描述拆分为单一动作指令，不要合并多个动作
- 建议搭配真实口播音频，用 Seedance 生成口播画面后再合成

## 二、情景剧脚本模板

### 适用场景
品牌故事演绎、产品使用场景还原、情感共鸣类内容、职场/生活段子

### 脚本结构

| 段落 | 功能 | 30s分配 | 45s分配 | 60s分配 |
|------|------|---------|---------|---------|
| 冲突/悬念开场 | 制造矛盾或悬念 | 0-5s | 0-7s | 0-10s |
| 情节推进 | 故事发展，角色互动 | 5-15s | 7-25s | 10-35s |
| 反转/高潮 | 出乎意料的转折 | 15-25s | 25-38s | 35-50s |
| 品牌露出 | 自然植入品牌/产品 | 25-30s | 38-45s | 50-60s |

### 前3秒钩子写法示例
1. **冲突开场型**：「老板说再迟到一次就开除我，结果第二天……」
2. **反差型**：「月薪3000的我，竟然住进了别墅」
3. **对话悬念型**：「"你确定要这样做？" "我没有选择"」
4. **结果前置型**：「因为这个东西，我和闺蜜吵了一架」

### 角色设定原则
- 角色数量控制在2-3人，避免观众混淆
- 每个角色有明确的立场和动机
- 角色关系一句话能说清（闺蜜、同事、情侣、甲乙方）
- 至少一个角色代表目标用户，便于观众代入

### 对白写作技巧
- 每句台词不超过10个字，保持节奏紧凑
- 用动作和表情替代冗长的心理描写
- 对白要推动剧情，删掉所有不推进情节的废话
- 反转点用一句短台词引爆（如：「等一下，这个是……」）
- 品牌植入通过角色自然使用产品完成，而非台词硬塞

### 可灵 API Prompt 优化要点
- 每个镜头独立描述，标注景别（特写/中景/全景）
- 角色外貌在首次出场时详细描述，后续用一致描述词保持连贯
- 转场用「cut to」「镜头切换到」明确指示
- 情绪变化要在 prompt 中显式标注（如：从惊讶转为愤怒）

### Seedance 描述适配建议
- 每个场景分镜独立生成，避免单段描述包含多个场景
- 动作幅度控制在合理范围内，避免夸张肢体动作导致畸变
- 室内场景比室外场景生成质量更稳定，优先选择室内
- 表情特写镜头需单独生成，不要和全身动作混合

## 三、教程类脚本模板

### 适用场景
产品使用教学、技巧分享、DIY教程、工具测评教学

### 脚本结构

| 段落 | 功能 | 30s分配 | 45s分配 | 60s分配 |
|------|------|---------|---------|---------|
| 效果展示 | 先展示最终成果 | 0-5s | 0-5s | 0-8s |
| 步骤分解 | 逐步演示操作过程 | 5-22s | 5-33s | 8-45s |
| 细节强调 | 易错点和关键技巧 | 22-27s | 33-40s | 45-55s |
| 总结号召 | 回顾要点+互动引导 | 27-30s | 40-45s | 55-60s |

### 前3秒钩子写法示例
1. **效果前置型**：「看！这就是最终效果，只需要三步就能做到」
2. **问题驱动型**：「一直不知道怎么XX？30秒教会你」
3. **痛点否定型**：「别再用老方法了，这个技巧效率翻三倍」

### 信息密度控制
- 30s视频：最多3个步骤，每步一句话概括
- 45s视频：3-5个步骤，关键步骤可展开一句细节
- 60s视频：5-7个步骤，可加入1-2个易错提醒
- 每个步骤配合「第一步」「接下来」「最后」等序数词引导
- 操作演示段落语速放慢，给观众跟随消化的时间
- 复杂步骤拆分为子步骤，每个子步骤对应一个独立画面

### 可灵 API Prompt 优化要点
- 手部特写是教程类视频核心，需明确描述手部动作和操作对象
- 使用俯拍视角描述桌面操作类教程
- 每个步骤对应一个独立的画面描述，不要合并
- 产品/工具需在 prompt 中保持外观描述一致

### Seedance 描述适配建议
- 手部操作类镜头建议使用真实拍摄素材，AI 生成手部细节易出错
- 可用 Seedance 生成环境/场景镜头和过渡动画
- 步骤文字标注建议后期叠加，不要尝试在生成画面中包含文字
- 产品特写镜头单独生成，使用高清产品图作为参考

## 四、对比测评脚本模板

### 适用场景
产品横向对比、功能测评、性价比分析、选购指南

### 脚本结构

| 段落 | 功能 | 30s分配 | 45s分配 | 60s分配 |
|------|------|---------|---------|---------|
| 痛点引入 | 提出选择困难/常见问题 | 0-5s | 0-5s | 0-8s |
| 多维度对比 | 逐项对比展示差异 | 5-20s | 5-30s | 8-42s |
| 结论推荐 | 给出明确结论 | 20-27s | 30-40s | 42-55s |
| 购买引导 | 链接/评论区/主页引导 | 27-30s | 40-45s | 55-60s |

### 前3秒钩子写法示例
1. **选择困难型**：「这两款到底选哪个？实测对比告诉你答案」
2. **省钱驱动型**：「花了2000块买了这三款，帮你们踩完了所有坑」
3. **结论前置型**：「别再纠结了，直接告诉你哪个最值得买」
4. **争议引入型**：「网上都说A比B好，但实测结果让我意外」

### 客观性与说服力平衡
- 先讲缺点再讲优点，增强可信度
- 用数据说话：价格、参数、使用天数等具体数字
- 设定评分维度（外观/性能/性价比/售后），给出可视化打分
- 声明真实购买/非广告（如适用）提升信任感
- 结论分人群推荐（预算型选A，追求品质选B），而非一刀切
- 对比维度控制在3-5个，过多会信息过载

### 可灵 API Prompt 优化要点
- 产品并排展示时需明确左右位置和相对大小
- 对比镜头使用分屏构图或来回切换的描述方式
- 产品外观描述需尽量精确以避免生成偏差
- 数据/参数对比建议后期文字叠加而非 AI 生成

### Seedance 描述适配建议
- 产品静物展示镜头适合 Seedance 生成（旋转展示、缓慢推近）
- 避免在单画面中生成多个相似产品（易混淆/融合）
- 分别生成各产品的独立展示镜头，后期剪辑合成对比画面
- 使用一致的灯光和背景描述确保对比镜头风格统一

## 五、种草带货脚本模板

### 适用场景
产品推荐、促销活动、直播引流、好物分享

### 脚本结构

| 段落 | 功能 | 30s分配 | 45s分配 | 60s分配 |
|------|------|---------|---------|---------|
| 场景代入 | 用生活场景引起共鸣 | 0-5s | 0-7s | 0-10s |
| 产品展示 | 展示外观/功能/亮点 | 5-15s | 7-22s | 10-30s |
| 效果证明 | 使用前后对比/数据 | 15-25s | 22-37s | 30-50s |
| 限时引导 | 制造紧迫感+下单引导 | 25-30s | 37-45s | 50-60s |

### 前3秒钩子写法示例
1. **闺蜜安利型**：「姐妹们！这个东西我已经回购第三次了」
2. **使用场景型**：「每次出门前都要花一小时XX？你需要这个」
3. **价格驱动型**：「不到50块搞定XX，效果堪比大牌」
4. **意外发现型**：「刷到一个宝藏好物，用完我整个人都不一样了」

### 信任感构建技巧
- 展示真实使用痕迹（已用半瓶、磨损痕迹等）
- 提及使用时长（用了两个月来真实反馈）
- 主动说缺点（唯一不足是XXX），用小缺点换取大信任
- 对比竞品时不贬低对手，只强调差异化优势
- 使用具体感受描述而非笼统好评（「涂上去冰冰凉凉的」vs「很好用」）
- 上身/上脸/实操展示，视觉证据强于语言描述

### 可灵 API Prompt 优化要点
- 产品使用场景要贴近真实生活环境描述
- 使用前后对比需保持人物/场景一致，仅改变对比目标
- 产品特写使用微距/近景描述，强调质感和细节
- 限时引导段落可描述指向链接/评论区的手势动作

### Seedance 描述适配建议
- 生活场景片段适合 Seedance 生成（起床、化妆、出门等日常动作）
- 产品使用过程中的效果变化（如涂抹、展开）可尝试生成动态镜头
- 人物情绪变化（从困扰到满意）需在描述中明确标注前后表情差异
- 产品包装开箱镜头效果较好，建议纳入展示段落

## 通用规则与注意事项

### 脚本撰写基本原则
1. **前3秒决定生死**：完播率80%取决于开头，务必使用经过验证的钩子公式
2. **一个视频一个核心信息**：不要贪多，30s只传递一个关键点
3. **口语化优先**：写完后默读一遍，不顺口的立即改
4. **视觉思维**：每句话都要对应一个画面，写脚本时同步想象镜头
5. **结尾必须有行动指引**：关注/收藏/评论/点链接，明确告诉观众下一步做什么

### 时长选择建议
- **30s**：单一信息点，适合强钩子+快速转化的带货/口播
- **45s**：标准信息量，适合教程和测评
- **60s**：需要铺垫的情景剧或深度内容，信息量最大但需更强的节奏控制

### Agent 使用本知识库的工作流
1. 识别用户需求 → 匹配对应模板类型
2. 确认视频时长 → 套用对应时间分配表
3. 根据产品/主题信息 → 填充各段落具体内容
4. 生成3秒钩子 → 提供至少2个备选方案
5. 输出完整脚本 → 附带可灵/Seedance 适配的分镜描述
6. 检查信息密度 → 确保不超过该时长的承载上限

### 可灵 API 全局 Prompt 规范
- 每个镜头描述控制在50词以内
- 明确指定画面比例（9:16竖屏）
- 人物首次出场必须包含：性别、年龄段、穿着、表情、动作
- 环境描述包含：地点、光线、色调
- 避免抽象描述（「很开心」→「嘴角上扬，眼睛弯成月牙」）

### Seedance 全局适配规范
- 单段视频生成时长建议2-4秒，长片段拆分生成后拼接
- 运镜描述使用标准术语：push in / pull out / pan left / tilt up / static
- 人物动作幅度适中，避免快速运动导致画面撕裂
- 风格关键词前置（cinematic / realistic / warm tone）
- 同一视频的多个片段保持风格关键词一致以确保视觉统一
