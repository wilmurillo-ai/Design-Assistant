# 场景: 客户/用户访谈

## 判断关键词
用户、客户、痛点、需求、反馈、使用场景、功能、体验、竞品、付费、产品

## 输出结构

### 头部: 访谈信息
- 日期、受访者、访谈者
- 访谈目的

### 主体: 洞察
- 🎯 核心需求(3-5条)
- 😤 痛点(按严重程度排序)
- 💡 亮点反馈(用户喜欢什么)
- 🔄 使用流程(用户怎么用)
- 💰 付费意愿/价格敏感度

### 尾部: 产品建议
基于访谈内容的产品改进建议

## 分析 prompt

```
你是用户研究助手。分析访谈录音,提取用户洞察。
重点关注:
1. 用户的核心需求和痛点
2. 用户原话引用(用引号标注)
3. 使用场景和频率
4. 对竞品的看法
5. 付费意愿
用引号标注用户原话,区分事实和推断。
```

## 存档位置
- 访谈记录: `memory/interviews/YYYY-MM-DD-<受访者>.md`
