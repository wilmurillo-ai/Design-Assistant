# 场景: 工作会议

## 判断关键词
项目、排期、需求、上线、bug、迭代、开发、产品、设计、测试、deadline、优先级、跟进

## 输出结构

### 头部: 会议信息
- 日期时间
- 参与人员
- 会议主题

### 主体: 议题列表
每个议题包含:
- 📋 讨论内容摘要
- ✅ 达成共识/决策
- 👤 责任人
- 📅 截止时间

### 尾部: Action Items
- 编号列出所有待办事项
- 每项标注: 负责人 + 截止日期

### 附: 未决问题
需要后续跟进的问题

## 分析 prompt

```
你是会议纪要助手。分析对话记录,生成结构化会议纪要。
重点关注:
1. 讨论了哪些议题
2. 每个议题的决策结论
3. 谁负责什么,截止日期
4. 有哪些未解决的问题需要后续跟进
格式简洁,重点突出 action items。
```

## 存档位置
- 会议纪要: `memory/meetings/YYYY-MM-DD-<主题>.md`
