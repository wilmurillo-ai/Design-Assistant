# 故障排除

## 问题1：记录无法匹配成员

**症状**：说"记录张三..."但 AI 无法识别

**原因**：
- 姓名拼写与 `skill-config.yaml` 不一致
- 使用了别名但未配置

**解决**：
1. 检查 `skill-config.yaml` 中的姓名拼写
2. 确认别名配置正确
3. 使用全名而非昵称

**示例**：
```yaml
# skill-config.yaml
members:
  - id: member-001
    name: "张三"
    alias: "ZS"  # 别名
```

---

## 问题2：生成的报告缺少数据

**症状**：年终总结内容很少

**原因**：
- 记录数量不足
- frontmatter 格式不正确

**解决**：
1. 确认记录文件的 frontmatter 格式正确
2. 保持每天记录的习惯
3. 事件描述要具体

**正确格式**：
```markdown
---
timestamp: 2024-01-15T09:30:00
member-id: member-001
categories:
  - technical
rating: positive
---

## 原始记录
张三主动优化了用户查询接口...
```

## 问题3：隐私顾虑

**症状**：担心数据安全

**解决**：
1. 使用 member-XXX 编号代替真名
2. 真名映射存储在加密文件或密码管理器中
3. 网盘同步前手动加密敏感文件
4. 定期归档离职成员数据到 `archive/`

**加密方案**：
```bash
# macOS/Linux
zip -e team-memory.zip team-memory/

# Windows (使用 7-Zip)
7z a -p team-memory.zip team-memory\
```

## 问题4：追踪提醒不工作

**症状**：AI 没有主动追踪承诺

**原因**：
- 记录时没有明确说"承诺"
- AI 没有创建追踪项

**解决**：
1. 记录时明确使用"承诺"关键词
2. 手动创建追踪文件
3. 定期查询"有哪些承诺到期了"

**示例**：
```
记录 张三 今天 1on1 鼓励沟通改进，承诺下月分配云原生任务
```

---

## 问题5：文件丢失

**症状**：找不到之前的记录

**解决**：
1. 检查是否在正确的目录
2. 确认 Git/网盘同步正常
3. 使用搜索功能查找

**搜索命令**：
```bash
# 在 members 目录搜索
grep -r "张三" data/members/

# 搜索特定日期
grep -r "2024-01" data/members/
```

---

## 问题6：OpenCode 无法读取文件

**症状**：AI 说找不到文件

**原因**：
- 文件路径不正确
- 文件名编码问题

**解决**：
1. 使用纯英文文件名
2. 避免特殊字符
3. 路径使用 `/` 而非 `\`

**推荐命名**：
```
张三-时间轴.md  ✅
zhangsan-timeline.md  ✅
张三_timeline.md  ⚠️
张三@时间轴.md  ❌
```
