## LLM 检索流程

### 场景：准备1:1谈话

1. **LLM 读取蒸馏文件** → 快速了解张三是谁
   - 技术扎实、主动性强、沟通需改进
   - 关键事件：性能优化10倍、通宵完成紧急需求

2. **如果需要更多细节** → 读取时间轴原文
   - 查看完整观察笔记
   - 查看之前的对话记录

3. **生成谈话要点** → 基于蒸馏定位 + 原文细节

### 场景：年终总结

1. **LLM 读取蒸馏文件** → 汇总所有关键事件
2. **读取时间轴原文** → 补充具体数据和证据
3. **生成结构化总结** → 有数据支撑的评估

---

## 优势

| 维度 | 只有原文 | 双层架构 |
|------|----------|----------|
| LLM 首次理解 | 需要读取大量文本 | 快速定位关键信息 |
| 上下文占用 | 高 | 低（蒸馏文件小） |
| 细节追溯 | 全文搜索 | 锚点引用 |
| 检索速度 | 慢 | 快 |
