# 杂乱信息整理示例

## 示例1：时间混乱的信息

### ❌ 用户原始输入

"我做过一个春节活动效果挺好的，然后之前国庆也做过一个活动，还有平时在做数据分析。对了，还有一个和搜索相关的项目，也是做优化什么的。在字节待了半年左右吧，大概2023年底到2024年中。"

### ✅ 整理后

**识别到的时间线：**
- 2023.12 - 2024.06：字节跳动实习（约6个月）

**拆分出的核心项目（按时间倒序）：**

| 项目 | 时间 | 优先级 | 说明 |
|:---|:---|:---:|:---|
| 春节活动 | 2024.01-2024.02 | P0 | 有明确成果，重点展开 |
| 搜索优化项目 | 2024.02-2024.04？ | P1 | 需要确认具体时间 |
| 数据分析工作 | 贯穿实习期？ | P1/P2 | 需要确认是独立项目还是支持性工作 |
| 国庆活动 | 2023.09-2023.10 | P1 | 时间冲突！需要确认是否在字节前 |

**需要向用户确认的问题：**
1. 国庆活动是在字节做的，还是之前的经历？
2. 春节活动和搜索项目是并行的，还是有先后顺序？
3. 数据分析是某个项目的一部分，还是独立的日常工作？

## 示例2：多段经历混杂

### ❌ 用户原始输入

"我在B站做过用户增长，DAU提升了很多。然后也在腾讯待过，做产品经理，写过PRD文档，也做过数据分析，还参加过一些会议。后来去字节做策略产品，做了推荐策略，也做过一些增长活动，还有搜索相关的优化。对了，在学校还做过一个创业项目。"

### ✅ 整理后

**识别到的公司/组织：**
1. 字节跳动 - 策略产品
2. 腾讯 - 产品经理
3. B站 - 用户增长
4. 学校 - 创业项目

**按时间倒序排列：**

| 公司 | 岗位 | 识别到的项目 | 优先级 | 备注 |
|:---|:---|:---|:---:|:---|
| 字节 | 策略产品 | 推荐策略 | P0 | 核心项目 |
| 字节 | 策略产品 | 增长活动 | P1 | 重要项目 |
| 字节 | 策略产品 | 搜索优化 | P1 | 重要项目 |
| 腾讯 | 产品经理 | PRD撰写+需求落地 | P1 | 需要确认具体项目 |
| 腾讯 | 产品经理 | 数据分析 | P2 | 可能是支持性工作 |
| B站 | 用户增长 | 用户增长项目 | P0 | 有数据成果，重点展开 |
| 学校 | 创始人 | 创业项目 | P1 | 非实习经历，视情况保留 |

**过滤掉的信息：**
- ❌ "参加过一些会议" → 日常事务，删除
- ⚠️ "写过PRD文档" → 需要确认是哪个项目的一部分

## 示例3：大量无关信息

### ❌ 用户原始输入

"我每天早上9点上班，6点下班，中午休息1小时。我们组有5个人，leader人很好，经常请我们喝奶茶。我每天的工作包括写日报，每周要写周报，每月还有月报。我还帮同事订过外卖，组织过一次团建。在工作上，我做过一个AI项目，用ChatGPT做一些事情，效果还不错。还有一个数据分析的工作，给运营提供了一些支持。"

### ✅ 整理后

**识别到的核心信息：**
1. AI项目（使用ChatGPT）- P0 核心项目
2. 数据分析工作（支持运营）- P1 重要项目

**过滤掉的信息：**
| 信息 | 原因 |
|:---|:---|
| 上下班时间 | 与岗位能力无关 |
| 团队人数 | 非项目内容 |
| leader请奶茶 | 无关信息 |
| 日报/周报/月报 | 日常事务 |
| 订外卖、组织团建 | 无关信息 |

**需要向用户确认：**
1. AI项目具体做了什么？使用ChatGPT解决了什么问题？有什么成果？
2. 数据分析支持运营，具体分析了什么？输出了什么结论？有什么效果？

## 示例4：项目边界模糊

### ❌ 用户原始输入

"我在美团做商家运营，做了很多工作。包括商家的入驻审核，还有一些商家的培训，然后也做过一些活动让商家参与，还分析过商家的数据，给商家做过一些运营方案。有时候也处理商家的投诉和问题。"

### ✅ 整理后

**识别到的核心模块：**

| 模块 | 内容 | 优先级 | 处理方式 |
|:---|:---|:---:|:---|
| 商家入驻 | 审核、培训 | P1 | 合并为"商家入驻体系" |
| 商家活动 | 活动策划、商家参与 | P0 | 独立项目，重点展开 |
| 数据分析 | 商家数据分析 | P1 | 可以合并到活动或独立展示 |
| 运营方案 | 给商家的运营方案 | P2 | 可能是日常支持工作 |
| 投诉处理 | 客服/支持工作 | P3 | 删除或与岗位无关则简写 |

**关联合并建议：**
```
建议合并为2-3个核心项目：

项目1（P0）：商家增长活动策划
- 商家入驻审核与培训体系搭建
- 商家活动策划与执行
- 数据分析与效果评估

项目2（P1）：商家数据分析与运营支持
- 商家数据监控与分析
- 输出商家运营方案
- （投诉处理可删除或一句话带过）
```

## 示例5：多段实习经历混乱

### ❌ 用户原始输入

"第一段在小红书，做内容运营，负责创作者的对接，还有做一些活动。第二段在字节，做的是产品，不过也做了一些运营的工作，还有数据分析。然后之前还在腾讯实习过，也是做产品，做了一个增长的项目。对了，字节的时候还有一个AI相关的项目。"

### ✅ 整理后

**按时间倒序排列（假设）：**

| 时间 | 公司 | 岗位 | 项目 | 优先级 |
|:---|:---|:---|:---|:---:|
| 最近 | 字节 | 产品 | AI项目 | P0 |
| 最近 | 字节 | 产品 | 增长项目 | P0 |
| 最近 | 字节 | 产品 | 数据分析 | P1/P2 |
| 中间 | 小红书 | 内容运营 | 创作者运营 | P1 |
| 中间 | 小红书 | 内容运营 | 活动策划 | P1 |
| 最早 | 腾讯 | 产品 | 增长项目 | P0 |

**整理说明：**
1. 字节的产品岗位需要拆分3个项目
2. 小红书的内容运营可以合并为1-2个项目
3. 腾讯的增长项目与字节的合并展示或分别展示

**需要确认：**
1. 具体时间顺序
2. 字节的数据分析是独立项目还是其他项目的一部分
3. 字节的AI项目和增长项目是并行的还是有先后

## 整理原则总结

### 1. 去噪原则
- ✅ 保留：项目成果、职责、数据、技能
- ❌ 删除：日常事务、无关细节、主观感受

### 2. 拆分原则
- 一段实习经历 → 2-4个核心项目
- 每个项目有明确的边界和成果

### 3. 合并原则
- 同一业务的不同环节 → 合并展示
- 同一技术的不同应用 → 合并展示
- 支持性工作 → 合并到主项目或简写

### 4. 排序原则
- 按时间倒序（最近→最早）
- 同一公司内按重要性（P0→P1→P2）

### 5. 优先级标记
| 优先级 | 特征 | 简历处理方式 |
|:---|:---|:---|
| P0 | 有数据、独立负责、与岗位高度相关 | 重点展开，3-4个bullet point |
| P1 | 有参与、有学习价值 | 正常描述，2-3个bullet point |
| P2 | 参与支持、技能相关 | 简写，1-2个bullet point |
| P3 | 日常事务、无关 | 删除 |
