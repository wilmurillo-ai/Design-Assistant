# 小红书AI创作者识别 - 使用说明书

## 📌 项目简介

自动搜索小红书平台上的AI创作者（AI动画、AI短剧、AI视频等），根据预设规则筛选符合条件的创作者，并导出到Excel/JSON文件。

---

## 🚀 快速开始

### 1. 触发方式

在OpenClaw中输入以下任一命令：

```
/小红书创作者识别
/找AI创作者  
/xhs-finder
```

或自然语言：
- "帮我找小红书上的AI动画创作者"
- "搜索小红书的AI短剧创作者"

### 2. 运行流程

```
触发命令 → 打开Chrome → 登录小红书 → 搜索关键词 → 
筛选创作者 → 导出文件 → 展示结果
```

---

## 💾 断点续跑功能（重要）

由于OpenClaw有**5分钟超时限制**，本工具支持**断点续跑**功能：

### 工作原理

1. **每处理完一个关键词，自动保存进度** 到 `output/checkpoint.json`
2. **超时中断后，下次运行自动从上次进度继续**
3. **所有关键词处理完成后，自动导出最终结果**

### 使用示例

假设你要处理10个关键词，每次只能处理2个：

**第一次运行：**
```
[CHECKPOINT] 已加载进度: 0/10 个关键词完成
[CHECKPOINT] 待处理关键词: ['AI动画', 'AI短剧', 'AI视频', ...]
...运行中...
[CHECKPOINT] 已保存进度: 2/10  ← 超时中断
```

**第二次运行（同一会话ID）：**
```
[CHECKPOINT] 已加载进度: 2/10 个关键词完成
[CHECKPOINT] 待处理关键词: ['AI视频', 'AI绘画', ...]  ← 自动跳过已处理的
...继续运行...
[CHECKPOINT] 已保存进度: 4/10  ← 再次超时
```

**直到全部完成：**
```
[CHECKPOINT] 已加载进度: 10/10 个关键词完成
[RESULT] 共 XX 条
[CHECKPOINT] 已清除断点文件
```

### 文件位置

- **断点文件:** `output/checkpoint.json`
- **结果文件:** `output/creators_YYYYMMDD_HHMMSS.xlsx`

### 注意事项

1. **同一会话ID** - 同一次任务的所有运行使用相同的session_id，结果会合并
2. **断点自动清理** - 任务完成后会自动删除checkpoint.json
3. **重复数据自动去重** - 基于创作者ID自动去重
4. **如需重新开始** - 手动删除 `output/checkpoint.json`

## ⚙️ 配置说明

### 修改搜索规则

编辑 `config/settings.json`：

```json
{
  "keywords": ["AI动画", "AI短剧", "AI视频"],
  "max_results": 15,
  "min_followers": 1000,
  "min_comments": 20,
  "output_format": "excel"
}
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `keywords` | 搜索关键词列表 | ["AI动画"] |
| `max_results` | 每个关键词搜索多少个视频 | 15 |
| `min_followers` | 最小粉丝量（少于这个数会过滤） | 1000 |
| `min_comments` | 最小评论数（少于这个数会过滤） | 20 |
| `output_format` | 输出格式：excel/json/feishu | excel |

### 修改关键词示例

搜索AI绘画创作者：
```json
{
  "keywords": ["AI绘画", "AI插画", "AI绘图"]
}
```

放宽筛选条件：
```json
{
  "min_followers": 500,
  "min_comments": 10
}
```

## 🔧 常见问题

### Q1: 第一次使用需要登录吗？
**A:** 是的。首次运行会弹出Chrome窗口，需要扫码登录小红书。Cookie会自动保存，后续运行无需再次登录。

### Q2: 运行时间多久？超时怎么办？
**A:** 
- 1个关键词 + 15个视频 ≈ 3-4分钟
- 3个关键词 + 30个视频 ≈ 10-15分钟

**注意:** OpenClaw单次命令有5分钟超时限制。

**解决方案 - 断点续跑：**
1. 本工具支持**断点续跑**，每处理完一个关键词自动保存进度
2. 超时中断后，**再次运行同一命令**，会自动从上次进度继续
3. 重复运行直到所有关键词处理完成，最终导出合并结果

**示例:**
```
第一次运行: 处理完2个关键词后超时
第二次运行: 自动跳过已处理的2个，继续处理第3、4个
第三次运行: 继续处理第5、6个...
直到全部完成，自动导出结果
```

### Q3: 为什么会"FILTER"或"SKIP"？
**A:** 
- `[FILTER]` - 不符合筛选条件（粉丝/评论不够）
- `[SKIP]` - 非视频内容（图文笔记）
- `[FAIL]` - 页面解析失败

### Q4: 如何导出到飞书？
**A:** 修改配置：
```json
{
  "output_format": "feishu",
  "feishu_sheet_url": "https://..."  // 飞书表格链接
}
```

### Q5: Cookie过期怎么办？
**A:** 删除 `config/cookies.json` 文件，重新运行并扫码登录。

### Q6: checkpoint.json是什么？
**A:** 这是**断点续跑**的进度文件，包含：
- 已完成的关键词列表
- 已采集的创作者数据
- 会话ID和时间戳

**什么时候删除？**
- 任务**正常完成**后会自动删除
- 如需**重新开始**，手动删除此文件

**文件位置:** `output/checkpoint.json`

## 📝 高级用法

### 批量运行多个关键词

编辑 `config/settings.json`：
```json
{
  "keywords": [
    "AI动画",
    "AI短剧", 
    "AI视频",
    "AI绘画",
    "AIGC"
  ],
  "max_results": 10
}
```

**提示:** 关键词越多，运行时间越长。建议分批执行。

### 调整筛选严格度

**严格模式**（只找头部创作者）：
```json
{
  "min_followers": 10000,
  "min_comments": 100
}
```

**宽松模式**（找新兴创作者）：
```json
{
  "min_followers": 100,
  "min_comments": 5
}
```
