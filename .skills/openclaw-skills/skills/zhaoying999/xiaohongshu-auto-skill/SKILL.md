# xiaohongshu-auto-skill — 小红书全平台 AI 自动化运营工具包

## 概述

本技能是一个面向小红书平台的端到端 AI Agent 运营工具包，涵盖从账号定位、内容研究、AI 辅助创作到自动化发布与数据复盘的全生命周期管理。采用多 Agent 协作与模块化复用架构。

## 核心工作流（四阶段）

```
阶段一：定位与研究 → 阶段二：AI 内容生成 → 阶段三：自动化发布 → 阶段四：数据复盘
```

---

## 阶段一：账号定位与研究

### 1.1 账号定位与人设设计

使用 LLM 基于用户提供的行业、目标受众、竞品账号等信息，生成完整的账号定位方案。

**输入要素：**
- 行业/品类（如：美妆、穿搭、美食、母婴、科技数码等）
- 目标受众画像（年龄、性别、消费力、兴趣）
- 竞品账号（可选，1-5 个小红书账号 ID 或主页链接）
- 差异化定位方向（可选）

**输出格式：**

```markdown
# 账号定位方案

## 基础定位
- **账号类型：** [个人号/品牌号/专家号/种草号]
- **核心领域：** [1-2 个垂直领域]
- **差异化标签：** [3-5 个关键词]

## 人设画像
- **昵称风格：** [建议格式]
- **简介模板：** [150 字以内的简介]
- **头像风格建议：** [描述]
- **视觉调性：** [主色调/风格关键词]

## 内容矩阵（建议比例）
| 内容类型 | 占比 | 发布频率 | 示例方向 |
|---------|------|---------|---------|
| 干货分享 | 40% | 每周 3 篇 | ... |
| 种草推荐 | 30% | 每周 2 篇 | ... |
| 生活日常 | 20% | 每周 1 篇 | ... |
| 互动话题 | 10% | 每周 1 篇 | ... |

## 竞品对标分析
- 竞品 A：[分析要点]
- 竞品 B：[分析要点]
```

> **实现提示：** 使用 LLM 推理能力直接生成。如果用户提供了竞品账号链接，可以结合 Web Search 和 Web Reader 技能获取竞品信息来增强分析。

### 1.3 关键词库建设

参考 `references/hot-keywords.md` 获取行业关键词库模板。关键词库用于：
- 选题灵感来源

## 阶段二：AI 内容生成

### 2.1 选题生成

**输入：** 行业领域 + 目标受众 +（可选）关键词/热点

**工作流程：**
1. 如果需要获取实时热点，使用 Web Search 搜索小红书相关热词趋势
2. 结合 `references/hot-keywords.md` 中的行业关键词
3. LLM 生成 5-10 个选题建议

**输出格式：**

```markdown
# 选题方案

## 热点选题（时效性强）
1. **[选题标题]** — 关键词：xxx | 预估热度：⭐⭐⭐⭐⭐
2. ...

## 常青选题（长尾流量）
1. **[选题标题]** — 关键词：xxx | 搜索量趋势：稳定
2. ...

## 选题推荐理由
对每个选题给出：
- 目标受众匹配度
- 竞争度评估
- 内容差异化建议
```

### 2.2 标题创作

小红书标题是流量的核心入口。标题创作遵循以下原则：

**爆款标题公式（参考 `references/content-template.md`）：**

```
公式1: 数字+痛点+解决方案
公式2: 身份+场景+结果
公式3: 反常识+揭秘/真相
公式4: 时间限定+效果承诺
公式5: 对比+选择建议
公式6: 情绪共鸣+共鸣点
```

**标题要求：**
- 长度控制在 20 字以内（最优展示）
- 必须包含核心关键词
- 使用 emoji 增加视觉吸引力（1-3 个）
- 避免标题党，内容必须匹配

**输出：** 为每个选题生成 3-5 个标题候选，标注推荐指数。

### 2.3 正文撰写

**内容结构（小红书风格）：**

```
【封面标题 / Hook】  ← 第一句话决定用户是否继续阅读

正文段落1：场景代入 / 痛点共鸣
正文段落2：核心内容 / 方法论
正文段落3：细节补充 / 案例佐证
正文段落4：总结 / 行动号召

#标签1 #标签2 #标签3 #标签4 #标签5
```

**写作规范：**
- 使用口语化、亲切的表达（"姐妹们"、"绝绝子"、"按头安利" 等适度使用）
- 每段 2-4 句，段落间空行
- 适当使用 emoji 分割段落和强调重点
- 结尾引导互动（提问/投票/引导收藏）
- 标签 5-10 个，混合大流量词和精准长尾词

**参考 `references/content-template.md` 获取完整的内容模板库。**

### 2.4 标签推荐

基于 `assets/tag-library.json` 标签库 + LLM 推理：

**标签策略：**
- 核心标签（2-3 个）：与内容最直接相关的大流量词
- 场景标签（2-3 个）：使用场景相关
- 长尾标签（2-4 个）：精准匹配目标搜索

### 2.5 封面设计

**设计指导：**

封面是小红书笔记的"门面"，直接影响点击率。提供以下设计方向：

```
封面类型选择：
├── 文字型封面（适用：干货/教程/攻略类）
├── 拼图型封面（适用：对比/清单/合集类）
├── 人像型封面（适用：穿搭/美妆/生活类）
├── 前后对比型（适用：改造/效果对比类）
└── 信息图表型（适用：数据分析/科普类）
```

**设计要素：**
- 比例：3:4（竖版），1080×1440px
- 字体：大而醒目的标题文字
- 配色：与账号调性一致，高对比度
- 关键元素不超过 3 个

> **实现提示：** 使用 Image Generation 技能生成封面图片。根据笔记类型选择合适的封面风格，将标题文字嵌入图片。

## 阶段三：自动化发布

### 3.1 图文发布

使用 `scripts/publish.py` 完成图文笔记发布：

```bash
python scripts/publish.py \
  --type image \
  --title "笔记标题" \
  --content "笔记正文内容" \
  --images "path/to/image1.jpg,path/to/image2.jpg" \
  --tags "标签1,标签2,标签3"
```

### 3.2 视频发布

```bash
python scripts/publish.py \
  --type video \
  --title "笔记标题" \
  --content "笔记正文内容" \
  --video "path/to/video.mp4" \
  --cover "path/to/cover.jpg" \
  --tags "标签1,标签2,标签3"
```

### 3.3 定时发布

**最佳发布时间参考：**

| 时段 | 时间 | 推荐指数 | 适用内容 |
|------|------|---------|---------|
| 早间通勤 | 7:30-9:00 | ⭐⭐⭐ | 新闻/资讯/干货 |
| 午间休息 | 12:00-13:30 | ⭐⭐⭐⭐ | 种草/教程/测评 |
| 下班通勤 | 17:30-19:00 | ⭐⭐⭐ | 日常/美食/生活 |
| 晚间休闲 | 20:00-23:00 | ⭐⭐⭐⭐⭐ | 所有类型（流量高峰） |

### 3.4 多账号管理

如果需要管理多个账号，在 `publish.py` 中通过 `--account` 参数切换：

```bash
python scripts/publish.py --account "账号昵称" ...
```

### 3.5 互动管理

**自动回复策略：**

评论分类与回复模板：
- **咨询类：** 提供产品/服务信息回复
- **好评类：** 感谢 + 引导关注
- **负面类：** 真诚回应 + 私信沟通引导
- **互动类：** 顺势互动 + 延展话题

> **实现提示：** 使用 LLM 生成个性化回复，避免千篇一律的模板回复。

## 阶段四：数据复盘

### 4.1 数据采集

使用 `scripts/crawler.py` 采集笔记数据：

```bash
python scripts/crawler.py \
  --action notes \
  --account "账号ID或昵称" \
  --limit 50 \
  --output "data/notes_data.csv"
```

**可采集指标：**
- 笔记基础数据（点赞、收藏、评论、分享）
- 互动率 = (点赞+收藏+评论+分享) / 曝光量
- 粉丝增长数据
- 搜索排名数据

### 4.2 竞品分析

```bash
python scripts/crawler.py \
  --action competitor \
  --targets "竞品1,竞品2,竞品3" \
  --limit 30 \
  --output "data/competitor_data.csv"
```

### 4.3 爆款拆解

**爆款笔记分析框架：**

```markdown
# 爆款拆解报告

## 笔记概览
- 标题：xxx

## 爆款因素分析
| 因素 | 分析 |
|------|------|
| 标题 | [分析标题的钩子策略] |
| 封面 | [分析封面设计亮点] |

| 情绪价值 | [分析情绪触发点] |
| 标签 | [分析标签选择] |

1. ...

## 优化方向
1. ...
2. ...
```

### 4.4 ROI 报告

使用 `scripts/report_builder.py` 生成运营报告：

```bash
python scripts/report_builder.py \
  --type daily \
  --account "账号ID" \
  --output "reports/daily_report.md"
```

**报告类型：**
- `daily` — 运营日报
- `weekly` — 运营周报
- `monthly` — 月度复盘
- `campaign` — 专题活动复盘

## 快速开始示例

**场景：为一个美妆品牌新号制定运营方案**

```
1. 运行阶段一：生成账号定位方案
   → LLM 基于行业信息生成定位与人设

2. 运行阶段一：竞品对标分析
   → crawler.py 采集竞品数据 → LLM 生成分析报告

3. 运行阶段二：生成一周选题 + 内容
   → content_generator.py 批量生成 7 天的选题、标题、正文

4. 运行阶段二：生成封面
   → Image Generation 技能生成封面图

5. 运行阶段三：定时发布
   → publish.py 按计划发布笔记

6. 运行阶段四：一周后数据复盘
   → crawler.py 采集数据 → report_builder.py 生成周报
```
