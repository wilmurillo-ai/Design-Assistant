# Daily News Briefing - Configuration Guide

## Prerequisites

### 1. API Keys

The briefing system uses Baidu search API for news aggregation.

**Required Environment Variables:**

```bash
# Baidu Search API Key (required)
export BAIDU_API_KEY="bce-v3/ALTAK-xxx/xxxx"

# Target user ID for QQ delivery (optional, default from config)
export QQ_TARGET_USER="9C12E02D9038B14FCEDCE1B69AAEAB3F"
```

**Where to set:**
- System-wide: `/etc/environment` or `/etc/profile.d/news-briefing.sh`
- User-specific: `~/.bashrc`, `~/.zshrc`
- Per-session: Export before running scripts

### 2. OpenClaw CLI

Required for automated delivery via messaging channels.

```bash
# Check installation
openclaw --version

# Verify QQ bot channel is configured
openclaw status
```

## Cron Job Configuration

### Default Schedule (9:00 AM Daily)

```cron
# Generate news at 9:00 AM
0 9 * * * source /etc/profile && cd /path/to/scripts && BAIDU_API_KEY="$BAIDU_API_KEY" python3 generate-briefing.py >> /var/log/daily-news.log 2>&1

# Deliver at 9:01 AM  
1 9 * * * source /etc/profile && bash /path/to/scripts/deliver-briefing.sh >> /var/log/news-delivery.log 2>&1
```

### Custom Schedules

| Time | Cron Entry | Use Case |
|------|-----------|----------|
| 7:00 AM | `0 7 * * *` | Early morning briefing |
| 7:30 AM | `30 7 * * *` | Standard (after system fully up) |
| 9:00 AM | `0 9 * * *` | Business hours start |
| 6:00 PM | `0 18 * * *` | Evening summary |

**Multiple deliveries:** You can set up multiple cron jobs with different times.

### Installing Cron Job

```bash
# Edit crontab
crontab -e

# Add the entries above, adjusting paths as needed

# Verify installation
crontab -l

# Check cron service
systemctl status cron
```

## Script Configuration

### generate-briefing.py

**Key parameters to customize:**

```python
# News file location (default: workspace/daily-news-YYYYMMDD.md)
WORKSPACE = '/root/.openclaw/workspace'

# Search categories and queries (line ~85)
china_tech = search_baidu('科技新闻 人工智能 芯片 AI 华为', count=3, freshness="pd")
intl_tech = search_baidu('NVIDIA Broadcom Apple Microsoft AI 美股科技股', count=3, freshness="pd")
china_finance = search_baidu('A 股 上证指数 港股 财经', count=3, freshness="pd")
intl_finance = search_baidu('美股 纳斯达克 道琼斯 美联储 原油 比特币', count=3, freshness="pd")
```

**Modify search queries:**
- Change keywords to target different news categories
- Adjust `count` parameter (1-5) for more/fewer articles
- `freshness`: "pd" (past day), "pw" (past week), "pm" (past month)

### deliver-briefing.sh

**Key variables:**

```bash
# Target user ID (QQ channel)
TARGET_USER="9C12E02D9038B14FCEDCE1B69AAEAB3F"

# Timeout for delivery attempts (seconds)
TIMEOUT=30
```

**Delivery methods priority:**
1. OpenClaw CLI (fastest, direct)
2. Python script (fallback with session scheduling)
3. Notification file (last resort, manual pickup)

## Log Files

| Log File | Purpose | Location |
|----------|---------|----------|
| daily-news.log | News generation status | `/var/log/daily-news.log` |
| news-delivery.log | Delivery attempt results | `/var/log/news-delivery.log` |

**Log rotation (optional):**
```bash
# Add to /etc/logrotate.d/daily-news-briefing
/var/log/daily-news.log
/var/log/news-delivery.log {
    daily
    rotate 7
    compress
    missingok
    notifempty
}
```

## Multi-Channel Delivery

### Supported Channels

The delivery scripts support any OpenClaw-configured channel:

```bash
# In deliver-briefing.sh, modify:
--channel qqbot      # Default (QQ Bot)
--channel telegram   # Telegram
--channel discord    # Discord
```

### Channel-Specific Targets

| Channel | Target Format | Example |
|---------|--------------|---------|
| QQBot | `c2c:USERID` | `c2c:9C12...` |
| Telegram | `@username` or chat ID | `@mychannel` |
| Discord | `guild-id/channel-id` | `123456/789012` |

## Advanced: Custom Commentary Rules

Modify `_generate_tech_commentary()` and `_generate_finance_commentary()` functions in `generate-briefing.py`:

```python
rules = [
    (['keyword1', 'keyword2'], "Your custom commentary here"),
    (['stock', 'market'], "Another commentary pattern"),
]
```

Pattern matching is case-insensitive and checks both title and content.
