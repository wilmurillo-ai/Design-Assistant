# Technical Report Style Guide

## 1. Tone & Voice
- **Technical**: Use precise terminology, include technical details (e.g., specs, code snippets).
- **Executive**: High-level summary, focus on business impact and recommendations.
- **General**: Simple language, avoid jargon, explain concepts clearly.

## 2. Formatting Rules
- **Headings**: Use `#` for title, `##` for main sections, `###` for subsections.
- **Lists**: Use `-` for unordered lists, `1.` for ordered lists.
- **Code**: Use inline code `` `code` `` and code blocks ```language ... ```.
- **Emphasis**: Use **bold** for key points, *italic* for definitions.
