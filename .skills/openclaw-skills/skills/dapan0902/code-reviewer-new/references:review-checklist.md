# Python 代码审查清单
## Error 级别（必须修复）
1. 无语法错误、缩进错误、未定义变量/函数
2. 异常处理：关键操作（文件读写、网络请求）必须捕获可能的异常，避免程序崩溃
3. 类型安全：核心函数的参数/返回值必须有类型注解

## Warning 级别（建议修复）
1. 符合 PEP 8 规范：行长度 ≤ 88 字符、变量/函数名小写+下划线、类名大驼峰
2. 注释：复杂逻辑（算法、特殊处理）必须加清晰注释，避免无意义注释
3. 导入规范：避免 `import *`，按需导入；导入顺序（标准库 → 第三方库 → 本地库）

## Info 级别（可选优化）
1. 避免冗余代码：删除重复的条件判断、无用变量
2. 循环优化：嵌套层级 ≤3 层，大列表遍历优先用生成器
3. 命名规范：变量/函数名见名知意（避免 a/b/c 等无意义命名）
