# Logic Patterns

This file stores reusable decomposition scaffolds.
These are not rigid templates. They are starting structures for reducing messy problems.

## Strategy

Reduce to:
- objective
- constraints
- core drivers
- leverage point
- irreversible risk

## Debugging

Reduce to:
- symptom
- possible mechanism
- failing dependency
- isolating test
- likely root cause

## Decision

Reduce to:
- objective
- options
- downside
- reversibility
- asymmetric upside

## Argument

Reduce to:
- claim
- evidence
- inference
- strongest objection
- synthesis

## Product

Reduce to:
- user need
- friction point
- value loop
- adoption constraint
- viability gap
