## 📋 隐私与透明度

**服务提供商**: Brand Marketing Platform
**计费平台**: 智灵 Skill 平台 (https://skills.zeelin.cn)
**数据处理**:
- 用户提交的品牌名称将用于采集全网公开数据
- 系统自动获取当日热点信息并进行匹配分析
- 生成一篇小红书图文内容基于 AI 分析结果

**数据保留**:
- 品牌分析数据保留 24 小时用于优化
- 请求日志保留 7 天用于故障排查
- 生成的内容不会被二次使用或分享

## 使用时机

当用户请求：
- 为品牌生成一篇小红书图文内容

- 追热点生成一篇营销内容
- 小红书图文创作

不适合：生成音乐、视频、代码开发等。

## 🔑 第一步（强制）：获取并验证用户 App-Key

**⚠️ 必须在任何创作操作之前完成此步骤，不得跳过！**

### 1-A. 检查本地存储的 App-Key

Agent 在每次启动或接收到品牌营销任务时，**优先读取本地持久化存储的 App-Key**。
存储位置：用户主目录下的 `~/.zeelin_config` 文件（或同级工作目录下的 `.zeelin_config`）。

**情况一：本地存在已保存的 App-Key（后续使用）**

读取文件中的 App-Key，自动调用校验接口（见 1-B）验证余额，然后展示：

> ✅ 检测到已保存的 App-Key：S6YVomNq2PYl...（显示前12位）
> 当前余额：**1000 额度**，可生成 14 篇内容。
>
> 如需更换 App-Key，请回复"更换密钥"或直接提供新的 Key。

**情况二：本地无存储记录（首次使用或配置文件已删除）**

Agent 向用户发起询问：

> 📊 **开始品牌分析前，我需要验证你的智灵账户额度。**
>
> 请提供你的智灵 App-Key（前往 https://skills.zeelin.cn 注册并创建应用获取）。
>
> 📌 每次生成一篇图文内容消耗 **70 额度**。

### 1-B. 调用额度校验接口

使用用户提供的 App-Key 调用校验接口：

```
POST https://skills.zeelin.cn/v2/api/skill/detail
Header: app-key: <用户的 zeelin_app_key>
Header: Content-Type: application/json
Body:
{
  "query": "品牌营销内容生成: <品牌名称>",
  "skill-id": "zeelin_BOyrUqap51qk4zxQOs92dch69"
}
```

### 1-C. 处理校验结果

**成功（code: 200）：**
```json
{
  "code": 200,
  "data": {
    "pre_order_id": "2026038787913be67748...",
    "remain_calls": 1000,
    "skill_id": "zeelin_BOyrUqap51qk4zxQOs92dch69"
  }

```
→ 将 App-Key **写入本地文件** `~/.zeelin_config` 持久化保存（首次验证通过时写入，更换密钥时覆盖）；
→ 保存 `pre_order_id`（2小时内有效），告知用户余额并继续：
> ✅ 验证通过！当前余额 **1000 额度**，可生成 14 篇内容。开始分析！

**余额不足（code: 402 或 remain_calls < 70）：**

> ❌ App-Key 无效，请检查是否正确复制。前往 https://skills.zeelin.cn/console/apps 查看你的 Key。

### 1-D. 更换 App-Key

当用户回复"更换密钥"或在已有 Key 的情况下直接提供新 Key 时：

1. 要求用户提供新的 App-Key

> ✅ 新 App-Key 验证通过并已保存！

## 📊 第二步：生成一篇品牌营销图文内容

**基础 URL**: `http://vane.zeelin.cn` (生产环境请替换为实际域名)

**POST** `/gateway/vane_marketing/content`

### ⏱️ 请求配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 请求超时 | **300 秒（5 分钟）** | 生成涉及品牌采集 + 热点分析 + AI 创作，耗时较长 |
| 禁止重复调用 | ✅ | 超时时间内不得重新发起请求 |
| 允许重试条件 | 明确网络错误或 HTTP 5xx | 不因响应慢而触发重试 |

### 🕐 等待策略

发起请求后立即向用户展示进度提示，保持等待直至收到响应或超时：

> ⏳ 正在为"XXX"品牌生成内容，包含数据采集 + 热点分析 + AI 创作，预计需要 2-3 分钟，请稍候...

**重试规则：**
- ✅ **允许重试**：明确的网络错误（连接失败、DNS 解析失败等）或 HTTP 5xx 服务器错误
- ❌ **禁止重试**：仅因响应时间较长，不重新发起请求
- ⏰ **超时处理**：等待超过 300 秒后提示：

> ❌ 生成超时（5 分钟），请稍后重试或联系技术支持（https://skills.zeelin.cn）。

### 请求示例

```bash
curl --location --request POST 'http://vane.zeelin.cn/gateway/vane_marketing/content' \
--header 'App-Key: <用户的 zeelin_app_key>' \
--header 'Content-Type: application/json' \
--data-raw '{
    "brand_name": "华为"
}'
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| brand_name | string | 是 | 品牌名称，如"华为"、"小米"等 |

## 📝 第三步：提取并展示小红书发布内容

从第二步返回的 `data` 列表中提取以下三个关键字段，整理为一篇可直接用于小红书发布的图文内容：

### 3-A. 字段提取规则

| 字段       | 来源路径 | 处理规则                                         |
|----------|----------|----------------------------------------------|
| **标题**   | `title` | 检查长度不超过 20 字，超出则自动截断并提示用户                    |
| **正文**   | `content` | 移除 Markdown 语法标记，保留纯文本、换行符和表情符号，长度不超过 1000 字 |
| **话题标签** | `tags` | 拼接在正文末尾                                      |
| **图片**   | `img_url` | 根据链接下载图片，存储在本地，保存图片存储路径                      |

### 3-B. 内容适配处理

**标题处理：**
- 原始标题：`华为AI手机来了！这才是未来该有的样子✨`
- 检查字数 ≤ 20 字 → ✅ 通过
- 若超出：截断为前 20 字 + `…`，并提示用户确认

**正文处理：**
- 移除所有 Markdown 语法标记（`#` `**` `*` `-` `>` 等）
- 保留纯文本内容、换行符、段落结构和表情符号
- 示例：`**粗体**` → `粗体`，`# 标题` → `标题`
- 检查正文字数 ≤ 1000 字 → 超出则提示用户精简

**话题标签处理：**
- 原始数组：`["华为", "AI手机", "科技好物"]`
- 转换结果：`#华为# #AI手机# #科技好物#`
- 拼接到正文末尾（换行分隔）

## ⚠️ 错误处理

| 错误码 | 原因 | 处理方式 |
|--------|------|----------|
| 402 | 余额不足 | 提示用户充值 https://skills.zeelin.cn |
| 404 | App-Key 不存在 | 提示用户检查 Key |
| 401 | 未传 App-Key | 要求用户提供 Key |
| 429 | 请求频率过高 | 等待 60 秒后重试 |
| DATA_COLLECTION_ERROR | 品牌数据采集失败 | 检查品牌名称是否正确，稍后重试 |
| HOT_TOPIC_ERROR | 热点获取失败 | 系统繁忙，稍后重试 |
| CONTENT_GENERATION_ERROR | 内容生成失败 | 重新发起请求，失败不扣费 |

### 自动发布相关错误

| 错误类型 | 处理方式 |
|----------|----------|
| 浏览器启动失败 | 提示用户检查环境，建议改为手动发布 |
| 未登录/登录超时 | 提示用户重新扫码，或改为手动发布 |
| 标题超过 20 字 | 自动截断并提示用户确认 |
| 正文超过 1000 字 | 提示用户精简内容后重试 |
| 图片超过 9 张 | 仅上传前 9 张并提示 |
| 图片格式不支持 | 提示仅支持 jpg/png 格式 |
| 发布失败/网络异常 | 提示稍后重试或改为手动发布 |
| 小红书风控拦截 | 提示可能触发敏感词，建议修改内容后手动发布 |

## 🔧 技术说明

### 工作流程

1. **品牌数据采集**：从全网公开数据源采集品牌相关信息
2. **热点分析**：获取当日热门话题并计算热度值
3. **智能匹配**：AI 分析品牌与热点的契合度
4. **内容生成**：基于小红书平台特点生成一篇图文内容
5. **内容提取**：从返回结果中提取标题、正文、话题标签三要素
6. **内容适配**：按小红书平台规范进行格式校验和转换
7. **发布选择**：用户自主选择手动发布或自动发布
8. **余额扣费**：成功生成后自动扣除 70 额度

### API 限制

- 请求频率：每分钟最多 10 次
- 超时时间：**300 秒（5 分钟）**
- 品牌名称长度：1-50 字符

### 小红书平台限制

- 标题：≤ 20 字
- 正文：≤ 1000 字
- 图片：最多 9 张，支持 jpg/png
- 发布频率：建议间隔 5 分钟以上
