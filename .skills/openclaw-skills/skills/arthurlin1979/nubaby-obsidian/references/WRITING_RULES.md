# WRITING_RULES

## 這份只管什麼
- 只管筆記命名與內文開頭格式
- 不管內容該放哪裡，落點回到 `ARTHUR_OS_STRUCTURE.md`
- 不管搜尋方式與 gateway

## Arthur-OS 根目錄命名規範
- 根目錄 `.md` 檔名預設使用：`YYYY-MM-DD title.md`
- 例：`2026-04-14 Harness Engineering 技術實作精華.md`

## Arthur-OS 根目錄內文開頭規範
- 內文最前面固定先放：`修改時間：YYYY-MM-DD HH:MM:SS`
- 後面空一行再進正文
- 不要在正文第一行再重複整個檔名

### 範例
```md
修改時間：2026-04-14 09:27:00

# Harness Engineering 技術實作精華
```

## 操作原則
- 新建根目錄筆記時，直接照這個格式建立
- 舊筆記若命名不一致，可批次改成同格式
- 若遇到空檔、孤兒檔、或無法安全推斷標題的檔案，不要硬改名；先檢查內容再決定
