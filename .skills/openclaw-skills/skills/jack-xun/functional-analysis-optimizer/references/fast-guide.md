# FAST 功能分析法详解

## 什么是 FAST

FAST（Function Analysis System Technique）是一种功能导向的系统分析工具，核心是把每个功能表达为"**动词+名词**"结构。

> 注意：功能≠特征。特征是静态描述，功能是动态动作。
> - ❌ "保温性"（特征）→ ✅ "保持温度"
> - ❌ "防水"（特征）→ ✅ "阻止水分渗透"

## FAST 图的基本结构

```
总功能（系统存在的根本理由）
│
├─ 上位功能（帮助总功能实现的一级子功能）
│   ├─ 上位功能A
│   └─ 上位功能B
│
└─ 从属功能（实现上位功能所必须的具体手段）
    ├─ 从属功能A
    └─ 从属功能B
```

**判断规则：**
- "如何"→找从属（手段）
- "为了"→找上位（目的）

## 案例：外卖保温袋

```
总功能：保持食物温度
│
├─ 隔绝热传导 [上位]
│   ├─ 从属：使用隔热材料（如锡纸）[从属]
│   └─ 从属：减少空气对流（封口设计）[从属]
│
├─ 减缓温度流失 [上位]
│   ├─ 从属：反射热辐射（银色内层）[从属]
│   └─ 从属：减少与外界接触面积（方正设计）[从属]
│
└─ 便于交付 [上位]
    ├─ 从属：提供手持结构（提手）[从属]
    └─ 从属：便于堆叠码放（标准尺寸）[从属]
```

## 案例：周报流程

```
总功能：传递工作进展信息
│
├─ 收集信息 [上位]
│   ├─ 从属：各成员填写日报[从属]
│   └─ 从属：自动拉取系统数据（工时/任务）[从属]
│
├─ 整合编辑 [上位]
│   ├─ 从属：汇总人整理成文[从属]
│   └─ 从属：模板自动填充[从属]
│
└─ 传递交付 [上位]
    ├─ 从属：邮件发送[从属]
    └─ 从属：共享文档实时查看[从属]
```

## 从功能到创新

功能分析的价值在于**打破"现状默认"**：

| 传统思维 | 功能思维 |
|----------|----------|
| "保温袋应该加厚" | "隔绝热传导"还能用什么实现？→ 真空层、相变材料 |
| "周报应该更早上交" | "传递信息"还能怎么实现？→ 日历共享、站会视频 |

## 常见场景

- 产品设计：分析竞品的功能树，找差异化切入点
- 流程优化：把流程步骤翻译为功能，发现冗余和断点
- 页面/UI优化：把页面元素翻译为用户功能诉求
- 组织诊断：分析部门职能的过载/缺失/低效
