# 功能建模常见错误与修正

## 错误 1：用特征代替功能

**错误示例：**
- "保温袋的保温性" → 不是功能，是特征
- "页面的美观性" → 不是功能，是特征

**修正方法：** 追问"这个特征帮用户做了什么"，找到动词。
- "保温袋的保温性" → "保持食物温度"
- "页面的美观性" → "建立用户信任"

---

## 错误 2：功能层级混乱

**错误示例：**
```
总功能：保持温度
├─ 隔绝热传导（这应该是一级）
└─ 使用锡纸（锡纸是"隔绝热传导"的从属手段，不是同级）
```

**修正方法：**
- "如何实现隔绝热传导？" → 回答是"从属功能"
- "隔绝热传导是为了什么？" → 回答是"上位功能"

---

## 错误 3：跳过总功能直接拆子功能

**错误表现：** 用户说"我想优化外卖袋的提手"，直接跳到提手，没有先定义总功能。

**修正方法：** 始终从总功能开始：
> "在优化提手之前，先确认一下——外卖袋的总功能是'保持食物温度'，提手只是实现'便于交付'这个子功能的手段。对吗？"

---

## 错误 4：诊断时跳过"正常"节点

**错误表现：** 只标注问题节点，不说正常节点，导致功能树不完整。

**修正方法：** 所有节点都要标注状态，即使正常也要标记：
> "隔绝热传导 [🟢 正常] — 当前铝箔方案成本和效果均衡，可接受"

---

## 错误 5：创新发散时替换总功能

**错误表现：** 创新路径直接改变了系统总功能（比如把"保温"改成了"制冷"），造成范围蔓延。

**修正方法：** 创新路径必须**保持总功能不变**，只改变实现手段。

---

## 错误 6：落地切片贪多

**错误表现：** 给出一个需要同时改 3 个子功能的"最小切片"。

**修正方法：** 最小切片**只改一个子功能**，验证一个假设。
> 如果要同时验证两个假设，那就算两个切片，不是最小切片。
