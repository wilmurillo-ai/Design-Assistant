# PPT标准格式生成指南 v4.1

## 严格的页面结构

### PPT生成严格按照以下结构：

#### 第1页：封面
- **标题**：DMP用户画像洞察分析报告
- **副标题**：基于N个维度的深度分析
- **人群名称**：显示分析的人群名称

#### 第2页：目录
- 用户画像总结
- 地域深耕方向（若无该维度则自动忽略）
- 各维度详细分析（X个维度）
- 综合策略建议
- 深度分析方向

#### 第3页：用户画像总结
- **一句话核心特征**：高度浓缩的用户特征描述
- **核心特征统计**：主要统计数据和特征

#### 第4页（可选）：地域深耕方向
**仅当有地域维度数据时出现**
- **优先布局**：优先布局的省份/城市
- **潜力市场**：潜力城市和地区推荐

#### 第5-N页：各维度详细分析
**每个维度一页**
- **核心特征列表**（表格形式）
  - 特征名称
  - TGI值
  - 占比（如有）
- **判断依据说明**
  - 该维度特征分析
  - 用户特点说明
- **数据分布说明**
  - 数据量
  - 平均TGI
  - 最高TGI

#### 倒数第2页：综合策略建议
- **产品定位**：面向的目标用户和产品方向
- **渠道布局策略**：推荐的营销渠道和策略
- **内容营销策略**：内容创意和营销方向

#### 最后一页：深度分析方向
- **单维度深度分析**：推荐进一步分析的维度
- **交叉维度分析**：推荐的维度交叉组合

```bash

```

## 页数计算公式

```
总页数 = 5 + 维度数 + (1 if 有地域数据 else 0)

示例：
- 5维度 + 无地域 = 10页
- 7维度 + 有地域 = 13页
- 10维度 + 有地域 = 16页
```

## 关键特性

### 1. 完全自动化的地域判断

脚本会自动检测是否有地域数据：
- ✅ 有地域维度 → 自动在P4生成地域深耕方向页
- ✅ 无地域维度 → 自动跳过，目录也相应调整

### 2. 动态分页

根据维度数自动调整：
```
- 3维度 = 9页PPT
- 5维度 = 11页PPT（无地域）或 12页PPT（有地域）
- 7维度 = 13页PPT（无地域）或 14页PPT（有地域）
```

### 3. 规范化内容提取

从Markdown自动准确提取：
- ✅ 一句话核心特征
- ✅ 核心定位标签
- ✅ 地域深耕方向（如有）
- ✅ 各维度的特征表、判断理由、数据分布
- ✅ 产品定位、渠道策略、内容营销
- ✅ 深度分析建议

### 4. 专业企业级设计

配色体系：
- **主色**：深蓝 #0070C0（标题、重点）
- **强调**：红色 #FF6B6B（分隔线）

- **背景**：深蓝 #065A82（封面、结束）

排版规范：
- 标题：44-54pt 加粗
- 副标题：24-28pt 加粗

## 常见问题

### Q1：为什么没有第4页地域深耕方向？

**原因**：你的Markdown报告中没有地域维度数据。

**检查方法**：
```bash
grep -n "### 地域深耕方向" reports/报告名称.md
```

**解决**：
- 如果数据本身无地域维度 → 这是正常的，目录会自动调整
- 如果应该有但被遗漏 → 检查分析数据是否包含地域维度

### Q2：维度顺序和目录对不对？

**可能原因**：Markdown中的维度顺序与目录编号不一致

**检查**：
```bash
grep "### 【.*.维度】" reports/报告名称.md
```

**解决**：脚本会按照Markdown中的顺序自动编号，确保一致

### Q3：某些内容在PPT中显示不全？

**原因**：内容太长，脚本自动截断以保证排版

**截断规则**：
- 核心特征：显示Top 8
- 判断理由：最多150字
- 数据说明：最多100字
- 策略建议：最多120字

**解决**：完整内容仍在Markdown报告中

### Q4：如何自定义PPT内容长度？

编辑 `generate_ppt_standard.py`，修改以下行：

```python
# 修改特征显示数量
for feature in dim['features'][:8]:  # 改为[:10]显示Top 10

# 修改文本截断长度
reason_text = dim['reason'][:150]  # 改为[:250]显示更多
```
