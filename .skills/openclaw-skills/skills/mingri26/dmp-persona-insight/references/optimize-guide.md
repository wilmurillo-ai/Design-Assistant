# DMP人群洞察分析 - v2.0优化指南

## 概述

`analyze_persona_v2.py` 是基于 **分析方法论** 优化后的增强版脚本，产出更加规范、可执行的洞察报告。

## 核心优化点

### 1️⃣ 规范化报告输出

#### 报告结构
```

  ↓
核心用户画像（关键指标表）
  ↓
多维度深度分析（维度1-7）
  ↓
用户分层与细分（3层）
  ↓
营销策略建议（渠道、内容、预算）
  ↓

```

#### 输出特点
- ✅ 完整的Markdown格式
- ✅ 包含所有必要的分析维度

- ✅ 可直接用于决策支持

#### 预算调整指引

**如果获客成本高于预期：**
- 减少高成本渠道投入
- 增加已验证高效渠道投入
- 优化创意和落地页

**如果转化率低于预期：**
- 检查内容创意质量
- 优化用户匹配度
- A/B测试不同营销文案

## 使用指南

### 安装与运行

```bash
# 1. 进入技能目录
cd ~/.openclaw/skills/dmp-persona-insight

# 2. 准备数据（Excel多sheet或CSV）
# 文件格式：
# - Sheet1: 基础标签
# - Sheet2: 兴趣偏好
# - Sheet3: 应用偏好
# ...

# 3. 运行脚本（v2.0优化版）
python scripts/analyze_persona_v2.py data.xlsx 人群名称 --output reports

# 4. 查看报告
cat reports/人群名称_洞察分析报告_v2.md
```

### 数据要求

#### 必需字段
- **标签名称**：标签的具体名称（一级标签、二级标签等）
- **TGI值**：目标群体指数（大于0）
- **其他字段**：可选，如占比、人数等

#### 支持格式
- Excel (.xlsx, .xls)：多个Sheet，每个Sheet代表一个维度
- CSV：单个文件，可包含多个维度

#### 数据质量要求
- ✅ 列名规范（包含标签、TGI关键词）
- ✅ TGI值为有效数值（>0）
- ✅ 无大量空值或缺失值
- ✅ 建议至少7-10个维度

## 最佳实践

### ✅ 做法

1. **数据优先**：确保输入数据完整和准确
2. **维度充足**：至少7-10个维度，确保分析深度
3. **关注TGI**：关注最高TGI和平均TGI的差异
4. **分层差异化**：为不同分层制定不同策略
5. **预算合理**：预算分配与ROI期望相匹配

### ❌ 避坑

1. **不要只看最高TGI**：要同时看平均TGI
2. **不要忽视维度含义**：理解每个维度的业务意义
3. **不要过度分层**：3-5层最优，太多无法执行
4. **不要预算过度集中**：要留有试验空间（P2渠道）
5. **不要忽视创意质量**：再好的渠道也需要好创意

## FAQ

**Q: 报告中的TGI > 100是什么意思？**
A: TGI > 100表示该人群该特征比例高于全库平均，值越高说明该人群该特征的优势越明显。

**Q: 为什么分成3层而不是更多？**
A: 3-5层是最优选择。太多层数无法制定差异化策略，太少层数覆盖不全。

**Q: 预算应该怎么分配？**
A: 基于渠道ROI期望分配。高ROI渠道多投（如微信），低ROI渠道少投。

**Q: 如何判断分析结果是否可靠？**
A: 检查三点：①所有结论都有数据支撑 ②用户分层差异明显 ③策略可直接执行

**Q: 报告多久更新一次？**
A: 建议每个季度或业务重大变化时更新一次。
