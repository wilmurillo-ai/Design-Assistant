# fortune-hub — 运势测算统一入口

## 快速开始

当用户说出以下任意模糊请求时触发本 Skill：

- "帮我看看运势"
- "算一卦"
- "今天运气怎么样"
- "帮我算算"
- "占卜一下"
- "测算"
- "命理分析"

触发后，先读取 `MEMORY.md` 判断用户意图，再决定直接路由还是展示领域菜单。

## 工作流程

### Step 1：识别意图

从用户输入中判断想测的领域：

- **明确指向某领域**（如"帮我看今天的星座运势"、"塔罗占卜一下"）→ 直接跳至 Step 2
- **意图模糊**（如"帮我算算"、"今天运气怎么样"）→ 展示领域选择菜单

**菜单展示格式：**

```
🔮 你想从哪个角度了解运势？

1. ♈ 星座运势 — 基于太阳星座的每日/周/月运势
2. 🔢 数字命理 — 基于姓名和生日的生命灵数分析 ✅
3. 🃏 塔罗占卜 — 针对具体问题的牌阵解读 ✅
4. 🀄 八字命理 — 基于出生年月日时的五行分析 ✅
5. ⭐ 紫微斗数 — 十四主星排盘的命运解读 ✅
6. 🌸 梅花易数 — 即时起卦的吉凶预测 ✅
7. 📿 六爻易经 — 铜钱起卦的吉凶断卦 ✅
8. 🕉️ 吠陀占星 — 印度占星术的星盘解读 ✅
9. 🧭 奇门遁甲 — 九宫排盘的择时决策 ✅
10. 🏠 风水顾问 — 居家/办公空间的风水分析 ✅

请选择编号，或直接描述你的问题。
```

### Step 2：检查用户档案

读取本目录下的 `MEMORY.md`，检查是否已有该领域所需的全部信息：

- **已有全部所需字段** → 跳过询问，直接进入 Step 4
- **部分缺失** → 只询问缺失字段，进入 Step 3
- **MEMORY.md 不存在** → 从头收集当前领域所需信息，进入 Step 3

### Step 3：收集缺失信息

**各领域所需字段：**

| 领域 | 必须字段 | 可选字段 |
|------|---------|---------|
| 星座运势 | 公历生日 | — |
| 数字命理 | 公历生日、英文全名 | — |
| 塔罗占卜 | — | 具体问题（每次现场询问） |
| 八字四柱 | 公历生日、出生时辰、性别 | 出生地 |
| 紫微斗数 | 农历生日（可从公历转换）、出生时辰、性别 | — |
| 梅花易数 | — | 具体问题（每次现场询问） |
| 六爻易经 | — | 具体问题（每次现场询问） |
| 吠陀占星 | 公历生日、出生时辰、出生地 | — |
| 奇门遁甲 | — | 具体问题、时间 |
| 风水 | — | 房屋朝向、楼层、布局 |

**收集信息时的交互要求：**

- 语气友善自然，不机械化
- 每次只问一个或一组相关字段，不要一口气抛出所有问题
- 示例用语：
  - "请告诉我你的出生日期 🎂"
  - "知道大概几点出生吗？不确定也没关系，可以用'不详'代替"
  - "性别是？（这会影响八字大运的排列方向）"
  - "有什么具体想问的问题吗？比如感情、事业、财运……直接说就好"

收集完成后，将新信息写入 `MEMORY.md`，然后进入 Step 4。

### Step 4：路由到对应 Skill

根据所选领域和 Skill 状态给出路由指引：

**已上线 Skill：**

告知用户使用对应 Skill 命令，格式：

> 已准备就绪！请使用 `/horoscope-daily` 查看你的星座运势 ✨

**开发中 Skill：**

友善告知状态，格式：

> 这个领域的 Skill（`bazi-fortune`）还在开发中 🚧，敬请期待！如果你想现在就了解，我可以基于已有知识为你提供一个简要分析，是否继续？

## User Context

本 Skill 的产出依赖以下用户信息，通过 `MEMORY.md` 跨领域、跨会话共享。

**读取**：每次触发时，**主动读取**本目录下的 `MEMORY.md`。文件存在且包含当前领域所需字段时，直接使用，不再重复询问。

**写入**：`MEMORY.md` 不存在或缺少必要字段时，按需引导用户提供，收集完成后写入 `MEMORY.md`。

**更新**：用户主动要求修改个人信息（如"我的生日填错了"）时，更新 `MEMORY.md` 中对应字段。

**MEMORY.md 格式：**

```markdown
# 用户玄学档案

## 基础信息
- 公历生日：1990-05-15
- 农历生日：庚午年四月廿一
- 出生时辰：未时（13:00-15:00）
- 出生地点：上海
- 性别：男
- 英文全名：San Zhang

## 星座信息
- 太阳星座：金牛座
- 星座区间：中段

## 已完成测算
- 2026-03-20: 星座日运势
```

**可持久化的字段说明：**

| 字段 | 说明 | 被哪些领域使用 |
|------|------|--------------|
| 公历生日 | YYYY-MM-DD 格式 | 星座运势、数字命理、八字、吠陀占星 |
| 农历生日 | 可由公历推算 | 紫微斗数 |
| 出生时辰 | 地支时辰或具体时间 | 八字、紫微斗数、吠陀占星 |
| 出生地点 | 城市名即可 | 八字（可选）、吠陀占星 |
| 性别 | 男/女 | 八字、紫微斗数 |
| 英文全名 | 护照/出生证上的全名 | 数字命理 |
| 太阳星座 | 由生日推算后缓存 | 星座运势 |

## 错误处理

| 错误类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 领域不识别 | 用户输入无法匹配任何领域 | 展示完整菜单，请用户重新选择 |
| 信息格式错误 | 生日格式不规范、时辰无法识别 | 给出正确格式示例，请用户重新输入 |
| MEMORY.md 读取失败 | 文件损坏或权限问题 | 提示用户重新提供信息，本次不写入 |
| 领域开发中 | 选择了尚未上线的 Skill | 友善说明，提供降级方案（基础分析） |

## 原子化设计

本 Skill 仅负责「运势测算入口路由」这一个原子化能力。不包含任何领域的具体测算功能。如需实际测算，请组合使用同仓库的对应 Skill（horoscope-daily、numerology-fortune 等），或直接调用目标领域 Skill。
