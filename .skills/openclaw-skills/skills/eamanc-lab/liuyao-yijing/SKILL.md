# 六爻易经占卜

基于京房纳甲体系，模拟铜钱起卦，完成纳甲装卦与生克制化断卦，为当下决策提供传统易学视角参考。

## 快速开始

```
"帮我起一卦"
"用六爻问一下这次投资的结果"
"铜钱起卦，问感情"
"六爻预测一下这次面试"
"帮我装一下这个卦：初爻少阳、二爻少阴..."
```

**完整示例**：

输入：`帮我起一卦，问这次跳槽的时机`

输出：

## User Context

六爻每次即时起卦，**不依赖用户个人生辰信息**。

**读取**：执行前依次检查：
1. 本目录 `MEMORY.md` — 优先使用（存储用户的起卦偏好和断卦风格）
2. 同仓库 `fortune-hub/MEMORY.md`（如果存在）— 补充共享基础字段

如有可用数据，直接使用，不再询问。

**写入**：仅当用户明确表达偏好时写入本目录的 `MEMORY.md`：

```markdown
# 用户偏好

## 起卦偏好
```

## 工作流程

### Step 1: 解析用户意图

从用户输入中识别起卦方式和问题主题：

| 参数 | 解析规则 | 默认值 |
|------|---------|--------|
| **起卦方式** | 用户说铜钱/掷铜钱 → 铜钱法；用户报数字 → 报数法；什么都没提供 → 铜钱法 | 铜钱法 |
| **问题/主题** | 用户描述的事项（事业/感情/财运/健康等） | 通用（无特定问题也可起卦） |
| **当前日期** | 用于确定月建、日辰、六神起排顺序 | 当前实际日期 |

### Step 2: 起卦

加载计算规则 [references/calculation-rules.md](references/calculation-rules.md)，按以下方式模拟起卦。

**必须严格按照规则计算，不可凭印象估算。** 每一步都要展示计算过程。

#### 铜钱法（默认）

模拟三枚铜钱掷六次，每次随机生成结果（正面=阳，反面=阴）：

| 组合 | 爻象 | 标记 | 说明 |
|------|------|------|------|
| 3 正 | 老阳 | ○ | 动爻，阳变阴 |
| 3 反 | 老阴 | × | 动爻，阴变阳 |
| 2 正 1 反 | 少阳 | — | 静爻，不变 |
| 1 正 2 反 | 少阴 | - - | 静爻，不变 |

**起卦展示格式（有仪式感）**：

```
🪙 掷铜钱中...

第一掷（初爻）：正正反 → 少阳 —
第二掷（二爻）：正反反 → 少阴 - -
第三掷（三爻）：正正正 → 老阳 ○ （动爻！阳→阴）
第四掷（四爻）：正反反 → 少阴 - -
第五掷（五爻）：正正反 → 少阳 —
第六掷（上爻）：反反反 → 老阴 × （动爻！阴→阳）

本卦 → 查六十四卦得：{卦名}
变卦 → 动爻阴阳互变后：{卦名}
```

#### 报数法

用户提供一个或多个数字时：

1. 单数：数字 ÷ 8 取余得上卦，数字 ÷ 6 取余得动爻，下卦从上卦+动爻推算
2. 双数：第一个数 ÷ 8 取余得上卦，第二个数 ÷ 8 取余得下卦，两数之和 ÷ 6 取余得动爻
3. 余数为 0 时取最大值（卦取 8，动爻取 6）

### Step 3: 装卦

加载计算规则 [references/calculation-rules.md](references/calculation-rules.md)，按以下顺序完成装卦。

#### 装卦顺序

1. **确定本卦和变卦**（由起卦结果确定）
2. **纳甲装天干**：查纳甲表，为本卦六爻装上天干
3. **纳支装地支**：查纳支表，为本卦六爻装上地支
4. **确定八宫归属**：查八宫六十四卦表，确定本卦属哪个宫
5. **确定世应位置**：查世应规则表，确定世爻和应爻在第几爻
6. **装六亲**：以世爻的五行为基准，按生克关系确定各爻六亲
7. **装六神**：以当日天干为起点，按青龙→朱雀→勾陈→螣蛇→白虎→玄武顺序，从初爻到上爻排列

#### 装卦展示格式

```
📋 装卦中...

本卦：{卦名}（{宫}宫，{世几世}）
当日日期：{年月日}，月建：{月支}，日辰：{日干支}

纳甲装天干：
乾卦（外）→ 甲壬 | 坤卦（内）→ 乙癸 | ...（按纳甲表）

六神起排：日干{X}，从初爻起青龙→朱雀→勾陈→螣蛇→白虎→玄武

世爻：第{N}爻 | 应爻：第{N}爻
世爻五行：{五行}，以此定六亲基准
```

### Step 4: 断卦

加载解读指南 [references/interpretation-guide.md](references/interpretation-guide.md)，按以下步骤断卦。

1. **定用神**：根据问题类型查用神选择表（求财→妻财，求官→官鬼...）
2. **看用神旺衰**：月建、日辰对用神地支的生克（旺/相/休/囚/死）
3. **看原神忌神**：原神（生用神）和忌神（克用神）是否在卦中出现，其旺衰如何
4. **看动爻影响**：动爻对用神的生克冲合，变爻是否持续影响
5. **综合判断**：整合以上信息，给出方向性结论

### Step 5: 行动建议

整合断卦分析，给出明确的方向判断和可执行的建议。

## 生成规则

**计算准确性**：
- 必须严格按照 calculation-rules.md 中的表格和规则计算，**绝不可凭印象估算**
- 铜钱法须逐爻展示掷卦结果，不可跳过
- 纳甲、纳支必须查表，不可凭记忆填写
- 世应位置必须查八宫归属后再查世应规则，不可猜测
- 六亲必须以世爻五行为基准按生克关系推导

**表达风格**：易学专业顾问型

- 先展示装卦过程再给断卦，有理有据
- 推导链必须展示（"用神妻财临月建，旺相有力，利财..."）
- 用"卦象显示""用神旺衰显示"而非"你一定"
- 负面结果必须附带建设性建议和可改变的时间窗口
- 六神六亲要解释其在当前问题中的具体含义

**禁用表达清单**：

| 禁止使用 | 替换为 |
|---------|-------|
| ❌ "此卦大凶，必然失败" | ✅ "用神受克，当前时机阻力较大，建议暂缓" |
| ❌ "你命中注定..." | ✅ "卦象在当下时机显示...的趋势" |
| ❌ "这件事绝对不行" | ✅ "当前格局不利，建议等待用神入旺之月" |
| ❌ "必须如何否则..." | ✅ "如果能...，局势可能有所改善" |

绝对禁止：绝对化断言、恐吓性语言、依赖性引导（"下次再来算"）。

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 用户没有提供起卦方式 | 默认使用铜钱法，告知用户并继续 |
| 用户提供了奇怪的数字组合 | 按报数法规则处理，展示计算过程并告知处理方式 |
| 用户没有具体问题 | 以"通用运势"为主题起卦，用神默认为官鬼或世爻 |
| 用户问梅花/塔罗/星座 | "建议使用 fortune-hub 选择对应领域 Skill，那里有专门的模块" |
| 用户质疑科学性 | "六爻是中国传统易学的重要分支，基于数千年观察总结的象数体系。可以把它当作当下时机的另一个观察角度 🪬" |

## 不适用场景

- **梅花易数** → meihua-yishu

- **数字命理** → numerology-fortune
