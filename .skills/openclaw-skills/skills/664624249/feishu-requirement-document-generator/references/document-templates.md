# 需求文档模板库

## 模板选择指南

### 按项目类型选择

| 项目类型 | 推荐模板 | 特点 |
|---------|---------|------|
| 业务系统 | BRD | 关注业务流程和规则 |
| 产品开发 | PRD | 关注用户体验和功能 |
| 技术系统 | TRD | 关注技术实现 |
| 综合项目 | SRS | 全面且结构化 |

### 按复杂度选择

- **简单需求：** 使用简化版模板
- **中等复杂：** 使用标准版模板
- **高度复杂：** 使用完整版模板

## 模板列表

### 1. 业务需求文档（BRD）
**用途：** 业务管理系统、ERP、CRM等
**文件：** [brd-template.md](brd-template.md)

**核心章节：**
- 业务背景
- 业务流程
- 功能需求
- 数据需求
- 非功能需求

### 2. 产品需求文档（PRD）
**用途：** 产品设计、APP开发、功能迭代
**文件：** [prd-template.md](prd-template.md)

**核心章节：**
- 产品概述
- 用户场景
- 功能详述
- 交互设计
- 验收标准

### 3. 技术需求文档（TRD）
**用途：** 技术系统、API开发、平台建设
**文件：** [trd-template.md](trd-template.md)

**核心章节：**
- 技术背景
- 系统架构
- 接口设计
- 性能要求
- 安全要求

### 4. 项目需求说明书（SRS）
**用途：** 综合性项目、定制开发
**文件：** [srs-template.md](srs-template.md)

**核心章节：**
- 项目概述
- 详细需求
- 设计方案
- 实施计划
- 验收标准

## 模板使用建议

### 首次使用
1. 阅读模板说明
2. 了解各章节用途
3. 根据实际情况调整
4. 删除不适用的章节

### 批量使用
1. 建立模板库
2. 统一文档标准
3. 定期更新模板
4. 收集使用反馈

### 自定义模板
1. 基于标准模板
2. 添加行业特定内容
3. 调整章节顺序
4. 优化表述方式

## 文档规范

### 格式规范
- 使用Markdown格式
- 标题层级不超过4级
- 表格简洁明了
- 适当使用图表

### 内容规范
- 客观准确描述
- 避免模糊用语
- 标注待确认事项
- 保留变更历史

### 审核规范
- 内部审核要点
- 客户确认流程
- 变更管理机制
- 版本控制方法
