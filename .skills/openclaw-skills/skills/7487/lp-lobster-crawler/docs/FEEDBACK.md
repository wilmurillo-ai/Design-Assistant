# FEEDBACK

> 用户反馈日志。每条记录一个实际遇到的问题或改进要求。
> Agent 每轮执行前必须阅读此文件，对 `status: open` 的条目进行自我修正。

---

## 反馈格式

```
### Fxxx — 简短标题
- **status**: open | resolved
- **date**: YYYY-MM-DD
- **category**: code-quality | docs | deps | workflow | bug | other
- **description**: 具体问题描述
- **root-cause**: 问题根因（可选，Agent 分析后补充）
- **action**: 要求的修正行为
- **resolved-by**: Txxx 或说明（resolved 后填写）
```
