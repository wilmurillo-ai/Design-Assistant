## 🔧 配置步骤

### 步骤 1：安装依赖

```bash
cd ~/.openclaw/skills/ssh-remote
npm install
```

这会安装 `ssh2` 模块，用于 SSH 连接。

### 步骤 2：创建服务器配置

```bash
mkdir -p ~/.openclaw/skills/ssh-remote/configs
```

创建 `configs/servers.json`:

```json
{
  "servers": [
    {
      "name": "production",
      "host": "192.168.1.100",
      "port": 22,
      "username": "root",
      "privateKeyPath": "/root/.ssh/id_rsa"
    },
    {
      "name": "staging",
      "host": "192.168.1.101",
      "port": 22,
      "username": "ubuntu",
      "password": "your_password"
    },
    {
      "name": "dev",
      "host": "dev.example.com",
      "port": 2222,
      "username": "deploy",
      "privateKeyPath": "/root/.ssh/deploy_key"
    }
  ]
}
```

### 步骤 3：配置 SSH 密钥（推荐）

```bash
# 生成 SSH 密钥（如果没有）
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa

# 复制公钥到服务器
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.1.100
```

### 步骤 4：测试连接

```bash
# 测试 SSH 连接
ssh -i ~/.ssh/id_rsa root@192.168.1.100
```

### 步骤 5：重启 Gateway

```bash
openclaw gateway restart
```

## ⚠️ 常见问题

### Q: 提示 "ssh2 模块未找到"
**A:** 运行 `npm install` 安装依赖

### Q: 提示 "连接超时"
**A:** 检查服务器 IP、端口、防火墙设置

### Q: 提示 "认证失败"
**A:** 检查用户名、密码或私钥配置

### Q: 提示 "权限拒绝"
**A:** 确保 SSH 用户有执行命令的权限

### Q: 文件上传失败
**A:** 检查远程目录权限和磁盘空间

### 定时监控

```javascript
// 每 5 分钟检查一次服务器状态
const cron = require('node-cron');
const { getSystemAlerts } = require('./src/monitor');

cron.schedule('*/5 * * * *', async () => {
  const alerts = await getSystemAlerts('production');
  if (alerts.hasAlerts) {
    console.log('⚠️ 服务器告警:', alerts.alerts);
  }
});
```
