# 执行计划模板

> **文档职责**：plan = 怎么做（技术方案、实现路径、任务拆解）。承接 spec.md 的业务需求，输出完整技术方案，是 task.md 生成的直接输入。
> plan.md 面向数仓开发同学，聚焦「技术实现」「任务拆解」「依赖哪些规范」「风险在哪里」。
>
> ⛔ **plan.md 不包含以下内容**（属于 task.md 职责）：
> - SQL 代码 / 伪代码
> - .job 文件配置
> - 完整 DDL 语句
>
> ⚠️ **规范声明**：所有项目特定配置（分层命名、存储格式、分区规范、调度参数、资源档位等）以 `resources/conventions/project-conventions.md` 和 `resources/conventions/platform-conventions.md` 为唯一依据。模板各节只提供结构和引用指针，不重复定义规范值。填写前须先读取对应公约文件。
