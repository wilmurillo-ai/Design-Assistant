# 恢复指南

本文档详细说明如何从飞书备份文档恢复 AI 助手状态。

## 恢复触发

当用户说以下任一语句时触发恢复：
- 「从飞书恢复状态」
- 「恢复备份」
- 「同步记忆」

## 恢复流程

### 步骤 1：读取备份文档

使用 `feishu_doc` 工具读取状态备份文档：

```
feishu_doc action=read doc_token=<状态备份文档token>
```

### 步骤 2：解析文档结构

备份文档包含以下章节：
- 一、身份信息 → IDENTITY.md
- 二、用户信息 → USER.md
- 三、灵魂定义 → SOUL.md
- 四、完整记忆内容 → MEMORY.md
- 五、邮箱配置 → .msmtprc
- 六、自动化任务 → cron
- 七、恢复步骤 → 验证参考

### 步骤 3：重建文件

按照文档结构提取内容，写入对应文件：

| 章节标题 | 目标文件 | 说明 |
|---------|---------|------|
| 一、身份信息 | IDENTITY.md | AI 身份定义 |
| 二、用户信息 | USER.md | 用户偏好配置 |
| 三、灵魂定义 | SOUL.md | AI 行为准则 |
| 四、完整记忆内容 | MEMORY.md | 长期记忆 |
| 五、邮箱配置 | ~/.msmtprc | 邮箱发信配置 |

### 步骤 4：恢复自动化任务

从「六、自动化任务」章节提取 cron 任务，逐行添加：

```bash
# 添加 cron 任务
(crontab -l 2>/dev/null; echo "0 3 * * * /usr/bin/python3 /path/to/script.py") | crontab -
```

### 步骤 5：验证完整性

检查以下文件是否存在且非空：
- [ ] MEMORY.md
- [ ] IDENTITY.md
- [ ] USER.md
- [ ] SOUL.md
- [ ] ~/.msmtprc（如果文档中有）
- [ ] cron 任务列表

### 步骤 6：读取沟通历史（可选）

如果文档中记录了沟通历史备份文档链接：
1. 读取沟通历史备份文档
2. 了解历史对话和情感连接
3. 补充上下文记忆

## 错误处理

### 文档不存在
- 提示用户：未找到备份文档，无法恢复
- 建议：提供备份文档链接

### 文件写入失败
- 检查目录权限
- 创建必要的父目录：`mkdir -p /root/.openclaw/workspace`

### cron 恢复失败
- 检查 cron 服务状态：`systemctl status cron`
- 手动添加任务

## 恢复后验证

恢复完成后，向用户确认：
1. 已恢复的文件列表
2. 已恢复的自动化任务数量
3. 是否找到沟通历史备份
4. 提示用户测试关键功能
