# AI 协作复盘 Skill

一个工具无关的 AI 辅助编程会话复盘框架。适用于任何能访问对话上下文的 AI 编程助手。

每次 AI 辅助开发结束后，触发一次系统化的 **八维度** 复盘分析，识别改进机会，生成结构化复盘报告。

**核心目标：让每一次 AI 协作都比上一次更高效。**

## 为什么需要这个

大多数开发者用 AI 的方式是被动的 —— 写代码、遇到问题、继续推进。很少有人停下来想一想：*"这次协作本可以更好吗？哪些模式一直在浪费我们的时间？"*

这个 Skill 把这种反思变成一个结构化、可重复的流程。它分析你和 AI 助手之间的对话，找到具体的浪费点，量化影响，给出可执行的改进建议。

## 八个分析维度

| # | 维度 | 分析内容 |
|---|------|---------|
| 1 | **AI 自我反思** | AI 的失误、延迟反应、遗漏判断（最高优先级，首先分析） |
| 2 | **验证策略** | AI 是否主动定义了验证标准，还是被动等待反馈 |
| 3 | **自动化机会** | 重复性工作流，可以封装成可复用的自动化 |
| 4 | **现有自动化调优** | 会话中使用的自动化/Skill/规则存在的缺口 |
| 5 | **工具集成机会** | 需要专用工具、插件或 API 连接的操作 |
| 6 | **知识沉淀** | 值得持久化的偏好、约定和决策，供未来会话使用 |
| 7 | **文档更新** | 需要更新的项目文档、规范或架构笔记 |
| 8 | **工作流效率** | 可以并行的串行步骤、重复劳动、次优工具选择 |

## 支持的 AI 工具

本 Skill 设计为工具无关，适用于任何 AI 编程助手。已测试的工具：

| 工具 | 安装方式 |
|------|---------|
| **WorkBuddy** | 复制到 `.workbuddy/skills/ai-retrospective/` |
| **Cursor** | 将 SKILL.md 内容复制到 `.cursor/rules/` 或在项目设置中引用 |
| **Claude Code** | 追加到 `CLAUDE.md` 或作为 slash command 引用 |
| **GitHub Copilot** | 在 `.github/copilot-instructions.md` 中引用 |
| **Windsurf** | 在 `.windsurfrules` 或 Cascade 工作流中引用 |
| **Cline** | 在 `.clinerules` 或自定义指令中引用 |

Skill 支持优雅降级 —— 如果你的工具缺少某些能力（比如不能写文件、不能持久化记忆），分析步骤照常工作，只有输出/持久化步骤会自适应调整。

## 快速开始

### 1. 安装

```bash
# 克隆到你的项目（或全局使用）
git clone https://github.com/AmosHC/ai-retrospective-skill.git

# 或者直接复制文件
cp -r ai-retrospective-skill/ your-project/.ai-retrospective/
```

### 2. 触发

在任何 AI 辅助编程会话结束时，说：
- "复盘" / "回顾一下"
- "retrospective" / "retro"
- "review this session"

### 3. 获取报告

Skill 会：
1. 扫描对话历史，构建事件时间线
2. 标记浪费点（可避免的步骤、延迟操作、重复工作）
3. 按八个维度进行分析，关联具体事件
4. 在对话中输出完整报告，同时保存到文件
5. 自动沉淀知识条目（如果工具支持）
6. 列出待确认的行动项

## 工作原理

本 Skill 是**纯 LLM 指令驱动**的 —— 没有脚本、没有外部依赖、不需要 API Key。完全通过结构化的 prompt 引导 AI 助手完成分析工作流。

核心洞察：你的 AI 助手已经在上下文中拥有完整的对话历史。这个 Skill 只是提供了一个系统化的框架，让 AI 去分析这段历史并提取可执行的洞察。

### 工作流概览

```
会话结束 → 触发词 → 步骤1: 提取事件 & 标记浪费
→ 步骤2: 八维度分析 → 步骤3: 生成报告文件
→ 步骤4: 在对话中显示完整分析 → 步骤5: 自动沉淀知识
→ 步骤6: 列出待确认行动项
```

## 适配指南

### 按工具安装

每个 AI 工具有自己的自定义指令约定。核心内容在 `SKILL.md` 和 `references/analysis_dimensions.md` 中。根据工具调整安装方式：

- **WorkBuddy**：直接使用 —— 目录结构就是 Skills 格式
- **Cursor**：创建 `.cursor/rules/retrospective.md`，合并 SKILL.md + analysis_dimensions.md 的关键部分
- **Claude Code**：在 `CLAUDE.md` 中追加「复盘」章节，包含工作流和维度摘要
- **其他工具**：在工具读取自定义指令的位置，放入 SKILL.md 的内容

### 自定义维度

八个维度设计为全面但不穷尽。你可以：
- 添加领域特定维度（如安全敏感项目加入「安全审查」维度）
- 调整各维度的优先级权重，匹配你的工作流
- 根据经验添加新的自检模式

### 自定义报告

编辑 `assets/report_template.md` 以匹配你的团队规范。模板使用占位符变量，由 AI 在生成报告时填充。
