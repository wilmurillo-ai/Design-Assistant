# Enforcement Checklist v2.1

Use this before sending any completion update.

## Source-of-Truth Gate
- [ ] Domain identified
- [ ] Authoritative Cerebro file(s) read
- [ ] Action mapped to explicit rule/section

## Execution Gate
- [ ] Work performed in correct project/domain context
- [ ] Privacy/routing constraints respected
- [ ] No undocumented assumptions used

## Proof Gate (required for "Done")
- [ ] Artifact path(s)
- [ ] Verification command/output
- [ ] If deploy/publish: resulting URL/version/id

## Write-Back Gate
- [ ] Cerebro updated if process/rules changed
- [ ] Daily memory updated with what/why/path

## Failure Handling
If any box is unchecked, status must be `In Progress` (not `Done`).
