# 首次建档与冲刺档案生成

本文件负责用户第一次使用本 skill 时的信息采集与冲刺档案建立。它只处理“先把情况摸清楚”这件事，不直接承担后续每日讲解与进度更新。

## 目标

系统需要在首次输入阶段尽快明确：

- 课程名称；
- 考试日期与剩余天数；
- 课程材料有哪些；
- 每天可投入的学习时间；
- 用户当前掌握程度；
- 明显薄弱章节或题型；
- 目标分数或最低目标。

如果信息不完整，也可以先建立初版档案，但必须标注哪些信息仍缺失，后续补齐后再细化。

## 处理原则

第一，优先提取与计划生成直接相关的信息，不在首次建档阶段展开大量讲解。第二，要把零散输入整理成结构化冲刺档案，便于后续 20 天调度使用。第三，如果用户上传了教材、PPT、笔记、往年题，系统应先判断这些材料分别服务于哪一类章节或题型，而不是简单按上传顺序使用。

## 推荐输出字段

### 冲刺档案至少包含
- 课程名；
- 考试时间；
- 剩余天数；
- 每日可用时间；
- 资料清单；
- 当前基础判断；
- 重点薄弱项；
- 目标结果；
- 是否已具备生成完整计划的条件。

## 课程识别原则

系统不应把课程预先写死，而应先判断课程更接近哪种类型，再决定后续计划与讲解方式：

- 理论型课程；
- 计算或推导型课程；
- 实验或编程型课程；
- 理论与计算混合型课程。

这一步只做识别和归类，不需要在这里展开详细复习内容。
