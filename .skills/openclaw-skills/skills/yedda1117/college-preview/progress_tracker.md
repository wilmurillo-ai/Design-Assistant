# 进度跟踪与动态调整

本文件负责维护学生自己的进度面板、每日记录与后续几天的局部调整逻辑。它不是家长监管面板，而是系统读取当前复习状态并决定后续如何推进的核心模块。

## 一、初始进度面板

首次生成计划后，要同步建立一份个人进度面板，至少包含以下信息：

- 当前是第几天；
- 20 天总进度；
- 各章节或知识板块当前掌握状态；
- 最近一次小测结果；
- 当前连续学习天数；
- 需要优先补强的知识点。

## 二、每日记录更新

当用户提交“今天完成了 80%，错了 2 道，某知识点还不稳”时，应把这些信息写入对应的日记录，并更新总面板中的知识点状态和风险提示。

### 日记录应包含
- 日期；
- 计划内容；
- 实际完成情况；
- 小测表现；
- 仍不清楚的知识点；
- 系统给出的次日建议。

### 进度面板应包含
- 章节或知识板块：已完成 / 进行中 / 薄弱；
- 阶段：基础梳理 / 重点突破 / 综合训练 / 考前回顾；
- 最近三天风险提醒；
- 是否建议调整未来 3 天计划。

## 三、读取逻辑

当用户发送“开始今天任务”时，系统先读取当前进度面板和上一天记录，再决定今天应正常推进、局部补强，还是压缩其他部分给重点让路。

## 四、节奏维持

### 连续学习
记录连续学习天数，但中断不做惩罚，只作为节奏提示。重点不是制造压力，而是帮助用户看到自己是否真正保持了复习连续性。

### 阶段完成提醒
每完成一个阶段，都应给出简短回顾，例如“基础梳理阶段已结束，接下来会把更多时间转向重点章节和综合题训练”。

### 低负担反馈
鼓励语应简短、自然，不夸张。更适合的方式是指出已经完成的内容和下一步重点，而不是用大量情绪化口号。
