# 字段说明

本文档详细说明涨跌停分析中的关键字段。

## 核心字段

### zt_days（涨停天数/连板数）

- **定义**：连续涨停的交易日数量
- **数值范围**：1, 2, 3, ...
- **解读**：
  - `zt_days = 1`：首板，当日首次涨停
  - `zt_days = 2`：2连板，连续两日涨停
  - `zt_days >= 3`：高标股，市场关注度高

**应用场景**：
- 连板股通常为板块龙头
- 连板数越高，市场情绪越强
- 但同时也意味着调整风险增加

---

### CS（Close-to-EMA Strength）

- **定义**：收盘价相对 EMA 的偏离强度
- **数值范围**：通常 -50 到 +50
- **解读**：
  - `CS > 0`：价格在 EMA 之上，相对强势
  - `CS < 0`：价格在 EMA 之下，相对弱势
  - `CS 越大`：强度越高

**应用场景**：
- 筛选强势涨停股
- 判断股票短期走势强度

---

### SCTR（StockCharts Technical Rank）

- **定义**：个股技术综合排名，综合多个技术指标（均线、动量、相对强度等）对全市场股票打分排名
- **数值范围**：0-100
- **解读**：
  - `SCTR = 60`：技术面强于市场 60% 的股票
  - `SCTR > 90`：全市场前 10%，技术面极强
  - `SCTR > 80`：全市场前 20%，技术面偏强
  - `SCTR < 30`：技术面偏弱，谨慎参与

**应用场景**：
- 识别技术面强势的涨停股
- 辅助判断个股是否具备持续上涨的技术基础
- 高 SCTR + 高连板数 = 龙头股的重要特征

---

## 分类字段

### industry（所属行业）

- **定义**：股票所属的行业分类
- **示例**：电子、医药生物、计算机、机械设备等

---

### concept（所属概念）

- **定义**：股票所属的概念标签数组
- **示例**：["人工智能", "ChatGPT", "算力"]

**注意事项**：
- 需排除无意义概念：机构重仓、AB股、涨跌停等
- 关注核心题材概念

---

## 统计指标

### 统计（N天M次涨停）

- **定义**：描述股票在近期窗口内的涨停历史，格式为 `"N天M次涨停"`
- **示例**：`"6天3次涨停"`、`"5天5次涨停"`、`"1天1次涨停"`、`"0天0次涨停"`
- **解读规则**：
  - `N == M`（如 `5天5次`）：真正的连续涨停，即连板股，连板数 = N
  - `N > M`（如 `6天3次`）：近 N 天内涨停了 M 次，中间有过炸板或未涨停的交易日，**不是连板股**
  - `N == 0`（如 `0天0次`）：出现在炸板池中，表示今日曾涨停但收盘未封住，历史上无近期连板记录

**重要区分**：

| 统计字段 | 含义 | 是否连板 |
|---------|------|--------|
| `5天5次涨停` | 连续5日涨停 | ✅ 是，连板数=5 |
| `6天3次涨停` | 6天内涨停3次，中间有断板 | ❌ 否 |
| `1天1次涨停` | 今日首板 | ✅ 是，连板数=1 |
| `0天0次涨停` | 炸板股，今日曾涨停未封住 | ❌ 否 |

**Agent 使用注意**：判断连板数时，必须同时满足 `N == M` 才能认定为连板，不能直接用 N 或 M 单独作为连板天数。

---

### 涨停数量

- **定义**：当日涨停股票总数
- **用途**：判断市场情绪

### 跌停数量

- **定义**：当日跌停股票总数
- **用途**：判断市场风险

### 涨跌停比例

- **定义**：涨停数 / 跌停数
- **解读**：
  - `比例 > 3:1`：市场情绪偏多
  - `比例 < 1:1`：市场情绪偏空
  - `比例约 1:1`：市场分歧较大

## 术语对照表

| 术语 | 英文 | 说明 |
|------|------|------|
| 涨停 | Limit Up | 股价涨幅达到当日上限（通常 10%） |
| 跌停 | Limit Down | 股价跌幅达到当日下限（通常 10%） |
| 炸板 | Broken Limit | 曾涨停但收盘未封住 |
| 连板 | Consecutive Limit | 连续多日涨停 |
| 首板 | First Limit | 首次涨停 |
| 龙头股 | Leading Stock | 板块内表现最强的股票 |
