# 个股分析大师 Skill

围绕"选股 + 选时 + 情绪资金"三条主线，以"短期看情绪，长期看价值"为核心，支持单只A股深度分析与多只A股横向对比；优先识别用户是综合对比、技术面、财报分析还是价值面对比意图，不明确时默认综合对比。

## Overview（功能概述）

该 Skill 通过 `daxiapi-cli` 与现有 xiapi 专项 Skill 协同，完成个股基本面、技术面、量化行为金融与风险提示的一体化分析。

输出强调：结论先行、数据可追溯、风险可落地，不给出确定性收益承诺。

## When to Use（何时使用）

当用户需要以下能力时触发：

- 分析某只股票是否值得长期持有（价值分析）
- 分析某只股票当前是否适合买入/减仓/观望（技术分析）
- 对 2-8 只股票做综合对比，输出相对强弱与优先级
- 跟随用户意图只做技术面对比、财报对比或价值面对比
- 对 ST/退市风险股做重整预期与摘帽概率分析

**具体触发词**：个股分析、个股对比、财报对比、价值分析、技术分析、基本面、买卖点、选股、选时、ST分析、重整预期、摘帽

## When Not to Use（何时不使用）

以下场景不应使用本 Skill：

- 用户未提供股票代码/公司名且无法确认标的
- 非A股标的（美股、港股、基金、期货、数字货币等）
- 一次性要求超大规模全市场对比（超出可读范围）
- 仅要求"马上给买卖指令/仓位指令"的高频交易场景
- 纯理论问答（如"MACD是什么"）

## Process（流程主体）

### Step 0: 前期准备（Token 与标的集合确认）

**触发条件**：首次使用、Token 未配置、或命令返回认证错误

**跳过条件**：Token 已配置且有效，且股票标的（单股或多股）已明确

**完成标准**：CLI 能正常返回目标股票数据，分析对象明确（单股或多股列表）

**步骤 0.1：检查 Token 配置**

```bash
npx daxiapi-cli@latest config get token
```

**步骤 0.2：如未配置，指导用户完成配置**

```bash
npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI
# 或
export DAXIAPI_TOKEN=YOUR_TOKEN_FROM_DAXIAPI
```

**步骤 0.3：确认标的代码（优先代码）**

```bash
npx daxiapi-cli@latest search <公司名> --type stock
```

- 单股模式：确认 1 个代码
- 对比模式：确认 2-8 个代码（建议不超过 8，避免信息过载）

**步骤 0.4：验证数据可用性**

```bash
# 单股
npx daxiapi-cli@latest stock info <code>

# 多股（CLI 支持逗号分隔或空格输入）
npx daxiapi-cli@latest stock info <code1>,<code2>,<code3>
```

### Step 1: 意图识别（综合 vs 技术 vs 财报 vs 价值）

**目标**：先判断用户需求是单股深度还是多股对比，并识别分析维度优先级

**判断规则**：

1. 先判断任务形态：
    - 单股深度：1 只股票
    - 多股对比：2 只及以上股票
2. 从用户当前问题提取关键词：
    - 综合对比关键词：综合对比、谁更好、优先级、组合选择
    - 技术面对比关键词：均线、MACD、RSI、K线、支撑位、压力位、买卖点
    - 财报对比关键词：财报、ROE、利润、现金流、负债率、增长质量
    - 价值面对比关键词：估值、低估、高估、安全边际、长期价值、护城河
3. 结合上下文判断主意图（可主次并存）
4. 若无法明确，默认"综合对比"

**分析侧重参考（单股）**：

- 价值主导：以基本面为主，技术面为辅，资金情绪做参考
- 技术主导：以技术面为主，基本面为辅，资金情绪做参考
- 财报主导：以财务质量为主，技术面为辅，资金情绪做参考
- 不明确：基本面与技术面并重，资金情绪做补充

**分析侧重参考（多股对比）**：

- 综合对比：基本面与技术面并重，财报质量次之，资金情绪做补充
- 技术面对比：以技术面为主，基本面与资金情绪为辅
- 财报对比：以财务质量为主，技术面与资金情绪为辅
- 价值面对比：以估值与护城河为主，财务质量次之，技术面做参考

**完成标准**：已明确任务形态与分析侧重点，并记录原因

### Step 2: 按意图收集数据（CLI + xiapi Skills）

**目标**：围绕主意图收集可分析数据；多股对比时保证各标的数据口径一致

> **详细命令参考**：[CLI 命令参考](references/cli-commands.md)

#### 2.1 基础信息与行情

```bash
# 单股
npx daxiapi-cli@latest stock info <code>
npx daxiapi-cli@latest kline <code>

# 多股（优先一次性拉取快照）
npx daxiapi-cli@latest stock info <code1>,<code2>,<code3>
# kline 按股票逐个拉取，保证时间区间一致
npx daxiapi-cli@latest kline <code1>
npx daxiapi-cli@latest kline <code2>
```

- 用于识别股票属性、近期价格行为、趋势结构
- 对比模式下必须统一观察窗口（如最近20/60个交易日）

#### 2.2 基本面与财务（价值分析/财报对比优先）

```bash
# 单股
npx daxiapi-cli@latest report finance <code>

# 多股（逐只执行，保持同一报告期口径）
npx daxiapi-cli@latest report finance <code1>
npx daxiapi-cli@latest report finance <code2>
```

- 必须联动 `xiapi-financial-roe-analysis` 的分析结论
- 重点抽取：ROE、毛利率、净利率、营收/净利增速、资产负债率、现金流质量
- 财报对比时必须注明口径（年报/季报/TTM）

#### 2.3 行业与市场主线（判断是否顺势）

```bash
npx daxiapi-cli@latest sector stocks --code <BK_CODE> --order cs
npx daxiapi-cli@latest sector heatmap --order cs --limit 20
npx daxiapi-cli@latest market temp
```

- 用于识别行业景气、板块强弱、市场风险偏好

#### 2.4 技术与形态（技术分析优先，K线为核心）

```bash
npx daxiapi-cli@latest stock info <code>
npx daxiapi-cli@latest kline <code>
npx daxiapi-cli@latest stock pattern trendUp
npx daxiapi-cli@latest stock pattern crossMa50
npx daxiapi-cli@latest stock pattern vcp
```

- 先用 `stock info` 做"强弱快照"，再用 `kline` 做"指标复算与形态确认"
- 以 `daxiapi stock info 600031` 为例，优先解读以下字段（字段含义见 `field-descriptions.md`）：
  - `zdf / zdf5 / zdf10 / zdf20`：判断1日到20日动量是否连续
  - `CS`（或 `bais`）：价格相对20日EMA乖离，判断是否过热或跌破趋势
  - `IBS`：收盘是否接近当日高点，判断当日K线收盘质量
  - `SCTR`：市场相对强弱排名，判断强势股属性是否成立
- 若 `stock info` 未提供某技术指标，必须以 `kline` 数据自行计算后再判断（禁止凭感觉补结论）
- 用于确认趋势方向、关键形态、突破有效性与信号一致性

#### 2.5 资金与情绪（量化行为维度）

```bash
npx daxiapi-cli@latest hotrank stock --type hour --list-type normal
npx daxiapi-cli@latest turnover
npx daxiapi-cli@latest zdt --type zt
```

- 用于判断市场关注度、换手活跃度与情绪拥挤度

#### 2.6 宏观政策与官方信息（必要时）

- 行业政策、监管导向、公司公告优先参考官方来源（如巨潮资讯）
- 若外部新闻无法稳定获取，必须明确写出"该部分数据不足，结论降级"

**完成标准**：已获取支持三大维度分析的数据；缺失项已明确标注

### Step 3: 执行分析与多股对比

**目标**：形成"选股 + 选时 + 博弈状态 + 风险收益"的完整判断，并在多股模式下给出可解释的对比结论

> **信号判断规则**：[策略逻辑信号表](references/strategy-signals.md)

#### 3.1 第一步：基本面分析（选股）

1. 宏观与行业景气度
    - 宏观周期对行业影响（复苏/过热/滞胀/衰退）
    - 政策导向（扶持/监管/中性）
    - 行业生命周期与竞争格局
2. 公司财务健康度（复用 ROE Skill 结果，精简汇总）
    - 盈利能力：ROE、毛利率、净利率趋势
    - 成长能力：营收和净利润增长
    - 偿债能力：资产负债率、流动/速动能力
    - 运营能力：存货/应收周转效率
3. 护城河与治理
    - 品牌、技术、规模、网络效应、特许经营能力
    - 管理层执行力与治理稳定性
4. 估值水平
    - 相对估值：PE/PB 历史分位
    - 绝对估值：仅在参数充分时给 DCF 方向性判断

#### 3.2 第二步：技术分析（选时，以K线复算为核心）

1. 快照判读（来自 `stock info`）
   - `zdf / zdf5 / zdf10 / zdf20`：短中期涨跌幅是否同向，判断趋势连续性
   - `CS`（或 `bais`）：
     - `0~15` 常见于健康偏强区间
     - `>15` 警惕短期偏离过大
     - `<-5` 视为明显跌破20EMA的风险区
   - `IBS`：高位收盘（接近100）通常强于低位收盘（接近0）
   - `SCTR`：高SCTR代表相对强势，但需与趋势和量能共同验证
2. K线趋势研判（来自 `kline`）
   - 中长期：基于K线计算 MA60/MA120/MA250，确认大级别方向
   - 短期：基于K线计算 MA5/MA10，识别短线节奏与拐点
3. 形态分析（来自 `kline` + `stock pattern`）
   - 使用 `trendUp / crossMa50 / vcp` 作为形态信号参考
   - 同时在K线中确认经典形态（W底、M头、三角形、旗形）与关键支撑/压力位
4. 指标复算（当CLI未直接返回时）
   - 使用K线数据计算常用技术指标（如 MACD、RSI、KDJ 等），与形态信号交叉验证
   - 基于成交量与价格同步性判断量价关系（放量突破/缩量回踩/量价背离）

#### 3.3 第三步：量化与行为金融（情绪与资金）

1. 资金流向
    - 主力资金净流向（daxiapi stock capital-flow <code>）
    - 北向资金持仓变化（若数据可得）
    - 融资融券变化（若数据可得）
2. 筹码与持仓结构
    - 当前价格与筹码峰关系（若数据可得）
    - 股东户数与人均持股变化（若数据可得）
3. 市场情绪
    - 换手率是否异常
    - 波动率变化
    - 热榜与新闻情绪倾向
4. 行为偏差观察
    - 关注是否存在羊群效应、锚定效应、拥挤交易等行为偏差迹象

#### 3.4 第四步：综合建议与风险提示

1. 选股结论：质地、主线契合度、长期价值
2. 选时结论：建仓期/持股期/减仓期/观望期
3. 综合评级：强烈推荐/谨慎推荐/中性/观望/回避
4. 高风险个股硬规则（必执行）：
   - 若出现"基本面恶化 + 技术破位 + 情绪退潮"中的两项及以上，必须给出明确结论：`不建议持有`
   - 必须给出至少 3 条可验证原因（示例：盈利持续下滑、CS长期为负且跌破关键均线、IBS低位弱收盘、行业景气下行、重大治理或合规风险）
5. 风险提示（必填）：
   - 数据与历史规律存在失效风险
   - 黑天鹅、政策、业绩不及预期等风险
   - 明确"仅供参考，不构成投资建议"

#### 3.5 多股对比矩阵（2只及以上触发）

**目标**：把多只股票放到统一框架中横向比较，输出"谁更强 + 为什么"。

1. 对比维度（随意图切换）：
   - 综合对比：基本面、技术面、财报质量、资金情绪
   - 技术面对比：趋势连续性、强弱指标、形态信号一致性
   - 财报对比：盈利质量、成长质量、偿债与现金流安全性
   - 价值面对比：估值分位、安全边际、护城河稳定性
2. 输出方式：
   - 建议给出对比表（每只股票一行）
   - 建议给出维度冠军与综合优先级（Top1/Top2/Top3）
   - 建议解释"为什么不是另一只"（至少1条反证）
3. 并列处理：
   - 若关键指标接近，不强行分胜负，可输出"同级候选"
   - 明确触发下一步决策所需补充数据

### Step 4: ST/退市风险股增强分析（条件触发）

**触发条件**：股票名称含 ST/\*ST，或用户明确提及退市/重整/摘帽

**增强维度**：

1. ST 原因与困境本质（连续亏损/规范问题/违规等）
2. 重整重组预期（进度、投资人、资产注入可能）
3. 摘帽可能性（条件、时间、催化剂）
4. 壳价值（行业、股本结构、注册地）
5. 风险收益比（最大风险、潜在收益、主观胜率）
6. 市场博弈（游资、散户情绪、机构态度）

**评级修正规则**：

- 强烈推荐：重整路径清晰且关键条件可验证
- 谨慎推荐：有预期但关键环节不确定
- 中性：博弈主导，基本面支撑弱
- 观望/回避：重整希望低或退市风险高

### Step 5: 输出结构化报告（含对比模式）

**目标**：按统一模板输出，避免信息堆砌

> **详细报告模板**：[报告模板](assets/report-template.md)

建议按以下顺序输出，可根据用户需求灵活调整：

1. 核心结论（先给结论）
2. 分析对象与数据时间
3. 基本面分析
4. 技术分析
5. 量化与行为金融分析
6. 多股对比结论（对比模式建议输出：对比表 + 优先级 + 反证）
7. 综合评级与交易阶段判断
8. 高风险结论（如触发，建议写明"不建议持有"及原因）
9. 关键风险提示
10. 免责声明
11. 互动追问引导（引导用户继续提问个股信息与解释需求）

> **必填项**（不可省略）：核心结论（1）、风险提示（9）、免责声明（10）。其余可根据数据可得性和用户关注点灵活裁剪。

## Quality Checks（质量检查）

### 必查项

- [ ] 已明确分析标的（名称+代码）
- [ ] 已识别用户意图（综合/技术/财报/价值）
- [ ] 至少覆盖"基本面 + 技术面"两个核心维度（对比模式按意图执行）
- [ ] 对比模式已输出对比表、优先级与反证
- [ ] 有数据支撑，不出现"无依据结论"
- [ ] 包含风险提示与免责声明
- [ ] 若为 ST，已输出增强分析模块

### Red Flags（危险信号）

| 危险信号           | 说明                   | 处理方式               |
| ------------------ | ---------------------- | ---------------------- |
| 数据缺失却强下结论 | 缺关键字段仍给确定判断 | 明确降级分析边界       |
| 指标口径混用       | 年报、季报、TTM 混用   | 标注口径并避免直接横比 |
| 绝对化措辞         | "一定涨""必翻倍"       | 改为概率与情景表达     |
| 缺少风险提示       | 只讲机会不讲风险       | 强制补充至少 3 条风险  |
| ST漏检             | ST 股票未做增强框架    | 触发 Step 4 重新分析   |

## Common Pitfalls（常见陷阱）

- 只做技术分析，忽略基本面变化导致"选时正确、选股错误"
- 多股对比时混用不同报告期口径，导致结论失真
- 只看财务指标，不看行业景气与政策方向
- 用单日波动解释长期价值
- 在数据不全时给出过度确定的评级
- 把 ST 博弈当作普通成长股逻辑分析

## Gotchas（闭坑指南）

> **详细错误处理**：[错误处理指南](references/error-handling.md)

### 标的不明确
- 先要求用户补充股票代码或公司全称
- 使用 `search` 辅助确认后再继续

### Token 认证失败（401）
- 错误示例：`Error: Authentication failed (401)`
- 解决方案：执行 `config get token` 检查，缺失或失效时重新配置

### 请求频率超限（429）
- 错误示例：`Error: Rate limit exceeded (429)`
- 解决方案：等待 30-60 秒后重试，避免短时间高频请求

### 数据缺失/接口异常
- 错误示例：`Error: No data available for code: XXXXXX`
- 解决方案：明确缺失字段与影响范围，用户确认后执行"降级分析"

### 多股对比口径不一致
- 常见表现：A 用年报、B 用季报，或A时间窗60日、B时间窗20日
- 解决方案：统一财报口径与K线时间窗口，再进行横向打分

### 宏观/新闻信息无法验证
- 明确写出"该部分证据不足"
- 不将其作为核心结论唯一依据

## Key Principles（重要原则）

1. **先识别任务形态（单股/多股）与分析意图**
2. **先取数后判断，禁止脑补缺失数据**
3. **对比场景必须统一口径（时间窗/报表期/指标定义）**
4. **价值与技术并重，不做单维度结论**
5. **结论必须附数据依据与时间边界**
6. **始终保留风险提示与免责声明**
