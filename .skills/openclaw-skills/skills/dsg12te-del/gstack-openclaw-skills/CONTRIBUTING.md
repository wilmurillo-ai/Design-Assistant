# 贡献指南

感谢你对 OpenClaw 项目的兴趣！

## 如何贡献

### 1. 报告问题

如果你发现任何问题或有功能建议：
- 检查是否已存在类似问题
- 使用清晰的问题描述
- 提供复现步骤（如果适用）

### 2. 提交更改

#### 步骤
1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'Add amazing feature'`)
4. 推送分支 (`git push origin feature/amazing-feature`)
5. 创建 Pull Request

### 3. 技能格式规范

如果你想添加新的技能，请确保：

#### 必需字段
```yaml
---
name: skill-name
description: 技能描述
version: 1.0.0
author: 作者名
tags: [tag1, tag2]
---
```

#### 内容结构
- 概述 (Overview)
- 触发条件
- 核心流程
- 使用示例

### 4. 代码规范

- 使用中文编写文档
- 保持格式一致性
- 添加适当的注释

## 行为准则

请阅读并遵守我们的 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)（如果有）。

## 联系方式

- GitHub Issues
- 提交 Pull Request

感谢你的贡献！
