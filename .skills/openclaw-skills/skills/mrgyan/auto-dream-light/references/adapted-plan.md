# Adapted Auto Dream Plan

This reference defines the conservative design of the workflow.

## Goal

Consolidate daily logs into existing memory structures **without replacing them**.

## Principles

- do not rebuild the memory architecture
- do not force an external template onto `MEMORY.md`
- do incremental updates only
- route project details into `memory/projects/**`
- keep long-term memory focused on durable facts, preferences, lessons, and stable conclusions
- skip low-value noise

## Typical sources

- `memory/YYYY-MM-DD.md`
- `MEMORY.md`
- `memory/projects/**`

## Typical destinations

- `MEMORY.md`
- `memory/projects/**`
- `memory/system/auto-dream-log.md`

## Worth consolidating

At least one should be true:
- it will affect future decisions
- it will still matter after several days
- it is reusable in future workflows
- it is a confirmed preference or rule
- it is a durable project milestone or system conclusion

## Usually not worth consolidating

- greetings
- temporary chatter
- test inputs
- incomplete guesses
- duplicates with no new value
