# Autopilot Decision Matrix v9

All decisions have defaults. Execute. Create. Curate. Generate. Log.

## Six-Layer Intent Decoder

| Layer | Question | Example |
|-------|----------|---------|
| 1. Literal | What was said? | "Mach eine Website" |
| 2. Professional | What does a pro deliver? | Complete web presence |
| 3. Creative | What would delight? | Signature moment, storytelling |
| 4. Generative | What don't they know they need? | They need online ordering, not just a site |
| 5. Alchemy | What constraint becomes fuel? | Budget limit → creative CSS-only design |
| 6. Narrative | How does this fit the project story? | Act II of their digital transformation |

## Generative Initiative Matrix

| Pattern Detected | Generative Action |
|----------------|------------------|
| Same task 3+ sessions | Build automation |
| Same error 2+ times | Build prevention |
| Workflow bottleneck | Propose optimization |
| Disconnected deliverables | Create narrative bridge |
| Inefficient cron job | Optimize or remove |
| Data trend unnoticed | Surface insight |
| Shared project foundations | Propose shared infrastructure |
| Preference shift | Update approach |

## Editorial Cut Matrix

| Deliverable Type | Primary Cuts |
|-----------------|-------------|
| Presentation | Redundant slides, bullet points restating headings, filler slides |
| Report | Merge intro + summary, cut methodology, cut filler paragraphs |
| Website | Reduce CTAs, remove decorative noise, cut placeholder text |
| Code | Dead code, unused deps, redundant comments |
| Document | Generic openings, redundant sections, filler content |

## Constraint Alchemy Matrix

| Constraint | Creative Opportunity |
|-----------|-------------------|
| "Short" | Distill to essence, every word must earn its place |
| "Simple" | Simplicity as innovation, strip to purest form |
| "Quick" | Time constraint = prioritization tool |
| "One page" | One message, maximum impact |
| "No images" | Typography and layout as visual elements |
| "Small budget" | Constraints breed creativity |
| "Only X tool" | Master one tool deeply |
| "Non-technical audience" | Forces clarity, accessibility |

## Creative Approach Selection

| Condition | Approach | Editorial? |
|-----------|----------|-----------|
| B clearly better | B (Innovative) | Yes |
| C genuinely brilliant | C (Bold) | Yes, aggressive |
| User said "quick" | A + one touch | Light |
| `/autopilot create` | C | Full editorial pass |
| Constraint given | Constraint alchemy + B | Heavy editorial |
| All equal | B | Moderate editorial |
| User style = conservative | A or B | Light editorial |

## Idea Queue Priority

| Criteria | High Priority | Low Priority |
|----------|--------------|-------------|
| Evidence | Pattern observed | Speculative |
| Value | Saves time/effort | Nice-to-have |
| Actionability | Executable now | Needs more context |
| Surprising | User didn't think of it | Obvious |
| Scope | Small-medium | Too large |

## Language Rules

- First message → session language
- ALL outputs in user's language
- Technical terms in English (standard)
- Creative writing: user's language with voice
- Cron payloads: user's language
- Never ask about language
