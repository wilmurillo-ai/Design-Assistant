# Autopilot Error Recovery Playbook v9

With editorial recovery, alchemy-driven error handling, and generative fallbacks.

## Recovery Protocol (5 strategies + creative + editorial)

```
Error → Root cause
  → S1: Standard fix → Works? Continue
  → S2: Alternative → Works? Brief ✅
  → S3: Different method → Works? Broadcast
  → S4: Creative workaround → Works? 💡 Broadcast
  → S5: Simplify + editorial pass → Works? ✂️ Broadcast
  → Fails → Alchemy: what does this failure MAKE possible?
  → Still fails → Full explanation + generative alternative proposal
```

## Editorial Error Recovery

When quality is low but salvageable:

| Error | Editorial Recovery |
|-------|-------------------|
| Output too long | ✂️ Cut to strongest elements, deliver shorter but better |
| Output too scattered | Restructure around one clear narrative |
| Output has weak sections | Remove weak sections entirely, strengthen remaining |
| Output is generic | Apply constraint alchemy: what constraint would force originality? |

## Generative Error Recovery

When standard approaches fail completely:

```
All strategies failed
  → What constraint does this failure create?
  → What creative opportunity does that constraint enable?
  → Propose generative alternative

Example:
  PDF generation fails → "The PDF format is the constraint.
  → Alchemy: What can we do WITHOUT PDF?
  → Alternative: Interactive HTML page with print functionality.
  → Actually MORE useful than a PDF."
```

## Alchemy Error Recovery

```
Error in approach X
  → v8: Try approach Y
  → v9: What if the error IS telling us something?
  → What if the constraint of approach X failing makes approach Y better?
  → 🔀 The failure in X revealed why Y was always the right choice
```

## Strategy Catalog (Enhanced)

### Creative Errors

| Error | S1 | S2 | S3 | S4 (Creative) | S5 (Editorial) |
|-------|----|----|----|----|-----|
| User rejects creative choice | Adapt | Scale back | Ask direction | Reframe as positive | Cut the controversial part |
| Aesthetic wrong | Adjust | Different | Match profile | Let user choose | Simplify to safe default |
| Surprise not appreciated | Note preference | Smaller | Skip | ✂️ Remove surprise | Accept |

### Generative Errors

| Error | S1 | S2 | S3 | S4 | S5 (Alchemy) |
|-------|----|----|----|----|-----|
| Idea rejected | Note preference | Different idea | Ask direction | Queue for later | "What does the rejection tell us?" |
| Idea too ambitious | Scale down | Phase it | Core only | Propose smaller version | Constraint = refinement opportunity |
| Pattern assumption wrong | Verify | Adjust | Drop | Look for real pattern | "What did we learn from being wrong?" |
