# 飞书日志技能更新说明

**更新日期**: 2026-03-13  
**更新原因**: 凭证配置优化，支持自动检测和引导

---

## 📋 更新内容

### 1. 凭证获取逻辑优化 (`log-work.mjs`)

**更新前**：
- 直接使用硬编码的内置凭证
- 凭证失效时只显示错误，不引导用户

**更新后**：
- 按优先级获取凭证：环境变量 → 配置文件 → 内置默认值
- 凭证不存在时显示详细的获取指南
- 支持配置文件 `~/.openclaw/feishu-credentials.json`

### 2. 新增配置工具 (`config-credentials.js`)

交互式命令行工具，帮助用户快速配置飞书凭证：

```bash
cd /Users/one/.openclaw/workspace/skills/feishu-log
node config-credentials.js
```

功能：
- 检测现有配置
- 交互式输入凭证
- 自动创建配置文件
- 显示下一步操作指南

### 3. 更新文档

**SKILL.md**:

- 说明 3 种配置方式（环境变量、配置文件、内置值）

**README.md**:
- 添加配置工具使用说明
- 更新配置章节结构

**feishu-credentials.example.json**:

## 🎯 使用流程

### 首次使用

1. **运行配置工具**
   ```bash
   node config-credentials.js
   ```

2. **按提示输入**
   - App ID（从飞书开放平台获取）
   - App Secret（从飞书开放平台获取）
   - Folder Token（从云空间 URL 获取，或使用 "root"）

3. **配置权限**
   - 在飞书开放平台添加 `drive:drive` 和 `docx:document` 权限
   - 发布应用

4. **测试**
   ```bash
   node log-work.mjs
   ```

### 已配置用户

直接使用，技能会自动从配置文件加载凭证。

## 🔧 技术细节

### 凭证优先级

```javascript
1. 环境变量 (FEISHU_APP_ID, FEISHU_APP_SECRET, FEISHU_LOG_ROOT_TOKEN)
2. 配置文件 (~/.openclaw/feishu-credentials.json)
3. 内置默认值 (可能已过期)
```

### 配置文件格式

```json
{
  "app_id": "cli_xxx",
  "app_secret": "xxx",
  "root_folder_token": "xxx"
}
```

### 错误处理

- 凭证不存在：显示获取指南
- 凭证无效：显示 API 错误和排查建议
- 权限不足：提示添加对应权限

---

## ⚠️ 注意事项

1. **内置凭证可能已过期** - 建议配置自己的应用凭证
2. **权限需要发布** - 添加权限后需要发布应用才能生效
3. **Folder Token** - 可以使用 `"root"` 表示云空间根目录
