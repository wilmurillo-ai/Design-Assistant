# Persian Transliteration Guide

Standard romanization rules for converting Persian script to Latin characters when Latin output is needed.

---

## When to Transliterate

- User requests romanized Persian (Finglish/Pinglish)
- Technical systems that don't support Unicode/RTL
- Domain names, usernames, or identifiers derived from Persian words
- Pronunciation guides for non-Persian speakers
- SMS/chat contexts where the user is typing in Latin script

## Finglish (فینگلیش) — Informal Romanization

Finglish is the informal Latin-script writing of Persian used in texting, social media, and chat. It has no strict standard but follows common conventions.

## Transliteration Rules

### General Rules

1. **Consistency within a single text** — pick one system (formal or Finglish) and stick with it
2. **Proper nouns** — transliterate as commonly recognized: تهران → Tehran (not Tehrān), اصفهان → Isfahan (not Esfahān) — use established English forms
3. **Ezafe** — write as -e or -ye: کتابِ من → ketāb-e man, خانه‌ی ما → khāne-ye mā
4. **Compound verbs** — keep verb parts together: کار کردن → kār kardan
5. **Prefixes** — attach with hyphen: می‌رود → mi-ravad, نمی‌داند → nemi-dānad

## When NOT to Transliterate

- If the output medium supports Persian script, prefer native script
- If the user is writing in Persian script, respond in Persian script unless asked otherwise
- Technical documentation for an Iranian audience should be in Persian script
- Only transliterate when there's a clear reason (system limitation, user request, pronunciation guide)
