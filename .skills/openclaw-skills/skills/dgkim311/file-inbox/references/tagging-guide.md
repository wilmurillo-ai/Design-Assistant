# Auto-Tagging Guide

## Extension → Tag Mapping

| Extension | Auto Tags |
|-----------|-----------|
| .pdf, .doc, .docx | #document |
| .pptx, .ppt | #slides |
| .xlsx, .xls | #spreadsheet |
| .csv, .tsv, .json, .jsonl, .parquet | #data |
| .png, .jpg, .jpeg, .gif, .webp, .svg | #image |
| .py, .js, .ts, .sh, .r, .ipynb | #code |
| .zip, .tar, .gz, .7z | #archive |
| .mp4 | #video |
| .mp3, .wav | #audio |
| .md | #markdown |
| .yaml, .yml, .toml | #config |

## Context Tags (manually added)

Add context tags to make files easier to find later:

- **Research**: #research #논문 #paper #review #rebuttal
- **Experiment**: #experiment #results #model #training #data
- **Teaching**: #lecture #assignment #syllabus #grade
- **Admin**: #contract #budget #form #report
- **Student name**: #지훈 #우세형 #광훈 (or English names)
- **Project**: #portfolio-opt #timemixer #adversarial

## Best Practices

- Always add at least one context tag beyond the auto-tags
- Korean and English tags are both searchable
- Keep tags short — prefer `#논문` over `#논문-리비전-작업`
- Consistent tag names across files improve search quality
