# workflow

## Recommended default routes

### Markdown -> WPS
1. Write or clean content in Markdown.
2. Convert to docx when editable office format is needed.
3. Open in WPS on macOS for final layout check.
4. Export PDF only after visual verification.

### PDF -> editable document
1. Extract content to Markdown first.
2. Repair headings, lists, and tables.
3. Rebuild into docx if the user needs editable output.
4. Open in WPS for final formatting.

### docx compatibility repair
1. Check heading levels and style usage.
2. Remove unnecessary direct formatting where possible.
3. Ensure page size is explicit (A4 vs Letter).
4. Verify headers, footers, page breaks, and tables.
5. Re-open in WPS and visually inspect key pages.

### PPT / Excel understanding workflow
1. Convert source file to Markdown or structured text when analysis is needed.
2. Let the agent summarize or rewrite content.
3. Rebuild presentation/table output only after structure is stable.

## Principle
Use WPS for final editing and visual confirmation; use conversion and text tooling for understanding and restructuring.
