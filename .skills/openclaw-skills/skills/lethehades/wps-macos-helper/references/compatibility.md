# compatibility

## Common WPS/Word compatibility risks
- Direct formatting can override styles and produce inconsistent layout.
- Page size mismatch (A4 vs Letter) can shift tables and pagination.
- Fonts may substitute differently on macOS.
- Floating images and text wrapping may drift between editors.
- Section breaks, headers/footers, and page numbering often need manual review.
- Tables with fixed widths may overflow after font substitution.

## Safer practices
- Prefer consistent heading styles over manual formatting.
- Set page size explicitly.
- Use common fonts available on the machine.
- Keep complex layouts simple before final export.
- Re-check tables, images, headers, footers, and page numbers in WPS before delivery.

## For AI-assisted preparation
- Prefer Markdown for content restructuring.
- Prefer docx for editable office exchange.
- Prefer PDF only at the final locked-output stage.
