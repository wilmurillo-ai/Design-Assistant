# report format

## Sections
1. Header
2. Blocking issues
3. Warnings
4. Suggested exclusions
5. Minimal publish surface
6. Identity leakage review
7. Action hints

## Decisions
- `ready`: safe enough to publish with no material concerns found
- `ready_after_fixes`: publishable after addressing listed warnings or exclusions
- `not_ready`: do not publish yet; blocking issues exist
