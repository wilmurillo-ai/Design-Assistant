# source types

## stable-rule
Primary source for long-term defaults and durable operating rules.
Typical source: `MEMORY.md`

## daily-fact
Primary source for dated events, recent project state, and day-specific history.
Typical source: `memory/YYYY-MM-DD.md`

## improvement
Primary source for learned workflow improvements, corrections, and reusable execution lessons.
Typical source: `~/self-improving/*.md`

## method
Primary source for reusable procedures and domain/task guidance.
Typical source: `skills/*/SKILL.md`, `skills/*/references/*.md`

## evidence
Primary source for operational proof such as publishing, visibility changes, config changes, and high-value actions.
Typical source: `logs/audit/*.jsonl`, `logs/audit/*.md`, audit indexes
