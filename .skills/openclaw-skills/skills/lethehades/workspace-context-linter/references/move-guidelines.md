# move guidelines

## Suggested destinations
- Move environment-specific details to `TOOLS.md`.
- Move long-term stable rules to `MEMORY.md`.
- Move temporary facts or one-off events to `memory/YYYY-MM-DD.md`.
- Move reusable project workflows or detailed procedures to a skill reference or project-specific document.
- Keep `SOUL.md` and `USER.md` lean; they should describe style and preference, not operational detail.
