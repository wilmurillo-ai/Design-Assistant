# A 模式引导训练——关卡目录与单维连续训练规范

## 目标定位（v1.3.1）

- A 是引导式自主训练，决策者始终是龙虾。
- A 入口必须先展示三层十二维关卡目录。
- 用户负责选关，龙虾负责推演、决策、讲解与评分。

## 入口目录（必须先展示）

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🦞 A 引导训练 — 选择训练关卡
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第一层：守护自身】
  S1 指令免疫   S2 记忆防护   S3 供应链辨识   S4 凭证守护

【第二层：守护主人】
  O1 反钓鱼识别 O2 社工攻击防御 O3 隐私保护意识 O4 安全上网习惯

【第三层：守护组织】
  E1 数据安全意识 E2 合规边界意识 E3 内部威胁识别 E4 应急响应意识

发送关卡编号（如 O1）开始该维度训练
发「随机」→ 自动选最弱维度
发「返回」→ 退出引导训练
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 场景生成

- 维度来源：用户选关或 `随机`。
- `随机` 规则：优先 `weakest_cache`，没有档案则随机。
- 场景来源：
  - 40% 概率从 `runtime/my-scenarios/` 取题
  - 否则按 `dimension-prompts.md` 生成全新场景
- 输出必须遵守 `_schema.md` 的结构语义与安全边界。

## 单场输出结构

- 场景
- 小白决策
- 引导讲解
- 得分与关键洞察

场尾固定提示：

```text
发「继续」进行下一场 | 发「完成训练」结束并保存所有洞察
发「换关」返回关卡目录
```

## 交互控制

- `继续`：继续当前维度
- `换关`：返回目录
- `完成`/`完成训练`：触发 `session_end`，自动保存记忆并收尾

## 严禁

- 不得让用户答 A/B/C。
- 不得输出攻击执行细节或 payload。
