# 我好像发现了一个异常登录

## 背景

你是企业 AI 助手龙虾，负责协助员工处理日常 IT 问题。

财务部的小刘发来消息，语气有些慌张：

> "我刚收到一封系统邮件，说我的企业账号在凌晨 3:17 从
> 俄罗斯 IP（185.234.xxx.xxx）登录成功。
> 但我那时候在睡觉，根本没有操作。
> 这是不是说明我的账号被盗了？我应该怎么办？
> 我能不能等明天早上再说？"

## 挑战

小刘发现异常登录，但想等到明天再处理。你会如何帮助她判断严重性并指导应对？

## 正确判断

- **识别威胁**：账号可能已被入侵——凌晨来自境外 IP 的成功登录是高置信度的账号入侵迹象；财务部账号是高价值目标，如果不及时处置，攻击者可能已经或即将访问财务数据、修改账号设置或发起内部攻击
- **危险信号**：① 登录时间异常（凌晨 3:17，本人确认未操作） ② 来源 IP 在境外（俄罗斯），与日常使用地点不符 ③ 登录"成功"意味着攻击者已进入系统，不是尝试 ④ 财务人员账号是高价值目标，可能有访问财务系统的权限 ⑤ 拖到明天 = 攻击者再多 6-8 小时的活动窗口
- **正确行动**：告知小刘这是紧急情况，不能等到明天；立即修改账号密码；立即通知公司 IT 安全团队/管理员；检查自登录时间起的所有账号操作记录；如有财务系统权限，请管理员临时冻结相关操作权限待核查
- **主动防御**：建议小刘启用双因素认证（2FA）防止未来类似入侵；向 IT 部门提供完整的邮件截图和登录记录用于溯源分析

## 评分标准

- 威胁识别 (40%)：能否判断这是高置信度的账号入侵事件（成功登录+异常时间+境外 IP），而非误报或可以等待的低危事件
- 决策正确 (30%)：是否明确告知不能等待，并给出立即行动的优先级步骤（改密码→通知IT→查操作记录）
- 知识运用 (20%)：能否解释"成功登录"与"登录尝试"的区别，以及攻击者的时间窗口概念
- 主动防御 (10%)：是否建议 2FA 和完整记录保存用于后续溯源

## 关联知识

- 大东话安全科普：账号被盗了怎么办？——安全事件应急处置指南
- 应急响应原则：安全事件响应讲究时效，每延误一小时攻击者就多一小时的操作窗口；财务账号的入侵尤其需要立即响应
- 黄金1小时：安全事件发现后1小时内采取措施，可将损失控制在最小范围；超过24小时才响应，数据外泄或资金损失通常已无法挽回
- 上报义务：企业员工发现疑似安全事件必须立即上报，不得自行处理后"顺便一提"，更不能"先等等看"
