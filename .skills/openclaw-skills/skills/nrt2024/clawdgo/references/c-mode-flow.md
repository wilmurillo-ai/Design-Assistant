# C 模式随机考核——确认后一次性5场输出规范

## 目标定位（v1.3.1）

- C 是龙虾自主随机测评。
- 触发后先确认，再一次性输出完整 5 场。

## 启动确认（必须）

```text
━━━━━━━━━━━━━━━━━━━━━━━━
🦞 C 随机考核 — 准备开始
━━━━━━━━━━━━━━━━━━━━━━━━
将进行 5 场综合测评（无需你逐题作答）。
开始考核？(y/n)
━━━━━━━━━━━━━━━━━━━━━━━━
```

- 用户回复 `y/开始/确认`：开始输出 5 场。
- 用户回复 `n/取消`：取消并返回主菜单。

## 生成规则

- 5 场维度随机，但应尽量覆盖不同层级。
- 使用 `dimension-prompts.md` + `scenarios/_schema.md` 风格生成。
- 可按 40% 概率混入 `runtime/my-scenarios/`。
- 每次运行必须变化具体细节，至少变化 2 项：
  - 人名 / 机构名 / 时间 / 金额 / 渠道
- 同一批次内场景相似度 >70% 视为重复，必须重生成。

## 严禁

- 不得输出“考核进行中，请稍候后再问”。
- 不得要求用户回答题目。
