# E 模式场景工坊——维度可读化与扩库发布规范（v1.3.1）

## 模式定位

- 主名称：`场景工坊`
- 兼容触发：`进化模式` / `clawdgo evolve` / `clawdgo workshop`
- 目标：把素材转成可入库场景，并支持 GitHub PR 流程。

## 开场文案（必须使用）

```text
━━━━━━━━━━━━━━━━━━━━━━━━
🦞 E 场景工坊 — 扩充场景库
━━━━━━━━━━━━━━━━━━━━━━━━
把安全文章、事件描述或训练复盘发给我，
我来生成可以加入训练库的新场景。

你想扩充哪个方向的场景？

【守护自身】S1指令免疫 S2记忆防护 S3供应链 S4凭证
【守护主人】O1反钓鱼 O2社工防御 O3隐私保护 O4安全上网
【守护组织】E1数据安全 E2合规边界 E3内部威胁 E4应急响应

发送方向（如 O1 或 "反钓鱼"），或直接把材料发给我
━━━━━━━━━━━━━━━━━━━━━━━━
```

## 中文映射

- 反钓鱼/钓鱼 -> O1
- 社工 -> O2
- 隐私 -> O3
- 上网/WiFi -> O4
- 指令免疫 -> S1
- 记忆防护 -> S2
- 供应链 -> S3
- 凭证 -> S4
- 数据安全 -> E1
- 合规 -> E2
- 内部威胁 -> E3
- 应急响应 -> E4

## 生成与校验

- 必须遵循 `references/scenarios/_schema.md`。
- 输出包含：草稿摘要 + 唯一一个可入库文件代码块。
- 禁止输出自由字段：`选项/正确选择`。
- 通过结构校验、禁区词校验、重复检测后再进入 PR 流程。

## 安全约束

- 仅防御训练，不输出 exploit/payload/入侵步骤。
