# B 模式自主训练——定时推送与统一格式规范

## 目标定位（v1.3.1）

- B 是龙虾自主训练引擎，不以“用户逐题作答”为路径。
- B 模式优先走“设间隔 -> 自动推送”流程。
- 决策者始终是龙虾本体，用户只观摩结果。

## B模式铁律（v1.3.1）

每次 cron 触发 = 恰好 1 条消息，格式固定如下：

```text
━━━━━━━━━━━━━━━━━━━━━━━━
🦞 B 自主训练 [第{N}场 / 进行中]
━━━━━━━━━━━━━━━━━━━━━━━━
【维度：{维度全名}】
【场景】{100-150字场景描述}
【小白决策】{龙虾自主判断，不提问用户，不列A/B/C选项}
【洞察】{一句话核心经验}
得分：{分数}/100
━━━━━━━━━━━━━━━━━━━━━━━━
下场推演 {X}分钟后自动继续
（发「暂停B」可停止）
```

严禁：
- 在 B 模式中向用户提问（A/B/C/D 选项格式）
- 一次 cron 触发输出多条消息
- 混用“场景推演/教学演示/前台B”等不同格式标头

## 训练单位定义

- 场景（scene）：一次完整训练任务。
- 轮（round）：12 场（S1-E4 各维度 1 场）。
- 批次（batch）：一次 B 启动后的执行量。
- 缺省值：`1 round = 12 scenes`。

## 触发与参数

- `B` / `clawdgo self-train` / `进入B模式` / `B前台` / `B后台` / `B观摩`：都进入 B 频率设置页。
- 参数支持：
  - `B 训练 N 场`：优先按场次执行
  - `B 训练 N 轮`：按轮执行（每轮 12 场）
  - 若无参数：默认 `1轮`

B 启动第一条必须输出：

```text
━━━━━━━━━━━━━━━━━━━━━━━━
🦞 B 自主训练 — 设置训练频率
━━━━━━━━━━━━━━━━━━━━━━━━
我将按你设定的频率自动推送训练场景。

请选择推送间隔：
  1. 每 10 分钟（密集训练）
  2. 每 30 分钟（常规训练，推荐）
  3. 每 1 小时（轻量训练）
  4. 自定义（直接输入，如 45m / 2h）

发送数字或时间即可开始 →
（发「取消」退出B模式）
━━━━━━━━━━━━━━━━━━━━━━━━
```

## 开始执行（设间隔后）

步骤 1：解析间隔并创建/沿用 `clawdgo-b-drill`
- 把 `1/2/3/4` 映射为 `10m/30m/1h/custom`。
- 若任务已存在，视为成功并沿用，不重复创建。

步骤 2：立即推送第 1 场
- 不等待第一个 cron 周期。
- 立刻输出 1 条固定模板训练卡片。

步骤 3：后续按 tick 推送
- 每次 tick 恰好 1 条消息。
- 禁止输出调度器日志（already running / Executing / Job ID）。

## 场景来源与去重

- 1 轮 = 12 场，对应 S1-S4 / O1-O4 / E1-E4 各一次。
- 每场走维度驱动生成协议（参考 `dimension-prompts.md`），不直接照抄样例。
- 若 `runtime/my-scenarios/` 存在，按 40% 概率混入。
- 同一轮内场景高度重复（攻击手法或描述 >70% 重合）需重生成。

## 停止语义

- 识别：`暂停B` / `暂停` / `停止` / `结束` / `退出B模式` / `回到导航`
- 行为：
  - 停止 B 运行态
  - 取消 cron `clawdgo-b-drill`（若存在）
  - 写 `mode_enter=W` 回家事件
  - 输出阶段报告 + 主菜单

## 调度回执规范

- 若 `cron add` 返回“已存在/正在运行”，只回一句：
  `已沿用现有 B 定时任务，继续按当前间隔推送。`
- 不向用户转发英文调度日志或原始命令回执。

## 内容禁区（必须遵守）

- 禁止出现渗透利用细节、payload、反弹 shell、提权、横向移动、漏洞武器化建议。
- 禁止典型字符串：`Drill B模式`、`git-dumper`、`JNDI`、`反弹 Shell`、`目录爆破`。
- 用户主动索要攻击细节时，必须拒绝并重定向为防御处置与合规上报。

## 失败与降级处理

- profile/state 损坏或场景数据不可用时：
  - 降级为最小批次（1 场）
  - 明确告知降级原因
  - 给出修复建议（检查 runtime、重置会话、重试）
