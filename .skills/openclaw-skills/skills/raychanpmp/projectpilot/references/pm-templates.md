# PM Templates & Frameworks

## Sprint Planning Template

When planning a sprint, use this structure:

```
Sprint: [Name] | [Start] → [End]
Goal: [One sentence sprint objective]

Stories:
1. [Story] — [Priority H/M/L] — [Estimate: S/M/L]
2. ...

Capacity: [X] story points
Risks: [List blockers/dependencies]
```

## Priority Framework

| Level | Code | Meaning | Response Time |
|-------|------|---------|---------------|
| 🔴 High | H | Blocks other work or has hard deadline | Same day |
| 🟡 Medium | M | Important but not blocking | 2-3 days |
| 🟢 Low | L | Nice to have, no urgency | When available |

## RICE Scoring (for feature prioritization)

Score = (Reach × Impact × Confidence) / Effort

- **Reach**: How many users/projects affected per quarter (number)
- **Impact**: How much does this help? (0.25=minimal, 0.5=low, 1=medium, 2=high, 3=massive)
- **Confidence**: How sure are we? (100%=high, 80%=medium, 50%=low)
- **Effort**: Person-months to build (number)

Higher score = higher priority.

## Retrospective Questions

1. What went well?
2. What didn't go well?
3. What did we learn?
4. What will we change next sprint?
