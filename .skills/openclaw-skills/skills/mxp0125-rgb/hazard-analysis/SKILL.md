# 危险性分析技能 · Hazard Analysis Skill

> 适用场景：产品（机械/食品/一般工业品）危险性分析、风险评估报告编写、CE标志合规、产品安全审核。
> 依据标准：ISO 12100:2011 / DIN EN ISO 12100、EU Machinery Regulation 2023/1230、Blue Guide 2014、 Machinery Directive 2006/42/EC (→ 2023/1230)、HACCP

---

## ⚡ 快速启动：三步完成风险评估

```
第一步：确定范围（边界）  →  识别产品/机器的用途、用户、环境、寿命
第二步：危险识别（S×P矩阵）→  列出所有危险（按生命阶段），打分
第三步：风险评价 + 整改   →  Pass（可接受）/ Fail（不可接受）
```

---

## 第一部分：ISO 12100 五步风险评估法（★★★★核心方法论）

> 依据：DIN EN ISO 12100:2011-03 / EN ISO 12100:2010
> 新版补充：EU Machinery Regulation 2023/1230（2027年强制执行）

### ISO 12100 五步流程（详细版）

| 步骤 | 名称 | 内容 | 产出 |
|------|------|------|------|
| **Step 1** | 确定机器的限定（Limits） | 预期用途、可合理预见的误用、用户特征（专业/公众/儿童/残疾人）、使用环境、机器寿命 | 限定说明书 |
| **Step 2** | 危险识别（Hazard Identification） | 按9大类危险逐阶段（7个Life Cycle阶段）识别所有危险 | 危险清单 |
| **Step 3** | 风险估计（Risk Estimation） | 评估伤害严重程度（S）× 发生可能性（P）= R | 风险值R |
| **Step 4** | 风险评价（Risk Evaluation） | 判断R是否可接受；不可接受则进入Step 5 | 接受/不接受 |
| **Step 5** | 风险降低（Risk Reduction） | 按优先级：本质安全设计 → 保护装置 → 使用信息 → PPE | 整改措施 |

> **迭代要求：** Step 5完成后须回到Step 3重新评估，直至R可接受

### 9大类危险识别清单（Step 2）

| 类别 | 具体危险 | 示例 |
|------|---------|------|
| **机械危险** | 挤压/剪切/切割/卷入/缠绕/射入/刺穿/撞击 | 刀具/旋转轴/液压冲床 |
| **电气危险** | 触电/电弧/静电/雷击 | 带电部件/高压/接地失效 |
| **热危险** | 高温/低温/火灾/爆炸/热辐射 | 热表面/蒸汽/冷却系统 |
| **噪声危险** | 听力损伤/语言干扰/心理压力 | 气动元件/发动机 |
| **振动危险** | 手臂振动/全身振动 | 研磨机/压缩机 |
| **辐射危险** | 电离/非电离/激光/紫外线 | X光机/焊接弧光 |
| **材料/物质危险** | 可燃/爆炸性/腐蚀性/有毒/过敏 | 溶剂/粉尘/油烟 |
| **人体工效学危险** | 姿势/负重/重复劳动/心理压力 | 操作高度/控制面板 |
| **综合危险** | 组合危险（ flammable gas + heat） | 多风险叠加场景 |

### ISO 12100 风险公式（扩展版）

```
R = f(S, P, E, T)

其中：
R  = 风险值（Risk）
S  = 伤害严重程度（Severity）: 1~5
P  = 暴露于危险的可能性（Probability of exposure）: A~E
E  = 避免危险的可能性（Evasion possibility）: A~E
T  = 伤害持续时间/累积效应（Time/Threshold）

简化公式（最常用）：R = S × P
```

### 生命周期待定（7阶段 — 来自案例报告样稿）

| 代码 | 阶段 | 关键行动 |
|------|------|---------|
| **A** | 设计 | 本质安全设计，从源头消除危险 |
| **B** | 制造 | 材料选用、制造过程危险控制 |
| **C** | 运输 | 防倾倒/防震动/包装 |
| **D** | 安装调试 | 安装人员培训、高压/低压电气安全 |
| **E** | 运行维护 | 最关键！机械/电气/热/噪声/辐射危险集中在此阶段 |
| **F** | 停机报废 | 能量隔离（LOTO）、意外再启动防护 |
| **G** | 处置 | 有害废弃物处置、材料回收 |

## 第二部分：EU Machinery Regulation 2023/1230（★★★★★重大更新，2027年强制）

> 取代：Directive 2006/42/EC（2027年1月20日起强制执行）
> 已有Directive 2006/42/EC证书：过渡期至2027年，之后须符合新Regulation

### 与旧指令的核心差异

| 项目 | 旧指令 2006/42/EC | 新法规 2023/1230（2027年起强制） |
|------|------------------|--------------------------------|
| **AI安全功能** | 无明确规定 | AI用于安全相关功能 → **高风险**，须第三方（Notified Body）认证 |
| **网络安全** | 无明确规定 | **网络安全成为直接安全风险**；须防护黑客入侵/篡改 |
| **数字文档** | 须纸质说明书 | **数字文档（电子版）允许**；除非消费者要求纸质的 |
| **实质性改装** | 无明确规定 | 实质性改装 → **改装者成为新制造商**，承担全部合规责任 |
| **风险评估范围** | 设计/制造/使用阶段 | **全生命周期**：设计→安装→维护→报废→可合理预见的误用 |
| **公告机构（Notified Body）** | 部分产品须 | 高风险产品（含AI）**强制**须公告机构参与 |
| **市场追溯** | 基本追溯 | 强化追溯：须记录批号/序列号以便快速溯源 |
| **技术文档保存** | ≥10年 | ≥10年（不变） |

### 过渡时间轴

```
2023年6月  →  新Regulation发布
2023年7月  →  正式生效（现行指令仍适用）
2027年1月20日  →  全面强制执行（42个月过渡期结束）
2027年后   →  所有新上市机械须符合 EU 2023/1230
```

## 第三部分：性能等级（PLr）与功能安全（★★★★★新增）

> 依据：ISO 13849-1:2023（等效替代 GB/T 16855.1-2025）

### PLr 等级定义

| PLr等级 | 目标失效量（PFHD） | 说明 |
|---------|-----------------|------|
| **a**（最低） | ≥ 10⁻⁵ ~ < 10⁻⁴ | 基本安全功能，低风险机器 |
| **b** | ≥ 10⁻⁶ ~ < 10⁻⁵ | 中低风险 |
| **c** | ≥ 10⁻⁷ ~ < 10⁻⁶ | 中高风险 |
| **d** | ≥ 10⁻⁸ ~ < 10⁻⁷ | 高风险（须冗余+自检） |
| **e**（最高） | ≥ 10⁻⁹ ~ < 10⁻⁸ | 极高风险（复杂系统） |

> PFHD = Probability of dangerous Failure per Hour（每小时危险失效概率）

### PLr 选用规则（风险图法）

```
风险评估 → 确定所需的PLr：
- 低风险区域 → PLr a
- 中等风险区域 → PLr b/c
- 高风险区域 → PLr d
- 极高风险区域 → PLr e
```

**PLr 设计步骤：**
1. 确定每个安全功能的PLr（Required Performance Level）
2. 计算系统实际能达到的PL（Achieved Performance Level）
3. 确保 **PL ≥ PLr**（否则需要改进安全设计）
4. 常用方法：冗余设计（Dual-channel）+ 诊断覆盖率（DC）

## 第四部分：HACCP 危险性分析与食品/产品安全（★★★★新增）

> 适用场景：食品工厂、产品安全风险管理（独立于机械风险，可交叉应用）
> 依据：Codex HACCP / FDA / USDA

### HACCP 七原则（与ISO 12100对比记忆）

| 原则 | 内容 | ISO 12100对应 |
|------|------|--------------|
| **原则1** | 危害分析（Hazards Identification） | Step 2 危险识别 |
| **原则2** | 确定关键控制点（CCP） | 识别高风险点（High Risk Areas） |
| **原则3** | 建立关键限值（CL） | 建立接受/不接受标准 |
| **原则4** | 建立监控程序（Monitoring） | 建立检查机制 |
| **原则5** | 建立纠正措施（Corrective Actions） | 制定整改方案 |
| **原则6** | 建立验证程序（Verification） | 重新评估（Iterative循环） |
| **原则7** | 建立文件记录（Documentation） | 技术文档保存 |

### HACCP vs ISO 12100 核心区别

| 对比项 | HACCP | ISO 12100 |
|--------|-------|-----------|
| **适用领域** | 食品/制药/化学品 | 机械/设备/工业产品 |
| **核心工具** | CCP判定树 | S×P风险矩阵 |
| **关注点** | 生物/化学/物理危害 | 机械/电气/热/噪声等危害 |
| **法规框架** | Codex/FDA/各国食品法规 | EU机械指令/ISO标准 |
| **CCP特殊要求** | 有关键控制点监控 | 须有安全联锁装置 |

## 第五部分：Bow-Tie分析法（屏障图）（★★★★新增）

> 适用场景：重大危险源（MAH）分析、复杂系统风险可视化

### Bow-Tie 结构

```
                    威胁（Threat）
                       ↓
  屏障A   屏障B   【顶事件 Top Event】  屏障C   屏障D
  ──────────────    (能量释放/危险事件)    ──────────────────
    左侧：预防       ↑               右侧：减轻
    右侧：后果（Consequences）

核心要素：
- 危险（Hazard）：危险物质/能量本身
- 顶事件（Top Event）：能量失控释放
- 威胁（Threat）：引发顶事件的原因
- 屏障（Barrier）：预防/控制的保护层
- 后果（Consequence）：顶事件后的实际损害
```

### 应用示例（机械压力机）

```
危险：高压液压系统
顶事件：液压油泄漏+点火源 → 火灾
威胁：密封圈老化 / 操作失误 / 维护不当
屏障A：定期检查密封圈（维护计划）
屏障B：液压站配置灭火系统
屏障C：紧急停机按钮
后果：人员烧伤 / 设备损坏 / 停产损失
```

## 第六部分：风险矩阵（5×5标准版）

| 可能性 ↓ / 严重程度 → | 1 极轻微 | 2 轻微 | 3 中等 | 4 严重（不可逆） | 5 极严重（死亡） |
|---|---|---|---|---|---|
| **A 极不可能** | 1A 极低 | 2A 低 | 3A 低 | 4A 中 | 5A 中 |
| **B 不太可能** | 1B 低 | 2B 低 | 3B 中 | 4B 中 | 5B 高 |
| **C 偶尔** | 1C 低 | 2C 中 | 3C 中高 | 4C 高 | 5C 极高 |
| **D 可能** | 1D 中 | 2D 中高 | 3D 高 | 4D 极高 | **4D 不可接受** |
| **E 很可能** | 1E 中 | 2E 高 | 3E 极高 | **4E 不可接受** | **5E 不可接受** |

**行动判定：**
- **R ≤ 4**：可接受（Acceptable）→ 记录存档
- **R = 5~9**：有条件接受（ALARP）→ 须有额外保护措施
- **R ≥ 10**：不可接受（Unacceptable）→ **必须整改**

## 第七部分：安全距离计算（ISO 13855:2024 ★★新增）

> 适用：为安全防护装置（光栅/安全门/双手控制）计算最小安全距离
> 新版 ISO 13855:2024（2024年末发布）扩展到 AMR/协作机器人场景

### 基本公式

```
S = K × T + C

其中：
S = 最小安全距离（mm）
K = 接近速度常数（mm/s）
T = 停止时间（s）= 安全装置响应时间 + 机器停止时间
C = 附加距离（mm）
  - C = 8 × (d - 14mm)（手指防护，d=光束间距）
  - C = 850mm（身体防护）
```

### K值速查

| 防护类型 | K值（mm/s） | 备注 |
|---------|------------|------|
| 手指防护（光栅） | 2000 | — |
| 手掌防护（光栅） | 2000 | d>8mm |
| 身体防护（安全门） | 1600 | — |
| 双手控制（Type IIIC） | 2000 | — |

## 第八部分：CE标志合规流程（全版本对照）

### 普通机械（自声明，Module A）

```
1. 确定产品分类（是否在协调标准覆盖范围内）
2. 进行风险评估（ISO 12100）
3. 确定适用的 Essential Health & Safety Requirements
4. 设计符合性：本质安全设计 → 安全装置 → 使用信息
5. 编制技术文档（TCF）≥10年保存
6. 起草并签署EU符合性声明（DoC）
7. 加贴CE标志（≥5mm，可见、不可磨灭）
8. 在产品/包装上标注：制造商名称/地址、型号、批号、制造年份
```

### 含AI的高风险机械（须Notified Body，2027年起强制）

```
在上述流程基础上增加：
9. AI安全功能须由公告机构（Notified Body）进行合格评定
10. 网络安全评估须证明系统防黑客/防篡改能力
11. 数字文档（e-TF）须符合MDR/ Machinery Regulation要求
12. 实质性改装后：改装者须重新完成全部CE流程
```

## 第九部分：常见不合格项与整改方向（扩展版）

| 不合格类型 | 原因 | 整改方向 |
|-----------|------|---------|
| **F（不可接受风险）** | 设计阶段未充分识别危险 | 重新进行危险识别，增加保护装置 |
| **PLr不达标** | 安全功能设计不满足所需PLr | 冗余设计（双通道）+自诊断，提高DC |
| **安全距离不足** | S = K×T+C 计算错误或未计算 | 按ISO 13855:2024重新计算安全距离 |
| **警告标签缺失** | 未按ISO 12100/ISO 7010格式标示 | 使用符合标准的警告标志 |
| **AI安全功能不合规** | AI用于安全功能但无Notified Body认证 | 委托公告机构进行AI安全评估（2027年起强制） |
| **网络安全缺失** | 未防护黑客入侵/篡改 | 按EU 2023/1230要求增加网络安全措施 |
| **实质性改装未重评** | 改装后未重新进行风险评估 | 改装者须承担新制造商全部责任 |
| **可追溯性不足** | 未标注批号/制造年份 | 重新标注完整追溯信息 |
| **CE标志不规范** | 高度不足/位置错误/格式错误 | 重新制作CE标志标贴 |
| **残余风险未记录** | 未进行残余风险评估 | 补充残余风险分析和记录 |
| **HACCP CCP未监控** | CCP监控程序缺失或不完整 | 建立CCP监控记录和纠正措施程序 |
| **数字文档不合规** | 仅有纸质文档，无数字备份 | 建立数字技术文档管理系统（e-TF） |
