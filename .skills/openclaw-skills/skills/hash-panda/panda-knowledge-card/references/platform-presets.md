# 社交平台预设

## 平台参数总览

| 平台 | 代码 | 推荐比例 | 备选比例 | 信息密度 | 文字大小 |
|------|------|----------|----------|----------|----------|
| 小红书 | `xhs` | 3:4 | 1:1, 4:5 | 中高 | 大号醒目 |
| 微信公众号 | `wechat` | 16:9 | 4:3, 1:1 | 中等 | 标准 |
| 微博 | `weibo` | 16:9 | 1:1, 9:16 | 中等 | 标准偏大 |
| X / Twitter | `x` | 16:9 | 2:1, 1:1 | 精简 | 大号 |
| 抖音 | `douyin` | 9:16 | 3:4 | 中等 | 超大醒目 |
| Instagram | `instagram` | 1:1 | 4:5, 9:16 | 中等 | 标准 |
| 通用 | `general` | 16:9 | 3:4, 1:1 | 标准 | 标准 |

## 小红书（xhs）

以竖屏图文为主，用户在信息流中快速滑动，卡片需要在第一眼吸引注意力。

| 属性 | 设置 |
|------|------|
| **推荐比例** | 3:4（竖屏，信息流最佳展示） |
| **备选比例** | 1:1（正方形，适合多图拼接）、4:5（接近竖屏） |
| **信息密度** | 中高 — 每张卡片 3-5 个信息点 |
| **文字处理** | 大号醒目，关键词突出，标题占比大 |
| **配色** | 明亮活泼，高饱和度，对比强烈 |

### 卡片排版建议

- 标题区域占画面上方 20-25%，字号要大
- 核心内容区域居中 50-60%
- 底部留白或放角色/CTA 15-25%
- 边距不少于画面宽度的 6%
- 编号/序号使用大圆形或色块，视觉引导强

---

## 微信公众号（wechat）

横屏长图文为主，卡片嵌入在文章段落之间，需要与文字排版协调。

| 属性 | 设置 |
|------|------|
| **推荐比例** | 16:9（横屏，与文章排版协调） |
| **备选比例** | 4:3（偏横屏）、1:1（居中展示） |
| **信息密度** | 中等 — 每张卡片 2-4 个信息点 |
| **文字处理** | 标准大小，清晰可读，与文章字体风格协调 |
| **配色** | 偏素雅，与文章白色背景和谐 |

### 卡片排版建议

- 横向布局，信息从左到右流动
- 留白充分，与文章白色背景融合
- 避免过于花哨的装饰
- 代码/命令文字使用等宽字体，浅灰底色

---

## 微博（weibo）

以横屏为主，也支持竖屏九宫格。卡片在时间线中需要快速传达信息。

| 属性 | 设置 |
|------|------|
| **推荐比例** | 16:9（横屏，时间线展示最佳） |
| **备选比例** | 1:1（九宫格）、9:16（竖屏长图） |
| **信息密度** | 中等 — 信息点精简，一图一观点 |
| **文字处理** | 标准到偏大，快速扫描即可理解 |
| **配色** | 较鲜明，在信息流中有辨识度 |

### 卡片排版建议

- 信息精简，一张卡片一个核心观点
- 关键数字/结论放大突出
- 适合对比卡和要点清单类型

---

## X / Twitter（x）

推文内嵌图片以横屏为主，Card 预览裁切为约 2:1，卡片需要在小尺寸下信息可读。

| 属性 | 设置 |
|------|------|
| **推荐比例** | 16:9（标准推文图片） |
| **备选比例** | 2:1（Card 预览最佳）、1:1（多图模式） |
| **信息密度** | 精简 — 每张卡片 1-3 个信息点 |
| **文字处理** | 大号，字少但冲击力强，英文优先 |
| **配色** | 高对比，深底亮字或亮底深字，避免中间灰 |

### 卡片排版建议

- 核心信息集中在画面中央 60% 区域（Card 预览裁切安全区）
- 上下边缘避免放重要文字（可能被裁切）
- 文字简洁，单张卡片不超过 30 个中文字或 50 个英文词
- 代码片段使用大号等宽字体，深色背景

---

## 抖音（douyin）

全屏竖屏浏览，用户上下滑动，卡片需要铺满屏幕且信息层级分明。

| 属性 | 设置 |
|------|------|
| **推荐比例** | 9:16（竖屏全屏） |
| **备选比例** | 3:4（接近竖屏） |
| **信息密度** | 中等 — 内容分层，从上到下阅读 |
| **文字处理** | 超大醒目，标题至少占画面 15% 高度 |
| **配色** | 高饱和度，强对比，暗色背景效果好 |

### 卡片排版建议

- 标题放在上方 20%，超大字号
- 内容区域中部 50-60%，从上到下排列
- 底部 20% 留给 CTA 或角色
- 边距至少 8%，避免被系统 UI 遮挡
- 避免在底部 15% 放重要信息（评论/互动栏遮挡）

---

## Instagram（instagram）

以正方形 Feed 为主，Stories 为竖屏。知识卡片以 Feed 正方形为主。

| 属性 | 设置 |
|------|------|
| **推荐比例** | 1:1（正方形 Feed） |
| **备选比例** | 4:5（Feed 竖屏最大比例）、9:16（Stories） |
| **信息密度** | 中等 — 视觉优先，文字辅助 |
| **文字处理** | 标准，英文优先，关键词大写 |
| **配色** | 干净统一，配色方案一致性强 |

### 卡片排版建议

- 正方形布局，信息居中对称
- 留白充足，不要堆满
- 多图轮播（Carousel）每张保持视觉一致
- 字体风格统一，避免混用过多字体

## 通用（general）

不针对特定平台，适用于博客文章、技术文档、PPT 演示等场景。

| 属性 | 设置 |
|------|------|
| **推荐比例** | 16:9（横屏标准，兼容性最佳） |
| **备选比例** | 3:4（竖屏）、1:1（正方形） |
| **信息密度** | 标准 — 根据内容复杂度自适应 |
| **文字处理** | 标准，注重可读性 |
| **配色** | 中性专业，适应各种背景 |

## 使用方式

### 在确认设置时选择

步骤 2 确认设置中会询问目标平台，根据选择自动应用对应的比例和排版规则。

### EXTEND.md 默认配置

```markdown
## 设置
默认平台: xhs
```

设置后无需每次选择。步骤 2 确认时显示当前平台并允许更改。
