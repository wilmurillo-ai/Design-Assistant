# 提示词构建指南（知识卡片专用）

## 概述

知识卡片的提示词与文章配图不同，核心关注点是 **文字排版** 和 **信息层级**。卡片本身就是内容载体，文字是画面的主角，视觉元素为文字服务。

---

## 提示词文件格式

```yaml
---
card_id: 01
layout: tips
style: notion
character_role: narrator
---

[下方为卡片类型化模板内容...]
```

---

## 通用构建原则

### 1. 文字优先

知识卡片中文字是核心内容，提示词必须：

- **明确指定标题文字内容**：使用文章中的原文
- **明确指定要点文字内容**：使用具体的知识点
- **指定字号层级**：主标题 > 副标题 > 正文 > 注释
- **指定文字排列方式**：居中/左对齐/缩进

### 2. 信息层级

每张卡片的信息层级：

```
第一层：主标题（最大、最醒目）
第二层：副标题或分类标签（中等）
第三层：核心内容（正文大小、最多的文字量）
第四层：补充信息/出处/CTA（最小）
```

### 3. 留白与间距

- 标题区与内容区之间有明显间距
- 各要点之间有一致的间距
- 卡片边缘留出安全边距（不少于画面宽度的 8%）
- 内容不要挤满，保持呼吸感

## 卡片提示词通用模板

```
[标题] - [卡片类型]知识卡片

布局：[卡片类型] — [具体布局描述]

标题区：
- 主标题：[大号醒目文字，具体内容]
- 副标题：[补充说明，可选]
- 序号标签：[如"1/5"，可选]

内容区：
- [类型化内容结构，见 card-layouts.md]

角色：（仅在有角色时包含）
- [角色名称]，一个[角色风格]的卡通角色（与参考图保持一致）
- 融合模式：[模式] — [位置描述]
- 动作：[具体动作]
- 表情：[根据内容语义]
- 占比：约占画面高度的 [百分比]%

装饰元素：[背景图案、分隔线、图标风格、装饰物]
配色：[语义化配色，具体色值]
风格：[风格特征描述]
比例：[宽高比，通常 3:4]
```

## 按卡片类型的内容区写法

### tips（要点清单）

```
内容区：
- ① [图标：灯泡] [要点标题]：[一句话说明，15-25字]
- ② [图标：星号] [要点标题]：[一句话说明]
- ③ [图标：箭头] [要点标题]：[一句话说明]
每条要点之间有等距分隔线或留白
```

### comparison（对比卡）

```
内容区：
左侧 — [A 方案名称]，使用[颜色A]：
- · [特点 1]
- · [特点 2]

右侧 — [B 方案名称]，使用[颜色B]：
- · [特点 1]
- · [特点 2]

分隔：[竖线/VS标识/渐变过渡]，居中
底部结论：[一句话推荐或总结，可选]
```

### definition（定义卡）

```
内容区：
术语区：[术语大号显示，装饰性引号或下划线]
定义区：[2-3句清晰定义，标准字号]
示例区：[标记为"例子"，带背景色块或缩进]
```

### steps（步骤卡）

```
内容区：
- Step 1 [圆形编号] [步骤名]：[说明]
  ↓ [连接箭头]
- Step 2 [圆形编号] [步骤名]：[说明]
  ↓
- Step 3 [圆形编号] [步骤名]：[说明]
步骤间有连接线表示顺序关系
```

### qa（问答卡）

```
内容区：
- Q [问题气泡/标签] [问题文字，醒目颜色]
  A [回答标签] [答案文字，标准颜色]
  [Q和A之间有视觉区分：不同背景色、缩进或气泡]
```

### matrix（矩阵卡）

```
内容区：
[2×2 网格，每个格子有独立背景色]
左上 [颜色1]：[象限名] — [要点]
右上 [颜色2]：[象限名] — [要点]
左下 [颜色3]：[象限名] — [要点]
右下 [颜色4]：[象限名] — [要点]
轴标签沿边缘标注
```

### quote（金句卡）

```
内容区：
[大号装饰性引号开头]
[金句文字，大号加粗，居中排列，行间距较大]
[装饰性引号结尾]
— [出处/作者]
[解读文字，小号，居中]
```

### mindmap（脑图卡）

```
内容区：
[中心：核心主题，大号圆形节点]
[分支1 向左上]：[名称] → [子节点1, 子节点2]
[分支2 向右上]：[名称] → [子节点1, 子节点2]
[分支3 向左下]：[名称] → [子节点1, 子节点2]
[分支4 向右下]：[名称] → [子节点1, 子节点2]
各分支使用不同颜色，从中心向外有连线
```

## 风格适配

提示词的装饰元素和配色需要随风格调整：

| 风格 | 装饰元素 | 配色 | 文字处理 |
|------|----------|------|----------|
| notion | 简笔画图标、手绘线条、淡色背景 | 黑白为主，一个强调色 | 简洁无衬线 |
| study-notes | 手写体文字、荧光笔划线、便签贴 | 蓝笔(#2563EB) + 红批注(#DC2626) + 黄高亮(#FDE68A) | 仿真手写 |
| chalkboard | 粉笔笔触、白色/彩色粉笔 | 黑板绿(#2D5016) 背景 + 白/彩色粉笔 | 粉笔手写 |
| cute | 圆角气泡、星星闪闪、云朵装饰 | 粉色(#FCA5A5) + 薄荷绿(#A7F3D0) + 浅紫(#C4B5FD) | 圆润字体 |
| bold | 粗黑边框、警示标识、大色块 | 高对比：黑+黄(#FCD34D) 或 黑+红(#EF4444) | 粗体大字 |
| warm | 圆润图标、柔和阴影、自然纹理 | 暖色：米色(#FEF3C7) + 珊瑚(#FCA5A5) + 橄榄(#A3E635) | 友好字体 |
| vector | 几何图标、粗轮廓线、色块 | 珊瑚红(#E07A5F) + 薄荷绿(#81B29A) + 芥末黄(#F2CC8F) | 现代无衬线 |

## 角色区块

有角色时，在"装饰元素"之前注入"角色"区块。知识卡片中角色的主要融合模式：

- **讲解者**（最常用）：站在卡片边缘，指向标题或核心内容
- **点缀**：缩小版出现在角落，用表情呼应内容

**讲解者在卡片中的模板**：

```
角色：
- [角色名称]（与参考图保持一致）
- 融合模式：讲解者 — 卡片[位置]
- 动作：[指向/展示] [标题/核心要点]
- 表情：[根据内容语义]
- 占比：约占画面高度的 15-20%
- 融合细节：角色与内容区域有清晰的空间分离，不遮挡文字
```

---

## 系列一致性

同一系列的所有卡片必须：

1. **使用相同风格**：配色、装饰元素、文字处理保持一致
2. **序号标识**：每张卡片标注序号（如"1/5"、"2/5"）
3. **角色一致**：使用同一张角色参考图（`--ref`）
4. **布局规范统一**：标题区高度、边距、字号层级保持一致

---

## 避免事项

- 文字过多（单张超过 150 字）
- 字号过小（在手机上不可读）
- 信息层级不清晰
- 装饰元素喧宾夺主
- 角色遮挡核心文字
- 各卡片风格不统一
