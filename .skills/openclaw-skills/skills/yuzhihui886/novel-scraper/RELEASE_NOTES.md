# novel-scraper v1.5.0 发布说明

## 🐛 关键 Bug 修复

**问题**：用户请求抓取第 301-400 章，实际抓取了第 300-419 章（120 章）

**根因**：使用索引切片 `data[300:400]` 而非按章节号筛选
- 索引 300-399 → 章节号 300-419（错误）
- 应为章节号 301-400（正确）

**修复**：
- 新增 `--chapters` 参数，按章节号精确筛选
- 自动提示缺失章节（网站目录中没有的章节）

## ✨ 新功能

### 按章节号范围抓取

```bash
# ✅ 推荐用法（v1.5.0）
python3 scripts/scraper_v5.py --chapters 301-400 --book "没钱修什么仙"

# 输出：
# 📋 章节号范围：301-400
# 📋 实际抓取：301-399 (83 章)
# ⚠️ 缺失 17 章（网站目录中没有）
```

### 辅助脚本

```bash
# 提取指定章节范围的 URL
python3 scripts/extract_urls.py 301 400
```

## 📊 对比

| 版本 | 参数 | 结果 | 问题 |
|------|------|------|------|
| v1.4.0 | `data[300:400]` | 章节号 300-419 | ❌ 索引切片错误 |
| v1.5.0 | `--chapters 301-400` | 章节号 301-399 | ✅ 按章节号筛选 |

## ⚠️ 注意事项

1. **缺失章节**：网站目录本身可能缺失某些章节（如第 345、367-370、381-384、389-396、400 章）
2. **向后兼容**：`--url` 和 `--urls` 参数继续有效
3. **推荐用法**：优先使用 `--chapters` 参数
