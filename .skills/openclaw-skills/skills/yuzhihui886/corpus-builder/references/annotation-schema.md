# 语料库标注体系详解

## 10 维度标注体系

### 1. 场景类型 (scene_type)

**类型**: 多选（1-3 个）

**选项**:
- 打斗 - 战斗、对决、比武等场景
- 修炼 - 吐纳、闭关、突破等修行场景
- 对话 - 以对话为主的交流场景
- 探险 - 探索秘境、遗迹、洞穴等
- 拍卖 - 拍卖会、竞价、宝物交易
- 会议 - 商议、讨论、计划安排
- 日常 - 吃饭、休息、闲聊等日常生活
- 过渡 - 时间过渡、场景转换

**识别关键词**:
```
打斗：拳、脚、剑、法术、攻击、防御、招式、灵力、轰、劈
修炼：灵气、经脉、丹田、突破、闭关、运功、打坐、炼化
对话：说道、问道、回答、沉默、冷笑、开口、声音
探险：遗迹、秘境、洞穴、探索、发现、入口、机关
```

---

### 2. 情绪基调 (emotion)

**类型**: 多选（1-2 个）

**选项**:
- 紧张 - 危险、警惕、杀气、围攻、生死关头
- 轻松 - 微笑、调侃、悠闲、放松、愉快
- 悲伤 - 泪水、悲痛、哀伤、哭泣、绝望
- 热血 - 怒吼、爆发、战意、激昂、不屈
- 温馨 - 温暖、柔和、关怀、拥抱、甜蜜
- 愤怒 - 怒吼、愤怒、咆哮、愤恨、怒火
- 恐惧 - 惊恐、害怕、颤抖、畏惧、恐慌
- 中性 - 平静、淡淡、默默、静静

---

### 3. 写作手法 (techniques)

**类型**: 多选（最多 3 个）

**选项**:
- 侧面描写 - 通过他人反应、环境烘托
- 对比 - 前后对比、人物对比、强弱对比
- 伏笔 - 埋下伏线、暗示后续发展
- 倒叙 - 回忆、插叙过去事件
- 插叙 - 插入相关背景或事件
- 白描 - 简洁直接的描写
- 细节描写 - 精细的动作、表情、心理描写
- 心理描写 - 内心活动、思想斗争

---

### 4. 节奏快慢 (pace)

**类型**: 单选

**选项**:
- 快节奏 - 动作密集、短句多、紧张刺激
- 慢节奏 - 描写细腻、长句多、舒缓平和
- 张弛有度 - 快慢结合、节奏变化

**判断标准**:
- 动作密度 > 描写密度 × 2 → 快节奏
- 描写密度 > 动作密度 × 2 → 慢节奏
- 其他 → 张弛有度

---

### 5. 视角类型 (pov)

**类型**: 单选

**选项**:
- 第一人称 - "我"、"我们"为主角
- 第三人称限知 - 跟随单一角色的视角
- 第三人称全知 - 上帝视角，多角色内心

---

### 6. 角色数量 (character_count)

**类型**: 单选

**选项**:
- 单人 - 只有一个主要角色
- 双人 - 两个角色互动
- 多人 (3+) - 三个或以上角色

---

### 7. 力量层级 (power_level)

**类型**: 单选（如有明确提及）

**选项**:
- 炼气期 - 练气、凝气、聚气
- 筑基期 - 筑基、结丹准备
- 金丹期 - 金丹、元婴准备
- 元婴期 - 元婴、化神准备
- 不明确 - 未明确提及修为

---

### 8. 关键元素 (key_elements)

**类型**: 多选（3-5 个关键词）

**示例**:
- 打斗场景：围攻、剑法、后发先至、以少胜多
- 修炼场景：突破、瓶颈、顿悟、走火入魔
- 对话场景：试探、威胁、谈判、表白

---

### 9. 适用场景 (usage)

**类型**: 多选（学习参考用途）

**示例**:
- 学习打斗描写
- 学习情绪渲染
- 学习对话写作
- 学习氛围营造
- 学习节奏把控
- 学习伏笔设置

---

### 10. 质量评分 (quality_score)

**类型**: 数值（1-10 分）

**评分标准**:
- 1-3 分：文笔生涩、描写生硬
- 4-6 分：文笔流畅、中规中矩
- 7-8 分：描写生动、有亮点
- 9-10 分：文笔精湛、堪称范文

**评分维度**:
- 文笔流畅度（30%）
- 描写生动性（40%）
- 节奏把控（30%）

## 标注数据格式

### JSON 格式
```json
{
    "scene_type": ["打斗", "修炼"],
    "emotion": ["紧张", "热血"],
    "techniques": ["细节描写", "侧面描写"],
    "pace": "快节奏",
    "pov": "第三人称限知",
    "character_count": "多人 (3+)",
    "power_level": "炼气期",
    "key_elements": ["围攻", "剑法", "后发先至"],
    "usage": ["学习打斗描写", "学习紧张氛围营造"],
    "quality_score": 8,
    "text_length": 1500,
    "annotation_model": "dashscope-coding/qwen3.5-plus",
    "original_text": "..."
}
```

### ChromaDB 元数据（扁平化）
```python
{
    "scene_type": "打斗，修炼",  # 逗号分隔
    "emotion": "紧张，热血",
    "techniques": "细节描写，侧面描写",
    "pace": "快节奏",
    "quality_score": "8",  # 字符串
    "source_file": "没钱修什么仙_第 1-10 章.txt"
}
```

## 标注流程

1. **规则预标注** - 使用关键词匹配快速预测
2. **LLM 精标注** - 调用 DashScope API 进行智能标注
3. **降级处理** - LLM 失败时使用规则标注
4. **质量验证** - 检查标注完整性和一致性
