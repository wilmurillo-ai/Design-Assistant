# 大众点评写作技能

生成自然、口语化的大众点评文案。

## 核心原则

### 1.1 要素选择原则
- 挑 **3-4个** 最想说的要素写，不用写全所有维度
- 每个点评侧重点不同，没有标准答案

### 1.2 结构自由原则
- **"最想说的"放前面**
- 其余要素自然穿插
- 不同点评用不同开头

### 1.3 语言自然原则（最重要！）

**必须删除的词：**
- "首先/其次/最后"
- "一是/二是"
- "总体来说/整体而言"

**口语化技巧：**
| 原版（AI感） | 口语版 |
|-------------|--------|
| 肉质嫩滑 | 肉质**挺**嫩滑**的** |
| 奶香味 | **有**奶香味 |
| 确实上头 | **还挺**上头的 |
| 人均150 | **他家**人均150 |
| 适合约会 | **挺**适合约会的 |

**核心句式：**
- 挺...的：如"挺适合约会的"
- 有+名词：如"有奶香味"
- 他家+描述：如"他家人均100左右"

**句子长短交错**，不要每句都是"XX的XX，XX的XX"

### 1.4 要素库

| 要素 | 说明 | 示例 |
|------|------|------|
| A | 位置/环境 | 望京麒麟社附近、装修挺有韩风的 |
| B | 人均/价格 | 人均100左右、他家人均150 |
| C | 菜品亮点1 | 五花肉外酥里嫩、有奶香味 |
| D | 菜品亮点2 | 牛肋条入味、口感新鲜 |
| E | 小不足 | 蔬菜种类少了点、等位有点久 |
| F | 服务特色 | 有服务员帮忙烤、挺省心的 |
| G | 适合场景 | 朋友聚餐、适合约会、打卡拍照 |

## 组合模式（随机选一种）

### 版本1：环境+菜品导向
`A → B → C → D → E → F → G`

示例：望京麒麟社附近的一家韩式烤肉，店面不大但装修还挺有韩风的。熟成五花肉烤得外酥里嫩，牛肋条也挺入味。人均100左右，有服务员帮忙烤，这点挺省心。适合朋友聚餐。

### 版本2：价格+体验导向
`B → C → D → E → A → F`

示例：人均100左右的韩式烤肉，熟成五花肉确实和其他店不一样，外酥里嫩。牛肋条也入味，就是蔬菜种类少了点。店面装修挺韩风的，服务也不错。可以来试试。

### 版本3：场景+服务导向
`G → F → C → D → B → A`

示例：朋友聚餐来的这家韩式烤肉，有服务员帮忙烤，挺省心的。熟成五花肉外酥里嫩，牛肋条也入味。人均100左右，地点在望京麒麟社这边。推荐来尝尝。

### 版本4：纯亮点版
`A → C → D → B → F → G`

示例：望京麒麟社附近的一家韩式烤肉，熟成五花肉烤得外酥里嫩，肥而不腻，确实和其他烤肉店的不一样。牛肋条也入味，口感挺新鲜的。人均100左右，有服务员帮忙烤，适合朋友聚餐或者打卡拍照。

## 生成流程

### 第1步：接收信息
用户发送菜品照片（1张或多张）

### 第2步：识别菜品
用模型识别每张照片的菜品，列出菜品列表

### 第3步：推断信息
根据菜品推断：
- 餐厅类型（如：韩式烤肉、烤鱼、火锅、面馆等）
- 人均档次（如：30-50元、80-100元、150元以上等）

### 第4步：补充信息
询问用户以下信息：
- 位置/地址
- 环境特点
- 服务特色
- 是否有不足之处
- 适合场景

**注意：** 如果用户已提供部分信息，跳过相应询问

### 第5步：选择要素
从要素库选 **3-4个** 要素，按"最想说的优先"排序

### 第6步：生成文案
按组合模式生成 **3个不同版本**，注意口语化

### 第7步：输出
输出给用户选择

---

## 注意事项

1. **口语化！口语化！口语化！** 多用"挺"、"的"、"有"、"还挺"、"他"
2. **不堆砌形容词** — 不用"色香味俱佳"、"服务周到热情"
3. **有棱角** — 敢写不足，如"蔬菜种类少了点"
4. **像朋友推荐** — 而不是写广告
5. **过渡词全删** — "首先、其次、最后、一是、二是"都不需要
