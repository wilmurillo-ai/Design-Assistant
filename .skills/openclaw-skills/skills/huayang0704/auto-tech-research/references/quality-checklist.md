# 技术调研质量检查清单

确保调研报告质量的系统性检查清单。

## 📊 整体质量评估

### 内容覆盖度（40分）

- [ ] **官方文档**（10分）
  - [ ] 是否查阅了官方文档？
  - [ ] 是否理解了核心概念？
  - [ ] 是否获取了最新版本信息？

- [ ] **学术论文**（10分）
  - [ ] 是否搜索了arXiv/Google Scholar？
  - [ ] 是否找到了核心论文？
  - [ ] 是否理解了技术原理？

- [ ] **工程实践**（10分）
  - [ ] 是否找到了开源实现？
  - [ ] 是否有实际案例？
  - [ ] 是否有最佳实践？

- [ ] **社区观点**（10分）
  - [ ] 是否有社区讨论？
  - [ ] 是否有使用反馈？
  - [ ] 是否有趋势分析？

### 内容质量（30分）

- [ ] **准确性**（10分）
  - [ ] 技术概念是否正确？
  - [ ] 版本信息是否准确？
  - [ ] 数据是否有来源？

- [ ] **时效性**（10分）
  - [ ] 是否优先近2年内容？
  - [ ] 是否标注了发布时间？
  - [ ] 是否反映了最新进展？

- [ ] **完整性**（10分）
  - [ ] 是否覆盖了主要方面？
  - [ ] 是否有足够的细节？
  - [ ] 是否有结论和建议？

### 结构质量（20分）

- [ ] **组织清晰**（10分）
  - [ ] 是否有明确的分级？
  - [ ] 是否有清晰的目录？
  - [ ] 是否有摘要和总结？

- [ ] **呈现友好**（10分）
  - [ ] 是否使用了表格/列表？
  - [ ] 是否有可视化辅助？
  - [ ] 是否有足够的链接？

### 实用价值（10分）

- [ ] **可操作性**（5分）
  - [ ] 是否有明确的学习路径？
  - [ ] 是否有推荐的下一步？
  - [ ] 是否有风险提示？

- [ ] **决策支持**（5分）
  - [ ] 是否回答了用户问题？
  - [ ] 是否有对比分析？
  - [ ] 是否有明确建议？

## 🔍 分级内容检查

### Level 1: 入门科普

**必要性检查**：
- [ ] 是否有概念定义？
- [ ] 是否有通俗解释？
- [ ] 是否有使用场景？

**质量检查**：
- [ ] 是否零基础可读？
- [ ] 是否有生动例子？
- [ ] 是否避免了过度简化？

**推荐标准**：
- [ ] 至少2篇高质量入门文章
- [ ] 至少1个视频教程
- [ ] 官方Quick Start

### Level 2: 实践教程

**必要性检查**：
- [ ] 是否有Step-by-Step步骤？
- [ ] 是否有可运行代码？
- [ ] 是否有预期输出？

**质量检查**：
- [ ] 代码是否完整？
- [ ] 步骤是否清晰？
- [ ] 是否有常见问题？

**推荐标准**：
- [ ] 至少2个完整教程
- [ ] 至少1个示例项目
- [ ] 官方Examples

### Level 3: 深度原理

**必要性检查**：
- [ ] 是否有架构分析？
- [ ] 是否有算法解释？
- [ ] 是否有性能分析？

**质量检查**：
- [ ] 原理是否准确？
- [ ] 分析是否深入？
- [ ] 是否有设计权衡？

**推荐标准**：
- [ ] 至少1篇深度分析
- [ ] 至少1篇论文/源码分析
- [ ] 官方Architecture文档

### Level 4: 前沿动态

**必要性检查**：
- [ ] 是否有最新进展？
- [ ] 是否有未来方向？
- [ ] 是否有社区热点？

**质量检查**：
- [ ] 内容是否最新（<6个月）？
- [ ] 是否标注了不确定性？
- [ ] 是否有多个观点？

**推荐标准**：
- [ ] 至少1篇最新论文
- [ ] 至少1个社区讨论
- [ ] 官方Roadmap

## ⚠️ 常见质量问题

### 内容问题

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 信息过时 | 引用3年前教程 | 增加时间筛选 |
| 来源单一 | 只来自一个平台 | 扩展搜索平台 |
| 深度不足 | 只有入门内容 | 增加学术资源 |
| 缺乏实践 | 只有理论 | 增加教程和代码 |

### 结构问题

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 没有分级 | 内容混在一起 | 强制按Level分类 |
| 缺乏摘要 | 只有链接列表 | 增加内容摘要 |
| 链接失效 | 点击404 | 定期检查和备份 |
| 格式混乱 | 排版不一致 | 使用统一模板 |

### 质量问题

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 错误信息 | 概念解释错误 | 交叉验证 |
| 夸大宣传 | 过度美化技术 | 标注优缺点 |
| 信息过载 | 内容太多无重点 | 精选Top N |
| 信息不足 | 关键内容缺失 | 扩大搜索范围 |

## ✅ 最终检查流程

### 发布前检查

```bash
# 1. 内容完整性检查
python3 check_content.py

# 2. 链接有效性检查
python3 check_links.py

# 3. 格式规范性检查
python3 check_format.py

# 4. 质量评分计算
python3 calculate_score.py
```

### 人工复核要点

1. **快速浏览**（5分钟）
   - 结构是否清晰
   - 是否有明显遗漏
   - 格式是否规范

2. **深度阅读**（15分钟）
   - 内容摘要是否准确
   - 分级是否合理
   - 链接是否有效

3. **价值评估**（5分钟）
   - 是否回答了问题
   - 是否有实用价值
   - 是否值得分享

### 质量等级

| 分数 | 等级 | 说明 | 处理方式 |
|------|------|------|----------|
| 90-100 | 优秀 | 内容全面，结构清晰 | 可直接发布 |
| 80-89 | 良好 | 内容完整，略有不足 | 小修后发布 |
| 70-79 | 合格 | 基本可用，有改进空间 | 需要补充 |
| 60-69 | 待改进 | 有明显缺陷 | 需要重写 |
| <60 | 不合格 | 无法使用 | 重新调研 |
