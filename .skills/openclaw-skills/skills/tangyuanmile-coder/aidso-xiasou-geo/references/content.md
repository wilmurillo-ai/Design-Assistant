# GEO内容生产

## 概述

接口成功返回的是 **Markdown 格式文本**，字段为：

- `data.final_article`

该字段内容已经包含以下全部内容：

1. GEO优化文章（终版）
2. 建议发布网站
3. 基础优化建议
4. 生成依据说明
5. 质检报告

> ⚠️ 成功返回后，必须将 `data.final_article`的 markdown链接 **原样返回给用户**。  
> 不要擅自修改、拆分、补充、重组、润色接口返回的任何内容。
> ⚠️ 本模块只负责 GEO内容生产。
> API key 绑定请查看：references/apikey.md

## 支持的触发方式

以下表达可视为 GEO内容生产意图：

- GEO优化
- 给 XX 生成 GEO优化内容
- 给 XX 生成一篇 GEO文章
- 帮我生成关于 XX 的 GEO优化文章
- 针对某个 AI问题生成 GEO内容
- 给 XX 生成适合大模型引用的内容
- 针对某平台生成 GEO优化内容

## 请求参数

以下表达可视为 GEO内容生产意图：

| 参数 | 类型 | 必填 | 说明 |
| ----- | ----- | ----- | ----- |
| `brand` | string | 是 | 要优化的品牌名称 |
| `issue` | string | 是 | 要优化的AI问题 |
| `platform` | string | 是 | 要优化的大模型平台 |

### 参数说明

1. `brand`
   
品牌名称
不要自行纠正品牌名，不要将用户输入的品牌映射为其他名称。用户输入什么品牌，就传什么品牌。
   
2. `issue`
   
用户希望围绕哪个 AI问题生成 GEO内容。
例如：
- 露露适合什么人群
- 欧莱雅小蜜罐值得买吗
- XX品牌安全吗
- XX品牌和竞品有什么区别
   
3. `platform`
   
用户指定的大模型平台。
调用 API 时，不能直接传中文平台名，必须转换为对应字段值。

## 参数缺失处理规则

### 1. 缺少品牌名称
若用户未提供品牌名称，则不要调用内容生产接口，先追问：

> 这篇 GEO内容是为哪个品牌生成的？请先提供品牌名称。

### 2. 缺少 AI问题
若用户未提供 AI问题，则不要调用内容生产接口，先追问：

> 你希望围绕哪个 AI问题生成 GEO内容？请补充具体问题。

### 3. 缺少大模型平台
若用户未提供平台，则不要调用内容生产接口，先追问：

> 你希望针对哪个大模型平台生成内容？请补充平台名称。

### 4. 平台无法识别
若用户输入的平台名称不在支持范围内，则不要调用内容生产接口，提示用户重新指定支持的平台：

> 当前仅支持以下平台：豆包、DeepSeek、腾讯元宝、千问、百度AI、文心、KIMI、AI抖音。  
> 请补充正确的平台名称后再生成。

## 积分确认规则

正式发起 GEO内容生产前，必须先提示用户确认积分消耗。

当前默认提示文案：

> 本次 GEO内容生产将消耗6积分，是否确认？

### 执行规则

- 用户未确认前，不要调用内容生产接口
- 用户确认后，才正式发起内容生产请求
- 若用户取消，则终止本次内容生产流程

## 内容生产接口

### 接口地址

```
POST https://api.aidso.com/openapi/skills/run_realtime_report
```

### 请求头

```
x-api-key: {用户已绑定的_api_key}
```

### 请求体

```
{
  "brand": "欧莱雅",
  "issue": "30岁左右抗老面霜推荐有哪些？",
  "platform": "DB"
}
```

## 成功返回结构

接口成功返回：

```
{
  "code": 200,
  "msg": "success",
  "data": {
    "final_article": "提示词3文章内容"
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
| ----- | ----- | ----- |
| `final_article` | string | Markdown 格式的最终内容文本 |

### final_article 内容说明

final_article 本身已经包含以下全部内容：

GEO优化文章（终版）
建议发布网站
基础优化建议
生成依据说明
质检报告

⚠️ 不需要 skill 再次拆解这些内容。
不需要 skill 再重组展示顺序。
不需要 skill 再做摘要。
不需要 skill 再生成“修订摘要”。
必须直接原样返回接口里的 final_article。

### 接口返回处理规则

**处理中**

{
    "code": 200,
    "msg": "处理中请稍后"
}

## 自动轮询规则

当内容生产接口返回“处理中”时，应进入自动轮询流程。

### 轮询频率

每 1 分钟轮询一次

### 最长轮询时间

- 最长 30 分钟
- 即最多轮询 30 次

### 轮询执行方式

使用独立轮询脚本

```
python3 scripts/content_poll.py "{brand}" "{issue}" "{platform}"
```

### 轮询方式

继续调用同一个内容生产接口，使用相同请求参数。

### 轮询结束条件

**成功**

满足以下条件即停止轮询并返回结果：

- code == 200
- msg == "success"
- data.final_article 有值

**失败**

不满足以下任一条件即停止轮询并返回错误 msg：

- 返回 data.final_article
- 返回处理中状态

**超时**

若超过 30 分钟仍未完成，回复：

> GEO内容暂未生成完成，请稍后请求获取结果。

## 手动继续查询规则

除自动轮询外，也支持用户手动继续查询当前待生成内容结果。

以下表达可视为继续查询：

- 查询GEO生产内容
- 查询文章

### 执行规则

- 若当前存在待生成任务，则继续调用同一个内容生产接口查询
- 若当前不存在待生成任务，则提示用户重新发起 GEO内容生产请求

推荐提示文案：

> 当前没有待查询的内容生产任务，请重新发起 GEO内容生产请求。
