# GEO 诊断

## 概述

用于发起 AIDSO GEO 品牌诊断，并返回最终诊断报告。

---

## 支持的触发方式

- 帮我做一个 XX 的 GEO 诊断报告
- 生成 XX 的 GEO 诊断报告
- 查一下 XX 的 GEO 表现

---

## 品牌提取规则

当用户输入类似以下表达时，需要从中提取 `brandName`：

- 帮我做一个**露露**的GEO诊断报告
- 做一个**欧莱雅**的GEO诊断报告
- 生成**XX品牌**的GEO诊断报告

### 建议提取规则

可按以下模式识别品牌名称：

- `帮我做一个(.+?)的GEO诊断报告`
- `做一个(.+?)的GEO诊断报告`
- `生成(.+?)的GEO诊断报告`
- `(.+?)的GEO诊断报告`

> ⚠️ 不要自行纠正品牌名，不要把数据外品牌名映射成其他别名。  
> 用户输入什么品牌，就传什么品牌。

---

## 品牌诊断接口

### 接口地址

```
GET https://api.aidso.com/openapi/skills/band_report/md?brandName={brandName}
```

### 请求头

```
x-api-key: {用户已绑定的_api_key}
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|----|----|----|----|
| `brandName` | string | 是 | 要诊断的品牌名称 |

---

## 积分确认规则

正式发起品牌诊断前，必须先提示用户确认积分消耗。

当前默认提示文案：

此次诊断将消耗20积分，是否确认？

执行规则
- 用户未确认前，不要调用诊断接口
- 用户确认后，才正式发起诊断请求
- 若用户取消，则终止本次诊断流程

---

## 诊断发起流程

### 步骤1：检查 API Key

若未绑定，则先进入 API key 绑定流程。

### 步骤2：提取品牌名

从用户请求中识别品牌名，并保存为当前待诊断品牌。

### 步骤3：提示积分确认

向用户提示：

> 此次诊断将消耗20积分，是否确认？

### 步骤4：用户确认后发起诊断

调用品牌诊断接口

---

## 接口返回处理规则

**成功并返回下载地址**

```
{
  "code": 200,
  "msg": "success",
  "data": "https://tcdn.aidso.com/skills/md/xxx.md"
}
```

**处理中**

```
{
  "code": 405,
  "msg": "正在处理中，请稍后",
}
```

**参数错误**

```
{
  "code": 401,
  "msg": "invalid token",
}
```

**积分不足**

```
{
  "code": 401,
  "msg": "积分不足",
}
```

**没有品牌名称**

```
{
    "code": 405,
    "msg": "没有品牌名称,请传入参数{brandName}",
    "data": ""
}
```

## 自动轮询规则

当品牌诊断接口返回“处理中请稍后”时，应进入自动轮询流程。

### 轮询频率

- 每 1 分钟轮询一次

### 最长轮询时间

- 最长 30 分钟
- 即最多轮询 30 次

### 推荐执行方式

使用辅助脚本执行轮询：

```
python3 scripts/diagnosis_poll.py "{brandName}"
```

### 轮询方式

继续调用同一个诊断接口：

```
GET https://api.aidso.com/openapi/skills/band_report/md?brandName={brandName}
x-api-key: {api_key}
```

### 轮询结束条件

**成功**

JSON中返回有效文件链接

**失败**

不满足以下任一条件即停止轮询并返回错误"msg"：

- JSON中返回有效文件链接
- 处理中

**超时**

若超过 30 分钟仍未完成，回复：

> 诊断结果暂未生成完成，请稍后请求获取结果。
> 也可以前往官网查看：https://geo.aidso.com/completeAnalysis

---

## 手动继续查询规则

除自动轮询外，也支持用户手动查询当前待诊断品牌结果。

以下表达可视为继续查询：

- 继续
- 查看结果
- 查询结果
- 查询报告

### 执行规则

- 若当前存在待诊断品牌，则继续调用同一个诊断接口查询
- 若当前不存在待诊断品牌，则提示用户重新发起品牌诊断请求

推荐提示文案：

> 当前没有待查询的诊断任务，请重新发起品牌诊断请求。

---

## 返回结果处理规则

接口返回的是 Markdown 链接

- 直接返回 Markdown 链接
- 拉取 Markdown 内容并返回

> ⚠️ 成功返回后，必须将接口返回的 Markdown 链接 **原样返回给用户**。  
> 不要擅自修改、拆分、补充、重组、润色接口返回的任何内容。
