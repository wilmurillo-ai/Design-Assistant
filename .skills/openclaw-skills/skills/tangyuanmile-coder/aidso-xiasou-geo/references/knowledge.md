# 品牌知识库

## 概述

品牌知识库用于补充 AI 对品牌的理解。

用户上传品牌知识内容时，必须明确该内容属于哪个品牌的知识库。  
因此，无论上传的是纯文本还是 `.md` 文件，上传时都必须提供 `brand_name`。

> ⚠️ 当前为知识库 **V1 简化版**。  
> 本版本只支持 **上传知识库内容**，**不支持修改、查询、删除**。  
> 仅支持上传：
> - “纯文本”
> - “Markdown 文件（.md）”

---

## 使用场景

品牌知识库适合以下情况：

- 品牌是新品牌、小品牌，互联网信息较少
- AI 对品牌认知不完整，影响 GEO 诊断准确性
- AI 内容生产与品牌实际情况不够贴合
- 用户希望上传品牌介绍、产品资料、卖点、场景、FAQ、品牌故事等资料，补充品牌信息基础

通过上传品牌知识库，后续 AI 在做 GEO 诊断与内容生产时，可以更贴近品牌真实信息。

## 当前支持范围

### ✅ 支持
- 上传纯文本知识
- 上传 Markdown 文件（`.md`）

### ❌ 不支持
- 查询知识库
- 修改知识库
- 删除知识库
- 上传链接
- 上传 PDF / Word / Excel / PPT / 图片 / 其他非 `.md` 文件

---

## 支持的知识库类型

| 类型 | 说明 | 支持 |
|------|------|------|
| `plain_text` | 纯文本 | ✅ 上传 |
| `file` | Markdown 文件（仅 .md，上传时自动转成文本） | ✅ 上传 |

---

## 不支持的文件类型

若用户上传以下类型文件：

- `.pdf`
- `.doc`
- `.docx`
- `.txt`
- `.xls`
- `.xlsx`
- `.ppt`
- `.pptx`
- 图片
- 其他非 `.md` 文件

则不要调用上传接口，直接提示用户：

> 当前品牌知识库仅支持上传纯文本或 `.md` 文件。  
> 请先将文件转换成 md 格式后再上传。

---

## 支持的触发方式

以下表达可视为品牌知识库上传意图：

- 上传品牌知识库
- 上传品牌资料
- 把这段内容上传到品牌知识库
- 把这个 md 文件上传到品牌知识库
- 先补充品牌资料到知识库
- 把品牌介绍加入知识库

---

## 上传输入规则

上传品牌知识库内容时，必须同时提供：

- `brand_name`：该内容所属品牌
- `content`：知识内容文本

### 1. 纯文本上传
当用户直接输入大段品牌资料文本时，视为纯文本上传。

### 2. Markdown 文件上传
当用户上传 `.md` 文件时：

- 先读取 `.md` 文件内容
- 将文件内容转成文本
- 再按纯文本方式上传

> ⚠️ `.md` 文件只是用户上传形式，不作为独立 API 类型处理。  
> 调用 API 时，统一按文本内容上传。

### 3. 品牌名称缺失
若用户上传知识内容时未提供品牌名称，则不要调用上传接口，应先追问：

> 这份知识内容属于哪个品牌？请输入品牌名称后再上传。

---

## 上传流程

### 步骤 1：检查 API key
若未绑定，则先进入 API key 绑定流程。

### 步骤 2：检查品牌名称
若用户未提供 `brand_name`，则不要上传，先要求用户补充品牌名称。

### 步骤 3：识别上传内容类型

#### 情况 A：纯文本
若用户输入的是文本内容，则直接按文本上传。

#### 情况 B：Markdown 文件
若用户上传的是 `.md` 文件：
- 先读取 `.md` 文件内容
- 将文件内容转成文本
- 再按文本方式上传

#### 情况 C：其他文件
若用户上传的是非 `.md` 文件，则拒绝上传并提示转换格式。

### 步骤 4：调用知识库上传接口
调用 API 时统一传：

- `brand_name`
- `content`

### 步骤 5：返回上传结果
- 成功：告知用户已上传成功
- 失败：返回后端错误文案
- API key 无效：要求重新绑定

---

## 知识库上传接口

### 接口地址

```
POST  https://api.aidso.com/openapi/skills/save_content/md
```

### 请求头

```
x-api-key: {用户已绑定的_api_key}
```

---

## 请求参数

```
{
    "content": "知识内容",
    "brand_name": "品牌名称"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `brand_name` | string | 是 | 知识内容所属品牌 |
| `content` | string | 是 | 知识内容正文；若用户上传的是 .md 文件，则传入读取后的 Markdown 文本内容 |

### 返回结构

{
    "code": 200,
    "msg": "success",
    "data": true
}
