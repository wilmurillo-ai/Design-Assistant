## 概览
本规范定义生成的每日资讯日报的格式要求，确保输出内容结构清晰、易于阅读。

## 日报结构

### 标准结构
1. **报头**
   - 日报标题
   - 日期
   - 统计数据

2. **分类板块**
   - 按主题分类的资讯列表
   - 每个板块包含相关资讯

3. **摘要**
   - 当日重点摘要（可选）

4. **报尾**
   - 数据来源说明
   - 生成时间

## 格式要求

### 1. 报头格式
```markdown
# [主题] 日报 - YYYY-MM-DD

**生成时间**: YYYY-MM-DD HH:MM:SS
**资讯总数**: XX 条
**分类数量**: XX 个
```

### 2. 分类板块格式
每个分类使用二级标题，资讯使用有序列表：

```markdown
## 分类名称

### [来源] 文章标题
- **链接**: URL
- **摘要**: 简要内容描述（1-2句话）
- **发布时间**: 时间信息（可选）

### [来源] 文章标题
- **链接**: URL
- **摘要**: 简要内容描述
```

### 3. 每日摘要格式（可选）
```markdown
## 今日重点摘要

- 要点1：重要资讯简述
- 要点2：重要资讯简述
- 要点3：重要资讯简述
```

### 4. 报尾格式
```markdown
---

**数据来源**: RSS订阅源、网页抓取
**生成工具**: Daily News Collector Skill
**备注**: 本报告由AI智能体自动生成
```

## 内容规范

### 1. 资讯标题
- 使用来源名称作为前缀，如 `[36氪]`
- 标题保持原文，不超过50个字
- 避免使用特殊符号和emoji

### 2. 摘要要求
- 简明扼要，1-2句话概括核心内容
- 字数控制在50-100字
- 突出关键信息（新产品、新技术、重要事件等）
- 避免重复原文中的冗长描述

### 3. 分类规则
- 按内容主题分类，常见分类：科技、商业、人工智能、综合
- 同一分类下的资讯按相关性排序
- 每个分类至少包含2条资讯，少于2条的合并到"其他"分类

### 4. 排序规则
- 每个分类内按发布时间倒序排列
- 同一时间发布的按相关性排序
- 重点摘要放在最前面

### 5. 格式一致性
- 统一使用Markdown格式
- 链接使用完整URL
- 时间格式统一为：YYYY-MM-DD 或 YYYY-MM-DD HH:MM

## 注意事项
1. 保持简洁，避免冗长描述
2. 确保链接可访问
3. 摘要要客观准确，不添加主观评价
4. 定期检查格式规范，保持一致性
5. 根据用户需求可适当调整分类和结构
