## 发债企业基本信息

接口路径：`chain/bond-issuers`
请求方式：**`POST`**
tool_id：`get_chain_bond_issuers`

接口说明：通过企业ID或企业名称（支持模糊匹配）查询上市发债企业的基本信息，包括企业统一社会信用代码、中英文名称、是否为发债公司、是否为上市公司以及相关机构ID等核心标识信息，用于快速识别和筛选符合条件的发债主体。

**输入参数**

| 参数名 | 必填 | 类型 | 说明 | 示例 |
|--------|:----:|------|------|------|
| `companyName` | ✅ | string | 企业名称 | `广东粤运交通股份有限公司` |

**输出参数**

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `companyId` | 企业ID | `CSF0000000010` |
| `creditCode` | 统一社会信用代码 | `9144xxx0719285123G` |
| `companyName` | 企业名称 | `广东粤运交通股份有限公司` |
| `companyNameEn` | 企业英文名称 | `Guangdong Yueyun Transportation Company Limited` |
| `isBondIssuer` | 是否为发债公司 | `1` |
| `isListed` | 是否为上市公司 | `1` |
| `refCompanyId` | 相关机构id | `CSF0000000010` |

**接口示例**

```bash
investoday-api chain/bond-issuers --method POST companyName=广东粤运交通股份有限公司
```

## 证券基础信息(产业链）

接口路径：`chain/sec-basic-info`
请求方式：**`POST`**
tool_id：`get_chain_sec_basic_info`

接口说明：通过股票代码（可批量）查询证券的基础信息，包括股票代码、中英文名称、所属公司信息、市场信息、上市状态、上市日期和退市日期，适用于获取股票的基本档案和上市状态。

**输入参数**

| 参数名 | 必填 | 类型 | 说明 | 示例 |
|--------|:----:|------|------|------|
| `stockCodes` | — | array | 交易代码 | `['000001', '600519']` |
| `pageNum` | — | integer | 页码。 最小值:1; | `1` |
| `pageSize` | — | integer | 页长。 最小值:1; 最大值:500; | `10` |

**输出参数**

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `stockCode` | 交易代码 | `002054` |
| `stockName` | 证券中文简称 | `德美化工` |
| `stockNameEn` | 证券英文简称 | `Dymatic Chemicals` |
| `companyId` | 机构标准代码 | `CSF0000000712` |
| `companyName` | 机构中文名称 | `广东德美精细化工集团股份有限公司` |
| `companyNameEn` | 机构英文名称 | `Dymatic Chemicals,Inc.` |
| `marketCode` | 市场编码 | `1002` |
| `marketName` | 市场中文名称 | `深圳证券交易所主板A股` |
| `marketNameEn` | 市场英文名称 | `SZSE Main Board A Share` |
| `listStatus` | 上市状态编码 | `LT` |
| `listDate` | 上市日期 | `2006-07-25 00:00:00.000` |
| `delistDate` | 退市日期 | `2025-07-25 00:00:00.000` |

**接口示例**

```bash
# 可选参数: stockCodes, pageNum, pageSize
investoday-api chain/sec-basic-info --method POST
```

## 行业信息(产业链）

接口路径：`chain/industry-info`
请求方式：**`POST`**
tool_id：`c`

接口说明：通过行业代码或行业名称（支持模糊匹配）查询产业链行业分类信息，返回行业代码、名称、父级代码、所有祖先代码、分类体系（如CSF、CSRC等）及行业层级等核心数据，用于构建和分析产业链上下游关系及行业分类体系。

**输入参数**

| 参数名 | 必填 | 类型 | 说明 | 示例 |
|--------|:----:|------|------|------|
| `industryName` | — | string | 行业名称【与industryCode组成多选多参数，必须传递其中一个至多个】 | `商品化工` |
| `industryCode` | — | string | 行业编码【与industryName组成多选多参数，必须传递其中一个至多个】 | `CSF_15101010` |

**输出参数**

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `industryCode` | 行业编码 | `CSF_101010` |
| `industryName` | 行业中文名称 | `能源设备与服务` |
| `parentCode` | 行业父层级 | `CSF_1010` |
| `ancestorsCodes` | 行业所有父层级 | `CSF_10, CSF_1010` |
| `industrySystem` | 行业分类体系 | `CSF` |
| `industryLevel` | 行业层级 | `3` |

**接口示例**

```bash
# 可选参数: industryName, industryCode
investoday-api chain/industry-info --method POST
```

## 供应链产品信息

接口路径：`chain/product-info`
请求方式：**`POST`**
tool_id：`get_chain_product_info`

接口说明：通过产品代码或产品名称（支持模糊查询）查询供应链产品信息，返回产品代码、名称、所属行业代码、产品层级、直属父级产品代码以及所有祖先层级产品代码，用于构建和分析产品层级关系及行业归属。

**输入参数**

| 参数名 | 必填 | 类型 | 说明 | 示例 |
|--------|:----:|------|------|------|
| `productCode` | — | string | 产品编码 | `P0007784` |
| `productName` | — | string | 产品名称 | `白茶` |

**输出参数**

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `parentCode` | 产业代码 | `P0007783` |
| `productName` | 产业名称 | `白茶` |
| `industryCode` | 行业代码 | `CSF_30202010` |
| `productCode` | 直属父层级产品编码 | `FP00100303` |
| `ancestorsCodes` | 所有父层级产品编码 | `FP001, FP00100303` |
| `productLevel` | 产业层级 | `3` |

**接口示例**

```bash
# 可选参数: productCode, productName
investoday-api chain/product-info --method POST
```
