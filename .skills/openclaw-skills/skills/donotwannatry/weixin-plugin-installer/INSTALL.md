# 微信插件安装指南

## 快速开始

本 skill 面向管理员私聊场景，不直接暴露原始 shell 命令给用户。

管理员触发“安装微信连接”后，固定脚本会依次执行：

1. 安装微信插件
2. 启用微信插件
3. 判断 Gateway 的重载模式
4. 如果插件已在当前运行态可用，则继续生成最新二维码
5. 如果刚完成安装或启用，则先返回“等待 Gateway 完成重载”的提示，下一条“刷新微信二维码”再继续
6. 检测到微信确认连接成功后，后台延迟执行 `openclaw gateway restart`

## 为什么不能在当前聊天里直接重启 Gateway

微信插件属于需要重载或重启才能生效的基础设施变更。

但如果当前这条“安装微信连接”消息本身就是通过 OpenClaw 网关处理的，直接在同一轮里执行 `openclaw gateway restart`，会把当前消息链路一起中断，导致回复发不出去。

因此本 skill 的策略是：

1. 安装并启用插件
2. 等待 Gateway 自动重载，或由管理员在宿主机外部重启
3. 再通过下一条“刷新微信二维码”消息进入二维码登录流程
4. 检测到用户已确认连接成功后，再在后台异步重启 Gateway

## 查询状态

管理员通过 skill 里的固定脚本查询当前二维码状态，不直接调用未白名单的命令。

## 取消配对

管理员通过 skill 里的固定脚本取消当前二维码刷新任务。
