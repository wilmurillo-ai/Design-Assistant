## 二、检测指标详解

### 2.1 评价指标

**Precision（精确率）**：预测为正例中实际为正例的比例
```
Precision = TP / (TP + FP)
```
- 高精确率 = 低误报率
- 关键场景：避免频繁无效报警

**Recall（召回率）**：实际正例中被预测为正例的比例
```
Recall = TP / (TP + FN)
```
- 高召回率 = 低漏报率
- 关键场景：不能漏掉真实事件

**F1分数**：精确率和召回率的调和平均
```
F1 = 2 × P × R / (P + R)
```

### 2.2 各类别检测特点

**拥堵（is_congestion）**：
- Recall高，Precision低
- 模型过于敏感，易误报
- 建议：调整阈值，增加负样本

**异常停车（is_abnormal_stop）**：
- Recall低，Precision高
- 模型过于保守，易漏检
- 建议：增加正样本，降低阈值

**非法车辆（is_illegal_vehicle）**：
- 仅Qwen-235B表现良好（F1=0.86）
- 其他模型几乎全部失效
- 建议：专用模型或规则辅助

**行人（is_pedestrian）**：
- Qwen系列Recall优秀（0.81）
- Think模式会降低Recall
- 建议：使用非Think版本

**事故（is_accident）**：
- Qwen-30B Recall最高（0.83）
- 大模型反而容易漏检
- 建议：小模型初筛，大模型复核

### 3.1 双模型方案

```
[视频输入]
```

- 资源充足

## 四、数据标注规范

### 4.1 标注要求

| 要求 | 说明 |
|-----|-----|
| 时间精度 | 精确到秒 |
| 空间范围 | 标注目标边界框 |
| 多标签 | 允许同时存在多个事件 |
| 层级关系 | 主事件+伴随事件 |

### 4.2 标注质量检查

**一致性检查**：
- 多人标注交叉验证
- 计算标注一致性系数

**完整性检查**：
- 验证时间连续性
- 检查遗漏标注

## 五、误报分析

### 5.1 常见误报原因

| 原因 | 示例 | 解决方案 |
|-----|-----|---------|
| 光照变化 | 阴影误判为异常停车 | 增加多样本 |
| 视角问题 | 远处车辆误判为行人 | 多角度训练 |
| 天气影响 | 雨雾天气误判 | 天气适应训练 |
| 动态背景 | 施工区域误判 | 区域标注 |

### 5.2 漏报分析

**罕见事件漏报**：
- 火灾样本少（4个）
- 抛洒物样本少（2个）
- 解决方案：数据增强、迁移学习

**小目标漏报**：
- 远处行人
- 小型抛洒物
- 解决方案：多尺度检测、注意力机制

## 六、系统优化建议

### 6.1 数据层面

1. **样本均衡**：增加罕见事件样本
2. **数据增强**：旋转、裁剪、色彩变换
3. **负样本挖掘**：增加难负样本

### 6.2 模型层面

1. **多尺度检测**：处理不同大小目标
2. **时序建模**：利用视频时序信息
3. **注意力机制**：关注关键区域

### 6.3 后处理层面

1. **时间平滑**：连续帧结果平滑
2. **置信度校准**：调整输出概率
3. **规则过滤**：业务规则约束
