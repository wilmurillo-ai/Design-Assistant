# 安全说明
## 🔒 安全承诺
我们高度重视用户的信息安全和隐私保护，严格遵循最小权限原则和安全开发规范，确保用户数据安全。

## 🛡️ 安全特性
### 1. API Key 安全存储
- 用户配置的所有API Key由OpenClaw平台统一**加密存储**，密钥由平台管理
- Skill代码中仅通过配置对象动态读取Key，不会持久化存储任何用户密钥
- 所有日志输出中的敏感信息会自动掩码处理，仅显示前4位和后4位

### 2. 数据隔离机制
- 每个用户的请求上下文完全隔离，不会出现跨用户数据访问
- 用户偏好、查询历史、使用统计等数据独立存储，用户之间完全隔离
- 支持用户一键删除自己的所有数据，符合隐私法规要求

### 3. 注入攻击防御
- **三重提示词注入防御机制**：
  1. 输入层：所有用户输入经过严格验证和过滤
  2. 检测层：基于正则和关键词的恶意内容检测
  3. 输出层：结果返回前再次进行安全检查
- 支持拦截系统指令覆盖、数据泄露诱导、代码执行诱导等常见注入攻击

### 4. 权限控制
- 仅申请必要的运行权限：网络访问、本地存储、消息发送、定时任务
- 文件系统访问限制在Skill专属隔离目录：`~/.openclaw/skills/travel-ai/`
- 无系统级调用权限，完全兼容OpenClaw安全模式(--safe-mode)

### 5. 第三方依赖安全
- 仅调用OpenClaw官方维护的Skill（baidu-search、multi-search-engine等）
- 所有第三方Skill都经过ClawHub安全审计，无恶意代码
- 调用第三方Skill时仅传递必要参数，不会泄露用户API Key
