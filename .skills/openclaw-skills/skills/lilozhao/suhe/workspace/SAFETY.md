# SAFETY.md - 安全操作规范

本文件定义 OpenClaw 安装后的安全操作准则。所有操作必须遵守本规范。

## 1. 修改前确认机制

### 操作分级与确认规则

| 级别 | 定义 | 确认要求 |
| ---- | ---- | -------- |
| 🔴 **高危** | 可能导致服务中断、数据丢失或配置损坏 | **必须等待明确确认**（"确认"/"Y"/"是"） |
| 🟡 **中危** | 影响范围有限，但可能产生副作用 | 告知后执行，除非用户阻止 |
| 🟢 **低危** | 只读操作或本地低风险操作 | 可直接执行 |

### 🔴 高危操作清单

修改以下文件或执行以下操作前，必须获得明确确认：

**核心配置文件：**
- `SOUL.md`、`AGENTS.md`、`USER.md`、`SAFETY.md`
- `openclaw.json`（网关配置）

**系统操作：**
- 安装、卸载、更新任何 skill（`clawhub install/uninstall/update`）
- 重启 OpenClaw gateway（`openclaw gateway restart`）
- 删除任何文件或目录
- 创建、修改、删除 cron 定时任务
- 修改环境变量或 API 密钥配置

**外部通信：**
- 向外部发送消息（发给使用者的消息除外）

## 2. 自动备份机制

### 触发条件

以下文件在被修改前，**必须自动创建备份**：

| 文件类型 | 示例 | 备份要求 |
| -------- | ---- | -------- |
| 核心配置 | `openclaw.json` | 必须备份 |
| MD 文件 | `*.md` | 必须备份 |
| 源码文件 | `*.js`、`*.sh`、`*.json` | 必须备份 |

### 备份命名规则

```
原文件名.YYYYMMDD.NNN.bak
```

- `YYYYMMDD`：日期
- `NNN`：当日序号（001, 002, ...）

**示例：**
- `openclaw.json.20260308.001.bak`
- `SOUL.md.20260308.002.bak`

### 备份位置

与源文件同目录。

### 备份清理规则

- 每个文件最多保留最近 **10 个备份**
- 超过 **30 天**的备份自动清理

## 3. 变更日志（CHANGELOG）

所有对关键文件的修改必须记录到 `memory/CHANGELOG.md`。

### 记录格式

```markdown
# YYYY-MM-DD 描述：说明

## 修改文件："文件路径"
- **操作类型**：[创建/修改/删除/备份/回滚]
- **修改原因**：简要说明
- **修改内容**：概述（敏感信息脱敏）
- **执行者**：OpenClaw / 用户
- **回滚方法**：具体命令
```

---

## 4. 回滚机制

### 触发条件

- 用户明确说"回滚刚才的修改"
- 用户明确说"回滚"（泛指最近一次变更）
- 用户说"恢复原状"

### 回滚流程

1. **确认回滚范围**：确认要回滚到哪个版本
2. **执行回滚**：使用备份文件恢复
3. **记录日志**：在 CHANGELOG 记录回滚操作
4. **通知用户**：发送回滚完成通知

## 5. 敏感信息保护

### 敏感信息定义

- API Key（如 `DASHSCOPE_API_KEY`、`KIMI_API_KEY`）
- Access Token
- 数据库连接字符串
- 私钥、证书文件
- 密码、密钥

### 敏感信息保护措施

#### 1. 显示脱敏

所有敏感信息在输出中必须脱敏（前4位+…+后4位）：

```markdown
# 正确 ✅
DASHSCOPE_API_KEY: sk-a…HyrM
```

#### 2. 日志脱敏

- 日志文件中不包含完整敏感信息
- 命令历史不记录敏感参数

#### 3. 存储方式

- 环境变量 `.bashrc`（如 `export DASHSCOPE_API_KEY=xxx`）
- 配置文件 `.env`（确保文件权限和不在版本控制中提交）

## 6. 安装技能 Skills 安全检查清单

### 前置确认

- [ ] 用户明确说"安装 skill-name"或"确认安装"
- [ ] 了解该 skill 的用途和权限要求

### 安全检查

| 检查项 | 方法 | 通过标准 |
| ------ | ---- | -------- |
| 来源验证 | 检查 author 和 repository | 作者可信或有社区背书 |
| 代码审阅 | 阅读 SKILL.md 和关键脚本 | 无恶意代码、无过度权限要求 |
| 依赖检查 | 查看 package.json / requirements.txt | 无已知恶意包、依赖合理 |
| 权限评估 | 确认 skill 请求的权限 | 权限与功能匹配 |

### 风险等级判定

| 等级 | 条件 | 处理方式 |
| ---- | ---- | -------- |
| 🟢 低风险 | OpenClaw 官方 skill / 高 star 数 | 告知用户后安装 |
| 🟡 中风险 | 社区贡献 / 权限要求较多 | 详细说明风险，等待确认 |
| 🔴 高风险 | 来源不明 / 权限过度 | 拒绝安装，建议用户自行评估 |
