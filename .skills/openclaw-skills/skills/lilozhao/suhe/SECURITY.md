# 安全政策

## 支持版本

目前，只有最新版本的 Ruolan 项目接收安全更新。如果您使用的是旧版本，请升级到最新版本以获得安全保护。

## 报告安全漏洞

如果您发现了安全漏洞，请通过以下方式向我们报告：

- **邮箱**: security@sumelabs.ai
- **GitHub**: 提交私密安全漏洞报告

请不要在公开的 Issue 中报告安全漏洞。

### 报告内容应包括

- 漏洞的详细描述
- 复现漏洞的步骤
- 潏洞的影响范围
- 可能的修复建议（如有）

## 安全最佳实践

### 对于用户

- 不要在配置文件中硬编码 API 密钥和其他敏感信息
- 定期更换 API 密钥
- 仅从官方渠道下载和安装 Ruolan
- 保持 OpenClaw 和相关组件更新到最新版本

### 对于开发者

- 严格遵守 [.gitignore](./.gitignore) 规则，防止敏感信息泄漏
- 在提交代码前检查是否包含敏感数据
- 使用环境变量管理 API 密钥和认证凭据
- 对用户输入进行适当验证和过滤

## 安全审计

我们定期对代码库进行安全审计，确保项目的安全性。如果您希望对项目进行安全审计并分享结果，请随时联系我们。
