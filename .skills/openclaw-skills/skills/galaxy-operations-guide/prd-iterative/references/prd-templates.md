# PRD 标准结构参考

## 完整版 PRD（适用复杂产品）

### 1. 产品概述
- **背景**：市场/用户/技术趋势
- **问题**：用户当前痛点
- **解决方案**：产品如何解决
- **产品定位**：一句话说明产品是什么

### 2. 用户画像
- 目标用户描述
- 用户场景
- 用户动机
- 用户障碍

### 3. 核心功能
按优先级分：
- **P0**（必须上线）
- **P1**（下个版本）
- **P2**（规划中）

### 4. 信息架构
- 功能结构图
- 页面清单
- 导航设计

### 5. 交互流程
- 核心路径（用户完成任务的主要流程）
- 异常路径（错误/空状态/边界情况）

### 6. 数据埋点
| 事件ID | 事件名称 | 触发时机 | 参数 |
|--------|---------|---------|------|
| view_home | 首页浏览 | 进入首页 | user_id, time |
| click_xxx | 点击xxx | 点击行为 | element_id, position |

### 7. MVP 范围
| 类型 | 功能 | 说明 |
|------|------|------|
| 必须 | ... | ... |
| 可选 | ... | ... |
| 排除 | ... | ... |

### 8. 非功能需求
- 性能要求
- 兼容性
- 监控指标

### 9. 附录
- 术语表
- 竞品对比
- 设计稿链接
