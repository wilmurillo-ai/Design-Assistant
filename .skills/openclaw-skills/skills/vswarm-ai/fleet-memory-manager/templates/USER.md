# USER.md — User Profile

<!--
=============================================================================
LAYER 3: TACIT KNOWLEDGE — USER PROFILE
=============================================================================

This is the tacit knowledge layer — the stuff that would take a new hire
weeks to learn about how your human works. Your agent builds this up over
time and it becomes one of the most valuable files in the system.

Think of it as: "everything I know about this person that changes how I
should work with them."

WHY THIS FILE EXISTS:
  Generic models give generic answers. An agent that knows YOU gives answers
  calibrated to your context, preferences, and working style. The difference
  between a competent assistant and an indispensable one is often just this file.

HOW TO FILL THIS IN:
  Start with what you know. Add to it as you observe patterns.
  When your human says "I prefer X" or "don't do Y" → update this file.
  When you notice a consistent pattern → add it here.

HOW TO USE THIS:
  Read this file every session. Let it shape HOW you respond, not just WHAT
  you respond with. A human who prefers brevity should get brief answers.
  A human who works late nights needs different check-in timing than one
  who works 9-5.

LOADING RULE:
  Load in both main sessions AND group/channel sessions.
  It helps calibrate responses without containing sensitive personal data.
  (Contrast with MEMORY.md which should only load in private sessions.)
=============================================================================
-->
