# 安全检查清单

## 输入处理
- 所有用户输入必须验证和消毒（sanitize）
- SQL 查询使用参数化查询，禁止字符串拼接
- 文件路径操作防范路径穿越（path traversal）
- 正则表达式防范 ReDoS

## 认证授权
- 密码和密钥禁止硬编码
- 敏感配置使用环境变量
- API 端点必须有权限校验
- Session/Token 有合理过期策略

## 数据安全
- 敏感数据不出现在日志中
- API 响应不泄漏内部实现细节
- 错误信息对用户友好且不暴露技术栈
- PII 数据加密存储

## 依赖安全
- 不引入有已知漏洞的依赖版本
- 优先使用社区活跃维护的库
