# 工作流组合

## 技能介绍

工作流组合是求是 OpenClaw Skills 系列中的核心技能，用于当一个任务明确需要多种方法串联时，组合使用多个技能形成完整的工作流程。

## 核心功能

- **任务分析**：分析任务的性质和需求
- **技能选择**：选择适合的技能组合
- **顺序确定**：确定技能的调用顺序
- **流程设计**：设计详细的工作流程
- **执行监控**：监控工作流程的执行情况

## 触发条件

当一个任务明确需要多种方法串联，需要组合使用多个技能时触发。

## 执行步骤

1. **任务分析**：分析任务的性质和需求
2. **技能选择**：选择适合的技能组合
3. **顺序确定**：确定技能的调用顺序
4. **流程设计**：设计详细的工作流程
5. **执行监控**：监控工作流程的执行情况
6. **调整优化**：根据执行情况调整工作流程

## 核心原则

根据任务需求，灵活组合使用各种技能，形成完整的工作流程。

## 不适用场景

- 任务只需要单一技能
- 任务流程简单，不需要复杂的技能组合
- 时间紧急，不需要详细的工作流程设计
- 已经有明确的工作流程

## 工作流组合的应用

- **复杂项目**：处理复杂的项目任务
- **多阶段任务**：处理需要多个阶段的任务
- **跨领域任务**：处理涉及多个领域的任务
- **创新任务**：处理需要创新思维的任务

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 技能选择不当 | 根据任务需求选择合适的技能 |
| 顺序安排不合理 | 合理安排技能的调用顺序 |
| 流程设计过于复杂 | 设计简洁有效的工作流程 |
| 缺乏监控和调整 | 监控执行情况，及时调整流程 |
| 忽视技能间的协同 | 注重技能间的协同效应 |

## 操作规程

当本 skill 被触发时，执行以下步骤：
1. **任务分析**：详细分析任务的性质、需求和挑战
2. **技能评估**：评估各种技能的适用性和可行性
3. **技能选择**：选择最适合的技能组合
4. **顺序设计**：设计合理的技能调用顺序
5. **流程制定**：制定详细的工作流程，包括每个阶段的目标和任务
6. **资源规划**：规划所需的资源和支持
7. **执行实施**：按照工作流程执行，监控执行情况
8. **评估调整**：评估工作流程的效果，根据情况调整流程

## 与其他 skill 的关系

- **矛盾分析法**：工作流组合需要分析任务中的矛盾
- **实践认识论**：工作流组合需要通过实践不断调整和优化
- **集中兵力**：在工作流程中，需要在关键阶段集中兵力
- **统筹兼顾**：工作流组合需要统筹兼顾各方面的因素
