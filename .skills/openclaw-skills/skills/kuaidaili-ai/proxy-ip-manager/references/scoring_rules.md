# 评分规则说明

## IP质量评分

### 评分公式

```
quality_score = success_rate * 0.7 + latency_score * 0.3
```

### 成功率权重（70%）

| 成功率 | 贡献分 |
|--------|--------|
| 100% | 70 |
| 95% | 66.5 |
| 80% | 56 |
| 60% | 42 |
| 50% | 35 |
| <50% | <35 |

### 延迟评分（30%）

| 响应时间 | 延迟评分 |
|---------|---------|
| < 1秒 | 100 |
| 1-3秒 | 80 |
| > 3秒 | 50 |

### 评分示例

| 成功率 | 平均延迟 | 延迟评分 | 最终评分 |
|--------|---------|---------|---------|
| 100% | 500ms | 100 | 100 |
| 95% | 1.5s | 80 | 90.5 |
| 80% | 2s | 80 | 80 |
| 60% | 3.5s | 50 | 57 |
| 50% | 4s | 50 | 50 |

## 健康状态分级

| 评分范围 | 状态 | 说明 |
|---------|------|------|
| 80-100 | 优秀 | 可放心使用 |
| 60-79 | 良好 | 正常使用 |
| 40-59 | 一般 | 建议监控 |
| 0-39 | 较差 | 建议检查 |

## 并发调整规则

### 基于成功率的调整

| success_rate | 并发倍数 |
|-------------|---------|
| ≥ 95% | base × 1.0 |
| 80-94% | base × 0.7 |
| 60-79% | base × 0.5 |
| 50-59% | base × 0.3 |
| < 50% | base × 0.2 |

### 失败保护

| 条件 | 动作 |
|------|------|
| 连续失败 > 10 | 并发降低50% |
| success_rate < 50% | 并发降至最低(1) |
