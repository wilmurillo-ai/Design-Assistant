# Writing a Custom SOUL.md

A SOUL.md defines who the persona *is*. It's the system prompt personality layer.

## Structure

Follow the pattern from the Emblem persona marketplace:

```markdown
# SOUL.md — <Name>

_<One-line motto or catchphrase>_

## Core Truths
What this persona believes. 3-6 bullet points that define behavior.

## Do NOT
Explicit anti-patterns. What this persona never does. Critical for tone control.

## Tone
2-3 sentences describing communication style.

## Quirks
Behavioral details that make the persona feel real. Sentence structure, habits, verbal tics.

## Expertise
Domain knowledge areas. What this persona is actually good at.

## Backstory
Brief origin story. Provides grounding for behavior and motivation.
```

## Tips

- **Tone is everything.** The difference between "persona" and "costume" is whether the tone is consistent.
- **Negative constraints matter more than positive ones.** "Never be verbose" shapes behavior more than "be concise."
- **Quirks sell authenticity.** Short sentences. Fragments. Repeated phrases. These make a persona recognizable.
- **Expertise should be real.** A persona that claims to be a security expert should have security-relevant behavioral patterns, not just the label.
- **Keep it under 1500 words.** Persona prompts compete for context with the actual task. Concise > comprehensive.

## IDENTITY.md

Minimal identity card:

```markdown
- **Name:** <Display Name>
- **Creature:** <What they are, one line>
- **Vibe:** <3-4 adjectives>
- **Emoji:** <Single emoji>
- **Inspired by:** <Source character, if applicable>

---

<1-2 sentence self-description in first person>
```
