# 新增方法准入与挂接协议（Method Addition Protocol）

## 目的

在持续新增方法时，确保：
1. 不与现有方法重复。
2. 结构层级清晰（主卡/子卡）。
3. 放置位置可解释、可维护。
4. 输出模板统一，调用成本可控。

## 一、新增前必做判定（4步）

1. 场景判定：该方法解决什么问题？
2. 关系判定：是新方法、细化、还是组合？
3. 模块判定：应落在哪个模块（A-F）？
4. 形态判定：主卡还是子卡？

## 二、模块挂接规则

- A 问题定义与结构化：问题界定、框架搭建、服务匹配。
- B 分析工具与根因：脑图/鱼骨/树图/系统展开/原因分解。
- C 战略与组织诊断：战略一致性、组织协同、行业/能力诊断。
- D 营销与增长：销售困境、增长因果链、对标增长。
- E 流程与BPR：流程优化、流程再造、组织承接。
- F 追问与收敛：5W2H/28问/5WHY类追问体系。

当方法跨两个模块时：
1. 选一个主模块存放。
2. 在另一个模块仅放“调用链接+触发条件”，不复制正文。

## 三、关系判定与处理动作

### 1) 新方法（独立）

判定：与现有方法核心目标和操作步骤重叠度 < 40%。
动作：新增`### 方法N`。

### 2) 细化方法（子卡）

判定：重叠度 40%-70%，但在某一步骤深度明显增强。
动作：作为“方法X子卡A/B”并挂到主卡下。

### 3) 组合方法（工作底稿）

判定：重叠度 > 70%，本质是多方法联用模板。
动作：不再新增完整方法，改为“组合工作底稿”条目。

## 四、主卡/子卡判定标准

主卡应满足：
1. 有独立适用边界。
2. 有完整实施路径（>=6步）。
3. 有独立交付模板。

子卡应满足：
1. 仅解决主卡中的某一关键步骤。
2. 单独使用价值有限，需要依附主卡。
3. 模板规模应小于主卡。

## 五、去重阈值

新增内容若满足以下任一条件，不单独成新方法：
1. 章节结构完全一致，仅换术语。
2. 操作步骤 > 60% 相同。
3. 输出模板字段 > 70% 相同。

此时应：
1. 合并到现有方法“扩展场景”。
2. 或改为“子卡/附录模板”。

## 六、命名与编号规范

1. 新增方法命名格式：`方法N：中文名（定位短语）`。
2. 子卡命名格式：`方法X：...`下的`子卡A/B`。
3. 不重排历史编号；只顺延新编号。
4. 如触发合并，保留旧编号的“重定向说明”。

## 七、新增方法最小交付结构

每个新增条目至少包含：
1. 来源要点（来自PPT/资料）。
2. 方法原理与适用边界。
3. 专家级操作步骤。
4. 常见误用与纠偏。
5. 交付模板（专家版）。
6. 与其他方法组合策略。

## 八、新增后校验清单

1. 是否已挂到唯一主模块？
2. 是否在主`SKILL.md`中可被路由到？
3. 是否新增了不必要重复模板？
4. 是否破坏了已有方法调用顺序？
5. 是否在对应模块“3分钟选型表”中可被选中？
