# Visual Delivery Mode

Use this mode when the outreach needs to feel like a mini-package rather than a plain note.

## Preferred delivery stack

1. Short HTML email body
2. Attached one-page brief as PDF
3. Approval preview before send

Fallback:
- HTML brief attachment when PDF rendering is unavailable

## What creates visual punch

- short intro paragraph
- 3-item bullet block
- highlighted top fit
- clean one-page brief with section cards
- enough whitespace to scan on mobile

## Avoid

- heavy branding
- image-heavy layouts
- salesy gradients/buttons
- long walls of copy

## Approval preview checklist

Before sending, show or summarize:
- subject line
- plain-text fallback body
- HTML body preview
- attached brief filename
- recipient

## Best default sequence

- generate structured JSON
- render markdown report
- render HTML brief from the same JSON
- render plain-text email
- render HTML email
- show preview
- wait for explicit approval
