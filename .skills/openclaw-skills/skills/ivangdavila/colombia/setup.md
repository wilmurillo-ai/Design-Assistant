# Setup — Colombia Travel Guide

## First-Time Setup

When Colombia travel appears for the first time:

- Create `~/colombia/` if it does not exist yet.
- Create `~/colombia/memory.md` from `memory-template.md`.
- Gather the trip backbone naturally in conversation instead of running a questionnaire.

Useful context to capture early:

- Passport nationality and whether entry is already solved or still uncertain
- Travel window, flexibility, and preferred arrival city
- Region bias: cities, Caribbean beaches, Coffee Region, Andes, Pacific, Amazon, islands
- Climate tolerance: altitude, heat, humidity, rain, boat comfort
- Style: food, culture, nightlife, nature, road trip, family, remote-work rhythm
- Constraints: budget, mobility, Spanish comfort, yellow-fever-sensitive areas, driving comfort

Save only the facts that improve the next answer.

## Returning Users

If `~/colombia/memory.md` already exists:

- Read it first and reuse stable preferences.
- Ask only what changed: dates, region scope, budget, flight vs bus tolerance, or health constraints.
- Update the memory with deltas instead of rewriting everything.

## Quick Start Responses

**"I want Medellin and Cartagena."**
Use `medellin-and-antioquia.md`, `cartagena-and-caribbean.md`, and `transport-domestic.md`.
Check pace, weather tolerance, and whether the user wants beach time or city depth.

**"I want coffee towns and nature."**
Use `coffee-region.md`, `national-parks.md`, and `road-trips-and-driving.md`.
Check rainy-season fit, altitude comfort, and whether the user accepts a car.

**"I need a full Colombia itinerary."**
Use `regions.md`, `itineraries.md`, `budget-and-costs.md`, and `weather-and-seasonality.md`.
Start by narrowing the trip to two or three strong bases.

## Important Notes

- Entry, stay length, and health requirements can block otherwise strong plans.
- Colombia rewards focused regional depth more than constant moving.
- Transfer time is shaped by mountains, weather, and airport friction, not just distance.
- Always include a backup plan for rain, landslides, or operational disruptions.
