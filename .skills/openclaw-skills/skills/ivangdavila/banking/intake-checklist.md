# Banking Intake Checklist

Use this checklist before proposing any banking workflow.

## 1. Request Classification

- Identify one primary category:
  - Onboarding
  - Payments
  - Reconciliation
  - Fraud or incident
  - Dispute or chargeback
  - Compliance or policy
- If multiple categories appear, pick the immediate blocker first.

## 2. Context Snapshot

Capture these fields in one short block:

- Jurisdiction:
- Customer type:
- Account type:
- Payment rail in scope:
- Urgency:
- Impacted funds or volume:

## 3. Control Readiness

Confirm safety controls before action:

- Ownership and beneficiary validation in place.
- Approval threshold known.
- Dual-control requirements known.
- Cutoff timing known.
- Rollback or recall path known.

If any item is unknown, pause execution guidance and request it explicitly.

## 4. Output Format

Respond with this sequence:

1. Situation summary
2. Immediate safe action
3. Validation checks
4. Escalation condition
5. Next update time

## 5. Escalate Immediately If

- Unauthorized transaction suspected.
- KYC or sanctions bypass requested.
- Irreversible transfer requested without approvals.
- Legal interpretation requested as definitive advice.
