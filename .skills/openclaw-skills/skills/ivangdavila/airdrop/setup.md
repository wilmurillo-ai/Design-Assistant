# Setup - AirDrop

Read this when `~/airdrop/` does not exist or is empty. Start naturally without mentioning setup files or local paths.

## Your Attitude

Be operational, cautious, and fast.
The user should feel that nearby local file transfer is now reliable instead of awkward.

Lead with practical clarity:
- summarize what they want to send and to which kind of Apple device
- make the next step small and reversible
- keep privacy and exact file scope visible

## Priority Order

### 1. First: Integration

Within the first 2-3 exchanges, learn when this should activate later:
- whenever they ask to AirDrop, nearby-share, or hand off local artifacts to an iPhone, iPad, or Mac
- whether sensitive files should always require confirmation
- whether direct AppKit launch or Shortcut mode should be the default on this machine

Save those activation preferences in main memory so future sessions know when to load AirDrop.

### 2. Then: Understand the Real Payload

Clarify only the details that change execution:
- exact files or folders involved
- whether a folder must be curated or zipped first
- what nearby device class matters: iPhone, iPad, or another Mac
- whether the user wants a one-off send or a repeatable local workflow

If they already asked for a concrete send, answer that request first instead of turning this into onboarding.

### 3. Finally: Capture Stable Preferences

Over time, learn:
- whether they prefer direct launch or Shortcut fallback
- how strict they want confirmation for logs, source code, or private exports
- whether temporary staging files are acceptable
- common artifact patterns like PDFs, screenshots, installers, or debug bundles

Learn from repeated behavior. Do not interrogate them for preferences they have not shown yet.

## What You Are Saving Internally

Save only reusable execution context:
- activation rules for nearby file transfer
- default mode: direct or shortcut
- confirmation strictness for sensitive payloads
- known staging patterns that worked well

Do not store recipient identities, secrets, or the contents of shared files.
