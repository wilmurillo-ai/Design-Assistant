# Search and Knowledge Workflows

Use this file for Baidu Search, AI Search, Baike, and Wenku tasks.

## Surface Roles

| Surface | Best use | Do not use it as |
|---------|----------|------------------|
| Baidu Search | Discovery, current pages, and official-page finding | Final proof by ranking order alone |
| Baike | Fast encyclopedia-style orientation | Sole evidence for high-stakes claims |
| Wenku | Long-form references, document leads, and terminology clues | Verified truth without source tracing |
| Official docs or product pages | Final verification | Discovery engine for broad exploration |

## Minimum Search Loop

1. Define the job: discovery, fact check, official-page lookup, or terminology mapping.
2. Write one primary query and one contradiction query.
3. Classify the result type: official, encyclopedia, document mirror, forum, or commercial page.
4. Verify the final claim with the strongest available source tier.

## Query Design Rules

- Keep proper nouns in their original language when that is the source language.
- Add product-family or endpoint hints when "Baidu" is too broad.
- Prefer narrower queries over a pile of operators.
- When a result is translation heavy, search again with the original Chinese term if available.

## Trust Shortcuts

- Search is for discovery.
- Baike is for orientation.
- Wenku is for leads and terminology.
- Official docs are for final claims.

## Failure Modes

- Treating a Baike summary as final product behavior
- Citing Wenku documents without checking provenance
- Assuming the top Baidu result is the most authoritative source
- Ignoring query-language drift between English and Chinese terms
