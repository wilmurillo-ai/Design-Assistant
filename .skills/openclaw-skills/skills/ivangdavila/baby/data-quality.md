# Data Quality - Baby

Baby tracking only helps if the log can answer real questions later.

## Minimum Rule

For any entry that might matter later, capture:
- timestamp
- event type
- amount, duration, or severity when relevant
- one short context note if something changed from usual

## Make Baselines Visible

Always clarify whether the note means:
- normal for this baby
- slightly off baseline
- clearly unusual

This matters more than generic severity words.

## Use One Unit System

Pick one volume and temperature unit system unless the user already mixes them on purpose.
If units change, normalize before summarizing.

## Separate Facts from Interpretation

Good:
- "Rectal temp 38.3 C at 21:10"
- "Fed 40 mL less than usual over last two bottles"

Bad:
- "Probably getting sick"
- "Not eating enough"

Observed facts can support decisions. Guesses should stay labeled as guesses.

## Preserve Continuity Across Caregivers

When more than one adult is involved, every handoff should answer:
- what happened last
- what is due next
- what is different from baseline
- what must not be missed

If a note does not help the next caregiver act, it probably does not belong in the handoff.

## Compress Noise

For stable days, roll up repeated normal events into a short summary instead of logging every tiny detail.
Return to detailed logging only when:
- symptoms appear
- intake or output changes
- medications start
- pediatric review is coming

## Missing Data Policy

If an entry is incomplete but still useful:
- store the confirmed facts
- mark the missing field clearly
- ask for the missing detail only if it changes action

Do not invent timings, amounts, or temperatures.

## Summary Quality Check

Before giving a summary, verify that it answers:
- what changed
- how long it has been happening
- what was tried
- whether the pattern is getting better, worse, or unclear

If those four points are missing, the summary is not ready.
