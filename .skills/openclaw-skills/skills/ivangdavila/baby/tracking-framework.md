# Tracking Framework - Baby

Use modular tracking instead of all-or-nothing logging.
Start with the smallest set that prevents confusion, then expand only when useful.

## Core Continuity Block

Run this by default:
- last feed
- last sleep
- last diaper
- active concern or medication
- what is due next

This is the minimum viable handoff.

## Optional Modules

### Feeding
- breastfeeding side and duration
- bottle amount and type
- pumping output
- spit-up, reflux, or feeding refusal

### Sleep
- sleep start and end
- wakes and settling notes
- location if that affects the pattern

### Diapers and Output
- wet diapers count
- stool timing and appearance
- output changes worth follow-up

### Symptoms and Recovery
- temperature and method
- congestion, cough, vomiting, rash, pain cues
- care actions tried and response

### Medications
- medication name
- dose and units
- exact time given
- next allowed time if already known

### Growth and Appointments
- dated measurements with units
- vaccine or visit notes
- pediatric questions

### Solids and Reactions
- food introduced
- amount or exposure
- reaction timing and severity

### Development and Routine Notes
- new skills or regressions
- soothing methods that worked
- schedule experiments and results

## Simplified Mode

When caregivers are overwhelmed, reduce to:
- last feed
- last sleep
- last diaper
- current concern
- next important task

## Escalation Mode

When symptoms or alerts are active, prioritize:
- symptom timeline
- fluids and output
- medications and response
- red or amber triggers from `triage-rules.md`

Routine optimization can wait until the alert resolves.
