# RALLY Brief

Use this before generating candidate names. A weak brief produces decorative options that fail in the real environment.

RALLY stands for:
- **Role**: what the thing is
- **Audience**: who must understand, say, search, or implement it
- **Limits**: hard constraints
- **Lexicon**: words, categories, metaphors, or anti-words
- **Yardstick**: how the winner will be judged

## Minimal Brief

```markdown
Asset:
Lane:
Current state:
Audience:
Primary job:
Hard limits:
System context:
Words to echo:
Words to avoid:
Winner looks like:
```

## What To Lock Before Ideation

### Role
- What is being named: company, product, feature, API path, file, taxonomy bucket, initiative
- Whether the name is public, internal, or transitional

### Audience
- Who sees it first
- Whether they are technical, non-technical, internal, external, global, or niche

### Limits
- Character count
- Existing namespace rules
- Localization or pronunciation constraints
- Backward compatibility or migration risk

### Lexicon
- Category words that help orientation
- Metaphors that fit the product or system
- Words that are overused, misleading, or off-brand

### Yardstick
- Fast comprehension
- Strong recall
- Lower collision risk
- Better fit with sibling names
- Better long-term brand stretch

## Brief Modes By Lane

| Lane | Brief emphasis |
|------|----------------|
| Product or brand | distinction, memorability, stretch, connotation |
| Feature or workflow | comprehension, UI length, action clarity |
| API or schema | predictability, symmetry, tense, nouns vs verbs |
| Package or repo | exactness, searchability, maintenance clarity |
| Rename | migration risk, alias plan, stakeholder confusion |

## Failure Signals

Pause and repair the brief if:
- the named object keeps changing during discussion
- the audience is described as "everyone"
- no one can say what success looks like
- the user wants a brand name and a UI label from the same list
- the brief depends on legal or domain availability that has not been checked
