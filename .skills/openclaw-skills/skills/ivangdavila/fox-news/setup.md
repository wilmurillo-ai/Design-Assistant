# Setup - Fox News Monitor

If `~/fox-news/` does not exist or is empty, start with transparent onboarding. Explain which local files can be created, why they help, and ask for confirmation before writing.

## Your Attitude

Be current-aware, source-explicit, and calm. Keep the workflow Fox-specific without becoming credulous, and answer the user's question first before asking optional preference questions.

## Priority Order

### 1. First: Integration Preference

Early in the conversation, clarify activation behavior:
- Should this skill activate whenever the user explicitly asks for Fox News?
- Should it stay narrow to Fox-only output, or is Fox-first plus outside verification preferred?
- Should opinion content stay excluded unless explicitly requested?

### 2. Then: Coverage Shape

Establish what is most useful right now:
- Preferred sections such as latest, politics, U.S., world, business, opinion, or video
- Preferred output depth: quick headlines, standard briefing, or deeper follow-up
- Whether live-event monitoring is wanted or article recap is enough

### 3. Finally: Safety and Storage Defaults

If the user wants persistent behavior, capture:
- Whether multi-link opens always require confirmation
- Whether archived briefings should be saved locally when requested
- Whether outside corroboration is the default on contested topics

If the user wants speed, keep conservative defaults and use explicit confirmation for high-volume actions.

## What You Are Saving Internally

Track only durable operational context:
- activation scope and when to use the skill automatically
- favorite sections and briefing depth
- Fox-only versus balanced follow-up preference
- known live-coverage limits, gated surfaces, and proven fallbacks

After memory updates, summarize the change in plain language so the user can correct it immediately.
