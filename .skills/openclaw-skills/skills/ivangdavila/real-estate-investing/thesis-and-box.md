# Thesis and Buy Box

Use this before serious deal review. Most bad acquisitions happen because the user skips strategy definition and starts reacting to listings.

## Lock the thesis first

Clarify these in plain language:
- target strategy: buy-and-hold, value-add, BRRRR, flip, house hack, short-term rental
- hold period: quick turn, 1-3 years, 5+ years, indefinite
- main objective: monthly cash flow, equity growth, tax benefits, learning, or optionality
- capital posture: preserve cash, use leverage aggressively, or stay moderate
- workload tolerance: light, moderate, heavy rehab, remote management only

If the user says "anything that makes money," slow down and force trade-offs.

## Define the buy box

Write a small set of hard filters:
- market or submarket
- property type and unit count
- price band
- minimum yield or return target
- rehab tolerance
- management model
- deal killers

Example:

```text
Strategy: small multifamily buy-and-hold
Markets: Midwest metros with stable rent growth
Price band: 150k-450k
Asset type: duplex to fourplex
Minimum: DSCR 1.25 on realistic rents
Rehab: cosmetic only
No-go: heavy foundation, flood friction, tenant concentration risk
```

## Guardrails that protect discipline

Set boundaries before getting attached:
- max all-in cash needed
- minimum reserve after closing
- minimum break-even occupancy
- maximum rehab timeline
- maximum exposure to one neighborhood or operator

## Questions that sharpen the frame

- What kind of pain is acceptable: slower growth, lower cash flow, heavier work, or more debt sensitivity?
- What would a "good enough" first win look like?
- Which risks are tolerable and which are immediate deal killers?
- Is the user buying a business, a stable coupon, or a value extraction project?

## Output standard

Any recommendation should reference the thesis and buy box explicitly:
- thesis fit: strong, medium, weak
- which guardrails are safe
- which guardrails are already broken
- whether the deal is exceptional enough to justify an exception
