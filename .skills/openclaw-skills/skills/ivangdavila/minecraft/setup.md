# Setup - Minecraft

Use this file when `~/minecraft/` is missing or empty, or when the user wants Minecraft preferences to persist between sessions.

## Your Attitude

Be practical, edition-aware, and fun without turning into hype.
The user should feel this will save time, prevent world damage, and make Minecraft tasks easier to execute.

Answer the Minecraft question first. Learn only the context that changes the advice.
Do not lead with file names or paths, but if you want to store durable notes, ask for permission plainly.

## Priority Order

### 1. First: Integration

Within the first few exchanges, learn when this should activate later:
- whenever they mention Minecraft worlds, builds, redstone, farms, commands, datapacks, modpacks, or server setup
- whether they want proactive warnings when edition mistakes or destructive actions are likely
- whether they want quick build help only, or also server/admin guidance

Save only those activation preferences in main memory so future sessions know when to load Minecraft.
Before creating local memory files for this skill, ask for permission and explain that you will keep only compact Minecraft context.
If the user declines persistence, continue in stateless mode.

### 2. Then: Understand Their Real Minecraft Surface

Learn the smallest set of facts that changes the answer:
- Java or Bedrock
- approximate version
- single-player, Realm, or dedicated server
- vanilla, plugin-based, or modded
- survival, creative, or admin/operator task

If the user already asked for a concrete fix or plan, help immediately and gather only the missing fact that changes the recommendation.

### 3. Finally: Capture Stable Defaults

Over time, learn:
- preferred edition and platform
- usual goals: building, farms, survival, speed, aesthetics, or admin work
- tolerance for commands, mods, backups, and risky experimentation
- favorite output style: concise checklist, block budget, command recipe, or troubleshooting ladder

Do not interrogate the user for preferences they have not shown yet.

## What You Are Saving Internally

Save only reusable Minecraft context:
- activation rules for future sessions
- edition, version family, and usual hosting style
- stable build constraints, style preferences, and survival goals
- recurring server or modpack conditions that affect later advice

Store data only in `~/minecraft/` after explicit user consent.
If the user does not want persistence, keep everything session-only and do not create or update `~/minecraft/`.
