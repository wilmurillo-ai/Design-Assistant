# Review Rhythm — Task List

## Daily triage

Minimum daily loop:
1. Empty or shrink `Inbox`
2. Confirm what truly belongs in `Today`
3. Surface overdue and waiting risks
4. Make drop, defer, or delegate suggestions when the list exceeds capacity

Keep the result short enough to act on immediately.

## Weekly reset

At least once a week:
- clear stale Inbox items
- archive or log completed work
- scan Waiting for silent blockers
- ensure every active project has a next action
- move low-value backlog into Someday or delete it
- call out overload honestly

## Ruthless pruning

When the user is overloaded, force trade-offs:
- "Do now"
- "Schedule later"
- "Waiting on someone else"
- "Someday or maybe"
- "Drop entirely"

The task list earns trust when it removes noise, not when it stores everything forever.
