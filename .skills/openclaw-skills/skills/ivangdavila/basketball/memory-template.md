## Supporting Files

For `~/basketball/possession-map.md`, keep:
- game context and date
- preview or review mode
- possession themes
- key matchups, triggers, and next actions

For `~/basketball/roster-notes.md`, keep:
- role map
- lineups that work or fail
- recruitment needs
- player-fit summaries

For `~/basketball/practice-log.md`, keep:
- weekly rhythm
- drill notes that actually transfer to games
- recurring constraints
- follow-up ideas for next cycles

## Principles

- Persistent memory is opt-in. If the user declines, keep the session ephemeral.
- Store context that improves future basketball work, not every observation.
- Prefer role-based summaries over raw transcript dumps.
- Update `last` whenever the skill is used.
- Never store credentials, private medical details, or copyrighted material when a short summary is enough.
