# Timeline and Run of Show

Use this when the wedding date exists and the user needs sequence, deadlines, or day-of clarity.

## Backward Plan

Start from the wedding date and move backward through the major locks:

| Timing | Focus |
|--------|-------|
| 12+ months | venue, rough budget, event size, priority list |
| 9-12 months | core vendors, guest-count range, dress or attire planning |
| 6-9 months | invitations plan, travel or hotel blocks, rentals, ceremony details |
| 3-6 months | tastings, fittings, final design choices, timeline draft |
| 1-3 months | RSVPs, seating, vendor confirmations, payment review |
| 2-4 weeks | final counts, emergency kit, run-of-show, contact sheet |
| week of | confirmations, packing, handoffs, weather or contingency checks |

Adjust the cadence for short engagements or elopements, but do not skip dependency logic.

## Milestone Rules

- Every milestone needs an owner, date, and consequence if late.
- If a deadline slips, update the downstream items immediately instead of pretending the schedule is intact.
- As the event approaches, prefer certainty over optimization.
