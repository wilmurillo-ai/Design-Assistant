# Listing Requirements — Play Store

## Store Listing Assets

### Required Graphics
| Asset | Dimensions | Format | Notes |
|-------|------------|--------|-------|
| App icon | 512x512 px | PNG (32-bit) | No alpha transparency edge |
| Feature graphic | 1024x500 px | PNG/JPEG | Shown in promotions, no text <24pt |
| Screenshots | Varies | PNG/JPEG | 2-8 per device type |
| TV banner | 1280x720 px | PNG/JPEG | Required for TV apps |

### Screenshot Best Practices

**First Two Screenshots:**
- Visible in search results before tap
- Show primary value proposition
- Include text overlays explaining benefit
- Device frame optional but professional

**Remaining Screenshots:**
- Feature progression or user journey
- Social proof elements where relevant
- Settings/customization options
- Before/after comparisons work well

**Device Frames:**
- Consistent style across all screenshots
- Modern device (current Pixel recommended)
- Avoid outdated bezels

- [ ] Short description localized
- [ ] Full description localized

- [ ] Keywords researched per market

## Description Structure

### Full Description Template
```
[Hook — 1-2 sentences, benefit-focused]

[Primary features — bullet points or short paragraphs]

• Feature 1 — benefit
• Feature 2 — benefit
• Feature 3 — benefit

[Social proof — awards, press, user count]

[Call to action — download now]

[Support info — email, website]
```

### Formatting That Works
- Emojis at bullet starts ✓
- Short paragraphs (2-3 sentences)
- White space between sections
- Key phrases in CAPS sparingly

### Avoid
- Walls of text
- ALL CAPS paragraphs
- Excessive emojis
- Unsubstantiated claims
- Competitor mentions
