# Landed Cost - DHgate

Never evaluate DHgate on listing price alone.

## Core Formula

```text
True Unit Cost =
  (item subtotal
  + shipping
  + duties or VAT
  + payment fees
  + inspection or defect reserve
  + domestic fulfillment costs if reselling)
  / usable units
```

If the order is a bundle or lot, divide by the number of units the user can actually sell or keep.

## Personal Buy Checklist

Use this sequence:

1. item price after coupon
2. shipping cost for the chosen method
3. expected import tax or VAT in the destination country
4. risk reserve if quality is uncertain
5. local substitute price for comparison

If the imported total is only slightly cheaper than a local option, local convenience often wins.

## Resale or Dropship Checklist

Add these layers:

1. all personal-buy inputs above
2. platform fees where the user will sell
3. packaging or relabeling cost
4. domestic shipping to the final buyer
5. refund and return reserve
6. slow-delivery cost in conversion or customer support time

## Quick Margin Guardrails

| Situation | Minimum standard |
|-----------|------------------|
| Unknown seller, first test | margin is secondary; keep cash risk tiny |
| Repeatable resale | enough margin to survive one bad batch or return wave |
| Slow-shipping category | margin must pay for delay and support friction |
| Fragile or sizing-sensitive item | reserve must be higher than normal |

If one defect-heavy batch could erase several good orders, the deal is not good enough.

## Coupon and MOQ Notes

- Treat coupons as a bonus, not proof of value
- Recompute unit cost when quantity thresholds change
- A lower unit price can still lose once shipping jumps at the next weight tier
- If MOQ forces dead stock, unsold units belong in the cost model
