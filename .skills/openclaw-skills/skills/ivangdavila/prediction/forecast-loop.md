# BRACE Forecast Loop

Use this file when the forecast matters enough to deserve a repeatable process.

BRACE keeps the agent from drifting into pure storytelling:

## 1. Base Rate

- Find the nearest reference class before reading the current case as unique.
- Prefer observed frequencies over vibes, anecdotes, or one memorable example.
- If the base rate is weak, say so and keep confidence lower.

## 2. Resolution Rule

- Define exactly what counts as yes, no, or unresolved.
- Name the threshold, date, and evidence source that will settle the question.
- If the resolution rule is fuzzy, rewrite the question first.

## 3. Arguments Both Ways

- List the strongest three reasons for yes and the strongest three reasons for no.
- Separate leading indicators from lagging signals.
- Demote arguments that are only vivid, recent, or emotionally satisfying.

## 4. Confidence Assignment

- Start from the base rate, then adjust in small defended steps.
- Convert the final view into a number, range, or explicit scenario split.
- Explain what evidence would move the forecast materially.

## 5. Evaluation Plan

- Log the forecast in `forecast-log.md`.
- Set a review date or trigger, not endless reactive updates.
- When the question resolves, score it in `scorecard.md` and update any reusable lesson.

## Minimum Output Contract

Every BRACE forecast should include:
- question
- resolution rule
- probability or scenarios
- main supporting evidence
- main contrary evidence
- update trigger
- next review date
