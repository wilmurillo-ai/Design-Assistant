# Growth and Marketing — Whop

Use this file when the user wants more buyers, cleaner attribution, or better distribution on Whop.

## Growth Order

Run growth in this order:

1. Tracking links
2. Promo logic
3. Affiliate design
4. External tracking integrations
5. Ads

That sequence prevents scaling blind.

## Tracking Links

Whop's tracking links are the simplest attribution tool and should be configured before serious promotion.

Use different links for:

- Email
- X or Twitter
- Instagram bio
- YouTube descriptions
- Communities
- Paid traffic campaigns
- Affiliate programs when you want a clean campaign view

The docs position tracking links as the way to see where users are coming from and which marketing efforts are working.

## Promo Codes

Whop supports percentage discounts and fixed-amount discounts at checkout.

Good promo-code uses:

- Closing a limited-time campaign
- Rewarding a warm audience segment
- Testing price sensitivity by channel
- Giving partners a controlled offer without rewriting the main pricing

Bad promo-code uses:

- Permanent discounting because the main offer does not convert
- Mixing multiple overlapping discounts that destroy attribution
- Using codes as a substitute for fixing the product or checkout experience

## Affiliates

Whop supports three distinct partner models. Pick the one that matches the relationship.

### Global affiliates

The docs describe global affiliates as the default marketplace-wide affiliate system. Listed products become promotable across Whop's network, with a 30% default commission and one-click referral links for affiliates.

Use this when the user wants broad distribution fast.

### Custom affiliates

Custom affiliates are for direct partner deals. The docs say they can be one-time or recurring and flat-fee or percentage-based, with personalized referral links and automatic conversion tracking.

Use this for:

- Influencers
- Closers
- Community owners
- Strategic partners

### Revenue share

Revenue share is for transactional partnerships. The docs say revenue splits are calculated after Whop fees and paid automatically into the partner's Whop balance.

Use this when the partner is part of the economics, not just the acquisition.

## Tracking Integrations

Whop supports external tracking integrations so the user can see more than dashboard-native traffic data.

Set these up before scale if the user is:

- Running paid acquisition
- Comparing multiple creators or channels
- Feeding marketing data into another analytics stack

## Whop Ads

Whop has a native Whop Ads product. The docs describe Meta ads with agency-grade accounts, done-for-you creatives, buyer audiences, and a unified ads-plus-sales dashboard.

Important constraint:

- The docs currently say Whop Ads is in beta and limited to top Whop merchants

So the right move is:

- Use it if the user already has access
- Treat it as an option, not the default assumption
- Keep a fallback paid-media plan outside this feature if access is unavailable

## Channel Review Checklist

- Does every channel have its own tracking path?
- Are promo codes tied to a deliberate campaign?
- Is the affiliate model matched to the partner type?
- Are external analytics connected before spend scales?
- Is paid traffic being added only after the offer already converts?
