# Trade-off Matrix

Use this file when the user must choose between options and the room is arguing from instinct.
The goal is not fake precision. The goal is visible reasoning.

## Standard Evaluation Axes

Score or describe each option across:
- performance
- cost now
- cost later
- schedule and lead time
- safety and compliance
- reliability and uptime
- maintainability
- reversibility
- training or operator burden

If an axis does not matter, say why it is excluded.

| A | | | | |

Use short, evidence-based statements instead of decorative scoring when the data is weak.

## Decision Rule

State the decision logic clearly:
- choose the safest reversible option
- choose the lowest lifecycle cost above required performance
- choose the fastest valid path to evidence
- choose the option with the best downside protection

Without a rule, teams cherry-pick the metric they already prefer.

## Minimum Output

Return:
1. options considered
2. decision rule
3. chosen option
4. rejected options and why
5. what evidence would reverse the decision
