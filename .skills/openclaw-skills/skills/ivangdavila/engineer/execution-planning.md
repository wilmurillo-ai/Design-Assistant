# Execution Planning

Use this file when the user already knows the direction and now needs a safe sequence.
A good engineering plan makes dependencies and irreversible points visible before work starts.

## Build the Sequence

Break the work into:
- prerequisites
- preparation work
- execution steps
- hold points
- restart criteria
- rollback path

Do not hide dependency risk inside a single bullet that says "implement."

## Critical Path Questions

- What must happen first for anything else to work?
- Which missing part, approval, tool, or condition can stop the plan cold?
- Which step is irreversible or expensive to undo?
- What evidence is required before moving to the next step?

## Readiness Check

Before launch, confirm:
- people available
- materials, tools, and spares ready
- instructions clear
- safety controls active
- fallback path prepared
- monitoring in place

## Fast Changeovers and Turnarounds

When time pressure is high:
- reduce surprises, not caution
- stage tools and materials early
- define stop conditions before the window opens
- protect restart quality, not just completion time

## Minimum Output

Return:
1. ordered plan
2. critical path
3. hold points and approvals
4. rollback or recovery path
5. restart criteria
