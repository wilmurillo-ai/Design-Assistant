# Deliverables

Every engagement should end with concrete artifacts the user can reuse.

## 1. Target Brief

```markdown
# Target Brief
- Target:
- Goal:
- Boundary:
- Available evidence:
- Fastest likely path:
```

## 2. Interface Map

Use the structure from `interface-map.md` to describe external behavior first.

## 3. Hypothesis Ledger

```markdown
# Hypothesis Ledger
| Hypothesis | Status | Evidence level | Cheapest next test |
|-----------|--------|----------------|--------------------|
|           | open   | 0              |                    |
```

Statuses:
- `open`
- `supported`
- `rejected`
- `parked`

## 4. Reproduction Note

```markdown
# Reproduction Note
- Trigger:
- Minimal sample:
- Steps:
- Expected result:
- Observed result:
- Confidence:
```

## 5. Remaining Unknowns

```markdown
# Remaining Unknowns
- What is still unclear
- Why it matters
- What evidence would resolve it
```

## Compression Rule

If the user wants a short answer, compress these artifacts into:
- mechanism summary
- evidence summary
- next action

Do not drop the unknowns ledger just because the answer is short.
