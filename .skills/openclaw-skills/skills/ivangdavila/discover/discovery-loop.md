# Discovery Loop - Discover

Use this loop when the user wants the agent to keep finding new things, not just summarize what is already obvious.

## Loop

1. Lock the topic and why it matters.
2. Check `~/discover/watchlist.md` and the latest matching findings.
3. Pick one lens that has not been overused.
4. Search narrowly for signal, not volume.
5. Run `novelty-test.md`.
6. Log the delta only if it is genuinely new.
7. Leave one next lens or next question for later.

## Useful Lenses

- direct: the obvious search path
- operator: how people actually execute this in the wild
- contrarian: what could break the thesis
- geographic: how the answer changes by country or jurisdiction
- regulatory: legal or policy constraints
- stakeholder: what changes for founder, employee, investor, family, or customer
- practical: what the user could do next week, not just understand abstractly

## Example

Topic: moving abroad for better tax efficiency

- direct -> country comparisons
- operator -> real relocation firms, tax advisors, or community playbooks
- contrarian -> exit tax, controlled foreign company risk, substance requirements
- geographic -> compare Andorra, UAE, Paraguay, or Spain regimes
- practical -> checklist, timing windows, documents, and first calls to make
