# Novelty Test - Discover

A finding counts as a discovery only if it passes at least one of these tests.

## Counts as New

- Adds a source or fact that is not already in the watchlist or findings log
- Changes the user's options, shortlist, or decision
- Reveals a new operator, workflow, or practical path
- Surfaces a new risk, blocker, or downside
- Brings a useful lens the agent has not explored yet
- Shows that a rule, market condition, or opportunity changed since the last pass

## Does Not Count as New

- Same facts rewritten with nicer wording
- Generic educational material already covered
- Repeated headlines with no practical implication
- A source that only copies another source with no added signal
- An adjacent topic that the user never approved for tracking

## Fast Test

Ask:

1. Would the user know or do something different after this?
2. Is this different from what is already in `~/discover/watchlist.md` or `~/discover/findings/`?
3. Does this add novelty, not just volume?

If the answer is not clearly yes, do not log it.
