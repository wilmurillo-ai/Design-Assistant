# Memory Routing

Use the smallest file that preserves the lesson.

## `memory.md`

Store durable operating rules:
- when critique should trigger by default
- how much interruption the user tolerates
- which domains need lighter or deeper critique

Good fit:
- "Before decomposing UI work, run standard critique because revisions are common."
- "Keep critique light for routine content edits unless the user flags quality issues."

## `checkpoints.md`

Store reusable breakpoint prompts tied to a workflow phase.

Good fit:
- "Architecture draft -> ask whether any assumption will poison all later tasks."
- "Before delegate packet -> ask what ambiguity will create rework for the receiver."

## `incidents.md`

Store late catches only when they teach a better insertion point.

Good fit:
- a user correction that exposed a missing mid-task review
- a failure that should have been caught before branching

Bad fit:
- every minor typo
- generic dissatisfaction with no process lesson
- long emotional summaries

## `archive/`

Move resolved incidents and obsolete checkpoints here so the live files stay small and readable.
