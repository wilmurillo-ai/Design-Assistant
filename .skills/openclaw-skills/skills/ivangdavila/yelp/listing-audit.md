# Yelp Listing Audit

Use this file for owner, operator, or agency-side Yelp work.

## Audit goals

A useful audit answers:
- can customers find the right listing quickly
- does the page set the right expectation
- what visible issues could reduce trust or conversion

## Audit checklist

Review these elements in order:
1. business identity: name, category, address, phone
2. hours and closure signals
3. price level and service attributes
4. photos, menu, services, or amenities
5. review pattern and unanswered concerns

## Priority buckets

Classify findings as:
- critical: wrong identity, wrong location, clearly wrong hours
- important: weak category fit, missing proof, repeated complaint cluster
- cleanup: thin descriptions, stale photos, weak differentiation

## Output format

Return findings like this:

| Priority | Issue | Why it matters | Suggested fix |
|----------|-------|----------------|---------------|
| Critical | Hours appear wrong | users show up at the wrong time | verify and correct hours |
| Important | Complaint cluster on wait times | trust and conversion drag | prepare ops note and response draft |
| Cleanup | Weak photo coverage | lower confidence before visit | refresh visible proof assets |

## Owner-side boundaries

- Draft suggested responses only if the user is authorized.
- Keep drafts specific, factual, and non-defensive.
- Do not claim a change is live until the user confirms it was applied.

## Common audit mistakes

- Treating a listing audit as the same thing as consumer comparison
- Focusing on average rating while missing trust-breaking details
- Writing reply drafts before understanding the complaint pattern
