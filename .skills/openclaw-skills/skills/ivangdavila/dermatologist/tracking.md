# Tracking and Case Separation

## One Folder Per Concern

Create one case folder for each distinct problem:
- one changing mole
- one eczema flare pattern
- one acne course
- one sudden rash episode

Do not merge unrelated concerns just because they happen at the same time.

## Case ID Pattern

Prefer a stable ID that captures first date, body site, and concern:

```text
YYYYMMDD-body-site-short-label
```

Examples:
- `20260308-left-cheek-acne-flare`
- `20260308-right-calf-new-patch`
- `20260308-upper-back-mole-watch`

## When to Split a Case

Start a new case if:
- the body site is different and not obviously linked
- the morphology is different
- the trigger or treatment plan is different
- the old case resolved and a clearly new episode appears later

## Timeline Discipline

Only log events that matter clinically or operationally:
- start
- spread
- pain or itch change
- bleeding or crusting
- treatment start or stop
- clinician visit, biopsy, or result

Avoid full diary prose. Short dated facts are better.

## Evolution Review

When asked "is it better or worse?":
1. compare the latest timeline entries
2. compare only standardized photos
3. separate symptom trend from visual trend
4. state uncertainty if the evidence is mixed

## Export Rule

When preparing a visit, reduce the case to:
- onset
- trend
- treatments tried
- triggers or exposures
- current concern
- top questions
