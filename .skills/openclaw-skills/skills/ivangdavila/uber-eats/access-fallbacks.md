# Access Fallbacks - Uber Eats

Use this when the web session is blocked, blank, or shows `access denied`.

## 1. Stop Blind Interaction

- Do not keep clicking on a blocked page.
- Re-read the title and URL first.
- Capture one screenshot only if the user approved browser inspection.

## 2. Try a Safer Web Path

- Reload once.
- Try the market-specific locale route if the generic root is unstable.
- If the daily browser profile looks poisoned for this session, stop before making it worse.

## 3. Switch to App or Manual Handoff

- If the user has the Uber Eats app available, prepare the merchant, items, and checkout summary for app continuation.
- If browser automation is not trustworthy, move to browse-only planning or manual instructions.

## 4. Treat Support Pages as Recovery, Not Ordering

- Use help pages for cancellation, address, and issue guidance.
- Do not confuse support access with a valid checkout session.

## 5. Save the Durable Lesson

After the issue is understood, store only the reusable part:
- browser or locale that works
- browser or profile state that triggers blocking
- whether app handoff is more reliable for this user
