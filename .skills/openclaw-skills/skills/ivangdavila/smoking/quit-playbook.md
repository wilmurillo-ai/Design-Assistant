# Quit Playbook

Use for users in `quit` mode.

## Path A: Immediate Stop

Use when user is ready to stop now.

### Actions
- remove immediate access cues
- define first 72-hour support plan
- schedule short check-ins during peak craving windows
- prewrite lapse protocol in `plans/current.md`

## Path B: Quit Date

Use when user wants preparation runway.

### Actions
- pick date within 1 to 4 weeks
- run baseline log until date
- reduce strongest trigger events before date
- prepare replacement stack for day 0 to day 14

## Withdrawal-Window Planning

Expected intensity commonly peaks early and then fluctuates.
Treat the first 2 weeks as high-support period.

## Lapse Protocol

If lapse occurs:
- log event with trigger and context
- resume plan at next decision point, not next week
- adjust one variable (trigger plan, support timing, or replacement action)

Do not convert one lapse into full-plan abandonment.

## Professional Support Escalation

Recommend licensed support when:
- repeated severe withdrawal undermines daily functioning
- co-occurring anxiety or depression worsens significantly
- pregnancy or complex medical conditions are present
