# Setup — Self-Evolving

Read this when `~/self-evolving/` does not exist or is empty. Start naturally. If you need to create local notes for the first time, tell the user that this skill keeps concise local notes for repeat improvement. Do not dump file paths or implementation detail unless they ask.

## Your Attitude

Treat evolution as practical improvement, not as a science project. The user should feel that the agent is becoming more reliable and easier to trust, not more complex.

## Priority Order

### 1. First: Integration

Within the first 2-3 exchanges, learn when this skill should activate:
- Should it activate whenever the user revisits a repeated workflow, recurring mistake, or optimization topic?
- Should it step in proactively after visible friction, or only on direct request?
- Are there situations where experimentation should never happen?

### 2. Then: Understand Their Friction

Ask broad questions first:
- Which workflow keeps repeating?
- What counts as a better result: speed, quality, consistency, or less back-and-forth?
- What kind of mistake is most expensive for them?

### 3. Finally: Learn Their Tolerance

Only if they want depth, learn:
- How much change is acceptable between runs
- What evidence is enough to promote a new rule
- Which surfaces must stay stable

## What You're Saving Internally

- Activation cues and always/never cases in the user's main memory
- Stable goals, quality signals, and no-go zones in `memory.md`
- Tentative mutations, failed experiments, and next checks in `experiments.md`
- Retired ideas in `archive/` only when they are clearly obsolete

## Working Style

- Answer the current task first; do not turn the conversation into an onboarding form
- Before the first persistent write, briefly state that this skill will keep concise local notes for future reuse
- Reflect back what you understood after each answer so the user feels progress
- Keep early questions open and short
- Prefer observing repeated behavior over interrogating for preferences
- If the user does not want any setup depth, work with the minimum needed and keep learning through normal use
