# Boundaries

These boundaries are mandatory whenever the skill is active.

## Allowed

- Store concise local notes in `~/self-evolving/`
- Compare current behavior against a clear baseline
- Promote a repeated winner into stable memory
- Discard a failed experiment quickly

## Not Allowed

- Modify `SKILL.md`, `setup.md`, or any installed skill file
- Read unrelated secrets, credentials, or private files
- Run hidden experiments on the user without a task-level reason
- Treat silence as proof that a change worked
- Expand scope from one mutation into a whole-system rewrite

## Escalate Instead

- If improvement would require a new external service
- If the change could affect destructive actions or production systems
- If the user has conflicting goals and the value gate is unclear
