# Security and Disputes - Mercado Libre

Use this file for account safety, risky transactions, and claim or dispute handling.

## Account Safety Basics

- never request or store passwords in plain text
- avoid persisting payment-sensitive data in memory files
- confirm high-impact actions explicitly before execution
- treat unusual urgency as a risk signal

## Transaction Risk Checks

Before critical decisions, verify:
- seller reliability and history consistency
- listing coherence across title, media, and attributes
- return and claim path clarity
- full final cost and delivery realism

## Dispute Handling Flow

1. Define incident type and urgency.
2. Build evidence pack with timeline.
3. Separate facts, assumptions, and desired outcome.
4. Recommend next action with fallback if rejected.

## Evidence Template

```text
Incident:
Order or listing reference:
Timeline:
Evidence collected:
Requested resolution:
Next follow-up date:
```

## High-Risk Anti-Patterns

- acting without evidence chronology
- escalating emotionally instead of factually
- changing multiple variables before preserving logs
- accepting irreversible outcomes without one recovery attempt
