# Member Lifecycle - Skool

## Treat the Lifecycle as One Funnel

The important path is:
- visitor
- applicant
- approved member
- activated member
- retained member
- expanded buyer or advocate

Every automation, question, course unlock, and event should move one step of that funnel.

## Applicant to Approved

- Membership questions should qualify intent, not create homework.
- Decide where manual approval is worth the friction.
- If instant approval is enabled, make the first in-group orientation stronger because the filter moved downstream.
- When the source is off-platform checkout or CRM enrollment, choose between native invite email, CSV import, Zapier invite, or webhook invite instead of inventing a new admission path.

## Approved to Activated

- Activation means more than joining. It means first lesson, first comment, first event, or first completed action.
- Use AutoDM, welcome posts, and classroom structure to create one obvious next move.
- Use built-in variables and preview in AutoDM rather than hand-waving the first-touch message.
- Remove dead ends where members arrive but do not know where to start.

## Retention

- Members stay when the next win is visible.
- Calendar cadence, lesson sequence, and community prompts should point to the same weekly loop.
- Track churn signals such as silent new members, missed live events, or repeated access confusion.

## Expansion

- If the group upsells coaching, masterminds, higher tiers, or extra courses, define the trigger for that expansion clearly.
- Expansion should follow demonstrated progress, not surprise gating or vague scarcity.
- Use grant/revoke access or Zapier-based course unlocks only when the exact unlock path and rollback path are clear.

## Safety Lens

- invites, removals, and access grants are write operations
- pricing, trials, and level unlocks should be checked twice
- never assume the admin view matches the member view
