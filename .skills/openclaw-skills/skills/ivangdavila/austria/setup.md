# Setup — Austria Travel Guide

When Austria comes up for the first time, create a lightweight local memory and gather only the context that changes the route.

## First-Time Setup

Create the workspace:

```bash
mkdir -p ~/austria
```

Create `~/austria/memory.md` from `memory-template.md`.

## What To Learn Early

Start with the trip-shaping facts instead of generic preferences:

- Entry path already solved or still unknown
- Travel month and flexibility
- Cities only, lakes and mountains, skiing, Christmas markets, food and wine, or mixed route
- Rail-first comfort vs willingness to rent a car
- Budget band and hotel style
- Hiking level, snow comfort, driving confidence, and kid or mobility needs
- Whether user wants Austria only or Austria plus neighboring countries

## Returning Users

If `~/austria/memory.md` exists:

1. Read it silently.
2. Reuse known trip style, transport bias, and constraint history.
3. Ask only what changed: dates, region focus, budget, weather tolerance, or border plans.
4. Update the memory with deltas and unresolved decisions.

## Quick Start Responses

**"I want Vienna and Salzburg."**
Then use: `vienna.md`, `salzburg-and-salzburgerland.md`, `transport-domestic.md`, `accommodation.md`.

**"I want lakes and mountains in summer."**
Then use: `regions.md`, `alps-lakes-and-outdoors.md`, `weather-and-seasonality.md`, `carinthia-and-lakes.md`.

**"I want Austria for skiing."**
Then use: `winter-ski-and-christmas.md`, `innsbruck-and-tyrol.md`, `road-trips-and-driving.md`, `safety-and-emergencies.md`.

**"I only have a few days in Austria."**
Then use: `regions.md`, `itineraries.md`, `transport-domestic.md`, `budget-and-costs.md`.

## Important Notes

- Rail works extremely well for classic Austria routes, but alpine edge cases still change the answer.
- Weather and daylight matter more than map distance.
- Hallstatt, Christmas markets, ski weeks, and top lake towns need earlier planning than casual visitors expect.
- Austria often works best as fewer bases with better local depth.
