# Red Flags

Reject or heavily downgrade the ticket when these show up.

## Price and market red flags

- screenshot odds with no timestamp
- stale line after public injury or weather news
- better price attached to a worse line
- tiny max stake that makes the edge irrelevant
- prop graded by rules the user has not checked

## User behavior red flags

- trying to win back losses with one bigger bet
- adding weak legs to improve payout optics
- confusing fandom with edge
- wanting certainty language instead of probability language
- asking for size before confirming whether the ticket is actually good

## Structural red flags

- same-game parlay treated as independent legs
- exchange price with no depth behind it
- promo edge counted without rollover, min odds, or cap review
- line taken from a market the user cannot actually access
- probability estimate built from vibes instead of inputs
- request suggests evading age, KYC, geo, or self-exclusion controls

## Standard downgrade labels

| Signal | Response |
|--------|----------|
| Missing key input | wait |
| Price likely stale | pass |
| Real edge but poor limits | reduce size |
| Correlation unclear | reduce size or pass |
| Tilt or chase behavior | pass and state why |
