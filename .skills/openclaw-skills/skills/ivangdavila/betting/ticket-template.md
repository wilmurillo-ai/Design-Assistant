# Ticket Memo Template

Use this format for a clean betting answer:

```markdown
## Ticket
- Event:
- Market:
- Book:
- Price:
- Stake Plan:

## Pricing
- Implied probability:
- Fair probability estimate:
- Fair price:
- Edge estimate:

## Checks
- Same market confirmed:
- Settlement or void concerns:
- Limit or liquidity concerns:
- News or latency concerns:

## Decision
- Label: positive edge | reduce size | wait | pass
- Maximum analytical size:
- Kill conditions:
- Why:
```

## Decision standard

Good memos are short, numeric, and explicit about uncertainty.

Bad memos:
- hide missing inputs
- skip the price conversion
- talk only about who is better
- push action without a pass condition
