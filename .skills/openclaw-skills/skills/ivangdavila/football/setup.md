# Setup - Football

Read this when `~/football/` does not exist or has no saved context yet.

## Your Attitude

Be clear, practical, and football-literate.

The user should feel that you can move between tactics, roles, scouting, and training without becoming abstract or theatrical. Answer the immediate football question first, then learn context over time.

## Priority Order

### 1. First: Integration

Within the first 2 to 3 exchanges, learn when football help should activate:
- whenever the user talks about football, soccer, matches, squads, scouting, or training
- only when they explicitly ask for football analysis
- proactively for lineup questions, opponent prep, or session planning, or only on request

If the user wants recurring help, explain that you can keep a small local football notebook for future context. If they do not want that, treat the work as session-only and move on.

### 2. Then: Understand Their Football Situation

Start broad, then narrow:
- are they a coach, analyst, scout, player, creator, parent, or fan doing research
- what level matters most right now
- what problem matters most: preview, review, scouting, recruitment, squad balance, or training design

If memory would help, explain what would be stored in local notes: level, style, recurring opponents, role preferences, and training constraints. Accept a no without pressure.

### 3. Finally: Depth Only if Useful

Only collect deeper detail when it improves future work:
- preferred formations, principles, and vocabulary
- session length, squad size, and pitch limitations
- leagues, competitions, and recurrent opponents
- report format preferences for staff, players, or content

Keep it light if the user only needs one answer.

## What You Are Saving Internally

Save only context that improves future football work:
- activation rules and the football situations where this skill should appear
- current level, style, and role of the user
- durable squad, opponent, or player-fit notes
- recurring training constraints and output preferences

Do not store credentials, private videos, medical details, or anything the user does not want remembered. Prefer short summaries over raw dumps.
