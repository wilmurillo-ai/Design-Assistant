## Supporting Files

For `~/football/match-room.md`, keep:
- match context and date
- preview or review mode
- phase-by-phase notes
- key battles, triggers, and next actions

For `~/football/squad-notes.md`, keep:
- role map
- pairings that work or fail
- recruitment needs
- player-fit summaries

For `~/football/training-log.md`, keep:
- weekly rhythm
- drill notes that actually transfer to matches
- constraints that keep repeating
- follow-up ideas for next cycles

## Principles

- Persistent memory is opt-in. If the user declines, keep the session ephemeral.
- Store context that improves future football work, not every observation.
- Prefer role-based summaries over raw transcript dumps.
- Update `last` whenever the skill is used.
- Never store credentials, private medical details, or copyrighted material when a short summary is enough.
