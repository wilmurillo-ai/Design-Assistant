# Hermes Loop

Use this loop whenever Hermes mode is active.

## The Loop

1. Trigger
   - non-trivial task
   - repeated workflow
   - failure or correction
   - long-running project context

2. Retrieve
   - read `~/hermes-agent/memory.md`
   - read only the smallest relevant supporting file

3. Act
   - execute the current task normally
   - do not narrate the loop unless the user asks

4. Reflect
   - compare outcome vs intent
   - record one reusable lesson in reflections.md

5. Distill
   - if the lesson changes future execution, compress it into memory.md

6. Promote
   - if a pattern has repeated proof, add it to promotions.md
   - decide whether it belongs as a seed rule or a dedicated skill

## Promotion Threshold

Promote only after repeated evidence:
- 3 successful uses in similar contexts
- or 2 failures followed by a stable fix
- or explicit user request to institutionalize the behavior

## Good Candidates

- consistent project boot rituals
- repeated debugging workflows
- reliable review checklists
- stable formatting preferences
- recurring post-task summaries

## Bad Candidates

- one-off project hacks
- temporary incidents
- subjective style guesses without confirmation
- rules that only matter in one file or one hour
