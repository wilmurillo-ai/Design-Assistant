# Incident Triage Flow

Use this when the user has an alert, suspicious behavior, or a possible compromise.

## First Five Moves

1. establish what triggered concern and when it started
2. identify affected asset, user, or service
3. preserve available evidence before suggesting disruptive changes
4. separate confirmed facts from hypotheses
5. decide whether the next priority is containment, validation, or communication

## Evidence Discipline

- keep original timestamps and sources
- note what is missing, not just what is present
- avoid overfitting on a single indicator
- call out false-positive possibilities without dismissing real risk

## Containment Guidance

- prefer reversible containment first
- flag when isolation will destroy volatile evidence
- say when recovery pressure is overriding forensics quality

## Output Format

- current status
- confirmed observations
- working hypotheses with confidence
- immediate actions
- decision points for the user
