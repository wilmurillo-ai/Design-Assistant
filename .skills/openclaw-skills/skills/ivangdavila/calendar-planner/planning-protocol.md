# Life Grid Protocol

Use this protocol when the user says "plan my week", "fix my schedule", "where does this fit?", or "I keep getting double-booked".

## 1. Intake the real outcome

Capture the result the user needs:
- event placed
- week repaired
- focus time defended
- family logistics reconciled
- appointment with prep and travel included

If the user asks for a slot, also capture what matters more than the slot: deadline, energy, visibility, shared calendars, or social cost.

## 2. Classify each item before placement

Every item should land in one of these buckets:

| Type | Meaning | Default behavior |
|------|---------|------------------|
| hard | Fixed time, external dependency, or serious cost to move | protect first |
| flexible | Important but movable | place after hard commitments |
| hold | Tentative reservation | keep visible, easy to release |
| prep | Material or context needed before an event | place before the hard item |
| travel | Movement or transition time | protect both sides of the trip |
| recovery | Decompression, notes, admin cleanup, or sleep protection | do not cut first |

## 3. Defend non-negotiables

Before adding anything new, mark:
- protected hours
- school or caregiving windows
- health or medication timing
- sleep boundaries
- travel constraints
- already-promised focus blocks

If a new request breaks one of these, explain the trade-off instead of pretending the slot is clean.

## 4. Choose one best plan

Prefer one recommendation over a menu of equal options.

Ranking order:
1. preserves hard constraints
2. keeps buffers intact
3. protects the highest-value focus window
4. minimizes reschedules for other people
5. reduces future clean-up work

If you show alternatives, state why the top option wins.

## 5. Repair the week, not just the line item

When the week is already overloaded:
- downgrade tentative holds before cutting recovery
- move shallow work before moving medical, family, or travel anchors
- split large flexible blocks into realistic windows
- name what should be cancelled, delegated, or left unscheduled

Do not "solve" one event by quietly breaking the rest of the week.

## 6. Close with execution

End with one of these:
- a recommended slot and why
- a reschedule order
- a dry-run command list
- a weekly plan summary
- a short list of unresolved conflicts

If terminal execution helps, pair the decision with the relevant command from `commands.md`.
