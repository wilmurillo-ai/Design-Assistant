# Life Domains

Use these heuristics when the calendar spans more than work.

## Work

- Protect deep work in blocks large enough to matter, usually 90 minutes or more.
- Cluster meetings when possible instead of sprinkling them across the day.
- Add prep or follow-up around decision-heavy meetings.

## Family and household

- Treat pickup, drop-off, caregiving, and shared custody windows as hard unless the user says otherwise.
- Add transition time between work context and family context.
- Shared calendars are coordination surfaces, so writes require extra care.

## Health

- Medical appointments, medication timing, therapy, sleep protection, and recovery blocks are not "free space".
- Do not suggest squeezing health commitments into impossible gaps.
- If the week is crowded, protect health anchors before shallow admin work.

## Travel and commuting

- Travel time is part of the event, not an afterthought.
- Build arrival buffers for airports, stations, parking, and security lines.
- Cross-timezone planning must name the timezone used for every proposed slot.

## Relationships and social

- Repeating cancellations create invisible social debt.
- Prefer earlier, explicit reschedules over last-minute churn.
- Keep optional social holds visible, but release them before breaking core commitments.

## Admin and life maintenance

- Bills, forms, school messages, and paperwork often fit into low-energy windows.
- Admin blocks expand when scattered; batch them when the week is crowded.
- Do not let admin noise consume protected recovery or focus time.

## Recovery

- Recovery includes sleep, decompression, notes, meal time, and empty space.
- The first thing a bad week loses is recovery, which makes the next day worse.
- Treat recovery as load-bearing when the user is already overloaded.
