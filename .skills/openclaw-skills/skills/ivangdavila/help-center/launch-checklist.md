# Launch Checklist — Help Center

Use this checklist before public rollout.

## Readiness

- [ ] Provider or custom stack decision documented with scoring.
- [ ] Taxonomy approved by support and product stakeholders.
- [ ] Top 20 ticket intents covered by published articles.
- [ ] Escalation routes tested from docs to ticket workflow.
- [ ] Redirect map validated for migrated URLs.

## Quality Controls

- [ ] Search relevance tested with real support queries.
- [ ] Broken-link scan completed with zero critical failures.
- [ ] Article owners assigned for every category.
- [ ] Review cadence scheduled (monthly or quarterly).

## Go-Live Operations

- [ ] Monitoring dashboard for deflection and search misses enabled.
- [ ] On-call owner assigned for launch week.
- [ ] Rollback plan documented and rehearsed.
- [ ] Communication sent to support and customer-facing teams.

## Success Criteria (first 30 days)

- [ ] Deflection rate meets target band.
- [ ] First response time improves or remains stable.
- [ ] Escalation noise decreases for documented issues.
