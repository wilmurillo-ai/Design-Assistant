# Content Operations — Help Center

Sustain quality with a simple editorial system tied to support outcomes.

## Article Standard

Every article should include:
- Problem statement in user language
- Resolution steps in ordered sequence
- Edge-case section and known limitations
- Escalation path when self-service fails
- Last reviewed date and owner

## Editorial Workflow

| Stage | Owner | Exit Criteria |
|-------|-------|---------------|
| Draft | Subject expert | Technically accurate content |
| Review | Support lead | Clear language and correct tagging |
| Publish | Help center owner | Search metadata and links validated |
| Monitor | Ops analyst | Performance metrics captured |

## Weekly Metrics Review

Track and discuss:
- Deflection rate by category
- Top unresolved search queries
- Articles with high exit-without-click rates
- Escalations that should have been solved by docs

## Update Triggers

Refresh articles when:
- Product behavior changes
- Same ticket issue appears 3+ times in a week
- Search miss pattern persists for 2 weeks
- SLA breaches cluster around one category
