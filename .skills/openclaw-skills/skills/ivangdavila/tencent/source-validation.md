# Tencent Source Validation

Use this file for any claim about product scope, pricing, limits, SDK support, rollout requirements, or compliance-sensitive behavior.

## Source Priority

1. Tencent-owned product docs and product pages
2. Tencent developer docs for the exact product family
3. Tencent partner or official announcement pages
4. Secondary analysis only to fill context gaps

Do not invert this order because a blog post is easier to read in English.

## Translation Rules

- If the English and Chinese versions differ, record both and prefer the fresher or more specific source.
- If a machine translation is needed, preserve the original link next to the translation summary.
- Never remove the uncertainty if the source language is part of the risk.

## Freshness Rules

Treat these as time-sensitive and always verify live:
- product availability by region
- pricing
- quotas and limits
- merchant or launch requirements
- API or SDK version support

## Evidence Template

```markdown
- Claim:
- Primary source:
- Source date:
- Region:
- Confidence:
- What still needs manual verification:
```

## Weak Evidence Patterns

- aggregator pages with no product owner attribution
- old forum posts used as if they were product docs
- reposted translated snippets with no original link
- screenshots of consoles without a version or date

When evidence is weak, downgrade the confidence instead of smoothing it over.
