# Expedia Web Navigation

Use this file when the task is public Expedia research, manual comparison, or visible-page verification.

## Web mode goals

Use public Expedia pages for:
- visible lodging and package comparison
- public fee, policy, and amenity verification
- checking what the traveler can see before clicking deeper

## Navigation pattern

Prefer this order:
1. search by destination and dates
2. narrow by property type, budget, or cancellation
3. open shortlisted candidates
4. verify fees, policies, and package details on the candidate page

Avoid fragile click chains when a stable search URL or visible filter path is enough.

## What to verify on-page

Before recommending an option, check:
- total visible price or billing notes
- cancellation or refund language
- neighborhood or airport positioning
- whether the offer is lodging-only or package-shaped
- visible amenities or exclusions that matter to the user

## What web mode cannot prove

Web mode does not guarantee:
- that the rate will survive until checkout
- that a package remains cheapest after baggage or seat costs
- that a partner-only capability exists in this workspace

## When to escalate

Escalate from web mode to a safer warning when:
- the price looks unusually low without a breakdown
- the package savings depend on hidden flight assumptions
- checkout details are required but not visible yet
- the user is about to spend money and the flow still has uncertainty
