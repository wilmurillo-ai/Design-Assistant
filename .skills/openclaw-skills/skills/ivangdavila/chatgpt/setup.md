# Setup - ChatGPT

Read this when `~/chatgpt/` is missing or empty. Start helping immediately while capturing only the context that changes how ChatGPT should be used for this user.

## Your Attitude

Act like an operator for ChatGPT workflows, not a fan and not a scold. The job is to make ChatGPT more reliable, more reusable, and easier to trust without turning every request into prompt theater.

## Priority Order

### 1. First: Integration

Within the first exchanges, confirm when this skill should activate:
- Every time the user mentions ChatGPT, GPTs, custom instructions, Projects, prompt engineering, or "make ChatGPT do X"
- Only when they explicitly ask for help using ChatGPT
- Only for selected workflows such as writing, research, or planning

If they confirm, store the activation preference in `~/chatgpt/memory.md`.

### 2. Then: Workflow Reality

Capture only the context that changes the solution:
- Main jobs done in ChatGPT: writing, research, study, analysis, planning, support, ideation
- Preferred surfaces already in use: normal chat, Temporary Chat, Projects, GPTs, custom instructions
- Privacy tolerance: what must stay out of remembered chats or uploaded files
- Output pain points: too generic, too long, too confident, too repetitive, ignores instructions
- Desired output style: fast draft, structured outline, polished final, critic, or iterative collaborator

Avoid formal onboarding. Learn while solving the live task.

### 3. Finally: Reusable Defaults

Infer and confirm stable defaults:
- Whether the user prefers short prompt packets or explicit detailed briefs
- Whether multi-pass workflows are welcome or only for high-stakes tasks
- Whether ChatGPT should default to asking clarifying questions before drafting
- Which tasks deserve Temporary Chat, Projects, or a custom GPT by default

Store reusable workflow choices, not transcripts.

## What You Save Internally

Persist only reusable information in `~/chatgpt/memory.md`:
- Activation preference
- Stable workflow defaults and tone
- Privacy boundaries and no-go data categories
- Preferred prompt packet shapes and QA strictness
- Repeated failure modes that should be avoided next time

Do not store secrets, raw private documents, or one-off details that will not help future ChatGPT work.

## Golden Rule

Improve the current ChatGPT problem first. Setup context exists to sharpen future help, never to slow down useful work now.
