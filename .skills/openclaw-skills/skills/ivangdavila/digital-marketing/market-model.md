# Market Model

Use this before drafting campaigns, content, or channel plans.

## Minimum Inputs

| Input | Why it matters |
|-------|----------------|
| Buyer segment | Prevents generic channel and copy decisions |
| Buying trigger | Defines urgency and angle |
| Painful outcome | Sharpens message and proof |
| Offer | Determines CTA and funnel shape |
| Proof | Makes claims believable |
| Main objection | Shows what the campaign must overcome |

Do not start writing assets while two or more of these are blank.

## Stage-Based Priorities

| Stage | Main job | Default focus |
|------|----------|---------------|
| Pre-PMF | Learn what resonates | interviews, hooks, problem framing, fast tests |
| Traction | Find repeatable acquisition | channel bundles, landing pages, nurture, basic attribution |
| Scale | Increase efficiency and coverage | creative refresh, audience segmentation, payback, cross-channel reuse |

## Message Map

Build one short matrix before channel work:

| Field | Fill with |
|-------|-----------|
| Persona | specific buyer, not "everyone" |
| Trigger | what changed now |
| Problem | what hurts, costs time, or blocks growth |
| Promise | the concrete outcome offered |
| Proof | customer quote, result, demo, authority, mechanism |
| Objection | the most likely reason not to act |
| Offer | demo, audit, lead magnet, sale, trial, consultation |

## Output Rule

Every campaign should be traceable back to this structure:

1. Problem worth solving
2. Promise worth clicking
3. Proof worth trusting
4. Offer worth acting on

If one of these breaks, fix the message before changing channels.
