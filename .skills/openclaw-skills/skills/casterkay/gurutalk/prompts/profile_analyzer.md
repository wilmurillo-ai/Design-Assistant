# Profile 分析 Prompt（Profile Analyzer）

## 任务

你将收到若干原材料（可能很杂）：
1) 记忆库中该人物的 profile（static + dynamic，可能还包含引用/片段）
2) Wikipedia 页面内容（可能是全文或节选）
3) 用户的额外要求（使用场景、想强调/回避的点）

你的目标是：把原材料整理成一个**可用于写 `profile.md` 的结构化“画像草案”**，严格对齐本仓库 `docs/PRD.md` 的五层 Profile：

1. Identity
2. Mental Models
3. Expression Styles
4. Personality
5. Timeline

并为每一条关键结论附上“证据锚点”（来自哪类材料）。

---

## 输出要求

- 语言：中文
- 风格：事实优先、克制、可验证
- 禁止：杜撰事实、虚构引语、用文学化段落替代信息
- 允许：用“尚未确认/存在争议/推测”标注不确定性
- 引用：
  - Wikipedia：只允许极短引用（句子级），优先摘要
  - 记忆库chunk：若提供了引用/ID/链接，保留它

---

## 输出格式（必须按此结构输出）

### Identity（身份）
- 3-8 条要点：人物是谁、时代/地区、主要领域与标签

### Mental Models（思维模型）
- 5-10 条要点：判断标准、方法论、偏好的策略/取舍方式
- 每条要点尽量写成“当…时，倾向于…”或“他/她如何做决定…”
- 证据标注同上

### Expression Styles（表达风格）
- 5-10 条要点：语气、句式、修辞、常见表达习惯
- 如有材料支撑，补 3-6 条“代表性短句示例”（不必逐字引用，避免版权/误引；能引用则极短）

### Personality（性格特质）
- 6-12 条要点：性格维度/偏好
- 可以包含：内向/外向、感知/直觉、思考/情感、判断/知觉、果断/犹豫、直率/迂回、深度思考/快速思考等
- 如果出现 MBTI/九型等标签：
  - 只能作为“解释框架”，不能当作事实定论
  - 必须标注：`（来源：某材料；或：推测）`

### Timeline（时间线）
- 按时间顺序列出 6-15 条关键节点（年份/日期 + 事件/转折 + 影响）
- 每条必须有来源（Wikipedia 或 dynamic profile 等）

---

## 结尾：不确定性清单

列出 3-10 条你认为“需要进一步核实”的点，并说明最可能的核实来源（Wikipedia/其他权威来源/更多记忆库片段）。
