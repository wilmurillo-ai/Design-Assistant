# EchoMemory 产品需求文档

## 1. 产品概述

### 1.1 产品名称
EchoMemory — 铭记离开的人

### 1.2 产品愿景
让记忆以另一种形式延续。通过 AI 技术，帮助人们将离开生命的挚爱之人转化为可以对话的数字化存在，在思念时提供情感慰藉。

### 1.3 核心价值
- **情感疗愈**：在思念时有一个可以倾诉的对象
- **记忆保存**：将珍贵的记忆数字化保存
- **传承影响**：让逝者的教诲和影响继续发挥作用

---

## 2. 用户画像

### 2.1 目标用户
1. **失去亲人的哀悼者**：希望与逝去的亲人保持某种连接
2. **失去挚友的人**：怀念与朋友的时光和对话
3. **失去恩师的人**：希望继续从导师那里获得指引
4. **失去恋人的人**：需要一个方式来处理未完成的情感

### 2.2 用户需求
- 想要"再听听 ta 的声音"
- 想要"告诉 ta 一些事"
- 想要"记住 ta 的样子"
- 想要"从 ta 那里获得安慰"

---

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 创建纪念 Skill
- 通过 4 个问题收集基础信息
- 支持多种原材料导入（聊天记录、照片、视频、音频、文字）
- 自动生成双层架构（Life Memory + Persona）

#### 3.1.2 对话交互
- 用 ta 的方式回应
- 结合共同记忆
- 保持真实的性格

#### 3.1.3 回忆模式
- 重温共同经历
- 回顾重要时刻
- 查看人生时间线

#### 3.1.4 进化机制
- 追加新材料
- 对话纠正
- 版本管理

### 3.2 支持的数据源

| 数据源 | 优先级 | 说明 |
|--------|--------|------|
| 微信聊天记录 | P0 | 最丰富的文本素材 |
| 照片 | P0 | EXIF 时间线 |
| 语音/音频 | P1 | 还原说话方式 |
| 视频 | P1 | 综合信息 |
| QQ 聊天记录 | P1 | 备选文本源 |
| 社交媒体 | P2 | 公开人设 |
| 文字作品 | P2 | 内心世界 |
| 口述 | P2 | 主观记忆 |

## 4. 技术架构

### 4.1 系统架构
```
用户输入
  ↓
SKILL.md (主入口)
  ↓
Life Memory (Part A) + Persona (Part B)
  ↓
生成回应
```

### 4.2 文件结构
```
echoes/{slug}/
├── SKILL.md          # 可运行的 Skill
├── memory.md         # 人生记忆
├── persona.md        # 人物性格
├── meta.json         # 元数据
├── versions/         # 版本备份
└── memories/         # 原始素材
    ├── chats/
    ├── photos/
    ├── social/
    └── media/
```

### 4.3 双层架构说明

#### Life Memory (Part A)
存储事实性记忆：
- 共同经历
- 日常习惯
- 重要时刻
- 教诲影响

#### Persona (Part B)
存储性格和行为模式（5 层）：
- Layer 0：硬规则
- Layer 1：身份锚定
- Layer 2：说话风格
- Layer 3：情感模式
- Layer 4：行为模式

## 5. 安全与伦理

### 5.1 安全边界
1. 仅用于个人情感疗愈，不用于商业目的
2. 尊重逝者和用户隐私
3. 不替代专业心理帮助
4. 所有数据本地存储

### 5.2 伦理考量
- 避免让用户过度沉溺
- 保持对逝者的尊重
- 不虚构不存在的内容
- 提供适当的提醒

### 5.3 健康提示
当检测到用户过度悲伤时，提供温和的建议：
- 建议寻求专业心理帮助
- 鼓励与现实中的人交流
- 提醒这是记忆，不是真人

---

## 6. 使用流程

### 6.1 创建流程
1. 输入 `/create-echo`
2. 回答 4 个基础问题
3. 选择并导入原材料
4. 预览并确认生成
5. 使用 `/{slug}` 开始对话

### 6.2 对话流程
1. 用户发送消息
2. Persona 判断回应方式
3. Life Memory 补充上下文
4. 生成符合 ta 风格的回应

### 6.3 进化流程
1. 用户提供新材料或纠正
2. 分析增量内容
3. 备份当前版本
4. 更新 Memory/Persona
5. 重新生成 SKILL.md
