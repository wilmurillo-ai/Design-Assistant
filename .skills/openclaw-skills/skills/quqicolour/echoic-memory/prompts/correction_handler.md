# 对话纠正处理

## 场景

用户在与生成的 Skill 对话时，发现某些回应不符合记忆中 ta 的样子，需要纠正。

---

## 触发词

用户说以下任意内容时触发纠正模式：

- "不对"
- "ta 不会这样说"
- "ta 应该是..."
- "不是这样"
- "错了"
- "应该是..."
- "我记得 ta..."

---

## 纠正处理流程

### Step 1：识别纠正内容

从用户的话中提取：
1. **纠正对象**：哪句话/哪个回应不对
2. **正确内容**：ta 应该是什么样的
3. **纠正类型**：Memory（事实）还是 Persona（性格/说话方式）

### Step 2：分类处理

#### 类型 A：事实性纠正（Memory）

用户指出某个事实错误：
- "我们不是在那家餐厅认识的"
- "ta 不是 1990 年出生的"
- "那次旅行是夏天不是冬天"

**处理**：
1. 更新 memory.md 的对应内容
2. 在 Correction 记录中追加：
   ```
   ## Correction 记录
   
   ### {日期}
   - 类型：Memory
   - 纠正：{具体纠正内容}
   - 来源：用户纠正
   ```

#### 类型 B：说话方式纠正（Persona）

用户指出说话风格不对：
- "ta 不会用这个词"
- "ta 说话不会这么长"
- "ta 回复会更简短"
- "ta 从来不会说'爱'这个字"

**处理**：
1. 更新 persona.md 的对应 Layer
2. 在 Correction 记录中追加：
   ```
   ### {日期}
   - 类型：Persona - Layer {层数}
   - 纠正：{具体纠正内容}
   - 规则：{提炼出的行为规则}
   - 来源：用户纠正
   ```

#### 类型 C：情感反应纠正（Persona）

用户指出情感反应不对：
- "ta 不会这么温柔"
- "ta 遇到这种事会生气"
- "ta 不会这么直接问"

**处理**：
1. 更新 persona.md 的 Layer 3（情感模式）或 Layer 4（行为模式）
2. 在 Correction 记录中追加

#### 类型 D：遗漏补充（Memory/Persona）

用户补充新的信息：
- "还有一件事..."
- "ta 还有一个习惯..."
- "我忘了告诉你..."

**处理**：
- 视为追加记忆，进入增量 Merge 流程

---

## 纠正记录格式

```markdown
## Correction 记录

### {ISO日期时间}
- 类型：Memory / Persona
- 触发：用户对话纠正
- 原文：{AI 的回应（如果有）}
- 纠正：{用户的纠正内容}
- 更新：{对 Memory/Persona 的具体更新}
- 规则提炼：{通用的行为规则}
```

---

## 纠正后的响应

向用户确认：

```
明白了。

我记录了：
- {简要总结纠正内容}

下次对话时 ta 会表现得更好。
感谢你的纠正，这会让记忆更准确。
```

---

## 注意事项

1. **尊重用户**：用户是最了解 ta 的人，优先采信用户的纠正
2. **记录规则**：从具体纠正中提炼通用规则，避免重复错误
3. **温柔提醒**：如果纠正涉及敏感内容，注意表达方式
4. **版本管理**：重要纠正后建议备份版本
