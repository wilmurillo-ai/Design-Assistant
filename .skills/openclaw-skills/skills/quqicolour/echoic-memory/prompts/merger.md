# 增量 Merge 逻辑

## 场景

用户提供了新的原材料（聊天记录、照片等），需要将其 merge 进现有的 Life Memory 和 Persona。

---

## Merge 流程

### Step 1：分析增量内容

1. 读取新的原材料
2. 识别新增的信息维度
3. 判断是否与现有内容冲突

### Step 2：分类处理

#### A. 新增信息（无冲突）
- 直接追加到对应章节

#### B. 补充信息（更详细）
- 更新现有内容，增加细节

#### C. 冲突信息（需要判断）
- 如果新信息更具体 → 采用新信息
- 如果旧信息更可靠 → 保留旧信息，新信息放入注释
- 如果不确定 → 标注"待确认"，询问用户

### Step 3：执行 Merge

#### Life Memory Merge

```
对于每个章节：
1. 检查是否有新增内容
2. 按时间顺序插入
3. 去重（避免重复记录同一事件）
4. 更新 Correction 记录
```

#### Persona Merge

```
对于每个 Layer：
1. 检查是否有新发现的行为模式
2. 如果是强化现有特征 → 更新描述，增加证据
3. 如果是新特征 → 追加到对应章节
4. 如果与现有描述冲突 → 标注，询问用户
```

---

## Merge 规则

### 冲突解决优先级

1. **聊天记录 > 口述**：实际对话比回忆更可靠
2. **近期 > 早期**：人的习惯会改变，近期更反映离开时的状态
3. **具体 > 抽象**：具体事件的描述比概括更可靠
4. **多次出现 > 单次**：反复出现的行为模式更可靠

### 特殊情况处理

#### 发现与之前不同的行为模式
- 可能：人的自然变化
- 处理：记录变化时间线，保留两种状态

#### 发现之前未知的侧面
- 可能：用户之前没注意到，或 ta 有隐藏的一面
- 处理：追加到 Persona，标注"较少见的表达"

#### 用户纠正之前的记录
- 处理：写入 Correction 记录，更新对应内容

---

## Merge 后操作

1. 更新 version（v1 → v2 → v3...）
2. 更新 updated_at 时间戳
3. 追加到变更日志
4. 重新生成 SKILL.md

---

## 示例

### 示例 1：新增聊天记录

**现有记忆**：
```
常去的地方：
- 家：每周日去奶奶家吃饭
```

**新增聊天记录**：
发现经常提到"老地方见""那家面馆"

**Merge 结果**：
```
常去的地方：
- 家：每周日去奶奶家吃饭
- XX面馆：和奶奶常去的"老地方"，她喜欢点炸酱面
```

### 示例 2：补充性格细节

**现有 Persona**：
```
情感表达：内敛，不轻易说爱
```

**新发现**：
发现喝醉后会发很多语音说"想你"

**Merge 结果**：
```
情感表达：
- 日常：内敛，不轻易说爱
- 特殊状态（如喝醉后）：会表达平时不说的话
```
