# Resume Writing Quality Guidelines

## Grammar and Style Rules

### Action Verbs
**Use strong, specific action verbs:**
- Developed, Implemented, Architected, Designed, Built, Created
- Optimized, Improved, Reduced, Increased, Scaled, Enhanced
- Led, Managed, Mentored, Collaborated, Coordinated
- Analyzed, Evaluated, Investigated, Researched
- Automated, Streamlined, Refactored, Migrated

**Avoid weak verbs:**
- Worked on, Helped with, Was responsible for, Assisted with

### Tense Consistency
- Current positions: Present tense ("Develop", "Lead", "Manage")
- Past positions: Past tense ("Developed", "Led", "Managed")
- Be consistent within each role

### Conciseness
- Eliminate redundant words ("in order to" → "to")
- Remove filler phrases ("was responsible for managing" → "managed")
- Each bullet should be one to two lines maximum
- Avoid complete sentences; use fragments starting with action verbs

### Common Grammar Mistakes
- Subject-verb agreement errors
- Incorrect use of "that" vs. "which"
- Misplaced modifiers
- Run-on sentences or sentence fragments (in summary sections)
- Inconsistent punctuation at end of bullets

### Technical Writing Clarity
- Be specific: "Reduced API response time from 500ms to 50ms" not "Made API faster"
- Use precise terminology: "PostgreSQL database" not "SQL database" when specific
- Avoid jargon the reader may not know without context
- Define acronyms on first use if not universally known

## Formatting Consistency

### Punctuation
- Choose bullets with OR without ending periods—be consistent
- Use parallel structure in bullet lists
- Consistent use of commas in lists

### Dates and Numbers
- Use consistent date format (e.g., "Jan 2023" or "January 2023" throughout)
- Spell out numbers one through nine; use numerals for 10 and above
- Use "1M" or "1 million" consistently
- Include units (%, ms, GB, users, etc.)

### Capitalization
- Consistent title capitalization for role names
- Proper nouns (company names, product names) capitalized
- Technology names with correct casing (JavaScript not Javascript, PostgreSQL not Postgresql)

## Content Quality Indicators

### Strong Bullet Points
- Start with action verb
- Include specific metric or outcome
- Provide context of scope or scale
- Show business or user impact
- Example: "Optimized database queries reducing page load time by 60% for 500K daily users, improving user retention by 15%"

### Weak Bullet Points
- Vague accomplishments: "Improved system performance"
- Lists of technologies without context: "Used React, Node.js, and MongoDB"
- Job description rather than achievement: "Responsible for maintaining codebase"
- No measurable impact: "Made the website better"

### Storytelling Through Bullets
- Each role should tell a progression story
- Show increasing responsibility and impact
- Demonstrate technical growth and leadership
- Connect individual contributions to team/company goals

## LaTeX-Specific Considerations

### Common Issues
- Inconsistent spacing around section headers
- Overuse of custom commands creating ATS parsing problems
- Special characters that don't render correctly
- Poor hyphenation in justified text
- Line breaks in inappropriate places

### Best Practices
- Use standard document classes when possible
- Test PDF text extraction to ensure ATS compatibility
- Avoid complex multi-column layouts for main content
- Use semantic markup rather than visual formatting
- Ensure links are clickable and formatted properly
