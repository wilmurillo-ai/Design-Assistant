## Resume Impact Indicators

### Strong Technical Indicators
- Quantified impact metrics (e.g., "reduced latency by 40%", "scaled to 10M users")
- Open source contributions with stars/downloads
- Production systems at scale
- Cross-functional collaboration
- Technical leadership and mentoring
- System design and architecture decisions

### Red Flags
- Buzzword stuffing without substance
- No metrics or measurable outcomes
- Only academic/tutorial projects for experienced candidates
- Technology lists without context of usage
- Vague responsibilities without accomplishments
- Outdated technology stacks for recent experience

## ATS Optimization Guidelines

### ATS-Friendly Practices
- Use standard section headers (Experience, Education, Skills)
- Include keywords from job descriptions naturally
- Use standard fonts (Arial, Calibri, Times New Roman)
- Avoid tables, text boxes, headers/footers for critical content
- Use standard bullet points (•, -, *)
- Include both acronyms and full terms (e.g., "CI/CD (Continuous Integration/Continuous Deployment)")
- Use chronological or reverse-chronological format
- Save as PDF for formatting preservation

### Common ATS Issues in LaTeX Resumes
- Multi-column layouts may confuse parsing
- Custom symbols or special characters
- Graphics and images
- Complex formatting that doesn't translate to plain text
- Non-standard section names

## Content Structure Best Practices

### Professional Summary (Optional)
- 2-3 sentences highlighting key expertise and value proposition
- Include years of experience and primary domain
- Focus on impact, not responsibilities

### Experience Section
- Use reverse chronological order
- Include company name, title, dates, location
- Lead with impact statements using strong action verbs
- Quantify achievements wherever possible
- Focus on business impact, not just technical tasks
- Use STAR method (Situation, Task, Action, Result) implicitly

### Skills Section
- Group by category (Languages, Frameworks, Tools, etc.)
- List technologies in order of proficiency or relevance
- Include both technical and soft skills when appropriate
- Avoid rating systems (e.g., "Expert in X" vs. bars/stars)

### Projects Section (for junior candidates)
- Include project name, tech stack, and link if available
- Focus on personal initiative and learning
- Highlight any real-world usage or adoption
- Show progression of complexity

### Education
- Include degree, institution, graduation year
- Add relevant coursework only if junior or career-changing
- Include GPA only if exceptional (>3.7) and recent graduate
- Certifications should be recent and relevant
