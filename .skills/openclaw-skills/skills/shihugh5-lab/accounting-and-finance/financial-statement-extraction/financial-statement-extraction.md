# 财务报表数据提取：分析工作的数据基石

## 📚 一、理论背景与核心概念

### 1.1 数据提取在财务分析中的基础地位

> "财务分析的质量首先取决于数据的质量。原始数据的不准确、不完整将直接导致分析结论的偏差，甚至误导投资决策。数据提取是财务分析的'地基工程'，必须扎实可靠。"

**数据质量的重要性**：
| 数据问题 | 对分析的影响 | 典型案例 |
|:---|:---|:---|
| 科目缺失 | 无法完整重构报表 | 遗漏少数股东权益 |
| 单位错误 | 数量级错误 | 万元当亿元使用 |
| 期间混淆 | 同比/环比错误 | 使用单季度而非TTM |
| 勾稽不平 | 数据内部矛盾 | 三张报表不一致 |

**数据提取的价值链**：
```
原始披露 → 数据提取 → 数据清洗 → 数据验证 → 结构化数据 → 财务分析
    │          │          │          │           │
   PDF      解析识别   标准化    勾稽检验    重构分析
  Excel     科目映射   单位统一   异常检测    估值建模
  API       多期整合   格式规范   完整性检查   投资决策
```

### 1.2 财务报告编制基础与数据来源

**中国财务报告体系**：
```
信息披露体系
├── 定期报告
│   ├── 年度报告（4月底前披露）
│   ├── 半年度报告（8月底前披露）
│   └── 季度报告（4月、10月）
├── 临时报告
│   ├── 重大事项公告
│   ├── 业绩预告
│   └── 问询函回复
└── 其他披露
    ├── 招股说明书
    ├── 增发说明书
    └── 债券募集说明书
```

**主要数据来源对比**：

| 数据源 | 优点 | 缺点 | 适用场景 |
|:---|:---|:---|:---|
| **PDF年报** | 原始、权威、完整 | 解析困难、格式不一 | 深度分析、验证 |
| **Excel导出** | 结构化、易处理 | 可能缺失注释 | 快速分析 |
| **金融终端** | 标准化、带计算 | 费用高、字段有限 | 批量分析 |
| **API接口** | 自动化、实时 | 需开发、有频率限制 | 系统构建 |

### 1.3 财务报表的结构化特征

**资产负债表的结构性**：
```
会计恒等式：资产 = 负债 + 所有者权益

结构性特征：
├── 流动性排序（流动→非流动）
├── 到期日结构（短期→长期）
├── 科目层级（一级科目→明细）
└── 合并范围（母公司→子公司）
```

**利润表的期间性**：
```
收入 - 费用 = 利润

期间特征：
├── 月度：管理需要，非公开
├── 季度：法定披露，未经审计
├── 半年度：法定披露，可选审计
└── 年度：法定披露，强制审计
```

**现金流量表的分类性**：
```
现金净增加额 = 经营现金流 + 投资现金流 + 筹资现金流

分类逻辑：
├── 经营活动：日常业务相关
├── 投资活动：长期资产相关
└── 筹资活动：融资活动相关
```

### 1.4 数据提取的理论挑战

**会计政策差异**：
| 差异类型 | 具体表现 | 处理策略 |
|:---|:---|:---|
| **收入确认** | 时点法vs时段法 | 识别政策，对比调整 |
| **折旧方法** | 直线法vs加速法 | 标准化调整 |
| **存货计价** | 先进先出vs加权平均 | 注意比较基准 |
| **租赁处理** | 经营租赁vs融资租赁 | 新准则已统一 |

**披露质量差异**：
- **详细程度**：有些公司附注详尽，有些简略
- **可比性**：历史数据口径可能变化
- **完整性**：表外项目披露不充分
- **及时性**：披露时点差异影响分析

```python
import pdfplumber
import pandas as pd

def 提取财务报表_from_pdf(pdf_path):
    """
    从PDF年报中提取三张财务报表
    """
    报表数据 = {
        '资产负债表': None,
        '利润表': None,
        '现金流量表': None
    }
    
    with pdfplumber.open(pdf_path) as pdf:
        总页数 = len(pdf.pages)
        
        for page_num in range(总页数):
            page = pdf.pages[page_num]
            text = page.extract_text()
            
            if not text:
                continue
            
            # 识别资产负债表
            if any(keyword in text for keyword in ['合并资产负债表', '母公司资产负债表']):
                print(f"在page {page_num + 1} 发现资产负债表")
                tables = page.extract_tables()
                if tables:
                    报表数据['资产负债表'] = 处理资产负债表表格(tables[0])
            
            # 识别利润表
            elif any(keyword in text for keyword in ['合并利润表', '合并损益表']):
                print(f"在page {page_num + 1} 发现利润表")
                tables = page.extract_tables()
                if tables:
                    报表数据['利润表'] = 处理利润表表格(tables[0])
            
            # 识别现金流量表
            elif any(keyword in text for keyword in ['合并现金流量表']):
                print(f"在page {page_num + 1} 发现现金流量表")
                tables = page.extract_tables()
                if tables:
                    报表数据['现金流量表'] = 处理现金流量表表格(tables[0])
    
    return 报表数据
```

**表格后处理函数**：

```python
def 处理资产负债表表格(raw_table):
    """
    处理提取的原始表格数据
    """
    # 转换为DataFrame
    df = pd.DataFrame(raw_table[1:], columns=raw_table[0])
    
    # 识别单位（通常在表头）
    单位 = 识别单位(df)
    
    # 清洗数据
    df = 清洗财务数据(df, 单位)
    
    # 标准化科目名称
    df['科目'] = df['科目'].apply(标准化科目名称)
    
    return df

def 识别单位(df):
    """
    从表头识别金额单位
    """
    表头文本 = ' '.join(df.columns.astype(str))
    
    if '亿元' in 表头文本 or '亿元' in str(df.iloc[0]):
        return '亿元'
    elif '万元' in 表头文本 or '万元' in str(df.iloc[0]):
        return '万元'
    elif '千元' in 表头文本:
        return '千元'
    else:
        return '元'

def 清洗财务数据(df, 单位):
    """
    清洗财务数据：去除千分位、处理负数、统一单位
    """
    转换因子 = {
        '元': 1,
        '千元': 1e3,
        '万元': 1e4,
        '亿元': 1e8
    }
    
    因子 = 转换因子.get(单位, 1)
    
    for col in df.columns:
        if col != '科目':
            df[col] = df[col].apply(lambda x: 转换数值(x, 因子))
    
    return df

def 转换数值(值, 因子):
    """
    将文本数值转换为标准数值
    """
    if pd.isna(值) or 值 == '' or 值 == '-':
        return 0
    
    # 转换为字符串
    值 = str(值)
    
    # 去除千分位逗号
    值 = 值.replace(',', '').replace('，', '')
    
    # 处理括号负数 (100) → -100
    if '(' in 值 and ')' in 值:
        值 = 值.replace('(', '-').replace(')', '')
    
    # 处理红色负数（某些PDF使用红色表示负数）
    # 注：需要OCR识别颜色，此处简化
    
    try:
        return float(值) / 因子  # 转换为亿元
    except:
        return 0
```

```python
def 标准化科目名称(原始名称):

    # 查找映射
```

### 步骤4：数据验证与质量控制

**勾稽关系检验**：

```python
def 检验报表勾稽关系(资产负债表, 利润表, 现金流量表):
    """
    检验三张报表的勾稽关系
    返回检验结果列表
    """
    检验结果 = []
    
    # 1. 资产负债表恒等式：资产 = 负债 + 权益
    资产 = 资产负债表.get('资产总计', 0)
    负债 = 资产负债表.get('负债合计', 0)
    权益 = 资产负债表.get('股东权益合计', 0)
    
    if abs(资产 - (负债 + 权益)) > 1:  # 1亿元误差容忍
        检验结果.append({
            '检验项': '资产负债表恒等式',
            '期望': f'{资产:.2f} = {负债:.2f} + {权益:.2f}',
            '实际': f'差异: {资产 - (负债 + 权益):.2f}',
            '状态': '❌ 不通过'
        })
    else:
        检验结果.append({
            '检验项': '资产负债表恒等式',
            '状态': '✅ 通过'
        })
    
    # 2. 利润表逻辑：收入 - 成本 - 费用 = 利润
    收入 = 利润表.get('营业收入', 0)
    成本 = 利润表.get('营业成本', 0)
    税金 = 利润表.get('税金及附加', 0)
    销售费用 = 利润表.get('销售费用', 0)
    管理费用 = 利润表.get('管理费用', 0)
    研发费用 = 利润表.get('研发费用', 0)
    财务费用 = 利润表.get('财务费用', 0)
    营业利润 = 利润表.get('营业利润', 0)
    
    计算营业利润 = 收入 - 成本 - 税金 - 销售费用 - 管理费用 - 研发费用 - 财务费用
    
    if abs(计算营业利润 - 营业利润) > 1:
        检验结果.append({
            '检验项': '营业利润计算',
            '期望': f'{营业利润:.2f}',
            '实际': f'{计算营业利润:.2f}',
            '差异': f'{计算营业利润 - 营业利润:.2f}',
            '状态': '❌ 不通过',
            '备注': '可能遗漏其他收支项目'
        })
    else:
        检验结果.append({
            '检验项': '营业利润计算',
            '状态': '✅ 通过'
        })
    
    # 3. 现金流量表与资产负债表现金勾稽
    现金增加 = 现金流量表.get('现金及现金等价物净增加额', 0)
```

```python
def 检测数据异常(本期数据, 历史数据=None):
    """
    检测财务数据中的异常值
    """
    异常点 = []
    
    # 1. 检查空值和零值
    关键科目 = ['营业收入', '净利润', '总资产']
    for 科目 in 关键科目:
        值 = 本期数据.get(科目)
        if 值 is None or 值 == 0:
            异常点.append({
                '类型': '数据缺失',
                '科目': 科目,
                '描述': '关键科目值为空或零',
                '建议': '检查原始数据提取是否完整'
            })
    
    # 2. 检查异常比例
    收入 = 本期数据.get('营业收入', 0)
    成本 = 本期数据.get('营业成本', 0)
    
    if 收入 > 0:
        毛利率 = (收入 - 成本) / 收入
        if 毛利率 < 0:
            异常点.append({
                '类型': '异常毛利率',
                '科目': '毛利率',
                '数值': f'{毛利率:.2%}',
                '描述': '毛利率为负，主营业务亏损',
                '建议': '核实收入和成本数据'
            })
        elif 毛利率 > 0.9:
            异常点.append({
                '类型': '异常毛利率',
                '科目': '毛利率',
                '数值': f'{毛利率:.2%}',
                '描述': '毛利率过高，可能成本数据不完整',
                '建议': '检查是否遗漏成本项目'
            })
    
    # 3. 与历史数据对比（如有）
    if 历史数据:
        for 科目 in 关键科目:
            本期值 = 本期数据.get(科目, 0)
            历史均值 = sum(历史数据.get(科目, [])) / len(历史数据.get(科目, [1]))
            
            if 历史均值 > 0 and 本期值 > 0:
                变动率 = (本期值 - 历史均值) / 历史均值
                if abs(变动率) > 0.5:  # 50%变动
                    异常点.append({
                        '类型': '大幅变动',
                        '科目': 科目,
                        '变动率': f'{变动率:.2%}',
                        '描述': f'较历史均值变动超过50%',
                        '建议': '分析变动原因，检查数据准确性'
                    })
    
    return 异常点
```

## 🔢 三、公式与计算逻辑

### 3.1 单位换算公式

**标准单位体系**（建议统一为亿元）：
```
1 亿元 = 10,000 万元 = 100,000,000 元

换算公式：
亿元 = 元 × 10^-8
亿元 = 万元 × 10^-4
亿元 = 百万元 × 10^-2
```

**自动识别与转换**：
```python
def 自动识别并转换(数值文本, 上下文=None):
    """
    自动识别单位并转换为亿元
    """
    数值文本 = str(数值文本).strip()
    
    # 提取数字部分
    import re
    数字匹配 = re.search(r'[-+]?[\d,]+\.?\d*', 数值文本)
    if not 数字匹配:
        return None
    
    数字 = float(数字匹配.group().replace(',', ''))
    
    # 判断单位
    if '亿' in 数值文本:
        return 数字
    elif '万' in 数值文本 and '百万' not in 数值文本:
        return 数字 * 1e-4
    elif '千' in 数值文本 and '千万' not in 数值文本:
        return 数字 * 1e-5
    elif '百万' in 数值文本 or 'M' in 数值文本:
        return 数字 * 1e-2
    elif '千' in 数值文本 or 'K' in 数值文本:
        return 数字 * 1e-5
    else:
        # 根据数值大小猜测单位
        if abs(数字) > 1e9:
            return 数字 * 1e-9  # 元
        elif abs(数字) > 1e5:
            return 数字 * 1e-4  # 万元
        else:
            return 数字  # 假设已经是亿元
```

### 3.2 数据完整性评分

```
完整性评分 = Σ(关键科目得分 × 权重) × 100

关键科目权重：
- 货币资金：10%
- 应收账款：10%
- 存货：10%
- 固定资产：10%
- 总资产：10%
- 营业收入：15%
- 营业成本：10%
- 净利润：15%
- 经营现金流：10%

得分规则：
- 数据完整且准确：10分
- 数据存在但存疑：5分
- 数据缺失：0分

评分等级：
- 90-100分：数据完整 ✅
- 70-89分：数据基本完整 ⚠️
- 50-69分：数据有缺失 ❌
- <50分：数据严重缺失 🔴
```

### 3.3 科目覆盖率计算

```
覆盖率 = 匹配的标准科目数 / 标准科目表总数 × 100%

标准科目表包含：
- 资产类：20个核心科目

- 损益类：20个核心科目
- 现金流类：15个核心科目

总计：78个标准科目

覆盖率等级：
- >90%：优秀
- 70-90%：良好
- 50-70%：一般
- <50%：较差
```

## ⚠️ 五、常见问题与注意事项

### 5.1 PDF解析常见问题

**问题1：表格识别不完整**
- **现象**：表格跨页被分割，或合并单元格识别错误
- **原因**：PDF生成方式不同，表格结构复杂
- **解决**：
  ```python
  # 跨页表格合并处理
  def 合并跨页表格(页1表格, 页2表格):
      # 识别续表标记
      if '续表' in str(页2表格.iloc[0]):
          return pd.concat([页1表格, 页2表格.iloc[1:]], ignore_index=True)
      return 页1表格
  ```

**问题2：OCR识别错误**
- **现象**：数字识别错误（如0和6混淆）
- **原因**：扫描版PDF质量差，字体特殊
- **解决**：
  - 优先使用文本层PDF
  - 人工抽查关键数据
  - 使用多引擎OCR对比

**问题3：单位识别错误**
- **现象**：把"万元"当成"元"
- **后果**：数量级错误，分析完全失真
- **解决**：
  - 强制检查单位标注
  - 与常识对比（如货币资金不可能只有几百元）
  - 使用勾稽关系反推

### 5.2 数据标准化陷阱

**陷阱1：科目名称变化**
- **现象**：同一家公司不同年度科目改名
- **例子**："应收账款"→"应收票据及应收账款"
- **解决**：维护科目历史映射表

**陷阱2：会计政策变更**
- **现象**：新收入准则实施导致科目调整
- **影响**：历史数据不可比
- **解决**：追溯调整或分段分析

**陷阱3：合并范围变化**
- **现象**：并购或处置子公司导致数据波动
- **识别**：关注"同一控制下企业合并"等注释
- **解决**：剔除非经常性影响或分部分析

### 5.3 数据源选择建议

**优先推荐顺序**：
1. **金融终端API**（Wind/Tushare）：结构化程度高，准确性有保障
2. **官方Excel下载**：格式规范，适合批量处理
3. **PDF年报**：权威原始数据，适合验证和深度分析
4. **第三方数据**：谨慎使用，需交叉验证

**数据交叉验证原则**：
```
单一数据源：可快速分析，但需接受风险
双数据源对比：推荐，可发现数据差异
三数据源确认：严谨，适合重要投资决策
```
