# 竞品情报警报系统

全网监控竞品动态，自动生成情报简报。

## 触发词

当用户提到以下内容时激活此skill：
- "监控竞品"
- "竞争对手动态"
- "竞品分析"
- "市场监控"
- "情报收集"
- "监控[公司名]"
- "竞品周报"
- "竞争对手在做什么"
- "商业情报"
- "舆情监控"

---

## 核心能力

### 1. 竞品名单管理

**管理维度：**
- 公司名称（全称+简称+曾用名）
- 官网地址
- 主要社交媒体账号
- 关键产品线
- 融资阶段（用于判断公司状态）
- 竞争对手优先级（核心/重要/一般）

**输出：** 结构化竞品数据库，可持续更新

### 2. 全网动态监控

**监控范围：**

| 监控类型 | 信息来源 | 频率 |
|---------|---------|------|
| 官网更新 | 官网公告、新闻 | 每天 |
| 社交媒体 | 微信公众号、微博、抖音、小红书 | 每天 |
| 内容发布 | 知乎、36氪、虎嗅、投资界 | 每天 |
| 招聘信息 | 拉勾、Boss直聘、猎聘 | 每周 |
| 融资动态 | IT桔子、天眼查、烯牛数据 | 每天 |
| 舆情监控 | 全网新闻、社交媒体提及 | 每天 |
| 价格变动 | 官网、电商平台 | 每周 |
| 监管信息 | 监管机构披露、企业信用信息 | 每周 |

### 3. 情报简报生成

**两种报告模式：**

#### 每日快报（5分钟生成）
```
📦 [公司名] 每日情报
📅 日期：XXXX年XX月XX日

【今日动态】
• [动态1描述] | 来源：[来源]
• [动态2描述] | 来源：[来源]

【重点关注】
⚠️ [需要重点关注的事项及分析]

【行动建议】
→ [对用户有价值的建议]
```

#### 每周竞品报告（完整版）
```
📊 [行业] 竞品周报
📅 周期：XXXX年XX月XX日 - XX日

【本周要闻】
• [竞品1]：[3条重要动态]+[简评]
• [竞品2]：[3条重要动态]+[简评]
• [竞品3]：[3条重要动态]+[简评]

【市场格局变化】
[本周市场格局的重要变化分析]

【竞品产品/战略动向】
[竞品在产品、战略、人员方面的重大变化]

【融资与估值动态】
[如有融资/上市动态，附简析]

【舆情摘要】
[正面/负面舆情一览]

【下周预测】
[基于现有信息的预判]

【战略建议】
→ [对用户的具体行动建议]
```

### 4. 重点警报触发

**自动触发条件：**
- 竞品发布重大产品更新
- 竞品获得大额融资
- 竞品出现负面舆情（危机公关）
- 竞品高管变动
- 竞品发起价格战
- 竞品进入新市场/新赛道

**触发时自动生成：**
- 事件详情（what）
- 影响分析（so what）
- 应对建议（now what）

## 工作流程

### 单次情报查询

```
Step 1：确认竞品名单
Step 2：全网搜索竞品最新动态
Step 3：筛选重要信息（过滤噪音）
Step 4：生成情报简报
Step 5：标注重点+行动建议
```

### 持续监控模式（配合Cron）

```
每日 09:00 → 生成前24小时竞品动态简报
每周一 09:00 → 生成上周完整竞品周报
实时监控 → 发现重大动态立即推送
```

```
📦 竞品每日情报 | 2026年4月14日

【御娘食品今日动态】
• 御娘官方小红书发布新笔记《配料表干净到可怕》| 互动：1,247 | 状态：正常
• 御娘醉卤汁在Weee!平台上新评分4.8/5 | 状态：关注

• [竞品A] 宣布完成B轮融资5000万元 | 来源：36氪 → [影响分析]
```

## 使用场景

- **投资人**：投前尽调+投后监控被投公司竞争环境
- **企业高管**：实时掌握竞争对手战略动向
- **商务/BD**：了解合作伙伴及竞争对手最新动态
- **市场研究**：持续追踪行业竞争格局变化
- **创业者**：监控目标赛道竞争环境

---

## 数据来源

本skill通过以下渠道收集信息：
- 全网搜索引擎（Google/Baidu）
- 企业官网及官方社交媒体
- 行业媒体（36氪、虎嗅、亿邦动力等）
- 融资数据平台（IT桔子、天眼查）
- 招聘平台（监测竞品团队扩张信号）
- 电商平台（价格、评分、评论）

---

## 局限性说明

- 本skill基于网络公开信息，无法获取未公开的内部信息
- 信息的时效性取决于网络公开时间
- 投资建议基于公开信息分析，不构成正式投资建议
- 建议将AI情报作为人工分析的补充，不完全替代人工判断
