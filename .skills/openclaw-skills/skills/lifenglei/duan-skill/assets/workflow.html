一、启动：确认 3 个决策
决策 01
协作模式
  • Full Auto — 确认主题直接交付
  • Guided — 3 个检查点确认 默认
  • Collaborative — 逐张审阅全程掌控
决策 02
制作路径
  • Path A：HTML 可编辑 → PPTX 默认
  • Path B：全 AI 视觉图像 → PPTX
  • A 文字可继续编辑；B 视觉更震撼
决策 03
个人形象融入
  • 检查 assets/character/ 目录
  • 有 *-sheet.png → 直接复用
  • 有 original.jpg → 生成角色设定图
  • 为空 → 询问用户需求
二、Step 01 · 内容梳理
01
Step 01
将原始材料转化为逐张 Slide 大纲
每张 Slide 明确定义：标题（断言句）· 要点（≤3–4条）· 视觉类型 · 是否需要 AI 插画 「Q3 销售增长 23%，新用户是主要驱动力」✓   「Q3 销售」✗ — 标题是结论，不是主题词
Checkpoint Guided / Collaborative 模式：表格展示大纲，用户确认后进入 Step 02
三、Step 02 · 风格选择
02
Step 02
按主题类型推荐 3 种风格，主动展示样例图
用 Read 工具展示 assets/style-samples/style-NN-name.png 视觉样例（不只是文字描述）
17+ 风格完整参数（色值 / 排版 / 构图 / Prompt 规范）→ references/proven-styles-gallery.md
Checkpoint Guided / Collaborative 模式：展示 3 个风格 + 样例图，用户选定后进入 Step 03
四、Step 03 · 构建（两条路径分叉）
Path A 可编辑 HTML + 选择性插画
Path B 全 AI 视觉（每张完整图像）
  • 每张 Slide 生成独立 HTML（720 × 405 pt）
  • HTML 规范 → prompt-templates.md
  • AI 插画可选，垫图机制保持风格统一
$ uv run [SKILL_SCRIPTS_DIR]/generate_image.py --input-image style-NN.png --prompt "场景描述 in STYLE" --filename slide-NN.png --resolution 2K
  • 每张 Slide 生成完整图片（含文字与布局）
  • 短 Prompt 比长 Prompt 效果更好
  • 描述 mood + 内容，不约束颜色比例数字
$ uv run [SKILL_SCRIPTS_DIR]/generate_image.py --input-image style-NN.png --prompt "布局+内容+情绪" --filename slide-NN.png --resolution 2K
五、Step 04 · 组装 PPTX
Path A html2pptx.js
Path B create_slides.py
$ node [SKILL_SCRIPTS_DIR]/html2pptx.js slide-01.html slide-02.html ... -o output.pptx
文字可在 PPT 中继续编辑 中文渲染完美
$ uv run [SKILL_SCRIPTS_DIR]/create_slides.py slide-01.png slide-02.png ... --layout fullscreen -o output.pptx
视觉震撼，风格高度统一 文字不可后期编辑

Step 05 · 最终交付
