# 合同审计 Skill - 设计文档

## 1. 概述

### 1.1 背景
公司内部合同审核依赖人工，效率低、易遗漏。需要 AI 辅助审计工具，提升审核质量和效率。

### 1.2 目标
- 自动化合同关键条款审查
- 识别潜在风险和不利条款
- 生成标准化审计报告
- 支持批量处理

### 1.3 范围
**支持文件格式**：
- **优先**：Word（.doc/.docx）
- 后续：PDF、图片（PNG/JPG）

**v1.0 MVP 仅支持 Word 格式**

**审查维度**：
- 金额条款（一致性、付款节点、违约金）
- 交付条款（时间、标准、延期责任）
- 合规性（签章、日期、关键条款）
- 风险提示（不利条款、模糊表述、纠纷点）

**不支持**：
- 法律有效性判定
- 复杂商业条款谈判建议
- 多语言合同（暂仅支持中文）

## 2. 用户场景

### 场景 1：单合同快速审查
**用户**：销售经理准备签署新客户合同
**流程**：
1. 私聊发送合同 PDF
2. AI 自动审查
3. 返回带批注的审计报告
4. 根据建议修改合同

### 场景 2：批量合同审查
**用户**：法务部月度合同审查
**流程**：
1. 发送多个合同文件
2. AI 批量处理
3. 生成汇总报告 + 各合同详细报告

### 场景 3：特定条款检查
**用户**：项目经理关注交付条款
**流程**：
1. 发送合同 + 指令"重点检查交付条款"
2. AI 针对性审查
3. 返回交付相关风险提示

## 5. 核心流程

### 5.1 主流程
```
用户发送文件 → 保存文件 → 提取文本 → 分析合同 → 生成报告 → 返回结果
```

### 5.2 详细流程

#### 步骤 1：接收文件
- 监听私聊文件消息
- 验证文件格式（PDF/Word/图片）
- 保存到临时目录
- 返回"收到文件，开始审计..."

#### 步骤 2：文本提取
| 格式 | 工具 | 说明 |
|:---|:---|:---|
| PDF | pdfplumber | 优先提取文字层，OCR 备选 |
| Word | python-docx | 保留段落结构 |
| 图片 | pytesseract | OCR 识别 |

#### 步骤 3：合同分析
**金额条款检查**：
- 大写金额与小写金额一致性
- 付款节点是否明确
- 违约金比例是否合理
- 发票类型与税率

**交付条款检查**：
- 交付时间是否具体
- 验收标准是否可量化
- 延期责任是否对等
- 交付物清单是否完整

**合规性检查**：
- 双方签章位置
- 签署日期完整性
- 关键条款是否缺失
- 法律适用条款

**风险提示**：
- 单方解除权
- 无限连带责任
- 模糊表述（"合理"、"尽力"）
- 争议解决条款偏向性

#### 步骤 4：报告生成
**格式选项**：
1. **文本报告**：Markdown 格式，适合快速阅读
2. **批注文档**：在原文件上添加批注（PDF 注释）
3. **汇总表格**：Excel/CSV，适合批量对比

### 5.3 异常处理
| 异常 | 处理 |
|:---|:---|
| 文件无法读取 | 提示用户检查文件完整性 |
| 文本提取失败 | 尝试 OCR，仍失败则人工介入 |
| 合同类型不匹配 | 提示当前仅支持服务/采购类合同 |
| 分析超时 | 分段处理，返回部分结果 |

## 6. 技术实现

### 6.1 依赖库

## 7. 安全与隐私

### 7.1 数据安全
- 合同文件仅临时存储，审计完成后删除
- 敏感信息（金额、公司名）在日志中脱敏
- 不支持云存储，本地处理

### 7.2 权限控制
- 仅响应私聊消息，不处理群聊
- 管理员可配置白名单
