# 技能懒加载与按需启动

> 轻量化Agent，按需启动

## 启动策略

### 常驻模块（始终加载）
- 核心交互模块
- 常用工具（计算器、翻译）
- 当前用户上下文

### 按需加载（触发时加载）
| 触发意图 | 加载模块 |
|----------|----------|
| 画图 | 绘图Agent |
| 查天气 | 天气模块 |
| 写代码 | 编程Agent |
| 交易 | 金融Agent |

## 技能优先级队列

| 优先级 | 类型 | 示例 | 内存状态 |
|--------|------|------|----------|
| P0 | 高频轻量 | 计算器、翻译 | 常驻 |
| P1 | 中频中量 | 天气、搜索 | 预加载 |
| P2 | 低频重量 | 分析、绘图 | 按需 |

## 懒加载流程

```
用户输入
   ↓
意图识别
   ↓
判断需要的技能
   ↓
检查是否已加载
   ├─ 是 → 直接使用
   └─ 否 → 动态加载
         ↓
       使用
         ↓
       用完释放（如需）
```

## 资源管理

- 内存占用过高 → 释放低优先级技能
- 用户切换场景 → 平滑切换技能
- 异常处理 → 加载失败降级
