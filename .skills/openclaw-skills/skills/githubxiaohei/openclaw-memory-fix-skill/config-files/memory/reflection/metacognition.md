> 反思思考过程，优化认知架构

## 核心思想

让系统不仅解决问题，还能反思自己的思考过程，从而不断优化自身的认知架构。

## 自我调参

### 动态调整参数

| 参数 | 调整条件 | 调整方式 |
|------|----------|----------|
| 记忆检索阈值 | 频繁未命中 | 降低阈值 |
| 工具调用超时 | 经常超时 | 增加超时时间 |
| 置信度标准 | 经常被纠正 | 提高置信度要求 |

### 调参流程

```
定期分析思维日志
    ↓
找出低效/错误模式
    ↓
制定调整策略
    ↓
执行调参
    ↓
验证效果
    ↓
固化最佳参数
```

## 反思触发

### 自动触发
- 任务完成后
- 遇到错误时
- 用户反馈负面时

### 手动触发
- 定期回顾（每天/每周）

## 反思格式

```markdown
### 反思: [任务名]

**思考过程**:
1. 步骤1 → 结果
2. 步骤2 → 结果

**问题分析**:
- 哪里可以优化？
- 有什么错误？

**改进策略**:
- 短期改进
- 长期改进

**效果验证**:
- 下次遇到类似任务怎么改
```

## 高层次策略抽象

从多次反思中抽象出通用策略：

**示例策略**：
- "遇到开放式问题，先进行意图澄清再执行"
- "复杂任务先拆解再执行"
- "不确定时先确认再行动"
