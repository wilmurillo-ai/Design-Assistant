## 核心功能

### 层间迁移
| 方向 | 触发条件 |
|------|---------|
| L1 → L2 | 任务完成、用户确认 |
| L2 → L3 | 相似情景3次+ |
| L3 → L4 | 知识验证10次+ |

### 动态衰减
- L4核心原则：永久保留
- L4画像：1%/月
- L3知识：5%/月
- L2情景：10%/月
- L1临时：50%/会话

### 置信度反馈
- 高(>80%) → 提升层级
- 中(30-80%) → 保持
- 低(<30%) → 降级或遗忘

## 记忆的持久化与备份（2026-03-08 新增）

### 定期快照

| 层级 | 频率 | 存储位置 |
|------|------|---------|
| L4 | 每日 | 本地 + 云端 |
| L3 | 每周 | 本地 + 云端 |
| L2 | 每月 | 本地 |

### 快照格式

```
memory_backups/
├── 2026-03-08/
│   ├── L4/
│   │   ├── MEMORY.md
│   │   ├── life/
│   │   └── tacit_knowledge.md
│   └── L3/
│       ├── knowledge/
│       └── lessons/
└── index.json    # 快照索引
```

### 自动备份流程

```
每日 02:00：
1. 导出L4内容
2. 加密压缩
3. 推送到云端（iCloud/Drive）
4. 保留最近30天快照

每周日：
1. 导出L3内容
2. 增量备份
3. 保留最近90天
```

### 恢复机制

```bash
# 查看可用快照
memory restore --list

# 恢复到指定快照
memory restore 2026-03-08

# 选择性恢复（仅L4）
memory restore 2026-03-08 --layers L4
```

## 加密与隐私

### 敏感信息分类

| 类型 | 示例 | 加密方式 |
|------|------|---------|
| 高度敏感 | 密码、API Key、身份证号 | AES-256 |
| 中度敏感 | 用户画像、偏好 | 可选加密 |
| 低度敏感 | 项目记录、知识库 | 不加密 |

### 加密流程

```
写入敏感信息时：
1. 检测敏感字段
2. 获取用户加密密钥（或自动生成）
3. AES-256加密
4. 存储密文
5. 密钥存储在Keychain

读取时：
1. 读取密文
2. 从Keychain获取密钥
3. 解密后使用
```

### 手动清理接口

```bash
# 查看敏感数据
memory privacy --scan

# 手动删除指定记忆
memory delete --id <id>

# 一键清理所有敏感信息
memory privacy --wipe

# 导出解密数据（供用户备份）
memory export --decrypted
```

### 隐私设置

```yaml
# 配置文件
privacy:
  encryption: auto    # auto/manual/off
  sensitive_fields: [password, api_key, id_card, phone]
  auto_wipe_after_days: 365
  key_storage: keychain
```

## 数据安全策略

### 多重备份

```
本地 → iCloud Drive → 外部硬盘
  ↓        ↓            ↓
 L4/L3    每日增量    每周全量
```

### 灾难恢复

| 场景 | 恢复方案 |
|------|---------|
| 本地数据损坏 | 从云端恢复 |
| 云端丢失 | 从外部硬盘恢复 |
| 全部丢失 | 从历史快照恢复 |

### 数据完整性校验

```
每次备份：
1. 计算SHA-256哈希
2. 存储哈希值
3. 恢复时验证完整性
```
