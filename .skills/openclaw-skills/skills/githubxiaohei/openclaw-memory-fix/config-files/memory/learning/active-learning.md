# 主动学习与知识缺口

> 主动发现知识空白，寻求填补

## 核心思想

系统不再被动等待指令，而是主动发现知识空白，并寻求填补。

## 不确定性检测

### 置信度标记

每次回答后评估置信度：

| 置信度 | 标记 | 行动 |
|--------|------|------|
| 90-100% | 🟢 确定 | 正常回答 |
| 70-89% | 🟡 较确定 | 回答但说明依据 |
| 50-69% | 🟠 不确定 | 提供备选方案 |
| <50% | 🔴 未知 | 标记"待探索" |

### 待探索标记

```json
{
  "id": "explore-001",
  "问题": "用户提到的XXX是什么",
  "当前理解": "不确定",
  "需要": "用户解释/搜索确认",
  "状态": "pending",
  "创建时间": "2026-03-07"
}
```

## 主动提问

### 提问场景

| 场景 | 示例 | 时机 |
|------|------|------|
| 意图不清晰 | "您是想问A还是B？" | 用户输入模糊 |
| 信息不足 | "需要更多细节" | 关键信息缺失 |
| 知识模糊 | "我不太确定，查一下？" | 置信度<50% |

### 提问策略

```
发现知识缺口
    ↓
判断严重程度
    ├─ 高 → 立即提问
    ├─ 中 → 完成后提问
    └─ 低 → 记录待探索
    ↓
选择合适时机提问
```

## 主动探索

### 自动搜索

当发现知识缺口时：

```
缺口: "用户提到的新技术"
    ↓
判断: 可以搜索？
    ↓
执行: 搜索相关资料
    ↓
更新: 存入记忆
    ↓
反馈: 告诉用户"我查了一下..."
```

### 用户空闲时

检测用户空闲（无操作>5分钟）：

```
用户空闲
    ↓
检查待探索列表
    ↓
有未填补的缺口？
    ├─ 是 → "关于刚才那个问题..."
    └─ 否 → 不打扰
```

## 知识进化

### 学习流程

```
1. 用户交互 → 发现缺口
2. 标记"待探索"
3. 主动搜索/提问
4. 获取答案
5. 更新记忆
6. 下次直接用
```

### 专家养成

```
长期缺口填补
    ↓
同一领域多次填补
    ↓
该领域知识权重增加
    ↓
成为"半专家"
    ↓
主动提供该领域建议
```
