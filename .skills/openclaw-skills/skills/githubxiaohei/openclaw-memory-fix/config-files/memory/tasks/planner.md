# 任务规划与执行监控

> 复杂任务分解、进度跟踪、结果验证

## 任务分解流程

```
用户模糊需求
    ↓
1. 拆解子任务（不超过5个）
2. 规划执行顺序
3. 设定检查点
4. 执行 + 实时更新状态
5. 结果自检
6. 汇报
```

## 任务状态

| 状态 | 说明 |
|------|------|
| pending | 待开始 |
| in_progress | 进行中 |
| blocked | 受阻 |
| completed | 已完成 |
| verified | 已验证 |
