# 文件写入规则

## description.md

背景信息、核心问题/目标简要描述、起因/动机。200-500 字。

追加模式下用 `---` 分隔并标注来源和日期。

## process.md

信息密度最大的文件。description 的扩展和细化，按时间线或逻辑顺序组织完整记录。

- 代码片段写入 code.md 而非此处
- 此处留索引引用：`> 相关代码见 [code.md](code.md) 中的「代码块名称」部分`
- 追加模式下用 `---` 分隔

## meta.md

独立、可封装的事实性知识。每条一行 `- ` 开头，1-2 句话。

```markdown
## 元信息

- xxx 接口的鉴权 key 是 private-00912ksi
- 项目部署服务器地址为 192.168.1.100:8080
```

追加模式下在末尾追加并去重。

## code.md

所有代码片段、脚本、配置文件内容。每个代码块标注语言和标题。保留原始代码不修改。

追加模式下在末尾追加新代码块。

## resource/

将用户提及的本地文件复制到此目录。

在 process.md 对应位置添加引用：`> 参考文件见 [resource/文件名](resource/文件名)`
