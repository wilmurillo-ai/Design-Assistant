# v0.7

## Summary

This version changes the repository from a dual-output character package generator into a canonical-source system with platform-specific runtime wrappers.

## Key Changes

- `characters/{slug}/` becomes the canonical static source for each generated character
- Codex is the default runtime install target at `./.agents/skills/{slug}/`
- OpenClaw export becomes optional and requires an explicit workspace path
- `canon.md`, `persona.md`, `style_examples.md`, and `meta.json` stay identical across Codex and OpenClaw runtime packages
- `SKILL.md` may differ by platform
- runtime exports copy necessary memory scripts but do not copy `.dreamlover-data/`

## Metadata Additions

- `canonical_source`
- `export_targets`
- `generated_for`
- `openclaw_exported_at`

## Runtime Rule

Always regenerate or re-export from the canonical source instead of hand-editing exported runtime packages.
