# V0.2

This contract revision promotes generated child skills into first-class Codex-installable packages.

## Scope

- generated child skills install to `./.agents/skills/{slug}/` by default
- `characters/{slug}/` remains available as an archive mirror
- child `SKILL.md` files require YAML front matter with `name` and `description`
- `tools/skill_writer.py` preserves existing layer content when updating packages
- `tools/skill_writer.py` runs a post-write lint pass by default
- `tools/skill_linter.py` validates package completeness, section boundaries, and placeholder leftovers

## Stability Promise

Until the next contract revision:

- the child skill runtime location remains `./.agents/skills/{slug}/` by default
- the required package files remain `SKILL.md`, `canon.md`, `persona.md`, `style_examples.md`, and `meta.json`
- the documented section headings for `canon`, `persona`, and `style_examples` remain stable
- linting remains part of the default writer workflow unless explicitly skipped
