# V0.3

This contract revision adds intake-first generation to the character skill workflow.

## Scope

- new character requests must gather the minimum intake bundle before generation
- `tools/skill_writer.py --interactive` prompts for intake information and writes the results into package outputs
- generated packages preserve target use, source types, and low-confidence persona policy in metadata
- child skills remain installed under `./.agents/skills/{slug}/` by default and can still mirror to `characters/{slug}/`

## Stability Promise

Until the next contract revision:

- the minimum intake fields remain character name, source work, target use, source types, and low-confidence persona policy
- `--interactive` remains the preferred CLI path when the user starts from an underspecified request
- package layout remains compatible with the v0.2 Codex-discoverable child skill format
