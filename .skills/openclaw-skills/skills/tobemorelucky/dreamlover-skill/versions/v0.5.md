# Version 0.5

This version adds a conditional memory system for generated child skills.

## New Runtime Pieces

- `scripts/memory_router.py`
- `scripts/memory_fetch.py`
- `scripts/memory_commit.py`
- `scripts/memory_summarize.py`
- `references/memory_policy.md`

## Contract Notes

- child skills must not read memory by default
- child skills must not write memory by default
- each turn must route memory decisions through `memory_router.py`
- local runtime memory must stay outside the skill package under `./.dreamlover-data/`
- child skills must not invent prior chat history when no relevant memory exists
