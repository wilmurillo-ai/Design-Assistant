# V0.4

This contract revision turns intake into a hard pre-write gate.

## Scope

- new character generation must collect and confirm source decision policy before any writes
- new character generation must collect and confirm input mode before any writes
- the intake summary must be repeated back to the user for confirmation
- file creation and modification are forbidden until the hard intake gate completes

## Stability Promise

Until the next contract revision:

- the hard intake gate remains mandatory for underspecified generation requests
- source decision policy and input mode remain part of the minimum intake bundle
- confirmation remains required before writing any character files
