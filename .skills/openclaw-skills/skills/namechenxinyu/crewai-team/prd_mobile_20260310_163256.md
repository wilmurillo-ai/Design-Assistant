# OpenClaw 移动端一页纸 PRD

**版本**：V1.0 (Mobile First)  
**owner**：移动端产品经理  
**原则**：手指友好，极简操作，砍掉桌面端思维  

---

### 1. 产品定位
**手机就是遥控器**：走到机器面前秒连，单手盲操抓取，成功后一键分享，彻底抛弃电脑配置。

### 2. 核心功能 (P0)
*只保留手机上最实用的 5 个功能，其余全部砍掉*
1. **蓝牙/NFC 秒连**：靠近机器触碰或点击即可连接，无需手动配置 IP/端口。
2. **单手模式操控**：屏幕下半区显示超大方向键与抓取键，支持站立单手盲操。
3. **纯净视角监控**：全屏显示摄像头画面，隐藏所有非必要参数，远程随时查看。
4. **故障远程复位**：接收异常推送后，支持一键远程重启恢复，无需现场处理。
5. **成果一键分享**：抓取成功后自动生成结果卡片，直接发送至社交软件。
*(注：明确不做代码编辑、日志查看、多窗口监控、手动网络配置)*

### 3. 技术方案
*能跨平台就别原生，一套代码搞定*
*   **跨平台框架**：**Flutter**。利用 Skia 引擎保证操控界面 60fps 流畅度，蓝牙/NFC 插件成熟，避免双倍测试成本。
*   **核心页面 (共 5 页)**：
    1.  **设备连接页**：自动扫描 + NFC 感应区，无设置页。
    2.  **核心操控页**：视频流 + 下半区大按钮，零键盘输入。
    3.  **纯净监控页**：横屏全屏视频，仅保留退出键。
    4.  **故障复位页**：故障描述 + 一键复位大按钮。
    5.  **成果分享页**：结果卡片 + 系统原生分享面板。
*   **数据与网络**：本地 **SQLite** 缓存指令队列，弱网环境下支持指令静默重发；API 超时设为 5s，失败自动重试 2 次。

### 4. 开发计划
*   **团队配置**：2 名 Flutter 高级开发（兼管 UI 与逻辑）。
*   **时间周期**：**3 周** (15 人天)。
    *   第 1 周：框架搭建、硬件插件调试（蓝牙/NFC/视频）。
    *   第 2 周：5 个核心页面 UI 与逻辑、API 联调。
    *   第 3 周：弱网重试机制、真机测试与打包发布。
*   **里程碑**：第 3 周五完成首版本迭代上线。

### 5. 发布标准
*   **主流程无 Bug**：连接->操控->抓取->分享 闭环通过率 100%。
*   **核心体验流畅**：操控指令延迟 <200ms，视频流首帧加载 <2s。
*   **手指友好**：所有可点击元素最小 44x44pt，核心按钮高度不低于 60pt。
*   **弱网可用**：无网状态下可查看本地历史记录，网络恢复后指令自动同步。

---
**备注**：本 PRD 严格遵循“这个功能在手机上真的用得上吗？”的审视原则。任何增加用户思考成本的功能（如手动配网、查看日志）均不予实现。移动端只负责“连得快、控得稳、发得爽”，复杂逻辑留给服务端。
