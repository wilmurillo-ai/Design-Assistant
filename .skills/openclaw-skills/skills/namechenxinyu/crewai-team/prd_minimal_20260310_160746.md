# 云端爬虫对接平台 - 一页纸 PRD v1.0

**原则：** 能跑就行，拒绝过度设计。  
**PM 口头禅：** 这个功能真的必要吗？

## 1. 产品定位
让非开发人员无需本地环境，通过可视化配置即可获取结构化数据并自动回调的云端工具。

## 2. 核心功能 (P0 Only)
1. **云端一键运行**：浏览器配置，云端执行，彻底解决本地环境依赖。
2. **可视化规则生成**：点选网页元素自动生成规则，零代码门槛。
3. **标准 Webhook 回调**：任务完成自动推送 JSON 至用户接口，无缝集成。
4. **预设主流网站模板**：内置常见站点配置，开箱即用。
5. **失败即时通知**：仅异常时发送邮件，不做复杂监控大盘。

## 3. 技术方案 (极简)
*   **技术栈**：Python 3.9 + Flask (后端) | Vue 3 + Element Plus (前端) | MySQL 8.0 (数据库) | Docker (部署)
*   **架构**：单体架构，后台线程池调度，无消息队列，无状态设计。
*   **核心表 (5 张)**：
    *   `users`: 用户配置 (Webhook/Email/Password)
    *   `tasks`: 任务定义 (URL/Status/Cron)
    *   `task_rules`: 抓取规则 (Selector/Field/Type)
    *   `templates`: 预设模板 (Site/Rules_JSON)
    *   `task_logs`: 运行日志 (Level/Message)

## 4. 开发计划 (2 周 2 人)
*   **人力**：1 后端 + 1 前端 (共 10 人天)
*   **Week 1**：后端骨架、抓取引擎 (Playwright)、数据库设计、核心 API。
*   **Week 2**：前端页面、可视化点选逻辑、联调测试、Docker 封装。
*   **上线**：第 3 周周一。

## 5. 发布标准 (Main Flow)
*   **核心用例 Pass**：登录 -> 创建任务 -> 配置规则 -> 手动运行 -> 收到 Webhook 回调。
*   **稳定性**：连续运行 24 小时容器不崩溃，内存无明显泄漏。
*   **Bug 标准**：无致命/严重 Bug，UI 瑕疵及边缘情况忽略。

## 6. 明确不做 (Cut List)
*   **不做**用户权限与团队管理 (初期单用户)。
*   **不做**内置数据库与数据清洗 (数据交付后由用户处理)。
*   **不做**数据可视化大屏 (这是 BI 工具的事)。
*   **不做**复杂代理池 (初期依赖用户自配或公开数据)。

*注：任何新增需求请先问自己“这个功能真的必要吗？”*
