# OpenClaw 移动端 PRD (MVP 版)
**版本号：** V1.0 | **状态：** 待评审 | **原则：** 手指友好，极简操作

## 1. 产品定位
让用户站在机器前，单手秒连秒玩，砍掉所有桌面配置思维，只保留“玩”的核心体验。

## 2. 核心功能 (P0, 不超过 5 个)
1. **NFC 一碰连接**：[场景：站立碎片化] 手机背面触碰机器即可连接，无需搜索 WiFi 或输入 IP。
2. **全屏沉浸式视图**：[场景：小屏看清爪子] 游戏时隐藏所有菜单，视频流占满屏幕。
3. **底部拇指操控区**：[场景：单手握持交互] 方向键和抓取键固定在屏幕下半区，支持拇指盲操。
4. **生物识别支付**：[场景：即时情绪分享] FaceID/指纹一键充值，避免输入繁琐导致流失。
5. **自动录制分享**：[场景：抓取成功瞬间] 游戏结束自动生成 15 秒高光视频，一键发送社交软件。

## 3. 技术方案
*   **跨平台框架：** **Flutter**。理由：高性能渲染视频流叠加 UI，一套代码搞定 iOS/Android，NFC 与生物识别插件一致。
*   **数据存储：** **SQLite**。理由：弱网环境下指令队列需要事务 integrity，优于 AsyncStorage。
*   **核心页面 (共 5 页)：**
    1.  **NFC 连接页**：监听标签，自动握手。
    2.  **游戏操控页**：全屏视频 + 底部拇指区。
    3.  **生物支付页**：仅显示金额与确认按钮。
    4.  **结果分享页**：播放缓存视频，唤起分享。
    5.  **个人中心页**：仅显示余额与列表，无图表。
*   **弱网策略：** 控制指令本地队列重试，视频流降级保控制。

## 4. 开发计划
*   **周期：** 3 周 (15 人天)。
*   **人员：** 1 名资深 Flutter 开发 (全栈移动端)。
*   **排期：**
    *   第 1 周：框架搭建 + 硬件插件集成 (NFC/生物识别)。
    *   第 2 周：核心页面开发 + 视频流对接。
    *   第 3 周：弱网队列逻辑 + 联调测试。
*   **备注：** 后端接口并行开发，移动端不阻塞。

## 5. 发布标准
1.  **主流程无 Bug：** NFC 连接 -> 支付 -> 操控 -> 分享，全流程打通无崩溃。
2.  **核心体验流畅：** 操控指令延迟 < 200ms，视频流卡顿不影响指令发送。
3.  **弱网可用：** 断网重连后，本地队列指令自动补发成功。
4.  **UI 达标：** 所有按钮点击区 >= 44x44pt，核心操作位于拇指热区。

## 6. 明确不做 (砍掉桌面端思维)
*   **不做** 爪子力度/电压校准 (只保留玩)。
*   **不做** 多机器同时管理 (一次只连一台)。
*   **不做** 复杂数据统计报表 (只关心还剩几次)。
*   **不做** 鼠标悬停交互 (触摸无悬停)。
*   **不做** 横向桌面式菜单 (拒绝顶部复杂导航)。

**PM 备注：** 这个功能在手机上真的用得上吗？如果不能单手操作，砍掉。如果不能秒连，砍掉。
