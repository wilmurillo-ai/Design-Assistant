# OpenClaw 对接简化工具 - 一页纸 PRD
**版本：** MVP 1.0 | **原则：** 少即是多，能跑通就行  
**PM 口头禅：** 这个功能真的必要吗？

## 1. 产品定位
让开发者在 5 分钟内完成 OpenClaw 接口连通性验证与 SDK 代码生成，消除配置与调试成本。

## 2. 核心功能（P0，仅保留必须）
1. **极简配置向导**：引导填入密钥并自动验证连通性，消除环境配置错误。
2. **可视化接口调试**：无需写代码即可发送请求并查看原始响应，快速定位问题。
3. **实时错误日志**：清晰展示接口返回的错误码与含义，避免盲目排查。
4. **一键代码生成**：根据调试结果生成可直接使用的 SDK 代码，缩短开发落地时间。
5. **本地数据持久化**：配置存浏览器 LocalStorage，不上传云端，保障安全且免登录。

## 3. 技术方案
**原则：** 能单体就不微服务，能单机就不集群。
*   **技术栈：** 前端 Vue3 + Element Plus | 后端 Python Flask | 数据库 SQLite | 部署 Docker 单体。
*   **核心表（仅 2 张）：**
    1. `sys_error_log`：存系统运行错误，不存用户隐私。
    2. `api_template`：存预置接口模板，减少手动输入。
*   **核心 API：** 验证 (/verify)、代理 (/proxy)、模板 (/templates)、生成 (/generate)、心跳 (/health)。

## 4. 开发计划
**原则：** 预留缓冲，但不注水。
*   **周期：** 1 周（5 人天）。
*   **人力：** 1 前端 + 1 后端（或 1 全栈）。
*   **排期：** 前端页面 2 天 | 后端逻辑 2 天 | 联调部署 1 天。

## 5. 发布标准
**原则：** 主流程能跑通就行，不追求完美。
*   **准入：** 10 项核心测试用例全部通过（含健康检查、密钥验证、代理转发、代码生成）。
*   **质量：** P0 级致命 Bug 为 0，P1 级严重 Bug 为 0。
*   **运维：** 容器重启后服务自动恢复，数据不丢失。

## 6. 明确不做（Out of Scope）
*   **不做**用户登录与云端同步（隐私风险高）。
*   **不做**数据分析看板（偏离核心）。
*   **不做**复杂工作流自动化（那是 iPaaS 的事）。
*   **不做**社区与交流功能（增加运营成本）。

**结论：** 方案已最简，立即开工。
