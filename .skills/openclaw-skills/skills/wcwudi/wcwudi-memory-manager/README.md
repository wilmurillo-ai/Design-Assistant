# memory-manager 技能 README.md 文档

### 技能概述
**memory-manager** 是一个专门用于管理 OpenClaw 记忆系统的技能，提供双向可控的记忆共享、归档和清理功能。

### 核心功能
1. **记忆共享管理** - 支持主代理与子代理之间的双向记忆交换
2. **手动归档系统** - 按用户指令进行记忆归档操作
3. **安全清理机制** - 提供多级确认的安全删除流程
4. **操作记录追踪** - 完整记录所有记忆操作的历史

### 快速开始

#### 1. 系统检查
```powershell

.\memory_check.ps1
```

#### 2. 代理检查
```powershell
# 检查代理配置和访问权限
.\agent_check.ps1
```

#### 3. 基础归档操作
```powershell
# 归档单个文件（需要用户确认）
.\archive_tool.ps1 -Action Archive -FilePath "path/to/file.md"
```

### 使用场景

#### 场景1：手动记忆清理
- 收到用户明确指令后执行清理
- 分类处理：保留/归档/删除
- 生成清理报告供用户确认

#### 场景2：项目记忆归档
- 用户指令下归档项目相关记忆
- 保留关键决策记录
- 清理临时讨论文件

#### 场景3：多代理协作
- 主代理统一管理所有记忆
- 子代理按需访问共享记忆
- 安全隔离不同代理空间

### 安全特性

#### 1. 手动确认机制
- **不自动执行**：所有操作必须收到用户明确指令
- **禁止假设确认**：不接受任何基于上下文的推断
- **明确文字回复**：只接受"是"、"允许"、"确认"、"好的"等明确回复

#### 2. 操作记录
- 所有操作记录到 `logs/operations/`
- 交换记录保存到 `logs/exchanges/`
- 可追溯的历史审计

#### 3. 安全边界
- 严格遵守工作空间限制
- 不越权访问其他代理空间
- 删除操作先移动到暂存区等待最终确认

### 维护建议

#### 手动管理原则
1. **不自动执行**：所有清理操作必须收到明确指令
2. **按需处理**：只在需要时进行记忆管理
3. **用户确认**：每次操作前必须获得用户明确授权

#### 操作流程
1. 用户发出清理指令
2. 系统检查当前状态并报告
3. 用户确认具体操作方案
4. 执行确认的操作

### 故障排除

#### 常见问题
1. **权限问题**：检查代理配置文件的访问权限
2. **路径错误**：确认工作空间路径正确
3. **脚本执行失败**：检查 PowerShell 执行策略
