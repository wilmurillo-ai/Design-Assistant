# ⏱️ 柳比歇夫时间记录法 - WorkBuddy Skill

基于柳比歇夫时间管理法的个人时间与精力追踪工具，通过飞书多维表格存储数据，支持自然语言记录与自动日报生成。

## 核心理念

- **时间管理的尽头是精力管理**：按"能量流向"分类胜过按"物理场景"分类
- **粗糙的真实，胜过虚假的精确**：不追求精确到分钟，追求持续记录
- **省 token 设计**：统计计算、全文拼接全由脚本完成，大模型只负责写2-4句能量洞察

## 6大分类

| 类别 | 描述 |
|------|------|
| 🛏️ 睡觉 | 夜睡、午休等底层基石 |
| 🏠 日常 | 做饭、通勤、家务等维持生活运转的基础动作 |
| 📖 输入 | 阅读、教程、听书等"能量向内"的自我增长行为 |
| ✍️ 输出 | 写文案、社群答疑、剪视频等"能量向外"的交付与资产构建行为 |
| 💕 关系 | 陪家人、老友深聊等"情感滋养与人际资产投资"行为 |
| 🔋 回血 | 运动、刷手机、发呆等"能量恢复"行为 |

## 功能

- **自然语言记录**：用日常语言描述活动，大模型自动分类+打标签
- **附带活动追踪**：记录并行发生的事（如边做饭边听书），计算"增量复利"
- **自动日报**：脚本从飞书读取记录→计算统计→生成可视化日报→写入飞书
- **Top3核心资产**：自动计算时间投入最多的3个标签（排除睡觉）
- **文本进度条**：日报中6大分类带有可视化进度条

## 安装与配置

### 🚀 快速安装（3步搞定）

#### Step 1：安装 Skill

```bash
# 方式1：从 ClawHub 安装（推荐）
# 在 WorkBuddy 中搜索 "liubischev-time-tracker" 一键安装

# 方式2：从 GitHub 安装
git clone https://github.com/ylstf/liubischev-time-tracker.git ~/.workbuddy/skills/liubischev-time-tracker
```

#### Step 2：安装依赖并登录飞书

```bash
# 安装 lark-cli（飞书命令行工具）
pip install lark-cli

# 登录飞书（按提示完成授权）
lark-cli auth login
```

#### Step 3：一键初始化

```bash
python3 scripts/setup.py
```

脚本会自动完成：
- ✅ 检查 lark-cli 和登录状态
- ✅ 创建飞书多维表格
- ✅ 创建记录表（13个字段，含单选项配置）
- ✅ 创建日报表（24个字段）
- ✅ 输出 Base Token 和 Table ID

初始化完成后保存输出的配置信息即可。

---

### 🔧 手动安装（备选）

如果自动初始化不成功，可以手动操作：

1. 打开飞书，新建一个**多维表格**
2. 在表格 URL 中找到 **Base Token**（格式如 `https://xxx.feishu.cn/base/BASE_TOKEN_HERE`）
3. 在这个多维表格中创建**两个数据表**（sheet），字段结构见下方

#### 📋 记录表（13个字段）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 记录日期 | 日期 | 记录所属日期 |
| 开始时间 | 文本 | 格式 HH:MM |
| 结束时间 | 文本 | 格式 HH:MM |
| 时长（分钟） | 数字 | AI写入时自动计算，无需手动填写 |
| 主活动描述 | 文本 | 活动内容 |
| 主类别 | 单选 | 选项：睡觉、日常、输入、输出、关系、回血 |
| 主Tag | 文本 | 如 #文案创作 |
| 附带活动描述 | 文本 | 并行活动 |
| 附带类别 | 单选 | 选项同主类别 |
| 附带Tag | 文本 | 并行活动标签 |
| 附带时长（分钟） | 数字 | 附带活动时长，直接填分钟数（如10） |
| 能量状态 | 单选 | 选项：高能、平稳、消耗 |
| 记录人 | 人员 | 记录人（便于筛选） |

> 💡 「时长（分钟）」由 AI 写入记录时自动计算，无需手动填写。

> 💡 「记录人」是人员类型字段，多人使用时可筛选统计。

#### 📊 日报表（24个字段）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 日期 | 日期 | 日报日期 |
| 追踪总时长 | 数字 | 当日追踪总时长（小时） |
| 黑洞时长 | 数字 | 24h - 追踪时长 |
| 黑洞占比 | 数字 | 黑洞/24h × 100 |
| 睡觉时长 | 数字 | 小时 |
| 睡觉占比 | 数字 | 百分比 |
| 日常时长 | 数字 | 小时 |
| 日常占比 | 数字 | 百分比 |
| 输入时长 | 数字 | 小时 |
| 输入占比 | 数字 | 百分比 |
| 输出时长 | 数字 | 小时 |
| 输出占比 | 数字 | 百分比 |
| 关系时长 | 数字 | 小时 |
| 关系占比 | 数字 | 百分比 |
| 回血时长 | 数字 | 小时 |
| 回血占比 | 数字 | 百分比 |
| Top1资产 | 文本 | 排除睡觉的 Tag #1 |
| Top2资产 | 文本 | Tag #2 |
| Top3资产 | 文本 | Tag #3 |
| 增量复利 | 数字 | 并行活动额外时长（小时） |
| 增量复利明细 | 文本 | 各并行 Tag 明细 |
| 能量洞察 | 文本 | AI 生成的洞察 |
| 日报全文 | 文本 | 完整日报文本 |
| 记录人 | 人员 | 日报所属记录人 |

4. 创建完成后，获取 Table ID：

```bash
lark-cli base +field-list --base-token YOUR_BASE_TOKEN
```

5. 获取你的飞书 open_id：

```bash
lark-cli contact +search-user --query 你的姓名
```

### Step 5：配置 WorkBuddy 上下文

在 SKILL.md 的「每次调用前平台需注入的5个字段」中，需要配置以下注入上下文：

| 字段 | 说明 | 示例 |
|------|------|------|
| 当前时间 | HH:MM 格式 | `09:35` |
| 当前日期 | YYYY-MM-DD 格式 | `2026-04-20` |
| 星期几 | 周一到周日 | `周一` |
| 上条记录结束时间 | HH:MM，无记录时填【无上条记录】 | `09:10` |
| 现有Tag列表 | #tag 逗号分隔，无时填【暂无Tag】 | `#社群运营, #文案创作` |

### Step 6：设置每日自动日报

在 WorkBuddy 中创建一个自动化任务：

- **名称**：柳比歇夫日报
- **触发时间**：每天 03:00
- **执行指令**：`执行柳比歇夫日报生成，日期为昨天`
- **工作目录**：`~/.workbuddy/skills/liubischev-time-tracker`

> AI Agent 会自动调用 `python3 scripts/daily_report.py` 并传入正确的参数。

### 运行日报脚本（手动）

```bash
python3 scripts/daily_report.py --date 2026-04-21 \
  --base-token YOUR_BASE_TOKEN \
  --records-table YOUR_RECORDS_TABLE_ID \
  --daily-table YOUR_DAILY_TABLE_ID \
  --recorder YOUR_NAME \
  --recorder-id YOUR_OPEN_ID
```

可选参数：
- `--insight "能量洞察文本"` — 补填大模型生成的能量洞察
- `--dry-run` — 只打印结果不写入飞书

## 使用方式

安装后在 WorkBuddy 中直接用自然语言：

- "刚才写了一个小时的文案" → 自动记录为 [输出] #文案创作
- "8点到9点在带孩子" → 自动记录为 [关系] #亲子陪伴
- "日报" → 自动生成当日日报

## 技术架构

```
用户自然语言 → 大模型分类(JSON) → 飞书多维表格(记录表)
                                          ↓
每日03:00定时/手动触发 → Python脚本 → 读取+计算+写入 → 飞书多维表格(日报表)
                                          ↓
                                    大模型补填能量洞察(2-4句)
```
