# 四种 Work-Oriented 方法总览与选型

## 为什么把它们放在一起？

它们都属于 **work-oriented**：关注"**做什么 / 怎么做 / 用什么工具 / 花多少时间**"，而非"做工作需要什么样的人"。课本提醒：放在同一章更多是编辑方便，四者之间可以自由组合（e.g. 在 task inventory 里嵌 critical incident 式问题，在 FJA 项目里抓 time study 数据）。

## 四方法并列表（课本第 2 章开篇）

| 方法 | 诞生/代表 | 主要输出 | 典型用途 | 关键限制 |
|---|---|---|---|---|
| **Time-and-Motion Study** | Frederick Taylor；Frank & Lillian Gilbreth | 标准用时、动作序列、therblig 串 | 效率提升、培训、推断能力需求、机器/机器人设计 | 多用于制造 / 物流 / 建筑；社会/认知工作难测；引"把人当机器"道德质疑 |
| **Functional Job Analysis (FJA)** | U.S. DOL (1972)《Handbook for Analyzing Jobs》；Fine & Cronshaw (1999) | 任务句 + data/people/things 功能等级；DOT 曾用它编出 54,000 个岗位 | 全国岗位分类（DOT）、岗位匹配、生涯咨询 | DOT 已停更 (1991)，被 O\*NET 取代；things 评级信度较差 |
| **Task Inventory** | USAF / CODAP（1950s–60s）；AT&T / WPSS（1970s） | 100+ 条 task statement 的大问卷，加上 importance / time / difficulty / frequency 评分 | 量化描述岗位、培训需求、绩效评估、法庭可用的证据 | 需要大样本；信度 ≈ .77（scales）、.61（general work activities） |
| **Critical Incident Technique (CIT)** | Flanagan (1954)，USAAF 二战项目起步 | 一系列 *context + behavior + consequence* 三段式行为事例 | 绩效评估表（BOS）、选拔、培训需求 | SME 容易说"showed good initiative"而不给具体行为；建议改口称 *behavioral examples* |

## 按"用户的目的"反查方法（基于 Table 2.10 精神扩展）

课本 Table 2.10 具体讨论的是 *task inventory 里该问哪些问题*。但它传达的思想可以推广到选方法：

| Purpose（用途） | 首选方法 | 次选 | 备注 |
|---|---|---|---|
| 改善一线生产效率 | Time-and-motion | — | 只有本方法直接量时 |
| 写岗位说明书 / 分类 | FJA（DOL 版） / O\*NET 查询 | Task inventory | DOT 已停更，改用 O\*NET |
| 生涯咨询 / 求职匹配 | FJA / O\*NET | — | DOT 保留在 Social Security Administration |
| 培训需求分析（要很细） | Task inventory + CIT | T&M（对动作类） | 培训用 task 要具体到工具层次 |
| 绩效评估量表（BOS / BARS） | CIT | Task inventory | BOS 由 critical incidents 构成，反映最佳 / 最差绩效 |
| 选拔测验依据 | CIT + Task inventory（或第 3 章方法） | — | 常需 MQ 合规证据（见 pharmacy MQ 案例） |
| 法定合规 / 反歧视诉讼 | Task inventory（量化） + CIT（行为证据） | — | Sanchez & Fraser 关于 importance 度量可法庭化 |
| 工作再设计 / 人机工效 | Time-and-motion（含 motion study） | — | 送餐 orderly 案例是经典 |

## 选型的三问

1. **要不要量化？** 要 → Task inventory；不要 → FJA / CIT 够用。
2. **关心"做多久"还是"做什么 / 发生了什么"？** 时间 → T&M；内容/分类 → FJA / Task inventory；优 vs 劣的鲜活例子 → CIT。
3. **样本能有多大？** 大（几十到上百 incumbent）→ Task inventory 最物有所值；小（<10 SME）→ FJA 观察–访谈 或 CIT 小组。

## "混搭"提示

课本原话：**"you can invent any number of methods by choosing from the building blocks in Chapter 1."** 第 2 章展示的是"常用成熟组合"，但：

- 可以在 Task inventory 问卷里问 *criticality*（来自 CIT 精神）。
- 可以给 CIT 收集到的行为先按 FJA 语法重写成任务句，再归类。
- Fine's FJA 本身就是一种"FJA + task-level 评分"的混合。

## 与第 1 章四大构件（building blocks）的对应

提醒学生：第 2 章这四方法，放到第 1 章框架里就是——

- **Descriptors**：T&M 给 *work activities*（细到 elemental motion）+ *performance standards*（时间）；FJA 给 *work activities* + 部分 worker characteristics；Task inventory 给 *work activities* + *time/frequency/importance*；CIT 给 *critical incidents*。
- **Methods of data collection**：T&M = observation / recording；FJA = observation + interview + SME panel；Task inventory = questionnaire；CIT = SME recall meeting。
- **Sources**：几乎都用 job incumbent + supervisor + 偶尔 technical expert；Fine's 新版明确主打 SME 面板（incumbents）。
- **Units of analysis**：四者共同核心是 **tasks**；T&M 再往下切到 elemental motions；FJA 往上汇到 data/people/things 功能类。

## 章末四方法再汇总（Chapter Summary 精神）

> **Time-and-motion** 优化效率；**FJA** 用统一语法描述"做什么并涉及数据–人–物的什么程度"；**Task inventory** 把 FJA 的任务句问卷化、量化；**CIT** 用生动的行为事例填进绩效/培训所需的肉。
