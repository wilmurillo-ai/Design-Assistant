# Performance Management and Appraisal

## 核心区分

- **Performance appraisal**：正式评价 individual workers 在工作中做了什么，常见形式是年度或半年度评价表和面谈。
- **Performance management**：更宽，包含 appraisal，也包含日常 coaching、绩效改进和未来职业发展。

课本强调：appraisal 常用于过去绩效的行政决策；management 还要帮助员工改进当前工作，并为更高层级或不同岗位的发展做准备。

## 三类目标

| 目标 | 说明 | 对工作分析的要求 |
|---|---|---|
| 行政奖惩 | 加薪、纪律、解雇、晋升等 | 评价维度必须 job-related 且可辩护 |
| 改进与发展 | feedback、coaching、career advancement | 需要具体情境和可改进行为，不只是分数 |
| 测验/干预验证 | 用绩效作为 criterion 验证 test 或 training | 需要可靠、有效、与工作内容相连的绩效标准 |

课本提示：同一个 appraisal 系统很难同时把行政奖惩和绩效改进两个目标都做得很好。

## 常见 rating formats

| 格式 | 核心做法 | 优点 | 问题 |
|---|---|---|---|
| Graphic rating scales | 对 traits 或通用工作要素打分，如 quality、cooperation、leadership | 易开发、易理解、可跨岗位用 | trait 太主观，法律风险高，反馈不利于改进行为 |
| BARS | 给评分点配 observable behavioral anchors | 比 trait 更行为化，帮助 rater 理解评分点 | 员工行为不一定刚好匹配 anchor；同一员工可能跨多个 anchor |
| BOS | 对每个行为项目评分，通常评频率 | rater 更容易判断；多项目可估 internal consistency reliability | 项目更多，填写更费时间 |
| Forced-choice | 从一组陈述中选最像/最不像员工的行为 | 信息质量高，rater 不易操纵初期结果 | 开发和计分复杂；managers 通常不喜欢，后来可能学会规避 |

## 格式比较

BARS、BOS 和 forced-choice 都需要大量 SME 工作：

- 生成优到劣的行为陈述。
- 判断陈述属于哪个绩效类别。
- 判断行为好坏程度是否清楚。
- forced-choice 还要匹配社会期许程度相似但绩效区分力不同的项目。

课本判断：

- BOS 的 user acceptance 通常最好。
- Forced-choice 初期信息质量可能最高，但 manager 接受度低。
- Graphic rating scales 容易做，但在法律和反馈质量上弱。
- 政治和组织接受度有时比量表格式本身更决定系统是否有用。

## 工作分析应提供什么

绩效评价需要 job analysis 产生能支持两件事的信息：

1. 行为主要受 incumbent 控制。
2. 行为能区分 good vs poor performance，并影响工作结果。

只知道 outcome 往往不够。例如 receiver 收货速度慢或损耗高很重要，但这不告诉 manager 如何 coaching。工作分析要揭示导致结果的行为过程。

## 为什么 CIT 特别适合

**Critical Incident Technique (CIT)** 记录：

- context of situation
- employee behavior
- consequence/outcome

这正好服务 performance appraisal：它过滤掉普通/典型行为，聚焦在员工行为导致好或坏结果的关键事件。因此 CIT 很适合开发 BARS、BOS 和 forced-choice 项目。

## 其他可用方法

- **FJA 和 task inventories**：提供具体 tasks，可汇总为 duties，作为绩效维度。
- **MPDQ**：对管理岗位可基于任务内容和职位层级生成定制 appraisal forms。
- **Competency models**：如果 competencies 清楚连接组织重视的绩效行为，可用于 coaching 和发展。
- **Worker attribute methods** 如 PAQ、TTA：通常不如行为/任务方法适合绩效评价，因为它们更像 capacity for behavior，而不是 actual on-the-job behavior。

## Behavioral job description

Behavioral 或 results-oriented job description 把 performance standards 写进 task descriptions。作用：

- 让标准更客观，减少 supervisor 临时判断负担。
- 清楚告诉 incumbent 什么表现算成功。

例子逻辑：

- 普通：沟通有效。
- 行为化：在明确时限内用书面形式批准或拒绝下属设备申请。
- 普通：平衡现金抽屉。
- 行为化：规定班末和月末允许的差额上限。

局限：绩效是多面向的，标准写得再具体也可能漏掉重要方面。一个 bank teller 可能现金极准，但对客户粗鲁、破坏同事关系、无视指示，仍然是绩效问题。

## Coaching 的重点

数字评分和全局行为标签很少足以支持 coaching。告诉员工"你在工作分配上是 3 分"并不会说明如何改进。coaching 需要具体 context 和可替代行为，例如：

- 是否把项目匹配给了合适技能的人。
- 是否让同一个人背负太多项目。
- 是否明确项目优先级。
- 是否沟通了 deadlines。

绩效管理最好是持续过程，而不是等年度评价才给反馈。
