# 第 7 章案例库

用途：回答时尽量调用这些 c7.pdf 原型案例，而不是凭空造例子。每个案例都标出适合说明的概念。

## Job description 案例

| 例子 | 用途 |
|---|---|
| Sales manager job description | 说明 job title、office/department、reporting relationships、job summary、duties、job evaluation points 等可放入 job description |
| Medical assistant job description | 说明 O*NET/SOC code、summary、administrative + clinical duties、what/why 的任务句 |
| Receiver job description | 说明 receiving job 的 mission、working conditions、FLSA status、supplementary information |

### Receiver summary

Receiver 的工作可概括为：从 shippers 接收 computers 和 parts，核对收到的物品与 receipts 是否一致，并把物品放到指定地点。

适合说明：

- 好的 job summary 要说明 job 为什么存在。
- duties/tasks 可以从 summary 展开。
- working conditions 如 heights、dust、shift work 可作为 other information。

### Medical assistant task

叙述中提到的 task 逻辑：interview patients 以获得 medical information，并测量 vital signs、weight、height。

适合说明：

- what = interview patients / measure signs。
- why = obtain medical information。
- 一条 task statement 可以同时包含多个 what。

### Receiver task

叙述中提到的 task 逻辑：

- unpack and examine products for damage and shortages。
- print barcode labels and put-away documents using computer and printers。

适合说明：

- what = unpack/examine/print。
- why = identify damage and shortages。
- how = using computer and printers。

## Job description 问题案例

### "Other assignments as required"

课本指出这类 summary 兜底句通常信息量低。可能目的包括防止员工拒绝未列任务，或表示团队时代成员可能做通常职责外的任务；但它不帮助读者理解 job essence。

适合说明：

- summary 应少用空泛兜底句。
- 工会可能不喜欢过松表述，因为它模糊工作边界。

### "Leaps tall buildings"

课本用夸张任务句说明 prescriptive description 的风险：如果岗位说明书写成 management 希望中的超人版本，会给 hiring、job evaluation 和 training 带来问题。

适合说明：

- descriptive 通常优于 prescriptive。
- 多名 incumbents 的实际工作数据可以缓解 job crafting 带来的漂移。

## Performance appraisal 案例

### College professor

课本用 professor 举例说明：

- 可以评价 lectures 和 exam design。
- Faculty 绩效维度常包括 scholarly/artistic productivity、teaching、service。
- BARS 示例围绕考试实践，BOS 示例也使用 professor 行为项目。

适合说明：

- 绩效维度最好来自 job functions，而不是抽象 traits。
- BARS/BOS 用 observable behavior 解释评分。

### Trait "leadership"

课本指出，评价 leadership 本身没有错，但如果不定义具体工作行为，raters 可能用不同理由认定一个人领导力好或差。

适合说明：

- Graphic rating scales 的法律和反馈风险。
- 抽象 trait 要转成 job behavior。

### Forced-choice professor items

示例中的行为类型包括：

- shows up late for class
- fails to use board/notes/outlines
- asks clear direct exam questions
- asks exam questions from lecture and text

适合说明：

- forced-choice 要求 rater 选最像和最不像。
- 开发难点在于区分社会期许和真实绩效区分力。

### Sales manager work assignments

课本用 sales manager 说明，"工作分配不好"背后可能有多种具体行为原因：

- 没把 individuals 的 skills 匹配给合适 projects。
- 同一人被分配太多 projects。
- projects priorities 没说明。
- deadlines 没沟通。

适合说明：

- 数字评分不足以 coaching。
- coaching 需要具体 context 和替代行为。

### Receiver performance outcome

Receiver 的绩效可用每小时接收 items 数量减去损失函数表示，例如 damaged items stocked、items signed for but never stocked。

适合说明：

- outcome 重要但不足以指导改进。
- 工作分析还要揭示导致 outcome 的 behaviors。

### Bank teller standards

课本把普通任务句 "balances cash drawer" 改成带标准的行为/结果导向版本：班末差额和月末总差额都有上限。

适合说明：

- behavioral job description 把 performance standards 写入 task。
- 但准确现金处理不等于完整好绩效；客户服务、同事关系、服从指示也可能重要。

## Job evaluation 案例

### Bank teller equity

课本用 teller 比较说明 equity theory：

- 与资历、努力和薪酬都相似的人比较，会觉得公平。
- 看到资历和努力更高的人拿更高薪，也可能觉得公平。
- 看到资历和努力相似的人拿高薪，会觉得不公平。

适合说明：

- inputs/outcomes ratio。
- 同一 job 内的公平更多属于 performance management，不是 job evaluation 主体。

### Salary surveys

银行 teller 如果在本银行薪酬低于附近银行，本银行难以吸引和保留好 teller；若高于市场太多，利润受损。

适合说明：

- same job, different companies 属于市场薪酬问题。
- salary surveys 用于让薪酬处于同一 ballpark。

### Hypothetical Internet company

Table 7.6 用 Internet company 展示 factor comparison：每个 job 在 skill、mental demand、physical demand、responsibility、working conditions 等因素上被分配 dollar value，各因素相加等于 total job rate。

适合说明：

- factor comparison 如何把 total pay 分配到 compensable factors。
- skill 在该例中似乎是最重要的 pay factor。

### PAQ 340 jobs

McCormick、Jeanneret 和 Mecham 的研究：PAQ 数据来自 45 个组织的 340 个 jobs，用 multiple regression 从 PAQ attributes 预测 salary，并交叉验证；平均 multiple correlation 约 .85。

适合说明：

- worker-oriented attributes 可用于 job evaluation。
- cross-validation 防止初始样本上的结果被高估。

### Air Force task inventory

Ray Christal 及同事用 task inventory 做 job evaluation。每个 job 由 task scores 或 task composites 表示，再用 regression 预测 salary。

适合说明：

- work-oriented task information 也可用于薪酬。
- task inventory 覆盖广泛 jobs 时会变得很长。
- 若采用 point system，仍需 incumbents/supervisors 澄清信息。

### Warehouse skill-based compensation

Warehouse worker 可随着掌握 packing/shipping、searching items、receiving/stocking、troubleshooting 等任务而涨薪。

适合说明：

- skill-based compensation。
- pay for what person knows/can do vs pay for fixed job。
- 管理层要权衡更高工资和效率收益。

## Job design/redesign 案例

### Pilot and flight management computer

Commercial aviation 可以让 flight management computer 自动飞行和降落，但如果 pilot 完全 out of the loop，系统失效时可能无法安全接管。

适合说明：

- people vs machines 不能只看自动化能力。
- 设计要保持 human readiness 和 joint custody。

### Volvo assembly crew

Volvo assembly plant 中，传统流水线被可移动 base 取代，assembly crew 被设计为 cross-trained，使成员最终几乎能建完整汽车。

适合说明：

- 技术改变 task allocation。
- cross-training 需要组织激励支持，常与 skill-based pay 连接。

### Race car driver and pit crew

Race car driver 依赖 pit crew 换胎、加油等。工作互依可能促使 team owner 讨论团队表现。

适合说明：

- job relations 可来自 functional interdependence。
- job design 不只分配个人任务，也塑造团队互动。

### Hospital messenger reasonable accommodation

坐轮椅的 paraplegic applicant 应聘 hospital messenger，需要移动血液/血浆样本。分析发现他能完成移动、开门、取样等要求，唯一问题是够不到电梯 emergency alarm；解决办法是给一根 stick。

适合说明：

- 残障便利中的 job analysis 是人-任务-环境的具体匹配。
- reasonable accommodation 可能很小但关键。

### Data entry analyst redesign

Morgeson & Campion 的 data entry analyst redesign：原 job 太 routine，主要做 clinical drug trials 的数据收集和录入。分析出 task clusters 后，保留多数 routine clusters，并加入系统开发/实施相关 clusters，提升 autonomy 和 motivational properties。

适合说明：

- task cluster 是 redesign 的可重组单位。
- redesign 要先定义、评分、组合 task clusters，再评估结果。
- redesign 后可通过问卷比较 motivational/mechanistic properties 和 satisfaction。
