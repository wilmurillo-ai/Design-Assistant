# Job Description

## 核心定义

- **Job analysis**：发现和理解工作的过程，通常产生书面报告或摘要。
- **Job description**：基于 job analysis 的简短书面 summary/snapshot，用来沟通 job 的本质。
- **Position description**：描述一个具体个人在组织中实际做什么；常见于独特管理岗位、分类、薪酬或 staffing。

课本立场：job 是多个足够相似、可被同等管理的 positions 的集合；job description 面向多个 incumbents，而 position description 面向某一个 position。

## Job description 的主要功能

最重要功能是向不熟悉该工作的人说明工作本质。SME 应该能认出这份工作，但主要读者不是 job expert，而是 job novice、管理者、求职者、HR 或其他需要理解岗位的人。

课本建议 job description 应短。Gael 建议 1-3 页；Ghorpade 也强调短，但不规定页数。

## 常见结构

| 部分 | 内容 | 学习提示 |
|---|---|---|
| Identifiers | job title、部门/地点、reporting relationships、job code 等 | 汇报关系应是 job title 之间的关系，不是人名之间的关系 |
| Job summary | 说明 job 为什么存在，mission/objective 是什么 | 最难写，关键在判断纳入什么、删除什么 |
| Duties and tasks | 说明 what/how/why | 通常用 duty/global task 层级，不列 100 条细 task |
| Other information | responsibility、qualification、working conditions、FLSA status、补充信息 | 往往服务特定 HR 目的，如 job evaluation、合规、招聘 |

## Duties and tasks 的 what-how-why

任务/职责句通常回答三件事：

1. **What** does the jobholder do?
2. **How** does the jobholder do it?
3. **Why** is it done?

写法上，句子通常以 action verb 开头，紧跟 direct object；如果读者不能自然推断，再补工具、设备、材料、目标或结果。信息详细程度取决于读者和用途：招聘沟通可能需要更多细节，内部专家可能需要更少。

## Other information 的用途

常见补充信息包括：

- Autonomy / discretion / responsibility：常与 job evaluation 系统相关。
- KSAO、education、minimum qualifications：可服务 staffing 或合规。
- Working conditions：危险、粉尘、噪声、极端温度、轮班等。
- FLSA exempt/nonexempt：分类取决于工作性质，而工作性质需要 job analysis 发现。

## 描述实际还是规定理想？

第 7 章建议 job description 通常应 **descriptive**：描述工作实际上如何完成，而不是 management 希望它如何完成。

原因：

- 实际岗位会因为 incumbent 兴趣、能力和习惯发生 job content drift 或 job crafting。
- 如果只写 prescriptive 的理想版本，招聘、评价、培训和薪酬都可能建立在错误信息上。
- 解决个体差异的办法不是写空想职责，而是分析多个 jobholders，让 job description 足够宽以覆盖当前大多数 incumbents。

## 当前还是未来？

Job description 通常是当前工作快照。环境变化越快，说明书越容易过期，但这并不意味着 job analysis 没用。相反，变化越快，管理者越需要知道系统当前怎样运转，才能监控、评价、指导和规划。

课本提到一种实用更新机制：在数据库中维护 job description；每当 manager 要填补某个 position 时，把现有说明书发给 manager 审阅和修订，并在必要时咨询其他 managers 判断变化是否适用于该 job title 的多数 positions。

## 适合的工作分析方法

为了产出 job description，工作分析至少要发现 essential duties/tasks 和 job purpose。适合方法包括：

- Functional Job Analysis (FJA)
- Critical Incident Technique (CIT)
- Combination Job Analysis Method (C-JAM)
- 其他能提供 work-oriented descriptors 的方法

如果 job description 还要服务薪酬、招聘、合规或绩效管理，就必须追加相应信息。HR purpose 决定"other information"里收什么。

## 学习时常见误区

- 把 job description 当成 job analysis 全部结果。实际上它只是简短产物。
- 写给专家看，导致外部读者无法理解。
- summary 里塞入空泛兜底句，而没有说明 job 为什么存在。
- duties/tasks 只写抽象名词，不写 action verb 和 direct object。
- 把 job evaluation 或 FLSA 分类信息当成自然会出现的内容，而不追问它服务什么目的。
