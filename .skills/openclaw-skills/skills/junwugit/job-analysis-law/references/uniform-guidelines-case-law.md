# Uniform Guidelines 与案例法中的工作分析要求

## Guidelines 的基本要求

Uniform Guidelines 要求 selection procedure 的选择建立在对工作的理解之上。某种 job analysis 应被用来获得这种理解，但 Guidelines 不强制某一种方法。

课本强调：

- Guidelines 更关注工作者的 observable work behaviors。
- 它们把 job analysis 当成类似 job description 的产品来谈。
- 它们反复要求记录重要性、关键性、任务、行为、工作产物、KSAO 与测验内容之间的关系。

## Criterion-related validity 下的要求

Guidelines 要求审查岗位信息，以确定与岗位相关的 work behaviors 或 performance measures。

这些 criteria 应代表：

- critical 或 important job duties
- work behaviors
- work outcomes

报告中应包括：

- work behaviors 或 outcomes 的完整描述
- 这些行为或结果的 criticality / importance 度量
- 判定其关键或重要的依据，例如时间占比、难度、执行频率、错误后果等
- 若把多个岗位合并做 validity study，要为每个岗位提供信息，并说明合并理由

学习结论：

> criterion-related validity 下的 job analysis 重点是重要任务、可观察行为和可用作绩效标准的工作结果。

## Content validity 下的要求

Content validity 要证明 selection procedure 的内容代表岗位绩效的重要方面。

Guidelines 期待 job analysis 包含：

- 使用了什么 job analysis 方法
- work behaviors、相关 tasks、以及 work products 的完整描述
- work behavior 的 criticality / importance 度量与确定方法
- 若识别 KSA，应给出操作性定义
- KSA 与 work behavior 之间关系的确定方法
- 工作情境，包括任务发生的 setting
- KSA 在行为中如何使用，以及复杂度和难度

还要建立明确链接：

- 测验每一部分意图抽样哪些行为
- 这些行为在岗位中对应哪里
- 若是 work sample，测试情境和复杂度应匹配岗位

课本例子：如果岗位要求把数学公式输入 spreadsheet，测试可以包含这一点；如果岗位不要求，就不应把它加入测试。

学习结论：

> content validity 的防御核心是"测验内容怎样代表岗位重要内容"。

## Construct validity 下的要求

课本指出，Principles 和 Standards 不再把 construct validity 当作单独策略，而是把它嵌入多种证据来源。但 Guidelines 仍讨论 construct validity。

Guidelines 下的底线：

- 被测 trait 或 construct 必须对岗位成功重要。
- job analysis 应显示重要 work behaviors。
- job analysis 应识别和定义将要测量的每个 construct。

报告应包括：

- job analysis 方法
- work behaviors、work outcomes 及其 criticality / importance
- 判定重要性的依据，如难度、频率、错误后果
- 若合并或比较多个岗位，要描述每个岗位的 behaviors 和 products，并说明岗位相似性

学习结论：

> construct validity 讨论中，工作分析要把心理构念连回岗位行为和 KSAO。

## Case Law：法院从工作分析中看什么

案例法重要，因为它显示法院如何解释法规和指南。Thompson and Thompson (1982) 总结了能帮助 personnel selection 经受法律审查的 job analysis 要点：

1. 必须针对将使用 selection instrument 的岗位做 job analysis。
2. 结果应书面报告。
3. 分析师应详细描述程序。
4. 由 knowledgeable job analysts 从多种当前来源收集资料。
5. 若使用 incumbents，应使用足够多且确实担任目标岗位的员工。
6. 分析应包括 tasks、duties、activities。
7. 选择工具应代表最重要任务。
8. 入门岗位应说明绩效标准。
9. 尤其在 content validation 下，应说明 KSA。

Arvey and Faley (1988) 进一步指出：

- 测验内容的权重应对应岗位中的重要性。
- 保留测验开发记录很重要。
- 法院似乎更关心 job analysis 信息质量，而不只是具体方法名称。

Brumback 的建议：

- 收集 task frequency 数据。
- 标准化程序，使他人可重复分析。
- 有方法判断岗位要求的重要性。
- 度量岗位要求的 trainability 或 difficulty。

## 课堂答题模板

如果题目问"怎样让一个选拔测验更能经受法律审查"，可以这样组织：

1. 先做目标岗位的系统工作分析。
2. 记录方法、来源、样本、时间和岗位版本。
3. 描述任务、职责、可观察行为、工作产物和工作环境。
4. 评估任务重要性、频率、难度、错误后果。
5. 明确 KSAO，并说明它们与任务的关系。
6. 把测验内容或构念逐项映射到重要任务/KSAO。
7. 保存测验开发和验证证据。
