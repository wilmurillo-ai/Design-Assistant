# Chapter 4 Examples and Teaching Anchors

## 使用方式

这些例子用于在对话中把抽象概念落地。不要一次性全讲；根据用户问题挑 1-2 个最贴近的例子。

## C-JAM 例子

**Task statement 写法**

- Personnel testing specialist：写多选题以评估申请者资格；向申请者解释考试分数申诉或投诉。
- Web designer：访谈客户以确定网页内容；写代码以呈现图形布局。
- Auto equipment mechanic：检查设备以判断维修或替换需要；改装设备以提高安全或效率。
- Water/fuels analyst：检查酸碱罐液位；维护燃油添加系统以满足环保标准。
- Clerk：折叠、装订、检查文件准确性。

教学用法：让用户指出每条任务的动词、宾语、目的，再判断是否太粗或太细。

**Task importance**

文员岗位表中，typing 和 filing duties 下的任务按 task importance 排序。用它说明 C-JAM 的输出不是普通任务清单，而是带重要性权重的任务清单。

**KSAO 与 ADA 风险**

颜色冲印岗位看似要求色觉，但教材例子显示，真正要求可能是能校准机器。用它提醒：感觉/运动要求必须贴近任务本质，避免无必要排除。

**Selection battery**

人员测验专家岗位可把"读写能力"设为 pass/fail，把"测验题编写知识"和"与困惑或愤怒申请者沟通的能力"做成可评分笔试/面试。用它说明：C-JAM 的 KSAO 评分能推导 selection decision 和测验权重。

## MJDQ 例子

**照明**

旅馆大厅和医院手术室对亮度要求不同；摄影师或珠宝匠可能需要可调光。用它说明 job design 可以改工作环境，不只是选人或培训人。

**Motivational**

音乐会演奏者需要发展高技能，并对演出结果负责。用它说明 skill variety、responsibility、task significance。

**Mechanistic**

披萨店员工按标准动作均匀撒蘑菇。用它说明 efficiency、standardization、motion economy。

**Biological**

长时间站立可以通过坐凳或休息安排减轻负担。用它说明 physiology、rest breaks、posture、work context。

**Perceptual/Motor**

汽车油门和刹车的标准位置很重要；显示颜色最好放在中央视野。用它说明 control-display compatibility、visual field、human factors。

**Trade-off**

从头到尾做一餐与只洗盘子相比，前者更有意义但训练复杂，后者更简单但可能无聊。用它说明 simplification 可能提升效率却降低满意度。

**Heart surgeon**

心脏外科医生高度自主、技能复杂、责任重大，通常 motivational 高、mechanistic 低。用它说明量表之间可能负相关。

## WDQ 例子

**用 social support 减轻 trade-off**

如果业务上不能增加自主性或任务丰富度，可以考虑增加同事支持、互动机会或反馈来源。用它说明 WDQ 比 MJDQ 多了 social characteristics，因此提供更多设计杠杆。

**专业 vs 非专业职业**

WDQ 能区分 occupational groupings。用它解释 construct validity：如果理论上专业岗位更高复杂度/信息处理，WDQ 分数应能反映这种差异。

## O*NET 例子

**Generalized work activity**

"Getting information" 可同时适用于律师查案例和地质学家分析水样。用它说明 GWA 足够宽，能跨职业比较。

**Nested specificity**

"Interacting with computers" 对程序员可能是写 C++ 代码，对秘书可能是用文字处理软件打信。用它说明 O*NET 的 broad-to-specific 层级。

**Work context**

监狱看守可能经常接触愤怒或不愉快的人；空管员的错误后果可能极其严重。用它说明 work context 与 consequences of error。

**Experience requirements**

护士和叉车司机需要 license。用它说明 licensure 是 experience requirements，而不是普通 education。

**Work values**

医疗工作者可能从帮助患者中获得 relationships 和 meaning。用它说明 work values 是 will do / finds rewarding，而不是 can do。

## 课堂提问模板

用 C-JAM 练应用：

1. 这个项目的 purpose 是 selection 还是 training？
2. 哪些 tasks 需要先生成和评分？
3. 哪些 KSAOs 是 new hires 必须具备的？
4. 哪些 KSAOs 应该做 pass/fail，哪些应排序？

用 MJDQ/WDQ 练应用：

1. 当前问题是满意度、效率、舒适度、安全、离职，还是绩效？
2. 这个岗位在哪些 design dimensions 上可能过高或过低？
3. 改某个维度会带来什么 trade-off？
4. WDQ 是否提供 MJDQ 没有的设计杠杆？

用 O*NET 练应用：

1. 用户需要 cross-occupation comparison 还是 organization-specific detail？
2. O*NET 能提供哪些 worker/work descriptors？
3. 哪些信息还需要本地 SME 重新收集？
