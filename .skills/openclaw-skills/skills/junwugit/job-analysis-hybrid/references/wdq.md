## 一句话定义

**WDQ (Work Design Questionnaire)** 是 Morgeson 与 Humphrey 开发的现代 work design measure，用 21 个因素更全面地描述工作设计，补足 MJDQ 和其他早期测量的不足。

## 为什么出现 WDQ

WDQ 的开发逻辑：

1. 回顾 work design 和 job analysis 文献。
2. 汇总既有研究中出现过的工作特征。
3. 用 sorting exercises 整理出大量 work design factors。
4. 基于 243 个岗位的 incumbent 数据形成 21-factor questionnaire。

教材强调：WDQ 包含 MJDQ 的核心元素，同时增加了其他工作特征，尤其是 social characteristics 和 work context，因此更完整。

## 21 个因素

| 大类 | 因素 |
|---|---|
| **Task characteristics** | work scheduling autonomy; decision-making autonomy; work methods autonomy; task variety; task significance; task identity; feedback from job |
| **Knowledge characteristics** | job complexity; information processing; problem solving; skill variety; specialization |
| **Social characteristics** | social support; initiated interdependence; received interdependence; interaction outside organization; feedback from others |
| **Work context** | ergonomics; physical demands; work conditions; equipment use |

记忆方式：

- Task + Knowledge：传统 motivational work characteristics 的核心区域。
- Social：MJDQ 较少覆盖，但对满意度和现代团队/服务型工作很重要。
- Context：身体、环境、设备等工作情境条件。

## 信效度证据

教材列出的核心证据：

- internal consistency reliability 平均约 .87。
- interrater agreement 平均约 .82。
- interrater reliability 平均约 .36，属于低到中等，但与岗位知觉数据的性质一致。
- WDQ 因素与 DOT、O*NET archival scales 的关系符合预期。
- WDQ 能区分 occupational groups，例如专业性和非专业性职业。

解释时可以说：不同 incumbents 对同一 job 的感知有足够 convergence，但个体知觉仍然会有差异。

## 如何处理 MJDQ Trade-offs

MJDQ 常见结论：mechanistically designed jobs 往往更无聊、更不满意；但组织可能不能简单提高 motivational characteristics，因为那可能影响质量或效率。

WDQ 的贡献是给出更多设计杠杆。例如：

- 如果不能增加自主性或任务丰富度，可以增加 **social support**。
- 社会支持高的岗位也往往更令人满意。
- 因此，MJDQ 中 motivational 与 mechanistic 的典型 trade-off，可能通过 social 或 context 维度被减轻。

这不是说 trade-offs 消失，而是 WDQ 提供了更细的设计空间。

## 用途与限制

适合使用 WDQ 的场景：

- 现代工作设计诊断。
- 比较岗位的 work design factors。
- 研究工作特征与满意度、绩效、离职、健康等结果的关系。
- 讨论如何补足 MJDQ 的四类 approach。

教材还提到 WDQ 后续使用很多，并被翻译成多种语言，包括中文。

限制与提醒：

- WDQ 是 general measure，不是某个岗位的详细 task inventory。
- 如果目标是开发 training curriculum、work sample、selection battery，WDQ 只能提供背景诊断，仍要做更细的 task/KSAO 分析。
- 如果用户只想记考试重点，抓住"21 factors + 4 大类 + 更全面地解决 MJDQ 不足"即可。

## 教学提醒

当用户混淆 MJDQ 与 WDQ 时：

- MJDQ：经典四 approach，重点是 trade-offs。
- WDQ：现代 21 factors，重点是全面描述和更多设计杠杆。

当用户问"为什么 WDQ 是 hybrid"时：

- 它不是简单 task list，也不是单纯 worker trait list。
- 它跨 task、knowledge、social、context 多类工作特征，覆盖 work design 的多个层面。
