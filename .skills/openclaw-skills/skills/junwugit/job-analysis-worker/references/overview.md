# Worker-Oriented Methods 总览与选型

## 本章主线

第 3 章的共同问题是：**一个人要把工作做好，需要具备什么 KSAOs、能力、特质、动机、人格或认知过程？**

它和第 2 章的差别是焦点不同：

| 方向 | 主要问法 | 常见输出 |
|---|---|---|
| Work-oriented | 这份工作做什么、怎么做、用什么工具？ | duties、tasks、activities、critical incidents、task ratings |
| Worker-oriented | 做好这份工作的人需要什么？ | KSAOs、traits、abilities、reinforcers、personality requirements、expert cognition |

课本提醒：两者不是绝对二分。比如 JEM 把 worker requirement 写成工作现场能理解的行为证据，所以很接近 work-oriented；CTA 通常也先做 task analysis，再深入专家的认知过程。

## KSAOs 快速框架

基于 Levine (1983) 的定义：

| 项 | 含义 | 简例 |
|---|---|---|
| Knowledge | 记忆中可调用的工作相关事实、概念、语言、程序 | 法规知识、蓝图符号、店内流程 |
| Skill | 训练形成的、使用工具/设备/机器完成任务的能力 | 用叉车、操作电钻、演奏乐器 |
| Ability | 相对稳定的容量，支持学习 K/S 或在不依赖工具时完成任务 | 空间能力、数学推理、色觉 |
| Other characteristics | 兴趣、偏好、气质、人格、对工作条件的适应性 | 独处耐受、责任心、喜欢变化 |

教学时不要把 KSAO 当词典背诵。更好的问法是：这个岗位的成功差异，主要来自"会什么"、"会操作什么"、"天生/较稳定的能力"、还是"日常风格与偏好"？

## 方法并列表

| 方法 | 核心 descriptor | 最典型用途 | 适合强调 |
|---|---|---|---|
| Job Element Method (JEM) | elements/subelements: 行为 + 证据 | 选拔、work sample tests、培训 | SME 熟悉的现场语言；直接把 job analysis 转成测验内容 |
| Position Analysis Questionnaire (PAQ) | 标准化 element/item | job evaluation、薪酬、job families、disability、选拔支持 | 跨岗位比较；统一数据库；抽象通用维度 |
| Threshold Traits Analysis System (TTAS) | 33 traits, can do/will do | 选拔、培训、岗位说明、job evaluation | 不遗漏广泛的人类特质 |
| Ability Requirements Scales (ARS) | 通用人类能力 + 对应测验 | 选拔、构建 job families | 把能力需求接到心理测验 |
| Occupational Reinforcer Pattern (ORP) | 岗位提供的 reinforcers | vocational guidance | 人的需要与岗位回报是否匹配 |
| AET | 设备、环境、工效负荷 | 工作再设计、人体工学、合理便利 | 减轻 stress/strain，提高绩效质量与数量 |
| Job Components Inventory (JCI) | 工具/设备使用，尤其 220 项工具 | 青少年职业指导、培训 | 岗位实际要碰什么工具 |
| Cognitive Task Analysis (CTA) | 专家知识与认知过程 | 专家培训、减少人为错误、系统设计 | novice vs expert；任务背后的策略和心智模型 |
| Personality-Oriented / PPRF | Big Five 相关岗位行为 | personality-based selection | 哪些人格要求对岗位成功有差异化意义 |

## 按目的选方法

| Purpose | 优先方法 | 选型理由 |
|---|---|---|
| 开发 work sample 或高度岗位相关的选拔题 | JEM | elements/subelements 本身能转成测验项目或工作样本 |
| 需要全公司岗位比较、薪酬点数、岗位族 | PAQ | 所有岗位使用同一标准 item，便于数据库比较 |
| 需要避免遗漏体力、心智、动机、社会特质 | TTAS | 33 traits 覆盖 can do 与 will do |
| 需要把工作要求接到现成能力测验 | ARS | Fleishman 能力清单与心理测验相连 |
| 生涯咨询：人想要什么工作回报 | ORP | 比较个人 needs 与岗位 reinforcers |
| 改善设备、工作姿势、工作条件、残障便利 | AET | ergonomics 视角最强 |
| 给年轻人解释入门岗位实际用哪些工具 | JCI | 工具/设备清单非常具体 |
| 提炼专家经验、训练新手、减少错误 | CTA | 捕捉常规 task analysis 看不到的认知过程 |
| 评估人格测验是否岗位相关 | PPRF | 把岗位行为要求映射到 Big Five 维度 |

## 常见混淆点

| 易混项 | 区分方式 |
|---|---|
| JEM element vs Ch.2 element vs PAQ element | JEM element = 行为 + 证据；Ch.2 element = 任务的最小动作；PAQ element = 标准问卷 item |
| ability vs skill | ability 更稳定，可支持学习；skill 多是训练形成，常涉及工具/设备 |
| TTAS trait vs ARS ability vs ORP reinforcer | trait/ability 描述人需要具备什么；reinforcer 描述岗位能给人什么 |
| JEM vs PAQ | JEM 从 SME 语言生成岗位特定 elements；PAQ 用同一套标准 items 比较所有岗位 |
| CTA vs task analysis | CTA 通常在 task analysis 后做，深入专家如何判断、表征、自动化处理任务 |
| Big Five vs PPRF | Big Five 是人格理论；PPRF 是把岗位行为要求连接到 Big Five 的 job analysis 工具 |

## 教学展开顺序

如果用户是新手，按三层讲：

1. 先讲 work-oriented vs worker-oriented，再给 KSAO。
2. 再讲"目的决定方法"：选拔/JEM，薪酬/PAQ，训练专家/CTA，人格测验/PPRF。
3. 用户追问具体方法时，读对应 reference，不要一次展开所有公式、表格和研究细节。

如果用户在做题，先定位题干中的关键词：

| 题干关键词 | 大概率方法 |
|---|---|
| B/S/T/P、TV、IT、TR、screenout、Primoff | JEM |
| 194/300 items、S-O-R、information input、DNA items | PAQ |
| can do/will do、33 traits、Lopez | TTAS |
| Fleishman、generic human abilities、job families | ARS |
| ability utilization、security、autonomy、vocational guidance | ORP |
| ergonomics、wheel ramp、stress and strain | AET |
| 220 tools、entry-level jobs、Britain | JCI |
| novice/expert、think aloud、airport screener、mental model | CTA |
| OCEAN、PPRF、Surgency、social desirability | Personality-oriented |

## 范围提示

Competency modeling 也很像 worker-oriented，但课本把它放到第 5 章，因为它常用于管理、领导与团队情境。用户问到时，给这个定位即可，不要把第 5 章内容伪装成本章内容。
