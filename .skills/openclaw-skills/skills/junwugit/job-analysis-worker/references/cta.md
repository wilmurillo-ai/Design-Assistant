# Cognitive Task Analysis (CTA)

## 一句话

**Cognitive Task Analysis (CTA)** 是一组用来 eliciting、analyzing、representing 工作中知识与认知过程的工具。它不只是问专家做了什么，而是问专家如何看、如何想、如何判断、如何快速做出高质量决策。

## 为什么它是 worker-oriented

CTA 聚焦的是 worker 内部的认知活动：

- 专家具备什么知识。
- 专家如何组织知识。
- 专家在关键情境下如何判断。
- 专家与新手的差异在哪里。
- 哪些 mental processes 能被训练或支持。

它通常不是替代 work-oriented job analysis，而是在先获得 duties/tasks 后，从中挑选关键 tasks 做深入认知分析。

## 何时使用

| 适用情境 | 说明 |
|---|---|
| 专家与新手差异很大 | 飞行员、安检员、外科医生、消防员 |
| 错误代价高 | 航空、医疗、军队、安全关键岗位 |
| 传统 task list 解释不了绩效差异 | 同样任务，不同人判断路径不同 |
| 需要设计专家训练 | 把隐性知识显性化 |
| 需要改进系统界面或警报 | 找出人会漏看、误判、认知过载的位置 |

CTA 通常昂贵、耗时、难实施，所以应用时会选择一部分关键 tasks，而不是对全岗位所有任务都做深度分析。

## Table 3.8: Work-oriented task analysis vs CTA

| Work-oriented task analysis | Cognitive task analysis |
|---|---|
| Emphasizes behavior | Emphasizes cognition |
| Analyzes target performance | Analyzes expertise and learning |
| Evaluates knowledge for each task separately | Evaluates interrelationship among knowledge elements |
| Segments tasks according to required behaviors | Segments tasks according to required cognitive skills |
| Usually does not address representational skills | Addresses representational skills |
| Often describes one way to perform | Accounts for individual differences |

教学提示：如果用户问"CTA 和 task inventory 的区别"，用这张表：task inventory 说"做哪些任务、频率/重要性如何"，CTA 说"专家如何完成这些任务，尤其是如何判断"。

## 四类 knowledge

| Knowledge type | 含义 | 示例 |
|---|---|---|
| Declarative knowledge | what，事实性知识 | worker-oriented 方法名称、法规、设备名称 |
| Procedural knowledge | how，步骤与程序 | 手动挡换挡步骤、故障排查步骤 |
| Generative knowledge | 新情境中推导怎么做 | 第一次到陌生机场仍能找到行李与交通 |
| Self-knowledge | 知道自己知道/不知道、能/不能 | 知道自己能修灯泡但不该给自己做手术 |

注意：Seamster 等人会把 knowledge 与 skill 区分开。Knowledge 是 jobholder 拥有的信息，skill 是使用信息的过程。所以他们会说 procedural skills，而不只是 procedural knowledge。

## 三类 cognitive skills

| Skill type | 含义 | 示例 |
|---|---|---|
| Automated skills | 快速、低努力的心理过程 | 熟练驾驶时保持车道和车速 |
| Representational skills | 使用 mental models 表征系统、设备、过程 | 根据汽车心智模型判断为什么打不着火 |
| Decision-making skills | 用 heuristics、mental simulation 等快速准确决策 | 黄灯出现时根据速度和距离决定刹车还是通过 |

Representational skills 不只用于机械系统，也用于手术、消防、组织内资源协调等涉及真实对象或社会系统的工作。

## 任务与认知矩阵

DuBois 等人建议把 knowledge requirements 与 task types 交叉成矩阵：

- knowledge requirements: declarative、procedural、generative、self。
- task types: technical tasks、organization-wide tasks、teamwork、communication 等。

这个矩阵用于避免只分析显眼任务，而漏掉团队、沟通、组织层面的认知要求。

## 五类 CTA 方法

| 方法 | 做法 | 例子 |
|---|---|---|
| Interviewing methods | 询问 SME 常规任务与 critical situations 中的 mental processes | 问警察："要让逮捕在法庭上站得住，需要知道什么？" |
| Team communication methods | 观察/记录团队沟通，从中推断认知过程 | 消防员通过向队友提问判断火源 |
| Diagramming methods | 用 path-goal diagrams、decision trees 等表示概念关系 | 画汽车无法启动的 troubleshooting 树 |
| Verbal report methods | 让专家执行任务时 think aloud，或前后报告想法 | 安检员看 X 光图时说出关注点与判断 |
| Psychological scaling methods | 让专家分类、排序、评分项目，再用 scaling/clustering 表示概念关系 | 飞行员把 25 张飞行问题卡片按相似性分堆 |

课本建议：结论至少基于两种不同数据收集方法。可以先用一种方法发展模型/假设，再用第二种方法高效检验。

## Airport screener 案例

岗位：机场安检员查看随身行李 X-ray 图像，识别枪支、爆炸物和其他危险品，同时保持速度以免安检线过长。

目的：提升培训。

专家每个 bag 有三种动作：

- pass the bag。
- search the bag manually。
- stop/trap the bag in the screening machine。

数据收集：

- 离开安检点访谈 screeners，让他们提供考验技能的 incidents。
- 在实际安检任务中 verbal protocol / think aloud。
- 分析师站在后方观察，并可问问题，例如某个刚通过的 briefcase 中如何藏武器。

决策模型简化：

1. 看 X-ray，判断是否有 odd/dark 的可疑区域。
2. 没有则 pass。
3. 有则更仔细识别是什么。
4. 如果是 threat，则 stop/trap。
5. 如果不是已识别 threat，但足够大到可能藏 threat，则 search。
6. 如果不够大，则 pass。

专家与新手差异：

- 专家有大量存储图像库，能快速识别 X-ray 中的物体。
- 这种识别对专家近似 automated skill。
- 新手识别不出更多物体，因此会搜索更多 baggage，速度更慢。

## CTA 的用途

| 用途 | 例子 |
|---|---|
| Reducing human error | 飞机 cockpit 信息太多，用 auditory warning 提醒关键风险 |
| Improving training | 军人训练快速识别敌我装备；外科训练 |
| Increasing systems reliability | 找出人机界面中容易漏看或误判的线索 |
| Performance measurement | 针对专家判断过程设计评估 |
| Medical education | 外科教育中用 CTA 提升时间、精度、准确性并减少错误 |

课本提到 Wingfield 等人 review 13 篇 surgical education 研究，其中超过 90% 显示 CTA 改善 surgical outcomes。也有研究显示 CTA 获取的知识多于较不系统的方法。

## Reliability 与 validity 注意点

CTA 不是一个固定单一程序，所以不能像 PAQ 那样简单讨论某套量表的信度。

关键风险：

- 结果取决于使用哪种方法。
- 结果取决于研究了哪些 experts。
- 如果 SME 不是真专家，分析会失真。
- CTA 允许 individual differences，所以单个 expert 的路径未必代表全部专家。

课本建议：

- 用至少两个指标选择 experts，例如 time in job、superior performance measures、peer nominations。
- 可行时研究多个 SMEs。
- 用两种不同方法交叉验证结论。

## 优点与限制

优点：

- 捕捉普通 task analysis 与 trait list 看不到的 mental processes。
- 特别适合培训、错误减少、专家知识显性化。
- 能解释为什么同样完成一项 task，专家表现明显不同。

限制：

- 成本高、耗时、难应用。
- 不适合所有任务。例行、低认知复杂度工作未必值得。
- 方法多样，质量依赖 analyst 能力与 expert 选择。

## 回答模板

当用户问"CTA 的步骤是什么"：

> CTA 通常先做 work-oriented task analysis，选出关键 tasks；再找真正的 experts，用访谈、团队沟通分析、图示、think aloud、心理 scaling 等方法收集认知过程；最后把任务与知识/技能联系起来，形成训练、系统设计或错误预防方案。

当用户问"为什么先做 task analysis"：

> 因为 CTA 不是从空中分析专家大脑，它需要知道哪些 duties/tasks 值得深入。task analysis 给出任务地图，CTA 再进入其中最关键、最依赖专业判断的节点。
