# Job Element Method (JEM)

## 一句话

**JEM** 是 Ernest Primoff 在 1950s 于 U.S. Civil Service Commission 发展出的早期 worker-oriented 方法。它用 SME 熟悉的现场语言，把"做好工作的人具备什么"写成 **elements** 与 **subelements**，再用 B/S/T/P 四量表推导出选拔、排序、培训或 screenout 用的指标。

## JEM 的 element 是什么

JEM element 不是第 2 章 time-and-motion 的最小动作，也不是 PAQ 的标准问卷 item。

在 JEM 中：

- **Element** = 某种工作相关行为/能力 + 现场可观察证据。
- **Subelement** = 用来说明 element 的具体行为例子。
- 命名应尽量用 incumbents 与 supervisors 能理解的工作现场语言，而不是心理学术语。

JEM elements 覆盖三类内容：

| 类别 | 说明 | 例子 |
|---|---|---|
| Cognitive | 识别、理解、计算、读取符号或知识 | 识别工具用途、读蓝图、计算均值/标准差 |
| Psychomotor | 感知与动作控制 | 色觉、操作电钻、开凿子、驾驶战斗机 |
| Work habits | 动机性工作习惯 | 愿意多承担工作、追求无差错完成 |

## 课本 Table 3.1 例子

| Job | Element / subelement 样例 |
|---|---|
| Office manager | Ability to gain conformance；Ability to apply procedures；Ability to meet deadlines |
| Police officer | Have good physical coordination；No fear of firearms；Ability to recall facts |
| Electrician | Understanding of theory and instruments；Understand ammeter/ohmmeter/voltmeter；Knowledge of shop math |

教学提示：这张表适合用来说明 JEM 的 wording 很接近工作现场，而不是抽象的 cognitive ability label。

## 基本流程

1. 找专业 job analyst 做项目负责人，通常配 6 名 SMEs，来源多为 incumbents 与 supervisors。
2. 第一次 3-5 小时会议：SMEs brainstorm elements，分析师追问并整理；随后让 SMEs 给每个 element 打 B/S/T/P。
3. 分析师把 0、check、plus 转成分数与百分比，再计算 TV、IT、TR 等导出指标。
4. 第二次 3-5 小时会议：把结果反馈给 SMEs，用于构建测验、绩效标准、培训项目等。
5. 根据用途把 elements/subelements 转成产品，并在产品表现不佳时回头修正 job analysis。

## 生成 elements 的追问

当 SMEs 想不出更多内容时，可以追问：

- 优秀员工会展示什么，才让你确信他/她很优秀？
- 如果只能给一个人特别奖金，你会看什么？
- 什么表现会让你想解雇一个人？
- 如果员工体力弱，什么地方可能出问题？

这些问题的关键不是问抽象能力，而是把 SME 的判断拉回到工作行为与证据。

## 四个 SME rating scales

每个 scale 使用三档响应：0、check、plus。通常 0=不太适用，check=有些适用，plus=高度适用。后续计算时可按 0/1/2 转换。

| Scale | 问题 | 直觉 |
|---|---|---|
| B: Barely Acceptable | 勉强合格者是否也必须具备？ | 基础最低要求 |
| S: Superior | 它能否区分优秀者与普通者？ | 选拔高绩效者 |
| T: Trouble Likely If Not Considered | 不纳入测验/评估会不会出麻烦？ | 避免漏测关键风险 |
| P: Practical | 如果要求它，应聘者中是否有人具备，岗位能否招满？ | 避免要求过高导致无人可招 |

教学例子：

- "呼吸"在 B 与 P 上很高，但对 S 与 T 的选拔意义很低，通常不值得测。
- 脑外科医生能熟练使用手术器械，T 很可能高。
- 杂货店员要求双语流利，P 在美国语境下可能低。

## 分数换算

如果有 6 个 SMEs，每人 0/check/plus 对应 0/1/2，则某 scale 最大分是 12。总分为 6 时就是 50%。课本 Table 3.2 中 B/S/T/P 列是这种最大可能得分百分比。

注意：TV、IT、TR 会在计算后重新 scaling，所以不能直接用表中 B/S/T/P 的百分比反推表中 TV/IT/TR。

## 三个导出量表

| Derived scale | 公式/规则 | 用途 |
|---|---|---|
| TV: Total Value | TV = (S - B - P) + (SP + T)，最后 scaling 到最大 150 | 选拔价值。TV > 100 视为 significant element |
| IT: Item Index | IT = SP + T | 测验内容或排序价值。IT > 50 可用于 rank applicants |
| TR: Training Value | TR = S + T + SP' - B，P' 是 P 的反向 | 培训价值。TR > 75 进入 training subelement |

TV 的直觉：

- 要选能区分优秀员工的 element。
- 不要选所有人都已经具备的 element。
- 不要选太稀缺、导致岗位招不满的 element。
- 如果漏掉会出大问题，则提高优先级。

TR 的直觉：

- 优秀者具备且缺失会出问题，值得教。
- 如果劳动市场已经普遍具备，就不一定需要培训。
- 如果勉强合格者也都具备，就不太像培训重点。

## 五类 element/subelement

| 类别 | 条件 | 意义 |
|---|---|---|
| E: Element | TV >= 100 | 重要选拔 element；如果标 E，就不再标其他字母 |
| S: Significant subelement | IT >= 50 | 可用于 applicant ranking |
| TS: Training subelement | TR >= 75 | 可进入培训内容 |
| SC: Screenout | B 与 P 都至少 75，且 T >= 50 | 最低要求，未满足可淘汰 |
| RS: Rankable screenout | 同时满足 S 与 SC | 最低要求之上还能排序 |

## Table 3.2 警察岗位例子

课本给出 police officer 的局部 JEM 结果。适合教学的例子：

| Element | 类别 | 教学用途 |
|---|---|---|
| Have sufficient height to see over roofs of cars | SC | 可以转成简单 work sample 或身高标准，但会带来 disparate impact 风险 |
| Ability to change a car tire | SC | 比较适合转成 work sample |
| Have honesty | RS | 数学上可排序/筛掉，但如何有效测量并不清楚 |
| Possess good judgment | E | TV 高，说明对整体选拔价值强 |
| Ability to function while in physical danger | E | 体现警察工作的高风险 element |

讲法律风险时，用"警察身高"提醒：SME 认为是 screenout 不等于法律上必然稳妥。若筛掉比例对某群体明显不利，必须回到 job relatedness 与业务必要性。

## JEM 和 work sample tests

JEM 的优势在于：subelements 可以直接进入测验内容，尤其是 work sample test。课本强调 JEM 不是从 job analysis 推断测验，而是让测验包含 subelement 的工作样例定义。

例子：

- 电工考试中不合格者通过了某部分，分析师回头发现好电工能使用与维护特定电气设备，于是把设备纳入考试，问题得到修正。
- "换车胎"可以直接转成现场样本测验。
- "诚实"难以直接转成有效工作样本，可能需要其他测量方式与法律审查。

## J-coefficient

**J-coefficient** 是 Primoff 提出的、用 job elements 估计某个 test 可能有多大 validity coefficient 的思路。它适用于传统验证研究不容易做的情境。

需要三类信息：

1. job elements 与总体 job performance 的相关。
2. tests 与 job elements 的相关。
3. job elements 彼此之间的相关。

Primoff 的原始思路是让 SMEs 判断 element-performance 关系，并通过长期研究积累 test-element 与 element-element 表。Hamilton 与 Dickinson (1987) 发现，多种生成 J-coefficient 所需信息的方法能得到比较一致的估计，并且与同样样本上的 empirical validity coefficient 接近。

教学提示：J-coefficient 可以解释成"当实证效标验证难做时，用结构化判断来预测测验效度"。

## 优点

- 现场语言强，SMEs 容易理解。
- elements 与 subelements 很容易转成 work samples、培训内容或绩效材料。
- 能把 selection、training、screenout 分开看。
- 对 job analysis 产品有反馈循环：测验不好用时能回查 job analysis。
- 对 KSAO 语言的传播影响很大。

## 限制

- 公式复杂，课本也指出并没有坚实证据说明复杂公式总是必要。
- 严重依赖 SME 判断，可能把偏见或过时做法固化成 screening tools。
- 当 JEM 用来选择 off-the-shelf tests 时，就不得不重新推断抽象能力，与其行为主义优势有张力。
- 对社会赞许或难以观察的特质，如 honesty，测量路径不清晰。
- screenout 类 element 尤其需要法律与 adverse impact 审查。

## 用于回答问题的模板

当用户问"JEM 是什么"：

> JEM 是一种用 SME 语言描述 worker requirements 的方法。它把岗位成功所需的行为证据写成 elements/subelements，再让 SMEs 对 B/S/T/P 四个维度评分，最后推导 TV、IT、TR，用于选拔、排序、培训或 screenout。

当用户给一个 element 让你判断：

1. 先问用途：选拔、培训、最低资格还是排序？
2. 看它更像 E、S、TS、SC 还是 RS。
3. 若涉及身高、体力、年龄、性别相关代理指标，提醒 legal/adverse impact 风险。
4. 若是抽象人格或道德词，追问可观察证据与测量方式。
