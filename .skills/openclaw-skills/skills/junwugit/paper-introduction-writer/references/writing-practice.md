# §1.6 Writing the Introduction（实战写作模式）

本文件收录 Glasman-Deal《Science Research Writing》第 1 单元 §1.6 的可操作规则，用于把前面学到的 model、language、writing skills 和 vocabulary 合并成一篇完整 Introduction。它不是新的结构模型，而是一个**执行模式**：先规划，再按四组件写出可读、有效、符合目标文章句型的引言。

## 何时读取本文件

用户出现以下任一需求时读取：

- 提到 "1.6 Writing the Introduction"、"Exercise 5"、"model answer / key"。
- 要求写 250-350 words 左右的短 Introduction。
- 给出一个假想研究项目，要求按 Glasman-Deal 的 streamlined model 写引言。
- 题目是 "A cover for the single-person pedal-powered vehicle"，或主题涉及 bicycle cover / SPPPV / cyclist safety / rain cover / pollution protection。

## §1.6 的核心写作要求

- 写作前先把 Introduction 按段落规划好，不要边写边决定结构。
- 使用 streamlined Introduction model：即使篇幅很短，也要包含 C1 背景、C2 研究地图、C3 gap、C4 本文。
- 语言要来自本单元的 Language section：前人研究连接语、gap 信号词、本文引入句、贡献标记词。
- 句型应尽量接近目标文章、书中示例引言、以及 `references/language-toolkit.md` 中的句型。
- 如果用户提供 target articles，优先模仿其 Introduction 的句子长度、段落节奏、时态和引用方式。
- 如果用户没有提供 target articles，使用 `references/templates.md` 的相近领域模板，并参考 `references/examples.md`。

## Exercise 5：SPPPV / Bicycle Cover 练习设定

练习场景：

- 假设作者刚完成一个研究项目，目标是设计一种 bicycle cover / SPPPV cover。
- 该 cover 可以保护骑行者免受 injury、pollution 或 rain 的影响。
- 研究角度可以自由选择：computer simulation、ventilation system modelling、aerodynamics、polymer construction、结构设计，或其他相关方面。
- 论文标题固定为：**A cover for the single-person pedal-powered vehicle**。
- Introduction 目标长度约 250-350 words。
- 必须尽量贴近四组件模型，并使用本单元语言资源。
- 练习题允许创造 fake research references；真实论文写作时不能这样做。

## 推荐段落规划（SPPPV 题目）

### Paragraph 1 — Component 1：宽端背景 + 一般问题

功能：

- 从城市空气污染、交通政策、低排放通勤等宽泛议题切入。
- 说明空气质量与车辆排放相关，并用政策目标或社会需求增强重要性。
- 将焦点收束到 Single-Person Pedal-Powered Vehicle (SPPPV)。
- 段末用 However / Although 引出一般问题：销量增长不足以自动带来大规模采用，safety and comfort issues 仍需解决。

可用句型：

```text
[BROAD ISSUE] has become a central issue in [POLICY / INDUSTRY / FIELD].
[OUTCOME] is linked to [CAUSE], and [POLICY / MARKET] aims to [GOAL] within [TIME].
As a result, much research has focused on developing [SOLUTION CLASS] such as [SPECIFIC SYSTEM].
However, although [POSITIVE TREND], [GENERAL PROBLEM] needs to be addressed if [LARGER GOAL] is to be achieved.
```

### Paragraph 2 — Component 2 + Component 3：研究地图 + gap

功能：

- 先用 topic sentence 概括前人已经研究 safety / comfort。
- 按时间顺序列举 safety 方向代表研究，例如 safety bars、reinforced polymer screen。
- 再转到 comfort / protection 方向，例如 rain/sun umbrella、mesh filter。
- 段末用 However 锁定 gap：这些设计可能牺牲 aerodynamic efficiency，形成 safety/comfort 与 aerodynamics 的 trade-off。

可用句型：

```text
Researchers have studied and improved many [ASPECTS] of [SYSTEM].
In [YEAR], [AUTHOR] responded to the need for [NEED] by [DESIGNING / INTRODUCING] [SOLUTION].
The issue of [SECOND ASPECT] has also been addressed by [GROUPS]; [AUTHOR] introduced [FEATURE], and more recently [AUTHOR] added [FEATURE].
However, it has been suggested that [FEATURE_1] and [FEATURE_2] negatively affect [PERFORMANCE METRIC].
```

### Paragraph 3 — Component 4：本文

功能：

- 用一句或两句介绍本文，保持短而密。
- 至少包含 aim、method outline、contribution。
- 对 SPPPV 题目，核心动作通常是 model / simulate / evaluate aerodynamic effects，并据此 propose a new design。

可用句型：

```text
In this study, we [MODEL / SIMULATE / EVALUATE] [EFFECT] and use the data thus obtained to propose [NEW DESIGN] which balances [DESIGN PARAMETERS] with [OPTIMUM PERFORMANCE].
```

## 9 句式映射（SPPPV Key 的结构）

| 句号 | 组件 | 功能 |
|---|---|---|
| S1 | C1 | 用城市空气污染 / 交通政策建立主题重要性 |
| S2 | C1 | 提供空气质量、车辆排放、政策目标等背景事实 |
| S3 | C1 | 把研究焦点从交通政策收束到 SPPPV |
| S4 | C1 bridge | 用 However / Although 提出 safety and comfort 的一般问题 |
| S5 | C2 | 开启研究地图：前人已改进 SPPPV 的 safety aspects |
| S6 | C2 | 列举 safety 方向的代表性研究 |
| S7 | C2 | 列举 comfort / protection 方向的代表性研究 |
| S8 | C3 | 用 However 锁定 gap：既有 safety/comfort 方案削弱 aerodynamic efficiency |
| S9 | C4 | 介绍本文：建模 + 用数据提出 balanced / optimum design |

## 这个 model answer 示范了什么

- **紧凑三段式**：第一段 C1，第二段 C2+C3，第三段 C4。
- **短引言也要有四组件**：不是因为篇幅短就省略 literature review 或 gap。
- **gap 可以是 trade-off**：不是所有 gap 都是"无人研究"；工程论文里常见的 gap 是前人解决 A 问题后损害了 B 性能。
- **文献可以按时间推进**：1999 -> 2002 -> 2008 -> more recently，使读者看到设计演进。
- **本文段可以只有一句**：但这句话必须同时包含 aim、method outline 和 contribution。
- **贡献标记词要克制**：对于平衡型设计，new、optimum、balanced 往往比 first、breakthrough 更合适。

## 生成时的自查

- [ ] 是否先从宽泛问题切入，而不是直接描述 cover 的材料或仿真细节？
- [ ] SPPPV 或其他缩略词是否在第一次出现时展开？
- [ ] 前人研究是否形成叙述线索，而不是引用清单？
- [ ] gap 是否用 However / Although / Despite 等信号词清楚标出？
- [ ] C4 是否回答了 C3 的 gap，而不是另起一个无关研究目标？
- [ ] 如果使用 fake references，是否只发生在练习模式中？
- [ ] 若输出中文版，是否逐段说明功能、逐句解释 function 而非复述内容？
