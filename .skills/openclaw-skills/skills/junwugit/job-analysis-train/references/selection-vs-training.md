# Selection Versus Training

## 核心区别

Selection 和 training 都关注工作中的重要任务，但解决路径不同：

- **Selection**：入职前识别已经具备所需 KSAOs 的人。
- **Training**：入职后通过教学让人获得所需 KSAs。

课本的关键句可转述为：selection 是让人匹配工作，training 是改变人来匹配工作。

## 二者共同关注的任务

两者通常都关注以下任务：

- 关键或重要任务。
- 发生频率高的任务。
- 难学或难做的任务。
- 错误后果严重的任务。

因此，不能只凭"重要"决定选拔还是培训；重要任务既可能要选拔，也可能要培训。

## 决策规则

用下面问题判断某项 KSA 更适合 selection 还是 training：

| 问题 | 倾向 selection | 倾向 training |
|---|---|---|
| 申请者池中多数人是否已经具备？ | 是 | 否 |
| 入职第一天是否必须具备？ | 是 | 否或可逐步掌握 |
| 该 KSA 是否稳定、难训练？ | 是，如一般能力、人格倾向 | 否，如特定知识、设备操作 |
| 组织是否想招熟手？ | 是 | 否，想按"公司方式"培养新人 |
| 错误代价是否高到不能边做边学？ | 可能需要先选拔或认证 | 也可能需要正式培训和认证 |

## Receiver Job 示例

接收员（receiver）工作中：

- **Perceptual speed and accuracy**：更像能力，适合 selection。
- **Arithmetic knowledge**：如果申请者一般已具备，适合 selection；若目标人群缺乏，也可以 training。
- **Forklift operation**：多数申请者可能不会开叉车，且有安全要求，适合 training 和 certification。

这个例子显示，同一份工作中不同 KSA 可以有不同处理方式。

## 方法选择的含义

如果目标是 selection：

- 需要 worker attributes / KSAOs。
- 也需要 task information 来支持 content validation、criterion development、work sample creation 和 legal/professional requirements。
- 可用 C-JAM、critical incidents、job elements、PAQ、TTAS、ARS、O*NET 等。

如果目标是 training：

- 需要更详细的 task content。
- 需要 task and KSA analysis 来设计内容。
- 需要 person analysis 来确定谁缺什么。
- 需要 instructional objectives 和 evaluation criteria。

## 课堂应用模板

分析一个案例时按四步：

1. 列出关键任务。
2. 列出每项任务背后的 KSA。
3. 判断目标申请者是否通常已有这些 KSA，以及是否入职即需。
4. 决定每项 KSA 是 select、train、还是二者结合。

示例句：

> 这个 KSA 不应自动写成招聘门槛。先问 applicant pool 中是否现实地具备它，以及组织是否愿意训练。如果多数合格申请者没有它，但它可以通过短期课程掌握，那么第 8 章会更支持把它放进 training，而不是作为 selection screen。
