# Selection 与 Test Validation

## Selection 的基本逻辑

Selection 是从申请者中选择组织想雇用的人。课本把逻辑拆成三步：

1. 分析工作，识别 tasks 和 KSAOs。
2. 评估申请者在 tasks 或 KSAOs 上的水平。
3. 把最可能表现好、或至少达到最低资格的人录用。

Selection 的复杂性来自两边：一边是法律和公平性约束，另一边是心理测验及其效度证据。

## Tests 的范围

课本把 employment test 理解得很宽：

- **Maximum performance tests**：能力、知识、推理等测试，通常有正确答案。
- **Typical performance tests**：人格、兴趣等，测典型行为倾向。
- **Work samples / simulations**：直接做工作样本，如焊接、打字、叉车操作、assessment center。
- **Interviews**：不管是否结构化，只要用于收集和评价信息并影响选拔，也属于测试/选拔程序。

## Test validation 的核心

Psychologists 讲 validity，法律语境常讲 job relatedness。就业测试中，test validation 是建立"测试分数对目标工作意味着什么"的证据链。

讲解时用这个句式：

> Validity 不是说一个 test 永远有效，而是说在某个用途、某个工作、某类推论中，test scores 有足够证据支持。

如果测试造成 adverse impact，雇主通常需要证明测试与工作相关并符合 business necessity。现实合规问题需另行核对法律和专业标准。

## Content-Oriented Strategies

Content-oriented validation 把测试内容和工作内容相连，最适合：

- work samples。
- simulations。
- job task knowledge tests。

它的论证最强时，test content 直接抽样自重要工作任务。例如 receiver job 中，用 packing list 核对零件、用 forklift 移动托盘，都与实际工作任务高度相似。

为 content-oriented strategy 服务的工作分析通常要关注：

- tasks 和 work activities。
- products/services。
- machines, tools, equipment, job aids。
- performance indicators and standards。
- 任务重要性、criticality，以及是否覆盖大部分 critical tasks。

注意：content evidence 主要支持 test stimuli/content 与工作相连，并不直接用数据证明 test scores 与 job performance 的关系。因此，高风险场景中常需要补充 criterion-oriented evidence。

## Criterion-Oriented Strategies

Criterion-oriented validation 用数据检验 test scores 与 criterion scores 的关系。criterion 通常是 job performance，也可以是 turnover、错误率、客户评价等结果。

常见设计：

- **Predictive design**：先测申请者，录用后过一段时间再收集 job performance。若研究用测试但暂不用于实际筛选，能更清楚看到测试影响；但实践中不常见。
- **Concurrent design**：同时测现有员工的测试分数和绩效。较快、较实用，但现有员工不一定代表未来申请者。

样本量提醒：小样本下相关系数不稳定。课本经验性地提醒，少于 100 人的 validation study 可疑，250 人以上更理想，但很多组织没有这么多同岗位员工。

## Construct-Oriented Strategies

Construct-oriented validation 关注测试分数在理论网络中的意义。它会检验 focal measure 与其他理论相关变量的关系模式。

例如，若开发外科医生焦虑测量，可检验它是否与既有焦虑测量正相关、与工作满意度负相关、与模拟外科表现相关。它在科学上信息量大，但组织实践中较少单独实施，因为组织通常更关心能否预测绩效。

## Synthetic Validity 与 Validity Generalization

当单一岗位人数不足时，可考虑 synthetic validity：不同工作或任务可能需要相同属性，合并跨岗位信息来研究这些属性与绩效的关系。

另一种思路是从类似工作或外部研究借用效度证据。Validity generalization 则汇总许多研究，判断某类测试是否跨工作普遍预测绩效。课本指出这种做法有争议，保守立场会要求目标工作与已有证据中的工作和情境足够接近。

## Signs vs Samples

- **Sample**：测试看起来就是工作的一部分，如 forklift driving work sample。
- **Sign**：测试是某种属性的指标，如 conscientiousness questionnaire。

样本通常更容易用 content-oriented evidence 支持；signs 通常更依赖 criterion-oriented evidence。无论哪一种，最终都在用 test scores 预测 job performance。

## 判断 KSAOs 的研究结论

课本总结：了解工作、使用系统程序、由专家小组而非单个判断者做判断，可以提升 KSAO 判断的可靠性和有效性。

实用要点：

- 单个专家对 KSAO 重要性的判断可能一致性低。
- 约 10 人左右的 expert panel 可显著提升平均判断的可靠性。
- 观察实际工作通常比只读 job description 更有助于准确判断。
- PAQ、TTAS、ARS、O*NET 等标准化方法便于跨岗位比较，但工作任务细节较少。

## Selection 中有用的工作分析方法

- **C-JAM**：先形成全面 task list，再形成完成任务所需 KSAOs；同时支持 criterion/content evidence 和 test choice。
- **Critical Incident Technique**：收集优秀/糟糕绩效事件，可形成绩效维度和 KSAO 维度。
- **Job Element Method**：从工作专家语言中的成功要素出发，可支持测试选择和评分表。
- **PAQ / TTAS / ARS / O*NET**：提供标准化 worker requirements，适合跨岗位比较和测试选择，但不提供足够任务内容。
