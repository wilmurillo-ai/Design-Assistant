# Staffing、Recruitment 与 Job Specification

## Staffing 的基本逻辑

Staffing 是把人和工作相匹配。课本把 staffing 放在 talent management 的大框架下，强调三个功能：

- **Recruitment**：吸引人来申请。
- **Selection**：从申请者中选择组织愿意雇用的人。
- **Placement**：把人放到合适岗位，并帮助其从外部人转为有效员工；本章较少展开。

从 staffing 角度看，工作通常被视为固定对象。组织不是先改变工作来适配个人，而是先弄清工作需要什么 KSAOs，再找具备这些属性的人。

## 从 job 到 KSAO 的推论风险

工作分析用于 staffing 时，核心难点是从具体任务推断人类属性：

1. 从 **job/tasks** 推断 **KSAOs**。
2. 从 **KSAOs** 选择或设计 **tests/assessments**。
3. 再把测试差异合成实际雇用决策。

课本反复提醒：越抽象的 KSAO，推论跳跃越大。例如，会计需要加减法知识是小跳跃；要求特定年限经验、高 verbal ability 或抗压性就是更大跳跃。工作会变化，人的属性分类也不完美，所以很难有一本稳定的"工作-要求字典"。

## Recruitment 的三个功能

招聘信息至少承担三个功能：

- 让人知道职位空缺，包括触达多元背景群体，支持 EEO 目标。
- 吸引合适的人申请，也就是把组织作为好雇主来营销。
- 告知潜在申请者工作要求、工作性质和组织特征。

对话中解释 recruitment 时，注意它有双重目标：一方面要吸引，另一方面要筛选。工作分析最直接支持的是"告知要求"和"自我筛选"。

## Job Specification

**Job specification** 是对工作要求的书面描述，通常从 job description 和 job analysis 推导而来。它可能包括：

- KSAOs，如 verbal ability、resistance to stress、specific job knowledge。
- 资质要求，如执照、证书、保险条件。
- 工作情境、个人工作负荷、worker activities、人类属性要求。

关键原则：job specification 中的要求要能追溯到工作分析。只看 task description 就写学历、经验或最低资格，风险较高；学历和年限经验尤其需要谨慎证明其与工作直接相关。

## 好的招聘工作分析应提供什么

为 recruitment 服务的 job analysis 至少要提供：

- 描述性 job title。
- job purpose 和 major duties。
- 最重要的 worker requirements。
- 可向申请者清楚说明的 minimum qualifications。
- 能支持申请者 self-screening 的信息。

如果用户问"为什么 job description 不够"，可答：job description 常能说明工作做什么，却未必能说明做这份工作需要什么 KSAOs；除非工作分析一开始就是为招聘/选拔要求设计的。

## 与 O*NET / worker-oriented analysis 的连接

课本提到，worker-oriented job analysis 可通过总结最重要的人类属性要求来生成 job specification。这种直接估计（direct estimation）思想也出现在 O*NET 的许多 KSAO 测量中。

## 课堂解释模板

可以这样回答：

> Recruitment 不是只发广告。第 8 章强调，招聘信息还要让申请者理解这份工作真正需要什么。工作分析的作用，是把 job description 中的 duties 转化为可说明、可辩护的 job specification，让合适的人愿意申请，也让不匹配的人可以自我筛选。
