# 核心术语（Chapter 1 "Definitions"）

## 工作内容的谱系：从最宽到最窄

课本用"从两端往中间逼近"的方式定义 job：

### 从宽端往下
- **World of work** — 所有工作的总和。太宽，实务上无用。
- **Branch** — 大约十来个分支。例：*public safety*（公共安全）分支包含执法、安保、消防、救灾等。
- **Group** — 每个 branch 再分约十来个组。例：public safety 下的 *law enforcement*。
- **Series** — 组内的职系。例：*sworn law enforcement officer series* 包含警员、警探、警长、副队长、队长、警监等。
- **Job** — 目标概念。例：*police officer*。

### 从窄端往上
- **Element**（要素） — 可辨识的最小工作单元，有清晰开头中间结尾。再往下就只剩生理动作或感官过程了。例：警察职业中的"拨打电话"。
- **Activity**（活动） — elements 的簇，指向一个工作要求。例：警察"接听房东–租客纠纷报警电话"。一个岗位通常有 100+ 活动。
- **Task**（任务） — activities 的集合，指向具体工作目标。课本建议**用动词开头**描述任务。例："Talks to conflicting parties to settle disturbances."（与冲突各方交谈以平息事端）。典型岗位 30–100 个任务。
- **Duty**（职责） — tasks 的集合，指向工作的一般性目标。例：警察的"vice activities"（涉恶 / 涉毒相关活动）。典型岗位 5–12 个职责。
- **Position**（职位） — 由 duties/tasks/activities/elements 构成，一个员工能做的整套工作。**每个在职员工拥有的是 position，不是 job**。
- **Job**（工作 / 岗位） — 一组彼此足够相似、被组织冠以同一头衔的 positions。

## Job Analysis 的两种定义

**短定义**：工作分析 = 发现一项工作之本质的过程。

**长定义**：工作分析是**系统性**地发现工作本质、将其**拆分为更小单元**、产出**一份或多份书面成果**，目标是描述"这份工作做什么"或"做好这份工作需要什么能力"。

课本强调三个补充要素：
1. **系统性流程** — 有预先规定的步骤，执行人称为 *job analyst*。
2. **拆分为更小单元** — 可以是 duties/tasks/activities/elements，也可以是其它实体（如视觉追踪要求、问题解决要求、握长柄工具的要求等）。
3. **书面成果** — 完整报告、岗位说明书、任务清单、任职资格说明等。

## 易混点

- **Work analysis vs job analysis**：work analysis 更宽，涵盖团队运作、流程、系统的分析；但课本"照惯例"统一用 job analysis。
- **Don't over-distinguish 层级**：怎么区分 activity / task / duty？——课本原话："don't worry; be happy." 切多细取决于用途：做 job evaluation 用粗，做脱产培训用细（甚至到 element 级）。

## KSAO / Competency

- **KSAO** = Knowledge, Skills, Abilities, Other characteristics（含人格等）。例：会计岗需要"使用当前税务软件的技能"。
- **Job specifications**（任职资格）= 雇主对某岗位的最低硬门槛（如工程学士、6 个月收银经验）。
- **Competency**（胜任力）= 课本引 Sanchez & Levine (2012)："有助于交付战略绩效结果的一组行为或行为主题"。通常跨多个岗位共享一组 competency，但具体行为表现因岗而异（程序员的"创新" vs 销售的"创新"长相不同）。
