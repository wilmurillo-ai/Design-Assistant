# Sources and Methods

## Sources of Job Analysis Data

### Incumbents

Incumbents 仍可能是未来最常用的数据来源，因为他们通常最了解工作。尤其在知识工作、虚拟工作和认知活动不可直接观察的场景中，jobholders 的解释仍非常关键。

局限：

- incumbents 不总是自己工作的最佳评估者
- 对噪音、身体疲劳、压力、学习难度等特征，incumbents 可能低估或判断不准
- 未来工作差异更大，平均多个 incumbents 的回答可能掩盖有意义的 role variation

### Customers

随着服务经济和 customer focus 增强，外部客户和内部客户都会成为更重要的数据来源。

可用方式：

- customer focus groups
- mystery shoppers
- internal customer panels
- 从完整 work process 角度组织跨部门人员讨论输入、输出和流程问题

适用场景：

- customer service job analysis
- police/community-facing roles
- retail/service encounters
- 内部流程中上游岗位产出直接影响下游岗位的工作

### Specialists

Specialists 在某些判断上可能比 incumbents 更准确：

- trainers/educational specialists：更适合判断 difficulty to learn
- psychologists：更适合判断 stress tolerance 等心理要求
- ergonomics/safety/noise experts：更适合判断身体负荷、噪音、伤害风险
- job analysts：可校准 incumbents 没有意识到或习以为常的工作条件

Expert panels 也可用来快速、低成本地获得信息。已有研究显示 panel 数据可能接近大样本问卷数据，但 panel 产物是否等价、panel composition 如何影响结果仍需研究。

### Computers, AI, and Big Data

未来 computers 可能不仅是工具，也会成为 job analysis data source：

- 系统日志显示员工使用哪些程序、文件和信息资源
- 传感器、手机、GPS、可穿戴设备记录位置、行动和工作状态
- text/web analytics 解释工作内容和信息流
- simulation models 预测工作设计变更的影响
- AI/machine learning 可能从大量行为数据中识别 task patterns

教学提醒：电脑数据更细、更自动化，但不自动等于理解"为什么"。解释行为目的、情境意义和伦理边界仍需要人类判断。

## Methods of Data Collection

传统 observation、interviews、questionnaires、diaries、records 仍会继续使用；技术变化主要扩展和改变方法组合。

### Computer Networks

网络会改变问卷和专家小组：

- paper-and-pencil task inventories 转向 web-based surveys
- respondents 直接录入数据，降低印刷、邮寄和录入成本
- geographically dispersed SMEs 可以在 virtual spaces 中开会
- 电脑搜索系统可快速汇总既有 job/occupation information

注意问题：

- 不同设备上的视觉呈现不一致
- respondent anonymity 需要保护
- over-surveying 可能降低回应意愿
- data integrity 需要防止无效回应和数据库被攻击
- 网络问卷的回应率可能低于纸本问卷，且 job analysis 问卷通常较长，不能简单套用短态度量表研究结论

### Electronic Performance Monitoring (EPM)

EPM 使用 computers 和 peripheral devices 监控工作与表现。例如：

- 车辆速度、怠速时间、位置
- keystrokes、mouse movements、timestamp
- 摄像头、屏幕记录、电话录音
- 系统自动识别员工正在执行的活动

优势：

- 比自我估计更细、更可靠地记录 frequency、duration、sequence
- 适合分析电脑化工作、物流、客服、远程工作等
- 可支持更实时的 work process analysis

伦理风险：

- 员工通常会把隐蔽监控视为侵入
- 需要考虑知情、同意、隐私、数据用途边界和信任
- 监控数据用于 job analysis、绩效管理、纪律处分时，伦理和法律风险不同

### Forecasting

当岗位还不存在时，不能直接观察或访谈 incumbents，只能预测 future tasks 和 future requirements。

方法：

- SME panels 识别并评价 current and future tasks
- hypothetical scenarios 提供 demographic、social、economic、political、technological trends，让专家推测未来工作
- simulations 模拟新技术、新环境或新流程下的工作
- retrospective job analysis 用于回溯过去某个时间点的岗位要求，常见于法院案件或 EEOC 调查

注意问题：

- forecasting 依赖专家的 mental representation，可能有偏差
- reliability、bias、consequential validity 需要研究
- retrospective accounts 也需要小心验证其可靠性和效度

## 课堂回答模板

如果用户问"未来最受技术影响的是哪个 building block？"：

> Methods of data collection 最直接受影响。网络问卷和虚拟小组改变了传统问卷/会议方式，EPM 和系统日志让电脑直接收集行为数据，simulation 和 forecasting 则让分析可以面向尚未出现的工作。

如果用户问"电脑会取代 incumbents 吗？"：

> 不应简单说取代。电脑能记录行为痕迹和频率，但 incumbents 仍更了解工作目的、情境判断、例外情况和隐性认知过程。未来更合理的是多源整合。
