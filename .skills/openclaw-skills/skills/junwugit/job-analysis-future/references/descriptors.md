# Future Descriptors

## 总体变化

Descriptors 是工作分析中要观察和记录的工作特征或人员要求。第 10 章认为，未来工作更灵活、更复杂、更团队化、更受技术影响，因此 descriptors 需要扩展到：

- 更宽的人员属性和 competencies
- 人际关系质量
- 团队、客户服务与跨文化互动
- roles 而不只是 tasks
- job design features
- tasks、concepts、actions 之间的 connections
- 与核心业务、外包和客户结果相关的新 rating scales

## Flexibility

未来岗位边界会更少、更模糊：

- workers 与 management 的边界减少
- expertise 与 organizational functions 之间的边界减少
- tasks 经常变化，甚至难以预先预测

工作分析的回应：

- 如果只是岗位轮换或职责包变大，可以把多个相关岗位合并成一个更包容的 job 来分析。
- 如果任务变化很快且难以预测，传统 task list 会不足，需要转向 broader work requirements，如 conscientiousness、adaptability、learning ability、problem solving 或 competencies。
- 但不能走向"只列普遍优秀品质"。如果完全不说明工作如何完成，分析对 job design、培训和法律防御会变弱。

## Interpersonal Relationships

传统 job analysis 常能列出 tasks，却不一定能抓住"关系质量"。本章强调以下场景：

- **Teams**：成员不同角色、共同责任、相互依赖。分析应关注成员依赖关系、shared mental models 和角色理解。
- **Customer service**：服务质量不仅是完成什么任务，也包括服务如何被交付。可能需要 service orientation、tactful questioning、emotional stability、emotional labor 等 descriptors。
- **Culture**：全球化和多元团队使跨文化互动更重要。相关 KSAOs 包括 flexibility、openness、social/perceptual skills、local customs knowledge，以及对 power distance、collectivism 等文化规范的理解。

可用 descriptor 来源：

- personality characteristics：agreeableness、sensitivity to others、preference for working with others
- interpersonal skills：serving、exchanging information、coaching、persuading、instructing、consulting、supervising、negotiating、mentoring、leading
- O*NET social skills 和 generalized work activities

## Roles

Roles 提供比 task list 更灵活的语言。Role 是某个群体位置中被期待的行为模式，能连接个人行为、他人期望和工作情境。

### Mintzberg managerial roles

| 大类 | 具体 roles | 用法 |
|---|---|---|
| Interpersonal | figurehead、leader、liaison | 分析管理者如何与人互动、代表组织、连接单位 |
| Informational | monitor、disseminator、representative | 分析管理者如何收集、过滤、传递信息 |
| Decisional | entrepreneur、disturbance handler、resource allocator | 分析管理者如何发现机会、处理扰动、分配资源 |

### Mumford/Campion/Morgeson team roles

| 大类 | Roles | 核心含义 |
|---|---|---|
| Task roles | contractor、creator、contributor、completer、critic | 组织任务、提出新方案、提供专长、执行承诺、质疑假设 |
| Social roles | cooperator、communicator、calibrator | 支持团队决策、维护协作氛围、观察并调整团队过程 |
| Boundary-spanning roles | consul、coordinator | 获取外部资源、代表团队、与外部相关方协调 |

教学提醒：team roles 被定义得很行为化，因此可与 task/domain descriptors 连接，而不是停留在抽象人格标签。

## Job Design Features

Campion 建议未来工作分析纳入 job design features。可借第 4 章 MJDQ 的四类：

- **Motivational**：autonomy、task significance、skill variety 等。
- **Mechanistic**：simplification、repetition、specialization 等。
- **Biological**：physical demands、endurance、ergonomic load 等。
- **Perceptual/motor**：lighting、information display、attention demands 等。

这些 features 可用于：

- 识别 selection criteria
- 预测员工差异会如何影响表现
- 评估 job changes 的影响
- 设计新岗位
- 改善系统可靠性和工作流程

## Connections

传统工作分析常产出 tasks 或 KSAOs 的列表，但较少说明它们之间如何连接。未来更需要分析连接关系：

- **Mental representations / schemas**：专家如何理解问题、分类情境、生成假设。
- **Cognitive task analysis**：用于不可直接观察的知识工作，揭示概念、任务和行动之间的关系。
- **Concept mapping**：用相似性、距离、接近性表示概念连接。
- **Flowcharts/process maps**：表示任务流程、系统状态和决策点。
- **Time charts**：表示顺序、并行和时间关系。

应用价值：

- 提供测验内容
- 支持 selection validity evidence
- 形成 training content
- 改善 project/team/job design

## New Scales

传统量表常评估 importance、frequency、difficulty to learn、是否需要入职前具备等。未来可能需要新量表：

- task 对组织核心功能的影响程度
- task 是否适合 outsourcing
- job 或 task 与战略使命的关系
- task 对 customer service outcomes 的影响
- task 是否需要保留在组织内部以维持核心能力

教学提醒：new scales 的价值在于把 job analysis 和组织战略、外包决策、客户结果连接起来；但这些新量表需要研究其 construct validity。
