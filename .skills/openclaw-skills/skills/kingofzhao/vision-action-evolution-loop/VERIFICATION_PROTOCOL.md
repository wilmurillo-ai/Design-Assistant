# VERIFICATION_PROTOCOL.md — vision-action-evolution-loop

## 验证目标

确保视觉-动作-进化五阶段闭环真正可自迭代，且每个阶段都有独立验证机制。

## 五步自验证流程

### Step 1 — 五阶段完整性
- [ ] 是否定义了全部5个阶段（Perceive/Plan/Execute/Evaluate/Evolve）？
- [ ] 每个阶段是否映射到SOUL五律中的一条？
- [ ] 阶段之间的数据流是否明确（输出→输入）？
- **通过标准**: 5/5阶段定义 + SOUL映射 + 数据流闭环

### Step 2 — 三阶段桥接验证
- [ ] 2D检测阶段是否引用已验证的diepre-vision-cognition？
- [ ] 3D空间理解是否有学术参考文献支撑？
- [ ] 动作规划阶段是否区分"工具增强"和"端到端VLA"两条路径？
- **通过标准**: 3/3阶段有引用支撑，路径选择有论证

### Step 3 — 工具增强策略验证
- [ ] 是否将现有OpenCV管道封装为可调用工具（而非替换）？
- [ ] VLA模型调用工具的方式是否有学术参考？
- **通过标准**: 工具封装架构完整 + 学术支撑

### Step 4 — 自迭代闭环验证
- [ ] `inject_feedback()` 是否能将评估阶段的反馈注入进化阶段？
- [ ] 多次迭代后内部世界模型是否有实质性更新（非重复）？
- [ ] 是否记录每次迭代的认知增量到文件？
- **通过标准**: 反馈→进化链路完整 + 文件记忆持久化

### Step 5 — 置信度 + 节点关系验证
- [ ] 每个阶段输出是否独立标注置信度？
- [ ] 是否明确定义了与父/子/兄弟Skill的节点关系？
- [ ] 综合置信度 < 70% 时是否触发人工介入？
- **通过标准**: 阶段级置信度 + 节点关系图 + 人工介入机制
