## 五步自验证

### Step 1 — RAPVL循环完整性
- [ ] 五步（Review/Analyze/Plan/Verify/Learn）是否都有明确定义？
- [ ] 每步是否有可执行的输出？
- [ ] 循环是否可自动触发（非手动）？
- **通过标准**: 5/5步定义 + 可执行输出 + 自动触发

### Step 2 — 成长度量化
- [ ] Growth Score公式是否明确定义？
- [ ] 是否支持不同职业的差异化度量？
- [ ] 是否区分成长/停滞/退步三种趋势？
- **通过标准**: 公式明确 + 差异化度量 + 趋势判断

### Step 3 — 职业适配层
- [ ] 是否提供≥5个职业的预设配置？
- [ ] 每个配置是否包含全部5个RAPVL要素？
- [ ] 是否支持自定义配置？
- **通过标准**: ≥5预设 + 5要素完整 + 自定义支持

### Step 4 — 自进化机制
- [ ] 是否定义了引擎自身的进化循环？
- [ ] 进化是否由Growth Score数据驱动（非随机）？
- [ ] 是否有"深度反思"模式（停滞时触发）？
- **通过标准**: 元进化循环 + 数据驱动 + 停滞反思

### Step 5 — 通用性验证
- [ ] 是否不依赖任何特定领域知识？
- [ ] 程序员和科研人员都能直接使用？
- [ ] 与programmer-cognition和researcher-cognition的关系是否清晰？
- **通过标准**: 领域无关 + 多职业适用 + 关系清晰
