# VERIFICATION_PROTOCOL.md — universal-occupation-adapter

## 五步自验证

### Step 1 — 五维度模型完整性
- [ ] 5个维度（known_sources/unknown_types/verification/memory/redlines）是否都有定义？
- [ ] 每个维度是否映射到SOUL五律中的一条或多条？
- **通过标准**: 5维度完整 + SOUL映射清晰

### Step 2 — 预设模板质量
- [ ] 是否有≥6个职业的完整预设模板？
- [ ] 每个模板是否5个维度都填充？
- [ ] 模板内容是否与职业实际匹配（非泛泛而谈）？
- **通过标准**: ≥6模板 + 5维度完整 + 职业准确性

### Step 3 — 自动生成能力
- [ ] 无预设模板时是否触发四向碰撞？
- [ ] 四向碰撞是否产出职业特有的认知（非通用套话）？
- [ ] 生成结果是否自动运行自验证？
- **通过标准**: 四向碰撞触发 + 职业特有 + 自动验证

### Step 4 — 批量生成
- [ ] 是否支持批量生成多职业Skill？
- [ ] 批量生成是否有数量控制？
- [ ] 每个批量生成是否独立验证？
- **通过标准**: 批量支持 + 数量控制 + 独立验证

### Step 5 — 无限扩展性
- [ ] 是否明确说明支持任意职业？
- [ ] 新生成的模板是否可保存为预设？
- [ ] 预设库是否可持续增长？
- **通过标准**: 任意职业 + 模板持久化 + 可持续增长
