# 地产故事线 · 专属知识库

## 使用规则

生成品牌故事线前，**优先检索本知识库**，匹配同类型案例作为写作参照。
优先级顺序：**内置知识库 > web_search 检索 > 内置案例库（兜底）**

检索逻辑：
1. **先读方法论**：进入 `故事线方法论/` 目录，读取 `01_切入角度选择.md`，确定本项目的主切入角度。根据项目特点按需读取其他方法论文件（历史溯源/国际案例/情感结构等）。
2. **再找案例**：根据「产品定位」字段进入对应项目类型目录，浏览案例的「核心标签」和「适用场景」，选取1-2个最匹配的案例。
3. **融合写作**：以方法论为框架，以案例为参照，结合项目具体信息进行创作。

优先级顺序：**内置知识库（方法论+案例）> web_search 检索 > 内置案例库（兜底）**

## 案例文件格式规范

每个案例为一个独立 `.md` 文件，文件名格式：`城市_项目名.md`
例：`广州_凯旋新世界.md`、`上海_汤臣一品.md`

### 必填字段

```markdown
---
项目名称: 
城市: 
区位: 
开发商: 
产品定位: （刚需 / 刚改 / 改善 / 豪宅 / 顶豪 / 城市或片区营造 / 产业园 / 商业）
产品类型: （住宅 / 综合体 / 文旅 / 商住 / 产业园 / 商业 / 城市更新）
核心标签: （3-5个关键词，如：滨水、低密、历史街区、TOD）
适用场景: （本案例最适合被哪类项目参照，1-2句）
---

## 项目背景
（100-200字：城市区位、开发商背景、地块特征、核心卖点）

## 故事线切入角度
（本项目品牌叙事的核心逻辑是什么？用了什么角度切入？100-150字）

## 核心视觉符号
（项目提炼的视觉符号/IP是什么？符号与生活方式如何连接？50-100字）

## 经典表达摘录
（品牌文案、Slogan、或故事线中最有代表性的1-3句话）

## 营销亮点
（本项目在营销推广上有哪些值得参照的创新做法？100-150字）

## 参照价值
（这个案例对同类项目最大的参照意义是什么？50字以内）
```

### 选填字段（有则填，无则省略）

```markdown
## 销售说辞特点
（如有独特的说辞逻辑或话术亮点，简述）

## 社群运营做法
（如有值得参照的私域或社群运营经验）

## 备注
（其他补充信息）
```
