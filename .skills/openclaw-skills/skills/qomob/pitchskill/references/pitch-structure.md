## 8段式标准结构

这是Pitch Workflow强制使用的结构。每段都有明确的功能、时间控制和表达目标。

| 段落 | 功能 | 时间占比 | 情绪目标 | 关键原则 |
|------|------|---------|---------|---------|
| 1. 开场 | 3秒抓注意力 | 5% | 震撼 | 一句话击中焦虑 |
| 2. 问题重构 | 展示深度理解 | 15% | 理性 | 三层问题递进 |
| 3. 洞察 | 打动情感 | 12% | 共情 | 消费者真相 |
| 4. 策略 | 展示独特性 | 20% | 兴奋 | 不可替代的逻辑 |
| 5. 执行 | 证明可落地 | 20% | 确定感 | 具体到可执行 |
| 6. 结果 | 量化回报 | 12% | 信心 | 决策语言 |
| 7. 风险控制 | 消除顾虑 | 10% | 安心 | 主动提出+兜底 |
| 8. 收尾 | 推动决策 | 6% | 决心 | 回扣开场 |

## Deck页面模板

### 开场段（P1-P3）

**P1: 封面页**
- 标题：用客户的语言描述问题（不是"XX方案提案"）
- 副标题：一句话策略主张
- 视觉：与开场Hook呼应的画面

**P2: Opening Hook**
- 核心冲击句（大字）
- 1-2个支撑数据或事实
- 过渡句引出问题重构

**P3: 从Hook到问题**
- 简短的过渡：从开场冲击到深度分析

### 问题重构段（P4-P8）

**P4: 官方问题**
- 复述Brief的核心需求
- 用客户的原话（展示你在听）

**P5: 表层问题**
- Brief背后的隐含需求
- "你们可能已经意识到了..."

**P6: 本质问题⭐**
- 用一句话揭示真正的问题
- 这是全场最重要的页面之一
- 视觉建议：大量留白，一句话居中

**P7-P8: 问题论证**
- 支持"本质问题"的数据和洞察
- 行业对标

### 洞察段（P9-P12）

**P9: 消费者真相**
- 消费者故事或数据
- "你的消费者实际上在想什么"

**P10: 洞察陈述**
- 一句话洞察（大字）
- 连接消费者真相和品牌资产

**P11-P12: 洞察论证**
- 支持洞察的数据和案例
- 为什么这个洞察只有这个品牌能用

### 策略段（P13-P20）

**P13: Challenge**
- 市场现实是什么
- 为什么需要改变

**P14: Strategic Idea**
- 一句话策略主张（大字）
- 这是全场第二重要的页面

**P15-P16: 策略框架**
- Challenge → Insight → Idea → Framework的可视化
- 展示策略的推导逻辑

**P17-P20: 策略展开**
- 按维度展开（人群/渠道/内容/节奏）
- 每个维度1页

### 执行段（P21-P28）

**P21: 执行时间线**
- 甘特图或路线图
- 清晰的阶段划分

**P22-P24: 各阶段详情**
- 每阶段的核心动作
- 关键里程碑

**P25-P26: 内容和渠道**
- 具体的内容方向
- 渠道组合和分配

**P27-P28: 团队和资源**
- 执行团队配置
- 关键资源配置

### 结果段（P29-P32）

**P29: 预期ROI**
- 量化预期（保守/中性/乐观三档）
- 用决策语言表达

**P30: 对标案例**
- 同行业成功案例
- 可量化的结果

**P31: KPI预测**
- 分阶段KPI
- 达成概率

**P32: 长期价值**
- 不只是短期ROI
- 品牌资产积累

### 风险控制段（P33-P35）

**P33: 主动风险识别**
- 列出Top 3风险
- 展示你比客户更早想到风险

**P34: 风险兜底方案**
- 每个风险的应对措施
- Plan B触发条件

**P35: 效果监控**
- 如何监控执行效果
- 调整机制

### 收尾段（P36-P38）

**P36: 回扣开场**
- 回到Opening Hook
- "回到我们最初的问题..."

**P37: 一句话总结**
- 整场提案浓缩为一句话
- 这句话要让决策者能向老板复述

**P38: 下一步**
- 明确的下一步行动
- 展示推进的紧迫感

## 情绪曲线参考

### 标准曲线（适用于大多数场景）
```
情绪强度
↑
│     ★开场                              ★收尾
│      ╱╲         ★策略    ★结果
│     ╱  ╲   ★洞察 ╱  ╲    ╱  ╲
│    ╱    ╲   ╱  ╲╱    ╲  ╱    ╲
│   ╱      ╲ ╱         ╲╱      ╲★风险
│  ╱  问题  ╱                     ╲
│ ╱      ╱                          ╲
└──────────────────────────────────────→ 时间
```

### Safety型曲线（强调确定感）
- 开场不那么激进，用数据震撼替代焦虑击穿
- 策略段增加数据支撑
- 风险控制段拉长

### Aggressive型曲线（强调冲击力）
- 开场更激进，制造紧迫感
- 策略段放大愿景
- 风险控制段缩短（但必须有）

## 开场模板库

### 焦虑击穿型
```
"过去6个月，你们在{渠道}上的{指标}下降了{X}%。
 你们最大的竞争对手正在{做某事}。
 如果今天不改变方向，{X}个月后你们会{不利结果}。"
```

### 数据震撼型
```
"我们研究了你们过去12个月的{数据维度}。
 发现了一个有意思的数字：{反常识数据}。
 这个数字背后藏着一个{行业趋势/消费者变化}。"
```

### 问题刺穿型
```
"今天我想回答一个问题。
 一个你们可能在想、但没有问出口的问题：
 {本质问题的表述}。"
```

### 反常识型
```
"行业里所有人都在做{X}。
 但我们的研究告诉我们，真正的机会不在{X}，而在{Y}。
 让我解释为什么。"
```

## 收尾模板库

### 回扣型收尾
```
"回到开头的问题——{重复开场Hook}。
 现在我们有了答案：{一句话策略总结}。
 这就是我们今天想分享的。
 下一步，我们建议{明确的行动}。"
```

### 决策推动型收尾
```
"今天你们面前有一个选择：
 继续沿着{现状路径}走，看到{预期结果}；
 或者和我们一起，走{策略路径}，看到{更好的预期结果}。
 我们准备好了。"
```

### 安心型收尾（Safety型客户）
```
"最后我想说：我们理解这个决定对你们的重要性。
 我们准备了{兜底方案}，设定了{触发条件}。
 你们不是在下赌注，而是在做一笔有保障的投资。"
```

## 情绪引擎模板

### 逐段落情感评估表

对8段式结构的每个段落进行情感冲击力评估：

| 段落 | 目标情绪 | 评估维度 | 及格线 | 满分 |
|------|---------|---------|--------|------|
| 开场 | 震撼/紧迫 | 3秒内是否制造情绪反应 | 7/10 | 10/10 |
| 问题重构 | 理性/深度 | 是否让客户觉得"你比他们更懂" | 7/10 | 10/10 |
| 洞察 | 共情/被打动 | 是否用消费者真相打动决策者 | 7/10 | 10/10 |
| 策略 | 兴奋/独特 | 是否展示不可替代性 | 7/10 | 10/10 |
| 执行 | 确定感 | 是否证明可落地 | 7/10 | 10/10 |
| 结果 | 信心 | 是否用量化数据说话 | 7/10 | 10/10 |
| 风险控制 | 安心 | 是否主动提出风险并有兜底 | 7/10 | 10/10 |
| 收尾 | 决心 | 是否推动决策 | 7/10 | 10/10 |

### 文案感染力优化模板

```
文案优化记录:

段落: {段落名}
原文: "{当前文案}"
问题诊断: {感染力不足的具体原因}
  - 可能原因A: 语言太理性，缺乏情感冲击
  - 可能原因B: 信息太多，核心信息被淹没
  - 可能原因C: 缺乏具体场景/数据/案例支撑
优化方向: {应该往什么方向改}
优化后: "{改写后的文案}"
提升点: {改写改变了什么——情感/节奏/具体性/冲击力}
```

### 情感冲击力提升技巧

| 问题 | 技巧 | 示例 |
|------|------|------|
| 太理性 | 加入"你"的直接称呼 | "你们正在失去XX" 比 "行业数据显示XX" 更有冲击力 |
| 太抽象 | 用具体数字替代模糊描述 | "下降了37%" 比 "大幅下降" 更可信 |
| 太平淡 | 用对比制造反差 | "竞品增长了20%，你们下降了5%" |
| 太冗长 | 一句话核心信息+支撑 | 先给结论，再给1-2个支撑点 |
| 缺乏紧迫感 | 用时间压力 | "如果不改变，6个月后..." |

## AIGC Demo 模板

### AIGC提示词标准格式

每个Demo的提示词必须包含以下要素：

```
Demo名称: {场景名}
对应Pitch页面: {PXX}
策略目的: {这个视觉要说明什么}

AIGC提示词（直接复制使用）:
"""
[主体描述], [动作/状态], [环境/背景], [风格], [色调], [构图], [氛围],
--ar 16:9 --v 6 --style raw
"""

风格参考: {参考什么风格的视觉}
关键元素: {必须出现的视觉元素}
避免元素: {不能出现的视觉元素}
推荐工具: {Midjourney / DALL-E / Stable Diffusion}
备选变体: {如果第一次生成不理想，可以调整的方向}
```

### 不同策略类型的视觉Demo建议

| 策略类型 | 推荐Demo场景 | 视觉风格 |
|---------|-------------|---------|
| 品牌焕新 | 新品牌视觉在真实场景中的应用 | 干净、现代、生活方式 |
| 产品创新 | 产品使用的场景化呈现 | 高端、细节感、真实感 |
| 体验营销 | 消费者互动的场景 | 活力、沉浸感、第一人称视角 |
| 数字营销 | 界面/社交平台效果预览 | 科技感、数据可视化 |
| 情感叙事 | 关键情感画面的定格 | 电影感、光影、人物特写 |

### AIGC提示词质量标准

- 必须是英文（Midjourney/DALL-E对英文理解更准确）
- 包含具体的视觉描述（不是抽象概念）
- 包含风格和氛围描述
- 包含构图和色调指导
- 标注宽高比（Pitch Deck通常16:9）
- 提供至少1个备选变体
