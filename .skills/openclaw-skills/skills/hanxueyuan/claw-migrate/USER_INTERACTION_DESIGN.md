# claw-migrate 用户交互设计

**核心原则**：让用户选择，而不是自动决定

---

## 📋 安装后自动配置流程

### 阶段 1：技能安装完成后的自动触发

```bash
# 用户安装技能
openclaw skill install claw-migrate

# ✅ 安装完成后，自动触发配置向导
🎉 claw-migrate 安装完成！

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 请选择你要进行的操作：

   1. 🔵 开始备份配置
      将本地配置备份到 GitHub 私有仓库
      适合：首次使用、定期备份

   2. 🟢 恢复/迁移配置
      从 GitHub 仓库恢复配置到本地
      适合：新机器、配置恢复

   3. ⚪ 稍后配置
      跳过向导，稍后手动配置
```

## 🎯 设计原则总结

1. **用户选择** - 所有内容都让用户选择，不自动决定
2. **清晰提示** - 敏感信息、风险等清晰提示
3. **预览确认** - 执行前预览，确认后执行
4. **灵活配置** - 随时可以修改配置
5. **状态透明** - 备份状态、历史清晰可见
