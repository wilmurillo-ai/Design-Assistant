# workflow-execution

Plan-first workflow for non-trivial work: define done criteria, track the work, package context as documents, route it to the right code destination, hand off cleanly, execute with minimal impact, and verify with evidence.

See [SKILL.md](./SKILL.md) for the full workflow and tracker-specific references.

## Install

```bash
clawhub install workflow-execution
```

## Includes

- **Planning guidance** — goal, done criteria, constraints, failure modes, ordered steps
- **Tracking patterns** — issue-first execution with structured documents
- **Context packaging** — `plan`, `design`, and `context` document conventions
- **Routing rules** — existing repo vs new repo vs local workspace
- **Handoff protocol** — spawn execution with issue references instead of fragile chat context
- **Verification loop** — evidence-based completion and closeout

## Tracker references

- Paperclip
- GitHub Issues
- Local-file fallback
