# Feature Requests Ledger

Use this ledger for requested capabilities the agent does not yet possess or does not perform reliably enough.

```markdown
## [FEAT-YYYYMMDD-XXX] capability_request

**Logged**: ISO-8601 timestamp
**Priority**: low | medium | high | critical
**Status**: open | triaged | planned | satisfied | declined
**Related Capability**: research | planning | tool-use | verification | synthesis | communication | coding | execution discipline | memory retrieval | long-horizon task handling

### Requested Capability
What the user wanted the agent to do.

### User Context
Why this matters and what job the user was trying to get done.

### Current Limitation
What the agent could not yet do well.

### Evolution Path
How this might become training, tooling, or a new stable policy.

### Linked Records
- CAP-...
- TRN-...
- EVL-...
- AGD-...

---
```
