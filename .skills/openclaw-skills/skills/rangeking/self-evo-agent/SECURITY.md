# Security Policy

## Supported Versions

Security fixes are currently prioritized for the latest state of the default branch.

| Version | Supported |
| --- | --- |
| Latest `main` | Yes |
| Older snapshots and forks | Best effort |

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive findings.

Preferred reporting paths:

1. Use GitHub private vulnerability reporting if it is enabled for this repository.
2. If private reporting is unavailable, contact the maintainer privately through GitHub: [@RangeKing](https://github.com/RangeKing).

Please include:

- a concise description of the issue
- affected files or workflow areas
- reproduction steps or proof of concept
- impact assessment
- any suggested mitigation

You can expect an acknowledgment as soon as practical, followed by triage and a remediation plan if the issue is confirmed.
