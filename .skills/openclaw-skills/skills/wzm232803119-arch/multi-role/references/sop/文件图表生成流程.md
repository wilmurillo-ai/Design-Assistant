# 文件图表生成流程 SOP

**适用类型：** 架构图、流程图、文档输出、数据图表、线框图等产出类任务
**版本：** v1.0
**创建日期：** 2026-03-11

---

## 流程步骤

1. **需求澄清**：大管家向用户确认以下要素
   - 文件/图表类型（架构图 / 流程图 / 线框图 / 文档 / 其他）
   - 输出格式（PNG / PDF / MD / DOCX 等）
   - 内容范围和核心要素
   - 风格要求（简洁线框 / 精美设计 / 技术风格 等）
2. **分配执行角色**：根据类型选择合适角色
   - 架构图 / 流程图 / 线框图 → 产品经理 负责内容，使用 canvas-design 或 PIL 工具生成
   - 技术方案文档 → CTO 负责
   - 运营内容文档 → 内容运营 负责
   - 通用文档 → 产品经理 负责
3. **中文渲染检查（强制）**：凡涉及中文内容的图片/PDF，必须使用 CJK 字体
   - macOS 系统字体：`/System/Library/Fonts/STHeiti Medium.ttc`
   - 禁止使用英文字体渲染中文（会产生方块乱码）
4. **大管家审核**：检查产出物是否符合需求
   - 内容是否完整、准确
   - 中文是否正常显示（无乱码）
   - 格式是否符合要求
   - 排版是否清晰可读
5. **提交用户确认**：将产出物发送给用户审查
6. **根据反馈修改**：如用户有调整意见，返回步骤 2 或 3 重新处理
7. **存档**：将最终版本存放到对应项目目录，并记录到对话归档

---

## 关键规则

- **中文图片必须用 CJK 字体**，否则一律打回重做
- **架构图 ≠ 流程图**：架构图展示模块结构和依赖关系；流程图展示步骤顺序
- 需求未澄清清楚前，**禁止开始生成**
- 大管家**禁止自己直接生成**，必须分配给对应角色执行
- 产出物必须经过**大管家审核**后才能提交用户

---

## 常见错误（血泪教训）

- ❌ 未确认输出格式就开始做，做完发现格式不对
- ❌ 使用英文字体渲染中文，产生乱码
- ❌ 把「架构图」做成了「流程图」，内容方向错误
- ❌ 大管家跳过类型判断、直接让角色执行，没有走本 SOP

---

## 字体使用参考

| 场景 | 推荐字体 |
|------|---------|
| 中文图表（macOS） | `/System/Library/Fonts/STHeiti Medium.ttc` |
| 中文图表（备选） | `/System/Library/Fonts/PingFang.ttc` |
| 英文图表 | `canvas-fonts/` 目录中的字体 |
