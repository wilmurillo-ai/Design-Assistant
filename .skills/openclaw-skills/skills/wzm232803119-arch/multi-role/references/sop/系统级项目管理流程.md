# 系统级项目管理流程 (SOP)

> 📌 **用途**: 规范系统级项目的创建、执行、归档流程
> 🎯 **适用范围**: 所有优化LobsterAI底层能力的项目
> 📝 **参考**: 日志管理规范 (`rules/日志管理规范.md`)

---

## 一、什么是系统级项目

### 定义
不产生直接业务价值,但优化LobsterAI底层能力的项目。

### 特征
- ✅ 无具体业务代码(或代码不在项目目录)
- ✅ 交付物是文档、规范、配置、架构调整
- ✅ 服务于所有业务项目

### 示例
- Skill库标准化 - 优化Skills目录结构
- 文档重构 - 优化governance文档体系
- 架构升级 - 调整多Agent系统架构
- 性能优化 - 全局Token优化、响应速度优化

### 与业务项目的区别

| 项目类型 | 交付物 | 目录位置 | 示例 |
|---------|--------|---------|------|
| 业务项目 | 可运行的应用程序 | `project/[项目名]/` | project-a, project-b |
| 系统级项目 | 文档/规范/优化方案 | `system-projects/[项目名]/` | skill-library, docs-refactor |

## 二、项目创建流程

### 步骤1: 创建项目目录

```bash
PROJECT_NAME="项目名"
mkdir -p ~/your-workspace/system-projects/${PROJECT_NAME}/{logs,deliverables}
```

### 步骤2: 创建项目元数据 (project.json)

```json
{
  "project_name": "项目全称",
  "project_type": "system",
  "status": "in_progress",
  "start_date": "YYYY-MM-DD",
  "end_date": null,
  "description": "项目简要描述",
  "objectives": ["目标1", "目标2"],
  "team": {
    "角色名": {
      "role": "职责描述",
      "performance": null
    }
  },
  "deliverables_location": "交付物主要位置",
  "logs_directory": "system-projects/[项目名]/logs/"
}
```

### 步骤3: (可选) 创建 deliverables/README.md

如果交付物在其他位置(如sessions/governance/、SKILLs/),创建引用说明:

```markdown
# [项目名] - 交付物位置

## 主要交付物

### 1. 交付物名称
**位置**: 绝对路径

**说明**: 简要说明

## 快速访问

\`\`\`bash
# 访问命令
\`\`\`
```

## 三、项目执行流程

### 阶段1: 方案设计

1. 分析现状问题
2. 提出优化方案
3. 用户确认方向
4. 保存方案到 `logs/YYYY-MM-DD_proposal.md`

### 阶段2: 实施执行

1. 按方案执行优化
2. 记录关键步骤到工作日志
3. 遇到问题及时记录和调整
4. 定期保存中间结果到 `logs/`

### 阶段3: 测试验证

1. 验证优化效果
2. 保存测试报告到 `logs/YYYY-MM-DD_test_report.md`
3. 发现问题返回阶段2修复

### 阶段4: 项目归档

1. 保存完成报告到 `logs/YYYY-MM-DD_completion.md`
2. 更新 `project.json`:
   - `status`: "completed"
   - `end_date`: "YYYY-MM-DD"
   - `team.[角色].performance`: 评分
3. 在 PROJECT_INDEX.md 中记录项目

## 四、日志管理规范

所有工作报告、测试报告、完成报告统一存放在 `logs/` 目录。

### 日志命名

- 方案文档: `logs/YYYY-MM-DD_proposal.md`
- 工作报告: `logs/YYYY-MM-DD_work_report.md`
- 测试报告: `logs/YYYY-MM-DD_test_report.md`
- 完成报告: `logs/YYYY-MM-DD_completion.md`

详见: `references/rules/日志管理规范.md`

---

## 五、交付物管理

### 交付物在项目目录内

直接存放在项目目录下,无需特殊说明。

### 交付物在其他位置

在 `deliverables/README.md` 创建引用:

```markdown
**位置**: `~/your-workspace/sessions/governance/`

**快速访问**:
\`\`\`bash
cd ~/your-workspace/sessions/governance
\`\`\`
```

## 六、项目归档检查清单

项目完成后,确认以下内容:

- [ ] `project.json` 状态更新为 "completed"
- [ ] `project.json` 填写了 end_date
- [ ] 所有团队成员都有performance评分
- [ ] `logs/` 目录包含完整的工作记录
- [ ] `deliverables/` 说明了交付物位置
- [ ] PROJECT_INDEX.md 中记录了本项目
