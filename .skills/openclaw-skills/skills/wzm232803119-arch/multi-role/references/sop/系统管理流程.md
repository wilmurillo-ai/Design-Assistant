# 系统管理流程 SOP

**适用类型：** SOP 维护、角色管理、治理文件更新、规则修订、宪法变更等系统管理任务
**版本：** v1.0
**创建日期：** 2026-03-11

---

## 流程步骤

### A. SOP 创建 / 更新

1. **识别触发时机**：以下情况需要创建或更新 SOP
   - 执行了无 SOP 覆盖的新型指令 → 创建新 SOP
   - 现有 SOP 流程不合理，用户或角色提出改进 → 更新现有 SOP
   - 发现 SOP 中存在漏洞或错误 → 修复并更新版本
2. **起草 SOP 内容**：包含以下要素
   - 适用类型（明确哪类指令触发此 SOP）
   - 版本和创建日期
   - 流程步骤（逐步清晰）
   - 关键规则（禁止项和强制项）
3. **存放位置**：`references/sop/[流程名称].md`
4. **更新索引**：在 SKILL.md 的 SOP 表格中补充该文件的引用
5. **更新角色宪法引用**：在 `references/constitutions/大管家-角色宪法-SOP.md` 的「四、方案审核清单」中补充
6. **向用户确认**：告知已创建/更新的 SOP 文件路径和主要内容

### B. 角色宪法 创建 / 更新

1. **触发条件**：
   - 角色首次创建 → 新建宪法文件
   - 角色触发重建阈值（P0×3 / P1×6 / P2×9）→ 创建新版本宪法
   - 用户要求更新角色定义 → 更新当前版本，追加版本记录
2. **文件位置**：`references/constitutions/[角色名]_v[版本号].md`
3. **内容要求**：职责定义、工作方式、强制检查清单、历史错误教训
4. **更新 README**：在宪法层表格中更新版本号和说明

### C. 错误记录 更新

1. **触发条件**：角色发生 P0/P1/P2 错误
2. **文件位置**：`templates/archives/errors/[角色名]_错误记录.md`
3. **内容格式**：
   - 错误编号、发生时间
   - 问题描述（发生了什么）
   - 根本原因分析
   - 解决方案和教训
4. **同步到宪法**：将教训要点同步到对应角色宪法的「教训」章节

### D. README 索引维护

1. **触发条件**：任何治理文件新增、删除、更新版本
2. **文件位置**：SKILL.md（路由索引）
3. **维护内容**：
   - 宪法层文件列表（确保版本号准确）
   - SOP 层文件列表（确保文件真实存在）
   - 规则层文件列表
   - 归档层文件列表

---

## 关键规则

- **禁止口头约定**：任何规则、流程、决策，必须写入文件，不能只停留在对话记录中
- **文件存放强制规则**：
  - 所有 SOP 文件 → `references/sop/`
  - 所有角色宪法 → `references/constitutions/`
  - 所有规则清单 → `references/rules/`
  - 所有历史档案 → `templates/archives/`
- **README 必须准确**：SKILL.md 中引用的文件必须真实存在，禁止列出不存在的文件
- **版本必须连续**：宪法版本号不得跳号，每次更新必须在文件末尾追加版本历史记录

---

## 禁止项

- ❌ 在不存在对应 SOP 的情况下，凭感觉执行任务
- ❌ 创建治理文件后不更新 README 索引
- ❌ 在对话中口头「约定」规则而不写入文件
- ❌ 将治理文件存放到 `references/` 目录以外的位置
