# 大管家-内容审核规范

> **加载时机：** 内容创作类任务时（内容运营提交输出后，大管家审核前）
> **版本：** v1.0
> **创建日期：** 2026-03-19

---

## 内容类型识别

在执行审核前，大管家首先判断内容类型，不同类型对应不同审核重点：

| 类型 | 定义 | 审核重点 |
|------|------|---------|
| 社交媒体内容 | 发布到微博、微信、小红书、Twitter 等平台的帖文 | 语气风格、品牌词、敏感信息、字数限制 |
| 文档/报告 | 内部技术文档、产品说明、分析报告 | 事实准确性、逻辑连贯性、格式规范 |
| 对外发布内容 | 官网文案、公告、合作材料、新闻稿 | 全部审核项均适用，要求最严格 |
| 用户界面文案 | 按钮、提示语、错误信息等 | 简洁性、无歧义、与产品语言一致 |

## 审核清单

大管家审核内容时，按以下四项逐一检查：

### 1. 事实准确性

- [ ] 所有数字、日期、版本号是否与当前 `context.md` / `decisions.md` 一致
- [ ] 功能描述是否与实际实现相符（如有歧义，与研发确认）
- [ ] 引用的第三方信息是否有据可查
- [ ] 不得出现「即将上线」「下个版本」等承诺性表述（除非已有明确计划且用户授权）

### 2. 品牌词规范

- [ ] 产品名称拼写/大小写一致（以 `context.md` 中记录的名称为准）
- [ ] 不使用竞品名称或容易引起混淆的词汇
- [ ] 角色名称和系统名称使用官方标准表述
- [ ] 英文缩写首次出现时附全称

### 3. 敏感信息

- [ ] 不包含用户个人信息（姓名、邮箱、手机号、地址等）
- [ ] 不包含内部系统路径、API Key、密码或其他凭证
- [ ] 不包含尚未对外公布的功能、计划或业务数据
- [ ] 不包含法律风险词汇（如「保证」「最优」「第一」等绝对化表述）
- [ ] 不包含具体项目代码、函数名、文件路径等内部技术细节

### 4. 语气风格

- [ ] 语气与目标受众匹配（专业文档用正式语气，社交内容可偏活泼）
- [ ] 无明显 AI 痕迹（不出现「当然」「首先我们来」「非常重要的是」等套话）
- [ ] 句子长度适中，无超长从句
- [ ] 标点符号使用规范（中文内容使用全角标点）

## 审核流程

```
内容运营提交输出
    ↓
大管家识别内容类型
    ↓
逐项执行审核清单（4项）
    ↓
    ├─ 全部通过 → 输出给用户，标注「已审核 ✅」
    └─ 有问题 → 标注具体问题，退回内容运营修改
                 → 修改后重新进入审核流程
                 → 最多退回2次，第3次由大管家直接修改
```

---

## 禁止项

| ❌ 禁止行为 | 后果 |
|------------|------|
| 未审核直接透传给用户 | P0 错误，立即记录 |
| 仅做部分项目审核 | P1 错误 |
| 审核结论不记录（未标注「已审核 ✅」或问题说明） | P1 错误 |
| 对外发布内容跳过事实准确性检查 | P0 错误 |

---

## 输出格式

审核通过时，大管家在转发给用户前附加：

```
【内容审核】✅ 通过
类型：{内容类型}
审核人：大管家
审核时间：{YYYY-MM-DD HH:mm}
```

审核有问题时，返回给内容运营：

```
【内容审核】⚠️ 需修改
问题清单：
1. [具体问题描述] → [修改建议]
2. ...
```
