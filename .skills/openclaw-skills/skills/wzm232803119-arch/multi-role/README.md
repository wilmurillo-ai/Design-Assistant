## 📦 安装说明

1. 将本 Skill 目录放置于本地路径（默认：`/Users/{user}/Documents/Skill/multi-role-governance/`）
2. 在 OpenClaw 中加载本 Skill
3. 首次对话，大管家会自动初始化目录结构并进行冷启动自检
4. 开始使用，无需额外配置

## 🧩 与 memory-index 配合使用

单独使用本 Skill：✅ 角色切换 + SOP 管理正常工作，但记忆按时间线性存储，多项目之间可能串扰。

配合 memory-index：✅ 记忆按项目/角色隔离，精确召回，完整体验。
> ⚠️ **推荐安装**：在 ClawHub 搜索 `memory-index` 安装，可获得完整的多项目记忆隔离能力。不安装也可正常使用，但多项目场景下记忆可能串扰。

## 📁 目录结构

```
multi-role-governance/
├── SKILL.md                        # Skill 核心规则（AI 加载）
├── README.md                       # 本文件（用户阅读）
├── SKILL-VALUE-REPORT-SPEC.md      # 跨 Skill 上报协议规范
├── references/
│   ├── constitutions/              # 角色宪法（6个角色 + 模板）
│   ├── sop/                        # SOP 流程文件（9个）
│   ├── rules/                      # 公共规则（6个，含任务状态机）
│   └── role-extras/                # 角色扩展文档（按需）
├── sessions/                       # 项目记忆目录（运行时自动创建）
└── templates/                      # 模板与归档
    ├── archives/
    └── iteration-reports/
```

## 📂 文件读写说明

本 Skill 会在其目录内自动创建和维护以下路径，**不会访问 Skill 目录以外的任何位置**：

| 路径 | 用途 | 创建时机 |
|------|------|---------|
| `sessions/{项目名}/` | 项目记忆（context、decisions、pitfalls、inbox） | 首次启动新项目时 |
| `sessions/{项目名}/task-logs/` | 任务执行日志 | 任务执行时 |
| `sessions/.skill-metrics.jsonl` | 跨 Skill 价值上报数据 | Skill 调用时追加 |
| `templates/archives/` | 归档记录 | 迭代收尾时 |

> 所有写入操作均限定在本 Skill 目录内。如需验证，可随时询问大管家"你读取了哪些文件"，大管家将如实回答。

## ❓ 常见问题

**Q：需要先说「CTO 你来」才能让 CTO 处理吗？**
A：不需要。大管家会根据你的意图自动路由。直接说你要做的事就行。

**Q：任务失败了怎么办？**
A：大管家会自动重试（最多3次），失败后写入 `sessions/{项目}/inbox.md`，你能看到失败原因和建议的下一步。

**Q：想扩展新角色怎么做？**
A：以 `references/constitutions/角色宪法-模板.md` 为起点创建新宪法文件，告诉大管家审批即可。

**Q：上下文满了怎么办？**
A：超过150K时大管家会自动压缩写入记忆文件，任务继续执行，不需要你手动干预。
