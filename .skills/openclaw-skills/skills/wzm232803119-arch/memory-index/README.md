# Memory Index - OpenClaw 结构化记忆引擎

为你的 OpenClaw 注入真正的“海马体”。

## 痛点终结：告别原生乱炖与烧 Token

在原生状态下，无序的上下文堆叠不仅让 Agent 频繁遗忘关键信息（原生乱炖），更在无形中燃烧了海量的 Token 预算。Memory Index 专为解决此痛点而生，让每一次对话都精准且高效。

## 四大核心特性

- **三层隔离**：物理级划分短期工作区、中期项目库与长期知识图谱，上下文互不干扰。
- **冷热拆分**：智能沉淀冷数据，极速检索热点上下文，确保大模型每次调用都切中要害。
- **防死锁机制**：独创记忆环路自检，彻底杜绝逻辑死循环与无意义的重复复读。
- **并发锁控制**：完美支持多 Subagent 异步高频写入，保障记忆严格一致，绝对不丢数据。

## 安装指南

```bash
clawhub install memory-index
```

## 使用指南（无痛上手）

零代码侵入！安装完毕后，Memory Index 会自动接管 OpenClaw 的底层存储路由。
你只需像往常一样与 Agent 对话，引擎会在后台默默为你调度记忆并节省 Token。

```bash
# 如需手动触发全量记忆冷热转换归档，可执行：
openclaw memory-index archive
```
