# Daily Review Spec

## 栏目说明

### 今日大事

放置具有公共重要性、影响范围广或需要优先建立背景感的事件。

### 变更与实践

放置能直接转化为个人实践、工作动作、工具使用或流程调整的内容。

### 安全与风险

放置漏洞、安全事件、合规变化、稳定性风险、重大政策约束。

### 开源与工具

放置工具发布、生态变动、项目演进、值得试用的新能力。

### 洞察与数据点

放置值得记住的数据、趋势信号、行业观察。

### 主题深挖

从当天候选中选一个最值得向趋势层面提升的话题，补上更深一层的分析框架。

## 固定栏目约束

- Daily Review 默认固定输出以下 6 个 `##` 栏目，顺序保持不变：
  - `今日大事`
  - `变更与实践`
  - `安全与风险`
  - `开源与工具`
  - `洞察与数据点`
  - `主题深挖`
- 不再额外创建题材型一级栏目，例如“产业与公司动态”“云与基础设施”“研究与治理”“今日观察”。
- 若某一栏当日没有足够高价值内容，也保留该栏目，并用 `### 暂无关键更新` 简短说明，而不是删掉栏目。
- 所有条目必须先判断认知任务，再决定进入哪个固定栏目，不能按来源类型或新闻题材机械分组。

## 单条内容推荐结构

- 标题
- 核心判断
- 为什么值得关注
- 对我的意义
- 参考链接（Markdown 格式）

## Markdown 排版规范

- 用 `#` / `##` / `###` 组织文档、栏目和子主题。
- 用短段落和列表提升扫描效率。
- 用 `**加粗**` 标出核心关键词、关键判断和风险点。
- 可适度使用 emoji：
  - `📌` 主题或重点
  - `✅` 明确结论
  - `💡` 洞察或启发
  - `⚠️` 风险或注意事项
- 不使用长篇连续正文覆盖多个主题。

## 标题层级规范

- `#`：整篇 Daily Review 标题，例如 `# Daily Review | 2026-03-28`
- `##`：固定栏目，只允许使用这 6 个栏目名
- `###`：单个独立主题，例如某一条工具更新、风险事件或产业动态
- 不要把同级主题混用为列表项和标题；需要强调结构时优先使用标题，而不是纯 emoji 文本行

## 英文文章处理规范

- 若内容来自英文原文，Review 中必须提供中文摘要。
- 中文摘要至少覆盖：
  - 发生了什么
  - 核心观点或关键结论
  - 为什么值得关注

## 链接规范

- 统一使用 `[标题文本](链接)`。
- 标题文本必须短、准、可识别。
- 若同一条内容有多个来源，正文只放主链接，其他链接放“补充来源”。

## 编辑准则

- 一份日报优先服务快速判断。
- 与其多，不如清楚。
- 只保留真正改变今日注意力分配的内容。
- 同一主题只表达一次核心结论，避免不同段落重复复述。
- 默认不包含“处理边界”区块。
- 默认不包含“后续动作”区块。
- 所有内容都要落入 6 个固定栏目中的一个，避免游离在栏目体系之外。
