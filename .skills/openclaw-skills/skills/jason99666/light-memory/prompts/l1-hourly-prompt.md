# L1 Hourly Distill Prompt

> 【全自动 Cron 任务】此 Prompt 由 Cron 定时自动触发，无需手动执行。
> 任务：每小时提炼对话中的关键信息，写入 SESSION-STATE.md

---

## 角色
你是记忆系统管理员，负责每小时从对话日志中提取关键信息。

---

## 路径初始化（第一步！）

**禁止使用 `~` 符号！Cron 环境中 ~ 可能被解析为 /root。**

执行以下命令获取绝对路径：

```bash
BASE=$(pwd)
WORKSPACE="${BASE}/workspace"
SESSION_STATE="${WORKSPACE}/SESSION-STATE.md"
SESSIONS_DIR="${BASE}/../../agents/main/sessions/"
```

**所有后续文件操作必须使用这些变量。**

---

## 输入

- **文件**：`${SESSIONS_DIR}` 下的 `.jsonl` 文件
- **范围**：最后 300 行（或 `[Last_Extracted_Time]` 之后的内容，取较新的）

---

## 提取目标

按以下优先级提取内容：

### 1. 活跃任务 (P0)
用户明确提到的待办事项，关键词：
- "任务："、"要"、"需要"、"计划"、"待办"
- "完成了"、"搞定了"（用于更新状态）

### 2. 关键决策 (P1)
用户做出的决定，关键词：
- "决策："、"结论："、"就用"、"选"、"放弃"
- "开始吧"、"执行"、"确认"

### 3. 洞察发现 (P2)
用户的深度思考，关键词：
- "洞察："、"发现"、"本质"、"根因"
- "理解"、"意识到"、"其实"

### 4. 完成项 (P3)
用户确认完成的任务，关键词：
- "完成了"、"搞定了"、"做完了"

---

## 过滤规则

**忽略**：
- ❌ 闲聊、寒暄、重复信息
- ❌ 已归档到 L2 的旧内容（检查 `[Last_Extracted_Time]` 游标）
- ❌ 工具调用的中间结果
- ❌ 错误堆栈和调试信息

**保留**：
- ✅ 未提炼的新内容
- ✅ 跨会话需要保持的上下文
- ✅ 明确的显式标记（"决策："、"洞察："等）

---

## 输出格式

写入 `${SESSION_STATE}`，格式如下：

```markdown
[Last_Extracted_Time: YYYY-MM-DD HH:MM:SS]

## 活跃任务 (Active Tasks)
- [ ] 任务描述（优先级 P0/P1/P2）

## 关键决策 (Key Decisions)
- 决策内容（带时间戳）

## 洞察发现 (Insights)
- 洞察内容（带时间戳）

## 完成项 (Completed)
- [x] 完成的任务

## 待归档上下文 (Pending Context)
- 需要保留到 L2 的对话摘要（200 字以内）
```

---

## 执行步骤

1. **读取日志**：使用 `read` 工具读取 `.jsonl` 文件（最后 300 行）
2. **查找游标**：如 SESSION-STATE 已存在，读取 `[Last_Extracted_Time]`
3. **识别标记**：扫描显式标记（决策/任务/洞察/完成）
4. **提炼内容**：分析对话，提取关键信息（忽略闲聊）
5. **写入文件**：使用 `write` 工具覆写 SESSION-STATE.md
6. **确认通知**：输出单行确认

---

## 确认通知

执行完成后，输出：

```
✅ L1 提炼完成 [HH:MM] | 提取：N 任务, M 决策, K 洞察
```

**最高静默纪律**：仅输出确认行，不输出中间思考过程。

---

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 日志文件不存在 | 跳过，输出 "⚠️ 日志文件不存在，跳过本次提炼" |
| 文件过大（>10MB） | 仅读取最后 300 行，忽略其余 |
| 写入失败 | 输出 "❌ 写入失败，请检查权限" |
| 无新内容 | 跳过写入，输出 "✅ 无新内容，跳过" |
