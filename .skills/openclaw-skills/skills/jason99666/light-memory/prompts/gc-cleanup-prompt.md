# GC Session Cleanup Prompt

> 任务：每天 23:10 清理不再需要的 Session 文件，释放磁盘空间

---

## 角色
你是系统管理员，负责清理不再需要的 Session 文件。

---

## 路径初始化（第一步！）

**禁止使用 `~` 符号！**

```bash
BASE=$(pwd)
SESSIONS_DIR="${BASE}/../../agents/main/sessions/"
```

**所有后续文件操作必须使用这些变量。**

---

## 删除规则

### 删除条件（满足任一即删除）

1. **Cron 临时文件**：文件名包含 `:run:` 标记
2. **空文件**：文件大小 < 1KB（1024 字节）

### 保留条件（满足任一即保留）

1. **Main 会话**：所有历史会话完整保留
2. **Feishu 会话**：所有历史会话完整保留
3. **用户创建的会话**：保留
4. **大文件**：文件大小 > 10KB（说明有实际内容）
   > 注：此为安全兜底规则，确保有实际内容的 session 不会被误删，
   > 即使不满足上述保留条件，只要文件足够大就说明不是临时文件。

---

## 执行步骤

### Step 1: 列出文件

扫描 `${SESSIONS_DIR}` 目录下所有 `.jsonl` 文件。

### Step 2: 分类判断

对每个文件：

```
检查文件名是否包含 :run: → 是 → 标记删除
检查文件大小是否 < 1KB → 是 → 标记删除
检查文件大小是否 > 10KB → 是 → 标记保留
否则 → 标记保留
```

### Step 3: 执行删除

删除标记为"删除"的文件。

### Step 4: 输出报告

```
🧹 GC 完成
删除：N 个文件（:run: X 个, 空文件 Y 个）
保留：M 个文件
释放：Z MB 空间
```

---

## 安全机制

- ✅ 执行前输出将被删除的文件列表
- ✅ 仅删除明确标记为临时的文件
- ✅ 保留所有用户可见的历史会话
- ✅ 不删除 Main/Feishu 核心会话

---

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| sessions 目录不存在 | 跳过，输出 "⚠️ sessions 目录不存在" |
| 无文件可删除 | 输出 "✅ 无需清理" |
| 删除失败 | 记录错误，继续处理下一个文件 |
