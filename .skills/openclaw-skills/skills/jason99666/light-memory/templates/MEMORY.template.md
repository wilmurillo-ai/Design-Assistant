```markdown
# 长期记忆法则库 (L3)

> 本文档由 L3 周度自我进化任务自动维护。
> 每周日 23:30 从 L2 每日归档中提炼 CAR 格式法则。

## 🔐 安全性原则

**核心论点**：不存储敏感凭证
- **触发场景**：任何配置修改
- **失效动作**：直接写入 API Key 到配置文件
- **成功动作**：使用环境变量或密钥管理
- **强制复用规则**：禁止在对话或文件中存储敏感信息

---

## 🧠 结构化法则库

### [分类名称]
**核心论点**：一句话总结法则
- **触发场景**：具体场景描述
- **失效动作**：第一次尝试的错误做法
- **成功动作**：修正后的正确做法
- **强制复用规则**：未来必须遵守的规则

### [配置安全]
**核心论点**：Cron 配置修改必须最小化，禁止重写整个文件
- **触发场景**：2026-03-31 重写 jobs.json 导致 L3 任务丢失
- **失效动作**：重写整个配置文件，未验证任务完整性
- **成功动作**：使用 Python 精确修改指定字段
- **强制复用规则**：任何生产配置文件修改必须备份→最小化修改→验证

---

## 📋 元规则

1. **文档即代码**：MEMORY.md 的每次更新必须通过 Self-Improving 流程
2. **CAR 格式强制**：所有新经验必须以 CAR 结构写入
3. **定期回顾**：每月回顾一次法则库，淘汰过时规则

---

*最后更新：YYYY-MM-DD*
```
