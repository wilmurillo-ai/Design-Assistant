# CAR 格式规范

> Context-Action-Result 经验萃取格式

---

## 什么是 CAR 格式？

CAR 是一种结构化的经验记录格式，确保每条经验都有：
- **Context**：什么场景下发生的
- **Action**：采取了什么行动（成功/失败）
- **Result**：得到了什么可复用的规则

---

## 完整格式

```markdown
### [分类名称]
**核心论点**：一句话总结法则
- **触发场景**：具体场景描述（什么情况下遇到这个问题）
- **失效动作**：第一次尝试的错误做法
- **成功动作**：修正后的正确做法
- **强制复用规则**：未来必须遵守的规则
```

## 示例

### 示例 1：配置安全

```markdown
### [配置安全]
**核心论点**：Cron 配置修改必须最小化，禁止重写整个文件
- **触发场景**：2026-03-31 重写 jobs.json 导致 L3 Self-Improving 任务和飞书待办任务丢失
- **失效动作**：重写整个配置文件，只复制了 3 个任务，漏掉了关键任务；修改后未验证完整性
- **成功动作**：使用 Python 精确修改指定字段；建立 checklist（备份→最小化修改→验证数量→验证 ID→验证 JSON 格式）
- **强制复用规则**：任何生产配置文件修改必须：①备份原文件并验证有效 ②只修改必要字段 ③修改后验证任务数量未变 ④验证关键任务 ID 存在 ⑤验证 JSON 格式正确
```

### 示例 2：技术调试

```markdown
### [技术调试]
**核心论点**：WSL2→Windows 网络问题需系统化分层排查
- **触发场景**：2026-04-02~04-03 web-access skill 调试，WSL2 无法连接 Windows Chrome CDP
- **失效动作**：盲目尝试 5 种方案均失败（端口转发、CDP Proxy、修改监听地址等）
- **成功动作**：建立系统化排查流程（网络层→传输层→应用层）；诊断根因（特定端口通信被阻断）
- **强制复用规则**：跨环境网络问题必须分层诊断（网络层 ping→传输层 TCP→应用层 HTTP）；备选方案（WSL2 安装 Chrome 或 cmd.exe 中转）；不盲目尝试
```

### 示例 3：效率工具

```markdown
### [效率工具]
**核心论点**：飞书文档操作统一使用官方扩展
- **触发场景**：2026-03-11 飞书文档写入失败排查
- **失效动作**：使用自建 feishu-workspace skill 进行写入/追加操作，API 过时导致失败
- **成功动作**：切换至官方 feishu_doc 扩展，写入/追加操作成功
- **强制复用规则**：飞书文档写入/追加操作必须使用 feishu_doc 扩展，禁止使用自建 skill 进行写入操作
```

## 萃取标准

### 高价值经验（提取）

| 类型 | 特征 | 示例 |
|------|------|------|
| **重复问题** | 出现 2 次以上 | 配置修改导致任务丢失 |
| **系统性风险** | 影响全局 | Cron 配置错误导致系统失联 |
| **关键决策** | 影响后续工作流 | 从 WAL 实时拦截改为显式标记 |
| **成功方法** | 可推广到其他场景 | 分层排查网络问题 |
| **错误教训** | 避免重复踩坑 | 禁止硬编码日期 |

### 低价值经验（跳过）

| 类型 | 特征 | 示例 |
|------|------|------|
| **一次性问题** | 特定环境/时间 | 某次特定的 API 超时 |
| **临时方案** | 后续需要修改 | 临时绕过某个 bug |
| **个人偏好** | 不具普遍性 | 喜欢用某种编辑器 |
| **闲聊内容** | 非技术/非决策 | 日常对话、寒暄 |

## 分类建议

| 分类 | 适用场景 |
|------|---------|
| **配置安全** | 配置文件修改、权限管理 |
| **架构设计** | 系统架构、数据流转 |
| **技术调试** | 排错、调试、故障恢复 |
| **效率工具** | 工具使用、工作流优化 |
| **产品思维** | 产品定位、用户体验 |
| **求职策略** | 职业发展、面试经验 |

---

## 质量检查

每条 CAR 法则应满足：

- [ ] **核心论点**一句话总结（20 字以内）
- [ ] **触发场景**具体明确（包含时间/环境）
- [ ] **失效动作**描述错误做法
- [ ] **成功动作**描述正确做法
- [ ] **强制复用规则**具体可执行（不是泛泛而谈）
- [ ] **无重复**（检查 MEMORY.md 已有内容）
