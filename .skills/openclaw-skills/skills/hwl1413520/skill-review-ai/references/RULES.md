# 审查规则详解

本文档详细说明 skill-review 工具的审查规则和评分标准。

## 评分概览

| 类别 | 满分 | 通过线 | 说明 |
|------|------|--------|------|
| SKILL.md 格式 | 25 | 20 | frontmatter 和正文格式 |
| 目录结构 | 20 | 15 | 目录组织和命名 |
| 脚本代码 | 35 | 25 | 代码质量和安全性 |
| 文件引用 | 20 | 15 | 引用完整性和正确性 |
| **总分** | **100** | **75** | 综合评分 |

## SKILL.md 格式规则

### 必需字段检查

#### name 字段
- **检查项**: 存在性、格式、长度
- **扣分**: 缺少字段 -5 分，格式错误 -3 分
- **验证规则**:
  - 必须存在
  - 1-64 字符
  - 仅小写字母、数字、连字符
  - 不能以连字符开头或结尾
  - 不能包含连续连字符

#### description 字段
- **检查项**: 存在性、长度
- **扣分**: 缺少字段 -5 分，过长/过短 -2 分
- **验证规则**:
  - 必须存在
  - 1-1024 字符
  - 建议至少 10 字符

### 可选字段检查

#### license 字段
- **检查项**: 格式
- **扣分**: 格式问题 -1 分（警告）

#### compatibility 字段
- **检查项**: 长度
- **扣分**: 超过 500 字符 -1 分（警告）

#### metadata 字段
- **检查项**: 格式
- **扣分**: 格式问题 -1 分（警告）

#### allowed-tools 字段
- **检查项**: 格式
- **扣分**: 格式问题 -1 分（警告）

### Frontmatter 格式

- **检查项**: YAML 格式正确性
- **扣分**: 格式错误 -10 分
- **说明**: 必须以 `---` 开头和结尾

## 目录结构规则

### 目录名匹配
- **检查项**: 目录名与 name 字段匹配
- **扣分**: 不匹配 -5 分
- **说明**: 目录名必须与 SKILL.md 中的 name 字段一致

### 嵌套深度
- **检查项**: 目录层级
- **扣分**: 超过 2 层嵌套 -1 分
- **说明**: 建议目录结构不超过 2 层深度

### 临时文件
- **检查项**: 不必要的文件
- **扣分**: 无（仅信息提示）
- **说明**: 检测 .DS_Store、Thumbs.db、*.tmp、*.log 等文件

## 脚本代码规则

### Shebang 检查
- **检查项**: 脚本开头是否有 shebang
- **扣分**: 缺少 shebang -1 分（警告）
- **适用**: .sh、.py 文件
- **示例**:
  ```bash
  #!/bin/bash
  #!/usr/bin/env python3
  ```

### 敏感信息检测
- **检查项**: 硬编码的敏感信息
- **扣分**: 发现敏感信息 -5 分（错误）
- **检测模式**:
  - `password = "..."`
  - `api_key = "..."`
  - `secret = "..."`
  - `token = "..."`

### 错误处理（Bash）
- **检查项**: 是否有错误处理
- **扣分**: 缺少错误处理 -1 分（警告）
- **说明**: 建议添加 `set -e` 或 `set -o errexit`

### 注释检查
- **检查项**: 是否有注释
- **扣分**: 无（仅信息提示）
- **说明**: 建议添加注释说明脚本功能

## 文件引用规则

### 引用存在性
- **检查项**: 引用的文件是否存在
- **扣分**: 文件不存在 -5 分（错误）
- **说明**: 检查 SKILL.md 中引用的所有文件

### 文件大小
- **检查项**: 文件大小
- **扣分**: 超过 1MB -1 分（警告）
- **说明**: 建议保持文件大小合理

## 问题级别定义

### ERROR（错误）
- 必须修复的问题
- 严重影响 skill 功能或安全性
- 示例：缺少必需字段、硬编码敏感信息、引用不存在

### WARN（警告）
- 建议修复的问题
- 可能影响使用体验或维护
- 示例：缺少 shebang、description 过短、目录嵌套过深

### INFO（信息）
- 仅供参考
- 不影响功能，但可改善质量
- 示例：缺少注释、存在临时文件

## 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 通过，无问题 |
| 1 | 失败，有错误 |
| 2 | 警告，有警告但无错误 |

## 使用建议

1. **开发阶段**: 频繁运行审查，及时修复问题
2. **发布前**: 确保所有错误已修复，警告已评估
3. **CI/CD 集成**: 设置阈值为 75 分，低于则阻止发布
