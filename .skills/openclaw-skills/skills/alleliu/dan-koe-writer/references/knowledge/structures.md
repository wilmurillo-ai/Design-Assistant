# Structures | 结构模式库

> 内容组织的方式，不同结构适合不同目的。
> 好的结构让复杂的观点变得易于理解和跟随。

---

## 通用结构

### APAG 完整结构（Dan Koe）

```
A - Attention（钩子）
P - Perspective（描绘敌人：指出错误观点）
A - Advantage（描绘英雄：提出正确视角）
G - Gamify（给出可操作步骤）
```

**适用:** Newsletter、长文章、YouTube 脚本

---

### PAS 结构

```
P - Problem（问题）
A - Agitation（放大问题）
S - Solution（解决方案）
```

**适用:** 短帖、推文、落地页

---

### Before/After 结构

```
Before（痛苦现状）
↓
转折点（顿悟/发现）
↓
After（解决后的状态）
```

**适用:** 转化型内容、个人故事

---

### 清单式结构

```
Hook（钩子）
↓
核心观点（一句话）
↓
3-5个要点（每个要点 = 故事/例子 + 可操作建议）
↓
结论 + CTA
```

**适用:** 教育类内容、教程

---

## 短内容结构

### 钩子 + 悖论 + CTA

```
第1行：抓住注意力的钩子
第2行：揭示悖论或反直觉观点
第3行：暗示解决方案
[可选] 第4行：行动号召
```

**适用:** Twitter/X、微博、Instagram

---

### 故事弧

```
简短场景设定
↓
冲突/问题
↓
顿悟（核心观点）
↓
行动建议
```

**适用:** LinkedIn、Medium、微信公众号

---

## 具体示例

### Dan Koe Newsletter 结构

```
1. Hook（个人经历或反直觉观点开头）
2. 描绘问题（放大痛苦，引用常见错误观点）
3. 提出新的思维框架（用故事/例子）
4. 分步解释（编号列表，清晰可操作）
5. 总结 + 行动号召
```

## 结构 (Structures)

### 问题-解决-未来 (来源: 40个AI Agent颠覆营销-J道, 日期: 2026-03-29)
