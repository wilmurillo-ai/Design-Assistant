# HEARTBEAT.md

> Periodic self-check. Read this file when a heartbeat poll arrives.
> If nothing needs attention, reply: HEARTBEAT_OK
> If something needs fixing, describe only the new issue — do not repeat old context.

Check these and report only NEW problems:

- Is `SESSION-STATE.md` stale, uncleaned, or out of date?
- Did a recently completed task miss closeout (hot state not reset, daily not written, stable content not promoted)?
- Are there daily entries from the last 1-2 days with stable preferences/facts that haven't been promoted to structured memory?
- Are there recurring issues that were only logged to daily but not hardened into enforcement layer?
- Is root `MEMORY.md` growing into a journal instead of staying as a short summary?
- Is `memory/system.md` showing clearly outdated information?
