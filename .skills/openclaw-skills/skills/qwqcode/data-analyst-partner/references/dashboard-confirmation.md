# New Dashboard Confirmation Checklist

Use this when someone asks for a new dashboard, new panel group, or a major dashboard extension.

## Required confirmations

1. Audience
- Who will use it?
- Product, content, operations, leadership, or mixed?

2. Decision purpose
- What decision should this dashboard support?
- Daily monitoring, diagnosis, weekly review, campaign follow-up, content optimization?

3. Metrics
- What are the primary metrics?
- Which metrics are must-have versus nice-to-have?

4. Dimensions
- Platform?
- Version?
- Channel?
- Region/language?
- Content type / content tag / speaker?
- Subscription or member status?

5. Time design
- Daily, weekly, monthly?
- Need rolling windows?
- Default time range?

6. Output shape
- Trend charts
- Funnel
- Breakdown table
- Ranking
- KPI tiles

7. Refresh and ownership
- How often should it refresh?
- Who owns metric definitions?
- Who signs off before publishing?

8. Reuse check
- Is there an existing dashboard or panel we can extend?

## What to produce after confirmation

Provide:
- dashboard goal
- target users
- metric list
- dimension list
- chart/module proposal
- reuse plan from existing Grafana assets
- open questions / risks
