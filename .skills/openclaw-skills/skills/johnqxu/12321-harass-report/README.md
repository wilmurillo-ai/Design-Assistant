# 12321 骚扰电话举报自动化

自动化填写 [12321网络不良与垃圾信息举报受理中心](https://www.12321.cn) 的骚扰电话举报表单，通过浏览器交互 + 聊天确认完成举报流程。

## 项目结构

```
12321-harass-report/
├── SKILL.md                    # 主技能定义 - 完整举报流程
├── references/
│   └── field-mapping.md        # 表单字段映射参考
└── README.md
```

## 功能

- 自动打开 12321 举报页面并绕过倒计时
- 自动识别图片验证码（OCR）
- 交互式确认举报信息，支持修改任意字段
- 自动填写并提交举报表单
- 支持短信验证码自动触发与填入

## 举报流程

1. **打开页面** - 访问 12321 举报页面，绕过同意倒计时
2. **验证码识别** - 提取验证码图片，OCR 识别，确认所有举报信息
3. **信息确认** - 用户确认或修改举报内容
4. **验证码 + 手机号** - 填入验证码和手机号，触发短信验证码
5. **填写表单** - 分步填入短信验证码、举报号码、日期时间、骚扰方式等字段
6. **校验提交** - 检查表单校验结果，确认无误后提交

## 默认举报信息

| 字段 | 默认值 |
|------|--------|
| 通话时长 | 3分钟以下 |
| 骚扰方式 | 自动语音骚扰 |
| 不良类型 | 贷款理财 |
| 来电描述 | 近期未经过我的授权，不同的机构、不同的人员给我拨打推销电话、骚扰电话。 |

## 触发方式

当用户说以下关键词时触发：

- "举报骚扰电话"
- "12321举报"
- "投诉骚扰电话"
- "填写举报表单"
- "打开12321"

## 技术要点

- 验证码填入使用 `el.value=` + `dispatchEvent('input')` 方式，必须在同一次 evaluate 中完成
- textarea 使用 `HTMLTextAreaElement.prototype` setter
- 日期/时间字段为 readonly，赋值后需触发 `.click()` 进行校验
- 详细字段映射参见 [references/field-mapping.md](references/field-mapping.md)
