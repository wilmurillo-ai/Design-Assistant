# 豆包视频创作助手 - 最佳实践指南

## 🎯 核心原则

### 1. 提示词确认优先 ✅

**原则**：提示词未确认，绝不开始生成视频！

**原因**：
- 视频生成成本高（¥0.15-0.50/场景）
- 生成时间长（1-2 分钟/场景）
- 提示词不准确会导致视频不符合预期
- 重新生成会产生额外成本

**流程**：
```
生成提示词 → 展示给用户 → 用户确认 → 开始生成
              ↓
         用户要求修改
              ↓
         修改提示词 → 重新确认
```

**确认提示词时展示的信息**：
- 完整提示词内容
- 将生成的画面描述
- 预计时长
- 预计成本
- ⚠️ 重要提醒（确认后无法撤销）

### 2. 逐个场景生成 ✅

**原则**：完成一个场景并确认后，再生成下一个场景！

**原因**：
- 及时发现问题，避免全部生成后才发现问题
- 减少浪费，一个场景不满意只重生成这一个
- 用户参与度高，可以随时调整方向
- 降低总体成本

**错误做法** ❌：
```
场景 1 生成 → 场景 2 生成 → 场景 3 生成 → 用户确认
（同时生成所有场景，用户最后才看到）
```

**正确做法** ✅：
```
场景 1 生成 → 用户确认 → 场景 2 生成 → 用户确认 → 场景 3 生成 → 用户确认
（逐个生成，逐个确认）
```

### 3. 生成方式选择 ✅

**原则**：根据场景复杂度选择合适的生成方式！

| 场景类型 | 推荐方式 | 原因 |
|---------|---------|------|
| 简单场景（产品特写） | 文生视频 | 快速，成本低 |
| 复杂场景（多人物、多元素） | 图生视频 | 画面可控，一致性好 |
| 抽象场景（概念、氛围） | 文生视频 | 无需精确控制 |
| 重要场景（关键镜头） | 图生视频 | 确保效果 |

### 4. 原型图确认（图生视频模式）✅

**原则**：原型图未确认，绝不生成视频！

**流程**：
```
生成原型图 → 用户确认 → 生成提示词 → 用户确认 → 生成视频
              ↓
         用户要求调整
              ↓
         重新生成原型图 → 重新确认
```

### 5. 项目记录完整 ✅

**原则**：所有配置、状态、日志必须完整保存！

**记录内容**：
- API 配置（项目级可覆盖全局）
- 每个场景的生成方式
- 每个场景的原型图路径（如适用）
- 每个场景的提示词及确认状态
- 每个场景的视频路径及确认状态
- 完整生成日志

**好处**：
- 支持中断后恢复
- 完整追溯生成历史
- 便于调试和优化
- 项目状态清晰可见

## 📋 标准操作流程

### 完整流程概览

```
1. 首次配置（仅第一次）
   ↓
2. 需求收集
   ↓
3. 脚本规划 → 用户确认
   ↓
4. 关键元素生成 → 用户确认
   ↓
5. 对每个场景（逐个进行）：
   a. 选择生成方式（文生/图生）
   b. 生成原型图（仅图生） → 用户确认
   c. 生成提示词 → 用户确认 ⚠️
   d. 生成视频
   e. 用户确认视频
   ↓
6. 视频合成
   ↓
7. 交付最终视频
```

## ⚠️ 常见错误及避免方法

### 错误 1：未确认提示词就生成视频 ❌

**后果**：
- 视频不符合预期
- 需要重新生成
- 浪费时间和成本

**避免方法**：
- 严格执行"提示词确认 → 生成视频"流程
- 在提示词确认界面显示⚠️警告
- 用户确认前不调用 API

### 错误 2：批量生成所有场景 ❌

**后果**：
- 发现问题时已生成多个场景
- 浪费更多成本
- 用户无法中途调整

**避免方法**：
- 逐个场景生成
- 每个场景确认后再继续
- 项目状态清晰标记

### 错误 3：忽略原型图确认 ❌

**后果**：
- 图生视频效果不可控
- 场景一致性差
- 需要重新生成

**避免方法**：
- 图生视频模式必须确认原型图
- 原型图不满意及时调整
- 确认后再进入提示词环节

### 错误 4：项目记录不完整 ❌

**后果**：
- 中断后无法恢复
- 无法追溯生成历史
- 难以调试问题

**避免方法**：
- 使用 project.json 完整记录
- 每个步骤都更新状态
- 保存所有确认记录

## 💡 优化建议

### 1. 提示词优化
- 使用具体描述，避免模糊词汇
- 包含镜头语言（推、拉、摇、移）
- 指定光影效果
- 添加质量词（高清、4K、电影级）

### 2. 成本控制
- 简单场景优先使用文生视频
- 重要场景使用图生视频
- 提示词确认后再开始生成
- 避免反复生成

### 3. 用户体验
- 清晰展示每个步骤
- 提供明确的选择选项
- 显示预计时间和成本
- 及时告知进度

### 4. 质量保证
- 每个环节都需用户确认
- 记录用户反馈
- 根据反馈优化提示词
- 确保场景一致性

---

**遵循这些最佳实践，可以最大程度减少反复，提高成功率，降低成本！** 🎯
