## 状态流转

```
collecting → planning → generating_elements → generating → confirming → completed
    │            │              │                │            │
    │            │              │                │            └─ 用户确认最终视频
    │            │              │                └─ 用户确认每个场景视频
    │            │              └─ 用户确认关键元素参考图
    │            └─ 用户确认脚本
    └─ 用户提供参考资料
```

## 项目记录文件结构

每个项目生成一个 `project.json` 文件，包含：

### 1. 基础信息
- `project_id`: 项目唯一标识
- `theme`: 视频主题
- `created_at`: 创建时间
- `updated_at`: 最后更新时间
- `status`: 当前状态

### 2. API 配置
- `api_key`: 豆包 API Key
- `text_to_video_model`: 文生视频模型
- `image_to_video_model`: 图生视频模型
- `configured_at`: 配置时间

### 3. 参考资料
- `documents`: 上传的文档列表
- `urls`: 抓取的网页内容
- `images`: 参考图片
- `notes`: 文字说明

### 4. 脚本
- `total_duration`: 总时长
- `scene_count`: 场景数量
- `scenes[]`: 每个场景的详细信息
  - `id`: 场景 ID
  - `time`: 时间范围
  - `title`: 场景标题
  - `description`: 画面描述
  - `copy`: 文案
  - `style`: 风格
  - `generation_mode`: 生成方式 (text_to_video/image_to_video)
  - `prototype_images`: 原型图路径（图生视频模式）
  - `prompt`: 视频提示词
  - `prompt_confirmed`: 提示词是否确认
  - `video_path`: 生成的视频路径
  - `video_status`: 视频状态 (pending/confirmed/rejected)
  - `status`: 场景状态

### 5. 关键元素
- `characters[]`: 人物参考图
- `scenes[]`: 场景参考图
- `objects[]`: 物品参考图
- 每个元素包含：
  - `id`: 元素 ID
  - `description`: 描述
  - `image_path`: 图片路径
  - `status`: 状态 (pending/confirmed/rejected)
  - `feedback`: 用户反馈（如拒绝）

### 6. 生成日志
- `generation_log[]`: 完整操作日志
  - `timestamp`: 时间戳
  - `action`: 操作类型
  - `details`: 详细信息

## 关键决策点

### 1. 首次配置
- **决策**: 是否已配置 API Key 和模型
- **保存**: `~/.openclaw/workspace/doubao-config.json`

### 2. 生成方式选择
- **决策**: 每个场景选择文生视频或图生视频
- **保存**: `project.json` → `scenes[].generation_mode`

### 3. 原型图确认（仅图生视频）
- **决策**: 是否确认原型图用于视频生成
- **保存**: `project.json` → `scenes[].prototype_images[]`

### 4. 提示词确认
- **决策**: 是否确认视频生成提示词
- **保存**: `project.json` → `scenes[].prompt_confirmed`

### 5. 视频确认
- **决策**: 是否确认生成的视频
- **保存**: `project.json` → `scenes[].video_status`

## 错误处理

### API 调用失败
- 重试机制：最多 3 次
- 记录失败原因到 `generation_log`
- 通知用户并提供手动重试选项

### 用户拒绝
- 记录反馈到 `scenes[].feedback` 或 `key_elements.[].feedback`
- 根据反馈调整提示词或重新生成
- 保留历史版本供参考
