# 技术参数确认指南

## 🎯 核心原则

**在开始脚本规划前，必须确认以下技术参数！**

## 📋 必须确认的技术参数

### 1. 视频总时长 ⏱️

**用户说的是整个视频的总时长！**

**豆包限制**：
- 单个视频生成：**2-15 秒**（推荐 5 秒）
- 通过合成多个场景实现更长视频

**总时长与场景拆分**：
| 用户希望的总时长 | 推荐场景数 | 单个场景时长 |
|----------------|-----------|-------------|
| 10-15 秒 | 2-3 个场景 | 5 秒/场景 |
| 20-30 秒 | 4-6 个场景 | 5 秒/场景 |
| 30-60 秒 | 6-12 个场景 | 5 秒/场景 |
| 60 秒+ | 12+ 个场景 | 5 秒/场景 |

**询问话术**：
```
⏱️ **视频总时长**：
您希望的整个视频时长是多少？
- 15 秒左右（短小精悍，适合社交媒体）
- 30 秒左右（标准时长，推荐）✨
- 60 秒左右（详细介绍）
- 更长（需要多个视频合成）

说明：豆包每个场景生成 5 秒，我会根据您的总时长推荐场景数量～
```

### 2. 视频分辨率/比例 📐

**选项**：
- **16:9** - 横屏（推荐：YouTube、B 站、官网）
- **9:16** - 竖屏（推荐：抖音、快手、朋友圈）
- **1:1** - 正方形（推荐：Instagram、小红书封面）
- **4:3** - 传统比例（较少用）

**询问话术**：
```
📐 **视频比例**：
- 16:9 横屏（YouTube、B 站、官网）📺
- 9:16 竖屏（抖音、快手、朋友圈）📱
- 1:1 正方形（Instagram、小红书）🖼️

主要在哪里发布？我帮您推荐～
```

**推荐逻辑**：
| 发布平台 | 推荐比例 |
|---------|---------|
| 抖音/快手/视频号 | 9:16 竖屏 |
| YouTube/B 站 | 16:9 横屏 |
| 朋友圈 | 9:16 竖屏 或 1:1 |
| 小红书 | 3:4 或 1:1 |
| 官网/展会 | 16:9 横屏 |

### 3. 分辨率质量 🎨

**选项**：
- 480p - 标清（快速测试）
- 720p - 高清（推荐）✨
- 1080p - 全高清（高质量需求）

**询问话术**：
```
🎨 **画质质量**：
- 480p（快速测试）
- 720p（高清，推荐）✨
- 1080p（全高清，高质量）

默认 720p 可以吗？
```

### 4. 帧率 🎬

**选项**：
- 24 fps - 电影感（推荐）✨
- 30 fps - 标准视频
- 60 fps - 流畅动作（游戏、运动）

**询问话术**：
```
🎬 **帧率**：
- 24 fps（电影感，推荐）✨
- 30 fps（标准）
- 60 fps（超流畅，适合运动场景）

默认 24 fps 可以吗？
```

### 5. 场景数量与时长计算 🎭

**重要**：场景数量根据总时长计算！

**计算公式**：
```
场景数量 = 总时长 ÷ 单个场景时长（5 秒）

例如：
- 总时长 30 秒 → 6 个场景（30÷5=6）
- 总时长 60 秒 → 12 个场景（60÷5=12）
```

**推荐配置**：
| 总时长 | 场景数 | 单个时长 | 适用场景 |
|--------|--------|---------|---------|
| 15 秒 | 3 个 | 5 秒 | 产品快闪、朋友圈 |
| 30 秒 | 6 个 | 5 秒 | 标准宣传片 ✨ |
| 60 秒 | 12 个 | 5 秒 | 详细介绍 |
| 90 秒+ | 18+ 个 | 5 秒 | 长篇内容（建议分段） |

**询问话术**：
```
🎭 **场景规划**：

根据您希望的总时长（{{total_duration}}秒），
我建议拆分为 **{{scene_count}}个场景**，每个场景 5 秒。

这样安排可以吗？
- 场景太少：每个场景内容会太拥挤
- 场景太多：制作时间长，成本高

推荐 6-8 个场景（30-40 秒），平衡效果和成本～
```

## ⚠️ 常见错误

### 错误 1：不确认参数就直接开始 ❌
```
用户：做个视频
助手：好的，开始生成...
（默认使用 16:9，但用户想要 9:16 发抖音）
```

**后果**：
- 视频比例不对，需要重新生成
- 浪费时间和成本

### 错误 2：使用专业术语 ❌
```
助手：请问需要 2.35:1 的宽屏比例吗？
用户：？？？
```

**正确做法**：
```
助手：横屏（像电影）还是竖屏（像抖音）？
```

### 错误 3：不给推荐值 ❌
```
助手：时长要多少秒？
用户：不知道，多少都可以
```

**正确做法**：
```
助手：时长 6 秒/12 秒/15 秒？推荐 12 秒，适合大多数场景～
```

## 💡 最佳实践

### 1. 提供推荐值
- 不要只给选项，要给出推荐
- 说明推荐原因
- 用户可以快速选择"标准"

### 2. 根据发布平台推荐
```
用户：发抖音
助手：那推荐 9:16 竖屏，时长 12 秒，适合抖音的信息流展示！
```

### 3. 允许快速选择
```
回复 "标准" - 使用推荐配置
回复 "定制" - 逐项确认
```
