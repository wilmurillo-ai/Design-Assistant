# 需求收集最佳实践

## 📋 核心原则

**主动询问 + 明确告知可提供的资料类型 = 更精准的视频**

## 🎯 需求收集清单

### 必问项（4 个核心问题）

1. **视频主题**：是什么产品/服务？想表达什么内容？
2. **目标受众**：视频给谁看的？
3. **视频风格**：科技感？温馨？专业？活泼？
4. **参考资料**：有没有可以提供的参考资料？

### 可提供的资料类型（明确告知用户）

#### 📄 文档资料
- 产品介绍文档
- 剧本/大纲
- 品牌指南
- 其他文字材料

#### 🔗 网页链接
- 产品官网
- 参考视频链接
- 竞品案例

#### 🖼️ 参考图片（强烈建议！）
- 产品图片
- 参考场景
- 人物形象
- 风格参考

> 💡 **提示**：有参考图片会让视频更精准！

#### 📝 文字描述
- 具体需求说明
- 特殊要求
- 禁忌内容

## 💬 标准对话模板

```
🎬 开始创作您的视频吧！

为了生成更符合您需求的视频，请告诉我：

**1. 视频主题**：是什么产品/服务？想表达什么内容？

**2. 目标受众**：视频给谁看的？

**3. 视频风格**：科技感？温馨？专业？活泼？

**4. 参考资料**（可选，但强烈建议提供）：

📄 **文档资料**：产品介绍、剧本、品牌指南等
🔗 **网页链接**：官网、参考视频、竞品案例
🖼️ **参考图片**：产品图、场景图、风格参考（有图片会让视频更精准！）
📝 **文字描述**：具体需求、特殊要求

您可以直接发送文件、图片、链接或文字描述给我～
```

## ⭐ 为什么要主动询问参考资料？

### 1. 用户可能不知道可以提供
- 很多用户以为只能文字描述
- 明确告知后，用户更愿意提供
- 提供资料后，视频质量显著提升

### 2. 参考资料的价值
| 资料类型 | 价值 |
|---------|------|
| 产品图片 | 确保产品外观准确 |
| 场景参考 | 确保场景风格一致 |
| 竞品案例 | 理解用户期望的效果 |
| 品牌指南 | 确保符合品牌规范 |

### 3. 提高成功率
- 有参考 → 图生视频 → 画面可控
- 无参考 → 文生视频 → 依赖提示词
- 参考资料越多，视频越精准

## 🎯 处理不同类型的参考资料

### 文档处理
1. 读取文档内容
2. 提取关键信息（产品名称、卖点、目标用户）
3. 生成脚本时参考文档内容

### URL 处理
1. 抓取网页内容
2. 提取产品信息和视觉风格
3. 询问用户是否有更具体的参考点

### 图片处理
1. 分析图片内容（产品、场景、风格）
2. 询问用户希望参考哪些方面
3. 生成关键元素参考图时参考

## 💡 追问技巧

### 用户说"随便"时
```
用户：风格随便，你看着办

助手：好的！那我来帮您规划～

为了更好地把握方向，我确认一下：
- 您的产品是面向年轻人还是成熟客户？
- 希望视频给人专业感还是亲和力？
- 有没有喜欢的颜色或色调偏好？

这样我可以更精准地为您设计风格！😊
```

### 用户说"没有参考资料"时
```
用户：没有参考资料

助手：没问题！我们可以用文生视频模式创作。

不过，如果您有以下任何一项，会让视频更精准：
- 产品照片（手机拍的也可以）
- 喜欢的风格参考（其他视频截图）
- 竞品案例链接

您看有可以提供的吗？没有的话我们直接用文字描述开始～
```

### 用户提供模糊描述时
```
用户：要高大上的感觉

助手：好的！"高大上"可以有很多表现形式～

具体来说，您更倾向于：
- 科技感（如苹果发布会风格）
- 奢华感（如奢侈品广告）
- 专业感（如企业宣传片）
- 还是其他风格？

有参考图片或视频链接的话会更好理解哦！
```

## 📊 需求收集检查清单

在开始脚本规划前，确认已收集：

- [ ] 视频主题/用途
- [ ] 目标受众
- [ ] 视频风格偏好
- [ ] 至少一种参考资料（文档/URL/图片/文字）
- [ ] 视频时长预期（默认 12 秒）
- [ ] 特殊要求或禁忌

## ⚠️ 常见错误

### 错误 1：不主动询问参考资料 ❌
```
用户：帮我做个产品视频
助手：好的，什么产品？
（直接开始，没有询问是否有参考资料）
```

**后果**：
- 视频可能不符合用户预期
- 只能依赖文字描述，准确性差

### 错误 2：不告知可提供的资料类型 ❌
```
助手：有参考资料吗？
用户：没有
（用户不知道可以提供图片、URL 等）
```

**后果**：
- 错失提高质量的机会
- 用户可能其实有资料但不知道可以提供

### 错误 3：收到资料后不充分利用 ❌
```
（用户上传了产品图片）
助手：好的，开始生成脚本...
（没有分析图片内容，直接用通用提示词）
```

**后果**：
- 参考资料形同虚设
- 视频与产品实际外观不符

## ✅ 正确做法示例

```
用户：帮我做个智能手表的宣传视频

助手：好的！我是豆包视频创作助手 🎬

为了生成更符合您需求的视频，请告诉我：

1. 视频主题：智能手表的哪些功能想重点展示？
2. 目标受众：给谁看的？（消费者/经销商/内部）
3. 视频风格：科技感？运动感？商务感？
4. 参考资料：
   - 📄 有产品介绍文档吗？
   - 🔗 有官网或参考视频吗？
   - 🖼️ 有产品图片或场景参考吗？（强烈建议！）

您可以直接发送文件、图片、链接给我～

用户：有产品图片，发给你
（发送图片）

助手：收到产品图片！让我分析一下...

✅ 产品分析完成：
- 外观：圆形表盘，银色金属表带
- 屏幕：AMOLED 彩色显示屏
- 风格：商务休闲

现在我来为您生成视频脚本...
```
