# Persona 分析 Prompt

## 任务

你将收到：
1. 用户手动填写的基础信息（称呼、关系阶段、依恋风格标签、关系特质标签、主观印象）
2. `chat_analyzer.md` 已输出的聊天分析结果

综合两部分，生成用于构建 Persona 的结构化人格数据。

**优先级规则：手动标签 > 聊天记录分析结果。**

---

## 综合分析维度

### 1. 核心性格定型

基于手动标签和聊天分析，提炼 3-5 条"核心法则"：

- **每条必须是具体行为，不能是形容词**
- 格式："当 [场景] 时，TA 会 [具体行为]，而不是 [你可能误以为的行为]"

示例（根据实际标签生成，不要照抄）：
```
- 当你回复慢了，TA 不会直接问你在干嘛，而是先发一个"..."然后等你
- 当 TA 生气了，TA 不会说"我很生气"，而是回复变成一个字或者直接不回
- 当 TA 想和好，TA 不会道歉，而是突然发一个跟什么都没关系的表情包
```

---

### 2. 表达风格画像

整合聊天分析的数据，输出：

```
口头禅：["xxx", ...]
高频词：["xxx", ...]
招牌 emoji：[列表，附场景]
句式风格：[短句/长句，结论前置/铺垫型，正式程度 1-5]
回复节奏：[秒回/慢回/看情绪，具体描述]
情绪信号：[TA 开心时怎么说/生气时怎么说/敷衍时怎么说]
```

---

### 3. 情感行为模式

输出以下结构（每项附 1-2 条原文引用）：

```
表达在乎：[方式 + 原文举例]
表达不满：[方式 + 原文举例]
道歉方式：[直接/绕弯子/用行动代替语言]
说"喜欢"的情境：[什么时候说，怎么说]
```

---

### 4. 冲突与和解链

输出完整的典型冲突序列：

```
触发点：[列表，附场景]
第一反应：[描述]
升级方式：[描述 + 典型话术]
冷战模式：[会/不会，持续多久，谁先开口]
和解信号：[TA 发出的和解信号，附原文]
```

---

### 5. 关系角色行为

```
主动联系频率：[描述]
主动联系触发场景：[列表]
消失预兆：[描述]
消失时的处理方式：[描述]
重新出现的开场：[描述 + 典型话术]
边界话题：[TA 会回避或拒绝的话题]
```

---

### 6. 关系动态总结

用 3-5 句话描述你们关系的整体模式：
- TA 在关系中扮演什么角色（追求者/被追者/轮流）
- TA 对这段关系的态度（认真/将就/不确定）
- TA 分手（如已分）的真实原因（基于聊天记录推断）

---

## 输出要求

- 语言：中文
- 没有依据的维度：标注 `（原材料不足）`
- 有原文依据的结论：直接引用（用引号）
- 手动标签与分析结果冲突：输出两个版本并注明
- 输出结果直接用于生成 persona.md，要求具体可执行
