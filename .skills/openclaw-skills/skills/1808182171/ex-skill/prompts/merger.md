# 增量 Merge Prompt

## 任务

用户追加了新的聊天记录或截图。将新内容的增量 merge 进已有的 `persona.md`，不覆盖已有结论。

---

## 输入

1. 已有的 `persona.md`（当前版本）
2. 新增内容（聊天记录文本 / 截图分析结果）

---

## Merge 规则

### 原则
- **只追加增量**，不删除或修改已有内容（除非新内容直接矛盾）
- **有冲突时**：新证据优先于旧推断；但如果旧内容是用户手动标注的（Correction 层），保留旧的并注明冲突

### 处理步骤

1. 分析新内容，用 `chat_analyzer.md` 提取新的行为数据
2. 对比新旧内容：
   - **相同结论**：跳过，不重复写
   - **新增证据**：追加到对应层，用"（新增: {日期}）"标注
   - **直接冲突**：在原文旁边加注 `⚠️ 新记录与此矛盾：{新内容摘要}`，供用户判断
3. 更新 `meta.json` 中的消息数量和版本号

### 各层 Merge 方向

| 新内容类型 | Merge 目标 |
|-----------|-----------|
| 新的口头禅/高频词 | Layer 2 表达风格 |
| 新的 emoji 用法 | Layer 2 招牌 emoji |
| 新的冲突场景 | Layer 4 冲突链 |
| 新的和解方式 | Layer 4 和解信号 |
| 新的情感表达 | Layer 3 情感行为模式 |
| 与现有结论矛盾的内容 | 加注 ⚠️ 标记，不自动覆盖 |

---

## 输出

输出 `persona.md` 的 diff（增量部分），格式：

```
[Merge 报告]
新增消息数：{N} 条
新增/修改的层：{列表}
发现冲突：{有/无，如有则列出}

[具体变更]
{按层列出变更内容}
```

然后输出更新后的完整 `persona.md`。
