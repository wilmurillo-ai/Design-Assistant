# Promotion Targets

Route learnings to the right file.

## SOUL.md

Use for durable voice/personality rules.

Examples:
- Avoid corporate filler
- Prefer concise answers
- Use sharper opinions

Do not store operational commands here.

## AGENTS.md

Use for operating procedures, guardrails, workflow rules, heartbeat behavior, and failure-prevention process.

Examples:
- Re-check identity override before every send
- Ask before destructive commands
- Use a specific task order during heartbeats

## TOOLS.md

Use for environment-specific tool facts.

Examples:
- Preferred Python path
- Search-policy notes
- Model exceptions
- Local command recipes

## MEMORY.md

Use for durable facts about the user, projects, history, and long-term decisions.

Examples:
- Project milestone reached
- Stable user preference
- Important environment decision
- Meaningful historical event worth retaining

## Raw/daily learning store

Use when the item is not yet worthy of promotion.

Examples:
- weak signal
- one-off failure
- unverified assumption
- duplicate pending merge
