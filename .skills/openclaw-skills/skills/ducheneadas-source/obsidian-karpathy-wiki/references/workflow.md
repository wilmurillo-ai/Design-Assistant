# Workflow

## Ingest

Use this when new material enters the vault.

1. Read the raw note, article, reflection, or draft.
2. Decide whether it is source material, output material, or already-stable formal knowledge.
3. If it is a loose root note, move it into a raw inbox unless the user wants otherwise.
4. Create or update a source summary page.
5. Extract the minimum stable upgrades:
   - question
   - method
   - decision
   - principle
   - concept
   - entity
6. Update index and log.

## Query

1. Read the index.
2. Start from question, method, and decision pages.
3. Pull principles for "why", and sources for evidence.
4. Save reusable outputs back into the wiki.
5. Revisit saved outputs during later maintenance and decide whether to promote them.

## Lint

Look for:

- orphan formal pages
- weak source-to-formal connections
- repeated questions without shared methods
- methods without boundaries
- decisions without current validity
- contradictions between pages
- stale claims superseded by newer sources
- principles that are not actually more general than adjacent methods
- outputs accumulating without write-back

## Migration

When converting an existing vault:

1. Do not immediately archive old material.
2. First classify old material into source batches.
3. Create source summaries.
4. Promote only the strongest stable themes.
5. Create or update outputs for useful existing syntheses.
6. Archive duplicates, stale governance, and obsolete copies last.
