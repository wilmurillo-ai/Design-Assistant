# Content Rules — Xiaohongshu

## Writing Style

- 接地气、直接、有技术自信，像程序员说话
- 能用数字就用数字，不说"很多""非常"
- 短句换行，移动端友好
- 结尾最多一个 👇

## Post Structure

```
标题：20字以内，用数字/痛点/对比/好奇四选一
正文：
  1. 开门见山（1-2句）
  2. 核心内容（2-4个要点，emoji 序号）
  3. 具体细节（数字、命令、真实案例）
  4. 互动问句（结尾一句）
标签：5-8个
字数：500字以内
```

## Sensitive Word Blacklist

Any of these trigger an audit failure:

**Platform names** (会被限流/封号):
Telegram, 微信, WeChat, API, MCP, 二维码

**Solicitation** (引流违规):
加我, 私信, 关注我, 点进主页

**Financial** (金融敏感):
Binance, 币安, SOL, ETH, 加密货币, 合约, 收益, 回报, 盈利, 投资建议, 跟我做

**Absolute claims** (广告法违规):
最, 第一, 100%, 绝对, 保证

**Safe alternatives**:
- API → 自动化接口 / 编程接口
- MCP → 本地服务 / 自动化工具
- 微信 → (avoid or use screenshot)

## Comment Rules

- 50-120 characters
- Add a real opinion not in the original post
- Include specific details
- No ads, no account mentions, no follow solicitation
- Different structure for each comment (no repetition)
- Random interval 3-8 min between posts
- Daily limit: 15 comments
