# Experiment Your Way：实验作为 PD 系统的底层引擎

> 来源：Ali Abdaal《Feel-Good Productivity》Experiment Your Way 部分，与 PD 系统联动整合
> 来源：复盘-迭代中枢模型（MEMORY.md）、PD 2.0 框架

---

## 核心命题

**不要死记硬背你的生产力系统。用实验的方式走出来。**

传统的生产力思维是线性的：找到一个"正确"的方法，然后坚持执行。但这种方法有一个致命缺陷——你和你的环境都在不断变化。上个月有效的东西，这个月可能就不再适用了。去年让你感觉良好的策略，今年可能让你倦怠。

FGP 和 PD 系统的共同底层假设是：**只有持续实验什么对你有效，才能长期感觉良好、持续产出。**

## 为什么实验心态是 PD 的底层引擎

### PD 系统的运转依赖实验

PD 系统的每个核心组件都内置了实验机制，但大多数时候我们没有把"实验"当成显性框架来使用：

**习惯建立（Atomic Habits）**  
每一条"替换习惯"本质上都是一个实验。你假设"这个替换方案会有效"，然后去测试它。Bullseye Framework 本身就是选择实验路径的工具。

**学习力系统（YJango）**  
知识晶体和 SM 卡片的制作本质上是实验——测试"这个知识在我需要时能否被提取出来"。如果提取失败，就重新制卡。这不是背诵，是持续优化。

**清晰思考（沙恩·帕里什）**  
决策四步骤是一个实验框架：界定问题（提出假设）→ 生成选项（设计实验）→ 评估（收集数据）→ 决策（得出结论）。每个决策都是一次实验。

**SuperMemo**  
整个渐进学习系统就是一场持续几十年的实验——测试"什么间隔、什么卡片格式、什么提取强度"最适合你的记忆和学习。

**Feel-Good Productivity**  
每个工具（便签魔法、自信开关、五分钟法则等）都是可测试的假设。你相信它会提升你的情绪/能量/生产力——但只有亲自测试才能知道。

### 复盘-迭代中枢 = 实验的执行层

复盘-迭代中枢模型（PD 系统的核心迭代机制）天然就是实验的执行框架：

**G（Goal）** → 定义当前实验的目标  
**R（Result）** → 运行实验，收集结果  
**A（Analysis）** → 分析结果：什么有效？什么无效？  
**I（Insight）** → 提炼洞察，更新你的个人方法论

没有实验，复盘就只是"事后诸葛亮"——评价自己做得怎么样，但没有改变未来的方向。

没有复盘，实验就只是"随机试错"——做了很多尝试，但没有提炼出可迁移的洞察。

**实验 + 复盘 = 有方向的高效迭代。**

## 实验心态的三个核心原则

### 原则1：低赌注，高频率

好的实验不是"赌上一切"的重大决策，而是**小步快跑的持续测试**。

**"低赌注"** 意味着：
- 你在尝试一个新工具时，不期待它立刻改变一切
- 失败是预期的，不是灾难
- 一个实验的失败不能否定你的价值

**"高频率"** 意味着：
- 每周至少有一个主动实验
- 实验之间有足够的时间让你收集数据，但不至于拖太久没有反馈
- 一年下来，你测试了50个不同的小策略

### 原则2：定义你的成功指标

每个实验开始前，先明确：**"我怎么知道这个实验成功了？"**

对于 FGP 工具来说，这个问题很简单：

> 这个工具对我的情绪有什么影响？对我的能量有什么影响？对我的生产力有什么影响？

对于 PD 系统的实验来说，指标可能更具体：

- 这个新习惯坚持了几天？中断的原因是什么？
- 这个任务分块方式让我感觉更清晰还是更焦虑？
- 这个问责机制真的让我更愿意开始了吗？

**没有量化指标，实验就变成了感受。感受是可以自欺欺人的，数据不能。**

### 原则3：区分实验失败和人生失败

惯性会让我们把"实验没效果"等同于"我不行"。但这两件事完全不同：

- **实验没效果**：这个策略不适合当前的我，调整它或放弃它
- **我不行**：我的身份有问题，需要自我否定

自我原谅（Forgive Yourself）之所以是 FGP 最被低估的工具，正是因为它打破了"实验失败 = 我很糟糕"的等式。

**当你原谅自己并继续实验时，你是科学家。**  
**当你自我打击并停止尝试时，你是放弃者。**

PD 系统里，复盘-迭代中枢的心智校准明确要求："拥抱失败视为低成本试错；关注可控因素。"这正是实验心态的体现。

## PD 系统中的实验运行指南

### 实验模板

在开始每个实验前，填写以下内容：

```
## 我的实验：[工具/策略名称]

**假设**：我相信这个会……
**测试周期**：X天（建议2-4周）
**Baseline（实验前）**：
  - 情绪状态（1-5）：
  - 能量水平（1-5）：
  - 目标活动产出：
**预期结果**：我期待看到……
**实际结果**（实验后填写）：
**分析**：为什么有效/无效？
**决策**：保留 / 调整后保留 / 放弃
**洞察**（可迁移的原则）：
```

### 优先实验清单

以下是你最值得优先测试的区域（基于 PD 系统的核心痛点）：

**高优先级实验区域**

| 区域 | 典型实验 | 对应 PD 组件 |
|:---|:---|:---|
| SM 节奏 | 测试不同复习时间块（Peak时间 vs 碎片时间）| 学习力 |
| 任务分块 | 测试不同分块粒度（25分钟 vs 55分钟 vs 90分钟）| 清晰思考 |
| 环境设计 | 测试不同工作环境（宿舍 vs 图书馆 vs 教室）| 习惯建立 |
| 问责机制 | 测试不同问责形式（AI伙伴 vs 真人伙伴 vs 公开承诺）| 承诺阶梯 |
| Burner 恢复 | 测试不同恢复策略对能量水平的实际影响 | FGP / 倦怠预防 |

### Alignment Experiments（对齐实验）

除了工具实验，还有一类更重要的实验：**价值观对齐实验**。

每周末问自己：

1. 这周我在做什么？
2. 我为什么在做这些？
3. 这些和我的核心价值观一致吗？
4. 如果不一致，我需要改变什么？

这不是一次性的问题——它是一个需要持续迭代的实验。你的"核心价值观"本身也会随着实验而演变。

## 实验与复盘的联动

### 日常层面

每天结束前，花2分钟做一个微型实验复盘：

- 今天我测试了什么？
- 结果怎么样？
- 明天要调整什么？

这不需要一个完整的 GRAI 复盘，只是保持实验意识的最小习惯。

### 周复盘层面

周复盘时，把"实验记录"作为一个标准板块：

```
## 本周实验回顾
- [实验1]：结果/决策
- [实验2]：结果/决策
- [实验3]：结果/决策

## 下周实验计划
- [实验A]：假设/预期/测试周期
```

### 月度层面

月度对齐实验：评估过去一个月，哪些实验洞察进入了你的日常行为？哪些实验结果彻底改变了你的方法论？

## 最终洞察

Ali 在 FGP 结尾的这段话，值得作为 PD 系统的底层信条：

> "不要死记硬背你的 feel-good productivity 之路。用实验的方式走出来。享受这个过程。这个过程不是关于追求完美，而是关于有策略地跌跌撞撞地找到适合自己的方式。从失败中学习，为成功而庆祝。把工作从消耗你资源的负担转变为给你赋能的来源。"

**生产力不是一个目的地，是一个持续进行的实验。**  
**你不是在"执行"一个系统，你是在通过实验不断进化你的系统。**
