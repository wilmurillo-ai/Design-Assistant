# Feel-Good Productivity 整合指南

> **整合状态**: Feel-Good 作为 PD Skill 的能量管理层
> **核心原则**: "感觉良好带来成功" + "精准资源配置"

---

## 整合架构

```
PD Skill (顶层生产力操作系统)
├── 框架层: 目标/项目/任务/习惯/复盘
├── 能量层: Feel-Good Productivity ← 你在这里
│   ├── Energise (Play/Power/People)
│   ├── Unblock (Uncertainty/Fear/Inertia)
│   └── Sustain (Four Burners)
├── 分析层: 控制论系统思维
└── 执行层: 七阶梯工具 (可选)
```

## 快速参考: Feel-Good + PD 工作流

### 每日流程
```
Morning (PD Planning + Feel-Good Check)
├── Energy level assessment
├── Burner status check
├── Energiser integration
└── Blocker prevention

Throughout Day (Execution)
├── Task execution
├── Blocker identification (if stuck)
├── Energiser top-up (if drained)
└── Burner awareness

Evening (PD Review + Feel-Good Reflection)
├── What flowed? (Energise)
├── What blocked? (Unblock)
├── Burner balance? (Sustain)
└── Identity votes cast
```

### 每周流程
```
Weekly Review (PD + Feel-Good)
├── Goal progress (PD)          → commitments/1_intent/active.md
├── Project status (PD)          → commitments/2_queued/active.md
├── Next actions update (PD)     → commitments/3_committed/next-actions.md + this-week.md
├── Waiting/blockers (PD)       → commitments/3_committed/waiting.md
├── Energise audit (Feel-Good)
├── Unblock patterns (Feel-Good)
├── Burner balance (Feel-Good)
└── System adjustments (Both)
```

## 关键原则

1. **能量优先于时间**: PD 规划前先检查 Feel-Good 能量状态
2. **阻力即信息**: 用 Unblock 框架诊断拖延，而非自责
3. **Burner 预警**: PD 的 dashboard 必须包含 Burner 检查
4. **身份驱动**: PD 的 goals 要连接到 Feel-Good 的身份认同
5. **系统整合**: 不是两个系统，而是 PD 框架 + Feel-Good 血肉

---

## 使用提示

**当 Feel-Good 单独不够时**:
- 需要结构化执行 → 使用 PD 的项目/任务管理
- 需要长期追踪 → 使用 PD 的 review 系统
- 需要习惯固化 → 使用 PD 的 habits/ 文件夹

**当 PD 单独不够时**:
- 感到 drained → 调用 Feel-Good Energise
- 无法启动 → 调用 Feel-Good Unblock
- 倦怠风险 → 调用 Feel-Good Sustain

**最佳实践**: 始终同时考虑两者——PD 给结构，Feel-Good 给能量。
