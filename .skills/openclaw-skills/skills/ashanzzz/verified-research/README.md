# Deep Research Skill — README

## 什么是深度研究？

深度研究不只是"搜一下"。它是一个**多轮交叉验证**的系统方法：
- 每个结论至少 3 个独立来源
- 4 级来源可信度分级（T1=官方/学术 → T4=社交媒体）
- BLOCKING 步骤：关键结论必须用不同关键词核实 2 次

## 研究缓存系统（2026-04-06 新增）

### 为什么有缓存？

研究一个复杂问题可能需要多轮搜索 + 数小时/数天时间。缓存系统确保：
1. **不丢进度** — 重启后继续上次的研究
2. **3天自动归档** — 研究完成后保留3天，无活动则自动写入 MEMORY.md 并清理
3. **可继续** — 说"继续研究XX"即可从上次断点继续

### 缓存放在哪？

```
/tmp/deep-research-cache/{话题slug}/{时间戳}/
```

> 注意：缓存在容器重启后会清空。如需永久保留报告，完成后请复制到 workspace 或 DSM。

### 3天清理规则

| 情况 | 发生什么 |
|------|----------|
| 研究完成后 3 天内继续研究 | 取消清理计划，继续追加证据 |
| 研究完成后 3 天无活动 | 摘要写入 MEMORY.md，缓存目录删除 |
| 报告已生成后复制到 workspace | 原始缓存仍按规则清理（MEMORY.md 已有摘要） |

### 如何继续研究？

说"继续研究{原话题}"，AI agent 会自动找到最新缓存目录继续。

### 如何保留报告？

研究完成后告诉 AI agent "帮我把报告保存到 workspace"，它会复制报告内容到 workspace 目录。

## 快速开始

1. 说"帮我研究一下XXX"
2. AI agent 开始多轮搜索
3. 每轮证据自动写入缓存
4. 研究完成后，报告显示给你，缓存保留 3 天
5. 如果需要继续，随时说"继续研究XXX"

## 脚本说明

| 脚本 | 作用 |
|------|------|
| `research.sh` | 初始化研究会话，创建缓存目录 |
| `claim-card.sh` | 写入单条证据卡（YAML frontmatter + 内容） |
| `manifest.sh` | 扫描所有证据卡，生成 manifest.json |
| `finalize.sh` | 生成 report_final.md，写入清理计划 |
| `cleanup.sh` | 检查并执行 3 天清理逻辑 |

## 来源可信度分级

| 等级 | 例子 | 权重 |
|------|------|------|
| T1 | 官方文档、学术论文、一手数据 | 最高 |
| T2 | 权威媒体、行业白皮书、官方公告 | 高 |
| T3 | 技术博客、社区讨论、GitHub Issues | 中 |
| T4 | 社交媒体、无法溯源的引用 | 低（需交叉验证） |
