# Docs Map

Use these docs for different questions.

## Architecture / design

- `docs/designs/MEMORYLAB_MEMORY_SYSTEM_BLUEPRINT.md`

Use when the question is about:
- why the system is shaped this way
- record types / layers / boundaries
- compatibility with context management

## Canonical implementation status

- `memory-system/MILESTONE_STATUS.md`

Use when the question is about:
- what is complete right now
- which milestone is closed
- current risks / follow-up notes

## Compact restore summary

- `docs/designs/MEMORYLAB_MEMORY_SYSTEM_MILESTONE_RESTORE.md`

Use when resuming work quickly without replaying old chat.

## Execution checklist

- `memory-system/BUILD_PLAN.md`

Use when you need the ordered checklist of work.
Do **not** use it as the canonical status source.

## Current live context

- `memory/ACTIVE_TASK_STATE.md`
- `memory/LIVE_CONTEXT_PACKET.md`

Use when you need the current working state rather than milestone/project status.
