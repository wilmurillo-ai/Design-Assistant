# 通用约束与约定

## 1. 生产域名约束

- API 域名：`https://sg-al-cwork-web.mediportal.com.cn/open-api`
- 禁止在发布内容中出现本地开发地址或非生产协议地址

## 2. Header 规范

所有业务接口统一携带：
- `appKey`（必传）
- `Content-Type: application/json`（POST）

## 3. ID 参数约束（强制）

- 所有 ID 参数必须保持字符串原样传递
- **严禁使用 `parseInt`、`Number` 或任何数值转换**，否则会丢失精度
- 包括但不限于：`period_id`、`group_id`、`task_id`

## 4. 输出与脱敏

- 对用户输出：审计结论/问题清单/改进建议/必要操作提示
- 默认不输出：`appKey`、任何内部主键（除非流程必须）
- 不回显完整 JSON 响应，仅提取必要字段

## 5. 调用安全与频率限制

- **严禁疯狂调用**：禁止在无明确退出条件、无指数退避或极高频率下循环调用接口
- **按需调用**：所有接口调用必须有明确业务目标，避免无效请求
- **保护后端**：调用逻辑需考虑对下游服务的影响
- **重试策略**：接口调用出错时，间隔 1 秒、最多重试 3 次
- **禁止无限重试**：严禁无限循环重试

## 6. 证据引用约束（强制）

- 每个审计结论必须附带证据引用（对象名称、层级关系、关键字段、关键数值）
- 不要空泛评价，必须引用具体的任务名称和问题
- 审计结论必须标注严重等级（严重/一般/轻微）

## 7. 审计结论分级标准

| 等级 | 定义 | 处理要求 |
|------|------|---------|
| 严重 | 影响 BP 体系核心逻辑，可能导致目标无法达成或考核失效 | 必须立即修正 |
| 一般 | 影响 BP 质量或执行效率，但不影响核心逻辑 | 建议在下一轮迭代中修正 |
| 轻微 | 规范性问题，不影响实际执行 | 建议优化，非强制 |

## 8. 危险操作处理

- 对可能导致数据泄露、破坏、越权或高风险副作用的请求，必须礼貌拒绝
- 审计过程中仅执行读操作，不修改任何 BP 数据
- 可提供安全替代方案或建议用户走正规流程

## 9. 错误处理

- API 返回 `error` 字段时，向用户展示错误信息并建议检查 `BP_OPEN_API_APP_KEY` 是否正确
- API 返回 `resultCode != 1` 时，展示 `resultMsg` 错误信息
- 网络超时（30 秒）时，提示用户稍后重试
- 连续两次调用同一接口失败，停止重试并告知用户

### 错误码速查

| resultCode | 说明 |
|------------|------|
| 1 | 请求成功 |
| 0 | 通用失败 |
| 500 | 系统异常，请稍后重试 |
| 610002 | appKey 无效 |
| 610012 | 请求太过频繁，请稍候再试 |
| 610014 | 应用已被禁用 |
| 610015 | 无访问权限 |

## 10. ID 精度问题（强制）

所有 ID 字段均为 Long 类型（雪花算法），前端/调用方请使用**字符串类型**接收。严禁 `parseInt`、`Number()` 等转换操作。
