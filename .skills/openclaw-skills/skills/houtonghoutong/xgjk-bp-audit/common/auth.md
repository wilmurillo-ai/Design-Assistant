# 认证与鉴权（统一前置规则）

## 1. 鉴权方式

BP Open-API 使用 `appKey` Header 鉴权，无需额外的 token 换取流程。

## 2. 统一前置鉴权规则（强约束）

1. **优先级 1（环境变量）**：读取 `XG_BIZ_API_KEY`。存在则直接作为 `appKey` Header 使用。
2. **优先级 2（上下文）**：若无环境变量，尝试从上下文中读取已配置的 appKey。
3. **优先级 3（向用户索取）**：仍无 appKey 时，向用户索取appKey。
4. **禁区**：禁止向用户暴露 appKey 的具体值或内部鉴权细节。

## 3. 请求头规范

所有业务请求统一携带：

| Header | 说明 |
|--------|------|
| `appKey` | appKey  认证密钥（必传） |
| `Content-Type` | POST 请求传 `application/json` |

## 4. 权限与生命周期（安全要求）

- **最小权限**：仅使用当前任务所需能力范围，不扩展权限范围。
- **禁止落盘**：`appKey` 不得写入文件或日志，仅允许环境变量或内存级使用。
