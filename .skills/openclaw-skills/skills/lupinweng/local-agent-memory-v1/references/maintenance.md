# Maintenance and Governance

## What belongs in long-term memory

Promote information when it is:
- stable across future tasks
- likely to matter across sessions
- a reusable workflow
- a durable operating or platform constraint

## What should stay in daily logs

Keep information in daily memory when it is:
- one-off
- noisy
- temporary
- historical detail without future value

## Heartbeat maintenance

During background maintenance:
1. Read `MEMORY.md`
2. Read recent daily logs
3. Look for repeated facts or workflows
4. Extract stable facts into semantic memory
5. Extract repeatable methods into procedural memory
6. Prune low-value lines from `MEMORY.md`

## Guardrails

- Do not mix memory maintenance with unrelated code edits
- Do not auto-migrate the whole memory tree without explicit direction
- Do not act on stale remembered facts without re-verification
