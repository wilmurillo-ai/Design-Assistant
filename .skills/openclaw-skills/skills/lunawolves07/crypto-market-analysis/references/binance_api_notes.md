# Binance API 使用说明与限制

本节提供 `crypto-market-analysis` 技能在与币安（Binance）API 交互时的一些说明和注意事项。

## 公共 API 数据获取

`crypto-market-analysis` 技能主要通过币安的公共 REST API 获取以下数据：

- **K线数据 (Klines)**: 用于计算技术指标，支持不同时间周期 (例如 `1h`, `1d`)。
  - API 端点: `/api/v3/klines`
  - 获取最近的数据，`limit` 参数通常用于控制返回的K线数量（本技能默认获取500条）。

- **24小时行情价格变动情况 (24hr Ticker Price Change Statistics)**: 用于获取当前价格、24小时变化百分比、24小时最高价、最低价和交易量。
  - API 端点: `/api/v3/ticker/24hr`

## 数据限制与缺失字段

目前技能版本通过上述公共 API 无法直接获取以下信息：

- **市值 (Market Cap)**
- **流通量 (Circulating Supply)**
- **总供应量 (Total Supply)**
- **历史最高价 (All-Time High)**
- **历史最低价 (All-Time Low)**

这些数据通常需要从专业的加密货币市场数据聚合平台（如 CoinGecko, CoinMarketCap）的 API 获取，或者可能需要通过更复杂的币安高级 API（例如涉及账户或交易数据）进行计算或查询。在本技能的当前实现中，这些字段将显示为 "N/A"。如果这些信息对用户非常重要，可以考虑未来集成第三方数据源。

## API 速率限制 (Rate Limits)

币安 API 对请求有严格的速率限制。如果在短时间内发送过多请求，可能会触发速率限制，导致请求被拒绝并返回错误。

- **请求权重 (Request Weight)**: 不同的 API 端点有不同的请求权重。`/api/v3/klines` 和 `/api/v3/ticker/24hr` 相对较低，但频繁调用仍需注意。
- **处理方式**: 本技能在设计时会尽量优化请求频率。如果遇到速率限制错误，系统可能会暂停一段时间后重试，或者报告错误。

## 错误处理

- **网络错误**: 如果遇到网络连接问题，技能会报告连接失败。
- **API 响应错误**: 如果币安 API 返回错误状态码（例如4xx或5xx），技能会捕获错误信息并报告。
- **数据解析错误**: 如果API返回的数据格式异常，技能会进行错误处理。

了解这些限制有助于更有效地使用技能，并对报告中的数据有清晰的预期。
