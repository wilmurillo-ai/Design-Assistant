# QA Checklist for Lodging Guides

Clear this checklist before exporting PDF.

## 1. Data integrity

- [ ] Every route number was checked after the latest hotel decision.
- [ ] Scenic latest entry / opening hours were verified.
- [ ] Hotel address and phone are correct if shown.
- [ ] Parking claims are verified if self-drive matters.
- [ ] No schedule-critical number was guessed.

## 2. Itinerary logic

- [ ] Day 1 arrival and check-in timing are realistic.
- [ ] Day 1 dinner logic still matches the chosen hotel location.
- [ ] Each scenic day leaves enough usable time on site.
- [ ] Holiday traffic buffer is explicitly reserved.
- [ ] If the hotel moved, dependent walking/driving assumptions were updated.

## 3. Visual quality

- [ ] No blank screenshots.
- [ ] No QR-heavy screenshots.
- [ ] No full-page clutter screenshots.
- [ ] Images justify recommendations instead of filling space.

## 4. Copy quality

- [ ] Tone is formal enough for a guide.
- [ ] The selected hotel and fallback hotel are clearly stated.
- [ ] Tradeoffs are explicit.
- [ ] Repetition was trimmed.

## 5. Output quality

- [ ] HTML renders cleanly before export.
- [ ] File name matches the lodging scenario.
- [ ] Local file delivery uses relative `MEDIA:./...` paths when applicable.
