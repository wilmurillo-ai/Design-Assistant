# Coupang Page Structure

이 문서는 쿠팡 페이지를 relay로 읽고 검수할 때 어떤 정보를 우선 확인해야 하는지 정리한 참조입니다.

이 구조는 2026-04-04 기준의 공개 쿠팡 검색/상품/장바구니 페이지와 실제 라이브 테스트 관찰을 함께 반영했습니다. 주문/결제 화면 항목은 로그인 후 보이는 동적 화면을 전제로 한 운영용 체크리스트이며, 실제 화면에 따라 일부 표현은 달라질 수 있습니다.

## 1. 검색 결과 페이지

검색 결과 페이지에서 우선 읽을 정보:
- 상품명
- 판매가
- 단위 가격이나 중량 정보
- 로켓배송, 로켓와우, 로켓프레시 같은 배송 배지
- 도착 예정 문구
- 평점과 리뷰 수
- 광고 여부가 명확하면 광고 표시

읽는 순서:
1. 검색어가 상단 검색창에 올바르게 들어갔는지 확인
2. 첫 화면에 보이는 결과 중 광고를 구분
3. 상품 카드에서 가격과 배송 문구를 먼저 읽음
4. 비슷한 상품은 용량, 옵션, 판매 단위로 구분

실전 메모:
- 공개 검색 결과 카드에는 상품명, 가격, `내일 도착 보장`, `와우는 무료배송`, 리뷰 수가 함께 보이는 경우가 많습니다.
- 식품은 `500g`, `2L 12개` 같은 중량/규격이 상품명 중간에 섞여 있으므로 상품명만 읽지 말고 규격도 함께 읽어야 합니다.
- 클릭 후 새 탭이 열릴 수 있으므로 검색 결과 번호는 현재 화면 기준 임시 식별자로만 사용합니다.

## 2. 상품 상세 페이지

상품 상세에서 우선 읽을 정보:
- 최종 상품명
- 선택된 옵션명 또는 필수 옵션 존재 여부
- 현재 판매가
- 배송 방식과 도착 예정
- 판매자 또는 판매 형태
- 수량 UI
- `장바구니 담기`, `바로구매` 버튼 위치

장바구니 담기 전에 확인할 것:
1. 검색 결과에서 본 상품과 상세 상품명이 크게 다르지 않은가
2. 용량, 옵션, 수량이 의도와 맞는가
3. 가격이 검색 결과 대비 비정상적으로 변하지 않았는가
4. 냉장/냉동/실온, 로켓배송/일반배송 등 중요한 조건이 달라지지 않았는가

옵션형 상품 주의:
- 색상, 사이즈, 맛, 세트 구성이 필요한 상품은 옵션을 임의로 선택하지 않습니다.
- 옵션 선택 드롭다운이나 버튼이 뜨면 사용자에게 정확한 옵션을 다시 확인합니다.
- 옵션이 선택되지 않으면 `장바구니 담기`가 실패하거나 다른 품목이 담길 수 있습니다.

실전 메모:
- 공개 상품 페이지에는 수량 입력 UI와 `장바구니 담기`, `바로구매` 버튼이 함께 보입니다.
- 상세 페이지에는 배송정보, 무료배송 여부, 적립, 판매자 평가 같은 보조 정보가 섞여 있습니다.
- `바로구매`가 `장바구니 담기` 옆에 붙어 있는 경우가 많으므로 클릭 대상 오인에 특히 주의해야 합니다.

## 3. 장바구니 페이지

장바구니에서 우선 읽을 정보:
- 항목 순서
- 상품명
- 옵션/규격
- 수량
- 개별 가격
- 묶음배송/판매자 구분
- 주문 예상 금액 영역

삭제나 수량 변경 전 확인할 것:
1. 같은 상품이 여러 개면 옵션과 가격까지 같이 읽음
2. 상품명만 비슷한 다른 품목이 없는지 확인
3. 변경하려는 품목이 이번 주문 대상인지 다시 확인

검수 포인트:
- 삭제 후에는 항목이 사라졌는지 확인
- 수량 변경 후에는 요청한 수량이 반영됐는지 확인
- 신규 담기 후에는 새 항목 추가 또는 기존 항목 수량 증가를 확인
- 장바구니 우측 또는 하단 요약 영역에서 총 수량, 총 금액, 할인, 배송비가 보이면 함께 읽음

실전 메모:
- 실제 장바구니는 동적 렌더링이라 스냅샷이 잘리거나 일부만 보일 수 있습니다.
- 장바구니를 읽은 직후에 바로 쓰기 액션을 하고, 그 다음 다시 장바구니를 읽는 루프가 가장 안정적입니다.
- 장바구니에는 이번 주문과 무관한 품목이 섞여 있을 수 있으므로 `이번 주문 대상`과 `비대상`을 나눠 보여주는 것이 좋습니다.

## 4. 주문/결제 직전 화면

이 문단은 현재 기본 모드에서 읽기 전용 검수 용도로 사용합니다. 최종 주문 제출은 기본 모드에서 하지 않습니다.

주문/결제 직전 화면에서 읽어야 할 핵심 항목:
- 배송지 또는 받는사람 정보 라벨
- 배송 요청사항
- 도착 예정 일시 또는 새벽/당일 도착 정보
- 주문 상품 목록
- 각 상품의 수량과 금액
- 상품금액 합계
- 할인, 쿠폰, 적립 예정
- 배송비
- 총 결제 예정 금액
- 결제수단 라벨

추가 입력이 필요한 대표 항목:
- 배송지 선택 또는 확인
- 배송 요청사항
- 결제수단 선택
- 카드사/할부 개월 수
- 해외직구 상품의 경우 개인통관고유부호
- 쿠팡페이 또는 카드 결제 승인 단계

운영 규칙:
1. 사용자가 직접 입력해야 하는 값이 보이면 자동 입력하지 말고 먼저 알림
2. 주문 대상 품목, 총액, 배송 시점, 결제수단 라벨을 한 번에 요약
3. 긴 화면은 텍스트 요약만 믿지 말고 스크린샷으로 한 번 더 확인
4. 해외직구나 제3자 판매 상품은 통관, 배송, 취소 조건이 달라질 수 있으니 별도 경고

## 5. 페이지별 실패 신호

검색 결과 페이지 실패 신호:
- 검색 결과 대신 로그인 유도나 차단 페이지가 뜸
- 카드 목록은 보이지만 클릭 대상 ref가 계속 바뀜

상품 상세 페이지 실패 신호:
- 클릭 후 URL이나 제목이 바뀌지 않음
- 새 탭이 열렸는데 포커스가 기존 탭에 남음
- 옵션 미선택 상태로 `장바구니 담기`가 불확실함

장바구니 페이지 실패 신호:
- 일부 품목만 보이고 목록이 잘림
- 수량 변경/삭제 후 재조회에서 동일 항목이 그대로 남음
- 신규 담기 후 신규 항목을 찾지 못함

주문/결제 직전 실패 신호:
- 배송지/결제수단 영역이 접혀 있어 핵심 필드를 읽지 못함
- 총 결제 예정 금액이 보이지 않음
- 추가 인증 화면으로 넘어감

## 6. 확인 우선순위

실전에서 가장 먼저 믿을 신호:
1. 현재 페이지 제목 또는 URL 변화
2. 장바구니 재조회 결과
3. 명시적 토스트나 확인 문구
4. 스크린샷으로 확인한 화면 상태

가장 덜 믿어야 할 신호:
- 클릭 직후의 일시적인 강조 표시
- 동적 ref만 맞았다는 사실
- 장바구니 버튼을 눌렀다는 로그만 있고, 장바구니 재조회가 없는 상태
