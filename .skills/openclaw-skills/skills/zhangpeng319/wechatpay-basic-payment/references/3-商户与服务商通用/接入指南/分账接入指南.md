# 分账

分账是基础支付的扩展能力（非独立支付方式），将支付成功的资金拆分给合作伙伴、员工等。只需在下单时传 `profit_sharing=true`，其他参数不变。支持 JSAPI/APP/H5/Native/小程序/付款码。零成本免手续费。

## 核心规则

| 规则 | 说明 |
|------|------|
| 最大分账比例 | 商户平台配置，默认最高 **30%**，最低 **1%** |
| 分账时间 | 支付成功 **1 分钟后**发起，**30 天**内必须完成，超时自动解冻；收付通模式为 **180 天** |
| 次数与方数 | 一笔订单最多 **50 次**，每次最多 **50 方**，单次不能向同一接收方分多笔 |
| 接收方 | 每商户最多 **2 万**个（商户账户或个人零钱），添加一次即可，需合法征得用户授权 |
| 个人收款限额 | 单月 **20 万** |
| 资金冻结 | 传了 `profit_sharing=true` 的订单资金被冻结，**必须调分账完结/解冻接口**才能释放 |
| 到账 | 准实时，可立即提现，无 T+1 限制 |

## 开通与流程

| | 商户模式 | 服务商模式 |
|--|---------|-----------|
| **发起方** | 商户自己 | 服务商（代子商户发起） |
| **资金来源** | 商户账户 | 子商户账户 |
| **比例配置** | 商户超管在商户平台配置 | 子商户配置允许服务商分账的最大比例 |
| **开通路径** | 商户平台 → 产品中心 → 支付扩展工具 → 开通分账 | ① 服务商平台 → 特约商户授权产品 → 服务商分账 → 邀请子商户；② 子商户商户平台 → 产品中心 → 分账 → 开通并授权 |
| **添加接收方** | 商户平台或 API | 服务商通过 API，或子商户在商户平台添加（需选择对应服务商） |

**业务流程**（参与角色：用户、商户系统/分账方、微信支付系统、分账接收方）：

```
=== 分账主流程 ===
1.  用户在商户侧使用微信支付进行支付
2.  商户系统带分账标识参数(profit_sharing=true)请求下单并支付
3.  微信支付系统回调支付结果给商户系统
4.  商户系统调用「添加分账接收方」接口（⚠️ 必须在请求分账前完成）
5.  微信支付系统返回添加结果
6.  添加成功后，商户系统调用「请求分账」接口
7.  微信支付系统按需求分账给接收方
8.  分账动账通知发送给接收方（⚠️ 非最终分账结果）
9.  若通知未收到或状态不明 → 调用「查询分账结果」确认

=== 分账后退款（可选） ===
10. 用户申请订单退款
11. 商户系统调用「分账回退」接口，请求将已分出的资金回退
12. 微信支付系统从接收方发起分账回退
13. 分账动账通知发送给回退方
14. 若未收到通知 → 调用「查询分账回退结果」确认
15. 回退成功后，资金回到商户可用余额
16. 发起退款流程（与普通退款相同）

=== 完结 ===
17. 分账结束后，调用「完结分账/解冻剩余资金」释放未分账资金
```

> `result=SUCCESS` 才是分账成功，`CLOSED` 为失败。分账单状态 `FINISHED` 只表示动作执行完毕，不代表成功。

## API 接口

| 接口 | 方法 | 路径 | 备注 |
|------|------|------|------|
| 添加/删除分账接收方 | POST | `.../receivers/add` 或 `.../delete` | |
| 请求分账 | POST | `/v3/profitsharing/orders` | 异步处理 |
| 查询分账结果 | GET | `.../orders/{out_order_no}` | |
| 解冻剩余资金 | POST | `.../orders/unfreeze` | 异步处理，不分账的订单也可直接调用解冻 |
| 分账回退 / 查询回退 | POST/GET | `.../return-orders` | 同步处理，仅限商户类型接收方，需接收方开启回退，最多50次，时限180天 |
| 查询剩余待分金额 | GET | `.../transactions/{transaction_id}/amounts` | |
| 申请分账账单 | GET | `/v3/profitsharing/bills` | 仅3个月内，金额单位"元"，字段前有 `` ` `` 字符 |
| 查询最大分账比例 | GET | `.../merchant-configs/{sub_mchid}` | 服务商独有 |

> 频率：请求分账 300QPS、查询 500QPS、回退 100QPS。合单支付分账按**子单维度**发起。

## 分账动账通知

仅针对分账**接收方**（`MERCHANT_ID` 类型），分账方不会收到通知。

- **回调地址**：接收方配置 `service_notify_url`（须 HTTPS）。商户模式在商户平台 → 分账接收设置；服务商模式在服务商平台配置
- **通知区分**：`summary` 为"分账"或"分账回退"
- **解密**：APIv3 密钥 + AES-256-GCM，解密后含 `transaction_id`、`order_id`、`out_order_no`、`receiver`、`success_time`（服务商模式额外含 `sp_mchid` + `sub_mchid`）
- **重试**：与支付回调一致（15次，24小时），需幂等处理

## 分账与退款

| 订单状态 | 退款处理 |
|---------|---------|
| 未分账（冻结中） | 直接从冻结资金退款 |
| 部分分账 | 退款 ≤ 剩余冻结金额直接退；超出需先解冻，再从可用余额退 |
| 已完结 | 从商户可用余额退款 |

> 退款前可先调回退接口将已分出资金退回。回退资金回到商户可用余额，与退款无强耦合。个人接收方**不支持**回退。
