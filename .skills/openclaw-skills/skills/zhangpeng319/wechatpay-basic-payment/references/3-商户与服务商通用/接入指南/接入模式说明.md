# 基础支付接入模式说明

基础支付（JSAPI/APP/H5/Native/小程序/付款码/合单支付）支持两种接入模式。使用本技能的任何能力前，需先确认用户属于哪种模式。

> 微信支付还有其他接入模式，但不在本技能覆盖范围内。

## 两种模式一览


|          | 商户模式（普通商户）                                                                                    | 服务商模式                                                                                                |
| -------- | --------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **适用对象** | 有开发能力的商户，自行接入微信支付                                                                             | 系统开发商/解决方案提供商，代中小商户接入                                                                                |
| **入驻平台** | 商户平台 [https://pay.weixin.qq.com/index.php/public](https://pay.weixin.qq.com/index.php/public) | 服务商平台 [https://pay.weixin.qq.com/partner/public/home](https://pay.weixin.qq.com/partner/public/home) |
| **资金流向** | 微信支付 → 商户结算账户                                                                                 | 微信支付 → 子商户结算账户（服务商账户不直接收款）                                                                           |
| **数据流向** | 商户系统 ↔ 微信支付                                                                                   | 服务商系统 ↔ 微信支付（代子商户发起）                                                                                 |
| **收款主体** | 商户自己                                                                                          | 子商户（特约商户）                                                                                            |


## 核心参数差异


| 参数       | 商户模式                                        | 服务商模式                                      |
| -------- | ------------------------------------------- | ------------------------------------------ |
| 商户号      | `mchid`                                     | `sp_mchid`（服务商）+ `sub_mchid`（子商户）          |
| 应用ID     | `appid`                                     | `sp_appid`（服务商）+ `sub_appid`（子商户，选填）       |
| 用户标识     | `openid`                                    | `sp_openid`（服务商应用下）或 `sub_openid`（子商户应用下）  |
| API证书    | **商户平台**申请；私钥用于 V3 请求签名 + 调起支付，不可外泄         | **服务商平台**申请；仅服务商持有，**子商户不需要**；代调用时以服务商身份签名 |
| APIv3密钥  | **商户平台**设置；只用于**回调**里 `resource` 解密，不参与请求签名 | **服务商平台**设置；**子商户不设**；解密发到服务商的回调           |
| 微信支付平台证书 | 商户身份调 `GET /v3/certificates`；验签应答/回调，注意轮换   | 服务商身份调同一接口；仍是**微信**平台证书，非子商户证书             |
| 微信支付公钥   | **商户平台**获取；与平台证书**二选一**验签                   | **服务商平台**获取；二选一验签；进件/开户意愿等敏感字段加密见专项文档      |


> **说明**：验微信侧数据 → 平台证书或公钥二选一；发请求签名 → 各自 API 证书私钥；回调解密 → APIv3 密钥。

## 基础支付 API 路径差异


| 接口             | 商户模式                                               | 服务商模式                                                                       |
| -------------- | -------------------------------------------------- | --------------------------------------------------------------------------- |
| JSAPI下单        | `/v3/pay/transactions/jsapi`                       | `/v3/pay/partner/transactions/jsapi`                                        |
| APP下单          | `/v3/pay/transactions/app`                         | `/v3/pay/partner/transactions/app`                                          |
| H5下单           | `/v3/pay/transactions/h5`                          | `/v3/pay/partner/transactions/h5`                                           |
| Native下单       | `/v3/pay/transactions/native`                      | `/v3/pay/partner/transactions/native`                                       |
| 商户订单号查单        | `/v3/pay/transactions/out-trade-no/{id}?mchid=xxx` | `/v3/pay/partner/transactions/out-trade-no/{id}?sp_mchid=xxx&sub_mchid=xxx` |
| 微信订单号查单        | `/v3/pay/transactions/id/{id}?mchid=xxx`           | `/v3/pay/partner/transactions/id/{id}?sp_mchid=xxx&sub_mchid=xxx`           |
| 关闭订单           | `/v3/pay/transactions/out-trade-no/{id}/close`     | `/v3/pay/partner/transactions/out-trade-no/{id}/close`                      |
| 退款（申请退款）       | `POST /v3/refund/domestic/refunds`                 | `POST /v3/refund/domestic/refunds`（请求体增加 `sub_mchid`）                       |
| 账单（申请交易账单）     | `GET /v3/bill/tradebill`                           | `GET /v3/bill/tradebill`（Query 增加 `sub_mchid`）                              |
| 分账（请求分账）       | `POST /v3/profitsharing/orders`                    | `POST /v3/profitsharing/orders`（请求体增加 `sub_mchid`）                          |
| 分账（查子商户分账比例上限） | 无对应接口（比例在商户平台配置）                                   | `GET /v3/profitsharing/merchant-configs/{sub_mchid}`                        |


> **规律**：下单、查单、关单在路径中多 `/partner/`；退款、账单、**请求分账**等路径在两种模式下相同，服务商多传 `**sub_mchid`**。上一行与下一行是**不同能力**：`orders` 是发起分账；`merchant-configs` 是服务商在调分账前查询某子商户允许的分账比例上限，普通商户侧不走该接口。其余分账相关路径见 `示例代码/接口索引.md`。

## 如何判断用户属于哪种模式


| 线索                                                  | 判断      |
| --------------------------------------------------- | ------- |
| 用户提到"子商户"、"特约商户"、"代商户接入"                            | → 服务商模式 |
| 参数中出现 `sp_mchid`、`sub_mchid`、`sp_appid`、`sub_appid` | → 服务商模式 |
| API 路径包含 `/partner/`                                | → 服务商模式 |
| 用户在"服务商平台"操作                                        | → 服务商模式 |
| 用户直接用自己的 `mchid` + `appid` 接入                       | → 商户模式  |
| 用户在"商户平台"操作且不涉及子商户                                  | → 商户模式  |


## 两种模式通用的部分

- 签名与验签规则（SHA256-RSA、调起支付签名格式）
- 回调通知处理（验签、解密、幂等应答）
- 退款/账单的业务逻辑和状态流转
- 各支付方式的前端调起方式（WeixinJSBridge、OpenSDK、wx.requestPayment 等）
- 产品选型逻辑（JSAPI/APP/H5/Native/小程序的适用场景）
