# 普通支付订单状态流转

## 状态定义

| trade_state | 含义 | 是否终态 | 说明 |
|------------|------|---------|------|
| NOTPAY | 未支付 | 否 | 待用户支付，可流转到 SUCCESS 或 CLOSED |
| SUCCESS | 支付成功 | ✅ 是 | 支付流程终态；可发起退款流转到 REFUND |
| CLOSED | 已关闭 | ✅ 是 | 不可再支付、不可退款 |
| REFUND | 转入退款 | ✅ 是 | 已发起退款（退款结果需查退款单确认） |

## 流转路径

```
下单成功
  │
  ▼
NOTPAY（未支付）
  │
  ├──── 用户支付成功 ────→ SUCCESS（支付成功）
  │                          │
  │                          └── 商户申请退款 ──→ REFUND（转入退款）
  │
  ├──── 用户支付失败 ────→ NOTPAY（状态不变，可重试）
  │     常见失败原因：余额不足、用户拒绝、支付密码错误、超过time_expire
  │
  └──── 商户关单 / 超时自动关单 ──→ CLOSED（已关闭）
```

## 关键规则

### 关单
- 商户可在 **7天内** 对未支付订单调用关单接口
- 超过7天未支付的订单，微信侧自动关单
- 关单后订单不可再支付
- 商户也可通过 `time_expire` 参数自定义支付截止时间

### 退款
- 仅 `SUCCESS` 状态的订单可申请退款
- 退款时限：支付成功后 **1年内**
- 支持部分退款（多次退款，总额不超过订单金额）
- 退款成功后订单状态变为 `REFUND`
- 退款结果需通过查询退款单接口确认，也可接收退款回调通知

### 支付超时与重试
- `prepay_id` 有效期为 **2小时**（JSAPI/APP/小程序/Native）
- `h5_url` 有效期为 **5分钟**（H5支付）
- 过期后需用原下单参数重新调下单接口获取新凭证
- 默认订单有效期 7天，可通过 `time_expire` 缩短

## 开发注意事项

### 订单状态以查单结果为准
- 前端回调（WeixinJSBridge / wx.requestPayment）返回的结果**不能**作为支付是否成功的最终判断
- 必须在收到前端回调后，调用查询订单 API 确认 `trade_state`
- 后端回调通知（notify_url）也需验签后处理

### 回调与查单双保险
- 微信支付会异步发送支付成功通知到 `notify_url`
- 但回调不保证 100% 送达（网络抖动、商户服务不可用等）
- 建议实现：收到回调处理 + 定时/延迟查单兜底
- 具体方案参考官方 [支付回调和查单实现指引](https://pay.weixin.qq.com/doc/v3/merchant/4012075249)

### 幂等处理
- 同一订单可能收到多次回调通知（微信有重试机制）
- 业务逻辑必须做幂等：已处理的订单重复通知时直接返回成功，不重复发货
