# 回调通知处理

## notify_url 配置要求

| 要求 | 说明 |
|------|------|
| 协议 | 必须以 `https://` 或 `http://` 开头 |
| 地址 | 必须是外网可访问的完整路径，不能用 localhost、127.0.0.1、192.168.x.x 等内网地址 |
| 参数 | URL 不能携带参数 |
| 备案 | 国内服务器域名需完成 ICP 备案 |
| APIv3密钥 | 必须在商户平台设置 APIv3 密钥，否则微信支付不会发送回调通知 |

常见错误示例：

| 错误 | 示例 |
|------|------|
| 只有域名没有路径 | `http://www.weixin.qq.com` |
| 缺少协议头 | `./PayNotify.aspx` |
| 使用内网地址 | `http://127.0.0.1/pay/notify.php` |

## 回调处理逻辑规范

1. **不做登录态校验** — notify_url 的处理逻辑不能要求用户登录
2. **5秒内返回应答** — 超时则微信支付认为通知失败，会重复发送
3. **幂等处理** — 微信支付可能对同一笔订单多次发送通知（如商户应答超时触发重试），商户收到通知后应先根据订单号检查是否已处理过：若已处理，直接返回成功应答，不要重复执行业务逻辑（如重复加款、重复发货等）
4. **先应答后处理** — 先返回 HTTP 200/204，再异步执行业务逻辑，避免业务处理耗时导致超时重试
5. **正确处理签名探测** — 微信支付会偶尔发送错误签名的探测请求（签名值包含 `WECHATPAY/SIGNTEST/` 前缀），验签失败时返回 4XX/5XX 即可，等待微信支付携带正确签名重新发送

## 重试机制

商户应答失败或超时（5秒）时，微信支付按以下频次重试：

15s / 15s / 30s / 3m / 10m / 20m / 30m / 30m / 30m / 60m / 3h / 3h / 3h / 6h / 6h（最多15次）

## 收不到回调的常见原因

### 配置类

- notify_url 拼写错误（域名错、路径漏）
- 使用了内网地址
- 域名未备案或 DNS 解析失效
- 未设置 APIv3 密钥
- 接口或商户平台未配置 notify_url

### 网络类

- 防火墙/安全组未对微信支付回调 IP 开通白名单
- WAF 或 CC 防护误拦截了回调请求
- 网络延迟过高（超过3秒）导致请求超时
- CDN 或反向代理（Nginx/Cloudflare）未正确转发到后端服务

### 逻辑类

- 回调处理代码做了登录态校验，导致返回 401/403
- 业务处理耗时超过5秒，未先返回 200 再异步处理
