# 分账常见问题

> 分账业务知识见 `分账接入指南.md`，本文仅收录分账相关的报错和常见问答。

## 分账失败处理（`fail_reason` 字段）

| fail_reason | 含义 | 处理方案 |
|-------------|------|---------|
| ACCOUNT_ABNORMAL | 接收账户异常 | 个人：引导用户检查微信是否被管控；商户：引导接收方申诉解除管控后重试 |
| NO_RELATION | 分账关系已解除 | 重新调用添加分账接收方接口 |
| RECEIVER_HIGH_RISK | 高风险接收方 | 无法分账，无需重试，需通过其他方式处理 |
| RECEIVER_REAL_NAME_NOT_VERIFIED | 接收方未实名 | 引导用户完成实名认证后重试 |
| NO_AUTH | 分账权限已解除 | 分出方商户重新申请分账权限 |
| RECEIVER_RECEIPT_LIMIT | 超出用户月收款限额 | 下月再分账，或引导用户开通商户号 |
| PAYER_ACCOUNT_ABNORMAL | 分出方账户异常 | 引导分出方商户申诉解除管控后重试 |
| INVALID_REQUEST | 描述参数设置失败 | 分账描述包含敏感词，修改后重试 |

## 常见报错

| 报错信息 | 原因与解决 |
|---------|-----------|
| "非分账订单不支持分账" | 下单时未传 `profit_sharing=true` 或订单号填错 |
| "分账金额不足" | 已退款/手续费扣除/30天自动解冻/超剩余可分金额 → 调查询待分金额 API |
| "分账金额超出最大分账比例" | 所有分账金额之和超过订单金额 × 最大比例 → 商户平台调整 |
| "分账接收方关系不存在" | 未添加接收方 → 先调添加接口 |
| "无分账权限" | 未开通或被冻结 → 商户平台开通/查站内信申诉 |
| "订单处理中，请稍后重试" | 支付不足1分钟/未结算/手续费账户余额低于100元 |
| "分账接收方与原请求不一致" | 重复使用已分过账的单号 → 多次分账用新单号 |
| "appid与openid不匹配" | openid 不是从请求分账传的 appid 下获取的 |
| "微信用户姓名与实名不一致" | `name` 字段与用户实名不匹配 |
| "账户不存在，请先点击充值" | 接收方商户未开通账户 → 商户平台 → 交易中心 → 充值 |
| "服务商和分账方无受理关系" | 子商户号不是在该服务商下进件的 |

## 常见问答

| 问题 | 答案 |
|------|------|
| 一次分账会部分成功部分失败吗？ | 会，需逐一查询每个接收方结果 |
| 全额退款后还能分账吗？ | 不能 |
| 关闭分账权限后冻结资金会解冻吗？ | 会（30天后自动解冻） |
| 删除接收方后还能回退吗？ | 能 |
| 收不到分账动账通知？ | ① 未配置URL；② 未设APIv3密钥；③ 只有接收方才收到 |
| 如何分账后立即解冻？ | 最后一次分账传 `unfreeze_unsplit=true`，或1分钟后调解冻接口 |
| 接收方收到资金会自动提现吗？ | 不会，需手动提现 |
| 分账失败资金怎么处理？ | 平台直接返还给分账方 |
| 冻结资金能用于其他订单退款吗？ | 不能，冻结资金不属于可用余额 |
| 服务商模式可由子商户自己发起分账吗？ | 不能，服务商模式订单必须由服务商发起 |
| 每个子商户都要分别添加接收方吗？ | 是 |
| 分账涉及税务吗？ | 仅资金分配，不涉及税务 |
| 分账请求失败计入50次上限吗？ | 未受理不计入；已受理计入 |
| 免充值券 vs 预充值券可分金额？ | 10-5 免充值券可分5元，预充值券可分10元 |

## 线上高频问题（来自真实工单）

### 调错接口类型

| 问题 | 答案 |
|------|------|
| 报"非电商商户暂不支持调用此API" | 普通服务商不能调电商收付通接口（`/v3/ecommerce/profitsharing/`），需用普通服务商分账接口（`/v3/profitsharing/`） |
| 报"电商商户不允许调用非电商分账API" | 电商收付通商户不能调普通分账接口，需用电商专用接口（`/v3/ecommerce/profitsharing/`） |
| 直连模式的订单能走收付通分账吗？ | 不能，什么模式下单就用什么模式的分账接口 |

### 解冻与资金状态

| 问题 | 答案 |
|------|------|
| 解冻后还能继续分账吗？ | **不能**，已解冻的订单不支持继续分账 |
| 解冻接口调成功了但没解冻？ | `out_order_no` 必须用**新的唯一单号**，不能复用分账请求的单号，否则接口返回成功但实际不解冻 |
| 分账失败后资金在哪？ | 仍冻结在分账方账户，需主动调解冻接口或等待自动解冻（30天） |
| "订单已完结分账，不支持分账"是什么意思？ | 订单已调过完结/解冻接口，不能再分账。如需重新分账需用新订单 |
| 逾期解冻是什么？ | 冻结期限内（30天）未分账，系统自动将资金解冻至商户账户 |
| 分账回退后还能用原可分金额继续分账吗？ | **不能**，回退资金进入可用余额，原订单可分金额不恢复 |
| 解冻后还能回退吗？ | **能**，解冻不影响分账回退 |

### 结算周期与管控

| 问题 | 答案 |
|------|------|
| "订单处理中，请在到达结算周期之后再请求分账" | 商户号被处罚延长了结算周期，需查站内信申诉。申诉成功后处罚期间的订单仍按处罚周期结算，不会加速 |
| "分账权限被处罚，请尽快处理违规记录" | 分账权限被冻结，需登录商户平台/合作伙伴平台查站内信申诉处理 |
| 交易类小程序分账提示要用户确认收货？ | 受公众平台运营规范管控，资金在用户确认收货或系统自动确认后才能结算，结算后方可分账 |

### 分账金额计算

| 问题 | 答案 |
|------|------|
| 可分金额怎么算？ | 订单金额 - 手续费 = 实际可分金额。如订单6.90元扣手续费后可能只剩6.86元 |
| 分账比例是按单次还是累计？ | **累计**。单笔订单所有分账金额的总和不得超过订单金额×最大比例，而非单次单独计算 |
| 部分退款后还能分账吗？ | 能，但分账金额需在剩余可分范围内；**全额退款**后不能分账 |

### 其他高频问题

| 问题 | 答案 |
|------|------|
| 分账中的 openid 能用 unionid 代替吗？ | **不能**，分账接口只接受 openid |
| 分账关系绑定支持幂等吗？ | 支持，重复绑定返回成功，不会覆盖原数据 |
| 分账动账通知中回退场景的 receiver 是谁？ | 指向**原分账方**的信息（即资金退回的目标方），而非接收方 |
| 分账接收方只有平台自己，也要调添加接口吗？ | 是，不管接收方是谁都需要先调添加分账接收方接口 |
| V2 分账完结接口对应 V3 哪个？ | V3 的「解冻剩余资金」接口 |
| 能否完全依赖分账动账通知确定结果？ | **不能**，需结合主动查询分账结果接口 |
| 分账能分给任何商户吗？ | 可分给微信支付体系内的商户或个人零钱，**不支持**分给其他第三方支付平台的账户 |
| 分账支持批量或定时打款吗？ | **不支持**，只能按订单维度逐笔分账 |
| "分账接收方与原请求不一致" | 重复使用了已分过账的商户分账单号并修改了接收方信息。多次分账/完结分账须使用**不同的**商户分账单号 |
| 分账没有电子回单吗？ | 没有，可通过分账账单和资金账单查询明细 |
| 子商户的 `sub_mchid` 是什么？ | 微信支付分配的特约商户号，**不是**特约商户的微信号 |
| 分账回退提示"收款功能受限" | 接收方或分出方被风控管控，需自行在商户平台查看违规记录并申诉 |
