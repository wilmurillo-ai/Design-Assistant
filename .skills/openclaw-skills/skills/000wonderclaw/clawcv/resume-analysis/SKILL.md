# 简历分析

对整份简历做结构化诊断，输出问题优先级和下一步改进方向。

## 适用场景

- 用户贴出完整简历，希望先看整体问题
- 用户想知道这份简历为什么效果一般
- 用户不确定应该先改哪一部分

## 不适用

- 只想改写某一段内容：用 `/resume-rewrite`
- 已经有明确 JD，要看岗位匹配：用 `/job-match`
- 目标是导出 PDF：用 `/pdf-export`

## 工作流程

### 第 1 步：确认分析上下文

优先确认以下信息：

- 目标岗位
- 当前身份或年限
- 简历是否为完整版本

如果用户没给目标岗位，可以先做通用分析。

### 第 2 步：按 5 个维度诊断

固定检查以下维度：

1. 完整性：核心板块是否齐全
2. 清晰度：表达是否直白、可读
3. 匹配度：是否贴近目标岗位
4. 成果感：是否有数据、结果、影响
5. 结构性：顺序、标题、层次是否合理

### 第 3 步：输出问题优先级

把问题分成高、中、低三个优先级，优先指出真正影响投递效果的问题，例如：

- 缺少个人总结
- 工作经历只有职责，没有结果
- 项目描述太像流水账
- 技能堆砌但和目标岗位不相关

### 第 4 步：给出分模块反馈

至少覆盖这些常见板块：

- 个人总结
- 工作经历
- 项目经历
- 教育经历
- 技能

### 第 5 步：给出下一步建议

分析结束后，明确下一步应该做什么：

- 需要逐段改写时，引导到 `/resume-rewrite`
- 有目标 JD 时，引导到 `/job-match`
- 准备投递最终版时，引导到 `/pdf-export`

## 输出格式

- 先给整体判断
- 再给 5 维评分或等级判断
- 然后列出高优先级问题
- 最后给出分板块建议和下一步动作

## 分析红线

- 不因为内容少就假设用户有不存在的经历
- 不把“写得少”误判成“经历差”
- 没有目标岗位时，不做过度行业化判断
- 如果简历明显不完整，先提醒补充，不输出虚高精度结论

## 示例说法

```text
帮我分析这份简历
看看我的简历主要问题在哪
我投产品经理，这份简历过不过关
```
