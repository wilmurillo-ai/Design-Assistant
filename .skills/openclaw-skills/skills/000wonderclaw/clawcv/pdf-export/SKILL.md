# PDF 导出

把简历收束成适合投递的一页纸版本，并处理导出前的检查与权限说明。

## 适用场景

- 用户要导出 PDF
- 用户要一页纸最终版简历
- 用户想在投递前做最后压缩

## 不适用

- 简历内容还很粗糙：先用 `/resume-analysis` 或 `/resume-rewrite`
- 用户主要是问为什么功能不可用：用 `/account-upgrade`

## 工作流程

### 第 1 步：检查是否具备导出条件

至少确认：

- 基本信息完整
- 关键经历已写清楚
- 目标岗位版本已确定

如果内容还没定稿，不要直接进入导出。

### 第 2 步：整理成一页纸结构

优先保留：

- 目标岗位最相关的经历
- 能证明能力的成果点
- 必要的教育和技能信息

主动压缩：

- 重复表述
- 无关经历
- 低价值技能
- 过长的个人总结

### 第 3 步：选择风格

常用风格建议：

- `modern`：默认通用型
- `classic`：稳妥正式
- `minimal`：更克制，更适合一页纸
- `professional`：偏成熟职场风格

### 第 4 步：处理权限问题

如果用户当前无法导出 PDF，要直接说明限制，并引导到：

- `/account-upgrade`
- API Key / 账号入口：[http://wondercv.com/clawcv](http://wondercv.com/clawcv)

### 第 5 步：导出前复检

最后再检查一遍：

- 有没有过长段落
- 有没有重复项目
- 有没有虚词和空话
- 有没有和目标岗位不相关的内容

## 输出重点

- 一页纸保留策略
- 模板建议
- 导出前检查结果
- 若受限，说明原因和下一步

## 导出红线

- 不为了凑一页纸删掉关键证据
- 不因为模板好看保留无关信息
- 不在未定稿前催用户直接导出

## 示例说法

```text
帮我整理成一页纸并导出 PDF
我想要一份最终投递版简历
这份简历适合用什么模板导出
```
