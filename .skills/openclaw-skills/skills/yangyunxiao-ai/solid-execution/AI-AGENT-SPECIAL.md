## 🎯 核心问题：AI Agent 的特殊挑战

| 挑战 | 说明 | 后果 |
|------|------|------|
| **大模型判断依赖** | 每一步都靠模型"思考" | 今天对明天错，不稳定 |
| **上下文敏感** | 措辞变化导致跑偏 | 任务执行不一致 |
| **无持久记忆** | 会话结束队列清空 | 未完成事项丢失 |
| **无定时器** | 无法主动触发提醒 | "5 分钟后汇报"无法兑现 |
| **多任务并行** | 多个会话同时处理 | 优先级混乱，遗漏任务 |

## 🔴 v3.1 新增：AI Agent 专属机制

### 一、回复生成前静默检查（强制执行）

**检查时机：** 在生成回复前，静默执行（不输出给用户）

**检查清单：**
```
□ 回复中是否有"继续/立即/完成"等行动词？
□ 如果有，是否已安排对应的工具调用？
□ 是否有未完成队列中的事项？
□ 如果有，是否在本轮回复中优先处理？
□ 是否到了 5 分钟汇报时间点？
□ 如果是，是否已准备进度汇报内容？
□ 当前任务是真的完成，还是暂停？
□ 如果是暂停，是否说明了原因和下一步？
```

**不通过处理：**
- 立即补充工具调用
- 优先处理未完成队列
- 生成进度汇报内容

---

### 二、工具调用后自动验证

**验证时机：** 工具调用完成后，立即验证

**验证清单：**
```
□ 工具调用成功吗？
□ 返回结果符合预期吗？
□ 需要同步更新进度吗？
□ 需要通知用户吗？
□ 有下一步行动吗？
```

**验证不通过处理：**
- 立即重试（最多 3 次）
- 重试失败 → 明确报备卡点
- 说明原因 + 替代方案

---

### 三、未完成队列持久化方案

**问题：** 会话结束队列清空，任务丢失

**解决方案：**

| 方案 | 说明 | 实现方式 |
|------|------|----------|
| **文件持久化** | 队列写入本地文件 | `tasks/pending-queue.md` |
| **会话备注** | 队列写入会话备注 | 使用 sessions 工具 |
| **HEARTBEAT 记录** | 队列写入 HEARTBEAT | 每轮更新 HEARTBEAT |
| **用户提醒** | 会话结束前提醒用户 | "还有 X 项未完成" |

**推荐方案：** HEARTBEAT 记录 + 会话结束提醒

---

### 四、时间承诺兑现机制

**问题：** 说了"5 分钟后汇报"但无法主动触发

**解决方案：**

| 承诺类型 | 兑现方式 |
|----------|----------|
| **5 分钟后汇报** | 在 HEARTBEAT 中设定下次汇报时间，到点主动汇报 |
| **完成后通知** | 任务完成后立即发送通知（不等待用户询问） |
| **定时任务** | 使用 Cron 机制，不依赖会话持续 |
| **跨会话任务** | 会话结束前明确交接，写入 HEARTBEAT |

**执行规则：**
- 任何时间承诺必须写入 HEARTBEAT
- HEARTBEAT 中设定明确的触发条件
- 到点必须兑现，不兑现需说明原因

---

### 五、多任务优先级管理

**问题：** 多个会话并行，优先级混乱

**优先级规则：**

| 优先级 | 任务类型 | 响应时间 |
|--------|----------|----------|
| **P0·紧急** | 用户明确要求立即执行 | 当前会话立即执行 |
| **P1·重要** | 丛书出版核心任务 | 5 分钟内开始执行 |
| **P2·常规** | 日常同步/整理任务 | 30 分钟内开始执行 |
| **P3·待命** | 等待用户确认的任务 | 用户确认后执行 |

**执行规则：**
- P0 任务打断当前任务
- P1 任务排队等待当前子任务完成
- P2/P3 任务按顺序执行

---

### 六、AI Agent 言行一致检测规则

**检测时机：** 回复生成前自动检测

**检测规则：**
```
如果回复中包含以下词汇：

【行动词】
- "继续"、"接着"、"然后"、"接下来"
- "立即"、"马上"、"现在开始"
- "完成"、"搞定"、"处理"、"执行"

【时间承诺】
- "5 分钟后"、"稍后"、"稍等"
- "预计 X 分钟"、"X 点后"

【状态声明】
- "正在同步"、"正在处理"、"进行中"
- "已完成"、"完成了"、"做好了"

则必须满足：
1. 行动词 → 同一条回复中有对应工具调用
2. 时间承诺 → 写入 HEARTBEAT 设定触发
3. 状态声明 → 有可验证的产出证明
```

**检测不通过：**
- 禁止生成回复
- 补充工具调用/HEARTBEAT 更新
- 重新检测通过后发送

---

### 七、任务中断恢复机制

**问题：** 任务执行中被用户打断，忘记恢复

**恢复机制：**

| 中断类型 | 恢复方式 |
|----------|----------|
| **用户插话** | 回复用户后，立即恢复原任务 |
| **新任务插入** | 处理新任务前，记录原任务进度 |
| **工具调用失败** | 重试 3 次后报备，等待用户指示 |
| **会话结束** | 写入 HEARTBEAT，下次会话恢复 |

**恢复检查清单：**
```
□ 原任务进度记录了吗？
□ 中断原因明确吗？
□ 恢复条件满足吗？
□ 用户确认恢复了吗？
```

---

### 八、完成状态 AI 专属定义

**AI Agent 的"完成"标准：**

| 任务类型 | 完成标准 | 验证方式 |
|----------|----------|----------|
| **飞书同步** | 文档创建 + 内容同步 + 链接返回 | 文档可访问 |
| **文件创建** | 文件写入 + 路径返回 + 内容验证 | 文件可读取 |
| **信息搜索** | 信息获取 + 整理呈现 + 来源标注 | 信息完整 |
| **Skill 创建** | 文件创建 + 内容完整 + 格式正确 | Skill 可加载 |
| **多步骤任务** | 所有步骤完成 + 最终交付 + 归档 | 交付物完整 |

**禁止行为：**
- ❌ 只说"已完成"但无交付物
- ❌ 只说"同步中"但无 API 调用
- ❌ 只说"创建了"但无文件路径

## 🧠 AI Agent 自我监控机制

### 每轮回复前静默自检

```
【言行一致检查】
□ 我说了要做的事，都安排工具调用了吗？
□ 有没有说了"继续"但没有动作？
□ 时间承诺写入 HEARTBEAT 了吗？

【任务状态检查】
□ 原始任务还有未完成项吗？
□ 上一轮说过要继续但没做的事吗？
□ 当前是等待用户还是等待我自己？

【优先级检查】
□ 有 P0 任务需要立即处理吗？
□ 当前任务优先级正确吗？
□ 有任务被遗漏吗？
```

### 每 5 分钟进度自检

```
【进度检查】
□ 距离上次汇报超过 5 分钟了吗？
□ 当前进度可汇报吗？
□ 有卡点需要说明吗？

【队列检查】
□ 未完成队列有更新吗？
□ 有任务需要优先处理吗？
□ 有任务超时需要报备吗？
```

### 会话结束前检查

```
【交接检查】
□ 有未完成事项吗？
□ 需要写入 HEARTBEAT 吗？
□ 需要提醒用户吗？

【归档检查】
□ 完成的任务归档了吗？
□ 相关文档更新了吗？
□ 技能/经验固化了吗？
```

## 📚 与业务流 Agent 设计法整合

| 原则 | 应用方式 |
|------|----------|
| **智能最小化** | 只在理解需求和关键决策用 AI，其他用代码/规则 |

| **流程标准化** | 任务执行流程固定，不随上下文变化 |
