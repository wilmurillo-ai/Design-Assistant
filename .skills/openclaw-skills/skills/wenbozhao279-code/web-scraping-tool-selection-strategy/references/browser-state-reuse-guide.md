# 浏览器状态复用指南

## Chrome标签页复用机制

### 复用优势
- **绕过登录验证**: 直接访问已登录状态的页面
- **绕过反爬机制**: 使用真实浏览器环境
- **提高效率**: 避免重复登录流程
- **获取完整数据**: 访问登录后可见的私有数据

### 标签页状态检查
```bash
# 检查当前Chrome标签页状态
playwright-cli goto "about:blank"
# 查看已打开的标签页数量和域名

# 验证登录状态
playwright-cli goto "https://www.jd.com"
# 检查是否显示用户名等登录标识
```

### 最佳实践
1. **启动顺序**: 先手动登录各平台账号，再启动抓取工具
2. **标签页管理**: 保持常用平台标签页处于活跃状态
3. **会话保持**: 避免关闭已登录的Chrome实例
4. **状态验证**: 抓取前先验证关键页面的登录状态

### 支持的平台
- 京东、淘宝、天猫、1688、抖音、拼多多
- 小红书、知乎、微博、B站
- 其他需要登录的网站

### 注意事项
- 确保Chrome浏览器已正确安装并可被工具访问
- 定期检查标签页是否因长时间未操作而登出
- 避免同时运行多个浏览器实例导致状态混乱
