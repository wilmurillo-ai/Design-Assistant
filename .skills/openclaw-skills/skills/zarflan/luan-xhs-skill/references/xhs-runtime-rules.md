# XHS 运行规则（引用自技能主文）

## 0.1 低 token 与快照约束

- 优先 `evaluate`，减少无意义 dump 与重复抓取。
- 只在关键节点做快照：登录确认、到发布页、填写完成、发布后确认成功。
- 避免 `fullPage`（除用户要求整页归档）；重复调用优先复用同一 `targetId`。
- 每个动作最多重试 1 次；第二次失败改稳健路径并汇报。
- 记录关键证据：账号名、页面状态、按钮可见、字数、发布后标题是否出现在笔记管理中。

## 0.2 运行入口与代理规则（最高优先）

- **发布、登录、会话校验一律优先走 `creator.xiaohongshu.com`。** 不要先用 `www.xiaohongshu.com` 判断“是否已登录可发布”。
- 技能配置 `config.useProxy=false` 时，先调用 `scripts/disable_proxy.js` 清空 `HTTP_PROXY/HTTPS_PROXY/http_proxy/https_proxy` 等代理变量。
- 社区首页可浏览 ≠ 创作服务平台可发布；真正的发布鉴权必须在创作服务平台内确认。
- 进入发布页后默认先检查当前 tab 是否落在“上传视频”；若是，主动切到“上传图文”。

## 0.3 创作平台会话校验

在任何“直接发布”动作前，按以下顺序校验：

1. 打开 `https://creator.xiaohongshu.com/new/home`
2. 若 URL 落到 `/login`，或页面出现“短信登录 / 发送验证码 / 收不到验证码 / 解锁创作者专属功能”，判定为 **会话失效**
3. 若页面出现“发布笔记 / 笔记管理 / 数据看板”，判定为 **会话有效**
4. 会话失效时，优先改走 `scripts/xhs_login_sms.js`，不要继续撞社区首页或反复扫码重试

## 0.4 已验证脚本优先级

在可执行脚本存在时，优先走脚本而不是临时拼 selector：

1. `scripts/xhs_login_sms.js`：短信验证码登录并保存 `xhs_user_info.json`
2. `scripts/xhs_login_qr.js`：创作平台专用二维码登录并保存 `xhs_user_info.json`
3. `scripts/xhs_publish_with_saved_session.js`：恢复保存会话并直接发布图文
4. `scripts/xhs_publish_video.js`：恢复保存会话并直接发布视频
5. `scripts/xhs_make_text_cover.py`：没有素材图时生成兜底封面

只有在脚本失效或页面结构明显变化时，才回退到临时页面探测。

## 3.5 搜索并浏览（核心约束）

1. 仅从搜索结果页点击进入帖子，禁止直接 `navigate` 到 `/explore/<id>`。
2. 默认跳过本账号作者内容（避免自刷）。
3. 进入后先校验：不是 404、可见评论/互动信息、可识别标题或作者。
4. 进入方式优先点卡片本体，避免点头像/作者名导致跳错。
5. 若评论控件为 `contenteditable` 或 `p.content-input`，需先触发输入事件再发送。
6. 两条点击失败或 404 后返回搜索页换下一条，不对同链接直跳重试。

## 6.0 回放与降级

- 若搜索结构变化先 snapshot 更新 selector 再继续，不盲跑旧路径。
- 关键页（创作页、探索页、用户页）尽量复用已打开 tab，不重复 `open`。
- 先告诉用户“已达异常节点”，避免无意义继续操作导致误发。
- 发布失败时，优先按这个顺序降级排查：**代理 → 创作平台登录态 → 是否被平台强制切到扫码登录 → 发布页 tab → 是否真正进入视频/图文编辑页 → 发布后笔记管理确认**。
- 视频发布时，若底部 `发布` 按钮可见但 disabled，先检查标题/正文是否填写完成，再判断是否失败。
- 视频草稿箱是本地浏览器草稿，不要假设能在另一套浏览器会话里看到。
