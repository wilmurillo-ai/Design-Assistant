# 微信公众号写作指南

## 写作原则

### 语言风格
- "说人话"，避免行业黑话（闭环、链路、抓手、颗粒度、赋能）
- 原创优先，原创内容流量和分成都更高
- 每条新闻精简到2-3句话，保留核心信息和关键数据
- 关键数据用红色加粗突出，制造视觉锚点

### 结构框架
```
开头（黄金3秒）→ 直接抛痛点/悬念/数据/故事
正文（分段+小标题）→ 2-3行一段，图文结合
结尾 → 总结观点 + 引导互动 + 引导关注
```

### 互动设计
- 设计"留言诱因"，留言权重最高
- 结尾提问引导评论（如"你怎么看？""你最关注哪条？"）
- 适当使用投票/选择题式互动

### 互动数据权重（高→低）
1. 留言（最高）
2. 分享
3. 在看
4. 点赞
5. 划线

## 排版规范

| 项目 | 设置 |
|------|------|
| 字体 | -apple-system, BlinkMacSystemFont, 'Helvetica Neue', 'PingFang SC' |
| 正文字号 | 14-15px |
| 正文颜色 | #555 或 #3f3f3f |
| 标题字号 | 17-22px |
| 标题颜色 | #1a1a2e |
| 行高 | 1.9-2.0 |
| 段落 | 2-3行一段，段间空一行 |
| 强调数据 | `<strong style="color: #e94560;">数据</strong>` |
| 配图小字 | 11px, #999, italic |

## 配图原则

- 每篇文章建议2-4张配图
- 头条区域必配图（场景化、有冲击力）
- 分区切换处配图（承上启下）
- 配图需与上下文内容强相关
- 配图下方标注说明文字
- 所有图片必须上传到微信CDN

## 避坑清单

- 版权：文章和图片侵权都可能赔钱
- 评论区：不妥内容可能导致限流
- 限流判断：标题没问题但阅读量不变 → 可能被限流
- 系统不通知限流，需自己在订阅号助手查看
- 正文图片用外部URL会被微信过滤，必须用upload-content-image
- 标题超32字会报错，摘要超128字会报错
