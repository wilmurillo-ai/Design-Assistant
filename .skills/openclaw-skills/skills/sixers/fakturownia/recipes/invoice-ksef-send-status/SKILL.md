# Send Existing Invoice To KSeF And Inspect Status

> Read [fakturownia-invoices](../../subskills/invoices/SKILL.md) first.

Use this when an invoice already exists and you want to send it to KSeF separately from create or update.

`send-gov` is the API-native CLI name; it means “send the invoice to KSeF”.

## Queue The Invoice For KSeF

```bash
fakturownia invoice send-gov --id 111 --json
```

## Inspect KSeF Status Fields

```bash
fakturownia invoice get --id 111 --fields id,number,gov_status,gov_id,gov_send_date,gov_error_messages[] --json
```

## Notes

- `gov_status` and related fields are the KSeF integration fields exposed by the upstream API.
- If the API reports send or validation errors, inspect `gov_error_messages[]` and retry after fixing the payload.
