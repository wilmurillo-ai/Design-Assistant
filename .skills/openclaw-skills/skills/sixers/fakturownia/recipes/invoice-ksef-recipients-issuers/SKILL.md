# KSeF Recipients And Issuers

> Read [fakturownia-invoices](../../subskills/invoices/SKILL.md) first.

Use this when KSeF invoice payloads need explicit recipient or issuer roles.

## Example

```bash
fakturownia invoice create --input '{
  "kind":"vat",
  "seller_name":"Wystawca Sp. z o.o.",
  "buyer_name":"Klient1 Sp. z o.o.",
  "positions":[{"name":"Produkt A1","tax":23,"total_price_gross":10.23,"quantity":1}],
  "recipients":[{
    "name":"Odbiorca1",
    "company":true,
    "tax_no":"1234567890",
    "tax_no_kind":"nip_with_id",
    "role":"Dokonujący płatności",
    "participation":100
  }],
  "issuers":[{
    "name":"Wystawca1",
    "company":true,
    "tax_no":"1234567890",
    "tax_no_kind":"",
    "country":"PL",
    "role":"Wystawca faktury"
  }]
}' --json
```

## Notes

- The CLI uses API-native field names, but the recipe language uses KSeF as the business concept.
- Inspect `request_body_schema` for the curated role and `tax_no_kind` enum values before automating payload generation.
