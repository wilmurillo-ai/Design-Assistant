# KSeF Correction And Correction To Zero

> Read [fakturownia-invoices](../../subskills/invoices/SKILL.md) first.

Use this when a KSeF invoice needs correction and the upstream workflow requires the KSeF number of the corrected invoice.

## Inspect The Existing Invoice

```bash
fakturownia invoice get --id 2432393 --fields id,number,gov_id,gov_status --json
```

## Create A KSeF Correction

```bash
fakturownia invoice create --input '{
  "kind":"correction",
  "invoice_id":2432393,
  "from_invoice_id":2432393,
  "gov_corrected_invoice_number":"KSEF-EXAMPLE-NUMBER",
  "correction_reason":"Korekta do zera",
  "positions":[{
    "name":"Usługa",
    "quantity":-1,
    "total_price_gross":"-123.00",
    "tax":"23",
    "kind":"correction"
  }]
}' --json
```

## Notes

- The API-native field name is `gov_corrected_invoice_number`; it is the KSeF number of the corrected invoice.
- Keep correction-to-zero logic in payloads and operator guidance rather than expecting a dedicated CLI verb.
