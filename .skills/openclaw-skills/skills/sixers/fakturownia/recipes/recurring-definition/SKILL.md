# Create Or Update A Recurring Definition

> Read [fakturownia-recurrings](../../subskills/recurrings/SKILL.md) first.

Use this when the task is about the recurring definition itself rather than a concrete invoice instance.

## Create

```bash
fakturownia recurring create --input '{
  "name":"Nazwa cyklicznosci",
  "invoice_id":1,
  "start_date":"2016-01-01",
  "every":"1m",
  "issue_working_day_only":false,
  "send_email":true,
  "buyer_email":"mail1@mail.pl, mail2@mail.pl",
  "end_date":"null"
}' --json
```

## Update Next Invoice Date

```bash
fakturownia recurring update --id 111 --input '{"next_invoice_date":"2016-02-01"}' --json
```
