# solution-architect - 解决方案架构师

**提供经过验证的商业解决方案框架**

---

## 📋 简介

`solution-architect` 不是生成报告的工具，而是**提供经过验证的解决方案框架**。

三大核心解决方案库：
1. **增长营销**（特斯拉台湾方案）- AI 驱动的内容飞轮×社群裂变×跨界破圈
2. **内容升级**（36 氪方案）- 一核两翼三驱动，从内容团队到产品化转型
3. **平台增长**（小米 MiMo 方案）- 三层驱动引擎，构建开发者生态

每个方案都包含：
- ✅ 核心框架
- ✅ 战术库（具体可执行）
- ✅ KPI 体系（量化目标）
- ✅ 实施路线图
- ✅ 风险与应对

---

## 🚀 快速开始

### 查看所有解决方案

```bash
python3 script.py list
```

### 查看方案详情

```bash
python3 script.py show growth-marketing
python3 script.py show content-upgrade
python3 script.py show platform-growth
```

### 根据问题找方案

```bash
python3 script.py solve --problem "内容团队效率低，想引入 AI"
```

### 根据目标找路径

```bash
python3 script.py plan --goal "6 个月内将开发者数量从 1000 提升到 10 万"
```

### 优化现有方案

```bash
python3 script.py enhance --my-plan 我的方案.md --reference growth-marketing
```

## 📦 核心解决方案

### 1. 增长营销（特斯拉方案）

**核心框架**: AI 赋能的内容飞轮 × 本地化社群裂变 × 跨界体验破圈

**适用场景**:
- 品牌需要提升社交媒体声量
- 需要通过内容驱动用户增长
- 需要构建高活跃度用户社群

**核心战术**:
- 视频内容矩阵（5 平台定位）
- AI 内容生产线（效率提升 70%）
- 社群裂变（10 万活跃用户）
- 跨界项目（2+ 标志性项目）

**实施周期**: 12-18 个月

---

### 2. 内容升级（36 氪方案）

**核心框架**: 一核两翼三驱动

**适用场景**:
- 内容团队需要引入 AI 提升效率
- 需要从"内容团队"转型为"产品团队"
- 需要构建深度研究能力

**核心战术**:
- 三级产品矩阵（流量层/核心层/产品层）
- 财报分析 SOP（五步流程）
- AI 生产管线（效率提升 50%+）
- 专家网络与高价值社群

**实施周期**: 12 个月

---

### 3. 平台增长（小米 MiMo 方案）

**核心框架**: 三层驱动增长引擎（产品力/运营力/生态力）

**适用场景**:
- 需要构建开发者生态
- 需要运营 API/平台产品
- 需要实现从"技术发布"到"生态繁荣"的跨越

**核心战术**:
- 产品力引擎（定价/API/解决方案）
- 运营力引擎（开发者全生命周期）
- 生态力引擎（内外联动）
- 增长飞轮设计

**实施周期**: 18 个月

## 💡 使用场景

### 场景 1: 我有业务问题

```bash
# 问题：我们的内容团队效率低，想引入 AI 但不知道从哪里开始
python3 script.py solve --problem "内容团队效率低，想引入 AI"

# 输出：
# ✅ 匹配解决方案：content-upgrade
# ✅ 推荐核心框架：AI 驱动的内容工业化生产
# ✅ 推荐战术：财报分析 SOP、AI 生产管线、前台 + 中台组织模式
```

### 场景 2: 我有目标

```bash
# 目标：6 个月内将开发者数量从 1000 提升到 10 万
python3 script.py plan --goal "6 个月内开发者从 1000 到 10 万"

# 输出：
# ✅ 推荐方案：platform-growth
# ✅ 核心引擎：三层驱动（产品力/运营力/生态力）
# ✅ 18 个月路线图（可压缩）
```

### 场景 3: 我有方案需要优化

```bash
python3 script.py enhance --my-plan 我的营销方案.md --reference growth-marketing

# 输出：
# ✅ 对比分析：你的方案 vs 特斯拉方案
# ✅ 缺失模块：识别差距
# ✅ 优化建议：具体改进点
```
