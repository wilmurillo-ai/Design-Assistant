# legal-brief-drafter v3.0

**法律文书整理与起草系统** - 基于实战经验提炼的智能法律文书生成工具

---

## 📋 简介

`legal-brief-drafter` 是一款专业的法律文书起草工具，源自微博粉丝头条行政诉讼案的实战经验提炼。系统提供 7 种法律文书模板、5 种法律推演方法、6 大论证策略库和 8 个分析框架，帮助用户快速生成高质量的法律文书。

**核心定位**：
- ✅ 不是简单的文书模板填充
- ✅ 而是法律论证的策略库和推演引擎
- ✅ 从"文书起草"升级为"法律推演系统"

---

## ✨ 核心功能

### 1️⃣ 7 种法律文书模板

| 模板 | 适用场景 | 说明 |
|------|---------|------|
| **行政诉讼驳斥意见** ⭐ | 起诉市场监管、行政复议等 | 四模块攻击框架（程序/事实/法律/价值） |
| **反驳意见书** | 民事诉讼反驳对方答辩 | 循环论证识别、逻辑矛盾揭示 |
| **庭后意见书** | 庭审后提交综合意见 | 程序问题 + 实体问题 + 专项论证 |
| **质证意见** | 对对方证据发表意见 | 三性分析（真实性/合法性/关联性） |
| **代理词** | 代理人庭上陈述 | 精简版核心观点阐述 |
| **上诉状** | 不服一审判决上诉 | 上诉理由 + 请求 |
| **答辩状** | 回应原告起诉 | 事实 + 法律 + 请求 |

### 2️⃣ 5 种法律推演方法

| 方法 | 结构 | 适用场景 |
|------|------|---------|
| **演绎推理** | 大前提→小前提→结论 | 法律规则明确的案件 |
| **归纳推理** | 多案例→裁判规则 | 类案检索与趋势分析 |
| **类比推理** | 案例 A→本案 | 新型案件法律适用 |
| **反事实推理** | 假设情景分析 | 因果关系与责任归属 |
| **成本效益分析** | 方案 A vs 方案 B | 责任分配与制度设计 |

### 3️⃣ 6 大论证策略库

| 策略 | 使用方法 | 示例 |
|------|---------|------|
| **归谬法/思想实验** | 推广到极端情形 | "如果涉案金额是 200 万，此规则是否公平？" |
| **证据反用法** | 以子之矛攻子之盾 | "对方依赖的协议正是其未公平提示的铁证" |
| **义务升级论证** | 强调优势推导更高义务 | "拥有技术优势即负有更高风险预见义务" |
| **类比论证** | 类比权威关系 | "平台与用户≈法庭与诉讼参与人" |
| **程序正义强调** | 形式审查 vs 实质审查 | "行政机关形式审查 vs 法律要求的实质审查" |
| **价值升华** | 个案上升到制度 | "关乎反对平台系统性傲慢" |

### 4️⃣ 8 个分析框架

1. **格式条款效力六维分析** - 性质/提示/公平/解释/平台义务/减损
2. **循环论证识别与破解** - 识别用"条款有效"证明"条款有效"
3. **逻辑矛盾揭示与利用** - 找出互斥主张，构建困境表
4. **行政诉讼合法性审查** - 职权/事实/法律/程序/合理性
5. **类案检索与裁判规则提炼** - 检索类似案例，提取裁判规则
6. **社会意义与示范效应论证** - 行业/消费者/法治影响
7. **行政诉讼程序合法性攻击** ⭐ - 形式 vs 实质审查
8. **法律推演置信度评估** ⭐ - 量化评估论证可靠性

## 🚀 快速开始

### 安装

```bash
# 从 ClawHub 安装
clawdhub install legal-brief-drafter

# 或手动安装
git clone <repo_url> ~/.openclaw/skills/legal-brief-drafter
```

### 基本使用

```bash
# 查看技能信息
python3 script.py --action info

# 起草行政诉讼驳斥意见
python3 script.py --action draft \
  --template 行政诉讼驳斥意见 \
  --input 案件素材.txt \
  --output 驳斥意见.md

# 法律推演分析
python3 script.py --action analyze \
  --inference-method 演绎 \
  --input 案件事实.txt \
  --output 分析结果.json

# 发布到 ClawHub 社区
python3 script.py --action push
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `--action` | ✅ | 执行动作（draft/analyze/push/info） |
| `--input` | ❌ | 输入文件路径（案件素材/庭审笔录等） |
| `--output` | ❌ | 输出文件路径 |
| `--template` | ❌ | 文书模板类型（7 种可选） |
| `--inference-method` | ❌ | 推演方法（5 种可选） |

## 📖 使用指南

### 场景 1：行政诉讼驳斥意见

**适用**：起诉市场监管局、行政复议决定等

**步骤**：

1. **准备素材**
   ```
   案件素材.txt 包含：
   - 行政决定书全文
   - 复议决定书全文
   - 你的核心质疑点
   - 相关证据材料
   ```

2. **执行起草**
   ```bash
   python3 script.py --action draft \
     --template 行政诉讼驳斥意见 \
     --input 案件素材.txt \
     --output 驳斥意见.md
   ```

3. **填充内容**
   - 模块一：攻击程序合法性（形式 vs 实质）
   - 模块二：攻击事实认定（表象 vs 本质）
   - 模块三：攻击法律适用（片面 vs 全面）
   - 模块四：攻击逻辑与价值（单方 vs 公平）

4. **输出结构**
   ```markdown
   # 关于对 [行政机关] [文书名称] 的驳斥意见
   
   ## 引言
   - 定性指控
   - 亮明目的
   
   ## 正文（四模块）
   - 模块一：攻击程序合法性
   - 模块二：攻击事实认定
   - 模块三：攻击法律适用
   - 模块四：攻击逻辑与价值
   
   ## 总结与请求
   - 总结错误
   - 明确请求
   - 高阶请求
   ```

### 场景 2：法律推演分析

**适用**：案件胜诉率评估、论证策略选择

**步骤**：

1. **选择推演方法**
   - 法律规则明确 → 演绎推理
   - 类案丰富 → 归纳推理
   - 新型案件 → 类比推理
   - 因果关系复杂 → 反事实推理
   - 责任分配争议 → 成本效益分析

2. **执行分析**
   ```bash
   python3 script.py --action analyze \
     --inference-method 演绎 \
     --input 案件事实.txt \
     --output 分析结果.json
   ```

3. **解读结果**
   ```json
   {
     "confidence": {
       "level": "高",
       "score": 0.85
     },
     "recommendations": ["建议 1", "建议 2"]
   }
   ```

## ⚠️ 注意事项

1. **不提供法律意见** - 本技能仅协助整理和起草文书，不替代律师
2. **法条需验证** - AI 引用的法条必须由用户核实原文
3. **隐私保护** - 文书中的个人信息建议使用 XXX 占位
4. **时效性** - 注意法律法规的最新修订版本
5. **置信度标注** - 推演结果应标注置信度等级（≥0.7 可作为主要论点）
6. **行政诉讼特殊规则** - 被告负举证责任，注意运用
