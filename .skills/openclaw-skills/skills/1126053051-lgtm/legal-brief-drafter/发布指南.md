# legal-brief-drafter v3.0.0 发布指南

## 当前状态

✅ **已登录**: `clawdhub whoami` 显示 `✔ 1126053051-lgtm`  
⚠️ **发布问题**: `clawdhub publish .` 报错 `--version must be valid semver`  
⚠️ **速率限制**: `clawdhub search` 显示 `Rate limit exceeded`（需等待 60 秒）

---

## 问题分析

**问题**: clawdhub 报版本号错误，但 `3.0.0` 是有效的 semver 格式

**可能原因**:
1. clawdhub 解析 YAML 时版本号被当作浮点数而非字符串
2. clawdhub 内部 bug
3. 需要特定格式的版本号

---

## 解决方案

### 方案 1：等待速率限制解除后重试

```bash
# 等待 60 秒
sleep 60

# 验证速率限制解除
clawdhub search legal-brief-drafter

# 尝试发布
cd ~/.openclaw/workspace/skills/legal-brief-drafter
clawdhub publish .
```

### 方案 2：使用简化版 skill.yaml

```bash
cd ~/.openclaw/workspace/skills/legal-brief-drafter

# 备份原文件
cp skill.yaml skill.yaml.backup

# 使用简化版
cp skill.minimal.yaml skill.yaml

# 发布
clawdhub publish .

# 还原
cp skill.yaml.backup skill.yaml
```

### 方案 3：手动上传到 ClawHub 网站

1. 访问 https://clawhub.com
2. 登录账号
3. 点击 "Publish Skill" 或 "Create New"
4. 上传文件：
   - SKILL.md
   - script.py
   - skill.yaml
   - README.md
   - LICENSE
5. 填写元数据：
   - Name: legal-brief-drafter
   - Version: 3.0.0
   - License: CC-BY-NC-SA-4.0
   - Description: 法律文书整理与起草系统
6. 提交发布

### 方案 4：联系 OpenClaw 团队

如果以上方案都失败，可能是 clawdhub CLI 的 bug：

```bash
# 报告问题
clawdhub --help
# 查看是否有报告问题的方式

# 或者访问 Discord 社区
https://discord.com/invite/clawd
```
