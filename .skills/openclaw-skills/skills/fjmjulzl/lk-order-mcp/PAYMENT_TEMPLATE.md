# 💳 lk-order-mcp 支付输出模板

## 📋 支付信息输出规范

下单成功后，支付信息必须以 **Markdown 格式** 输出，包含以下内容：

---

```markdown

| 项目 | 内容 |
|------|------|
| **订单号** | `{orderId}` |
| **支付方式** | {payType} |
| **支付金额** | ¥{amount} |
| **门店** | {shopName} |
| **取餐方式** | {deliveryType} |

### 🔗 支付链接

[点击打开支付页面]({payUrl})

或复制链接到浏览器：
```
{payUrl}
```

### 📸 支付二维码

![支付二维码]({qrImageUrl})

> 💡 使用支付宝或微信扫一扫即可完成支付

---

### 📦 订单详情

| 商品 | 规格 | 数量 |
|------|------|------|
| {productName} | {skuDesc} | {amount} |

**原价**: ¥{originalPrice}  
**优惠**: ¥{discount}  
**实付**: ¥{finalPrice}

---

### ⏰ 取餐信息

- **门店**: {shopName}
- **地址**: {shopAddress}
```

---

## 🎨 输出元素说明

### 1. 支付链接

**格式**：
```markdown
[点击打开支付页面]({payUrl})
```

**要求**：
- 必须是可点击的 Markdown 链接
- 同时提供纯文本链接方便复制
- 链接格式：`https://openpre.lkcoffee.com/pay?orderId={orderId}`

---

### 2. 二维码图片

**格式**：
```markdown
![支付二维码]({qrImageUrl})
```

**要求**：
- 使用 `![alt text](url)` Markdown 图片语法
- 二维码 URL 格式：`https://api.qrserver.com/v1/create-qr-code/?size=300x300&data={payUrl}`
- 图片尺寸建议：300x300 像素
- 必须确保图片可公开访问

**示例**：
```markdown
![支付二维码](https://api.qrserver.com/v1/create-qr-code/?size=300x300&data=https%3A%2F%2Fopenpre.lkcoffee.com%2Fpay%3ForderId%3D7629296683999035401)
```

---

### 3. 订单详情表格

**格式**：
```markdown
| 商品 | 规格 | 数量 |
|------|------|------|
| {productName} | {skuDesc} | {amount} |
```

**要求**：
- 使用 Markdown 表格
- 包含商品名称、规格、数量
- 如有多个商品，每行一个

---

### 4. 价格明细

**格式**：
```markdown
**原价**: ¥{originalPrice}  
**优惠**: ¥{discount}  
**实付**: ¥{finalPrice}
```

**要求**：
- 使用加粗突出关键信息
- 使用 `¥` 符号
- 换行使用两个空格或 `<br>`

---

### 5. 取餐信息

**格式**：
```markdown
- **门店**: {shopName}
- **地址**: {shopAddress}
- **营业时间**: {workTime}
- **取餐号**: {pickUpCode}
```

**要求**：
- 使用无序列表
- 关键信息加粗
- 地址完整清晰

## 📋 注意事项

1. **必须输出二维码图片**
   - 使用 `![alt](url)` 格式
   - 确保图片 URL 可公开访问
   - 尺寸适中（建议 300x300）

2. **必须输出可点击链接**
   - 使用 `[text](url)` 格式
   - 同时提供纯文本方便复制

3. **使用表格展示信息**
   - 支付信息用表格
   - 订单详情用表格
   - 清晰易读

4. **关键信息加粗**
   - 订单号
   - 金额
   - 门店名称

5. **分段清晰**
   - 使用 `---` 分隔不同部分
   - 每个部分有明确标题
