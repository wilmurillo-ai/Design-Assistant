# Weekend Scout -- Design Changes Log

Historical design drift log for Weekend Scout.
Rows are kept in reverse chronological order and track material design and
contract changes relative to the current design in `docs/weekend-scout-design-v2.md`.

| Date | Design Section | Change | Reason |
|------|---------------|--------|--------|
| 2026-04-22 | 5. Skill Definition / 8. Installation | Repo-root `SKILL.md` is now a permanent bundle bootstrap/dispatcher entrypoint: it checks the bundle package version, runs `install/install_skill.py --with-pip --runtime-only` when the local runtime is missing or stale, then dispatches into the matching nested `.openclaw/`, `.agents/`, or `.claude/` runtime skill; `install/install_skill.py` gained `--runtime-only` to skip shared/global skill-copy side effects during that bootstrap | Workspace-installed bundles shadow shared/global skill directories, so the root skill cannot safely self-promote or self-replace; it must remain the stable entrypoint and bootstrap the runtime in place |
| 2026-04-22 | 6. Project Structure / 5. | 2026-04-17 | 7. CLI Command Reference / 5.2 Skill Workflow (platform-transport) | `init` / `init-skill` now emit `cache_dir` as a POSIX forward-slash string on every platform, `_load_json_argument` rejects drive-letter paths with no separator via a `malformed_path` error, and the bundled `platform-transport.md` tells the skill to use `cache_dir` verbatim | A Windows + git-bash run corrupted `--cities-file` / `--events-file` arguments — bash ate the unquoted backslashes in `D:\Work\...\_tmp_*.tmp`, producing a drive-relative path that Cygwin bash wrote as a stray `D:WorkWeekend-Scout.weekend_scoutcache_tmp_events_1.tmp` file at the repo root.
