# On-Page SEO Reference

Use this file for page-level relevance and SERP presentation work.

## Priority order

1. Search intent match
2. Title tag
3. Meta description
4. H1 / heading structure
5. Canonical correctness
6. Internal linking
7. Image alt text

## Titles

### Strong heuristics
- Ideal display range: roughly 50–60 characters
- Put the main topic early when natural
- Avoid duplicate titles across important pages
- Brand suffix is optional if space is tight

### Diagnose
- Too vague → weak relevance signal
- Too long → snippet truncation risk
- Too short → underuses SERP real estate
- Repeated across pages → cannibalization / ambiguity risk

### Good response pattern
- State char count
- Explain mismatch vs intent
- Offer 3–5 better options

## Meta descriptions

### Strong heuristics
- Usually best around 120–156 chars
- Include the core query/theme naturally
- Optimize for CTR, not keyword stuffing
- Each important page should have a distinct description

### Diagnose
- Missing → Google may rewrite snippet
- Generic → low CTR potential
- Too long → truncation risk
- Repeated → weak page differentiation

## H1 / heading structure

### Rules of thumb
- Exactly one H1 is preferred for most pages
- H2s should map to major subtopics or user questions
- H3s should sit beneath H2s, not float alone
- Don’t manufacture headings just to stuff keywords

### What to flag
- Missing H1
- Multiple H1s with no clear reason
- Heading hierarchy skipped or chaotic
- Headings that don’t reflect search intent

## Canonicals

### Basics
- Self-referential canonical is usually the safest default for canonical pages
- Missing canonical is not always fatal, but is worth noting
- Conflicting canonicals or canonicalizing to the wrong URL can suppress rankings

### Common issues
- Pagination or faceted pages canonicalized incorrectly
- Canonical pointing to homepage/site root in error
- Different canonical than indexable target without rationale

## Internal linking

### Heuristics
- Important pages should receive contextual internal links from relevant pages
- Pages should not be dead ends when a useful onward path exists
- Use descriptive anchor text; avoid generic anchors when context matters

### Audit questions
- Is this page orphaned or weakly linked?
- Are internal anchors descriptive?
- Are related pages visibly connected?
- Is the nav doing all the work while body links are absent?

## Image alt text

### What good alt text does
- Describes the image meaningfully
- Supports accessibility first
- Can reinforce relevance when used naturally

### What not to do
- Stuff keywords unnaturally
- Add alt text that says nothing useful
- Treat decorative images like content images

## Suggested page-audit output

- Title: current + count + verdict
- Meta: current + count + verdict
- H1/H2/H3 summary
- Canonical status
- Internal linking observations
- Image alt summary
- Exact rewrites where useful
