# Claude Code 架构设计模式

> 从源码泄露事件（2026年3月31日，512,000行TypeScript）学到的设计思路

## 背景

2026年3月31日，Anthropic 的 Claude Code 通过 npm 包的 source map 文件意外泄露了完整源码。这是第二次泄露（同年2月24日也曾发生）。

源码揭示了 Anthropic 内部构建 AI 编程工具的核心架构，这些设计模式对构建类似系统有重要参考价值。

## 核心发现

**AI 编程工具的护城河不是模型，而是 harness（框架）**

模型能力固然重要，但真正的竞争优势在于：
- 任务编排能力
- 工具系统设计
- 上下文管理
- 安全机制

---

## 架构设计原则

### 1. Initiative 与 Execution 分离

将"想做什么"（计划）与"怎么做"（执行）解耦。

```
┌─────────────────┐     ┌─────────────────┐
│   Initiative    │────▶│   Execution     │
│   (规划层)       │     │   (执行层)       │
└─────────────────┘     └─────────────────┘
```

**好处：**
- 一个 Master Agent 可以指挥多个 Worker Agent 并行执行
- 任务规划与具体操作解耦，更灵活
- 便于实现多Agent协作

---

### 2. 读写分离并发

- **只读操作**：可以并发执行，互不阻塞
- **写操作**：必须排队，确保数据一致性

```typescript
// 示例逻辑
if (operation.isRead) {
  await Promise.all(parallelOperations)
} else {
  await serializeOperations()  // 写操作排队
}
```

**安全默认：**
- 工具没有声明只读属性 = **默认危险**
- 需要显式权限确认才能执行

---

### 3. 工具按需加载

**不是一次性加载所有工具**，而是：

1. 先给模型简版工具描述（名称+用途）
2. 模型选中某个工具后，再加载完整参数
3. 按需获取详细信息，省 Token

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  工具列表    │───▶│  选中工具    │───▶│  完整参数    │
│  (轻量)      │    │  (请求)      │    │  (按需)      │
└──────────────┘    └──────────────┘    └──────────────┘
```

---

### 4. 记忆不记代码

**代码事实实时从源码读取**，而不是依赖记忆。

- 代码可能经常变化
- 记忆会失效，但源码是最新的
- 避免"我以为是这样但实际不是"的幻觉

```python
# 好的做法
def get_function_signature(func):
    return read_source_code(func)  # 实时读取
```

---

### 5. 五级上下文压缩

当上下文快溢出时，自动压缩：

| 级别 | 方法 | 压缩比 |
|------|------|--------|
| 1 | 剪裁 | 删除低价值消息 |
| 2 | 微压缩 | 精简长消息 |
| 3 | 折叠 | 摘要长对话 |
| 4 | 自动压缩 | AI 总结 |
| 5 | 硬压缩 | 强制保留关键信息 |

---

### 6. 插件式工具架构

每个能力都是**独立、权限门控**的单元：

```
┌─────────────────────────────────────────────┐
│                 Tool System                  │
├─────────────┬─────────────┬─────────────────┤
│  BashTool   │   GitTool   │   FileTool      │
├─────────────┼─────────────┼─────────────────┤
│ 权限模型     │  权限模型    │   权限模型      │
│ 验证逻辑    │  验证逻辑    │   验证逻辑      │
│ 输出格式    │  输出格式    │   输出格式      │
└─────────────┴─────────────┴─────────────────┘
```

**每个工具都有：**
- 独立的权限模型
- 独立的验证逻辑
- 独立的输出格式化

---

### 7. Fail-closed 安全设计

**默认拒绝**未知操作：

- 工具没有声明"只读" = 危险
- 需要显式权限确认才能执行
- 黑名单优先于白名单

```typescript
// 默认行为
const isSafe = tool.declaredReadOnly && !tool.hasSideEffects
if (!isSafe) {
  requireExplicitPermission()
}
```

## 代码安全实践

### Undercover Mode

当 Anthropic 员工在内部仓库工作时，自动隐藏 AI 身份：

```typescript
// 检查是否在内部仓库
if (isInternalRepo(repoUrl)) {
  injectUndercoverPrompt()  // 隐藏模型代号
}
```

**规则：**
- 不在 commit message 里透露模型代号（如 Capybara、Tengu）
- 不写 Unreleased 模型版本号
- 不留 AI 身份痕迹
- 默认开启，无法关闭
