## 概述

对公司对外交流材料进行系统性的风险审查与质量评估，识别潜在的保密信息泄露、内容错误、
法律合规问题及专业性不足之处，帮助团队在材料发出前发现并修正问题，保护公司利益和声誉。

# 审查提示词：对外材料（External Materials）

**适用文件夹名**：`对外材料`
**范围**：拟投 SCI 等学术论文、专利交底书、对外汇报等。
**核心价值**：规范传递价值并控制对外风险。

## Agent 执行说明

执行 Phase B 时，将本节 **审查维度表** 作为该类文档的 **唯一** 内容审查 checklist：对每一行维度逐条对照原文给出结论（**合规与数据安全** 须结合主 Skill 附录中的 **AI 对话链接检测结果**），写入审查报告「3. 内容审查结论」；问题写入「4. 分级问题清单」并按主 Skill 的严重度规则 @ 作者。

**报告输出**：审查完成后，创建飞书云文档保存报告，标题为 `review_<文档短标题>_<YYYYMMDD>`，移动至 `审查报告/对外材料/` 文件夹。

如需更细对外话术与品牌类审查，可对照工作区 [material-review/SKILL.md](../../material-review/SKILL.md) 作补充参考（本提示词未涵盖的条款再以该 Skill 为准）。

## 审查维度（Agent 自检），按六大维度审查

### 维度一：合规与保密风险（权重最高，优先审查）

**检查项：**
- [ ] 是否包含未经授权的客户名称、项目名称、案例数据
- [ ] 是否包含公司内部财务数据（营收、利润、成本）
- [ ] 是否包含员工个人信息（姓名+职位+联系方式组合）
- [ ] 是否包含竞争对手的负面表述或对比（可能引发法律纠纷）
- [ ] 是否包含"保密"、"内部"、"仅供参考"等标注但对外发送
- [ ] 是否引用了未公开发布的产品功能、路线图
- [ ] 是否包含政治敏感内容（国家主权、领土、政治立场）
- [ ] 是否包含违反当地法规的内容（广告法、数据保护法等）
- [ ] 承诺表述是否存在**极端夸张**的绝对化用语（见下方判断标准）

**绝对化用语判断标准（重要）：**

常见的技术性描述语言（如"精准"、"稳定"、"高效"、"彻底解决"、"智能感知"等）
属于行业惯用表达，**不应标记为合规风险**，请直接忽略。

只有同时满足以下特征时才应标记：
- 使用"**100%**"、"**绝对**"、"**全球最**"、"**业界唯一**"、"**永不**"等极端量化或排他性表述
- 或明确以书面形式"**保证**达到……效果"、"**承诺**……不会发生"等带有合同性质的绝对承诺

以下示例**不应标记**（常见技术文档用语）：
- "精准提取故障特征"、"稳定识别高频调制带"
- "实现智能感知"
- "高精度"、"高可靠"、"全面覆盖"

以下示例**才应标记**（极端夸张表述）：
- "100% 准确率，绝不误报"
- "业界唯一能做到……的产品，保证效果"
- "永远不会出现故障"

**风险分级：**
- 🔴 Critical：客户机密数据外泄、政治敏感内容 → **必须删除，禁止发出**
- 🟠 High：内部财务数据、未授权客户案例 → **发出前必须处理**
- 🟡 Medium：极端夸张承诺表述（符合上述标准的） → **建议修改措辞**

---

### 维度二：品牌与视觉规范

**检查项：**
- [ ] 公司名称/产品名称拼写是否正确（注意大小写、空格）
- [ ] Logo 使用是否规范（尺寸、颜色、保护区域）
- [ ] 品牌色彩是否与视觉识别手册一致
- [ ] 字体是否使用授权字体
- [ ] 版式是否使用官方模板
- [ ] 页眉/页脚格式是否统一（公司信息、版权声明、页码）
- [ ] 图片分辨率是否清晰（≥ 96dpi，建议 150dpi+）
- [ ] 是否使用了未经授权的第三方图片/图标
- [ ] 联系方式（邮箱、电话、网址）是否为官方渠道且准确

---

### 维度三：内容准确性

**检查项：**
- [ ] 数据和统计数字是否标注来源
- [ ] 行业数据是否为最新（检查年份，超过2年的数据需确认）
- [ ] 产品功能描述是否与当前版本一致
- [ ] 价格/费率信息是否为最新
- [ ] 公司信息（成立时间、员工规模、办公地点）是否准确
- [ ] 合作伙伴/认证资质信息是否有效
- [ ] 引用的第三方报告、新闻是否真实可查
- [ ] 专业术语使用是否正确

**⚠️ 重要：以下情况不应标记为"需确认"问题**
- **产品组件/模块的列举**：文档中对自有产品功能模块、算法模型、子系统的逐条列举
  （例如"趋势分析大模型、波形分析大模型、轴承分析大模型……"），属于对产品构成的
  描述性说明，**无需要求作者确认真实性**，不应纳入问题清单
- **架构设计的描述**：对系统层次、引擎模块、数据流程的文字描述，不属于需要数据源
  佐证的"声明"，不应标记为风险项
- **需要标记的情形**（满足以下之一才应标记）：
  - 涉及具体性能指标（准确率、响应时间、处理量等量化数据）但无来源
  - 与已知产品版本存在明显矛盾
  - 使用"保证"、"业界最优"、"唯一"等绝对化表述

---

### 维度四：逻辑与结构

**检查项：**
- [ ] 整体逻辑是否清晰（有明确的开篇→展开→结论）
- [ ] 目标受众定位是否准确（内容深度与受众匹配）
- [ ] 核心信息（价值主张）是否突出、重复强调
- [ ] 章节顺序是否合理，前后是否有逻辑矛盾
- [ ] 数据与结论是否对应，有无过度推断
- [ ] Call-to-Action（行动召唤）是否明确
- [ ] PPT：每页是否只表达一个核心观点
- [ ] 文档：段落是否有清晰的主题句

---

### 维度五：语言表达

**检查项：**
- [ ] 是否有错别字
- [ ] 语法是否正确（尤其双语材料的英文部分）
- [ ] 表述是否专业，避免口语化
- [ ] 是否有冗长啰嗦的句子（建议单句不超过50字）
- [ ] 专业术语是否有必要的解释（针对非专业受众）
- [ ] 数字格式是否统一（千位分隔符、小数点、单位）
- [ ] 日期格式是否统一（YYYY年MM月DD日 或 YYYY-MM-DD）

**⚠️ 以下项目不纳入审查，请忽略：**
- **标点符号格式**：中英文标点混用（全角/半角括号、冒号、逗号、句号等）不作为问题标记，
  即使全文存在不统一情况也请直接略过，不生成相关问题条目
- **中英文之间是否有空格**：不作为审查项

**⚠️ 重要：产品名称/缩写格式不应标记为问题**
- 中文文档中使用中文括号（如"多模态故障诊断智能体（PHMGPT）"）是正确格式，
  **不应标记为括号格式错误**
- 产品缩写（如 PHMGPT）在首次出现后单独使用，是正常的行文习惯，**不算格式不统一**
- 只有全文从未给出全称、或同一段落内括号中英文混用且显然笔误时，才应标记

---

### 维度六：视觉呈现质量（PPT/图文材料专项）

**检查项：**
- [ ] 文字是否过密（每页超过100字需警惕）
- [ ] 字号是否可读（正文不小于14pt，标题不小于24pt）
- [ ] 颜色对比度是否足够（文字与背景）
- [ ] 图表是否有标题、单位、数据来源
- [ ] 排版是否整齐，元素是否对齐
- [ ] 动画/过渡是否适度（避免过度花哨）
- [ ] 整体风格是否统一

### 生成报告

按照下方【报告模板】输出 Markdown 格式报告，**创建为飞书云文档**。

**飞书操作**：
1. 使用 `createDocument` API 创建新文档，标题为 `review_<文档短标题>_<YYYYMMDD>`
2. 将报告内容（Markdown 格式）写入文档
3. 将文档移动到飞书知识库的 `审查报告/文献调研/` 文件夹
4. 将新创建的飞书文档链接记录到汇总表中

## 特殊场景处理

### 双语材料（中英文）
- 中英文内容需分别审查，重点检查：翻译是否准确、专业术语是否一致、
  格式（标点、空格）是否符合各语言规范

### 含数据图表的材料
- 需特别关注：图表数据与正文描述是否一致、Y轴起点是否从0开始（防止误导）、
  样本量和置信区间是否标注

### PDF 字体渲染导致的兼容字符问题（**忽略，不纳入审查问题**）
- 部分 PDF 使用思源黑体、康熙字典体等特殊字体，文字提取工具可能输出
  CJK 兼容字符（如"⼯"、"⼒"、"⼼"等 Kangxi 部首字符）
- 这属于工具层面的字体渲染问题，**不代表文档本身存在错误**，审查时请直接忽略，
  不要将其列为问题或建议修改字体

### PDF 中的嵌入图片/图表（**忽略，不纳入审查问题**）
- 当提取的文本中出现 `[📷 第N页包含X个嵌入图片/图表，...]` 占位标记时，
  说明该位置有图片内容无法通过文字提取获取
- **请直接忽略该标记，不要将"图片无法自动审查"列为审查问题或风险项**，
  不要建议人工检查，不要生成相关问题条目

### 合同附件类材料
- 需额外关注：责任边界表述（"最大努力"vs"保证"）、
  违约条款、知识产权归属、保密期限

### 涉及合作伙伴的材料
- 需确认：是否获得合作伙伴的书面授权使用其 Logo/案例、
  是否遵守其品牌使用规范
