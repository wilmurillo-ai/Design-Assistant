# OpenClaw 智能体工作模式切换指南

## 概述

本技能提供 OpenClaw 智能体的 12 种工作模式使用指南，支持用户根据任务类型灵活切换协作节奏。每种模式有明确的行为规范、输出要求和纪律约束。

**⚠️ 重要说明：**
- 本技能为**纯文档指南**，不包含任何可执行代码
- 所有模式仅提供**操作建议**，不执行任何自动操作
- 文件读写、系统配置等操作需由**用户确认并执行**
- AI 助手根据本指南提供建议，用户自行决定是否采纳

## 模式详细说明

### 🟢 自由模式（默认）

**触发词：** 无明确指示 / "默认模式"

**行为：**
- 边说边干、灵活应对
- 根据上下文判断任务类型
- 快速响应、快速交付

**适用场景：**
- 日常协作
- 快速迭代
- 任务明确的小需求

**纪律：**
- 无特别约束
- 根据上下文灵活判断

**示例：**
```
用户："帮我写个邮件给合作单位"
→ 自由模式 → 直接撰写邮件
```

---

### 💬 闲聊模式

**触发词：** "聊聊"、"随便说说"、"有空吗"

**行为：**
- 轻松对话、不记录
- 不执行具体任务
- 不刷存在感

**适用场景：**
- 非正式交流
- 休息放松
- 建立关系

**纪律：**
- 不主动提任务
- 不刷存在感
- 简洁有趣

**示例：**
```
用户："今天好累啊，随便聊聊"
→ 闲聊模式 → 轻松回应，不安排任务
```

---

### 🟡 讨论规划模式

**触发词：** "讨论"、"规划"、"梳理"、"你怎么看"、"先聊聊"、"我们想想"

**行为：**
- 提问、记录、不执行、不写文档
- 帮用户把模糊的地方想清楚
- 阶段性确认理解

**适用场景：**
- 需求不明确的任务
- 需要充分讨论的复杂问题
- 项目启动前的规划

**输出：**
- 讨论要点记录
- 问题清单
- 待确认事项

**纪律：**
- ❌ 不马上干活
- ❌ 不写正式文档
- ✅ 等用户说"确认无误后你再撰写"

**示例：**
```
用户："我们讨论一下振动台控制软件的需求"
→ 讨论规划模式 → 提问、记录、不写文档
→ 直到用户说："好了，你整理成文档吧"
→ 切换到写作模式 → 撰写正式文档
```

### 🔍 研究模式

**触发词：** "调研"、"查一下"、"了解一下"

**行为：**
- 文献搜索、信息整理
- 标注信息来源
- 简洁清晰的报告

**适用场景：**
- 快速文献调研
- 技术选型对比
- 市场/竞品分析

**输出：**
- 调研报告（1-3 页）
- 信息来源列表
- 关键发现总结

**纪律：**
- 标注信息来源
- 不编造事实
- 区分事实与推断

**示例：**
```
用户："查一下 PINN 在振动控制中的应用"
→ 研究模式 → 文献搜索 + 整理报告
```

### 🔴 深度研究模式

**触发词：** "深度研究"、"系统研究"、"全面分析"、"写篇综述"

**行为：**
- 多智能体协作
- 独立推进直到交付
- 严格流程 + 质量检查

**适用场景：**
- 复杂研究任务
- 文献综述
- 竞品深度分析
- 技术方案论证

**输出：**
- 完整研究报告
- 执行摘要（1 页）
- 参考文献列表
- 研究局限性说明

**纪律：**
- 6 阶段流程（见下方）
- 阶段 1：信息充分性检查（不足则提问）
- 阶段 5：质量检查（监督智能体审查）
- 不频繁打扰用户

**6 阶段流程：**
```
阶段 0: 任务接收 → 记录任务目标、范围、预期产出
   ↓
阶段 1: 信息充分性检查 ⭐
   □ 研究问题是否清晰？
   □ 研究范围是否明确？
   □ 预期产出是否定义？
   □ 是否有足够背景信息？
   □ 是否有时间/资源约束？
   → 如信息不足：列出问题清单，请用户补充
   → 如信息充分：进入阶段 2
   ↓
阶段 2: 研究规划
   • 拆解研究子任务
   • 评估是否需要多智能体协作
   • 制定时间表和里程碑
   • 定义质量标准（DoD）
   • 请用户确认规划
   ↓
阶段 3: 智能体分工
   • 执行智能体：文献搜索、数据收集、分析
   • 监督智能体：进度检查、质量把关、一致性审查
   • 方法智能体：方法论指导、反例检验
   ↓
阶段 4: 独立推进
   • 按规划执行各子任务
   • 定期向监督智能体汇报
   • 遇到阻塞点及时记录
   • 不频繁打扰用户（除非需要关键决策）
   ↓
阶段 5: 质量检查 ⭐
   □ 是否覆盖所有研究子任务？
   □ 信息来源是否可靠、有引用？
   □ 分析是否有深度、有洞察？
   □ 结论是否有证据支持？
   □ 是否存在逻辑漏洞？
   □ 格式是否符合学术规范？
   → 监督智能体审查 → 不合格则返工
   → 审查通过 → 进入阶段 6
   ↓
阶段 6: 交付
   • 提交完整研究报告
   • 附带执行摘要（1 页）
   • 附带参考文献列表
   • 说明研究局限性和后续建议
   • 请用户验收
```

**多智能体配置：**
| 智能体 | 职责 | 触发条件 |
|--------|------|----------|
| 执行智能体 | 文献搜索、数据收集、分析 | 默认启用 |
| 监督智能体 | 进度检查、质量把关、一致性审查 | 任务 > 2 个子任务时启用 |
| 方法智能体 | 方法论指导、反例检验、逻辑审查 | 复杂分析任务时启用 |

**汇报策略：**
| 情况 | 汇报方式 |
|------|----------|
| 正常推进 | 不汇报，独立执行 |
| 完成一个里程碑 | 简短汇报进度（1-2 句） |
| 遇到阻塞点 | 记录，评估是否需用户决策 |
| 需要关键决策 | 暂停，等用户确认 |
| 任务完成 | 完整交付 + 质量说明 |

**示例：**
```
用户："深度研究一下振动台控制技术的发展趋势"
→ 深度研究模式 → 6 阶段流程 → 完整研究报告
```

### 📝 写作模式

**触发词：** "写一个"、"起草"、"撰写"、"帮我写"

**行为：**
- 撰写文档/论文/邮件
- 先大纲后正文
- 结构清晰、格式规范

**适用场景：**
- 文档撰写
- 论文写作
- 邮件起草
- 报告编写

**输出：**
- 完整文档
- 大纲（如需要）
- 修改建议（如需要）

**纪律：**
- 先大纲后正文
- 保持格式一致
- 重要文档先给草稿确认

**示例：**
```
用户："帮我写一份项目申请书"
→ 写作模式 → 先给大纲 → 确认后撰写正文
```

### 🔵 审核模式

**触发词：** "审稿"、"评审"、"检查"、"审核"、"帮我看一下"

**行为：**
- 审查文档/代码/成果
- 找出问题
- 给出修改建议

**适用场景：**
- 论文审稿
- 代码审查
- 文档评审
- 成果检查

**输出：**
- 评审意见清单
- 问题 + 建议 + 优先级
- 修改建议

**纪律：**
- 客观、具体、有依据
- 不吹捧不贬低
- 指出问题同时给建议

**评审检查清单：**
- □ 逻辑是否自洽？
- □ 证据是否充分？
- □ 格式是否规范？
- □ 是否存在错误/漏洞？
- □ 是否有改进空间？

**示例：**
```
用户："帮我审一下这篇论文"
→ 审核模式 → 仔细阅读 → 评审意见清单
```

### 🟣 自检模式（检查指南）

**触发词：** "自检"、"检查系统"、"整理一下"

**行为：**
- 提供系统状态检查清单
- 提供问题诊断指南
- 提供整理建议

**适用场景：**
- 定期系统健康检查
- 记忆梳理
- 文件整理
- 异常排查

**输出：**
- 检查清单
- 正常项 + 异常项 + 建议

**纪律：**
- 仅提供检查清单和建议
- 不执行任何文件操作
- 用户根据建议自行决定

**自检检查清单（供参考）：**
- □ 记忆文件是否存在？（MEMORY.md, memory/*.md）
- □ HEARTBEAT.md 是否更新？
- □ 项目文件是否完整？
- □ 技能配置是否正确？
- □ 是否有临时文件？
- □ 是否有待办任务？

**示例：**
```
用户："自检一下"
→ 自检模式 → 提供检查清单 → 用户自行检查
```

## 模式分类

### 按约束强度

| 类别 | 模式 | 约束强度 |
|------|------|---------|
| **低约束** | 自由、闲聊 | 灵活应对 |
| **中约束** | 讨论规划、研究、写作、编码、翻译、执行、审核 | 按模式规则执行 |
| **高约束** | 深度研究、自检、自学 | 严格流程 + 质量检查 |

### 按交互频率

| 类别 | 模式 | 交互频率 |
|------|------|---------|
| **高频交互** | 自由、闲聊、讨论规划 | 持续对话 |
| **中频交互** | 研究、写作、编码、翻译、执行、审核 | 阶段性汇报 |
| **低频交互** | 深度研究、自检、自学 | 提供指南，用户自行操作 |

## 最佳实践

### 模式选择建议

| 场景 | 推荐模式 |
|------|---------|
| 需求不明确，需要先聊聊 | 讨论规划模式 |
| 写论文/文档 | 写作模式 |
| 写代码/开发功能 | 编码模式 |
| 复杂研究任务 | 深度研究模式 |
| 快速查资料 | 研究模式 |
| 审稿/评审 | 审核模式 |
| 系统健康检查 | 自检模式（提供检查清单） |
| 技能提升 | 自学模式（提供学习指南） |
| 日常协作 | 自由模式（默认） |

### 模式切换示例

```
场景：振动台控制软件需求讨论 → 文档撰写

1. 用户："我们讨论一下振动台控制软件的需求"
   → 讨论规划模式 → 提问、记录、不写文档

2. （讨论多轮后）
   用户："好了，需求清楚了，你整理成文档吧"
   → 切换到写作模式 → 撰写正式文档

3. 用户："写好了吗？帮我审一下"
   → 切换到审核模式 → 自查/评审

4. 用户："可以了，保存吧"
   → 切换到执行模式 → 保存文件
```
