### 2. 时间推理引擎

**解决问题**：时间表达式解析

```typescript
parseTimeQuery("下周有什么重要事项？")
// → { relativeTime: 'next_week', keywords: ['3月24日-3月30日', '重要'] }

resolveConditionalTime("如果明天是周三，我需要准备什么？")
// → { targetDay: '周三', keywords: ['周三', '准备'] }
```

**改进效果**：
- "如果明天是周三..." 从 20% → **命中**

## 各方案优缺点

### OpenClaw 原生 MEMORY.md

**优点：**
- ✅ 最高准确率（100%）
- ✅ 最强语义理解（LLM 能力）
- ✅ 无需额外配置

**缺点：**
- ❌ 受 LLM 上下文窗口限制（<200条）
- ❌ 存储耗时较长
- ❌ 无结构化管理
- ❌ 无回收站机制

**适用场景：** 小规模记忆（<100条），需要最高准确率

---

### Memory Palace（全优化）

**优点：**
- ✅ 与原生相同准确率（100%）
- ✅ 检索最快（19.5ms）
- ✅ 无限扩展
- ✅ 结构化存储
- ✅ 语义理解能力强
- ✅ 时间推理能力
- ✅ 概念关联能力
- ✅ 回收站机制

**缺点：**
- ⚠️ 首次加载模型较慢（~7s）
- ⚠️ 需要 ~200MB 内存
- ⚠️ 排序权重有优化空间

**适用场景：** 大规模记忆管理，需要语义检索和时间推理
