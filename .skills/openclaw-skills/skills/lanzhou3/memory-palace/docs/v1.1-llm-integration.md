## 概述

v1.1 版本的核心目标是将所有需要 LLM 能力的功能统一使用 **OpenClaw 原生 Subagent 方式** 实现，不引入额外的 LLM SDK 或 API 调用。

## 架构设计

### 核心原则

1. **使用 OpenClaw 原生能力** - 通过 `sessions_spawn` 或 `runEmbeddedPiAgent` 调用 LLM
2. **功能独立封装** - 每个 LLM 功能作为独立模块
3. **优雅降级** - LLM 不可用时回退到规则引擎
4. **异步优先** - 批量操作使用 Subagent 异步处理

### 调用方式

**统一使用 Subagent 方式**（`sessions_spawn`）

| 场景 | 调用方式 | 原因 | 超时 |
|------|----------|------|------|
| 单条记忆总结 | Subagent | 统一架构，独立会话上下文 | 30s |
| 批量经验提取 | Subagent | 异步处理，避免阻塞主会话 | 60s |
| **复杂时间解析** | Subagent | 精简 Prompt，快速响应 | **10s** |
| **概念动态扩展** | Subagent | 精简 Prompt，快速响应 | **15s** |
| 智能压缩 | Subagent | 批量处理，异步执行 | 60s |

**实测数据**（2026-03-18 验证）：
| 功能 | 实测耗时 | 目标 | 状态 |
|------|----------|------|------|
| 时间解析 | 7s | 10s | ✅ 达标 |
| 概念扩展 | 12s | 15s | ✅ 达标 |

**10-15s 快速任务优化策略**：
- 精简 Prompt，去掉冗余说明
- 只要求返回核心结果，不要推理过程
- 使用更短的示例
- 限制输出长度

**优势**：
- 架构统一，维护简单
- 每个 LLM 任务独立会话，互不干扰
- 可使用不同的模型配置
- 支持异步批量处理
- 失败可独立重试

## 功能模块

### 1. LLMSummarizer - 智能总结模块

**位置**: `src/llm/summarizer.ts`

**功能**:
- 总结单条记忆内容
- 提取关键信息点
- 评估记忆重要性
- 建议标签分类

**接口**:
```typescript
interface SummarizeResult {
  summary: string;           // 一句话总结
  keyPoints: string[];       // 关键要点
  importance: number;        // LLM 评估的重要性 (0-1)
  suggestedTags: string[];   // 建议的标签
  category: string;          // 分类（工作/生活/技术/等）
}

async function summarizeMemory(content: string): Promise<SummarizeResult>;
```

**Prompt 模板**:
```
请分析以下记忆内容，提取关键信息：

{content}

请以 JSON 格式返回：
{
  "summary": "一句话总结（不超过50字）",
  "keyPoints": ["要点1", "要点2", "要点3"],
  "importance": 0.8,
  "suggestedTags": ["标签1", "标签2"],
  "category": "分类"
}
```

### 2. ExperienceExtractor - 经验提取模块

**位置**: `src/llm/experience-extractor.ts`

**功能**:
- 从对话/长文本中提取可复用经验
- 识别教训和最佳实践
- 发现模式和规律

**接口**:
```typescript
interface ExtractedExperience {
  experience: string;        // 经验描述
  context: string;           // 适用场景
  lessons: string[];         // 教训
  bestPractices: string[];   // 最佳实践
  relatedTopics: string[];   // 相关主题
}

async function extractExperiences(
  memories: Memory[], 
  options?: { focusArea?: string }
): Promise<ExtractedExperience[]>;
```

**Prompt 模板**:
```
请从以下记忆中提取可复用的经验和教训：

{memories}

请识别：
1. 成功的经验（最佳实践）
2. 失败的教训
3. 可复用的模式
4. 适用场景

以 JSON 数组格式返回。
```

### 3. TimeParserLLM - 时间推理增强模块

**位置**: `src/llm/time-parser.ts`

**功能**:
- 解析复杂时间表达
- 处理模糊时间描述
- 理解时间条件句

**接口**:
```typescript
interface ParsedTime {
  resolvedDate: string;      // 解析后的日期 YYYY-MM-DD
  confidence: number;        // 置信度 0-1
}

async function parseComplexTime(expression: string): Promise<ParsedTime>;
```

**示例**:
- "下周三之前的那天" → 2026-03-25
- "三个月后的第一个周一" → 具体日期
- "如果明天是周五，我要准备什么" → 条件推理

**Prompt 模板** (精简版，目标 10s):
```
今日: {currentDate}
解析: {expression}
返回JSON: {"date":"YYYY-MM-DD","confidence":0.9}
仅返回JSON，无解释。
```

**优化要点**：
- 去掉冗余说明和推理过程
- 只返回核心结果（日期 + 置信度）
- 限制输出长度到 50 字符以内

**Prompt 模板** (精简版，目标 10s):

**优化要点**：
- 去掉意图分析等冗余字段
- 只返回关键词列表
- 限制关键词数量到 5-10 个
- 输出控制在 100 字符以内

## 技术实现参考

### OpenClaw Subagent 调用

**核心接口**: `sessions_spawn` 工具

所有 LLM 功能统一通过 Subagent 调用，在 SKILL.md 中定义工具时调用 `sessions_spawn`：

```typescript
// 在 SKILL.md 工具实现中调用 Subagent
async function summarizeMemory(content: string): Promise<SummarizeResult> {
  // 通过 sessions_spawn 创建总结专用 Subagent
  const result = await sessions_spawn({
    runtime: "subagent",
    mode: "run",  // 一次性任务
    task: `请分析以下记忆内容，提取关键信息并总结：

${content}

请以 JSON 格式返回：
{
  "summary": "一句话总结（不超过50字）",
  "keyPoints": ["要点1", "要点2", "要点3"],
  "importance": 0.8,
  "suggestedTags": ["标签1", "标签2"],
  "category": "分类"
}`,
    timeoutSeconds: 30,
  });
  
  return JSON.parse(result);
}
```

### Subagent 封装模式

```typescript
// src/llm/subagent-client.ts
interface SubagentOptions {

async function callSubagent(options: SubagentOptions): Promise<string> {
```

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| Subagent 调用超时 | 设置合理超时（默认 30s，批量 60s） |
| LLM 返回格式错误 | JSON 解析容错 + 重试机制 |
| 批量处理阻塞 | Subagent 天然异步，不阻塞主会话 |
| Token 消耗过大 | 控制输入长度，分段处理 |
| Subagent 失败 | 失败可独立重试，不影响主流程 |
| 并发限制 | 控制同时运行的 Subagent 数量 |

## 用户体验设计

### 操作反馈机制

每次记忆操作返回明确的结果状态：

```typescript
interface OperationResult {
  success: boolean;
  message: string;      // 简短描述
  data?: any;           // 返回数据
  duration?: number;    // 耗时（毫秒）
}

// 示例
{ success: true, message: "记忆已保存", data: { id: "xxx" }, duration: 150 }
{ success: false, message: "保存失败：存储空间不足" }
```

### 异步任务通知

长时间任务（压缩、经验提取）完成后主动通知用户：

```typescript
// 任务状态
interface TaskStatus {
  taskId: string;
  status: "running" | "completed" | "failed";
  progress?: number;    // 0-100
  result?: any;
}

// 完成后通知
"✅ 经验提取完成：从 15 条记忆中提取了 3 条经验"
```

### 错误处理流程

| 错误类型 | 处理方式 |
|----------|----------|
| Subagent 超时 | 返回超时错误，建议稍后重试 |
| JSON 解析失败 | 返回原始响应，标记格式错误 |
| LLM 返回空 | 返回空结果，提示"未找到相关信息" |
| 降级触发 | 静默降级到规则引擎，记录日志 |

### 记忆纠错机制

提供 `memory_palace_correct` 工具让用户修正 AI 记忆错误：

```typescript
// 参数
{
  id: "memory-id",
  correction: "正确的信息",
  reason: "原记忆有误"
}

// 返回
{ success: true, message: "记忆已更正" }
```

### 操作预览

在压缩/总结前让用户确认是否执行：

```typescript
// 返回预览信息
{
  affectedMemories: 15,
  estimatedSaving: "80%",
  preview: "压缩后的内容预览..."
}

// 用户确认后执行
```
