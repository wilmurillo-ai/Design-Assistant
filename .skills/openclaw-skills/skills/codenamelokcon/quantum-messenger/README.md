通过此技能，您可以将量子密信作为 AI 机器人的前端界面，实现文本对话、图片生成、文件传输等多种交互功能，并能在回复中实时查看到 AI 模型的运行状态（CTX、时间等）。

## 🏗️ 逻辑原理
1. **监听层**: 使用 Node.js 启动一个 HTTP Server (`listener.mjs`)，监听来自量子密信 Webhook 的 POST 请求。
2. **大脑层**: 接收到消息后，通过 OpenClaw CLI 指令将内容传递给指定的 AI Agent。
3. **分发层**:
    - **文本**: AI 返回的文本（含系统状态栏）通过量子密信回调接口原样返回。
    - **多模态**: 自动识别 AI 回复中的文件路径（IMAGE:/FILE:），先上传至量子密信存储，再将 fileId 以媒体消息形式推送给用户。

## 🚀 部署指南

### 1. 准备工作
确保您的服务器已安装 **Node.js 18+** 和 **OpenClaw**。

### 2. 创建量子密信机器人
1. 在量子密信管理后台（或 APP 内）创建机器人。
2. **重要**: 必须选择 **“自定义会话机器人”**。
3. 获取机器人的 **KEY**。

### 3. 设置机器人 URL
在量子密信 APP 或管理后台设置机器人的“消息回调地址”：
- **格式**: `http://<您的服务器公网IP>:<端口号>`
- **示例**: `http://1.2.3.4:9001`
- **注意**: 请确保该端口号已在您服务器的 **安全组/防火墙** 中放通。

### 4. 修改配置
编辑 `scripts/listener.mjs` 和 `scripts/send.mjs`，将 `KEY` 替换为您获取到的真实 KEY。

### 5. 启动服务
在技能目录下运行：
```bash
nohup node scripts/listener.mjs > listener.log 2>&1 &
```

## 🛠️ 功能特性
- **文本交互**: 支持直接对话。
- **状态感知**: 自动引用 OpenClaw 状态栏，显示当前模型、上下文 (CTX) 使用率及 UTC 时间。
- **多模态传输**: 
    - 脚本中已预留图片 (`isImage=true`) 和 附件 (`isImage=false`) 的处理逻辑。
    - AI 只要在回复中输出 `IMAGE:/path/to/file` 或 `FILE:/path/to/file`，即可自动触发媒体发送。
