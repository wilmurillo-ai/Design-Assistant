# MCP 服务器连接指南

## 概述

RedQuant 提供托管式 MCP 服务，当前对外开放 **46 个只读金融分析工具**，覆盖 A 股行情、财务、行业、新闻、策略概览、量化选股与 LLM 优化能力。

当前统一使用 **`streamable-http`** 传输协议。

> 公开 Skill 包仅保留连接原则与权限边界；具体托管地址由部署维护方提供，不在本目录中内嵌固定访问地址。

## 连接原则

- 使用 RedQuant 团队提供的**远程托管 MCP 地址**
- `transport` 固定为 `streamable-http`
- 默认走托管服务，不要求用户自建本地进程
- 若平台已有预置接入信息，以平台侧信息为准

## OpenClaw / Claude Desktop 接入示例

将下列占位符替换为团队提供的托管地址：

```json
{
  "mcpServers": {
    "redquant": {
      "url": "<REDQUANT_HOSTED_MCP_ENDPOINT>",
      "transport": "streamable-http"
    }
  }
}
```

## 审查与权限说明

- 默认仅连接 RedQuant 托管 MCP 服务
- 所有 MCP 工具均为**只读查询**
- 不进行买卖、资金变更、写入型操作或环境级指令执行
- 公开信息补充能力仅在需要补充背景资料时使用
- 面向普通用户仅开放安全可展示的策略说明能力

## 连接失败时的排查顺序

1. 确认 `transport` 已设置为 `streamable-http`
2. 确认使用的是团队提供的托管地址，而非临时示例值
3. 确认当前网络环境允许访问外部托管服务
4. 若平台侧出现限流或暂时不可用，稍后重试

## 客户端建议

- 建议超时设置：60-120 秒
- 批量财务、历史数据等复杂查询可能需要更长时间
- 回答中应明确标注风险提示，不将结果表述为收益承诺

## 常见问题

**Q: 连接超时怎么办？**  
A: 先确认 `transport` 是否为 `streamable-http`，再检查托管地址与当前网络环境是否一致。

**Q: 工具调用报错“未找到数据”？**  
A: 检查股票代码格式（需带 `.SH` / `.SZ` 后缀），并确认日期范围合理。

**Q: 为什么公开 Skill 包里没有固定地址？**  
A: 为保持公开包简洁与审查友好，固定托管地址由部署维护方在环境侧统一提供。
