# 招投标专项分析

本文档专门针对招投标场景，提供从信息收集到投标策略的全流程指导。

---

## 一、招投标信息收集

### 1.1 信息来源

| 渠道 | 网址 | 特点 |
|------|------|------|
| 中国招标投标网 | https://www.cec.org.cn | 官方权威 |
| 中国政府采购网 | http://www.ccgp.gov.cn | 政府采购 |
| 省级政府采购网 | 各省财政厅网站 | 地方项目 |
| 公共资源交易中心 | 各地市交易中心 | 建设工程 |
| 招标雷达/乙方宝 | 商业平台 | 数据全但付费 |
| 企业官网 | 自行发布 | 真实性强 |

### 1.2 关键信息提取

收集招标信息时，必须提取以下内容：

```
□ 项目名称
□ 招标编号
□ 采购单位
□ 预算金额
□ 报名截止时间
□ 投标截止时间
□ 开标时间
□ 资质要求
□ 评分标准（核心！）
□ 采购需求/技术参数
□ 联系人/联系方式
```

### 1.3 信息时效性

- **黄金72小时**：招标信息发布后72小时内是收集关键期
- **追踪变更**：关注澄清、变更公告
- **历史数据**：查询该单位历史中标记录，分析竞争格局

## 二、评分标准分析（核心！）

### 2.1 常见评分办法

| 办法 | 特点 | 应对策略 |
|------|------|---------|
| 综合评分法 | 技术+价格综合打分 | 技术分争取高分 |
| 最低价中标 | 价格最低者得 | 成本核算精准 |
| 合理低价 | 去掉最高最低价平均 | 报价技巧 |
| 询价采购 | 价低者得 | 价格战 |

### 2.2 综合评分法细分

**常见权重**：
- 技术/服务方案：40-60%
- 价格：30-50%
- 商务（资质、业绩）：10-20%

**评分细则分析**：
```
重点分析：
1. 哪些分数是"送的"（固定分）
2. 哪些分数可以争取
3. 竞争焦点在哪里
4. 我方优劣势
```

### 2.3 评分标准拆解

| 评分项 | 满分 | 我方得分预估 | 差距 | 弥补措施 |
|--------|------|-------------|------|---------|
| 技术方案 | 30 | 25 | -5 | 方案优化 |
| 项目团队 | 20 | 18 | -2 | 人员提升 |
| 售后服务 | 15 | 15 | 0 | 保持 |
| 业绩案例 | 15 | 10 | -5 | 找联合体 |
| 价格 | 20 | 18 | -2 | 报价策略 |
| **合计** | **100** | **86** | **-14** | |

## 三、投标策略制定

### 3.1 竞争态势分析

**SWOT分析框架**：

| | 我方优势 | 我方劣势 |
|--|---------|---------|
| **机会** | SO策略：放大优势抓机会 | WO策略：补短板抓机会 |
| **威胁** | ST策略：优势规避威胁 | WT策略：收缩防御 |

**竞争格局判断**：
- [ ] 有没有围标（陪标）现象
- [ ] 核心竞争对手有几家
- [ ] 各自的优劣势
- [ ] 业主倾向谁

### 3.2 投标决策模型

```
决策公式：
预期收益 = 中标概率 × 利润 - 投标成本

决策阈值：
- 预期收益 > 投标成本 → 参与
- 预期收益 < 投标成本 → 放弃
- 战略意义项目 → 可以牺牲短期利润
```

**需要考虑的因素**：
- [ ] 战略价值（标杆意义）
- [ ] 长期合作可能
- [ ] 现金流情况
- [ ] 我方产能/人员
- [ ] 竞争激烈程度

### 3.3 报价策略

| 策略 | 适用场景 | 具体做法 |
|------|---------|---------|
| 低价冲标 | 首次进入、战略布局 | 保本或微利 |
| 正常报价 | 正常竞争 | 市场合理价 |
| 高价保利润 | 技术领先、独家优势 | 报出价值 |
| 差异报价 | 方案有亮点 | 技术分高 |

**报价技巧**：
- 调研历史中标价
- 分析竞争对手报价习惯
- 把握业主预算心理
- 注意不平衡报价

## 四、技术方案撰写

### 4.1 方案结构

```
技术方案框架：
1. 项目理解（得分点：体现理解深度）
2. 总体设计思路
3. 技术架构
4. 功能模块详述
5. 项目实施计划
6. 质量保障措施
7. 售后服务方案
8. 项目团队介绍
9. 成功案例
10. 差异化优势
```

### 4.2 评分要点把握

**技术方案得分技巧**：

| 评分点 | 得分要点 | 写作技巧 |
|--------|---------|---------|
| 项目理解 | 体现深度 | 用客户原话、行业术语 |
| 技术架构 | 先进性 | 采用主流技术、国产化 |
| 功能满足 | 全覆盖 | 逐条响应需求 |
| 实施计划 | 可落地 | 时间合理、节点清晰 |
| 售后服务 | 有保障 | 响应快、承诺实 |

### 4.3 加分项

常见加分项（投标时务必争取）：
- [ ] 自主知识产权
- [ ] ISO认证
- [ ] 高新企业认证
- [ ] 行业奖项
- [ ] 专利技术
- [ ] 产学研合作
- [ ] 信创适配证明
- [ ] 当地服务能力

## 五、投标全流程

### 5.1 时间节点把控

| 阶段 | 时间节点 | 关键动作 |
|------|---------|---------|
| 信息收集 | T-30天 | 发现项目、初步分析 |
| 立项决策 | T-20天 | 是否参与、团队组建 |
| 现场考察 | T-15天 | 踏勘现场、见业主 |
| 文件购买 | T-10天 | 获取招标文件 |
| 方案编写 | T-7天 | 技术/商务文件 |
| 内部评审 | T-3天 | 方案审核、成本核算 |
| 封标提交 | T-1天 | 打印装订、签字盖章 |
| 开标 | T日 | 述标答辩（如有） |
| 中标公示 | T+3天 | 等待结果 |
| 中标通知书 | T+7天 | 确认中标 |

### 5.2 常见废标点

**商务类**：
- [ ] 资质证书过期
- [ ] 授权委托书缺签字盖章
- [ ] 保证金未按时缴纳
- [ ] 业绩证明材料不全
- [ ] 投标报价超过预算

**技术类**：
- [ ] 方案严重偏离需求
- [ ] 缺项漏项
- [ ] 技术参数不满足

**格式类**：
- [ ] 密封不符合要求
- [ ] 份数不够
- [ ] 签章不全

## 六、述标与答辩

### 6.1 述标准备

**述标内容框架**（10分钟为例）：
```
1. 公司介绍（1分钟）
2. 项目理解（2分钟）—— 体现深度
3. 解决方案（4分钟）—— 核心亮点
4. 实施保障（2分钟）—— 让业主放心
5. 案例分享（1分钟）—— 增加信任
```

### 6.2 常见答辩问题

| 问题类型 | 示例问题 | 回答要点 |
|---------|---------|---------|
| 技术类 | "你们怎么实现这个功能" | 演示+案例 |
| 实施类 | "人员能到位吗" | 团队实力+承诺 |
| 售后类 | "出了问题怎么解决" | 响应时间+流程 |
| 商务类 | "价格还能降吗" | 价值导向、套餐优惠 |
| 质疑类 | "为什么要选你们" | 差异化+成功率 |

### 6.3 述标技巧

- **前3分钟定胜负**：第一印象很重要
- **讲业主关心的**：少讲技术，多讲价值
- **用数据说话**：成功率、客户评价
- **差异化突出**：与竞争对手的区别
- **自信从容**：眼神交流、语速适中

## 七、中标后工作

### 7.1 合同签订注意事项

**必争条款**：
- [ ] 付款节点（预付款、进度款、验收款、质保金）
- [ ] 验收标准（避免扯皮）
- [ ] 实施周期（预留缓冲）
- [ ] 售后服务（响应时间、范围）
- [ ] 违约责任（对等原则）

**风险条款**：
- 无限责任→有限责任
- 高额罚款→合理赔偿
- 固定价格→调价机制

### 7.2 项目启动

- [ ] 组建项目团队
- [ ] 召开启动会
- [ ] 制定项目计划
- [ ] 需求调研启动
- [ ] 商务转交实施

## 八、招投标话术

### 8.1 客户拜访话术

**初次接触**：
> "王总您好，我们是专注企业信息化的服务商，之前服务过XX行业XX客户，他们通过我们的系统实现了XX效率提升。今天来是想了解一下贵公司目前的数字化建设情况，看看有没有能帮上忙的地方。"

**了解需求**：
> "我们注意到贵司最近发布了XX项目的招标公告，想了解一下这个项目主要是解决哪些业务问题？我们在这方面有一些经验，可以提供一些参考建议。"

**邀约考察**：
> "方便的话，欢迎您到我们实施的XX项目现场实地考察，了解一下实际使用效果。"

### 8.2 异议处理

**价格太高**：
> "X总，价格确实是我们价值的体现。我们提供的不仅是软件，更是成功率。您看，我们同类项目的成功率是XX%，这能为贵公司节省大量的试错成本。"

**已有供应商**：
> "我们理解贵司已有合作方。如果贵司后续有新的需求或者想对比了解，我们可以提供免费的技术交流，不一定马上合作，但多一个参考总是好的。"

**决策周期长**：
> "理解，信息化项目确实需要慎重。我们建议可以先做一个小的试点，成功后再推广，这样风险可控。"

## 十、案例：招投标分析示例

### 案例：某省政务云项目

**项目概况**：
- 预算：3000万
- 周期：18个月
- 评分：技术60%，价格30%，商务10%

**竞争分析**：
| 竞争对手 | 技术实力 | 价格策略 | 优势 |
|---------|---------|---------|------|
| 用友 | 强 | 中 | 本地化服务 |
| 华为 | 强 | 高 | 品牌 |
| 阿里云 | 中 | 中 | 云计算 |
| 本地小企业 | 弱 | 低 | 关系 |

**我方策略**：
1. **技术方案**：突出政务行业经验、信创适配
2. **报价策略**：报出中等偏上价格，保证利润
3. **商务加分**：找当地合作伙伴联合投标
4. **差异化**：专注财政、人社业务场景

**结果**：成功中标，综合得分第一
