# 客户分析后自动生成推进策略（智能匹配引擎）

## 使用说明

**完成客户分析后，自动调用本模块，输出：**
1. 用友卖点匹配建议（针对该客户的定制化卖点）
2. 产品推荐（最适合的用友产品）
3. 推进策略（下一步行动建议）
4. 竞争风险预警（可能遇到的竞争对手及应对）

## 智能匹配规则

### 第一步：根据规模确定产品

```
营收 > 50亿 OR 员工 > 5000 OR 多法人/集团型
  → 推荐：用友BIP（旗舰版/超级版）
  → 定价参考：数百万~数千万级项目

营收 5-50亿 AND（制造 OR 商贸 OR 流通）
  → 推荐：用友U9C / YonSuite
  → 定价参考：几十万~数百万

营收 5-50亿 AND 服务型企业
  → 推荐：用友YonSuite
  → 定价参考：几十万~数百万

营收 1-5亿
  → 推荐：用友T+Cloud
  → 定价参考：几万~几十万

政府/事业单位
  → 推荐：用友GRP
  → 走政府采购流程
```

## 推进策略模板库

### 场景A：初次接触，客户有意向

```
第1步（当天）：发送定制化邮件 + 行业案例
第2步（2天内）：微信跟进，确认是否收到
第3步（1周内）：约15分钟电话，了解核心需求
第4步（2周内）：约现场拜访 或 产品演示
第5步（1个月内）：提交解决方案报告
```

### 场景B：客户说"在跟金蝶谈"

```
立即行动：
1. 不要慌，保持自信
2. 提出对比评测建议：
   "我建议您在决定前让我们也做一次演示，
   就2小时，我带[行业]案例来，
   您对比看看哪家更符合您的需求"
3. 准备一份"用友 vs 金蝶"对比表（针对该客户场景定制）
4. 找到客户内部支持者，协助推进
```

### 场景C：央企/国企客户

```
核心策略：信创为切入点，配合政治正确

话术：
"您这边有没有IT信创改造的时间节点要求？
现在很多央企已经开始这个工作了，
用友是信创认证覆盖最全的ERP厂商，
招商局集团和中国人寿都是我们帮助完成改造的。"

关键动作：
- 提供信创认证证书
- 提供同类央企案例
- 走正式政府采购/招投标流程
- 联系内部IT部门+主管领导
```

### 场景D：正在POC/评测阶段

```
策略：让POC成为你的优势展示舞台

行动：
1. 选择最有差异化的场景做演示
2. 提前了解评测标准，针对性准备
3. 安排技术专家全程陪同
4. POC结束后当天跟进，收集反馈
5. 快速响应问题，展示服务能力

关键话术：
"POC期间遇到任何问题，我的技术顾问24小时响应，
这也是我们将来实施服务能力的预演。"
```

## 销售漏斗管理建议

### 对每个客户的跟进频率

| 销售阶段 | 判断标准 | 跟进频率 | 优先级 |
|---------|---------|---------|-------|
| 热线索 | 有明确预算+时间节点+决策人联系 | 每3天 | P0 |
| 温线索 | 有需求但预算/节点不明 | 每1-2周 | P1 |
| 冷线索 | 有意向但无行动 | 每月1次 | P2 |
| 沉睡线索 | 超过3个月无回应 | 季度激活 | P3 |

### 赢单关键检查清单

在提交报价前，确认：
- [ ] 是否见过CEO/总经理？（大项目必须）
- [ ] 是否见过CFO？（财务系统必须）
- [ ] 是否有内部"Champion"（支持者）？
- [ ] 是否了解竞争对手现状？
- [ ] 是否有试用/POC计划？
- [ ] 是否确认预算范围？
- [ ] 是否有明确的决策时间节点？

**7项均确认 = 可以提交正式报价**
**少于5项 = 需要补充情报再推进**
