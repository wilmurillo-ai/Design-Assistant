# 拜访记录分析指南

## 功能概述

智能解析用户的拜访记录（文字/语音），自动提取：
- 关键信息点
- 客户动态变化
- 商机进展更新
- 下一步行动项

---

## 支持的输入形式

### 1. 文字记录
```
客户：张三集团
今天拜访了李总，整体沟通顺畅，他们计划Q2上新的ERP系统，预算大概500万左右...
```

### 2. 语音记录
用户发送语音，AI转写后分析

### 3. 关键事件提取
用户描述："今天见了XX客户，他说要考虑考虑"
→ 自动识别：商机可能受阻，需要更多支撑

---

## 信息抽取框架

### 必须提取的信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 客户名称 | 企业全称 | 北京张三科技集团 |
| 拜访时间 | 日期时间 | 2024-03-15 14:00 |
| 拜访人 | 客户方联系人 | 李总（采购总监） |
| 参会人 | 我方人员 | 销售小王 |
| 拜访类型 | 初次/跟进/演示/商务 | 产品演示 |
| 客户态度 | 积极/中性/消极 | 积极 |

---

### 可选提取的信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 预算信息 | 金额/预算来源 | 500万，自筹 |
| 需求描述 | 客户想要什么 | 集团ERP升级 |
| 痛点问题 | 客户面临的挑战 | 现有系统不稳定 |
| 决策时间 | 计划何时决策 | Q2结束前 |
| 竞争对手 | 客户还在谈谁 | 金蝶在接触 |
| 承诺事项 | 双方承诺的事项 | 周三发方案 |
| 阻碍因素 | 遇到的障碍 | 预算需审批 |
| 客户评价 | 对我方产品的反馈 | 功能齐全，界面一般 |

---

## 商机阶段更新

### 阶段定义（销售漏斗）

| 阶段 | 定义 | 判断依据 |
|------|------|---------|
| 线索 | 初步接触 | 刚获取联系方式 |
| 需求确认 | 明确需求 | 客户说明白要什么 |
| 方案制定 | 准备方案 | 正在准备或已提交 |
| 商务谈判 | 谈价格/合同 | 进入价格协商 |
| 签约 | 合同签订 | 已签合同 |
| 赢单/输单 | 结果 | 明确结果 |

### 自动更新逻辑

```
初次拜访成功 → 线索 → 需求确认
提交方案后 → 方案制定
进入价格讨论 → 商务谈判
收到合同 → 签约
```

## 触发方式

- "分析一下拜访记录"
- "帮我整理一下今天的拜访"
- "这个客户什么情况"
- "更新一下客户状态"
- "提取一下关键信息"
- "（直接发送语音或文字记录）"

---

## 自动触发的后续动作

1. **更新客户画像** - 将新信息补充到客户档案
2. **更新商机状态** - 自动推进漏斗阶段
3. **生成下次跟进任务** - 自动创建待办事项
4. **提醒相关人员** - 如有协同事项，通知相关同事

## 注意事项

1. **隐私保护** - 记录中的敏感信息仅用于销售跟进
2. **信息核实** - 关键信息需用户确认后再更新
3. **持续学习** - 根据用户反馈优化提取准确率
