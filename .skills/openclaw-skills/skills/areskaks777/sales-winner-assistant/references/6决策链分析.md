## 决策模式深度识别

### 模式一：一把手工程（最快成交）

```
特征信号：
- CEO亲自参与调研会
- 招投标文件由CEO/董事长授权签字
- 公司战略规划中明确提到数字化
- 上过媒体/行业论坛谈数字化转型

攻关策略：
✓ 约高层拜访，突出战略价值而非功能
✓ 提供同规模竞争对手的成功案例
✓ 给CEO一个"拿得出手"的决策理由（行业第一、国产替代、AI赋能）
✓ 邀请CEO参加用友高峰论坛/行业大会
```

### 模式二：CFO把关（财务主导）

```
特征信号：
- 采购流程中财务部门频繁审核
- 对方不断追问TCO（总拥有成本）
- 要求ROI测算和投资回收期
- CFO参加了多轮技术评审

攻关策略：
✓ 准备详细ROI测算模型（效率提升×人效×工资=节省成本）
✓ 分期付款方案降低一次性投入
✓ 提供同行案例的财务回报数据
✓ 强调合规价值（降低审计风险、财税合规）

CFO关心的核心数字：
- 系统上线后可减少多少财务人员？
- 月结时间从多少天缩短到多少天？
- 报表误差率降低多少？
```

### 模式三：委员会制（多方博弈）

```
特征信号：
- 采购流程需要多个部门签字
- 技术评审由多部门参与
- 招标文件评分项目繁多

攻关策略：
✓ 分别约谈每个关键角色，针对性诉求
✓ 找到内部"Champion"（支持者）作为线人
✓ 投标文件全方位覆盖各部门关注点
✓ 组织调研参观（让决策委员会成员都看到效果）
```

### 模式四：CIO/IT主导（技术优先）

```
特征信号：
- CIO技术背景强，自主权大
- 前期大量技术评测和POC
- 对接口、架构、安全要求细

攻关策略：
✓ 深度技术交流，用技术语言沟通
✓ 安排架构师/技术专家对接
✓ POC项目快速出成果
✓ 强调开放API、集成能力、二次开发
```

## 关键人信息采集

### 采集渠道（实时）

```
搜索指令（触发 web_search）：
1. "[企业名] CIO 信息化 总监" → 信息化负责人
2. "[企业名] CFO 财务总监" → 财务决策人
3. "[企业名] 高管 管理团队" → 决策层全貌
4. site:linkedin.com "[企业名] IT director CIO" → 领英信息
5. "[企业名] 招聘 CIO 副总裁 VP" → 关键人变动信号
```

### 关键人变动价值

| 变动类型 | 销售机会 | 行动速度 |
|---------|---------|---------|
| 新CIO上任 | 高（倾向树立成绩）| 立即行动 |
| 新CEO/总经理 | 极高（战略重置）| 立即行动 |
| CFO换人 | 高（财务系统评估）| 快速跟进 |
| 公司上市前 | 高（财务规范化需求）| 重点关注 |
| 并购重组 | 极高（系统整合需求）| 立即行动 |
