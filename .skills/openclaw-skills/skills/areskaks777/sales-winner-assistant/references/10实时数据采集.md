# 实时数据采集指引（C类升级）

## 快速采集框架

当用户要求分析某家企业时，按以下顺序触发搜索：

---

## 第一步：企业基本面（必查）

```
触发 web_search（同时搜索）：

1. "[企业名] 简介 主营业务 成立时间"
2. "[企业名] 营收 利润 2024 年报"
3. "[企业名] 员工规模 人数"
4. "[企业名] 新闻 动态 2025"
```

---

## 第二步：IT/数字化现状（必查）

```
触发 web_search：

1. "[企业名] 招聘 IT CIO 信息化 数字化"
   → 判断IT团队规模和技术方向

2. "[企业名] 信息化 数字化 系统 ERP"
   → 了解数字化程度和现有系统

3. "[企业名] 用友 OR 金蝶 OR SAP OR Oracle"
   → 排查现有ERP系统

4. "[企业名] 大模型 AI 人工智能 2024 2025"
   → 判断AI投入意愿
```

---

## 第三步：招投标记录（重要）

```
触发 web_search：

1. "[企业名] 中标 软件 系统 信息化"
   → 近期IT采购记录

2. "[企业名] 招标 采购 ERP CRM OA"
   → 主动采购需求

3. site:zbtb.gov.cn OR site:ccgp.gov.cn "[企业名] IT"
   → 政府采购平台记录（国企/央企适用）
```

---

## 第四步：决策人信息（按需查）

```
触发 web_search：

1. "[企业名] CIO 信息总监 IT负责人"
2. "[企业名] CFO 财务总监"
3. "[企业名] CEO 总经理 总裁 管理团队"
4. "[企业名] 高管 变动 任命 2024 2025"
   → 关注关键人变动（新官上任是极好机会）
```

---

## 第五步：竞争态势（按需查）

```
触发 web_search：

1. "[行业] 龙头企业 市占率 排名 2024"
2. "[企业名] 竞争对手 市场份额"
3. "[企业名] 合作伙伴 战略合作"
```

## 招投标专项采集

### 核心信息网站

| 网站 | 类型 | 适用场景 |
|------|------|---------|
| 中国政府采购网 (ccgp.gov.cn) | 政府采购 | 国企/政府单位 |
| 中国招投标公共服务平台 | 工程+服务 | 基建/工程类 |
| 企查查 / 天眼查 | 综合工商 | 所有企业 |
| 招标云 / 采采网 | 企业招标 | 大型民营企业 |

### 招标信息解读

```
当搜索到招投标记录时，重点提取：

✓ 采购时间 → 判断系统上线时间（使用年限）
✓ 采购金额 → 判断IT预算规模
✓ 中标供应商 → 确认现有系统厂商
✓ 项目名称关键词 → 判断具体业务模块
✓ 联系人/签字人 → 找决策人线索

信号解读：
- 3年内有IT采购 → 近期不太可能更换，但可做扩展
- 5年以上无IT采购 → 系统老化，替换机会大
- 正在招标 → 立即联系！黄金窗口期
```

## 采集结果评级

采集完成后，对企业打分（影响跟进优先级）：

| 维度 | 1分 | 3分 | 5分 |
|------|-----|-----|-----|
| 规模 | <1亿 | 5-50亿 | >50亿 |
| 数字化成熟度 | 初级 | 成长 | 成熟/引领 |
| IT预算信号 | 无记录 | 有采购记录 | 正在招标 |
| 决策人信号 | 联系不上 | 有线索 | 直接接触 |
| 替换机会 | 低 | 中 | 高 |

**总分 ≥ 15分 → 重点跟进**
**总分 10-14分 → 正常跟进**
**总分 < 10分 → 长线维护**
