# 企业核心业务分析（增强版）

## 分析框架

### 基础信息

| 字段 | 说明 |
|------|------|
| 企业全称 | 工商注册名称 |
| 成立时间 | 工商注册日期 |
| 注册资本 | 万元 |
| 总部地址 | 实际办公地址 |
| 上市情况 | 主板/创业板/科创板/港股/未上市 |
| 员工规模 | 总人数及分布 |
| 营收规模 | 最近一年营收（亿元） |
| 数字化评级 | 初级/成长/成熟/引领（按数字化程度） |

### 主营业务

```
1. 产品/服务描述
   - 核心产品是什么（按收入贡献排序）
   - 主要服务模式（SaaS/订阅/一次性）
   - 是否涉及政府业务/军工/国企特殊属性
   
2. 目标市场
   - 客户群体（B2B/B2C/B2G）
   - 地域范围（区域/全国/出海）
   - 客户规模分布（中小企/大B/集团）
   
3. 商业模式
   - 收入来源（产品/服务/订阅/增值）
   - 定价模式（按模块/按用户/按交易量）
   - 合作生态（代理商/集成商/自营）
```

### 业务亮点（重点挖掘）

| 亮点类型 | 描述 | 对数字化需求的影响 |
|---------|------|----------------|
| 技术优势 | 核心技术、专利数量 | 研发管理需求高 |
| 市场优势 | 市场份额、渠道宽度 | 销售管理/CRM需求高 |
| 品牌优势 | 行业地位、上下游话语权 | 供应链/SCM需求高 |
| 生产优势 | 产能、智能制造程度 | MES/智能工厂需求高 |
| 国际化 | 海外业务比例 | 多币种/跨境财务需求 |

## 行业专项分析

### 制造业（重点客群）

```
关注维度：
- 生产模式：离散/流程/混合
- 制造类型：按订单/按库存/按工程
- 智能制造程度：自动化程度、MES是否部署
- 供应链复杂度：供应商数量、多级BOM
- 质量管理：是否ISO认证、是否需要追溯
- 出口比例：是否需要跨境合规

数字化痛点：
✓ 计划与执行脱节（ERP计划≠车间实际）
✓ 质量追溯困难（问题定位慢）
✓ 供应商协同弱（采购效率低）
✓ 财务成本核算不准（工时/物料核算复杂）
```

### 零售/连锁（高频客群）

```
关注维度：
- 门店数量及分布
- 线上线下比例（O2O程度）
- 会员数量和运营质量
- 库存周转率
- 供应商数量

数字化痛点：
✓ 多渠道库存协同（超卖/积压问题）
✓ 门店数据孤岛（总部看不清门店）
✓ 会员精细化运营缺失
✓ 财务合并报表复杂（多法人/多业态）
```

### 医疗/医药（专业客群）

```
关注维度：
- 机构类型：医院/连锁诊所/医药制造/医疗器械
- 是否涉及医保结算
- DRG/DIP支付方式改革影响
- 是否需要合规（GSP/GMP/等保）

数字化痛点：
✓ 药品追溯（一物一码监管要求）
✓ 医保合规管理（控费压力大）
✓ 院内业财融合（科室成本核算）
✓ 多院区统一管控
```

### 数字化转型成熟度评估

| 评级 | 特征 | 典型场景 |
|------|------|---------|
| 引领级 | 数字原生、AI驱动 | 拓展AIAgent/智能决策 |
| 成熟级 | 核心系统已上线 | 系统集成/数据中台升级 |
| 成长级 | 部分系统上线 | 补齐短板/扩大覆盖 |
| 初级 | 主要靠Excel | 基础ERP/财务系统首购 |
