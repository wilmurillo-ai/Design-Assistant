# Step 3: GESP 知识点等级判定

## 🎯 目标

判定题目涉及的知识点等级（GESP 1-8级）和题目难度等级。

---

## 📊 知识点等级判定总览表

| 级别 | 算法要求 | 数据结构要求 | 典型题目特征 |
|------|---------|-------------|-------------|
| **一级** | 无特定算法 | 基本数据类型 | 顺序/分支/循环结构、四则运算、简单输入输出 |
| **二级** | 无特定算法 | 无新增 | 多层嵌套、数学函数(abs/sqrt/max/min)、ASCII编码 |
| **三级** | 枚举法、模拟法 | 一维数组、字符串 | 进制转换、位运算、数组遍历、字符串操作 |
| **四级** | 递推算法、排序 | 二维数组、结构体、指针 | 函数定义调用、二维数组操作、排序实现 |
| **五级** | 二分查找/答案、递归、贪心、分治 | 链表 | 高精度运算、数论(GCD/LCM/素数) |
| **六级** | DFS、BFS、简单DP | 树、栈、队列 | 树遍历、搜索、01背包 |
| **七级** | 复杂DP(LIS/LCS)、图遍历 | 图、哈希表 | 二维DP、状态压缩、flood fill |
| **八级** | 最小生成树、最短路径 | 无新增 | Kruskal/Prim、Dijkstra/Floyd |

---

## 🔧 判定规则

### 规则1：取知识点最高等级
题目涉及多个等级的知识点 → 取最高等级

### 规则2：算法优先
以算法复杂度为主要判定依据，数据结构辅助

### 规则3：知识点等级 ≠ 题目难度
- 知识点在高级别，但题目简单 → 题面备注实际难度较低
- 知识点在低级别，但题目难 → 题面备注实际难度较高

---

## 📊 难度等级判定

| 难度 | 特征 |
|------|------|
| 入门 | 直接应用知识点，无思维难度 |
| 普及- | 简单变形，需要一定思考 |
| 普及/提高- | 知识点组合或中等思维量 |
| 提高/省选- | 复杂问题，需要深度分析 |
| 省选/NOI | 高难度，需要创新解法 |

---

## 🔧 快速判定流程

```
1. 分析题目类型
   ↓
2. 识别数据结构
   - 基本类型 → 一二级
   - 数组/字符串 → 三级起
   - 二维数组/结构体 → 四级起
   - 链表 → 五级起
   - 树/栈/队列 → 六级起
   - 图/哈希 → 七级起
   ↓
3. 识别算法
   - 无特定算法 → 一二级
   - 枚举/模拟 → 三级
   - 排序/递推 → 四级
   - 二分/贪心/分治 → 五级
   - DFS/BFS/简单DP → 六级
   - 复杂DP/图算法 → 七级
   - 最短路/MST → 八级
   ↓
4. 确定知识点等级（取最高）
   ↓
5. 输出：等级 + 具体知识点
```
