# 文字处理系统

图片生成时文字处理的规范。

---

## 核心原则

1. **文字必须嵌入prompt**：AI根据场景自主决定是否在图片中加入文字
2. **语言自动匹配**：根据用户输入语言自动选择文字语言
3. **防混淆机制**：文字与背景区分，确保可读性

---

## 文字包含判定规则

| 场景 | 是否包含文字 | 文字内容 |
|------|-------------|---------|
| 知识可视化 | ✅ 包含 | 标题、关键概念、简要说明 |
| 算法教学 | ✅ 包含 | 步骤标签、公式、变量名 |
| 概念解释 | ✅ 包含 | 核心定义、关键术语 |
| 纯艺术插画 | ❌ 不包含 | 无 |
| 场景氛围图 | ❌ 不包含 | 无 |
| 信息图表 | ✅ 包含 | 数据标签、百分比 |
| 流程图解 | ✅ 包含 | 步骤名称、箭头说明 |

---

## 语言默认规则

| 用户输入语言 | 图片文字语言 |
|-------------|-------------|
| 中文 | 中文 |
| 英文 | 英文 |
| 中英混合 | 中文为主 |
| 未指定 | 中文 |

---

## 文字内容规范

### 知识可视化类

```
文字要求：
- 标题：简洁明了，2-6字
- 关键概念：精炼概括，4-10字
- 说明文字：一行不超过15字
- 字体风格：与整体风格协调
```

### 算法教学类

```
文字要求：
- 步骤标签：Step 1, Step 2 或 步骤一、步骤二
- 公式：保持原样，清晰呈现
- 变量名：使用代码风格字体
- 注释：简短，不超过一行
```

---

## 样式属性规范

| 属性 | 要求 |
|------|------|
| **字体** | 与风格协调（技术用代码字体，艺术用书法字体） |
| **颜色** | 与背景区分，高对比度 |
| **大小** | 标题大、正文中、注释小 |
| **位置** | 标题居上，说明居下或侧边 |
| **效果** | 可加阴影增强可读性 |

---

## 防混淆机制

### 绝对禁止

- ❌ 文字颜色与背景相近
- ❌ 文字被图片元素遮挡
- ❌ 文字重叠不可读
- ❌ 字体风格与主题冲突

### 推荐做法

- ✅ 文字区域背景纯色或留白
- ✅ 文字颜色与背景对比度 > 50%
- ✅ 文字区域独立，不与图形重叠
- ✅ 关键文字放大或加粗强调
