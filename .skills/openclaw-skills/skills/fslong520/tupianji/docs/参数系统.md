# 参数系统与决策流程

图片姬采用**AI自主决策**为核心的参数体系。

---

## 六维参数体系

| 参数维度 | AI推理依据 | 输出内容 |
|---------|-----------|---------|
| **目标受众** | 内容复杂度、专业程度、使用场景 | 受众群体判断 |
| **艺术风格** | 主题类型、情感基调、文化背景 | 视觉风格选择 |
| **主题氛围** | 内容属性、表达目的、情感诉求 | 氛围基调确定 |
| **构图布局** | 信息密度、重点分布、叙事需求 | 布局方式选择 |
| **配色方案** | 情感表达、风格特征、视觉舒适度 | 配色方案生成 |
| **知识结构** | 内容逻辑、教学需求、信息架构 | 呈现结构设计 |

---

## 决策流程

```
内容深度分析 → 特征提取 → 多维度关联推理 → 参数组合生成 → 验证优化 → 输出方案
```

---

## 多方案输出机制

AI决策后，必须输出**2-3个不同方案**供用户选择。

### 输出格式

```markdown
## 方案一：[方案名称]
- 目标受众：[推理结果]
- 艺术风格：[推理结果]
- 主题氛围：[推理结果]
- 构图方式：[推理结果]
- 配色方案：[推理结果]
- 推荐理由：[简要说明推理依据]

## 方案二：[方案名称]
[同上结构]

## 方案三：[方案名称]（可选）
[同上结构]

请选择您偏好的方案，或提出调整意见。
```

### 方案差异原则

| 差异维度 | 方案一 | 方案二 | 方案三 |
|---------|-------|-------|-------|
| **风格倾向** | 主推荐 | 保守安全 | 创新大胆 |
| **配色风格** | 情感匹配 | 专业通用 | 艺术创意 |
| **构图复杂度** | 信息密集 | 清晰简洁 | 艺术留白 |

---

## 用户参与机制

### 快速决策模式

当用户说"就用方案X"或"直接生成"时，跳过确认环节，直接输出prompt。

### 用户选择模式

完整流程：方案展示 → 用户选择 → 参数确认 → Prompt生成

---

## 用户提示语识别

| 用户表达 | 判断 |
|---------|------|
| "就用方案一" | 快速决策，直接生成 |
| "直接生成吧" | 快速决策，使用方案一 |
| "我喜欢方案二的风格" | 选择方案二 |
| "把方案一的配色换成冷色调" | 方案一基础上调整配色 |
| "再给我一个新方案" | 增加方案三或重新推理 |
