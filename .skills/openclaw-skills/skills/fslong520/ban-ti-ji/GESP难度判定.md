## 一、判定总览表

| 级别 | 算法要求 | 数据结构要求 | 典型题目特征 |
|------|---------|-------------|-------------|
| **一级** | 无特定算法 | 基本数据类型(int/long long/float/double/char/bool) | 顺序/分支/循环结构、四则运算、简单输入输出 |
| **二级** | 无特定算法 | 无新增 | 多层嵌套分支/循环、数学函数(abs/sqrt/max/min)、ASCII编码 |
| **三级** | 枚举法、模拟法 | 一维数组、字符串 | 进制转换、位运算、数组遍历、字符串操作 |
| **四级** | 递推算法、冒泡/插入/选择排序 | 二维数组、结构体、指针 | 函数定义调用、二维数组操作、排序实现 |
| **五级** | 二分查找/答案、递归、贪心、分治(归并/快排) | 链表 | 高精度运算、链表操作、数论(GCD/LCM/素数) |
| **六级** | DFS、BFS、简单动态规划 | 树、二叉树、栈、队列 | 树的遍历、搜索、01背包、栈队列操作 |
| **七级** | 复杂DP(LIS/LCS/区间DP)、图遍历 | 图、哈希表 | 二维DP、状态压缩、图DFS/BFS、flood fill |
| **八级** | 最小生成树、最短路径、算法优化 | 无新增 | Kruskal/Prim、Dijkstra/Floyd、组合数学 |

## 二、各级别详细判定标准

### 一级

**算法要求**：无特定算法

**数据结构**：基本数据类型
- int、long long
- float、double
- char、bool

**典型题目特征**：
- 顺序结构（按顺序执行语句）
- 分支结构（if/switch）
- 循环结构（for/while/do-while）
- 四则运算（加减乘除取余）
- 简单输入输出（cin/cout、scanf/printf）

**判定示例**：
| 题目 | 级别 |
|------|------|
| 输入两个数，输出它们的和 | 一级 |
| 输入一个数，判断是奇数还是偶数 | 一级 |
| 输入n，输出1到n的和 | 一级 |
| 输入成绩，输出等级（ABCD） | 一级 |

---

### 二级

**算法要求**：无特定算法

**数据结构**：无新增

**典型题目特征**：
- 多层嵌套的分支结构
- 多层嵌套的循环结构
- 数学函数：abs()、sqrt()、max()、min()、rand()
- ASCII编码与字符转换
- 数据类型转换（强制/隐式）

**判定示例**：
| 题目 | 级别 |
|------|------|
| 判断闰年（多层if嵌套） | 二级 |
| 输出九九乘法表（双层循环） | 二级 |
| 字母大小写转换（ASCII） | 二级 |
| 使用max/min求多个数的极值 | 二级 |

---

### 三级

**算法要求**：枚举法、模拟法

**数据结构**：一维数组、字符串

**典型题目特征**：
- 一维数组的遍历、查找、统计
- 字符串操作（大小写转换、查找、分割、替换）
- 进制转换（二/八/十/十六进制）
- 位运算（& | ^ ~ << >>）
- 原码、反码、补码
- 枚举所有情况找答案
- 模拟过程执行

**判定示例**：
| 题目 | 级别 |
|------|------|
| 统计数组中奇数个数 | 三级 |
| 字符串大小写转换 | 三级 |
| 十进制转二进制 | 三级 |
| 判断一个数是否为素数（枚举） | 三级 |
| 模拟简单游戏过程 | 三级 |

---

### 四级

**算法要求**：递推算法、冒泡/插入/选择排序

**数据结构**：二维数组、结构体、指针

**典型题目特征**：
- 二维数组操作（矩阵）
- 结构体定义与使用
- 指针基础操作
- 函数定义与调用
- 参数传递（值传递、引用传递）
- 冒泡排序、插入排序、选择排序
- 递推关系式求解
- 文件读写操作

**判定示例**：
| 题目 | 级别 |
|------|------|
| 矩阵转置 | 四级 |
| 学生成绩排序（结构体） | 四级 |
| 实现冒泡排序 | 四级 |
| 斐波那契数列第n项（递推） | 四级 |
| 函数实现两数交换 | 四级 |

---

### 五级

**算法要求**：二分查找/答案、递归、贪心、分治（归并/快排）

**数据结构**：链表

**典型题目特征**：
- 二分查找（有序数组查找）
- 二分答案（最大化最小值/最小化最大值）
- 递归求解问题
- 贪心策略选择
- 归并排序、快速排序
- 链表创建、插入、删除、遍历
- 高精度运算（加减乘除）
- 数论：GCD、LCM、素数筛、质因数分解

**判定示例**：
| 题目 | 级别 |
|------|------|
| 有序数组中查找元素（二分） | 五级 |
| 分巧克力（二分答案） | 五级 |
| 链表反转 | 五级 |
| 高精度加法 | 五级 |
| 求两个数的GCD（欧几里得） | 五级 |
| 活动选择问题（贪心） | 五级 |

---

### 六级

**算法要求**：DFS、BFS、简单动态规划

**数据结构**：树、二叉树、栈、队列

**典型题目特征**：
- 树的基本概念与遍历
- 二叉树的前/中/后序遍历
- 栈的操作与应用
- 队列的操作与应用
- 深度优先搜索（DFS）
- 广度优先搜索（BFS）
- 简单一维DP
- 01背包问题
- 哈夫曼树、二叉排序树
- 面向对象基础

**判定示例**：
| 题目 | 级别 |
|------|------|
| 二叉树的前序遍历 | 六级 |
| 迷宫问题（DFS/BFS） | 六级 |

| 爬楼梯问题（DP） | 六级 |

**算法要求**：复杂DP（LIS/LCS/区间DP）、图遍历

**数据结构**：图、哈希表

**典型题目特征**：

**判定示例**：

### 八级

**算法要求**：最小生成树、最短路径、算法优化

**数据结构**：无新增（综合应用）

**典型题目特征**：
- 最小生成树（Kruskal、Prim）
- 最短路径（Dijkstra、Floyd）
- 算法复杂度分析与优化
- 组合数学（排列、组合）
- 计数原理（加法、乘法）

- 倍增法
- 图论算法综合应用

**判定示例**：

## 三、判定规则

### 规则1：取知识点最高等级

如果题目涉及多个等级的知识点，取知识点所在的最高等级。

**示例**：
- 题目既涉及一维数组（三级），又涉及二分查找（五级）→ **五级**

### 规则2：算法优先

以算法复杂度为主要判定依据，数据结构作为辅助判定。

**示例**：
- 题目使用数组（三级），但涉及二分答案（五级）→ **五级**

### 规则3：知识点等级 ≠ 题目难度

**重要**：判定的等级是知识点所在等级，不是题目难度。

| 情况 | 说明 |
|------|------|
| 知识点在高级别，但题目简单 | 题目可能涉及五级知识点（如二分查找），但数据范围小、实现简单，实际难度较低 |
| 知识点在低级别，但题目难 | 题目可能只涉及三级知识点（如枚举），但需要巧妙的优化或复杂实现，实际难度较高 |
| 建议做法 | 在题面中同时标注知识点等级和实际难度预估 |

### 规则4：边界情况调整

| 情况 | 调整方向 |
|------|---------|
| 纯模拟题但数据范围极大（需要优化） | 知识点等级不变，可在题面备注实际难度 |
| 涉及高级算法但数据范围极小（暴力可过） | 知识点等级按最优解判定，备注暴力可过 |
| 题目有多种解法，最优解难度较高 | 按最优解判定知识点等级 |

### 规则5：特殊情况

| 特殊情况 | 等级判定 |
|---------|---------|
| 纯输入输出题 | 一级 |
| 纯数学公式计算 | 一级或二级 |
| 需要算法优化才能过 | 根据优化算法判定 |
| 需要特殊数据结构（如线段树） | 超出GESP范围，标记为"超出八级" |

## 四、快速判定流程

```
1. 分析题目类型
   ↓
2. 识别涉及的数据结构
   - 基本类型 → 一二级
   - 数组/字符串 → 三级起
   - 二维数组/结构体 → 四级起
   - 链表 → 五级起
   - 树/栈/队列 → 六级起
   - 图/哈希 → 七级起
   ↓
3. 识别涉及的算法
   - 无特定算法 → 一二级
   - 枚举/模拟 → 三级
   - 排序/递推 → 四级
   - 二分/贪心/分治 → 五级
   - DFS/BFS/简单DP → 六级
   - 复杂DP/图算法 → 七级
   - 最短路/MST → 八级
   ↓
4. 确定知识点等级（取最高）
   ↓
5. 输出结果：等级 + 具体知识点
```

**示例输出**：
- "本题涉及知识点为 **GESP三级**：进制转换、位运算"
- "本题涉及知识点为 **GESP五级**：二分查找、链表"
