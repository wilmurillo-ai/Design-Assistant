# agent — OpenClaw phishing-detection skill

OpenClaw skill that scans the user's Gmail inbox for phishing, crypto scams,
impersonation, and BEC, and (opt-in) reports offending identities to the
on-chain [`OpenClawRegistry`](../contracts/README.md) on Ethereum Sepolia.

- **Runtime**: [OpenClaw](https://www.npmjs.com/package/openclaw) ≥ 0.10
- **Python**: 3.11+
- **Gmail access**: OpenClaw's `gog` CLI (OAuth via `gog auth add`)
- **Participation tiers**: `watcher` (read-only, no crypto) or `reporter`
  (stakes 0.0001 Sepolia ETH, submits on-chain reports)

## Detection pipeline (summary)

Three stages, cheapest first, with short-circuits:

| Stage | What runs | When it short-circuits |
|---|---|---|
| **0 — cache**   | Dedup by Message-ID, trusted-sender allowlist, per-sender verdict TTL | Always first |
| **1 — gates**   | SPF/DKIM/DMARC fail, display-name spoof, registry identity/domain hit | Scores 75–95 deterministically; gate ≥ 90 ⇒ skip stage 2 |
| **2 — heuristics** | Mohammad URL red flags + Naive Bayes on subject+body | Both scores < 20 and no gate ⇒ mark safe, skip LLM |
| **3 — LLM**     | Claude classifies ambiguous middle, returns category + confidence | Only runs when 0–2 didn't resolve |

Final: `confidence = max(gate_score, 0.25 × url + 0.25 × bayes + 0.50 × llm)`.
Report if ≥ 70 (Reporter mode); alert if 50–70; silent if < 50.

See [`../README.md`](../README.md) for the full protocol overview and papers
cited in the scoring.

## Install the skill into OpenClaw

```bash
cp -r agent/ ~/.openclaw/skills/phishing-detection/
openclaw skills list
```

You should see `phishing-detection` in the list. Then in an OpenClaw session:

```
> install phishing detection
```

The skill walks you through the setup wizard (mode choice, Gmail account,
optional wallet + stake for Reporter). See [`SKILL.md`](./SKILL.md) for the
full conversation flow.

## Watcher vs Reporter (quick reference)

|                          | Watcher                       | Reporter |
|---|---|---|
| Wallet required          | No                            | Yes (generated by skill) |
| Sepolia ETH required     | 0                             | ≥ 0.0002 |
| On-chain writes          | None                          | stake + report_identity |
| Detects phishing locally | ✓                             | ✓ |
| Reads registry           | ✓                             | ✓ |
| Earns reward pool share  | –                             | ✓ (future) |
| Setup time               | ~2 min                        | ~10 min |

Upgrade from Watcher to Reporter at any time by re-running the wizard.

## Retraining the Bayes model

The shipped `data/bayes_model.json` is trained on an inline hackathon corpus
(25 phish + 25 ham samples + the 8 famous historical cases, with a ×50
phish-weight boost on famous-case tokens). Regenerate with:

```bash
python -m bin.trainer
```

## Manual end-to-end (Reporter tier)

After running the skill's setup wizard:

1. Send a test phishing email to your connected Gmail account (e.g. from a
   second account, subject "URGENT: verify account", body with wire/invoice
   language).
2. Wait for the next heartbeat cycle (~60s) or trigger one manually.
3. Watch the agent detect, classify, and ask whether to report on-chain.
4. Approve. Agent submits to the registry, returns tx hash.
5. `python -m bin.stats` prints the local scan stats.
6. Dashboard (`app/`, PR #3) shows the report appear in the Live Threat Feed.

## Security notes

- **Transaction authorization policy** is enforced in Python: `registry_client.py`
  hardcodes the registry address and only exposes stake / unstake / reportIdentity.
  Even a successful prompt injection can't redirect funds.
- Agent wallet private key lives in `~/.openclaw/phishing-detection/config.json`.
  In production this would move into OpenClaw's encrypted vault.
- `gog` owns Gmail OAuth; this skill never sees the user's Gmail password.
- On-chain reports carry only `keccak256(from_address)` + the plaintext domain.
  Full email addresses never hit the chain.
