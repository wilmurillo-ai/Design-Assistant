# daily-assistant MCP Server

> AI-powered daily task management via MCP (Model Context Protocol)

A personal task management MCP Server that works with Claude Code, OpenClaw, and any MCP-compatible AI agent. Deterministic operations (parsing, sorting, statistics) run in code with zero token cost — AI only steps in when creativity is needed.

## Features

### 6 Tools

| Tool | Description |
|------|------------|
| `recommend_next` | Recommends the most important task to work on next |
| `get_today` | Reads today's daily task file |
| `inherit_tasks` | Carries over uncompleted tasks from yesterday |
| `check_overdue` | Detects overdue task files |
| `generate_review` | Generates end-of-day review with statistics |
| `scan_split` | Flags tasks that are too large (>80min) or missing time estimates |

### 3 Resources

| Resource | Content |
|----------|---------|
| `daily://today` | Today's task file |
| `daily://dashboard` | Dashboard overview |
| `daily://history` | 7-day completion statistics |

## Quick Start

### Prerequisites

- Python 3.10+
- One of: [Claude Code](https://claude.ai/code) / [Cursor](https://cursor.com) / [Kiro](https://kiro.dev) / [Windsurf](https://windsurf.com) or any MCP-compatible AI agent

### One-Command Install

```bash
# macOS / Linux
git clone https://github.com/AsuraNale/daily-assistant-mcp.git && cd daily-assistant-mcp && python3 src/setup.py --auto

# Windows
git clone https://github.com/AsuraNale/daily-assistant-mcp.git && cd daily-assistant-mcp && py src/setup.py --auto
```

That's it. No manual `pip install` needed. The setup wizard automatically:
- Creates `.venv` and installs `fastmcp` (isolated, no system pollution)
- Creates data directory (`~/Desktop/日常小助手/Daily/` + `Dashboard.md`)
- Generates `config.json`
- Detects installed AI editors and writes MCP config for each one

**Supported editors** (auto-detected):

| Editor | Config Path |
|--------|------------|
| Claude Code | `~/.claude.json` |
| Cursor | `~/.cursor/mcp.json` |
| Kiro | `~/.kiro/settings/mcp.json` |
| Windsurf | `~/.codeium/windsurf/mcp_config.json` |

### Interactive Install (advanced)

If you prefer to choose a custom data directory:

```bash
python3 src/setup.py   # Windows: py src/setup.py
```

### Start Using

Open your AI editor and try:
- "What should I work on next?" → calls `recommend_next`
- "Show me today's tasks" → calls `get_today`
- "Carry over yesterday's tasks" → calls `inherit_tasks`
- "Any overdue tasks?" → calls `check_overdue`
- "Generate today's review" → calls `generate_review`
- "Any tasks that need splitting?" → calls `scan_split`

## Task Format

Tasks are written in Markdown with emoji markers:

```markdown
- [ ] Task description ⏱️45min 📅 2026-03-30 ⏫
```

| Marker | Meaning | Required |
|--------|---------|:--------:|
| `⏱️30min` | Time estimate (recommended ≤ 80min) | Yes |
| `📅 YYYY-MM-DD` | Deadline | Yes |
| `⏫` | Highest priority | No |
| `🔼` | High priority | No |
| `🔽` | Low priority | No |

Completed tasks: `- [x] Task description ...`

## Configuration

`config.json` (auto-generated by `setup.py`):

```json
{
  "daily_dir": "/Users/you/Desktop/日常小助手/Daily",
  "dashboard_file": "/Users/you/Desktop/日常小助手/Dashboard.md",
  "language": "zh"
}
```

### Language / 语言

Set `"language"` in `config.json` to switch all tool output between Chinese and English:

| Value | Language | Example output |
|-------|---------|---------------|
| `"zh"` | 中文 (default) | 🎉 今日任务清零！ |
| `"en"` | English | 🎉 All tasks done! |

> **Note**: MCP tool names and descriptions remain in English (MCP standard).
> Only the **output text** (recommendations, reviews, status messages) is affected.

You can also pass a custom config path:
```bash
python3 src/server.py --config /path/to/config.json
```

## Design Philosophy

- **Zero-token deterministic ops**: Parsing, sorting, and statistics run in Python code — no AI tokens consumed
- **AI only when needed**: Task splitting suggestions, creative advice, context-aware recommendations
- **Platform-independent**: Works on Windows, macOS, Linux. No dependency on Obsidian or any specific editor
- **Simple data format**: Plain Markdown files — edit with any text editor, version-control friendly
