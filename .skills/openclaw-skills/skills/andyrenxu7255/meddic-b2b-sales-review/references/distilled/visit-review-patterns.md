# 拜访后复盘蒸馏模式

## 用途

当复盘来源于一次拜访、会议、沟通纪要时，优先读取本文件。

## 核心流程

1. 整合本次拜访记录 + 该客户历史关键信息
2. 用六步法判断当前所处阶段
3. 用 MEDDIC 检查哪些要素已明确、哪些缺失
4. 输出：
   - 当前机会健康度
   - 当前最关键缺口
   - 下一步具体动作
   - 需要补充的关键信息

## 关键原则

- 不是展示理论，而是把隐性销售经验转成标准化动作
- 每次拜访后都要形成“下一步必须做什么”的明确结论
- 复盘结论要能承接到下一步跟进计划中
