# 阶段 Gate 校验规则

## 用途

本文件用于判断当前阶段是否“真正通过”，避免销售把主观感觉误当成阶段推进。

## Discovery 通过条件

至少满足：
- 客户问题 / 业务目标已基本明确
- 客户愿意继续探索，不是泛兴趣
- 已有基础关键人信息
- 不是伪需求或纯礼貌交流

## Scope 通过条件

至少满足：
- 项目边界和优先级比 Discovery 更清楚
- 客户承认这是一个值得继续推进的项目
- 范围轮廓基本明确
- 已从“聊需求”进入“收敛范围”

## Go/No-Go 通过条件

至少满足：
- EB 可触达且有兴趣
- Champion 真实存在
- 痛点与业务结果绑定
- 决策流程基本可判断
- 预算或预算逻辑基本成立

## Validation Solution 条件

至少满足：
- 客户认可方案方向
- 已有关键验证动作或计划
- 不是还停留在泛讨论

## Business Case 条件

至少满足：
- 客户已开始从价值、ROI、采购合理性角度评估项目
- 已有预算、立项、审批逻辑

## Negotiate & Close 条件

至少满足：
- 商务、法务、采购、签约流程真实启动
- 当前阻塞主要在成交环节，而不是前面阶段未完成

## 使用原则

- 如果 gate 没通过，不要直接套该阶段赢率
- gate 不成立时，优先考虑阶段回退，而不是轻微打折
