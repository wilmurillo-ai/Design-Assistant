# 单个机会 / 单子复盘

## 目标

用于从销售管理者视角复盘**单个销售机会**，判断它当前处于什么阶段、为什么推进/卡住/可能输、销售做得对不对，以及管理者下一步该推什么动作。

## 建议读取顺序

1. 先判断阶段：需要时读 `methods/six-step.md`
2. 再查缺口：需要时读 `methods/meddic.md`
3. 涉及去留判断：需要时读 `methods/go-no-go.md`
4. 输出结论前：读 `templates/output-format.md`
5. 信息不足时：读 `templates/questions.md`

## 复盘流程

### 1. 识别当前阶段
- 用客户行为判断，而不是销售自我感觉
- 必须说明为什么在这一步、为什么还没进下一步

### 2. 识别核心卡点
优先判断是不是以下问题：
- 找不到 EB
- 没有 Champion
- 痛点不成立或不紧急
- 决策流程不清
- 销售动作错位（该挖痛点时讲产品、该找 EB 时做演示）

### 3. 区分问题类型
- 机会质量问题
- 销售动作问题
- 策略问题
- 资源问题

### 4. 输出管理动作
至少输出：
- 是否继续推进
- 销售下一步最关键动作（1-3 个）
- 管理者是否介入
- 下次复盘前补齐什么证据

## 重点判断句式

- 当前阶段是 X，不是 Y
- 这单卡住的根因不是“跟进慢”，而是 X
- 现在最该补的不是方案，而是 X
- 如果两周内仍拿不到 X，建议下调预测/暂停投入
