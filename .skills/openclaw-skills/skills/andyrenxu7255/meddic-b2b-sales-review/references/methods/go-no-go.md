# Go / No-Go 与 Forecast 校准参考

## 用途

当任务需要判断“是否继续投入”“预测应不应该上调/下调”“这单要不要放弃”时读取本文件。

## Go 的最低条件

一般至少要满足：
- EB 可触达
- 有真实 Champion
- 痛点与业务结果绑定
- 有预算可能
- 决策流程可预见

## 常见 No-Go / 暂缓信号

- 找不到 EB
- 没有真实 Champion
- 痛点不急，不值得预算支持
- 决策流程不清
- 客户没有明确下一步承诺

## Forecast 校准

- **Commit**：硬证据充分，EB/Champion/流程都较清晰
- **Best Case**：方向对，但仍缺关键证据
- **Pipeline**：还在探索，不能美化

## 管理动作

- 硬证据不够，不要高报预测
- 两周内关键短板补不齐，考虑降预测
- 机会质量低于门槛时，优先止损而不是继续堆资源
