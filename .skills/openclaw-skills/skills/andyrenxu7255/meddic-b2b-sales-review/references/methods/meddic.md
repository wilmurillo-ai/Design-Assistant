# MEDDIC / MEDDPICC 方法参考

## 用途

当任务需要识别机会缺口、判断赢率、分析为什么卡住/为什么会输时读取本文件。

## 核心维度

- **Metrics**：业务价值是否量化
- **Economic Buyer**：是否识别并触达经济决策者
- **Decision Criteria**：客户如何评估供应商
- **Decision Process**：客户内部怎么决策
- **Identify Pain**：痛点是否真实、紧迫、能支撑预算
- **Champion**：是否有真实支持者

如任务明确涉及 MEDDPICC / MEDDPIC，再补充：
- **Paper Process**：采购/合同/法务流程
- **Competition**：竞争格局与替代方案

## 复盘时的使用方式

对每个维度标记：
- 已验证
- 部分验证
- 未验证

优先盯住高风险缺口：
- EB 不可触达
- 没有 Champion
- 痛点不紧急
- 决策流程不清晰

## 管理判断原则

- 先找关键缺口，再谈策略
- 没有 EB / Champion 时，不要假装机会健康
- 没有 Metrics 时，Business Case 很难成立
