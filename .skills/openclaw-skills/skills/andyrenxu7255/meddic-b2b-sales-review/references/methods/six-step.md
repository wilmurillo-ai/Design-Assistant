# 六步法方法参考

## 用途

当任务需要判断机会/项目当前处于哪一步，以及为什么不是下一步时读取本文件。

## 六步法

1. **Discover / 发现**：客户问题、业务目标、关键人、时间背景是否清楚
2. **Scope / 范围**：项目边界、优先级、范围轮廓是否清楚
3. **Go / No-Go**：是否值得继续投入
4. **Validation Solution / 验证方案**：客户是否认可方案方向与有效性
5. **Business Case / 商业案例**：ROI、预算、立项逻辑是否成立
6. **Negotiate & Close / 谈判成交**：商务、法务、采购、竞争、风险是否可控

## 判断原则

- 用客户行为判断阶段，不用销售感觉判断
- 必须说明“为什么在这一步”与“为什么没进下一步”
- 阶段可以回退，不要硬维持高阶段

## 常见误判

- 只做了需求交流就误判到 Validation
- 客户说有兴趣就误判为能签
- 开始谈价格就误判为接近成交
