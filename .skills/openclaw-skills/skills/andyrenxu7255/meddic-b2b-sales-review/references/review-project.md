# 项目复盘 / 项目推进 Review

## 目标

用于复盘**项目型销售机会**或已经进入项目推进期的单子，判断项目当前卡在哪个节点、问题是销售问题还是项目推进问题，以及管理者如何推动跨部门资源与客户关键人。

## 适用场景

- 项目推进变慢
- 客户反复变更需求
- 方案验证迟迟过不去
- 已立项但迟迟不能签约
- 实施、商务、采购、法务多方缠绕

## 建议读取顺序

1. 先读 `methods/six-step.md` 判断项目当前在六步法哪个节点
2. 再读 `methods/meddic.md` 判断关键人、决策流程、痛点是否成立
3. 如果在投入与放弃之间摇摆，读 `methods/go-no-go.md`
4. 输出前读 `templates/output-format.md`

## 项目复盘重点

### 1. 不只看“项目进度”，要看“项目动因”
- 客户为什么还要推进？
- 项目是不是已经失去业务优先级？
- 有没有真实 sponsor 在推动？

### 2. 区分三种卡点
- **项目管理卡点**：排期、接口、资源、实施复杂度
- **商务卡点**：采购、法务、条款、价格
- **机会质量卡点**：EB 不在、痛点弱、Champion 弱

### 3. 管理者重点动作
- 是否需要高层出面
- 是否需要方案、交付、产品跨部门一起进场
- 是否需要重设阶段，不再虚高判断

## 输出重点

项目复盘要明确：
- 卡在项目推进哪个节点
- 这是不是“假项目推进”
- 接下来是补资源，补关键人，还是直接降温
