# 输单复盘

## 目标

用于从销售管理者视角复盘输单原因，不停留在表面理由，而是识别真正的输单逻辑、可改进动作以及以后如何避免重演。

## 建议读取顺序

1. 先读 `methods/meddic.md` 看哪些关键维度缺失
2. 再读 `methods/six-step.md` 判断是不是阶段错判
3. 如果涉及是否早该放弃，读 `methods/go-no-go.md`
4. 输出前读 `templates/output-format.md`
5. 信息不足时读 `templates/questions.md`

## 输单分类

必须先区分输给谁：
- 输给竞争对手
- 输给“不做项目”
- 输给内部优先级变化
- 输给采购/法务/周期
- 输给销售动作失误

## 深挖原则

不要停在“价格高”这种表象。
继续追问：
- 为什么客户觉得贵？
- 为什么价值没有建立起来？
- 为什么 EB 没有愿意拍板？
- 为什么 Champion 没推动起来？

## 输出重点

- 表象原因
- 真正原因
- 如果重来一次，最该改的动作
- 对销售和管理者的复盘教训
