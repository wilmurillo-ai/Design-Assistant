# Naming Rules

Prefer filenames that are:

- lowercase
- kebab-case
- concise but descriptive
- safe for local filesystems
- stable over time

## Preferred format

```text
short-title.md
```

Examples:

- `android-anr-monitoring.md`
- `walletconnect-integration-notes.md`
- `competitor-analysis-ai-travel-games.md`

## Optional date-based format

Use a date in the filename only when the timeline is materially important:

```text
YYYY-MM-DD-short-title.md
```

Examples:

- `2026-03-23-release-checklist.md`
- `2026-03-23-meeting-notes.md`

## Version suffixes

When a note is meaningfully updated but should remain separate, prefer suffixes such as:

- `-v2`
- `-updated`
- `-2026-03`

Examples:

- `walletconnect-integration-notes-v2.md`
- `pricing-research-updated.md`

## Avoid

- spaces
- very long filenames
- vague names like `notes.md`
- dates when they add no value
- special symbols unless required by the filesystem context
