# 费曼虾心跳

每次心跳触发时，执行以下流程。心跳不再是按概念清单轮询考试，而是读项目、发现盲区、按需提问。

## 1. 检查活跃契约

读取 `contracts.json`，筛选 `status: "active"`。

- 没有活跃契约 → `HEARTBEAT_OK`，不做任何事
- 有活跃契约 → 继续

## 2. 读取用户项目/笔记（只读）

读取契约 `resources` 中 `type: "local"` 的 path 指向的文件内容。

- 若无可读路径 → 退回到基于概念清单的追问（按 instructions/feynman.md 的兜底逻辑）
- 若有 → 将最新内容作为上下文

## 3. 发现可追问的点

在读到的内容中寻找「用户可能不理解但正在使用」的知识点，例如：

- 代码里用了某 API/概念，但概念清单中该条 status 为 pending 或 in_progress
- 代码里写了某配置（如 learning_rate），笔记里没解释其影响
- 笔记里提到了某概念，但解释不完整或可能有盲区

优先从**最近有变化**的部分找（若可推断：新写的代码、新增的笔记段落）。

## 4. 执行费曼追问

按照 `instructions/feynman.md` 第二部分的完整流程执行：

准备上下文（契约状态 + 项目内容 + 记忆）→ 生成基于项目内容的问题 → 推送给用户 → 等待回答 → 判定。

## 5. 检查截止日期

若活跃契约的 deadline 距今 ≤ 3 天，追问后附加：

```
📅 提醒：这份契约还剩 {N} 天，还有 {M} 个概念待掌握。
```

## 6. 检查长时间未互动

若最近活跃契约的 `last_active_at` 超过 48 小时，在费曼问题前加一句：

```
🦞 好久不见。上次我们聊到了「{上次的概念}」。
```
