# 同步质量检查清单（Reviewer）

> 此文件是 feishu-sync-obsidian 的**审查标准**。
> Step 4 质量检查时，逐条对照执行。

---

## 审查协议

**Step 1**：加载本清单
**Step 2**：检查每个同步的文档
**Step 3**：按严重程度分类输出

---

## 检查项

### 必须修复（error）

| # | 检查项 | 验证方法 |
|---|--------|---------|
| E1 | frontmatter 完整 | 文件开头有 `---` 包裹的 YAML |
| E2 | feishu_doc_token 正确 | 读取 frontmatter，确认 feishu_doc_token 存在 |
| E3 | feishu_node_token 正确 | 读取 frontmatter，确认 feishu_node_token 存在 |
| E4 | feishu_wiki 正确 | frontmatter 中的 wiki 名称与来源一致 |
| E5 | 文件路径符合 PARA | 路径在 routing-rules.md 的映射范围内 |

### 建议修复（warning）

| # | 检查项 | 验证方法 |
|---|--------|---------|
| W1 | frontmatter 格式正确 | YAML 可解析，无语法错误 |
| W2 | 文档内容非空 | 内容长度 > 10 字符 |
| W3 | 文件名安全 | 不含 `\/:*?"<>|` 等非法字符 |
| W4 | 路径已创建 | 目标目录存在 |

### 可考虑（info）

| # | 检查项 | 验证方法 |
|---|--------|---------|
| I1 | created 时间合理 | 不超过当前时间 7 天 |
| I2 | 标签完整 | frontmatter 包含 tags 字段 |

---

## 输出格式

```
【同步质量报告】

## Summary
- 写入文档数：X
- 通过：X
- 失败（必须修复）：X
- 警告：X

## Findings

### 🔴 必须修复（X）
- **E1** [文件名]：[原因] → [修复建议]
- ...

### 🟡 建议修复（X）
- **W1** [文件名]：[原因] → [修复建议]
- ...

### 🟢 通过（X）
- [文件名列表]

## Top 3 改进建议
1. [最严重的问题]
2. [次严重的问题]
3. [第三严重的问题]
```

---

## 修复规则

- **E1-E5**：必须修复，同步失败
- **W1-W4**：建议修复，可能影响后续同步
- **I1-I2**：可选，不影响同步

---

## 何时触发审查

- 每次同步执行后自动触发（Step 4）
- 用户要求"检查同步结果"时触发
- 发现同步问题时触发
