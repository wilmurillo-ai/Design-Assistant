# Tool Discovery — Reference Guide

Finding new tools, plugins, APIs, and services that could help the company.
The systematic approach to staying current with what's available.

---

## 1. TOOL DISCOVERY CHANNELS

### Where to Find New Tools
```
PRODUCT HUNT (producthunt.com):
  Best for: Newly launched tools, startup software
  Check: Daily "Top Products" and weekly digest
  Filter by: Category (e.g., "Productivity", "AI", "No-code")
  Signal: 500+ upvotes = significant interest

HACKER NEWS:
  "Ask HN: What tools do you use for X?"
  "Show HN:" posts for new tools
  Best for: Developer-focused tools, technical solutions

REDDIT:
  r/SaaS, r/Entrepreneur, r/nocode, r/webdev
  "What's your stack for X?" threads
  Tool recommendation threads

TWITTER/X:
  Search: "[problem] tool" or "[use case] software"
  Follow practitioners who share their tool stacks
  "Just launched" searches

APPSUMO:
  Lifetime deals on tools
  Good signal: 500+ reviews on an AppSumo deal = validated product
```

## 2. TOOL EVALUATION CRITERIA

### The Tool Evaluation Framework
```
For any potential tool, evaluate:

FIT SCORE:
  Does it solve a problem we actually have? (1-10)
  Is it better than our current solution? (1-10)
  Does it fit our workflow? (1-10)

ECONOMICS:
  Price: Free tier available? / Monthly cost?
  Value: ROI positive vs. time saved?
  Runway impact: Is this a cost we can afford?

RISK:
  Maturity: How established is it? (Startup risk = data loss)
  Data: What data does it access? (Privacy/security)
  Dependency: If they shut down, how hard is migration?

EFFORT:
  Learning curve: Hours to productive use?
  Integration: Does it work with our existing stack?
  Maintenance: Ongoing time required?
```
