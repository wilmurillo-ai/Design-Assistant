# Research Synthesis — Reference Guide

Structuring findings, extracting insights, writing briefings, and turning raw data
into decision-ready intelligence.

---

## 1. THE SYNTHESIS PROCESS

### From Raw Data to Decision-Grade Intelligence
```
STEP 1: COLLECT
  Gather all data points from all sources
  Don't filter yet — collect broadly

STEP 2: SORT
  Group similar data points together
  Identify themes and patterns
  Tag each point with source and confidence level

STEP 3: PRUNE
  Remove duplicates
  Remove low-confidence data that doesn't add to the picture
  Keep only data that informs the specific decision

STEP 4: INTERPRET
  For each theme: What does this mean?
  Look for: Surprises, confirmations, contradictions
  Ask: What would this data have to be true for our recommendation to be right?

STEP 5: CONCLUDE
  What is the single most important finding?
  What should we do about it?
  How confident are we?

STEP 6: DOCUMENT
  Write the briefing in SEIAR format
  Lead with conclusion (not how you got there)
```

## 2. INSIGHT EXTRACTION TECHNIQUES

### The "So What?" Test
For every data point, ask "so what does this mean for us?"

```
DATA: "Google Trends shows 'family planner' searches up 45% in 5 years"
SO WHAT: Growing market for family planning products — good timing to enter

DATA: "Top Amazon book in this category has 3,200 reviews"
SO WHAT: Validated large demand, but also established competition we need to differentiate from

DATA: "Reddit posts about this problem average 50+ upvotes"
SO WHAT: Real, validated pain point with engaged community = good audience to market to

DATA: "Two competing Gumroad products have less than 10 sales combined"
SO WHAT: Either very new market (wait for more validation) or wrong platform for this product
```

### Pattern Recognition
```
Look for these patterns across data:

CONVERGENCE: Multiple sources agree → High confidence finding
  3 sources all show growing demand = strong signal

DIVERGENCE: Sources contradict each other → Investigate further
  High search volume but no competing products = investigate why

OUTLIER: One source shows something different from all others
  Usually: Either the outlier is wrong OR the others are missing something

ABSENCE: Something that should be there isn't
  No community discussions about obvious problem = either wrong audience or wrong problem framing
```

## 3. SYNTHESIS ANTI-PATTERNS

### Common Synthesis Failures

**Confirmation Bias**
Collecting data that confirms what you already believe, ignoring contradicting evidence.
Fix: Explicitly search for evidence AGAINST your hypothesis.

**False Precision**
"Our addressable market is exactly $47.3 million"
Fix: Use ranges. "Estimated $40-60 million, based on [methodology]."

**Missing the Forest for Trees**
So focused on individual data points you miss the big picture pattern.
Fix: After all data is collected, step back and ask: what is the story here?

**Premature Synthesis**
Concluding before enough data is collected.
Fix: Decide minimum evidence threshold before starting research. Don't conclude until met.
