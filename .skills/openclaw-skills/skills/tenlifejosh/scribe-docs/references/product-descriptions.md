## 1. THE PRODUCT DESCRIPTION FRAMEWORK

### The 5-Part Description Structure
```
PART 1 — THE HOOK (lines 1-2):
  The buyer's problem stated in their language OR
  The transformation they want stated specifically.
  NOT: "This is a guide to family organization."
  YES: "Sunday nights shouldn't feel like dread."

PART 2 — THE BRIDGE (lines 3-5):
  Connect the problem to why this product exists.
  This is where the creator's story often appears briefly.
  Makes the product feel human, not manufactured.

PART 3 — THE SOLUTION (lines 6-12):
  What the product IS.
  What's specifically inside it.
  Benefits connected to each major component.
  Format: "This is a [X]-page [type] that [does what for whom]."

PART 4 — THE PROOF (lines 13-17):
  Social proof if available (specific quotes or numbers).
  Creator credibility if relevant.
  One concrete result or use case.

PART 5 — THE CTA (lines 18-20):
  Who this is for (include audience).
  One clear invitation to purchase.
  Remove any remaining friction.
```

## 2. GUMROAD-SPECIFIC COPY

### Gumroad Title Formula
```
FORMULA: [Specific Outcome] + [For Specific Audience] + [Key Differentiator]

EXAMPLES:
  "FamliClaw — Family Organization System That Actually Works"
  "Budget Binder — Monthly Finance Tracker for Families Starting Over"
  "The First Job Playbook — Career Guide for Young Adults Who Feel Lost"

RULES:
  - Under 80 characters (fully visible in listings)
  - Contains primary keyword naturally
  - Specific (not generic)
  - Benefit or outcome in title
```

### Gumroad Description Template
```
[Problem or transformation statement — 1-2 sentences]

[Creator's origin story / why this exists — 2-3 sentences]

What's inside:

• [Component 1]: [What it does for them]
• [Component 2]: [What it does for them]
• [Component 3]: [What it does for them]
[Continue for main components]

[Social proof or use case — 1-2 sentences]

This is for you if:
• [Specific characteristic 1]
• [Specific characteristic 2]
• [Specific characteristic 3]

[$Price]. Instant download. [Any guarantee language].
```

## 3. KDP BOOK DESCRIPTION FORMULA

### Amazon Book Description (HTML Support Available)
```
KDP descriptions support basic HTML:
  <b>Bold text</b>
  <br> Line break
  <p>Paragraph</p>
  <ul><li>Bullet item</li></ul>

OPTIMAL LENGTH: 600-1000 words (Amazon rewards longer descriptions)

STRUCTURE:
  Opening hook (1 paragraph)
  Problem validation (1-2 paragraphs)
  Solution introduction (1 paragraph)
  What's inside (bulleted list, 5-10 items)
  Who it's for (1 paragraph)
  What changes after (1 paragraph)
  Social proof (if available)
  Final CTA

KEYWORD PLACEMENT:
  Primary keyword: First sentence, title area
  Secondary keywords: Naturally throughout description
  Amazon factors: keywords, conversion rate, reviews
```

### KDP Category and Keywords
```
CATEGORIES:
  Select 2 categories (maximum)
  Choose most specific applicable category
  Niche categories = easier to rank
  
KEYWORDS (7 maximum):
  Use long-tail phrases (3-5 words each)
  Think: what would someone search to find this?
  NOT: "family" (too broad)
  YES: "family organization printable" or "family planner workbook"
  
KEYWORD RESEARCH:
  Amazon autocomplete: type keyword, note suggestions
  Publisher Rocket: paid tool for KDP keyword research
  Review competitor keywords: look at their subtitles and descriptions
```

## 4. ETSY LISTING COPY

### Etsy Title Formula
```
Etsy title is critical for search — put ALL keywords in title

FORMULA: [Primary keyword] [Product type] [secondary keyword] [audience]
"Family Planner Printable | Monthly Organization PDF | Digital Download for Busy Moms"

RULES:
  - 140 character limit
  - Most important keywords first
  - Use | or / to separate keyword groups
  - Don't worry about readability — optimize for search
```

### Etsy Description
```
Etsy descriptions are mostly for conversion, not search (tags handle search)

STRUCTURE:
  What it is (1-2 sentences)
  What's included (specific list with page counts/sizes)
  How to use it (1-2 sentences)
  What you'll receive (file format, how to download)
  Technical details (dimensions, file types, resolution)
  Usage rights (personal use or commercial?)
  Contact/support information
  
ETSY TAGS (13 tags):
  Mix specific (2-3 word phrases): "family planner PDF", "printable budget tracker"
  Mix broad (1-2 word): "printable planner", "digital download"
  Include audience terms: "for moms", "for families"
```

## 6. DESCRIPTION QUALITY STANDARDS

### Quality Checklist for Any Product Description
```
OPENING:
- [ ] First line is the reader's problem OR desired outcome (not a product description)
- [ ] Would work as a social media hook
- [ ] Doesn't start with "This is a guide to..."

BODY:
- [ ] Contains specific numbers (pages, components, time savings)
- [ ] Each component listed with benefit attached
- [ ] Uses "you" not "customers" or "users"
- [ ] No jargon or insider language
- [ ] Specific enough to build trust

PROOF:
- [ ] Includes at least one trust signal (reviews, specificity, creator story)
- [ ] Doesn't oversell or make unprovable claims

CLOSING:
- [ ] Who it's for is clear
- [ ] Purchase action is easy and clear
- [ ] Price stated (where platform allows)
- [ ] Format/delivery stated (PDF, instant download, etc.)

VOICE:
- [ ] Sounds like a real person (Josh's voice if first person)
- [ ] No AI-obvious phrases
- [ ] Direct, not hedging
```
