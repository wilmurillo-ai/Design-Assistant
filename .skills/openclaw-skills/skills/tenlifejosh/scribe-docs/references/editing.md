# Editing — Reference Guide

Self-editing checklist, cutting filler, tightening copy, and improving any piece of writing.
The discipline that turns a draft into something worth reading.

---

## 1. THE EDITING MINDSET

The first draft is for getting ideas down. The edit is for the reader.
Switch hats completely: write as creator, edit as reader.
Wait before editing when possible — distance reveals problems.

---

## 2. THE EDITING CHECKLIST

### Pass 1: Big Picture (Structure)
```
Read straight through without editing. Answer:
- [ ] Does it do what it's supposed to do? (sell, inform, connect)
- [ ] Is the structure correct? (right info in right order)
- [ ] Does the opening earn continued reading?
- [ ] Is the ending strong? (not just trailing off)
- [ ] Is anything important missing?
```

### Pass 2: Paragraph Level (Flow)
```
- [ ] Each paragraph has one clear purpose
- [ ] Paragraphs are in the right order (each leads to the next)
- [ ] No orphan paragraphs (introduced ideas that go nowhere)
- [ ] Transitions don't announce transitions ("Now, let's discuss...")
```

### Pass 3: Sentence Level (Clarity)
```
- [ ] Cut every sentence that doesn't move things forward
- [ ] Simplify every sentence that tries to do too much
- [ ] Passive voice → active voice ("is being used" → "people use")
- [ ] Weak verbs → strong verbs ("make an improvement" → "improve")
- [ ] "Very", "really", "quite", "rather" → delete all of them
```

### Pass 4: Word Level (Precision)
```
- [ ] Every adjective earns its place or gets cut
- [ ] No redundant modifiers ("completely finished" → "finished")
- [ ] Specific nouns over generic nouns ("2 months" not "a couple months")
- [ ] No AI-obvious vocabulary (delve, plethora, holistic, leverage, etc.)
- [ ] Short words over long words when meaning is the same
```

## 3. THE CUT LIST

### Always Delete These Phrases
```
"In order to" → "To"
"Due to the fact that" → "Because"
"At this point in time" → "Now"
"In the event that" → "If"
"A large number of" → "Many"
"It is worth noting that" → [Delete + state the thing]
"That being said" → [Delete]
"With that in mind" → [Delete]
"It goes without saying" → [Delete — if it goes without saying, don't say it]
"Needless to say" → [Same — delete]
"As I mentioned earlier" → [Delete — they remember, or remind them cleanly]
"In conclusion" → [Delete — just conclude]
```

### Common Bloat Patterns
```
THROAT-CLEARING OPENERS (delete entire first paragraph if it just warms up):
  "In today's fast-paced world..."
  "When it comes to [topic]..."
  "Many people struggle with..."
  "It's no secret that..."

HEDGE WORDS (delete):
  might, could, perhaps, possibly, potentially, seemingly, arguably
  (Exception: use these when genuine uncertainty exists)

INTENSIFIERS THAT WEAKEN (delete):
  very, really, quite, extremely, incredibly, absolutely, definitely

REDUNDANCIES:
  "completely finished" → "finished"
  "absolutely free" → "free"
  "end result" → "result"
  "future plans" → "plans"
  "past experience" → "experience"
```

---

## 4. THE VOICE EDIT

### Checking for Josh's Voice After Editing
```
After any structural and clarity edits, do a voice pass:
- [ ] Reads like a person said it (not a committee wrote it)
- [ ] Short sentences where punchy moments call for them
- [ ] No sentences over 25 words without purpose
- [ ] First sentence is the real sentence (no warmup)
- [ ] No AI-vocabulary
- [ ] Opinion present (not just information)
```

---

## 5. THE READ-ALOUD TEST

The final edit pass: read it out loud.

If you stumble over a sentence → rewrite it.
If you take a breath in the middle of a sentence → it's too long.
If you feel awkward saying something → the reader will feel awkward reading it.
If you hear yourself speed up through a section → it's filler.
If you hear yourself slow down → good writing.

---

## 6. EDITING BY FORMAT

```
COLD EMAIL EDITING (after first draft):
  1. Can you cut it by 30%? (You usually can)
  2. Is the ask in one sentence? (Should be)
  3. Is the personalization real? (Vague = cut)
  4. Does it end with one small ask? (Not three)

SOCIAL POST EDITING:
  1. Does the first line stop a scroll?
  2. Can the last line be cut? (Often yes)
  3. Does every tweet in a thread earn its place?
  4. Are there any hedge words? (Delete all of them)

PRODUCT DESCRIPTION EDITING:
  1. Does the opening hook work without context?
  2. Are benefits stated for every feature?
  3. Is there any corporate language? (Replace it)
  4. Is the CTA the clearest sentence in the piece?
```
