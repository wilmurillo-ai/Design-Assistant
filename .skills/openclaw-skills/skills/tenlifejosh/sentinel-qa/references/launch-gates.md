## 1. THE GATE SYSTEM OVERVIEW

### What Are Launch Gates?
Launch gates are binary pass/fail checkpoints. Every item in every gate must pass.
One failed gate item = asset does not ship until that item is fixed.

There is no "mostly passes." There is PASS or FAIL.

### The Five Gates
```
GATE 1: Content Completeness
  "Is all the content actually there?"
  Catches: missing sections, placeholders, incomplete work

GATE 2: Brand Quality  
  "Does this represent us well?"
  Catches: off-brand voice, inconsistent design, quality below standard

GATE 3: Risk & Legal
  "Could this hurt us?"
  Catches: legal exposure, reputational damage, platform violations

GATE 4: Technical Integrity
  "Does this actually work?"
  Catches: broken links, corrupt files, failed downloads, rendering errors

GATE 5: Commercial Readiness
  "Is this ready to sell/publish?"
  Catches: wrong price, wrong category, missing metadata, incomplete listing
```

## 2. GATE 1: CONTENT COMPLETENESS

**Purpose:** Verify all content is present, final, and complete.

```
MANDATORY CHECKS — ALL MUST PASS:

□ G1.1 — No placeholder text exists anywhere in the asset
  Check for: "Lorem ipsum", [PLACEHOLDER], TBD, "Insert X here", blank fields
  FAIL = any placeholder found → STOP, return to creator

□ G1.2 — All promised content is present
  For products: Every section mentioned in description or table of contents exists
  For emails: All sections present (header, body, CTA, footer)
  For social: All referenced images/media are attached
  FAIL = any referenced content missing → STOP

□ G1.3 — Word count meets minimums
  Product descriptions: 200+ words
  Email body: 100+ words (excluding footer)
  Long-form content: meets stated length
  FAIL = under minimum → STOP

□ G1.4 — Final version confirmed
  File name contains no "draft", "WIP", "v0", or "temp"
  Date on content is not from a prior year
  Author confirmed this is final (not still revising)
  FAIL = any version ambiguity → STOP

□ G1.5 — All required elements present by type
  Emails: subject, body, CTA, unsubscribe, footer
  Products: title, description, file, cover image, price
  Social posts: caption, image (where required), CTA
  FAIL = any required element missing → STOP
```

**Gate 1 Verdict:** PASS (all boxes checked) / FAIL (any box unchecked)

## 3. GATE 2: BRAND QUALITY

**Purpose:** Ensure the asset meets our quality standard and brand identity.

```
MANDATORY CHECKS — ALL MUST PASS:

□ G2.1 — Voice is consistent with Ten Life Creatives brand
  Direct, authentic, builder energy — not corporate, not generic
  FAIL = clearly off-brand voice that would feel wrong to audience

□ G2.2 — No AI-obvious language patterns
  Check: "delve into", "it is worth noting", "furthermore", "plethora", etc.
  FAIL = 3+ AI-obvious phrases in a single piece

□ G2.3 — Visual quality meets standard (for assets with visuals)
  No blurry images, no broken formatting, no ugly layout
  FAIL = obviously low-quality visual that would reflect poorly

□ G2.4 — Tone is appropriate for audience and context
  Matches the emotional context (celebratory for wins, serious for hard topics)
  FAIL = tone mismatch that could confuse or alienate

□ G2.5 — Claims are honest and specific
  Vague claims ("best product") replaced with honest, specific ones
  FAIL = obviously puffery without any specificity
```

**Gate 2 Verdict:** PASS / FAIL

## 4. GATE 3: RISK & LEGAL

**Purpose:** Verify no legal or reputational risk that could harm the company.

```
MANDATORY CHECKS — ALL MUST PASS:

□ G3.1 — No income/results claims without required disclaimers
  Any earnings claims have proof and "results not typical" language
  FAIL = specific income/results claim without disclaimer → HARD STOP + ESCALATE

□ G3.2 — No copyrighted material used without license
  All images are licensed (Unsplash, Pexels, purchased, or original)
  Statistics sourced or original
  FAIL = identifiable copyright violation → STOP + ESCALATE

□ G3.3 — No false claims about product capabilities
  Every feature/benefit claim is true and deliverable
  FAIL = claim that cannot be substantiated → STOP

□ G3.4 — No private information disclosed
  No customer data, private communications, or personal info
  FAIL = any identifiable private information → HARD STOP + ESCALATE

□ G3.5 — Platform TOS compliant
  Content complies with where it's being published
  FAIL = known TOS violation → STOP

□ G3.6 — CAN-SPAM / legal email requirements met (for emails only)
  Physical address, unsubscribe link, accurate from name
  FAIL = missing any required legal element → STOP
```

**Gate 3 Verdict:** PASS / FAIL
**Any HARD STOP = Escalate to Hutch before doing anything else.**

## 5. GATE 4: TECHNICAL INTEGRITY

**Purpose:** Verify all technical elements work correctly.

```
MANDATORY CHECKS — ALL MUST PASS:

□ G4.1 — Primary file/link tested and working
  PDF opens without error | URL loads correct page | Download works
  FAIL = any technical failure on primary asset → STOP

□ G4.2 — Purchase/conversion flow tested
  For products: test purchase completed successfully
  For landing pages: form submissions work
  For email: test email received by test address
  FAIL = flow broken at any step → STOP

□ G4.3 — Mobile rendering verified
  Content displays correctly on mobile screen size
  Images load, text readable, CTA accessible
  FAIL = broken mobile experience → STOP (lower priority, can fix quickly)

□ G4.4 — File format and size compliant
  Format accepted by target platform
  File size within platform limits
  FAIL = file rejected by platform → STOP

□ G4.5 — No broken internal links or references
  All links in the asset tested and working
  All cross-references (page numbers, section titles) accurate
  FAIL = broken link to primary destination → STOP
  (Minor internal cross-reference issues = flag but don't block)
```

**Gate 4 Verdict:** PASS / FAIL

## 6. GATE 5: COMMERCIAL READINESS

**Purpose:** Ensure the asset is set up correctly on the platform to convert.

```
MANDATORY CHECKS — ALL MUST PASS:

□ G5.1 — Price is set correctly and intentional
  Not $0.00, not $0.01, not placeholder amount
  Matches pricing strategy for this product category
  FAIL = any obviously wrong price → STOP

□ G5.2 — Platform configuration complete
  Category set | Tags set | All required fields filled
  FAIL = incomplete platform setup → STOP

□ G5.3 — Thank-you / success page configured
  Post-purchase redirect works
  Thank-you message is correct and complete
  FAIL = broken post-purchase experience → STOP

□ G5.4 — Asset is in the correct published state
  Set to "publish" not "draft" (when ready to go live)
  Visibility settings correct (public vs. unlisted)
  FAIL = wrong publication state → note but can fix at publish time

□ G5.5 — Sales notifications configured (for products)
  Creator will receive notification of new sales
  Analytics tracking active (where applicable)
  FAIL = won't know when sales happen → fix but not blocking
```

**Gate 5 Verdict:** PASS / FAIL

## 7. GATE WAIVER PROTOCOL

Certain gate items can be waived by founder approval. Here's how:

### What CAN be waived:
- G2.3 (visual quality) — if intentionally rough aesthetic
- G4.3 (mobile rendering) — if proven mobile traffic is < 5% for this channel
- G5.4 (publication state) — set correctly at publish time
- G5.5 (notifications) — low priority, can configure after launch

### What CANNOT be waived (ever):
- G1.1 (placeholders) — non-negotiable
- G3.1 (income claims) — legal risk
- G3.2 (copyright) — legal risk
- G3.4 (private info) — legal risk
- G4.1 (primary file works) — product integrity
- G5.1 (correct price) — commercial integrity

### Waiver Process:
1. Sentinel documents the specific gate item being waived
2. Specific reason for waiver documented
3. Hutch verbal or written approval
4. Waiver logged with date and reason

## 8. LAUNCH GATE BY ASSET TYPE

### New Digital Product (Gumroad)
Required gates: G1 + G2 + G3 + G4 + G5 (all five, full review)
Estimated review time: 15-20 minutes

### Email Campaign
Required gates: G1 + G2 + G3 + G4 (Gate 5 = N/A)
Estimated review time: 10-15 minutes

### Social Media Post
Required gates: G1 (partial) + G2 + G3 + G4 (partial)
Estimated review time: 3-5 minutes

### KDP Submission
Required gates: G1 + G2 + G3 + G4 (full KDP specs) + G5
Estimated review time: 30-45 minutes (includes file verification)

### Product Update (Price Change)
Required gates: G5.1 only
Estimated review time: 2 minutes

### Product Update (Content)
Required gates: G1 + G4 + relevant G3 checks
Estimated review time: 10 minutes
