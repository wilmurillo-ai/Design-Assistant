## 1. THE PRE-PUBLISH PROTOCOL

### When to Run This Review
Run a pre-publish review on EVERY asset before it goes public:
- Digital products (before listing on Gumroad/KDP)
- Email campaigns (before sending to list)
- Social media posts (before publishing)
- Landing pages (before going live)
- Blog posts / articles (before publishing)
- Product description updates
- Price changes
- Any public-facing documentation

### Who Runs It
Sentinel Agent runs all pre-publish reviews. No asset goes live without Sentinel sign-off.
Exception: internal notes, drafts not near publishing.

### The Review Standard
You are not looking for perfection — you are looking for anything that would:
1. Damage trust with the buyer/audience
2. Create legal or reputational risk
3. Confuse the reader or buyer
4. Misrepresent the product or company
5. Technically fail (broken links, missing files, wrong prices)

## 2. UNIVERSAL PRE-PUBLISH CHECKLIST

Run this for EVERY asset type:

### Identity & Attribution
- [ ] Creator/author attribution is correct (Ten Life Creatives, or specific brand)
- [ ] Year/date is current (not a previous year's template)
- [ ] Version number is correct (if versioned asset)
- [ ] Contact information is accurate and functional

### Content Quality
- [ ] No placeholder text remaining ("Lorem ipsum", "TBD", "INSERT X HERE", "[EXAMPLE]")
- [ ] No broken formatting (orphaned bullet points, unmatched brackets, unclosed tags)
- [ ] Headline/title is final (not "Draft Title" or "Working Title")
- [ ] All referenced materials/links actually exist
- [ ] Numbers and statistics are accurate and sourced
- [ ] No inconsistent terminology (calling the same thing by two different names)

### Brand & Voice
- [ ] Tone is consistent with our brand voice (direct, authentic, builder energy)
- [ ] No corporate jargon or overly formal language (we're not a Fortune 500)
- [ ] No AI-obvious language ("it is worth noting", "delve into", "comprehensive guide")
- [ ] Claims are specific and honest (no "world's best" without evidence)
- [ ] Faith references (if any) are authentic, not performative

### Technical Integrity
- [ ] All links tested and working (including purchase/download links)
- [ ] File opens without errors on a clean device
- [ ] File format is correct for the destination platform
- [ ] File size is within platform limits
- [ ] Images display correctly (no broken images, correct aspect ratios)

### Legal & Risk
- [ ] No claims that could constitute false advertising
- [ ] No unauthorized use of third-party trademarks, copyrights, or images
- [ ] No personal information published without consent
- [ ] No promises that cannot be kept
- [ ] Refund policy accurately represented (if mentioned)

### Final Mechanics
- [ ] Price is intentional and matches our pricing strategy
- [ ] Publish date/time is set correctly (not accidentally in the past)
- [ ] Category/tags are appropriate
- [ ] Privacy settings are correct (public vs. private vs. unlisted)

## 3. ASSET-SPECIFIC CHECKLISTS

### Digital Product (Gumroad)
```
PRODUCT FILE:
- [ ] PDF opens without password or errors
- [ ] All pages load correctly
- [ ] Images are sharp (300 DPI for print, 150+ DPI for screen)
- [ ] No blank pages (unless intentional journal pages)
- [ ] Page count matches what's advertised
- [ ] Table of contents links work (if included)
- [ ] Cover page looks professional
- [ ] File is the final version (not a draft or WIP)
- [ ] File named correctly: [Product-Name]-v1.0.pdf (no spaces)

LISTING:
- [ ] Title is compelling and specific (outcome + method)
- [ ] Description is 200+ words
- [ ] First 50 words sell the product (visible before "read more")
- [ ] Price is set intentionally
- [ ] Cover image is 1280×720px or larger
- [ ] Category is correct
- [ ] Tags set (3+ minimum)
- [ ] Thank-you page URL set correctly

PURCHASE FLOW:
- [ ] Make a test purchase (on Gumroad sandbox or live)
- [ ] Confirmation email received
- [ ] Download link works
- [ ] Downloaded file matches what was uploaded
```

### Email Campaign
```
CONTENT:
- [ ] Subject line is compelling (tested in inbox simulator)
- [ ] Preview text complements subject (not cut off awkwardly)
- [ ] Opening line is strong (not "Hey [FIRST_NAME]!")
- [ ] CTA is clear and singular (one main action)
- [ ] CTA link works and goes to correct page
- [ ] All personalization tokens filled or have fallbacks
- [ ] Unsubscribe link present and functional
- [ ] Footer has physical address (CAN-SPAM requirement)

RENDERING:
- [ ] Preview on mobile (most opens are mobile)
- [ ] Preview on desktop
- [ ] Images display correctly with images disabled
- [ ] Alt text on all images
- [ ] No broken HTML tags

SEGMENTATION:
- [ ] Sending to correct list/segment
- [ ] Not double-sending to anyone
- [ ] Suppression list applied (unsubscribes, complaints)
- [ ] Send time appropriate for recipient timezone
```

### Social Media Post
```
CONTENT:
- [ ] Post makes sense without any context (standalone)
- [ ] Hashtags are relevant (not spam-y)
- [ ] Tag (@mentions) are intentional and correct
- [ ] Any links are shortened and working
- [ ] No typos in the first 3 words (most visible)

MEDIA:
- [ ] Images are correct aspect ratio for platform
  (Twitter/X: 16:9 or 1:1 | Instagram: 1:1 or 4:5 | LinkedIn: 1.91:1)
- [ ] Images are not blurry or pixelated
- [ ] No watermarks from other platforms
- [ ] Text in images is readable on mobile

RISK:
- [ ] Post cannot be misread as political, offensive, or divisive
- [ ] No info that should be private
- [ ] No competitors named negatively
- [ ] Complies with platform community guidelines
```

### KDP Book Submission
```
INTERIOR:
- [ ] Page size exact match to selected trim size
- [ ] Margins meet KDP minimums for page count
- [ ] All fonts embedded
- [ ] No RGB colors (B&W = grayscale only)
- [ ] Page count is even
- [ ] First page is right-hand page (odd page number)
- [ ] No content in margin/bleed area
- [ ] Images 300 DPI minimum
- [ ] File size under 650MB

COVER:
- [ ] Dimensions match KDP cover calculator output exactly
- [ ] 0.125" bleed on all sides
- [ ] Critical content outside safe zone (0.25" from trim)
- [ ] Spine text centered and readable
- [ ] Author name and title on cover AND spine
- [ ] Barcode area clear on back cover
- [ ] CMYK color mode
- [ ] 300 DPI

METADATA:
- [ ] Title matches cover exactly
- [ ] Description is compelling (500-1500 words)
- [ ] Keywords researched and set (7 keywords max)
- [ ] Categories appropriate and strategic
- [ ] Author bio complete
- [ ] Price set (print list price + royalty calculation verified)
```

## 4. THE QUICK SCAN (2-MINUTE REVIEW)

When you need a rapid quality check (not a full review):

1. **Read the first line** — Is it free of typos and compelling?
2. **Check for placeholders** — Scan for [brackets], TBD, lorem ipsum
3. **Verify the key link** — Click the main CTA/download link
4. **Check the price** — Is it what we intend?
5. **Test on mobile** — Does it display correctly?

If all 5 pass → can proceed with standard review
If any fail → stop, do full review before shipping

## 5. THE DEEP REVIEW (15-Minute Review)

For high-stakes assets (product launches, email campaigns to full list):

**Minutes 0-3: First Impressions**
Read without stopping. Note anything that felt off. Don't fix yet — just observe.

**Minutes 3-7: Checklist Execution**
Run through the universal pre-publish checklist systematically. Check every box.

**Minutes 7-10: The Stranger Test**
Pretend you've never heard of this product or company. Does this make complete sense?
- Would you understand what you're buying?
- Is the value clear?
- Do you trust the seller?

**Minutes 10-13: Risk Pass**
Run through the risk assessment checklist (see risk-assessment.md).
Look specifically for: legal claims, factual errors, anything that could go viral negatively.

**Minutes 13-15: Technical Verification**
Test every link. Download the file. Check on mobile. Verify price.

## 6. COMMON LAST-MINUTE FAILURES

These are the failures that slip through most often. Check these explicitly:

1. **Stale year in copyright** — "© 2023 Ten Life Creatives" when it's 2024
2. **Wrong product name** — Template reuse with old product name not updated
3. **Placeholder price** — $XX.XX or $0.00 still in place
4. **Draft in filename** — "FamliClaw-DRAFT-v3-FINAL-edited.pdf" instead of clean name
5. **Sandbox link** — Test/sandbox purchase link instead of live link
6. **Missing alt text** — Images without alt text (accessibility + SEO issue)
7. **Wrong template variable** — {{first_name}} or [[NAME]] still in email
8. **Timezone-wrong send time** — 9 AM UTC instead of 9 AM Mountain Time
9. **Wrong file version** — Old version of product file uploaded (check file dates)
10. **Broken table formatting** — Table looks fine in editor but breaks on publish

## 7. SIGN-OFF PROCEDURE

### Standard Sign-Off
After completing the review and all issues are resolved:

```
✅ PRE-PUBLISH REVIEW COMPLETE

Asset: [Name]
Reviewed by: Sentinel Agent
Date: [Date]
Verdict: APPROVED

Checklist: All items passed
Issues found: None / [X resolved]
Test purchase/preview: Completed
Ready to publish: YES
```

### Conditional Sign-Off
If minor issues remain but asset is safe to publish:

```
⚠️ CONDITIONAL APPROVAL

Asset: [Name]
Verdict: APPROVED with follow-up required

May publish: YES
Condition: [Specific issue] must be fixed within [timeframe]
Assigned to: [Agent/Role]
Follow-up due: [Date]
```

### Rejection Sign-Off
If asset cannot be published:

```
❌ PUBLICATION BLOCKED

Asset: [Name]
Verdict: REJECTED

Blocked by: [List of blocking issues]
Required action: [Specific fixes needed]
Re-review required: YES
Estimated time to resolve: [Estimate]
```
