# Bottleneck Analysis — Reference Guide

Identifying what's slowing the company down, the Theory of Constraints applied
to a founder-led business, and the interventions that unlock momentum.

---

## 1. THE THEORY OF CONSTRAINTS

### Goldratt's Core Insight
Every system has exactly one bottleneck at any given time.
The throughput of the system = the throughput of the bottleneck.
Making anything OTHER than the bottleneck faster doesn't improve the system.

Application: Before optimizing anything, identify the bottleneck first.

---

## 2. IDENTIFYING THE BOTTLENECK

### The Five Bottleneck Questions
```
Q1: What is the ONE thing, if it happened, that would most accelerate progress?
  [This is often the bottleneck]

Q2: Where do things pile up waiting? What creates the biggest backlog?
  [The point of pile-up is usually just after the bottleneck]

Q3: What are we working around because one thing isn't working?
  [Workarounds reveal bottlenecks]

Q4: If we had 3 more hours per week to spend anywhere, where would it have the most impact?
  [The answer is usually the bottleneck]

Q5: What's the single thing we'd solve first if we could solve anything?
  [Direct bottleneck identification]
```

### Common TLC Bottlenecks by Business Area
```
GND BOTTLENECKS:
  Common: "Not enough outreach" → Solution: More emails sent per day
  Common: "No follow-up system" → Solution: Systematic follow-up sequence
  Common: "Poor conversion in proposals" → Solution: Better proposal, more specificity

DIGITAL PRODUCTS BOTTLENECKS:
  Common: "Products not found" → Solution: Platform SEO, more platforms, marketing
  Common: "People find but don't buy" → Solution: Conversion copy, price test
  Common: "Builds take too long" → Solution: Template system, faster generation tools

PRAYFUL BOTTLENECKS:
  Common: "Unclear product definition" → Solution: Define MVP scope exactly
  Common: "No users to test with" → Solution: Beta recruiting before build complete
```

## 3. THE BOTTLENECK RESOLUTION PROTOCOL

```
STEP 1: NAME THE BOTTLENECK
  Be specific: not "marketing" but "we don't have a consistent lead generation system"
  Not "Prayful is behind" but "we haven't defined what the MVP actually is"

STEP 2: TEMPORARILY EXPLOIT
  Get maximum output from current constraint without major changes
  Example bottleneck "not enough cold emails": 
  → Before building a complex system, just send more emails

STEP 3: SUBORDINATE EVERYTHING ELSE
  Stop optimizing non-bottlenecks (waste of time)
  Redirect resources to the bottleneck

STEP 4: ELEVATE IF NEEDED
  If exploitation didn't resolve it: invest to improve the constraint
  Add resources, change the approach, solve it fundamentally

STEP 5: FIND THE NEW BOTTLENECK
  After resolving one, a new bottleneck will appear
  Go back to Step 1
```
