# 一文读懂“见、知、用、安”数据管理之道

原文链接：https://mp.weixin.qq.com/s/d6bYkOiz2yqy8FtgOLDfYA?token=1267526467&lang=zh_CN

去年春天，我写了一篇《面对数字化实施困境，CIO应当吸取的五大经验教训》，在同行中产生很多共鸣。在数字化实施过程中，很多企业会遇到文章中总结的五大问题。其中，困扰数字化管理者的主要问题是：项目投入大，但业务价值不清晰，最终领导不认可，员工无感知。这个问题对于数据中台类项目尤为明显。

如何解决这个问题？我建议从谋（规划）、道（方法）和器（工具）这三个方面思考。

## 1 数据管理之谋

总的来说，数字化实施过程中这些问题的出现，往往是因为以下三点做得不够：

- 可落地的总体规划
- 日常可遵循的架构设计
- 完善的 IT 管理体系

推进数字化转型，我们遵循四步路径：

1. 战略驱动：从明确的业务战略出发，制定数字化转型的战略目标和原则，聚焦业务价值。
2. 顶层规划：设计总体架构蓝图和管理体系，确定项目举措和建设节奏，建立确保架构遵循的机制。
3. 变革项目：按照管理变革的方式，推进数字化转型项目，强化 PMO 的作用，做好内外部沟通。
4. 产品迭代：项目完成后还要持续迭代，用产品思维管理数字化服务，注重用户体验的持续优化。

第二点“顶层规划”尤为重要。在顶层规划中，我们主要做两件事：一是架构设计，二是管理体系。

架构设计包括四个方面：业务架构、数据架构、应用架构和技术架构。其中，有两个关键任务：

- 流程优化：梳理端到端业务流程，找出增值活动，提升企业能力。
- 数据治理：建立数据管理体系，形成数据资产，提供统一的数据服务。

这两者，要同步设计和实施，互相融合。关于流程，我在《大卫说流程》系列文章中有很多阐述。本文重点讨论数据这部分。

## 2 数据管理之道

数据的利用，有很多困难：

1. 我们要知道都有哪些数据，分别在哪里？数据资源有没有清晰的整体视图。
2. 我们要知道谁在管理这些数据，管理的标准如何，数据的质量怎么样？
3. 数据也不能只被技术部门了解，业务人员怎么使用数据，管理层如何能看见数据的价值？
4. 我们把数据采集、汇聚、集中管理起来，那么数据的安全管控又成为一个挑战。

总结起来，我们要解决四大数据问题：数据不可见、不可知、不可用、不可安。于是，我们用“见、知、用、安”数据管理之道，体系化解决这些问题。

举一个实际案例说明：

- “见”数据：通过咨询规划摸清数据家底，构建数据管理制度体系，形成数据资源目录，促进跨部门数据融合贯通。
- “知”数据：建设统一数据平台，建立数据标准，确定各部门责任人，数据质量形成闭环管理，推进数据模型与算法建设。
- “用”数据：用数据驱动智能化运营，建立经营分析指标体系，建设可视化的数字驾驶舱、经营决策监督分析系统。
- “安”数据：制定数据合规办理办法，对数据分类分级，评估数据合规风险，建立数据全生命周期管理。

遵循这套方法，这家企业从顶层设计开始，盘点、治理、分析和利用跨部门的数据资产。虽然数据建设项目周期长，但是分阶段实施，目标清晰，每阶段必有成果。最终，企业实现数字化经营管理，提升了决策效率和管理能力，取得非常好的落地效果。

方法对了，还要有合适的技术工具。在上述案例当中，企业建设了统一的数据平台。在平台上对数据进行治理，形成数据资产目录和各个主题库。部署使用 BI 软件，基于主题数据和指标数据，构建了数字驾驶舱等应用。

## 3 数据管理之器

那么，如何建数据平台呢？

我们知道 DAMA 数据管理框架，其中著名的车轮图，将数据治理置为中心，数据架构、数据质量等 10 个知识域环绕周围。彼得·艾肯设计的金字塔图，则将 DAMA 的 11 个知识域，与数据管理的四个阶段结合起来，展示企业为了达成业务战略，管理和利用数据的演进过程。借鉴艾肯的金字塔图，我做了一些调整，以更好地展现演进过程。

阶段 1：数据接入  
企业对数据的管理，起源于业务系统使用数据库来存储和操作数据。大数据时代，则需要专门处理大数据的存储和计算组件。

阶段 2：数据治理  
当这样的业务系统越来越多，企业的数据规模亦越大。此时需要对各种数据进行融合治理，实施整体的数据安全管控。

阶段 3：数据服务  
治理好的数据，进入到数据仓库，通过 BI 软件可以直观展现数据洞察。有些企业还会建设主数据管理系统。

阶段 4：数据应用  
最上层是对数据的高级应用，包括使用统计分析、机器学习、大模型等技术，结合实际场景，深度挖掘数据的价值。

在这个过程中，我们需要多种技术和工具来提供支撑。尤其是大数据领域，涉及的功能组件非常多。企业在构建数据平台能力时，可以按照以下三个层面考虑：

1. 大数据底座：把数据从数据源采集、汇聚、存储起来，并提供计算处理能力。目前，基于 Hadoop 等开放生态的各种组件已经很成熟，重点在于大数据平台集群的规划设计和组件优化。
2. 数据治理平台：对多源异构的数据进行集成和融合治理，形成可复用的数据资产，对外提供数据服务。重点关注数据集成能力和全面的治理能力。
3. 数据应用平台：提供数据可视化，算法与模型库，以及低代码开发能力，用来快速构建数据应用。重点在于数据驱动的应用构建能力。

数据平台建设不要贪大求全，应结合应用场景和需求痛点。我在上一篇文章《能源行业数据治理的五点观察与思考》中，提出“一横一纵、以用促治”的方法，可供参考。简而言之，建议企业在统一的架构设计的前提下，找出纵向细分场景，打通数据采集治理应用的链路，确保数据价值彰显。在这个过程中，同步建设和增强数据管理平台的能力。
