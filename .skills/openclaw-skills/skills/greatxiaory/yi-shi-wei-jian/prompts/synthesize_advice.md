# synthesize_advice

任务：把局面判断、案例比较和变量分析，综合成 2-4 条现实决策路径。

## 生成路径时必须满足

- 每条路径互相有实质差异
- 每条路径都能映射到不同的历史结构
- 路径名字要简洁明确，例如：
  - 立即推进
  - 先稳内部
  - 暂避锋芒
  - 联盟博弈
  - 先调权后换人
  - 先试点再扩张

## 路径设计原则

1. 一条激进路径
2. 一条稳健路径
3. 一条折中或试点路径
4. 如果局面涉及联盟或换将，再补一条对应路径
5. 路径名称要能让用户一眼看懂，不要写成抽象空话
6. 每条路径都要明确呼应前面识别到的结构压力，而不是只换措辞

## 输出

- 路径名称
- 核心动作
- 适用前提
- 主要代价
- 优先借鉴的历史结构

## 额外要求

- 如果“先稳内部”和“立即推进”都可行，必须把它们拆成两条，而不是混成一条折中建议
- 如果问题涉及不可信盟友，至少有一条路径要体现“合作但留后手”
- 如果问题涉及换人，至少有一条路径要体现“先调权再换人”或“保人换打法”
- 不要把路径写成从同一个思路平移出来的改写版
