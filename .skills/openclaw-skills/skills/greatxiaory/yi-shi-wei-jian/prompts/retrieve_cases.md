# retrieve_cases

任务：从案例库中选择 2-4 个与当前问题结构最接近的中国历史案例。

## 检索原则

1. 优先看结构语义，不优先看知名度，也不优先看字面词是否一致。
2. 重点对齐：
   - situation_tags
   - objective
   - hidden_constraints
   - chosen_action
   - key_reasons
   - modern_analogy_keywords 仅作辅助
3. 选出的案例要尽量形成对照：
   - 至少一个成功案例
   - 如有必要，加入一个失败或代价过高的案例
   - 如果问题是改革、联盟或控制权问题，优先加入一个“代价被低估”的反例
4. 如果多个案例都像，优先保留约束条件更像的那个。
5. 如果问题里出现“合作方翻脸、不完全信任、表面支持实则拖延、先削权再换人、先守后攻”等现代语句，不要只找同词案例，要优先匹配具有相同结构约束的案例。

## 每个案例必须输出

- 事件
- 核心局面
- 关键决策
- 成败原因
- 为什么像当前问题

## 避免事项

- 不要把“著名”误当成“相似”
- 不要只罗列故事，不解释对应关系
- 不要把历史结果直接当成现实预言
- 不要把“同属联盟/改革/换将”当成已经足够，相似点必须落到约束条件
- 不要把检索退化成关键词列表对碰
