# classify_situation

任务：把用户的现实问题归类为一个主局面和一到两个次级局面。

## 操作步骤

1. 先做结构语义拆解，不要先盯字面词。
2. 至少抽取这些维度：目标、资源强弱、内部摩擦、外部压力、联盟依赖、信任风险、时间窗口、合法性约束、执行阻力、行动取向。
2. 在以下类别中选主局面：
   - 以弱对强
   - 内部冲突
   - 改革推进
   - 联盟不稳
   - 守攻抉择
   - 权力控制
   - 用人换将
3. 如果用户信息不足，最多提出 3 个澄清问题。
4. 如果不提问，必须明确你使用了哪些默认假设。
5. 如果用户只是随口说“我该怎么办”，优先把问题翻译成结构化局面，而不是先展开泛泛建议。

## 分类标准

- `以弱对强`：资源、地位、份额、正式权威明显弱于对手
- `内部冲突`：团队、股东、派系或关键执行层出现对抗
- `改革推进`：制度、流程、激励、组织结构需要改变
- `联盟不稳`：合作能带来收益，但承诺不牢、目标不一
- `守攻抉择`：要决定先守基本盘还是主动出击
- `权力控制`：控制权、任命权、继承、关键中枢归属
- `用人换将`：要不要换人、提拔、空降、削权、重配岗位

## 触发表达提示

下列表达通常意味着应该触发本 skill：

- “先稳内部还是先推进”
- “要不要换掉负责人”
- “合作方不太可信，还要不要联手”
- “我比对手弱很多，该怎么破局”
- “该守基本盘还是主动扩张”
- “控制权被架空了怎么办”

如果问题更像纯历史问答，就不要勉强套进决策辅助模式。

## 额外要求

- 不要把“出现了某个词”当成分类完成
- 要解释这个问题在结构上为什么更像某一类局面
- 如果一个问题同时有“推进”和“先稳住”的张力，优先把它视为真实的守攻抉择，而不是简单按单个词归类

## 输出

- 主局面
- 次局面
- 3 条以内判断依据
- 关键缺口或默认假设
- 如果有歧义，说明最容易混淆的另一个局面是什么
