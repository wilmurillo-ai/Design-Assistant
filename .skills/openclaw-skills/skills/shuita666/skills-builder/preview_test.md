# Skill Trigger Test Phrases

After installing your new Skill, use the phrases below to verify it activates correctly. Copy and paste them into a new conversation with Claude — if the Skill is working, Claude should respond according to its defined rules rather than its default behavior.

---

## How to Use This Document

1. Install your Skill via Claude's Settings → Customize
2. Start a **new conversation** (Skills don't apply retroactively)
3. Try the test phrases below one at a time
4. If Claude doesn't seem to follow the Skill's rules, check that the description field contains matching keywords

---

## Universal Trigger Phrases
*(These should activate any Skill created with skill-builder)*

- "I want to create a Skill"
- "Help me turn this workflow into a Skill"
- "I want you to remember how I like things done"
- "Can you customize how you respond to me?"
- "Help me make a reusable template for how you work"
- "I always have to re-explain this to you — can we save it?"

---

## How to Test a Specific Skill You Just Created

Once your Skill is installed, test it with a **real task** — not a question about the Skill itself.

### Example: If you created a writing-style Skill
❌ Don't test with: "Do you have my writing Skill activated?"
✅ Do test with: "Help me write an intro paragraph about climate change."
Then check: Did Claude follow your tone, format, and rules without being told?

### Example: If you created a document-editing Skill
❌ Don't test with: "What are the rules in my editing Skill?"
✅ Do test with: "Please edit this paragraph: [paste your text]"
Then check: Did Claude apply your terminology, citation format, and style rules automatically?

---

## What Good Activation Looks Like

Your Skill is working correctly if Claude:
- Follows your behavioral rules **without you restating them**
- Uses the tone and format you specified
- Avoids the things you put in the Off-Limits section
- Produces output that matches your ideal example

## What to Do If It Doesn't Trigger

If Claude ignores the Skill and responds in its default way:
1. Check that your trigger phrases match keywords in the Skill's `description` field
2. Make your request more specific — e.g., add the exact words from the description
3. Consider updating the description to include more natural trigger phrases
4. Start a fresh conversation (context from old chats can interfere)
