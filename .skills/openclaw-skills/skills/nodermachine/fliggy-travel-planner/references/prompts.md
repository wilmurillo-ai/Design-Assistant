# 旅行信息提取 Prompt 模板

用于从小红书攻略笔记中提取结构化旅行信息的大模型 Prompt。

---

## 📋 基础提取 Prompt

```
你是一位专业的旅行规划助手。请从以下小红书攻略笔记中提取关键旅行信息。

## 输入数据

笔记标题：{title}
笔记作者：{author}
点赞数：{likes} | 收藏数：{collects} | 评论数：{comments}
笔记内容：
{content}

## 提取要求

请提取以下字段（如果内容中没有提到，填写"待补充"）：

1. **目的地**：具体的城市/地区名称
2. **最佳季节**：推荐游玩的月份或季节
3. **推荐天数**：建议游玩几天
4. **预算范围**：人均预算（不含机票），格式如"2000-4000 元"
5. **必去景点**：3-8 个核心景点，按推荐度排序
6. **美食推荐**：3-6 种当地特色美食
7. **住宿建议**：推荐住宿区域或具体酒店
8. **交通建议**：当地交通方式（租车/公交/打车等）
9. **注意事项**：2-5 条重要提示（避雷、穿搭、证件等）
10. **推荐行程**：如果笔记中有详细行程，按天提取

## 输出格式

请严格按照以下 JSON 格式输出（不要包含其他文字）：

{
  "destination": "目的地名称",
  "best_season": "最佳季节",
  "recommended_days": "推荐天数",
  "budget_range": "预算范围",
  "highlights": ["景点 1", "景点 2", "景点 3"],
  "food": ["美食 1", "美食 2", "美食 3"],
  "accommodation": ["住宿建议 1", "住宿建议 2"],
  "transportation": "交通建议",
  "tips": ["注意事项 1", "注意事项 2"],
  "itinerary": ["Day 1: 行程", "Day 2: 行程"]
}
```

## 🎯 多笔记整合 Prompt

当有多篇笔记时，使用整合 Prompt：

```
你是一位专业的旅行规划助手。请综合分析以下 {note_count} 篇小红书攻略笔记，
提取最全面、最准确的旅行信息。

## 笔记列表

{notes_json}

## 分析要求

1. **交叉验证**：如果多篇笔记提到相同景点/美食，优先保留
2. **热度加权**：点赞/收藏数高的笔记内容权重更高
3. **时效性**：优先采用最近发布的笔记信息
4. **冲突处理**：如果有冲突信息（如不同预算），取平均值或标注范围
5. **去重合并**：相似内容合并，避免重复

## 输出格式

{
  "destination": "目的地名称",
  "best_season": "最佳季节",
  "recommended_days": "推荐天数",
  "budget_range": "预算范围（综合多篇）",
  "highlights": [
    {"name": "景点 1", "mentions": 5, "avg_rating": 4.8},
    {"name": "景点 2", "mentions": 3, "avg_rating": 4.5}
  ],
  "food": [
    {"name": "美食 1", "mentions": 4},
    {"name": "美食 2", "mentions": 3}
  ],
  "accommodation": [
    {"area": "区域 1", "reason": "推荐理由"},
    {"area": "区域 2", "reason": "推荐理由"}
  ],
  "transportation": "综合交通建议",
  "tips": ["最重要的注意事项 1", "注意事项 2"],
  "itinerary": ["Day 1: 综合最优行程", "Day 2: ..."],
  "source_notes": [
    {"title": "笔记 1 标题", "likes": 1000, "url": "链接"},
    {"title": "笔记 2 标题", "likes": 800, "url": "链接"}
  ]
}
```

## ✈️ 行程规划 Prompt

根据提取的信息生成详细行程：

```
你是一位专业的旅行规划师。请根据以下信息生成详细的 {days}天{destination}旅行行程。

## 基础信息

- 目的地：{destination}
- 游玩天数：{days}天
- 最佳季节：{best_season}
- 必去景点：{highlights}
- 住宿区域：{accommodation}
- 交通方式：{transportation}

## 规划要求

1. **合理分配**：每天安排 3-5 个景点，避免过于紧凑
2. **路线优化**：相邻景点安排在同一天，减少路途时间
3. **劳逸结合**：每天安排休息时间，不要全程赶路
4. **餐饮安排**：每餐推荐附近特色美食
5. **弹性时间**：每天保留 1-2 小时自由活动时间
6. **实用信息**：标注每个景点的建议游玩时长、门票价格、开放时间

## 输出格式

{
  "day_1": {
    "theme": "第一天主题（如：古城初探）",
    "schedule": [
      {
        "time": "09:00-11:00",
        "activity": "景点 1",
        "duration": "2 小时",
        "tips": "游玩提示"
      },
      {
        "time": "11:30-13:00",
        "activity": "午餐（推荐餐厅）",
        "duration": "1.5 小时",
        "tips": "必点菜品"
      }
    ],
    "summary": "第一天总结"
  },
  ...
}
```

## 💰 预算估算 Prompt

```
请根据以下信息估算{destination}{days}天旅行的详细预算。

## 基础信息

- 目的地：{destination}
- 天数：{days}天
- 出发地：{from_city}
- 出行人数：{people}人
- 机票价格：¥{flight_price}/人（往返）

## 估算项目

1. **交通**
   - 机票：已知
   - 当地交通：租车/打车/公交

2. **住宿**
   - 酒店/民宿：{accommodation_level}档次
   - 单价：{price_range}元/晚

3. **餐饮**
   - 每日餐标：{meal_budget}元/人/天

4. **门票**
   - 各景点门票总和

5. **其他**
   - 购物、纪念品、应急等

## 输出格式

{
  "total_per_person": "人均总预算",
  "total_for_all": "总预算（所有人）",
  "breakdown": {
    "flights": "机票小计",
    "accommodation": "住宿小计",
    "food": "餐饮小计",
    "tickets": "门票小计",
    "transportation": "当地交通小计",
    "others": "其他小计"
  },
  "daily_average": "日均消费"
}
```

## 🎨 文案优化 Prompt

生成小红书风格的旅行文案：

```
你是一位小红书旅行博主。请将以下旅行信息改写成小红书风格的种草文案。

## 原始信息

{travel_info}

## 文案要求

1. **标题**：20 字以内，包含数字、emoji、吸引眼球的关键词
   - 示例："大理 3 天 2 夜｜人均 2k+💰超全攻略不踩雷"
   
2. **开头钩子**：1-2 句，制造共鸣或悬念
   - 示例："刚从大理回来！这份攻略一定要收好～"
   
3. **正文结构**：
   - 分段清晰，每段 3-5 行
   - 使用 emoji 点缀（但不过量）
   - 重点信息加粗或用符号标注
   
4. **互动引导**：结尾提问或引导收藏
   - 示例："有问题评论区见～记得点赞收藏⭐"
   
5. **话题标签**：5-8 个相关话题
   - 示例：#大理旅行 #云南旅游 #旅行攻略

## 输出格式

标题：{title}

正文：
{content}

话题：{tags}
```

---

## ⚠️ 使用注意事项

1. **温度设置**：信息提取用低温（0.2-0.3），文案创作用高温（0.7-0.8）
2. **最大 token**：设置合理的输出长度限制
3. **重试机制**：JSON 解析失败时自动重试
4. **人工校验**：关键信息（如价格、日期）建议人工确认
