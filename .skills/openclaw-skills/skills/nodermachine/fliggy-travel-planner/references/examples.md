# 使用示例

展示 `xiaohongshu-travel-planner` 技能的实际使用场景。

---

## 📌 示例 1：简单查询

**用户输入：**
```
帮我规划一下去大理的旅行，看下小红书的攻略和机票价格
```

**技能执行流程：**

1. 解析用户意图：目的地=大理，需要攻略 + 机票
2. 询问出发地和日期（如果未提供）
3. 搜索小红书"大理攻略"
4. 提取 Top 10 高赞笔记信息
5. 查询飞猪机票（出发地→大理/昆明）
6. 生成整合报告

**输出示例：**
```markdown
# 🌟 大理旅行规划

## 📖 小红书攻略精选

**最佳季节：** 3-5 月/9-11 月  
**推荐天数：** 4-5 天  
**预算参考：** 2000-4000 元（不含机票）

### 🎯 必去景点
1. 洱海骑行 - 环海西路最美
2. 大理古城 - 晚上更有氛围
3. 苍山索道 - 俯瞰洱海全景
...

### ✈️ 机票信息（飞猪）
**出发地：** 北京  
**目的地：** 大理（昆明转机）
| 去程 | 返程 | 价格 | 预订 |
|------|------|------|------|
| MU5101 08:00 | MU5102 20:00 | ¥1,580 | [立即预订](链接) |
```

## 📌 示例 2：指定日期往返

**用户输入：**
```
计划 4 月 1 日 -4 月 5 日去日本旅游，帮我找攻略和机票
```

**技能执行流程：**

1. 解析参数：
   - 目的地：日本（需 уточнить具体城市）
   - 出发日期：2026-04-01

2. 追问具体城市（东京/大阪/京都等）
3. 搜索"日本 东京 攻略 4 月"
4. 特别关注樱花季相关内容
5. 查询往返机票
6. 生成带樱花季特色的报告

**输出亮点：**
- 樱花观赏地点推荐
- 樱花季特别提示（人流、预订提前量）
- 樱花季价格上浮说明

2. 分别搜索三个目的地攻略
3. 查询到三个目的地的机票（或昆明转机）
4. 生成对比表格

## 📌 示例 7：价格监控

**用户输入：**
```
我想去三亚，帮我盯着机票，低于 1000 元时通知我，顺便看看最新攻略
```

**技能执行流程：**

1. 设置价格监控 cron 任务
2. 每天查询北京 - 三亚机票
3. 价格<1000 元时发送通知
4. 通知中包含最新攻略摘要

**Cron 配置：**
```json
{
  "name": "三亚机票监控",
  "schedule": {"kind": "every", "everyMs": 86400000},
  "payload": {
    "kind": "systemEvent",
    "text": "监控三亚机票价格，低于 1000 元时通知用户"
  }
}
```

## 🎯 最佳实践

### ✅ 推荐做法

1. **提供明确信息**：出发地、日期、预算、人数
2. **说明偏好**：休闲/探险、美食/景点、经济/舒适
3. **特殊需求**：带娃、老人、素食、无障碍等

### ❌ 避免做法

1. **信息过于模糊**："想去玩"（无目的地、无时间）
2. **预算不切实际**：500 元去欧洲
3. **临时抱佛脚**：明天出发查今天的机票
