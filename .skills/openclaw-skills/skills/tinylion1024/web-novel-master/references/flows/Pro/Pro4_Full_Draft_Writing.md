# Professional 模式 - Phase 4：正文撰写

**模式**: Professional 专业模式（作者・完整但不繁琐）
**Phase数**: 5/8

**重要**：全程无需向用户确认，逐章创作直到完成

---

## 4.1 启动检测

1. 读取 `03-写作计划.json`
2. 如有 `status: "in_progress"` → 从该章节继续
3. 如有 `status: "failed"` → 从第一个 failed 章节开始
4. 全部 `pending` → 从第1章开始

---

## 4.2 逐章创作流程

### Step 1: 写前分析

1. 读取 `03-写作计划.json` — 确定待创作章节
2. 读取 `00-大纲.md` — 提取核心事件、爽点、承接、钩子、情绪弧线、伏笔
3. **一致性检查**（非第1章）：读取上章摘要，核对人物状态
4. JSON `status` → `"in_progress"`

### Step 2: 撰写

5. 创建章节文件
6. 基于大纲创作 — 严格按核心事件和爽点
7. 正文 2000-3000 字
   - 前20%有即时爽点
   - 情绪曲线：憋→爽
   - 人物语气：对话符合性格
8. 字数检查

### Step 3: 优化

9. 爽点验证
10. 深度润色：
    - 爽感强化、情绪外放、节奏加快
    - 金句植入、人物语气、每500字推进检查
11. 字数再检查
12. 伏笔检查

### Step 4: 收尾

13. 章节摘要（200-300字）
14. JSON `status` → `"completed"`，填入 `wordCount`

---

## 4.3 串行模式

```
WHILE 存在 pending 章节:
    执行逐章创作流程
    完成后立即认领下一章
所有完成 → 进入 Phase 5
```

---

## 4.4 并行模式

主 Agent 分批（每批5-8章）派生子 Agent：

```
1. 计算批次分配
2. 并行派生子 Agent
3. 子 Agent 执行逐章创作
4. 所有完成 → 进入 Phase 5
```
