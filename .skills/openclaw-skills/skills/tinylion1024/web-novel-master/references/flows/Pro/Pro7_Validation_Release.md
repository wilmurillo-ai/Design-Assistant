# Professional 模式 - Phase 7：校验发布

**模式**: Professional 专业模式（作者・完整但不繁琐）
**Phase数**: 8/8

---

## 7.1 初始化校验

`03-写作计划.json` `status` → `"validating"`

---

## 7.2 逐章校验

对每一章：

1. **文件存在性**
2. **字数检查** — `python scripts/check_chapter_wordcount.py <文件>`
3. **爽点检查** — 有核心爽点？密度达标？
4. **金句检查** — 有让读者想截图的句子？
5. **情绪曲线** — 有压抑→释放弧线？
6. **人物一致性** — 对话语气？状态衔接？
7. **伏笔检查** — 规划回收的伏笔已回收？

---

## 7.3 校验维度

| 维度 | 目标 | 不合格处理 |
|-----|------|----------|
| 字数 | 2000-3000 | 扩充/压缩 |
| 爽点密度 | 每2000字≥1 | 增强描写 |
| 金句 | 每章≥1 | 强化结尾 |
| 开头钩子 | 前20%有爽点 | 重写开头 |
| 情绪曲线 | 憋→爽弧线 | 调整节奏 |
| 人物语气 | 符合性格 | 调整对话 |
| 伏笔回收 | 已规划需兑现 | 补充回收 |

---

## 7.4 自动修复（最多3轮）

不合格章节：
1. `status` → `"failed"`，`retryCount` +1
2. 重新执行逐章创作流程
3. 重写后重新校验
4. 更新 JSON
