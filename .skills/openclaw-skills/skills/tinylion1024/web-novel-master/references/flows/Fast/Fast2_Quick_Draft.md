# Fast 模式 - Phase 2：快速起草

**模式**: Fast 快速模式（爱好者・极简）
**Phase数**: 3/5

---

## 2.1 快速规划

根据核心信息，快速生成：

### 大纲（简版）

```markdown
# [小说名称]

## 核心设定
- 类型：[类型]
- 金手指：[金手指]
- 爽点：[爽点]

## 章节规划（每章一句话）

第1章：[核心爽点开场]
第2章：[冲突升级]
第3章：[首次打脸/突破]
...
```

---

## 2.2 串行写作

**重要**：全程不中断，逐章创作

```
WHILE 有待写章节:
    读取章节规划
    创作本章（2000-3000字）
    前20%必须有即时爽点
    结尾设置钩子
    字数检查
    更新 JSON status → completed
```

### 每章创作步骤

1. **读取规划** — 获取本章核心事件
2. **创作正文** — 2000-3000字，前20%要有爽点
3. **字数检查** — `python scripts/check_chapter_wordcount.py <文件>`
4. **设置钩子** — 结尾留悬念

---

## 2.3 章节数确定

- 用户指定章节数 → 按指定创作
- 未指定 → 创作到有完整闭环的第一个高潮后停止，询问用户是否继续
