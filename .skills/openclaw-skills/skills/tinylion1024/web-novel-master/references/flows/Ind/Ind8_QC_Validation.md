# Industrial 模式 - Phase 8：QC校验

**模式**: Industrial 工业模式（团队・最完整・可流水线）
**Phase数**: 9/10

---

## 8.1 多重校验体系

### 校验层级

| 层级 | 校验内容 | 执行者 |
|------|---------|--------|
| L1 自检 | 字数+爽点 | 写作 Agent |
| L2 互检 | 人物一致性+伏笔 | 主 Agent |
| L3 终检 | 全文节奏+质量评分 | 独立校验 Agent |

---

## 8.2 L1 自检

每章自动执行：

1. **字数检查** — `python scripts/check_chapter_wordcount.py <文件>`
2. **爽点检查** — 本章是否有核心爽点？
3. **金句检查** — 是否有让读者想截图的句子？

**通过标准**：
- 字数 2000-3000
- 有核心爽点
- 有至少1个金句

---

## 8.3 L2 互检

### 人物一致性检查

- [ ] 人物对话语气全程一致？
- [ ] 人物能力值变化有交代？
- [ ] 人物关系进展有铺垫？

### 伏笔系统检查

- [ ] 所有埋设伏笔有回收？
- [ ] 回收逻辑合理？
- [ ] 无断头伏笔？

### 节奏一致性检查

- [ ] 每章情绪弧线完整？
- [ ] 憋→爽节奏符合阶段定位？
- [ ] 无连续3章以上无爽点？

---

## 8.4 L3 终检

### 质量评分

| 维度 | 权重 | 评分 | 说明 |
|------|------|------|------|
| 字数控制 | 15% | /15 | 2000-3000范围 |
| 爽点设计 | 25% | /25 | 爽点是否到位 |
| 开头钩子 | 15% | /15 | 前20%是否吸引 |
| 金句密度 | 10% | /10 | 是否有金句 |
| 节奏控制 | 15% | /15 | 张弛有度 |
| 伏笔系统 | 10% | /10 | 埋设回收完整 |
| 文风统一 | 10% | /10 | AI味清除 |

**总分** = (得分/100) × 100%

**通过标准**：总分 ≥ 70%

---

## 8.5 自动修复

### 修复流程

```
不合格章节 → status: "failed" → retryCount +1
         ↓
    重写 → L1自检 → L2互检 → L3终检
         ↓
    通过 → status: "completed"
         ↓
   失败 → retryCount ≥ 3 → 标记保留 → 人工介入
```

### 修复限制

- 每章最多重写 3 次
- 超过 3 次仍不合格 → 标记 `needs_human_review: true`
