## 9.3 平台发布策略

### 平台选择

| 平台 | 特点 | 适合类型 |
|------|------|---------|
| 起点中文网 | 付费为主，用户成熟 | 玄幻/都市 |
| 番茄小说 | 免费为主，流量大 | 都市/系统流 |
| 飞卢小说 | 更新快，追更强 | 都市/同人 |
| 七猫小说 | 免费+付费 | 大众向 |

### 更新策略

```markdown
## 发布计划

### 前期（1-10章）
- 发布节奏：日更
- 发布时间：每日 12:00 或 20:00
- 诱饵章节：前3章必须最强爽点

### 中期（11-30章）
- 发布节奏：日更/周更
- 质量把控：存稿10章以上

### 后期（31-完本）
- 发布节奏：周更
- 大结局：提前预告
```

## 9.4 运营规划

### 读者互动

- [ ] 建立读者群
- [ ] 收集读者反馈
- [ ] 根据评论调整节奏

### 数据监控

| 指标 | 目标 |
|------|------|
| 点击率 | >15% |
| 追读率 | >30% |
| 完读率 | >20% |
| 留存率 | >40%（7日） |

### 应急处理

| 问题 | 应对 |
|------|------|
| 读者差评 | 分析原因，调整节奏或爽点 |
| 流量下滑 | 增加名场面密度 |
| 抄袭投诉 | 保留创作记录 |

## 9.5 完本规划

### 完本标准

```markdown
## 完本条件

1. 主线剧情完整（主角目标达成）
2. 主要情感线有结局
3. 重要伏笔全部回收
4. 读者满意度 > 60%

## 结局类型
- 【大团圆】主角达成所有目标
- 【悬念式】留有开放式结局
- 【反转式】结局有意外反转
```
