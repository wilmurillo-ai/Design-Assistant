# Industrial 模式 - Phase 6：团队写作

**模式**: Industrial 工业模式（团队・最完整・可流水线）
**Phase数**: 7/10

**重要**：全程无需向用户确认，严格按写作计划执行

---

## 6.1 启动检测

1. 读取 `05-写作计划.json`
2. 确认 `writingMode`：
   - `serial` → 串行模式
   - `subagent-parallel` → 并行模式
   - `agent-teams` → Teams 模式
3. 如有 `status: "in_progress"` → 从该章节继续
4. 如有 `status: "failed"` → 从第一个 failed 章节开始

---

## 6.2 逐章创作标准流程

### Step 1: 读取规划

1. 读取 `05-写作计划.json` — 确定待创作章节
2. 读取 `04-章节详细规划.md` — 获取本章详细规划
3. 读取 `00-大纲.md` — 确认本章在模块中的位置

### Step 2: 一致性检查

**非第1章必须执行**：
- 读取上章摘要
- 核对人物状态、能力值、关系进展
- 检查待回收伏笔是否已埋设

### Step 3: 章节创作

4. 创建章节文件（路径：`chapters/第XX章.md`）
5. 基于详细规划创作
6. 正文 2000-3000 字：
   - 前20%有即时爽点
   - 情绪曲线：憋→爽节奏
   - 人物语气：对话符合性格设定
   - 伏笔埋设/回收（如规划中有）
7. 字数检查

### Step 4: 自我校验

8. 爽点验证 — 确认爽点到位
9. 深度润色 — 爽感强化、情绪外放、金句植入
10. 字数再检查
11. 伏笔确认 — 如规划中有回收，本章是否兑现

### Step 5: 收尾

12. 生成章节摘要（200-300字）
13. 更新 `05-写作计划.json` — `status: "completed"`
14. 伏笔系统更新 — 标记伏笔状态

---

## 6.3 串行模式

主 Agent 逐章创作，全程不中断。

```
WHILE 存在 pending 章节:
    执行标准流程
    完成后立即认领下一章
所有完成 → 进入 Phase 7
```

---

## 6.4 并行模式

主 Agent 分批派生子 Agent。

### 批次分配规则

| 总章数 | 每批章数 | 批次数量 |
|--------|---------|---------|
| ≤30章 | 5-6章 | 5-6批 |
| 31-50章 | 6-8章 | 6-8批 |
| 51-100章 | 8-10章 | 6-12批 |
| >100章 | 10-12章 | 10+批 |

### 主 Agent 流程

```
1. 计算批次分配
2. 创建章节目录结构
3. 并行派生子 Agent（每批一个）
4. 监控子 Agent 进度
5. 所有完成 → 进入 Phase 7
```

### 子 Agent Prompt

```
你是网文批量创作 Agent。创作第 {start} 到 {end} 章。

项目路径：{projectPath}
网文类型：{genre}
金手指：{goldenFinger}
核心爽点：{coreTropes}

创作流程（严格按 Ind6_Team_Writing.md 第6.2节执行）：
1. 读取章节详细规划
2. 一致性检查
3. 章节创作
4. 自我校验
5. 收尾

约束：
- 不要使用 AskUserQuestion
- 每章必须读取详细规划
- 字数必须达到 2000 字以上
- 伏笔必须按规划埋设/回收
- 负责的所有章节必须全部完成

完成后报告：各章编号、字数、校验结果
```

**并发安全**：每批不重叠，无写入冲突

## 6.5 Teams 模式

Claude Code 多 Agent 协作。

### 主 Agent 流程

```
1. TeamCreate 创建写作团队
2. 派生 3-5 个团队成员
3. TaskCreate 为每章创建任务
4. 团队成员各自：
   - TaskList 读取任务
   - TaskUpdate 认领任务
   - 执行标准创作流程
   - TaskUpdate 标记完成
5. 所有任务完成 → 关闭团队 → 进入 Phase 7
```

### 团队成员 Prompt

```
你是网文创作团队成员。创作分配给你的章节。

项目路径：{projectPath}
网文类型：{genre}
金手指：{goldenFinger}

工作流程：
1. TaskList 读取任务
2. TaskUpdate 认领任务
3. 读取章节详细规划
4. 执行标准创作流程（详见 Ind6_Team_Writing.md 第6.2节）
5. TaskUpdate 标记完成
6. 回到 TaskList 认领下一个

约束：
- 不要使用 AskUserQuestion
- 字数必须达到 2000 字以上
- 通过 SendMessage 与团队协调
```

## 6.6 质量监控

### 进度跟踪

```markdown
## 写作进度

| 章节 | 状态 | 字数 | 校验结果 |
|------|------|------|---------|
| 第1章 | ✅ | 2500 | 通过 |
| 第2章 | 🔄 | - | - |
| ... | ⏳ | - | - |
```

### 问题章节处理

- 校验失败 → `status: "failed"`，`retryCount +1`
- 重试3次仍失败 → 标记保留，主 Agent 接管或人工介入
