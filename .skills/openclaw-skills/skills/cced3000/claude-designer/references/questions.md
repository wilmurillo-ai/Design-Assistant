# Question Banks

Use these as a starting point when beginning a new design task. Pick questions that matter for the specific deliverable. Aim for 10+ questions in one batch; it's better to over-ask than under-ask.

---

## Always ask (any deliverable)

1. What's the starting point? Do you have a design system, UI kit, brand guide, screenshots, Figma, or codebase I should work from?
2. If no existing design context: do you want me to commit to a specific aesthetic direction, or use a well-known UI kit as a baseline?
3. How many variations do you want? Across what dimensions (visual, layout, interaction, copy, motion)?
4. Do you want options grounded in existing patterns, novel/experimental solutions, or a mix?
5. What's the target audience and venue?
6. Which aspect matters most — flows, copy, visuals, motion?
7. Are there any designs you like or want to reference?
8. Anything to avoid (aesthetic dislikes, brand constraints, etc.)?

---

## Slide decks

9. How many slides (roughly)?
10. Target length in minutes for the talk?
11. Is this projected (16:9, 1920×1080) or printed/shared as PDF?
12. Do you want speaker notes? If yes, should they be full scripts or bullet cues?
13. How text-heavy vs image-heavy? Are there photos / diagrams to use?
14. Is there a required template or brand deck to match?
15. Any specific slides you know you want (title, agenda, thank you, etc.)?

---

## Interactive prototypes

9. What device / form factor? (iPhone, Android, tablet, desktop, responsive)
10. What's the core flow to demo? (onboarding, purchase, a specific feature)
11. How many screens should the flow cover?
12. Which interactions need to feel real vs can be stubbed? (e.g., real search results vs a static list)
13. Should screens persist state across refreshes?
14. Is this for user testing, a design review, or an investor demo? (Affects fidelity expectations.)
15. Do we need dark mode, localization, or accessibility-specific treatments?

---

## Animated video

9. Rough target duration? (15s / 30s / 60s / 90s)
10. What's the single sentence the video is communicating?
11. Is there narration / voiceover, or is it silent / text-on-screen?
12. Output format — MP4 export, looping HTML, live playback?
13. Aspect ratio? (16:9, 9:16 vertical, 1:1 square)
14. Any existing animation style to match?

---

## Landing page / marketing page

9. What's the single goal of the page? (sign up, buy, book demo, collect email)
10. Who's the audience and what do they care about?
11. What's above the fold — headline + visual, or headline + form?
12. What sections do you need? (hero, social proof, features, pricing, FAQ, CTA)
13. Is there real imagery / product screenshots to use, or should I use placeholders?
14. Mobile-first, desktop-first, or equal?

---

## Dashboard

9. Who uses this and how often? (glance daily vs deep-dive weekly)
10. What's the top 1–2 metrics they MUST see?
11. What data sources / API shapes should I assume?
12. Real-time, periodic refresh, or static daily snapshot?
13. Filtering / drill-down needed, or just view?
14. Dark mode, data density preferences, charting library preference?

---

## Logo / brand / visual identity

9. Industry and target audience?
10. 2–3 adjectives describing the feeling? (e.g., "calm, precise, technical")
11. Any visual references — brands you admire, moods, palettes?
12. Wordmark, symbol, or both?
13. Where will it appear? (screens only, print, merch, signage)
14. Any constraints — color restrictions, must work in one color, etc.?

---

## Design system components

9. Which components do you need? (buttons, inputs, cards, modals, nav, etc.)
10. What states should each component have? (default, hover, active, disabled, loading, error)
11. Framework to target? (React + Tailwind, vanilla CSS, tokens only)
12. Should I document usage, or just produce the components?
13. Any existing tokens (colors, spacing scale, type scale) to use?

---

## When the user gave you "enough"

Skip the questions when the user's brief contains:

- A clear deliverable + audience + venue ("10-min deck for eng all-hands from this PRD")
- A direct reference ("recreate this screen", "match our marketing site")
- Explicit fidelity ("rough wireframes", "hi-fi pixel-perfect")
- Explicit scope ("just the hero section", "full onboarding flow")

When in doubt, ask. One round of focused questions early saves two rounds of rework later.
