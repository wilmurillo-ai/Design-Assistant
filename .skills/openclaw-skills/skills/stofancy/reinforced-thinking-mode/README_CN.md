# 强化思考模式 (Reinforced Thinking Mode)

## Skill 名称
**强化思考模式 (Reinforced Thinking Mode)**

## 一句话描述
通过强制性多轮独立深度思考，生成高质量、全面的解决方案，严格约束防止偷懒思维和幻觉。

## 详细描述

强化思考模式是一种解决复杂问题的严格方法，确保 AI Agent 在多轮思考中进行彻底、独立、高质量的思考。

### 它解决的问题

标准的多轮思考通常存在以下问题：
- **偷懒思维**：Agent 留下 TODO 和"稍后完善"的标记
- **上下文污染**：携带过多前几轮的上下文，限制创造力
- **内部幻觉**：假装完成了10轮思考，实际上只是总结
- **迭代依赖**：依赖"下一轮"来修复当前轮的缺陷

### 解决方案

本 skill 通过严格约束确保：
1. **真正的独立性**：每轮只读取 `problem.md` 和上一轮方案
2. **最终交付质量**：每轮必须完整，无 TODO 或"稍后完善"
3. **防止幻觉**：文件读写强制从头思考，防止内部捷径
4. **内部自我审视**：自我检查但不记录，避免影响下一轮

## 核心特性

- ✅ **心智模式转变**："每轮都是最后一轮"的心态
- ✅ **严格文件约束**：每轮只读取必要文件
- ✅ **禁止关键词**：自动防止偷懒模式（TODO、下一轮等）
- ✅ **独立宣言**：每个方案声明其独立性
- ✅ **无审视文件**：仅内部自我检查，保持下一轮真正独立
- ✅ **灵活输出**：无固定模板，适应具体问题需求

## 使用场景

- 复杂系统架构设计
- 多维度决策制定
- 创新方案探索
- 全面风险评估
- 多方案战略规划

## 使用方法

1. 创建工作目录
2. 在 `problem.md` 中定义你的问题
3. 运行 skill，指定轮数（默认：5，推荐：10）
4. 每轮生成独立的 `round_{n}.md` 方案
5. 最终报告整合所有方案

### 示例

```bash
# 启动强化思考
reinforced_thinking --problem "设计一个可扩展的微服务架构" --rounds 10

# 输出结构
reinforced-thinking/
├── problem.md      # 问题定义
├── round_1.md      # 独立方案 1
├── round_2.md      # 独立方案 2
├── ...
├── round_10.md     # 独立方案 10
└── final_report.md # 整合推荐
```

## 为什么有效

核心洞察：**写文件防止内部幻觉**。当 AI 必须每轮读取文件时，它无法依赖内部记忆或假装进行了深度思考。每轮都重新开始，确保真正的独立分析。
