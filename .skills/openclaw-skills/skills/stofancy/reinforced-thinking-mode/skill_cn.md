# 强化思考模式

## 何时使用

当用户需要深度、多角度分析时激活：
- 关键词："深度思考"、"多角度"、"全面评估"、"设计方案"、"架构规划"
- 意图：复杂系统设计、战略决策、创新、风险评估

**复杂度 → 轮数：**
- 简单（事实性、明确）：2-3 轮
- 中等（功能设计、权衡）：4-5 轮
- 复杂（架构、战略）：5-7 轮
- 棘手（未定义、冲突）：7-10 轮

## 核心原则

**N 次独立的、完整的思考会话——而非迭代完善。**

每轮必须达到：*"如果这是最后一轮，我完全满意。"*

### 心智模式

✅ **思考：** "这是我唯一的机会——全力以赴"

❌ **绝不想：** "下一轮再改进这个"

---

## 硬性约束

### 文件访问
- **每轮只读取：** `problem.md` + `round_{i-1}.md`（第1轮：仅 problem.md）
- **绝不**读取其他轮次的详细内容

### 禁用词汇
方案文件中绝不使用：`TODO`、`待完善`、`下一轮`、`稍后`、`再细化`

### 信息缺口
- 事实不确定 → 立即搜索
- 需求不确定 → 立即询问用户
- **绝不**假设并继续

---

## 执行流程

### 阶段 1：初始化

1. **评估复杂度** → 确定轮数（简单2-3，中等4-5，复杂5-7，棘手7-10）
2. **创建目录** `reinforced-thinking/`
3. **写入问题定义** `problem.md`：背景、当前状态、核心问题、约束、成功标准

---

### 阶段 2：迭代（每轮执行）

对于第 X 轮：

1. **读取文件**：`problem.md` + `round_{X-1}.md`（第1轮仅 problem.md）
2. **重置思维**：从全新角度思考，不复制上一轮方法
3. **自由选择角度**：必要时完全推翻上一轮
4. **写入方案** `round_X.md`，包含：
   - 独立思考宣言
   - 核心洞察
   - 方案设计
   - 预期结果
   - 风险与对策
   - 为什么方案完整
5. **自我审视**：检查是否有禁用词、是否完整、是否可直接实施
6. **如不通过 → 重做步骤4
7. **提前终止检查**（第2轮及以后）：
   - 对比当前轮与上一轮的核心思路、解决方案
   - 如相似度 > 70%，说明缺乏新意，**建议提前终止**
   - 将终止建议写入当前轮方案中，由用户决定是否终止

---

### 阶段 3：合成

1. 读取所有轮次 + problem.md
2. 分析各方案独特价值、冲突、互补点
3. **红队审查**：对每个方案进行批判性审视
   - 假设缺陷：方案基于哪些假设？假设不成立怎么办？
   - 漏洞风险：有哪些潜在漏洞或被绕过的可能？
   - 失败模式：在什么场景下会失效？
   - 对抗测试：如果有人故意破坏，最脆弱的点在哪里？
4. 生成最终报告 `final_report.md`（包含红队审查结论）

### 阶段 4：清理

自动删除中间文件（problem.md, round_*.md），只保留 final_report.md。

如需保留，可在prompt中注明。

---

## 质量保证

### 强制重做规则

满足以下任一条件，必须重做当前轮：
1. 包含禁用词（TODO、待完善、下一轮、稍后、再细化）
2. 不是最终质量，需"后续补充"
3. 读取了 problem.md 和 round_{X-1}.md 以外的文件
4. 方案不完整

### 检查清单（每轮必做）

- [ ] 选择了与上一轮不同的角度
- [ ] 没有复制上一轮的方案
- [ ] 无禁用词
- [ ] 如果这是最后一轮，我会满意
- [ ] 所有细节已包含，可直接实施
- [ ] 只读取了 problem.md 和上一轮

## 常见错误与纠正

| 错误 | 示例 | 纠正方法 |
|------|------|----------|
| 携带上下文 | "基于R1的UX和R2的技术..." | 只写"基于 problem.md 和 R3 的方法..." |
| 留下TODO | "细节下一轮再设计" | 现在就给出核心设计细节 |
| 假设事实 | "用户可能想要X" | 搜索确认或询问用户 |
| 预设方向 | "下一轮：安全角度" | 不预设，让每轮自由选择 |
| 迭代思维 | "在R1基础上改进" | 每轮独立，从头思考 |
| 不写细节 | "详见相关文档" | 直接在文件中写出完整细节 |

---

## 最佳实践

- **发散思维**：无角度矩阵——自由选择任何视角
- **真正独立**：只读 problem.md + 上一轮，而非全部历史
- **每轮全力以赴**：不为"下次迭代"保留实力
- **透明公开**：在文件中说明选择的视角和理由
- **提前终止**：如连续2轮缺乏真正创新或与上轮相似度>70%，建议提前终止
