```
  乐乐 (发起方)              老板                    开心 (接收方)
       │                      │                        │
       │  1. 申请通信窗口       │                        │
       │ ────────────────────► │                        │
       │                      │                        │
       │  2. 审批通知           │                        │
       │ ◄──── ✅ 批准 ──────  │                        │
       │                      │                        │
       │  3. 开启窗口 + 发送任务                          │
       │ ─────────────────────────────────────────────► │
       │                                                │
       │  4. 任务执行 + 返回结果                          │
       │ ◄───────────────────────────────────────────── │
       │                                                │
       │  ... (更多消息交换) ...                          │
       │                                                │
       │  5. 窗口到期 / 手动关闭                          │
       │ ──── 自动生成摘要 ────                          │

       │                      │  (老板随时可查日志)       │
       │                      │                        │
```

### 2.2 Token 保护流程

```
  消息发送
      │
      ▼
  计算 Token 消耗
      │
      ├─── < 80% 预算 ──→ 正常通信
      │
      ├─── ≥ 80% 预算 ──→ ⚠️ 发送告警给老板 + 节点
      │
      └─── ≥ 100% 预算 ─→ 🛑 强制关闭窗口
```

## 3. 安全模型

### 3.1 三层安全

```
  ┌─────────────────────────────────────┐
  │  Layer 1: 网络隔离                    │
  │  仅限 192.168.110.0/24 内网通信       │
  ├─────────────────────────────────────┤
  │  Layer 2: Token 认证                 │
  │  每个节点持有唯一 API Token           │
  │  所有请求必须携带 Bearer Token        │
  ├─────────────────────────────────────┤
  │  Layer 3: 窗口授权                    │
  │  通信必须在已批准的时间窗口内          │
  │  老板审批 + 自动到期 + Token 上限     │
  └─────────────────────────────────────┘
```

### 3.2 审计链

```
  消息产生
      │
      ▼
  comm_logger.sh 记录
      │
      ├──→ memory/node-comms/YYYY-MM-DD/window-xxx.jsonl  (实时日志)
      │
      ├──→ memory/node-comms/active-windows.json           (窗口状态)
      │
      └──→ memory/node-comms/reports/YYYY-MM.md            (月度报表)

  老板可以：
  - 随时 replay 任何窗口的完整对话
  - 搜索历史通信记录
  - 查看月度通信报表
  - 审查 Token 使用情况
```

## 4. 与 v2.0 的协同

### 4.1 兼容性设计

v3.0 是 v2.0 的**增量升级**，不破坏现有架构：

```
  v2.0 三层记忆架构 (保留)
  ├── Layer 1: Session Context (不变)
  ├── Layer 2: Working Memory  (不变)
  └── Layer 3: Long-Term Memory (不变)
  
  v3.0 新增层
  └── Layer 4: Node Communication
      ├── N2N Protocol (节点间通信)
      ├── Communication Windows (通信窗口)
      └── Cross-Node Memory (跨节点记忆)
```

### 4.2 子脑 → 远程节点

v2.0 的子脑概念在 v3.0 中扩展：

| v2.0 | v3.0 | 说明 |
|------|------|------|
| 本地子脑 | 本地 Agent | 同一节点内的子进程 |
| — | 远程节点 | 通过 N2N 协议通信的独立节点 |
| — | 混合模式 | 本地子脑 + 远程节点协作 |

子脑可以是：
- **本地进程** — 同 v2.0，直接文件共享
- **远程 Agent** — 通过 N2N 协议通信，需要通信窗口

### 4.3 记忆同步

```
  乐乐 (controller)
  memory/
  ├── YYYY-MM-DD.md          ← 本地记忆 (v2.0)
  ├── node-comms/            ← N2N 通信记录 (v3.0 新增)
  │   ├── active-windows.json
  │   ├── 2026-03-24/
  │   │   ├── win-001.jsonl
  │   │   └── win-002.jsonl
  │   └── reports/
  │       └── 2026-03.md
  └── strategies.json        ← 可含跨节点策略 (v2.0 扩展)
```
