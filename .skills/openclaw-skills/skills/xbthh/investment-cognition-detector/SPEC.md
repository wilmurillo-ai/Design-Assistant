# 投资认知检测器 (Investment Cognition Detector)

## 核心理念

**帮助用户发现"以为自己懂但实际不懂"的认知盲点**

基于苏格拉底式追问和费曼学习法原理，通过系统化四个核心问题的提问，验证用户对投资标的的真实理解深度。

## 设计理论基础

| 理论 | 应用 |
|------|------|
| 段永平/巴菲特能力圈理论 | "懂"的四条标准：敢重仓、敢长期持有、下跌时开心、不会到处问 |
| 费曼学习法 | 能清晰解释=真懂，不能解释=假懂 |
| 达克效应 | 帮助用户从"不知道自己不知道"进入"知道自己不知道" |
| 米勒定律 | 4-5个核心问题是人类工作记忆的最佳处理量 |

## 核心框架：四问模型

每个问题权重25%，追问机制贯穿始终：

| # | 问题 | 核心评估内容 | 典型追问 |
|---|------|------------|----------|
| Q1 | 这家公司是靠什么赚钱的？ | 商业模式理解深度 | "为什么是这三件事？5年后还成立吗？" |
| Q2 | 这家公司能否持续赚钱？ | 护城河持续性判断 | "这个护城河5年后是变强还是变弱？" |
| Q3 | 这家公司能否赚更多的钱？ | 成长性/第二增长曲线 | "主业放缓后靠什么增长？" |
| Q4 | 这家公司赚了钱能否分给我？ | 股东回报/分红 | "如果不分红，钱去哪了？再投资回报高吗？" |

## 评估等级

| 等级 | 分值 | 含义 |
|------|------|------|
| 🔴 不懂 | 0-40% | 存在严重认知盲点，不建议重仓 |
| 🟡 初步懂 | 40-70% | 有基础认知，但存在重要漏洞 |
| 🟢 基本懂 | 70-90% | 理解核心逻辑，少量盲点可接受 |
| ✅ 完全懂 | 90%+ | 认知较为完整，可考虑重仓 |

## 工作流程

```
1. 用户指定公司 →
2. AI快速研究公司（公开资料） →
3. 基于四问框架生成4道核心问题 + 追问 →
4. 交互式问答（每题含追问） →
5. 汇总诊断报告 →
6. 认知盲点反馈 + 学习建议
```

## 追问机制

每道核心问题后，AI应根据用户回答进行追问：

- **追问方向**：
  - "为什么这么认为？"（追问因果）
  - "有什么证据支持？"（追问依据）
  - "如果...会怎样？"（追问边界条件）
  - "你确定吗？有没有考虑相反的情况？"（追问假设）

- **追问触发**：当用户回答过于笼统、缺乏具体逻辑、或暴露矛盾时

## 报告输出

### 诊断报告结构

1. **总体评分**：四档等级 + 四问得分分布
2. **问题详解**：每问得分 + 具体反馈
3. **认知盲点**：明确列出"你以为懂但实际有问题的点"
4. **追问启示**：在交互中暴露的有价值的问题
5. **学习建议**：针对盲点的具体研究方向

### 报告语言风格

- 直接、犀利，不回避问题
- 引用用户原话作为证据
- 给出具体改进方向，不只是批评

## 使用场景

- **持仓前检验**：买入前确认自己真的懂
- **定期复审**：每季度/重大事件后重新检测认知变化
- **持仓后跟踪**：通过持续检测追踪认知是否随时间深化或退化

## 技术实现

- 基于公开资料研究公司（Web Search/Fetch）
- 问题生成基于公司特点个性化
- 追问逻辑基于用户回答动态调整
- 最终报告Markdown格式
