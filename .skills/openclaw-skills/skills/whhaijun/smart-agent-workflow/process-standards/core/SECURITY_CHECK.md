# 安全诊断规范 v1.0

> 所有操作前必须执行安全检查
> 创建时间：2026-03-28
> 适用范围：所有团队成员

---

## 🔒 核心原则

**任何操作前，先问自己 5 个问题：**
1. 是否涉及敏感数据？
2. 是否会对外发送？
3. 是否不可逆？
4. 是否涉及用户隐私？
5. 是否需要用户确认？

---

## 📋 操作前安全检查清单

### 必须检查（每次操作前）

```markdown
- [ ] 是否涉及敏感数据？（API key、密码、token）
- [ ] 是否会对外发送？（git push、API 调用、消息发送）
- [ ] 是否不可逆？（删除、覆盖、发布）
- [ ] 是否涉及用户隐私？（姓名、手机、邮箱、地址）
- [ ] 是否需要用户确认？
```

**如果任意一项为"是" → 执行对应的安全措施**

---

## 🔍 敏感数据识别规则

### 自动识别模式

| 类型 | 正则模式 | 示例 |
|------|---------|------|
| API Key | `/api[_-]?key/i`, `/token/i`, `/secret/i` | `api_key`, `API_TOKEN` |
| 密码 | `/password/i`, `/pwd/i`, `/passwd/i` | `password`, `user_pwd` |
| 手机号 | `/1[3-9]\d{9}/` | `13812345678` |
| 邮箱 | `/[\w.-]+@[\w.-]+\.\w+/` | `user@example.com` |
| 身份证 | `/\d{17}[\dXx]/` | `110101199001011234` |
| 银行卡 | `/\d{16,19}/` | `6222021234567890123` |

### 处理方式

**发现敏感数据时：**
1. **立即停止操作**
2. **脱敏处理**
   ```
   原始：api_key = "sk-1234567890abcdef"
   脱敏：api_key = "sk-****...cdef"
   
   原始：手机号 13812345678
   脱敏：手机号 138****5678
   ```
3. **询问用户**
   ```
   ⚠️ 检测到敏感数据：[类型]
   是否需要脱敏处理？
   [A] 脱敏后继续
   [B] 跳过该数据
   [C] 取消操作
   ```

## 🚦 安全操作分级

### 级别定义

| 级别 | 操作类型 | 处理方式 | 示例 |
|------|---------|---------|------|
| 🟢 安全 | 读文件、分析、本地写入 | 直接执行 | `read()`, `write()` |
| 🟡 注意 | 修改代码、删除文件 | 执行前说明影响 | `edit()`, `trash()` |
| 🔴 危险 | git push、对外发送、删除敏感数据 | 必须用户确认 | `git push`, `message()` |

### 🟢 安全操作

**可以直接执行，无需确认：**
- 读取文件（read）
- 分析代码
- 本地写入（write）
- 搜索（web_search）
- 查询（sessions_list）

### 🟡 注意操作

**执行前必须说明影响：**
- 修改代码（edit）
- 删除文件（trash）
- 重命名文件
- 移动文件

**说明格式：**
```
即将执行：[操作]
影响范围：[文件/模块]
可能风险：[风险描述]
是否继续？
```

### 🔴 危险操作

**必须用户明确确认：**
- git push（代码发布）
- message()（对外发送消息）
- exec() 涉及网络请求
- 删除敏感数据
- 修改配置文件

**确认格式：**
```
🚨 危险操作警告

操作：[具体操作]
影响：[影响范围]
风险：[可能的后果]
不可逆：[是/否]

请明确确认：
[A] 确认执行
[B] 取消操作
```

## 🛡️ 特殊场景处理

### 场景1：Git 操作

**git push 前必须检查：**
```markdown
- [ ] 是否包含敏感数据？（.env、密钥文件）
- [ ] 是否包含大文件？（> 10MB）
- [ ] 是否包含临时文件？（.DS_Store、node_modules）
- [ ] commit 信息是否规范？
- [ ] 是否推送到正确的分支？
```

**检查通过后，输出摘要：**
```
准备推送：
- 分支：[branch]
- 文件：[数量] 个
- 改动：+[行数] -[行数]
- commit: [message]

确认推送？
```

### 场景2：对外发送消息

**message() 前必须检查：**
```markdown
- [ ] 消息内容是否包含敏感信息？
- [ ] 发送对象是否正确？
- [ ] 是否需要用户审核内容？
```

**检查通过后，输出预览：**
```
准备发送消息：
- 目标：[channel/user]
- 内容预览：[前100字]
- 是否包含附件：[是/否]

确认发送？
```

### 场景3：删除操作

**删除前必须检查：**
```markdown
- [ ] 是否有备份？
- [ ] 是否可恢复？（trash vs rm）
- [ ] 是否影响其他模块？
```

**优先使用 trash（可恢复）：**
```
❌ 不推荐：exec("rm -rf xxx")
✅ 推荐：exec("trash xxx")
```

## 🎯 实战示例

### 示例1：读取配置文件

**操作：** `read("config.yml")`

**安全检查：**
- 级别：🟢 安全
- 敏感数据：可能包含 API key
- 处理：读取后自动脱敏

**执行：**
```python
content = read("config.yml")
# 自动脱敏
content = content.replace(/api_key:\s*"([^"]+)"/, 'api_key: "****"')
```

### 示例2：推送代码

**操作：** `exec("git push origin main")`

**安全检查：**
- 级别：🔴 危险
- 检查：是否包含敏感数据
- 处理：必须用户确认

**执行：**
```
🚨 危险操作警告

操作：git push origin main
影响：代码将推送到远程仓库
风险：不可撤回，可能影响生产环境
不可逆：是

请明确确认：
[A] 确认执行
[B] 取消操作
```

### 示例3：发送消息

**操作：** `message(action="send", to="user", message="...")`

**安全检查：**
- 级别：🔴 危险
- 检查：消息内容是否包含敏感信息
- 处理：预览后用户确认

**执行：**
```
准备发送消息：
- 目标：user@example.com
- 内容预览：您的验证码是 1234...
- 是否包含附件：否

⚠️ 检测到可能的敏感信息：验证码
是否继续发送？
```
