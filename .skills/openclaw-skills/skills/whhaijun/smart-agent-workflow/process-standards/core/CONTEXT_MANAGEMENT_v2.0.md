# Context 管理规范 v2.0

> 避免 Token 超限，提升执行效率
> 最后更新：2026-03-28
> 变更：新增自动监控、预警机制、优化策略

---

## 📊 问题说明

**Context 过长会导致：**
- Token 使用率过高（>90%）
- 响应变慢
- 费用增加
- 可能超出模型限制

**触发条件：**
- 单次对话处理多个复杂任务
- 大量代码读写操作
- 频繁的文件操作
- 长时间连续对话

---

## 🎯 核心策略

### 三大原则

1. **预防为主** — 任务开始前评估 Context 需求
2. **实时监控** — 执行过程中持续监控使用率
3. **及时优化** — 达到阈值立即采取措施

---

## 📈 Context 监控指标

### 实时监控（每次操作后）

| 指标 | 说明 | 阈值 |
|------|------|------|
| Token 使用率 | 当前/总量 | <70% 健康 / 70-85% 注意 / 85-95% 警告 / >95% 危险 |
| 对话轮次 | 连续对话次数 | <10 健康 / 10-20 注意 / >20 警告 |
| 文件读取次数 | 累积读取文件数 | <5 健康 / 5-10 注意 / >10 警告 |
| 任务执行时长 | 累积执行时间 | <15分钟 健康 / 15-30分钟 注意 / >30分钟 警告 |

### 预警级别

```
🟢 健康（<70%）
   - 正常执行
   - 无需特殊处理

🟡 注意（70-85%）
   - 提示压缩 MEMORY.md
   - 减少文件全量读取
   - 使用引用代替复制

🟠 警告（85-95%）
   - 建议创建 Task Brief
   - 停止非必要操作
   - 准备开新对话

🔴 危险（>95%）
   - 强制创建 Task Brief
   - 立即停止当前任务
   - 必须开新对话继续
```

## ✅ 解决方案（4种）

### 方案1：Task Brief + 新对话（推荐）

**适用场景：** 任务复杂，需要多步操作

**触发条件（满足任意1条）：**
- Context 使用率 > 80%
- 任务预计耗时 > 30 分钟
- 需要跨 session 传递任务
- 连续任务达到断点

**步骤：**

1. **创建 Task Brief**
```markdown
# Task Brief: [任务名称]

## 任务目标
[清晰描述要达成的目标]

## 当前状态
- ✅ 已完成：[列出已完成的工作]
- ❌ 待完成：[列出待完成的工作]

## 关键信息
- 文件位置：[路径]
- 配置信息：[关键配置]
- 注意事项：[重要提醒]

## 下一步
[具体的执行步骤]
```

2. **保存 Task Brief**
```bash
# 保存到 workspace-shared/tasks/
echo "Task Brief 内容" > workspace-shared/tasks/TASK_BRIEF_XXX.md
```

3. **告知用户**
```
已创建 Task Brief: workspace-shared/tasks/TASK_BRIEF_XXX.md
建议开启新对话继续任务，减少 context 压力。
```

### 方案2：压缩 MEMORY.md

**适用场景：** 对话中积累了大量历史信息

**触发条件：**
- Context 使用率 70-85%
- MEMORY.md 内容过长（>500行）

**步骤：**

1. **识别关键信息**
   - 保留：核心决策、重要配置、待办事项
   - 删除：详细过程、临时信息、已完成的细节

2. **压缩示例**

❌ **压缩前（冗长）：**
```markdown
- 已学习 pydantic-ai 核心特性（类型安全、依赖注入、工具注册、结构化输出、动态指令）
- 已学习 MetaGPT 核心特性（角色系统、SOP 流程、消息传递、工作流编排）
- 已输出 smart-agent-template v2.0 优化方案（见 SMART_AGENT_OPTIMIZATION_PLAN.md）
- 已完成多渠道接入要求调研（见 MULTI_CHANNEL_REQUIREMENTS.md）
```

✅ **压缩后（精简）：**
```markdown
- 已学习 pydantic-ai、MetaGPT 核心特性
- 已输出 smart-agent-template v2.0 优化方案
- 已完成多渠道接入调研
```

3. **执行压缩**
```python
# 编辑 MEMORY.md，保留核心信息
edit(path="MEMORY.md", oldText="...", newText="...")
```

### 方案3：分批处理

**适用场景：** 需要处理多个独立任务

**原则：**
- 每次只处理 1-2 个任务
- 完成后汇报，等待下一批
- 避免一次性处理 5+ 个任务

**示例：**
```
用户：帮我完成 A、B、C、D、E 五个任务

正确做法：
1. 先完成 A、B
2. 汇报结果
3. 等待用户确认
4. 再处理 C、D、E
```

### 方案4：优化文件读取

**适用场景：** 需要读取多个文件

**问题操作：**
```python
# ❌ 一次性读取多个大文件
read(file1)
read(file2)
read(file3)
```

**优化操作：**
```python
# ✅ 只读取必要的部分
read(file1, limit=50)  # 只读前 50 行
read(file2, offset=100, limit=20)  # 只读特定区域
```

## 📋 最佳实践

### 1. 任务开始前评估

**评估清单：**
```markdown
- [ ] 预计需要读取多少文件？
- [ ] 预计需要多少步骤？
- [ ] 预计耗时多久？
- [ ] 是否需要跨 session 传递？
```

**评估结果：**
- 文件 > 5 个 → 使用 limit/offset
- 步骤 > 10 个 → 插入断点
- 耗时 > 30 分钟 → 创建 Task Brief
- 跨 session → 使用 Task Brief 格式

### 2. 执行过程中监控

**每次操作后检查：**
```markdown
- [ ] Token 使用率是否 > 70%？
- [ ] 对话轮次是否 > 10？
- [ ] 文件读取是否 > 5 个？
```

**达到阈值立即采取措施：**
- 70-85% → 压缩 MEMORY.md
- 85-95% → 创建 Task Brief
- >95% → 强制停止，开新对话

### 3. 避免大量文件操作

**优化策略：**
```python
# ❌ 全量读取
content = read("large_file.md")

# ✅ 分段读取
header = read("large_file.md", limit=10)
middle = read("large_file.md", offset=100, limit=20)
```

### 4. 使用引用而非复制

**问题做法：**
```markdown
详细内容见 XXX.md：
[复制整个文件内容]
```

**正确做法：**
```markdown
详细内容见 XXX.md（已保存，不在此重复）
```

### 5. 及时清理临时信息

**定期清理：**
- 删除已完成任务的详细记录
- 移除临时文件路径
- 清理调试信息

## 🎯 实战示例

### 示例1：Context 使用率 75%

**场景：** 正在执行任务，Token 使用率达到 75%

**处理：**
1. 检查 MEMORY.md，识别可压缩内容
2. 压缩历史记录，保留核心信息
3. 后续文件读取使用 limit/offset
4. 继续执行任务

### 示例2：Context 使用率 88%

**场景：** 任务进行到一半，Token 使用率达到 88%

**处理：**
1. 立即停止非必要操作
2. 创建 Task Brief，记录当前状态
3. 保存已完成的产出
4. 提示用户开新对话继续

### 示例3：连续任务 15 个步骤

**场景：** 任务包含 15 个连续步骤

**处理：**
1. 任务开始前插入断点（每 5 步一个）
2. 批次1：步骤 1-5 → 汇报
3. 批次2：步骤 6-10 → 汇报
4. 批次3：步骤 11-15 → 汇报
5. 每个断点检查 Context 使用率

## 📊 Context 使用率计算

### 简单估算

```
Token 使用率 = (已使用 Token / 总 Token) × 100%

已使用 Token ≈ 
  系统提示词 Token 
  + 对话历史 Token 
  + 文件内容 Token 
  + 工具调用 Token
```

### 优化目标

| 任务类型 | 目标使用率 |
|---------|-----------|
| 简单任务（<5步） | <50% |
| 中等任务（5-10步） | 50-70% |
| 复杂任务（>10步） | 70-85% |

**超过 85% → 必须优化**

## 🆘 紧急情况处理

### Context 使用率 > 95%

**立即执行：**
1. 停止所有操作
2. 创建 Task Brief（使用模板）
3. 保存当前状态
4. 输出紧急汇报：
   ```
   ⚠️ Context 使用率超限（XX%）
   
   已完成：
   - [列出已完成的工作]
   
   待完成：
   - [列出待完成的工作]
   
   Task Brief 已创建：[路径]
   
   建议立即开新对话继续任务。
   ```
