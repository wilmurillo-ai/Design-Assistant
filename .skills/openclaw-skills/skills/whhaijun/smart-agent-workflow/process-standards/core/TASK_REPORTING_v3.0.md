# 任务汇报规范 v3.0（简化版）

> 核心原则：**简单任务简单汇报，复杂任务详细汇报**
> 最后更新：2026-03-28
> 变更：简化为 P0/P1 分级，减少强制检查点

---

## 🎯 核心原则

**有进展就汇报，卡住就上报，完成就汇总**

---

## ✅ P0 规范（所有任务必须遵守）

### 核心 3 条

**1. 卡住必须上报**

**触发条件：**
- 失败 ≥ 2 次
- 等待 > 30 秒
- 不知道怎么继续

**格式：**
```
⚠️ 卡住了

问题：[具体问题]
已尝试：[尝试过的方法]
需要：[需要什么帮助]
```

**2. 完成必须汇报**

**格式（3句话）：**
```
目标：[原始需求]
结果：[一句话结论]
产出：[文件路径或结论]
```

**示例：**
```
目标：读取 README.md
结果：文件内容已输出
产出：无
```

**3. 需要决策必须上报**

**触发条件：**
- 需求不明确
- 有多个方向不知道选哪个
- 涉及钱/权限/对外发送

**格式：**
```
🚨 需要你决策

情况：[简述]
选项：[A] xxx / [B] xxx
我的建议：[A/B] — 理由：[...]
```

---

## 📋 P1 规范（复杂任务建议遵守）

### 额外 2 条

**4. 批次完成汇报**

**适用场景：** 复杂任务，有多个批次

**格式（简短）：**
```
批次X完成

完成：[任务名] — [结果+关键数据]
下一步：[行动] / 预计：[时间]
```

**示例：**
```
批次1完成

完成：分析了 3 个文件，识别了 5 个问题
下一步：设计新架构 / 预计：30分钟
```

**5. 断点汇报**

**适用场景：** 连续任务 > 15 分钟

**格式（极简）：**
```
进度更新：已完成 X/Y，剩余 Z 分钟
```

## 🎯 实战示例

### 示例1：简单任务（P0）

**任务：** "帮我读一下 README.md"

**汇报：**
```
目标：读取 README.md
结果：文件内容已输出
产出：无
```

### 示例2：复杂任务（P0 + P1）

**任务：** "帮我重构支付模块"

**拆分方案汇报：**
```
任务拆分：
批次1：分析现有代码（30分钟）
批次2：设计新架构（30分钟）
批次3：重构代码（1小时）
```

**批次1完成汇报：**
```
批次1完成

完成：分析了 3 个文件，识别了 5 个问题
下一步：设计新架构 / 预计：30分钟
```

**批次2完成汇报：**
```
批次2完成

完成：设计文档已输出
下一步：开始重构 / 预计：1小时
```

**最终汇报：**
```
目标：重构支付模块
结果：完成重构，代码质量提升
产出：3个重构后的文件
```

### 示例3：卡住上报（P0）

**场景：** 接口调用失败 2 次

**汇报：**
```
⚠️ 卡住了

问题：手续费接口调用失败
已尝试：检查参数、重试 2 次
需要：后端同学检查接口状态
```

### 示例4：需要决策（P0）

**场景：** 有多个实现方案

**汇报：**
```
🚨 需要你决策

情况：转账确认页有两种设计方案
选项：
[A] 单页显示所有信息 — 优点：简洁 / 缺点：信息密集
[B] 分步显示 — 优点：清晰 / 缺点：多一步操作

我的建议：A — 理由：用户更习惯一次性确认
```

## 📊 汇报频率对照表

| 任务类型 | P0 汇报 | P1 汇报 | 总汇报次数 |
|---------|---------|---------|-----------|
| 简单任务（<15分钟） | 完成汇报 | 无 | 1次 |
| 复杂任务（3批次） | 完成汇报 | 批次汇报×3 | 4次 |
| 长连续任务（>30分钟） | 完成汇报 | 批次汇报 + 断点汇报 | 5-8次 |

---

## 🚨 禁止行为

### 绝对不允许

- ❌ 任务卡住超过 10 分钟无任何反馈
- ❌ 失败后默默重试，不说原因
- ❌ 等待用户主动来问进度

### 必须做到

- ✅ 卡住就上报（失败≥2次 / 等待>30秒）
- ✅ 完成就汇报（3句话即可）
- ✅ 需要决策就上报

---

## 🔄 与 v2.0 的区别

| 项目 | v2.0 | v3.0（简化版） |
|------|------|---------------|
| 强制检查点 | 5个 | 3个（P0） |
| 汇报格式 | 详细模板 | 简短格式（3句话） |
| 批次汇报 | 强制 | 建议（P1） |
| 适用场景 | 所有任务 | 分级：简单任务用P0，复杂任务用P0+P1 |
