# 记忆压缩规则

## 触发条件

当 `memory/hot.md` 超过 100 行时，自动触发压缩。

## 压缩策略

### 1. 标记系统

在 `hot.md` 中使用以下标记：

| 标记 | 含义 | 处理方式 |
|------|------|---------|
| `[deprecated]` | 过时内容 | 删除 |
| `[warm]` | 低频使用 | 移动到 `memory/projects/*.md` |
| `[merge]` | 可合并规则 | 手动合并后删除 |

### 2. 压缩流程

```bash
# 自动压缩
cd ~/my-agent
./scripts/compress_hot.sh

# 手动标记示例
## 代码规范
- 函数命名：必须用驼峰（camelCase） [deprecated]  # 已过时
- 变量命名：用有意义的名字 [warm]  # 低频使用
```

### 3. 压缩原则

| 原则 | 说明 |
|------|------|
| **保留高频规则** | 每周使用 ≥ 3 次的规则保留在 HOT |
| **降级低频规则** | 每月使用 < 2 次的规则降级到 WARM |
| **删除过时规则** | 已被新规则替代的旧规则删除 |
| **合并相似规则** | 多条相似规则合并为一条 |

### 4. 验证机制

压缩后运行健康检查：

```bash
./scripts/health_check.sh
```

确保：
- `hot.md` ≤ 100 行
- 必需文件完整
- 目录结构正确

## 示例

### 压缩前（120 行）

```markdown
## 代码规范
- 函数命名：必须用驼峰（camelCase）
- 函数命名：不要用下划线
- 函数命名：首字母小写
- 变量命名：用有意义的名字
- 变量命名：不要用单字母
```

### 压缩后（80 行）

```markdown
## 代码规范
- 命名规范：函数/变量用驼峰，首字母小写，不用下划线/单字母
```

## 自动化

建议在 cron 中定期运行：

```bash
# 每周日凌晨 2 点自动压缩
0 2 * * 0 cd ~/my-agent && ./scripts/compress_hot.sh
```
