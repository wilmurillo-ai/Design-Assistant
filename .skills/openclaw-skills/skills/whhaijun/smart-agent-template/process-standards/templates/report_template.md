# 任务汇报模板

> 用于任务汇报
> 根据汇报类型选择对应模板

---

## 1. 任务开始汇报（拆分方案）

```markdown
## 任务拆分方案

**任务名称:** [任务名]
**触发原因:** [为什么拆]

**依赖关系:**
  A // B         → 并行
  A → B          → 串行

**批次安排:**
  批次1（并行，最多3个）：A / B / C
  批次2（依赖批次1）：D

**健康度:** 🟢 On Track / 🟡 At Risk / 🔴 Off Track

**预计时间:** [总时长]

**风险预案:**
  - [风险] 可能性：H/M/L × 影响：H/M/L → 应对：[措施]
```

---

## 2. 批次完成汇报

```markdown
## 批次X完成汇报

**健康度:** 🟢 / 🟡 / 🔴

**完成:**
- [任务名]：[结果+关键数据]（不要只说「完成了」）
- [任务名]：[结果+关键数据]

**风险/阻塞:**
- [描述] — 可能性：M，影响：H → [应对]

**下一步:**
- [行动] / 负责人：[谁] / 预计：[时间]
```

---

## 3. 问题上报

```markdown
## ⚠️ 问题上报

**任务ID:** [task_id]
**问题描述:** [具体问题]
**失败原因:** [根因分析]
**已尝试:** [尝试过的方法]
**需要:** [需要什么帮助]

**影响:**
- 阻塞任务：[哪些任务]
- 预计延误：[时间]
```

---

## 4. 决策请求

```markdown
## 🚨 需要决策

**任务ID:** [task_id]
**情况:** [简述]
**需要你决定:** [具体问题]

**选项:**
- [A] [选项A] — 优点：[...] / 缺点：[...]
- [B] [选项B] — 优点：[...] / 缺点：[...]

**我的建议:** [A/B] — 理由：[...]
```

---

## 5. 任务完成汇报

```markdown
## ✅ 任务完成

**任务ID:** [task_id]
**目标:** [原始需求]
**结果:** [一句话结论]

**产出:**
- [文件路径1]
- [文件路径2]

**关键发现:**
- [重要发现1]
- [重要发现2]

**需要 Mark 确认的:**
- [真正需要决策的事项，没有则写"无"]

**下一步建议:** [可选]
```

---

## 汇报质量自检清单

使用前检查：

- [ ] 是否包含关键数据？（不能只说"完成了"）
- [ ] 是否说明下一步？（不能只汇报结果）
- [ ] 是否标注健康度？（让用户快速了解状态）
- [ ] 是否简洁明了？（2 分钟内读完）
- [ ] 是否使用标准格式？（便于解析和追踪）
