
### 3. 自动 Task Brief 生成

**触发条件（满足任意 1 条）：**
- Context 使用率 > 80%
- 任务预计耗时 > 30 分钟
- 需要跨 session 传递任务
- 连续任务达到断点且剩余工作量大

**自动生成流程：**
```
1. 检测到触发条件
2. 自动生成 Task Brief 文件
   - task_id: 自动生成（项目前缀-日期-序号）
   - 背景: 自动提取关键上下文（≤100字）
   - 任务: 当前待完成的具体操作
   - 输入: 已完成的产出文件路径
   - 期望输出: 下一步需要产出什么
3. 保存到 workspace-shared/tasks/
4. 提示用户："已创建 Task Brief: xxx.md，建议开新对话继续"
```

## ✅ 合格 Task 的 4 个硬标准

每个子任务必须同时满足，缺一不可：

| 标准 | 说明 | 反例 |
|------|------|------|
| **单一** | 只做一件事 | "修复 KYC 和转账 bug" ❌ |
| **可量化** | 验收条件具体到步骤 | "优化页面" ❌ |
| **可控** | 1-2 个工作周期内完成 | "重构钱包模块" ❌ |
| **独立** | 不依赖未完成的其他 Task | 若有依赖 → 先完成依赖 |

## 📋 拆分方案模板（每次派任务前必填）

```markdown
任务名称：
触发原因：[为什么拆]

依赖关系：
  A // B         → 并行（同时 spawn）
  A → B          → 串行（A 完成后才能启动 B）
  A → (B // C)   → A 完成后，B 和 C 并行

批次安排：
  批次1（并行，最多3个）：A / B / C
  批次2（依赖批次1）：D

健康度：🟢 On Track / 🟡 At Risk / 🔴 Off Track

风险预案：
  - [风险描述] 可能性：H/M/L × 影响：H/M/L → 应对：[措施]

完成标准（DoD）：
  ✅ 文件写入路径 xxx
  ✅ 覆盖 X 个要点
  ✅ 无报错
  ✅ 最后一行：[TASK_RESULT] 完成
```

---

## ⚠️ 执行约束（强制）

| 规则 | 限制 |
|------|------|
| 并行上限 | 同时 spawn ≤ 3 个 subagent |
| Context 控制 | 读 > 5 个文件 → 分段（offset/limit），只传路径不粘贴内容 |
| 失败处理 | 工具失败 → 标注跳过，继续；不卡死，不重试超 2 次 |
| 最后必做 | 无论中间发生什么，最后一步永远是完整汇报 |
| 🆕 断点汇报 | 每个断点必须输出阶段汇报，不可跳过 |

---

## 📊 批次完成汇报格式

```markdown
健康度：🟢 / 🟡 / 🔴

完成：
- [任务名]：[结果+关键数据]（不要只说「完成了」）

风险/阻塞：
- [描述] — 可能性：M，影响：H → [应对]

下一步：
- [行动] / 负责人：[谁] / 截止：[时间]
```

**汇报原则：** 结论优先 → 数据支撑 → 下一步行动，2 分钟读完。

## 🎯 Tiqmo 实战示例

**❌ 不合格：**
- "优化转账流程" — 太宽泛
- "修复 KYC 和转账 bug" — 两件事
- "重构钱包" — 体量过大

**✅ 合格：**
- "转账确认页加手续费，格式 `SAR X.XX`，位于总金额下方"
- "KYC 上传失败 Toast 提示『上传失败，请重试』，3 秒消失"
- "余额刷新超时从 10s 改 5s，超时显示上次缓存值"
