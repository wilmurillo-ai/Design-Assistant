# WBS 任务拆分规范 v3.0（简化版）

> 核心原则：**先判断任务类型，再选择规范**
> 最后更新：2026-03-28
> 变更：简化为 P0/P1 分级，增加任务类型判断流程

---

## 🎯 任务类型判断流程（30秒内）

### Step 1：判断新任务 or 连续任务

| 特征 | 判断 |
|------|------|
| 包含"继续"、"刚才"、"上次" | 连续任务 |
| 引用了之前的产出 | 连续任务 |
| 完全独立的新需求 | 新任务 |

### Step 2：判断简单 or 复杂

**满足任意 2 条 → 复杂任务：**
- 文件 ≥ 3 个
- 步骤 ≥ 5 个
- 预计耗时 > 15 分钟

**否则 → 简单任务**

### Step 3：选择规范

| 任务类型 | 使用规范 | 核心动作 |
|---------|---------|---------|
| **简单新任务** | P0（核心3条） | 直接执行 + 完成汇报 |
| **复杂新任务** | P0 + P1（6条） | WBS拆分 + 批次汇报 + 完成汇报 |
| **简单连续任务** | P0（核心3条） | 继续执行 + 完成汇报 |
| **复杂连续任务** | P0 + P1 + 断点 | 插入断点 + 批次汇报 + 完成汇报 |

---

## ✅ P0 规范（所有任务必须遵守）

### 核心 3 条

**1. 复杂任务必须拆分**
- 满足任意2条：文件≥3 / 步骤≥5 / 耗时>15分钟
- 输出拆分方案（简短即可，不需要详细模板）

**2. 卡住必须上报**
- 失败≥2次 / 等待>30秒 / 不知道怎么继续
- 立即告知用户

**3. 完成必须汇报**
- 格式：目标 + 结果 + 产出（3句话即可）

---

## 📋 P1 规范（复杂任务建议遵守）

### 额外 3 条

**4. 批次完成汇报**
- 每个批次完成后简短汇报
- 格式：完成了什么 + 下一步

**5. 危险操作确认**
- git push / 对外发送 / 删除
- 先说明影响，再执行

**6. 连续任务插断点**
- 连续执行 > 15分钟 → 插入断点
- 断点处汇报进度

## 🎯 实战示例

### 示例1：简单新任务

**任务：** "帮我读一下 README.md"

**判断：**
- 新任务 ✓
- 简单（1个文件，1步，<1分钟）✓
- **使用规范：** P0

**执行：**
1. 直接读文件
2. 输出内容
3. 完成汇报：读取 README.md，内容已输出

### 示例2：复杂新任务

**任务：** "帮我重构支付模块"

**判断：**
- 新任务 ✓
- 复杂（多个文件，多步骤，>1小时）✓
- **使用规范：** P0 + P1

**执行：**
1. 输出拆分方案：
   - 批次1：分析现有代码（30分钟）
   - 批次2：设计新架构（30分钟）
   - 批次3：重构代码（1小时）

2. 批次1完成汇报：
   - 完成：分析了 3 个文件，识别了 5 个问题
   - 下一步：设计新架构

3. 批次2完成汇报：
   - 完成：设计文档已输出
   - 下一步：开始重构

4. 最终汇报：
   - 目标：重构支付模块
   - 结果：完成重构，代码质量提升
   - 产出：3个重构后的文件

### 示例3：简单连续任务

**任务：** "继续刚才的工作"

**判断：**
- 连续任务 ✓
- 简单（基于之前的上下文，1-2步）✓
- **使用规范：** P0

**执行：**
1. 继续执行
2. 完成汇报：继续刚才的工作，已完成

### 示例4：复杂连续任务

**任务：** "继续完成剩余的 10 个模块重构"

**判断：**
- 连续任务 ✓
- 复杂（10个模块，预计>1小时）✓
- **使用规范：** P0 + P1 + 断点

**执行：**
1. 输出拆分方案：
   - 批次1：模块1-3（30分钟）
   - 批次2：模块4-6（30分钟）
   - 批次3：模块7-10（40分钟）

2. 每批次完成后汇报
3. 每15分钟插入断点汇报进度
4. 最终汇报

## 📊 决策树（快速参考）

```
收到任务
    ↓
包含"继续"/"刚才"？
    ├─ 是 → 连续任务
    └─ 否 → 新任务
    ↓
文件≥3 or 步骤≥5 or 耗时>15分钟？（满足2条）
    ├─ 是 → 复杂任务
    └─ 否 → 简单任务
    ↓
┌─────────────┬─────────────┐
│ 简单任务    │ 复杂任务    │
│ P0规范      │ P0+P1规范   │
│ 3条核心规则 │ 6条规则     │
└─────────────┴─────────────┘
```

## 🔄 与 v2.0 的区别

| 项目 | v2.0 | v3.0（简化版） |
|------|------|---------------|
| 核心规则 | 22个检查点 | 6条规则（P0 3条 + P1 3条） |
| 适用场景 | 所有任务 | 分级：简单任务用P0，复杂任务用P0+P1 |
| 执行难度 | 高（容易忘记） | 低（核心3条易记） |
| 灵活性 | 低（全部强制） | 高（P1为建议） |
