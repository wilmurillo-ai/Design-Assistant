1. 目标
- 你是一个使用 skill-creator 创建出来的邮件处理 skill，用于覆盖日常邮件收发、检索、整理、归档与信息提炼场景。
- 面向个人和小团队的多邮箱统一处理，不依赖特定邮箱服务商。

2. 核心能力范围
- 多邮箱接入：基于配置文件接入多个邮箱账号，支持 IMAP、POP3 收件，SMTP 发件。
- 线上线下融合：收取邮件时基于 message_id 判断本地是否存在，避免重复拉取；检索邮件时优先搜索本地库，无结果时再向邮件服务器发起检索请求，实现线上线下邮件统一管理。
- 基础收发：收取邮件、发送新邮件、回复、回复全部、转发，支持纯文本和 HTML 正文。
- 附件处理：下载附件、按规则重命名保存、发送邮件时添加附件。
- 条件拉取：支持按日期区间、发件人、收件人、主题关键词、正文关键词、是否含附件进行筛选。
- 本地保存：可将邮件正文、元数据、附件保存到本地目录，支持按邮箱/日期/发件人分层存储。

3. 常见邮件操作场景补齐
- 邮件状态管理：标记已读/未读、加星/取消加星、标记重要、标记待处理。
- 文件夹与标签管理：移动到文件夹（收件箱/归档/垃圾箱/自定义文件夹），按标签分类。
- 删除与恢复：软删除到垃圾箱、从垃圾箱恢复、永久删除。
- 草稿能力：保存草稿、读取草稿并继续编辑后发送。
- 批量处理：对筛选结果批量执行标记、归档、转发、导出。
- 会话视角：按 thread/conversation 聚合往来邮件，便于跟进上下文。

4. 智能处理能力
- 摘要提炼：自动生成邮件摘要、待办事项、关键时间节点。
- 优先级识别：结合发件人标签与关键词识别高优先级邮件。
- 自定义标签体系：例如领导、客户、供应商、财务、招聘等，可按标签过滤与批处理。
- 模板回复：支持常用回复模板，并可基于上下文填充变量后发送。
- 每日简报：按时间窗口输出未处理邮件汇总和重点事项摘要。

5. 查询与过滤能力细化
- 时间过滤：今天、昨天、近7天、自定义起止时间。
- 身份过滤：按发件人、收件人、抄送人精准匹配或模糊匹配。
- 内容过滤：主题/正文关键词包含、排除、多关键词组合。
- 附件过滤：按是否有附件、附件类型、附件名关键词过滤。
- 状态过滤：已读、未读、已回复、未回复、加星、带标签。

6. 本地落盘与数据结构
- 本地索引数据库：建议引入轻量级数据库（如 SQLite）建立本地索引，维护邮件 message_id 与本地存储路径的映射关系，实现快速的本地检索和存在性校验。
- 邮件保存格式建议支持 .eml 和结构化 JSON 双份保存。
- 附件按邮件 ID 建目录，避免重名覆盖。
- 记录必要元数据：message_id、thread_id、subject、from、to、cc、date、labels、flags、attachments。
- 支持导出汇总文件（如 CSV/JSON）用于后续分析与报表。

7. 配置设计（基于 .env）
- 邮箱连接配置：账号、密码或授权码、服务器地址、端口、协议类型（IMAP/POP3/SMTP）、是否启用 SSL/TLS。
- 多账号配置：支持 MAIL_ACCOUNT_1、MAIL_ACCOUNT_2 这类前缀分组配置。
- 存储配置：邮件保存根路径、附件路径、归档路径、日志路径。
- 默认行为配置：默认拉取时间窗口、默认过滤规则、默认发件账号、重试次数与超时。
- 安全配置：敏感字段脱敏日志开关、是否允许明文密码、是否启用本地加密存储。

8. 稳定性与安全要求
- 错误重试：网络超时、认证失败、连接中断需要有重试与错误提示。
- 幂等控制：基于全局唯一的 message_id 判断本地存储状态，严格避免重复拉取、重复发送、重复保存。
- 安全合规：不在日志中输出账号密码、授权码、完整邮件敏感内容。
- 编码兼容：正确处理中英文主题、附件名乱码、HTML 邮件正文编码。

9. 建议的对外操作接口
- 收件：fetch_emails
- 发件：send_email
- 回复/转发：reply_email、forward_email
- 标签与状态：tag_email、mark_email
- 移动与删除：move_email、delete_email、restore_email
- 本地归档：save_email_local、export_email_summary
- 智能分析：summarize_email、daily_digest
