# Landing Page Anatomy

> Reference for structuring landing page sections in optimal order.
> Load with `read_file("references/landing-page-anatomy.md")` during the Strategy and Build phases.

---

## Section Order — High-Converting Layout

This is the recommended section order. Not every page needs every section. The skill selects sections based on product type, conversion goal, and complexity.

### Essential Sections (Every Page)

| # | Section | Purpose | Above Fold? |
|---|---------|---------|-------------|
| 1 | **Navigation** | Logo + minimal nav + CTA button | Yes |
| 2 | **Hero** | Value proposition + primary CTA | Yes |
| 3 | **Social Proof Bar** | Logos, user count, or trust signal | Yes (if short) |
| 4 | **Problem/Pain** | Why the status quo is broken | No |
| 5 | **Solution/Benefits** | How your product solves it | No |
| 6 | **How It Works** | 3-step process | No |
| 7 | **CTA (mid-page)** | Reinforce the ask | No |
| 8 | **Final CTA** | Last conversion opportunity + recap | No |
| 9 | **Footer** | Legal, links, contact | No |

### Optional Sections (Based on Product)

| Section | When to Include | Position |
|---------|-----------------|----------|
| **Features Grid** | Product has 3+ distinct features | After Solution |
| **Testimonials** | Social proof available | After Features or How It Works |
| **Case Study** | Complex/expensive product | After Testimonials |
| **Pricing** | Price is a selling point or needs transparency | Before Final CTA |
| **Comparison Table** | Competitive market, users shopping | After Features |
| **FAQ** | Common objections or complex product | Before Final CTA |
| **Team/Founder** | Trust matters (consulting, coaching, services) | After Testimonials |
| **Guarantee** | Risk reversal needed (money-back, free trial) | Before or with Final CTA |
| **Stats/Numbers** | Impressive metrics available | After Social Proof or Features |
| **Video/Demo** | Product is visual or hard to explain in text | In Hero or after Problem |
| **Integration Logos** | Product connects to other tools | After Features |

## Section Templates

### 1. Navigation

```
[Logo]                                    [CTA Button]
```

Rules:
- **Minimal links** — a landing page is not a website. 0-3 nav links max.
- **No competing CTAs** — one primary action in the nav
- **Sticky on scroll** — nav should stay visible
- **Mobile**: hamburger menu or just logo + CTA

### 2. Hero Section

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   [Headline — 5-12 words]                        │
│   [Subheadline — 15-25 words, expands on above]  │
│                                                  │
│   [Primary CTA Button]    [Secondary CTA link]   │
│                                                  │
│   [Hero image / screenshot / demo gif]           │
│                                                  │
└──────────────────────────────────────────────────┘
```

Layout variants:
- **Split**: Text left, image right (most common)
- **Centered**: Text centered, image below (clean, minimal)
- **Full-width image**: Text overlaid on background (bold, visual)
- **Video background**: Text over looping video (attention-grabbing)

### 3. Social Proof Bar

```
Trusted by [number]+ [type of users]
[Logo] [Logo] [Logo] [Logo] [Logo]
```

Or: `★★★★★ Rated 4.9/5 by 500+ users`

Types:
- Client/partner logos
- User count ("Join 10,000+ founders")
- Rating/review aggregate
- Media mentions ("As seen in...")
- Award badges

### 4. Problem Section

```
## [Problem headline — name the pain]

[2-3 sentences describing the problem in the reader's words]

- Pain point 1 — specific consequence
- Pain point 2 — specific consequence
- Pain point 3 — specific consequence
```

Rules:
- Use the reader's language, not technical jargon
- Be specific: "spending 4 hours a week on invoices" > "wasting time on admin"
- Max 3 pain points — more dilutes the impact

### 5. Solution / Benefits Section

```
## [Solution headline — what changes with your product]

[Brief transition from problem to solution]

┌─────────┐  ┌─────────┐  ┌─────────┐
│ [Icon]  │  │ [Icon]  │  │ [Icon]  │
│Benefit 1│  │Benefit 2│  │Benefit 3│
│[Detail] │  │[Detail] │  │[Detail] │
└─────────┘  └─────────┘  └─────────┘
```

Rules:
- Lead with **benefits** (outcomes), not features (capabilities)
- 3–6 benefits, each with icon + short title + 1-2 sentence description
- "Save 10 hours a week" > "Automated workflow engine"

### 6. How It Works

```
## How It Works

Step 1: [Action]     →  Step 2: [Action]     →  Step 3: [Result]
[Brief description]     [Brief description]     [Brief description]
```

Rules:
- **Always 3 steps** — cognitive sweet spot
- Steps should feel effortless: "Sign up → Connect → Done"
- Last step should be the outcome, not an action
- Optional: include small screenshots or icons per step

### 7. Features Grid

```
┌────────────┐  ┌────────────┐  ┌────────────┐
│ [Icon]     │  │ [Icon]     │  │ [Icon]     │
│ Feature 1  │  │ Feature 2  │  │ Feature 3  │
│ [Benefit   │  │ [Benefit   │  │ [Benefit   │
│  sentence] │  │  sentence] │  │  sentence] │
└────────────┘  └────────────┘  └────────────┘
┌────────────┐  ┌────────────┐  ┌────────────┐
│ Feature 4  │  │ Feature 5  │  │ Feature 6  │
└────────────┘  └────────────┘  └────────────┘
```

Rules:
- Grid of 3, 4, or 6 (avoid odd numbers > 3)
- Each feature: icon + title + benefit-oriented description
- Title is the feature; description is the benefit

### 8. Testimonials

```
┌──────────────────────────────────────────┐
│ "[Specific result quote]"                │
│                                          │
│ — Name, Title/Company                    │
│   [Photo]                                │
└──────────────────────────────────────────┘
```

Rules:
- Use specific outcomes: "Reduced my billing time from 4 hours to 15 minutes"
- NOT generic praise: "Great product, highly recommend!"
- Include name + title + photo (or company) for credibility
- 2–3 testimonials is optimal; more than 5 causes scrolling fatigue
- If no real testimonials yet, skip this section entirely — don't fake it

### 9. Pricing

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│  Basic   │  │   Pro    │  │Enterprise│
│ $X/mo    │  │ $Y/mo    │  │ Contact  │
│          │  │ ★Popular │  │          │
│ Feature  │  │ Feature  │  │ Feature  │
│ Feature  │  │ Feature  │  │ Feature  │
│ [CTA]    │  │ [CTA]    │  │ [CTA]    │
└──────────┘  └──────────┘  └──────────┘
```

Rules:
- Highlight recommended plan
- Include monthly/annual toggle if applicable
- Show what's included AND what's not
- Anchor with the most expensive or most popular plan

### 10. FAQ

```
▶ Question 1?
  [Answer — direct, 1-3 sentences]

▶ Question 2?
  [Answer]
```

Rules:
- Address objections disguised as questions
- 5-8 questions max
- Lead with the most common objection
- Real questions > manufactured questions

### 11. Final CTA

```
┌──────────────────────────────────────────┐
│                                          │
│  [Recap headline — remind of value]      │
│  [One line of urgency or reassurance]    │
│                                          │
│        [Primary CTA Button]              │
│                                          │
│  [Risk reversal: "No credit card needed" │
│   or "30-day money-back guarantee"]      │
│                                          │
└──────────────────────────────────────────┘
```

## Product Type → Recommended Sections

| Product Type | Recommended Sections |
|-------------|---------------------|
| **SaaS tool** | Hero, Social Proof, Problem, Solution, How It Works, Features, Testimonials, Pricing, FAQ, Final CTA |
| **Digital product** (course, ebook, template) | Hero, Problem, Solution, What's Inside, Testimonials, Pricing, Guarantee, FAQ, Final CTA |
| **Service** (consulting, coaching) | Hero, Problem, Solution, How It Works, Testimonials, Founder/Team, Pricing, FAQ, Final CTA |
| **Waitlist / Coming Soon** | Hero (with email capture), Problem, Solution teaser, Social Proof (if any), How It Works preview |
| **Free tool / Open source** | Hero, How It Works, Features, Demo/Screenshot, Community proof, Getting Started |
| **Newsletter / Community** | Hero (with email capture), What You'll Get, Sample content, Social Proof, About the Creator |
| **Physical product** | Hero (with product photo), Benefits, How It Works, Testimonials, Specs, Pricing, Guarantee, FAQ |

## Page Type Templates

When the Strategy phase determines a page type, use the corresponding forced section order below. Do not add sections from other page types unless the user explicitly requests it.

### Waitlist Page (`page_type: "waitlist"`)

For pre-launch products with no evidence (Tier 3) or products not yet ready for full pages.

**Forced section order:**
1. **Navigation** — logo + "Join Waitlist" CTA button
2. **Hero** — teaser headline, subheadline describing what's coming, single email-capture field inline with submit button, reassurance text ("We'll only email you once, when we launch")
3. **Problem / Opportunity teaser** — 1-2 sentences only, not a full section with cards
4. **Solution preview** — what it will do, 3 bullet points maximum
5. **Timeline** — "Launching Q2 2026" or similar, with optional waitlist count ("432 people already waiting")
6. **Social proof bar** — only if available (beta signups count, advisor logos). Otherwise omit.
7. **Footer** — minimal: copyright only

**Sections explicitly excluded**: Pricing, FAQ (more than 2-3 items), Features grid, Testimonials, How It Works (full), Case Study, Comparison Table.

**Template**: Use `templates/waitlist-page.html` as structural reference.

---

### Demo / Booking Page (`page_type: "demo_booking"`)

For services or complex products where the conversion goal is scheduling a demo or consultation.

**Forced section order:**
1. **Navigation** — logo + "Book a Demo" / "Schedule a Call" CTA
2. **Hero** — outcome-focused headline, calendar/booking CTA button
3. **Credibility bar** — logos, client count, years of experience
4. **Process / How It Works** — what happens in the demo or engagement (3-4 steps)
5. **Scope section** — what's included, what's not
6. **Social proof / Testimonials** — if available. Otherwise omit.
7. **FAQ** — 3-5 objection-handling questions ("What happens after the demo?", "How long is the call?")
8. **Final CTA** — calendar/booking CTA repeated
9. **Footer** — contact info, privacy policy, terms

---

### Direct Purchase Page (`page_type: "direct_purchase"`)

For launched products with pricing where the conversion goal is an immediate purchase.

**Forced section order:**
1. **Navigation** — logo + primary buy CTA
2. **Hero** — benefit-led headline, primary buy CTA button
3. **Social proof bar** — user count, logos, ratings
4. **Problem section** — why the status quo is broken
5. **Solution / Benefits** — how the product solves it
6. **How It Works** — 3-step process
7. **Features grid** — if product has 3+ distinct features
8. **Pricing section** — **required**. Tiers or single price with clear what's-included.
9. **Risk reversal section** — **required**. Money-back guarantee, free trial, refund policy.
10. **FAQ / Objection handling** — 5-8 questions addressing purchase objections
11. **Testimonials** — if available. Otherwise omit.
12. **Final CTA** — buy action repeated with recap headline
13. **Footer** — privacy policy, terms of service, contact

---

### Service Lead-Gen Page (`page_type: "service_lead_gen"`)

For service businesses where the conversion goal is a contact form submission or quote request.

**Forced section order:**
1. **Navigation** — logo + "Get in Touch" / "Request a Quote" CTA
2. **Hero** — outcome-focused headline, "Get in Touch" CTA button
3. **Credibility section** — founder story, years of experience, notable clients, relevant background
4. **Methodology / How We Work** — process description (Discovery → Proposal → Engagement → Delivery), not product features
5. **Scope / What's Included** — clear deliverables, optionally with "What's Not Included" column
6. **Case studies or results** — if available. Otherwise omit.
7. **Testimonials** — if available. Otherwise omit.
8. **FAQ** — 5 questions focused on service objections ("How long does an engagement take?", "What if it's not a good fit?")
9. **Final CTA** — contact form (name + email + message) or calendar booking link
10. **Footer** — contact info, privacy policy, terms of service

**Template**: Use `templates/service-page.html` as structural reference.

## Above-the-Fold Checklist

The hero section (visible without scrolling) must answer these 3 questions in < 5 seconds:

1. **What is this?** → Headline
2. **Why should I care?** → Subheadline
3. **What do I do next?** → CTA button

If any of these are unclear, the visitor leaves.

---

## Mobile Considerations

- Hero image stacks below text on mobile
- Navigation collapses to hamburger or just CTA
- Feature grids stack vertically (1 column)
- CTAs should be full-width buttons on mobile
- Font sizes: minimum 16px body, 24px+ headlines
- Tap targets: minimum 44×44px
- Remove decorative elements that add scroll length without value
