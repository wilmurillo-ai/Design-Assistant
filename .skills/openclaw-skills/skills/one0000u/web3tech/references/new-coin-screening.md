# New Coin Screening

## Use This Path For

- New coin discovery
- Novelty-based screening
- Shortlist generation
- First-pass early risk review
- Watchlist creation

## Stage 1: Broad Screen

1. Run `web3tech_new_coins`.
2. Remove candidates with weak descriptions, vague positioning, or no sign of substance.
3. Keep only a small shortlist.

## Stage 2: Shortlist Review

1. Run `web3tech_new_coin_detail` for shortlisted names.
2. Look for genuine novelty, a clear angle, and evidence the project is more than marketing.
3. Use `web3tech_hot_coins` to cross-check if any candidates are already gaining traction.

## Stage 3: Early Risk Pass

Use deeper tools selectively:

- `web3tech_coin_analysis` for broader technical confirmation
- `web3tech_project_timeline` for execution evidence
- `web3tech_project_developers` for team credibility
- `web3tech_code_similarity` for clone or fork risk
- `web3tech_whitepaper_analysis` for top picks or unusually strong claims

## Ranking Guidance

Prefer a practical shortlist over a fake-precision leaderboard. Rank by:

- Novelty signal
- Clarity of thesis
- Technical plausibility
- Execution evidence
- Contributor credibility
- Obvious red flags or missing evidence

If evidence is thin, label the result as a `watchlist`, not a recommendation.

## Red Flags

- No clear differentiation
- No credible execution evidence
- Suspiciously generic whitepaper or pitch
- Obvious code reuse without explanation
- Thin or opaque contributor picture
- Heavy claims with little supporting detail
