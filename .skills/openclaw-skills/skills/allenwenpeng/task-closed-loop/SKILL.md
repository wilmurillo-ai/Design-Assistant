# 任务闭环管理技能

## 📚 技能概述

**任务闭环管理**是一种确保任务从创建到完成全程可追踪、可验证的管理方法。

**核心原则**: 凡事有交代，件件有着落，事事有回音

**学习时间**: 2026-04-10  
**适用范围**: 量化团队全体成员 (Master, Radar, Analyst, Risk, Coder)

---

## 🎯 任务闭环流程

### 阶段 1: 任务创建 (Create)

| 要素 | 说明 | 示例 |
|------|------|------|
| **任务名称** | 简洁明确 | 数据质量修复 |
| **任务描述** | 详细说明 | 修复 4 月 1-7 日数据重复问题 |
| **负责人** | 明确 Agent | Master(主导) + Radar + Coder |
| **截止时间** | 明确时间点 | 2026-04-10 20:00 |
| **优先级** | P0/P1/P2/P3 | P0(紧急重要) |
| **验收标准** | 可验证的结果 | 数据无重复，验证通过 |

**任务创建模板**:
```markdown
## 任务：[任务名称]

**描述**: [详细说明]
**负责人**: [Agent]
**截止时间**: YYYY-MM-DD HH:MM
**优先级**: P0/P1/P2/P3
**验收标准**:
- [ ] 标准 1
- [ ] 标准 2
```

---

### 阶段 2: 任务分配 (Assign)

**分配原则**:
1. **能力匹配** - 任务与 Agent 职责匹配
2. **负荷均衡** - 避免单 Agent 过载
3. **明确期望** - 清晰传达验收标准

**分配流程**:
```
Master 创建任务 → 确认负责人 → 负责人确认接收 → 开始执行
```

---

### 阶段 3: 任务执行 (Execute)

**执行要求**:
1. **进度汇报** - 每 2 小时汇报一次进展
2. **问题上报** - 遇到阻碍立即汇报
3. **变更申请** - 范围/时间变更需申请

**进度汇报模板**:
```markdown
【任务进度汇报】
任务：[任务名称]
当前进度：XX%
已完成：[完成内容]
待完成：[剩余内容]
问题/风险：[如有]
下一步：[下一步计划]
```

---

### 阶段 4: 任务验证 (Verify)

**验证流程**:
```
负责人提交 → Master 验证 → Boss 确认 → 闭环完成
```

**验证标准**:
1. **结果验证** - 是否达到验收标准
2. **质量验证** - 是否符合质量要求
3. **文档验证** - 是否有完整记录

**验证检查表**:
- [ ] 任务目标已达成
- [ ] 验收标准已满足
- [ ] 文档已归档
- [ ] 经验已总结

---

### 阶段 5: 任务总结 (Summarize)

**总结内容**:
1. **结果总结** - 任务完成情况
2. **经验教训** - 做得好的/需要改进的
3. **知识沉淀** - 形成文档/流程/规范

**总结模板**:
```markdown
## 任务总结：[任务名称]

### 结果
- 状态：✅ 完成 / ❌ 未完成
- 完成时间：YYYY-MM-DD HH:MM
- 实际耗时：X 小时

### 经验教训
**做得好的**:
1. ...
2. ...

**需要改进的**:
1. ...
2. ...

### 知识沉淀
- [ ] 文档已更新
- [ ] 流程已优化
- [ ] 规范已建立
```

## 📋 闭环管理红线

### 必须遵守的规则

1. **任务创建必须明确验收标准** - 无标准不创建
2. **任务执行必须进度汇报** - 无汇报不执行
3. **任务完成必须验证确认** - 无验证不关闭
4. **任务结束必须总结归档** - 无总结不闭环

### 禁止行为

- ❌ 任务无明确负责人
- ❌ 任务无明确截止时间
- ❌ 任务执行无进度汇报
- ❌ 任务完成无验证确认
- ❌ 任务结束无总结归档
