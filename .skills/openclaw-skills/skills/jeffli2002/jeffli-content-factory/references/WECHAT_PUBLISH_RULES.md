# 微信公众号发布规范补充：Jeff 文章优化规律（2026-04-02 新增）

来源：对比 Jeff 公众号最终版 vs Content Agent 输出版 → 小米SU7选址分析文章

---

## Jeff 优化的核心规律

### 1. 叙事视角：去掉"客户委托"外壳，改为"个人实验"
- **Agent输出**：帮一个新能源汽车客户做北京市场选址咨询，客户问：小米在北京到底开了多少家店？
- **Jeff改后**：最近一直在思考怎么把OpenClaw的能力和过往十年在汽车行业的业务场景做真正的融合。今天无意中看到地图API，眼睛一亮。
- **规律**：Jeff 会把商业咨询的壳子拆掉，改成「我自己做了一次实验」，更真实、更有个人色彩，读者更容易产生共鸣

### 2. 免责声明提前到文章最顶部
- **Agent输出**：免责声明通常在文末
- **Jeff改后**：免责声明直接放在正文第一段之前，作为文章的门
- **规律**：免责声明提前既保护自己，也建立读者信任（说明是真实实验而非软文）

### 3. 错误公开化：专门建「修正框」
- **Agent输出**：在正文叙述中提及西城区判断有误
- **Jeff改后**：专门建一个醒目的红色左边框框，写「⚠️重要修正」，把错误单拿出来说清楚
- **规律**：主动暴露错误并解释根因，反而增加可信度；读者会认为「这个人不怕承认错误，内容更可靠」

### 4. 竞品品牌用全称（但主品牌用*匿名）
- **Agent输出**：小*汽车，特斯拉，蔚来，比亚迪
- **Jeff改后**：小*汽车，特斯拉，蔚来（都用简称）
- **规律**：如果文章是分析某一品牌，该品牌用*匿名（规避平台审核风险），但竞品可以用全称

### 5. 关键结论用彩色背景高亮
- **Agent输出**：重要句子用 `color:#556b2f` 或 `border-bottom:dotted`
- **Jeff改后**：在关键句外层再加一层 `background-color:#fff8e7; padding:1px 4px; border-radius:3px;` 奶黄底色
- **规律**：重要结论用双重视觉标记（颜色+背景），在手机端更醒目

### 6. 删除所有自我介绍类结尾
- **Agent输出**：文末常有「我们下期再见」「如果你觉得文章对你有所帮助，请关注就行」等
- **Jeff改后**：保留CTA但移到结论之后、免责声明之前；删除「我们下期再见」
- **规律**：CTA紧跟结论，不要拖泥带水加一堆废话

### 7. 延伸阅读只保留最相关的1-2篇
- **Agent输出**：文章末尾加很多历史文章链接
- **Jeff改后**：只保留「内容工厂skill开源」这一篇
- **规律**：延伸阅读不是越多越好，选一篇最相关的，引流效果更集中

## Skill 改进建议

1. **Hook生成规则**：增加「叙事视角判断」，如果是商业分析文章，主动建议改成「个人实验/经历」视角
2. **免责声明**：默认插入到正文第一段之前，不放文末
3. **修正内容处理**：如果文章有「发现原判断有误」的情况，自动生成「⚠️修正框」
4. **重要结论高亮**：使用 `墨绿文字 + 奶黄背景` 双重视觉标记，不用纯文字样式
5. **结尾精简**：CTA后直接接免责声明，不加「下期再见」等废话
6. **延伸阅读**：限制最多2篇，选最相关的，不要堆砌
