# 智能文档识别功能说明

## 功能概述

智能文档识别功能允许用户通过上传护照、机票、酒店订单等文档，自动提取入境卡填写所需的信息，大大提升填写效率和准确性。

## 支持的文档类型

### 1. 📘 护照识别

#### 可提取信息
- 护照号码（Passport Number）
- 姓（Surname）
- 名（Given Name）
- 性别（Gender）
- 出生日期（Date of Birth）
- 国籍（Nationality）
- 护照签发国（Country of Issue）
- 护照有效期（Expiry Date）

#### 识别技术
- **MRZ 识别**：Machine Readable Zone（机器可读区）是护照底部的两行或三行字符，包含所有关键信息
- **OCR 技术**：光学字符识别，提取护照信息页的文字内容
- **AI 视觉理解**：使用 Claude 的视觉能力理解护照布局和内容

#### 最佳实践
```
✅ 推荐做法：
- 拍摄护照信息页（有照片的那一页）
- 确保光线充足，避免反光
- 护照平放，四个角都在画面内
- MRZ 区域（底部两行字符）清晰可读

❌ 避免：
- 模糊、倾斜的照片
- 部分遮挡或裁剪
- 过度曝光或阴影
```

#### 识别示例
```
输入：护照照片
输出：
✅ 已识别护照信息：
- 姓名：ZHANG SAN
- 护照号：E12345678
- 出生日期：01/01/1990
- 性别：Male
- 国籍：China
- 签发国：China
- 有效期：31/12/2030

MRZ 原始数据：
P<CHNZHANG<<SAN<<<<<<<<<<<<<<<<<<<<<<<<<<<<
E123456780CHN9001011M3012315<<<<<<<<<<<<<<06
```

### 2. ✈️ 机票识别

#### 可提取信息
- 航班号（Flight Number）
- 出发日期（Departure Date）
- 抵达日期（Arrival Date）
- 出发城市/机场（Departure City/Airport）
- 目的地城市/机场（Destination City/Airport）
- 航空公司（Airline）
- 订单号/PNR（Booking Reference）
- 乘客姓名（Passenger Name）

#### 识别技术
- **结构化数据提取**：识别电子机票的标准格式
- **关键字段定位**：定位航班号、日期、城市等关键信息
- **日期格式转换**：自动转换为标准格式（DD/MM/YYYY）

#### 支持的机票格式
- 电子机票 PDF
- 航空公司 APP 截图
- 邮件中的机票确认单
- 第三方订票平台（携程、飞猪等）的订单截图

## 信息整合与验证

### 自动整合流程

1. **文档分类**
   - 自动识别上传的文档类型（护照/机票/酒店）
   - 支持一次性上传多个文档

2. **信息提取**
   - 从每个文档中提取关键信息
   - 验证提取信息的格式和完整性

3. **数据整合**
   - 将多个文档的信息整合到统一的数据结构
   - 自动填充入境卡所需的字段
   - 标记已提取和待补充的字段

4. **信息确认**
   - 生成信息摘要供用户确认
   - 允许用户修正错误信息
   - 补充缺失的信息（如邮箱、手机号）

## 技术实现

### 使用的工具和技术

1. **read_image 工具**
   - 读取用户上传的图片
   - 支持 JPG、PNG、HEIC 等格式
   - 自动处理图片方向和大小

2. **Claude 视觉能力**
   - 理解文档布局和结构
   - 提取文字和关键信息
   - 验证信息的合理性

3. **MRZ 解析**
   - 解析护照 MRZ 区域
   - 验证校验位
   - 提取标准化信息

4. **正则表达式**
   - 验证护照号格式
   - 验证日期格式
   - 提取航班号、订单号等

### 识别准确性

| 文档类型 | 准确率 | 常见问题 | 解决方案 |
|---------|--------|---------|---------|
| 护照 | 95%+ | 模糊、反光 | 重新拍照，确保清晰 |
| 机票 | 90%+ | 格式多样 | 支持主流平台格式 |

## 优势与限制

### ✅ 优势

1. **节省时间**
   - 无需手动输入 10+ 个字段
   - 一次上传，多次使用
   - 平均节省 5-10 分钟

2. **减少错误**
   - 避免手动输入的拼写错误
   - 避免日期格式错误
   - 避免姓名顺序错误

3. **提高准确性**
   - 直接从官方文档提取
   - 自动格式验证
   - 信息一致性检查

4. **用户体验好**
   - 操作简单，拍照即可
   - 即时反馈
   - 支持多文档批量上传

### ⚠️ 限制

1. **依赖文档质量**
   - 需要清晰的照片或扫描件
   - 模糊、倾斜的照片可能识别失败

2. **格式多样性**
   - 不同平台的机票、酒店订单格式不同
   - 可能需要人工补充或修正

3. **隐私考虑**
   - 护照等敏感信息需要安全处理
   - 建议使用后删除上传的文档

4. **网络依赖**
   - 需要上传文档到云端处理
   - 大文件可能上传较慢

## 安全与隐私

### 数据处理原则

1. **最小化原则**
   - 只提取必要的信息
   - 不保存原始文档
   - 处理完成后删除临时数据

2. **加密传输**
   - 文档上传使用 HTTPS 加密
   - 敏感信息不明文存储

3. **用户控制**
   - 用户可以选择不上传文档
   - 用户可以手动输入信息
   - 用户可以修正识别错误

4. **透明度**
   - 明确告知用户提取了哪些信息
   - 允许用户确认和修正
   - 提供信息删除选项

## 常见问题

### Q1: 识别失败怎么办？
**A:** 如果识别失败，可以：
1. 重新拍照，确保清晰
2. 手动输入信息
3. 联系客服获取帮助

### Q2: 支持哪些语言的文档？
**A:** 目前主要支持：
- 中文
- 英文
- 中英文混合

### Q3: 识别需要多长时间？
**A:** 通常：
- 护照：2-5 秒
- 机票：3-8 秒
- 酒店订单：3-8 秒

### Q4: 我的信息安全吗？
**A:** 是的，我们：
- 使用加密传输
- 不保存原始文档
- 处理完成后删除数据
- 符合隐私保护法规

### Q5: 可以识别旧版护照吗？
**A:** 可以，只要：
- 护照信息页清晰
- MRZ 区域完整
- 照片光线充足

### Q6: 识别错误可以修正吗？
**A:** 可以，系统会：
- 显示识别结果供确认
- 允许用户手动修正
- 重新验证修正后的信息

### Q7: 支持批量处理吗？
**A:** 支持，可以：
- 一次上传多个文档
- 自动分类和提取
- 批量确认信息

### Q8: 需要特殊的拍照技巧吗？
**A:** 建议：
- 光线充足
- 护照平放
- 避免反光
- 四角完整
- 对焦清晰
