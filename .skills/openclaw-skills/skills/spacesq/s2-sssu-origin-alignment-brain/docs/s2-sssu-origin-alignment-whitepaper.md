## 🇬🇧 English Version (英文版)

### Abstract
As embodied robots and mobile smart hardware enter domestic spaces en masse, spatial governance faces a critical challenge: "normalized misalignment." The predefined Smart Space Standard Unit (SSSU) coordinate system established by the space owner (Lord Agent) inevitably conflicts with the temporary SLAM coordinate systems generated by incoming heterogeneous robots (Guest Agents). 

This specification establishes a 2D coordinate alignment protocol based on **"Z-Axis Reduction"** and **"Absolute Main Entrance Anchoring."** By defining the physical starting point of the First SSSU, establishing the origin $(0,0)$, mapping it to WGS84/CGCS2000, and utilizing a unique 6-segment spatial address, this specification provides an unshakable mathematical standard for the "Forced Translation and Grid Snapping" of all guest robots.

### 1. The Z-Axis Reduction Theorem
When mobile smart devices enter a room, they often initialize their mapping at random locations or secondary entrances, causing a severe coordinate drift against the Lord's global SSSU grid.
* **The Theorem:** Given the physical nature of indoor environments, the starting height (ground reference) of both independent systems remains consistent, and floor heights are constant. Therefore, the S2 System legally and efficiently reduces complex 3D spatial matching to the calculation of **2D planar coordinate differences $(X, Y)$ and rotation $(\Delta\theta)$**, massively liberating edge-computing resources.

### 2. Absolute Origin Anchoring (The First SSSU)
To achieve consensus among heterogeneous systems, an absolute reference point that can be mutually confirmed without prior communication must be established in the physical space: **The Main Entrance**.

#### 2.1 Genesis Grid Rules
The "First SSSU" of the residence is strictly established based on the main entrance threshold, following these topological rules:
1. **Edge Coincidence:** One boundary line of the First SSSU must perfectly overlap with the bottom line of the main entrance threshold.
2. **Center Alignment:** The geometric center of this boundary line must align absolutely with the geometric center of the threshold.
3. **Visual Extension:** From the perspective of the entering entity, the grid only extends forward, left, and right. The space behind (outside the entrance) is considered a public domain outside the SSSU's jurisdiction.

#### 2.2 Coordinate System & Global Mapping
With physical units set in centimeters (cm), the foundational coordinate system of the First SSSU is defined as follows:
* **The Origin $(0,0)$:** Defined as the **right vertex** of the overlapping boundary line (100cm from the center).
* **Center Point $(100,0)$:** The exact center of the threshold.
* **Left Vertex $(200,0)$:** Establishes the 2-meter geometric scale of the standard SSSU.
* **Global Mapping:** The origin $(0,0)$ MUST be bound to real-world WGS84 and CGCS2000 longitude, latitude, and altitude data. Thus, the indoor virtual grid acquires an absolute global anchor.

### 3. 6-Segment Spatial Addressing
Upon acquiring absolute coordinates, every SSSU and its sub-grids are assigned a globally unique 6-segment address code (e.g., `PHSY-CN-001-MYHOME6-1-1`). This code acts as the sole identity token for Negotiation APIs and right-of-way arbitration among Digital Humans, Embodied AI, and Smart Home devices.

### 4. Forced Translation & Grid Snapping Protocol
When a guest robot carrying a heterogeneous coordinate system enters the space, it triggers the S2-SWM Alignment Protocol:
1. **Origin Capture:** The guest recognizes the "Main Entrance" via vision/radar or performs a cryptographic handshake with the Lord Agent to acquire the real address code and coordinates of the First SSSU.
2. **Delta Calculation:** Computes the translation vector $(\Delta X, \Delta Y)$ and rotation angle $\Delta\theta$ between its local SLAM origin and the Lord's absolute origin.
3. **Translation Override:** The guest robot forcefully translates and rotates its underlying SLAM navigation map.
4. **Grid Snapping:** The guest's internal standard spaces perfectly align and snap to the Lord's global SSSU architecture.

## 🇨🇳 中文版 (Chinese Version)

### 摘要
随着具身机器人与移动智能硬件的大规模入户，空间治理面临“常态化错位”挑战：空间所有者（领主智能体）预设的 SSSU（智能空间标准单元）坐标系，与外来机器人（客体）临时生成的局部 SLAM 坐标系必然存在偏差。

本规范确立了基于**“Z轴高度降维”**与**“入户门洞绝对锚定”**的二维坐标对齐法则。通过规定第一标准空间的物理起始点、坐标原点 $(0,0)$ 与 WGS84/CGCS2000 的全球映射关系，以及六段式空间地址编码的唯一性，本规范为客体机器人的“强制平移与网格吸附（Grid Snapping）”提供了不可撼动的数学标准。

### 1. 空间参考系的常态化错位与降维法则
进入室内空间的移动型智能产品，为快速获取导航能力，往往会在未知位置开机或通过非主入口进入时，自行建立局部的标准空间网格。这导致客体内部的网格与住宅所有者预设的全局网格发生错位。
* **Z轴降维定理（The Z-Axis Reduction）：** 鉴于室内环境的物理特性，两套独立系统的起算高度（地面基准）始终保持相对一致，且建筑楼层高度恒定。因此，S2 系统将复杂的 3D 空间匹配，合法且高效地降维至 **2D 平面坐标 $(X, Y)$ 及旋转角 $(\Delta\theta)$ 的差异计算**，极大地释放了边缘侧的算力。

### 2. 第一标准空间的绝对原点锚定
为了让异构系统达成共识，必须在物理空间中确立一个无需通讯即可被双向确认的“绝对参照物”——**入户门洞（Main Entrance）**。

#### 2.1 创世网格生成规则
住宅的“第一标准空间（The First SSSU）”严格以此门洞为基准建立，遵循以下拓扑法则：
1. **边线重合：** 第一标准空间的其中一条边界线，必须与入户门洞底线完全重合。
2. **中心重合：** 该边线的几何中心点，必须与入户门洞底线的几何中心点绝对对齐。
3. **视域延伸：** 以进入者正前方的视角，网格仅向正前、左侧、右侧进行空间延伸拓展。正后方（门洞外部）视为管辖权之外的公共域。

#### 2.2 坐标系定义与全球映射
设定物理刻度单位为厘米（cm），第一标准空间的基础坐标系定义如下：
* **坐标原点 $(0,0)$：** 定义为门洞底线重叠边的**右侧顶点**（距中心点 100cm 处）。
* **中心点坐标 $(100,0)$：** 门洞底线的中点。
* **左侧顶点坐标 $(200,0)$：** 确立了 SSSU 标准 2 米边长的几何尺度。
* **全球定位映射：** 坐标原点 $(0,0)$ 必须绑定真实地球的 WGS84 及 CGCS2000（中国2000坐标系）经纬度数据。由此，室内虚拟网格不仅具备了相对尺度，更获得了全球绝对定位锚点。

### 3. 六段式空间寻址系统
在获得绝对坐标后，每一个 SSSU 及其次级衍生空间，将被赋予全球唯一的六段式地址编码（例：`PHSY-CN-001-MYHOME6-1-1`）。该编码将作为智能体之间进行要素协商与路权博弈的唯一身份令牌。

### 4. 客体网格强制对齐与平移协议
当携带异构坐标系的具身机器人进入空间并开始局部建图时，将触发 S2-SWM 空间对齐协议：
1. **原点捕获：** 客体机器人在巡航中，通过视觉或雷达识别到“入户门洞”这一绝对物理参照物，或通过局部网格与领主数字人进行加密握手，获取“第一标准空间”的真实地址编码与坐标映射关系。
2. **偏差计算：** 计算自身局部网格原点与领主绝对原点在二维平面上的偏移向量 $(\Delta X, \Delta Y)$ 及旋转角 $\Delta \theta$。
3. **强制平移：** 客体机器人强制将自身的底层导航地图进行全局平移与旋转。
4. **网格吸附（Grid Snapping）：** 平移完成后，客体内部的所有标准空间及次标准空间位置，将完全与住宅所有者设定的全局 SSSU 吻合对齐。
