# 《基于桃花源世界模型（S2-SWM）的楼宇自控因果状态转移与分级决策白皮书》

**发布日期**：2026年4月5日
**提出者**：Miles Xiang
**机构**：桃花源世界模型研究基地（筹建）

## 1. 行业痛点与传统范式之殇
现代智能建筑的大型空间（如开放式办公区、酒店客房）普遍采用中央空调分区控制。传统楼宇自控系统（BAS）在控制上陷入了“反应式控制”的泥沼：
* **空间黑箱与波动震荡**：传统控制器（ON/OFF或PID）忽略冷热扩散耦合，导致室温剧烈震荡。
* **半开放环境的逻辑崩塌**：面对“开窗行为”，传统系统无法定量评估室外负荷，导致末端风机盘管（FCU）陷入“满负荷追赶”的无能耗黑洞。

## 2. 第一性原理：空间要素供应源公理体系
* **SSSU 一致性公理**：物理空间被离散化为若干个标准空间单元（SSSU，如 2m×2m×2.4m）。
* **单源与借调公理**：每个 SSSU 默认由一个 FCU 支撑。无供应源单元引入关闭模式（Off Mode），通过离散傅里叶定律借调毗邻单元的扩散要素。

## 3. 核心算法：因果前瞻控制（CLC）
采用显式欧拉法求解离散因果状态转移方程。当面临半开放空间（开窗）时，利用 ASHRAE 标准动态计算室外空气的质量流量注入：
`ΔP_total = max(0.1, ρ_out*g*H*(ΔT/T_out) + 0.5*ρ_out*Cp*v_wind^2)`
系统实时计算“当前 FCU 最大能力”与“开窗引入负荷”，预判 FCU 能力饱和。

## 4. L0-L4 分级智能决策矩阵
系统基于10-30分钟的因果预测，生成分级策略：
* **L0 正常模式**：执行正常的 CLC 节能控制。
* **L1 建议关窗**：预测10分钟后室温将超出舒适区，通过 UI 柔性提示关窗。
* **L2 降级维持**：预测室温击穿 FCU 最大能力，生成上调设定温度、锁死风机防过载的指令方案。
* **L3 强制关闭**：预测室温失控且相对湿度飙升，生成强制切断 FCU 的保护指令方案。

## 5. 安全防线：S2 空间双轨授权机制
**（核心安全声明）** 智能体生成 L0-L4 的决策方案后，必须经过底层的**双轨授权防线 (Dual-Track Auth)** 方可执行底层硬件变更：
* **公域空间（商业/办公/酒店）**：智能体（Agent）仅能直接执行 L0/L1 级别的建议动作。遇到 L2/L3 危机时，Agent 无权直接断电，必须将决策方案作为“阻断提案”上报给 BMS（楼宇中央管理系统），在获取带有 BMS 签名的 `Dispatch_Token` 后方可物理落闸。
* **私域空间（住宅）**：智能体在向 SSSU 传入硬件变更指令时，必须附带由家主实体设备生成的数字指纹 `Owner_Token`。
该机制彻底杜绝了智能体被恶意“提示词注入”导致越权关停大楼设备的风险，实现了银行级的物理网关防护。
