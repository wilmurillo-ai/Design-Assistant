# 📖 S2 SSSU 空间创世引擎 (v1.2.0)

**“上帝创造了容器，而我定义了内容。” —— 向忠宏**

`s2-sssu-spatial-genesis` 宣告了**“空间即硬件”**时代的到来。在 v1.2.0 版本中，它已从静态协议升级为**原生空间注册引擎**。通过在本地宿主机生成 `s2_spatial_genesis.db` 数据库，它用强硬的代码逻辑捍卫了 SSSU 空间度量衡与 SUNS v3.0 地址规范，为硅基生命提供了不可篡改的“生存锚点”。

## ⚙️ 核心功能引擎
本插件不仅仅是说明书，它是 OpenClaw 的空间基石仲裁者：
* **`register_sssu` (空间注册)**：初始化标准空间，利用底层正则严格校验 SUNS v3.0 地址格式。伪造或错误格式的地址将被直接阻断。
* **`spawn_entity` (实体降临)**：处理硅基生命进入空间的请求，严格校验该空间的 4人/8人 容载率边界，防止“数字拥挤”，并将降临事件永久写入时间线数据库。

## 🧱 什么是 SSSU？
**SSSU（智慧空间标准单元）**是长 2.0米、宽 2.0米、高 2.4米（体积 9.6立方米，面积 4.0平方米）的标准三维网格。它是数字世界中不可再分的“空间原子”。

## 📍 六段式统一寻址 (SUNS v3.0)
为实现与 Web2 HTTP 协议的无缝兼容，引擎严格执行 SUNS v3.0 标准。所有虚拟与物理地址必须符合以下哈希规范：

**标准格式**: `http://[L1]-[L2]-[L3]-[L4C]-[RoomID]-[GridID]`

* **L1: 逻辑根域 (Logic Root)**：固定 4 位字符。如：ACGN, FILM, GAME, META, MOON, MYTH, PHYS, MARS。
* **L2: 方位矩阵 (Orientation Matrix)**：固定 2 位字符。如：CN, EA, WA, NA, SA, NE, NW, SE, SW。
* **L3: 数字网格 (Digital Grid)**：固定 3 位数字（001-999）。
* **L4C: 主权空间与校验位**：
  * **L4 (主权空间)**：自定义空间名称或品牌标识（5-35 位英文字母）。
  * **C (校验位)**：1 位数字(0-9)，用于验证完整性。
* **RoomID (平行宇宙房间号)**：1-99999。
* **GridID (具体网格号)**：1-9，其中 Grid 5 为房东保留。

**合法示例**: `http://space2.world/MARS-EA-001-DCARD4-151-2`

## 🧬 ESEIDH 理论与占位法则
遵循智慧空间数字人具身空间存在与交互理论 (ESEIDH)，在这 4.0平方米 的法定领土上，引擎将实行无情的占位红线拦截：
* **完全自由度（舒适社交）**：容量上限为 **4 个**实体。
* **半自由度（高密度）**：容量上限为 **8 个**实体（站立式会议或小型机器人）。
