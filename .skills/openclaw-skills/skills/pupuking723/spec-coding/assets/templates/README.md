# 规格模板说明

## 模板列表

### feature-spec.md - 功能规格模板

**适用场景**：
- 新业务功能开发
- 用户故事实现
- 后台管理功能
- 数据处理流程

**核心章节**：
- 背景与目标（问题描述、目标用户、成功指标）
- 输入/输出规格（表格形式）
- 行为规格（正常流程、边界情况、错误处理）
- 非功能需求（性能、兼容性、安全性）
- 验收标准（Checklist）

---

### api-spec.md - API 规格模板

**适用场景**：
- REST API 设计
- GraphQL Schema 定义
- Webhook 事件规范
- 第三方集成接口

**核心章节**：
- 接口概述（路径、认证、速率限制）
- 请求规格（Header、路径参数、查询参数、请求体）
- 响应规格（成功响应、错误响应、状态码）
- 边界情况与错误处理
- 验收测试用例（curl 示例）

---

### component-spec.md - 组件规格模板

**适用场景**：
- React/Vue 组件开发
- UI 控件设计
- 表单组件
- 交互组件

**核心章节**：
- 组件概述（路径、类型、依赖）
- Props 规格（TypeScript 接口 + 详细说明表）
- 状态规格（内部状态、状态流转图）
- 行为规格（用户交互、自动行为）
- UI 规格（布局结构、响应式断点、状态样式）
- 可访问性（A11y）检查项
- 测试用例（Jest/Vitest 示例）

---

## 使用建议

1. **不要删减核心章节** - 每个章节都有其价值
2. **表格优先** - 能用表格的不用段落
3. **示例驱动** - 每个规格点配一个示例
4. **状态标记** - 始终在文档顶部标注当前状态
