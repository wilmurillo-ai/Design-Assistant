# 聚合支付错误码

> 本页返回码主要用于排查和联调定位，不建议把 `resp_code` 直接写成订单终态逻辑；订单终态仍以 `trans_stat`、异步通知和主动查询结果为准。

## 通用错误码

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 00000000 | 本次接口处理完成，实际交易结果仍以 `trans_stat` / 查询结果为准 | 继续结合交易状态字段确认终态 |
| 00000100 | 交易正在处理中 | 等待异步通知或轮询查询 |
| 10000000 | 无效参数 | 检查必填字段、格式、长度 |
| 98888888 | 未知系统错误 | 联系汇付技术支持 |
| 99999999 | 系统异常，请稍后重试 | 稍后重试或联系技术支持 |

## 参数校验类 (10000000)

| 返回描述 | 处理建议 |
|---------|---------|
| 请求内容体不能为空 | 检查请求 body |
| %s不能为空 | 检查对应必填字段 |
| %s长度固定%d位 | 检查字段长度 |
| %s最大长度为%d位 | 缩短字段值 |
| %s的传入枚举[%s]不存在 | 检查枚举值是否合法 |
| %s不符合%s格式 | 检查日期/金额等格式 |

## 下单类错误码

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 20000000 | 重复交易 | 使用新的 req_seq_id |
| 20000001 | 操作过于频繁 | 稍后重试 |
| 22000000 | 产品配置信息异常 | 检查 product_id 配置 |
| 22000002 | 商户配置信息异常 | 检查 huifu_id |
| 90000000 | 交易受限 / 单笔金额超限 / 交易存在风险 | 查看 resp_desc 详情 |

## 查询类错误码

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 20000004 | 交易不存在 | 检查流水号是否正确 |
| 21000000 | 参数逻辑校验不合法（流水号、全局流水号不能同时为空） | 至少传入一个查询条件 |

## 关单类错误码

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 10000015 | 不允许关闭一分钟以内的订单 | 等待一分钟后重试 |
| 10000016 | 原订单已为终态,请发起查询交易获取 | 订单已完成，无需关单 |
| 10000018 | 关单失败 | 查看 resp_desc 详情 |
| 23000000 | 原交易订单已失败不允许关单 / 关单状态为终态 | 订单已完成或已关单 |
| 23000004 | 不支持的交易（银联二维码不支持关单） | 银联不支持关单 |

## 退款类错误码

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 10000001 | 原交易没有处理完成 | 等待原交易完成 |
| 10000002 | 退款金额大于可退金额 | 检查退款金额 |
| 10000009 | 该交易不支持部分退款 | 只能全额退款 |
| 21000000 | 原交易请求流水号、商户单号、全局流水号不能同时为空 | 至少传入一个原交易标识 |
| 22000004 | 暂未开通退款权限/分账退款权限 | 联系汇付开通权限 |
| 23000002 | 数据权限不足 / 退款手续费承担方不一致 | 检查手续费配置 |
| 23000003 | 金额校验异常（退款额>可退额 / 余额不足） | 检查退款金额和账户余额 |
| 23000004 | 不支持的交易（预授权撤销/优惠交易部分退款） | 使用其他方式处理 |
| 30000000 | 调用收银台退款接口失败 | 稍后重试 |
| 90000000 | 交易受限 / 可用余额不足 / 交易存在风险 | 查看 resp_desc 详情 |

## 对账单查询错误码

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 00000000 | 查询成功 | 正常返回 |
| — | 当前huifuId请求过于频繁 | 每天不要超过 3 次查询 |

## 使用边界

- `resp_code`、`resp_desc` 主要用于排查，不直接驱动订单成功/失败终态。
- `00000000` 不等于“交易最终成功”，仍要结合 `trans_stat`、异步通知和查单结果判断。
- `00000100` 表示当前接口仍在处理中，应该继续等待异步通知或轮询查询。
- 其他返回码优先结合 `resp_desc`、请求参数、渠道场景和原交易标识排查。
