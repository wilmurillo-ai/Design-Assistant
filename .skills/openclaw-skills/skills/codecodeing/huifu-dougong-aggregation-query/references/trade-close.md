## 接口概览

| 项 | 值 |
| --- | --- |
| 接口名称 | 扫码交易关单 |
| 汇付端点 | `POST /v2/trade/payment/scanpay/close` |
| 支持格式 | `JSON` |
| 适用对象 | 已开通微信/支付宝权限的商户 |
| 不支持场景 | 银联、数字货币订单不支持关单 |
| 签名说明 | [接入指引-开发指南](https://paas.huifu.com/open/doc/guide/#/api_v2jqyq) |

服务商/商户系统通过本接口发起订单关闭请求。原交易已是终态时，关单会失败。

## SDK 映射

| 项 | 值 |
| --- | --- |
| SDK Request 类 | `TradePaymentScanpayCloseRequest` |
| 包路径 | `com.huifu.dg.lightning.models.payment` |
| SDK Client 方法 | `Factory.Payment.Common().close(request)` |

`TradePaymentScanpayCloseRequest` 常用字段映射：

| 字段 | setter | 必填 | 说明 |

| `req_date` | `setReqDate()` | Y | 请求日期，`yyyyMMdd` |
| `req_seq_id` | `setReqSeqId()` | Y | 请求流水号，同一 `huifu_id` 下当天唯一 |
| `huifu_id` | `setHuifuId()` | Y | 商户号 |
| `org_req_date` | `setOrgReqDate()` | Y | 原交易请求日期 |
| `org_hf_seq_id` | `setOrgHfSeqId()` | C | 原交易汇付全局流水号，二选一 |
| `org_req_seq_id` | `setOrgReqSeqId()` | C | 原交易请求流水号，二选一 |

## 公共参数

### 公共请求参数

| 参数 | 定义 | 长度 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `sys_id` | String | 32 | Y | 渠道商或商户的 `huifu_id` |
| `product_id` | String | 32 | Y | 汇付分配的产品号，例如 `MCS` |
| `sign` | String | 512 | Y | 请求签名 |
| `data` | JSON | - | Y | 业务请求参数 |

### 公共返回参数

| 参数 | 定义 | 长度 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `sign` | String | 512 | Y | 响应签名 |
| `data` | JSON | - | N | 业务返回体 |

## 请求参数

### `data`

| 参数 | 定义 | 长度 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `req_date` | String | 8 | Y | 请求日期，格式 `yyyyMMdd` |
| `req_seq_id` | String | 128 | Y | 请求流水号，同一 `huifu_id` 下当天唯一 |
| `huifu_id` | String | 32 | Y | 商户号 |
| `org_req_date` | String | 8 | Y | 原交易请求日期，格式 `yyyyMMdd` |
| `org_hf_seq_id` | String | 128 | C | 原交易返回的全局流水号，和 `org_req_seq_id` 二选一 |
| `org_req_seq_id` | String | 128 | C | 原交易请求流水号，和 `org_hf_seq_id` 二选一 |

### 原交易定位规则

- `org_req_date` 必填。
- `org_hf_seq_id`、`org_req_seq_id` 不能同时为空。
- 关单仅支持微信、支付宝交易；银联二维码、数字货币订单不支持。
- 不允许关闭一分钟以内的订单。

## 同步返回参数

### `data`

| 参数 | 定义 | 长度 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `resp_code` | String | 8 | Y | 业务响应码，见[业务返回码](#业务返回码) |
| `resp_desc` | String | 512 | Y | 业务响应信息 |
| `huifu_id` | String | 32 | Y | 商户号 |
| `req_date` | String | 8 | Y | 请求日期，原样返回 |
| `req_seq_id` | String | 128 | Y | 请求流水号，原样返回 |
| `org_req_date` | String | 8 | Y | 原交易请求日期 |
| `org_req_seq_id` | String | 128 | N | 原交易请求流水号 |
| `org_hf_seq_id` | String | 128 | N | 原交易全局流水号 |
| `org_trans_stat` | String | 1 | N | 原交易状态，`P` 处理中，`S` 成功，`F` 失败 |
| `trans_stat` | String | 1 | Y | 关单状态，`P` 处理中，`S` 成功，`F` 失败 |

### 结果判定

- `resp_code=00000000` 且 `trans_stat=S`：关单成功。
- `resp_code=00000100` 或 `trans_stat=P`：关单处理中，应继续调用 [close-query.md](close-query.md) 确认结果。
- `trans_stat=F`：关单失败，应结合 `resp_desc` 和 `org_trans_stat` 判断是否因原交易已终态或不支持关单导致失败。

## Webhook

斗拱交易完成后除同步返回外，也支持发送 webhook 关单事件 `trans.close`。可用于驱动业务系统中的订单关闭后置流程。

参考文档：
[webhook 的使用说明](https://paas.huifu.com/open/doc/devtools/#/webhook/webhook_jieshao?id=%e5%a6%82%e4%bd%95%e4%bd%bf%e7%94%a8webhook)

## 业务返回码

| 返回码 | 返回描述 |
| --- | --- |
| `00000000` | 交易成功 |
| `00000100` | 交易处理中 |
| `10000000` | 入参数据不符合接口要求 |
| `20000001` | 不允许关闭一分钟以内的订单 |
| `20000001` | 并发冲突，请稍后重试 |
| `21000000` | 原请求流水号和原全局流水号不能同时为空 |
| `22000000` | 产品号不存在 |
| `22000000` | 产品号状态异常 |
| `23000000` | 原订单已为终态，无法发起关单操作 |
| `23000000` | 关单状态为终态，不能重复关单 |
| `23000001` | 原交易不存在 |
| `23000004` | 原订单为银联二维码交易，不支持关单 |
| `90000000` | 业务执行失败 |
| `98888888` | 系统错误 |
| `99999999` | 系统异常，请稍后重试 |

## 实现备注与文档勘误

1. 官方把返回示例标题写成“成功示例”，但示例体中的 `resp_code=10000016`、`trans_stat=F`，实际是失败示例，不应按成功场景理解。
2. 官方返回码表未列出 `10000016`，但返回示例出现了该值，含义从示例文案看是“原订单已为终态，请改为查询交易结果”。
3. `resp_code` 表示接口调用结果，关单最终状态仍应结合 `trans_stat` 判断；处理中场景不要直接判定失败。
4. 若 `org_trans_stat=S`，说明原交易已支付成功，后续应走退款而不是重复关单。
