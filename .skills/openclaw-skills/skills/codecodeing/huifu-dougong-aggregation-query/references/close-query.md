## 接口概览

| 项 | 值 |
| --- | --- |
| 接口名称 | 扫码交易关单查询 |
| 汇付端点 | `POST /v2/trade/payment/scanpay/closequery` |
| 支持格式 | `JSON` |
| 适用对象 | 开通微信、支付宝、云闪付、数字人民币权限的商户 |
| 查询限制 | 只能通过原交易查询关单 |
| 签名说明 | [接入指引-开发指南](https://paas.huifu.com/open/doc/guide/#/api_v2jqyq) |

服务商或商户发起关单请求后，若未收到关单结果，可通过本接口查询关单状态。

## 请求参数

### `data`

| 参数 | 定义 | 长度 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `req_date` | String | 8 | Y | 请求日期，格式 `yyyyMMdd` |
| `req_seq_id` | String | 128 | Y | 请求流水号，同一 `huifu_id` 下当天唯一 |
| `huifu_id` | String | 32 | Y | 商户号 |
| `org_req_date` | String | 8 | Y | 原交易请求日期，格式 `yyyyMMdd` |
| `org_req_seq_id` | String | 128 | C | 原交易请求流水号，和 `org_hf_seq_id` 二选一 |
| `org_hf_seq_id` | String | 128 | C | 原交易返回的全局流水号，和 `org_req_seq_id` 二选一 |

### 查询规则

- 只能通过原交易维度查询关单状态，不能仅靠关单请求本身定位。
- `org_req_date` 必填。
- `org_req_seq_id`、`org_hf_seq_id` 不能同时为空。
- 查询返回中的 `trans_stat` 是关单状态，不是原交易支付状态。

## 同步返回参数

### `data`

| 参数 | 定义 | 长度 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| `resp_code` | String | 8 | Y | 业务响应码，见[业务返回码](#业务返回码) |
| `resp_desc` | String | 512 | Y | 业务响应信息 |
| `huifu_id` | String | 32 | Y | 商户号 |
| `req_date` | String | 8 | Y | 请求日期，原样返回，格式 `yyyyMMdd` |
| `req_seq_id` | String | 128 | Y | 请求流水号，原样返回 |
| `org_req_date` | String | 8 | Y | 原交易请求日期 |
| `org_req_seq_id` | String | 128 | N | 原交易请求流水号 |
| `org_hf_seq_id` | String | 128 | N | 原交易全局流水号 |
| `org_trans_stat` | String | 1 | N | 原交易状态，`P` 处理中，`S` 成功，`F` 失败 |
| `trans_stat` | String | 1 | Y | 关单状态，`P` 处理中，`S` 成功，`F` 失败 |

### 结果判定

- `resp_code=00000000` 表示查询动作成功，不代表关单成功。
- 关单最终状态以 `trans_stat` 为准。
- 当 `trans_stat=S` 时，表示关单成功。
- 当 `trans_stat=F` 时，表示关单失败；可结合 `org_trans_stat` 判断原交易是否已成功。
- 当 `trans_stat=P` 时，表示关单仍处理中，应继续轮询或等待系统回执。

## 业务返回码

| 返回码 | 返回描述 |
| --- | --- |
| `00000000` | 查询成功 |
| `10000000` | 入参数据不符合接口要求 |
| `21000000` | 原请求流水号和原全局流水号不能同时为空 |
| `22000000` | 产品号不存在 |
| `22000000` | 产品号状态异常 |
| `23000001` | 原交易不存在 |
| `98888888` | 系统错误 |

## 实现备注与文档勘误

1. 官方把返回示例标题写成“成功示例”，这里的“成功”应理解为查询成功，不是关单成功；示例中的 `trans_stat=F` 明确表示关单失败。
2. 该接口是“关单查询”，但官方说明“只能通过原交易查询关单”，实现时不要误以为可以只依赖关单请求流水号查询。
3. 当 `org_trans_stat=S` 且 `trans_stat=F` 时，通常表示原交易已支付成功，关单未生效，后续应走退款流程而不是重复关单。
4. 官方关单查询页的“适用对象”口径包含云闪付、数字人民币，但这不代表发起关单接口支持这些渠道；能否发起关单仍以 [trade-close.md](trade-close.md) 中“仅支持微信、支付宝”的限制为准。
