# 框架集成提示：React / Vue / 原生 JS

> 这份文档不提供完整工程模板，只给出当前版本最有用的集成边界提示，避免把 checkout-js 误解成某个前端框架专属方案。

## 总原则

当前 Skill 提供的是组件式前端支付能力，不绑定 React、Vue 或原生 JS。

无论你用什么框架，核心都只有三步：
1. 在页面生命周期中初始化 `HFPay`
2. 提供稳定的 `createPreOrder` 与 `callback`
3. 在容器节点挂载 `checkout` 或单按钮组件

## React 提示

- 在 `useEffect` 中初始化并挂载组件。
- 确保 `createPreOrder`、`callback` 不要在每次 render 时无意义重建。
- 组件卸载时，至少移除你自己加的事件、loading 状态和轮询逻辑。

## Vue 提示

- 在 `onMounted` 中初始化并挂载组件。
- 不要把支付组件实例直接塞进深层响应式对象，避免调试和序列化噪音。
- 页面切换时，保留好订单上下文，方便回调后通知后端确认。

## 原生 JS 提示

- DOM Ready 后再初始化 `HFPay`。
- 容器节点必须先存在再 `mount()`。
- 如果页面会局部重渲染，注意不要重复挂载多个同类组件。

## 所有框架都要注意

- 不要在前端存放 RSA 私钥。
- 不要把支付成功判断做死在前端状态机里。
- `createPreOrder` 返回字段名保持官方示例使用的下划线命名。
- 页面层可以传对象给自有服务端，但服务端调用汇付接口时仍要遵循服务端文档的 JSON 字符串约束。

## 当前版本不做什么

- 不提供 React / Vue 完整 demo 工程
- 不生成脚手架
- 不提供 TypeScript 类型包

这些能力可以后续再扩，但当前上架版本至少应把接入边界讲清楚。
