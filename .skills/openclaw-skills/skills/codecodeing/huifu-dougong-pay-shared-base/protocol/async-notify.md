# 异步通知规则

这份文档只说异步通知的公共规则。  
不绑定某一种语言。

## 基本规则

- 通知方式：HTTP(S) POST
- 编码：UTF-8
- 默认超时：5 秒
- 失败重试：3 次
- 不支持 HTTP 重定向
- URL 不要带参数
- 自定义端口建议在 `8000-9005`

## 返回要求

商户收到通知后，需要：

1. 返回 HTTP `200`
2. 响应体返回 `RECV_ORD_ID_` + 请求流水号

如果没有按这个要求返回，汇付会继续重试。

## 回调数据体

这里有一个容易混淆的点：

- 支付交易类回调，业务数据一般在 `resp_data`
- 商户配置类回调，业务数据一般在 `data`

处理时不要把这两个字段混用。

## 验签规则

- 回调也要验签
- 验签原文是回调里的业务数据体
- 异步验签不需要再做字段排序

## 幂等要求

同一条异步通知可能会发多次。  
所以业务系统必须做幂等。

建议至少做到：

- 以 `hf_seq_id` 作为幂等键
- 业务落库有唯一约束
- 已处理过的通知直接返回成功

## 状态确认

异步通知很重要。  
但它不能代替主动查询。

更稳的做法是：

1. 收到异步通知先验签
2. 做幂等判断
3. 再调用查询接口做二次确认
4. 最后再落业务状态

特别是非终态订单，不能只靠前端回跳或单次通知判断成功。

## 不应该放到这里的内容

这份文档不放下面这些内容：

- `@PostMapping`
- `HttpServletRequest`
- `Flask`、`Gin`、`FastAPI`、`ASP.NET` 的具体写法

这些都属于语言或工程实现层。
