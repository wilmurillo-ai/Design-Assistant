# V2 接口签名规则

这份文档只说协议规则。  
不绑定某一种语言。

## 适用范围

适用于 `https://api.huifu.com/v2/` 下的接口。

## 请求报文结构

```json
{
  "sys_id": "调用方 huifu_id",
  "product_id": "汇付分配的产品号",
  "sign": "签名结果",
  "data": {
    "业务字段": "值"
  }
}
```

重点只有一条：

- **只对 `data` 做签名**

## 签名规则

1. `data` 必须是 JSON 对象，不是 JSON 字符串。
2. 只对 `data` 第一层字段参与签名。
3. 字段名按 ASCII 字典序排序。
4. 多层对象如果存在，按 JSON 字符串传入，不参与内层排序。
5. 使用 RSA 私钥按 `SHA256WithRSA` 签名。
6. 签名结果 Base64 编码后放到 `sign` 字段。

## 验签规则

### 同步返回

同步返回验签时：

1. 取返回里的 `data`
2. 按 ASCII 字典序排序第一层字段
3. 用汇付 RSA 公钥验签

### 异步返回

异步返回验签时：

- **不需要重新排序**
- 直接按回调里的原始 `data` 或 `resp_data` 验签

这是同步和异步最容易搞混的地方。

## 建模规则

下面这条很重要：

- 文档里写 `String(JSON Object)` 或 `String(JSON Array)`  
  只代表 SDK 边界是字符串。  
  不代表业务层应该直接拼裸字符串。

更稳的做法是：

1. 业务层先正常建对象
2. 校验通过后再统一序列化
3. SDK 边界再转成字符串字段

## 不应该放到这里的内容

这份文档不放下面这些内容：

- Java 的 `setProcutId()` 这种 SDK 特有坑
- Spring Boot 写法
- fastjson、Jackson、Newtonsoft 之类库差异
- 某个语言 SDK 的类名

这些都应该放到语言适配层。
