# 服务端 SDK 能力矩阵

这份表有两个用途：

1. 告诉读者官方支持哪些语言
2. 告诉读者当前仓库在 `v1.1.0` 里先覆盖到什么程度

## 官方语言支持

| 语言 | 官方安装方式 | 最低运行时 | 官方包/仓库 |
| --- | --- | --- | --- |
| Java | Maven | JDK 1.8+ | `dg-java-sdk` / `dg-lightning-sdk` |
| PHP | Composer | PHP 7.4+ | `huifurepo/dg-php-sdk` |
| C# | NuGet | .NET 4.8+ | `dg-csharp-sdk` |
| Python | pip | Python 3.x | `dg-sdk` |
| Go | go mod | Go 1.16+ | `bspay-go-sdk` |

## 当前仓库在 v1.1.0 的覆盖方式

| 能力 | Java | PHP | C# | Python | Go |
| --- | --- | --- | --- | --- | --- |
| 协议规则说明 | ✅ | ✅ | ✅ | ✅ | ✅ |
| SDK 安装说明 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 初始化说明 | ✅ | ✅ | ✅ | ✅ | ✅ |

## v1.1.0 的落地原则

这一版先做“最小可用”。

也就是说：

- 先把语言入口建起来
- 先把协议层抽出来
- 先把初始化和最小示例说明补齐

完整 Demo、复杂工程模板、自动生成能力，放到后续版本。
