# 前端 SDK 能力矩阵

当前 v1.1.0 的前端能力定位是：商户自定义收银台页面 + 内嵌支付组件。

## 当前入口

| 项目 | 说明 |
| --- | --- |
| Skill | `huifu-dougong-hostingpay-checkout-js` |
| 逻辑定位 | `frontend-base + hostingpay-checkout adapter` |
| 页面归属 | 商户侧自定义页面 |
| 组件能力 | `checkout` 组件、支付宝按钮、微信按钮、云闪付按钮 |
| 结果回调 | `callback` 事件模型 |
| 最终结果确认 | 服务端查询 / 异步通知 |

## 组件化接入边界

当前前端能力不是模板型前端工程，也不是一套固定页面。  
它要求商户自己拥有页面，再把汇付支付组件嵌入到现有页面流里。

你需要自己负责：

- 页面布局、品牌样式、订单展示
- 调用服务端预下单接口
- 监听 `callback` 事件并驱动自己的页面状态
- 在支付完成后回到服务端查单或等待异步通知确认最终结果

汇付前端 SDK 当前负责：

- 渲染 `checkout` 组件
- 提供支付宝 / 微信 / 云闪付单支付按钮
- 按当前环境决定展示哪些支付方式
- 回传前端支付过程中的 `callback` 事件

## 环境展示规则

- 支付宝内：只展示支付宝
- 微信内：只展示微信
- 云闪付内：只展示云闪付
- 系统浏览器：默认展示支付宝与微信

## 当前版本覆盖范围

`v1.1.0` 先覆盖：

- SDK 安装
- `HFPay` 初始化
- `createPreOrder` 约定
- `pre_order_type` 对应关系
- `callback` 回调结构
- 环境相关的支付方式展示

这一版不做：

- 前端工程模板
- React/Vue/原生 JS 三套完整示例
- 自动生成前端代码
