## 文档勘误与实现备注

- 官方顶部“支持的支付方式”列表漏写了 `T_APP`，但请求参数 `trade_type` 枚举明确包含 `T_APP`；当前按参数表收录。
- 官方请求参数把 `req_date` 标成 `N`，但 SDK 示例和回调字段都依赖它；实现时仍建议始终传入。
- 官方请求参数把 `method_expand` 标成 `Y`，但是否真正必填取决于 `trade_type`；例如 `A_NATIVE`、`U_NATIVE` 并不是每次都有强制子字段。
- 官方 `A_JSAPI` / `A_NATIVE` 表中 `body` 与重复的 `ali_promo_params` 行被拼到了同一行；当前按两个独立字段理解。
- 同步返回里字段名是 `trade_type`，异步回调里字段名是 `trans_type`；不要混用。
- 同步返回 `trade_type` 的官方枚举含 `D_NATIVE`、`T_H5`、`T_NATIVE`，但当前请求参数页并未把这些值列入下单枚举；不要据此自行扩展请求值。
- 官方另有 Webhook 能力说明，但它属于事件分发机制，不是本接口固定响应体结构。
