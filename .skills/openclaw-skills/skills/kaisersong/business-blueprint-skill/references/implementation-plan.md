在现有 SKILL.md 末尾添加 "Architecture Diagram Generation" 章节：

```markdown
## Architecture Diagram Generation

When user requests an architecture diagram (keywords: "架构图", "architecture diagram",
"--export", "diagram"):

1. Read `references/architecture-design-system.md` for the complete design system.
2. Read the appropriate template from `references/architecture-templates/` based on the user's domain.
3. Read the blueprint JSON (if available) or extract entities from user's text.
4. Generate a self-contained HTML file with inline SVG following the design system rules.
5. Write the output file to the same directory as the blueprint JSON.

### Generation Rules
- Use dark mode by default (#020617 bg + 40px grid)
- L→R data flow: Clients(左) → Frontend → Backend → Database(右)
- Map `systems[].category` to semantic colors
- Map `systems[].properties.type == "aws"` → AWS Region boundary box
- Use `flowSteps[].seqIndex` for L→R ordering
- Component sizing: 0-1 cap = small(60px), 2-4 = medium(80px), 5+ = large(120px)
- Z-order: arrows → masks → styles → text → legend

### Output
- Single HTML file: `{blueprint_name}.html`
- No external dependencies (except Google Fonts CDN for JetBrains Mono)
- Opens in any browser, printable to PDF
```

def _build_architecture_svg(blueprint: dict[str, Any], colors: dict, theme: str) -> str:
    """Return the architecture SVG for the HTML viewer.
    
    In v2, the SVG content is expected to be pre-generated by the Agent
    following the architecture design system. If not available, falls back
    to the classic three-layer layout via export_svg.
    """
    # Fallback: use classic three-layer layout
    import io
    buf = io.StringIO()
    from .export_svg import export_svg
    from pathlib import Path
    import tempfile
    with tempfile.NamedTemporaryFile(suffix=".svg", delete=False) as f:
        tmp_path = Path(f.name)
    export_svg(blueprint, tmp_path, theme=theme)
    result = tmp_path.read_text(encoding="utf-8")
    tmp_path.unlink()
    return result

**或者更简洁的方案：直接删除 `_build_architecture_svg()`，改为 `export_html_viewer()` 接受 `svg_content` 参数：**
```python
def export_html_viewer(blueprint: dict[str, Any], target: Path, theme: str = "light",
                       svg_content: str | None = None) -> None:
    """Generate self-contained HTML viewer.
    
    If svg_content is provided, embed it directly. Otherwise, skip SVG
    entirely (the viewer becomes a JSON inspector only).
    """
    if svg_content is None:
        # Generate fallback SVG
        import tempfile
        from .export_svg import export_svg
        with tempfile.NamedTemporaryFile(suffix=".svg", delete=False) as f:
            tmp_path = Path(f.name)
        export_svg(blueprint, tmp_path, theme=theme)
        svg_content = tmp_path.read_text(encoding="utf-8")
        tmp_path.unlink()
    # ... rest of HTML template with svg_content embedded
```

### 测试脚本

```bash
# tests/test_agent_generated_diagram.sh
# 用法：Agent 生成 aws-serverless.html 后运行此脚本

SVG_FILE="aws-serverless.html"  # 或 .svg，取决于输出格式
PASS=0
FAIL=0

check() {
    local desc="$1" pattern="$2" file="$3"
    if grep -q "$pattern" "$file"; then
        echo "PASS: $desc"
        PASS=$((PASS+1))
    else
        echo "FAIL: $desc — expected '$pattern'"
        FAIL=$((FAIL+1))
    fi
}

check "Dark mode bg" '#020617' "$SVG_FILE"
check "40px grid" 'width="40"' "$SVG_FILE"
check "AWS Region box" 'stroke-dasharray="8,4"' "$SVG_FILE"
check "Region amber stroke" '#F59E0B' "$SVG_FILE"
check "Lambda green stroke" '#34D399' "$SVG_FILE"
check "DynamoDB violet stroke" '#A78BFA' "$SVG_FILE"
check "Arrow marker" 'arrow-solid' "$SVG_FILE"
check "Summary cards" 'summary-card' "$SVG_FILE"
check "Node count (7)" 'class="node"' "$SVG_FILE"  # 至少 7 处

echo ""
echo "Results: $PASS passed, $FAIL failed"
[ "$FAIL" -eq 0 ] && exit 0 || exit 1
```
