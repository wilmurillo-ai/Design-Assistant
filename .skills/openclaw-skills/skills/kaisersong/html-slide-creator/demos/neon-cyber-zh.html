// 问题所在

演示文稿不应该需要
构建系统

→ 痛点 01
复杂的工具链
npm install、webpack 配置、300MB node_modules。只是为了做幻灯片。令人难以接受。
→ 痛点 02
通用输出
每个 AI 演示文稿看起来都一样——Inter 字体、紫色渐变。零信号、最大噪音。
→ 痛点 03
缺乏所有权
封闭平台。导出限制。月度费用。你的幻灯片被扣为人质。
03 / 08
// 界面

真正有效的
风格探索

你无法用言语描述设计偏好。slide-creator 生成 3 个视觉预览——你反应，而不是描述。

01//
询问氛围
输入气感。温暖、锐利、俏皮、技术风。自然语言处理。
02//
生成 3 个迷你预览
并排呈现三个不同的视觉系统。零配置。
03//
选择看起来对的那个
直觉获胜。这就是整个架构。

// 工作流程

从创意到演示
仅需分钟

01//
/slide-creator
回答几个问题。内容、氛围、受众、语调。最多两分钟。
02//
从 3 个预览中选择
对并排呈现的三种视觉风格做出反应。直觉胜于色彩理论。
03//
打开你的 HTML
方向键 → 空格 → 滑动。任何浏览器。零安装。
