为什么做幻灯片这么痛苦

PPT

  • 软件笨重，启动慢
  • 协作冲突频发
  • 格式在不同设备间崩坏

Keynote

  • 仅限 macOS
  • 导出兼容性差
  • 学习曲线陡峭

Markdown

  • 排版简陋
  • 无动画、无视觉效果
  • 不像"成品"
痛点：开发者需要一个能快速生成、视觉精美、零依赖的演示方案。

一句话生成完整演示文稿

核心功能演示

播放模式、编辑模式、演讲者视图 — 全在浏览器内运行

F5 全屏，任何屏幕都完美适配

快捷键功能
F5进入播放模式
Esc退出播放模式
ArrowRight / Space下一页
ArrowLeft上一页
B黑屏 / 恢复
P打开演讲者视图
# 播放模式快捷键速查
F5 → 全屏   Esc → 退出   B → 黑屏   P → 演讲者视图

浏览器内直接编辑，Ctrl+S 保存

E

进入编辑模式

左上角热区悬停显示 ✎ Edit，点击进入 contenteditable 编辑

N

备注面板

每页独立备注，演讲者视图中同步显示

S

Ctrl+S 保存

浏览器原生保存，修改直接写入 HTML 文件

提示：编辑模式不影响播放模式，切换自如。

演讲者视图，备注计时器一应俱全

主窗口

  • 观众看到全屏幻灯片
  • F5 进入，Esc 退出
  • 黑屏模式（B 键）

演讲者窗口

  • 实时备注显示
  • 计时器（从打开开始）
  • 当前页 / 总页数
  • 独立翻页控制
通信机制：通过 BroadcastChannel 实现两个窗口实时同步，无需服务器。

从想法到演示，只需一个命令

命令行为耗时
/slide-creator --plan生成 PLANNING.md 大纲~1 min
/slide-creator --generate从大纲生成 HTML3-5 min
/slide-creator --review16 项检查点优化~2 min
两种深度：自动模式快速出稿，精修模式深度规划叙事结构与视觉锁定。
