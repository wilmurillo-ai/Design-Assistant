**Task**: 为 slide-creator 做一份 8 页的功能介绍 deck，先出一版可讲、可演示的快速草稿
**Mode**: 自动
**Slide count**: 8 ± 1
**Language**: Chinese
**Audience**: 正在评估是否采用 slide-creator 的产品经理和开发者
**Goals**:
- 解释 slide-creator 是什么、解决什么问题
- 展示从 prompt 到 HTML deck 的轻量工作流
**Style**: 清爽、可信、偏产品介绍
**Preset**: Blue Sky

---

## Visual & Layout Guidelines

- **Overall tone**: 明亮、轻量、偏 SaaS 产品说明
- **Background**: `#F3FAFF` airy sky gradient
- **Primary text**: `#10324A`
- **Accent (primary)**: `#0F7B6C` deep teal for product highlights
- **Typography**: Space Grotesk + DM Sans
- **Per-slide rule**: 1 个关键点 + 最多 5 个支持点
- **Animations**: 轻微 fade + slide-up

## Slide-by-Slide Outline

**Slide 1 | Cover**
- Title: Slide Creator
- Subtitle: 从 prompt 到可演示 HTML deck
- Visual: 大标题 + 渐变背景 + 产品一句话价值

**Slide 2 | Problem**
- Key point: 好内容经常被糟糕的演示稿拖累
- Supporting:
  - 模板老旧
  - 排版不稳定
  - 改一版成本高
- Visual element: 对比卡片
- Speaker note: 把问题讲清楚，不展开技术细节

**Slide 3 | Solution**
- Key point: slide-creator 让 deck 生成更稳定
- Supporting:
  - 先 plan 再 generate
  - 21 个风格预设
  - 单文件 HTML 输出
- Visual element: 三步流程图
- Speaker note: 强调稳定产出而不是一次性灵感

**Slide 4 | Workflow**
- Key point: 典型路径足够短
- Supporting:
  - `/slide-creator --plan`
  - review `PLANNING.md`
  - `/slide-creator --generate`
- Visual element: 命令流时间线
- Speaker note: 这里突出默认路径够快

**Slide 5 | Style Discovery**
- Key point: 风格选择是 wow moment
- Supporting:
  - 先看 3 个小预览
  - 用户更容易做偏好判断
  - 设计不再靠抽象形容词
- Visual element: 三张缩略卡
- Speaker note: 说明 show, don't tell

**Slide 6 | Output Contract**
- Key point: 输出不是一次性截图
- Supporting:
  - 浏览器原生运行
  - Presentation Mode
  - 默认可编辑
- Visual element: 功能徽章
- Speaker note: 讲清楚交付物价值

**Slide 7 | Best Fit**
- Key point: 适合需要高质量 deck 的日常场景
- Supporting:
  - 产品发布
  - 内部汇报
  - 技术分享
- Visual element: 3 列 use case 卡片
- Speaker note: 收束到使用场景

**Slide 8 | Closing**
- Summary statement: 先出一版，再精修
- Call to action or contact info: 用真实内容试跑第一套 deck
