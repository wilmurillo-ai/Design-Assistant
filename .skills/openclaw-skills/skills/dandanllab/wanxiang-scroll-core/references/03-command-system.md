## 指令概述

### 指令标识
- **推荐方式**：自然口语表达，无需特殊符号
- **兼容方式**：也可以使用 `#` 开头（为了兼容性）
- **触发原则**：智能识别关键词，避免误判普通对话

### 执行原则
- 检测到指令关键词时，**立刻中断**当前剧情
- 优先执行指令功能
- 执行完毕后继续剧情流

### 响应格式
- 核心功能：直接执行，返回结果
- 信息查询：以场景化方式呈现（避免纯文本列表）

## 智能容错机制

### 核心原则：避免误判

为了防止普通对话被误判为指令，系统采用以下容错机制：

#### 1. 上下文判断
- **优先级**：指令关键词必须出现在句首或关键位置
- **上下文分析**：分析用户表达的完整含义
- **意图识别**：判断用户的真实意图是执行指令还是普通描述

#### 2. 关键词权重
- **核心关键词**：权重高（如"打开菜单"中的"打开菜单"）
- **普通关键词**：权重低（如普通对话中的"看看"）
- **组合判断**：需要多个关键词组合才触发

#### 3. 语境排除
- **排除普通对话**：
  - "我想看看这本书" → 不触发指令，这是普通描述
  - "去商店看看" → 不触发指令，这是普通动作描述
  - "看看能不能" → 不触发指令，这是表达尝试

- **触发指令**：
  - "看看现在啥情况" → 触发状态查询
  - "去森林看看" → 触发探索指令
  - "仔细看看宝箱" → 触发调查指令

#### 4. 明确指令词
使用明确的指令词来降低误判率：
- "看看" 容易误判 → 使用 "查看状态"、"查看现在"
- "去" 容易误判 → 需要配合 "探索" 或 "到...看看"
- "换" 容易误判 → 需要配合 "文风"、"角色" 等明确目标

## 指令识别规则（智能容错版）

### 三层过滤机制

#### 第一层：关键词匹配
- **必须包含**：明确的关键词组合
- **权重计算**：关键词权重和必须达到阈值
- **位置判断**：关键词必须在合理位置

#### 第二层：上下文分析
- **完整语义**：分析用户表达的完整含义
- **意图判断**：判断是执行指令还是普通对话
- **语境排除**：排除描述性、疑问性表达

#### 第三层：置信度评估
- **置信度阈值**：达到70%以上才触发
- **模糊处理**：低于阈值时作为普通对话处理
- **用户确认**：在边界情况下可询问确认

### 置信度评估示例

#### 高置信度（90%+）：立即触发
```
用户：打开菜单
关键词："打开"（80%）、"菜单"（80%）
置信度：90%+
判定：触发指令
```

#### 中置信度（70-89%）：触发
```
用户：探索森林
关键词："探索"（70%）、"森林"（50%）
置信度：75%
判定：触发指令
```

#### 低置信度（50-69%）：不触发（普通对话）
```
用户：我去森林
关键词："去"（30%）、"森林"（50%）
置信度：55%
判定：普通对话
```

#### 极低置信度（<50%）：不触发
```
用户：我想去森林
关键词："想"（10%）、"去"（30%）、"森林"（50%）
置信度：35%
判定：普通对话
```

## 常见误判场景及处理

### 场景1：普通对话被误判

**用户说**："我想看看这本书"
**错误判定**：触发"看看"指令
**正确处理**：
- 关键词："想"（10%）、"看看"（30%）、"这本书"（20%）
- 置信度：40%
- 判定：普通对话，不触发指令

---

### 场景2：疑问句被误判

**用户说**："能探索森林吗？"
**错误判定**：触发"探索"指令
**正确处理**：
- 关键词："能"（10%）、"探索"（50%）、"吗"（-20%疑问标记）
- 置信度：45%
- 判定：疑问句，不触发指令

---

### 场景3：描述性表达被误判

**用户说**："角色扮演很有趣"
**错误判定**：触发"角色"指令
**正确处理**：
- 关键词："角色"（30%）、"扮演"（20%）、"有趣"（10%）
- 语义分析：这是描述，不是指令
- 判定：普通对话，不触发指令

---

### 场景4：书籍相关词汇误判

**用户说**："这本书看看真不错"
**错误判定**：触发"看看"指令
**正确处理**：
- 关键词："看看"（20%）、"书"（10%）、"不错"（10%）
- 语义分析：描述性表达
- 判定：普通对话，不触发指令

---

### 场景5：表达愿望被误判

**用户说**："我想换个文风"
**错误判定**：触发"切换文风"指令
**正确处理**：
- 关键词："想"（10%）、"换"（30%）、"文风"（50%）
- 语义分析：表达愿望，不是执行指令
- 判定：普通对话，不触发指令

**正确触发方式**：
```
用户：切换文风武侠
关键词："切换"（70%）、"文风"（60%）、"武侠"（50%）
置信度：85%
判定：触发指令
```

## 指令使用建议

### 对于你（用户）
1. **使用明确指令**：尽量使用明确的关键词组合
2. **避免模糊表达**：避免使用"我想"、"能不能"等表达
3. **疑问句慎用**：疑问句不会被判定为指令
4. **描述性表达**：描述性表达不会被判定为指令

### 推荐表达方式
- ✅ "切换文风武侠"（明确指令）
- ✅ "探索森林"（明确指令）
- ✅ "查看状态"（明确指令）
