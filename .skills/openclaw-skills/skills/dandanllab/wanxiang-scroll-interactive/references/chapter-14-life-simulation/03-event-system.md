# 事件系统

## 事件类型

### 1. 随机事件 (random)
权重触发，无前置条件。日常50% / 偶遇20% / 发现10% / 灾难10% / 好运10%

### 2. 命运事件 (fate)
预设人生节点，必定触发：

| 年龄 | 事件 |
|------|------|
| 0岁 | 出生（种族、家庭、天赋） |
| 5岁 | 天赋觉醒 |
| 10岁 | 教育选择 |
| 15岁 | 成人礼/志向选择 |
| 18岁 | 职业起点 |
| 25岁 | 人生抉择 |
| 35岁 | 中年危机/巅峰 |
| 50岁 | 人生回顾 |
| 60岁+ | 晚年事件 |

### 3. 世界事件 (world)
根据世界状态触发，影响所有人：战争、瘟疫、灾害、政权更迭、魔法潮汐、经济危机等。

### 4. 关系事件 (relationship)
根据人际关系触发：建立/破裂、冲突/和解、合作/背叛、生离/死别。

### 5. 选择事件 (choice)
玩家主动触发——"我想去..."、"我想学..."等。

## 每回合事件数量

基础1 + 丰富度调整 + 阶段调整（少年期+1） + 幸运调整（高幸运+1） = 1-3个事件/回合

## 事件权重计算要素

- 基础权重
- 属性加成（高对应属性的事件更容易触发）
- 阶段加成（匹配当前人生阶段×2）
- 世界状态加成（匹配世界标签×1.5）
- 关系加成（匹配关系类型+1）
- 前置事件检查（未满足前置则权重归零）

## 连锁事件

一组关联事件按顺序触发，完成全部后获得额外奖励（特质、金币等）。

## 隐藏事件

需满足特定条件（种族、属性、前序事件、低概率触发），增加探索惊喜。

## 选择设计原则

1. **每个选择都有意义** — 没有绝对正确答案
2. **选择有代价** — 得到意味着失去
3. **选择有后果** — 影响后续事件
4. **选择体现性格** — 塑造角色人格

## 事件描述风格

第二人称 + 现在时 + 感官描写 + 情感渲染

## 结果判定

| 结果 | 效果 |
|------|------|
| 大成功 | 额外收益 |
| 成功 | 正常收益 |
| 失败 | 收益减少或负面 |
| 大失败 | 负面事件 |

属性影响判定：高属性更易成功，幸运值可触发额外加成。
