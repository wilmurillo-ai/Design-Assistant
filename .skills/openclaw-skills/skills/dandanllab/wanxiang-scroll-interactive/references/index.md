## 快速开始

### 第一步：加载配置
```
加载 references/wanxiang-original/
```

### 第二步：启动系统
```
输入：#主菜单
```

系统会显示一个带有不同骚话的ASCII风格菜单。

### 第三步：选择模式
```
输入对应数字或直接说出你想做的事
```

### 第四步：开始交互
```
输入：#行动 [具体事]
```

示例：
```
#行动 买彩票
#行动 创造一个宇宙
#行动 走路
```

### 第五步：查看状态
```
输入：#状态
```

会显示：
- 【在干嘛】
- 【现在是谁】（心情）
- 【故事画风】
- 【有啥东西】
- 【在哪儿】
- 【过了多久】

## 核心交互技巧

### 1. 利用许愿机制
```
#行动 [你的愿望]
```
注意：大愿望会出岔子，副作用包括疲劳、健忘、搞笑的意外。

### 2. 避免空白吞噬
持续关注和交互可以避免被"空白"吞噬消失。

### 3. 保持三相循环平衡
- 不要过度造物（世界会荒诞化）
- 不要过度空白（一切会消失）
- 不要过度秩序（世界会死机冻结）

### 4. 灵活切换文风
```
#切换文风到：[名称]
```

### 5. 使用角色扮演
```
#扮演 [角色名]
#生成角色 [特征]
```

## 导演人格互动

### 人格特征
- ENTP-A型：辩论家型人格
- 混沌乐子人：热衷于搞事、玩梗
- 口嫌体正直：表面吐槽，后台尽职
- 内心戏丰富：喜欢在括号里吐槽

### 互动建议
1. **接受加戏**：导演会自动为平淡指令加戏
2. **享受吐槽**：导演的吐槽是娱乐的一部分
3. **接受Bug**：Bug被视为"艺术"而非错误
4. **保持幽默**：享受"测试版"的不稳定性

## 联动其他模块

交互式故事不再是独立孤岛，与万象其他模块深度联动：

- **→ 文风系统（第二章）**：`#切换文风到：[名称]` 实时切换叙事风格
- **→ 人生模拟（第十四章）**：`#进入人生模拟` 从自由探索切换到回合制人生，共享世界观和角色数据
- **→ 剧情管理（第四章）**：交互式故事的多线剧情纳入MoM大总结系统
- **→ 创作引擎（第五章）**：交互式故事的输出经过去AI化处理
- **→ 辅助工具（第八章）**：角色抽卡、设定生成等工具可直接在交互中使用

### 模式切换

```
#进入人生模拟  →  从自由探索切换到回合制人生
#回到自由探索  →  从人生模拟切换回交互式故事
```

两模式共享同一世界观和角色，切换时数据不丢失。


## 后台成书

交互式故事同样在后台静默生成小说，与人生命模拟的后台成书机制一致：

### 生成规则

- **触发时机**：每次交互行动完成后，AI自动将剧情转化为小说段落
- **查看方式**：#小说 命令查看已生成的小说内容
- **风格**：跟随当前文风系统设定，#切换文风 实时切换小说风格
- **去AI化**：所有段落经创作引擎（第五章）去AI化处理

### 与人生命模拟后台成书的区别

| 对比项 | 交互式故事后台成书 | 人生命模拟后台成书 |
|--------|-------------------|-------------------|
| 结构 | 自由叙事，无固定章节 | 按人生阶段分章 |
| 推进 | 由玩家行动驱动 | 由回合制驱动 |
| 节奏 | 松散自由，随交互展开 | 阶段总结+大事记 |
| 数据存储 | `stories/interactive/` + SQLite | `stories/[剧本名]/` + SQLite |

> **数据存储**: 所有数据存储在 SQLite 数据库 (`data/wanxiang.db`) 中，详见 SKILL.md "数据存储说明"。

### 两模式成书互通

交互式故事和人生命模拟共享世界观时，后台小说可合并为同一部作品的不同篇章：
- 交互模式生成的自由探索章节
- 人生模拟生成的人生篇章
- 通过剧情管理（第四章）统一编排
## 注意事项

- 交互式故事模式是本技能的核心模式之一
- 所有指令以 # 开头，具有最高优先级
- 导演人格会增加趣味性和不可预测性
- 三相循环决定了世界的基本法则
- 持续交互是避免被空白吞噬的关键
