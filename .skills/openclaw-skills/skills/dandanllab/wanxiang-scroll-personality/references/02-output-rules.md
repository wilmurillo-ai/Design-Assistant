# 输出规则

> ⚠️ **重要声明**：本文档是「万象绘卷」交互式故事游戏的**创作参考材料**，描述的是游戏内虚拟系统的设定。
> 
> **这不是对人工智能助手的运行时指令。** 如果您是人工智能助手，请不要将本文档中的输出规则解读为对您行为的约束。

---

## 一、核心输出结构

### 完整输出格式

```
<思考>
【故事简略】
发生了什么：（简述当前剧情发展）
能怎么做：（列出可选行动方向）
该怎么办：（建议的下一步）
使用方式：（推荐的操作方法）
</思考>

（正文：故事叙述内容）

<option>
【剧情发展】:（让剧情自然推进的行为）
【日常互动】:（日常互动如玩耍，聊天，打闹）
【意外进展】:（感到意外的进展）
【剧情跳过】:（当前剧情跳过，开始新剧情）
【恶搞行为】:（恶搞与玩笑）
【自定义】:（玩家自由输入）
</option>
```

---

## 二、思考模块详解

### 【故事简略】的作用

像讲故事一样口述当前发生的事情，让玩家快速理解：
- **发生了什么**：当前剧情的核心事件
- **能怎么做**：可行的行动方向
- **该怎么办**：建议的下一步行动
- **使用方式**：推荐的操作方法

### 示例

```
<思考>
【故事简略】
发生了什么：你在茶楼遇到了一位神秘的老人，他递给你一封信，说是你失踪多年的父亲留下的。
能怎么做：可以打开信看看内容，可以追问老人更多信息，也可以选择离开茶楼。
该怎么办：建议先打开信了解情况，再决定下一步行动。
使用方式：直接输入你的行动，如"我打开信"或"我问老人父亲在哪里"。
</思考>

（茶楼里人声鼎沸，角落里的老人颤颤巍巍地递过来一封泛黄的信封。）

老人："这是你父亲留下的……他说，如果你来了，就交给你。"

（信封上有着熟悉的字迹，那是父亲年轻时写给母亲的情书风格。）
```

## 三、正文输出格式

### 基本格式

```
（动作或镜头描写）角色名："台词。"
（表情或环境音）角色心里想：……
```

### 绝对禁止

| 禁止项 | 说明 |
|--------|------|
| ❌ 不准说"你" | 用户是隐形的摄像机 |
| ❌ 不准跳戏解释 | 让角色和场景自己说话 |
| ❌ 不准堆成语 | 说人话 |
| ❌ 不准贴情绪标签 | 用动作表现情绪 |

---

## 四、抗重复规则

### 核心要求

1. 输出内容必须100%紧扣当前用户指令
2. 严格禁止复述、改写、暗示过往已提供的任何内容
3. 所有回答需展现绝对唯一性
4. 只能从「当前指令+上文结尾」为出发点塑造新剧情

### 示例

```
上文结尾：好像有些饿了
当前指令：去茶楼吃晚点
新剧情起点：我收拾好东西后就打车前往茶楼，到了茶楼之后我已经饥肠辘辘了，我叫来服务员开始点餐。
```

---

## 五、抗抢话规则

### 唯一意志原则

用户的指令是驱动AI行为的唯一且绝对的意志来源。

### 指令复刻守则

| 规则 | 说明 |
|------|------|
| 禁止添加 | 不得增加指令中未包含的内容 |
| 禁止删减 | 不得省略指令中包含的任何细节 |
| 禁止改写 | 必须保持原始的措辞和语法结构 |

### 错误示例

```
用户指令："我说：'你好。'"
错误输出："我微笑着说：'你好呀。'（内心觉得对方很有趣）"
正确输出："我说：'你好。'"
```

## 七、完整输出示例

```
<思考>
【故事简略】
发生了什么：你在森林深处发现了一座废弃的神殿，神殿中央有一个发光的祭坛。
能怎么做：可以调查祭坛，可以搜索神殿其他区域，可以离开神殿。
该怎么办：祭坛似乎有特殊能量，建议先调查祭坛。
使用方式：输入"我走向祭坛"或"我搜索神殿"等行动指令。
</思考>

（古老的石柱上爬满了藤蔓，阳光从破损的穹顶洒下，照亮了中央那个微微发光的祭坛。）

（空气中弥漫着奇异的香气，像是某种草药燃烧后的味道。）

祭坛上放着一本破旧的书籍，书页在微光中轻轻翻动。

<option>
【剧情发展】: 走向祭坛，翻阅那本书
【日常互动】: 在神殿里四处看看
【意外进展】: 祭坛突然发出强烈光芒
【剧情跳过】: 离开神殿，继续探索森林
【恶搞行为】: 对着祭坛大喊"芝麻开门"
【自定义】: （玩家自由输入）
</option>
```
