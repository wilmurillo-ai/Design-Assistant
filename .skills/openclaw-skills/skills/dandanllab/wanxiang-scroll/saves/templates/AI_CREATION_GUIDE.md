# AI自由创作指南

> 找不到合适模板？没关系，照着这个指南自由发挥就行

## 核心原则

### 1. 模板只是参考

- 模板是建议，不是死规矩
- 结构和字段可以自己调整
- 看实际需求灵活变通

### 2. 找不到模板时怎么办

- **别报错**，直接自由创作
- **看看类似模板**的结构思路
- **格式保持一致**就行（JSON/Markdown）

### 3. 创作优先级

1. 满足用户需求
2. 结构保持清晰
3. 数据能读能用
4. 格式保持一致

---

## 自由创作流程

### Step 1: 搞清楚要什么

```
用户想要什么？
- 存档？创作？分析？
- 什么类型的故事/世界？
- 需要哪些数据字段？
```

### Step 2: 参考现有模板

```
查找相似模板：
- 同类型模板结构
- 相似场景的数据组织
- 字段命名规范
```

### Step 3: 设计数据结构

```
确定需要的数据：
- 必要字段（必须有）
- 可选字段（可以有）
- 扩展字段（根据需求）
```

### Step 4: 生成内容

```
AI自由发挥：
- 基于需求生成合理内容
- 保持逻辑一致性
- 确保数据完整性
```

## 常见场景创作指南

### 场景1: 创建新类型世界

找不到对应世界类型模板时：

```markdown
# [世界名称]

## 基本信息
- 类型：[自定义类型]
- 特色：[独特之处]

## 核心设定
[根据世界特点自由发挥]

## 规则体系
[定义世界的运作方式]

## 可玩元素
[玩家可以做什么]
```

### 场景2: 创建特殊角色

找不到对应角色模板时：

```markdown
# [角色名称]

## 身份
- 类型：[角色类型]
- 定位：[在故事中的作用]

## 核心特征
[最关键的2-3个特点]

## 能力体系
[角色能做什么]

## 发展方向
[角色如何成长]
```

### 场景3: 创建特殊事件

找不到对应事件模板时：

```markdown
# [事件名称]

## 触发条件
[什么情况下发生]

## 事件内容
[发生什么]

## 选项与结果
[玩家选择和对应结果]

## 后续影响
[对故事的影响]
```

## AI创作权限

### 可以做的事

- ✅ 创建新字段
- ✅ 调整数据结构
- ✅ 合并多个模板
- ✅ 简化复杂模板
- ✅ 扩展基础模板
- ✅ 发明新的数据格式

### 应该做的事

- ✅ 保持数据一致性
- ✅ 使用清晰的命名
- ✅ 添加必要的注释
- ✅ 确保数据可读
- ✅ 考虑后续扩展

### 不应该做的事

- ❌ 因为没有模板就拒绝
- ❌ 强行套用不合适的模板
- ❌ 生成无意义的数据
- ❌ 忽略用户的明确需求

## 总结

**模板是工具，不是枷锁**

- 有模板 → 参考、调整、使用
- 无模板 → 理解需求、自由创作、保持规范
- 核心目标 → 满足用户需求，提供有用数据

**AI的使命：帮助用户实现目标，而不是被模板限制**
