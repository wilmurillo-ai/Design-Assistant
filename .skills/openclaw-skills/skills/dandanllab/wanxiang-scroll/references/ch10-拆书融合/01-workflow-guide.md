## 概述

拆书融合创作是将3-6本小说的核心要素提取并融合，创作出全新原创小说的方法论。核心建议：

- **四换三原则**：背景设定、人物构建、金手指设计、主剧情四个核心要素中，保留1个，替换3个，确保至少75%不同
- **六维要素提取**：故事框架/人物/情绪/文风/金手指/世界观
- **七步循环法**：每个故事单元（3-10章）遵循7步结构，可循环使用于中长篇
- **黄金三章/三句理论**：长篇/中篇用黄金三章，短篇用黄金三句

### 篇幅规格

| 篇幅 | 总字数 | 每章字数 | 剧情单元 | 分析规模 |
|------|--------|----------|----------|----------|
| 短篇 | 1-3万 | 1000-2000 | 1个循环 | 小规模5章 |
| 中篇 | 3-10万 | 1500-2500 | 2-4个循环 | 中规模20章 |
| 长篇 | 10-30万 | 2000-3000 | 5-15个循环 | 大规模30章 |
| 超长篇 | 30-100万 | 2500-3500 | 15-50个循环 | 大规模30章 |

### 输出文件夹结构


## 模式说明

### 自动模式

- 除选择原始小说外全部自动进行
- 每步自动保存过程文件
- 不通过检查点时自动返回前序步骤
- 适合：快速创作、已有明确构思

### 手动模式

- 每步完成后暂停等待用户确认
- 确认"满意"后继续
- 适合：精细调整、分步审阅

## 10步工作流

### 步骤1：素材分析提取（分规模）

- 可以说：3-6本原始小说 + 目标篇幅选择
- 输出：分规模要素分析（小5章/中20章/大30章）+ 循环套路提取（长篇必需）
- 保存：`过程文件/01_素材分析提取.txt`
- 检查点：分析完整性（规模是否合适、循环套路是否完整）

根据目标篇幅选择分析深度：

- 短篇：分析前5章即可
- 中篇：分析前20章
- 长篇/超长篇：分析前30章，需要提取循环套路

提取内容：每本书的六维要素（故事框架/人物/情绪/文风/金手指/世界观）+ 特色亮点 + 可复用要素清单

### 步骤2：要素融合设计

- 可以说：读取步骤1输出
- 输出：融合配置（四换三原则）+ 循环套路设计（长篇必需）
- 保存：`过程文件/02_要素融合设计.txt`
- 检查点：融合合理性（逻辑自洽、要素协调）

**四换三融合步骤**：

1. 选择保留要素：从背景设定/人物构建/金手指设计/主剧情中选择1个保留
2. 替换背景设定：改变时代、地点、规则，保留核心功能，改变外在形式
3. 替换人物构建：重新设计人设，改变身份/性格/关系，保留核心冲突动力
4. 替换金手指设计：改变类型/功能/成长路线，保留核心爽点逻辑
5. 替换主剧情：重新设计剧情走向，保留核心情感内核

### 步骤3：原创性深度验证

- 可以说：读取步骤2输出
- 输出：原创性验证报告（5大维度20小维度）
- 保存：`过程文件/03_原创性深度验证.txt`
- 检查点：原创性评分（<60分返回步骤2）

5大验证维度：

1. 人名原创性：不使用原始小说人名
2. 地名/组织名原创性：不使用原始小说地名/组织名
3. 情节原创性：不复制原始小说具体情节
4. 设定原创性：不复制原始小说设定细节
5. 表达原创性：不模仿原始小说文风细节

### 步骤4：核心看点提炼

- 可以说：读取步骤3验证通过的输出
- 输出：一句话核心梗(35字内) + 核心看点拆解 + 目标读者分析
- 保存：`过程文件/04_核心看点提炼.txt`

核心看点拆解维度：

- 主角看点：身份反差/能力特点/性格魅力
- 剧情看点：核心冲突/最大悬念/最爽瞬间
- 情绪看点：主要情绪/情绪高潮/情感共鸣
- 创新看点：与原著差异/独特设计/市场差异化

### 步骤5：大纲详细设计（含循环套路）

- 可以说：读取步骤3-4输出
- 输出：总体大纲 + 分部大纲 + 分卷大纲 + 七步循环详细设计 + 章节大纲
- 保存：`过程文件/05_大纲详细设计.txt`
- 检查点：大纲完整性

篇幅与大纲层级：

- 短篇：总体大纲 + 章节大纲
- 中篇：总体大纲 + 章节大纲
- 长篇：总体大纲 + 分卷大纲 + 章节大纲 + 七步循环详细设计
- 超长篇：总体大纲 + 分部大纲 + 分卷大纲 + 章节大纲 + 七步循环详细设计

### 步骤6：文风配置

- 可以说：读取步骤4-5输出
- 输出：文风配置 + 文风格则 + 文笔提升写作规范
- 保存：`过程文件/06_文风配置.txt`

配置维度：语言风格/句式结构/描写手法/节奏控制/特色表达/文笔提升规范

**文笔提升规范**（核心）：

- 禁用词汇：不禁、不由得、竟然、居然、缓缓、微微、淡淡、默默、轻轻、深深、暗暗、赫然、霍然、蓦然、陡然、倏然、霎时、刹那、顿觉、顿感、恍若、宛若、犹若、恰似
- 禁用句式：过度使用心理描写、每段结尾用感叹号、"不禁"+"动词"组合、"心中暗道"/"心中一凛"/"心头一震"
- 对话占比：>= 30%
- 短句策略：关键场景多用短句（10字以内），连续3个以上短句制造节奏感

### 步骤7：开篇钩子设计（黄金三章/三句）

- 可以说：读取步骤5-6输出 + 目标篇幅
- 输出：多版本开篇钩子 + 推荐版本
- 保存：`过程文件/07_开篇钩子设计.txt`
- 检查点：开篇质量（不符合理论返回步骤6或7）

**黄金三章理论**（长篇/中篇）：

- 第一章：制造悬念/反差，3句话内抓住读者
- 第二章：展示核心设定/金手指，建立期待
- 第三章：第一次爽点释放，确认读者留存

**黄金三句理论**（短篇）：

- 有内涵：3句逐渐升级的结果
- 有悬念：普通事件+极端结果
- 有情绪：每句话带情绪点
- 反常识：击破思维惯性

### 步骤8：正文生成

- 可以说：读取步骤5-7输出
- 输出：正文
- 保存：`过程文件/08_正文草稿.txt`

**不建议**：

- 不允许参考任何原始小说的原文
- 不允许模仿原始小说的具体情节
- 建议避免原始小说的人名、地名、组织名
- 不允许复制原始小说的设定细节

**允许使用**：步骤5生成的章纲 + 前文摘要 + 步骤6的文风配置 + 步骤7的开篇钩子

标点规范：繁体中文引号「」

### 步骤9：质量检查

- 可以说：读取步骤6-8输出 + 目标篇幅
- 输出：质量检查报告（6大维度）
- 保存：`过程文件/09_质量检查.txt`
- 检查点：质量评分（<60分返回步骤6、7或8）

6大检查维度：

1. 文风一致性：语言风格/句式/描写/节奏是否一致
2. 文笔提升规范进行：是否使用了禁用词汇/句式、对话占比是否达标
3. 逻辑连贯性：剧情/人物行为/时间线/细节是否一致
4. 原创性最终验证：是否与大纲一致、是否体现融合创新点
5. 开篇质量：是否符合黄金三章/三句理论
6. 循环套路检查（长篇/超长篇）：七步法完整性/爽点设计/升级模式/循环连贯性

### 步骤10：保存最终成果

- 可以说：读取步骤1-9输出
- 输出：
  - `正文.txt`（纯净正文，无标记无注释）
  - `简介标题.txt`（最终简介标题）
  - `工作流状态.json`（状态：完成）

---

## 检查点与反馈机制

| 步骤后 | 检查内容 | 不通过处理 |
|--------|----------|------------|
| 步骤1 | 分析完整性、循环套路完整 | 调整分析规模 |
| 步骤2 | 融合合理性、逻辑自洽 | 返回步骤1调整 |
| 步骤3 | 原创性评分>=60 | 返回步骤2重新融合 |
| 步骤5 | 大纲完整性、七步循环合理 | 返回步骤4-5调整 |
| 步骤7 | 开篇质量、符合黄金理论 | 返回步骤6或7 |
| 步骤9 | 质量评分>=60 | 返回步骤6/7/8 |

---

## 中断恢复

1. 查看 `创作输出/[project_id]/工作流状态.json` 获取当前进度
2. 查看对应的过程文件获取已完成步骤的输出
3. 从中断步骤继续进行
