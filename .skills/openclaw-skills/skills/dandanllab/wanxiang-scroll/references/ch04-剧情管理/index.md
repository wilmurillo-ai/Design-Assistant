## 剧情控制核心功能

### 1. 剧情推进控制

#### 标准推进

```
剧情推进 标准模式
```

- 正常推进剧情
- 维持节奏和人设一致性

#### 加速推进

```
剧情推进 加速模式
```

- 加快时间流逝
- 快速推进情节
- 增加转折和新事件

#### 减缓推进

```
剧情推进 减缓模式
```

- 细化描写
- 深挖人物
- 延展对话

### 2. 大总结系统

#### 新增大总结

```
剧情总结 新增
```

- 只总结新增内容
- 保留之前的大总结
- 适合阶段性总结

#### 全文大总结

```
剧情总结 全文
```

- 合并之前的大总结
- 生成完整的剧情总结
- 适合长篇故事回顾

#### 总结格式

```markdown
### 大总结：[时间点]

#### 1. 剧情概览
- 主要事件
- 角色发展
- 世界变化

#### 2. 关键伏笔
- 未解决的线索
- 隐藏的信息
- 未来可能的剧情点

#### 3. 角色状态
- [角色名]：[状态]
- [角色名]：[状态]

#### 4. 世界状态
- 当前时间
- 主要地点
- 特殊事件
```

#### 隐藏楼层策略

```
隐藏楼层 [内容]
```

- 记录伏笔或备注
- 不显示给你
- 用于创作者自我备注

> **数据存储**：隐藏楼层数据存储在 SQLite 数据库的 `hidden_floors` 表中，支持跨会话持久化。详见 SKILL.md "数据存储说明"。

### 3. 剧情节奏调节

#### 加速剧情

```
剧情节奏 加速
```

效果：
- 加快时间流逝
- 增加情节转折
- 引入新事件
- 快速推进到关键点

#### 减缓剧情

```
剧情节奏 减缓
```

效果：
- 细化场景描写
- 深挖人物心理
- 延展对话内容
- 慢节奏积累情绪

### 4. 多故事线管理

#### 故事线切换

```
故事线切换 [故事线名]
```

- 切换到不同的故事线
- 保持各故事线的独立性
- 支持多线并进

#### 对话模式

```
对话模式 开启
对话模式 关闭
```

- 适合多角色互动
- 专注于对话交流
- 减少环境描写

## MoM系统（高级功能）

MoM系统是MoM大总结与剧情QR功能的整合，提供更强大的剧情管理能力。

### 主要功能

#### 1. MoM大总结

- 智能剧情总结
- 自动识别关键信息
- 伏笔自动标记
- 角色关系图谱

#### 2. 剧情QR（Query & Response）

- 智能剧情查询
- 上下文管理
- 情节关联分析
- 时间线管理

### 使用建议

**长篇创作**：
1. 每10章做一次大总结
2. 用MoM系统管理复杂剧情
3. 定期检查伏笔和线索
4. 维护角色关系图谱

**多线故事**：
1. 每个故事线单独总结
2. 用故事线切换来管理
3. 注意时间线的一致性
4. 定期合并相关故事线

## 跟其他模块怎么联动

剧情管理是万象的"骨架"，串联所有长篇叙事：

- **→ 交互式故事（第三章）**：交互方式的多线剧情纳入MoM大总结系统
- **→ 人生模拟（第十二章）**：人生阶段作为剧情大弧，连锁事件作为支线标记，关系变化纳入角色图谱
- **→ 网文创作（第六章）**：细纲生成→剧情管理的大弧结构，节奏调节→三相循环节奏
- **→ 文风系统（第二章）**：剧情转折点可引发文风切换（如悲剧转折切换到压抑风）
- **→ 创作引擎（第五章）**：剧情推进过程中的输出经文笔润色处理
- **→ 质量控制（第七章）**：剧情逻辑自洽性检查，防止崩坏

## 注意

- 大总结系统适合长篇故事
- 隐藏楼层用于创作者自我备注
- 剧情节奏调节会影响整体氛围
- 多故事线管理需要清晰的标签
- MoM系统提供更高级的剧情管理能力
