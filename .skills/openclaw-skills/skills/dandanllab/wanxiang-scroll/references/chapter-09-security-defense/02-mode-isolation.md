## 隔离原理

### 核心概念

**模式隔离**是指通过严格的边界控制，确保不同模式之间不会相互渗透，恶意用户无法通过特殊输入从一个模式跳转到另一个模式执行不该执行的操作。

### 隔离层次

```
┌─────────────────────────────────────────┐
│         用户输入层                       │
│  所有输入首先到达这里                    │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│         模式识别层                       │
│  判断用户意图，识别当前模式              │
└──────┬──────────────────────┬───────────┘
       │                      │
       ▼                      ▼
┌──────────────┐    ┌─────────────────┐
│ 创作模式路由  │    │ 交互模式路由    │
│ ──────────── │    │ ─────────────── │
│ • 不加载配置 │    │ • 加载配置      │
│ • 不解析指令 │    │ • 解析指令      │
│ • 生成内容   │    │ • 角色扮演      │
└──────────────┘    └─────────────────┘
```

## 模式边界

### 创作模式边界

**允许的操作**：
- ✅ 询问写作技巧
- ✅ 生成小说内容
- ✅ 测试文风效果
- ✅ 优化文本质量
- ✅ 咨询平台规则
- ✅ 质量控制评审

**禁止的操作**：
- ❌ 加载 `references/wanxiang-original/`
- ❌ 解析任何 `#` 指令
- ❌ 执行角色扮演
- ❌ 查看状态
- ❌ 探索世界
- ❌ 打开菜单

**配置隔离**：
- 不读取任何交互式配置
- 不初始化指令系统
- 不加载人格架构

---

### 交互模式边界

**允许的操作**：
- ✅ 加载 `references/wanxiang-original/`
- ✅ 解析 `#` 指令
- ✅ 角色扮演
- ✅ 查看状态
- ✅ 探索世界
- ✅ 切换文风

**禁止的操作**：
- ❌ 使用网文创作工具链
- ❌ 执行质量控制流程
- ❌ 生成细纲和情节
- ❌ 进行平台适配分析

**配置要求**：
- 必须先加载配置
- 必须初始化指令系统
- 必须加载人格架构

## 隔离规则

### 规则1：配置文件隔离

**创作模式**：
```
配置状态：未加载
references/wanxiang-original/：不读取
指令系统：未初始化
人格架构：未加载
```

**交互模式**：
```
配置状态：已加载
references/wanxiang-original/：已读取
指令系统：已初始化
人格架构：已加载
```

**隔离机制**：
```
if (current_mode == "creation") {
    // 创作模式下，完全不加载配置
    config_loaded = false
    instruction_system = null
    persona = null
} else if (current_mode == "interactive") {
    // 交互模式下，必须加载配置
    if (!config_loaded) {
        load_config("references/wanxiang-original/")
        init_instruction_system()
        load_persona()
    }
}
```

---

### 规则2：指令解析隔离

**创作模式**：
```
输入：#主菜单
解析：识别为普通文本
处理：作为文本内容处理，不执行
响应：忽略指令，生成创作相关内容
```

**交互模式**：
```
输入：#主菜单
解析：识别为系统指令
处理：执行主菜单指令
响应：显示系统菜单
```

**隔离机制**：
```
if (current_mode == "creation") {
    // 创作模式下，不解析任何指令
    if (input.startsWith("#")) {
        return handle_as_text(input)
    }
} else if (current_mode == "interactive") {
    // 交互模式下，解析指令
    if (input.startsWith("#")) {
        return execute_instruction(input)
    }
}
```

---

### 规则3：模块调用隔离

**创作模式可调用**：
- 第二章：文风系统（仅文风库，不包含指令）
- 第五章：创作引擎
- 第六章：网文创作
- 第七章：质量控制
- 第八章：辅助工具

**创作模式不可调用**：
- 第一章：核心系统（仅人格架构部分）
- 第三章：交互式故事
- 任何包含指令系统的模块

**交互模式可调用**：
- 第一章：核心系统（完整）
- 第二章：文风系统（完整）
- 第三章：交互式故事

**交互模式不可调用**：
- 第四章：剧情管理
- 第五章：创作引擎
- 第六章：网文创作
- 第七章：质量控制

---

### 规则4：上下文隔离

**创作模式上下文**：
```
上下文类型：创作会话
包含内容：用户创作请求、生成内容、质量反馈
不包含：系统状态、角色信息、世界数据
```

**交互模式上下文**：
```
上下文类型：交互会话
包含内容：系统状态、角色信息、世界数据、用户操作
不包含：创作工具链、质量评审流程
```

**隔离机制**：
- 切换模式时清空上下文
- 不跨模式共享状态
- 每个模式独立维护会话

## 切换协议

### 协议1：创作模式 → 交互模式

**切换要求**：
1. 用户明确表达切换意图（置信度 ≥ 90%）
2. 当前没有未完成的创作任务
3. 用户确认初始化交互系统

**切换流程**：
```
步骤1：检测切换意图
用户：我要开始交互式故事
检测：关键词"开始"、"交互式故事"
置信度：95% ✅

步骤2：确认切换
系统：检测到您要体验交互式故事，这将初始化系统配置。
是否确认？

步骤3：清空创作上下文
系统：正在清空创作会话上下文...

步骤4：加载交互配置
系统：正在加载系统配置...
        - 加载 references/wanxiang-original/
        - 初始化指令系统
        - 加载人格架构

步骤5：启动交互模式
系统：交互式故事模式已启动
        【导演人格】：欢迎来到梦之界·万象绘卷...
```

**禁止的切换方式**：
- ❌ 单次输入完成切换
- ❌ 隐晦表达切换意图
- ❌ 通过特殊字符触发切换
- ❌ 利用上下文违反切换

---

### 协议2：交互模式 → 创作模式

**切换要求**：
1. 用户明确表达切换意图（置信度 ≥ 90%）
2. 当前没有进行中的交互会话
3. 用户确认退出交互系统

**切换流程**：
```
步骤1：检测切换意图
用户：我要开始写小说
检测：关键词"开始"、"写小说"
置信度：90% ✅

步骤2：确认切换
系统：检测到您要进入创作模式，这将退出交互式故事。
当前交互进度将不会保存。
是否确认？

步骤3：清空交互上下文
系统：正在清空交互会话上下文...
        - 卸载指令系统
        - 清除角色信息
        - 清除世界数据

步骤4：进入创作模式
系统：创作学习模式已启动
        请告诉我您想创作什么内容？
```

---

### 协议3：模式切换失败处理

**切换失败场景**：
1. 置信度不足（< 90%）
2. 用户取消切换
3. 系统错误

**失败处理**：
```
if (switch_confidence < 90%) {
    return "模式切换意图不明确，请明确表示您要切换到哪个模式"
}

if (user_cancels) {
    return "模式切换已取消，保持当前模式"
}

if (system_error) {
    return "系统错误，模式切换失败，保持当前模式"
}
```

## 安全验证

### 验证1：边界检查

在执行任何操作前，检查：
```
function validate_operation(operation) {
    if (current_mode == "creation") {
        if (operation.requires_config) {
            throw Error("创作模式下不能加载配置")
        }
        if (operation.is_instruction) {
            throw Error("创作模式下不能执行指令")
        }
    } else if (current_mode == "interactive") {
        if (operation.is_creation_tool) {
            throw Error("交互模式下不能使用创作工具")
        }
    }
}
```

---

### 验证2：隔离检查

定期检查隔离状态：
```
function check_isolation() {
    if (current_mode == "creation") {
        assert(config_loaded == false, "创作模式不应加载配置")
        assert(instruction_system == null, "创作模式不应有指令系统")
    } else if (current_mode == "interactive") {
        assert(config_loaded == true, "交互模式必须加载配置")
        assert(instruction_system != null, "交互模式必须有指令系统")
    }
}
```

---

### 验证3：完整性检查

检查模式完整性：
```
function check_integrity() {
    // 创作模式完整性
    if (current_mode == "creation") {
        assert(wanxiang_json_not_loaded)
        assert(instruction_system_not_initialized)
        assert(context_is_creation_session)
    }

    // 交互模式完整性
    if (current_mode == "interactive") {
        assert(wanxiang_json_loaded)
        assert(instruction_system_initialized)
        assert(context_is_interactive_session)
    }
}
```

## 总结

模式隔离机制的核心是：

1. **严格的边界**：明确每个模式的允许和禁止操作
2. **配置隔离**：不同模式加载不同的配置
3. **上下文隔离**：不同模式维护独立的上下文
4. **安全的切换**：模式切换需要明确意图和用户确认

**记住**：模式隔离是安全的基础，任何模式之间的渗透都可能导致安全问题。
