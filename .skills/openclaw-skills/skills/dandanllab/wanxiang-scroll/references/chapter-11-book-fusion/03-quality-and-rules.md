## 原创性保障机制

### 四换三原则
背景设定、人物构建、金手指设计、主剧情四个核心要素中，保留1个，替换3个，确保至少75%的核心要素不同。

### 生成时严格禁止
1. 禁止参考任何原始小说的原文
2. 禁止模仿原始小说的具体情节
3. 禁止使用原始小说的人名、地名、组织名
4. 禁止复制原始小说的设定细节

### 生成时允许使用
1. 步骤5生成的章纲
2. 前文摘要
3. 步骤6的文风配置
4. 步骤7的开篇钩子

## 字数要求与篇幅规格

| 篇幅类型 | 总字数 | 每章字数 | 剧情单元数 |
|---------|--------|---------|-----------|
| 短篇 | 1-3万 | 1000-2000 | 1个循环 |
| 中篇 | 3-10万 | 1500-2500 | 2-4个循环 |
| 长篇 | 10-30万 | 2000-3000 | 5-15个循环 |
| 超长篇 | 30-100万 | 2500-3500 | 15-50个循环 |

标点规范：繁体中文引号「」

## 质量检查6大维度

### 维度1：文风一致性检查
- 语言风格是否一致
- 句式结构是否符合要求
- 描写手法是否得当
- 节奏控制是否合理

### 维度2：降AI规范执行检查
- 是否使用了禁用词汇
- 是否使用了禁用句式
- 心理描写是否过多
- 对话占比是否达标（>=30%）
- 短句使用是否合理

禁用词汇清单：不禁、不由得、竟然、居然、缓缓、微微、淡淡、默默、轻轻、深深、暗暗、赫然、霍然、蓦然、陡然、倏然、霎时、刹那、顿觉、顿感、恍若、宛若、犹若、恰似

禁用句式：过度心理描写、每段结尾感叹号、"不禁+动词"、"心中暗道"/"心中一凛"/"心头一震"

### 维度3：逻辑连贯性检查
- 剧情是否连贯
- 人物行为是否合理
- 时间线是否清晰
- 细节是否一致

### 维度4：原创性最终验证
- 是否与大纲一致
- 是否体现了融合创新点
- 是否有独特的表达

### 维度5：开篇质量检查
- 长篇/中篇：是否符合黄金三章理论
- 短篇：是否符合黄金三句理论
- 开头3句/3章是否抓住眼球
- 是否建立了核心冲突
- 是否制造了悬念
- 是否留下了钩子

### 维度6：循环套路检查（长篇/超长篇）
- 循环七步法是否完整
- 爽点设计是否到位
- 升级模式是否合理
- 循环之间是否连贯

## 占位符与数据传递

### 占位符列表
| 占位符 | 说明 | 替换来源 |
|--------|------|----------|
| `{{SOURCE_NOVELS}}` | 原始小说列表 | 用户输入 |
| `{{STEP1_OUTPUT}}` | 步骤1素材分析 | 步骤1输出 |
| `{{STEP2_OUTPUT}}` | 步骤2融合设计 | 步骤2输出 |
| `{{STEP3_OUTPUT}}` | 步骤3原创验证 | 步骤3输出 |
| `{{STEP4_OUTPUT}}` | 步骤4核心看点 | 步骤4输出 |
| `{{STEP5_OUTPUT}}` | 步骤5大纲 | 步骤5输出 |
| `{{STEP6_OUTPUT}}` | 步骤6文风配置 | 步骤6输出 |
| `{{STEP7_OUTPUT}}` | 步骤7开篇钩子 | 步骤7输出 |
| `{{STEP8_OUTPUT}}` | 步骤8正文草稿 | 步骤8输出 |
| `{{NOVEL_CONFIG}}` | 小说配置信息 | 系统自动生成 |

### NOVEL_CONFIG 字段
| 字段 | 说明 | 来源 |
|------|------|------|
| 小说标题 | 从步骤4提取 | 步骤4或临时 |
| 篇幅类型 | 短篇/中篇/长篇/超长篇 | 步骤1提取 |
| 字数目标 | 根据篇幅类型 | 固定值 |
| 结构类型 | 七步循环法 | 固定值 |
| 剧情单元数 | 根据篇幅类型 | 步骤5提取 |
| 原始小说 | 选择的原始小说列表 | 用户输入 |
| 叙事视角 | 根据题材智能选择 | 步骤2提取 |

### 数据传递流程
```
原始小说 → 步骤1 → {{STEP1_OUTPUT}}
                ↓
           步骤2 → {{STEP2_OUTPUT}}
                ↓
           步骤3 → {{STEP3_OUTPUT}}（不通过返回步骤2）
                ↓
           步骤4 → {{STEP4_OUTPUT}}
                ↓
           步骤5 → {{STEP5_OUTPUT}}
                ↓
           步骤6 → {{STEP6_OUTPUT}}
                ↓
           步骤7 → {{STEP7_OUTPUT}}（不通过返回步骤6或7）
                ↓
           步骤8 → {{STEP8_OUTPUT}}
                ↓
           步骤9 → 质量检查（不通过返回步骤6/7/8）
                ↓
           步骤10 → 保存完成
```

## 步骤输出验证

每个步骤执行前应验证输入数据：

| 步骤 | 必需输入 | 格式要求 |
|------|---------|---------|
| 步骤1 | `{{SOURCE_NOVELS}}` | 至少3本小说，最多6本 |
| 步骤2 | `{{STEP1_OUTPUT}}` | 包含素材分析报告 |
| 步骤3 | `{{STEP2_OUTPUT}}` | 包含6维要素 |
| 步骤4 | `{{STEP3_OUTPUT}}` | 包含融合配置 |
| 步骤5 | `{{STEP3-4_OUTPUT}}` | 包含验证通过的融合+核心看点 |
| 步骤6 | `{{STEP4-5_OUTPUT}}` | 包含大纲 |
| 步骤7 | `{{STEP5-6_OUTPUT}}` | 包含文风配置 |
| 步骤8 | `{{STEP5-7_OUTPUT}}` | 包含开篇钩子 |
| 步骤9 | `{{STEP6-8_OUTPUT}}` | 包含正文草稿 |

### 验证失败处理
- 必需字段缺失：终止执行并提示用户
- 格式错误：尝试修复，无法修复则终止

### 标准输出格式
每个步骤输出须包含：
```
【步骤X完成】
【输出内容】
（步骤的具体输出内容）
【数据提取】
关键字段=值
```
