# Content Pipeline 项目分析

## 项目概述

**作者**: OrangeViolin (01fish)  
**定位**: Claude Code Skill，端到端内容生产与分发  
**核心流程**: 素材收集 → 写文章 → 排版 → 封面图 → 多平台内容 → 一键分发

## 核心优势技术

### 1. 多平台分发架构 (Multi-Platform Distribution)

**技术亮点**:
- 统一 manifest.json 配置，描述所有平台的内容
- 平台模块化设计，每个平台独立 publisher
- Chrome CDP (Chrome DevTools Protocol) 自动化
- 支持 6 大平台：公众号、小红书、即刻、小宇宙、抖音、视频号

**代码结构**:
```
scripts/distribute/
├── distribute.ts          # 主调度器
├── cdp-utils.ts           # CDP 工具集
├── platforms/
│   ├── wechat.ts          # 公众号
│   ├── xiaohongshu.ts     # 小红书
│   ├── jike.ts            # 即刻
│   ├── xiaoyuzhou.ts      # 小宇宙
│   ├── douyin.ts          # 抖音
│   └── shipinhao.ts       # 视频号
```

**可借鉴点**:
- 我们的 skill 也可以支持多平台（目前只有公众号）
- manifest.json 统一管理内容是个好思路
- CDP 自动化比纯 API 更灵活（但稳定性稍差）

### 2. 品牌色彩系统 (Brand Color System)

**01fish 品牌色**:
- 墨绿 `#1A3328` (85%) - 主背景
- 鱼红 `#C44536` (5%) - 强调色
- 宣纸 `#F2EDE3` (10%) - 文字/浅色背景

**应用层次**:
1. **排版主题**: Markdown → HTML 时注入品牌色
2. **封面图**: HTML 模板使用品牌色
3. **文章配图**: 暗底/浅底交替，保持品牌一致性

**可借鉴点**:
- 我们的主题系统可以加入「品牌色」概念
- 封面图也可以用 HTML + html2canvas 生成（我们目前用 SVG）
- 暗底/浅底交替的配图设计值得参考

---

### 3. 封面图生成技术 (Cover Image Generation)

**技术方案**:
- HTML + CSS 设计封面
- html2canvas 渲染为 Canvas
- FileSaver 导出 PNG
- 支持 2x 高清导出

**设计规范**:
- 主头图: 900×383px (2.35:1)，2x 导出 1800×766
- 次头图: 200×200px 正方形（从主图中心裁切）
- 暗底墨绿 + 宣纸白文字 + 鱼红强调
- 品牌角标 + 底部标签

**可借鉴点**:
- 我们的 SVG 方案可以保留，但可以增加 HTML+Canvas 选项
- 2x 高清导出值得实现
- 品牌角标和标签系统可以增加

### 4. 写作方法论 (Writing Framework)

**深度拆解饭统戴老板 5 篇 10w+ 文章**，提炼出：

#### 4.1 人设定位
- 会讲故事的极客朋友
- 硬核但说人话
- 有观点，敢下判断

#### 4.2 语言风格
- **句子节奏**: 长短交替（长句 40-60 字铺展，短句 15 字内收束）
- **排比句**: 三段以上，最后一段升华
- **破折号**: 关键判断前制造停顿
- **口语/书面配比**: 70% 书面 + 25% 口语 + 5% 黑话

#### 4.3 标题技术
六种模式：
1. 反转/颠覆: "人类还配不上贾维斯"
2. 对仗/并列: "旧战场和大革命"
3. 进程/弧线: "从仰望，到猛攻"
4. 文学化用: "悬疑剧已过万重山"
5. 好奇缺口: "改变商业世界的一张表"
6. 人物+判断: "[人名]的主战场"

#### 4.4 文章结构
- **序言**: 故事开头 + 背景 + 论点引出（15%，~1500 字）
- **01**: 铺设背景/建立框架（25%，~2500 字）
- **02**: 核心论述/深入分析（25%，~2500 字）
- **03**: 转折/案例/新视角（20%，~2000 字）
- **04**: 升华/哲学/收束（15%，~1500 字）

**可借鉴点**:
- 可以加入写作模板/框架系统
- 标题生成可以参考这 6 种模式
- 文章结构可以作为写作指导

### 5. 平台适配策略 (Platform Adaptation)

**小红书**:
- 图文轮播（Carousel）
- 标题 20 字以内
- 正文 1000 字以内
- 话题标签 3-5 个

**即刻**:
- 短动态形式
- 适合金句、观点

**小宇宙**:
- 播客脚本生成
- AI 语音合成

**抖音/视频号**:
- 视频画布生成
- 可录制的内容模板

**可借鉴点**:
- 一鱼多吃：一篇文章 → 多平台内容
- 每个平台有专门的格式模板
- 可以加入 AI 语音合成（TTS）

---

### 6. CDP 自动化技术

**核心能力**:
- 启动 Chrome with remote-debugging-port
- 通过 Chrome DevTools Protocol 控制页面
- 模拟点击、输入、上传文件
- 等待元素、执行 JS

**优势**:
- 不需要平台开放 API
- 可以操作任何网页
- 绕过一些反爬机制

**劣势**:
- 依赖 Chrome 浏览器
- UI 变化会失效
- 需要处理登录态

**可借鉴点**:
- 可以作为 API 方案的补充
- 适合小红书、抖音等没有开放 API 的平台

## 总结

Content Pipeline 的优势在于：
1. **完整的内容生产链路**（从素材到多平台分发）
2. **成熟的品牌色彩系统**
3. **基于 CDP 的多平台自动化**
4. **深度拆解的写作方法论**

我们的 skill 优势在于：
1. **更简单的使用方式**（一键推送）
2. **更丰富的主题系统**（5 种预设主题）
3. **更完善的文档**（SKILL.md 详细指南）

**融合方向**:
- 保留我们的主题系统和简单用法
- 吸收其品牌色彩、写作框架、多平台适配思路
- 逐步增加多平台分发能力
