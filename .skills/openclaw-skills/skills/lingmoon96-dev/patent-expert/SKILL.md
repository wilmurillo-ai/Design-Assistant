# Patent Expert

面向专利工作全流程的综合技能，适用于技术方案挖掘、检索分析、交底书整理、权利要求分析和布局规划。

## 核心能力

### 1. 专利挖掘

从产品、技术方案、业务流程中识别可专利的技术点。

工作流程：
1. 获取用户提供的技术方案、产品文档或流程描述。
2. 按核心算法、系统架构、交互流程、软硬件结合、数据结构、网络通信等维度拆解技术点。
3. 评估新颖性、创造性、实用性。
4. 输出按优先级排序的技术点清单。

输出格式：

```text
专利技术点清单
===============
#1 [核心] 技术点名称
   - 技术问题：
   - 解决方案：
   - 预期效果：
   - 新颖性评估：5/5

#2 [重要] 技术点名称
   ...
```

### 2. 专利布局

围绕核心技术做系统性申请规划。

布局策略：
- 城墙布局
- 星状布局
- 伞状布局
- 糖衣布局

工作流程：
1. 确定核心技术。
2. 横向展开应用场景、终端和行业。
3. 纵向展开基础方案、改进方案、优化方案。
4. 识别竞争对手可能的绕行路径。
5. 输出布局地图和时间规划。

输出格式：

```text
专利布局地图
===============
核心技术：[名称]
+-- 基础专利
|   +-- P1: [专利名称] - 优先级：5/5
|   +-- P2: [专利名称] - 优先级：4/5
+-- 应用场景扩展
|   +-- P3: [场景A方案] - 优先级：3/5
|   +-- P4: [场景B方案] - 优先级：3/5
+-- 竞品防御
|   +-- P5: [绕路方向防御] - 优先级：4/5
+-- 时间规划
    +-- 第1批（立即）：P1, P2
    +-- 第2批（3个月内）：P3, P5
    +-- 第3批（6个月内）：P4
```

### 3. 专利检索

利用公开资源进行专利检索和分析。

检索数据库：
1. Google Patents: `https://patents.google.com`
2. CNIPA: `https://pss-system.cponline.cnipa.gov.cn`
3. USPTO: `https://ppubs.uspto.gov`
4. Espacenet: `https://worldwide.espacenet.com`

工作流程：
1. 确定检索目标。
2. 提取关键词和分类号。
3. 构建检索式。
4. 执行检索并筛选。
5. 输出检索分析报告。

### 4. 交底书撰写

撰写标准发明专利交底书。

推荐结构：
- 发明名称
- 技术领域
- 背景技术
- 发明目的
- 技术方案
- 有益效果
- 附图说明
- 具体实施方式

规范要求：
- 对应权利要求层级。
- 详细到本领域技术人员可实现。
- 有益效果与技术问题一一对应。
- 优先使用规范专利术语。

模板：见 `assets/invention-disclosure-template.md`

### 5. 权利要求分析

分析现有专利权利要求范围，逐条对比技术特征，识别保护范围边界，并输出技术特征对比表。

### 6. 交底书配图建议

根据交底书中的附图说明，组织专利配图需求，统一图纸风格和命名规则。

绘图参考：`references/patent-drawing-guide.md`

建议流程：
1. 读取附图说明，提取图号、图名和核心关系。
2. 在系统架构图、流程图、模块结构图、DAG 图、时间示意图、多设备示意图中选择合适图型。
3. 依据指南生成绘图脚本或绘图说明。
4. 检查输出图是否与交底书一致。

说明：
- 本技能可提供配图结构和绘制要求。
- 若实际执行绘图，需要结合当前主机环境确认 Python、Graphviz、字体和输出路径配置。

## 工作流路由

| 用户意图 | 路由 |
|----------|------|
| 方案能申请专利吗 | 专利挖掘 -> 交底书 |
| 做专利规划 | 专利挖掘 -> 专利布局 |
| 查现有专利 | 专利检索 |
| 写交底书 | 交底书撰写 |
| 分析权利要求 | 权利要求分析 |
| 画交底书图/补图/生成配图 | 交底书配图建议 |
| 写权利要求/检查 claim 合规 | 结合其他权利要求撰写技能协同处理 |

## 协作建议

- 交底书初稿形成后，可与专利申请起草类技能串联。
- 权利要求审查或质量校验时，可与专利审稿类技能交叉使用。
- 需要管理层汇报时，可将结果压缩为技术点优先级、申请批次和业务价值摘要。
