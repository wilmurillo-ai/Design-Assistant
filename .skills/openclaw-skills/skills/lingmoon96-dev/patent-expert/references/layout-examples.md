# 专利布局策略案例

## 城墙布局

适用：保护单一核心技术

示例：某公司研发了一种新型AI推荐算法

```
核心：AI推荐算法（基础专利 P1）
+-- P2: 基于用户行为的实时推荐方法
+-- P3: 推荐结果的个性化展示方法
+-- P4: 推荐系统的冷启动优化方法
+-- P5: 推荐效果的评估与反馈方法
```

## 星状布局

适用：一个核心技术的多领域应用

示例：大语言模型技术

```
核心：大语言模型训练方法（P1）
+-- P2: 大模型在客服领域的应用
+-- P3: 大模型在代码生成领域的应用
+-- P4: 大模型在医疗诊断领域的应用
+-- P5: 大模型在自动驾驶领域的应用
+-- P6: 大模型在金融风控领域的应用
```

## 伞状布局

适用：垂直领域技术栈全覆盖

示例：智能驾驶技术

```
顶层：自动驾驶决策系统（P1）
+-- 感知层：P2 环境感知方法 / P3 传感器融合方法
+-- 决策层：P4 路径规划方法 / P5 行为预测方法
+-- 执行层：P6 车辆控制方法 / P7 紧急制动方法
+-- 数据层：P8 数据标注方法 / P9 模型训练方法
```

## 布局优先级评估标准

| 维度 | 高优先级（5分） | 中优先级（3分） | 低优先级（1分） |
|------|----------------|----------------|----------------|
| 新颖性 | 全新方案 | 组合创新 | 微小改进 |
| 创造性 | 非显而易见 | 有一定创造性 | 本领域常规 |
| 商业价值 | 核心产品依赖 | 辅助产品使用 | 潜在应用 |
| 竞争壁垒 | 竞品难绕过 | 竞品可绕过但有成本 | 竞品易绕过 |
| 申请紧迫性 | 竞品可能抢先 | 暂无紧迫性 | 长期技术 |

## 时间规划原则

1. **基础专利优先** — 越底层的核心技术越早申请
2. **公开风险控制** — 论文发表/产品发布前必须提交
3. **分批申请** — 核心第一批，改进型后续跟进
4. **PCT途径** — 有海外布局需求时优先走PCT
