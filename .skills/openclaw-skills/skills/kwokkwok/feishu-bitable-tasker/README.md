# 飞书多维表格任务管理

基于飞书知识库的多维表格和文档能力的任务管理工具。

![添加应用到多维表格](images/demo.png)

## 为什么需要这个 Skill

AI 当然可以直接调用飞书 API 来管理任务——但实际体验很差：

- **每次都要重新理解 API**：飞书的多维表格、知识库、文档块是三套独立的 API 体系，AI 需要反复查文档、试错、处理各种边界情况
- **认证和权限问题反复出现**：应用权限、资源级授权、token 类型选择……每次对话都可能踩一遍同样的坑
- **复杂操作容易出错**：比如「创建任务文档」其实需要 6 步——查记录、找/建月文档、找/建日文档、创建任务文档、插入子页面列表块、回写链接到记录。AI 临时拼凑很容易漏掉或搞错顺序
- **上下文无法复用**：上一次对话中解决过的配置问题、摸索出的 API 细节，下次对话全部丢失

这个 Skill 把这些经验固化成了可靠的脚本。AI 不需要理解飞书 API 的细节，只需要调用几个原子命令，就能稳定地完成任务管理。配置一次，后续使用零成本。

## 快速使用

### 安装

建议安装到项目级别（在项目根目录执行）：

```bash
npx skills add KwokKwok/feishu-bitable-tasker
```

安装后 skill 位于 `.claude/skills/feishu-bitable-tasker/`，仅对当前项目生效。

> 纯 Node.js 原生模块实现，无外部依赖，无需运行 `npm install`。

---

### 使用步骤

#### 步骤 1：创建飞书应用并配置权限

> 如果已有飞书应用，可以直接使用，但需确认权限是否满足。

1. 访问 [飞书开放平台](https://open.feishu.cn/)，点击「创建应用」→「创建企业自建应用」

2. 填写应用名称和描述，创建后在应用详情页记下 **App ID** 和 **App Secret**

3. 在应用管理页面，进入「权限管理」→「批量导入权限」，粘贴以下内容：

```json
{
  "scopes": {
    "tenant": ["bitable:app", "wiki:wiki", "docx:document"]
  }
}
```

4. 如修改了权限，需要点击「发布」→「创建版本」，填写版本号和日志，发布应用后权限才会生效

| 权限            | 用途                                    |
| --------------- | --------------------------------------- |
| `bitable:app`   | 多维表格读写（创建/查询/更新任务记录）  |
| `wiki:wiki`     | 知识库读写（创建文档节点、查询节点）    |
| `docx:document` | 文档块读写（为月/日文档添加子页面列表） |

---

#### 步骤 2：创建知识库多维表格并授权

1. 打开飞书知识库，创建一个新知识库（或使用现有的）

2. 在知识库中创建一个**多维表格**

3. 打开多维表格，点击右上角 **"..."** → **"更多"** → **"添加文档应用"**

4. 搜索并选择你创建的应用，**将权限设置为"可管理"**（默认为"可编辑"权限不足）

![添加应用到多维表格](images/add-app.png)

![设置应用权限](images/modify-app-permission.png)

5. 点击多维表格右上角的分享按钮，复制多维表格链接并保存

> ⚠️ **重要：** 必须在知识库中创建多维表格，不能使用独立的多维表格。链接格式应为 `https://xxx.feishu.cn/wiki/xxxxx`

---

#### 步骤 3：在 Claude Code 中配置

打开 Claude Code / OpenClaw 等工具，复制并修改下面的提示词发给 Agent：

```
请帮我配置一下飞书任务管理
App ID: <第一步复制的 App ID>
App Secret: <第一步复制的 App Secret>
多维表格链接: <第二步复制的多维表格链接>
任务表名称: <可选，默认与多维表格名称相同>
```

配置成功后，你就可以开始使用飞书任务管理功能了！

---

## 功能

- **创建任务** — 在多维表格中插入记录，支持自定义类型和状态
- **创建文档** — 在知识库中按「任务文档根节点 → 年月 → 日期 → 任务名」自动创建关联文档
- **修改状态** — 更新任务状态
- **查看统计** — 按状态分组展示任务数量

## 原理说明

#### 为什么需要手动创建资源？

你可能会问：为什么不让机器人自动创建多维表格？

这是一个**有意为之的安全设计**。如果让机器人自动创建资源，就需要赋予它更高级别的权限（如整个知识库空间的写入权限）。而采用「人工创建资源 + 授权」的方式，机器人**只能在你明确授权的资源范围内操作**，无法访问或修改其他任何文档。

这种最小权限原则确保了：即使凭证意外泄露，影响范围也仅限于你授权的那个多维表格。

### 多应用支持

同一个 skill 安装可以管理多个飞书应用。在 `config/` 目录下创建不同名称的凭证文件即可：

```
config/
├── credentials.json          # 默认凭证
├── team-a.json               # 团队 A 的应用
└── team-b.json               # 团队 B 的应用
```

使用时指定对应的凭证文件：

```bash
node scripts/task-manager.js config/team-a.json create "任务标题"
node scripts/task-manager.js config/team-b.json list
```

每个凭证文件独立配置各自的飞书应用和知识库多维表格，互不影响。

### 验证配置

配置完成后，Skill 会自动运行验证脚本确保所有功能正常：

```bash
echo "n" | node scripts/validate.js config/credentials.json
```

验证成功后会显示所有测试通过。

> 💡 **小贴士：** 验证成功后，你可以在飞书多维表格中将视图切换为「看板」模式，按「状态」字段分组，更加直观地管理任务进度。

![验证成功](images/test-done.png)

## 技术实现

纯 JavaScript，使用 Node.js 原生 `https` 模块，无外部依赖。

```
scripts/
├── feishu-client.js    # 飞书 API 客户端
├── task-manager.js     # 任务管理（create / create-doc / list）
├── admin.js            # 配置管理（config-from-links / list-fields）
└── validate.js         # 端到端验证
```
