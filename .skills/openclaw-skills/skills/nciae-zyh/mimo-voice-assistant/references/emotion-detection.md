# 🎭 Emotion-Aware TTS — 情绪感知语音生成

## 概述

情绪感知通过分析回复文本的语言特征，自动判断情绪类型，
并相应调整语音参数（语速、音量、停顿），让语音回复更自然。

**核心原则**：通过文本分析（规则 + LLM），而非额外模型。

## 情绪分类

| 情绪 | 标识 | 语速系数 | 停顿特征 | 使用场景 |
|------|------|----------|----------|----------|
| 😊 开心 | `happy` | 1.1 | 短停顿 | 祝贺、感谢、好消息 |
| 😐 平静 | `neutral` | 1.0 | 正常 | 一般回答、说明 |
| 😢 难过 | `sad` | 0.85 | 长停顿 | 慰藉、道歉、坏消息 |
| 😡 生气 | `angry` | 1.2 | 短促 | 警告、纠正、表达不满 |
| 🤩 激动 | `excited` | 1.15 | 频繁短停顿 | 发现、成功、惊奇 |
| 😰 紧张 | `anxious` | 1.05 | 不规律停顿 | 催促、提醒、警报 |
| 🤔 思考 | `thoughtful` | 0.95 | 长停顿 | 分析、建议、决策 |
| 💕 温柔 | `gentle` | 0.9 | 舒缓 | 安慰、鼓励、亲密 |

## 自动情绪判断规则

### 第一层：关键词匹配（即时判断）

```
开心触发词：太好了/太棒了/哈哈/厉害/恭喜/完美/🎉/❤️/👍
难过触发词：抱歉/对不起/可惜/遗憾/难过/😢/🙏
生气触发词：不对/错误/禁止/警告/绝不/你怎么能/😕
激动触发词：哇/天哪/竟然/震惊/不可思议/居然/😱/🤯
紧张触发词：紧急/注意/马上/立刻/快点/危险/⚠️/🚨
温柔触发词：别担心/没关系/慢慢来/加油/我在这里/💕
```

### 第二层：句式结构分析

```
疑问句 → thoughtful (语速稍慢)
感叹句 → excited (语速加快)
短句 + 命令 → angry/anxious
长句 + 说明 → neutral
引用道歉 → sad
```

### 第三层：上下文情绪（可选）

如果对话中有明确情绪指令，优先使用：
- `[[tts:happy]]` — 强制指定情绪
- `[[tts:sad]]` — 强制指定
- 等等

## 实现方式

### Agent 自行判断（推荐）

Agent 在回复前分析自己的回复内容：

```javascript
// 伪代码 — Agent 内部逻辑
function detectEmotion(text) {
  // 1. 关键词扫描
  if (检测到开心关键词) return 'happy';
  if (检测到难过关键词) return 'sad';
  if (检测到生气关键词) return 'angry';
  if (检测到激动关键词) return 'excited';
  if (检测到紧张关键词) return 'anxious';
  if (检测到温柔关键词) return 'gentle';
  
  // 2. 句式分析
  if (主要是问号) return 'thoughtful';
  if (感叹号 > 2) return 'excited';
  if (句子很短且有力) return 'angry';
  
  // 3. 默认
  return 'neutral';
}
```

### TTS 标记语法

在回复文本中插入情绪标记（proxy 会忽略这些标记的发音）：

```
[emotion:happy] 恭喜你！太厉害了！
[emotion:sad] 抱歉，这个我做不到...
[emotion:excited] 哇！这也太酷了吧！
```

或者使用 OpenClaw 的 TTS 标签：

```
[[tts:happy]]恭喜你！[[/tts:happy]]
```

## 语速调整

情绪判断后，通过文本前缀暗示语速（如果 proxy 支持）：

```
<speed:fast> 你怎么能这样！
<speed:slow> 我想这个问题需要认真考虑一下...
<speed:normal> 今天天气不错。
```

## 使用示例

### 场景：用户发来好消息

```
用户: "我考上研究生了！"
Agent 分析 → 检测到"了！"感叹 → 情绪: excited
TTS 输出: "哇！太棒了！恭喜你！这真是个好消息！🎉"
语速系数: 1.15 (稍快，欢快)
```

### 场景：用户发来坏消息

```
用户: "我的猫生病了..."
Agent 分析 → 检测到"生病" → 情绪: sad
TTS 输出: "别太担心，猫咪会好起来的...要不我们查查最近的宠物医院？"
语速系数: 0.85 (缓慢，温柔)
```

### 场景：用户问复杂问题

```
用户: "你帮我分析一下这个投资方案的风险"
Agent 分析 → 分析/方案/风险 → 情绪: thoughtful
TTS 输出: "让我想想...这个方案的主要风险有三个方面..."
语速系数: 0.95 (稍慢，沉思)
```

## 高级：对话情绪连续性

在长对话中，情绪应该是连贯的：

```
用户说开心 → Agent 回复也开心
用户说难过 → Agent 先共情再建议
用户愤怒 → Agent 先冷静再回应
```

不要在情绪上与用户反差太大（除非是安慰场景）。

## 配置选项

在 skill 使用时，可以通过以下方式控制情绪行为：

| 选项 | 说明 | 默认 |
|------|------|------|
| `emotionMode` | `auto` / `manual` / `off` | `auto` |
| `defaultEmotion` | 无情绪标记时的默认值 | `neutral` |
| `emotionIntensity` | `low` / `medium` / `high` | `medium` |

## 注意事项

- 情绪判断是启发式的，不是精确的
- 不同语言/文化的情绪表达可能不同
- 中文的感叹号/省略号是强信号
- 英文的 "lol", "haha", "omg" 也是信号
- 避免过度情绪化 — 自然最重要
