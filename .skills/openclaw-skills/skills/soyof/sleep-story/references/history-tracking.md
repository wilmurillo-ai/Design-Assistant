# 故事历史记录追踪

## 文件位置

`memory/sleep-story-history.json`

## 用途

记录已创作的助眠故事元素，避免多篇故事出现重复内容。

## 去重规则

### 冷却期

| 元素类型 | 冷却期 | 说明 |
|----------|--------|------|
| `scenes` (主场景) | 3 篇 | 同一场景至少间隔 3 篇故事 |
| `characters` (角色) | 2 篇 | 同角色类型至少间隔 2 篇故事 |
| `journeys` (旅程) | 2 篇 | 同探索方式至少间隔 2 篇故事 |
| `discoveries` (发现) | 2 篇 | 同温暖发现至少间隔 2 篇故事 |
| `sensoryFocus` (感官) | 1 篇 | 连续故事应切换主要感官 |
| `types` (故事类型) | 1 篇 | 尽量轮换四大类型 |

### 检查流程

创作新故事前：

1. **读取历史记录**
   ```bash
   cat memory/sleep-story-history.json
   ```

2. **检查最近使用的元素**
   - 查看 `recentElements` 中的各项
   - 确认冷却期是否已过

3. **选择新鲜元素**
   - 优先选择未使用过的场景、角色、旅程
   - 如必须使用，确保冷却期已过

4. **创作后更新**
   - 添加新故事到 `stories` 数组
   - 更新 `recentElements`
   - 更新 `lastCreated` 和 `totalStories`

## 用户偏好追踪

### 记录内容

- `favoriteTypes` - 用户偏好的故事类型
- `favoriteCharacters` - 用户喜欢的角色（如猫、鹿、鲸鱼）
- `avoidElements` - 用户不喜欢的元素（如某些场景、声音）
- `preferredLength` - 偏好的故事长度

### 更新时机

- 用户明确表达喜好时
- 用户对某篇故事反馈积极时
- 用户要求"不要 XXX"时

## 清理规则

- 保留最近 10 篇故事的完整记录
- 超过 10 篇的故事，只保留统计信息
- 用户偏好永久保留（除非用户要求清除）

---

**记住**：去重的目的是让每篇故事都有新鲜感，而不是限制创作。如果用户明确说"我还想听类似的故事"，可以灵活调整。
