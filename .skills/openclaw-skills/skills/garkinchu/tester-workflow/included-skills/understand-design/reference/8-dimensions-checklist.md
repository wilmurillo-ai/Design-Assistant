## 使用说明

### 如何使用这个检查清单

1. **按顺序检查**：从维度1到维度8依次检查
2. **不跳过任何维度**：即使某个维度看起来"没问题"
3. **记录所有问题**：不要因为"太多"而省略
4. **标注优先级**：P0/P1/P2
5. **提供建议**：不只是发现问题，还要给出解决方向

### 优先级判断标准

- **P0（阻塞上线）**：安全漏洞、数据丢失风险、阻塞核心功能
- **P1（重要）**：影响用户体验、性能瓶颈、可测试性问题
- **P2（一般）**：优化建议、文档完善、非核心功能

### 常见遗漏点

- 安全性：忘记问日志审计、会话管理
- 数据一致性：忘记问并发场景、事务边界
- 边界条件：忘记问状态转换、超时重试
- 接口设计：忘记问错误码、限流熔断
- 数据库设计：忘记问索引、软删除
- 缓存/中间件：忘记问降级方案、Key设计
- 性能：忘记问QPS指标、瓶颈分析
- 可测试性：忘记问测试环境、Mock点
