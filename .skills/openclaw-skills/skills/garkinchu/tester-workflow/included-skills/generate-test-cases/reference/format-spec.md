# CSV格式规范

## 文件格式要求

### 1. 编码格式

**必须使用 UTF-8 BOM 编码**

```
✅ 正确：UTF-8 BOM（文件开头有 \ufeff 标记）
❌ 错误：UTF-8（无BOM）
❌ 错误：GBK
❌ 错误：ANSI
```

**如何实现：**
```
在文件内容开头添加 BOM 标记：
﻿用例ID||测试模块||测试点||测试步骤||预期结果||优先级
```

**为什么需要BOM：**
- Excel打开中文CSV时需要BOM才能正确识别编码
- 测试管理工具导入时需要BOM
- 避免中文乱码问题

---

### 2. 分隔符

**必须使用双竖线 `||` 作为分隔符**

```
✅ 正确：用例ID||测试模块||测试点
❌ 错误：用例ID,测试模块,测试点
❌ 错误：用例ID|测试模块|测试点
❌ 错误：用例ID;测试模块;测试点
```

**为什么使用双竖线：**
- 测试步骤和预期结果中经常包含逗号
- 双竖线不会与内容冲突
- 便于解析和导入

---

### 3. 列定义

**标准列顺序（不可更改）：**

```
用例ID||测试模块||测试点||测试步骤||预期结果||优先级
```

**列说明：**

| 列名 | 必填 | 格式 | 说明 |
|------|------|------|------|
| 用例ID | 是 | TC-001, LC-001 | 单点测试用TC，流程测试用LC |
| 测试模块 | 是 | 文本 | 功能模块名称 |
| 测试点 | 是 | 文本 | 具体测试点描述 |
| 测试步骤 | 是 | 文本 | 详细操作步骤，用中文逗号分隔 |
| 预期结果 | 是 | 文本 | 明确的预期结果 |
| 优先级 | 是 | P0/P1/P2 | P0最高，P2最低 |

---

## 用例ID格式

### 单点测试用例（TC-XXX）

**格式：** `TC-` + 三位数字

```
✅ 正确示例：
TC-001
TC-002
TC-010
TC-099
TC-100

❌ 错误示例：
TC-1（缺少前导零）
TC001（缺少连字符）
tc-001（小写）
TC_001（下划线）
```

**编号规则：**
- 从 TC-001 开始
- 连续递增
- 保持三位数字（001-999）
- 先生成所有TC用例，再生成LC用例

---

### 流程测试用例（LC-XXX）

**格式：** `LC-` + 三位数字

```
✅ 正确示例：
LC-001
LC-002
LC-010

❌ 错误示例：
LC-1（缺少前导零）
LC001（缺少连字符）
lc-001（小写）
```

**编号规则：**
- 从 LC-001 开始
- 连续递增
- 按业务流程顺序编号
- 有依赖关系的用例编号连续

---

## 测试步骤格式

### 基本要求

**融入前置条件和测试数据：**

```
❌ 错误示例：
1. 打开页面
2. 输入数据
3. 点击保存

✅ 正确示例：
打开团队管理页面，选择"驻点组"类型，输入团队名称"康复一组"，选择所属科室"康复科"，点击保存按钮
```

**不使用序号：**

```
❌ 错误示例：
1、打开页面
2、输入数据
3、点击保存

✅ 正确示例：
打开页面，输入数据，点击保存按钮
```

**使用中文逗号连接：**

```
✅ 正确：打开页面，输入数据，点击保存按钮
❌ 错误：打开页面。输入数据。点击保存按钮。
❌ 错误：打开页面；输入数据；点击保存按钮
```

---

### 专业简洁的语言风格

**避免口语化：**

```
❌ 错误：
然后我们打开页面，接着输入一些数据，最后点一下保存按钮

✅ 正确：
打开页面，输入测试数据，点击保存按钮
```

**避免冗余描述：**

```
❌ 错误：
首先打开系统，然后登录系统，接着进入团队管理模块，最后打开团队列表页面

✅ 正确：
打开团队管理列表页面
```

**明确具体的数据：**

```
❌ 错误：
输入一些数据

✅ 正确：
输入团队名称"康复一组"，选择类型"驻点组"
```

---

### 流程测试的特殊格式

**标注依赖关系：**

```
✅ 正确示例：
基于LC-001创建的团队"康复一组"，添加成员"张三"（工号001），点击保存按钮

或者在测试点中标注：
测试点: 添加成员-正常添加（累积LC-001）
```

**说明数据来源：**

```
✅ 正确示例：
使用LC-001创建的团队ID，调用添加成员接口，传入用户ID=123
```

---

## 预期结果格式

### 基本要求

**明确具体：**

```
❌ 错误示例：
保存成功

✅ 正确示例：
系统提示"保存成功"，团队列表中显示新增的团队"康复一组"，类型为"驻点组"，所属科室为"康复科"
```

**包含验证点：**

```
❌ 错误示例：
操作成功

✅ 正确示例：
1. 系统提示"操作成功"
2. 数据库中team表新增一条记录
3. 团队状态为"启用"
4. 创建时间为当前时间
```

---

### 数据计算说明

**说明计算过程：**

```
❌ 错误示例：
金额为100元

✅ 正确示例：
金额为100元（50元基础费用 + 50元服务费）
```

**说明数据来源：**

```
❌ 错误示例：
治疗人次为5

✅ 正确示例：
治疗人次为5（张三2次 + 李四3次，同一人同一项目只计1人次）
```

---

### 错误提示格式

**完整的错误提示：**

```
❌ 错误示例：
提示错误

✅ 正确示例：
系统提示"团队名称不能为空"，数据不保存，焦点定位到团队名称输入框
```

**多个验证点：**

```
✅ 正确示例：
1. 系统提示"保存失败：团队名称已存在"
2. 数据不保存
3. 页面停留在当前页
4. 已输入的数据保留
```

## 优先级标注

### 优先级定义

| 优先级 | 定义 | 示例 |
|--------|------|------|
| P0 | 核心功能，阻塞上线 | 登录、核心业务流程、数据安全 |
| P1 | 重要功能，影响体验 | 查询、导出、提示信息 |
| P2 | 辅助功能，可后续优化 | 界面美化、非核心提示 |

### 优先级判断标准

**P0（必须测试）：**
- 核心业务流程
- 数据安全相关
- 权限控制
- 必填字段校验
- 关键业务规则

**P1（建议测试）：**
- 查询功能
- 导出功能
- 边界值测试
- 错误提示
- 用户体验

**P2（可选测试）：**
- 界面美化
- 非核心提示
- 优化建议
- 辅助功能

## 完整示例

### 单点测试用例示例

```csv
﻿用例ID||测试模块||测试点||测试步骤||预期结果||优先级
TC-001||团队管理||新增团队-正常新增||打开团队管理页面，点击"新增团队"按钮，输入团队名称"康复一组"，选择类型"驻点组"，选择所属科室"康复科"，点击保存按钮||系统提示"保存成功"，团队列表中显示新增的团队"康复一组"，类型为"驻点组"，所属科室为"康复科"，状态为"启用"||P0
TC-002||团队管理||新增团队-团队名称为空||打开团队管理页面，点击"新增团队"按钮，团队名称留空，选择类型"驻点组"，选择所属科室"康复科"，点击保存按钮||系统提示"团队名称不能为空"，数据不保存，焦点定位到团队名称输入框||P0
TC-003||团队管理||新增团队-团队名称重复||打开团队管理页面，点击"新增团队"按钮，输入已存在的团队名称"康复一组"，选择类型"驻点组"，选择所属科室"康复科"，点击保存按钮||系统提示"团队名称已存在"，数据不保存，已输入的数据保留||P0
```

### 流程测试用例示例

```csv
﻿用例ID||测试模块||测试点||测试步骤||预期结果||优先级
LC-001||团队管理||团队成员管理流程-创建团队||打开团队管理页面，创建团队"康复一组"，类型为"驻点组"，所属科室"康复科"||团队创建成功，团队ID为T001，状态为"启用"||P0
LC-002||团队管理||团队成员管理流程-添加成员（累积LC-001）||打开团队T001的成员管理页面，点击"添加成员"，选择用户"张三"（工号001），点击保存||成员添加成功，成员列表显示"张三"，所属团队为"康复一组"||P0
LC-003||团队管理||团队成员管理流程-同类型团队约束验证（累积LC-001、LC-002）||创建另一个驻点组团队"康复二组"，尝试添加成员"张三"||系统提示"该成员已加入同类型团队'康复一组'，不能重复添加"，数据不保存||P0
```

## 格式检查清单

生成CSV文件后，必须检查：

- [ ] 文件编码为UTF-8 BOM（文件开头有 `﻿`）
- [ ] 使用 `||` 双竖线分隔符
- [ ] 列顺序正确：用例ID||测试模块||测试点||测试步骤||预期结果||优先级
- [ ] 用例ID格式正确（TC-001, LC-001）
- [ ] 测试步骤融入前置条件和测试数据
- [ ] 测试步骤使用中文逗号连接，不使用序号
- [ ] 预期结果明确具体
- [ ] 优先级标注合理（P0/P1/P2）
- [ ] 流程测试标注依赖关系
- [ ] 无多余空行或空列
