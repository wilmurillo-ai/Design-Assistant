# 测试覆盖策略详解

## 5个必须覆盖的策略

### 策略1：正常业务流程

**定义：** 按照设计预期的正常路径执行业务操作

**覆盖要求：** 100%覆盖所有正常业务流程

#### 覆盖场景

**基本CRUD操作：**
- 新增（Create）
- 查询（Read）
- 修改（Update）
- 删除（Delete）

**业务流程：**
- 主流程（最常见的业务路径）
- 分支流程（条件分支）
- 端到端流程（完整业务场景）

#### 示例

```
模块：团队管理

正常流程用例：
TC-001: 新增团队-正常新增
TC-002: 查询团队-列表查询
TC-003: 查询团队-详情查询
TC-004: 修改团队-正常修改
TC-005: 删除团队-正常删除

LC-001: 团队管理完整流程（创建→添加成员→绑定项目→删除）
```

---

### 策略2：异常业务流程

**定义：** 非正常路径、错误场景、异常情况的处理

**覆盖要求：** ≥80%覆盖常见异常场景

#### 覆盖场景

**权限异常：**
- 无权限访问
- 权限不足操作
- 登录超时
- 账号被禁用

**数据异常：**
- 数据不存在（404）
- 数据已删除
- 数据已过期
- 数据状态不允许操作

**业务规则冲突：**
- 唯一性冲突
- 业务约束冲突
- 状态转换不允许
- 依赖数据缺失

**系统异常：**
- 网络超时
- 服务异常（500）
- 数据库异常
- 第三方服务异常

**并发异常：**
- 并发修改冲突
- 乐观锁冲突
- 资源竞争

#### 示例

```
模块：团队管理

异常流程用例：
TC-010: 新增团队-权限不足
TC-011: 新增团队-团队名称重复
TC-012: 修改团队-团队不存在
TC-013: 修改团队-团队已删除
TC-014: 删除团队-存在下级团队
TC-015: 删除团队-存在关联成员
TC-016: 添加成员-成员已加入同类型团队
TC-017: 添加成员-并发添加冲突
```

### 策略3：边界值测试

**定义：** 测试输入值的边界条件和极限值

**覆盖要求：** ≥90%覆盖关键字段的边界值

#### 覆盖场景

**数值边界：**
- 最小值（0、1、最小允许值）
- 最大值（最大允许值）
- 最大值+1（超出范围）
- 负数
- 小数
- 科学计数法

**字符串边界：**
- 空字符串
- 单字符
- 最小长度
- 最大长度
- 最大长度+1
- 特殊字符

**日期边界：**
- 过去日期
- 当前日期
- 未来日期
- 最小日期（1900-01-01）
- 最大日期（2099-12-31）
- 非法日期（2月30日）

**集合边界：**
- 空集合
- 单个元素
- 最大数量
- 最大数量+1

**状态边界：**
- 初始状态
- 中间状态
- 终止状态
- 非法状态转换

#### 示例

```
模块：团队管理

边界值用例：
TC-020: 新增团队-团队名称为空
TC-021: 新增团队-团队名称1个字符
TC-022: 新增团队-团队名称50个字符（最大长度）
TC-023: 新增团队-团队名称51个字符（超长）
TC-024: 新增团队-团队名称包含特殊字符
TC-025: 查询团队-每页0条（最小值）
TC-026: 查询团队-每页100条（最大值）
TC-027: 查询团队-每页101条（超出范围）
```

**覆盖要求：** 100%覆盖所有输入字段的校验规则

#### 覆盖场景

**必填校验：**
- 必填字段为空
- 必填字段为null
- 必填字段为空格

**格式校验：**
- 邮箱格式
- 手机号格式
- 身份证格式
- 日期格式
- 时间格式
- URL格式
- IP地址格式

**长度校验：**
- 最小长度
- 最大长度
- 固定长度

**范围校验：**
- 数值范围
- 日期范围
- 枚举值范围

**唯一性校验：**
- 重复值
- 大小写敏感
- 空格敏感

**安全校验：**
- SQL注入
- XSS攻击
- 脚本注入
- 路径遍历

**业务规则校验：**
- 自定义规则
- 关联字段校验
- 条件校验

## 覆盖度计算

### 计算公式

**正常流程覆盖率：**
```
覆盖率 = (已覆盖的正常流程数 / 总正常流程数) × 100%
```

**异常流程覆盖率：**
```
覆盖率 = (已覆盖的异常场景数 / 总异常场景数) × 100%
```

**边界值覆盖率：**
```
覆盖率 = (已覆盖的边界值数 / 总边界值数) × 100%
```

**输入校验覆盖率：**
```
覆盖率 = (已覆盖的校验规则数 / 总校验规则数) × 100%
```

**用户体验覆盖率：**
```
覆盖率 = (已覆盖的体验场景数 / 总体验场景数) × 100%
```

### 覆盖度目标

| 策略 | 目标覆盖率 | 最低要求 |
|------|-----------|---------|
| 正常流程 | 100% | 100% |
| 异常流程 | 80% | 60% |
| 边界值 | 90% | 70% |
| 输入校验 | 100% | 100% |
| 用户体验 | 50% | 30% |

## 常见问题

### Q1: 如何判断覆盖度是否达标？

**A:** 使用覆盖度计算公式，确保：
- 正常流程覆盖率 = 100%
- 异常流程覆盖率 ≥ 80%
- 边界值覆盖率 ≥ 90%
- 输入校验覆盖率 = 100%
- 用户体验覆盖率 ≥ 50%

### Q2: 如果某个策略的用例数量很少怎么办？

**A:** 
- 不是所有功能都需要大量用例
- 关键是覆盖率，不是绝对数量
- 简单功能可能只需要几个用例
- 复杂功能可能需要几十个用例

### Q3: 5个策略的优先级是什么？

**A:** 
1. 正常流程（最高优先级，必须100%覆盖）
2. 输入校验（高优先级，必须100%覆盖）
3. 异常流程（中高优先级，≥80%覆盖）
4. 边界值（中优先级，≥90%覆盖）
5. 用户体验（中低优先级，≥50%覆盖）

### Q4: 如何避免遗漏覆盖场景？

**A:** 
- 使用本文档的覆盖场景清单
- 逐项检查每个场景是否有用例
- 使用覆盖策略检查清单
- 计算覆盖率确保达标
