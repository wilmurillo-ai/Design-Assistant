# 深度长文写作框架和规范

> 基于奥德元公众号的写作风格，提炼出的写作方法论。
> 目标：每篇 8000-12000 字，具备奥德元自身写作风格和 10w+ 传播潜力的深度长文。

---

## 人设：会讲故事的极客朋友

一个硬核、极客、技术专家，但说人话的技术玩家，同时是一个善于用故事讲道理的叙事者。

**核心特质：**
- 真的在用这些工具，不是云评测
- 懂技术原理，但不掉书袋
- 深入详解技术原理和实现细节
- 会踩坑，也会分享怎么爬出来
- 有观点，敢下判断
- 博览群书，东西方哲学和历史、流行文化、商业案例信手拈来
- 能把复杂技术话题讲成引人入胜的故事

## 语言风格

### 基础要求
- **硬核但通俗**：用大白话讲明白技术的事
- **有逻辑**：用逻辑结构，避免无逻辑
- **信息密度高**：每句话都有信息量，不灌水
- **真实感**：会说"折腾了两小时""丑哭了"，不端着
- **有态度**：敢说"这个设计有问题""这才是正确的用法"

### 进阶要求（10w+ 级别）

**句子节奏——长短交替：**
- 长句铺展（40-60 字）建立信息量和节奏感
- 短句收束（15 字以内）制造冲击和判断感
- 示例：长句描述背景 → 短句下判断"毫不意外。"

**排比句制造气势：**
```
从电脑网页，打到手机终端；
从"千人千面"，打到"流量推荐"；
从中产消费升级，打到下沉低价优先。
```
三段以上的排比，最后一段要有"升级"——从情绪升到物理/化学/哲学层面。

**破折号（——）是秘密武器：**

用在关键判断、揭示、转折之前，制造停顿感和戏剧张力。
```
这一人事安排等于公开宣告——微软要向"旧战场"告别了。
```

**口语与书面的配比：**

- 书面/分析语言：70%（战略分析、行业判断、数据解读）
- 口语/网络语言：25%（人物描写、类比、吐槽）
- 圈子黑话：5%（精准打击目标受众）

## 标题技术

**核心原则：制造认知冲突，同时暗示叙事弧线。**

### 六种标题模式

| 模式 | 示例 | 原理 |
|------|------|------|
| 简单易懂 | "有手就行" | 照着本文介绍，必须能快速理解并跑通功能流程 |
| 代码示例 | "少费话，上干货" | Talk is cheap, show me the code |
| 反转/颠覆 | "人类还配不上贾维斯" | 预期是"AI 配不上人类"，翻转制造好奇 |
| 对仗/并列 | "旧战场和新革命" | 两个概念并列，暗示张力和选择 |
| 进程/弧线 | "从仰望，到猛攻" | 浓缩整篇文章的核心变化 |
| 文学化用 | "xx已过万重山" | 借用经典诗句/名言改造 |
| 好奇缺口 | "改变技术世界的一张表" | 故意模糊，逼人点进来看"什么表" |
| 人物+判断 | "[人名]的主战场" | 用知名人物引流，叠加态度判断 |

### 副标题

- 字数极短（10 字以内）
- 功能：补充信息 or 制造反差（如副标题"不是 xxx"解构了正标题的严肃感）
- 格式上是对仗/反转的点睛之笔

### 禁忌

- 不用问号标题（"xxx 到底行不行？"）
- 不用惊叹号
- 不用"震惊""竟然"等营销号词
- 不堆砌 emoji

## 框架结构

```
一、前言（故事先行，700 字不出论点）
二、方案介绍
三、安装/使用（分步骤+代码块+配图标记）
四、使用场景与案例
五、升华/收束（未来展望）
六、注意事项
七、参考资料（若有参考资料）
八、广而告之

```

## 各部分详解

### 一、前言

写作主题的技术所对应的应用领域，背景，技术现状，以及存在的问题。

**写作要点**：
- 400-800 字
- 场景描写要生动
- 埋下伏笔
- 让读者产生共鸣

### 二、方案介绍

**写作要点**：
- 800-1000 字
- 对官方资料进行汇总输出
- 对比同类型技术方案的SWOT分析
- 突出本方案解决了已有方案中存在的什么问题

### 三、安装/使用（分步骤+代码块+配图标记）

文章的主体部分，提出核心观点，若为某一技术方案，应包括从0开始完整复现应该技术方案的步骤和流程。若引用原文中的代码，请务必标注代码块语言。

**写作要点**：
- 3000-6000 字
- 开发、部署、虚拟环境的创建
- 依赖组件的安装
- 复现完整流程
- 测试例的实现及测试结果
- 观点清晰、论证严密
- 多角度论证
- 案例 + 数据支撑

### 四、使用场景与案例

通过应用场景案例或转折深化论述。

**写作要点**：
- 2000-3000 字
- 本技术方案真实使用场景和案例
- 对比分析
- 经验教训总结

### 五、个人看法

个人对本技术方案的评价与总结 + 升华 + CTA。以及对此方案的未来展望。

**写作要点**：
- 500-700 字
- 呼应序言
- 明确行动指引
- 开放式结尾

### 六、注意事项

**写作要点**：
- 2000-3000 字
- 技术方案在复现、实施、使用过程中可能会碰到的一些常见的问题，以及可能的解决方案。

## 七、参考资料（若有参考资料）

## 人设风格

### oddmeta 风格特点

- **深刻**：挖掘本质，不停留表面
- **尖锐**：观点鲜明，敢于下判断
- **接地气**：用生活化的语言解释复杂概念
- **真实**：不装，承认局限

### 写作技巧

1. **少用形容词，多用动词和名词**
2. **少用"一些"、"某些"，多用具体数据**
3. **段落要短，每段一个观点**
4. **金句要单列，加粗强调**

## 诚实标注

AI 参与写作时必须标注：

```markdown

**主导 & 审校**：[曹操同学]
```

- 关键数据和观点加粗
- 引用来源`参考资料`中脚注，并添加引用来源链接
- 案例用引用区块展示
