# 教程类文章框架

本文件定义了教程类文章的标准化写作框架，适用于 4000-8000 字的技术教程。

## 框架结构

```
→ 一、前言
→ 二、方案介绍
→ 三、安装/使用（分步骤+代码块+配图标记）
→ 四、使用场景与案例（分阶段+表格展示+人机协作）
→ 五、拿走即用（快速安装命令+使用方式表格）
→ 六、注意事项
→ 七、广而告之
```

## 各部分详解

### 一、前言

写作主题的技术所对应的应用领域，背景，技术现状，以及存在的问题。

**写作要点**：
- 用故事性的场景切入
- 埋下伏笔
- 让读者产生共鸣
- 控制在 200-300 字

### 二、方案介绍

用表格 + 一句话定义的方式介绍本方案的核心概念。

| 概念 | 定义 |
|------|------|
| 概念A | 一句话解释 |
| 概念B | 一句话解释 |

**写作要点**：
- 挑选 3-5 个核心概念
- 用通俗的语言解释
- 对官方资料进行汇总输出
- 对比同类型技术方案的SWOT分析
- 突出本方案解决了已有方案中存在的什么问题


展示最终效果，包含：
- 成品截图/效果展示
- 在线体验链接
- 核心数据（性能提升、功能数量等）
- 素材中若有图片，下载并引用素材配图

### 三、安装/使用

分步骤讲解，包含：
- 代码块（带语言标记）
- 配图标记（提示在什么位置插入图片）
- 命令行示例

**写作要点**：
- 步骤要清晰编号
- 代码块要完整可运行
- 关键步骤要有配图标记
- 控制在 600-800 字

### 四、使用场景与案例

通过应用场景案例或转折深化论述。

- 阶段划分
- 表格展示关键配置
- 人机协作说明（AI 负责什么，人类负责什么）

**写作要点**：
- 真实项目案例
- 展示完整流程
- 体现人机协作
- 控制在 800-1000 字

### 五、拿走即用

总结部分，包含：
- 一行命令安装
- 使用方式速查表
- 相关资源链接

升华 + CTA，包含：
- 价值总结
- 行动召唤
- 互动问题

**写作要点**：
- 简洁明了
- 方便复制
- 控制在 200-300 字

### 六、注意事项

**写作要点**：
- 500-3000 字
- 技术方案在复现、实施、使用过程中可能会碰到的一些常见的问题，以及可能的解决方案。
- 所有的参考资料都要引用，引用时请在代码块中引用，并在文本中添加引用来源及其链接。

### 七、广而告之

添加一段固定内容：

```markdown

<br>

<br>

<span>若需联系作者，请加微信：</span><span style="font-size: 1.2em; font-weight: bold; color: blue;"><b>oddmeta</b></span>
</div>
```

## 八、参考资料（若有参考资料）

若存在参考资料，请务必标注，并标识来源地址。

参考资料格式示例：

```markdown
- [1] **OddTTS项目**: https://github.com/oddmeta/oddtts
- [2] **小落同学**: https://www.oddmeta.net/project-xl
```

## 格式规范

- 标题层级：h1 → h2 → h3
- 代码块：标注语言
- 表格：使用 Markdown 表格
- 图片：用配图标记说明
- 引用：使用 > 区块引用

## 人机协作标注

在文章中适当位置标注：

```markdown
**本文由 AI 辅助调研和撰写**
**人类主导 + AI 协作**
```
