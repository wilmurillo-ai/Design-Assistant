# Debugging example

## Input style

User reports that a machine step does not advance, or an alarm cannot be reset, and suspects timer logic, scan order, or interlock logic.

## Expected behavior

The skill should:
- restate the symptom clearly
- separate known facts from hypotheses
- propose likely fault paths
- provide a practical debug checklist
- avoid claiming certainty without evidence

## Preferred output shape

1. symptom
2. known facts
3. assumptions
4. likely causes
5. debug plan
6. likely code corrections
7. safe verification points
