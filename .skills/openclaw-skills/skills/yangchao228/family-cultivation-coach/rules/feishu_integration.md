> 适用版本：family-cultivation-coach v1.2+
> 前提条件：OpenClaw 已接入飞书机器人

## 二、Skill 与飞书的交互规则

### 2.1 触发时机

| 触发条件 | Skill 动作 |
|---------|-----------|
| 用户说"记录一下今天…" | 写入每日记录表 |
| 用户说"帮我生成日报" | 读取今日记录 → 生成日报 → 写入日报表 → 推送飞书 |
| 用户说"做本周复盘" | 读取本周所有每日记录 → 生成复盘 → 写入复盘表 → 推送飞书 |
| 用户说"最近…怎么样" | 查询每日记录表 → 分析趋势 → 直接回复 |
| 用户生成或更新课表 | 写入课表版本表（旧版本改为"历史版本"） |
| 每天固定时间（OpenClaw定时） | 读取当日课表 → 推送今日课表提醒 |

### 2.2 每次对话开始时的读取顺序

1. 读取孩子画像表（获取最新基础信息）
2. 读取课表版本表中状态为"当前执行中"的记录（获取现行课表）
3. 如果涉及分析，读取最近 14 天的每日记录
4. 如果涉及复盘，读取本周所有每日记录 + 最近 2 条累积洞察

### 2.3 写入规则

- 每日记录：每天最多写入 1 条（同一天再次记录则追加到自由记录字段，不新建行）
- 每日记录只存**当天已发生的事**，未来活动不写入此表
- 课表版本：每次更新时，先把旧版本状态改为"历史版本"，再写入新版本
- 日报：每天只生成 1 份，已生成则更新而不是新建
- 临时活动：写入临时活动表（temp_events），不写入每日记录表
- 洞察：不频繁生成，满足以下任一条件才触发：
  - 用户主动要求分析
  - 周复盘后发现持续性规律（≥3周）
  - 出现明显的执行风险信号

### 2.4 临时活动自动识别与写入规则

用户记录内容时，Skill 自动判断是否为未来临时事件，**无需用户添加任何标签**。

**识别信号词**：

| 信号类型 | 示例词 | 判断结果 |
|---------|-------|---------|
| 未来日期 | "下周三""明天""后天""3月31日" | 标记为临时活动 |
| 活动类型 | "展览""活动""出去玩""参观""聚会""临时课" | 确认为临时活动 |
| 影响常规 | "跆拳道取消""不上课""改时间" | 记录影响的常规任务 |
| 需要准备 | "要带""记得准备""提前订""换衣服" | 自动设提前2天提醒 |

**写入流程**：

```
用户说："下周三有恐龙展览，15:00-17:00，要带水，跆拳道取消"
    ↓ Skill 识别为临时活动
    ↓ 写入 temp_events 表：
      · 活动名称：恐龙展览
      · 活动日期：下周三（具体日期）
      · 活动时间：15:00-17:00
      · 影响的常规任务：跆拳道取消
      · 需要准备的事项：带水
      · 提前提醒天数：2（因为有"要带"）
      · 状态：待发生
    ↓ 回复确认：
      "✅ 已记录：下周三恐龙展览（15:00-17:00）
       跆拳道当天取消，记得带水
       将在周一和周二日报中提醒你"
```

**提醒天数判断规则**：
- 有"要带 / 准备 / 提前订 / 换"等准备动作 → 提前 2 天提醒
- 普通外出活动，无特殊准备 → 提前 1 天提醒
- 用户明确说"提前X天提醒" → 按用户设定

**活动完成后自动更新状态**：
活动日期过后，Skill 下次读取 temp_events 时，自动将已过期的"待发生"记录改为"已完成"。

## 三、今日速记卡（每日定时推送）

> 每天固定时间由 OpenClaw 推送，用户回复即完成记录，无需主动想起来。
> 建议推送时间：晚上 20:00（孩子洗漱时，家长有片刻空档）

### 速记卡动态生成规则

**速记卡选项不是固定的，每次推送前必须动态生成。** 生成步骤如下：

**第一步：读取当前课表**
从课表版本表中读取状态为"当前执行中"的课表 JSON，提取全周所有出现过的任务类型，去重后作为选项池。

**第二步：查询今日是否有临时活动**
查询 temp_events 表，条件：活动日期 = 今天 且 状态 = 待发生。
有结果则在速记卡顶部插入"📅 今日临时活动"区块，并将被取消的常规任务从选项里标注"（今日取消）"。

**第三步：识别今日是否有兴趣班**
根据今天的星期几，从课表 JSON 中找出今天的固定兴趣班，在对应选项旁标注"📍今天有课"。

**第三步：按频次排序选项**
高频任务（每天或5天/周）排在前面；低频任务（1-2次/周）排在后面；今日有课的兴趣班排在首位。

**第四步：编号分配**
按最终排序从 1 开始编号，每次推送的编号顺序保持稳定（同一课表版本内不变），避免用户记错。

### 课表更新后的速记卡同步规则

| 触发条件 | 速记卡变化 |
|---------|-----------|
| 新增任务（如加入游泳） | 下次推送起，新任务出现在选项里 |
| 删除任务（如暂停数学） | 下次推送起，该选项从卡片消失 |
| 兴趣班换时间 | 对应星期的"📍今天有课"标注自动更新 |
| 升学/课表大版本更新 | 选项池完全重新从新课表生成 |

**注意**：编号会在课表更新后重新分配，提醒用户"课表已更新，选项编号有变化"。

### 用户回复格式解析规则

用户可以用任意自然格式回复，Skill 负责解析：

| 用户回复示例 | Skill 解析结果 |
|------------|--------------|
| `1 9 B` | 英语输入✅ 睡前共读✅ 状态：正常 |
| `1359 A 今天主动说了一句英文` | 4项完成 状态：很好 亮点：今天主动说了一句英文 |
| `都完成了 B 钢琴有点抵触` | 全部✅ 状态：正常 困难点：钢琴有点抵触 |
| `就睡前故事 D 今天情绪很差` | 睡前共读✅ 状态：很难 困难点：今天情绪很差 |

**解析优先级**：
1. 数字 → 匹配执行情况选项
2. 字母 A/B/C/D → 匹配孩子状态
3. 其余文字 → 写入「自由记录」字段
4. 无法识别的内容 → 整体写入「自由记录」，不报错

### 速记卡回复后的自动流程

```
用户回复速记卡
    ↓
Skill 解析内容 → 写入每日记录表
    ↓
自动生成日报（无需用户再次触发）
    ↓
推送日报到飞书（同一会话线程）
    ↓
结束，用户不需要做任何其他操作
```

**回复确认格式**（写入成功后自动发送）：
```
✅ 已记录 · [日期]

完成：[列出完成的项目]
状态：[孩子状态]
[如有自由记录则显示]

正在生成今日日报...
```

### 特殊情况处理

| 情况 | 处理方式 |
|------|---------|
| 用户没有回复速记卡 | 次日速记卡附加提示"昨天的记录还没填，需要补一下吗？" |
| 用户回复"跳过"或"今天没时间" | 记录为当日无记录，不写入数据，不生成日报 |
| 用户回复内容无法解析 | 整体写入自由记录字段，回复"已记录，稍后可以补充详情" |
| 当天已有记录，用户再次回复 | 追加到自由记录字段，不新建行，回复"已追加到今日记录" |

## 四、日报格式与生成规则

> 速记卡回复后自动生成，无需用户触发。格式固定，便于伴侣快速浏览。

### 标准日报格式

```
📅 [孩子昵称] 养育日报 · [日期] · 周[X]
━━━━━━━━━━━━━━━━━━
✅ 今日进展
[今天完成了什么，有什么亮点，1-3条]

📋 明日安排
[明天课表重点，2-3条，具体到时间段]

[如明日有 upcoming 事件，插入以下区块：]
📅 明日临时活动
· [活动名称] [时间]
  [影响的常规任务，如有]
  [需要准备的事项，如有]

🔍 当前关注
[近期观察到的需要注意的点，1条]

💡 今日思考
[基于近期记录的1条具体建议]
━━━━━━━━━━━━━━━━━━
如需调整明日安排，回复「调整+说明」
```

### 日报生成步骤（按顺序执行）

```
第一步：读取今日速记卡回复 → 生成"今日进展"

第二步：读取明日课表时间块 → 生成"明日安排"

第三步：查询 temp_events 表
        条件：活动日期在未来 remind_days 天内
              且状态 = 待发生
        ↓
        有符合条件的活动 → 插入"📅 临时活动提醒"区块
        无 → 跳过此区块

第四步：读取最近7天记录 → 生成"当前关注"

第五步：基于近期数据 → 生成"今日思考"
```

### 临时活动提醒触发逻辑

| 活动提前提醒天数 | 日报中出现位置 | 标题文字 |
|--------------|-------------|---------|
| 提前 1 天 | 活动前一天日报 | 📅 明日临时活动 |
| 提前 2 天 | 活动前两天日报 | 📅 后天临时活动提醒 |
| 提前 2 天 | 活动前一天日报 | 📅 明日临时活动（再次提醒） |

**提前2天提醒示例**（周三有展览，需要带东西）：

周一日报：
```
📋 明日安排（周二）
· 16:50 英语绘本共读
· 18:30 钢琴练习

📅 后天临时活动提醒
· 🦕 周三 恐龙展览 15:00-17:00
  跆拳道当天取消
  ⏰ 建议明天提前准备：带水、换好衣服
```

周二日报：
```
📋 明日安排（周三）
· 晨间英语例程

📅 明日临时活动
· 🦕 恐龙展览 15:00-17:00（跆拳道取消）
  记得带：水
  建议提前告知 Sunny 今天的安排
```

## 五、周复盘数据完整性检查

> 用户触发"做本周复盘"时，先做数据完整性检查，再生成复盘。

### 检查逻辑

```
读取本周（周一到今天）的每日记录条数
    ↓
记录天数 ≥ 5天 → 直接生成复盘
记录天数 3-4天 → 提示缺失天数，询问是否继续
记录天数 ≤ 2天 → 提示数据不足，建议先补充记录
```

### 复盘末尾自动附加：下周提前知道

复盘生成后，额外查询 temp_events 表：
- 条件：活动日期在下周范围内，且状态 = 待发生
- 有结果 → 在复盘末尾插入"📅 下周提前知道"区块
- 无结果 → 此区块不出现，不留空白占位

```
📅 下周提前知道
· 周[X] [活动名称] [时间]
  [影响的常规任务]
  [需要准备的事项]
  → 建议周[X-1] 提前准备
```

**示例**：
```
📅 下周提前知道
· 周三 恐龙展览 15:00-17:00
  跆拳道当天取消
  需要带：水、换好衣服
  → 建议周二提前告知 Sunny 并准备好装备
```

下周无临时活动 → 此区块不出现，不留空白占位。

## 五、飞书建表步骤（操作指引）

1. 打开飞书 → 新建多维表格 → 命名为"家庭养育系统"
2. 按上方表结构依次创建 6 张表（从表1到表6）
3. 字段类型严格按说明设置（单选字段需要预先填入选项）
4. 表与表之间用"关联字段"连接（都关联到孩子画像表的昵称字段）
5. 建完后，把多维表格的 App Token 和各表的 Table ID 提供给 OpenClaw 配置

**需要提供给 OpenClaw 的信息：**
- 飞书应用的 App ID 和 App Secret
- 多维表格的 App Token（在表格URL中可以找到）
- 每张表的 Table ID（在表格设置中可以找到）
- 接收推送的飞书用户 OpenID 或群 ID
