# Cloud Code 源码泄露 — 系列第一期

## 视频主题
Cloud Code 源码泄露：最强编码 Agent 的 Harness 工程教科书级实现

## 目标时长
60 秒竖屏（视频号格式）

## 脚本内容

### 开场 Hook（0-3秒）
「Cloud Code 源码泄露，这是目前公认最强的编码 Agent。而它的核心竞争力，根本不在模型。」

### 核心循环（4-15秒）
「Cloud Code 的核心骨架，就是一个 While 循环加一组工具。源码里叫 Query Engine：用户输入一句话，构建提示词，调用模型，模型说要执行工具就执行，把结果喂回去，继续循环，直到模型给出最终回答。没有复杂的状态图，没有条件路由，就是一个循环，但这个循环周围包裹着一整套精密的 Harness 系统。」

### 第一层：上下文管理（16-35秒）
「第一层是上下文管理，这是整个 Harness 里工程含量最高的部分。
Cloud Code 设计了三级压缩流水线来管理上下文窗口。
第一级叫微压缩，不调用模型，纯规则驱动。它按工具类型保留最近 N 个结果，怎么清有讲究——如果用户连续对话，API 前缀被缓存了，就通过 cache editing API 精细删除，不动前缀，缓存继续有效。如果用户离开一段时间缓存过期了，直接替换成占位符。这一步源码里就有三条执行路径。
第二级叫绘画记忆压缩，它不是做摘要，而是从对话中提取结构化事实——项目结构、用户偏好、任务进度，持久化到本地记忆目录。
第三级才是完整压缩，用独立模型调用把整段对话总结成一条精简上下文边界消息，之前的消息从模型视野中移除，但 UI 保留完整滚动历史。」

### 第二层：安全与约束（36-50秒）
「第二层是安全与约束。Cloud Code 有四个权限模式，从逐一确认到完全自动，每次工具调用都要过五步评估流水线。
最有意思的是分类器，而且它有两层。
第一层叫 Bash Classifier，纯规则匹配，只读命令自动放行，速度极快。
第二层叫 Yellow Classifier，是一个完整的 LLMS Judge 系统，用独立模型调用审查主 Agent 的每一个操作。它采用两阶段架构：第一阶段快速判断，yes 就放行，只有需要拦截时才进入第二阶段完整推理。
源码里有 23 条编号的 Bash 安全检查，防御 $9 等号展开、Unicode 零宽空格注入、IFS 空字节注入等。其中一条是 HackerOne 安全审查中发现的真实漏洞。
核心思想是：Prompt 里的规则是建议，代码里的规则是法律。」

### 第三层：多 Agent 与未发布功能（51-60秒）
「第三层是多 Agent 与未来架构。Cloud Code 的 Swarm 系统里，有 1 个主 Agent 叫 Leader，可以生成多个子 Agent 叫 Teammate。上下文隔离，但关键发现会通过 team memory sync 机制同步回 Leader。
还有一个 coordinator mode，开启后 Cloud Code 变成纯粹编排器，系统提示词里有一句话：不许偷懒委派。
此外还有几个藏在 feature flag 后面的未发布功能——Speculation 投机执行，Kos 永远在线的持久助手，Ultra Plan 把复杂规划卸载到云端容器。
回头看，Cloud Code 的每一个子系统都在做同一件事：在概率性的模型周围，构建确定性的工程系统。模型可能输出错误参数，Z Schema 拦截它；模型可能执行危险命令，权限分类器阻止它。
没有一行代码试图让模型变得更聪明。所有的工程投入，都在让模型周围的系统变得更可靠。
这就是 Cloud Code 源码证明的一件事：最强的编码 Agent，核心竞争力不在模型，在模型周围那套精密的控制系统。模型是引擎，Harness 是整辆车。」
