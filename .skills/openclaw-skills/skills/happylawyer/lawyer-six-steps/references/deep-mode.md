# 深度模式操作细则

## Phase 1 事实梳理
输出：
1. 时间轴对照表（己方/对方事实与证据）
2. 7W2H表
3. 主线图（文字版）
4. 事实分层：有利/不利/争议/待查

## Phase 2 法律分析
分三层：
1. 法律与行政法规
2. 最高法院司法解释（优先）
3. 地方审判政策/纪要/行业规则

输出《法律依据清单》：层次、规范名称、条号、适用点、链接。

## Phase 3 法理研究
按需使用：文义、体系、法意、目的、扩张、限缩、当然、合宪、社会学解释。
输出：
- 立法目的与本案适配性
- 法律漏洞与填补路径
- 法律冲突选择规则（上位法/新法/特别法）

## Phase 4 案例检索
优先级：
1. 指导案例
2. 公报/人民司法/人民法院案例选
3. 受案法院及上级法院同类案例

输出《案例对照表》：案号、法院、关键事实、要旨、异同、链接。

## Phase 5 裁判规则
提炼：
- 要件认定规则
- 免责及例外
- 非要件但高影响因素

输出《影响因素排序表》：影响因素、类型、认定标准、方向、本案状态。

## Phase 6 结果预判
三段论涵摄：
- 大前提：请求权基础规范
- 小前提：已证实事实
- 结论：每一诉请支持度

再做两层修正：
- 政策/价值衡量
- 个案执行可行性（履行能力、调解窗口）

最终输出：胜诉概率区间 + 每项诉请预判 + 风险与行动。

## 最终报告输出

Phase 6 完成并经用户确认后，必须进入最终报告输出阶段。

- 简易模式：使用 `references/output-templates.md` 中“简易模式案件分析报告模板”
- 深度模式：使用 `references/output-templates.md` 中“深度模式案件分析报告模板”
- 报告须覆盖前六步，不得机械拼接聊天记录
- 聊天界面只提示“已写入并打开”，不要完整展开报告正文
