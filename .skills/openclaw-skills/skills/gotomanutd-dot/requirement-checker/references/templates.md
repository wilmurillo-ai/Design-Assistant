## 📄 模板 1: 功能改造需求

```markdown
# 【功能名称】改造需求

## 一、需求背景

[简述需求背景和业务价值]

## 二、改造类型

- **类型**: 标准化改造 / 个性化改造（针对 XX 客户）
- **授权控制**: 
  - 新增角色：【角色名称】
  - 权限范围：【菜单/功能可见范围】
  - 数据范围：【本机构/全公司/指定范围】

## 三、涉及元素

### 3.1 接口
- 新增接口：【interfaceName】
- 修改接口：【existingInterface】

### 3.2 菜单
- 管理台位置：【一级菜单】>【二级菜单】>【功能名称】

### 3.3 定时任务
- 新增任务：【TaskName】（执行时间：每日 02:00）

### 3.4 清算节点
- 涉及节点：【ClearingNode001】

## 四、改造内容

### 4.1 查询功能改造

1. **新增多条件组合查询**
   - 查询条件：
     | 条件名 | 类型 | 必填 | 取值规则 |
     |--------|------|------|----------|
     | 资产类型 | 下拉 | 否 | STOCK/FUND/BOND |
     | 日期范围 | 日期 | 是 | yyyy-MM-dd |
   
   - 用户操作流程：
     1. 用户进入【资产查询】页面
     2. 选择查询条件
     3. 点击【查询】按钮
     4. 系统展示查询结果

   - 交互逻辑：
     - 操作前：校验必填字段
     - 操作中：调用【assetQueryApi】
     - 操作后：成功显示结果，失败显示错误提示

2. **支持导出 Excel 功能**
   - 触发方式：点击【导出】按钮
   - 导出字段：[字段 1]、[字段 2]、[字段 3]
   - 文件格式：.xlsx

### 4.2 报表功能改造

1. **新增资产汇总报表**
   - 报表名称：【资产汇总日报】
   - 统计维度：按日/周/月
   - 报表字段：
     | 字段名 | 类型 | 说明 |
     |--------|------|------|
     | 日期 | Date | 统计日期 |
     | 资产总额 | Decimal(18,2) | 单位：元 |
   
   - 计算公式：
     ```
     资产总额 = Σ(持仓数量 × 单位净值 × 汇率)
     ```
     参数说明：
     - 持仓数量：该资产的持有份额
     - 单位净值：资产当日单位净值
     - 汇率：外币资产兑换人民币汇率

## 五、界面原型

- 原型图：见附件 query_page.png
- 或在线原型：https://www.figma.com/file/xxx

## 六、数据字典

### 资产类型
| 代码 | 名称 | 说明 |
|------|------|------|
| STOCK | 股票 | 上市股票 |
| FUND | 基金 | 公募基金 |
| BOND | 债券 | 国债/企业债 |

## 七、异常处理

| 异常场景 | 处理方式 | 用户提示 |
|----------|----------|----------|
| 网络异常 | 自动重试 3 次 | "网络连接失败" |
| 数据校验失败 | 高亮错误字段 | "请填写完整信息" |
| 权限不足 | 隐藏功能按钮 | - |
| 系统超时 | 记录日志 | "操作超时，请重试" |

## 八、查询关联规则

- 查询范围：当前数据 + 历史数据（近 3 年）
- 关联表：asset_table JOIN asset_history_table ON asset_id
- 关联规则：查询条件包含日期范围时自动关联历史表
```

## 📄 模板 3: 报表需求

```markdown
# 【报表名称】需求

## 一、需求背景

[简述报表用途和使用场景]

## 二、报表要素

### 2.1 报表字段

| 字段名 | 类型 | 精度 | 说明 | 数据来源 |
|--------|------|------|------|----------|
| 日期 | Date | - | 统计日期 | trade_date |
| 资产代码 | String | - | 资产唯一标识 | asset_code |
| 资产名称 | String | - | 资产名称 | asset_name |
| 持仓数量 | Decimal | 2 位 | 单位：份 | position_qty |
| 单位净值 | Decimal | 4 位 | 单位：元 | unit_nav |
| 估值总额 | Decimal | 2 位 | 持仓×净值 | calculated |

### 2.2 查询条件

| 条件名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| 统计日期 | 日期 | 是 | T-1 | 支持单选/范围 |
| 资产类型 | 下拉 | 否 | 全部 | STOCK/FUND/BOND |
| 客户号 | 输入 | 否 | - | 支持模糊查询 |

### 2.3 统计维度

- 支持按日统计
- 支持按周统计（每周一为起始）
- 支持按月统计（每月 1 日为起始）

## 三、计算公式

```
估值总额 = 持仓数量 × 单位净值

合计行：
- 持仓数量合计 = Σ(持仓数量)
- 估值总额合计 = Σ(估值总额)
```

## 四、导出格式

- 支持 Excel 导出（.xlsx）
- 支持 CSV 导出
- 文件名规则：报表名称_统计日期.xlsx

## 五、权限控制

- 可见角色：【投资经理】、【风控专员】
- 数据范围：仅可查看本管理组合数据

## 六、性能要求

- 查询响应时间：< 3 秒
- 导出响应时间：< 10 秒
- 支持最大数据量：10 万条

## 七、界面原型

- 原型图：见附件 report_design.png
```

## 📄 模板 4: 系统对接需求

```markdown
# 【功能名称】系统对接需求

## 一、需求背景

[简述系统对接的背景和目的]

## 二、对接方说明

### 2.1 对接系统信息

| 项目 | 内容 |
|------|------|
| 对接系统 | XX理财系统 |
| 对接部门 | 产品运营部 |
| 系统负责人 | 张三（产品经理） |
| 技术对接人 | 李四（开发工程师） |
| 联系方式 | zhangsan@example.com |

### 2.2 对接环境

- **测试环境**: https://test-api.example.com
- **生产环境**: https://api.example.com
- **接口文档**: https://docs.example.com/api

## 三、对接数据内容

### 3.1 数据字段

| 字段名 | 类型 | 长度 | 必填 | 说明 | 示例值 |
|--------|------|------|------|------|--------|
| fund_code | String | 10 | 是 | 基金代码 | 000001 |
| fund_name | String | 100 | 是 | 基金名称 | 华夏成长混合 |
| nav | Decimal | 10,4 | 是 | 单位净值 | 1.2345 |
| nav_date | Date | - | 是 | 净值日期 | 2026-04-05 |
| status | String | 2 | 是 | 状态 | 01-正常 |

### 3.2 数据量估算

- **单次数据量**: 约1000条
- **日增量**: 约100条
- **历史数据量**: 约50万条（近3年）

### 3.3 数据质量要求

- 数据完整性：必填字段不能为空
- 数据准确性：基金代码符合规范
- 数据一致性：净值日期≤当前日期

## 四、对接数据应用

### 4.1 应用场景

1. **客户持仓展示**
   - 用途：在客户持仓页面展示最新净值
   - 频率：客户查询时实时展示
   
2. **收益计算**
   - 用途：每日计算客户收益
   - 频率：每日定时计算（T+1）
   
3. **历史收益统计**
   - 用途：展示客户历史收益曲线
   - 频率：客户查询时统计

### 4.2 数据流程

```
对接系统 → 数据同步 → 本地存储 → 业务应用
    ↓          ↓          ↓          ↓
  基金净值    定时任务    数据库     持仓展示
```

## 五、对接频率

| 数据类型 | 频率 | 时间 | 说明 |
|---------|------|------|------|
| 基金净值 | 每日 | 每日02:00 | T+1数据 |
| 基金信息 | 每周 | 周一03:00 | 全量更新 |
| 临时同步 | 按需 | 手动触发 | 特殊情况 |

**触发方式**：定时任务自动执行

## 六、对接方式

### 6.1 技术方案

| 项目 | 内容 |
|------|------|
| 对接方式 | REST API |
| 接口地址 | https://api.example.com/funds/nav |
| 请求方式 | POST |
| 认证方式 | OAuth 2.0 |
| 超时设置 | 30秒 |
| 重试策略 | 失败重试3次，间隔5秒 |

### 6.2 接口说明

**请求参数**:
```json
{
  "fund_codes": ["000001", "000002"],
  "nav_date": "2026-04-05"
}
```

**响应数据**:
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "fund_code": "000001",
      "fund_name": "华夏成长混合",
      "nav": 1.2345,
      "nav_date": "2026-04-05"
    }
  ]
}
```

### 6.3 备选方案

- **主方案**: REST API
- **备选方案**: SFTP文件传输（API不可用时）
- **文件格式**: CSV文件，字段顺序同上

## 七、数据时效性

### 7.1 数据时间

| 数据类型 | 时效性 | 说明 |
|---------|--------|------|
| 基金净值 | T+1 | 前一交易日数据 |
| 基金信息 | T+0 | 实时数据 |

### 7.2 时效性要求

- **同步时间**: 每日08:00前完成同步
- **数据延迟**: 客户查询时数据应≤1天
- **历史数据**: 保留近3年历史数据

### 7.3 数据版本

- 每日数据生成版本号（格式：YYYYMMDD）
- 支持历史数据查询和对比

## 八、异常处理

### 8.1 对接失败

| 异常场景 | 处理方式 | 用户提示 | 告警 |
|---------|---------|---------|------|
| API超时 | 自动重试3次 | 无 | 钉钉告警 |
| 认证失败 | 记录日志 | - | 邮件通知负责人 |
| 数据格式错误 | 跳过错误数据 | - | 记录异常日志 |
| 网络异常 | 使用上一日数据 | "数据更新中" | 钉钉告警 |

### 8.2 数据异常

| 异常场景 | 处理方式 | 说明 |
|---------|---------|------|
| 数据缺失 | 标记异常 | 人工确认后补充 |
| 数据重复 | 去重处理 | 保留最新数据 |
| 数据错误 | 记录日志 | 对接方确认后修正 |

## 九、数据一致性

### 9.1 对账机制

- **对账时间**: 每日09:00
- **对账内容**: 数据量核对、字段完整性核对
- **对账方式**: 自动对账 + 人工抽检

### 9.2 差异处理

- 自动告警：数据量差异>5%
- 人工核查：关键字段不一致
- 数据修复：确认后重新同步

### 9.3 监控告警

- 同步失败：钉钉即时告警
- 数据延迟：每日10:00未完成同步则告警
- 数据异常：关键字段异常则告警

## 十、测试计划

### 10.1 测试环境

- 使用测试环境接口
- 模拟数据验证
- 性能压力测试

### 10.2 测试内容

- 接口连通性测试
- 数据正确性测试
- 异常场景测试
- 性能压力测试

## 十一、上线计划

### 11.1 上线步骤

1. 测试环境验证
2. 生产环境联调
3. 灰度发布（10%流量）
4. 全量上线

### 11.2 回滚方案

- 保留旧数据源访问权限
- 准备应急脚本
- 制定回滚流程
```
