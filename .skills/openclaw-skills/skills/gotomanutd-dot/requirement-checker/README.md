## 📖 简介

使用 AI 智能检查需求文档是否符合规范，生成具体问题说明、原文引用、针对性建议和 GWT 验收标准。

**核心特点**：
- ✅ **AI 驱动** - 使用 LLM 智能检查，不是简单规则匹配
- ✅ **温柔话术** - 从"挑毛病"变成"建议式"，拿不准时温柔建议
- ✅ **智能引导** - 首次使用自动引导设置目录
- ✅ **多格式支持** - .docx, .md, .txt, .pdf, .jpg, .png, .html
- ✅ **GWT 自动生成** - 结合实际业务场景生成验收标准
- ✅ **建议分级** ⭐新增 - 核心建议/完善建议/优化建议三级分类
- ✨ **v2.5.0 新增** - 系统对接检查项 + 建议分级体系

---

## 📋 检查内容（13 项规范）

### 核心建议（影响理解的关键项）
1. 流程描述 - 业务流程图/流程说明
2. 异常处理 - 异常场景和错误提示

### 完善建议（提升文档质量）
3. 改造内容标注 - 新增/改造/优化内容标注
4. 元素完整性 - 输入/输出/处理/界面元素完整
5. 交互逻辑 - 用户操作流程和系统响应逻辑
6. 算法公式 - 计算逻辑说明
7. 查询关联 - 数据查询逻辑、表关联关系
8. GWT 验收标准 - Given-When-Then 格式验收标准
9. 系统对接描述 ⭐新增 - 如涉及系统对接，检查是否包含：
   - 对接方（哪个系统/部门）
   - 对接数据内容（具体字段、数据类型）
   - 对接数据应用（用途、业务场景）
   - 对接频率（实时/定时/批量）
   - 对接方式（API/文件/MQ/数据库）
   - 数据时效性（T+0/T+1/延迟要求）

### 优化建议（锦上添花）
10. 分项描述 - 功能点拆分清晰
11. 界面细节 - UI/UX 说明、原型图
12. 改造类型 - 新增/改造/优化类型标注
13. 原型附件 - 原型图/截图附件

## 🔧 依赖配置

### ✨ v2.4 优化：智能配置引导

**无需手动配置！** 首次运行时自动检测并保存：

1. **自动检测** - 扫描环境变量、OpenClaw 配置
2. **自动选择** - 按优先级选择最优 provider
3. **自动保存** - 保存到本地 config.json，下次无需重复

### 配置检测顺序

```
1️⃣ 本地缓存 (config.json)     → 最快，避免重复检测
2️⃣ 环境变量                  → 通用方案
3️⃣ OpenClaw 配置             → 自动扫描所有 provider
4️⃣ 引导用户配置              → 简化交互
```

### 支持的 Provider

自动检测以下 provider（按优先级）：
1. **bailian** (阿里云百炼，性价比高) ⭐ 推荐
2. **moonshot** (Kimi，长文本)
3. **openai** (通用)
4. **zhipu** (智谱)
5. 其他自定义 provider

### 手动配置（可选）

如果需要手动配置，可以选择：

#### 方式 1️⃣：OpenClaw 配置

```json
{
  "models": {
    "providers": {
      "bailian": {
        "baseUrl": "https://coding.dashscope.aliyuncs.com/v1",
        "apiKey": "your-api-key-here"
      }
    }
  }
}
```

#### 方式 2️⃣：环境变量

```bash
export OPENAI_API_KEY="your-api-key-here"
export OPENAI_BASE_URL="https://coding.dashscope.aliyuncs.com/v1"
export OPENAI_MODEL="glm-5"
```

#### 方式 3️⃣：运行时输入

首次运行时会引导输入，可选择保存到 config.json。

## 🚀 使用方式

### 在 OpenClaw 中使用

#### 1️⃣ 首次使用（自动引导）

```
请检查需求文档
```

智能体会引导你设置目录：
```
📁 需要设置需求文档检查目录

推荐配置：
- 输入目录：~/Downloads/requirementDocs
- 输出目录：~/Downloads/requirementReports

请告诉我：使用默认配置，或自定义目录
```

#### 2️⃣ 已配置后（直接使用）

```
请检查需求文档
```

智能体自动使用配置的目录执行检查。

#### 3️⃣ 需要汇总报告

```
请检查需求文档，并生成汇总报告
```

会额外生成 `00_汇总报告.md`。

#### 4️⃣ 自定义目录

```
请检查需求文档，输入目录是 ~/Documents/需求文档/
```

---

### 命令行直接运行

```bash
# 进入技能目录
cd ~/.openclaw/workspace/skills/requirement-checker

# 设置环境变量（如果没有 OpenClaw 配置）
export OPENAI_API_KEY="your-api-key"
export OPENAI_BASE_URL="https://coding.dashscope.aliyuncs.com/v1"

# 运行检查
python3 scripts/batch_check_ai.py
```

## 📊 评分机制（v2.5.0 更新）

### 建议分级扣分

| 优先级 | 扣分 | 说明 |
|--------|------|------|
| 核心建议 | 15分/项 | 影响理解的关键内容 |
| 完善建议 | 10分/项 | 提升文档质量 |
| 优化建议 | 5分/项 | 锦上添花 |

### 质量等级

| 等级 | 评分 | 说明 |
|------|------|------|
| 🌟 质量优秀 | ≥85分 | 文档完整清晰 |
| ✅ 通过 | 70-84分 | 整体不错，有优化空间 |
| 💡 建议自检 | 50-69分 | 有核心建议需要确认 |
| 🔧 建议完善 | <50分 | 需要补充核心内容 |

## 💡 温柔话术说明

本技能采用"建议式"话术，而不是"挑毛病"：

| 传统话术 | 本技能话术 |
|---------|-----------|
| "⚠️ 需要注意的问题" | "💡 可能需要关注的地方" |
| "未说明..." | "如果能...会更好" |
| "建议补充" | "可以补充" |
| "缺失" | "可补充" |
| "打回" | "待改进" |

**核心理念**：拿不准时温柔建议，让用户自己判断是否需要。

## ❓ 常见问题

### Q1: 没有 OpenClaw 配置怎么办？

**答**：可以设置环境变量：
```bash
export OPENAI_API_KEY="your-api-key"
export OPENAI_BASE_URL="https://coding.dashscope.aliyuncs.com/v1"
export OPENAI_MODEL="glm-5"
```

### Q2: 支持哪些文档格式？

**答**：支持 `.docx`, `.md`, `.txt`, `.pdf`, `.jpg`, `.png`, `.html`

### Q3: 检查结果不准确怎么办？

**答**：这是 AI 检查，可能存在误判。建议：
- 把报告当作"提醒"，不是"问题清单"
- 自己判断是否需要修改
- 温柔话术就是为了减少误判带来的压力

### Q4: 可以自定义检查规则吗？

**答**：可以修改 `references/checklist.md` 自定义检查规则。

### Q5: 支持其他平台吗？

**答**：v2.3 主要支持 OpenClaw 和环境变量。v2.4 将支持更多 Claw 平台。
