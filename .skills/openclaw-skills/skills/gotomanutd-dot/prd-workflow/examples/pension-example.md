## Step 1: 深度访谈记录

### 用户初始需求
```
我想给 APP 加个养老规划功能
```

### 访谈过程（20 个问题）

#### 产品定位（3 个）
1. **目标用户年龄段？** → 35-50 岁规划期人群
2. **收入水平定位？** → 中产阶层（年收入 30-100 万）
3. **养老测算精度？** → 详细测算（10-15 分钟）

#### 核心功能（3 个）
4. **是否需要产品推荐？** → 测算 + 产品推荐
5. **数据来源？** → 手动输入 + 对接社保
6. **输出形式？** → 详细报告 + 交互式方案

#### 合规要求（3 个）
7. **需要风险测评吗？** → 完整测评（20 题+，符合监管）
8. **适当性管理？** → 需要（产品风险等级匹配）
9. **冷静期/回访？** → 不需要

#### 技术约束（4 个）
10. **使用渠道？** → 手机 APP + 企业微信
11. **现有系统对接？** → 已有客户系统和产品库
12. **上线时间？** → 2-3 个月

#### 业务目标（2 个）
13. **核心业务目标？** → 提升理财经理效率，从 30-45 分钟降到 10-15 分钟
14. **成功指标？** → 测算时间缩短 67%，产品转化率提升 20%

#### 用户场景（2 个）
15. **主要使用场景？** → 理财经理在网点为客户做规划，或企业微信远程服务
16. **使用频率？** → 每天 5-10 次

## Step 2: 需求拆解结果

### 功能清单（MoSCoW 优先级）

#### P0 - Must Have（必须有）

| 功能模块 | 功能点 | 描述 |
|---------|--------|------|
| 客户信息 | 基本信息录入 | 年龄/收入/家庭结构/所在城市 |
| 客户信息 | 社保信息查询 | 对接社保系统获取缴纳记录 |
| 养老测算 | 缺口计算引擎 | 基于替代率法计算养老金缺口 |
| 养老测算 | 三条曲线展示 | 资产曲线/支出曲线/缺口曲线 |
| 风险测评 | 20 题测评问卷 | 符合监管要求的风险测评 |
| 风险测评 | 风险等级判定 | R1-R5 风险等级 |
| 产品推荐 | 产品库对接 | 对接现有产品库 |
| 产品推荐 | 适当性匹配 | 根据风险等级推荐产品 |
| 报告生成 | 详细报告 | 5-10 页 PDF 报告 |

#### P1 - Should Have（应该有）

| 功能模块 | 功能点 | 描述 |
|---------|--------|------|
| 养老测算 | 参数调整 | 可调整退休年龄/预期寿命等 |
| 养老测算 | 方案对比 | 对比不同方案的效果 |
| 产品推荐 | 配置方案 | 资产配置建议（比例） |
| 报告生成 | 交互式报告 | H5 交互式报告 |

#### P2 - Could Have（可以有）

| 功能模块 | 功能点 | 描述 |
|---------|--------|------|
| 养老测算 | 社保测算 | 详细社保养老金测算 |
| 产品推荐 | 动态调整 | 根据市场变化调整配置 |
| 报告生成 | 报告分享 | 分享给家人/理财经理 |

### 用户故事（部分）

**US-001：客户信息录入**
```
作为 理财经理
我想要 快速录入客户基本信息
以便 开始养老测算
```

**验收标准**：
```
Given 理财经理打开客户信息页面
When 填写年龄、收入、城市等信息
Then 系统验证必填项完整
And 保存客户信息
```

**US-002：养老缺口测算**
```
作为 理财经理
我想要 查看客户的养老金缺口
以便 推荐合适的产品配置
```

**验收标准**：
```
Given 客户信息完整（年龄/收入/社保等）
When 点击"开始测算"
Then 显示三条曲线（资产/支出/缺口）
And 显示养老金缺口总额
And 显示缺口产生年份
```
