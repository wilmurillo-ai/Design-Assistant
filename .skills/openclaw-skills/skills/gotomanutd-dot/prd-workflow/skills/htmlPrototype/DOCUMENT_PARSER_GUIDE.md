# 从需求文档生成原型

## 🎯 功能说明

支持读取**需求文档**（PRD），自动提取页面类型、功能需求、组件需求，生成对应的 HTML 原型。

---

## 📋 支持的文档格式

| 格式 | 扩展名 | 支持度 |
|------|--------|--------|
| **Markdown** | `.md` | ✅ 完全支持 |
| **纯文本** | `.txt` | ✅ 完全支持 |
| **Word** | `.docx` | ✅ 支持（需安装 python-docx） |
| **PDF** | `.pdf` | ⚠️ 需要转换为文本 |

---

## 📊 提取的信息

### 1. 页面类型检测

自动识别文档描述的页面类型：

| 关键词 | 识别为 |
|--------|--------|
| 列表、清单、表格 | 列表页 |
| 表单、填写、输入 | 表单页 |
| 仪表盘、看板、统计 | 仪表盘 |
| 详情、明细、查看 | 详情页 |
| 登录、登陆 | 登录页 |

### 2. 业务领域识别

识别业务关键词：

| 关键词 | 识别为 |
|--------|--------|
| 产品、商品 | 产品管理 |
| 用户、客户、会员 | 用户管理 |
| 订单、订购 | 订单管理 |
| 销售、销售额 | 销售管理 |
| 电商、商城 | 电商系统 |
| 财务、资金、账单 | 财务管理 |
| 库存、仓储 | 库存管理 |

### 3. 组件需求提取

识别需要的 UI 组件：

| 关键词 | 提取组件 |
|--------|----------|
| 表格、列表、数据展示 | 表格组件 |
| 筛选、过滤、搜索 | 筛选组件 |
| 分页、翻页 | 分页组件 |
| 按钮、操作 | 按钮组件 |
| 图表、统计图 | 图表组件 |
| 卡片、数据卡片 | 卡片组件 |

### 4. 风格偏好识别

识别界面风格：

| 关键词 | 风格 | 主色调 |
|--------|------|--------|
| 商务、专业、企业、正式 | business | 深蓝色 |
| 活泼、清新、年轻、活力 | fresh | 绿色系 |
| 其他 | modern | 紫色系（默认） |

## 💡 使用示例

### 示例 1：产品管理 PRD

**文档内容**（requirements.md）：
```markdown
# 产品管理系统需求

## 功能需求

需要一个产品列表页面，包含：
- 数据表格：显示产品名称、价格、库存、状态
- 筛选功能：支持关键词搜索、状态筛选
- 操作按钮：新建、导出、批量删除
- 分页功能

## 界面要求

商务专业风格，蓝色主题
```

**运行命令**：
```bash
python3 main.py --doc requirements.md
```

**输出**：
```

📋 从文档提取:
   页面类型：列表页
   关键词：产品
   组件：表格、筛选、按钮、分页
   风格：business

✅ 已生成商务专业风格的产品列表页
📁 ~/Desktop/prototype_list.html
```

## 📁 文档结构建议

为了获得最佳效果，建议按以下结构编写需求文档：

```markdown
# [项目名称]

## 1. 项目概述
[简要说明项目背景和目标]

## 2. 用户角色
[列出用户角色和权限]

## 3. 功能需求
### 3.1 [页面/功能名称]
- 数据表格：[字段列表]
- 筛选功能：[筛选条件]
- 操作按钮：[按钮列表]
- 其他组件：[组件说明]

## 4. 界面要求
- 风格：[商务/活泼/现代]
- 色调：[颜色偏好]
- 布局：[布局要求]

## 5. 非功能需求
- 性能要求
- 安全性要求
```

## 🔧 高级用法

### 组合使用

```bash
# 文档 + 交互模式
python3 main.py --doc requirements.md -i

# 文档 + 指定页面类型
python3 main.py --doc requirements.md --page dashboard

# 文档 + 自定义输出
python3 main.py --doc requirements.md --output ~/Projects/prototype

# 文档 + 不截图
python3 main.py --doc requirements.md --no-screenshot
```

### 解析器 API

```python
from parser.requirement_parser import parse_requirement_doc

# 解析文档
result = parse_requirement_doc('requirements.md')

print(f"页面类型：{result['page_type']}")
print(f"关键词：{result['keywords']}")
print(f"组件：{result['components']}")
print(f"风格：{result['style_preference']}")
```

## 🐛 故障排查

### 问题 1：文档解析失败

```bash
# 检查文件是否存在
ls -l requirements.md

# 检查文件格式
file requirements.md

# 尝试转换为纯文本
cat requirements.md | python3 main.py -
```

### 问题 2：Word 文档无法读取

```bash
# 安装 python-docx
pip install python-docx

# 或导出为 Markdown/文本
```

### 问题 3：解析结果不准确

**原因**：文档描述不够清晰

**解决**：
1. 添加更多关键词（列表、表格、筛选等）
2. 明确说明页面类型
3. 使用交互模式补充
