# 一致性检查引擎

**版本**: v1.0  
**功能**: 检查文档前后文是否矛盾

---

## 🔍 检查项清单

### 1. 风险等级一致性 ⭐

**检查规则**:
- 提取"产品概述"中的风险等级
- 提取"风险揭示"中的风险等级
- 对比是否一致

**常见错误**:
```
❌ 产品概述：风险等级 R3
   风险揭示：风险等级 R2

❌ 产品概述：中风险
   风险揭示：R4（中高风险）
```

**修复建议**:
```
✅ 统一为 R3（以产品概述为准）
✅ 或重新评估产品风险等级
```

---

### 2. 费率结构一致性 ⭐

**检查规则**:
- 提取"费率结构"章节的所有费率
- 提取"产品概述"或其他章节提到的费率
- 对比是否一致

**常见错误**:
```
❌ 费率结构：申购费率 1.5%
   产品概述：申购费率 1.2%

❌ 费率结构：管理费 1.0%/年
   收益测算：管理费按 1.2% 计算
```

**修复建议**:
```
✅ 以费率结构章节为准
✅ 统一修改为 1.5%
```

---

### 3. 术语一致性 ⭐

**检查规则**:
- 提取文档中所有金融术语
- 检查同一概念是否使用同一术语
- 对比标准术语库

**常见错误**:
```
❌ 第 1 章：基金份额
   第 3 章：基金单位
   第 5 章：份额

❌ 第 2 章：认购费
   第 4 章：申购费（开放式应统一用申购）
```

**修复建议**:
```
✅ 统一使用"基金份额"
✅ 开放式基金统一用"申购费"
```

---

### 4. 数字一致性 ⭐

**检查规则**:
- 提取所有关键数字（投资比例、限额等）
- 检查同一指标在不同位置是否一致
- 检查计算是否正确

**常见错误**:
```
❌ 投资范围：股票投资比例 80%-95%
   投资策略：股票仓位 80%-90%

❌ 最低持有：1000 份
   申购起点：100 元（未说明对应份额）
```

**修复建议**:
```
✅ 统一为 80%-95%
✅ 补充说明：1000 份≈1000 元（按净值 1 元计）
```

---

### 5. 逻辑一致性 ⭐

**检查规则**:
- 检查业务流程是否自洽
- 检查时间顺序是否合理
- 检查条件依赖是否正确

**常见错误**:
```
❌ 业务流程：先认购 → 后开户（应为先开户后认购）

❌ 交易规则：T 日 15:00 后申请，T+1 日确认
   费用说明：T 日 15:00 后申请按 T+1 日净值（矛盾）

❌ 风险提示：本产品不保本
   收益测算：预期年化收益 5%（暗示保本）
```

**修复建议**:
```
✅ 修正流程：开户 → 认购 → 确认 → 持有
✅ 统一交易规则描述
✅ 修改收益测算为"业绩比较基准"，并说明不保本
```

## 🔧 检查逻辑

```python
def check_consistency(document_content):
    """
    检查文档一致性
    
    返回:
        {
            "score": 90,  # 一致性得分
            "issues": [   # 发现的问题
                {
                    "type": "risk_level",
                    "severity": "high",
                    "location1": "产品概述",
                    "value1": "R3",
                    "location2": "风险揭示",
                    "value2": "R2",
                    "suggestion": "统一为 R3"
                }
            ],
            "suggestions": []
        }
    """
    
    issues = []
    
    # 1. 检查风险等级
    risk_issue = check_risk_level(document_content)
    if risk_issue:
        issues.append(risk_issue)
    
    # 2. 检查费率
    fee_issues = check_fees(document_content)
    issues.extend(fee_issues)
    
    # 3. 检查术语
    term_issues = check_terminology(document_content)
    issues.extend(term_issues)
    
    # 4. 检查数字
    number_issues = check_numbers(document_content)
    issues.extend(number_issues)
    
    # 5. 检查逻辑
    logic_issues = check_logic(document_content)
    issues.extend(logic_issues)
    
    # 计算得分
    score = calculate_score(issues)
    
    return {
        "score": score,
        "issues": issues,
        "suggestions": generate_suggestions(issues)
    }
```

## 📊 评分规则

| 问题类型 | 严重程度 | 扣分 | 示例 |
|---------|---------|------|------|
| 风险等级不一致 | 高 | -15 分 | R3 vs R2 |
| 费率不一致 | 高 | -10 分 | 1.5% vs 1.2% |
| 逻辑矛盾 | 高 | -15 分 | 流程颠倒 |
| 数字不一致 | 中 | -5 分 | 80%-95% vs 80%-90% |
| 术语不统一 | 低 | -2 分 | 份额 vs 单位 |

**基础分**: 100 分  
**最低分**: 0 分
