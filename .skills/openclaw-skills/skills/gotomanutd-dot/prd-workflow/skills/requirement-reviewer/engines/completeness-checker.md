# 完整性检查引擎

**版本**: v1.0  
**功能**: 检查 PRD 文档章节完整性

---

## 📋 必需章节清单

### P0 级章节（必需）

| 章节 | 关键词 | 检查规则 |
|-----|-------|---------|
| 产品概述 | 产品定位、目标用户、核心价值 | 至少包含 2 个关键词 |
| 功能需求 | 功能清单、功能描述 | 至少包含 1 个关键词 |
| 非功能需求 | 性能、安全、兼容性 | 至少包含 1 个关键词 |
| 业务流程 | 流程、流程图、步骤 | 至少包含 1 个关键词 |
| 合规要求 | 合规、监管、法规 | 至少包含 1 个关键词 |
| 风险揭示 | 风险、风险提示 | 至少包含 1 个关键词 |
| 验收标准 | 验收、测试、标准 | 至少包含 1 个关键词 |

### P1 级章节（建议）

| 章节 | 关键词 | 检查规则 |
|-----|-------|---------|
| 数据需求 | 数据、字段、数据库 | 至少包含 1 个关键词 |
| 界面原型 | 原型、界面、UI、设计稿 | 至少包含 1 个关键词 |
| 运营需求 | 运营、监控、统计 | 至少包含 1 个关键词 |

## 🔧 检查逻辑

```python
def check_completeness(document_content):
    """
    检查文档完整性
    
    参数:
        document_content: PRD 文档内容（字符串）
    
    返回:
        {
            "score": 85,  # 完整性得分
            "missing_p0": [],  # 缺失的 P0 章节
            "missing_p1": [],  # 缺失的 P1 章节
            "suggestions": []  # 补充建议
        }
    """
    
    # 1. 提取章节标题
    chapters = extract_chapters(document_content)
    
    # 2. 检查 P0 章节
    missing_p0 = []
    for chapter in P0_CHAPTERS:
        if not has_chapter(chapters, chapter):
            missing_p0.append(chapter)
    
    # 3. 检查 P1 章节
    missing_p1 = []
    for chapter in P1_CHAPTERS:
        if not has_chapter(chapters, chapter):
            missing_p1.append(chapter)
    
    # 4. 计算得分
    score = calculate_score(missing_p0, missing_p1)
    
    # 5. 生成建议
    suggestions = generate_suggestions(missing_p0, missing_p1)
    
    return {
        "score": score,
        "missing_p0": missing_p0,
        "missing_p1": missing_p1,
        "suggestions": suggestions
    }
```

---

## 📊 评分规则

| 缺失类型 | 扣分 | 示例 |
|---------|------|------|
| 缺失 1 个 P0 章节 | -15 分 | 缺失"风险揭示" |
| 缺失 2 个 P0 章节 | -30 分 | 缺失"风险揭示"+"合规要求" |
| 缺失 3+ 个 P0 章节 | -50 分 | 多个核心章节缺失 |
| 缺失 1 个 P1 章节 | -5 分 | 缺失"数据需求" |
| 缺失 2-3 个 P1 章节 | -10 分 | 缺失 2-3 个建议章节 |
| 缺失 4+ 个 P1 章节 | -15 分 | 多个建议章节缺失 |

**基础分**: 100 分  
**最低分**: 0 分
