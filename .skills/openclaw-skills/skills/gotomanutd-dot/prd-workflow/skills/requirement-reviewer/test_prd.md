# 测试 PRD - 下单支付功能

## 一、产品概述

这是一个简单的下单支付功能的 PRD。

## 二、业务流程

用户下单 → 系统处理 → 用户支付 → 支付完成

## 三、功能需求

### 功能 1：下单功能

用户点击下单按钮，系统创建订单。

**输入**
- 金额：数字，必填
- 产品代码：字符串，必填

**输出**
- 订单号：字符串

### 功能 2：支付功能

用户选择支付方式，完成支付。

## 四、验收标准

Given 用户已登录
When 用户点击购买按钮
Then 系统创建订单并跳转到支付页面

## 五、系统对接

上游对接核心系统获取用户信息，下游对接支付系统处理支付。
