# Requirement Reviewer 设计文档

## 一、分层架构

```
requirement-reviewer/
├── engines/
│   ├── core/                      # 形式检查层（规则匹配，秒级，无成本）
│   │   ├── completeness_checker.py
│   │   ├── consistency_checker.py
│   │   ├── terminology_checker.py
│   │   └── acceptance_criteria_checker.py
│   │
│   ├── semantic/                  # 内容检查层（AI理解，分钟级，有成本）
│   │   ├── section_parser.py      # 章节结构解析
│   │   ├── check_items.py         # 13项检查定义
│   │   ├── section_matcher.py     # 章节-检查项映射
│   │   └── ai_checker.py          # AI调用引擎
│   │
│   └── plugins/                   # 领域插件（可选）
│       └── financial/
```

## 二、执行流程

```
review 步骤：
  │
  ├─ 形式检查（规则匹配）
  │     ↓ 发现问题
  │   自动修补 → 重新检查 → 通过
  │
  ├─ 内容检查（AI理解）
  │     ↓ 发现问题
  │   问答引导用户修补
  │     ↓
  │   问题处理完毕
  │
  └─ 生成评审报告
```

## 三、13项检查

## 四、内容检查交互流程

### 按优先级、逐个问答

```
┌─────────────────────────────────────┐
│ 【核心问题】第 1/2 项                        │
├─────────────────────────────────────┤
│ 问题：流程描述不完整                         │
│ 章节：业务流程                               │
│ 说明：缺少异常分支说明（超时处理、失败重试）      │
├─────────────────────────────────────┤
│ 请选择：                                     │
│   1. AI 自动修补                             │
│   2. 告诉 AI 怎么修补                        │
│   3. 误判，可忽略                            │
└─────────────────────────────────────┘
```

### 三种选项的处理

| 选项 | 用户操作 | 系统行为 |
|------|---------|---------|
| 1. AI 自动修补 | 无需输入 | AI 生成修补内容，自动更新 PRD |
| 2. 告诉 AI 怎么修补 | 输入修补描述 | AI 按用户描述生成内容，更新 PRD |
| 3. 误判，可忽略 | 无需输入 | 标记忽略，记入报告，不修补 |

## 五、评审报告内容

- 检查时间
- 形式问题（已自动修补）
- 内容问题（已处理/已忽略）
- 评分
- 结论
