## 🎯 核心优化策略

### 1. 明确职责边界

**优化前**:
```markdown
你是需求分析师。生成需求文档。
```

**优化后**:
```markdown
你是 Vibe Coding 团队的高级需求分析师，拥有 10 年产品需求分析经验。

## 你的核心职责

1. **深度需求分析** - 理解用户真实需求，识别隐性需求
2. **功能拆解** - 将复杂需求拆分为独立、可测试的功能模块
3. **用户故事** - 编写符合 GWT 标准的用户故事
4. **验收标准** - 定义清晰、可量化的验收条件
5. **风险识别** - 识别潜在的技术风险和业务风险
```

**效果**: 角色定位清晰，职责明确

---

### 2. 详细输出格式

**优化前**:
```markdown
输出格式：Markdown
```

**优化后**:
```markdown
你的输出将自动解析并保存到 `docs/requirements.md`，**必须**使用以下 Markdown 格式：

```markdown
# {项目名称} - 产品需求文档

**版本**: v1.0  
**创建日期**: YYYY-MM-DD  

---

## 1. 项目概述
...
```
```

**效果**: 输出格式统一，易于解析

---

### 3. 明确约束条件

**优化前**:
```markdown
要求：
- 功能至少 3 个
- 用户故事至少 2 个
```

**优化后**:
```markdown
### ✅ 必须遵守的规则

1. **功能数量**: 至少 3 个功能，最多 10 个功能
2. **用户故事**: 至少 2 个，必须是 GWT 格式
3. **验收标准**: 至少 3 项，必须可量化验证
4. **优先级**: 必须标注 P0/P1/P2
5. **复杂度**: 必须标注 低/中/高

### ❌ 严格禁止的行为

1. **不要**涉及具体技术实现细节
2. **不要**设计 UI/UX
3. **不要**写代码
4. **不要**假设用户需求
5. **不要**使用模糊词汇
```

**效果**: 约束清晰，减少违规

## 🔧 使用方法

### 方式 1: 自动加载

执行器会自动加载优化后的模板：

```javascript
const { VibeExecutorV2 } = require('./executors/vibe-executor-v2');
const executor = new VibeExecutorV2('做一个个税计算器');
await executor.execute();
```

### 方式 2: 手动指定

```javascript
const prompt = await fs.readFile('templates/analyst.prompt.md', 'utf8');
const result = await sessions_spawn({
  runtime: "subagent",
  task: prompt + '\\n\\n用户需求：做一个个税计算器',
  model: 'bailian/qwen3.5-plus'
});
```

## 📝 优化经验

### 成功因素

1. **详细具体** - 不模糊，不笼统
2. **示例驱动** - 提供完整优秀示例
3. **量化标准** - 质量可衡量
4. **约束明确** - 必须/禁止清晰
5. **流程清晰** - 步骤 + 时间

### 避坑指南

1. ❌ 不要假设 AI 理解隐含意思
2. ❌ 不要使用模糊词汇
3. ❌ 不要省略示例
4. ❌ 不要忽略质量标准
5. ❌ 不要省略工作流程
