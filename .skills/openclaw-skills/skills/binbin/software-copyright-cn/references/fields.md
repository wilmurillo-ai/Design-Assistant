# 软著登记申请字段说明

## 基本信息字段

### 软件全称
- **必填**，申请著作权登记的软件全称
- 应简短明确、针对性强
- 全部申请材料中名称保持一致
- 示例：`智慧校园教务管理系统`、`基于深度学习的图像识别平台`

### 软件简称
- **选填**，对全称的简化
- 无简称则**留空**，不要填"无"
- 示例：全称"智慧校园教务管理系统" → 简称"智慧教务系统"

### 版本号
- **必填**，格式 `V数字` 或 `数字`
- 示例：`V1.0`、`V2.0.1`、`1.0`

### 软件分类
- **必填**，四选一：
  - 应用软件
  - 嵌入式软件
  - 中间件
  - 操作系统

## 环境字段（均50字以内）

### 开发的硬件环境
- 开发时使用的硬件配置
- 示例：`Intel Core i7处理器，16GB内存，512GB SSD硬盘`

### 运行的硬件环境
- 软件运行所需的硬件配置
- 示例：`PC服务器，Intel Xeon处理器，8GB及以上内存，100GB以上硬盘`

### 开发该软件的操作系统
- 开发时使用的操作系统及版本
- 示例：`Windows 11 专业版`、`macOS 14 Sonoma`、`Ubuntu 22.04 LTS`

### 软件开发环境/开发工具
- 开发使用的IDE、编辑器、工具链
- 示例：`Visual Studio Code 1.85、Node.js 20.10`、`IntelliJ IDEA 2023.3`

### 该软件的运行平台/操作系统
- 运行软件所需的操作系统及版本
- 示例：`Windows 10及以上`、`Android 10及以上`、`CentOS 7.9`

### 软件运行支撑环境/支持软件
- 运行所需的依赖软件、数据库、中间件等
- 示例：`MySQL 8.0、Redis 7.0、Nginx 1.24`、`JDK 17、Apache Tomcat 10`

## 编程语言
- **必填**，可多选（最多10项）
- 预设选项：Assembly language、C、C#、C++、Delphi/Object Pascal、Go、HTML、Java、JavaScript、MATLAB、Objective-C、PHP、PL/SQL、Perl、Python、R、Ruby、SQL、Swift、Visual Basic、Visual Basic .Net
- 不在列表中可文字补充（如 TypeScript、Kotlin、Rust、Dart 等）

## 源程序量
- **必填**，整数，单位：行
- 统计项目全部源代码行数
- 示例：`52360`

## 文本描述字段

### 开发目的
- **必填**，50字以内，至少8字
- 描述创作该软件的目的
- 示例：`为解决中小企业进销存管理效率低下的问题，提供一体化的智能管理解决方案`

### 面向领域/行业
- **必填**，50字以内，至少4字
- 描述软件面向或适用的行业或领域
- 示例：`教育行业`、`金融科技`、`智慧医疗与健康管理`

### 软件的主要功能
- **必填**，110-200字（含标点）
- 描述软件的核心功能模块
- 应涵盖主要功能点，层次清晰

### 软件的技术特点
- **必填**，分两部分：
  1. **技术标签**（最多3项）：APP、游戏、软件、教育软件、金融软件、医疗软件、地理信息软件、云计算软件、信息安全软件、大数据软件、人工智能软件、VR软件、5G软件、小程序、物联网软件、智慧城市软件
  2. **技术特点描述**（**必填**，100字以内）：除选择技术标签外，还必须填写一段技术特点的文字描述

## 鉴别材料

### 程序鉴别材料
- **必填**，PDF格式
- 源程序连续的前30页 + 连续的后30页
- 每页不少于50行
- 源程序不足60页则提交全部
- 前后各30页可按开发时间排序或按功能主次自定义排序
- 页眉应包含软件名称和版本号

### 文档鉴别材料
- **必填**，PDF格式
- 任何一种文档（用户手册、设计说明书等）的连续前30页 + 连续后30页
- 每页不少于30行
- 文档不足60页则提交整个文档
- 文档中的权利人署名、软件名称、版本号须与其他申请材料一致
- 文档类型：程序设计说明书、流程图、用户手册等
- **至少60页**

## 联系地址
- **必填**字段：真实姓名、国家、省/市/区、详细地址、邮编、手机号
- **选填**字段：收件单位
