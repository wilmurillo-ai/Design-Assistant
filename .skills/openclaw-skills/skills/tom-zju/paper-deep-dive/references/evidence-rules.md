# Paper Deep Dive - 证据规则与不确定性标注

如何正确标注证据来源、表达不确定性，避免推测写成事实。

---

## 证据类型体系

### 五类证据标签

| 标签 | 英文 | 含义 | 可信度 | 使用场景 |
|------|------|------|--------|---------|
| **论文明确声称** | Explicit Claim | 作者在文中直接提出的结论 | ⭐⭐⭐⭐⭐ | 引用作者原话或明确声明 |
| **直接证据** | Direct Evidence | 实验结果直接支撑 | ⭐⭐⭐⭐⭐ | 表格/图表数据直接支持 |
| **部分证据** | Partial Evidence | 有证据但不够完整 | ⭐⭐⭐ | 某些条件下成立 |
| **基于证据的推断** | Evidence-based Inference | 结合结果进行的合理推断 | ⭐⭐ | 逻辑推导但非直接证明 |
| **尚未验证** | Unverified | 论文暗示但没有严谨验证 | ⭐ | 提及但未测试 |

---

## 标签使用规范

### 论文明确声称 (Explicit Claim)

**定义**：作者在论文中明确写出的结论，可直接引用。

**识别标志**：
- "We propose..."
- "Our method achieves..."
- "The key insight is..."
- "We demonstrate that..."

**使用方式**：
```markdown
论文明确声称该方法在ImageNet上的准确率达到85.4%（Section 4.1）。
```

**注意事项**：
- 注意声称的条件（如"在特定设置下"、"使用额外数据"）
- 区分主要声称和次要声称

---

### 直接证据 (Direct Evidence)

**定义**：实验结果、图表数据直接支持结论，无需额外推断。

**识别标志**：
- 表格中的对比数字
- 图中的性能曲线
- 统计检验结果

**使用方式**：
```markdown
直接证据来自Table 2：本文方法在准确率上比Baseline高3.2个百分点（82.1% vs 78.9%）。
```

**注意事项**：
- 检查实验设置是否一致
- 注意统计显著性（p-value）

---

### 部分证据 (Partial Evidence)

**定义**：有证据支持，但只在特定条件、数据集或设置下成立。

**识别标志**：
- "在CIFAR-10上..."（但其他数据集未测试）
- "当模型规模大于1B时..."（小规模未验证）
- 只在部分消融实验中成立

**使用方式**：
```markdown
部分证据显示该方法在CIFAR-10上有效（Table 3），但论文未报告ImageNet上的结果。
```

**注意事项**：
- 明确指出证据的适用范围
- 说明哪些情况未覆盖

---

### 基于证据的推断 (Evidence-based Inference)

**定义**：基于实验结果进行逻辑推导，但不是直接观测。

**识别标志**：
- 需要多步推理
- 解释"为什么"有效
- 归因于特定机制

**使用方式**：
```markdown
基于证据的推断，该组件可能起主要作用：消融实验显示去掉它性能下降5%，而去掉其他组件只下降1-2%。
```

**注意事项**：
- 明确标注这是推断
- 提供推断的逻辑链条
- 承认可能存在其他解释

---

### 尚未验证 (Unverified)

**定义**：论文中提及或暗示，但没有进行实验验证。

**识别标志**：
- "We leave X for future work"
- "It is possible that..."
- 只在Introduction中提及，实验部分未测试

**使用方式**：
```markdown
这一点尚未验证：论文仅在Discussion中提到该方法可能适用于多模态场景，但未进行相关实验。
```

**注意事项**：
- 不要将其作为已验证的事实
- 区分"作者说未来工作"和"本文已完成"

---

## 不确定性表达模板

### 通用模板

| 情境 | 模板 | 示例 |
|------|------|------|
| 论文没讲清楚 | "论文没有完整说明……" | 论文没有完整说明该超参数的选取策略 |
| 基于推断 | "这一点更像是根据……推断出来的" | 这一点更像是根据Figure 3的趋势推断出来的 |
| 结果未隔离验证 | "结果表明……，但论文没有隔离验证……" | 结果表明整体提升显著，但论文没有隔离验证该组件的独立贡献 |
| 依赖前提 | "这个解释依赖于……前提" | 这个解释依赖于训练充分收敛的前提 |
| 代码与论文不一致 | "代码实现中使用了X，而论文描述为Y" | 代码实现中使用了AdamW，而论文描述为SGD |
| 特定条件 | "这一点只在……条件下成立" | 这一点只在大规模数据集上成立 |
| 未充分测试 | "论文仅在……上测试，未覆盖……" | 论文仅在英文数据上测试，未覆盖其他语言 |

## 具体场景处理

### 场景1：实现细节缺失

**问题**：论文描述与实现细节不一致，或缺少关键细节。

**处理方式**：
```markdown
论文描述使用X优化器，但官方代码实现中使用Y优化器，此处存在差异。

论文没有完整说明学习率调度策略的具体参数，只提到"使用cosine annealing"。
```

---

### 场景2：结果归因不明

**问题**：知道结果好，但不确定是哪个因素导致的。

**处理方式**：
```markdown
消融实验（Table 4）显示同时去掉A和B性能下降10%，
但论文没有单独测试只去掉A或只去掉B的情况，
因此A和B各自的贡献无法从直接证据中分离。

基于证据的推断，A可能是主要因素，
因为文中提到A解决了前任方法的核心问题，
但这一推断未被实验直接验证。
```

---

### 场景3：泛化性存疑

**问题**：只在特定条件下测试，泛化性未知。

**处理方式**：
```markdown
论文明确声称该方法在数据集X、Y、Z上有效（Section 4.1）。

然而，部分证据显示在数据集W上效果不佳（Figure 5的小字注脚），
这一点论文没有充分讨论。

该方法在低资源场景下的表现尚未验证。
```

---

### 场景4：对比不公平

**问题**：与baseline的比较可能存在不公平因素。

**处理方式**：
```markdown
直接证据显示本文方法比Baseline A高2个百分点（Table 1）。

但论文没有明确说明是否使用了相同的训练数据增强策略，
这一点可能影响比较的公平性。

Baseline B的结果直接引用了其原论文（2020年），
而本文方法使用了更新的训练技巧，
因此提升可能部分来自训练优化而非方法本身。
```

---

### 场景5：机制解释推测

**问题**：论文对"为什么有效"的解释可能是推测性的。

**处理方式**：
```markdown
论文声称该机制有效是因为"捕捉了长程依赖"（Section 3.2），
但这一解释更像是基于注意力权重可视化的推断，
而非直接的因果验证。

论文没有设计实验来验证"长程依赖"是否真的是关键因素，
其他解释（如更强的局部特征提取）也可能成立。
```

## 常见错误与避免

### ❌ 错误：把推测写成事实

**错误示例**：
```markdown
该方法通过引入X机制提升了性能。
```

**问题**：没说是论文声称的还是你推断的。

**正确写法**：
```markdown
论文明确声称X机制是性能提升的关键（Section 3），
直接证据来自消融实验（Table 2）：去掉X后性能下降5%。
```

---

### ❌ 错误：混淆相关与因果

**错误示例**：
```markdown
因为使用了更大的模型，所以性能更好。
```

**问题**：可能有其他因素。

**正确写法**：
```markdown
实验结果显示模型规模与性能正相关（Figure 4），
但这一基于证据的推断没有排除其他因素
（如训练时间更长、数据更多）的影响。
```

---

### ❌ 错误：过度解读图表

**错误示例**：
```markdown
从图3可以看出，该方法在所有情况下都最优。
```

**问题**：图中可能只展示了部分结果。

**正确写法**：
```markdown
直接证据来自Figure 3：在展示的6个测试案例中，
该方法在5个上达到最优，1个上接近最优。

但论文没有说明这6个案例是如何选取的，
是否存在选择偏差尚未验证。
```

---

### ❌ 错误：忽视边界条件

**错误示例**：
```markdown
该方法的时间复杂度是O(n)。
```

**问题**：可能有特定条件。

**正确写法**：
```markdown
论文明确声称在稀疏假设下，
该方法的时间复杂度为O(n)（Section 3.2，Theorem 1）。

在一般稠密情况下，复杂度为O(n²)，
与标准方法相同。
```

## 证据标注实践

### 实践1：解读实验结果

```markdown
### 主实验分析

**论文明确声称**：
该方法在四个标准数据集上达到SOTA（Abstract, Section 4.1）。

**直接证据**：
Table 2显示在Dataset A、B、C上分别超过前任SOTA 1.2%、0.8%、2.1%，
但在Dataset D上低于最近的工作Y（-0.5%）。

**部分证据**：
Dataset D上的性能在附录中报告，正文未讨论，
可能暗示该方法在该数据集上优势不明显。

**基于证据的推断**：
该方法可能更适合具有X特性的数据集，
因为表现好的A、B、C都具有X特性，
而D不具备（文中提到D的特性是Z）。

**尚未验证**：
论文没有测试在人工构造的"反X"数据集上的表现，
因此上述推断未被直接验证。
```

---

### 实践2：解读方法贡献

```markdown
### 核心贡献分析

**论文明确声称**：
该工作的主要贡献是提出了X机制（Section 1, Abstract）。

**直接证据**：
消融实验（Table 4）显示：
- 完整模型：85.2%
- 去掉X机制：80.1%（-5.1%）
- 去掉Y组件：83.9%（-1.3%）
- 去掉Z技巧：84.5%（-0.7%）

**基于证据的推断**：
X机制似乎是主要贡献来源，
因为其移除导致的性能下降远大于其他组件。

**不确定性**：
然而，论文没有说明这些组件之间是否存在交互效应。
如果X、Y、Z必须同时使用才能生效，
单独测试可能低估了Y和Z的贡献。

**代码与论文的不一致**：
官方代码实现中还包含了一个未在论文中描述的W技巧，
其贡献程度尚未验证。
```

## 质量检查清单

### 证据标注检查

- [ ] 重要结论是否标注了证据来源？
- [ ] 是否区分了"论文声称"和"实验证据"？
- [ ] 推断性结论是否明确标注为"推断"？
- [ ] 未验证的内容是否明确标注？
- [ ] 是否检查了代码与论文的一致性？

### 不确定性表达检查

- [ ] 是否有推测被写成了事实？
- [ ] 是否混淆了相关性和因果性？
- [ ] 是否忽视了边界条件？
- [ ] 是否过度解读了图表？
- [ ] 是否对不确定性保持诚实？

## 示例：完整的证据分析段落

```markdown
### 性能分析

**论文明确声称**（Abstract, Section 4.1）：
该方法在ImageNet上达到85.4%的top-1准确率，
超过前任SOTA 1.2个百分点。

**直接证据**（Table 1）：
- 本文方法：85.4%
- 前任SOTA X：84.2%
- Baseline Y：82.1%

然而，这一比较的公平性存在以下不确定性：

1. **训练设置不一致**：
   论文明确声称使用了额外的数据增强（Section 4.2），
   但未明确说明前任方法是否也使用了相同的增强策略。
   如果前任方法未使用，1.2%的提升可能部分来自数据增强。

2. **模型规模差异**：
   本文方法使用150M参数，前任SOTA使用100M参数。
   基于证据的推断，规模差异可能解释0.3-0.5%的提升
   （根据Figure 3的scaling曲线），
   但论文没有明确隔离验证这一点。

3. **未充分测试的场景**：
   论文仅在ImageNet上报告结果，
   该方法在其他数据集上的泛化性尚未验证。

**综合评估**：
直接证据支持"本文方法在特定条件下表现优异"，
但"方法本身优于前任"这一结论的证据强度为中等，
受到训练设置和模型规模的混杂因素影响。
```
