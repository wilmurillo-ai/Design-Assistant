# SP Engine Rubric

## Scoring Model

Total default score: 10

- 现病史完整性: 3
- 关键危险信号追问: 2
- 既往/过敏/用药史: 2
- 个人/家族/相关史: 1
- 沟通闭环（归纳+确认）: 1
- 结构与逻辑: 1

## Must-Catch Items by Complaint Type

## Acute abdominal pain

1. 起病方式、部位、性质、程度、持续时间
2. 放射痛
3. 发热、恶心呕吐、黄疸、黑便/血便
4. 饮食诱因、体位关系
5. 既往腹部手术史与同类发作
6. 用药史（尤其止痛药、抗凝药）
7. 过敏史

## Chest pain

1. 胸痛性质与持续时间
2. 放射至肩背/上肢/下颌
3. 呼吸困难、出汗、恶心、晕厥
4. 心血管危险因素
5. 既往冠心病/高血压/糖尿病
6. 当前用药与抗凝史

## Fever

1. 体温峰值与热型
2. 畏寒寒战、皮疹、咳嗽/腹泻/尿路症状
3. 流行病学暴露
4. 免疫状态与基础病
5. 抗菌药使用史

## Deduction Rules

1. Missing a high-risk red flag: -2 each.
2. Missing allergy or medication safety history: -2 each.
3. Missing closure summary: -1.
4. Disordered interview causing major logic gaps: -1 to -2.

## Feedback Contract

After each session, always output:

1. Deduction table
2. Top 3 missed points
3. Corrected questioning script (<= 6 lines)
4. One immediate retest scenario
