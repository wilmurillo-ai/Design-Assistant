# 会议纪要质检清单

在输出最终结果前，逐项检查：

1. 是否包含执行摘要，且为 3-5 句。
2. 是否按“问题描述 -> 原因分析 -> 解决方案 -> 最终决议”组织每个议题。
3. 是否明确区分“已决议”与“待确认”。
4. 是否提供“核心决议清单”，且可独立阅读。
5. `Action Items` 是否为 Markdown 表格，且四列齐全：
   - 任务项
   - 负责人
   - 交付物/截止日期
   - 备注
6. 是否对所有“无 Action Item 的结论”给出风险提示。
7. 是否避免输出“今天大家讨论了...”等空话开场。
8. TTS 稿是否只覆盖“决议 + 待办 + 风险”，且含节奏标记。
9. 对缺失字段是否使用 `待指定` / `待确认`，并给出补全建议。
10. 若用户要求语音文件：是否输出 `m4a` 或 `mp3` 且非空，并注明“实际使用音色”与文件路径。
11. 语音是否执行后处理（或明确说明未执行原因，如缺少 ffmpeg）。
12. Action Items 是否通过硬门禁（负责人与交付物/截止日期均不可为空/占位符）。
13. ASR 结果是否包含低置信度片段清单，并完成人工复核标记。
14. 是否生成独立口播稿文件（`.spoken.txt`），且 TTS 使用的是口播稿而非原文档。
15. 是否输出思维导图产物（`.mindmap.json` + `.html` + `.xmind`），且主题与会议纪要一致。
