# skill-setup-flow 技能

## 概述

这是一个**元技能**（meta-skill），用于为其他已安装的 OpenClaw 技能创建标准化的设置流程。

## 问题

每个 OpenClaw 技能安装后都需要：
- 创建目录结构
- 创建配置文件
- 更新核心文件（SOUL.md, AGENTS.md 等）
- 记录设置日志
- 提供使用说明

手动完成这些步骤既繁琐又容易遗漏。

## 解决方案

`skill-setup-flow` 提供标准化的设置流程模板，可以：
- 自动分析技能类型
- 创建必要的目录和文件
- 更新核心配置文件
- 记录完整的设置日志
- 提供快速参考和使用指南

## 使用方法

### 基本用法

```
为 {skill-name} 创建设置流程
初始化 {skill-name} 配置
设置 {skill-name} 技能
启用 {skill-name}
```

### 示例

```bash
# 为 self-improving 创建设置
为 self-improving 创建设置流程

# 为 weather 创建配置
为 weather 创建设置流程

# 为 proactive-agent 初始化
初始化 proactive-agent 配置
```

## 文件结构

```
skills/skill-setup-flow/
├── SKILL.md           # 技能定义（本文件）
├── README.md          # 使用说明（本文件）
├── QUICKSTART.md      # 快速参考
├── setup-template.md  # 设置模板
└── examples/
    └── self-improving-setup.md  # 完整示例
```

## 支持的技能类型

### A. 配置型技能
需要 API 密钥或配置项
- 示例：weather, news-summary, tavily
- 创建：config.md, 配置模板

### B. 数据型技能
需要存储和管理数据
- 示例：self-improving, memory 相关
- 创建：数据目录，索引文件

### C. 行为型技能
改变 AI 的行为模式
- 示例：self-improving, proactive-agent
- 创建：memory.md, 行为规则

### D. 工具型技能
提供新的工具或命令
- 示例：mcporter, oracle
- 创建：工具配置，快速参考

## 标准流程

### 步骤 1: 分析技能
- 读取 `SKILL.md` 了解需求
- 确定技能类型（配置/数据/行为/工具）
- 识别必要的配置项

### 步骤 2: 创建结构
- 创建必要的目录
- 创建配置文件模板
- 创建状态跟踪文件

### 步骤 3: 集成配置
- 更新 `SOUL.md` 添加行为指导
- 更新 `AGENTS.md` 添加使用说明
- 更新 `MEMORY.md` 记录设置事件

### 步骤 4: 测试验证
- 基本功能测试
- 集成测试
- 记录测试结果

### 步骤 5: 文档记录
- 创建设置日志
- 提供快速参考
- 记录配置选项

## 输出内容

每次设置流程会创建：

1. **设置日志** (`setup-log.md`)
   - 设置日期和时间
   - 创建的文件清单
   - 更新的核心文件
   - 配置选项
   - 测试结果

2. **配置文件** (根据技能类型)
   - `config.md` - 配置选项
   - `memory.md` - 技能记忆
   - `state.md` - 运行状态

3. **核心文件更新**
   - `SOUL.md` - 行为指导
   - `AGENTS.md` - 使用说明
   - `MEMORY.md` - 事件记录

4. **快速参考卡**
   - 常用命令
   - 配置位置
   - 故障排除

## 示例输出

查看 `examples/self-improving-setup.md` 查看完整的设置流程示例。

## 最佳实践

1. **最小化配置** - 只创建必要的文件
2. **提供默认值** - 所有配置都有合理默认值
3. **记录一切** - 详细记录设置过程
4. **可逆操作** - 提供重置/卸载方法
5. **测试导向** - 设置完成后立即测试

## 与其他技能的关系

- `skill-creator` - 创建新技能后使用此技能设置
- `skill-vetter` - 审查技能后再安装和设置
- `find-skills` - 找到技能后安装并设置
