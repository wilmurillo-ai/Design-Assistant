# EXTEND.md

本文件用于配置你的本地运行偏好。它不是文案模板，而是技能运行参数说明。

## 一、默认输出偏好
- 默认输出模式：auto
- 默认发布平台：moments
- 默认文案气质：轻故事 + 生活方式 + 热点轻联动
- 默认是否带联系方式：否
- 默认是否生成图片：是
- 默认是否优先直出图：是
- 默认是否允许图上文字：是
- 默认是否允许人物叙事：是
- 默认图片尺寸：1024x1792
- 默认生成张数：1

## 二、微博热搜来源
本技能已经**内置并整合**了 `weibo-hot-trend`，默认不再依赖外部占位命令。

默认行为：
- 优先调用本技能目录下 `vendor/weibo-hot-trend/scripts/weibo.js`
- 若你想改用外部安装版本，可在 `config/runtime.json` 中指定命令，例如：
  - `node ~/.openclaw/workspace/skills/weibo-hot-trend/scripts/weibo.js 20`

注意：
- 不允许跳过微博热搜扫描
- 未扫描热搜时，不允许宣称已做“今日热点联动”

## 三、图片生成来源
默认通道：`direct_prompt`

含义：
- 由本技能先生成主图提示词
- 再由本技能把提示词包装成一段“生成图片”标准指令
- 然后由当前宿主/本地大模型直接执行出图

这条逻辑不是“给你一个 prompt 自己想办法”，而是：
- 技能必须明确地下发“生成图片”请求
- 如果宿主不支持，才回退为仅输出提示词
- 提示词默认要和主文案强绑定，允许图上文字与人物故事场景

可选扩展：
- 以后你也可以在 `config/runtime.json` 中加上 `local_command` 或 `local_api`
- 但默认完整链路就是 `direct_prompt`

## 四、豆包说明（固定保留）
无论本地出图是否成功，最终输出中都保留这句话：

> 你可以把以上提示词直接发给豆包，由豆包生成图片；如果当前环境支持，本技能也会优先直接尝试出图。
