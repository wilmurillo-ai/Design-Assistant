# 02-estate-profile.md

## 目标
当楼盘没有现成 seed 档案或本地档案时，必须触发**楼盘信息检索工作流**，先建立一个可复用的楼盘基础档案，再进入热点联动与文案生成。

本工作流的核心不是“随便搜一点”，而是：
1. 明确使用互联网检索
2. 明确优先搜索哪些平台
3. 明确要收集哪些信息
4. 明确如何交叉验证
5. 明确何时允许进入后续文案环节

---

## 触发条件
满足任一条件时，必须执行本工作流：
- `data/seed-estates/` 中没有该楼盘档案
- 本地缓存档案不存在
- 本地缓存档案字段缺失严重，无法支撑文案生成
- 用户明确要求分析一个新楼盘

若已存在完整档案，可跳过检索流程，直接进入“楼盘理解摘要”。

---

## 强制规则
- 必须明确使用互联网检索，不得只靠模型记忆补全楼盘信息。
- 必须至少搜索 **5 个来源**，其中至少 **2 个房产信息源** + **2 个内容/舆情源** + **1 个综合搜索源**。
- 必须优先从以下平台中检索：
  - 房产信息源：安居客、贝壳、房天下
  - 内容/舆情源：抖音、快手、微博、今日头条
  - 综合搜索源：百度
- 若某平台当前环境无法直接访问，也必须说明“该平台当前无法直连”，并改用可访问平台或公开搜索结果补足。
- 不得把单一来源的营销话术直接当作事实；必须进行交叉验证。
- 未经验证的信息不得写入“已确认信息”，只能写入“待核实线索”。

---

## 推荐搜索顺序
### 第一步：综合确认楼盘身份
先确认最基本的识别信息，避免搜错盘。

推荐搜索关键词：
- `楼盘名 城市`
- `楼盘名 项目地址`
- `楼盘名 开发商`
- `楼盘名 安居客`
- `楼盘名 贝壳`
- `楼盘名 房天下`

目标：
- 确认楼盘标准名称
- 确认所在城市 / 板块 / 地址
- 确认产品类型（住宅 / 改善 / 豪宅 / 现房等）
- 识别是否有重名项目

### 第二步：优先抓房产基础信息
优先搜索以下来源：
1. 安居客
2. 贝壳
3. 房天下
4. 百度搜索结果页中的公开项目页/新闻页

必须重点提取：
- 项目定位
- 面积段 / 户型段
- 现房 / 期房 / 交付状态
- 区位 / 交通 / 商业 / 医疗 / 教育等基础配套
- 项目卖点原始描述

### 第三步：补充内容平台与舆情材料
优先搜索以下来源：
1. 抖音
2. 快手
3. 微博
4. 今日头条
5. 百度补充搜索

必须重点提取：
- 用户讨论重点
- 常被提及的优点
- 常被提及的疑虑或争议点
- 哪些视觉元素最常被拍摄或传播（外立面、园林、水景、车库、会所、连廊等）
- 是否存在可形成传播感的话题标签

### 第四步：图片线索采集
从可访问的公开页面里收集“图片线索”，而不是直接下载和复用他人图片。

要记录：
- 高频视觉元素
- 常见拍摄角度
- 适合重建的画面类型
- 不适合生成的误导性画面

---

## 搜索执行模板
当你开始检索新楼盘时，先在内部按下面顺序执行：

1. 综合搜索：
- `楼盘名 + 城市`
- `楼盘名 + 地址`
- `楼盘名 + 安居客`
- `楼盘名 + 贝壳`
- `楼盘名 + 房天下`

2. 内容平台搜索：
- `楼盘名 + 抖音`
- `楼盘名 + 快手`
- `楼盘名 + 微博`
- `楼盘名 + 今日头条`

3. 补充搜索：
- `楼盘名 + 评价`
- `楼盘名 + 口碑`
- `楼盘名 + 优缺点`
- `楼盘名 + 实景`
- `楼盘名 + 园林 / 外立面 / 车库 / 会所`

---

## 输出字段要求
检索完成后，必须产出一个“楼盘基础档案”，至少包含：

### A. 已确认信息
- 标准名称
- 城市 / 板块 / 地址
- 项目定位
- 面积段 / 主力户型
- 当前状态（现房 / 在售 / 已交付等）
- 核心卖点
- 主要配套

### B. 舆情与内容线索
- 常见正面评价
- 常见负面评价 / 顾虑点
- 高频视觉元素
- 适合传播的生活方式关键词
- 不建议硬写的风险点

### C. 待核实线索
- 单一平台提到、但未交叉验证的信息
- 容易夸大或带强营销性质的信息

### D. 来源记录
至少记录 5 个来源，格式建议：
- 来源平台：
- 页面标题：
- 链接或检索描述：
- 用途：基础信息 / 舆情 / 视觉线索 / 配套验证

---

## 通过标准
只有当以下条件满足时，才允许进入后续环节：
- 已确认信息字段完整度达到 80% 以上
- 至少有 5 个来源
- 至少有 3 个核心卖点可用于文案
- 至少有 2 个视觉线索可用于图片提示词
- 至少有 1 个负面或风险点被识别并记录

如果不满足，必须继续检索，不能直接开始写文案。

---

## 写入本地档案
检索完成后，必须把结果写入本地档案，建议输出两个文件：
- `profile.md`：给人读的楼盘摘要
- `profile.json`：给后续工作流读的结构化数据

写入后，后续再次调用同一楼盘时优先复用本地档案。

---

## 注意事项
- 抖音、快手、微博、头条等平台的公开搜索可用于发现内容线索，但不应把单条短视频文案直接当事实依据。
- 房产平台页面信息通常更适合确认面积段、配套、状态等基础信息。
- 若用户后续提供了官方项目资料，应以用户提供资料优先修正本地档案。
