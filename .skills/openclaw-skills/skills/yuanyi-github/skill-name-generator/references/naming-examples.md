## 英文命名模式

### 模式 1: 动作 + 对象
适用场景:强调操作行为
- `pdf-parser` - 解析 PDF (SEO: pdf 高频词，2个单词)
- `image-compress` - 压缩图片 (SEO: image 通用词，2个单词)
- `text-analyze` - 分析文本 (SEO: text 搜索量大，2个单词)
- `data-convert` - 转换数据 (SEO: data 核心词，2个单词)
- `file-split` - 分割文件 (SEO: file 直观，2个单词)
- `code-format` - 格式化代码 (SEO: code 极高频，2个单词)
- `email-send` - 发送邮件 (SEO: email 通用，2个单词)
- `video-encode` - 编码视频 (SEO: video 高频，2个单词)

### 模式 2: 对象 + 动作
适用场景:强调处理对象
- `data-export` - 数据导出 (SEO: data + export 双关键词，2个单词)
- `document-scan` - 文档扫描 (SEO: document 专业词，2个单词)
- `message-filter` - 消息过滤 (SEO: message 通用，2个单词)
- `resource-optimize` - 资源优化 (SEO: resource 专业，2个单词)
- `content-generate` - 内容生成 (SEO: content 高频，2个单词)
- `system-monitor` - 系统监控 (SEO: system 通用，2个单词)
- `workflow-automate` - 工作流自动化 (SEO: workflow 专业，2个单词)

### 模式 3: 工具 + 领域
适用场景:强调专业领域
- `code-review` - 代码审查 (SEO: code + review 双高频，2个单词)
- `email-marketing` - 邮件营销 (SEO: email + marketing，2个单词)
- `finance-calc` - 金融计算 (SEO: finance + calc，2个单词)
- `music-tag` - 音乐标签 (SEO: music 高频，2个单词)
- `social-share` - 社交分享 (SEO: social 高频，2个单词)

### 模式 4: 领域 + 功能
适用场景:强调功能应用
- `finance-analyzer` - 金融分析器 (SEO: finance + analyzer，2个单词)
- `education-tutor` - 教育辅导 (SEO: education 专业，2个单词)
- `health-tracker` - 健康追踪 (SEO: health 高频，2个单词)

### 模式 5: 平台 + 功能
适用场景:针对特定平台的工具

适用场景:强调功能性质的平台工具

## 常见后缀词库

### 处理类
- `parser`: 解析器，用于解析文件或数据 (SEO 中频词)
- `extractor`: 提取器，用于提取信息 (SEO 中频词)
- `converter`: 转换器，用于格式转换 (SEO 高频词)
- `compressor`: 压缩器，用于压缩文件 (SEO 中频词)
- `optimizer`: 优化器，用于优化性能或质量 (SEO 高频词)

### 分析类
- `analyzer`: 分析器，用于分析数据或内容 (SEO 高频词)
- `scanner`: 扫描器，用于扫描检查 (SEO 中频词)
- `monitor`: 监控器，用于监控状态 (SEO 高频词)
- `inspector`: 检查器，用于检查代码或文档 (SEO 中频词)
- `detector`: 检测器，用于检测问题或特征 (SEO 中频词)

### 生成类

- `builder`: 构建器，用于构建项目或对象 (SEO 高频词)

## SEO 命名指南

### SEO 评分标准 (满分 10 分)
1. **关键词匹配度** (3 分): 名称是否包含用户搜索的核心关键词
2. **词汇热度** (2 分): 使用的词汇是否为高频搜索词
3. **简洁直观性** (2 分): 名称是否让用户一眼理解功能
4. **行业认可度** (2 分): 命名是否符合行业标准
5. **独特性** (1 分): 是否在同类工具中具有辨识度

### SEO 优化策略

#### 策略 1: 关键词前置
- 将用户最可能搜索的关键词放在名称第一位
- 示例: `pdf-parser` 优于 `parser-pdf`
- 原因: 搜索算法更重视前置词汇

#### 策略 2: 使用高频词
- 优先使用用户搜索频率高的词汇
- 极高频词: code、data、pdf、image、api、web、app
- 高频词: text、file、document、video、music、email
- 中频词: parser、converter、generator、analyzer

#### 策略 3: 避免缩写
- 除非行业通用(如 pdf、api、http)，否则避免缩写
- 错误示例: `img-comp` (image compress 的缩写)
- 正确示例: `image-compress`
- 原因: 用户很少搜索缩写

#### 策略 4: 精准匹配搜索意图
- 分析用户搜索此功能时的心理和表达习惯
- 示例: 用户搜"压缩图片" > `image-compress` 优于 `pic-shrink`
- 原因: 匹配用户搜索习惯提升点击率

#### 策略 5: 竞品参考
- 参考市场上成功工具的命名方式
- 识别行业标准术语(如 code-review、data-export)
- 原因: 用户已被教育了这些术语

## 命名最佳实践

### DO 推荐做法
- 使用简洁、明确且高频的词汇
- 遵循小写字母+连字符的格式
- 确保名称能准确传达功能
- 保持名称长度适中(20-40字符)
- 使用领域内通用的术语
- **优先考虑 SEO 效果**，包含高频搜索词
- 将核心关键词前置以提升搜索权重
- 参考行业标准术语和竞品命名

### DON'T 避免做法
- 使用驼峰命名或下划线
- 添加 "-skill" 后缀
- 使用过于复杂或生僻的词汇
- 名称过长或过短
- 使用容易引起歧义的词汇
- 使用非行业标准的缩写
- 忽略用户搜索习惯和关键词
