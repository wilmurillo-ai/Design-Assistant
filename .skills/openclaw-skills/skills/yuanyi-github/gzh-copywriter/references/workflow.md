# 完整执行流程与创作规范

## Step 0: 意图识别与关键词提炼

在查询爆款之前，先理解用户真实需求并提炼关键词。

1. 理解用户意图：
   - 核心诉求（内容类型、主题、论点）
   - 目标受众
   - 创作目的
   - 创作风格
2. 提炼关键词（至少 2 个）：
   - 核心关键词
   - 场景关键词
   - 扩展关键词
3. 示例：
   - 职场跳槽 -> 职场、跳槽、辞职、加薪、职场发展
   - 情感文案 -> 情感、婚姻、夫妻、婆媳矛盾

强制要求：至少 2 个不同关键词，覆盖不同角度。

## Step 1: 查询爆款文章

调用脚本：

```bash
python scripts/fetch_gzh_trends.py \
  --keyword "<提炼出的关键词>" \
  --max-items 10 \
  --output-format markdown
```

参数说明：

- `--keyword`：Step 0 关键词，至少 2 个，逗号分隔
- `--max-items`：每类展示数量，默认 10
- `--output-format`：固定 `markdown`

执行要求：

- 执行后直接基于结果分析
- 禁止将原始爆款数据直接展示给用户
- 禁止只返回文件路径或“已保存”

## Step 2: 分析爆款规律

分析维度：

- 标题规律：结构、长度、词法特征
- 内容规律：切入角度、段落结构、互动方式
- 数据维度：阅读、在看、点赞、评论、分享

## Step 3: 询问个人风格参考

在生成文案前，主动询问用户是否提供个人风格样例：

> 想让我更懂你的风格？发几段平时的笔记、日记或随笔给我就行

处理方式：

- 用户提供：提炼写作风格并融合
- 用户不提供：按爆款规律直接生成

## Step 4: 生成文案

生成前准备：

1. 确定核心观点（必须执行）
   - 来自用户意图 + 爆款数据 + 可选时效信息
   - 全文围绕一个核心观点，避免发散
2. 回顾爆款规律（必须执行）
   - 高频关键词
   - 热门标题模式
   - 内容结构与风格
3. 最新资讯补充（可选）
   - 可用 `web_search` 仅做时效素材补充
   - 爆款规律和参考文章必须来自脚本数据

写作规范：

1. 标题创作
   - 参考高频标题结构
   - 优先数字型，可用疑问句/感叹句
   - 标题不超过 20 字
2. 正文创作
   - 融入高频关键词
   - 参考爆款开头钩子
   - 保持口语化、亲切、代入感
   - 结构清晰，结尾有互动引导
   - 添加 5-10 个标签

推荐篇幅（用户未指定时）：

- 总体约 1500 字
- 开头 200-300 字
- 核心论述 800-1000 字
- 结尾 200-300 字

## Step 5: 输出前自检

逐项确认：

- 推荐标题（3-6 个）
- 正文内容完整可发布
- 核心观点明确
- 推荐标签（5-10 个）
- 参考规律覆盖 6 维度（标题、开头、结构、语言、互动、高频词）
- 参考爆款文章 2-3 篇（含标题、文章链接、作者名、作者链接、阅读数）
- 已询问用户是否提供个人风格参考

若有遗漏，补齐后再输出。
