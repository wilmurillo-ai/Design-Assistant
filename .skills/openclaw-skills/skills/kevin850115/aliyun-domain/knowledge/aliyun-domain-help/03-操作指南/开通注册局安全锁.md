## 📄 文档内容

**注册局安全锁**（Registry Lock）是一项由域名注册局提供的顶级域名安全服务。它在顶级域名服务器层面上锁定您的域名，可防止您的核心域名被恶意转移、篡改或删除。

与阿里云平台提供的"注册商安全锁"相比，注册局安全锁提供了更高层级的安全保障。任何解锁操作都必须通过人工授权认证流程，确保每一次变更都经过域名持有人的明确许可。

| **费用** | 付费，按年购买和续费 | 免费 |
| **安全性** | 无法避免阿里云平台以外的因素对域名带来的安全隐患 | 最高级别安全保护 |

## 申请解锁与再次加锁

完成授权认证后，您可以通过预留的指令邮箱向阿里云发送邮件，申请解锁或再次加锁域名。

**指令邮箱**: safetylock@service.aliyun.com

**操作要求**：
- 必须使用授权委托书中指定的指令邮箱发送邮件
- 处理时长：**2-3 个工作日**。请提前规划域名维护操作，以免影响业务

### 申请解锁

修改域名 DNS、持有人信息等操作前，需先申请解锁。为了域名安全，建议在解锁时明确告知预计恢复锁定的时间。

**邮件示例**：
```
邮件主题：example.com 域名解锁申请

邮件正文：
解锁域名 example.com
解锁时间：20**年**月**日
恢复时间：20**年**月**日
```

> 邮件中申请的时间应考虑到 2-3 个工作日的处理周期，以免影响您的后续操作。

### 申请再次加锁

完成域名维护操作后，应立即申请重新加锁。

**邮件示例**：
```
邮件主题：example.com 域名加锁申请

邮件正文：
加锁域名 example.com
```

## 服务管理

### 续费

注册局安全锁服务到期后会自动失效，域名将解除锁定状态。为确保持续保护，请在服务到期前及时续费。

**续费条件**：续费前，请确保域名本身的有效期大于一年。

**操作步骤**：
1. 登录阿里云域名控制台
2. 在目标域名的基本信息页面，找到注册局安全锁进行续费
3. 续费后，请检查您的授权认证是否仍在有效期内。如果已过期，您需要重新提交资料进行授权认证

### 更新授权信息

在授权有效期内，如果被授权人或指令邮箱发生变更，您需要重新提交授权认证。操作流程与设置授权认证信息相同。

新的授权认证通过后，旧的授权信息将自动作废。

## 常见问题

### 为何无法续费？

注册局安全锁的最小服务周期为 1 年，且要求域名的有效期需大于安全锁服务的有效期。为安全锁续费前请确保域名的有效期大于 1 年，如果您的域名有效期小于 1 年，需先完成域名续费再进行安全锁续费。

### 续费后还需要重新进行授权认证吗？

授权认证需以您的授权认证书的有效时间为准：
- 如果您的授权认证在续费后仍在有效期内，则**不需要**重新进行授权认证
- 如果您的授权认证已过有效期，则需要重新提交资料进行授权认证

### 注册局安全锁授权认证失败原因与解决方法

| 原因 | 描述 | 解决方法 |
|------|------|----------|
| **信息不一致** | 域名持有人信息与域名信息不一致 | 域名持有人为企业：需提供与域名持有人信息一致的企业证件（如营业执照）<br>域名持有人为个人：需提供与域名持有人信息一致的个人身份证件（如身份证） |
| **资料无效** | 无域名持有人主体认证 | 域名持有人为企业：需要在提供的所有授权资料上加盖企业公章<br>域名持有人为个人：需要在提供的所有授权资料上进行本人签字确认 |
| **授权书内容不完整** | - | 请检查授权书，确保授权人、域名、授权时间、被授权人身份证、被授权人联系方式、指令邮箱等内容填写完整 |
| **资料缺失** | 未提交全部授权认证资料 | 域名持有人为企业：需提供企业的有效证件、授权认证书及被授权人的身份证复印件<br>域名持有人为个人：授权他人操作需提供域名持有人身份证复印件、授权认证书及被授权人的身份证复印件；本人操作需提供本人的身份证复印件 |

### 为什么购买了安全锁后被告知无法转移域名？

安全锁与域名及账号关联，如果其中任何一项信息发生变化，安全锁都将失效。因此注册局安全锁服务期间，**无论您的域名是否处于锁定状态，阿里云均会关闭您的域名转移功能**。

如果您仍要对域名进行转移操作，阿里云将视为您自动放弃安全锁服务，购买注册局安全锁的订单金额不予以退还。
