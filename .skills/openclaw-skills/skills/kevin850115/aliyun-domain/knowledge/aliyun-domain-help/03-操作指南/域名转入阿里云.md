## 转入前检查

请对照以下清单，检查您的域名能否转入阿里云：

| 检查项 | 说明与操作指引 |
|--------|----------------|
| **域名后缀** | 确保域名后缀（TLD）支持转入。在域名价格总览页面，确认域名后缀在"转入"列有明确标价 |
| **域名状态** | 域名必须处于 **ok** 或 **active** 状态。不得处于 clientTransferProhibited（注册商禁止转移锁）、serverTransferProhibited（注册局禁止转移锁）等限制状态。可通过 WHOIS 查询确认 |
| **注册时间** | 在 WHOIS 查询页查询域名注册时间需**满 60 天** |
| **到期时间** | 在 WHOIS 查询页查询域名距离到期日必须**大于 15 天，小于 10 年** |
| **上一次续费时间** | 域名在原注册商完成续费/赎回**不足 45 天**时不建议转入，在此期间转入可能导致您在原注册商处的续费失效，续费年限减少 |
| **续费状态** | 域名续费年限未达到上限（10 年）。如果域名续费已至最大年限（10 年），暂不支持转入 |
| **联系人信息** | 请确认域名的管理联系人信息正确，后续所有转入相关的确认邮件会发送至管理联系人的邮箱 |
| **DNSSEC** | 请在原注册商平台**禁用 DNSSEC** |
| **法律状态** | 域名未处于任何纠纷、仲裁或法院限制转出的状态 |

## 域名转入阿里云流程

> **说明**: 当域名转入阿里云注册商后，如果您依然使用原注册商的服务器进行域名解析，则无需再次备案。如果您选择使用阿里云服务器，请您重新进行备案。

### 迁移 DNS 解析

域名转入仅变更域名的注册商（即域名归属平台），而 DNS 解析负责将域名指向您的网站/邮箱服务器，两者是独立服务。

域名转入流程本身不会主动修改您的 DNS 配置，但您的原注册商在检测到域名被转出后，可能会单方面终止其捆绑提供的 DNS 解析服务。

**建议**: 为避免因域名转移导致网站和企业邮箱无法正常使用，建议您先将域名解析转入阿里云。

### 创建信息模板

信息模板是在阿里云代表域名持有者身份的档案（包含姓名、地址、邮箱等）。

在信息模板列表中创建一个和域名当前持有人相同的信息模板，作为承接域名的持有人。

### 获取域名转移密码

您需要在原域名注册商处获取**域名转移密码**（有的注册商也称为转移码、转移口令、域名密码）。

在 WHOIS 查询页，查询待转移域名的注册商，联系注册商获取域名转移密码。

**部分注册商获取转移密码的操作指导**：
- 华为云 | 腾讯云 | 火山云 | 新网 | 西部数码
- 爱名网 | 聚名网 | 美橙互联 | GoDaddy | Cloudflare
- AWS | dynadot | Porkbun

> **如原域名注册商拒绝提供域名转移密码**：
> - **国内域名**: 投诉邮箱 supervise@cnnic.cn，投诉电话 010-58813000
> - **国际域名**: 在线填写 Transfer Complaint Form 进行投诉

### 提交转移信息

1. 登录**域名转入页面**
2. 填入待转入的域名和转移密码，然后单击**立即转入**
3. 填写格式：待转移域名 + 空格 + 转移密码，每行一条转移记录
4. 一次最多支持转入 **200 个域名**
5. 选择一个信息模板，单击下一步
6. 核对订单金额并支付费用（通常为一年续费费用）

> **说明**: 转入订单支付后，系统会自动验证您输入的转移密码是否正确。转移密码验证通过后，域名开始转入阿里云。域名转入订单自付款后会有 15 天的有效期。

### 检查转入状态

根据注册局的规定，域名转入处理一般需要 **5~7 天**。

您可以通过域名转入页面查询转入状态、提交时间等详细信息。当域名转入状态显示为**转入成功**，则表示域名已成功转入阿里云。

**若域名转入失败**: 请参见域名转入失败常见问题及处理方法进行排查和处理。

### 域名转入后的后续操作

| 操作 | 说明 |
|------|------|
| **检查域名状态** | 域名成功转入后，通过 WHOIS 工具查询域名状态。如果域名被原注册商锁定为禁止更新（clientUpdateProhibited 或 clientHold）状态，需解除禁止更新锁定 |
| **域名信息补全** | 多数域名成功转入阿里云时会一并完成域名过户和实名认证，少数域名由于有注册局锁或注册局规则限制，转入后需手动操作一次域名过户来补全域名信息 |
| **".gov.cn"域名实名认证** | ".gov.cn"域名转入后需通过域名过户完成实名认证。若原单位与变更后单位一致，盖同一公章即可 |
| **".hk"域名实名认证** | ".hk"域名转入后需通过域名过户完成实名认证，过户时请选择"管理者信息"、"付费者信息"、"技术者信息"均过户，过户前后的信息可一致 |

### 取消转入

转入域名结算成功后至转移密码验证通过前，或转移密码验证通过后的 **5 天**（5*24 小时）内，您可以在域名转入列表中单击正在转入的域名，取消域名转入，转入订单将退款。

> **说明**: 如果阿里云账号存在欠票金额，则退款至账号余额；如果账号无欠票金额，则退款至原付款渠道。

## 常见问题

### 为什么域名转入失败了？

常见原因包括：
- 域名在原注册商处被锁定（状态非 ok）
- 转移密码错误或已过期
- 未满足转入前置检查的所有条件

### 为什么我支付了转入费用，但域名到期日没有延长一年？

这通常是因为您的域名在原注册商续费后 **45 天内**就进行了转入操作。根据 ICANN 政策，这会导致原续费行为被取消。

**建议**: 请在续费 45 天后再提交转入。

### 如何将 5-7 天的转入时间缩短？

提交域名转入申请并支付后，您可以登录原注册商管理后台，找到域名转移管理相关功能，**主动批准该域名的转出请求**。这样可以将等待时间从 5-7 天缩短至几分钟到一小时。

### 什么是"信息模板"，为什么必须创建它？

信息模板是您在阿里云的域名所有者身份档案，包含姓名、联系方式等信息，并需通过实名认证。转入域名时选择一个信息模板，就是指定该域名的新所有者。

### 待转入域名的持有人信息是 A，域名转入时能否选择和 A 不同的 B 持有人的信息模板？

可以选择 B 信息模板，域名也会被过户给 B。但建议您使用与原注册商基本一致的信息模板，确保转入平稳快速后，再变更域名信息，减少失败风险。

### 域名在原注册服务商没有做实名认证，但是在转入的时候选择了已实名认证模板 B，待域名转入成功后，持有人信息也会变为 B 吗？

会的。
