## 什么是域名带价 Push

**域名带价 Push** 是买卖双方点对点的交易模式。

卖家将域名转给指定的一个买家，并明确填写转让价格。当买家成功接收并支付转让金后，实现域名交易闭环。

**支付方式**: 
- 阿里云域名交易平台全面接入支付宝进行买家付款和卖家收款
- 买家通过支付宝付款成功，交易款项会直接转入卖家的支付宝账户
- 支付宝支付额度或支付宝转账、提现额度等限制，需遵循支付宝的规则

> **说明**: 因阿里云只是提供此域名交易的服务平台，所以无法对域名交易涉及的款项开具发票。

## 域名带价 Push 交易须知

### 买家须知

- 购买域名仅支持**支付宝付款**，不支持信用卡支付
- 确保支付宝已完成实名认证，并建议您绑定与阿里云账号的实名认证主体保持一致的支付宝账号
- 在接收域名 PUSH 时，可以根据支付宝内显示的选项选择支付渠道
- 成功购买域名后，**不支持退款和退回域名**

### 卖家须知

- 发布域名带价 Push 必须绑定实名支付宝
- 支持批量发布域名带价 Push，批量域名 Push 流程和单个域名 Push 流程相同
- 域名带价 Push 发布成功，有效时间为 **3 天**（3*24 小时），3 天内买家未接收的带价 Push 域名将失效
- 在买家接收带价 Push 域名前，卖家可以取消已发布的带价 Push 域名

## 域名带价 Push 交易费用

| 费用项 | 说明 |
|--------|------|
| **购买域名的费用** | 买家需支付购买域名的费用，最终支付价格为卖家 Push 域名时所带的域名价格 |
| **手续费** | 由卖家承担，费率为整笔订单成交金额的 **1%**，最低收取手续费为 0.01 元 |

> **说明**: 
> - 自 2021 年 07 月 12 日 10:00 起，带价 PUSH 交易类型将开始收取交易手续费
> - 如果有促销活动，需以促销费率为准
> - 手续费和域名买家无关，买家只需按照订单价格全额支付购买域名的费用

## 域名带价 Push 常见问题

### 域名带价 Push 交易是实时交割吗？

买家完成订单支付后，系统**实时交割**。

> **说明**: 本交易类型不支持申请发票。

### 不能发布域名带价 Push 的原因有哪些？

| 原因 | 说明 |
|------|------|
| 域名不在当前登录的阿里云账号下 | - |
| 注册时间不足 | ".cn/.中国/.公司/.网络"域名注册后 7 天内不允许发布；其他域名注册成功后 1 天内不能发布 |
| 域名已过期 | - |
| 账号间转移功能被限制 | - |
| 特殊域名后缀 | 微站域名、".gov.cn"域名、微海报".cn"域名、CNNIC 注册局保留词域名、诚信通".xin"域名或.hk、.商标等域名注册局有域名过户材料和主体限制等特殊要求 |
| 域名状态异常 | 处于过户中或转出中等状态 |
| 法律纠纷 | 非法域名，域名处于法律仲裁中或存在争议 |
| 已有交易 | 已经发起了带价 Push 或一口价域名等交易类型 |
| 国际站限制 | 带价 Push 不支持卖家将域名转给阿里云国际站用户 |

### 发布域名后哪些情况会导致域名 Push 失效？

- 域名已被删除或已转移到其他注册商（包含转移中的域名）
- 域名发生账号间转移
- 域名发生过户行为
- 域名被设置黑名单、限制账号间转移等
- 用户设置的发布时间已到期
- 域名的转移密码被获取
- 违反上游管理机构政策、法规
- 小二后台手工下架域名

### 批量发布多个 Push 域名，其中一个或多个域名发布失败怎么办？

为确保买卖双方权益，当卖家批量发布多个 Push 域名时，如果其中有一个或多个域名发布失败，会导致**所有 Push 域名发布失败**。

**处理方法**: 卖家需删除不可发布的域名，重新批量发布。

为确保买家权益，当买家批量接收并购买 Push 域名时，如果有一个域名不能完成购买，会导致所有域名 Push 失败，买家支付的金额会退还到买家支付宝账户中。

### 已接收的带价 Push 失效怎么办？

买家接收带价 Push 的域名后需要在 **2 小时内** 完成订单支付，否则该带价 Push 交易会关闭。

买家接收带价 Push 后，未支付之前，也可主动关闭 Push 订单。订单关闭后，如需交易需要卖家重新发起。
