# 知乎推广文案 — 一稿多发助手
# 适合回答的问题：「怎么高效运营多个自媒体平台？」「自媒体创作者有什么提效工具？」

---

运营 3 个以上平台的人都懂这个痛苦：同一个选题，在不同平台上要完全重写。

**问题在哪**

不同平台的内容逻辑差异非常大：
- 公众号读者习惯长文，需要大标题、分节、引导关注
- 知乎用户更在意逻辑和数据，喜欢专业口吻
- 小红书要口语化、情绪化，标题要有 emoji
- 抖音是口播，前三秒没钩子直接划走
- Twitter 是英文，每条 280 字，要能独立理解

如果每个平台都从头写，时间成本极高。如果直接复制粘贴，效果很差，平台调性完全不对。

**现在的做法**

用「一稿多发助手」这个 OpenClaw Skill。把原文（或主题）放进去，一次生成全部平台的版本。

测试了几次，每个版本确实是按平台调性重写的，不是简单缩短格式：
- 小红书版是第一人称、口语化，带话题标签
- 抖音版是口播脚本，带「[镜头切换]」「[展示画面]」这种提示
- 知乎版自动生成一个配套的问题，以专业角度回答
- Twitter Thread 是英文，每条独立成句，末条有行动号召

还支持从主题开始，不需要有现成文章，给关键词或大纲就能展开。

生成后可以继续调整：「小红书那版改得更口语一点」「公众号标题帮我想 5 个备选做 A/B 测试」。

openclaw skills install content-repurposer-cn
