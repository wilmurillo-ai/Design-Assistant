# tour-compare 标准输出流程

## 📤 标准输出结构

每次对比分析，系统会输出三部分内容：

### 1️⃣ 文字版对比总结（核心）
### 2️⃣ 可视化报告链接（HTML）
### 3️⃣ 相关问题追问（引导）

## 🎯 输出要素说明

### 1. 文字版对比总结

**必须包含**：
- ✅ 一句话结论（推荐哪个 + 理由）
- ✅ 核心差异表格（价格/评分/天数/酒店/购物）
- ✅ 关键决策因素（2-3 个）
- ✅ 性价比分析
- ✅ 场景化推荐（怎么选）

**特点**：
- 简洁明了，一眼看懂
- 突出差异，不罗列信息
- 有明确推荐和建议

---

### 2. 可视化报告链接

**两种类型**：
- **同目的地**: `tour-compare-packages.html`
- **跨目的地**: `tour-compare-destinations.html`

**报告内容**：
- 决策树流程图
- 详细对比表格
- 用户评价对比
- 费用明细对比
- 风险提示
- 最终建议

**提示语**：
```
💡 提示：点击链接在浏览器中打开完整报告
```

---

### 3. 相关问题追问

**问题类型**：

#### 基于价格差异
- "想知道价格差异主要体现在哪些方面吗？"
- "需要我详细对比预算明细吗？"

#### 基于行程
- "需要我详细对比每日行程安排吗？"
- "想了解具体景点和游玩时间吗？"

#### 基于酒店
- "想了解具体酒店信息和位置吗？"
- "需要对比酒店设施和评价吗？"

#### 基于评价
- "需要查看真实用户评价摘要吗？"
- "想知道好评/差评关键词吗？"

#### 基于政策
- "需要对比退改政策和风险提示吗？"
- "想了解取消扣费标准吗？"

#### 基于目的地
- "想知道这两个地方的最佳旅行时间吗？"
- "需要我分析签证/语言/货币等差异吗？"

#### 通用问题
- "需要我推荐其他类似商品吗？"

**输出规则**：
- 每次输出 3-4 个问题
- 优先输出与当前对比最相关的问题
- 最后加一个通用问题

## 📖 设计原则

### 1. 用户友好
- 先说结论，再说细节
- 用表格不用纯文字
- 用图标增加可读性

### 2. 信息层次
- 第一层：一句话结论
- 第二层：核心差异
- 第三层：详细分析（报告链接）

### 3. 引导深入
- 每次提供 3-4 个相关问题
- 引导用户深入了解
- 但不一次性输出太多

### 4. 行动召唤
- 提供可视化报告链接
- 告诉用户点击打开
- 增强交互体验
