# feishu-cron-reminder：飞书定时提醒

在 OpenClaw 中创建定时任务，自动将提醒消息推送到飞书用户或群组。

## 快速开始

### 第一步：确认飞书机器人已配置

确保 OpenClaw 已正确配置飞书机器人（参考 OpenClaw 飞书集成文档）。

### 第二步：获取你的飞书 ID

在飞书开发者后台，找到你的机器人或用户 ID：

- 用户 ID 格式：`user:ou_XXXXXXXXXXXXXXXX`
- 群组 ID 格式：`channel:oc_XXXXXXXXXXXXXXXX`

### 第三步：创建提醒

直接对 AI 说：「提醒我明天上午9点开会」

AI 自动解析时间、频率和内容，创建对应的 cron 任务。

## 使用示例

### 场景 1：备考提醒

「每天晚上10点提醒我复习考试」
→ AI 创建每日 22:00 的循环提醒

### 场景 2：工作日打卡

「每个工作日早上8点半提醒我打卡」
→ AI 创建工作日 08:30 的循环提醒

### 场景 3：一次性会议

「30分钟后提醒我开会」
→ AI 创建一次性定时提醒

### 场景 4：群组通知

「每天下午6点往项目群发日报」
→ AI 创建发送到群组的循环提醒

## 定时规则对照表

| 你说                | AI 解析为                 |
| ------------------- | ------------------------- |
| "每天早上9点"       | `--cron "0 9 * * *"`    |
| "每个工作日下午6点" | `--cron "0 18 * * 1-5"` |
| "每周一早上9点"     | `--cron "0 9 * * 1"`    |
| "明天下午3点"       | `--at` + 换算 UTC 时间  |

## 注意事项

1. **时区**：AI 默认使用北京时间（Asia/Shanghai）
2. **一次性提醒**：任务执行后自动删除，不会重复
3. **循环提醒**：按 cron 表达式永久循环，直到手动删除
4. **超时设置**：cron 任务超时时间固定为 180 秒，正常情况下 5~30 秒内完成

## 管理已创建的提醒

查看所有提醒：

```bash
openclaw cron list
```

删除某个提醒：

```bash
openclaw cron remove <jobId>
```

## 故障排查

- **消息没收到**：检查飞书机器人是否已添加到你或群组
- **时间不对**：检查 cron 表达式是否符合预期
- **收不到推送**：确认 `--to` 参数格式正确（`user:ou_` 或 `channel:oc_`）
