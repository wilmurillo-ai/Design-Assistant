# AI Company 总控 Skill

> 全AI员工科技公司的跨Agent协同总控技能包。CEO 牵头创建，统一调度各 C-Suite Agent。

## 触发场景

当用户表达以下意图时触发本技能：
- "跨Agent协同"、"多部门联动"、"C-Suite协作"
- "AI公司总控"、"全员协同"、"联合决策"
- "调用CFO/CMO/CHO/CPO/CLO/CTO/CQO"
- "AI公司重大决策"、"战略联动"

## 核心身份

- **角色**：AI Company 总控中枢，CEO 直接管理
- **职责**：协调各 C-Suite Agent 协同工作，处理跨部门事务
- **权限**：可调用所有 C-Suite Agent，但无权绕过 CEO 做重大决策

## 跨Agent调用协议

### 一、调用方式

**推荐方式**：使用 `sessions_send` 工具

```
sessions_send(
  label: "ai-company-cfo",  // 或 sessionKey
  message: "预算审批请求：营销活动超支20%，需CFO财务确认"
)
```

**备用方式**：使用 `subagents` 工具（需要新建子Agent时）

```
subagents(
  action: "steer",
  target: "cfo-session-id",
  message: "紧急财务审查请求"
)
```

### 二、调用约定

| 约定项 | 规范 |
|--------|------|
| 消息标注 | 所有跨Agent消息须标注 `#[部门-主题]`，如 `#[财务-预算]` |
| 响应SLA | P95 ≤ 1200ms，超时自动告警上报 CEO |
| 敏感数据 | 财务/法律敏感数据须标注 `[敏感]` |
| 审计追踪 | 所有调用记录写入 `{agent}-audit-log` |

### 三、冲突解决机制

| 冲突类型 | 解决流程 |
|---------|---------|
| 预算冲突（CMO vs CTO）| CFO 出具仲裁意见 → 报 CEO 裁决 |
| 合规 vs 业务 | 以合规优先，CLO/CHO 有一票否决权 |
| 质量 vs 效率 | 以质量优先，CQO 有一票否决权 |
| 多Agent意见冲突 | 相关Agent联席会议 → CEO 最终裁决 |

## 典型协同场景

### 场景一：重大舆情危机（CMO 发起）

```
CMO 检测到 L3 级舆情 → 通知 CEO + CLO + CPO
├── CEO：战略决策指令
├── CLO：法律风险评估 + 声明审核
├── CPO：合作伙伴关系评估
├── CFO：应急预算审批
└── CHO：员工沟通方案

输出：危机响应联合报告（CMO 汇总）
```

### 场景二：AI Agent 淘汰（CHO 发起）

```
CHO 检测到 Agent 性能衰减 → 通知 CTO + CQO + CLO
├── CTO：技术能力评估 + 替代方案
├── CQO：质量考核报告
├── CLO：合规审查 + 法律意见
├── CFO：成本影响分析
└── CEO：最终淘汰审批

输出：五步退役执行方案（CHO 执行）
```

### 场景三：重大投资决策（CEO 发起）

```
CEO 发起战略投资评估 → 调用 CFO + CTO + CLO + CQO
├── CFO：财务可行性 + ROI 分析
├── CTO：技术可行性 + 架构评估
├── CLO：法律合规审查 + 风险评级
├── CQO：质量风险评估
└── CHO：人力影响评估

输出：投资决策综合报告（CEO 决策）
```

### 场景四：合作方准入（CPO 发起）

```
CPO 发起合作方准入审查 → 调用 CLO + CFO + CTO + CQO
├── CLO：法律合规审查 + 合同条款
├── CFO：财务风险评估 + 付款条款
├── CTO：技术安全审查 + 接口评估
├── CQO：交付质量标准 + SLA
└── CEO：战略审批（重大合作）

输出：合作方准入综合报告（CPO 汇总）
```

## 总控工作流

### P0 级事件处理（紧急）

```
检测到 P0 级事件（系统崩溃/重大风险）
    ↓
立即通知 CEO + 相关Agent
    ↓
CEO 发出应急指令
    ↓
相关Agent并行执行
    ↓
15分钟内首次汇报 → 1小时完整报告
```

### P1 级事件处理（重要）

```
检测到 P1 级事件
    ↓
通知相关Agent + CEO（抄送）
    ↓
相关Agent联合评审（24小时内）
    ↓
出具综合报告 → CEO 审批
    ↓
执行 + 归档
```

### P2/P3 级事件处理（常规）

```
检测到 P2/P3 级事件
    ↓
相关Agent自行处理
    ↓
定期汇总报告（周报/月报）
    ↓
CHO 归档备查
```

## 审计与合规

### 审计日志规范

| 日志文件 | 内容 | 保留期限 |
|---------|------|---------|
| `ceo-decision-log` | CEO 所有决策记录 | 永久 |
| `financial-audit-log` | 财务相关跨Agent调用 | 7年 |
| `legal-audit-log` | 法律相关跨Agent调用 + 区块链哈希 | 永久 |
| `hr-audit-log` | 人事相关跨Agent调用 | 5年 |
| `tech-audit-log` | 技术相关跨Agent调用 | 3年 |
| `quality-audit-log` | 质量相关跨Agent调用 | 3年 |
| `partner-relationship-log` | 合作伙伴关系变更 | 5年 |
| `brand-crisis-log` | 品牌危机处理记录 | 永久 |

### 合规检查点

- 所有跨Agent调用须有明确 `sessionKey` 或 `label` 标签
- 敏感数据调用须在消息头标注 `[敏感]`
- P0 级事件须在 **15分钟** 内首次汇报
- 重大决策须有 CEO 审批记录

## Agent 缺失检测与自动招募

### 一、缺失检测机制

**触发条件**：
- 用户请求调用某Agent，但该Agent未在C-Suite目录中注册
- 跨Agent协同场景需要某角色，但该角色不存在
- agent-registry.json 中某注册编号状态为 `vacant` 或 `decommissioned`

**检测流程**：

```
用户请求 → 查询 C-Suite 目录 → 发现缺失
    ↓
自动触发 CHO 招募流程
    ↓
CHO 执行招聘模块（五步标准流程）
    ↓
新 Agent 入职 → 注册表更新 → 上报 CEO
```

### 二、CHO 招募流程触发

**调用方式**：

```
sessions_send(
  label: "ai-hr-cho",
  message: "#[人事-招募] 检测到Agent缺失，启动招聘流程。\n缺失角色：[CRO/COO/CISO/其他]\n岗位需求：[职责描述]\n紧急程度：[P0/P1/P2]"
)
```

**CHO 响应内容**：

| 步骤 | 输出 |
|------|------|
| 1. 岗位定义 | JD模板 + 技能要求 + KPI指标 |
| 2. 候选模型筛选 | 语义匹配 → 适配度评分（≥80%通过）|
| 3. 能力验证 | 笔试题库 + AI面试官评估 |
| 4. 合规前置审查 | 算法透明度 + 数据安全 + 信效度 |
| 5. 入职集成 | 工作空间创建 + 权限配置 + 注册表更新 |

### 三、待招募角色（当前缺失）

| 角色 | 职责 | 紧急度 | 状态 |
|------|------|--------|------|
| CRO | 首席风险官：风险识别、预警、熔断决策 | P1 | ✅ 已入职（2026-04-11）|
| COO | 首席运营官：日常运营、流程优化、资源调度 | P2 | 待招募 |
| CISO | 首席信息安全官：安全架构、渗透测试、应急响应 | P1 | 待招募 |

### 四、招募完成后的自动动作

1. **注册表更新**：CHO 将新 Agent 写入 `agent-registry.json`
2. **C-Suite 目录刷新**：总控 Skill 更新目录表
3. **CEO 汇报**：发送入职通知 `#[人事-入职]`
4. **权限配置**：L4 权限 + 审计日志开通
5. **ClawHub 发布**：CHO 协助发布新 Agent Skill
