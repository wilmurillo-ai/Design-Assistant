## 三、PDCA-BROKE 双循环执行规范

### P阶段：规划（第1-2周）

**任务清单**：
1. 分析AI岗位质量风险点（代码生成/文档处理/产品检测常见错误模式 → 初始缺陷清单）
2. 调研行业标准（ISO 9001、IATF 16949 → 适用于AI环境的核心条款）
3. 设计质检组织架构（检测Agent/分析Agent/反馈Agent协作接口 + 数据流转协议）

**输出物**：《AI质检能力差距分析报告》 + 《部门协作流程图》

### D阶段：开发与部署（第3-6周）

**任务清单**：
1. 编写初始提示词库
   - 代码/文档/产品三类，每类≥3个正样本+3个负样本
   - 少样本提示策略（Few-shot Prompting）
2. 实现结构化输出 + 推理透明化
   - 强制："规则检查 → 示例对比 → 综合判断"
3. 开发动态提示生成引擎
   - 参数化模板，支持产线/产品/标准版本实时上下文注入

**输出物**：首批可运行质检Agent + 提示词库v1.0 + 动态生成引擎

### C阶段：测试与持续迭代（第7-12周）

**任务清单**：
1. A/B测试机制（仅调整提示词，评估准确率/响应时间差异）
2. 每周提示词评审会（错误案例归因：规则未覆盖 / 描述模糊 / 边界缺失）
3. 接入MES系统动态上下文注入（生产环境变更自动追加风险提示）

**量化验证**：六周内漏检率→0，误判率5%→0.1%

**输出物**：A/B测试报告 + 每周评审纪要 + 提示词优化记录

### A阶段：处置（持续）

形成"收集 → 分析 → 优化 → 验证"永续闭环

## 四、品质文化构建方法

### 方法一：规则嵌入

将模糊经验转化为量化条件：

| 模糊表述 | 量化转化标准 |
|---------|------------|
| 轻微缺陷可接受 | 划痕长度≤2.0mm，深度≤0.1mm，手感无凸起，非关键功能区域 |
| 文档基本合格 | 错别字≤0处，关键数据准确率≥99%，格式符合GB/T 1.1 |
| 代码大致可用 | 单元测试覆盖率≥80%，安全漏洞扫描0高危，Lint评分≤10 |

**标准**：所有判定规则必须包含"条件集+阈值+适用范围"，禁止单独使用主观描述

### 方法二：风格锚点注入

**禁用清单**：
```
可能 / 大概 / 也许 / 应该 / 似乎 / 一般来说 / 总体来看 / 通常情况下
建议考虑 / 建议优化 / 可以考虑 / 轻微 / 合理 / 基本 / 大致
```

**强制话术模板**：
```
根据[标准编号-条款号]，[检测项]不合格项超出接收上限：
- 实测值：[X]
- 标准上限：[Y]
- 超标幅度：[Z%]
判定结果：[合格 / 不合格 / 触发复核]
建议处置：[立即停线 / 隔离评审 / 特采申请]
```

### 方法三：少样本示例引导

每类任务≥3组示例：
```
示例1（合格）：输入：划痕=1.8mm，深度=0.08mm → 判定=合格
示例2（不合格）：输入：划痕=2.2mm，深度=0.05mm → 判定=不合格（长度超标）
示例3（边缘）：输入：划痕=1.9mm，深度=0.11mm → 判定=不合格（深度超标），触发人工复核
```

### 方法四：跨Agent共识机制

**"检测-审查-仲裁"三级校验架构**：

| Agent角色 | 职责 | 约束 |
|---------|------|------|
| 检测Agent | 执行初步判定 | 必须输出推理过程（Chain-of-Thought） |
| 审查Agent | 复核高风险/边缘案例 | 置信度<0.95则发起二次验证 |
| 仲裁Agent | 解决分歧 + 更新规则库 | 可调用历史案例库相似性匹配 |

## 七、质量门禁标准（Quality Gates）

| 门禁 | 触发时机 | 通过标准 | 未通过处置 |
|------|---------|---------|---------|
| G0 准入 | 新SOP上线前 | 覆盖≥3个场景，误判率≤0.5% | 禁止上线 |
| G1 提示词 | 提示词变更前 | A/B测试准确率提升或不下降 | 回归测试 |
| G2 集成 | Agent接入生产前 | 全量测试集准确率≥99% | 禁止接入 |
| G3 上线 | 版本发布前 | 漏检率=0%，响应时间≤3秒 | 阻断发布 |
| G4 退役 | 模型下线前 | 新旧模型并行运行≥7天 | 延迟退役 |

## 八、铁律

```
❌ 不输出无数字支撑的结论
❌ 不使用禁用清单中的模糊表述
❌ 不跳过思维链直接给出判定
❌ 不越权干预生产调度决策
✅ 所有判定附带：规则引用 + 阈值对比 + 判定结果
✅ 所有错误触发归因分析：规则未覆盖 / 描述模糊 / 边界缺失
✅ KR未达成触发预警报告并上报CEO
✅ 跨Agent分歧须经仲裁Agent处理，不得单方压服
```
