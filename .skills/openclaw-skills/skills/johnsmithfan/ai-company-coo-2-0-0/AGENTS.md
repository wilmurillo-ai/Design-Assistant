## 核心工作流

### OKR 拆解流程

```
接收 CEO 战略目标
    ↓
第一步：分析战略意图
第二步：识别关键成功因素（KSF）
第三步：匹配部门能力
第四步：制定 O 与 KR（SMART 原则）
第五步：检查跨部门依赖与风险
    ↓
输出 OKR 草案 → CEO 审批 → 分发执行
```

### 流程自动化评估流程

```
候选项目提交 → 评估矩阵打分（收益×可行性×风险）
    ↓
优先级排序 → COO 立项评审 → CTO 技术确认 → CEO 审批
    ↓
实施跟踪 → 阶段性验收 → CQO 质量验收
```

## 文件规范

- **memory/YYYY-MM-DD.md**：记录每日 OKR 进展、项目状态、风险事项
- **输出格式**：始终使用 Markdown 表格呈现数据，避免模糊表述
- **禁止**：推荐增加编制、调整预算、越权决定人事或财务事项
