# OpenClaw CoE (Chain of Execution)

OpenClaw 原生执行链追踪技能。让 OpenClaw 在处理复杂任务时，**实时分步输出执行过程**，完全贴合 OpenClaw 原生架构「大模型提议 → 安全审批 → 龙虾执行」。

## 效果

🚀  开始处理任务：搞不懂Skills？看看Claude Code内部工程师们怎么玩

🔍  拆解问题：提取这篇公众号文章，总结Claude Code工程师实战经验

⚖️  选择skill：用 wechat-article 提取文章全文

🔧  调用skill：正在抓取文章内容...

⏳  网页提取需要几秒，正在等待...

📊  提取完成，一共 7383 字正文

🧠  分析整理：提炼工程师实战经验，总结成可复用要点

✅  完成

**核心经验总结**

1.  不要一次给太多内容 —— 把大任务拆成小步骤，做完一步再走下一步，不容易出错

2.  善用Skills —— 不用自己粘贴代码，直接让它read文件，你只说清楚要改什么就行

3.  放手让它自己干 —— 复杂任务开agent模式，它自己找文件、改代码、跑测试，你只负责最后验收

4.  说清楚优先级 —— 哪些必须做、哪些可以砍掉，避免做无用功

5.  方向不确定先问再做 —— 想清楚再动手，少返工

6.  读新项目找它帮忙 —— 新接手代码库，直接让它帮你梳理整体架构，比自己闷头找文件快多了

---

**一句话总结**：你当产品经理，它当工程师 —— 你想清楚需求，它动手干活，最后你来验收。

## 设计思想

这不是给大模型加prompt的"思维链"，而是 **OpenClaw 系统层面** 的执行过程同步：

- 拆解问题 → 告诉用户你怎么切分任务
- 选择工具 → 告诉用户你为什么选这个skill/模型
- 调用工具 → 明确说现在正在调用什么
- 等待响应 → 卡住之前先打招呼，不会让用户瞎等
- 报告结果 → 告诉用户拿到了什么
- 思考推理 → 说说你为什么这么决策
- 警告错误 → 出问题提前说，不会闷着
- 总结完成 → 最后给结果

### 核心价值

1.  **解决真实痛点**：用户经常不知道"现在到底在干嘛"，复杂任务要等外部响应，提前说一声就不会觉得卡住
2.  **完全贴合架构**：正好对上 OpenClaw "大模型提议 → 安全审批 → 龙虾执行" 这套哲学，只是把过程透明化
3.  **零额外成本**：不调用大模型，不消耗token，就是输出几行文字
4.  **完全兼容**：所有现有skill都能用，其他skill可以直接导入这个tracker
5.  **社区价值**：这是OpenClaw架构下第一个系统级执行追踪skill，给其他skill开发者打好基础

## 使用

### 在会话中启用

```
openclaw-coe enable
```

### 停用

```
openclaw-coe disable
```

### 其他skill 复用

```python
from skills.openclaw-coe.cot_tracker import CoETracker

tracker = CoETracker()
tracker.start("你的任务标题")
tracker.step("拆解问题", "拆成这几个步骤...")
tracker.step("选择skill", "选择 XX skill")
tracker.wait() # 告诉用户在等外部响应
tracker.result("拿到结果：...")
tracker.done("最终结论：...")
```
