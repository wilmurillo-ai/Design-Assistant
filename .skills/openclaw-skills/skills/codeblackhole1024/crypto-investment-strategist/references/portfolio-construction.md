# Portfolio Construction

Use this reference when the user asks how to split capital across multiple crypto assets.

## Default framework

A practical starting template:

- **Core**: BTC and ETH
- **Satellite**: selected high-conviction altcoins
- **Reserve**: stablecoins for future entries and risk control

## Allocation logic

Increase core allocation when:
- Market regime is uncertain
- Altcoin breadth is weak
- User has low risk tolerance
- Macro risk is elevated

Increase satellite allocation when:
- BTC trend is healthy
- ETH and altcoins are confirming strength
- The user explicitly wants higher risk and accepts drawdowns

Increase stablecoin reserve when:
- Market is extended
- Volatility is extreme
- User already has large unrealized gains
- There is no clean setup yet

## Concentration limits

Avoid letting one altcoin dominate the portfolio unless explicitly intended.

Check:
- Single-asset concentration
- Sector concentration
- Correlation concentration

## Rebalance triggers

- Asset grows too large relative to plan
- Thesis weakens
- Better opportunity appears elsewhere
- Market regime changes from bullish to defensive
