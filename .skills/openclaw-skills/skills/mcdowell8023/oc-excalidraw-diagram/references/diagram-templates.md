# 图表模板

## 通用约定

- 坐标起点：x=100, y=100
- 水平间距：250px（节点中心到中心）
- 垂直间距：120px（节点中心到中心）
- 箭头 gap：8px（从节点边缘出发）

---

## 1. 流程图（Flowchart）

**结构：** 开始(椭圆) → 步骤(矩形) → 判断(菱形) → 步骤(矩形) → 结束(椭圆)

**布局：** 竖向，从上到下

```
椭圆[开始]
  ↓ arrow
矩形[步骤1]
  ↓ arrow
菱形[判断?]
  ↓(是) arrow    →(否) arrow
矩形[步骤2]      矩形[步骤3]
  ↓ arrow
椭圆[结束]
```

**坐标模板（单列流程）：**
- 开始椭圆：x=200, y=100, w=140, h=60
- 步骤1矩形：x=200, y=220, w=160, h=60
- 判断菱形：x=200, y=340, w=160, h=80
- 步骤2矩形：x=200, y=480, w=160, h=60
- 结束椭圆：x=200, y=600, w=140, h=60

**颜色：**
- 开始/结束：backgroundColor: `#b2f2bb`（绿色）
- 步骤：backgroundColor: `#a5d8ff`（蓝色）
- 判断：backgroundColor: `#ffd43b`（黄色）

---

## 2. 架构图（Architecture Diagram）

**结构：** 分层矩形，用箭头表示调用/依赖关系

**布局：** 横向分层，每层节点横排

```
[前端层]  [前端层]
    ↓         ↓
   [API 网关]
    ↓         ↓
[服务A]    [服务B]
    ↓         ↓
   [数据库层]
```

**颜色规则：**
- 前端层：`#a5d8ff`
- 服务层：`#b2f2bb`
- 数据层：`#ffc9c9`
- 网关/代理：`#ffd43b`

---

## 3. 思维导图（Mind Map）

**结构：** 中心主题 + 放射状分支

**布局：** 中心节点居中，分支均匀分布四周

```
        [分支1]
           ↑
[分支4] ← [主题] → [分支2]
           ↓
        [分支3]
```

**坐标参考（4分支）：**
- 主题椭圆：x=400, y=300, w=200, h=80
- 上方分支：x=400, y=140, w=160, h=60
- 右方分支：x=650, y=300, w=160, h=60
- 下方分支：x=400, y=460, w=160, h=60
- 左方分支：x=140, y=300, w=160, h=60

**颜色：**
- 主题：`#ffd43b`（黄色，突出）
- 分支：交替使用 `#a5d8ff` 和 `#b2f2bb`

---

## 4. ER 图（Entity-Relationship）

**结构：** 实体(矩形) + 属性(文本) + 关系(箭头+标注)

**布局：** 网格排列，实体间用带标签箭头连接

**元素规则：**
- 实体：rectangle，宽200, 高包含字段（每字段约25px）
- 主键字段：加粗或用 `---` 分隔
- 关系标注：在箭头旁加 text 元素（1:N, N:M 等）

**表格样式（使用文本行模拟）：**
```
矩形背景 #a5d8ff
┌─────────────┐
│ User（表名）  │  ← 矩形，backgroundColor: #a5d8ff
├─────────────┤
│ id PK        │  ← 每行用独立 text 元素
│ name         │
│ email        │
└─────────────┘
```

---

## 5. 时序图（Sequence Diagram）

**结构：** 参与者横排，时间轴纵向，消息用横向箭头

**布局：**
```
[A]      [B]      [C]
 |        |        |
 |--msg1->|        |
 |        |--msg2->|
 |        |<-resp--|
 |<-resp--|        |
```

**坐标规则：**
- 参与者矩形：y=100, 宽120, 高60，水平间距250
- 生命线（虚线）：每个参与者下方画竖线（用 line 或多个短 text 代替）
- 消息箭头：水平 arrow，y坐标递增（+60px/消息）

---

## 6. 泳道图（Swimlane）

**结构：** 横向泳道 + 竖向流程

**布局：**
```
┌────────────────────────────────┐
│ 角色A  │ [步骤1] → [步骤2]      │
├────────────────────────────────┤
│ 角色B  │          [步骤3] → [步骤4] │
└────────────────────────────────┘
```

**实现方式：**
- 泳道背景：大矩形，backgroundColor: `#f1f3f5`（浅灰），无描边
- 泳道标题：左侧矩形，backgroundColor: `#a5d8ff`
- 步骤节点：普通矩形，按角色归属放入对应泳道行

---

## 7. 类图（Class Diagram）

**结构：** 类(矩形，三段式) + 继承/组合箭头

**布局：** 层次排列，父类在上，子类在下

**类矩形格式：**
```
┌──────────────┐
│  ClassName   │  ← 类名，backgroundColor: #a5d8ff
├──────────────┤
│ -field1: str │  ← 属性（text）
├──────────────┤
│ +method(): void │ ← 方法（text）
└──────────────┘
```

**箭头类型：**
- 继承：endArrowhead: `"triangle"`
- 实现接口：strokeStyle: `"dashed"`, endArrowhead: `"triangle"`
- 组合：endArrowhead: `"diamond"`
- 关联：endArrowhead: `"arrow"`

---

## 8. 数据流图（DFD）

**结构：** 外部实体(矩形) + 处理过程(圆角矩形) + 数据存储(双横线矩形) + 数据流(箭头)

**颜色：**
- 外部实体：`#ffc9c9`
- 处理过程：`#a5d8ff`
- 数据存储：`#b2f2bb`

**布局：** 自由布局，注意数据流方向清晰

---

## 9. 关系图（Relationship）

**结构：** 节点(圆/矩形) + 关系(有向/无向箭头) + 标签

**布局：** 力导向自由布局，保证不重叠

**元素要点：**
- 节点：ellipse，backgroundColor交替使用
- 关系标签：在箭头中点放 text 元素（可用 `containerId` 绑定到 arrow）
- 双向关系：两根箭头或 startArrowhead + endArrowhead 同时设置
