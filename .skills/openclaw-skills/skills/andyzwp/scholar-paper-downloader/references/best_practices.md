# 学术文献下载最佳实践

## 概述

本文档提供学术文献下载的最佳实践,包括速率限制、版权合规、性能优化等方面的建议。

## 速率限制与礼貌性

### 为什么需要速率限制?

1. **保护服务器**: 避免对学术数据库造成过大负载
2. **避免封禁**: 频繁请求可能导致IP被封
3. **维护学术社区**: 保持资源的可持续访问

### 推荐的请求频率

| 数据源 | 推荐频率 | 说明 |
|--------|----------|------|
| arXiv | 1-3秒/请求 | 相对宽松 |
| Semantic Scholar | 1-2秒/请求 | 有API限制 |
| Google Scholar | 5-10秒/请求 | 通过SerpAPI访问 |
| 其他来源 | 3-5秒/请求 | 保守估计 |

### 实现延迟

```python
import time;

for paper in papers:
    # 处理论文
    process(paper);
    
    # 延迟
    time.sleep(2);  # 2秒延迟
```

## 版权与合规

### 合法使用范围

1. **预印本 (arXiv等)**: 通常可以自由下载和使用
2. **开放获取论文**: 根据开放获取许可使用
3. **机构订阅**: 通过机构网络访问时可用于个人研究

### 禁止行为

1. **商业获利**: 不得将论文用于商业目的
2. **大规模爬取**: 避免一次性下载大量文献
3. **绕过付费墙**: 不得通过技术手段绕过付费验证
4. **侵犯版权**: 尊重作者的版权声明

### 正确引用

下载论文后,请务必:

1. 保留原始元数据信息
2. 在使用时正确引用作者和发表信息
3. 遵守论文的许可协议

## 付费文献处理

### 多源fallback策略

技能实现了智能的付费文献处理策略:

1. **优先免费来源**: 首先尝试arXiv(完全免费)
2. **备选来源**: 尝试Semantic Scholar等开放获取来源
3. **标记手动获取**: 无法下载时,标记为需要手动获取

### 处理流程

```
搜索论文
    ↓
arXiv有免费版本? → 是 → 下载
    ↓否
Semantic Scholar有开放获取? → 是 → 下载
    ↓否
标记为"需要手动获取"
```

## 性能优化

### 并发下载

```python
from concurrent.futures import ThreadPoolExecutor;

with ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(download, paper) for paper in papers];
    results = [f.result() for f in futures];
```

### 断点续传

技能支持断点续传:

1. 检查已存在的文件,避免重复下载
2. 记录下载日志,便于恢复
3. 分块下载,支持中断后继续

### 资源管理

1. **限制并发数**: 默认3个并发连接
2. **设置超时**: 默认30秒超时
3. **重试机制**: 失败时自动重试3次

## 数据组织

### 文件命名规范

推荐的命名格式:

- `{first_author}_{year}_{title_abbr}.pdf`
- 例如: `Hinton_2018_Deep_Learning_Review.pdf`

### 目录结构

```
papers/
├── arxiv/
│   ├── 2103.00001.pdf
│   └── 2103.00002.pdf
├── semantic_scholar/
│   └── paper-id.pdf
├── papers_index.md
└── papers_index.json
```

### 元数据存储

1. **Markdown索引**: 便于阅读和分享
2. **JSON索引**: 便于程序处理
3. **PDF内嵌元数据**: 使用PyMuPDF提取

## 故障排除

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 下载超时 | 网络不稳定 | 增加超时时间 |
| 403错误 | 需要认证 | 尝试其他来源 |
| 404错误 | 论文不存在 | 检查论文ID |
| 解析错误 | 元数据格式变化 | 更新解析逻辑 |

### 日志使用

技能会生成详细的日志:

```bash
# 查看日志
python batch_downloader.py -q "query" --log-level DEBUG;
```

## 维护建议

1. **定期更新依赖**: 保持库版本最新
2. **监控API变化**: 学术数据库可能随时更新API
3. **备份索引**: 定期备份生成的索引文件
4. **清理失败文件**: 定期清理损坏或失败的下载
