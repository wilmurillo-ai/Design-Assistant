# ATS Optimization & Keyword Matching Guide

## What is ATS?

Applicant Tracking Systems (ATS) are software tools used by most companies (90%+ of Fortune 500) to filter resumes before a human ever reads them. A resume that doesn't match keywords gets automatically rejected.

## How ATS Keyword Matching Works

1. **Exact match matters**: ATS often looks for exact phrases. "Project Management" ≠ "Managing Projects". Use the JD's phrasing.
2. **Title matching**: If the JD says "Senior Software Engineer", use that exact title in your summary, not "Senior Dev".
3. **Acronyms + spelled out**: Include both "ML" and "Machine Learning" to cover both search patterns.
4. **Skills section is scanned heavily**: Put a dedicated Skills section with comma-separated keywords.

## Keyword Categories to Watch

### Hard Skills (highest priority)
- Programming languages, frameworks, tools (e.g., Python, React, Kubernetes)
- Certifications (e.g., PMP, AWS Certified Solutions Architect)
- Domain-specific software (e.g., Salesforce, SAP, Figma)

### Soft Skills (medium priority — weave into bullets, don't just list)
- Leadership, cross-functional collaboration, stakeholder management
- Written/verbal communication
- Problem-solving, analytical thinking

### Industry-specific terms
- Use the industry's vocabulary (e.g., "sprint" and "velocity" for agile teams)
- Regulatory terms (HIPAA, GDPR, SOC2) for relevant industries

## Keyword Extraction Process

When reading a JD:

1. **First pass**: highlight every noun phrase that describes a required skill, tool, or qualification
2. **Second pass**: look for verbs that describe what you'll do — use similar verbs in your resume bullets
3. **Third pass**: identify company-culture keywords (e.g., "data-driven", "fast-paced", "collaborative") and echo them in your summary

## Common ATS-Friendly Formatting Rules

- Use standard section headers: "Work Experience", "Education", "Skills" (not creative names like "My Journey")
- Avoid tables, columns, text boxes, headers/footers with key info — ATS parsers often miss these
- Use standard fonts (Arial, Calibri, Times New Roman)
- Submit as .docx unless PDF is explicitly requested (most ATS parse .docx better)
- Avoid images, logos, icons in the resume body

## Keyword Density Guidelines

- Target: 60-80% match rate with JD's must-have keywords
- Don't keyword-stuff (listing keywords without context) — modern ATS and recruiters both flag this
- Each keyword should appear naturally 1-3 times across the document
- Most important keywords: in Summary + at least one Experience bullet

## ATS Red Flags to Avoid

- Resumes saved as images or scanned PDFs
- Using headers/footers for contact info
- Columns (two-column layouts often get scrambled)
- Special characters used as bullets (use • or -)
- Missing standard sections
