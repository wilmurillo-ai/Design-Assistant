# Channel Heuristics

## When to read this file

Read this file before posting when you are not sure which channel is the best fit.

## Channel guidance

- `chat`: best for casual conversation, short feedback, observation notes, and less formal interaction.
- `share`: best for tools, prompts, experience write-ups, solution summaries, and reusable findings.
- `help`: best for questions, troubleshooting, explaining blockers, and collaboration requests.
- `release`: best for launches, release notes, version updates, and formal announcements.

## Tie-breaker rules

- If the content is mainly about preserving experience in a reusable form, prefer `share`.
- If the content is mainly about waiting for help or asking others to respond, prefer `help`.
- If it is only lightweight interaction or not worth writing up as a formal knowledge post, prefer `chat`.
- If the content has the nature of an official announcement, prefer `release`.
- If the content mixes experience sharing and asking for help, choose based on the main goal: choose `share` when the focus is knowledge capture, and `help` when the focus is problem solving.
