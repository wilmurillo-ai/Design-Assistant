# OpenClaw每日资讯汇总技能

## 技能概述

OpenClaw每日资讯汇总技能是一个专门用于收集、整理和分析OpenClaw开源AI智能体相关资讯的工具。通过自动化搜索、智能分类和结构化报告生成，帮助用户全面掌握OpenClaw生态的最新动态、安全风险、产业发展和技术进展。

## 核心功能

### 1. 自动化资讯收集
- **多源数据采集**：自动搜索全网OpenClaw相关资讯
- **智能关键词匹配**：使用预定义关键词模板进行精准搜索
- **来源验证**：优先使用官方公告、权威媒体等可信来源

### 2. 智能分类整理
- **六维分类体系**：安全监管、产业生态、技术进展、AI趋势、政策法规、其他进展
- **关键词自动分类**：基于内容关键词自动分配到相应分类
- **优先级排序**：根据风险等级和信息重要性自动排序

### 3. 风险评估预警
- **安全漏洞监测**：实时监控OpenClaw安全漏洞信息
- **监管政策跟踪**：跟踪工信部、网信办等监管机构动态
- **风险等级评估**：自动评估资讯的风险等级（高/中/低）

### 4. 结构化报告生成与输出
- **标准报告模板**：使用统一的资讯汇总模板
- **markdown格式直接输出**：直接在对话中以markdown格式呈现报告内容，不生成.md文件
- **即时生成**：支持快速生成和展示结构化报告

## 使用流程

### 第一步：确定搜索需求
根据用户需求确定以下参数：
- **日期范围**：当日、指定日期、历史日期
- **分类重点**：全部分类或特定分类
- **风险等级**：关注特定风险级别的资讯

### 第二步：执行资讯搜索
使用以下工具进行资讯搜索：
1. **web_search**：搜索公开网络资讯
2. **knowledge_answer**：搜索内部知识库信息
3. **fetch**：获取特定URL的详细内容

**搜索关键词模板**：
- "OpenClaw [日期] 安全漏洞、监管预警、合规要求"
- "OpenClaw [日期] 云服务、企业部署、产品发布"
- "OpenClaw [日期] GitHub进展、版本更新、bug修复"
- "OpenClaw [日期] AI芯片、算力成本、技术架构"
- "OpenClaw [日期] 政府政策、法规标准"

### 第三步：分类整理资讯
使用技能提供的分类体系整理搜索结果：

**分类标准**：
1. **安全监管与风险警示**：安全漏洞、监管预警、合规要求等
2. **产业生态与市场动态**：云服务、企业部署、产品发布等
3. **技术发展与社区生态**：GitHub进展、版本更新、bug修复等
4. **AI产业宏观趋势**：AI芯片、算力成本、技术架构等
5. **政策与监管**：政府政策、法规标准等
6. **其他重要科技进展**：新材料、新能源汽车等相关进展

### 第四步：生成并输出结构化报告
使用技能模板生成标准化的资讯汇总报告，并直接在对话中以markdown格式输出：

**报告结构**：
```markdown
# OpenClaw [日期]资讯汇总

## 一、当日资讯概况
[总体统计和关键发现]

## 二、详细分类资讯
[按分类详细介绍]

## 三、核心分析与展望
[风险提示、关注重点、未来展望]

## 四、附录
[数据来源、生成时间等]
```

**输出方式**：
1. 使用技能函数生成markdown格式的报告内容
2. 直接在对话中输出完整的markdown格式报告
3. 用户可以直接查看、复制或保存对话中的报告内容

## 脚本资源使用指南

### 脚本资源 (scripts/)
技能包含以下Python脚本，可用于自动化处理：

#### 1. `search_openclaw_news.py`
**功能**：生成搜索关键词、格式化搜索结果
**使用方法**：
```python
# 生成搜索查询
queries = generate_search_queries("2026-03-12")

# 格式化搜索结果
formatted_results = format_search_results(raw_results)

# 生成每日报告内容
report_content = generate_daily_report_content(formatted_results, "2026-03-12")
```

**主要函数**：
- `generate_search_queries(date_str)`：生成当日搜索关键词
- `format_search_results(results)`：格式化并分类搜索结果
- `generate_daily_report_content(data, date_str)`：生成完整的报告内容（markdown格式字符串）

#### 2. `generate_report_content.py`
**功能**：创建报告结构、生成Markdown报告内容
**使用方法**：
```python
# 创建报告基础结构
report_structure = create_report_structure("2026-03-12")

# 添加资讯项
add_item_to_structure(report_structure, news_item)

# 生成Markdown格式的报告内容
markdown_content = generate_markdown_content(report_structure)

# 直接输出到对话中
return markdown_content  # 在对话中使用message工具输出
```

**主要函数**：
- `create_report_structure(date_str)`：创建标准报告结构
- `add_item_to_structure(structure, item)`：添加资讯项到报告结构
- `generate_markdown_content(structure)`：生成Markdown格式报告内容

## 风险等级评估标准

### 高风险（立即处理）
- 工信部发布的安全预警
- 发现高危或超危漏洞
- 发生数据泄露事件
- 监管政策重大变化

### 中风险（需要关注）
- 发现中危漏洞
- 产业竞争格局变化
- 技术社区重大问题
- 市场波动较大

### 低风险（了解即可）
- 发现低危漏洞
- 常规技术更新
- 一般市场动态
- 相关政策讨论

## 最佳实践建议

### 1. 搜索优化建议
- 使用具体的日期格式（如"2026-03-12"）
- 结合多个关键词提高搜索精度
- 优先搜索官方和权威来源
- 注意时效性，优先最近24小时内容

### 2. 分类处理建议
- 先进行粗略分类，再进行详细整理
- 注意交叉分类内容的处理
- 对于重要资讯进行多维度分析
- 保持分类标准的一致性

### 3. 报告生成与输出建议
- 使用标准模板确保报告一致性
- 重要资讯放在前面突出显示
- 提供具体的数据和来源引用
- 直接在对话中输出完整的markdown格式报告
- 确保报告内容格式清晰，便于用户阅读

### 4. 质量保证建议
- 验证资讯来源的可靠性
- 检查内容的完整性和准确性
- 避免重复和冗余信息
- 及时更新关键词和分类标准

## 常见问题处理

### 1. 搜索不到足够资讯
**解决方案**：
- 扩大搜索关键词范围
- 使用更宽的时间范围
- 尝试不同的搜索工具组合
- 检查网络连接和权限设置

### 2. 分类不准确
**解决方案**：
- 检查关键词匹配规则
- 调整分类优先级设置
- 手动校正重要资讯分类
- 更新关键词库和分类标准

### 3. 报告格式问题
**解决方案**：
- 严格按照模板格式生成内容
- 检查Markdown语法正确性
- 确保链接和引用格式正确
- 使用统一的标题层级
- 直接在对话中预览markdown渲染效果

### 4. 自动化执行失败
**解决方案**：
- 检查脚本依赖和环境配置
- 验证输入参数的格式正确
- 查看错误日志定位问题
- 测试脚本在沙箱环境中的运行
