# OpenClaw每日资讯汇总技能使用示例

## 基本使用场景

### 示例1：生成当日资讯汇总报告
**用户请求**："帮我生成今天的OpenClaw资讯汇总报告"

**执行步骤**：
1. 确定日期：2026-03-12（当日）
2. 执行搜索：使用web_search搜索"OpenClaw 2026-03-12 安全漏洞、监管预警、合规要求"等关键词
3. 分类整理：按六维分类体系整理搜索结果
4. 生成报告：使用模板生成markdown格式报告内容，不生成.md文件
5. 输出方式：直接在对话中输出完整的markdown报告内容

**输出示例**：
```markdown
# OpenClaw 2026-03-12资讯汇总

## 一、当日资讯概况
### 1.1 总体统计
- 发现资讯总数：18条
- 高风险资讯：2条（安全漏洞预警）
- 中风险资讯：6条（产业动态）
- 低风险资讯：10条（社区讨论）
- 主要来源：工信部官网、GitHub、科技媒体

### 1.2 关键发现
- **最重要发现**：发现OpenClaw高危安全漏洞CVE-2026-1234
- **风险最高**：安全监管类资讯，涉及数据泄露风险
- **趋势变化**：相比昨日，安全相关资讯增加50%

## 二、详细分类资讯
### 2.1 安全监管与风险警示
#### 高风险资讯
- **标题**：OpenClaw发现高危远程代码执行漏洞
  - **来源**：国家信息安全漏洞共享平台(https://example.com)
  - **内容摘要**：发现CVE-2026-1234漏洞，攻击者可远程执行任意代码
  - **风险等级**：高
  - **影响范围**：所有使用OpenClaw v1.2.0以下版本的系统
  - **建议措施**：立即升级到v1.2.1版本

（后续内容省略...）
```

## 输出格式说明

### Markdown格式规范
1. **标题层级**：使用#、##、###等标准标题
2. **列表格式**：使用-或*开头的无序列表
3. **链接格式**：[链接文本](URL)
4. **代码块**：使用```标记代码或示例
5. **加粗强调**：使用**文本**强调重要内容
6. **输出方式**：直接在对话中输入markdown内容，不生成.md文件

### 报告内容标准
1. **结构完整**：必须包含所有标准章节
2. **来源清晰**：每个资讯点都要注明来源
3. **风险明确**：明确标注风险等级
4. **建议具体**：提供可操作的建议措施
5. **格式统一**：保持一致的格式风格

## 最佳实践建议

### 搜索优化建议
1. **关键词组合**：使用多个相关关键词组合搜索
2. **时间范围**：根据需求调整搜索时间范围
3. **来源优先级**：优先搜索官方和权威来源
4. **验证信息**：重要资讯需要交叉验证多个来源

### 分类处理建议
1. **先粗后细**：先进行大类分类，再进行详细整理
2. **交叉分类**：注意可能属于多个分类的资讯
3. **优先级排序**：按风险等级和重要性排序
4. **定期调整**：根据实际情况调整分类标准

### 报告生成建议
1. **使用模板**：严格按照模板格式生成报告
2. **突出重点**：重要资讯放在前面和加粗强调
3. **保持客观**：基于事实进行分析，避免主观判断
4. **提供价值**：除了整理资讯，还要提供分析和建议

### 对话输出建议
1. **分段输出**：对于长报告可以分段输出
2. **格式检查**：输出前检查markdown格式是否正确
3. **用户体验**：考虑用户在对话中的阅读体验
4. **交互便捷**：便于用户复制、保存或进一步询问

## 常见问题处理

### 搜索不到信息时
- 扩大搜索关键词范围
- 调整时间范围
- 尝试不同的搜索引擎
- 检查网络连接和权限

### 分类不准确时
- 手动调整重要资讯的分类
- 更新关键词库和分类规则
- 增加新的分类类别
- 记录特殊情况供后续优化

### 报告格式问题
- 使用markdown预览工具检查格式
- 确保链接和引用格式正确
- 统一标题层级和列表格式
- 保持整体风格一致
