# 周边 POI 搜索：查找用户附近某类地点

> **适用摘要**: 用户想找附近的餐厅、药店、加油站等，基于当前位置搜索周边 POI。

## 触发意图

遇到以下用户需求时使用本 recipe：
- "找一下我附近2公里内的星巴克"
- "附近有哪些医院？"
- "周边500米内的停车场"
- "我当前位置附近的便利店"
- "离我最近的加油站在哪里"

## 前置条件

| 条件 | 要求 |
|---|---|
| 必要输入 | 用户当前位置坐标（纬度,经度）、搜索关键词（POI类型） |
| 搜索半径 | 默认 1000 米，可自定义，超过城市边界自动降级为城市范围搜索 |
| 坐标系 | BD09LL（百度坐标系）；GPS 原始坐标需声明坐标系类型 |
| 权限 | 标准 AK 即可 |

## 调用链

```
用户位置坐标 + 搜索关键词
  → circular_region_search（圆形区域搜索）
  → [可选] place_detail_search（获取 POI 详情）
  → 返回附近 POI 列表（名称、坐标、地址、电话等）
```

> **直接搜索 vs 先逆地理编码**:
> 如果用户给的是 GPS 坐标，可以声明坐标类型直接传入圆形搜索，无需先逆地理编码。
> 只有需要知道用户"在什么地方"的语义描述时，才需要先调用逆地理编码。

## 分步说明

### Step 1: circular_region_search（圆形区域 POI 搜索）

**目的**: 在用户位置附近指定半径内搜索 POI

**参考**: `../references/circular_region_search.md`

**需要提供**:
- 搜索关键词（如"星巴克"、"医院"、"停车场"）
- 中心坐标（用户位置）
- 搜索半径（米），建议范围 200-5000
- 是否严格限制在半径内（推荐开启，避免返回较远结果）
- 坐标系类型
- 是否返回详情（开启后可获取电话、评分、营业时间等）

**从响应获取**:
- 每个 POI 的名称、坐标、地址、电话
- POI 的唯一标识符（可传入详情接口获取更多信息）
- 评分、营业时间（开启详情模式时返回）

### [可选] Step 2: place_detail_search（获取指定 POI 的完整详情）

**目的**: 获取某个 POI 的完整详情（营业时间、评分、图片、描述等）

**参考**: `../references/place_detail_search.md`

**需要提供**: Step 1 结果中获取的 POI 唯一标识符

## 常见错误

| 错误 | 原因 | 解决方法 |
|---|---|---|
| 结果包含半径外的 POI | 未启用严格半径限制 | 开启严格半径限制选项 |
| GPS 坐标搜索结果偏移 | 未声明坐标系为 WGS84 | GPS 坐标需声明使用 WGS84 坐标系 |
| 无结果返回 | 关键词太具体或半径太小 | 扩大搜索半径或换用更通用的关键词 |
| 返回数据缺少详情字段 | 未开启详情返回模式 | 开启详情模式可获取评分、电话、营业时间 |
| 半径过大但只返回城市内结果 | 超过城市边界时自动变为城市范围搜索 | 这是正常行为，超大范围建议改用行政区划检索 |

## 变体

- **多关键词并集搜索**: 同时搜索多个类型（如肯德基、麦当劳、必胜客），最多10个
- **按评分排序**: 指定按评分从高到低排序（餐饮类支持）
- **分页获取更多结果**: 通过分页参数获取更多搜索结果
