# 坐标转结构化地址：GPS 坐标 → 行政区划 + 附近 POI

> **适用摘要**: 将经纬度坐标（WGS84/GCJ02/BD09LL）逆解析为可读地址、行政区划信息及周边 POI。

## 触发意图

遇到以下用户需求时使用本 recipe：
- "把坐标(39.9042, 116.4074)转成地址"
- "我有一批 GPS 坐标，需要知道对应的省市区名称"
- "根据经纬度获取详细地址和行政区划编码"
- "这个坐标在哪个区？"
- "逆地理编码"

## 前置条件

| 条件 | 要求 |
|---|---|
| 必要输入 | 经纬度坐标（格式: `纬度,经度`） |
| 坐标系 | 需明确坐标类别：GPS=WGS84，GCJ02，百度=BD09LL |
| 权限 | 标准 AK 即可（境外坐标需开通境外权限） |

## 调用链

```
输入坐标（注明坐标系）
  → global_reverse_geocoding（全球逆地理编码）
  → 返回结构化地址 + 行政区划 + 附近 POI（可选）
```

> 此接口一步完成，无需多步串联。
> **关键**: 坐标系参数必须与实际坐标系匹配，否则结果会有明显偏差（可达数百米）。

## 分步说明

### Step 1: global_reverse_geocoding（一步完成逆地理编码）

**目的**: 将坐标转换为结构化地址和行政区划信息

**参考**: `../references/global_reverse_geocoding.md`

**从响应获取**:
- 标准地址字符串（如"北京市朝阳区东坝路"）
- 含 POI 的详细地址（需开启 POI 扩展）
- 省、市、区县、乡镇街道名称及编码
- 所在商圈名称
- 语义化描述（如"百度大厦南150米"）
- 周边 POI 列表（需开启 POI 扩展）

## 常见错误

| 错误 | 原因 | 解决方法 |
|---|---|---|
| 地址解析结果偏差数百米 | 坐标系参数与实际坐标系不匹配 | GPS 数据需声明 WGS84，GCJ02 坐标需声明 GCJ02，百度坐标声明 BD09LL |
| 城市字段为空 | 坐标在省辖县或县级市区域 | 属正常现象，使用区县字段作为城市级别信息 |
| 周边 POI 为空 | 未开启 POI 扩展选项 | 调用时开启 POI 扩展功能 |
| 海外坐标返回错误 | 未开通境外权限 | 联系商务开通境外逆地理编码权限 |

## 坐标系选择速查

| 坐标类别 | 坐标系 |
|---|---|
| GPS 设备 / 手机原始 GPS | WGS84 |
| GCJ02（国测局坐标） | GCJ02（国测局） |
| 百度地图 | BD09LL |
| 百度地图米制坐标 | BD09MC |

## 变体

- **仅获取行政区划，不需要 POI**: 不开启 POI 扩展，速度更快
- **获取开发区信息**: 结果中包含开发区名称字段
- **选择行政区划数据源**: 默认使用民政部数据；如需含开发区信息，可切换为统计局数据源
