# 建议出发时间：想几点到，几点出发合适？

> **适用摘要**: 用户指定期望到达时间，系统反推建议出发时间。⚠️ 需要高级权限。

## 触发意图

遇到以下用户需求时使用本 recipe：
- "我想下午3点到公司，几点出发合适？"
- "明天早上9点要开会，我应该几点出门？"
- "帮我算一下几点出发能准时到达"
- "想在[时间]前到达[地点]，建议出发时间"

## 前置条件

| 条件 | 要求 |
|---|---|
| 必要输入 | 起点坐标（BD09LL）、终点坐标或地名、期望到达时间 |
| 坐标系 | BD09LL（百度坐标系） |
| 权限 | ⚠️ **高级权限**：需在百度地图控制台开通「建议出发时间」高级服务 |
| 时间范围 | 期望到达时间必须是当前时间之后 **15分钟** 以上的时刻 |

## 调用链

```
用户输入: 起终点 + 期望到达时间
  → [可选] Step 1: administrative_region_search（地名 → 坐标 + uid）
  → Step 2: driving_route_planning（含期望到达时间参数）
  → 返回建议出发时间
```

> 如果起点或终点是坐标，可以直接跳过 Step 1。
> 设置期望到达时间后，路线规划服务将依据该时间预测路况，并返回建议出发时间。
> 若算出的建议时间早于当前时间，则返回特殊值表示应立即出发。

## 分步说明

### [可选] Step 1: administrative_region_search（当终点为地名时）

**目的**: 将目的地名称转为坐标和 uid

**参考**: `../references/administrative_region_search.md`

**需要提供**: 目的地名称、城市名（提高准确率）

**从响应获取**: 目的地坐标和 POI 唯一标识符（uid）

### Step 2: driving_route_planning（含期望到达时间）

**目的**: 规划路线并计算建议出发时间

**参考**: `../references/driving_route_planning.md` → 高级权限: `../references/capabilities/suggested_departure_time.md`

**需要提供**: 起点坐标、终点坐标、终点 uid（若有）、期望到达时间（UNIX 时间戳，秒级）

**从响应获取**:
- 建议出发时间（UNIX 时间戳，特殊值表示应立即出发）
- 预计行驶耗时
- 路线距离

## 常见错误

| 错误 | 原因 | 解决方法 |
|---|---|---|
| 建议出发时间字段不返回 | 未开通高级权限 | 确认已在控制台开通「建议出发时间」服务 |
| 接口返回参数无效错误 | 期望到达时间早于当前时间 + 15 分钟 | 确保期望到达时间至少晚于当前时间 15 分钟 |
| 返回"立即出发"提示 | 按预测路况，应立即出发才能准时到达 | 提示用户"建议立即出发"，否则可能迟到 |
| 时间戳单位错误 | 使用了毫秒级时间戳 | 使用秒级 UNIX 时间戳 |

## 变体

- **终点是地名**: 先调用搜索建议获取坐标，再传入本接口（参考 `route_to_named_place.md`）
- **只想知道路线不需要建议时间**: 不传期望到达时间参数，使用普通路线规划
- **需要预测未来某时刻出发的耗时**: 改用出发时间参数，参考 `traffic_aware_route.md`
