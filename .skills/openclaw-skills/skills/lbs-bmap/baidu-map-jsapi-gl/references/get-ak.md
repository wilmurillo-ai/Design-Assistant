# 获取密钥（AK）

百度地图 JSAPI 需要使用 AK（Access Key）进行鉴权。使用任何地图功能前，必须先申请 AK。

## 当用户无 AK 或 AK 失效时

如果用户尚未拥有 AK，或者现有 AK 已失效（如被禁用、配额用尽、白名单不匹配等），请提示用户按照以下流程获取或更新 AK：

> 您需要一个有效的百度地图 AK 才能使用 JSAPI。请前往 [百度地图开放平台控制台](https://lbsyun.baidu.com/apiconsole/key) 创建或管理您的应用密钥。

## 申请流程

1. **进入控制台**：登录 [百度地图开放平台](https://lbsyun.baidu.com/)，点击页面上方的 **控制台**，然后在左侧边栏点击 **应用管理** → **我的应用** → **创建应用**。

2. **填写应用信息**：
   - **应用名称**：自定义名称
   - **应用类型**：JS API 必须选择 **浏览器端**
   - **启用服务**：默认全部勾选，可按需调整
   - **请求校验方式**：根据实际需求选择（IP 白名单 / Referer 白名单）
   - **白名单**：填写允许调用的域名或 IP（开发阶段可填写 `*`）

3. **提交并复制 AK**：创建完成后，在应用列表中复制 AK 即可使用。

## 注意事项

- **应用类型必须匹配**：JS API 只支持 **浏览器端** 类型的 AK，移动端需创建 Android/iOS SDK 类型，服务端需创建服务端类型。
- **权限配置**：每个 AK 可对应不同的基础服务权限，请确保所需服务已勾选启用。
- **配额监控**：可在控制台查看每个 AK 的配额使用量，避免因配额耗尽导致服务不可用。
- **白名单安全**：生产环境请务必配置准确的白名单，避免使用 `*` 通配符。

## AK 无效排查

当地图加载失败或控制台提示 AK 相关错误时，请先确认以下几点：

1. **AK 是否为浏览器端类型**：JS API 只支持浏览器端类型的 AK，使用服务端或移动端类型的 AK 会导致鉴权失败。
2. **AK 是否正确填写**：检查 script 标签中的 `ak` 参数是否完整、无多余空格。
3. **AK 是否仍然有效**：前往 [控制台](https://lbsyun.baidu.com/apiconsole/key) 确认 AK 状态正常、未被禁用或删除。

如确认 AK 无误仍无法使用，建议在控制台重新创建一个浏览器端类型的 AK。

## 在代码中使用 AK

```html
<!-- 将 YOUR_AK 替换为您申请的密钥 -->
<script src="//api.map.baidu.com/api?v=1.0&type=webgl&ak=YOUR_AK"></script>
```
