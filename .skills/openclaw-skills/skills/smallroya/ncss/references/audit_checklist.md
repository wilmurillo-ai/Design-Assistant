## 概览
本清单用于审计章节质量，确保内容符合创作规范和连续性要求。每项检查通过打"✓"，不通过打"✗"。

## 连续性检查

### 时间连续性
- [ ] 章节时间与上一章连续
- [ ] 时间流逝合理（不会突然跳过或重复）
- [ ] 角色年龄随时间推进

### 空间连续性
- [ ] 角色位置与上一章一致
- [ ] 场景转换合理
- [ ] 地理位置逻辑正确

### 物资连续性
- [ ] 角色携带的物品连续（不会突然出现或消失）
- [ ] 武器、装备状态一致
- [ ] 资源（金钱、能量等）数量合理

### 信息连续性
- [ ] 角色已知信息一致
- [ ] 不会出现角色"突然知道"未提及的信息
- [ ] 信息传递有明确来源

## 角色一致性

### 行为一致性
- [ ] 角色行为符合其性格设定
- [ ] 角色决策符合其动机和目标
- [ ] 不会出现角色行为突兀改变（无解释的情况下）

### 能力一致性
- [ ] 角色能力与设定一致
- [ ] 能力使用有逻辑限制
- [ ] 不会出现角色突然获得未提及的能力

### 说话风格
- [ ] 每个角色有独特的说话风格
- [ ] 角色用词习惯一致
- [ ] 不会出现角色说话风格突然改变

### 关系一致性
- [ ] 角色关系发展合理
- [ ] 不会出现关系突兀变化（无铺垫）
- [ ] 角色对彼此的态度一致

## 伏笔管理

### 伏笔推进
- [ ] 本章推进了至少一个未解决的伏笔
- [ ] 伏笔推进方式自然
- [ ] 不会强行推进不相关的伏笔

### 新伏笔
- [ ] 新增伏笔有必要性
- [ ] 伏笔暗示清晰（读者能意识到）
- [ ] 伏笔设置有解决方向

### 伏笔回收
- [ ] 本章回收了部分伏笔
- [ ] 回收方式出人意料但合乎逻辑
- [ ] 不会突然解决不相关的伏笔

### 伏笔追踪
- [ ] 在 `pending_hooks.md` 中更新伏笔状态
- [ ] 伏笔状态标记准确（open/progressing/resolved）
- [ ] 避免伏笔无限期拖延

## 叙事节奏

### 节奏变化
- [ ] 章节内有节奏变化（快慢交替）
- [ ] 长短段落搭配合理
- [ ] 场景转换自然

### 情节密度
- [ ] 情节推进适度（不会过于密集或稀疏）
- [ ] 每个场景都有明确目标
- [ ] 删除了不必要的场景

### 悬念设置
- [ ] 章节结尾有悬念或期待点
- [ ] 悬念设置合理
- [ ] 不会强行制造悬念

### 章节完整性
- [ ] 本章有完整的故事弧线
- [ ] 不会在中途突兀中断
- [ ] 为下一章做好铺垫

## 语言表达

### 词汇选择
- [ ] 用词准确、恰当
- [ ] 避免重复使用相同词汇
- [ ] 动词有力，避免过多副词

### 句式变化
- [ ] 句式多样化（长短句搭配）
- [ ] 避免句式单调重复
- [ ] 特殊句式使用适度

### 对话质量
- [ ] 对话自然、真实
- [ ] 每句对话都有功能（推进情节、揭示性格等）
- [ ] 避免信息堆砌式的对话

### 描写质量
- [ ] 描写生动、有画面感
- [ ] 调动五感（视、听、嗅、味、触）
- [ ] 描写服务于情节和角色

### 去AI味检查
- [ ] 避免过度使用"突然"、"顿时"等词
- [ ] 避免过度总结和概括
- [ ] 避免句式机械重复
- [ ] 避免过度使用形容词和副词

## 逻辑自洽

### 因果关系
- [ ] 每个结果都有明确原因
- [ ] 不会出现无原因的结果
- [ ] 因果链条完整清晰

### 行为逻辑
- [ ] 角色行为有明确动机
- [ ] 角色决策符合其认知水平
- [ ] 不会出现角色做出不合理选择

### 世界规则
- [ ] 符合世界观设定的规则
- [ ] 不会违反已设定的物理/魔法规则
- [ ] 新设定的规则有充分解释

### 情节逻辑
- [ ] 情节发展符合内在逻辑
- [ ] 不会出现机械降神
- [ ] 转折有铺垫和暗示

## 情感弧线

### 情感连贯
- [ ] 角色情感变化自然连贯
- [ ] 不会出现情感突兀变化
- [ ] 情感反应符合性格和情境

### 情感深度
- [ ] 情感描写有深度，不止于表面
- [ ] 读者能够共情角色的情感
- [ ] 情感高潮有充分铺垫

### 情感推进
- [ ] 本章推进了角色的情感弧线
- [ ] 情感发展有明确方向
- [ ] 不会重复相同的情感状态

### 情感与情节
- [ ] 情感变化与情节发展同步
- [ ] 情感不是孤立的，而是融入情节
- [ ] 情感对角色行为有影响

## 世界观一致性

### 设定一致
- [ ] 符合已设定的世界观
- [ ] 不会出现设定矛盾
- [ ] 新设定与旧设定兼容

### 细节一致
- [ ] 地名、物品名等细节一致
- [ ] 不会出现同一事物不同名称
- [ ] 历史事件描述一致

### 文化一致
- [ ] 符合设定的文化背景
- [ ] 不会出现文化冲突（除非是故事冲突）
- [ ] 习俗、礼仪等细节准确

### 规则一致
- [ ] 魔法/科技规则一致
- [ ] 不会随意改变规则
- [ ] 新规则有充分解释

## 格式规范

### 章节格式
- [ ] 章节标题符合规范（"第X章"或"Chapter X"）
- [ ] 段落分隔清晰
- [ ] 对话格式正确

### 字数要求
- [ ] 字数符合目标范围（±10%）
- [ ] 不会过短（少于1000字）或过长（超过6000字）
- [ ] 字数与内容密度匹配

### 特殊格式
- [ ] Markdown 格式正确
- [ ] 不会出现格式错误
- [ ] 标题层级合理

### 文件命名
- [ ] 文件名符合规范（chapter-XX.md）
- [ ] 章节编号连续
- [ ] 不会出现编号跳跃
