# 概念解释记录与用户档案存储规范

本文件定义 `my-study-pal` 的本地概念解释记录结构与用户档案结构。默认在**当前工作空间根目录**下创建并维护 `mystudy/`。

与其他文件的分工：
- `setup.md` 负责首次启用、补齐骨架和可运行状态检查
- `references/blueprint.md` 负责定义最小可运行骨架
- `study-storage.md` 负责定义目录结构、文件职责和模板内容

## 目录结构

```text
mystudy/
  study-summary.md
  study-detail/
    主题名YYYYMMDD.md
  user-profile.md
```

## 文件职责

### 1. study-summary.md

用于保存学习主题概览，适合作为长期总表。

至少记录以下字段：
- 时间（精确到时分秒）
- 学习主题
- 是否完成

### 2. study-detail/

用于保存单个学习主题的对话记录。

文件命名规则：
- `主题名YYYYMMDD.md`
- 文件名尽量简洁，去掉非法路径字符
- 同一天同主题若重复记录，可在文件名后追加序号，例如：
  - `提示词工程20260410-2.md`
- 作为 skill 内置模板文件时，优先使用 ASCII 文件名保存模板本体，例如 `study-detail-template.md`

每个详情文件至少包含：
- 学习主题
- 日期
- 对话记录（直接保留完整对话过程，不做摘要版纪要）

### 3. user-profile.md

用于保存用户画像与长期教学偏好。

建议包含以下结构：

- 用户工作 / 专业 / 业务领域
- 用户爱好领域
- 用户最近的 5 个学习知识点
- 用户教学偏好
  - 讲解式
    - 教学模块
    - 展开程度
    - 术语风格
  - 引导式（预留）
- 语言风格偏好

运行时规则：
- 该文件是概念解释过程中优先读取的实际生效配置
- 如果该文件缺少某些偏好字段，再回退到对应 reference 文件中的默认建议
- 用户提出长期偏好时，必须在同一轮内实时更新该文件
- 用户提出临时偏好时，本轮可先临时生效，不必默认改写该文件
- 不允许出现“回复里说已记住，但用户档案未更新”的状态

## 初始化与可运行状态

当 skill 第一次需要落盘概念解释记录或用户画像时，应先读取 `setup.md`，并按 `references/blueprint.md` 补齐骨架。

### 补齐要求

1. 检查工作空间下是否存在 `mystudy/`
2. 若不存在，则创建：
   - `mystudy/`
   - `mystudy/study-detail/`
3. 若 `study-summary.md` 不存在，则创建默认模板
4. 若 `user-profile.md` 不存在，则创建默认模板
5. 不要覆盖用户已有内容

### 可运行状态

概念解释系统至少满足以下条件时，才算达到可运行状态：
- `mystudy/` 存在
- `mystudy/study-detail/` 存在
- `study-summary.md` 存在
- `user-profile.md` 存在
- `user-profile.md` 中至少包含讲解式默认配置

核心原则是：**第一次需要记录概念解释内容或用户画像时，目录一定要自动补齐；未达到可运行状态前不要假装初始化已经完成。**

## 推荐模板

### study-summary.md 模板

```markdown
# 学习概览

| 时间 | 学习主题 | 是否完成 |
|---|---|---|
| 2026-04-10 09:30:00 | 示例主题 | 进行中 |
```

### user-profile.md 模板

```markdown
# user-profile

## 用户工作 / 专业 / 业务领域
- 暂无记录

## 用户爱好领域
- 暂无记录

## 用户最近的 5 个学习知识点
- 暂无记录

## 用户教学偏好

### 讲解式
- 教学模块：正常（定义解释、示例呈现、特征构成、关键点、追问）
- 展开程度：标准
- 术语风格：白话为主，术语点到为止

### 引导式
- 预留

## 语言风格偏好

### 1. 扮演角色：默认风格（清晰、自然、像可靠同事）

#### 核心气质与立场
- 清晰、自然、可靠

#### 句式与措辞
- 直接表达，少绕弯，默认不过度风格化

#### 口头禅
- 暂无固定口头禅
- 若用户明确指定，应优先自然穿插，增强识别度

### 2. 其他语言配置
- emoji：按需少量使用
```

### study-detail 单次记录模板

```markdown
# 主题对话记录

- 学习主题：
- 日期：

## 对话过程
用户：
助手：
```
