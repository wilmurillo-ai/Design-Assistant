# Contributing to AetherCore

Thank you for your interest in contributing to AetherCore! This document provides guidelines for contributing.

## 🎯 Code of Conduct

Please be respectful and constructive in all interactions.

## 🚀 How to Contribute

### 1. Reporting Issues
- Check if the issue already exists
- Provide clear steps to reproduce
- Include error messages and system information

### 2. Feature Requests
- Explain the problem you're trying to solve
- Describe your proposed solution
- Provide examples if possible

### 3. Pull Requests
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add or update tests
5. Update documentation
6. Submit a pull request

## 🛠️ Development Setup

### Prerequisites
- Python 3.8+
- Git

### Installation
```bash
git clone https://clawhub.ai/aethercore
cd AetherCore
pip install -r requirements.txt
```

### Running Tests
```bash
python run_simple_tests.py
```

## 📝 Coding Standards

### Python Code
- Follow PEP 8 style guide
- Add type hints where appropriate
- Write docstrings for public functions
- Keep functions focused and small

### Documentation
- Update README.md for user-facing changes
- Update CHANGELOG.md for significant changes
- Keep documentation clear and concise

## 🧪 Testing
- Write tests for new functionality
- Ensure existing tests pass
- Test edge cases and error conditions

1. All pull requests will be reviewed
2. Feedback will be provided within a reasonable timeframe
3. Changes may be requested before merging
