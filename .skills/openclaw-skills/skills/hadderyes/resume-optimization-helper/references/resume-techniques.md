# 简历优化技巧 - 完整参考

> SKILL.md 已包含核心技巧。本文档提供行业适配和进阶技巧，按需加载。

## 行业优化要点

### 互联网/技术岗
- 突出技术栈版本和深度（如"React 18 + TypeScript"比"熟悉前端"强10倍）
- 量化性能优化成果（"首屏加载时间从4.2s降至1.1s"）
- GitHub开源项目、技术博客、演讲经历加分
- 关键词：架构设计、性能优化、敏捷开发、CI/CD、微服务

### 销售/市场岗
- 必须量化：客户数、销售额、转化率、市场占有率
- 突出大客户经验和从0到1的开拓能力
- 标明销售区域范围和团队管理规模
- 关键词：拓客、转化、客户关系管理、KPI达成率、市占率

### 财务/金融岗
- 突出管理资金规模和节省成本数据
- 注明使用的财务系统（SAP、Oracle等）
- 强调合规和风控经验
- 关键词：预算编制、财务分析、成本控制、审计、合规

### 产品/运营岗
- 用DAU/MAU/GMV等数据指标说话
- 突出从需求分析到上线落地的全流程能力
- A/B测试和增长黑客经验是加分项
- 关键词：用户增长、转化率、留存率、产品迭代、数据驱动

### 新人/应届生
- 突出实习经历和项目经验（课设也行）
- 强调学习能力和快速上手经历
- 列出与岗位相关的课程和证书
- 学生会/社团经历突出领导力和组织能力

## STAR法则深度指南

### 常见错误
- 只写了Action，缺少Situation背景
- Result太模糊（"取得了很好的效果"）
- 把团队成果写成个人成果
- 动词不够具体（"做了"→"主导"）

### 数据化技巧
- **时间**: "提前X周/月完成"、"缩短X%工期"
- **金额**: "节约X万成本"、"创造X万营收"
- **人数**: "管理X人团队"、"服务X万用户"
- **比例**: "提升X%"、"降低X%"、"转化率X%"

### 无数据时的替代方案
- 描述规模："负责3条产品线" "覆盖华东5省"
- 描述频次："日均处理200+工单" "每周产出3篇原创"
- 描述排名："部门Top 3" "连续2季度超额完成"
- 描述认证："通过XX认证" "获得XX资质"

## 简历自检清单

### 必查项
- [ ] 篇幅控制在一页纸（应届生）或两页（5年以上经验）
- [ ] 每段工作经历都有量化的成果
- [ ] 核心优势在简历前1/3区域
- [ ] 没有错别字和语法错误
- [ ] 联系方式清晰完整

### 加分项
- [ ] 针对JD定制了关键词
- [ ] 包含了LinkedIn/GitHub等链接
- [ ] 自我评价有数据支撑而非空洞形容词
- [ ] 时间线连贯，无大段空白

### 减分项（立即删除）
- [ ] 照片（除非外企或特定行业要求）
- [ ] 身高体重年龄等无关信息
- [ ] "精通"但实际只是了解的技能
- [ ] 虚假或夸大的数据
- [ ] 与求职岗位无关的经历

## HR筛简历的3秒法则

HR平均花3-6秒浏览一份简历，视线轨迹通常是：

1. **基本信息区**（1秒）：学校、学历、工作年限
2. **个人优势/摘要**（1秒）：是否有亮点关键词
3. **最近一段工作**（1-2秒）：公司知名度 + 职位 + 成果
4. **技能区**（1秒）：是否匹配岗位要求

因此：**前1/3决定生死**。把最亮的信息放在最上面。
