# Publish Notes

## Why this skill first

This is the clearest and most broadly useful of the OpenClaw-adapted development skills.
It captures a simple but high-value behavior:
- do not confuse command success with task success
- verify real outcomes before reporting completion

## Intended audience

AI agents and coding assistants that perform:
- setup work
- bug fixes
- automation changes
- configuration updates
- browser or script tasks

## Value proposition

Prevents false completion reports and improves trust.

## Suggested publish summary

Verify real outcomes before saying work is complete. Useful for setup, fixes, automation, and configuration tasks where edited text or exit code is not enough proof.

## Suggested changelog

Initial OpenClaw-adapted release. Focuses on outcome verification, partial-verification reporting, and practical examples for setup, scripts, reminders, and browser automation.
