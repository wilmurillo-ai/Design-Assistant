# hypothesis-lab

## Use when

- A strategic claim needs validation before heavy implementation.
- Team needs a fast test instead of a long roadmap debate.

## Inputs

- Hypothesis statement
- Target segment and context
- Current baseline metric
- Experiment constraints (time/budget/tools)

## Steps

1. Rewrite hypothesis into falsifiable form.
2. Pick the smallest viable test.
3. Define success and failure thresholds up front.
4. Run for 7 days or fixed sample size.
5. Decide continue, stop, or pivot.

## Experiment card

```md
Hypothesis:
If we [change], then [target segment] will [measurable behavior].

Test design:
- Duration:
- Channel:
- Sample size:

Success threshold:
Failure threshold:

Decision rule:
- Continue if ...
- Stop if ...
- Pivot if ...
```

## Acceptance criteria

- Hypothesis is falsifiable.
- Success/failure thresholds are numeric.
- Decision rule is defined before running.
