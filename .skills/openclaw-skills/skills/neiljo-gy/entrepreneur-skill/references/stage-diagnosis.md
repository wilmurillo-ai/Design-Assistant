# stage-diagnosis

## Use when

- Team is unsure whether the main challenge is problem, distribution, retention, pricing, or operations.
- Founder asks "What should we focus on this week?"

## Inputs

- Current stage guess (`0->1` or `1->10`)
- Current product and target segment
- Last 2-4 weeks of key metrics
- Main constraints (time, budget, headcount)

## Steps

1. Identify stage by evidence, not self-label.
2. Find one dominant bottleneck.
3. Quantify impact if bottleneck is solved.
4. Propose 1-2 highest-leverage moves for next 7 days.

## Output template

```md
Stage: [0->1 | 1->10]
Primary bottleneck: [...]
Evidence: [...]
Top moves (7 days):
1) [...]
2) [...]
North-star metric this week: [...]
```

## Acceptance criteria

- A single primary bottleneck is chosen.
- Each proposed move has measurable expected impact.
- At least one "stop doing" decision is included.
