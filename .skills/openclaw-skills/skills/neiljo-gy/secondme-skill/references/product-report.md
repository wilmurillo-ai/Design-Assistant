# secondme 产品报告

## 1. 产品定义

`secondme` 是一个“个人智能体分身技能包”，目标是让每个人都能拥有可持续进化的 AI 分身，并具备以下能力：

- 代表个人：复现用户的表达风格、决策偏好与价值边界。
- 管理数据：数据本地优先、可追溯、可增量更新、可审计导出。
- 理解自我：基于长期知识与记忆产物形成结构化自我洞察。

## 2. 用户价值与场景

### 目标用户

- 创作者、研究者、创业者、学生、长期记录者

### 核心场景

- 个体认知镜像：理解“我如何做决策”与“我为何这样表达”。
- 个人协作代理：以用户风格完成整理、草拟、复盘。
- 长期成长回顾：基于时序数据追踪兴趣、关系与行为模式变化。

## 3. 对标 Second-Me 的启发

基于 `mindverse/Second-Me` 的可借鉴点：

- 本地训练与部署优先：强调数据主权与控制权。
- 接口兼容思路：通过标准化接口降低接入成本。
- 长期记忆驱动：分身基于持续数据更新而不是一次性提示词。
- 身份化定位：分身不仅会答复，还可承载个人上下文。

## 4. 现有能力与架构映射

`secondme` 以“编排层”串联现有三套技能：

1. `anyone-skill`：身份蒸馏入口（4 维提炼 + 证据分级）
2. `persona-knowledge`：数据治理中台（摄取、PII 扫描、去重、wiki/KG、导出）
3. `persona-model-trainer`：本地模型具身（训练、评测、导出、版本管理）

结论：底层能力基本齐备，核心工作是流程编排与治理固化。

## 5. MVP 编排流程

`init -> ingest -> distill -> train -> eval -> integrate -> report`

### 门槛策略

- 数据门槛：assistant 样本 >= 200，且来源类别 >= 2
- 训练门槛：完成训练并产出可运行导出
- 质量门槛：voice >= 3.5、probe >= 0.8、perplexity 劣化 <= 20%
- 治理门槛：PII 通过、高风险动作审批留痕、导出审计留痕

## 6. 治理与安全策略

### 权限原则

- 本地优先
- 最小权限
- 默认拒绝外发

### 审批门

以下动作必须显式人工确认：

- 法律与金融承诺
- 账号写操作或不可逆操作
- 对外发布/共享身份资产

## 7. 交付产物标准

### 状态文件

- 路径：`state/pipeline-state.json`
- 关键字段：`current_stage`、`last_success_stage`、`dataset_version`、`model_version`、`error_code`、`retry_count`

### 报告目录

- 数据报告：`reports/data/`
- 模型报告：`reports/model/`
- 部署报告：`reports/deploy/`

### 报告命名

`{slug}_{datasetVersion}_{modelVersion}_{timestamp}_{reportType}.md`

## 8. 风险与缓解

- 数据不足：阻断训练并引导补充来源
- 算力受限：按硬件分层自动降级（MLX/Unsloth/Colab）
- 人格漂移：immutableTraits + 周期性回归评测
- 隐私外溢：默认本地，导出必须审计

## 9. 实施结论

`secondme` 可在现有能力上快速落地，无需重造训练与知识底层。  
推荐采用 OpenPersona 标准骨架作为主路径，在此之上叠加编排协议、门禁规则与报告体系，实现可运行、可治理、可复制的个人智能体分身。
