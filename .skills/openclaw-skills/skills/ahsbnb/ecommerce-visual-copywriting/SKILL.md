# 电商视觉文案设计 SOP

> 输入：产品信息 + 主图初稿 → 输出：合规精简的主图+详情页执行方案（每张图只含：**画面内容 + 图内文案 + 设计场景说明**）

## 何时使用

用户提到以下任一关键词时触发：
- 主图文案 / 详情页文案 / 电商文案 / 商品文案
- listing / 产品页面 / 转化率 / 点击率
- 合规审查 / 广告法 / 平台审核
- 淘宝 / 天猫 / 京东 / 拼多多 / 抖音小店 文案

## 核心工作流

```
1. 收集信息 → 2. 提炼必卖理由 → ⏸️ 用户确认 → 3. 主图5张 → 4. 详情页模块 → 5. 自审评分 → 6. 输出
                              ↑
                         不通过则修改
```

> **⏸️ 门禁规则**：Step 2 完成后，必须将**合规版必卖理由**输出给用户审阅确认。用户确认后才能进入 Step 3 及后续环节。用户提出修改意见则调整后再提交。

### Step 1: 收集信息（必须拿到才能开工）

向用户索要以下材料，缺项标记清楚：

| # | 必须有 | 用途 |
|---|--------|------|
| 1 | **主图文案初稿**（用户已有或口述）| 基础素材 |
| 2 | **产品类型判定**：保健食品(蓝帽子) / 普通食品 / 运动器材 / 其他 | 决定套哪套合规规则 |
| 3 | **品牌名 + 公司全称** | 品牌信任模块、法律声明署名 |
| 4 | **SKU列表+价格** | 规格参数表 |

强烈建议：
| # | 建议有 | 用途 |
|---|--------|------|
| 5 | 商机分析/目标人群画像 | 必卖理由提炼方向 |
| 6 | 竞品链接/截图 | 差异化定位 |
| 7 | 产品实拍图 | 场景规划更具体 |

### Step 2: 提炼必卖理由卡片（⏸️ 输出后等用户确认）

从产品信息中提取 **3-5条必卖理由**，每条含：

```
目标人群 → 痛点 → 解决方案 → 利益翻译 → 信任证据 → 优先级 → 适用模块
```

规则：
- 每条必须基于**可验证的事实**（专利号/检测报告/认证/工艺特点）
- 优先级 ★★★★★ = 主推核心差异 | ★★★★ = 重要辅助 | ★★★ = 补充
- **必须经过合规审查**：无绝对化用语、无功效暗示、无医疗化表达
- 输出后**暂停**，等待用户确认或修改意见

> ⚠️ **门禁：未获用户明确确认前，禁止进入 Step 3 主图阶段。**

### Step 3: 主图5张（⚠️ 必须在 Step 2 用户确认后才能执行）

#### 图1 首图・点击率核心
- **画面**：产品居中 + 背景氛围 + 动效钩子
- **图内文案**：≤5行（标题+副标题+卖点+兜底声明）
- **设计说明**：构图重点、字号层级、点击率钩子是什么

#### 图2 痛点共鸣・用户代入
- **画面**：问题场景 vs 解决方案（对比构图）
- **图内文案**：问句引入 + 对比描述 + 利益点 ≤5行
- **设计说明**：情绪引导方式、不承诺效果

#### 图3 差异化优势・竞品区隔
- **画面**：对比实拍（左旧右新）或数据可视化
- **图内文案**：差异化标题 + 对比表 + 证据 ≤5行
- **设计说明**：真实性要求、视觉引导路径

#### 图4 场景适配・使用价值
- **画面**：2-3个使用场景并列（三宫格）
- **图内文案**：场景标签 + 一句话利益 + 规格 ≤5行
- **设计说明**：风格调性、生活化程度

#### 图5 CTA行动号召
- **画面**：产品全家福 + CTA按钮区域
- **图内文案**：情绪价值语 + 信任背书清单 + 价格 + CTA ≤5行
- **设计说明**：CTA跳色处理、价格展示方式

### Step 4: 详情页模块（按需取舍，不必全做）

根据产品需求选择模块组合。**每个模块同样只写三样：画面 / 图内文案 / 设计说明**

#### M1 首屏痛点共鸣
- 目标：3秒抓住人，让用户说"这就是我"

#### M2 核心优势展开
- 将必卖理由Top2-3逐个展开：痛点→方案→证据

#### M3 配方/工艺深度
- 配料表图解 + 工艺流程 + 营养成分数据

#### M4 使用场景
- 2-3个典型场景代入（工位/出差/通勤/运动…）

#### M5 品牌/资质信任 ⚠️ 可选
- 专利证书 + 生产许可 + 品牌介绍

#### M6 规格参数+竞品对比
- 参数表格 + 属性差异对照表

#### M7 FAQ
- 从必卖理由反向推演顾虑，6道题左右
- 最后一题必须是产品属性界定+免责声明

#### M8 购买引导+法律声明 ⚠️ 可选
- CTA + 免责声明完整版

> **注意**：M5/M8 根据产品需要决定是否做，不是必须。

### Step 5: 自审评分（强制！≥80分才允许输出）

写完初稿后，用以下评分表逐项打分：

| # | 评审项 | 分值 | 扣分标准 |
|---|--------|------|---------|
| 1 | **文案精简度**：无多余文字、无教学式解释 | 20 | 每处多余解释 -2 |
| 2 | **图内可放性**：文案量适合放入图片（≤5行/主图，≤6行/详情）| 20 | 每超1行 -3 |
| 3 | **合规性**：无绝对化用语、无功效暗示、免责声明完整 | 25 | 每处违规 -5 |
| 4 | **结构清晰度**：每张图只有「画面+文案+设计说明」三项 | 15 | 多余板块每个 -3 |
| 5 | **实用性**：设计师拿到就能直接开工，无需再问 | 20 | 缺关键信息每处 -4 |

**总分 < 80？自行重写，不打扰用户直到合格。**

### Step 6: 输出

- 以 Markdown 格式输出完整执行方案
- 作为 artifact 制品交付（`isArtifact=true`）

## 合规规则库（详见 references/compliance-rules.md）

按产品类型自动匹配对应规则层：

| 类型 | 规则层 | 核心红线 |
|------|--------|---------|
| 所有产品 | **零层：通用** | 绝对化用语清零、数据必须有背书、不贬低竞品 |
| 蓝帽子保健食品 | **一层：批文铁律** | 只能说批准功能，一字不多不少 |
| 运动器材/体态管理 | **二层：去医疗化** | 禁医学术语、效果必须弱化、禁治疗动词 |
| 普通食品（非保健非药品）| **三层：最严** | **什么功能都不能说**，连暗示都不行 |

详细替换表、禁用词清单、免责声明模板见 `references/compliance-rules.md`

## 文案量标准速查

| 位置 | 上限 | 目标 |
|------|------|------|
| 主图标题 | ≤10字 | 5-8字 |
| 主图副标题 | ≤15字 | 8-12字 |
| 单张主图总行数 | ≤5行 | 3-4行 |
| 详情页单模块 | ≤6行有效文字 | 4-5行 |
| FAQ单条回答 | ≤4行 | 2-3行 |
