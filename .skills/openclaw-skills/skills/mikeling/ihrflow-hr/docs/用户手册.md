## 1. 这是什么

iHRFlow AI 招聘助手是一个 **OpenClaw 技能（Skill）**，让你的 AI 助手直接连接 iHRFlow 人才管理平台，用自然语言完成招聘操作。

**你可以用它做什么：**

- "帮我搜一下会 Python 的候选人"
- "今天有什么面试？"
- "把张三推荐到全栈工程师岗位"
- "王五通过了 HR 初筛，推进到下一步"
- "帮我看看整体招聘数据"

**你不需要：** 打开 iHRFlow 后台、手动翻简历、在多个页面之间来回切换。

## 2. 安装与配置

### 前提条件

| 条件 | 说明 |
|------|------|
| iHRFlow 后端 | 已部署且 MCP Server 正在运行 |
| OpenClaw | 已安装（版本 2026.3.x 或更高） |
| 系统依赖 | `curl` 和 `jq`（macOS/Linux 通常已预装） |

### 第一步：安装技能

**方式 A — 从 ClawHub 安装（推荐）：**

```bash
clawhub install ihrflow-hr
```

**方式 B — 手动安装：**

```bash
cp -r ihrflow-skill/ ~/.openclaw/skills/ihrflow-hr/
```

### 第二步：配置环境变量

有两种配置方式，任选其一：

#### 方式 A：通过 Dashboard 图形界面配置（推荐）

1. 打开 OpenClaw 控制台：

```bash
openclaw dashboard
```

2. 在左侧导航栏点击 **设置 → 配置**

3. 左侧分类列表中点击 **Environment**

4. 选择顶部的 **Environment Variable Overrides** 标签页

5. 点击 **"+ Add Entry"** 按钮，逐条添加以下键值对：

| Key（左侧输入框） | Value（右侧输入框） | 是否必填 |
|---|---|---|
| `IHRFLOW_MCP_URL` | `https://ihrflow.com/mcp` | 必填 |
| `IHRFLOW_USERNAME` | 你的 iHRFlow 用户名 | 必填 |
| `IHRFLOW_PASSWORD` | 你的 iHRFlow 密码 | 必填 |
| `IHRFLOW_TENANT_ID` | 你的租户 ID | 多租户时必填 |
| `IHRFLOW_API_KEY` | MCP 传输层 API 密钥 | 看服务器配置 |

6. 全部填完后，点击页面右上角的 **Save** 按钮保存

7. 点击 **Apply** 或手动执行 `openclaw gateway restart` 使配置生效

> **进阶提示：** 如果你只想让这些变量对 iHRFlow 技能生效而不影响其他技能，可以点击右上角 **Raw** 按钮进入 JSON 编辑模式，找到或新建 `skills.entries` 节点：
>
> ```json
> {
>   "skills": {
>     "entries": {
>       "ihrflow-hr": {
>         "enabled": true,
>         "env": {
>           "IHRFLOW_MCP_URL": "https://ihrflow.com/mcp",
>           "IHRFLOW_USERNAME": "你的用户名",
>           "IHRFLOW_PASSWORD": "你的密码",
>           "IHRFLOW_TENANT_ID": "你的租户ID",
>           "IHRFLOW_API_KEY": "你的API密钥"
>         }
>       }
>     }
>   }
> }
> ```

#### 方式 B：通过命令行配置

```bash
# 必填：MCP 服务器地址
openclaw config set env.IHRFLOW_MCP_URL "https://ihrflow.com/mcp"

# 必填：你的 iHRFlow 登录账号
openclaw config set env.IHRFLOW_USERNAME "你的用户名"

# 必填：你的 iHRFlow 登录密码（纯数字密码需加引号）
openclaw config set env.IHRFLOW_PASSWORD '"你的密码"' --strict-json

# 可选：租户 ID（多租户部署时必填）
openclaw config set env.IHRFLOW_TENANT_ID "你的租户ID"

# 可选：MCP 传输层 API Key（服务器配置了 API Key 时必填）
openclaw config set env.IHRFLOW_API_KEY "你的API密钥"
```

> **提示：** 如果密码是纯数字（如 `111111`），必须用 `'"111111"' --strict-json` 格式，否则会被当作数字而非字符串。

### 第三步：重启并验证

```bash
# 重启 OpenClaw Gateway 使配置生效
openclaw gateway restart

# 验证技能状态（应显示 ✓ Ready）
openclaw skills info ihrflow-hr
```

正确输出：

```
briefcase ihrflow-hr ✓ Ready
...
Requirements:
  Binaries: ✓ curl, ✓ jq
  Environment: ✓ IHRFLOW_MCP_URL, ✓ IHRFLOW_USERNAME, ✓ IHRFLOW_PASSWORD
```

如果任何一项显示 `✗`，请检查对应的配置是否正确设置。

## 4. 日常使用场景

### 📅 场景一：每日早会 / 招聘日报

```
你: 给我一份今天的招聘日报
```

AI 会整合今日日程、招聘统计、在招岗位，生成一份结构化报告。

### 🔍 场景二：为岗位寻找候选人

```
你: 帮我找适合"高级Python后端工程师"的候选人

你: 看看这个岗位的详细要求    （AI 会先查岗位需求）
你: 帮我搜3年以上Python后端经验的人
你: 第一个人的简历详情看看
你: 把他推荐到这个岗位
```

```
你: 看看全栈工程师岗位下的候选人
你: 张三已经通过 HR 初筛了，帮我推进
你: 给他的简历加个备注：沟通能力优秀，建议加速推进
```

```
你: 看看目前已安排的面试

### 📊 场景五：面试结束后的评价

```

### 🆕 场景六：创建新岗位

```

```

## 6. 招聘流程说明

### 候选人筛选流程

```
简历投递 → HR初筛 → [笔试] → 部门筛选 → 面试（一面/二面/三面）→ 终审 → 发Offer
```

每个阶段可以「通过」或「拒绝」：

| 阶段 | 通过操作 | 拒绝操作 |
|------|---------|---------|
| HR 初筛 | "HR初筛通过" | "HR初筛拒绝" |
| 部门筛选 | "部门审核通过" | "部门审核拒绝" |
| 终审 | "终审通过" | "终审拒绝" |

> **注意：** 笔试阶段由系统自动管理，不需要通过 AI 助手操作。

### 岗位生命周期

```
创建（待发布）→ 发布（招聘中）→ 暂停 ↔ 恢复 → 关闭
```

### 面试状态流转

```
待安排 → 已安排 → 待评价 → 已完成
            ↓
          已取消
```

## 7. 常见问题

### Q: AI 回复说「未认证」或「Session expired」怎么办？

告诉 AI："重新登录 iHRFlow"，它会自动重新初始化会话。如果反复出现，检查用户名密码是否正确：

```bash
openclaw config get env
```

### Q: 搜索候选人没有结果？

- 尝试换个关键词，支持技能名、姓名、学校名等
- 用 `search_talent` 语义搜索描述需求而非精确关键词，例如："3年以上后端经验，熟悉微服务架构"
- 确认 iHRFlow 后端数据库中已导入简历数据

### Q: AI 没有自动调用 iHRFlow，只是在"聊天"？

确认 Skill 处于 Ready 状态：

```bash
openclaw skills info ihrflow-hr
```

如果显示 `blocked` 或环境变量缺失，请回到 [安装与配置](#2-安装与配置) 部分重新设置。

### Q: Dashboard 上显示 Skill 是 blocked 状态？

这通常是因为环境变量没有持久化配置。在终端中 `export` 的变量不会被 Gateway 进程继承。

**解决方法（任选其一）：**

- **图形界面：** 打开 Dashboard → 设置 → 配置 → Environment → Environment Variable Overrides → 添加缺失的变量 → Save → Apply
- **命令行：** 使用 `openclaw config set env.xxx` 写入配置

具体步骤见 [第二步：配置环境变量](#第二步配置环境变量)。

### Q: 技能页面的「API key」输入框是干什么的？

Dashboard「技能」页面中 ihrflow-hr 旁边的「API key」输入框只对应 Skill 声明的 `primaryEnv`（即 `IHRFLOW_MCP_URL`）这 **一个** 变量。iHRFlow Skill 需要多个环境变量，因此仅靠这个输入框不够。请到 **配置 → Environment → Environment Variable Overrides** 中添加全部变量（见 [方式 A](#方式-a通过-dashboard-图形界面配置推荐)）。

### Q: 我是多租户部署，每个人的数据不同怎么办？

iHRFlow Skill 支持多租户。在配置中设置 `IHRFLOW_TENANT_ID`，不同用户在各自的 OpenClaw 实例中配置自己的用户名、密码和租户 ID，彼此数据完全隔离。

### Q: 密码是纯数字，配置时报错？

纯数字密码会被识别为数字类型。使用以下格式：

```bash
openclaw config set env.IHRFLOW_PASSWORD '"123456"' --strict-json
```

### Q: 我可以在手机上用吗？

可以。将 OpenClaw 连接到 Telegram / Discord / 飞书等聊天渠道后，你可以在任何设备上与 AI 助手对话，它会通过 Skill 操作 iHRFlow。

## 8. 附录

### 环境变量

| 变量名 | 必填 | 说明 | 示例 |
|--------|------|------|------|
| `IHRFLOW_MCP_URL` | ✅ | MCP 服务器地址 | `https://ihrflow.com/mcp` |
| `IHRFLOW_USERNAME` | ✅ | iHRFlow 用户名 | `hr_admin` |
| `IHRFLOW_PASSWORD` | ✅ | iHRFlow 密码 | `MyP@ssw0rd` |
| `IHRFLOW_TENANT_ID` | 多租户时必填 | 租户 ID | `acme-corp` |
| `IHRFLOW_API_KEY` | 看服务器配置 | MCP 传输层 API 密钥 | `4lUvNRfG...` |

### 技能不覆盖的功能

以下功能需要在 iHRFlow 后台操作，AI 助手暂不支持：

- 用户/角色/租户管理
- 笔试题库管理与评分
- 简历附件上传与批量导入
- 系统配置与权限设置
- Offer 审批与发放

### 安全说明

- 你的密码和 API Key 在 OpenClaw 配置中加密存储，CLI 查询时显示为 `__OPENCLAW_REDACTED__`
- 如需更高安全性，可启用 OpenClaw 内置的 `apikeys-ui` Vault 技能，将敏感变量存入 `~/.openclaw/secrets.json`（权限 0600），配置中仅保留引用
- MCP 服务器支持 API Key 传输层认证 + 用户密码登录双重验证
- 每个用户的会话独立，JWT Token 自动管理和刷新
- 多租户数据完全隔离
