# Code Hug - 使用示例

## 基本工作流编排
```
为我的 Java Spring Boot 项目启动一个新的 Code Hug 会话，并启用商业智能提取功能。
```

## 商业智能重点
```
使用 Code Hug 从这个遗留的 PHP 代码库中提取业务规则并生成 PRD 文档。
```

## 针对性分析
```
专门对支付处理模块运行 Code Hug，重点关注工作流映射和安全分析。
```

## 智能诊断
```
诊断为什么我的 Node.js 应用程序构建失败，并尝试自动修复。
```

## 恢复之前的会话
```
从会话 payment-service-001 停止的地方继续 Code Hug 分析。
```

## 预期输出结构

当 Code Hug 完成分析后，您将收到涵盖以下内容的综合交付物：

### 商业智能成果物
- **业务规则目录**: 提取的业务规则，包含实现细节和验证场景
- **PRD文档**: 功能规格、用户故事、验收标准和非功能性需求
- **工作流图**: 端到端业务流程、状态转换和集成点
- **数据模型图**: 实体关系、数据流和模式演进历史

### 技术评估报告
- **执行摘要**: 关键发现、风险评估和现代化路线图
- **架构分析**: 组件关系、依赖图和技术栈评估
- **安全审计**: 带严重性评级和修复指导的漏洞分析
- **性能概况**: 瓶颈识别和优化机会
- **技术债务清单**: 维护挑战和优先级改进建议
- **质量门禁状态**: 各阶段验证结果和合规状态

### 工作流管理成果物
- **会话审计轨迹**: 完整的决策日志和操作历史
- **阶段产出物**: 每个工作流阶段产生的所有成果物
- **集成验证结果**: 测试结果和验证报告
- **部署准备包**: 生产就绪的部署成果物和检查清单

完整的分析结果存储在您工作区中的 `.code-hug/` 目录中，供将来参考、增量更新和审计使用。

## 多语言支持

Code Hug 支持跨多种技术栈的全面分析：
- **Java**: 企业应用、微服务、Android 项目
- **JavaScript/TypeScript**: Web 应用、Node.js 后端、React/Vue 前端  
- **Python**: 数据科学项目、Web 应用、自动化脚本
- **PHP**: 遗留企业系统、现代框架
- **混合项目**: 多语言架构和微服务

启动会话时只需指定您的技术栈，Code Hug 就会自动配置相应的分析工具和工作流。
