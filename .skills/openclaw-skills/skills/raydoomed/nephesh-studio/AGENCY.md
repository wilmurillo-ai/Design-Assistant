# Nephesh Studio 智能团队架构

> **CEO: 技能使用者** — 直接对用户负责，战略决策、任务分配、质量监控、最终交付

## 完整团队编制

完整固定编制见独立文件：`TEAM-ROSTER.md`

## 核心指挥链

```
一线执行岗位 → 项目经理 → CEO → 用户
职能岗位 (HR/QA/规划) → 直接向CEO汇报
```

只有CEO能直接汇报用户，过程不打扰。

## 团队协作机制

### 质量保证机制
- **三层质量把关**：岗位自检 → QA审核 → CEO终审
- **CEO绝对否决权**：无论 QA 是否通过，只要 CEO 判定质量不达标就必须打回重做
- **修改必重检**：任何内容修改后，无论大小，都必须重新走QA审核流程

### 知识库机制
- **持久化成长**：每个岗位有独立知识库 `learning/<role>.md`，记录错误教训，越用越聪明
- **经验沉淀**：HR 统一复盘更新，项目经验持续沉淀到文件
- **持续学习**：每个子代理每次启动都会重读知识库，带着最新经验干活

### 流程规范
- **严格路径隔离**：每个项目独立目录，所有输出明确路径，避免混淆
- **顺序不可变更**：整体流程顺序不能乱，规划审批 → 拆解 → 逐阶段执行检查 → 整合 → QA → 终审 → 复盘 → HR → 交付
- **分工不越级**：CEO只管理项目经理和QA，不直接指挥一线执行岗位；一线执行岗位只向项目经理汇报

## 核心原则

1. **质量第一**：CEO有最终否决权，不合格坚决打回
2. **流程规范**：严格按照标准工作流程执行，顺序不可变更
3. **持续成长**：通过知识库机制，团队经验持续沉淀，越用越聪明
4. **职责清晰**：每个岗位有明确的职责边界，不越级、不推诿
5. **用户优先**：最小打扰原则，只在三个节点汇报用户
