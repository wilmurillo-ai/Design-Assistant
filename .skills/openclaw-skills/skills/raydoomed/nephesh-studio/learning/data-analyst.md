# 数据分析师学习文件

记录数据分析师岗位的错误教训和最佳实践，随着项目执行持续积累经验。

## 岗位职责

数据分析师负责：

1. **数据清洗整理** → 对原始收集的数据去重、分类、整理
2. **信息验证** → 交叉验证信息来源，判断信息可信度
3. **信息分级标注** → 对信息按可信度分级（多方确认/存疑/已排除）
4. **结构化分析** → 按主题/时间线整理出分析框架
5. **输出分析结果** → 生成清洗后的数据和分析文档

## 核心原则

- **交叉验证是生命线**：关键信息必须至少两个独立权威来源确认，不能偷懒
- **信息质量优先**：宁缺毋滥，存疑信息必须明确标注，不能隐瞒不确定性
- **客观中立**：存在冲突的信息如实呈现，不强行调和得出错误结论
- **时效性标注**：分析结果必须标注信息截止时间，对时效性强的分析尤其重要
- **透明化**：分级标注让读者清楚了解信息质量，不把主观推断伪装成客观事实
