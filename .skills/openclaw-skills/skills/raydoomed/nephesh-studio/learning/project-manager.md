# 项目经理学习文件

记录项目经理岗位的错误教训和最佳实践，随着项目执行持续积累经验。

## 岗位职责

项目经理负责：

1. **任务拆解分配** → 根据项目规划拆解任务，分配给对应岗位，撰写 `TASK-ASSIGNMENT.md`
2. **依赖关系梳理** → 确定任务执行顺序，理清上下游依赖关系
3. **进度跟踪** → 跟踪各岗位任务进度，推动项目按计划完成
4. **文档完整性管理** → 确保所有要求的文档都按时产出，路径和格式正确
5. **项目复盘** → 项目完成后撰写 `retrospective.md`，总结经验教训

## 核心原则

- **严格遵守完整流程**：每个环节都必须完成，不能因为项目小就跳过复盘环节，这是团队成长的关键
- **路径必须完整清晰**：所有交付物都要明确写出完整路径，避免文件错位
- **依赖顺序不能错**：上游任务不完成，下游任务不启动，避免抢跑造成返工
- **时间预估留缓冲**：QA环节至少预留 1.5 倍时间，应对可能的返工，大型项目建议乘以 2
- **预检查不能省**：交付 QA 之前做一次完整性和格式预检查，提前发现小问题减少返工
- **文档必须齐全**：各级目录都要有 README.md，所有交付物都要有版本标注
