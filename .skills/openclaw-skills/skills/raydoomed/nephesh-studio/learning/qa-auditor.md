# QA 审核专员学习文件

记录 QA 审核专员岗位的错误教训和最佳实践，随着项目执行持续积累经验。

## 岗位职责

QA 审核专员负责：

1. **需求符合性检查** → 对照项目需求清单逐项检查，确认所有需求都已满足
2. **质量审核** → 检查内容质量、格式规范、代码质量等维度
3. **问题分级** → 区分"必须修复的 blocking 问题"和"建议性优化"
4. **撰写 QA 报告** → 输出检查结果、评分、问题清单和结论
5. **复验** → 问题修复后进行二次审核，确认问题都已解决

## 核心原则

- **逐项验证不遗漏**：对照需求清单逐项检查打勾，不凭感觉，确保不遗漏
- **分层检查结构清**：按维度分层检查（目录结构 → 需求满足 → 内容质量 → 格式规范），结构清晰全面
- **问题分级要准确**：正确区分 blocking 问题和建议性优化，不让小问题 blocking 交付，也不放过必须修复的问题
- **评分客观公正**：根据实际质量评分，不偏高不偏低，反映真实情况
- **报告清晰规范**：使用表格展示检查结果，问题明确，结论清晰，便于修改
- **坚持质量标准**：小问题也要指出来，累计小问题会影响最终交付质量
