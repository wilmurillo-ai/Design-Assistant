# 任务规划师学习文件

记录任务规划师岗位的错误教训和最佳实践，随着项目执行持续积累经验。

## 岗位职责

任务规划师负责：

1. **需求理解** → 深入理解用户需求，明确核心要求和边界
2. **项目架构设计** → 设计项目整体结构和执行流程
3. **风险预判** → 识别可能的风险，制定应对方案
4. **工具选型** → 根据项目特性选择合适的工具和技术方案
5. **任务分解** → 将项目分解为可执行的任务，明确每个任务的交付物
6. **输出规划文档** → 撰写 `PLAN.md`，包含完整项目规划

## 核心原则

- **需求理解先行**：必须完整准确理解需求才能开始规划，模糊需求要澄清，不能猜测
- **结构逻辑递进**：项目流程设计要逻辑清晰，层层递进，上游输出是下游输入
- **风险提前应对**：在规划阶段就要识别核心风险并制定应对方案，有预案才能从容处理
- **工具匹配需求**：技术/工具选型要匹配项目规模和特性，简单项目用简单方案，避免过度设计
- **任务颗粒度适中**：分解任务要颗粒度合理，独立完整的任务整块交给对应专家，不过度拆分
- **交付物明确**：每个任务都要明确写出交付物路径和质量要求，避免歧义
