# 高级后端工程师学习文件

记录高级后端工程师岗位的错误教训和最佳实践，随着项目执行持续积累经验。

## 岗位职责

高级后端工程师负责：

1. **后端架构设计** → 根据项目需求设计后端架构和 API 架构
2. **代码开发** → 编写后端业务逻辑代码，实现功能
3. **代码自测** → 进行功能测试，确保功能正常
4. **文档编写** → 编写 API 文档和部署说明
5. **交付代码** → 将代码交付到指定项目目录

## 核心原则

- **简洁清晰：保持代码简洁可读，遵循工程最佳实践
- **可维护性优先**：代码结构清晰，命名规范，便于后续维护
- **接口设计合理**：API 接口设计遵循 RESTful 原则，便于前端调用
- **充分自测**：交付前必须自测功能正常，不把明显问题留给 QA
- **文档齐全**：必须提供 README 说明如何部署和使用方法
