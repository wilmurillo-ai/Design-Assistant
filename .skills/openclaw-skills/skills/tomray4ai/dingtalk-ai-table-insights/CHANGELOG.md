# dingtalk-ai-table-insights 更新日志

## v1.6.10 (2026-03-02) - 抽样逻辑优化

### 🎯 核心改进

**抽样逻辑完全优化** - 修复数据抽样策略，确保大模型接收完整数据，同时控制 token 消耗。

### ✨ 修复内容

1. **数据表抽样** - 每表最多处理 10 个数据表
2. **记录数抽样** - 每数据表最多随机抽 100 条（读取阶段）
3. **字段数限制** - 每条记录默认取前 8 个字段（适配不同表格结构）
4. **移除固定字段限制** - 不再强制限定 8 个字段名，自动取前 8 个

### 📊 抽样策略

| 阶段 | 策略 | 数值 |
|------|------|------|
| 数据表数量 | 每表最多 | 10 个 |
| 每表记录数 | 每数据表最多 | 100 条（随机） |
| 每记录字段数 | 每条记录最多 | 8 个（前 8 个） |
| 总数据量 | 最多 | ~1000 条 |

### 🔄 对比

| 版本 | 字段限制 | 数据利用率 | 适配性 |
|------|---------|-----------|--------|
| v1.6.9 | 固定 8 个字段名 | 可能遗漏关键字段 | 依赖表格结构 |
| **v1.6.10** | 自动取前 8 个字段 | ✅ 充分利用数据 | ✅ 适配任意表格 |

**大模型分析完全修复** - 修复 `openclaw agent` 调用方式，发送详细数据样本，实现完整的 LLM 深度分析。

   - 降级处理：LLM 失败时自动使用本地模板

## v1.3.0 (2026-02-28) - 大模型分析支持

### 🎉 主要更新

**默认启用大模型分析** - 脚本现在默认使用大模型进行深度分析，提供更智能的洞察和建议。

### ✨ 新功能

1. **大模型分析（默认启用）**
   - 自动尝试通过 OpenClaw 会话调用大模型
   - 支持 MCP 大模型服务器配置
   - 多级降级策略，确保分析总能完成

2. **--no-llm 参数**
   - 可手动选择使用本地模板生成报告
   - 适用于快速分析或大模型不可用的场景

### 🔄 调用策略

大模型调用按以下优先级尝试：

1. **OpenClaw 会话** - 如果在 OpenClaw 会话中运行，优先使用 sessions_send
2. **MCP 大模型服务器** - 如果配置了 MCP 大模型服务器（如 qwen、chat 等）
3. **本地模板** - 降级方案，确保分析总能完成

### 📝 使用示例

```bash
# 默认使用大模型分析
python3 scripts/analyze_tables.py --keyword "销售"

# 使用本地模板（不使用大模型）
python3 scripts/analyze_tables.py --keyword "销售" --no-llm

# 全量扫描
python3 scripts/analyze_tables.py
```

### 🔧 技术改进

- 优化 prompt 构建，增加数据样本量（3 条 → 5 条）
- 添加临时文件支持，避免命令行长度限制
- 改进错误处理和日志输出
- 添加降级提示，引导用户配置大模型

### 📊 对比

| 特性 | 本地模板 (v1.2) | 大模型分析 (v1.3) |
|------|----------------|------------------|
| 数据一致性检查 | ❌ 简单规则 | ✅ 智能跨表对比 |
| 趋势洞察 | ❌ 无 | ✅ 发现关联模式 |
| 风险识别 | ⚠️ 基础规则 | ✅ 上下文感知 |

| 报告流畅度 | ⚠️ 机械化 | ✅ 自然语言 |
