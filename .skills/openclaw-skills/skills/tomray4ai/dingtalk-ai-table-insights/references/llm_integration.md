# 🤖 大模型集成说明

**数据如何送给大模型分析**

---

## 📊 数据处理流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 获取表格列表                                              │
│     search_accessible_ai_tables(keyword="仪表盘")             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 读取表格数据（分页读取）                                   │
│     read_all_sheets_data(doc_id, limit=50)                   │
│     - 每个数据表最多读取 50 条                                  │
│     - 使用 cursor 分页                                        │
│     - 临时文件避免 JSON 截断                                   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 数据预处理（精简和抽样）                                   │
│     - 统计每个表格的数据表数量                                │
│     - 统计每个数据表的记录数                                  │
│     - 每个表格抽样 3 条记录作为示例                             │
│     - 提取关键字段（标题、优先级、状态等）                      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 构建 Prompt（系统提示 + 用户数据）                          │
│     System: 你是数据分析专家...                               │
│     User: 请分析以下数据...                                   │
│     Data: JSON 格式的表格摘要                                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  5. 调用大模型（3 种方式，自动降级）                             │
│     方法 1: OpenClaw sessions_spawn（优先）                    │
│     方法 2: OpenClaw CLI（备用）                              │
│     方法 3: 本地模板生成（回退）                               │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  6. 输出报告（Markdown 格式）                                  │
│     - 洞察报告：业务分析（给用户看）                           │
│     - 执行日志：技术细节（给开发看）                           │
└─────────────────────────────────────────────────────────────┘
```

### 步骤 3: 数据预处理（精简）

**目的**: 减少 token 消耗，保留关键信息

```python
data_summary = []
for table in tables_data:
    table_info = {
        "表格名称": "仪表盘需求表",
        "数据表数": 2,
        "总记录数": 185,
        "数据表详情": [
            {"数据表名称": "反馈整理表", "记录数": 92},
            {"数据表名称": "汇报", "记录数": 93}
        ],
        "数据示例": [
            {"标题": "词云优化", "优先级": "P1"},
            {"标题": "组合布局移动端适配", "优先级": "P0"},
            {"标题": "图表联动功能", "优先级": "P0"}
        ]
    }
    data_summary.append(table_info)
```

**精简策略**:
1. ✅ 保留统计信息（表格数、记录数）
2. ✅ 保留 3 条示例记录
3. ✅ 提取关键字段（标题、优先级、状态）
4. ❌ 移除详细字段（创建时间、UID 等）
5. ❌ 移除完整记录（避免 token 超限）

### 步骤 4: 构建 Prompt

**System Prompt**（系统提示）:
```
你是一个数据分析专家，擅长从企业数据中发现洞察和风险。

## 分析维度
1. 数据一致性检查 - 跨表格对比相同指标，发现矛盾
2. 趋势洞察 - 从多个表格中发现关联和模式
3. 风险预警 - 识别异常和高风险项（按优先级排序）
4. 行动建议 - 给出具体可执行的建议（做什么 + 谁来做 + 何时完成）

## 输出格式
- 使用 Markdown 格式
- 适当使用 emoji 增强可读性
- 字数控制在 800-1500 字
- 适合在钉钉中查看
- 不要提及技术细节（如权限问题、API 错误等）
```

**User Prompt**（用户数据）:
```
请分析以下钉钉 AI 表格数据，生成洞察分析报告。

## 分析范围
- 筛选关键词：仪表盘
- 分析表格数：5 个
- 总记录数：956 条

## 表格数据摘要
[
  {
    "表格名称": "AI 表格专题直播：仪表盘",
    "数据表数": 3,
    "总记录数": 612,
    "数据表详情": [...],
    "数据示例": [...]
  },
  ...
]

## 请生成报告，包含以下内容
1. 执行摘要（关键指标）
2. 详细数据分析（每个表格的数据表详情、示例、洞察）
3. 风险与异常识别（自动发现数据问题）
4. 行动建议（具体可执行，包含优先级和时间）

请开始生成报告：
```

### 步骤 5: 调用大模型（3 种方式）

#### 方法 1: OpenClaw sessions_spawn（优先）

```python
from openclaw import sessions_spawn

result = sessions_spawn(
    task=full_prompt,
    timeoutSeconds=120,
    cleanup="delete"
)
report = result
```

**优点**:
- ✅ 原生集成
- ✅ 异步执行
- ✅ 自动清理

---

#### 方法 2: OpenClaw CLI（备用）

```python
import subprocess

cmd = ['openclaw', 'ask', full_prompt]
result = subprocess.run(cmd, capture_output=True, text=True, timeout=120)
report = result.stdout.strip()
```

**优点**:
- ✅ 不依赖 Python API
- ✅ 适用于所有环境

---

#### 方法 3: 本地模板生成（回退）

```python
report = generate_insight_report(tables_data, keyword)
```

**优点**:
- ✅ 无需大模型
- ✅ 快速生成

**缺点**:

- ❌ 基于规则生成

## 🔒 数据安全

### 数据脱敏

**脱敏字段**:
- ✅ 用户 UID → 保留（用于统计）
- ✅ 创建时间 → 保留（用于趋势）
- ✅ 标题/内容 → 保留（用于分析）

**不脱敏原因**:
- 数据仅用于当次分析
- 不存储到外部服务
- 本地处理，不传输

### Token 限制

**优化策略**:
1. 每个表格只抽样 3 条记录
2. 只提取关键字段（5-8 个）
3. 使用 JSON 压缩（去除空白）
4. 限制总记录数（<1000 条）

**Token 估算**:
- System Prompt: ~200 tokens
- User Prompt: ~100 tokens
- 数据摘要：~500-1000 tokens（取决于数据量）
- **总计**: ~800-1300 tokens
