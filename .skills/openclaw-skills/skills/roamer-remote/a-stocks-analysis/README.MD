## 功能一览
- **特色功能**：
  1. 持续集成各路专家的选股分析方法，也支持定制私有选股策略，可对个股或市场热点股进行全面分析。
  2. 分析要求可通过自然语言描述，覆盖并不限于个股、行业、整体市场面及其他相关维度。
  3. 支持从“行情 -> 基本面 -> 指数/文本”的组合分析链路，帮助从数据查询过渡到结构化判断。
- **行情查询**：日线/周线/月线、每日指标、资金流向、复权因子
- **基本面分析**：公司资料、财务报表（利润表/资产负债表/现金流）、财务指标、分红送转、股东信息
- **指数数据**：沪深300、上证指数、深证成指等指数的行情、成分股、权重
- **文本数据**：董秘问答（上证E互动/深证互动易）、政策法规、研报元数据

## 常见使用场景

### 0. 推荐分析工作流（先做这个）

当你希望得到更稳定的分析结论时，建议按以下顺序提问和调用：

1. **先看行情（价格与量能）**  
   先确认趋势、波动和交易活跃度，避免脱离走势谈基本面。
2. **再看基本面（质量与估值）**  
   结合 ROE、毛利率、利润表等判断“涨跌背后是否有基本面支撑”。
3. **最后做指数/文本交叉验证（环境与叙事）**  
   用指数强弱、成分股结构、董秘问答/研报元数据确认市场环境与公司叙事是否一致。

> 这不是投资建议，而是一个通用的数据分析顺序模板。

### 1. 查询股票行情

| 你想问 | 示例问题 |
|--------|----------|
| 最近股价走势怎么样 | "帮我看看贵州茅台最近一个月的走势" |
| 今天/历史收盘价 | "招商银行上周五收盘价是多少" |
| 周线/月线K线 | "平安银行近半年月线行情" |
| 涨跌停价格 | "中金公司今天的涨跌停价格" |

**调用示例**

```
# 日线行情
GET /stocks/daily?ts_code=600519.SH&start_date=20250301&end_date=20250401

# 股票列表查询
GET /stocks/stock-basic
```

### 2. 分析基本面

| 你想问 | 示例问题 |
|--------|----------|
| 公司概况 | "中国平安这家公司是做什么的" |
| 财务指标 | "比亚迪的ROE和毛利率怎么样" |
| 财报数据 | "万科近三年利润表" |
| 分红送转 | "格力电器最近一次分红是什么时候" |
| 股东结构 | "茅台的前十大股东有哪些" |

**调用示例**

```

GET /stocks/fina-indicator?ts_code=000001.SZ

# 利润表
GET /stocks/income?ts_code=601318.SH&start_date=20230101&end_date=20241231

# 分红送股
GET /stocks/dividend?ts_code=000001.SZ
```

### 3. 指数分析

| 你想问 | 示例问题 |
|--------|----------|
| 大盘点位 | "沪深300现在多少点" |
| 指数成分股 | "沪深300有哪些股票" |

**调用示例**

### 4. 上市公司互动问答

| 你想问 | 示例问题 |
|--------|----------|
| 公司公告 | "华为概念股最近有什么董秘问答" |
| 投资者关系 | "隆基绿能最近在上证E互动回复了哪些问题" |

**调用示例**

```
GET /stocks/irm-qa-sh?ts_code=600438.SH&start_date=20250301
```

### 5. 问题到接口路由（快速对照）

| 你想解决的问题 | 建议优先接口组合 |
|--------|----------|
| 最近走势强不强 | `GET /stocks/daily` + `GET /stocks/daily-basic` |
| 公司质量有没有改善 | `GET /stocks/fina-indicator` + `GET /stocks/income` |
| 行情上涨是否有行业共振 | `GET /stocks/daily` + `GET /stocks/index-daily` + `GET /stocks/index-member` |
| 热点是否有公司层面信号 | `GET /stocks/irm-qa-sh`（或深市对应接口）+ 行情接口 |
| 分红能力是否稳定 | `GET /stocks/dividend` + `GET /stocks/income` |

## 股票代码怎么写

支持两种格式：

| 格式 | 示例 | 说明 |
|------|------|------|
| 裸码 | 600000 | 自动识别交易所 |
| 标准格式 | 600000.SH | `.SH` 沪市，`.SZ` 深市，`.BJ` 北交所 |

> 代码前导0不可省略，如平安银行是 000001，不是 1。

---

## 日期格式

所有日期参数使用 `YYYYMMDD` 格式：

| 参数 | 说明 | 示例 |
|------|------|------|
| start_date | 起始日期 | 20250101 |
| end_date | 结束日期 | 20250401 |

## 常见问题

**Q: 数据是实时的吗？**
A: 提供的是 Tushare 行情源，通常收盘后当晚更新。不保证实时涨跌。

**Q: 能查港股吗？**
A: 支持港股代码（需加 .HK 后缀，如 00700.HK）。

**Q: 缺少 TAX_API_KEY 怎么办？**
A: 访问 https://tax.yyyou.top/ 微信扫码创建授权码，然后配置到你的 Agent 工具。

**Q: 查询失败怎么办？**
A: 检查：1) 股票代码是否正确；2) 日期是否在交易日内；3) API Key 是否有效。可稍后重试。

### 常见失败样例（含修正）

**样例1：股票代码不规范**

错误示例：
```
GET /stocks/daily?ts_code=1
```

修正示例：
```
GET /stocks/daily?ts_code=000001.SZ
```

**样例2：日期格式错误**

错误示例：
```
GET /stocks/daily?ts_code=600519.SH&start_date=2025-03-01
```

修正示例：
```
GET /stocks/daily?ts_code=600519.SH&start_date=20250301
```

**样例3：起止日期颠倒**

错误示例：
```
GET /stocks/daily?ts_code=600519.SH&start_date=20250401&end_date=20250301
```

修正示例：
```
GET /stocks/daily?ts_code=600519.SH&start_date=20250301&end_date=20250401
```

## 最小可用案例（MVP）

目标问题：**“贵州茅台最近一个月上涨是否有基本面支撑？”**

建议最小链路：

1. 看走势  
   `GET /stocks/daily?ts_code=600519.SH&start_date=20250301&end_date=20250401`
2. 看财务指标  
   `GET /stocks/fina-indicator?ts_code=600519.SH`
3. 看利润趋势（可选）  
   `GET /stocks/income?ts_code=600519.SH&start_date=20230101&end_date=20241231`

输出结论时建议包含三句：

- **行情观察**：最近一个月涨跌幅、波动、阶段高低点。
- **基本面观察**：ROE/毛利率/利润趋势是否支持当前走势。
- **一致性判断**：价格变化与基本面是否同向，若背离需提示风险。
