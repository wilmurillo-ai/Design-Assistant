## 二、维度详解

### 2.1 完整性评估 (Completeness)

**评估目的**：检查商业模式画布9大模块是否都有内容

**评分标准**：

| 模块 | 分值 | 评分标准 |
|------|------|----------|
| 客户细分 | 2分 | 有明确的客户群体描述 |
| 价值主张 | 3分 | 有清晰的价值描述 |
| 渠道通路 | 2分 | 有具体的渠道说明 |
| 客户关系 | 2分 | 有关系类型描述 |
| 收入来源 | 3分 | 有明确的收入方式 |
| 核心资源 | 2分 | 有资源清单 |
| 关键业务 | 2分 | 有业务描述 |
| 重要合作 | 2分 | 有合作伙伴说明 |
| 成本结构 | 2分 | 有主要成本项 |
| **合计** | **20分** | |

**检查清单**：
- [ ] 客户细分是否具体（非"所有人"）
- [ ] 价值主张是否清晰（非空泛描述）
- [ ] 收入来源是否明确（非"赚钱"）
- [ ] 成本结构是否有主要项目
- [ ] 各模块内容是否有实际意义

---

### 2.2 一致性评估 (Consistency)

**评估目的**：检查各模块之间逻辑是否自洽

**评分标准**：

| 检查项 | 分值 | 通过标准 |
|--------|------|----------|
| 客户-价值匹配 | 4分 | 价值主张与客户细分匹配 |
| 价值-渠道匹配 | 4分 | 渠道能传递价值主张 |
| 客户-收入匹配 | 4分 | 收入来源来自目标客户 |
| 资源-业务匹配 | 4分 | 资源支撑关键业务 |
| 成本-收入匹配 | 4分 | 收入能覆盖成本 |
| **合计** | **20分** | |

**常见不一致问题**：

| 问题 | 说明 | 示例 |
|------|------|------|
| 有收入无客户 | 定义了收入来源但没有客户细分 | "卖广告"但没有定义谁来看广告 |
| 有渠道无价值 | 定义了渠道但没有价值主张 | "通过抖音销售"但卖什么没说 |
| 价值客户不匹配 | 价值主张与客户需求不符 | 给价格敏感客户提供高端服务 |
| 资源业务不匹配 | 资源无法支撑业务 | 想做AI但没有技术团队 |
| 成本收入不匹配 | 收入无法覆盖成本 | 获客成本高于客户终身价值 |

---

### 2.3 可行性评估 (Feasibility)

**评估目的**：评估商业模式是否具备实施条件

**评分标准**：

| 检查项 | 分值 | 评估内容 |
|--------|------|----------|
| 资源可获取 | 5分 | 核心资源是否能获得 |
| 能力可建立 | 5分 | 关键业务是否能开展 |
| 合作可达成 | 5分 | 重要合作是否能建立 |
| 渠道可触达 | 5分 | 渠道通路是否可行 |
| **合计** | **20分** | |

**可行性检查清单**：

**资源可获取**：
- [ ] 资金是否充足
- [ ] 人才是否能招聘到
- [ ] 技术是否能获取
- [ ] 设备/设施是否能获得

**能力可建立**：
- [ ] 核心能力是否在团队能力范围内
- [ ] 是否需要大量学习/培训
- [ ] 是否有相关经验

**合作可达成**：
- [ ] 潜在合作伙伴是否愿意合作
- [ ] 合作条件是否合理
- [ ] 是否有备选方案

**渠道可触达**：
- [ ] 渠道成本是否可承受
- [ ] 渠道效率是否足够
- [ ] 是否有渠道资源

---

### 2.4 盈利性评估 (Profitability)

**评估目的**：评估商业模式的盈利潜力

**评分标准**：

| 检查项 | 分值 | 评估内容 |
|--------|------|----------|
| 收入规模 | 5分 | 收入天花板是否足够高 |
| 毛利率 | 5分 | 毛利空间是否足够 |
| 成本结构 | 5分 | 成本是否可控 |
| 盈利路径 | 5分 | 盈利逻辑是否清晰 |
| **合计** | **20分** | |

**盈利性分析框架**：

```
收入分析
├── 收入来源多样性（单一/多元）
├── 收入可预测性（一次性/经常性）
├── 收入规模潜力（市场规模 × 渗透率 × 客单价）
└── 定价能力（议价能力）

成本分析
├── 固定成本 vs 可变成本比例
├── 规模经济效应
├── 成本优化空间
└── 关键成本驱动因素

盈利路径
├── 单位经济模型（LTV > CAC）
├── 盈亏平衡点
├── 盈利时间线
└── 现金流状况
```

**关键指标**：
- **LTV/CAC ratio** > 3 为健康
- **毛利率** > 50% 为优秀（软件/SaaS）
- ** payback period** < 12个月为健康

---

### 2.5 差异化评估 (Differentiation)

**评估目的**：评估商业模式的竞争优势

**评分标准**：

| 检查项 | 分值 | 评估内容 |
|--------|------|----------|
| 独特性 | 5分 | 与竞争对手的差异程度 |
| 护城河 | 5分 | 竞争壁垒的高度 |
| 可持续性 | 5分 | 优势的可持续性 |
| 客户认知 | 5分 | 客户是否感知到差异 |
| **合计** | **20分** | |

**差异化来源**：

| 来源 | 说明 | 可持续性 |
|------|------|----------|
| **技术专利** | 独特的技术或专利 | 高（有保护期） |
| **品牌** | 强大的品牌认知 | 高（需持续投入） |
| **网络效应** | 用户越多价值越大 | 极高（赢家通吃） |
| **规模经济** | 规模带来的成本优势 | 中高 |
| **转换成本** | 客户转换的代价 | 中 |
| **渠道控制** | 独特的渠道资源 | 中 |

**竞争分析框架**：

```
竞争对手识别
├── 直接竞争对手（提供相同产品/服务）
├── 间接竞争对手（解决相同问题）
├── 替代品（完全不同的解决方案）
└── 潜在进入者（可能进入的新玩家）

差异化评估
├── 价值主张差异
├── 客户细分差异
├── 渠道差异
├── 成本结构差异
└── 收入模式差异
```

---

### 2.6 风险性评估 (Risk)

**评估目的**：识别和评估商业模式的潜在风险

**评分标准**：

| 风险等级 | 分值 | 说明 |
|----------|------|------|
| 低风险 | 16-20分 | 风险可控，有应对预案 |
| 中风险 | 11-15分 | 存在风险，需要关注 |
| 高风险 | 6-10分 | 风险较高，需要重点应对 |
| 极高风险 | 0-5分 | 风险极高，可能致命 |

**风险类型检查清单**：

**市场风险**：
- [ ] 市场需求是否真实存在
- [ ] 市场规模是否足够大
- [ ] 市场增长趋势如何
- [ ] 市场是否会被颠覆

**竞争风险**：
- [ ] 竞争对手反应如何
- [ ] 是否有巨头可能进入
- [ ] 价格战风险
- [ ] 技术替代风险

**运营风险**：
- [ ] 关键资源是否稳定
- [ ] 关键人才是否可保留
- [ ] 供应链是否稳定
- [ ] 质量风险

**财务风险**：
- [ ] 现金流风险
- [ ] 融资风险
- [ ] 汇率风险（如涉及）
- [ ] 坏账风险

**政策/法律风险**：
- [ ] 监管政策变化
- [ ] 合规风险
- [ ] 知识产权风险
- [ ] 数据安全风险

## 三、评估流程

### 3.1 评估步骤

```
步骤1：准备
├── 收集商业模式画布
├── 收集相关数据和信息
└── 确定评估目的

步骤2：完整性评估
├── 检查9大模块
├── 标记缺失内容
└── 计算完整性得分

步骤3：一致性评估
├── 检查模块间逻辑
├── 识别矛盾点
└── 计算一致性得分

步骤4：可行性评估
├── 评估资源可获取性
├── 评估能力可建立性
└── 计算可行性得分

步骤5：盈利性评估
├── 分析收入潜力
├── 分析成本结构
└── 计算盈利性得分

步骤6：差异化评估
├── 竞争分析
├── 护城河评估
└── 计算差异化得分

步骤7：风险评估
├── 识别风险点
├── 评估风险等级
└── 计算风险得分

步骤8：汇总报告
├── 计算总分
├── 生成评估报告
└── 提出改进建议
```
