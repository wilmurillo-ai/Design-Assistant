## 🔄 开发流程

### 新功能开发流程

```
1. 需求分析 (人类)
       ↓
2. Claude Code - 核心实现
       ↓
3. OpenCode - 测试覆盖
       ↓
4. Pi/Cursor - 文档完善
       ↓
5. 集成测试 (OpenCode)
       ↓
6. 代码审查 (Claude Code)
       ↓
7. 发布准备 (Pi/Cursor)
       ↓
8. 人类审核 + 发布
```

---

### Bug 修复流程

```
1. Bug 报告 (人类)
       ↓
2. Claude Code - 定位 + 修复
       ↓
3. OpenCode - 回归测试
       ↓
4. Pi/Cursor - 更新 CHANGELOG
       ↓
5. 人类审核 + 发布
```

---

### 版本发布流程

```
1. 代码冻结 (Claude Code)
       ↓
2. 测试验证 (OpenCode)
       ↓
3. 文档更新 (Pi/Cursor)
       ↓
4. 发布说明 (Pi/Cursor)
       ↓
5. GitHub 发布 (人类)
       ↓
6. ClawHub 发布 (人类)
```

## 🎓 最佳实践

### 代码开发

1. **小步提交**: 每次提交一个功能点
2. **测试先行**: 先写测试再写代码
3. **代码审查**: 每次修改都要审查
4. **文档同步**: 代码和文档同步更新

### 测试开发

1. **覆盖核心**: 优先测试核心功能
2. **边界条件**: 测试所有边界情况
3. **回归测试**: 每次修改后回归
4. **性能测试**: 关键功能性能基准

### 文档编写

1. **用户视角**: 从用户角度编写
2. **示例驱动**: 提供可运行示例
3. **更新及时**: 功能完成即更新
4. **安全说明**: 安全考虑明确说明
