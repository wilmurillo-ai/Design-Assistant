## 未来规划 (v1.3.0+)

### 架构改进

```
当前架构: 直接调用脚本
         ↓
建议架构: 状态机 + 队列调度

        ┌──────────────┐
 用户 ──▶│ 调度器(Scheduler) │◀── 健康检查
        └──────┬───────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
 备份队列   恢复队列    导出队列
    │          │          │
    ▼          ▼          ▼
 执行器     执行器      执行器
    │          │          │
    └──────────┼──────────┘
               ▼
          状态存储
```

**优势**:
- 支持并发任务控制
- 任务持久化 (中断后可恢复)
- 更好的错误隔离
- 支持异步操作

### 功能增强

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 恢复测试 | High | 验证备份可恢复性，不实际写入 |
| 定时任务 | High | 集成 cron/systemd 自动备份 |
| 备份加密 | Medium | 整包加密 (不只是配置文件) |
| 敏感脱敏 | Medium | 自动识别并脱敏敏感信息 |
| 云存储支持 | Medium | AWS S3, Azure Blob, GCS |
| Web UI | Low | 图形化管理界面 |
| 增量恢复 | Low | 仅恢复变更文件 |
