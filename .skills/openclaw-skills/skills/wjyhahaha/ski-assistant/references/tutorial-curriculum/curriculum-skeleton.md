# 教程动态生成协议

本文件定义 Agent 如何基于 `_index.md` 中的技能骨架动态组装教程内容。`modules/tutorial.md` 读取此协议后，按规则生成符合质量标准的教程。

---

## 架构总览

```
用户请求教程
    → modules/tutorial.md 读取用户 profile（板型、级别、风格偏好）
    → 读取 _index.md 获取级别定义、适配规则、技能包索引
    → 读取 levels/{board}-{level}.md 获取对应级别技能元数据
    → 按需读取 training.md / faq.md / gear.md 中对应级别部分
    → 读取 curriculum-skeleton.md（本文件）获取生成规则
    → 按本文件定义的模板和约束动态生成教程内容
    → 输出完整教程 Markdown
```

**核心原则**：规则控质量，Agent 生成内容。不预写全文，但严格控制输出格式和内容标准。

---

## 生成流程

### Step 1：确定输出范围

根据用户 `board_type` 和 `level` 确定：
- 读取 `levels/sb-{X}.md`（单板）或 `levels/ski-{X}.md`（双板）获取技能元数据
- 读取 `training.md` 中对应级别部分获取训练计划
- 读取 `faq.md` 中对应级别部分获取 FAQ 要点
- 读取 `gear.md` 中对应级别部分获取装备规则

### Step 2：生成教程结构

每个教程必须包含以下章节（按顺序）：

```markdown
# [板型] Level [X] — [名称] 教程

> 版本：v3.0 | 免责声明：AI 生成内容，不能替代专业教练指导

## 本级别适合你吗

[基于级别定义表生成自我评估引导]

## 训练计划

[基于 training_framework 生成天数、时长、技能顺序表格]

## 核心技能详解

[逐个技能生成，每个技能包含以下子章节]

### [技能名称]

**为什么重要**：[基于 why_important 展开为 2-3 句自然语言]

**怎么做**：[基于 action_guide_template 展开为可操作的分步指导]

**常见错误**：[基于 common_mistakes_template 展开，每个错误附带 1 句纠正建议]

**练习方法**：[基于 practice_template 展开为具体的练习步骤，包含数量和组数]

**成功信号**：[基于 success_signal 展开为 1-2 句可自检的描述]

[Level 2+ 核心技能附加术语对照（如果有 terms 字段）]

## 装备清单

[基于 gear_framework 生成按优先级分类的装备清单]

## 常见问题

[基于 faq_framework 生成自然语言问答，每个问题 3-5 句]

## 下一步

[基于级别递进关系生成进阶指引]
```

## 内容生成规则

### 规则 1：技能讲解必须完整覆盖模板

每个技能必须生成以下 5 个子章节，不可省略：
1. **为什么重要** — 将 `why_important` 展开为 2-3 句有说服力的自然语言，说明该技能与整体滑行能力的关系
2. **怎么做** — 将 `action_guide_template` 展开为分步指导（至少 3 步），包含身体部位的具体动作描述
3. **常见错误** — 将 `common_mistakes_template` 每个错误展开为「错误描述 + 后果 + 纠正方法」三段式
4. **练习方法** — 将 `practice_template` 每个练习展开为具体步骤，保留数量和组数信息
5. **成功信号** — 将 `success_signal` 展开为 1-2 句可自检的描述，让读者能明确判断是否达标

### 规则 2：语言风格

- 使用第二人称（你）直接对话
- 语气友好、专业、鼓励性，避免居高临下
- 技术术语首次出现时标注英文（如：反弓 Angulation）
- Level 0-1 不使用 CASI/CSIA 术语对照
- Level 2+ 核心技能在讲解末尾添加术语对照（从 `terms` 字段读取）

### 规则 3：训练计划表格格式

必须包含以下列：

| 阶段 | 技能 | 时长 | 权重 | 关键目标 |

- 阶段：Day 1 / Day 2 / ... 或 Phase 1 / Phase 2 / ...
- 技能：技能中文名称
- 时长：从 `duration_hours` 或 `duration_minutes` 读取
- 权重：从 `weight` 读取，显示为百分比
- 关键目标：基于技能 `why_important` 提炼为 1 句话

### 规则 4：装备清单格式

按优先级分类输出：
- **必须佩戴**（must_have）— 安全和基础体验必备
- **强烈推荐**（recommended）— 提升舒适度和安全性
- **考虑购买**（consider_buy / rental_or_buy）— Level 1+ 可考虑自购的装备
- **风格专精**（carving / freestyle / powder / mogul）— Level 3 按风格路线推荐

每个装备条目格式：`- [装备名称]：[1 句选择建议或注意事项]`

### 规则 5：FAQ 生成

- 基于 `answer_key_points` 展开为 3-5 句自然语言
- 语气共情，先肯定问题合理性，再给出具体建议
- 涉及安全风险的问题（如道外安全、摔倒）必须强调请专业教练
- 不涉及具体医疗建议，身体酸痛等问题给出一般性建议并注明"如持续不适请咨询医生"

### 规则 6：Level 3 风格路线处理

Level 3 教程按用户 `freestyle` 偏好（从 `user_profile.json` 读取）选择风格路线：
- `carving`：只生成刻滑路线技能（单板 L3-SB-04/05/06，双板 L3-SK-05/06/07）
- `park` / `freestyle`：只生成公园路线技能（L3-SB-07/08/09，仅单板）
- `freeride` / `powder`：只生成野雪路线技能（单板 L3-SB-10/11/12，双板 L3-SK-11/12/13）
- `mogul`：只生成蘑菇路线技能（L3-SK-08/09/10，仅双板）
- 未指定或 `all`：生成全部风格路线，但标注推荐优先级

## 质量检查清单

生成完成后必须自检：

- [ ] 所有技能都包含 5 个子章节（为什么重要、怎么做、常见错误、练习方法、成功信号）
- [ ] 技能顺序与 `skill_sequence` 一致
- [ ] 训练计划表格包含所有必需列
- [ ] 装备清单按优先级分类
- [ ] FAQ 覆盖该级别全部问题
- [ ] Level 2+ 核心技能包含术语对照
- [ ] 免责声明在文件顶部
- [ ] 版本标注在文件顶部
- [ ] 下一步章节指向正确的下一级别
- [ ] 全文无英文混用（除技术术语和术语对照外）

## 与旧版教程的对比要求

动态生成的教程内容质量不得低于旧版预写文件。具体标准：

1. **信息量**：每个技能的讲解不少于旧版对应技能的字数
2. **可操作性**：练习方法必须包含具体数量和组数（与旧版一致或更详细）
3. **安全性**：安全相关技能（安全摔倒、道外安全、缆车穿脱）的讲解不得简化
4. **术语准确性**：术语对照与旧版一致
5. **结构一致性**：8 个教程（4 级别 x 2 板型）结构完全一致

## 个性化适配

基于 `user_profile.json` 和 `records.json` 动态调整：

1. **级别匹配**：根据 `records.json` 最近一次评分的 `posture`、`turning`、`overall` 三项分数，自动匹配到对应级别
2. **弱项匹配**：找出评分最低的子项，通过技能的 `rubric_mapping` 字段映射到对应技能。例如用户 `turning: 刻滑质量 5.5/10`，映射到 L2-SB-04 刻滑入门
3. **风格偏好**：Level 3 根据 `freestyle` 字段选择风格路线
4. **历史进度**：如果有历史记录，在"下一步"中提及上次进度

### 弱项注入强制规则（有 records.json 时必执行）

当 `records.json` 存在且最近一次评分有任一维度低于当前级别目标分时，必须执行以下注入：

**Step 1：提取弱项数据**
- 从最近一条记录读取 `scores` 和 `issues` 数组
- 识别低于目标分的维度（对照 _index.md 各级别目标分数）
- 记录具体子项分数（如 `turning: 刻滑质量 5.5/10`）

**Step 2：匹配技能**
- 通过每个技能的 `rubric_mapping` 字段，将弱项子项映射到对应技能
- 一个弱项可能对应多个技能，优先匹配当前级别
- 如果当前级别无对应技能，向上或向下找最近级别

**Step 3：注入内容格式（在匹配技能的讲解中强制添加）**

在"为什么重要"之后、"怎么做"之前，插入以下个性化段落：

```markdown
> ⚡ **针对你的情况**：
> - 上次分析得分：[弱项维度] [子项名] [分数]/10，目标 ≥ [目标分]/10
> - 你的具体表现：[从 issues 数组中提取描述，如无则写"建议下次分析时关注此维度"]
> - 优先练这个：[说明该技能与弱项的因果关系，1-2 句]
```

**Step 4：训练计划标记**
- 弱项相关技能在训练计划表格中标注"⚡ 你的重点"
- 在训练计划末尾追加"重点突破路径"：说明从哪个基础技能练起、因果关系是什么

**Step 5：FAQ 优先展示**
- 与弱项相关的 FAQ 问题提前到最前面
- 在对应问题后标注"⚡ 你目前最需要关注的"

**无 records.json 时的降级**
- 跳过上述所有注入步骤
- 输出通用版教程，末尾提示"完成练习后发视频给 AI 教练打分，下次教程将基于你的实际表现个性化调整"

## 维护说明

1. 修改 `levels/*.md` 中的技能元数据后，无需修改本文件，生成内容自动更新
2. 修改 `training.md` / `faq.md` / `gear.md` 后，无需修改本文件
3. 如需调整生成格式（如增加新的子章节），修改本文件的「生成教程结构」部分
4. 如需新增质量检查项，修改「质量检查清单」部分
5. 每次修改本文件后，重新生成 8 个教程并对比旧版验证质量
