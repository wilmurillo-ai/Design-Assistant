# Module 4: Presale Monitoring — 早鸟预售

**触发词**：早鸟票、预售、雪季卡、季卡、什么时候买划算、帮我盯着、开售提醒、early bird, season pass, presale

**加载知识**：按需读取 `references/resorts-reference.md`（各雪场早鸟规律，仅读取目标雪场相关段落）

---

## MUST 步骤（不可跳过，跳过须告知用户原因）

### 单个雪场查询

1. **确认需求**：雪场名称、滑雪次数（预估）、预算范围
2. **提供历史规律**：从 resorts-reference.md 提取该雪场的历年早鸟时间线和价格规律
3. **搜索当季公告**：WebSearch 搜索 `"{雪场名} 早鸟票 {年份}"` / `"{雪场名} 季卡 预售"`
4. **性价比分析**：对比单次票 vs 次卡 vs 季卡的每滑次成本

### 批量查询（用户问"今年早鸟票情况"或"检查预售状态"）

1. **读取 watchlist**：读取 `~/.ski-assistant/watchlist.json`，如为空则告知用户"监听列表为空，说'帮我盯着XX'可添加"
2. **逐个搜索**：对每个雪场执行 WebSearch 搜索当季公告
3. **状态分组**：按搜索结果分为"已开售"和"未开始"两组，不可混合输出
4. **输出结构化报告**：按下方"输出格式"模板输出

**常见时间线参考**：
- 崇礼 4-6 月、东北 5-7 月、新疆 6-8 月
- Ikon/Epic Pass 3-4 月
- 日本 7-9 月

## 预售监听

**纯被动机制**：技能脚本不创建任何后台任务或定时任务。

1. 用户说"帮我盯着XX" → 保存到 `~/.ski-assistant/watchlist.json`：
   ```json
   {
     "resorts": [
       {
         "name": "万龙滑雪场",
         "keywords": ["万龙 早鸟", "Wanlong season pass"],
         "sources": ["雪场官网", "飞猪"],
         "notify_channels": ["im", "console"],
         "added_at": "2026-04-10T10:00:00+08:00",
         "last_status": "未开始",
         "last_found": null,
         "history": []
       }
     ],
     "last_check": null
   }
   ```

2. 检查方式（仅用户主动触发）：
   - "检查预售状态" → 读取 watchlist.json + WebSearch 查最新公告
   - 直接问"XX早鸟开始了吗" → WebSearch 搜索对应信息

3. 定时检查依赖 Agent 平台的 cron 调度能力，必须经用户明确同意后才能创建。

## 输出格式

### 场景 A：单个雪场查询

必须包含以下四部分：

**1. 历史规律**
该雪场历年早鸟开售时间线和价格规律（从 resorts-reference.md 提取）

**2. 当季公告**
搜索到的最新公告。如无则说明"未搜索到当季公告"，不可省略此节。

**3. 性价比对比表**

| 票种 | 总价 | 预估滑次 | 每滑次成本 | 适合场景 |
|------|------|---------|-----------|---------|
| 单次票 | ¥X | 1 | ¥X | 偶尔滑 1-2 次 |
| 次卡 | ¥X | N | ¥X/次 | 季内滑 3-8 次 |
| 季卡 | ¥X | 不限 | ¥X/次（按N次算） | 季内滑 10 次+ |

**4. 购买建议**
结合用户画像（水平、预估滑次、预算）给出具体建议

### 场景 B：批量查询（watchlist 全量检查）

必须包含以下四部分，顺序不可变：

**1. 已开售列表**（表格，按折扣力度从高到低排序）

| 雪场 | 预售内容 | 折扣力度 | 价格 | 截止时间 | 来源 |
|------|---------|---------|------|---------|------|
| 万龙 | 早鸟季卡 | -40% | ¥3500 | 2026-05-31 | 官网 |

如全部未开售，输出"暂无已开售雪场"，不可省略此节。

**2. 未开始列表**（表格，按预计开售时间从近到远排序）

| 雪场 | 预计开售时间 | 依据 | 监听产品 |
|------|------------|------|---------|
| 北大湖 | 2026年5月 | 历史规律（去年5月中旬） | 早鸟季卡 |

如全部已开售，输出"所有监听雪场均已开售"，不可省略此节。

**3. 时间线**（按月份排列，当年 4 月至次年 1 月）

```
2026年4月: 万龙（已开售）、太舞（已开售）
2026年5月: 北大湖（预计）、松花湖（预计）
2026年6月: 将军山（预计）、可可托海（预计）
2026年7月: Niseko（预计）
...
```

**4. 个性化建议**
结合用户画像（`~/.ski-assistant/user_profile.json`）给出：
- 根据水平推荐适合的票种（如高级用户适合季卡，新手适合次卡）
- 根据预估滑次计算最优选择
- 预算范围内的最优组合

### 场景 C：用户问"XX早鸟开始了吗"

直接回答"是/否"，附具体信息：
- 已开售 → 价格、折扣、购买链接/渠道
- 未开售 → 预计时间、依据、建议关注时间

## 价格趋势

用户手动记录价格追加到 `~/.ski-assistant/price_history.json`。查看时分析最低/最高/平均/当前价位，给出购买时机建议。
