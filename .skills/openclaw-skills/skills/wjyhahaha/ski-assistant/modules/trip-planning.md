# Module 1: Trip Planning — 行程规划

**触发词**：推荐雪场、去哪滑、帮我规划、去 XX 滑雪、滑雪攻略、recommend resort, where to ski, plan my trip

**加载知识**：按需读取 `references/travel-guide.md`（攻略模板）、`references/gear-guide.md`（装备）、`references/budget-templates.md`（预算）、`references/resorts-reference.md`（雪场软信息）。这 4 个文件无依赖关系，**可并行读取**。

**加载数据**：读取 `data/search_strategies.json` 获取当前区域的搜索策略（用于实时查价步骤）

---

## MUST 步骤（不可跳过，跳过须告知用户原因）

### 场景 A：信息不全（用户只说"周末去哪滑"）

1. **读取用户画像**：读取 `~/.ski-assistant/user_profile.json`
2. **缺失必填项时提问**：水平、出发城市、天数/日期、预算、单板/双板。缺一项问一项。用户明确说"不限"可跳过
3. 用户回答后，进入场景 B

**用户画像关键字段**：
| 字段 | 说明 | 示例值 |
|------|------|--------|
| `level` | 滑雪水平 | beginner / intermediate / advanced |
| `city` | 出发城市 | "北京" |
| `board_type` | 单板/双板 | "snowboard" / "ski" |
| `style` | 风格偏好（单板） | "carving"（刻滑）/ "park"（平花/公园）/ "freeride"（野雪）/ "all-mountain" |
| `style` | 风格偏好（双板） | "carving"（卡宾）/ "mogul"（蘑菇）/ "freeride"（野雪）/ "all-mountain" |
| `days` | 天数 | 2 |
| `budget` | 预算（元） | 2000 |
| `preferences` | 偏好标签 | ["粉雪", "夜滑", "陡坡"] |
| `avoid` | 避免项 | ["人多", "排队久"] |
| `snow_hours` | 累计雪时（小时） | 50 |
| `ski_days` | 上雪天数 | 10 |
| `mileage` | 累计滑行里程（km） | 100 |

### 场景 B：信息完整（直接出攻略）

1. **读取数据库并打分**：读取 `data/resorts_db.json`，合并 `~/.ski-assistant/custom_resorts.json`（如有）。对候选雪场用打分公式计算（见下方），取 top 3。不可主观判断替代公式
2. **尝试查实时价格**：运行 `python3 tools/price_api.py check`
   - flyai 可用：先执行 `keyword-search`（从 `data/search_strategies.json` 提取关键词）查飞猪套餐，再执行 `search-flight` 和 `search-hotel` 查交通住宿
   - flyai 不可用：WebSearch 搜索（标注"网络搜索参考价"）
3. **国际场景必调汇率**：涉及非 CNY 币种，运行 `python3 tools/exchange_rate.py <金额> <源币种> <目标币种>` 获取实时汇率（如 `python3 tools/exchange_rate.py 10000 JPY CNY`）。失败时用内置备用汇率并标注"参考汇率"
4. **检查数据 freshness**：读取 `data/resorts_db.json` 的 `_meta.updated`。超过 90 天未更新，输出时提醒用户

## 打分公式（必须逐项计算）

对每个候选雪场计算总分（满分约 100）：

| 维度 | 规则 | 分值范围 |
|------|------|---------|
| 基础分 | 固定 | 50 |
| 偏好匹配 | 每项 +8，封顶 4 项 | 0 ~ +32 |
| 板型适配 | 单板+公园雪场 +10，单板+无公园 -5 | -5 ~ +10 |
| 风格适配 | 刻滑+宽陡道 +10，平花+平地地形 +10，野雪+粉雪区 +10，不匹配 -5 | -5 ~ +10 |
| 距离 | <300km: +15, <800km: +8, <2000km: +3 | 0 ~ +15 |
| 时间适配 | 短途+远距: -10, 长途+远距: +3 | -10 ~ +3 |
| 预算适配 | 合预算: +12, 超30%: +4, 超更多: -8 | -8 ~ +12 |
| 经验适配 | 雪时/天数/里程与雪场难度匹配: +10，不匹配（新手去高级场或老手去初级场） -8 | -8 ~ +10 |
| 规模加分 | 落差/面积/雪道数加权 | 0 ~ +10 |
| 雪季适配 | 非雪季室内 +15, 南半球反季 +10, 过季 -30 | -30 ~ +15 |
| 避免项 | 每项 -12 | 每项 -12 |

输出时展示每个推荐雪场的**打分依据简述**（如"偏好+24（粉雪/夜滑/陡坡），距离+8（200km），总分94"），不可只给分数不解释。

## 输出格式

### 场景 A：信息不全（需提问）

必须包含以下两部分：

**1. 缺失信息清单**

| 缺失项 | 示例回答 | 用途 |
|--------|---------|------|
| 滑雪水平 | 初/中/高级 | 匹配雪道难度 |
| 单板/双板 | 单板/双板 | 推荐公园/刻滑道/野雪区 |
| 风格偏好 | 刻滑/平花/野雪 | 匹配雪道类型和地形 |
| 累计雪时 | 约50小时 | 评估实际经验，推荐匹配难度的雪道 |
| 上雪天数 | 约10天 | 判断熟练程度，推荐进阶或休闲雪场 |
| 滑行里程 | 约100km | 衡量体能和耐力，推荐合适规模雪场 |
| 出发城市 | 北京 | 计算距离/交通 |
| 天数/日期 | 周末2天 | 筛选合适雪场 |
| 预算范围 | ¥2000以内 | 过滤超预算选项 |

**2. 引导语**
"告诉我这些信息后，我会从 854 座雪场中用打分公式为你匹配最优选择。单板/双板、风格偏好（刻滑/平花/野雪）和累计雪时会影响推荐结果，帮你找到难度匹配的雪场。"
