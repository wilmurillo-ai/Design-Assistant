# Module 5B: Pre-Departure Reminder — 出行提醒

**触发词**：行前检查、出发提醒、装备清单、明天去滑雪、pre-departure, checklist, reminder, packing list

**加载知识**：必须读取 `references/gear-guide.md`（装备数据库）、`references/pre-departure-checklist.md`（行前检查模板）。这 2 个文件无依赖关系，**可并行读取**。

**加载数据**：读取 `~/.ski-assistant/user_profile.json`（用户画像）、`~/.ski-assistant/upcoming_trips.json`（行程记录，如不存在则初始化为空数组 `[]` 并询问用户行程信息）

---

## MUST 步骤（不可跳过，跳过须告知用户原因）

### 场景 A：用户主动触发行前检查（"我明天去滑雪了"、"帮我做行前检查"）

1. **读取行程信息**：读取 `~/.ski-assistant/upcoming_trips.json`，如无记录则询问用户目的地、出发日期、返回日期
2. **读取用户画像**：读取 `~/.ski-assistant/user_profile.json` 获取 level、board_type、style 等
3. **查询天气**：调用 Open-Meteo API 获取目的地山顶天气（必须传 elevation 参数）
4. **生成 checklist**：按下方"输出格式 - 场景 A"生成行前检查清单
5. **可选：设置提醒**：询问用户是否需要创建出发前一天+当天的定时提醒

### 场景 B：用户确认出行日期后自动创建提醒（从 trip-planning 模块流转）

1. 用户在 trip-planning 中确认了具体出行日期
2. 将行程信息保存到 `~/.ski-assistant/upcoming_trips.json`：
   ```json
   {
     "resort": "万龙",
     "city": "河北崇礼",
     "departure_date": "2026-04-15",
     "return_date": "2026-04-17",
     "created_at": "2026-04-11",
     "reminders_set": false
   }
   ```
3. 询问用户是否创建定时提醒任务（必须经用户明确同意）
4. 用户同意后，创建两个定时任务：
   - **出发前一天 20:00**：发送装备检查清单 + 天气速报
   - **出发当天早上 07:00**：发送最终确认清单 + 实时天气 + 交通提醒

### 场景 C：定时提醒触发（由 cron 调度执行）

1. 读取对应行程信息
2. 查询实时天气（Open-Meteo API）
3. 生成检查清单（同场景 A 格式）
4. 将结果发送到用户 IM 会话（如已连接）或在对话中输出

## 行前 Checklist 生成规则

根据以下维度动态生成检查清单：

| 维度 | 影响内容 |
|------|---------|
| 目的地气候 | 崇礼/东北需极寒装备，日本需防水装备，室内雪场精简 |
| 用户水平 | 初学者必加护腕/护臀，高级加野雪装备 |
| 板型 | 单板加护腕/护臀/护背，双板加雪杖 |
| 行程天数 | 决定内衣/袜子数量（参考 gear-guide.md 穿衣数量建议） |
| 是否国际 | 国际加签证、保险（含救援）、通讯、装备托运 |
| 交通方式 | 自驾加防滑链，公共交通加装备打包建议 |
| 天气预测 | 大风加面罩/护目镜，降雪加防水套，极寒加暖宝宝 |

## 输出格式

### 场景 A：用户主动触发行前检查

必须包含以下五部分，顺序不可变：

**1. 行程速览**

| 项目 | 信息 |
|------|------|
| 目的地 | 万龙滑雪场，河北崇礼 |
| 出发日期 | 2026-04-15（周三） |
| 返回日期 | 2026-04-17（周五） |
| 行程天数 | 2天1晚 |
| 山顶天气 | -8°C ~ -2°C，小雪，风力 3-4 级 |
| 穿衣建议 | 极寒模式：加厚保暖层+围脖+暖宝宝 |

如行程信息不全，输出"未找到你的行程记录，请告诉我目的地和出发日期"。

**2. 装备检查清单**（按类别分组，每项带勾选框）

根据用户画像和目的地气候动态生成，必须包含以下分类：

| 类别 | 物品 | 数量 | 已确认 | 备注 | 适合你因为 |
|------|------|------|--------|------|-----------|
| 安全必备 | 头盔 | 1 | □ | CE/ASTM 认证 | 你说过安全第一 |
| 安全必备 | 滑雪专项险 | 2天 | □ | 含救援 | 必买！普通旅游险不含滑雪 |
| 安全必备 | 滑雪镜 | 1 | □ | 阴天用增光镜片 | 预报有小雪， visibility 低 |
| 服装 | 防水滑雪外套 | 1 | □ | 防水 10000mm+ | 崇礼-8°C，需防风防水 |
| 服装 | 速干内衣 | 3件 | □ | 聚酯纤维，非棉质 | 2天行程，备1件替换 |
| 服装 | 滑雪袜 | 3双 | □ | 长筒压缩款 | 不要穿两双影响血循环 |
| 服装 | 保暖中间层 | 2件 | □ | 抓绒/轻薄羽绒 | -8°C 需加强保暖 |
| 服装 | 围脖/面罩 | 1 | □ | 防风款 | 崇礼风寒效应明显 |
| 护具 | 护腕 | 1对 | □ | 单板必戴 | 你是单板，保护手腕 |
| 护具 | 护臀/小乌龟 | 1 | □ | 初学者推荐 | 你是初学，减少摔跤痛苦 |
| 配件 | 暖宝宝 | 1包 | □ | 贴手套/鞋里 | -8°C 极寒必备 |
| 配件 | 防晒霜 SPF50+ | 1 | □ | 高海拔紫外线强 | 雪地反射紫外线更强 |
| 电子 | 运动相机 | 1 | □ | GoPro/Insta360 | 方便 AI 教练分析动作 |
| 电子 | 充电宝 | 1 | □ | 低温耗电快 | 崇礼低温手机掉电快 |

**物品数量规则**：
- 速干内衣：按 1件/天 + 1件备用
- 滑雪袜：按 1双/天 + 1双备用
- 中间层：1-2 天带 1 件，3 天+ 带 2 件
- 暖宝宝：崇礼/东北/新疆必带，北京周边/室内可不带

**3. 证件与文档**

| 物品 | 状态 | 备注 |
|------|------|------|
| 身份证/护照 | □ | 国内带身份证，国际带护照+签证 |
| 保险保单 | □ | 电子保单截图存手机 |
| 雪票凭证 | □ | 电子票截图或纸质票 |
| 酒店预订单 | □ | 确认含雪具寄存 |
| 紧急联系人 | □ | 存 1-2 个紧急联系人电话 |

国际目的地额外增加：
| 签证 | □ | 确认有效期 |
| 国际保险（含救援） | □ | 确认覆盖滑雪和医疗转运 |
| 通讯（SIM/漫游） | □ | 确认能接打电话 |
| 信用卡/现金 | □ | 部分雪场只收现金 |

**4. 交通确认**

| 项目 | 状态 | 备注 |
|------|------|------|
| 交通方式 | □ | 自驾/大巴/高铁 |
| 路况查询 | □ | 出发前查实时路况 |
| 停车信息 | □ | 确认雪场停车场位置 |
| 装备打包 | □ | 单板包/双板包或雪具袋 |

如行程信息包含交通方式，自动填充对应检查项。

**5. 天气速报与穿衣建议**

| 时段 | 温度 | 天气 | 风力 | 穿衣建议 |
|------|------|------|------|---------|
| 山顶（实时） | -8°C | 小雪 | 3-4级 | 加厚保暖层+围脖 |
| 出发日预报 | -10°C ~ -3°C | 阴转小雪 | 4-5级 | 防风外套+暖宝宝 |
| 返回日预报 | -6°C ~ 0°C | 多云 | 2-3级 | 正常滑雪着装 |

如天气 API 不可用，使用 WebSearch 搜索并标注"网络搜索天气"。

### 场景 C：定时提醒（出发当天 07:00）

必须包含以下三部分：

**1. 今日天气速报**

| 项目 | 信息 |
|------|------|
| 山顶实时 | -8°C，小雪，风力 3 级 |
| 体感温度 | -12°C（风寒效应） |

**2. 最终确认清单**（精简版，只列关键项）

- [ ] 头盔 ✓

**3. 交通提醒**

## 数据存储

### upcoming_trips.json 结构

```json
[
  {
    "resort": "万龙",
    "city": "河北崇礼",
    "departure_date": "2026-04-15",
    "return_date": "2026-04-17",
    "created_at": "2026-04-11",
    "reminders_set": true,
    "reminder_ids": ["cron-xxx", "cron-yyy"],
    "status": "upcoming"
  }
]
```

状态流转：`upcoming` → `in_progress`（出发当天）→ `completed`（返回后）

### 过期清理

返回日期已过 7 天的行程，标记为 `completed`。超过 30 天的记录可清理（告知用户后执行）。

## 定时任务管理

### 创建提醒（需用户明确同意）

```
用户："我 4 月 15 号去万龙滑雪"
Agent：好的，已记录你的行程。是否需要我在出发前一天（4 月 14 号晚上 8 点）和出发当天（4 月 15 号早上 7 点）提醒你检查装备？
用户："好的"
Agent：已创建两个提醒：
  - 4 月 14 日 20:00 装备检查提醒
  - 4 月 15 日 07:00 出发确认提醒
```

### 取消提醒

用户说"取消提醒"或"不用提醒了"时：
1. 读取 upcoming_trips.json 找到对应行程
2. 删除关联的 cron 任务
3. 更新 reminders_set 为 false
4. 告知用户"已取消提醒"

### 查看即将出行的行程

用户说"我近期有什么滑雪行程"时：
1. 读取 upcoming_trips.json
2. 筛选 status 为 upcoming 且 departure_date 在未来 30 天内的记录
3. 按日期排序输出

## 输出格式：查看近期行程

必须包含以下表格：

| 目的地 | 出发日期 | 返回日期 | 提醒状态 | 操作 |
|--------|---------|---------|---------|------|
| 万龙 | 2026-04-15 | 2026-04-17 | 已设置 | 可取消/可立即检查 |
| 太舞 | 2026-05-01 | 2026-05-03 | 未设置 | 可设置提醒 |

如无记录，输出"你还没有 upcoming 的滑雪行程，说'我X月X号去XX滑雪'可以添加"。
