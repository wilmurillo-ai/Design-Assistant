# 学习风格自适应

## 数据采集

学习 2-3 天后，分析用户行为模式：

| 行为指标 | 说明 |
|---|---|
| 图文偏好 | 用户回复中提到图还是文字更多 |
| 阅读时长 | 图文推送 vs 纯文字推送的完成时间差异 |
| 费曼质量 | 哪种风格下用户的复述更准确 |
| 互动深度 | 哪种风格下用户追问更多 |
| 主动要求 | 用户说「多给点图」「给代码示例」等直接反馈 |

## 自适应规则

| 检测到 | 调整 |
|---|---|
| 用户经常说「这个图好清楚」 | 增加图解比重，复杂概念多用 Mermaid |
| 用户经常追问代码细节 | 增加代码示例，减少纯文字解释 |
| 用户的类比特别好 | 推送时加入更多类比，少用专业术语 |
| 用户经常给数据/指标提问 | 增加数据驱动的表格和图表 |
| 用户说「太啰嗦」 | 精简到 300 字以内，只保留要点 |
| 用户说「再多讲讲」 | 增加深度解析和延伸阅读 |

## 风格偏好存储

在 path.json 中记录：

```json
{
  "learning_style": {
    "visual_weight": "high",
    "code_weight": "medium",
    "analogy_weight": "high",
    "data_weight": "low",
    "verbosity": "concise",
    "updated_at": "2026-04-08",
    "confidence": 0.75
  }
}
```

## 渐进式调整

- **前 2 天**：使用均衡风格（图文各半，有代码有类比）
- **第 3 天起**：根据采集的数据开始微调
- **第 5 天起**：风格稳定，confidence > 0.7
- **用户手动调整**：随时说「多给图/多给代码/精简点」，立即调整
