# 遗忘曲线复习提醒

## 复习节点（基于 Ebbinghaus 遗忘曲线）

| 复习次数 | 间隔时间 | 遗忘率（无复习） | 复习后记忆保持 |
|---|---|---|---|
| 第 1 次复习 | 1 天后 | ~60% 已忘 | ~90% |
| 第 2 次复习 | 3 天后 | ~75% 已忘 | ~95% |
| 第 3 次复习 | 7 天后 | ~80% 已忘 | ~98% |

## 复习卡格式

```
🔄 复习提醒 | {topic} · Day {n} 回顾

还记得昨天学的「{概念}」吗？快速回顾：

📌 一句话总结：
{用 1 句话概括核心}

🧠 快速自测（30 秒）：
{1 道简短问题}

💡 提示：
{给一个小提示，但不给答案}

回复你的答案，我来确认 👇
```

## 实现方式

用户完成某天的费曼简化后，立即创建复习 cron：

```json
{
  "name": "quick-learn-review: {slug}-day{n}",
  "schedule": {
    "kind": "at",
    "at": "{完成时间 + 24小时}"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "读取 learning-data/{slug}/path.json 和 daily-log.json，找到 Day {n} 的学习内容。生成一张 5 分钟复习卡推送给用户。若用户答对，安排下一次复习（+3 天）；答错则 1 天后再复习。"
  },
  "sessionTarget": "isolated"
}
```

**链式调度逻辑：**
1. 复习卡推送 → 用户回答 → 判断对错
2. **答对** → 标记通过，创建下一次复习 cron（当前时间 +3 天）
3. **答错** → 回补资料，1 天后再次推送同一复习卡
4. **连续 3 次答对** → 标记该概念为「已内化」，不再安排复习
5. 最后一次复习（7 天节点）完成后，不再创建新 cron

## 自适应调整

| 用户表现 | 调整 |
|---|---|
| 答对 | 跳过该复习节点，进入下一轮 |
| 答错 | 回到 1 天节奏重新复习 |
| 连续 3 次答对 | 标记「已内化」，终止复习 |
| 无响应（24h） | 跳过本次复习，下次节点照常 |
