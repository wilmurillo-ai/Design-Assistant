# Learning Patterns & Advanced Paths / 学习模式与进阶路径

> Day titles and explanations match user's language. Technical topic names stay in English.

## Domain Patterns

### Programming/Tech / 编程技术
```
Day 1: Concept + Hello World | Day 2-3: Core Syntax/API | Day 4-5: Hands-on
Day 6: Best Practices + Pitfalls | Day 7: Summary + Self-test
```
Each session includes practice. Material: Docs 40% + Tutorials 40% + Video 20%.

### Theory/Concepts / 理论概念
```
Day 1: Big Picture (What/Why) | Day 2-3: Core Breakdown | Day 4-5: Cases/Scenarios
Day 6: Advanced/Controversies | Day 7: Summary + Self-test
```
Material: Surveys/Books 50% + Articles 30% + Video 20%. Suitable for TTS reading.

### Products/Business / 产品业务
```
Day 1: Industry Panorama | Day 2-3: Competitor/Model | Day 4-5: Ops/Growth
Day 6: Data/Metrics | Day 7: Summary + Self-test
```
Priority: Industry reports, top company shares. Material: Reports 40% + Articles 40% + Podcasts 20%.

### Universal Rules
- **Default 7 days**, customizable 3-30. Daily 30 min default. Last 1-2 days = summary+self-test.
- "Crash course" / 「速成」→ 3 days, core only. "Systematic" / 「系统学」→ 14+ days, more depth.
