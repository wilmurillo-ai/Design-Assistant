# tool 工具接口规范

本文档定义投资研究 tool 工具的接口规范。

> **重要**：所有 tool 工具都是 **LLM 工作流**，输出为结构化文本（Markdown），而非纯 JSON。

---

## 通用规范

### 工具分类

| 类别 | 工具 | 用途 |
|------|------|------|
| 基础工具 | `extract_assets` | 从文本提取资产 |
| 基础工具 | `investment_search_pro` | 检索专业投资信息 |
| 快速概览 | `get_asset_overview` | 获取资产核心逻辑和主要矛盾 |
| 单维度分析 | `analyze_*` | 深度分析单个维度 |
| 深度分析 | 5 个 `analyze_*` 并行 | 完整投资研究报告 |

### 调用方式

```python
# 工具调用示例
result = await tool.call_tool("get_asset_overview", {
    "symbol": "600519.SH"
})
```

### 返回格式

**单维度分析工具**返回 Markdown 文本，包含：

```markdown
[公司名称]

[一句话总结 + 核心数据]

[4 个带 emoji 的关键发现]
💰 发现1
🌍 发现2
📉 发现3
🔧 发现4

1. **[详细分析1]**。具体内容...
2. **[详细分析2]**。具体内容...
3. **[详细分析3]**。具体内容...
4. **[详细分析4]**。具体内容...
```

### 错误处理

| 错误类型 | 返回内容 |
|----------|----------|
| 股票代码无效 | `⚠️ 股票代码 [code] 不存在或格式错误` |
| 数据暂不可用 | `⚠️ [维度] 数据暂不可用，请稍后重试` |
| 网络错误 | `⚠️ 数据获取失败: [错误信息]` |

## 0. 资产提取 `extract_assets`

### 功能
从任意文本中提取金融资产名称和代码，用于识别用户意图中的分析对象。

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `text` | string | 是 | 用户输入的原始文本 |

### 输出结构

```json
{
  "success": true,
  "assets": [
    {
      "name": "宁德时代",
      "symbol": "300750.SZ",
      "market": "A股",
      "confidence": 0.95
    }
  ],
  "ambiguous": [],
  "suggestions": []
}
```

### 输出字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `assets` | array | 识别到的资产列表 |
| `assets[].name` | string | 资产名称（标准化） |
| `assets[].symbol` | string | 股票代码 |
| `assets[].market` | string | 市场类型：A股/港股/美股/基金 |
| `assets[].confidence` | float | 识别置信度 (0-1) |
| `ambiguous` | array | 歧义资产（需用户确认） |
| `suggestions` | array | 推荐的资产（如用户输入简称） |

### 输出示例

**输入**: "帮我分析下宁王和茅台的技术面"

```json
{
  "success": true,
  "assets": [
    {
      "name": "宁德时代",
      "symbol": "300750.SZ",
      "market": "A股",
      "confidence": 0.92,
      "alias": "宁王"
    },
    {
      "name": "贵州茅台",
      "symbol": "600519.SH",
      "market": "A股",
      "confidence": 0.98
    }
  ],
  "ambiguous": [],
  "suggestions": []
}
```

**输入**: "比亚迪怎么样"

```json
{
  "success": true,
  "assets": [
    {
      "name": "比亚迪",
      "symbol": "002594.SZ",
      "market": "A股",
      "confidence": 0.95
    }
  ],
  "ambiguous": [],
  "suggestions": [
    {
      "name": "比亚迪股份",
      "symbol": "1211.HK",
      "market": "港股",
      "note": "港股比亚迪，如需分析港股请明确"
    }
  ]
}
```

**输入**: "今天大盘怎么样"（无明确资产）

```json
{
  "success": true,
  "assets": [],
  "ambiguous": [],
  "suggestions": [],
  "hint": "未识别到具体股票，可询问用户想分析哪只股票"
}
```

### 处理逻辑

| 情况 | 处理方式 |
|------|----------|
| 无资产 | 询问用户想分析哪只股票 |
| 单资产 + 高置信度(>0.8) | 直接使用，继续分析流程 |
| 单资产 + 低置信度(<0.8) | 确认资产名称是否正确 |
| 多资产 | 询问分析哪一只，或触发行业对比模式 |
| 存在歧义资产 | 展示选项让用户确认 |

### 支持的资产类型

| 类型 | 识别模式 | 示例 |
|------|----------|------|
| A股 | 6位数字代码、公司名称、简称、别名 | 600519、贵州茅台、茅台、茅王 |
| 港股 | 数字代码+.HK、公司名称 | 00700.HK、腾讯控股 |
| 美股 | 英文代码、公司名称 | AAPL、苹果 |
| 基金 | 基金代码、基金名称 | 110022、易方达消费行业 |

## 1. 投资信息检索 `investment_search_pro`

### 功能
从新闻/研报/公众号等公开信息中获取专业的金融类信息内容，用于理解市场动态、公司事件、行业趋势等。

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | 是 | 用于检索的文本 |

### 输出结构

```json
{
  "success": true,
  "result": [
    {
      "title": "文档标题",
      "content": "文档内容",
      "effectiveTime": "文档创建时间"
    }
  ]
}
```

### 输出字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 文档标题 |
| `content` | string | 文档内容 |
| `effectiveTime` | string | 文档创建的时间 |

### 输出示例

**输入**: query="宁德时代 2025年业绩"

```json
{
  "success": true,
  "result": [
    {
      "title": "宁德时代2025年Q3财报解读：储能业务爆发",
      "content": "宁德时代发布2025年第三季度财报...",
      "effectiveTime": "2025-10-30"
    },
    {
      "title": "全球动力电池市场格局分析",
      "content": "根据最新数据，宁德时代市场份额...",
      "effectiveTime": "2025-11-15"
    }
  ]
}
```

### 使用场景

| 场景 | 调用方式 |
|------|----------|
| 用户询问公司最新动态 | `investment_search_pro` → 搜索相关新闻和研报 |
| 用户询问行业趋势 | `investment_search_pro` → 搜索行业分析报告 |
| 用户询问市场事件 | `investment_search_pro` → 搜索相关事件报道 |

## 2. 快速资产概览 `get_asset_overview`

### 功能
获取资产的**核心逻辑和主要矛盾**，用于快速分析场景。一次性输出投资该资产最关键的信息，帮助用户快速决策是否值得深入研究。

### 输入参数

| 参数 | 类型 | 必填 |说明                |
|------|------|------|-------------------|
| `symbol` | string | 是 | 股票代码（如 600519.SH） |

### 输出结构

```markdown
[公司名称] - 快速概览

## 核心逻辑
[2-3 句话说明投资该资产的核心逻辑]

## 主要矛盾
[当前面临的主要多空因素]

## 关键数据
| 指标 | 数值 | 评价 |
|------|------|------|
| 营收增速 | X% | [评价] |
| ROE | X% | [评价] |
| PE | X | [评价] |
| 技术趋势 | [多/空/震荡] | [评价] |

## 投资建议
- **投资倾向**: [偏多/中性/偏空]
- **适合人群**: [长期投资者/短期交易者/观望]
- **核心风险**: [1-2 个主要风险]

---
*如需更详细分析，可以说"给我深度分析"*
```

### 输出示例

**输入**: symbol="300750.SZ"

```markdown
宁德时代 - 快速概览
```

## 4. 技术面 `analyze_technical`

### 功能
分析股价技术形态，包括均线系统、MACD/KDJ 指标、布林带、量价关系。

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `symbol` | string | 是 | 股票代码（必须带市场后缀） |

### 输出结构

```markdown
[公司名称]

[日期]股价[位置描述]，[均线/指标状态]，[量能描述]，[趋势判断]

当前[公司名称]技术面呈现典型的『[形态描述]』结构，[操作建议]。具体分析如下:
- **均线系统**: [5/10/20/60日均线状态]
- **MACD指标**: [MACD/DIF/DEA状态及信号]
- **KDJ指标**: [KDJ值及超买超卖判断]
- **布林带形态**: [上下轨/中轨/带宽/突破状态]
- **量价关系**: [成交量变化及资金流向]
综合判断:[操作建议及关键位置]
```

### 输出示例

```markdown
新奥股份

3月11日股价在21.29附近企稳，5日均线向上拐头，MACD金叉动能增强，KDJ超卖区反弹，布林带收口后价格突破中轨，量能温和放大，显示短期反弹动能正在形成，但上方压力仍存。

- **均线系统**:5日均线（21.43）略高于当前股价（21.29），且呈向上拐头趋势；10日均线（21.62）与20日均线（20.60）形成金叉雏形...
- **MACD指标**:MACD值0.20，DIF（0.48）与DEA（0.38）形成金叉，柱状线虽未大幅放大，但动能持续抬升...
- **KDJ指标**:当前KDJ值18.29，处于超卖区域，但近期连续三日从低位回升...
- **布林带形态**:布林带上下轨分别为23.31和17.89，当前价格位于中轨（约20.6）附近，带宽明显收窄...
- **量价关系**:近两日成交量维持在19万手以上，较前期低谷放大数倍...
综合判断:当前股价处于关键支撑位之上，技术面共振向上，建议在股价有效突破10日均线（21.62）并伴随成交量放大时，确认买入信号...
```

### 输出结构

```markdown
[公司名称]

[日期][公司名称]资金面呈现[主力资金状态]、[融资融券状态]、[趋势判断]

总体来看，[公司名称]当前资金面[整体判断]，[操作建议]。具体分析如下:
- **资金净流入与主力动向**: [整体资金流向及主力行为]
- **大单与超大单结构**: [大单/超大单净流入数据]

- **杠杆资金风险敞口**: [融资融券数据及风险判断]

因此，[操作建议及关键观察点]
```

### 输出示例

```markdown
联科科技
```

## 调用示例

### 单维度调用

```python
# 用户输入: "分析下宁德时代的技术面"
result = await tool.call_tool("analyze_technical", {
    "symbol": "300750.SZ"
})
```

### 多维度调用（深度分析）

```python
# 用户输入: "给我茅台的深度分析"
tools = [
    "analyze_fundamentals_financial",
    "analyze_fundamentals_valuation",
    "analyze_technical",
    "analyze_capital_flow",
    "analyze_market_sentiment",
]

results = await asyncio.gather(*[
    tool.call_tool(tool_name, {"symbol": "600519.SH"})
    for tool_name in tools
])
```
