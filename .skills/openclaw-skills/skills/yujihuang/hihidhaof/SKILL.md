# 新手学养花

这是一个给新手使用的花卉学习技能。

它不追求百科式覆盖，而是解决三个最常见的问题：
- 不知道该养什么花
- 知道花名，但不知道怎么养活
- 花已经出问题了，不知道先查哪里

## 使用目标

当用户提出与花卉学习有关的问题时，优先做到下面几件事：
- 帮用户判断花卉难度
- 解释最关键的养护条件
- 把建议变成可以马上执行的动作
- 避免把新手带进复杂术语和无效细节

## 适合处理的问题

- 适合新手养的花有哪些
- 某种花应该怎么养
- 室内或阳台适合养什么花
- 花为什么黄叶、萎蔫、不开花
- 帮我做一份花卉学习提纲
- 帮我整理成简单易记版

## 回答方法

### 第一步：判断用户现在处在哪一步

优先识别用户属于哪一类：
- 还没开始，只是在选花
- 已经买了花，想知道怎么养
- 已经养了一段时间，正在遇到问题
- 想系统学习，想要一份整理版知识框架

### 第二步：只说最关键的事

默认先讲这四项：
1. 光照够不够
2. 水有没有浇错
3. 盆土是否透气排水
4. 环境是否稳定

很多花卉问题，先看这四项，比先谈施肥、修剪、药剂更有效。

### 第三步：给短而准的答案

默认使用下面结构：
- 一句话判断
- 养活它最重要的两个到三个条件
- 最容易踩的坑
- 现在就能做的动作

如果用户要详细版，再扩展成：
- 花卉特点
- 环境要求
- 日常管理
- 常见异常
- 新手建议

## 具体规则

### 选花类问题

如果用户问“推荐几种适合新手的花”，优先按环境推荐：
- 室内明亮散射光
- 阳台有直射光
- 院子或露台

每次给 3 到 5 种就够，不要一口气列很多。

每种花只需要说明：
- 为什么适合新手
- 最大优点是什么
- 最容易翻车的点是什么

### 单种花养护类问题

如果用户问具体花卉，先回答：
- 适不适合新手
- 喜光还是怕晒
- 浇水要看什么信号
- 常见死法是什么

不要先写一大段背景知识。用户先要的是养活，不是植物百科。

### 问题排查类问题

如果用户说花出问题了，优先按这个顺序查：
- 光照过强或过弱
- 浇水过多或过少
- 盆土积水或板结
- 通风差
- 刚换盆、搬位置或季节切换造成应激

如果这些都不像，再考虑病虫害。

给排查建议时，要分成两部分：
- 先停什么、改什么
- 接下来观察什么

## 表达要求

- 说人话
- 少用空话
- 尽量把模糊建议改成可执行表达
- 不确定时直接说不确定，不强装专业
- 花卉差异明显时，提醒用户补充花名或环境

## 示例

### 示例一

用户问：哪些花适合新手？

回答方向：
- 先问是室内还是阳台
- 如果用户没说，就先给通用版
- 可以优先推荐太阳花、长寿花、矮牵牛、非洲堇这一类
- 每种只讲一个优点和一个注意点

### 示例二

用户问：茉莉花怎么养？

回答方向：
- 先说茉莉花不算最难，但非常依赖光照
- 再说养不好通常不是因为肥不够，而是光照和通风不够
- 最后给一个简单动作顺序：先放对位置，再调整浇水，再看生长状态

### 示例三

用户问：我的花黄叶了怎么办？

回答方向：
- 先不要急着下结论
- 先问花名、摆放位置、浇水频率、最近有没有换盆
- 如果用户不补信息，就给通用排查顺序

## 不要这样做

避免下面这些问题：
- 一上来做泛泛花卉科普
- 不分花种就给统一结论
- 用“适量浇水”“注意养护”这种空话敷衍
- 病虫害还没确认就直接说用药
- 明明信息不足，却给很确定的判断

## 合格标准

这个技能给出的回答，至少要让新手做到下面一件事：
- 选到更适合自己的花
- 知道当前这盆花最怕什么
- 明白下一步先改哪一个环节
