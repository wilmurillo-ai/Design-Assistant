## ⚠️ 历史错误教训

**2026-04-02 宁德时代报告数据错误事件：**

### 错误内容
1. **归母净利润错误**：将"净利润"767.86 亿元误作"归母净利润"（实际 722.01 亿元）
2. **分业务收入错误**：
   - 动力电池：估计 2850 亿 vs 实际 3165 亿（偏差 -10%）
   - 储能电池：估计 950 亿 vs 实际 624 亿（偏差 +52%）❌
   - 电池材料：估计 350 亿 vs 实际 219 亿（偏差 +60%）❌
3. **分业务增速错误**：
   - 储能：估计 +45% vs 实际 +9%
   - 电池材料：估计 +5% vs 实际 -24%
4. **分业务毛利率错误**：
   - 动力电池：估计 20% vs 实际 26%
   - 储能：估计 25% vs 实际 30%

### 错误原因
1. **MCP 工具调用失败**：脚本使用错误的工具名称（如 `IncomeStatement`），实际应使用 `FinancialDataAPI`
2. **使用缓存旧数据**：从其他技能复制的 JSON 文件只包含 2021-2024 年数据
3. **手动估计数据**：分业务收入、毛利率等使用行业知识估计，未实际查询 MCP

## ✅ 数据校验清单

### 🎯 获取最新财报数据的标准流程（必须严格执行）

**⚠️ 重要教训（2026-04-02 明阳电气案例）：**
- 查询 2025 年年报时，返回的 `date` 字段为 "2018-12-31"，说明 2025 年年报数据不存在
- 必须降级查询 2025 年三季报，`date` 字段为 "2025-09-30"，确认是最新数据
- 在报告中必须明确标注数据截止日期和报告期类型

---

**第一步：确定当前年份和报告期**
```bash
# 当前年份：2026 年
# 最新年报：2025 年年报（当前年份 -1）
# 最新季报：2025 年三季报/中报/一季报
```

**第二步：优先查询最新年报**
```bash
# 1. 优先查询最新年报（当前年份 -1）
mcporter call jy-financedata-api.FinancialDataAPI query="公司名 股票代码 2025 年 年报 营业收入 净利润"
```

**⚠️ 关键验证：检查返回数据的 `date` 字段**
```json
// 正确示例：2025 年年报存在
{
  "date": "2025-12-31",
  "reportdate": "年报",
  "financevalue": "64.44"
}

// 错误示例：2025 年年报不存在，返回旧数据
{
  "date": "2018-12-31",  // ⚠️ 这是 2018 年数据，不是 2025 年！
  "reportdate": "年报",
  "financevalue": "0.20"
}
```

**如 `date` 字段显示为多年前，说明该年报数据不存在，必须降级查询最新季报！**

**第三步：如年报无数据，查询最新季报**
```bash
# 2. 如年报无数据，查询最新季报（按优先级）
# 按优先级查询：三季报 > 中报 > 一季报

# 2.1 先查询三季报（10 月 - 次年 3 月可用）
mcporter call jy-financedata-api.FinancialDataAPI query="公司名 股票代码 2025 年 三季报 营业收入 净利润"

# 2.2 如三季报无数据，查询中报（7 月 -9 月可用）
mcporter call jy-financedata-api.FinancialDataAPI query="公司名 股票代码 2025 年 中报 营业收入 净利润"

# 2.3 如中报无数据，查询一季报（4 月 -6 月可用）
mcporter call jy-financedata-api.FinancialDataAPI query="公司名 股票代码 2025 年 一季报 营业收入 净利润"
```

**第四步：验证数据时效性（必须执行）**
- [ ] 检查返回数据的 `date` 字段，确认是最新报告期（如 2025-09-30）
- [ ] 如 `date` 字段显示为多年前（如 2018-12-31），说明数据不存在，必须降级
- [ ] 在报告中明确标注报告期类型（年报/三季报/中报/一季报）
- [ ] 在报告中标注数据截止日期（如 2025-09-30）

**第五步：获取上年同期数据**
```bash
# 根据最新报告期，获取上年同期数据计算同比增速
# 如最新为 2025 年三季报，则获取 2024 年三季报
mcporter call jy-financedata-api.FinancialDataAPI query="公司名 股票代码 2024 年 三季报 营业收入 净利润"
```

### 生成报告前必须核实的数据

#### 1. 核心财务数据（必须通过 MCP 查询）
- [ ] 营业收入（合并利润表）
- [ ] 归母净利润（不是净利润！）
- [ ] 净利润
- [ ] 少数股东损益
- [ ] 验证：净利润 = 归母净利润 + 少数股东损益
- [ ] **确认数据报告期（年报/季报）**

#### 2. 分业务收入（必须通过 MainOperIncData 查询）
- [ ] 各业务板块收入（动力电池、储能、材料回收等）
- [ ] 各业务收入占比
- [ ] 各业务同比增速
- [ ] 验证：各业务收入合计 = 主营业务收入

#### 3. 盈利能力数据（必须通过 MCP 查询）
- [ ] 毛利率（整体）
- [ ] 净利率（整体）
- [ ] 分业务毛利率（如可获取）
- [ ] 期间费用率（销售、管理、研发、财务）

#### 4. 同比数据验证
- [ ] 获取上一年同期数据进行对比
- [ ] 验证同比增速计算正确
- [ ] 检查异常波动（>50% 或方向相反需重点核实）

#### 5. 数据时效性验证 ⭐（强制要求）
- [ ] 检查返回数据的 `date` 字段（如 "2025-09-30"）
- [ ] 确认 `date` 字段是最新报告期，不是多年前（如 "2018-12-31"）
- [ ] 如 `date` 字段为多年前，已降级查询上一报告期
- [ ] 报告中已明确标注数据截止日期（如 2025-09-30）
- [ ] 报告中已明确标注报告期类型（年报/三季报/中报/一季报）

**⚠️ 验证示例：**
```json
// ✅ 正确：2025 年三季报数据
{
  "date": "2025-09-30",
  "reportdate": "三季报",
  "financevalue": "52.00"
}

// ❌ 错误：查询 2025 年年报，但返回 2018 年数据
{
  "date": "2018-12-31",  // ⚠️ 这不是 2025 年数据！
  "reportdate": "年报",
  "financevalue": "0.20"
}
// 处理：立即降级查询 2025 年三季报
```

  query="公司名 股票代码 年份 年报 营业收入 净利润 归母净利润"

## 📋 报告生成流程

### 步骤 1：环境检查
```bash
# 检查 mcporter 是否安装
mcporter --version

# 检查 MCP 服务配置
mcporter list
# 应包含：gildata_datamap-sse
```

### 步骤 2：获取核心财务数据
```bash
# 获取利润表数据
mcporter call gildata_datamap-sse.FinancialDataAPI \
  query="公司名 股票代码 年份 年报 营业收入 净利润 归母净利润"

# 保存结果并记录关键数字
```

### 步骤 3：获取分业务数据
```bash
# 获取主营构成
mcporter call gildata_datamap-sse.MainOperIncData \
  query="公司名 股票代码 年份 分产品 主营业务收入"

# 保存结果并记录各业务收入
```

### 步骤 4：获取财务指标
```bash
# 获取盈利能力指标
mcporter call gildata_datamap-sse.FinancialDataAPI \
  query="公司名 股票代码 年份 年报 毛利率 净利率 销售费用 管理费用 研发费用"
```

### 步骤 5：数据校验
- [ ] 核对各数据源的一致性
- [ ] 验证同比计算正确
- [ ] 检查异常值（如毛利率>50% 或<5% 需核实）
- [ ] 验证分业务收入合计与总收入匹配

### 步骤 6：生成报告
- [ ] 使用实际查询数据，不使用估计值
- [ ] 标注数据来源和截止日期
- [ ] 添加免责声明

## 🚨 常见错误及避免方法

| 错误类型 | 表现 | 避免方法 |
|---------|------|----------|
| 工具名称错误 | 调用 `IncomeStatement` 返回空 | 使用 `FinancialDataAPI` |
| 服务名称错误 | 调用 `jy-financedata-tool` 失败 | 使用 `gildata_datamap-sse` |
| 数据混淆 | 净利润 vs 归母净利润 | 明确查询"归属于母公司所有者的净利润" |
| 使用旧数据 | 报告年份与实际不符 | 查询时明确指定年份 |
| 手动估计 | 分业务收入偏差大 | 必须通过 `MainOperIncData` 查询 |
| 未验证同比 | 增速计算错误 | 获取上年同期数据对比验证 |
