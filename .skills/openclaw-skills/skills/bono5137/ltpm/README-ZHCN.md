# 长期任务进度管理 (LTPM)

> 用于管理长期、多阶段 AI 项目的通用协议

**版本**: 2.1  
**用途**: 将短期对话记忆转换为结构化长期文档，实现项目无缝继续。

---

## 概述

LTPM 确保你在数天或数周后返回项目时能够立即理解：

1. **项目目标**是什么
2. **进度**在哪里
3. **下一步**需要做什么

### 设计理念

- **被动机制**：利用 HEARTBEAT.md 机制防止进度丢失
- **主动机制**：关键里程碑主动记录
- **双轨协同**：平衡系统健壮性与资源消耗

---

## 快速开始

```bash
# 创建项目结构
projects/
└── {项目名称}/
    ├── MISSION.md        # 项目目标
    ├── PROGRESS.md       # 当前状态
    └── NEXT_STEPS.md     # 下一步行动
```

### 核心命令

| 命令 | 说明 |
|------|------|
| `init_project {名称}` | 初始化新项目 |
| `checkpoint` | 保存当前进度（主动） |
| `resume {项目}` | 加载项目上下文 |
| `next` | 显示下一步行动 |
| `mark_milestone {描述}` | 标记关键里程碑 |
| `auto_sync_status` | 检查同步状态 |

## v2.1 新功能

### 1. 动态阈值
根据项目复杂度和会话活跃度自适应调整静默阈值：

| 复杂度 | 文件数 | 基础阈值 |
|--------|--------|----------|
| 简单 | <10 | 15 分钟 |
| 中等 | 10-50 | 10 分钟 |
| 复杂 | 50-200 | 5 分钟 |
| 非常复杂 | >200 | 3 分钟 |

### 2. 文件监听（低资源模式）
当系统资源允许时，使用 `fswatch` 进行文件变化检测。

### 3. 冲突解决
- 基于时间戳的冲突检测
- 最后写入者胜出策略
- 覆盖前自动 `.auto-save` 备份

### 4. 明确触发提示
提示模板，提醒用户何时记录进度：

```
✓ 完成重要任务
✓ 做出重要决定
✓ 结束工作会话
✓ 达成里程碑
```

### 5. 自然语言触发器

**中文**: 开始/进度/完成/暂停/结束/里程碑  
**English**: start/progress/completed/pause/end/milestone

### 6. 结构化格式支持
可选的 JSON/YAML 格式（支持 Markdown/JSON/YAML）。

## 文档模板

### MISSION.md
```markdown
# {项目名称}

**创建时间**: {YYYY-MM-DD}
**状态**: 进行中 / 已暂停 / 已完成

## 愿景
我们要构建什么？

## 成功标准
- [ ] 标准 1
- [ ] 标准 2
```

### PROGRESS.md
```markdown
# 进度：{项目名称}

**最后更新**: {YYYY-MM-DD}
**总体进度**: {x%}

## 当前阶段
{阶段描述}

## 里程碑
- [x] 已完成
- [ ] 进行中
- [ ] 待处理

## 最近活动
- {YYYY-MM-DD}: {活动内容}
```

### NEXT_STEPS.md
```markdown
# 下一步：{项目名称}

## 当前优先级
1. [ ] 任务 1
2. [ ] 任务 2
```

## 配置

在项目根目录创建 `.ltpm/config.json`：

```json
{
  "version": "2.1",
  "thresholds": {
    "simple": 15,
    "medium": 10,
    "complex": 5
  },
  "backup": {
    "enabled": true,
    "max_count": 5
  },
  "format": {
    "default": "markdown"
  }
}
```
