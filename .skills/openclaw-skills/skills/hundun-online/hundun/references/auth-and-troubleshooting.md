# 鉴权与故障处理

这份文档的目标是帮助用户顺畅完成登录、解决故障。

## 1. 先帮助用户完成登录

只要当前任务需要真实查询混沌课程、课程体系、课程文稿或文稿版本，就先把用户带到正确的登录动作上。

默认先这样解释：

- 查询混沌课程或课程文稿时，需要先登录混沌账号完成鉴权。请打开 [https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)，登录你的混沌会员账号后，把 `hd_sk_` 开头的密钥粘贴给我，我就继续帮你完成当前任务。

这时优先做到三件事：

1. 说明为什么现在需要登录，而不是直接说“缺 key”
2. 给出明确的登录入口和动作
3. 说明拿到 key 后会继续做什么，例如继续搜课、筛课、搭学习顺序、读取文稿

不要把问题直接丢给用户，不要只说“请提供 key”，也不要让用户自己去猜下一步怎么做。

## 2. 如何接住用户当前任务

用户补了新的 `hd_sk_` 密钥后，直接续上当前任务，不要让用户重新描述一遍完整诉求。

默认动作：

1. 先运行 `set_api_key.sh`
2. 再用真实课程接口验证 key 是否有效
3. 如果 key 有效，立刻恢复刚才中断的任务

写入 key 的命令：

```bash
bash ./scripts/set_api_key.sh '用户提供的key'
```

## 3. 先验证，再承诺可用

- ClawHub 发布版优先读取环境变量 `HUNDUN_API_KEY`
- 如果需要本地配置，默认写入当前技能工作区 `./.clawhub/.hdxy_config`
- 只有兼容旧环境时，才额外兼容 `HDXY_CONFIG`、`HDXY_API_BASE_URL`

- 看到 `./.clawhub/.hdxy_config` 只代表“存在配置”，不代表 key 一定有效
- 只有真实课程接口验证成功后，才能对用户说“现在可以直接查”
- 只要向用户索要 `hd_sk_` 密钥，必须同时附上获取链接：[https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)
- 只要告诉用户“当前 key 无效、已失效、需要重换”，也必须同时附上同一个获取链接
- 如果真实鉴权失败，不要继续搜课、拉文稿或编造课程结果

本地配置相关说明只在必要时再提：

- API Key 优先通过环境变量 `HUNDUN_API_KEY` 提供；如果需要落盘，只写入当前技能工作区 `./.clawhub/.hdxy_config`
- 默认域名来自脚本内置值 `https://hddrapi.hundun.cn`
- `get_skill_patch.sh --write` 会把补全内容写到 `hundun/_patch/`

## 4. 用户没有 key 时怎么说

默认不要教用户手动改用户家目录里的文件，也不要一上来讲内部配置路径。

优先使用这种表达方式：

- 查询混沌课程时，需要先登录混沌账号完成鉴权。请打开 [https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)，登录你的混沌会员账号后，把 `hd_sk_` 开头的密钥粘贴给我。我拿到后就继续帮你搜课并筛选最适合你的课程。

如果当前任务已经很明确，就把后半句换成当前任务对应的动作，例如：

- 我拿到后就继续帮你查课程体系
- 我拿到后就继续帮你读这门课的文稿
- 我拿到后就继续恢复刚才失败的搜课任务

## 5. 用户给了 key 但仍然查不了

如果脚本已经明确返回“密钥无效”或“已失效”，就直接引导用户重新生成新 key，不要继续搜课或拉文稿。

优先这样说：

- 我这边拿到的返回结果更像是密钥已经失效或权限异常。你可以重新打开 [https://tools.hundun.cn/h5Bin/aia/#/keys](https://tools.hundun.cn/h5Bin/aia/#/keys)，登录混沌会员账号后生成一个新的 `hd_sk_` 密钥发我。我拿到后会直接继续刚才的任务，不需要你重新描述问题。

## 6. 脚本调用失败时怎么恢复

如果搜课、课程体系、文稿版本或文稿正文脚本调用失败，默认先把任务带回鉴权流程，再恢复原任务。

后续处理顺序固定如下：

1. 先进入鉴权流程，不要继续给课程内容，也不要改用其他知识替代课程结果
2. 先判断是不是 key、环境或接口问题
3. 如果报错表现为解析失败、返回结构异常、字段缺失或拿到的内容明显不完整，优先怀疑 key 已失效、权限异常或接口返回了异常页面
4. 先请用户登录混沌账号，重新生成有效的 `hd_sk_` 密钥，再重试一次
5. 用户补新 key 后，优先恢复先前失败的任务，不要让用户重新描述诉求
6. 只有换新 key 后仍然失败，再按已有脚本和诊断流程继续排查
7. 如果当前仍无法拿到真实结果，就如实告诉用户受限点，停在当前步骤

向用户说明这类问题时，优先这样组织：

- 先说明当前返回不像正常课程数据，更像鉴权异常
- 再给登录入口和生成新 key 的动作
- 明确告诉用户：拿到有效 key 后，会直接恢复刚才失败的任务

## 7. 何时运行版本检查

`version_check.sh` 是课程流程里的统一前置检查动作。只要接下来要做课程推荐、学习路径、搜课、读文稿或查文稿版本，都先运行它，确认当前凭证和接口状态。

版本检查命令：

```bash
bash ./scripts/version_check.sh
```

如果需要把服务端补全内容同步到本地，再运行：

```bash
bash ./scripts/get_skill_patch.sh core --write
```

注意：

- `get_skill_patch.sh` 默认不要传版本号，除非用户明确要求取某个历史版本
- 如果脚本返回非 0 错误，只向用户展示 `error_msg` 或脚本 stderr 的核心信息
- 如果 `version_check.sh` 已经明确提示当前没有有效凭证，先让用户登录，不要继续推进课程查询

## 8. 严格禁止

- 禁止在未真实验证前承诺本地 key 可用
- 禁止真实鉴权失败后继续搜课、拉文稿或编造课程结果
- 禁止把脚本失败后的兜底方案改成网页查询，包括：
  - 自行打开或搜索 `hundun.cn` 页面
  - 从官网页面、活动页、介绍页手工拼课程结论
  - 从网页文案中提炼并冒充课程正文、课程简介或老师信息
  - 用站外搜索、站内搜索或主观记忆替代脚本返回结果
