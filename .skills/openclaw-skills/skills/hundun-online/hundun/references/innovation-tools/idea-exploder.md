# 创意点子爆破器

当用户想找新方向、新产品、新服务、新组合，但思路被行业惯性卡住时，用这个流程。

先读共享规则：[创新工具通用打法](./_shared-playbook.md)

## 适用问题

- “我想做个新东西，但不知道从哪里开始想”
- “这个行业还能有什么新机会”
- “我们现在只会模仿，怎么找到新点子”

## 先读哪些 foundations

- 要识别行业共识、隐含假设和基石问题，先读 [第一性原理.md](../foundations/第一性原理.md)
- 要生成新方向时，先读 [第二曲线创新.md](../foundations/第二曲线创新.md) 里和 `组合创新`、`分形创新`、`边缘分化`、`破界创新` 对应的章节

## 额外方法

这份工具默认同时使用：

- `MECE`：把问题拆得更干净，避免混成一团
- `供给-需求-连接`：判断新点子到底改了哪一层

## 最少输入

- 当前想探索的领域或问题
- 手里最能用的资源、能力或渠道
- 最想避开的边界或代价

如果信息还不够，先基于上下文假设一个最可能的目标场景，给一版方向参考，再请用户修正。

## 处理流程

1. 先把用户真正想推进的结果说清楚，不被行业名词带着跑
2. 用 MECE 和 `供给-需求-连接` 把当前系统拆开，看现有方案、目标人群、关键场景和限制条件
3. 找出当前做法默认成立的 1 到 3 个隐含假设
4. 用第一性原理追问这些假设为什么成立，以及如果它们不成立，会冒出什么新可能
5. 再用第二曲线模型生成新方向：
   - 用 `组合创新` 重组要素
   - 用 `分形创新` 从现有能力里长出新侧枝
   - 用 `边缘分化` 找边缘人群、边缘场景和非主流入口
   - 用 `破界创新` 在必要时直接换题
6. 收敛出 2 到 3 个方向，优先推荐一个最值得先试的，并告诉用户先验证什么

## 输出要求

- 先说你理解到的真正问题，再给方向
- 默认给 2 到 3 个方向，不一次铺满一堆点子
- 每个方向都要说清楚新在哪里，为什么不是旧思路换皮
- 最后一定落到一个最小验证动作
- 结尾主动请用户判断哪条方向更像他真正想做的，再继续深挖

## 严格禁止

- 禁止把“更好、更便宜、更高效”当成创新本身
- 禁止只做表面拼接，没有触及隐含假设
- 禁止脱离用户资源和现实约束空想
