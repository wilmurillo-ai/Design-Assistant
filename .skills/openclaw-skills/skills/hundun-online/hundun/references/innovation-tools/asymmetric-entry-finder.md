# 以小博大探测器

当用户想进入一个热门赛道、竞争激烈的市场或被巨头占据的领域，但希望找到错位竞争的小切口时，用这个流程。

先读共享规则：[创新工具通用打法](./_shared-playbook.md)

## 适用问题

- “这个赛道很热，但都是大公司，我还有机会吗”
- “我不想跟头部正面打，怎么切进去”
- “红海市场里有没有适合小团队的破局点”

## 先读哪些 foundations

- 要判断市场默认前提和真正约束，先读 [第一性原理.md](../foundations/第一性原理.md)
- 要寻找错位切口，先读 [第二曲线创新.md](../foundations/第二曲线创新.md) 里和 `错位竞争`、`颠覆式创新`、`边缘分化`、`价值网络`、`破界创新` 对应的章节
- 如果要解释为什么大公司跟不进来，也可以补读 `组织心智`

## 额外方法

这份工具默认同时使用：

- `系统思考`：看主流市场是怎么被一整套结构锁住的
- `非消费者视角`：看哪些人今天其实还没被服务到

## 最少输入

- 想进入的赛道或市场
- 手里的资源、能力或渠道优势
- 最不想卷进去的竞争方式

如果信息不足，先按最可能的资源结构给一版切口判断，并说明这建立在哪个假设上。

## 处理流程

1. 先判断用户真正想赢的是什么，是进入市场、活下来，还是先找到第一批能成交的人
2. 用系统思考和 `价值网络` 拆开主流价值网：谁掌控客户、渠道、交付、利润和规则
3. 看头部玩家是不是已经出现性能过度、服务过度，或者根本看不上某些边缘需求
4. 用第一性原理追问这个市场默认成立的前提是什么，以及它有没有被高估
5. 再用第二曲线模型找切口：
   - 用 `错位竞争` 找不同位置
   - 用 `颠覆式创新` 找低端、非主流或非消费者入口
   - 用 `边缘分化` 找边缘人群和边缘场景
   - 用 `破界创新` 在必要时重写市场定义
6. 收敛出 2 到 3 个切口，默认推荐 1 个最适合当前资源结构的方向，并明确先验证什么

## 输出要求

- 先说这个市场值不值得以小博大切入
- 再给 2 到 3 个可行切口，并说明为什么巨头不愿做、做不好或做了会伤到自己
- 推荐方向必须结合用户现有资源，不给抽象蓝海口号
- 最后一定落到一个最小验证动作
- 结尾主动问用户更想沿着哪个切口继续，我再往 MVP、用户或渠道层面细化

## 严格禁止

- 禁止把“做高端、做细分、做差异化”当成完成答案
- 禁止建议用户正面硬刚头部主战场
- 禁止给出和用户资源完全不匹配的空想切口
