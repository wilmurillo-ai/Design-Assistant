# 一句话价值提亮器

当用户已经有一段关于自己、公司、产品或项目的介绍，但表达平、长、散、记不住时，用这个流程。

先读共享规则：[创新工具通用打法](./_shared-playbook.md)

## 适用问题

- “帮我把这段介绍说得更有记忆点”
- “我们到底该怎么一句话介绍自己”
- “这段官网标题 / 电梯介绍 / 小红书简介太普通了”

## 先读哪些 foundations

- 要追问用户真正买的是什么、原表述背后的隐含假设是什么，先读 [第一性原理.md](../foundations/第一性原理.md)
- 要突出差异、重写问题边界或重组表达要素时，先读 [第二曲线创新.md](../foundations/第二曲线创新.md) 里和 `组合创新`、`错位竞争`、`破界创新` 对应的章节

## 额外方法

这份工具默认同时使用：

- `Job-to-be-Done`：判断用户真正雇这段介绍去完成什么任务
- `黄金圈`：把 Why、How、What 重新排顺序

## 最少输入

- 当前介绍原文
- 主要想打动谁
- 最希望对方记住什么

如果信息不全，先按最可能的对象和场景给一版参考表达，并明确这是基于当前信息的假设。

## 处理流程

1. 先把原话拆成对象、问题、结果、差异线索
2. 判断原表述现在更像在讲功能、场景，还是身份感
3. 用第一性原理和 JTBD 追问：用户真正买的不是这句话本身，而是它帮自己传达了什么价值
4. 用黄金圈重排表达重心，让 Why 先于 How，How 先于 What
5. 如果需要更强张力，再用 `错位竞争` 找出它和常见替代说法的差异；如果原问题本身就问错了，再用 `破界创新` 改写表达边界
6. 最后用 `组合创新` 重组对象、问题、结果和差异，收敛成一条主表达，再补 1 到 2 条备选

## 输出要求

- 默认先给 1 条最推荐的主表达，再给 1 到 2 条不同气质的备选
- 每条都顺手点一下为什么它更容易被记住
- 回答优先像一起打磨一句话，不要写成长篇文案分析
- 如果你做了场景假设，要明确说出来，请用户纠偏
- 结尾主动问用户更想靠近哪一种方向，再继续细修

## 严格禁止

- 禁止用行业套话、空形容词替代价值判断
- 禁止把介绍写得比原来更长、更虚
- 禁止脱离用户真实业务硬造噱头
