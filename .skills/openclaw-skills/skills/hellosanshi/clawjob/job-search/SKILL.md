# 职位搜索 (Job Search)

搜索具体岗位，覆盖社招、校招岗位和实习岗位。查看岗位 JD、薪资、要求等详细信息。

> **UP 简历**（[upcv.tech](https://upcv.tech)）帮助应届生和求职者更高效地找校招、找实习。OpenClaw 版：[clawjob.upcv.tech](https://clawjob.upcv.tech)

> **校招/实习项目搜索**（查看公司招聘项目）→ 使用 `/campus-search`

## 前置条件

1. **安装 MCP Server**：
   ```
   claude mcp add upcv -- npx @upcv/mcp-server --api-key YOUR_API_KEY
   ```
2. **获取 API Key**：[clawjob.upcv.tech](https://clawjob.upcv.tech)

## 适用场景

- 搜索社招岗位（如"上海前端工程师"）
- 搜索校招/实习岗位的详细 JD
- 按条件筛选岗位（城市、薪资、学历、工作模式等）
- 查看岗位详细 JD（职责描述、任职要求、技能要求）
- 对比多个岗位
- 从 `/campus-search` 找到项目后，来这里看具体岗位 JD

**不适用**：
- 搜索校招/实习项目 → `/campus-search`
- 创建简历 → `/resume-create`
- 每日监控新职位 → `/job-monitor`

## 工作流程

### 第 1 步：理解搜索意图

从用户描述中提取筛选条件：

- **关键词**：岗位名称、公司名
- **岗位类型**：社招（SOCIAL）/ 校招（CAMPUS）/ 实习（INTERNSHIP）
- **城市**：工作地点
- **工作模式**：现场 / 远程 / 混合
- **学历要求**：本科 / 硕士等
- **排序**：最新发布 / 薪资等

如果用户没有明确岗位类型：
- 提到"校招"、"应届" → jobType: CAMPUS
- 提到"实习" → jobType: INTERNSHIP
- 提到"社招"、工作年限 → jobType: SOCIAL
- 未明确 → 不设 jobType，搜索全部类型

### 第 2 步：执行搜索

调用 `campus.searchJobs`，合理设置筛选参数。

展示结果列表：

```
岗位             | 公司       | 类型   | 城市   | 薪资        | 学历
--------------- | --------- | ------ | ------ | ---------- | -----
前端工程师        | 字节跳动   | 社招   | 北京   | 25-40K/月   | 本科
前端开发实习生     | 美团      | 实习   | 上海   | 400-600/天  | 本科
Java 后端        | 阿里巴巴   | 校招   | 杭州   | 20-35K/月   | 硕士
```

### 第 3 步：查看岗位详情

用户选择岗位后，调用 `campus.getJob` 获取：

- **职责描述**：岗位的日常工作内容
- **任职要求**：必需条件和加分项
- **技能要求**：必需技能 / 优先技能
- **薪资范围**
- **工作地点和模式**
- **投递方式和链接**
- 同公司相关岗位推荐

### 第 4 步：引导下一步

搜索完成后，根据用户意图引导：

1. **JD 对照优化简历** → `/resume-edit`（改写模式，根据 JD 优化）
2. **准备投递** → `/auto-apply`（准备投递数据）
3. **提供投递链接**，方便用户直接前往
4. **设置岗位监控** → `/job-monitor`（关注类似岗位的最新发布）

## 筛选参数速查

### campus.searchJobs

| 参数 | 说明 | 示例值 |
|------|------|--------|
| search | 关键词 | "前端工程师" |
| jobType | 岗位类型 | CAMPUS / INTERNSHIP / SOCIAL |
| workMode | 工作模式 | ON_SITE / REMOTE / HYBRID |
| cityIds | 城市 | 逗号分隔 |
| degreeId | 学历要求 | - |
| sortBy | 排序 | newest |

## 依赖的 MCP Tools

## 错误处理

| 错误 | 处理方式 |
|------|---------|
| 搜索无结果 | 建议放宽筛选条件（减少关键词、扩大城市范围） |
| API Token 无效 | 引导重新获取 Token |

## 注意事项

1. **分页处理**：默认每页 12 条，结果较多时提示翻页或缩小范围
2. **类型区分**：展示结果时明确标注社招/校招/实习
3. **投递链接**：展示岗位时一定包含投递链接
4. **组合使用**：搜索 JD → 优化简历 → 投递，形成完整工作流
