# 提示词生成器

基于100+验证模板生成高质量AI提示词。通过自动选择最佳模板、填充占位符，简化用户与AI工具的交互。

## 核心理念

**好的提示词 = 明确意图 + 合适模板 + 具体细节 + 输出格式**

本技能帮助你：

1. 理解你想达成什么目标
2. 选择最合适的模板
3. 填充具体细节
4. 优化输出格式

## 快速开始

### 基础用法

简单描述你想完成的任务：

```
"帮我写一篇关于健身的文章"
"生成一个用于Claude的代码调试提示词"
"我想创建一个Python爬虫"
```

技能会：

1. 识别最佳模板类别
2. 如有需要，询问澄清问题
3. 生成完整的、可直接使用的提示词
4. 提供优化建议

### 高级用法

如需更多控制，可指定模板类别：

```
"用小红书种草笔记的风格，帮我写一篇关于XX产品的推荐"
"生成一个用于Claude的代码调试提示词，使用角色扮演模式"
```

## 模板类别

### 1. 写作类

**适用场景：** 用户想要创建文字内容

| 子类别   | 最佳用途 | 关键要素            |
| ----- | ---- | --------------- |
| 公众号文章 | 长篇内容 | 目标读者、痛点、故事+洞察   |
| 小红书笔记 | 社交媒体 | Emoji、轻松语气、产品亮点 |
| 知乎回答  | 问答格式 | 专业视角、有理有据       |
| 商业文案  | 营销推广 | 产品优势、行动号召       |
| 职场写作  | 工作文档 | 结构清晰、结果导向       |

**模板模式：**

```
【角色】你是一个[角色]，【任务】帮我[做什么]
【背景】[相关背景信息]
【要求】[具体要求]
【格式】[输出格式]
【示例】[参考示例]
```

### 2. 绘图类

**适用场景：** 用户想要生成图片

| 子类别  | 最佳用途 | 关键要素           |
| ---- | ---- | -------------- |
| 人物肖像 | 人像图片 | 年龄、性别、外貌、表情、服装 |
| 场景插画 | 场景图片 | 地点、时间、天气、光线、氛围 |
| 商业设计 | 商业用途 | 产品、背景、光线、角度    |
| 社交媒体 | 社交图片 | 风格、元素、颜色、比例    |
| 艺术创作 | 创意作品 | 主题、风格、颜色、艺术家参考 |

**模板模式：**

```
[主体]，[风格]风格，[细节描述]，[技术参数]
```

**关键参数：**

- `--ar`：宽高比（如 16:9、3:4、1:1）
- `--niji`：动漫风格（Midjourney专用）
- 质量标签：`--quality high --v 6`

## 工作流程

### 第一步：理解用户意图

如有需要，询问澄清问题：

- 主要目标是什么？
- 目标受众是谁？
- 偏好什么风格/语气？
- 有什么特定要求或限制？

### 第二步：选择模板类别

根据用户描述，识别：

1. 主要类别（如写作、绘图）
2. 子类别（如公众号文章、人物肖像）
3. 具体模板模式

### 第三步：生成提示词

填充模板：

- 用户的具体要求
- 合适的角色和背景
- 清晰的输出格式
- 必要的约束条件

### 第四步：优化（可选）

如用户提供反馈：

- 调整语气和风格
- 添加缺失要素
- 根据需要简化或扩展
- 提供替代版本

## 最佳实践

### 写作类提示词

1. **明确角色定位** - 给AI一个清晰的角色和专业领域
2. **具体任务描述** - 明确说明你想要什么
3. **提供背景信息** - 背景帮助AI更好理解
4. **设定输出格式** - 指定你想要的输出结构
5. **给出示例** - 示例能澄清期望

### 绘图类提示词

1. **主体清晰** - 清晰描述主要对象
2. **风格明确** - 指定艺术风格（写实、动漫、油画等）
3. **细节丰富** - 包含光线、颜色、氛围
4. **技术参数** - 添加宽高比（--ar）、质量标签
5. **参考艺术家** - 提及特定艺术家作为风格参考

### 编程类提示词

1. **语言和框架** - 指定编程语言和框架
2. **功能需求** - 清晰描述代码要做什么
3. **输入输出** - 定义预期的输入和输出
4. **约束条件** - 提及任何限制或要求
5. **代码风格** - 指定编码风格偏好

## 常用模式

### 角色-任务-背景模式

```
你是一个[角色]，擅长[能力]。
帮我[做什么]。
[背景信息]
[具体要求]
```

### 问题-解决方案模式

```
我遇到了[问题]。
背景：[背景信息]
已尝试：[做过什么]
期望：[想要的结果]
```

### 对比分析模式

```
对比以下选项：
选项A：[描述]
选项B：[描述]
对比维度：[从哪些方面对比]
```

## 使用技巧

1. **越具体越好** - 提供尽可能多的细节
2. **提供示例** - 展示你想要的样例
3. **迭代优化** - 从简单开始，根据结果逐步完善
4. **保存好用的** - 保存效果好的提示词供将来使用
5. **多轮对话** - 不要期望一次完美，持续迭代

## 问题排查

### 问题：生成的提示词太笼统

**解决方案：** 向用户询问：

- 具体目标受众
- 期望的语气/风格
- 要涵盖的要点
- 输出格式偏好

### 问题：模板不适合用户需求

**解决方案：** 使用万能公式：

```
【角色】你是一个[角色]
擅长[能力]
【任务】帮我[做什么]
【背景】[相关背景信息]
【要求】[具体要求]
【格式】[输出格式]
【示例】[参考示例]
```

### 问题：用户想优化现有提示词

**解决方案：** 使用迭代优化模式：

```
当前提示词：[用户提供的提示词]
存在问题：[分析问题]
优化方向：[改进建议]
优化后提示词：[改进版本]
```
