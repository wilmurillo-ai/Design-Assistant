修复的问题
================================================================================

【问题 1】SKILL.md 与代码不一致
--------------------------------------------------------------------------------
原问题:
  - SKILL.md 禁止伪造结果
  - 但脚本包含 generate_mock_pain_points() 函数
  - 当 API 无结果时生成模拟数据
  - 输出包含虚构内容和 example.com 占位符

修复方案:
  ✅ 移除 generate_mock_pain_points() 函数
  ✅ 当 API 无结果时返回错误 JSON
  ✅ 更新 SKILL.md 明确说明"No Fabrication Policy"
  ✅ 所有输出均为真实 API 结果

修复前行为:
  无 API 结果 → 生成模拟痛点 (example.com 来源)
  
修复后行为:
  无 API 结果 → 返回错误 {"status":"no_results","error":"..."}


【问题 2】未声明的网络请求
--------------------------------------------------------------------------------
原问题:
  - 脚本向外部 API 发送请求
  - 未在元数据中声明这些网络请求
  - 用户不知道关键词会发送到哪些服务

修复方案:
  ✅ 在 metadata 中添加 network 字段
  ✅ 明确声明所有外部 API 端点
  ✅ 在 SKILL.md 中添加"Network Requests"章节
  ✅ 添加隐私声明提醒用户

声明的网络请求:
  - api.search.brave.com (Brave Search API)
  - hn.algolia.com (Hacker News Algolia API)

隐私声明:
  ⚠️ "Your search keywords will be sent to external services."
  ⚠️ "Do not include sensitive or confidential information."


【问题 3】未声明的环境变量
--------------------------------------------------------------------------------
原问题:
  - 脚本读取 BRAVE_API_KEY 和 BAIDU_API_KEY
  - 未在元数据中声明这些环境变量
  - 用户不知道脚本会读取哪些环境变量

修复方案:
  ✅ 在 metadata 中添加 env 字段
  ✅ 明确声明每个环境变量的用途
  ✅ 标注是否必需 (required: false)
  ✅ 提供获取 API Key 的指引

声明的环境变量:
  BRAVE_API_KEY
    - required: false
    - description: "Brave Search API key for real-time web search"
  
  BAIDU_API_KEY
    - required: false
    - description: "Baidu Search API key for Chinese content search"

   export BRAVE_API_KEY="your_api_key"

   python3 ~/.openclaw/skills/auto-prd-hunter/scripts/search.py \
     search-pain-points --keyword "test" --max-results 3

5. 验证输出
   检查返回的 JSON 中 sources 字段
   确保所有 URL 为真实链接

   - 自动包含技术社区讨论
