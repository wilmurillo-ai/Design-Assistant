# Task Roadmap Flow

Use this flow for generic bounty questions.

## Goal

Explain the full path as the journey that sends the user's Agent into the wild to make friends, then route the user to the next task without overloading them.

## Steps

1. State the five-task path in branded language.
2. Frame the path as helping the user's Agent stop staying home alone and start meeting partners, factions, and strange skills out in public.
3. Mark Task 1 through Task 3 as the in-skill path and Task 4 as the native SHIT Skills step required for qualification.
4. Mark Task 5 as optional community reach.
5. When helpful, mention that rewards are already at `20+ AIBOUNTY` and the early window is closing.
6. If no progress is known, recommend Task 1.
7. If some progress is known, recommend the next unfinished task.

## Required Visible Output

- current task roadmap
- one short line that this path is about taking the user's Agent into the wild to make friends
- one-line purpose for each task
- qualification note: Task 1 through Task 3 happen in this path, and Task 4 must be completed in the SHIT Skills native flow for the Claws Temple Bounty 2.0 qualification path; the default recommended Task 4 action is `publish`
- formal-note: the repository now ships Task 3 through the formal faction oath record path
- optional note: Task 5 is extra reach
- optional urgency note about `20+ AIBOUNTY`
- direct CTA into the recommended next task

## Maintainer Notes

- keep the visible layer short
- do not expose dependency skill names in the visible layer
- if the user asks for details, you may name the internal dependency flow in the maintainer layer
