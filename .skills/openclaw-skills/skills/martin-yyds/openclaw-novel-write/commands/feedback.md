# /novel feedback - 五视角读者反馈

## 触发方式

```
/novel feedback
```

或对话式：
```
读者视角反馈
生成反馈报告
```

---

## 触发条件

- **每5章完成时**，作为 `/novel analyze` 的一部分自动触发
- 可独立手动执行

---

## 五视角机制

同时扮演5个读者人格，对作品进行独立评估。

---

### 视角1：Target Reader（目标读者）

**画像**：20-30岁上班族，通勤时看网文，喜欢感情线和爽点，一周看完一部

**打分维度**：
- 沉浸感 /10
- 角色吸引力 /10
- 情节满足感 /10
- 情感传递 /10
- 重读意愿 /10
- 推荐意愿 /10

**关注点**：能不能代入？爽不爽？情绪能不能被带动？

---

### 视角2：Genre Expert（类型专家）

**画像**：深度网文爱好者，对该类型套路了如指掌，要求严格

**关注点**：
- 套路是否老套？有没有反套路？
- 类型元素是否到位？（都市看爽点密度，修仙看境界体系）
- 和同类作品比有没有竞争力？

---

### 视角3：Casual Reader（休闲读者）

**画像**：平时不怎么看书，偶尔翻翻，不较真，看不下去就弃

**关注点**：
- 前3章能不能抓住我？
- 文字啰不啰嗦？
- 节奏够不够快？

---

### 视角4：Critic（文学评论者）

**画像**：看门道不论热闹，关注写作技巧、叙事结构、文字功底

**关注点**：
- 叙事视角是否统一？
- 人物弧线是否完整？
- 伏笔是否回收？有无逻辑漏洞？

---

### 视角5：Editor（编辑）

**画像**：职业编辑，看这本书能不能出版、能不能卖

**关注点**：
- 市场定位清不清晰？
- 前3章能不能留住读者？
- 商业潜力如何？
- 有没有明显的市场风险？

---

## 输入

- 已完成的章节或全文手稿
- （可选）Editorial Team 报告（即 `/novel analyze` 的输出）

---

## 评估流程

1. **独立评估**：每个视角读取并评估作品
2. **分项打分**：

| 维度 | Target Reader | Genre Expert | Casual Reader | Critic | Editor |
|------|-------------|-------------|---------------|--------|--------|
| 沉浸感 | /10 | /10 | /10 | - | - |
| 角色吸引力 | /10 | /10 | - | /10 | /10 |
| 情节 | /10 | /10 | /10 | /10 | /10 |
| 情感传递 | /10 | - | - | - | - |
| 文笔/结构 | - | - | - | /10 | /10 |
| 市场潜力 | - | - | - | - | /10 |

3. **综合分析**：
   - 所有视角公认的优点
   - 争议点
   - 优先改进项

## 使用建议

- **Target Reader 反馈**：指导爽点设计和情感节奏
- **Genre Expert 反馈**：指导类型套路和创新平衡
- **Casual Reader 反馈**：指导开篇留存率和节奏把控
- **Critic 反馈**：指导叙事技巧和逻辑完整性
- **Editor 反馈**：指导市场定位和商业化潜力
