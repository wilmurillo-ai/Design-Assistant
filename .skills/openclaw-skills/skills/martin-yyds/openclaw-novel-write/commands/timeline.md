# /novel timeline - 生成时间线文档

## 触发方式

```
/novel timeline
```

或对话式：
```
生成时间线
创建时间线文档
```

---

## 目标

在创作计划完成后，生成独立的时间线文档，记录故事中所有事件的时间顺序，并与创作计划互相校验，确保一致性。

---

## 前置条件

**必须先完成 `/novel plan`**

---

## 执行步骤

### 1. 加载前置文档

- `memory/constitution.md` - 创作宪法
- `stories/<story-name>/specification.md` - 故事规格
- `stories/<story-name>/clarify-answers.md` - 关键决策澄清
- `stories/<story-name>/creative-plan.md` - 创作计划

### 2. 生成时间线

从创作计划中提取所有事件，按「真实时间顺序」排列，生成 `stories/<story-name>/timeline.md`

### 3. 与创作计划互相校验

#### 3.1 时间线 → 创作计划
- 时间线中的每个事件必须在 creative-plan.md 中能找到对应情节
- 找不到则报错，要求补充到 creative-plan.md

#### 3.2 创作计划 → 时间线
- creative-plan.md 中的关键情节点必须反映在时间线中
- 缺少则报错，要求补充到 timeline.md

#### 3.3 冲突检测
- 时间顺序矛盾（如：事件A发生在事件B之前，但B的章节在A之前）
- 逻辑矛盾（如：角色在事件C时已死亡，但事件D中仍活着）

#### 校验失败处理
```
❌ 时间线与创作计划不一致

问题类型：[冲突类型]
具体问题：[描述]
影响范围：[哪些章节/事件]

请修复后再继续。
```

### 4. 校验通过后保存

- 时间线文档保存到 `stories/<story-name>/timeline.md`
- 同步更新 `spec/tracking/plot-tracker.json` 中的时间线字段

---

## 时间线文档格式

```markdown
# [故事名称] - 时间线

## 时间线概览

| 序号 | 时间点 | 事件 | 章节对应 | 重要程度 |
|------|--------|------|---------|---------|
| 001 | 第1年3月 | 主角出生 | - | ⭐⭐⭐ |
| 002 | 第1年7月 | 主角离家 | 第1章 | ⭐⭐⭐ |
| 003 | 第3年5月 | 遇到恩师 | 第3章 | ⭐⭐ |
| 004 | 第5年2月 | 恩师去世 | 第8章 | ⭐⭐⭐ |

---

## 阶段划分

### 童年期（第1年 - 第5年）
- 事件：001、002
- 主线：建立主角背景

### 成长期（第6年 - 第10年）
- 事件：003、004
- 主线：主角能力觉醒

---

## 关键事件详情

### 001 - 主角出生
- **时间**：第1年3月
- **地点**：某医院
- **涉及人物**：主角、母亲
- **章节对应**：序章
- **备注**：特殊体质出生伴随异象

### 002 - 主角离家
- **时间**：第1年7月
- **地点**：家中
- **涉及人物**：主角、父亲
- **章节对应**：第1章
- **备注**：父亲送别，埋下伏笔
```

## 输出文件

- `stories/<story-name>/timeline.md`

---

## 与创作计划的关系

```
creative-plan.md  ──互相校验──►  timeline.md
       │                              │
       └── 情节节点 ──────────────────┘
       └── 时间顺序 ─────────────────┘
```

**两者必须一致，不一致时阻断流程。**

### 时间线变更重校验

如果在后续创作过程中需要修改 `timeline.md`（如新增事件、调整时间点），**必须重新进行互相校验**：

1. 修改 timeline.md 后
2. 对照 creative-plan.md 检查是否一致
3. 如有不一致 → 同步修正 creative-plan.md 或 timeline.md
4. track --check 时也会检测时间线与章节内容的一致性
