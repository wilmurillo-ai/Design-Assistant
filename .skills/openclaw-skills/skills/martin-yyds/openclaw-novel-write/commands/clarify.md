# /novel clarify - 澄清关键决策

## 触发方式

```
/novel clarify
```

或对话式：
```
澄清关键决策
回答关于故事的问题
```

---

## 目标

澄清故事中的关键模糊点，确保创作者对故事的关键决策有清晰一致的理解。

---

## 执行步骤

### 1. 读取现有规格

读取 `stories/<story-name>/specification.md` 和 `memory/constitution.md`

### 2. 识别模糊点

从规格中识别需要澄清的关键问题，通常包括：

#### 角色相关
- 主角的行动逻辑是什么？
- 角色如何在关键情节点做出选择？

#### 情节相关
- 核心冲突如何升级？
- 关键转折点的触发条件？
- 结局的走向？

#### 世界观相关
- 金手指的边界在哪里？
- 特殊设定的运作规则？

#### 读者体验相关
- 爽点/甜点的频率和强度？
- 虐点/冲突的承受阈值？

---

### 3. 提问并记录

提出 5 个最关键的问题，引导用户思考：

```
在开始详细规划之前，我需要澄清以下关键决策：

1. [问题1]
2. [问题2]
3. [问题3]
4. [问题4]
5. [问题5]

请回答这些问题，帮助我更准确地制定创作计划。
```

---

### 4. 更新规格文档

将澄清的决策追加到 `stories/<story-name>/specification.md`

```markdown
## 关键决策记录

### Q: [问题]
A: [答案]

### Q: [问题]
A: [答案]
```

---

## 输出

更新文件：`stories/<story-name>/specification.md`

---

## 提示下一步

澄清完成后，建议执行：
```
/novel plan - 制定创作计划
```
