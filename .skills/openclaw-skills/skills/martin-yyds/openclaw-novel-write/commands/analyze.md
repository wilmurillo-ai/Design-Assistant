# /novel analyze - 质量分析

## 触发方式

```
/novel analyze
```

或对话式：
```
分析写作质量
质量检查
```

---

## 触发条件

- **每5章完成时提醒用户触发**（章节编号 % 5 == 0）
- 手动执行检查

---

## 分析维度

### 1. 一致性检查

- **角色一致性**：性格、行为、语言是否一致
- **世界观一致性**：设定是否自洽
- **时间线一致性**：事件顺序是否正确

### 2. 节奏分析

- 高潮点分布是否合理
- 章节间的节奏变化
- 是否有拖沓或过赶的部分

### 3. 视角检查

- 视角是否统一
- 是否有跳视角的情况
- 内心独白是否过多

### 4. 对话质量

- 对话是否自然
- 是否推动情节
- 是否有说明式对话

### 5. 具象化程度

- 是否过度抽象描写
- 是否有"AI腔"痕迹
- 时间/人物/数量是否具体

---
