# /novel tasks - 分解任务清单

## 触发方式

```
/novel tasks
```

或对话式：
```
分解任务清单
生成写作任务
```

---

## 目标

将创作计划分解为具体的、可执行的任务清单，**按卷分拆，细化到每一章节，含字数要求**。

---

## 输出结构

```
stories/<story-name>/
├── tasks.md              # 总纲（索引 + 统计）
├── tasks-volume-1.md     # 第1卷详细任务
├── tasks-volume-2.md     # 第2卷详细任务
└── ...
```

---

## 执行步骤

### 1. 加载计划文档

读取：
- `stories/<story-name>/specification.md` - 章节规划
- `stories/<story-name>/creative-plan.md` - 创作计划

### 2. 生成各卷任务文件

按卷分别生成，输出到 `stories/<story-name>/tasks-volume-{N}.md`

### 3. 生成总纲

输出到 `stories/<story-name>/tasks.md`

---

## tasks.md 总纲模板

```markdown
# [故事名称] - 任务清单

## 卷纲

| 卷 | 章节范围 | 核心任务 | 状态 |
|----|---------|---------|------|
| 第1卷 | 第1-10章 | 建立主角、冲突、世界观 | ⏳ 进行中 |
| 第2卷 | 第11-25章 | 深化冲突、角色成长 | 🔲 待开始 |
| 第3卷 | 第26-30章 | 核心冲突爆发、收束线索 | 🔲 待开始 |

## 任务统计

| 项目 | 数值 |
|------|------|
| 总章节数 | 30 |
| 总卷数 | 3 |
| 预估总字数 | 90,000-105,000字 |
| 预估完成时间 | 6-8周（每日1章） |

### 里程碑

- 第10章完成 → 第一卷完成
- 第25章完成 → 第二卷完成
- 第30章完成 → 全书完成

## 当前进度

| 卷 | 完成 | 进行中 | 待开始 |
|----|------|--------|--------|
| 第1卷 | 0/10 | 0 | 10 |
| 第2卷 | 0/15 | 0 | 15 |
| 第3卷 | 0/5 | 0 | 5 |

## 详细任务

- [第1卷任务清单](./tasks-volume-1.md)
- [第2卷任务清单](./tasks-volume-2.md)
- [第3卷任务清单](./tasks-volume-3.md)
```

---

## tasks-volume-N.md 卷任务模板

```markdown
# [故事名称] - 第{N}卷任务清单

## 卷信息

- **卷名**: {卷名}
- **章节范围**: 第{起始章}-第{结束章}章（共{章数}章）
- **核心任务**: {卷的核心目标}
- **预估字数**: {卷的总字数范围}
- **卷状态**: 🔲 待开始

---

## 任务列表

### T{编号} - 第{章号}章：{章节标题}

- **状态**: pending
- **字数要求**: {最小字数}-{最大字数}字
- **核心任务**: {本章的主要任务}
- **关键情节**: {具体情节点}
- **涉及线索**:
  - PL-{编号}({线索名}) ⭐⭐⭐ 主推进
  - PL-{编号}({线索名}) ⭐ 背景
- **交汇点**: {如适用} X-{编号}({交汇点描述})
- **伏笔埋设/揭晓**: {如适用} F-{编号}埋设 / F-{编号}揭晓
- **必须包含**: {关键元素列表}
- **章末钩子**: {悬念设置}
- **依赖**: {依赖任务}（无依赖则写"无"）
- **输出**: `content/volume{N}/chapter-{章号:02d}.md`

---
```

## 提示下一步

任务清单创建完成后，可以开始执行：
```
/novel write - 开始第一章写作
```

**注意**：必须先完成前5步（constitution → specify → clarify → plan → tasks）才能开始写作。
