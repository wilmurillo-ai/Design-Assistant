# /novel plan - 制定创作计划

## 触发方式

```
/novel plan [技术偏好]
```

或对话式：
```
制定创作计划
生成章节大纲
```

---

## 目标

将"要创造什么"（规格）转化为"如何创造"（计划），包括章节架构、线索分布、节奏设计。

---

## 执行步骤

### 1. 加载前置文档

- `memory/constitution.md` - 创作宪法
- `stories/<story-name>/specification.md` - 故事规格
- `stories/<story-name>/clarify-answers.md` - 澄清记录（如有）

### 2. 检查黄金开篇法则（如适用）

如果总字数 < 10000字或规划第1-3章：
```bash
test -f spec/presets/golden-opening.md && echo "found" || echo "not-found"
```
如存在，读取并应用五大黄金法则。

### 3. 设计章节架构

#### 整体结构
- **开头**（第1-3章）：建立世界观、角色、主要冲突
- **发展**（中段）：冲突升级、角色成长、支线展开
- **高潮**（后段）：核心冲突爆发、关键抉择
- **结尾**（最后几章）：收束所有线索、满足读者期待

#### 卷结构（如多卷）

```markdown
## 第1卷：破冰（约10章）
- 核心任务：建立主角、冲突、世界观
- 关键节点：第3章（第一个高潮）、第5章（关系确立）

## 第2卷：升温（约15章）
- 核心任务：深化冲突、角色成长
- 关键节点：第8章（重大转折）、第12章（关系突破）
```

### 4. 线索设计

识别主要线索和伏笔：

| 线索ID | 线索名称 | 类型 | 主线/支线 | 埋设章节 | 揭晓章节 |
|--------|---------|------|----------|---------|---------|
| PL-01 | 身份秘密 | 核心 | 主线 | 第1章 | 第20章 |
| PL-02 | 家族恩怨 | 背景 | 支线 | 第3章 | 第15章 |

### 5. 节奏设计

- 每X章一个小高潮
- 每Y章一个中等高潮
- 关键转折点安排

### 6. 生成创作计划

输出到 `stories/<story-name>/creative-plan.md`

```markdown
# [故事名称] - 创作计划

## 整体结构

### 卷划分
[各卷的任务和节奏]

### 章节概览
[总章节数和每卷比例]

## 章节架构

### 第1卷：[卷名]

| 章节 | 标题 | 核心任务 | 关键情节 | 涉及线索 |
|------|------|---------|---------|---------|
| 1 | 意外相逢 | 主角出场 | 建立冲突 | PL-01 ⭐⭐⭐ |
| 2 | 暗流涌动 | 深化冲突 | 关系建立 | PL-01 ⭐⭐, PL-02 ⭐ |

## 线索分布

[各线索的埋设和揭晓计划]

## 伏笔记录

| 伏笔ID | 内容 | 埋设章节 | 揭晓章节 |
|-------|------|---------|---------|
| F-01 | 身份线索 | 第2章 | 第20章 |

## 节奏规划

[高潮点分布、爽点频率]

## 技术方案

[技术偏好、实现细节]
```
