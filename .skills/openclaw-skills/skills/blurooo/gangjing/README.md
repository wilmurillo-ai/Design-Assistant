# 杠精（gangjing）—— 根治 AI 幻觉，始终保持清醒

> 你写代码的时候有没有一个人在旁边说"你确定？"
> 没有的话，现在有了。

---

## 【亮点】

**一句话：你说"没问题"，杠精说"真的吗"——然后用六把刀和真实代码崩溃证明给你看。**

![杠精主视觉](https://raw.githubusercontent.com/agent-dance/harness-engineering/main/skills/gangjing/assets/gangjing-hero.png)

---

## 【介绍】

### 你一定经历过这些场景：

- 周五下午："这个测试先不写了，赶紧上线。" → 周一凌晨三点被电话叫醒
- 技术评审会："大家有意见吗？" → 全场沉默 → "那就这样定了" → 三个月后全部重写
- 代码写完了："肯定没bug。" → 第一个用户输入了一个 `NaN`，系统原地爆炸

**痛点很简单：你的方案在被真正挑战之前，你永远不知道它有多脆弱。**

同事不好意思说、Code Review 太温柔、自己又看不出自己的盲区——
你需要一个**不留情面但真正有用的杠精**。

### 杠精能干什么？

![没有杠精 vs 有杠精](https://raw.githubusercontent.com/agent-dance/harness-engineering/main/skills/gangjing/assets/gangjing-before-after.png)

**两层全覆盖：杠你的决策，也锤你的代码。**

| 你说 | 杠精干什么 |
|------|-----------|
| "我想做个实时游戏平台" | "'实时'是多实时？50ms？100ms？'平台'是做匹配还是就搓个房间？你这八个字我能杠出六个没想清楚的问题。" |
| "我决定用 MongoDB 存订单" | "MongoDB存订单！怎么不用Excel呢？也是NoSQL，还自带透视表。" → 然后认真分析事务、运维、备选方案 |
| "不写测试，赶紧上线" | "不写测试？那不如把lint也关了，编译也省了，直接祈祷部署。" → 然后给出最小可行测试方案 |
| "代码绝对没bug" | 二话不说跑攻击引擎 → "15次攻击，7次崩溃，评分20/100。你说的'绝对'呢？" |

### 融合四大经典杠精流派

不是学术论文式的"我有一些concerns"，而是**真正的杠精味道**：

| 流派 | 手法 | 经典台词 |
|------|------|----------|
| 🎭 **无中生有** | 挖你没提到的问题 | "你说了用Redis，但你没说Redis挂了怎么办。没提到 = 没想到 = 最后出事。" |
| 📢 **极端夸大** | 小缺陷放大到荒谬 | "先不写测试？那不如连类型检查也关了，自由！解放！直到线上炸了。" |
| 🎯 **以偏概全** | 一个反例击穿你的"一定" | "Shopify从微服务退回单体了。你比Shopify更需要微服务？" |
| ✂️ **断章取义** | 抓你每个模糊措辞 | "'性能应该够用'？'应该'是量化过还是猜的？" |

**关键区别：杠精不是喷子。** 每条杠都带论据，杠完都给解法。
光说"有问题"不给方案的那叫喷子——杠精看不起喷子。

## 【使用建议】

### 什么时候用？

**简单原则：做决策之前，先让杠精过一遍。**

| 场景 | 强度 | 效果 |
|------|------|------|
| "我想做个XX" | 🟡 正经杠 | 帮你在动手之前把需求杠清楚，省得做到一半发现方向错了 |
| 选技术栈 / 数据库 / 框架 | 🟡 正经杠 | 找出你没考虑过的备选方案和隐藏陷阱 |
| 定对外 API / 数据模型 | 🔴 往死里杠 | 不可逆决策，一旦上线回滚成本极高，必须严格审查 |
| "这个绝对没问题" | 🔴 自动拉满 | 第十人规则：越是全员共识，越需要一个反对票 |
| 改个变量名 | 🟢 微杠 | 阴阳怪气一句："data和payload你都分不清？算了你开心就好。" |

![杠精强度计](https://raw.githubusercontent.com/agent-dance/harness-engineering/main/skills/gangjing/assets/gangjing-intensity.png)

### 怎么用？

**安装后开箱即用。** 你只需要正常说话，杠精就会自动介入：

```
你: 我想做个电商后台系统
杠精: "电商后台"？卖什么？日单量多少？"后台"是管理端还是整个服务端？
      你这一句话我能拆出五个没定义的词……
```

**你可以随时控制它：**
- 说 **"使劲杠"** → 火力全开
- 说 **"轻点杠"** → 降到微杠
- 说 **"够了"** → 停止并输出裁决书

### 六把刀，刀刀见血

![六把刀](https://raw.githubusercontent.com/agent-dance/harness-engineering/main/skills/gangjing/assets/gangjing-six-knives.png)

| 刀 | 杠什么 | 直觉理解 |
|----|--------|----------|
| 🔪 假设刺穿 | 你方案成立的前提条件 | "谁告诉你网络一定可靠的？" |
| 🗡️ 备选轰炸 | 你没看过的其他方案 | "你连PostgreSQL都没试就说MongoDB最好？" |
| ⚔️ 规模预言 | 10倍增长后的崩溃点 | "现在能跑，10万用户的时候呢？" |
| 🪓 不可逆评估 | 回滚成本 | "选数据库就像结婚，离婚很贵的。" |
| 🔨 Pre-mortem | 假设已经失败了，为什么 | "6个月后项目黄了，死因是什么？" |
| 💀 代码实锤 | 跑真实攻击，用崩溃说话 | "你说没bug？15次攻击7次崩溃。" |

## 【创作思路】

### 灵感来源：把互联网杠精变成生产力

互联网上的杠精人人喊打——但仔细想想，**杠精的核心能力恰恰是技术审查最需要的**：

- "无中生有"→ 发现别人没提到的隐患
- "极端夸大"→ 把风险的趋势线画到终点
- "以偏概全"→ 用一个反例击破过度自信
- "断章取义"→ 逼迫模糊的需求变得精确

**把令人讨厌的性格特质，重新定向到有价值的场景——这就是杠精技能的核心设计哲学。**

### 技术巧思 1: 四大流派映射为技术审查手法

不是简单地让 AI 说"我有不同意见"——而是给它**四种具体的攻击范式**，
每种都有技术场景的经典台词模板。这样 AI 不会泛泛而谈，
而是像真正的杠精一样精准找茬。

### 技术巧思 2: 强度自动校准 + 第十人规则

不是所有决策都值得全力杠。技能内置了一个**不可逆性 × 影响范围**的二维矩阵，
自动判定用微杠、正经杠还是往死里杠。

更精妙的是**第十人规则**——当用户说出"绝对没问题"、"大家都同意了"这类
过度自信的表述时，**自动升级到最高强度**。因为历史告诉我们，
灾难发生前最常听到的就是这两句话。

### 技术巧思 3: 嘴炮 + 实锤双引擎

大多数 AI 审查工具只会说"我觉得可能有问题"——是观点，不是证据。

杠精的第六把刀"代码实锤"内置了完整的攻击引擎（harness.py / harness.js），
能自动生成恶意输入、在进程隔离环境下执行攻击、
**用真实的崩溃和漏洞数据**作为杠的论据。

不是"我觉得你的代码不安全"，而是"你的代码在 NaN 输入下崩溃了，这是证据，
评分 20/100，等级 F。"——**用事实杠，杠到你无话可说。**

### 技术巧思 4: Step 0 — 模糊意图是最大的靶子

在审查方案之前，先审查意图本身。当用户说"我想做个XX"的时候，
大多数 AI 会直接开始帮忙做——杠精不会。

杠精会先把"我想做个XX"里的每个模糊词拆开：
"做"是做MVP还是做完整产品？"XX"的核心功能定义了吗？目标用户是谁？

**模糊的意图不值得讨论方案。先把需求杠清楚，再杠方案——顺序不能反。**

这个设计让杠精成为了需求阶段最好的合作伙伴——不是帮你做，
而是先确保你知道自己在做什么。
