# Quality Checklist

Every generated asset should pass these checks before use.

## Structure

- clear point early
- one primary thesis
- one clear CTA or closing principle

## Specificity

- specific examples or mechanisms present
- no vague transformation claims

## Evidence

- claims use approved proof or labeled assumptions
- no invented certainty

## Voice

- matches `voice.md`
- avoids banned phrasing

## Channel Fit

- matches `channels.md`
- fits expected intensity and structure

## Final Gate

If any item fails, rewrite instead of shipping.
