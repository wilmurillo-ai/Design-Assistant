# Promotion Rules

## Rule 1: Human approval required

No candidate update becomes canon without review.

## Rule 2: Preserve provenance

Every promoted claim should be traceable to a source or explicitly labeled as a hypothesis.

## Rule 3: Promote into the right file

- objections go to `audience.md`
- proof goes to `proof.md`
- tone corrections go to `style-corrections.md`
- messaging changes go to `messaging.md`

## Rule 4: Reject rawness

Do not promote:

- raw transcript fragments without normalization
- emotional venting
- speculative claims presented as fact
- duplicate updates

## Rule 5: Canon is concise

Canon should capture stable knowledge, not every observation.
