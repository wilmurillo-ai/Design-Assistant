# Research Prompt: Mental Category Positioning Analysis

**To be executed by:** Perplexity, Gemini, or other research-capable LLM
**Estimated time:** 30-45 minutes of research
**Output format:** Markdown document with tables and visual matrix

---

## Your Task

You are a positioning strategist. Your client is struggling to explain what they are because prospects don't have a mental model for "AI agents run your growth operation but you approve everything."

**Your job:** Map the existing mental categories B2B buyers use for "who helps with growth/marketing," identify where competitors sit, and recommend a positioning strategy (claim existing category OR create new one).

---

## Background Context

**What this type of service actually is:**
- AI agents that run outbound, content, pipeline building
- Delivered as managed service (not SaaS you run yourself)
- Transparent approval mode = client approves/rejects all work before it goes live
- Price: $3-15K/mo
- Target: B2B service companies, $500K-$5M revenue

**The problem:**
When prospects hear the pitch, they mentally bucket it as either:
- "AI sales tool" (like Clay, Apollo) - "I have to run this myself"
- "Growth agency" (traditional) - "Black box that wastes money"
- "AI consultant" - "They'll build me something then leave"

None of these are accurate. We need to understand what categories exist and where to position.

---

## Research Questions

1. What are the discrete mental categories B2B buyers use for growth/marketing help?
2. What are the default assumptions for each category (price, delivery, results)?
3. Where do competitors position themselves?
4. What white space exists (categories that should exist but don't)?
5. How have other companies successfully created new categories?

---

## Research Method

### Part 1: Category Mapping from Competitor Self-Description

**Find 20+ companies across these categories:**
- **AI Sales Tools:** Clay, Apollo, Instantly, 11x, Seamless, Lemlist, Woodpecker
- **Traditional Agencies:** Search "B2B growth agency", "demand gen agency" (pick 5-7)
- **AI Agencies/Services:** Search "AI marketing agency", "AI SDR service" (pick 5-7)
- **Consultants:** Search "RevOps consultant", "growth consultant" (pick 3-5)

**For each, document:**
1. How do they describe themselves? (homepage H1 + first sentence)
2. What category label do they claim? ("platform", "agency", "service", "partner", "system")
3. Delivery model: Self-serve SaaS / Managed service / Done-for-you / Consulting
4. Price tier: <$1K/mo / $1-10K/mo / >$10K/mo
5. Transparency: Black box (no visibility) / Glass box (show process)
6. Proof metric: Logos / Lead volume / Meeting count / Revenue impact / ROI %

**Compile into a table:**

| Company | Self-Description | Category Label | Delivery | Price | Transparency | Proof Metric |
|---------|------------------|----------------|----------|-------|--------------|--------------|
| Clay | "Data enrichment platform" | Platform | Self-serve | <$1K | Black box | Lead volume |
| 11x.ai | "Digital workers for GTM teams" | Service | Managed | $1-10K | Black box | Meeting count |
| etc. | | | | | | |

### Part 2: Category Positioning Matrix

Create a 2x2 matrix plotting all competitors:

**X-axis:** Delivery Model
- Left: Self-serve (you run it)
- Right: Done-for-you (they run it)

**Y-axis:** Price
- Bottom: Low (<$1K/mo)
- Top: High (>$10K/mo)

**For each competitor, plot them on this matrix and annotate:**
- Company name
- Category they claim
- Transparency level (icon or color code)

**Identify:**
- Where is a transparent AI managed service positioned vs where prospects think it is?
- What's the white space (empty quadrants or underserved areas)?
- What categories exist vs what categories are missing?

### Part 3: Blue Ocean Category Creation Case Studies

**Research how these companies created new categories:**

1. **HubSpot** - "Inbound Marketing"
   - Old category they escaped: Outbound sales/cold calling
   - New category they created: Inbound marketing
   - Core distinction: Attract vs interrupt
   - How they educated: Blog, certification, "Inbound methodology"

2. **Salesforce** - "No Software"
   - Old category: Installed CRM software
   - New category: Cloud CRM / SaaS
   - Core distinction: No installation, pay-as-you-go
   - How they educated: "No Software" billboards, demos

3. **Slack** - "Where Work Happens"
   - Old category: Email + chat tools
   - New category: Team collaboration platform
   - Core distinction: Channels + integrations, not just messaging
   - How they educated: Bottom-up adoption, free tier

4. **Stripe** - "Payments for Developers"
   - Old category: Merchant accounts
   - New category: API-first payments
   - Core distinction: 7 lines of code vs weeks of setup
   - How they educated: Developer docs, drop-in code

**For each, document:**

| Company | Old Category (Escaped) | New Category (Created) | Core 1-Sentence Distinction | Education Strategy |
|---------|----------------------|------------------------|----------------------------|-------------------|
| HubSpot | Outbound sales | Inbound marketing | Attract customers vs interrupt them | Blog + certification |
| etc. | | | | |

### Part 4: Category Definition Options

Based on the white space identified and category creation patterns, propose **3-5 category options**:

| Category Name | 1-Sentence Definition | Why Different from Existing | Risk/Downside | Education Needed |
|---------------|----------------------|---------------------------|---------------|------------------|
| "Transparent AI Service" | AI does the work, you approve everything before it goes live | NOT black box agency, NOT self-serve tool | "Transparent" might not resonate | Show approval workflow UI |
| "AI Service Bureau" | Your growth team is AI agents with human oversight | NOT traditional agency, NOT tool you run | "Bureau" sounds old-fashioned | Explain zero-employee model |
| "Supervised Automation" | Automation runs, but nothing executes without your approval | NOT autonomous AI, NOT manual work | Might sound too cautious | Demo approval workflow |
| etc. | | | | |

## Expected Output

Deliver a markdown document with these sections:

### 1. Competitor Category Table
The full table of 20+ competitors with their positioning (from Part 1).

### 2. Category Positioning Matrix
A visual representation (can be ASCII art or describe it clearly enough for someone to sketch it):

```

```

Annotate the white space and where to position.

### 3. Blue Ocean Category Case Studies
The table from Part 3 showing how 4+ companies created categories.

### 4. Category Definition Options
The table from Part 4 with 3-5 options.

### 5. Recommended Positioning
Pick ONE category option and write:

**Category Name:** [chosen name]

**Positioning Statement (1 sentence):**
> [Service] is a [category] that [core distinction].

**Expanded Explanation (2-3 sentences):**
> [What it means, why it's different]

**Comparison (2 sentences):**
> Unlike [old category A], we [distinction A]. Unlike [old category B], we [distinction B].

**Proof to Show:**
> [What evidence makes this believable - approval workflow UI, case study, etc.]

### 6. Why This Category Wins
3-5 bullet points:
- Why this category is ownable (competitors can't claim it)
- Why this category is valuable (buyers want it)
- Why this category is clear (easy to understand)
- What risk it mitigates (doesn't trigger existing objections)

---

## Research Tips

- Use Perplexity's search for real-time competitor homepages
- Look for patterns across 20+ companies, not just a few
- Pay attention to self-description language (exact words matter)
- White space = opportunity (where no one is positioned)
- Best category names are descriptive, not clever

A single markdown file with all sections, tables, and the final positioning recommendation.
